# 核心概念 4: 懒加载模式

> 理解如何避免内存溢出

---

## 核心问题

为什么 LangChain 的 DocumentLoader 强调懒加载(lazy loading)?它如何解决大文件处理的内存问题?

---

## 什么是懒加载?

### 定义

**懒加载**: 按需加载数据,而不是一次性加载所有数据到内存。

**对比**:

```python
# 急切加载(Eager Loading) - 一次性加载
def load_all():
    data = []
    for item in data_source:
        data.append(process(item))  # 全部加载到内存
    return data

# 懒加载(Lazy Loading) - 按需加载
def lazy_load():
    for item in data_source:
        yield process(item)  # 逐个生成,用完即释放
```

---

## 为什么需要懒加载?

### 问题 1: 内存溢出

**场景**: 加载 1GB 的 PDF 文档集合

```python
# 急切加载 - 内存溢出
loader = PyPDFLoader("large_file.pdf")
docs = loader.load()  # 一次性加载所有页面

# 假设 PDF 有 1000 页,每页 1MB
# 内存占用: 1000 * 1MB = 1GB
# 如果服务器只有 512MB 内存 → 内存溢出!
```

**懒加载解决**:

```python
# 懒加载 - 节省内存
loader = PyPDFLoader("large_file.pdf")
for doc in loader.lazy_load():  # 逐页加载
    process(doc)  # 处理完立即释放内存

# 内存占用: 1 * 1MB = 1MB (只保留当前页)
# 节省内存: 99.9%
```

[来源: reference/fetch_4_297d49_07.md | Reddit 内存优化讨论]

### 问题 2: 启动延迟

**场景**: 用户等待文档加载

```python
# 急切加载 - 用户等待
start = time.time()
docs = loader.load()  # 等待所有文档加载完成
print(f"等待时间: {time.time() - start}秒")  # 30秒
process(docs[0])  # 用户只需要第一个文档

# 懒加载 - 立即响应
start = time.time()
for doc in loader.lazy_load():
    print(f"首个文档: {time.time() - start}秒")  # 0.1秒
    process(doc)
    break  # 只处理第一个
```

### 问题 3: 资源浪费

**场景**: 提前终止处理

```python
# 急切加载 - 浪费资源
docs = loader.load()  # 加载 1000 个文档
for i, doc in enumerate(docs):
    if i >= 10:  # 只需要前 10 个
        break
    process(doc)
# 浪费: 加载了 990 个不需要的文档

# 懒加载 - 按需加载
count = 0
for doc in loader.lazy_load():
    process(doc)
    count += 1
    if count >= 10:  # 只加载前 10 个
        break
# 节省: 只加载了需要的 10 个文档
```

## 懒加载的实现

### Python 生成器

**基础概念**:

```python
# 普通函数 - 返回列表
def get_numbers():
    result = []
    for i in range(1000000):
        result.append(i)  # 全部存储在内存
    return result

# 生成器函数 - 逐个生成
def lazy_numbers():
    for i in range(1000000):
        yield i  # 逐个生成,用完即释放

# 使用对比
numbers = get_numbers()  # 内存占用: ~40MB
print(numbers[0])

for num in lazy_numbers():  # 内存占用: ~40 bytes
    print(num)
    break
```

### DocumentLoader 中的实现

```python
class BaseLoader(ABC):
    def lazy_load(self) -> Iterator[Document]:
        """懒加载实现"""
        # 使用生成器逐个返回 Document
        for item in self.data_source:
            # 处理单个项目
            doc = self.process_item(item)
            # 立即返回,不等待后续项目
            yield doc

    def load(self) -> list[Document]:
        """急切加载实现"""
        # 内部调用 lazy_load,转换为列表
        return list(self.lazy_load())
```

## 实际应用

### 场景 1: 大文件处理

```python
from langchain_community.document_loaders import PyPDFLoader

# 处理 1GB PDF 文件
loader = PyPDFLoader("large_report.pdf")

# 懒加载 - 逐页处理
for page_num, doc in enumerate(loader.lazy_load()):
    print(f"处理第 {page_num} 页")

    # 向量化
    embedding = embed(doc.page_content)

    # 存储
    vector_store.add(embedding)

    # 处理完立即释放内存
    del doc, embedding
```

### 场景 2: 流式响应

```python
from langchain_community.document_loaders import DirectoryLoader

# 流式加载目录中的所有文档
loader = DirectoryLoader("docs/", glob="**/*.pdf")

# 实时显示进度
for i, doc in enumerate(loader.lazy_load()):
    print(f"已加载 {i+1} 个文档")
    process(doc)

    # 可以随时终止
    if should_stop():
        break
```

### 场景 3: 批量处理

```python
# 批量处理 - 结合懒加载
def batch_process(loader, batch_size=10):
    batch = []
    for doc in loader.lazy_load():
        batch.append(doc)

        if len(batch) >= batch_size:
            # 批量处理
            process_batch(batch)
            # 清空批次,释放内存
            batch = []

    # 处理剩余文档
    if batch:
        process_batch(batch)

loader = DirectoryLoader("docs/")
batch_process(loader, batch_size=10)
```

## 类比理解

### 前端开发类比

**懒加载** 就像前端的 **虚拟滚动**:

```javascript
// 急切加载 - 渲染所有数据
const allItems = data.map(item => <Item key={item.id} data={item} />)
// 内存占用: 10000 个组件

// 懒加载 - 只渲染可见部分
<VirtualScroll>
  {visibleItems.map(item => <Item key={item.id} data={item} />)}
</VirtualScroll>
// 内存占用: 10 个组件
```

### 日常生活类比

**懒加载** 就像 **按需点餐**:

**急切加载** = 自助餐:
- 一次性拿所有菜
- 占用大桌子
- 吃不完浪费

**懒加载** = 点餐:
- 吃完一道再点下一道
- 只占用当前盘子
- 不浪费

## 常见误区

### ❌ 误区 1: 懒加载总是更慢

**错误**: 认为懒加载比急切加载慢

**正确**: 懒加载的总时间可能相同,但:
- 首次响应更快
- 内存占用更小
- 可以提前终止

### ❌ 误区 2: 懒加载不能获取数量

**错误**: 认为懒加载无法获取文档数量

**正确**: 可以先遍历计数,再处理

```python
# 方法 1: 转换为列表
docs = list(loader.lazy_load())
print(f"文档数量: {len(docs)}")

# 方法 2: 先计数
count = sum(1 for _ in loader.lazy_load())
print(f"文档数量: {count}")

# 方法 3: 边处理边计数
count = 0
for doc in loader.lazy_load():
    count += 1
    process(doc)
print(f"处理了 {count} 个文档")
```

### ❌ 误区 3: 懒加载不能并行

**错误**: 认为懒加载无法并行处理

**正确**: 可以结合多线程/多进程

```python
from concurrent.futures import ThreadPoolExecutor

# 懒加载 + 并行处理
def parallel_process(loader, max_workers=4):
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        # 懒加载生成任务
        futures = []
        for doc in loader.lazy_load():
            future = executor.submit(process, doc)
            futures.append(future)

        # 等待完成
        for future in futures:
            future.result()
```

## 最佳实践

### 1. 优先使用懒加载

```python
# ✓ 推荐: 默认使用懒加载
for doc in loader.lazy_load():
    process(doc)

# ✗ 避免: 除非确定文件很小
docs = loader.load()
```

### 2. 批量处理优化

```python
# 结合懒加载和批量处理
def batch_lazy_load(loader, batch_size=10):
    batch = []
    for doc in loader.lazy_load():
        batch.append(doc)
        if len(batch) >= batch_size:
            yield batch
            batch = []
    if batch:
        yield batch

# 使用
for batch in batch_lazy_load(loader):
    process_batch(batch)  # 批量处理提升效率
```

### 3. 内存监控

```python
import psutil

# 监控内存使用
process = psutil.Process()
for doc in loader.lazy_load():
    process_doc(doc)

    # 检查内存
    memory_mb = process.memory_info().rss / 1024 / 1024
    if memory_mb > 1000:  # 超过 1GB
        print("警告: 内存占用过高")
```

## 总结

### 核心原理

1. **按需加载**: 只加载当前需要的数据
2. **生成器模式**: 使用 Python 生成器实现
3. **内存优化**: 处理完立即释放内存

### 设计优势

| 优势 | 说明 |
|------|------|
| 内存效率 | 节省 90%+ 内存 |
| 响应速度 | 立即开始处理 |
| 灵活控制 | 可随时终止 |
| 无限数据 | 支持无限大数据集 |
