## 核心要素拆解

### 1. 是什么？

**对文档序列进行清洗、过滤、增强的统一接口**

- 不是单一的内容转换工具
- 不是只处理一个文档的函数
- 是一个统一的抽象，涵盖四种操作：清洗、过滤、增强、重排序

```python
from langchain_core.documents.transformers import BaseDocumentTransformer
from langchain_core.documents import Document
from typing import Sequence, Any
from abc import abstractmethod

class BaseDocumentTransformer:
    """所有文档转换器的抽象基类"""

    @abstractmethod
    def transform_documents(
        self, documents: Sequence[Document], **kwargs: Any
    ) -> Sequence[Document]:
        """核心方法：文档序列 → 文档序列"""

    async def atransform_documents(
        self, documents: Sequence[Document], **kwargs: Any
    ) -> Sequence[Document]:
        """异步版本：默认包装同步方法"""
```

[来源: reference/source_document_transformer_01.md | LangChain 源码分析]

**关键特征：**
- 输入输出类型一致：`Sequence[Document]` → `Sequence[Document]`
- 纯函数式设计：不修改原始文档，返回新的文档序列
- 接口极简：只有两个方法

### 2. 做什么？

**四种核心操作：**

| 操作 | 说明 | 文档数量 | 内容变化 | 元数据变化 | 代表转换器 |
|------|------|---------|---------|-----------|-----------|
| **清洗** | 去除噪声和无关内容 | 不变 | ✅ 变 | 不变 | Html2TextTransformer |
| **过滤** | 去除冗余或低质量文档 | ⬇️ 减少 | 不变 | 不变 | EmbeddingsRedundantFilter |
| **增强** | 添加结构化元数据 | 不变 | 不变 | ✅ 变 | OpenAIMetadataTagger |
| **重排序** | 优化文档顺序 | 不变 | 不变 | 不变 | LongContextReorder |

```python
from langchain_core.documents import Document

# 原始文档
docs = [
    Document(page_content="<h1>RAG 入门</h1><p>RAG 是...</p>", metadata={"source": "web"}),
    Document(page_content="RAG 是检索增强生成的缩写", metadata={"source": "web"}),
    Document(page_content="RAG 是 Retrieval Augmented Generation", metadata={"source": "web"}),
]

# 清洗：去除 HTML 标签
from langchain_community.document_transformers import Html2TextTransformer
clean_docs = Html2TextTransformer().transform_documents(docs)
# 结果：HTML 标签被移除，保留纯文本

# 过滤：去除语义重复
from langchain_community.document_transformers import EmbeddingsRedundantFilter
from langchain_openai import OpenAIEmbeddings
unique_docs = EmbeddingsRedundantFilter(
    embeddings=OpenAIEmbeddings()
).transform_documents(clean_docs)
# 结果：语义重复的文档被移除（3 → 2）

# 重排序：优化顺序
from langchain_community.document_transformers import LongContextReorder
reordered_docs = LongContextReorder().transform_documents(unique_docs)
# 结果：最相关的文档放在首尾，避免"Lost in the Middle"
```

### 3. 为什么？

**是 RAG 管道中提升检索质量的关键环节**

RAG 管道的核心流程是：加载 → **转换** → 向量化 → 检索 → 生成。

DocumentTransformer 处于"转换"这个关键位置，直接决定了：

1. **向量化质量**：噪声文档会产生低质量的向量，导致检索不准
2. **检索效率**：冗余文档浪费存储和计算资源
3. **生成质量**：无关内容混入上下文会导致 LLM 产生幻觉

```
加载原始文档 → [DocumentTransformer] → 高质量文档 → 向量化 → 精准检索 → 优质生成
                    ↑
              清洗 + 过滤 + 增强 + 重排序
              这一步决定了整个 RAG 管道的上限
```

**实际影响：**

```python
# ❌ 不使用 DocumentTransformer
# 原始 HTML 文档直接向量化
raw_docs = loader.load()
vectorstore = Chroma.from_documents(raw_docs, embeddings)
# 问题：HTML 标签、导航栏、广告都被向量化
# 结果：检索质量差，LLM 生成的回答包含噪声

# ✅ 使用 DocumentTransformer
# 清洗后再向量化
clean_docs = Html2TextTransformer().transform_documents(raw_docs)
split_docs = text_splitter.split_documents(clean_docs)
vectorstore = Chroma.from_documents(split_docs, embeddings)
# 结果：检索质量高，LLM 生成的回答准确
```

## 一句话记忆法

**"文档净化工厂"**：像工厂流水线一样，原材料（原始文档）经过清洗、筛选、贴标签、排序等工序，变成高质量的成品（干净文档），才能送入下一道工序（向量化和检索）。

## 与相关概念的区别

| 概念 | DocumentTransformer | DocumentLoader | TextSplitter | Retriever |
|------|-------------------|----------------|--------------|-----------|
| **定位** | 文档转换器 | 文档加载器 | 文本分块器 | 文档检索器 |
| **输入** | `Sequence[Document]` | 文件路径/URL | `Sequence[Document]` | 查询字符串 |
| **输出** | `Sequence[Document]` | `List[Document]` | `List[Document]` | `List[Document]` |
| **核心方法** | `transform_documents()` | `load()` | `split_documents()` | `invoke()` |
| **关系** | 父类 | 独立组件 | 子类（继承自 Transformer） | 下游消费者 |

**关键关系：**
- TextSplitter **继承自** BaseDocumentTransformer
- `TextSplitter.transform_documents()` 内部调用 `split_documents()`
- 这意味着所有 TextSplitter 都可以作为 DocumentTransformer 使用

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

# TextSplitter 既是分块器，也是转换器
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)

# 作为分块器使用
chunks = splitter.split_documents(docs)

# 作为转换器使用（等价）
chunks = splitter.transform_documents(docs)
```

## 核心价值

### 1. 提升检索质量
清洗噪声 → 纯净文档 → 高质量向量 → 精准检索

### 2. 优化 Context Window
过滤冗余 → 减少文档数量 → 节省 token → 降低成本

### 3. 增强元数据
自动标注 → 结构化元数据 → 支持混合检索 → 提升精度

### 4. 解决 Lost in the Middle
重排序 → 关键文档放首尾 → LLM 注意力集中 → 生成质量提升

---

## 典型应用场景

1. **网页文档清洗**：爬取的 HTML 页面去除标签和噪声
2. **检索结果去重**：向量检索返回的冗余文档过滤
3. **自动分类标注**：用 LLM 为文档自动添加类别、主题等元数据
4. **长上下文优化**：重排序文档顺序，解决 LLM 的注意力衰减问题
5. **多语言处理**：文档翻译和跨语言检索
6. **QA 对提取**：从文档中自动提取问答对，用于训练或评估
