## 4.1 BM25 Function 创建与使用

### 核心概念

**BM25 Function** 是 Milvus 2.6 的核心特性，自动从文本生成稀疏向量。

**关键特点：**
- 自动分词
- 自动生成稀疏向量
- 无需手动 embedding

### 最小可用代码

```python
from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection
from pymilvus import Function, FunctionType

# 1. 连接 Milvus
connections.connect(host="localhost", port="19530")

# 2. 定义 Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR)
]

# 3. 创建 BM25 Function
bm25_function = Function(
    name="bm25_fn",
    input_field_names=["content"],
    output_field_names="sparse_vector",
    function_type=FunctionType.BM25
)

schema = CollectionSchema(fields=fields, functions=[bm25_function])
collection = Collection(name="my_collection", schema=schema)

# 4. 插入数据（自动生成稀疏向量）
data = [
    {"content": "如何配置 Kubernetes 集群？"},
    {"content": "Docker 容器化部署指南"},
    {"content": "Kubernetes 网络配置详解"}
]
collection.insert(data)
```

### 实际应用场景

**场景：文档问答系统**
- 用户输入查询："Kubernetes 集群配置"
- BM25 Function 自动生成稀疏向量
- 精确匹配关键词"Kubernetes"、"集群"、"配置"

**数据来源：** `reference/context7_pymilvus_01.md` - PyMilvus 官方文档

## 4.2 稀疏向量索引配置

### 核心概念

**稀疏向量索引** 用于加速 BM25 检索。

**两种索引类型：**
1. **SPARSE_INVERTED_INDEX**：倒排索引，精度优先
2. **SPARSE_WAND**：WAND 算法，性能优先（4 倍加速）

### 最小可用代码

```python
# 创建稀疏向量索引
collection.create_index(
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",  # 或 SPARSE_WAND
        "metric_type": "BM25",
        "params": {
            "bm25_k1": 1.2,           # 词频饱和参数
            "bm25_b": 0.75,           # 文档长度归一化参数
            "drop_ratio_build": 0.2   # 构建时丢弃低值条目
        }
    }
)

# 加载 Collection
collection.load()
```

### 索引选择建议

| 场景 | 推荐索引 | 原因 |
|------|----------|------|
| **小规模数据（< 100万）** | SPARSE_INVERTED_INDEX | 精度高，性能足够 |
| **大规模数据（> 100万）** | SPARSE_WAND | 4 倍加速，适合大规模检索 |
| **精度要求高** | SPARSE_INVERTED_INDEX | 不丢失任何信息 |
| **性能要求高** | SPARSE_WAND | 通过上界估计跳过不相关文档 |

### 实际应用场景

**场景：代码搜索引擎**
- 10 亿行代码
- 使用 SPARSE_WAND 索引
- 检索速度提升 4 倍

**数据来源：** `reference/search_wand_algorithm_01.md` - WAND 算法详解

## 4.3 混合检索实现

### 核心概念

**混合检索** = 稠密向量（语义相似度）+ 稀疏向量（关键词匹配）

**融合策略：**
1. **RRFRanker**：无需手动调整权重，适合快速上手
2. **WeightedRanker**：手动设置权重，适合精确控制

### 最小可用代码

```python
from pymilvus import AnnSearchRequest, RRFRanker
import numpy as np

# 1. 准备查询
query_text = "Kubernetes 集群配置"
query_dense = np.random.rand(768).tolist()  # 假设使用 768 维稠密向量

# 2. 创建稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense],
    anns_field="dense_vector",
    param={"metric_type": "L2", "params": {"ef": 100}},
    limit=10
)

# 3. 创建稀疏向量检索请求（BM25）
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=10
)

# 4. 执行混合检索
results = collection.hybrid_search(
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),  # RRF 融合策略
    limit=5,
    output_fields=["content"]
)

# 5. 输出结果
for i, hit in enumerate(results[0]):
    print(f"{i+1}. {hit.entity.get('content')} (score: {hit.score})")
```

### 融合策略对比

| 策略 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **RRFRanker** | 无需调参，鲁棒性好 | 无法精确控制权重 | 快速上手，不确定权重 |
| **WeightedRanker** | 精确控制权重 | 需要实验调优 | 明确各字段重要性 |

### 实际应用场景

**场景：电商商品搜索**
- 稠密向量：图像相似度
- 稀疏向量：文本关键词匹配
- 混合检索：结合视觉和文本信息

**数据来源：** `reference/search_hybrid_weight_01.md` - 混合检索权重调优

## 4.4 BM25 参数调优

### 核心概念

**BM25 参数** 控制检索行为：
- **bm25_k1**：词频饱和参数（默认 1.2）
- **bm25_b**：文档长度归一化参数（默认 0.75）
- **drop_ratio**：丢弃低值条目，减少索引大小

### 参数调优指南

#### 参数 1：bm25_k1（词频饱和参数）

**作用：** 控制词频对分数的影响

```python
# k1 = 0：退化为布尔检索（词频不影响分数）
# k1 = 1.2：默认值，平衡词频影响
# k1 = 2.0：词频影响更大
# k1 = ∞：退化为 TF-IDF（线性增长）

# 调优示例
collection.create_index(
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "BM25",
        "params": {"bm25_k1": 1.5}  # 增加词频影响
    }
)
```

**调优建议：**
- 短文档（< 100 词）：k1 = 1.0-1.2
- 长文档（> 500 词）：k1 = 1.5-2.0
- 代码搜索：k1 = 1.0（关键词精确匹配）

#### 参数 2：bm25_b（文档长度归一化参数）

**作用：** 控制文档长度对分数的影响

```python
# b = 0：不考虑文档长度
# b = 0.75：默认值，平衡文档长度影响
# b = 1：完全归一化（长文档不占优势）

# 调优示例
collection.create_index(
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "BM25",
        "params": {"bm25_b": 0.5}  # 减少文档长度影响
    }
)
```

**调优建议：**
- 文档长度差异大：b = 0.75-1.0
- 文档长度相近：b = 0.3-0.5
- 短文档优先：b = 0.0-0.3

#### 参数 3：drop_ratio（丢弃低值条目）

**作用：** 减少索引大小，加速搜索

```python
# drop_ratio_build：构建时丢弃低值条目
# drop_ratio_search：搜索时丢弃低值条目

# 调优示例
collection.create_index(
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "BM25",
        "params": {
            "drop_ratio_build": 0.2,   # 丢弃 20% 低值条目
            "drop_ratio_search": 0.1   # 搜索时丢弃 10% 低值条目
        }
    }
)
```

**调优建议：**
- 精度优先：drop_ratio = 0.0-0.1
- 性能优先：drop_ratio = 0.2-0.3
- 平衡：drop_ratio = 0.1-0.2

### 实际应用场景

**场景：新闻推荐系统**
- 文档长度差异大（100-5000 词）
- 参数配置：k1 = 1.5, b = 0.8, drop_ratio = 0.15
- 效果：准确率提升 15%，检索速度提升 30%

**数据来源：** `reference/search_bm25_practices_01.md` - BM25 最佳实践

## 4.5 生产级混合检索系统

### 核心概念

**生产级系统** 需要考虑：
- 性能优化
- 错误处理
- 监控与日志
- 动态参数调整

### 最小可用代码

```python
from pymilvus import connections, Collection, AnnSearchRequest, RRFRanker
import logging
import time

# 配置日志
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

class HybridSearchSystem:
    def __init__(self, collection_name: str):
        """初始化混合检索系统"""
        self.collection_name = collection_name
        self.collection = None
        self._connect()

    def _connect(self):
        """连接 Milvus"""
        try:
            connections.connect(host="localhost", port="19530")
            self.collection = Collection(name=self.collection_name)
            self.collection.load()
            logger.info(f"Connected to collection: {self.collection_name}")
        except Exception as e:
            logger.error(f"Failed to connect: {e}")
            raise

    def search(self, query_text: str, query_dense: list, top_k: int = 5):
        """执行混合检索"""
        start_time = time.time()

        try:
            # 创建检索请求
            dense_req = AnnSearchRequest(
                data=[query_dense],
                anns_field="dense_vector",
                param={"metric_type": "L2", "params": {"ef": 100}},
                limit=top_k * 2
            )

            sparse_req = AnnSearchRequest(
                data=[query_text],
                anns_field="sparse_vector",
                param={"metric_type": "BM25"},
                limit=top_k * 2
            )

            # 执行混合检索
            results = self.collection.hybrid_search(
                reqs=[dense_req, sparse_req],
                ranker=RRFRanker(k=60),
                limit=top_k,
                output_fields=["content"]
            )

            # 记录性能
            elapsed_time = time.time() - start_time
            logger.info(f"Search completed in {elapsed_time:.3f}s")

            return results[0]

        except Exception as e:
            logger.error(f"Search failed: {e}")
            return []

    def close(self):
        """关闭连接"""
        if self.collection:
            self.collection.release()
        connections.disconnect("default")
        logger.info("Connection closed")

# 使用示例
system = HybridSearchSystem("my_collection")
results = system.search(
    query_text="Kubernetes 集群配置",
    query_dense=[0.1] * 768,
    top_k=5
)

for i, hit in enumerate(results):
    print(f"{i+1}. {hit.entity.get('content')} (score: {hit.score})")

system.close()
```

## 常见问题

### Q1: 什么时候使用 BM25？

**A:** 当你需要关键词精确匹配时，例如：
- 搜索特定技术名词（"Kubernetes"、"Docker"）
- 搜索代码片段
- 搜索产品型号

### Q2: 混合检索一定比单一检索好吗？

**A:** 不一定，取决于场景：
- 关键词匹配场景：BM25 更好
- 语义理解场景：稠密向量更好
- 综合场景：混合检索更好

### Q3: 如何选择索引类型？

**A:**
- 小规模数据（< 100万）：SPARSE_INVERTED_INDEX
- 大规模数据（> 100万）：SPARSE_WAND
- 精度优先：SPARSE_INVERTED_INDEX
- 性能优先：SPARSE_WAND

### Q4: 如何调优 BM25 参数？

**A:** 实验驱动：
1. 准备测试集（查询 + 相关文档）
2. 尝试不同参数组合
3. 评估准确率和召回率
4. 选择最优参数

### Q5: 混合检索的权重如何设置？

**A:**
- 快速上手：使用 RRFRanker（无需调参）
- 精确控制：使用 WeightedRanker（需要实验）
- 建议：先用 RRF，不满意再用 Weighted
