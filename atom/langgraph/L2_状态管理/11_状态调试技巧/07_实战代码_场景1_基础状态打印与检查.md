# 实战代码 - 场景1：基础状态打印与检查

## 场景说明

本场景演示如何在 LangGraph 中使用最基础的调试手段排查问题。你将看到：

1. 使用 `print()` 在节点内部打印状态，快速定位数据流向
2. 使用 `logging` 模块输出分级日志，区分调试信息和正式日志
3. 使用 `get_state()` 在图执行后检查完整状态快照
4. 使用 `get_state_history()` 遍历每一步的状态变化
5. 组合以上手段，形成一套完整的基础调试工作流

**核心原则：** 调试的本质是"让不可见的状态变得可见"。print 解决 80% 的问题，logging 解决需要持久化的问题，get_state 解决需要回溯的问题。

## 完整代码

```python
"""
LangGraph 状态调试 - 场景1：基础状态打印与检查
演示：print 调试、logging 集成、get_state 检查、get_state_history 历史追溯

核心知识点：
- print() 快速调试：在节点函数内打印输入/输出状态
- logging 分级日志：DEBUG/INFO/WARNING 分层记录
- get_state() 状态快照：执行后检查完整状态
- get_state_history() 历史遍历：回溯每一步的状态变化
- 调试辅助函数：封装通用的状态打印逻辑

运行环境：Python 3.13+, langgraph
安装依赖：uv add langgraph
"""

import logging
import operator
from typing import TypedDict, Annotated

from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver


# ============================================================
# 1. 配置 logging —— 调试的第一步
# ============================================================

# 创建专用 logger，避免污染全局日志
# 命名为 "graph_debug"，后续可以单独控制这个 logger 的级别
logger = logging.getLogger("graph_debug")
logger.setLevel(logging.DEBUG)

# 创建控制台处理器，设置格式
# 格式说明：时间 [logger名] 级别: 消息
console_handler = logging.StreamHandler()
console_handler.setLevel(logging.DEBUG)
formatter = logging.Formatter(
    "%(asctime)s [%(name)s] %(levelname)s: %(message)s",
    datefmt="%H:%M:%S",
)
console_handler.setFormatter(formatter)
logger.addHandler(console_handler)

# 防止日志重复（如果在 Jupyter 中多次运行）
logger.propagate = False


# ============================================================
# 2. 定义状态 —— 一个模拟 RAG 管道的多步骤状态
# ============================================================

class RAGDebugState(TypedDict):
    """模拟 RAG 管道的状态，用于演示调试技巧

    字段说明：
    - query: 用户的原始查询
    - steps: 已执行的步骤列表（使用 operator.add 累积）
    - documents: 检索到的文档列表
    - answer: 最终生成的回答
    - debug_info: 调试信息字典（记录每个节点的执行细节）
    """
    query: str
    steps: Annotated[list[str], operator.add]
    documents: list[dict]
    answer: str
    debug_info: dict


# ============================================================
# 3. 调试辅助函数 —— 封装通用的状态打印逻辑
# ============================================================

def print_state_summary(label: str, state: dict) -> None:
    """打印状态摘要 —— 调试时最常用的辅助函数

    为什么需要这个函数？
    直接 print(state) 在状态很大时（比如包含长文档列表）会刷屏。
    这个函数只打印关键信息：有哪些键、每个值的类型和长度。
    """
    print(f"\n{'='*50}")
    print(f"  [{label}] 状态摘要")
    print(f"{'='*50}")
    for key, value in state.items():
        if isinstance(value, list):
            print(f"  {key}: list[{len(value)}项]")
        elif isinstance(value, dict):
            print(f"  {key}: dict{{{', '.join(value.keys())}}}")
        elif isinstance(value, str) and len(value) > 50:
            print(f"  {key}: '{value[:50]}...'")
        else:
            print(f"  {key}: {value!r}")
    print(f"{'='*50}")


def debug_node(node_name: str, state: dict, output: dict) -> dict:
    """通用节点调试包装器 —— 同时使用 print 和 logging

    使用方法：在节点函数的 return 之前调用
    作用：
    1. print 输出人类可读的摘要（开发时看控制台）
    2. logging 记录详细信息（可以输出到文件）
    3. 返回原始 output，不影响图的执行
    """
    # print 输出 —— 快速查看
    print(f"\n>>> 节点 [{node_name}] 执行完毕")
    print(f"    输入键: {list(state.keys())}")
    print(f"    输出键: {list(output.keys())}")

    # logging 输出 —— 详细记录
    logger.debug(f"节点 [{node_name}] 输入状态: {state}")
    logger.info(f"节点 [{node_name}] 输出更新: {output}")

    return output


# ============================================================
# 4. 节点函数 —— 每个节点都内置调试输出
# ============================================================

def analyze_query(state: RAGDebugState) -> dict:
    """步骤1：分析用户查询

    调试要点：
    - 检查 query 是否正确传入
    - 确认 steps 列表的初始状态
    """
    query = state["query"]

    # === 调试技巧1：print 直接打印关键变量 ===
    print(f"\n[analyze_query] 收到查询: '{query}'")
    print(f"[analyze_query] 当前步骤数: {len(state['steps'])}")

    # === 调试技巧2：logging 记录详细信息 ===
    logger.info(f"开始分析查询: {query}")
    logger.debug(f"完整输入状态: {state}")

    # 模拟查询分析
    analysis = f"已分析: {query} -> 类型=知识问答, 领域=AI"

    output = {
        "steps": ["analyze_query"],
        "debug_info": {
            "analyze_query": {
                "input_query": query,
                "analysis": analysis,
            }
        },
    }

    # === 调试技巧3：使用辅助函数统一输出 ===
    return debug_node("analyze_query", state, output)


def retrieve_documents(state: RAGDebugState) -> dict:
    """步骤2：检索相关文档

    调试要点：
    - 确认 analyze_query 的结果已经在 steps 中
    - 检查检索到的文档数量和内容
    """
    query = state["query"]

    logger.info(f"开始检索文档, 查询: {query}")
    print(f"\n[retrieve_documents] 已完成步骤: {state['steps']}")

    # 模拟文档检索
    docs = [
        {"content": f"RAG 是检索增强生成的缩写", "score": 0.95, "source": "wiki"},
        {"content": f"RAG 结合了检索和生成两个阶段", "score": 0.88, "source": "paper"},
        {"content": f"RAG 可以减少大模型的幻觉问题", "score": 0.76, "source": "blog"},
    ]

    # === 调试技巧4：打印中间计算结果 ===
    print(f"[retrieve_documents] 检索到 {len(docs)} 篇文档")
    for i, doc in enumerate(docs):
        print(f"  文档{i+1}: score={doc['score']}, source={doc['source']}")

    # === 调试技巧5：logging.warning 标记需要注意的情况 ===
    low_score_docs = [d for d in docs if d["score"] < 0.8]
    if low_score_docs:
        logger.warning(f"有 {len(low_score_docs)} 篇文档评分低于 0.8，可能影响回答质量")

    output = {
        "steps": ["retrieve_documents"],
        "documents": docs,
        "debug_info": {
            **state.get("debug_info", {}),
            "retrieve_documents": {
                "doc_count": len(docs),
                "avg_score": sum(d["score"] for d in docs) / len(docs),
                "sources": [d["source"] for d in docs],
            },
        },
    }

    return debug_node("retrieve_documents", state, output)


def generate_answer(state: RAGDebugState) -> dict:
    """步骤3：生成回答

    调试要点：
    - 确认 documents 已经被正确填充
    - 检查生成的回答是否合理
    """
    docs = state["documents"]
    query = state["query"]

    logger.info(f"开始生成回答, 文档数: {len(docs)}")
    print(f"\n[generate_answer] 使用 {len(docs)} 篇文档生成回答")

    # === 调试技巧6：断言检查前置条件 ===
    # 在开发阶段使用 assert，确保上游节点正确执行
    assert len(docs) > 0, "generate_answer: 没有检索到文档！"
    assert all("content" in d for d in docs), "文档缺少 content 字段！"

    # 模拟回答生成
    context = " | ".join(d["content"] for d in docs[:2])
    answer = f"关于「{query}」：{context}。综合以上信息，RAG 是一种结合检索与生成的技术。"

    print(f"[generate_answer] 生成回答长度: {len(answer)} 字符")

    output = {
        "steps": ["generate_answer"],
        "answer": answer,
        "debug_info": {
            **state.get("debug_info", {}),
            "generate_answer": {
                "context_length": len(context),
                "answer_length": len(answer),
                "docs_used": len(docs[:2]),
            },
        },
    }

    return debug_node("generate_answer", state, output)


# ============================================================
# 5. 构建并编译图
# ============================================================

print("=" * 60)
print("=== 构建 RAG 调试演示图 ===")
print("=" * 60)

builder = StateGraph(RAGDebugState)

# 添加节点
builder.add_node("analyze", analyze_query)
builder.add_node("retrieve", retrieve_documents)
builder.add_node("generate", generate_answer)

# 添加边（线性流程）
builder.add_edge(START, "analyze")
builder.add_edge("analyze", "retrieve")
builder.add_edge("retrieve", "generate")
builder.add_edge("generate", END)

# 编译图 —— 必须传入 checkpointer 才能使用 get_state
# InMemorySaver 是开发调试的最佳选择：零配置、速度快、重启即清空
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

print("图编译完成，节点: analyze -> retrieve -> generate")


# ============================================================
# 6. 执行图 —— 观察 print 和 logging 的输出
# ============================================================

print("\n")
print("=" * 60)
print("=== 执行图 ===")
print("=" * 60)

# config 中的 thread_id 是 get_state 的关键
# 同一个 thread_id 的所有执行共享同一条 checkpoint 链
config = {"configurable": {"thread_id": "debug-demo-1"}}

# 初始状态
initial_state = {
    "query": "什么是 RAG？",
    "steps": [],
    "documents": [],
    "answer": "",
    "debug_info": {},
}

# invoke 会依次执行所有节点
result = graph.invoke(initial_state, config)

# 执行完毕后打印最终结果
print_state_summary("最终结果", result)


# ============================================================
# 7. get_state —— 执行后检查状态快照
# ============================================================

print("\n")
print("=" * 60)
print("=== get_state 状态快照检查 ===")
print("=" * 60)

# get_state 返回 StateSnapshot 对象
# 它包含图执行完毕后的完整状态信息
snapshot = graph.get_state(config)

# --- 7.1 查看当前状态值 ---
print("\n--- 7.1 当前状态值 (snapshot.values) ---")
values = snapshot.values
print(f"  query: {values['query']}")
print(f"  steps: {values['steps']}")
print(f"  documents 数量: {len(values['documents'])}")
print(f"  answer 前50字: {values['answer'][:50]}...")
print(f"  debug_info 键: {list(values['debug_info'].keys())}")

# --- 7.2 查看下一步节点 ---
print("\n--- 7.2 下一步节点 (snapshot.next) ---")
print(f"  next: {snapshot.next}")
print(f"  解读: {'图已执行完毕' if not snapshot.next else f'下一步执行 {snapshot.next}'}")

# --- 7.3 查看元数据 ---
print("\n--- 7.3 元数据 (snapshot.metadata) ---")
metadata = snapshot.metadata
print(f"  source: {metadata.get('source')}")
print(f"  step: {metadata.get('step')}")
print(f"  writes: {metadata.get('writes') is not None}")

# --- 7.4 查看配置信息 ---
print("\n--- 7.4 配置信息 (snapshot.config) ---")
print(f"  thread_id: {snapshot.config['configurable']['thread_id']}")
print(f"  checkpoint_id: {snapshot.config['configurable']['checkpoint_id']}")

# --- 7.5 查看创建时间 ---
print("\n--- 7.5 创建时间 (snapshot.created_at) ---")
print(f"  created_at: {snapshot.created_at}")

# --- 7.6 查看任务信息 ---
print("\n--- 7.6 任务信息 (snapshot.tasks) ---")
print(f"  tasks 数量: {len(snapshot.tasks)}")
if snapshot.tasks:
    for task in snapshot.tasks:
        print(f"    任务: {task.name}, id={task.id[:8]}...")


# ============================================================
# 8. get_state_history —— 遍历每一步的状态变化
# ============================================================

print("\n")
print("=" * 60)
print("=== get_state_history 状态历史遍历 ===")
print("=" * 60)

print("\n历史记录按时间倒序排列（最新的在前）：\n")

for i, snap in enumerate(graph.get_state_history(config)):
    step = snap.metadata.get("step", "?")
    source = snap.metadata.get("source", "?")
    next_nodes = snap.next

    print(f"--- 历史记录 #{i} ---")
    print(f"  step: {step}")
    print(f"  source: {source}")
    print(f"  next: {next_nodes}")

    # 打印状态中的 steps 字段，追踪执行进度
    if isinstance(snap.values, dict) and "steps" in snap.values:
        print(f"  steps: {snap.values['steps']}")

    # 打印 checkpoint_id（用于时间旅行调试）
    cp_id = snap.config["configurable"].get("checkpoint_id", "N/A")
    print(f"  checkpoint_id: {cp_id[:20]}...")
    print()


# ============================================================
# 9. 综合调试示例 —— 定位"为什么回答不对"
# ============================================================

print("=" * 60)
print("=== 综合调试示例：定位问题 ===")
print("=" * 60)

print("""
假设用户反馈"回答不够准确"，我们用以下步骤排查：

步骤1: 用 get_state 查看最终状态
步骤2: 检查 documents 的评分分布
步骤3: 检查 debug_info 中的执行细节
步骤4: 用 get_state_history 找到检索步骤的状态
""")

# 步骤1: 查看最终状态
final_snapshot = graph.get_state(config)
final_values = final_snapshot.values

# 步骤2: 检查文档评分
print("--- 文档评分分析 ---")
docs = final_values["documents"]
scores = [d["score"] for d in docs]
avg_score = sum(scores) / len(scores)
print(f"  文档数量: {len(docs)}")
print(f"  评分范围: {min(scores):.2f} ~ {max(scores):.2f}")
print(f"  平均评分: {avg_score:.2f}")
if avg_score < 0.85:
    print(f"  [提示] 平均评分偏低，建议优化检索策略")

# 步骤3: 检查 debug_info
print("\n--- debug_info 执行细节 ---")
debug_info = final_values.get("debug_info", {})
for node_name, info in debug_info.items():
    print(f"  {node_name}: {info}")

# 步骤4: 找到检索步骤的状态
print("\n--- 检索步骤的历史状态 ---")
for snap in graph.get_state_history(config):
    if snap.metadata.get("source") == "loop":
        steps = snap.values.get("steps", [])
        if "retrieve_documents" in steps and "generate_answer" not in steps:
            print(f"  找到检索完成时的状态快照:")
            print(f"  steps: {steps}")
            print(f"  documents 数量: {len(snap.values.get('documents', []))}")
            print(f"  checkpoint_id: {snap.config['configurable']['checkpoint_id'][:20]}...")
            break

print("\n调试完成！通过以上步骤，可以精确定位问题出在哪个环节。")
```

## 代码解析

### 调试技巧速查表

| 技巧 | 适用场景 | 代码示例 |
|------|---------|---------|
| `print()` | 快速查看变量值 | `print(f"[节点名] 变量: {var}")` |
| `logging.info()` | 记录正常执行流程 | `logger.info(f"开始处理: {query}")` |
| `logging.debug()` | 记录详细调试信息 | `logger.debug(f"完整状态: {state}")` |
| `logging.warning()` | 标记异常但不中断 | `logger.warning("评分偏低")` |
| `get_state()` | 执行后检查最终状态 | `snapshot = graph.get_state(config)` |
| `get_state_history()` | 回溯每一步的变化 | `for s in graph.get_state_history(config):` |
| `assert` | 检查前置条件 | `assert len(docs) > 0` |
| 辅助函数 | 统一调试输出格式 | `debug_node("name", state, output)` |

### 关键 API 说明

**StateSnapshot 对象的属性：**

```python
snapshot = graph.get_state(config)

snapshot.values       # dict - 当前状态的所有字段值
snapshot.next         # tuple - 下一步要执行的节点名（空 tuple 表示已完成）
snapshot.config       # dict - 包含 thread_id 和 checkpoint_id
snapshot.metadata     # dict - 包含 source、step、writes 等元信息
snapshot.created_at   # str - checkpoint 创建时间（ISO 格式）
snapshot.tasks        # tuple - 待执行的任务列表
snapshot.parent_config  # dict | None - 父 checkpoint 的配置
```

**get_state_history 的遍历顺序：**

```
最新 (step=3, 图执行完毕)     ← 第一个返回
  ↑
step=2 (generate 执行前)
  ↑
step=1 (retrieve 执行前)
  ↑
step=0 (analyze 执行前)
  ↑
step=-1 (input, 初始输入)     ← 最后一个返回
```

历史记录按时间倒序排列。`step=-1` 是初始输入，`source=input`；后续步骤 `source=loop`，表示图的执行循环。

## 关键要点

1. **InMemorySaver 是开发调试的最佳选择** —— 零配置、速度快、进程结束自动清空，不会留下垃圾数据
2. **get_state 需要 checkpointer** —— 没有 checkpointer 的图无法调用 `get_state()`，会抛出异常
3. **历史记录按时间倒序排列** —— `get_state_history()` 返回的第一条是最新状态，最后一条是初始输入
4. **print 和 logging 互补** —— print 用于快速开发调试，logging 用于需要持久化或分级过滤的场景
5. **debug_info 字段是好习惯** —— 在状态中专门留一个字段记录调试信息，比散落的 print 更容易管理
6. **assert 只在开发阶段使用** —— 生产环境应该用 `if` + 异常处理替代 assert

## 在 RAG 系统中的应用

基础状态调试在 RAG 系统中的典型用途：

- **检索质量排查**：通过 `get_state` 检查检索到的文档评分分布，判断是检索问题还是生成问题
- **管道断点分析**：通过 `get_state_history` 找到出错步骤的前一个状态，定位问题节点
- **对话上下文追踪**：在多轮对话中，用 logging 记录每轮的 query 和 context 变化
- **A/B 测试对比**：用不同 thread_id 运行同一查询的不同策略，用 get_state 对比最终状态
