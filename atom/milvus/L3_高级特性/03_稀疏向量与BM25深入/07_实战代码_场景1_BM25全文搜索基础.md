## 场景描述

- 检索速度 < 100ms

### 学习目标

- 掌握 BM25 Function 的创建和使用
- 理解稀疏向量索引的配置
- 学会执行 BM25 全文搜索

"""
场景1：BM25 全文搜索基础
演示：从零开始创建 BM25 全文搜索系统

数据来源：
- reference/context7_pymilvus_01.md - PyMilvus 官方文档
- reference/source_sparse_bm25_01.md - Milvus 源码分析
"""

import os
from pymilvus import (
    connections,
    FieldSchema,
    CollectionSchema,
    DataType,
    Collection,
    Function,
    FunctionType,
    utility
)
import time

# ===== 1. 连接 Milvus =====
print("=== 步骤1：连接 Milvus 2.6 ===")

connections.connect(
    alias="default",
    host=os.getenv("MILVUS_HOST", "localhost"),
    port=os.getenv("MILVUS_PORT", "19530")
)
print("✓ 成功连接到 Milvus")

# ===== 2. 定义 Collection Schema =====
print("\n=== 步骤2：定义 Collection Schema ===")

# 定义字段
fields = [
    FieldSchema(
        name="id",
        dtype=DataType.INT64,
        is_primary=True,
        auto_id=True,
        description="文档 ID"
    ),
    FieldSchema(
        name="title",
        dtype=DataType.VARCHAR,
        max_length=512,
        description="文档标题"
    ),
    FieldSchema(
        name="content",
        dtype=DataType.VARCHAR,
        max_length=65535,
        description="文档内容（用于 BM25）"
    ),
    FieldSchema(
        name="sparse_vector",
        dtype=DataType.SPARSE_FLOAT_VECTOR,
        description="BM25 生成的稀疏向量"
    )
]

print("✓ 定义了 4 个字段：id, title, content, sparse_vector")

# ===== 3. 创建 BM25 Function =====
print("\n=== 步骤3：创建 BM25 Function ===")

# BM25 Function 自动从 content 字段生成 sparse_vector
bm25_function = Function(
    name="bm25_fn",
    input_field_names=["content"],  # 输入字段
    output_field_names=["sparse_vector"],  # 输出字段
    function_type=FunctionType.BM25  # 函数类型
)

print("✓ 创建 BM25 Function：content -> sparse_vector")
print(f"  - 输入字段：{bm25_function.input_field_names}")
print(f"  - 输出字段：{bm25_function.output_field_names}")
print(f"  - 函数类型：{bm25_function.function_type}")

# ===== 4. 创建 Collection =====
print("\n=== 步骤4：创建 Collection ===")

collection_name = "tech_docs_bm25"

# 删除已存在的 Collection
if utility.has_collection(collection_name):
    utility.drop_collection(collection_name)
    print(f"✓ 删除已存在的 Collection: {collection_name}")

# 创建 Schema（包含 BM25 Function）
schema = CollectionSchema(
    fields=fields,
    functions=[bm25_function],  # 添加 BM25 Function
    description="技术文档 BM25 全文搜索"
)

# 创建 Collection
collection = Collection(
    name=collection_name,
    schema=schema
)

print(f"✓ 创建 Collection: {collection_name}")
print(f"  - 字段数量：{len(fields)}")
print(f"  - 函数数量：1 (BM25)")

# ===== 5. 创建稀疏向量索引 =====
print("\n=== 步骤5：创建稀疏向量索引 ===")

# 配置索引参数
index_params = {
    "index_type": "SPARSE_INVERTED_INDEX",  # 倒排索引
    "metric_type": "BM25",  # BM25 度量
    "params": {
        "bm25_k1": 1.2,  # 词频饱和参数（默认值）
        "bm25_b": 0.75,  # 文档长度归一化参数（默认值）
        "drop_ratio_build": 0.0  # 不丢弃任何条目（精度优先）
    }
}

collection.create_index(
    field_name="sparse_vector",
    index_params=index_params
)

print("✓ 创建稀疏向量索引")
print(f"  - 索引类型：{index_params['index_type']}")
print(f"  - 度量类型：{index_params['metric_type']}")
print(f"  - BM25 参数：k1={index_params['params']['bm25_k1']}, b={index_params['params']['bm25_b']}")

# ===== 6. 准备测试数据 =====
print("\n=== 步骤6：准备测试数据 ===")

# 技术文档数据（中英文混合）
documents = [
    {

## 关键点解析

### 1. BM25 Function 的工作原理

**核心机制**：
```python
bm25_function = Function(
    name="bm25_fn",
    input_field_names=["content"],  # 输入：原始文本
    output_field_names=["sparse_vector"],  # 输出：稀疏向量
    function_type=FunctionType.BM25
)
```

**处理流程**：
1. **分词**：使用 Analyzer 将文本分词
2. **哈希**：将 token 转换为 uint32（使用 `HashString2LessUint32`）
3. **词频统计**：统计每个 token 的出现次数
4. **稀疏向量**：生成 `map[uint32]float32`

**数据来源**：`reference/source_sparse_bm25_01.md` - Milvus 源码分析

---

### 2. 稀疏向量索引配置

**索引类型选择**：

| 索引类型 | 适用场景 | 性能 | 精度 |
|----------|----------|------|------|
| **SPARSE_INVERTED_INDEX** | 小规模数据（< 100万） | 中等 | 高 |
| **SPARSE_WAND** | 大规模数据（> 100万） | 高（4倍加速） | 中等 |

**参数说明**：
- `bm25_k1`：词频饱和参数（默认 1.2）
  - 值越大，词频对分数的影响越大
  - 推荐范围：1.0-2.0
- `bm25_b`：文档长度归一化参数（默认 0.75）
  - 值越大，文档长度对分数的影响越大
  - 推荐范围：0.3-1.0
- `drop_ratio_build`：构建时丢弃低值条目
  - 0.0 = 不丢弃（精度优先）
  - 0.2 = 丢弃 20%（性能优先）

**数据来源**：`reference/context7_pymilvus_01.md` - PyMilvus 官方文档

---

### 3. BM25 搜索的优势

**关键词精确匹配**：
- 查询"Kubernetes 集群配置"
- 精确匹配包含"Kubernetes"、"集群"、"配置"的文档
- 分数越高，匹配度越高

**与向量检索的对比**：

| 特性 | BM25 | 向量检索 |
|------|------|----------|
| **匹配方式** | 关键词精确匹配 | 语义相似度 |
| **适用场景** | 技术名词、产品型号 | 自然语言理解 |
| **速度** | 快（倒排索引） | 中等（ANN） |
| **准确率** | 高（精确匹配） | 中等（语义理解） |

**数据来源**：`reference/search_bm25_practices_01.md` - BM25 最佳实践

### 4. 性能优化建议

**索引优化**：
```python
# 小规模数据（< 100万）
index_params = {
    "index_type": "SPARSE_INVERTED_INDEX",
    "params": {"drop_ratio_build": 0.0}  # 精度优先
}

# 大规模数据（> 100万）
index_params = {
    "index_type": "SPARSE_WAND",
    "params": {"drop_ratio_build": 0.2}  # 性能优先
}
```

**搜索优化**：
```python
# 添加 drop_ratio_search 参数
search_params = {
    "metric_type": "BM25",
    "params": {"drop_ratio_search": 0.1}  # 搜索时丢弃 10%
}
```

### 5. 常见问题

**Q1: BM25 Function 支持哪些语言？**

A: 支持多种语言，包括中文、英文、日文等。通过配置 Analyzer 参数可以指定分词器。

**Q2: 如何处理中英文混合文本？**

A: Milvus 2.6 的 BM25 Function 自动处理中英文混合文本，无需额外配置。

**Q3: BM25 搜索的延迟是多少？**

A: 通常 < 100ms（取决于数据规模和索引类型）。使用 SPARSE_WAND 索引可以进一步降低延迟。

**Q4: 如何调优 BM25 参数？**

A: 建议从默认值开始（k1=1.2, b=0.75），根据实际效果调整。详见场景4。
