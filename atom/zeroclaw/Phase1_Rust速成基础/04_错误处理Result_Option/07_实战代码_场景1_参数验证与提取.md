# 实战代码 场景1：参数验证与提取

> **前置知识**：03_核心概念_4_组合子方法.md, 03_核心概念_5_anyhow灵活错误处理.md
> **预计阅读**：30 分钟
> **难度**：入门 → 实战
> **目标**：模拟 ZeroClaw Tool 的参数解析模式，掌握 Option 组合子链 + bail!/ensure! 验证

---

## 场景说明

ZeroClaw 的每个 Tool 都实现了 `execute(&self, args: Value) -> anyhow::Result<ToolResult>` 方法。这个方法的第一件事就是**从 JSON 参数中安全提取值并验证合法性**。

本场景模拟一个 `WebFetchTool`，从 JSON 参数中提取 URL 和可选配置，验证参数合法性，返回结构化结果。

### 涉及的 ZeroClaw 源码模式

| 模式 | ZeroClaw 源文件 | 本场景对应 |
|------|----------------|-----------|
| Option 组合子链 `.get().and_then().ok_or_else()` | `src/tools/shell.rs` | `extract_url()`, `extract_method()` |
| `bail!` 验证 | `src/tools/web_fetch.rs` | `validate_url()` |
| `ensure!` 断言 | `src/security/secrets.rs` | `validate_url()` |
| match 处理最终结果 | `src/tools/web_fetch.rs` | `main()` 中的 match |
| `unwrap_or` 安全默认值 | `src/tools/shell.rs` | 可选参数提取 |

[来源: reference/source_error_handling_01.md - Option 组合子链, bail! 提前返回, ensure! 条件断言]

### src/main.rs

```rust
// ============================================================
// 场景1：模拟 ZeroClaw Tool 的参数验证与提取
// ============================================================
//
// 对应 ZeroClaw 源码：
//   - src/tools/traits.rs   → ToolResult 结构体、Tool trait 定义
//   - src/tools/shell.rs    → .get().and_then().ok_or_else() 参数提取
//   - src/tools/web_fetch.rs → bail! 验证 URL、match 处理验证结果
//   - src/security/secrets.rs → ensure! 条件断言
//   - src/config/schema.rs  → Option<T> 可选配置字段 + unwrap_or
//
// 每个函数上方的注释标注了对应的 ZeroClaw 源码位置。
// ============================================================

use anyhow::{Result, bail, ensure, anyhow};
use serde_json::{json, Value};

// ----------------------------------------------------------
// 1. ToolResult 结构体（模拟 src/tools/traits.rs）
// ----------------------------------------------------------
// ZeroClaw 的 ToolResult 包含 output、success、error 等字段。
// Tool 执行失败时不 panic，而是返回 success: false + error 信息。

#[derive(Debug)]
struct ToolResult {
    output: String,
    success: bool,
    error: Option<String>,
}

impl Default for ToolResult {
    fn default() -> Self {
        Self {
            output: String::new(),
            success: true,
            error: None,
        }
    }
}

// ----------------------------------------------------------
// 2. 参数提取函数（模拟 src/tools/shell.rs 的参数提取模式）
// ----------------------------------------------------------
// 核心模式：args.get("key").and_then(|v| v.as_str()).ok_or_else(|| anyhow!("..."))?
//
// 这是 ZeroClaw 每个 Tool 的 execute 方法中出现频率最高的模式。
// 三步链式操作：
//   1. .get("key")        → Option<&Value>  （key 可能不存在）
//   2. .and_then(|v| ...) → Option<&str>    （值可能不是字符串）
//   3. .ok_or_else(...)   → Result<&str, E> （None 转为带信息的 Err）

/// 提取必填的 url 参数（对应 src/tools/web_fetch.rs）
fn extract_url(args: &Value) -> Result<String> {
    // ★ ZeroClaw 最高频模式：Option 组合子链
    let url = args.get("url")                              // Option<&Value>
        .and_then(|v| v.as_str())                          // Option<&str>
        .ok_or_else(|| anyhow!("Missing 'url' parameter"))?; // Result<&str, Error>
    Ok(url.to_string())
}

/// 提取可选的 method 参数，默认 "GET"（对应 src/tools/shell.rs 的 unwrap_or 模式）
fn extract_method(args: &Value) -> String {
    // ★ 可选参数模式：and_then + unwrap_or
    args.get("method")
        .and_then(|v| v.as_str())
        .unwrap_or("GET")          // 缺失时用默认值，不报错
        .to_string()
}

/// 提取可选的 timeout 参数，默认 30（对应 src/config/schema.rs 的 Option 字段）
fn extract_timeout(args: &Value) -> u64 {
    args.get("timeout")
        .and_then(|v| v.as_u64())
        .unwrap_or(30)
}

/// 提取可选的 headers（演示更深层的 Option 链）
fn extract_headers(args: &Value) -> Vec<(String, String)> {
    // .and_then 链可以任意深入，遇到 None 就短路
    let headers = args.get("headers")
        .and_then(|v| v.as_object());

    match headers {
        Some(map) => map.iter()
            .filter_map(|(k, v)| {
                // filter_map：闭包返回 Option，None 的项被过滤掉
                v.as_str().map(|s| (k.clone(), s.to_string()))
            })
            .collect(),
        None => Vec::new(),
    }
}

// ----------------------------------------------------------
// 3. 参数验证函数（模拟 src/tools/web_fetch.rs 的 validate_url）
// ----------------------------------------------------------
// ZeroClaw 在提取参数后，会进行业务规则验证。
// 使用 bail! 和 ensure! 在验证失败时提前返回错误。

/// 验证 URL 合法性（对应 src/tools/web_fetch.rs 的 validate_url 方法）
fn validate_url(url: &str) -> Result<()> {
    // ★ bail! 模式：条件不满足，立即返回错误
    // 对应 ZeroClaw: anyhow::bail!("URL cannot be empty")
    if url.is_empty() {
        bail!("URL cannot be empty");
    }

    // ★ ensure! 模式：一行条件断言
    // 对应 ZeroClaw: anyhow::ensure!(blob.len() > NONCE_LEN, "...")
    ensure!(
        url.starts_with("http://") || url.starts_with("https://"),
        "Only http:// and https:// URLs are allowed, got: {url}"
    );

    // 额外验证：URL 长度限制
    ensure!(
        url.len() <= 2048,
        "URL too long: {} chars (max 2048)", url.len()
    );

    Ok(())
}

/// 验证 HTTP method（演示 bail! 的格式化能力）
fn validate_method(method: &str) -> Result<()> {
    let allowed = ["GET", "POST", "PUT", "DELETE", "PATCH", "HEAD"];
    if !allowed.contains(&method) {
        bail!(
            "Invalid HTTP method '{}'. Allowed: {}",
            method,
            allowed.join(", ")
        );
    }
    Ok(())
}

// ----------------------------------------------------------
// 4. execute 函数（模拟 Tool trait 的 execute 方法）
// ----------------------------------------------------------
// 这是 ZeroClaw 每个 Tool 的核心方法。
// 流程：提取参数 → 验证参数 → 执行业务逻辑 → 返回 ToolResult
//
// 对应 src/tools/web_fetch.rs 的 execute 实现：
//   1. 提取 url 参数（Option 组合子链）
//   2. validate_url（bail!/ensure!）
//   3. 执行 HTTP 请求
//   4. 返回 ToolResult

fn execute(args: Value) -> Result<ToolResult> {
    // 第一步：提取参数
    // 必填参数用 ? 传播错误，可选参数用 unwrap_or 给默认值
    let url = extract_url(&args)?;
    let method = extract_method(&args);
    let timeout = extract_timeout(&args);
    let headers = extract_headers(&args);

    // 第二步：验证参数
    // 对应 ZeroClaw web_fetch.rs 中的 validate_url 调用
    validate_url(&url)?;
    validate_method(&method)?;

    // 第三步：模拟执行（真实 ZeroClaw 这里会发 HTTP 请求）
    let output = format!(
        "Fetched {} {} (timeout={}s, headers={})",
        method, url, timeout, headers.len()
    );

    Ok(ToolResult {
        output,
        success: true,
        error: None,
    })
}

// ----------------------------------------------------------
// 5. main 函数：测试多种场景
// ----------------------------------------------------------
// 对应 ZeroClaw 的调用层（Agent 调用 Tool 的 execute 方法后处理结果）。
// 使用 match 处理最终结果，不让错误 panic。

fn main() {
    println!("=== 场景 A：正常参数 ===");
    let args_ok = json!({
        "url": "https://example.com/api",
        "method": "POST",
        "timeout": 60,
        "headers": {
            "Authorization": "Bearer token123",
            "Content-Type": "application/json"
        }
    });
    // ★ match 处理最终结果（对应 ZeroClaw 的顶层调用）
    match execute(args_ok) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
    }

    println!("\n=== 场景 B：缺少必填参数 url ===");
    let args_missing = json!({
        "method": "GET"
    });
    match execute(args_missing) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
        //                         ^^^ 会打印 "Missing 'url' parameter"
    }

    println!("\n=== 场景 C：url 为空字符串 ===");
    let args_empty_url = json!({
        "url": ""
    });
    match execute(args_empty_url) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
        //                         ^^^ 会打印 "URL cannot be empty"
    }

    println!("\n=== 场景 D：url 协议不合法 ===");
    let args_bad_proto = json!({
        "url": "ftp://files.example.com/data"
    });
    match execute(args_bad_proto) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
        //                         ^^^ 会打印 "Only http:// and https:// URLs are allowed"
    }

    println!("\n=== 场景 E：HTTP method 不合法 ===");
    let args_bad_method = json!({
        "url": "https://example.com",
        "method": "CONNECT"
    });
    match execute(args_bad_method) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
    }

    println!("\n=== 场景 F：url 是数字而非字符串 ===");
    let args_wrong_type = json!({
        "url": 12345
    });
    match execute(args_wrong_type) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
        //                         ^^^ 会打印 "Missing 'url' parameter"
        //                         因为 .as_str() 对数字返回 None
    }

    println!("\n=== 场景 G：只有 url，其余用默认值 ===");
    let args_minimal = json!({
        "url": "https://example.com"
    });
    match execute(args_minimal) {
        Ok(result) => println!("  success={}, output={}", result.success, result.output),
        Err(e) => println!("  Error: {e}"),
        //                         ^^^ method 默认 GET，timeout 默认 30，headers 默认空
    }
}
```

## 逐模式解析

### 模式 1：Option 组合子链（最高频）

```rust
// ZeroClaw 源码：src/tools/shell.rs
let command = args.get("command")
    .and_then(|v| v.as_str())
    .ok_or_else(|| anyhow!("Missing 'command' parameter"))?;

// 本场景对应：
let url = args.get("url")
    .and_then(|v| v.as_str())
    .ok_or_else(|| anyhow!("Missing 'url' parameter"))?;
```

**数据流**：

```
args.get("url")          → Some(&Value::String("https://..."))  或  None
  .and_then(|v| v.as_str()) → Some("https://...")               或  None
  .ok_or_else(|| anyhow!()) → Ok("https://...")                 或  Err(anyhow::Error)
  ?                          → "https://..."                     或  函数提前返回 Err
```

**TypeScript 对照**：

```typescript
// TypeScript 版本——需要手动检查类型
const url = args?.url;
if (typeof url !== "string") {
  throw new Error("Missing 'url' parameter");
}

// Rust 的组合子链把类型检查和空值检查合并成一行
// .as_str() 同时检查了"值存在"和"值是字符串类型"
```

[来源: reference/source_error_handling_01.md - Option 组合子链]

### 模式 2：bail! 提前返回

```rust
// ZeroClaw 源码：src/tools/web_fetch.rs
if url_str.is_empty() {
    anyhow::bail!("URL cannot be empty");
}

// 本场景对应：
if url.is_empty() {
    bail!("URL cannot be empty");
}
```

`bail!("msg")` 等价于 `return Err(anyhow!("msg"))`。用于验证失败时立即退出函数。

**TypeScript 对照**：

```typescript
if (url === "") throw new Error("URL cannot be empty");
```

[来源: reference/source_error_handling_01.md - bail! 提前返回]

### 模式 3：ensure! 条件断言

```rust
// ZeroClaw 源码：src/security/secrets.rs
anyhow::ensure!(blob.len() > NONCE_LEN, "Encrypted value too short");

// 本场景对应：
ensure!(
    url.starts_with("http://") || url.starts_with("https://"),
    "Only http:// and https:// URLs are allowed, got: {url}"
);
```

`ensure!(condition, "msg")` 等价于 `if !condition { bail!("msg") }`。更简洁。

**bail! vs ensure! 选择**：
- 条件简单、一行能写完 → `ensure!`
- 条件复杂、需要 if/else → `if ... { bail!(...) }`

[来源: reference/source_error_handling_01.md - ensure! 条件断言]

### 模式 4：unwrap_or 安全默认值

```rust
// ZeroClaw 源码：src/tools/shell.rs
let approved = args.get("approved")
    .and_then(|v| v.as_bool())
    .unwrap_or(false);

// 本场景对应：
let method = args.get("method")
    .and_then(|v| v.as_str())
    .unwrap_or("GET");
```

**关键区分**：
- 缺失是**错误**（必填参数）→ `.ok_or_else(|| anyhow!("..."))?`
- 缺失是**正常**（可选参数）→ `.unwrap_or(默认值)`

[来源: reference/source_error_handling_01.md - 安全默认值]

### 模式 5：match 处理最终结果

```rust
// ZeroClaw 源码：src/tools/web_fetch.rs
let url = match self.validate_url(url) {
    Ok(v) => v,
    Err(e) => return Ok(ToolResult { success: false, error: Some(e.to_string()), .. }),
};

// 本场景对应（简化版）：
match execute(args) {
    Ok(result) => println!("success={}, output={}", result.success, result.output),
    Err(e) => println!("Error: {e}"),
}
```

**规则**：中间层用 `?` 传播，顶层用 `match` 决定成功/失败的处理方式。

[来源: reference/source_error_handling_01.md - match 模式匹配 Result]

## 小结

本场景覆盖了 ZeroClaw Tool 参数处理的 5 个核心模式：

| 模式 | 用途 | 一句话记忆 |
|------|------|-----------|
| `.get().and_then().ok_or_else()?` | 必填参数提取 | "取值 → 转型 → 报错 → 传播" |
| `.get().and_then().unwrap_or()` | 可选参数提取 | "取值 → 转型 → 默认值" |
| `bail!("msg")` | 主动返回错误 | "验证不通过，立即退出" |
| `ensure!(cond, "msg")` | 条件断言 | "一行 if + bail" |
| `match result { Ok/Err }` | 顶层结果处理 | "最终决定成功/失败怎么办" |

**核心原则**：必填参数用 `ok_or_else` + `?`，可选参数用 `unwrap_or`，验证用 `bail!`/`ensure!`，顶层用 `match`。
