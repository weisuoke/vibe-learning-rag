# 核心概念07：并行执行与状态聚合

> 多个节点并行执行时的状态管理

## 概念定义

**并行执行允许多个节点同时运行，状态聚合机制通过Reducer自动合并并行节点的状态更新，确保数据一致性。**

## 并行执行基础

### 1. 并行节点定义

```python
from typing import TypedDict, Annotated
import operator
from langgraph.graph import StateGraph

class State(TypedDict):
    messages: Annotated[list, operator.add]
    results: Annotated[list, operator.add]

graph = StateGraph(State)

# 定义并行节点
def node1(state: State) -> dict:
    return {"messages": ["node1 executed"]}

def node2(state: State) -> dict:
    return {"messages": ["node2 executed"]}

def node3(state: State) -> dict:
    return {"messages": ["node3 executed"]}

graph.add_node("node1", node1)
graph.add_node("node2", node2)
graph.add_node("node3", node3)

# 并行执行
graph.add_edge("START", "node1")
graph.add_edge("START", "node2")
graph.add_edge("START", "node3")
```

### 2. 并行执行流程

```
START
  ├─> node1 ─┐
  ├─> node2 ─┤
  └─> node3 ─┴─> 聚合状态 -> END
```

## 状态聚合机制

### 1. Reducer聚合

```python
class State(TypedDict):
    messages: Annotated[list, operator.add]

# 并行执行
def node1(state: State) -> dict:
    return {"messages": ["a"]}

def node2(state: State) -> dict:
    return {"messages": ["b"]}

def node3(state: State) -> dict:
    return {"messages": ["c"]}

# StateGraph聚合流程：
# 1. 收集所有更新: [["a"], ["b"], ["c"]]
# 2. 依次应用Reducer:
#    [] + ["a"] = ["a"]
#    ["a"] + ["b"] = ["a", "b"]
#    ["a", "b"] + ["c"] = ["a", "b", "c"]
# 3. 最终状态: {"messages": ["a", "b", "c"]}
```

### 2. 聚合顺序

```python
# StateGraph内部实现（简化）
def aggregate_parallel_updates(state, node_results):
    """聚合并行节点的更新"""
    # 1. 按节点执行顺序收集更新
    updates_by_field = {}
    for result in node_results:
        for key, value in result.items():
            if key not in updates_by_field:
                updates_by_field[key] = []
            updates_by_field[key].append(value)

    # 2. 对每个字段应用Reducer
    for key, values in updates_by_field.items():
        channel = channels[key]
        for value in values:
            channel.update(value)

    return state
```

**注意**：聚合顺序可能不确定，取决于节点完成顺序。

## 并行模式

### 1. Fan-out模式

```python
# 一个节点触发多个并行节点
graph.add_edge("start_node", "parallel_node1")
graph.add_edge("start_node", "parallel_node2")
graph.add_edge("start_node", "parallel_node3")

# 流程：
# start_node -> [parallel_node1, parallel_node2, parallel_node3]
```

### 2. Fan-in模式

```python
# 多个并行节点汇聚到一个节点
graph.add_edge("parallel_node1", "merge_node")
graph.add_edge("parallel_node2", "merge_node")
graph.add_edge("parallel_node3", "merge_node")

# 流程：
# [parallel_node1, parallel_node2, parallel_node3] -> merge_node
```

### 3. 完整并行流程

```python
# Fan-out + Fan-in
graph.add_edge("start", "parallel1")
graph.add_edge("start", "parallel2")
graph.add_edge("start", "parallel3")

graph.add_edge("parallel1", "merge")
graph.add_edge("parallel2", "merge")
graph.add_edge("parallel3", "merge")

# 流程：
# start -> [parallel1, parallel2, parallel3] -> merge
```

```python
class State(TypedDict):
    data: list
    processed: Annotated[list, operator.add]

    chunk = state["data"][:100]

    return {"processed": result}
```

## 并发控制

### 1. 无Reducer的字段

```python
class State(TypedDict):
    # 无Reducer，并行更新会导致不确定结果
    status: str

def node1(state: State) -> dict:
    return {"status": "node1"}

def node2(state: State) -> dict:
    return {"status": "node2"}

# 并行执行后，status可能是"node1"或"node2"
# 取决于哪个节点最后完成
```

**建议**：并行节点避免更新无Reducer的字段。

### 2. 使用Reducer确保一致性

```python
class State(TypedDict):
    # 使用Reducer确保一致性
    logs: Annotated[list, operator.add]

def node1(state: State) -> dict:
    return {"logs": ["node1 log"]}

def node2(state: State) -> dict:
    return {"logs": ["node2 log"]}

# 并行执行后，logs包含所有日志
# logs = ["node1 log", "node2 log"]（顺序可能不同）
```

### 2. 资源管理

```python
# 限制并行度
from concurrent.futures import ThreadPoolExecutor

class State(TypedDict):
    tasks: list
    results: Annotated[list, operator.add]

def process_with_limit(state: State) -> dict:
    """限制并行任务数量"""
    with ThreadPoolExecutor(max_workers=5) as executor:
        futures = [executor.submit(process_task, task)
                   for task in state["tasks"]]
        results = [f.result() for f in futures]
    return {"results": results}
```

## 错误处理

### 1. 部分失败

```python
def node1(state: State) -> dict:
    try:
        result = risky_operation()
        return {"results": [result]}
    except Exception as e:
        return {"errors": [str(e)]}

def node2(state: State) -> dict:
    try:
        result = another_operation()
        return {"results": [result]}
    except Exception as e:
        return {"errors": [str(e)]}

# 即使node1失败，node2仍可成功
# 错误和结果分别聚合
```

### 2. 全部失败处理

```python
class State(TypedDict):
    results: Annotated[list, operator.add]
    errors: Annotated[list, operator.add]

def check_results(state: State) -> dict:
    """检查并行执行结果"""
    if not state["results"] and state["errors"]:
        # 所有节点都失败
        return {"status": "all_failed"}
    elif state["errors"]:
        # 部分失败
        return {"status": "partial_success"}
    else:
        # 全部成功
        return {"status": "success"}
```

## 最佳实践

### 1. 独立节点

```python
# ✓ 节点之间独立，适合并行
def node1(state: State) -> dict:
    return {"results": [process_independently()]}

def node2(state: State) -> dict:
    return {"results": [another_independent_process()]}

# ❌ 节点之间有依赖，不适合并行
def node1(state: State) -> dict:
    return {"temp_data": "..."}

def node2(state: State) -> dict:
    # 依赖node1的temp_data
    return {"results": [process(state["temp_data"])]}
```

### 2. 使用Reducer

```python
# ✓ 使用Reducer聚合结果
class State(TypedDict):
    results: Annotated[list, operator.add]

# ❌ 无Reducer，结果不确定
class State(TypedDict):
    result: dict  # 并行更新会覆盖
```

### 3. 错误隔离

```python
# ✓ 每个节点独立处理错误
def node(state: State) -> dict:
    try:
        result = operation()
        return {"results": [result]}
    except Exception as e:
        return {"errors": [str(e)]}

# ❌ 错误传播到其他节点
def node(state: State) -> dict:
    result = risky_operation()  # 可能抛出异常
    return {"results": [result]}
```

## 总结

**并行执行与状态聚合的核心**：
- 多个节点同时执行
- Reducer自动聚合更新
- 确保数据一致性
- 提高执行效率

**关键点**：
- 节点之间应该独立
- 使用Reducer确保一致性
- 注意聚合顺序的不确定性
- 独立处理错误
