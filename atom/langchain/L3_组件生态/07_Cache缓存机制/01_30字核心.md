# 30字核心

**LangChain缓存机制通过存储LLM调用和Embedding计算结果，避免重复计算，降低成本30-50%，提升响应速度20-100倍。**

---

## 核心要点拆解

### 1. 两种缓存类型

**LLM缓存**：
- 缓存LLM的调用结果（prompt → response）
- 全局配置，自动生效
- 支持多种后端（InMemory、Redis、SQLite等）

**Embedding缓存**：
- 缓存Embedding的计算结果（text → vector）
- 使用包装器模式（CacheBackedEmbeddings）
- 避免重复计算相同文本的向量

### 2. 核心价值

**成本节省**：
- 减少API调用次数
- 降低30-50%的运营成本
- 特别适合高频重复查询场景

**性能提升**：
- 缓存命中速度提升20-100倍
- 从秒级降至毫秒级
- 显著改善用户体验

### 3. 工作原理

**缓存键设计**：
- LLM缓存：`(prompt, llm_string)` 二元组
- Embedding缓存：文本内容的哈希值
- 确保配置一致性才命中缓存

**缓存流程**：
1. 查询缓存（lookup）
2. 命中返回，未命中计算
3. 更新缓存（update）
4. 返回结果

## 一句话记忆法

**缓存 = 用空间换时间，存储计算结果避免重复，成本降一半速度快百倍。**

---

## 快速验证

```python
from langchain_core.caches import InMemoryCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI
import time

# 启用缓存
set_llm_cache(InMemoryCache())

llm = ChatOpenAI(model="gpt-4o-mini")

# 首次调用（无缓存）
start = time.time()
response1 = llm.invoke("What is LangChain?")
print(f"首次调用耗时: {time.time() - start:.2f}秒")

# 再次调用（缓存命中）
start = time.time()
response2 = llm.invoke("What is LangChain?")
print(f"缓存命中耗时: {time.time() - start:.2f}秒")

# 输出示例：
# 首次调用耗时: 2.34秒
# 缓存命中耗时: 0.01秒  ← 速度提升200倍+
```

## 与其他概念的关系

**vs Memory（记忆管理）**：
- Memory：管理对话历史，维护上下文
- Cache：缓存计算结果，避免重复调用
- 两者互补，不冲突

**vs 批处理**：
- 批处理：一次性处理多个请求，降低成本
- Cache：存储结果避免重复计算
- 可以结合使用
