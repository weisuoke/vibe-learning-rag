## 核心知识点

### 1. 最常用的分块器：RecursiveCharacterTextSplitter

**为什么选择它？**
- LangChain 官方推荐的默认分块器
- 平衡了语义完整性和性能
- 适用于 90% 的文本分块场景
- 支持 20+ 种编程语言

**基本用法：**

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 创建分块器
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,        # 每块最大 1000 字符
    chunk_overlap=200,      # 块之间重叠 200 字符
    add_start_index=True    # 添加起始索引（便于溯源）
)

# 分块文本
text = "很长的文本内容..."
chunks = text_splitter.split_text(text)

print(f"分块数量: {len(chunks)}")
print(f"第一块: {chunks[0][:100]}...")
```

---

### 2. 两个核心参数

#### chunk_size（块大小）

**作用**：控制每个块的最大大小

**推荐配置：**
```python
# 通用场景
chunk_size = 1000  # 1000 characters

# 成本优化场景
chunk_size = 512   # 512 tokens（使用 from_tiktoken_encoder）

# 大文档场景
chunk_size = 1500  # 1500 characters
```

**选择原则：**
- 根据 LLM 的 context window 调整
- 考虑文档的平均段落长度
- 通过实验找到最佳值

#### chunk_overlap（块重叠）

**作用**：在块之间保持重叠，避免切断上下文

**推荐配置：**
```python
# 通常设置为 chunk_size 的 15-20%
chunk_overlap = 200  # chunk_size = 1000 时

# 最小值
chunk_overlap = 100  # 保证基本连续性

# 最大值
chunk_overlap = 300  # chunk_size 的 30%
```

**为什么需要重叠？**
- 保持上下文连续性
- 避免在句子或段落中间切断
- 提高检索召回率

### 3. 完整的 RAG 分块管道

**场景**：构建一个文档问答系统

```python
"""
完整的 RAG 文档分块管道
演示：文档加载 → 分块 → 向量化 → 检索 → 生成
"""

import os
from dotenv import load_dotenv

# LangChain 组件
from langchain_community.document_loaders import TextLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma
from langchain_openai import ChatOpenAI
from langchain.chains import RetrievalQA

# 加载环境变量
load_dotenv()

# ===== 1. 加载文档 =====
print("=== 步骤 1: 加载文档 ===")

# 创建示例文档
sample_text = """
LangChain 是一个用于开发由语言模型驱动的应用程序的框架。
它提供了一系列工具和组件，帮助开发者构建复杂的 AI 应用。

TextSplitter 是 LangChain 的核心组件之一。
它负责将长文本分割成适合 LLM 处理的小块。
这对于 RAG（检索增强生成）应用至关重要。

RecursiveCharacterTextSplitter 是最常用的分块器。
它通过递归尝试多个分隔符来保持语义完整性。
推荐配置是 chunk_size=1000 和 chunk_overlap=200。

在实际应用中，合理的分块策略可以显著提高检索质量。
chunk_size 太小会导致上下文不足，太大会降低检索精度。
chunk_overlap 确保重要信息不会在块边界处丢失。
"""

# 保存为临时文件
with open("/tmp/langchain_doc.txt", "w", encoding="utf-8") as f:
    f.write(sample_text)

# 加载文档
loader = TextLoader("/tmp/langchain_doc.txt", encoding="utf-8")
documents = loader.load()

print(f"加载文档数量: {len(documents)}")
print(f"文档内容长度: {len(documents[0].page_content)} 字符")

# ===== 2. 文本分块 =====
print("\n=== 步骤 2: 文本分块 ===")

# 创建分块器
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,         # 为演示设置较小的值
    chunk_overlap=50,       # 25% 重叠
    add_start_index=True    # 添加起始索引
)

# 分块文档
splits = text_splitter.split_documents(documents)

print(f"分块数量: {len(splits)}")
print("\n前 3 个块的内容：")
for i, split in enumerate(splits[:3]):
    print(f"\n块 {i + 1}:")
    print(f"  内容: {split.page_content[:80]}...")
    print(f"  元数据: {split.metadata}")

# ===== 3. 向量化和存储 =====
print("\n=== 步骤 3: 向量化和存储 ===")

# 创建 Embedding 模型
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small"
)

# 创建向量存储
vectorstore = Chroma.from_documents(
    documents=splits,
    embedding=embeddings,
    collection_name="langchain_docs"
)

print(f"向量存储创建成功，包含 {len(splits)} 个文档块")

# ===== 4. 检索测试 =====
print("\n=== 步骤 4: 检索测试 ===")

# 创建检索器
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 3}  # 返回 top-3 相关块
)

# 测试查询
query = "什么是 RecursiveCharacterTextSplitter？"
print(f"查询: {query}")

# 检索相关文档
relevant_docs = retriever.get_relevant_documents(query)

print(f"\n检索到 {len(relevant_docs)} 个相关块：")
for i, doc in enumerate(relevant_docs):
    print(f"\n块 {i + 1}:")
    print(f"  内容: {doc.page_content[:100]}...")
    print(f"  相关度: 高")

# ===== 5. RAG 生成 =====
print("\n=== 步骤 5: RAG 生成 ===")

# 创建 LLM
llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0
)

# 创建 RAG 链
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=retriever,
    return_source_documents=True
)

# 执行查询
result = qa_chain.invoke({"query": query})

print(f"\n问题: {query}")
print(f"\n答案: {result['result']}")
print(f"\n使用的源文档数量: {len(result['source_documents'])}")

# ===== 6. 清理 =====
print("\n=== 步骤 6: 清理 ===")
import shutil
if os.path.exists("/tmp/langchain_doc.txt"):
    os.remove("/tmp/langchain_doc.txt")
print("临时文件已清理")

print("\n=== RAG 管道执行完成 ===")
```

### 4. 常见文档类型处理

#### 处理 PDF 文档

```python
from langchain_community.document_loaders import PyPDFLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 加载 PDF
loader = PyPDFLoader("document.pdf")
pages = loader.load()

# 分块
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
splits = text_splitter.split_documents(pages)

print(f"PDF 分块数量: {len(splits)}")
```

#### 处理 Markdown 文档

```python
from langchain_community.document_loaders import UnstructuredMarkdownLoader
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 加载 Markdown
loader = UnstructuredMarkdownLoader("README.md")
docs = loader.load()

# 分块
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
splits = text_splitter.split_documents(docs)

print(f"Markdown 分块数量: {len(splits)}")
```

### 5. Token 优化分块

**场景**：精确控制 token 数量，优化成本

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 使用 tiktoken 计数
text_splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
    model_name="gpt-4",
    chunk_size=512,      # 512 tokens
    chunk_overlap=100    # 100 tokens
)

text = "很长的文本内容..."
chunks = text_splitter.split_text(text)

# 验证 token 数量
import tiktoken
enc = tiktoken.encoding_for_model("gpt-4")

for i, chunk in enumerate(chunks):
    token_count = len(enc.encode(chunk))
    print(f"块 {i + 1}: {token_count} tokens")
```

## 常见问题

### Q1: 如何选择 chunk_size？

**A**: 根据以下因素决定：
1. **LLM 的 context window**：确保 chunk_size < context window
2. **文档类型**：
   - 短文档（< 1000 字符）：chunk_size = 500
   - 中等文档（1000-5000 字符）：chunk_size = 1000
   - 大文档（> 5000 字符）：chunk_size = 1500
3. **检索精度**：chunk_size 越小，检索越精确，但上下文越少
4. **实验验证**：通过基准测试找到最佳值

### Q2: chunk_overlap 设置多大合适？

**A**: 推荐设置为 chunk_size 的 15-20%：
- chunk_size = 1000 → chunk_overlap = 150-200
- chunk_size = 512 → chunk_overlap = 80-100
- 最小值：100 characters（保证基本连续性）
- 最大值：chunk_size 的 30%（避免过多重复）

### Q3: 如何处理中文文档？

**A**: RecursiveCharacterTextSplitter 原生支持中文：
```python
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,      # 中文字符数通常较少
    chunk_overlap=100,
    separators=["\n\n", "\n", "。", "！", "？", " ", ""]  # 中文分隔符
)
```

### Q4: 如何验证分块效果？

**A**: 使用以下方法：
```python
# 1. 检查分块数量
print(f"分块数量: {len(chunks)}")

# 2. 检查每块大小
for i, chunk in enumerate(chunks):
    print(f"块 {i + 1}: {len(chunk)} 字符")

# 3. 检查重叠部分
for i in range(len(chunks) - 1):
    overlap = set(chunks[i].split()) & set(chunks[i + 1].split())
    print(f"块 {i + 1} 和块 {i + 2} 的重叠词数: {len(overlap)}")

# 4. 可视化分块
for i, chunk in enumerate(chunks[:3]):
    print(f"\n块 {i + 1}:")
    print(chunk)
    print("-" * 50)
```

### Q5: 如何优化分块性能？

**A**: 使用以下技巧：
1. **批处理**：一次处理多个文档
2. **缓存分块器**：避免重复创建
3. **合适的 chunk_size**：避免过小（增加块数量）或过大（降低检索精度）
4. **使用 CharacterTextSplitter**：如果文本简单，使用更快的分块器
