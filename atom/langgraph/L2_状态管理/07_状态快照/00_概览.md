# 状态快照（State Snapshot）- 知识点概览

## 知识点简介

**状态快照是 LangGraph 在每个执行步骤自动捕获完整图状态的机制，通过 `StateSnapshot` 数据结构记录当前值、待执行节点、元数据等信息，支持状态查看、历史回溯和时间旅行调试。**

在 LangGraph 中，图的执行是多步骤的——每个节点处理完后，状态会发生变化。状态快照就像给每一步拍一张"照片"，记录下那一刻的完整状态。有了这些"照片"，你可以回看任意历史状态、从某个历史节点重新执行（重放），甚至修改历史状态后走一条全新的执行路径（分叉）。这对调试 AI Agent 的非确定性行为至关重要。

## 为什么需要状态快照？

### 没有快照时会发生什么？

```python
# 图执行完毕后，你只能看到最终结果
result = graph.invoke({"query": "什么是 RAG？"})
# result 只有最终状态，中间发生了什么？不知道。
# LLM 在第 3 步做了什么决策？无法追溯。
# 想从第 2 步重新执行？做不到。
```

AI Agent 的执行具有非确定性——同样的输入，LLM 可能给出不同的输出。当结果不符合预期时，你需要知道"哪一步出了问题"，而不是只看到最终的错误结果。

### 状态快照解决三个核心问题

| 问题 | 说明 | 示例 |
|------|------|------|
| **执行不透明** | 多步骤执行过程中间状态不可见 | Agent 做了 5 步决策，只看到最终结果 |
| **调试困难** | 非确定性行为难以复现 | LLM 上次返回了正确答案，这次却错了 |
| **无法回溯** | 想从某个历史状态重新开始 | 第 3 步的检索结果很好，但第 4 步的生成出了问题 |

## 核心概念速览

本知识点包含 6 个核心概念：

| 编号 | 概念 | 一句话说明 | 重要程度 |
|------|------|-----------|----------|
| 1 | StateSnapshot 数据结构 | NamedTuple，包含 values/next/config/metadata 等 8 个字段 | ★★★★★ |
| 2 | Checkpoint 存储格式 | 底层 TypedDict，包含 channel_values/channel_versions/versions_seen | ★★★★☆ |
| 3 | get_state() 获取快照 | 从 checkpointer 获取最新或指定 checkpoint 的状态快照 | ★★★★★ |
| 4 | get_state_history() 历史遍历 | 逆时间顺序遍历所有历史快照，支持 filter 和 limit | ★★★★☆ |
| 5 | update_state() 状态修改 | 手动创建新 checkpoint，遵循 reducer 规则 | ★★★★★ |
| 6 | 时间旅行机制 | Replay（重放）从历史点重新执行 + Fork（分叉）修改后走新路径 | ★★★★★ |

### 场景1 预览：基础快照操作

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver
from typing_extensions import TypedDict

class State(TypedDict):
    query: str
    answer: str

def answer_node(state: State) -> dict:
    return {"answer": f"关于 '{state['query']}' 的回答"}

builder = StateGraph(State)
builder.add_node("answer", answer_node)
builder.add_edge(START, "answer")
builder.add_edge("answer", END)

# 关键：必须配置 checkpointer 才能使用快照
graph = builder.compile(checkpointer=MemorySaver())

config = {"configurable": {"thread_id": "1"}}
result = graph.invoke({"query": "什么是 RAG？"}, config=config)

# 获取状态快照
snapshot = graph.get_state(config)
print(snapshot.values)      # {'query': '什么是 RAG？', 'answer': '关于...的回答'}
print(snapshot.next)        # () — 空元组表示执行已完成
print(snapshot.metadata)    # {'source': 'loop', 'step': 1, ...}
print(snapshot.created_at)  # '2026-02-27T...'
```

## 源码对应

本知识点涉及的 LangGraph 源码文件：

| 文件 | 说明 | 关键函数/类 |
|------|------|------------|
| `langgraph/types.py` | StateSnapshot、PregelTask、Interrupt 类型定义 | `StateSnapshot` |
| `langgraph/pregel/main.py` | get_state/get_state_history/_prepare_state_snapshot | `Pregel.get_state()` |
| `langgraph/pregel/_checkpoint.py` | Checkpoint 工具函数 | `empty_checkpoint()`, `create_checkpoint()` |
| `langgraph/checkpoint/base/__init__.py` | Checkpoint、CheckpointMetadata、BaseCheckpointSaver | `Checkpoint`, `CheckpointTuple` |
| `langgraph/checkpoint/memory/__init__.py` | 内存存储实现 | `InMemorySaver` |

### 核心数据流

```
graph.invoke(input, config)
    ↓
每个节点执行后 → checkpointer.put(checkpoint)  ← 自动保存快照
    ↓
graph.get_state(config)
    ↓
checkpointer.get_tuple(config)  → CheckpointTuple
    ↓
_prepare_state_snapshot(config, CheckpointTuple)
    ├─ channels_from_checkpoint()  → 恢复 channel 值
    ├─ prepare_next_tasks()        → 确定下一步任务
    └─ return StateSnapshot        → 用户可读的快照
```
