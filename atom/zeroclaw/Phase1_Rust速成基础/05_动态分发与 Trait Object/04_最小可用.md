# 动态分发与 Trait Object - 最小可用

> **20% 核心知识，解决 80% 问题。** 掌握以下 5 个核心知识，就能在 ZeroClaw 源码中理解和使用动态分发。

---

## 4.1 Box\<dyn Trait\> 基础用法

**这是你最常用的动态分发形式。**

### 创建

把具体类型装进 `Box`，类型自动擦除为 `dyn Trait`：

```rust
// 定义 Trait
trait Tool: Send + Sync {
    fn name(&self) -> &str;
    fn execute(&self, input: &str) -> String;
}

// 具体类型实现 Trait
struct ShellTool;
impl Tool for ShellTool {
    fn name(&self) -> &str { "shell" }
    fn execute(&self, input: &str) -> String {
        format!("执行命令: {}", input)
    }
}

// 创建 Box<dyn Tool> —— 类型从 ShellTool 擦除为 dyn Tool
let tool: Box<dyn Tool> = Box::new(ShellTool);
```

### 调用方法

和普通对象一样调用，编译器自动通过 vtable 找到正确的方法：

```rust
println!("{}", tool.name());           // "shell"
println!("{}", tool.execute("ls -la")); // "执行命令: ls -la"
// 看起来和直接调用 ShellTool 的方法完全一样
// 区别在于：这里是通过 vtable 间接调用（运行时查找）
```

### 存入集合

**异构集合**是 `Box<dyn Trait>` 最高频的用途——不同类型放进同一个 `Vec`：

```rust
struct FileReadTool;
impl Tool for FileReadTool {
    fn name(&self) -> &str { "file_read" }
    fn execute(&self, input: &str) -> String {
        format!("读取文件: {}", input)
    }
}

// Vec<Box<dyn Tool>> —— 不同类型的 Tool 放在一起
let tools: Vec<Box<dyn Tool>> = vec![
    Box::new(ShellTool),       // ShellTool 类型
    Box::new(FileReadTool),    // FileReadTool 类型
];

// 遍历调用——每个 tool 自动调用自己的实现
for tool in &tools {
    println!("[{}] {}", tool.name(), tool.execute("test"));
}
// 输出:
// [shell] 执行命令: test
// [file_read] 读取文件: test
```

```typescript
// TypeScript 等价 —— 天然支持，不需要 Box
const tools: Tool[] = [new ShellTool(), new FileReadTool()];
tools.forEach(t => console.log(`[${t.name()}] ${t.execute("test")}`));
```

**记住：** 在 Rust 中，`Vec<Box<dyn Tool>>` = TypeScript 的 `Tool[]`。

## 4.2 工厂函数模式

**这是 ZeroClaw 中最常见的模式：根据配置字符串返回不同的实现。**

### 基本工厂

```rust
use anyhow::{bail, Result};

fn create_tool(name: &str) -> Result<Box<dyn Tool>> {
    match name {
        "shell"     => Ok(Box::new(ShellTool)),
        "file_read" => Ok(Box::new(FileReadTool)),
        _           => bail!("未知工具: {}", name),
    }
}

// 使用
let tool = create_tool("shell")?;
println!("{}", tool.name());  // "shell"
```

### ZeroClaw 真实场景：Provider 工厂

```rust
fn create_provider(config: &Config) -> Result<Box<dyn Provider>> {
    match config.provider.as_str() {
        "openai"    => Ok(Box::new(OpenAIProvider::new(&config.api_key)?)),
        "anthropic" => Ok(Box::new(AnthropicProvider::new(&config.api_key)?)),
        "ollama"    => Ok(Box::new(OllamaProvider::new(&config.endpoint)?)),
        _           => bail!("未知 Provider: {}", config.provider),
    }
}
```

### ZeroClaw 真实场景：Memory 工厂

```rust
fn create_memory(backend: &str) -> Result<Box<dyn Memory>> {
    match backend {
        "sqlite"   => Ok(Box::new(SqliteMemory::new()?)),
        "postgres" => Ok(Box::new(PostgresMemory::new()?)),
        "markdown" => Ok(Box::new(MarkdownMemory::new())),
        "none"     => Ok(Box::new(NoneMemory::new())),
        _          => bail!("未知 Memory 后端: {}", backend),
    }
}
```

**模式总结：** `match 配置字符串 => Ok(Box::new(具体实现))` —— 这一个模式覆盖了 ZeroClaw 中 80% 的动态分发使用场景。

```typescript
// TypeScript 等价
function createProvider(config: Config): Provider {
  switch (config.provider) {
    case "openai":    return new OpenAIProvider(config.apiKey);
    case "anthropic": return new AnthropicProvider(config.apiKey);
    default:          throw new Error(`未知 Provider: ${config.provider}`);
  }
}
```

## 4.3 Arc\<dyn Trait\> 共享

**当多个组件需要使用同一个 trait object 时，用 `Arc<dyn Trait>`。**

### 核心场景：共享 Memory

在 ZeroClaw 中，`MemoryStoreTool`、`MemoryRecallTool`、`MemoryForgetTool` 三个工具需要操作同一个数据库连接。不能创建三个独立的连接——需要共享。

```rust
use std::sync::Arc;

// 创建一个 Memory 实例
let memory: Arc<dyn Memory> = Arc::new(SqliteMemory::new()?);

// 多个工具共享同一个 Memory
// clone() 只增加引用计数（原子操作，~1ns），不复制数据
let store_tool = MemoryStoreTool::new(memory.clone());   // 引用计数: 2
let recall_tool = MemoryRecallTool::new(memory.clone()); // 引用计数: 3
let forget_tool = MemoryForgetTool::new(memory);         // 引用计数: 4（转移）

// 三个工具指向堆上同一个 SqliteMemory 实例
// 最后一个 Arc 被销毁时，SqliteMemory 才会被释放
```

### Box vs Arc 速查

```
Box<dyn Trait>  → 只有一个主人     → 用于 Agent 独占的 Provider
Arc<dyn Trait>  → 可以有多个主人   → 用于多个 Tool 共享的 Memory
```

| 场景 | 选择 | ZeroClaw 例子 |
|------|------|---------------|
| 结构体独占字段 | `Box<dyn Trait>` | `provider: Box<dyn Provider>` |
| 多组件共享 | `Arc<dyn Trait>` | `memory: Arc<dyn Memory>` |
| 跨异步任务 | `Arc<dyn Trait>` | `runtime: Arc<dyn RuntimeAdapter>` |
| 工厂函数返回 | `Box<dyn Trait>` | `create_memory() -> Box<dyn Memory>` |

```typescript
// TypeScript 不需要区分 —— JS 的引用天然就是共享的
const memory = new SqliteMemory();
const store = new MemoryStoreTool(memory);   // 共享
const recall = new MemoryRecallTool(memory); // 共享
// JS 有垃圾回收器，不需要手动管理所有权
```

**记住：** 需要 `.clone()` 传给多个地方 → `Arc`。只有一个所有者 → `Box`。

## 4.4 对象安全速查

**不是所有 Trait 都能变成 `dyn Trait`。** 编译器会检查你的 Trait 是否满足"对象安全"规则。

### ✅ 能用 dyn 的写法

```rust
trait Tool: Send + Sync {
    fn name(&self) -> &str;                // ✅ 方法用 &self
    fn execute(&mut self, input: &str);    // ✅ 方法用 &mut self
    fn description(&self) -> String {      // ✅ 有默认实现也可以
        String::from("no description")
    }
}
```

### ❌ 不能用 dyn 的写法

```rust
trait NotObjectSafe {
    fn process<T>(&self, input: T);    // ❌ 泛型方法（vtable 无法存储）
    fn clone(&self) -> Self;           // ❌ 返回 Self（大小不确定）
    fn create() -> Self;              // ❌ 没有 self 参数（不知道调用谁的）
}
```

### 速查表

| 写法 | 能否 dyn？ | 原因 |
|------|-----------|------|
| `fn name(&self) -> &str` | ✅ 可以 | 标准方法签名 |
| `fn execute(&mut self, input: &str)` | ✅ 可以 | 可变借用也行 |
| `fn process<T>(&self, input: T)` | ❌ 不行 | 泛型方法无法放入 vtable |
| `fn clone(&self) -> Self` | ❌ 不行 | 返回 Self 需要知道具体大小 |
| `fn create() -> Self` | ❌ 不行 | 无 self 参数 |
| `async fn execute(&self)` | ⚠️ 需要 `#[async_trait]` | 原生 async 在 trait 中有限制 |

### 处理 async 方法

ZeroClaw 的所有核心 Trait 都有 async 方法，统一用 `#[async_trait]` 宏解决：

```rust
use async_trait::async_trait;

#[async_trait]
trait Provider: Send + Sync {
    async fn chat(&self, prompt: &str) -> Result<String>;
    // #[async_trait] 宏会把这行转换为：
    // fn chat<'a>(&'a self, prompt: &'a str) -> Pin<Box<dyn Future<Output = Result<String>> + Send + 'a>>;
    // 这样就能放进 vtable 了
}
```

**记住三条就够了：**
1. 方法必须有 `&self` 或 `&mut self`
2. 不能有泛型参数，不能返回 `Self`
3. async 方法用 `#[async_trait]` 宏

## 4.5 静态 vs 动态：一句话决策

### 默认用泛型（静态分发）

```rust
// 静态分发：编译时确定，零开销，可内联
fn process<T: Tool>(tool: &T) {
    println!("{}", tool.name());
    // 编译器为每种 T 生成一份专用代码
    // ShellTool → 直接调用 ShellTool::name()
    // FileReadTool → 直接调用 FileReadTool::name()
}
```

### 两种情况切换到 dyn（动态分发）

**情况 1：异构集合** —— 不同类型放同一个 Vec

```rust
// 泛型做不到：Vec<T> 中的 T 只能是一种类型
// let tools: Vec<???> = vec![ShellTool, FileReadTool]; // 类型不同，编译错误

// 必须用 dyn：
let tools: Vec<Box<dyn Tool>> = vec![
    Box::new(ShellTool),
    Box::new(FileReadTool),
];
```

**情况 2：运行时选择** —— 编译时不知道用哪个实现

```rust
// 泛型做不到：match 的每个分支返回不同类型
fn create_tool(name: &str) -> Box<dyn Tool> {  // 必须用 dyn
    match name {
        "shell" => Box::new(ShellTool),          // 类型是 ShellTool
        "file"  => Box::new(FileReadTool),       // 类型是 FileReadTool
        _ => panic!(),
    }
}
```

### 决策流程图

```
需要把不同类型放进同一个集合吗？
  └─ 是 → 用 dyn（异构集合）
  └─ 否 ↓

需要运行时决定使用哪个实现吗？
  └─ 是 → 用 dyn（工厂模式）
  └─ 否 ↓

默认用泛型（静态分发，零开销）
```
