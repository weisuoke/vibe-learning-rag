# 核心概念 6：异步工具支持

> 本文档是【自定义Tool开发】知识点的第6个核心概念，讲解 LangChain 工具的异步执行机制

---

## 概念定义

**异步工具支持**是指 LangChain 工具能够以非阻塞方式执行 I/O 密集型操作（如 API 调用、数据库查询），通过 Python 的 `async/await` 机制提升并发性能。

在 Agent 系统中，工具经常需要调用外部服务（天气 API、数据库、搜索引擎），如果使用同步方式，每个工具调用都会阻塞整个执行流程。异步工具允许多个工具并发执行，显著提升响应速度。

---

## 为什么需要异步工具？

### 问题场景

假设一个 Agent 需要同时调用 3 个工具：
- 天气 API（耗时 2 秒）
- 数据库查询（耗时 1 秒）
- 搜索引擎（耗时 3 秒）

**同步执行**：总耗时 = 2 + 1 + 3 = **6 秒**
**异步执行**：总耗时 = max(2, 1, 3) = **3 秒**

在生产环境中，这种性能差异会直接影响用户体验。

### RAG/Agent 中的应用

1. **RAG 检索阶段**：同时查询多个向量数据库
2. **Agent 工具调用**：并发执行多个独立工具
3. **批量处理**：同时处理多个用户请求
4. **外部 API 集成**：避免阻塞等待 HTTP 响应

---

## 三种异步工具实现方式

### 方式 1：@tool 装饰器 + async 函数（推荐）

**最简单的方式**，直接在函数定义前加 `async` 关键字。

```python
from langchain.tools import tool
import httpx

@tool
async def get_weather(city: str) -> str:
    """Get current weather for a city.

    Args:
        city: City name (e.g., "Beijing", "New York")
    """
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"https://api.weather.com/v1/current?q={city}"
        )
        data = response.json()
        return f"Weather in {city}: {data['temp']}°C, {data['condition']}"

# 异步调用
import asyncio
result = asyncio.run(get_weather.ainvoke({"city": "Beijing"}))
print(result)
```

**关键点**：
- 函数定义使用 `async def`
- 内部使用 `await` 调用异步操作
- 调用时使用 `.ainvoke()` 而非 `.invoke()`
- LangChain 自动识别异步函数

[来源: reference/source_tool_development_01.md | LangChain 源码分析]

### 方式 2：StructuredTool + coroutine 参数

**适合包装现有异步函数**，显式指定异步实现。

```python
from langchain.tools import StructuredTool
from pydantic import BaseModel, Field
import httpx

class WeatherInput(BaseModel):
    city: str = Field(description="City name")
    units: str = Field(default="metric", description="Temperature units")

async def fetch_weather(city: str, units: str = "metric") -> str:
    """Async function to fetch weather data."""
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"https://api.weather.com/v1/current",
            params={"q": city, "units": units}
        )
        data = response.json()
        return f"{city}: {data['temp']}° ({units})"

# 创建工具时指定 coroutine
weather_tool = StructuredTool.from_function(
    name="get_weather",
    description="Get weather information",
    coroutine=fetch_weather,  # 异步函数
    args_schema=WeatherInput,
)

# 异步调用
result = await weather_tool.ainvoke({"city": "Shanghai", "units": "celsius"})
```

**关键点**：
- 使用 `coroutine` 参数而非 `func`
- 可以同时提供 `func`（同步）和 `coroutine`（异步）
- LangChain 会根据调用方式自动选择

[来源: reference/source_tool_development_01.md | StructuredTool 源码]

### 方式 3：BaseTool + _arun 方法

**最大控制权**，适合需要状态管理的复杂工具。

```python
from langchain.tools import BaseTool
from langchain_core.runnables import RunnableConfig
from pydantic import BaseModel, Field
from typing import Any
import httpx

class WeatherInput(BaseModel):
    city: str = Field(description="City name")

class WeatherTool(BaseTool):
    name = "weather_tool"
    description = "Get weather information for a city"
    args_schema = WeatherInput

    # 工具状态（如 API 密钥、连接池）
    api_key: str = ""
    client: Any = None

    def __init__(self, api_key: str, **kwargs):
        super().__init__(**kwargs)
        self.api_key = api_key
        # 初始化异步 HTTP 客户端
        self.client = httpx.AsyncClient(
            headers={"Authorization": f"Bearer {api_key}"}
        )

    def _run(self, city: str, config: RunnableConfig = None) -> str:
        """同步版本（回退方案）"""
        # 注意：这里使用同步客户端
        with httpx.Client() as client:
            response = client.get(f"https://api.weather.com/v1/current?q={city}")
            return response.text

    async def _arun(self, city: str, config: RunnableConfig = None) -> str:
        """异步版本（推荐）"""
        response = await self.client.get(
            f"https://api.weather.com/v1/current",
            params={"q": city}
        )
        data = response.json()
        return f"Weather in {city}: {data['temp']}°C"

    async def __del__(self):
        """清理资源"""
        if self.client:
            await self.client.aclose()

# 使用
tool = WeatherTool(api_key="your_api_key")
result = await tool.ainvoke({"city": "Tokyo"})
```

**关键点**：
- 必须实现 `_run()` 方法（同步版本）
- 可选实现 `_arun()` 方法（异步版本）
- 如果只提供 `_run()`，LangChain 会自动在线程池中执行
- 适合需要初始化/清理资源的场景

[来源: reference/source_tool_development_01.md | BaseTool 源码]

## 自动回退机制

### LangChain 的智能处理

如果工具只提供同步实现，LangChain 会自动使用线程池执行：

```python
# 源码逻辑（简化版）
async def ainvoke(self, input, config=None, **kwargs):
    if not self.coroutine:
        # 没有异步实现，回退到线程池
        return await run_in_executor(config, self.invoke, input, config, **kwargs)

    # 有异步实现，直接调用
    return await super().ainvoke(input, config, **kwargs)
```

**这意味着**：
- 所有工具都可以用 `.ainvoke()` 调用
- 同步工具不会阻塞事件循环（在线程池中执行）
- 但性能不如原生异步实现

[来源: reference/source_tool_development_01.md | StructuredTool.ainvoke 实现]

## 实战示例：并发调用多个工具

### 场景：同时查询天气和新闻

```python
"""
演示：并发执行多个异步工具
场景：用户询问"北京的天气和最新新闻"
"""

import asyncio
import httpx
from langchain.tools import tool
from typing import List

# ===== 1. 定义异步工具 =====

@tool
async def get_weather(city: str) -> str:
    """Get current weather for a city."""
    async with httpx.AsyncClient() as client:
        # 模拟 API 调用（2秒延迟）
        await asyncio.sleep(2)
        return f"Weather in {city}: 22°C, Sunny"

@tool
async def get_news(city: str, limit: int = 3) -> str:
    """Get latest news for a city."""
    async with httpx.AsyncClient() as client:
        # 模拟 API 调用（3秒延迟）
        await asyncio.sleep(3)
        return f"Top {limit} news in {city}: [News1, News2, News3]"

@tool
async def get_traffic(city: str) -> str:
    """Get traffic information for a city."""
    async with httpx.AsyncClient() as client:
        # 模拟 API 调用（1秒延迟）
        await asyncio.sleep(1)
        return f"Traffic in {city}: Moderate congestion"

# ===== 2. 串行执行（慢）=====
print("=== 串行执行 ===")
import time

async def sequential_execution():
    start = time.time()

    weather = await get_weather.ainvoke({"city": "Beijing"})
    news = await get_news.ainvoke({"city": "Beijing", "limit": 3})
    traffic = await get_traffic.ainvoke({"city": "Beijing"})

    elapsed = time.time() - start
    print(f"Weather: {weather}")
    print(f"News: {news}")
    print(f"Traffic: {traffic}")
    print(f"Total time: {elapsed:.2f}s")  # 约 6 秒

asyncio.run(sequential_execution())

# ===== 3. 并发执行（快）=====
print("\n=== 并发执行 ===")

async def concurrent_execution():
    start = time.time()

    # 使用 asyncio.gather 并发执行
    results = await asyncio.gather(
        get_weather.ainvoke({"city": "Beijing"}),
        get_news.ainvoke({"city": "Beijing", "limit": 3}),
        get_traffic.ainvoke({"city": "Beijing"}),
    )

    elapsed = time.time() - start
    print(f"Weather: {results[0]}")
    print(f"News: {results[1]}")
    print(f"Traffic: {results[2]}")
    print(f"Total time: {elapsed:.2f}s")  # 约 3 秒（最慢的工具）

asyncio.run(concurrent_execution())

# ===== 4. 在 Agent 中使用 =====
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain_openai import ChatOpenAI
from langchain import hub

async def agent_with_async_tools():
    llm = ChatOpenAI(model="gpt-4", temperature=0)
    tools = [get_weather, get_news, get_traffic]

    prompt = hub.pull("hwchase17/openai-functions-agent")
    agent = create_openai_functions_agent(llm, tools, prompt)
    agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

    # Agent 会自动并发调用工具（如果 LLM 决定同时调用多个）
    result = await agent_executor.ainvoke({
        "input": "What's the weather, news, and traffic in Beijing?"
    })

    print(result["output"])

# asyncio.run(agent_with_async_tools())
```

**运行输出示例**：
```
=== 串行执行 ===
Weather: Weather in Beijing: 22°C, Sunny
News: Top 3 news in Beijing: [News1, News2, News3]
Traffic: Traffic in Beijing: Moderate congestion
Total time: 6.02s

=== 并发执行 ===
Weather: Weather in Beijing: 22°C, Sunny
News: Top 3 news in Beijing: [News1, News2, News3]
Traffic: Traffic in Beijing: Moderate congestion
Total time: 3.01s
```

**性能提升**：并发执行比串行快 **2 倍**！

## 异步工具的最佳实践

### 1. 何时使用异步？

**适合异步的场景**：
- ✅ HTTP API 调用
- ✅ 数据库查询
- ✅ 文件 I/O（大文件）
- ✅ 向量数据库检索
- ✅ 外部服务调用

**不适合异步的场景**：
- ❌ CPU 密集型计算（如大规模矩阵运算）
- ❌ 纯内存操作（如列表排序）
- ❌ 简单的字符串处理

### 2. 使用异步 HTTP 客户端

**推荐库**：`httpx`（支持 async/await）

```python
import httpx

# ✅ 推荐：异步客户端
async with httpx.AsyncClient() as client:
    response = await client.get("https://api.example.com")

# ❌ 避免：同步客户端（会阻塞事件循环）
import requests
response = requests.get("https://api.example.com")  # 阻塞！
```

### 3. 复用连接池

**不推荐**：每次调用都创建新客户端
```python
@tool
async def bad_tool(url: str) -> str:
    async with httpx.AsyncClient() as client:  # 每次都创建
        response = await client.get(url)
        return response.text
```

**推荐**：使用 BaseTool 管理连接池
```python
class GoodTool(BaseTool):
    name = "good_tool"
    description = "Efficient HTTP tool"

    client: httpx.AsyncClient = None

    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.client = httpx.AsyncClient(timeout=10.0)

    async def _arun(self, url: str, config: RunnableConfig = None) -> str:
        response = await self.client.get(url)
        return response.text

    async def __del__(self):
        if self.client:
            await self.client.aclose()
```

### 4. 错误处理

异步工具中的错误处理与同步工具相同：

```python
from langchain.tools import ToolException

@tool
async def safe_api_call(url: str) -> str:
    """Safe API call with error handling."""
    try:
        async with httpx.AsyncClient() as client:
            response = await client.get(url, timeout=5.0)
            response.raise_for_status()
            return response.text
    except httpx.TimeoutException:
        raise ToolException("API request timed out")
    except httpx.HTTPStatusError as e:
        raise ToolException(f"API returned error: {e.response.status_code}")
    except Exception as e:
        raise ToolException(f"Unexpected error: {str(e)}")
```

### 5. 超时控制

**始终设置超时**，避免工具无限等待：

```python
@tool
async def api_with_timeout(query: str) -> str:
    """API call with timeout."""
    async with httpx.AsyncClient(timeout=5.0) as client:  # 5秒超时
        response = await client.get(f"https://api.example.com?q={query}")
        return response.text

# 或使用 asyncio.wait_for
@tool
async def api_with_asyncio_timeout(query: str) -> str:
    """API call with asyncio timeout."""
    async with httpx.AsyncClient() as client:
        try:
            response = await asyncio.wait_for(
                client.get(f"https://api.example.com?q={query}"),
                timeout=5.0
            )
            return response.text
        except asyncio.TimeoutError:
            raise ToolException("Request timed out after 5 seconds")
```

## 与 RAG/Agent 的关系

### 在 RAG 系统中的应用

```python
from langchain.tools import tool
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings

@tool
async def rag_retrieval(query: str, top_k: int = 5) -> str:
    """Retrieve relevant documents from vector database.

    Args:
        query: User query
        top_k: Number of documents to retrieve
    """
    # 异步向量检索
    vectorstore = Chroma(
        embedding_function=OpenAIEmbeddings(),
        persist_directory="./chroma_db"
    )

    # 并发查询多个集合
    results = await asyncio.gather(
        vectorstore.asimilarity_search(query, k=top_k, filter={"source": "docs"}),
        vectorstore.asimilarity_search(query, k=top_k, filter={"source": "code"}),
    )

    docs = results[0] + results[1]
    return "\n\n".join([doc.page_content for doc in docs])
```

### 在 Agent 系统中的应用

```python
# Agent 自动并发调用工具
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain_openai import ChatOpenAI

async def create_async_agent():
    llm = ChatOpenAI(model="gpt-4")

        "input": "Compare weather in Beijing and Shanghai, and find related news"
    })

    return result
```

## 常见陷阱与解决方案

### 陷阱 1：在异步函数中使用同步库

**错误示例**：
```python
@tool
async def bad_tool(url: str) -> str:
    import requests  # 同步库
    response = requests.get(url)  # 阻塞事件循环！
    return response.text
```

**解决方案**：使用异步库或在线程池中执行
```python
@tool
async def good_tool(url: str) -> str:
    import httpx  # 异步库
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
        return response.text

# 或者在线程池中执行同步代码
import asyncio
from concurrent.futures import ThreadPoolExecutor

@tool
async def sync_in_thread(url: str) -> str:
    import requests
    loop = asyncio.get_event_loop()
    with ThreadPoolExecutor() as executor:
        response = await loop.run_in_executor(
            executor,
            requests.get,
            url
        )
        return response.text
```

### 陷阱 2：忘记使用 await

**错误示例**：
```python
@tool
async def bad_tool(city: str) -> str:
    result = get_weather(city)  # 忘记 await，返回 coroutine 对象！
    return result
```

**解决方案**：始终 await 异步调用
```python
@tool
async def good_tool(city: str) -> str:
    result = await get_weather(city)  # 正确
    return result
```

### 陷阱 3：混用同步和异步调用

**错误示例**：
```python
# 在同步代码中直接调用异步工具
result = get_weather.ainvoke({"city": "Beijing"})  # 返回 coroutine，未执行！
```

**解决方案**：使用 asyncio.run() 或在异步上下文中调用
```python
# 方式 1：使用 asyncio.run()
result = asyncio.run(get_weather.ainvoke({"city": "Beijing"}))

# 方式 2：在异步函数中调用
async def main():
    result = await get_weather.ainvoke({"city": "Beijing"})
    print(result)

asyncio.run(main())
```

## 性能对比

### 基准测试

```python
import time
import asyncio
from langchain.tools import tool

@tool
def sync_tool(n: int) -> str:
    """Synchronous tool (simulates 1s API call)."""
    time.sleep(1)
    return f"Result {n}"

@tool
async def async_tool(n: int) -> str:
    """Asynchronous tool (simulates 1s API call)."""
    await asyncio.sleep(1)
    return f"Result {n}"

# 测试：调用 5 个工具
async def benchmark():
    # 同步工具（串行）
    start = time.time()
    for i in range(5):
        sync_tool.invoke({"n": i})
    sync_time = time.time() - start
    print(f"Sync tools: {sync_time:.2f}s")  # 约 5 秒

    # 异步工具（并发）
    start = time.time()
    await asyncio.gather(*[
        async_tool.ainvoke({"n": i}) for i in range(5)
    ])
    async_time = time.time() - start
    print(f"Async tools: {async_time:.2f}s")  # 约 1 秒

    print(f"Speedup: {sync_time / async_time:.1f}x")

asyncio.run(benchmark())
```

**输出**：
```
Sync tools: 5.01s
Async tools: 1.00s
Speedup: 5.0x
```
