## 卡片 1：多向量检索的直觉理解

**一句话**：多向量检索就像用多个维度同时评价一个人，既看学历（稠密向量），又看工作经验（稀疏向量），综合打分找到最合适的候选人。

**举例**：
```python
# Collection = 候选人数据库
# 多个向量字段 = 多个评价维度
schema.add_field("dense_vector", DataType.FLOAT_VECTOR, dim=768)    # 学历背景
schema.add_field("sparse_vector", DataType.SPARSE_FLOAT_VECTOR)     # 工作经验

# 同时检索多个维度
results = client.hybrid_search(
    collection_name="candidates",
    reqs=[dense_req, sparse_req],  # 两个维度同时评价
    ranker=RRFRanker(),            # 综合打分
    limit=10
)
```

**应用**：多模态检索（图片+文本）、全文搜索增强（BM25+语义）、多语言检索（中文+英文）

## 卡片 2：AnnSearchRequest API 使用

**一句话**：AnnSearchRequest 定义单个向量字段的检索请求，指定查询向量、字段名、检索参数和返回数量。

**关键代码**：
```python
from pymilvus import AnnSearchRequest

# 定义稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense_embedding],      # 查询向量
    anns_field="dense_vector",         # 向量字段名
    param={"metric_type": "IP"},       # 检索参数（内积）
    limit=10                           # 返回结果数量
)

# 定义稀疏向量检索请求
sparse_req = AnnSearchRequest(
    data=[query_sparse_embedding],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)
```

**实战技巧**：
- `data` 参数：稠密向量传入 list，稀疏向量传入 csr_matrix
- `metric_type`：稠密向量常用 IP 或 L2，稀疏向量常用 IP
- `limit`：控制每个检索请求的返回数量，最终结果由 hybrid_search 的 limit 控制

## 卡片 3：RRF 算法原理与应用

**一句话**：RRF（Reciprocal Rank Fusion）通过倒数排名融合多个检索结果，公式为 score = 1 / (k + rank)，适合不确定权重的场景。

**算法原理**：
```python
# RRF 算法示例
def rrf_score(rank, k=60):
    """
    rank: 结果在检索列表中的排名（从1开始）
    k: 平滑参数，默认60
    """
    return 1 / (k + rank)

# 示例：某个文档在两个检索结果中的排名
dense_rank = 3   # 在稠密向量检索中排第3
sparse_rank = 5  # 在稀疏向量检索中排第5

# 计算融合分数
score = rrf_score(dense_rank) + rrf_score(sparse_rank)
# score = 1/(60+3) + 1/(60+5) = 0.0159 + 0.0154 = 0.0313
```

**使用示例**：
```python
from pymilvus import RRFRanker

# 创建 RRF 排序器（k=60 是默认值）
ranker = RRFRanker()

# 执行混合检索
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[dense_req, sparse_req],
    ranker=ranker,
    limit=10
)
```

**应用**：不确定各向量字段权重时的首选方案，自动平衡多个检索结果

## 卡片 4：WeightedRanker 权重策略

**一句话**：WeightedRanker 使用加权求和融合多个检索结果，权重可以是任意正数，适合已知权重的场景。

**关键代码**：
```python
from pymilvus import WeightedRanker

# 创建加权排序器
ranker = WeightedRanker(
    sparse_weight=0.7,   # 稀疏向量权重（精确匹配）
    dense_weight=1.0     # 稠密向量权重（语义理解）
)

# 执行混合检索
results = col.hybrid_search(
    [sparse_req, dense_req],
    rerank=ranker,
    limit=10,
    output_fields=["text"]
)
```

**权重调整策略**：
- **精确匹配优先**：sparse_weight=1.0, dense_weight=0.5（关键词搜索）
- **语义理解优先**：sparse_weight=0.5, dense_weight=1.0（问答系统）
- **平衡策略**：sparse_weight=0.7, dense_weight=1.0（通用场景）

**实战技巧**：
- 权重不需要归一化，Milvus 会自动处理
- 可以通过 A/B 测试找到最优权重
- 不同场景使用不同权重配置

## 卡片 5：混合检索完整流程

**一句话**：混合检索流程包括定义多个 AnnSearchRequest、选择 Ranker、调用 hybrid_search API、获取融合结果四个步骤。

**完整示例**：
```python
from pymilvus import AnnSearchRequest, WeightedRanker

def hybrid_search_pipeline(
    col,
    query_dense_embedding,
    query_sparse_embedding,
    sparse_weight=1.0,
    dense_weight=1.0,
    limit=10,
):
    # 步骤1：定义稠密向量检索请求
    dense_req = AnnSearchRequest(
        [query_dense_embedding],
        "dense_vector",
        {"metric_type": "IP", "params": {}},
        limit=limit
    )

    # 步骤2：定义稀疏向量检索请求
    sparse_req = AnnSearchRequest(
        [query_sparse_embedding],
        "sparse_vector",
        {"metric_type": "IP", "params": {}},
        limit=limit
    )

    # 步骤3：创建加权排序器
    rerank = WeightedRanker(sparse_weight, dense_weight)

    # 步骤4：执行混合检索
    res = col.hybrid_search(
        [sparse_req, dense_req],
        rerank=rerank,
        limit=limit,
        output_fields=["text"]
    )[0]

    return [hit.get("text") for hit in res]
```

**应用**：RAG 系统、文档问答、知识库检索

## 卡片 6：BM25 + 稠密向量混合检索

**一句话**：BM25 函数自动将文本转换为稀疏向量，结合稠密向量实现精确关键词匹配 + 语义理解的混合检索。

**关键代码**：
```python
from pymilvus import AnnSearchRequest, RRFRanker

# 查询文本
query = "what is hybrid search"

# 获取稠密向量
query_embedding = get_embeddings([query])[0]

# 步骤1：BM25 稀疏向量检索（直接传入文本）
sparse_request = AnnSearchRequest(
    [query],                    # 直接传入文本，BM25 函数自动转换
    "sparse_vector",
    {"metric_type": "BM25"},
    limit=5
)

# 步骤2：稠密向量检索（传入向量）
dense_request = AnnSearchRequest(
    [query_embedding],          # 传入向量
    "dense_vector",
    {"metric_type": "IP"},
    limit=5
)

# 步骤3：使用 RRF 融合结果
results = client.hybrid_search(
    collection_name,
    [sparse_request, dense_request],
    ranker=RRFRanker(),
    limit=5,
    output_fields=["content", "metadata"]
)
```

**优势**：
- **BM25**：精确关键词匹配，捕获术语重要性
- **稠密向量**：语义理解，捕获概念关系
- **混合检索**：兼顾精确匹配和语义理解

## 卡片 7：多模态检索实战

**一句话**：使用 CLIP 等多模态模型为图片和文本生成向量，通过多向量检索实现图文混合搜索。

**应用场景**：
```python
# 场景：电商平台图文混合搜索
# 用户输入："红色连衣裙"

# 步骤1：定义多向量字段
schema.add_field("text_vector", DataType.FLOAT_VECTOR, dim=512)   # 文本向量
schema.add_field("image_vector", DataType.FLOAT_VECTOR, dim=512)  # 图片向量

# 步骤2：使用 CLIP 模型生成查询向量
from transformers import CLIPModel, CLIPProcessor

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

# 生成文本向量
text_inputs = processor(text=["红色连衣裙"], return_tensors="pt")
text_vector = model.get_text_features(**text_inputs)[0].tolist()

# 步骤3：定义检索请求
text_req = AnnSearchRequest(
    [text_vector],
    "text_vector",
    {"metric_type": "IP"},
    limit=10
)

image_req = AnnSearchRequest(
    [text_vector],  # CLIP 模型的文本和图片向量在同一空间
    "image_vector",
    {"metric_type": "IP"},
    limit=10
)

# 步骤4：混合检索
results = client.hybrid_search(
    "products",
    [text_req, image_req],
    ranker=RRFRanker(),
    limit=10
)
```

**应用**：电商搜索、内容推荐、图片检索

## 卡片 8：索引创建的关键要求

**一句话**：所有向量字段都必须创建索引才能加载 Collection，部分索引会导致 LoadCollection 失败。

**关键约束**（来源：源码分析）：
```python
# 错误示例：只为部分向量字段创建索引
schema.add_field("dense_vector", DataType.FLOAT_VECTOR, dim=128)
schema.add_field("sparse_vector", DataType.SPARSE_FLOAT_VECTOR)

# 只创建稠密向量索引
collection.create_index(
    field_name="dense_vector",
    index_params={"index_type": "IVF_FLAT", "metric_type": "L2"}
)

# 尝试加载 Collection（会失败）
collection.load()  # ❌ 错误：sparse_vector 没有索引
```

**正确示例**：
```python
# 为所有向量字段创建索引
collection.create_index(
    field_name="dense_vector",
    index_params={"index_type": "IVF_FLAT", "metric_type": "L2"}
)

collection.create_index(
    field_name="sparse_vector",
    index_params={"index_type": "SPARSE_INVERTED_INDEX", "metric_type": "IP"}
)

# 现在可以加载 Collection
collection.load()  # ✅ 成功
```

**实战技巧**：
- 不同类型的向量字段使用不同的索引类型
- FloatVector：IVF_FLAT、HNSW
- SparseFloatVector：SPARSE_INVERTED_INDEX
- BinaryVector：BIN_IVF_FLAT

## 卡片 9：性能优化关键数据

**一句话**：多向量检索通过并行检索和结果融合，在保持高召回率的同时提升检索效率。

**关键数据**：
- **并行检索**：多个向量字段同时检索，不增加延迟
- **召回率提升**：混合检索比单一向量检索召回率提升 15-30%
- **推荐 limit 配置**：每个 AnnSearchRequest 的 limit 设置为最终结果的 2-3 倍
- **索引类型选择**：
  - 稠密向量：HNSW（高性能）或 IVF_FLAT（平衡）
  - 稀疏向量：SPARSE_INVERTED_INDEX（唯一选择）

**性能优化示例**：
```python
# 优化前：每个请求 limit=10，最终结果 limit=10
dense_req = AnnSearchRequest([vector], "dense_vector", {"metric_type": "IP"}, limit=10)
sparse_req = AnnSearchRequest([vector], "sparse_vector", {"metric_type": "IP"}, limit=10)

# 优化后：每个请求 limit=30，最终结果 limit=10
dense_req = AnnSearchRequest([vector], "dense_vector", {"metric_type": "IP"}, limit=30)
sparse_req = AnnSearchRequest([vector], "sparse_vector", {"metric_type": "IP"}, limit=30)

# 融合后取 top 10，召回率更高
results = client.hybrid_search(
    collection_name,
    [dense_req, sparse_req],
    ranker=RRFRanker(),
    limit=10  # 最终返回 10 个结果
)
```

**应用**：生产环境性能调优、大规模检索系统

## 卡片 10：多向量检索最佳实践

**一句话**：选择合适的 Ranker、配置合理的权重、监控性能并及时调优。

**最佳实践**：

1. **Ranker 选择**：
   - 不确定权重 → RRFRanker（推荐）
   - 已知权重 → WeightedRanker
   - 需要时间衰减 → Decay Ranker

2. **权重配置**：
   - 关键词搜索：sparse_weight=1.0, dense_weight=0.5
   - 语义搜索：sparse_weight=0.5, dense_weight=1.0
   - 通用场景：sparse_weight=0.7, dense_weight=1.0

3. **索引配置**：
   - 所有向量字段都必须创建索引
   - 稠密向量使用 HNSW 或 IVF_FLAT
   - 稀疏向量使用 SPARSE_INVERTED_INDEX

4. **性能优化**：
   - 每个 AnnSearchRequest 的 limit 设置为最终结果的 2-3 倍
   - 使用 BM25 函数自动生成稀疏向量
   - 监控 QPS、延迟、召回率

5. **LangChain 集成**：
```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Milvus
from langchain_community.vectorstores.utils import BM25BuiltInFunction

# 定义多个 Embedding 模型
embedding1 = OpenAIEmbeddings(model="text-embedding-ada-002")
embedding2 = OpenAIEmbeddings(model="text-embedding-3-large")

# 创建 Milvus 向量存储（支持多向量字段）
vectorstore = Milvus.from_documents(
    documents=docs,
    embedding=[embedding1, embedding2],
    builtin_function=BM25BuiltInFunction(output_field_names="sparse"),
    vector_field=["dense1", "dense2", "sparse"],
    connection_args={"uri": URI},
)
```

**应用**：生产环境部署、RAG 系统优化
