## 4.1 五分钟创建并运行项目

**这是什么**：从零到 "Hello, world!" 只需一条命令。

```bash
# 创建项目——等同于 npm init + 自动生成骨架
cargo new my_project
cd my_project
```

**生成的目录结构**：

```
my_project/
├── Cargo.toml      ← package.json（项目配置 + 依赖声明）
├── .gitignore       ← 自动生成
└── src/
    └── main.rs      ← 入口文件（等同于 index.ts）
```

**Cargo.toml 内容**：

```toml
[package]
name = "my_project"        # 包名（= package.json 的 "name"）
version = "0.1.0"          # 语义化版本
edition = "2024"           # Rust 版本（类似 TS 的 target）

[dependencies]             # 运行时依赖（目前为空）
```

**创建库项目**用 `--lib` 标志，生成 `src/lib.rs` 而不是 `main.rs`：

```bash
cargo new --lib my_lib     # 创建库项目——等同于创建 npm 包
```

**三个最常用命令**：

```bash
cargo run          # 编译并运行  = npm start
cargo build        # 只编译      = npm run build
cargo test         # 运行测试    = npm test
```

**关键区别**：Rust 没有 `npm install` 这一步！`cargo build` 会**自动下载并编译**所有依赖。[^1]

**记忆口诀**：**`cargo new` 创建，`cargo run` 运行，`cargo test` 测试——三板斧够用。**

## 4.2 添加和使用依赖（三步走）

**这是什么**：引入外部库，只需三步。

### Step 1：添加依赖

```bash
cargo add serde --features derive      # 添加序列化库
cargo add serde_json                   # 添加 JSON 处理
```

自动修改 `Cargo.toml`：

```toml
[dependencies]
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
```

### Step 2：在代码中使用

```rust
use serde::{Serialize, Deserialize};  // ← 类似 import { x } from 'serde'

#[derive(Serialize, Deserialize, Debug)]
struct User {
    name: String,
    age: u32,
}

fn main() {
    let user = User { name: "Alice".to_string(), age: 30 };

    // 序列化（= JSON.stringify）
    let json = serde_json::to_string_pretty(&user).unwrap();
    println!("JSON:\n{}", json);

    // 反序列化（= JSON.parse）
    let parsed: User = serde_json::from_str(&json).unwrap();
    println!("解析: {:?}", parsed);
}
```

### Step 3：编译运行

```bash
cargo build   # 自动下载 serde + serde_json 并编译
cargo run     # 运行
```

### 与 npm 的关键差异

| 维度 | npm | Cargo |
|------|-----|-------|
| 安装命令 | `npm install pkg` | `cargo add pkg` |
| 依赖存放 | `node_modules/`（项目内） | `~/.cargo/registry/`（全局缓存） |
| 安装时机 | 显式 `npm install` | `cargo build` 时自动 |
| 版本锁定 | `package-lock.json` | `Cargo.lock` |
| 特性开关 | 无（全量安装） | `--features` 按需启用 |

**记忆口诀**：**`cargo add` 加依赖，`use` 引模块，`cargo build` 全搞定。**

## 4.3 组织代码到模块（最常用模式）

**这是什么**：把代码拆分到多个文件。这是与 TypeScript 差异最大的地方。

### 核心规则（记住这一条就够了）

> **Rust 的文件不会自动成为模块！必须在父模块中用 `mod xxx;` 显式声明。**[^2]

### 最小示例：一个有 3 个模块的项目

**目录结构**：

```
user_app/
├── Cargo.toml
└── src/
    ├── main.rs        ← 入口：声明模块 + 使用模块
    ├── models.rs      ← 数据模型
    ├── services.rs    ← 业务逻辑
    └── utils.rs       ← 工具函数
```

**src/models.rs**：

```rust
#[derive(Debug, Clone)]
pub struct User {           // pub = export（公开给外部）
    pub name: String,
    pub email: String,
}

impl User {
    pub fn new(name: &str, email: &str) -> Self {
        User { name: name.to_string(), email: email.to_string() }
    }
}
```

**src/utils.rs**：

```rust
pub fn format_display_name(name: &str) -> String {
    let trimmed = name.trim();
    if trimmed.is_empty() { return "Anonymous".to_string(); }
    let mut chars = trimmed.chars();
    match chars.next() {
        None => String::new(),
        Some(first) => first.to_uppercase().to_string() + chars.as_str(),
    }
}
```

**src/services.rs**：

```rust
use crate::models::User;              // crate:: = 从项目根开始
use crate::utils::format_display_name;

pub struct UserService {
    users: Vec<User>,
}

impl UserService {
    pub fn new() -> Self { UserService { users: Vec::new() } }

    pub fn add_user(&mut self, name: &str, email: &str) {
        let display_name = format_display_name(name);
        let user = User::new(&display_name, email);
        println!("添加用户: {:?}", user);
        self.users.push(user);
    }

    pub fn list_users(&self) {
        println!("\n=== 用户列表 ({} 人) ===", self.users.len());
        for user in &self.users {
            println!("  - {} <{}>", user.name, user.email);
        }
    }
}
```

**src/main.rs**（关键！）：

```rust
// ─── 第一步：声明模块（必须！文件存在 ≠ 模块存在）───
mod models;       // ← 告诉编译器：去找 src/models.rs
mod services;     // ← 告诉编译器：去找 src/services.rs
mod utils;        // ← 告诉编译器：去找 src/utils.rs

// ─── 第二步：引入具体类型 ───
use services::UserService;

fn main() {
    let mut svc = UserService::new();
    svc.add_user("alice", "alice@example.com");
    svc.add_user("  bob  ", "bob@example.com");
    svc.list_users();
}
```

### TypeScript 对照

```typescript
// TypeScript——文件即模块，直接 import
import { UserService } from './services';
```

```rust
// Rust——必须先 mod 声明，再 use 引用
mod services;                    // 第一步：声明
use services::UserService;      // 第二步：引入
```

### 路径前缀速记

| 前缀 | 含义 | TypeScript 等价 |
|------|------|-----------------|
| `crate::` | 从项目根开始 | `@/`（路径别名） |
| `super::` | 从父模块开始 | `../` |
| `self::` | 从当前模块开始 | `./` |

**记忆口诀**：**先 `mod` 声明，后 `use` 引用，`pub` 控制可见性。**

## 4.4 读懂 ZeroClaw 的项目结构

有了前三节基础，你已经能看懂 ZeroClaw 的源码组织了。

### Cargo.toml 关键部分

```toml
[workspace]
members = [".", "crates/robot-kit"]    # 两个 crate 组成 workspace
resolver = "2"

[package]
name = "zeroclaw"
version = "0.1.7"
edition = "2024"

[features]
default = []                                     # 默认不启用可选功能
hardware = ["dep:nusb", "dep:tokio-serial"]      # USB 硬件支持
channel-matrix = ["dep:matrix-sdk"]              # Matrix 协议

[profile.release]
opt-level = "z"       # 优化目标：最小体积（嵌入式设备）
lto = "fat"           # 链接时优化
strip = true          # 移除调试符号
```

| Cargo.toml 段落 | 等同于 | 作用 |
|-----------------|--------|------|
| `[workspace]` | `workspaces` in package.json | monorepo 配置 |
| `[package]` | `name`/`version` | 包元数据 |
| `[dependencies]` | `dependencies` | 运行时依赖 |
| `[features]` | webpack DefinePlugin | 编译时功能开关 |
| `[profile.release]` | webpack production config | 生产环境优化 |

### src/lib.rs 模块声明

```rust
// src/lib.rs — ZeroClaw 的模块声明（精简版）
pub mod agent;          // 代理循环
pub mod config;         // 配置管理
pub mod providers;      // LLM 提供商（OpenAI、Anthropic…）
pub mod tools;          // 可调用工具（40+ 个）
pub mod channels;       // 消息通道（Telegram、Discord…）

#[cfg(feature = "hardware")]
pub mod hardware;       // ← 只在启用 hardware feature 时编译

// Re-export 核心类型（用户直接 use zeroclaw::Config）
pub use config::Config;
pub use agent::{Agent, AgentBuilder};
pub use tools::Tool;
```

**怎么读？**[^3]
1. `pub mod agent;` → 公开模块，源码在 `src/agent/mod.rs`
2. `#[cfg(feature = "hardware")]` → 条件编译，只在启用时编译
3. `pub use config::Config;` → re-export，外部可直接 `use zeroclaw::Config`

### `#[cfg(feature = "xxx")]` 的含义

```rust
#[cfg(feature = "hardware")]
pub mod hardware;       // 整个模块按需编译
```

```bash
cargo build                          # 不含 hardware
cargo build --features hardware      # 包含 hardware
cargo build --features "hardware,channel-matrix"  # 多个 feature
```

**TypeScript 类比**：`#[cfg]` 是编译时移除代码，比 `process.env` 更彻底——代码完全不存在于产物中。

**记忆口诀**：**`Cargo.toml` 管配置，`lib.rs` 管模块，`#[cfg]` 管开关。**

## 4.5 Cargo 日常命令速查表

| 命令 | npm 等价物 | 一行解释 |
|------|-----------|---------|
| `cargo new name` | `npm init` | 创建新项目 |
| `cargo build` | `npm run build` | 编译（首次自动下载依赖） |
| `cargo run` | `npm start` | 编译 + 运行 |
| `cargo test` | `npm test` | 运行所有测试 |
| `cargo add pkg` | `npm install pkg` | 添加依赖 |
| `cargo check` | `tsc --noEmit` | 快速检查错误，不生成二进制 |
| `cargo clippy` | `eslint` | 代码质量检查 |
| `cargo fmt` | `prettier` | 自动格式化 |
| `cargo doc --open` | `typedoc` | 生成并打开文档 |
| `cargo tree` | `npm ls` | 显示依赖树 |

**Feature 相关命令**：

```bash
cargo build --features "hardware"       # 启用特定 feature
cargo build --all-features              # 启用全部 feature
cargo build --no-default-features       # 禁用默认 feature
cargo tree -e features                  # 查看 feature 依赖树
```

**使用频率排序**：

```
每次写代码：  cargo check → cargo run → cargo test
每天使用：    cargo clippy → cargo fmt
偶尔使用：    cargo add → cargo update → cargo doc
项目初始化：  cargo new
```

## 综合示例：从零构建一个迷你多模块项目

把 4.1 - 4.5 全部组合：

```bash
cargo new mini_tool && cd mini_tool
cargo add serde --features derive
cargo add serde_json
```

**src/config.rs**：

```rust
use serde::{Serialize, Deserialize};

#[derive(Debug, Serialize, Deserialize)]
pub struct AppConfig {
    pub app_name: String,
    pub debug: bool,
}

impl AppConfig {
    pub fn default_config() -> Self {
        AppConfig { app_name: "MiniTool".to_string(), debug: false }
    }

    pub fn from_json(json: &str) -> Result<Self, serde_json::Error> {
        serde_json::from_str(json)
    }
}
```

**src/processor.rs**：

```rust
use crate::config::AppConfig;

pub struct Processor { config: AppConfig }

impl Processor {
    pub fn new(config: AppConfig) -> Self { Processor { config } }

    pub fn run(&self) {
        println!("🚀 {} 启动", self.config.app_name);
        if self.config.debug {
            println!("  [DEBUG] 配置: {:?}", self.config);
        }
        println!("  处理完成！");
    }
}
```

**src/main.rs**：

```rust
mod config;
mod processor;

use config::AppConfig;
use processor::Processor;

fn main() {
    let custom_json = r#"{ "app_name": "MyTool", "debug": true }"#;

    match AppConfig::from_json(custom_json) {
        Ok(cfg) => Processor::new(cfg).run(),
        Err(e) => eprintln!("配置解析失败: {}", e),
    }
}
```

```bash
cargo run
# 🚀 MyTool 启动
#   [DEBUG] 配置: AppConfig { app_name: "MyTool", debug: true }
#   处理完成！
```

**这个例子用到了**：`cargo new` + `cargo add`（4.1 & 4.2），`mod` + `use` + `pub`（4.3），`crate::` 路径（4.3），`cargo run`（4.5）。
