### Embeddings API Parameters

Source: https://docs.voyageai.com/reference/contextualized-embeddings-api

This section details the parameters available for the Voyage AI Embeddings API, including input type, output dimension, output data type, and encoding format.

```APIDOC
## POST /embeddings

### Description
This endpoint generates embeddings for the provided text inputs. You can customize the embedding process by specifying input type, output dimension, output data type, and encoding format.

### Method
POST

### Endpoint
/embeddings

### Parameters
#### Request Body
- **inputs** (array[string]) - Required - The list of texts to embed.
- **input_type** (string) - Optional - Type of the input text. Defaults to `null`. Other options: `query`, `document`. 
  - When `input_type` is `null`, embeddings are generated directly.
  - When `input_type` is `query` or `document`, Voyage prepends specific prompts for retrieval-optimized embeddings.
    - `query`: "Represent the query for retrieving supporting documents: "
    - `document`: "Represent the document for retrieval: "
- **output_dimension** (integer) - Optional - The number of dimensions for resulting output embeddings. Defaults to `null` (1024). `voyage-context-3` supports 2048, 1024, 512, and 256.
- **output_dtype** (string) - Optional - The data type for the embeddings to be returned. Defaults to `float`. Other options: `int8`, `uint8`, `binary`, `ubinary`.
  - `float`: 32-bit single-precision floating-point numbers (default, highest precision).
  - `int8`: 8-bit integers (-128 to 127).
  - `uint8`: 8-bit unsigned integers (0 to 255).
  - `binary`: Bit-packed, quantized single-bit embedding values (uses offset binary method).
  - `ubinary`: Bit-packed, quantized single-bit embedding values (unsigned).
- **encoding_format** (string) - Optional - Format in which the embeddings are encoded. Defaults to `null`. Other options: `base64`.
  - `null`: Embeddings are returned as arrays of numbers (float or integers based on `output_dtype`).
  - `base64`: Embeddings are returned as a Base64-encoded NumPy array.

### Request Example
```json
{
  "inputs": ["What is the capital of France?", "Who wrote 'Hamlet'?"],
  "input_type": "query",
  "output_dimension": 512,
  "output_dtype": "float",
  "encoding_format": null
}
```

### Response
#### Success Response (200)
- **embeddings** (array[array[number]]) - A list of embeddings, where each embedding is a list of numbers.
- **model_info** (object) - Information about the model used.
  - **id** (string) - The model ID.
  - **created** (integer) - Timestamp of model creation.
  - **owned_by** (string) - The owner of the model.

#### Response Example
```json
{
  "embeddings": [
    [0.001, 0.002, ...],
    [0.003, 0.004, ...]
  ],
  "model_info": {
    "id": "voyage-01",
    "created": 1677610600,
    "owned_by": "voyageai"
  }
}
```
```

### Convert Float Embeddings to Int8 Embeddings in Python

Source: https://docs.voyageai.com/discuss/67814163e1852e0010dba25e

This Python snippet demonstrates how to convert existing float embeddings (e.g., 2048 dimensions) to int8 embeddings (e.g., 1024 dimensions) using the Voyage AI client. It includes steps for scaling, clipping, casting to int8, and truncating the embeddings. The code also verifies if the derived embeddings match those directly generated by the API.

```python
import numpy as np
import voyageai

vo = voyageai.Client()

# Get float and int8 embeddings
embd_float = vo.embed('Sample text 1', model='voyage-3-large', output_dimension=2048).embeddings[0]
embd_int8 = vo.embed('Sample text 1', model='voyage-3-large', output_dimension=1024, output_dtype='int8').embeddings[0]

# Compute int8 embeddings from float
embd_int8_calc = ((np.array(embd_float)/ 0.16).clip(-1, 1) * 127).astype(np.int8).tolist() # Scale
embd_int8_1024_calc = embd_int8_calc[0:1024] # Truncate

if (embd_int8 == embd_int8_1024_calc):
    print("The returned and derived int8 embeddings are the same.")
else:
    print("The returned and derived int8 embeddings are NOT the same.")
print("The first ten elements of the embeddings are")
print(f"\tReturned: {embd_int8[0:10]}")
print(f"\tDerived: {embd_int8_1024_calc[0:10]}")

```

### Generate Contextualized Embeddings with Voyage AI Python Package

Source: https://docs.voyageai.com/docs/contextualized-chunk-embeddings

This snippet demonstrates how to use the `voyageai.Client.contextualized_embed()` function to generate contextualized embeddings. It requires the `voyageai` package to be installed and an API key to be set up. The function takes a list of lists of strings as input, along with a model name, and can optionally accept input type, output dimension, and output data type.

```python
import voyageai

# Initialize the client with your API key (or ensure it's set as an environment variable)
client = voyageai.Client()

# Example inputs: a list of lists, where each inner list contains text chunks from a document
inputs = [
    ["chunk 1 of document 1", "chunk 2 of document 1", "chunk 3 of document 1"],
    ["chunk 1 of document 2", "chunk 2 of document 2"]
]

# Generate contextualized embeddings
response = client.contextualized_embed(
    inputs=inputs,
    model="voyage-context-3",
    input_type="document",
    output_dimension=1024,
    output_dtype="float"
)

# The response contains the embeddings
print(response.embeddings)
```

### Embeddings API > Parameters > output_dimension

Source: https://docs.voyageai.com/reference/embeddings-api

The `output_dimension` parameter specifies the number of dimensions for the resulting output embeddings. Most models support a single default dimension when `output_dimension` is set to `null`. However, specific models like `voyage-4-large`, `voyage-4`, `voyage-4-lite`, `voyage-3-large`, `voyage-3.5`, `voyage-3.5-lite`, and `voyage-code-3` offer flexibility and support `output_dimension` values of 2048, 1024 (which is the default), 512, and 256.

### Python API > voyageai.Client.embed()

Source: https://docs.voyageai.com/docs/embeddings

The `voyageai.Client.embed()` function is used to vectorize inputs. It accepts a list of texts, a model name, and optional parameters such as `input_type`, `truncation`, `output_dimension`, and `output_dtype`. The `texts` parameter can be a single string or a list of strings, with constraints on the maximum number of items and total tokens depending on the model used. The `model` parameter specifies which embedding model to use, with recommended options provided. The `input_type` parameter can be set to `None`, `query`, or `document` to tailor embeddings for retrieval tasks. `truncation` controls whether input texts exceeding context length are truncated or raise an error. `output_dimension` allows specifying the dimensionality of the output embeddings for certain models, and `output_dtype` determines the data type of the returned embeddings, with options like `float`, `int8`, `uint8`, `binary`, and `ubinary` offering different trade-offs between precision and storage.
