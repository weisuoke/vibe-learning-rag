## 4.1 基本语法：Annotated[类型, Reducer函数]

**一句话：** 使用 `Annotated` 为状态字段指定合并策略

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator

class State(TypedDict):
    # 使用 Annotated 指定 reducer
    messages: Annotated[list, operator.add]  # 累积策略

    # 不使用 Annotated
    counter: int  # 覆盖策略（默认）
```

**关键点：**
- `Annotated[类型, reducer]` 的第二个参数是 reducer 函数
- Reducer 函数签名：`(old_value, new_value) -> merged_value`
- 没有 Annotated 的字段使用默认覆盖策略

**实际应用：**
在对话系统中，消息需要累积而不是覆盖：
```python
class ConversationState(TypedDict):
    messages: Annotated[list[BaseMessage], operator.add]  # 累积消息
    user_id: str  # 覆盖用户ID
```

## 4.2 三个内置 Reducer：operator.add / operator.or_ / add_messages

**一句话：** 使用内置 reducer 处理常见的状态合并场景

### operator.add - 列表/字符串拼接

```python
class State(TypedDict):
    items: Annotated[list, operator.add]
    text: Annotated[str, operator.add]

# 使用示例
def node1(state):
    return {"items": [1, 2]}  # 只返回新增项

def node2(state):
    return {"items": [3, 4]}  # 只返回新增项

# 最终状态：{"items": [1, 2, 3, 4]}
```

### operator.or_ - 字典合并

```python
class State(TypedDict):
    metadata: Annotated[dict, operator.or_]

# 使用示例
def node1(state):
    return {"metadata": {"user": "alice"}}

def node2(state):
    return {"metadata": {"session": "123"}}

# 最终状态：{"metadata": {"user": "alice", "session": "123"}}
```

### add_messages - 消息列表管理

```python
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]

# 特性：按 ID 更新消息
# - 新消息追加到列表
# - 相同 ID 的消息会被覆盖
```

**实际应用：**
- `operator.add`：累积推理步骤、工具调用结果
- `operator.or_`：合并配置、元数据
- `add_messages`：管理对话历史

## 4.3 节点返回增量更新（不是完整状态）

**一句话：** 节点只返回需要更新的字段，不要返回完整状态

```python
class State(TypedDict):
    messages: Annotated[list, operator.add]
    counter: int

# ❌ 错误：返回完整状态
def bad_node(state):
    return {
        "messages": state["messages"] + [new_message],  # 包含旧消息
        "counter": state["counter"] + 1
    }

# ✅ 正确：只返回增量
def good_node(state):
    return {
        "messages": [new_message],  # 只返回新消息
        "counter": state["counter"] + 1
    }
```

**为什么这样做？**
- Reducer 会自动合并旧值和新值
- 返回完整状态会导致重复累积
- 增量更新更清晰、更高效

**实际应用：**
在多步推理系统中，每个节点只返回当前步骤的结果：
```python
def reasoning_step(state):
    # 执行推理
    thought = llm.invoke(state["messages"])

    # 只返回新的推理步骤
    return {"thoughts": [thought]}  # 不包含旧的 thoughts
```

## 4.4 默认覆盖 vs Reducer 累积

**一句话：** 理解两种状态更新策略的区别

### 默认覆盖策略（无 Annotated）

```python
class State(TypedDict):
    current_step: str  # 无 Annotated

def node1(state):
    return {"current_step": "step1"}

def node2(state):
    return {"current_step": "step2"}

# 最终状态：{"current_step": "step2"}  # 覆盖
```

### Reducer 累积策略（有 Annotated）

```python
class State(TypedDict):
    steps: Annotated[list[str], operator.add]  # 有 Annotated

def node1(state):
    return {"steps": ["step1"]}

def node2(state):
    return {"steps": ["step2"]}

# 最终状态：{"steps": ["step1", "step2"]}  # 累积
```

**选择原则：**
- **需要累积**：使用 `Annotated[list, operator.add]`
  - 消息历史、推理步骤、工具结果
- **需要覆盖**：不使用 Annotated
  - 当前状态、用户ID、配置项

**实际应用：**
```python
class AgentState(TypedDict):
    # 累积：对话历史
    messages: Annotated[list[BaseMessage], add_messages]

    # 累积：工具调用结果
    tool_results: Annotated[list[ToolResult], operator.add]

    # 覆盖：当前步骤
    current_step: str

    # 覆盖：用户信息
    user_id: str
```
