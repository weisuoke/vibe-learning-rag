# Agent类型与选择 - 30字核心

## 核心概念（30字）

**根据模型能力和工具复杂度选择Agent类型：函数调用用OpenAI Functions，开源模型用ReAct，复杂工具用Structured Chat。**

---

## 为什么这30字重要？

选择合适的Agent类型是构建高效Agent系统的第一步，直接影响：

1. **执行效率**：不同Agent类型的推理开销差异巨大
2. **工具兼容性**：某些Agent类型对工具输入格式有严格要求
3. **模型适配性**：不是所有模型都支持所有Agent类型
4. **开发成本**：选错类型可能导致大量返工

这30字浓缩了Agent选型的核心决策树：
- **模型能力** → 是否支持函数调用？
- **工具复杂度** → 工具输入是否需要结构化？
- **场景需求** → 需要多少轮推理？

---

## 展开说明（核心决策逻辑）

### 1. 三大主流Agent类型

LangChain提供多种Agent类型，但核心常用的是三种：

| Agent类型 | 适用模型 | 工具要求 | 推理方式 | 典型场景 |
|-----------|----------|----------|----------|----------|
| **OpenAI Functions** | GPT-3.5/4 | 任意 | 函数调用 | 生产环境首选 |
| **ReAct** | 任意LLM | 任意 | 思维链 | 开源模型/教学 |
| **Structured Chat** | 任意LLM | 结构化输入 | 思维链+格式化 | 复杂工具场景 |

### 2. 选型决策树

```
开始
  ↓
是否使用OpenAI模型？
  ├─ 是 → OpenAI Functions Agent（最优选择）
  │       - 速度快（原生支持）
  │       - 可靠性高（格式稳定）
  │       - 成本低（Token消耗少）
  │
  └─ 否 → 工具是否需要复杂输入？
          ├─ 是 → Structured Chat Agent
          │       - 支持多参数工具
          │       - 强制JSON格式
          │       - 适合API调用
          │
          └─ 否 → ReAct Agent
                  - 通用性最强
                  - 适合简单工具
                  - 推理过程可见
```

### 3. 核心差异对比

#### OpenAI Functions Agent
```python
# 模型原生理解工具定义
tools = [
    {
        "name": "search",
        "description": "搜索网络信息",
        "parameters": {"query": "string"}
    }
]
# 模型直接返回：{"name": "search", "arguments": {"query": "..."}}
```

**优势**：
- ⚡ 速度快（1轮对话完成）
- 🎯 准确率高（格式固定）
- 💰 成本低（无需额外Prompt）

#### ReAct Agent
```python
# 模型通过Prompt学习推理格式
prompt = """
Thought: 我需要搜索信息
Action: search
Action Input: "LangChain Agent"
Observation: [搜索结果]
Thought: 现在我知道答案了
Final Answer: ...
"""
```

**优势**：
- 🌍 通用性强（任何LLM）
- 👁️ 可解释性好（推理可见）
- 🔧 灵活性高（易于调试）

#### Structured Chat Agent
```python
# 强制工具输入为JSON格式
tool_input = {
    "action": "complex_api",
    "action_input": {
        "param1": "value1",
        "param2": {"nested": "value2"}
    }
}
```

**优势**：
- 📦 支持复杂参数（嵌套JSON）
- 🔒 格式强约束（减少错误）
- 🔌 适合API集成（结构化输入）

### 4. 实际选型建议

**生产环境（推荐优先级）：**
1. **首选**：OpenAI Functions（如果用GPT）
2. **次选**：Structured Chat（如果工具复杂）
3. **备选**：ReAct（如果需要可解释性）

**开发/学习环境：**
- 优先使用ReAct（推理过程清晰）
- 便于理解Agent工作原理

**成本敏感场景：**
- OpenAI Functions > Structured Chat > ReAct
- （Token消耗递增）

## 与RAG开发的联系

在RAG Agent系统中，Agent类型选择直接影响：

1. **检索工具调用**：
   - 向量检索工具（简单参数）→ 任何Agent
   - 混合检索工具（复杂参数）→ Structured Chat

2. **多步推理场景**：
   - Query改写 → 检索 → ReRank → 生成
   - ReAct能清晰展示每步推理

3. **生产环境部署**：
   - 延迟敏感 → OpenAI Functions
   - 成本敏感 → 开源模型 + ReAct

**典型RAG Agent选型：**
```python
# 生产环境：OpenAI Functions + 向量检索
agent = create_openai_functions_agent(
    llm=ChatOpenAI(model="gpt-4"),
    tools=[vector_search_tool, rerank_tool]
)

# 开发环境：ReAct + 可视化推理
agent = create_react_agent(
    llm=ChatOpenAI(model="gpt-3.5-turbo"),
    tools=[vector_search_tool],
    verbose=True  # 查看推理过程
)
```
