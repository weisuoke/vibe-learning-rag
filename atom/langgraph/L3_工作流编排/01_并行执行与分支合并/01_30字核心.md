# 并行执行与分支合并 - 30字核心

**并行执行与分支合并是让多个节点同时执行并自动合并结果的机制，是 LangGraph 构建高性能工作流的核心能力。**

---

## 核心要素拆解

### 1. 并行执行 (Parallel Execution)

**定义**: 多个节点在同一个超步中同时执行

**关键机制**:
- **静态并行**: 使用多条边（`add_edge`）实现
- **动态并行**: 使用 Send API 实现

**价值**: 显著提升工作流性能，减少总执行时间

---

### 2. 分支合并 (Branch Merge)

**定义**: 使用 Reducer 函数自动合并并行节点的结果

**关键机制**:
- **Reducer 函数**: 如 `operator.add`
- **状态注解**: `Annotated[list, operator.add]`

**价值**: 避免状态覆盖，确保所有并行结果都被保留

---

### 3. 在 LangGraph 中的价值

**性能提升**:
- 串行执行 12 秒 → 并行执行 5 秒（提升 2.4 倍）

**模式支持**:
- Map-Reduce 工作流
- 多智能体协作
- 多数据源聚合

**架构优势**:
- 基于 Bulk Synchronous Parallel 模型
- 保证执行顺序和同步
- 支持复杂的分支逻辑

---

## 一句话记忆

**并行执行 = 同时做多件事，分支合并 = 自动汇总结果。**

---

## 快速示例

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict, Annotated
import operator

# 定义状态（使用 reducer 合并结果）
class State(TypedDict):
    results: Annotated[list, operator.add]

# 定义节点
def task_a(state):
    return {"results": ["A"]}

def task_b(state):
    return {"results": ["B"]}

def task_c(state):
    return {"results": ["C"]}

# 构建图（并行执行）
builder = StateGraph(State)
builder.add_node("task_a", task_a)
builder.add_node("task_b", task_b)
builder.add_node("task_c", task_c)

# Fan-out: 从 START 扇出到三个任务
builder.add_edge(START, "task_a")
builder.add_edge(START, "task_b")
builder.add_edge(START, "task_c")

# Fan-in: 三个任务扇入到 END
builder.add_edge("task_a", END)
builder.add_edge("task_b", END)
builder.add_edge("task_c", END)

# 编译并运行
graph = builder.compile()
result = graph.invoke({"results": []})
print(result)  # {'results': ['A', 'B', 'C']}
```

**运行结果**: 三个任务并行执行，结果自动合并为 `['A', 'B', 'C']`

## 核心对比

| 特性 | 串行执行 | 并行执行 |
|------|----------|----------|
| 执行方式 | 一个接一个 | 同时执行 |
| 总时间 | 所有任务时间之和 | 最慢任务的时间 |
| 状态管理 | 简单覆盖 | 需要 Reducer 合并 |
| 适用场景 | 有依赖关系的任务 | 独立任务 |

---

## 两种并行方式对比

| 特性 | 静态并行（多条边） | 动态并行（Send API） |
|------|-------------------|---------------------|
| 并行度 | 编译时确定 | 运行时确定 |
| 灵活性 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 性能 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 代码复杂度 | 简单 | 稍复杂 |
| 适用场景 | 固定数量任务 | 动态数量任务 |

---

## 关键类比

**前端类比**: 就像 `Promise.all()`，同时执行多个异步任务，等待所有任务完成后合并结果

**日常生活类比**: 就像餐厅厨房，多个厨师同时做不同的菜（并行执行），最后服务员把所有菜端上桌（分支合并）

---

## 核心公式

```
并行执行时间 = max(task1_time, task2_time, task3_time)
串行执行时间 = task1_time + task2_time + task3_time

性能提升 = 串行时间 / 并行时间
```

**示例**:
- 任务 A: 3 秒
- 任务 B: 5 秒
- 任务 C: 4 秒

串行: 3 + 5 + 4 = 12 秒
并行: max(3, 5, 4) = 5 秒
提升: 12 / 5 = 2.4 倍

---

## 核心机制

### Fan-out (扇出)

```python
# 从一个节点分支到多个节点
builder.add_edge("start", "task_a")
builder.add_edge("start", "task_b")
builder.add_edge("start", "task_c")
```

### Fan-in (扇入)

```python
# 多个节点合并到一个节点
builder.add_edge("task_a", "end")
builder.add_edge("task_b", "end")
builder.add_edge("task_c", "end")
```

### Reducer (合并器)

```python
# 使用 operator.add 自动合并列表
class State(TypedDict):
    results: Annotated[list, operator.add]
```
