# 实战代码 场景2：循环控制与错误处理

> 给 AgentExecutor 装上刹车和安全气囊——max_iterations 防失控，handle_parsing_errors 防崩溃

---

## 场景概述

场景1 演示了 AgentExecutor 的基本用法，但那是"晴天场景"。生产环境中，Agent 会遇到各种问题：

- LLM 反复调用工具却不给最终答案（死循环）
- LLM 输出格式不对，解析器无法识别（解析错误）
- 工具执行超时或失败（工具异常）
- 整体执行时间过长（用户等不及）

本文演示如何用 AgentExecutor 的控制参数应对这些问题：

- `max_iterations`：限制最大循环次数
- `max_execution_time`：限制最大执行时间
- `handle_parsing_errors`：自动处理 LLM 输出格式错误
- `early_stopping_method`：超限后的善后策略

## 示例1：max_iterations——防止 Agent 死循环

**解决的问题：** LLM 可能反复调用工具却永远不给出最终答案。`max_iterations` 是最基本的安全阀。

```python
"""
max_iterations 演示
演示：限制 Agent 最大循环次数，观察超限后的行为
"""
from dotenv import load_dotenv
load_dotenv()

from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate


@tool
def search(query: str) -> str:
    """搜索信息，但总是返回不够完整的结果"""
    # 模拟一个"永远不够满意"的工具——Agent 可能反复调用
    return f"关于 '{query}' 的部分结果：找到一些相关信息，但可能不够全面，建议进一步搜索。"


tools = [search]
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个搜索助手。尽可能全面地回答问题。如果搜索结果不够，继续搜索。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
agent = create_tool_calling_agent(llm, tools, prompt)


# ===== 对比不同的 max_iterations 设置 =====

# 设置 1：宽松限制（默认 15）
executor_loose = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=15,  # 默认值
    verbose=True,
    return_intermediate_steps=True,
)

# 设置 2：严格限制
executor_strict = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=3,  # 最多 3 轮
    verbose=True,
    return_intermediate_steps=True,
)


if __name__ == "__main__":
    question = "详细介绍 Python 3.13 的所有新特性"

    # 严格限制：3 轮后强制停止
    print("=" * 60)
    print("严格限制：max_iterations=3")
    print("=" * 60)
    result = executor_strict.invoke({"input": question})
    print(f"\n工具调用次数: {len(result['intermediate_steps'])}")
    print(f"最终输出: {result['output'][:100]}...")
    # 如果 Agent 在 3 轮内没有给出最终答案，
    # 输出会是 "Agent stopped due to iteration limit or time limit."
```

### 预期输出

```
============================================================
严格限制：max_iterations=3
============================================================

> Entering new AgentExecutor chain...

Invoking: `search` with `{'query': 'Python 3.13 新特性'}`
关于 'Python 3.13 新特性' 的部分结果：找到一些相关信息...

Invoking: `search` with `{'query': 'Python 3.13 详细特性列表'}`
关于 'Python 3.13 详细特性列表' 的部分结果：找到一些相关信息...

Invoking: `search` with `{'query': 'Python 3.13 完整更新日志'}`
关于 'Python 3.13 完整更新日志' 的部分结果：找到一些相关信息...

> Finished chain.

工具调用次数: 3
最终输出: Agent stopped due to iteration limit or time limit....
```

> **关键观察：** Agent 连续搜索了 3 次，每次都觉得结果不够全面想继续搜。但 `max_iterations=3` 在第 3 轮后强制停止，返回了默认的停止信息。这就是 `early_stopping_method="force"` 的效果。

## 示例2：early_stopping_method——超限后的善后策略

**解决的问题：** `"force"` 返回的固定字符串对用户不友好。`"generate"` 让 LLM 基于已有信息给出一个"尽力而为"的答案。

```python
"""
early_stopping_method 对比
演示："force" vs "generate" 在超限后的不同表现
"""
from dotenv import load_dotenv
load_dotenv()

from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate


@tool
def search(query: str) -> str:
    """搜索信息"""
    return f"搜索结果：关于 '{query}' 的信息——这是一段模拟的搜索内容，包含部分相关信息。"


tools = [search]
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个搜索助手。尽可能全面地回答。如果信息不够，继续搜索不同角度。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
agent = create_tool_calling_agent(llm, tools, prompt)


# 策略 1：force（默认）——直接返回固定字符串
executor_force = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=2,
    early_stopping_method="force",
    verbose=False,
)

# 策略 2：generate——让 LLM 基于已有步骤总结答案
executor_generate = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=2,
    early_stopping_method="generate",
    verbose=False,
)


if __name__ == "__main__":
    question = "什么是 RAG？有哪些应用场景？"

    print("=" * 60)
    print('策略 1：early_stopping_method="force"')
    print("=" * 60)
    result_force = executor_force.invoke({"input": question})
    print(f"输出: {result_force['output']}")

    print(f"\n{'=' * 60}")
    print('策略 2：early_stopping_method="generate"')
    print("=" * 60)
    result_gen = executor_generate.invoke({"input": question})
    print(f"输出: {result_gen['output'][:200]}...")
```

### 预期输出

```
============================================================
策略 1：early_stopping_method="force"
============================================================
输出: Agent stopped due to iteration limit or time limit.

============================================================
策略 2：early_stopping_method="generate"
============================================================
输出: 根据搜索到的信息，RAG（检索增强生成）是一种结合检索和生成的技术...
```

> **选择建议：**
> - 内部工具 / 调试阶段 → `"force"`（快速失败，零额外成本）
> - 面向用户的产品 → `"generate"`（多花一次 LLM 调用，但用户体验好得多）

## 示例3：handle_parsing_errors——让 Agent 自我修复

**解决的问题：** LLM 输出格式不对时，默认行为是直接抛异常。`handle_parsing_errors=True` 让 Agent 把错误反馈给 LLM，LLM 自己修正格式后重试。

```python
"""
handle_parsing_errors 演示
演示：三种错误处理模式的效果对比
"""
from dotenv import load_dotenv
load_dotenv()

from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.exceptions import OutputParserException


@tool
def get_weather(city: str) -> str:
    """获取城市天气信息"""
    weather_data = {"北京": "晴天 25°C", "上海": "多云 22°C"}
    return weather_data.get(city, f"未找到 {city} 的天气数据")


@tool
def calculate(expression: str) -> str:
    """计算数学表达式。输入必须是合法的数学表达式。"""
    try:
        result = eval(expression, {"__builtins__": {}})
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


tools = [get_weather, calculate]
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有用的中文助手。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
agent = create_tool_calling_agent(llm, tools, prompt)


# ===== 模式 1：handle_parsing_errors=False（默认）=====
# 解析错误时直接抛异常
executor_no_handle = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=False,
)

# ===== 模式 2：handle_parsing_errors=True =====
# 自动将错误信息反馈给 LLM，让它自我修正
executor_auto = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=True,
    max_iterations=5,  # 给 LLM 几次修正机会
    verbose=True,
)

# ===== 模式 3：handle_parsing_errors=自定义字符串 =====
# 用自定义的错误提示引导 LLM 修正
executor_custom = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=(
        "你的输出格式不正确。请使用工具时严格按照要求格式输出，"
        "或者直接给出最终答案。"
    ),
    max_iterations=5,
    verbose=True,
)

# ===== 模式 4：handle_parsing_errors=自定义函数 =====
# 根据错误类型给出不同的修正指导
def smart_error_handler(error: OutputParserException) -> str:
    """根据错误类型给出精确的修正指导"""
    error_text = str(error)
    if "Could not parse" in error_text:
        return "输出格式错误。请直接调用工具或给出最终答案，不要输出多余内容。"
    elif "action_input" in error_text:
        return "工具输入格式错误。请确保输入是有效的字符串或 JSON。"
    else:
        return f"发生了解析错误：{error_text[:100]}。请重新组织你的回复。"


executor_smart = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=smart_error_handler,
    max_iterations=5,
    verbose=True,
)


if __name__ == "__main__":
    # 正常场景：所有模式都能正常工作
    print("=" * 60)
    print("正常场景（handle_parsing_errors=True）")
    print("=" * 60)
    result = executor_auto.invoke({"input": "北京天气怎么样？"})
    print(f"输出: {result['output']}")

    # 展示四种模式的配置方式
    print(f"\n{'=' * 60}")
    print("四种 handle_parsing_errors 配置方式")
    print("=" * 60)
    print("""
    # 模式 1：False（默认）——不处理，直接抛异常
    handle_parsing_errors=False

    # 模式 2：True——自动反馈错误信息给 LLM
    handle_parsing_errors=True

    # 模式 3：字符串——用自定义提示引导 LLM
    handle_parsing_errors="你的格式不对，请重试..."

    # 模式 4：函数——根据错误类型动态处理
    handle_parsing_errors=smart_error_handler
    """)
```

> **自修复流程：**
> ```
> LLM 输出格式错误
>   → OutputParserException
>   → handle_parsing_errors 捕获
>   → 错误信息作为 observation 反馈给 LLM
>   → LLM 看到错误，修正格式
>   → 重新输出（可能成功，也可能再次失败）
>   → 直到成功或达到 max_iterations
> ```
>
> 这对应源码中 `_iter_next_step` 的 `except OutputParserException` 分支，
> 错误被包装成 `AgentAction(tool="_Exception")` 并通过 `ExceptionTool` 处理。

## 示例4：组合配置——生产环境推荐方案

**解决的问题：** 把所有控制参数组合起来，形成一个适合生产环境的配置。

```python
"""
生产环境推荐配置
演示：多个控制参数协同工作
"""
from dotenv import load_dotenv
load_dotenv()

from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate


@tool
def search_docs(query: str) -> str:
    """搜索知识库文档"""
    docs = {
        "RAG": "RAG 通过检索外部知识增强 LLM 生成，核心流程：索引→检索→生成。",
        "Embedding": "Embedding 将文本映射到向量空间，语义相近的文本距离更近。",
        "Chunking": "文本分块常见策略：固定大小、语义分块、递归分块。",
    }
    for key, value in docs.items():
        if key.lower() in query.lower():
            return value
    return f"未找到与 '{query}' 相关的文档"


@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    try:
        result = eval(expression, {"__builtins__": {}})
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


tools = [search_docs, calculate]
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个 RAG 学习助手。用中文简洁回答。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])
agent = create_tool_calling_agent(llm, tools, prompt)


# ===== 开发环境配置 =====
dev_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=25,              # 宽松，方便调试
    max_execution_time=120.0,       # 2 分钟
    early_stopping_method="force",  # 快速看到是否超限
    handle_parsing_errors=False,    # 不处理，直接看原始错误
    verbose=True,                   # 打印每一步
    return_intermediate_steps=True, # 返回推理过程
)

# ===== 生产环境配置 =====
prod_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=10,              # 严格限制
    max_execution_time=30.0,        # 30 秒 SLA
    early_stopping_method="generate",  # 超限也给用户有意义的回答
    handle_parsing_errors=True,     # 自动处理解析错误
    verbose=False,                  # 不打印日志
    return_intermediate_steps=False,# 不返回中间步骤
)

# ===== 批处理配置 =====
batch_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=20,              # 允许更多步骤
    max_execution_time=300.0,       # 5 分钟
    early_stopping_method="force",  # 失败就跳过
    handle_parsing_errors=True,     # 自动处理
    verbose=False,
)


if __name__ == "__main__":
    # 用生产配置运行
    print("=" * 60)
    print("生产环境配置测试")
    print("=" * 60)

    questions = [
        "什么是 RAG？",
        "Embedding 和 Chunking 有什么关系？",
        "计算 1024 * 768",
        "你好",  # 纯对话，不需要工具
    ]

    for q in questions:
        result = prod_executor.invoke({"input": q})
        print(f"\nQ: {q}")
        print(f"A: {result['output'][:100]}")
        print("-" * 40)
```
