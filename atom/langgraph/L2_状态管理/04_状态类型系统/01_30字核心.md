# 01_30字核心

## 30字核心定义

**LangGraph 状态类型系统通过 TypedDict/Pydantic/dataclass 定义状态结构，用泛型参数和 Annotated 元数据实现类型安全的状态管理。**

[来源: reference/source_状态类型系统_01.md]

---

## 5个关键点

### 1. 三种类型定义方式

**TypedDict（轻量快速）、Pydantic（运行时验证）、dataclass（默认值支持）**

[来源: reference/search_状态类型系统_01.md, reference/source_状态类型系统_01.md]

### 2. 四个泛型参数

**StateT（状态）、ContextT（上下文）、InputT（输入）、OutputT（输出）**

[来源: reference/source_状态类型系统_01.md]

### 3. Annotated 绑定 Reducer

**通过 `__metadata__` 属性将 Reducer 函数绑定到字段，实现状态累积**

[来源: reference/source_状态类型系统_01.md, reference/context7_typing_extensions_01.md]

### 4. 类型修饰符控制

**Required/NotRequired（必需性）、ReadOnly（只读）、Optional（可选）**

[来源: reference/context7_typing_extensions_01.md, reference/source_状态类型系统_01.md]

### 5. 自动类型推断

**从函数签名推断输入类型，优先级：显式 > 推断 > 默认**

[来源: reference/source_状态类型系统_01.md]

## 为什么重要

### 理由 1：类型安全保障

静态检查 + 运行时验证双重保障，减少 80% 的类型错误。IDE 提示错误避免将错误代码提交，Pydantic 捕获无效数据防止系统崩溃。

[来源: reference/context7_pydantic_01.md, reference/search_状态类型系统_01.md]

### 理由 2：灵活的状态管理

支持 TypedDict、Pydantic、dataclass 三种定义方式，通过泛型参数分离状态、上下文、输入、输出，满足从简单到复杂的各种场景。

[来源: reference/source_状态类型系统_01.md, reference/search_状态类型系统_01.md]

### 理由 3：自动化与智能推断

自动从函数签名推断输入类型，通过 Annotated 元数据绑定 Reducer 函数，减少 50% 的样板代码，提高开发效率。

[来源: reference/source_状态类型系统_01.md, reference/context7_typing_extensions_01.md]

## 与其他概念的关系

### 与 Python 类型系统

基于 `typing` 和 `typing_extensions`，扩展了 `Annotated` 的语义，兼容 PEP 655（Required/NotRequired）和 PEP 705（ReadOnly）。

[来源: reference/context7_typing_extensions_01.md, reference/source_状态类型系统_01.md]

### 与 Pydantic

使用 `model_fields_set` 跟踪显式设置的字段，只更新与默认值不同的字段，特殊处理 `None` 值。

[来源: reference/context7_pydantic_01.md, reference/source_状态类型系统_01.md]

### 与 StateGraph

StateGraph 的泛型参数（StateT、ContextT、InputT、OutputT）直接依赖类型系统，类型定义决定了状态管理的行为。

[来源: reference/source_状态类型系统_01.md]

### 与 Reducer 函数

通过 `Annotated[type, reducer]` 将 Reducer 函数绑定到字段，Reducer 函数必须是二元操作符（接受两个参数）。

[来源: reference/source_状态类型系统_01.md, reference/context7_typing_extensions_01.md]

## 快速示例

```python
from typing_extensions import TypedDict, Annotated
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph

# 方式1: TypedDict（推荐）
class State1(TypedDict):
    messages: Annotated[list, lambda x, y: x + y]
    user_id: str

# 方式2: Pydantic（边界验证）
class State2(BaseModel):
    query: str = Field(..., min_length=1, max_length=1000)
    user_id: str = Field(..., pattern=r'^[a-zA-Z0-9_-]+$')

# 方式3: 泛型类型系统
class Input(TypedDict):
    query: str

class Output(TypedDict):
    result: str

graph = StateGraph(
    state_schema=State1,
    input_schema=Input,
    output_schema=Output
)
```

[来源: reference/source_状态类型系统_01.md, reference/search_状态类型系统_01.md]

## 核心设计原则

1. **TypedDict 优先**：轻量、快速、官方推荐
2. **边界用 Pydantic**：防止无效数据进入系统
3. **类型修饰符精细控制**：Required/NotRequired/ReadOnly
4. **Reducer 实现累积**：消息列表、历史记录等
5. **泛型类型分离关注点**：State/Context/Input/Output
