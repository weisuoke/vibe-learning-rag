# 30字核心

> **一句话理解数据插入与查询**

---

## 核心本质（30字）

**Milvus 2.6 让你直接插入原始文本并查询，自动调用嵌入模型完成向量化，3步替代传统5步流程。**

---

## 拆解理解

### 传统方式（5步）
```
1. 准备原始数据（文本）
2. 调用嵌入API生成向量
3. 插入向量到Milvus
4. 查询时生成查询向量
5. 映射结果回原始数据
```

### Milvus 2.6 方式（3步）
```
1. 直接插入原始文本 → Milvus自动生成向量
2. 直接用文本查询 → Milvus自动生成查询向量
3. 获取结果 → 自动返回原始数据
```

---

## 核心价值

### 对开发者
- **省代码**: 不用写嵌入调用逻辑
- **省时间**: 不用管理嵌入服务
- **省心**: 插入和查询自动使用同一模型

### 对RAG应用
- **更简单**: 3行代码完成文档问答
- **更一致**: 避免插入和查询用不同模型
- **更高效**: 减少网络往返次数

---

## 一个例子说明

### 传统方式代码
```python
# 步骤1: 手动生成嵌入
from openai import OpenAI
client = OpenAI()

docs = ["Milvus is a vector database"]
embeddings = []
for doc in docs:
    emb = client.embeddings.create(
        input=doc,
        model="text-embedding-3-small"
    )
    embeddings.append(emb.data[0].embedding)

# 步骤2: 插入向量
milvus_client.insert(collection, {
    "id": 1,
    "vector": embeddings[0],
    "text": docs[0]
})

# 步骤3: 查询时再次生成嵌入
query = "What is Milvus?"
query_emb = client.embeddings.create(
    input=query,
    model="text-embedding-3-small"
).data[0].embedding

# 步骤4: 搜索
results = milvus_client.search(
    collection,
    data=[query_emb],
    limit=3
)
```

### Milvus 2.6 方式代码
```python
# 步骤1: 配置一次嵌入函数（在创建collection时）
# 步骤2: 直接插入原始文本
milvus_client.insert(collection, {
    "id": 1,
    "document": "Milvus is a vector database"  # 原始文本
})

# 步骤3: 直接用文本查询
results = milvus_client.search(
    collection,
    data=["What is Milvus?"],  # 原始文本查询
    anns_field="dense",
    limit=3
)
```

**代码量对比**: 20行 vs 5行，减少75%

## 适用场景

✅ **适合使用 Embedding Functions**:
- 文档问答系统
- 知识库检索
- 语义搜索
- 内容推荐
- 智能客服

❌ **不适合使用**:
- 需要自定义嵌入逻辑
- 使用非标准嵌入模型
- 需要嵌入前预处理（如特殊分词）

---

## 关键术语

| 术语 | 含义 | 类比 |
|------|------|------|
| **Embedding Functions** | Milvus 2.6内置的自动向量化功能 | 自动挡汽车 vs 手动挡 |
| **Data-in, Data-out** | 输入输出都是原始数据，不是向量 | 输入中文，输出中文（不用管中间的编码） |
| **Provider** | 嵌入模型提供商（OpenAI, Cohere等） | 外卖平台上的不同餐厅 |
