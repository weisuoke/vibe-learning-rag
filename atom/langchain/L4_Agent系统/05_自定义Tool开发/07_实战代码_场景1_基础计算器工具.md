- 工具的输入输出机制

## 完整代码实现

### 场景 1.1：最简单的计算器

```python
"""
场景 1.1：使用 @tool 装饰器创建最简单的计算器工具
演示：基础装饰器用法、类型提示、docstring 编写
"""

from langchain.tools import tool

# ===== 1. 最简单的工具定义 =====
@tool
def calculator(expression: str) -> str:
    """Evaluate mathematical expressions.

    Args:
        expression: Math expression like "2 + 2" or "10 * 5"

    Returns:
        Calculation result as string
    """
    try:
        # 使用 eval 计算表达式（注意：生产环境需要更安全的方法）
        result = eval(expression)
        return f"Result: {result}"
    except Exception as e:
        return f"Error: {str(e)}"


# ===== 2. 测试工具 =====
print("=== 测试 1：基础计算 ===")
result1 = calculator.invoke({"expression": "2 + 2"})
print(f"输入: 2 + 2")
print(f"输出: {result1}\n")

result2 = calculator.invoke({"expression": "10 * 5 + 3"})
print(f"输入: 10 * 5 + 3")
print(f"输出: {result2}\n")

# ===== 3. 查看工具的自动生成 schema =====
print("=== 工具 Schema（自动推断）===")
print(f"工具名称: {calculator.name}")
print(f"工具描述: {calculator.description}")
print(f"参数 Schema:")
import json
print(json.dumps(calculator.args_schema.model_json_schema(), indent=2))
```

**运行输出**：
```
=== 测试 1：基础计算 ===
输入: 2 + 2
输出: Result: 4

输入: 10 * 5 + 3
输出: Result: 53

=== 工具 Schema（自动推断）===
工具名称: calculator
工具描述: Evaluate mathematical expressions.
参数 Schema:
{
  "type": "object",
  "properties": {
    "expression": {
      "type": "string",
      "description": "Math expression like \"2 + 2\" or \"10 * 5\""
    }
  },
  "required": ["expression"]
}
```

**关键点解析**：
1. **@tool 装饰器**：自动将函数转换为 LangChain 工具
2. **类型提示**：`expression: str` 告诉 LangChain 参数类型
3. **Docstring**：Google-style 格式，自动解析为工具描述和参数说明
4. **invoke() 方法**：工具的标准调用接口，接受字典参数

### 场景 1.2：自定义工具名称和描述

```python
"""
场景 1.2：自定义工具名称和描述
演示：装饰器参数、更清晰的工具定义
"""

from langchain.tools import tool

# ===== 1. 自定义名称和描述 =====
@tool(
    "math_calculator",  # 自定义工具名称
    description="Performs arithmetic calculations. Use this for any math problems like addition, subtraction, multiplication, division."
)
def calc(expression: str) -> str:
    """Evaluate mathematical expressions safely."""
    try:
        # 限制可用的函数，提高安全性
        allowed_names = {
            "abs": abs,
            "round": round,
            "min": min,
            "max": max,
        }
        result = eval(expression, {"__builtins__": {}}, allowed_names)
        return f"Result: {result}"
    except Exception as e:
        return f"Error: {str(e)}"


# ===== 2. 测试自定义工具 =====
print("=== 测试 2：自定义名称和描述 ===")
print(f"工具名称: {calc.name}")
print(f"工具描述: {calc.description}\n")

result = calc.invoke({"expression": "max(10, 20, 5)"})
print(f"输入: max(10, 20, 5)")
print(f"输出: {result}")
```

**运行输出**：
```
=== 测试 2：自定义名称和描述 ===
工具名称: math_calculator
工具描述: Performs arithmetic calculations. Use this for any math problems like addition, subtraction, multiplication, division.

输入: max(10, 20, 5)
输出: Result: 20
```

**为什么需要自定义描述？**
- LLM 通过描述决定何时调用工具
- 清晰的描述 = 更准确的工具选择
- 描述应该说明"何时使用"而非"如何使用"

### 场景 1.3：多参数计算器

```python
"""
场景 1.3：多参数工具
演示：使用 Pydantic 定义复杂输入 schema
"""

from langchain.tools import tool
from pydantic import BaseModel, Field
from typing import Literal

# ===== 1. 定义输入 Schema =====
class CalculatorInput(BaseModel):
    """Input schema for advanced calculator."""
    operation: Literal["add", "subtract", "multiply", "divide"] = Field(
        description="Mathematical operation to perform"
    )
    a: float = Field(description="First number")
    b: float = Field(description="Second number")
    precision: int = Field(
        default=2,
        description="Number of decimal places in result"
    )


# ===== 2. 创建工具 =====
@tool(args_schema=CalculatorInput)
def advanced_calculator(
    operation: str,
    a: float,
    b: float,
    precision: int = 2
) -> str:
    """Perform arithmetic operations with configurable precision.

    Supports addition, subtraction, multiplication, and division.
    """
    operations = {
        "add": lambda x, y: x + y,
        "subtract": lambda x, y: x - y,
        "multiply": lambda x, y: x * y,
        "divide": lambda x, y: x / y if y != 0 else None,
    }

    if operation not in operations:
        return f"Error: Unknown operation '{operation}'"

    result = operations[operation](a, b)

    if result is None:
        return "Error: Division by zero"

    # 格式化结果
    formatted_result = round(result, precision)
    return f"{a} {operation} {b} = {formatted_result}"


# ===== 3. 测试多参数工具 =====
print("=== 测试 3：多参数计算器 ===")

test_cases = [
    {"operation": "add", "a": 10.5, "b": 5.3, "precision": 1},
    {"operation": "multiply", "a": 7, "b": 8, "precision": 0},
    {"operation": "divide", "a": 22, "b": 7, "precision": 3},
]

for test in test_cases:
    result = advanced_calculator.invoke(test)
    print(f"输入: {test}")
    print(f"输出: {result}\n")

# ===== 4. 查看 Schema =====
print("=== 参数 Schema ===")
import json
print(json.dumps(advanced_calculator.args_schema.model_json_schema(), indent=2))
```

**运行输出**：
```
=== 测试 3：多参数计算器 ===
输入: {'operation': 'add', 'a': 10.5, 'b': 5.3, 'precision': 1}
输出: 10.5 add 5.3 = 15.8

输入: {'operation': 'multiply', 'a': 7, 'b': 8, 'precision': 0}
输出: 7 multiply 8 = 56

输入: {'operation': 'divide', 'a': 22, 'b': 7, 'precision': 3}
输出: 22 divide 7 = 3.143

=== 参数 Schema ===
{
  "type": "object",
  "properties": {
    "operation": {
      "type": "string",
      "enum": ["add", "subtract", "multiply", "divide"],
      "description": "Mathematical operation to perform"
    },
    "a": {
      "type": "number",
      "description": "First number"
    },
    "b": {
      "type": "number",
      "description": "Second number"
    },
    "precision": {
      "type": "integer",
      "default": 2,
      "description": "Number of decimal places in result"
    }
  },
  "required": ["operation", "a", "b"]
}
```

**Pydantic Schema 的优势**：
1. **类型约束**：`Literal` 限制 operation 只能是 4 个值之一
2. **默认值**：precision 默认为 2，可选参数
3. **详细描述**：每个参数都有清晰的说明
4. **自动验证**：LangChain 自动验证输入是否符合 schema

### 场景 1.4：与 Agent 集成

```python
"""
场景 1.4：将计算器工具集成到 Agent
演示：工具如何被 LLM 调用
"""

import os
from langchain.tools import tool
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_core.prompts import ChatPromptTemplate

# ===== 1. 定义工具 =====
@tool
def calculator(expression: str) -> str:
    """Evaluate mathematical expressions like "2 + 2" or "10 * 5 + 3"."""
    try:
        result = eval(expression)
        return f"Result: {result}"
    except Exception as e:
        return f"Error: {str(e)}"


# ===== 2. 创建 Agent =====
# 配置 LLM
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0,
    api_key=os.getenv("OPENAI_API_KEY")
)

# 定义 Prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant with access to a calculator tool. Use it when you need to perform calculations."),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

# 创建 Agent
tools = [calculator]
agent = create_tool_calling_agent(llm, tools, prompt)
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,  # 显示详细执行过程
)


# ===== 3. 测试 Agent =====
print("=== 测试 4：Agent 调用计算器 ===\n")

questions = [
    "What is 25 multiplied by 4?",
    "Calculate the sum of 123 and 456",
    "If I have 100 dollars and spend 37, how much do I have left?",
]

for question in questions:
    print(f"问题: {question}")
    result = agent_executor.invoke({"input": question})
    print(f"回答: {result['output']}\n")
    print("-" * 60 + "\n")
```

**运行输出示例**：
```
=== 测试 4：Agent 调用计算器 ===

问题: What is 25 multiplied by 4?

> Entering new AgentExecutor chain...

Invoking: `calculator` with `{'expression': '25 * 4'}`

Result: 100

The result of 25 multiplied by 4 is **100**.

> Finished chain.
回答: The result of 25 multiplied by 4 is **100**.

------------------------------------------------------------

问题: Calculate the sum of 123 and 456

> Entering new AgentExecutor chain...

Invoking: `calculator` with `{'expression': '123 + 456'}`

Result: 579

The sum of 123 and 456 is **579**.

> Finished chain.
回答: The sum of 123 and 456 is **579**.

------------------------------------------------------------
```

**Agent 执行流程解析**：
1. **LLM 理解问题**：识别需要计算
2. **决策调用工具**：选择 calculator 工具
3. **构造工具输入**：生成 `{'expression': '25 * 4'}`
4. **执行工具**：调用 calculator.invoke()
5. **处理结果**：将 "Result: 100" 返回给 LLM
6. **生成最终回答**：LLM 用自然语言回答用户

## 核心知识点总结

### 1. @tool 装饰器的三种用法

```python
# 用法 1：无参数装饰器（最简单）
@tool
def simple_tool(param: str) -> str:
    """Tool description."""
    return f"Result: {param}"

# 用法 2：自定义名称和描述
@tool("custom_name", description="Custom description")
def named_tool(param: str) -> str:
    return f"Result: {param}"

# 用法 3：显式指定 schema
@tool(args_schema=MySchema)
def schema_tool(param: str) -> str:
    return f"Result: {param}"
```

### 2. Schema 自动推断机制

LangChain 通过以下方式推断 schema：
1. **函数签名**：提取参数名和类型提示
2. **Docstring**：解析 Google-style 格式的参数描述
3. **Pydantic 模型**：如果提供 args_schema，直接使用

### 3. 工具调用流程

```
用户问题 → LLM 分析 → 决定调用工具 → 构造输入 → 执行工具 → 返回结果 → LLM 生成回答
```

## 最佳实践

### ✅ 推荐做法

1. **清晰的 Docstring**
```python
@tool
def good_tool(query: str) -> str:
    """Search for information about a topic.

    Args:
        query: The search query, e.g., "Python tutorials"

    Returns:
        Search results as formatted text
    """
    pass
```

2. **使用 Pydantic 约束输入**
```python
class ToolInput(BaseModel):
    param: Literal["option1", "option2"] = Field(description="Choose an option")

@tool(args_schema=ToolInput)
def constrained_tool(param: str) -> str:
    pass
```

3. **错误处理**
```python
@tool
def safe_tool(param: str) -> str:
    """Tool with error handling."""
    try:
        result = risky_operation(param)
        return f"Success: {result}"
    except Exception as e:
        return f"Error: {str(e)}"
```

### ❌ 避免的做法

1. **缺少类型提示**
```python
@tool
def bad_tool(param):  # ❌ 没有类型提示
    return param
```

2. **模糊的描述**
```python
@tool
def vague_tool(x: str) -> str:
    """Does something."""  # ❌ 描述不清晰
    return x
```

3. **不安全的 eval**
```python
@tool
def unsafe_tool(code: str) -> str:
    return eval(code)  # ❌ 生产环境禁止直接 eval
```
