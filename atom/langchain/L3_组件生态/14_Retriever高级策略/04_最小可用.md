## 4.1 EnsembleRetriever 混合检索（最常用）

### 核心思路

```
混合检索 = BM25（关键词匹配）+ VectorStore（语义匹配）+ RRF 融合

BM25 擅长：精确关键词、专有名词、代码片段
向量检索擅长：语义理解、同义词、模糊表达
两者互补，覆盖几乎所有查询场景
```

### 推荐权重

```python
# 通用场景推荐权重
weights = [0.4, 0.6]  # BM25 0.4 + 向量 0.6

# 技术文档场景（关键词更重要）
weights = [0.5, 0.5]

# 对话式查询场景（语义更重要）
weights = [0.3, 0.7]
```

### 完整代码示例

```python
"""
EnsembleRetriever 混合检索 - 最小可用示例
BM25 + ChromaDB 向量检索，RRF 自动融合
"""

from langchain_community.retrievers import BM25Retriever
from langchain.retrievers import EnsembleRetriever
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_core.documents import Document

# ===== 1. 准备文档 =====
docs = [
    Document(page_content="Python 的 asyncio 库提供了异步编程支持"),
    Document(page_content="FastAPI 是基于 Starlette 的高性能 Web 框架"),
    Document(page_content="async/await 语法让异步代码像同步一样易读"),
    Document(page_content="Django 是 Python 最流行的全栈 Web 框架"),
    Document(page_content="协程是 Python 异步编程的核心概念"),
    Document(page_content="uvicorn 是 ASGI 服务器，常与 FastAPI 搭配使用"),
]

# ===== 2. 创建两个检索器 =====
# BM25 检索器（关键词匹配）
bm25_retriever = BM25Retriever.from_documents(docs, k=4)

# 向量检索器（语义匹配）
vectorstore = Chroma.from_documents(docs, OpenAIEmbeddings())
vector_retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

# ===== 3. 创建混合检索器（一行代码） =====
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.4, 0.6],  # BM25 权重 0.4，向量权重 0.6
)

# ===== 4. 检索 =====
results = ensemble_retriever.invoke("Python 异步编程怎么用？")

for i, doc in enumerate(results, 1):
    print(f"{i}. {doc.page_content}")
```

**预期输出：**
```
1. Python 的 asyncio 库提供了异步编程支持
2. async/await 语法让异步代码像同步一样易读
3. 协程是 Python 异步编程的核心概念
4. FastAPI 是基于 Starlette 的高性能 Web 框架
```

**关键点：**
- `weights` 控制两个检索器的相对重要性，总和不需要为 1
- RRF 融合在内部自动完成，`c=60` 是默认值
- `k` 参数分别控制每个子检索器的召回数量

## 4.2 ContextualCompressionRetriever 重排序（第二常用）

### 核心思路

```
重排序 = 先粗筛（多召回）+ 再精排（精选 Top-N）

第一阶段：基础检索器召回 20 篇文档（快但粗）
第二阶段：重排序模型精选 5 篇文档（慢但准）
```

### FlashrankRerank（推荐：免费本地方案）

```python
"""
ContextualCompressionRetriever + FlashrankRerank
免费本地重排序，无需 API Key
"""

from langchain.retrievers import ContextualCompressionRetriever
from langchain_community.document_compressors import FlashrankRerank
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_core.documents import Document

# ===== 1. 准备文档和基础检索器 =====
docs = [
    Document(page_content="Python 的 asyncio 库提供了异步编程支持，包括事件循环和协程"),
    Document(page_content="JavaScript 的 Promise 和 async/await 也支持异步编程"),
    Document(page_content="Python 的 GIL 限制了多线程的并行执行能力"),
    Document(page_content="asyncio.run() 是启动异步程序的推荐方式"),
    Document(page_content="多进程可以绕过 GIL 实现真正的并行计算"),
    Document(page_content="协程比线程更轻量，适合 I/O 密集型任务"),
]

vectorstore = Chroma.from_documents(docs, OpenAIEmbeddings())

# 基础检索器：多召回（k=6，尽量不遗漏）
base_retriever = vectorstore.as_retriever(search_kwargs={"k": 6})

# ===== 2. 创建重排序压缩器 =====
compressor = FlashrankRerank(
    top_n=3,  # 精选 Top-3
    model="ms-marco-MultiBERT-L-12",  # 默认模型，无需下载
)

# ===== 3. 组合：基础检索器 + 重排序 =====
compression_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=base_retriever,
)

# ===== 4. 检索（自动完成粗筛 + 精排） =====
results = compression_retriever.invoke("Python 异步编程")

for i, doc in enumerate(results, 1):
    print(f"{i}. [相关度: {doc.metadata.get('relevance_score', 'N/A'):.4f}]")
    print(f"   {doc.page_content}")
```

**预期输出：**
```
1. [相关度: 0.9812]
   Python 的 asyncio 库提供了异步编程支持，包括事件循环和协程
2. [相关度: 0.8934]
   asyncio.run() 是启动异步程序的推荐方式
3. [相关度: 0.8567]
   协程比线程更轻量，适合 I/O 密集型任务
```

**关键点：**
- `top_n` 控制最终返回的文档数量
- FlashrankRerank 是免费本地方案，不需要 API Key
- 重排序后的文档会带上 `relevance_score` 元数据

## 4.3 MultiQueryRetriever 查询扩展

### 核心思路

```
查询扩展 = LLM 生成多个查询变体 → 分别检索 → 合并去重

用户查询: "Python 异步编程"
  → 变体1: "Python asyncio 库的使用方法"
  → 变体2: "如何在 Python 中编写异步代码"
  → 变体3: "Python 协程和事件循环的工作原理"
  → 分别检索 → 合并去重 → 返回更全面的结果
```

### from_llm 一行创建

```python
"""
MultiQueryRetriever - 查询扩展
LLM 自动生成多个查询变体，提高召回率
"""

from langchain.retrievers.multi_query import MultiQueryRetriever
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_chroma import Chroma
from langchain_core.documents import Document

# ===== 1. 准备文档和基础检索器 =====
docs = [
    Document(page_content="asyncio 是 Python 标准库中的异步 I/O 框架"),
    Document(page_content="协程通过 async def 定义，用 await 暂停执行"),
    Document(page_content="事件循环是 asyncio 的核心，负责调度协程"),
    Document(page_content="aiohttp 是基于 asyncio 的 HTTP 客户端/服务端库"),
    Document(page_content="Python 3.11 引入了 TaskGroup 简化并发任务管理"),
    Document(page_content="异步生成器用 async for 迭代，适合流式数据处理"),
]

vectorstore = Chroma.from_documents(docs, OpenAIEmbeddings())
base_retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# ===== 2. 一行创建 MultiQueryRetriever =====
llm = ChatOpenAI(temperature=0, model="gpt-4o-mini")

multi_query_retriever = MultiQueryRetriever.from_llm(
    retriever=base_retriever,
    llm=llm,
    include_original=True,  # 保留原始查询
)

# ===== 3. 检索（LLM 自动生成变体 + 多路检索 + 去重） =====
results = multi_query_retriever.invoke("Python 异步编程怎么入门？")

print(f"检索到 {len(results)} 篇去重后的文档：")
for i, doc in enumerate(results, 1):
    print(f"{i}. {doc.page_content}")
```

**预期输出：**
```
检索到 5 篇去重后的文档：
1. asyncio 是 Python 标准库中的异步 I/O 框架
2. 协程通过 async def 定义，用 await 暂停执行
3. 事件循环是 asyncio 的核心，负责调度协程
4. Python 3.11 引入了 TaskGroup 简化并发任务管理
5. 异步生成器用 async for 迭代，适合流式数据处理
```

**关键点：**
- `from_llm` 内部自动构建查询生成链，默认生成 3 个变体
- `include_original=True` 保留原始查询，避免 LLM 改写偏离原意
- 多个变体的检索结果自动去重（基于 `page_content`）

## 4.4 三个关键参数

### 参数速查表

| 参数 | 所属组件 | 默认值 | 推荐值 | 作用 |
|------|----------|--------|--------|------|
| `weights` | EnsembleRetriever | 等权重 | `[0.4, 0.6]` | 控制 BM25 和向量检索的相对重要性 |
| `top_n` | FlashrankRerank | 3 | 3-5 | 重排序后返回的文档数量 |
| `k` / `search_kwargs` | 各种 Retriever | 4 | 10-20（粗筛）/ 3-5（精排） | 每个检索器的召回数量 |

### 参数调优口诀

```python
# 口诀：粗筛多召回，精排少而精

# 基础检索器：多召回（宁可多，不可漏）
base_retriever = vectorstore.as_retriever(search_kwargs={"k": 20})

# 重排序：精选（只留最相关的）
compressor = FlashrankRerank(top_n=5)

# 混合检索：权重看场景
# 技术文档 → BM25 权重高（关键词重要）
# 对话问答 → 向量权重高（语义重要）
ensemble = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.4, 0.6],
)
```

## 4.5 一个集成模式：LCEL 管道

### 完整 RAG 管道

```python
"""
LCEL 管道：Retriever → 格式化 → Prompt → LLM → 解析
这是生产环境中最常用的集成模式
"""

from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_chroma import Chroma
from langchain_core.documents import Document
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_core.runnables import RunnablePassthrough
from langchain_community.retrievers import BM25Retriever
from langchain.retrievers import EnsembleRetriever, ContextualCompressionRetriever
from langchain_community.document_compressors import FlashrankRerank

# ===== 1. 准备文档 =====
docs = [
    Document(page_content="LangChain 是一个用于构建 LLM 应用的框架，支持链式调用和 Agent"),
    Document(page_content="LCEL（LangChain Expression Language）用管道符 | 连接组件"),
    Document(page_content="Retriever 是 LangChain 中负责文档检索的抽象接口"),
    Document(page_content="ChromaDB 是一个轻量级的向量数据库，适合快速原型开发"),
    Document(page_content="EnsembleRetriever 组合多个检索器，使用 RRF 融合结果"),
    Document(page_content="ContextualCompressionRetriever 在检索后对文档进行重排序或压缩"),
]

# ===== 2. 构建混合检索 + 重排序 =====
bm25_retriever = BM25Retriever.from_documents(docs, k=4)
vectorstore = Chroma.from_documents(docs, OpenAIEmbeddings())
vector_retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

# 混合检索
ensemble_retriever = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.4, 0.6],
)

# 重排序
compressor = FlashrankRerank(top_n=3)
final_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=ensemble_retriever,
)

# ===== 3. 构建 LCEL 管道 =====
def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

prompt = ChatPromptTemplate.from_template("""
基于以下上下文回答问题。如果上下文中没有相关信息，请说"我不确定"。

上下文：
{context}

问题：{question}

回答：""")

llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 核心管道：retriever | format_docs | prompt | llm | parser
rag_chain = (
    {"context": final_retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

# ===== 4. 使用 =====
answer = rag_chain.invoke("LangChain 的 LCEL 是什么？")
print(f"回答：{answer}")
```

**预期输出：**
```
回答：LCEL（LangChain Expression Language）是 LangChain 中用管道符 | 连接组件的表达式语言。
它可以将 Retriever、Prompt、LLM 等组件串联成完整的处理管道。
```

**管道结构图：**
```
用户问题
    │
    ├──→ final_retriever（混合检索 + 重排序）──→ format_docs ──→ context
    │
    └──→ RunnablePassthrough() ──→ question
                │
                ▼
            prompt（模板填充）
                │
                ▼
            llm（生成回答）
                │
                ▼
            StrOutputParser（提取文本）
                │
                ▼
            最终回答
```

## 常见问题

### Q1: EnsembleRetriever 的 weights 怎么选？

**A**: 通用场景用 `[0.4, 0.6]`（向量略高）。技术文档场景用 `[0.5, 0.5]`。对话式查询用 `[0.3, 0.7]`。没有万能值，建议用评估集测试。

### Q2: FlashrankRerank 和 CohereRerank 怎么选？

**A**: FlashrankRerank 免费、本地运行、无需 API Key，适合开发和小规模生产。CohereRerank 效果更好但需要付费 API，适合对精度要求高的生产环境。

### Q3: MultiQueryRetriever 会增加多少延迟？

**A**: 主要增加一次 LLM 调用（生成查询变体）+ 多次检索。用 gpt-4o-mini 生成变体约 300-500ms，3 次检索约 100-200ms。总延迟约 500-700ms。

### Q4: 混合检索 + 重排序可以叠加使用吗？

**A**: 可以，而且推荐叠加。EnsembleRetriever 负责高召回粗筛，FlashrankRerank 负责高精度精排。4.5 节的 LCEL 管道就是这个模式。
