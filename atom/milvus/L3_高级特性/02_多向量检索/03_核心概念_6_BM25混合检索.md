# 核心概念 6：BM25 混合检索

## 概述

BM25 混合检索是 Milvus 多向量检索的一个重要应用场景，它结合了 BM25 稀疏向量（精确关键词匹配）和稠密向量（语义理解）的优势，实现了兼顾语义理解和精确匹配的高质量检索。

**核心价值**：
- **语义理解**：稠密向量捕获文本的深层语义关系
- **精确匹配**：BM25 稀疏向量实现精确关键词匹配
- **优势互补**：混合检索克服单一方法的局限性
- **自动化**：BM25 函数自动生成稀疏向量，无需手动处理

## 1. BM25 算法原理

### 1.1 什么是 BM25？

BM25（Best Matching 25）是一种经典的全文检索算法，广泛应用于搜索引擎中。它基于词频（TF）和逆文档频率（IDF）计算文档与查询的相关性分数。

**核心思想**：
- 查询词在文档中出现越多，相关性越高（但有饱和度限制）
- 查询词在整个语料库中越罕见，权重越高
- 考虑文档长度归一化，避免长文档占优势

### 1.2 BM25 公式

```
score(D, Q) = Σ IDF(qi) * (f(qi, D) * (k1 + 1)) / (f(qi, D) + k1 * (1 - b + b * |D| / avgdl))
```

**参数说明**：
- `D`：文档
- `Q`：查询
- `qi`：查询中的第 i 个词
- `f(qi, D)`：词 qi 在文档 D 中的频率
- `|D|`：文档 D 的长度
- `avgdl`：语料库中文档的平均长度
- `k1`：词频饱和度参数（通常取 1.2-2.0）
- `b`：长度归一化参数（通常取 0.75）
- `IDF(qi)`：词 qi 的逆文档频率

### 1.3 BM25 vs 稠密向量

| 维度 | BM25 稀疏向量 | 稠密向量 |
|------|---------------|----------|
| **表示方式** | 高维稀疏向量（大部分为 0） | 低维稠密向量（所有维度有值） |
| **语义理解** | 无语义理解，基于词匹配 | 深层语义理解 |
| **精确匹配** | 精确关键词匹配 | 可能遗漏精确词 |
| **同义词** | 无法识别同义词 | 可识别同义词 |
| **计算效率** | 高效（稀疏计算） | 相对较慢 |
| **适用场景** | 关键词搜索、专业术语 | 语义搜索、问答系统 |

## 2. Milvus BM25 函数

### 2.1 BM25 函数定义

Milvus 提供了内置的 BM25 函数，可以自动将文本字段转换为稀疏向量，无需手动计算。

**Python SDK 示例**：

```python
from pymilvus import Function, FunctionType, DataType

# 定义 BM25 函数
bm25_function = Function(
    name="text_bm25",                          # 函数名称
    function_type=FunctionType.BM25,           # 函数类型
    input_field_names=["text"],                # 输入文本字段
    output_field_names="sparse_vector",        # 输出稀疏向量字段
    params={}                                  # BM25 参数（可选）
)

# 将函数添加到 Schema
schema.add_function(bm25_function)
```

**关键点**：
- `input_field_names`：输入的文本字段，需要启用 `enable_analyzer=True`
- `output_field_names`：输出的稀疏向量字段，类型为 `SPARSE_FLOAT_VECTOR`
- `params`：BM25 参数配置（如 k1、b），默认使用标准值

### 2.2 完整 Schema 定义

```python
from pymilvus import CollectionSchema, FieldSchema, DataType, Function, FunctionType

# 创建 Schema
schema = CollectionSchema(
    fields=[
        # 主键字段
        FieldSchema(
            name="id",
            dtype=DataType.INT64,
            is_primary=True,
            auto_id=True
        ),

        # 文本字段（BM25 输入）
        FieldSchema(
            name="text",
            dtype=DataType.VARCHAR,
            max_length=65535,
            enable_analyzer=True,           # 启用分词器
            enable_match=True,              # 启用全文匹配
            analyzer_params={
                "type": "standard"          # 分词器类型
            }
        ),

        # 稠密向量字段
        FieldSchema(
            name="dense_vector",
            dtype=DataType.FLOAT_VECTOR,
            dim=768
        ),

        # 稀疏向量字段（BM25 输出）
        FieldSchema(
            name="sparse_vector",
            dtype=DataType.SPARSE_FLOAT_VECTOR
        )
    ],
    description="BM25 + Dense Vector Hybrid Search Collection"
)

# 添加 BM25 函数
bm25_function = Function(
    name="text_bm25",
    function_type=FunctionType.BM25,
    input_field_names=["text"],
    output_field_names="sparse_vector",
    params={}
)
schema.add_function(bm25_function)
```

**重要配置**：
- `enable_analyzer=True`：启用文本分词器，BM25 函数必需
- `enable_match=True`：启用全文匹配功能
- `analyzer_params`：分词器配置，支持 `standard`、`english`、`chinese` 等

### 2.3 数据插入

插入数据时，只需提供文本和稠密向量，BM25 函数会自动生成稀疏向量。

```python
from pymilvus import MilvusClient
from sentence_transformers import SentenceTransformer

# 初始化 Embedding 模型
embedding_model = SentenceTransformer('all-MiniLM-L6-v2')

# 插入数据

    {
        "text": text,
        "dense_vector": dense_vector

    for text, dense_vector in zip(texts, dense_vectors)

    collection_name="hybrid_collection",
    data=data
)

# 注意：sparse_vector 字段由 BM25 函数自动生成，无需手动提供
```

## 3. BM25 混合检索实现

### 3.1 基础混合检索

```python
from pymilvus import AnnSearchRequest, RRFRanker

def bm25_hybrid_search(
    client,
    collection_name,
    query_text,
    embedding_model,
    limit=10
):
    """
    BM25 + 稠密向量混合检索

    Args:
        client: MilvusClient 实例
        collection_name: Collection 名称
        query_text: 查询文本
        embedding_model: Embedding 模型
        limit: 返回结果数量

    Returns:
        检索结果列表
    """
    # 1. 生成查询向量
    query_dense_vector = embedding_model.encode([query_text])[0].tolist()

    # 2. 定义 BM25 稀疏向量检索请求
    sparse_request = AnnSearchRequest(
        data=[query_text],                      # 直接传入文本
        anns_field="sparse_vector",             # 稀疏向量字段
        param={"metric_type": "BM25"},          # 使用 BM25 度量
        limit=limit
    )

    # 3. 定义稠密向量检索请求
    dense_request = AnnSearchRequest(
        data=[query_dense_vector],              # 传入向量
        anns_field="dense_vector",              # 稠密向量字段
        param={"metric_type": "IP"},            # 使用内积度量
        limit=limit
    )

    # 4. 使用 RRF 融合结果
    results = client.hybrid_search(
        collection_name=collection_name,
        reqs=[sparse_request, dense_request],
        ranker=RRFRanker(),                     # RRF 排序器
        limit=limit,
        output_fields=["text"]
    )

    return results[0]
```

**关键点**：
- BM25 检索请求直接传入文本，不需要手动生成稀疏向量
- `metric_type: "BM25"` 指定使用 BM25 算法
- 稠密向量检索使用标准的向量检索方式
- RRFRanker 自动融合两个检索结果

### 3.2 使用 WeightedRanker

如果你知道稀疏向量和稠密向量的相对重要性，可以使用 WeightedRanker。

```python
from pymilvus import AnnSearchRequest, WeightedRanker

def bm25_hybrid_search_weighted(
    client,
    collection_name,
    query_text,
    embedding_model,
    sparse_weight=0.7,      # BM25 权重
    dense_weight=1.0,       # 稠密向量权重
    limit=10
):
    """
    使用加权排序器的 BM25 混合检索
    """
    # 生成查询向量
    query_dense_vector = embedding_model.encode([query_text])[0].tolist()

    # 定义检索请求
    sparse_request = AnnSearchRequest(
        data=[query_text],
        anns_field="sparse_vector",
        param={"metric_type": "BM25"},
        limit=limit
    )

    dense_request = AnnSearchRequest(
        data=[query_dense_vector],
        anns_field="dense_vector",
        param={"metric_type": "IP"},
        limit=limit
    )

    # 使用加权排序器
    ranker = WeightedRanker(sparse_weight, dense_weight)

    results = client.hybrid_search(
        collection_name=collection_name,
        reqs=[sparse_request, dense_request],
        ranker=ranker,
        limit=limit,
        output_fields=["text"]
    )

    return results[0]
```

**权重调整策略**：
- **关键词搜索场景**：增加 BM25 权重（如 0.7 vs 0.3）
- **语义搜索场景**：增加稠密向量权重（如 0.3 vs 0.7）
- **平衡场景**：使用相等权重（如 0.5 vs 0.5）或 RRFRanker

### 3.3 完整示例

```python
from pymilvus import MilvusClient, CollectionSchema, FieldSchema, DataType
from pymilvus import Function, FunctionType, AnnSearchRequest, RRFRanker
from sentence_transformers import SentenceTransformer

# 1. 连接 Milvus
client = MilvusClient(uri="http://localhost:19530")

# 2. 创建 Collection
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(
            name="text",
            dtype=DataType.VARCHAR,
            max_length=65535,
            enable_analyzer=True,
            enable_match=True
        ),
        FieldSchema(name="dense_vector", dtype=DataType.FLOAT_VECTOR, dim=384),
        FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR)
    ]
)

# 添加 BM25 函数
bm25_function = Function(
    name="text_bm25",
    function_type=FunctionType.BM25,
    input_field_names=["text"],
    output_field_names="sparse_vector"
)
schema.add_function(bm25_function)

client.create_collection(
    collection_name="bm25_hybrid_demo",
    schema=schema
)

# 3. 创建索引
client.create_index(
    collection_name="bm25_hybrid_demo",
    index_params={
        "field_name": "dense_vector",
        "index_type": "HNSW",
        "metric_type": "IP",
        "params": {"M": 16, "efConstruction": 200}
    }
)

client.create_index(
    collection_name="bm25_hybrid_demo",
    index_params={
        "field_name": "sparse_vector",
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "BM25"
    }
)

# 4. 加载 Collection
client.load_collection("bm25_hybrid_demo")

# 5. 插入数据
embedding_model = SentenceTransformer('all-MiniLM-L6-v2')

documents = [
    "Milvus is an open-source vector database built for AI applications",
    "BM25 is a probabilistic ranking function used in information retrieval",
    "Hybrid search combines semantic understanding with keyword matching",
    "Vector databases store and retrieve high-dimensional vectors efficiently",
    "Full-text search uses inverted indexes for fast keyword lookup"
]

dense_vectors = embedding_model.encode(documents).tolist()

data = [
    {"text": text, "dense_vector": vector}
    for text, vector in zip(documents, dense_vectors)
]

client.insert(collection_name="bm25_hybrid_demo", data=data)

# 6. 执行混合检索
query = "vector database for AI"
query_vector = embedding_model.encode([query])[0].tolist()

# BM25 检索请求
sparse_req = AnnSearchRequest(
    data=[query],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=5
)

# 稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="dense_vector",
    param={"metric_type": "IP"},
    limit=5
)

# 混合检索
results = client.hybrid_search(
    collection_name="bm25_hybrid_demo",
    reqs=[sparse_req, dense_req],
    ranker=RRFRanker(),
    limit=3,
    output_fields=["text"]
)

# 7. 输出结果
print(f"Query: {query}\n")
for i, hit in enumerate(results[0], 1):
    print(f"{i}. {hit['entity']['text']}")
    print(f"   Score: {hit['distance']}\n")
```

## 4. BM25 混合检索的优势

### 4.1 优势对比

| 检索方式 | 优势 | 劣势 |
|----------|------|------|
| **纯稠密向量** | 语义理解强，可识别同义词 | 可能遗漏精确关键词 |
| **纯 BM25** | 精确关键词匹配 | 无语义理解，无法识别同义词 |
| **BM25 混合检索** | 兼顾语义理解和精确匹配 | 计算成本略高 |

### 4.2 实际应用场景

**1. 文档问答系统**

```python
# 查询："什么是向量数据库？"
#
# BM25 检索：精确匹配"向量数据库"关键词
# 稠密向量检索：理解"什么是"的语义，找到定义性文本
# 混合检索：返回既包含关键词又符合语义的最佳答案
```

**2. 专业术语搜索**

```python
# 查询："HNSW 算法原理"
#
# BM25 检索：精确匹配"HNSW"专业术语
# 稠密向量检索：理解"算法原理"的语义
# 混合检索：返回关于 HNSW 算法原理的文档
```

**3. 多语言检索**

```python
# 查询："machine learning"（英文）
# 文档包含："机器学习"（中文）
#
# BM25 检索：无法匹配（不同语言）
# 稠密向量检索：可以匹配（跨语言语义理解）
# 混合检索：结合两者优势
```

## 6. 常见问题

### 6.1 BM25 函数 vs 手动生成稀疏向量

**Q：我可以手动生成稀疏向量吗？**

A：可以，但不推荐。Milvus 的 BM25 函数提供了以下优势：
- 自动分词和向量化
- 自动维护 IDF 统计信息
- 与 Milvus 索引深度集成
- 简化代码和维护成本

### 6.2 RRFRanker vs WeightedRanker

**Q：什么时候使用 RRFRanker，什么时候使用 WeightedRanker？**

A：
- **RRFRanker**：不确定权重时使用，自动平衡多个检索结果
- **WeightedRanker**：已知权重时使用，可精确控制各向量字段的重要性

### 6.3 性能影响

**Q：BM25 混合检索会影响性能吗？**

A：会有一定影响，但可以接受：
- 需要执行两次检索（BM25 + 稠密向量）
- 需要融合两个结果集
- 通过索引优化和参数调优可以缓解

```python
def safe_hybrid_search(client, collection_name, query, embedding_model):
    """
    带错误处理的混合检索
    """
    try:
        # 生成查询向量
        query_vector = embedding_model.encode([query])[0].tolist()

        # 定义检索请求
        sparse_req = AnnSearchRequest(
            data=[query],
            anns_field="sparse_vector",
            param={"metric_type": "BM25"},
            limit=10
        )

        dense_req = AnnSearchRequest(
            data=[query_vector],
            anns_field="dense_vector",
            param={"metric_type": "IP"},
            limit=10
        )

        # 执行混合检索
        results = client.hybrid_search(
            collection_name=collection_name,
            reqs=[sparse_req, dense_req],
            ranker=RRFRanker(),
            limit=10,
            output_fields=["text"]
        )

        return results[0]

    except Exception as e:
        print(f"Hybrid search failed: {e}")
        # 降级到单一检索
        return client.search(
            collection_name=collection_name,
            data=[query_vector],
            anns_field="dense_vector",
            limit=10,
            output_fields=["text"]
        )
```

## 8. 总结

BM25 混合检索是 Milvus 多向量检索的核心应用之一，它通过结合 BM25 稀疏向量和稠密向量的优势，实现了兼顾语义理解和精确匹配的高质量检索。

**核心要点**：
1. **BM25 函数**：自动将文本转换为稀疏向量，简化开发流程

3. **Ranker 选择**：RRFRanker（自动平衡）或 WeightedRanker（精确控制）
4. **性能优化**：索引优化、参数调优、分词器选择
