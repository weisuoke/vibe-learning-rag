## 二、断点调试

### 2.1 设置断点

```typescript
// src/index.ts
export async function main() {
  console.log('Starting OpenClaw...');  // 在这里设置断点

  const config = loadConfig();
  debugger;  // 代码断点

  await startGateway(config);
}
```

### 2.2 条件断点

```typescript
// 只在特定条件下暂停
for (let i = 0; i < 100; i++) {
  processItem(i);  // 断点条件: i === 50
}
```

## 三、日志调试

### 3.1 配置日志系统

```typescript
// src/utils/logger.ts
import pino from 'pino';

export const logger = pino({
  level: process.env.LOG_LEVEL || 'info',
  transport: {
    target: 'pino-pretty',
    options: {
      colorize: true,
      translateTime: 'HH:MM:ss',
      ignore: 'pid,hostname'
    }
  }
});
```

### 3.2 使用日志

```typescript
import { logger } from './utils/logger';

logger.debug('Debug message');
logger.info('Info message');
logger.warn('Warning message');
logger.error('Error message');
```

## 四、性能分析

### 4.1 使用 Node.js Profiler

```bash
# 启动性能分析
node --prof dist/index.js

# 生成报告
node --prof-process isolate-*.log > profile.txt
```

### 4.2 使用 Chrome DevTools

```bash
# 启动调试模式
node --inspect dist/index.js

# 或使用 inspect-brk（启动时暂停）
node --inspect-brk dist/index.js
```

打开 Chrome: `chrome://inspect`

## 五、内存调试

### 5.1 检测内存泄漏

```bash
# 启动内存分析
node --inspect --expose-gc dist/index.js
```

### 5.2 生成堆快照

```typescript
import v8 from 'v8';
import fs from 'fs';

// 生成堆快照
const snapshot = v8.writeHeapSnapshot();
console.log(`Heap snapshot written to ${snapshot}`);
```

## 八、总结

### 核心配置

1. **VSCode 调试**: launch.json + tasks.json
2. **断点调试**: 条件断点、日志点
3. **日志调试**: pino + pino-pretty
4. **性能分析**: --prof, --inspect
5. **内存调试**: --expose-gc, 堆快照

### 常用命令

```bash
# 调试模式
node --inspect dist/index.js

# 性能分析
node --prof dist/index.js

# 内存分析
node --inspect --expose-gc dist/index.js
```
