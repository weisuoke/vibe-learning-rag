## 场景描述

在实际应用中，我们经常需要从多个数据源并行获取数据，然后合并结果。例如：

- **智能助手**: 同时获取新闻、天气、股票信息
- **数据聚合**: 从多个 API 并行获取数据
- **监控系统**: 并行检查多个服务状态
- **RAG 系统**: 从多个知识库并行检索

本文档展示如何使用 LangGraph 的并行执行机制优雅地实现这些场景，并处理各种错误情况。

---

## 核心概念

### 1. 并行数据获取模式

从 START 节点扇出到多个数据获取节点：

```python
# 从 START 扇出到三个数据源
builder.add_edge(START, "fetch_news")
builder.add_edge(START, "fetch_weather")
builder.add_edge(START, "fetch_stocks")
```

### 2. 数据合并节点

等待所有数据源完成后，合并结果：

```python
# 三个数据源扇入到合并节点
builder.add_edge("fetch_news", "combine_data")
builder.add_edge("fetch_weather", "combine_data")
builder.add_edge("fetch_stocks", "combine_data")
```

### 3. 错误处理策略

- **部分失败容忍**: 某个数据源失败不影响其他数据源
- **超时控制**: 设置合理的超时时间
- **降级策略**: 失败时返回默认值或缓存数据

# ===== 4. 定义数据合并节点 =====

def combine_data(state: DataState):
    """
    合并节点: 合并所有数据源的结果

    关键点:
    - 等待所有并行节点完成后才执行
    - 处理部分数据源失败的情况
    - 生成友好的用户报告
    """
    print("\n[合并节点] 正在合并所有数据...")

    # 统计成功和失败的数据源
    success_count = 0
    total_count = 3

    if state.get("news_data"):
        success_count += 1
    if state.get("weather_data"):
        success_count += 1
    if state.get("stock_data"):
        success_count += 1

    # 构建报告
    report = f"""
{'=' * 60}
数据聚合报告
查询: {state['query']}
时间: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}
成功率: {success_count}/{total_count}
{'=' * 60}

"""

    # 添加成功获取的数据
    if state.get("news_data"):
        report += state["news_data"] + "\n"
    else:
        report += "【新闻】数据获取失败\n\n"

    if state.get("weather_data"):
        report += state["weather_data"] + "\n"
    else:
        report += "【天气】数据获取失败\n\n"

    if state.get("stock_data"):
        report += state["stock_data"] + "\n"
    else:
        report += "【股票】数据获取失败\n\n"

    # 添加错误信息（如果有）
    if state.get("errors"):
        report += "─" * 60 + "\n"
        report += "错误信息:\n"
        for error in state["errors"]:
            report += f"  - {error}\n"

    report += "=" * 60

    print("[合并节点] ✓ 数据合并完成")
    return {"combined_report": report}

def fetch_with_retry(api_func, query: str, max_retries: int = 3):
    """
    带重试的数据获取函数

    参数:
    - api_func: API 调用函数
    - query: 查询参数
    - max_retries: 最大重试次数
    """
    for attempt in range(max_retries):
        try:
            result = api_func(query)
            return result
        except Exception as e:
            if attempt == max_retries - 1:
                raise e
            print(f"    重试 {attempt + 1}/{max_retries}...")
            time.sleep(1)


def fetch_news_with_retry(state: DataState):
    """带重试的新闻获取"""
    print(f"[节点1] 开始获取新闻数据（带重试）...")

    try:
        result = fetch_with_retry(fetch_news_api, state["query"])

        if result["status"] == "success":
            articles = result["data"]["articles"]
            news_text = f"【新闻】{result['data']['title']}\n"
            for i, article in enumerate(articles, 1):
                news_text += f"{i}. {article['headline']} ({article['source']})\n"

            print(f"[节点1] ✓ 新闻数据获取成功")
            return {"news_data": news_text}
        else:
            raise Exception("API 返回失败状态")

    except Exception as e:
        error_msg = f"新闻获取失败（已重试3次）: {str(e)}"
        print(f"[节点1] ✗ {error_msg}")
        return {
            "news_data": None,
            "errors": [error_msg]
        }


print("✓ 重试机制示例构建完成")

import signal
from contextlib import contextmanager


class TimeoutException(Exception):
    pass


@contextmanager
def timeout(seconds):
    """超时上下文管理器"""
    def timeout_handler(signum, frame):
        raise TimeoutException("操作超时")

    # 设置信号处理器
    old_handler = signal.signal(signal.SIGALRM, timeout_handler)
    signal.alarm(seconds)

    try:
        yield
    finally:
        signal.alarm(0)
        signal.signal(signal.SIGALRM, old_handler)


def fetch_news_with_timeout(state: DataState):
    """带超时控制的新闻获取"""
    print(f"[节点1] 开始获取新闻数据（超时控制：5秒）...")

    try:
        with timeout(5):
            result = fetch_news_api(state["query"])

            if result["status"] == "success":
                articles = result["data"]["articles"]
                news_text = f"【新闻】{result['data']['title']}\n"
                for i, article in enumerate(articles, 1):
                    news_text += f"{i}. {article['headline']} ({article['source']})\n"

                print(f"[节点1] ✓ 新闻数据获取成功")
                return {"news_data": news_text}
            else:
                raise Exception("API 返回失败状态")

    except TimeoutException as e:
        error_msg = f"新闻获取超时: {str(e)}"
        print(f"[节点1] ✗ {error_msg}")
        return {
            "news_data": None,
            "errors": [error_msg]
        }
    except Exception as e:
        error_msg = f"新闻获取失败: {str(e)}"
        print(f"[节点1] ✗ {error_msg}")
        return {
            "news_data": None,
            "errors": [error_msg]
        }

## 关键技术点

### 1. 并行执行机制

LangGraph 自动管理并行执行：

- 所有从 START 出发的节点在同一个超步中并行执行
- 合并节点等待所有并行节点完成后才执行
- 不需要手动管理线程或进程

### 2. 错误处理策略

**部分失败容忍**：

```python
# 每个节点独立处理错误
try:
    result = fetch_api(query)
    return {"data": result}
except Exception as e:
    return {
        "data": None,
        "errors": [str(e)]
    }
```

**合并节点处理缺失数据**：

```python
if state.get("news_data"):
    report += state["news_data"]
else:
    report += "【新闻】数据获取失败\n"
```

### 3. 状态更新策略

每个节点只更新自己负责的字段：

```python
# 新闻节点只更新 news_data
return {"news_data": news_text}

# 天气节点只更新 weather_data
return {"weather_data": weather_text}

# 合并节点可以访问所有字段
state["news_data"]  # ✓ 可用
state["weather_data"]  # ✓ 可用
state["stock_data"]  # ✓ 可用
```

## 性能优化建议

### 1. 控制并行度

根据系统资源调整并行任务数量：

```python
# 适中的并行度（3-5个任务）
builder.add_edge(START, "fetch_news")
builder.add_edge(START, "fetch_weather")
builder.add_edge(START, "fetch_stock")
```

### 2. 设置合理的超时

避免某个慢速数据源拖慢整体：

```python
def fetch_with_timeout(api_func, timeout_seconds=5):
    with timeout(timeout_seconds):
        return api_func()
```

### 3. 使用缓存

对于不常变化的数据，使用缓存减少 API 调用：

```python
from functools import lru_cache

@lru_cache(maxsize=100)
def cached_fetch(query: str):
    return fetch_api(query)
```
