## 步骤 1：生产构建配置

### 1.1 环境变量配置

**文件**：`.env.production`

```bash
# 创建生产环境配置
cat > .env.production << 'EOF'
# 生产环境
NODE_ENV=production

# API Keys（使用环境变量注入）
ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
OPENAI_API_KEY=${OPENAI_API_KEY}

# Gateway 配置
OPENCLAW_PORT=18789
OPENCLAW_HOST=0.0.0.0

# 日志配置
LOG_LEVEL=info
LOG_FORMAT=json

# 性能配置
NODE_OPTIONS=--max-old-space-size=4096
EOF
```

### 1.2 TypeScript 生产配置

**文件**：`tsconfig.production.json`

```json
{
  "extends": "./tsconfig.json",
  "compilerOptions": {
    "sourceMap": false,
    "inlineSourceMap": false,
    "declaration": true,
    "declarationMap": false,
    "removeComments": true,
    "importHelpers": true
  },
  "exclude": [
    "**/*.test.ts",
    "**/*.spec.ts",
    "**/test/**",
    "**/tests/**"
  ]
}
```

**关键配置**：
- `sourceMap: false`：不生成 source map（减小体积）
- `removeComments: true`：移除注释
- `importHelpers: true`：使用 tslib 减小体积

    bundle: true,

  clean: true,

**关键配置**：

- `splitting: true`：代码分割

## 步骤 4：Docker 构建

### 4.1 多阶段 Dockerfile

**文件**：`Dockerfile`

```dockerfile
# 阶段 1: 构建
FROM node:22-alpine AS builder

# 设置工作目录
WORKDIR /app

# 安装 pnpm
RUN npm install -g pnpm@10.23.0

# 复制依赖文件
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY ui/package.json ./ui/
COPY packages/*/package.json ./packages/
COPY extensions/*/package.json ./extensions/

# 安装依赖
RUN pnpm install --frozen-lockfile

# 复制源码
COPY . .

# 构建
RUN NODE_ENV=production pnpm build

# 阶段 2: 生产
FROM node:22-alpine

# 设置工作目录
WORKDIR /app

# 安装 pnpm
RUN npm install -g pnpm@10.23.0

# 复制依赖文件
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./

# 安装生产依赖
RUN pnpm install --frozen-lockfile --prod

# 复制构建产物
COPY --from=builder /app/dist ./dist

# 创建非 root 用户
RUN addgroup -g 1001 -S openclaw && \
    adduser -S openclaw -u 1001

# 切换用户
USER openclaw

# 暴露端口
EXPOSE 18789

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:18789/health', (r) => process.exit(r.statusCode === 200 ? 0 : 1))"

# 启动命令
CMD ["node", "dist/daemon-cli.js", "gateway"]
```

**关键特性**：
- **多阶段构建**：分离构建和运行环境
- **Alpine 镜像**：减小镜像体积
- **非 root 用户**：提升安全性
- **健康检查**：自动检测服务状态

## 步骤 5：Docker Compose 部署

### 5.1 docker-compose.yml

**文件**：`docker-compose.yml`

```yaml
version: '3.8'

services:
  openclaw:
    build:
      context: .
      dockerfile: Dockerfile
    image: openclaw:latest
    container_name: openclaw-gateway
    ports:
      - "18789:18789"
    environment:
      - NODE_ENV=production
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - LOG_LEVEL=info
    volumes:
      - openclaw-data:/app/data
      - openclaw-logs:/app/logs
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "node", "-e", "require('http').get('http://localhost:18789/health', (r) => process.exit(r.statusCode === 200 ? 0 : 1))"]
      interval: 30s
      timeout: 3s
      retries: 3
      start_period: 5s
    networks:
      - openclaw-network

  postgres:
    image: postgres:16-alpine
    container_name: openclaw-postgres
    environment:
      - POSTGRES_USER=openclaw
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-openclaw}
      - POSTGRES_DB=openclaw
    volumes:
      - postgres-data:/var/lib/postgresql/data
    restart: unless-stopped
    networks:
      - openclaw-network

  redis:
    image: redis:7-alpine
    container_name: openclaw-redis
    volumes:
      - redis-data:/data
    restart: unless-stopped
    networks:
      - openclaw-network

volumes:
  openclaw-data:
  openclaw-logs:
  postgres-data:
  redis-data:

networks:
  openclaw-network:
    driver: bridge
```

## 核心洞察

### 1. 生产构建不是简单的 build

**表面**：运行 `pnpm build`

**实际**：
- 优化配置（minify, treeshake）
- 移除开发依赖
- 压缩构建产物
- 验证完整性

---

### 2. Docker 多阶段构建很重要

**表面**：一个 Dockerfile

**实际**：
- 构建阶段：完整工具链
- 运行阶段：最小依赖
- 减小镜像体积
- 提升安全性

---

### 3. 监控和日志是必需的

**表面**：服务能跑就行

**实际**：
- 结构化日志便于分析
- Prometheus 指标监控性能
- 健康检查自动恢复
- 负载测试验证容量

---

## 一句话总结

**OpenClaw 生产构建的关键是：优化 tsdown 配置，使用 Docker 多阶段构建，配置监控和日志，验证部署质量。**
