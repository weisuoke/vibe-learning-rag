# 一句话总结

**Pi-mono 的 Prompt Templates 是用 Markdown + YAML frontmatter + Bash 风格变量替换实现的极简模板系统，通过文件系统多源加载和 `/template-name args` 调用方式，让提示词像代码一样可复用、可版本控制、可团队协作。**

---

## 核心本质

Prompt Templates 的本质是：**将提示词工程化**。

就像代码需要函数、类、模块来组织一样，提示词也需要模板、变量、加载机制来管理。Pi-mono 选择了最简单但最有效的方式：

- **格式**：Markdown（人类可读）+ YAML frontmatter（机器可解析）
- **变量**：Bash 风格（开发者熟悉）
- **加载**：文件系统（无需数据库）
- **调用**：命令行风格（符合 CLI 工具习惯）

---

## 为什么这个设计是极简的？

### 1. 零学习成本的格式

```markdown
---
description: Review PRs from URLs
---
You are given one or more GitHub PR URLs: $@

Please analyze each PR and provide:
1. Summary of changes
2. Potential issues
3. Suggestions for improvement
```

- Markdown：所有开发者都会
- YAML frontmatter：Git commit、Jekyll、Hugo 都在用
- Bash 变量：Shell 脚本的标准语法

### 2. 零配置的加载机制

```
~/.pi/agent/prompts/     # 全局模板，所有项目共享
./.pi/prompts/           # 项目模板，项目特定
```

- 不需要数据库
- 不需要配置文件
- 不需要注册流程
- 放文件就能用

### 3. 零依赖的调用方式

```bash
/pr https://github.com/user/repo/pull/123
```

- 不需要编程
- 不需要 API 调用
- 不需要额外工具
- 命令行直接用

## 核心洞察

**Prompt Templates 不是技术创新，而是工程实践的回归。**

它没有发明新的模板语法，而是复用了 Markdown、YAML、Bash 这些已经被验证的标准。它没有创造新的加载机制，而是直接使用文件系统。它没有设计新的调用方式，而是采用命令行风格。

这就是极简主义的力量：**不是做得更多，而是做得更少，但做得更好。**
