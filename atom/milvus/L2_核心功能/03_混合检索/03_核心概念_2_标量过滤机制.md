# 核心概念_2_标量过滤机制

## 概述

标量过滤是混合检索中的重要组成部分，允许在向量检索的基础上添加元数据条件，实现更精确的检索结果。本文档深入讲解标量过滤的原理、语法和实现。

---

## 核心概念1：过滤表达式语法

**过滤表达式是使用布尔逻辑和比较运算符构建的条件语句，用于在向量检索时筛选符合特定条件的文档。**

### 什么是过滤表达式？

过滤表达式（Filter Expression）是一种类似SQL WHERE子句的条件语句，用于在Milvus中筛选数据。

**基本语法：**
```python
# 简单条件
filter = "price > 100"

# 复合条件
filter = "price > 100 AND category == 'electronics'"

# 复杂条件
filter = "(price > 100 AND price < 500) OR (rating >= 4.5 AND stock > 0)"
```

### 比较运算符

Milvus支持以下比较运算符：

| 运算符 | 说明 | 示例 |
|-------|------|------|
| `==` | 等于 | `category == 'electronics'` |
| `!=` | 不等于 | `status != 'deleted'` |
| `>` | 大于 | `price > 100` |
| `<` | 小于 | `price < 1000` |
| `>=` | 大于等于 | `rating >= 4.0` |
| `<=` | 小于等于 | `stock <= 100` |
| `IN` | 在列表中 | `category IN ['electronics', 'computers']` |
| `LIKE` | 模糊匹配 | `title LIKE '%Python%'` |

**代码示例：**
```python
from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 等于
results = client.query(
    collection_name="products",
    filter="category == 'electronics'",
    output_fields=["name", "price"]
)

# 大于
results = client.query(
    collection_name="products",
    filter="price > 100",
    output_fields=["name", "price"]
)

# 范围查询
results = client.query(
    collection_name="products",
    filter="price >= 50 AND price <= 200",
    output_fields=["name", "price"]
)

# IN运算符
results = client.query(
    collection_name="products",
    filter='category IN ["electronics", "computers", "phones"]',
    output_fields=["name", "category"]
)

# LIKE运算符
results = client.query(
    collection_name="products",
    filter="name LIKE '%laptop%'",
    output_fields=["name", "price"]
)
```

### 逻辑运算符

Milvus支持以下逻辑运算符：

| 运算符 | 说明 | 示例 |
|-------|------|------|
| `AND` | 逻辑与 | `price > 100 AND stock > 0` |
| `OR` | 逻辑或 | `category == 'electronics' OR category == 'computers'` |
| `NOT` | 逻辑非 | `NOT (category == 'electronics')` |

**代码示例：**
```python
# AND运算符
results = client.query(
    collection_name="products",
    filter="category == 'electronics' AND price > 100 AND stock > 0",
    output_fields=["name", "price", "stock"]
)

# OR运算符
results = client.query(
    collection_name="products",
    filter="category == 'electronics' OR category == 'computers'",
    output_fields=["name", "category"]
)

# NOT运算符
results = client.query(
    collection_name="products",
    filter="NOT (category == 'electronics')",
    output_fields=["name", "category"]
)

# 复杂组合
results = client.query(
    collection_name="products",
    filter="(category == 'electronics' AND price > 100) OR (category == 'computers' AND price > 500)",
    output_fields=["name", "category", "price"]
)
```

### 括号优先级

使用括号控制运算优先级：

```python
# 错误：可能不是预期的结果
filter = "category == 'electronics' AND price > 100 OR stock > 0"
# 解释：(category == 'electronics' AND price > 100) OR stock > 0

# 正确：使用括号明确优先级
filter = "category == 'electronics' AND (price > 100 OR stock > 0)"
# 解释：category == 'electronics' AND (price > 100 OR stock > 0)
```

## 核心概念2：JSON字段过滤

**JSON字段过滤允许对JSON类型字段的嵌套属性进行条件筛选，支持复杂的数据结构查询。**

### JSON字段访问

Milvus支持通过键路径访问JSON字段：

```python
# Schema定义
schema.add_field(field_name="metadata", datatype=DataType.JSON)

# 插入数据
data = [{
    "id": 1,
    "metadata": {
        "color": "red",
        "size": "large",
        "weight": 150,
        "tags": ["new", "sale"]
    }
}]

# 访问JSON字段
filter = 'metadata["color"] == "red"'
filter = 'metadata["weight"] > 100'
filter = 'metadata["size"] == "large" AND metadata["color"] == "red"'
```

### JSON运算符

Milvus提供专门的JSON运算符：

| 运算符 | 说明 | 示例 |
|-------|------|------|
| `JSON_CONTAINS` | 检查是否包含元素 | `json_contains(tags, "sale")` |
| `JSON_CONTAINS_ALL` | 检查是否包含所有元素 | `json_contains_all(tags, ["electronics", "sale"])` |
| `JSON_CONTAINS_ANY` | 检查是否包含任一元素 | `json_contains_any(tags, ["sale", "clearance"])` |

**代码示例：**
```python
# JSON_CONTAINS
results = client.query(
    collection_name="products",
    filter='json_contains(metadata["tags"], "sale")',
    output_fields=["name", "metadata"]
)

# JSON_CONTAINS_ALL
results = client.query(
    collection_name="products",
    filter='json_contains_all(metadata["tags"], ["electronics", "sale", "new"])',
    output_fields=["name", "metadata"]
)

# JSON_CONTAINS_ANY
results = client.query(
    collection_name="products",
    filter='json_contains_any(metadata["tags"], ["sale", "clearance", "discount"])',
    output_fields=["name", "metadata"]
)

# 组合使用
results = client.query(
    collection_name="products",
    filter='metadata["color"] == "red" AND json_contains(metadata["tags"], "sale")',
    output_fields=["name", "metadata"]
)
```

### 嵌套JSON访问

```python
# 嵌套JSON结构
data = [{
    "id": 1,
    "metadata": {
        "product": {
            "name": "Laptop",
            "specs": {
                "cpu": "Intel i7",
                "ram": 16
            }
        }
    }
}]

# 访问嵌套字段
filter = 'metadata["product"]["specs"]["ram"] >= 16'
filter = 'metadata["product"]["specs"]["cpu"] == "Intel i7"'
```

## 核心概念3：数组字段过滤

**数组字段过滤允许对数组类型字段进行条件筛选，支持元素包含、长度检查等操作。**

### 数组字段定义

```python
from pymilvus import DataType

# Schema定义
schema.add_field(
    field_name="tags",
    datatype=DataType.ARRAY,
    element_type=DataType.VARCHAR,
    max_capacity=20,
    max_length=50
)

schema.add_field(
    field_name="features",
    datatype=DataType.ARRAY,
    element_type=DataType.VARCHAR,
    max_capacity=10,
    max_length=100
)
```

### 数组运算符

Milvus提供专门的数组运算符：

| 运算符 | 说明 | 示例 |
|-------|------|------|
| `ARRAY_CONTAINS` | 检查是否包含元素 | `ARRAY_CONTAINS(tags, "sale")` |
| `ARRAY_CONTAINS_ALL` | 检查是否包含所有元素 | `ARRAY_CONTAINS_ALL(tags, ["sale", "new"])` |
| `ARRAY_CONTAINS_ANY` | 检查是否包含任一元素 | `ARRAY_CONTAINS_ANY(tags, ["sale", "clearance"])` |
| `ARRAY_LENGTH` | 检查数组长度 | `ARRAY_LENGTH(tags) >= 3` |

**代码示例：**
```python
# ARRAY_CONTAINS
results = client.query(
    collection_name="products",
    filter="ARRAY_CONTAINS(tags, 'sale')",
    output_fields=["name", "tags"]
)

# ARRAY_CONTAINS_ALL
results = client.query(
    collection_name="products",
    filter="ARRAY_CONTAINS_ALL(tags, ['sale', 'new'])",
    output_fields=["name", "tags"]
)

# ARRAY_CONTAINS_ANY
results = client.query(
    collection_name="products",
    filter="ARRAY_CONTAINS_ANY(tags, ['sale', 'clearance', 'discount'])",
    output_fields=["name", "tags"]
)

# ARRAY_LENGTH
results = client.query(
    collection_name="products",
    filter="ARRAY_LENGTH(tags) >= 3",
    output_fields=["name", "tags"]
)

# 组合使用
results = client.query(
    collection_name="products",
    filter="ARRAY_CONTAINS(tags, 'electronics') AND ARRAY_LENGTH(tags) >= 2",
    output_fields=["name", "tags"]
)
```

### 数组索引访问

```python
# 访问数组特定位置的元素
filter = "tags[0] == 'electronics'"
filter = "tags[1] == 'sale'"

# 组合使用
filter = "tags[0] == 'electronics' AND tags[1] == 'sale'"
```

## 标量过滤在混合检索中的应用

### Pre-filter（预过滤）

**Pre-filter在向量检索之前应用过滤条件，减少检索空间。**

**适用场景：**
- 过滤条件选择性高（<20%）
- 需要提升检索速度
- 元数据条件明确

**代码示例：**
```python
from pymilvus import AnnSearchRequest, RRFRanker

# 稠密向量检索 + Pre-filter
dense_req = AnnSearchRequest(
    data=[dense_query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20,
    expr="category == 'electronics' AND price < 1000"  # Pre-filter
)

# 稀疏向量检索 + Pre-filter
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20,
    expr="category == 'electronics' AND price < 1000"  # Pre-filter
)

# 混合检索
results = client.hybrid_search(
    collection_name="products",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=10,
    output_fields=["name", "price", "category"]
)
```

**性能特点：**
- ✅ 减少向量检索的候选集
- ✅ 提升检索速度（高选择性时）
- ❌ 可能降低召回率（过滤太严格）

---

### Post-filter（后过滤）

**Post-filter在向量检索之后应用过滤条件，保证召回率。**

**适用场景：**
- 过滤条件选择性低（>50%）
- 需要保证召回率
- 元数据条件不明确

**代码示例：**
```python
# 向量检索（不带过滤）
dense_req = AnnSearchRequest(
    data=[dense_query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=100  # 召回更多候选
)

sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=100
)

# 混合检索
results = client.hybrid_search(
    collection_name="products",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=50,  # 先召回50个
    output_fields=["name", "price", "category", "rating"]
)

# Post-filter（应用层过滤）
filtered_results = []
for hits in results:
    for hit in hits:
        if hit.entity.get("price") < 1000 and hit.entity.get("rating") >= 4.0:
            filtered_results.append(hit)
            if len(filtered_results) >= 10:
                break
    if len(filtered_results) >= 10:
        break
```

**性能特点：**
- ✅ 保证召回率
- ✅ 适合低选择性过滤
- ❌ 需要召回更多候选（延迟增加）

## 过滤表达式模板化

### 为什么需要模板化？

当过滤条件包含CJK字符（中文、日文、韩文）时，解析开销较大。模板化可以优化性能。

**传统方式（慢）：**
```python
filter = "age > 25 AND city IN ['北京', '上海']"
```

**模板化方式（快）：**
```python
filter = "age > {age} AND city IN {city}"
filter_params = {"age": 25, "city": ["北京", "上海"]}
```

### 模板化示例

```python
# 单参数模板
results = client.query(
    collection_name="users",
    filter="age > {age}",
    filter_params={"age": 25},
    output_fields=["name", "age"]
)

# 多参数模板
results = client.query(
    collection_name="users",
    filter="age > {min_age} AND age < {max_age} AND city IN {cities}",
    filter_params={
        "min_age": 25,
        "max_age": 40,
        "cities": ["北京", "上海", "深圳"]
    },
    output_fields=["name", "age", "city"]
)

# 混合检索中使用模板
dense_req = AnnSearchRequest(
    data=[dense_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20,
    expr="price > {min_price} AND price < {max_price} AND category IN {categories}",
    expr_params={
        "min_price": 100,
        "max_price": 1000,
        "categories": ["电子产品", "计算机"]
    }
)
```

## 标量索引优化

### 为什么需要标量索引？

标量索引可以加速过滤操作，特别是在大规模数据集上。

### 标量索引类型

| 索引类型 | 适用字段类型 | 说明 |
|---------|-------------|------|
| `INVERTED` | VARCHAR | 倒排索引，适合字符串字段 |
| `STL_SORT` | INT64, FLOAT | 排序索引，适合数值字段 |
| `TRIE` | VARCHAR | 前缀树索引，适合前缀匹配 |

### 创建标量索引

```python
from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 为字符串字段创建倒排索引
index_params = client.prepare_index_params()

index_params.add_index(
    field_name="category",
    index_type="INVERTED"
)

index_params.add_index(
    field_name="brand",
    index_type="INVERTED"
)

# 为数值字段创建排序索引
index_params.add_index(
    field_name="price",
    index_type="STL_SORT"
)

index_params.add_index(
    field_name="rating",
    index_type="STL_SORT"
)

# 创建索引
client.create_index(
    collection_name="products",
    index_params=index_params
)
```

### 索引性能对比

| 场景 | 无索引 | 有索引 | 提升 |
|-----|-------|-------|------|
| 字符串等值查询 | 100ms | 5ms | 20x |
| 数值范围查询 | 80ms | 8ms | 10x |
| 复合条件查询 | 150ms | 12ms | 12.5x |

## 过滤性能优化建议

### 1. 使用标量索引

```python
# 为频繁过滤的字段创建索引
index_params.add_index(field_name="category", index_type="INVERTED")
index_params.add_index(field_name="price", index_type="STL_SORT")
```

### 2. 选择性高的条件放前面

```python
# 好：选择性高的条件在前
filter = "tenant_id == 'tenant_a' AND category == 'electronics'"

# 差：选择性低的条件在前
filter = "category == 'electronics' AND tenant_id == 'tenant_a'"
```

### 3. 使用IN代替多个OR

```python
# 好：使用IN
filter = "category IN ['electronics', 'computers', 'phones']"

# 差：使用多个OR
filter = "category == 'electronics' OR category == 'computers' OR category == 'phones'"
```

### 4. 避免复杂计算

```python
# 好：预计算字段
filter = "total_price > 1000"

# 差：运行时计算
filter = "price * quantity > 1000"
```

### 5. 使用模板化（CJK字符）

```python
# 好：使用模板
filter = "city IN {cities}"
filter_params = {"cities": ["北京", "上海"]}

# 差：直接嵌入
filter = "city IN ['北京', '上海']"
```

## 总结

### 核心要点

1. **过滤表达式**：使用布尔逻辑和比较运算符构建条件
2. **JSON过滤**：支持嵌套JSON字段的条件筛选
3. **数组过滤**：支持数组元素包含和长度检查
4. **Pre-filter vs Post-filter**：根据选择性选择策略
5. **标量索引**：加速过滤操作，提升性能

### 最佳实践

1. **创建标量索引**：为频繁过滤的字段创建索引
2. **选择合适的过滤策略**：Pre-filter（高选择性）vs Post-filter（低选择性）
3. **使用模板化**：优化CJK字符的过滤性能
4. **优化过滤条件**：选择性高的条件放前面，使用IN代替OR
5. **避免复杂计算**：预计算字段，避免运行时计算
