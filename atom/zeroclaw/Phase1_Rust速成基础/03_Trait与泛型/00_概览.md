### 4.2 为什么用 Trait 而不是继承？

| 维度 | 继承（Java/C++） | Trait（Rust） |
|------|-----------------|--------------|
| 关系 | "是一个"（is-a） | "能做什么"（can-do） |
| 耦合 | 父类改动影响所有子类 | 实现者之间完全独立 |
| 扩展 | 需要修改类层次 | 新增 impl 即可 |
| 组合 | 多重继承问题（菱形继承） | 自由组合多个 Trait |

**ZeroClaw 的选择**：Anthropic Provider 和 Ollama Provider 之间没有继承关系，它们只是都实现了 `Provider` Trait。这意味着新增一个 Provider 只需要写一个 `impl Provider for MyProvider`，不需要改动任何现有代码。

### 4.3 Trait + 泛型 = 零成本多态

```rust
// ZeroClaw 的 Tool Trait -- 最简洁的核心 Trait
#[async_trait]
pub trait Tool: Send + Sync {
    fn name(&self) -> &str;
    fn description(&self) -> &str;
    fn parameters_schema(&self) -> serde_json::Value;
    async fn execute(&self, args: serde_json::Value) -> anyhow::Result<ToolResult>;
    fn spec(&self) -> ToolSpec { /* 默认实现 */ }
}
```
[来源: sourcecode/zeroclaw/src/tools/traits.rs]

任何 struct 只要实现这 4 个必需方法（第 5 个有默认实现），就能作为 ZeroClaw 的工具使用。这就是 Trait 的力量。

## 5. 前端开发者快速理解

### 5.1 Trait 约等于 TypeScript interface（但更强大）

```typescript
// TypeScript interface
interface Tool {
  name(): string;
  description(): string;
  execute(args: any): Promise<ToolResult>;
}

class ShellTool implements Tool {
  name() { return "shell"; }
  description() { return "Execute shell commands"; }
  async execute(args: any) { /* ... */ }
}
```

```rust
// Rust trait -- 几乎一样的结构
trait Tool {
    fn name(&self) -> &str;
    fn description(&self) -> &str;
    async fn execute(&self, args: Value) -> Result<ToolResult>;
}

impl Tool for ShellTool {
    fn name(&self) -> &str { "shell" }
    fn description(&self) -> &str { "Execute shell commands" }
    async fn execute(&self, args: Value) -> Result<ToolResult> { /* ... */ }
}
```

**关键差异**：
- Rust Trait 可以有**默认实现**（TypeScript interface 不行，abstract class 才行）
- Rust Trait 可以为**已有类型**添加行为（TypeScript 做不到给 `string` 加方法）
- Rust Trait 是**编译期检查**，TypeScript interface 是类型擦除

### 5.2 泛型 约等于 TypeScript generics（但编译时单态化）

```typescript
// TypeScript 泛型 -- 运行时类型擦除
function first<T>(arr: T[]): T | undefined {
  return arr[0];
}
// 编译后：function first(arr) { return arr[0]; }  -- T 消失了
```

```rust
// Rust 泛型 -- 编译时单态化
fn first<T>(arr: &[T]) -> Option<&T> {
    arr.first()
}
// 编译后：为每个具体类型生成专用版本
// first_i32, first_string, first_tool ... 各自优化
```

**核心区别**：TypeScript 泛型在运行时消失（类型擦除），Rust 泛型在编译时展开为具体类型的专用代码（单态化），所以**零运行时开销**。

### 5.3 对照速查表

| TypeScript 概念 | Rust 对应 | 差异 |
|----------------|-----------|------|
| `interface Tool { }` | `trait Tool { }` | Trait 可有默认实现 |
| `implements Tool` | `impl Tool for X` | 可为外部类型实现 |
| `<T>` 泛型参数 | `<T>` 泛型参数 | Rust 编译时单态化 |
| `<T extends Foo>` | `<T: Foo>` 或 `where T: Foo` | 语法不同，语义相似 |
| `type Item` (type alias) | `type Item` (关联类型) | Rust 的由实现者决定 |
| `abstract class` | `trait` + 默认方法 | Rust 无继承 |
| 运行时多态 | `dyn Trait`（动态分发） | 需显式 Box/Arc 包装 |
| 编译时多态 | 泛型（静态分发） | 零成本，默认选择 |
