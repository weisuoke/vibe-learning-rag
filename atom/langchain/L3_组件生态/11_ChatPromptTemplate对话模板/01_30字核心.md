## 30字核心

**多角色消息模板，支持历史占位符，构建对话式AI的标准化提示词接口**

（恰好30个中文字符）

---

## 核心解读

### 为什么这30个字是核心？

这30个字包含了 ChatPromptTemplate 的三个最本质特征：

#### 1. 多角色消息模板（核心特征）

**来源**: [源码分析](reference/source_chatprompt_01.md) - `chat.py:789-1484`

**本质**:
- 不是简单的字符串模板，而是**消息列表模板**
- 每条消息都有明确的**角色**（System/Human/AI/Chat）
- 输出是**结构化的消息对象**，而非纯文本

**与 PromptTemplate 的本质区别**:
```python
# PromptTemplate: 输出字符串
prompt = PromptTemplate.from_template("You are {role}. Answer: {question}")
result = prompt.format(role="assistant", question="What is AI?")
# 输出: "You are assistant. Answer: What is AI?" (字符串)

# ChatPromptTemplate: 输出消息列表
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are {role}."),
    ("human", "{question}")
])
result = prompt.format_messages(role="assistant", question="What is AI?")
# 输出: [SystemMessage(content="You are assistant."),
#        HumanMessage(content="What is AI?")] (消息对象列表)
```

**为什么这是核心**:
- 对话式 AI 的本质是**多轮消息交互**，而非单一文本生成
- ChatModel 需要的输入是**消息列表**，而非字符串
- 角色区分让 LLM 能够理解**对话上下文**和**说话者身份**

**数据支持**:
- 源码: `chat.py:663-688` 定义了 4 种消息模板类型
- Context7: [消息类型与角色](reference/context7_langchain_02.md) - 官方文档明确说明角色的重要性
- 社区: [GeeksforGeeks 教程](reference/search_messagesplaceholder_01.md) - 强调多角色是对话系统的基础

---

#### 2. 历史占位符（关键创新）

**来源**: [源码分析](reference/source_chatprompt_01.md) - `chat.py:52-217` (MessagesPlaceholder)

**本质**:
- `MessagesPlaceholder` 是 ChatPromptTemplate 的**杀手级特性**
- 允许在模板中**动态插入消息列表**（如对话历史）
- 支持 `optional=True` 和 `n_messages` 限制

**核心价值**:
```python
# 没有 MessagesPlaceholder: 无法处理对话历史
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("human", "{input}")
])
# 问题: 无法注入之前的对话历史

# 有 MessagesPlaceholder: 轻松管理对话历史
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    MessagesPlaceholder("history"),  # 动态插入历史
    ("human", "{input}")
])

prompt.format_messages(
    history=[
        ("human", "What's 5 + 2?"),
        ("ai", "5 + 2 is 7")
    ],
    input="Now multiply that by 4"
)
# 输出包含完整对话上下文
```

**为什么这是核心**:
- 对话式 AI 的核心需求是**保持上下文**
- 没有历史占位符，每次都要手动拼接历史消息
- MessagesPlaceholder 让对话历史管理**标准化**和**自动化**

**数据支持**:
- 源码: `chat.py:52-217` - MessagesPlaceholder 实现细节
- Context7: [对话历史管理](reference/context7_langchain_01.md) - 官方示例
- 社区: [AWS 聊天机器人案例](reference/search_messagesplaceholder_01.md) - 生产环境应用

---

#### 3. 标准化提示词接口（架构价值）

**来源**: [Context7 文档](reference/context7_langchain_02.md) - LCEL 集成

**本质**:
- ChatPromptTemplate 是 LangChain 生态的**标准接口**
- 与 ChatModel、OutputParser、Memory 等组件**无缝集成**
- 支持 LCEL 管道操作符 `|`

**核心价值**:
```python
# 标准化接口: 可组合、可复用、可测试
chain = prompt | llm | output_parser

# 与 Memory 集成
chain_with_memory = (
    RunnablePassthrough.assign(
        history=RunnableLambda(memory.load_memory_variables)
        | itemgetter("history")
    )
    | prompt
    | llm
)

# 与 Agent 集成
agent_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant with tools."),
    MessagesPlaceholder("history"),
    ("human", "{input}"),
    MessagesPlaceholder("agent_scratchpad")  # Agent 工具调用历史
])
```

**为什么这是核心**:
- 标准化接口让**复杂系统变得可组合**
- 降低了对话式 AI 的开发门槛
- 提升了代码的**可维护性**和**可测试性**

**数据支持**:
- Context7: [LCEL 集成](reference/context7_langchain_02.md) - 官方集成文档
- 社区: [Mirascope 指南](reference/search_messagesplaceholder_01.md) - 强调标准化的重要性

## 为什么不是其他特性？

### 为什么不是"多模态支持"？

**理由**:
- 多模态（文本+图片）是**扩展特性**，而非核心
- 大多数对话场景只需要文本
- 多模态支持是在核心架构上的**增量功能**

**数据支持**:
- 源码: `chat.py:396-662` - 多模态是 `_StringImageMessagePromptTemplate` 的扩展
- 社区: [Reddit 讨论](reference/search_multimodal_01.md) - 多模态是高级场景

### 为什么不是"Partial Variables"？

**理由**:
- Partial Variables 是**便利特性**，而非必需
- 可以通过其他方式（如直接传递变量）实现相同功能
- 存在兼容性问题（GitHub Issue #17560）

**数据支持**:
- Context7: [Partial Variables](reference/context7_langchain_03.md) - 定位为"高级特性"
- GitHub: [兼容性问题](reference/search_partial_01.md) - Issue #17560, #30049

### 为什么不是"模板组合"？

**理由**:
- 模板组合是**工程优化**，而非核心能力
- 可以通过 `from_messages` 直接构建完整模板
- 组合主要用于代码复用，而非功能实现

**数据支持**:
- 源码: `chat.py:1006-1044` - `__add__` 方法是辅助功能

## 在 RAG 开发中的核心应用

### 应用场景 1: 对话式 RAG 系统

**核心价值**: 多角色 + 历史占位符

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser

# RAG 对话模板
rag_prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Answer based on the context: {context}"),
    MessagesPlaceholder("history"),  # 对话历史
    ("human", "{question}")
])

# 构建 RAG 链
rag_chain = rag_prompt | ChatOpenAI() | StrOutputParser()

# 使用
result = rag_chain.invoke({
    "context": "LangChain is a framework for building LLM applications.",
    "history": [
        ("human", "What is LangChain?"),
        ("ai", "LangChain is a framework for building LLM applications.")
    ],
    "question": "What can I build with it?"
})
```

**为什么需要 ChatPromptTemplate**:
- System 消息注入 RAG 上下文
- MessagesPlaceholder 保持对话连贯性
- Human 消息传递用户问题

**数据支持**:
- 社区: [AWS 聊天机器人](reference/search_messagesplaceholder_01.md) - 完整 RAG 对话系统

## 核心原理的第一性思考

### 为什么对话需要角色？

**第一性原理**: 对话是**多方交互**，而非单向输出

- 人类对话中，我们通过**说话者身份**理解语境
- LLM 需要知道**谁在说话**才能生成合适的回复
- System 角色定义**规则**，Human 角色提供**输入**，AI 角色展示**期望输出**

**类比**: 就像剧本中的角色标注
```
系统: [旁白] 你是一个翻译助手
用户: [观众提问] Hello
AI: [演员回答] Bonjour
```

---

### 为什么需要历史占位符？

**第一性原理**: 对话是**有状态的**，而非无状态的

- 人类对话依赖**上下文记忆**
- 每次对话都需要**之前的信息**
- 占位符让历史注入**标准化**和**可控**

**类比**: 就像聊天软件的消息记录
```
[历史消息区域]
- 用户: What's 5 + 2?
- AI: 5 + 2 is 7
[当前输入]
- 用户: Now multiply that by 4
```

---

### 为什么需要标准化接口？

**第一性原理**: 复杂系统需要**模块化**和**可组合性**

- 对话系统涉及多个组件（LLM、Memory、Tools）
- 标准化接口让组件**解耦**
- 可组合性提升**开发效率**

**类比**: 就像 USB 接口
- 不同设备（键盘、鼠标、硬盘）都用同一接口
- 可以自由组合和替换
- 降低了系统复杂度

## 核心误区警示

### 误区 1: "ChatPromptTemplate 只是多了角色标签"

**错误**: 认为只是在字符串前加了 "system:", "human:" 等标签

**真相**:
- 输出是**消息对象**，而非字符串
- 消息对象包含**元数据**（如 additional_kwargs）
- ChatModel 直接处理消息对象，无需解析

**数据支持**:
- 源码: `chat.py:789-1484` - 返回 `ChatPromptValue` 对象

---

### 误区 2: "MessagesPlaceholder 只是变量占位符"

**错误**: 认为和 `{variable}` 一样，只是替换变量

**真相**:
- MessagesPlaceholder 接受**消息列表**，而非单个值
- 支持 `optional=True`（普通变量不支持）
- 支持 `n_messages` 限制（普通变量不支持）
- 自动转换元组为消息对象

**数据支持**:
- 源码: `chat.py:52-217` - MessagesPlaceholder 实现

---

### 误区 3: "可以用 PromptTemplate 替代 ChatPromptTemplate"

**错误**: 认为手动拼接字符串可以实现相同功能

**真相**:
- ChatModel 需要**消息对象**，而非字符串
- 手动拼接无法保留**消息元数据**
- 无法利用 LangChain 生态的**标准化接口**

**数据支持**:
- Context7: [官方建议](reference/context7_langchain_01.md) - 推荐使用 ChatPromptTemplate
