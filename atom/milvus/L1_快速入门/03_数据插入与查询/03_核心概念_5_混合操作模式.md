# 核心概念 5: 混合操作模式

> **深入理解 Milvus 的混合搜索：多向量搜索、Dense+Sparse 混合、重排序**

---

## 概述

混合操作模式（Hybrid Operations）是 Milvus 的高级特性，它允许在单次查询中结合多种搜索策略，以提升检索质量和准确性。在 RAG 应用中，混合搜索能够同时利用语义理解（Dense 向量）和关键词匹配（Sparse 向量）的优势，显著提升检索效果。

### 核心价值

在 RAG 应用中，混合操作的作用是：
1. **提升召回率**: 结合多种搜索策略，减少漏检
2. **提升精确度**: 通过重排序优化结果顺序
3. **平衡语义和关键词**: Dense 向量捕捉语义，Sparse 向量匹配关键词
4. **适应复杂查询**: 处理需要多维度匹配的查询

---

## 混合搜索的类型

### 1. 多向量搜索（Multi-Vector Search）

在同一个 Collection 中存储多个向量字段，搜索时对多个向量字段同时进行 ANN 搜索。

**应用场景**:
- 多模态搜索（文本 + 图片）
- 多字段搜索（标题 + 内容 + 摘要）
- 多模型搜索（不同嵌入模型的向量）

### 2. Dense + Sparse 混合搜索

结合密集向量（语义）和稀疏向量（关键词）进行搜索。

**应用场景**:
- 文档检索（需要语义理解 + 关键词匹配）
- 问答系统（理解问题意图 + 匹配关键术语）
- 代码搜索（理解功能 + 匹配函数名）

### 3. 向量搜索 + 标量过滤

在向量搜索的基础上添加标量字段的过滤条件。

**应用场景**:
- 按类别过滤的语义搜索
- 按时间范围的相似推荐
- 按权限控制的内容检索

## 多向量搜索

### 基本概念

多向量搜索允许在一个 Collection 中定义多个向量字段，并在搜索时对这些字段同时进行 ANN 搜索，然后合并结果。

### Collection Schema 定义

```python
from pymilvus import MilvusClient, DataType, Function, FunctionType

client = MilvusClient(uri="http://localhost:19530")

# 创建 Schema
schema = client.create_schema()

# 添加字段
schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
schema.add_field("document", DataType.VARCHAR, max_length=5000)

# 添加多个向量字段
schema.add_field("title_vector", DataType.FLOAT_VECTOR, dim=768)    # 标题向量
schema.add_field("content_vector", DataType.FLOAT_VECTOR, dim=768)  # 内容向量
schema.add_field("summary_vector", DataType.FLOAT_VECTOR, dim=768)  # 摘要向量

# 配置多个 Embedding Functions
schema.add_function(Function(
    name="title_embedding",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["title"],
    output_field_names=["title_vector"],
    params={"provider": "openai", "model_name": "text-embedding-3-small"}
))

schema.add_function(Function(
    name="content_embedding",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["content"],
    output_field_names=["content_vector"],
    params={"provider": "openai", "model_name": "text-embedding-3-small"}
))
```

### 混合搜索实现

```python
from pymilvus import AnnSearchRequest, RRFRanker

# 准备查询向量
query_text = "What is Milvus?"

# 创建多个搜索请求
search_req_title = AnnSearchRequest(
    data=[query_text],  # Embedding Function 自动嵌入
    anns_field="title_vector",
    param={"metric_type": "COSINE"},
    limit=100  # 每个搜索返回 100 个候选
)

search_req_content = AnnSearchRequest(
    data=[query_text],
    anns_field="content_vector",
    param={"metric_type": "COSINE"},
    limit=100
)

# 执行混合搜索
results = client.hybrid_search(
    collection_name="multi_vector_collection",
    search_requests=[search_req_title, search_req_content],
    ranker=RRFRanker(),  # 使用 RRF 重排序
    limit=10,  # 最终返回 10 个结果
    output_fields=["document", "title"]
)

# 打印结果
for hit in results[0]:
    print(f"ID: {hit['id']}, 分数: {hit['distance']:.4f}")
    print(f"标题: {hit['entity']['title']}\n")
```

## Dense + Sparse 混合搜索

### 什么是 Dense 和 Sparse 向量？

**Dense 向量（密集向量）**:
- 维度: 通常 384-1536 维
- 特点: 大部分维度非零
- 优势: 捕捉语义关系
- 生成: 神经网络嵌入模型（BERT, OpenAI 等）

**Sparse 向量（稀疏向量）**:
- 维度: 通常 10,000+ 维
- 特点: 大部分维度为零
- 优势: 精确关键词匹配
- 生成: BM25, SPLADE 等算法

### 为什么需要混合？

| 场景 | Dense 向量 | Sparse 向量 | 混合搜索 |
|------|-----------|------------|----------|
| "vector database" | ✅ 找到 "Milvus", "Pinecone" | ❌ 可能漏掉精确匹配 | ✅ 最佳 |
| "Milvus 2.6" | ❌ 可能找到 "Milvus 2.5" | ✅ 精确匹配版本号 | ✅ 最佳 |
| "如何优化性能" | ✅ 理解意图 | ❌ 关键词不明确 | ✅ 最佳 |

### Collection Schema 定义

```python
# 创建支持 Dense + Sparse 的 Collection
schema = client.create_schema()

schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
schema.add_field("document", DataType.VARCHAR, max_length=5000)

# Dense 向量字段
schema.add_field("dense_vector", DataType.FLOAT_VECTOR, dim=768)

# Sparse 向量字段
schema.add_field("sparse_vector", DataType.SPARSE_FLOAT_VECTOR)

# 配置 Dense Embedding Function
schema.add_function(Function(
    name="dense_embedding",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["document"],
    output_field_names=["dense_vector"],
    params={"provider": "openai", "model_name": "text-embedding-3-small"}
))

# 配置 Sparse Embedding Function（如 BM25）
schema.add_function(Function(
    name="sparse_embedding",
    function_type=FunctionType.BM25,
    input_field_names=["document"],
    output_field_names=["sparse_vector"]
))
```

### 混合搜索实现

```python
# 准备查询
query_text = "Milvus 2.6 new features"

# 创建 Dense 搜索请求
dense_req = AnnSearchRequest(
    data=[query_text],
    anns_field="dense_vector",
    param={"metric_type": "COSINE"},
    limit=100
)

# 创建 Sparse 搜索请求
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},  # Sparse 通常使用 IP
    limit=100
)

# 执行混合搜索
results = client.hybrid_search(
    collection_name="hybrid_collection",
    search_requests=[dense_req, sparse_req],
    ranker=RRFRanker(),  # RRF 重排序
    limit=10,
    output_fields=["document"]
)
```

## 重排序策略（Reranker）

### 什么是重排序？

重排序（Reranking）是在多个搜索结果合并后，重新计算每个结果的最终分数并排序的过程。

### 支持的重排序器

**1. RRFRanker（Reciprocal Rank Fusion）**

**原理**: 基于倒数排名融合，无需参数。

```python
from pymilvus import RRFRanker

ranker = RRFRanker()
```

**计算公式**:
```
RRF_score = Σ (1 / (k + rank_i))
```
其中 k 是常数（默认 60），rank_i 是结果在第 i 个搜索中的排名。

**优点**:
- ✅ 无需参数调优
- ✅ 对不同搜索结果的分数范围不敏感
- ✅ 鲁棒性好

**2. WeightedRanker（加权重排序）**

**原理**: 为每个搜索请求分配权重，加权求和。

```python
from pymilvus import WeightedRanker

ranker = WeightedRanker(weights=[0.7, 0.3])  # Dense 70%, Sparse 30%
```

**计算公式**:
```
Weighted_score = w1 * score1 + w2 * score2
```

**优点**:
- ✅ 可以控制不同搜索的重要性
- ✅ 适合已知某个搜索更重要的场景

**缺点**:
- ❌ 需要手动调优权重
- ❌ 对分数范围敏感

### 重排序器对比

| 重排序器 | 参数 | 适用场景 | 优点 | 缺点 |
|----------|------|----------|------|------|
| **RRFRanker** | 无 | 通用场景 | 无需调参，鲁棒 | 无法控制权重 |
| **WeightedRanker** | 权重列表 | 已知重要性 | 可控制权重 | 需要调参 |

### 重排序示例

```python
# 示例：比较不同重排序器的效果

# 1. RRF 重排序（推荐）
results_rrf = client.hybrid_search(
    collection_name="hybrid_collection",
    search_requests=[dense_req, sparse_req],
    ranker=RRFRanker(),
    limit=10
)

# 2. 加权重排序（Dense 更重要）
results_weighted = client.hybrid_search(
    collection_name="hybrid_collection",
    search_requests=[dense_req, sparse_req],
    ranker=WeightedRanker(weights=[0.8, 0.2]),  # Dense 80%, Sparse 20%
    limit=10
)

# 3. 加权重排序（Sparse 更重要，适合关键词查询）
results_weighted2 = client.hybrid_search(
    collection_name="hybrid_collection",
    search_requests=[dense_req, sparse_req],
    ranker=WeightedRanker(weights=[0.3, 0.7]),  # Dense 30%, Sparse 70%
    limit=10
)
```

## 向量搜索 + 标量过滤

### 基本用法

```python
# 在向量搜索中添加标量过滤
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    anns_field="dense_vector",
    filter='category == "tech" and score > 0.8',  # 标量过滤
    limit=10,
    output_fields=["document", "category", "score"]
)
```

### 过滤时机

**Pre-filtering（预过滤）**:
```python
# 先过滤，再搜索
# Milvus 默认使用预过滤
results = client.search(
    data=[query_vector],
    filter='category == "tech"',  # 先过滤出 tech 类别
    limit=10  # 然后在过滤结果中搜索 top-10
)
```

**Post-filtering（后过滤）**:
```python
# 先搜索，再过滤
# 需要增大 limit 以确保过滤后有足够结果
results = client.search(
    data=[query_vector],
    filter='category == "tech"',
    limit=100  # 搜索 100 个，过滤后可能只剩 10 个
)
```

## RAG 应用场景

### 场景1: 文档问答（Dense + Sparse）

```python
def hybrid_document_qa(question, category=None):
    """混合搜索文档问答"""
    # 创建 Dense 搜索请求（语义理解）
    dense_req = AnnSearchRequest(
        data=[question],
        anns_field="dense_vector",
        param={"metric_type": "COSINE"},
        limit=50
    )

    # 创建 Sparse 搜索请求（关键词匹配）
    sparse_req = AnnSearchRequest(
        data=[question],
        anns_field="sparse_vector",
        param={"metric_type": "IP"},
        limit=50
    )

    # 执行混合搜索
    results = client.hybrid_search(
        collection_name="doc_collection",
        search_requests=[dense_req, sparse_req],
        ranker=RRFRanker(),
        limit=5,
        output_fields=["document"]
    )

    # 提取文档内容
    contexts = [hit['entity']['document'] for hit in results[0]]

    # 构建 prompt 并调用 LLM
    context_text = "\n\n".join(contexts)
    prompt = f"基于以下上下文回答问题：\n\n{context_text}\n\n问题：{question}"

    answer = call_llm(prompt)
    return answer

# 使用示例
answer = hybrid_document_qa("What are the new features in Milvus 2.6?")
```

### 场景2: 多字段搜索

```python
def multi_field_search(query):
    """搜索标题、内容、摘要"""
    # 创建多个搜索请求
    title_req = AnnSearchRequest(
        data=[query],
        anns_field="title_vector",
        param={"metric_type": "COSINE"},
        limit=50
    )

    content_req = AnnSearchRequest(
        data=[query],
        anns_field="content_vector",
        param={"metric_type": "COSINE"},
        limit=50
    )

    summary_req = AnnSearchRequest(
        data=[query],
        anns_field="summary_vector",
        param={"metric_type": "COSINE"},
        limit=50
    )

    # 执行混合搜索
    results = client.hybrid_search(
        collection_name="multi_field_collection",
        search_requests=[title_req, content_req, summary_req],
        ranker=WeightedRanker(weights=[0.5, 0.3, 0.2]),  # 标题最重要
        limit=10,
        output_fields=["title", "content", "summary"]
    )

    return results[0]
```

### 场景3: 带过滤的混合搜索

```python
def filtered_hybrid_search(query, category, min_score):
    """带标量过滤的混合搜索"""
    # 构建过滤条件
    filter_expr = f'category == "{category}" and score >= {min_score}'

    # Dense 搜索
    dense_req = AnnSearchRequest(
        data=[query],
        anns_field="dense_vector",
        param={"metric_type": "COSINE"},
        limit=50
    )

    # Sparse 搜索
    sparse_req = AnnSearchRequest(
        data=[query],
        anns_field="sparse_vector",
        param={"metric_type": "IP"},
        limit=50
    )

    # 执行混合搜索（带过滤）
    results = client.hybrid_search(
        collection_name="filtered_collection",
        search_requests=[dense_req, sparse_req],
        ranker=RRFRanker(),
        filter=filter_expr,  # 添加过滤条件
        limit=10,
        output_fields=["document", "category", "score"]
    )

    return results[0]

# 使用示例
results = filtered_hybrid_search(
    query="vector database tutorial",
    category="tech",
    min_score=0.8
)
```

## 性能优化

### 1. 合理设置候选数量

```python
# ❌ 候选数量过小
dense_req = AnnSearchRequest(..., limit=10)  # 可能导致重排序效果不佳

# ✅ 候选数量适中
dense_req = AnnSearchRequest(..., limit=50-100)  # 平衡性能和效果
```

### 2. 选择合适的重排序器

```python
# 通用场景：使用 RRF
ranker = RRFRanker()

# 已知重要性：使用 Weighted
ranker = WeightedRanker(weights=[0.7, 0.3])
```

### 3. 优化过滤条件

```python
# ❌ 复杂过滤
filter='a > 0 and b < 10 and c in [1,2,3]'

# ✅ 简单过滤
filter='category == "tech"'
```

## 常见问题

### Q1: 混合搜索比单一搜索慢多少？

**A**: 混合搜索的时间约为单一搜索的 N 倍（N 为搜索请求数量），但通常在可接受范围内：
- 单一搜索: ~10ms
- 双向量混合: ~20ms
- 三向量混合: ~30ms

### Q2: 如何选择重排序器？

**A**:
- **默认选择**: RRFRanker（无需调参，效果好）
- **已知重要性**: WeightedRanker（可控制权重）
- **实验对比**: 在实际数据上对比效果

### Q3: Dense 和 Sparse 的权重如何设置？

**A**: 经验值：
- **通用查询**: Dense 70%, Sparse 30%
- **关键词查询**: Dense 30%, Sparse 70%
- **语义查询**: Dense 80%, Sparse 20%

建议在实际数据上实验调优。

### Q4: 混合搜索的候选数量如何设置？

**A**: 推荐值：
- 最终返回 10 个：候选 50-100 个
- 最终返回 20 个：候选 100-200 个
- 候选数量 = 最终返回数量 × 5-10

## 总结

### 核心要点

1. **多向量搜索**: 在多个向量字段上同时搜索，适合多模态或多字段场景
2. **Dense + Sparse**: 结合语义和关键词，提升检索质量
3. **重排序**: RRFRanker（推荐）或 WeightedRanker
4. **标量过滤**: 在向量搜索基础上添加过滤条件
5. **性能优化**: 合理设置候选数量、选择合适的重排序器

### 最佳实践

| 场景 | 推荐配置 |
|------|----------|
| **文档问答** | Dense + Sparse, RRFRanker, 候选 50-100 |
| **多字段搜索** | 多向量, WeightedRanker, 候选 50-100 |
| **带过滤搜索** | Dense + 标量过滤, 候选 50-100 |
| **关键词查询** | Dense + Sparse, Weighted(0.3, 0.7) |
