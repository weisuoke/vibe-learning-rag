## 基础概念题

### Q1: 什么是Retriever?它与VectorStore有什么区别?

**标准答案**:

Retriever是LangChain中的统一检索接口,而VectorStore是具体的向量存储实现。

**核心区别**:
1. **抽象层级**:
   - Retriever:抽象接口,定义统一的检索协议
   - VectorStore:具体实现,提供向量存储和检索功能

2. **接口定义**:
   - Retriever:输入str,输出list[Document]
   - VectorStore:提供多种检索方法(similarity_search, mmr等)

3. **Runnable集成**:
   - Retriever:实现Runnable接口,可在LCEL管道中使用
   - VectorStore:不是Runnable,需要转换

4. **转换关系**:
```python
retriever = vectorstore.as_retriever()
```

**追问**: 为什么需要Retriever抽象层?

**答**: 统一不同检索实现的接口,支持LCEL集成,提供回调机制和LangSmith追踪。

---

### Q2: as_retriever()方法的作用是什么?

**标准答案**:

as_retriever()是VectorStore的工厂方法,用于将VectorStore转换为Retriever。

**核心功能**:
1. **适配器模式**:将VectorStore适配到BaseRetriever接口
2. **策略配置**:通过search_type选择检索策略
3. **参数管理**:通过search_kwargs配置检索参数
4. **标签生成**:自动生成追踪标签

**代码示例**:
```python
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 10, "lambda_mult": 0.5},
    tags=["production"]
)
```

**追问**: as_retriever()返回的是什么类型?

**答**: VectorStoreRetriever,它继承自BaseRetriever。

---

### Q3: LangChain支持哪三种检索策略?各有什么特点?

**标准答案**:

**三种策略**:

1. **similarity**(相似度搜索):
   - 算法:余弦相似度Top-K
   - 特点:简单快速,适合大多数场景
   - 参数:k(返回文档数量)
   - 适用:通用文档检索

2. **mmr**(最大边际相关性):
   - 算法:平衡相关性和多样性
   - 特点:避免结果重复
   - 参数:k, fetch_k, lambda_mult
   - 适用:推荐系统,需要多样性的场景

3. **similarity_score_threshold**(阈值过滤):
   - 算法:只返回高于阈值的文档
   - 特点:质量控制严格
   - 参数:k, score_threshold
   - 适用:高精度问答

**对比表**:

| 策略 | 速度 | 多样性 | 质量控制 | 适用场景 |
|------|------|--------|----------|----------|
| similarity | 快 | 低 | 中 | 通用检索 |
| mmr | 慢 | 高 | 中 | 推荐系统 |
| threshold | 快 | 低 | 高 | 精准问答 |

## 参数配置题

### Q4: MMR的lambda_mult参数如何影响检索结果?

**标准答案**:

lambda_mult控制相关性和多样性的平衡,取值范围[0, 1]。

**公式**:
```
MMR = λ * Sim(q, d) - (1-λ) * max(Sim(d, d_i))
```

**影响**:
- **λ=1**: 最大相关性,最小多样性(退化为similarity搜索)
- **λ=0.8-0.9**: 高相关性,低多样性
- **λ=0.5**: 平衡相关性和多样性(默认值)
- **λ=0.2-0.3**: 低相关性,高多样性
- **λ=0**: 最大多样性,最小相关性

**选择建议**:
- 文档问答:λ=0.7-0.9
- 推荐系统:λ=0.3-0.5
- 探索性搜索:λ=0.1-0.3

**追问**: fetch_k参数的作用是什么?

**答**: fetch_k是MMR算法的候选文档数量,从fetch_k个候选中选择k个最终文档。推荐fetch_k >= 2*k。

### Q5: 如何选择合适的k值?

**标准答案**:

k值需要根据场景权衡,不是越大越好。

**考虑因素**:
1. **LLM上下文窗口**:k太大会超出限制
2. **文档质量**:高质量文档可以用较小的k
3. **查询复杂度**:复杂查询需要更多上下文
4. **Token成本**:k越大,成本越高
5. **精确率vs召回率**:k太大降低精确率

**推荐值**:
- 简单问答:k=3-5
- 复杂分析:k=10-20
- 推荐系统:k=20-50

**优化策略**:
```python
# 策略1:动态调整
def get_optimal_k(query_complexity):
    if complexity == "simple":
        return 3
    elif complexity == "medium":
        return 5
    else:
        return 10

# 策略2:使用threshold控制质量
retriever = vectorstore.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={"k": 20, "score_threshold": 0.75}
)
```

## 实现原理题

### Q6: Retriever如何实现Runnable接口?

**标准答案**:

BaseRetriever继承自RunnableSerializable[str, list[Document]]。

**核心方法**:
```python
class BaseRetriever(RunnableSerializable[str, list[Document]]):
    def invoke(self, query: str) -> list[Document]:
        # 1. 配置回调
        # 2. 触发on_retriever_start
        # 3. 调用_get_relevant_documents
        # 4. 触发on_retriever_end
        # 5. 返回结果
        pass

    @abstractmethod
    def _get_relevant_documents(self, query: str) -> list[Document]:
        """子类实现"""
        pass
```

**Runnable特性**:
- invoke/ainvoke:同步/异步调用
- batch/abatch:批量处理
- stream/astream:流式输出
- __or__:支持|操作符

**LCEL集成**:
```python
rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
)
```

### Q7: VectorStoreRetriever如何选择检索策略?

**标准答案**:

VectorStoreRetriever使用策略模式,根据search_type选择不同的检索方法。

**实现**:
```python
class VectorStoreRetriever(BaseRetriever):
    search_type: str = "similarity"
    search_kwargs: dict = {}

    def _get_relevant_documents(self, query: str) -> list[Document]:
        if self.search_type == "similarity":
            return self.vectorstore.similarity_search(query, **self.search_kwargs)
        elif self.search_type == "mmr":
            return self.vectorstore.max_marginal_relevance_search(query, **self.search_kwargs)
        elif self.search_type == "similarity_score_threshold":
            docs_and_scores = self.vectorstore.similarity_search_with_relevance_scores(query, **self.search_kwargs)
            return [doc for doc, _ in docs_and_scores]
```

**设计模式**:
- 策略模式:通过search_type选择策略
- 适配器模式:将VectorStore适配到BaseRetriever
- 工厂方法:as_retriever()创建实例

### Q8: 如何优化Retriever的性能?

**标准答案**:

**优化策略**:

1. **批量处理**:
```python
# 使用batch而非多次invoke
batch_results = retriever.batch(queries)
```

2. **异步处理**:
```python
# 使用异步并发
tasks = [retriever.ainvoke(q) for q in queries]
results = await asyncio.gather(*tasks)
```

```python
@lru_cache(maxsize=100)
def cached_retrieve(query):
```

- 减小k值
- 使用threshold过滤
- 优化fetch_k(MMR)

- 建立元数据索引

**性能指标**:
- 延迟:< 500ms
- 吞吐量:> 100 QPS
- 召回率:85-90%

### Q9: 如何处理Retriever的错误?

**标准答案**:

**错误处理策略**:

1. **基础错误处理**:
```python
try:
    docs = retriever.invoke(query)
except Exception as e:
    logger.error(f"检索失败: {e}")
    return []
```

2. **重试机制**:
```python
for attempt in range(max_retries):
    try:
        return retriever.invoke(query)
    except Exception as e:
        if attempt < max_retries - 1:
            time.sleep(2 ** attempt)
        else:
            raise
```

3. **降级策略**:
```python
try:
    docs = retriever.invoke(query)
except Exception:
    # 使用备用检索器或默认文档
    docs = fallback_retriever.invoke(query)
```

4. **监控告警**:
```python
# 使用回调监控
class MonitorCallback(BaseCallbackHandler):
    def on_retriever_error(self, error, **kwargs):
        alert_system.send_alert(error)
```

## 应用场景题

### Q10: 如何在RAG应用中集成Retriever?

**标准答案**:

**基础RAG管道**:
```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_openai import ChatOpenAI

# 1. 创建Retriever
retriever = vectorstore.as_retriever(search_kwargs={"k": 4})

# 2. 定义Prompt
prompt = ChatPromptTemplate.from_template("""
根据以下上下文回答问题:

上下文: {context}
问题: {question}

回答:
""")

# 3. 格式化函数
def format_docs(docs):
    return "\n\n".join([doc.page_content for doc in docs])

# 4. 构建管道
rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | ChatOpenAI()
)

# 5. 使用
answer = rag_chain.invoke("What is LangChain?")
```

**高级特性**:
- 对话历史管理
- 流式输出
- 批量处理
- 错误处理
