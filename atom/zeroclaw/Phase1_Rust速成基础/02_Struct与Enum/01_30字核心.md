## 30 字核心

**Struct 组合数据（AND），Enum 表达变体（OR），match 保证穷尽处理。**

这 30 个字浓缩了 Rust 数据建模的全部精髓。下面逐字拆解。

---

## 逐字拆解

### 关键词 1：「Struct 组合数据」

Struct 把多个字段**组合**在一起，形成一个完整的数据单元。

```rust
// Rust: 一个 Agent 同时拥有所有这些字段
struct Agent {
    name: String,           // 名称
    model: String,          // LLM 模型
    temperature: f64,       // 温度参数
    max_tokens: u32,        // 最大 token 数
}
```

```typescript
// TypeScript 等价物：interface
interface Agent {
  name: string;
  model: string;
  temperature: number;
  maxTokens: number;
}
```

一个 `Agent` 实例**必须同时拥有**所有四个字段。缺一不可，少一个都编译不过。

### 关键词 2：「AND」

Struct 是**积类型**（Product Type）-- 字段之间是 AND 关系。

**为什么叫"积类型"？** Config 的可能取值 = String 的可能值 x u16 的可能值 x bool 的可能值。笛卡尔积（乘法），所以叫 Product Type。

**前端类比**：React 组件的 props -- `<Button size="lg" color="blue" disabled={false} />`，三个 prop 都要传，它们是 AND 关系。

**日常生活类比**：快递单 -- 收件人 AND 地址 AND 电话 AND 物品。每项都要填，少一项发不出去。

### 关键词 3：「Enum 表达变体」

Enum 定义一个类型的**多种可能形态**，某一时刻只能是其中一种。

```rust
// Rust: ConversationMessage 在任意时刻只能是三种之一
enum ConversationMessage {
    Chat(ChatMessage),                  // 形态 1：普通聊天
    AssistantToolCalls(Vec<ToolCall>),   // 形态 2：AI 调用工具
    ToolResults(Vec<ToolResult>),        // 形态 3：工具返回结果
}
```

```typescript
// TypeScript 等价物：discriminated union
type ConversationMessage =
  | { type: "Chat"; data: ChatMessage }
  | { type: "AssistantToolCalls"; data: ToolCall[] }
  | { type: "ToolResults"; data: ToolResult[] };
```

每个 Rust enum 变体可以**携带完全不同的数据**，远比 C/Java 枚举（只是常量数字）强大。

### 关键词 4：「OR」

Enum 是**和类型**（Sum Type）-- 数据的不相交联合。

```rust
enum AutonomyLevel {
    ReadOnly,     // 只读模式 OR
    Supervised,   // 监督模式 OR
    Full,         // 完全自主模式
}
```

某一时刻只能是 ReadOnly **或者** Supervised **或者** Full。不能同时是两种。

**前端类比**：React 状态机 -- 页面要么 `loading`，要么 `success`，要么 `error`。

**日常生活类比**：交通信号灯 -- 红灯 OR 黄灯 OR 绿灯。任意时刻只亮一种颜色。

### 关键词 5-6：「match 保证穷尽处理」

`match` 要求你**必须处理所有可能的变体**，漏掉任何一个都无法编译。

```rust
fn describe_autonomy(level: &AutonomyLevel) -> &str {
    match level {
        AutonomyLevel::ReadOnly => "AI can only read, no actions",
        AutonomyLevel::Supervised => "AI acts with human approval",
        AutonomyLevel::Full => "AI acts autonomously",
    }
}
// 删掉 Full 那一行？编译器报错：
// error[E0004]: non-exhaustive patterns: `AutonomyLevel::Full` not covered
```

```typescript
// TypeScript: switch 不强制穷尽，漏了 "full" 不会报错
function describe(level: "readonly" | "supervised" | "full"): string {
  switch (level) {
    case "readonly": return "read only";
    case "supervised": return "supervised";
    // 忘记 "full"？TypeScript 默认不报错！
  }
}
```

Rust 在**编译阶段**就帮你找到遗漏，TypeScript 在运行时才暴露（还可能是 `undefined`）。

## 与 TypeScript 的深度对比

### 对比 1：interface vs struct

| 维度 | TypeScript interface | Rust struct |
|------|---------------------|-------------|
| 运行时存在？ | 不存在（类型擦除） | 存在（决定内存布局） |
| 类型系统 | **结构类型**（形状相同就兼容） | **名义类型**（必须是同一个 struct） |
| 可以有方法？ | 不能（只有 class 可以） | 可以（通过 impl 块） |
| 字段默认可见性 | 全部公有 | **全部私有**（需要 pub） |
| 可选字段 | `field?: type` | `field: Option<type>` |
| 所有权 | 无（GC 管理） | 有（所有权系统） |

```rust
// ZeroClaw 的 SecurityPolicy struct
pub struct SecurityPolicy {
    pub autonomy_level: AutonomyLevel,  // pub: 外部可访问
    allowed_tools: Vec<String>,          // 私有：外部不可直接访问
    max_iterations: u32,                 // 私有：需要通过方法访问
}
```

> Rust struct 的**默认私有**迫使你思考 API 设计：哪些字段应该暴露给外部？

### 对比 2：union type vs enum

| 维度 | TypeScript union type | Rust enum |
|------|----------------------|-----------|
| 判别方式 | 手动加 `kind`/`type` 字段 | **自动内置判别符** |
| 穷尽检查 | 需要额外配置 `never` | **默认强制穷尽** |
| 可以有方法？ | 不可以 | **可以**（impl 块） |
| 内存布局 | 每个对象独立 | **共用最大变体的空间** |
| 拼写错误？ | `"circl"` 运行时才爆 | **编译时就报错** |

```rust
// Rust: enum 自带判别，无需手动 "kind" 字段
enum Shape {
    Circle(f64),
    Rectangle { width: f64, height: f64 },
    Triangle { base: f64, height: f64 },
}

fn area(shape: &Shape) -> f64 {
    match shape {
        Shape::Circle(r) => std::f64::consts::PI * r * r,
        Shape::Rectangle { width, height } => width * height,
        Shape::Triangle { base, height } => 0.5 * base * height,
    }
}
```

### 对比 3：switch vs match

match 远比 switch 强大：

```rust
// 1. 解构变体并绑定数据（switch 做不到）
match message {
    ConversationMessage::Chat(msg) => {
        println!("Role: {}", msg.role);  // msg 自动绑定为 ChatMessage
    },
    ConversationMessage::AssistantToolCalls(calls) => {
        println!("Calling {} tools", calls.len());
    },
    ConversationMessage::ToolResults(results) => {
        println!("Got {} results", results.len());
    },
}

// 2. 守卫条件（guard）
match temperature {
    t if t < 0.0 => println!("Too cold"),
    t if t > 2.0 => println!("Too hot"),
    t => println!("Temperature {} is fine", t),
}

// 3. match 是表达式，可以返回值（switch 是语句）
let label = match level {
    AutonomyLevel::ReadOnly => "safe",
    AutonomyLevel::Supervised => "moderate",
    AutonomyLevel::Full => "powerful",
};
```

**switch 的其他劣势**：无法嵌套解构、需要 break 防止 fall-through、无法直接赋值。

## 编译时保障：穷尽性检查的价值

### 场景：ZeroClaw 新增一个 AutonomyLevel 变体

```rust
pub enum AutonomyLevel {
    ReadOnly,
    Supervised,
    Full,
    Custom(String),    // 新增：自定义安全级别
}
```

**Rust**：项目中所有 `match AutonomyLevel` 的代码**立即编译失败**，精确指出需要更新的位置。

**TypeScript**：旧代码中的 switch 不会报错，默默返回 `undefined`。

### ZeroClaw 真实案例：MemoryCategory

```rust
// src/memory/traits.rs
pub enum MemoryCategory {
    Conversation,       // 对话记忆
    Knowledge,          // 知识记忆
    Task,               // 任务记忆
    Custom(String),     // 自定义类别
}

fn category_label(cat: &MemoryCategory) -> String {
    match cat {
        MemoryCategory::Conversation => "conversation".to_string(),
        MemoryCategory::Knowledge => "knowledge".to_string(),
        MemoryCategory::Task => "task".to_string(),
        MemoryCategory::Custom(name) => format!("custom:{}", name),
        // 漏掉 Custom？编译器报错
        // 未来新增 Metadata 变体？编译器也报错
    }
}
```

## 常见误解澄清

### 误解 1：「Rust enum 就像 C/Java 的枚举」

**错！** C/Java 枚举本质是**命名的整数常量**。Rust enum 是**代数数据类型**（ADT），每个变体可携带不同数据：

```rust
// Rust enum: 每个变体是完全不同的"形状"
enum ConversationMessage {
    Chat(ChatMessage),                   // 携带一个 struct
    AssistantToolCalls(Vec<ToolCall>),    // 携带一个 Vec
    ToolResults(Vec<ToolResult>),         // 携带另一个 Vec
}
```

**正确理解**：Rust enum 更接近 TypeScript 的 discriminated union，但有编译器强制穷尽检查和内置方法。

### 误解 2：「Rust struct 就像 JavaScript class」

**错！** 关键区别：

| JavaScript class | Rust struct |
|-----------------|-------------|
| `class Dog extends Animal` | **没有继承** |
| 隐式 `this` | 显式 `&self` / `&mut self` / `self` |
| `constructor()` 特殊语法 | `fn new()` 只是约定名称 |
| 数据+行为一起定义 | **分开**：struct 定义数据，impl 定义行为 |
| 有垃圾回收 | **没有 GC**，靠所有权管理 |

**正确理解**：struct + impl 是"数据与行为分离"的设计。没有继承，用组合代替。

### 误解 3：「match 就是 switch 换个名字」

**错！** match 有四个 switch 做不到的能力：
1. **自动解构并绑定变量** -- `Chat(msg)` 中 msg 直接可用
2. **守卫条件** -- `t if t < 0.0`
3. **嵌套解构** -- 可以解构 struct 内部的 struct
4. **是表达式** -- 可以 `let x = match ... { }`

## ZeroClaw 源码中的真实案例

### 案例 1：AutonomyLevel -- 安全模式控制

```rust
impl SecurityPolicy {
    pub fn can_execute_tool(&self, tool_name: &str) -> bool {
        match &self.autonomy_level {
            AutonomyLevel::ReadOnly => false,              // 一律禁止
            AutonomyLevel::Supervised => {
                self.requires_approval(tool_name)           // 需要审批
            },
            AutonomyLevel::Full => {
                self.is_tool_allowed(tool_name)             // 检查白名单
            },
        }
    }
}
```

### 案例 2：ConversationMessage -- 为什么用 enum 不用 struct

```rust
// 错误设计：用 struct + 可选字段表达互斥数据
struct ConversationMessage {
    chat: Option<ChatMessage>,              // 可能有
    tool_calls: Option<Vec<ToolCall>>,      // 可能有
    tool_results: Option<Vec<ToolResult>>,  // 可能有
}
// 问题：允许同时存在 chat 和 tool_calls，这是无效状态！

// 正确设计：enum 保证互斥（Make Invalid States Unrepresentable）
enum ConversationMessage {
    Chat(ChatMessage),
    AssistantToolCalls(Vec<ToolCall>),
    ToolResults(Vec<ToolResult>),
}
// 某一时刻只能是三种之一，不可能同时是 Chat 又是 ToolResults
```

> **设计原则**：数据是互斥关系（OR）用 enum；数据是共存关系（AND）用 struct。

## AND vs OR 可视化

```
┌─────────────────────────────────────────────┐
│              Struct（AND 类型）                │
│                                             │
│   Agent = name AND model AND temperature    │
│                                             │
│   ┌──────┐ ┌──────┐ ┌─────────────┐        │
│   │ name │+│model │+│ temperature │        │
│   └──────┘ └──────┘ └─────────────┘        │
│   所有字段同时存在                             │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│              Enum（OR 类型）                   │
│                                             │
│   Message = Chat OR ToolCalls OR Results    │
│                                             │
│   ┌──────┐   ┌───────────┐   ┌─────────┐  │
│   │ Chat │ | │ ToolCalls │ | │ Results │  │
│   └──────┘   └───────────┘   └─────────┘  │
│   同一时刻只能是其中一种                       │
└─────────────────────────────────────────────┘
```

## 一句话记忆口诀

> **「Struct 是快递箱（所有东西打包在一起），Enum 是交通灯（同一时刻只亮一种颜色），match 是安检门（每种情况都要过检）。」**

- **Struct = 快递箱 = AND**：所有字段打包在一起
- **Enum = 交通灯 = OR**：某一时刻只能是一种状态
- **match = 安检门 = 穷尽**：每种可能都必须检查
