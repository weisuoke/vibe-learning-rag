## 技巧 1：动态节点注册与运行时图修改

### 适用场景
- 插件系统：根据配置动态加载节点
- 可配置工作流：用户自定义流程
- A/B 测试：动态切换节点实现

### 原理解析

StateGraph 采用 Builder 模式，在 `compile()` 前可以动态添加节点。

**源码依据**（state.py:289-354）：
```python
def add_node(self, node: StateNode, ...) -> Self:
    # 返回 Self，支持链式调用
    # 编译前可以多次调用
```

### 完整代码示例

```python
"""
动态节点注册实战
演示：根据配置动态构建 RAG 工作流
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict
from typing import Callable

class RAGState(TypedDict):
    query: str
    documents: list[str]
    answer: str

# 节点配置
NODE_REGISTRY: dict[str, Callable] = {}

def register_node(name: str):
    """节点注册装饰器"""
    def decorator(func: Callable):
        NODE_REGISTRY[name] = func
        return func
    return decorator

@register_node("retriever_v1")
def retriever_v1(state: RAGState) -> dict:
    return {"documents": ["doc1", "doc2"]}

@register_node("retriever_v2")
def retriever_v2(state: RAGState) -> dict:
    return {"documents": ["doc1", "doc2", "doc3"]}

@register_node("generator")
def generator(state: RAGState) -> dict:
    docs = "\n".join(state["documents"])
    return {"answer": f"Answer based on: {docs}"}

# 动态构建图
def build_graph(config: dict) -> StateGraph:
    builder = StateGraph(RAGState)

    # 根据配置选择检索器
    retriever_name = config.get("retriever", "retriever_v1")
    builder.add_node(retriever_name, NODE_REGISTRY[retriever_name])
    builder.add_node("generator", NODE_REGISTRY["generator"])

    builder.add_edge(START, retriever_name)
    builder.add_edge(retriever_name, "generator")
    builder.add_edge("generator", END)

    return builder.compile()

# 使用
config_a = {"retriever": "retriever_v1"}
graph_a = build_graph(config_a)

config_b = {"retriever": "retriever_v2"}
graph_b = build_graph(config_b)
```

### 实际应用场景
- **多租户系统**：每个租户自定义工作流
- **实验平台**：快速切换不同实现进行对比
- **插件架构**：第三方开发者注册自定义节点

### 注意事项
1. 节点注册必须在 `compile()` 前完成
2. 节点名称必须唯一，避免冲突
3. 动态节点的类型签名要一致

## 技巧 2：自定义 Reducer 函数实现复杂状态聚合

### 适用场景
- 消息去重：避免重复处理相同消息
- 优先级队列：按优先级聚合任务
- 状态合并策略：自定义合并逻辑

### 原理解析

StateGraph 使用 `Annotated` 类型添加 reducer 函数。

**源码依据**（state.py:197-250）：
```python
class State(TypedDict):
    messages: Annotated[list, reducer]  # reducer 函数
```

### 完整代码示例

```python
"""
自定义 Reducer 函数实战
演示：实现消息去重和优先级队列
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import Annotated, TypedDict
from typing import Any

# 1. 消息去重 Reducer
def dedupe_messages(existing: list[str], new: list[str]) -> list[str]:
    """去重合并消息"""
    seen = set(existing)
    result = existing.copy()
    for msg in new:
        if msg not in seen:
            result.append(msg)
            seen.add(msg)
    return result

# 2. 优先级队列 Reducer
def priority_merge(existing: list[dict], new: list[dict]) -> list[dict]:
    """按优先级合并任务"""
    all_tasks = existing + new
    # 按优先级排序（高优先级在前）
    return sorted(all_tasks, key=lambda x: x.get("priority", 0), reverse=True)

# 3. 字典深度合并 Reducer
def deep_merge_dict(existing: dict, new: dict) -> dict:
    """深度合并字典"""
    result = existing.copy()
    for key, value in new.items():
        if key in result and isinstance(result[key], dict) and isinstance(value, dict):
            result[key] = deep_merge_dict(result[key], value)
        else:
            result[key] = value
    return result

# 定义状态
class AdvancedState(TypedDict):
    messages: Annotated[list[str], dedupe_messages]
    tasks: Annotated[list[dict], priority_merge]
    metadata: Annotated[dict, deep_merge_dict]

# 使用示例
builder = StateGraph(AdvancedState)

def node_a(state: AdvancedState) -> dict:
    return {
        "messages": ["msg1", "msg2"],
        "tasks": [{"id": 1, "priority": 5}],
        "metadata": {"source": "node_a", "count": 1}
    }

def node_b(state: AdvancedState) -> dict:
    return {
        "messages": ["msg2", "msg3"],  # msg2 会被去重
        "tasks": [{"id": 2, "priority": 10}],  # 高优先级
        "metadata": {"source": "node_b", "count": 2}
    }

builder.add_node("node_a", node_a)
builder.add_node("node_b", node_b)
builder.add_edge(START, "node_a")
builder.add_edge("node_a", "node_b")
builder.add_edge("node_b", END)

graph = builder.compile()
result = graph.invoke({"messages": [], "tasks": [], "metadata": {}})

print(result["messages"])  # ['msg1', 'msg2', 'msg3']
print(result["tasks"])     # [{'id': 2, 'priority': 10}, {'id': 1, 'priority': 5}]
```

### 实际应用场景
- **多代理系统**：聚合多个代理的输出
- **增量更新**：只更新变化的部分
- **冲突解决**：自定义冲突解决策略

### 注意事项
1. Reducer 函数必须是纯函数（无副作用）
2. 返回类型必须与字段类型一致
3. 考虑性能：避免在 reducer 中进行重量级操作

## 技巧 3：条件路由的高级模式

### 适用场景
- 多分支决策：根据状态选择不同路径
- 循环执行：重复执行直到满足条件
- 动态终止：根据状态决定是否结束

### 原理解析

`add_conditional_edges` 支持复杂的路由逻辑。

**源码依据**（state.py:839-887）：
```python
def add_conditional_edges(
```

### 完整代码示例

```python
"""
高级条件路由实战
演示：循环执行 + 多分支决策 + 动态终止
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

class IterativeState(TypedDict):
    query: str
    answer: str
    iteration: int
    max_iterations: int
    quality_score: float

def generate_answer(state: IterativeState) -> dict:
    """生成答案"""
    iteration = state.get("iteration", 0) + 1
    quality = 0.5 + (iteration * 0.15)  # 模拟质量提升

    return {
        "answer": f"Answer v{iteration}",
        "iteration": iteration,
        "quality_score": quality
    }

def evaluate_quality(state: IterativeState) -> dict:
    """评估答案质量"""
    # 实际应用中可以调用 LLM 评估
    return {}

def route_decision(state: IterativeState) -> str:
    """
    路由决策函数
    返回值：
    - "regenerate": 重新生成（循环）
    - "refine": 优化答案
    - "end": 结束
    """
    iteration = state.get("iteration", 0)
    max_iter = state.get("max_iterations", 3)
    quality = state.get("quality_score", 0.0)

    # 达到最大迭代次数
    if iteration >= max_iter:
        return "end"

    # 质量足够好
    if quality >= 0.8:
        return "end"

    # 质量中等，需要优化
    if quality >= 0.6:
        return "refine"

    # 质量不够，重新生成
    return "regenerate"

def refine_answer(state: IterativeState) -> dict:
    """优化答案"""
    return {
        "answer": state["answer"] + " (refined)",
        "quality_score": state["quality_score"] + 0.1
    }

# 构建图
builder = StateGraph(IterativeState)

builder.add_node("generate", generate_answer)
builder.add_node("evaluate", evaluate_quality)
builder.add_node("refine", refine_answer)

builder.add_edge(START, "generate")
builder.add_edge("generate", "evaluate")

# 条件路由
builder.add_conditional_edges(
    "evaluate",
    route_decision,
    {
        "regenerate": "generate",  # 循环回去
        "refine": "refine",
        "end": END
    }
)

builder.add_edge("refine", END)

graph = builder.compile()

# 执行
result = graph.invoke({
    "query": "What is LangGraph?",
    "answer": "",
    "iteration": 0,
    "max_iterations": 5,
    "quality_score": 0.0
})

print(f"Final answer: {result['answer']}")
print(f"Iterations: {result['iteration']}")
print(f"Quality: {result['quality_score']}")
```

### 实际应用场景
- **自我反思 Agent**：生成 → 评估 → 改进循环
- **多策略尝试**：失败后尝试不同策略
- **渐进式优化**：逐步提升输出质量

### 注意事项
1. 必须有明确的终止条件，避免无限循环
2. 路由函数返回值必须在 `path_map` 中定义
3. 考虑循环次数限制，防止资源耗尽

## 技巧 4：子图嵌套与状态传递

### 适用场景
- 模块化设计：将复杂流程拆分为子图
- 可复用组件：子图可以在多个地方使用
- 团队协作：不同团队开发不同子图

### 原理解析

CompiledStateGraph 可以作为节点添加到父图中。

**源码依据**（_node.py:200-211）：
```python
StateNode: TypeAlias = (
    ...
    | Runnable[NodeInputT, Any]  # 支持 Runnable
)
```

### 完整代码示例

```python
"""
子图嵌套实战
演示：构建模块化 RAG 系统
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

# 子图 1：文档检索子图
class RetrievalState(TypedDict):
    query: str
    documents: list[str]

def build_retrieval_subgraph():
    builder = StateGraph(RetrievalState)

    def dense_retrieval(state: RetrievalState) -> dict:
        return {"documents": ["dense_doc1", "dense_doc2"]}

    def sparse_retrieval(state: RetrievalState) -> dict:
        docs = state.get("documents", [])
        return {"documents": docs + ["sparse_doc1"]}

    def rerank(state: RetrievalState) -> dict:
        # 重排序逻辑
        return {"documents": state["documents"][:2]}

    builder.add_node("dense", dense_retrieval)
    builder.add_node("sparse", sparse_retrieval)
    builder.add_node("rerank", rerank)

    builder.add_edge(START, "dense")
    builder.add_edge("dense", "sparse")
    builder.add_edge("sparse", "rerank")
    builder.add_edge("rerank", END)

    return builder.compile()

# 子图 2：答案生成子图
class GenerationState(TypedDict):
    query: str
    documents: list[str]
    answer: str

def build_generation_subgraph():
    builder = StateGraph(GenerationState)

    def generate(state: GenerationState) -> dict:
        docs = "\n".join(state["documents"])
        return {"answer": f"Answer based on: {docs}"}

    def validate(state: GenerationState) -> dict:
        # 验证逻辑
        return {}

    builder.add_node("generate", generate)
    builder.add_node("validate", validate)

    builder.add_edge(START, "generate")
    builder.add_edge("generate", "validate")
    builder.add_edge("validate", END)

    return builder.compile()

# 父图：组合子图
class MainState(TypedDict):
    query: str
    documents: list[str]
    answer: str

def build_main_graph():
    builder = StateGraph(MainState)

    # 添加子图作为节点
    retrieval_subgraph = build_retrieval_subgraph()
    generation_subgraph = build_generation_subgraph()

    builder.add_node("retrieval", retrieval_subgraph)
    builder.add_node("generation", generation_subgraph)

    builder.add_edge(START, "retrieval")
    builder.add_edge("retrieval", "generation")
    builder.add_edge("generation", END)

    return builder.compile()

# 使用
graph = build_main_graph()
result = graph.invoke({"query": "What is LangGraph?", "documents": [], "answer": ""})
print(result)
```

### 实际应用场景
- **企业级 RAG**：检索、重排、生成各自独立
- **多代理系统**：每个代理是一个子图
- **工作流编排**：组合多个独立的工作流

### 注意事项
1. 子图的状态类型必须与父图兼容
2. 子图的输出会合并到父图状态中
3. 子图内部的 Checkpoint 不会自动传递到父图

## 技巧 5：错误处理与重试策略

### 适用场景
- API 调用失败：自动重试
- 临时性错误：指数退避重试
- 降级策略：失败后使用备用方案

### 原理解析

`add_node` 支持 `retry_policy` 参数。

**源码依据**（state.py:289-354）：
```python
def add_node(
    self,
    node: StateNode,
    *,
    retry_policy: RetryPolicy | Sequence[RetryPolicy] | None = None,
    ...
) -> Self:
```

### 完整代码示例

```python
"""
错误处理与重试策略实战
演示：生产级容错设计
"""

from langgraph.graph import StateGraph, START, END
from langgraph.pregel import RetryPolicy
from typing_extensions import TypedDict
import random

class RobustState(TypedDict):
    query: str
    result: str
    error_count: int

def unreliable_api_call(state: RobustState) -> dict:
    """
    模拟不稳定的 API 调用
    30% 概率失败
    """
    if random.random() < 0.3:
        raise Exception("API call failed")

    return {"result": "Success", "error_count": 0}

def fallback_handler(state: RobustState) -> dict:
    """
    降级处理：使用缓存或默认值
    """
    return {
        "result": "Fallback result (from cache)",
        "error_count": state.get("error_count", 0) + 1
    }

def error_logger(state: RobustState) -> dict:
    """
    错误日志记录
    """
    error_count = state.get("error_count", 0)
    if error_count > 0:
        print(f"Warning: {error_count} errors occurred")
    return {}

# 构建图
builder = StateGraph(RobustState)

# 配置重试策略
retry_policy = RetryPolicy(
    max_attempts=3,
    backoff_factor=2.0,  # 指数退避
    retry_on=(Exception,)
)

# 添加节点（带重试）
builder.add_node(
    "api_call",
    unreliable_api_call,
    retry_policy=retry_policy
)

builder.add_node("fallback", fallback_handler)
builder.add_node("logger", error_logger)

# 条件路由：失败后走降级路径
def route_after_api(state: RobustState) -> str:
    if state.get("result"):
        return "logger"
    return "fallback"

builder.add_edge(START, "api_call")
builder.add_conditional_edges(
    "api_call",
    route_after_api,
    {
        "logger": "logger",
        "fallback": "fallback"
    }
)
builder.add_edge("fallback", "logger")
builder.add_edge("logger", END)

graph = builder.compile()

# 测试
for i in range(5):
    print(f"\n=== Test {i+1} ===")
    result = graph.invoke({"query": "test", "result": "", "error_count": 0})
    print(f"Result: {result['result']}")
```

### 实际应用场景
- **LLM API 调用**：处理速率限制和临时故障
- **数据库查询**：处理连接超时
- **外部服务集成**：处理网络波动

### 注意事项
1. 重试次数不宜过多，避免雪崩效应
2. 使用指数退避避免频繁重试
3. 记录错误日志用于监控和调试

## 技巧 6：状态持久化与断点续传

### 适用场景
- 长时间运行的工作流：支持中断和恢复
- 人机循环：等待人工审批后继续
- 故障恢复：系统崩溃后从检查点恢复

### 原理解析

`compile()` 支持 `checkpointer` 参数。

**源码依据**（state.py:1035-1153）：
```python
def compile(
    self,
    checkpointer: Checkpointer | None = None,
    interrupt_before: list[str] | None = None,
    interrupt_after: list[str] | None = None,
    ...
) -> CompiledStateGraph:
```

### 完整代码示例

```python
"""
状态持久化与断点续传实战
演示：人机循环审批流程
"""

from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver
from typing_extensions import TypedDict

class ApprovalState(TypedDict):
    request: str
    analysis: str
    approved: bool
    reason: str

def analyze_request(state: ApprovalState) -> dict:
    """分析请求"""
    return {
        "analysis": f"Analysis of: {state['request']}"
    }

def make_decision(state: ApprovalState) -> dict:
    """做出决策（需要人工审批）"""
    # 这里会被中断，等待人工输入
    return {}

def execute_action(state: ApprovalState) -> dict:
    """执行操作"""
    if state.get("approved"):
        return {"reason": "Action executed successfully"}
    else:
        return {"reason": "Action rejected"}

# 构建图
builder = StateGraph(ApprovalState)

builder.add_node("analyze", analyze_request)
builder.add_node("decision", make_decision)
builder.add_node("execute", execute_action)

builder.add_edge(START, "analyze")
builder.add_edge("analyze", "decision")
builder.add_edge("decision", "execute")
builder.add_edge("execute", END)

# 配置检查点和中断
checkpointer = MemorySaver()
graph = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["decision"]  # 在 decision 节点前中断
)

# 第一次执行：运行到中断点
config = {"configurable": {"thread_id": "approval_001"}}
result = graph.invoke(
    {"request": "Deploy to production", "analysis": "", "approved": False, "reason": ""},
    config=config
)

print("=== After first invoke ===")
print(f"Analysis: {result['analysis']}")
print("Waiting for human approval...")

# 模拟人工审批
# 在实际应用中，这里会等待人工输入
human_decision = {"approved": True}

# 第二次执行：从检查点恢复并继续
result = graph.invoke(
    {**result, **human_decision},
    config=config
)

print("\n=== After approval ===")
print(f"Approved: {result['approved']}")
print(f"Reason: {result['reason']}")

# 查看执行历史
state_history = list(graph.get_state_history(config))
print(f"\n=== State history ===")
print(f"Total checkpoints: {len(state_history)}")
```

### 实际应用场景
- **审批工作流**：等待人工审批后继续
- **长时间任务**：支持暂停和恢复
- **调试和回放**：从任意检查点重新执行

### 注意事项
1. Checkpointer 需要持久化存储（生产环境使用数据库）
2. `thread_id` 必须唯一，用于区分不同的执行实例
3. 中断点的选择要合理，避免在关键操作中间中断
