## 概述

你在 ChatGPT 网页上看到的"逐字输出"效果，底层就是**流式响应（Streaming）**——LLM 不是算完所有内容再一次性返回，而是每生成一个 token 就推送一个。

ZeroClaw 的 Provider trait 定义了流式接口：

```rust
fn stream_chat_with_system(
    &self,
    system_prompt: Option<&str>,
    message: &str,
    model: &str,
    temperature: f64,
    options: StreamOptions,
) -> stream::BoxStream<'static, StreamResult<StreamChunk>>;
```

本节从零理解 `BoxStream`、`Stream` trait 和 `tokio-stream`，通过 4 个可运行示例掌握流式处理。

## 1. 什么是 Stream？

### 前端类比

```
Rust Stream          ←→    JS/TS 对应物
─────────────────────────────────────────
Stream trait          ←→    AsyncIterator / AsyncIterable
stream.next().await   ←→    for await (const chunk of stream)
BoxStream             ←→    ReadableStream
StreamExt (map/filter)←→    TransformStream
SSE (Server-Sent Events) ←→ EventSource / fetch + getReader()
```

### 核心概念

```
Iterator（同步迭代器）：
  vec![1, 2, 3].iter()   →  Some(1) → Some(2) → Some(3) → None
  每次调用 next() 立即返回

Stream（异步迭代器）：
  async_stream           →  Some(1) → [等待] → Some(2) → [等待] → Some(3) → None
  每次调用 next().await 可能需要等待

两者关系：
  Iterator : Stream  =  fn : async fn
  同步世界 : 异步世界 = 立即返回 : 可能需要等待
```

### 示例1：从最简单的 Stream 开始

```rust
use tokio_stream::StreamExt; // 提供 next()、map()、filter() 等方法

#[tokio::main]
async fn main() {
    println!("═══ Stream 基础：从 Vec 创建 ═══\n");

    // ─── 最简单的 Stream：从 Vec 创建 ───
    // 类似 JS: async function* () { yield 1; yield 2; yield 3; }
    let mut stream = tokio_stream::iter(vec![1, 2, 3, 4, 5]);

    // 方式1：while let 逐个消费
    println!("--- while let 消费 ---");
    while let Some(value) = stream.next().await {
        println!("  收到: {}", value);
    }
    println!("  Stream 结束（返回 None）\n");

    // ─── 带延迟的 Stream（模拟异步数据源） ───
    println!("--- 带延迟的 Stream ---");
    use std::time::Duration;

    // throttle：每个元素之间至少间隔 200ms
    let mut stream = tokio_stream::iter(vec!["Hello", " ", "World", "!"])
        .throttle(Duration::from_millis(200));

    while let Some(chunk) = stream.next().await {
        print!("{}", chunk); // 逐字输出效果
        // 注意：print! 不换行，模拟 ChatGPT 打字效果
    }
    println!("\n  （模拟逐字输出完成）\n");

    // ─── Stream 组合操作 ───
    println!("--- Stream 组合操作（map + filter） ---");
    let stream = tokio_stream::iter(1..=10)
        .filter(|x| *x % 2 == 0)    // 只保留偶数
        .map(|x| x * x);            // 平方

    // collect 方式消费
    let results: Vec<i32> = stream.collect().await;
    println!("  偶数的平方: {:?}", results);
    // 输出: [4, 16, 36, 64, 100]
}
```

```

│ 操作          │ Rust (tokio-stream)    │ JS/TS (Array/AsyncIter)    │
├──────────────┼────────────────────────┼────────────────────────────┤
│ 创建          │ tokio_stream::iter()   │ async function*() {}       │
```

## 3. BoxStream 类型解析

### ZeroClaw 中的 BoxStream

```rust
// ZeroClaw Provider trait 中的流式方法：
fn stream_chat_with_system(
    &self,
    system_prompt: Option<&str>,
    message: &str,
    model: &str,
    temperature: f64,
    options: StreamOptions,
) -> stream::BoxStream<'static, StreamResult<StreamChunk>>;
//   ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//   这是什么类型？让我们拆解它
```

### 逐层拆解

```
BoxStream<'static, StreamResult<StreamChunk>>

拆解1：BoxStream<'a, T>
= Pin<Box<dyn Stream<Item = T> + Send + 'a>>
  │    │    │                    │      │
  │    │    │                    │      └── 'static: Stream 可以活任意长
  │    │    │                    └── Send: 可以跨线程传递
  │    │    └── dyn Stream: 动态分发的 Stream trait object
  │    └── Box: 堆分配（因为 dyn 大小不确定）
  └── Pin: 固定在内存中不能移动（async 状态机需要这个）

拆解2：StreamResult<StreamChunk>
= Result<StreamChunk, StreamError>

拆解3：StreamChunk
= 一个包含文本片段的结构体，比如：
  StreamChunk { content: "Hello" }
  StreamChunk { content: " World" }
  StreamChunk { content: "!" }

完整含义：
"一个堆分配的、可跨线程的、不可移动的异步流，
 每次迭代返回一个可能成功（StreamChunk）或失败（StreamError）的结果"
```

### 为什么需要 Box？

```rust
// ─── 问题：不同 Provider 返回不同类型的 Stream ───

// AnthropicProvider 的 stream 内部是 reqwest 的 HTTP 流
// OpenAIProvider 的 stream 内部是 eventsource 的 SSE 流
// MockProvider 的 stream 内部是 tokio_stream::iter

// 这三种 Stream 的具体类型不同（大小不同），
// 但 trait 定义需要统一的返回类型

// ❌ 不行：返回 impl Stream（每个实现的具体类型不同，不能统一）
// trait Provider {
//     fn stream(&self) -> impl Stream<Item = StreamChunk>;
//     // 编译错误：trait 方法不能返回 impl Trait（Rust 2024 已放宽，
//     // 但在需要 trait object 的场景仍然需要 Box）
// }

// ✅ 正确：Box 擦除具体类型，统一为 dyn Stream
trait Provider {
    fn stream(&self) -> BoxStream<'static, StreamChunk>;
    // 所有实现都返回 BoxStream，调用方不需要知道具体类型
}

// 前端类比：
// TypeScript 不需要这个，因为所有 AsyncIterator 都是同一种类型
// interface Provider {
//     stream(): AsyncIterableIterator<StreamChunk>;  // 简单！
// }
```

## 4. 模拟 LLM 流式输出（完整可运行）

```rust
use futures::stream::{self, BoxStream, StreamExt};
use std::time::Duration;
use anyhow::Result;

/// 模拟 LLM 的流式输出 chunk
#[derive(Debug, Clone)]
struct StreamChunk {
    content: String,
    is_done: bool,
}

/// 模拟 LLM Provider 的流式响应
///
/// 对应 ZeroClaw:
///   fn stream_chat_with_system(...) -> BoxStream<'static, StreamResult<StreamChunk>>
fn mock_llm_stream(prompt: &str) -> BoxStream<'static, Result<StreamChunk>> {
    // 模拟 LLM 逐 token 输出的内容
    let tokens: Vec<String> = format!("关于「{}」，我的回答是：这是一个很好的问题。", prompt)
        .chars()
        .map(|c| c.to_string())
        .collect();

    // 把 Vec 转成 Stream，每个 token 之间加延迟
    let token_count = tokens.len();

    let stream = stream::iter(tokens.into_iter().enumerate())
        .then(move |(i, token)| async move {
            // 模拟 LLM 生成每个 token 的延迟（50-150ms）
            let delay = 50 + (i as u64 * 7) % 100;
            tokio::time::sleep(Duration::from_millis(delay)).await;

            Ok(StreamChunk {
                content: token,
                is_done: i == token_count - 1,
            })
        })
        .boxed(); // .boxed() 把具体类型转成 BoxStream

    stream
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ 模拟 LLM 流式输出 ═══\n");

    // ─── 场景1：基本流式消费 ───
    println!("--- 场景1：逐 token 打印（模拟 ChatGPT 打字效果） ---\n");
    print!("🤖 ");

    let mut stream = mock_llm_stream("Rust 异步编程");
    let mut total_tokens = 0;

    while let Some(chunk) = stream.next().await {
        match chunk {
            Ok(c) => {
                print!("{}", c.content);  // 不换行，逐字输出
                total_tokens += 1;
                if c.is_done {
                    println!("\n");
                    println!("  📊 共 {} 个 token", total_tokens);
                }
            }
            Err(e) => {
                println!("\n  ❌ 流式错误: {}", e);
                break;
            }
        }
    }

    // ─── 场景2：收集完整响应 ───
    println!("\n--- 场景2：收集完整响应（等全部 token 输出完） ---\n");
    let mut stream = mock_llm_stream("Tokio 运行时");
    let mut full_response = String::new();

    while let Some(chunk) = stream.next().await {
        if let Ok(c) = chunk {
            full_response.push_str(&c.content);
        }
    }
    println!("  完整响应: {}", full_response);

    // ─── 场景3：带超时的流式消费 ───
    println!("\n--- 场景3：带超时的流式消费 ---\n");
    consume_stream_with_timeout("超时测试").await;

    Ok(())
}

/// 带超时保护的流式消费
async fn consume_stream_with_timeout(prompt: &str) {
    use tokio_stream::StreamExt as TokioStreamExt;

    let stream = mock_llm_stream(prompt);

    // 给整个流设置超时：如果 500ms 内没有新 chunk，认为超时
    let mut stream = tokio_stream::StreamExt::timeout(
        stream,
        Duration::from_millis(500),
    );

    print!("🤖 ");
    let mut count = 0;

    while let Some(item) = TokioStreamExt::next(&mut stream).await {
        match item {
            Ok(Ok(chunk)) => {
                // 正常收到 chunk
                print!("{}", chunk.content);
                count += 1;
            }
            Ok(Err(e)) => {
                // Stream 内部错误
                println!("\n  ❌ Stream 错误: {}", e);
                break;
            }
            Err(_elapsed) => {
                // 超时：500ms 没有新数据
                println!("\n  ⏰ 流式超时！已收到 {} 个 chunk", count);
                break;
            }
        }
    }
    println!();
}
```

## 5. 用 async-stream 创建自定义 Stream

`async-stream` crate 提供了 `stream!` 宏，让创建 Stream 像写 async fn 一样自然。

```rust
use async_stream::stream;
use futures::stream::BoxStream;
use futures::StreamExt;
use std::time::Duration;
use anyhow::Result;

/// 用 async-stream 创建 LLM 流式响应
/// 比手动构建 Stream 更直观
fn create_llm_stream(prompt: &str) -> BoxStream<'static, Result<String>> {
    let prompt = prompt.to_string();

    let s = stream! {
        // 像写普通 async 函数一样！
        let words = vec!["这是", "一个", "流式", "响应", "的", "示例", "。"];

        for (i, word) in words.iter().enumerate() {
            // 模拟生成延迟
            tokio::time::sleep(Duration::from_millis(100)).await;

            // yield 发送一个值给消费者
            // 对应 JS 的 generator: yield* value
            yield Ok(format!("{}", word));

            // 模拟第 4 个 word 有时出错
            if i == 3 && prompt.contains("错误") {
                yield Err(anyhow::anyhow!("模拟的流式错误"));
                return; // 终止 stream
            }
        }

        // stream! 结束后自动发送 None（stream 结束信号）
    };

    Box::pin(s) // 转成 BoxStream
}

/// 用 async-stream 创建带心跳的事件流（SSE 风格）
fn create_sse_stream() -> BoxStream<'static, String> {
    let s = stream! {
        let mut event_id = 0;

        loop {
            event_id += 1;

            if event_id <= 5 {
                // 前 5 个是数据事件
                yield format!("event: message\ndata: 第 {} 条消息\nid: {}\n",
                    event_id, event_id);
            } else if event_id <= 7 {
                // 第 6-7 个是心跳
                yield format!("event: heartbeat\ndata: ping\nid: {}\n",
                    event_id);
            } else {
                // 第 8 个后结束
                yield "event: done\ndata: stream_complete\n".to_string();
                return;
            }

            tokio::time::sleep(Duration::from_millis(200)).await;
        }
    };

    Box::pin(s)
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ async-stream 创建自定义 Stream ═══\n");

    // ─── 场景1：正常的 LLM 流式响应 ───
    println!("--- 场景1：正常流式响应 ---\n");
    print!("🤖 ");
    let mut stream = create_llm_stream("正常请求");
    while let Some(chunk) = stream.next().await {
        match chunk {
            Ok(text) => print!("{}", text),
            Err(e) => {
                println!("\n  ❌ 错误: {}", e);
                break;
            }
        }
    }
    println!("\n");

    // ─── 场景2：带错误的流式响应 ───
    println!("--- 场景2：带错误的流式响应 ---\n");
    print!("🤖 ");
    let mut stream = create_llm_stream("触发错误测试");
    while let Some(chunk) = stream.next().await {
        match chunk {
            Ok(text) => print!("{}", text),
            Err(e) => {
                println!("\n  ❌ 流中断: {}", e);
                break;
            }
        }
    }
    println!();

    // ─── 场景3：SSE 风格事件流 ───
    println!("\n--- 场景3：SSE 风格事件流 ---\n");
    let mut stream = create_sse_stream();
    while let Some(event) = stream.next().await {
        for line in event.lines() {
            println!("  {}", line);
        }
        println!("  ---");
    }

    Ok(())
}
```

### stream! 宏 vs 手动构建

```rust
// ─── 手动构建（复杂，不直观） ───
fn manual_stream() -> BoxStream<'static, i32> {
    let items = vec![1, 2, 3];
    futures::stream::iter(items.into_iter())
        .then(|x| async move {
            tokio::time::sleep(Duration::from_millis(100)).await;
            x * 2
        })
        .boxed()
}

// ─── async-stream 宏（直观，像写 async fn） ───
fn macro_stream() -> BoxStream<'static, i32> {
    let s = stream! {
        for x in vec![1, 2, 3] {
            tokio::time::sleep(Duration::from_millis(100)).await;
            yield x * 2;
        }
    };
    Box::pin(s)
}

// 两者功能完全相同，但 stream! 宏可读性好得多
// 特别是逻辑复杂时（条件 yield、循环 yield、错误处理）
```

## 6. 流式处理高级模式

### 模式1：map/filter/take 流式管道

```rust
use futures::StreamExt;
use tokio_stream::StreamExt as TokioStreamExt;

async fn stream_pipeline() {
    let stream = tokio_stream::iter(1..=20)
        .filter(|x| *x % 2 == 0)      // 只保留偶数
        .map(|x| x * x)               // 平方
        .take(5);                      // 只取前 5 个

    let results: Vec<i32> = stream.collect().await;
    println!("结果: {:?}", results);
    // 输出: [4, 16, 36, 64, 100]
}
```

### 模式2：合并多个 Stream

```rust
use tokio_stream::StreamExt;

async fn merge_streams() {
    let stream1 = tokio_stream::iter(vec!["A1", "A2", "A3"]);
    let stream2 = tokio_stream::iter(vec!["B1", "B2", "B3"]);

    // merge：两个流交替输出
    let mut merged = stream1.merge(stream2);

    while let Some(value) = merged.next().await {
        println!("  收到: {}", value);
    }
    // 输出顺序不确定（取决于哪个先准备好）
}
```

### 模式3：Stream 转 Iterator 和反向

```rust
// Vec → Stream
let stream = tokio_stream::iter(vec![1, 2, 3]);

// Stream → Vec（需要 .await）
let vec: Vec<i32> = stream.collect().await;

// Channel receiver → Stream
let (tx, rx) = tokio::sync::mpsc::channel::<String>(32);
let stream = tokio_stream::wrappers::ReceiverStream::new(rx);
// 现在可以用 StreamExt 的方法处理 mpsc 消息了！
```

### Rust vs TypeScript 流式处理关键差异

```
┌──────────────────┬─────────────────────────┬──────────────────────────┐
│ 维度              │ Rust (tokio-stream)      │ TypeScript               │
├──────────────────┼─────────────────────────┼──────────────────────────┤
│ 创建流            │ stream! { yield x; }    │ async function* () {}    │
│ 消费流            │ while let Some(x) = ... │ for await (const x of s) │
│ 类型擦除          │ BoxStream (Pin+Box+dyn) │ 不需要（动态类型）        │
│ 背压(backpressure)│ .await 自然背压         │ ReadableStream 内置       │
│ 取消              │ drop stream 即取消      │ reader.cancel() / abort  │
│ 超时              │ stream.timeout(dur)     │ AbortSignal.timeout()    │
│ 错误处理          │ Stream<Item=Result<T>>  │ try/catch in for-await   │
│ 线程安全          │ Send + 'static          │ 不需要（单线程）          │
│ 内存管理          │ 编译时确定              │ GC 管理                  │
└──────────────────┴─────────────────────────┴──────────────────────────┘
```

**最大区别**：Rust 的 Stream 有"背压"——消费者不调 `next().await`，生产者就不会继续产生数据。JS 的 ReadableStream 也有类似机制，但 EventSource/WebSocket 默认没有背压，数据会堆积在内存中。

## 8. 完整示例：模拟 ZeroClaw 流式聊天

把前面学到的所有内容组合成一个完整的、接近 ZeroClaw 风格的示例。

```rust
use async_stream::stream;
use futures::stream::BoxStream;
use futures::StreamExt;
use std::time::Duration;
use anyhow::Result;

// ─── 类型定义（简化版 ZeroClaw） ───

#[derive(Debug, Clone)]
enum StreamChunk {
    Text(String),
    ToolCall { name: String, args: String },
    Done,
    Error(String),
}

// ─── Provider trait（简化版） ───

trait StreamProvider: Send + Sync {
    fn stream_chat(&self, message: &str) -> BoxStream<'static, Result<StreamChunk>>;
}

// ─── 模拟 Provider 实现 ───

struct MockLLM;

impl StreamProvider for MockLLM {
    fn stream_chat(&self, message: &str) -> BoxStream<'static, Result<StreamChunk>> {
        let message = message.to_string();

        let s = stream! {
            // 模拟 LLM 思考
            tokio::time::sleep(Duration::from_millis(200)).await;

            if message.contains("天气") {
                // 需要调用工具
                yield Ok(StreamChunk::Text("让我查一下".to_string()));
                tokio::time::sleep(Duration::from_millis(100)).await;

                yield Ok(StreamChunk::ToolCall {
                    name: "weather".to_string(),
                    args: "{\"city\": \"北京\"}".to_string(),
                });
                tokio::time::sleep(Duration::from_millis(100)).await;

                yield Ok(StreamChunk::Text("天气晴朗，25°C".to_string()));
            } else {
                // 普通回复，逐 token 输出
                let reply = format!("关于「{}」：这是一个好问题！", message);
                for ch in reply.chars() {
                    tokio::time::sleep(Duration::from_millis(50)).await;
                    yield Ok(StreamChunk::Text(ch.to_string()));
                }
            }

            yield Ok(StreamChunk::Done);
        };

        Box::pin(s)
    }
}

// ─── 流式响应消费者 ───

async fn handle_stream_response(
    provider: &dyn StreamProvider,
    message: &str,
) -> Result<String> {
    println!("👤 用户: {}", message);
    print!("🤖 AI: ");

    let mut stream = provider.stream_chat(message);
    let mut full_response = String::new();

    // 带超时消费：每个 chunk 最多等 2 秒
    while let Some(chunk) = stream.next().await {
        match chunk {
            Ok(StreamChunk::Text(text)) => {
                print!("{}", text);
                full_response.push_str(&text);
            }
            Ok(StreamChunk::ToolCall { name, args }) => {
                println!("\n  🔧 [调用工具: {} ({})]", name, args);
                print!("🤖 AI: ");
            }
            Ok(StreamChunk::Done) => {
                println!("\n  ✅ 流式响应完成");
                break;
            }
            Ok(StreamChunk::Error(e)) => {
                println!("\n  ❌ 流式错误: {}", e);
                break;
            }
            Err(e) => {
                println!("\n  💥 传输错误: {}", e);
                break;
            }
        }
    }

    Ok(full_response)
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ 完整示例：模拟 ZeroClaw 流式聊天 ═══\n");

    let provider = MockLLM;

    // 普通问题
    let response = handle_stream_response(&provider, "Rust 怎么学").await?;
    println!("  📋 完整响应: {}\n", response);

    // 需要工具调用的问题
    let response = handle_stream_response(&provider, "今天天气怎么样").await?;
    println!("  📋 完整响应: {}\n", response);

    Ok(())
}
```

## 9. 速查表

```
创建 Stream：
  tokio_stream::iter(vec)          ← 从 Vec/Iterator
  stream! { yield x; }            ← 用 async-stream 宏（最灵活）
  ReceiverStream::new(rx)         ← 从 mpsc::Receiver

消费 Stream：
  while let Some(x) = s.next().await { ... }     ← 逐个消费
  let vec: Vec<T> = s.collect().await;            ← 收集全部

类型转换：
  stream.boxed()                  ← 具体类型 → BoxStream
  Box::pin(stream)                ← 同上

流式管道：
  stream.map(|x| ...)             ← 映射
  stream.filter(|x| ...)          ← 过滤
  stream.take(n)                  ← 取前 N 个
  stream.timeout(dur)             ← 超时
  stream.throttle(dur)            ← 限速
  stream.merge(other)             ← 合并

ZeroClaw 模式：
  BoxStream<'static, Result<T>>   ← Provider 的流式返回类型
  stream! { yield Ok(chunk); }    ← 创建流
  while let Some(Ok(chunk)) = ... ← 消费流
```

### 一句话总结

> **Stream 就是异步的 Iterator，BoxStream 就是装箱后的统一类型**——ZeroClaw 用 `BoxStream<'static, Result<StreamChunk>>` 实现 LLM 的逐 token 流式输出，本质上和前端用 `ReadableStream` / `EventSource` 处理 SSE 是同一个模式：生产者异步 yield，消费者 await 消费。掌握 `stream!` 宏 + `StreamExt` 的 map/filter/timeout，你就能处理任何流式场景。
