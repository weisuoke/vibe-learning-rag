## 知识点概述

### 什么是边？

在 LangGraph 中，**边（Edge）是连接节点的有向路径，控制图的执行流程**。

**核心作用**：
- 定义节点之间的执行顺序
- 控制状态在节点间的流转
- 实现条件分支和动态路由

**边的类型**：
1. **普通边（Normal Edge）**：固定路由，直接连接两个节点
2. **条件边（Conditional Edge）**：动态路由，基于状态选择目标节点

### 为什么需要边？

**问题场景**：
- 如何定义节点的执行顺序？
- 如何实现条件分支逻辑？
- 如何根据状态动态选择执行路径？

**解决方案**：
- **普通边**：解决固定流程的连接问题
- **条件边**：解决动态决策的路由问题

## 常见问题

### Q1: 什么时候用普通边，什么时候用条件边？

**简单规则**：
- **固定流程** → 使用普通边
- **需要决策** → 使用条件边

**示例**：
```python
# 固定流程：使用普通边
graph.add_edge(START, "step1")
graph.add_edge("step1", "step2")
graph.add_edge("step2", END)

# 需要决策：使用条件边
graph.add_conditional_edges(
    "check",
    route_func,  # 根据状态决定下一步
    {"pass": "next_step", "fail": "retry"}
)
```

### Q2: 路由函数应该如何设计？

**核心原则**：
1. **清晰简洁**：逻辑清晰，易于理解
2. **类型提示**：使用 `Literal` 明确返回值
3. **错误处理**：处理边界情况
4. **性能优化**：避免耗时操作

**示例**：
```python
from typing import Literal

def route_func(state: State) -> Literal["path_a", "path_b"]:
    """清晰的路由函数"""
    # 简单的条件判断
    if state["score"] > 0.5:
        return "path_a"
    return "path_b"
```

### Q3: 如何调试条件路由？

**调试技巧**：
1. **添加日志**：在路由函数中打印决策信息
2. **可视化图**：使用 `get_graph()` 查看图结构
3. **单步测试**：单独测试路由函数
4. **状态检查**：检查路由函数接收的状态

**示例**：
```python
def route_func(state: State) -> str:
    score = state["score"]
    print(f"[DEBUG] Routing decision: score={score}")

    if score > 0.5:
        print("→ Taking path_a")
        return "path_a"
    else:
        print("→ Taking path_b")
        return "path_b"
```
