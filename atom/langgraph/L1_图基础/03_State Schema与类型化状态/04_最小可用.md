## 核心知识清单

掌握以下5个核心知识点，就能开始使用LangGraph的State Schema：

### 1. TypedDict定义状态

```python
from typing import TypedDict

class State(TypedDict):
    messages: list
    user_id: str
    count: int
```

**核心要点**：
- 使用TypedDict定义状态结构
- 每个字段指定类型
- 这是最基础的状态定义方式

**能做什么**：
- 定义图的数据结构
- 获得类型检查支持
- 清晰的数据契约

### 2. Annotated添加Reducer

```python
from typing import TypedDict, Annotated
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]  # 列表追加
    count: Annotated[int, operator.add]      # 数值累加
    user_id: str                              # 覆盖更新
```

**核心要点**：
- `Annotated[类型, reducer]`指定聚合规则
- `operator.add`用于追加/累加
- 无Annotated的字段默认覆盖

**能做什么**：
- 自动聚合并行节点的更新
- 累积消息历史
- 计数器累加

### 3. 创建和使用图

```python
from langgraph.graph import StateGraph

# 创建图
graph = StateGraph(State)

# 添加节点
def my_node(state: State) -> dict:
    return {"messages": ["new message"]}

graph.add_node("my_node", my_node)

# 设置入口和出口
graph.set_entry_point("my_node")
graph.set_finish_point("my_node")

# 编译
app = graph.compile()

# 运行
result = app.invoke({"messages": [], "user_id": "user123", "count": 0})
```

**核心要点**：
- `StateGraph(State)`创建图
- 节点返回部分状态更新
- `compile()`编译成可执行图
- `invoke()`运行图

**能做什么**：
- 构建完整的状态化图
- 执行节点逻辑
- 获取最终状态

### 4. 状态持久化

```python
from langgraph.checkpoint.memory import MemorySaver

# 创建检查点
checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)

# 多轮对话
config = {"configurable": {"thread_id": "conversation1"}}

# 第一轮
app.invoke({"messages": ["hello"]}, config)

# 第二轮 - 保留历史
app.invoke({"messages": ["world"]}, config)
```

**核心要点**：
- `MemorySaver()`创建内存检查点
- `thread_id`标识会话
- 状态在多次调用间保留

**能做什么**：
- 实现多轮对话
- 保存执行状态
- 中断和恢复

### 5. 多Schema架构

```python
class OverallState(TypedDict):
    query: str
    user_id: str
    internal_data: dict  # 内部字段
    answer: str

class InputState(TypedDict):
    query: str
    user_id: str

class OutputState(TypedDict):
    answer: str

graph = StateGraph(
    state_schema=OverallState,
    input_schema=InputState,
    output_schema=OutputState
)
```

**核心要点**：
- `state_schema`：内部完整状态
- `input_schema`：用户输入接口
- `output_schema`：用户输出接口

**能做什么**：
- 隐藏内部实现细节
- 清晰的API接口
- 灵活的状态管理

## 快速参考

### 常用Reducer

```python
import operator

# 列表追加
messages: Annotated[list, operator.add]

# 字典合并
metadata: Annotated[dict, operator.or_]

# 数值累加
count: Annotated[int, operator.add]

# 自定义Reducer
def merge_unique(existing: list, new: list) -> list:
    seen = set(existing)
    return existing + [x for x in new if x not in seen]

items: Annotated[list, merge_unique]
```

### 节点模式

```python
# 基础节点
def node(state: State) -> dict:
    return {"field": "value"}

# 条件更新
def node(state: State) -> dict:
    if condition:
        return {"field": "value"}
    return {}

# 多字段更新
def node(state: State) -> dict:
    return {
        "field1": "value1",
        "field2": "value2"
    }
```

### Checkpointer选择

```python
# 开发环境 - 内存
from langgraph.checkpoint.memory import MemorySaver
```

## 完整示例

```python
from typing import TypedDict, Annotated
import operator
from langgraph.graph import StateGraph
from langgraph.checkpoint.memory import MemorySaver

# 1. 定义状态
class ChatState(TypedDict):
    messages: Annotated[list, operator.add]
    user_id: str

# 2. 创建图
graph = StateGraph(ChatState)

# 3. 添加节点
def chat_node(state: ChatState) -> dict:
    messages = state["messages"]
    # 简单回复逻辑
    last_message = messages[-1] if messages else ""
    response = f"Echo: {last_message}"
    return {"messages": [response]}

graph.add_node("chat", chat_node)

# 4. 设置入口和出口
graph.set_entry_point("chat")
graph.set_finish_point("chat")

# 5. 编译（带持久化）
checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)

# 6. 使用
config = {"configurable": {"thread_id": "user123"}}

# 第一轮
result1 = app.invoke({"messages": ["Hello"], "user_id": "user123"}, config)
print(result1["messages"])  # ["Hello", "Echo: Hello"]

# 第二轮
result2 = app.invoke({"messages": ["How are you?"], "user_id": "user123"}, config)
print(result2["messages"])  # ["Hello", "Echo: Hello", "How are you?", "Echo: How are you?"]
```

## 常见问题

**Q: TypedDict vs Pydantic，用哪个？**
A: 优先TypedDict，除非需要验证或默认值。

**Q: Reducer什么时候执行？**
A: 节点返回后、并行执行后、手动更新时。

**Q: 如何调试状态问题？**
A: 在节点中打印状态，使用Checkpointer查看历史。

**Q: 状态太大怎么办？**
A: 只保留必要字段，使用Reducer控制增长。

**Q: 如何实现多轮对话？**
A: 使用Checkpointer + thread_id。
