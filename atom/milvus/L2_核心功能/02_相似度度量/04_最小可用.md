## 必须掌握的4个核心知识

### 4.1 理解三大基础度量

#### L2距离（欧几里得距离）

**一句话：** 测量向量之间的直线距离

```python
import numpy as np

def l2_distance(a, b):
    """L2距离：两点间的直线距离"""
    return np.sqrt(np.sum((a - b) ** 2))

# 示例
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

distance = l2_distance(vec_a, vec_b)
print(f"L2距离: {distance:.4f}")  # 1.7321
```

**何时使用：**
- 向量长度有意义（图像特征、坐标数据）
- 需要考虑绝对位置差异

---

#### 余弦相似度（COSINE）

**一句话：** 测量向量之间的角度，忽略长度

```python
def cosine_similarity(a, b):
    """余弦相似度：向量夹角的余弦值"""
    dot_product = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)

# 示例
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 4.0, 6.0])  # vec_a 的2倍

similarity = cosine_similarity(vec_a, vec_b)
print(f"余弦相似度: {similarity:.4f}")  # 1.0000（完全同向）
```

**何时使用：**
- 文本语义检索（Embedding模型输出通常归一化）
- 只关注方向，不关注长度

**⭐ 推荐：** 80%的场景使用COSINE

---

#### 内积（IP）

**一句话：** 测量向量的内积，考虑长度和方向

```python
def inner_product(a, b):
    """内积：向量点积"""
    return np.dot(a, b)

# 示例
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

ip = inner_product(vec_a, vec_b)
print(f"内积: {ip:.4f}")  # 20.0000
```

**何时使用：**
- 推荐系统（向量长度代表偏好强度）
- Embedding模型输出已归一化（此时IP ≈ COSINE）

---

### 4.2 掌握度量选型原则

**决策树（3步选型法）：**

```
步骤1：检查数据类型
├─ 稠密向量（float） → 继续步骤2
├─ 二值向量（binary） → HAMMING
└─ 文本关键词 → BM25

步骤2：检查Embedding模型
├─ 输出已归一化 → COSINE（推荐）
└─ 输出未归一化 → 继续步骤3

步骤3：检查业务场景
├─ 语义检索（文档问答） → COSINE
├─ 推荐系统（偏好强度） → IP
└─ 图像检索（绝对位置） → L2
```

**快速记忆：**
- **不确定用什么？→ 用COSINE**
- **文本检索？→ 用COSINE**
- **推荐系统？→ 用IP**

```python
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=128)

schema = CollectionSchema(fields, description="相似度检索示例")

# 创建Collection（指定度量方式）
collection = Collection(name="demo_collection", schema=schema)

# 创建索引（指定度量方式）
index_params = {
    "index_type": "HNSW",
    "metric_type": "COSINE",  # ← 关键：指定度量方式
    "params": {"M": 16, "efConstruction": 200}

```

### 4.4 理解归一化的影响

**关键认知：** 归一化后，COSINE ≈ IP

```python
# 未归一化向量
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

cosine = cosine_similarity(vec_a, vec_b)
ip = inner_product(vec_a, vec_b)
print(f"未归一化 - COSINE: {cosine:.4f}, IP: {ip:.4f}")
# COSINE: 0.9926, IP: 20.0000（差异很大）

# 归一化向量
vec_a_norm = vec_a / np.linalg.norm(vec_a)
vec_b_norm = vec_b / np.linalg.norm(vec_b)

cosine_norm = cosine_similarity(vec_a_norm, vec_b_norm)
ip_norm = inner_product(vec_a_norm, vec_b_norm)
print(f"归一化后 - COSINE: {cosine_norm:.4f}, IP: {ip_norm:.4f}")
# COSINE: 0.9926, IP: 0.9926（几乎相同）
```

**实践建议：**
- 如果Embedding模型输出已归一化 → 用COSINE（语义更直观）
- 如果需要考虑向量长度 → 用IP

## 快速实战：完整示例

```python
"""
最小可用示例：文档问答系统
演示：使用COSINE度量进行语义检索
"""

import numpy as np
from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection

# ===== 1. 连接Milvus =====
connections.connect("default", host="localhost", port="19530")

# ===== 2. 创建Collection（使用COSINE度量）=====
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=500),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=128)
]
schema = CollectionSchema(fields, description="文档问答")
collection = Collection(name="qa_docs", schema=schema)

# ===== 3. 创建索引（COSINE度量）=====
index_params = {
    "index_type": "HNSW",
    "metric_type": "COSINE",  # 文本检索推荐COSINE
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="embedding", index_params=index_params)

# ===== 4. 插入数据 =====
# 模拟Embedding（实际应使用Embedding模型）
docs = [
    "Milvus是一个开源向量数据库",
    "向量检索用于语义搜索",
    "COSINE度量适合文本检索"
]
embeddings = [np.random.rand(128).tolist() for _ in range(3)]

data = [
    docs,
    embeddings
]
collection.insert(data)
collection.load()

# ===== 5. 相似度检索 =====
query_embedding = [np.random.rand(128).tolist()]
search_params = {"metric_type": "COSINE", "params": {"ef": 100}}

results = collection.search(
    data=query_embedding,
    anns_field="embedding",
    param=search_params,
    limit=3,
    output_fields=["text"]
)

# ===== 6. 输出结果 =====
print("=== 检索结果 ===")
for hits in results:
    for hit in hits:
        print(f"文档: {hit.entity.get('text')}")
        print(f"相似度: {hit.distance:.4f}")
        print("---")

# 清理
collection.drop()
connections.disconnect("default")
```

**运行输出示例：**
```
=== 检索结果 ===
文档: COSINE度量适合文本检索
相似度: 0.8523
---
文档: 向量检索用于语义搜索
相似度: 0.7891
---
文档: Milvus是一个开源向量数据库
相似度: 0.7234
---
```
