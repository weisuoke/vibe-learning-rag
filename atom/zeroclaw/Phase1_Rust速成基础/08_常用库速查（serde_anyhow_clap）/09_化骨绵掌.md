## 卡片 1：Rust 的"npm" —— Cargo.toml 依赖声明

### 🎯 核心点

**在 Cargo.toml 的 `[dependencies]` 中添加库，cargo 自动下载和编译。**

```toml
[dependencies]
serde = { version = "1.0", features = ["derive"] }
serde_json = "1.0"
toml = "1.0"
anyhow = "1.0"
clap = { version = "4.5", features = ["derive"] }
tracing = "0.1"
tracing-subscriber = { version = "0.3", features = ["env-filter"] }
```

**前端对照：** = `package.json` 的 `dependencies` + `npm install`，但 `cargo build` 一步到位（下载+编译）。`features = ["derive"]` 是 Rust 特有的——不开启就无法使用 `#[derive(Serialize, Parser)]` 等宏。

**ZeroClaw：** Cargo.toml 有 30+ 依赖，这四个库全部在列。

## 卡片 2：derive 宏 —— 一行代码自动实现功能

### 🎯 核心点

**`#[derive(X)]` 让编译器在编译时自动为你的类型生成实现代码——零运行时开销。**

```rust
#[derive(
    Debug,                  // 自动生成 {:?} 打印格式
    Clone,                  // 自动生成 .clone() 方法
    Serialize, Deserialize, // serde: 自动序列化/反序列化
    Parser,                 // clap: 自动 CLI 解析
)]
struct MyConfig {
    name: String,
    verbose: bool,
}
```

**前端对照：** TypeScript 装饰器 `@Serializable` 是运行时通过反射实现的；Rust derive 是编译时代码生成，编译器直接把实现代码"写"出来。

| 特性 | TS 装饰器 | Rust derive |
|------|----------|-------------|
| 执行时机 | 运行时 | 编译时 |
| 性能开销 | 有（反射） | 零 |
| 错误发现 | 运行时 | 编译时 |

**ZeroClaw：** 几乎每个 struct/enum 上方都有 `#[derive(...)]`，这是 Rust 生态的标准写法。

## 卡片 3：serde —— 一个 struct 通吃所有格式

### 🎯 核心点

**derive `Serialize`/`Deserialize`，struct 就能自动转换 JSON/TOML/YAML。**

```rust
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, Debug)]
struct User { name: String, age: u32 }

fn main() -> anyhow::Result<()> {
    let user = User { name: "Alice".into(), age: 30 };

    // struct → JSON
    let json = serde_json::to_string(&user)?;    // {"name":"Alice","age":30}

    // JSON → struct
    let parsed: User = serde_json::from_str(&json)?;

    // 同一个 struct，换个函数就能转 TOML
    let toml_str = toml::to_string(&user)?;       // name = "Alice"\nage = 30
    Ok(())
}
```

**前端对照：** TS 需要 `JSON.parse()` + zod 校验两步；Rust serde `from_str` 一步搞定解析+类型校验。

**ZeroClaw：** `config/schema.rs` 的 `Config` struct 用 serde 从 TOML 反序列化——20+ 字段全靠 serde 自动处理。

## 卡片 4：serde 属性 —— 声明式配置序列化行为

### 🎯 核心点

**用 `#[serde(...)]` 属性控制字段如何序列化。** 四个最常用：

```rust
#[derive(Serialize, Deserialize, Debug)]
struct ServerConfig {
    #[serde(default)]                    // 缺失时用默认值（String→"", u16→0）
    host: String,

    #[serde(default = "default_port")]   // 指定默认值函数
    port: u16,

    #[serde(skip)]                       // 不参与序列化/反序列化
    internal_secret: String,

    #[serde(rename = "server_name")]     // JSON 中叫 server_name，Rust 中叫 name
    name: String,

    #[serde(alias = "addr")]             // 反序列化时也接受 "addr" 这个名字
    address: String,
}
fn default_port() -> u16 { 8080 }
```

**前端对照：** `default` ≈ `z.string().default("")`；`skip` ≈ 手动删字段；`rename` ≈ `.transform()`；`alias` 无直接对应。

**ZeroClaw：** `Config` 大量使用 `#[serde(default)]` 省略可选项、`#[serde(alias)]` 兼容旧配置名、`#[serde(skip)]` 隐藏内部状态。

## 卡片 5：anyhow::Result —— 万能错误容器

### 🎯 核心点

**函数返回 `anyhow::Result<T>` 配合 `?`，任何错误自动传播，不需要手动定义错误类型。**

```rust
use anyhow::Result;
use std::fs;

fn load_config(path: &str) -> Result<String> {
    let content = fs::read_to_string(path)?;  // io::Error → anyhow::Error
    let _config: toml::Value = toml::from_str(&content)?;  // toml Error → anyhow::Error
    Ok(content)
}

fn main() -> Result<()> {
    let config = load_config("config.toml")?;
    println!("Loaded: {} bytes", config.len());
    Ok(())
}
```

**关键理解：** `anyhow::Result<T>` = `Result<T, anyhow::Error>`。`anyhow::Error` 是"万能容器"——任何实现了 `std::error::Error` 的错误都能自动放进去，不需要定义 `enum MyError { Io(...), Parse(...) }`。

**前端对照：** TS 的 `catch(e)` 中 `e` 是 `unknown`；anyhow 统一为 `anyhow::Error` 但保留原始类型信息。

## 卡片 6：.context() —— 给错误加"面包屑"

### 🎯 核心点

**`.context("描述")` 在错误链上添加一层上下文——出错时看到完整的"面包屑路径"。**

```rust
use anyhow::{Context, Result};

fn load_user_config(user_id: u64) -> Result<String> {
    let path = format!("/etc/app/users/{user_id}/config.toml");
    let content = std::fs::read_to_string(&path)
        .context("failed to read config file")?;        // 第一层
    let _: toml::Value = toml::from_str(&content)
        .context("failed to parse TOML")?;               // 第二层
    Ok(content)
}

fn init_system() -> Result<()> {
    load_user_config(42)
        .context("failed to initialize user 42")?;      // 第三层
    Ok(())
}

// 错误输出（从外到内）：
// Error: failed to initialize user 42
// Caused by:
//   0: failed to read config file
//   1: No such file or directory (os error 2)
```

**前端对照：** TS 用 `new Error("msg", { cause: prevError })` 手动嵌套；Rust `.context()` 一行搞定，自动链式。

**ZeroClaw：** 几乎每个 `?` 后都跟 `.context()`，错误链如 `"tool execution failed" → "failed to call shell" → "permission denied"`。

## 卡片 7：clap Parser —— 类型安全的 CLI 解析

### 🎯 核心点

**derive `Parser`，struct 字段自动变成 CLI 参数——帮助信息、错误提示全自动生成。**

```rust
use clap::Parser;

/// 一个简单的文件处理工具          ← 自动成为 --help 描述
#[derive(Parser, Debug)]
#[command(version, about)]
struct Args {
    /// 输入文件路径                 ← 自动成为参数帮助信息
    #[arg(short, long)]
    input: String,

    /// 是否显示详细输出
    #[arg(short, long, default_value_t = false)]
    verbose: bool,

    /// 处理的最大行数
    #[arg(short, long, default_value_t = 100)]
    max_lines: usize,
}

fn main() {
    let args = Args::parse();
    println!("Input: {}, Verbose: {}", args.input, args.verbose);
}
// $ mytool -i data.csv -v -m 50
// $ mytool --help   ← 自动生成帮助信息
```

**前端对照：** yargs 用 `.option('input', { type: 'string' })` 链式调用，类型运行时才知道；clap 编译时就确定。

## 卡片 8：clap Subcommand —— 命令行子命令

### 🎯 核心点

**derive `Subcommand`，enum 变体就是子命令——用 `match` 分发处理。**

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "mytool")]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// 初始化新项目
    Init { #[arg(short, long)] name: String },
    /// 构建项目
    Build { #[arg(long)] release: bool },
    /// 运行测试
    Test,
}

fn main() -> anyhow::Result<()> {
    let cli = Cli::parse();
    match cli.command {
        Commands::Init { name } => println!("Init: {name}"),
        Commands::Build { release } => println!("Build release={release}"),
        Commands::Test => println!("Testing..."),
    }
    Ok(())
}
// $ mytool init --name my_project
// $ mytool build --release
```

**前端对照：** yargs 用 `.command('init', ...)` + 回调函数；clap 用 enum + match，编译时保证所有子命令都被处理。

**ZeroClaw：** `Commands` enum 有 30+ 变体，`main.rs` 一个 `match` 分发到对应处理函数。

## 卡片 9：tracing —— 比 println! 更强的日志

### 🎯 核心点

**`info!/warn!/error!` 宏替代 `println!`，`RUST_LOG` 环境变量控制输出级别。**

```rust
use tracing::{info, warn, error, debug};

fn main() -> anyhow::Result<()> {
    // 初始化（整个程序只需一次）
    tracing_subscriber::fmt()
        .with_env_filter(tracing_subscriber::EnvFilter::from_default_env())
        .init();

    info!("Server started on port {}", 8080);
    debug!("Loading config...");           // RUST_LOG=debug 才输出
    warn!("Rate limit approaching: {}/100", 95);
    error!("Database connection failed");
    info!(port = 8080, host = "0.0.0.0", "Server started");  // 结构化字段
    Ok(())
}
```

```bash
$ RUST_LOG=info cargo run               # 只看 info 及以上
$ RUST_LOG=debug cargo run              # 看 debug 及以上
$ RUST_LOG=mytool=debug cargo run       # 只看 mytool 模块的 debug
```

**前端对照：** `info!` ≈ `console.log`，`warn!` ≈ `console.warn`，`error!` ≈ `console.error`。但 tracing 能通过环境变量动态控制，不需要改代码。

**ZeroClaw：** 全项目 tracing 替代 `println!`，`RUST_LOG=zeroclaw=debug` 看详细信息，生产环境 `RUST_LOG=info`。

## 卡片 10：四库协作 —— ZeroClaw 的基础模式

### 🎯 核心点

**clap 接收输入 → serde 解析配置 → anyhow 处理错误 → tracing 记录日志。**

```rust
use anyhow::{Context, Result};
use clap::Parser;
use serde::Deserialize;
use tracing::info;

#[derive(Deserialize)]
struct AppConfig {
    #[serde(default = "default_port")]
    port: u16,
    database_url: String,
}
fn default_port() -> u16 { 3000 }

#[derive(Parser)]
#[command(name = "myapp", version, about = "A demo app")]
struct Args {
    #[arg(short, long, default_value = "config.toml")]
    config: String,
}

fn main() -> Result<()> {
    let args = Args::parse();                                   // 1. clap

    tracing_subscriber::fmt()                                   // 2. tracing
        .with_env_filter("info")
        .init();
    info!("Starting myapp");

    let content = std::fs::read_to_string(&args.config)
        .context("failed to read config file")?;                // 3. anyhow

    let config: AppConfig = toml::from_str(&content)
        .context("failed to parse config")?;                    // 4. serde + anyhow

    info!(port = config.port, "Config loaded");
    Ok(())
}
```

```
clap 解析参数 → serde 解析配置 → anyhow 传播错误 → tracing 全程记录
```

### ZeroClaw 关联

这就是 `src/main.rs` 的核心流程：`Cli::parse()` → `tracing_subscriber` 初始化 → `toml::from_str()` 解析配置 → 所有函数返回 `anyhow::Result`。**学完这四个库，你就能读懂 ZeroClaw 的整个启动流程。**
