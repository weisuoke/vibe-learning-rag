# 核心概念：类型推断

## 概述

LangGraph 的类型推断机制允许框架自动从函数签名中提取输入类型，减少显式类型声明的需要，提高开发效率。

**核心功能**：
- 从函数签名推断输入类型
- 推断优先级：显式 > 推断 > 默认
- 支持多种函数类型（函数、方法、可调用对象）
- 推断失败的优雅处理

**为什么需要类型推断**：
- 减少样板代码
- 提高开发效率
- 保持类型安全
- 支持渐进式类型系统

## 1. 从函数签名推断输入类型

### 1.1 核心机制

LangGraph 在 `add_node()` 方法中自动推断节点函数的输入类型。

**关键代码**（来自源码分析）：

```python
from typing import get_type_hints
import inspect

def add_node(
    self,
    node: str | StateNode[NodeInputT, ContextT],
    action: StateNode[NodeInputT, ContextT] | None = None,
    *,
    input_schema: type[NodeInputT] | None = None,
) -> Self:
    inferred_input_schema = None

    try:
        # 检查是否为函数、方法或可调用对象
        if (
            isfunction(action)
            or ismethod(action)
            or ismethod(getattr(action, "__call__", None))
        ) and (
            hints := get_type_hints(getattr(action, "__call__"))
            or get_type_hints(action)
        ):
            if input_schema is None:
                # 获取第一个参数名
                first_parameter_name = next(
                    iter(
                        inspect.signature(
                            cast(FunctionType, action)
                        ).parameters.keys()
                    )
                )
                # 提取第一个参数的类型提示
                if input_hint := hints.get(first_parameter_name):
                    if isinstance(input_hint, type) and get_type_hints(input_hint):
                        inferred_input_schema = input_hint
    except (NameError, TypeError, StopIteration):
        pass

    # 应用推断结果
    if input_schema is not None:
        # 优先级 1: 显式指定
        self.nodes[node] = StateNodeSpec[NodeInputT, ContextT](
            input_schema=input_schema,
        )
    elif inferred_input_schema is not None:
        # 优先级 2: 推断的类型
        self.nodes[node] = StateNodeSpec(
            input_schema=inferred_input_schema,
        )
    else:
        # 优先级 3: 图的 state_schema
        self.nodes[node] = StateNodeSpec[StateT, ContextT](
            input_schema=self.state_schema,
        )
```

[来源: sourcecode/langgraph/.../state.py]

**推断步骤**：
1. 检查 action 是否为函数、方法或可调用对象
2. 使用 `get_type_hints()` 提取类型提示
3. 获取第一个参数名
4. 提取第一个参数的类型提示
5. 验证类型提示是否为有效的类型

### 1.2 类型提示提取

**使用 `get_type_hints()`**：

```python
from typing import get_type_hints
from typing_extensions import TypedDict

class State(TypedDict):
    x: int
    y: str

def my_node(state: State) -> dict:
    return {"x": state["x"] + 1}

# 提取类型提示
hints = get_type_hints(my_node)
print(hints)
# 输出: {'state': <class '__main__.State'>, 'return': <class 'dict'>}

# 获取第一个参数的类型
first_param = next(iter(inspect.signature(my_node).parameters.keys()))
print(first_param)  # 'state'
print(hints[first_param])  # <class '__main__.State'>
```

**关键点**：
- `get_type_hints()` 返回字典：参数名 → 类型
- 第一个参数通常是状态参数
- 返回类型不用于推断（仅输入类型）

### 1.3 验证推断的类型

**验证逻辑**：

```python
if isinstance(input_hint, type) and get_type_hints(input_hint):
    inferred_input_schema = input_hint
```

**验证条件**：
1. `isinstance(input_hint, type)`：必须是类型（不是实例）
2. `get_type_hints(input_hint)`：类型必须有类型提示（如 TypedDict、Pydantic、dataclass）

**示例**：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    x: int

# 有效：State 是类型且有类型提示
isinstance(State, type)  # True
get_type_hints(State)  # {'x': <class 'int'>}

# 无效：int 是类型但没有类型提示
isinstance(int, type)  # True
get_type_hints(int)  # {}
```

## 2. 推断优先级

### 2.1 三级优先级

LangGraph 按以下优先级确定节点的输入类型：

1. **显式 `input_schema`**：用户显式指定
2. **推断的类型**：从函数签名推断
3. **图的 `state_schema`**：默认使用图的状态类型

**优先级代码**：

```python
if input_schema is not None:
    # 优先级 1: 显式指定
    self.nodes[node] = StateNodeSpec[NodeInputT, ContextT](
        input_schema=input_schema,
    )
elif inferred_input_schema is not None:
    # 优先级 2: 推断的类型
    self.nodes[node] = StateNodeSpec(
        input_schema=inferred_input_schema,
    )
else:
    # 优先级 3: 图的 state_schema
    self.nodes[node] = StateNodeSpec[StateT, ContextT](
        input_schema=self.state_schema,
    )
```

[来源: sourcecode/langgraph/.../state.py]

### 2.2 优先级示例

**场景 1：显式指定（优先级 1）**

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph

class State(TypedDict):
    x: int

class CustomInput(TypedDict):
    y: str

def my_node(state: State) -> dict:
    return {"x": 1}

graph = StateGraph(State)
graph.add_node("node1", my_node, input_schema=CustomInput)
# 使用 CustomInput（显式指定）
```

**场景 2：推断类型（优先级 2）**

```python
class State(TypedDict):
    x: int

class NodeInput(TypedDict):
    y: str

def my_node(state: NodeInput) -> dict:
    return {"x": 1}

graph = StateGraph(State)
graph.add_node("node1", my_node)
# 使用 NodeInput（从函数签名推断）
```

**场景 3：默认类型（优先级 3）**

```python
class State(TypedDict):
    x: int

def my_node(state) -> dict:  # 无类型提示
    return {"x": 1}

graph = StateGraph(State)
graph.add_node("node1", my_node)
# 使用 State（图的 state_schema）
```

### 2.3 优先级选择建议

| 场景 | 推荐优先级 | 原因 |
|------|-----------|------|
| 节点输入与状态相同 | 优先级 3（默认） | 简洁，无需额外声明 |
| 节点输入是状态子集 | 优先级 2（推断） | 类型安全，自动推断 |
| 节点输入完全不同 | 优先级 1（显式） | 明确意图，避免混淆 |
| 复杂类型系统 | 优先级 1（显式） | 清晰，易于维护 |

## 3. 支持的函数类型

### 3.1 普通函数

**定义**：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    x: int

def my_function(state: State) -> dict:
    return {"x": state["x"] + 1}

# 推断成功：my_function 是函数
graph.add_node("node1", my_function)
```

**检查逻辑**：

```python
from inspect import isfunction

isfunction(my_function)  # True
```

### 3.2 方法

**定义**：

```python
class MyClass:
    def my_method(self, state: State) -> dict:
        return {"x": state["x"] + 1}

obj = MyClass()

# 推断成功：my_method 是方法
graph.add_node("node1", obj.my_method)
```

**检查逻辑**：

```python
from inspect import ismethod

ismethod(obj.my_method)  # True
```

**注意**：第一个参数是 `self`，第二个参数才是状态。

### 3.3 可调用对象

**定义**：

```python
class MyCallable:
    def __call__(self, state: State) -> dict:
        return {"x": state["x"] + 1}

obj = MyCallable()

# 推断成功：obj 是可调用对象
graph.add_node("node1", obj)
```

**检查逻辑**：

```python
from inspect import ismethod

ismethod(getattr(obj, "__call__", None))  # True
```

### 3.4 Lambda 函数

**定义**：

```python
my_lambda = lambda state: {"x": state["x"] + 1}

# 推断失败：lambda 没有类型提示
graph.add_node("node1", my_lambda)
# 使用图的 state_schema
```

**限制**：Lambda 函数通常没有类型提示，无法推断。

### 3.5 支持的函数类型总结

| 函数类型 | 推断支持 | 示例 |
|---------|---------|------|
| 普通函数 | ✓ | `def func(state: State)` |
| 方法 | ✓ | `obj.method(state: State)` |
| 可调用对象 | ✓ | `class C: __call__(state: State)` |
| Lambda | ✗ | `lambda state: ...` |
| 无类型提示函数 | ✗ | `def func(state)` |

## 4. 推断失败的处理

### 4.1 推断失败的场景

**场景 1：无类型提示**

```python
def my_node(state):  # 无类型提示
    return {"x": 1}

# 推断失败：无法提取类型
# 使用图的 state_schema
```

**场景 2：类型提示不是类型**

```python
def my_node(state: dict) -> dict:  # dict 没有类型提示
    return {"x": 1}

# 推断失败：dict 没有 __annotations__
# 使用图的 state_schema
```

**场景 3：Lambda 函数**

```python
my_lambda = lambda state: {"x": 1}

# 推断失败：lambda 没有类型提示
# 使用图的 state_schema
```

**场景 4：类型提示错误**

```python
def my_node(state: "NonExistentType") -> dict:
    return {"x": 1}

# 推断失败：NameError
# 使用图的 state_schema
```

### 4.2 异常处理

**捕获的异常**：

```python
try:
    # 推断逻辑
    ...
except (NameError, TypeError, StopIteration):
    pass
```

**异常类型**：
- `NameError`：类型名称不存在（如字符串类型提示）
- `TypeError`：类型提示无效
- `StopIteration`：没有参数（空函数签名）

### 4.3 优雅降级

**降级策略**：

```python
if input_schema is not None:
    # 使用显式指定的类型
    ...
elif inferred_input_schema is not None:
    # 使用推断的类型
    ...
else:
    # 降级：使用图的 state_schema
    self.nodes[node] = StateNodeSpec[StateT, ContextT](
        input_schema=self.state_schema,
    )
```

**关键点**：
- 推断失败不会抛出异常
- 自动降级到图的 `state_schema`
- 保证系统正常运行

### 4.4 调试推断失败

**手动检查推断逻辑**：

```python
from typing import get_type_hints
import inspect

def debug_inference(func):
    """调试类型推断"""
    try:
        # 检查是否为函数
        print(f"Is function: {inspect.isfunction(func)}")

        # 提取类型提示
        hints = get_type_hints(func)
        print(f"Type hints: {hints}")

        # 获取第一个参数
        params = list(inspect.signature(func).parameters.keys())
        print(f"Parameters: {params}")

        if params:
            first_param = params[0]
            input_hint = hints.get(first_param)
            print(f"First param type: {input_hint}")

            # 验证类型
            if isinstance(input_hint, type):
                print(f"Is type: True")
                print(f"Has type hints: {bool(get_type_hints(input_hint))}")
            else:
                print(f"Is type: False")
    except Exception as e:
        print(f"Error: {e}")

# 测试
class State(TypedDict):
    x: int

def my_node(state: State) -> dict:
    return {"x": 1}

debug_inference(my_node)
```

## 5. 完整代码示例

### 5.1 基础类型推断

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph

class State(TypedDict):
    x: int
    y: str

def node1(state: State) -> dict:
    """推断成功：使用 State"""
    return {"x": state["x"] + 1}

graph = StateGraph(State)
graph.add_node("node1", node1)
# 自动推断 input_schema = State
```

### 5.2 子集类型推断

```python
class State(TypedDict):
    x: int
    y: str
    z: float

class NodeInput(TypedDict):
    x: int  # 只需要 x

def node2(state: NodeInput) -> dict:
    """推断成功：使用 NodeInput"""
    return {"x": state["x"] + 1}

graph = StateGraph(State)
graph.add_node("node2", node2)
# 自动推断 input_schema = NodeInput
```

### 5.3 显式指定类型

```python
class State(TypedDict):
    x: int

class CustomInput(TypedDict):
    y: str

def node3(state: State) -> dict:
    """显式指定：使用 CustomInput"""
    return {"x": 1}

graph = StateGraph(State)
graph.add_node("node3", node3, input_schema=CustomInput)
# 显式指定 input_schema = CustomInput
```

### 5.4 推断失败的处理

```python
class State(TypedDict):
    x: int

def node4(state) -> dict:  # 无类型提示
    """推断失败：使用 State（默认）"""
    return {"x": 1}

graph = StateGraph(State)
graph.add_node("node4", node4)
# 推断失败，使用 State（图的 state_schema）
```

### 5.5 方法和可调用对象

```python
class MyProcessor:
    def process(self, state: State) -> dict:
        """方法：推断成功"""
        return {"x": state["x"] + 1}

class MyCallable:
    def __call__(self, state: State) -> dict:
        """可调用对象：推断成功"""
        return {"x": state["x"] + 1}

processor = MyProcessor()
callable_obj = MyCallable()

graph = StateGraph(State)
graph.add_node("node5", processor.process)  # 推断成功
graph.add_node("node6", callable_obj)  # 推断成功
```

### 5.6 复杂类型系统

```python
from pydantic import BaseModel

class State(TypedDict):
    x: int

class PydanticInput(BaseModel):
    y: str
    z: int

def node7(state: PydanticInput) -> dict:
    """Pydantic 模型：推断成功"""
    return {"x": state.z}

graph = StateGraph(State)
graph.add_node("node7", node7)
# 自动推断 input_schema = PydanticInput
```

## 6. 最佳实践

### 6.1 何时使用类型推断

**推荐使用**：
- 节点输入是状态的子集
- 简单的类型系统
- 快速原型开发

**示例**：

```python
class State(TypedDict):
    x: int
    y: str

class NodeInput(TypedDict):
    x: int  # 只需要 x

def my_node(state: NodeInput) -> dict:
    return {"x": state["x"] + 1}

# 推荐：使用类型推断
graph.add_node("node", my_node)
```

### 6.2 何时显式指定类型

**推荐显式指定**：
- 节点输入与状态完全不同
- 复杂的类型系统
- 需要清晰的文档

**示例**：

```python
class State(TypedDict):
    x: int

class CustomInput(TypedDict):
    y: str  # 完全不同的字段

def my_node(state: State) -> dict:
    return {"x": 1}

# 推荐：显式指定
graph.add_node("node", my_node, input_schema=CustomInput)
```

### 6.3 类型提示最佳实践

**推荐**：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    x: int
    y: str

# 推荐：使用 TypedDict
def my_node(state: State) -> dict:
    return {"x": state["x"] + 1}
```

**避免**：

```python
# 避免：使用 dict（无法推断）
def my_node(state: dict) -> dict:
    return {"x": 1}

# 避免：无类型提示（无法推断）
def my_node(state):
    return {"x": 1}
```

### 6.4 调试类型推断

**使用日志**：

```python
import logging

logging.basicConfig(level=logging.DEBUG)

# LangGraph 会输出类型推断的日志
graph.add_node("node", my_node)
```

**手动验证**：

```python

hints = get_type_hints(my_node)
print(f"Inferred type: {hints.get('state')}")
```

## 7. 常见问题

### Q1: 为什么 Lambda 函数无法推断类型？

**A**: Lambda 函数通常没有类型提示，无法提取类型信息。

```python
# 无法推断
my_lambda = lambda state: {"x": 1}

# 解决方案：使用普通函数
def my_node(state: State) -> dict:
    return {"x": 1}
```

### Q2: 如何推断方法的类型？

**A**: LangGraph 会跳过 `self` 参数，从第二个参数推断类型。

```python
class MyClass:
    def my_method(self, state: State) -> dict:
        return {"x": 1}

obj = MyClass()
graph.add_node("node", obj.my_method)
# 推断成功：使用 State
```

### Q3: 推断失败会抛出异常吗？

**A**: 不会。推断失败会优雅降级到图的 `state_schema`。

```python
def my_node(state):  # 无类型提示
    return {"x": 1}

graph.add_node("node", my_node)
# 推断失败，使用 State（图的 state_schema）
```

### Q4: 如何调试类型推断？

**A**: 使用 `get_type_hints()` 和 `inspect.signature()` 手动检查。

```python
from typing import get_type_hints
import inspect

hints = get_type_hints(my_node)
params = list(inspect.signature(my_node).parameters.keys())
print(f"Type hints: {hints}")
print(f"Parameters: {params}")
```

## 8. 总结

LangGraph 的类型推断机制提供了：
1. **自动类型推断**：从函数签名提取输入类型
2. **三级优先级**：显式 > 推断 > 默认
3. **多种函数类型支持**：函数、方法、可调用对象
4. **优雅降级**：推断失败时使用默认类型

**关键要点**：
- 类型推断减少样板代码
- 推断优先级确保灵活性
- 推断失败不会影响系统运行
- 支持多种函数类型
