# 30字核心

**Partition 是 Collection 的子集，通过数据隔离和自动路由实现多租户场景下的高效检索和资源优化。**

---

## 核心要素拆解

### 1. Partition 是什么？

**Collection 的子集**
- Partition 是 Collection 内部的逻辑数据分区
- 每个 Partition 与父 Collection 共享相同的 Schema
- 一个 Collection 可以包含多个 Partition

### 2. Partition 的核心功能

**数据隔离**
- 实现租户级别的数据隔离
- 支持物理隔离（手动分区）和逻辑隔离（Partition Key）

**自动路由**
- Partition Key 自动将数据路由到 16 个物理分区
- 无需手动管理分区

### 3. Partition 的应用场景

**多租户场景**
- SaaS 应用：每个租户独立的数据隔离
- RAG 系统：租户级别的知识库隔离

**高效检索**
- 避免 99% 的无效扫描
- 延迟从秒级降至毫秒级

**资源优化**
- 热冷数据管理：只加载需要的分区
- 节省内存资源

## 与其他概念的关系

### Partition vs Collection

| 维度 | Collection | Partition |
|------|-----------|-----------|
| 定义 | 数据表 | 数据表的子集 |
| Schema | 独立 Schema | 共享 Collection Schema |
| 数据隔离 | 强隔离 | 中等隔离 |
| 使用场景 | 不同业务 | 同一业务的不同租户 |

### Partition vs Partition Key

| 维度 | 手动分区 | Partition Key |
|------|----------|---------------|
| 管理方式 | 手动创建 | 自动管理 |
| 最大数量 | 1,024 个 | 支持百万租户 |
| 数据隔离 | 物理隔离 | 逻辑隔离 |
| 适用场景 | 中等数量租户 | 大规模多租户 |

## 一句话记忆法

**Partition = 数据分区 + 自动路由 + 多租户隔离**

## 快速验证

```python
from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection

# 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 方式 1：手动分区
collection = Collection("my_collection")
collection.create_partition("tenant_a")  # 手动创建分区

# 方式 2：Partition Key 自动分区（推荐）
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
        FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
        FieldSchema(name="tenant_id", dtype=DataType.VARCHAR, max_length=64,
                   is_partition_key=True)  # 自动分区
    ]
)
collection = Collection("my_collection", schema)
```
