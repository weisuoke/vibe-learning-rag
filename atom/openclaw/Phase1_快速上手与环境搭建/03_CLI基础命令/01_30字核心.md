# 01_30字核心

## 30字核心定义

**OpenClaw CLI 是多通道 AI 网关的统一控制接口，通过 5 个核心命令实现网关管理、消息路由、Agent 交互和通道配置的完整生命周期控制。**

---

## 核心要素拆解

### 1. 统一控制接口
- CLI 是 OpenClaw 的**唯一入口**
- 所有操作通过命令行完成
- 本地和远程 Gateway 统一管理

### 2. 5 个核心命令
```bash
openclaw gateway        # 网关生命周期管理
openclaw message send   # 跨通道消息发送
openclaw agent          # AI Agent 交互
openclaw channels       # 通道状态管理
openclaw config         # 配置持久化
```

### 3. 完整生命周期
- **启动阶段**: `gateway` 启动控制平面
- **配置阶段**: `config` 设置模型和通道
- **运行阶段**: `message` 和 `agent` 处理消息
- **监控阶段**: `channels` 查看状态

---

## 为什么是这 30 字？

### 关键词 1: 多通道 AI 网关
- OpenClaw 的核心定位
- 不是单一聊天工具，是**消息路由中枢**
- 支持 WhatsApp、Telegram、Slack、Discord 等 10+ 通道

### 关键词 2: 统一控制接口
- CLI 抽象了底层复杂性
- 开发者无需关心各通道的 API 差异
- 一套命令操作所有通道

### 关键词 3: 5 个核心命令
- 最小化学习成本
- 覆盖 80% 使用场景
- 其他命令都是这 5 个的扩展

### 关键词 4: 完整生命周期控制
- 从启动到监控的闭环
- 不仅是"发消息"，是**系统管理**
- 支持开发、测试、生产全流程

## 与其他 CLI 工具的对比

| 工具 | 定位 | 核心命令数 | 适用场景 |
|------|------|-----------|---------|
| **OpenClaw CLI** | 多通道 AI 网关控制 | 5 个核心 | 跨平台消息 + AI Agent |
| Docker CLI | 容器管理 | 10+ 核心 | 容器生命周期 |
| Git CLI | 版本控制 | 6 个核心 | 代码管理 |
| AWS CLI | 云服务管理 | 100+ 服务 | 云资源操作 |

**OpenClaw CLI 的优势**：
- 命令数量少，学习曲线平缓
- 专注于消息和 AI，不做无关功能
- 开发者友好的设计（TypeScript 生态）

## 实际应用场景

### 场景 1: 快速启动 AI 助手
```bash
# 3 条命令启动完整系统
openclaw gateway --port 18789
openclaw message send --to +1234567890 --message "Hello"
openclaw agent --message "帮我总结今天的任务"
```

### 场景 2: 多通道消息分发
```bash
# 一条命令发送到所有通道
openclaw message send --to @all --message "系统维护通知"
```

### 场景 3: 生产环境监控
```bash
# 实时查看通道状态
openclaw channels status
openclaw gateway health
```
