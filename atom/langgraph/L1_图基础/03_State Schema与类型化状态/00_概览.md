## 核心价值

State Schema 是 LangGraph 的核心概念，定义了图中数据的结构和流转规则。掌握 State Schema 设计是构建可靠、可维护的 LangGraph 应用的基础。

**为什么重要**：
- 类型安全：编译时发现错误，而非运行时崩溃
- 状态管理：自动处理状态更新和聚合
- 代码可维护：清晰的数据结构，易于理解和修改
- 性能优化：高效的状态序列化和持久化

## 常见问题

**Q1: TypedDict vs Pydantic，应该用哪个？**
A: 优先使用 TypedDict，除非需要验证或复杂默认值。详见 `03_核心概念_01_TypedDict状态定义.md`。

**Q2: Reducer 什么时候执行？**
A: 节点返回后、并行执行后、手动更新时。详见 `03_核心概念_02_Annotated与Reducer.md`。

**Q3: 如何处理 Optional 字段？**
A: 使用 NotRequired 而非 Optional，语义更清晰。详见 `07_实战代码_场景5_Optional字段与默认值.md`。

**Q4: 状态太大怎么办？**
A: 使用 reducer 控制状态增长，定期清理。详见 `03_核心概念_10_生产环境最佳实践.md`。

**Q5: 如何调试状态问题？**
A: 使用 Checkpointer 查看状态历史。详见 `03_核心概念_09_状态持久化与检查点.md`。
