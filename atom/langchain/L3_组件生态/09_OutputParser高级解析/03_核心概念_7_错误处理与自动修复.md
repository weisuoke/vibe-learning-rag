# 核心概念 7：错误处理与自动修复

> 本文档详细讲解 LangChain OutputParser 的错误处理与自动修复机制，包括 OutputParserException、OutputFixingParser、RetryOutputParser 和 OUTPUT_PARSING_FAILURE 错误。

---

## 概述

OutputParser 在解析 LLM 输出时经常遇到格式错误、不完整输出、无效 JSON 等问题。LangChain 提供了完善的错误处理和自动修复机制，确保应用的鲁棒性。

**核心价值**:
- **自动修复**：使用 LLM 自动修复格式错误
- **重试机制**：自动重试失败的解析
- **错误反馈**：提供详细的错误信息
- **生产级可靠性**：确保应用稳定运行

**三层防护体系**:
1. **Prompt 优化**：在 prompt 中明确指定输出格式
2. **部分解析**：使用 `partial=True` 处理不完整输入
3. **错误修复**：使用 OutputFixingParser 或 RetryOutputParser

## 1. OutputParserException：核心异常类

### 1.1 异常定义

OutputParserException 是 LangChain 中所有解析错误的基类。

**类定义**（来源：search_outputparser_02_流式解析错误处理.md）:

```python
from langchain_core.exceptions import OutputParserException

class OutputParserException(Exception):
    """Exception raised when output parsing fails."""
    pass
```

**关键特性**:
- 继承自 Python 标准 `Exception`
- 用于捕获所有解析错误
- 在 streaming 和非 streaming 模式下均可抛出

### 1.2 常见错误场景

**场景 1：JSON 格式错误**

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.exceptions import OutputParserException

parser = JsonOutputParser()

try:
    # 无效的 JSON
    result = parser.parse('{"name": "Alice", "age": }')
except OutputParserException as e:
    print(f"解析失败: {e}")
    # 输出: 解析失败: Invalid JSON: Expecting value: line 1 column 28 (char 27)
```

**场景 2：Pydantic 验证失败**

```python
from langchain_core.output_parsers import PydanticOutputParser
from pydantic import BaseModel, Field

class Person(BaseModel):
    name: str = Field(description="Person's name")
    age: int = Field(description="Person's age", gt=0)

parser = PydanticOutputParser(pydantic_object=Person)

try:
    # age 为负数，验证失败
    result = parser.parse('{"name": "Alice", "age": -5}')
except OutputParserException as e:
    print(f"验证失败: {e}")
    # 输出: 验证失败: 1 validation error for Person
    # age: Input should be greater than 0
```

**场景 3：Markdown 代码块问题**

```python
try:
    # LLM 输出被 markdown 包裹
    result = parser.parse('```json\n{"name": "Alice"}\n```')
except OutputParserException as e:
    print(f"解析失败: {e}")
```

**注意**：JsonOutputParser 已支持自动检测并去除 markdown 包裹，但其他解析器可能不支持。

### 1.3 异常处理最佳实践

**基础错误处理**:

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.exceptions import OutputParserException

parser = JsonOutputParser()

try:
    result = parser.parse(llm_output)
except OutputParserException as e:
    # 记录错误
    logger.error(f"解析失败: {e}")
    # 返回默认值
    result = {}
```

**异常类型过滤**（来源：context7_langchain_01_错误处理.md）:

```python
from langchain_core.exceptions import OutputParserException

def handle_error(error):
    """根据错误类型返回不同的处理策略"""
    if isinstance(error, ToolInputParsingException):
        return "Error: Invalid tool input format. Please check your JSON syntax."
    elif isinstance(error, OutputParserException):
        return f"Error: {str(error)}"
    else:
        return "Error: Unknown error occurred."
```

## 2. OUTPUT_PARSING_FAILURE：官方错误标识

### 2.1 错误定义

OUTPUT_PARSING_FAILURE 是 LangChain 中最常见的解析错误标识。

**官方文档说明**（来源：search_outputparser_02_流式解析错误处理.md）:
> This is the most common parsing error in LangChain. It usually occurs when the LLM output format does not match expectations, especially in streaming mode.

**常见原因**:
- LLM 输出格式不符合预期
- Streaming 模式下输出不完整
- JSON 格式错误
- Pydantic 验证失败

### 2.2 解决方案

**方案 1：使用 Tool Calling**（推荐）

```python
from langchain_openai import ChatOpenAI
from pydantic import BaseModel

class Person(BaseModel):
    name: str
    age: int

# 使用原生结构化输出（2025+ 推荐）
model = ChatOpenAI(model="gpt-4")
structured_model = model.with_structured_output(Person)

result = structured_model.invoke("Alice is 30 years old")
print(result)  # Person(name='Alice', age=30)
```

**方案 2：优化 Prompt 格式**

```python
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

prompt = PromptTemplate.from_template(
    "Extract information about the person.\n"
    "{format_instructions}\n"
    "Person: {person}\n"
)

# 注入格式指令
formatted_prompt = prompt.format(
    format_instructions=parser.get_format_instructions(),
    person="Alice is 30 years old"
)

chain = model | parser
result = chain.invoke(formatted_prompt)
```

**方案 3：升级模型**

```python
# 使用更强大的模型提高输出质量
model = ChatOpenAI(model="gpt-4")  # 而不是 gpt-3.5-turbo
```

### 2.3 Streaming 模式下的特殊处理

**问题描述**（来源：search_outputparser_02_流式解析错误处理.md）:
- XML 解析器因不完整 chunk 失败
- JSON 解析器无法处理部分 JSON
- 累积缓冲区导致内存问题

**解决方案**:

```python
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

# 方案 1：使用部分解析
for chunk in chain.stream(input):
    try:
        # 部分解析支持
        result = parser.parse_result([Generation(text=chunk)], partial=True)
        print(result)
    except OutputParserException:
        # 忽略不完整的 chunk
        continue

# 方案 2：累积完整输出再解析
buffer = ""
for chunk in chain.stream(input):
    buffer += chunk

result = parser.parse(buffer)
```

## 3. OutputFixingParser：自动修复解析器

### 3.1 核心原理

OutputFixingParser 使用 LLM 自动修复格式错误的输出。

**工作流程**:
1. 原始解析器尝试解析输出
2. 如果失败，捕获 OutputParserException
3. 将错误信息和原始输出发送给 LLM
4. LLM 生成修复后的输出
5. 再次尝试解析

**类定义**（来源：search_outputparser_02_流式解析错误处理.md）:

```python
from langchain.output_parsers import OutputFixingParser

fixing_parser = OutputFixingParser.from_llm(
    parser=PydanticOutputParser(pydantic_object=Person),
    llm=ChatOpenAI()
)
```

### 3.2 使用示例

**基础使用**:

```python
from langchain_core.output_parsers import PydanticOutputParser
from langchain.output_parsers import OutputFixingParser
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

class Person(BaseModel):
    name: str = Field(description="Person's name")
    age: int = Field(description="Person's age")

# 创建原始解析器
base_parser = PydanticOutputParser(pydantic_object=Person)

# 创建修复解析器
fixing_parser = OutputFixingParser.from_llm(
    parser=base_parser,
    llm=ChatOpenAI(model="gpt-4")
)

# 错误的输出（缺少引号）
bad_output = '{name: Alice, age: 30}'

# 原始解析器失败
try:
    result = base_parser.parse(bad_output)
except OutputParserException as e:
    print(f"原始解析失败: {e}")

# 修复解析器成功
result = fixing_parser.parse(bad_output)
print(result)  # Person(name='Alice', age=30)
```

**与 LCEL 集成**:

```python
from langchain_core.prompts import PromptTemplate

prompt = PromptTemplate.from_template("Extract person info: {text}")
model = ChatOpenAI()

# 构建链（带自动修复）
chain = prompt | model | fixing_parser

result = chain.invoke({"text": "Alice is 30 years old"})
print(result)  # Person(name='Alice', age=30)
```

### 3.3 工作原理详解

**修复 Prompt 模板**:

```python
# OutputFixingParser 内部使用的 prompt
FIXING_PROMPT = """
Instructions:
{instructions}

Completion:
{completion}

Above, the Completion did not satisfy the constraints given in the Instructions.
Error:
{error}

Please try again. Please only respond with an answer that satisfies the constraints laid out in the Instructions:
"""
```

**修复流程**:

```
1. 原始输出: {name: Alice, age: 30}
   ↓
2. 解析失败: JSONDecodeError
   ↓
3. 发送给 LLM:
   - Instructions: 输出格式要求
   - Completion: 原始输出
   - Error: 错误信息
   ↓
4. LLM 修复: {"name": "Alice", "age": 30}
   ↓
5. 再次解析: 成功
```

### 3.4 性能考虑

**优势**:
- 自动修复格式错误
- 提高应用鲁棒性
- 减少人工干预

**劣势**:
- 额外的 LLM 调用（增加延迟和成本）
- 可能无法修复所有错误
- 不适合实时场景

**最佳实践**:
- 优先优化 prompt，减少错误发生
- 仅在必要时使用 OutputFixingParser
- 考虑使用缓存减少重复修复

## 4. RetryOutputParser：重试解析器

### 4.1 核心原理

RetryOutputParser 在解析失败时自动重试，提供原始 prompt 和错误信息给 LLM。

**与 OutputFixingParser 的区别**:

| 特性 | OutputFixingParser | RetryOutputParser |
|------|-------------------|-------------------|
| **输入** | 仅错误输出 | 原始 prompt + 错误输出 |
| **上下文** | 无原始 prompt | 有原始 prompt |
| **适用场景** | 格式修复 | 内容和格式修复 |
| **成功率** | 中 | 高 |

### 4.2 使用示例

**基础使用**（来源：search_outputparser_02_流式解析错误处理.md）:

```python
from langchain.output_parsers import RetryOutputParser
from langchain_core.output_parsers import PydanticOutputParser
from langchain_openai import ChatOpenAI
from pydantic import BaseModel

class Person(BaseModel):
    name: str
    age: int

# 创建原始解析器
base_parser = PydanticOutputParser(pydantic_object=Person)

# 创建重试解析器
retry_parser = RetryOutputParser.from_llm(
    parser=base_parser,
    llm=ChatOpenAI(model="gpt-4"),
    max_retries=3
)

# 使用重试解析器
result = retry_parser.parse_with_prompt(
    completion="Alice, age unknown",
    prompt_value="Extract person info: Alice is 30 years old"
)
print(result)  # Person(name='Alice', age=30)
```

**与 LCEL 集成**:

```python
from langchain_core.prompts import PromptTemplate
from langchain_core.runnables import RunnablePassthrough

prompt = PromptTemplate.from_template("Extract person info: {text}")
model = ChatOpenAI()

# 构建链（带重试）
chain = (
    RunnablePassthrough.assign(
        prompt_value=lambda x: prompt.format(**x)
    )
    | model
    | retry_parser
)

result = chain.invoke({"text": "Alice is 30 years old"})
```

### 4.3 重试策略

**配置重试次数**:

```python
retry_parser = RetryOutputParser.from_llm(
    parser=base_parser,
    llm=ChatOpenAI(),
    max_retries=3  # 最多重试 3 次
)
```

**重试流程**:

```
1. 第一次解析失败
   ↓
2. 重试 1: 提供原始 prompt + 错误信息
   ↓
3. 如果仍失败，重试 2
   ↓
4. 如果仍失败，重试 3
   ↓
5. 如果仍失败，抛出 OutputParserException
```

### 4.4 实战示例：生产级错误处理

```python
from langchain.output_parsers import RetryOutputParser, OutputFixingParser
from langchain_core.output_parsers import PydanticOutputParser
from langchain_openai import ChatOpenAI
from pydantic import BaseModel
import logging

class Person(BaseModel):
    name: str
    age: int

# 配置日志
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

# 创建解析器
base_parser = PydanticOutputParser(pydantic_object=Person)
retry_parser = RetryOutputParser.from_llm(
    parser=base_parser,
    llm=ChatOpenAI(model="gpt-4"),
    max_retries=2
)
fixing_parser = OutputFixingParser.from_llm(
    parser=base_parser,
    llm=ChatOpenAI(model="gpt-4")
)

def parse_with_fallback(text: str, prompt: str = None) -> Person:
    """生产级解析函数，带多层回退"""

    # 第一层：直接解析
    try:
        logger.info("尝试直接解析")
        return base_parser.parse(text)
    except OutputParserException as e:
        logger.warning(f"直接解析失败: {e}")

    # 第二层：重试解析（如果有 prompt）
    if prompt:
        try:
            logger.info("尝试重试解析")
            return retry_parser.parse_with_prompt(text, prompt)
        except OutputParserException as e:
            logger.warning(f"重试解析失败: {e}")

    # 第三层：修复解析
    try:
        logger.info("尝试修复解析")
        return fixing_parser.parse(text)
    except OutputParserException as e:
        logger.error(f"修复解析失败: {e}")
        raise

# 使用
result = parse_with_fallback(
    text='{name: Alice, age: 30}',
    prompt="Extract person info: Alice is 30 years old"
)
```

## 5. 错误处理工作流程

### 5.1 多个结构化输出错误处理

**场景**：LLM 尝试返回多个结构化输出，但只期望一个（来源：context7_langchain_01_错误处理.md）

**错误处理流程**:

```
1. 模型错误：返回多个工具调用（ContactInfo 和 EventDetails）
   ↓
2. 错误反馈：系统自动生成 ToolMessage 提示错误
   ↓
3. 模型重试：模型根据错误反馈重新生成正确的单个输出
   ↓
4. 成功返回：最终返回正确的结构化响应
```

**代码示例**:

```python
from pydantic import BaseModel, Field
from typing import Union
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy

class ContactInfo(BaseModel):
    name: str = Field(description="Person's name")
    email: str = Field(description="Email address")

class EventDetails(BaseModel):
    event_name: str = Field(description="Name of the event")
    date: str = Field(description="Event date")

agent = create_agent(
    model="gpt-4",
    tools=[],
    response_format=ToolStrategy(Union[ContactInfo, EventDetails])  # 默认: handle_errors=True
)

result = agent.invoke({
    "messages": [{"role": "user", "content": "Extract info: John Doe (john@email.com) is organizing Tech Conference on March 15th"}]
})
```

### 5.2 流式工具调用的错误处理

**挑战**（来源：context7_langchain_01_错误处理.md）:
> When using fine-grained tool streaming, the system may emit invalid or partial JSON inputs during the streaming process. It is important to implement proper error handling and validation in your code to account for these edge cases.

**解决方案**:

```python
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

async def stream_with_error_handling(chain, input):
    """流式解析带错误处理"""
    buffer = ""

    async for chunk in chain.astream(input):
        buffer += chunk

        try:
            # 尝试部分解析
            result = parser.parse_result(
                [Generation(text=buffer)],
                partial=True
            )
            yield result
        except OutputParserException:
            # 忽略不完整的 chunk
            continue
```

## 6. 错误处理最佳实践

### 6.1 三层防护策略

**第一层：Prompt 优化**

```python
from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

# 在 prompt 中明确指定输出格式

    format_instructions=parser.get_format_instructions(),
    text="Alice is 30 years old"

```

**第二层：部分解析**

```python
# 使用 partial=True 处理不完整输入
result = parser.parse_result(
    [Generation(text=incomplete_json)],
    partial=True
)
```

**第三层：错误修复**

```python
# 使用 OutputFixingParser 或 RetryOutputParser
fixing_parser = OutputFixingParser.from_llm(parser=base_parser, llm=llm)
```

### 6.2 选择合适的错误处理策略

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| 简单格式错误 | OutputFixingParser | 快速修复，成本低 |
| 内容和格式错误 | RetryOutputParser | 提供上下文，成功率高 |
| 实时应用 | Prompt 优化 + 部分解析 | 低延迟 |
| 批处理任务 | RetryOutputParser | 可容忍延迟 |
| 生产环境 | 多层回退 | 最高可靠性 |

### 6.3 性能优化

**减少错误发生**:
1. 使用更强大的模型（如 GPT-4）
2. 优化 prompt 格式
3. 提供清晰的格式指令
4. 使用原生结构化输出（2025+ 推荐）

**减少修复成本**:
1. 缓存修复结果
2. 限制重试次数
3. 使用更便宜的模型进行修复

**监控和日志**:

```python
import logging

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

def parse_with_logging(text: str) -> dict:
    try:
        result = parser.parse(text)
        logger.info("解析成功")
        return result
    except OutputParserException as e:
        logger.error(f"解析失败: {e}")
        logger.error(f"原始输出: {text}")
        raise
```

## 总结

错误处理与自动修复是 LangChain OutputParser 的核心特性，通过三层防护体系确保应用的鲁棒性：

**核心要点**:
1. **OutputParserException**：所有解析错误的基类，用于捕获和处理错误
2. **OUTPUT_PARSING_FAILURE**：最常见的解析错误，通过 prompt 优化、tool calling 或模型升级解决
3. **OutputFixingParser**：使用 LLM 自动修复格式错误，适合简单场景
4. **RetryOutputParser**：提供原始 prompt 和错误信息，成功率更高
5. **三层防护**：Prompt 优化 → 部分解析 → 错误修复

**最佳实践**:
- 优先优化 prompt，减少错误发生
- 使用原生结构化输出（2025+ 推荐）
- 根据场景选择合适的错误处理策略
- 实现多层回退机制确保可靠性
- 监控和日志记录所有解析错误
