## 必会 1：函数返回 Result

### 概念

Rust 中"这个操作可能失败"的表达方式是：**函数返回 `Result<T, E>`**。成功返回 `Ok(值)`，失败返回 `Err(错误)`。

不像 TypeScript 用 throw 抛异常，Rust 把错误当作**普通返回值**——函数签名就告诉你"我可能失败"。

### Rust 代码

```rust
// 返回 Result：成功是 f64，失败是 String
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err("division by zero".to_string())  // 失败：返回错误
    } else {
        Ok(a / b)                             // 成功：返回结果
    }
}
```

### TypeScript 对照

```typescript
// TypeScript：throw 抛异常——函数签名看不出会失败
function divide(a: number, b: number): number {
    if (b === 0) throw new Error("division by zero");
    return a / b;
}

// Rust 的 Result 等价于这种显式模式（但 TS 没有语言级支持）
type Result<T, E> = { ok: true; value: T } | { ok: false; error: E };
```

### ZeroClaw 中的使用

ZeroClaw 所有 trait 方法都返回 `anyhow::Result<T>`（即 `Result<T, anyhow::Error>` 的简写）：

```rust
// src/tools/traits.rs
async fn execute(&self, args: serde_json::Value) -> anyhow::Result<ToolResult>;

// src/memory/traits.rs
async fn store(&self, entry: MemoryEntry) -> anyhow::Result<()>;  // () = 成功但无返回值
async fn recall(&self, query: &str, limit: usize) -> anyhow::Result<Vec<MemoryEntry>>;
```

**记住**：看到 `-> anyhow::Result<T>` 就知道"这个函数可能失败"。

## 必会 2：用 ? 传播错误

### 概念

`?` 是 Rust 最重要的错误处理语法糖：
- 遇到 `Ok(value)` → 解包，拿到 value，继续执行
- 遇到 `Err(e)` → **自动 return Err(e)**，函数提前退出

一个字符替代十几行 match。**看到 `?` 就想"这里可能失败并提前返回"。**

### Rust 代码

```rust
use anyhow::Result;
use std::fs;

fn load_config(path: &str) -> Result<String> {
    let content = fs::read_to_string(path)?;  // 失败？自动 return Err
    let trimmed = content.trim().to_string();
    Ok(trimmed)                                // 成功：返回结果
}
```

没有 `?` 的等价写法（你不需要这样写，但要理解 `?` 做了什么）：

```rust
fn load_config(path: &str) -> Result<String> {
    let content = match fs::read_to_string(path) {
        Ok(c) => c,
        Err(e) => return Err(e.into()),  // ? 自动做的事
    };
    Ok(content.trim().to_string())
}
```

### TypeScript 对照

```typescript
// TypeScript：异常自动冒泡（隐式）
function loadConfig(path: string): string {
    const content = fs.readFileSync(path, "utf-8");  // 失败？自动 throw
    return content.trim();
}

// Rust 的 ? 类似于 TS 的自动冒泡，但是显式标记的
// 你能看到哪一行可能失败（有 ? 的那行）
```

### ZeroClaw 中的使用

```rust
// src/security/secrets.rs — 链式 ? 传播
pub fn encrypt(&self, plaintext: &str) -> anyhow::Result<String> {
    let key_bytes = self.load_or_create_key()?;  // 第一个 ?：读密钥可能失败
    let cipher = build_cipher(&key_bytes)?;       // 第二个 ?：构建加密器可能失败
    let ciphertext = cipher.encrypt(&nonce, plaintext.as_bytes())
        .map_err(|e| anyhow::anyhow!("Encryption failed: {e}"))?;  // 第三个 ?
    Ok(base64_encode(&ciphertext))
}
```

**规则**：`?` 只能在返回 `Result`（或 `Option`）的函数中使用。忘记写返回类型会编译报错。

## 必会 3：用 match 处理最终结果

### 概念

`?` 负责在中间层传播错误。但在**调用链顶层**（比如 main 函数、HTTP handler），你需要用 `match` 决定"成功怎么办、失败怎么办"。

### Rust 代码

```rust
fn main() {
    match load_config("config.json") {
        Ok(config) => println!("Loaded: {config}"),
        Err(e) => eprintln!("Error: {e}"),  // 打印错误，不崩溃
    }
}
```

也可以用 `if let` 只处理一种情况：

```rust
// 只关心成功的情况
if let Ok(config) = load_config("config.json") {
    apply_config(&config);
}
```

### TypeScript 对照

```typescript
// TypeScript：try/catch 是顶层处理
try {
    const config = loadConfig("config.json");
    console.log(`Loaded: ${config}`);
} catch (e) {
    console.error(`Error: ${e}`);
}
```

### ZeroClaw 中的使用

```rust
// src/tools/web_fetch.rs — match 区分业务错误和程序错误
let validated_url = match self.validate_url(url) {
    Ok(v) => v,
    Err(e) => return Ok(ToolResult {
        success: false,
        error: Some(e.to_string()),
        ..Default::default()
    }),
};

// src/tools/shell.rs — 嵌套 match 处理超时 + 执行结果
match tokio::time::timeout(Duration::from_secs(30), cmd.output()).await {
    Ok(Ok(output)) => { /* 成功 */ },
    Ok(Err(e))     => { /* 执行失败 */ },
    Err(_)         => { /* 超时 */ },
}
```

**规则**：中间层用 `?` 传播，顶层用 `match` 处理。

## 必会 4：Option 的安全提取

### 概念

`Option<T>` 表示"值可能存在（Some）也可能不存在（None）"。Rust 没有 null，所有"可能为空"都用 Option。

你需要掌握两个最常用的提取方式：
- `.unwrap_or(默认值)` — None 时用默认值
- `.ok_or_else(|| 错误)` — None 转成 Err，然后用 `?` 传播

### Rust 代码

```rust
// 方式1：unwrap_or — 给默认值（适合配置字段）
let port: Option<u16> = None;
let actual_port = port.unwrap_or(8080);  // None → 8080

// 方式2：ok_or_else + ? — 转成 Result 传播（适合必填参数）
fn get_name(args: &serde_json::Value) -> anyhow::Result<String> {
    let name = args.get("name")
        .and_then(|v| v.as_str())
        .ok_or_else(|| anyhow::anyhow!("Missing 'name' parameter"))?;
    Ok(name.to_string())
}
```

### TypeScript 对照

```typescript
// unwrap_or  ≈  ?? 空值合并
const port = config.port ?? 8080;

// ok_or_else ≈  手动检查 + throw
const name = args.name ?? (() => { throw new Error("Missing 'name'"); })();
```

### ZeroClaw 中的使用

```rust
// 配置默认值 — unwrap_or
let streaming = config.streaming.unwrap_or(false);

// 参数提取 — and_then + ok_or_else + ?（ZeroClaw 最高频模式）
// src/tools/shell.rs
let command = args.get("command")
    .and_then(|v| v.as_str())
    .ok_or_else(|| anyhow::anyhow!("Missing 'command' parameter"))?;

// HTTP 头提取 — and_then + unwrap_or
// src/tools/web_fetch.rs
let content_type = response.headers()
    .get(reqwest::header::CONTENT_TYPE)
    .and_then(|v| v.to_str().ok())
    .unwrap_or("");
```

**记住**：
- 缺失是正常的（配置可选字段）→ `unwrap_or(默认值)`
- 缺失是错误的（必填参数）→ `ok_or_else(|| anyhow!("..."))?`

## 必会 5：anyhow 三件套

### 概念

anyhow 是 Rust 应用层错误处理的标准库。ZeroClaw 全面使用 anyhow。你只需要记住三样东西：

| 工具 | 作用 | 用法 |
|------|------|------|
| `anyhow::Result<T>` | 统一返回类型 | 函数签名写 `-> Result<T>` |
| `.context("msg")` | 给错误加描述 | `operation().context("what failed")?` |
| `?` | 传播错误 | 放在可能失败的表达式后面 |

### Rust 代码

```rust
use anyhow::{Context, Result};
use std::fs;

fn load_and_parse(path: &str) -> Result<serde_json::Value> {
    let content = fs::read_to_string(path)
        .context("Failed to read config file")?;          // 三件套：context + ?

    let parsed = serde_json::from_str(&content)
        .context("Failed to parse JSON")?;                 // 三件套：context + ?

    Ok(parsed)
}
// 错误输出：Failed to parse JSON
//          Caused by: expected value at line 1 column 1
// 既有你写的上下文，也保留了原始错误——排查问题极其方便
```

额外两个常用宏（加分项）：

```rust
use anyhow::{bail, ensure};

fn validate(input: &str) -> Result<()> {
    ensure!(!input.is_empty(), "input cannot be empty");  // 条件不满足 → 返回 Err
    if input.len() > 1000 {
        bail!("input too long: {} chars", input.len());   // 主动返回 Err
    }
    Ok(())
}
```

### TypeScript 对照

```typescript
// anyhow::Result<T>  ≈  Promise<T>（统一的"可能失败"容器）
// .context("msg")    ≈  .catch(e => { throw new Error(`msg: ${e}`) })
// ?                  ≈  await（自动传播错误）
// bail!("msg")       ≈  throw new Error("msg")
// ensure!(cond, msg) ≈  if (!cond) throw new Error(msg)
```

### ZeroClaw 中的使用

```rust
// src/memory/sqlite.rs — context 添加上下文
Connection::open(&path_buf)
    .context("SQLite failed to open database")?;

// src/tools/web_fetch.rs — bail! 提前退出
if url_str.is_empty() {
    anyhow::bail!("URL cannot be empty");
}

// src/security/secrets.rs — ensure! 条件断言
anyhow::ensure!(blob.len() > NONCE_LEN, "Encrypted value too short");
```

### 速查：ZeroClaw 错误处理模式 Top 5

```rust
// 模式1：? 传播（最常用）
let data = some_operation()?;

// 模式2：context + ?（推荐，带上下文）
let data = some_operation().context("描述哪里出错了")?;

// 模式3：Option → Result + ?（参数提取）
let val = args.get("key")
    .and_then(|v| v.as_str())
    .ok_or_else(|| anyhow::anyhow!("Missing 'key'"))?;

// 模式4：bail!（主动返回错误）
if invalid_condition {
    anyhow::bail!("说明为什么失败");
}

// 模式5：unwrap_or（安全默认值）
let flag = config.optional_field.unwrap_or(false);
```
