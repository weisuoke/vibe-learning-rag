## 卡片 1：VectorStore 统一接口

**核心概念**：

LangChain VectorStore 提供统一的抽象接口，所有向量存储后端都实现相同的方法，便于切换。

**关键代码**：

```python
from langchain_core.vectorstores import VectorStore
from langchain_openai import OpenAIEmbeddings

# 所有后端都支持这些方法
vector_store.add_texts(["文档1", "文档2"])
results = vector_store.similarity_search("查询", k=5)
vector_store.delete(ids=["doc1"])
```

**实战技巧**：

1. **工厂模式**：使用工厂函数创建向量存储，便于切换后端
2. **统一接口**：依赖抽象而非具体实现，降低耦合
3. **异步支持**：高并发场景使用异步方法（`aadd_texts`, `asimilarity_search`）

**[来源: reference/source_vectorstore_base_01.md | LangChain 源码分析]**

## 卡片 2：InMemory 快速原型

**核心概念**：

InMemory 是纯内存向量存储，零配置启动，适合快速原型和测试。

**关键代码**：

```python
from langchain_core.vectorstores import InMemoryVectorStore
from langchain_openai import OpenAIEmbeddings

# 零配置启动
vector_store = InMemoryVectorStore(OpenAIEmbeddings())

# 添加文档
vector_store.add_texts(["文档1", "文档2"])

# 检索
results = vector_store.similarity_search("查询", k=5)
```

**实战技巧**：

1. **测试用例**：单元测试和集成测试的理想选择
2. **快速验证**：验证 RAG 流程是否正确
3. **限制**：数据不持久化，程序重启丢失

**适用场景**：< 1K 文档，快速原型，测试

**[来源: reference/source_inmemory_02.md | LangChain 源码分析]**

## 卡片 3：Chroma 本地开发

**核心概念**：

Chroma 是本地优先的向量数据库，支持持久化，适合本地开发和中小规模生产。

**关键代码**：

```python
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings

# 持久化到本地
chroma = Chroma(
    collection_name="my_docs",
    embedding_function=OpenAIEmbeddings(),
    persist_directory="./chroma_db"  # 持久化目录
)

# 添加文档
chroma.add_texts(["文档1", "文档2"])

# 带元数据过滤的检索
results = chroma.similarity_search(
    "查询",
    k=5,
    filter={"category": "tech"}
)
```

**实战技巧**：

1. **持久化**：使用 `persist_directory` 保存数据
2. **元数据过滤**：支持复杂的元数据查询
3. **MMR 检索**：使用 `max_marginal_relevance_search` 提高多样性

**适用场景**：< 100K 文档，本地开发，中小规模生产

**[来源: reference/source_chroma_03.md | LangChain 源码分析]**

## 卡片 4：FAISS 高性能检索

**核心概念**：

FAISS 是 Meta 开发的高性能向量检索库，C++ 实现，适合静态数据集。

**关键代码**：

```python
from langchain_community.vectorstores import FAISS
from langchain_openai import OpenAIEmbeddings

# 创建索引
faiss = FAISS.from_texts(["文档1", "文档2"], OpenAIEmbeddings())

# 保存索引
faiss.save_local("./faiss_index")

# 加载索引
faiss = FAISS.load_local("./faiss_index", OpenAIEmbeddings())

# 检索
results = faiss.similarity_search("查询", k=5)
```

**实战技巧**：

1. **持久化**：使用 `save_local` 和 `load_local` 保存/加载索引
2. **批量操作**：使用 `merge_from` 合并多个索引
3. **GPU 加速**：安装 `faiss-gpu` 提升性能

**限制**：不支持增量更新，不支持元数据过滤

**适用场景**：< 1M 文档，静态数据，高性能需求

**[来源: reference/source_faiss_pinecone_milvus_05.md | 集成概述]**

## 卡片 5：Qdrant 生产部署

**核心概念**：

Qdrant 是 Rust 实现的生产级向量数据库，支持分布式部署和混合检索。

**关键代码**：

```python
from qdrant_client import QdrantClient
from langchain_qdrant import QdrantVectorStore
from langchain_openai import OpenAIEmbeddings

# 连接 Qdrant 服务
client = QdrantClient(url="http://localhost:6333")

# 创建向量存储
qdrant = QdrantVectorStore(
    client=client,
    collection_name="my_docs",
    embeddings=OpenAIEmbeddings()
)

# 添加文档
qdrant.add_texts(["文档1", "文档2"])

# 检索
results = qdrant.similarity_search("查询", k=5)
```

**实战技巧**：

1. **Docker 部署**：`docker run -p 6333:6333 qdrant/qdrant`
2. **距离策略**：选择合适的距离度量（COSINE, EUCLID, DOT）
3. **异步支持**：使用 `AsyncQdrantClient` 提高并发性能

**适用场景**：> 100K 文档，生产环境，高并发

## 卡片 6：Pinecone 云端托管

**核心概念**：

Pinecone 是完全托管的向量数据库服务，零运维，自动扩展。

**关键代码**：

```python
from pinecone import Pinecone, ServerlessSpec
from langchain_pinecone import PineconeVectorStore
from langchain_openai import OpenAIEmbeddings

# 初始化 Pinecone
pc = Pinecone(api_key='YOUR_API_KEY')

# 创建索引
pc.create_index(
    name="my-index",
    dimension=1536,
    metric="cosine",
    spec=ServerlessSpec(cloud="aws", region="us-east-1")
)

# 创建向量存储
index = pc.Index("my-index")
pinecone = PineconeVectorStore(
    index=index,
    embedding=OpenAIEmbeddings(),
    namespace="production"
)

# 添加文档
pinecone.add_texts(["文档1", "文档2"])
```

**实战技巧**：

1. **命名空间**：使用 namespace 隔离不同环境或租户
2. **Serverless 模式**：按使用付费，适合不可预测负载
3. **成本优化**：定期清理无用数据，使用命名空间共享索引

**适用场景**：快速上线，无运维团队，大规模应用

**[来源: reference/context7_pinecone_02.md | 官方文档]**

## 卡片 8：后端选择决策树

**核心概念**：

根据数据规模、预算、运维能力选择合适的向量存储后端。

**决策流程**：

```
数据规模？
  ├─ < 1K → InMemory（测试）或 Chroma（开发）
  ├─ 1K-10K → Chroma（持久化）或 FAISS（高性能）
  ├─ 10K-100K → 预算？
  │             ├─ 充足 → Pinecone
  │             └─ 有限 → Qdrant
  └─ > 100K → 运维能力？
              ├─ 有 → Milvus（企业级）
              └─ 无 → Pinecone（托管）
```

**关键代码**：

```python
def recommend_backend(num_docs, budget, ops_capability):
    # 推荐向量存储后端
    if num_docs < 1000:
        return "InMemory" if not need_persistence else "Chroma"
    elif num_docs < 10000:
        return "Chroma" if need_persistence else "FAISS"
    elif num_docs < 100000:
        return "Pinecone" if budget == "high" else "Qdrant"
    else:
        return "Milvus" if ops_capability else "Pinecone"
```

**实战技巧**：

1. **从小规模开始**：使用 Chroma 快速验证
2. **评估真实负载**：基于实际数据和查询模式测试
3. **保持灵活性**：使用 LangChain 抽象，便于切换后端

## 卡片 9：性能优化技巧

**核心概念**：

通过索引优化、批量操作、缓存策略提升 VectorStore 性能。

**关键技巧**：

**1. 索引优化**：
```python
# 选择合适的索引类型
# HNSW > IVF > FLAT（性能）
# FLAT > IVF > HNSW（准确率）
```

**2. 批量操作**：
```python
# 批量添加文档
batch_size = 100
for i in range(0, len(documents), batch_size):
    batch = documents[i:i+batch_size]
    vector_store.add_documents(batch)
```

**3. 缓存策略**：
```python
from functools import lru_cache

@lru_cache(maxsize=1000)
def cached_search(query, k):
    return vector_store.similarity_search(query, k=k)
```

**4. 异步操作**：
```python
# 高并发场景使用异步
results = await vector_store.asimilarity_search("查询", k=5)
```

**5. 向量维度优化**：
```python
# 使用较小的 embedding 模型
# 384 维 vs 1536 维（性能提升 4 倍）
```

**实战技巧**：

1. **监控性能**：查询延迟、吞吐量、内存占用
2. **定期清理**：删除无用数据降低存储负担

```python
# Chroma 没有直接导出功能
```

**2. 数据导入**：
```python
# 导入到 Qdrant
from langchain_qdrant import QdrantVectorStore

qdrant = QdrantVectorStore.from_documents(
    documents=exported_docs,
    embedding=OpenAIEmbeddings(),
```

**3. 渐进式迁移**：
```python
# 双写策略
def add_documents_dual_write(documents):
    # 同时写入新旧后端
    old_store.add_documents(documents)
    new_store.add_documents(documents)

# 灰度发布
def search_with_fallback(query, k):
    try:
        # 优先使用新后端
        return new_store.similarity_search(query, k=k)
    except Exception:
        # 失败时回退到旧后端
        return old_store.similarity_search(query, k=k)
```

**实战技巧**：

1. **保留备份**：迁移前备份原始数据
2. **验证数据**：对比新旧后端的查询结果
3. **监控指标**：监控性能和错误率
4. **准备回滚**：出现问题时快速回滚

**常见迁移路径**：
- Chroma → Qdrant（规模增长）
- FAISS → Qdrant（需要增量更新）
- 自托管 → Pinecone（降低运维成本）
- Pinecone → 自托管（降低使用成本）
