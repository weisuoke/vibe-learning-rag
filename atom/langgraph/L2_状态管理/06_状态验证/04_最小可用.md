## 4.1 用 Pydantic BaseModel 替代 TypedDict

**一句话：** 把 TypedDict 换成 BaseModel，立刻获得运行时验证能力

```python
from pydantic import BaseModel
from langgraph.graph import StateGraph, START, END

# ❌ TypedDict：没有运行时验证
# from typing_extensions import TypedDict
# class State(TypedDict):
#     query: str
#     answer: str

# ✅ Pydantic BaseModel：有运行时验证
class State(BaseModel):
    query: str
    answer: str = ""

def process(state: State):
    return {"answer": f"回答: {state.query}"}

builder = StateGraph(State)
builder.add_node("process", process)
builder.add_edge(START, "process")
builder.add_edge("process", END)
graph = builder.compile()

# ✅ 正常调用
result = graph.invoke({"query": "什么是 RAG？"})
print(result)  # {'query': '什么是 RAG？', 'answer': '回答: 什么是 RAG？'}

# ❌ 缺少必填字段 query，Pydantic 会报 ValidationError
# graph.invoke({"answer": "没有问题"})
```

**关键点：**
- 只需把 `TypedDict` 换成 `BaseModel`，其他代码几乎不变
- `StateGraph(State)` 同时接受 TypedDict 和 BaseModel
- BaseModel 的字段可以直接用 `=` 设置默认值，比 TypedDict 更直观
- 必填字段（无默认值）在调用时必须提供，否则报错

## 4.2 使用 Field 约束

**一句话：** 用 `Field` 为字段添加范围、长度等约束，不用写自定义验证器

```python
from pydantic import BaseModel, Field

class RAGState(BaseModel):
    # 字符串长度约束：查询不能为空，不能超过 500 字
    query: str = Field(min_length=1, max_length=500)

    # 数值范围约束：检索数量在 1-20 之间
    top_k: int = Field(default=5, ge=1, le=20)

    # 浮点数范围约束：温度在 0-2 之间
    temperature: float = Field(default=0.7, ge=0.0, le=2.0)

    # 带描述的字段（方便调试和文档生成）
    answer: str = Field(default="", description="LLM 生成的回答")
```

**验证效果：**

```python
# ✅ 正常
state = RAGState(query="什么是向量数据库？")

# ❌ 查询为空字符串
try:
    RAGState(query="")
except Exception as e:
    print("拦截：查询不能为空")

# ❌ top_k 超出范围
try:
    RAGState(query="测试", top_k=100)
except Exception as e:
    print("拦截：top_k 必须在 1-20 之间")

# ✅ 自动类型转换：字符串 "3" 转为整数 3
state = RAGState(query="测试", top_k="3")
print(state.top_k)  # 3 (int)
```

**常用 Field 约束速查：**

| 约束 | 适用类型 | 说明 | 示例 |
|------|---------|------|------|
| `min_length` | str, list | 最小长度 | `Field(min_length=1)` |
| `max_length` | str, list | 最大长度 | `Field(max_length=500)` |
| `ge` | int, float | 大于等于 | `Field(ge=0)` |
| `le` | int, float | 小于等于 | `Field(le=100)` |
| `gt` | int, float | 大于 | `Field(gt=0)` |
| `lt` | int, float | 小于 | `Field(lt=1.0)` |
| `pattern` | str | 正则匹配 | `Field(pattern=r"^gpt-.*$")` |
| `default` | 任意 | 默认值 | `Field(default=5)` |

## 4.3 添加 field_validator

**一句话：** 用 `field_validator` 实现 Field 无法覆盖的自定义验证逻辑

```python
from pydantic import BaseModel, Field, field_validator

class RAGState(BaseModel):
    query: str = Field(min_length=1)
    top_k: int = Field(default=5, ge=1, le=20)
    model_name: str = "gpt-4"

    @field_validator('query')
    @classmethod
    def clean_query(cls, v: str) -> str:
        """清洗查询：去除首尾空格，防止纯空格绕过 min_length"""
        v = v.strip()
        if not v:
            raise ValueError('查询不能为纯空格')
        return v

    @field_validator('model_name')
    @classmethod
    def validate_model(cls, v: str) -> str:
        """限制可用模型列表"""
        allowed = ['gpt-4', 'gpt-3.5-turbo', 'claude-3-opus']
        if v not in allowed:
            raise ValueError(f'模型必须是 {allowed} 之一')
        return v
```

**使用效果：**

```python
# ✅ 自动清洗空格
state = RAGState(query="  什么是 RAG？  ")
print(state.query)  # "什么是 RAG？"（已去除空格）

# ❌ 纯空格被拦截
try:
    RAGState(query="   ")
except Exception as e:
    print("拦截：查询不能为纯空格")

# ❌ 不支持的模型
try:
    RAGState(query="测试", model_name="llama-3")
except Exception as e:
    print("拦截：模型不在允许列表中")
```

**field_validator 的两个核心用途：**
1. **数据清洗**：去空格、统一大小写、格式化
2. **业务校验**：白名单检查、格式验证、依赖检查

## 4.4 捕获验证错误

**一句话：** 用 `try/except ValidationError` 优雅处理非法输入

```python
from pydantic import BaseModel, Field, ValidationError

class RAGState(BaseModel):
    query: str = Field(min_length=1)
    top_k: int = Field(default=5, ge=1, le=20)

# 方式1：直接捕获 Pydantic 验证错误
try:
    state = RAGState(query="", top_k=100)
except ValidationError as e:
    print(f"验证失败，共 {e.error_count()} 个错误：")
    for error in e.errors():
        print(f"  字段: {error['loc']}, 原因: {error['msg']}")
    # 输出：
    # 验证失败，共 2 个错误：
    #   字段: ('query',), 原因: String should have at least 1 character
    #   字段: ('top_k',), 原因: Input should be less than or equal to 20
```

**在 LangGraph 图调用中捕获：**

```python
from langgraph.graph import StateGraph, START, END

def process(state: RAGState):
    return {"answer": f"回答: {state.query}"}

builder = StateGraph(RAGState)
builder.add_node("process", process)
builder.add_edge(START, "process")
builder.add_edge("process", END)
graph = builder.compile()

# 安全调用模式
def safe_invoke(user_input: dict) -> dict:
    try:
        return graph.invoke(user_input)
    except ValidationError as e:
        return {
            "error": True,
            "message": "输入验证失败",
            "details": e.errors()
        }

# ✅ 正常输入
result = safe_invoke({"query": "什么是 RAG？"})

# ❌ 非法输入 → 返回友好错误
result = safe_invoke({"query": "", "top_k": 100})
print(result)
# {'error': True, 'message': '输入验证失败', 'details': [...]}
```

**关键点：**
- `ValidationError` 来自 `pydantic`，不是 Python 内置异常
- `e.errors()` 返回结构化的错误列表，适合 API 响应
- `e.error_count()` 返回错误数量
- 在生产环境中，永远不要让 ValidationError 直接暴露给用户

## 4.5 混合架构入门

**一句话：** 边界用 Pydantic 验证输入，内部用 TypedDict 保持性能

```python
from pydantic import BaseModel, Field
from typing_extensions import TypedDict
from typing import Annotated
import operator
from langgraph.graph import StateGraph, START, END

# 外部输入 Schema：Pydantic 严格验证
class InputState(BaseModel):
    query: str = Field(min_length=1, max_length=500)
    top_k: int = Field(default=5, ge=1, le=20)

# 内部状态 Schema：TypedDict 轻量传递
class InternalState(TypedDict):
    query: str
    top_k: int
    results: list[str]
    answer: str

# 构建图时指定两个 Schema
def search(state):
    return {"results": [f"结果{i}" for i in range(state["top_k"])]}

def generate(state):
    context = "\n".join(state["results"])
    return {"answer": f"基于 {len(state['results'])} 条结果的回答"}

builder = StateGraph(InternalState, input=InputState)
builder.add_node("search", search)
builder.add_node("generate", generate)
builder.add_edge(START, "search")
builder.add_edge("search", "generate")
builder.add_edge("generate", END)
graph = builder.compile()

# ✅ 输入经过 Pydantic 验证
result = graph.invoke({"query": "什么是 RAG？", "top_k": 3})

# ❌ 非法输入在入口处被拦截
# graph.invoke({"query": "", "top_k": 100})
```

**混合架构的核心思路：**

```
用户输入 → [Pydantic InputState 验证] → 内部 TypedDict 状态 → 节点处理 → 输出
              ↑ 严格验证                    ↑ 零开销传递
```

**为什么推荐混合架构？**
- Pydantic 验证有性能开销，不需要在每个节点都执行
- TypedDict 支持 Annotated + Reducer，Pydantic BaseModel 不直接支持
- 边界验证 + 内部轻量 = 安全与性能的最佳平衡
