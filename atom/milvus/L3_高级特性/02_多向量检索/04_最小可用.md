## 4.1 理解多向量检索的本质

**多向量检索 = 在多个向量字段上同时进行 ANN 检索 + 融合结果**

```python
# 核心概念
Collection = {
    "id": Int64,
    "dense_vector": FloatVector(dim=768),      # 稠密向量（语义理解）
    "sparse_vector": SparseFloatVector,        # 稀疏向量（关键词匹配）
    "image_vector": FloatVector(dim=512),      # 图片向量（多模态）
}

# 混合检索流程
results = hybrid_search(
    [dense_req, sparse_req, image_req],  # 多个检索请求
    ranker=RRFRanker()                   # 融合策略
)
```

**为什么需要多向量检索？**
- **多模态检索**：图片 + 文本混合检索（电商、内容平台）
- **全文搜索增强**：BM25 + 稠密向量混合检索（文档问答、知识库）
- **多语言检索**：中文向量 + 英文向量混合检索（跨语言搜索）
- **多粒度检索**：标题向量 + 内容向量混合检索（精准匹配）

## 4.2 选择 Ranker 策略

### 方案 1：RRFRanker（推荐，不确定权重时）

**适用场景**：不确定各向量字段的重要性，让算法自动平衡

**RRF 算法原理**：
```python
# Reciprocal Rank Fusion
score = 1 / (k + rank)  # k 默认值为 60
```

**Python 示例**：
```python
from pymilvus import AnnSearchRequest, RRFRanker, MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 1. 定义稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense_embedding],      # 查询向量
    anns_field="dense_vector",         # 向量字段名
    param={"metric_type": "IP"},       # 内积距离
    limit=10                           # 返回结果数量
)

# 2. 定义稀疏向量检索请求
sparse_req = AnnSearchRequest(
    data=[query_sparse_embedding],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 3. 使用 RRF 融合结果
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(),                # 自动平衡
    limit=10,
    output_fields=["text", "metadata"]
)

# 4. 处理结果
for hit in results[0]:
    print(f"ID: {hit['id']}, Score: {hit['distance']}, Text: {hit['text']}")
```

### 方案 2：WeightedRanker（已知权重时）

**适用场景**：明确知道各向量字段的重要性，需要精确控制

**Python 示例**：
```python
from pymilvus import AnnSearchRequest, WeightedRanker, Collection

collection = Collection("my_collection")

# 1. 定义稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense_embedding],
    anns_field="dense_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 2. 定义稀疏向量检索请求
sparse_req = AnnSearchRequest(
    data=[query_sparse_embedding],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 3. 使用加权融合（稀疏向量权重 0.7，稠密向量权重 1.0）
ranker = WeightedRanker(0.7, 1.0)

# 4. 执行混合检索
results = collection.hybrid_search(
    [sparse_req, dense_req],
    rerank=ranker,
    limit=10,
    output_fields=["text"]
)

# 5. 处理结果
for hit in results[0]:
    print(f"ID: {hit.id}, Score: {hit.distance}, Text: {hit.get('text')}")
```

**权重调整建议**：
- 关键词匹配重要：稀疏向量权重 > 稠密向量权重（如 1.0 vs 0.5）
- 语义理解重要：稠密向量权重 > 稀疏向量权重（如 1.0 vs 0.3）
- 平衡场景：权重相等（如 1.0 vs 1.0）

## 4.3 核心操作

### 步骤 1：定义多向量字段 Schema

```python
from pymilvus import FieldSchema, CollectionSchema, DataType, Collection

# 定义 Schema
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=2048),

        # 稠密向量字段
        FieldSchema(name="dense_vector", dtype=DataType.FLOAT_VECTOR, dim=768),

        # 稀疏向量字段
        FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR),
    ],
    description="Multi-vector collection for hybrid search"
)

# 创建 Collection
collection = Collection("multi_vector_collection", schema)
```

### 步骤 2：为所有向量字段创建索引

**关键约束**：所有向量字段都必须创建索引才能加载 Collection

```python
# 为稠密向量创建索引
dense_index_params = {
    "index_type": "HNSW",
    "metric_type": "IP",
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="dense_vector", index_params=dense_index_params)

# 为稀疏向量创建索引
sparse_index_params = {
    "index_type": "SPARSE_INVERTED_INDEX",
    "metric_type": "IP",
    "params": {"drop_ratio_build": 0.2}
}
collection.create_index(field_name="sparse_vector", index_params=sparse_index_params)

# 加载 Collection
collection.load()
```

### 步骤 3：插入多向量数据

```python
# 准备数据
data = [
    ["Document 1", "Document 2", "Document 3"],  # text
    [
        [0.1] * 768,  # dense_vector for doc1
        [0.2] * 768,  # dense_vector for doc2
        [0.3] * 768,  # dense_vector for doc3
    ],
    [
        {1: 0.5, 10: 0.3, 100: 0.8},  # sparse_vector for doc1
        {2: 0.6, 20: 0.4, 200: 0.9},  # sparse_vector for doc2
        {3: 0.7, 30: 0.5, 300: 1.0},  # sparse_vector for doc3
    ],
]

# 插入数据
collection.insert(data)
collection.flush()
```

### 步骤 4：执行混合检索

```python
from pymilvus import AnnSearchRequest, RRFRanker

# 准备查询向量
query_dense = [0.15] * 768
query_sparse = {1: 0.5, 10: 0.3}

# 定义检索请求
dense_req = AnnSearchRequest(
    data=[query_dense],
    anns_field="dense_vector",
    param={"metric_type": "IP", "params": {"ef": 100}},
    limit=10
)

sparse_req = AnnSearchRequest(
    data=[query_sparse],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 执行混合检索
results = collection.hybrid_search(
    [dense_req, sparse_req],
    rerank=RRFRanker(),
    limit=5,
    output_fields=["text"]
)

# 输出结果
for hit in results[0]:
    print(f"Text: {hit.get('text')}, Score: {hit.distance}")
```

## 4.4 实际应用场景

### 场景 1：BM25 + 稠密向量混合检索（推荐）

**适用场景**：文档问答、知识库检索

```python
from pymilvus import (
    FieldSchema, CollectionSchema, DataType, Collection,
    Function, FunctionType, AnnSearchRequest, RRFRanker
)

# 1. 定义 Schema（包含 BM25 函数）
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=2048,
                   enable_analyzer=True),  # 启用分析器
        FieldSchema(name="dense_vector", dtype=DataType.FLOAT_VECTOR, dim=768),
        FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR),
    ]
)

# 2. 添加 BM25 函数（自动生成稀疏向量）
bm25_function = Function(
    name="bm25_function",
    function_type=FunctionType.BM25,
    input_field_names=["text"],
    output_field_names="sparse_vector",
    params={}
)
schema.add_function(bm25_function)

# 3. 创建 Collection
collection = Collection("bm25_hybrid_collection", schema)

# 4. 创建索引
collection.create_index("dense_vector", {
    "index_type": "HNSW",
    "metric_type": "IP",
    "params": {"M": 16, "efConstruction": 200}
})
collection.create_index("sparse_vector", {
    "index_type": "SPARSE_INVERTED_INDEX",
    "metric_type": "IP"
})
collection.load()

# 5. 插入数据（BM25 自动生成稀疏向量）
data = [
    ["What is RAG?", "How to use Milvus?", "Vector database tutorial"],  # text
    [[0.1]*768, [0.2]*768, [0.3]*768],  # dense_vector
]
collection.insert(data)

# 6. 混合检索
query_text = "RAG tutorial"
query_dense = get_embedding(query_text)  # 使用 Embedding 模型

# BM25 检索请求（直接传入文本）
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=10
)

# 稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense],
    anns_field="dense_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 执行混合检索
results = collection.hybrid_search(
    [sparse_req, dense_req],
    rerank=RRFRanker(),
    limit=5,
    output_fields=["text"]
)
```

### 场景 2：多模态检索（图片 + 文本）

**适用场景**：电商搜索、内容平台

```python
from pymilvus import FieldSchema, CollectionSchema, DataType, Collection

# 1. 定义多模态 Schema
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(name="product_name", dtype=DataType.VARCHAR, max_length=512),
        FieldSchema(name="text_vector", dtype=DataType.FLOAT_VECTOR, dim=768),
        FieldSchema(name="image_vector", dtype=DataType.FLOAT_VECTOR, dim=512),
    ]
)

collection = Collection("multimodal_collection", schema)

# 2. 创建索引
collection.create_index("text_vector", {
    "index_type": "HNSW",
    "metric_type": "COSINE",
    "params": {"M": 16, "efConstruction": 200}
})
collection.create_index("image_vector", {
    "index_type": "HNSW",
    "metric_type": "COSINE",
    "params": {"M": 16, "efConstruction": 200}
})
collection.load()

# 3. 插入数据
data = [
    ["Product A", "Product B", "Product C"],  # product_name
    [text_embedding_a, text_embedding_b, text_embedding_c],  # text_vector
    [image_embedding_a, image_embedding_b, image_embedding_c],  # image_vector
]
collection.insert(data)

# 4. 多模态检索
query_text_vector = get_text_embedding("red dress")
query_image_vector = get_image_embedding("uploaded_image.jpg")

text_req = AnnSearchRequest(
    data=[query_text_vector],
    anns_field="text_vector",
    param={"metric_type": "COSINE"},
    limit=10
)

image_req = AnnSearchRequest(
    data=[query_image_vector],
    anns_field="image_vector",
    param={"metric_type": "COSINE"},
    limit=10
)

# 使用加权融合（图片权重更高）
results = collection.hybrid_search(
    [text_req, image_req],
    rerank=WeightedRanker(0.3, 0.7),  # 文本权重 0.3，图片权重 0.7
    limit=5,
    output_fields=["product_name"]
)
```

## 4.5 完整可运行示例

```python
"""
多向量混合检索完整示例
环境要求：pymilvus >= 2.4.0
"""

from pymilvus import (
    connections, FieldSchema, CollectionSchema, DataType,
    Collection, AnnSearchRequest, RRFRanker, utility
)
import numpy as np

# 1. 连接 Milvus
connections.connect(host="localhost", port="19530")

# 2. 删除已存在的 Collection（如果存在）
collection_name = "hybrid_search_demo"
if utility.has_collection(collection_name):
    utility.drop_collection(collection_name)

# 3. 定义 Schema
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=512),
        FieldSchema(name="dense_vector", dtype=DataType.FLOAT_VECTOR, dim=128),
        FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR),
    ],
    description="Hybrid search demo collection"
)

# 4. 创建 Collection
collection = Collection(collection_name, schema)

# 5. 创建索引
print("Creating indexes...")
collection.create_index("dense_vector", {
    "index_type": "HNSW",
    "metric_type": "IP",
    "params": {"M": 16, "efConstruction": 200}
})
collection.create_index("sparse_vector", {
    "index_type": "SPARSE_INVERTED_INDEX",
    "metric_type": "IP"
})

# 6. 插入数据
print("Inserting data...")
texts = [
    "Milvus is a vector database",
    "RAG combines retrieval and generation",
    "Hybrid search improves accuracy",
    "Multi-vector search supports multiple fields",
    "BM25 is good for keyword matching"
]

dense_vectors = np.random.rand(5, 128).tolist()
sparse_vectors = [
    {1: 0.5, 10: 0.3, 100: 0.8},
    {2: 0.6, 20: 0.4, 200: 0.9},
    {3: 0.7, 30: 0.5, 300: 1.0},
    {4: 0.8, 40: 0.6, 400: 0.7},
    {5: 0.9, 50: 0.7, 500: 0.6},
]

data = [texts, dense_vectors, sparse_vectors]
collection.insert(data)
collection.flush()

# 7. 加载 Collection
print("Loading collection...")
collection.load()

# 8. 执行混合检索
print("\nPerforming hybrid search...")
query_dense = np.random.rand(128).tolist()
query_sparse = {1: 0.5, 10: 0.3, 100: 0.8}

dense_req = AnnSearchRequest(
    data=[query_dense],
    anns_field="dense_vector",
    param={"metric_type": "IP", "params": {"ef": 100}},
    limit=5
)

sparse_req = AnnSearchRequest(
    data=[query_sparse],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=5
)

results = collection.hybrid_search(
    [dense_req, sparse_req],
    rerank=RRFRanker(),
    limit=3,
    output_fields=["text"]
)

# 9. 输出结果
print("\nSearch Results:")
for i, hit in enumerate(results[0], 1):
    print(f"{i}. Text: {hit.get('text')}")
    print(f"   Score: {hit.distance:.4f}")
    print(f"   ID: {hit.id}\n")

# 10. 清理
print("Cleaning up...")
collection.release()
utility.drop_collection(collection_name)
connections.disconnect("default")

print("Demo completed!")
```
