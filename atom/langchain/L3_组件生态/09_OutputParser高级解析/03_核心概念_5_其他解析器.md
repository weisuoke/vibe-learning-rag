## 概述

本文档介绍 LangChain 中两个重要但常被忽视的解析器：StrOutputParser（最简单的解析器）和 XMLOutputParser（XML 格式解析器）。它们分别代表了解析器设计的两个极端：极简主义和安全优先。

**核心特性**：
- **StrOutputParser**：零开销、流式友好、LCEL 兼容
- **XMLOutputParser**：安全解析、流式支持、格式指令生成

## 1. StrOutputParser - 最简单的解析器

### 1.1 类定义

```python
class StrOutputParser(BaseTransformOutputParser[str]):
    """Extract text content from model outputs as a string.

    Converts model outputs (such as `AIMessage` or `AIMessageChunk` objects) into plain
    text strings. It's the simplest output parser and is useful when you need string
    responses for downstream processing, display, or storage.

    Supports streaming, yielding text chunks as they're generated by the model.

    Example:
        ```python
        from langchain_core.output_parsers import StrOutputParser
        from langchain_openai import ChatOpenAI

        model = ChatOpenAI(model="gpt-4o")
        parser = StrOutputParser()

        # Get string output from a model
        message = model.invoke("Tell me a joke")
        result = parser.invoke(message)
        print(result)  # plain string

        # With streaming - use transform() to process a stream
        stream = model.stream("Tell me a story")
        for chunk in parser.transform(stream):
            print(chunk, end="", flush=True)
        ```
    """
```

**关键特性**：
- 继承自 `BaseTransformOutputParser`（支持流式解析）
- 返回类型固定为 `str`
- 最简单的 OutputParser，几乎不做任何处理

### 1.2 parse() 方法 - 直接返回

```python
@override
def parse(self, text: str) -> str:
    """Returns the input text with no changes."""
    return text
```

**关键发现**：
- 不做任何转换，直接返回输入
- 用于从 `AIMessage` 中提取文本内容
- 支持流式输出（继承自 `BaseTransformOutputParser`）

### 1.3 序列化支持

```python
@classmethod
def is_lc_serializable(cls) -> bool:
    """`StrOutputParser` is serializable.

    Returns:
        `True`
    """
    return True

@classmethod
def get_lc_namespace(cls) -> list[str]:
    """Get the namespace of the LangChain object.

    Returns:
        `["langchain", "schema", "output_parser"]`
    """
    return ["langchain", "schema", "output_parser"]
```

**用途**：
- 支持序列化和反序列化
- 属于 LangChain 核心命名空间
- 可以保存和加载解析器配置

### 1.4 _type 属性

```python
@property
def _type(self) -> str:
    """Return the output parser type for serialization."""
    return "default"
```

**用途**：
- 用于序列化时的类型标识
- `"default"` 表示这是默认的字符串解析器

### 1.5 使用示例

#### 基础用法

```python
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

model = ChatOpenAI()
parser = StrOutputParser()

chain = model | parser

result = chain.invoke("Tell me a joke")
# 输出: "Why did the chicken cross the road? ..."
```

#### 流式用法

```python
# 流式输出
for chunk in chain.stream("Tell me a story"):
    print(chunk, end="", flush=True)
    # 输出: "Once", " ", "upon", " ", "a", " ", "time", ...
```

#### 与 LCEL 集成

```python
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("Tell me about {topic}")
chain = prompt | model | parser

result = chain.invoke({"topic": "AI"})
# 输出: "Artificial Intelligence is..."
```

## 2. XMLOutputParser - XML 格式解析器

### 2.1 类定义

```python
"""Output parser for XML format."""

import contextlib
import re
import xml
import xml.etree.ElementTree as ET
from collections.abc import AsyncIterator, Iterator
from typing import Any, Literal
from xml.etree.ElementTree import TreeBuilder

from typing_extensions import override

from langchain_core.exceptions import OutputParserException
from langchain_core.messages import BaseMessage
from langchain_core.output_parsers.transform import BaseTransformOutputParser
from langchain_core.runnables.utils import AddableDict

try:
    from defusedxml import ElementTree  # type: ignore[import-untyped]
    from defusedxml.ElementTree import XMLParser  # type: ignore[import-untyped]

    _HAS_DEFUSEDXML = True
except ImportError:
    _HAS_DEFUSEDXML = False
```

**关键特性**：
- 继承自 `BaseTransformOutputParser`（支持流式解析）
- 支持两种 XML 解析器：`defusedxml`（安全）和 `xml`（标准）
- 提供格式指令模板（`XML_FORMAT_INSTRUCTIONS`）

### 2.2 XML 格式指令

```python
XML_FORMAT_INSTRUCTIONS = """The output should be formatted as a XML file.
1. Output should conform to the tags below.
2. If tags are not given, make them on your own.
3. Remember to always open and close all the tags.

As an example, for the tags ["foo", "bar", "baz"]:
1. String "<foo>\n   <bar>\n      <baz></baz>\n   </bar>\n</foo>" is a well-formatted instance of the schema.
2. String "<foo>\n   <bar>\n   </foo>" is a badly-formatted instance.
3. String "<foo>\n   <tag>\n   </tag>\n</foo>" is a badly-formatted instance.

Here are the output tags:
```
{tags}
```"""
```

**用途**：
- 生成 Prompt 指令，告诉 LLM 如何生成 XML
- 包含示例和反例
- 可以注入到 Prompt 中

### 2.3 _StreamingParser - 流式 XML 解析器

```python
class _StreamingParser:
    """Streaming parser for XML.

    This implementation is pulled into a class to avoid implementation drift between
    `transform` and `atransform` of the `XMLOutputParser`.
    """

    def __init__(self, parser: Literal["defusedxml", "xml"]) -> None:
        """Initialize the streaming parser.

        Args:
            parser: Parser to use for XML parsing.

                Can be either `'defusedxml'` or `'xml'`. See documentation in
                `XMLOutputParser` for more information.

        Raises:
            ImportError: If `defusedxml` is not installed and the `defusedxml` parser is
                requested.
        """
        if parser == "defusedxml":
            if not _HAS_DEFUSEDXML:
                msg = (
                    "defusedxml is not installed. "
                    "Please install it to use the defusedxml parser."
                    "You can install it with `pip install defusedxml` "
                )
                raise ImportError(msg)
            parser_ = XMLParser(target=TreeBuilder())
        else:
            parser_ = None
        self.pull_parser = ET.XMLPullParser(["start", "end"], _parser=parser_)
        self.xml_start_re = re.compile(r"<[a-zA-Z:_]")
        self.current_path: list[str] = []
        self.current_path_has_children = False
        self.buffer = ""
        self.xml_started = False
```

**关键特性**：
- 使用 `XMLPullParser` 实现流式解析
- 支持 `defusedxml`（防止 XXE 攻击）
- 维护当前路径和缓冲区状态

### 2.4 defusedxml vs xml

#### defusedxml（推荐）

**优点**：
- 防止 XML 外部实体（XXE）攻击
- 防止 XML 炸弹攻击
- 更安全，适合生产环境

**安装**：
```bash
pip install defusedxml
```

#### xml（标准库）

**优点**：
- Python 标准库，无需额外安装
- 性能更好

**缺点**：
- 不安全，仅适合可信输入

#### 选择建议

```python
# 生产环境（推荐）
parser = XMLOutputParser(parser="defusedxml")

# 开发环境（可选）
parser = XMLOutputParser(parser="xml")
```

## 3. 流式 XML 解析工作原理

### 3.1 parse() 方法

```python
def parse(self, chunk: str | BaseMessage) -> Iterator[AddableDict]:
    """Parse a chunk of text.

    Args:
        chunk: A chunk of text to parse. This can be a `str` or a `BaseMessage`.

    Yields:
        A `dict` representing the parsed XML element.

    Raises:
        xml.etree.ElementTree.ParseError: If the XML is not well-formed.
    """
    if isinstance(chunk, BaseMessage):
        # extract text
        chunk_content = chunk.content
        if not isinstance(chunk_content, str):
            # ignore non-string messages (e.g., function calls)
            return
        chunk = chunk_content
    # add chunk to buffer of unprocessed text
    self.buffer += chunk
    # ... (省略具体实现)
```

**关键逻辑**：
1. **缓冲区管理**：累积输入块到 `buffer`
2. **XML 检测**：使用正则表达式检测 XML 开始标签
3. **增量解析**：使用 `XMLPullParser` 逐个解析 XML 元素
4. **路径跟踪**：维护当前 XML 路径（`current_path`）
5. **元素输出**：逐个输出完整的 XML 元素

### 3.2 工作流程示例

**输入流**：
```xml
"<person>" → "<name>Alice" → "</name>" → "<age>30</age>" → "</person>"
```

**解析过程**：

1. **第一个块**：`"<person>"`
   - buffer = `"<person>"`
   - 检测到 XML 开始标签
   - 等待更多内容

2. **第二个块**：`"<name>Alice"`
   - buffer = `"<person><name>Alice"`
   - 解析到 `<person>` 和 `<name>` 开始标签
   - 等待闭合标签

3. **第三个块**：`"</name>"`
   - buffer = `"<person><name>Alice</name>"`
   - 解析到完整的 `<name>` 元素
   - 输出：`{"name": "Alice"}`

4. **第四个块**：`"<age>30</age>"`
   - buffer = `"<person><name>Alice</name><age>30</age>"`
   - 解析到完整的 `<age>` 元素
   - 输出：`{"age": "30"}`

5. **第五个块**：`"</person>"`
   - buffer = `"<person><name>Alice</name><age>30</age></person>"`
   - 解析到完整的 `<person>` 元素
   - 输出：`{"person": {"name": "Alice", "age": "30"}}`

## 4. 安全考虑

### 4.1 XXE 攻击（XML 外部实体攻击）

#### 攻击示例

```xml
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<foo>&xxe;</foo>
```

**危害**：
- 读取服务器文件
- 执行远程代码
- 拒绝服务攻击

#### 防御措施

```python
# 使用 defusedxml 解析器
parser = XMLOutputParser(parser="defusedxml")

# defusedxml 会自动禁用外部实体引用
```

### 4.2 XML 炸弹攻击

#### 攻击示例

```xml
<!DOCTYPE lolz [
  <!ENTITY lol "lol">
  <!ENTITY lol2 "&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;&lol;">
  <!ENTITY lol3 "&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;&lol2;">
]>
<lolz>&lol3;</lolz>
```

**危害**：
- 指数级内存消耗
- 服务器崩溃
- 拒绝服务攻击

#### 防御措施

```python
# 使用 defusedxml 解析器
parser = XMLOutputParser(parser="defusedxml")

# 限制 XML 大小
max_xml_size = 1024 * 1024  # 1MB
if len(xml_text) > max_xml_size:
    raise ValueError("XML too large")

# 设置解析超时
import signal
signal.alarm(5)  # 5 秒超时
```

### 4.3 安全最佳实践

```python
from langchain_core.output_parsers import XMLOutputParser

# 1. 使用 defusedxml（推荐）
parser = XMLOutputParser(parser="defusedxml")

# 2. 验证 XML 输入
def validate_xml(xml_text: str) -> bool:
    # 检查 XML 大小
    if len(xml_text) > 1024 * 1024:  # 1MB
        return False

    # 检查是否包含 DOCTYPE（可能包含实体定义）
    if "<!DOCTYPE" in xml_text:
        return False

    # 检查是否包含外部实体引用
    if "<!ENTITY" in xml_text:
        return False

    return True

# 3. 使用前验证
if validate_xml(xml_text):
    result = parser.parse(xml_text)
else:
    raise ValueError("Invalid XML")
```

### 5.4 完整示例：结构化人物信息提取

```python
from langchain_core.output_parsers import XMLOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 初始化

## 6. 架构设计亮点

### 6.1 StrOutputParser

1. **极简设计**：不做任何转换，直接返回输入
2. **流式友好**：继承自 `BaseTransformOutputParser`
3. **序列化支持**：可以保存和加载配置
4. **LCEL 兼容**：可以在 LCEL 中使用
5. **零开销**：不需要缓冲区，无延迟

### 6.2 XMLOutputParser

1. **安全优先**：支持 `defusedxml` 防止 XXE 攻击
2. **流式解析**：使用 `XMLPullParser` 实现增量解析
3. **格式指令**：提供 Prompt 模板
4. **路径跟踪**：维护当前 XML 路径
5. **错误处理**：捕获 XML 解析错误

## 7. 性能考虑

### 7.1 StrOutputParser

- **零开销**：不做任何转换
- **流式友好**：逐块输出，无延迟
- **内存友好**：不需要缓冲区

### 7.2 XMLOutputParser

- **解析开销**：XML 解析比 JSON 慢
- **缓冲区开销**：需要维护缓冲区
- **安全开销**：`defusedxml` 比标准 `xml` 慢

**性能对比**：

| 解析器 | 解析速度 | 内存占用 | 安全性 |
|--------|----------|----------|--------|
| StrOutputParser | 极快 | 极低 | N/A |
| XMLOutputParser (xml) | 中等 | 中等 | 低 |
| XMLOutputParser (defusedxml) | 较慢 | 中等 | 高 |

## 10. 总结

### 10.1 StrOutputParser

- **最简单的解析器**：不做任何转换
- **流式友好**：逐块输出，无延迟
- **LCEL 兼容**：可以在 LCEL 中使用
- **适用场景**：需要纯文本输出时

### 10.2 XMLOutputParser

- **XML 格式解析**：支持流式解析
- **安全优先**：支持 `defusedxml` 防止攻击
- **格式指令**：提供 Prompt 模板
- **适用场景**：需要结构化 XML 输出时

### 10.3 选择建议

| 场景 | 推荐解析器 |
|------|-----------|
| 纯文本输出 | `StrOutputParser` |
| JSON 输出 | `JsonOutputParser` |
| Pydantic 验证 | `PydanticOutputParser` |
| XML 输出（生产） | `XMLOutputParser(parser="defusedxml")` |
| XML 输出（开发） | `XMLOutputParser(parser="xml")` |
| 列表输出 | `ListOutputParser` 系列 |
| 工具调用 | `JsonOutputToolsParser` 系列 |

### 10.4 不适用场景

- **StrOutputParser**：需要结构化输出时
- **XMLOutputParser**：需要 JSON 格式时、需要严格类型验证时
