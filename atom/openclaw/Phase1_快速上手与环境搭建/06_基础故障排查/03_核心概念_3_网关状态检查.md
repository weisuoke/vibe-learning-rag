# OpenClaw 基础故障排查 - 核心概念 3：网关状态检查

## 一、概念定义

**网关状态检查是 OpenClaw 故障排查的核心环节，通过多层次的状态验证和探测机制，快速识别 Gateway 的运行状态、端口绑定、RPC 连接和服务健康度。**

**来源**: `reference/source_gateway_troubleshooting.md`

---

## 二、核心价值

### 2.1 为什么需要网关状态检查？

**问题场景**：
- Gateway 启动后无响应
- 端口 18789 无法访问
- 通道连接但消息不流动
- 不知道 Gateway 是否正常运行
- 升级后 Gateway 行为异常

**传统解决方式的问题**：
- ❌ 手动检查进程 → 进程存在不代表正常
- ❌ 手动测试端口 → 不知道内部状态
- ❌ 查看日志 → 信息分散难以判断
- ❌ 重启试试 → 可能掩盖真正问题

**网关状态检查的价值**：
- ✅ 快速判断 Gateway 是否运行
- ✅ 验证端口绑定状态
- ✅ 检查 RPC 连接健康度
- ✅ 识别配置问题
- ✅ 提供详细的诊断信息

**来源**: `reference/source_gateway_troubleshooting.md`

### 3.2 状态判断表

| 状态 | 含义 | 下一步操作 |
|------|------|-----------|
| `Runtime: running` | Gateway 正常运行 | 检查 RPC 连接 |
| `Runtime: stopped` | Gateway 未启动 | `openclaw gateway start` |
| `Runtime: starting` | Gateway 启动中 | 等待几秒后重新检查 |
| `Runtime: error` | Gateway 异常 | 查看日志 `openclaw logs` |
| `RPC probe: ok` | RPC 连接正常 | 检查通道状态 |
| `RPC probe: failed` | RPC 连接失败 | 检查端口和配置 |
| `RPC probe: timeout` | RPC 超时 | 检查网络和防火墙 |

## 四、工作原理

### 4.1 状态检查流程

```typescript
// Gateway 状态检查流程
async function checkGatewayStatus(): Promise<GatewayStatus> {
  // 1. 检查进程是否运行
  const processRunning = await checkProcess();
  if (!processRunning) {
    return { status: 'stopped' };
  }

  // 2. 检查端口是否监听
  const portListening = await checkPort(18789);
  if (!portListening) {
    return { status: 'error', error: 'Port not listening' };
  }

  // 3. 检查 RPC 连接
  const rpcStatus = await probeRPC();
  if (rpcStatus !== 'ok') {
    return { status: 'error', error: 'RPC connection failed' };
  }

  // 4. 检查配置
  const configValid = await validateConfig();
  if (!configValid) {
    return { status: 'error', error: 'Invalid configuration' };
  }

  // 5. 返回健康状态
  return {
    status: 'running',
    pid: process.pid,
    uptime: process.uptime(),
    port: 18789,
    mode: config.gateway.mode
  };
}
```

**来源**: `reference/source_gateway_troubleshooting.md`

### 4.2 RPC 探测机制

```typescript
// RPC 探测实现
async function probeRPC(): Promise<RPCProbe> {
  const startTime = Date.now();

  try {
    // 1. 建立 RPC 连接
    const client = await createRPCClient('http://localhost:18789');

    // 2. 发送 ping 请求
    const response = await client.ping();

    // 3. 计算延迟
    const latency = Date.now() - startTime;

    // 4. 验证响应
    if (response.status === 'ok') {
      return {
        status: 'ok',
        latency,
        version: response.version
      };
    } else {
      return {
        status: 'failed',
        error: response.error
      };
    }
  } catch (error) {
    // 5. 处理错误
    if (error.code === 'ECONNREFUSED') {
      return { status: 'failed', error: 'Connection refused' };
    } else if (error.code === 'ETIMEDOUT') {
      return { status: 'timeout', error: 'Request timeout' };
    } else {
      return { status: 'failed', error: error.message };
    }
  }
}
```

**来源**: `reference/source_troubleshooting_main.md`

### 4.3 端口检查实现

```typescript
// 端口检查实现
import * as net from 'net';

async function checkPort(port: number): Promise<boolean> {
  return new Promise((resolve) => {
    const socket = new net.Socket();

    // 设置超时
    socket.setTimeout(1000);

    // 尝试连接
    socket.connect(port, 'localhost', () => {
      socket.destroy();
      resolve(true);  // 端口可访问
    });

    // 连接失败
    socket.on('error', () => {
      resolve(false);  // 端口不可访问
    });

    // 超时
    socket.on('timeout', () => {
      socket.destroy();
      resolve(false);
    });
  });
}

// 使用示例
const isListening = await checkPort(18789);
if (!isListening) {
  console.error('Gateway port 18789 is not listening');
}
```

## 六、常见问题

### 6.1 Gateway 显示 running 但无法连接？

**问题**：`openclaw gateway status` 显示 running，但 `openclaw gateway probe` 失败

**可能原因**：
1. RPC 服务未正常启动
2. 端口被防火墙阻止
3. 配置错误导致 RPC 无法响应

**解决方法**：
```bash
# 1. 检查端口是否真的在监听
lsof -i :18789
# 或
netstat -an | grep 18789

# 2. 检查防火墙
# macOS
sudo pfctl -s rules | grep 18789

# Linux
sudo iptables -L | grep 18789

# 3. 检查配置
openclaw doctor

# 4. 查看详细日志
openclaw logs --follow | grep ERROR

# 5. 如果仍然失败，重启 Gateway
openclaw gateway restart
```

**来源**: `reference/source_gateway_troubleshooting.md`

**判断标准**：

  sleep 60

### 7.3 故障自动恢复

```typescript
// auto-recovery.ts
import { spawn } from 'child_process';

interface RecoveryConfig {
  maxRetries: number;
  retryDelay: number;
  healthCheckInterval: number;
}

class GatewayRecovery {
  private config: RecoveryConfig;
  private retryCount: number = 0;

  constructor(config: RecoveryConfig) {
    this.config = config;
  }

  async start(): Promise<void> {
    // 启动健康检查循环
    setInterval(() => this.healthCheck(), this.config.healthCheckInterval);
  }

  private async healthCheck(): Promise<void> {
    const status = await this.getGatewayStatus();

    if (status !== 'running') {
      console.warn('Gateway is not running, attempting recovery...');
      await this.recover();
    } else {
      // 重置重试计数
      this.retryCount = 0;
    }
  }

  private async recover(): Promise<void> {
    if (this.retryCount >= this.config.maxRetries) {
      console.error('Max retries reached, giving up');
      // 发送告警
      await this.sendAlert('Gateway recovery failed');
      return;
    }

    this.retryCount++;
    console.log(`Recovery attempt ${this.retryCount}/${this.config.maxRetries}`);

    // 1. 停止 Gateway
    await this.runCommand('openclaw', ['gateway', 'stop']);

    // 2. 等待
    await this.sleep(this.config.retryDelay);

    // 3. 运行 doctor
    await this.runCommand('openclaw', ['doctor', '--repair']);

    // 4. 启动 Gateway
    await this.runCommand('openclaw', ['gateway', 'start']);

    // 5. 验证
    await this.sleep(5000);
    const status = await this.getGatewayStatus();

    if (status === 'running') {
      console.log('✓ Gateway recovered successfully');
      this.retryCount = 0;
    } else {
      console.error('✗ Gateway recovery failed');
    }
  }

  private async getGatewayStatus(): Promise<string> {
    try {
      const result = await this.runCommand('openclaw', ['gateway', 'status']);
      return result.includes('running') ? 'running' : 'stopped';
    } catch {
      return 'stopped';
    }
  }

  private runCommand(cmd: string, args: string[]): Promise<string> {
    return new Promise((resolve, reject) => {
      const proc = spawn(cmd, args);
      let output = '';

      proc.stdout.on('data', (data) => {
        output += data.toString();
      });

      proc.on('close', (code) => {
        if (code === 0) {
          resolve(output);
        } else {
          reject(new Error(`Command failed with code ${code}`));
        }
      });
    });
  }

  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }

  private async sendAlert(message: string): Promise<void> {
    // 实现告警通知（邮件、Slack、钉钉等）
    console.error(`ALERT: ${message}`);
  }
}

// 使用
const recovery = new GatewayRecovery({
  maxRetries: 3,
  retryDelay: 5000,
  healthCheckInterval: 60000
});

recovery.start();
```
