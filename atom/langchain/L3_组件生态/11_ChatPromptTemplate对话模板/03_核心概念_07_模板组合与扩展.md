# 核心概念7：模板组合与扩展

> 本文档深入讲解 LangChain ChatPromptTemplate 中的模板组合与扩展机制，包括 + 操作符组合、append/extend 动态添加、切片操作和模板复用策略。

---

## 什么是模板组合与扩展？

**模板组合与扩展**是 ChatPromptTemplate 的高级特性，允许通过多种方式动态构建和修改对话模板。

**核心价值**：
- 支持模块化提示词设计
- 提升模板复用性
- 简化复杂对话流程构建
- 支持运行时动态调整

## 1. 使用 + 操作符组合模板

### 1.1 基础概念

ChatPromptTemplate 重载了 `+` 操作符（`__add__` 方法），使得模板组合变得非常直观。

```python
from langchain_core.prompts import ChatPromptTemplate

# 创建基础模板
base = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 创建用户输入模板
user_input = ChatPromptTemplate.from_messages([
    ("human", "{input}")
])

# 使用 + 操作符组合
combined = base + user_input

# 使用组合后的模板
messages = combined.format_messages(input="Hello")
print(messages)
```

**输出**：
```
[
    SystemMessage(content="You are a helpful assistant."),
    HumanMessage(content="Hello")
]
```

**关键特性**：
- 简单直观的语法
- 自动合并消息列表
- 自动处理变量合并

[来源: reference/source_chatprompt_01.md | 源码分析]

### 1.2 支持的组合方式

ChatPromptTemplate 的 `+` 操作符支持多种类型的组合：

#### 方式1：ChatPromptTemplate + ChatPromptTemplate

```python
from langchain_core.prompts import ChatPromptTemplate

template1 = ChatPromptTemplate.from_messages([
    ("system", "You are a {role}.")
])

template2 = ChatPromptTemplate.from_messages([
    ("human", "{input}")
])

# 组合两个 ChatPromptTemplate
combined = template1 + template2

messages = combined.format_messages(role="assistant", input="Hello")
```

**特点**：
- 合并两个模板的所有消息
- 自动合并 input_variables
- 自动合并 partial_variables（检查冲突）

#### 方式2：ChatPromptTemplate + 元组

```python
from langchain_core.prompts import ChatPromptTemplate

base = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 直接添加元组格式的消息
extended = base + ("human", "{input}")

messages = extended.format_messages(input="Hello")
```

**特点**：
- 元组格式：`(role, template)`
- 支持的角色：system, human, ai
- 自动转换为对应的消息模板

#### 方式3：ChatPromptTemplate + 列表

```python
from langchain_core.prompts import ChatPromptTemplate

base = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 添加多个消息
extended = base + [
    ("human", "Example: {example}"),
    ("ai", "I understand."),
    ("human", "{input}")
]

messages = extended.format_messages(example="test", input="Hello")
```

**特点**：
- 一次添加多个消息
- 支持混合格式
- 适合批量扩展

#### 方式4：ChatPromptTemplate + BaseMessage

```python
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.messages import HumanMessage, SystemMessage

base = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 添加固定的消息对象
extended = base + HumanMessage(content="Fixed message")

messages = extended.format_messages()
```

**特点**：
- 添加固定内容的消息
- 不包含变量
- 适合添加固定的示例或说明

#### 方式5：ChatPromptTemplate + 字符串

```python
from langchain_core.prompts import ChatPromptTemplate

base = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 添加字符串（默认为 human 消息）
extended = base + "{input}"

messages = extended.format_messages(input="Hello")
```

**特点**：
- 字符串默认作为 human 消息
- 简化常见场景的代码
- 等价于 `+ ("human", "{input}")`

[来源: reference/source_chatprompt_01.md | 源码分析]

### 1.3 变量合并机制

当组合模板时，ChatPromptTemplate 会自动处理变量合并：

```python
from langchain_core.prompts import ChatPromptTemplate

# 模板1 有变量 role
template1 = ChatPromptTemplate.from_messages([
    ("system", "You are a {role}.")
])

# 模板2 有变量 input
template2 = ChatPromptTemplate.from_messages([
    ("human", "{input}")
])

# 组合后自动合并变量
combined = template1 + template2

# 查看合并后的变量
print(combined.input_variables)  # ['role', 'input']

# 使用时需要传递所有变量
messages = combined.format_messages(role="assistant", input="Hello")
```

**合并规则**：
- `input_variables` 取并集
- `partial_variables` 检查冲突（同名变量会报错）
- `optional_variables` 也会合并

## 2. append() 和 extend() 动态添加

### 2.1 append() 方法

`append()` 方法用于追加单个消息到模板末尾。

```python
from langchain_core.prompts import ChatPromptTemplate

# 创建基础模板
template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 追加单个消息
template.append(("human", "{input}"))

# 再追加一个消息
template.append(("ai", "I'm ready to help!"))

messages = template.format_messages(input="Hello")
print(messages)
```

**输出**：
```
[
    SystemMessage(content="You are a helpful assistant."),
    HumanMessage(content="Hello"),
    AIMessage(content="I'm ready to help!")
]
```

**关键特性**：
- 直接修改原模板（不返回新实例）
- 支持元组、消息对象等多种格式
- 适合逐步构建模板

### 2.2 extend() 方法

`extend()` 方法用于追加多个消息到模板末尾。

```python
from langchain_core.prompts import ChatPromptTemplate

# 创建基础模板
template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 一次追加多个消息
template.extend([
    ("human", "Example: Hello"),
    ("ai", "Hi there!"),
    ("human", "{input}")
])

messages = template.format_messages(input="How are you?")
```

**关键特性**：
- 批量添加消息
- 直接修改原模板
- 比多次 append 更高效

[来源: reference/source_chatprompt_01.md | 源码分析]

### 2.3 append vs + 操作符

两者的主要区别：

```python
from langchain_core.prompts import ChatPromptTemplate

base = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# 方式1：使用 + 操作符（返回新实例）
new_template = base + ("human", "{input}")
print(len(base.messages))      # 1（原模板未改变）
print(len(new_template.messages))  # 2（新模板）

# 方式2：使用 append（修改原模板）
base.append(("human", "{input}"))
print(len(base.messages))      # 2（原模板已改变）
```

**选择建议**：
- 需要保留原模板：使用 `+` 操作符
- 需要修改原模板：使用 `append()` / `extend()`
- 链式组合：使用 `+` 操作符更清晰

## 3. 切片操作与索引访问

### 3.1 索引访问

ChatPromptTemplate 支持通过索引访问单个消息：

```python
from langchain_core.prompts import ChatPromptTemplate

template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("human", "Example: {example}"),
    ("ai", "I understand."),
    ("human", "{input}")
])

# 访问第一个消息
first_message = template[0]
print(first_message)  # SystemMessagePromptTemplate

# 访问最后一个消息
last_message = template[-1]
print(last_message)  # HumanMessagePromptTemplate
```

**特点**：
- 支持正向索引（0, 1, 2...）
- 支持反向索引（-1, -2, -3...）
- 返回消息模板对象

### 3.2 切片操作

ChatPromptTemplate 支持切片操作来提取部分消息：

```python
from langchain_core.prompts import ChatPromptTemplate

template = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("human", "Example 1: Hello"),
    ("ai", "Hi!"),
    ("human", "Example 2: Goodbye"),
    ("ai", "Bye!"),
    ("human", "{input}")
])

# 提取前3个消息
first_three = template[:3]
print(len(first_three.messages))  # 3

# 提取示例部分（索引1到4）
examples = template[1:5]
print(len(examples.messages))  # 4

# 提取最后2个消息
last_two = template[-2:]
print(len(last_two.messages))  # 2
```

**特点**：
- 返回新的 ChatPromptTemplate 实例
- 保留原模板不变
- 支持标准 Python 切片语法

### 3.3 实际应用场景

#### 场景1：提取系统提示

```python
from langchain_core.prompts import ChatPromptTemplate

full_template = ChatPromptTemplate.from_messages([
    ("system", "You are a {role}."),
    ("human", "Example: {example}"),
    ("ai", "I understand."),
    ("human", "{input}")
])

# 只提取系统提示
system_only = full_template[:1]

messages = system_only.format_messages(role="assistant")
```

#### 场景2：移除示例

```python
from langchain_core.prompts import ChatPromptTemplate

template_with_examples = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("human", "Example: Hello"),
    ("ai", "Hi!"),
    ("human", "{input}")
])

# 移除示例，只保留系统提示和用户输入
template_without_examples = ChatPromptTemplate.from_messages([
    template_with_examples[0],  # system
    template_with_examples[-1]  # human input
])

messages = template_without_examples.format_messages(input="Hello")
```

## 4. 模板复用策略

### 4.1 基础模板 + 动态扩展

创建可复用的基础模板，根据场景动态扩展：

```python
from langchain_core.prompts import ChatPromptTemplate

# 定义基础模板
BASE_TEMPLATE = ChatPromptTemplate.from_messages([
    ("system", "You are a {role} assistant.")
])

# 场景1：简单问答
qa_template = BASE_TEMPLATE + ("human", "{question}")

# 场景2：带示例的问答
qa_with_examples = BASE_TEMPLATE + [
    ("human", "Example: What is Python?"),
    ("ai", "Python is a programming language."),
    ("human", "{question}")
]

# 场景3：带历史记录的对话
from langchain_core.prompts import MessagesPlaceholder

qa_with_history = BASE_TEMPLATE + [
    MessagesPlaceholder("history"),
    ("human", "{question}")
]

# 使用不同的模板
messages1 = qa_template.format_messages(role="Python", question="What is a list?")
messages2 = qa_with_examples.format_messages(role="Python", question="What is a dict?")
messages3 = qa_with_history.format_messages(
    role="Python",
    history=[("human", "Hi"), ("ai", "Hello!")],
    question="What is a tuple?"
)
```

**优势**：
- 统一管理基础配置
- 根据需求灵活扩展
- 减少重复代码

### 4.2 模板工厂模式

使用工厂函数创建不同配置的模板：

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

def create_assistant_template(
    role: str,
    with_examples: bool = False,
    with_history: bool = False
):
    """创建助手模板的工厂函数"""

    # 基础模板
    messages = [("system", f"You are a {role} assistant.")]

    # 添加示例
    if with_examples:
        messages.extend([
            ("human", "Example question"),
            ("ai", "Example answer")
        ])

    # 添加历史记录占位符
    if with_history:
        messages.append(MessagesPlaceholder("history"))

    # 添加用户输入
    messages.append(("human", "{input}"))

    return ChatPromptTemplate.from_messages(messages)

# 使用工厂函数
simple_template = create_assistant_template("Python")
example_template = create_assistant_template("Python", with_examples=True)
history_template = create_assistant_template("Python", with_history=True)
full_template = create_assistant_template("Python", with_examples=True, with_history=True)
```

**优势**：
- 集中管理模板创建逻辑
- 支持参数化配置
- 易于测试和维护

### 4.3 模板继承与组合

通过组合实现模板继承：

```python
from langchain_core.prompts import ChatPromptTemplate

# 基础助手模板
BASE_ASSISTANT = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant.")
])

# Python 专家模板（继承基础模板）
PYTHON_EXPERT = BASE_ASSISTANT + [
    ("system", "You specialize in Python programming."),
    ("system", "Always provide code examples.")
]

# RAG 助手模板（继承基础模板）
RAG_ASSISTANT = BASE_ASSISTANT + [
    ("system", "Answer based on the provided context."),
    ("human", "Context: {context}"),
    ("human", "Question: {question}")
]

# 使用继承的模板
python_messages = (PYTHON_EXPERT + ("human", "{input}")).format_messages(
    input="How to use list comprehension?"
)

rag_messages = RAG_ASSISTANT.format_messages(
    context="Python is a programming language.",
    question="What is Python?"
)
```

**优势**：
- 实现模板继承
- 支持多层组合
- 提升代码复用性

## 5. 高级技巧

### 5.1 条件组合

根据条件动态组合模板：

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

def create_conditional_template(
    include_system: bool = True,
    include_examples: bool = False,
    include_history: bool = False
):
    """根据条件创建模板"""

    messages = []

    if include_system:
        messages.append(("system", "You are a helpful assistant."))

    if include_examples:
        messages.extend([
            ("human", "Example: Hello"),
            ("ai", "Hi there!")
        ])

    if include_history:
        messages.append(MessagesPlaceholder("history"))

    messages.append(("human", "{input}"))

    return ChatPromptTemplate.from_messages(messages)

# 根据不同条件创建模板
minimal_template = create_conditional_template(include_system=False)
full_template = create_conditional_template(
    include_system=True,
    include_examples=True,
    include_history=True
)
```

### 5.2 模板链式构建

使用链式调用构建复杂模板：

```python
from langchain_core.prompts import ChatPromptTemplate

# 链式构建
template = (
    ChatPromptTemplate.from_messages([("system", "You are a {role}.")])
    + ("human", "Example: {example}")
    + ("ai", "I understand.")
    + ("human", "{input}")
)

messages = template.format_messages(
    role="assistant",
    example="Hello",
    input="How are you?"
)
```

**优势**：
- 代码更简洁
- 逻辑更清晰
- 易于阅读和维护

### 5.3 模板组合与 Partial Variables

组合模板时结合 partial variables：

```python
from langchain_core.prompts import ChatPromptTemplate

# 创建基础模板并预填充
base = ChatPromptTemplate.from_messages([
    ("system", "You are a {role} assistant.")
]).partial(role="Python")

# 扩展模板
extended = base + [
    ("human", "Language: {language}"),
    ("human", "{input}")
]

# 再次预填充
final = extended.partial(language="English")

# 使用时只需传递 input
messages = final.format_messages(input="Hello")
```

**注意事项**：
- 组合后可以继续 partial
- 注意 partial variables 的冲突
- 参考核心概念6的兼容性问题

## 7. 注意事项与最佳实践

### 7.1 变量冲突处理

组合模板时注意变量冲突：

```python
from langchain_core.prompts import ChatPromptTemplate

template1 = ChatPromptTemplate.from_messages([
    ("system", "Role: {role}")
]).partial(role="assistant")

template2 = ChatPromptTemplate.from_messages([
    ("system", "Role: {role}")
]).partial(role="expert")

# ⚠️ 组合时可能出现 partial variables 冲突
# 参考核心概念6的兼容性问题
try:
    combined = template1 + template2
except Exception as e:
    print(f"Error: {e}")
```

**建议**：
- 避免在组合的模板中使用相同的 partial variable
- 在组合后再进行 partial
- 使用不同的变量名

### 7.2 性能考虑

```python
from langchain_core.prompts import ChatPromptTemplate

# ❌ 低效：多次使用 + 操作符
template = ChatPromptTemplate.from_messages([("system", "System")])
template = template + ("human", "Message 1")
template = template + ("human", "Message 2")
template = template + ("human", "Message 3")

# ✅ 高效：使用 extend 批量添加
template = ChatPromptTemplate.from_messages([("system", "System")])
template.extend([
    ("human", "Message 1"),
    ("human", "Message 2"),
    ("human", "Message 3")
])

# ✅ 或者一次性创建
template = ChatPromptTemplate.from_messages([
    ("system", "System"),
    ("human", "Message 1"),
    ("human", "Message 2"),
    ("human", "Message 3")
])
```

### 7.3 模板可读性

```python
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

# ✅ 清晰的模板结构
template = ChatPromptTemplate.from_messages([
    # 系统提示
    ("system", "You are a {role} assistant."),

    # Few-shot 示例
    ("human", "Example: Hello"),
    ("ai", "Hi there!"),

    # 对话历史
    MessagesPlaceholder("history"),

    # 用户输入
    ("human", "{input}")
])
```

**建议**：
- 添加注释说明每部分的作用
- 使用空行分隔不同部分
- 保持一致的代码风格

## 8. 总结

### 核心要点

1. **+ 操作符组合**：支持多种类型的组合，自动合并变量
2. **append/extend**：动态修改模板，适合运行时构建
3. **切片操作**：提取部分消息，支持标准 Python 切片语法
4. **模板复用**：通过基础模板 + 动态扩展实现复用
5. **工厂模式**：集中管理模板创建逻辑
6. **链式构建**：使用链式调用构建复杂模板

### 最佳实践

- ✅ 使用 + 操作符进行不可变组合
- ✅ 使用 extend 批量添加消息
- ✅ 创建可复用的基础模板
- ✅ 使用工厂函数管理模板创建
- ✅ 添加注释提升可读性
- ❌ 避免 partial variables 冲突
- ❌ 避免多次使用 + 操作符（性能）
