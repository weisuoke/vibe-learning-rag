## 4.1 使用 @tool 装饰器创建工具

### 最简单的工具

```python
from langchain.tools import tool

@tool
def calculator(expression: str) -> str:
    """计算数学表达式

    Args:
        expression: 数学表达式，如 "2 + 2" 或 "10 * 5"

    Returns:
        计算结果
    """
    try:
        result = eval(expression)
        return f"结果: {result}"
    except Exception as e:
        return f"计算错误: {e}"

# 测试工具
print(calculator.invoke({"expression": "2 + 2"}))
# 输出: 结果: 4
```

**关键要点**：
- 函数必须有**类型提示**（`expression: str` 和 `-> str`）
- 使用 **Google-style docstring** 描述功能和参数
- 工具名称默认为函数名（`calculator`）
- 使用 `.invoke()` 方法调用工具

**这个工具可以做什么**：
- ✅ 被 Agent 自动调用
- ✅ 自动生成 JSON schema
- ✅ 参数验证（必须是字符串）
- ✅ 与 LangChain Agent 无缝集成

## 4.2 定义复杂参数（Pydantic Schema）

### 多参数工具

```python
from pydantic import BaseModel, Field
from typing import Literal

class WeatherInput(BaseModel):
    """天气查询输入参数"""
    location: str = Field(description="城市名称，如 '北京' 或 'Beijing'")
    units: Literal["celsius", "fahrenheit"] = Field(
        default="celsius",
        description="温度单位"
    )

@tool(args_schema=WeatherInput)
def get_weather(location: str, units: str = "celsius") -> str:
    """获取指定城市的天气信息

    Args:
        location: 城市名称
        units: 温度单位（celsius 或 fahrenheit）
    """
    # 模拟 API 调用
    temp = 22 if units == "celsius" else 72
    return f"{location} 当前温度: {temp}°{units[0].upper()}"

# 测试
print(get_weather.invoke({"location": "北京"}))
# 输出: 北京 当前温度: 22°C

print(get_weather.invoke({"location": "New York", "units": "fahrenheit"}))
# 输出: New York 当前温度: 72°F
```

**关键要点**：
- 使用 `BaseModel` 定义参数结构
- 使用 `Field()` 添加参数描述（帮助 LLM 理解）
- 使用 `Literal` 限制可选值（防止无效输入）
- 提供默认值提升易用性

**Pydantic 常用类型**：
```python
from typing import List, Dict, Optional, Literal
from pydantic import BaseModel, Field

class ToolInput(BaseModel):
    # 基础类型
    name: str = Field(description="名称")
    age: int = Field(description="年龄", ge=0, le=150)  # 范围限制
    score: float = Field(description="分数")
    is_active: bool = Field(description="是否激活")

    # 可选参数
    email: Optional[str] = Field(default=None, description="邮箱（可选）")

    # 枚举类型
    status: Literal["pending", "active", "inactive"] = Field(
        default="pending",
        description="状态"
    )

    # 列表和字典
    tags: List[str] = Field(default=[], description="标签列表")
    metadata: Dict[str, str] = Field(default={}, description="元数据")
```

## 4.3 基础错误处理

### 使用 ToolException

```python
from langchain.tools import tool, ToolException
import requests

@tool
def search_api(query: str) -> str:
    """搜索 API（带错误处理）

    Args:
        query: 搜索关键词
    """
    try:
        response = requests.get(
            f"https://api.example.com/search?q={query}",
            timeout=5
        )
        response.raise_for_status()
        return response.json()["results"]

    except requests.Timeout:
        # 可恢复错误 → ToolException（不会中断 Agent）
        raise ToolException("API 超时，请稍后重试")

    except requests.HTTPError as e:
        if e.response.status_code == 404:
            raise ToolException(f"未找到 '{query}' 的结果")
        else:
            raise ToolException(f"API 错误: {e.response.status_code}")

    except KeyError:
        raise ToolException("API 返回数据格式错误")
```

**错误处理原则**：
- **可恢复错误** → 使用 `ToolException`
  - 用户输入错误
  - 外部服务临时不可用
  - 数据不存在
- **不可恢复错误** → 使用普通异常
  - 配置错误（API 密钥缺失）
  - 权限不足
  - 代码逻辑错误

**简化版错误处理**：
```python
@tool(handle_tool_error=True)  # 自动捕获所有错误
def simple_tool(param: str) -> str:
    """简单工具（自动错误处理）"""
    # 任何异常都会被捕获，返回通用错误消息
    raise ValueError("Some error")
    # Agent 看到: "Error: Some error"
```

## 4.4 与 Agent 集成

### 创建 Agent 并使用工具

```python
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain import hub

# 1. 定义工具
@tool
def get_current_time() -> str:
    """获取当前时间"""
    from datetime import datetime
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

@tool
def calculator(expression: str) -> str:
    """计算数学表达式"""
    return str(eval(expression))

# 2. 创建工具列表
tools = [get_current_time, calculator]

# 3. 创建 LLM
llm = ChatOpenAI(model="gpt-4", temperature=0)

# 4. 加载 prompt 模板
prompt = hub.pull("hwchase17/openai-functions-agent")

# 5. 创建 Agent
agent = create_openai_functions_agent(llm, tools, prompt)

# 6. 创建 Agent Executor
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,  # 显示执行过程
)

# 7. 运行 Agent
result = agent_executor.invoke({
    "input": "现在几点？2 + 2 等于多少？"
})

print(result["output"])
```

**执行流程**：
```
User: "现在几点？2 + 2 等于多少？"
    ↓
Agent: 思考 → 需要调用 get_current_time 和 calculator
    ↓
Agent: 调用 get_current_time()
Tool: 返回 "2026-03-02 14:30:00"
    ↓
Agent: 调用 calculator("2 + 2")
Tool: 返回 "4"
    ↓
Agent: 整合结果 → "现在是 2026-03-02 14:30:00，2 + 2 等于 4"
```

## 4.5 异步工具（可选）

### 异步 API 调用

```python
import httpx
from langchain.tools import tool

@tool
async def async_weather(location: str) -> str:
    """异步获取天气（性能更好）

    Args:
        location: 城市名称
    """
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"https://api.weather.com/v1/current?q={location}",
            timeout=5.0
        )
        data = response.json()
        return f"{location}: {data['temp']}°C, {data['condition']}"

# 异步调用
import asyncio
result = asyncio.run(async_weather.ainvoke({"location": "北京"}))
print(result)
```

**何时使用异步**：
- ✅ 调用外部 API（网络 I/O）
- ✅ 数据库查询
- ✅ 文件读写
- ❌ CPU 密集型计算（用同步即可）

**性能对比**：
```python
# 同步：串行执行（慢）
result1 = tool1.invoke(...)  # 等待 1 秒
result2 = tool2.invoke(...)  # 等待 1 秒
# 总耗时: 2 秒

# 异步：并行执行（快）
result1, result2 = await asyncio.gather(
    tool1.ainvoke(...),  # 并行执行
    tool2.ainvoke(...),  # 并行执行
)
# 总耗时: 1 秒
```

## 完整示例：RAG 文档检索工具

### 实战：构建一个可用的 RAG 工具

```python
from langchain.tools import tool, ToolException
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from pydantic import BaseModel, Field
from typing import Optional

# 1. 定义输入 schema
class SearchInput(BaseModel):
    """文档检索输入参数"""
    query: str = Field(description="搜索查询，用自然语言描述需求")
    top_k: int = Field(
        default=3,
        description="返回结果数量",
        ge=1,
        le=10
    )
    filter_category: Optional[str] = Field(
        default=None,
        description="文档分类过滤（可选）"
    )

# 2. 初始化向量数据库（假设已有数据）
embeddings = OpenAIEmbeddings()
vector_store = Chroma(
    persist_directory="./chroma_db",
    embedding_function=embeddings
)

# 3. 创建检索工具
@tool(args_schema=SearchInput)
def search_documents(
    query: str,
    top_k: int = 3,
    filter_category: Optional[str] = None
) -> str:
    """从知识库检索相关文档

    Args:
        query: 搜索查询
        top_k: 返回结果数量（1-10）
        filter_category: 文档分类过滤（可选）

    Returns:
        检索到的文档内容
    """
    try:
        # 构建过滤条件
        filter_dict = None
        if filter_category:
            filter_dict = {"category": filter_category}

        # 执行检索
        results = vector_store.similarity_search(
            query,
            k=top_k,
            filter=filter_dict
        )

        # 检查结果
        if not results:
            raise ToolException(
                f"未找到与 '{query}' 相关的文档，请尝试更通用的关键词"
            )

        # 格式化输出
        output = f"找到 {len(results)} 个相关文档:\n\n"
        for i, doc in enumerate(results, 1):
            output += f"文档 {i}:\n{doc.page_content}\n\n"

        return output

    except Exception as e:
        # 数据库连接错误等
        raise ToolException(f"检索失败: {str(e)}")

# 4. 测试工具
print(search_documents.invoke({
    "query": "如何使用 LangChain 创建 Agent？",
    "top_k": 2
}))
```

**这个工具的特点**：
- ✅ 完整的参数验证（top_k 范围限制）
- ✅ 可选的分类过滤
- ✅ 友好的错误处理
- ✅ 格式化的输出
- ✅ 可直接集成到 Agent

### 工具创建速查表

| 场景 | 代码模板 | 适用情况 |
|------|----------|----------|
| **简单工具** | `@tool`<br>`def func(param: str) -> str:` | 单参数，简单逻辑 |
| **多参数工具** | `@tool(args_schema=Schema)`<br>`def func(...) -> str:` | 多参数，需要验证 |
| **异步工具** | `@tool`<br>`async def func(...) -> str:` | 外部 API 调用 |
| **错误处理** | `raise ToolException("...")` | 可恢复错误 |
| **自定义名称** | `@tool("custom_name")` | 工具名称与函数名不同 |

### 常见问题速查

**Q: 工具没有被 Agent 调用？**
```python
# 检查清单：
# 1. 是否有类型提示？
@tool
def bad_tool(param):  # ❌ 缺少类型提示
    return "result"

@tool
def good_tool(param: str) -> str:  # ✅ 有类型提示
    return "result"

# 2. 描述是否清晰？
@tool
def bad_tool(param: str) -> str:
    """Tool"""  # ❌ 描述太简单
    return "result"

@tool
def good_tool(param: str) -> str:
    """搜索文档。当用户询问知识库内容时使用此工具。"""  # ✅ 描述清晰
    return "result"
```

**Q: 如何调试工具？**

**Q: 如何处理敏感信息？**
```python
import os
from langchain.tools import tool

@tool
def secure_tool(query: str) -> str:
    """安全工具（不暴露 API 密钥）"""
    # ✅ 从环境变量读取
    api_key = os.getenv("API_KEY")

    # ❌ 不要硬编码
    # api_key = "sk-1234567890"

    # ❌ 不要在返回值中暴露
    # return f"API Key: {api_key}"

    # ✅ 只返回业务结果
    return call_api(query, api_key)
```

