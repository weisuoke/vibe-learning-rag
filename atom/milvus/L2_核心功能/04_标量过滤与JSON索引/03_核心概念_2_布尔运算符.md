# 核心概念 2：布尔运算符

> **目标**：掌握 Milvus 布尔运算符的使用方法和优化技巧

---

## 概述

布尔运算符是构建复杂过滤条件的核心工具，通过 AND、OR、NOT 的组合，可以实现任意复杂的业务逻辑。

**核心价值**：
- 组合多个过滤条件
- 实现复杂的业务规则
- 支持嵌套逻辑表达式
- 位图操作优化性能

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

---

## 1. AND 操作符

### 1.1 基本语法

**语法**：
```python
"condition1 AND condition2"
```

**语义**：
- 所有条件都必须为真
- 结果是条件的交集
- 短路求值（左侧为假则不计算右侧）

### 1.2 使用示例

**简单组合**：
```python
# 年龄和城市同时满足
"age > 25 AND city == '北京'"

# 价格范围
"price >= 100 AND price <= 500"

# 多个条件
"age > 25 AND city == '北京' AND is_vip == true"
```

**实际场景**：
```python
# 电商场景：有库存且价格合适的商品
filter = "stock > 0 AND price >= 100 AND price <= 500"

# 用户画像：特定年龄段且活跃的用户
filter = "age >= 25 AND age <= 35 AND last_login_at > 1640995200"

# RAG场景：特定主题且高质量的文档
filter = "topic == 'AI' AND quality_score >= 0.8 AND language == 'zh'"
```

### 1.3 性能特点

**选择性优化**：
```python
# 好的写法（选择性高的条件在前）
"stock > 0 AND price >= 100 AND price <= 500"
# 假设 stock > 0 只保留10%数据，可以快速过滤掉90%

# 不好的写法（选择性低的条件在前）
"price >= 100 AND price <= 500 AND stock > 0"
# 假设价格范围保留80%数据，需要处理更多数据
```

**原理**：
- Milvus 从左到右执行 AND 条件
- 选择性高的条件先执行可以快速减少候选集
- 减少后续条件的计算量

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

## 2. OR 操作符

### 2.1 基本语法

**语法**：
```python
"condition1 OR condition2"
```

**语义**：
- 至少一个条件为真
- 结果是条件的并集
- 不支持短路求值（所有条件都会计算）

### 2.2 使用示例

**简单组合**：
```python
# 城市选择
"city == '北京' OR city == '上海'"

# 状态选择
"status == 'active' OR status == 'pending'"

# 多个条件
"color == 'red' OR color == 'blue' OR color == 'green'"
```

**实际场景**：
```python
# 电商场景：多个品牌选择
filter = "brand == 'Apple' OR brand == 'Samsung' OR brand == 'Huawei'"

# 用户画像：多个城市的用户
filter = "city == '北京' OR city == '上海' OR city == '深圳' OR city == '广州'"

# RAG场景：多个主题的文档
filter = "topic == 'AI' OR topic == 'ML' OR topic == 'NLP'"
```

### 2.3 优化建议

**使用 IN 替代多个 OR**：
```python
# 不推荐（多个 OR）
"city == '北京' OR city == '上海' OR city == '深圳'"

# 推荐（使用 IN）
"city IN ['北京', '上海', '深圳']"
```

**原因**：
- IN 操作符性能更好
- 代码更简洁易读
- 支持索引优化

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

## 3. NOT 操作符

### 3.1 基本语法

**语法**：
```python
"NOT condition"
```

**语义**：
- 否定条件
- 排除特定值
- 结果是条件的补集

### 3.2 使用示例

**简单否定**：
```python
# 排除特定值
"NOT color == 'green'"

# 排除特定状态
"NOT status == 'deleted'"

# 排除特定范围
"NOT (age >= 60)"
```

**实际场景**：
```python
# 电商场景：排除已下架商品
filter = "NOT status == 'discontinued'"

# 用户画像：排除已注销用户
filter = "NOT status == 'deleted'"

# RAG场景：排除低质量文档
filter = "NOT quality_score < 0.5"
```

### 3.3 等价写法

**NOT 与不等于**：
```python
# 使用 NOT
"NOT color == 'green'"

# 使用 !=（推荐）
"color != 'green'"
```

**NOT 与范围**：
```python
# 使用 NOT
"NOT (age >= 60)"

# 使用 <（推荐）
"age < 60"
```

**建议**：
- 优先使用 `!=`、`<`、`>` 等直接操作符
- NOT 主要用于复杂表达式的否定

## 4. 复合条件组合

### 4.1 AND 与 OR 组合

**基本组合**：
```python
# 年龄大于25，且在北京或上海
"age > 25 AND (city == '北京' OR city == '上海')"

# 价格合适，且（有库存或可预订）
"price <= 500 AND (stock > 0 OR pre_order == true)"
```

**复杂组合**：
```python
# 电商场景：多维度筛选
filter = '''
    (price >= 100 AND price <= 500)
    AND (brand IN ['Apple', 'Samsung', 'Huawei'])
    AND (stock > 0 OR pre_order == true)
    AND NOT status == 'discontinued'
'''

# 用户画像：精准定位
filter = '''
    (age >= 25 AND age <= 35)
    AND (city IN ['北京', '上海', '深圳'])
    AND (vip_level >= 3 OR total_purchase > 10000)
    AND NOT status == 'deleted'
'''
```

### 4.2 优先级规则

**默认优先级**：
```
1. 括号 ()
2. 比较操作符 (==, !=, >, <, >=, <=)
3. NOT
4. AND
5. OR
```

**示例**：
```python
# 不使用括号（可能产生歧义）
"age > 25 AND city == '北京' OR city == '上海'"
# 实际执行：(age > 25 AND city == '北京') OR city == '上海'
# 结果：年龄>25且在北京，或者在上海（不限年龄）

# 使用括号（明确意图）
"age > 25 AND (city == '北京' OR city == '上海')"
# 结果：年龄>25，且在北京或上海
```

**建议**：
- 始终使用括号明确优先级
- 避免依赖默认优先级
- 提高代码可读性

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

## 5. 位图操作优化

### 5.1 底层实现

**表达式树**：
```
filter = "age > 25 AND (city == '北京' OR city == '上海')"

解析为表达式树：
        AND
       /   \
    age>25  OR
           /  \
    city==北京 city==上海
```

**执行流程**：
```cpp
// 递归执行表达式树
VectorPtr eval(ExprNode* node) {
    if (node->type == COMPARE) {
        return eval_compare(node);
    } else if (node->type == AND) {
        auto left = eval(node->left);
        auto right = eval(node->right);
        return bitwise_and(left, right);
    } else if (node->type == OR) {
        auto left = eval(node->left);
        auto right = eval(node->right);
        return bitwise_or(left, right);
    }
}
```

[来源: reference/source_expression.md | LogicalBinaryExpr.cpp]

### 5.2 位图表示

**TargetBitmap**：
```cpp
// 使用位图表示结果
TargetBitmap result(n, false);  // n个文档，初始为false

// AND操作：位图与运算
result = left_bitmap & right_bitmap;

// OR操作：位图或运算
result = left_bitmap | right_bitmap;

// NOT操作：位图取反
result = ~bitmap;
```

**示例**：
```
假设有8个文档：

age > 25:        [1, 1, 0, 1, 0, 1, 1, 0]
city == '北京':  [1, 0, 1, 1, 0, 0, 1, 1]

AND 结果:        [1, 0, 0, 1, 0, 0, 1, 0]
OR 结果:         [1, 1, 1, 1, 0, 1, 1, 1]
```

**性能优势**：
- 使用 SIMD 指令加速位图操作
- 内存占用小（每个文档1位）
- 支持快速的集合运算

[来源: reference/source_expression.md | LogicalBinaryExpr.cpp]

## 7. 常见错误

### 7.1 优先级错误

**错误示例**：
```python
# 错误：没有使用括号
"age > 25 AND city == '北京' OR city == '上海'"
# 实际执行：(age > 25 AND city == '北京') OR city == '上海'
# 结果：年龄>25且在北京，或者在上海（不限年龄）

# 正确：使用括号明确优先级
"age > 25 AND (city == '北京' OR city == '上海')"
# 结果：年龄>25，且在北京或上海
```

### 7.2 逻辑错误

**错误示例**：
```python
# 错误：矛盾条件
"age > 30 AND age < 25"  # ❌ 永远为 false

# 错误：冗余条件
"age > 25 OR age > 30"  # ❌ 等价于 age > 25

# 正确
"age >= 25 AND age <= 35"  # ✅
```

### 7.3 性能错误

**错误示例**：
```python
# 错误：选择性低的条件在前
"price >= 100 AND price <= 500 AND stock > 0"
# 假设价格范围保留80%数据，需要处理更多数据

# 正确：选择性高的条件在前
"stock > 0 AND price >= 100 AND price <= 500"
# 假设 stock > 0 只保留10%数据，可以快速过滤掉90%
```

## 8. 性能优化

### 8.1 条件顺序优化

**原则**：
- 选择性高的条件放在前面
- 计算成本低的条件放在前面
- 有索引的字段优先

**示例**：
```python
# 好的写法
"stock > 0 AND brand == 'Apple' AND price >= 100 AND price <= 500"
# 假设：stock > 0 (10%), brand == 'Apple' (5%), price range (80%)

# 不好的写法
"price >= 100 AND price <= 500 AND brand == 'Apple' AND stock > 0"
```

### 8.2 使用 IN 替代多个 OR

**原则**：
- 3个以上的 OR 条件考虑使用 IN
- IN 支持索引优化
- 代码更简洁

**示例**：
```python
# 不推荐
"city == '北京' OR city == '上海' OR city == '深圳' OR city == '广州'"

# 推荐
"city IN ['北京', '上海', '深圳', '广州']"
```

### 8.3 避免复杂的 NOT

**原则**：
- 优先使用 `!=`、`<`、`>` 等直接操作符
- 避免对复杂表达式使用 NOT
- NOT 可能导致全表扫描

**示例**：
```python
# 不推荐
"NOT (age >= 60)"

# 推荐
"age < 60"
```

## 9. 类比理解

### 前端开发类比

**JavaScript 逻辑运算**：
```javascript
// JavaScript
const filtered = users.filter(u =>
    u.age > 25 &&
    (u.city === '北京' || u.city === '上海') &&
    u.active === true
);

// Milvus
filter = "age > 25 AND (city == '北京' OR city == '上海') AND active == true"
```

### SQL 类比

**SQL WHERE 子句**：
```sql
-- SQL
SELECT * FROM users
WHERE age > 25
  AND (city = '北京' OR city = '上海')
  AND active = true;

-- Milvus
filter = "age > 25 AND (city == '北京' OR city == '上海') AND active == true"
```

### 生活类比

**餐厅筛选**：
```
找餐厅的条件：
- AND：价格100-500元 且 距离<3km 且 营业中
- OR：川菜 或 粤菜 或 湘菜
- NOT：不要 火锅

组合：
"价格100-500元 AND 距离<3km AND 营业中 AND (川菜 OR 粤菜 OR 湘菜) AND NOT 火锅"
```
