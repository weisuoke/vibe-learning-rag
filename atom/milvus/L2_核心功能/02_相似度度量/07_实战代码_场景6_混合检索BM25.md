# 实战代码 - 场景6：混合检索BM25

## 场景概述

**目标**：实现 Milvus 2.5+ BM25 全文检索与向量语义检索的混合检索系统

**适用场景**:
- RAG 系统（推荐模式）
- 文档检索系统

## 完整可运行代码

```python
"""
Milvus BM25 混合检索实战
演示：全文检索 + 向量检索 + RRF 重排序

基于 2025-2026 最佳实践：
- Milvus 2.5+ BM25 Function
- SPARSE_INVERTED_INDEX
- RRFRanker 重排序
- 生产级混合检索

参考来源：
- Milvus 2.5 Hybrid Search Tutorial (December 17, 2024)
- Milvus Full Text Search Documentation
"""

import os
from typing import List, Dict, Tuple
from openai import OpenAI
from pymilvus import (
    MilvusClient,
    DataType,
    Function,
    FunctionType,
    AnnSearchRequest,
    RRFRanker
)

# ===== 1. 混合检索系统类 =====
print("=== Milvus BM25 混合检索系统 ===\n")

class HybridSearchSystem:
    """
    BM25 + 向量混合检索系统

    架构：
    1. 文本字段（enable_analyzer=True）
    2. 稠密向量（语义检索）
    3. 稀疏向量（BM25 全文检索，自动生成）
    4. RRF 重排序
    """

    def __init__(
        self,
        openai_client: OpenAI,
        milvus_client: MilvusClient,
        collection_name: str = "hybrid_search_demo"
    ):
        self.openai_client = openai_client
        self.milvus_client = milvus_client
        self.collection_name = collection_name

        # 模型配置
        self.embedding_model = "text-embedding-3-small"
        self.embedding_dim = 1536

        print(f"✅ 混合检索系统初始化完成")
        print(f"  - Embedding 模型: {self.embedding_model}")
        print(f"  - Collection: {self.collection_name}")

    def _get_embeddings(self, texts: List[str]) -> List[List[float]]:
        """生成文本 Embeddings"""
        if not texts:
            return []
        response = self.openai_client.embeddings.create(
            input=texts,
            model=self.embedding_model
        )
        return [embedding.embedding for embedding in response.data]

    def create_collection(self):
        """
        创建支持混合检索的 Collection

        关键配置：
        - content: VARCHAR with enable_analyzer=True
        - dense_vector: FLOAT_VECTOR for semantic search
        - sparse_vector: SPARSE_FLOAT_VECTOR for BM25 (auto-generated)
        - BM25 Function: auto-generate sparse vectors
        """
        print(f"\n=== 创建 Collection ===")

        # 删除已存在的 collection
        if self.milvus_client.has_collection(self.collection_name):
            self.milvus_client.drop_collection(self.collection_name)

        # 定义 schema
        schema = self.milvus_client.create_schema()

        # 主键
        schema.add_field(
            field_name="id",
            datatype=DataType.VARCHAR,
            is_primary=True,
            auto_id=True,
            max_length=100
        )

        # 文本字段（启用分析器）
        schema.add_field(
            field_name="content",
            datatype=DataType.VARCHAR,
            max_length=65535,
            enable_analyzer=True,  # 启用词法分析
            enable_match=True       # 启用文本匹配
        )

        # 稠密向量（语义检索）
        schema.add_field(
            field_name="dense_vector",
            datatype=DataType.FLOAT_VECTOR,
            dim=self.embedding_dim
        )

        # 稀疏向量（BM25，自动生成）
        schema.add_field(
            field_name="sparse_vector",
            datatype=DataType.SPARSE_FLOAT_VECTOR
        )

        # 元数据
        schema.add_field(
            field_name="metadata",
            datatype=DataType.JSON
        )

        # 定义 BM25 Function
        bm25_function = Function(
            name="bm25",
            function_type=FunctionType.BM25,
            input_field_names=["content"],
            output_field_names="sparse_vector"
        )

        schema.add_function(bm25_function)

        # 创建索引
        index_params = self.milvus_client.prepare_index_params()

        # 稀疏向量索引（BM25）
        index_params.add_index(
            field_name="sparse_vector",
            index_type="SPARSE_INVERTED_INDEX",
            metric_type="BM25"
        )

        # 稠密向量索引（语义检索）
        index_params.add_index(
            field_name="dense_vector",
            index_type="AUTOINDEX",
            metric_type="COSINE"
        )

        # 创建 collection
        self.milvus_client.create_collection(
            collection_name=self.collection_name,
            schema=schema,
            index_params=index_params
        )

        print(f"✅ Collection 创建完成")
        print(f"  - 文本字段: content (enable_analyzer=True)")
        print(f"  - 稠密向量: dense_vector (COSINE)")
        print(f"  - 稀疏向量: sparse_vector (BM25, auto-generated)")

    def insert_documents(self, documents: List[Dict]):
        """
        插入文档

        参数：
        - documents: [{"content": str, "metadata": dict}, ...]

        注意：只需提供 content 和 metadata，sparse_vector 自动生成
        """
        print(f"\n=== 插入文档 ===")
        print(f"文档数量: {len(documents)}")

        # 生成 embeddings
        texts = [doc["content"] for doc in documents]
        embeddings = self._get_embeddings(texts)

        # 准备数据
        entities = []
        for i, doc in enumerate(documents):
            entities.append({
                "content": doc["content"],
                "dense_vector": embeddings[i],
                "metadata": doc.get("metadata", {})
            })

        # 插入数据
        self.milvus_client.insert(
            collection_name=self.collection_name,
            data=entities
        )

        print(f"✅ 文档插入完成: {len(documents)} 条")

    def search_semantic(self, query: str, top_k: int = 5) -> List[Dict]:
        """
        语义检索（仅使用稠密向量）

        返回：[{"id": str, "distance": float, "content": str, "metadata": dict}, ...]
        """
        query_embedding = self._get_embeddings([query])[0]

        results = self.milvus_client.search(
            collection_name=self.collection_name,
            data=[query_embedding],
            anns_field="dense_vector",
            limit=top_k,
            output_fields=["content", "metadata"]
        )

        return [
            {
                "id": hit["id"],
                "distance": hit["distance"],
                "content": hit["entity"]["content"],
                "metadata": hit["entity"]["metadata"]
            }
            for hit in results[0]
        ]

    def search_fulltext(self, query: str, top_k: int = 5) -> List[Dict]:
        """
        全文检索（仅使用 BM25 稀疏向量）

        返回：[{"id": str, "distance": float, "content": str, "metadata": dict}, ...]
        """
        results = self.milvus_client.search(
            collection_name=self.collection_name,
            data=[query],
            anns_field="sparse_vector",
            limit=top_k,
            output_fields=["content", "metadata"]
        )

        return [
            {
                "id": hit["id"],
                "distance": hit["distance"],
                "content": hit["entity"]["content"],
                "metadata": hit["entity"]["metadata"]
            }
            for hit in results[0]
        ]

    def search_hybrid(self, query: str, top_k: int = 5) -> List[Dict]:
        """
        混合检索（BM25 + 语义检索 + RRF 重排序）

        返回：[{"id": str, "distance": float, "content": str, "metadata": dict}, ...]
        """
        query_embedding = self._get_embeddings([query])[0]

        # BM25 搜索请求
        sparse_search_params = {"metric_type": "BM25"}
        sparse_request = AnnSearchRequest(
            [query],
            "sparse_vector",
            sparse_search_params,
            limit=top_k
        )

        # 稠密向量搜索请求
        dense_search_params = {"metric_type": "COSINE"}
        dense_request = AnnSearchRequest(
            [query_embedding],
            "dense_vector",
            dense_search_params,
            limit=top_k
        )

        # 混合检索 + RRF 重排序
        results = self.milvus_client.hybrid_search(
            self.collection_name,
            [sparse_request, dense_request],
            ranker=RRFRanker(),  # Reciprocal Rank Fusion
            limit=top_k,
            output_fields=["content", "metadata"]
        )

        return [
            {
                "id": hit["id"],
                "distance": hit["distance"],
                "content": hit["entity"]["content"],
                "metadata": hit["entity"]["metadata"]
            }
            for hit in results[0]
        ]
```

## 核心要点

### 1. BM25 Function 配置（Milvus 2.5+）

**关键组件**（来源：Milvus 2.5 Tutorial, December 17, 2024）:

```python
# 文本字段（启用分析器）
schema.add_field(
    field_name="content",
    datatype=DataType.VARCHAR,
    enable_analyzer=True,  # 启用词法分析
    enable_match=True       # 启用文本匹配
)

# 稀疏向量（自动生成）
schema.add_field(
    field_name="sparse_vector",
    datatype=DataType.SPARSE_FLOAT_VECTOR
)

# BM25 Function
bm25_function = Function(
    name="bm25",
    function_type=FunctionType.BM25,
    input_field_names=["content"],
    output_field_names="sparse_vector"
)
```

**自动生成**: 用户只需提供文本，Milvus 自动生成稀疏向量。

### 2. 索引配置

| 字段类型 | 索引类型 | 度量类型 | 用途 |
|---------|---------|---------|------|
| **sparse_vector** | SPARSE_INVERTED_INDEX | BM25 | 全文检索 |
| **dense_vector** | AUTOINDEX / HNSW | COSINE / IP | 语义检索 |

### 3. RRFRanker 重排序

**公式**:
```
RRF_score(doc) = Σ (1 / (k + rank_i))
```

**参数**:
- `k`: 常数（默认 60）
- `rank_i`: 文档在第 i 个搜索结果中的排名

**优势**:
- 无需分数归一化
- 自动平衡多个检索源
- 简单高效

### 4. 三种检索方式对比

| 检索方式 | 优势 | 劣势 | 适用场景 |
|---------|------|------|---------|
| **语义检索** | 理解语义和上下文 | 可能错过精确关键词 | 自然语言问题 |
| **全文检索** | 精确关键词匹配 | 无法理解语义 | 关键词搜索 |
| **混合检索** | 结合两者优点 | 计算成本略高 | 生产 RAG 系统 |

### 5. 生产环境配置

**Top-K 策略**:
- 初始检索: top_k = 20-50（每个检索源）
- RRF 重排序后: top_k = 5-10（最终返回）

**索引选择**:
- 稠密向量: HNSW（高性能）
- 稀疏向量: SPARSE_INVERTED_INDEX（BM25 专用）

**度量类型**:
- 稠密向量: COSINE（推荐）或 IP
- 稀疏向量: BM25（固定）

### 实验1：不同 Top-K 的影响

```python
top_k_values = [5, 10, 20, 50]
```

### 实验2：二次重排序（Cross-Encoder）

```python
# 初始混合检索（top-20）
initial_results = hybrid_system.search_hybrid(query, top_k=20)

# 二次重排序（cross-encoder）
from sentence_transformers import CrossEncoder
reranker = CrossEncoder('cross-encoder/ms-marco-MiniLM-L-6-v2')

# 计算相关性分数
pairs = [[query, result['content']] for result in initial_results]
scores = reranker.predict(pairs)

# 重排序
final_results = sorted(
    zip(initial_results, scores),
    key=lambda x: x[1],
    reverse=True
)[:5]
```
