## 1. 稀疏向量概述

### 1.1 什么是稀疏向量？

**稀疏向量（Sparse Vector）** 是一种只存储非零元素的向量表示方法，与稠密向量（Dense Vector）相对。

**核心特点：**
- 只存储非零元素的位置和值
- 维度通常很高（10000+ 维）
- 非零元素很少（1-5%）
- 内存效率高

**在 Milvus 2.6 中的应用：**
- BM25 全文搜索
- 关键词匹配
- 混合检索（与稠密向量结合）

---

### 1.2 稀疏向量 vs 稠密向量

| 特性 | 稀疏向量 | 稠密向量 |
|------|----------|----------|
| **维度** | 10000+ | 128-1536 |
| **非零元素** | 1-5% | 100% |
| **存储格式** | positions + values | 连续数组 |
| **内存占用** | 低（只存储非零元素） | 高（存储所有元素） |
| **应用场景** | 关键词匹配、BM25 | 语义相似度 |
| **索引类型** | SPARSE_INVERTED_INDEX, SPARSE_WAND | HNSW, IVF_FLAT |

## 2. Milvus 稀疏向量数据结构

### 2.1 源码分析

根据 Milvus 源码（`client/entity/sparse.go`），稀疏向量使用以下数据结构：

```go
// 稀疏向量接口
type SparseEmbedding interface {
    Dim() int                                      // 维度
    Len() int                                      // 实际元素数量
    Get(idx int) (pos uint32, value float32, ok bool)  // 获取元素
    Serialize() []byte                             // 序列化
    FieldType() FieldType
}

// 稀疏向量实现
type sliceSparseEmbedding struct {
    positions []uint32  // 位置数组
    values    []float32 // 值数组
    dim       int       // 维度
    len       int       // 元素数量
}
```

**关键点：**
- 使用两个数组存储：`positions[]` 和 `values[]`
- `positions[i]` 和 `values[i]` 对应同一个非零元素
- 自动按 position 升序排列
- 维度 = max(positions) + 1

---

### 2.2 存储格式

**内存格式：**
```
positions: [0, 100, 500, 1000]
values:    [0.5, 0.8, 0.3, 0.9]
```

表示：
- 第 0 维：0.5
- 第 100 维：0.8
- 第 500 维：0.3
- 第 1000 维：0.9
- 其他维度：0

**维度计算：**
```
dim = max(positions) + 1 = 1000 + 1 = 1001
```

---

### 2.3 序列化机制

**序列化格式：** 每个元素 8 字节（4 字节位置 + 4 字节值）

```go
func (e sliceSparseEmbedding) Serialize() []byte {
    row := make([]byte, 8*e.Len())
    for idx := 0; idx < e.Len(); idx++ {
        pos, value, _ := e.Get(idx)
        binary.LittleEndian.PutUint32(row[idx*8:], pos)
        binary.LittleEndian.PutUint32(row[idx*8+4:], math.Float32bits(value))
    }
    return row
}
```

**示例：**
```
positions: [0, 100]
values:    [0.5, 0.8]

序列化后（16 字节）：
[0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x3F,  // pos=0, value=0.5
 0x64, 0x00, 0x00, 0x00, 0xCD, 0xCC, 0x4C, 0x3F]  // pos=100, value=0.8
```

```python
index_params = {
    "metric_type": "IP",
    "index_type": "SPARSE_INVERTED_INDEX",
    "params": {"drop_ratio_build": 0.2}

```

**drop_ratio_search**：搜索时丢弃低值条目

```python

    "metric_type": "IP",
```

- drop_ratio 越高，索引越小，搜索越快，但精度越低
- drop_ratio 越低，索引越大，搜索越慢，但精度越高

## 5. 内存优化策略

### 5.1 稀疏度分析

```python
import numpy as np

def analyze_sparsity(sparse_vectors):
    """分析稀疏向量的稀疏度"""
    total_elements = 0
    non_zero_elements = 0

    for vec in sparse_vectors:
        dim = max(vec.keys()) + 1
        total_elements += dim
        non_zero_elements += len(vec)

    sparsity = 1 - (non_zero_elements / total_elements)
    print(f"稀疏度: {sparsity:.2%}")
    print(f"非零元素比例: {(1-sparsity):.2%}")
    print(f"平均维度: {total_elements / len(sparse_vectors):.0f}")
    print(f"平均非零元素: {non_zero_elements / len(sparse_vectors):.0f}")

# 示例
sparse_vectors = [
    {0: 0.5, 100: 0.8, 500: 0.3},
    {10: 0.7, 200: 0.6, 600: 0.4},
    {5: 0.9, 150: 0.5, 700: 0.2}
]

analyze_sparsity(sparse_vectors)
```

**输出：**
```
稀疏度: 99.57%
非零元素比例: 0.43%
平均维度: 701
平均非零元素: 3
```

## 6. 实战案例：BM25 稀疏向量生成

### 6.1 手动生成稀疏向量

```python
from collections import Counter
import hashlib

def text_to_sparse_vector(text: str) -> dict:
    """将文本转换为稀疏向量（简化版 BM25）"""
    # 1. 分词（简单按空格分割）
    tokens = text.lower().split()

    # 2. 词频统计
    token_counts = Counter(tokens)

    # 3. 生成稀疏向量
    sparse_vector = {}
    for token, count in token_counts.items():
        # 使用哈希函数将 token 转换为 uint32
        hash_value = int(hashlib.md5(token.encode()).hexdigest(), 16) % (2**32)
        sparse_vector[hash_value] = float(count)

    return sparse_vector

# 示例
text = "Milvus is a vector database for AI applications"
sparse_vec = text_to_sparse_vector(text)

print(f"文本: {text}")
print(f"稀疏向量维度: {max(sparse_vec.keys()) + 1}")
print(f"非零元素数量: {len(sparse_vec)}")
print(f"稀疏向量: {sparse_vec}")
```

## 7. 性能优化技巧

### 7.1 批量插入优化

```python
import time

# 准备大量数据
documents = [{"content": f"Document {i}"} for i in range(10000)]

# 方法 1：逐条插入（慢）
start = time.time()
for doc in documents:
    collection.insert([doc])
time1 = time.time() - start

# 方法 2：批量插入（快）
start = time.time()
batch_size = 1000
for i in range(0, len(documents), batch_size):
    batch = documents[i:i+batch_size]
    collection.insert(batch)
time2 = time.time() - start

print(f"逐条插入时间: {time1:.2f}s")
print(f"批量插入时间: {time2:.2f}s")
print(f"加速比: {time1/time2:.2f}x")
```

## 8. 常见问题与解决方案

### 8.1 维度过高问题

**问题：** 稀疏向量维度过高（100000+），导致内存占用大

**解决方案：**
1. 使用 drop_ratio 丢弃低值条目
2. 限制词汇表大小
3. 使用哈希技巧（Hash Trick）

```python
def hash_trick(token: str, max_dim: int = 10000) -> int:
    """使用哈希技巧限制维度"""
    hash_value = int(hashlib.md5(token.encode()).hexdigest(), 16)
    return hash_value % max_dim
```

---

### 8.2 稀疏度过低问题

**问题：** 非零元素过多（>10%），失去稀疏向量优势

**解决方案：**
1. 使用 TF-IDF 过滤常见词
2. 设置词频阈值
3. 使用 drop_ratio 参数

```python
def filter_common_words(sparse_vector: dict, threshold: float = 0.1) -> dict:
    """过滤低值条目"""
    return {k: v for k, v in sparse_vector.items() if v > threshold}
```
