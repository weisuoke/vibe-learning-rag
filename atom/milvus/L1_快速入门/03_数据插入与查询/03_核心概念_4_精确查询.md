# 核心概念 4: 精确查询

> **深入理解 Milvus 的精确查询操作：Query, Get, QueryIterator**

---

## 概述

除了向量相似度搜索（Search），Milvus 还提供基于标量字段的精确查询功能。精确查询不涉及向量计算，而是通过主键或标量字段的过滤条件来检索数据，类似于传统数据库的 SQL 查询。

### 核心价值

在 RAG 应用中，精确查询的作用是：
1. **元数据检索**: 根据文档 ID、标签、时间等精确获取数据
2. **数据验证**: 检查特定数据是否存在
3. **批量获取**: 根据主键列表批量获取数据
4. **分页浏览**: 遍历所有满足条件的数据

---

## 三种查询操作对比

Milvus 提供三种精确查询操作：**Get**、**Query**、**QueryIterator**

| 特性 | Get | Query | QueryIterator |
|------|-----|-------|---------------|
| **查询方式** | 按主键 | 按过滤表达式 | 按过滤表达式（分页） |
| **适用场景** | 已知主键，快速获取 | 条件过滤，返回部分结果 | 条件过滤，遍历所有结果 |
| **必需参数** | Collection名称<br>主键列表 | Collection名称<br>过滤表达式 | Collection名称<br>过滤表达式<br>每页数量 |
| **可选参数** | 分区名称<br>输出字段 | 分区名称<br>返回数量<br>输出字段 | 分区名称<br>总返回数量<br>输出字段 |
| **返回结果** | 指定主键的 Entity | 满足条件的 Entity（部分） | 满足条件的 Entity（全部，分页） |

---

## Get 操作

### 什么是 Get？

Get 是最简单的查询操作，通过主键列表直接获取对应的 Entity。

### 基本用法

```python
from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 通过主键获取 Entity
results = client.get(
    collection_name="my_collection",
    ids=[1, 2, 3],  # 主键列表
    output_fields=["document", "dense"]  # 返回的字段
)

# 打印结果
for entity in results:
    print(f"ID: {entity['id']}")
    print(f"文档: {entity['document']}\n")
```

### 参数详解

```python
client.get(
    collection_name="my_collection",  # Collection 名称
    ids=[1, 2, 3],                    # 主键列表
    output_fields=["*"],              # 返回的字段
    partition_names=None              # 可选：指定分区
)
```

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `collection_name` | str | ✅ | Collection 名称 |
| `ids` | List[int/str] | ✅ | 主键列表 |
| `output_fields` | List[str] | ❌ | 返回的字段列表（默认所有） |
| `partition_names` | List[str] | ❌ | 搜索的分区列表 |

### 使用场景

**场景1: 根据 ID 获取文档**

```python
def get_document_by_id(doc_id):
    """根据文档 ID 获取内容"""
    results = client.get(
        collection_name="doc_collection",
        ids=[doc_id],
        output_fields=["document", "title", "author"]
    )

    if results:
        return results[0]
    return None

# 使用示例
doc = get_document_by_id(123)
if doc:
    print(f"标题: {doc['title']}")
    print(f"作者: {doc['author']}")
    print(f"内容: {doc['document']}")
```

**场景2: 批量获取文档**

```python
def get_documents_batch(doc_ids):
    """批量获取文档"""
    results = client.get(
        collection_name="doc_collection",
        ids=doc_ids,
        output_fields=["document", "title"]
    )
    return results

# 使用示例
docs = get_documents_batch([1, 2, 3, 4, 5])
for doc in docs:
    print(f"{doc['id']}: {doc['title']}")
```

**场景3: 验证数据存在性**

```python
def document_exists(doc_id):
    """检查文档是否存在"""
    results = client.get(
        collection_name="doc_collection",
        ids=[doc_id],
        output_fields=["id"]  # 只返回 ID，减少数据传输
    )
    return len(results) > 0

# 使用示例
if document_exists(123):
    print("文档存在")
else:
    print("文档不存在")
```

### Get 的特点

**优点**:
- ✅ **最快**: 直接通过主键索引查找，O(1) 复杂度
- ✅ **简单**: 不需要写过滤表达式
- ✅ **精确**: 保证返回指定主键的数据

**限制**:
- ❌ **只能按主键**: 不支持其他字段的查询
- ❌ **不支持条件**: 无法添加额外的过滤条件

## Query 操作

### 什么是 Query？

Query 是基于过滤表达式的查询操作，可以根据标量字段的条件来检索数据。

### 基本用法

```python
# 查询所有 category 为 "tech" 的文档
results = client.query(
    collection_name="my_collection",
    filter='category == "tech"',  # 过滤表达式
    output_fields=["document", "title"],
    limit=10  # 最多返回 10 条
)

# 打印结果
for entity in results:
    print(f"ID: {entity['id']}")
    print(f"标题: {entity['title']}\n")
```

### 参数详解

```python
client.query(
    collection_name="my_collection",  # Collection 名称
    filter="",                        # 过滤表达式
    output_fields=["*"],              # 返回的字段
    limit=None,                       # 返回数量限制
    offset=0,                         # 跳过前 N 条
    partition_names=None              # 搜索的分区
)
```

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `collection_name` | str | ✅ | Collection 名称 |
| `filter` | str | ✅ | 过滤表达式 |
| `output_fields` | List[str] | ❌ | 返回的字段列表 |
| `limit` | int | ❌ | 最多返回多少条（默认无限制） |
| `offset` | int | ❌ | 跳过前 N 条（分页） |
| `partition_names` | List[str] | ❌ | 搜索的分区列表 |

### 过滤表达式

Query 支持丰富的过滤表达式：

**基本比较**:
```python
# 等于
filter='category == "tech"'

# 不等于
filter='status != "deleted"'

# 大于、小于
filter='score > 0.8'
filter='price < 100'

# 大于等于、小于等于
filter='age >= 18'
filter='count <= 1000'
```

**逻辑运算**:
```python
# AND
filter='category == "tech" and score > 0.8'

# OR
filter='category == "tech" or category == "ai"'

# 复杂组合
filter='(category == "tech" or category == "ai") and score > 0.8'
```

**集合操作**:
```python
# IN
filter='category in ["tech", "ai", "ml"]'

# NOT IN
filter='status not in ["deleted", "archived"]'
```

**字符串匹配**:
```python
# LIKE（前缀匹配）
filter='title like "Milvus%"'

# 包含（需要使用 JSON 函数）
filter='tags like "%vector%"'
```

**范围查询**:
```python
# 时间范围
filter='created_at > "2026-01-01" and created_at < "2026-12-31"'

# 数值范围
filter='price >= 10 and price <= 100'
```

### 使用场景

**场景1: 按类别查询**

```python
def get_documents_by_category(category, limit=10):
    """按类别查询文档"""
    results = client.query(
        collection_name="doc_collection",
        filter=f'category == "{category}"',
        output_fields=["document", "title"],
        limit=limit
    )
    return results

# 使用示例
tech_docs = get_documents_by_category("tech", limit=5)
```

**场景2: 按时间范围查询**

```python
def get_recent_documents(days=7):
    """获取最近 N 天的文档"""
    from datetime import datetime, timedelta

    cutoff_date = (datetime.now() - timedelta(days=days)).isoformat()

    results = client.query(
        collection_name="doc_collection",
        filter=f'created_at > "{cutoff_date}"',
        output_fields=["document", "title", "created_at"],
        limit=100
    )
    return results

# 使用示例
recent_docs = get_recent_documents(days=7)
```

**场景3: 复杂条件查询**

```python
def search_documents(category=None, min_score=None, tags=None):
    """复杂条件查询"""
    conditions = []

    if category:
        conditions.append(f'category == "{category}"')

    if min_score:
        conditions.append(f'score >= {min_score}')

    if tags:
        tag_list = ', '.join([f'"{tag}"' for tag in tags])
        conditions.append(f'tags in [{tag_list}]')

    filter_expr = ' and '.join(conditions) if conditions else 'id > 0'

    results = client.query(
        collection_name="doc_collection",
        filter=filter_expr,
        output_fields=["document", "title", "score"],
        limit=50
    )
    return results

# 使用示例
docs = search_documents(
    category="tech",
    min_score=0.8,
    tags=["vector", "database"]
)
```

**场景4: 分页查询**

```python
def paginated_query(filter_expr, page_size=10, page_num=1):
    """分页查询"""
    offset = (page_num - 1) * page_size

    results = client.query(
        collection_name="doc_collection",
        filter=filter_expr,
        output_fields=["document", "title"],
        limit=page_size,
        offset=offset
    )
    return results

# 使用示例
page1 = paginated_query('category == "tech"', page_size=10, page_num=1)
page2 = paginated_query('category == "tech"', page_size=10, page_num=2)
```

## QueryIterator 操作

### 什么是 QueryIterator？

QueryIterator 是用于遍历大量查询结果的迭代器，它会自动分页获取所有满足条件的数据。

### 基本用法

```python
# 创建迭代器
iterator = client.query_iterator(
    collection_name="my_collection",
    filter='category == "tech"',
    output_fields=["document", "title"],
    batch_size=100  # 每次获取 100 条
)

# 遍历所有结果
all_results = []
while True:
    batch = iterator.next()
    if not batch:
        break
    all_results.extend(batch)

iterator.close()

print(f"总共获取了 {len(all_results)} 条数据")
```

### 参数详解

```python
client.query_iterator(
    collection_name="my_collection",  # Collection 名称
    filter="",                        # 过滤表达式
    output_fields=["*"],              # 返回的字段
    batch_size=1000,                  # 每批次数量
    limit=None,                       # 总返回数量限制
    partition_names=None              # 搜索的分区
)
```

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `collection_name` | str | ✅ | Collection 名称 |
| `filter` | str | ✅ | 过滤表达式 |
| `output_fields` | List[str] | ❌ | 返回的字段列表 |
| `batch_size` | int | ✅ | 每次获取的数量 |
| `limit` | int | ❌ | 总共返回的数量限制 |
| `partition_names` | List[str] | ❌ | 搜索的分区列表 |

### 使用场景

**场景1: 导出所有数据**

```python
def export_all_documents(category):
    """导出指定类别的所有文档"""
    iterator = client.query_iterator(
        collection_name="doc_collection",
        filter=f'category == "{category}"',
        output_fields=["document", "title", "author"],
        batch_size=1000
    )

    all_docs = []
    while True:
        batch = iterator.next()
        if not batch:
            break
        all_docs.extend(batch)
        print(f"已获取 {len(all_docs)} 条数据...")

    iterator.close()
    return all_docs

# 使用示例
all_tech_docs = export_all_documents("tech")
print(f"总共导出了 {len(all_tech_docs)} 条技术文档")
```

**场景2: 批量处理**

```python
def process_documents_in_batches(filter_expr, process_func):
    """批量处理文档"""
    iterator = client.query_iterator(
        collection_name="doc_collection",
        filter=filter_expr,
        output_fields=["id", "document"],
        batch_size=100
    )

    total_processed = 0
    while True:
        batch = iterator.next()
        if not batch:
            break

        # 处理当前批次
        for doc in batch:
            process_func(doc)
            total_processed += 1

        print(f"已处理 {total_processed} 条数据...")

    iterator.close()
    return total_processed

# 使用示例
def update_document(doc):
    # 处理逻辑
    pass

total = process_documents_in_batches('score < 0.5', update_document)
```

**场景3: 统计分析**

```python
def analyze_documents(category):
    """分析文档统计信息"""
    iterator = client.query_iterator(
        collection_name="doc_collection",
        filter=f'category == "{category}"',
        output_fields=["score", "word_count"],
        batch_size=1000
    )

    total_count = 0
    total_score = 0
    total_words = 0

    while True:
        batch = iterator.next()
        if not batch:
            break

        for doc in batch:
            total_count += 1
            total_score += doc.get('score', 0)
            total_words += doc.get('word_count', 0)

    iterator.close()

    return {
        'count': total_count,
        'avg_score': total_score / total_count if total_count > 0 else 0,
        'avg_words': total_words / total_count if total_count > 0 else 0
    }

# 使用示例
stats = analyze_documents("tech")
print(f"文档数量: {stats['count']}")
print(f"平均分数: {stats['avg_score']:.2f}")
print(f"平均字数: {stats['avg_words']:.0f}")
```

## Query vs Search

### 核心区别

| 特性 | Query | Search |
|------|-------|--------|
| **查询方式** | 标量字段过滤 | 向量相似度 |
| **是否涉及向量** | ❌ 否 | ✅ 是 |
| **查询速度** | 快（索引查找） | 较慢（向量计算） |
| **结果排序** | 插入顺序或主键 | 相似度降序 |
| **适用场景** | 精确匹配、元数据查询 | 语义搜索、相似推荐 |

### 使用建议

**使用 Query 的场景**:
- ✅ 根据 ID、类别、标签等精确查询
- ✅ 时间范围查询
- ✅ 数据验证和存在性检查
- ✅ 批量获取特定条件的数据

**使用 Search 的场景**:
- ✅ 语义搜索（找相似内容）
- ✅ RAG 应用中的文档检索
- ✅ 推荐系统
- ✅ 相似度排序

### 组合使用

```python
# 先用 Search 找到相似文档，再用 Query 获取详细信息
def hybrid_search(query_text, category):
    """混合搜索：向量搜索 + 标量过滤"""
    # 1. 向量搜索
    search_results = client.search(
        collection_name="doc_collection",
        data=[query_text],  # Embedding Functions 自动嵌入
        anns_field="dense",
        filter=f'category == "{category}"',  # 标量过滤
        limit=10,
        output_fields=["id"]
    )

    # 2. 获取详细信息
    doc_ids = [hit['id'] for hit in search_results[0]]
    detailed_results = client.get(
        collection_name="doc_collection",
        ids=doc_ids,
        output_fields=["document", "title", "author", "created_at"]
    )

    return detailed_results
```

## 性能优化

### 1. 使用主键查询

```python
# ✅ 最快：使用 Get
results = client.get(ids=[1, 2, 3])

# ❌ 较慢：使用 Query
results = client.query(filter='id in [1, 2, 3]')
```

### 2. 限制返回字段

```python
# ❌ 返回所有字段
results = client.query(filter='...', output_fields=["*"])

# ✅ 只返回需要的字段
results = client.query(filter='...', output_fields=["id", "title"])
```

### 3. 使用分区

```python
# ❌ 搜索所有分区
results = client.query(filter='...')

# ✅ 只搜索相关分区
results = client.query(filter='...', partition_names=["tech_docs"])
```

### 4. 合理使用 limit

```python
# ❌ 不限制返回数量
results = client.query(filter='...')  # 可能返回百万条

# ✅ 限制返回数量
results = client.query(filter='...', limit=100)
```

## 常见问题

### Q1: Query 和 Get 哪个更快？

**A**: Get 更快，因为它直接通过主键索引查找（O(1)），而 Query 需要扫描和过滤（O(n)）。如果已知主键，优先使用 Get。

### Q2: 如何查询所有数据？

**A**: 使用 QueryIterator：
```python
iterator = client.query_iterator(
    collection_name="my_collection",
    filter='id > 0',  # 匹配所有数据
    batch_size=1000
)
```

### Q3: Query 支持模糊查询吗？

**A**: 支持前缀匹配（LIKE），但不支持中间匹配：
```python
# ✅ 支持：前缀匹配
filter='title like "Milvus%"'

# ❌ 不支持：中间匹配
filter='title like "%Milvus%"'  # 不支持
```

### Q4: 如何统计满足条件的数据数量？

**A**: 使用 Query 并计数：
```python
results = client.query(
    collection_name="my_collection",
    filter='category == "tech"',
    output_fields=["id"],  # 只返回 ID，减少数据传输
    limit=None  # 不限制数量
)
count = len(results)
```

## 总结

### 核心要点

1. **Get**: 通过主键快速获取，最快的查询方式
2. **Query**: 通过过滤表达式查询，支持复杂条件
3. **QueryIterator**: 分页遍历大量数据，适合批量处理
4. **Query vs Search**: Query 用于精确匹配，Search 用于语义搜索
5. **性能优化**: 使用主键、限制字段、使用分区、合理设置 limit

### 选择指南

| 场景 | 推荐操作 | 原因 |
|------|----------|------|
| 已知主键 | Get | 最快 |
| 条件查询（少量结果） | Query | 灵活 |
| 条件查询（大量结果） | QueryIterator | 分页 |
| 语义搜索 | Search | 相似度 |
| 混合查询 | Search + Query | 结合优势 |
