## 4.1 async fn + .await：基本读写模式

**这是什么**：声明异步函数，调用并等待结果。90% 的异步代码都是这个模式。

```rust
// ─── 声明异步函数 ───
async fn fetch_data(url: &str) -> Result<String, reqwest::Error> {
    let response = reqwest::get(url).await?;   // .await 等待 HTTP 响应
    let text = response.text().await?;          // .await 等待读取 body
    Ok(text)
}

// ─── 调用异步函数 ───
#[tokio::main]   // ← 这个宏创建 Tokio 运行时
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let data = fetch_data("https://api.example.com/data").await?;
    println!("获取到: {}", data);
    Ok(())
}
```

**TypeScript 对照**：

```typescript
// TypeScript — 几乎一模一样！
async function fetchData(url: string): Promise<string> {
    const response = await fetch(url);
    const text = await response.text();
    return text;
}

async function main() {
    const data = await fetchData("https://api.example.com/data");
    console.log(`获取到: ${data}`);
}
```

**关键区别**：
| | TypeScript | Rust |
|---|---|---|
| 返回类型 | `Promise<string>` | `Result<String, Error>` |
| 错误传播 | `try/catch` 或 `.catch()` | `?` 运算符 |
| 调用即执行 | ✅ 调用立刻开始执行 | ❌ 调用只返回 Future，必须 `.await` 才执行 |
| 运行时 | 内置（V8 事件循环） | 需要手动配置（`#[tokio::main]`） |

**实际使用场景**：

```rust
// ZeroClaw 中 90% 的代码就是这个模式
async fn handle_message(provider: &dyn Provider, message: &str) -> Result<String> {
    // 调用 LLM
    let response = provider.simple_chat(message, "gpt-4", 0.7).await?;
    Ok(response)
}
```

**记忆口诀**：**`async` 声明，`.await` 等待，`?` 传播错误。**

## 4.2 tokio::spawn：后台任务

**这是什么**：把一段异步代码扔到后台执行，不阻塞当前任务。

```rust
use tokio::task::JoinHandle;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // ─── 启动后台任务 ───
    let handle: JoinHandle<String> = tokio::spawn(async move {
        // 这段代码在后台执行——可能在另一个线程上
        tokio::time::sleep(std::time::Duration::from_secs(1)).await;
        "后台任务完成".to_string()
    });

    // 当前任务继续做别的事
    println!("后台任务已启动，我继续做事...");

    // 等需要结果时再 .await
    let result = handle.await?;   // ? 处理 JoinError（任务 panic 等）
    println!("结果: {}", result);

    Ok(())
}
```

**TypeScript 对照**：

```typescript
// TypeScript 中没有真正的"后台线程"概念
// 但 Promise.all 和 setTimeout 是类似思路

// Rust 的 tokio::spawn ≈ 不等待的 Promise + 稍后 await
const promise = asyncWork();  // 立刻开始执行（注意不加 await）
console.log("继续做事...");
const result = await promise;  // 稍后获取结果
```

**多任务并发**：

```rust
// 同时启动多个后台任务
let handle1 = tokio::spawn(fetch_data("https://api1.com"));
let handle2 = tokio::spawn(fetch_data("https://api2.com"));
let handle3 = tokio::spawn(fetch_data("https://api3.com"));

// 等待所有任务完成
let (r1, r2, r3) = tokio::join!(handle1, handle2, handle3);
// 三个请求并发执行，总时间 ≈ 最慢那个（而非三者之和）
```

**核心约束**：`tokio::spawn` 要求闭包是 **`Send + 'static`**
- `Send`：因为任务可能在另一个线程执行
- `'static`：闭包不能引用外部的栈变量（用 `move` 转移所有权）

```rust
let name = String::from("hello");

// ❌ 编译错误：name 是借用，不是 'static
tokio::spawn(async {
    println!("{}", name);
});

// ✅ 用 move 转移所有权
tokio::spawn(async move {
    println!("{}", name);  // name 被 move 进来了
});
// 注意：move 之后，外面不能再用 name

// ✅ 如果需要共享，用 Arc
let name = Arc::new(String::from("hello"));
let name_clone = name.clone();
tokio::spawn(async move {
    println!("{}", name_clone);
});
// name 还可以继续使用
```

**记忆口诀**：**`spawn` 扔后台，`move` 转所有权，`handle.await` 取结果。**

## 4.3 tokio::select!：超时与竞争

**这是什么**：同时等待多个异步操作，**第一个完成的赢**，其他自动取消。

```rust
use std::time::Duration;

#[tokio::main]
async fn main() {
    // ─── 最常用模式：超时控制 ───
    tokio::select! {
        result = do_work() => {
            println!("工作完成: {:?}", result);
        }
        _ = tokio::time::sleep(Duration::from_secs(5)) => {
            println!("超时了！5 秒内没完成");
        }
    }
}

async fn do_work() -> String {
    // 模拟耗时操作
    tokio::time::sleep(Duration::from_secs(2)).await;
    "done".to_string()
}
```

**TypeScript 对照**：

```typescript
// TypeScript 中的超时模式
const result = await Promise.race([
    doWork(),
    new Promise((_, reject) =>
        setTimeout(() => reject(new Error("超时")), 5000)
    ),
]);

// Rust 的 select! ≈ Promise.race
// 但更强大：可以有 N 个分支、可以模式匹配、自动取消输家
```

**ZeroClaw 中的 select! 用法**（Agent 循环）：

```rust
loop {
    tokio::select! {
        // 分支 1：收到用户消息
        Some(msg) = rx.recv() => {
            process_message(msg).await;
        }
        // 分支 2：收到取消信号
        _ = cancellation_token.cancelled() => {
            println!("收到关闭信号，退出循环");
            break;
        }
    }
}
```

**记忆口诀**：**`select!` = Promise.race，先到先得，其余取消。**

## 4.4 mpsc 通道：任务间消息传递

**这是什么**：异步任务之间安全传递数据的管道。多个发送者 → 一个接收者。

```rust
use tokio::sync::mpsc;

#[tokio::main]
async fn main() {
    // ─── 创建通道 ───
    let (tx, mut rx) = mpsc::channel::<String>(32);  // 缓冲区 32 条消息

    // ─── 生产者（可以有多个）───
    let tx1 = tx.clone();
    tokio::spawn(async move {
        tx1.send("来自任务1".to_string()).await.unwrap();
    });

    let tx2 = tx.clone();
    tokio::spawn(async move {
        tx2.send("来自任务2".to_string()).await.unwrap();
    });

    // 重要：drop 原始 tx！否则通道永远不会关闭
    drop(tx);

    // ─── 消费者（只有一个）───
    while let Some(msg) = rx.recv().await {
        println!("收到: {}", msg);
    }
    // 所有 tx 被 drop 后，recv() 返回 None，循环退出
    println!("所有消息已处理");
}
```

**TypeScript 对照**：

```typescript
// mpsc ≈ EventEmitter / postMessage / RxJS Subject
const bus = new EventEmitter();

// 消费者
bus.on("message", (msg) => console.log(`收到: ${msg}`));

// 多个生产者
bus.emit("message", "来自任务1");
bus.emit("message", "来自任务2");

// 区别：Rust 的 mpsc 在编译时保证类型安全 + 有背压机制
```

**ZeroClaw 的 Channel 架构就是 mpsc**：

```
Telegram ──► tx.clone() ──┐
Discord  ──► tx.clone() ──┤──► [缓冲区] ──► rx ──► Agent 循环
CLI      ──► tx.clone() ──┘
```

**三个必记要点**：
1. `tx.clone()` 给多个生产者——Sender 可以 clone
2. `drop(tx)` 释放原始发送端——否则通道永不关闭
3. `while let Some(msg) = rx.recv().await` 消费消息——标准写法

**记忆口诀**：**`channel` 建管道，`tx.send()` 发消息，`rx.recv()` 收消息，`drop(tx)` 关管道。**

## 4.5 #[async_trait]：异步 Trait

**这是什么**：让 Trait 的方法可以是 `async fn`，支持动态分发（`dyn Trait`）。

```rust
use async_trait::async_trait;
use std::sync::Arc;

// ─── 定义异步 Trait ───
#[async_trait]                          // ← 加在 trait 定义上
trait MyService: Send + Sync {          // ← Send + Sync 用于多线程安全
    fn name(&self) -> &str;             // 同步方法——不需要特殊处理
    async fn process(&self, input: &str) -> anyhow::Result<String>;  // 异步方法
}

// ─── 实现异步 Trait ───
struct EchoService;

#[async_trait]                          // ← impl 块也要加！
impl MyService for EchoService {
    fn name(&self) -> &str { "echo" }

    async fn process(&self, input: &str) -> anyhow::Result<String> {
        // 模拟异步操作
        tokio::time::sleep(std::time::Duration::from_millis(100)).await;
        Ok(format!("Echo: {}", input))
    }
}

// ─── 使用：动态分发 ───
async fn run(service: &dyn MyService) {
    let result = service.process("hello").await.unwrap();
    println!("[{}] {}", service.name(), result);
}

#[tokio::main]
async fn main() {
    // 通过 Arc<dyn Trait> 持有
    let service: Arc<dyn MyService> = Arc::new(EchoService);
    run(service.as_ref()).await;
}
```

**TypeScript 对照**：

```typescript
// Rust 的 #[async_trait] ≈ TypeScript 的 interface + async 方法
interface MyService {
    name(): string;
    process(input: string): Promise<string>;
}

class EchoService implements MyService {
    name() { return "echo"; }
    async process(input: string) { return `Echo: ${input}`; }
}

async function run(service: MyService) {
    const result = await service.process("hello");
    console.log(`[${service.name()}] ${result}`);
}
```

**ZeroClaw 四大 Trait 全部使用这个模式**：

```rust
// Provider：LLM 调用
#[async_trait]
pub trait Provider: Send + Sync {
    async fn chat(&self, ...) -> Result<ChatResponse>;
}

// Channel：消息收发
#[async_trait]
pub trait Channel: Send + Sync {
    async fn listen(&self, tx: mpsc::Sender<ChannelMessage>) -> Result<()>;
}

// Tool：工具执行
#[async_trait]
pub trait Tool: Send + Sync {
    async fn execute(&self, args: Value) -> Result<ToolResult>;
}

// Memory：存储检索
#[async_trait]
pub trait Memory: Send + Sync {
    async fn store(&self, key: &str, content: &str, ...) -> Result<()>;
    async fn recall(&self, query: &str, limit: usize, ...) -> Result<Vec<MemoryEntry>>;
}
```

**三个必记规则**：
1. **Trait 定义上加 `#[async_trait]`**
2. **每个 impl 块也要加 `#[async_trait]`**（最常见的遗漏！）
3. **Trait 加 `: Send + Sync`**（多线程安全）

**记忆口诀**：**`#[async_trait]` 标两处（定义 + 实现），`Send + Sync` 保线程安全。**

## 综合示例：用 5 个模式写一个迷你 Agent

把全部 5 个模式组合在一起：

```rust
use async_trait::async_trait;
use tokio::sync::mpsc;
use std::sync::Arc;
use std::time::Duration;

// ── 模式5：#[async_trait] 定义可插拔接口 ──
#[async_trait]
trait SimpleProvider: Send + Sync {
    async fn chat(&self, message: &str) -> String;
}

struct MockProvider;

#[async_trait]
impl SimpleProvider for MockProvider {
    // ── 模式1：async fn + .await ──
    async fn chat(&self, message: &str) -> String {
        tokio::time::sleep(Duration::from_millis(100)).await;
        format!("AI 回复: {}", message)
    }
}

#[tokio::main]
async fn main() {
    let provider: Arc<dyn SimpleProvider> = Arc::new(MockProvider);

    // ── 模式4：mpsc 通道接收消息 ──
    let (tx, mut rx) = mpsc::channel::<String>(32);

    // ── 模式2：tokio::spawn 后台监听 ──
    tokio::spawn(async move {
        let messages = vec!["你好", "今天天气怎么样", "再见"];
        for msg in messages {
            tx.send(msg.to_string()).await.unwrap();
            tokio::time::sleep(Duration::from_millis(200)).await;
        }
        // tx 在这里被 drop → 通道关闭
    });

    // ── Agent 主循环 ──
    loop {
        // ── 模式3：select! 处理消息或超时 ──
        tokio::select! {
            msg = rx.recv() => {
                match msg {
                    Some(message) => {
                        println!("用户: {}", message);
                        let reply = provider.chat(&message).await;
                        println!("AI: {}", reply);
                    }
                    None => {
                        println!("通道关闭，退出");
                        break;
                    }
                }
            }
            _ = tokio::time::sleep(Duration::from_secs(10)) => {
                println!("10 秒无消息，超时退出");
                break;
            }
        }
    }
}
```

**运行输出**：

```
用户: 你好
AI: AI 回复: 你好
用户: 今天天气怎么样
AI: AI 回复: 今天天气怎么样
用户: 再见
AI: AI 回复: 再见
通道关闭，退出
```

**这个例子浓缩了 ZeroClaw 的核心架构**：
- `SimpleProvider` → ZeroClaw 的 `Provider` Trait
- `mpsc::channel` → ZeroClaw 的 `Channel.listen(tx)` 架构
- `tokio::spawn` → ZeroClaw 的后台监听器
- `select!` + 超时 → ZeroClaw 的 Agent 循环 + 取消控制
- `async/await` → ZeroClaw 全局使用的异步模式
