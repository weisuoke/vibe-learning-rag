# 实战代码 - 场景4：语义缓存实现

> **场景目标**：掌握语义缓存的实现原理和应用场景，提升缓存命中率

---

## 场景说明

### 什么是语义缓存

**精确匹配缓存**：
- 只有完全相同的文本才命中
- 示例："What is AI?" ≠ "What is artificial intelligence?"
- 命中率受限

**语义缓存**：
- 基于 embedding 相似度匹配
- 语义相似的查询也能命中
- 显著提升命中率

### 适用场景
- 问答系统（相似问题频繁）
- 对话应用（表达方式多样）
- 客服机器人（同义问题）
- 搜索系统（查询改写）

### 核心特性
- **语义理解**：理解查询意图
- **相似度匹配**：基于 embedding 距离
- **阈值控制**：平衡命中率和准确性
- **成本优化**：减少重复 API 调用

## 关键点解释

### 1. 语义缓存工作原理

**流程**：
```
1. 用户查询 → 计算 embedding
2. 在 Redis 中搜索相似向量
3. 计算相似度（余弦距离）
4. 如果相似度 < 阈值 → 命中缓存
5. 返回缓存的结果
```

**与精确匹配的区别**：
- 精确匹配：字符串完全相同
- 语义缓存：embedding 相似度

### 2. 相似度阈值

**score_threshold 含义**：
- 使用余弦距离（0-2）
- 距离越小越相似
- 阈值越小要求越严格

**推荐值**：
- 0.1-0.2：严格匹配（问答系统）
- 0.2-0.3：适中（对话应用）
- 0.3-0.5：宽松（搜索系统）

### 3. Embedding 计算成本

**额外成本**：
- 每次查询需要计算 embedding
- text-embedding-3-small：~$0.0001/次
- 相比 LLM 调用成本低很多

**优化策略**：
- 缓存查询 embedding
- 使用更小的模型
- 批量处理

### 4. 误命中问题

**什么是误命中**：
- 语义相似但答案不同的查询
- 返回了错误的缓存结果

**示例**：
```python
# 查询1："What is Python?"
# 查询2："What is Java?"
# 如果阈值太大，可能误命中
```

**解决方案**：
- 降低阈值
- 监控误命中率
- 人工审核高频查询

### 5. Redis 向量搜索

**实现原理**：
- Redis 使用 RediSearch 模块
