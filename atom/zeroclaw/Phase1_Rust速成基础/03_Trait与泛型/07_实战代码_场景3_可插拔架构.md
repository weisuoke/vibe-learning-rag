## 第一步：定义 Provider Trait（简化版）

ZeroClaw 真实的 Provider Trait 有 12+ 个方法，我们简化为 2 个核心方法：

```rust
#[async_trait]
pub trait Provider: Send + Sync {
    fn name(&self) -> &str;
    async fn chat(&self, message: &str) -> anyhow::Result<String>;
}
// Send + Sync: ZeroClaw 所有核心 Trait 的标配
// tokio 会在不同线程上调度任务，必须保证线程安全
```

```typescript
// TypeScript 对照：不需要 Send + Sync（JS 单线程，无数据竞争）
interface Provider {
  name(): string;
  chat(message: string): Promise<string>;
}
```

## 第二步：实现多个 Provider

```rust
// EchoProvider -- 回显输入（用于测试）
pub struct EchoProvider;

#[async_trait]
impl Provider for EchoProvider {
    fn name(&self) -> &str { "echo" }
    async fn chat(&self, message: &str) -> anyhow::Result<String> {
        Ok(format!("[Echo] {}", message))
    }
}

// MockLlmProvider -- 模拟 LLM 响应（带配置）
pub struct MockLlmProvider { model: String }

impl MockLlmProvider {
    pub fn new(model: impl Into<String>) -> Self {
        Self { model: model.into() }
    }
}

#[async_trait]
impl Provider for MockLlmProvider {
    fn name(&self) -> &str { "mock_llm" }
    async fn chat(&self, message: &str) -> anyhow::Result<String> {
        Ok(format!("[{}] 收到: '{}'. 这是一个模拟回复。", self.model, message))
    }
}
```

注意 `impl Into<String>` -- ZeroClaw 源码高频模式，让 `&str` 和 `String` 都能传：

```rust
MockLlmProvider::new("gpt-4");                  // &str -> 自动转换
MockLlmProvider::new(String::from("gpt-4"));     // String -> 直接使用
```

## 第三步：构建 Agent（使用 Trait Object）

```rust
struct Agent {
    provider: Box<dyn Provider>,       // 动态分发：运行时决定用哪个 Provider
    tools: Vec<Box<dyn Tool>>,         // 动态分发：可以混装不同类型的 Tool
}
```

### 为什么用 `Box<dyn Provider>` 而不是泛型 `<P: Provider>`？

```rust
// 方案 A：泛型（静态分发）
struct Agent<P: Provider> { provider: P }
// 问题：Agent<OpenAI> 和 Agent<Ollama> 是不同类型！无法运行时切换

// 方案 B：Trait Object（动态分发）-- ZeroClaw 的选择
struct Agent { provider: Box<dyn Provider> }
// 优势：运行时根据 config.toml 动态选择，代价仅 ~1-2ns 虚函数表查找
```

```typescript
// TypeScript 天然就是动态分发
class Agent { constructor(private provider: Provider) {} }
// Rust 需要 Box<dyn T> 显式选择 -- "零成本抽象"：你不用的东西不付费
```

## 第五步：完整运行示例

将以下代码保存为 `main.rs`，用 `cargo run` 即可运行：

```rust
use async_trait::async_trait;
use serde_json::{json, Value};

// ========== Trait 定义 ==========

#[async_trait]
pub trait Provider: Send + Sync {
    fn name(&self) -> &str;
    async fn chat(&self, message: &str) -> anyhow::Result<String>;
}

#[async_trait]
pub trait Tool: Send + Sync {
    fn name(&self) -> &str;
    async fn execute(&self, args: Value) -> anyhow::Result<String>;
}

// ========== Provider 实现 ==========

pub struct EchoProvider;

#[async_trait]
impl Provider for EchoProvider {
    fn name(&self) -> &str { "echo" }
    async fn chat(&self, message: &str) -> anyhow::Result<String> {
        Ok(format!("[Echo] {}", message))
    }
}

pub struct MockLlmProvider { model: String }

impl MockLlmProvider {
    pub fn new(model: impl Into<String>) -> Self { Self { model: model.into() } }
}

#[async_trait]
impl Provider for MockLlmProvider {
    fn name(&self) -> &str { "mock_llm" }
    async fn chat(&self, message: &str) -> anyhow::Result<String> {
        Ok(format!("[{}] 收到: '{}'. 这是一个模拟回复。", self.model, message))
    }
}

// ========== Tool 实现 ==========

pub struct GreetTool;

#[async_trait]
impl Tool for GreetTool {
    fn name(&self) -> &str { "greet" }
    async fn execute(&self, args: Value) -> anyhow::Result<String> {
        let name = args.get("name").and_then(|v| v.as_str()).unwrap_or("World");
        Ok(format!("Hello, {}!", name))
    }
}

// ========== Agent ==========

struct Agent {
    provider: Box<dyn Provider>,
    tools: Vec<Box<dyn Tool>>,
}

impl Agent {
    fn new(provider: Box<dyn Provider>) -> Self {
        Self { provider, tools: Vec::new() }
    }

    fn add_tool(mut self, tool: Box<dyn Tool>) -> Self {
        self.tools.push(tool);
        self  // 返回 self 实现链式调用（Builder 模式）
    }

    async fn chat(&self, message: &str) -> anyhow::Result<String> {
        println!("[Agent] Provider: {}", self.provider.name());
        println!("[Agent] Tools: {:?}",
            self.tools.iter().map(|t| t.name()).collect::<Vec<_>>());
        self.provider.chat(message).await
    }

    async fn use_tool(&self, tool_name: &str, args: Value) -> anyhow::Result<String> {
        self.tools.iter()
            .find(|t| t.name() == tool_name)
            .ok_or_else(|| anyhow::anyhow!("Tool not found: {}", tool_name))?
            .execute(args).await
    }
}

// ========== 工厂函数 ==========

fn create_provider(name: &str) -> anyhow::Result<Box<dyn Provider>> {
    match name {
        "echo" => Ok(Box::new(EchoProvider)),
        "mock_llm" => Ok(Box::new(MockLlmProvider::new("gpt-4-mock"))),
        _ => anyhow::bail!("Unknown provider: {}", name),
    }
}

// ========== 主函数 ==========

#[tokio::main]
async fn main() -> anyhow::Result<()> {
    // 1. 工厂创建 Provider（模拟从 config.toml 读取）
    let agent = Agent::new(create_provider("mock_llm")?)
        .add_tool(Box::new(GreetTool));

    // 2. 对话
    let reply = agent.chat("什么是 Trait？").await?;
    println!("回复: {}\n", reply);

    // 3. 调用工具
    let result = agent.use_tool("greet", json!({"name": "ZeroClaw"})).await?;
    println!("工具结果: {}\n", result);

    // 4. 切换 Provider -- 可插拔的威力：只改一个字符串
    println!("=== 切换到 Echo Provider ===\n");
    let agent2 = Agent::new(create_provider("echo")?);
    let reply2 = agent2.chat("什么是 Trait？").await?;
    println!("回复: {}", reply2);

    Ok(())
}

// ========== 测试 ==========

#[cfg(test)]
mod tests {
    use super::*;

    #[tokio::test]
    async fn test_echo_provider() {
        let p = EchoProvider;
        assert_eq!(p.name(), "echo");
        assert!(p.chat("hello").await.unwrap().contains("hello"));
    }

    #[tokio::test]
    async fn test_factory() {
        assert_eq!(create_provider("echo").unwrap().name(), "echo");
        assert!(create_provider("nonexistent").is_err());
    }

    #[tokio::test]
    async fn test_agent_with_tool() {
        let agent = Agent::new(Box::new(EchoProvider)).add_tool(Box::new(GreetTool));
        assert_eq!(agent.use_tool("greet", json!({"name": "Test"})).await.unwrap(), "Hello, Test!");
        assert!(agent.use_tool("nonexistent", json!({})).await.is_err());
    }
}
```

## TypeScript 对照：依赖注入模式

```typescript
interface Provider {
  name(): string;
  chat(message: string): Promise<string>;
}

class EchoProvider implements Provider {
  name() { return "echo"; }
  async chat(message: string) { return `[Echo] ${message}`; }
}

class MockLlmProvider implements Provider {
  constructor(private model: string) {}
  name() { return "mock_llm"; }
  async chat(message: string) {
    return `[${this.model}] 收到: '${message}'. 这是一个模拟回复。`;
  }
}

class Agent {
  constructor(private provider: Provider) {}
  async chat(message: string) { return this.provider.chat(message); }
}

function createProvider(name: string): Provider {
  switch (name) {
    case "echo": return new EchoProvider();
    case "mock_llm": return new MockLlmProvider("gpt-4-mock");
    default: throw new Error(`Unknown: ${name}`);
  }
}

const agent = new Agent(createProvider("mock_llm"));
await agent.chat("Hello!");
```

| 方面 | Rust | TypeScript |
|------|------|-----------|
| 动态分发 | `Box<dyn Provider>`（显式） | `Provider`（隐式，天然动态） |
| 线程安全 | `Send + Sync` 编译时保证 | 无需（单线程） |
| 工厂返回 | `Result<Box<dyn Provider>>` | `Provider` |
| 内存管理 | `Box` 堆分配，自动释放 | GC 自动管理 |

## ZeroClaw 真实架构对照

| 方面 | 简化版 | ZeroClaw 真实版 |
|------|--------|----------------|
| Provider 方法数 | 2 个 | 12+ 个（含流式、工具调用、vision） |
| Provider 实现数 | 2 个 | 12 个（Anthropic, OpenAI, Ollama...） |
| Agent 构建 | `Agent::new(provider)` | `AgentBuilder::new().provider(p).channel(c).build()` |
| Tool 注册 | `.add_tool(Box::new(t))` | `ToolDispatcher` 统一调度 |
| 配置来源 | 硬编码字符串 | `config.toml` + 环境变量 + CLI 参数 |

真实的 AgentBuilder 模式（源码 `src/agent/`）：

```rust
// ZeroClaw 真实代码风格（简化展示）
let agent = AgentBuilder::new()
    .provider(create_provider("anthropic")?)    // Box<dyn Provider>
    .channel(create_channel("telegram")?)       // Box<dyn Channel>
    .tools(vec![Box::new(ShellTool), Box::new(FileTool)])
    .memory(create_memory("sqlite")?)           // Box<dyn Memory>
    .build()?;
// 所有组件都是 Box<dyn Trait> -- 完全可插拔
```

核心洞察：**ZeroClaw 的整个架构就是 `Box<dyn Trait>` 的组合**。理解了本节的简化版，你就理解了 ZeroClaw 70+ 个模块的组织方式。

## 关键学习点

**1. Trait Object 是可插拔架构的基石** -- `Box<dyn Provider>` = 堆分配 + 类型擦除 + 动态分发，让 Agent 不关心具体用的是哪个 Provider。

**2. 工厂模式 + match = 运行时组件选择** -- `config.toml` 里写 `"ollama"`，工厂就创建 `OllamaProvider`，其他代码零修改。

**3. Send + Sync 是异步 Trait 的必备约束** -- 没有它，`Box<dyn Provider>` 无法跨 `.await` 使用（tokio 可能在不同线程恢复任务）。

**4. Builder 模式实现链式调用** -- `fn add_tool(mut self, ...) -> Self` 返回 self，支持 `.add_tool(a).add_tool(b)` 链式写法。

**5. impl Into\<String\> 让 API 更灵活** -- `&str` 和 `String` 都能传，ZeroClaw 构造函数的标配模式。
