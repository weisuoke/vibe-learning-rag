# L2_Multi-Agent协作 - 知识点列表

> 掌握 Codex CLI 的多代理协作机制，实现复杂任务的并行执行

---

## 学习目标

- 深入理解 Multi-Agent 架构原理
- 掌握 Spawn 机制和子代理管理
- 学会使用 Git Worktrees 实现并行执行
- 能够编排 Agent Team 完成复杂任务

---

## 知识点列表

### 01. Multi-Agent 架构与原理 ⭐⭐⭐

**核心内容：**
- 官方 multi_agent 配置启用
- Orchestrator-Worker 模式
- 子代理上下文隔离与转录持久化
- 并发限制与配额管理（最多 6-24 个代理）

**学习重点：**
- 理解 Orchestrator-Worker 架构模式
- 掌握子代理的上下文隔离机制
- 学会管理并发代理数量

**实战场景：**
- 启用 multi_agent 实验功能
- 配置并发代理限制
- 查看子代理转录日志

---

### 02. Spawn 机制深度解析 ⭐⭐⭐

**核心内容：**
- spawn_agent 原生命令
- 显式 vs 自动 spawn
- 子代理生命周期管理（wait, close_agent）
- 并行 vs 顺序执行策略

**学习重点：**
- 掌握 spawn_agent 命令的使用
- 理解显式和自动 spawn 的区别
- 学会管理子代理的生命周期

**实战场景：**
- 手动 spawn 子代理执行任务
- 配置自动 spawn 策略
- 等待子代理完成并收集结果

---

### 03. Git Worktrees 并行执行 ⭐⭐⭐

**核心内容：**
- 多 worktree 隔离工作空间
- 避免文件冲突的最佳实践
- 社区工具：TSK, Emdash 2.0
- 并行运行多个 Codex 实例

**学习重点：**
- 理解 Git Worktrees 的工作原理
- 掌握多 worktree 的创建和管理
- 学会使用社区工具实现并行化

**实战场景：**
- 创建多个 worktree 隔离工作空间
- 在不同 worktree 中并行运行 Codex
- 使用 TSK 或 Emdash 工具管理并行任务

---

### 04. Agent Team 编排实战 ⭐⭐

**核心内容：**
- 真实 multi-agent group chat 实现
- 团队领导完全自主管理子代理
- 复杂任务分解与结果收集
- 审查循环与质量控制

**学习重点：**
- 理解 Agent Team 的编排模式
- 掌握任务分解和分配策略
- 学会实现审查循环

**实战场景：**
- 创建 Agent Team 完成复杂项目
- 实现团队领导自主管理子代理
- 设置审查循环确保代码质量

---

## 学习路径建议

1. **先学架构原理**：理解 Multi-Agent 的设计理念
2. **再学 Spawn 机制**：掌握子代理的创建和管理
3. **实践并行执行**：使用 Git Worktrees 实现真正的并行化
4. **综合编排**：完成 Agent Team 实战项目
