# 实战代码 - 场景1：基础快照操作

## 场景说明

本场景演示如何在 LangGraph 中使用 `get_state()` 获取状态快照，并逐字段解读 `StateSnapshot` 的完整结构。你将看到：

1. 配置 `InMemorySaver` 启用 checkpoint 持久化
2. 执行多步骤图后获取最新快照
3. 逐一解读 `StateSnapshot` 的 8 个字段
4. 通过 `checkpoint_id` 获取特定历史快照
5. 通过 `parent_config` 回溯快照链

这是状态快照的最基础用法——理解 `StateSnapshot` 每个字段的含义，是后续历史遍历、状态修改、时间旅行的前提。

[来源: reference/source_状态快照_01.md | LangGraph 源码分析]
[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

## 完整代码

```python
"""
LangGraph 状态快照实战 - 场景1：基础快照操作
演示：get_state() 获取快照 + 解读 StateSnapshot 各字段

核心知识点：
- InMemorySaver 配置
- graph.get_state(config) 获取最新快照
- StateSnapshot 的 8 个字段含义
- 通过 checkpoint_id 获取特定快照
- 通过 parent_config 回溯快照链

运行环境：Python 3.13+, langgraph
安装依赖：uv add langgraph
"""

from typing import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver


# ============================================================
# 1. 定义状态 —— 带 Reducer 的多字段状态
# ============================================================

class PipelineState(TypedDict):
    """模拟一个简单的数据处理管道状态

    messages 使用 add reducer（追加模式），
    step_count 和 current_step 使用覆盖模式。
    """
    messages: Annotated[list[str], add]  # 追加模式：每个节点的日志会累积
    step_count: int                       # 覆盖模式：记录当前步骤编号
    current_step: str                     # 覆盖模式：记录当前步骤名称


# ============================================================
# 2. 定义节点函数
# ============================================================

def step_load(state: PipelineState) -> dict:
    """步骤 A：数据加载"""
    print(f"  [step_load] 执行数据加载...")
    return {
        "messages": ["数据加载完成：读取了 100 条记录"],
        "step_count": 1,
        "current_step": "load",
    }


def step_process(state: PipelineState) -> dict:
    """步骤 B：数据处理"""
    print(f"  [step_process] 执行数据处理...")
    return {
        "messages": ["数据处理完成：清洗并转换了 95 条有效记录"],
        "step_count": 2,
        "current_step": "process",
    }


def step_save(state: PipelineState) -> dict:
    """步骤 C：数据保存"""
    print(f"  [step_save] 执行数据保存...")
    return {
        "messages": ["数据保存完成：写入向量数据库"],
        "step_count": 3,
        "current_step": "save",
    }


# ============================================================
# 3. 构建图 —— 线性三步管道
# ============================================================

builder = StateGraph(PipelineState)
builder.add_node("step_load", step_load)
builder.add_node("step_process", step_process)
builder.add_node("step_save", step_save)
builder.add_edge(START, "step_load")
builder.add_edge("step_load", "step_process")
builder.add_edge("step_process", "step_save")
builder.add_edge("step_save", END)

# 关键：配置 InMemorySaver 作为 checkpointer
# 没有 checkpointer，get_state() 会报错
memory = InMemorySaver()
graph = builder.compile(checkpointer=memory)


# ============================================================
# 4. 执行图 —— 必须提供 thread_id
# ============================================================

print("=" * 60)
print("=== 执行数据处理管道 ===")
print("=" * 60)

# thread_id 是 checkpoint 的命名空间，同一个 thread_id 下的快照形成一条链
config = {"configurable": {"thread_id": "pipeline-001"}}

result = graph.invoke(
    {"messages": [], "step_count": 0, "current_step": "init"},
    config,
)

print(f"\n最终结果:")
print(f"  messages: {result['messages']}")
print(f"  step_count: {result['step_count']}")
print(f"  current_step: {result['current_step']}")


# ============================================================
# 5. 获取最新快照 —— get_state() 核心用法
# ============================================================

print()
print("=" * 60)
print("=== StateSnapshot 字段逐一解读 ===")
print("=" * 60)

snapshot = graph.get_state(config)

# --- 字段 1: values ---
# 当前状态的所有 channel 值，就是图执行完毕后的最终状态
print("\n--- 字段 1: values（当前状态值） ---")
print(f"  类型: {type(snapshot.values).__name__}")
print(f"  内容: {snapshot.values}")
# --- 字段 2: next ---
# 下一步要执行的节点名元组。图执行完毕时为空元组 ()
# 如果图在中间被中断（interrupt），这里会显示下一个待执行的节点
print("\n--- 字段 2: next（下一步节点） ---")
print(f"  类型: {type(snapshot.next).__name__}")
print(f"  内容: {snapshot.next}")
print(f"  说明: 空元组表示图已执行完毕")

# --- 字段 3: config ---
# 获取此快照的配置，包含 thread_id 和 checkpoint_id
# checkpoint_id 是这个快照的唯一标识符
print("\n--- 字段 3: config（快照配置） ---")
print(f"  thread_id: {snapshot.config['configurable']['thread_id']}")
print(f"  checkpoint_id: {snapshot.config['configurable']['checkpoint_id']}")
print(f"  checkpoint_ns: {snapshot.config['configurable'].get('checkpoint_ns', '')}")

# --- 字段 4: metadata ---
# 快照的元数据，包含 source、step、run_id 等
# source 表示这个 checkpoint 是怎么产生的：
#   "input"  → 来自 invoke/stream 的初始输入
#   "loop"   → 来自图执行循环中的节点
#   "update" → 来自手动 update_state()
#   "fork"   → 从另一个 checkpoint 复制
print("\n--- 字段 4: metadata（元数据） ---")
print(f"  source: {snapshot.metadata.get('source')}")
print(f"  step: {snapshot.metadata.get('step')}")
print(f"  writes: {snapshot.metadata.get('writes')}")

# --- 字段 5: created_at ---
# ISO 8601 格式的创建时间戳
print("\n--- 字段 5: created_at（创建时间） ---")
print(f"  时间戳: {snapshot.created_at}")

# --- 字段 6: parent_config ---
# 父快照的配置。通过它可以回溯到上一个快照
# 最早的快照（初始输入）的 parent_config 为 None
print("\n--- 字段 6: parent_config（父快照配置） ---")
if snapshot.parent_config:
    parent_cp_id = snapshot.parent_config["configurable"]["checkpoint_id"]
    print(f"  父 checkpoint_id: {parent_cp_id}")
    print(f"  说明: 可以用这个 ID 获取上一步的快照")
else:
    print(f"  值: None（这是最早的快照）")

# --- 字段 7: tasks ---
# 当前步骤的任务列表。图执行完毕时为空元组
# 每个 PregelTask 包含 id, name, path, error, interrupts, state, result
print("\n--- 字段 7: tasks（当前任务） ---")
print(f"  类型: {type(snapshot.tasks).__name__}")
print(f"  数量: {len(snapshot.tasks)}")
if snapshot.tasks:
    for task in snapshot.tasks:
        print(f"    任务: {task.name}, ID: {task.id}")

# --- 字段 8: interrupts ---
# 待处理的中断列表。没有中断时为空元组
print("\n--- 字段 8: interrupts（中断列表） ---")
print(f"  数量: {len(snapshot.interrupts)}")
print(f"  说明: 空元组表示没有待处理的中断")


# ============================================================
# 6. 通过 checkpoint_id 获取特定快照
# ============================================================

print()
print("=" * 60)
print("=== 通过 checkpoint_id 获取特定快照 ===")
print("=" * 60)

# 先记录当前快照的 checkpoint_id
current_cp_id = snapshot.config["configurable"]["checkpoint_id"]
print(f"\n当前快照 checkpoint_id: {current_cp_id}")

# 如果有父快照，获取它
if snapshot.parent_config:
    parent_cp_id = snapshot.parent_config["configurable"]["checkpoint_id"]

    # 构造包含 checkpoint_id 的 config 来获取特定快照
    parent_config = {
        "configurable": {
            "thread_id": "pipeline-001",
            "checkpoint_id": parent_cp_id,
        }
    }
    parent_snapshot = graph.get_state(parent_config)

    print(f"\n父快照信息:")
    print(f"  checkpoint_id: {parent_cp_id}")
    print(f"  values: {parent_snapshot.values}")
    print(f"  metadata.source: {parent_snapshot.metadata.get('source')}")
    print(f"  metadata.step: {parent_snapshot.metadata.get('step')}")
    print(f"  next: {parent_snapshot.next}")


# ============================================================
# 7. 回溯快照链 —— 从最新快照一路追溯到初始输入
# ============================================================

print()
print("=" * 60)
print("=== 回溯快照链（parent_config 链式追溯） ===")
print("=" * 60)

current = snapshot
chain_index = 0

while current is not None:
    cp_id = current.config["configurable"]["checkpoint_id"]
    source = current.metadata.get("source", "unknown") if current.metadata else "unknown"
    step = current.metadata.get("step", "?") if current.metadata else "?"
    next_nodes = current.next

    print(f"\n  [{chain_index}] checkpoint_id: {cp_id}")
    print(f"      source: {source}, step: {step}")
    print(f"      next: {next_nodes}")
    print(f"      step_count: {current.values.get('step_count', 'N/A')}")
    print(f"      current_step: {current.values.get('current_step', 'N/A')}")
    print(f"      messages 数量: {len(current.values.get('messages', []))}")

    # 尝试获取父快照
    if current.parent_config:
        current = graph.get_state(current.parent_config)
        chain_index += 1
    else:
        print(f"\n  到达链的起点（无 parent_config）")
        current = None

print(f"\n  快照链总长度: {chain_index + 1}")


# ============================================================
# 8. 多线程隔离验证
# ============================================================

print()
print("=" * 60)
print("=== 多线程隔离验证 ===")
print("=" * 60)

# 用不同的 thread_id 执行同一个图
config_thread2 = {"configurable": {"thread_id": "pipeline-002"}}
result2 = graph.invoke(
    {"messages": [], "step_count": 0, "current_step": "init"},
    config_thread2,
)

# 获取两个线程的快照
snap1 = graph.get_state(config)
snap2 = graph.get_state(config_thread2)

print(f"\n线程 1 (pipeline-001):")
print(f"  checkpoint_id: {snap1.config['configurable']['checkpoint_id']}")
print(f"  messages 数量: {len(snap1.values.get('messages', []))}")

print(f"\n线程 2 (pipeline-002):")
print(f"  checkpoint_id: {snap2.config['configurable']['checkpoint_id']}")
print(f"  messages 数量: {len(snap2.values.get('messages', []))}")

print(f"\n两个线程的 checkpoint_id 不同: "
      f"{snap1.config['configurable']['checkpoint_id'] != snap2.config['configurable']['checkpoint_id']}")
print(f"说明: 不同 thread_id 的快照完全隔离，互不影响")
```

## 关键知识点解析

### 1. StateSnapshot 字段速查表

| 字段 | 类型 | 含义 | 图执行完毕时的值 |
|------|------|------|-----------------|
| `values` | `dict` | 当前状态的所有 channel 值 | 最终状态 |
| `next` | `tuple[str, ...]` | 下一步要执行的节点 | `()` 空元组 |
| `config` | `RunnableConfig` | 包含 thread_id 和 checkpoint_id | 最新 checkpoint 的配置 |
| `metadata` | `CheckpointMetadata` | source、step、writes 等元信息 | `source="loop"` |
| `created_at` | `str` | ISO 8601 时间戳 | 最后一步的时间 |
| `parent_config` | `RunnableConfig \| None` | 父快照配置，用于回溯 | 上一步的配置 |
| `tasks` | `tuple[PregelTask, ...]` | 当前步骤的任务 | `()` 空元组 |
| `interrupts` | `tuple[Interrupt, ...]` | 待处理的中断 | `()` 空元组 |

### 2. metadata.source 的四种来源

| source 值 | 含义 | 触发时机 |
|-----------|------|---------|
| `"input"` | 初始输入 | `graph.invoke()` 的输入被保存为第一个 checkpoint |
| `"loop"` | 循环执行 | 图执行过程中每个节点完成后自动保存 |
| `"update"` | 手动更新 | 调用 `graph.update_state()` 时创建 |
| `"fork"` | 分叉 | 从另一个 checkpoint 复制并修改时创建 |

### 3. 快照链的结构

一次完整的图执行会产生如下快照链（以本例的三步管道为例）：

```
[input, step=-1] → [loop, step=0] → [loop, step=1] → [loop, step=2] → [loop, step=3]
   __start__         step_load        step_process       step_save          END
```

- `step=-1` 是初始输入的 checkpoint（source="input"）
- `step=0` 是 `__start__` 节点处理输入后的 checkpoint
- `step=1~3` 是各业务节点执行后的 checkpoint
- 每个快照的 `next` 字段指向下一个要执行的节点

### 4. 必须配置 checkpointer

```python
# 没有 checkpointer → get_state() 会报错
graph = builder.compile()  # 不传 checkpointer
graph.get_state(config)    # ValueError!

# 有 checkpointer → 正常工作
memory = InMemorySaver()
graph = builder.compile(checkpointer=memory)
graph.get_state(config)    # 返回 StateSnapshot
```

`InMemorySaver` 是最简单的 checkpointer，数据存在内存中，进程结束就丢失。生产环境应使用 `PostgresSaver` 或 `SqliteSaver`。

## 在 RAG 系统中的应用

状态快照在 RAG 系统中的典型用途：

- **调试检索质量**：获取检索步骤的快照，查看 `retrieved_docs` 字段，判断检索结果是否相关
- **监控管道状态**：通过 `metadata.step` 判断管道执行到了哪一步
- **错误定位**：当生成结果不理想时，回溯快照链找到问题出在哪个步骤
- **多轮对话恢复**：通过 `thread_id` 隔离不同用户的对话状态
