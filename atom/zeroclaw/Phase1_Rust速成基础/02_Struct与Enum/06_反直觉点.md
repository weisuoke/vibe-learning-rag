# 反直觉点：Struct 与 Enum 的 10 个认知陷阱

> **一句话定位：** 从 TypeScript 转 Rust 时，这 10 个"我以为应该这样"会让你踩坑——提前知道，少走弯路。

---

## 反直觉 1：Enum 变体可以携带完全不同的数据

### 直觉

"Enum 就是一组常量，像 TypeScript 的 `enum Direction { Up, Down }`"

### 现实

Rust 的 enum 变体可以各自携带**完全不同类型和数量**的数据：

```rust
enum ConversationMessage {
    Chat { role: String, content: String },  // 两个 String 字段
    ToolCall { name: String, input: String }, // 两个 String 字段
    Error(String),                           // 一个 String
}
```

### 为什么

Rust 的 enum 是**tagged union**（带标签的联合体），不是简单的整数常量集。它等价于 TypeScript 的 discriminated union，但内置在语言层面。

## 反直觉 2：Enum 的内存大小 = 最大变体

### 直觉

"每个变体只占自己需要的空间"

### 现实

```rust
enum Event {
    Click,                       // 0 字节数据
    Data(String),                // 24 字节
    BigPayload([u8; 1024]),      // 1024 字节
}
// size_of::<Event>() ≈ 1032 字节——所有变体都占这么多！
```

### 为什么

Enum 在栈上是固定大小的。编译器必须为**最大变体**预留空间，因为运行时可能是任何一个变体。解决方案：`Box` 大变体。

```rust
enum EventOptimized {
    Click,
    Data(String),
    BigPayload(Box<[u8; 1024]>),  // 栈上只有 8 字节指针
}
// size_of::<EventOptimized>() ≈ 32 字节
```

## 反直觉 3：Struct 创建时必须填所有字段

### 直觉

"像 TypeScript 一样，没填的字段自动是 undefined"

### 现实

```rust
struct Agent { name: String, model: String, temperature: f64 }

// ❌ 编译错误：missing field `temperature`
let a = Agent { name: "bot".into(), model: "gpt-4".into() };
```

### 为什么

Rust 没有 `null`/`undefined`。每个字段必须有确定的值。想要可选字段？用 `Option<T>`：

```rust
struct Agent {
    name: String,
    description: Option<String>,  // 显式标记"可以没有"
}
let a = Agent { name: "bot".into(), description: None };  // ✓
```

## 反直觉 4：match 的 `_` 通配符是双刃剑

### 直觉

"加个 `_ =>` 兜底就安全了"

### 现实

```rust
enum Level { ReadOnly, Supervised, Full }

fn check(level: &Level) -> &str {
    match level {
        Level::Full => "dangerous",
        _ => "safe",  // 看起来安全...
    }
}

// 半年后新增变体：
// enum Level { ReadOnly, Supervised, Full, Admin }
// Admin 被 _ 默默吞掉，返回 "safe"——这是 bug！
```

### 为什么

`_` 匹配所有未列出的变体，包括**未来新增的**。编译器不会提醒你。优先写全所有分支，只在确实不关心剩余情况时才用 `_`。

## 反直觉 5：Copy 和 Clone 不是一回事

### 直觉

"Clone 就是 Copy，都是复制"

### 现实

| | Copy | Clone |
|--|------|-------|
| 触发方式 | 隐式（赋值时自动） | 显式（`.clone()`） |
| 性能 | 按位拷贝（极快） | 可能深拷贝（可能慢） |
| 限制 | 只能用于栈上小类型 | 任何类型都可以 |

```rust
// Copy：隐式拷贝，原值仍可用
let a = 42;
let b = a;       // Copy，a 仍然可用
println!("{}", a); // ✓

// Clone：显式调用
let s1 = String::from("hello");
let s2 = s1.clone();  // 必须显式调用
println!("{}", s1);    // ✓（因为 clone 了）

// 没有 Clone 也没有 Copy：move
let s3 = String::from("world");
let s4 = s3;          // move！s3 不再可用
// println!("{}", s3); // ❌ 编译错误
```

### 为什么

Copy 是 Clone 的子集。Copy 要求类型完全在栈上（不含堆数据）。String/Vec/Box 有堆数据，不能 Copy，只能 Clone。

## 反直觉 6：derive(PartialEq) 不等于 derive(Eq)

### 直觉

"能 `==` 比较就够了，PartialEq 和 Eq 有什么区别？"

### 现实

```rust
let x = f64::NAN;
println!("{}", x == x);  // false！NaN 不等于自身
```

- `PartialEq`：允许 `a == a` 为 false（如 NaN）
- `Eq`：保证 `a == a` 永远为 true（自反性）

**实际影响**：`HashMap` 的 key 必须实现 `Eq + Hash`。含 `f64` 的类型不能 derive `Eq`。

## 反直觉 7：Rust 没有继承，但不缺多态

### 直觉

"没有 class extends，怎么实现多态？"

### 现实

Rust 用 **trait + 组合** 替代继承：

```rust
// TypeScript 思维：继承链
// class Base { ... }
// class Child extends Base { ... }

// Rust 思维：trait 定义行为，struct 组合数据
trait Provider { fn chat(&self, msg: &str) -> String; }

struct Agent {
    provider: Box<dyn Provider>,  // 多态：可以是任何 Provider 实现
}
```

### 为什么

继承的问题：脆弱基类、菱形继承、紧耦合。组合 + trait 更灵活——每个组件独立，通过接口通信。

## 反直觉 8：Option\<T\> 和 T 是完全不同的类型

### 直觉

"Option\<String\> 就是 String，只是可能为空"

### 现实

```rust
let name: String = "bot".into();
let maybe: Option<String> = Some("bot".into());

// ❌ 不能直接对 Option<String> 调 String 的方法
// maybe.to_uppercase();  // 编译错误！

// 必须先"解包"
let upper = maybe.map(|s| s.to_uppercase());  // Option<String>
let upper = maybe.unwrap_or_default().to_uppercase();  // String
```

### 为什么

`Option<T>` 是一个独立的 enum（`Some(T)` 或 `None`），不是 `T` 的"可空版本"。编译器强制你在使用前确认值存在。这消除了 null pointer exception。

## 反直觉 9：struct update 语法 `..` 会 move 源对象

### 直觉

"`..default` 只是复制默认值，原对象还能用"

### 现实

```rust
let base = AgentConfig { name: "a".into(), model: "gpt-4".into(), temperature: 0.7 };
let custom = AgentConfig { name: "b".into(), ..base };
// println!("{:?}", base);  // ❌ 编译错误！base.model 被 move 了
```

### 为什么

`..base` 会 move 未显式赋值的字段。String 不是 Copy，所以 `model` 被 move 到 `custom`。解决方案：

```rust
let custom = AgentConfig { name: "b".into(), ..base.clone() };  // clone 后再用
```

## 反直觉 10：空 struct 和空 enum 都有用

### 直觉

"没有字段的 struct / 没有变体的 enum 有什么用？"

### 现实

```rust
// 单元 struct：用作类型标记（零大小，Zero-Sized Type）
struct Validated;
struct Unvalidated;

struct Form<State> { data: String, _state: std::marker::PhantomData<State> }
// Form<Unvalidated> 和 Form<Validated> 是不同类型，编译器阻止你跳过验证

// 空 enum：表示"不可能发生"
enum Never {}
// 函数返回 Never 意味着永远不会正常返回（如 infinite loop 或 panic）
```

### 为什么

Rust 的类型系统不只是存数据——类型本身就是信息。空类型用于编译期约束，运行时零开销。

## 速查卡

```
陷阱速查：
  1. Enum ≠ 常量集        变体可携带不同数据（tagged union）
  2. Enum 大小 = 最大变体  Box 大变体优化内存
  3. 必须填所有字段        用 Option<T> 表达可选
  4. _ 通配符吞新变体      优先写全分支
  5. Copy ≠ Clone          Copy 隐式栈拷贝，Clone 显式可深拷贝
  6. PartialEq ≠ Eq        f64 有 NaN，不能 Eq
  7. 无继承但有多态        trait + Box<dyn Trait> 组合
  8. Option<T> ≠ T         必须解包才能用
  9. ..base 会 move        用 ..base.clone() 保留原对象
  10. 空类型有意义          类型标记 + 编译期约束
```
