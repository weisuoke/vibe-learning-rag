## 案例 1：简化版 with_connection

ZeroClaw 的 Cron 模块（`src/cron/store.rs`）有一个经典的泛型工厂函数：

```rust
// ZeroClaw 源码：src/cron/store.rs（简化版）
fn with_connection<T>(config: &Config, f: impl FnOnce(&Connection) -> Result<T>) -> Result<T> {
    let db_path = config.workspace_dir.join("cron").join("jobs.db");
    let conn = Connection::open(&db_path)?;
    f(&conn)
}
```

### 逐行解读

```rust
fn with_connection<T>(
//                 ^^ 泛型参数 T -- 调用者决定返回什么类型
    config: &Config,
//          ^^^^^^^ 普通参数：借用一个配置对象
    f: impl FnOnce(&Connection) -> Result<T>,
//     ^^^^ ^^^^^^^ impl Trait 语法糖，等价于 where F: FnOnce(...)
//          FnOnce = 只能调用一次的闭包（消耗所有权）
//          接收 &Connection 引用，返回 Result<T>
) -> Result<T> {
//   ^^^^^^^^^ 返回类型也用了泛型 T -- 闭包返回什么，函数就返回什么
    let conn = Connection::open(&db_path)?;
//                                       ^ ? 操作符：打开失败则提前返回错误
    f(&conn)
//  ^^^^^^^ 把连接的引用传给闭包，闭包执行具体操作
}
```

**核心思想：** 函数负责"打开连接 + 清理资源"，调用者通过闭包决定"用连接做什么"。T 让返回值完全灵活。

### TypeScript 对照

```typescript
// TypeScript 版本 -- 泛型 + 回调，结构几乎一样
function withConnection<T>(config: Config, f: (conn: Connection) => T): T {
  const conn = new Connection(config.dbPath);
  try {
    return f(conn);
  } finally {
    conn.close(); // TS 需要手动清理；Rust 自动 Drop（RAII）
  }
}
```

## 案例 2：create_memory_with_builders 完整解读

这是 ZeroClaw 源码中最能体现"泛型 + where 子句"威力的函数（`src/memory/mod.rs`）：

```rust
// ZeroClaw 源码：src/memory/mod.rs（完整签名）
fn create_memory_with_builders<F, G>(
    backend_name: &str,
    workspace_dir: &Path,
    mut sqlite_builder: F,
    mut postgres_builder: G,
    unknown_context: &str,
) -> anyhow::Result<Box<dyn Memory>>
where
    F: FnMut() -> anyhow::Result<SqliteMemory>,
    G: FnMut() -> anyhow::Result<Box<dyn Memory>>,
{
    match classify_memory_backend(backend_name) {
        MemoryBackendKind::Sqlite  => Ok(Box::new(sqlite_builder()?)),
        MemoryBackendKind::Lucid   => {
            let local = sqlite_builder()?;
            Ok(Box::new(LucidMemory::new(workspace_dir, local)))
        }
        MemoryBackendKind::Postgres => postgres_builder(),
        MemoryBackendKind::Markdown => Ok(Box::new(MarkdownMemory::new(workspace_dir))),
        MemoryBackendKind::None     => Ok(Box::new(NoneMemory::new())),
        MemoryBackendKind::Unknown  => {
            tracing::warn!("Unknown memory backend '{backend_name}'{unknown_context}");
            Ok(Box::new(MarkdownMemory::new(workspace_dir)))
        }
    }
}
```

### 逐参数解读

```rust
fn create_memory_with_builders<F, G>(
//                             ^  ^ 两个泛型参数，代表两种不同的构建器
    backend_name: &str,        // 后端名称，如 "sqlite"、"postgres"
    workspace_dir: &Path,      // 工作目录路径
    mut sqlite_builder: F,     // SQLite 构建器（mut 因为要调用 FnMut）
    mut postgres_builder: G,   // Postgres 构建器
    unknown_context: &str,     // 错误提示上下文
) -> anyhow::Result<Box<dyn Memory>>
//   ^^^^^^^^^^^^^^ ^^^^^^^^^^^^^^^^
//   可能失败       返回 Trait Object -- 调用者不需要知道具体类型
```

### 逐 where 行解读

```rust
where
    F: FnMut() -> anyhow::Result<SqliteMemory>,
//  ^  ^^^^^^     ^^^^^^^^^^^^^^ ^^^^^^^^^^^^^
//  F  可多次调用  可能失败        返回具体类型 SqliteMemory
    G: FnMut() -> anyhow::Result<Box<dyn Memory>>,
//  ^  ^^^^^^     ^^^^^^^^^^^^^^ ^^^^^^^^^^^^^^^^^
//  G  可多次调用  可能失败        返回 Trait Object（任意 Memory 实现）
```

### 为什么用 FnMut 而不是 FnOnce？

```rust
// FnOnce -- 只能调用一次（消耗闭包自身）
// FnMut  -- 可以调用多次（可变借用闭包）
// Fn     -- 可以调用多次（不可变借用闭包）

// 看源码：sqlite_builder 被调用了两次！
MemoryBackendKind::Sqlite => Ok(Box::new(sqlite_builder()?)),  // 第 1 次
MemoryBackendKind::Lucid  => {
    let local = sqlite_builder()?;                              // 第 2 次
    // ...
}
```

虽然 match 只会走一个分支，但编译器看到 `sqlite_builder` 出现在两个分支中，就要求它**可以被调用多次** -- 所以必须用 `FnMut`，不能用 `FnOnce`。

```typescript
// TypeScript 对照：TS 没有这个区分
// 所有函数都可以调用任意次，不需要 FnOnce/FnMut/Fn 的概念
type SqliteBuilder = () => SqliteMemory;  // 就一种
```

### 返回 Box<dyn Memory>：泛型 + Trait Object 结合

```rust
-> anyhow::Result<Box<dyn Memory>>
//                 ^^^^^^^^^^^^^^^^
//                 Box  = 堆分配（不同实现大小不同，需要统一）
//                 dyn  = 动态分发（运行时决定调用哪个实现）
//                 Memory = 保证有 store/recall/forget 等方法
```

这是 Rust 工厂模式的标准返回类型。TypeScript 天然动态分发（`function createMemory(): Memory`），Rust 需要 `Box<dyn Trait>` 实现同样效果。

## 案例 3：自己写一个泛型工厂

从硬编码版本 `fn create_echo_tool() -> Box<dyn Tool> { Box::new(EchoTool) }` 开始 -- 问题是只能创建一种工具。升级为泛型版本：

```rust
fn create_tool_with_builder<F, T>(builder: F) -> anyhow::Result<Box<dyn Tool>>
where
    F: FnOnce() -> anyhow::Result<T>,  // 构建器：生产一个 T
    T: Tool + 'static,                  // T 必须实现 Tool，'static 表示不含临时引用
{
    let tool = builder()?;
    Ok(Box::new(tool))
}
```

这里用 `FnOnce` 就够了 -- 构建器只需要调用一次。

### 使用示例

```rust
// 创建不同的工具，同一个工厂函数
let echo: Box<dyn Tool> = create_tool_with_builder(|| Ok(EchoTool))?;
let calc: Box<dyn Tool> = create_tool_with_builder(|| {
    // 构建器可以包含初始化逻辑
    let precision = load_config()?.precision;
    Ok(CalculatorTool { precision })
})?;
```

## 完整可运行代码（简化版，不依赖外部库，用 println 模拟）

将以下代码保存为 `main.rs`，用 `cargo run` 即可运行：

```rust
// main.rs -- 泛型工厂函数完整示例
// 运行方式：cargo run（无外部依赖）

// ========== Trait 定义 ==========

trait Memory {
    fn name(&self) -> &str;
    fn store(&self, key: &str, value: &str);
}

trait Tool {
    fn name(&self) -> &str;
    fn execute(&self, input: &str) -> String;
}

// ========== Memory 实现 ==========

struct SqliteMemory { db_path: String }
struct MarkdownMemory { dir: String }

impl Memory for SqliteMemory {
    fn name(&self) -> &str { "sqlite" }
    fn store(&self, key: &str, value: &str) {
        println!("  [SQLite] INSERT (key={}, value={})", key, value);
    }
}
impl Memory for MarkdownMemory {
    fn name(&self) -> &str { "markdown" }
    fn store(&self, key: &str, value: &str) {
        println!("  [Markdown] Writing {}.md: {}", key, value);
    }
}

// ========== Tool 实现 ==========

struct EchoTool;
struct UpperTool;

impl Tool for EchoTool {
    fn name(&self) -> &str { "echo" }
    fn execute(&self, input: &str) -> String { input.to_string() }
}
impl Tool for UpperTool {
    fn name(&self) -> &str { "upper" }
    fn execute(&self, input: &str) -> String { input.to_uppercase() }
}

// ========== 泛型工厂函数 ==========

/// 案例 1：with_connection 模式 -- 泛型 T 让返回值由闭包决定
fn with_db<T>(db_name: &str, f: impl FnOnce(&str) -> T) -> T {
    println!("[DB] Opening '{}'", db_name);
    let result = f(db_name);
    println!("[DB] Closed");
    result
}

/// 案例 2：多构建器 + where 子句 + Box<dyn Trait>
fn create_memory_with_builders<F, G>(
    backend: &str,
    mut sqlite_builder: F,
    mut markdown_builder: G,
) -> Result<Box<dyn Memory>, String>
where
    F: FnMut() -> Result<SqliteMemory, String>,
    G: FnMut() -> Result<MarkdownMemory, String>,
{
    match backend {
        "sqlite" => Ok(Box::new(sqlite_builder()?)),
        "markdown" => Ok(Box::new(markdown_builder()?)),
        other => {
            println!("[WARN] Unknown '{}', fallback to markdown", other);
            Ok(Box::new(markdown_builder()?))
        }
    }
}

/// 案例 3：泛型 Tool 工厂
fn create_tool_with_builder<F, T>(builder: F) -> Result<Box<dyn Tool>, String>
where
    F: FnOnce() -> Result<T, String>,
    T: Tool + 'static,
{
    Ok(Box::new(builder()?))
}

// ========== 主函数 ==========

fn main() {
    // 案例 1：同一个函数，不同返回类型
    let count: usize = with_db("jobs.db", |_| 42);           // T = usize
    let names: Vec<String> = with_db("jobs.db", |_| vec!["backup".into()]); // T = Vec<String>
    println!("count={}, names={:?}\n", count, names);

    // 案例 2：根据配置创建不同 Memory 后端
    for backend in &["sqlite", "markdown", "redis"] {
        let mem = create_memory_with_builders(
            backend,
            || Ok(SqliteMemory { db_path: "memory.db".into() }),
            || Ok(MarkdownMemory { dir: "./memories".into() }),
        ).unwrap();
        println!("Created: {} backend", mem.name());
        mem.store("greeting", "hello");
    }

    // 案例 3：泛型 Tool 工厂
    println!();
    let tools: Vec<Box<dyn Tool>> = vec![
        create_tool_with_builder(|| Ok(EchoTool)).unwrap(),
        create_tool_with_builder(|| Ok(UpperTool)).unwrap(),
    ];
    for tool in &tools {
        println!("{} -> \"{}\"", tool.name(), tool.execute("hello zeroclaw"));
    }
}
```

`Cargo.toml`（无外部依赖）：

```toml
[package]
name = "generic-factory"
version = "0.1.0"
edition = "2021"
```

## 关键学习点总结

### 1. 泛型 T 让返回值由调用者决定

```rust
fn with_db<T>(db: &str, f: impl FnOnce(&str) -> T) -> T
//         ^                                     ^      ^
//         声明                                  闭包返回 T  函数也返回 T

// 调用时 T 被自动推断，不需要手动指定
let count: usize = with_db("db", |_| 42);        // T = usize
let names: Vec<String> = with_db("db", |_| vec![]); // T = Vec<String>
```

### 2. FnOnce vs FnMut vs Fn 的选择

```rust
// FnOnce -- 闭包只调用一次（最宽松，接受所有闭包）
// FnMut  -- 闭包可调用多次，可修改捕获的变量
// Fn     -- 闭包可调用多次，不可修改捕获的变量（最严格）

// 选择原则：用最宽松的那个，除非你需要多次调用
fn once_ok<F: FnOnce()>(f: F) { f(); }           // 只调用一次 -> FnOnce
fn need_mut<F: FnMut()>(mut f: F) { f(); f(); }  // 多次调用 -> FnMut
```

### 3. where 子句让复杂约束可读

```rust
// 不用 where -- 签名挤在一行，难以阅读
fn create<F: FnMut() -> Result<SqliteMemory, String>, G: FnMut() -> Result<Box<dyn Memory>, String>>(f: F, g: G) {}

// 用 where -- 参数和约束分离，清晰明了
fn create<F, G>(f: F, g: G)
where
    F: FnMut() -> Result<SqliteMemory, String>,
    G: FnMut() -> Result<Box<dyn Memory>, String>,
{}
```

### 4. Box<dyn Trait> 是工厂函数的标准返回类型

```rust
// 工厂函数不知道会返回哪个具体类型 -> 用 Trait Object
fn create_memory(...) -> Box<dyn Memory>
//                       ^^^^^^^^^^^^^^^
// Box  = 堆分配（不同实现大小不同，需要统一大小）
// dyn  = 动态分发（运行时查表调用正确的方法）
// Memory = 保证返回值一定有 name()/store()/recall() 等方法

// TypeScript 天然就是动态分发，不需要 Box<dyn>
// function createMemory(): Memory { ... }
```

## 挑战练习

实现一个 `create_provider` 泛型工厂函数：

**要求：**
- 接收 `provider_name: &str` 和两个构建器闭包
- 根据名称选择调用哪个构建器
- 返回 `Result<Box<dyn Provider>, String>`

**提示骨架：**

```rust
trait Provider {
    fn name(&self) -> &str;
    fn chat(&self, message: &str) -> String;
}

struct OllamaProvider { model: String }
struct OpenAIProvider { api_key: String }

impl Provider for OllamaProvider {
    fn name(&self) -> &str { "ollama" }
    fn chat(&self, msg: &str) -> String { format!("[Ollama/{}] {}", self.model, msg) }
}

impl Provider for OpenAIProvider {
    fn name(&self) -> &str { "openai" }
    fn chat(&self, msg: &str) -> String { format!("[OpenAI] {}", msg) }
}

// 你来实现这个函数：
fn create_provider<F, G>(
    provider_name: &str,
    ollama_builder: F,
    openai_builder: G,
) -> Result<Box<dyn Provider>, String>
where
    F: FnOnce() -> Result<OllamaProvider, String>,
    G: FnOnce() -> Result<OpenAIProvider, String>,
{
    match provider_name {
        "ollama" => Ok(Box::new(ollama_builder()?)),
        "openai" => Ok(Box::new(openai_builder()?)),
        other => Err(format!("Unknown provider: {}", other)),
    }
}

// 测试你的实现：
fn main() {
    let provider = create_provider(
        "ollama",
        || Ok(OllamaProvider { model: "llama3".into() }),
        || Ok(OpenAIProvider { api_key: "sk-test".into() }),
    ).unwrap();

    println!("{}: {}", provider.name(), provider.chat("Hello!"));
    // 输出：ollama: [Ollama/llama3] Hello!
}
```
