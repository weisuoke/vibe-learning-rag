# 核心概念 7：守护进程管理

**本文档深入讲解守护进程（Daemon）的管理，包括 LaunchAgent（macOS）和 systemd（Linux）的配置和故障排查。**

---

## 概述

守护进程（Daemon）是在后台持续运行的服务进程，负责：

1. **自动启动**：系统启动时自动启动 Gateway
2. **崩溃重启**：Gateway 崩溃时自动重启
3. **日志管理**：记录 Gateway 运行日志
4. **资源控制**：限制 Gateway 的资源使用

OpenClaw 支持两种守护进程管理方式：
- **LaunchAgent**（macOS）
- **systemd user service**（Linux）

---

## macOS LaunchAgent

### LaunchAgent 概述

LaunchAgent 是 macOS 的用户级守护进程管理系统，特点：

- **用户级**：每个用户有独立的 LaunchAgent
- **自动启动**：用户登录时自动启动
- **持久化**：用户登出后继续运行（需要 KeepAlive）
- **权限隔离**：以用户权限运行，不需要 root

### LaunchAgent 配置文件

**位置**：

```
~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

**完整配置示例**：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <!-- 服务标识 -->
  <key>Label</key>
  <string>ai.openclaw.gateway</string>

  <!-- 程序路径和参数 -->
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/openclaw</string>
    <string>gateway</string>
    <string>run</string>
    <string>--port</string>
    <string>18789</string>
  </array>

  <!-- 工作目录 -->
  <key>WorkingDirectory</key>
  <string>/Users/username/.openclaw</string>

  <!-- 环境变量 -->
  <key>EnvironmentVariables</key>
  <dict>
    <key>OPENCLAW_GATEWAY_TOKEN</key>
    <string>oc_1a2b3c4d5e6f7g8h9i0j</string>
    <key>NODE_ENV</key>
    <string>production</string>
  </dict>

  <!-- 自动启动 -->
  <key>RunAtLoad</key>
  <true/>

  <!-- 崩溃重启 -->
  <key>KeepAlive</key>
  <dict>
    <key>SuccessfulExit</key>
    <false/>
  </dict>

  <!-- 日志输出 -->
  <key>StandardOutPath</key>
  <string>/Users/username/.openclaw/logs/gateway.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/username/.openclaw/logs/gateway-error.log</string>

  <!-- 资源限制 -->
  <key>SoftResourceLimits</key>
  <dict>
    <key>NumberOfFiles</key>
    <integer>4096</integer>
  </dict>

  <!-- 进程类型 -->
  <key>ProcessType</key>
  <string>Interactive</string>
</dict>
</plist>
```

### 配置字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| **Label** | String | 服务唯一标识（必需） |
| **ProgramArguments** | Array | 程序路径和参数（必需） |
| **WorkingDirectory** | String | 工作目录 |
| **EnvironmentVariables** | Dict | 环境变量 |
| **RunAtLoad** | Boolean | 加载时自动启动 |
| **KeepAlive** | Boolean/Dict | 崩溃重启策略 |
| **StandardOutPath** | String | 标准输出日志路径 |
| **StandardErrorPath** | String | 标准错误日志路径 |
| **SoftResourceLimits** | Dict | 资源限制 |
| **ProcessType** | String | 进程类型（Interactive/Background） |

### LaunchAgent 管理命令

```bash
# 加载服务
launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# 卸载服务
launchctl unload ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# 启动服务

# 停止服务

# 查看服务状态

# 查看服务详情
launchctl list ai.openclaw.gateway

# 重启服务
launchctl stop ai.openclaw.gateway && launchctl start ai.openclaw.gateway
```

## Linux systemd

### systemd 概述

systemd 是 Linux 的系统和服务管理器，特点：

- **用户级服务**：`systemctl --user` 管理用户服务
- **依赖管理**：支持服务依赖和启动顺序
- **资源控制**：支持 cgroup 资源限制
- **日志集成**：集成 journald 日志系统

### systemd 配置文件

**位置**：

```
~/.config/systemd/user/openclaw-gateway.service
```

**完整配置示例**：

```ini
[Unit]
Description=OpenClaw Gateway
Documentation=https://docs.openclaw.ai
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/openclaw gateway run --port 18789
WorkingDirectory=/home/username/.openclaw
Restart=always
RestartSec=5

# 环境变量
Environment="OPENCLAW_GATEWAY_TOKEN=oc_1a2b3c4d5e6f7g8h9i0j"
Environment="NODE_ENV=production"

# 资源限制
LimitNOFILE=4096
MemoryMax=2G
CPUQuota=200%

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=openclaw-gateway

[Install]
WantedBy=default.target
```

### 配置字段说明

| 字段 | 说明 |
|------|------|
| **[Unit]** | 服务元数据 |
| `Description` | 服务描述 |
| `After` | 在指定服务之后启动 |
| `Wants` | 弱依赖（推荐但非必需） |
| **[Service]** | 服务配置 |
| `Type` | 服务类型（simple/forking/oneshot） |
| `ExecStart` | 启动命令 |
| `WorkingDirectory` | 工作目录 |
| `Restart` | 重启策略（always/on-failure/no） |
| `RestartSec` | 重启延迟（秒） |
| `Environment` | 环境变量 |
| `LimitNOFILE` | 文件描述符限制 |
| `MemoryMax` | 内存限制 |
| `CPUQuota` | CPU 配额（100% = 1 核） |
| **[Install]** | 安装配置 |
| `WantedBy` | 被哪个 target 依赖 |

## 守护进程安装流程

### 安装实现

```typescript
// 伪代码示例
async function installGatewayDaemon(options: {
  port: number;
  token?: string;
  runtime: "node" | "bun";
}): Promise<void> {
  const { port, token, runtime } = options;

  // 1. 检测平台
  const platform = process.platform;

  if (platform === "darwin") {
    // macOS: 安装 LaunchAgent
    await installLaunchAgent({ port, token, runtime });
  } else if (platform === "linux") {
    // Linux: 安装 systemd service
    await installSystemdService({ port, token, runtime });
  } else {
    throw new Error(`Unsupported platform: ${platform}`);
  }
}

async function installLaunchAgent(options: {
  port: number;
  token?: string;
  runtime: "node" | "bun";
}): Promise<void> {
  const { port, token, runtime } = options;

  // 1. 生成 plist 文件
  const plistPath = path.join(os.homedir(), "Library/LaunchAgents/ai.openclaw.gateway.plist");
  const plistContent = generateLaunchAgentPlist({ port, token, runtime });

  // 2. 写入文件
  await fs.writeFile(plistPath, plistContent);

  // 3. 加载服务
  await exec(`launchctl load ${plistPath}`);

  // 4. 启动服务
  await exec("launchctl start ai.openclaw.gateway");
}

async function installSystemdService(options: {
  port: number;
  token?: string;
  runtime: "node" | "bun";
}): Promise<void> {
  const { port, token, runtime } = options;

  // 1. 检查 systemd 可用性
  const systemdAvailable = await isSystemdUserServiceAvailable();
  if (!systemdAvailable) {
    throw new Error("systemd user services are unavailable");
  }

  // 2. 生成 service 文件
  const servicePath = path.join(os.homedir(), ".config/systemd/user/openclaw-gateway.service");
  const serviceContent = generateSystemdService({ port, token, runtime });

  // 3. 创建目录
  await fs.mkdir(path.dirname(servicePath), { recursive: true });

  // 4. 写入文件
  await fs.writeFile(servicePath, serviceContent);

  // 5. 重新加载配置
  await exec("systemctl --user daemon-reload");

  // 6. 启用服务
  await exec("systemctl --user enable openclaw-gateway");

  // 7. 启动服务
  await exec("systemctl --user start openclaw-gateway");
}
```

## systemd 可用性检查

### 检查逻辑

```typescript
async function isSystemdUserServiceAvailable(): Promise<boolean> {
  try {
    // 1. 检查 systemctl 命令
    const result = await exec("systemctl --user status");
    return result.exitCode === 0;
  } catch (err) {
    // 2. 检查错误信息
    const errorMessage = err instanceof Error ? err.message : String(err);

    if (errorMessage.includes("DBUS_SESSION_BUS_ADDRESS") ||
        errorMessage.includes("XDG_RUNTIME_DIR")) {
      // D-Bus 会话总线不可用
      return false;
    }

    return false;
  }
}
```

### 常见问题

#### 问题 1：DBUS_SESSION_BUS_ADDRESS 未定义

**错误信息**：

```
Failed to connect to bus: $DBUS_SESSION_BUS_ADDRESS and $XDG_RUNTIME_DIR not defined
```

**原因**：

- 使用 `su` 或 `sudo -iu` 切换用户（不创建完整会话）
- 缺少 `dbus-user-session` 包
- 未启用 lingering

**解决方案**：

```bash
# 1. 安装 dbus-user-session
sudo apt install -y dbus-user-session

# 2. 启用 lingering
sudo loginctl enable-linger $USER

# 3. 重新登录
exit
# SSH 重新连接

# 4. 验证
systemctl --user status
```

#### 问题 2：Lingering 未启用

**症状**：

用户登出后 systemd 会话被终止，Gateway 停止运行。

**解决方案**：

```bash
# 启用 lingering
sudo loginctl enable-linger $USER

# 验证
loginctl show-user $USER | grep Linger
# 应该输出：Linger=yes
```

#### 问题 3：SSH PAM 配置问题

**症状**：

SSH 登录后 systemd 用户会话不可用。

**解决方案**：

```bash
# 1. 编辑 sshd_config
sudo vim /etc/ssh/sshd_config

# 2. 确保以下配置
UsePAM yes

# 3. 验证 PAM 配置
grep pam_systemd /etc/pam.d/sshd
# 应该包含：session optional pam_systemd.so

# 4. 如果缺失，安装
sudo apt install --reinstall libpam-systemd

# 5. 重启 SSH 服务
sudo systemctl restart sshd

# 6. 重新登录
exit
# SSH 重新连接
```

**详细排查**：查看 `temp/troubleshooting/systemd_issue_1818.md`

## 守护进程生命周期

### 启动流程

```
用户运行 openclaw gateway start
    ↓
检测平台（macOS/Linux）
    ↓
macOS: launchctl start ai.openclaw.gateway
Linux: systemctl --user start openclaw-gateway
    ↓
守护进程启动 Gateway 进程
    ↓
Gateway 监听端口 18789
    ↓
健康检查（/health 端点）
    ↓
启动成功
```

### 停止流程

```
用户运行 openclaw gateway stop
    ↓
检测平台（macOS/Linux）
    ↓
macOS: launchctl stop ai.openclaw.gateway
Linux: systemctl --user stop openclaw-gateway
    ↓
守护进程发送 SIGTERM 信号
    ↓
Gateway 优雅关闭（完成当前请求）
    ↓
超时后发送 SIGKILL（强制终止）
    ↓
停止成功
```

### 重启流程

```
用户运行 openclaw gateway restart
    ↓
停止 Gateway（SIGTERM）
    ↓
等待进程退出
    ↓
启动 Gateway
    ↓
健康检查
    ↓
重启成功
```

### 崩溃重启

```
Gateway 进程崩溃（退出码非 0）
    ↓
守护进程检测到退出
    ↓
等待 RestartSec 秒（默认 5 秒）
    ↓
自动重启 Gateway
    ↓
健康检查
    ↓
恢复运行
```

## 日志管理

### macOS 日志

**日志位置**：

```
~/.openclaw/logs/gateway.log          # 标准输出
~/.openclaw/logs/gateway-error.log    # 标准错误
```

**查看日志**：

```bash
# 查看标准输出
tail -f ~/.openclaw/logs/gateway.log

# 查看错误日志
tail -f ~/.openclaw/logs/gateway-error.log

# 使用 OpenClaw CLI
openclaw logs gateway
openclaw logs gateway --follow
openclaw logs gateway --tail 100
```

### Linux 日志

**日志位置**：

systemd 日志存储在 journald 中，不是文件。

**查看日志**：

```bash
# 查看所有日志
journalctl --user -u openclaw-gateway

# 实时跟踪日志
journalctl --user -u openclaw-gateway -f

# 查看最近 100 行
journalctl --user -u openclaw-gateway -n 100

# 查看特定时间范围
journalctl --user -u openclaw-gateway --since "2026-02-22 00:00:00"

# 查看错误日志
journalctl --user -u openclaw-gateway -p err

# 使用 OpenClaw CLI
openclaw logs gateway
```

### 日志轮转

**macOS**：

手动配置日志轮转（使用 `logrotate` 或自定义脚本）。

**Linux**：

journald 自动管理日志轮转。

**配置 journald 日志大小**：

```bash
# 编辑配置
sudo vim /etc/systemd/journald.conf

# 设置日志大小限制
SystemMaxUse=1G
SystemMaxFileSize=100M

# 重启 journald
sudo systemctl restart systemd-journald
```

## 故障排查

### 问题 1：服务无法启动

**macOS**：

```bash
# 查看 launchctl 错误
launchctl list ai.openclaw.gateway

# 查看日志
tail -f ~/.openclaw/logs/gateway-error.log

# 检查 plist 文件
plutil -lint ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# 手动启动测试
/usr/local/bin/openclaw gateway run --foreground
```

**Linux**：

```bash
# 查看服务状态
systemctl --user status openclaw-gateway

# 查看详细日志
journalctl --user -u openclaw-gateway -n 50

# 检查配置文件
systemd-analyze --user verify openclaw-gateway.service

# 手动启动测试
/usr/local/bin/openclaw gateway run --foreground
```

### 问题 2：服务频繁重启

**症状**：

Gateway 启动后立即崩溃，守护进程不断重启。

**排查步骤**：

```bash
# 1. 查看崩溃日志
openclaw logs gateway | grep -i error

# 2. 检查端口占用
lsof -i :18789

# 3. 检查配置文件
openclaw doctor

# 4. 手动启动调试
openclaw gateway run --foreground --log-level debug
```

### 问题 3：服务未自动启动

**macOS**：

```bash
# 检查 RunAtLoad 配置
grep RunAtLoad ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# 重新加载服务
launchctl unload ~/Library/LaunchAgents/ai.openclaw.gateway.plist
launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

**Linux**：

```bash
# 检查服务是否启用
systemctl --user is-enabled openclaw-gateway

# 启用服务
systemctl --user enable openclaw-gateway

# 检查 lingering
loginctl show-user $USER | grep Linger
```

### 问题 4：日志文件过大

**macOS**：

```bash
# 清理日志
rm ~/.openclaw/logs/gateway.log
rm ~/.openclaw/logs/gateway-error.log

# 重启服务（重新创建日志文件）
openclaw gateway restart
```

**Linux**：

```bash
# 清理 journald 日志
journalctl --user --vacuum-time=7d
journalctl --user --vacuum-size=100M

# 查看日志大小
journalctl --user --disk-usage
```

## 总结

守护进程管理的核心要点：

1. **平台差异**：macOS 使用 LaunchAgent，Linux 使用 systemd
2. **自动启动**：开机自启和崩溃重启
3. **日志管理**：macOS 使用文件日志，Linux 使用 journald
4. **资源控制**：限制内存、CPU、文件描述符
5. **故障排查**：systemd 可用性检查、lingering 配置、PAM 配置

理解守护进程管理，可以帮助你确保 OpenClaw Gateway 稳定运行。
