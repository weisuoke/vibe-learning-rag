# 核心概念2：条件边（Conditional Edge）

## 一句话定义

**条件边是根据路由函数返回值动态决定下一个节点的边，通过 `add_conditional_edges()` 添加，实现工作流的分支决策逻辑。**

---

## API 签名

```python
def add_conditional_edges(
    self,
    source: str,
    path: Callable[..., Hashable | Sequence[Hashable]],
    path_map: dict[Hashable, str] | list[str] | None = None,
) -> Self
```

**参数说明：**
- `source`：起始节点名称，条件边从该节点出发
- `path`：路由函数，接收状态并返回目标节点名称或 Send 对象
- `path_map`：可选的映射字典，将路由函数返回值映射到节点名称

---

## 快速示例

```python
from langgraph.graph import StateGraph, START, END
from typing import Literal
from typing_extensions import TypedDict

class State(TypedDict):
    value: int
    path_taken: str

def process_input(state: State) -> dict:
    """处理输入节点"""
    return {"value": state["value"]}

def route_by_value(state: State) -> Literal["high_path", "low_path"]:
    """路由函数：根据值大小决定路径"""
    if state["value"] > 50:
        return "high_path"
    return "low_path"

def high_handler(state: State) -> dict:
    """高值处理节点"""
    return {"path_taken": "high", "value": state["value"] * 2}

def low_handler(state: State) -> dict:
    """低值处理节点"""
    return {"path_taken": "low", "value": state["value"] + 10}

# 构建图
builder = StateGraph(State)
builder.add_node("process", process_input)
builder.add_node("high_path", high_handler)
builder.add_node("low_path", low_handler)

# 添加条件边
builder.add_edge(START, "process")
builder.add_conditional_edges(
    "process",                    # 起始节点
    route_by_value,               # 路由函数
    {                             # path_map 映射
        "high_path": "high_path",
        "low_path": "low_path"
    }
)
builder.add_edge("high_path", END)
builder.add_edge("low_path", END)

graph = builder.compile()

# 测试
result = graph.invoke({"value": 60})
print(result)  # {"value": 120, "path_taken": "high"}
```

## 详细解释

### 1. 条件边的工作原理

条件边通过**路由函数**实现动态路由决策：

```
状态 → 路由函数 → 返回值 → path_map 映射 → 目标节点
```

**执行流程：**
1. 起始节点执行完毕
2. 调用路由函数，传入当前状态
3. 路由函数返回一个值（字符串或 Send 对象）
4. 如果提供了 `path_map`，将返回值映射到节点名称
5. 执行目标节点

### 2. BranchSpec 内部实现

条件边在内部使用 `BranchSpec` 类存储：

```python
class BranchSpec(NamedTuple):
    path: Runnable[Any, Hashable | list[Hashable]]  # 路由函数
    ends: dict[Hashable, str] | None                # path_map 映射
    input_schema: type[Any] | None = None           # 输入类型
```

**关键特性：**
- `path`：路由函数，可以是普通函数或 Runnable
- `ends`：映射字典，将路由函数返回值映射到节点名称
- `input_schema`：可选的输入类型，影响路由函数接收的状态类型

**存储位置：**
```python
# 条件边存储在 self.branches[source][name] 中
self.branches[source][name] = BranchSpec(path, path_map)
```

### 3. 路由函数设计

路由函数是条件边的核心，决定了工作流的分支逻辑。

**基本要求：**
- 接收状态作为参数
- 返回节点名称（字符串）或 Send 对象
- 应该是纯函数（无副作用）

**类型注解的重要性：**

使用 `Literal` 类型注解可以自动推断 `path_map`：

```python
from typing import Literal

def route_by_category(state: State) -> Literal["cat1", "cat2", "cat3"]:
    """类型注解明确了所有可能的返回值"""
    if state["category"] == "A":
        return "cat1"
    elif state["category"] == "B":
        return "cat2"
    else:
        return "cat3"

# 自动推断 path_map，无需手动指定
builder.add_conditional_edges(
    "source",
    route_by_category
    # path_map 自动推断为 {"cat1": "cat1", "cat2": "cat2", "cat3": "cat3"}
)
```

**不使用类型注解时：**

```python
def route_by_category(state: State):
    """没有类型注解，必须手动指定 path_map"""
    if state["category"] == "A":
        return "cat1"
    elif state["category"] == "B":
        return "cat2"
    else:
        return "cat3"

# 必须手动指定 path_map
builder.add_conditional_edges(
    "source",
    route_by_category,
    {
        "cat1": "handler1",
        "cat2": "handler2",
        "cat3": "handler3"
    }
)
```

### 4. path_map 映射机制

`path_map` 将路由函数返回值映射到实际的节点名称。

**三种使用方式：**

**方式1：字典映射（推荐）**
```python
builder.add_conditional_edges(
    "source",
    route_function,
    {
        "return_value_1": "node_a",
        "return_value_2": "node_b",
        "return_value_3": "node_c"
    }
)
```

**方式2：列表形式（路由函数直接返回节点名称）**
```python
builder.add_conditional_edges(
    "source",
    route_function,
    ["node_a", "node_b", "node_c"]  # 路由函数必须返回这些名称之一
)
```

**方式3：自动推断（使用 Literal 类型注解）**
```python
def route_function(state: State) -> Literal["node_a", "node_b"]:
    return "node_a" if condition else "node_b"

builder.add_conditional_edges(
    "source",
    route_function
    # path_map 自动推断
)
```

### 5. 路由到特殊节点

条件边可以路由到 `END` 节点，实现提前退出：

```python
def route_with_exit(state: State) -> Literal["continue", "exit"]:
    """根据条件决定是否继续"""
    if state["should_continue"]:
        return "continue"
    return "exit"

builder.add_conditional_edges(
    "process",
    route_with_exit,
    {
        "continue": "next_node",
        "exit": END  # 路由到 END 节点
    }
)
```

### 6. 状态传递机制

路由函数接收的状态类型取决于节点的 `input_schema`：

```python
# 情况1：没有 input_schema，接收完整状态
def route_full_state(state: State) -> str:
    # state 是完整的 State 类型
    return "next"

# 情况2：有 input_schema，接收部分状态
class InputState(TypedDict):
    value: int

def route_partial_state(state: InputState) -> str:
    # state 只包含 InputState 定义的字段
    return "next"
```

**常见陷阱：**

入口节点的条件边可能接收 `InputState` 而非完整的 `State`：

```python
# ❌ 错误：假设路由函数总是接收完整状态
def route_by_full_state(state: State) -> str:
    # 如果这是入口节点的条件边，state 可能只是 InputState
    return state["some_field"]  # 可能不存在

# ✅ 正确：明确指定输入类型
def route_by_input_state(state: InputState) -> str:
    return state["value"]
```

## 完整可运行示例

### 示例1：二分支决策

```python
"""
条件边示例1：二分支决策
演示：根据用户输入的年龄判断是否成年
"""

from langgraph.graph import StateGraph, START, END
from typing import Literal
from typing_extensions import TypedDict

# ===== 1. 定义状态 =====
class AgeCheckState(TypedDict):
    age: int
    message: str
    is_adult: bool

# ===== 2. 定义节点 =====
def check_age(state: AgeCheckState) -> dict:
    """检查年龄节点"""
    print(f"检查年龄: {state['age']}")
    return {"age": state["age"]}

def adult_handler(state: AgeCheckState) -> dict:
    """成年人处理节点"""
    return {
        "message": f"您已成年（{state['age']}岁），可以访问所有内容",
        "is_adult": True
    }

def minor_handler(state: AgeCheckState) -> dict:
    """未成年人处理节点"""
    return {
        "message": f"您未成年（{state['age']}岁），访问受限",
        "is_adult": False
    }

# ===== 3. 定义路由函数 =====
def route_by_age(state: AgeCheckState) -> Literal["adult", "minor"]:
    """根据年龄路由"""
    if state["age"] >= 18:
        return "adult"
    return "minor"

# ===== 4. 构建图 =====
builder = StateGraph(AgeCheckState)
builder.add_node("check", check_age)
builder.add_node("adult", adult_handler)
builder.add_node("minor", minor_handler)

builder.add_edge(START, "check")
builder.add_conditional_edges(
    "check",
    route_by_age,
    {
        "adult": "adult",
        "minor": "minor"
    }
)
builder.add_edge("adult", END)
builder.add_edge("minor", END)

graph = builder.compile()

# ===== 5. 测试 =====
print("=== 测试1：成年人 ===")
result1 = graph.invoke({"age": 25})
print(f"结果: {result1['message']}")
print(f"是否成年: {result1['is_adult']}")

print("\n=== 测试2：未成年人 ===")
result2 = graph.invoke({"age": 15})
print(f"结果: {result2['message']}")
print(f"是否成年: {result2['is_adult']}")
```

**运行输出：**
```
=== 测试1：成年人 ===
检查年龄: 25
结果: 您已成年（25岁），可以访问所有内容
是否成年: True

=== 测试2：未成年人 ===
检查年龄: 15
结果: 您未成年（15岁），访问受限
是否成年: False
```

### 示例2：多分支决策

```python
"""
条件边示例2：多分支决策
演示：根据文档类型选择不同的处理器
"""

from langgraph.graph import StateGraph, START, END
from typing import Literal
from typing_extensions import TypedDict

# ===== 1. 定义状态 =====
class DocumentState(TypedDict):
    doc_type: str
    content: str
    processed_content: str

# ===== 2. 定义节点 =====
def classify_document(state: DocumentState) -> dict:
    """文档分类节点"""
    print(f"分类文档: {state['doc_type']}")
    return {"doc_type": state["doc_type"], "content": state["content"]}

def process_pdf(state: DocumentState) -> dict:
    """PDF 处理节点"""
    return {"processed_content": f"PDF处理: {state['content']}"}

def process_word(state: DocumentState) -> dict:
    """Word 处理节点"""
    return {"processed_content": f"Word处理: {state['content']}"}

def process_text(state: DocumentState) -> dict:
    """文本处理节点"""
    return {"processed_content": f"文本处理: {state['content']}"}

def process_unknown(state: DocumentState) -> dict:
    """未知类型处理节点"""
    return {"processed_content": f"未知类型: {state['content']}"}

# ===== 3. 定义路由函数 =====
def route_by_doc_type(state: DocumentState) -> Literal["pdf", "word", "text", "unknown"]:
    """根据文档类型路由"""
    doc_type = state["doc_type"].lower()
    if doc_type == "pdf":
        return "pdf"
    elif doc_type in ["doc", "docx"]:
        return "word"
    elif doc_type == "txt":
        return "text"
    else:
        return "unknown"

# ===== 4. 构建图 =====
builder = StateGraph(DocumentState)
builder.add_node("classify", classify_document)
builder.add_node("pdf", process_pdf)
builder.add_node("word", process_word)
builder.add_node("text", process_text)
builder.add_node("unknown", process_unknown)

builder.add_edge(START, "classify")
builder.add_conditional_edges(
    "classify",
    route_by_doc_type,
    {
        "pdf": "pdf",
        "word": "word",
        "text": "text",
        "unknown": "unknown"
    }
)
builder.add_edge("pdf", END)
builder.add_edge("word", END)
builder.add_edge("text", END)
builder.add_edge("unknown", END)

graph = builder.compile()

# ===== 5. 测试 =====
test_cases = [
    {"doc_type": "pdf", "content": "这是一个PDF文档"},
    {"doc_type": "docx", "content": "这是一个Word文档"},
    {"doc_type": "txt", "content": "这是一个文本文件"},
    {"doc_type": "xlsx", "content": "这是一个Excel文件"}
]

for i, test_case in enumerate(test_cases, 1):
    print(f"\n=== 测试{i}: {test_case['doc_type']} ===")
    result = graph.invoke(test_case)
    print(f"处理结果: {result['processed_content']}")
```

**运行输出：**
```
=== 测试1: pdf ===
分类文档: pdf
处理结果: PDF处理: 这是一个PDF文档

=== 测试2: docx ===
分类文档: docx
处理结果: Word处理: 这是一个Word文档

=== 测试3: txt ===
分类文档: txt
处理结果: 文本处理: 这是一个文本文件

=== 测试4: xlsx ===
分类文档: xlsx
处理结果: 未知类型: 这是一个Excel文件
```

### 示例3：循环与退出

```python
"""
条件边示例3：循环与退出
演示：重试机制，最多重试3次
"""

from langgraph.graph import StateGraph, START, END
from typing import Literal
from typing_extensions import TypedDict

# ===== 1. 定义状态 =====
class RetryState(TypedDict):
    attempt: int
    max_attempts: int
    success: bool
    message: str

# ===== 2. 定义节点 =====
def attempt_task(state: RetryState) -> dict:
    """尝试执行任务"""
    attempt = state.get("attempt", 0) + 1
    print(f"第 {attempt} 次尝试...")

    # 模拟：第3次尝试成功
    success = attempt >= 3

    return {
        "attempt": attempt,
        "success": success,
        "message": f"第{attempt}次尝试{'成功' if success else '失败'}"
    }

# ===== 3. 定义路由函数 =====
def route_by_result(state: RetryState) -> Literal["retry", "success", "failed"]:
    """根据结果决定是否重试"""
    if state["success"]:
        return "success"
    elif state["attempt"] < state["max_attempts"]:
        return "retry"
    else:
        return "failed"

# ===== 4. 构建图 =====
builder = StateGraph(RetryState)
builder.add_node("attempt", attempt_task)

builder.add_edge(START, "attempt")
builder.add_conditional_edges(
    "attempt",
    route_by_result,
    {
        "retry": "attempt",    # 循环：重新执行 attempt 节点
        "success": END,        # 成功：退出
        "failed": END          # 失败：退出
    }
)

graph = builder.compile()

# ===== 5. 测试 =====
print("=== 测试：重试机制 ===")
result = graph.invoke({"max_attempts": 5})
print(f"\n最终结果:")
print(f"  尝试次数: {result['attempt']}")
print(f"  是否成功: {result['success']}")
print(f"  消息: {result['message']}")
```

**运行输出：**
```
=== 测试：重试机制 ===
第 1 次尝试...
第 2 次尝试...
第 3 次尝试...

最终结果:
  尝试次数: 3
  是否成功: True
  消息: 第3次尝试成功
```

## 在 LangGraph 工作流中的应用

### 1. RAG 工作流中的条件边

条件边在 RAG 系统中用于实现智能决策：

```python
"""
RAG 工作流示例：根据文档质量决定是否需要网络搜索
"""

from langgraph.graph import StateGraph, START, END
from typing import Literal
from typing_extensions import TypedDict

class RAGState(TypedDict):
    query: str
    documents: list[str]
    doc_quality: str
    answer: str

def retrieve_documents(state: RAGState) -> dict:
    """检索文档"""
    # 模拟检索
    docs = ["文档1", "文档2"]
    return {"documents": docs}

def grade_documents(state: RAGState) -> dict:
    """评估文档质量"""
    # 模拟评估
    quality = "good" if len(state["documents"]) > 0 else "poor"
    return {"doc_quality": quality}

def web_search(state: RAGState) -> dict:
    """网络搜索"""
    return {"documents": state["documents"] + ["网络搜索结果"]}

def generate_answer(state: RAGState) -> dict:
    """生成答案"""
    return {"answer": f"基于 {len(state['documents'])} 个文档生成答案"}

def route_by_quality(state: RAGState) -> Literal["search", "generate"]:
    """根据文档质量决定是否需要网络搜索"""
    if state["doc_quality"] == "poor":
        return "search"
    return "generate"

# 构建 RAG 工作流
builder = StateGraph(RAGState)
builder.add_node("retrieve", retrieve_documents)
builder.add_node("grade", grade_documents)
builder.add_node("search", web_search)
builder.add_node("generate", generate_answer)

builder.add_edge(START, "retrieve")
builder.add_edge("retrieve", "grade")
builder.add_conditional_edges(
    "grade",
    route_by_quality,
    {
        "search": "search",
        "generate": "generate"
    }
)
builder.add_edge("search", "generate")
builder.add_edge("generate", END)

graph = builder.compile()
```

### 2. 多条件边组合

复杂工作流可以使用多个条件边：

```python
# Self-RAG 工作流示例
workflow.add_edge(START, "retrieve")
workflow.add_edge("retrieve", "grade_documents")

# 第一个条件边：决定是否需要转换查询
workflow.add_conditional_edges(
    "grade_documents",
    decide_to_generate,
    {
        "transform_query": "transform_query",
        "generate": "generate",
    },
)

workflow.add_edge("transform_query", "retrieve")  # 形成循环

# 第二个条件边：评估生成结果
workflow.add_conditional_edges(
    "generate",
    grade_generation,
    {
        "not supported": "generate",      # 重新生成
        "useful": END,                    # 结束
        "not useful": "transform_query",  # 转换查询
    },
)
```

## 常见问题与最佳实践

### 1. 避免与普通边混用

**问题：** 条件边和普通边同时指向同一节点会导致节点重复执行

```python
# ❌ 错误：混用导致重复执行
builder.add_conditional_edges("A", route_func, {"path1": "B"})
builder.add_edge("A", "B")  # B 节点会被执行两次

# ✅ 正确：只使用一种边类型
builder.add_conditional_edges("A", route_func, {"path1": "B", "path2": "C"})
```

### 2. 使用类型注解提升可维护性

```python
# ✅ 推荐：使用 Literal 类型注解
def route_function(state: State) -> Literal["path_a", "path_b"]:
    return "path_a" if condition else "path_b"

# 自动推断 path_map，减少错误
builder.add_conditional_edges("source", route_function)
```

### 3. 路由函数应该是纯函数

```python
# ❌ 错误：路由函数有副作用
def route_with_side_effect(state: State) -> str:
    state["counter"] += 1  # 修改状态
    return "next"

# ✅ 正确：路由函数只读取状态
def route_pure(state: State) -> str:
    if state["counter"] > 10:
        return "next"
    return "retry"
```

### 4. 明确指定输入类型

```python
# ✅ 推荐：明确指定路由函数的输入类型
def route_by_input(state: InputState) -> Literal["path_a", "path_b"]:
    # 明确知道 state 的类型
    return "path_a" if state["value"] > 0 else "path_b"
```
