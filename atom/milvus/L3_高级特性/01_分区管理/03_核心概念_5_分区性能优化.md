# 核心概念 5：分区性能优化

> 通过分区提升检索效率的策略和技巧

---

## 概述

**分区性能优化**是指通过合理使用分区功能，提升 Milvus 检索性能的策略。核心思想是减少检索范围，避免无效扫描。

**核心价值**：
- 性能提升：避免 99% 无效扫描
- 延迟降低：从秒级降至毫秒级
- 资源优化：节省内存和计算资源

---

## 1. Partition Key 隔离特性

### 1.1 功能介绍

**来源**：Context7 官方文档

**定义**：通过启用 Partition Key 隔离，可以进一步提升搜索性能。当前仅支持 HNSW 索引类型。启用后，搜索必须在过滤器中包含单个特定的 Partition Key 值。

**核心机制**：
- 启用后，Milvus 会为每个 Partition Key 值创建独立的 HNSW 图
- 搜索时只遍历特定 Partition Key 对应的 HNSW 图
- 避免跨分区的图遍历，提升性能

### 1.2 配置方法

**Python 示例**：
```python
from pymilvus import Collection, FieldSchema, CollectionSchema, DataType

# 定义 Schema
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
        FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
        FieldSchema(name="tenant_id", dtype=DataType.VARCHAR, max_length=64,
                   is_partition_key=True)
    ]
)

# 创建 Collection，启用 Partition Key 隔离
collection = Collection(
    name="my_collection",
    schema=schema,
    properties={"partitionkey.isolation": True}  # 启用 Partition Key 隔离
)
```

**Go 示例**：
```go
err = client.CreateCollection(ctx,
    milvusclient.NewCreateCollectionOption("my_collection", schema).
        WithProperty("partitionkey.isolation", true))
if err != nil {
    fmt.Println(err.Error())
}
```

**Java 示例**：
```java
Map<String, String> properties = new HashMap<>();
properties.put("partitionkey.isolation", "true");

CreateCollectionReq createCollectionReq = CreateCollectionReq.builder()
        .collectionName("my_collection")
        .collectionSchema(schema)
        .properties(properties)
        .build();
client.createCollection(createCollectionReq);
```

### 1.3 使用限制

**索引类型限制**：
- 仅支持 HNSW 索引类型
- 不支持 IVF_FLAT、IVF_SQ8 等其他索引类型

**搜索限制**：
- 搜索时必须在过滤器中包含单个特定的 Partition Key 值
- 不支持跨多个 Partition Key 值的搜索

**示例**：
```python
# 正确：包含单个 Partition Key 值
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"ef": 100}},
    limit=10,
    expr='tenant_id == "tenant_a"'  # 单个值
)

# 错误：包含多个 Partition Key 值
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"ef": 100}},
    limit=10,
    expr='tenant_id in ["tenant_a", "tenant_b"]'  # 多个值（不支持）
)
```

## 2. 避免 99% 无效扫描

### 2.1 性能数据

**来源**：Twitter/X 性能优化数据

**核心数据**：
- **99% 无效扫描避免**：使用 Partition Key 可以避免扫描 99% 的无关数据
- **延迟优化**：多租户搜索延迟从秒级降至毫秒级
- **数据量减少**：搜索时仅处理数千向量而非百万

### 2.2 性能对比

**场景**：1000 个租户，每个租户 10 万条数据，总数据量 1 亿条

**不使用 Partition Key**：
```python
# 扫描 1 亿条数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    expr='tenant_id == "tenant_a"'  # 需要扫描 1 亿条数据
)
# 延迟：> 1 秒
```

**使用 Partition Key**：
```python
# 只扫描 10 万条数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    expr='tenant_id == "tenant_a"'  # 只扫描租户 A 的分区
)
# 延迟：< 10 毫秒
# 性能提升：100 倍
```

### 2.3 性能提升原理

**核心机制**：
1. **数据路由**：Milvus 根据 Partition Key 值自动将数据路由到 16 个物理分区
2. **分区过滤**：搜索时只扫描相关分区，避免全表扫描
3. **索引优化**：每个分区独立建立索引，提升检索效率

**示例**：
```python
# 假设有 1000 个租户，16 个物理分区
# 每个物理分区大约包含 1000 / 16 ≈ 62 个租户的数据

# 搜索租户 A 的数据
# Milvus 只扫描租户 A 所在的分区（1/16 的数据）
# 数据量减少：1 亿 → 625 万（减少 93.75%）
```

## 3. 分区数量建议

### 3.1 推荐配置

**来源**：GitHub 最佳实践

**Partition Key 物理分区数**：
- **默认**：16 个物理分区
- **推荐**：100-200 个物理分区（生产环境）
- **最大**：根据系统资源调整

**手动分区数**：
- **推荐分区大小**：20-100K 行
- **推荐分区数**：100-200 个
- **最大分区数**：1,024 个

### 3.2 配置方法

**配置 Partition Key 物理分区数**：
```python
from pymilvus import Collection

collection = Collection(
    name="my_collection",
    schema=schema,
    properties={"partition.num": 100}  # 配置 100 个物理分区
)
```

### 3.3 选择建议

**根据租户数量选择**：
- **小规模**（< 1 万租户）：16 个物理分区
- **中等规模**（1-10 万租户）：64 个物理分区
- **大规模**（> 10 万租户）：100-200 个物理分区

**根据数据量选择**：
- **小数据量**（< 100 万条）：16 个物理分区
- **中等数据量**（100 万-1000 万条）：64 个物理分区
- **大数据量**（> 1000 万条）：100-200 个物理分区

## 4. 热冷数据管理

### 4.1 概念

**热数据**：频繁访问的数据，需要保持在内存中
**冷数据**：不常访问的数据，可以释放内存

### 4.2 手动分区的热冷数据管理

**场景**：按时间分区，只加载最近的数据

**示例**：
```python
from pymilvus import Collection
from datetime import datetime, timedelta

collection = Collection("time_series_data")

# 创建按月份的分区
current_month = datetime.now().strftime("%Y_%m")
last_month = (datetime.now().replace(day=1) - timedelta(days=1)).strftime("%Y_%m")
old_month = "2025_12"

collection.create_partition(current_month)
collection.create_partition(last_month)
collection.create_partition(old_month)

# 只加载最近两个月的数据（热数据）
collection.load(partition_names=[current_month, last_month])

# 搜索热数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=[current_month, last_month]
)

# 如果需要搜索历史数据（冷数据），临时加载
collection.load(partition_names=[old_month])
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=[old_month]
)

# 搜索完成后释放冷数据
collection.release(partition_names=[old_month])
```

### 4.3 内存优化

**优化策略**：
1. **只加载需要的分区**：节省内存
2. **定期释放冷数据**：释放不常用的分区
3. **按需加载**：临时加载冷数据，使用后释放

**示例**：
```python
# 场景：100 个分区，每个分区 100 万条数据，总数据量 1 亿条

# 策略 1：加载所有分区
collection.load()  # 内存占用：1 亿条数据

# 策略 2：只加载最近 10 个分区
recent_partitions = [f"partition_{i}" for i in range(90, 100)]
collection.load(partition_names=recent_partitions)  # 内存占用：1000 万条数据
# 内存节省：90%
```

## 5. 分区搜索优化

### 5.1 指定分区搜索

**核心策略**：在搜索时指定分区，减少检索范围

**示例**：
```python
from pymilvus import Collection

collection = Collection("my_collection")

# 策略 1：搜索所有分区（慢）
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10
    # 不指定 partition_names，搜索所有分区
)

# 策略 2：搜索特定分区（快）
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=["partition_a"]  # 只搜索 partition_a
)
# 性能提升：取决于分区数量
```

### 5.2 分区 + 标量过滤

**核心策略**：结合分区和标量过滤，进一步减少检索范围

**示例**：
```python
# 策略 1：只使用标量过滤（慢）
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    expr='category == "electronics"'  # 标量过滤
)

# 策略 2：分区 + 标量过滤（快）
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=["partition_a"],  # 分区过滤
    expr='category == "electronics"'  # 标量过滤
)
# 性能提升：先通过分区减少数据量，再通过标量过滤
```

## 6. 性能监控

### 6.1 监控指标

**关键指标**：
- **QPS**：每秒查询数
- **延迟**：查询响应时间
- **内存使用**：加载的分区占用的内存
- **分区负载**：每个分区的数据量和访问频率

### 6.2 监控方法

**使用 Milvus 内置监控**：
```python
from pymilvus import utility

# 获取 Collection 统计信息
stats = utility.get_collection_stats("my_collection")
print(stats)

# 获取分区统计信息
for partition in collection.partitions:
    stats = utility.get_partition_stats("my_collection", partition.name)
    print(f"Partition: {partition.name}, Stats: {stats}")
```

### 6.3 性能调优

**调优策略**：
1. **增加物理分区数**：如果分区负载不均衡，增加物理分区数
2. **调整分区大小**：保持每个分区 20-100K 行
3. **优化索引参数**：根据实际场景调整索引参数
4. **监控和告警**：设置性能监控和告警

## 7. 实际应用案例

### 7.1 多租户 SaaS 应用

**场景**：1000 个租户，每个租户 10 万条数据

**优化策略**：
```python
# 使用 Partition Key 自动分区
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
        FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
        FieldSchema(name="tenant_id", dtype=DataType.VARCHAR, max_length=64,
                   is_partition_key=True)
    ]
)

# 启用 Partition Key 隔离
collection = Collection(
    name="saas_app",
    schema=schema,
    properties={
        "partitionkey.isolation": True,  # 启用 Partition Key 隔离
        "partition.num": 100  # 配置 100 个物理分区
    }
)

# 创建 HNSW 索引
index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="vector", index_params=index_params)

# 搜索租户 A 的数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"ef": 100}},
    limit=10,
    expr='tenant_id == "tenant_a"'
)
# 性能：延迟 < 10 毫秒
```

### 7.2 时间序列数据

**场景**：按月份分区，只查询最近的数据

**优化策略**：
```python
from datetime import datetime, timedelta

collection = Collection("time_series_data")

# 创建按月份的分区
for i in range(12):
    month = (datetime.now() - timedelta(days=30*i)).strftime("%Y_%m")
    collection.create_partition(month)

# 只加载最近 3 个月的数据
recent_months = [
    (datetime.now() - timedelta(days=30*i)).strftime("%Y_%m")
    for i in range(3)
]
collection.load(partition_names=recent_months)

# 搜索最近 3 个月的数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=recent_months
)
# 性能：内存节省 75%，延迟降低 50%
```

1. **Partition Key 隔离特性**：启用后可进一步提升性能（仅支持 HNSW 索引）
2. **避免 99% 无效扫描**：使用 Partition Key 可以避免扫描 99% 的无关数据
3. **分区数量建议**：推荐 100-200 个物理分区（Partition Key）
4. **热冷数据管理**：只加载需要的分区，节省内存
5. **分区搜索优化**：指定分区搜索，结合标量过滤
6. **性能监控**：监控 QPS、延迟、内存使用、分区负载
