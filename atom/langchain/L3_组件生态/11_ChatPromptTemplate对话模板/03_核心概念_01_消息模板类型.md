# 核心概念 01: 消息模板类型

## 概述

ChatPromptTemplate 的核心是**消息模板类型系统**,它定义了对话中不同角色的消息如何被构建和管理。

**核心问题**: 为什么需要不同的消息模板类型?

**答案**: 因为对话系统中的每条消息都有明确的**角色**和**用途**,不同角色需要不同的处理方式。

---

## 四大消息模板类型

### 类型 1: SystemMessagePromptTemplate - 系统消息模板

#### 定义

**SystemMessagePromptTemplate** 用于构建系统消息,定义 AI 的角色、行为规则和任务说明。

**源码位置**: `chat.py:681-688`

```python
class SystemMessagePromptTemplate(_StringImageMessagePromptTemplate):
    """System message prompt template."""

    def format(self, **kwargs: Any) -> BaseMessage:
        return SystemMessage(content=self.prompt.format(**kwargs))
```

#### 核心特性

1. **角色定义**: 设置 AI 的身份和行为
2. **规则说明**: 定义 AI 应该遵循的规则
3. **上下文设置**: 提供任务背景信息
4. **优先级最高**: 通常放在消息列表的最前面

#### 使用场景

**场景 1: 基础角色定义**

```python
from langchain_core.prompts import SystemMessagePromptTemplate

# 创建系统消息模板
system_template = SystemMessagePromptTemplate.from_template(
    "你是一个{role}。请{instruction}。"
)

# 格式化
message = system_template.format(
    role="专业的 Python 开发者",
    instruction="用简洁的语言回答问题"
)

print(message)
# SystemMessage(content="你是一个专业的 Python 开发者。请用简洁的语言回答问题。")
```

**场景 2: RAG 系统指令**

```python
# RAG 系统消息
rag_system_template = SystemMessagePromptTemplate.from_template("""
你是一个基于知识库的问答助手。

规则:
1. 仅基于提供的上下文回答问题
2. 如果上下文中没有答案,明确说明"根据提供的信息无法回答"
3. 引用具体的上下文片段
4. 保持回答简洁准确

上下文:
{context}
""")

# 使用
message = rag_system_template.format(
    context="Python 是一种高级编程语言,由 Guido van Rossum 于 1991 年创建。"
)
```

**场景 3: 多模态系统消息**

```python
# 支持图片的系统消息
multimodal_system = SystemMessagePromptTemplate.from_template([
    {"type": "text", "text": "你是一个图像分析助手。"},
    {"type": "image_url", "image_url": "{instruction_image}"}
])
```

#### 在 RAG 中的应用

```python
# RAG 对话系统的系统消息
rag_system = SystemMessagePromptTemplate.from_template("""
你是一个智能文档问答助手。

能力:
- 理解用户问题的意图
- 从提供的文档中提取相关信息
- 综合多个文档片段给出完整答案

限制:
- 不要编造信息
- 不要超出文档范围回答
- 不确定时要说明

当前文档库: {doc_source}
""")
```

### 类型 2: HumanMessagePromptTemplate - 用户消息模板

#### 定义

**HumanMessagePromptTemplate** 用于构建用户消息,表示用户的输入、问题或请求。

**源码位置**: `chat.py:663-670`

```python
class HumanMessagePromptTemplate(_StringImageMessagePromptTemplate):
    """Human message prompt template."""

    def format(self, **kwargs: Any) -> BaseMessage:
        return HumanMessage(content=self.prompt.format(**kwargs))
```

#### 核心特性

1. **用户输入**: 表示用户的问题或请求
2. **变量替换**: 支持动态内容注入
3. **多模态支持**: 可以包含文本和图片
4. **灵活组合**: 可以与其他消息类型组合

#### 使用场景

**场景 1: 简单问题**

```python
from langchain_core.prompts import HumanMessagePromptTemplate

# 创建用户消息模板
human_template = HumanMessagePromptTemplate.from_template(
    "{question}"
)

# 格式化
message = human_template.format(question="什么是 Python?")

print(message)
# HumanMessage(content="什么是 Python?")
```

**场景 2: 带上下文的问题**

```python
# RAG 查询模板
rag_query_template = HumanMessagePromptTemplate.from_template("""
问题: {question}

请基于上面提供的上下文回答这个问题。
""")

# 使用
message = rag_query_template.format(
    question="Python 是什么时候创建的?"
)
```

**场景 3: 多模态查询**

```python
# 图片 + 文本查询
multimodal_query = HumanMessagePromptTemplate.from_template([
    {"type": "text", "text": "分析这张图片:\n{description}"},
    {"type": "image_url", "image_url": "{image_path}"}
])

# 使用
message = multimodal_query.format(
    description="这是一个架构图",
    image_path="https://example.com/architecture.png"
)
```

**场景 4: 结构化输入**

```python
# 结构化查询模板
structured_query = HumanMessagePromptTemplate.from_template("""
查询类型: {query_type}
查询内容: {query_content}
过滤条件: {filters}
""")

# 使用
message = structured_query.format(
    query_type="语义搜索",
    query_content="如何使用 LangChain?",
    filters="文档类型=教程"
)
```

#### 在 RAG 中的应用

```python

{question}

(如果需要参考之前的对话,请结合上下文理解我的问题)
""")

# 使用
message = conversational_rag_query.format(
    question="它的主要特点是什么?"
)
```

### 类型 3: AIMessagePromptTemplate - AI 消息模板

#### 定义

**AIMessagePromptTemplate** 用于构建 AI 消息,通常用于 Few-shot 学习示例或对话历史。

**源码位置**: `chat.py:672-679`

```python
class AIMessagePromptTemplate(_StringImageMessagePromptTemplate):
    """AI message prompt template."""

    def format(self, **kwargs: Any) -> BaseMessage:
        return AIMessage(content=self.prompt.format(**kwargs))
```

#### 核心特性

1. **示例回答**: 提供 AI 应该如何回答的示例
2. **对话历史**: 表示之前的 AI 回复
3. **格式引导**: 引导 LLM 的回答格式
4. **Few-shot 学习**: 通过示例教 LLM 如何回答

#### 使用场景

**场景 1: Few-shot 学习**

```python
from langchain_core.prompts import AIMessagePromptTemplate

# 创建 AI 消息模板
ai_template = AIMessagePromptTemplate.from_template(
    "{example_response}"
)

# 格式化
message = ai_template.format(
    example_response="Python 是一种高级编程语言。"
)

print(message)
# AIMessage(content="Python 是一种高级编程语言。")
```

**场景 2: 翻译示例**

### 类型 4: ChatMessagePromptTemplate - 自定义角色消息模板

#### 定义

**ChatMessagePromptTemplate** 用于构建自定义角色的消息,当需要使用 system/human/ai 之外的角色时使用。

**源码位置**: `chat.py:353-386`

```python
class ChatMessagePromptTemplate(BaseStringMessagePromptTemplate):
    """Chat message prompt template."""

    role: str  # 自定义角色名称

    def format(self, **kwargs: Any) -> BaseMessage:
        return ChatMessage(
            content=self.prompt.format(**kwargs),
            role=self.role
        )
```

#### 核心特性

1. **自定义角色**: 可以定义任意角色名称
2. **灵活性**: 适用于特殊场景
3. **兼容性**: 某些 LLM 支持自定义角色
4. **扩展性**: 可以模拟多角色对话

#### 使用场景

**场景 1: 工具角色**

```python
from langchain_core.prompts import ChatMessagePromptTemplate

# 创建工具角色消息
tool_template = ChatMessagePromptTemplate.from_template(
    template="{tool_output}",
    role="tool"
)

# 格式化
message = tool_template.format(
    tool_output="搜索结果: Python 是一种编程语言"
)

print(message)
# ChatMessage(content="搜索结果: Python 是一种编程语言", role="tool")
```

**场景 2: 多角色对话**

```python
# 模拟多角色讨论
discussion_template = ChatPromptTemplate.from_messages([
    ("system", "这是一个技术讨论。"),
    ChatMessagePromptTemplate.from_template(
        template="我认为 {opinion_1}",
        role="developer"
    ),
    ChatMessagePromptTemplate.from_template(
        template="我觉得 {opinion_2}",
        role="architect"
    ),
    ("human", "{question}")
])
```

**场景 3: 函数调用结果**

```python
# 函数调用结果作为消息
function_result_template = ChatPromptTemplate.from_messages([
    ("human", "查询天气"),
    ChatMessagePromptTemplate.from_template(
        template="函数返回: {function_result}",
        role="function"
    ),
    ("ai", "根据函数返回的结果...")
])
```

#### 在 RAG 中的应用

```python
# RAG 检索器角色
rag_retriever_template = ChatPromptTemplate.from_messages([
    ("system", "你是一个 RAG 系统。"),
    ("human", "{question}"),

        role="retriever"
```

## 消息格式转换

### 格式 1: 元组格式 (Tuple Format)

**最简洁的格式**,适合快速构建简单消息。

```python
from langchain_core.prompts import ChatPromptTemplate

# 元组格式
template = ChatPromptTemplate.from_messages([
    ("system", "你是一个助手。"),
    ("human", "{input}")
])

# 自动转换为对应的消息模板类型
# ("system", "...") → SystemMessagePromptTemplate
# ("human", "...") → HumanMessagePromptTemplate
# ("ai", "...") → AIMessagePromptTemplate
```

**支持的角色名**:
- `"system"` → SystemMessage
- `"human"` / `"user"` → HumanMessage
- `"ai"` / `"assistant"` → AIMessage
- 其他字符串 → ChatMessage(role=字符串)

**优点**:
- 简洁易读
- 快速开发
- 适合简单场景

**缺点**:
- 功能有限
- 无法设置额外参数
- 不支持复杂配置

### 格式 2: 消息对象格式 (Message Object Format)

**直接使用消息对象**,功能最完整。

```python
from langchain_core.messages import SystemMessage, HumanMessage, AIMessage

# 消息对象格式
template = ChatPromptTemplate.from_messages([
    SystemMessage(content="你是一个助手。"),
    HumanMessage(content="{input}")
])
```

**优点**:
- 功能完整
- 可以设置额外参数(如 additional_kwargs)
- 灵活性最高

**缺点**:
- 代码较长
- 不支持变量替换(内容是固定的)

### 格式 3: 模板对象格式 (Template Object Format)

**使用模板类**,功能和灵活性的平衡。

```python
from langchain_core.prompts import (
    SystemMessagePromptTemplate,
    HumanMessagePromptTemplate,
    AIMessagePromptTemplate
)

# 模板对象格式
template = ChatPromptTemplate.from_messages([
    SystemMessagePromptTemplate.from_template("你是一个{role}。"),
    HumanMessagePromptTemplate.from_template("{input}")
])
```

**优点**:
- 支持变量替换
- 可以设置模板格式(f-string/mustache/jinja2)
- 可以设置额外参数

**缺点**:
- 代码最长
- 需要导入多个类

### 格式对比表

| 特性 | 元组格式 | 消息对象格式 | 模板对象格式 |
|------|----------|--------------|--------------|
| 简洁性 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |
| 变量支持 | ✅ | ❌ | ✅ |
| 额外参数 | ❌ | ✅ | ✅ |
| 模板格式 | ❌ | ❌ | ✅ |
| 适用场景 | 简单对话 | 固定消息 | 复杂模板 |

### 混合使用

**最佳实践**: 根据需求混合使用不同格式

```python
from langchain_core.prompts import (
    ChatPromptTemplate,
    SystemMessagePromptTemplate,
    MessagesPlaceholder
)
from langchain_core.messages import AIMessage

# 混合格式
template = ChatPromptTemplate.from_messages([
    # 元组格式 - 简单系统消息
    ("system", "你是一个 RAG 助手。"),

    # 模板对象格式 - 复杂系统消息
    SystemMessagePromptTemplate.from_template(
        "上下文:\n{context}",
        template_format="f-string"
    ),

    # 占位符 - 对话历史
    MessagesPlaceholder("history"),

    # 消息对象 - 固定示例
    AIMessage(content="我会基于上下文回答问题。"),

    # 元组格式 - 用户问题
    ("human", "{question}")
])
```

## 在 RAG 中的实际应用

### 应用 1: 基础 RAG 模板

```python
from langchain_core.prompts import ChatPromptTemplate
```

## 核心洞察

### 洞察 1: 角色即语义

消息的角色不仅是标签,更是语义信息。LLM 会根据角色理解消息的用途。

### 洞察 2: 格式选择原则

- 简单场景 → 元组格式
- 固定消息 → 消息对象格式
- 复杂模板 → 模板对象格式
- 实际项目 → 混合使用

### 洞察 3: RAG 的消息层次

RAG 系统的消息通常有三层:
1. **角色层**: System 消息定义 AI 角色
2. **知识层**: System 消息注入上下文
3. **交互层**: Human/AI 消息进行对话

### 洞察 4: Few-shot 的威力

通过 AI 消息提供示例,可以显著提升 LLM 的回答质量,特别是在 RAG 场景中。

## 最佳实践

### 实践 1: System 消息分层

```python
# ✅ 好的做法 - 分层清晰
template = ChatPromptTemplate.from_messages([
    ("system", "角色: 你是一个 RAG 助手。"),
    ("system", "规则: 仅基于上下文回答。"),
    ("system", "上下文:\n{context}"),
    ("human", "{question}")

# ❌ 不好的做法 - 混在一起
template = ChatPromptTemplate.from_messages([

    ("human", "{question}")

```

### 实践 2: 使用 Few-shot 引导格式

```python
# ✅ 好的做法 - 提供示例
template = ChatPromptTemplate.from_messages([
    ("system", "回答时引用来源。"),
    ("human", "示例问题"),
    ("ai", "示例回答 [来源: ...]"),
    ("system", "上下文: {context}"),
    ("human", "{question}")
])
```

### 实践 3: 选择合适的格式

```python
# ✅ 简单场景用元组
simple = ChatPromptTemplate.from_messages([
    ("system", "你是助手。"),
    ("human", "{input}")

# ✅ 复杂场景用模板对象
complex = ChatPromptTemplate.from_messages([

        "你是{role}。规则:\n{rules}",
        template_format="f-string"
    ),
    HumanMessagePromptTemplate.from_template("{input}")
])
```

## 总结

消息模板类型是 ChatPromptTemplate 的基础:

1. **SystemMessagePromptTemplate**: 定义角色和规则
2. **HumanMessagePromptTemplate**: 表示用户输入
3. **AIMessagePromptTemplate**: 提供示例和历史
4. **ChatMessagePromptTemplate**: 自定义角色

三种格式各有优势:
- **元组格式**: 简洁快速
- **消息对象格式**: 功能完整
- **模板对象格式**: 灵活强大

在 RAG 开发中,正确使用消息模板类型可以:
- 清晰地组织对话结构
- 有效地注入上下文
- 引导 LLM 的回答格式
- 支持复杂的对话场景

理解消息模板类型,就理解了 ChatPromptTemplate 的核心设计。
