> **知识点**: 常用库速查（serde/anyhow/clap）
> **层级**: Phase1_Rust速成基础
> **维度**: 实战代码（场景2 — 用 clap + anyhow + tracing 构建完整 CLI 工具骨架）

---

## 场景说明

**目标**：用 clap + anyhow + tracing 三库联合，构建一个模仿 ZeroClaw 结构的迷你 CLI 工具 `miniclaw`。

**涵盖技能**：
- `#[derive(Parser)]` 定义顶层 CLI 入口
- `#[derive(Subcommand)]` 定义子命令 + 嵌套子命令
- `#[derive(ValueEnum)]` 限制参数取值
- 自定义 `value_parser` 校验参数范围
- `anyhow::Result` 统一错误处理 + `.context()` 上下文
- `tracing_subscriber` 根据 `-v` 次数动态设置日志级别

**前端类比**：这就像用 `commander.js` + `winston` + `try/catch` 搭建 Node.js CLI 工具，但类型更安全、帮助文档自动生成。

## 2. 完整代码 — src/main.rs

```rust
use anyhow::{bail, Context, Result};
use clap::{Parser, Subcommand, ValueEnum};
use tracing::{debug, error, info, warn};

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 1. 自定义 value_parser：温度范围验证
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

/// 解析温度参数，确保在 0.0~2.0 范围内
fn parse_temperature(s: &str) -> Result<f64, String> {
    let temp: f64 = s
        .parse()
        .map_err(|_| format!("'{s}' is not a valid number"))?;
    if (0.0..=2.0).contains(&temp) {
        Ok(temp)
    } else {
        Err(format!(
            "temperature must be between 0.0 and 2.0, got {temp}"
        ))
    }
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 2. ValueEnum：受限枚举参数
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

/// 输出格式（只允许 json / table / plain 三种）
#[derive(ValueEnum, Clone, Debug)]
enum OutputFormat {
    Json,
    Table,
    Plain,
}

// default_value_t 需要 Display trait
impl std::fmt::Display for OutputFormat {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            OutputFormat::Json => write!(f, "json"),
            OutputFormat::Table => write!(f, "table"),
            OutputFormat::Plain => write!(f, "plain"),
        }
    }
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 3. 嵌套子命令：provider 的二级操作
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#[derive(Subcommand, Debug)]
enum ProviderAction {
    /// List all available providers
    List,
    /// Set the default provider
    Set {
        /// Provider name (e.g., openai, anthropic)
        name: String,
    },
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 4. 子命令枚举（模仿 ZeroClaw 的 Commands）
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#[derive(Subcommand, Debug)]
enum Commands {
    /// Initialize configuration and workspace
    Init {
        /// Overwrite existing configuration
        #[arg(long)]
        force: bool,
    },

    /// Run a task with the AI agent
    Run {
        /// The message/prompt to send
        #[arg(short, long)]
        message: String,

        /// Model temperature (0.0 ~ 2.0)
        #[arg(short, long, default_value = "0.7", value_parser = parse_temperature)]
        temperature: f64,

        /// Output format
        #[arg(short, long, default_value_t = OutputFormat::Plain)]
        format: OutputFormat,
    },

    /// Show system status
    Status,

    /// Manage AI providers
    Provider {
        #[command(subcommand)]
        action: ProviderAction,
    },
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 5. 顶层 CLI 结构
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

/// A mini ZeroClaw-style CLI tool
#[derive(Parser, Debug)]
#[command(name = "miniclaw", version, about, long_about = None)]
struct Cli {
    /// Path to config file
    #[arg(long, global = true)]
    config: Option<String>,

    /// Increase verbosity (-v = info, -vv = debug, -vvv = trace)
    #[arg(short, long, global = true, action = clap::ArgAction::Count)]
    verbose: u8,

    #[command(subcommand)]
    command: Commands,
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 6. 子命令处理函数（每个返回 anyhow::Result）
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

fn cmd_init(force: bool, config_path: &Option<String>) -> Result<()> {
    let path = config_path.as_deref().unwrap_or("~/.miniclaw/config.toml");
    info!(path = path, force = force, "Initializing configuration");

    if !force {
        // 模拟检查文件是否已存在
        debug!("Checking if config already exists at {}", path);
        // 实际项目中这里会检查文件系统
    }

    println!("✅ Configuration initialized at {}", path);
    if force {
        warn!("Existing configuration was overwritten (--force)");
    }
    Ok(())
}

fn cmd_run(message: &str, temperature: f64, format: &OutputFormat) -> Result<()> {
    info!(
        temperature = temperature,
        format = %format,
        "Running task"
    );

    // 模拟业务逻辑验证
    if message.trim().is_empty() {
        bail!("message cannot be empty");
    }

    debug!(message = message, "Sending to AI model");

    // 模拟 AI 响应
    let response = format!("Echo: {}", message);

    match format {
        OutputFormat::Json => {
            println!("{{\"response\": \"{}\", \"temperature\": {}}}", response, temperature);
        }
        OutputFormat::Table => {
            println!("┌─────────────┬────────────────────┐");
            println!("│ Response    │ {:<18} │", response);
            println!("│ Temperature │ {:<18} │", temperature);
            println!("└─────────────┴────────────────────┘");
        }
        OutputFormat::Plain => {
            println!("{}", response);
        }
    }
    Ok(())
}

fn cmd_status(config_path: &Option<String>) -> Result<()> {
    let path = config_path.as_deref().unwrap_or("(default)");
    info!("Checking system status");

    println!("🔧 MiniClaw Status");
    println!("  Config:   {}", path);
    println!("  Version:  {}", env!("CARGO_PKG_VERSION"));
    println!("  Status:   OK");
    Ok(())
}

fn cmd_provider_list() -> Result<()> {
    info!("Listing providers");
    println!("Available providers:");
    println!("  1. openai     (default)");
    println!("  2. anthropic");
    println!("  3. local");
    Ok(())
}

fn cmd_provider_set(name: &str) -> Result<()> {
    let valid = ["openai", "anthropic", "local"];
    if !valid.contains(&name) {
        bail!(
            "unknown provider '{}'. Valid: {}",
            name,
            valid.join(", ")
        );
    }
    info!(provider = name, "Setting default provider");
    println!("✅ Default provider set to: {}", name);
    Ok(())
}

// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
// 7. main 函数：初始化 + 解析 + 分发
// ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

fn main() -> Result<()> {
    // 7a. 解析 CLI 参数（失败会自动打印帮助并退出）
    let cli = Cli::parse();

    // 7b. 根据 -v 次数设置日志级别
    let log_level = match cli.verbose {
        0 => "warn",
        1 => "info",
        2 => "debug",
        _ => "trace",
    };
    let filter = format!("{},hyper=warn,reqwest=warn", log_level);

    tracing_subscriber::fmt()
        .with_env_filter(
            tracing_subscriber::EnvFilter::try_from_default_env()
                .unwrap_or_else(|_| filter.parse().unwrap()),
        )
        .with_target(false)
        .init();

    debug!(verbose = cli.verbose, "Log level: {}", log_level);

    // 7c. match 分发子命令（enum 穷举，编译器保证不遗漏）
    match cli.command {
        Commands::Init { force } => {
            cmd_init(force, &cli.config)
                .context("failed to initialize configuration")?;
        }
        Commands::Run { message, temperature, format } => {
            cmd_run(&message, temperature, &format)
                .context("failed to run task")?;
        }
        Commands::Status => {
            cmd_status(&cli.config)
                .context("failed to get status")?;
        }
        Commands::Provider { action } => match action {
            ProviderAction::List => {
                cmd_provider_list().context("failed to list providers")?;
            }
            ProviderAction::Set { name } => {
                cmd_provider_set(&name)
                    .context("failed to set provider")?;
            }
        },
    }

    Ok(())
}
```

## 3. 运行示例

```bash
$ cargo run -- --help              # 自动生成帮助（/// 注释 → 帮助文本）
$ cargo run -- init --force        # → ✅ Configuration initialized
$ cargo run -- run -m "Hello AI" -t 0.9 -f json
# → {"response": "Echo: Hello AI", "temperature": 0.9}

$ cargo run -- run -m "test" -t 3.0   # value_parser 自动拦截越界
# → error: temperature must be between 0.0 and 2.0, got 3

$ cargo run -- status              # → 🔧 MiniClaw Status ...
$ cargo run -- provider list       # → Available providers: openai, anthropic, local
$ cargo run -- provider set anthropic  # → ✅ Default provider set to: anthropic

$ cargo run -- -vv run -m "Hello"  # -vv = debug 级别，显示调试日志
# → DEBUG Sending to AI model message="Hello"
# → Echo: Hello
```

## 4. 前端对比：等价的 Node.js 版本

```typescript
// 等价的 commander.js 实现（简化版）
import { Command } from "commander";

const program = new Command()
  .name("miniclaw").version("0.1.0")
  .option("--config <path>", "Config file")
  .option("-v, --verbose", "Verbosity", (_, prev) => prev + 1, 0);

program.command("init").option("--force").action(({ force }) => {
  console.log(`✅ Initialized (force=${force ?? false})`);
});

program.command("run")
  .requiredOption("-m, --message <msg>", "Message")
  .option("-t, --temperature <n>", "Temp", parseFloat, 0.7)
  .action(({ message, temperature }) => {
    // ❌ 校验需手动写，忘了就是 bug；类型是 any
    if (temperature < 0 || temperature > 2) process.exit(1);
    console.log(`Echo: ${message}`);
  });
program.parse();
```

| 对比项 | Rust (clap) | TypeScript (commander) |
|--------|-------------|------------------------|
| 类型安全 | 编译时保证 | 运行时 `any` |
| 帮助文档 | `///` 注释自动生成 | `.description()` 手动写 |
| 参数校验 | `value_parser` 解析时拦截 | `action` 里手动 `if` |
| 子命令遗漏 | enum `match` 编译器强制穷举 | 忘写无提示 |
| 错误传播 | `?` + `.context()` 链式 | `try/catch` 手动 |

## 5. ZeroClaw 关联说明

本骨架简化自 ZeroClaw 的 `main.rs`，核心结构一一对应：

```
ZeroClaw 真实结构                    本例 miniclaw
─────────────────────────────────────────────────────
struct Cli (config_dir, command)  →  struct Cli (config, verbose, command)
enum Commands (30+ 子命令)        →  enum Commands (4 个子命令)
  Onboard { force, ... }         →    Init { force }
  Agent { message, temp, ... }   →    Run { message, temperature, format }
  Status                         →    Status
  Service { action }             →    Provider { action }
enum CompletionShell (ValueEnum) →  enum OutputFormat (ValueEnum)
parse_temperature (value_parser) →  parse_temperature (value_parser)
tracing_subscriber 初始化        →  tracing_subscriber 初始化（+ verbose 联动）
match cli.command { ... }        →  match cli.command { ... }
```

学完本例后，你可以直接去读 ZeroClaw 的 `src/main.rs`，结构完全相同。

## 6. 代码架构要点速查

```
miniclaw CLI 架构：

  Cli::parse()                      ← 一行解析所有参数
       │
       ├── --config <path>          ← global 参数，所有子命令共享
       ├── -v / -vv / -vvv          ← ArgAction::Count 计数器
       │
       └── match command {          ← enum 穷举分发
             Init { force }         ← cmd_init() → Result<()>
             Run { msg, temp, fmt } ← cmd_run()  → Result<()>
             Status                 ← cmd_status() → Result<()>
             Provider { action }    ← 嵌套 match
               ├── List             ← cmd_provider_list()
               └── Set { name }    ← cmd_provider_set()
           }

  关键模式：
    1. 每个处理函数返回 anyhow::Result<()>
    2. 调用处加 .context("描述") 附加上下文
    3. tracing 宏替代 println! 做调试输出
    4. value_parser 在解析阶段就拦截非法参数
```
