## 核心三步骤

### 步骤 1: 启动 Gateway（30秒）

```bash
# 启动 OpenClaw Gateway
openclaw gateway start

# 预期输出:
# ✓ Gateway started successfully
# ✓ Listening on port 3000
# ✓ Ready to accept connections
```

**作用**: Gateway 是消息的统一入口，必须先启动。

---

### 步骤 2: 配对通道（2分钟）

```bash
# 启动配对流程
openclaw pairing start

# 预期输出:
# ? Select a channel to pair: (Use arrow keys)
#   ❯ WhatsApp
#     Telegram
#     Discord
#     Slack
```

**操作流程**:

1. **选择通道**: 用方向键选择你想使用的通道（如 WhatsApp）
2. **扫描二维码**: 如果是 WhatsApp，会显示二维码，用手机扫描
3. **发送验证消息**: 系统会提示你发送一条私信（DM）到特定账号
4. **确认配对**: 发送后，系统会自动确认配对成功

```bash
# 配对成功后的输出:
# ✓ Pairing successful!
# ✓ Channel: WhatsApp
# ✓ User: +1234567890
# ✓ Pairing ID: abc123def456
```

**重要**: 配对是一次性操作，配对成功后会保存配置，下次不需要重复。

---

### 步骤 3: 发送消息（1分钟）

```bash
# 发送第一条消息
openclaw message send "Hello from OpenClaw!"

# 预期输出:
# ✓ Message sent successfully
# ✓ Channel: WhatsApp
# ✓ Recipient: +1234567890
# ✓ Message ID: msg_abc123
# ✓ Status: delivered
```

**验证**: 打开你的 WhatsApp（或你选择的通道），应该能看到这条消息。

## 常见问题快速解决

### 问题 1: Gateway 启动失败

```bash
# 错误: Port 3000 already in use

# 解决方案 1: 停止占用端口的进程
lsof -ti:3000 | xargs kill -9

# 解决方案 2: 使用其他端口
openclaw gateway start --port 3001
```

### 问题 2: 配对失败

```bash
# 错误: Pairing timeout

# 解决方案:
# 1. 确保 Gateway 正在运行
openclaw gateway status

# 2. 检查网络连接
ping google.com

# 3. 重新启动配对
openclaw pairing start
```

### 问题 3: 消息发送失败

```bash
# 错误: No active pairing found

# 解决方案: 检查配对状态
openclaw pairing list

# 如果没有配对，重新配对
openclaw pairing start
```

### 问题 4: 找不到 openclaw 命令

```bash
# 错误: command not found: openclaw

# 解决方案: 全局安装 OpenClaw
npm install -g @openchatai/openclaw

# 或者使用 npx
npx @openchatai/openclaw message send "Hello"
```

## 核心概念速记

### Gateway 架构
- **作用**: 统一入口，多通道适配
- **类比**: 机场转机大厅
- **必须**: 发送消息前必须启动

### DM 配对
- **作用**: 安全验证，确保消息发送到正确的人
- **类比**: 加好友验证
- **一次性**: 配对成功后会保存

### 消息发送
- **命令**: `openclaw message send "text"`
- **前提**: Gateway 运行 + 已配对
- **双向**: 可以发送和接收响应

## 最小可用配置文件

如果你想跳过交互式配对，可以手动创建配置文件：

```yaml
# ~/.openclaw/config.yaml
gateway:
  port: 3000
  host: localhost

pairings:
  - channel: whatsapp
    user: "+1234567890"
    pairingId: "abc123def456"
    status: active
```

**注意**: 手动配置需要有效的 `pairingId`，建议使用交互式配对。
