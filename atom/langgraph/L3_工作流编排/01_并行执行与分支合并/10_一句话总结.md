## 核心机制回顾

### 1. Send API - 动态并行的核心

**定义**: 在条件边中动态创建并行任务的类

**关键特性**:
- 运行时决定并行任务数量
- 可以发送自定义状态到目标节点
- 支持 Map-Reduce 模式

**代码示例**:
```python
from langgraph.types import Send

def continue_to_jokes(state):
    return [Send("generate_joke", {"subject": s}) for s in state["subjects"]]
```

---

### 2. Reducer 函数 - 自动合并的保证

**定义**: 用于合并并行节点状态更新的函数

**关键特性**:
- 避免状态覆盖问题
- 使用 `Annotated[type, reducer]` 语法
- 支持内置和自定义 reducer

**代码示例**:
```python
from typing import Annotated
import operator

class State(TypedDict):
    results: Annotated[list, operator.add]  # 自动追加
```

---

### 3. BSP 模型 - 同步机制的基础

**定义**: Bulk Synchronous Parallel 模型，确保并行执行的顺序和同步

**关键特性**:
- 超步（Superstep）：并行执行周期
- 同步点：超步之间的屏障
- 确定性：相同输入产生相同输出

---

### 4. Fan-out/Fan-in - 并行模式

**定义**: 从一个节点扇出到多个并行节点，再扇入到一个节点

**关键特性**:
- 静态并行：编译时确定（多条边）
- 动态并行：运行时确定（Send API）
- 自动同步：等待所有并行任务完成

## 性能提升对比

| 场景 | 串行时间 | 并行时间 | 提升倍数 |
|------|----------|----------|----------|
| 3 个任务（各 5 秒） | 15 秒 | 5 秒 | 3 倍 |
| 5 个任务（各 3 秒） | 15 秒 | 3 秒 | 5 倍 |
| 10 个文档处理 | 30 秒 | 3 秒 | 10 倍 |
| 多智能体协作 | 15 秒 | 5 秒 | 3 倍 |

**核心公式**:
```
并行时间 = max(task1_time, task2_time, ..., taskN_time)
串行时间 = task1_time + task2_time + ... + taskN_time
性能提升 = 串行时间 / 并行时间
```

## 常见误区回顾

### 误区 1: 所有任务都应该并行执行 ❌

**正确理解**: 只有独立任务才适合并行，有依赖关系的任务必须串行。

### 误区 2: 并行度越高越好 ❌

**正确理解**: 需要考虑资源限制（CPU、内存、API 配额），合理控制并行度。

### 误区 3: Reducer 会被每个节点调用 ❌

**正确理解**: Reducer 只在合并并行节点的结果时调用。

### 误区 4: 静态并行和动态并行性能相同 ❌

**正确理解**: 静态并行性能更好，动态并行更灵活。

### 误区 5: 并行执行总是比串行快 ❌

**正确理解**: 需要考虑并行开销，任务太小时并行可能更慢。

---

## 最佳实践总结

### 1. 选择合适的并行策略
- 任务数量固定 → 静态并行（多条边）
- 任务数量动态 → 动态并行（Send API）
- 性能敏感 → 静态并行
- 灵活性优先 → 动态并行

### 2. 正确使用 Reducer
- 列表追加：`Annotated[list, operator.add]`
- 数值累加：`Annotated[int, operator.add]`
- 自定义合并：定义自己的 reducer 函数
- 覆盖模式：不使用 Annotated（默认）

### 3. 控制并行度
- 设置最大并行任务数量
- 使用批处理减少任务数
- 监控资源使用情况
- 考虑 API 限流

### 4. 错误处理
- 在并行任务中捕获异常
- 返回错误状态而非抛出异常
- 在聚合节点中处理错误
- 提供降级策略

## 快速参考卡

### Send API 语法
```python
from langgraph.types import Send

# 创建单个并行任务
Send("node_name", {"key": "value"})

# 创建多个并行任务
[Send("node_name", {"item": item}) for item in items]
```

### Reducer 语法
```python
from typing import Annotated
import operator

# 列表追加
results: Annotated[list, operator.add]

# 数值相加
total: Annotated[int, operator.add]

# 自定义 reducer
def custom_reducer(current, new):
    return current + new

data: Annotated[list, custom_reducer]
```

### Fan-out/Fan-in 语法
```python
# Fan-out（扇出）
builder.add_edge("source", "target1")
builder.add_edge("source", "target2")

# Fan-in（扇入）
builder.add_edge("target1", "sink")
builder.add_edge("target2", "sink")
```

### 条件边语法
```python
# 返回 Send 对象列表触发并行
def route(state):
    return [Send("node", {"item": item}) for item in state["items"]]

builder.add_conditional_edges("source", route, ["node"])
```
