## 卡片1：Dijkstra算法直觉

**一句话：** Dijkstra就像导航系统，每次选择"目前看起来最近"的路口探索。

**举例：**
```python
# 贪心策略
while 未访问节点不为空:
    选择距离最小的节点u
    标记u为已访问
    更新u的邻居的距离
```

**应用：** 在知识图谱中找最可靠的推理路径，保证找到最优解。

---

## 卡片2：A*搜索加速

**一句话：** A*比Dijkstra多了"方向感"，优先探索接近目标的节点。

**公式：**
```
f(n) = g(n) + h(n)
g(n) = 起点到n的实际代价
h(n) = n到终点的启发式估计
```

**应用：** 在大规模知识图谱中快速找到推理路径，比Dijkstra快5-10倍。

---

## 卡片3：权重设计原则

**一句话：** 权重量化"代价"，设计好坏直接决定路径质量。

**公式：**
```python
weight = 基础代价 + 不确定性惩罚 + 时间衰减 + 相关性代价
```

**应用：** 在AI Agent中，权重设计决定推理链的可信度和相关性。

---

## 卡片4：启发式函数约束

**一句话：** 启发式函数h(n)必须可采纳（不高估），否则A*不保证最优。

**约束：**
```
h(n) ≤ h*(n)  (可采纳性)
h(n) ≤ cost(n,n') + h(n')  (一致性)
```

**应用：** 在知识图谱中，用语义相似度作为启发式，需要校准以保证可采纳性。

---

## 卡片5：推理链建模

**一句话：** 推理过程 = 图遍历，节点是知识点，边是推理步骤。

**建模：**
```
节点 = 实体/概念
边 = 关系/推理步骤
权重 = 推理代价/可信度
```

**应用：** 将抽象的"推理"具象化为"路径搜索"，让推理过程可解释、可优化。

---

## 卡片6：多跳推理优化

**一句话：** 多跳推理需要平衡路径长度和路径质量。

**策略：**
```
- 启发式剪枝：提前剪掉不可能的路径
- 束搜索：每步只保留top-k候选
- K最短路径：提供多样化的推理路径
```

**应用：** 复杂问答需要3-5跳推理，优化策略减少搜索空间。

---

## 卡片7：GraphRAG架构

**一句话：** GraphRAG = 图检索 + 向量检索 + 生成，结合结构化和语义检索。

**流程：**
```
问题 → 提取实体 → 图检索(最短路径) → 向量检索 → LLM生成
```

**应用：** 2025年StepChain框架用BFS-based推理流，多跳问答准确率提升30%+。

---

## 卡片8：2025年理论突破

**一句话：** STOC 2025突破40年排序屏障，O(m log^{2/3} n)算法。

**突破点：**
```
传统：Dijkstra需要完全排序 → O(m log n)
新算法：只需部分排序 → O(m log^{2/3} n)
```

**应用：** 大规模知识图谱（百万级节点）性能提升2-3倍。

---

## 卡片9：iLLM-A*创新

**一句话：** 用LLM生成waypoints引导A*搜索，实现1000×加速。

**原理：**
```
LLM（战略层）：生成关键路径点
A*（战术层）：在waypoints之间精确搜索
```

**应用：** 大规模路径规划、机器人导航、知识图谱推理。

---

## 卡片10：生产级架构

**一句话：** LangGraph + RAG + UCP = 2026年生产级路径规划三位一体。

**架构：**
```
LangGraph：状态图管理推理流程
RAG：知识图谱检索提供上下文
UCP：统一上下文协议保证一致性
```

**应用：** 可观测、可监控、可维护的企业级AI Agent系统。
