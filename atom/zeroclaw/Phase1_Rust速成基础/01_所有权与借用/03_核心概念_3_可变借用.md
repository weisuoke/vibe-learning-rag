# 核心概念 3：可变借用 &mut T

> **一句话定义：** `&mut T` 是对数据的独占可变访问权——编译器保证同一时刻只有一个"写者"，从根源杜绝数据竞争。

---

## 1. 什么是可变借用

### 1.1 &mut T 的含义：独占可变访问

不可变借用 `&T` 是"只读共享"——多人可以同时看同一份文件。可变借用 `&mut T` 则是"独占写入"——一个人拿走文件修改时，其他人既不能看也不能改。

```rust
fn main() {
    let mut name = String::from("ZeroClaw");
    let r = &mut name;        // 创建可变借用
    r.push_str(" Agent");     // 通过可变借用修改数据
    println!("{}", r);        // "ZeroClaw Agent"
}
```

### 1.2 为什么需要 `let mut` + `&mut` 双重标记

Rust 要求你在两个地方都明确表达"可变"意图：

```rust
fn main() {
    let mut score = 100;    // 第一层：变量本身声明为 mut
    let r = &mut score;     // 第二层：借用时用 &mut
    *r += 50;
    println!("{}", score);  // 150
}
```

| 标记 | 含义 | 类比 |
|------|------|------|
| `let mut x` | "这个变量允许被修改" | 文件设置为"可编辑" |
| `&mut x` | "我现在要借来修改它" | 有人打开文件进入编辑模式 |

漏掉任何一层编译器都会拒绝：

```rust
let name = String::from("hello");
// let r = &mut name;  // 错误: cannot borrow `name` as mutable,
                       //       as it is not declared as mutable
```

### 1.3 与 TypeScript 的自由修改对比

```typescript
// TypeScript: 任何引用都能随时修改，无任何保护
const config = { name: "ZeroClaw", port: 8080 };
const ref1 = config;
const ref2 = config;
ref1.name = "Agent";  // 同时修改——多线程/异步下就是 data race
ref2.port = 3000;
```

```rust
fn main() {
    let mut config = Config { name: "ZeroClaw".into(), port: 8080 };
    let r1 = &mut config;
    // let r2 = &mut config;  // 编译错误：不能同时有两个可变借用
    r1.name = "Agent".into();
}
```

**核心区别：** TypeScript 信任程序员不会犯错；Rust 通过编译器规则让你不可能犯错。

## 2. 可变借用的规则

### 规则 1：任一时刻只能有一个 &mut T

```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &mut s;
    // let r2 = &mut s;  // error[E0499]: cannot borrow `s` as mutable more than once
    r1.push_str(" world");
    println!("{}", r1);
}
```

### 规则 2：&mut T 和 &T 不能同时存在

```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;       // OK
    let r2 = &s;       // OK
    // let r3 = &mut s; // error[E0502]: cannot borrow as mutable because also borrowed as immutable
    println!("{} and {}", r1, r2);
}
```

### 规则 3：可变借用期间所有者不能使用

```rust
fn main() {
    let mut data = vec![1, 2, 3];
    let r = &mut data;
    r.push(4);
    // println!("{:?}", data);  // 编译错误: 所有者被冻结
    println!("{:?}", r);        // OK: 通过可变借用访问
}
```

**三条规则总结表：**

| 场景 | 是否允许 | 原因 |
|------|---------|------|
| 多个 `&T` | 允许 | 只读不冲突 |
| 一个 `&mut T` | 允许 | 独占写入 |
| 多个 `&mut T` | 禁止 | 写-写冲突 |
| `&T` + `&mut T` 同时存在 | 禁止 | 读-写冲突 |
| 所有者 + `&mut T` 同时使用 | 禁止 | 等同读-写冲突 |

## 3. 为什么限制这么严格？

### 3.1 防止数据竞争

数据竞争需要三个条件同时满足：多指针访问同一数据 + 至少一个在写 + 无同步机制。Rust 的借用规则在编译期让前两个条件不可能同时成立。

### 3.2 防止迭代器失效

TypeScript 中的经典 bug：

```typescript
const items = [1, 2, 3, 4, 5];
for (const item of items) {
    if (item === 3) items.push(100); // 迭代时修改集合——未定义行为!
}
```

Rust 编译器直接阻止：

```rust
fn main() {
    let mut items = vec![1, 2, 3, 4, 5];
    for item in &items {
        if *item == 3 {
            // items.push(100);  // 编译错误! 不能在遍历时修改
        }
    }
}
```

### 3.3 Vec push + 引用的经典案例

```rust
fn main() {
    let mut v = vec![1, 2, 3];
    let first = &v[0];  // 不可变借用
    v.push(4);           // 编译错误! push 可能触发 Vec 扩容重分配内存
    // println!("{}", first); // first 将指向已释放的旧内存——悬垂指针!
}
```

`Vec` 在 `push` 时可能因容量不足而重新分配内存，之前的引用就指向已释放的旧内存。Rust 在编译期阻止这种灾难。

## 4. &mut self 方法模式

### 4.1 &self vs &mut self vs self 选择指南

```rust
struct Counter { value: u64, label: String }

impl Counter {
    fn get(&self) -> u64 { self.value }              // 只读
    fn increment(&mut self) { self.value += 1; }      // 需要修改
    fn into_label(self) -> String { self.label }       // 消费所有权
}
```

| 签名 | 含义 | 适用场景 |
|------|------|---------|
| `&self` | 只读借用 | getter、查询、格式化 |
| `&mut self` | 可变借用 | setter、状态变更、原地修改 |
| `self` | 转移所有权 | 转换、消费、Builder.build() |

### 4.2 Builder 模式中的 &mut self

```rust
struct RequestBuilder { url: String, method: String, timeout_ms: u64 }

impl RequestBuilder {
    fn new(url: &str) -> Self {
        Self { url: url.into(), method: "GET".into(), timeout_ms: 5000 }
    }
    fn method(&mut self, m: &str) -> &mut Self { self.method = m.into(); self }
    fn timeout(&mut self, ms: u64) -> &mut Self { self.timeout_ms = ms; self }
}

fn main() {
    let mut b = RequestBuilder::new("https://api.example.com");
    b.method("POST").timeout(10000);  // 链式调用
}
```

返回 `&mut Self` 支持链式调用，同时不转移所有权——build 后还能复用 builder。

## 5. 可变借用的生命周期

### 5.1 NLL 精确跟踪 &mut 的使用范围

非词法生命周期（NLL）让编译器追踪每个引用的**实际使用范围**，而非看花括号：

```rust
fn main() {
    let mut s = String::from("hello");
    let r1 = &s;
    let r2 = &s;
    println!("{} and {}", r1, r2);
    // --- r1, r2 的最后使用在这里，NLL 结束其借用 ---

    let r3 = &mut s;  // OK! r1、r2 已经"用完了"
    r3.push_str(" world");
    println!("{}", r3);
}
```

### 5.2 &mut 使用结束后可以创建新借用

```rust
fn main() {
    let mut data = vec![1, 2, 3];

    let writer = &mut data;   // 可变借用
    writer.push(4);
    // writer 最后一次使用在这里，NLL 结束其借用

    let snapshot: Vec<&i32> = data.iter().collect();  // 不可变借用——安全
    println!("data: {:?}", snapshot);

    let writer2 = &mut data;  // 又一个可变借用
    writer2.retain(|x| *x > 2);
    println!("filtered: {:?}", writer2); // [3, 4]
}
```

## 6. ZeroClaw 实战实例

### 6.1 apply_env_overrides(&mut self) —— 就地修改配置

```rust
// [来源: sourcecode/zeroclaw/src/config/schema.rs]
impl Config {
    pub fn apply_env_overrides(&mut self) {
        if let Ok(key) = std::env::var("ZEROCLAW_API_KEY")
            .or_else(|_| std::env::var("API_KEY"))
        {
            if !key.is_empty() { self.api_key = Some(key); }
        }
        if let Ok(provider) = std::env::var("ZEROCLAW_PROVIDER") {
            if !provider.is_empty() { self.default_provider = Some(provider); }
        }
        if let Ok(model) = std::env::var("ZEROCLAW_MODEL") {
            if !model.is_empty() { self.default_model = Some(model); }
        }
    }
}
```

用 `&mut self` 就地修改，避免克隆整个 Config 结构体。

### 6.2 SOP 引擎的 &mut self 方法族

```rust
// [来源: sourcecode/zeroclaw/src/sop/engine.rs]
impl SopEngine {
    pub fn new(config: SopConfig) -> Self { /* ... */ }       // 构造，不需要 mut
    pub fn sops(&self) -> &[Sop] { &self.sops }              // 只读查询
    pub fn active_runs(&self) -> &HashMap<String, SopRun> {   // 只读查询
        &self.active_runs
    }
    pub fn reload(&mut self, workspace_dir: &Path) { /* ... */ }     // 修改 sops
    pub fn start_run(&mut self, sop_name: &str, event: SopEvent)     // 修改 active_runs
        -> Result<SopRunAction> { /* ... */ }
    pub fn advance_step(&mut self, run_id: &str, result: SopStepResult) // 修改运行状态
        -> Result<SopRunAction> { /* ... */ }
    pub fn cancel_run(&mut self, run_id: &str) -> Result<()> { /* ... */ } // 移除运行
}
```

观察分界线：只读查询用 `&self`，状态变更用 `&mut self`，调用者一目了然。

### 6.3 Drop trait 中的 &mut self

```rust
// [来源: sourcecode/zeroclaw/src/tools/shell.rs]
struct EnvGuard { key: &'static str, original: Option<String> }

impl Drop for EnvGuard {
    fn drop(&mut self) {  // Drop 签名固定为 &mut self
        match &self.original {
            Some(val) => std::env::set_var(self.key, val),
            None => std::env::remove_var(self.key),
        }
    }
}
```

`Drop::drop` 用 `&mut self` 而非 `self`：避免所有权转移导致递归 drop，让你修改字段和清理资源，Rust 在 `drop` 返回后自动释放内存。

## 7. 常见错误与解决方案

### 错误场景：同时持有 &T 和 &mut T

```rust
let mut data = vec![1, 2, 3];
let first = &data[0];   // 不可变借用
data.push(4);            // 需要可变借用 -> 冲突!
println!("{}", first);
```

### 解决 1：重新组织代码，提前完成不可变借用

```rust
let mut data = vec![1, 2, 3];
let first = data[0];    // 复制值（i32 实现了 Copy），不产生借用
println!("first = {}", first);
data.push(4);            // 安全
```

### 解决 2：使用 clone() 打破借用依赖

```rust
let mut names = vec!["Alice".to_string(), "Bob".to_string()];
let saved = names[0].clone();  // clone 创建独立副本，切断借用
names.push("Charlie".to_string());
names[0] = "Alex".to_string();
println!("saved: {}, current: {:?}", saved, names);
```

注意：`clone()` 有运行时开销，不要滥用——先尝试重组代码。

### 解决 3：使用 Cell/RefCell 进行内部可变性

当设计确实需要在 `&self` 约束下修改数据时：

```rust
use std::cell::RefCell;

struct Cache {
    data: RefCell<Vec<String>>,         // 运行时借用检查
    hit_count: std::cell::Cell<u64>,    // Copy 类型用 Cell
}

impl Cache {
    fn get_or_insert(&self, key: &str) -> String {  // 注意：&self，不是 &mut self
        let data = self.data.borrow();
        if let Some(item) = data.iter().find(|s| s.starts_with(key)) {
            self.hit_count.set(self.hit_count.get() + 1);
            return item.clone();
        }
        drop(data);  // 必须先释放不可变借用
        let value = format!("{}_cached", key);
        self.data.borrow_mut().push(value.clone());
        value
    }
}
```

| 类型 | 适用场景 | 运行时开销 |
|------|---------|-----------|
| `Cell<T>` | T 实现了 Copy（如 i32, bool） | 几乎为零 |
| `RefCell<T>` | T 是任意类型（如 String, Vec） | 运行时借用检查 |

**警告：** `RefCell` 违反借用规则时会 panic，不是编译错误。它是逃生舱口，不是常规工具。
