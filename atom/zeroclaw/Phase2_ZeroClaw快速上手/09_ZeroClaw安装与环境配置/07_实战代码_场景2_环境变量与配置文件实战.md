# 实战代码 · 场景 2：环境变量与配置文件实战

> **场景目标：** 从零配置 ZeroClaw 的运行环境，包括编写 config.toml、设置环境变量、切换 Provider、配置代理，并理解配置优先级链的实际行为。

---

## 场景概述

### 你将完成什么？

```
onboard 引导 → 手写 config.toml → 环境变量配置 → Provider 切换 → 代理设置 → 优先级验证
  (2分钟)       (5分钟)            (3分钟)         (5分钟)        (3分钟)      (5分钟)
```

### 前置条件

| 条件 | 说明 |
|------|------|
| ZeroClaw 已安装 | 完成场景 1，`zeroclaw --version` 可输出版本号 |
| 至少一个 API Key | OpenRouter / OpenAI / Anthropic 任选（或本地 Ollama 免费） |

### 前端类比概览

```
~/.zeroclaw/config.toml    ≈    next.config.js     （项目级稳定配置）
ZEROCLAW_* 环境变量         ≈    .env.local          （环境级敏感配置）
命令行前缀临时覆盖          ≈    PORT=4000 npm start （单次临时覆盖）
zeroclaw onboard           ≈    npx create-next-app （交互式生成配置）
```

## 场景 2.1：从 onboard 到手动配置

> **目标：** 理解 `zeroclaw onboard` 做了什么，以及它生成的 config.toml 长什么样。

### 方式 1：交互式引导（推荐新手）

```bash
zeroclaw onboard
```

**交互过程：**

```
🧭 Welcome to ZeroClaw!

? Select your LLM provider:
  ❯ openrouter  (推荐，一个 key 访问多种模型)
    openai / anthropic / ollama

? Enter your API key: sk-or-v1-xxxxxxxxxxxx
? Select default model: anthropic/claude-sonnet-4-6
? Set default temperature (0.0-2.0) [0.7]: 0.7

✅ Config saved to ~/.zeroclaw/config.toml
💡 Try: zeroclaw agent -m "What can you do?"
```

> **前端类比：** 就像 `npx create-next-app` 问你要不要 TypeScript，然后一次性生成配置文件。

### 方式 2：非交互式（脚本 / CI）

```bash
zeroclaw onboard \
  --api-key "sk-or-v1-xxxxxxxxxxxx" \
  --provider openrouter \
  --model "anthropic/claude-sonnet-4-6"
```

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh — `--onboard`、`--api-key`、`--provider`、`--model` 参数处理逻辑]

### 查看 + 验证生成的配置

```bash
# 查看生成的配置
cat ~/.zeroclaw/config.toml
# 预期输出:
#   api_key = "sk-or-v1-xxxxxxxxxxxx"
#   default_provider = "openrouter"
#   default_model = "anthropic/claude-sonnet-4-6"
#   default_temperature = 0.7

# 验证配置生效
zeroclaw doctor
# 预期: All checks passed! 🎉

# 端到端验证
zeroclaw agent -m "Hello! Tell me which provider you're using."
# 预期末尾: [provider: openrouter | model: claude-sonnet-4-6]
```

## 场景 2.2：手写 config.toml 完整模板

> **目标：** 理解 config.toml 的完整结构，能根据需要手动编写和修改。

```bash
mkdir -p ~/.zeroclaw
nano ~/.zeroclaw/config.toml    # 或 vim / code
```

### 完整模板（复制后按需修改）

```toml
# ~/.zeroclaw/config.toml — 完整配置模板
# 未注释 = 常用字段，已注释 = 按需启用

# === 基础配置 ===
# api_key = "sk-or-v1-xxx"            # ⚠️ 建议走环境变量，不写文件里
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
default_temperature = 0.7
# api_url = "https://your-proxy.com/v1"  # 自定义端点

# === Gateway（内置 HTTP 网关）===
[gateway]
port = 42617                           # 类似 PORT=3000 npm start
host = "127.0.0.1"                     # 127.0.0.1 = 仅本机，[::] = 所有网卡
# allow_public_bind = false            # 生产环境改 true

# === 内存后端 ===
[memory]
backend = "sqlite"                     # sqlite = 零配置，postgres = 生产级

# === 安全策略 ===
[autonomy]
workspace_only = true                  # Agent 只能操作工作区内文件

# === 运行时 ===
[runtime]
kind = "native"                        # native / docker

# === 按需启用 ===
# [reliability]
# retry_max = 3
# fallback_model = "openai/gpt-4o-mini"

# [skills]
# open_skills_enabled = false

# [observability]
# log_level = "info"                   # trace / debug / info / warn / error
```

> **核心原则：** 最小配置只需 `default_provider` + `default_model`（API Key 走环境变量）。

### 各节速查表

| 节名 | 功能 | 最常改的字段 | 前端类比 |
|------|------|-------------|---------|
| 顶层字段 | Provider、模型、温度 | `default_provider`, `default_model` | `next.config.js` 顶层 |
| `[gateway]` | HTTP 网关端口和地址 | `port`, `host` | `devServer` 配置 |
| `[memory]` | 对话持久化后端 | `backend` | localStorage vs DB |
| `[autonomy]` | Agent 操作权限 | `workspace_only` | CSP 安全策略 |
| `[reliability]` | 重试和降级 | `retry_max`, `fallback_model` | `axios-retry` |
| `[observability]` | 日志级别 | `log_level` | `debug` / `loglevel` |

[来源: sourcecode/zeroclaw/src/config/mod.rs — 配置结构体定义与各节解析]

## 场景 2.3：环境变量配置实战

> **目标：** 掌握三种设置方式：持久化、项目级、单次覆盖。

### 方式 1：写入 Shell 配置文件（持久化）

```bash
# 确认你的 Shell
echo $SHELL    # /bin/zsh → ~/.zshrc，/bin/bash → ~/.bashrc

# 追加环境变量
cat >> ~/.zshrc << 'EOF'

# === ZeroClaw 配置 ===
export ZEROCLAW_API_KEY="sk-or-v1-xxxxxxxxxxxx"
export ZEROCLAW_PROVIDER="openrouter"
export ZEROCLAW_MODEL="anthropic/claude-sonnet-4-6"
EOF

source ~/.zshrc

# 验证
env | grep ZEROCLAW
# ZEROCLAW_API_KEY=sk-or-v1-xxxxxxxxxxxx
# ZEROCLAW_PROVIDER=openrouter
# ZEROCLAW_MODEL=anthropic/claude-sonnet-4-6
```

### 方式 2：使用 .env 文件（项目级隔离）

```bash
cat > ~/my-project/.env << 'EOF'
ZEROCLAW_API_KEY=sk-or-v1-xxxxxxxxxxxx
ZEROCLAW_PROVIDER=openrouter
ZEROCLAW_MODEL=anthropic/claude-sonnet-4-6
EOF

echo ".env" >> ~/my-project/.gitignore  # ⚠️ 防止泄露 API Key

# 加载（set -a 让变量自动 export，子进程可见）
set -a && source ~/my-project/.env && set +a
```

> **前端类比：** `set -a && source .env && set +a` = `dotenv` 库自动加载 `.env` 文件。

### 方式 3：单次命令覆盖（临时测试）

```bash
# 只对这一次命令生效
ZEROCLAW_PROVIDER=ollama ZEROCLAW_MODEL=llama3.1:8b zeroclaw agent -m "Hello from Ollama!"

# 验证：原始值不变
echo $ZEROCLAW_PROVIDER   # → openrouter
```

> **前端类比：** `PORT=4000 npm start` — 只对这次启动生效。

## 场景 2.4：Provider 切换实战

> **目标：** 在 4 个主流 Provider 之间自由切换。

### Provider 对比表

| Provider | API Key 格式 | 模型示例 | 适用场景 |
|----------|-------------|---------|---------|
| **OpenRouter** ⭐ | `sk-or-v1-...` | `anthropic/claude-sonnet-4-6` | 日常开发（一个 Key 用所有模型） |
| **Ollama** | 不需要 | `llama3.1:8b` | 离线/隐私/免费 |
| **OpenAI** | `sk-...` | `gpt-4o` | GPT 系列直连 |
| **Anthropic** | `sk-ant-...` | `claude-sonnet-4-6` | Claude 系列直连 |

### 切换方式（通用模式）

```bash
# 切换到任意 Provider 的通用三步：
export ZEROCLAW_API_KEY="<对应格式的 Key>"
export ZEROCLAW_PROVIDER="<provider 名>"
export ZEROCLAW_MODEL="<模型名>"
zeroclaw agent -m "Test"
```

### 实战：切换到 Ollama（本地免费）

```bash
# 1. 确保 Ollama 运行中
curl -fsSL https://ollama.com/install.sh | sh   # 安装（如未安装）
ollama pull llama3.1:8b                          # 下载模型

# 2. 切换 ZeroClaw
export ZEROCLAW_PROVIDER="ollama"
export ZEROCLAW_MODEL="llama3.1:8b"
unset ZEROCLAW_API_KEY                           # Ollama 不需要 Key

# 3. 测试
zeroclaw agent -m "Are you running locally?"
# [tokens: 30 | latency: 2.5s | provider: ollama | model: llama3.1:8b]
```

### 快速切换技巧：不改配置，单次覆盖

```bash
# 配置是 OpenRouter，临时用 Ollama 跑一次
ZEROCLAW_PROVIDER=ollama ZEROCLAW_MODEL=llama3.1:8b zeroclaw agent -m "Quick test"

# 或临时用 config.toml（持久切换）
cat > ~/.zeroclaw/config.toml << 'EOF'
default_provider = "ollama"
default_model = "llama3.1:8b"
EOF
```

> **前端类比：** 切换 Provider = 切换后端 API 地址。`API_BASE_URL=http://staging npm start` 让你连到不同后端。

## 场景 2.5：代理配置实战（公司/受限网络）

> **目标：** 在需要代理的网络环境中配置 ZeroClaw。

### HTTP 代理（最常见）

```bash
export ZEROCLAW_PROXY_ENABLED=true                        # ⚠️ 总开关，不设则其他代理变量不生效
export ZEROCLAW_HTTPS_PROXY="http://proxy.company.com:8080"
export ZEROCLAW_NO_PROXY="localhost,127.0.0.1,.company.internal"
```

### SOCKS 代理 / 本地代理工具

```bash
# SOCKS5 代理
export ZEROCLAW_PROXY_ENABLED=true
export ZEROCLAW_ALL_PROXY="socks5://proxy.company.com:1080"

# 本地 Clash
export ZEROCLAW_PROXY_ENABLED=true
export ZEROCLAW_HTTPS_PROXY="http://127.0.0.1:7890"
```

### 验证代理

```bash
zeroclaw doctor
# ✅ Proxy: http://proxy.company.com:8080 (enabled)
# ✅ Provider: openrouter (reachable via proxy)
```

### 代理作用范围

```bash
export ZEROCLAW_PROXY_SCOPE="zeroclaw"  # 默认：只有 ZeroClaw API 请求走代理
export ZEROCLAW_PROXY_SCOPE="system"    # Agent 执行的 curl/git 等子进程也走代理
```

> **前端类比：** `npm config set proxy` 只影响 npm 请求。`ZEROCLAW_PROXY_SCOPE` 控制代理是只影响 ZeroClaw 还是也影响子进程。

## 场景 2.6：配置优先级验证实战

> **目标：** 亲手验证"环境变量覆盖 config.toml"的行为。

### 优先级链

```
最高                                          最低
 ▼                                             ▼
命令行前缀  →  Shell 环境变量  →  config.toml  →  代码默认值
(单次生效)    (终端会话内)      (持久文件)      (硬编码回退)
```

### 实验 1：环境变量覆盖 config.toml

```bash
# config.toml 设的是 openrouter
grep default_provider ~/.zeroclaw/config.toml
# → default_provider = "openrouter"

# 不设环境变量 → 用 config.toml 的值
zeroclaw agent -m "Which provider?"
# → [provider: openrouter]  ← 来自 config.toml

# 设环境变量 → 覆盖 config.toml
export ZEROCLAW_PROVIDER="ollama"
export ZEROCLAW_MODEL="llama3.1:8b"
zeroclaw agent -m "Which provider now?"
# → [provider: ollama]  ← 环境变量赢了！

# 清理
unset ZEROCLAW_PROVIDER ZEROCLAW_MODEL
```

### 实验 2：命令行前缀覆盖环境变量

```bash
export ZEROCLAW_PROVIDER="openrouter"
ZEROCLAW_PROVIDER=ollama ZEROCLAW_MODEL=llama3.1:8b zeroclaw agent -m "Who?"
# → [provider: ollama]  ← 命令行前缀赢了

zeroclaw agent -m "And now?"
# → [provider: openrouter]  ← 环境变量恢复
```

### 实验 3：回退变量

```bash
unset ZEROCLAW_API_KEY
export API_KEY="sk-or-v1-xxxxxxxxxxxx"   # 通用回退变量

zeroclaw doctor
# → ✅ API key: sk-or-v1-xxxx...xxxx    ← ZeroClaw 回退到了 API_KEY
```

### 优先级总结

| 配置来源 | 实际使用 | 为什么 |
|---------|---------|-------|
| 只有 config.toml | config.toml 的值 | 没有更高优先级来源 |
| config.toml + 环境变量 | 环境变量 | 环境变量 > config.toml |
| 三者都有 + 命令行前缀 | 命令行前缀 | 前缀 > 环境变量 > config.toml |
| 什么都没设 | `openrouter` | 代码硬编码默认值 |

> **前端类比（CSS 优先级）：** 命令行前缀 = `!important`，环境变量 = inline style，config.toml = class，默认值 = 浏览器默认。

## 配置项速查表

| 优先级 | 配置项 | 推荐位置 | 说明 |
|--------|-------|---------|------|
| ⭐⭐⭐ | API Key | 环境变量 | 敏感信息，不写入文件 |
| ⭐⭐⭐ | Provider | config.toml | 不常变，持久化 |
| ⭐⭐⭐ | Model | config.toml | 不常变，持久化 |
| ⭐⭐ | Temperature | config.toml | 个人偏好 |
| ⭐⭐ | Gateway Port | config.toml | 端口冲突时改 |
| ⭐ | Proxy | 环境变量 | 网络环境相关 |
| ⭐ | Memory Backend | config.toml | sqlite 默认即可 |

## 常见配置问题排查

| # | 现象 | 解决方案 |
|---|------|---------|
| 1 | `API key not configured` | `export ZEROCLAW_API_KEY="sk-..."` |
| 2 | 改了 config.toml 但不生效 | `env \| grep ZEROCLAW` 检查环境变量是否覆盖了，`unset` 掉 |
| 3 | `Provider unreachable` | 设置 `ZEROCLAW_PROXY_ENABLED=true` + `ZEROCLAW_HTTPS_PROXY` |
| 4 | `TOML parse error` | 字符串要双引号、数字不加引号、布尔值小写 |
| 5 | `Model not found` | 确认 Provider 和模型对应（OpenRouter 用 `anthropic/claude-sonnet-4-6`，直连用 `claude-sonnet-4-6`） |
| 6 | 代理设了但超时 | 检查 `ZEROCLAW_PROXY_ENABLED=true`（总开关） |
| 7 | Ollama `connection refused` | `ollama serve` 启动服务 |
| 8 | 环境变量重开终端就没了 | 写入 `~/.zshrc` 或 `~/.bashrc` |

### 配置排查三板斧

```bash
zeroclaw doctor                    # 看 ZeroClaw 认为的当前配置
env | grep ZEROCLAW                # 看环境变量是否意外覆盖
cat ~/.zeroclaw/config.toml        # 看配置文件内容
```
