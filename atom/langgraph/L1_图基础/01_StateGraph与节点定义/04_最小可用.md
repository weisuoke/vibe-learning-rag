## 4.1 创建 StateGraph 实例

### 概念说明

StateGraph 是 LangGraph 的核心类，采用 **Builder 模式**设计。你需要先创建一个 builder 实例，然后逐步添加节点和边，最后编译成可执行的图。

**关键点**：
- StateGraph 本身不可执行，必须先编译
- 创建时需要传入 State Schema（状态结构定义）
- 支持泛型设计，提供类型安全

**来源**：[源码分析 - state.py:112-127]

### 代码示例

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph

# 1. 定义状态结构
class State(TypedDict):
    message: str
    count: int

# 2. 创建 StateGraph 实例
builder = StateGraph(State)

# 此时 builder 是一个构建器，还不能执行
# 需要添加节点、边，然后编译
```

**来源**：[Context7 文档 - use-graph-api]

### 实际应用场景

- **文档问答系统**：State 包含 `query`, `documents`, `answer`
- **多步推理任务**：State 包含 `input`, `reasoning_steps`, `final_answer`
- **审批流程**：State 包含 `request`, `approvals`, `status`

## 4.2 定义 State Schema

### 概念说明

State Schema 使用 **TypedDict** 定义状态结构，是图中所有节点共享的数据容器。

**关键特性**：
- 使用 `Annotated[type, reducer]` 添加 reducer 函数
- Reducer 函数控制状态更新策略（如追加、覆盖、合并）
- 节点函数只需返回部分状态更新，框架自动合并

**来源**：[源码分析 - state.py:197-250] + [Context7 文档 - use-graph-api]

### 代码示例

```python
import operator
from typing import Annotated
from typing_extensions import TypedDict, NotRequired

# 示例 1：基础 State（默认覆盖策略）
class BasicState(TypedDict):
    message: str
    count: int

# 示例 2：带 Reducer 的 State（追加策略）
class AppendState(TypedDict):
    # operator.add 使列表变为 append-only
    messages: Annotated[list[str], operator.add]
    count: int

# 示例 3：可选字段
class OptionalState(TypedDict):
    topic: NotRequired[str]  # 可选字段
    joke: NotRequired[str]   # 可选字段

# 节点函数示例
def node_a(state: AppendState):
    # 只返回部分状态更新
    return {"messages": ["A"]}  # 会追加到 messages 列表

def node_b(state: AppendState):
    return {"messages": ["B"], "count": 1}  # 同时更新多个字段
```

**来源**：[Context7 文档 - use-graph-api]

### 实际应用场景

- **对话历史管理**：使用 `Annotated[list, operator.add]` 追加消息
- **多代理协作**：每个代理更新自己负责的状态字段
- **审批流程**：使用 `NotRequired` 标记可选的审批意见

**类比**：
- **前端类比**：TypedDict = TypeScript 接口定义，Reducer = Redux reducer
- **日常类比**：State = 工作台状态，Reducer = 状态更新规则（覆盖 vs 追加）

## 4.3 添加节点函数

### 概念说明

节点函数是图中的处理单元，接受当前状态作为输入，返回部分状态更新。

**节点函数规范**：
- **输入**：接受完整的 State（TypedDict）
- **输出**：返回部分状态更新（dict）
- **命名**：可以自动推断函数名，或显式指定
- **类型**：支持普通函数、异步函数、Runnable

**来源**：[源码分析 - _node.py:16-93] + [Context7 文档 - use-time-travel]

### 代码示例

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph

class State(TypedDict):
    input: str
    output: str

# 方式 1：自动推断节点名称
def process_input(state: State):
    """处理输入"""
    processed = state["input"].upper()
    return {"output": processed}

# 方式 2：显式指定节点名称
def generate_response(state: State):
    """生成响应"""
    response = f"Response: {state['output']}"
    return {"output": response}

# 创建图并添加节点
builder = StateGraph(State)
builder.add_node(process_input)  # 节点名称自动为 "process_input"
builder.add_node("generator", generate_response)  # 显式指定名称为 "generator"
```

**来源**：[源码分析 - state.py:328-349]

### 实际应用场景

- **RAG 系统**：`retrieve_node`, `rerank_node`, `generate_node`
- **多步推理**：`analyze_node`, `reason_node`, `conclude_node`
- **审批流程**：`submit_node`, `review_node`, `approve_node`

**最佳实践**：
- 每个节点职责单一（单一职责原则）
- 节点函数保持简洁（< 50 行）
- 使用描述性的函数名

**来源**：[Reddit 社区 - 最佳实践]

## 4.4 连接节点（边）

### 概念说明

边（Edge）定义节点之间的转换关系，控制状态流转的路径。

**边的类型**：
- **普通边**：`add_edge(start, end)` - 无条件转换
- **START 边**：`add_edge(START, node)` - 图的入口点
- **END 边**：`add_edge(node, END)` - 图的出口点

**来源**：[源码分析 - state.py:785-837] + [Context7 文档 - use-graph-api]

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

class State(TypedDict):
    message: str

def step_1(state: State):
    return {"message": "Step 1 done"}

def step_2(state: State):
    return {"message": state["message"] + " -> Step 2 done"}

def step_3(state: State):
    return {"message": state["message"] + " -> Step 3 done"}

# 创建图
builder = StateGraph(State)

# 添加节点
builder.add_node(step_1)
builder.add_node(step_2)
builder.add_node(step_3)

# 连接节点（顺序执行）
builder.add_edge(START, "step_1")    # 入口点
builder.add_edge("step_1", "step_2") # step_1 -> step_2
builder.add_edge("step_2", "step_3") # step_2 -> step_3
builder.add_edge("step_3", END)      # 出口点
```

**来源**：[Context7 文档 - use-graph-api]

### 实际应用场景

- **线性流程**：START -> 节点A -> 节点B -> 节点C -> END
- **分支流程**：使用条件边（下一节介绍）
- **循环流程**：节点A -> 节点B -> 节点A（需要终止条件）

**类比**：
- **前端类比**：边 = React Router 路由规则
- **日常类比**：边 = 工厂流水线的传送带

## 4.5 编译并执行图

### 概念说明

编译（compile）是将 StateGraph builder 转换为可执行图的必需步骤。编译后的图是 **Pregel 实例**，支持 `invoke`, `stream`, `ainvoke` 等执行方法。

**关键点**：
- 编译时会验证图结构（检查节点连接、循环等）
- 编译后才能执行
- 可以配置 checkpointer（持久化）、interrupt（中断点）等

**来源**：[源码分析 - state.py:1035-1153] + [Context7 文档 - pregel]

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

# 1. 定义 State
class State(TypedDict):
    input: str
    output: str

# 2. 定义节点函数
def process(state: State):
    return {"output": state["input"].upper()}

# 3. 构建图
builder = StateGraph(State)
builder.add_node(process)
builder.add_edge(START, "process")
builder.add_edge("process", END)

# 4. 编译图
graph = builder.compile()

# 5. 执行图
result = graph.invoke({"input": "hello world"})
print(result)
# 输出: {'input': 'hello world', 'output': 'HELLO WORLD'}
```

**来源**：[Context7 文档 - use-graph-api]

### 完整可运行示例

```python
"""
最小可用 StateGraph 示例
演示：创建、添加节点、编译、执行
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# ===== 1. 定义 State Schema =====
class JokeState(TypedDict):
    topic: str
    joke: str

# ===== 2. 定义节点函数 =====
def generate_topic(state: JokeState):
    """生成笑话主题"""
    return {"topic": "程序员"}

def write_joke(state: JokeState):
    """根据主题写笑话"""
    topic = state["topic"]
    joke = f"为什么{topic}喜欢黑色？因为 RGB(0,0,0) 最省电！"
    return {"joke": joke}

# ===== 3. 构建图 =====
builder = StateGraph(JokeState)

# 添加节点
builder.add_node("generate_topic", generate_topic)
builder.add_node("write_joke", write_joke)

# 连接节点
builder.add_edge(START, "generate_topic")
builder.add_edge("generate_topic", "write_joke")
builder.add_edge("write_joke", END)

# ===== 4. 编译图 =====
graph = builder.compile()

# ===== 5. 执行图 =====
result = graph.invoke({"topic": "", "joke": ""})

print("=== 执行结果 ===")
print(f"主题: {result['topic']}")
print(f"笑话: {result['joke']}")
```

**运行输出**：
```
=== 执行结果 ===
主题: 程序员
笑话: 为什么程序员喜欢黑色？因为 RGB(0,0,0) 最省电！
```

**来源**：[Context7 文档 - use-time-travel] + [GitHub 教程]

### 实际应用场景

- **文档问答**：加载文档 -> 检索 -> 生成答案
- **内容审核**：提交内容 -> 自动审核 -> 人工复审 -> 发布

## 这些知识足以做什么？

掌握以上 5 个核心操作，你可以：

### ✅ 构建基础工作流

- **顺序执行流程**：多个节点按顺序执行
- **状态传递**：节点间通过 State 共享数据
- **部分更新**：节点只需返回需要更新的字段

### ✅ 实现常见应用场景

- **RAG 问答系统**：检索 -> 重排序 -> 生成答案
- **多步推理任务**：分析 -> 推理 -> 总结

### ✅ 为后续学习打基础

- **条件路由**：根据状态动态选择下一个节点（下一节）
- **循环执行**：节点可以循环执行直到满足条件
- **并行执行**：多个节点同时执行
- **持久化**：使用 Checkpointer 保存状态
- **人机循环**：在关键节点等待人工输入

## 最小可用代码模板

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义 State
class State(TypedDict):
    # 在这里定义你的状态字段
    input: str
    output: str

# 2. 定义节点函数
def my_node(state: State):
    # 在这里处理状态
    return {"output": state["input"]}

# 3. 构建图
builder = StateGraph(State)
builder.add_node(my_node)
builder.add_edge(START, "my_node")
builder.add_edge("my_node", END)

# 4. 编译并执行
graph = builder.compile()
result = graph.invoke({"input": "hello", "output": ""})
print(result)
```

**使用建议**：
1. 复制这个模板作为起点
2. 根据你的需求修改 State 定义
3. 添加你的节点函数
4. 连接节点形成工作流
5. 编译并测试

## 常见错误与解决

### 错误 1：忘记编译

```python
# ❌ 错误：直接调用 builder
builder.invoke({"input": "hello"})
# AttributeError: 'StateGraph' object has no attribute 'invoke'

# ✅ 正确：先编译再调用
graph = builder.compile()
graph.invoke({"input": "hello"})
```

### 错误 2：节点返回完整 State

```python
# ❌ 错误：返回完整 State
def my_node(state: State):
    return State(input=state["input"], output="result")

# ✅ 正确：只返回需要更新的字段
def my_node(state: State):
    return {"output": "result"}
```

### 错误 3：START/END 使用错误

```python
# ❌ 错误：START 作为终点
builder.add_edge("my_node", START)

# ❌ 错误：END 作为起点
builder.add_edge(END, "my_node")

# ✅ 正确：START 只能作为起点，END 只能作为终点
builder.add_edge(START, "my_node")
builder.add_edge("my_node", END)
```
