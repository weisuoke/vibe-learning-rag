## 二、核心概念预览

### 2.1 三层架构

OpenClaw 的开发环境可以分为三个层次：

```
┌─────────────────────────────────────┐
│     依赖管理层（pnpm workspace）      │
│  - package.json 依赖结构             │
│  - pnpm-lock.yaml 版本锁定           │
│  - workspace 包管理                  │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│     构建工具链层（TypeScript）        │
│  - tsconfig.json 编译配置            │
│  - tsdown 快速编译器                 │
│  - 构建脚本和输出管理                │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│     源码结构层（Monorepo）           │
│  - src/ 核心代码                     │
│  - ui/ 用户界面                      │
│  - extensions/ 扩展插件              │
└─────────────────────────────────────┘
```

### 2.2 关键技术栈

| 技术 | 版本 | 用途 |
|------|------|------|
| Node.js | 22.12.0+ | 运行时环境 |
| pnpm | 10.23.0 | 包管理器 |
| TypeScript | 5.9.3 | 类型系统 |
| tsdown | 最新 | 编译器 |

## 三、实战场景概览

### 3.1 基础编译流程

**场景描述**: 从零开始搭建 OpenClaw 开发环境

**核心步骤**:
```bash
# 1. 克隆仓库
git clone https://github.com/openclaw/openclaw.git

# 2. 安装依赖
pnpm install

# 3. 构建项目
pnpm ui:build && pnpm build

# 4. 运行
pnpm gateway:watch
```

**预期结果**: OpenClaw 成功启动，可以进行开发

---

### 3.2 开发模式配置

**场景描述**: 配置自动重载的开发环境

**核心配置**:
- watch 模式：代码变更自动重新编译
- 热重载：无需重启即可看到变更
- 增量编译：只编译变更的文件

**关键文件**: `scripts/run-node.mjs`

---

### 3.3 调试配置与技巧

**场景描述**: 配置 VSCode 调试环境

**核心配置**:
```json
{
  "type": "node",
  "request": "launch",
  "name": "Debug OpenClaw",
  "runtimeExecutable": "pnpm",
  "runtimeArgs": ["dev"]
}
```

---

### 3.4 故障排查实战

**场景描述**: 解决常见的开发环境问题

**常见问题**:
1. pnpm install 失败 → 检查 Node.js 版本
2. 编译错误 → 检查 TypeScript 配置
3. 运行时错误 → 检查环境变量

## 六、核心价值

### 6.1 为什么要从源码搭建？

**对比表**:

| 维度 | npm 安装 | 源码搭建 |
|------|----------|----------|
| 安装速度 | ⚡ 快 | 🐢 慢 |
| 定制能力 | ❌ 无 | ✅ 完全 |
| 调试能力 | ⚠️ 有限 | ✅ 完整 |
| 学习价值 | ⭐ 低 | ⭐⭐⭐ 高 |
| 贡献代码 | ❌ 不可 | ✅ 可以 |

**适用场景**:
- ✅ 需要修改 OpenClaw 源码
- ✅ 想要贡献代码到 OpenClaw
- ✅ 需要深入理解 OpenClaw 内部实现
- ✅ 需要调试 OpenClaw 的问题
- ❌ 只是想快速使用 OpenClaw（用 npm 安装）

### 6.2 学习收益

完成本知识点学习后，你将获得：

1. **技术能力提升**：
   - 掌握 monorepo 项目的开发流程
   - 理解 TypeScript 项目的构建过程
   - 熟悉现代前端工具链

2. **实战经验积累**：
   - 能够独立搭建复杂项目的开发环境
   - 能够排查和解决常见的开发环境问题
   - 能够配置高效的开发工作流

3. **职业发展助力**：
   - 为开源贡献做好准备
   - 提升简历竞争力
   - 积累大型项目经验
