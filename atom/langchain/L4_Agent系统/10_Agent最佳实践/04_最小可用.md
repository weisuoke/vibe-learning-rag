# Agent最佳实践 - 最小可用

> 用最少的代码，搭建一个具备基本可靠性和安全性的生产级 Agent。

---

## 最小可用 = 3 个 Middleware

一个"能上线"的 Agent 至少需要三个 Middleware：

```python
from langchain.agents import create_agent
from langchain.agents.middleware import (
    ModelCallLimitMiddleware,
    ModelRetryMiddleware,
    ModelFallbackMiddleware,
)

# 最小可用的生产级 Agent
agent = create_agent(
    "gpt-4.1",
    tools=[search_tool, calculator_tool],
    system_prompt="你是一个有用的助手。",
    middleware=[
        ModelCallLimitMiddleware(run_limit=25),     # 防无限循环
        ModelRetryMiddleware(max_retries=3),          # 防瞬时故障
        ModelFallbackMiddleware(fallbacks=["gpt-4.1-mini"]),  # 防服务宕机
    ],
)

# 使用
result = agent.invoke({
    "messages": [{"role": "user", "content": "今天北京天气怎么样？"}]
})
```

---

## 为什么是这三个？

| Middleware | 防御什么 | 没有会怎样 |
|-----------|---------|-----------|
| `ModelCallLimitMiddleware` | 无限循环 | Agent 死循环，Token 费用爆炸 |
| `ModelRetryMiddleware` | API 超时/限流 | 偶尔的网络抖动就让 Agent 崩溃 |
| `ModelFallbackMiddleware` | 模型宕机 | 供应商维护时整个系统不可用 |

## 对比：没有 vs 有最佳实践

### 没有最佳实践（原型 Agent）

```python
# 能跑，但不能上线
from langchain.agents import create_agent

agent = create_agent(
    "gpt-4.1",
    tools=[search_tool],
)
# 问题：
# 1. LLM 陷入循环 → 烧钱
# 2. API 超时 → 用户看到错误
# 3. 模型宕机 → 系统完全不可用
```

### 有最佳实践（最小可用）

```python
from langchain.agents import create_agent
from langchain.agents.middleware import (
    ModelCallLimitMiddleware,
    ModelRetryMiddleware,
    ModelFallbackMiddleware,
)

agent = create_agent(
    "gpt-4.1",
    tools=[search_tool],
    middleware=[
        ModelCallLimitMiddleware(run_limit=25),
        ModelRetryMiddleware(max_retries=3),
        ModelFallbackMiddleware(fallbacks=["gpt-4.1-mini"]),
    ],
)
# 解决了基本的可靠性问题
```

### 完整版（生产级）

```python
from langchain.agents import create_agent
from langchain.agents.middleware import (
    ModelCallLimitMiddleware,
    ToolCallLimitMiddleware,
    ModelRetryMiddleware,
    ModelFallbackMiddleware,
    ToolRetryMiddleware,
    PIIMiddleware,
    HumanInTheLoopMiddleware,
    SummarizationMiddleware,
)

agent = create_agent(
    "gpt-4.1",
    tools=[search_tool, db_query, send_email],
    middleware=[
        # 限制保护
        ModelCallLimitMiddleware(run_limit=25),
        ToolCallLimitMiddleware(allowed={"search": 10, "db_query": 5}),
        # 可靠性
        ModelRetryMiddleware(max_retries=3, backoff_factor=2.0),
        ModelFallbackMiddleware(fallbacks=["gpt-4.1-mini", "claude-3.5-sonnet"]),
        ToolRetryMiddleware(max_retries=2, tools=["search"]),
        # 安全
        PIIMiddleware(strategy="mask", pii_types=["email", "phone"]),
        HumanInTheLoopMiddleware(interrupt_on={"send_email": {"always": True}}),
        # 上下文管理
        SummarizationMiddleware(trigger=("fraction", 0.6), keep=5),
    ],
)
```

## 渐进式增强路径

```
Level 0: 原型 Agent
  create_agent("gpt-4.1", tools=[...])
  → 能跑，但不可靠

Level 1: 最小可用（+3 行）
  + ModelCallLimitMiddleware     → 防循环
  + ModelRetryMiddleware         → 防故障
  + ModelFallbackMiddleware      → 防宕机
  → 基本可靠

Level 2: 安全加固（+2 行）
  + PIIMiddleware               → 防泄密
  + HumanInTheLoopMiddleware    → 防越权
  → 安全可靠

Level 3: 生产就绪（+2 行）
  + SummarizationMiddleware     → 防上下文溢出
  + ToolCallLimitMiddleware     → 精细控制
  → 生产级别
```

**核心理念：从 Level 0 到 Level 3，只是在 `middleware=[]` 列表中多加几行，不需要修改 Agent 核心逻辑。**
