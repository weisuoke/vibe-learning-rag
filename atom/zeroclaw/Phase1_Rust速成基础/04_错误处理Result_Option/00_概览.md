# 错误处理 Result/Option - 概览

> **一句话定位**：Rust 用 `Result<T,E>` 和 `Option<T>` 两个枚举替代了 try/catch 和 null，在编译期强制你处理每一个可能的错误和空值 -- ZeroClaw 的每个 trait 方法都返回 `anyhow::Result<T>`，不理解错误处理就看不懂源码。

---

## 1. 为什么学这个？

### 1.1 Rust 没有 try/catch

这是从 TypeScript 转到 Rust 最大的思维冲击之一：

```typescript
// TypeScript -- 异常是隐式的，编译器不管你处不处理
async function fetchData(url: string): Promise<string> {
  const res = await fetch(url);  // 可能抛异常
  return res.text();              // 可能抛异常
  // 编译器完全不会提醒你处理错误
}
```

```rust
// Rust -- 错误是显式的，编译器强制你处理
async fn fetch_data(url: &str) -> Result<String, reqwest::Error> {
    let res = reqwest::get(url).await?;  // ? 传播错误
    let text = res.text().await?;         // ? 传播错误
    Ok(text)  // 必须显式包装成功值
}
```

**核心区别**：TypeScript 的错误像地雷（运行时才爆炸），Rust 的错误像路标（编译时就告诉你前方有坑）。

### 1.2 ZeroClaw 全局使用 anyhow::Result

打开 ZeroClaw 任何一个 trait 文件，你会看到：

```rust
// src/tools/traits.rs
#[async_trait]
pub trait Tool: Send + Sync {
    async fn execute(&self, args: Value) -> anyhow::Result<ToolResult>;
    //                                      ^^^^^^^^^^^^^^^^^^^^^^^^
    //                                      每个方法都返回 Result
}

// src/memory/traits.rs
#[async_trait]
pub trait Memory: Send + Sync {
    async fn store(&self, entry: MemoryEntry) -> anyhow::Result<()>;
    async fn search(&self, query: &str, limit: usize) -> anyhow::Result<Vec<MemoryEntry>>;
}

// src/channels/traits.rs
#[async_trait]
pub trait Channel: Send + Sync {
    async fn receive(&mut self) -> anyhow::Result<ChannelMessage>;
    async fn send(&self, message: ChannelMessage) -> anyhow::Result<()>;
}
```

[来源: reference/source_error_handling_01.md]

**不理解 `Result` 和 `?` 运算符 = 看不懂 ZeroClaw 90% 的代码。**

### 1.3 Rust 也没有 null/undefined

TypeScript 开发者的另一个痛点：

```typescript
// TypeScript -- null/undefined 是运行时炸弹
interface Config {
  apiKey?: string;       // 可能是 undefined
  baseUrl: string | null; // 可能是 null
}
// 忘记检查 → TypeError: Cannot read properties of undefined
```

```rust
// Rust -- Option<T> 在编译期强制你处理"没有值"的情况
struct Config {
    api_key: Option<String>,   // 明确表示"可能没有"
    base_url: String,          // 一定有值
}
// 不处理 Option → 编译不通过
```

## 5. ZeroClaw 中的错误处理

### 5.1 依赖配置

```toml
# Cargo.toml
[dependencies]
anyhow = "1.0"      # 应用层灵活错误处理
thiserror = "2.0"    # 库层自定义错误类型
```

[来源: reference/source_error_handling_01.md]

### 5.2 统一模式：所有 trait 方法返回 anyhow::Result<T>

ZeroClaw 的设计原则是 **trait 层统一使用 `anyhow::Result<T>`**，不在 trait 定义中使用自定义错误枚举：

```rust
// 所有 trait 方法的返回类型都是 anyhow::Result<T>
trait Tool:     async fn execute(&self, args: Value) -> anyhow::Result<ToolResult>;
trait Memory:   async fn store(&self, entry: MemoryEntry) -> anyhow::Result<()>;
trait Channel:  async fn receive(&mut self) -> anyhow::Result<ChannelMessage>;
trait Provider: async fn chat(&self, messages: Vec<Message>) -> anyhow::Result<Response>;
```

### 5.3 关键源文件与错误处理模式

| 源文件 | 主要错误处理模式 | 说明 |
|--------|-----------------|------|
| `src/tools/traits.rs` | `anyhow::Result<ToolResult>` | Tool trait 定义 |
| `src/tools/shell.rs` | Option 组合子 + 嵌套 match | 参数提取 + 超时处理 |
| `src/tools/web_fetch.rs` | `bail!` + match Result | URL 验证 + 错误包装 |
| `src/memory/sqlite.rs` | `.context()` 错误上下文 | 数据库操作上下文注入 |
| `src/security/secrets.rs` | `ensure!` + `map_err` | 加密操作验证 |
| `src/config/schema.rs` | 大量 `Option<T>` 字段 | 可选配置项 |
| `src/gateway/api.rs` | 自定义 Result 类型 | HTTP 响应错误映射 |

[来源: reference/source_error_handling_01.md]

### 5.4 ZeroClaw 错误处理四原则

1. **零 panic**：生产代码不使用 `unwrap()`，仅测试中使用
2. **安全优先**：错误信息不泄露敏感数据（密钥、路径等）
3. **上下文丰富**：每个错误都附带描述性上下文（`.context("...")`）
4. **Trait 一致性**：所有 trait 方法统一返回 `anyhow::Result<T>`

[来源: reference/source_error_handling_01.md]

## 6. 与 TypeScript 的核心区别

### 6.1 对比总览

| 特性 | TypeScript | Rust |
|------|-----------|------|
| **错误机制** | try/catch 异常（隐式） | `Result<T,E>` 枚举（显式） |
| **空值处理** | `null` / `undefined`（运行时炸弹） | `Option<T>`（编译时强制处理） |
| **编译检查** | 不强制处理异常 | 编译器强制处理每个 Result/Option |
| **错误传播** | `throw` 自动冒泡 | `?` 运算符显式传播 |
| **错误类型** | `any`（通常无类型） | 泛型 `E`（类型安全） |
| **空值安全** | `?.` optional chaining | `Option` 的 `map` / `and_then` |
| **错误上下文** | 手动拼接 message | `.context("msg")` 自动链式 |
| **panic** | `throw new Error()` | `panic!()` -- 仅用于不可恢复的 bug |

### 6.2 思维转变

| TypeScript 思维 | Rust 思维 |
|----------------|-----------|
| "错误会自动冒泡到 catch" | "每个错误必须显式用 `?` 传播" |
| "null 检查靠自觉" | "Option 不处理就编译不过" |
| "catch 里 error 是 any" | "Err(e) 的 e 有明确类型" |
| "try/catch 包一大段代码" | "每行可能出错的代码都用 `?`" |
| "错误处理是可选的" | "错误处理是强制的" |

## 7. 快速参考

### 7.1 Result 速查

```rust
// 定义
enum Result<T, E> {
    Ok(T),    // 成功，携带值
    Err(E),   // 失败，携带错误
}

// 创建
let ok: Result<i32, String> = Ok(42);
let err: Result<i32, String> = Err("failed".to_string());

// 模式匹配
match result {
    Ok(value) => println!("成功: {}", value),
    Err(e) => println!("失败: {}", e),
}

// ? 运算符（在返回 Result 的函数中使用）
let value = some_operation()?;  // Err 时自动返回，Ok 时解包
```

### 7.2 Option 速查

```rust
// 定义
enum Option<T> {
    Some(T),  // 有值
    None,     // 没有值
}

// 创建
let some: Option<i32> = Some(42);
let none: Option<i32> = None;

// 安全取值
let value = opt.unwrap_or(0);           // None 时用默认值
let value = opt.unwrap_or_default();    // None 时用类型默认值
let value = opt.ok_or(anyhow!("missing"))?;  // None 转 Err
```

### 7.3 anyhow 速查

```rust
use anyhow::{Result, bail, ensure, Context, anyhow};

// 函数签名
fn do_something() -> Result<String> { ... }  // anyhow::Result<T>

// 提前返回错误
bail!("something went wrong: {}", detail);

// 条件断言
ensure!(!input.is_empty(), "input cannot be empty");

// 添加上下文
let data = fs::read_to_string(path)
    .context("failed to read config file")?;

// 创建临时错误
return Err(anyhow!("invalid state: {}", state));
```

### 7.4 ZeroClaw 常见模式速查

```rust
// 模式 A: Option 组合子提取参数（最常见）
let command = args.get("command")
    .and_then(|v| v.as_str())
    .ok_or_else(|| anyhow!("Missing 'command' parameter"))?;

// 模式 B: .context() 添加错误上下文
Connection::open(&path).context("SQLite failed to open database")?;

// 模式 C: bail! 验证输入
if url.is_empty() {
    bail!("URL cannot be empty");
}

// 模式 D: ensure! 条件断言
ensure!(blob.len() > NONCE_LEN, "Encrypted value too short");

// 模式 E: map_err 转换错误类型
cipher.encrypt(&nonce, data)
    .map_err(|e| anyhow!("Encryption failed: {e}"))?;

// 模式 F: 嵌套 match 处理超时
match tokio::time::timeout(duration, operation).await {
    Ok(Ok(result)) => { /* 成功 */ },
    Ok(Err(e))     => { /* 操作失败 */ },
    Err(_)         => { /* 超时 */ },
}
```
