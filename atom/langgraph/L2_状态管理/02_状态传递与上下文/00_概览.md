## 核心价值

### 为什么这个知识点重要？

在 LangGraph 中，**状态传递与上下文** 是构建复杂 AI 工作流的基础设施。它解决了以下核心问题：

1. **节点间数据共享**：多个节点如何安全、高效地共享数据？
2. **配置传递**：如何在不污染状态的情况下传递运行时配置？
3. **状态流转控制**：如何精确控制状态在图中的流转路径？
4. **类型安全**：如何在编译时检查状态结构的正确性？
5. **可追溯性**：如何追踪状态的变化历史？

### 实际应用场景

- **多 Agent 协作系统**：多个 Agent 共享任务状态和中间结果
- **RAG 系统**：检索、重排、生成节点间传递文档和查询
- **复杂决策流**：多步推理中保持上下文连贯性
- **人机协作**：在人工审核节点间保持状态一致性

## 状态传递的完整流程

### 从源码角度理解

根据 `reference/source_状态传递_01.md` 的源码分析，状态传递的完整流程如下：

```python
# 1. StateGraph 初始化时创建 Channels
class StateGraph:
    channels: dict[str, BaseChannel]  # 状态存储容器

# 2. PregelNode 声明输入输出
class PregelNode:
    channels: str | list[str]  # 输入声明
    triggers: list[str]        # 触发条件
    writers: list[Runnable]    # 输出声明
    bound: Runnable            # 节点逻辑

# 3. ChannelRead 读取状态
class ChannelRead:
    @staticmethod
    def do_read(config: RunnableConfig, select: str, fresh: bool):
        read: READ_TYPE = config[CONF][CONFIG_KEY_READ]
        return read(select, fresh)

# 4. 节点执行逻辑
def my_node(state: State) -> dict:
    # 处理状态
    return {"key": "value"}  # 返回部分更新

# 5. ChannelWrite 写入状态
class ChannelWrite:
    @staticmethod
    def do_write(config: RunnableConfig, writes: Sequence):
        write: TYPE_SEND = config[CONF][CONFIG_KEY_SEND]
        write(_assemble_writes(writes))

# 6. Channel 更新状态
class BaseChannel:
    def update(self, values: Sequence[Update]) -> bool:
        # 更新逻辑（由具体实现决定）
        pass
```

### 状态流转的时序图

```
用户调用 graph.invoke(initial_state, context=...)
    ↓
Pregel 引擎初始化 RunnableConfig
    ├── 注入 CONFIG_KEY_READ → ChannelRead.do_read
    ├── 注入 CONFIG_KEY_SEND → ChannelWrite.do_write
    └── 注入 CONFIG_KEY_RUNTIME → Runtime Context
    ↓
触发 Node_1 执行
    ├── ChannelRead 读取 channels 声明的状态
    ├── 节点函数执行（state → partial_update）
    └── ChannelWrite 写入 partial_update
    ↓
Channel 更新状态（通过 Reducer 合并）
    ↓
检查 triggers，触发下一个节点
    ↓
重复上述流程...
    ↓
到达 END 节点，返回最终状态
```

## State vs Context：核心区别

### 概念对比

| 维度 | State（状态） | Context（上下文） |
|------|--------------|------------------|
| **可变性** | 可变（Mutable） | 不可变（Immutable） |
| **传递方式** | Channel 机制 | RunnableConfig |
| **更新方式** | 节点返回部分更新 | 运行时传入，不可修改 |
| **典型用途** | 消息历史、中间结果、工作流状态 | 用户 ID、模型配置、数据库连接 |
| **生命周期** | 随执行不断变化 | 整个执行期间保持不变 |
| **类型定义** | TypedDict | dataclass 或 TypedDict |
| **存储位置** | StateGraph.channels | RunnableConfig.CONF |
| **访问方式** | 节点参数 `state: State` | 节点参数 `runtime: Runtime[Context]` |

### 代码示例对比

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph
from langgraph.runtime import Runtime
from dataclasses import dataclass

# State：可变数据
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]  # 会不断累积
    current_step: str                        # 会被覆盖更新
    results: list                            # 会不断追加

# Context：不可变配置
@dataclass
class AgentContext:
    user_id: str          # 运行时传入，不会改变
    model_name: str       # 运行时传入，不会改变
    temperature: float    # 运行时传入，不会改变

# 节点函数
def process_node(state: AgentState, runtime: Runtime[AgentContext]) -> dict:
    # 读取 state（可变）
    messages = state["messages"]

    # 读取 context（不可变）
    user_id = runtime.context.user_id
    model = runtime.context.model_name

    # 返回 state 更新
    return {
        "current_step": "processing",
        "results": [{"user": user_id, "model": model}]
    }
```

## 设计模式与架构决策

### 核心设计模式

根据 `reference/source_状态传递_01.md` 的分析，LangGraph 使用了以下设计模式：

1. **依赖注入模式**
   - 通过 RunnableConfig 注入读写函数
   - 节点不需要知道状态如何存储

2. **策略模式**
   - Channel 的不同实现（LastValue, BinaryOperator 等）
   - 不同的状态更新策略

3. **命令模式**
   - ChannelWrite 封装写入操作
   - 支持批量写入和延迟执行

4. **观察者模式**
   - triggers 机制触发节点执行
   - 状态变化自动通知依赖节点

### 架构优势

1. **解耦性**：节点逻辑与状态存储分离
2. **可测试性**：可以独立测试节点函数
3. **可扩展性**：可以自定义 Channel 实现
4. **类型安全**：TypedDict 提供编译时检查
5. **可追溯性**：所有状态变化都可追踪

## 快速参考

### 核心 API 速查

```python
# 1. 定义状态 Schema
from typing import TypedDict, Annotated
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]
    data: dict

# 2. 定义上下文 Schema
from dataclasses import dataclass

@dataclass
class Context:
    user_id: str
    config: dict

# 3. 创建图
from langgraph.graph import StateGraph

builder = StateGraph(State, context_schema=Context)

# 4. 定义节点（接收 state 和 runtime）
from langgraph.runtime import Runtime

def my_node(state: State, runtime: Runtime[Context]) -> dict:
    # 读取 state
    messages = state["messages"]

    # 读取 context
    user_id = runtime.context.user_id

    # 返回部分更新
    return {"data": {"user": user_id}}

# 5. 添加节点
builder.add_node("my_node", my_node)

# 6. 编译图
graph = builder.compile()

# 7. 执行图
result = graph.invoke(
    {"messages": []},  # 初始状态
    context=Context(user_id="123", config={})  # 运行时上下文
)
```
