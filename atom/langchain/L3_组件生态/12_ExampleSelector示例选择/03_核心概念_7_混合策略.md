# 核心概念7：混合策略（语义相似度 + 长度限制）

## 概念定义

**混合策略是将语义相似度选择和长度限制结合的示例选择方法，既保证选择最相关的示例，又控制 Prompt 总长度不超过模型限制。**

这是 2025-2026 年 LangChain 社区的新趋势，解决了单一策略的局限性。

### 为什么需要混合策略？

**单一策略的局限性：**

1. **纯语义相似度选择的问题**
   - 可能选择过多示例导致超过 token 限制
   - 不考虑示例长度，可能选择很长的示例
   - 在生产环境中容易导致 API 调用失败

2. **纯长度限制选择的问题**
   - 按顺序选择示例，不考虑相关性
   - 可能选择不相关的示例
   - Few-shot learning 效果差

**混合策略的优势：**
- ✅ 既保证相关性，又控制长度
- ✅ 避免 token 超限
- ✅ 提高实际应用效果
- ✅ 生产环境更稳定

---

## 核心原理

### 两阶段选择机制

```
输入查询
    ↓
【阶段1：语义相似度筛选】
    ↓
选择 top-k 最相似的示例（如 k=10）
    ↓
【阶段2：长度限制过滤】
    ↓
按长度过滤，确保总长度 ≤ max_length
    ↓
最终示例列表
```

**关键点：**
1. 先用语义相似度保证相关性
2. 再用长度限制保证不超限
3. 两者结合，取长补短

## 实现方式

### 方案1：手动组合（基础实现）

**思路：** 先用 `SemanticSimilarityExampleSelector` 选择，再手动过滤长度。

```python
from langchain_core.example_selectors import SemanticSimilarityExampleSelector
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings

# 1. 创建语义相似度选择器
semantic_selector = SemanticSimilarityExampleSelector.from_examples(
    examples=examples,
    embeddings=OpenAIEmbeddings(model="text-embedding-3-small"),
    vectorstore_cls=Chroma,
    k=10  # 先选择 10 个最相似的
)

# 2. 手动过滤长度
def select_with_length_limit(query: str, max_length: int = 2048):
    # 语义相似度选择
    selected = semantic_selector.select_examples({"query": query})
    
    # 长度过滤
    filtered = []
    total_length = 0
    
    for example in selected:
        example_text = f"{example['input']}\n{example['output']}"
        example_length = len(example_text.split())
        
        if total_length + example_length <= max_length:
            filtered.append(example)
            total_length += example_length
        else:
            break
    
    return filtered

# 使用
query = "如何实现用户认证？"
selected_examples = select_with_length_limit(query, max_length=2048)
print(f"选择了 {len(selected_examples)} 个示例，总长度: {sum(len(f'{e['input']}\n{e['output']}'.split()) for e in selected_examples)} 词")
```

**优点：**

- 完全控制逻辑
- 易于调试

**缺点：**

### 方案3：自定义选择器（高级）

**思路：** 继承 `BaseExampleSelector`，实现自定义逻辑。

```python
from langchain_core.example_selectors import BaseExampleSelector
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from typing import Dict, List

class HybridExampleSelector(BaseExampleSelector):
    """混合策略选择器：语义相似度 + 长度限制"""
    
    def __init__(
        self,
        examples: List[Dict],
        embeddings: OpenAIEmbeddings,
        k: int = 10,
        max_length: int = 2048
    ):
        self.examples = examples
        self.k = k
        self.max_length = max_length
        
        # 创建向量存储
        texts = [f"{ex['input']}\n{ex['output']}" for ex in examples]
        self.vectorstore = Chroma.from_texts(
            texts,
            embeddings,
            metadatas=examples
        )
    
    def add_example(self, example: Dict) -> None:
        """添加示例"""
        self.examples.append(example)
        text = f"{example['input']}\n{example['output']}"
        self.vectorstore.add_texts([text], metadatas=[example])
    
    def select_examples(self, input_variables: Dict) -> List[Dict]:
        """选择示例：语义相似度 + 长度限制"""
        query = input_variables.get("query", "")
        
        # 阶段1：语义相似度选择
        docs = self.vectorstore.similarity_search(query, k=self.k)
        candidates = [doc.metadata for doc in docs]
        
        # 阶段2：长度限制过滤
        selected = []
        total_length = 0
        
        for example in candidates:
            text = f"{example['input']}\n{example['output']}"
            length = len(text.split())
            
            if total_length + length <= self.max_length:
                selected.append(example)
                total_length += length
            else:
                break
        
        return selected

# 使用
selector = HybridExampleSelector(
    examples=examples,
    embeddings=OpenAIEmbeddings(model="text-embedding-3-small"),
    k=10,
    max_length=2048
)

selected = selector.select_examples({"query": "如何实现用户认证？"})
print(f"选择了 {len(selected)} 个示例")
```

**优点：**
- ✅ 完全自定义
- ✅ 灵活扩展
- ✅ 符合 LangChain 规范

**缺点：**
- 代码量大
- 需要维护

## 最佳实践

### 1. 参数配置建议

**语义相似度参数：**
- `k=5-10`：先选择 5-10 个候选示例
- `embeddings=text-embedding-3-small`：成本低、速度快

**长度限制参数：**
- `max_length=2048`：根据模型 context window 设置
- 预留空间给输入和输出（如 4096 context，设置 max_length=2048）

### 2. 性能优化

**缓存 Embeddings：**
```python
# 使用持久化向量存储
vectorstore = Chroma(
    persist_directory="./chroma_db",
    embedding_function=embeddings
)
```

**批量处理：**
```python
# 批量选择示例
queries = ["query1", "query2", "query3"]
results = [select_with_length_limit(q) for q in queries]
```

### 3. 常见陷阱

**陷阱1：k 值设置过小**
- ❌ 错误：k=2，长度限制 2048
- ✅ 正确：k=10，长度限制 2048
- 原因：k 太小可能选不到足够的示例

**陷阱2：长度计算不准确**
- ❌ 错误：按字符数计算
- ✅ 正确：按 token 数计算（使用 tiktoken）
- 原因：模型按 token 计费和限制

**陷阱3：不考虑输入长度**
- ❌ 错误：max_length 只考虑示例
- ✅ 正确：max_length 包含输入、示例、输出
- 原因：总长度可能超过 context window

function recommendProducts(userQuery, maxItems = 10, maxPrice = 1000) {
  // 阶段1：相关性筛选（类似语义相似度）
  const relevant = products.filter(p =>

### 类比2：混合策略 = 智能缓存

**前端类比：** 浏览器缓存策略

```javascript
// 前端：智能缓存（LRU + 大小限制）
class SmartCache {
  constructor(maxSize = 1024 * 1024) {  // 1MB
    this.cache = new Map();
    this.maxSize = maxSize;
    this.currentSize = 0;
  }
  
  set(key, value) {
    const size = value.length;
    
    // 阶段1：LRU 策略（类似语义相似度）
    // 保留最近使用的
    
    // 阶段2：大小限制（类似长度限制）
    while (this.currentSize + size > this.maxSize) {
      const oldest = this.cache.keys().next().value;
      this.delete(oldest);
    }
    
    this.cache.set(key, value);
    this.currentSize += size;
  }
}
```

2. **阶段2：空间限制**（长度限制）
   - 根据书架大小，决定放多少本
   - 确保不超过书架容量

**相似性：**
- 都是资源管理
- 先保证质量，再控制数量
