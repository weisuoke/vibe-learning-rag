## 1. 普通边的基础用法

### 核心概念

普通边用于建立固定的执行顺序，从一个节点直接连接到另一个节点。

### 基础语法

```python
graph.add_edge(start_key, end_key)
```

### 最小示例

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    value: int

def step1(state: State) -> dict:
    return {"value": state["value"] * 2}

def step2(state: State) -> dict:
    return {"value": state["value"] + 10}

# 构建图
graph = StateGraph(State)
graph.add_node("step1", step1)
graph.add_node("step2", step2)

# 添加普通边
graph.add_edge(START, "step1")
graph.add_edge("step1", "step2")
graph.add_edge("step2", END)

# 编译并执行
app = graph.compile()
result = app.invoke({"value": 5})
print(result)  # {'value': 20}
```

### 关键点

- 使用 `add_edge` 方法
- 参数：起始节点、结束节点
- 执行顺序固定，不会改变

## 2. 条件边的基础用法

### 核心概念

条件边用于根据状态值动态选择执行路径，实现分支决策。

### 基础语法

```python
graph.add_conditional_edges(
    source,           # 起始节点
    route_func,       # 路由函数
    path_map          # 路径映射
)
```

### 最小示例

```python
from typing import Literal

def route_by_value(state: State) -> Literal["high", "low"]:
    """根据值路由"""
    if state["value"] > 50:
        return "high"
    return "low"

def process_high(state: State) -> dict:
    return {"value": state["value"] * 2}

def process_low(state: State) -> dict:
    return {"value": state["value"] + 10}

# 构建图
graph = StateGraph(State)
graph.add_node("check", lambda s: s)
graph.add_node("high", process_high)
graph.add_node("low", process_low)

graph.add_edge(START, "check")
graph.add_conditional_edges(
    "check",
    route_by_value,
    {"high": "high", "low": "low"}
)
graph.add_edge("high", END)
graph.add_edge("low", END)

app = graph.compile()
```

### 关键点

- 使用 `add_conditional_edges` 方法
- 需要路由函数决定下一个节点
- 使用 `Literal` 类型提示明确返回值

[来源: reference/source_edges_02_add_conditional_edges.md]

## 3. 路由函数的基本结构

### 核心概念

路由函数接受状态作为输入，返回目标节点名称。

### 基础结构

```python
from typing import Literal

def route_func(state: State) -> Literal["node_a", "node_b"]:
    """路由函数
    
    Args:
        state: 当前状态
        
    Returns:
        目标节点名称
    """
    if state["condition"]:
        return "node_a"
    return "node_b"
```

### 关键点

1. **输入参数**：`state: State`
2. **返回类型**：`Literal[...]` 类型提示
3. **返回值**：字符串（节点名称）
4. **决策逻辑**：简单的条件判断

[来源: reference/context7_langgraph_01_edges_routing.md]

## 4. START 和 END 的使用

### 核心概念

- `START`：图的入口标记
- `END`：图的终止标记

### 基础用法

```python
from langgraph.graph import START, END

# START 作为起始节点
graph.add_edge(START, "first_node")

# END 作为结束节点
graph.add_edge("last_node", END)

# 条件边也可以路由到 END
def route_func(state: State) -> Literal["continue", END]:
    if state["done"]:
        return END
    return "continue"
```

### 关键点

- `START` 不能作为结束节点
- `END` 不能作为起始节点
- 条件边可以路由到 `END`

[来源: reference/source_edges_01_add_edge.md]

## 5. 一个完整的示例

### 场景：简单的审批流程

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict, Literal

class ApprovalState(TypedDict):
    request: str
    score: float
    status: str

def evaluate(state: ApprovalState) -> dict:
    """评估请求"""
    # 模拟评分
    score = 0.75
    return {"score": score}

def route_decision(state: ApprovalState) -> Literal["approve", "reject"]:
    """决策路由"""
    if state["score"] > 0.7:
        return "approve"
    return "reject"

def approve(state: ApprovalState) -> dict:
    """批准"""
    return {"status": "approved"}

def reject(state: ApprovalState) -> dict:
    """拒绝"""
    return {"status": "rejected"}

# 构建图
graph = StateGraph(ApprovalState)

# 添加节点
graph.add_node("evaluate", evaluate)
graph.add_node("approve", approve)
graph.add_node("reject", reject)

# 添加边
graph.add_edge(START, "evaluate")

# 条件边：根据评分决定
graph.add_conditional_edges(
    "evaluate",
    route_decision,
    {
        "approve": "approve",
        "reject": "reject"
    }
)

graph.add_edge("approve", END)
graph.add_edge("reject", END)

# 编译并执行
app = graph.compile()
result = app.invoke({
    "request": "申请内容",
    "score": 0.0,
    "status": ""
})
print(result)
# {'request': '申请内容', 'score': 0.75, 'status': 'approved'}
```

### 流程图

```
START → evaluate → (score > 0.7) → approve → END
                 → (score ≤ 0.7) → reject → END
```

## 快速参考

### 普通边 vs 条件边

| 特性 | 普通边 | 条件边 |
|------|--------|--------|
| 方法 | `add_edge` | `add_conditional_edges` |
| 路由 | 固定 | 动态 |
| 参数 | 起始节点、结束节点 | 起始节点、路由函数、路径映射 |
| 使用场景 | 线性流程 | 分支决策 |

### 常用模式

#### 模式 1：线性流程

```python
graph.add_edge(START, "step1")
graph.add_edge("step1", "step2")
graph.add_edge("step2", END)
```

#### 模式 2：简单分支

```python
def route(state):
    return "a" if state["flag"] else "b"

graph.add_conditional_edges("check", route, {"a": "node_a", "b": "node_b"})
```

#### 模式 3：多路由

```python
def route(state) -> Literal["a", "b", "c"]:
    if state["type"] == "A":
        return "a"
    elif state["type"] == "B":
        return "b"
    return "c"

graph.add_conditional_edges("router", route, {"a": "node_a", "b": "node_b", "c": "node_c"})
```

## 常见错误

### 错误 1：忘记添加节点

```python
# ❌ 错误
graph.add_edge("node1", "node2")  # 节点未添加

# ✅ 正确
graph.add_node("node1", func1)
graph.add_node("node2", func2)
graph.add_edge("node1", "node2")
```

### 错误 2：路由函数无类型提示

```python
# ❌ 错误
def route(state):
    return "a"

# ✅ 正确
def route(state: State) -> Literal["a", "b"]:
    return "a"
```

### 错误 3：END 作为起始节点

```python
# ❌ 错误
graph.add_edge(END, "node1")

# ✅ 正确
graph.add_edge("node1", END)
```
