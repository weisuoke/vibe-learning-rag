# 核心概念 6：多分支路由

> 本文档详细讲解 LangGraph 条件边中的多分支路由机制

---

## 什么是多分支路由？

**多分支路由是指路由函数可以返回多个目标节点，图会并行执行这些节点，然后在汇聚节点等待所有节点完成。**

在 LangGraph 中，条件边不仅可以路由到单个节点，还可以路由到多个节点，实现并行执行和汇聚的模式。

[来源: reference/context7_langgraph_01_edges_routing.md]

---

## 多分支路由的核心特性

### 1. 返回多个目标节点

路由函数可以返回一个节点名称列表，而不是单个节点名称。

```python
from langgraph.graph import StateGraph, START, END
from typing import Sequence
from typing_extensions import TypedDict

class State(TypedDict):
    value: int
    results: list[str]

def multi_route(state: State) -> Sequence[str]:
    """返回多个目标节点"""
    targets = []

    if state["value"] > 50:
        targets.append("high_handler")

    if state["value"] % 2 == 0:
        targets.append("even_handler")

    if state["value"] < 100:
        targets.append("low_handler")

    return targets  # 返回多个节点名称

graph = StateGraph(State)
graph.add_node("router", lambda s: s)
graph.add_node("high_handler", lambda s: {"results": s.get("results", []) + ["high"]})
graph.add_node("even_handler", lambda s: {"results": s.get("results", []) + ["even"]})
graph.add_node("low_handler", lambda s: {"results": s.get("results", []) + ["low"]})

graph.add_edge(START, "router")
graph.add_conditional_edges("router", multi_route)
```

**关键点**：
1. **返回类型**：`Sequence[str]` 表示返回多个节点名称
2. **动态决策**：根据状态值动态决定执行哪些节点
3. **并行执行**：所有返回的节点会并行执行

[来源: reference/context7_langgraph_01_edges_routing.md]

---

### 2. 并行执行

当路由函数返回多个节点时，这些节点会并行执行。

```python
def parallel_route(state: State) -> list[str]:
    """并行执行多个处理节点"""
    return ["process_a", "process_b", "process_c"]

graph.add_conditional_edges("start", parallel_route)
```

**执行流程**：
```
start
  ↓
parallel_route() → ["process_a", "process_b", "process_c"]
  ↓
┌─────────┬─────────┬─────────┐
│process_a│process_b│process_c│  (并行执行)
└─────────┴─────────┴─────────┘
```

**优势**：
- 提高执行效率
- 充分利用计算资源
- 适合独立的处理任务

---

### 3. 汇聚节点

使用 `add_edge` 的多起始节点功能，可以等待所有并行节点完成后再执行汇聚节点。

```python
# 并行执行
graph.add_conditional_edges("router", multi_route)

# 汇聚节点：等待所有并行节点完成
graph.add_edge(["high_handler", "even_handler", "low_handler"], "merge")
graph.add_edge("merge", END)
```

**执行流程**：
```
router
  ↓
multi_route() → ["high_handler", "even_handler"]
  ↓
┌────────────┬────────────┐
│high_handler│even_handler│  (并行执行)
└────────────┴────────────┘
  ↓           ↓
  └───────────┘
       ↓
     merge  (等待所有节点完成)
       ↓
      END
```

**关键点**：
- **AND 逻辑**：汇聚节点会等待所有起始节点完成
- **状态合并**：多个节点的状态更新会合并到一起
- **同步点**：汇聚节点是并行执行的同步点

[来源: reference/source_edges_01_add_edge.md]

## 多分支路由的应用场景

### 场景 1：多步推理

在复杂的推理任务中，可能需要并行执行多个推理步骤。

```python
class ReasoningState(TypedDict):
    question: str
    reasoning_paths: list[str]
    final_answer: str

def reasoning_route(state: ReasoningState) -> list[str]:
    """并行执行多个推理路径"""
    return ["logical_reasoning", "analogical_reasoning", "causal_reasoning"]

def logical_reasoning(state: ReasoningState) -> dict:
    # 逻辑推理
    return {"reasoning_paths": state.get("reasoning_paths", []) + ["逻辑推理结果"]}

def analogical_reasoning(state: ReasoningState) -> dict:
    # 类比推理
    return {"reasoning_paths": state.get("reasoning_paths", []) + ["类比推理结果"]}

def causal_reasoning(state: ReasoningState) -> dict:
    # 因果推理
    return {"reasoning_paths": state.get("reasoning_paths", []) + ["因果推理结果"]}

def merge_reasoning(state: ReasoningState) -> dict:
    # 合并推理结果
    return {"final_answer": f"综合{len(state['reasoning_paths'])}个推理路径的结果"}

graph = StateGraph(ReasoningState)
graph.add_node("start", lambda s: s)
graph.add_node("logical_reasoning", logical_reasoning)
graph.add_node("analogical_reasoning", analogical_reasoning)
graph.add_node("causal_reasoning", causal_reasoning)
graph.add_node("merge", merge_reasoning)

graph.add_edge(START, "start")
graph.add_conditional_edges("start", reasoning_route)
graph.add_edge(["logical_reasoning", "analogical_reasoning", "causal_reasoning"], "merge")
graph.add_edge("merge", END)
```

**优势**：
- 并行执行多个推理路径
- 提高推理效率
- 综合多个视角的结果

[来源: reference/context7_langgraph_01_edges_routing.md]

### 场景 2：并行处理

在数据处理任务中，可能需要并行执行多个处理步骤。

```python
class ProcessingState(TypedDict):
    data: str
    validation_result: str
    transformation_result: str
    enrichment_result: str

def processing_route(state: ProcessingState) -> list[str]:
    """并行执行多个处理步骤"""
    return ["validate", "transform", "enrich"]

def validate(state: ProcessingState) -> dict:
    # 数据验证
    return {"validation_result": "验证通过"}

def transform(state: ProcessingState) -> dict:
    # 数据转换
    return {"transformation_result": "转换完成"}

def enrich(state: ProcessingState) -> dict:
    # 数据增强
    return {"enrichment_result": "增强完成"}

def finalize(state: ProcessingState) -> dict:
    # 最终处理
    return {"data": "所有处理完成"}

graph = StateGraph(ProcessingState)
graph.add_node("start", lambda s: s)
graph.add_node("validate", validate)
graph.add_node("transform", transform)
graph.add_node("enrich", enrich)
graph.add_node("finalize", finalize)

graph.add_edge(START, "start")
graph.add_conditional_edges("start", processing_route)
graph.add_edge(["validate", "transform", "enrich"], "finalize")
graph.add_edge("finalize", END)
```

**优势**：
- 并行执行独立的处理步骤
- 减少总执行时间
- 提高系统吞吐量

### 场景 3：RAG 多路检索

在 RAG 系统中，可能需要从多个数据源并行检索。

```python
class RAGState(TypedDict):
    query: str
    vector_results: list[str]
    keyword_results: list[str]
    graph_results: list[str]
    final_docs: list[str]

def retrieval_route(state: RAGState) -> list[str]:
    """并行执行多种检索策略"""
    return ["vector_search", "keyword_search", "graph_search"]

def vector_search(state: RAGState) -> dict:
    # 向量检索
    return {"vector_results": ["向量检索结果1", "向量检索结果2"]}

def keyword_search(state: RAGState) -> dict:
    # 关键词检索
    return {"keyword_results": ["关键词检索结果1", "关键词检索结果2"]}

def graph_search(state: RAGState) -> dict:
    # 图检索
    return {"graph_results": ["图检索结果1", "图检索结果2"]}

def merge_results(state: RAGState) -> dict:
    # 合并检索结果
    all_results = (
        state.get("vector_results", []) +
        state.get("keyword_results", []) +
        state.get("graph_results", [])
    )
    return {"final_docs": all_results}

graph = StateGraph(RAGState)
graph.add_node("query_analysis", lambda s: s)
graph.add_node("vector_search", vector_search)
graph.add_node("keyword_search", keyword_search)
graph.add_node("graph_search", graph_search)
graph.add_node("merge_results", merge_results)

graph.add_edge(START, "query_analysis")
graph.add_conditional_edges("query_analysis", retrieval_route)
graph.add_edge(["vector_search", "keyword_search", "graph_search"], "merge_results")
graph.add_edge("merge_results", END)
```

**优势**：
- 并行检索多个数据源
- 提高检索召回率
- 综合多种检索策略的结果

[来源: reference/context7_langgraph_01_edges_routing.md]

## 多分支路由的实现细节

### 1. 路由函数签名

```python
from typing import Sequence, Literal

# 返回多个节点名称
def multi_route(state: State) -> Sequence[str]:
    return ["node_a", "node_b", "node_c"]

# 使用 Literal 类型提示
def multi_route_typed(state: State) -> Sequence[Literal["node_a", "node_b", "node_c"]]:
    return ["node_a", "node_b"]
```

**关键点**：
- **返回类型**：`Sequence[str]` 或 `list[str]`
- **类型提示**：使用 `Literal` 明确可能的节点名称
- **空列表**：可以返回空列表，表示不执行任何节点

---

### 2. 状态合并机制

当多个节点并行执行时，它们的状态更新会合并到一起。

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    value: int
    results: Annotated[list[str], add]  # 使用 add reducer 合并列表

def node_a(state: State) -> dict:
    return {"results": ["A"]}

def node_b(state: State) -> dict:
    return {"results": ["B"]}

def node_c(state: State) -> dict:
    return {"results": ["C"]}

# 并行执行后，results = ["A", "B", "C"]
```

**关键点**：
- **Reducer 函数**：使用 `Annotated` 和 reducer 函数定义状态合并规则
- **add reducer**：将多个列表合并为一个列表
- **自定义 reducer**：可以定义自定义的合并逻辑

[来源: reference/source_edges_01_add_edge.md]

---

### 3. 条件多分支

路由函数可以根据状态动态决定执行哪些节点。

```python
def conditional_multi_route(state: State) -> list[str]:
    """根据状态动态决定执行哪些节点"""
    targets = []

    # 条件1：值大于50
    if state["value"] > 50:
        targets.append("high_handler")

    # 条件2：值是偶数
    if state["value"] % 2 == 0:
        targets.append("even_handler")

    # 条件3：值小于100
    if state["value"] < 100:
        targets.append("low_handler")

    # 如果没有满足任何条件，返回空列表
    return targets
```

**执行示例**：
- `value = 60`：返回 `["high_handler", "even_handler", "low_handler"]`
- `value = 25`：返回 `["low_handler"]`
- `value = 150`：返回 `["even_handler"]`

## 多分支路由的最佳实践

### 1. 使用类型提示

```python
# ✅ 推荐：使用 Literal 类型提示
def multi_route(state: State) -> Sequence[Literal["node_a", "node_b", "node_c"]]:
    return ["node_a", "node_b"]

# ❌ 不推荐：没有类型提示
def multi_route(state):
    return ["node_a", "node_b"]
```

**优势**：
- 帮助 IDE 提供代码补全
- 帮助图可视化工具理解路由路径
- 提前发现类型错误

---

### 2. 处理空列表情况

```python
def multi_route(state: State) -> list[str]:
    targets = []

    if state["condition_a"]:
        targets.append("node_a")

    if state["condition_b"]:
        targets.append("node_b")

    # 如果没有满足任何条件，返回空列表
    # 图会跳过这个路由，直接执行后续节点
    return targets
```

**关键点**：
- **空列表**：表示不执行任何节点
- **跳过路由**：图会跳过这个路由，直接执行后续节点
- **默认行为**：可以在空列表时返回一个默认节点

---

### 3. 避免过多并行节点

```python
# ❌ 不推荐：过多并行节点
def multi_route(state: State) -> list[str]:
    return ["node1", "node2", "node3", "node4", "node5", "node6", "node7", "node8"]

# ✅ 推荐：合理控制并行节点数量
def multi_route(state: State) -> list[str]:
    # 最多并行3个节点
    targets = []
    if state["condition_a"]:
        targets.append("node_a")
    if state["condition_b"]:
        targets.append("node_b")
    if state["condition_c"]:
        targets.append("node_c")
    return targets[:3]  # 限制最多3个
```

**原因**：
- 过多并行节点会消耗大量资源
- 可能导致性能下降
- 增加调试难度

---

### 4. 使用有意义的节点名称

```python
# ✅ 推荐：节点名称清晰
def multi_route(state: State) -> list[str]:
    return ["validate_data", "transform_data", "enrich_data"]

# ❌ 不推荐：节点名称不清晰
def multi_route(state: State) -> list[str]:
    return ["node1", "node2", "node3"]
```

---

### 5. 添加日志和调试信息

```python
def multi_route(state: State) -> list[str]:
    targets = []

    if state["value"] > 50:
        print("添加 high_handler")
        targets.append("high_handler")

    if state["value"] % 2 == 0:
        print("添加 even_handler")
        targets.append("even_handler")

    print(f"最终路由目标: {targets}")
    return targets
```

## 多分支路由与单分支路由的对比

| 特性 | 单分支路由 | 多分支路由 |
|------|-----------|-----------|
| **返回值** | 单个节点名称 | 多个节点名称 |
| **执行方式** | 顺序执行 | 并行执行 |
| **适用场景** | 简单决策 | 复杂并行任务 |
| **性能** | 较低 | 较高（并行） |
| **复杂度** | 低 | 中等 |
| **调试难度** | 低 | 中等 |

## 完整示例：RAG 多路检索系统

```python
from langgraph.graph import StateGraph, START, END
from typing import Sequence, Literal, Annotated
from typing_extensions import TypedDict
from operator import add

class RAGState(TypedDict):
    query: str
    retrieval_strategy: str
    vector_results: list[str]
    keyword_results: list[str]
    graph_results: list[str]
    final_docs: Annotated[list[str], add]

def analyze_query(state: RAGState) -> dict:
    """分析查询，决定检索策略"""
    query = state["query"]

    # 简单的策略决策
    if len(query) > 50:
        strategy = "comprehensive"  # 综合检索
    elif "?" in query:
        strategy = "semantic"  # 语义检索
    else:
        strategy = "keyword"  # 关键词检索

    return {"retrieval_strategy": strategy}

def retrieval_route(state: RAGState) -> Sequence[Literal["vector_search", "keyword_search", "graph_search"]]:
    """根据策略决定检索方法"""
    strategy = state["retrieval_strategy"]

    if strategy == "comprehensive":
        # 综合检索：使用所有方法
        return ["vector_search", "keyword_search", "graph_search"]
    elif strategy == "semantic":
        # 语义检索：使用向量和图检索
        return ["vector_search", "graph_search"]
    else:
        # 关键词检索：只使用关键词检索
        return ["keyword_search"]

def vector_search(state: RAGState) -> dict:
    """向量检索"""
    print(f"执行向量检索: {state['query']}")
    return {"final_docs": ["向量检索结果1", "向量检索结果2"]}

def keyword_search(state: RAGState) -> dict:
    """关键词检索"""
    print(f"执行关键词检索: {state['query']}")
    return {"final_docs": ["关键词检索结果1", "关键词检索结果2"]}

def graph_search(state: RAGState) -> dict:
    """图检索"""
    print(f"执行图检索: {state['query']}")
    return {"final_docs": ["图检索结果1", "图检索结果2"]}

def merge_results(state: RAGState) -> dict:
    """合并检索结果"""
    docs = state["final_docs"]
    print(f"合并 {len(docs)} 个检索结果")
    return {"final_docs": docs}

# 构建图
graph = StateGraph(RAGState)

# 添加节点
graph.add_node("analyze_query", analyze_query)
graph.add_node("vector_search", vector_search)
graph.add_node("keyword_search", keyword_search)
graph.add_node("graph_search", graph_search)
graph.add_node("merge_results", merge_results)

# 添加边
graph.add_edge(START, "analyze_query")
graph.add_conditional_edges("analyze_query", retrieval_route)
graph.add_edge(["vector_search", "keyword_search", "graph_search"], "merge_results")
graph.add_edge("merge_results", END)

# 编译图
app = graph.compile()

# 测试
result = app.invoke({
    "query": "什么是 LangGraph 的多分支路由？",
    "retrieval_strategy": "",
    "vector_results": [],
    "keyword_results": [],
    "graph_results": [],
    "final_docs": []
})

print(f"\n最终结果: {result['final_docs']}")
```

**输出**：
```
执行向量检索: 什么是 LangGraph 的多分支路由？
执行图检索: 什么是 LangGraph 的多分支路由？
合并 4 个检索结果

最终结果: ['向量检索结果1', '向量检索结果2', '图检索结果1', '图检索结果2']
```

## 总结

**多分支路由的核心特性**：
1. **返回多个节点**：路由函数可以返回多个目标节点
2. **并行执行**：所有返回的节点会并行执行
3. **汇聚节点**：使用 `add_edge` 的多起始节点功能等待所有节点完成
4. **状态合并**：使用 reducer 函数合并多个节点的状态更新

**应用场景**：
- 多步推理
- 并行处理
- RAG 多路检索
- 数据验证和转换

**最佳实践**：
- 使用类型提示（`Literal`）
- 处理空列表情况
- 避免过多并行节点
- 使用有意义的节点名称
- 添加日志和调试信息

通过合理使用多分支路由，可以构建高效的并行处理工作流，提高系统性能和吞吐量。
