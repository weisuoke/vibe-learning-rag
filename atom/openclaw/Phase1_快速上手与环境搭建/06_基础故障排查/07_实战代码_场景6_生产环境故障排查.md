# OpenClaw 基础故障排查 - 实战场景 6：生产环境故障排查

## 一、场景概述

**场景描述**：综合诊断生产环境的复杂问题，使用完整的故障排查流程

**适用场景**：
- 生产环境服务异常
- 多个症状同时出现
- 需要快速定位根因
- 需要完整的诊断报告

**核心流程**：
1. 60秒快速诊断
2. 深度探测分析
3. 日志特征识别
4. 配置验证修复
5. 服务恢复验证

## 二、TypeScript 综合诊断系统

### 2.1 诊断编排器

```typescript
// production-troubleshooter.ts
import { exec } from 'child_process';
import { promisify } from 'util';
import * as fs from 'fs/promises';

const execAsync = promisify(exec);

// 诊断结果
interface DiagnosticResult {
  step: string;
  status: 'pass' | 'fail' | 'warning';
  message: string;
  suggestions?: string[];
}

// 诊断报告
interface DiagnosticReport {
  timestamp: number;
  duration: number;
  results: DiagnosticResult[];
  summary: {
    total: number;
    passed: number;
    failed: number;
    warnings: number;
  };
  rootCause?: string;
  recommendations: string[];
}

// 生产环境故障排查器
class ProductionTroubleshooter {
  private results: DiagnosticResult[] = [];
  private startTime: number = 0;

  // 完整诊断流程
  async diagnose(): Promise<DiagnosticReport> {
    this.startTime = Date.now();
    this.results = [];

    console.log('=== OpenClaw Production Troubleshooting ===\n');

    // 阶段 1: 60秒快速诊断
    console.log('Phase 1: 60-Second Quick Diagnosis');
    await this.quickDiagnosis();

    // 阶段 2: 深度探测
    console.log('\nPhase 2: Deep Probe');
    await this.deepProbe();

    // 阶段 3: 日志分析
    console.log('\nPhase 3: Log Analysis');
    await this.logAnalysis();

    // 阶段 4: 配置验证
    console.log('\nPhase 4: Configuration Validation');
    await this.configValidation();

    // 生成报告
    return this.generateReport();
  }

  // 阶段 1: 60秒快速诊断
  private async quickDiagnosis(): Promise<void> {
    await this.runCommand('openclaw status', 'Quick Status Check');
    await this.runCommand('openclaw gateway status', 'Gateway Status');
    await this.runCommand('openclaw doctor', 'Health Check');
    await this.runCommand('openclaw channels status --probe', 'Channel Connectivity');
  }

  // 阶段 2: 深度探测
  private async deepProbe(): Promise<void> {
    await this.runCommand('openclaw status --deep', 'Deep Status Analysis');

    const gatewayResult = await this.runCommand(
      'openclaw gateway status --json',
      'Gateway JSON Status'
    );

    if (gatewayResult.status === 'pass') {
      try {
        const gatewayData = JSON.parse(gatewayResult.details?.stdout || '{}');
        this.analyzeGatewayStatus(gatewayData);
      } catch (error) {
        // JSON 解析失败，继续
      }
    }

    await this.runCommand('openclaw doctor --deep', 'Deep Health Check');
  }

  // 阶段 3: 日志分析
  private async logAnalysis(): Promise<void> {
    const logResult = await this.runCommand(
      'openclaw logs --tail 100',
      'Recent Logs'
    );

    if (logResult.status === 'pass' && logResult.details?.stdout) {
      this.analyzeLogPatterns(logResult.details.stdout);
    }
  }

  // 阶段 4: 配置验证
  private async configValidation(): Promise<void> {
    const doctorResult = await this.runCommand(
      'openclaw doctor',
      'Configuration Validation'
    );

    if (doctorResult.status === 'fail') {
      this.results.push({
        step: 'Auto-Repair Suggestion',
        status: 'warning',
        message: 'Configuration issues detected',
        suggestions: [
          'Run: openclaw doctor --repair',
          'This will automatically fix common configuration issues'
        ]
      });
    }
  }

  // 运行命令并记录结果
  private async runCommand(
    command: string,
    step: string
  ): Promise<DiagnosticResult> {
    console.log(`  Running: ${command}`);

    try {
      const { stdout, stderr } = await execAsync(command, { timeout: 30000 });

      const result: DiagnosticResult = {
        step,
        status: 'pass',
        message: `${step}: PASS`,
        details: { stdout, stderr }
      };

      this.results.push(result);
      console.log(`  ✓ ${step}`);
      return result;
    } catch (error: any) {
      const result: DiagnosticResult = {
        step,
        status: 'fail',
        message: `${step}: FAIL`,
        suggestions: this.getSuggestionsForError(error)
      };

      this.results.push(result);
      console.log(`  ✗ ${step}`);
      return result;
    }
  }

  // 分析 Gateway 状态
  private analyzeGatewayStatus(gatewayData: any): void {
    if (gatewayData.runtime !== 'running') {
      this.results.push({
        step: 'Gateway Runtime Analysis',
        status: 'fail',
        message: `Gateway runtime is ${gatewayData.runtime}`,
        suggestions: [
          'Run: openclaw gateway restart',
          'Check logs: openclaw logs --follow'
        ]
      });
    }

    if (gatewayData.rpcProbe !== 'ok') {
      this.results.push({
        step: 'RPC Probe Analysis',
        status: 'fail',
        message: 'RPC probe failed',
        suggestions: [
          'Check gateway authentication',
          'Verify gateway.auth.token in config'
        ]
      });
    }
  }

  // 分析日志模式
  private analyzeLogPatterns(logs: string): void {
    const patterns = [
      {
        pattern: /EADDRINUSE.*18789/,
        issue: 'Port 18789 Already in Use',
        severity: 'fail' as const,
        suggestions: ['Run: lsof -i :18789', 'Kill the process or change port']
      },
      {
        pattern: /unauthorized|401|403/i,
        issue: 'Authentication Failure',
        severity: 'fail' as const,
        suggestions: ['Check gateway.auth.token', 'Run: openclaw doctor']
      },
      {
        pattern: /mention required/i,
        issue: 'Mention Gating Active',
        severity: 'warning' as const,
        suggestions: ['Messages require bot mention', 'Check requireMention in config']
      },
      {
        pattern: /pairing request|pairing required/i,
        issue: 'Pairing Approval Pending',
        severity: 'warning' as const,
        suggestions: ['Run: openclaw pairing list <channel>', 'Approve pending pairings']
      }
    ];

    for (const { pattern, issue, severity, suggestions } of patterns) {
      if (pattern.test(logs)) {
        this.results.push({
          step: 'Log Pattern Analysis',
          status: severity,
          message: `Detected: ${issue}`,
          suggestions
        });
      }
    }
  }

  // 根据错误获取建议
  private getSuggestionsForError(error: any): string[] {
    const errorMsg = error.message.toLowerCase();

    if (errorMsg.includes('enoent') || errorMsg.includes('not found')) {
      return ['OpenClaw CLI not found', 'Install: npm install -g openclaw'];
    }

    if (errorMsg.includes('timeout')) {
      return ['Command timed out', 'Check: openclaw gateway status'];
    }

    if (errorMsg.includes('econnrefused')) {
      return ['Cannot connect to gateway', 'Run: openclaw gateway start'];
    }

    return ['Check command output for details'];
  }

  // 生成诊断报告
  private generateReport(): DiagnosticReport {
    const duration = Date.now() - this.startTime;

    const summary = {
      total: this.results.length,
      passed: this.results.filter(r => r.status === 'pass').length,
      failed: this.results.filter(r => r.status === 'fail').length,
      warnings: this.results.filter(r => r.status === 'warning').length
    };

    return {
      timestamp: this.startTime,
      duration,
      results: this.results,
      summary,
      rootCause: this.identifyRootCause(),
      recommendations: this.generateRecommendations()
    };
  }

  // 识别根因
  private identifyRootCause(): string | undefined {
    const criticalFailures = this.results.filter(r => r.status === 'fail');

    if (criticalFailures.length === 0) {
      return undefined;
    }

    const portIssue = criticalFailures.find(r =>
      r.message.includes('Port') || r.message.includes('EADDRINUSE')
    );
    if (portIssue) {
      return 'Port 18789 is already in use by another process';
    }

    const authIssue = criticalFailures.find(r =>
      r.message.includes('Authentication') || r.message.includes('unauthorized')
    );
    if (authIssue) {
      return 'Gateway authentication is misconfigured';
    }

    return 'Multiple issues detected, see detailed results';
  }

  // 生成建议
  private generateRecommendations(): string[] {
    const recommendations: string[] = [];
    const failedSteps = this.results.filter(r => r.status === 'fail');

    if (failedSteps.length === 0) {
      recommendations.push('✓ All checks passed - system is healthy');
      return recommendations;
    }

    const allSuggestions = new Set<string>();
    for (const result of this.results) {
      if (result.suggestions) {
        result.suggestions.forEach(s => allSuggestions.add(s));
      }
    }

    const prioritySuggestions = [
      'Run: openclaw doctor --repair',
      'Run: openclaw gateway restart',
      'Check logs: openclaw logs --follow'
    ];

    for (const suggestion of prioritySuggestions) {
      if (allSuggestions.has(suggestion)) {
        recommendations.push(suggestion);
        allSuggestions.delete(suggestion);
      }
    }

    recommendations.push(...Array.from(allSuggestions));
    return recommendations;
  }

  // 打印报告
  printReport(report: DiagnosticReport): void {
    console.log('\n=== Diagnostic Report ===\n');
    console.log(`Duration: ${(report.duration / 1000).toFixed(2)}s`);
    console.log(`Total Checks: ${report.summary.total}`);
    console.log(`  ✓ Passed: ${report.summary.passed}`);
    console.log(`  ✗ Failed: ${report.summary.failed}`);
    console.log(`  ⚠ Warnings: ${report.summary.warnings}`);

    if (report.rootCause) {
      console.log(`\nRoot Cause: ${report.rootCause}`);
    }

    console.log('\nRecommendations:');
    for (const rec of report.recommendations) {
      console.log(`  - ${rec}`);
    }

    console.log('\n=== End of Report ===');
  }

  // 保存报告到文件
  async saveReport(report: DiagnosticReport, filename: string): Promise<void> {
    await fs.writeFile(filename, JSON.stringify(report, null, 2));
    console.log(`\nReport saved to: ${filename}`);
  }
}

// 使用示例
async function main() {
  const troubleshooter = new ProductionTroubleshooter();

  try {
    const report = await troubleshooter.diagnose();
    troubleshooter.printReport(report);

    const timestamp = new Date().toISOString().replace(/[:.]/g, '-');
    await troubleshooter.saveReport(
      report,
      `openclaw-diagnostic-${timestamp}.json`
    );
  } catch (error) {
    console.error('Diagnostic failed:', error);
    process.exit(1);
  }
}

main().catch(console.error);
```

## 三、类比理解

### 3.1 后端开发类比

**类比 1：Kubernetes 健康检查**

```
OpenClaw 生产诊断 ≈ Kubernetes Probes

相似点：
1. 快速诊断 = Liveness Probe（服务是否存活）
2. 深度探测 = Readiness Probe（服务是否就绪）
3. 日志分析 = kubectl logs（日志排查）
4. 配置验证 = kubectl describe（配置检查）
5. 自动修复 = kubectl rollout restart（重启恢复）
```

**类比 2：Nginx 故障排查**

```
OpenClaw 诊断流程 ≈ Nginx 故障排查

相似点：
1. 快速状态 = nginx -t（配置测试）
2. 深度探测 = nginx -T（完整配置）
3. 日志分析 = tail -f access.log（日志跟踪）
4. 配置验证 = nginx -t && nginx -s reload（验证重载）
```

**来源**: 后端开发经验

## 四、实战场景

### 4.1 场景 1：Gateway 无法启动

```bash
# 症状：Gateway 启动失败

# 1. 快速诊断
openclaw gateway status
# Output: Runtime: stopped

# 2. 查看日志
openclaw logs --tail 50
# Output: Error: listen EADDRINUSE: address already in use :::18789

# 3. 识别根因
lsof -i :18789
# Output: node 12345 user ... (TCP *:18789 (LISTEN))

# 4. 解决方案
kill 12345
openclaw gateway start

# 5. 验证
openclaw gateway status
# Output: Runtime: running ✓
```

## 五、总结

### 5.1 核心要点

1. **系统化诊断**：60秒快速诊断 → 深度探测 → 日志分析 → 配置验证
2. **根因识别**：通过日志特征和状态分析快速定位问题
3. **自动化修复**：使用 doctor --repair 自动修复常见问题
4. **完整验证**：修复后必须验证服务状态
5. **报告生成**：保存诊断报告用于后续分析

### 5.2 使用建议

- ✅ 遵循标准诊断流程
- ✅ 保存诊断报告
- ✅ 验证修复效果
- ✅ 记录根因和解决方案
- ❌ 不要跳过验证步骤

### 5.3 生产环境最佳实践

1. **定期健康检查**：每天运行 `openclaw doctor`
2. **日志监控**：使用 `openclaw logs --follow` 实时监控
3. **配置备份**：定期备份 `~/.openclaw/openclaw.json`
4. **自动化诊断**：集成到 CI/CD 流程
5. **告警机制**：配置健康检查失败告警
