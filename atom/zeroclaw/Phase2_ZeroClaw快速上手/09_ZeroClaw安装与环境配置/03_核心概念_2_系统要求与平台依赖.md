# 核心概念 2：系统要求与平台依赖

> **一句话定义：** 系统要求与平台依赖是运行或构建 ZeroClaw 之前必须准备好的"地基和建材"——Rust 工具链是"万能建造工具"，平台依赖是"各地特产的建材"，构建资源是"施工场地大小"。

---

## 1. 为什么需要了解系统要求？

如果你是前端开发者，可能从没想过"装个工具还要检查内存和磁盘"。因为 `npm install -g typescript` 只是下载 JS 文件，Node.js 运行时帮你搞定了一切。

但 ZeroClaw 是 **Rust 编译型程序**。从源码构建意味着：

```
源码 → [编译器 + 链接器 + 系统库] → 机器码二进制
```

这个过程需要：
1. **编译器**（Rust 工具链）—— 把 `.rs` 翻译成机器码
2. **链接器**（系统提供）—— 把目标文件"粘合"成可执行文件
3. **系统库**（如 OpenSSL）—— 提供加密、网络等底层能力
4. **足够的资源**（RAM、磁盘）—— 编译是内存和 CPU 密集型操作

> **前端类比：** 遇到过 `node-gyp` 编译 `bcrypt`、`sharp` 失败吗？报错缺 `build-essential`、缺 `Visual Studio Build Tools`？那就是"编译型代码需要系统依赖"。ZeroClaw 整个程序都是编译型的，所以这个问题覆盖了全部安装流程。

**好消息：** 如果你选择**预编译二进制**或 **Homebrew** 安装，大部分系统依赖都不需要。本文主要面向源码构建、源码贡献、CI/CD 构建和安装报错排查场景。

## 2. Rust 工具链详解

### 2.1 什么是 Rust 工具链？

**Rust 工具链 = 编译器 + 包管理器 + 标准库**，由 `rustup` 统一管理。

```
rustup（版本管理器）
├── rustc     — 编译器（.rs → 机器码）
├── cargo     — 包管理器 + 构建系统（npm + webpack 合体）
├── rustfmt   — 代码格式化（prettier）
├── clippy    — 代码检查（eslint）
└── std       — 标准库（Node.js 内置模块）
```

| Rust 工具 | 前端对应 |
|----------|---------|
| `rustup` | `nvm` / `fnm` |
| `cargo` | `npm` / `pnpm` |
| `rustc` | V8（内置在 Node.js 中） |
| `Cargo.lock` | `package-lock.json` |
| crates.io | npmjs.com |

### 2.2 安装 rustup

```bash
# Linux / macOS
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"

# 验证
rustc --version    # 预期: rustc 1.87.0 或更高
cargo --version
```

**Windows：** 访问 https://rustup.rs/ 下载 `rustup-init.exe`，需先安装 Visual Studio Build Tools（见 3.6 节）。

> **日常生活类比：** 安装 `rustup` 就像装"应用商店"——装好后就能安装、更新、管理所有 Rust 工具。

### 2.3 MSRV（最低支持 Rust 版本）

**ZeroClaw 的 MSRV 是 1.87。** Rust 编译器版本必须 >= 1.87 才能编译。

[来源: sourcecode/zeroclaw/Cargo.toml]

```
rustc 1.85  → ❌ 编译失败
rustc 1.87  → ✅ 恰好满足
rustc 1.93  → ✅ 推荐
```

> **前端类比：** MSRV 就像 `package.json` 中的 `engines.node: ">=18.0.0"`。

### 2.4 rust-toolchain.toml 固定版本

ZeroClaw 仓库根目录的 `rust-toolchain.toml` 会**自动固定 Rust 版本**：

```toml
[toolchain]
channel = "stable"
```

`cd` 进入源码目录时，`rustup` 自动读取并切换版本，缺少则自动下载。

[来源: sourcecode/zeroclaw/rust-toolchain.toml]

| 文件 | 作用 | 自动切换？ |
|------|------|-----------|
| `.nvmrc` | 指定 Node.js 版本 | 需要 `nvm use` |
| `rust-toolchain.toml` | 指定 Rust 版本 | ✅ 全自动 |

### 2.5 版本管理常用命令

```bash
rustup update stable              # 更新到最新 stable
rustup install 1.87.0             # 安装特定版本
rustup default stable             # 设置全局默认
rustup toolchain list             # 查看已安装的工具链
rustup target add aarch64-unknown-linux-gnu  # 添加交叉编译目标
```

## 3. 各平台构建依赖详解

### 3.1 依赖作用速查

| 依赖 | 作用 | 前端类比 |
|------|------|---------|
| **gcc / build-essential** | C/C++ 编译器 + 链接器 | `node-gyp` 背后调用的就是它 |
| **pkg-config** | 帮编译器找到系统库位置 | webpack 的 `resolve.alias` |
| **libssl-dev** | TLS/SSL 加密库 | Node.js 内置了 OpenSSL |
| **perl** | OpenSSL 构建脚本依赖 | 不需要你写 Perl |
| **git / curl** | 下载源码和文件 | 和前端一样 |
| **ca-certificates** | HTTPS 证书信任链 | 浏览器内置了 |

> **日常生活类比（盖房子）：** gcc = 锤子电钻 | pkg-config = 建材采购清单 | libssl-dev = 门锁防盗窗 | perl = 门锁安装说明翻译器 | git = 运输卡车

### 3.2 Linux (Debian / Ubuntu)

```bash
sudo apt install -y build-essential pkg-config git curl libssl-dev perl ca-certificates
```

- `build-essential` = gcc + g++ + make + libc-dev 全家桶
- `libssl-dev` = OpenSSL 开发头文件（注意 `-dev` 后缀）

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 3.3 Linux (Fedora / RHEL)

```bash
sudo dnf install -y gcc gcc-c++ make pkgconf-pkg-config git curl openssl-devel perl
```

与 Debian 的区别：`build-essential` → 拆分为 `gcc` + `gcc-c++` + `make`；`libssl-dev` → `openssl-devel`（Red Hat 系用 `-devel`）。

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 3.4 Linux (Arch)

```bash
sudo pacman -S --noconfirm gcc make pkgconf git curl openssl perl ca-certificates
```

装过 `base-devel` 的话，大部分已有。

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 3.5 Linux (Alpine)

```bash
apk add bash build-base pkgconf git curl openssl-dev perl ca-certificates
```

Alpine 特殊：使用 `musl libc`（体积小，某些 crate 可能兼容性问题）；`bash` 需显式安装（默认是 `ash`）。

> **前端类比：** `node:alpine` vs `node:slim` —— 体积小但偶尔有原生模块编译问题。

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 3.6 macOS

```bash
xcode-select --install
```

**只需一行！** Xcode CLI Tools 包含 clang、链接器、make、系统 SDK。macOS 不需要手动装 OpenSSL——ZeroClaw 使用 `rustls`（纯 Rust TLS）。

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 3.7 Windows

1. **Visual Studio Build Tools 2022**（免费）—— 勾选 **"Desktop development with C++"**
2. **Rust 工具链** —— `rustup-init.exe` 会自动检测 MSVC

Windows 最复杂是因为链接器 `link.exe` 绑定在 Visual Studio Build Tools 中。

## 4. 构建资源规划

### 4.1 资源要求

| 资源 | 最低要求 | 推荐 | 说明 |
|------|---------|------|------|
| **RAM + swap** | 2 GB | 4 GB+ | 编译消耗大量内存 |
| **磁盘空间** | 6 GB | 10 GB+ | 工具链 ~1GB + 编译产物 ~4-5GB |

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 4.2 Bootstrap 预检环境变量

安装脚本编译前**自动检查资源**：

```bash
ZEROCLAW_BOOTSTRAP_MIN_RAM_MB=2048     # 默认最低 2 GB
ZEROCLAW_BOOTSTRAP_MIN_DISK_MB=6144    # 默认最低 6 GB

# 低配机器可降低门槛
ZEROCLAW_BOOTSTRAP_MIN_RAM_MB=1024 ./bootstrap.sh
```

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

### 4.3 低配机器三种方案

**方案 A：预编译二进制（推荐，零资源消耗）**
```bash
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --prebuilt-only
```

**方案 B：交叉编译（在高配机器上编译）**
```bash
rustup target add armv7-unknown-linux-gnueabihf
cargo build --release --target armv7-unknown-linux-gnueabihf
# scp 到目标设备
```

**方案 C：增加 swap**
```bash
sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile
sudo mkswap /swapfile && sudo swapon /swapfile
# 编译完后: sudo swapoff /swapfile && sudo rm /swapfile
```

> **日常生活类比：** A = 买成品家具 | B = 工厂定制快递到家 | C = 临时搭棚子扩展场地

### 4.4 为什么 Rust 编译这么吃资源？

| 阶段 | TypeScript | Rust |
|------|-----------|------|
| 类型检查 | ✅ | ✅（更严格） |
| 所有权检查 | ❌ | ✅（独特步骤） |
| 代码优化 | ❌ 交给 V8 JIT | ✅ 编译时完成 |
| 代码生成 | JS 文本 | 机器码（LLVM） |
| 链接 | ❌ | ✅ 合成单一二进制 |
| 依赖编译 | ❌ 运行时加载 | ✅ 全部编译 |

编译时的大量工作换来了运行时极致的性能——~8.8 MB 二进制、<5 MB 内存、<10ms 启动。

## 5. 运行时性能数据

| 指标 | 数值 | 对比 |
|------|------|------|
| **二进制大小** | ~8.8 MB | Electron 应用 ~200-300 MB |
| **运行内存** | ~3.9-4.1 MB | 空白 Node.js 进程 ~30 MB |
| **启动时间** | <10ms (0.8GHz) | Next.js 冷启动 ~2-5 秒 |
| **编译时间** | ~30-40 分钟 | 一次编译，终身受益 |

这些数据是 Release Profile 优化的结果（`opt-level = "z"`, `lto = "fat"`, `strip = true`）。ZeroClaw 把 22+ 个 Provider、工具系统、内存引擎全部压进 8.8 MB，可在 $10 ARM 开发板上运行。

[来源: sourcecode/zeroclaw/Cargo.toml, scripts/bootstrap.sh]

## 6. 支持的目标平台

### 6.1 一级支持（官方预编译）

| 平台 | Target Triple | 典型设备 |
|------|--------------|---------|
| Linux x86_64 | `x86_64-unknown-linux-gnu` | PC、云服务器 |
| Linux ARM64 | `aarch64-unknown-linux-gnu` | ARM 服务器、树莓派 4/5 |
| Linux ARM32 | `armv7-unknown-linux-gnueabihf` | 树莓派 3 |
| macOS Intel | `x86_64-apple-darwin` | Intel Mac |
| macOS ARM | `aarch64-apple-darwin` | M1/M2/M3/M4 Mac |
| Windows | `x86_64-pc-windows-msvc` | Windows PC |

[来源: sourcecode/zeroclaw/.github/workflows/release.yml]

### 6.2 二级支持（可编译，无预编译）

Android（armv7 / aarch64）、MUSL targets（x86_64 / aarch64 静态链接）。

### 6.3 Target Triple 解读

```
aarch64-unknown-linux-gnu
  │        │       │     │
  │        │       │     └── ABI（gnu = glibc, musl = 静态链接）
  │        │       └──── 操作系统
  │        └──────────── 厂商（unknown = 通用）
  └───────────────────── CPU 架构
```

> **前端类比：** Target Triple 类似 User-Agent 字符串——告诉编译器"目标环境是什么"。

## 7. 常见问题排查

### 7.1 OpenSSL 缺失

```
error: failed to run custom build command for `openssl-sys`
```

```bash
sudo apt install -y libssl-dev          # Debian/Ubuntu
sudo dnf install -y openssl-devel       # Fedora
brew install openssl && export OPENSSL_DIR=$(brew --prefix openssl)  # macOS
```

ZeroClaw 默认用 `rustls`，但某些可选 Feature 可能触发 OpenSSL 依赖。

### 7.2 链接器错误

```
error: linker `cc` not found
```

```bash
sudo apt install -y build-essential     # Debian/Ubuntu
xcode-select --install                  # macOS
# Windows: 安装 Visual Studio Build Tools → "Desktop development with C++"
```

### 7.3 编译被 OOM kill

```
Killed (signal 9)
```

```bash
cargo build --release -j 1              # 单线程编译省内存
# 或直接用预编译: --prebuilt-only
```

### 7.4 Rust 版本过低

```
requires rustc 1.87.0 or newer
```

```bash
rustup update stable
```

### 7.5 pkg-config 找不到库

```bash
sudo apt install -y pkg-config libssl-dev
pkg-config --libs openssl               # 验证: 应输出 -lssl -lcrypto
```

## 8. 在 ZeroClaw 开发中的应用

### 8.1 源码开发环境清单

```bash
# 1. Rust 工具链
rustup update stable && rustc --version   # >= 1.87

# 2. 系统依赖（选择你的平台）
xcode-select --install                    # macOS
# sudo apt install -y build-essential pkg-config git curl libssl-dev perl ca-certificates  # Ubuntu

# 3. 克隆并构建
git clone https://github.com/zeroclaw-labs/zeroclaw.git && cd zeroclaw
cargo build                               # Debug 构建
cargo build --release                     # Release 构建

# 4. 测试 + 质量检查
cargo test && cargo clippy && cargo fmt --check
```

### 8.2 Docker 中的依赖最小化

```dockerfile
# Builder 阶段：安装所有构建依赖
FROM rust:1.93-slim AS builder
RUN apt-get update && apt-get install -y pkg-config libssl-dev perl ca-certificates

# Release 阶段：只有二进制，零依赖
FROM gcr.io/distroless/cc-debian13 AS release
COPY --from=builder /app/target/release/zeroclaw /zeroclaw
```

构建依赖只存在于 builder 阶段，最终镜像只包含编译好的二进制。

[来源: sourcecode/zeroclaw/Dockerfile]
