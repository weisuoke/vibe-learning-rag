# PromptTemplate高级用法 - 概览

> 本文档是 LangChain PromptTemplate 高级用法的学习指南，帮助你掌握模板组合、部分变量、多格式支持等高级特性

---

## 知识点简介

**PromptTemplate 高级用法**是 LangChain 中用于构建复杂、可复用、动态提示词模板的核心技术。它不仅仅是简单的字符串替换，而是提供了模板组合、部分变量预填充、多种模板格式支持、文件加载等高级特性，让你能够构建模块化、可维护的提示词工程系统。

在 AI Agent 开发中，提示词是与 LLM 交互的核心接口。掌握 PromptTemplate 的高级用法，能够让你：
- 构建可复用的提示词模板库
- 通过组合实现复杂的提示词逻辑
- 动态注入上下文信息（如日期、用户信息）
- 支持多语言、多场景的提示词管理
- 提升提示词工程的开发效率和可维护性

**为什么需要学习高级用法？**

基础的 PromptTemplate 只能做简单的变量替换，但在实际的 AI Agent 开发中，你会遇到：
- 需要在多个模板中共享相同的变量（如系统设置、日期）
- 需要将多个小模板组合成复杂的提示词
- 需要支持不同的模板格式（f-string、mustache、jinja2）
- 需要从文件加载模板以便于管理和版本控制
- 需要与 ChatPromptTemplate、LCEL 等其他组件集成

这些场景都需要 PromptTemplate 的高级特性来解决。

## 核心概念列表

本知识点包含 **6 个核心概念**，按学习顺序排列：

### 1. Partial Variables（部分变量）
**一句话定义：** 预填充模板变量，避免每次调用时重复传递相同的值

**核心特性：**
- 支持字符串固定值（如系统设置、常量）
- 支持函数动态值（如当前日期、动态计算）
- 在模板格式化时自动合并
- 在模板组合时检查冲突

**使用场景：**
- 共享系统设置（如语言、模型名称）
- 动态注入日期时间
- 预填充用户信息
- 减少重复代码

**来源：** [源码分析](reference/source_prompttemplate_01.md) | [Context7 文档](reference/context7_langchain_02.md) | [社区实践](reference/search_partial_01.md)

---

### 2. Template Composition（模板组合）
**一句话定义：** 使用 `+` 操作符将多个小模板组合成复杂的提示词

**核心特性：**
- 使用 `+` 操作符组合两个 PromptTemplate
- 自动合并 input_variables（取并集）
- 自动合并 partial_variables（检查冲突）
- 支持 PromptTemplate + str 的组合
- 要求两个模板的 template_format 必须一致

**使用场景：**
- 构建模块化的提示词系统
- 复用常见的提示词片段
- 动态组合不同的提示词部分
- 提升提示词的可维护性

**来源：** [源码分析](reference/source_prompttemplate_01.md) | [社区实践](reference/search_composition_01.md)

---

### 3. Template Formats（模板格式）
**一句话定义：** 支持三种模板格式（f-string、mustache、jinja2），满足不同的使用需求

**核心特性：**
- **f-string（默认）**：简单快速，使用 `{variable}` 语法
- **mustache**：支持嵌套变量 `{{obj.bar}}`、section `{{#foo}}...{{/foo}}`
- **jinja2**：功能强大，支持复杂逻辑，但有安全风险

**使用场景：**
- 简单场景使用 f-string
- 需要嵌套变量时使用 mustache
- 需要复杂逻辑时使用 jinja2（注意安全）

**来源：** [源码分析](reference/source_prompttemplate_01.md)

---

### 4. Template Validation（模板验证）
**一句话定义：** 可选的模板验证机制，在开发时捕获变量错误

**核心特性：**
- 通过 `validate_template=True` 启用
- 检查模板中的变量是否与 input_variables 匹配
- Mustache 模板不支持验证
- 需要提供 input_variables 才能验证

**使用场景：**
- 开发环境启用验证
- 捕获变量名拼写错误
- 确保模板的正确性

**来源：** [源码分析](reference/source_prompttemplate_01.md)

---

### 5. 从文件加载模板
**一句话定义：** 使用 `from_file` 方法从文件加载模板，便于管理和版本控制

**核心特性：**
- 支持多种文件编码（UTF-8、CP-1252 等）
- 自动提取 input_variables
- 便于模板的版本控制和团队协作

**使用场景：**
- 管理大量的提示词模板
- 多语言模板管理
- 团队协作和版本控制

**来源：** [源码分析](reference/source_prompttemplate_01.md)

---

### 6. 与其他组件的集成
**一句话定义：** PromptTemplate 可以与 ChatPromptTemplate、LCEL、SequentialChain 等组件无缝集成

**核心特性：**
- 与 ChatPromptTemplate 组合构建对话模板
- 使用 `|` 操作符与 LLM 链式组合（LCEL）
- 与 SequentialChain 组合构建复杂工作流

**使用场景：**
- 构建对话式 AI Agent
- 实现多步推理工作流
- 集成到 RAG 系统中

**来源：** [Context7 文档](reference/context7_langchain_01.md) | [Context7 文档](reference/context7_langchain_03.md) | [社区实践](reference/search_composition_01.md)

## 常见问题（FAQ）

### Q1: Partial Variables 和直接传递变量有什么区别？
**A:** Partial Variables 是预填充变量，在模板创建时就固定了值，后续调用时不需要再传递。直接传递变量是在每次调用时都需要传递。Partial Variables 适合共享的、不变的变量（如系统设置），直接传递适合每次调用都不同的变量（如用户输入）。

### Q2: 什么时候使用模板组合，什么时候使用 PipelinePromptTemplate？
**A:** 模板组合（`+` 操作符）适合简单的线性组合，推荐优先使用。PipelinePromptTemplate 适合复杂的多步骤模板组合，但在 2025-2026 年的最新实践中，推荐使用模板组合 + LCEL 表达式来替代 PipelinePromptTemplate。

### Q3: 如何选择模板格式（f-string、mustache、jinja2）？
**A:**
- **f-string**：默认选择，简单快速，适合 90% 的场景
- **mustache**：需要嵌套变量或 section 时使用
- **jinja2**：需要复杂逻辑（循环、条件）时使用，但要注意安全风险

### Q4: ChatPromptTemplate 支持 partial_variables 吗？
**A:** 在某些版本中存在兼容性问题（参考 [GitHub Issue #17560](https://github.com/langchain-ai/langchain/issues/17560)）。建议在使用前测试，或者使用 PromptTemplate 组合的方式来实现类似功能。

### Q5: 模板组合时如何处理变量冲突？
**A:**
- `input_variables` 会自动合并（取并集），不会冲突
- `partial_variables` 如果有相同的 key，会抛出 ValueError
- 建议使用命名空间前缀来避免冲突（如 `system_date`、`user_date`）
