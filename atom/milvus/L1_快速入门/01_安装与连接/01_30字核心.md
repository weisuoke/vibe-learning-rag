# 30字核心

**Milvus 2.6 通过 Docker Compose 快速部署，使用 pymilvus 2.6+ SDK 连接，是构建 RAG 系统的向量数据库基础。**

---

## 为什么这30个字很重要？

这句话浓缩了 Milvus 2.6 入门的三个关键要素：

1. **部署方式**：Docker Compose（一键启动，环境隔离）
2. **连接工具**：pymilvus 2.6+ SDK（Python 编程接口）
3. **应用场景**：RAG 系统的向量数据库基础（实战导向）

---

## 核心要素拆解

### 1. Docker Compose 快速部署

**含义**：使用 Docker Compose 配置文件一键启动 Milvus 及其依赖服务。

**为什么重要**：
- 无需手动安装依赖（etcd、MinIO 等）
- 环境隔离，不污染本地系统
- 一条命令启动/停止，适合开发和测试

**2026 年最新特性**：
- Milvus 2.6.11 引入 **Woodpecker WAL**，替代 Kafka/Pulsar
- **Streaming Node** 架构，简化部署复杂度
- 支持 **100K collections**，适合大规模应用

### 2. pymilvus 2.6+ SDK

**含义**：Milvus 官方 Python 客户端库，用于连接和操作 Milvus。

**为什么重要**：
- 提供 Pythonic API，易于集成到 RAG 系统
- 支持 MilvusClient（简化接口）和 connections（高级接口）
- 自动处理连接池、重连、序列化等底层细节

**2026 年最新特性**：
- 支持 **Embedding Functions**（内置向量化）
- 原生支持 **多向量检索**（Multi-Vector Search）
- 改进的 **异步 API**（async/await）

### 3. RAG 系统的向量数据库基础

**含义**：Milvus 在 RAG（Retrieval-Augmented Generation）系统中扮演向量存储和检索的核心角色。

**为什么重要**：
- RAG 需要高效的语义检索能力
- Milvus 提供毫秒级向量检索性能
- 支持标量过滤、混合检索等高级功能

**典型 RAG 流程**：
```
文档 → Embedding → Milvus 存储 → 用户查询 → Milvus 检索 → LLM 生成答案
```

## 快速验证

用 3 条命令验证你理解了核心概念：

```bash
# 1. 启动 Milvus（Docker Compose）
docker compose -f milvus-standalone-docker-compose.yml up -d

# 2. 安装 pymilvus
pip install pymilvus>=2.6.0

# 3. 连接并验证
python -c "from pymilvus import MilvusClient; client = MilvusClient('http://localhost:19530'); print('Connected:', client.list_collections())"
```

如果输出 `Connected: []`，说明你已经成功掌握了核心概念！

## 与其他向量数据库对比

| 特性 | Milvus 2.6 | Pinecone | Weaviate |
|------|-----------|----------|----------|
| **部署方式** | Docker Compose（本地） | 云服务（托管） | Docker/K8s |
| **Python SDK** | pymilvus | pinecone-client | weaviate-client |
| **RAG 集成** | 原生支持 | 原生支持 | 原生支持 |
| **开源** | ✅ 是 | ❌ 否 | ✅ 是 |
| **2026 新特性** | Woodpecker WAL | 无 | GraphQL API |

**选择 Milvus 的理由**：
- 开源免费，完全掌控数据
- 本地部署，无需依赖云服务
- 2.6 版本性能和易用性大幅提升
