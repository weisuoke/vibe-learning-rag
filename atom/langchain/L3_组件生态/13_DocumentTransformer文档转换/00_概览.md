### 1.2 为什么需要文档转换？

**问题场景：**

```python
from langchain_community.document_loaders import WebBaseLoader

# ❌ 直接加载的 HTML 文档充满噪声
loader = WebBaseLoader("https://example.com/article")
docs = loader.load()

print(docs[0].page_content[:200])
# 输出：<nav>Home | About | Contact</nav><div class="sidebar">广告...</div>
#       <article>这才是你真正需要的内容...</article><footer>版权信息</footer>

# 问题 1：HTML 标签和导航栏混入正文
# 问题 2：广告和侧边栏干扰语义
# 问题 3：直接向量化会严重影响检索质量
```

**DocumentTransformer 的解决方案：**

```python
from langchain_community.document_transformers import Html2TextTransformer

# ✅ 使用 DocumentTransformer 清洗文档
html2text = Html2TextTransformer()
clean_docs = html2text.transform_documents(docs)

print(clean_docs[0].page_content[:200])
# 输出：这才是你真正需要的内容...
# 干净、纯文本、适合向量化
```

## 二、核心概念地图

### 2.1 知识点全景图


## 四、重点与难点

### 4.1 核心重点

#### 重点 1：BaseDocumentTransformer 接口设计（⭐⭐⭐⭐⭐）

**为什么重要？**
- 所有转换器的统一抽象
- 理解接口才能正确使用和扩展
- 纯函数式设计：`Sequence[Document]` → `Sequence[Document]`

**核心用法：**

```python
from langchain_core.documents.transformers import BaseDocumentTransformer
from langchain_core.documents import Document

# 所有转换器都遵循同一接口
class MyTransformer(BaseDocumentTransformer):
    def transform_documents(self, documents, **kwargs):
        """输入文档序列，输出文档序列"""
        return [
            Document(
                page_content=doc.page_content.strip(),
                metadata=doc.metadata
            )
            for doc in documents
        ]
```

[来源: reference/source_document_transformer_01.md | LangChain 源码分析]

#### 重点 2：DocumentCompressorPipeline 组合模式（⭐⭐⭐⭐⭐）

**为什么重要？**
- 实际项目中几乎总是需要多步转换
- Pipeline 模式是 LangChain 推荐的组合方式
- 与 ContextualCompressionRetriever 无缝集成

**核心用法：**

```python
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain_community.document_transformers import (
    EmbeddingsRedundantFilter,
    LongContextReorder,
)

# 组合多个转换器
pipeline = DocumentCompressorPipeline(
    transformers=[
        EmbeddingsRedundantFilter(embeddings=embeddings),  # 先去重
        LongContextReorder(),                               # 再重排序
    ]
)
```

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

#### 重点 3：EmbeddingsRedundantFilter 冗余过滤（⭐⭐⭐⭐）

**为什么重要？**
- 检索结果经常包含语义重复的文档
- 冗余文档浪费 context window
- 过滤后能显著提升生成质量

**核心用法：**

```python
from langchain_community.document_transformers import EmbeddingsRedundantFilter
from langchain_openai import OpenAIEmbeddings

# 基于向量相似度过滤冗余文档
filter = EmbeddingsRedundantFilter(
    embeddings=OpenAIEmbeddings(),
    similarity_threshold=0.95  # 相似度超过 95% 视为冗余
)
unique_docs = filter.transform_documents(docs)
```

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

### 4.2 核心难点

#### 难点 1：理解"转换"的多种含义（⭐⭐⭐⭐）

**难在哪里？**
DocumentTransformer 不只是"转换内容"，它包含四种完全不同的操作：

| 操作类型 | 文档数量变化 | 内容变化 | 元数据变化 | 示例 |
|----------|-------------|---------|-----------|------|
| 清洗 | 不变 | 变 | 不变 | Html2TextTransformer |
| 过滤 | 减少 | 不变 | 不变 | EmbeddingsRedundantFilter |
| 增强 | 不变 | 不变 | 变 | OpenAIMetadataTagger |
| 重排序 | 不变 | 不变 | 不变 | LongContextReorder |

**突破方法：** 把 DocumentTransformer 理解为"文档序列的纯函数变换"，不限于内容修改。

#### 难点 2：选择合适的转换器组合（⭐⭐⭐⭐）

**难在哪里？**
- 11+ 种内置转换器，如何选择？
- 转换器的执行顺序影响结果
- 不同场景需要不同组合

**突破方法：** 遵循标准管道顺序：清洗 → 分块 → 过滤 → 增强 → 重排序

#### 难点 3：异步转换与性能优化（⭐⭐⭐）

**难在哪里？**
- 某些转换器（如 OpenAIMetadataTagger）需要调用 LLM，耗时较长
- 大量文档的批量转换需要异步处理
- 需要平衡转换质量和处理速度

**突破方法：** 使用 `atransform_documents()` 异步方法，结合批量处理策略。

## 五、实际应用场景

### 5.1 场景 1：HTML 文档清洗管道

**场景描述：**
从网页爬取的文档包含大量 HTML 标签、导航栏、广告等噪声，需要清洗后才能用于 RAG。

**实现方案：**
```python
from langchain_community.document_loaders import WebBaseLoader
from langchain_community.document_transformers import Html2TextTransformer

loader = WebBaseLoader("https://example.com/docs")
raw_docs = loader.load()

# 清洗 HTML
html2text = Html2TextTransformer()
clean_docs = html2text.transform_documents(raw_docs)
```

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

### 5.2 场景 2：冗余文档过滤与聚类

**场景描述：**
检索返回的文档中有大量语义重复的内容，需要去重后再送入 LLM。

**实现方案：**
```python
from langchain_community.document_transformers import (
    EmbeddingsRedundantFilter,
    EmbeddingsClusteringFilter,
)

# 方案 A：直接去重
filter = EmbeddingsRedundantFilter(embeddings=embeddings)
unique_docs = filter.transform_documents(retrieved_docs)

# 方案 B：聚类后选代表
cluster_filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=5,
    num_closest=1,
)
representative_docs = cluster_filter.transform_documents(retrieved_docs)
```

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

### 5.3 场景 3：自动元数据标注

**场景描述：**
文档缺少结构化元数据（如类别、主题、语言），需要用 LLM 自动标注。

**实现方案：**
```python
from langchain_community.document_transformers import OpenAIMetadataTagger
from langchain_openai import ChatOpenAI

schema = {
    "properties": {
        "category": {"type": "string", "enum": ["技术", "商业", "科学"]},
        "language": {"type": "string"},
    }
}

tagger = OpenAIMetadataTagger(
    metadata_schema=schema,
    llm=ChatOpenAI(model="gpt-4o-mini"),
)
tagged_docs = tagger.transform_documents(docs)
# 每个文档的 metadata 中自动添加 category 和 language 字段
```

### 5.4 场景 4：RAG 管道完整文档转换

**场景描述：**
构建完整的 RAG 管道，从加载到检索全流程使用 DocumentTransformer。

**实现方案：**
```python
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain.retrievers import ContextualCompressionRetriever

# 构建转换管道
pipeline = DocumentCompressorPipeline(
    transformers=[
        EmbeddingsRedundantFilter(embeddings=embeddings),
        LongContextReorder(),
    ]
)

# 集成到检索器

    base_compressor=pipeline,
    base_retriever=vectorstore.as_retriever(),
)

# 使用

```

## 七、常见误区与陷阱

### 误区 1：DocumentTransformer 只是"内容转换" ❌

**正确理解：** 它包含清洗、过滤、增强、重排序四种操作，不仅仅是修改文档内容。

### 误区 2：TextSplitter 和 DocumentTransformer 是两个东西 ❌

**正确理解：** TextSplitter 继承自 BaseDocumentTransformer，`split_documents()` 就是 `transform_documents()` 的特化。

### 误区 3：转换器的顺序无所谓 ❌

**正确理解：** 顺序很重要。先清洗再分块，先过滤再重排序，顺序错误会导致结果不佳。
