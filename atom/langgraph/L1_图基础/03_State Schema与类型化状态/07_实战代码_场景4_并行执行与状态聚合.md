# 实战代码 - 场景4：并行执行与状态聚合

> 完整可运行的并行执行与状态聚合示例

## 场景描述

在 LangGraph 中，多个节点可以并行执行，它们的输出需要通过 reducer 自动聚合。本场景演示如何使用 State Schema 和 Reducer 实现并行执行与状态聚合。

**应用场景**：
- 多数据源并行查询
- 并行 API 调用
- 分布式任务处理
- Fan-out/Fan-in 模式

## 核心知识点

1. **并行执行**：多个节点同时执行
2. **Reducer 聚合**：自动合并并行节点的输出
3. **Fan-out 模式**：一个节点触发多个并行节点
4. **Fan-in 模式**：多个并行节点汇聚到一个节点

## 完整代码

```python
from typing import TypedDict, Annotated
import operator
from langgraph.graph import StateGraph, START, END

# 1. 定义状态 Schema
class State(TypedDict):
    query: str
    results: Annotated[list, operator.add]  # 列表追加
    metadata: Annotated[dict, operator.or_]  # 字典合并

# 2. 定义并行节点
def search_database(state: State) -> dict:
    """从数据库搜索"""
    print(f"Searching database for: {state['query']}")
    return {
        "results": [
            {"source": "database", "content": "Database result 1"},
            {"source": "database", "content": "Database result 2"}
        ],
        "metadata": {"database_count": 2}
    }

def search_api(state: State) -> dict:
    """从 API 搜索"""
    print(f"Searching API for: {state['query']}")
    return {
        "results": [
            {"source": "api", "content": "API result 1"}
        ],
        "metadata": {"api_count": 1}
    }

def search_cache(state: State) -> dict:
    """从缓存搜索"""
    print(f"Searching cache for: {state['query']}")
    return {
        "results": [
            {"source": "cache", "content": "Cache result 1"}
        ],
        "metadata": {"cache_count": 1}
    }

def aggregate_results(state: State) -> dict:
    """聚合结果"""
    print(f"\nAggregated {len(state['results'])} results")
    print(f"Metadata: {state['metadata']}")
    return {}

# 3. 构建图
graph = StateGraph(State)

# 添加节点
graph.add_node("search_database", search_database)
graph.add_node("search_api", search_api)
graph.add_node("search_cache", search_cache)
graph.add_node("aggregate", aggregate_results)

# Fan-out: START 触发三个并行节点
graph.add_edge(START, "search_database")
graph.add_edge(START, "search_api")
graph.add_edge(START, "search_cache")

# Fan-in: 三个并行节点汇聚到 aggregate
graph.add_edge("search_database", "aggregate")
graph.add_edge("search_api", "aggregate")
graph.add_edge("search_cache", "aggregate")

# 结束
graph.add_edge("aggregate", END)

# 编译图
app = graph.compile()

# 4. 执行
if __name__ == "__main__":
    result = app.invoke({
        "query": "LangGraph tutorial",
        "results": [],
        "metadata": {}
    })

    print("\n=== Final State ===")
    print(f"Query: {result['query']}")
    print(f"Total results: {len(result['results'])}")
    print(f"Results: {result['results']}")
    print(f"Metadata: {result['metadata']}")
```

## 执行流程

```
START
  ├─> search_database ─┐
  ├─> search_api ──────┤
  └─> search_cache ────┴─> aggregate -> END
```

**Superstep 1**：三个搜索节点并行执行
**Superstep 2**：aggregate 节点执行

## 进阶示例：自定义 Reducer

```python
from typing import TypedDict, Annotated

def merge_results_with_dedup(existing: list, new: list) -> list:
    """去重合并结果"""
    seen = {r['content'] for r in existing}
    return existing + [r for r in new if r['content'] not in seen]

class AdvancedState(TypedDict):
    query: str
    results: Annotated[list, merge_results_with_dedup]  # 自定义 reducer
    metadata: Annotated[dict, operator.or_]

# 使用方式相同，但 results 会自动去重
```

## 并发控制

### 限制并行度

```python
from langgraph.graph import StateGraph

# 使用条件路由控制并行度
def route_to_sources(state: State) -> list[str]:
    """根据条件选择数据源"""
    sources = []
    if state.get("use_database"):
        sources.append("search_database")
    if state.get("use_api"):
        sources.append("search_api")
    if state.get("use_cache"):
        sources.append("search_cache")
    return sources

# 动态并行执行
graph.add_conditional_edges(
    START,
    route_to_sources
)
```

## 常见陷阱

### 陷阱1：Reducer 顺序依赖

```python
# 错误：假设 reducer 按特定顺序执行
def order_dependent_reducer(a: list, b: list) -> list:
    # 这个假设是错误的！
    return a + b  # 顺序不确定

# 正确：Reducer 应该是交换律的
def order_independent_reducer(a: list, b: list) -> list:
    return sorted(a + b, key=lambda x: x['id'])
```

### 陷阱2：可变对象共享

```python
# 错误：返回可变对象
def bad_node(state: State) -> dict:
    shared_list = []  # 危险！
    return {"results": shared_list}

# 正确：返回新对象
def good_node(state: State) -> dict:
    return {"results": []}  # 每次返回新列表
```

### 陷阱3：忘记初始化状态

```python
# 错误：未初始化列表
result = app.invoke({"query": "test"})  # KeyError: 'results'

# 正确：初始化所有字段
result = app.invoke({
    "query": "test",
    "results": [],
    "metadata": {}
})
```

## 最佳实践

1. **Reducer 幂等性**：确保 reducer 可以重复执行
2. **错误隔离**：一个节点失败不影响其他节点
3. **超时控制**：避免慢节点阻塞整个流程
4. **结果验证**：聚合后验证结果完整性
5. **日志记录**：记录每个节点的执行时间和结果
