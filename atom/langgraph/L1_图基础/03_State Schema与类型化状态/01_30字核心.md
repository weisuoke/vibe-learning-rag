# 30字核心

> State Schema与类型化状态的核心定义

## 核心定义

**TypedDict 定义状态结构，Annotated 指定 reducer，Channel 自动管理更新与聚合。**

## 核心拆解

### TypedDict 定义状态结构

```python
from typing import TypedDict

class State(TypedDict):
    messages: list
    user_id: str
    count: int
```

**本质**：用类型化字典定义图的数据结构。

### Annotated 指定 reducer

```python
from typing import Annotated
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]
```

**本质**：用元数据注解定义状态聚合规则。

### Channel 自动管理

```python
# StateGraph 自动为每个字段创建 Channel
# - LastValue: 存储最后一个值
# - BinaryOperatorAggregate: 使用 reducer 聚合
```

**本质**：底层机制，自动处理状态更新和聚合。

### 更新与聚合

```python
def node(state: State) -> dict:
    return {"messages": ["new message"]}

# StateGraph 自动应用 reducer 合并状态
```

**本质**：节点返回部分状态，自动与当前状态合并。

## 为什么这样设计

1. **类型安全**：TypedDict 提供编译时类型检查
2. **声明式**：Annotated 声明聚合规则，无需手动编写
3. **自动化**：Channel 自动处理复杂的状态管理
4. **灵活性**：支持多种 reducer，适应不同场景

## 一句话记忆

**TypedDict 定义结构，Annotated 定义规则，Channel 自动执行。**

## 快速示例

```python
from typing import TypedDict, Annotated
import operator
from langgraph.graph import StateGraph

# 1. 定义状态结构
class State(TypedDict):
    messages: Annotated[list, operator.add]  # 列表追加
    user_id: str  # 最后一个值

# 2. 创建图
graph = StateGraph(State)

# 3. 添加节点
def node1(state: State) -> dict:
    return {"messages": ["hello"]}

def node2(state: State) -> dict:
    return {"messages": ["world"]}

graph.add_node("node1", node1)
graph.add_node("node2", node2)

# 4. 并行执行，自动聚合
# messages 会自动合并为 ["hello", "world"]
```

## 核心价值

- **简化开发**：无需手动管理状态合并
- **提高可靠性**：类型检查减少错误
- **支持并发**：自动处理并行节点的状态聚合
- **易于维护**：清晰的状态结构和聚合规则
