## 完整代码（50行）

```python
"""
最小可用 ReAct Agent
演示：Thought→Action→Observation 循环的核心机制

运行方式：python examples/minimal_react_agent.py
"""

import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import PromptTemplate
from langchain.agents import create_react_agent, AgentExecutor

# 加载环境变量
load_dotenv()

# 1. 初始化 LLM
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 2. 定义工具（Agent 的"手和脚"）
@tool
def calculator(expression: str) -> str:
    """计算数学表达式。输入示例：'2 + 3' 或 'sqrt(16)'"""
    try:
        import math
        result = eval(expression, {"__builtins__": {}}, {"sqrt": math.sqrt, "pow": pow})
        return f"结果：{result}"
    except Exception as e:
        return f"计算错误：{e}"

@tool
def search(query: str) -> str:
    """搜索知识库。输入示例：'Python 是什么'"""
    kb = {
        "python": "Python 是一种高级编程语言，由 Guido van Rossum 于 1991 年创建。",
        "langchain": "LangChain 是用于构建 LLM 应用的开源框架，支持 Agent、RAG 等模式。"
    }
    for key, value in kb.items():
        if key in query.lower():
            return value
    return f"未找到 '{query}' 的相关信息"

tools = [calculator, search]

# 3. 定义 ReAct Prompt（告诉 LLM 如何推理）
prompt = PromptTemplate.from_template("""Answer the following question. You have access to these tools:

{tools}

Use this format:
Question: the input question
Thought: think about what to do
Action: one of [{tool_names}]
Action Input: the input to the action
Observation: the result of the action
... (repeat Thought/Action/Observation as needed)
Thought: I now know the final answer
Final Answer: the final answer

Question: {input}
Thought:{agent_scratchpad}""")

# 4. 创建 Agent 和 Executor
agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True, max_iterations=3)

# 5. 运行示例
print("=" * 60)
print("示例：多步推理（搜索 + 计算）")
print("=" * 60)

result = agent_executor.invoke({
    "input": "Python 是哪一年创建的？从那年到 2025 年过了多少年？"
})

print(f"\n最终答案：{result['output']}")
```

## 关键机制解析

### 1. ReAct 循环的本质

```
用户提问
  ↓
Thought: "我需要先搜索 Python 的创建年份"  ← LLM 推理
  ↓
Action: search                              ← LLM 选择工具
  ↓
Action Input: "Python"                      ← LLM 提供参数
  ↓
Observation: "Python 创建于 1991 年"        ← 工具返回结果（真实的！）
  ↓
Thought: "现在需要计算年份差"                ← LLM 基于结果继续推理
  ↓
Action: calculator                          ← LLM 选择新工具
  ↓
... 循环继续，直到 LLM 输出 "Final Answer"
```

### 2. create_react_agent 做了什么？

```python
# 内部逻辑（简化版）：
def create_react_agent(llm, tools, prompt):
    # 1. 绑定 stop sequence，防止 LLM 自己编造 Observation
    llm_with_stop = llm.bind(stop=["\nObservation"])

    # 2. 构建 LCEL 链
    agent = (
        RunnablePassthrough.assign(
            agent_scratchpad=lambda x: format_log_to_str(x["intermediate_steps"])
        )
        | prompt  # 填充 prompt 模板
        | llm_with_stop  # 调用 LLM（到 Observation 就停）
        | ReActSingleInputOutputParser()  # 解析输出
    )
    return agent
```

### 3. AgentExecutor 的循环控制

```python
# AgentExecutor 的核心循环（伪代码）：
while True:
    # 调用 agent，得到决策
    decision = agent.invoke({"input": question, "intermediate_steps": steps})

    if isinstance(decision, AgentFinish):
        # LLM 认为可以回答了 → 停止循环
        return decision.return_values["output"]

    elif isinstance(decision, AgentAction):
        # LLM 想调用工具 → 执行工具，继续循环
        tool_result = execute_tool(decision.tool, decision.tool_input)
        steps.append((decision, tool_result))

    if len(steps) >= max_iterations:
        # 防止无限循环
        raise MaxIterationsError()
```

### 4. 为什么需要 stop=["\nObservation"]？

```python
# 没有 stop sequence 时，LLM 可能会这样：
"""
Thought: 我需要搜索 Python
Action: search
Action Input: Python
Observation: Python 创建于 1991 年  ← LLM 自己编的！不是真实结果！
Thought: 好的，1991 年...          ← 基于幻觉继续推理
"""

# 有 stop sequence 时：
"""
Thought: 我需要搜索 Python
Action: search
Action Input: Python
[STOP]  ← LLM 在这里被强制停止，等待真实工具返回
"""
# 然后 AgentExecutor 执行真实的 search 工具，把结果追加到 scratchpad
```

## 核心要点

| 要点 | 说明 |
|------|------|
| **ReAct 循环** | Thought → Action → Observation，不断重复直到 Final Answer |
| **工具定义** | 用 `@tool` 装饰器定义，描述要清晰（LLM 靠描述选工具） |
| **Prompt 格式** | 必须包含 `{tools}`, `{tool_names}`, `{input}`, `{agent_scratchpad}` |
| **Stop Sequence** | 防止 LLM 自己编造 Observation，确保使用真实工具结果 |
| **AgentExecutor** | 负责循环控制、工具执行、错误处理 |
| **verbose=True** | 开启后可以看到每一步推理过程，调试必备 |

## 常见问题

**Q1：Agent 选错了工具怎么办？**

改善工具描述，让 LLM 更容易选对：
```python
@tool
def calculator(expression: str) -> str:
    """计算数学表达式。
    适用：数学运算、数值计算
    不适用：搜索、查询事实
    输入示例：'2 + 3' 或 'sqrt(16)'"""
    ...
```

**Q2：如何查看完整的 Prompt？**

```python
from langchain.agents.format_scratchpad import format_log_to_str

full_prompt = prompt.format(
    tools="calculator: 计算\nsearch: 搜索",
    tool_names="calculator, search",
    input="你的问题",
    agent_scratchpad="",
)
print(full_prompt)
```

**Q3：Agent 陷入无限循环怎么办？**

设置 `max_iterations` 和 `max_execution_time` 限制。
