# 30字核心 - 部分状态更新

> 用一句话说清楚：部分状态更新到底是什么？

---

## 30字核心定义

**部分状态更新是节点函数只返回需要修改的字段，未返回字段保持原值，通过 Reducer 函数实现增量合并的机制。**

[来源: reference/source_部分状态更新_01.md, reference/context7_langgraph_01.md]

---

## 为什么这个定义准确？

### 1. 抓住了核心本质

这个定义包含了部分状态更新的三个核心要素：

**要素 1：节点函数只返回需要修改的字段**
- 这是"部分"的含义
- 节点不需要返回完整状态
- 只关注自己负责的字段

**要素 2：未返回字段保持原值**
- 这是"更新"的含义
- 不是替换整个状态
- 保留其他节点的工作成果

**要素 3：通过 Reducer 函数实现增量合并**
- 这是"机制"的核心
- 不是简单覆盖
- 支持累加、追加等操作

[来源: reference/search_部分状态更新_01.md]

### 2. 区分了关键概念

**vs 完整状态更新：**
```python
# 完整状态更新（不推荐）
def node_full(state: State) -> State:
    return {
        "messages": state["messages"],
        "counter": state["counter"] + 1,
        "analysis": state["analysis"]
    }

# 部分状态更新（推荐）
def node_partial(state: State) -> dict:
    return {"counter": state["counter"] + 1}
```

**vs 状态覆盖：**
```python
# 状态覆盖（没有 Reducer）
class State(TypedDict):
    messages: list[str]  # 会被覆盖

# 增量合并（有 Reducer）
class State(TypedDict):
    messages: Annotated[list[str], operator.add]  # 会追加
```

[来源: reference/context7_langgraph_01.md]

### 3. 突出了实际价值

**价值 1：简化节点逻辑**
- 节点只需要知道自己负责的字段
- 不需要了解完整状态结构
- 降低耦合度

**价值 2：避免状态丢失**
- 不会因为遗漏字段而丢失数据
- 多个节点可以独立更新不同字段
- 提高系统可靠性

**价值 3：支持增量操作**
- 可以累加数值
- 可以追加列表
- 可以合并字典

[来源: reference/search_部分状态更新_01.md]

## 定义的深入解读

### 第一部分：节点函数只返回需要修改的字段

**核心思想：** 节点函数返回字典，只包含需要更新的字段

```python
from typing import TypedDict

class State(TypedDict):
    messages: list[str]
    counter: int
    analysis: str
    summary: str

def analyze_node(state: State) -> dict:
    # 只返回 analysis 字段
    result = analyze(state["messages"])
    return {"analysis": result}
    # messages, counter, summary 保持不变

def summarize_node(state: State) -> dict:
    # 只返回 summary 字段
    result = summarize(state["analysis"])
    return {"summary": result}
    # messages, counter, analysis 保持不变
```

[来源: reference/source_部分状态更新_01.md]

**关键特性：**
- 返回值是字典（dict）
- 只包含需要更新的字段
- 未返回的字段保持原值
- 节点职责清晰

**实际应用：**
```python
# 多步推理工作流
def step1_node(state):
    return {"thoughts": ["第一步思考"]}

def step2_node(state):
    return {"thoughts": ["第二步思考"]}  # 追加而非覆盖

def conclude_node(state):
    return {"answer": "最终答案"}  # 不影响 thoughts
```

[来源: reference/context7_langgraph_01.md]

### 第二部分：未返回字段保持原值

**核心思想：** LangGraph 自动保留未更新的字段

```python
# 初始状态
initial_state = {
    "messages": ["Hello"],
    "counter": 0,
    "analysis": ""
}

# 节点 A 只更新 counter
def node_a(state):
    return {"counter": 1}

# 执行后的状态
# {
#     "messages": ["Hello"],  # 保持不变
#     "counter": 1,           # 已更新
#     "analysis": ""          # 保持不变
# }
```

[来源: reference/source_部分状态更新_01.md]

**关键特性：**
- 自动保留未返回的字段
- 不需要手动复制状态
- 避免状态丢失
- 支持并行更新

**实际应用：**
```python
# 并行节点更新不同字段
def node_a(state):
    return {"field_a": "value_a"}

def node_b(state):
    return {"field_b": "value_b"}

# 两个节点可以并行执行，互不影响
```

[来源: reference/search_部分状态更新_01.md]

### 第三部分：通过 Reducer 函数实现增量合并

**核心思想：** 使用 Reducer 函数定义如何合并当前值和新值

```python
from typing import Annotated
import operator

class State(TypedDict):
    # 使用 operator.add 作为 Reducer
    messages: Annotated[list[str], operator.add]
    # 没有 Reducer，使用覆盖策略
    counter: int

def node_a(state):
    return {
        "messages": ["新消息"],  # 追加到列表
        "counter": 1             # 覆盖原值
    }
```

[来源: reference/context7_langgraph_01.md]

**关键特性：**
- `Annotated[type, reducer]` 语法
- Reducer 接收两个参数：当前值和新值
- 返回合并后的值
- 支持自定义 Reducer

**常用 Reducer：**

**1. operator.add - 加法合并**
```python
from typing import Annotated
import operator

class State(TypedDict):
    # 数值累加
    count: Annotated[int, operator.add]
    # 列表拼接
    items: Annotated[list[str], operator.add]

# 使用示例
def node_a(state):
    return {"count": 1, "items": ["a"]}

def node_b(state):
    return {"count": 2, "items": ["b"]}

# 最终状态：{"count": 3, "items": ["a", "b"]}
```

[来源: reference/source_部分状态更新_01.md]

**2. add_messages - 消息列表专用**
```python
from typing import Annotated, Sequence
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]

# 使用示例
def chat_node(state):
    return {"messages": [AIMessage(content="回复")]}
    # 追加而非替换
```

[来源: reference/context7_langgraph_01.md]

**3. 自定义 Reducer**
```python
def merge_dicts(current: dict, new: dict) -> dict:
    """合并字典"""
    return {**current, **new}

class State(TypedDict):
    metadata: Annotated[dict, merge_dicts]
```

[来源: reference/search_部分状态更新_01.md]

## 与其他概念的对比

### 1. 部分状态更新 vs 完整状态返回

| 特性 | 部分状态更新 | 完整状态返回 |
|------|-------------|-------------|
| 返回内容 | 只返回需要更新的字段 | 返回所有字段 |
| 节点职责 | 只关注自己负责的字段 | 需要了解完整状态 |
| 状态丢失风险 | 低（自动保留） | 高（容易遗漏） |
| 代码复杂度 | 低 | 高 |
| 推荐程度 | ✅ 推荐 | ❌ 不推荐 |

[来源: reference/search_部分状态更新_01.md]

### 2. Reducer 合并 vs 简单覆盖

| 特性 | Reducer 合并 | 简单覆盖 |
|------|-------------|---------|
| 定义方式 | `Annotated[type, reducer]` | `type` |
| 更新策略 | 自定义合并逻辑 | 直接覆盖 |
| 适用场景 | 累加、追加、合并 | 替换、重置 |
| 示例 | `Annotated[list, operator.add]` | `list` |

[来源: reference/context7_langgraph_01.md]

### 3. 部分更新 vs Pydantic 模型更新

| 特性 | 字典部分更新 | Pydantic 模型更新 |
|------|-------------|------------------|
| 数据结构 | 字典 | Pydantic 模型 |
| 字段跟踪 | 手动指定 | 自动跟踪（model_fields_set） |
| 类型验证 | 无 | 有 |
| 默认值处理 | 无 | 自动跳过 |

[来源: reference/source_部分状态更新_01.md]

## 实际应用场景

### 场景 1：多步推理工作流

```python
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
import operator

class ReasoningState(TypedDict):
    question: str
    thoughts: Annotated[list[str], operator.add]
    answer: str

def think_step1(state):
    # 只返回第一步思考
    return {"thoughts": ["分析问题：" + state["question"]]}

def think_step2(state):
    # 只返回第二步思考
    return {"thoughts": ["推理过程：基于已有信息..."]}

def think_step3(state):
    # 只返回第三步思考
    return {"thoughts": ["得出结论：..."]}

def conclude(state):
    # 只返回最终答案
    all_thoughts = "\n".join(state["thoughts"])
    return {"answer": f"基于以下思考：\n{all_thoughts}\n答案是..."}

# 构建图
graph = StateGraph(ReasoningState)
graph.add_node("step1", think_step1)
graph.add_node("step2", think_step2)
graph.add_node("step3", think_step3)
graph.add_node("conclude", conclude)
graph.add_edge(START, "step1")
graph.add_edge("step1", "step2")
graph.add_edge("step2", "step3")
graph.add_edge("step3", "conclude")
graph.add_edge("conclude", END)

app = graph.compile()

# 运行
result = app.invoke({"question": "什么是 LangGraph？", "thoughts": [], "answer": ""})
print(result)
# {
#     "question": "什么是 LangGraph？",
#     "thoughts": [
#         "分析问题：什么是 LangGraph？",
#         "推理过程：基于已有信息...",
#         "得出结论：..."
#     ],
#     "answer": "基于以下思考：\n...\n答案是..."
# }
```

[来源: reference/context7_langgraph_01.md]

### 场景 2：对话历史维护

```python
from typing import Annotated, Sequence
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph.message import add_messages
from langgraph.graph import StateGraph, START, END

class ChatState(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]

def chatbot(state):
    # 只返回新消息
    last_message = state["messages"][-1]
    response = f"你说：{last_message.content}"
    return {"messages": [AIMessage(content=response)]}

# 构建图
graph = StateGraph(ChatState)
graph.add_node("chatbot", chatbot)
graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)

app = graph.compile()

# 第一轮对话
result1 = app.invoke({"messages": [HumanMessage(content="你好")]})
print(result1["messages"])
# [HumanMessage(content="你好"), AIMessage(content="你说：你好")]

# 第二轮对话（继续追加）
result2 = app.invoke({
    "messages": result1["messages"] + [HumanMessage(content="再见")]
})
print(result2["messages"])
# [
#     HumanMessage(content="你好"),
#     AIMessage(content="你说：你好"),
#     HumanMessage(content="再见"),
#     AIMessage(content="你说：再见")
# ]
```

[来源: reference/context7_langgraph_01.md]

### 场景 3：增量数据累积

```python
from typing import Annotated
import operator

class CounterState(TypedDict):
    count: Annotated[int, operator.add]
    total: Annotated[float, operator.add]
    items: Annotated[list[str], operator.add]

def process_item_a(state):
    return {
        "count": 1,
        "total": 10.5,
        "items": ["item_a"]
    }

def process_item_b(state):
    return {
        "count": 1,
        "total": 20.3,
        "items": ["item_b"]
    }

def process_item_c(state):
    return {
        "count": 1,
        "total": 15.7,
        "items": ["item_c"]
    }

# 构建图
graph = StateGraph(CounterState)
graph.add_node("a", process_item_a)
graph.add_node("b", process_item_b)
graph.add_node("c", process_item_c)
graph.add_edge(START, "a")
graph.add_edge("a", "b")
graph.add_edge("b", "c")
graph.add_edge("c", END)

app = graph.compile()

# 运行
result = app.invoke({"count": 0, "total": 0.0, "items": []})
print(result)
# {
#     "count": 3,
#     "total": 46.5,
#     "items": ["item_a", "item_b", "item_c"]
# }
```

## 核心要点总结

### 1. 三个核心要素

**要素 1：部分返回**
- 节点函数只返回需要更新的字段
- 返回值是字典（dict）
- 未返回的字段保持原值

**要素 2：自动保留**
- LangGraph 自动保留未更新的字段
- 不需要手动复制状态
- 避免状态丢失

**要素 3：增量合并**
- 使用 Reducer 函数定义合并逻辑
- 支持累加、追加、合并等操作
- 没有 Reducer 则使用覆盖策略

[来源: reference/source_部分状态更新_01.md, reference/context7_langgraph_01.md]

### 2. 两种更新策略

**策略 1：覆盖（默认）**
```python
class State(TypedDict):
    counter: int  # 没有 Reducer，使用覆盖

def node(state):
    return {"counter": 10}  # 直接覆盖原值
```

**策略 2：合并（Reducer）**
```python
class State(TypedDict):
    counter: Annotated[int, operator.add]  # 有 Reducer，使用合并

def node(state):
    return {"counter": 10}  # 累加到原值
```

[来源: reference/context7_langgraph_01.md]

### 3. 三个最佳实践

**实践 1：节点应该是纯函数**
```python
# ✅ 好的实践
def good_node(state: State) -> dict:
    result = process(state["input"])
    return {"output": result}

# ❌ 坏的实践
def bad_node(state: State) -> dict:
    state["output"] = process(state["input"])  # 直接修改状态
    return state
```

**实践 2：明确指定 Reducer**
```python
# ✅ 好的实践
class State(TypedDict):
    messages: Annotated[list[str], operator.add]  # 明确追加

# ❌ 坏的实践
class State(TypedDict):
    messages: list[str]  # 不清楚是追加还是覆盖
```

**实践 3：避免过度使用 Overwrite**
```python
# ✅ 好的实践：只在必要时使用
def reset_node(state):
    if state["error"]:
        return {"messages": Overwrite([])}  # 明确需要重置
    return {"messages": [new_message]}

# ❌ 坏的实践：滥用 Overwrite
def bad_node(state):
    return {"messages": Overwrite([new_message])}  # 不必要
```
