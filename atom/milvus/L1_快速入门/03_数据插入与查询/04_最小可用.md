## 最小可用代码

### 完整示例（50行代码）

```python
"""
Milvus 2.6 最小可用示例：文档问答
演示 Embedding Functions 的 Data-in, Data-out 模式
"""

import os
from dotenv import load_dotenv
from pymilvus import MilvusClient, DataType, Function, FunctionType

# 1. 加载环境变量
load_dotenv()

# 2. 初始化 Milvus 客户端
client = MilvusClient(uri="./milvus_demo.db")  # 使用 Milvus Lite

# 3. 创建 Collection Schema
schema = client.create_schema()

# 添加字段
schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
schema.add_field("document", DataType.VARCHAR, max_length=5000)
schema.add_field("dense", DataType.FLOAT_VECTOR, dim=1536)

# 4. 配置 Embedding Function（核心！）
embedding_function = Function(
    name="openai_embedding",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["document"],
    output_field_names=["dense"],
    params={
        "provider": "openai",
        "model_name": "text-embedding-3-small",
    }
)
schema.add_function(embedding_function)

# 5. 创建索引
index_params = client.prepare_index_params()
index_params.add_index(
    field_name="dense",
    index_type="AUTOINDEX",
    metric_type="COSINE"
)

# 6. 创建 Collection
collection_name = "doc_qa_demo"
if client.has_collection(collection_name):
    client.drop_collection(collection_name)

client.create_collection(
    collection_name=collection_name,
    schema=schema,
    index_params=index_params
)

# 7. 插入文档（直接插入原始文本！）
documents = [
    {"document": "Milvus is an open-source vector database designed for AI applications."},
    {"document": "Milvus 2.6 introduces Embedding Functions for automatic vectorization."},
    {"document": "The Data-in, Data-out pattern simplifies RAG development significantly."},
    {"document": "You can use OpenAI, Cohere, Bedrock, and other providers with Milvus."},
    {"document": "Milvus supports hybrid search combining dense and sparse vectors."},
]

client.insert(collection_name=collection_name, data=documents)
print(f"✅ 插入了 {len(documents)} 条文档")

# 8. 查询（直接用文本查询！）
query = "What is Milvus 2.6's new feature?"

results = client.search(
    collection_name=collection_name,
    data=[query],  # 直接传入文本，不是向量！
    anns_field="dense",
    limit=3,
    output_fields=["document"]
)

# 9. 显示结果
print(f"\n🔍 查询: {query}\n")
print("📄 最相关的文档:\n")
for i, hit in enumerate(results[0], 1):
    print(f"{i}. [相似度: {hit['distance']:.4f}]")
    print(f"   {hit['entity']['document']}\n")

# 10. 清理（可选）
# client.drop_collection(collection_name)
```

## 代码解析

### 核心步骤

```python
# 步骤1: 配置 Embedding Function
embedding_function = Function(
    name="openai_embedding",           # 函数名称
    function_type=FunctionType.TEXTEMBEDDING,  # 类型：文本嵌入
    input_field_names=["document"],    # 输入字段：原始文本
    output_field_names=["dense"],      # 输出字段：向量
    params={
        "provider": "openai",          # 提供商
        "model_name": "text-embedding-3-small",  # 模型
    }
)
schema.add_function(embedding_function)  # 添加到 schema

# 步骤2: 插入原始文本（自动嵌入）
client.insert(collection_name, data=[
    {"document": "原始文本"}  # 只需提供文本，不需要向量！
])

# 步骤3: 用文本查询（自动嵌入）
results = client.search(
    collection_name,
    data=["查询文本"],  # 只需提供文本，不需要向量！
    anns_field="dense",
    limit=3
)
```

### 关键点说明

**1. Schema 定义**
```python
schema.add_field("document", DataType.VARCHAR, max_length=5000)  # 原始文本字段
schema.add_field("dense", DataType.FLOAT_VECTOR, dim=1536)       # 向量字段
```
- `document`: 存储原始文本
- `dense`: 存储向量（由 Embedding Function 自动生成）

**2. Embedding Function 配置**
```python
params={
    "provider": "openai",                      # 使用 OpenAI
    "model_name": "text-embedding-3-small",    # 模型名称
}
```
- 配置一次，全局生效
- 插入和查询自动使用同一模型

**3. 自动向量化**
```python
# 插入时：document → Milvus 调用 OpenAI → 生成向量 → 存储到 dense 字段
client.insert(data=[{"document": "文本"}])

# 查询时：查询文本 → Milvus 调用 OpenAI → 生成查询向量 → 搜索
client.search(data=["查询文本"])
```

## 对比：传统方式 vs Milvus 2.6

### 传统方式（需要手动嵌入）

```python
from openai import OpenAI

openai_client = OpenAI()

# 插入时：手动生成嵌入
docs = ["Milvus is a vector database"]
embeddings = []
for doc in docs:
    emb = openai_client.embeddings.create(
        input=doc,
        model="text-embedding-3-small"
    )
    embeddings.append(emb.data[0].embedding)

# 手动插入向量
milvus_client.insert({
    "id": 1,
    "vector": embeddings[0],
    "text": docs[0]
})

# 查询时：手动生成查询向量
query = "What is Milvus?"
query_emb = openai_client.embeddings.create(
    input=query,
    model="text-embedding-3-small"
).data[0].embedding

# 手动搜索
results = milvus_client.search(data=[query_emb])
```

**代码量**: ~25行

### Milvus 2.6 方式（自动嵌入）

```python
# 配置一次（在创建 collection 时）
schema.add_function(embedding_function)

# 插入：直接插入文本
milvus_client.insert({"document": "Milvus is a vector database"})

# 查询：直接用文本查询
results = milvus_client.search(data=["What is Milvus?"])
```

**代码量**: ~5行

**减少**: 80% 代码量

## 常见问题

### Q1: 为什么需要配置 OpenAI API Key？

**A**: Embedding Functions 需要调用 OpenAI API 生成向量。如果你使用其他提供商（如 Cohere, Bedrock），需要配置相应的凭证。

### Q2: 可以使用本地嵌入模型吗？

**A**: 可以！Milvus 2.6 支持多种提供商：
- **云服务**: OpenAI, Cohere, Bedrock, Voyage, Jina
- **本地模型**: Sentence Transformers, Hugging Face TEI

示例（使用 Sentence Transformers）：
```python
# 需要先安装: pip install sentence-transformers
params={
    "provider": "sentence_transformers",
    "model_name": "all-MiniLM-L6-v2",
}
```

### Q3: Milvus Lite 和 Milvus Standalone 有什么区别？

**A**:
- **Milvus Lite**: 嵌入式版本，数据存储在本地文件，适合学习和原型开发
- **Milvus Standalone**: 独立服务，使用 Docker 部署，适合开发和小规模生产
- **Milvus Cluster**: 分布式集群，适合大规模生产环境

### Q4: 如何查看生成的向量？

**A**: 向量由 Milvus 自动生成和管理，通常不需要查看。如果需要调试：
```python
results = client.query(
    collection_name=collection_name,
    filter="id > 0",
    output_fields=["document", "dense"],
    limit=1
)
print(results[0]["dense"])  # 查看向量
```

### Q5: 插入大量文档时性能如何？

**A**: Milvus 会批量调用嵌入 API 以提升性能。建议：
- 单次插入 100-1000 条文档
- 使用批量插入而非逐条插入
- 监控 API 调用成本
