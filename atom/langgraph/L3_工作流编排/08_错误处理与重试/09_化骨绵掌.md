## 卡片 1：RetryPolicy 参数速查

```python
RetryPolicy(
    max_attempts=3,          # 总尝试次数（含首次）
    initial_interval=0.5,    # 首次重试等待秒数
    backoff_factor=2.0,      # 间隔倍增因子
    max_interval=128.0,      # 重试间隔上限（秒）
    jitter=True,             # 是否添加随机抖动
    retry_on=default_retry_on  # 异常匹配策略
)
```

**记忆口诀：** 最多试几次，首次等多久，倍增多少倍，上限多少秒，要不要抖动，重试啥异常。

## 卡片 2：退避时间计算公式

```python
interval = min(
    max_interval,
    initial_interval * (backoff_factor ** (attempts - 1))
)

# 启用 jitter 时：
sleep_time = interval + random.uniform(0, 1)
```

**示例（默认参数）：**
```
0.5s → 1.0s → 2.0s → 4.0s → 8.0s → ... → 128.0s
```

## 卡片 3：默认重试行为

```python
# ✅ 会重试
- ConnectionError          # 网络连接错误
- HTTP 5xx                 # 服务端错误
- 其他未知异常             # 保守策略

# ❌ 不会重试
- ValueError, TypeError    # 编程错误
- ImportError, NameError   # 代码错误
- OSError, RuntimeError    # 系统错误
- HTTP 4xx                 # 客户端错误
```

```python
# 方式 1：单一类型
RetryPolicy(retry_on=ConnectionError)

# 方式 2：类型序列
RetryPolicy(retry_on=(ConnectionError, TimeoutError))

    return isinstance(exc, ConnectionError)
```

## 卡片 7：断路器三种状态

```
CLOSED（关闭）
  ↓ 失败次数 >= 阈值
OPEN（打开）
  ↓ 超时后
HALF_OPEN（半开）
  ↓ 成功次数 >= 阈值
CLOSED（关闭）
```

**配置参数：**
- `failure_threshold`: 触发断路的失败次数
- `timeout`: 断路器保持打开的时长
- `success_threshold`: 恢复需要的成功次数

```python
- 平均执行时间

- Fallback 率 > 10% → P2 告警

- 执行时间 > 10s → P3 告警
```

## 快速决策树

```
遇到异常
  │
  ├─ 是 GraphBubbleUp？
  │   └─ 是 → 直接抛出（不重试）
  │
  ├─ 是编程错误（ValueError）？
  │   └─ 是 → 直接抛出（不重试）
  │
  ├─ 是网络错误或 5xx？
  │   └─ 是 → 重试
  │       ├─ 尝试次数 < max_attempts？
  │       │   └─ 是 → 退避等待 → 重试
  │       └─ 否 → 尝试 Fallback
  │           ├─ 有备用方案？
  │           │   └─ 是 → 执行 Fallback
  │           └─ 否 → 抛出异常
  │
  └─ 其他异常
      └─ 根据 retry_on 判断
```

## 常见错误速查

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 重试次数过多 | max_attempts 设置过大 | 降低到 3-5 次 |
| 等待时间过长 | max_interval 未设置 | 设置为 60s 或更小 |
| 重试风暴 | 未启用 jitter | 设置 `jitter=True` |
| 编程错误被重试 | 未正确配置 retry_on | 使用 default_retry_on |
| 所有异常都不重试 | retry_on 返回 False | 检查自定义函数逻辑 |
| 断路器一直打开 | timeout 设置过长 | 降低到 30-60s |

## 记忆技巧

1. **max_attempts**：记住是"总尝试次数"，不是"重试次数"
2. **指数退避**：记住公式 `initial * (factor ** (n-1))`
3. **jitter**：记住"防止重试风暴"
4. **default_retry_on**：记住"不重试编程错误"
5. **GraphBubbleUp**：记住"流程控制，不重试"
6. **断路器**：记住"三种状态：关闭、打开、半开"
7. **Fallback**：记住"降级链：主 → 备 → 缓存 → 默认"
