## 🔑 核心知识点预览

### 消息发送的本质

OpenClaw 的消息发送不是简单的 API 调用，而是一个完整的 **Gateway 架构**：

```
用户命令 → CLI → Message Command → Action Runner → Channel Selection → Outbound Service → Channel Plugin → 目标平台
```

**关键特性**：
- 🔐 **安全配对**：通过 DM 配对确保消息发送到正确的接收者
- 🔄 **双向通信**：支持发送消息和接收响应
- 🌐 **多通道支持**：统一接口操作 WhatsApp、Telegram、Discord 等
- 🤖 **Agent 集成**：消息可以触发 Agent 自动响应

### 快速开始命令

```bash
# 1. 发送第一条消息（需要先配对）
openclaw message send "Hello from OpenClaw!"

# 2. 查看配对状态
openclaw pairing list

# 3. 启动 Agent 响应
openclaw agent start

# 4. 检查系统状态
openclaw doctor
```

## ⚠️ 常见问题预警

在学习本知识点时，你可能会遇到以下问题：

1. **配对失败** - 确保你已经正确配置了通道插件
2. **消息发送失败** - 检查 `openclaw doctor` 输出的系统状态
3. **Agent 无响应** - 确认 Agent 已启动且配置正确
4. **权限错误** - 某些通道需要特定的 API 权限

详细的故障排查方法请参考 [07_实战代码_场景5_故障排查](./07_实战代码_场景5_故障排查.md)。
