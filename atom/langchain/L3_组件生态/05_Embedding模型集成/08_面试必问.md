## 一、Embeddings 抽象接口设计

### 问题1：LangChain 为什么要将 embed_documents 和 embed_query 分离？

**标准答案：**

LangChain 将 Embedding 操作分为两个方法的原因：

1. **语义区分**：文档嵌入用于索引构建（批量操作），查询嵌入用于检索（单个操作）
2. **优化空间**：某些模型对文档和查询使用不同的 prompt 或优化策略
3. **灵活性**：允许未来扩展，如查询重写、查询增强等
4. **批量优化**：embed_documents 支持批量处理，embed_query 针对单个查询优化

### 问题2：LangChain Embeddings 的异步方法是如何设计的？

**标准答案：**

LangChain 的异步方法采用"渐进式异步"设计：

1. **默认实现**：使用 `run_in_executor` 将同步方法包装为异步
2. **可选优化**：子类可以覆盖异步方法，提供原生异步实现
3. **向后兼容**：旧代码只需实现同步方法即可工作
4. **性能提升**：原生异步实现可以避免线程池开销，提高并发性能

```python
async def aembed_documents(self, texts: list[str]) -> list[list[float]]:
    return await run_in_executor(None, self.embed_documents, texts)
```

### 问题3：如何实现一个自定义的 Embeddings 类？

**标准答案：**

实现自定义 Embeddings 类需要以下步骤：

1. **继承基类**：继承 `langchain_core.embeddings.Embeddings`
2. **实现抽象方法**：必须实现 `embed_documents` 和 `embed_query`
3. **可选优化**：可以覆盖 `aembed_documents` 和 `aembed_query` 提供原生异步
4. **集成测试**：确保与 LangChain 生态兼容

```python
from langchain_core.embeddings import Embeddings

class CustomEmbeddings(Embeddings):
    def embed_documents(self, texts: list[str]) -> list[list[float]]:
        # 实现批量嵌入逻辑
        return [self._embed_single(text) for text in texts]

    def embed_query(self, text: str) -> list[float]:
        # 实现单个查询嵌入逻辑
        return self._embed_single(text)
```

### 问题4：OpenAI text-embedding-3 系列模型如何选择？dimensions 参数有什么作用？

**标准答案：**

OpenAI text-embedding-3 系列模型选择策略：

1. **text-embedding-3-small**：
   - 默认维度：1536
   - 成本低，性能好
   - 适合大规模 RAG 应用

2. **text-embedding-3-large**：
   - 默认维度：3072
   - 性能最好，成本较高
   - 适合高精度场景

3. **dimensions 参数**：
   - 支持自定义维度（仅 text-embedding-3+）
   - 降低维度可节省 50-75% 成本
   - 权衡：维度 ↓ → 性能 ↓，但成本 ↓

## 三、批量处理与优化

### 问题7：OpenAIEmbeddings 如何处理超过 token 限制的长文本？

**标准答案：**

OpenAIEmbeddings 自动处理长文本：

1. **自动分块**：使用 tiktoken 分词，超过 `embedding_ctx_length` 自动分块
2. **独立嵌入**：每个分块独立调用 API
3. **加权平均**：根据 token 数量加权平均合并
4. **归一化**：最终向量归一化

```python
# 加权平均算法
total_weight = sum(num_tokens_in_batch)
average = [
    sum(val * weight for val, weight in zip(embedding, num_tokens))
    / total_weight
    for embedding in zip(*embeddings)
]
# 归一化
magnitude = sum(val**2 for val in average) ** 0.5
normalized = [val / magnitude for val in average]
```

### 问题8：批量处理 Embeddings 时，chunk_size 参数如何设置？

**标准答案：**

chunk_size 参数设置策略：

1. **API 限制**：OpenAI API 有批量大小限制（通常 2048）
2. **内存管理**：避免一次性处理过多文本
3. **推荐值**：
   - 小规模：100-500

4. **权衡**：
   - 批次大 → 减少 API 调用次数，但内存占用高
   - 批次小 → 内存占用低，但 API 调用次数多

## 四、缓存机制

### 问题9：CacheBackedEmbeddings 的装饰器模式是如何工作的？

**标准答案：**

CacheBackedEmbeddings 使用装饰器模式添加缓存：

1. **包装原始 Embeddings**：不修改原始实现
2. **缓存查找**：先从缓存中批量获取（mget）
3. **增量计算**：只计算缺失的嵌入
4. **批量更新**：使用 mset 批量更新缓存

```python
def embed_documents(self, texts: list[str]) -> list[list[float]]:
    # 1. 从缓存中批量获取
    vectors = self.document_embedding_store.mget(texts)

    # 2. 找出缺失的文本
    missing_indices = [i for i, v in enumerate(vectors) if v is None]

    # 3. 计算缺失的嵌入
    missing_vectors = self.underlying_embeddings.embed_documents(
        [texts[i] for i in missing_indices]
    )

    # 4. 更新缓存
    self.document_embedding_store.mset(zip(missing_texts, missing_vectors))

    return vectors
```

### 问题10：为什么 CacheBackedEmbeddings 默认不缓存查询嵌入？

**标准答案：**

默认不缓存查询嵌入的原因：

1. **查询多样性**：用户查询通常不重复，缓存命中率低
2. **存储成本**：避免存储大量低价值缓存
3. **性能权衡**：查询缓存的收益小于存储开销

**何时缓存查询：**
- 固定查询集（如预定义问题）
- 查询模板（如 Few-shot 示例）
- 评估和测试场景

```python
# 启用查询缓存
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,

)
```

### 问题12：如何在生产环境中部署 Embedding 缓存？

**标准答案：**

生产环境 Embedding 缓存部署策略：

1. **存储选择**：
   - 小规模：LocalFileStore（本地文件）
   - 中规模：Redis（内存缓存）
   - 大规模：Elasticsearch（分布式缓存）

2. **缓存键策略**：
   - 使用 BLAKE2b 或 SHA-256（安全性）
   - 添加 namespace 区分不同模型

3. **监控和维护**：
   - 监控缓存命中率
   - 定期清理过期缓存
   - 备份缓存数据

4. **高可用性**：
   - 缓存失效时自动降级到 API 调用
   - 使用分布式缓存避免单点故障

```python
from langchain_classic.embeddings import CacheBackedEmbeddings
from langchain_classic.storage import RedisStore

# 生产环境配置
store = RedisStore(redis_url="redis://localhost:6379")
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    namespace="prod-text-embedding-3-small",
    key_encoder="blake2b"  # 安全哈希
)
```
