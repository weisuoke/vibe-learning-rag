# 实战代码 - 场景3：加密持久化配置

## 场景说明

本场景演示 LangGraph 的加密持久化功能——使用 `EncryptedSerializer` 对 checkpoint 数据进行 AES 加密，确保存储在数据库中的状态数据即使被泄露也无法被读取。

核心要点：
1. **EncryptedSerializer 是装饰器模式**：它包装默认的 `JsonPlusSerializer`，在序列化后加密、反序列化前解密
2. **AES EAX 认证加密**：同时提供机密性（数据不可读）和完整性（数据不可篡改）
3. **向后兼容**：加密序列化器可以读取未加密的旧数据（通过类型标签判断）
4. **密钥管理**：通过 `LANGGRAPH_AES_KEY` 环境变量或参数传入

> 注意：本场景需要安装 `pycryptodome` 库来提供 AES 加密支持。

[来源: reference/source_状态持久化准备_01.md | LangGraph 源码分析]
[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

---

## 核心知识点

| 知识点 | 说明 |
|--------|------|
| `EncryptedSerializer` | 装饰器模式，包装任何 SerializerProtocol 并添加加密层 |
| `from_pycryptodome_aes()` | 工厂方法，创建基于 PyCryptodome 的 AES 加密序列化器 |
| `LANGGRAPH_AES_KEY` | 环境变量，存放 AES 密钥（16/24/32 字节） |
| 类型标签格式 | `"msgpack"` → `"msgpack+aes"`，用 `+` 分隔加密方法 |
| 向后兼容 | 没有 `+` 的标签视为未加密，加密序列化器可直接读取 |
| AES EAX 模式 | 认证加密，线格式：nonce(16B) + tag(16B) + ciphertext |

```python
# ----- 1. 定义状态 -----

class ChatState(TypedDict):
    """聊天状态"""
    messages: Annotated[list[str], add]
    user_name: str
    secret_context: str  # 敏感信息，需要加密存储


# ----- 2. 定义节点 -----

def greet_node(state: ChatState) -> dict:
    """问候节点"""
    name = state.get("user_name", "用户")
    greeting = f"你好，{name}！有什么可以帮你的？"
    print(f"  greet_node: {greeting}")
    return {"messages": [greeting]}


def process_node(state: ChatState) -> dict:
    """处理节点：使用敏感上下文生成回答"""
    messages = state.get("messages", [])
    context = state.get("secret_context", "")
    last_msg = messages[-1] if messages else ""
    response = f"基于上下文处理: {last_msg} (上下文长度: {len(context)}字)"
    print(f"  process_node: {response}")
    return {"messages": [response]}


# ----- 3. 构建图 -----

builder = StateGraph(ChatState)
builder.add_node("greet", greet_node)
builder.add_node("process", process_node)
builder.add_edge(START, "greet")
builder.add_edge("greet", "process")
builder.add_edge("process", END)


# ----- 4. 使用加密 SqliteSaver -----

db_path = ":memory:"  # 内存数据库（演示用）
conn = sqlite3.connect(db_path, check_same_thread=False)

# 创建加密序列化器
encrypted_serde = EncryptedSerializer.from_pycryptodome_aes()

# 创建加密的 SqliteSaver
encrypted_checkpointer = SqliteSaver(conn, serde=encrypted_serde)
encrypted_checkpointer.setup()  # 初始化数据库表

# 编译图
graph = builder.compile(checkpointer=encrypted_checkpointer)

# ----- 5. 执行图 -----

config = {"configurable": {"thread_id": "encrypted-thread-1"}}

print("\n--- 第一次对话 ---")
result = graph.invoke(
    {
        "messages": ["你好"],
        "user_name": "小明",
        "secret_context": "用户VIP等级: 钻石, 账户余额: ¥50000",
    },
    config,
)
print(f"结果: {result['messages']}")

# ----- 6. 验证状态已持久化 -----

print("\n--- 验证持久化状态 ---")
state = graph.get_state(config)
print(f"持久化的消息: {state.values['messages']}")
print(f"持久化的用户名: {state.values['user_name']}")
print(f"持久化的敏感上下文: {state.values['secret_context']}")

# ----- 7. 验证数据库中的数据是加密的 -----

print("\n--- 验证数据库中数据已加密 ---")
cursor = conn.execute("SELECT * FROM checkpoints LIMIT 1")
row = cursor.fetchone()
if row:
    print(f"数据库行列数: {len(row)}")
    # 检查存储的二进制数据
    for i, col in enumerate(row):
        if isinstance(col, bytes):
            print(f"  列{i} (bytes): 长度={len(col)}, 前20字节={col[:20]!r}")
        elif isinstance(col, str) and len(col) > 50:
            print(f"  列{i} (str): 长度={len(col)}, 前50字符={col[:50]!r}...")
        else:
            print(f"  列{i}: {col!r}")
    print("数据库中存储的是加密后的二进制数据，无法直接读取原始状态")

conn.close()


# ============================================================
# 示例6：对比——加密 vs 未加密存储
# ============================================================

print()
print("=" * 60)
print("=== 示例6：对比——加密 vs 未加密存储 ===")
print("=" * 60)

# 未加密存储
conn_plain = sqlite3.connect(":memory:", check_same_thread=False)
plain_checkpointer = SqliteSaver(conn_plain)
plain_checkpointer.setup()
graph_plain = builder.compile(checkpointer=plain_checkpointer)

config_plain = {"configurable": {"thread_id": "plain-thread-1"}}
graph_plain.invoke(
    {
        "messages": ["你好"],
        "user_name": "小明",
        "secret_context": "用户VIP等级: 钻石, 账户余额: ¥50000",
    },
    config_plain,
)

# 加密存储
conn_enc = sqlite3.connect(":memory:", check_same_thread=False)
enc_checkpointer = SqliteSaver(conn_enc, serde=encrypted_serde)
enc_checkpointer.setup()
graph_enc = builder.compile(checkpointer=enc_checkpointer)

config_enc = {"configurable": {"thread_id": "enc-thread-1"}}
graph_enc.invoke(
    {
        "messages": ["你好"],
        "user_name": "小明",
        "secret_context": "用户VIP等级: 钻石, 账户余额: ¥50000",
    },
    config_enc,
)

# 对比数据库内容
print("\n未加密数据库:")
for row in conn_plain.execute("SELECT thread_id, checkpoint FROM checkpoints"):
    tid = row[0]
    data = row[1]
    if isinstance(data, str):
        # 检查是否能看到明文
        has_vip = "VIP" in data or "钻石" in data or "50000" in data
        print(f"  thread={tid}, 包含敏感信息明文: {has_vip}")
        if has_vip:
            # 找到敏感信息的位置
            for keyword in ["VIP", "钻石", "50000"]:
                if keyword in data:
                    idx = data.index(keyword)
                    print(f"    发现 '{keyword}' 在位置 {idx}")

print("\n加密数据库:")
for row in conn_enc.execute("SELECT thread_id, checkpoint FROM checkpoints"):
    tid = row[0]
    data = row[1]
    if isinstance(data, str):
        has_vip = "VIP" in data or "钻石" in data or "50000" in data
        print(f"  thread={tid}, 包含敏感信息明文: {has_vip}")
    elif isinstance(data, bytes):
        print(f"  thread={tid}, 数据为加密字节, 长度={len(data)}")

conn_plain.close()
conn_enc.close()

print("\n结论：加密存储确保敏感数据在数据库中不可读")
print("即使数据库文件被泄露，没有 AES 密钥也无法恢复原始数据")
```

## 代码解析

### 1. EncryptedSerializer 的装饰器模式

```
                    ┌─────────────────────────┐
                    │   EncryptedSerializer    │
                    │                         │
  Python 对象 ────► │  JsonPlusSerializer      │ ────► (type+cipher, 密文)
                    │       ↓ dumps_typed      │
                    │  (type, 明文字节)         │
                    │       ↓ encrypt          │
                    │  (type+aes, 密文字节)     │
                    └─────────────────────────┘
```

EncryptedSerializer 不自己做序列化，而是：
1. 委托内部的 `JsonPlusSerializer` 完成 Python 对象 ↔ 字节的转换
2. 在字节层面添加 AES 加密/解密
3. 在类型标签上追加 `+aes` 后缀标记加密状态

### 2. AES EAX 认证加密

```
加密输出格式（线格式）:
┌──────────┬──────────┬─────────────────┐
│ nonce    │ tag      │ ciphertext      │
│ 16 bytes │ 16 bytes │ 变长            │
└──────────┴──────────┴─────────────────┘

- nonce: 随机数，确保相同明文产生不同密文
- tag: 认证标签，验证数据完整性（防篡改）
- ciphertext: 加密后的数据
```

EAX 模式的优势：
- 同时提供机密性（不可读）和完整性（不可改）
- 密钥错误或数据被篡改时，解密直接失败（MAC check failed）
- 不需要额外的 HMAC 计算

### 3. 向后兼容机制

```python
def loads_typed(self, data: tuple[str, bytes]) -> Any:
    type_tag, payload = data
    if "+" in type_tag:
        # 有加密后缀 → 分离标签，先解密
        original_tag, cipher_name = type_tag.rsplit("+", 1)
        decrypted = self.cipher.decrypt(payload)
        return self.serde.loads_typed((original_tag, decrypted))
    else:
        # 无加密后缀 → 直接反序列化（旧数据）
        return self.serde.loads_typed(data)
```

这个设计让你可以：
- 在不停机的情况下升级到加密存储
- 旧的未加密 checkpoint 照常读取
- 新写入的 checkpoint 自动加密
- 逐步迁移，无需一次性重新加密所有数据

## 关键要点

1. **EncryptedSerializer 是装饰器**：包装 JsonPlusSerializer，在字节层面添加加密，不改变序列化逻辑
2. **类型标签自描述**：`msgpack+aes` 格式让系统知道如何解密，也实现了向后兼容
3. **AES EAX 提供双重保护**：机密性（不可读）+ 完整性（不可改），密钥错误直接报错
4. **无缝升级**：加密序列化器可以读取未加密旧数据，新数据自动加密
5. **密钥管理是关键**：使用环境变量或密钥管理服务，永远不要硬编码密钥
6. **pycryptodome 是必需依赖**：`pip install pycryptodome` 或 `uv add pycryptodome`

---

## 扩展思考

1. **密钥轮换怎么做？** 目前 EncryptedSerializer 不直接支持密钥轮换。一种方案是：用旧密钥读取所有 checkpoint，用新密钥重新写入。LangGraph 的向后兼容设计让这个过程可以渐进式完成。

2. **加密对性能的影响？** AES 加密非常快（现代 CPU 有硬件加速），对于典型的 checkpoint 数据（几 KB 到几 MB），加密开销可以忽略不计。主要开销在于额外的 32 字节（nonce + tag）。

3. **哪些场景需要加密持久化？** 当 checkpoint 包含用户隐私数据（PII）、API 密钥、商业机密等敏感信息时。特别是使用文件数据库（SQLite）或共享数据库（PostgreSQL）时，加密是必要的安全措施。

4. **能否只加密部分字段？** 目前 EncryptedSerializer 是全量加密。如果需要选择性加密，可以在应用层对敏感字段单独加密后再存入状态。
