# 实战代码 - 场景8：多代理系统

## 场景描述

构建一个 RAG 系统的多代理协作工作流，使用 Supervisor 模式协调多个专业代理完成复杂任务。

**实际应用场景**：智能文档问答系统
- Query Analyzer：分析用户查询意图
- Document Retriever：检索相关文档
- Answer Generator：生成答案
- Quality Reviewer：审核答案质量
- Supervisor：协调各代理的工作流程

## 核心挑战

1. **代理协调**：如何让 Supervisor 正确调度各个代理
2. **状态共享**：多个代理如何共享和更新状态
3. **条件路由**：根据任务进度动态决定下一步
4. **错误处理**：某个代理失败时如何处理
5. **循环控制**：避免无限循环，设置合理的终止条件

## 完整代码

```python
"""
LangGraph 多代理系统实战
场景：RAG 系统中的多代理协作
"""

from typing import Annotated, Literal, TypedDict
from langgraph.graph import StateGraph, START, END
import operator


# ===== 1. 定义共享状态 =====
class AgentState(TypedDict):
    """多代理共享状态"""
    # 用户输入
    query: str

    # 各代理的工作结果
    query_analysis: str  # 查询分析结果
    retrieved_docs: list[str]  # 检索到的文档
    answer: str  # 生成的答案
    quality_score: float  # 质量评分

    # 工作流控制
    current_agent: str  # 当前执行的代理
    next_agent: str  # 下一个要执行的代理
    messages: Annotated[list[str], operator.add]  # 工作日志
    retry_count: int  # 重试次数


# ===== 2. Supervisor 节点 =====
def supervisor(state: AgentState) -> dict:
    """
    Supervisor：协调各个代理的工作
    根据当前状态决定下一步执行哪个代理
    """
    current = state.get("current_agent", "start")
    retry_count = state.get("retry_count", 0)

    # 决策逻辑
    if current == "start":
        next_agent = "query_analyzer"
    elif current == "query_analyzer":
        next_agent = "document_retriever"
    elif current == "document_retriever":
        # 检查是否检索到文档
        if state.get("retrieved_docs"):
            next_agent = "answer_generator"
        else:
            next_agent = "END"  # 没有文档，结束流程
    elif current == "answer_generator":
        next_agent = "quality_reviewer"
    elif current == "quality_reviewer":
        # 根据质量评分决定
        quality_score = state.get("quality_score", 0)
        if quality_score >= 0.7:
            next_agent = "END"  # 质量合格，结束
        elif retry_count < 2:
            next_agent = "answer_generator"  # 重新生成
        else:
            next_agent = "END"  # 重试次数用完，结束
    else:
        next_agent = "END"

    return {
        "next_agent": next_agent,
        "messages": [f"[Supervisor] 当前代理: {current}, 下一步: {next_agent}"]
    }


# ===== 3. Query Analyzer 节点 =====
def query_analyzer(state: AgentState) -> dict:
    """
    Query Analyzer：分析用户查询
    提取关键信息和意图
    """
    query = state["query"]

    # 模拟查询分析（实际应用中会调用 LLM）
    analysis = f"分析查询 '{query}': 类型=问答, 领域=技术文档, 关键词=[LangGraph, 多代理]"

    return {
        "query_analysis": analysis,
        "current_agent": "query_analyzer",
        "messages": [f"[Query Analyzer] {analysis}"]
    }


# ===== 4. Document Retriever 节点 =====
def document_retriever(state: AgentState) -> dict:
    """
    Document Retriever：检索相关文档
    基于查询分析结果检索文档
    """
    query = state["query"]
    analysis = state.get("query_analysis", "")

    # 模拟文档检索（实际应用中会查询向量数据库）
    docs = [
        "LangGraph 支持多代理协作，通过 StateGraph 实现状态共享",
        "Supervisor 模式是多代理系统的常见架构",
        "条件路由可以实现动态的代理调度"
    ]

    return {
        "retrieved_docs": docs,
        "current_agent": "document_retriever",
        "messages": [f"[Document Retriever] 检索到 {len(docs)} 个文档"]
    }


# ===== 5. Answer Generator 节点 =====
def answer_generator(state: AgentState) -> dict:
    """
    Answer Generator：生成答案
    基于检索到的文档生成答案
    """
    query = state["query"]
    docs = state.get("retrieved_docs", [])
    retry_count = state.get("retry_count", 0)

    # 模拟答案生成（实际应用中会调用 LLM）
    if retry_count == 0:
        answer = f"根据文档，{query} 的答案是：LangGraph 通过 StateGraph 和条件路由实现多代理协作。"
    else:
        answer = f"改进后的答案：LangGraph 的多代理系统使用 Supervisor 模式协调各代理，通过共享状态实现协作。"

    return {
        "answer": answer,
        "current_agent": "answer_generator",
        "messages": [f"[Answer Generator] 生成答案 (尝试 {retry_count + 1})"]
    }


# ===== 6. Quality Reviewer 节点 =====
def quality_reviewer(state: AgentState) -> dict:
    """
    Quality Reviewer：审核答案质量
    评估答案的准确性和完整性
    """
    answer = state.get("answer", "")
    retry_count = state.get("retry_count", 0)

    # 模拟质量评估（实际应用中会使用评估模型）
    # 第一次生成的答案质量较低，需要重试
    if retry_count == 0:
        quality_score = 0.6
        verdict = "需要改进"
    else:
        quality_score = 0.85
        verdict = "质量合格"

    return {
        "quality_score": quality_score,
        "current_agent": "quality_reviewer",
        "retry_count": retry_count + 1,
        "messages": [f"[Quality Reviewer] 质量评分: {quality_score}, {verdict}"]
    }


# ===== 7. 条件路由函数 =====
def route_next(state: AgentState) -> Literal["query_analyzer", "document_retriever",
                                              "answer_generator", "quality_reviewer", "END"]:
    """
    条件路由：根据 Supervisor 的决策路由到下一个代理
    """
    next_agent = state.get("next_agent", "END")

    if next_agent == "END":
        return END

    return next_agent


# ===== 8. 构建多代理图 =====
def build_multi_agent_graph():
    """构建多代理协作图"""

    # 创建 StateGraph
    builder = StateGraph(AgentState)

    # 添加所有节点
    builder.add_node("supervisor", supervisor)
    builder.add_node("query_analyzer", query_analyzer)
    builder.add_node("document_retriever", document_retriever)
    builder.add_node("answer_generator", answer_generator)
    builder.add_node("quality_reviewer", quality_reviewer)

    # 设置入口点：从 Supervisor 开始
    builder.add_edge(START, "supervisor")

    # Supervisor 的条件路由
    builder.add_conditional_edges(
        "supervisor",
        route_next,
        {
            "query_analyzer": "query_analyzer",
            "document_retriever": "document_retriever",
            "answer_generator": "answer_generator",
            "quality_reviewer": "quality_reviewer",
            END: END
        }
    )

    # 各代理完成后返回 Supervisor
    builder.add_edge("query_analyzer", "supervisor")
    builder.add_edge("document_retriever", "supervisor")
    builder.add_edge("answer_generator", "supervisor")
    builder.add_edge("quality_reviewer", "supervisor")

    # 编译图
    graph = builder.compile()

    return graph


# ===== 9. 运行示例 =====
if __name__ == "__main__":
    print("=== LangGraph 多代理系统实战 ===\n")

    # 构建图
    graph = build_multi_agent_graph()

    # 初始状态
    initial_state = {
        "query": "LangGraph 如何实现多代理协作？",
        "current_agent": "start",
        "messages": [],
        "retry_count": 0
    }

    # 执行图
    print("开始执行多代理工作流...\n")
    result = graph.invoke(initial_state)

    # 输出结果
    print("\n=== 执行日志 ===")
    for msg in result["messages"]:
        print(msg)

    print("\n=== 最终结果 ===")
    print(f"查询: {result['query']}")
    print(f"查询分析: {result.get('query_analysis', 'N/A')}")
    print(f"检索文档数: {len(result.get('retrieved_docs', []))}")
    print(f"答案: {result.get('answer', 'N/A')}")
    print(f"质量评分: {result.get('quality_score', 0)}")
    print(f"重试次数: {result.get('retry_count', 0)}")
```

## 关键点解析

### 1. Supervisor 模式的核心

**集中式协调**：
- Supervisor 作为中央协调者，不执行具体任务
- 根据当前状态决定下一步执行哪个代理
- 所有代理完成后都返回 Supervisor

**决策逻辑**：
```python
def supervisor(state: AgentState) -> dict:
    current = state.get("current_agent", "start")

    # 根据当前代理决定下一步
    if current == "query_analyzer":
        next_agent = "document_retriever"
    elif current == "document_retriever":
        next_agent = "answer_generator"
    # ...

    return {"next_agent": next_agent}
```

### 2. 共享状态设计

**状态字段分类**：
- **输入数据**：`query`（用户查询）
- **工作结果**：`query_analysis`, `retrieved_docs`, `answer`
- **控制信息**：`current_agent`, `next_agent`, `retry_count`
- **日志记录**：`messages`（使用 `operator.add` 累积）

**部分更新机制**：
```python
def query_analyzer(state: AgentState) -> dict:
    # 只返回需要更新的字段
    return {
        "query_analysis": analysis,
        "current_agent": "query_analyzer",
        "messages": [f"[Query Analyzer] {analysis}"]
    }
```

### 3. 条件路由实现

**路由函数**：
```python
def route_next(state: AgentState) -> Literal[...]:
    next_agent = state.get("next_agent", "END")

    if next_agent == "END":
        return END

    return next_agent
```

**路由配置**：
```python
builder.add_conditional_edges(
    "supervisor",
    route_next,
    {
        "query_analyzer": "query_analyzer",
        "document_retriever": "document_retriever",
        # ...
        END: END
    }
)
```

### 4. 循环与重试机制

**质量检查循环**：
- Quality Reviewer 评估答案质量
- 质量不合格时返回 Answer Generator 重新生成
- 使用 `retry_count` 限制重试次数

**循环控制**：
```python
def supervisor(state: AgentState) -> dict:
    retry_count = state.get("retry_count", 0)

    if current == "quality_reviewer":
        quality_score = state.get("quality_score", 0)
        if quality_score >= 0.7:
            next_agent = "END"
        elif retry_count < 2:
            next_agent = "answer_generator"  # 重试
        else:
            next_agent = "END"  # 超过重试次数
```

### 5. 图结构设计

**星型拓扑**：
```
         ┌─────────────┐
    ┌───→│ Supervisor  │←───┐
    │    └─────────────┘    │
    │           │            │
    │           ↓            │
    │    ┌─────────────┐    │
    └────│Query Analyzer│────┘
         └─────────────┘
```

所有代理都连接到 Supervisor，形成星型结构。

### 6. 错误处理策略

**空文档处理**：
```python
if state.get("retrieved_docs"):
    next_agent = "answer_generator"
else:
    next_agent = "END"  # 没有文档，提前结束
```

**重试限制**：
- 设置 `retry_count` 上限
- 避免无限循环

**累积日志**：

使用 `operator.add` 确保每个代理的日志都被累积。

## 常见问题

### Q1: Supervisor 和普通节点有什么区别？

**A**: Supervisor 是一个特殊的协调节点：
- **不执行具体任务**：只负责决策和路由
- **连接所有代理**：所有代理完成后都返回 Supervisor
- **包含决策逻辑**：根据状态决定下一步

普通节点执行具体任务，Supervisor 负责协调。

### Q2: 如何避免无限循环？

**A**: 三种策略：
1. **计数器限制**：使用 `retry_count` 限制重试次数
2. **状态检查**：检查状态是否满足终止条件
3. **超时机制**：设置最大执行步数

```python
if retry_count >= max_retries:
    return {"next_agent": "END"}
```

### Q3: 多个代理可以并行执行吗？

**A**: 当前示例是顺序执行，但 LangGraph 支持并行：

```python
# 并行执行多个代理
builder.add_edge(START, ["agent1", "agent2", "agent3"])
builder.add_edge(["agent1", "agent2", "agent3"], "aggregator")
```

所有代理完成后才进入下一步。

### Q4: 如何处理代理执行失败？

**A**: 添加错误处理节点：

```python
def error_handler(state: AgentState) -> dict:
    error = state.get("error")
    return {
        "next_agent": "END",
        "messages": [f"[Error] {error}"]
    }

# 在 Supervisor 中检查错误
if state.get("error"):
    next_agent = "error_handler"
```

### Q5: 状态会不会越来越大？

**A**: 合理设计状态字段：
- **只保留必要信息**：不要存储大量中间结果
- **使用引用**：存储文档 ID 而非完整文档
- **定期清理**：在 Supervisor 中清理过期数据

```python
def supervisor(state: AgentState) -> dict:
    # 清理不再需要的数据
    if current == "quality_reviewer":
        return {
            "next_agent": "END",
            "retrieved_docs": []  # 清空文档
        }
```

## 最佳实践

### 1. 明确代理职责

**单一职责原则**：
- 每个代理只负责一个明确的任务
- 避免代理职责重叠
- 便于测试和维护

### 2. 设计清晰的状态结构

**状态字段分类**：
```python
class AgentState(TypedDict):
    # 输入
    query: str

    # 工作结果
    analysis: str
    docs: list[str]

    # 控制信息
    current_agent: str
    next_agent: str

    # 元数据
    messages: Annotated[list[str], operator.add]
```

### 3. 使用类型提示

**明确路由目标**：
```python
def route_next(state: AgentState) -> Literal["agent1", "agent2", "END"]:
```

## 扩展方向
