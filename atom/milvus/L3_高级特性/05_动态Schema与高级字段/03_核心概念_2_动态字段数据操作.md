# 核心概念2：动态字段数据操作

> 本文档详细讲解 Milvus 2.6 动态字段的插入、查询、过滤操作和实际应用技巧

---

## 一句话定义

**动态字段数据操作是指在启用动态 Schema 的 Collection 中，对未预定义字段进行插入、查询和过滤的完整流程，所有动态字段自动存储在 `$meta` 字段中，可以像固定字段一样进行查询和过滤。**

---

## 动态字段插入操作

### 1. 基础插入

**定义**：在插入数据时，可以包含任意未在 Schema 中定义的字段。

**Python 示例**（来源：reference/context7_pymilvus_01.md）：

```python
from pymilvus import MilvusClient

client = MilvusClient("http://localhost:19530")

# 插入带有动态字段的数据
data = [
    {
        "id": 1,
        "vector": [0.1, 0.2, 0.3] * 256,  # 固定字段
        "title": "Document 1",             # 动态字段
        "author": "Alice",                 # 动态字段
        "tags": ["AI", "ML"],              # 动态字段（数组）
        "page_count": 100,                 # 动态字段（数字）
        "is_published": True               # 动态字段（布尔）
    },
    {
        "id": 2,
        "vector": [0.4, 0.5, 0.6] * 256,
        "title": "Document 2",
        "category": "Tech",                # 不同的动态字段
        "published_date": "2026-01-01"     # 不同的动态字段
    }
]

# 插入数据
client.insert(collection_name="documents", data=data)
```

**关键特性**：
- 不同记录可以有不同的动态字段
- 动态字段类型自动推断
- 无需预先声明字段

---

### 2. 批量插入

**场景**：大量数据插入时，动态字段的处理。

```python
# 批量插入不同结构的数据
batch_data = []

# PDF 文档
for i in range(100):
    batch_data.append({
        "id": i,
        "vector": [0.1] * 768,
        "text": f"PDF content {i}",
        "source": "pdf",
        "page_number": i % 10,
        "total_pages": 100
    })

# 网页文档
for i in range(100, 200):
    batch_data.append({
        "id": i,
        "vector": [0.2] * 768,
        "text": f"Web content {i}",
        "source": "web",
        "url": f"https://example.com/{i}",
        "crawled_at": "2026-01-01"
    })

# 批量插入
client.insert(collection_name="documents", data=batch_data)
```

**性能考虑**：
- 批量插入比单条插入效率高
- 建议每批 1000-10000 条记录
- 动态字段不影响插入性能

---

### 3. 动态字段类型推断

**类型推断规则**（来源：reference/source_动态Schema_01.md）：

| Python 类型 | Milvus 动态字段类型 | 示例 |
|------------|-------------------|------|
| `str` | VARCHAR | `"Alice"` |
| `int` | INT64 | `100` |
| `float` | DOUBLE | `29.99` |
| `bool` | BOOL | `True` |
| `list` | ARRAY | `["AI", "ML"]` |
| `dict` | JSON | `{"key": "value"}` |

**示例**：

```python
data = {
    "id": 1,
    "vector": [0.1] * 768,
    # 类型推断示例
    "title": "Document 1",           # VARCHAR
    "page_count": 100,               # INT64
    "price": 29.99,                  # DOUBLE
    "is_published": True,            # BOOL
    "tags": ["AI", "ML"],            # ARRAY<VARCHAR>
    "metadata": {                    # JSON
        "author": "Alice",
        "department": "Engineering"
    }
}
```

## 动态字段查询操作

### 1. 基础查询

**定义**：在 `output_fields` 中指定动态字段名称，即可查询动态字段。

```python
# 查询动态字段
results = client.search(
    collection_name="documents",
    data=[query_vector],
    output_fields=["title", "author", "tags"],  # 包含动态字段
    limit=10
)

# 访问结果
for hit in results[0]:
    print(f"ID: {hit['id']}")
    print(f"Title: {hit['title']}")      # 动态字段
    print(f"Author: {hit['author']}")    # 动态字段
    print(f"Tags: {hit['tags']}")        # 动态字段
    print(f"Distance: {hit['distance']}")
```

**关键特性**：
- 动态字段和固定字段查询方式相同
- 无需通过 `$meta` 前缀访问
- 如果字段不存在，返回 `None`

---

### 2. 查询所有字段

**使用 `output_fields=["*"]` 查询所有字段**：

```python
# 查询所有字段（包括动态字段）
results = client.search(
    collection_name="documents",
    data=[query_vector],
    output_fields=["*"],  # 返回所有字段
    limit=10
)

# 访问结果
for hit in results[0]:
    print(f"All fields: {hit}")
    # 输出示例：
    # {
    #   'id': 1,
    #   'vector': [0.1, 0.2, ...],
    #   'title': 'Document 1',
    #   'author': 'Alice',
    #   'tags': ['AI', 'ML']
    # }
```

**注意事项**：
- `*` 会返回所有固定字段和动态字段
- 大量动态字段会增加网络传输开销
- 建议只查询需要的字段

---

### 3. 处理缺失字段

**问题**：不同记录可能有不同的动态字段，查询时需要处理缺失字段。

```python
# 查询可能缺失的动态字段
results = client.search(
    collection_name="documents",
    data=[query_vector],
    output_fields=["title", "author", "category"],
    limit=10
)

# 安全访问动态字段
for hit in results[0]:
    title = hit.get('title', 'Unknown')        # 使用默认值
    author = hit.get('author', 'Anonymous')
    category = hit.get('category', 'Uncategorized')

    print(f"Title: {title}")
    print(f"Author: {author}")
    print(f"Category: {category}")
```

**最佳实践**：
- 使用 `dict.get()` 方法提供默认值
- 在应用层处理缺失字段
- 避免假设所有记录都有相同的动态字段

## 动态字段过滤操作

### 1. 基础过滤

**定义**：在 `filter` 参数中使用动态字段进行过滤。

```python
# 字符串过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='author == "Alice"',  # 动态字段过滤
    output_fields=["title", "author"],
    limit=10
)

# 数值过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='page_count > 50',
    output_fields=["title", "page_count"],
    limit=10
)

# 布尔过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='is_published == true',
    output_fields=["title", "is_published"],
    limit=10
)
```

---

### 2. 复合过滤

**使用逻辑运算符组合多个动态字段过滤条件**：

```python
# AND 逻辑
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='author == "Alice" and page_count > 50',
    limit=10
)

# OR 逻辑
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='author == "Alice" or author == "Bob"',
    limit=10
)

# 复杂组合
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='(author == "Alice" or author == "Bob") and page_count > 50',
    limit=10
)
```

---

### 3. 数组字段过滤

**使用 `array_contains` 函数过滤数组字段**：

```python
# 检查数组是否包含特定值
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='array_contains(tags, "AI")',  # tags 是动态数组字段
    output_fields=["title", "tags"],
    limit=10
)

# 组合数组过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='array_contains(tags, "AI") and author == "Alice"',
    limit=10
)
```

---

### 4. JSON 字段过滤

**使用 JSON Path 语法过滤 JSON 字段**（来源：reference/context7_milvus_02.md）：

```python
# JSON 字段过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='metadata["author"] == "Alice"',  # metadata 是动态 JSON 字段
    output_fields=["title", "metadata"],
    limit=10
)

# 嵌套 JSON 过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='metadata["permissions"]["read"][0] == "user1"',
    limit=10
)
```

**注意事项**：
- JSON Path 语法支持嵌套访问
- 需要 JSON Path Index 优化性能（Milvus 2.6+）

---

### 5. 范围过滤

**数值范围过滤**：

```python
# 范围过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='page_count >= 50 and page_count <= 100',
    limit=10
)

# IN 操作符
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='author in ["Alice", "Bob", "Charlie"]',
    limit=10
)

# NOT IN 操作符
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='category not in ["Draft", "Archived"]',
    limit=10
)
```

## 动态字段的性能考虑

### 1. 查询性能

**问题**（来源：reference/search_动态Schema_01.md）：
- 动态字段无法创建索引
- 过滤动态字段需要全表扫描
- 大数据量下性能显著下降

**性能对比**：

| 操作 | 固定字段（有索引） | 动态字段（无索引） |
|------|------------------|------------------|
| 等值过滤 | 快（索引查找） | 慢（全表扫描） |
| 范围过滤 | 快（索引范围查询） | 慢（全表扫描） |
| 数组过滤 | 快（倒排索引） | 慢（全表扫描） |
| JSON 过滤 | 快（JSON Path Index） | 慢（全表扫描） |

**优化建议**：
- 将高频过滤字段定义为固定字段
- 为固定字段创建索引
- 减少动态字段的过滤操作

---

### 2. 存储开销

**动态字段存储在 `$meta` 字段中**：

```
固定字段：每个字段独立存储
动态字段：统一存储在 $meta 字段（JSON 格式）

存储开销：
- 固定字段：字段值大小
- 动态字段：字段名 + 字段值 + JSON 序列化开销
```

**示例**：

```python
# 固定字段存储
{
  "id": 1,
  "vector": [0.1, 0.2, ...],
  "title": "Document 1"  # 直接存储
}

# 动态字段存储
{
  "id": 1,
  "vector": [0.1, 0.2, ...],
  "$meta": {  # JSON 格式存储
    "title": "Document 1",
    "author": "Alice",
    "tags": ["AI", "ML"]
  }
}
```

**存储开销对比**：
- 固定字段：~10 字节（字段值）
- 动态字段：~20 字节（字段名 + 字段值 + JSON 开销）

---

### 3. group_by 问题

**问题**（来源：reference/search_动态Schema_01.md，Issue #47438）：
- 动态字段的 `group_by` 对缺失字段处理错误
- 缺失字段会被视为一个独立的组

**示例**：

```python
# 数据
data = [
    {"id": 1, "vector": [0.1]*768, "category": "A"},
    {"id": 2, "vector": [0.2]*768, "category": "B"},
    {"id": 3, "vector": [0.3]*768}  # 缺少 category 字段
]

# group_by 查询
results = client.search(
    collection_name="documents",
    data=[query_vector],
    group_by_field="category",  # 动态字段
    limit=10
)

# 问题：id=3 的记录会被视为一个独立的组（category=null）
# 预期：应该被过滤或单独处理
```

**解决方案**：
- 在应用层过滤缺失字段
- 或使用固定字段进行 group_by

```python
# 解决方案1：应用层过滤
results = client.search(
    collection_name="documents",
    data=[query_vector],
    filter='category != null',  # 过滤缺失字段
    group_by_field="category",
    limit=10
)

# 解决方案2：使用固定字段
# 将 category 定义为固定字段，设置默认值
schema.add_field(
    field_name="category",
    datatype=DataType.VARCHAR,
    max_length=128,
    default_value="Uncategorized"
)
```

## 在 Milvus 2.6 中的应用

### 2026 年生产环境标准配置

```python
from pymilvus import MilvusClient, DataType

client = MilvusClient("http://localhost:19530")

# 创建 Schema（启用动态字段）
schema = client.create_schema(
    auto_id=False,
    enable_dynamic_field=True
)

# 核心字段（固定）
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="text", datatype=DataType.VARCHAR, max_length=65535)

# 高频过滤字段（固定）
schema.add_field(field_name="tenant_id", datatype=DataType.VARCHAR, max_length=64)
schema.add_field(field_name="source", datatype=DataType.VARCHAR, max_length=32)
schema.add_field(field_name="created_at", datatype=DataType.INT64)

# 复杂元数据（固定 JSON 字段）
schema.add_field(field_name="metadata", datatype=DataType.JSON)

# 创建索引
index_params = client.prepare_index_params()
index_params.add_index(field_name="vector", index_type="AUTOINDEX", metric_type="COSINE")
index_params.add_index(field_name="tenant_id", index_type="INVERTED")
index_params.add_index(field_name="source", index_type="INVERTED")
index_params.add_index(field_name="created_at", index_type="INVERTED")

# 创建 Collection
client.create_collection(
    collection_name="documents",
    schema=schema,
    index_params=index_params
)

# 插入数据（包含动态字段）
data = [
    {
        "id": 1,
        "vector": [0.1] * 768,
        "text": "Document content...",
        "tenant_id": "tenant_a",
        "source": "pdf",
        "created_at": 1640000000,
        "metadata": {"author": "Alice"},
        # 动态字段
        "page_number": 5,
        "total_pages": 100
    }
]

client.insert(collection_name="documents", data=data)

# 查询数据（包含动态字段）
results = client.search(
    collection_name="documents",
    data=[[0.1] * 768],
    filter='tenant_id == "tenant_a" and source == "pdf"',
    output_fields=["text", "page_number", "total_pages"],
    limit=10
)
```

## 总结

动态字段数据操作包括插入、查询和过滤三个核心操作。插入时可以包含任意未定义字段，查询时可以像固定字段一样访问，过滤时支持多种运算符和复杂条件。

**核心要点**：
1. 插入：可以包含任意动态字段，类型自动推断
2. 查询：在 `output_fields` 中指定动态字段名称
3. 过滤：支持等值、范围、数组、JSON 等多种过滤方式
4. 性能：动态字段无法创建索引，查询性能较低
5. 最佳实践：核心字段固定，业务字段动态，高频过滤字段创建索引

**2026 年生产标准**：
- 核心字段和高频过滤字段定义为固定字段
- 使用 JSON 字段存储复杂元数据
- 为固定字段创建索引优化性能
- 安全处理缺失字段，使用默认值
