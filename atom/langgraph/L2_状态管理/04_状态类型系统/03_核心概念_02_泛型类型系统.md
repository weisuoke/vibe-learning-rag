# 核心概念:泛型类型系统

> StateT、ContextT、InputT、OutputT - LangGraph 的四大类型参数

---

## 概述

LangGraph 的 StateGraph 使用泛型类型系统，支持 4 个类型参数：**StateT**、**ContextT**、**InputT**、**OutputT**。这种设计提供了强大的类型安全和灵活性。

**核心问题**：
- 为什么需要 4 个不同的类型参数？
- 每个类型参数的作用是什么？
- 如何正确使用泛型类型系统？

**本文目标**：
- 深入理解 4 个类型参数的含义
- 掌握类型参数的默认值和推断机制
- 学习类型参数的使用场景和最佳实践
- 通过代码示例理解实际应用

## 1. StateGraph 泛型定义

### 1.1 源码定义

```python
class StateGraph(Generic[StateT, ContextT, InputT, OutputT]):
    """A graph whose nodes communicate by reading and writing to a shared state.

    The signature of each node is `State -> Partial<State>`.
    """

    def __init__(
        self,
        state_schema: type[StateT],
        context_schema: type[ContextT] | None = None,
        *,
        input_schema: type[InputT] | None = None,
        output_schema: type[OutputT] | None = None,
        **kwargs: Unpack[DeprecatedKwargs],
    ) -> None:
        self.state_schema = state_schema
        self.input_schema = cast(type[InputT], input_schema or state_schema)
        self.output_schema = cast(type[OutputT], output_schema or state_schema)
        self.context_schema = context_schema
```

[来源: reference/source_状态类型系统_01.md | sourcecode/langgraph/.../state.py]

**关键发现**：
- StateGraph 是一个泛型类，使用 4 个类型参数
- `state_schema` 是必需的，其他 schema 可选
- `input_schema` 和 `output_schema` 默认使用 `state_schema`
- `context_schema` 用于暴露不可变的运行时上下文

### 1.2 类型参数概览

| 类型参数 | 必需性 | 默认值 | 作用 |
|---------|--------|--------|------|
| **StateT** | 必需 | - | 图的内部状态类型 |
| **ContextT** | 可选 | None | 不可变的运行时上下文类型 |
| **InputT** | 可选 | StateT | 图的输入类型 |
| **OutputT** | 可选 | StateT | 图的输出类型 |

## 2. StateT：内部状态类型

### 2.1 基本概念

**StateT** 是图的核心状态类型，定义了节点之间共享的数据结构。

**特性**：
- 必需参数，必须显式提供
- 所有节点默认使用此类型
- 支持 TypedDict、Pydantic、dataclass

### 2.2 基础用法

```python
from typing_extensions import TypedDict, Annotated
from langgraph.graph import StateGraph

def add_messages(left: list, right: list) -> list:
    return left + right

# 定义状态类型
class State(TypedDict):
    messages: Annotated[list[str], add_messages]
    user_id: str
    count: int

# 创建图（StateT = State）
graph = StateGraph(State)

def node1(state: State) -> dict:
    return {
        "messages": ["Hello"],
        "count": state["count"] + 1
    }

graph.add_node("node1", node1)
```

[来源: reference/source_状态类型系统_01.md | sourcecode/langgraph/.../state.py]

### 2.3 StateT 的作用域

**StateT 影响的范围**：
- 节点函数的输入类型（默认）
- 节点函数的输出类型（部分更新）
- 图的内部状态存储
- Channel 的类型定义

**示例**：
```python
class State(TypedDict):
    x: int
    y: str

graph = StateGraph(State)

# 节点函数默认使用 State 类型
def node(state: State) -> dict:
    # state 的类型是 State
    # 返回值是 Partial<State>
    return {"x": state["x"] + 1}
```

## 3. ContextT：不可变上下文

### 3.1 基本概念

**ContextT** 用于暴露不可变的运行时上下文，节点可以读取但不能修改。

**特性**：
- 可选参数，默认为 None
- 只读，节点不能修改
- 适合存储配置、用户信息等

[来源: reference/source_状态类型系统_01.md | sourcecode/langgraph/.../state.py]

### 3.2 基础用法

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph

# 定义状态类型
class State(TypedDict):
    messages: list[str]
    count: int

# 定义上下文类型
class Context(TypedDict):
    user_id: str
    api_key: str
    max_retries: int

# 创建图（StateT = State, ContextT = Context）
graph = StateGraph(State, context_schema=Context)

def node1(state: State, *, context: Context) -> dict:
    # 可以读取 context
    user_id = context["user_id"]
    max_retries = context["max_retries"]

    # 但不能修改 context（只读）
    # context["user_id"] = "new_id"  # 错误！

    return {
        "messages": [f"Hello {user_id}"],
        "count": state["count"] + 1
    }

graph.add_node("node1", node1)
```

### 3.3 Context 的使用场景

**适合使用 Context 的场景**：
- 用户身份信息（user_id、tenant_id）
- API 密钥和配置
- 请求元数据（request_id、trace_id）
- 全局配置（max_retries、timeout）

**不适合使用 Context 的场景**：
- 需要在节点间传递的数据 → 使用 State
- 需要累积的数据 → 使用 State + Reducer
- 需要修改的数据 → 使用 State

## 4. InputT：输入类型

### 4.1 基本概念

**InputT** 定义了图的输入类型，用于验证和转换外部输入。

**特性**：
- 可选参数，默认使用 StateT
- 用于 API 边界验证
- 支持与 StateT 不同的类型

### 4.2 基础用法

```python
from typing_extensions import TypedDict
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph

# 内部状态（TypedDict）
class InternalState(TypedDict):
    messages: list[str]
    context: dict
    results: list[dict]

# 外部输入（Pydantic）
class APIInput(BaseModel):
    query: str = Field(..., min_length=1, max_length=1000)
    user_id: str = Field(..., pattern=r'^[a-zA-Z0-9_-]+$')
    max_results: int = Field(default=10, ge=1, le=100)

# 创建图（StateT = InternalState, InputT = APIInput）
graph = StateGraph(
    state_schema=InternalState,
    input_schema=APIInput
)

def entry_node(state: InternalState) -> dict:
    # 输入已经被验证和转换
    return {
        "messages": [state.get("query", "")],
        "results": []
    }

graph.add_node("entry", entry_node)
```

### 4.3 InputT 的作用

**InputT 的处理流程**：
1. 外部输入 → InputT 验证
2. InputT → StateT 转换
3. StateT → 节点处理

**示例**：
```python
# 使用 Pydantic 验证输入
class APIInput(BaseModel):
    query: str = Field(..., min_length=1)
    user_id: str

class State(TypedDict):
    messages: list[str]
    user_id: str

graph = StateGraph(State, input_schema=APIInput)

# 调用时自动验证
try:
    result = graph.invoke({"query": "test", "user_id": "123"})
except ValidationError as e:
    print(f"Invalid input: {e}")
```

## 5. OutputT：输出类型

### 5.1 基本概念

**OutputT** 定义了图的输出类型，用于控制返回给外部的数据。

**特性**：
- 可选参数，默认使用 StateT
- 用于过滤和转换输出
- 支持与 StateT 不同的类型

### 5.2 基础用法

```python
from typing_extensions import TypedDict
from pydantic import BaseModel
from langgraph.graph import StateGraph

# 内部状态（包含所有字段）
class InternalState(TypedDict):
    messages: list[str]
    user_id: str
    internal_data: dict
    debug_info: dict

# 外部输出（只暴露部分字段）
class APIOutput(BaseModel):
    messages: list[str]
    user_id: str

# 创建图（StateT = InternalState, OutputT = APIOutput）
graph = StateGraph(
    state_schema=InternalState,
    output_schema=APIOutput
)

def process_node(state: InternalState) -> dict:
    return {
        "messages": ["Result"],
        "internal_data": {"key": "value"},
        "debug_info": {"step": 1}
    }

graph.add_node("process", process_node)

# 输出只包含 APIOutput 定义的字段
result = graph.invoke({"messages": [], "user_id": "123"})
# result = {"messages": ["Result"], "user_id": "123"}
# internal_data 和 debug_info 不会暴露
```

### 5.3 OutputT 的使用场景

**适合使用 OutputT 的场景**：
- 隐藏内部实现细节
- 过滤敏感信息
- 简化 API 响应
- 格式化输出数据

**示例**：
```python
# 内部状态包含敏感信息
class InternalState(TypedDict):
    user_id: str
    api_key: str  # 敏感
    internal_cache: dict  # 内部
    result: str

# 输出只暴露结果
class PublicOutput(TypedDict):
    user_id: str
    result: str

graph = StateGraph(InternalState, output_schema=PublicOutput)
```

## 6. 类型参数的组合使用

### 6.1 完整示例

```python
from typing_extensions import TypedDict, Annotated
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph

# 1. 内部状态（TypedDict）
def add_messages(left: list, right: list) -> list:
    return left + right

class InternalState(TypedDict):
    messages: Annotated[list[str], add_messages]
    user_id: str
    context: dict
    results: list[dict]
    debug_info: dict

# 2. 上下文（TypedDict）
class Context(TypedDict):
    api_key: str
    max_retries: int
    timeout: int

# 3. 输入（Pydantic）
class APIInput(BaseModel):
    query: str = Field(..., min_length=1, max_length=1000)
    user_id: str = Field(..., pattern=r'^[a-zA-Z0-9_-]+$')

# 4. 输出（Pydantic）
class APIOutput(BaseModel):
    messages: list[str]
    results: list[dict]

# 创建图（使用所有 4 个类型参数）
graph = StateGraph(
    state_schema=InternalState,
    context_schema=Context,
    input_schema=APIInput,
    output_schema=APIOutput
)

def process_node(state: InternalState, *, context: Context) -> dict:
    # 读取 context
    max_retries = context["max_retries"]

    # 处理逻辑
    return {
        "messages": [f"Processed with {max_retries} retries"],
        "results": [{"id": 1, "score": 0.9}],
        "debug_info": {"step": 1}
    }

graph.add_node("process", process_node)
graph.set_entry_point("process")
graph.set_finish_point("process")

app = graph.compile()

# 使用
try:
    # 输入验证（APIInput）
    result = app.invoke(
        {"query": "test", "user_id": "user123"},
        config={"configurable": {
            "api_key": "secret",
            "max_retries": 3,
            "timeout": 30
        }}
    )
    # 输出过滤（APIOutput）
    print(result)  # {"messages": [...], "results": [...]}
    # debug_info 不会暴露
except Exception as e:
    print(f"Error: {e}")
```

[来源: reference/source_状态类型系统_01.md | sourcecode/langgraph/.../state.py]

### 6.2 类型参数的关系

```
外部输入 (APIInput)
    ↓ 验证和转换
内部状态 (InternalState) + 上下文 (Context)
    ↓ 节点处理
内部状态 (InternalState)
    ↓ 过滤和转换
外部输出 (APIOutput)
```

## 7. 类型推断机制

### 7.1 节点输入类型推断

LangGraph 支持从节点函数签名自动推断输入类型：

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph

class State(TypedDict):
    x: int
    y: str

class NodeInput(TypedDict):
    x: int

graph = StateGraph(State)

# 方式 1：显式指定 input_schema
def node1(state: NodeInput) -> dict:
    return {"x": state["x"] + 1}

graph.add_node("node1", node1, input_schema=NodeInput)

# 方式 2：自动推断（从函数签名）
def node2(state: NodeInput) -> dict:
    return {"x": state["x"] + 1}

graph.add_node("node2", node2)  # 自动推断 input_schema = NodeInput
```

[来源: reference/source_状态类型系统_01.md | sourcecode/langgraph/.../state.py]

**推断逻辑**（源码）：
```python
def add_node(
    self,
    node: str | StateNode[NodeInputT, ContextT],
    action: StateNode[NodeInputT, ContextT] | None = None,
    *,
    input_schema: type[NodeInputT] | None = None,
) -> Self:
    inferred_input_schema = None

    try:
        if (
            isfunction(action)
            or ismethod(action)
            or ismethod(getattr(action, "__call__", None))
        ) and (
            hints := get_type_hints(getattr(action, "__call__"))
            or get_type_hints(action)
        ):
            if input_schema is None:
                first_parameter_name = next(
                    iter(
                        inspect.signature(
                            cast(FunctionType, action)
                        ).parameters.keys()
                    )
                )
                if input_hint := hints.get(first_parameter_name):
                    if isinstance(input_hint, type) and get_type_hints(input_hint):
                        inferred_input_schema = input_hint
    except (NameError, TypeError, StopIteration):
        pass

    # 优先级：显式 input_schema > 推断的类型 > 图的 state_schema
    if input_schema is not None:
        # 使用显式指定的类型
        pass
    elif inferred_input_schema is not None:
        # 使用推断的类型
        input_schema = inferred_input_schema
    else:
        # 使用图的 state_schema
        input_schema = self.state_schema
```

[来源: reference/source_状态类型系统_01.md | sourcecode/langgraph/.../state.py]

### 7.2 推断优先级

**类型推断优先级**：
1. 显式指定的 `input_schema` 参数
2. 从函数签名推断的类型
3. 图的 `state_schema`

**示例**：
```python
class State(TypedDict):
    x: int
    y: str

class NodeInput(TypedDict):
    x: int

graph = StateGraph(State)

# 优先级 1：显式指定（最高优先级）
def node1(state: State) -> dict:
    return {"x": 1}

graph.add_node("node1", node1, input_schema=NodeInput)
# 使用 NodeInput，忽略函数签名中的 State

# 优先级 2：推断（中等优先级）
def node2(state: NodeInput) -> dict:
    return {"x": 1}

graph.add_node("node2", node2)
# 推断出 input_schema = NodeInput

# 优先级 3：默认（最低优先级）
def node3(state) -> dict:  # 无类型注解
    return {"x": 1}

graph.add_node("node3", node3)
# 使用 State（图的 state_schema）
```

## 8. 最佳实践

### 8.1 类型参数选择建议

**决策树**：
```
开始
  |
  ├─ 需要验证外部输入？
  |   ├─ 是 → 使用 InputT（Pydantic）
  |   └─ 否 → InputT = StateT
  |
  ├─ 需要隐藏内部数据？
  |   ├─ 是 → 使用 OutputT
  |   └─ 否 → OutputT = StateT
  |
  └─ 需要只读上下文？
      ├─ 是 → 使用 ContextT
      └─ 否 → ContextT = None
```

### 8.2 常见模式

**模式 1：简单图（只使用 StateT）**
```python
class State(TypedDict):
    messages: list[str]

graph = StateGraph(State)
```

**模式 2：API 边界验证（StateT + InputT + OutputT）**
```python
class InternalState(TypedDict):
    messages: list[str]
    internal_data: dict

class APIInput(BaseModel):
    query: str = Field(..., min_length=1)

class APIOutput(BaseModel):
    messages: list[str]

graph = StateGraph(
    state_schema=InternalState,
    input_schema=APIInput,
    output_schema=APIOutput
)
```

**模式 3：多租户系统（StateT + ContextT）**
```python
class State(TypedDict):
    messages: list[str]

class Context(TypedDict):
    tenant_id: str
    user_id: str

graph = StateGraph(State, context_schema=Context)
```

**模式 4：完整企业级（所有类型参数）**
```python
graph = StateGraph(
    state_schema=InternalState,
    context_schema=Context,
    input_schema=APIInput,
    output_schema=APIOutput
)
```

### 8.3 性能考虑

**类型参数对性能的影响**：
- StateT：无影响（必需）
- ContextT：轻微影响（额外的参数传递）
- InputT：中等影响（Pydantic 验证）
- OutputT：轻微影响（字段过滤）

**优化建议**：
- 内部状态使用 TypedDict（性能最优）
- 边界验证使用 Pydantic（安全性优先）
- 避免过度使用类型参数

## 9. 常见问题

### Q1: 为什么需要 4 个类型参数？

**A**:
- **StateT**：内部状态，节点间通信
- **ContextT**：只读上下文，配置和元数据
- **InputT**：输入验证，API 边界安全
- **OutputT**：输出过滤，隐藏内部细节

每个类型参数解决不同的问题，提供灵活性和类型安全。

### Q2: InputT 和 StateT 有什么区别？

**A**:
- **InputT**：外部输入类型，用于验证
- **StateT**：内部状态类型，用于节点通信

InputT 会被转换为 StateT，两者可以不同。

### Q3: 什么时候使用 ContextT？

**A**:
- 需要在所有节点间共享只读数据
- 数据不应该被节点修改
- 典型场景：用户信息、配置、API 密钥

### Q4: 如何在节点中访问 Context？

**A**:
```python
def node(state: State, *, context: Context) -> dict:
    # 使用 * 分隔位置参数和关键字参数
    user_id = context["user_id"]
    return {"messages": [f"Hello {user_id}"]}
```

## 10. 总结

### 10.1 核心要点

1. **StateT**：必需，内部状态类型
2. **ContextT**：可选，只读上下文
3. **InputT**：可选，输入验证（默认 = StateT）
4. **OutputT**：可选，输出过滤（默认 = StateT）

### 10.2 使用原则

- **简单场景**：只使用 StateT
- **API 边界**：使用 InputT + OutputT
- **多租户**：使用 ContextT
- **企业级**：使用所有类型参数

### 10.3 类型安全

泛型类型系统提供：
- 编译时类型检查
- IDE 自动补全
- 运行时验证（Pydantic）
- 清晰的 API 边界
