## 5个核心知识点

### 知识点1：混合检索 = 向量检索 + BM25检索

**核心理解：**
```
混合检索 = 稠密向量检索（语义匹配） + 稀疏向量检索（关键词匹配）
```

**为什么需要？**
- 向量检索：理解"意思相近"，但可能漏掉精确关键词
- BM25检索：匹配"字面相同"，但可能漏掉同义词
- 混合检索：两者结合，提升检索质量

**类比：**
- 向量检索 = 看整体轮廓（模糊匹配）
- BM25检索 = 看细节特征（精确匹配）
- 混合检索 = 双眼视觉（立体感知）

---

### 知识点2：Milvus 2.5+内置BM25

**核心理解：**
Milvus 2.5+内置BM25功能，无需外部搜索引擎（如Elasticsearch）。

**Schema配置：**
```python
from pymilvus import MilvusClient, DataType, Function, FunctionType

# 1. 创建Schema
schema = client.create_schema()

# 2. 添加字段
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True, auto_id=True)
schema.add_field(field_name="text", datatype=DataType.VARCHAR, max_length=1000, enable_analyzer=True)  # 启用分词
schema.add_field(field_name="dense_vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="sparse_vector", datatype=DataType.SPARSE_FLOAT_VECTOR)

# 3. 定义BM25函数（自动生成稀疏向量）
bm25_function = Function(
    name="text_bm25_emb",
    input_field_names=["text"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)
schema.add_function(bm25_function)
```

**关键点：**
- `enable_analyzer=True`：启用文本分词
- `SPARSE_FLOAT_VECTOR`：稀疏向量类型
- `Function`：自动将text转换为sparse_vector

### 知识点3：RRF重排序算法

**核心理解：**
RRF（Reciprocal Rank Fusion）通过排名位置融合多个检索结果。

**公式：**
```
score_rrf(d) = Σ(1 / (k + rank_i(d)))
```

**参数：**
- `d`：文档
- `k`：平滑参数（通常为60）
- `rank_i(d)`：文档d在第i个检索结果中的排名

**示例：**
```python
# 向量检索结果
vector_results = {"doc1": rank=1, "doc2": rank=2, "doc3": rank=3}

# BM25检索结果
bm25_results = {"doc2": rank=1, "doc1": rank=2, "doc4": rank=3}

# RRF计算（k=60）
doc1_score = 1/(60+1) + 1/(60+2) = 0.0325
doc2_score = 1/(60+2) + 1/(60+1) = 0.0325
doc3_score = 1/(60+3) + 0 = 0.0159
doc4_score = 0 + 1/(60+3) = 0.0159

# 最终排序：doc1, doc2, doc3, doc4
```

**为什么有效？**
- 排名位置比分数更可靠（不同检索方式的分数量纲不同）
- 平滑参数k避免排名靠后的文档权重过低
- 民主化融合，每个检索方式贡献相等

### 知识点4：AnnSearchRequest构建检索请求

**核心理解：**
`AnnSearchRequest`用于构建单个检索请求，`hybrid_search`用于执行多个请求并融合结果。

**代码结构：**
```python
from pymilvus import AnnSearchRequest, RRFRanker

# 1. 稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[dense_query_vector],           # 查询向量
    anns_field="dense_vector",           # 向量字段名
    param={"metric_type": "COSINE", "params": {"ef": 64}},  # 检索参数
    limit=20                             # 召回数量
)

# 2. 稀疏向量检索请求（BM25）
sparse_req = AnnSearchRequest(
    data=[query_text],                   # 查询文本
    anns_field="sparse_vector",          # 稀疏向量字段名
    param={"metric_type": "BM25"},       # BM25度量
    limit=20
)

# 3. 混合检索
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[dense_req, sparse_req],        # 多个检索请求
    ranker=RRFRanker(k=60),              # RRF重排序
    limit=10,                            # 最终返回数量
    output_fields=["text"]               # 返回字段
)
```

**关键点：**
- `data`：稠密向量传向量，稀疏向量传文本
- `limit`：每个检索请求的召回数量（通常设为最终返回数量的2倍）
- `ranker`：重排序策略（RRF或Weighted）

### 知识点5：索引配置

**核心理解：**
稠密向量和稀疏向量需要不同的索引类型。

**代码：**
```python
# 创建索引参数
index_params = client.prepare_index_params()

# 稠密向量索引（HNSW）
index_params.add_index(
    field_name="dense_vector",
    index_type="HNSW",
    metric_type="COSINE",
    params={"M": 16, "efConstruction": 200}
)

# 稀疏向量索引（倒排索引）
index_params.add_index(
    field_name="sparse_vector",
    index_type="SPARSE_INVERTED_INDEX",
    metric_type="BM25"
)

# 创建Collection
client.create_collection(
    collection_name="my_collection",
    schema=schema,
    index_params=index_params
)
```

**索引选择：**
- 稠密向量：`HNSW`（推荐）、`IVF_FLAT`、`GPU_CAGRA`
- 稀疏向量：`SPARSE_INVERTED_INDEX`（BM25专用）

## 最小可用代码（50行）

### 完整示例：文档问答系统

```python
from pymilvus import MilvusClient, DataType, Function, FunctionType, AnnSearchRequest, RRFRanker
from sentence_transformers import SentenceTransformer

# 1. 初始化
client = MilvusClient(uri="http://localhost:19530")
model = SentenceTransformer('all-MiniLM-L6-v2')

# 2. 创建Schema
schema = client.create_schema()
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True, auto_id=True)
schema.add_field(field_name="text", datatype=DataType.VARCHAR, max_length=1000, enable_analyzer=True)
schema.add_field(field_name="dense_vector", datatype=DataType.FLOAT_VECTOR, dim=384)
schema.add_field(field_name="sparse_vector", datatype=DataType.SPARSE_FLOAT_VECTOR)

bm25_function = Function(
    name="text_bm25_emb",
    input_field_names=["text"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)
schema.add_function(bm25_function)

# 3. 创建索引
index_params = client.prepare_index_params()
index_params.add_index(field_name="dense_vector", index_type="HNSW", metric_type="COSINE", params={"M": 16, "efConstruction": 200})
index_params.add_index(field_name="sparse_vector", index_type="SPARSE_INVERTED_INDEX", metric_type="BM25")

# 4. 创建Collection
client.create_collection(collection_name="docs", schema=schema, index_params=index_params)

# 5. 插入数据
docs = [
    "Python是一种高级编程语言，广泛用于数据科学和机器学习。",
    "机器学习是人工智能的一个分支，通过数据训练模型。",
    "深度学习使用神经网络处理复杂的模式识别任务。"
]

data = [{"text": doc, "dense_vector": model.encode(doc).tolist()} for doc in docs]
client.insert(collection_name="docs", data=data)

# 6. 混合检索
query = "如何学习Python机器学习？"
query_vector = model.encode(query).tolist()

dense_req = AnnSearchRequest(data=[query_vector], anns_field="dense_vector", param={"metric_type": "COSINE", "params": {"ef": 64}}, limit=10)
sparse_req = AnnSearchRequest(data=[query], anns_field="sparse_vector", param={"metric_type": "BM25"}, limit=10)

results = client.hybrid_search(collection_name="docs", reqs=[dense_req, sparse_req], ranker=RRFRanker(k=60), limit=3, output_fields=["text"])

# 7. 输出结果
for hits in results:
    for hit in hits:
        print(f"Score: {hit['score']:.4f}, Text: {hit['entity']['text']}")
```

**输出示例：**
```
Score: 0.0325, Text: Python是一种高级编程语言，广泛用于数据科学和机器学习。
Score: 0.0325, Text: 机器学习是人工智能的一个分支，通过数据训练模型。
Score: 0.0159, Text: 深度学习使用神经网络处理复杂的模式识别任务。
```

## 你能做什么？

### 场景1：文档问答系统

**需求：** 用户提问，从知识库中检索相关文档

**实现：**
```python
def rag_search(query: str, top_k: int = 5):
    query_vector = model.encode(query).tolist()
    
    dense_req = AnnSearchRequest(data=[query_vector], anns_field="dense_vector", param={"metric_type": "COSINE", "params": {"ef": 64}}, limit=20)
    sparse_req = AnnSearchRequest(data=[query], anns_field="sparse_vector", param={"metric_type": "BM25"}, limit=20)
    
    results = client.hybrid_search(collection_name="docs", reqs=[dense_req, sparse_req], ranker=RRFRanker(k=60), limit=top_k, output_fields=["text"])
    
    return [hit['entity']['text'] for hits in results for hit in hits]

# 使用
context = rag_search("如何优化Python代码性能？", top_k=3)
print("\n".join(context))
```

## 常见问题

### Q1: 为什么插入数据时不需要提供sparse_vector？

**答案：**
因为定义了BM25函数，Milvus会自动将`text`字段转换为`sparse_vector`。

```python
bm25_function = Function(
    name="text_bm25_emb",
    input_field_names=["text"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)
```

---

### Q2: 为什么limit设为20，最终返回10个？

**答案：**
- `limit=20`：每个检索请求召回20个候选
- `limit=10`：RRF重排序后返回10个最终结果

**推荐：** 召回数量设为最终返回数量的2倍

---

### Q3: 如何选择k值？

**答案：**
- `k=60`：默认值，适合大多数场景
- `k`越大：排名靠后的文档权重越低
- `k`越小：排名靠后的文档权重越高

**建议：** 使用默认值60，除非有特殊需求

---

### Q4: 如何提升检索速度？

**答案：**
1. **索引优化**：使用HNSW索引（推荐）
2. **批量检索**：一次传入多个查询向量
3. **Pre-filter**：高选择性过滤条件放在检索前
4. **减少召回数量**：limit设为最终返回数量的1.5-2倍

---

### Q5: 如何提升检索准确率？

**答案：**
1. **增加召回数量**：limit设为最终返回数量的2-3倍
2. **优化Embedding模型**：选择更好的模型（如BGE-M3）
3. **调整重排序策略**：尝试Weighted Ranker
4. **优化Chunking**：合理分块文档
