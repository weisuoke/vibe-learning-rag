# 核心概念 4：长期记忆与跨会话

> Store + 语义搜索 + LangMem — 让 Agent 拥有跨对话的持久记忆

---

## 一句话定义

**长期记忆是 Agent 在不同对话之间记住用户偏好、历史事实和学到的知识的能力，通过 LangGraph Store 或第三方工具（LangMem/Mem0/Zep）实现。**

---

## 短期 vs 长期记忆：为什么还需要长期记忆？

### 短期记忆的局限

```
对话 1 (thread-1): "我对花生过敏"
    → Agent 记住了 ✓

对话 2 (thread-2): "帮我推荐个餐厅"
    → Agent 推荐了花生料理 ✗  ← 忘了用户过敏！

原因：Checkpointer 的记忆是 thread 级别的
      不同 thread_id 之间互相隔离
```

### 长期记忆解决的问题

```
短期记忆: 像便签纸 → 对话结束就扔了
长期记忆: 像笔记本 → 永久保存，随时翻阅

┌────────────────────────────────────────────────┐
│  长期记忆用途                                    │
│                                                │
│  1. 用户画像：姓名、偏好、习惯                     │
│  2. 历史事实：做过什么决定、讨论过什么                │
│  3. 学到的知识：用户纠正过的错误、特定术语            │
│  4. 系统优化：哪些回答策略效果好                    │
└────────────────────────────────────────────────┘
```

| 类型 | 人类记忆 | AI 实现 | 示例 |

| **语义记忆** | 知识和事实 | 存储用户事实 | "用户叫张三，对花生过敏" |

## 方案一：LangGraph Store（官方推荐）

### 核心概念

```
Store = 跨线程的键值存储 + 可选的语义搜索

与 Checkpointer 的区别：
  Checkpointer: 自动保存，thread 内
  Store: 手动读写，跨 thread

数据组织方式：
  namespace: ("user_123", "memories")  ← 谁的记忆
  key: "fact_1"                        ← 记忆的唯一 ID
  value: {"text": "用户对花生过敏"}      ← 记忆内容
```

### 基本用法

```python
from langgraph.store.memory import InMemoryStore

# 创建 Store
store = InMemoryStore()

# 存储记忆
store.put(
    namespace=("user_123", "facts"),   # 命名空间
    key="allergy",                      # 键
    value={"text": "用户对花生过敏", "importance": "high"}
)

store.put(
    namespace=("user_123", "facts"),
    key="name",
    value={"text": "用户名叫张三", "importance": "medium"}
)

store.put(
    namespace=("user_123", "preferences"),
    key="style",
    value={"text": "喜欢简洁的回答风格", "importance": "medium"}
)

# 读取特定记忆
item = store.get(("user_123", "facts"), "allergy")
print(item.value)  # {"text": "用户对花生过敏", "importance": "high"}

# 列出所有记忆
items = store.search(("user_123", "facts"))
for item in items:
    print(f"{item.key}: {item.value['text']}")
```

### 带语义搜索的 Store

```python
from langgraph.store.memory import InMemoryStore
from langchain.embeddings import init_embeddings

# 创建带 Embedding 的 Store（支持语义搜索）
embeddings = init_embeddings("openai:text-embedding-3-small")
store = InMemoryStore(
    index={
        "embed": embeddings,
        "dims": 1536,
    }
)

# 存储记忆
store.put(("user_123", "memories"), "1", {"text": "我喜欢吃意大利面"})
store.put(("user_123", "memories"), "2", {"text": "我在一家科技公司工作"})
store.put(("user_123", "memories"), "3", {"text": "我周末喜欢打篮球"})
store.put(("user_123", "memories"), "4", {"text": "我对海鲜过敏"})

# 语义搜索 — 不用精确匹配关键词
results = store.search(
    ("user_123", "memories"),
    query="推荐什么食物",   # ← 语义相关搜索
    limit=2
)
for r in results:
    print(f"[{r.score:.2f}] {r.value['text']}")
# → [0.92] 我喜欢吃意大利面
# → [0.87] 我对海鲜过敏
```

### 在 Agent 中集成 Store

```python
from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.store.memory import InMemoryStore
from langgraph.graph import MessagesState, StateGraph
from langgraph.runtime import Runtime
from langchain.embeddings import init_embeddings

# 初始化
embeddings = init_embeddings("openai:text-embedding-3-small")
checkpointer = InMemorySaver()
store = InMemoryStore(index={"embed": embeddings, "dims": 1536})

# 自定义节点：使用长期记忆
async def chat_with_memory(state: MessagesState, runtime: Runtime):
    user_msg = state["messages"][-1].content
    user_id = runtime.config["configurable"].get("user_id", "default")

    # 1. 从长期记忆中检索相关信息
    memories = await runtime.store.asearch(
        (user_id, "memories"),
        query=user_msg,
        limit=5
    )
    memory_text = "\n".join(
        f"- {m.value['text']}" for m in memories
    ) or "暂无历史记忆"

    # 2. 构建带记忆的 Prompt
    system_prompt = f"""你是一个贴心的助手。

关于这个用户的已知信息：
{memory_text}

请基于这些信息提供个性化回答。"""

    # 3. 调用 LLM
    response = await llm.ainvoke([
        {"role": "system", "content": system_prompt},
        *state["messages"]
    ])

    # 4. 提取新的记忆（可选：让 LLM 判断哪些值得记住）
    # 简单版：直接保存用户消息
    from uuid import uuid4
    await runtime.store.aput(
        (user_id, "memories"),
        str(uuid4()),
        {"text": user_msg, "timestamp": "2026-03-06"}
    )

    return {"messages": [response]}

# 构建图
builder = StateGraph(MessagesState)
builder.add_node("chat", chat_with_memory)
builder.set_entry_point("chat")
graph = builder.compile(checkpointer=checkpointer, store=store)

# 使用 — 跨线程也能记住
config1 = {"configurable": {"thread_id": "t1", "user_id": "user_123"}}
graph.invoke({"messages": [{"role": "user", "content": "我对花生过敏"}]}, config1)

config2 = {"configurable": {"thread_id": "t2", "user_id": "user_123"}}
graph.invoke({"messages": [{"role": "user", "content": "推荐个午餐"}]}, config2)
# → 会避免推荐花生相关的食物（从 Store 中检索到过敏信息）
```

## 方案二：LangMem SDK

### 什么是 LangMem？

LangMem 是 LangChain 在 2025 年推出的专用长期记忆 SDK：

```
LangMem = Store 的高级封装

提供：
  - 自动记忆提取（LLM 判断什么值得记住）
  - 记忆去重和更新（不重复存储相同信息）
  - 记忆衰减（旧记忆权重降低）
  - 结构化记忆组织
```

[来源: atom/langchain/L4_Agent系统/07_Agent Memory集成/reference/search_agent_memory_02.md]

### 基本概念

```python
# LangMem 的核心功能（概念示例）

# 1. 自动提取 — LLM 判断哪些对话值得记住
memory_extractor = LangMemExtractor(model="gpt-4.1-mini")
facts = memory_extractor.extract(conversation)
# → ["用户名张三", "对花生过敏", "喜欢简洁回答"]

# 2. 智能更新 — 新信息覆盖旧信息
# 旧记忆: "用户住在北京"
# 新对话: "我最近搬到上海了"
# → 自动更新为: "用户住在上海（之前住北京）"

# 3. 记忆检索 — 基于当前对话检索相关记忆
relevant = memory_store.search(
    query="推荐个餐厅",
    user_id="user_123"
)
# → ["用户对花生过敏", "喜欢意大利菜"]
```

### LangMem vs 手动 Store

| 维度 | 手动 Store | LangMem |
|------|-----------|---------|
| **记忆提取** | 手动编写逻辑 | LLM 自动提取 |
| **去重更新** | 需自行实现 | 内置 |
| **记忆衰减** | 无 | 内置 |
| **使用成本** | 低 | 有 LLM 调用费 |
| **灵活性** | 高（完全控制） | 中（框架约束） |
| **适用** | 简单场景 | 复杂记忆需求 |

## 长期记忆的设计模式

### 模式 1：被动记忆（读多写少）

```
用户注册时，管理员预存用户画像到 Store
Agent 每次对话前读取，但不自动写入

适用：客服系统、企业助手
```

### 模式 2：主动记忆（自动提取）

```
每次对话后，LLM 自动判断是否有值得记住的信息
如果有，自动存入 Store

适用：个人助手、学习系统
```

### 模式 3：反思记忆（定期总结）

```
后台任务定期回顾所有对话历史
提取模式、总结用户特征
更新长期记忆

适用：高级 Agent、个性化推荐
```

## 长期记忆的注意事项

### 1. 隐私保护

```python
# 敏感信息处理
async def save_memory_safely(store, user_id, text):
    """保存记忆前过滤敏感信息"""
    # 不存储密码、银行卡号等
    sensitive_patterns = [r'\d{16}', r'密码', r'password']
    for pattern in sensitive_patterns:
        if re.search(pattern, text, re.IGNORECASE):
            return  # 跳过敏感信息

    await store.aput(
        (user_id, "memories"),
        str(uuid4()),
        {"text": text, "created_at": datetime.now().isoformat()}
    )
```

### 2. 记忆容量管理

```python
# 定期清理旧记忆
async def cleanup_old_memories(store, user_id, max_memories=100):
    """保留最近 N 条记忆"""
    items = store.search((user_id, "memories"))
    if len(list(items)) > max_memories:
        # 按时间排序，删除最旧的
        sorted_items = sorted(items, key=lambda x: x.value.get("created_at", ""))
        for item in sorted_items[:-max_memories]:
            store.delete((user_id, "memories"), item.key)
```

### 3. 记忆一致性

```python
# 新信息与旧记忆冲突时的处理
# 旧: "用户住在北京"  新: "我搬到上海了"
# → 应该更新，而不是同时保留两条矛盾的记忆

async def update_or_create_memory(store, user_id, new_fact, llm):
    """检查是否与现有记忆冲突，如果是则更新"""
    existing = await store.asearch(
        (user_id, "memories"),
        query=new_fact,
        limit=3
    )

    for item in existing:
        if item.score > 0.85:  # 高度相似 → 可能需要更新
            # 让 LLM 判断是否矛盾
            result = await llm.ainvoke(
                f"这两条信息是否矛盾？\n"
                f"旧: {item.value['text']}\n"
                f"新: {new_fact}\n"
                f"回答 '是' 或 '否'"
            )
            if "是" in result.content:
                # 更新旧记忆
                store.put(
                    (user_id, "memories"),
                    item.key,
                    {"text": new_fact, "updated_at": datetime.now().isoformat()}
                )
                return

    # 没有冲突，创建新记忆
    store.put(
        (user_id, "memories"),
        str(uuid4()),
        {"text": new_fact, "created_at": datetime.now().isoformat()}
    )
```

## 与 RAG 开发的关系

| RAG 场景 | 长期记忆用途 | 实现方式 |
|----------|-------------|----------|
| **个性化文档问答** | 记住用户常问的领域 | Store + 偏好标签 |
| **知识库助手** | 记住用户已读过的文档 | Store + 阅读记录 |
| **智能客服** | 记住用户历史工单 | Store + 工单摘要 |
| **学习系统** | 记住用户掌握程度 | Store + 知识图谱 |

**长期记忆 + RAG 的协同**：

```
用户提问 "如何优化数据库性能？"

1. 从长期记忆检索：用户是 DBA，熟悉 PostgreSQL
2. 从 RAG 知识库检索：数据库优化的相关文档
3. 结合两者：给出针对 PostgreSQL DBA 的高级优化建议
   （而不是给初学者的基础教程）
```

## 本节小结

> **长期记忆让 Agent 从「每次从零开始」变成「了解你的个人助手」。**
> - **LangGraph Store**：官方推荐，支持语义搜索，与 Checkpointer 无缝配合
