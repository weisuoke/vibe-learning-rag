# 一句话总结：Struct 与 Enum 全景速记

> **一句话定位：** 用一句话浓缩每个核心概念的本质，作为复习和速查的终极索引。

---

## 总纲

**Struct 是 AND（所有字段同时存在），Enum 是 OR（同一时刻只有一种变体），match 是编译器强制你处理每种 OR 的机制。**

---

## 逐概念总结

| # | 概念 | 一句话 |
|---|------|--------|
| 1 | Struct 定义 | 用命名字段把相关数据绑在一起，创建时必须填满所有字段——没有 undefined。 |
| 2 | impl 块 | 数据（struct）和行为（方法）分开写，`Self` 是当前类型，`&self` 是只读借用。 |
| 3 | Enum 变体 | 每个变体可以携带完全不同的数据，是 TypeScript discriminated union 的编译器原生版。 |
| 4 | match | 强制穷尽的 switch——漏掉任何变体都编译不过，新增变体时编译器帮你找到所有需要更新的地方。 |
| 5 | derive 宏 | 一行 `#[derive(...)]` 让编译器自动生成 Debug/Clone/Copy/PartialEq 等实现，零运行时开销。 |
| 6 | 组合嵌套 | Rust 没有继承，用 struct 嵌套 struct + enum 字段实现组合——乐高式拼装，比继承更灵活。 |
| 7 | Option\<T\> | 用 `Some(T)` / `None` 替代 null，编译器强制你处理"没有值"的情况，消灭空指针异常。 |
| 8 | Builder 模式 | 全 Option 字段 + 链式 `mut self` 方法 + `build()` 验证——构造复杂对象的标准姿势。 |
| 9 | Copy vs Clone | Copy 是隐式栈上按位拷贝（小类型），Clone 是显式可深拷贝（任何类型）。 |
| 10 | 内存布局 | Enum 大小 = 最大变体，编译器自动重排 struct 字段减少 padding，Option\<&T\> 零开销。 |

## ZeroClaw 映射

| ZeroClaw 类型 | 对应概念 | 一句话 |
|--------------|---------|--------|
| `Agent` struct | 组合嵌套 | 把 identity、security、tools 组合成一个完整的 Agent |
| `AgentBuilder` | Builder 模式 | Option 字段 + 链式设置 + build() 验证必填项 |
| `ConversationMessage` enum | Enum 变体 | Chat / ToolCalls / ToolResults 三选一，match 强制处理 |

| `AutonomyLevel` enum | 简单 enum | ReadOnly / Supervised / Full，derive(Copy) 因为是小类型 |
| `ObserverEvent` enum | 复杂 enum | 11 个变体各携带不同数据，match 分发到处理器 |
| `SecurityPolicy` struct | 组合 | autonomy_level + max_actions，嵌入 Agent |

## 决策速查

```
要存一组同时存在的数据？          → struct
要表达互斥的状态/类型？           → enum
要给类型加方法？                  → impl 块
要处理 enum 的每种情况？          → match（编译器强制穷尽）
要快速判断是否匹配？              → matches! 宏
字段可能没有值？                  → Option<T>
构造参数多且部分可选？            → Builder 模式
小类型要隐式拷贝？                → derive(Copy, Clone)
要打印调试信息？                  → derive(Debug)
大变体拖累 enum 大小？            → Box 包装大变体
```
