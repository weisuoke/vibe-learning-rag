# 实战代码：场景4 - 多模态对话（文本+图片）

> **目标**：掌握 ChatPromptTemplate 的多模态消息处理，实现文本+图片的对话系统，集成 Vision 模型（GPT-4o）

---

## 场景概述

**适用场景**：
- 图像描述生成
- 图像问答系统
- 多模态 RAG（文档+图表）
- 产品图片分析
- 视觉内容理解

**技术栈**：
- `langchain_core.prompts.ChatPromptTemplate`
- `langchain_openai.ChatOpenAI` (gpt-4o)
- `langchain_core.output_parsers.StrOutputParser`
- `PIL` (图像处理)
- `base64` (图像编码)

**核心概念**：
- **多模态消息**: 在一条消息中同时包含文本和图像
- **图像格式**: URL、本地路径、base64 编码
- **Vision 模型**: 支持视觉理解的 LLM（如 GPT-4o）

## 实战1：基础图像描述生成

### 代码实现

```python
"""
实战1：基础图像描述生成
演示如何使用 ChatPromptTemplate 处理图像 URL
"""

from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser
from dotenv import load_dotenv

# 加载环境变量
load_dotenv()

def demo_image_description():
    """演示基础图像描述"""
    print("=== 实战1：基础图像描述生成 ===\n")

    # 1. 创建多模态模板
    template = ChatPromptTemplate.from_messages([
        ("system", "You are a professional image analyst. Describe images in detail."),
        ("human", [
            {"text": "Please describe this image:"},
            {"image_url": "{image_url}"}
        ])
    ])

    print("1. 创建多模态模板")
    print(f"   输入变量: {template.input_variables}")

    # 2. 配置 Vision 模型
    model = ChatOpenAI(
        model="gpt-4o",  # 支持视觉的模型
        temperature=0
    )
    print("\n2. 配置 Vision 模型: gpt-4o")

    # 3. 创建处理链
    chain = template | model | StrOutputParser()
    print("3. 创建处理链: template | model | parser")

    # 4. 测试图像描述
    # 使用公开的测试图像 URL
    test_images = [
        {
            "url": "https://upload.wikimedia.org/wikipedia/commons/thumb/d/dd/Gfp-wisconsin-madison-the-nature-boardwalk.jpg/2560px-Gfp-wisconsin-madison-the-nature-boardwalk.jpg",
            "name": "自然风景"
        },
        {
            "url": "https://upload.wikimedia.org/wikipedia/commons/thumb/3/3a/Cat03.jpg/1200px-Cat03.jpg",
            "name": "猫咪照片"
        }
    ]

    print("\n4. 测试图像描述:")
    for i, img in enumerate(test_images, 1):
        print(f"\n   [{i}] 图像: {img['name']}")
        print(f"       URL: {img['url'][:60]}...")

        try:
            result = chain.invoke({"image_url": img["url"]})
            print(f"       描述: {result[:200]}...")
        except Exception as e:
            print(f"       错误: {str(e)}")

if __name__ == "__main__":
    demo_image_description()
```

"""
实战2：本地图像处理（Base64 编码）
演示如何处理本地图像文件

from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import StrOutputParser
from dotenv import load_dotenv
import base64
from pathlib import Path

load_dotenv()

def encode_image_to_base64(image_path: str) -> str:
    """将本地图像编码为 base64"""
    with open(image_path, "rb") as image_file:
        return base64.b64encode(image_file.read()).decode('utf-8')

def demo_local_image_processing():
    """演示本地图像处理"""
    print("=== 实战2：本地图像处理（Base64 编码） ===\n")

        ("system", "You are an image analyst. Analyze the image and answer questions."),
        ("human", [
            {"text": "Question: {question}\n\nImage:"},
            {"image_url": "data:image/jpeg;base64,{image_data}"}
        ])
    ])

    print("1. 创建多模态模板（支持 base64）")
    print(f"   输入变量: {template.input_variables}")

    # 2. 配置模型
    model = ChatOpenAI(model="gpt-4o", temperature=0)
    chain = template | model | StrOutputParser()
    print("\n2. 配置模型和处理链")

    # 3. 模拟本地图像处理
    # 注意：这里使用示例，实际使用时需要提供真实的图像路径
    print("\n3. 处理本地图像:")
    print("   注意: 需要提供真实的图像文件路径")

## 最佳实践总结

### 1. 图像格式选择

```python
# 方式1: 公开 URL（推荐用于测试）
{"image_url": "https://example.com/image.jpg"}

# 方式2: Base64 编码（推荐用于本地图像）
{"image_url": f"data:image/jpeg;base64,{base64_data}"}

# 方式3: 文件路径（需要转换为 base64）
image_data = encode_image_to_base64("./image.jpg")
{"image_url": f"data:image/jpeg;base64,{image_data}"}
```

### 2. 图像优化

```python
from PIL import Image
import io
import base64

def optimize_image(image_path: str, max_size: tuple = (1024, 1024)) -> str:
    """优化图像大小"""
    # 打开图像
    img = Image.open(image_path)

    # 调整大小
    img.thumbnail(max_size, Image.Resampling.LANCZOS)

    # 转换为 JPEG
    buffer = io.BytesIO()
    img.convert('RGB').save(buffer, format='JPEG', quality=85)

    # 编码为 base64
    img_str = base64.b64encode(buffer.getvalue()).decode()
    return f"data:image/jpeg;base64,{img_str}"
```

### 3. 模型配置

```python
# 推荐配置
model = ChatOpenAI(
    model="gpt-4o",           # 必须使用支持视觉的模型
    temperature=0,            # 确保输出稳定
    max_tokens=1000           # 根据需求调整

```

### 4. 错误处理

```python
def safe_multimodal_invoke(chain, inputs: dict):
    """安全的多模态调用"""
    try:
        return chain.invoke(inputs)
    except Exception as e:
        if "image" in str(e).lower():
            print(f"图像处理错误: {e}")
            # 降级为纯文本模式
            return "无法处理图像，请检查图像 URL 或格式"
        else:
            raise e
```

## 常见问题

### Q1: 支持哪些图像格式？

**答案**：
- **URL**: JPEG, PNG, GIF, WebP
- **Base64**: JPEG, PNG（推荐 JPEG，文件更小）
- **大小限制**: 建议 < 20MB（实际限制取决于 API）

### Q2: 如何处理多张图像？

**答案**：
```python
# 在一条消息中包含多张图像
("human", [
    {"text": "Compare these images:"},
    {"image_url": "{image1}"},
    {"image_url": "{image2}"},
    {"image_url": "{image3}"},
    {"text": "What are the differences?"}
])
```

### Q3: 图像处理的 Token 消耗？

**答案**：
- **低分辨率** (detail="low"): ~85 tokens
- **高分辨率** (detail="high"): 根据图像大小，可能数百到数千 tokens
- **建议**: 非必要不使用高分辨率

### Q4: 如何在 RAG 中集成图像？

**答案**：
1. 在文档索引时保存图像 URL 或路径
2. 检索时同时返回文本和图像引用
3. 使用多模态模板将文本和图像一起传递给 LLM
