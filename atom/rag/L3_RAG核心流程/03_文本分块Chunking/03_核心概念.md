# 核心概念

掌握三种最常用的分块策略，覆盖 90% 的 RAG 开发场景。

---

## 核心概念1：固定大小分块（Fixed-size Chunking）

**按固定字符数或 Token 数切分文本，是最简单直接的分块方式。**

```python
def fixed_size_chunk(text: str, chunk_size: int = 500, overlap: int = 50) -> list[str]:
    """
    固定大小分块

    Args:
        text: 原始文本
        chunk_size: 每个块的大小（字符数）
        overlap: 相邻块的重叠部分（字符数）

    Returns:
        分块后的文本列表
    """
    chunks = []
    start = 0

    while start < len(text):
        end = start + chunk_size
        chunk = text[start:end]
        chunks.append(chunk)
        start = end - overlap  # 重叠部分

    return chunks

# 示例
text = "Python是一种广泛使用的编程语言..." * 100
chunks = fixed_size_chunk(text, chunk_size=200, overlap=20)
print(f"原文长度: {len(text)}, 分块数: {len(chunks)}")
```

### 核心参数

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `chunk_size` | 每个块的大小 | 200-1000 字符 |
| `overlap` | 重叠大小 | chunk_size 的 10-20% |

### 重叠（Overlap）的作用

```
没有重叠：
┌─────────┐┌─────────┐┌─────────┐
│ Chunk 1 ││ Chunk 2 ││ Chunk 3 │
└─────────┘└─────────┘└─────────┘
     ↑           ↑
   边界处的句子可能被切断

有重叠：
┌─────────────┐
│   Chunk 1   │
└───────┬─────┘
        │ 重叠
    ┌───┴─────────┐
    │   Chunk 2   │
    └───────┬─────┘
            │ 重叠
        ┌───┴─────────┐
        │   Chunk 3   │
        └─────────────┘
     ↑
   边界处的内容在两个块中都有
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 实现简单 | ❌ 可能切断句子/段落 |
| ✅ 块大小可控 | ❌ 不考虑语义边界 |
| ✅ 处理速度快 | ❌ 可能切断代码块 |

### 在 RAG 开发中的应用

**适用场景：**
- 纯文本内容（小说、文章）
- 对分块质量要求不高的快速原型
- 文本结构不明显的场景

**不适用场景：**
- 代码文档（会切断函数）
- 结构化文档（会破坏表格）
- 问答对（会分离问题和答案）

## 核心概念2：递归字符分块（Recursive Character Splitting）

**按层级分隔符递归切分，优先在自然边界（段落、句子）处切分，是 LangChain 的默认策略。**

```python
def recursive_split(
    text: str,
    chunk_size: int = 500,
    separators: list[str] = None
) -> list[str]:
    """
    递归字符分块

    按分隔符优先级递归切分：
    1. 先尝试按段落切分（\n\n）
    2. 段落太长则按换行切分（\n）
    3. 还是太长则按句子切分（。！？）
    4. 最后按字符切分
    """
    if separators is None:
        separators = ["\n\n", "\n", "。", "！", "？", ".", "!", "?", " ", ""]

    chunks = []

    def split_recursive(text: str, seps: list[str]) -> list[str]:
        if not seps:
            # 没有分隔符了，直接按字符切分
            return [text[i:i+chunk_size] for i in range(0, len(text), chunk_size)]

        sep = seps[0]
        if sep == "":
            # 空分隔符，按字符切分
            return [text[i:i+chunk_size] for i in range(0, len(text), chunk_size)]

        parts = text.split(sep)
        result = []
        current = ""

        for part in parts:
            if len(current) + len(part) + len(sep) <= chunk_size:
                current += part + sep
            else:
                if current:
                    result.append(current.strip())
                if len(part) > chunk_size:
                    # 这部分还是太长，用下一级分隔符继续切
                    result.extend(split_recursive(part, seps[1:]))
                else:
                    current = part + sep

        if current:
            result.append(current.strip())

        return result

    return split_recursive(text, separators)
```

### 分隔符优先级

```
优先级从高到低：

1. "\n\n"  → 段落边界（最自然的切分点）
2. "\n"    → 换行边界
3. "。"    → 句子边界（中文）
4. "."     → 句子边界（英文）
5. " "     → 单词边界
6. ""      → 字符边界（最后手段）

示例：
┌────────────────────────────────────────┐
│ 第一段内容...                           │
│                                        │
│ 第二段内容...                           │  ← 优先在 \n\n 处切分
│                                        │
│ 第三段是一个很长很长的段落，超过了       │
│ chunk_size，所以需要在句子边界切分。     │  ← 段落太长，在句号处切分
│ 这是第二句。这是第三句。                 │
└────────────────────────────────────────┘
```

### LangChain 实现

```python
from langchain_text_splitters import RecursiveCharacterTextSplitter

# 创建分块器
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,           # 目标块大小
    chunk_overlap=50,         # 重叠大小
    separators=["\n\n", "\n", "。", ".", " ", ""],  # 分隔符优先级
    length_function=len,      # 长度计算函数
)

# 分块
text = """
第一段内容，介绍了Python的基本概念。

第二段内容，详细讲解了变量和数据类型。这是一个比较长的段落，
包含了很多重要的知识点。我们需要理解变量的本质是什么。

第三段内容，讲解了控制流程。
"""

chunks = splitter.split_text(text)
for i, chunk in enumerate(chunks):
    print(f"Chunk {i+1} ({len(chunk)} chars): {chunk[:50]}...")
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 保持语义完整性 | ❌ 块大小不完全均匀 |
| ✅ 在自然边界切分 | ❌ 需要调整分隔符列表 |
| ✅ 适用于多种文档 | ❌ 对特殊格式支持有限 |

### 在 RAG 开发中的应用

**适用场景：**
- 通用文档（文章、报告、书籍）
- Markdown 文档
- 大多数 RAG 应用的默认选择

**最佳实践：**
- 中文文档：添加中文标点到分隔符列表
- 代码文档：添加 `\n\n\n`（函数间空行）
- Markdown：添加 `#`（标题）作为高优先级分隔符

## 核心概念3：语义分块（Semantic Chunking）

**基于语义相似度切分，在语义变化的地方切分，保证每个块内容语义一致。**

```python
import numpy as np
from typing import Callable

def semantic_chunk(
    text: str,
    embed_func: Callable[[str], list[float]],
    threshold: float = 0.5,
    min_chunk_size: int = 100
) -> list[str]:
    """
    语义分块

    原理：
    1. 先按句子切分
    2. 计算相邻句子的语义相似度
    3. 在相似度低于阈值的地方切分

    Args:
        text: 原始文本
        embed_func: 向量化函数
        threshold: 相似度阈值，低于此值则切分
        min_chunk_size: 最小块大小
    """
    # 1. 按句子切分
    import re
    sentences = re.split(r'(?<=[。！？.!?])', text)
    sentences = [s.strip() for s in sentences if s.strip()]

    if len(sentences) <= 1:
        return [text]

    # 2. 计算每个句子的向量
    embeddings = [embed_func(s) for s in sentences]

    # 3. 计算相邻句子的相似度
    def cosine_similarity(a, b):
        return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

    similarities = []
    for i in range(len(embeddings) - 1):
        sim = cosine_similarity(embeddings[i], embeddings[i + 1])
        similarities.append(sim)

    # 4. 在相似度低的地方切分
    chunks = []
    current_chunk = sentences[0]

    for i, sim in enumerate(similarities):
        if sim < threshold and len(current_chunk) >= min_chunk_size:
            # 相似度低，切分
            chunks.append(current_chunk)
            current_chunk = sentences[i + 1]
        else:
            # 相似度高，合并
            current_chunk += sentences[i + 1]

    if current_chunk:
        chunks.append(current_chunk)

    return chunks
```

### 工作原理

```
原文：
"Python是一种编程语言。它简单易学。  ← 相似度高，不切分

 机器学习是AI的分支。深度学习很流行。" ← 相似度高，不切分
      ↑
   这里语义变化大，相似度低，切分！

结果：
Chunk 1: "Python是一种编程语言。它简单易学。"
Chunk 2: "机器学习是AI的分支。深度学习很流行。"
```

### 相似度阈值的影响

```
阈值 = 0.8（高）：
- 切分点多
- 块更小
- 每个块语义更纯粹

阈值 = 0.3（低）：
- 切分点少
- 块更大
- 可能混合多个主题
```

### LangChain 实现

```python
from langchain_experimental.text_splitter import SemanticChunker
from langchain_openai import OpenAIEmbeddings

# 创建语义分块器
embeddings = OpenAIEmbeddings()
semantic_splitter = SemanticChunker(
    embeddings=embeddings,
    breakpoint_threshold_type="percentile",  # 使用百分位数作为阈值
    breakpoint_threshold_amount=95,          # 95% 分位数
)

# 分块
chunks = semantic_splitter.split_text(text)
```

### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 语义完整性最好 | ❌ 需要调用 Embedding API |
| ✅ 块内容主题一致 | ❌ 处理速度慢 |
| ✅ 适合主题多变的文档 | ❌ 成本较高 |

### 在 RAG 开发中的应用

**适用场景：**
- 主题频繁变化的文档
- 对检索质量要求极高的场景
- 文档结构不规则的情况

**不适用场景：**
- 大规模文档处理（成本高）
- 实时处理场景（速度慢）
- 预算有限的项目

## 三种策略对比

| 特性 | 固定大小 | 递归字符 | 语义分块 |
|------|----------|----------|----------|
| **实现复杂度** | ⭐ | ⭐⭐ | ⭐⭐⭐ |
| **处理速度** | 🚀🚀🚀 | 🚀🚀 | 🚀 |
| **语义完整性** | ❌ | ✅ | ✅✅ |
| **块大小可控** | ✅✅ | ✅ | ❌ |
| **API 成本** | 无 | 无 | 高 |
| **推荐场景** | 快速原型 | 通用场景 | 高质量要求 |

## 选择策略的决策树

```
开始
  │
  ├─ 是否需要快速原型？
  │     │
  │     ├─ 是 → 固定大小分块
  │     │
  │     └─ 否 ↓
  │
  ├─ 文档是否有明显结构（段落、标题）？
  │     │
  │     ├─ 是 → 递归字符分块（推荐）
  │     │
  │     └─ 否 ↓
  │
  ├─ 是否对质量要求极高且预算充足？
  │     │
  │     ├─ 是 → 语义分块
  │     │
  │     └─ 否 → 递归字符分块
  │
  └─ 默认推荐：递归字符分块
```
