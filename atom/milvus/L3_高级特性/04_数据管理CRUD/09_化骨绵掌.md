## 卡片1：直觉理解 - CRUD 是什么

**一句话：** CRUD 是数据库的四个基本操作：创建（Create）、读取（Read）、更新（Update）、删除（Delete）。

**举例：**
```python
# Create - 插入数据
client.insert(collection_name, data)

# Read - 查询数据
client.query(collection_name, ids=[1, 2, 3])
client.search(collection_name, data=[vector])

# Update - 更新数据
client.upsert(collection_name, data)

# Delete - 删除数据
client.delete(collection_name, ids=[1, 2, 3])
```

**应用：** 在 RAG 系统中，CRUD 操作管理知识库的完整生命周期：导入文档（Insert）、检索文档（Search）、更新元数据（Upsert）、清理过期数据（Delete）。

---

## 卡片2：形式化定义 - CRUD 操作分类

**一句话：** Milvus 的 CRUD 操作分为写操作（Insert/Upsert/Delete）和读操作（Query/Search）。

**精确表述：**
- **写操作**：修改数据状态，需要 Flush 确保持久化
  - Insert：插入新数据
  - Upsert：插入或更新数据
  - Delete：删除数据
- **读操作**：查询数据，不修改状态
  - Query：基于主键或标量字段的精确查询
  - Search：基于向量相似度的检索

**应用：** 理解操作分类有助于选择合适的 API 和优化性能。

---

## 卡片3：Insert 操作 - 数据插入

**一句话：** Insert 是向 Collection 中插入新数据的操作，支持批量插入。

**举例：**
```python
data = [
    {"id": 1, "vector": [0.1, 0.2], "title": "Doc 1"},
    {"id": 2, "vector": [0.3, 0.4], "title": "Doc 2"}
]
result = client.insert(collection_name, data)
# 返回: {'insert_count': 2, 'ids': [1, 2]}
```

**关键点：**
- 批量插入比单条插入快 30-40倍
- 返回插入数量和 ID 列表
- 支持 AutoID 和手动指定 ID

**应用：** RAG 系统初始化知识库时，使用批量 Insert 导入历史文档。

---

## 卡片4：Query 操作 - 精确查询

**一句话：** Query 是基于主键或标量字段的精确查询，类似 SQL 的 WHERE 条件。

**举例：**
```python
# 按主键查询
result = client.query(collection_name, ids=[1, 2, 3])

# 按表达式查询
result = client.query(
    collection_name,
    filter='category == "tech" and score >= 50',
    output_fields=["id", "title"]
)
```

**关键点：**
- 性能快（5-10ms）
- 返回精确匹配的结果
- 支持复杂过滤表达式

**应用：** RAG 系统中根据文档 ID 获取详情，或根据类别筛选文档。

---

## 卡片5：Search 操作 - 向量检索

**一句话：** Search 是基于向量相似度的检索，返回 Top-K 最相似的结果。

**举例：**
```python
query_vector = [0.1, 0.2, 0.3, ...]
result = client.search(
    collection_name,
    data=[query_vector],
    limit=10,
    output_fields=["id", "title"]
)
# 返回: Top-10 相似文档
```

**关键点：**
- 性能较慢（10-20ms）
- 使用 ANN 算法
- 支持混合检索（向量 + 标量过滤）

**应用：** RAG 系统的核心操作，根据用户查询检索相关文档。

---

## 卡片6：Delete 操作 - 数据删除

**一句话：** Delete 是删除指定数据的操作，支持按主键或表达式删除。

**举例：**
```python
# 按主键删除
client.delete(collection_name, ids=[1, 2, 3])

# 按表达式删除
client.delete(collection_name, filter='score < 50')
```

**关键点：**
- 删除不会立即释放存储空间
- 需要 Compaction 回收空间
- 删除后数据在 Query 和 Search 中不可见

**应用：** RAG 系统中清理过期文档或删除低质量数据。

---

## 卡片7：Upsert 操作 - 插入或更新

**一句话：** Upsert 是插入或更新数据的操作，如果主键存在则更新，否则插入。

**举例：**
```python
data = [
    {"id": 1, "vector": [0.5, 0.6], "title": "Updated Doc 1"}
]
result = client.upsert(collection_name, data)
# 如果 id=1 存在，则更新；否则插入
```

**关键点：**
- 工作流程：检查主键 → 删除旧数据 → 插入新数据
- 性能比 Insert 慢约 2倍
- 支持部分更新（保留原有向量）

**应用：** RAG 系统中更新文档元数据（如标签、分类），但保留原有向量。

---

## 卡片8：对比区分 - Query vs Search

**一句话：** Query 是精确查询，Search 是相似度检索。

**对比表：**
| 特性 | Query | Search |
|------|-------|--------|
| 基于 | 主键/标量字段 | 向量距离 |
| 返回 | 精确匹配 | Top-K 相似 |
| 性能 | 更快（5-10ms） | 较慢（10-20ms） |
| 排序 | 无排序 | 按相似度排序 |
| 场景 | 已知ID或条件 | 语义搜索 |

**应用：** RAG 系统中，Query 用于获取文档详情，Search 用于语义检索。

---

## 卡片9：进阶理解 - 数据一致性

**一句话：** Milvus 提供三种一致性级别：Strong、Bounded、Eventually。

**关键点：**
- **Strong**：插入后立即可见，性能最低
- **Bounded**：插入后在指定时间内可见（默认），性能适中
- **Eventually**：插入后最终可见，性能最高

**常见问题：**
- Insert 后立即 Query 查询不到数据？
- 原因：使用了 Eventually 一致性
- 解决方案：使用 Strong 一致性或手动 Flush

**应用：** RAG 系统中，批量导入后使用 Flush 确保数据可见。

---

## 卡片10：总结与延伸 - 最佳实践

**一句话：** CRUD 操作的最佳实践：批量操作、合理一致性、定期 Compaction。

**核心原则：**
1. **批量操作优于单条操作**：性能提升 30-40倍
2. **使用 output_fields 减少数据传输**：不要返回向量字段
3. **合理选择一致性级别**：平衡性能和准确性
4. **定期 Compaction**：回收删除操作的存储空间
5. **Upsert 后验证数据一致性**：避免已知 Bug

**延伸学习：**
- 混合检索（向量 + 标量过滤）
- 分区管理（Partition）
- 动态 Schema
- 性能调优

**应用：** 在 RAG 系统中，遵循这些最佳实践可以显著提升性能和稳定性。
