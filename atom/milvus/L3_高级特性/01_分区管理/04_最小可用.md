## 4.1 理解分区的本质

**分区 = Collection 的子集**

```python
# 核心概念
Collection C = {entity₁, entity₂, ..., entityₙ}
Partition P ⊆ C
```

**为什么需要分区？**
- 减少检索范围：只搜索相关分区
- 数据隔离：实现租户级别的数据隔离
- 资源优化：只加载需要的分区

## 4.2 选择分区策略

### 方案 1：Partition Key 自动分区（推荐）

**适用场景**：大规模多租户（> 1,024 个租户）

```python
from pymilvus import FieldSchema, CollectionSchema, DataType, Collection

# 定义 Schema，标记 Partition Key
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
        FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=384),
        FieldSchema(name="tenant_id", dtype=DataType.VARCHAR, max_length=64,
                   is_partition_key=True),  # 标记为 Partition Key
        FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=2048)
    ]
)

# 创建 Collection
collection = Collection("my_collection", schema)

# 创建索引
index_params = {
    "index_type": "HNSW",
    "metric_type": "COSINE",
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="vector", index_params=index_params)

# 加载 Collection
collection.load()

# 插入数据（自动路由到相应分区）
data = [
    [[0.1]*384, [0.2]*384, [0.3]*384],  # vector
    ["tenant_a", "tenant_a", "tenant_b"],  # tenant_id
    ["doc1", "doc2", "doc3"]  # content
]
collection.insert(data)

# 搜索租户 A 的数据（自动只搜索相关分区）
query_vector = [[0.15]*384]
results = collection.search(
    data=query_vector,
    anns_field="vector",
    param={"metric_type": "COSINE", "params": {"ef": 100}},
    limit=10,
    expr='tenant_id == "tenant_a"'  # 自动路由
)
```

### 方案 2：手动分区管理

**适用场景**：中等数量租户（< 1,024 个租户）

```python
from pymilvus import Collection

collection = Collection("my_collection")

# 创建分区
collection.create_partition("tenant_a")
collection.create_partition("tenant_b")

# 插入数据到指定分区
data_a = [
    [1, 2, 3],  # id
    [[0.1]*128, [0.2]*128, [0.3]*128],  # vector
    ["doc1", "doc2", "doc3"]  # content
]
collection.insert(data_a, partition_name="tenant_a")

# 加载分区
collection.load(partition_names=["tenant_a"])

# 搜索指定分区
query_vector = [[0.15]*128]
results = collection.search(
    data=query_vector,
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=["tenant_a"]  # 只搜索 tenant_a
)
```

## 4.3 核心操作

### 创建分区

```python
collection.create_partition("partition_name")
```

### 列出分区

```python
partitions = collection.partitions
for partition in partitions:
    print(partition.name)
```

### 加载分区

```python
collection.load(partition_names=["partition_a"])
```

### 释放分区

```python
collection.release(partition_names=["partition_a"])
```

### 删除分区

```python
collection.drop_partition("partition_a")
```

## 4.4 性能优化

### 启用 Partition Key 隔离

```python
# 创建 Collection，启用 Partition Key 隔离
collection = Collection(
    name="my_collection",
    schema=schema,
    properties={
        "partitionkey.isolation": True,  # 启用 Partition Key 隔离
        "partition.num": 100  # 配置 100 个物理分区
    }
)
```

**效果**：
- 避免 99% 无效扫描
- 延迟从秒级降至毫秒级

## 4.5 热冷数据管理

```python
from datetime import datetime, timedelta

collection = Collection("time_series_data")

# 创建按月份的分区
current_month = datetime.now().strftime("%Y_%m")
last_month = (datetime.now().replace(day=1) - timedelta(days=1)).strftime("%Y_%m")

collection.create_partition(current_month)
collection.create_partition(last_month)

# 只加载最近两个月的数据（热数据）
collection.load(partition_names=[current_month, last_month])

# 搜索热数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10,
    partition_names=[current_month, last_month]
)
```
