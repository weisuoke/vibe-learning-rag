# 一句话总结

**Partition 是 Collection 的子集，通过 Partition Key 自动分区或手动分区实现多租户数据隔离，避免 99% 无效扫描，延迟从秒级降至毫秒级，支持百万级租户和热冷数据管理。**

---

## 核心要点

### 1. Partition 的本质

**Partition = Collection 的子集**

- 形式化定义：`Partition P ⊆ Collection C`
- 所有 Partition 的并集 = Collection
- 用于数据隔离和性能优化

### 2. 两种分区方式

**手动分区（Partition-level）**：
- 物理隔离，最多 1,024 个分区
- 支持热冷数据管理
- 适用于中等数量租户

**Partition Key（Partition Key-level，推荐）**：
- 逻辑隔离，支持百万级租户
- 自动管理，无需手动创建
- 适用于大规模多租户

### 3. 性能优化关键数据

- **99% 无效扫描避免**：只扫描相关分区
- **延迟优化**：从秒级降至毫秒级
- **推荐分区数**：100-200 个物理分区（Partition Key）
- **推荐分区大小**：20-100K 行（手动分区）

### 4. 多租户策略选择

| 租户数量 | 推荐策略 |
|----------|----------|
| < 10 | Database-level |
| < 100 | Collection-level |
| < 1,024 | Partition-level |
| > 1,024 | Partition Key-level |

### 5. 核心操作

```python
# Partition Key 自动分区（推荐）
schema = CollectionSchema(
    fields=[
        FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
        FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
        FieldSchema(name="tenant_id", dtype=DataType.VARCHAR, max_length=64,
                   is_partition_key=True)  # 自动分区
    ]
)

# 手动分区
collection.create_partition("partition_a")
collection.load(partition_names=["partition_a"])
collection.search(data=[query_vector], partition_names=["partition_a"])
```
