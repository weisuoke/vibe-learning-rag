# 30字核心

**RAG（检索增强生成）通过检索外部知识库为 LLM 提供上下文，解决知识过时和幻觉问题。**

---

## 拆解理解

| 关键词 | 含义 |
|-------|------|
| **检索（Retrieval）** | 从知识库中找到与问题相关的内容 |
| **增强（Augmented）** | 用检索到的内容增强 LLM 的输入 |
| **生成（Generation）** | LLM 基于增强后的上下文生成答案 |
| **外部知识库** | 企业文档、产品手册、FAQ 等私有数据 |

---

## 为什么这句话重要？

理解这句话，你就理解了：

1. **RAG 的本质是"开卷考试"** → LLM 不需要记住所有知识，只需要会"查资料"
2. **RAG 解决的核心问题** → 让 LLM 能回答它训练时没见过的问题
3. **RAG 的价值** → 知识可更新、答案可追溯、成本比微调低

---

## RAG 的一句话公式

```
RAG = 检索相关知识 + 注入到 Prompt + LLM 生成答案
```

用前端的话说：

```
RAG = fetch(知识库) + 拼接到请求体 + 调用 LLM API
```
