# 30字核心

**知识点**: 开发环境搭建（从源码）

---

## 核心要点

**OpenClaw 从源码搭建开发环境的核心是：pnpm workspace 管理依赖 + tsdown 快速编译 + Node.js 22 运行时，三层架构协同工作实现高效开发流程。**

---

## 拆解说明

### 1. pnpm workspace 管理依赖

**作用**: 依赖管理层

**核心价值**:
- 使用 workspace 协议管理 monorepo
- 统一版本锁定（pnpm-lock.yaml）
- 高效的磁盘空间利用

**关键命令**:
```bash
pnpm install          # 安装所有依赖
pnpm --filter <pkg>   # 针对特定包操作
```

**数据来源**: `reference/01_pnpm_workspace_docs.md`

---

### 2. tsdown 快速编译

**作用**: 构建工具链层

**核心价值**:
- 基于 Rolldown 和 Oxc 的高性能编译器
- 比传统 tsc 快数倍
- 自动生成类型声明文件

**关键命令**:
```bash
pnpm build           # 编译 TypeScript
pnpm dev             # watch 模式
```

**数据来源**: `reference/05_typescript_build_github.md`

---

### 3. Node.js 22 运行时

**作用**: 运行时环境

**核心价值**:
- ESM 模块系统优先
- 更好的 TypeScript 支持
- 性能显著提升

**版本要求**:
```json
{
  "engines": {
    "node": ">=22.12.0"
  }
}
```

**数据来源**: `reference/02_nodejs_22_docs.md`

---

## 三层架构协同

```
用户修改代码
    ↓
pnpm workspace 管理依赖
    ↓
tsdown 快速编译
    ↓
Node.js 22 运行
    ↓
开发环境就绪
```

---

## 为什么是这三个？

### 1. 为什么选择 pnpm？

**对比表**:

| 特性 | npm | yarn | pnpm |
|------|-----|------|------|
| 磁盘空间 | 大 | 大 | 小 |
| 安装速度 | 慢 | 中 | 快 |
| workspace | 支持 | 支持 | 最佳 |
| 严格性 | 低 | 中 | 高 |

**数据来源**: `reference/07_pnpm_workspace_best_practices.md`

---

### 2. 为什么选择 tsdown？

**对比表**:

| 特性 | tsc | esbuild | tsdown |
|------|-----|---------|--------|
| 编译速度 | 慢 | 快 | 最快 |
| 类型检查 | 完整 | 无 | 完整 |
| DTS 生成 | 支持 | 不支持 | 支持 |
| 配置复杂度 | 高 | 中 | 低 |

**数据来源**: `reference/05_typescript_build_github.md`

---

### 3. 为什么选择 Node.js 22？

**关键特性**:
- ESM 模块系统成熟
- V8 引擎性能提升
- 更好的 TypeScript 集成
- LTS 版本稳定性

**数据来源**: `reference/02_nodejs_22_docs.md`

## 快速验证

### 检查环境

```bash
# 检查 Node.js 版本
node --version
# 应该输出: v22.12.0 或更高

# 检查 pnpm 版本
pnpm --version
# 应该输出: 10.23.0

# 检查 Git
git --version
# 任意版本即可
```

### 快速搭建

```bash
# 1. 克隆仓库
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# 2. 安装依赖
pnpm install

# 3. 编译构建
pnpm ui:build && pnpm build

# 4. 运行开发模式
pnpm gateway:watch
```

**预期结果**: OpenClaw 成功启动，可以进行开发

**数据来源**: `reference/06_openclaw_dev_setup_github_reddit.md`

## 常见误区

### ❌ 误区 1: 可以用 npm 代替 pnpm

**错误原因**: OpenClaw 使用 pnpm workspace 特性，npm 不完全兼容

**正确做法**: 必须使用 pnpm 10.23.0

---

### ❌ 误区 2: 可以用 tsc 代替 tsdown

**错误原因**: OpenClaw 的构建脚本依赖 tsdown

**正确做法**: 使用 pnpm build（内部调用 tsdown）

---

### ❌ 误区 3: Node.js 20 也可以

**错误原因**: OpenClaw 依赖 Node.js 22 的特性

**正确做法**: 必须使用 Node.js 22.12.0 或更高版本
