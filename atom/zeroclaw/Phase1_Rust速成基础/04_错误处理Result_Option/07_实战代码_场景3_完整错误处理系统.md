## 场景说明

构建一个完整的错误处理系统，展示 Rust 社区 2025-2026 年的最佳实践：

- **底层**：用 thiserror 定义具体错误枚举，调用者可以 match 不同变体
- **顶层**：用 anyhow 包装一切，附加上下文信息方便调试

这就是 ZeroClaw 的错误处理哲学——内部精确，边界灵活。

```typescript
// TypeScript 类比：你要构建的东西
// 底层：具体的 Error 子类（类似 thiserror）
class ConfigError extends Error { /* ... */ }
class AuthError extends Error { /* ... */ }

// 顶层：统一 catch 处理（类似 anyhow）
try { await run(); }
catch (e) {
  if (e instanceof AuthError) { /* 特殊处理 */ }
  else { console.error(e); }
}
```

### src/main.rs

```rust
//! 完整错误处理系统：thiserror 自定义错误 + anyhow 应用层组合
//!
//! 运行: cargo run

use std::io;
use thiserror::Error;
use anyhow::{Context, Result};

// ===== 第一部分：用 thiserror 定义自定义错误枚举 =====
// TypeScript 类比：多个 class XxxError extends Error {}
// Rust 用一个枚举统一表达，编译器强制 match 所有变体。

/// 应用级错误枚举——thiserror 自动生成 Display + Error + From
#[derive(Error, Debug)]
pub enum AppError {
    #[error("config error at '{path}': {reason}")]
    Config { path: String, reason: String },

    #[error("network error for '{url}'")]
    Network {
        url: String,
        #[source] // Error::source() 返回此字段
        source: io::Error,
    },

    #[error("auth failed: {0}")]
    Auth(String),

    #[error("not found: {resource} (id={id})")]
    NotFound { resource: String, id: String },

    // #[from] = #[source] + 自动 impl From<io::Error>，让 ? 自动转换
    #[error("I/O error")]
    Io(#[from] io::Error),

    #[error("JSON parse error")]
    Json(#[from] serde_json::Error),
}
// [来源: reference/context7_thiserror_01.md - derive Error, #[from], #[error], #[source]]

// ===== 第二部分：底层函数——返回具体错误类型 =====

/// 模拟加载配置文件
fn load_config(path: &str) -> std::result::Result<serde_json::Value, AppError> {
    let content = std::fs::read_to_string(path)?; // io::Error → AppError::Io（#[from]）
    let config: serde_json::Value = serde_json::from_str(&content)?; // → AppError::Json
    if config.get("version").is_none() {
        return Err(AppError::Config {
            path: path.to_string(),
            reason: "missing 'version' field".to_string(),
        });
    }
    Ok(config)
}

/// 模拟网络请求
fn fetch_data(url: &str) -> std::result::Result<String, AppError> {
    if url.is_empty() {
        return Err(AppError::Config {
            path: "url".to_string(),
            reason: "URL cannot be empty".to_string(),
        });
    }
    if url.contains("timeout") {
        return Err(AppError::Network {
            url: url.to_string(),
            source: io::Error::new(io::ErrorKind::TimedOut, "connection timed out"),
        });
    }
    if url.contains("private") {
        return Err(AppError::Auth("invalid API key".to_string()));
    }
    Ok(format!("{{\"data\": \"response from {url}\"}}"))
}

/// 模拟查找资源
fn find_resource(resource: &str, id: &str) -> std::result::Result<String, AppError> {
    if id == "999" {
        return Err(AppError::NotFound {
            resource: resource.to_string(),
            id: id.to_string(),
        });
    }
    Ok(format!("{resource}/{id}: found"))
}
// [来源: reference/source_error_handling_01.md - 错误处理模式分类]

// ===== 第三部分：match 具体错误变体 =====
// 核心价值：调用者 match 每个变体，做不同恢复策略
// TS 类比：catch(e) { if (e instanceof AuthError) ... }
// Rust 的 match 更强：编译器确保穷尽所有变体

fn demo_match_variants() {
    println!("===== 场景1：match 具体错误变体 =====\n");

    let test_urls = [
        "https://api.example.com/data",   // 成功
        "",                                // 配置错误
        "https://api.example.com/timeout", // 网络超时
        "https://api.example.com/private", // 认证失败
    ];

    for url in test_urls {
        let label = if url.is_empty() { "(empty)" } else { url };
        print!("  fetch(\"{label}\"): ");
        match fetch_data(url) {
            Ok(data) => println!("OK → {data}"),
            Err(AppError::Config { reason, .. }) => println!("CONFIG → {reason}"),
            Err(AppError::Network { url, source }) => println!("NETWORK → 重试 {url} ({source})"),
            Err(AppError::Auth(msg)) => println!("AUTH → 刷新令牌: {msg}"),
            Err(AppError::NotFound { resource, id }) => println!("404 → {resource}/{id}"),
            Err(e) => println!("OTHER → {e}"),
        }
    }
}
// [来源: reference/context7_thiserror_01.md - 调用者 match 具体变体]

// ===== 第四部分：顶层用 anyhow 包装 =====
// AppError impl Error（thiserror 生成），anyhow 对所有 Error 实现 From
// 所以 ? 自动转换：AppError → anyhow::Error

fn run_app() -> Result<()> {
    let _config = load_config("app.json")
        .context("Failed to initialize application config")?;
    let _data = fetch_data("https://api.example.com/data")
        .context("Failed to fetch initial data")?;
    let _user = find_resource("user", "42")
        .context("Failed to load current user")?;
    Ok(())
}
// [来源: reference/context7_anyhow_01.md - Context trait]

// ===== 第五部分：downcast_ref 从 anyhow::Error 提取具体类型 =====
// TS 类比：catch(e: unknown) { if (e instanceof AuthError) ... }
// Rust：downcast_ref 返回 Option，不会 panic

fn demo_downcast() {
    println!("\n===== 场景2：downcast_ref 提取具体类型 =====\n");

    // AppError → anyhow::Error
    let anyhow_err: anyhow::Error = AppError::Auth("token expired".to_string()).into();

    // downcast_ref 尝试提取
    if let Some(AppError::Auth(msg)) = anyhow_err.downcast_ref::<AppError>() {
        println!("  downcast 成功: Auth({msg}) → 执行刷新令牌");
    }

    // 提取错误类型 → None
    println!("  downcast 为 io::Error: {:?}", anyhow_err.downcast_ref::<io::Error>());

    // 带 context 的错误链遍历
    println!("\n  --- 错误链遍历 ---");
    let inner = AppError::Network {
        url: "https://api.example.com/timeout".to_string(),
        source: io::Error::new(io::ErrorKind::TimedOut, "connection timed out"),
    };
    let chained: anyhow::Error = anyhow::Error::from(inner).context("Data sync failed");

    for (i, cause) in chained.chain().enumerate() {
        println!("  链[{i}]: {cause}");
    }

    // 从链中提取具体类型
    let found = chained.chain().find_map(|c| c.downcast_ref::<AppError>());
    if let Some(AppError::Network { url, .. }) = found {
        println!("\n  从链中提取 Network 错误，url={url} → 可针对性重试");
    }
}
// [来源: reference/context7_anyhow_01.md - 错误链遍历, downcast_ref]

// ===== 第六部分：ZeroClaw 风格综合演示 =====

fn demo_zeroclaw_style() {
    println!("\n===== 场景3：ZeroClaw 风格错误处理 =====\n");

    let ops: Vec<(&str, Result<()>)> = vec![
        ("加载配置", load_config("app.json").map(|_| ()).context("初始化配置失败")),
        ("获取数据", fetch_data("https://api.example.com/private").map(|_| ()).context("数据获取失败")),
        ("查找用户", find_resource("user", "999").map(|_| ()).context("用户查找失败")),
    ];

    for (name, result) in ops {
        print!("  [{name}] ");
        match result {
            Ok(()) => println!("成功"),
            Err(e) => {
                println!("失败:");
                for (i, cause) in e.chain().enumerate() {
                    let tag = if i == 0 { "    错误" } else { "    原因" };
                    println!("{tag}: {cause}");
                }
                // downcast 做特殊处理
                if let Some(app_err) = e.downcast_ref::<AppError>() {
                    match app_err {
                        AppError::Auth(_) => println!("    → 建议: 检查 API 密钥"),
                        AppError::NotFound { .. } => println!("    → 建议: 确认资源 ID"),
                        AppError::Io(_) => println!("    → 建议: 检查文件路径"),
                        _ => println!("    → 建议: 查看日志详情"),
                    }
                }
                println!();
            }
        }
    }
}
// [来源: reference/source_error_handling_01.md - 统一错误类型, 设计原则]

fn main() {
    demo_match_variants();
    demo_downcast();
    demo_zeroclaw_style();

    println!("===== 场景4：顶层 anyhow 错误链 =====\n");
    if let Err(e) = run_app() {
        println!("  应用启动失败:");
        println!("  Debug: {:?}\n", e);
        println!("  Display: {}", e);
    }
}
```

## 代码结构解析

### 错误类型转换链

```
io::Error          ──#[from]──→  AppError::Io      ──?──→  anyhow::Error
serde_json::Error  ──#[from]──→  AppError::Json    ──?──→  anyhow::Error
(手动构造)         ──────────→  AppError::Config   ──?──→  anyhow::Error
(手动构造)         ──────────→  AppError::Auth     ──?──→  anyhow::Error
```

**为什么能自动转换？**
1. `#[from]` 让 thiserror 生成 `impl From<io::Error> for AppError`
2. thiserror 生成 `impl std::error::Error for AppError`
3. anyhow 对所有 `std::error::Error` 实现了 `From`
4. 所以 `?` 一路自动转换：`io::Error → AppError → anyhow::Error`

[来源: reference/context7_thiserror_01.md - #[from] 自动实现 From trait]

### 职责分工

```
┌──────────────────────────────────────────────┐
│  应用层（main / run_app）                     │
│  返回: anyhow::Result<()>                     │
│  职责: .context() 附加上下文，打印错误链       │
│  ┌──────────────────────────────────────┐    │
│  │  业务层（load_config / fetch_data）   │    │
│  │  返回: Result<T, AppError>            │    │
│  │  职责: 返回具体变体，调用者可 match    │    │
│  │  ┌──────────────────────────┐        │    │
│  │  │  标准库 / 第三方库        │        │    │
│  │  │  通过 #[from] 自动转换    │        │    │
│  │  └──────────────────────────┘        │    │
│  └──────────────────────────────────────┘    │
└──────────────────────────────────────────────┘
```

## 速查卡

```
thiserror:
  #[derive(Error, Debug)]        启用 derive 宏
  #[error("msg {field}")]        格式化错误消息
  #[from]                        自动 From（让 ? 转换）
  #[source]                      标记错误源（不实现 From）

anyhow:
  .context("msg")?               附加上下文后传播
  error.chain()                  遍历错误链
  error.downcast_ref::<T>()      提取具体错误类型

组合模式:
  底层 → Result<T, AppError>     调用者可 match
  顶层 → anyhow::Result<T>      统一包装 + 上下文
```
