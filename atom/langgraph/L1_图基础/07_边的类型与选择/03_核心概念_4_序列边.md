# 核心概念4：序列边（Sequence Edge）

## 一句话定义

**序列边是 LangGraph 提供的语法糖，通过 `add_sequence()` 方法自动添加一系列顺序执行的节点和边，简化线性工作流的定义。**

---

## API 签名

```python
def add_sequence(
    self,
    nodes: Sequence[
        StateNode[NodeInputT, ContextT]
        | tuple[str, StateNode[NodeInputT, ContextT]]
    ],
) -> Self:
    """Add a sequence of nodes that will be executed in the provided order.

    Args:
        nodes: A sequence of `StateNode` (callables that accept a `state` arg)
               or `(name, StateNode)` tuples.

    Returns:
        Self: The graph builder instance for method chaining.
    """
```

**来源**: `langgraph/graph/state.py:889-934`

---

## 详细解释

### 什么是序列边？

序列边不是一种新的边类型，而是一个**便捷方法**（语法糖），用于简化线性工作流的定义。它在内部自动完成以下操作：

1. **自动添加节点**：调用 `add_node()` 为每个节点注册
2. **自动添加边**：调用 `add_edge()` 连接相邻节点
3. **自动推断名称**：如果未指定节点名称，自动从函数名推断

### 核心特征

#### 1. 语法糖本质

序列边是对重复操作的封装：

```python
# 传统方式：手动添加节点和边
builder.add_node("step1", step1_func)
builder.add_node("step2", step2_func)
builder.add_node("step3", step3_func)
builder.add_edge("step1", "step2")
builder.add_edge("step2", "step3")

# 序列边方式：一行搞定
builder.add_sequence([
    ("step1", step1_func),
    ("step2", step2_func),
    ("step3", step3_func)
])
```

#### 2. 自动节点名称推断

如果不提供节点名称，`add_sequence()` 会自动从函数名推断：

```python
def process_input(state):
    return {"value": state["value"] * 2}

def validate_result(state):
    return {"valid": state["value"] > 0}

# 自动推断节点名称为 "process_input" 和 "validate_result"
builder.add_sequence([process_input, validate_result])
```

#### 3. 顺序执行保证

序列边确保节点按照列表顺序依次执行：

```
nodes[0] → nodes[1] → nodes[2] → ... → nodes[n]
```

每个节点必须等待前一个节点完成后才能开始执行。

## 实现原理

### 源码分析

```python
def add_sequence(
    self,
    nodes: Sequence[
        StateNode[NodeInputT, ContextT]
        | tuple[str, StateNode[NodeInputT, ContextT]]
    ],
) -> Self:
    """Add a sequence of nodes that will be executed in the provided order."""

    # 1. 验证至少有一个节点
    if len(nodes) < 1:
        raise ValueError("Sequence requires at least one node.")

    previous_name: str | None = None

    # 2. 遍历所有节点
    for node in nodes:
        # 3. 解析节点名称和函数
        if isinstance(node, tuple) and len(node) == 2:
            name, node = node  # 手动指定名称
        else:
            name = _get_node_name(node)  # 自动推断名称

        # 4. 检查节点名称唯一性
        if name in self.nodes:
            raise ValueError(
                f"Node names must be unique: node with the name '{name}' already exists."
            )

        # 5. 添加节点
        self.add_node(name, node)

        # 6. 添加边（连接到前一个节点）
        if previous_name is not None:
            self.add_edge(previous_name, name)

        # 7. 更新前一个节点名称
        previous_name = name

    return self
```

**关键步骤**：
1. **验证输入**：确保至少有一个节点
2. **遍历节点**：按顺序处理每个节点
3. **解析名称**：支持手动指定或自动推断
4. **唯一性检查**：防止节点名称冲突
5. **添加节点**：调用 `add_node()` 注册节点
6. **添加边**：调用 `add_edge()` 连接相邻节点
7. **链式调用**：返回 `self` 支持方法链

## 完整代码示例

### 示例1：基础序列边使用

```python
"""
序列边基础示例
演示：使用 add_sequence() 简化线性工作流定义
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict


# ===== 1. 定义状态 =====
class State(TypedDict):
    value: int
    history: list[str]


# ===== 2. 定义节点函数 =====
def step1_double(state: State) -> dict:
    """第一步：将值翻倍"""
    new_value = state["value"] * 2
    return {
        "value": new_value,
        "history": state.get("history", []) + [f"step1: {state['value']} -> {new_value}"]
    }


def step2_add_ten(state: State) -> dict:
    """第二步：加10"""
    new_value = state["value"] + 10
    return {
        "value": new_value,
        "history": state["history"] + [f"step2: {state['value']} -> {new_value}"]
    }


def step3_square(state: State) -> dict:
    """第三步：平方"""
    new_value = state["value"] ** 2
    return {
        "value": new_value,
        "history": state["history"] + [f"step3: {state['value']} -> {new_value}"]
    }


# ===== 3. 构建图（使用序列边） =====
builder = StateGraph(State)

# 方式1：自动推断节点名称
builder.add_edge(START, "step1_double")
builder.add_sequence([
    step1_double,
    step2_add_ten,
    step3_square
])
builder.add_edge("step3_square", END)

graph = builder.compile()


# ===== 4. 运行测试 =====
print("=== 序列边基础示例 ===\n")

result = graph.invoke({"value": 5, "history": []})

print(f"初始值: 5")
print(f"最终值: {result['value']}")
print(f"\n执行历史:")
for step in result["history"]:
    print(f"  - {step}")

# 预期输出：
# 初始值: 5
# 最终值: 400
#
# 执行历史:
#   - step1: 5 -> 10
#   - step2: 10 -> 20
#   - step3: 20 -> 400
```

**运行输出**：
```
=== 序列边基础示例 ===

初始值: 5
最终值: 400

执行历史:
  - step1: 5 -> 10
  - step2: 10 -> 20
  - step3: 20 -> 400
```

### 示例2：手动指定节点名称

```python
"""
序列边进阶示例
演示：手动指定节点名称，提高可读性
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict


# ===== 1. 定义状态 =====
class ProcessState(TypedDict):
    text: str
    steps: list[str]


# ===== 2. 定义处理函数 =====
def clean_text(state: ProcessState) -> dict:
    """清理文本"""
    cleaned = state["text"].strip().lower()
    return {
        "text": cleaned,
        "steps": state.get("steps", []) + ["cleaned"]
    }


def tokenize(state: ProcessState) -> dict:
    """分词"""
    tokens = state["text"].split()
    return {
        "text": " ".join(tokens),
        "steps": state["steps"] + ["tokenized"]
    }


def remove_stopwords(state: ProcessState) -> dict:
    """移除停用词"""
    stopwords = {"the", "a", "an", "is", "are"}
    tokens = [w for w in state["text"].split() if w not in stopwords]
    return {
        "text": " ".join(tokens),
        "steps": state["steps"] + ["removed_stopwords"]
    }


# ===== 3. 构建图（手动指定名称） =====
builder = StateGraph(ProcessState)

builder.add_edge(START, "clean")
builder.add_sequence([
    ("clean", clean_text),
    ("tokenize", tokenize),
    ("remove_stopwords", remove_stopwords)
])
builder.add_edge("remove_stopwords", END)

graph = builder.compile()


# ===== 4. 运行测试 =====
print("=== 手动指定节点名称示例 ===\n")

result = graph.invoke({
    "text": "  The Quick Brown Fox is Jumping  ",
    "steps": []
})

print(f"原始文本: '  The Quick Brown Fox is Jumping  '")
print(f"处理后: '{result['text']}'")
print(f"\n处理步骤: {' -> '.join(result['steps'])}")

# 预期输出：
# 原始文本: '  The Quick Brown Fox is Jumping  '
# 处理后: 'quick brown fox jumping'
#
# 处理步骤: cleaned -> tokenized -> removed_stopwords
```

**运行输出**：
```
=== 手动指定节点名称示例 ===

原始文本: '  The Quick Brown Fox is Jumping  '
处理后: 'quick brown fox jumping'

处理步骤: cleaned -> tokenized -> removed_stopwords
```

### 示例3：序列边与普通边混用

```python
"""
序列边混合使用示例
演示：序列边与普通边、条件边的组合使用
"""

from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict
from typing import Literal


# ===== 1. 定义状态 =====
class WorkflowState(TypedDict):
    data: str
    validated: bool
    processed: bool


# ===== 2. 定义节点函数 =====
def load_data(state: WorkflowState) -> dict:
    """加载数据"""
    return {"data": "raw_data", "validated": False, "processed": False}


def validate_data(state: WorkflowState) -> dict:
    """验证数据"""
    is_valid = len(state["data"]) > 0
    return {"validated": is_valid}


def transform_data(state: WorkflowState) -> dict:
    """转换数据"""
    return {"data": state["data"].upper()}


def enrich_data(state: WorkflowState) -> dict:
    """丰富数据"""
    return {"data": f"[ENRICHED] {state['data']}"}


def save_data(state: WorkflowState) -> dict:
    """保存数据"""
    return {"processed": True}


def route_after_validation(state: WorkflowState) -> Literal["process", "error"]:
    """验证后路由"""
    return "process" if state["validated"] else "error"


def handle_error(state: WorkflowState) -> dict:
    """错误处理"""
    return {"data": "ERROR", "processed": True}


# ===== 3. 构建图（混合使用） =====
builder = StateGraph(WorkflowState)

# 起始边
builder.add_edge(START, "load")

# 加载和验证节点
builder.add_node("load", load_data)
builder.add_node("validate", validate_data)
builder.add_edge("load", "validate")

# 条件边：根据验证结果路由
builder.add_conditional_edges(
    "validate",
    route_after_validation,
    {
        "process": "transform",
        "error": "error_handler"
    }
)

# 序列边：处理流程（transform -> enrich -> save）
builder.add_sequence([
    ("transform", transform_data),
    ("enrich", enrich_data),
    ("save", save_data)
])

# 错误处理节点
builder.add_node("error_handler", handle_error)

# 结束边
builder.add_edge("save", END)
builder.add_edge("error_handler", END)

graph = builder.compile()


# ===== 4. 运行测试 =====
print("=== 序列边混合使用示例 ===\n")

# 测试成功路径
print("测试1：成功路径")
result1 = graph.invoke({"data": "", "validated": False, "processed": False})
print(f"结果: {result1}\n")

# 测试失败路径
print("测试2：失败路径（空数据）")
result2 = graph.invoke({"data": "", "validated": False, "processed": False})
print(f"结果: {result2}")

# 预期输出：
# 测试1：成功路径
# 结果: {'data': '[ENRICHED] RAW_DATA', 'validated': True, 'processed': True}
#
# 测试2：失败路径（空数据）
# 结果: {'data': 'ERROR', 'validated': False, 'processed': True}
```

## 在 LangGraph 工作流中的应用

### 应用场景1：数据处理管道

序列边非常适合构建数据处理管道，其中每个步骤依次处理数据：

```python
# 数据处理管道
builder.add_sequence([
    ("extract", extract_data),      # 提取数据
    ("transform", transform_data),  # 转换数据
    ("validate", validate_data),    # 验证数据
    ("load", load_data)             # 加载数据
])
```

**优势**：
- 清晰的数据流向
- 易于理解和维护
- 减少样板代码

### 应用场景2：文档处理流程

在 RAG 系统中，文档处理通常是线性的：

```python
# RAG 文档处理流程
builder.add_sequence([
    ("load_docs", load_documents),
    ("chunk_docs", chunk_documents),
    ("embed_docs", embed_documents),
    ("store_docs", store_in_vectordb)
])
```

### 应用场景3：多步骤验证

需要多个验证步骤时，序列边可以简化定义：

```python
# 多步骤验证流程
builder.add_sequence([
    ("check_format", check_format),
    ("check_schema", check_schema),
    ("check_business_rules", check_business_rules),
    ("finalize", finalize_validation)
])
```

## 序列边 vs 普通边

### 对比表格

| 特性 | 序列边 `add_sequence()` | 普通边 `add_edge()` |
|------|------------------------|---------------------|
| **用途** | 添加多个顺序节点 | 连接两个节点 |
| **节点添加** | 自动添加节点 | 需要手动添加节点 |
| **边添加** | 自动添加边 | 手动添加边 |
| **名称推断** | 支持自动推断 | 不涉及 |
| **代码量** | 少（一行） | 多（多行） |
| **适用场景** | 线性流程 | 任意连接 |
| **灵活性** | 低（固定顺序） | 高（任意连接） |

### 何时使用序列边？

**适合使用序列边**：
- 线性工作流（A → B → C → D）
- 数据处理管道
- 多步骤验证流程
- 顺序执行的任务链

**不适合使用序列边**：
- 需要条件分支的流程
- 需要并行执行的流程
- 需要循环的流程
- 复杂的路由逻辑

## 常见问题

### Q1: 序列边可以与 START/END 连接吗？

**答**：序列边本身不直接连接 START/END，需要额外添加边：

```python
# 正确方式
builder.add_edge(START, "first_node")
builder.add_sequence([
    ("first_node", func1),
    ("second_node", func2)
])
builder.add_edge("second_node", END)
```

### Q2: 序列边中的节点可以重复吗？

**答**：不可以。序列边会检查节点名称唯一性，重复会抛出 `ValueError`：

```python
# 错误示例
builder.add_sequence([
    ("process", process_func),
    ("process", process_func)  # ValueError: Node names must be unique
])
```

### Q3: 序列边可以为空吗？

**答**：不可以。至少需要一个节点，否则抛出 `ValueError`：

```python
# 错误示例
builder.add_sequence([])  # ValueError: Sequence requires at least one node
```

### Q4: 序列边可以嵌套使用吗？

**答**：不可以直接嵌套，但可以多次调用 `add_sequence()`：

```python
# 正确方式：多次调用
builder.add_sequence([func1, func2])
builder.add_sequence([func3, func4])
builder.add_edge("func2", "func3")  # 手动连接两个序列
```

## 性能考虑

### 编译时开销

序列边在编译时会展开为多个 `add_node()` 和 `add_edge()` 调用，因此：

- **编译时间**：与手动添加相同
- **运行时性能**：与手动添加完全相同
- **内存占用**：与手动添加完全相同

**结论**：序列边是纯粹的语法糖，没有性能损失。

## 最佳实践

### 1. 优先使用序列边简化线性流程

```python
# 推荐：使用序列边
builder.add_sequence([step1, step2, step3])

# 不推荐：手动添加（除非有特殊需求）
builder.add_node("step1", step1)
builder.add_node("step2", step2)
builder.add_node("step3", step3)
builder.add_edge("step1", "step2")
builder.add_edge("step2", "step3")
```

### 2. 手动指定有意义的节点名称

```python
# 推荐：清晰的名称
builder.add_sequence([
    ("load", load_data),
    ("validate", validate_data),
    ("process", process_data)
])

# 不推荐：依赖自动推断（函数名可能不够清晰）
builder.add_sequence([load_data, validate_data, process_data])
```

### 3. 结合条件边处理分支逻辑

```python
# 推荐：序列边 + 条件边
builder.add_sequence([step1, step2])
builder.add_conditional_edges("step2", route_func, {...})
builder.add_sequence([step3, step4])
```

### 4. 保持序列长度适中

```python
# 推荐：3-5个节点
builder.add_sequence([step1, step2, step3, step4])

# 不推荐：过长的序列（难以维护）
builder.add_sequence([step1, step2, ..., step20])  # 考虑拆分
```

## 总结

**序列边是 LangGraph 提供的便捷方法，用于简化线性工作流的定义。**

**核心特点**：
1. **语法糖**：内部调用 `add_node()` 和 `add_edge()`
2. **自动化**：自动添加节点和边，减少样板代码
3. **名称推断**：支持自动推断或手动指定节点名称
4. **顺序保证**：确保节点按列表顺序依次执行
5. **零性能损失**：编译后与手动添加完全相同

**适用场景**：
- 数据处理管道
- 文档处理流程
- 多步骤验证
- 任何线性工作流

**最佳实践**：
- 优先使用序列边简化线性流程
- 手动指定有意义的节点名称
- 结合条件边处理分支逻辑
- 保持序列长度适中（3-5个节点）

序列边是构建清晰、简洁的 LangGraph 工作流的重要工具，掌握它可以显著提高开发效率。
