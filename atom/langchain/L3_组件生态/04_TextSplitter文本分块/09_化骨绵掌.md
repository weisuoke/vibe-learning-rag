## 1. `_merge_splits` 算法详解

### 1.1 算法原理

`_merge_splits()` 是 TextSplitter 的核心算法，负责将小块合并成中等大小的块，同时处理重叠。这个算法是所有 TextSplitter 子类的基础。

**核心思想：**
- 遍历所有小块，累加长度
- 当累加长度超过 `chunk_size` 时，保存当前块
- 处理重叠：从当前块开头移除元素，直到满足 `chunk_overlap` 约束
- 继续添加新块

**为什么需要这个算法？**
- 保证每个块的大小在合理范围内
- 处理块之间的重叠，保持上下文连续性
- 避免在句子或段落中间切断

### 1.2 源码分析

```python
def _merge_splits(self, splits: Iterable[str], separator: str) -> list[str]:
    """将小块合并成中等大小的块，同时处理重叠"""

    # 计算分隔符长度
    separator_len = self._length_function(separator)

    docs = []  # 最终的文档块列表
    current_doc = []  # 当前正在构建的块
    total = 0  # 当前块的总长度

    for d in splits:
        # 计算当前小块的长度
        len_ = self._length_function(d)

        # 检查是否超过 chunk_size
        if (
            total + len_ + (separator_len if len(current_doc) > 0 else 0)
            > self._chunk_size
        ):
            if len(current_doc) > 0:
                # 保存当前块
                doc = self._join_docs(current_doc, separator)
                if doc is not None:
                    docs.append(doc)

                # 处理重叠：移除开头的元素，直到满足 chunk_overlap 约束
                while (
                    total > self._chunk_overlap
                    or (
                        total + len_ + (separator_len if len(current_doc) > 0 else 0)
                        > self._chunk_size
                        and total > 0
                    )
                ):
                    total -= self._length_function(current_doc[0]) + (
                        separator_len if len(current_doc) > 1 else 0
                    )
                    current_doc = current_doc[1:]

        # 添加当前小块
        current_doc.append(d)
        total += len_ + (separator_len if len(current_doc) > 1 else 0)

    # 保存最后一个块
    doc = self._join_docs(current_doc, separator)
    if doc is not None:
        docs.append(doc)

    return docs
```

**关键点：**
1. **长度计算**：使用 `length_function` 计算长度，支持字符计数或 token 计数
2. **分隔符处理**：只在块之间添加分隔符，不在块的开头或结尾
3. **重叠处理**：从当前块开头移除元素，确保重叠部分不超过 `chunk_overlap`
4. **边界条件**：处理最后一个块，避免遗漏

### 1.3 算法流程图

```
开始
  ↓
遍历所有小块
  ↓
计算当前小块长度
  ↓
是否超过 chunk_size？
  ├─ 是 → 保存当前块 → 处理重叠 → 继续
  └─ 否 → 添加到当前块 → 继续
  ↓
所有小块处理完毕？
  ├─ 是 → 保存最后一个块 → 结束
  └─ 否 → 继续遍历
```

## 2. `length_function` 机制

### 2.1 原理讲解

`length_function` 是 TextSplitter 的核心参数，用于计算文本长度。它允许你自定义长度计算方式，支持字符计数、token 计数、单词计数等。

**为什么需要 length_function？**
- **灵活性**：支持不同的长度计算方式（字符、token、自定义）
- **可扩展性**：无需修改基类即可支持新的计量方式
- **性能**：避免在基类中硬编码特定的计算逻辑

**常见的 length_function：**
1. **字符计数**：`len`（默认）
2. **Token 计数**：使用 tiktoken 或 HuggingFace tokenizer
3. **单词计数**：`lambda text: len(text.split())`
4. **自定义计数**：根据业务需求自定义

### 2.2 源码分析

**基类中的 length_function：**

**使用 tiktoken 的工厂方法：**

```python
@classmethod
def from_tiktoken_encoder(
    cls,
    encoding_name: str = "gpt2",
    model_name: str | None = None,
    allowed_special: Literal["all"] | AbstractSet[str] = set(),
    disallowed_special: Literal["all"] | Collection[str] = "all",
    **kwargs: Any,
) -> Self:
    """使用 tiktoken 编码器计算长度"""

    # 获取编码器
    if model_name is not None:
        enc = tiktoken.encoding_for_model(model_name)
    else:
        enc = tiktoken.get_encoding(encoding_name)

    # 定义 length_function
    def _tiktoken_encoder(text: str) -> int:
        return len(
            enc.encode(
                text,
                allowed_special=allowed_special,
                disallowed_special=disallowed_special,
            )
        )

    # 返回实例
    return cls(length_function=_tiktoken_encoder, **kwargs)
```

**使用 HuggingFace tokenizer 的工厂方法：**

```python
@classmethod
def from_huggingface_tokenizer(
    cls, tokenizer: PreTrainedTokenizerBase, **kwargs: Any
) -> TextSplitter:
    """使用 HuggingFace tokenizer 计算长度"""

    def _huggingface_tokenizer_length(text: str) -> int:
        return len(tokenizer.tokenize(text))

    return cls(length_function=_huggingface_tokenizer_length, **kwargs)
```

## 3. 源码分析

### 3.1 TextSplitter 基类架构

**继承关系：**

```python
class TextSplitter(BaseDocumentTransformer, ABC):
    """Interface for splitting text into chunks."""
```

- 继承自 `BaseDocumentTransformer`（文档转换器基类）
- 使用 ABC（抽象基类）定义接口规范

**核心方法：**

1. **抽象方法：split_text()**
   - 所有子类必须实现此方法
   - 输入：文本字符串
   - 输出：文本块列表

2. **create_documents()**
   - 将文本列表转换为 Document 对象列表
   - 支持为每个文本关联元数据
   - 自动调用 `split_text()` 进行分块

3. **split_documents()**
   - 分割 Document 对象列表
   - 提取文本和元数据
   - 调用 `create_documents()` 处理

4. **_merge_splits()**
   - 核心算法：将小块合并成中等大小的块
   - 处理重叠

### 3.2 RecursiveCharacterTextSplitter 递归算法

**核心思想：**
- 递归尝试多个分隔符
- 优先使用更大的分隔符（如段落、句子、单词）
- 只有当前分隔符无法满足 chunk_size 时，才使用下一个分隔符

**算法流程：**

```python
def _split_text(self, text: str, separators: list[str]) -> list[str]:
    final_chunks = []
    separator = separators[-1]  # 默认使用最后一个分隔符
    new_separators = []

    # 1. 找到第一个存在于文本中的分隔符
    for i, s_ in enumerate(separators):
        separator_ = s_ if self._is_separator_regex else re.escape(s_)
        if not s_:
            separator = s_
            break
        if re.search(separator_, text):
            separator = s_
            new_separators = separators[i + 1:]  # 剩余的分隔符
            break

    # 2. 使用找到的分隔符分割文本
    separator_ = separator if self._is_separator_regex else re.escape(separator)
    splits = _split_text_with_regex(
        text, separator_, keep_separator=self._keep_separator
    )

    # 3. 合并和递归处理
    good_splits = []
    separator_ = "" if self._keep_separator else separator

    for s in splits:
        if self._length_function(s) < self._chunk_size:
            good_splits.append(s)  # 小于 chunk_size，保留
        else:
            # 大于 chunk_size，需要进一步处理
            if good_splits:
                merged_text = self._merge_splits(good_splits, separator_)
                final_chunks.extend(merged_text)
                good_splits = []

            if not new_separators:
                final_chunks.append(s)  # 没有更多分隔符，直接添加
            else:
                # 递归使用下一个分隔符
                other_info = self._split_text(s, new_separators)
                final_chunks.extend(other_info)

    # 4. 最后合并所有保留的小块
    if good_splits:
        merged_text = self._merge_splits(good_splits, separator_)
        final_chunks.extend(merged_text)

    return final_chunks
```

**关键点：**
1. **分隔符优先级**：段落 → 行 → 单词 → 字符
2. **递归深度**：最多为分隔符数量
3. **性能优化**：只有当前分隔符无法满足时，才递归

## 4. 性能优化技巧

### 4.1 内存优化

**问题：** 大文件分块时，内存占用过高

**解决方案：**

1. **使用生成器模式**

```python
def split_text_generator(text: str, splitter: TextSplitter):
    """使用生成器逐个返回块，避免一次性加载所有块"""
    for chunk in splitter.split_text(text):
        yield chunk

# 使用
for chunk in split_text_generator(large_text, splitter):
    process_chunk(chunk)  # 逐个处理，不占用大量内存
```

2. **流式处理**

```python
def process_large_file(file_path: str, splitter: TextSplitter):
    """流式处理大文件"""
    with open(file_path, 'r', encoding='utf-8') as f:
        buffer = ""
        for line in f:
            buffer += line
            # 当 buffer 达到一定大小时，进行分块
            if len(buffer) > 10000:
                chunks = splitter.split_text(buffer)
                for chunk in chunks[:-1]:  # 保留最后一个块
                    yield chunk
                buffer = chunks[-1] if chunks else ""

        # 处理剩余内容
        if buffer:
            chunks = splitter.split_text(buffer)
            for chunk in chunks:
                yield chunk
```

### 4.2 速度优化

**问题：** 分块速度慢，影响性能

**解决方案：**

1. **缓存 tokenizer 实例**

```python
# 不推荐：每次都创建新的 tokenizer
for text in texts:
    splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
        model_name="gpt-4"
    )
    chunks = splitter.split_text(text)

# 推荐：复用 tokenizer 实例
splitter = RecursiveCharacterTextSplitter.from_tiktoken_encoder(
    model_name="gpt-4"
)
for text in texts:
    chunks = splitter.split_text(text)
```

2. **批处理**

```python
def batch_split(texts: list[str], splitter: TextSplitter, batch_size: int = 100):
    """批量处理文本"""
    all_chunks = []
    for i in range(0, len(texts), batch_size):
        batch = texts[i:i + batch_size]
        for text in batch:
            chunks = splitter.split_text(text)
            all_chunks.extend(chunks)
    return all_chunks
```

3. **并行处理**

```python
from concurrent.futures import ThreadPoolExecutor

def parallel_split(texts: list[str], splitter: TextSplitter, max_workers: int = 4):
    """并行处理文本"""
    with ThreadPoolExecutor(max_workers=max_workers) as executor:
        results = executor.map(splitter.split_text, texts)
    return [chunk for chunks in results for chunk in chunks]
```

### 5.1 自定义分块器

**场景：** 需要实现特殊的分块逻辑

**示例 1：按句子分块（保留完整句子）**

```python
from langchain_text_splitters import TextSplitter
import re

class SentenceTextSplitter(TextSplitter):
    """按句子分块，保留完整句子"""

    def split_text(self, text: str) -> list[str]:
        # 使用正则表达式分割句子
        sentences = re.split(r'(?<=[.!?])\s+', text)

        # 使用 _merge_splits 合并句子
        return self._merge_splits(sentences, " ")

# 使用
splitter = SentenceTextSplitter(chunk_size=500, chunk_overlap=50)

```

**示例 2：按段落分块（保留段落结构）**

```python
class ParagraphTextSplitter(TextSplitter):
    """按段落分块，保留段落结构"""

    def split_text(self, text: str) -> list[str]:
```

```python
class PostProcessTextSplitter(RecursiveCharacterTextSplitter):
    """添加后处理逻辑的分块器"""

    def split_text(self, text: str) -> list[str]:

        # 后处理：去除空白、过滤短块
        processed_chunks = []
```

**示例 2：添加元数据提取**

```python
class MetadataTextSplitter(RecursiveCharacterTextSplitter):
    """添加元数据提取的分块器"""

    def create_documents(
        self, texts: list[str], metadatas: list[dict] | None = None

        # 调用父类方法
        docs = super().create_documents(texts, metadatas)

        # 添加自定义元数据
        for doc in docs:
            doc.metadata["word_count"] = len(doc.page_content.split())
            doc.metadata["char_count"] = len(doc.page_content)

        return docs
```
