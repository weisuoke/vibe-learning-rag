# async/await 与 Tokio - 一句话总结

> **知识点**: async/await 与 Tokio
> **层级**: Phase1_Rust速成基础
> **维度**: 一句话总结
> **目标受众**: 有 TypeScript/前端经验但零 Rust 经验的开发者

---

## 一句话总结

**Rust 的 `async fn` 将异步操作编译为惰性状态机，由 Tokio 多线程运行时通过 poll 驱动执行，配合 `spawn` / `select!` / `mpsc` / `#[async_trait]` 构建并发、可取消、可插拔的异步架构，是 ZeroClaw 1939 个异步函数和四大核心 Trait 的底层引擎。**

---

## 扩展解读

这句话包含三个层次：

### 第一层：定义——async fn 是什么

`async fn` 不是"异步执行的函数"，而是"返回惰性 Future 的函数"。调用它不执行任何代码，只创建一个状态机值。这与 JavaScript 的 Promise（调用即执行）截然不同，是理解 Rust 异步的第一道门槛。

### 第二层：核心机制——怎么工作

- **Tokio 运行时**：多线程工作窃取调度器，驱动 Future 到完成
- **spawn**：生成后台独立任务（需 Send + 'static）
- **select!**：多 Future 竞赛，第一个完成的赢，其余取消
- **mpsc**：有界通道实现任务间消息传递，防内存泄漏
- **#[async_trait]**：让 `dyn Trait` 支持 async fn，启用动态分发

### 第三层：价值——在 ZeroClaw 中做什么

ZeroClaw 是 IO 密集型 AI Agent 框架（LLM 调用、工具执行、消息收发全是 IO）。async/await 让它在等待一个 LLM 响应时，可以同时接收新消息、执行工具、监控组件健康——一个运行时驱动整个异步世界。

## 关键记忆点

### 6 个核心概念一句话

| 概念 | 一句话 |
|------|--------|
| async fn + Future | async fn 返回惰性状态机，`.await` 才驱动执行 |
| Tokio 运行时 | 多线程工作窃取调度器，`#[tokio::main]` 创建入口 |
| tokio::spawn | 后台生成独立任务，需 `Send + 'static` |
| tokio::select! | 多 Future 竞赛，第一个完成即返回，其余自动取消 |
| mpsc 通道 | 多生产者单消费者，有界通道提供背压防止内存泄漏 |
| #[async_trait] | 将 async fn 转为 `Pin<Box<dyn Future>>`，让 dyn Trait 支持异步 |

### ZeroClaw 速记

- **1939** 个 async fn —— 几乎所有函数都是异步的
- **124** 个 #[async_trait] —— 四大核心 Trait 全部异步
- **14** 个 tokio::spawn —— 组件监控、Channel 监听
- **7** 个 tokio::select! —— Agent 循环取消、优雅停止
- **31** 个 mpsc 通道 —— Channel → Agent 消息传递
- **48** 个 tokio::time —— 定时器、超时、退避重试

### Rust vs JavaScript 异步对照速记

| 维度 | JavaScript | Rust |
|------|-----------|------|
| 执行时机 | 调用即执行（eager） | 惰性，`.await` 才执行（lazy） |
| 运行时 | V8 内置事件循环 | 需显式选择（Tokio） |
| 线程模型 | 单线程 | 多线程工作窃取 |
| 取消 | AbortController | drop Future 即取消 |
| 并发 | `Promise.all()` | `tokio::join!` |
| 竞赛 | `Promise.race()` | `tokio::select!` |
| 跨线程约束 | 不存在 | `Send + 'static` |
| 开销 | 微任务队列 + 堆分配 | 零开销状态机 |

### 一句话决策表

| 你想做什么？ | 用什么？ |
|-------------|---------|
| 等一个异步操作完成 | `.await` |
| 同时等多个操作全部完成 | `tokio::join!` |
| 同时等多个操作，只要第一个 | `tokio::select!` |
| 生成后台独立任务 | `tokio::spawn` |
| 任务间传递消息 | `mpsc::channel` |
| 给操作加超时 | `tokio::time::timeout` |
| 在 dyn Trait 中用 async fn | `#[async_trait]` |
| 调用阻塞的同步代码 | `spawn_blocking` |
