## 实际应用场景总结

### 场景1：构建新的 LangGraph 工作流

**应用最佳实践**：
```python
from typing_extensions import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy

# 1. 状态设计：TypedDict + Reducer
class State(TypedDict):
    messages: Annotated[list, add]  # 使用 reducer 聚合
    user_id: str                     # 不可变字段
    result: str

# 2. 节点设计：单一职责 + 纯函数
def search_node(state: State) -> dict:
    """只负责搜索"""
    results = search(state["messages"][-1])
    return {"messages": [f"搜索结果: {results}"]}

def validate_node(state: State) -> dict:
    """只负责验证"""
    is_valid = validate(state["messages"][-1])
    return {"result": "valid" if is_valid else "invalid"}

# 3. 错误处理：RetryPolicy
retry_policy = RetryPolicy(
    max_attempts=3,
    initial_interval=1.0,
    retry_on=ValueError
)

# 4. 图结构：Builder 模式 + 模块化
builder = StateGraph(State)
builder.add_node("search", search_node, retry_policy=retry_policy)
builder.add_node("validate", validate_node)
builder.add_edge(START, "search")
builder.add_edge("search", "validate")
builder.add_edge("validate", END)

graph = builder.compile()
```

**遵循的最佳实践**：
- 使用 TypedDict 明确状态类型
- 使用 Annotated[list, add] 聚合消息
- 节点函数单一职责、纯函数
- 为可能失败的节点配置 RetryPolicy
- 使用 Builder 模式构建图

### 场景2：重构现有的复杂工作流

**问题代码**：
```python
# ❌ 反模式：节点做太多事情
def complex_node(state: State):
    # 搜索 + 验证 + 格式化 + 日志
    results = search(state["query"])
    is_valid = validate(results)
    formatted = format(results)
    log(formatted)
    return {"results": formatted, "is_valid": is_valid}
```

**应用最佳实践重构**：
```python
# ✅ 最佳实践：拆分为多个单一职责节点
def search_node(state: State):
    """只负责搜索"""
    results = search(state["query"])
    return {"search_results": results}

def validate_node(state: State):
    """只负责验证"""
    is_valid = validate(state["search_results"])
    return {"is_valid": is_valid}

def format_node(state: State):
    """只负责格式化"""
    formatted = format(state["search_results"])
    return {"results": formatted}

# 使用子图组织相关节点
def create_search_subgraph():
    builder = StateGraph(State)
    builder.add_node("search", search_node)
    builder.add_node("validate", validate_node)
    builder.add_node("format", format_node)
    builder.add_edge(START, "search")
    builder.add_edge("search", "validate")
    builder.add_edge("validate", "format")
    builder.add_edge("format", END)
    return builder.compile()
```

**改进效果**：
- 可读性提升：每个节点职责清晰
- 可测试性提升：可以单独测试每个节点
- 可维护性提升：修改一个功能不影响其他功能
- 可复用性提升：节点可以在其他工作流中复用

### 场景3：生产环境部署

**应用最佳实践**：
```python
from langgraph.checkpoint.postgres import PostgresSaver
from langgraph.types import RetryPolicy
import logging

# 1. 配置持久化
checkpointer = PostgresSaver(
    connection_string="postgresql://user:pass@localhost/dbname"
)

# 2. 配置日志和监控
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

def monitored_node(state: State):
    """带监控的节点"""
    logger.info(f"Node started: user_id={state['user_id']}")
    try:
        result = process(state)
        logger.info(f"Node completed: result={result}")
        return result
    except Exception as e:
        logger.error(f"Node failed: error={e}")
        raise

# 3. 配置重试策略
retry_policy = RetryPolicy(
    max_attempts=5,
    initial_interval=1.0,
    backoff_factor=2.0,
    jitter=True,  # 生产环境启用
    retry_on=lambda e: isinstance(e, (ConnectionError, TimeoutError))
)

# 4. 构建生产级图
builder = StateGraph(State)
builder.add_node("process", monitored_node, retry_policy=retry_policy)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile(checkpointer=checkpointer)

# 5. 使用 thread_id 实现会话隔离
config = {
    "configurable": {
        "thread_id": f"user_{user_id}_{session_id}"
    }
}

result = graph.invoke(input, config=config)
```

**生产环境价值**：
- 持久化：使用 PostgresSaver 确保状态可恢复
- 监控：添加日志记录关键事件
- 稳定性：配置重试策略处理临时性错误
- 隔离性：使用 thread_id 实现多用户隔离
