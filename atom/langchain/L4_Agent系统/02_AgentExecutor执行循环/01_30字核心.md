# 【30字核心】AgentExecutor 执行循环

> **AgentExecutor 是一个 while 循环，反复执行 Think→Act→Observe 直到任务完成或触发限制。**

---

## 核心公式

```
while 未完成 and 未超限:
    决策 = LLM.plan(输入, 历史步骤)     # Think
    if 决策 == 完成:
        return 最终答案
    结果 = Tool.run(决策.工具, 决策.参数)  # Act
    历史步骤.append(决策, 结果)           # Observe
```

这就是 AgentExecutor 的全部。一个 while 循环，三个步骤，两个退出条件。

---

## 展开理解

### 为什么是 while 循环？

Agent 和普通 Chain 的本质区别在于：**Chain 是线性管道，Agent 是循环决策**。

一个 Chain 的执行路径在编写时就确定了——先做 A，再做 B，最后做 C。但 Agent 不同，它在运行时才决定下一步做什么。LLM 可能决定先搜索，再计算，再搜索一次，最后总结。这种"运行时动态决策"的能力，只能通过循环实现。

AgentExecutor 的 `_call` 方法（源码位于 `agent.py:1570`）就是这个循环的实现。它维护一个 `intermediate_steps` 列表，记录每一轮的 Action 和 Observation，然后把这个列表传给 LLM，让 LLM 基于完整的历史做出下一步决策。

### 循环的三个角色

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   LLM (大脑)          Tools (手脚)               │
│   ┌─────────┐        ┌─────────┐                │
│   │ 分析输入 │        │ 搜索    │                │
│   │ 选择工具 │◄──────►│ 计算    │                │
│   │ 判断完成 │        │ 查询    │                │
│   └─────────┘        └─────────┘                │
│        ▲                                         │
│        │                                         │
│        ▼                                         │
│   intermediate_steps (记忆)                       │
│   ┌─────────────────────────────────┐            │
│   │ Step1: search("天气") → "晴天"  │            │
│   │ Step2: calc("25+3") → "28"     │            │
│   │ ...                             │            │
│   └─────────────────────────────────┘            │
│                                                  │
└──────────────────────────────────────────────────┘
```

- **LLM**：大脑，负责 Think（决定做什么）和判断是否完成
- **Tools**：手脚，负责 Act（执行具体操作）
- **intermediate_steps**：记忆，负责 Observe（记录历史供 LLM 参考）

### 两个退出条件

循环不会无限运行。AgentExecutor 有两道安全阀：

1. **正常退出**: LLM 返回 `AgentFinish`，表示"我已经有了足够的信息，可以给出最终答案"
2. **强制退出**: 达到 `max_iterations`（默认 15 次）或 `max_execution_time` 限制

```python
# 源码简化版 (agent.py:1235)
def _should_continue(self, iterations, time_elapsed):
    if self.max_iterations is not None and iterations >= self.max_iterations:
        return False
    return self.max_execution_time is None or time_elapsed < self.max_execution_time
```

这两道安全阀至关重要——没有它们，一个"犹豫不决"的 LLM 可能会无限循环，烧光你的 API 额度。

## 核心模式可视化

```
输入: "北京明天天气怎么样？"

迭代 1:
  Think  → AgentAction(tool="weather_search", tool_input="北京明天天气")
  Act    → "北京明天：晴，25°C，东风3级"
  Observe → intermediate_steps = [(action1, "北京明天：晴，25°C，东风3级")]

迭代 2:
  Think  → AgentFinish(return_values={"output": "北京明天天气晴朗，气温25°C，东风3级。"})
  ✅ 循环结束，返回最终答案
```

大多数简单任务只需要 1-3 次迭代。复杂任务（如多步研究、数据分析）可能需要 5-10 次。超过 10 次通常意味着 Prompt 设计有问题或任务拆分不合理。

## 与 AI Agent 开发的关系

这个 while 循环模式是**所有 Agent 框架的共同基因**：

| 你在用的框架 | 底层都是这个循环 |
|-------------|-----------------|
| LangChain AgentExecutor | `_call` 中的 while 循环 |
| LangChain create_agent (v1) | LangGraph 状态机中的循环边 |
| OpenAI Assistants API | 服务端的 Run 循环 |
| Anthropic Tool Use | 消息循环中的 tool_use → tool_result |
| AutoGPT / BabyAGI | 主任务循环 |

**理解 AgentExecutor 的执行循环 = 理解所有 Agent 系统的运行原理。**

在 RAG 场景中，这个循环让 Agent 能够：
- 多次检索不同角度的文档
- 根据第一次检索结果调整查询策略
- 自主判断信息是否充分
- 在信息不足时主动补充检索

这就是从"被动检索"到"主动研究"的质变。
