# 核心概念2：组合模式全景

> LangChain 提供了一套完整的组合原语，让你像搭积木一样把独立组件拼成完整管道。本文基于 `langchain_core.runnables` 模块源码分析，覆盖 RunnableSequence、RunnableParallel、RunnableBranch、RunnablePassthrough、RunnableAssign、RunnablePick、ConfigurableFields 六大核心类。

---

## 一句话定义

**组合模式是 LangChain 提供的将独立组件连接成完整管道的标准方式，包括串行、并行、条件、数据传递四大类。**

把它想成乐高积木的连接方式——积木块是组件，连接方式就是组合模式。你不需要自己写胶水代码，LangChain 已经帮你定义好了所有"连接件"。

---

## 模式1：串行组合（RunnableSequence）

### 原理

串行组合是 LangChain 中**最重要、最常用**的组合操作符。

- 通过 `|` 操作符连接，前一个的输出是后一个的输入
- 自动支持 sync / async / batch / streaming 四种执行模式
- batch 默认使用线程池并行优化，不需要你手动管理

**数据流向：**

```
输入 → 组件A → 中间结果 → 组件B → 中间结果 → 组件C → 最终输出
```

### 源码洞察

从 `base.py` line 2817 的 `RunnableSequence` 类：

```python
class RunnableSequence(RunnableSerializable):
    """由 | 操作符创建的 Runnable 序列"""
    first: Runnable     # 第一个组件
    middle: list        # 中间组件列表
    last: Runnable      # 最后一个组件
```

**三个关键设计：**

1. **内部结构**：`first + middle[] + last`，不是简单的列表，而是有明确的头尾
2. **自动展平**：嵌套的 RunnableSequence 会自动展平（通过 `steps_flat` 属性），`(A | B) | C` 等价于 `A | B | C`
3. **流式传播**：如果所有组件实现了 `transform` 方法，整个序列可以端到端流式传输

### 代码示例

```python
from langchain_core.runnables import RunnableLambda
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

# ===== 基础串行：prompt → model → parser =====
chain = (
    ChatPromptTemplate.from_template("翻译成英文：{text}")
    | ChatOpenAI(model="gpt-4o-mini")
    | StrOutputParser()
)
result = chain.invoke({"text": "你好世界"})
print(result)  # "Hello, World"

# ===== 多步串行：预处理 → 核心逻辑 → 后处理 =====
def preprocess(text: str) -> str:
    """清洗输入文本"""
    return text.strip().lower()

def postprocess(text: str) -> dict:
    """结构化输出"""
    return {"result": text, "length": len(text)}

prompt = ChatPromptTemplate.from_template("总结以下内容：{text}")
model = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

chain = (
    RunnableLambda(preprocess)
    | prompt
    | model
    | parser
    | RunnableLambda(postprocess)
)
```

### 注意事项

| 要点 | 说明 |
|------|------|
| RunnableLambda 阻断流式 | 默认不支持 `transform`，放在链中会阻断流式传播 |
| 类型匹配 | 前一个组件的输出类型必须匹配后一个的输入类型 |
| 嵌套自动展平 | `(A \| B) \| (C \| D)` 自动变成 `A \| B \| C \| D` |
| batch 优化 | batch 调用时，每个步骤内部使用线程池并行处理多个输入 |

> **实战建议**：如果需要流式 + 自定义逻辑，考虑子类化 `Runnable` 并实现 `transform` 方法，而不是用 `RunnableLambda`。

## 模式2：并行组合（RunnableParallel）

### 原理

并行组合让你**同时执行多个 Runnable**，它们接收相同的输入，各自独立运行，最终结果合并成一个字典。

**数据流向：**

```
           ┌→ 组件A → 结果A ─┐
输入 ──────┼→ 组件B → 结果B ─┼──→ {"a": 结果A, "b": 结果B, "c": 结果C}
           └→ 组件C → 结果C ─┘
```

### 源码洞察

从 `base.py` line 3565 的 `RunnableParallel` 类：

```python
class RunnableParallel(RunnableSerializable):
    """并发执行多个 Runnable"""
    steps__: Mapping[str, Runnable]  # 内部存储：名称 → Runnable 的映射
```

**关键行为**：在 RunnableSequence 中使用 dict 字面量时，会**自动转换**为 RunnableParallel。这就是为什么你可以直接写 `{} | prompt` 而不需要显式包装。

### 三种构建方式

```python
from langchain_core.runnables import RunnableParallel, RunnablePassthrough

# ===== 方式1：dict 字面量（最常用，RAG 标准写法）=====
chain = (
    {
        "question": RunnablePassthrough(),
        "context": retriever | format_docs,
    }
    | prompt
    | model
    | parser
)

# ===== 方式2：显式 RunnableParallel =====
parallel = RunnableParallel(
    {
        "joke": joke_chain,
        "poem": poem_chain,
    }
)
result = parallel.invoke({"topic": "猫"})
# 输出: {"joke": "为什么猫不玩扑克...", "poem": "窗台上的小猫..."}

# ===== 方式3：关键字参数 =====
parallel = RunnableParallel(joke=joke_chain, poem=poem_chain)
# 效果与方式2 完全相同
```

> **三种方式完全等价**，方式1 最简洁，是社区约定俗成的写法。

### 典型用途

1. **RAG 中同时传递 question 和 context**——最经典的用法
2. **同时调用多个 LLM** 获取不同角度的回答，然后合并
3. **同时执行多个独立处理步骤**，比如同时做摘要和关键词提取

## 模式3：条件路由（RunnableBranch）

### 原理

条件路由让你**根据输入内容选择不同的执行路径**。它接受一组 `(条件函数, 执行链)` 的元组，加上一个默认分支。

**数据流向：**

```
           条件1为True? ──→ 链A
输入 ──→  条件2为True? ──→ 链B
           都不满足?    ──→ 默认链
```

### 源码洞察

从 `branch.py` line 42 的 `RunnableBranch` 类：

```python
class RunnableBranch(RunnableSerializable):
    """基于条件选择执行分支"""
    branches: Sequence[Tuple[Runnable, Runnable]]  # (条件, 执行体) 列表
    default: Runnable                               # 默认分支
```

**关键设计：**
- 至少需要 2 个分支（包括默认分支）
- condition 和 runnable 都会被 `coerce_to_runnable` 自动转换（所以你可以传 lambda）
- 第一个为 True 的条件被选中，后续条件不再检查
- 支持 `stream` / `astream`，选中的分支可以流式输出

### 代码示例

```python
from langchain_core.runnables import RunnableBranch

# ===== 根据意图类型路由到不同处理链 =====
branch = RunnableBranch(
    # (条件函数, 对应的处理链)
    (lambda x: x["type"] == "question", qa_chain),
    (lambda x: x["type"] == "summary", summary_chain),
    (lambda x: x["type"] == "translate", translate_chain),
    default_chain,  # 最后一个参数是默认分支，不需要条件
)

# 使用
result = branch.invoke({"type": "question", "text": "什么是RAG？"})
# → 走 qa_chain 分支

result = branch.invoke({"type": "unknown", "text": "hello"})
# → 走 default_chain 分支
```

### 典型用途

1. **意图分类后路由**——先用 LLM 判断用户意图，再分发到对应处理链
2. **根据输入长度选择模型**——短文本用小模型，长文本用大模型
3. **根据语言选择 prompt**——中文用中文 prompt，英文用英文 prompt
4. **根据数据类型选择解析器**——JSON 用 JSON parser，文本用 text parser

## 模式4：数据传递（RunnablePassthrough + RunnableAssign + RunnablePick）

这一组模式解决的是**数据在管道中如何流转**的问题。

### 4.1 RunnablePassthrough：透传输入

**作用**：把输入原封不动地传递下去，可选执行一个副作用函数。

```python
from langchain_core.runnables import RunnablePassthrough

# 最简单的用法：原样传递
passthrough = RunnablePassthrough()
result = passthrough.invoke("hello")
# 输出: "hello"

# 带副作用函数（比如日志记录）
def log_input(x):
    print(f"[LOG] 收到输入: {x}")
    return x  # 必须返回原始输入

passthrough = RunnablePassthrough(func=log_input)
```

**在 RAG 中的经典用法**：在并行组合中保留原始 question：

```python
chain = (
    {
        "question": RunnablePassthrough(),       # 透传原始问题
        "context": retriever | format_docs,      # 同时检索上下文
    }
    | prompt
    | model
    | parser
)
# 输入: "什么是RAG？"
# 并行后: {"question": "什么是RAG？", "context": "RAG是..."}
```

### 4.2 RunnablePassthrough.assign()：保留原始 + 添加新字段

**作用**：保留输入字典的所有字段，同时计算并添加新字段。这是 `RunnableAssign` 的快捷方式。

```python
from langchain_core.runnables import RunnablePassthrough
from datetime import datetime

# assign 保留原始输入，同时添加新字段
chain = RunnablePassthrough.assign(
    context=lambda x: retriever.invoke(x["question"]),
    timestamp=lambda x: datetime.now().isoformat(),
)

# 输入: {"question": "什么是RAG？"}
# 输出: {"question": "什么是RAG？", "context": [...], "timestamp": "2026-..."}
#        ↑ 原始字段保留           ↑ 新增字段        ↑ 新增字段
```

> **关键区别**：`RunnablePassthrough()` 透传单个值，`RunnablePassthrough.assign()` 在字典上添加字段。

### 4.3 RunnablePick：从字典中选择字段

**作用**：从字典输入中只选取指定的字段，丢弃其他字段。

```python
from langchain_core.runnables import RunnablePick

# 选择单个字段
pick_question = RunnablePick("question")
result = pick_question.invoke({"question": "什么是RAG？", "context": "...", "meta": "..."})
# 输出: "什么是RAG？"

# 选择多个字段
pick_fields = RunnablePick(["question", "context"])
result = pick_fields.invoke({"question": "什么是RAG？", "context": "...", "meta": "..."})
# 输出: {"question": "什么是RAG？", "context": "..."}
```

### 数据传递模式对比

| 模式 | 输入 | 输出 | 用途 |
|------|------|------|------|
| `RunnablePassthrough()` | 任意值 | 原样返回 | 在并行中保留原始输入 |
| `.assign(key=fn)` | 字典 | 原字典 + 新字段 | 给数据添加计算字段 |
| `RunnablePick("key")` | 字典 | 指定字段的值 | 从字典中提取需要的字段 |
| `RunnablePick(["a","b"])` | 字典 | 子字典 | 从字典中提取多个字段 |

## 模式5：动态配置（ConfigurableFields）

### 原理

动态配置让你在**运行时**修改组件的参数，而不需要重新构建链。这对于需要根据用户请求切换模型、调整参数的生产场景非常有用。

两种配置方式：
- `configurable_fields()`：暴露组件的特定字段，运行时可修改
- `configurable_alternatives()`：提供完全不同的替代组件，运行时可切换

### 源码洞察

从 `configurable.py` line 49 的 `DynamicRunnable` 类：

```python
class DynamicRunnable(RunnableSerializable):
    """运行时动态配置的 Runnable"""
    default: RunnableSerializable       # 默认组件
    config: RunnableConfig              # 配置信息
```

### 代码示例

```python
from langchain_core.runnables import ConfigurableField
from langchain_openai import ChatOpenAI

# ===== configurable_fields：运行时修改参数 =====
model = ChatOpenAI(model="gpt-4o-mini").configurable_fields(
    model_name=ConfigurableField(
        id="model_name",
        name="Model Name",
        description="选择使用的模型",
    )
)

chain = prompt | model | parser

# 默认使用 gpt-4o-mini
result = chain.invoke({"question": "简单问题"})

# 运行时切换到 gpt-4o（通过 config 参数）
result = chain.invoke(
    {"question": "复杂问题"},
    config={"configurable": {"model_name": "gpt-4o"}}
)
```

`configurable_alternatives` 则更进一步，可以在运行时切换到完全不同的组件（比如从 OpenAI 切换到 Anthropic）。

### 典型用途

1. **A/B 测试**——同一个链，运行时切换不同模型对比效果
2. **多租户系统**——不同客户使用不同的模型配置
3. **参数调优**——运行时调整 temperature、max_tokens 等参数

## 组合模式速查表

| 模式 | 操作符 / 方法 | 输入 → 输出 | 典型场景 |
|------|--------------|-------------|----------|
| 串行 | `\|` | A → B → C | 标准处理管道 |
| 并行 | `{}` / `RunnableParallel` | A → {B, C} → D | 同时处理多个任务 |
| 条件 | `RunnableBranch` | A → (B or C) | 意图路由 |
| 透传 | `RunnablePassthrough()` | A → A | 保留原始输入 |
| 赋值 | `.assign()` | A → A + B | 添加新字段 |
| 选取 | `RunnablePick` | {A, B, C} → A | 提取特定字段 |
| 配置 | `.configurable_fields()` | A(config) → B | 运行时切换参数 |
| 替代 | `.configurable_alternatives()` | A or B | 运行时切换组件 |

## 组合模式的嵌套

这些模式可以**自由嵌套**，构建任意复杂的管道。下面是一个综合示例，展示串行 + 并行 + 条件 + 数据传递的组合：

```python
from langchain_core.runnables import (
    RunnablePassthrough,
    RunnableBranch,
    RunnableLambda,
    RunnableParallel,
)

# ===== 综合示例：意图分类 → 条件路由 → 不同处理管道 =====

# 第一步：用 assign 添加意图分类结果
# 第二步：用 RunnableBranch 根据意图路由
# 第三步：不同分支内部使用并行组合

chain = (
    # 1. 保留原始输入，同时添加意图分类
    RunnablePassthrough.assign(
        intent=classify_chain,  # 用 LLM 分类意图
    )
    # 2. 根据意图路由到不同处理链
    | RunnableBranch(
        # QA 分支：并行获取 question + context，然后生成回答
        (
            lambda x: x["intent"] == "qa",
            {
                "question": lambda x: x["text"],
                "context": (lambda x: x["text"]) | retriever | format_docs,
            }
            | qa_prompt
            | model
            | parser,
        ),
        # 闲聊分支：直接对话
        (
            lambda x: x["intent"] == "chat",
            chat_prompt | model | parser,
        ),
        # 默认分支
        default_prompt | model | parser,
    )
)

# 使用
result = chain.invoke({"text": "什么是向量数据库？"})
# → intent="qa" → 走 QA 分支 → 检索 + 生成
```

**嵌套层次图：**

```
RunnableSequence (串行)
├── RunnableAssign (赋值: 添加 intent)
└── RunnableBranch (条件路由)
    ├── 分支1: RunnableSequence (串行)
    │   ├── RunnableParallel (并行: question + context)
    │   ├── qa_prompt
    │   ├── model
    │   └── parser
    ├── 分支2: RunnableSequence (串行)
    │   ├── chat_prompt
    │   ├── model
    │   └── parser
    └── 默认: RunnableSequence (串行)
        ├── default_prompt
        ├── model
        └── parser
```

## 模式选择决策树

不确定该用哪个模式？按这个流程走：

```
需要组合组件？
├── 数据需要依次处理？ ──→ 串行（|）
├── 多个任务可以同时做？ ──→ 并行（{}）
├── 需要根据条件走不同路？ ──→ 条件（RunnableBranch）
├── 需要保留原始数据？
│   ├── 同时添加新字段？ ──→ assign()
│   ├── 只要原样传递？ ──→ RunnablePassthrough()
│   └── 只要部分字段？ ──→ RunnablePick()
└── 需要运行时切换？ ──→ configurable_fields / configurable_alternatives
```

## 在 RAG 中的应用

| RAG 环节 | 常用组合模式 | 说明 |
|----------|-------------|------|
| 输入处理 | assign / Passthrough | 保留 question，添加 context |
| 检索 | 串行 | query → retriever → format |
| 多路检索 | 并行 | 同时查向量库 + 关键词库 |
| 意图路由 | 条件 | QA / 闲聊 / 摘要分流 |
| 生成 | 串行 | prompt → model → parser |
| 模型切换 | 配置 | 运行时切换不同 LLM |

---

## 小结

六种组合模式覆盖了管道构建的所有场景：

- **串行**（`|`）是骨架，定义数据流向
- **并行**（`{}`）是加速器，同时处理多个任务
- **条件**（`RunnableBranch`）是路由器，根据输入分流
- **数据传递**（`Passthrough` / `assign` / `Pick`）是管道工，控制数据怎么流
- **动态配置**（`configurable_fields`）是开关，运行时调整行为

掌握这五类模式，你就能组合出任意复杂度的 LangChain 管道。
