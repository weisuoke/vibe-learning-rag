# 30字核心

## 核心定义

**StateGraph 是 Builder 模式的图构建器，节点通过共享状态通信，是 LangGraph 构建状态化工作流的核心基础。**

---

## 核心要点拆解

### 1. StateGraph 是什么？

**Builder 模式的图构建器**

- StateGraph 本身不可直接执行，必须通过 `.compile()` 编译成可执行图
- 使用链式调用逐步构建图结构：`add_node()` → `add_edge()` → `compile()`
- 编译后生成 `CompiledStateGraph`（Pregel 实例），支持 `invoke()`、`stream()` 等执行方法

**来源**：源码分析 `state.py:112-127`，Context7 官方文档

---

### 2. 节点如何工作？

**通过共享状态通信**

- 每个节点函数接受完整的 `State`（TypedDict）作为输入
- 节点返回**部分状态更新**（dict），框架自动合并到共享状态
- 支持 9 种节点签名：基础节点、带 config、带 writer、带 store、带 runtime 等
- 节点可以是函数、Runnable、甚至编译后的子图

**来源**：源码分析 `_node.py:16-93`，Context7 官方文档

---

### 3. 为什么是核心基础？

**状态化工作流的基石**

- **检查点机制**：每个节点完成后自动保存状态，支持断点续传
- **条件路由**：通过 `add_conditional_edges` 实现动态决策和循环
- **多代理协作**：共享状态管理，支持 Supervisor 模式
- **生产级特性**：重试特定节点、失败重放、状态持久化

**来源**：Twitter 最佳实践，Reddit 社区案例

---

## 最小示例

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

class State(TypedDict):
    message: str

def node_a(state: State):
    return {"message": state["message"] + " -> A"}

# 构建图
builder = StateGraph(State)
builder.add_node("node_a", node_a)
builder.add_edge(START, "node_a")
builder.add_edge("node_a", END)

# 编译并执行
graph = builder.compile()
result = graph.invoke({"message": "Start"})
# {'message': 'Start -> A'}
```

**来源**：Context7 官方文档示例
