# Token与Context Window - 30字核心

**Token 是 LLM 处理文本的最小单位，Context Window 是一次能处理的 Token 上限，这是 RAG 设计的核心约束。**

---

## 关键词解析

| 关键词 | 含义 |
|--------|------|
| Token | LLM 的"文字积木"，比字符大，比单词小 |
| Context Window | 上下文窗口，LLM 的"工作台大小" |
| RAG 核心约束 | 决定能注入多少检索内容 |

---

## 为什么这是核心？

```
检索到 10 篇文档 → 但 Context Window 只能放 3 篇 → 必须选择/压缩
                              ↑
                    这就是 Token 限制的影响
```

不理解 Token 限制，RAG 系统就会频繁报错或答案不完整。
