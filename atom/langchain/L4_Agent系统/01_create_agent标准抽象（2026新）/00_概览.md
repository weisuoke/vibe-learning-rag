# create_agent标准抽象（2026新） - 学习概览

> LangChain 1.0 的核心创新：用一个工厂函数 + Middleware 组合模式，取代过去所有 Agent 创建方式

---

## 知识点定位

- **所属层级**: L4_Agent系统 - 第 1 个知识点
- **重要程度**: 极高（LangChain 1.0 后构建 Agent 的唯一推荐方式）
- **发布时间**: 2025 年 10 月（LangChain 1.0 正式发布）
- **核心变化**: 统一取代 AgentExecutor、create_react_agent、create_openai_functions_agent 等旧版 API

### 为什么这个知识点重要？

LangChain 近三年的 Agent 抽象存在一个共同问题：**开发者在非简单场景下缺乏对上下文工程的控制力**。随着 Agent 复杂度增加，旧版 API 的参数爆炸、参数间依赖关系混乱等问题日益严重。

`create_agent()` + Middleware 系统是 LangChain 团队对这一问题的终极回答。

## 核心内容概述

### 1. create_agent() 工厂函数

统一的 Agent 创建入口，一个函数搞定所有场景：

```python
from langchain.agents import create_agent

agent = create_agent(
    model="openai:gpt-4o",          # 字符串或 BaseChatModel
    tools=[web_search, calculator],   # 工具列表
    system_prompt="你是一个助手",      # 系统提示词
    middleware=[logging, retry],       # 中间件链
    response_format=MyOutput,         # 结构化输出
)
```

**关键设计决策：**
- `model` 支持字符串标识符（如 `"openai:gpt-4o"`），内部通过 `init_chat_model` 转换
- 返回值是 `CompiledStateGraph`（基于 LangGraph），天然支持流式、持久化、中断恢复
- 工具支持 `BaseTool`、普通函数、字典三种格式

### 2. AgentMiddleware 中间件系统

**LangChain 1.0 最大的创新**。提供 6 种钩子，让你在 Agent 循环的任意阶段插入自定义逻辑：

| 钩子 | 触发时机 | 典型用途 |
|------|----------|----------|
| `before_agent` | Agent 循环开始前 | 初始化、权限检查 |
| `after_agent` | Agent 循环结束后 | 清理、日志记录 |
| `before_model` | 模型调用前 | 状态更新、消息摘要 |
| `after_model` | 模型调用后 | 结果验证、跳转控制 |
| `wrap_model_call` | 包装模型调用 | 重试、降级、缓存 |
| `wrap_tool_call` | 包装工具调用 | 权限控制、结果转换 |

支持**装饰器**和**类**两种定义方式，19 个内置中间件覆盖常见场景。

### 3. AgentState 状态管理

基于 TypedDict 的轻量状态设计：

```python
class AgentState(TypedDict, Generic[ResponseT]):
    messages: list[AnyMessage]              # 消息历史（核心）
    jump_to: JumpTo | None                  # 跳转控制（临时）
    structured_response: ResponseT          # 结构化响应（输出）
```

### 4. ModelRequest / ModelResponse 数据流

不可变请求模式，通过 `override()` 创建新实例：

```python
# 在 wrap_model_call 中修改请求
def wrap_model_call(self, request, handler):
    new_request = request.override(
        tools=filtered_tools,
        model_settings={"temperature": 0}
    )
    return handler(new_request)
```

### 5. 结构化输出 ResponseFormat

三种策略适配不同场景：

| 策略 | 原理 | 适用场景 |
|------|------|----------|
| `ToolStrategy` | 将输出格式包装为工具 | 通用，兼容性最好 |
| `ProviderStrategy` | 使用模型原生结构化输出 | OpenAI/Anthropic，性能最优 |
| `AutoStrategy` | 自动选择最佳策略 | 默认推荐 |

### 6. 与旧版对比

| 维度 | 旧版 (AgentExecutor) | 新版 (create_agent) |
|------|----------------------|---------------------|
| 创建方式 | 多个 create_xxx_agent 函数 | 统一 create_agent() |
| 自定义 | 参数爆炸，难以组合 | Middleware 组合模式 |
| 运行时 | 自定义循环 | LangGraph 状态图 |
| 状态管理 | AgentAction/AgentFinish | AgentState TypedDict |
| 流式支持 | 有限 | 原生完整支持 |
| 持久化 | 需额外实现 | Checkpointer 内置 |

## 架构全景图

```
┌─────────────────────────────────────────────────────┐
│                   create_agent()                     │
│  model + tools + system_prompt + middleware + ...     │
└──────────────────────┬──────────────────────────────┘
                       │ 构建
                       ▼
┌─────────────────────────────────────────────────────┐
│              CompiledStateGraph (LangGraph)           │
│                                                       │
│  ┌─────────┐    ┌──────────┐    ┌─────────────┐     │
│  │  START   │───▶│  model   │───▶│  tools      │     │
│  └─────────┘    │  node    │◀───│  node       │     │
│                  └────┬─────┘    └─────────────┘     │
│                       │                               │
│                       ▼                               │
│                  ┌─────────┐                          │
│                  │   END   │                          │
│                  └─────────┘                          │
│                                                       │
│  ┌─────────────────────────────────────────────┐     │
│  │           Middleware 链                       │     │
│  │  before_agent → before_model → wrap_model    │     │
│  │  → after_model → wrap_tool → after_agent     │     │
│  └─────────────────────────────────────────────┘     │
│                                                       │
│  ┌─────────────────────────────────────────────┐     │
│  │           AgentState                         │     │
│  │  messages | jump_to | structured_response    │     │
│  └─────────────────────────────────────────────┘     │
└─────────────────────────────────────────────────────┘
```

**核心循环**: model 生成 → 判断是否调用工具 → 调用工具 → 回到 model → 直到不再需要工具 → 输出结果

**Middleware 介入点**: 在循环的每个关键节点，Middleware 都可以拦截、修改、重试或跳转。

```python
from langchain.agents import create_agent

agent = create_agent(
    model="openai:gpt-4o-mini",
    tools=[],
    system_prompt="你是一个友好的助手",
)
result = agent.invoke({"messages": [{"role": "user", "content": "你好"}]})
print(result["messages"][-1].content)
```

## 关键术语速查

| 术语 | 含义 |
|------|------|
| `create_agent()` | LangChain 1.0 统一的 Agent 工厂函数 |
| `AgentMiddleware` | 中间件基类，定义 6 种钩子方法 |
| `AgentState` | Agent 状态容器（TypedDict），核心字段为 messages |
| `ModelRequest` | 模型调用请求对象，支持 override() 不可变修改 |
| `ModelResponse` | 模型调用响应对象，包含消息和结构化输出 |
| `CompiledStateGraph` | create_agent 的返回类型，基于 LangGraph |
| `ResponseFormat` | 结构化输出配置，支持 Tool/Provider/Auto 三种策略 |
| `context_schema` | 运行时上下文类型定义，用于传递请求级别的参数 |
| `Checkpointer` | 状态持久化组件，支持中断恢复 |
| `jump_to` | 状态字段，用于在 Middleware 中控制流程跳转 |

## 19 个内置 Middleware 速览

| 类别 | Middleware | 用途 |
|------|-----------|------|
| 模型控制 | `ModelRetryMiddleware` | 模型调用失败自动重试 |
| 模型控制 | `ModelFallbackMiddleware` | 主模型失败时降级到备用模型 |
| 模型控制 | `ModelCallLimitMiddleware` | 限制模型最大调用次数 |
| 工具控制 | `ToolRetryMiddleware` | 工具调用失败自动重试 |
| 工具控制 | `LLMToolSelectorMiddleware` | 动态选择相关工具 |
| 人机协作 | `HumanInTheLoopMiddleware` | 人工审核关键操作 |
| 安全合规 | `PIIMiddleware` | 个人信息脱敏 |
| 上下文管理 | `SummarizationMiddleware` | 长对话自动摘要 |
| 上下文管理 | `ContextEditingMiddleware` | 动态编辑上下文 |
