## 1. Load/Release 管理

### 1.1 Load 的定义

**Load 是将 Collection 的数据和索引加载到内存的操作，是执行 Search 和 Query 的前提。**

### 1.2 Load 的核心概念

```python
from pymilvus import Collection, connections

connections.connect("default", host="localhost", port="19530")

# 创建 Collection
collection = Collection("my_collection")

# Load Collection（加载到内存）
collection.load()

# 执行 Search（需要先 Load）
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10
)

# Release Collection（释放内存）
collection.release()
```

**Load 流程**：
```
Load 请求
   ↓
Query Coordinator（查询协调器）
   ↓
Query Node（查询节点）
   ↓
加载数据到内存
   ↓
加载索引到内存
   ↓
Load 完成
```

### 1.3 Load 的高级用法

#### 部分加载（Partition）

```python
# 只加载特定分区
collection.load(partition_names=["partition_2025", "partition_2026"])

# 执行 Search（只在已加载的分区中检索）
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10,
    partition_names=["partition_2025"]
)
```

#### 副本数量配置

```python
# 配置副本数量（提高查询并发能力）
collection.load(replica_number=2)

# 查看副本信息
replicas = collection.get_replicas()
for replica in replicas:
    print(f"Replica ID: {replica.id}, Nodes: {replica.node_ids}")
```

### 1.4 Release 的最佳实践

```python
# 释放不再使用的 Collection
collection.release()

# 释放特定分区
collection.release(partition_names=["partition_2024"])

# 验证 Release 状态
print(f"Load state: {collection.load_state}")  # NotLoad
```

**Release 的时机**：
- Collection 不再使用时
- 内存资源紧张时
- 需要重新加载数据时

## 2. 数据一致性级别

### 2.1 一致性级别的定义

**一致性级别决定了读操作能看到多新的数据，是性能和准确性的权衡。**

### 2.2 三种一致性级别

#### Strong Consistency（强一致性）

```python
# Strong Consistency：保证读取最新数据
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10,
    consistency_level="Strong"  # 强一致性
)
```

**特点**：
- 保证读取最新数据
- 包括 Growing Segment 中的数据
- 性能较低
- 适合实时推荐、金融系统

#### Bounded Consistency（有界一致性）

```python
# Bounded Consistency：允许一定延迟
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10,
    consistency_level="Bounded"  # 有界一致性
)
```

**特点**：
- 允许一定延迟（通常几秒）
- 平衡性能和准确性
- 适合一般 RAG 应用
- **推荐的默认选择**

#### Eventually Consistency（最终一致性）

```python
# Eventually Consistency：性能最高
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10,
    consistency_level="Eventually"  # 最终一致性
)
```

**特点**：
- 性能最高
- 可能读到旧数据
- 适合离线分析、批量处理

### 2.3 一致性级别对比

| 一致性级别 | 数据新鲜度 | 性能 | 延迟 | 适用场景 |
|-----------|-----------|------|------|---------|
| **Strong** | 最新数据 | 较低 | 最高 | 实时推荐、金融系统 |
| **Bounded** | 允许延迟 | 中等 | 中等 | 一般 RAG 应用 |
| **Eventually** | 可能旧数据 | 最高 | 最低 | 离线分析、批量处理 |

### 2.4 一致性级别的选择策略

```python
def choose_consistency_level(use_case: str) -> str:
    """根据使用场景选择一致性级别"""
    if use_case in ["real_time_recommendation", "financial_system"]:
        return "Strong"
    elif use_case in ["rag_application", "search_engine"]:
        return "Bounded"  # 推荐
    elif use_case in ["offline_analysis", "batch_processing"]:
        return "Eventually"
    else:
        return "Bounded"  # 默认
```

## 3. 批量操作优化

### 3.1 批量插入优化

```python
import numpy as np

# ❌ 不推荐：单条插入
for i in range(10000):
    data = [{"id": i, "vector": np.random.rand(128).tolist(), "name": f"Item {i}"}]
    collection.insert(data)

# ✅ 推荐：批量插入
batch_size = 1000
for i in range(0, 10000, batch_size):
    data = [
        {"id": j, "vector": np.random.rand(128).tolist(), "name": f"Item {j}"}
        for j in range(i, min(i + batch_size, 10000))
    ]
    collection.insert(data)
```

**批量插入的优势**：
- 减少网络开销
- 提高吞吐量
- 降低系统负载

### 3.2 批量查询优化

```python
# ❌ 不推荐：单条查询
for query_vector in query_vectors:
    results = collection.search(
        data=[query_vector],
        anns_field="vector",
        param={"metric_type": "L2"},
        limit=10
    )

# ✅ 推荐：批量查询
results = collection.search(
    data=query_vectors,  # 批量查询向量
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10
)
```

### 3.3 批量删除优化

```python
# ❌ 不推荐：单条删除
for id in range(1, 10001):
    collection.delete(expr=f"id == {id}")

# ✅ 推荐：批量删除
collection.delete(expr="id >= 1 and id <= 10000")

# 或者按 ID 列表删除
batch_ids = list(range(1, 10001))
collection.delete(expr=f"id in {batch_ids}")
```

### 3.4 批量操作的性能对比

```python
import time

# 单条插入性能测试
start = time.time()
for i in range(1000):
    data = [{"id": i, "vector": np.random.rand(128).tolist()}]
```

## 4. 数据导入流程优化

### 4.1 标准数据导入流程

```python
from pymilvus import Collection, FieldSchema, CollectionSchema, DataType
import numpy as np

# 1. 创建 Collection
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=False),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
    FieldSchema(name="name", dtype=DataType.VARCHAR, max_length=100)
]
schema = CollectionSchema(fields=fields)
collection = Collection(name="my_collection", schema=schema)

# 2. 批量插入数据
batch_size = 10000
total_records = 100000

for i in range(0, total_records, batch_size):
    data = [
        {"id": j, "vector": np.random.rand(128).tolist(), "name": f"Item {j}"}
        for j in range(i, min(i + batch_size, total_records))
    ]
    collection.insert(data)
    print(f"Inserted {min(i + batch_size, total_records)} / {total_records} records")

# 3. 创建索引
index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="vector", index_params=index_params)
print("Index created")

# 4. Load Collection
collection.load()
print("Collection loaded")

# 5. 执行查询
query_vector = np.random.rand(128).tolist()
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"ef": 64}},
    limit=10
)
print(f"Search completed, found {len(results[0])} results")
```

### 4.2 Load Collection 后的数据导入

```python
# Load Collection
collection.load()

# 继续插入数据（新数据会进入 Growing Segment）
new_data = [
    {"id": 100001, "vector": np.random.rand(128).tolist(), "name": "New Item 1"},
    {"id": 100002, "vector": np.random.rand(128).tolist(), "name": "New Item 2"}
]
collection.insert(new_data)

# 使用 Strong Consistency 确保新数据立即可见
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10,
    consistency_level="Strong"  # 确保新数据可见
)
```

### 4.3 Flush 操作

```python
# Flush 操作：将 Growing Segment 持久化到磁盘
collection.flush()

# 等待 Flush 完成
print("Flushing...")
collection.flush()
print("Flush completed")
```

**Flush 的作用**：
- 将 Growing Segment 持久化到磁盘
- 触发 Segment Seal 操作
- 提高数据可靠性

## 5. Compaction 机制

### 5.1 Compaction 的定义

**Compaction 是数据压缩和空间回收的操作，用于合并小 Segment 和回收删除数据的空间。**

### 5.2 Compaction 的触发

```python
from pymilvus import Collection

collection = Collection("my_collection")

# 手动触发 Compaction
collection.compact()

# 等待 Compaction 完成
collection.wait_for_compaction_completed()

# 查看 Compaction 状态
compaction_state = collection.get_compaction_state()
print(f"Compaction state: {compaction_state}")

# 查看 Compaction 计划
compaction_plans = collection.get_compaction_plans()
for plan in compaction_plans:
    print(f"Plan: {plan}")
```

### 5.3 Compaction 的应用场景

**场景 1：删除操作后回收空间**

```python
# 删除数据
collection.delete(expr="status == 'archived'")

# 执行 Compaction 回收空间
collection.compact()
collection.wait_for_compaction_completed()
```

**场景 2：Upsert 操作后确保数据一致性**

```python
# Upsert 数据
collection.upsert(data)

# 执行 Compaction 确保数据一致性
collection.compact()
collection.wait_for_compaction_completed()
```

**场景 3：定期维护**

```python
import schedule
import time

def daily_compaction():
    """每天执行一次 Compaction"""
    collection = Collection("my_collection")
    collection.compact()
    collection.wait_for_compaction_completed()
    print("Daily compaction completed")

# 每天凌晨 2 点执行
schedule.every().day.at("02:00").do(daily_compaction)

while True:
    schedule.run_pending()
    time.sleep(60)
```

## 6. 错误处理与重试策略

### 6.1 连接错误处理

```python
from pymilvus import connections, Collection
import time

def connect_with_retry(max_retries=3, retry_delay=5):
    """带重试的连接"""
    for i in range(max_retries):
        try:
            connections.connect(
                alias="default",
                host="localhost",
                port="19530",
                timeout=10
            )
            print("Connected to Milvus")
            return True
        except Exception as e:
            print(f"Connection failed (attempt {i+1}/{max_retries}): {e}")
            if i < max_retries - 1:
                time.sleep(retry_delay)
    return False

# 使用
if connect_with_retry():
    collection = Collection("my_collection")
else:
    print("Failed to connect to Milvus")
```

### 6.2 插入错误处理

```python
def insert_with_retry(collection, data, max_retries=3):
    """带重试的插入"""
    for i in range(max_retries):
        try:
            result = collection.insert(data)
            print(f"Inserted {result.insert_count} records")
            return result
        except Exception as e:
            print(f"Insert failed (attempt {i+1}/{max_retries}): {e}")
            if i < max_retries - 1:
                time.sleep(2)
            else:
                raise

# 使用
try:
    insert_with_retry(collection, data)
except Exception as e:
    print(f"Insert failed after retries: {e}")
```

### 6.3 Search 错误处理

```python
def search_with_fallback(collection, query_vector, consistency_level="Bounded"):
    """带降级的搜索"""
    try:
        # 尝试使用指定的一致性级别
        results = collection.search(
            data=[query_vector],
            anns_field="vector",
            param={"metric_type": "L2"},
            limit=10,
            consistency_level=consistency_level
        )
        return results
    except Exception as e:
        print(f"Search failed with {consistency_level} consistency: {e}")

        # 降级到 Eventually Consistency
        if consistency_level != "Eventually":
            print("Falling back to Eventually consistency")
            return search_with_fallback(collection, query_vector, "Eventually")
        else:
            raise

# 使用
try:
    results = search_with_fallback(collection, query_vector, "Strong")
except Exception as e:
    print(f"Search failed: {e}")
```

## 7. 监控与健康检查

### 7.1 Collection 状态监控

```python
from pymilvus import Collection, utility

def check_collection_health(collection_name: str):
    """检查 Collection 健康状态"""
    try:
        # 检查 Collection 是否存在
        if not utility.has_collection(collection_name):
            print(f"Collection {collection_name} does not exist")
            return False

        collection = Collection(collection_name)

        # 检查 Load 状态
        load_state = collection.load_state
        print(f"Load state: {load_state}")

        # 检查实体数量
        num_entities = collection.num_entities
        print(f"Number of entities: {num_entities}")

        # 检查索引状态
        indexes = collection.indexes
        for index in indexes:
            print(f"Index: {index.field_name}, Type: {index.params['index_type']}")

        return True
    except Exception as e:
        print(f"Health check failed: {e}")
        return False

# 使用
check_collection_health("my_collection")
```

### 7.2 性能监控

```python
import time

def monitor_search_performance(collection, query_vectors, num_runs=10):
    """监控搜索性能"""
    latencies = []

    for i in range(num_runs):
        start = time.time()
        results = collection.search(
            data=query_vectors,
            anns_field="vector",
            param={"metric_type": "L2"},
            limit=10
        )
        latency = time.time() - start
        latencies.append(latency)
        print(f"Run {i+1}: {latency:.3f}s")

    avg_latency = sum(latencies) / len(latencies)
    print(f"Average latency: {avg_latency:.3f}s")
    print(f"Min latency: {min(latencies):.3f}s")
    print(f"Max latency: {max(latencies):.3f}s")

# 使用
query_vectors = [np.random.rand(128).tolist() for _ in range(10)]
monitor_search_performance(collection, query_vectors)
```

### 8.2 查询最佳实践

```python
def production_search(collection_name: str, query_vector, top_k=10):
    """生产环境搜索"""
    collection = Collection(collection_name)

    # 1. 检查 Collection 是否已加载
    if collection.load_state != "Loaded":
        collection.load()

    # 2. 执行搜索（使用 Bounded Consistency）
    try:
        results = collection.search(
            data=[query_vector],
            anns_field="vector",
            param={"metric_type": "L2", "params": {"ef": 64}},
            limit=top_k,
            consistency_level="Bounded",  # 推荐
            output_fields=["id", "name"]  # 只返回需要的字段
        )
        return results[0]
    except Exception as e:
        print(f"Search failed: {e}")
        # 降级到 Eventually Consistency
        results = collection.search(
            data=[query_vector],
            anns_field="vector",
            param={"metric_type": "L2", "params": {"ef": 32}},  # 降低 ef
            limit=top_k,
            consistency_level="Eventually",
            output_fields=["id", "name"]
        )
        return results[0]
```

## 9. 常见问题

### 9.1 Load 失败

**原因**：
- 内存不足
- Collection 不存在
- 索引未构建

**解决方案**：
```python
# 检查内存使用情况
import psutil
memory = psutil.virtual_memory()
print(f"Available memory: {memory.available / (1024**3):.2f} GB")

# 释放其他 Collection
other_collection.release()

# 重新 Load
collection.load()
```

### 9.2 Search 返回空结果

**原因**：
- Collection 未加载
- 一致性级别导致数据不可见
- 索引未构建

**解决方案**：
```python
# 确保 Collection 已加载
collection.load()

# 使用 Strong Consistency
results = collection.search(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2"},
    limit=10,
    consistency_level="Strong"
)
```

### 9.3 插入性能慢

**原因**：
- 单条插入
- 批量大小不合适
- 网络延迟

**解决方案**：
```python
# 使用批量插入
batch_size = 10000  # 调整批量大小
for i in range(0, len(data), batch_size):
    batch = data[i:i+batch_size]
    collection.insert(batch)
```

## 10. 总结

**数据管理最佳实践是确保 Milvus 系统高效、稳定运行的关键，包括：**

**Load/Release 管理**：
1. **Load 是 Search 的前提**：必须先 Load 才能执行 Search
2. **部分加载优化内存**：只加载需要的分区
3. **副本数量提升并发**：配置多个副本提高查询并发能力
4. **及时 Release 释放内存**：不再使用的 Collection 及时释放

**数据一致性级别**：
1. **Strong**：实时推荐、金融系统
2. **Bounded**：一般 RAG 应用（推荐）
3. **Eventually**：离线分析、批量处理

**批量操作优化**：
1. **批量插入**：性能远优于单条插入
2. **批量查询**：减少网络开销
3. **批量删除**：使用表达式批量删除

**数据导入流程**：
1. 批量插入数据
2. 创建索引
3. Load Collection
4. 执行查询

**Compaction 机制**：
1. 删除操作后回收空间
2. Upsert 操作后确保数据一致性
3. 定期维护

**错误处理与重试**：
1. 连接错误重试
2. 插入错误重试
3. Search 降级策略
