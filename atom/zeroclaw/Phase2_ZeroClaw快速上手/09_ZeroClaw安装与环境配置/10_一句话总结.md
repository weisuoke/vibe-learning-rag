## 一句话总结

**ZeroClaw 安装与环境配置是将一个 8.8MB 的 Rust 编译产物部署到目标平台、并通过"环境变量 > 命令行 > config.toml > 默认值"的分层配置驱动 AI Agent 运行的过程，5 种安装方式覆盖全场景、3 行 TOML 即可启动、Feature Flags 按需裁剪实现最小构建。**

---

## 核心要点回顾

1. **安装 ≠ 编译** — 推荐方式（一键脚本 / Homebrew / 预编译下载）直接下载编译好的二进制文件，和 `cargo install` 的编译过程无关，30 秒可完成
2. **3 行 TOML 就能跑** — `api_key` + `default_provider` + `default_model`，写入 `~/.zeroclaw/config.toml` 或通过 `zeroclaw onboard` 自动生成
3. **配置分层各司其职** — 密钥走环境变量（安全）、偏好走 config.toml（持久）、临时需求走命令行（灵活），三者协作互补而非二选一
4. **默认零 Feature = 设计选择** — `default = []` 已覆盖 90% 使用场景（CLI + Agent + 所有 Provider + SQLite），Feature Flags 是编译时菜单、按需启用
5. **优先级链是排错利器** — 遇到"改了配置不生效"时，`zeroclaw config show` 可查看每个值的来源，99% 的问题是高优先级配置覆盖了低优先级
