# 核心概念 3：数据检索方法（Search）

> 本文档详细讲解 Milvus 2.6 中的 Search 向量相似度检索操作和混合检索

---

## 概述

Search 是 Milvus 中基于向量相似度的检索操作，是 RAG 系统实现语义检索的核心功能。Milvus 2.6 还支持混合检索（向量 + BM25），进一步提升检索准确性。

---

## 1. Search 的定义

**Search 是基于向量相似度的检索操作，返回与查询向量最相似的 Top-K 个实体。**

### 核心特征

1. **向量相似度计算**：基于向量距离（L2、IP、COSINE）计算相似度
2. **Top-K 返回**：返回相似度最高的 K 个结果
3. **标量过滤**：支持在检索时添加标量字段过滤条件
4. **混合检索**：支持向量检索 + BM25 全文检索的混合模式

### 与 Query 的区别

| 特性 | Search | Query |
|------|--------|-------|
| **查询方式** | 基于向量相似度 | 基于主键/标量字段 |
| **返回结果** | 相似度排序的实体 | 精确匹配的实体 |
| **性能特点** | 向量计算，相对慢 | 索引查询，快速 |
| **应用场景** | 语义检索、相似度匹配 | 精确查找、元数据过滤 |
| **是否需要向量** | 需要 | 不需要 |

## 2. Search 的核心概念

### 2.1 基础向量检索

**通过查询向量检索最相似的实体。**

```python
from pymilvus import MilvusClient
import numpy as np

client = MilvusClient("http://localhost:19530")

# 准备查询向量
query_vector = np.random.rand(128).tolist()

# 基础向量检索
results = client.search(
    collection_name="my_collection",
    data=[query_vector],  # 查询向量列表
    limit=10,  # 返回 Top-10 结果
    output_fields=["id", "name", "price"]  # 返回的字段
)

# 处理结果
for hits in results:
    for hit in hits:
        print(f"ID: {hit['id']}, Score: {hit['score']}, Name: {hit['entity']['name']}")
```

**关键参数**：
- `data`：查询向量列表（支持批量查询）
- `limit`：返回的结果数量（Top-K）
- `output_fields`：返回的字段列表
- `search_params`：搜索参数（如 `ef`、`nprobe` 等）

### 2.2 带标量过滤的检索

**在向量检索时添加标量字段过滤条件。**

```python
# 带标量过滤的检索
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    filter="price > 100 and status == 'active'",  # 标量过滤
    output_fields=["id", "name", "price", "status"]
)

for hits in results:
    for hit in hits:
        print(f"ID: {hit['id']}, Score: {hit['score']}, Price: {hit['entity']['price']}")
```

**过滤策略**：
- **Pre-filtering**：先过滤再检索（默认）
- **Post-filtering**：先检索再过滤

### 2.3 混合检索（向量 + BM25）

**Milvus 2.6 支持向量检索和 BM25 全文检索的混合模式。**

```python
from pymilvus import AnnSearchRequest, RRFRanker

# 1. 向量检索请求
vector_search_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=10
)

# 2. BM25 全文检索请求
bm25_search_req = AnnSearchRequest(
    data=["search query text"],
    anns_field="sparse_vector",  # BM25 稀疏向量字段
    param={"metric_type": "BM25"},
    limit=10
)

# 3. 混合检索（使用 RRF 融合）
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[vector_search_req, bm25_search_req],
    ranker=RRFRanker(),  # Reciprocal Rank Fusion
    limit=10,
    output_fields=["id", "title", "content"]
)

for hits in results:
    for hit in hits:
        print(f"ID: {hit['id']}, Score: {hit['score']}, Title: {hit['entity']['title']}")
```

**混合检索优势**：
- 向量检索捕捉语义相似性
- BM25 检索捕捉关键词匹配
- RRF 融合两种检索结果，提升准确性

## 3. Search 的实现原理

### 3.1 检索流程

```
用户请求（查询向量）
   ↓
Query Coordinator（查询协调器）
   ↓
Query Node（查询节点）
   ↓
向量索引检索（HNSW/IVF_FLAT/GPU_CAGRA）
   ↓
标量过滤（如果有）
   ↓
Top-K 排序
   ↓
结果返回
```

### 3.2 Growing Segment 和 Sealed Segment

**Milvus 的数据分为两种 Segment：**

```python
# Growing Segment（增长段）
# - 新插入的数据
# - 实时可搜索（brute-force）
# - 性能较低

# Sealed Segment（封闭段）
# - 达到阈值后 Seal
# - 构建索引
# - 高性能检索
```

**数据流转**：
```
Insert → WAL → Growing Segment → Seal → Sealed Segment → 构建索引
```

**搜索可见性**：
- **Strong Consistency**：立即可见（包括 Growing Segment）
- **Bounded Consistency**：允许延迟（可能不包括最新数据）
- **Eventually Consistency**：最终一致（性能最高）

### 3.3 一致性级别对比

```python
# Strong Consistency（强一致性）
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Strong"  # 保证读取最新数据
)

# Bounded Consistency（有界一致性）
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Bounded"  # 允许一定延迟
)

# Eventually Consistency（最终一致性）
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Eventually"  # 性能最高，但可能读到旧数据
)
```

**一致性级别选择**：

| 一致性级别 | 数据新鲜度 | 性能 | 适用场景 |
|-----------|-----------|------|---------|
| **Strong** | 最新数据 | 较低 | 实时推荐、金融系统 |
| **Bounded** | 允许延迟 | 中等 | 一般 RAG 应用 |
| **Eventually** | 可能旧数据 | 最高 | 离线分析、批量处理 |

## 4. Search 的高级用法

### 4.1 批量检索

```python
# 批量查询向量
query_vectors = [
    np.random.rand(128).tolist(),
    np.random.rand(128).tolist(),
    np.random.rand(128).tolist()
]

# 批量检索
results = client.search(
    collection_name="my_collection",
    data=query_vectors,  # 多个查询向量
    limit=10,
    output_fields=["id", "name"]
)

# 处理每个查询的结果
for i, hits in enumerate(results):
    print(f"Query {i} results:")
    for hit in hits:
        print(f"  ID: {hit['id']}, Score: {hit['score']}")
```

### 4.2 搜索参数调优

```python
# 调整搜索参数
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    search_params={
        "metric_type": "L2",
        "params": {
            "nprobe": 10,  # IVF_FLAT 参数
            "ef": 64       # HNSW 参数
        }
    },
    output_fields=["id", "name"]
)
```

**常用参数**：
- **HNSW**：`ef`（搜索时的候选数量，越大越准确但越慢）
- **IVF_FLAT**：`nprobe`（搜索的聚类数量，越大越准确但越慢）
- **GPU_CAGRA**：`itopk_size`、`search_width`

### 4.3 分区检索

```python
# 在特定分区中检索
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    partition_names=["partition_2025", "partition_2026"],  # 指定分区
    output_fields=["id", "name"]
)
```

### 4.4 Range Search（范围检索）

```python
# 范围检索（返回距离在指定范围内的所有结果）
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    search_params={
        "metric_type": "L2",
        "params": {
            "radius": 10.0,  # 最大距离
            "range_filter": 0.0  # 最小距离
        }
    },
    output_fields=["id", "name"]
)
```

## 5. Search 的性能优化

### 5.1 选择合适的索引类型

```python
from pymilvus import Collection

collection = Collection("my_collection")

# 小数据集（< 100万）：FLAT
collection.create_index(
    field_name="vector",
    index_params={
        "index_type": "FLAT",
        "metric_type": "L2"
    }
)

# 中等数据集（100万 - 1000万）：HNSW
collection.create_index(
    field_name="vector",
    index_params={
        "index_type": "HNSW",
        "metric_type": "L2",
        "params": {"M": 16, "efConstruction": 200}
    }
)

# 大数据集（> 1000万）：IVF_FLAT
collection.create_index(
    field_name="vector",
    index_params={
        "index_type": "IVF_FLAT",
        "metric_type": "L2",
        "params": {"nlist": 1024}
    }
)

# GPU 加速：GPU_CAGRA
collection.create_index(
    field_name="vector",
    index_params={
        "index_type": "GPU_CAGRA",
        "metric_type": "L2",
        "params": {"intermediate_graph_degree": 64}
    }
)
```

### 5.2 减少返回字段

```python
# 只返回需要的字段
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    output_fields=["id", "title"]  # 不返回向量字段
)
```

### 5.3 使用合适的一致性级别

```python
# RAG 应用使用 Bounded Consistency
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Bounded"  # 平衡性能和准确性

```

### 5.4 批量检索优化

```python
# 批量检索比单次检索更高效
query_vectors = [vector1, vector2, vector3]  # 批量查询

    collection_name="my_collection",
```

## 6. Search 的应用场景

### 6.1 RAG 文档检索

```python
from openai import OpenAI

openai_client = OpenAI()

# 1. 用户查询
user_query = "What is the capital of France?"

# 2. 生成查询向量
response = openai_client.embeddings.create(
    model="text-embedding-3-small",
    input=user_query
)
query_vector = response.data[0].embedding

# 3. 向量检索
results = client.search(
    collection_name="documents",
    data=[query_vector],
    limit=5,
    filter="doc_type == 'knowledge_base'",
    output_fields=["id", "title", "content"]
)

# 4. 构建上下文
context = "\n\n".join([hit['entity']['content'] for hit in results[0]])

# 5. 生成回答
response = openai_client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {user_query}"}
    ]
)

print(response.choices[0].message.content)
```

## 7. Search 的常见问题

### 7.1 Search 返回空结果

**原因**：
- Collection 未加载到内存
- 一致性级别导致数据不可见
- 向量索引未构建

**解决方案**：
```python
# 确保 Collection 已加载
collection.load()

# 使用 Strong Consistency
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Strong"
)

# 确保索引已构建
collection.create_index(
    field_name="vector",
    index_params={"index_type": "HNSW", "metric_type": "L2"}
)
```

### 7.2 Search 性能慢

**原因**：
- 索引类型不合适
- 搜索参数设置不当
- 返回字段过多

**解决方案**：
```python
# 使用合适的索引
collection.create_index(
    field_name="vector",
    index_params={"index_type": "HNSW", "metric_type": "L2"}
)

# 调整搜索参数
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    search_params={"params": {"ef": 32}},  # 降低 ef 提升性能
    output_fields=["id"]  # 减少返回字段
)
```

### 7.3 Search 结果不准确

**原因**：
- 向量质量问题
- 索引参数设置不当
- 标量过滤过于严格

**解决方案**：
```python
# 提高索引质量
collection.create_index(
    field_name="vector",
    index_params={
        "index_type": "HNSW",
        "metric_type": "L2",
        "params": {"M": 32, "efConstruction": 400}  # 提高参数
    }
)

# 提高搜索精度
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    search_params={"params": {"ef": 128}},  # 提高 ef
    output_fields=["id", "name"]
)
```

## 8. 源码实现（Go 客户端）

根据 `client/milvusclient/read.go` 的源码分析：

```go
func (c *Client) Search(ctx context.Context, option SearchOption, callOptions ...grpc.CallOption) ([]ResultSet, error) {
    startTime := time.Now()
    collectionName := option.CollectionName()
    result := []ResultSet{}

    // 获取 Collection 信息
    collection, err := c.getCollection(ctx, option.CollectionName())
    if err != nil {
        return result, err
    }

    // 构建 Search 请求
    req, err := option.SearchRequest(collection)
    if err != nil {
        return result, err
    }

    // 调用 gRPC 服务
    err = c.callService(func(milvusService milvuspb.MilvusServiceClient) error {
        resp, err := milvusService.Search(ctx, req, callOptions...)
        err = merr.CheckRPCCall(resp, err)
        if err != nil {
            return err
        }

        // 解析结果
        result, err = c.handleSearchResult(collection.Schema, option.OutputFields(), resp)
        return err
    })

    // 记录操作
    c.recordOperation("Search", collectionName, startTime, err)
    return result, err
}
```

**关键点**：
- Search 返回 `[]ResultSet`（支持批量查询）
- 支持 wildcard 输出字段（`*`）
- 支持动态字段
- 支持 group-by 值

## 9. 混合检索深入

### 9.1 RRF（Reciprocal Rank Fusion）

**RRF 融合算法**：
```
RRF_score(d) = Σ (1 / (k + rank_i(d)))

其中：
- d：文档
- rank_i(d)：文档 d 在第 i 个检索结果中的排名
- k：常数（通常为 60）
```

**Python 实现**：
```python
from pymilvus import RRFRanker

# 使用 RRF 融合
ranker = RRFRanker(k=60)

results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[vector_search_req, bm25_search_req],
    ranker=ranker,
    limit=10
)
```

### 9.2 Weighted Ranker（加权融合）

```python
from pymilvus import WeightedRanker

# 使用加权融合
ranker = WeightedRanker(weights=[0.7, 0.3])  # 向量检索权重 0.7，BM25 权重 0.3

results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[vector_search_req, bm25_search_req],
    ranker=ranker,
    limit=10
)
```

## 10. 总结

**Search 是 Milvus 中基于向量相似度的检索操作，具有以下特点：**

1. **语义检索**：基于向量相似度计算，捕捉语义相似性
2. **Top-K 返回**：返回相似度最高的 K 个结果
3. **标量过滤**：支持在检索时添加标量字段过滤条件
4. **混合检索**：支持向量 + BM25 的混合模式（Milvus 2.6）
5. **一致性保证**：支持多种一致性级别

**与 Query 的互补**：
- Search 用于语义检索和相似度匹配
- Query 用于精确查找和元数据过滤
- 两者结合可以实现强大的混合检索

**2026 年生产标准**：
- 混合检索（向量 + BM25）是标准配置
- RRF 融合算法提升检索准确性
- Bounded Consistency 平衡性能和准确性
