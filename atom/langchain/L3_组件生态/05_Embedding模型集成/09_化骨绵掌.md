## 卡片1：Embeddings 抽象接口直觉理解

**一句话：** Embeddings 接口定义了文本向量化的统一契约，让你可以无缝切换不同的 Embedding 提供商。

**核心要点：**
- 使用 ABC 抽象基类定义接口
- 只有 4 个方法：embed_documents、embed_query、aembed_documents、aembed_query
- 所有 Embedding 模型（OpenAI、HuggingFace、Cohere）都遵循这个接口

**代码示例：**
```python
from langchain_core.embeddings import Embeddings

class Embeddings(ABC):
    @abstractmethod
    def embed_documents(self, texts: list[str]) -> list[list[float]]:
        """批量嵌入文档"""

    @abstractmethod
    def embed_query(self, text: str) -> list[float]:
        """嵌入单个查询"""
```

**实战场景：** 在 RAG 系统中，你可以轻松从 OpenAI 切换到本地模型，只需更换实例化的类，其他代码无需修改。

**常见误区：** 认为 embed_documents 和 embed_query 底层实现不同，实际上大多数模型两者相同，分离只是为了语义清晰和未来扩展。

[来源: reference/source_embeddings_base_01.md]

## 卡片2：embed_documents vs embed_query

**一句话：** embed_documents 用于批量索引构建，embed_query 用于单个查询检索，虽然底层可能相同但语义不同。

**核心要点：**
- **embed_documents(texts: list[str])** - 输入列表，输出二维数组，用于批量处理
- **embed_query(text: str)** - 输入单个字符串，输出一维数组，用于查询
- 某些模型对文档和查询使用不同的 prompt 或优化策略

**代码示例：**
```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 批量嵌入文档（索引构建）
docs = ["文档1", "文档2", "文档3"]
doc_vectors = embeddings.embed_documents(docs)  # [[...], [...], [...]]

# 单个查询嵌入（检索）
query = "用户查询"
query_vector = embeddings.embed_query(query)  # [...]
```

**实战场景：** 在 RAG 系统中，索引构建阶段使用 embed_documents 批量处理所有文档，用户查询时使用 embed_query 处理单个问题。

**常见误区：** 认为必须用 embed_query 处理查询，实际上可以用 embed_documents([query])[0]，但不推荐，因为语义不清晰。

[来源: reference/source_embeddings_base_01.md, reference/context7_langchain_openai_02.md]

## 卡片3：OpenAI 模型选择策略

**一句话：** text-embedding-3-small 性价比最高适合大规模应用，text-embedding-3-large 精度最高适合高价值场景。

**核心要点：**
- **text-embedding-3-small**: 1536维，$0.02/1M tokens，性价比之王
- **text-embedding-3-large**: 3072维，$0.13/1M tokens，精度最高
- **text-embedding-ada-002**: 1536维，旧模型，逐渐淘汰
- 支持自定义维度（text-embedding-3+），降维可节省成本

**代码示例：**
```python
# 场景1：成本敏感（推荐）
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512  # 降维节省 67% 存储
)

# 场景2：高精度需求
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-large"
)
```

**实战场景：** 大规模文档库（10万+文档）使用 small 模型 + 降维，企业知识库使用 large 模型保证检索质量。

**常见误区：** 认为维度越高越好，实际上降维到 512 维性能下降不到 5%，但存储成本降低 67%。

[来源: reference/source_openai_embeddings_03.md, reference/search_embedding_integration_01.md]

## 卡片4：批量处理与 chunk_size

**一句话：** 使用 chunk_size 控制批量大小，将 1000 次 API 调用减少到 10 次，效率提升 100 倍。

**核心要点：**
- **chunk_size**: 每次 API 调用包含的最大文本数量（默认 1000）
- 批量处理减少网络开销和 API 调用次数
- 自动处理超长文本（超过 8191 tokens 自动分块）

**代码示例：**
```python
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    chunk_size=100,  # 每批处理 100 个文本
    show_progress_bar=True  # 显示进度条
)

# 批量处理 1000 个文档
texts = [f"文档 {i}" for i in range(1000)]
vectors = embeddings.embed_documents(texts)
# 只调用 10 次 API（1000 / 100）
```

**实战场景：** 索引构建时批量处理大量文档，设置合理的 chunk_size 避免速率限制，同时提高吞吐量。

**常见误区：** 认为 chunk_size 越大越好，实际上过大会导致单次请求超时，推荐 50-200 之间。

[来源: reference/source_openai_embeddings_03.md, 03_核心概念_3_批量处理与优化.md]

## 卡片5：缓存机制原理

**一句话：** CacheBackedEmbeddings 使用装饰器模式包装任何 Embeddings，避免重复计算节省 90%+ 成本。

**核心要点：**
- 装饰器模式：不侵入原始实现，透明添加缓存功能
- 默认只缓存文档嵌入，查询嵌入可选
- 支持多种哈希算法（SHA-1、BLAKE2b、SHA-256）
- 批量缓存更新（mget/mset）

**代码示例：**
```python
from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings

# 创建缓存存储
store = LocalFileStore("./embedding_cache")

# 包装 Embeddings 添加缓存
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    namespace="openai-ada-002"
)

# 第一次调用：计算并缓存
vectors1 = cached_embeddings.embed_documents(["hello", "world"])

# 第二次调用：从缓存读取（快 100 倍）
vectors2 = cached_embeddings.embed_documents(["hello", "world"])
```

**实战场景：** 开发迭代时频繁重建索引，使用缓存避免重复计算相同文档，节省 API 调用和时间。

**常见误区：** 认为查询也应该缓存，实际上查询通常不重复，缓存命中率低，浪费存储空间。

[来源: reference/source_cache_backed_02.md, 03_核心概念_4_缓存机制.md]

## 卡片6：异步批量处理

**一句话：** 使用 aembed_documents 异步方法提高并发性能，在高并发 RAG 应用中吞吐量提升 5-10 倍。

**核心要点：**
- 异步方法：aembed_documents、aembed_query
- 默认实现：使用 run_in_executor 包装同步方法
- 可覆盖：提供原生异步实现以获得更好性能
- 并发控制：max_concurrency 参数限制并发数

**代码示例：**
```python
import asyncio
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

async def process_documents():
    texts = [f"文档 {i}" for i in range(100)]

    # 异步批量处理
    vectors = await embeddings.aembed_documents(texts)
    return vectors

# 运行异步任务
vectors = asyncio.run(process_documents())
```

**实战场景：** FastAPI 异步 API 服务中，使用异步方法避免阻塞事件循环，提高并发处理能力。

**常见误区：** 认为异步一定更快，实际上单个请求异步和同步速度相同，只有高并发场景才有优势。

[来源: reference/source_embeddings_base_01.md, 07_实战代码_场景1_基础使用流程.md]

## 卡片7：多提供商集成

**一句话：** 统一的 Embeddings 接口让你可以轻松切换 OpenAI、HuggingFace、Cohere 等提供商，避免供应商锁定。

**核心要点：**
- **OpenAI**: 高质量，API 调用，成本较高
- **Sentence Transformers**: 本地部署，免费，速度快
- **Cohere**: 多语言支持，API 调用
- 切换只需更换实例化的类，其他代码不变

**代码示例：**
```python
# OpenAI（云端）
from langchain_openai import OpenAIEmbeddings
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# Sentence Transformers（本地）
from langchain_community.embeddings import HuggingFaceEmbeddings
embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")

# 使用方式完全相同
vectors = embeddings.embed_documents(["hello", "world"])
```

**实战场景：** 开发环境使用本地模型节省成本，生产环境使用 OpenAI 保证质量，切换只需修改一行代码。

**常见误区：** 认为不同提供商的向量可以混用，实际上不同模型的向量空间不同，不能直接比较。

[来源: reference/context7_sentence_transformers_01.md, 07_实战代码_场景2_多提供商集成.md]

## 卡片8：自定义 Embeddings 实现

**一句话：** 继承 Embeddings 基类实现 embed_documents 和 embed_query 两个方法，即可集成任何自定义模型。

**核心要点：**
- 继承 `langchain_core.embeddings.Embeddings`
- 实现两个抽象方法：embed_documents、embed_query
- 可选：覆盖异步方法以提供原生异步支持
- 可选：添加缓存、批量处理等优化

**代码示例：**
```python
from langchain_core.embeddings import Embeddings
import numpy as np

class MyCustomEmbeddings(Embeddings):
    def __init__(self, model_path: str):
        self.model = self._load_model(model_path)

    def embed_documents(self, texts: list[str]) -> list[list[float]]:
        # 批量嵌入逻辑
        return [self._embed_single(text) for text in texts]

    def embed_query(self, text: str) -> list[float]:
        # 单个查询嵌入逻辑
        return self._embed_single(text)

    def _embed_single(self, text: str) -> list[float]:
        # 实际的嵌入逻辑
        return self.model.encode(text).tolist()

# 使用自定义 Embeddings
embeddings = MyCustomEmbeddings(model_path="./my_model")
vectors = embeddings.embed_documents(["hello", "world"])
```

**实战场景：** 集成公司内部训练的领域特定模型，或使用非主流的 Embedding 服务。

**常见误区：** 认为必须实现所有方法，实际上只需实现两个抽象方法，异步方法有默认实现。

[来源: reference/search_custom_embeddings_02.md, 07_实战代码_场景4_自定义Embedding实现.md]

## 卡片9：RAG 集成实战

**一句话：** 完整的 RAG 管道包括文档加载、分块、嵌入、存储、检索、生成六个步骤，Embeddings 在第三步发挥作用。

**核心要点：**
- **步骤1**: 加载文档（PyPDFLoader、TextLoader）
- **步骤2**: 文本分块（RecursiveCharacterTextSplitter）
- **步骤3**: 嵌入向量化（OpenAIEmbeddings）
- **步骤4**: 存储到向量数据库（Chroma、FAISS）
- **步骤5**: 检索相似文档（retriever.get_relevant_documents）
- **步骤6**: 生成答案（LLM + Prompt）

**代码示例：**
```python
from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma

# 1. 加载文档
loader = PyPDFLoader("document.pdf")
documents = loader.load()

# 2. 分块
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
splits = splitter.split_documents(documents)

# 3. 嵌入 + 4. 存储
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = Chroma.from_documents(splits, embeddings)

# 5. 检索
retriever = vectorstore.as_retriever()
docs = retriever.get_relevant_documents("用户查询")
```

**实战场景：** 构建企业知识库问答系统，用户提问时检索相关文档片段，结合 LLM 生成答案。

**常见误区：** 认为 Embeddings 是 RAG 的全部，实际上只是其中一个环节，文档分块和检索策略同样重要。

[来源: reference/search_rag_integration_04.md, 07_实战代码_场景3_RAG集成实战.md]

## 卡片10：生产环境优化

**一句话：** 综合使用缓存、批量处理、异步并发三大优化策略，将 RAG 系统性能提升 10-100 倍。

**核心要点：**
- **缓存优化**: 使用 CacheBackedEmbeddings 避免重复计算
- **批量优化**: 设置合理的 chunk_size（50-200）
- **异步优化**: 使用 aembed_documents 提高并发
- **维度优化**: 降低维度节省存储和计算成本
- **监控优化**: 记录 API 调用次数、缓存命中率、延迟

**代码示例：**
```python
from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings

# 综合优化配置
base_embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512,  # 降维节省成本
    chunk_size=100,  # 批量处理
    max_retries=3,  # 错误重试
    show_progress_bar=True  # 进度监控
)

# 添加缓存层
store = LocalFileStore("./cache")
embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    namespace="prod-v1"
)

# 异步批量处理
async def process():
    texts = load_documents()  # 加载大量文档
    vectors = await embeddings.aembed_documents(texts)
    return vectors
```

**实战场景：** 生产环境 RAG 系统，处理百万级文档，需要平衡成本、性能、质量三个维度。

**常见误区：** 过度优化，实际上应该先测量瓶颈，再针对性优化，避免过早优化。

[来源: reference/search_cache_optimization_03.md, reference/search_embedding_integration_01.md]
