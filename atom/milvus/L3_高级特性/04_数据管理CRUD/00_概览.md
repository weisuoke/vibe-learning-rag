# 04_数据管理CRUD - 概览

## 知识点简介

**数据管理CRUD（Create, Read, Update, Delete）** 是 Milvus 2.6 中管理向量数据的核心操作集合，涵盖数据插入（Insert）、查询（Query）、检索（Search）、删除（Delete）、更新（Upsert）等完整生命周期管理。

在 RAG 系统中，CRUD 操作是知识库管理的基础：
- **Insert**：将文档向量化后存入 Milvus
- **Search**：根据用户查询检索相似文档
- **Query**：精确查询特定文档元数据
- **Upsert**：更新文档内容或元数据
- **Delete**：删除过期或无效文档

## 核心要点速览

### CRUD 操作对比

| 操作 | 用途 | 性能 | 使用场景 |
|------|------|------|----------|
| **Insert** | 插入新数据 | 高 | 初始数据导入、增量数据添加 |
| **Query** | 精确查询 | 中 | 按主键或表达式查询元数据 |
| **Search** | 相似度检索 | 中-高 | 向量相似度搜索、RAG 检索 |
| **Delete** | 删除数据 | 中 | 数据清理、合规性删除 |
| **Upsert** | 插入或更新 | 低 | 数据更新、增量同步 |

### 数据写入流程

```
Insert/Upsert
    ↓
WAL (Write-Ahead Log)
    ↓
Growing Segment (实时可搜索)
    ↓
Sealed Segment (达到阈值后)
    ↓
构建索引
    ↓
加载到内存
```

### 关键概念

1. **Growing Segment**：新插入的数据首先进入 Growing Segment，使用 brute-force 搜索，实时可见
2. **Sealed Segment**：Growing Segment 达到阈值后 Seal，构建索引，提供高性能搜索
3. **Compaction**：删除操作不会立即释放空间，需要 Compaction 来回收空间
4. **一致性级别**：Strong（立即可见）、Bounded（延迟可见）、Eventually（最终一致）

### 常见误区

1. ❌ **"Upsert 性能和 Insert 一样"** - Upsert 需要先查询再操作，性能开销更大
2. ❌ **"Delete 后空间立即释放"** - 需要 Compaction 才能回收空间
3. ❌ **"Insert 后立即可搜索"** - 取决于一致性级别，可能有延迟

## 实际应用场景

### RAG 文档问答系统

```python
# 1. 初始数据导入
client.insert("knowledge_base", documents)

# 2. 用户查询检索
results = client.search("knowledge_base", query_vector, limit=5)

# 3. 文档更新
client.upsert("knowledge_base", updated_documents)

# 4. 过期文档删除
client.delete("knowledge_base", filter="timestamp < '2025-01-01'")
```

### 多租户知识库

```python
# 按租户分区管理
client.insert("knowledge_base", documents, partition_name="tenant_001")
client.search("knowledge_base", query_vector, partition_names=["tenant_001"])
```

### 增量数据同步

```python
# 使用 Upsert 实现增量同步
for batch in data_stream:
    client.upsert("knowledge_base", batch)
```
