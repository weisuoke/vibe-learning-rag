## 场景概述

OutputParser 在实际应用中经常遇到解析失败的情况，原因包括：
- LLM 输出格式不符合预期
- JSON/XML 语法错误
- 缺少必需字段
- 类型不匹配

LangChain 提供了三种错误处理机制：
1. **OutputParserException**：基础异常捕获
2. **OutputFixingParser**：使用 LLM 自动修复错误
3. **RetryOutputParser**：重新调用 LLM 生成正确输出

## 示例1：基础异常处理（OutputParserException）

**场景**：捕获并处理 OutputParser 的解析错误。

**关键技术**：
- `OutputParserException` 异常类
- try-except 错误捕获
- 降级策略

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import PydanticOutputParser
from langchain_core.exceptions import OutputParserException
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from pydantic import BaseModel, Field

load_dotenv()

# 定义 Pydantic 模型
class Person(BaseModel):
    name: str = Field(description="Person's name")
    age: int = Field(description="Person's age")
    email: str = Field(description="Person's email")

# 初始化
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
parser = PydanticOutputParser(pydantic_object=Person)

# 创建 prompt（故意不包含格式指令，增加失败概率）
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("user", "Tell me about a person named Alice.")
])

chain = prompt | model | parser

print("=== 基础异常处理 ===\n")

try:
    result = chain.invoke({})
    print(f"Success: {result}")
except OutputParserException as e:
    print(f"Parse failed: {e}")
    print(f"\nError type: {type(e).__name__}")

    # 降级策略：返回默认值
    default_person = Person(
        name="Unknown",
        age=0,
        email="unknown@example.com"
    )
    print(f"\nFallback to default: {default_person}")

"""
预期输出：
=== 基础异常处理 ===

Parse failed: Failed to parse Person from completion ...
Error type: OutputParserException

Fallback to default: name='Unknown' age=0 email='unknown@example.com'
"""
```

**关键点**：
- `OutputParserException` 包含原始输出和错误信息
- 可以实现降级策略（返回默认值、使用缓存等）
- 适合简单的错误处理场景

## 示例2：OutputFixingParser - 自动修复

**场景**：使用 LLM 自动修复格式错误的输出。

**关键技术**：
- `OutputFixingParser.from_llm()`
- 自动调用 LLM 修复错误
- 保留原始 parser 的验证逻辑

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import PydanticOutputParser, OutputFixingParser
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

load_dotenv()

# 定义模型
class Product(BaseModel):
    name: str = Field(description="Product name")
    price: float = Field(description="Product price")
    in_stock: bool = Field(description="Whether product is in stock")

# 初始化
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
base_parser = PydanticOutputParser(pydantic_object=Product)

# 创建 OutputFixingParser
fixing_parser = OutputFixingParser.from_llm(
    parser=base_parser,
    llm=model
)

print("=== OutputFixingParser 自动修复 ===\n")

# 模拟错误的 JSON 输出
bad_output = """
{
    "name": "Laptop",
    "price": "1299.99",  # 错误：应该是数字而不是字符串
    "in_stock": "yes"     # 错误：应该是布尔值而不是字符串
}
"""

print(f"Bad output:\n{bad_output}\n")

# 尝试用基础 parser 解析（会失败）
print("--- Using base parser ---")
try:
    result = base_parser.parse(bad_output)
    print(f"Success: {result}")
except Exception as e:
    print(f"Failed: {type(e).__name__}")

# 使用 OutputFixingParser 自动修复
print("\n--- Using OutputFixingParser ---")
try:
    result = fixing_parser.parse(bad_output)
    print(f"Success: {result}")
    print(f"Type of price: {type(result.price)}")
    print(f"Type of in_stock: {type(result.in_stock)}")
except Exception as e:
    print(f"Failed: {e}")

"""
预期输出：
=== OutputFixingParser 自动修复 ===

Bad output:
{
    "name": "Laptop",
    "price": "1299.99",
    "in_stock": "yes"
}

--- Using base parser ---
Failed: ValidationError

--- Using OutputFixingParser ---
Success: name='Laptop' price=1299.99 in_stock=True
Type of price: <class 'float'>
Type of in_stock: <class 'bool'>
"""
```

**工作原理**：
1. 基础 parser 尝试解析，失败并抛出异常
2. OutputFixingParser 捕获异常和原始输出
3. 构造修复 prompt，包含错误信息和原始输出
4. 调用 LLM 生成修复后的输出
5. 再次使用基础 parser 解析

**优势**：
- 自动修复常见格式错误
- 无需手动编写修复逻辑
- 保留原始 parser 的验证能力

## 示例3：RetryOutputParser - 重试机制

**场景**：当输出不符合要求时，重新调用 LLM 生成正确输出。

**关键技术**：
- `RetryOutputParser.from_llm()`
- 提供原始 prompt 和错误输出
- 自动重试直到成功或达到最大次数

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import PydanticOutputParser
from langchain.output_parsers import RetryOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from pydantic import BaseModel, Field, validator

load_dotenv()

# 定义带验证的模型
class Employee(BaseModel):
    name: str = Field(description="Employee name")
    age: int = Field(description="Employee age")
    department: str = Field(description="Department name")

    @validator('age')
    def age_must_be_positive(cls, v):
        if v <= 0 or v > 120:
            raise ValueError('Age must be between 1 and 120')
        return v

# 初始化
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
base_parser = PydanticOutputParser(pydantic_object=Employee)

# 创建 RetryOutputParser
retry_parser = RetryOutputParser.from_llm(
    parser=base_parser,
    llm=model,
    max_retries=3
)

# 创建 prompt（故意不包含年龄限制，增加失败概率）
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs JSON."),
    ("user", "Generate an employee profile for someone in the Engineering department.")
])

print("=== RetryOutputParser 重试机制 ===\n")

# 使用 RetryOutputParser
# 注意：RetryOutputParser 需要原始 prompt 和 LLM 输出
chain = prompt | model

# 获取原始输出
completion = chain.invoke({})
print(f"Original LLM output:\n{completion.content}\n")

# 使用 RetryOutputParser 解析（如果失败会自动重试）
try:
    result = retry_parser.parse_with_prompt(
        completion=completion.content,
        prompt_value=prompt.format_prompt()
    )
    print(f"Success after retry: {result}")
except Exception as e:
    print(f"Failed after max retries: {e}")

"""
预期输出：
=== RetryOutputParser 重试机制 ===

Original LLM output:
{"name": "John Doe", "age": 150, "department": "Engineering"}

Success after retry: name='John Doe' age=30 department='Engineering'
"""
```

**工作原理**：
1. 获取 LLM 的原始输出
2. 尝试用基础 parser 解析
3. 如果失败，构造重试 prompt（包含错误信息）
4. 重新调用 LLM 生成输出
5. 重复步骤 2-4，直到成功或达到最大重试次数

**与 OutputFixingParser 的区别**：
- **OutputFixingParser**：修复已有的错误输出
- **RetryOutputParser**：重新生成正确的输出

## 示例4：三层错误处理策略

**场景**：构建生产级的错误处理系统，结合三种机制。

**关键技术**：
- 第一层：Prompt 优化（预防）
- 第二层：RetryOutputParser（重试）
- 第三层：OutputFixingParser（修复）
- 第四层：异常捕获（降级）

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import PydanticOutputParser, OutputFixingParser
from langchain.output_parsers import RetryOutputParser
from langchain_core.exceptions import OutputParserException
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from pydantic import BaseModel, Field
from typing import Optional

load_dotenv()

class UserProfile(BaseModel):
    username: str = Field(description="User's username")
    email: str = Field(description="User's email address")
    age: int = Field(description="User's age", ge=18, le=100)
    bio: Optional[str] = Field(description="User's bio", default=None)

class RobustOutputParser:
    """生产级 OutputParser 包装器"""

    def __init__(self, pydantic_object, llm, max_retries=2):
        self.base_parser = PydanticOutputParser(pydantic_object=pydantic_object)
        self.retry_parser = RetryOutputParser.from_llm(
            parser=self.base_parser,
            llm=llm,
            max_retries=max_retries
        )
        self.fixing_parser = OutputFixingParser.from_llm(
            parser=self.base_parser,
            llm=llm
        )
        self.llm = llm

    def parse_with_fallback(self, text: str, prompt_value=None) -> tuple:
        """
        多层错误处理解析
        返回: (result, strategy_used)
        """
        # 第一层：直接解析
        try:
            result = self.base_parser.parse(text)
            return result, "direct"
        except OutputParserException as e:
            print(f"Direct parse failed: {type(e).__name__}")

        # 第二层：重试（如果有 prompt）
        if prompt_value:
            try:
                result = self.retry_parser.parse_with_prompt(
                    completion=text,
                    prompt_value=prompt_value
                )
                return result, "retry"
            except Exception as e:
                print(f"Retry parse failed: {type(e).__name__}")

        # 第三层：修复
        try:
            result = self.fixing_parser.parse(text)
            return result, "fixing"
        except Exception as e:
            print(f"Fixing parse failed: {type(e).__name__}")

        # 第四层：降级到默认值
        print("All strategies failed, using default")
        return None, "default"

# 使用示例
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
robust_parser = RobustOutputParser(
    pydantic_object=UserProfile,
    llm=model,
    max_retries=2
)

# 创建 prompt（包含格式指令）
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs JSON.\n{format_instructions}"),
    ("user", "Generate a user profile for a software engineer named Alice.")
])

prompt_with_format = prompt.partial(
    format_instructions=robust_parser.base_parser.get_format_instructions()
)

print("=== 三层错误处理策略 ===\n")

# 测试1：正常情况
print("--- Test 1: Normal case ---")
chain = prompt_with_format | model
completion = chain.invoke({})
result, strategy = robust_parser.parse_with_fallback(
    text=completion.content,
    prompt_value=prompt_with_format.format_prompt()
)
print(f"Result: {result}")
print(f"Strategy used: {strategy}\n")

# 测试2：错误输出
print("--- Test 2: Bad output ---")
bad_output = '{"username": "alice", "email": "invalid", "age": "thirty"}'
result, strategy = robust_parser.parse_with_fallback(
    text=bad_output,
    prompt_value=prompt_with_format.format_prompt()
)
print(f"Result: {result}")
print(f"Strategy used: {strategy}\n")

"""
预期输出：
=== 三层错误处理策略 ===

--- Test 1: Normal case ---
Result: username='alice' email='alice@example.com' age=30 bio='Software engineer...'
Strategy used: direct

--- Test 2: Bad output ---
Direct parse failed: ValidationError
Retry parse failed: ValidationError
Result: username='alice' email='alice@example.com' age=30 bio=None
Strategy used: fixing
"""
```

**设计亮点**：
- **分层防护**：从预防到修复，逐层降级
- **策略追踪**：记录使用的策略，便于监控
- **灵活配置**：可调整重试次数和降级策略
- **生产就绪**：完善的错误处理和日志

## 示例5：流式解析的错误处理

**场景**：在流式模式下处理解析错误。

**关键技术**：
- 流式输出的部分解析
- 错误累积和批量处理
- 优雅降级

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.exceptions import OutputParserException
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

load_dotenv()

model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
parser = JsonOutputParser()

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs JSON."),
    ("user", "Generate a list of 5 products with name and price.")
])

chain = prompt | model | parser

print("=== 流式解析错误处理 ===\n")

error_count = 0
success_count = 0
last_valid_chunk = None

try:
    for i, chunk in enumerate(chain.stream({})):
        try:
            # 验证 chunk 的有效性
            if isinstance(chunk, dict):
                last_valid_chunk = chunk
                success_count += 1
                print(f"Chunk {i}: Valid - {len(str(chunk))} chars")
            else:
                error_count += 1
                print(f"Chunk {i}: Invalid type - {type(chunk)}")
        except Exception as e:
            error_count += 1
            print(f"Chunk {i}: Error - {type(e).__name__}")

except OutputParserException as e:
    print(f"\nStream failed: {e}")
    print(f"Using last valid chunk: {last_valid_chunk}")

print(f"\n=== Statistics ===")
print(f"Success: {success_count}")
print(f"Errors: {error_count}")
print(f"Success rate: {success_count / (success_count + error_count) * 100:.1f}%")

"""
预期输出：
=== 流式解析错误处理 ===

Chunk 0: Valid - 2 chars
Chunk 1: Valid - 25 chars
Chunk 2: Valid - 58 chars
...

=== Statistics ===
Success: 15
Errors: 0
Success rate: 100.0%
"""
```

## 常见错误与解决方案

### 错误1：忘记提供 format_instructions

**问题**：
```python
# 错误：没有告诉 LLM 输出格式
prompt = ChatPromptTemplate.from_template("Generate a person profile")
```

**解决方案**：
```python
# 正确：包含格式指令
prompt = ChatPromptTemplate.from_template(
    "Generate a person profile\n{format_instructions}"
)
prompt = prompt.partial(format_instructions=parser.get_format_instructions())
```

### 错误2：OutputFixingParser 和 RetryOutputParser 混淆

**区别**：
- **OutputFixingParser**：修复已有输出（不需要原始 prompt）
- **RetryOutputParser**：重新生成输出（需要原始 prompt）

**使用场景**：
- 格式错误 → OutputFixingParser
- 内容错误 → RetryOutputParser

### 错误3：无限重试

**问题**：
```python
# 错误：没有限制重试次数
while True:
    try:
        result = parser.parse(text)
        break
    except:
        continue  # 可能无限循环
```

**解决方案**：
```python
# 正确：限制重试次数
max_retries = 3
for attempt in range(max_retries):
    try:
        result = parser.parse(text)
        break
    except OutputParserException:
        if attempt == max_retries - 1:
            # 使用降级策略
            result = default_value
```

## 错误处理策略对比

| 策略 | 成本 | 成功率 | 延迟 | 适用场景 |
|------|------|--------|------|---------|
| 直接解析 | 低 | 中 | 低 | 格式规范的输出 |
| OutputFixingParser | 中 | 高 | 中 | 格式错误 |
| RetryOutputParser | 高 | 高 | 高 | 内容错误 |
| 降级策略 | 低 | 100% | 低 | 兜底方案 |

1. **预防优于修复**：

   - 使用 `get_format_instructions()`
   - 提供示例输出

2. **分层错误处理**：
   - 第一层：Prompt 优化
   - 第二层：重试机制
   - 第三层：自动修复
   - 第四层：降级策略

3. **监控和日志**：
   - 记录解析失败率
   - 追踪使用的错误处理策略
   - 分析失败原因

4. **成本控制**：
   - 限制重试次数
   - 使用更便宜的模型进行修复
   - 缓存成功的解析结果

5. **用户体验**：
   - 提供有意义的错误消息
   - 实现优雅降级
   - 避免完全失败
