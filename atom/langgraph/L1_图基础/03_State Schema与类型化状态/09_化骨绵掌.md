# 化骨绵掌

> State Schema与类型化状态的10个两分钟知识卡片

## 卡片1：TypedDict 状态定义

**核心内容**：

```python
from typing import TypedDict, Annotated
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]
    user_id: str
    count: int
```

**关键点**：
- TypedDict 定义状态结构
- 类型注解提供编译时检查
- 轻量级，无运行时开销

**记忆口诀**：
> TypedDict 定结构，类型安全无开销

## 卡片2：Annotated 与 Reducer

**核心内容**：

```python
from typing import Annotated
import operator

# Annotated[类型, reducer函数]
messages: Annotated[list, operator.add]
metadata: Annotated[dict, operator.or_]
```

**关键点**：
- Annotated 附加元数据
- Reducer 定义聚合规则
- 自动应用于状态更新

**记忆口诀**：
> Annotated 加元数据，Reducer 定规则

## 卡片3：Reducer 执行时机

**核心内容**：

Reducer 在三个时机执行：
1. 节点返回后
2. 并行执行后
3. 手动更新时

**关键点**：
- 自动触发，无需手动调用
- 确保状态一致性
- 支持并发聚合

**记忆口诀**：
> 节点返回、并行执行、手动更新，三时机触发

## 卡片4：Channel 类型系统

**核心内容**：

```python
# LastValue: 存储最后一个值
user_id: str

# BinaryOperatorAggregate: 使用 reducer 聚合
messages: Annotated[list, operator.add]

# EphemeralValue: 临时值，不持久化
temp_data: EphemeralValue[dict]
```

**关键点**：
- Channel 是状态管理的底层机制
- 不同类型适应不同场景
- 自动创建，无需手动管理

**记忆口诀**：
> LastValue 存最后，Aggregate 用 reducer，Ephemeral 不持久

## 卡片5：NotRequired vs Optional

**核心内容**：

```python
from typing import Optional
from typing_extensions import NotRequired

# NotRequired: 字段可以不存在
class State1(TypedDict):
    field: NotRequired[int]

# Optional: 字段必须存在，但值可以是 None
class State2(TypedDict):
    field: Optional[int]
```

**关键点**：
- NotRequired 更符合语义
- Optional 需要处理 None
- 推荐使用 NotRequired

**记忆口诀**：
> NotRequired 可不存在，Optional 必须有但可为 None

## 卡片6：并行执行与 Superstep

**核心内容**：

```
Superstep 1: [node1, node2, node3] 并行执行
  -> Reducer 聚合状态
Superstep 2: [node4] 执行
  -> Reducer 聚合状态
```

**关键点**：
- Superstep 是执行单元
- 并行节点在同一 superstep
- Superstep 结束后状态聚合

**记忆口诀**：
> Superstep 为单元，并行后聚合

# InputState: 节点接收

## 卡片9：状态大小控制

**核心内容**：

```python
def trim_messages(existing: list, new: list) -> list:
    """只保留最近100条"""
    all_messages = existing + new
    return all_messages[-100:]

class State(TypedDict):
    messages: Annotated[list, trim_messages]
```

**关键点**：
- 使用 reducer 控制增长
- 避免状态无限膨胀
- 定期清理过期数据

**记忆口诀**：
> Reducer 控增长，定期清理防膨胀
