# 一句话总结

**相似度度量是向量检索系统的核心评判机制,通过L2距离、余弦相似度、内积等数学函数量化向量间的相似程度,不同度量方式适用于不同的数据类型和业务场景,Milvus 2.6支持9种度量方式覆盖稠密向量、二值向量和文本检索,度量选型直接决定检索结果的准确性、相关性和性能表现,2026年混合检索(BM25+向量)已成为生产环境标准配置。**

---

## 核心要素拆解

### 1. 本质定义
**相似度度量** = 量化向量间相似程度的数学函数

### 2. 主要方法
- **L2距离**：直线距离（欧几里得空间）
- **余弦相似度**：角度相似（方向）
- **内积**：投影长度（长度+方向）

### 3. 数据类型覆盖
- 稠密向量：L2、IP、COSINE
- 二值向量：HAMMING、JACCARD、SUBSTRUCTURE、SUPERSTRUCTURE
- 文本检索：BM25
- 多向量：MaxSim

### 4. 应用价值
- 决定检索准确性
- 影响性能表现
- 适配业务场景

### 5. 2026年标准
- 混合检索成为标配
- 多向量检索用于复杂场景
- 度量选型需要基准测试验证

## 三个关键认知

### 认知1：度量方式决定检索质量

```
相同的向量数据 + 不同的度量方式 = 不同的检索结果

示例：
查询："苹果公司的产品"
候选1："iPhone是苹果公司的手机"
候选2："苹果是一种水果"

COSINE度量：候选1排名更高（语义相似）
关键词匹配：两者都包含"苹果"（无法区分）
```

---

### 认知2：没有"最好"的度量

```
最好的度量 = 最适合你场景的度量

文本检索 → COSINE（语义相似）
推荐系统 → IP（偏好强度）
图像去重 → HAMMING（位级比较）
混合检索 → BM25 + COSINE（2026标准）
```

---

### 认知3：2026年混合检索是标准

```
单一度量的局限：
- 纯向量检索：可能漏掉关键词精确匹配
- 纯关键词检索：无法理解语义

混合检索的优势：
BM25（精确匹配） + 向量（语义理解）
= 更高的检索质量
```

## 学习成果检验

如果你能回答以下问题，说明已经掌握了相似度度量：

1. **L2、IP、COSINE有什么区别？**
   - L2：直线距离
   - IP：内积（长度+方向）
   - COSINE：角度（只看方向）

2. **什么时候用COSINE，什么时候用L2？**
   - COSINE：文本检索（Embedding归一化）
   - L2：图像检索（绝对位置有意义）

3. **Milvus支持哪些度量方式？**
   - 稠密向量：L2、IP、COSINE
   - 二值向量：HAMMING、JACCARD、SUBSTRUCTURE、SUPERSTRUCTURE
   - 专用：BM25、MaxSim

4. **如何选择合适的度量方式？**
   - 看数据类型（稠密/二值/文本）
   - 看Embedding模型（是否归一化）
   - 看业务场景（语义/推荐/去重）
   - 基准测试验证

5. **2026年的标准配置是什么？**
   - 混合检索（BM25 + 向量）
   - 多向量检索（MaxSim）
   - 度量选型基准测试

## 核心价值总结

**相似度度量的三大价值：**

1. **量化抽象概念**
   - 将"相似"转换为可计算的数值
   - 使计算机能够理解和比较数据

2. **支持大规模检索**
   - 提供统一的比较标准
   - 从百万、千万向量中找到最相似的

3. **适配不同场景**
   - 不同度量捕捉不同的相似性
   - 根据业务需求选择最优度量
