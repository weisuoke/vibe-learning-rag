## 问题 1：依赖安装失败

### 1.1 pnpm 未安装

**症状**：
```bash
bash: pnpm: command not found
```

**诊断**：
```bash
# 检查 pnpm 是否安装
which pnpm

# 检查 npm 版本
npm --version
```

**解决方案**：
```bash
# 安装 pnpm
npm install -g pnpm

# 验证安装
pnpm --version
# 输出: 10.23.0
```

---

### 1.2 Node.js 版本过低

**症状**：
```bash
ERR_PNPM_UNSUPPORTED_ENGINE Unsupported environment (bad pnpm and/or Node.js version)

Your pnpm version is compatible with the following Node.js versions: >=22.12.0

Got:
  Node.js: v20.10.0
  pnpm: 10.23.0
```

**诊断**：
```bash
# 检查 Node.js 版本
node --version

# 检查 package.json 要求
cat package.json | grep engines
```

**解决方案**：
```bash
# 使用 nvm 安装 Node.js 22
nvm install 22
nvm use 22

# 验证版本
node --version
# 输出: v22.12.0

# 重新安装依赖
pnpm install
```

---

### 1.3 原生依赖编译失败

**症状（macOS）**：
```bash
gyp ERR! build error
gyp ERR! stack Error: `make` failed with exit code: 2
gyp ERR! stack     at ChildProcess.onExit (/usr/local/lib/node_modules/npm/node_modules/node-gyp/lib/build.js:194:23)
```

**诊断**：
```bash
# 检查 Xcode Command Line Tools
xcode-select -p

# 检查 Python 版本
python3 --version

# 检查编译工具
which make
which gcc
```

**解决方案（macOS）**：
```bash
# 安装 Xcode Command Line Tools
xcode-select --install

# 重新安装依赖
rm -rf node_modules
pnpm install
```

**症状（Linux）**：
```bash
gyp ERR! build error
gyp ERR! stack Error: not found: make
```

**解决方案（Linux）**：
```bash
# 安装构建工具
sudo apt-get update
sudo apt-get install build-essential python3

# 重新安装依赖
rm -rf node_modules
pnpm install
```

**症状（Windows）**：
```bash
gyp ERR! find VS
gyp ERR! find VS msvs_version not set from command line or npm config
```

**解决方案（Windows）**：
```bash
# 安装 Visual Studio Build Tools
# 下载: https://visualstudio.microsoft.com/downloads/

# 或使用 windows-build-tools
npm install --global windows-build-tools

# 重新安装依赖
rm -rf node_modules
pnpm install
```

### 2.2 TypeScript 类型错误

**症状**：
```bash
src/core/config.ts:45:7 - error TS2322: Type 'string | undefined' is not assignable to type 'string'.

45   const apiKey: string = process.env.ANTHROPIC_API_KEY
         ~~~~~~
```

**诊断**：
```bash
# 检查 TypeScript 配置
cat tsconfig.json

# 检查类型定义
ls node_modules/@types/
```

**解决方案**：
```typescript
// 方案 1: 使用类型断言
const apiKey: string = process.env.ANTHROPIC_API_KEY!

// 方案 2: 使用默认值
const apiKey: string = process.env.ANTHROPIC_API_KEY || ''

// 方案 3: 使用类型守卫
const apiKey = process.env.ANTHROPIC_API_KEY
if (!apiKey) {
  throw new Error('ANTHROPIC_API_KEY is required')
}
// 此时 apiKey 类型为 string
```

### 2.4 内存不足

**症状**：
```bash
FATAL ERROR: Reached heap limit Allocation failed - JavaScript heap out of memory
```

**诊断**：
```bash
# 检查系统内存
free -h  # Linux
vm_stat  # macOS

# 检查 Node.js 内存限制
node -e "console.log(v8.getHeapStatistics().heap_size_limit / 1024 / 1024)"
```

**解决方案**：
```bash
# 增加 Node.js 内存限制
export NODE_OPTIONS='--max-old-space-size=4096'

# 重新构建
pnpm build

# 或在 package.json 中配置
{
  "scripts": {
    "build": "NODE_OPTIONS='--max-old-space-size=4096' pnpm canvas:a2ui:bundle && tsdown"
  }
}
```

## 问题 3：运行时错误

### 3.1 端口被占用

**症状**：
```bash
Error: listen EADDRINUSE: address already in use :::18789
```

**诊断**：
```bash
# 查找占用端口的进程
lsof -i :18789

# 预期输出
COMMAND   PID   USER   FD   TYPE DEVICE SIZE/OFF NODE NAME
node    12345  user   23u  IPv6  0x...      0t0  TCP *:18789 (LISTEN)
```

**解决方案**：
```bash
# 方案 1: 杀死占用进程
kill -9 12345

# 方案 2: 使用其他端口
OPENCLAW_PORT=18790 pnpm gateway:dev

# 方案 3: 修改配置
# ~/.openclaw/config/settings.json
{
  "gateway": {
    "port": 18790
  }
}
```

## 问题 4：性能问题

### 4.1 构建速度慢

**症状**：
```bash
Build completed in 5m 30s
```

**诊断**：
```bash
# 检查 CPU 使用率
top

# 检查磁盘 I/O
iostat

# 检查构建配置
cat tsdown.config.ts
```

**解决方案**：
```typescript
// tsdown.config.ts
export default defineConfig({
  // 启用并行构建
  workers: true,

  // 启用缓存
  cache: true,

  // 减少入口点
  entry: [
    'src/index.ts',
    'src/entry.ts'
  ],

  // 禁用 source map
  sourcemap: false,

  // 启用增量构建
  incremental: true
})
```

---

### 4.2 测试运行慢

**症状**：
```bash
Test Files  45 passed (45)
     Tests  234 passed (234)
  Duration  5m 30s
```

**诊断**：
```bash
# 检查测试配置
cat vitest.unit.config.ts

# 检查测试文件数量
find . -name "*.test.ts" | wc -l
```

**解决方案**：
```typescript
// vitest.unit.config.ts
export default defineConfig({
  test: {
    // 使用 threads pool
    pool: 'threads',

    // 增加并发数
    poolOptions: {
      threads: {
        maxThreads: 8,
        minThreads: 4
      }
    },

    // 禁用覆盖率
    coverage: {
      enabled: false
    },

    // 只运行修改的测试
    changed: true
  }
})
```

### 4.3 Gateway 响应慢

**症状**：
```bash
Request took 5000ms
```

**诊断**：
```bash
# 使用 Node.js profiler
node --prof dist/daemon-cli.js gateway

# 运行一些请求
curl http://localhost:18789/api/message -d '{"text":"test"}'

# 停止 Gateway (Ctrl+C)

# 生成性能报告
node --prof-process isolate-0x*.log > profile.txt

# 查看报告
cat profile.txt
```

**解决方案**：
```typescript
// 1. 启用缓存
import NodeCache from 'node-cache'

const cache = new NodeCache({ stdTTL: 600 })

export async function processMessage(text: string) {
  // 检查缓存
  const cached = cache.get(text)
  if (cached) return cached

  // 处理消息
  const response = await generateResponse(text)

  // 缓存结果
  cache.set(text, response)

  return response
}

// 2. 使用连接池
import { Pool } from 'pg'

const pool = new Pool({
  max: 20,
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000
})

// 3. 启用压缩
import compression from 'compression'

app.use(compression())
```

## 问题 5：跨平台兼容性

### 5.1 路径分隔符问题

**症状（Windows）**：
```bash
Error: ENOENT: no such file or directory, open 'src\core\config.ts'
```

**诊断**：
```typescript
// 检查路径
console.log(__dirname)
console.log(__filename)
```

**解决方案**：
```typescript
import path from 'path'

// ❌ 不好：硬编码路径分隔符
const configPath = 'src/core/config.ts'

// ✅ 好：使用 path.join
const configPath = path.join('src', 'core', 'config.ts')

// ✅ 好：使用 path.resolve
const configPath = path.resolve(__dirname, 'src', 'core', 'config.ts')
```

---

### 5.2 换行符问题

**症状**：
```bash
Git shows all files as modified on Windows
```

**诊断**：
```bash
# 检查 Git 配置
git config core.autocrlf

# 检查文件换行符
file src/core/config.ts
```

**解决方案**：
```bash
# 配置 Git
git config --global core.autocrlf true  # Windows
git config --global core.autocrlf input # macOS/Linux

# 重新检出文件
git rm --cached -r .
git reset --hard
```

---

### 5.3 权限问题

**症状（Linux/macOS）**：
```bash
EACCES: permission denied, open '/usr/local/lib/node_modules/pnpm'
```

**诊断**：
```bash
# 检查文件权限
ls -la /usr/local/lib/node_modules/

# 检查当前用户
whoami
```

**解决方案**：
```bash
# 方案 1: 修复权限
sudo chown -R $(whoami) /usr/local/lib/node_modules

# 方案 2: 使用 nvm（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 22
nvm use 22

# 方案 3: 使用 sudo（不推荐）
sudo npm install -g pnpm
```

## 核心洞察

### 1. 问题排查需要系统化

**表面**：遇到错误就改代码

**实际**：
- 收集完整的错误信息
- 理解错误的根本原因
- 验证解决方案的有效性
- 记录问题和解决方案

---

### 2. 日志是最好的朋友

**表面**：console.log 够用了

**实际**：
- 结构化日志便于分析
- 日志级别控制输出
- 日志聚合便于监控
- 日志持久化便于回溯

---

### 3. 性能问题需要数据支持

**表面**：感觉慢就优化

**实际**：
- 使用 profiler 找到瓶颈
- 测量优化前后的性能
- 避免过早优化
- 关注关键路径

---

## 一句话总结

**OpenClaw 问题排查的关键是：系统化收集信息，理解错误根因，使用正确的调试工具，验证解决方案有效性。**
