## 实战代码

### 场景1：基础状态设计

```python
"""
场景1：基础状态设计
演示：TypedDict 定义、Reducer 函数、状态更新
"""

from typing_extensions import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver

# ===== 1. 状态定义 =====
print("=== 场景1：基础状态设计 ===\n")

class CustomerServiceState(TypedDict):
    """客服系统状态定义"""
    # 不可变字段（会被覆盖）
    user_id: str
    session_id: str
    query: str
    
    # 可变字段（使用 Reducer 聚合）
    messages: Annotated[list[str], add]  # 对话历史
    search_results: Annotated[list[dict], add]  # 搜索结果
    
    # 状态标记
    is_resolved: bool
    confidence_score: float


# ===== 2. 节点函数 =====

def understand_query(state: CustomerServiceState) -> dict:
    """理解用户查询"""
    query = state["query"]
    print(f"[理解查询] 用户问题: {query}")
    
    # 返回部分状态更新
    return {
        "messages": [f"系统: 正在理解您的问题..."],
        "confidence_score": 0.85
    }


def search_knowledge(state: CustomerServiceState) -> dict:
    """搜索知识库"""
    query = state["query"]
    print(f"[搜索知识] 查询: {query}")
    
    # 模拟搜索结果
    results = [
        {"title": "常见问题1", "content": "答案1"},
        {"title": "常见问题2", "content": "答案2"}
    ]
    
    return {
        "search_results": results,
        "messages": [f"系统: 找到 {len(results)} 条相关信息"]
    }


def generate_response(state: CustomerServiceState) -> dict:
    """生成回复"""
    results = state["search_results"]
    print(f"[生成回复] 基于 {len(results)} 条搜索结果")
    
    response = f"根据您的问题，我找到了 {len(results)} 条相关信息..."
    
    return {
        "messages": [f"客服: {response}"],
        "is_resolved": True
    }


# ===== 3. 构建图 =====

builder = StateGraph(CustomerServiceState)

# 添加节点
builder.add_node("understand", understand_query)
builder.add_node("search", search_knowledge)
builder.add_node("generate", generate_response)

# 添加边
builder.add_edge(START, "understand")
builder.add_edge("understand", "search")
builder.add_edge("search", "generate")
builder.add_edge("generate", END)

# 编译图
graph = builder.compile()

# ===== 4. 执行测试 =====

initial_state = {
    "user_id": "user_001",
    "session_id": "session_001",
    "query": "如何重置密码？",
    "messages": [],
    "search_results": [],
    "is_resolved": False,
    "confidence_score": 0.0
}

result = graph.invoke(initial_state)

print("\n=== 最终状态 ===")
print(f"用户ID: {result['user_id']}")
print(f"会话ID: {result['session_id']}")
print(f"是否解决: {result['is_resolved']}")
print(f"置信度: {result['confidence_score']}")
print(f"\n对话历史:")
for msg in result["messages"]:
    print(f"  {msg}")
print(f"\n搜索结果数: {len(result['search_results'])}")
```

**运行输出**：
```
=== 场景1：基础状态设计 ===

[理解查询] 用户问题: 如何重置密码？
[搜索知识] 查询: 如何重置密码？
[生成回复] 基于 2 条搜索结果

=== 最终状态 ===
用户ID: user_001
会话ID: session_001
是否解决: True
置信度: 0.85

对话历史:
  系统: 正在理解您的问题...
  系统: 找到 2 条相关信息
  客服: 根据您的问题，我找到了 2 条相关信息...

搜索结果数: 2
```

### 场景2：Context Schema 使用

```python
"""
场景2：Context Schema 使用
演示：运行时上下文传递、不可变参数
"""

from langgraph.runtime import Runtime

# ===== 1. 定义 Context Schema =====
print("\n=== 场景2：Context Schema 使用 ===\n")

class ServiceContext(TypedDict):
    """服务上下文（不可变）"""
    api_key: str
    max_results: int
    timeout: int
    language: str


class SimpleState(TypedDict):
    """简单状态"""
    query: str
    results: list[dict]


# ===== 2. 使用 Runtime 访问 Context =====

def search_with_context(state: SimpleState, runtime: Runtime[ServiceContext]) -> dict:
    """使用 Context 中的配置进行搜索"""
    # 访问运行时上下文
    max_results = runtime.context.get("max_results", 5)
    language = runtime.context.get("language", "zh")
    
    print(f"[搜索配置] 最大结果数: {max_results}, 语言: {language}")
    
    # 模拟搜索
    results = [
        {"id": i, "title": f"结果{i}"}
        for i in range(max_results)
    ]
    
    return {"results": results}


# ===== 3. 构建图 =====

builder = StateGraph(
    state_schema=SimpleState,
    context_schema=ServiceContext
)

builder.add_node("search", search_with_context)
builder.add_edge(START, "search")
builder.add_edge("search", END)

graph = builder.compile()

# ===== 4. 执行测试 =====

# 传递 context 参数
result = graph.invoke(
    {"query": "测试查询", "results": []},
    context={
        "api_key": "sk-xxx",
        "max_results": 3,
        "timeout": 30,
        "language": "zh"
    }
)

print(f"\n搜索结果数: {len(result['results'])}")
for r in result["results"]:
    print(f"  - {r['title']}")
```

**运行输出**：
```
=== 场景2：Context Schema 使用 ===

[搜索配置] 最大结果数: 3, 语言: zh

搜索结果数: 3
  - 结果0
  - 结果1
  - 结果2
```

### 场景3：多用户状态隔离

```python
"""
场景3：多用户状态隔离
演示：thread_id 隔离、Checkpoint 持久化
"""

# ===== 1. 多用户状态定义 =====
print("\n=== 场景3：多用户状态隔离 ===\n")

class MultiUserState(TypedDict):
    """多用户状态"""
    user_id: str
    messages: Annotated[list[str], add]
    counter: int


def chat_node(state: MultiUserState) -> dict:
    """聊天节点"""
    user_id = state["user_id"]
    counter = state.get("counter", 0) + 1
    
    message = f"用户 {user_id} 的第 {counter} 条消息"
    print(f"[聊天] {message}")
    
    return {
        "messages": [message],
        "counter": counter
    }


# ===== 2. 构建图（带 Checkpoint）=====

builder = StateGraph(MultiUserState)
builder.add_node("chat", chat_node)
builder.add_edge(START, "chat")
builder.add_edge("chat", END)

# 使用 MemorySaver 实现持久化
checkpointer = MemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# ===== 3. 模拟多用户交互 =====

# 用户1 的第一次对话
config_user1 = {"configurable": {"thread_id": "user_001"}}
result1 = graph.invoke(
    {"user_id": "user_001", "messages": [], "counter": 0},
    config=config_user1
)
print(f"用户1 消息数: {len(result1['messages'])}, 计数器: {result1['counter']}")

# 用户2 的第一次对话
config_user2 = {"configurable": {"thread_id": "user_002"}}
result2 = graph.invoke(
    {"user_id": "user_002", "messages": [], "counter": 0},
    config=config_user2
)
print(f"用户2 消息数: {len(result2['messages'])}, 计数器: {result2['counter']}")

# 用户1 的第二次对话（状态会恢复）
result1_2 = graph.invoke(
    {"user_id": "user_001", "messages": [], "counter": 0},
    config=config_user1
)
print(f"用户1 消息数: {len(result1_2['messages'])}, 计数器: {result1_2['counter']}")

print("\n=== 用户1 的完整对话历史 ===")
for msg in result1_2["messages"]:
    print(f"  {msg}")
```

**运行输出**：
```
=== 场景3：多用户状态隔离 ===

[聊天] 用户 user_001 的第 1 条消息
用户1 消息数: 1, 计数器: 1
[聊天] 用户 user_002 的第 1 条消息
用户2 消息数: 1, 计数器: 1
[聊天] 用户 user_001 的第 2 条消息
用户1 消息数: 2, 计数器: 2

=== 用户1 的完整对话历史 ===
  用户 user_001 的第 1 条消息
  用户 user_001 的第 2 条消息
```

### 场景4：并行执行与状态聚合

```python
"""
场景4：并行执行与状态聚合
演示：扇出/扇入模式、Reducer 聚合
"""

# ===== 1. 并行状态定义 =====
print("\n=== 场景4：并行执行与状态聚合 ===\n")

class ParallelState(TypedDict):
    """并行执行状态"""
    query: str
    results: Annotated[list[dict], add]  # 使用 add reducer 聚合


def search_source_a(state: ParallelState) -> dict:
    """搜索源A"""
    print("[源A] 开始搜索...")
    return {
        "results": [
            {"source": "A", "title": "结果A1"},
            {"source": "A", "title": "结果A2"}
        ]
    }


def search_source_b(state: ParallelState) -> dict:
    """搜索源B"""
    print("[源B] 开始搜索...")
    return {
        "results": [
            {"source": "B", "title": "结果B1"},
            {"source": "B", "title": "结果B2"}
        ]
    }


def search_source_c(state: ParallelState) -> dict:
    """搜索源C"""
    print("[源C] 开始搜索...")
    return {
        "results": [
            {"source": "C", "title": "结果C1"}
        ]
    }


def aggregate_results(state: ParallelState) -> dict:
    """聚合结果"""
    results = state["results"]
    print(f"\n[聚合] 收到 {len(results)} 条结果")
    
    # 按来源分组
    by_source = {}
    for r in results:
        source = r["source"]
        by_source.setdefault(source, []).append(r)
    
    print("按来源统计:")
    for source, items in by_source.items():
        print(f"  源{source}: {len(items)} 条")
    
    return {}


# ===== 2. 构建并行图 =====

builder = StateGraph(ParallelState)

# 添加节点
builder.add_node("source_a", search_source_a)
builder.add_node("source_b", search_source_b)
builder.add_node("source_c", search_source_c)
builder.add_node("aggregate", aggregate_results)

# 扇出：从 START 到三个搜索源（并行执行）
builder.add_edge(START, "source_a")
builder.add_edge(START, "source_b")
builder.add_edge(START, "source_c")

# 扇入：三个搜索源到聚合节点
builder.add_edge("source_a", "aggregate")
builder.add_edge("source_b", "aggregate")
builder.add_edge("source_c", "aggregate")

builder.add_edge("aggregate", END)

graph = builder.compile()

# ===== 3. 执行测试 =====

result = graph.invoke({
    "query": "测试查询",
    "results": []
})

print(f"\n=== 最终结果 ===")
print(f"总结果数: {len(result['results'])}")
for r in result["results"]:
    print(f"  [{r['source']}] {r['title']}")
```

**运行输出**：
```
=== 场景4：并行执行与状态聚合 ===

[源A] 开始搜索...
[源B] 开始搜索...
[源C] 开始搜索...

[聚合] 收到 5 条结果
按来源统计:
  源A: 2 条
  源B: 2 条
  源C: 1 条

=== 最终结果 ===
总结果数: 5
  [A] 结果A1
  [A] 结果A2
  [B] 结果B1
  [B] 结果B2
  [C] 结果C1
```

## 最佳实践总结

### 1. 状态设计原则

**TypedDict 定义**：
- 使用 TypedDict 明确定义状态类型
- 区分可变字段和不可变字段
- 使用清晰的字段命名

**Reducer 函数**：
- 使用 `Annotated[type, reducer]` 定义需要聚合的字段
- `operator.add` 适合列表追加
- 自定义 reducer 处理复杂聚合逻辑
- Reducer 函数应该是纯函数

### 2. Context Schema 使用

**适用场景**：
- API 密钥、数据库连接等不可变配置
- 用户ID、租户ID等会话标识
- 超时、重试次数等运行时参数

**最佳实践**：
- Context 在整个图执行过程中保持不变
- 通过 `Runtime[Context]` 访问上下文
- 不要在 Context 中存储可变状态

### 3. 多用户状态隔离

**隔离策略**：
- 使用 `thread_id` 实现会话隔离
- 使用 Checkpointer 实现状态持久化
- 定期清理过期的 Checkpoint

**命名规范**：
```python
thread_id = f"user_{user_id}_session_{session_id}"
```

### 4. 并行执行与聚合

**扇出/扇入模式**：
- 使用多条边实现扇出（并行执行）
- 使用 Reducer 聚合并行节点的结果
- 确保并行节点之间没有依赖关系

**Reducer 选择**：
- `operator.add`：列表追加
- 自定义函数：复杂聚合逻辑

## 常见误区

### 误区1：在状态中存储大对象

**错误示例**：
```python
class State(TypedDict):
    large_data: bytes  # 存储大文件内容
```

**正确做法**：
```python
class State(TypedDict):
    file_path: str  # 只存储文件路径
```

### 误区2：直接修改状态

**错误示例**：
```python
def node(state: State):
    state["messages"].append("new")  # 直接修改
    return state
```

**正确做法**：
```python
def node(state: State):
    return {"messages": ["new"]}  # 返回更新
```

### 误区3：忽略 Reducer 函数

**错误示例**：
```python
class State(TypedDict):
    messages: list  # 没有 reducer，会被覆盖
```

**正确做法**：
```python
class State(TypedDict):
    messages: Annotated[list, add]  # 使用 reducer 聚合
```
