## 五、一键脚本

### 5.1 完整安装脚本

```bash
#!/bin/bash

# 检查 Node.js 版本
if ! command -v node &> /dev/null; then
    echo "❌ Node.js not found"
    exit 1
fi

NODE_VERSION=$(node --version | cut -d'v' -f2)
REQUIRED_VERSION="22.12.0"

if [ "$(printf '%s\n' "$REQUIRED_VERSION" "$NODE_VERSION" | sort -V | head -n1)" != "$REQUIRED_VERSION" ]; then
    echo "❌ Node.js version must be >= $REQUIRED_VERSION"
    exit 1
fi

echo "✅ Node.js version: $NODE_VERSION"

# 检查 pnpm
if ! command -v pnpm &> /dev/null; then
    echo "⚙️  Enabling pnpm..."
    corepack enable pnpm
fi

echo "✅ pnpm version: $(pnpm --version)"

# 克隆仓库
if [ ! -d "openclaw" ]; then
    echo "⚙️  Cloning repository..."
    git clone https://github.com/openclaw/openclaw.git
fi

cd openclaw

# 安装依赖
echo "⚙️  Installing dependencies..."
pnpm install

# 构建 UI
echo "⚙️  Building UI..."
pnpm ui:build

# 编译 TypeScript
echo "⚙️  Building TypeScript..."
pnpm build

# 验证
echo "⚙️  Running checks..."
pnpm check

echo "✅ OpenClaw development environment is ready!"
echo "Run 'pnpm gateway:watch' to start development mode"
```

**使用方法**:

```bash
# 保存为 setup.sh
chmod +x setup.sh
./setup.sh
```

### 5.2 快速重置脚本

#!/bin/bash

## 六、常见问题快速排查

### 问题 1: pnpm install 失败

**错误信息**:

```
ERR_PNPM_UNSUPPORTED_ENGINE  Unsupported environment
```

**解决方案**:

```bash
# 检查 Node.js 版本
node --version

# 如果版本不符合，升级 Node.js
asdf install nodejs 22.12.0
asdf global nodejs 22.12.0
```

**数据来源**: `reference/06_openclaw_dev_setup_github_reddit.md` - 故障排查

---

### 问题 2: pnpm build 失败

**错误信息**:

```
Error: Cannot find module 'tsdown'
```

**解决方案**:

```bash
# 重新安装依赖
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

---

### 问题 3: 端口被占用

**错误信息**:

```
Error: Port 3000 is already in use
```

**解决方案**:

```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=3001 pnpm gateway:watch
```

---

### 问题 4: 类型检查失败

**错误信息**:

```
error TS2792: Cannot find module 'openclaw/plugin-sdk'
```

**解决方案**:

```bash
# 检查 tsconfig.json 的 paths 配置
cat tsconfig.json | grep paths

# 重新编译
pnpm build
```

## 七、开发工作流

### 7.1 日常开发流程

```bash
# 1. 启动 watch 模式
pnpm gateway:watch

# 2. 修改代码
vim src/index.ts

# 3. 自动重新编译（watch 模式会自动触发）

# 4. 运行测试
pnpm test

# 5. 提交代码
git add .
git commit -m "feat: add new feature"
```

---

### 7.2 代码质量检查流程

```bash
# 1. 格式检查
pnpm check

# 2. 类型检查
tsc --noEmit

# 3. 代码检查
oxlint

# 4. 运行测试
pnpm test

# 5. 如果全部通过，提交代码
git commit -m "feat: add new feature"
```

---

### 7.3 构建发布流程

```bash
# 1. 清理旧产物
rm -rf dist ui/dist

# 2. 构建 UI
pnpm ui:build

# 3. 编译 TypeScript
pnpm build

# 4. 运行测试
pnpm test

# 5. 发布（如果有权限）
pnpm publish
```
