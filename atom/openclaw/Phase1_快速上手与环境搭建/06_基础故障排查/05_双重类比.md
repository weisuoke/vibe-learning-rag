# OpenClaw 基础故障排查 - 双重类比

**通过后端开发类比和日常生活类比，帮助你快速理解 OpenClaw 基础故障排查的核心概念。**

---

## 类比 1：60秒诊断流程

### 后端开发类比：Kubernetes 健康检查

```typescript
// OpenClaw 60秒诊断 ≈ Kubernetes 健康检查流程
openclaw status              // kubectl get pods
openclaw gateway status      // kubectl describe pod
openclaw doctor              // kubectl doctor (if exists)
openclaw logs --follow       // kubectl logs -f

// 相似点：
// 1. 分层检查（Pod → Container → Application）
// 2. 标准化命令序列
// 3. 快速定位问题
// 4. 提供修复建议
```

**来源**: `reference/source_troubleshooting_main.md`

### 日常生活类比：医院急诊流程

```
OpenClaw 60秒诊断 = 医院急诊分诊流程

步骤对应：
1. openclaw status          → 测量生命体征（体温、血压、心率）
2. openclaw gateway status  → 检查主要器官（心脏、肺部）
3. openclaw doctor          → 医生初步诊断
4. openclaw logs --follow   → 持续监测病情变化

相似点：
- 标准化流程
- 快速定位问题
- 分层检查
- 60秒内完成初步诊断
```

## 类比 2：openclaw doctor 命令

### 后端开发类比：数据库自动修复

```typescript
// OpenClaw doctor ≈ MySQL mysqlcheck
openclaw doctor          // mysqlcheck --check-only
openclaw doctor --repair // mysqlcheck --auto-repair

// 相似点：
// 1. 自动检测问题
// 2. 一键修复
// 3. 修复前备份
// 4. 提供详细报告
// 5. 支持深度检查
```

**来源**: `reference/source_doctor_command.md`

### 日常生活类比：汽车诊断仪

```
OpenClaw doctor = 汽车诊断仪

操作流程：
1. 插上诊断仪        → 运行 openclaw doctor
2. 自动扫描故障码    → 检查配置、服务、环境
3. 显示故障原因      → 显示检查结果
4. 一键修复          → openclaw doctor --repair
5. 修复后验证        → 再次运行 doctor

相似点：
- 自动化诊断
- 一键修复
- 修复前备份
- 详细报告
```

## 类比 4：网关状态检查

### 后端开发类比：Nginx 状态检查

```bash
# OpenClaw Gateway ≈ Nginx
openclaw gateway status  # nginx -t && systemctl status nginx
openclaw gateway probe   # curl http://localhost/health

# 相似点：
# 1. 检查进程状态
# 2. 检查端口监听
# 3. 检查配置有效性
# 4. 健康探测
```

**来源**: `reference/source_gateway_troubleshooting.md`

### 日常生活类比：机场安检

```
OpenClaw Gateway = 机场安检站

检查项目：
1. 安检站是否开放    → Gateway 是否运行
2. 通道是否畅通      → 端口是否监听
3. 安检设备是否正常  → RPC probe 是否 ok
4. 人员是否就位      → 服务是否启动

相似点：
- 关键节点
- 必须正常运行
- 影响整体流程
- 需要持续监控
```

## 类比 7：端口冲突

### 后端开发类比：端口占用检查

```bash
# OpenClaw 端口 18789 ≈ 常见服务端口
openclaw gateway status  # netstat -tulpn | grep 18789
EADDRINUSE              # Address already in use

# 相似点：
# 1. 端口唯一性
# 2. 冲突检测
# 3. 进程管理
# 4. 端口释放
```

**来源**: `reference/search_openclaw_troubleshooting.md`

### 日常生活类比：停车位占用

```
OpenClaw 端口 18789 = 固定停车位

场景：
1. 你的车位（18789）被别人占了
   → 另一个 Gateway 实例在运行

2. 解决方法：
   - 让对方挪车    → openclaw gateway stop
   - 停到你的车位  → openclaw gateway start
   - 验证停好了    → openclaw gateway status

相似点：
- 资源唯一性
- 冲突检测
- 释放占用
- 重新分配
```
