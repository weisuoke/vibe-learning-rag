# 实战代码 - 场景 6：分块策略对比

> 对比 7 种分块策略的性能和效果，提供选型指南

---

## 场景描述

在 RAG 开发中，选择合适的分块策略至关重要。本场景实现一个完整的分块策略对比系统，对比 7 种主流分块策略的性能和效果，帮助开发者做出明智的选型决策。

**对比的 7 种策略**：
1. CharacterTextSplitter - 基础字符分块
2. RecursiveCharacterTextSplitter - 递归字符分块
3. TokenTextSplitter - Token 分块
4. HTMLHeaderTextSplitter - HTML 结构化分块
5. MarkdownHeaderTextSplitter - Markdown 结构化分块
6. PythonCodeTextSplitter - Python 代码分块
7. SemanticChunker - 语义分块

**评估维度**：
- 性能指标：速度、内存占用
- 效果指标：检索质量、上下文保留
- 适用场景：文档类型、数据规模

---

## 完整代码实现

```python
"""
分块策略对比系统
对比 7 种分块策略的性能和效果
"""

import time
import tracemalloc
from typing import List, Dict, Any, Tuple
from dataclasses import dataclass
import statistics

from langchain_text_splitters import (
    CharacterTextSplitter,
    RecursiveCharacterTextSplitter,
    TokenTextSplitter,
    HTMLHeaderTextSplitter,
    MarkdownHeaderTextSplitter,
    RecursiveCharacterTextSplitter as CodeSplitter,
    Language
)
from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma
from langchain.schema import Document


@dataclass
class PerformanceMetrics:
    """性能指标"""
    split_time: float  # 分块耗时（秒）
    memory_usage: float  # 内存占用（MB）
    chunks_count: int  # 分块数量
    avg_chunk_size: float  # 平均块大小
    min_chunk_size: int  # 最小块大小
    max_chunk_size: int  # 最大块大小


@dataclass
class QualityMetrics:
    """质量指标"""
    retrieval_precision: float  # 检索精确度
    retrieval_recall: float  # 检索召回率
    context_preservation: float  # 上下文保留度
    semantic_coherence: float  # 语义连贯性


@dataclass
class ComparisonResult:
    """对比结果"""
    strategy_name: str
    performance: PerformanceMetrics
    quality: QualityMetrics
    适用场景: List[str]
    优点: List[str]
    缺点: List[str]


class ChunkingStrategyComparator:
    """分块策略对比器"""

    def __init__(self, chunk_size: int = 1000, chunk_overlap: int = 200):
        """
        初始化对比器

        Args:
            chunk_size: 目标块大小
            chunk_overlap: 块重叠大小
        """
        self.chunk_size = chunk_size
        self.chunk_overlap = chunk_overlap
        self.embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

    def _measure_performance(
        self,
        splitter: Any,
        text: str
    ) -> Tuple[List[str], PerformanceMetrics]:
        """
        测量分块性能

        Args:
            splitter: 分块器
            text: 待分块文本

        Returns:
            (分块结果, 性能指标)
        """
        # 开始内存追踪
        tracemalloc.start()

        # 记录开始时间
        start_time = time.time()

        # 执行分块
        if isinstance(splitter, (HTMLHeaderTextSplitter, MarkdownHeaderTextSplitter)):
            chunks_docs = splitter.split_text(text)
            chunks = [doc.page_content for doc in chunks_docs]
        else:
            chunks = splitter.split_text(text)

        # 记录结束时间
        split_time = time.time() - start_time

        # 获取内存使用
        current, peak = tracemalloc.get_traced_memory()
        tracemalloc.stop()
        memory_usage = peak / 1024 / 1024  # 转换为 MB

        # 计算统计信息
        chunk_sizes = [len(chunk) for chunk in chunks]

        metrics = PerformanceMetrics(
            split_time=split_time,
            memory_usage=memory_usage,
            chunks_count=len(chunks),
            avg_chunk_size=statistics.mean(chunk_sizes) if chunk_sizes else 0,
            min_chunk_size=min(chunk_sizes) if chunk_sizes else 0,
            max_chunk_size=max(chunk_sizes) if chunk_sizes else 0
        )

        return chunks, metrics

    def _measure_quality(
        self,
        chunks: List[str],
        test_queries: List[str],
        ground_truth: Dict[str, List[str]]
    ) -> QualityMetrics:
        """
        测量分块质量

        Args:
            chunks: 分块结果
            test_queries: 测试查询
            ground_truth: 真实答案（查询 -> 相关块内容）

        Returns:
            质量指标
        """
        # 创建向量存储
        vectorstore = Chroma.from_texts(
            texts=chunks,
            embedding=self.embeddings,
            collection_name=f"test_{time.time()}"
        )

        retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

        # 计算检索指标
        precisions = []
        recalls = []

        for query in test_queries:
            # 检索相关文档
            retrieved_docs = retriever.get_relevant_documents(query)
            retrieved_contents = [doc.page_content for doc in retrieved_docs]

            # 获取真实相关内容
            relevant_contents = ground_truth.get(query, [])

            if not relevant_contents:
                continue

            # 计算精确度和召回率
            true_positives = sum(
                1 for content in retrieved_contents
                if any(rel in content for rel in relevant_contents)
            )

            precision = true_positives / len(retrieved_contents) if retrieved_contents else 0
            recall = true_positives / len(relevant_contents) if relevant_contents else 0

            precisions.append(precision)
            recalls.append(recall)

        # 计算上下文保留度（检查块之间的连续性）
        context_scores = []
        for i in range(len(chunks) - 1):
            # 检查相邻块是否有重叠
            overlap = self._calculate_overlap(chunks[i], chunks[i + 1])
            context_scores.append(overlap)

        # 计算语义连贯性（简化版：检查块内句子的完整性）
        coherence_scores = []
        for chunk in chunks:
            # 检查块是否以完整句子结束
            ends_with_period = chunk.strip().endswith(('.', '!', '?', '。', '！', '？'))
            coherence_scores.append(1.0 if ends_with_period else 0.5)

        return QualityMetrics(
            retrieval_precision=statistics.mean(precisions) if precisions else 0,
            retrieval_recall=statistics.mean(recalls) if recalls else 0,
            context_preservation=statistics.mean(context_scores) if context_scores else 0,
            semantic_coherence=statistics.mean(coherence_scores) if coherence_scores else 0
        )

    def _calculate_overlap(self, chunk1: str, chunk2: str) -> float:
        """计算两个块之间的重叠度"""
        words1 = set(chunk1.split())
        words2 = set(chunk2.split())

        if not words1 or not words2:
            return 0.0

        intersection = words1 & words2
        union = words1 | words2

        return len(intersection) / len(union) if union else 0.0

    def compare_strategies(
        self,
        text: str,
        test_queries: List[str],
        ground_truth: Dict[str, List[str]]
    ) -> List[ComparisonResult]:
        """
        对比所有分块策略

        Args:
            text: 待分块文本
            test_queries: 测试查询列表
            ground_truth: 真实答案

        Returns:
            对比结果列表
        """
        results = []

        # 1. CharacterTextSplitter
        print("\n" + "="*60)
        print("测试策略 1: CharacterTextSplitter")
        print("="*60)

        splitter1 = CharacterTextSplitter(
            separator="\n\n",
            chunk_size=self.chunk_size,
            chunk_overlap=self.chunk_overlap
        )

        chunks1, perf1 = self._measure_performance(splitter1, text)
        quality1 = self._measure_quality(chunks1, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="CharacterTextSplitter",
            performance=perf1,
            quality=quality1,
            适用场景=["简单文本", "段落分明的文档", "快速原型"],
            优点=["速度快", "实现简单", "内存占用低"],
            缺点=["可能切断语义", "不适合复杂文档", "缺少结构信息"]
        ))

        # 2. RecursiveCharacterTextSplitter
        print("\n" + "="*60)
        print("测试策略 2: RecursiveCharacterTextSplitter")
        print("="*60)

        splitter2 = RecursiveCharacterTextSplitter(
            chunk_size=self.chunk_size,
            chunk_overlap=self.chunk_overlap
        )

        chunks2, perf2 = self._measure_performance(splitter2, text)
        quality2 = self._measure_quality(chunks2, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="RecursiveCharacterTextSplitter",
            performance=perf2,
            quality=quality2,
            适用场景=["通用文本", "RAG 应用", "大多数场景"],
            优点=["语义完整性好", "通用性强", "推荐使用"],
            缺点=["速度略慢", "配置复杂度中等"]
        ))

        # 3. TokenTextSplitter
        print("\n" + "="*60)
        print("测试策略 3: TokenTextSplitter")
        print("="*60)

        splitter3 = TokenTextSplitter(
            chunk_size=self.chunk_size // 4,  # Token 约为字符的 1/4
            chunk_overlap=self.chunk_overlap // 4
        )

        chunks3, perf3 = self._measure_performance(splitter3, text)
        quality3 = self._measure_quality(chunks3, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="TokenTextSplitter",
            performance=perf3,
            quality=quality3,
            适用场景=["精确 Token 控制", "LLM API 调用", "成本优化"],
            优点=["精确控制 Token", "避免超出限制", "成本可控"],
            缺点=["速度最慢", "依赖 tiktoken", "内存占用高"]
        ))

        # 4. HTMLHeaderTextSplitter (需要 HTML 文本)
        print("\n" + "="*60)
        print("测试策略 4: HTMLHeaderTextSplitter")
        print("="*60)

        html_text = self._convert_to_html(text)
        splitter4 = HTMLHeaderTextSplitter(
            headers_to_split_on=[
                ("h1", "Header 1"),
                ("h2", "Header 2"),
                ("h3", "Header 3"),
            ]
        )

        chunks4, perf4 = self._measure_performance(splitter4, html_text)
        quality4 = self._measure_quality(chunks4, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="HTMLHeaderTextSplitter",
            performance=perf4,
            quality=quality4,
            适用场景=["HTML 文档", "网页内容", "结构化文档"],
            优点=["保留层级结构", "元数据丰富", "便于导航"],
            缺点=["仅适用 HTML", "需要预处理", "配置复杂"]
        ))

        # 5. MarkdownHeaderTextSplitter (需要 Markdown 文本)
        print("\n" + "="*60)
        print("测试策略 5: MarkdownHeaderTextSplitter")
        print("="*60)

        md_text = self._convert_to_markdown(text)
        splitter5 = MarkdownHeaderTextSplitter(
            headers_to_split_on=[
                ("#", "Header 1"),
                ("##", "Header 2"),
                ("###", "Header 3"),
            ]
        )

        chunks5, perf5 = self._measure_performance(splitter5, md_text)
        quality5 = self._measure_quality(chunks5, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="MarkdownHeaderTextSplitter",
            performance=perf5,
            quality=quality5,
            适用场景=["Markdown 文档", "技术文档", "README 文件"],
            优点=["保留标题层级", "元数据完整", "适合技术文档"],
            缺点=["仅适用 Markdown", "需要格式规范"]
        ))

        # 6. PythonCodeTextSplitter
        print("\n" + "="*60)
        print("测试策略 6: PythonCodeTextSplitter")
        print("="*60)

        code_text = self._convert_to_code(text)
        splitter6 = CodeSplitter.from_language(
            language=Language.PYTHON,
            chunk_size=self.chunk_size,
            chunk_overlap=self.chunk_overlap
        )

        chunks6, perf6 = self._measure_performance(splitter6, code_text)
        quality6 = self._measure_quality(chunks6, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="PythonCodeTextSplitter",
            performance=perf6,
            quality=quality6,
            适用场景=["Python 代码", "代码仓库", "代码问答"],
            优点=["语法感知", "保持代码完整", "支持多语言"],
            缺点=["仅适用代码", "配置复杂"]
        ))

        # 7. SemanticChunker (简化实现)
        print("\n" + "="*60)
        print("测试策略 7: SemanticChunker (简化版)")
        print("="*60)

        chunks7, perf7 = self._semantic_chunking(text)
        quality7 = self._measure_quality(chunks7, test_queries, ground_truth)

        results.append(ComparisonResult(
            strategy_name="SemanticChunker",
            performance=perf7,
            quality=quality7,
            适用场景=["高质量要求", "语义完整性重要", "预算充足"],
            优点=["语义完整性最好", "检索质量高", "上下文保留好"],
            缺点=["速度最慢", "成本最高", "需要 Embedding"]
        ))

        return results

    def _convert_to_html(self, text: str) -> str:
        """将文本转换为 HTML 格式（简化版）"""
        lines = text.split("\n\n")
        html_lines = ["<html><body>"]

        for i, line in enumerate(lines):
            if i % 3 == 0:
                html_lines.append(f"<h1>{line}</h1>")
            elif i % 3 == 1:
                html_lines.append(f"<h2>{line}</h2>")
            else:
                html_lines.append(f"<p>{line}</p>")

        html_lines.append("</body></html>")
        return "\n".join(html_lines)

    def _convert_to_markdown(self, text: str) -> str:
        """将文本转换为 Markdown 格式（简化版）"""
        lines = text.split("\n\n")
        md_lines = []

        for i, line in enumerate(lines):
            if i % 3 == 0:
                md_lines.append(f"# {line}")
            elif i % 3 == 1:
                md_lines.append(f"## {line}")
            else:
                md_lines.append(line)

        return "\n\n".join(md_lines)

    def _convert_to_code(self, text: str) -> str:
        """将文本转换为 Python 代码格式（简化版）"""
        lines = text.split("\n\n")
        code_lines = []

        for i, line in enumerate(lines):
            if i % 2 == 0:
                code_lines.append(f"class Section{i}:")
                code_lines.append(f'    """"{line}"""')
                code_lines.append("    pass")
            else:
                code_lines.append(f"def function_{i}():")
                code_lines.append(f'    """"{line}"""')
                code_lines.append("    return None")

        return "\n\n".join(code_lines)

    def _semantic_chunking(self, text: str) -> Tuple[List[str], PerformanceMetrics]:
        """
        语义分块（简化实现）

        使用句子 Embedding 相似度来分块
        """
        # 开始内存追踪
        tracemalloc.start()
        start_time = time.time()

        # 按句子分割
        sentences = text.split(". ")

        # 计算句子 Embeddings
        embeddings_list = self.embeddings.embed_documents(sentences)

        # 基于相似度合并句子
        chunks = []
        current_chunk = [sentences[0]]
        current_embedding = embeddings_list[0]

        for i in range(1, len(sentences)):
            # 计算余弦相似度
            similarity = self._cosine_similarity(
                current_embedding,
                embeddings_list[i]
            )

            # 如果相似度高，合并到当前块
            if similarity > 0.7 and len(". ".join(current_chunk)) < self.chunk_size:
                current_chunk.append(sentences[i])
                # 更新当前 Embedding（平均）
                current_embedding = [
                    (a + b) / 2
                    for a, b in zip(current_embedding, embeddings_list[i])
                ]
            else:
                # 否则，开始新块
                chunks.append(". ".join(current_chunk))
                current_chunk = [sentences[i]]
                current_embedding = embeddings_list[i]

        # 添加最后一个块
        if current_chunk:
            chunks.append(". ".join(current_chunk))

        # 记录性能
        split_time = time.time() - start_time
        current, peak = tracemalloc.get_traced_memory()
        tracemalloc.stop()
        memory_usage = peak / 1024 / 1024

        chunk_sizes = [len(chunk) for chunk in chunks]

        metrics = PerformanceMetrics(
            split_time=split_time,
            memory_usage=memory_usage,
            chunks_count=len(chunks),
            avg_chunk_size=statistics.mean(chunk_sizes) if chunk_sizes else 0,
            min_chunk_size=min(chunk_sizes) if chunk_sizes else 0,
            max_chunk_size=max(chunk_sizes) if chunk_sizes else 0
        )

        return chunks, metrics

    def _cosine_similarity(self, vec1: List[float], vec2: List[float]) -> float:
        """计算余弦相似度"""
        dot_product = sum(a * b for a, b in zip(vec1, vec2))
        norm1 = sum(a * a for a in vec1) ** 0.5
        norm2 = sum(b * b for b in vec2) ** 0.5

        if norm1 == 0 or norm2 == 0:
            return 0.0

        return dot_product / (norm1 * norm2)

def prepare_test_data() -> Tuple[str, List[str], Dict[str, List[str]]]:
    """
    准备测试数据

    Returns:
        (测试文本, 测试查询, 真实答案)
    """
    # 测试文本
    test_text = """
LangChain 是一个用于开发由语言模型驱动的应用程序的框架。
它提供了一系列工具和组件，帮助开发者构建复杂的 AI 应用。

TextSplitter 是 LangChain 的核心组件之一。
它负责将长文本分割成适合 LLM 处理的小块。
这对于 RAG（检索增强生成）应用至关重要。

CharacterTextSplitter 是最基础的分块器。
它使用简单的字符分隔符来分割文本。
适合快速原型开发和简单文本处理。

RecursiveCharacterTextSplitter 是最常用的分块器。
它通过递归尝试多个分隔符来保持语义完整性。
推荐配置是 chunk_size=1000 和 chunk_overlap=200。

TokenTextSplitter 基于 Token 进行分块。
它使用 tiktoken 库来精确控制 Token 数量。
适合需要精确控制 LLM API 成本的场景。

HTMLHeaderTextSplitter 专门用于 HTML 文档。
它可以根据 HTML 标签层级进行分块。
保留了文档的结构信息和元数据。

MarkdownHeaderTextSplitter 专门用于 Markdown 文档。
它可以根据标题层级进行分块。
非常适合技术文档和 README 文件。

PythonCodeTextSplitter 专门用于 Python 代码。
它具有语法感知能力，避免在函数中间切断。
支持多种编程语言的代码分块。

SemanticChunker 基于语义相似度进行分块。
它使用 Embedding 模型来计算句子之间的相似度。
提供最好的语义完整性，但速度较慢。

在实际应用中，合理的分块策略可以显著提高检索质量。
chunk_size 太小会导致上下文不足，太大会降低检索精度。
chunk_overlap 确保重要信息不会在块边界处丢失。

RAG 系统的性能取决于多个因素：
1. 文档分块策略
2. Embedding 模型选择
3. 向量存储配置
4. 检索器参数
5. LLM 提示词设计

优化这些因素需要大量实验和调优。
不同的文档类型需要不同的分块策略。
选择合适的分块器可以提高 RAG 应用的质量。
```

## 核心要点

### 1. 性能 vs 质量权衡

**速度排序**（快 → 慢）：
1. CharacterTextSplitter - 最快
2. MarkdownHeaderTextSplitter
3. HTMLHeaderTextSplitter
4. RecursiveCharacterTextSplitter
5. PythonCodeTextSplitter
6. TokenTextSplitter
7. SemanticChunker - 最慢

**质量排序**（高 → 低）：
1. SemanticChunker - 最高
2. RecursiveCharacterTextSplitter
3. TokenTextSplitter
4. MarkdownHeaderTextSplitter
5. HTMLHeaderTextSplitter
6. PythonCodeTextSplitter
7. CharacterTextSplitter - 最低

### 2. 选型决策矩阵

| 场景 | 推荐策略 | 理由 |
|------|---------|------|
| 通用 RAG 应用 | RecursiveCharacterTextSplitter | 平衡性能和质量 |
| 成本敏感 | TokenTextSplitter | 精确控制 Token |
| 高质量要求 | SemanticChunker | 最佳语义完整性 |
| HTML 文档 | HTMLHeaderTextSplitter | 保留结构信息 |
| Markdown 文档 | MarkdownHeaderTextSplitter | 保留标题层级 |
| 代码仓库 | PythonCodeTextSplitter | 语法感知 |
| 快速原型 | CharacterTextSplitter | 最快速度 |

### 3. 生产环境建议

**推荐配置**：
```python
# 通用场景
RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)

# 成本优化
TokenTextSplitter(
    chunk_size=512,  # Token 数量
    chunk_overlap=50
)

# 高质量场景
SemanticChunker(
    embeddings=OpenAIEmbeddings(),
    breakpoint_threshold_type="percentile"
)
```

**性能优化**：
- 使用缓存减少重复分块
- 批量处理文档
- 异步处理大文件
- 监控内存使用
