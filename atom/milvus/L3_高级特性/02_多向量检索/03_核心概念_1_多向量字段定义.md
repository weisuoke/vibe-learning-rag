## 1. 什么是多向量字段

### 1.1 定义

**多向量字段（Multi-Vector Fields）**：在一个 Milvus Collection 中定义多个向量字段，每个字段可以有不同的类型和维度。

**核心能力**：
- 一个 Collection 可以存储多个向量字段
- 向量字段可以是稠密或稀疏类型
- 不同向量字段可以有不同的维度
- 支持在多个向量字段上同时进行检索

**来源**：
> Milvus enables the storage of multiple vector fields within a single collection, which can be either sparse or dense and may vary in dimensionality. This capability supports multi-vector and hybrid search scenarios, allowing users to perform complex searches across different types of vector representations.
>
> — Milvus 官方文档 v2.6.x

### 1.2 应用场景

| 场景 | 向量字段组合 | 用途 |
|------|-------------|------|
| **多模态检索** | 图片向量 + 文本向量 | 电商商品搜索、内容平台 |
| **全文搜索增强** | BM25 稀疏向量 + 稠密向量 | 文档问答、知识库检索 |
| **多语言检索** | 中文向量 + 英文向量 | 跨语言搜索系统 |
| **多粒度检索** | 标题向量 + 内容向量 | 精准匹配 + 语义理解 |

## 2. 支持的向量类型

Milvus 支持 4 种向量类型，每种类型有不同的特点和应用场景。

### 2.1 FloatVector（浮点向量）

**定义**：使用 32 位浮点数表示的稠密向量。

**特点**：
- 最常用的向量类型
- 精度高，适合语义理解
- 内存占用：`维度 × 4 字节`

**适用场景**：
- 文本 Embedding（OpenAI、Sentence-Transformers）
- 图片 Embedding（CLIP、ResNet）
- 通用语义检索

**Python 定义**：
```python
from pymilvus import DataType, FieldSchema

# 定义 FloatVector 字段
field = FieldSchema(
    name="dense_vector",
    dtype=DataType.FLOAT_VECTOR,
    dim=768,  # 维度
    description="text dense vector"
)
```

**Go 定义**：
```go
&schemapb.FieldSchema{
    Name:     "dense_vector",
    DataType: schemapb.DataType_FloatVector,
    TypeParams: []*commonpb.KeyValuePair{
        {Key: common.DimKey, Value: "768"},
    },
}
```

### 2.2 SparseFloatVector（稀疏浮点向量）

**定义**：使用稀疏表示的浮点向量，只存储非零元素。

**特点**：
- 维度可以非常高（数万到数十万）
- 只存储非零值，节省内存
- 适合关键词匹配

**适用场景**：
- BM25 全文检索
- TF-IDF 向量
- 关键词精确匹配

**Python 定义**：
```python
from pymilvus import DataType, FieldSchema

# 定义 SparseFloatVector 字段（不需要指定维度）
field = FieldSchema(
    name="sparse_vector",
    dtype=DataType.SPARSE_FLOAT_VECTOR,
    description="text sparse vector"
)
```

**Go 定义**：
```go
&schemapb.FieldSchema{
    Name:     "sparse_vector",
    DataType: schemapb.DataType_SparseFloatVector,
    // 稀疏向量不需要指定维度
}
```

### 2.3 BinaryVector（二值向量）

**定义**：使用二进制位表示的向量，每个维度只有 0 或 1。

**特点**：
- 内存占用极小：`维度 / 8 字节`
- 计算速度快（位运算）
- 精度较低

**适用场景**：
- 图片指纹（SimHash）
- 快速去重
- 大规模检索（内存受限）

**Python 定义**：
```python
from pymilvus import DataType, FieldSchema

# 定义 BinaryVector 字段
field = FieldSchema(
    name="binary_vector",
    dtype=DataType.BINARY_VECTOR,
    dim=128,  # 维度必须是 8 的倍数
    description="binary fingerprint"
)
```

**Go 定义**：
```go
&schemapb.FieldSchema{
    Name:     "binary_vector",
    DataType: schemapb.DataType_BinaryVector,
    TypeParams: []*commonpb.KeyValuePair{
        {Key: common.DimKey, Value: "128"},
    },
}
```

### 2.4 BFloat16Vector（半精度浮点向量）

**定义**：使用 16 位浮点数表示的稠密向量。

**特点**：
- 内存占用减半：`维度 × 2 字节`
- 精度略低于 FloatVector
- 计算速度更快

**适用场景**：
- 大规模向量存储（内存优化）
- 对精度要求不高的场景
- GPU 加速计算

**Python 定义**：
```python
from pymilvus import DataType, FieldSchema

# 定义 BFloat16Vector 字段
field = FieldSchema(
    name="bfloat16_vector",
    dtype=DataType.BFLOAT16_VECTOR,
    dim=768,
    description="half precision vector"
)
```

**Go 定义**：
```go
&schemapb.FieldSchema{
    Name:     "bfloat16_vector",
    DataType: schemapb.DataType_BFloat16Vector,
    TypeParams: []*commonpb.KeyValuePair{
        {Key: common.DimKey, Value: "768"},
    },
}
```

## 3. Schema 定义方法

### 3.1 Python SDK 完整示例

```python
from pymilvus import CollectionSchema, FieldSchema, DataType

# 1. 定义主键字段
id_field = FieldSchema(
    name="id",
    dtype=DataType.INT64,
    is_primary=True,
    auto_id=True
)

# 2. 定义标量字段
text_field = FieldSchema(
    name="text",
    dtype=DataType.VARCHAR,
    max_length=1000
)

# 3. 定义多个向量字段
dense_vector_field = FieldSchema(
    name="dense_vector",
    dtype=DataType.FLOAT_VECTOR,
    dim=768,
    description="text dense embedding"
)

sparse_vector_field = FieldSchema(
    name="sparse_vector",
    dtype=DataType.SPARSE_FLOAT_VECTOR,
    description="BM25 sparse embedding"
)

image_vector_field = FieldSchema(
    name="image_vector",
    dtype=DataType.FLOAT_VECTOR,
    dim=512,
    description="image CLIP embedding"
)

# 4. 创建 Schema
schema = CollectionSchema(
    fields=[
        id_field,
        text_field,
        dense_vector_field,
        sparse_vector_field,
        image_vector_field
    ],
    description="Multi-vector collection for multimodal search"
)

# 5. 创建 Collection
from pymilvus import Collection

collection = Collection(
    name="multimodal_collection",
    schema=schema
)

print(f"Collection created with {len(schema.fields)} fields")
print(f"Vector fields: {[f.name for f in schema.fields if 'VECTOR' in str(f.dtype)]}")
```

### 3.2 Go SDK 完整示例

```go
package main

import (
    "github.com/milvus-io/milvus-proto/go-api/v2/commonpb"
    "github.com/milvus-io/milvus-proto/go-api/v2/schemapb"
)

func createMultiVectorSchema() *schemapb.CollectionSchema {
    return &schemapb.CollectionSchema{
        Name:        "multimodal_collection",
        Description: "Multi-vector collection for multimodal search",
        Fields: []*schemapb.FieldSchema{
            // 1. 主键字段
            {
                FieldID:      100,
                Name:         "id",
                IsPrimaryKey: true,
                AutoID:       true,
                DataType:     schemapb.DataType_Int64,
            },
            // 2. 标量字段
            {
                FieldID:  101,
                Name:     "text",
                DataType: schemapb.DataType_VarChar,
                TypeParams: []*commonpb.KeyValuePair{
                    {Key: "max_length", Value: "1000"},
                },
            },
            // 3. 稠密向量字段 1
            {
                FieldID:  102,
                Name:     "dense_vector",
                DataType: schemapb.DataType_FloatVector,
                TypeParams: []*commonpb.KeyValuePair{
                    {Key: "dim", Value: "768"},
                },
            },
            // 4. 稀疏向量字段
            {
                FieldID:  103,
                Name:     "sparse_vector",
                DataType: schemapb.DataType_SparseFloatVector,
            },
            // 5. 稠密向量字段 2（图片）
            {
                FieldID:  104,
                Name:     "image_vector",
                DataType: schemapb.DataType_FloatVector,
                TypeParams: []*commonpb.KeyValuePair{
                    {Key: "dim", Value: "512"},
                },
            },
        },
    }
}
```

## 4. 字段配置参数

### 4.1 通用参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✓ | 字段名称 |
| `dtype` | DataType | ✓ | 数据类型 |
| `description` | string | ✗ | 字段描述 |
| `is_primary` | bool | ✗ | 是否为主键（仅一个字段） |
| `auto_id` | bool | ✗ | 是否自动生成 ID |

### 4.2 向量字段特定参数

| 向量类型 | 必需参数 | 可选参数 | 说明 |
|---------|---------|---------|------|
| **FloatVector** | `dim` | - | 维度必须固定 |
| **SparseFloatVector** | - | - | 维度动态，不需要指定 |
| **BinaryVector** | `dim` | - | 维度必须是 8 的倍数 |
| **BFloat16Vector** | `dim` | - | 维度必须固定 |

### 4.3 维度配置示例

```python
# FloatVector - 固定维度
FieldSchema(name="vec1", dtype=DataType.FLOAT_VECTOR, dim=768)

# SparseFloatVector - 无需指定维度
FieldSchema(name="vec2", dtype=DataType.SPARSE_FLOAT_VECTOR)

# BinaryVector - 维度必须是 8 的倍数
FieldSchema(name="vec3", dtype=DataType.BINARY_VECTOR, dim=128)  # ✓
FieldSchema(name="vec4", dtype=DataType.BINARY_VECTOR, dim=100)  # ✗ 错误

# BFloat16Vector - 固定维度
FieldSchema(name="vec5", dtype=DataType.BFLOAT16_VECTOR, dim=512)
```

## 5. 索引创建要求

### 5.1 核心约束

**关键规则**：
- **所有向量字段都必须创建索引**才能加载 Collection
- 如果只为部分向量字段创建索引，`LoadCollection` 会失败
- 不同类型的向量字段使用不同的索引类型

**来源**：
```go
// 源码证据：tests/integration/hellomilvus/hybridsearch_test.go:98-148
// load without index on vector fields
loadStatus, err := c.MilvusClient.LoadCollection(ctx, &milvuspb.LoadCollectionRequest{
    DbName:         dbName,
    CollectionName: collectionName,
})
s.NoError(err)
s.Error(merr.Error(loadStatus))  // 预期失败：缺少索引
```

### 5.2 索引类型选择

| 向量类型 | 推荐索引类型 | 度量类型 |
|---------|-------------|---------|
| **FloatVector** | HNSW, IVF_FLAT, IVF_PQ | L2, IP, COSINE |
| **SparseFloatVector** | SPARSE_INVERTED_INDEX, SPARSE_WAND | IP |
| **BinaryVector** | BIN_IVF_FLAT, BIN_FLAT | JACCARD, HAMMING |
| **BFloat16Vector** | HNSW, IVF_FLAT | L2, IP, COSINE |

### 5.3 Python 索引创建示例

```python
from pymilvus import Collection

collection = Collection("multimodal_collection")

# 1. 为稠密向量创建 HNSW 索引
collection.create_index(
    field_name="dense_vector",
    index_params={
        "index_type": "HNSW",
        "metric_type": "IP",
        "params": {"M": 16, "efConstruction": 256}
    },
    index_name="dense_vector_idx"
)

# 2. 为稀疏向量创建索引
collection.create_index(
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "IP"
    },
    index_name="sparse_vector_idx"
)

# 3. 为图片向量创建索引
collection.create_index(
    field_name="image_vector",
    index_params={
        "index_type": "IVF_FLAT",
        "metric_type": "L2",
        "params": {"nlist": 128}
    },
    index_name="image_vector_idx"
)

# 4. 加载 Collection（所有向量字段都有索引后才能成功）
collection.load()
print("Collection loaded successfully")
```

### 5.4 Go 索引创建示例

```go
// 1. 为 FloatVector 创建索引
createIndexStatus, err := c.MilvusClient.CreateIndex(ctx, &milvuspb.CreateIndexRequest{
    CollectionName: collectionName,
    FieldName:      "dense_vector",
    IndexName:      "dense_vector_idx",
    ExtraParams:    constructIndexParam(768, "HNSW", "IP"),
})

// 2. 为 SparseFloatVector 创建索引
createIndexStatus, err = c.MilvusClient.CreateIndex(ctx, &milvuspb.CreateIndexRequest{
    CollectionName: collectionName,
    FieldName:      "sparse_vector",
    IndexName:      "sparse_vector_idx",
    ExtraParams:    constructIndexParam(0, "SPARSE_INVERTED_INDEX", "IP"),
})

// 3. 为 BinaryVector 创建索引
createIndexStatus, err = c.MilvusClient.CreateIndex(ctx, &milvuspb.CreateIndexRequest{
    CollectionName: collectionName,
    FieldName:      "binary_vector",
    IndexName:      "binary_vector_idx",
    ExtraParams:    constructIndexParam(128, "BIN_IVF_FLAT", "JACCARD"),

// 4. 加载 Collection
loadStatus, err := c.MilvusClient.LoadCollection(ctx, &milvuspb.LoadCollectionRequest{
    DbName:         dbName,
    CollectionName: collectionName,
})
```

## 6. BM25 函数集成

### 6.1 什么是 BM25 函数

**BM25 函数**：Milvus 内置的全文检索函数，可以自动将文本字段转换为稀疏向量。

**优势**：
- 无需手动计算 BM25 向量
- 自动分词和索引
- 与稠密向量无缝集成

### 6.2 Python 配置示例

```python
from pymilvus import (
    CollectionSchema, FieldSchema, DataType,
    Function, FunctionType
)

# 1. 定义文本字段（启用分词）
text_field = FieldSchema(
    name="text",
    dtype=DataType.VARCHAR,
    max_length=1000,
    enable_analyzer=True,  # 启用分词器
    enable_match=True      # 启用全文匹配
)

# 2. 定义稀疏向量字段（BM25 输出）
sparse_vector_field = FieldSchema(
    name="sparse_vector",
    dtype=DataType.SPARSE_FLOAT_VECTOR
)

# 3. 定义 BM25 函数
bm25_function = Function(
    name="bm25_fn",
    function_type=FunctionType.BM25,
    input_field_names=["text"],        # 输入：文本字段
    output_field_names=["sparse_vector"],  # 输出：稀疏向量字段
    params={}
)

# 4. 创建 Schema 并添加函数
schema = CollectionSchema(
    fields=[id_field, text_field, sparse_vector_field, dense_vector_field],
    description="Collection with BM25 function"
)
schema.add_function(bm25_function)

# 5. 创建 Collection
collection = Collection(name="bm25_collection", schema=schema)
```

### 6.3 数据插入（自动生成 BM25 向量）

```python
# 插入数据时，只需提供文本，BM25 向量会自动生成
data = [
    {
        "text": "Milvus is a vector database",
        "dense_vector": [0.1, 0.2, ...]  # 768 维
        # sparse_vector 会自动生成，无需提供
    },
    {
        "text": "Hybrid search combines dense and sparse vectors",
        "dense_vector": [0.3, 0.4, ...]
    }
]

collection.insert(data)
```

```python
# 场景 1：内存充足 - 使用 FloatVector
FieldSchema(name="vec", dtype=DataType.FLOAT_VECTOR, dim=768)
# 内存占用：768 × 4 = 3072 字节/向量

# 场景 2：内存受限 - 使用 BFloat16Vector
FieldSchema(name="vec", dtype=DataType.BFLOAT16_VECTOR, dim=768)
# 内存占用：768 × 2 = 1536 字节/向量（减半）

# 场景 3：极限优化 - 使用 BinaryVector
FieldSchema(name="vec", dtype=DataType.BINARY_VECTOR, dim=768)
# 内存占用：768 / 8 = 96 字节/向量（减少 32 倍）
```

## 8. 常见问题

### Q1: 一个 Collection 最多可以有多少个向量字段？

**答**：Milvus 没有明确限制向量字段数量，但建议不超过 10 个。过多的向量字段会：
- 增加索引创建时间
- 增加内存占用
- 降低检索性能

### Q2: 不同向量字段可以有不同的维度吗？

**答**：可以。每个向量字段独立配置维度。

```python
# ✓ 合法：不同维度的向量字段
FieldSchema(name="text_vec", dtype=DataType.FLOAT_VECTOR, dim=768)
FieldSchema(name="image_vec", dtype=DataType.FLOAT_VECTOR, dim=512)
FieldSchema(name="title_vec", dtype=DataType.FLOAT_VECTOR, dim=384)
```

### Q3: 稀疏向量的维度如何确定？

**答**：稀疏向量的维度是动态的，由实际数据决定。

```python
# 稀疏向量不需要指定维度
FieldSchema(name="sparse_vec", dtype=DataType.SPARSE_FLOAT_VECTOR)

# 插入数据时，维度由数据决定
from scipy.sparse import csr_matrix
sparse_data = csr_matrix([[0, 0.5, 0, 0.8, 0]])  # 维度 = 5
```

### Q4: 如果忘记为某个向量字段创建索引会怎样？

**答**：`LoadCollection` 会失败，报错信息类似：

```
Error: index not found for vector field 'xxx'
```

**解决方法**：为所有向量字段创建索引后再加载。

### Q5: BM25 函数生成的稀疏向量需要手动创建索引吗？

**答**：需要。BM25 函数只负责生成稀疏向量，索引仍需手动创建。

```python
# 1. 定义 BM25 函数（生成稀疏向量）
schema.add_function(bm25_function)

# 2. 创建 Collection
collection = Collection(name="bm25_col", schema=schema)

# 3. 为稀疏向量创建索引（必需）
collection.create_index(
    field_name="sparse_vector",
    index_params={"index_type": "SPARSE_INVERTED_INDEX", "metric_type": "IP"}
)

# 4. 加载 Collection
collection.load()
```

## 9. 总结

### 9.1 核心要点

1. **多向量字段能力**：一个 Collection 可以包含多个向量字段，支持不同类型和维度
2. **4 种向量类型**：FloatVector、SparseFloatVector、BinaryVector、BFloat16Vector
3. **索引创建要求**：所有向量字段都必须创建索引才能加载 Collection
4. **BM25 函数集成**：自动将文本转换为稀疏向量，无需手动计算

- **多模态检索**：图片 + 文本混合检索
