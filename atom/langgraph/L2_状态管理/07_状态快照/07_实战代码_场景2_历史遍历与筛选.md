# 实战代码 - 场景2：历史遍历与筛选

## 场景说明

本场景演示 `get_state_history()` 的完整用法。你将看到：

1. 基础遍历：列出所有快照的 step、source 和 checkpoint_id
2. `limit` 参数：只获取最近 N 个快照
3. `filter` 参数：按 metadata 字段过滤（如只看 `source="loop"` 的快照）
4. `before` 参数：获取指定 checkpoint 之前的历史
5. 实际应用：在历史中精准定位特定步骤的快照

`get_state_history()` 返回的是逆时间顺序的迭代器——最新的快照排在最前面。这是时间旅行和状态修改的基础。

[来源: reference/source_状态快照_01.md | LangGraph 源码分析]
[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

## 完整代码

```python
"""
LangGraph 状态快照实战 - 场景2：历史遍历与筛选
演示：get_state_history() 的 filter、before、limit 参数

核心知识点：
- get_state_history(config) 获取完整历史
- limit 限制返回数量
- filter 按 metadata 字段过滤
- before 获取指定 checkpoint 之前的历史
- 逆时间顺序遍历

运行环境：Python 3.13+, langgraph
安装依赖：uv add langgraph
"""

from typing import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver


# ============================================================
# 1. 定义状态与节点 —— 模拟 RAG 管道
# ============================================================

class RAGState(TypedDict):
    """模拟 RAG 管道状态"""
    query: str
    messages: Annotated[list[str], add]
    step_count: int
    docs_count: int


def node_rewrite(state: RAGState) -> dict:
    """查询改写"""
    rewritten = f"[改写] {state['query']} → 更精确的查询"
    return {
        "messages": [f"查询改写完成: {rewritten}"],
        "step_count": 1,
    }


def node_retrieve(state: RAGState) -> dict:
    """文档检索"""
    return {
        "messages": ["检索完成: 找到 5 篇相关文档"],
        "step_count": 2,
        "docs_count": 5,
    }


def node_rerank(state: RAGState) -> dict:
    """重排序"""
    return {
        "messages": ["重排序完成: 保留 Top-3 文档"],
        "step_count": 3,
        "docs_count": 3,
    }


def node_generate(state: RAGState) -> dict:
    """生成回答"""
    return {
        "messages": [f"生成完成: 基于 {state['docs_count']} 篇文档生成回答"],
        "step_count": 4,
    }


# ============================================================
# 2. 构建并执行图
# ============================================================

builder = StateGraph(RAGState)
builder.add_node("rewrite", node_rewrite)
builder.add_node("retrieve", node_retrieve)
builder.add_node("rerank", node_rerank)
builder.add_node("generate", node_generate)
builder.add_edge(START, "rewrite")
builder.add_edge("rewrite", "retrieve")
builder.add_edge("retrieve", "rerank")
builder.add_edge("rerank", "generate")
builder.add_edge("generate", END)

memory = InMemorySaver()
graph = builder.compile(checkpointer=memory)

config = {"configurable": {"thread_id": "rag-001"}}

print("=" * 60)
print("=== 执行 RAG 管道 ===")
print("=" * 60)

result = graph.invoke(
    {"query": "什么是向量数据库？", "messages": [], "step_count": 0, "docs_count": 0},
    config,
)
print(f"最终 messages: {result['messages']}")
print(f"最终 step_count: {result['step_count']}")


# ============================================================
# 3. 基础遍历 —— 列出所有历史快照
# ============================================================

print()
print("=" * 60)
print("=== 基础遍历：所有历史快照（逆时间顺序） ===")
print("=" * 60)

# get_state_history 返回迭代器，转为列表方便操作
all_states = list(graph.get_state_history(config))

print(f"\n总快照数: {len(all_states)}")
print(f"{'序号':<4} {'source':<8} {'step':<6} {'next':<20} {'checkpoint_id（前12位）'}")
print("-" * 70)

for i, state in enumerate(all_states):
    source = state.metadata.get("source", "?")
    step = state.metadata.get("step", "?")
    next_nodes = state.next
    cp_id = state.config["configurable"]["checkpoint_id"][:12] + "..."
    print(f"{i:<4} {source:<8} {step:<6} {str(next_nodes):<20} {cp_id}")

# 注意：逆时间顺序意味着 index=0 是最新的，index=-1 是最早的
print(f"\n第一个（index=0）是最新快照: step={all_states[0].metadata.get('step')}")
print(f"最后一个（index={len(all_states)-1}）是最早快照: step={all_states[-1].metadata.get('step')}")


# ============================================================
# 4. limit 参数 —— 只获取最近 N 个快照
# ============================================================

print()
print("=" * 60)
print("=== limit 参数：只获取最近 3 个快照 ===")
print("=" * 60)

# limit=3 只返回最近的 3 个快照
recent_states = list(graph.get_state_history(config, limit=3))

print(f"\n返回快照数: {len(recent_states)}（limit=3）")
for i, state in enumerate(recent_states):
    source = state.metadata.get("source", "?")
    step = state.metadata.get("step", "?")
    print(f"  [{i}] source={source}, step={step}, next={state.next}")

print(f"\n说明: limit 从最新的快照开始截取，适合只关心最近几步的场景")


# ============================================================
# 5. filter 参数 —— 按 metadata 字段过滤
# ============================================================

print()
print("=" * 60)
print("=== filter 参数：按 metadata 过滤 ===")
print("=" * 60)

# filter 接受一个字典，按 metadata 的字段进行精确匹配
# 常用过滤条件：source="loop" 只看循环执行的快照（排除 input）

# 5a: 只看 source="loop" 的快照（节点执行产生的）
print("\n--- 5a: filter={'source': 'loop'}（只看节点执行的快照） ---")
loop_states = list(graph.get_state_history(config, filter={"source": "loop"}))
print(f"  匹配数量: {len(loop_states)}")
for state in loop_states:
    step = state.metadata.get("step", "?")
    writes = state.metadata.get("writes", {})
    node_name = list(writes.keys())[0] if writes else "?"
    print(f"    step={step}, 写入节点={node_name}")

# 5b: 只看 source="input" 的快照（初始输入）
print("\n--- 5b: filter={'source': 'input'}（只看初始输入的快照） ---")
input_states = list(graph.get_state_history(config, filter={"source": "input"}))
print(f"  匹配数量: {len(input_states)}")
for state in input_states:
    step = state.metadata.get("step", "?")
    print(f"    step={step}, values.query={state.values.get('query', 'N/A')}")

# 5c: 按 step 过滤
print("\n--- 5c: filter={'step': 2}（只看 step=2 的快照） ---")
step2_states = list(graph.get_state_history(config, filter={"step": 2}))
print(f"  匹配数量: {len(step2_states)}")
for state in step2_states:
    source = state.metadata.get("source", "?")
    print(f"    source={source}, next={state.next}")
    print(f"    messages: {state.values.get('messages', [])}")


# ============================================================
# 6. before 参数 —— 获取指定 checkpoint 之前的历史
# ============================================================

print()
print("=" * 60)
print("=== before 参数：获取指定 checkpoint 之前的历史 ===")
print("=" * 60)

# 先找到 step=2（retrieve 执行后）的快照作为锚点
anchor_state = None
for state in all_states:
    if state.metadata.get("step") == 2:
        anchor_state = state
        break

if anchor_state:
    anchor_config = anchor_state.config
    anchor_cp_id = anchor_config["configurable"]["checkpoint_id"]
    print(f"\n锚点快照: step=2, checkpoint_id={anchor_cp_id[:12]}...")

    # before 参数接受一个 config，返回该 checkpoint 之前（不包含自身）的所有快照
    before_states = list(graph.get_state_history(config, before=anchor_config))

    print(f"锚点之前的快照数: {len(before_states)}")
    for i, state in enumerate(before_states):
        step = state.metadata.get("step", "?")
        source = state.metadata.get("source", "?")
        print(f"  [{i}] step={step}, source={source}, next={state.next}")

    print(f"\n说明: before 不包含锚点自身，只返回更早的快照")


# ============================================================
# 7. 组合使用 —— filter + limit + before
# ============================================================

print()
print("=" * 60)
print("=== 组合使用：filter + limit ===")
print("=" * 60)

# 只看 source="loop" 的快照，且只取最近 2 个
print("\n--- filter={'source': 'loop'} + limit=2 ---")
combined = list(graph.get_state_history(
    config,
    filter={"source": "loop"},
    limit=2,
))
print(f"  返回数量: {len(combined)}")
for state in combined:
    step = state.metadata.get("step", "?")
    writes = state.metadata.get("writes", {})
    node_name = list(writes.keys())[0] if writes else "?"
    print(f"    step={step}, node={node_name}")


# ============================================================
# 8. 实际应用：找到特定步骤的快照
# ============================================================

print()
print("=" * 60)
print("=== 实际应用：精准定位检索步骤的快照 ===")
print("=" * 60)

# 场景：RAG 管道执行完毕后，想查看检索步骤的状态
# 方法 1：通过 filter + step 定位
print("\n--- 方法 1：通过 filter={'step': 2} 定位 ---")
retrieve_states = list(graph.get_state_history(config, filter={"step": 2}))
if retrieve_states:
    rs = retrieve_states[0]
    print(f"  找到检索步骤快照:")
    print(f"    docs_count: {rs.values.get('docs_count')}")
    print(f"    messages: {rs.values.get('messages')}")
    print(f"    next: {rs.next}")
    print(f"    checkpoint_id: {rs.config['configurable']['checkpoint_id'][:12]}...")

# 方法 2：遍历历史，按 writes 中的节点名定位
print("\n--- 方法 2：通过 writes 中的节点名定位 ---")
for state in all_states:
    writes = state.metadata.get("writes", {})
    if "retrieve" in writes:
        print(f"  找到 retrieve 节点的快照:")
        print(f"    step: {state.metadata.get('step')}")
        print(f"    writes['retrieve']: {writes['retrieve']}")
        print(f"    checkpoint_id: {state.config['configurable']['checkpoint_id'][:12]}...")
        break

# 方法 3：遍历历史，按 next 字段定位（找到 next 包含目标节点的快照）
print("\n--- 方法 3：通过 next 字段定位（检索前的快照） ---")
for state in all_states:
    if "retrieve" in state.next:
        print(f"  找到 retrieve 执行前的快照:")
        print(f"    step: {state.metadata.get('step')}")
        print(f"    next: {state.next}")
        print(f"    说明: 这是 retrieve 即将执行但还未执行的状态")
        print(f"    checkpoint_id: {state.config['configurable']['checkpoint_id'][:12]}...")
        break


# ============================================================
# 9. 性能提示：迭代器 vs 列表
# ============================================================

print()
print("=" * 60)
print("=== 性能提示 ===")
print("=" * 60)

# get_state_history 返回的是迭代器，不是列表
# 对于大量快照，用迭代器可以避免一次性加载所有数据到内存

# 示例：只需要找到第一个匹配的快照，不需要遍历全部
print("\n--- 用迭代器提前终止（找到即停） ---")
for state in graph.get_state_history(config):
    if state.metadata.get("step") == 1:
        print(f"  找到 step=1 的快照，提前终止遍历")
        print(f"  checkpoint_id: {state.config['configurable']['checkpoint_id'][:12]}...")
        break  # 不会继续遍历剩余快照

# 示例：用 limit 替代 list + 切片
print("\n--- 用 limit 替代 list()[:N]（更高效） ---")
print(f"  推荐: list(graph.get_state_history(config, limit=3))")
print(f"  避免: list(graph.get_state_history(config))[:3]")
print(f"  原因: limit 在存储层就限制了返回数量，避免加载全部数据")
```

## 关键知识点解析

### 1. get_state_history 参数速查表

| 参数 | 类型 | 默认值 | 作用 |
|------|------|--------|------|
| `config` | `RunnableConfig` | 必填 | 指定 thread_id |
| `filter` | `dict` | `None` | 按 metadata 字段精确匹配过滤 |
| `before` | `RunnableConfig` | `None` | 只返回指定 checkpoint 之前的快照 |
| `limit` | `int` | `None` | 限制返回的最大快照数 |

### 2. filter 支持的字段

`filter` 字典的 key 对应 `CheckpointMetadata` 的字段：

```python
# 常用过滤条件
filter={"source": "loop"}     # 只看循环执行的快照
filter={"source": "input"}    # 只看初始输入
filter={"source": "update"}   # 只看手动更新的快照
filter={"step": 2}            # 只看特定步骤
```

### 3. 三种定位快照的方法对比

| 方法 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| `filter={"step": N}` | 知道目标步骤编号 | 简洁高效 | 需要知道 step 编号 |
| 遍历 `writes` 字段 | 知道目标节点名 | 按节点名定位，直观 | 需要遍历 |
| 遍历 `next` 字段 | 需要节点执行前的状态 | 能找到"即将执行"的状态 | 需要遍历 |

### 4. 逆时间顺序的含义

```
get_state_history 返回顺序:
  index=0  → 最新快照（图执行完毕后的状态）
  index=1  → 倒数第二个快照
  ...
  index=-1 → 最早快照（初始输入）

这与 git log 的顺序一致：最新的在最前面。
```

## 在 RAG 系统中的应用

历史遍历在 RAG 系统中的典型用途：

- **检索质量审计**：用 `filter={"step": N}` 定位检索步骤，检查返回的文档列表
- **管道性能分析**：遍历所有快照的 `created_at`，计算每步耗时
- **错误回溯**：当生成结果有问题时，从最新快照逆序回溯，找到第一个出错的步骤
- **A/B 测试**：对比不同 thread 的历史快照，分析不同参数对结果的影响
