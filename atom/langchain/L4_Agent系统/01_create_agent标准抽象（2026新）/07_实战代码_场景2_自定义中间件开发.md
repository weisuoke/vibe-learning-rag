# 实战代码 场景2：自定义 Middleware 开发

> 从装饰器到类继承——5 个递进示例，手把手教你写出生产级 Agent 中间件

---

## 场景概述

内置 Middleware 覆盖了常见需求，但真实项目总有定制化场景。本文用 5 个递进示例，从装饰器到类方式，展示自定义 Middleware 的开发模式。

---

## 示例1：装饰器方式——日志中间件

**解决的问题：** 快速给 Agent 加上调用日志，不想写一整个类。

```python
"""
装饰器方式定义 Middleware
演示：@before_model, @after_model, @wrap_model_call 三种装饰器
"""
from langchain.agents import create_agent
from langchain.agents.middleware import (
    before_model,
    after_model,
    wrap_model_call,
    AgentState,
)
from langchain_core.tools import tool
from langgraph.types import Runtime
import time


# ===== 1. 定义一个简单工具 =====
@tool
def search_docs(query: str) -> str:
    """搜索文档知识库"""
    return f"找到关于'{query}'的 3 篇文档：最佳实践、入门指南、常见问题"


# ===== 2. 装饰器方式定义三个中间件 =====

@before_model
def log_before(state: AgentState, runtime: Runtime) -> None:
    """模型调用前：记录当前消息数和最新消息"""
    msg_count = len(state["messages"])
    last_msg = state["messages"][-1].content if state["messages"] else "无"
    # 截断过长的消息，只显示前 50 个字符
    preview = last_msg[:50] + "..." if len(last_msg) > 50 else last_msg
    print(f"[BEFORE] 消息数: {msg_count}, 最新: {preview}")


@after_model
def log_after(state: AgentState, runtime: Runtime) -> None:
    """模型调用后：记录模型的决策（调工具还是直接回复）"""
    last_msg = state["messages"][-1]
    if hasattr(last_msg, "tool_calls") and last_msg.tool_calls:
        tools = [tc["name"] for tc in last_msg.tool_calls]
        print(f"[AFTER] 模型决定调用工具: {tools}")
    else:
        content = last_msg.content[:50]
        print(f"[AFTER] 模型直接回复: {content}...")


@wrap_model_call
def timing_wrapper(request, handler):
    """计时中间件：测量每次模型调用的耗时"""
    start = time.time()
    response = handler(request)
    elapsed = time.time() - start
    print(f"[TIMING] 模型调用耗时: {elapsed:.2f}s")
    return response


# ===== 3. 组装 Agent =====
agent = create_agent(
    "openai:gpt-4o-mini",
    tools=[search_docs],
    system_prompt="你是一个文档助手，用中文回答。需要时搜索知识库。",
    middleware=[log_before, log_after, timing_wrapper],
)

# ===== 4. 调用 =====
result = agent.invoke(
    {"messages": [{"role": "user", "content": "RAG 的最佳实践有哪些？"}]}
)
print(f"\n最终回复: {result['messages'][-1].content[:100]}...")
```

### 要点

- `@before_model` / `@after_model` 返回 `None` 表示不修改状态
- `@wrap_model_call` 必须调用 `handler(request)` 并返回结果，否则模型调用不会执行
- 三个装饰器各自独立，按 `middleware` 列表顺序执行

## 示例2：类方式——重试中间件

**解决的问题：** 需要持有实例状态（重试次数、延迟配置），装饰器不够用了。

```python
"""
类方式定义 Middleware
演示：AgentMiddleware 子类，实现带指数退避的重试逻辑
"""
from langchain.agents import create_agent
from langchain.agents.middleware import AgentMiddleware
from langchain_core.tools import tool
import time


class RetryMiddleware(AgentMiddleware):
    """自动重试中间件：模型调用失败时指数退避重试"""

    def __init__(self, max_retries: int = 3, base_delay: float = 1.0):
        self.max_retries = max_retries
        self.base_delay = base_delay

    def wrap_model_call(self, request, handler):
        last_error = None
        for attempt in range(self.max_retries):
            try:
                return handler(request)
            except Exception as e:
                last_error = e
                if attempt < self.max_retries - 1:
                    # 指数退避：1s → 2s → 4s
                    wait = self.base_delay * (2 ** attempt)
                    print(
                        f"[RETRY] 第 {attempt + 1}/{self.max_retries} 次失败，"
                        f"等待 {wait:.1f}s 后重试: {e}"
                    )
                    time.sleep(wait)
        # 所有重试都失败，抛出最后一个错误
        raise last_error


# ===== 使用 =====
@tool
def get_weather(city: str) -> str:
    """查询城市天气"""
    return f"{city}今天晴，25°C"


agent = create_agent(
    "openai:gpt-4o-mini",
    tools=[get_weather],
    system_prompt="你是天气助手，用中文回答。",
    middleware=[RetryMiddleware(max_retries=3, base_delay=1.0)],
)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "北京今天天气怎么样？"}]}
)
print(result["messages"][-1].content)
```

### 类方式 vs 装饰器

装饰器 `@wrap_model_call` 是一个函数，无法持有 `max_retries` 等配置。类方式通过 `__init__` 接收参数，实例变量随时可用。

## 示例3：动态 Prompt 中间件

**解决的问题：** 不同用户角色需要不同的系统提示，system_prompt 写死不够灵活。

```python
"""
动态 Prompt 中间件
演示：根据运行时 context 动态切换系统提示
"""
from typing import TypedDict
from langchain.agents import create_agent
from langchain.agents.middleware import dynamic_prompt, ModelRequest
from langchain_core.tools import tool


# ===== 1. 定义运行时上下文类型 =====
class UserContext(TypedDict):
    user_role: str   # developer / manager / general
    language: str    # zh / en


# ===== 2. 定义动态 Prompt 中间件 =====
@dynamic_prompt
def adaptive_prompt(request: ModelRequest) -> str:
    """根据用户角色和语言动态生成系统提示"""
    ctx = request.runtime.context
    role = ctx.get("user_role", "general")
    lang = ctx.get("language", "zh")

    role_prompts = {
        "developer": "你是技术专家。回答要包含代码示例和底层原理。",
        "manager": "你是项目顾问。用简洁语言总结要点，避免技术细节。",
        "general": "你是友好的助手。用通俗易懂的语言回答。",
    }
    base = role_prompts.get(role, role_prompts["general"])
    if lang == "en":
        base += " Please respond in English."
    return base


# ===== 3. 定义工具 =====
@tool
def search_knowledge(query: str) -> str:
    """搜索技术知识库"""
    return f"关于'{query}'的知识：RAG 是检索增强生成的缩写，结合检索和生成两个步骤。"


# ===== 4. 创建 Agent（注意 context_schema）=====
agent = create_agent(
    "openai:gpt-4o-mini",
    tools=[search_knowledge],
    # 不传 system_prompt，完全由 dynamic_prompt 控制
    middleware=[adaptive_prompt],
    context_schema=UserContext,
)

# ===== 5. 不同角色调用同一个 Agent =====
# 开发者视角：会得到技术细节和代码示例
result_dev = agent.invoke(
    {"messages": [{"role": "user", "content": "什么是 RAG？"}]},
    context={"user_role": "developer", "language": "zh"},
)
print("=== 开发者视角 ===")
print(result_dev["messages"][-1].content[:200])

# 管理者视角：会得到简洁的业务总结
result_mgr = agent.invoke(
    {"messages": [{"role": "user", "content": "什么是 RAG？"}]},
    context={"user_role": "manager", "language": "zh"},
)
print("=== 管理者视角 ===")
print(result_mgr["messages"][-1].content[:200])
```

`context_schema=UserContext` 的作用是类型检查——传入的 context 缺少必要字段会报错。`@dynamic_prompt` 内部通过 `request.runtime.context` 读取运行时上下文，返回的字符串会被设为本次调用的 system_message。

## 示例4：自定义状态中间件（指标收集）

**解决的问题：** 想在 Agent 状态中追踪模型调用次数和 token 消耗，但 `AgentState` 默认只有 `messages`。

```python
"""
自定义状态中间件
演示：通过 state_schema 扩展 AgentState，收集调用指标
"""
from typing import Annotated, Any
from langchain.agents import create_agent, AgentState
from langchain.agents.middleware import AgentMiddleware
from langchain.agents.middleware.types import OmitFromSchema
from langchain_core.tools import tool
from langgraph.types import Runtime


# ===== 1. 扩展状态类型 =====
class MetricsState(AgentState):
    """在标准 AgentState 基础上添加指标字段"""
    # OmitFromSchema: 不暴露给输入/输出，纯内部使用
    model_call_count: Annotated[int, OmitFromSchema(input=True, output=True)]
    total_input_tokens: Annotated[int, OmitFromSchema(input=True, output=True)]
    total_output_tokens: Annotated[int, OmitFromSchema(input=True, output=True)]


# ===== 2. 指标收集中间件 =====
class MetricsMiddleware(AgentMiddleware):
    """收集模型调用指标的中间件"""

    # 告诉 create_agent 合并这个扩展状态
    state_schema = MetricsState

    def before_model(self, state: MetricsState, runtime: Runtime) -> dict[str, Any]:
        """每次模型调用前：递增计数器"""
        count = state.get("model_call_count", 0) + 1
        print(f"[METRICS] 第 {count} 次模型调用")
        return {"model_call_count": count}

    def after_model(self, state: MetricsState, runtime: Runtime) -> dict[str, Any] | None:
        """每次模型调用后：累计 token 消耗"""
        last_msg = state["messages"][-1]
        usage = getattr(last_msg, "usage_metadata", None)

        if usage:
            input_tokens = state.get("total_input_tokens", 0) + usage.get("input_tokens", 0)
            output_tokens = state.get("total_output_tokens", 0) + usage.get("output_tokens", 0)
            print(
                f"[METRICS] 本次: input={usage.get('input_tokens', 0)}, "
                f"output={usage.get('output_tokens', 0)} | "
                f"累计: input={input_tokens}, output={output_tokens}"
            )
            return {
                "total_input_tokens": input_tokens,
                "total_output_tokens": output_tokens,
            }
        return None

    def after_agent(self, state: MetricsState, runtime: Runtime) -> None:
        """Agent 结束时：打印汇总"""
        total_calls = state.get("model_call_count", 0)
        total_in = state.get("total_input_tokens", 0)
        total_out = state.get("total_output_tokens", 0)
        print(f"\n[METRICS SUMMARY] 模型调用 {total_calls} 次, "
              f"总 token: {total_in + total_out} (in={total_in}, out={total_out})")


# ===== 3. 定义工具 =====
@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    try:
        result = eval(expression)
        return f"计算结果: {result}"
    except Exception as e:
        return f"计算错误: {e}"


# ===== 4. 创建 Agent =====
agent = create_agent(
    "openai:gpt-4o-mini",
    tools=[calculate],
    system_prompt="你是一个计算助手，用中文回答。需要计算时调用工具。",
    middleware=[MetricsMiddleware()],
)

# ===== 5. 调用 =====
result = agent.invoke(
    {"messages": [{"role": "user", "content": "帮我算一下 (100 + 200) * 1.15 等于多少"}]}
)
print(f"\n回复: {result['messages'][-1].content}")
```

### 预期输出

```
[METRICS] 第 1 次模型调用
[METRICS] 本次: input=85, output=22 | 累计: input=85, output=22
[METRICS] 第 2 次模型调用
[METRICS] 本次: input=120, output=45 | 累计: input=205, output=67
[METRICS SUMMARY] 模型调用 2 次, 总 token: 272 (in=205, out=67)
```

多个 middleware 各自定义 `state_schema` 时，`create_agent` 会自动合并所有字段到统一的状态类型中。

## 示例5：组合多个自定义中间件

**解决的问题：** 真实项目需要同时使用多个中间件，理解它们的执行顺序至关重要。

```python
"""
组合多个自定义中间件
演示：装饰器 + 类方式混合使用，理解洋葱模型执行顺序
"""
from typing import Annotated, Any, TypedDict
from langchain.agents import create_agent, AgentState
from langchain.agents.middleware import (
    AgentMiddleware, before_model, after_model, dynamic_prompt, ModelRequest,
)
from langchain.agents.middleware.types import OmitFromSchema
from langchain_core.tools import tool
from langgraph.types import Runtime
import time


# ===== 1. 扩展状态 + 上下文类型 =====
class AppState(AgentState):
    call_count: Annotated[int, OmitFromSchema(input=True, output=True)]

class AppContext(TypedDict):
    user_role: str


# ===== 2. 装饰器方式：日志 + 动态 Prompt =====
@before_model
def log_request(state: AgentState, runtime: Runtime) -> None:
    print(f"  [LOG] -> 准备调用模型，当前 {len(state['messages'])} 条消息")

@after_model
def log_response(state: AgentState, runtime: Runtime) -> None:
    last = state["messages"][-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        print(f"  [LOG] <- 模型调用工具: {[tc['name'] for tc in last.tool_calls]}")
    else:
        print(f"  [LOG] <- 模型回复: {last.content[:60]}...")

@dynamic_prompt
def role_prompt(request: ModelRequest) -> str:
    role = request.runtime.context.get("user_role", "general")
    if role == "developer":
        return "你是技术专家，回答要包含代码和原理。用中文。"
    return "你是友好的助手，用简单语言回答。用中文。"


# ===== 3. 类方式：重试 + 计数 =====
class ProductionMiddleware(AgentMiddleware):
    state_schema = AppState

    def __init__(self, max_retries: int = 2):
        self.max_retries = max_retries

    def before_model(self, state: AppState, runtime: Runtime) -> dict[str, Any]:
        count = state.get("call_count", 0) + 1
        print(f"[PROD] 第 {count} 次模型调用")
        return {"call_count": count}

    def wrap_model_call(self, request, handler):
        for attempt in range(self.max_retries):
            try:
                start = time.time()
                result = handler(request)
                print(f"[PROD] 调用成功，耗时 {time.time() - start:.2f}s")
                return result
            except Exception as e:
                if attempt < self.max_retries - 1:
                    print(f"[PROD] 重试 {attempt + 1}: {e}")
                    time.sleep(1)
                else:
                    raise

    def after_agent(self, state: AppState, runtime: Runtime) -> None:
        print(f"[PROD] Agent 结束，共 {state.get('call_count', 0)} 次模型调用")


# ===== 4. 工具 =====
@tool
def search_docs(query: str) -> str:
    """搜索文档"""
    return f"找到关于'{query}'的文档：RAG 最佳实践指南"


# ===== 5. 组装——顺序决定执行层次 =====
agent = create_agent(
    "openai:gpt-4o-mini",
    tools=[search_docs],
    middleware=[
        role_prompt,                       # 最外层：动态 Prompt
        ProductionMiddleware(max_retries=2),  # 中间层：重试 + 计数
        log_request, log_response,         # 内层：日志
    ],
    context_schema=AppContext,
)

# ===== 6. 调用 =====
result = agent.invoke(
    {"messages": [{"role": "user", "content": "RAG 系统怎么做检索优化？"}]},
    context={"user_role": "developer"},
)
print(f"最终回复: {result['messages'][-1].content[:150]}...")
```

### 执行顺序与预期输出

```
role_prompt(最外层 wrap) → ProductionMiddleware.before_model(计数)
  → log_request → [模型调用] → log_response
  → ProductionMiddleware.wrap_model_call(计时)
  → (有 tool_calls? → 执行工具 → 回到 before_model)
  → ProductionMiddleware.after_agent(汇总)
```

```
[PROD] 第 1 次模型调用
  [LOG] -> 准备调用模型，当前 1 条消息
[PROD] 调用成功，耗时 1.35s
  [LOG] <- 模型调用工具: ['search_docs']
[PROD] 第 2 次模型调用
  [LOG] -> 准备调用模型，当前 4 条消息
[PROD] 调用成功，耗时 2.10s
  [LOG] <- 模型回复: 根据搜索结果，RAG 检索优化可以从以下几个方面入手...
[PROD] Agent 结束，共 2 次模型调用
```

## 开发速查表

| 需求 | 推荐方式 | 钩子 |
|------|----------|------|
| 简单日志 | `@before_model` / `@after_model` | before/after |
| 动态 prompt | `@dynamic_prompt` | wrap（内部） |
| 重试/回退 | `class + wrap_model_call` | wrap_model_call |
| 扩展状态 | `class + state_schema` | 任意 |
| 工具权限 | `class + wrap_tool_call` | wrap_tool_call |
| 输入验证 | `class + before_agent` | before_agent |

决策规则：只需一个钩子且无状态 → 装饰器；多钩子/有配置/需扩展状态 → 类。

---

## 关键要点

1. **装饰器方式**适合单钩子简单场景：`@before_model`、`@after_model`、`@wrap_model_call`、`@dynamic_prompt`
2. **类方式**适合多钩子复杂场景：继承 `AgentMiddleware`，在 `__init__` 中接收配置
3. `wrap_model_call` 是最强大的钩子——能重试、回退、缓存、短路返回
4. `state_schema` 允许 Middleware 扩展 Agent 状态，多个 Middleware 的 schema 会自动合并
5. 多个 Middleware 按洋葱模型执行：列表第一个是最外层，最后一个最接近模型调用
