# DocumentTransformer文档转换 - 最小可用知识

**学习目标：** 5分钟快速上手 DocumentTransformer，掌握20%核心知识解决80%问题

**适用场景：** 需要在 RAG 管道中对文档进行清洗、过滤、重排序的开发者

---

## 核心理念

**最小可用原则：** 掌握以下内容，就能开始使用 DocumentTransformer

```
DocumentTransformer 的本质 = 输入一组 Document，输出一组经过变换的 Document
```

**为什么需要？**
- 原始文档往往包含 HTML 标签、重复内容、噪音信息
- 直接送入 LLM 会浪费 Token、降低回答质量
- DocumentTransformer 在检索之后、生成之前，对文档做"精加工"

## 最小可用知识点

### 4.1 理解 transform_documents 接口

**一句话：** 所有 DocumentTransformer 只有一个核心方法——输入 Document 列表，输出 Document 列表

**为什么重要？**
- 这是整个 DocumentTransformer 体系的统一接口
- 理解了这个接口，就能使用任何转换器
- 接口极简，学习成本极低

**最简单的用法：**

```python
from langchain_core.documents import Document
from langchain_core.documents.transformers import BaseDocumentTransformer

# ===== 1. 理解接口签名 =====
# transform_documents(documents: Sequence[Document], **kwargs) -> Sequence[Document]
#
# 输入：Document 列表
# 输出：Document 列表（可能数量不同、内容不同）

# ===== 2. 准备测试文档 =====
docs = [
    Document(page_content="Python 是一门编程语言", metadata={"source": "wiki"}),
    Document(page_content="Python 用于数据科学", metadata={"source": "blog"}),
    Document(page_content="Java 是另一门编程语言", metadata={"source": "wiki"}),
]

# ===== 3. 任何转换器都遵循同一接口 =====
# transformer = SomeTransformer()
# result = transformer.transform_documents(docs)
# result 仍然是 Document 列表，可以继续传给下一个转换器

print(f"输入: {len(docs)} 个文档")
print(f"每个文档都有 page_content 和 metadata")
print(f"转换后: 仍然是 Document 列表，可以链式传递")
```

**运行输出：**
```
输入: 3 个文档
每个文档都有 page_content 和 metadata
转换后: 仍然是 Document 列表，可以链式传递
```

**什么时候用？**
- 任何时候你需要对检索到的文档做后处理
- 构建 RAG 管道时，在 Retriever 和 LLM 之间插入转换步骤

**关键理解：**
- 输入输出类型一致 → 转换器可以任意串联
- 不修改原始文档 → 返回新的 Document 对象
- 支持异步 → `atransform_documents()` 自动可用

[来源: reference/source_document_transformer_01.md | BaseDocumentTransformer 核心架构分析]

### 4.2 Html2TextTransformer 清洗 HTML

**一句话：** 把 HTML 文档转成干净的纯文本，去掉所有标签和格式噪音

**为什么重要？**
- 网页抓取的文档都是 HTML 格式
- HTML 标签会干扰 Embedding 质量和 LLM 理解
- 这是 RAG 管道中最常见的文档清洗操作

**最简单的用法：**

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import Html2TextTransformer

# ===== 1. 模拟从网页抓取的 HTML 文档 =====
html_docs = [
    Document(
        page_content="""
        <html>
        <head><title>Python 教程</title></head>
        <body>
            <h1>Python 入门</h1>
            <p>Python 是一门<strong>简单易学</strong>的编程语言。</p>
            <ul>
                <li>语法简洁</li>
                <li>生态丰富</li>
            </ul>
            <script>console.log('noise');</script>
            <style>.hidden { display: none; }</style>
        </body>
        </html>
        """,
        metadata={"source": "https://example.com/python"}
    ),
]

print("=== 转换前（HTML） ===")
print(html_docs[0].page_content[:100], "...")

# ===== 2. 使用 Html2TextTransformer 清洗 =====
html2text = Html2TextTransformer()
clean_docs = html2text.transform_documents(html_docs)

print("\n=== 转换后（纯文本） ===")
print(clean_docs[0].page_content)

# ===== 3. metadata 保持不变 =====
print(f"\nmetadata 保留: {clean_docs[0].metadata}")
```

**运行输出：**
```
=== 转换前（HTML） ===
        <html>
        <head><title>Python 教程</title></head>
        <body>
            <h1>Python 入门</h ...

=== 转换后（纯文本） ===
# Python 入门

Python 是一门**简单易学**的编程语言。

  * 语法简洁
  * 生态丰富

metadata 保留: {'source': 'https://example.com/python'}
```

**什么时候用？**
- 从网页抓取文档后，送入 Embedding 或 LLM 之前
- 处理包含 HTML 标签的任何文档
- RAG 管道的第一步清洗操作

**安装依赖：**
```bash
uv add html2text
```

### 4.3 EmbeddingsRedundantFilter 去重

**一句话：** 通过 Embedding 向量相似度自动去除重复或高度相似的文档

**为什么重要？**
- 检索结果经常包含内容高度重复的文档
- 重复文档浪费 Token，不提供新信息
- 去重后 LLM 能看到更多样化的上下文，回答质量更高

**最简单的用法：**

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import EmbeddingsRedundantFilter
from langchain_openai import OpenAIEmbeddings
import os
from dotenv import load_dotenv

load_dotenv()

# ===== 1. 模拟检索结果（包含重复内容） =====
retrieved_docs = [
    Document(page_content="Python 是一门解释型编程语言，语法简洁优雅"),
    Document(page_content="Python 是解释型语言，以简洁的语法著称"),  # 与第1条高度相似
    Document(page_content="Python 广泛用于数据科学和机器学习"),
    Document(page_content="Python 在数据科学和 ML 领域应用广泛"),  # 与第3条高度相似
    Document(page_content="FastAPI 是基于 Python 的现代 Web 框架"),
]

print(f"去重前: {len(retrieved_docs)} 个文档")

# ===== 2. 创建去重过滤器 =====
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
redundant_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.95  # 相似度 > 0.95 视为重复
)

# ===== 3. 执行去重 =====
filtered_docs = redundant_filter.transform_documents(retrieved_docs)

print(f"去重后: {len(filtered_docs)} 个文档")
print("\n保留的文档:")
for i, doc in enumerate(filtered_docs, 1):
    print(f"  {i}. {doc.page_content[:50]}...")
```

**运行输出：**
```
去重前: 5 个文档
去重后: 3 个文档

保留的文档:
  1. Python 是一门解释型编程语言，语法简洁优雅...
  2. Python 广泛用于数据科学和机器学习...
  3. FastAPI 是基于 Python 的现代 Web 框架...
```

**什么时候用？**
- 检索结果中有多个来源返回相似内容时
- 多路检索（混合检索）合并结果后
- 需要在有限 Token 内最大化信息多样性时

**关键参数：**
- `embeddings`：用于计算文档向量的 Embedding 模型
- `similarity_threshold`：相似度阈值，默认 0.95（越高越严格）

### 4.4 LongContextReorder 重排序

**一句话：** 把最相关的文档放在开头和结尾，解决 LLM "中间遗忘"问题

**为什么重要？**
- 研究表明 LLM 对长上下文的中间部分关注度最低（"Lost in the Middle" 问题）
- 简单的重排序就能显著提升回答质量
- 零成本优化——不需要额外的 API 调用

**最简单的用法：**

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import LongContextReorder

# ===== 1. 模拟按相关度排序的检索结果 =====
# 假设文档已按相关度从高到低排列
docs = [
    Document(page_content="[相关度最高] RAG 的核心是检索增强生成"),
    Document(page_content="[相关度次高] RAG 结合了检索和生成两个步骤"),
    Document(page_content="[相关度中等] 向量检索是 RAG 的关键技术"),
    Document(page_content="[相关度较低] Embedding 模型将文本转为向量"),
    Document(page_content="[相关度最低] LLM 可以理解自然语言"),
]

print("=== 重排序前（按相关度降序） ===")
for i, doc in enumerate(docs, 1):
    print(f"  位置{i}: {doc.page_content[:30]}...")

# ===== 2. 使用 LongContextReorder =====
reorder = LongContextReorder()
reordered_docs = reorder.transform_documents(docs)

print("\n=== 重排序后（重要文档在首尾） ===")
for i, doc in enumerate(reordered_docs, 1):
    print(f"  位置{i}: {doc.page_content[:30]}...")
```

**运行输出：**
```
=== 重排序前（按相关度降序） ===
  位置1: [相关度最高] RAG 的核心是检索增强生成...
  位置2: [相关度次高] RAG 结合了检索和生成两个步骤...
  位置3: [相关度中等] 向量检索是 RAG 的关键技术...
  位置4: [相关度较低] Embedding 模型将文本转为向量...
  位置5: [相关度最低] LLM 可以理解自然语言...

=== 重排序后（重要文档在首尾） ===
  位置1: [相关度最低] LLM 可以理解自然语言...
  位置2: [相关度较低] Embedding 模型将文本转为向量...
  位置3: [相关度中等] 向量检索是 RAG 的关键技术...
  位置4: [相关度次高] RAG 结合了检索和生成两个步骤...
  位置5: [相关度最高] RAG 的核心是检索增强生成...
```

**重排序策略图解：**
```
原始顺序（按相关度降序）：
  [最高] [次高] [中等] [较低] [最低]

重排序后（重要文档在首尾）：
  [最低] [较低] [中等] [次高] [最高]
         ↑ 不重要的放中间 ↑
  ↑ LLM 关注开头              LLM 关注结尾 ↑
```

**什么时候用？**
- 检索结果超过 3-4 个文档时
- 使用长上下文窗口的 LLM 时
- 需要零成本提升回答质量时

### 4.5 DocumentCompressorPipeline 组合多个转换器

**一句话：** 把多个转换器串联成管道，按顺序依次处理文档

**为什么重要？**
- 实际 RAG 管道通常需要多步转换（清洗 → 去重 → 重排序）
- Pipeline 让你用一行代码组合多个转换器
- 可以直接集成到 ContextualCompressionRetriever 中

**最简单的用法：**

```python
from langchain_core.documents import Document
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain_community.document_transformers import (
    EmbeddingsRedundantFilter,
    LongContextReorder,
)
from langchain_openai import OpenAIEmbeddings
import os
from dotenv import load_dotenv

load_dotenv()

# ===== 1. 创建各个转换器 =====
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 步骤1: 去除冗余文档
redundant_filter = EmbeddingsRedundantFilter(embeddings=embeddings)

# 步骤2: 重排序（重要文档放首尾）
reorder = LongContextReorder()

# ===== 2. 组合成 Pipeline =====
pipeline = DocumentCompressorPipeline(
    transformers=[redundant_filter, reorder]
)

# ===== 3. 使用 Pipeline 处理文档 =====
docs = [
    Document(page_content="RAG 是检索增强生成技术"),
    Document(page_content="RAG 即检索增强生成，是一种 AI 技术"),  # 重复
    Document(page_content="向量数据库存储 Embedding 向量"),
    Document(page_content="LangChain 是流行的 LLM 框架"),
]

print(f"处理前: {len(docs)} 个文档")

# Pipeline 会依次执行: 去重 → 重排序
result = pipeline.compress_documents(docs, query="什么是 RAG？")

print(f"处理后: {len(result)} 个文档")
print("\n处理后的文档:")
for i, doc in enumerate(result, 1):
    print(f"  {i}. {doc.page_content}")
```

**运行输出：**
```
处理前: 4 个文档
处理后: 3 个文档

处理后的文档:
  1. LangChain 是流行的 LLM 框架
  2. 向量数据库存储 Embedding 向量
  3. RAG 是检索增强生成技术
```

**什么时候用？**
- 需要多步文档处理时（清洗 + 过滤 + 重排序）
- 构建生产级 RAG 管道时
- 需要与 ContextualCompressionRetriever 集成时

**与 ContextualCompressionRetriever 集成：**

```python
from langchain.retrievers import ContextualCompressionRetriever

# 将 Pipeline 集成到检索器中
compression_retriever = ContextualCompressionRetriever(
    base_compressor=pipeline,
    base_retriever=vectorstore.as_retriever(search_kwargs={"k": 10})
)

# 检索时自动执行: 检索 → 去重 → 重排序
results = compression_retriever.invoke("什么是 RAG？")
```

## 完整可运行示例

**场景：** 构建一个完整的文档转换管道——清洗 HTML → 去重 → 重排序

```python
"""
DocumentTransformer 完整示例：RAG 文档转换管道
演示：HTML 清洗 → 冗余过滤 → 长上下文重排序
"""

import os
from dotenv import load_dotenv
from langchain_core.documents import Document
from langchain_community.document_transformers import (
    Html2TextTransformer,
    EmbeddingsRedundantFilter,
    LongContextReorder,
)
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain_openai import OpenAIEmbeddings

# ===== 1. 环境配置 =====
load_dotenv()

# ===== 2. 模拟从网页抓取的原始文档 =====
print("=== 步骤1: 准备原始 HTML 文档 ===")
raw_docs = [
    Document(
        page_content="<h1>RAG 简介</h1><p>RAG 是<b>检索增强生成</b>技术。</p>",
        metadata={"source": "page1.html"}
    ),
    Document(
        page_content="<h1>RAG 概述</h1><p>RAG 即<b>检索增强生成</b>，是一种 AI 技术。</p>",
        metadata={"source": "page2.html"}
    ),
    Document(
        page_content="<h1>向量数据库</h1><p>向量数据库用于存储和检索 <em>Embedding 向量</em>。</p>",
        metadata={"source": "page3.html"}
    ),
    Document(
        page_content="<h1>LangChain</h1><p>LangChain 是构建 LLM 应用的<b>流行框架</b>。</p>",
        metadata={"source": "page4.html"}
    ),
    Document(
        page_content="<h1>Prompt 工程</h1><p>Prompt 工程是优化 LLM 输入的<b>关键技术</b>。</p>",
        metadata={"source": "page5.html"}
    ),
]
print(f"原始文档数量: {len(raw_docs)}")

# ===== 3. 步骤1: HTML 清洗 =====
print("\n=== 步骤2: HTML 清洗 ===")
html2text = Html2TextTransformer()
clean_docs = html2text.transform_documents(raw_docs)

for doc in clean_docs:
    print(f"  [{doc.metadata['source']}] {doc.page_content.strip()[:50]}")

# ===== 4. 步骤2: 去除冗余文档 =====
print("\n=== 步骤3: 去除冗余文档 ===")
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
redundant_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.95
)
filtered_docs = redundant_filter.transform_documents(clean_docs)
print(f"去重后文档数量: {len(filtered_docs)}（去除了 {len(clean_docs) - len(filtered_docs)} 个重复文档）")

# ===== 5. 步骤3: 长上下文重排序 =====
print("\n=== 步骤4: 长上下文重排序 ===")
reorder = LongContextReorder()
final_docs = reorder.transform_documents(filtered_docs)

print("最终文档顺序（重要文档在首尾）:")
for i, doc in enumerate(final_docs, 1):
    print(f"  位置{i}: {doc.page_content.strip()[:50]}")

# ===== 6. 总结 =====
print(f"\n=== 处理总结 ===")
print(f"原始文档: {len(raw_docs)} 个")
print(f"清洗后: {len(clean_docs)} 个")
print(f"去重后: {len(filtered_docs)} 个")
print(f"最终输出: {len(final_docs)} 个")
```

**运行输出：**
```
=== 步骤1: 准备原始 HTML 文档 ===
原始文档数量: 5

=== 步骤2: HTML 清洗 ===
  [page1.html] # RAG 简介  RAG 是**检索增强生成**技术。
  [page2.html] # RAG 概述  RAG 即**检索增强生成**，是一种 AI 技术。
  [page3.html] # 向量数据库  向量数据库用于存储和检索 _Embedding 向量_。
  [page4.html] # LangChain  LangChain 是构建 LLM 应用的**流行框架**。
  [page5.html] # Prompt 工程  Prompt 工程是优化 LLM 输入的**关键技术**。

=== 步骤3: 去除冗余文档 ===
去重后文档数量: 4（去除了 1 个重复文档）

=== 步骤4: 长上下文重排序 ===
最终文档顺序（重要文档在首尾）:
  位置1: # Prompt 工程  Prompt 工程是优化 LLM 输入的**关键技术**。
  位置2: # LangChain  LangChain 是构建 LLM 应用的**流行框架**。
  位置3: # 向量数据库  向量数据库用于存储和检索 _Embedding 向量_。
  位置4: # RAG 简介  RAG 是**检索增强生成**技术。

=== 处理总结 ===
原始文档: 5 个
清洗后: 5 个
去重后: 4 个
最终输出: 4 个
```

## 快速参考卡

### 最常用的代码模板

```python
# 1. HTML 清洗
from langchain_community.document_transformers import Html2TextTransformer
html2text = Html2TextTransformer()
clean_docs = html2text.transform_documents(html_docs)

# 2. 去重过滤
from langchain_community.document_transformers import EmbeddingsRedundantFilter
redundant_filter = EmbeddingsRedundantFilter(embeddings=embeddings)
filtered_docs = redundant_filter.transform_documents(docs)

# 3. 长上下文重排序
from langchain_community.document_transformers import LongContextReorder
reorder = LongContextReorder()
reordered_docs = reorder.transform_documents(docs)

# 4. 组合 Pipeline
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
pipeline = DocumentCompressorPipeline(transformers=[redundant_filter, reorder])
result = pipeline.compress_documents(docs, query="...")
```

### 转换器速查表

| 转换器 | 用途 | 输入→输出数量 | 需要 Embedding |
|--------|------|---------------|----------------|
| `Html2TextTransformer` | HTML 转纯文本 | N → N | 否 |
| `EmbeddingsRedundantFilter` | 去除重复文档 | N → ≤N | 是 |
| `EmbeddingsClusteringFilter` | 聚类选代表 | N → K | 是 |
| `LongContextReorder` | 重排序 | N → N | 否 |
| `OpenAIMetadataTagger` | 自动标注元数据 | N → N | 否（需要 LLM） |
| `BeautifulSoupTransformer` | HTML 提取 | N → N | 否 |

### 推荐管道顺序

```
原始文档 → 清洗(Html2Text) → 去重(RedundantFilter) → 重排序(LongContextReorder) → LLM
```

## 这些知识足以

掌握以上5点，你就能在 RAG 管道中有效处理文档转换了：

1. 理解 `transform_documents` 统一接口——所有转换器的使用方式一模一样
2. 用 `Html2TextTransformer` 清洗 HTML——RAG 管道的第一步
3. 用 `EmbeddingsRedundantFilter` 去重——减少噪音，提升质量
4. 用 `LongContextReorder` 重排序——零成本优化 LLM 注意力
