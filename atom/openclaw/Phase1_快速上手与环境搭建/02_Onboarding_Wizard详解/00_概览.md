## 核心知识点预览

### 1. Onboarding Wizard 是什么？

**交互式配置向导**，通过问答式流程自动完成 OpenClaw 的全部设置。

**核心价值**：
- 降低配置门槛（无需手动编辑 JSON5）
- 避免配置错误（自动验证输入）
- 一键安装守护进程（LaunchAgent/systemd）

### 2. 两种模式

| 模式 | 交互次数 | 适用场景 |
|------|---------|---------|
| **QuickStart** | 最少（3-5 个问题） | 本地开发、快速体验 |
| **Advanced** | 完整（10+ 个问题） | 生产部署、多机访问 |

### 3. 配置流程（7 步）

```
1. Model/Auth     → 选择模型提供商（Anthropic/OpenAI/Custom）
2. Workspace      → 设置 Agent 工作目录
3. Gateway        → 配置网关（端口/绑定/认证）
4. Channels       → 连接聊天渠道（WhatsApp/Telegram/Discord）
5. Daemon         → 安装守护进程（LaunchAgent/systemd）
6. Health Check   → 验证 Gateway 启动
7. Skills         → 安装推荐技能
```

### 4. 关键文件

```bash
~/.openclaw/openclaw.json          # 配置文件（向导生成）
~/.openclaw/workspace/             # 默认工作目录
~/Library/LaunchAgents/            # macOS 守护进程
~/.config/systemd/user/            # Linux 守护进程
```

## 实战场景

本知识点包含 5 个实战场景，覆盖常见使用场景：

### 场景 1：基础 CLI 向导使用
```bash
openclaw onboard
```
适合：首次安装，本地开发

### 场景 2：非交互式自动化配置
```bash
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY"
```
适合：CI/CD 部署，VPS 自动化

### 场景 3：自定义 Provider 配置
```bash
openclaw onboard --non-interactive \
  --auth-choice custom-api-key \
  --custom-base-url "https://llm.example.com/v1" \
  --custom-model-id "foo-large"
```
适合：使用自托管模型或第三方 API

### 场景 4：多 Agent 环境搭建
```bash
openclaw agents add work
openclaw agents add personal
```
适合：工作/个人分离，多用户环境

### 场景 5：Channel 集成实战
```bash
openclaw onboard
# 选择 WhatsApp/Telegram/Discord
```
适合：聊天渠道集成

## 常见问题速查

### Q1: 向导会覆盖已有配置吗？

**A**: 不会。向导会检测已有配置，提供"保留"、"更新"、"重置"三种选项。

### Q2: QuickStart 和 Advanced 有什么区别？

**A**: QuickStart 使用安全默认值减少交互，Advanced 暴露全部控制。功能完全相同。

### Q3: 如何跳过某些步骤？

**A**: 使用 `--skip-health`、`--no-install-daemon` 等参数。

### Q4: 非交互模式如何传递 API 密钥？

**A**: 通过 `--anthropic-api-key` 参数或 `ANTHROPIC_API_KEY` 环境变量。

### Q5: systemd 安装失败怎么办？

**A**: 查看 `temp/troubleshooting/systemd_issue_1818.md`，通常需要安装 `dbus-user-session` 并启用 lingering。
