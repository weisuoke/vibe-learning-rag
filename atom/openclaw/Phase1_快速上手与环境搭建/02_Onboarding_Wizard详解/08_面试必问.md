# 面试必问 - Onboarding Wizard 技术深度

**本文档涵盖 Onboarding Wizard 相关的常见面试问题，从设计理念到实现细节。**

---

## 基础概念题

### Q1: 什么是 Onboarding Wizard？它解决什么问题？

**答案**：

Onboarding Wizard 是 OpenClaw 的交互式配置向导，通过 CLI 问答式流程引导用户完成从零到可用的全部设置。

**解决的核心问题**：

1. **降低配置门槛**：避免手动编辑复杂的 JSON5 配置文件
2. **减少配置错误**：自动验证输入，防止语法错误和类型错误
3. **简化守护进程安装**：一键安装 LaunchAgent (macOS) 或 systemd (Linux)
4. **提供安全默认值**：Loopback 绑定 + Token 认证 + Pairing 策略

**类比**：就像 `rails new` 或 `create-react-app`，一条命令完成项目初始化。

---

### Q2: QuickStart 和 Advanced 模式有什么区别？

**答案**：

| 维度 | QuickStart | Advanced |
|------|-----------|----------|
| **交互次数** | 最少（3-5 个问题） | 完整（10+ 个问题） |
| **Gateway 绑定** | Loopback (127.0.0.1) | 可选 LAN/Tailscale/自定义 |
| **认证方式** | Token（自动生成） | Token 或 Password |
| **Tailscale** | 关闭 | 可选 Serve/Funnel |
| **适用场景** | 本地开发、快速体验 | 生产部署、多机访问 |

**设计理念**：渐进式披露（Progressive Disclosure）

- QuickStart：隐藏复杂选项，使用安全默认值
- Advanced：暴露全部控制，适合高级用户

**实现方式**：

```typescript
const flow: WizardFlow = await prompter.select({
  message: "Onboarding mode",
  options: [
    { value: "quickstart", label: "QuickStart" },
    { value: "advanced", label: "Manual" },
  ],
});

if (flow === "quickstart") {
  // 使用默认值，减少交互
  port = DEFAULT_GATEWAY_PORT;
  bind = "loopback";
  authMode = "token";
} else {
  // 完整交互
  port = await prompter.text({ message: "Gateway port" });
  bind = await prompter.select({ message: "Gateway bind", options: [...] });
  authMode = await prompter.select({ message: "Gateway auth", options: [...] });
}
```

### Q3: 向导配置哪些核心组件？

**答案**：

向导配置 7 个核心组件：

1. **Model/Auth** - 模型提供商和 API 密钥
   - Anthropic (推荐)
   - OpenAI
   - Custom Provider (OpenAI-compatible / Anthropic-compatible / Unknown)

2. **Workspace** - Agent 工作目录
   - 默认：`~/.openclaw/workspace`
   - 包含 Bootstrap 文件、会话数据、文档

3. **Gateway** - 网关服务器
   - Port（默认 18789）
   - Bind（Loopback / LAN / Tailnet / Custom）
   - Auth（Token / Password）
   - Tailscale（Off / Serve / Funnel）

4. **Channels** - 聊天渠道
   - WhatsApp（QR 扫码配对）
   - Telegram（Bot Token）
   - Discord（OAuth 授权）
   - 其他（Signal、BlueBubbles、iMessage 等）

5. **Daemon** - 守护进程
   - macOS：LaunchAgent (`~/Library/LaunchAgents/`)
   - Linux：systemd user service (`~/.config/systemd/user/`)

6. **Health Check** - 健康检查
   - 验证 Gateway 启动
   - 检查模型提供商连接
   - 确认 Workspace 存在

7. **Skills** - 技能安装
   - 推荐技能（web-search、file-ops、code-exec）
   - 依赖安装（通过 Homebrew 或手动）

## 设计理念题

### Q4: 为什么 QuickStart 模式也要生成 Token（即使是 Loopback）？

**答案**：

**原因**：

1. **安全默认值**：即使本地访问也应该有认证保护
2. **防止本地恶意软件**：本机可能有恶意软件尝试访问 Gateway
3. **多用户环境**：共享机器上防止其他用户访问
4. **未来扩展性**：后续可能需要远程访问，Token 已经准备好
5. **一致性**：所有模式使用相同的认证机制，简化代码逻辑

**设计原则**：Secure by Default（安全默认值）

**实现**：

```typescript
if (authMode === "token") {
  if (flow === "quickstart") {
    gatewayToken = quickstartGateway.token ?? randomToken();
  } else {
    const tokenInput = await prompter.text({
      message: "Gateway token (blank to generate)",
    });
    gatewayToken = normalizeGatewayTokenInput(tokenInput) || randomToken();
  }
}
```

### Q5: 为什么 Tailscale Funnel 强制使用 Password 认证？

**答案**：

**原因**：

Tailscale Funnel 将 Gateway 暴露到公网，Token 认证存在泄露风险：

1. **Token 在 URL 中可见**：
   ```
   http://example.com?token=oc_abc123
   ```
   - 浏览器历史记录
   - 服务器日志
   - 分析工具（Google Analytics）
   - 代理服务器日志

2. **Password 使用 Basic Auth**：
   ```
   Authorization: Basic base64(username:password)
   ```
   - 在 HTTP Header 中，不在 URL
   - 不会出现在浏览器历史
   - 不会被日志记录（大多数服务器）

**实现**：

```typescript
if (tailscaleMode === "funnel" && authMode !== "password") {
  await prompter.note("Tailscale funnel requires password auth.", "Note");
  authMode = "password";
}
```

**设计原则**：最小权限 + 防止泄露

---

### Q6: 为什么 Tailscale 模式强制 Loopback 绑定？

**答案**：

**原因**：

Tailscale Serve/Funnel 是反向代理，Gateway 应该只绑定到 Loopback：

```
Tailscale Serve/Funnel (反向代理)
    ↓
Gateway (127.0.0.1:18789)
    ↓
Agent
```

**防止的安全风险**：

1. **同时暴露 LAN + Tailnet**：
   ```
   Gateway 绑定到 0.0.0.0
       ↓
   LAN 用户可以直接访问（绕过 Tailscale 认证）
       ↓
   安全风险！
   ```

2. **正确配置**：
   ```
   Gateway 绑定到 127.0.0.1
       ↓
   只有 Tailscale 可以访问
       ↓
   所有访问都经过 Tailscale 认证
   ```

**实现**：

```typescript
if (tailscaleMode !== "off" && bind !== "loopback") {
  await prompter.note("Tailscale requires bind=loopback. Adjusting bind to loopback.", "Note");
  bind = "loopback";
  customBindHost = undefined;
}
```

**设计原则**：最小权限 + 单一入口

## 实现细节题

### Q7: 向导如何处理已有配置？

**答案**：

向导会检测已有配置，提供三种选项：

**检测逻辑**：

```typescript
const snapshot = await readConfigFileSnapshot();
let baseConfig: OpenClawConfig = snapshot.valid ? snapshot.config : {};

if (snapshot.exists) {
  await prompter.note(
    summarizeExistingConfig(baseConfig),
    "Existing config detected"
  );

  const action = await prompter.select({
    message: "Config handling",
    options: [
      { value: "keep", label: "Use existing values" },
      { value: "modify", label: "Update values" },
      { value: "reset", label: "Reset" },
    ],
  });

  if (action === "reset") {
    const resetScope = await prompter.select({
      message: "Reset scope",
      options: [
        { value: "config", label: "Config only" },
        { value: "config+creds+sessions", label: "Config + creds + sessions" },
        { value: "full", label: "Full reset (config + creds + sessions + workspace)" },
      ],
    });
    await handleReset(resetScope, workspaceDir, runtime);
    baseConfig = {};
  }
}
```

**Reset Scope 行为**：

| Scope | 删除内容 |
|-------|---------|
| `config` | `~/.openclaw/openclaw.json` |
| `config+creds+sessions` | 配置 + `~/.openclaw/creds/` + `~/.openclaw/sessions/` |
| `full` | 配置 + 凭证 + 会话 + Workspace |

**设计原则**：幂等性 + 可恢复性

### Q8: 向导如何生成和验证 Gateway Token？

**答案**：

**生成逻辑**：

```typescript
function randomToken(): string {
  const bytes = crypto.randomBytes(16);
  return `oc_${bytes.toString('hex')}`;
}

// 示例输出：oc_1a2b3c4d5e6f7g8h9i0j1k2l3m4n5o6p
```

**验证逻辑**：

```typescript
function normalizeGatewayTokenInput(input: string | symbol): string | undefined {
  if (typeof input !== "string") return undefined;
  const trimmed = input.trim();
  if (trimmed === "") return undefined;

  // 验证格式：oc_ 前缀 + 32 个十六进制字符
  if (!/^oc_[0-9a-f]{32}$/.test(trimmed)) {
    throw new Error("Invalid token format. Expected: oc_<32 hex chars>");
  }

  return trimmed;
}
```

**安全特性**：

1. **加密随机**：使用 `crypto.randomBytes`（不是 `Math.random`）
2. **足够长度**：16 字节 = 128 位熵
3. **格式验证**：防止用户输入弱 Token
4. **前缀标识**：`oc_` 前缀便于识别和日志过滤

### Q9: 向导如何安装守护进程？

**答案**：

**macOS (LaunchAgent)**：

```typescript
async function installLaunchAgent() {
  const plistPath = path.join(
    os.homedir(),
    "Library/LaunchAgents/ai.openclaw.gateway.plist"
  );

  const plistContent = `
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>ai.openclaw.gateway</string>
  <key>ProgramArguments</key>
  <array>
    <string>${cliPath}</string>
    <string>gateway</string>
    <string>run</string>
    <string>--port</string>
    <string>${port}</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>${logPath}/stdout.log</string>
  <key>StandardErrorPath</key>
  <string>${logPath}/stderr.log</string>
</dict>
</plist>
  `;

  await fs.writeFile(plistPath, plistContent);
  await exec("launchctl load " + plistPath);
}
```

**Linux (systemd)**：

```typescript
async function installSystemdService() {
  const servicePath = path.join(
    os.homedir(),
    ".config/systemd/user/openclaw-gateway.service"
  );

  const serviceContent = `
[Unit]
Description=OpenClaw Gateway
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=${cliPath} gateway run --port ${port}
Restart=always
RestartSec=5
Environment="OPENCLAW_GATEWAY_TOKEN=${token}"

[Install]
WantedBy=default.target
  `;

  await fs.writeFile(servicePath, serviceContent);
  await exec("systemctl --user daemon-reload");
  await exec("systemctl --user enable openclaw-gateway");
  await exec("systemctl --user start openclaw-gateway");
}
```

**关键步骤**：

1. 生成服务配置文件（plist / service unit）
2. 写入到标准位置
3. 加载服务（launchctl load / systemctl daemon-reload）
4. 启用开机自启（systemctl enable）
5. 启动服务（launchctl start / systemctl start）

### Q10: 向导如何验证 Gateway 启动？

**答案**：

**Health Check 流程**：

```typescript
async function waitForGatewayReachable(params: {
  url: string;
  token?: string;
  password?: string;
  timeoutMs: number;
}): Promise<{ ok: boolean; error?: string }> {
  const startTime = Date.now();

  while (Date.now() - startTime < params.timeoutMs) {
    try {
      const response = await fetch(`${params.url}/health`, {
        headers: {
          ...(params.token && { "Authorization": `Bearer ${params.token}` }),
          ...(params.password && { "Authorization": `Basic ${btoa(`:${params.password}`)}` }),
        },
      });

      if (response.ok) {
        return { ok: true };
      }
    } catch (err) {
      // Gateway 未启动，继续轮询
    }

    await sleep(1000); // 每秒轮询一次
  }

  return { ok: false, error: "Gateway did not become ready" };
}
```

**轮询策略**：

- 超时时间：12 秒（默认）
- 轮询间隔：1 秒
- 成功条件：`/health` 端点返回 200 OK

**错误处理**：

```typescript
const probe = await waitForGatewayReachable({
  url: localUrl,
  token: gatewayToken,
  timeoutMs: 12000,
});

if (!probe.ok) {
  await prompter.note(
    "Gateway did not become ready. Check logs: openclaw logs gateway",
    "Health check failed"
  );
  // 继续向导，不强制退出
}
```

## 故障排查题

### Q11: Linux 上 systemd 安装失败，如何排查？

**答案**：

**常见错误**：

```
Failed to connect to bus: $DBUS_SESSION_BUS_ADDRESS and $XDG_RUNTIME_DIR not defined
```

**排查步骤**：

1. **检查 dbus-user-session**：
   ```bash
   dpkg -l | grep dbus-user-session
   # 如果未安装：
   sudo apt install -y dbus-user-session
   ```

2. **检查 lingering**：
   ```bash
   loginctl show-user $USER | grep Linger
   # 应该输出：Linger=yes

   # 如果未启用：
   sudo loginctl enable-linger $USER
   ```

3. **检查环境变量**：
   ```bash
   echo $XDG_RUNTIME_DIR
   # 应该输出：/run/user/1000（或类似）

   echo $DBUS_SESSION_BUS_ADDRESS
   # 应该输出：unix:path=/run/user/1000/bus
   ```

4. **重新登录**（重要！）：
   ```bash
   exit
   # SSH 重新连接
   ```

5. **验证 systemd 可用**：
   ```bash
   systemctl --user status
   # 应该显示用户服务列表
   ```

**根本原因**：

- `su` 或 `sudo -iu` 不创建完整用户会话
- 缺少 D-Bus 会话总线
- 未启用 lingering（登出后会话被终止）

**解决方案**：

使用 `machinectl` 或 SSH 登录（创建完整会话）：

```bash
# 方式 1：machinectl
sudo machinectl shell username@.host

# 方式 2：SSH
ssh username@localhost
```

### Q12: macOS 上 Gateway "未就绪"，如何排查？

**答案**：

**常见原因**：

1. **CLI 不在 PATH**：
   ```bash
   which openclaw
   # 如果未找到，添加到 PATH：
   export PATH="/usr/local/bin:$PATH"
   ```

2. **Accessibility 权限未授予**：
   - System Settings → Privacy & Security → Accessibility
   - 启用 OpenClaw

3. **端口被占用**：
   ```bash
   lsof -i :18789
   # 如果被占用，杀死进程或使用其他端口
   ```

4. **LaunchAgent 配置错误**：
   ```bash
   cat ~/Library/LaunchAgents/ai.openclaw.gateway.plist
   # 检查 ProgramArguments 路径是否正确
   ```

**排查步骤**：

1. **手动启动 Gateway**（调试）：
   ```bash
   openclaw gateway run --foreground --log-level debug
   ```

2. **查看日志**：
   ```bash
   openclaw logs gateway
   # 或
   tail -f ~/Library/Logs/OpenClaw/gateway.log
   ```

3. **检查 LaunchAgent 状态**：
   ```bash
   launchctl list | grep openclaw
   # 应该显示：ai.openclaw.gateway
   ```

4. **重新安装 LaunchAgent**：
   ```bash
   openclaw gateway uninstall
   openclaw gateway install
   ```

```bash
openclaw configure
```

```bash
npm uninstall -g openclaw
```

**验证修复**：


**根本原因**：

向导流程中的 Bug，Model/Auth 步骤被意外跳过（已在 GitHub Issue #16134 报告）。

### Q15: 向导的配置热重载机制是如何实现的？

**答案**：

**热重载模式**：

```typescript
type ReloadMode = "hybrid" | "hot" | "restart" | "off";

const reloadConfig = {
  mode: "hybrid",  // 默认
  debounceMs: 300,
};
```

**模式说明**：

| 模式 | 行为 |
|------|------|
| `hybrid` | 热重载安全配置，自动重启关键配置 |
| `hot` | 仅热重载安全配置，关键配置需手动重启 |
| `restart` | 任何配置变更都重启 Gateway |
| `off` | 禁用文件监听，需手动重启 |

**实现**：

```typescript
async function watchConfigFile() {
  const watcher = fs.watch(configPath, { persistent: true });

  for await (const event of watcher) {
    if (event.eventType === "change") {
      await debounce(async () => {
        const newConfig = await readConfigFile();
        const changes = detectChanges(currentConfig, newConfig);

        if (needsRestart(changes)) {
          if (reloadMode === "hybrid") {
            await restartGateway();
          } else if (reloadMode === "hot") {
            logger.warn("Config change requires restart. Run: openclaw gateway restart");
          }
        } else {
          await applyHotReload(changes);
        }

        currentConfig = newConfig;
      }, reloadConfig.debounceMs);
    }
  }
}

function needsRestart(changes: ConfigChanges): boolean {
  const restartRequired = [
    "gateway.port",
    "gateway.bind",
    "gateway.auth",
    "gateway.tailscale",
    "discovery",
    "plugins",
  ];

  return changes.some(change =>
    restartRequired.some(path => change.path.startsWith(path))
  );
}
```

**热重载支持**：

- ✅ Channels
- ✅ Agent & Models
- ✅ Automation (hooks, cron)
- ✅ Sessions & Messages
- ✅ Tools & Media
- ❌ Gateway Server (port, bind, auth) - 需要重启
- ❌ Infrastructure (discovery, plugins) - 需要重启
