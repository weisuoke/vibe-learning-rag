# 第三方可观测性平台

## 1. 【30字核心】

**第三方可观测性平台是专门为 LLM 应用设计的监控工具，通过 CallbackHandler 集成，提供追踪、评估、调试和成本分析等功能。**

---

## 2. 【第一性原理】

### 什么是第一性原理？

**第一性原理**：回到事物最基本的真理，从源头思考问题

### 第三方可观测性平台的第一性原理

#### 1. 最基础的定义

**第三方可观测性平台 = 专门为 LLM 应用设计的监控和调试工具**

仅此而已！没有更基础的了。

#### 2. 为什么需要第三方可观测性平台？

**核心问题：如何在生产环境中监控和优化 LLM 应用？**

在 RAG 开发中，我们面临三个根本性挑战：

1. **黑盒问题**：LLM 调用、检索、工具执行都是不透明的
2. **成本控制**：Token 使用量、API 调用成本难以追踪
3. **质量保证**：如何评估 RAG 系统的输出质量

#### 3. 第三方可观测性平台的三层价值

##### 价值1：完整的追踪能力

**让 LLM 应用的执行过程变得透明**

```python
# 传统方式：无法追踪
result = chain.invoke("问题")

# 使用 Langfuse：完整追踪
from langfuse.callback import CallbackHandler

langfuse_handler = CallbackHandler()
result = chain.invoke("问题", config={"callbacks": [langfuse_handler]})
# 在 Langfuse Dashboard 中查看完整追踪
```

**在 RAG 中的应用**：追踪检索质量、LLM 生成过程、成本分析

##### 价值2：评估与优化

**提供自动化的质量评估**

```python
# Langfuse 自动评估
langfuse.score(
    trace_id=trace_id,
    name="retrieval_quality",
    value=0.85,
    comment="检索到相关文档"
)
```

**在 RAG 中的应用**：评估检索质量、生成质量、用户满意度

##### 价值3：成本与性能监控

**实时追踪成本和性能指标**

```python
# 自动追踪 Token 使用量和成本
# Langfuse 自动计算每次调用的成本
```

**在 RAG 中的应用**：优化 Token 使用、降低 API 成本、提升响应速度

## 3. 【核心概念】

### 核心概念1：Langfuse - 最流行的开源平台

**Langfuse 是最常用的开源 LLM 可观测性平台，提供追踪、评估、Prompt 管理和成本分析。**

#### 核心特性

| 特性 | 说明 |
|------|------|
| **追踪** | 完整的 LLM 调用追踪 |
| **评估** | 自动化质量评估 |
| **Prompt 管理** | Prompt 版本控制 |
| **成本分析** | Token 使用量和成本追踪 |
| **开源** | 可自托管 |

#### 基础集成

```python
from langfuse.callback import CallbackHandler
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

# 初始化 Langfuse
langfuse_handler = CallbackHandler(
    public_key="pk-lf-***",
    secret_key="sk-lf-***",
    host="https://cloud.langfuse.com"
)

# 创建链
prompt = ChatPromptTemplate.from_template("回答问题：{question}")
model = ChatOpenAI(model="gpt-4")
chain = prompt | model

# 使用 Langfuse 追踪
result = chain.invoke(
    {"question": "什么是 RAG？"},
    config={"callbacks": [langfuse_handler]}
)

# 在 Langfuse Dashboard 中查看追踪
```

#### Prompt 管理

```python
from langfuse import Langfuse

langfuse = Langfuse()

# 创建 Prompt
langfuse.create_prompt(
    name="rag_qa_prompt",
    prompt="基于以下上下文回答问题：\n\n{context}\n\n问题：{question}",
    is_active=True  # 设为生产版本
)

# 获取 Prompt（带缓存）
prompt = langfuse.get_prompt("rag_qa_prompt", cache_ttl_seconds=300)
```

#### 评估集成

```python
# 添加评分
langfuse.score(
    trace_id=trace_id,
    name="retrieval_quality",
    value=0.85,
    comment="检索到 3 个相关文档"
)

# 添加用户反馈
langfuse.score(
    trace_id=trace_id,
    name="user_feedback",
    value=1,  # 1 = 满意, 0 = 不满意
    comment="回答准确"
)
```

**在 RAG 中的应用：**

```python

# 追踪 RAG 执行

# 获取 trace_id
```

| **商业支持** | 官方技术支持 |

#### 基础集成

```python

# 配置环境变量

os.environ["LANGSMITH_API_KEY"] = "your_api_key"
os.environ["LANGSMITH_PROJECT"] = "my_rag_project"

# 自动追踪所有 LangChain 调用
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

prompt = ChatPromptTemplate.from_template("回答问题：{question}")

chain = prompt | model

# 自动追踪到 LangSmith

```

#### @traceable 装饰器

```python
from langsmith import traceable

@traceable(
    run_type="chain",
    tags=["rag", "production"],
    metadata={"version": "1.0"}
)
def rag_pipeline(question: str) -> str:
    # 检索
    docs = retriever.invoke(question)
    
    # 生成
    context = "\n\n".join([doc.page_content for doc in docs])
    response = llm.invoke(f"基于以下上下文回答：\n\n{context}\n\n问题：{question}")
    
    return response.content

# 自动追踪到 LangSmith
result = rag_pipeline("什么是向量数据库？")
```

```python

    model="gpt-4",
    messages=[{"role": "user", "content": "什么是 RAG？"}]
```

## 6. 【反直觉点】

### 误区1：必须选择一个平台 ❌

**为什么错？**
- 不同平台有不同的优势
- 可以同时使用多个平台
- 开发环境和生产环境可以使用不同平台

**为什么人们容易这样错？**
因为传统监控工具通常只选择一个，但 LLM 可观测性平台可以组合使用。

**正确理解：**

```python
# 开发环境：使用 Phoenix 快速调试
import phoenix as px
px.launch_app()
LangChainInstrumentor().instrument()

# 生产环境：使用 Langfuse 监控
from langfuse.callback import CallbackHandler
langfuse_handler = CallbackHandler()

# 同时使用两个平台
result = chain.invoke(
    input,
    config={"callbacks": [langfuse_handler]}  # 生产环境追踪
)
# Phoenix 自动追踪（开发环境）
```

**在 RAG 中的应用：**
- 开发阶段用 Phoenix 调试 Embedding 质量
- 生产环境用 Langfuse 监控成本和性能
- 敏感数据用 AgentReplay 本地存储

---

**为什么错？**
- Langfuse 功能非常完善
- 开源平台更新速度快
- 社区贡献丰富

**为什么人们容易这样错？**
因为传统观念认为"免费的不如付费的"，但开源 LLM 工具生态非常成熟。

**正确理解：**

**Langfuse（开源）功能对比：**

**社区推荐：**

### 误区3：可观测性平台会显著影响性能 ❌

**为什么错？**
- 追踪是异步的，不阻塞主流程
- 开销通常 < 5%
- 可以通过采样降低开销

**为什么人们容易这样错？**
因为"监控"通常意味着额外开销，但现代可观测性平台设计为低开销。

**正确理解：**

```python
# 性能测试
import time

# 不使用追踪
start = time.time()
result = chain.invoke(input)
print(f"无追踪耗时: {time.time() - start:.2f}秒")

# 使用 Langfuse 追踪
start = time.time()
result = chain.invoke(input, config={"callbacks": [langfuse_handler]})
print(f"有追踪耗时: {time.time() - start:.2f}秒")

# 性能差异通常 < 5%
```

**优化建议：**
- 生产环境使用采样（如 10% 的请求）
- 只追踪关键路径
- 使用异步上传

## 7. 【实战代码】

```python
"""
第三方可观测性平台实战示例
演示：Langfuse、LangSmith、Phoenix、AgentReplay 集成
"""

import os
import asyncio
from typing import List
from dotenv import load_dotenv

from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_community.vectorstores import Chroma
from langchain_core.runnables import RunnablePassthrough
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.document_loaders import TextLoader

# 加载环境变量
load_dotenv()

# ===== 1. Langfuse 集成 =====
print("=== Langfuse 集成 ===")

from langfuse.callback import CallbackHandler
from langfuse import Langfuse

# 初始化 Langfuse
langfuse = Langfuse(
    public_key=os.getenv("LANGFUSE_PUBLIC_KEY"),
    secret_key=os.getenv("LANGFUSE_SECRET_KEY"),
    host="https://cloud.langfuse.com"
)

langfuse_handler = CallbackHandler()

# 创建 RAG 链
loader = TextLoader("sample_docs.txt")
documents = loader.load()

text_splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = text_splitter.split_documents(documents)

embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

template = """基于以下上下文回答问题：

上下文：
{context}

问题：{question}

回答："""

prompt = ChatPromptTemplate.from_template(template)
model = ChatOpenAI(model="gpt-4", temperature=0)

rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | model
    | StrOutputParser()
)

# 使用 Langfuse 追踪
question = "什么是向量数据库？"
result = rag_chain.invoke(
    question,
    config={"callbacks": [langfuse_handler]}
)

print(f"回答: {result[:100]}...")

# 获取 trace_id
trace_id = langfuse_handler.get_trace_id()
print(f"Trace ID: {trace_id}")

# 添加评分
langfuse.score(
    trace_id=trace_id,
    name="retrieval_quality",
    value=0.85,
    comment="检索到相关文档"
)

print("✅ Langfuse 追踪完成，查看 Dashboard: https://cloud.langfuse.com")

# ===== 2. LangSmith 集成 =====
print("\n=== LangSmith 集成 ===")

# 配置环境变量
os.environ["LANGSMITH_TRACING"] = "true"
os.environ["LANGSMITH_API_KEY"] = os.getenv("LANGSMITH_API_KEY", "your_key")
os.environ["LANGSMITH_PROJECT"] = "rag_demo"

# 自动追踪（无需额外代码）
result = rag_chain.invoke(question)
print(f"回答: {result[:100]}...")
print("✅ LangSmith 自动追踪完成，查看 Dashboard: https://smith.langchain.com")

# 使用 @traceable 装饰器
from langsmith import traceable

@traceable(
    run_type="chain",
    tags=["rag", "demo"],
    metadata={"version": "1.0"}
)
def rag_pipeline(question: str) -> str:
    """RAG 管道"""
    docs = retriever.invoke(question)
    context = "\n\n".join([doc.page_content for doc in docs])
    response = model.invoke(f"基于以下上下文回答：\n\n{context}\n\n问题：{question}")
    return response.content

result = rag_pipeline(question)
print(f"@traceable 回答: {result[:100]}...")

# ===== 3. Phoenix 集成 =====
print("\n=== Phoenix 集成 ===")

import phoenix as px
from phoenix.trace.langchain import LangChainInstrumentor

# 启动 Phoenix（在后台运行）
session = px.launch_app()
print(f"Phoenix UI: http://localhost:6006")

# 自动追踪 LangChain
LangChainInstrumentor().instrument()

# 执行 RAG（自动追踪到 Phoenix）
result = rag_chain.invoke(question)
print(f"回答: {result[:100]}...")
print("✅ Phoenix 自动追踪完成，查看 UI: http://localhost:6006")

# ===== 4. AgentReplay 集成 =====
print("\n=== AgentReplay 集成 ===")

# 注意：需要先启动 AgentReplay 桌面应用
try:
    from agentreplay import AgentReplay
    
    # 初始化（自动连接本地应用）
    ar = AgentReplay()
    
    # 使用 AgentReplay 追踪
    result = rag_chain.invoke(
        question,
        config={"callbacks": [ar.get_callback_handler()]}
    )
    
    print(f"回答: {result[:100]}...")
    print("✅ AgentReplay 追踪完成，查看桌面应用")
except ImportError:
    print("⚠️ AgentReplay 未安装，跳过")

# ===== 5. 多平台同时使用 =====
print("\n=== 多平台同时使用 ===")

# 同时使用 Langfuse 和 LangSmith
os.environ["LANGSMITH_TRACING"] = "true"

result = rag_chain.invoke(
    question,
    config={"callbacks": [langfuse_handler]}  # Langfuse
)
# LangSmith 自动追踪

print(f"回答: {result[:100]}...")
print("✅ 同时追踪到 Langfuse 和 LangSmith")

# ===== 6. Prompt 管理（Langfuse）=====
print("\n=== Prompt 管理 ===")

# 创建 Prompt
langfuse.create_prompt(
    name="rag_qa_prompt",
    prompt=template,
    is_active=True
)

# 获取 Prompt（带缓存）
prompt_obj = langfuse.get_prompt("rag_qa_prompt", cache_ttl_seconds=300)
print(f"Prompt 版本: {prompt_obj.version}")

# ===== 7. 成本分析 =====
print("\n=== 成本分析 ===")

# Langfuse 自动追踪 Token 使用量
# 在 Dashboard 中查看成本报告

# 手动计算成本
def calculate_cost(trace_id: str):
    """计算单次调用成本"""
    # 从 Langfuse 获取 trace 数据
    # 这里简化为示例
    total_tokens = 1000  # 示例值
    cost_per_1k_tokens = 0.03  # GPT-4 价格
    cost = (total_tokens / 1000) * cost_per_1k_tokens
    return cost

cost = calculate_cost(trace_id)
print(f"估算成本: ${cost:.4f}")

# ===== 8. 评估集成 =====
print("\n=== 评估集成 ===")

# 自动评估检索质量
def evaluate_retrieval(question: str, docs: List) -> float:
    """评估检索质量"""
    # 简化的评估逻辑
    if len(docs) >= 3:
        return 0.9
    elif len(docs) >= 1:
        return 0.6
    else:
        return 0.3

# 执行并评估
docs = retriever.invoke(question)
score = evaluate_retrieval(question, docs)

# 记录到 Langfuse
langfuse.score(
    trace_id=trace_id,
    name="retrieval_quality",
    value=score,
    comment=f"检索到 {len(docs)} 个文档"
)

print(f"检索质量评分: {score}")

# ===== 9. 生产环境配置 =====
print("\n=== 生产环境配置 ===")

# 采样配置（只追踪 10% 的请求）
import random

def should_trace() -> bool:
    """是否追踪当前请求"""
    return random.random() < 0.1  # 10% 采样率

if should_trace():
    result = rag_chain.invoke(
        question,
        config={"callbacks": [langfuse_handler]}
    )
else:
    result = rag_chain.invoke(question)

print("✅ 生产环境采样追踪")

# ===== 10. 错误监控 =====
print("\n=== 错误监控 ===")

try:
    # 模拟错误
    result = rag_chain.invoke("这是一个会导致错误的问题" * 1000)
except Exception as e:
    # 记录错误到 Langfuse
    langfuse.score(
        trace_id=trace_id,
        name="error",
        value=0,
        comment=f"错误: {str(e)}"
    )
    print(f"❌ 错误已记录: {str(e)[:100]}...")

print("\n=== 实战示例完成 ===")
```

## 8. 【面试必问】

### 问题："如何选择合适的可观测性平台？"

**普通回答（❌ 不出彩）：**
"根据需求选择，Langfuse 是开源的，LangSmith 是官方的。"

**出彩回答（✅ 推荐）：**

> **选择可观测性平台需要考虑四个维度：**
>
> 1. **部署方式**：
>    - 云服务：LangSmith（官方托管）、Langfuse Cloud
>    - 自托管：Langfuse（开源）、Phoenix（本地）
>    - 本地优先：AgentReplay（桌面应用）
>
> 2. **成本考虑**：
>    - 免费：Phoenix、AgentReplay、Langfuse（自托管）
>    - 付费：LangSmith、Langfuse Cloud（企业版）
>    - 按使用量计费 vs 固定订阅
>
> 3. **功能需求**：
>    - 基础追踪：所有平台都支持
>    - Prompt 管理：Langfuse、LangSmith
>    - Embedding 监控：Phoenix（专长）
>    - 评估框架：Langfuse、LangSmith
>
> 4. **数据隐私**：
>    - 敏感数据：AgentReplay（完全本地）、Langfuse（自托管）
>    - 一般数据：LangSmith、Langfuse Cloud
>
> **在 RAG 开发中的实际选择**：
> - 开发阶段：Phoenix（快速调试）
> - 生产环境：Langfuse（开源 + 功能完善）
> - 敏感数据：AgentReplay（本地存储）
> - 快速上线：LangSmith（官方支持）
>
> **组合使用策略**：
> - 开发环境用 Phoenix 调试
> - 生产环境用 Langfuse 监控
> - 敏感项目用 AgentReplay 本地存储

1. ✅ 从多个维度系统分析（部署、成本、功能、隐私）
2. ✅ 提供具体的选择建议

## 9. 【化骨绵掌】

### 卡片1：可观测性平台的本质

**一句话：** 可观测性平台是专门为 LLM 应用设计的监控和调试工具

**举例：**
- Langfuse：开源、功能完善
- LangSmith：官方、深度集成
- Phoenix：轻量、专注调试
- AgentReplay：本地、隐私保护

**应用：** 在 RAG 开发中，根据场景选择合适的平台

---

### 卡片2：Langfuse 的核心优势

**一句话：** Langfuse 是最流行的开源平台，功能完善且可自托管

**举例：**
```python
from langfuse.callback import CallbackHandler
langfuse_handler = CallbackHandler()
result = chain.invoke(input, config={"callbacks": [langfuse_handler]})
```

**应用：** 生产环境首选，平衡功能和成本

---

### 卡片3：LangSmith 的原生集成

**一句话：** LangSmith 是 LangChain 官方平台，提供原生支持和深度集成

**举例：**
```python
os.environ["LANGSMITH_TRACING"] = "true"
# 自动追踪所有 LangChain 调用
```

**应用：** 快速上线项目，无需额外配置

---

### 卡片4：Phoenix 的轻量级调试

**一句话：** Phoenix 专注于 Embedding 和检索质量监控，适合开发调试

**举例：**
```python
import phoenix as px
px.launch_app()
LangChainInstrumentor().instrument()
```

**应用：** 开发阶段快速调试 RAG 检索质量

---

### 卡片5：AgentReplay 的本地优先

**一句话：** AgentReplay 是本地桌面应用，数据完全本地存储

**举例：**
```python
from agentreplay import AgentReplay
ar = AgentReplay()
result = chain.invoke(input, config={"callbacks": [ar.get_callback_handler()]})
```

**应用：** 敏感数据场景，保护隐私

---

### 卡片6：多平台组合使用

**一句话：** 可以同时使用多个平台，发挥各自优势

**举例：**
- 开发：Phoenix 调试
- 生产：Langfuse 监控
- 敏感：AgentReplay 本地

**应用：** 根据环境和需求灵活组合
