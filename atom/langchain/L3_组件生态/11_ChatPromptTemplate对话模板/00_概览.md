# ChatPromptTemplate对话模板 - 概览

> 本文档是 LangChain ChatPromptTemplate 的学习指南，帮助你掌握对话式 AI 的标准化提示词接口

---

## 知识点简介

**ChatPromptTemplate** 是 LangChain 中用于构建对话式 AI 提示词的核心组件。它不是简单的字符串模板，而是提供了多角色消息管理、对话历史占位符、多模态支持等高级特性，让你能够构建结构化、可维护的对话系统。

在 AI Agent 和 RAG 开发中，ChatPromptTemplate 是与 ChatModel 交互的标准接口。掌握 ChatPromptTemplate，能够让你：
- 构建多角色对话模板（System/Human/AI）
- 使用 MessagesPlaceholder 管理对话历史
- 实现 Few-shot 学习和示例驱动
- 支持多模态输入（文本+图片）
- 与 LCEL、Memory、Agent 等组件无缝集成
- 提升对话系统的开发效率和可维护性

**为什么需要学习 ChatPromptTemplate？**

基础的 PromptTemplate 只能生成字符串，但在实际的对话式 AI 开发中，你会遇到：
- 需要区分不同角色的消息（系统指令、用户输入、AI 回复）
- 需要动态注入对话历史以保持上下文
- 需要提供 Few-shot 示例来引导 LLM 行为
- 需要处理多模态输入（如图片描述任务）
- 需要与 ChatModel 的消息格式对接

这些场景都需要 ChatPromptTemplate 来解决。

## 常见问题（FAQ）

### Q1: ChatPromptTemplate 和 PromptTemplate 有什么区别？
**A:** ChatPromptTemplate 输出消息列表，支持多角色和对话历史；PromptTemplate 输出字符串，适合简单文本生成。对话式 AI 应该使用 ChatPromptTemplate。

### Q2: MessagesPlaceholder 和普通变量有什么区别？
**A:** MessagesPlaceholder 接受消息列表，支持 optional 和 n_messages 参数，自动转换元组为消息对象；普通变量只能接受单个值。

### Q3: ChatPromptTemplate 支持 partial_variables 吗？
**A:** 支持，但在某些版本中存在兼容性问题（参考 GitHub Issue #17560, #30049）。建议在使用前测试，或使用最新版本。

### Q4: 如何选择消息格式（元组、对象、模板）？
**A:** 简单场景使用元组格式 `("human", "Hello")`；需要变量替换使用模板格式；需要额外参数使用消息对象。

### Q5: 如何处理大规模对话历史？
**A:** 使用 MessagesPlaceholder 的 n_messages 参数限制消息数量，或使用 Memory 组件的摘要功能。
