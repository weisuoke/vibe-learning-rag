## 4.1 基础配置（5分钟上手）

### 核心步骤

```python
from pymilvus import MilvusClient, Function, DataType, FunctionType

# 步骤1：创建客户端
client = MilvusClient("http://localhost:19530")

# 步骤2：定义embedding函数
embedding_function = Function(
    name="my_embedding",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["text"],
    output_field_names=["vector"],
    params={
        "provider": "openai",
        "model_name": "text-embedding-3-small",
        "api_key": "sk-xxx",
        "dim": 512
    }
)

# 步骤3：创建schema
schema = client.create_schema()
schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
schema.add_field("text", DataType.VARCHAR, max_length=1000)
schema.add_field("vector", DataType.FLOAT_VECTOR, dim=512)
schema.add_function(embedding_function)

# 步骤4：创建collection
client.create_collection("my_collection", schema=schema)

# 步骤5：插入数据（自动embedding）
client.insert("my_collection", [
    {"text": "Milvus is a vector database"},
    {"text": "RAG combines retrieval and generation"}
])

# 步骤6：搜索（自动embedding查询）
results = client.search(
    "my_collection",
    data=["What is Milvus?"],
    anns_field="vector",
    limit=5
)
```

**这就是全部！** 5个步骤，不到20行代码，就能运行一个完整的RAG系统。

## 4.2 三个最常用的提供商

### 提供商1：OpenAI（推荐新手）

**为什么选择**：
- 最稳定可靠
- 文档最完善
- 社区支持最好

**配置示例**：
```python
params = {
    "provider": "openai",
    "model_name": "text-embedding-3-small",  # 或 "text-embedding-3-large"
    "api_key": "sk-xxx",
    "dim": 512  # 可选：降低维度节省成本
}
```

**成本**：
- text-embedding-3-small: $0.02/百万token
- text-embedding-3-large: $0.13/百万token

**批量大小**：128条/批

### 提供商2：VoyageAI（推荐生产环境）

**为什么选择**：
- 速度最快（~200ms延迟）
- 批量大（128条/批）
- RAG场景优化

**配置示例**：
```python
params = {
    "provider": "voyageai",
    "model_name": "voyage-3-large",  # 或 "voyage-3", "voyage-3-lite"
    "api_key": "pa-xxx",
    "dim": 1024  # 可选
}
```

**成本**：$0.12/百万token

**批量大小**：128条/批

### 提供商3：Cohere（推荐多语言）

**为什么选择**：
- 多语言支持好
- 支持Int8量化
- 截断策略灵活

**配置示例**：
```python
params = {

}
```

**成本**：$0.10/百万token

**批量大小**：96条/批

## 4.3 数据操作（插入和查询）

### 插入数据

```python
# 单条插入
client.insert("my_collection", [{"text": "Hello world"}])

# 批量插入
texts = [f"Document {i}" for i in range(1000)]
client.insert("my_collection", [{"text": t} for t in texts])
# Milvus自动按MaxBatch分批调用embedding API
```

### 查询数据

```python
# 相似度搜索
results = client.search(
    "my_collection",
    data=["search query"],
    anns_field="vector",
    limit=5
)

# 访问结果
for hit in results[0]:
    print(f"ID: {hit['id']}, Distance: {hit['distance']}")
```

### 混合查询（向量+标量过滤）

```python
# 添加标量字段
schema.add_field("category", DataType.VARCHAR, max_length=50)

# 插入带标量的数据
client.insert("my_collection", [
    {"text": "Python tutorial", "category": "programming"},
    {"text": "Cooking recipe", "category": "food"}
])

# 混合查询
results = client.search(
    "my_collection",
    data=["programming guide"],
    anns_field="vector",
    filter='category == "programming"',
    limit=5
)
```

## 4.4 常见错误处理

### 错误1：空字符串

**错误信息**：
```
MilvusException: empty string is not allowed
```

**原因**：所有提供商都拒绝空字符串

**解决方案**：
```python
# 过滤空字符串
data = [{"text": t} for t in texts if t.strip()]
client.insert("my_collection", data)
```

### 错误2：批量大小超限

**错误信息**：
```
MilvusException: batch size exceeds limit
```

**原因**：超过了提供商的MaxBatch限制

**解决方案**：
```python
# Milvus会自动分批，但如果手动分批：
max_batch = 128  # OpenAI/VoyageAI
for i in range(0, len(texts), max_batch):
    batch = texts[i:i+max_batch]
    client.insert("my_collection", [{"text": t} for t in batch])
```

### 错误3：维度不匹配

**错误信息**：
```
MilvusException: dimension mismatch
```

**原因**：schema中定义的维度与embedding输出维度不一致

**解决方案**：
```python
# 确保维度一致
params = {
    "provider": "openai",
    "model_name": "text-embedding-3-small",
    "dim": 512  # 与schema中的dim一致
}

schema.add_field("vector", DataType.FLOAT_VECTOR, dim=512)  # 必须匹配
```

### 错误4：API密钥无效

**错误信息**：
```
MilvusException: invalid api key
```

**解决方案**：
```python
# 方法1：直接传递
params = {"api_key": "sk-xxx"}

# 方法2：环境变量
import os
os.environ["OPENAI_API_KEY"] = "sk-xxx"
params = {}  # Milvus自动从环境变量读取

# 方法3：YAML配置（生产环境推荐）
# 在milvus.yaml中配置
```

## 4.5 性能优化基础

### 优化1：选择批量大的提供商

```python
# ✅ 好：批量大（128条/批）
providers = ["openai", "voyageai", "vertexai"]

# ❌ 差：批量小（1条/批）
providers = ["bedrock"]  # 避免用于大规模场景
```

**性能差异**：
- OpenAI (128条/批): 10000条数据 = 79次API调用
- Bedrock (1条/批): 10000条数据 = 10000次API调用
- **性能差距：126倍**

### 优化2：降低embedding维度

```python
# 原始维度：1536
params = {
    "provider": "openai",
    "model_name": "text-embedding-3-small",
    "dim": 1536
}

# 降低维度：512
params = {
    "provider": "openai",
    "model_name": "text-embedding-3-small",
    "dim": 512  # 降低67%
}
```

**收益**：
- 存储节省：67%
- 计算加速：~3倍
- 质量损失：<5%

### 优化3：使用较小的模型

```python
# ❌ 大模型：更贵，更慢
params = {
    "provider": "openai",
    "model_name": "text-embedding-3-large",  # 3072维，$0.13/M

    "provider": "openai",

```

**收益**：
- 成本降低：85%

- 质量损失：~10%（大多数场景可接受）

## 4.6 完整的最小可用示例

### 场景：构建一个简单的文档问答系统

```python
from pymilvus import MilvusClient, Function, DataType, FunctionType
import os

# ===== 1. 配置 =====
os.environ["OPENAI_API_KEY"] = "sk-xxx"

client = MilvusClient("http://localhost:19530")

# ===== 2. 定义embedding函数 =====
embedding_function = Function(
    name="openai_ef",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["text"],
    output_field_names=["vector"],
    params={
        "provider": "openai",
        "model_name": "text-embedding-3-small",
        "dim": 512
    }
)

# ===== 3. 创建collection =====
schema = client.create_schema()
schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
schema.add_field("text", DataType.VARCHAR, max_length=2000)
schema.add_field("source", DataType.VARCHAR, max_length=100)
schema.add_field("vector", DataType.FLOAT_VECTOR, dim=512)
schema.add_function(embedding_function)

client.create_collection("doc_qa", schema=schema)

# ===== 4. 插入文档 =====
documents = [
    {
        "text": "Milvus is an open-source vector database built for AI applications.",
        "source": "milvus_intro.txt"
    },
    {
        "text": "RAG combines retrieval and generation for better LLM responses.",
        "source": "rag_guide.txt"
    },
    {
        "text": "Embedding functions automate the text-to-vector conversion process.",
        "source": "embedding_guide.txt"
    }
]

client.insert("doc_qa", documents)
print(f"Inserted {len(documents)} documents")

# ===== 5. 查询 =====
def ask_question(question):
    results = client.search(
        "doc_qa",
        data=[question],
        anns_field="vector",
        limit=3,
        output_fields=["text", "source"]
    )

    print(f"\nQuestion: {question}")
    print("Top 3 relevant documents:")
    for i, hit in enumerate(results[0], 1):
        print(f"{i}. [Distance: {hit['distance']:.4f}] {hit['entity']['text']}")
        print(f"   Source: {hit['entity']['source']}")

# 测试查询
ask_question("What is Milvus?")
ask_question("How does RAG work?")
ask_question("What are embedding functions?")

# ===== 6. 清理 =====
# client.drop_collection("doc_qa")
```

**运行输出**：
```

Question: What is Milvus?
Top 3 relevant documents:
1. [Distance: 0.1234] Milvus is an open-source vector database built for AI applications.
   Source: milvus_intro.txt
2. [Distance: 0.4567] Embedding functions automate the text-to-vector conversion process.
   Source: embedding_guide.txt
3. [Distance: 0.7890] RAG combines retrieval and generation for better LLM responses.
   Source: rag_guide.txt

Question: How does RAG work?
Top 3 relevant documents:
1. [Distance: 0.0987] RAG combines retrieval and generation for better LLM responses.
   Source: rag_guide.txt
...
```
