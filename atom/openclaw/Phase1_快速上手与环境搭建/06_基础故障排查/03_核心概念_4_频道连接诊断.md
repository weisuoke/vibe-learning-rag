# OpenClaw 基础故障排查 - 核心概念 4：频道连接诊断

## 一、概念定义

**频道连接诊断是 OpenClaw 多通道系统的核心排查机制，通过状态探测、配对验证和消息流追踪，快速识别并解决 8+ 个通道的连接、认证和消息传递问题。**

## 二、核心价值

### 2.1 为什么需要频道连接诊断？

**问题场景**：
- 通道显示已连接但消息不流动
- 配对请求待处理
- 提及门控阻止消息
- 特定通道频繁断开
- 多通道状态不一致

**频道连接诊断的价值**：
- ✅ 快速定位通道问题
- ✅ 识别配对和权限问题
- ✅ 验证消息流完整性
- ✅ 支持 8+ 个通道类型
- ✅ 提供统一的诊断接口

**来源**: `reference/source_channels_troubleshooting.md`

### 4.2 配对机制

```typescript
// 配对机制实现
interface PairingRequest {
  channel: ChannelType;
  sender: string;
  timestamp: number;
  status: 'pending' | 'approved' | 'rejected';
}

class PairingManager {
  private requests: Map<string, PairingRequest> = new Map();

  async handleMessage(channel: ChannelType, sender: string, message: string): Promise<boolean> {
    // 1. 检查是否已配对
    const paired = await this.isPaired(channel, sender);
    if (paired) {
      return true;  // 允许消息通过
    }

    // 2. 创建配对请求
    const request: PairingRequest = {
      channel,
      sender,
      timestamp: Date.now(),
      status: 'pending'
    };
    this.requests.set(`${channel}:${sender}`, request);

    // 3. 记录日志
    logger.warn('Pairing request', { channel, sender });

    // 4. 阻止消息
    return false;
  }

  async approve(channel: ChannelType, sender: string): Promise<void> {
    const key = `${channel}:${sender}`;
    const request = this.requests.get(key);

    if (request) {
      request.status = 'approved';
      await this.savePairing(channel, sender);
      logger.info('Pairing approved', { channel, sender });
    }
  }
}
```

### 4.3 提及门控机制

```typescript
// 提及门控实现
interface MentionGating {
  enabled: boolean;
  botName: string;
  patterns: RegExp[];
}

function checkMentionRequired(
  config: MentionGating,
  message: string,
  isGroupChat: boolean
): boolean {
  // 1. 如果不是群组，不需要提及
  if (!isGroupChat) {
    return true;
  }

  // 2. 如果禁用提及门控，允许通过
  if (!config.enabled) {
    return true;
  }

  // 3. 检查是否提及机器人
  for (const pattern of config.patterns) {
    if (pattern.test(message)) {
      return true;  // 找到提及，允许通过
    }
  }

  // 4. 未提及，阻止消息
  logger.debug('Message dropped (mention required)', { message });
  return false;
}

// 使用示例
const mentionConfig: MentionGating = {
  enabled: true,
  botName: 'MyBot',
  patterns: [
    /@MyBot/i,
    /MyBot/i
  ]
};

const allowed = checkMentionRequired(
  mentionConfig,
  'Hello @MyBot, how are you?',
  true  // 群组消息
);
// allowed = true
```

## 七、最佳实践

### 7.1 通道监控脚本

```bash
#!/bin/bash
# channel-monitor.sh

# 监控所有通道状态

while true; do
  echo "=== Channel Status Check ==="
  date

  # 检查所有通道
  STATUS=$(openclaw channels status --probe 2>&1)
  echo "$STATUS"

  # 检查是否有失败的通道
  if echo "$STATUS" | grep -q "failed"; then
    echo "⚠️  Some channels failed!"

    # 尝试重连
    FAILED_CHANNELS=$(echo "$STATUS" | grep "failed" | awk '{print $1}' | tr -d ':')
    for channel in $FAILED_CHANNELS; do
      echo "Reconnecting $channel..."
      openclaw channels reconnect "$channel"
    done
  fi

  # 检查配对请求
  for channel in telegram whatsapp discord; do
    PENDING=$(openclaw pairing list "$channel" 2>&1 | grep -c "Pending")
    if [ "$PENDING" -gt 0 ]; then
      echo "⚠️  $channel has $PENDING pending pairing requests"
    fi
  done

  sleep 60
done
```

### 7.2 自动配对批准

```typescript
// auto-pairing.ts
import { spawn } from 'child_process';

interface PairingConfig {
  autoApprove: boolean;
  whitelist?: string[];
  blacklist?: string[];
}

class AutoPairingManager {
  private config: PairingConfig;

  constructor(config: PairingConfig) {
    this.config = config;
  }

  async start(): Promise<void> {
    // 监听日志中的配对请求
    const logs = spawn('openclaw', ['logs', '--follow']);

    logs.stdout.on('data', (data) => {
      const lines = data.toString().split('\n');

      for (const line of lines) {
        // 识别配对请求
        const match = line.match(/pairing request.*channel=(\w+).*sender=(\w+)/);
        if (match) {
          const [, channel, sender] = match;
          this.handlePairingRequest(channel, sender);
        }
      }
    });
  }

  private async handlePairingRequest(channel: string, sender: string): Promise<void> {
    // 检查是否自动批准
    if (!this.config.autoApprove) {
      console.log(`Pairing request from ${sender} on ${channel} (manual approval required)`);
      return;
    }

    // 检查黑名单
    if (this.config.blacklist?.includes(sender)) {
      console.log(`Rejecting ${sender} (blacklisted)`);
      await this.runCommand('openclaw', ['pairing', 'reject', channel, sender]);
      return;
    }

    // 检查白名单
    if (this.config.whitelist && !this.config.whitelist.includes(sender)) {
      console.log(`Rejecting ${sender} (not in whitelist)`);
      return;
    }

    // 自动批准
    console.log(`Auto-approving ${sender} on ${channel}`);
    await this.runCommand('openclaw', ['pairing', 'approve', channel, sender]);
  }

  private runCommand(cmd: string, args: string[]): Promise<void> {
    return new Promise((resolve, reject) => {
      const proc = spawn(cmd, args);
      proc.on('close', (code) => {
        if (code === 0) {
          resolve();
        } else {
          reject(new Error(`Command failed with code ${code}`));
        }
      });
    });
  }
}

// 使用
const manager = new AutoPairingManager({
  autoApprove: true,
  whitelist: ['user123', 'user456']
});

manager.start();
```
