# 一句话总结

**多向量检索通过在一个 Collection 中定义多个向量字段（稠密/稀疏），使用 AnnSearchRequest 定义检索请求，用 RRFRanker 或 WeightedRanker 融合结果，实现多模态检索、BM25+语义混合检索，兼顾精确匹配和语义理解。**

---

## 核心要点

### 1. 多向量检索的本质

**Multi-Vector Hybrid Search = 多个向量字段 + 多个 ANN 检索 + 结果融合**

- 一个 Collection 可以包含多个向量字段
- 每个向量字段独立进行 ANN 检索
- 使用 Ranker 融合多个检索结果
- 支持不同类型和维度的向量字段共存

### 2. 核心 API 组件

**三大核心组件**：

- **AnnSearchRequest**：定义单个向量字段的检索请求
  - 指定查询向量数据（data）
  - 指定向量字段名（anns_field）
  - 配置检索参数（metric_type、limit）

- **RRFRanker**：Reciprocal Rank Fusion 排序器
  - 算法：`score = 1 / (k + rank)`
  - 默认 k=60（可配置）
  - 适合不确定权重的场景

- **WeightedRanker**：加权排序器
  - 为每个检索请求指定权重
  - 权重可以是任意正数
  - 适合已知权重的场景

### 3. RRF 算法原理

**Reciprocal Rank Fusion（倒数排名融合）**：

```
score = 1 / (k + rank)
```

- **k 参数**：默认 60，控制排名衰减速度
- **rank**：结果在单个检索中的排名（从 1 开始）
- **融合方式**：对每个结果的多个 score 求和
- **优势**：自动平衡多个检索结果，无需手动调权重

### 4. 索引创建要求

**关键约束**：

- **所有向量字段都必须创建索引**
- 不同类型的向量字段使用不同的索引类型
  - FloatVector → HNSW、IVF_FLAT
  - SparseFloatVector → SPARSE_INVERTED_INDEX
  - BinaryVector → BIN_IVF_FLAT
- 索引创建完成后才能加载 Collection
- 部分索引会导致 LoadCollection 失败

### 5. 应用场景

| 场景 | 向量组合 | 融合策略 |
|------|----------|----------|
| **多模态检索** | 图片向量 + 文本向量 | RRFRanker |
| **全文搜索增强** | BM25 稀疏向量 + 稠密向量 | RRFRanker |
| **多语言检索** | 中文向量 + 英文向量 | WeightedRanker |
| **多粒度检索** | 标题向量 + 内容向量 | WeightedRanker |

### 6. BM25 混合检索

**BM25 + 稠密向量 = 精确匹配 + 语义理解**：

```python
# BM25 函数自动生成稀疏向量
bm25_function = Function(
    name="sparse_vector",
    function_type=FunctionType.BM25,
    input_field_names=["text"],
    output_field_names="sparse_vector"
)

# 混合检索
sparse_req = AnnSearchRequest([query], "sparse_vector", {"metric_type": "BM25"}, limit=5)
dense_req = AnnSearchRequest([query_embedding], "dense_vector", {"metric_type": "IP"}, limit=5)
results = client.hybrid_search(collection_name, [sparse_req, dense_req], ranker=RRFRanker(), limit=5)
```

### 7. 核心操作流程

```python
from pymilvus import AnnSearchRequest, RRFRanker, WeightedRanker

# 1. 定义多个向量字段
schema.add_field("dense_vector", DataType.FLOAT_VECTOR, dim=768)
schema.add_field("sparse_vector", DataType.SPARSE_FLOAT_VECTOR)

# 2. 为所有向量字段创建索引
collection.create_index("dense_vector", {"index_type": "HNSW", "metric_type": "IP"})
collection.create_index("sparse_vector", {"index_type": "SPARSE_INVERTED_INDEX", "metric_type": "IP"})

# 3. 定义检索请求
dense_req = AnnSearchRequest([query_dense], "dense_vector", {"metric_type": "IP"}, limit=10)
sparse_req = AnnSearchRequest([query_sparse], "sparse_vector", {"metric_type": "IP"}, limit=10)

# 4. 执行混合检索（使用 RRF）
results = collection.hybrid_search([dense_req, sparse_req], ranker=RRFRanker(), limit=10)

# 或使用加权融合
results = collection.hybrid_search([dense_req, sparse_req], ranker=WeightedRanker(1.0, 0.7), limit=10)
```

### 8. Ranker 选择策略

| 场景 | 推荐 Ranker | 理由 |
|------|-------------|------|
| 不确定权重 | RRFRanker | 自动平衡，无需调参 |
| 已知权重 | WeightedRanker | 精确控制各字段重要性 |
| 多模态检索 | RRFRanker | 不同模态难以确定权重 |
| BM25+稠密 | RRFRanker | 官方推荐，效果稳定 |
| 业务场景明确 | WeightedRanker | 根据业务需求调整权重 |
