# 最小可用

> 从克隆到首次运行的最小步骤

---

## 核心目标

**最小可用**：用最少的步骤，让 OpenClaw 源码编译并运行起来。

**时间目标**：15-20 分钟完成首次构建和运行。

**前置要求**：
- Node.js 22+
- pnpm 10.23.0+
- Git
- 8GB+ RAM
- 10GB+ 磁盘空间

---

## 最小故障排查

### 问题 1：依赖安装失败

**症状**：
```
ERR_PNPM_NO_MATCHING_VERSION
```

**解决方案**：
```bash
# 清理缓存
pnpm store prune

# 重新安装
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

---

### 问题 2：构建失败

**症状**：
```
Error: Cannot find module 'tsdown'
```

**解决方案**：
```bash
# 重新安装依赖
pnpm install --force

# 重新构建
pnpm build
```

---

### 问题 3：Gateway 启动失败

**症状**：
```
Error: Port 18789 is already in use
```

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :18789

# 杀死进程
kill -9 <PID>

# 或使用其他端口
OPENCLAW_PORT=18790 pnpm gateway:dev
```

---

### 问题 4：原生依赖编译失败

**症状**：
```
gyp ERR! build error
gyp ERR! stack Error: `make` failed with exit code: 2
```

**解决方案 (macOS)**：
```bash
# 安装 Xcode Command Line Tools
xcode-select --install

# 重新安装依赖
pnpm install
```

**解决方案 (Linux)**：
```bash
# 安装构建工具
sudo apt-get install build-essential python3

# 重新安装依赖
pnpm install
```

**解决方案 (Windows)**：
```bash
# 安装 Visual Studio Build Tools
# https://visualstudio.microsoft.com/downloads/

# 重新安装依赖
pnpm install
```

## 核心洞察

### 1. 最小可用不是简化

**表面**：减少步骤

**实际**：
- 保留必要步骤
- 移除可选步骤
- 优化执行顺序
- 提供故障排查

---

### 2. 依赖安装是关键

**表面**：运行 `pnpm install`

**实际**：
- 9 个原生依赖需要编译
- 需要正确的编译工具链
- 可能需要 10-15 分钟
- 是最容易出错的环节

---

### 3. 构建流程很复杂

**表面**：运行 `pnpm build`

**实际**：
- 9 步构建流程
- 8 个入口点
- 资源复制和元数据生成
- 需要 2-3 分钟

---

## 一句话总结

**OpenClaw 的最小可用流程是：克隆 → 安装依赖（15分钟）→ 构建（2分钟）→ 启动 Gateway，总计约 20 分钟。**
