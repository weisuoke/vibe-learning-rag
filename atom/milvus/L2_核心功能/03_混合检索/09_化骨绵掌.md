## 卡片1：混合检索基础概念（2分钟）

### 核心定义

**混合检索 = 向量检索 + BM25检索 + RRF重排序**

```
用户查询
    ↓
┌─────────────────────────────┐
│  并行检索                    │
│  ├─ 稠密向量检索（语义）     │
│  └─ 稀疏向量检索（关键词）   │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  RRF重排序                   │
│  score = Σ(1 / (k + rank))  │
└─────────────────────────────┘
    ↓
TopK结果
```

### 为什么需要混合检索？

**单一检索的局限：**
- 纯向量：语义相似 ≠ 精确匹配
- 纯BM25：关键词匹配 ≠ 语义理解

**混合检索的优势：**
- Recall提升8%
- Precision提升10%
- 兼顾语义和关键词

### 快速记忆

**类比：** 混合检索 = 双眼视觉
- 左眼（向量）：看整体轮廓
- 右眼（BM25）：看细节特征
- 大脑（RRF）：融合两个视角

## 卡片2：稠密向量检索原理（2分钟）

### 核心概念

**稠密向量：** 每个维度都有值的连续向量

```python
# 示例：384维稠密向量
dense_vector = [0.23, -0.45, 0.67, ..., 0.89]  # 384个浮点数
```

### 生成方式

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
text = "Python机器学习库"
vector = model.encode(text)  # 384维向量
```

### 相似度度量

**COSINE（推荐用于文本）：**
```
similarity = Σ(xi×yi) / (‖x‖×‖y‖)
值域：[-1, 1]，越大越相似
```

### 检索流程

```python
dense_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20
)
```

### 快速记忆

**类比：** 稠密向量 = 照片压缩
- 原始文本 = 高清照片
- 向量 = 压缩后的缩略图
- 相似度 = 图片相似度

## 卡片3：稀疏向量检索（BM25）（2分钟）

### 核心概念

**稀疏向量：** 大部分维度为0，只存储非零维度

```python
# 示例：稀疏向量（字典格式）
sparse_vector = {
    27: 0.5,      # 第27维：权重0.5
    100: 0.3,     # 第100维：权重0.3
    5369: 0.6     # 第5369维：权重0.6
}
```

### BM25算法

**公式：**
```
BM25(D, Q) = Σ IDF(qi) × (f(qi, D) × (k1 + 1)) / (f(qi, D) + k1 × (1 - b + b × |D| / avgdl))
```

**直觉理解：**
- 词频越高，分数越高（但有饱和效应）
- 文档越短，分数越高（长度惩罚）
- 词越稀有（IDF高），分数越高

### Milvus内置BM25

```python
# 定义BM25函数（自动生成稀疏向量）
bm25_function = Function(
    name="text_bm25_emb",
    input_field_names=["text"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)
schema.add_function(bm25_function)
```

### 快速记忆

**类比：** BM25 = 关键词索引
- 文本 = 书籍内容
- 稀疏向量 = 书籍索引（只记录关键词页码）
- BM25分数 = 关键词匹配度

## 卡片4：RRF重排序算法（2分钟）

### 核心公式

```
score_rrf(d) = Σ(1 / (k + rank_i(d)))
```

**参数：**
- `d`：文档
- `k`：平滑参数（默认60）
- `rank_i(d)`：文档d在第i个检索结果中的排名

### 计算示例

```python
# 向量检索结果
vector_results = {"doc1": rank=1, "doc2": rank=2, "doc3": rank=3}

# BM25检索结果
bm25_results = {"doc2": rank=1, "doc1": rank=2, "doc4": rank=3}

# RRF计算（k=60）
doc1_score = 1/(60+1) + 1/(60+2) = 0.0325
doc2_score = 1/(60+2) + 1/(60+1) = 0.0325
doc3_score = 1/(60+3) + 0 = 0.0159
doc4_score = 0 + 1/(60+3) = 0.0159

# 最终排序：doc1 = doc2 > doc3 = doc4
```

### 为什么RRF有效？

1. **排名位置比分数更可靠**
   - 不同检索方式的分数量纲不同
   - 排名位置是相对的，更公平

2. **民主化融合**
   - 每个检索方式的贡献相等
   - 避免某个检索方式主导结果

### 快速记忆

**类比：** RRF = 民主投票
- 每个检索方式 = 一个评委
- 排名位置 = 评委的投票
- RRF分数 = 综合得分

## 卡片5：标量过滤机制（2分钟）

### 核心概念

**标量过滤：** 在向量检索基础上添加元数据条件

```python
# 过滤表达式
filter = "category == 'electronics' AND price > 100 AND stock > 0"
```

### 支持的运算符

**比较运算符：**
```python
==, !=, >, <, >=, <=, IN, LIKE
```

**逻辑运算符：**
```python
AND, OR, NOT
```

**JSON字段过滤：**
```python
filter = 'metadata["wireless"] == true'
```

**数组字段过滤：**
```python
filter = "ARRAY_CONTAINS(tags, 'wireless')"
filter = "ARRAY_CONTAINS_ALL(tags, ['wireless', 'premium'])"
filter = "ARRAY_CONTAINS_ANY(tags, ['5G', 'premium'])"
```

### 应用示例

```python
# 电商商品搜索
filter = """
    category == 'audio' AND
    price >= 200 AND price <= 500 AND
    rating > 4.5 AND
    ARRAY_CONTAINS(tags, 'wireless')
"""

dense_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20,
    expr=filter  # 标量过滤
)
```

### 快速记忆

**类比：** 标量过滤 = 筛选条件
- 向量检索 = 找相似的商品
- 标量过滤 = 按价格、类别、评分筛选
- 结果 = 既相似又符合条件的商品

## 卡片6：Pre-filter vs Post-filter（2分钟）

### 核心概念

**选择性 = 过滤后文档数 / 总文档数**

### 决策规则

```
选择性 < 20%？
├─ 是 → Pre-filter（先过滤，再检索）
└─ 否 → 选择性 > 50%？
    ├─ 是 → Post-filter（先检索，再过滤）
    └─ 否 → 两者性能相近，优先Pre-filter
```

### 性能对比

| 选择性 | Pre-filter延迟 | Post-filter延迟 | 推荐策略 |
|-------|---------------|----------------|---------|
| 5% | 15ms | 50ms | Pre-filter（3.3x快） |
| 20% | 35ms | 55ms | Pre-filter（1.6x快） |
| 50% | 65ms | 60ms | Post-filter（1.1x快） |
| 80% | 85ms | 50ms | Post-filter（1.7x快） |

### 代码示例

**Pre-filter（高选择性）：**
```python
dense_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20,
    expr="category == 'rare_category'"  # 5%选择性
)
```

**Post-filter（低选择性）：**
```python
dense_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=100  # 召回更多候选
)
# 应用层过滤
filtered = [hit for hit in results if hit.entity.get("price") < 1000][:20]
```

### 快速记忆

**类比：** Pre-filter vs Post-filter = 筛选时机
- Pre-filter = 先筛选简历，再面试（候选人少）
- Post-filter = 先面试，再筛选（候选人多）
- 选择性 = 候选人比例

## 卡片7：多向量检索（2分钟）

### 核心概念

**多向量检索：** 一个文档有多个向量表示

```python
# Schema设计
schema.add_field(field_name="title_vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="description_vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="image_vector", datatype=DataType.FLOAT_VECTOR, dim=512)
schema.add_field(field_name="sparse_vector", datatype=DataType.SPARSE_FLOAT_VECTOR)
```

### 应用场景

1. **文本+图像**：产品描述 + 产品图片
2. **标题+正文**：文章标题 + 文章正文
3. **稠密+稀疏**：语义向量 + 关键词向量

### 重排序策略

**RRF（等权重）：**
```python
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, image_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=10
)
```

**Weighted Ranker（可控权重）：**
```python
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, image_req, sparse_req],
    ranker=WeightedRanker(weights=[0.4, 0.3, 0.2, 0.1]),
    limit=10
)
```

### 快速记忆

**类比：** 多向量检索 = 多维度评价
- 标题向量 = 第一印象

## 卡片8：性能优化策略（2分钟）

### 索引选择

**HNSW（推荐）：**
```python
index_params.add_index(
    field_name="dense_vector",
    index_type="HNSW",
    metric_type="COSINE",
    params={"M": 16, "efConstruction": 200}
)
search_params = {"ef": 64}
```

**适用场景：**
- 10M-100M向量
- 高召回率、低延迟
- 内存充足

### 标量索引

```python
# 字符串字段：倒排索引
index_params.add_index(field_name="category", index_type="INVERTED")

# 数值字段：排序索引
index_params.add_index(field_name="price", index_type="STL_SORT")
```

### 批量检索

```python
# 批量查询（提升吞吐量）
query_vectors = [vector1, vector2, vector3, ...]
results = client.search(
    collection_name="documents",
    data=query_vectors,  # 多个查询向量
    anns_field="dense_vector",
    limit=10
)
```

### 优化建议

1. **创建标量索引**：为频繁过滤的字段创建索引
2. **选择性高的条件放前面**：先过滤选择性高的条件
3. **使用IN代替多个OR**：`category IN ['a', 'b']` 比 `category == 'a' OR category == 'b'` 快
4. **避免复杂计算**：预计算字段，避免运行时计算

### 快速记忆

**类比：** 性能优化 = 高速公路系统
- HNSW索引 = 高速公路（快速到达）
- 标量索引 = 出口标识（快速筛选）
- 批量检索 = 拼车（提升吞吐量）

## 卡片9：RAG应用场景（2分钟）

### 核心流程

```
用户问题
    ↓
┌─────────────────────────────┐
│  1. 生成查询向量             │
│  query_vector = model.encode(query) │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  2. 混合检索                 │
│  - 稠密向量检索（语义）      │
│  - 稀疏向量检索（关键词）    │
│  - RRF重排序                 │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  3. 提取上下文               │
│  context = "\n\n".join([hit.entity.get("content") for hit in results]) │
└─────────────────────────────┘
    ↓
┌─────────────────────────────┐
│  4. LLM生成答案              │
│  answer = llm.generate(query, context) │
└─────────────────────────────┘
    ↓
最终答案
```

### 代码示例

```python
class RAGSystem:
    def ask(self, query: str, top_k: int = 5) -> Dict:
        # 1. 生成查询向量
        query_vector = self.model.encode(query)
        
        # 2. 混合检索
        dense_req = AnnSearchRequest(
            data=[query_vector],
            anns_field="dense_vector",
            param={"metric_type": "COSINE", "params": {"ef": 64}},
            limit=20
        )
        sparse_req = AnnSearchRequest(
            data=[query],
            anns_field="sparse_vector",
            param={"metric_type": "BM25"},
            limit=20
        )
        results = self.client.hybrid_search(
            collection_name="knowledge_base",
            reqs=[dense_req, sparse_req],
            ranker=RRFRanker(k=60),
            limit=top_k
        )
        
        # 3. 提取上下文
        context = "\n\n".join([hit.entity.get("content") for hits in results for hit in hits])
        
        # 4. LLM生成答案
        answer = self.llm.generate(query, context)
        
        return {"query": query, "answer": answer, "sources": results}
```

### 快速记忆

**类比：** RAG = 开卷考试
- 混合检索 = 快速翻书找相关内容
- 上下文 = 找到的相关段落
- LLM = 根据段落回答问题

## 卡片10：生产级实践（2分钟）

### 核心组件

```python
class ProductionHybridSearch:
    def __init__(self, config: SearchConfig):
        self.client = MilvusClient(uri=config.milvus_uri)
        self.model = SentenceTransformer(config.model_name)
        self.config = config
        self.cache = {}  # LRU缓存
        
    def _retry_on_failure(self, func, *args, **kwargs):
        """重试机制"""
        for attempt in range(self.config.retry_times):
            try:
                return func(*args, **kwargs)
            except MilvusException as e:
                if attempt < self.config.retry_times - 1:
                    time.sleep(self.config.retry_delay * (attempt + 1))
                else:
                    raise
```

### 关键特性

1. **错误处理**：
   ```python
   try:
       results = self.hybrid_search(query)
   except MilvusException as e:
       logger.error(f"Search failed: {e}")
       return {"error": str(e)}
   ```

2. **重试机制**：
   ```python
   @retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=10))
   def hybrid_search(self, query):
       # 检索逻辑
   ```

3. **缓存策略**：
   ```python
   @lru_cache(maxsize=100)
   def get_embedding(self, text: str):
       return self.model.encode(text)
   ```

4. **监控指标**：
   ```python
   # Prometheus metrics
   search_requests_total = Counter('search_requests_total', 'Total search requests')
   search_latency = Histogram('search_latency_seconds', 'Search latency')
   search_errors_total = Counter('search_errors_total', 'Total search errors')
   ```

5. **健康检查**：
   ```python
   @app.get("/health")
   def health_check():
       try:
           client.get_server_version()
           return {"status": "healthy"}
       except Exception as e:
           return {"status": "unhealthy", "error": str(e)}
   ```

### 快速记忆

**类比：** 生产级系统 = 商业航班
- 错误处理 = 应急预案
- 重试机制 = 备用方案
- 缓存策略 = 快速通道
