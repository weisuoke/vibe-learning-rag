# 实战代码 - 场景2：自定义验证器实战

## 场景说明

本场景演示 Pydantic 的两种自定义验证器在 LangGraph 中的组合使用：

1. `@field_validator`：字段级验证——对单个字段做清洗、转换、校验
2. `@model_validator`：模型级验证——跨字段联合校验，处理字段间的依赖关系

场景1 中的 `Field` 约束只能做简单的范围/格式检查。当你需要「清洗输入」「字段间联动」「复杂业务规则」时，就需要自定义验证器。

[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]
[来源: reference/context7_pydantic_01.md | Pydantic 官方文档]

## 完整代码

```python
"""
LangGraph 状态验证实战 - 场景2：自定义验证器
演示：field_validator + model_validator 组合使用

运行环境：Python 3.13+, langgraph, pydantic
安装依赖：uv add langgraph pydantic
"""

from pydantic import BaseModel, Field, field_validator, model_validator, ValidationError
from typing_extensions import Self
from langgraph.graph import StateGraph, START, END


# ============================================================
# 1. 定义带自定义验证器的状态
# ============================================================

class RAGSearchState(BaseModel):
    """RAG 检索状态 - 演示 field_validator + model_validator

    验证规则：
    - query: 自动去除首尾空格，至少 2 个字符
    - search_type: 只允许 semantic / keyword / hybrid
    - top_k: 1-50 之间
    - threshold: 0.0-1.0 之间
    - 跨字段规则: keyword 搜索时 threshold 自动归零;
                  top_k > 20 时 threshold 不能低于 0.5
    """
    query: str
    search_type: str = "semantic"
    top_k: int = Field(default=5, ge=1, le=50)
    threshold: float = Field(default=0.7, ge=0.0, le=1.0)
    results: list = Field(default_factory=list)
    error_message: str = Field(default="")

    # ----------------------------------------------------------
    # field_validator：字段级验证
    # ----------------------------------------------------------

    @field_validator("query")
    @classmethod
    def clean_and_validate_query(cls, v: str) -> str:
        """清洗并验证查询字符串

        1. 去除首尾空格（清洗）
        2. 检查是否为空（校验）
        3. 检查最小长度（校验）
        """
        v = v.strip()
        if not v:
            raise ValueError("查询不能为空（去除空格后）")
        if len(v) < 2:
            raise ValueError(f"查询至少需要 2 个字符，当前: '{v}'（{len(v)} 字符）")
        return v

    @field_validator("search_type")
    @classmethod
    def validate_search_type(cls, v: str) -> str:
        """验证搜索类型是否在允许列表中"""
        allowed = {"semantic", "keyword", "hybrid"}
        if v not in allowed:
            raise ValueError(
                f"search_type 必须是 {allowed} 之一，收到: '{v}'"
            )
        return v

    # ----------------------------------------------------------
    # model_validator：模型级验证（跨字段）
    # ----------------------------------------------------------

    @model_validator(mode="after")
    def validate_search_params(self) -> Self:
        """跨字段联合验证

        规则 1: keyword 搜索不使用相似度阈值，自动归零
        规则 2: top_k > 20 时，threshold 不能低于 0.5
                （大量结果 + 低阈值 = 垃圾结果泛滥）
        """
        # 规则 1：keyword 搜索自动归零 threshold
        if self.search_type == "keyword" and self.threshold > 0:
            self.threshold = 0.0

        # 规则 2：大 top_k + 低 threshold 的组合不允许
        if self.top_k > 20 and self.threshold < 0.5:
            raise ValueError(
                f"top_k={self.top_k}(>20) 时，threshold 不能低于 0.5"
                f"（当前: {self.threshold}）。大量结果需要更高的质量门槛。"
            )
        return self


# ============================================================
# 2. 定义节点函数
# ============================================================

def search_node(state: RAGSearchState):
    """模拟 RAG 检索节点"""
    print(f"  搜索类型: {state.search_type}")
    print(f"  查询内容: {state.query}")
    print(f"  Top-K: {state.top_k}, 阈值: {state.threshold}")

    # 模拟检索结果
    mock_results = [
        {"title": f"文档_{i}", "score": round(1.0 - i * 0.1, 2)}
        for i in range(min(state.top_k, 5))
    ]
    return {"results": mock_results}


def format_node(state: RAGSearchState):
    """格式化输出节点"""
    print(f"  检索到 {len(state.results)} 条结果")
    for r in state.results:
        print(f"    - {r['title']} (score: {r['score']})")
    return {}


# ============================================================
# 3. 构建 LangGraph 图
# ============================================================

builder = StateGraph(RAGSearchState)
builder.add_node("search", search_node)
builder.add_node("format", format_node)
builder.add_edge(START, "search")
builder.add_edge("search", "format")
builder.add_edge("format", END)
graph = builder.compile()
```

## 关键知识点解析

### 1. field_validator vs model_validator 的分工

| 维度 | `@field_validator` | `@model_validator` |
|------|-------------------|-------------------|
| 作用范围 | 单个字段 | 整个模型（所有字段） |
| 触发时机 | 字段类型检查之后 | 所有 field_validator 之后 |
| 典型用途 | 清洗、格式化、单字段校验 | 跨字段依赖、联合约束 |
| 能修改值吗 | 可以（返回新值） | 可以（修改 self 属性） |
| 装饰器要求 | 必须加 `@classmethod` | 不需要（mode='after'） |

**决策流程**：
```
需要验证 → 只涉及一个字段？
              ├── 是 → 能用 Field 约束？
              │         ├── 是 → 用 Field（最简单）
              │         └── 否 → 用 @field_validator
              └── 否 → 用 @model_validator
```

### 2. field_validator 的关键语法

```python
@field_validator("字段名")
@classmethod
def 方法名(cls, v: 字段类型) -> 字段类型:
    # v 是字段的值（已经过类型转换）
    # 验证失败：raise ValueError("错误信息")
    # 验证通过：return 处理后的值（可以修改）
    return v
```

注意事项：
- 必须加 `@classmethod`（Pydantic v2 要求）
- 参数 `v` 是已经过类型转换的值（比如 `"5"` 已经变成 `5`）
- 可以同时验证多个字段：`@field_validator("field1", "field2")`
- 返回值会替换原始值（这就是「清洗」能力的来源）

### 3. model_validator 的两种模式

```python
# mode='before'：在所有字段验证之前运行，接收原始字典
@model_validator(mode='before')
@classmethod
def pre_process(cls, data: dict) -> dict:
    return data

# mode='after'：在所有字段验证之后运行（推荐），接收已验证实例
@model_validator(mode='after')
def post_validate(self) -> Self:
    return self
```

本场景使用 `mode='after'`，因为需要在字段值都确认合法后再检查它们之间的关系。

### 4. 验证器执行顺序

```
输入 → Step 1: 类型检查 + Field 约束
     → Step 2: @field_validator（按字段定义顺序）
     → Step 3: @model_validator(mode='after')
```

如果 Step 1/2 有错误，Step 3 不会执行。Pydantic 会尽可能收集 Step 1/2 中的所有错误一次性报告，而不是遇到第一个就停止。

## 在 RAG 系统中的应用

自定义验证器在 RAG 系统中的典型场景：

```python
class ProductionRAGState(BaseModel):
    """生产级 RAG 状态 - 验证器组合示例"""
    query: str
    language: str = "zh"
    search_type: str = "hybrid"
    top_k: int = Field(default=5, ge=1, le=50)
    rerank: bool = False

    @field_validator("query")
    @classmethod
    def sanitize_query(cls, v: str) -> str:
        v = v.strip()
        if not v:
            raise ValueError("查询不能为空")
        return v[:500] if len(v) > 500 else v  # 截断超长输入

    @model_validator(mode="after")
    def adjust_for_language(self) -> Self:
        if self.language == "zh" and self.search_type == "keyword":
            self.search_type = "hybrid"  # 中文用混合搜索更好
        if self.rerank and self.top_k < 10:
            self.top_k = 10  # rerank 需要更多初始候选
        return self
```

所有业务规则集中在状态定义中，修改一处即可全局生效。
