---

## 场景描述

在 RAG 系统中，我们经常需要按照不同的维度对数据进行物理隔离：
- **按时间分区**：将历史数据和最新数据分开存储
- **按业务类型分区**：将不同类型的文档分开管理
- **按数据来源分区**：将不同来源的数据分开存储

本场景演示如何使用 Milvus 的手动分区管理功能，实现基础的分区创建、加载、释放和删除操作。

---

## 核心概念

### 1. 手动分区 vs 自动分区

**手动分区（本场景）**：
- 开发者手动创建分区
- 插入数据时指定分区名称
- 搜索时指定分区名称
- 最多支持 1,024 个分区

**自动分区（Partition Key）**：
- Milvus 自动根据字段值创建分区
- 无需手动指定分区名称
- 支持数百万租户

### 2. 分区生命周期

```
创建分区 → 加载分区 → 使用分区 → 释放分区 → 删除分区
   ↓          ↓          ↓          ↓          ↓
create    load       insert     release    drop
partition  partition  /search    partition  partition
```

### 3. 默认分区

- 每个 Collection 都有一个名为 `_default` 的默认分区
- 如果不指定分区，数据会插入到默认分区
- 默认分区无法删除

## 完整代码示例

```python
"""
Milvus 基础分区操作实战
场景：按业务类型管理文档知识库

功能：
1. 创建分区（按文档类型：技术文档、产品文档、营销文档）
2. 向不同分区插入数据
3. 加载和释放分区
4. 在指定分区中搜索
5. 删除分区

依赖：
- pymilvus
- numpy
"""

from pymilvus import (
    connections,
    Collection,
    CollectionSchema,
    FieldSchema,
    DataType,
    utility
)
import numpy as np
from typing import List, Dict, Any
import time


class PartitionManager:
    """分区管理器 - 演示基础分区操作"""

    def __init__(self, collection_name: str = "document_knowledge_base"):
        """
        初始化分区管理器

        Args:
            collection_name: Collection 名称
        """
        self.collection_name = collection_name
        self.collection = None

    def connect_to_milvus(self, host: str = "localhost", port: str = "19530"):
        """连接到 Milvus"""
        print(f"连接到 Milvus: {host}:{port}")
        connections.connect(
            alias="default",
            host=host,
            port=port
        )
        print("✓ 连接成功")

    def create_collection(self):
        """创建 Collection（如果不存在）"""
        # 检查 Collection 是否存在
        if utility.has_collection(self.collection_name):
            print(f"Collection '{self.collection_name}' 已存在，删除后重新创建")
            utility.drop_collection(self.collection_name)

        # 定义 Schema
        fields = [
            FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
            FieldSchema(name="title", dtype=DataType.VARCHAR, max_length=512),
            FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=2000),
            FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=128)
        ]

        schema = CollectionSchema(
            fields=fields,
            description="文档知识库 - 按业务类型分区"
        )

        # 创建 Collection
        self.collection = Collection(
            name=self.collection_name,
            schema=schema
        )

        print(f"✓ Collection '{self.collection_name}' 创建成功")

    def create_partitions(self, partition_names: List[str]):
        """
        创建多个分区

        Args:
            partition_names: 分区名称列表
        """
        print("\n=== 创建分区 ===")
        for partition_name in partition_names:
            self.collection.create_partition(partition_name)
            print(f"✓ 分区 '{partition_name}' 创建成功")

    def list_partitions(self):
        """列出所有分区"""
        print("\n=== 分区列表 ===")
        partitions = self.collection.partitions
        for partition in partitions:
            print(f"- {partition.name}")
        return [p.name for p in partitions]

    def check_partition_exists(self, partition_name: str) -> bool:
        """检查分区是否存在"""
        exists = self.collection.has_partition(partition_name)
        print(f"分区 '{partition_name}' 存在: {exists}")
        return exists

    def insert_data_to_partition(
        self,
        partition_name: str,
        titles: List[str],
        contents: List[str]
    ):
        """
        向指定分区插入数据

        Args:
            partition_name: 分区名称
            titles: 文档标题列表
            contents: 文档内容列表
        """
        print(f"\n=== 向分区 '{partition_name}' 插入数据 ===")

        # 生成随机 embedding（实际应用中应使用真实的 embedding 模型）
        embeddings = np.random.rand(len(titles), 128).tolist()

        # 准备数据
        data = [
            titles,
            contents,
            embeddings
        ]

        # 插入数据到指定分区
        insert_result = self.collection.insert(
            data=data,
            partition_name=partition_name
        )

        print(f"✓ 插入 {insert_result.insert_count} 条数据到分区 '{partition_name}'")
        return insert_result

    def create_index(self):
        """创建向量索引"""
        print("\n=== 创建索引 ===")
        index_params = {
            "index_type": "IVF_FLAT",
            "metric_type": "L2",
            "params": {"nlist": 128}
        }

        self.collection.create_index(
            field_name="embedding",
            index_params=index_params
        )
        print("✓ 索引创建成功")

    def load_partitions(self, partition_names: List[str]):
        """
        加载指定分区到内存

        Args:
            partition_names: 要加载的分区名称列表
        """
        print(f"\n=== 加载分区 ===")
        self.collection.load(partition_names=partition_names)

        # 等待加载完成
        time.sleep(2)

        # 检查加载状态
        for partition_name in partition_names:
            state = utility.load_state(
                collection_name=self.collection_name,
                partition_names=[partition_name]
            )
            print(f"✓ 分区 '{partition_name}' 加载状态: {state}")

    def search_in_partition(
        self,
        partition_name: str,
        query_embedding: List[float],
        top_k: int = 5
    ):
        """
        在指定分区中搜索

        Args:
            partition_name: 分区名称
            query_embedding: 查询向量
            top_k: 返回结果数量
        """
        print(f"\n=== 在分区 '{partition_name}' 中搜索 ===")

        search_params = {
            "metric_type": "L2",
            "params": {"nprobe": 10}
        }

        results = self.collection.search(
            data=[query_embedding],
            anns_field="embedding",
            param=search_params,
            limit=top_k,
            partition_names=[partition_name],
            output_fields=["title", "content"]
        )

        print(f"找到 {len(results[0])} 条结果:")
        for i, hit in enumerate(results[0]):
            print(f"{i+1}. ID: {hit.id}, 距离: {hit.distance:.4f}")
            print(f"   标题: {hit.entity.get('title')}")
            print(f"   内容: {hit.entity.get('content')[:50]}...")

        return results

    def release_partitions(self, partition_names: List[str]):
        """
        释放指定分区（从内存中卸载）

        Args:
            partition_names: 要释放的分区名称列表
        """
        print(f"\n=== 释放分区 ===")
        self.collection.release(partition_names=partition_names)

        # 等待释放完成
        time.sleep(1)

        for partition_name in partition_names:
            print(f"✓ 分区 '{partition_name}' 已释放")

    def drop_partition(self, partition_name: str):
        """
        删除分区

        Args:
            partition_name: 要删除的分区名称
        """
        print(f"\n=== 删除分区 '{partition_name}' ===")
        self.collection.drop_partition(partition_name)
        print(f"✓ 分区 '{partition_name}' 已删除")

    def get_partition_stats(self, partition_name: str):
        """获取分区统计信息"""
        partition = self.collection.partition(partition_name)
        num_entities = partition.num_entities
        print(f"分区 '{partition_name}' 包含 {num_entities} 条数据")
        return num_entities


def main():
    """主函数 - 演示完整的分区操作流程"""

    # 1. 初始化
    manager = PartitionManager()
    manager.connect_to_milvus()
    manager.create_collection()

    # 2. 创建分区（按文档类型）
    partition_names = [
        "technical_docs",  # 技术文档
        "product_docs",    # 产品文档
        "marketing_docs"   # 营销文档
    ]
    manager.create_partitions(partition_names)

    # 3. 列出所有分区
    all_partitions = manager.list_partitions()

    # 4. 检查分区是否存在
    manager.check_partition_exists("technical_docs")

    # 5. 向不同分区插入数据
    # 技术文档
    manager.insert_data_to_partition(
        partition_name="technical_docs",
        titles=["API 文档", "架构设计", "性能优化指南"],
        contents=[
            "本文档介绍了 REST API 的使用方法...",
            "系统采用微服务架构，包含以下模块...",
            "性能优化的关键在于减少数据库查询次数..."
        ]
    )

    # 产品文档
    manager.insert_data_to_partition(
        partition_name="product_docs",
        titles=["产品介绍", "功能列表", "使用手册"],
        contents=[
            "我们的产品是一款智能知识管理系统...",
            "核心功能包括：文档管理、智能搜索、协作编辑...",
            "使用手册：第一步，创建知识库..."
        ]
    )

    # 营销文档
    manager.insert_data_to_partition(
        partition_name="marketing_docs",
        titles=["市场分析", "竞品对比", "营销策略"],
        contents=[
            "当前市场规模达到 100 亿美元...",
            "与竞品相比，我们的优势在于...",
            "营销策略：聚焦企业客户，提供定制化服务..."
        ]
    )

    # 6. 创建索引
    manager.create_index()

    # 7. 加载部分分区（只加载技术文档和产品文档）
    manager.load_partitions(["technical_docs", "product_docs"])

    # 8. 在指定分区中搜索
    query_embedding = np.random.rand(128).tolist()
    manager.search_in_partition("technical_docs", query_embedding, top_k=3)

    # 9. 获取分区统计信息
    manager.get_partition_stats("technical_docs")
    manager.get_partition_stats("product_docs")

    # 10. 释放分区
    manager.release_partitions(["technical_docs", "product_docs"])

    # 11. 删除分区（演示删除营销文档分区）
    manager.drop_partition("marketing_docs")

    # 12. 再次列出所有分区
    manager.list_partitions()

    print("\n=== 演示完成 ===")


if __name__ == "__main__":
    main()
```

## 代码说明

### 1. 核心操作

**创建分区**：
```python
collection.create_partition("partition_name")
```

**插入数据到分区**：
```python
collection.insert(data=data, partition_name="partition_name")
```

**加载分区**：
```python
collection.load(partition_names=["partition1", "partition2"])
```

**在分区中搜索**：
```python
collection.search(
    data=[query_embedding],
    partition_names=["partition_name"],
    ...
)
```

**释放分区**：
```python
collection.release(partition_names=["partition_name"])
```

**删除分区**：
```python
collection.drop_partition("partition_name")
```

### 2. 关键注意事项

1. **默认分区**：每个 Collection 都有一个 `_default` 分区，无法删除
2. **加载要求**：搜索前必须加载分区，否则会报错
3. **分区数量限制**：最多 1,024 个分区
4. **内存管理**：不使用的分区应及时释放，节省内存

## RAG 应用场景

### 场景 1：按时间分区管理历史数据

```python
# 创建按月份的分区
partitions = [
    "docs_2024_01",
    "docs_2024_02",
    "docs_2024_03"
]

# 只加载最近 3 个月的数据
collection.load(partition_names=["docs_2024_01", "docs_2024_02", "docs_2024_03"])

# 搜索时只在最近的数据中查找
results = collection.search(
    data=[query_embedding],
    partition_names=["docs_2024_03"],  # 只搜索最新月份
    limit=10
)
```

### 场景 2：按业务类型隔离数据

```python
# 创建按业务类型的分区
partitions = [
    "customer_service",  # 客服文档
    "technical_support", # 技术支持文档
    "sales"              # 销售文档
]

# 根据用户角色加载不同的分区
if user_role == "customer_service":
    collection.load(partition_names=["customer_service"])
elif user_role == "technical":
    collection.load(partition_names=["technical_support"])
```

### 场景 3：热冷数据管理

```python
# 热数据：最近 7 天的文档
hot_partitions = ["docs_recent"]

# 冷数据：历史文档
cold_partitions = ["docs_archive"]

# 默认只加载热数据
collection.load(partition_names=hot_partitions)

# 需要时再加载冷数据
if need_historical_data:
    collection.load(partition_names=cold_partitions)
```

## 性能优化建议

### 1. 分区大小

**推荐**：每个分区 20-100K 行数据

**原因**：
- 分区过小：管理开销大
- 分区过大：加载和释放耗时长

### 2. 分区数量

**推荐**：控制在 100-200 个分区以内

**原因**：
- 分区过多：元数据管理开销大
- 分区过少：无法充分利用分区优势

### 3. 加载策略

**按需加载**：
```python
# 只加载需要搜索的分区
collection.load(partition_names=["active_partition"])

# 搜索完成后释放
collection.release(partition_names=["active_partition"])
```

**预加载**：
```python
# 预加载常用分区
collection.load(partition_names=["hot_partition1", "hot_partition2"])
```

## 常见问题

### Q1: 如何选择手动分区还是 Partition Key？

**手动分区**：
- 适用于中等数量的分区（< 1,024）
- 需要物理隔离
- 需要热冷数据管理

**Partition Key**：
- 适用于大规模多租户（百万级）
- 自动管理，无需手动创建
- 数据隔离较弱

### Q2: 分区是否会影响搜索性能？

**会**，但影响是正面的：
- 指定分区搜索：只扫描相关分区，性能提升
- 不指定分区搜索：扫描所有已加载分区，性能下降

### Q3: 如何删除分区中的数据？

```python
# 方法 1：删除整个分区
collection.drop_partition("partition_name")

# 方法 2：使用 delete 表达式（需要先加载分区）
collection.delete(expr="id in [1, 2, 3]", partition_name="partition_name")
```
