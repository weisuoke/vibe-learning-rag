## 卡片1：传统Memory弃用

**核心信息**：
- 所有传统 Memory 类已被标记为 `@deprecated`
- 自 0.3.1 版本起弃用，将在 1.0.0 版本移除
- 不支持原生工具调用，会静默失败

**弃用原因**：
- 架构设计在工具调用能力之前
- 无法与现代 LLM 兼容
- 缺乏 LCEL 管道集成

**迁移路径**：
```python
# 旧方式（已弃用）
from langchain.memory import ConversationBufferMemory

# 新方式（推荐）
from langchain_core.runnables.history import RunnableWithMessageHistory
```

## 卡片2：RunnableWithMessageHistory核心

**核心机制**：
- 实现 Runnable 接口，支持 LCEL 管道
- 自动管理对话历史
- 支持 LangSmith 追踪

**基础用法**：
```python
conversation = RunnableWithMessageHistory(
    llm,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history"
)

response = conversation.invoke(
    {"input": "用户消息"},
    config={"configurable": {"session_id": "session_001"}}
)
```

**关键优势**：
- 统一的 Runnable 接口
- 标准的 invoke、ainvoke、batch 方法
- 灵活的 Session 管理

## 卡片3：Session管理模式

**核心概念**：
- Session ID：隔离不同对话
- get_session_history：获取历史的回调函数
- config：传递 session_id 的配置对象

**实现模式**：
```python
store = {}

def get_session_history(session_id: str):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]
```

**关键点**：
- 每个 session_id 对应一个独立的对话历史
- 通过 config 传递 session_id
- 支持多用户多会话

## 卡片4：持久化存储选择

**存储方案对比**：

| 场景 | 方案 | 特点 |
|------|------|------|
| 开发/测试 | InMemory | 简单、快速 |
| 生产（高性能） | Redis | 高性能、TTL |
| 生产（持久化） | PostgreSQL | 持久化、复杂查询 |
| 小型应用 | SQLite | 轻量级、无需额外服务 |

**选择因素**：
- 会话时长：短期用 Redis，长期用 PostgreSQL
- 并发量：低并发用 SQLite，高并发用 Redis
- 数据量：小数据用内存，大数据用 PostgreSQL

## 卡片5：异步操作优势

**性能对比**：
- 同步操作：100 请求/秒
- 异步操作：478 请求/秒（4.78x 提升）

**实现方式**：
```python
# 异步 Session History
async def get_session_history_async(session_id: str):
    pool = await asyncpg.create_pool(DATABASE_URL)
    return AsyncPostgresChatMessageHistory(session_id, pool)

# 异步调用
response = await conversation.ainvoke(
    {"input": "用户消息"},
    config={"configurable": {"session_id": "session_001"}}
)
```

**关键点**：
- 使用 asyncpg、aioredis 等异步库
- 配置连接池（min_size、max_size）
- 使用 ainvoke、aadd_message 等异步方法

## 卡片6：多用户隔离

**隔离机制**：
- Thread ID：每个用户有独立的 thread ID
- Session 管理：通过 session ID 区分不同会话
- 并发控制：使用异步锁保证线程安全

**实现方式**：
```python
class MultiUserSessionManager:
    def __init__(self):
        # 存储结构: {user_id: {session_id: history}}
        self.store = {}

    def get_session_history(self, user_id: str, session_id: str):
        if user_id not in self.store:
            self.store[user_id] = {}
        if session_id not in self.store[user_id]:
            self.store[user_id][session_id] = InMemoryChatMessageHistory()
        return self.store[user_id][session_id]
```

**关键点**：
- 用户隔离：每个用户有独立的存储空间
- 会话隔离：同一用户可以有多个会话
- 权限验证：确保用户只能访问自己的数据

## 卡片7：内存限制策略

**三种策略**：

1. **Token 限制**：
```python
total_tokens = count_tokens_approximately(messages)
while total_tokens > max_tokens and len(messages) > 2:
    messages.pop(0)
```

2. **消息数量限制**：
```python
if len(history.messages) > max_messages:
    history.messages = history.messages[-max_messages:]
```

3. **滑动窗口**：
```python
# 保留最近 N 轮对话
max_conversation_messages = window_size * 2
conversation_messages = conversation_messages[-max_conversation_messages:]
```

**选择建议**：
- 短对话（<10轮）：完整历史
- 中等对话（10-50轮）：滑动窗口
- 长对话（>50轮）：自动总结

## 卡片8：LangGraph Checkpointer

**核心机制**：
- 在每个节点执行后保存图的状态
- 支持跨会话恢复对话
- Thread ID 隔离不同对话
- 支持时间旅行（回溯到历史状态）

**基础用法**：
```python
from langgraph.checkpoint.memory import InMemorySaver

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "thread_001"}}
graph.invoke({"messages": "message"}, config)
```

**适用场景**：
- 复杂的 Agent 系统
- 需要管理完整图状态
- 多节点工作流

## 卡片9：LangMem SDK

**核心特性**（2025年2月发布）：
- 语义知识提取：从对话中自动提取关键信息
- 跨会话记忆：维护长期用户记忆
- 自动优化提示：根据记忆优化 Agent 提示
- LangGraph 集成：无缝集成到 LangGraph 工作流

**使用方式**：
```python
from langmem import MemoryStore

# 短期记忆：Checkpointer
checkpointer = InMemorySaver()

# 长期记忆：LangMem Store
store = MemoryStore()

# 集成
graph = builder.compile(
    checkpointer=checkpointer,
    store=store
)
```

**应用场景**：
- 个人助理、客户服务、教育辅导、长期用户交互

## 卡片10：生产环境架构

**架构设计**：
```
用户请求
    ↓
API 网关（FastAPI）
    ↓
Session 管理层
    ├─ Session ID 生成
    ├─ 用户认证
    └─ 权限验证
    ↓
对话历史管理层
    ├─ RunnableWithMessageHistory
    ├─ 持久化存储（Redis/PostgreSQL）
    └─ 内存限制策略
    ↓
LLM 调用层
    ├─ ChatOpenAI
    ├─ 异步操作
    └─ 错误处理
    ↓
响应返回
```

**关键组件**：
- 持久化存储：Redis（主）+ PostgreSQL（备）+ Memory（降级）
- 内存限制：Token 限制 + 消息数量限制 + 自动总结
- 错误处理：重试机制 + 降级策略 + 监控告警
- 性能优化：异步操作 + 连接池 + 批量操作

**性能指标**：
- 响应时间：< 100ms
- 并发支持：> 1000 请求/秒
- 成功率：> 99.9%
