**60秒快速诊断是 OpenClaw 故障排查的黄金流程，通过7个标准化命令的组合执行，在1分钟内快速定位90%的常见问题。**

## 二、诊断流程

### 2.1 完整诊断命令序列

```bash
# 60秒快速诊断流程（7个命令）
openclaw status                      # 5秒  - 快速状态概览
openclaw status --all                # 10秒 - 完整诊断报告
openclaw gateway probe               # 5秒  - RPC 连接测试
openclaw gateway status              # 5秒  - 网关运行状态
openclaw doctor                      # 15秒 - 健康检查
openclaw channels status --probe     # 10秒 - 通道连接探测
openclaw logs --follow               # 10秒 - 实时日志观察
```

**来源**: `reference/source_troubleshooting_main.md`

### 2.2 健康输出标准

| 命令 | 健康输出 | 异常信号 |
|------|---------|---------|
| `openclaw status` | 显示已配置通道，无认证错误 | 认证失败、通道缺失 |
| `openclaw status --all` | 完整报告可分享 | 大量错误、服务不可达 |
| `openclaw gateway probe` | 网关目标可达 | connection refused, timeout |
| `openclaw gateway status` | `Runtime: running`, `RPC probe: ok` | stopped, failed |
| `openclaw doctor` | 无阻塞性错误 | Config invalid, service error |
| `openclaw channels status --probe` | `connected` 或 `ready` | disconnected, auth error |
| `openclaw logs --follow` | 稳定活动，无重复致命错误 | FATAL, ERROR 重复出现 |

## 三、TypeScript 实现

### 3.1 诊断脚本核心实现

```typescript
// quick-diagnostic.ts
import { spawn } from 'child_process';

interface DiagnosticCommand {
  cmd: string;
  args: string[];
  timeout: number;
  description: string;
}

interface CommandResult {
  command: string;
  output: string;
  exitCode: number;
  duration: number;
  healthy: boolean;
}

interface DiagnosticReport {
  timestamp: number;
  totalDuration: number;
  results: CommandResult[];
  overallHealth: 'healthy' | 'degraded' | 'unhealthy';
  issues: string[];
  recommendations: string[];
}

class QuickDiagnostic {
  private commands: DiagnosticCommand[] = [
    {
      cmd: 'openclaw',
      args: ['status'],
      timeout: 5000,
      description: 'Quick status overview'
    },
    {
      cmd: 'openclaw',
      args: ['status', '--all'],
      timeout: 10000,
      description: 'Full diagnostic report'
    },
    {
      cmd: 'openclaw',
      args: ['gateway', 'probe'],
      timeout: 5000,
      description: 'RPC connection test'
    },
    {
      cmd: 'openclaw',
      args: ['gateway', 'status'],
      timeout: 5000,
      description: 'Gateway runtime status'
    },
    {
      cmd: 'openclaw',
      args: ['doctor'],
      timeout: 15000,
      description: 'Health check'
    },
    {
      cmd: 'openclaw',
      args: ['channels', 'status', '--probe'],
      timeout: 10000,
      description: 'Channel connectivity probe'
    }
  ];

  private results: CommandResult[] = [];
  private issues: string[] = [];

  async run(): Promise<DiagnosticReport> {
    const startTime = Date.now();

    console.log('=== OpenClaw 60-Second Quick Diagnostic ===\n');

    // 执行所有诊断命令
    for (const command of this.commands) {
      const result = await this.runCommand(
        command.cmd,
        command.args,
        command.timeout
      );
      this.results.push(result);

      // 实时输出结果
      this.printResult(result);

      // 分析结果
      this.analyzeResult(result);
    }

    // 生成报告
    const totalDuration = Date.now() - startTime;
    const report = this.generateReport(totalDuration);

    // 输出总结
    this.printSummary(report);

    return report;
  }

  private async runCommand(
    cmd: string,
    args: string[],
    timeout: number
  ): Promise<CommandResult> {
    const startTime = Date.now();
    const commandStr = `${cmd} ${args.join(' ')}`;

    return new Promise((resolve) => {
      const proc = spawn(cmd, args);
      let output = '';
      let timedOut = false;

      // 设置超时
      const timer = setTimeout(() => {
        timedOut = true;
        proc.kill();
      }, timeout);

      // 收集输出
      proc.stdout.on('data', (data) => {
        output += data.toString();
      });

      proc.stderr.on('data', (data) => {
        output += data.toString();
      });

      // 处理完成
      proc.on('close', (code) => {
        clearTimeout(timer);
        const duration = Date.now() - startTime;

        resolve({
          command: commandStr,
          output,
          exitCode: timedOut ? -1 : (code || 0),
          duration,
          healthy: !timedOut && code === 0
        });
      });
    });
  }

  private printResult(result: CommandResult): void {
    const status = result.healthy ? '✓' : '✗';
    const color = result.healthy ? '\x1b[32m' : '\x1b[31m';
    const reset = '\x1b[0m';

    console.log(`${color}${status}${reset} ${result.command} (${result.duration}ms)`);

    if (!result.healthy) {
      console.log(`  Output: ${result.output.substring(0, 200)}...`);
    }
  }

  private analyzeResult(result: CommandResult): void {
    if (!result.healthy) {
      this.issues.push(`Command failed: ${result.command}`);
    }

    // 分析特定命令的输出
    if (result.command.includes('gateway status')) {
      if (result.output.includes('Runtime: stopped')) {
        this.issues.push('Gateway is not running');
      }
      if (result.output.includes('RPC probe: failed')) {
        this.issues.push('RPC connection failed');
      }
    }

    if (result.command.includes('doctor')) {
      if (result.output.includes('Config') && result.output.includes('invalid')) {
        this.issues.push('Configuration validation failed');
      }
    }

    if (result.command.includes('channels status')) {
      if (result.output.includes('disconnected')) {
        this.issues.push('Some channels are disconnected');
      }
    }
  }

  private generateReport(totalDuration: number): DiagnosticReport {
    const healthyCount = this.results.filter(r => r.healthy).length;
    const totalCount = this.results.length;

    let overallHealth: 'healthy' | 'degraded' | 'unhealthy';
    if (healthyCount === totalCount) {
      overallHealth = 'healthy';
    } else if (healthyCount >= totalCount * 0.7) {
      overallHealth = 'degraded';
    } else {
      overallHealth = 'unhealthy';
    }

    const recommendations = this.generateRecommendations();

    return {
      timestamp: Date.now(),
      totalDuration,
      results: this.results,
      overallHealth,
      issues: this.issues,
      recommendations
    };
  }

  private generateRecommendations(): string[] {
    const recommendations: string[] = [];

    if (this.issues.some(i => i.includes('Gateway is not running'))) {
      recommendations.push('Run: openclaw gateway start');
    }

    if (this.issues.some(i => i.includes('RPC connection failed'))) {
      recommendations.push('Check gateway logs: openclaw logs --follow');
      recommendations.push('Run: openclaw doctor --repair');
    }

    if (this.issues.some(i => i.includes('Configuration validation failed'))) {
      recommendations.push('Run: openclaw doctor --repair');
    }

    if (this.issues.some(i => i.includes('channels are disconnected'))) {
      recommendations.push('Check channel configuration');
      recommendations.push('Run: openclaw channels reconnect <channel>');
    }

    if (recommendations.length === 0) {
      recommendations.push('System is healthy, no action needed');
    }

    return recommendations;
  }

  private printSummary(report: DiagnosticReport): void {
    console.log('\n=== Diagnostic Summary ===');
    console.log(`Total Duration: ${report.totalDuration}ms`);
    console.log(`Overall Health: ${report.overallHealth.toUpperCase()}`);

    if (report.issues.length > 0) {
      console.log('\nIssues Found:');
      report.issues.forEach(issue => console.log(`  - ${issue}`));
    }

    console.log('\nRecommendations:');
    report.recommendations.forEach(rec => console.log(`  - ${rec}`));
  }
}

// 使用示例
const diagnostic = new QuickDiagnostic();
diagnostic.run().then(report => {
  // 可以将报告保存到文件或发送到监控系统
  console.log('\nDiagnostic complete!');
  process.exit(report.overallHealth === 'healthy' ? 0 : 1);
});
```

### 3.2 决策树实现

```typescript
// diagnostic-decision-tree.ts
enum IssueType {
  NO_REPLIES = 'no_replies',
  DASHBOARD_CONNECTIVITY = 'dashboard_connectivity',
  GATEWAY_STARTUP = 'gateway_startup',
  CHANNEL_FLOW = 'channel_flow',
  AUTOMATION = 'automation',
  NODE_TOOLS = 'node_tools',
  BROWSER_TOOLS = 'browser_tools'
}

interface DiagnosticPath {
  issueType: IssueType;
  commands: string[];
  logSignatures: string[];
  solutions: string[];
}

class DiagnosticDecisionTree {
  private paths: Map<IssueType, DiagnosticPath> = new Map([
    [IssueType.NO_REPLIES, {
      issueType: IssueType.NO_REPLIES,
      commands: [
        'openclaw status',
        'openclaw gateway status',
        'openclaw channels status --probe',
        'openclaw pairing list <channel>',
        'openclaw logs --follow'
      ],
      logSignatures: [
        'drop guild message (mention required)',
        'pairing request',
        'blocked',
        'allowlist'
      ],
      solutions: [
        'Approve pairing: openclaw pairing approve <channel> <sender>',
        'Mention bot in group: @BotName hello',
        'Check agent status: openclaw status --all'
      ]
    }],
    [IssueType.GATEWAY_STARTUP, {
      issueType: IssueType.GATEWAY_STARTUP,
      commands: [
        'openclaw status',
        'openclaw gateway status',
        'openclaw logs --follow',
        'openclaw doctor'
      ],
      logSignatures: [
        'Gateway start blocked: set gateway.mode=local',
        'refusing to bind gateway ... without auth',
        'another gateway instance is already listening',
        'EADDRINUSE'
      ],
      solutions: [
        'Set gateway mode: openclaw config set gateway.mode local',
        'Stop other instances: openclaw gateway stop',
        'Run doctor: openclaw doctor --repair'
      ]
    }]
  ]);

  identifyIssue(symptoms: string[]): IssueType | null {
    // 根据症状识别问题类型
    for (const [issueType, path] of this.paths) {
      const matchCount = symptoms.filter(symptom =>
        path.logSignatures.some(sig => symptom.includes(sig))
      ).length;

      if (matchCount > 0) {
        return issueType;
      }
    }

    return null;
  }

  getDiagnosticPath(issueType: IssueType): DiagnosticPath | undefined {
    return this.paths.get(issueType);
  }

  async runDiagnosticPath(issueType: IssueType): Promise<void> {
    const path = this.getDiagnosticPath(issueType);
    if (!path) {
      console.log('Unknown issue type');
      return;
    }

    console.log(`\n=== Diagnosing: ${issueType} ===\n`);

    // 执行诊断命令
    for (const command of path.commands) {
      console.log(`Running: ${command}`);
      // 实际执行命令的逻辑
    }

    // 输出解决方案
    console.log('\n=== Recommended Solutions ===');
    path.solutions.forEach(solution => {
      console.log(`  - ${solution}`);
    });
  }
}
```

## 四、Bash 自动化脚本

### 4.1 完整诊断脚本

```bash
#!/bin/bash
# quick-diagnostic.sh

echo "=== OpenClaw 60-Second Quick Diagnostic ==="
echo "Start time: $(date)"
echo ""

# 初始化计数器
TOTAL_CHECKS=7
PASSED_CHECKS=0
FAILED_CHECKS=0

# 颜色定义
GREEN='\033[0;32m'
RED='\033[0;31m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# 检查函数
check_command() {
  local cmd="$1"
  local description="$2"
  local timeout="${3:-10}"
  
  echo -n "[$((PASSED_CHECKS + FAILED_CHECKS + 1))/$TOTAL_CHECKS] $description... "
  
  # 执行命令并捕获输出
  if timeout "$timeout" $cmd > /tmp/openclaw_check_$$.txt 2>&1; then
    echo -e "${GREEN}✓${NC}"
    PASSED_CHECKS=$((PASSED_CHECKS + 1))
    return 0
  else
    echo -e "${RED}✗${NC}"
    FAILED_CHECKS=$((FAILED_CHECKS + 1))
    echo "  Output: $(head -n 3 /tmp/openclaw_check_$$.txt)"
    return 1
  fi
}

# 1. 快速状态检查
check_command "openclaw status" "Quick status overview" 5

# 2. 完整状态报告
check_command "openclaw status --all" "Full diagnostic report" 10

# 3. RPC 探测
check_command "openclaw gateway probe" "RPC connection test" 5

# 4. 网关状态
check_command "openclaw gateway status" "Gateway runtime status" 5

# 5. 健康检查
check_command "openclaw doctor" "Health check" 15

# 6. 通道探测
check_command "openclaw channels status --probe" "Channel connectivity probe" 10

# 7. 日志检查（非阻塞）
echo -n "[7/$TOTAL_CHECKS] Log observation... "
timeout 10 openclaw logs --follow > /tmp/openclaw_logs_$$.txt 2>&1 &
LOG_PID=$!
sleep 3
kill $LOG_PID 2>/dev/null

if grep -q "FATAL\|ERROR" /tmp/openclaw_logs_$$.txt; then
  echo -e "${RED}✗${NC} (Errors found in logs)"
  FAILED_CHECKS=$((FAILED_CHECKS + 1))
  echo "  Recent errors:"
  grep "FATAL\|ERROR" /tmp/openclaw_logs_$$.txt | head -n 3
else
  echo -e "${GREEN}✓${NC}"
  PASSED_CHECKS=$((PASSED_CHECKS + 1))
fi

# 清理临时文件
rm -f /tmp/openclaw_check_$$.txt /tmp/openclaw_logs_$$.txt

# 输出总结
echo ""
echo "=== Diagnostic Summary ==="
echo "Total Checks: $TOTAL_CHECKS"
echo -e "Passed: ${GREEN}$PASSED_CHECKS${NC}"
echo -e "Failed: ${RED}$FAILED_CHECKS${NC}"

# 健康度评估
if [ $FAILED_CHECKS -eq 0 ]; then
  echo -e "\nOverall Health: ${GREEN}HEALTHY${NC}"
  echo "✓ All systems operational"
  exit 0
elif [ $FAILED_CHECKS -le 2 ]; then
  echo -e "\nOverall Health: ${YELLOW}DEGRADED${NC}"
  echo "⚠ Some issues detected, but system is functional"
  echo ""
  echo "Recommendations:"
  echo "  - Run: openclaw doctor --repair"
  echo "  - Check logs: openclaw logs --follow"
  exit 1
else
  echo -e "\nOverall Health: ${RED}UNHEALTHY${NC}"
  echo "✗ Critical issues detected"
  echo ""
  echo "Immediate Actions:"
  echo "  1. Check if Gateway is running: openclaw gateway status"
  echo "  2. Run repair: openclaw doctor --repair"
  echo "  3. Restart Gateway: openclaw gateway restart"
  echo "  4. Check logs: openclaw logs --follow | grep ERROR"
  exit 2
fi
```

```bash
      exit 0
    fi
  fi
  
  echo "✗ Health check failed, manual intervention required"
```

## 六、常见问题

### 6.1 诊断超时怎么办？

**问题**：某个诊断命令超时

**解决方法**：
```bash
# 1. 跳过超时命令，继续其他检查
timeout 5 openclaw status || echo "Status check timed out, skipping..."

# 2. 增加超时时间
timeout 30 openclaw status --all

# 3. 检查网络连接
ping -c 3 localhost

# 4. 检查 Gateway 是否响应
lsof -i :18789
```

**来源**: `reference/source_troubleshooting_main.md`

