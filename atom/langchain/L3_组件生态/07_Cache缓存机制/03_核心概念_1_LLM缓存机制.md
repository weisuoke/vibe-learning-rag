# 核心概念1：LLM缓存机制

> LLM调用缓存的原理、实现与最佳实践

---

## 概述

LLM缓存机制通过存储LLM的调用结果，避免重复的API调用，从而降低成本和延迟。LangChain提供了统一的缓存接口和多种后端实现。

---

## 1. BaseCache抽象接口

### 核心设计

```python
from abc import ABC, abstractmethod
from typing import Any, List, Optional

class BaseCache(ABC):
    """LLM和Chat模型的缓存接口"""

    @abstractmethod
    def lookup(self, prompt: str, llm_string: str) -> Optional[List[Generation]]:
        """根据prompt和llm_string查询缓存"""

    @abstractmethod
    def update(self, prompt: str, llm_string: str, return_val: List[Generation]) -> None:
        """更新缓存"""

    @abstractmethod
    def clear(self, **kwargs: Any) -> None:
        """清空缓存"""
```

**关键点**：
- 所有缓存实现都继承自`BaseCache`
- 提供同步和异步两套API
- 缓存键由`(prompt, llm_string)`组成

### 缓存键设计

**prompt**：
- 对于普通LLM：提示词字符串
- 对于Chat模型：序列化后的消息列表

**llm_string**：
- LLM配置的字符串表示
- 包含：模型名、温度、max_tokens、停止词等
- 确保相同配置才命中缓存

**示例**：
```python
# 缓存键示例
prompt = "What is artificial intelligence?"
llm_string = "model=gpt-4o-mini,temperature=0.7,max_tokens=100"

# 缓存键 = (prompt, llm_string)
cache_key = (prompt, llm_string)
```

## 2. InMemoryCache实现

### 核心代码

```python
class InMemoryCache(BaseCache):
    """内存缓存实现"""

    def __init__(self, *, maxsize: Optional[int] = None) -> None:
        """初始化内存缓存

        Args:
            maxsize: 最大缓存条目数，None表示无限制
        """
        self._cache: dict[tuple[str, str], RETURN_VAL_TYPE] = {}
        self._maxsize = maxsize

    def lookup(self, prompt: str, llm_string: str) -> Optional[RETURN_VAL_TYPE]:
        """查询缓存"""
        return self._cache.get((prompt, llm_string))

    def update(self, prompt: str, llm_string: str, return_val: RETURN_VAL_TYPE) -> None:
        """更新缓存"""
        if self._maxsize is not None and len(self._cache) == self._maxsize:
            # FIFO淘汰策略
            del self._cache[next(iter(self._cache))]
        self._cache[(prompt, llm_string)] = return_val

    def clear(self, **kwargs: Any) -> None:
        """清空缓存"""
        self._cache.clear()
```

**关键特性**：
- 使用Python字典存储
- 支持maxsize限制
- FIFO（先进先出）淘汰策略
- 异步方法直接调用同步方法

### 使用示例

```python
from langchain_core.caches import InMemoryCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI
import time

# 1. 启用内存缓存
set_llm_cache(InMemoryCache(maxsize=100))

# 2. 创建LLM实例
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)

# 3. 首次调用（无缓存）
print("=== 首次调用 ===")
start = time.time()
response1 = llm.invoke("What is LangChain?")
print(f"耗时: {time.time() - start:.2f}秒")
print(f"响应: {response1.content[:100]}...")

# 4. 再次调用（缓存命中）
print("\n=== 缓存命中 ===")
start = time.time()
response2 = llm.invoke("What is LangChain?")
print(f"耗时: {time.time() - start:.2f}秒")
print(f"响应: {response2.content[:100]}...")

# 5. 不同配置（缓存未命中）
print("\n=== 不同配置 ===")
llm_different = ChatOpenAI(model="gpt-4o-mini", temperature=0.9)
start = time.time()
response3 = llm_different.invoke("What is LangChain?")
print(f"耗时: {time.time() - start:.2f}秒")
```

**输出示例**：
```
=== 首次调用 ===
耗时: 2.34秒
响应: LangChain is a framework for developing applications powered by language models...

=== 缓存命中 ===
耗时: 0.01秒  ← 速度提升200倍+
响应: LangChain is a framework for developing applications powered by language models...

=== 不同配置 ===
耗时: 2.18秒  ← 配置不同，缓存未命中
```

## 3. 全局缓存配置

### 配置方式

```python
from langchain_core.globals import set_llm_cache, get_llm_cache
from langchain_core.caches import InMemoryCache

# 设置全局缓存
set_llm_cache(InMemoryCache())

# 获取当前缓存
current_cache = get_llm_cache()

# 禁用缓存
set_llm_cache(None)
```

**特点**：
- 全局配置，所有LLM实例自动使用
- 无需修改业务代码
- 可以动态切换缓存后端

### 缓存生效范围

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

# 设置全局缓存

llm1 = ChatOpenAI(model="gpt-4o-mini")
llm2 = ChatAnthropic(model="claude-3-5-sonnet-20241022")

# 两个实例都会自动缓存
response1 = llm1.invoke("Hello")  # 缓存生效

```

## 4. 缓存命中逻辑

### 命中条件

缓存命中需要满足：
1. **prompt完全相同**（字符串精确匹配）
2. **llm_string完全相同**（配置参数一致）

### 配置参数影响

```python
from langchain_openai import ChatOpenAI

# 相同配置 → 缓存命中
llm1 = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)
llm2 = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)

response1 = llm1.invoke("Hello")  # 首次调用
response2 = llm2.invoke("Hello")  # 缓存命中 ✅

# 不同配置 → 缓存未命中
llm3 = ChatOpenAI(model="gpt-4o-mini", temperature=0.9)
response3 = llm3.invoke("Hello")  # 缓存未命中 ❌
```

### Chat模型的prompt序列化

```python
from langchain_core.messages import HumanMessage, SystemMessage

# 消息列表会被序列化为字符串
messages = [
    SystemMessage(content="You are a helpful assistant."),
    HumanMessage(content="What is AI?")
]

# 序列化后的prompt（简化示例）
# "System: You are a helpful assistant.\nHuman: What is AI?"

# 相同消息 → 缓存命中
response1 = llm.invoke(messages)  # 首次调用
response2 = llm.invoke(messages)  # 缓存命中 ✅

# 消息顺序不同 → 缓存未命中
messages_different = [
    HumanMessage(content="What is AI?"),
    SystemMessage(content="You are a helpful assistant.")
]
response3 = llm.invoke(messages_different)  # 缓存未命中 ❌
```

## 5. 缓存性能分析

### 性能对比

| 场景 | 耗时 | 成本 | 说明 |
|------|------|------|------|
| 首次调用（无缓存） | 2-3秒 | $0.01 | API调用 |
| 缓存命中 | 10-50ms | $0 | 内存读取 |
| 性能提升 | 20-100倍 | 100% | 显著改善 |

### 缓存命中率

**影响因素**：
- 查询重复度
- 配置参数一致性
- 缓存容量限制

**典型命中率**：
- 问答系统：50-70%
- 文档处理：30-50%
- 对话应用：20-40%

### 成本节省计算

```python
# 假设场景
total_requests = 10000  # 总请求数
cache_hit_rate = 0.65   # 缓存命中率65%
cost_per_request = 0.01  # 每次请求成本$0.01

# 无缓存成本
cost_without_cache = total_requests * cost_per_request
print(f"无缓存成本: ${cost_without_cache}")

# 有缓存成本
cache_hits = total_requests * cache_hit_rate
cache_misses = total_requests * (1 - cache_hit_rate)
cost_with_cache = cache_misses * cost_per_request
print(f"有缓存成本: ${cost_with_cache}")

# 节省成本
savings = cost_without_cache - cost_with_cache
savings_percent = (savings / cost_without_cache) * 100
print(f"节省成本: ${savings} ({savings_percent:.1f}%)")

# 输出：
# 无缓存成本: $100.0
# 有缓存成本: $35.0
# 节省成本: $65.0 (65.0%)
```

## 6. 缓存清理策略

### 手动清理

```python
from langchain_core.globals import get_llm_cache

# 获取当前缓存
cache = get_llm_cache()

# 清空所有缓存
if cache:
    cache.clear()
    print("缓存已清空")
```

### FIFO淘汰策略

```python
# InMemoryCache使用FIFO策略
cache = InMemoryCache(maxsize=100)

# 当缓存满时，删除最早的条目
# 第101次插入时，删除第1次插入的条目
```

### 定期清理

```python
import schedule
import time

def clear_cache():
    """定期清理缓存"""
    cache = get_llm_cache()
    if cache:
        cache.clear()
        print(f"缓存已清理: {time.strftime('%Y-%m-%d %H:%M:%S')}")

# 每天凌晨2点清理缓存
schedule.every().day.at("02:00").do(clear_cache)

# 运行调度器
while True:
    schedule.run_pending()
    time.sleep(60)
```

## 7. 异步支持

### 异步API

```python
from langchain_core.caches import InMemoryCache
from langchain_openai import ChatOpenAI
import asyncio

async def async_cache_demo():
    """异步缓存示例"""
    # 设置缓存
    set_llm_cache(InMemoryCache())

    llm = ChatOpenAI(model="gpt-4o-mini")

    # 异步调用（自动使用缓存）
    response1 = await llm.ainvoke("What is AI?")
    response2 = await llm.ainvoke("What is AI?")  # 缓存命中

    print("异步缓存工作正常")

# 运行异步函数
asyncio.run(async_cache_demo())
```

**特点**：
- InMemoryCache的异步方法直接调用同步方法
- 其他缓存后端（如Redis）有真正的异步实现
- 无需修改代码，自动支持异步

### 场景3：多轮对话

## 9. 最佳实践

### 1. 保持配置一致性

```python
# ✅ 推荐：使用固定配置
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7,
    max_tokens=1000
)

# ❌ 避免：动态配置
temperature = random.uniform(0.5, 1.0)  # 每次不同
llm = ChatOpenAI(model="gpt-4o-mini", temperature=temperature)
```

### 2. 合理设置maxsize

```python
# 根据应用场景设置
set_llm_cache(InMemoryCache(maxsize=1000))  # 中等规模
set_llm_cache(InMemoryCache(maxsize=10000))  # 大规模
set_llm_cache(InMemoryCache())  # 无限制（注意内存）
```

### 3. 监控缓存效果

```python
class MonitoredCache(InMemoryCache):
    """带监控的缓存"""

    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.hits = 0
        self.misses = 0

    def lookup(self, prompt: str, llm_string: str):
        result = super().lookup(prompt, llm_string)
        if result is not None:
            self.hits += 1
        else:
            self.misses += 1
        return result

    def get_hit_rate(self) -> float:
        """计算命中率"""
        total = self.hits + self.misses
        return self.hits / total if total > 0 else 0.0

# 使用监控缓存
cache = MonitoredCache(maxsize=100)
set_llm_cache(cache)

# ... 使用LLM ...

# 查看统计
print(f"缓存命中率: {cache.get_hit_rate():.2%}")
print(f"命中次数: {cache.hits}")
print(f"未命中次数: {cache.misses}")
```

### 4. 定期清理

```python
# 定期清理过期缓存
import time

last_clear_time = time.time()
CLEAR_INTERVAL = 3600  # 1小时

def maybe_clear_cache():
    """定期清理缓存"""
    global last_clear_time
    if time.time() - last_clear_time > CLEAR_INTERVAL:
        cache = get_llm_cache()
        if cache:
            cache.clear()
        last_clear_time = time.time()
```

## 10. 常见问题

### Q1: 为什么缓存没有命中？

**可能原因**：
1. prompt不完全相同（空格、换行等）
2. llm_string不同（配置参数变化）
3. 消息ID影响（Chat模型）
4. 缓存已被清空或淘汰

**解决方案**：
- 规范化prompt（去除多余空格）
- 固定LLM配置参数
- 检查消息序列化逻辑

### Q2: InMemoryCache适合生产环境吗？

**不适合**，原因：
- 不持久化（重启丢失）
- 不支持分布式
- 内存泄漏风险

**生产环境推荐**：
- 使用RedisCache
- 配置持久化
- 监控缓存性能

### Q3: 如何处理缓存过期？

InMemoryCache不支持TTL（过期时间），需要：
- 定期手动清理
- 使用支持TTL的后端（Redis）
- 实现自定义缓存类
