# 核心概念 4：错误处理机制

## 概述

LangGraph 节点函数的错误处理机制是构建健壮工作流的关键。本文档深入讲解异常捕获、RetryPolicy 配置、错误传播和流式错误处理的核心机制。

## 1. 异常捕获机制

### 1.1 基础异常捕获

节点函数中的异常捕获遵循 Python 标准的 try-except 模式：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    result: str
    error: str | None

def my_node(state: State) -> dict:
    """带异常捕获的节点函数"""
    try:
        # 可能抛出异常的操作
        result = risky_operation()
        return {"result": result, "error": None}
    except ValueError as e:
        # 捕获特定异常
        return {"result": "", "error": f"ValueError: {str(e)}"}
    except Exception as e:
        # 捕获所有其他异常
        return {"result": "", "error": f"Unexpected error: {str(e)}"}
```

[来源: reference/context7_langgraph_01.md]

### 1.2 异常传播

如果节点函数不捕获异常，异常会向上传播到图执行层：

```python
def failing_node(state: State) -> dict:
    """不捕获异常的节点"""
    # 异常会传播到图执行层
    raise ValueError("Something went wrong")

# 图执行时会抛出异常
try:
    result = graph.invoke({"input": "test"})
except ValueError as e:
    print(f"Graph execution failed: {e}")
```

**关键点**：
- 未捕获的异常会中断图执行
- 异常会附加节点名称和 ID 信息
- 可以在图层面统一处理异常

[来源: reference/source_节点函数约定_02.md]

### 1.3 特殊异常处理

LangGraph 有几种特殊异常需要特别处理：

```python
import asyncio
from langgraph.errors import GraphBubbleUp
from langgraph.types import Command

async def special_error_node(state: State) -> dict:
    """处理特殊异常"""
    try:
        result = await some_async_operation()
        return {"result": result}
    except asyncio.CancelledError:
        # 异步取消错误需要重新抛出
        print("Task was cancelled")
        raise
    except GraphBubbleUp:
        # 图冒泡异常不应被捕获
        raise
    except Exception as e:
        # 其他异常可以正常处理
        return {"error": str(e)}
```

**特殊异常类型**：
1. `asyncio.CancelledError` - 异步任务取消
2. `GraphBubbleUp` - 图层面的控制流异常
3. `ParentCommand` - 父图通信异常

## 2. RetryPolicy 配置

### 2.1 RetryPolicy 结构

RetryPolicy 定义了节点失败时的重试行为：

```python
from langgraph.types import RetryPolicy

retry_policy = RetryPolicy(
    max_attempts=3,           # 最大重试次数
    initial_interval=1.0,     # 初始重试间隔（秒）
    backoff_factor=2.0,       # 退避因子（指数退避）
    max_interval=10.0,        # 最大重试间隔（秒）
    jitter=True,              # 是否添加随机抖动
    retry_on=Exception        # 重试条件
)
```

[来源: reference/source_节点函数约定_02.md, reference/context7_langgraph_01.md]

### 2.2 指数退避算法

RetryPolicy 使用指数退避算法计算重试间隔：

```python
# 伪代码展示退避算法
interval = initial_interval
for attempt in range(1, max_attempts):
    # 计算退避间隔
    interval = min(
        max_interval,
        initial_interval * (backoff_factor ** (attempt - 1))
    )

    # 添加抖动（如果启用）
    if jitter:
        sleep_time = interval + random.uniform(0, 1)
    else:
        sleep_time = interval

    # 等待后重试
    await asyncio.sleep(sleep_time)
```

**退避示例**（initial_interval=1.0, backoff_factor=2.0）：
- 第 1 次重试：1.0 秒
- 第 2 次重试：2.0 秒
- 第 3 次重试：4.0 秒
- 第 4 次重试：8.0 秒
- 第 5 次重试：10.0 秒（达到 max_interval）

[来源: reference/source_节点函数约定_02.md]

### 2.3 重试条件配置

RetryPolicy 支持三种重试条件：

#### 方式 1：单个异常类型

```python
retry_policy = RetryPolicy(
    max_attempts=3,
    retry_on=ValueError  # 只重试 ValueError
)
```

#### 方式 2：多个异常类型

```python
retry_policy = RetryPolicy(
    max_attempts=3,
    retry_on=[ValueError, TypeError, ConnectionError]  # 重试多种异常
)
```

#### 方式 3：自定义判断函数

```python
def retry_on_rate_limit(error: Exception) -> bool:
    """自定义重试条件"""
    return "rate limit" in str(error).lower()

retry_policy = RetryPolicy(
    max_attempts=5,
    retry_on=retry_on_rate_limit  # 使用自定义函数
)
```

[来源: reference/source_节点函数约定_02.md, reference/context7_langgraph_01.md]

### 2.4 节点级别配置

在添加节点时配置 RetryPolicy：

```python
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy

builder = StateGraph(State)

# 为单个节点配置重试策略
builder.add_node(
    "api_call",
    unreliable_api_call,
    retry_policy=RetryPolicy(
        max_attempts=3,
        initial_interval=1.0,
        backoff_factor=2.0,
        max_interval=10.0,
        jitter=True,
        retry_on=lambda e: isinstance(e, (ConnectionError, TimeoutError))
    )
)

builder.add_edge(START, "api_call")
builder.add_edge("api_call", END)

graph = builder.compile()
```

## 3. 错误传播

### 3.1 错误传播流程

错误在 LangGraph 中的传播路径：

```
节点函数异常
    ↓
检查 RetryPolicy
    ↓
是否应该重试？
    ├─ 是 → 等待 → 重试节点
    └─ 否 → 附加节点信息 → 向上传播
        ↓
    图执行层捕获
        ↓
    用户代码处理
```

[来源: reference/source_节点函数约定_02.md]

### 3.2 错误信息增强

LangGraph 会自动为异常添加上下文信息：

```python
# 源码中的错误增强逻辑
try:
    return task.proc.invoke(task.input, config)
except Exception as exc:
    # 添加节点信息到异常
    if SUPPORTS_EXC_NOTES:
        exc.add_note(f"During task with name '{task.name}' and id '{task.id}'")
    raise
```

**错误信息示例**：
```
ValueError: Invalid input
During task with name 'process_data' and id 'abc123'
```

[来源: reference/source_节点函数约定_02.md]

### 3.3 状态中的错误标志

推荐在状态中添加错误标志来传播错误信息：

```python
class State(TypedDict):
    data: str
    error: str | None
    error_count: int

def node_with_error_flag(state: State) -> dict:
    """使用状态标志传播错误"""
    try:
        result = process_data(state["data"])
        return {"data": result, "error": None}
    except Exception as e:
        return {
            "error": str(e),
            "error_count": state.get("error_count", 0) + 1
        }

def error_handler_node(state: State) -> dict:
    """根据错误标志决策"""
    if state.get("error"):
        # 处理错误情况
        return {"data": "fallback_value", "error": None}
    return state
```

## 4. 流式错误处理

### 4.1 流式执行中的错误

流式执行（streaming）中的错误处理需要特别注意：

```python
async def stream_with_error_handling(graph, input_data):
    """流式执行中的错误处理"""
    try:
        async for event in graph.astream_events(input_data):
            try:
                # 处理每个事件
                if event["event"] == "on_chain_end":
                    print(f"Node completed: {event['name']}")
                elif event["event"] == "on_chain_error":
                    print(f"Node failed: {event['name']}, Error: {event['error']}")
            except Exception as e:
                # 在循环内处理错误
                print(f"Error processing event: {e}")
                # 决定是否继续
                continue
    except Exception as e:
        # 捕获整体流式执行错误
        print(f"Stream execution failed: {e}")
```

**关键点**：
- 在 `for await` 循环内处理错误
- 区分事件处理错误和流式执行错误
- 使用 `on_chain_error` 事件监听节点错误

[来源: reference/search_节点函数约定_01.md]

### 4.2 流式错误的常见问题

根据社区讨论，流式错误处理有几个常见问题：

#### 问题 1：try-catch 在 for await 外层失效

```python
# ❌ 错误做法
try:
    async for event in graph.astream_events(input_data):
        process_event(event)  # 错误不会被外层 try 捕获
except Exception as e:
    print(f"Error: {e}")  # 可能不会执行

# ✅ 正确做法
async for event in graph.astream_events(input_data):
    try:
        process_event(event)
    except Exception as e:
        print(f"Error: {e}")
```

#### 问题 2：ToolNode 错误未通过 on_tool_end 通知

```python
# 需要监听 on_chain_error 事件
async for event in graph.astream_events(input_data):
    if event["event"] == "on_chain_error":
        # 处理工具节点错误
        print(f"Tool error: {event['error']}")
```

[来源: reference/search_节点函数约定_01.md]

### 4.3 异步生成器节点的问题

根据社区反馈，异步生成器节点可能存在问题：

```python
# ⚠️ 不推荐：异步生成器节点
async def async_generator_node(state: State):
    """可能有问题的异步生成器"""
    async for item in some_async_generator():
        yield {"item": item}

# ✅ 推荐：普通异步函数
async def async_node(state: State) -> dict:
    """推荐的异步节点"""
    items = []
    async for item in some_async_generator():
        items.append(item)
    return {"items": items}
```

## 5. 完整代码示例

### 示例 1：基础错误处理

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy
import random

class State(TypedDict):
    result: str
    attempts: int
    error: str | None

def unreliable_node(state: State) -> dict:
    """模拟不可靠的节点"""
    attempts = state.get("attempts", 0) + 1

    # 50% 概率失败
    if random.random() < 0.5:
        raise ConnectionError("Network error")

    return {
        "result": "Success!",
        "attempts": attempts,
        "error": None
    }

# 构建图
builder = StateGraph(State)
builder.add_node(
    "unreliable",
    unreliable_node,
    retry_policy=RetryPolicy(
        max_attempts=3,
        initial_interval=0.5,
        backoff_factor=2.0,
        max_interval=5.0,
        jitter=True,
        retry_on=ConnectionError
    )
)
builder.add_edge(START, "unreliable")
builder.add_edge("unreliable", END)

graph = builder.compile()

# 执行
try:
    result = graph.invoke({"result": "", "attempts": 0, "error": None})
    print(f"Result: {result['result']}, Attempts: {result['attempts']}")
except Exception as e:
    print(f"Failed after retries: {e}")
```

### 示例 2：自定义重试条件

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy
import httpx

class APIState(TypedDict):
    url: str
    response: str | None
    status_code: int | None
    error: str | None

def retry_on_rate_limit_or_server_error(error: Exception) -> bool:
    """自定义重试条件：只重试速率限制和服务器错误"""
    error_str = str(error).lower()
    return (
        "rate limit" in error_str or
        "429" in error_str or
        "500" in error_str or
        "503" in error_str
    )

async def api_call_node(state: APIState) -> dict:
    """调用外部 API"""
    async with httpx.AsyncClient() as client:
        response = await client.get(state["url"])
        response.raise_for_status()
        return {
            "response": response.text,
            "status_code": response.status_code,
            "error": None
        }

# 构建图
builder = StateGraph(APIState)
builder.add_node(
    "api_call",
    api_call_node,
    retry_policy=RetryPolicy(
        max_attempts=5,
        initial_interval=1.0,
        backoff_factor=2.0,
        max_interval=30.0,
        jitter=True,
        retry_on=retry_on_rate_limit_or_server_error
    )
)
builder.add_edge(START, "api_call")
builder.add_edge("api_call", END)

graph = builder.compile()
```

### 示例 3：错误恢复流程

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class ProcessState(TypedDict):
    data: str
    processed: str | None
    error: str | None
    retry_count: int

def process_node(state: ProcessState) -> dict:
    """主处理节点"""
    try:
        # 模拟处理逻辑
        if not state["data"]:
            raise ValueError("Empty data")

        processed = state["data"].upper()
        return {
            "processed": processed,
            "error": None
        }
    except Exception as e:
        return {
            "error": str(e),
            "retry_count": state.get("retry_count", 0) + 1
        }

def error_handler_node(state: ProcessState) -> dict:
    """错误处理节点"""
    if state.get("error"):
        # 使用默认值恢复
        return {
            "processed": "DEFAULT_VALUE",
            "error": None
        }
    return state

def should_retry(state: ProcessState) -> str:
    """决定是否重试"""
    if state.get("error") and state.get("retry_count", 0) < 3:
        return "process"
    elif state.get("error"):
        return "error_handler"
    else:
        return "end"

# 构建图
builder = StateGraph(ProcessState)
builder.add_node("process", process_node)
builder.add_node("error_handler", error_handler_node)

builder.add_edge(START, "process")
builder.add_conditional_edges(
    "process",
    should_retry,
    {
        "process": "process",
        "error_handler": "error_handler",
        "end": END
    }
)
builder.add_edge("error_handler", END)

graph = builder.compile()

# 测试
result = graph.invoke({
    "data": "",
    "processed": None,
    "error": None,
    "retry_count": 0
})
print(f"Processed: {result['processed']}")
```

### 示例 4：流式错误处理

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import asyncio

class StreamState(TypedDict):
    items: list[str]
    processed: list[str]

async def stream_processor_node(state: StreamState) -> dict:
    """流式处理节点"""
    processed = []
    for item in state["items"]:
        try:
            # 模拟处理
            await asyncio.sleep(0.1)
            processed.append(item.upper())
        except Exception as e:
            # 记录错误但继续处理
            processed.append(f"ERROR: {item}")

    return {"processed": processed}

# 构建图
builder = StateGraph(StreamState)
builder.add_node("processor", stream_processor_node)
builder.add_edge(START, "processor")
builder.add_edge("processor", END)

graph = builder.compile()

# 流式执行
async def run_with_streaming():
    input_data = {
        "items": ["item1", "item2", "item3"],
        "processed": []
    }

    try:
        async for event in graph.astream_events(input_data, version="v2"):
            try:
                if event["event"] == "on_chain_start":
                    print(f"Started: {event['name']}")
                elif event["event"] == "on_chain_end":
                    print(f"Completed: {event['name']}")
                elif event["event"] == "on_chain_error":
                    print(f"Error in {event['name']}: {event.get('error')}")
            except Exception as e:
                print(f"Error processing event: {e}")
                continue
    except Exception as e:
        print(f"Stream failed: {e}")

# 运行
asyncio.run(run_with_streaming())
```

## 6. 最佳实践

### 6.1 错误处理策略

1. **分层错误处理**
   - 节点内部：捕获预期异常，返回错误状态
   - RetryPolicy：处理临时性错误（网络、速率限制）
   - 图层面：处理致命错误，提供降级方案

2. **错误信息记录**
   ```python
   import logging

   logger = logging.getLogger(__name__)

   def node_with_logging(state: State) -> dict:
       try:
           result = risky_operation()
           return {"result": result}
       except Exception as e:
           logger.error(f"Node failed: {e}", exc_info=True)
           return {"error": str(e)}
   ```

3. **错误恢复机制**
   - 使用条件边实现错误恢复流程
   - 提供降级方案（fallback）
   - 限制重试次数避免无限循环

### 6.3 避免的常见错误

1. **不要捕获所有异常**
   ```python
   # ❌ 错误
   try:
       result = operation()
   except:  # 捕获所有异常包括 KeyboardInterrupt
       pass

   # ✅ 正确
   try:
       result = operation()
   except Exception as e:  # 只捕获 Exception 及其子类
       handle_error(e)
   ```

2. **不要忽略 CancelledError**
   ```python
   # ❌ 错误
   try:
       await async_operation()
   except Exception:  # 会捕获 CancelledError
       pass

   # ✅ 正确
   try:
       await async_operation()
   except asyncio.CancelledError:
       raise  # 重新抛出
   except Exception as e:
       handle_error(e)
   ```

3. **不要在流式执行外层捕获错误**
   ```python
   # ❌ 错误
   try:
       async for event in graph.astream_events(input):
           process(event)
   except Exception:
       pass  # 可能不会捕获到事件处理错误

   # ✅ 正确
   async for event in graph.astream_events(input):
       try:
           process(event)
       except Exception as e:
           handle_error(e)
   ```

## 7. 总结

LangGraph 的错误处理机制提供了多层次的错误管理能力：

1. **节点级别**：使用 try-except 捕获预期异常
2. **重试机制**：通过 RetryPolicy 处理临时性错误
3. **错误传播**：未捕获异常向上传播到图层面
4. **流式处理**：在事件循环内处理流式错误

**关键要点**：
- 合理配置 RetryPolicy 参数（max_attempts, backoff_factor, jitter）
- 使用自定义重试条件精确控制重试行为
- 在状态中添加错误标志实现错误传播
- 流式执行中在循环内处理错误
- 特殊异常（CancelledError, GraphBubbleUp）需要特殊处理

通过正确使用这些错误处理机制，可以构建健壮、可靠的 LangGraph 工作流。
