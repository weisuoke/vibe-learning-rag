## 1. 标量过滤

### 1.1 前端开发类比

**JavaScript 数组过滤**：
```javascript
// JavaScript
const products = [
  { id: 1, price: 99, stock: 100, brand: 'Apple' },
  { id: 2, price: 199, stock: 50, brand: 'Samsung' },
  { id: 3, price: 299, stock: 0, brand: 'Huawei' }
];

// 过滤：价格100-200，有库存
const filtered = products.filter(p =>
  p.price >= 100 && p.price <= 200 && p.stock > 0
);

// Milvus 标量过滤（等价）
filter = "price >= 100 AND price <= 200 AND stock > 0"
```

**SQL WHERE 子句**：
```sql
-- SQL
SELECT * FROM products
WHERE price >= 100
  AND price <= 200
  AND stock > 0;

-- Milvus
filter = "price >= 100 AND price <= 200 AND stock > 0"
```

### 1.2 日常生活类比

**餐厅筛选**：
```
找餐厅的条件：
- 价格：100-500元（标量过滤：price >= 100 AND price <= 500）
- 距离：< 3km（标量过滤：distance < 3）
- 营业中（标量过滤：status == 'open'）
- 口味相似（向量检索：similarity）

组合：
"在满足价格、距离、营业状态的餐厅中，找口味最相似的"
= 标量过滤 + 向量检索
```

**购物筛选**：
```
在电商平台买手机：
- 品牌：Apple 或 Samsung（标量过滤：brand IN ['Apple', 'Samsung']）
- 价格：5000-8000元（标量过滤：price >= 5000 AND price <= 8000）
- 有库存（标量过滤：stock > 0）
- 外观相似（向量检索：similarity）

组合：
"在满足品牌、价格、库存的手机中，找外观最相似的"
= 标量过滤 + 向量检索
```

## 2. 布尔运算符

### 2.1 前端开发类比

**AND 操作符 = 多条件筛选**：
```javascript
// JavaScript
const result = users.filter(u =>
  u.age > 25 &&
  u.city === '北京' &&
  u.active === true
);

// Milvus
filter = "age > 25 AND city == '北京' AND active == true"
```

**OR 操作符 = 多选一**：
```javascript
// JavaScript
const result = users.filter(u =>
  u.city === '北京' ||
  u.city === '上海' ||
  u.city === '深圳'
);

// Milvus
filter = "city IN ['北京', '上海', '深圳']"
```

**NOT 操作符 = 排除**：
```javascript
// JavaScript
const result = users.filter(u =>
  u.status !== 'deleted'
);

// Milvus
filter = "status != 'deleted'"
```

### 2.2 日常生活类比

**AND = 同时满足**：
```
找工作的条件：
- 薪资 > 20k（必须）
- 地点在北京（必须）
- 五险一金（必须）

所有条件都要满足 = AND
```

**OR = 满足任一**：
```
交通方式选择：
- 地铁
- 公交
- 打车

任选一种即可 = OR
```

**NOT = 排除**：
```
找房子的条件：
- 不要顶楼（NOT floor == 'top'）
- 不要一楼（NOT floor == 1）
- 不要临街（NOT location == 'street'）

排除不想要的 = NOT
```

## 3. JSON Path

### 3.1 前端开发类比

**JavaScript 对象访问**：
```javascript
// JavaScript
const product = {
  metadata: {
    product: {
      category: 'electronics',
      brand: 'Apple'
    },
    pricing: {
      base: 999.99,
      final: 899.99
    },
    tags: ['hot', 'new', '5G']
  }
};

// 访问嵌套属性
const category = product.metadata.product.category;
const price = product.metadata.pricing.final;
const firstTag = product.metadata.tags[0];

// Milvus JSON Path（语法相同）
"metadata['product']['category']"
"metadata['pricing']['final']"
"metadata['tags'][0]"
```

**Python 字典访问**：
```python
# Python
product = {
    'metadata': {
        'product': {
            'category': 'electronics',
            'brand': 'Apple'
        }
    }
}

# 访问嵌套字典
category = product['metadata']['product']['category']

# Milvus JSON Path（语法相同）
"metadata['product']['category']"
```

### 3.2 日常生活类比

**商品详情页**：
```
商品信息结构：
├── 基本信息
│   ├── 名称：iPhone 15
│   ├── 品牌：Apple
│   └── 类别：电子产品
├── 价格信息
│   ├── 原价：999.99
│   └── 现价：899.99
└── 标签
    ├── 热销
    ├── 新品
    └── 5G

访问路径：
- 品牌：基本信息 → 品牌 = metadata['product']['brand']
- 现价：价格信息 → 现价 = metadata['pricing']['final']
- 第一个标签：标签 → 第一个 = metadata['tags'][0]
```

**文件系统路径**：
```
文件路径：
/Users/wuxiao/Documents/project/README.md

类比 JSON Path：
metadata['user']['documents']['project']['README.md']

都是层层访问，逐级深入
```

## 4. JSON 索引

### 4.1 前端开发类比

**无索引 = 数组查找**：
```javascript
// 无索引：线性查找
const result = products.find(p => p.id === 123);
// O(n) 时间复杂度，需要遍历所有元素
```

**有索引 = Map 查找**：
```javascript
// 有索引：哈希表查找
const productsById = new Map();
productsById.set(123, product);
const result = productsById.get(123);
// O(1) 时间复杂度，直接定位
```

**性能对比**：
```javascript
// 100万条数据
// 无索引：1000ms
// 有索引：1ms
// 性能提升：1000倍
```

### 4.2 日常生活类比

**图书馆找书**：
```
无索引（JsonFlatIndex）：
- 逐本翻书找内容
- 时间：1小时
- 效率：低

有索引（JsonInvertedIndex）：
- 查目录直接定位
- 时间：1分钟
- 效率：高

性能提升：60倍
```

**超市购物**：
```
无索引：
- 在整个超市逐个货架找商品
- 时间：30分钟

有索引：
- 查看分类标识，直接到对应区域
- 时间：3分钟

性能提升：10倍
```

## 5. 索引类型

### 5.1 前端开发类比

**INVERTED 索引 = Map**：
```javascript
// INVERTED 索引
const categoryIndex = new Map();
categoryIndex.set('electronics', [1, 5, 10, 15]);
categoryIndex.set('books', [2, 6, 11, 16]);

// 查询
const result = categoryIndex.get('electronics');
// O(1) 时间复杂度
```

**BITMAP 索引 = Set**：
```javascript
// BITMAP 索引（低基数）
const activeUsers = new Set([1, 3, 5, 7, 9]);
const inactiveUsers = new Set([2, 4, 6, 8, 10]);

// 查询
const isActive = activeUsers.has(5);
// O(1) 时间复杂度
```

**STL_SORT 索引 = 排序数组 + 二分查找**：
```javascript
// STL_SORT 索引
const sortedAges = [18, 20, 25, 30, 35, 40, 45, 50];

// 二分查找
function binarySearch(arr, target) {
  let left = 0, right = arr.length - 1;
  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if (arr[mid] === target) return mid;
    if (arr[mid] < target) left = mid + 1;
    else right = mid - 1;
  }
  return -1;
}
// O(log n) 时间复杂度
```

### 5.2 日常生活类比

**INVERTED 索引 = 电话簿**：
```
电话簿：
- 按姓名排序
- 快速查找某个人的电话
- 适用于精确匹配

Milvus INVERTED：
- 按值排序
- 快速查找某个值的文档
- 适用于精确匹配和范围查询
```

**BITMAP 索引 = 座位图**：
```
电影院座位图：
- 已售：红色
- 可售：绿色
- 只有2种状态（低基数）

Milvus BITMAP：
- 适用于低基数字段
- 内存占用小
- 查询速度快
```

**TRIE 索引 = 字典**：
```
字典查词：
- 按首字母分类
- 快速找到以某字母开头的词
- 适用于前缀匹配

Milvus TRIE：
- 适用于前缀匹配
- 如：name LIKE 'John%'
```

## 6. 混合查询执行策略

### 6.1 前端开发类比

**Pre-filtering = 先筛选再排序**：
```javascript
// Pre-filtering
const filtered = products.filter(p => p.stock > 0);  // 先筛选
const sorted = filtered.sort((a, b) => similarity(a) - similarity(b));  // 再排序
const top10 = sorted.slice(0, 10);

// 适用场景：筛选后数据量小（高选择性）
```

**Post-filtering = 先排序再筛选**：
```javascript
// Post-filtering
const sorted = products.sort((a, b) => similarity(a) - similarity(b));  // 先排序
const top100 = sorted.slice(0, 100);  // 取前100
const filtered = top100.filter(p => p.stock > 0);  // 再筛选
const top10 = filtered.slice(0, 10);

// 适用场景：筛选后数据量大（低选择性）
```

### 6.2 日常生活类比

**Pre-filtering = 先筛选简历再面试**：
```
招聘流程（高选择性）：
1. 筛选简历：1000份 → 100份（选择性90%）
2. 面试：100人 → 10人

优势：减少面试成本
劣势：可能错过优秀候选人
```

**Post-filtering = 先面试再筛选**：
```
招聘流程（低选择性）：
1. 面试：1000人 → 100人
2. 筛选：100人 → 10人（选择性10%）

优势：不会错过优秀候选人
劣势：面试成本高
```

## 7. 性能优化

### 7.1 前端开发类比

**索引优化 = 缓存**：
```javascript
// 无缓存（无索引）
function getUser(id) {
  return users.find(u => u.id === id);  // O(n)
}

// 有缓存（有索引）
const userCache = new Map();
function getUser(id) {
  return userCache.get(id);  // O(1)
}

// 性能提升：n 倍
```

**查询优化 = 短路求值**：
```javascript
// 不好的写法
if (expensiveCheck1() && expensiveCheck2() && cheapCheck()) {
  // ...
}

// 好的写法（选择性高的条件在前）
if (cheapCheck() && expensiveCheck1() && expensiveCheck2()) {
  // ...
}

// 如果 cheapCheck() 为 false，后面的不会执行
```

### 7.2 日常生活类比

**索引优化 = 商品分类**：
```
无分类（无索引）：
- 所有商品混在一起
- 找一件商品需要翻遍整个仓库
- 时间：1小时

有分类（有索引）：
- 按类别分区存放
- 直接到对应区域找
- 时间：5分钟

性能提升：12倍
```

**查询优化 = 安检流程**：
```
不好的流程：
1. 详细检查行李（慢）
2. 检查身份证（快）
3. 检查票（快）

好的流程（选择性高的在前）：
1. 检查票（快，可能没票）
2. 检查身份证（快，可能证件不符）
3. 详细检查行李（慢）

如果前面不通过，后面不需要检查
```

## 8. 常见误区

### 8.1 前端开发类比

**误区1："索引越多越好"**：
```javascript
// 错误：为所有字段创建索引
const indexes = {
  id: new Map(),
  name: new Map(),
  age: new Map(),
  email: new Map(),
  phone: new Map(),
  // ... 100个字段
};

// 问题：
// 1. 内存占用大
// 2. 更新成本高
// 3. 大部分索引用不到

// 正确：只为常用字段创建索引
const indexes = {
  id: new Map(),  // 主键，必须
  email: new Map(),  // 常用查询
};
```

**误区2："JSON 查询一定很慢"**：
```javascript
// 错误认知：JSON 查询慢
const result = data.find(d =>
  d.metadata.product.category === 'electronics'
);
// 认为嵌套访问慢

// 正确认知：有索引就快
const categoryIndex = new Map();
const result = categoryIndex.get('electronics');
// 有索引，查询很快
```

### 8.2 日常生活类比

**误区1："索引越多越好"**：
```
图书馆索引：
- 按书名索引（必须）
- 按作者索引（常用）
- 按出版社索引（常用）
- 按页数索引（不常用，浪费）
- 按纸张类型索引（不常用，浪费）

结论：只为常用查询创建索引
```

**误区2："JSON 查询一定很慢"**：
```
商品详情查询：
- 无索引：翻遍所有商品详情页（慢）
- 有索引：查目录直接定位（快）

结论：有索引就快，无索引才慢
```

```

**JSON Path**：
```
JSON → 嵌套 → 路径 → 访问
Path → 层级 → 导航 → ['key']

```
索引 → 目录 → 快速 → 查找
优化 → 性能 → 提升 → 30-100倍
```
