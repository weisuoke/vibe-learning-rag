# 核心概念:类型修饰符

> LangGraph 状态类型系统中的字段必需性与可变性控制

---

## 概述

类型修饰符用于精确控制 TypedDict 字段的必需性和可变性。LangGraph 支持 `Required`、`NotRequired`、`ReadOnly` 和 `Optional` 等修饰符，提供灵活的状态字段定义。

**核心价值**:
- 精确控制字段必需性
- 提高类型安全性
- 支持只读字段保护
- 与 Python 类型系统无缝集成

---

## 1. Required / NotRequired

### 1.1 基础概念

`Required` 和 `NotRequired` 用于显式标记 TypedDict 字段是否必需。

**引入版本**:
- Python 3.11+ 的 `typing` 模块
- `typing_extensions` 4.0.0+（向后兼容）
- 关联 PEP 655

[来源: reference/context7_typing_extensions_01.md | typing_extensions 官方文档]

### 1.2 基础用法

```python
from typing_extensions import TypedDict, Required, NotRequired

# 方式 1: 使用 total=True（默认）
class State1(TypedDict):
    name: str  # 必需
    age: int  # 必需
    email: NotRequired[str]  # 可选

# 方式 2: 使用 total=False
class State2(TypedDict, total=False):
    name: Required[str]  # 必需
    age: Required[int]  # 必需
    email: str  # 可选
```

**关键特性**:
- `total=True`（默认）: 所有字段必需，除非标记 `NotRequired`
- `total=False`: 所有字段可选，除非标记 `Required`
- 显式标记优先级高于 `total` 参数

### 1.3 在 LangGraph 中的应用

```python
from typing_extensions import TypedDict, NotRequired, Annotated
from langgraph.graph import StateGraph

def add_messages(left: list, right: list | None) -> list:
    if right is None:
        return left
    return left + right

class ChatState(TypedDict):
    messages: Annotated[list[str], add_messages]  # 必需
    user_id: str  # 必需
    session_id: NotRequired[str]  # 可选
    metadata: NotRequired[dict]  # 可选

graph = StateGraph(ChatState)

# 使用示例
def node1(state: ChatState) -> dict:
    # 只需提供必需字段
    return {"messages": ["Hello"]}

def node2(state: ChatState) -> dict:
    # 可以提供可选字段
    return {
        "messages": ["World"],
        "session_id": "abc123"
    }
```

### 1.4 源码实现

LangGraph 通过 `_is_required_type` 函数检测字段必需性:

```python
def _is_required_type(type_: Any) -> bool | None:
    """检查注解是否标记为 Required/NotRequired"""
    origin = get_origin(type_)
    if origin is Required:
        return True
    if origin is NotRequired:
        return False
    if origin is Annotated or getattr(origin, "__args__", None):
        return _is_required_type(type_.__args__[0])
    return None
```

**检查逻辑**:
1. 检查类型的 origin 是否为 `Required` 或 `NotRequired`
2. 递归处理嵌套的 `Annotated` 类型
3. 返回 `True`（必需）、`False`（可选）或 `None`（未标记）

## 2. ReadOnly

### 2.1 基础概念

`ReadOnly` 表示 TypedDict 字段初始化后不能修改。

**引入版本**:
- Python 3.13+ 的 `typing` 模块
- `typing_extensions` 4.9.0+
- 关联 PEP 705

[来源: reference/context7_typing_extensions_01.md | typing_extensions 官方文档]

### 2.2 基础用法

```python
from typing_extensions import TypedDict, ReadOnly

class Config(TypedDict):
    api_key: ReadOnly[str]  # 只读
    endpoint: ReadOnly[str]  # 只读
    timeout: int  # 可变

# 使用示例
config: Config = {
    "api_key": "secret",
    "endpoint": "https://api.example.com",
    "timeout": 30
}

# 类型检查器会警告（运行时不强制）
# config["api_key"] = "new_secret"  # 类型错误
config["timeout"] = 60  # OK
```

### 2.3 特殊属性

TypedDict 使用 `__readonly_keys__` 和 `__mutable_keys__` 跟踪字段:

```python
from typing_extensions import TypedDict, ReadOnly

class State(TypedDict):
    user_id: ReadOnly[str]
    name: str

# 内部属性
print(State.__readonly_keys__)  # frozenset({'user_id'})
print(State.__mutable_keys__)  # frozenset({'name'})
```

[来源: reference/context7_typing_extensions_01.md | typing_extensions 官方文档]

### 2.4 在 LangGraph 中的应用

```python
from typing_extensions import TypedDict, ReadOnly, Annotated
from langgraph.graph import StateGraph

def add_messages(left: list, right: list | None) -> list:
    if right is None:
        return left
    return left + right

class State(TypedDict):
    user_id: ReadOnly[str]  # 只读（初始化后不变）
    messages: Annotated[list[str], add_messages]  # 可变（累积）
    counter: int  # 可变（覆盖）

graph = StateGraph(State)

def node1(state: State) -> dict:
    # 不应修改 user_id（类型检查器会警告）
    return {"messages": ["Hello"], "counter": 1}

def node2(state: State) -> dict:
    # 可以修改 messages 和 counter
    return {"messages": ["World"], "counter": 2}
```

**注意**: LangGraph 不在运行时强制 `ReadOnly`，仅作为类型提示。

### 2.5 源码实现

```python
def _is_readonly_type(type_: Any) -> bool:
    """检查注解是否标记为 ReadOnly"""
    origin = get_origin(type_)
    if origin is Annotated:
        return _is_readonly_type(type_.__args__[0])
    if origin is ReadOnly:
        return True
    return False
```

## 3. Optional / Union[T, None]

### 3.1 基础概念

`Optional[T]` 等价于 `Union[T, None]` 或 `T | None`（Python 3.10+），表示值可以是 `T` 或 `None`。

**语法演变**:
```python
from typing import Optional, Union

# Python 3.9-
x: Optional[str]  # 推荐
y: Union[str, None]  # 等价

# Python 3.10+
z: str | None  # 推荐（PEP 604）
```

### 3.2 与 NotRequired 的区别

```python
from typing_extensions import TypedDict, NotRequired

class State1(TypedDict):
    # NotRequired: 字段可以不存在
    name: NotRequired[str]

class State2(TypedDict):
    # Optional: 字段必须存在，但值可以是 None
    name: str | None

# 使用对比
state1: State1 = {}  # OK，name 不存在
state2: State2 = {"name": None}  # OK，name 存在但为 None
# state2_invalid: State2 = {}  # 错误，name 必须存在
```

**关键区别**:
- `NotRequired[T]`: 字段可以不存在于字典中
- `Optional[T]`: 字段必须存在，但值可以是 `None`

### 3.3 在 LangGraph 中的应用

```python
from typing_extensions import TypedDict, NotRequired, Annotated
from langgraph.graph import StateGraph

def add_messages(left: list, right: list | None) -> list:
    if right is None:
        return left
    return left + right

class State(TypedDict):
    messages: Annotated[list[str], add_messages]
    user_id: str
    # 可选字段（可以不存在）
    session_id: NotRequired[str]
    # 可空字段（必须存在，但可以是 None）
    error: str | None

graph = StateGraph(State)

def node1(state: State) -> dict:
    return {
        "messages": ["Hello"],
        "error": None  # 必须提供 error 字段
        # session_id 可以不提供
    }

def node2(state: State) -> dict:
    return {
        "messages": ["World"],
        "error": "Something went wrong",
        "session_id": "abc123"  # 可选提供
    }
```

### 3.4 源码实现

```python
def _is_optional_type(type_: Any) -> bool:
    """检查类型是否为 Optional"""
    # 处理新联合语法（PEP 604）: str | None
    if isinstance(type_, types.UnionType):
        return any(
            arg is type(None) or _is_optional_type(arg) for arg in type_.__args__
        )

    if hasattr(type_, "__origin__") and hasattr(type_, "__args__"):
        origin = get_origin(type_)
        if origin is Optional:
            return True
        if origin is Union:
            return any(
                arg is type(None) or _is_optional_type(arg) for arg in type_.__args__
            )
        if origin is Annotated:
            return _is_optional_type(type_.__args__[0])
    return False
```

**检查逻辑**:
1. 处理 `T | None` 语法（Python 3.10+）
2. 处理 `Optional[T]` 和 `Union[T, None]`
3. 递归处理嵌套的 `Annotated` 类型

## 4. 类型修饰符的优先级和组合

### 4.1 优先级规则

LangGraph 按以下优先级推断字段默认值:

```
__optional_keys__ > Required/NotRequired > dataclass 默认值 > 类型注解
```

**源码实现**:
```python
def get_field_default(name: str, type_: Any, schema: type[Any]) -> Any:
    """确定字段的默认值"""
    optional_keys = getattr(schema, "__optional_keys__", _DEFAULT_KEYS)
    irq = _is_required_type(type_)

    # 优先级 1: __optional_keys__
    if name in optional_keys:
        if irq:  # 除非显式标记 Required
            return ...
        return None

    # 优先级 2: Required/NotRequired
    if irq is not None:
        if irq:
            return ...  # Required
        return None  # NotRequired

    # 优先级 3: dataclass 默认值
    if dataclasses.is_dataclass(schema):
        field_info = next(
            (f for f in dataclasses.fields(schema) if f.name == name), None
        )
        if field_info:
            if field_info.default is not dataclasses.MISSING:
                return field_info.default
            elif field_info.default_factory is not dataclasses.MISSING:
                return field_info.default_factory()

    # 优先级 4: 类型注解
    if _is_optional_type(type_):
        return None
    return ...  # 必需字段
```

[来源: reference/source_状态类型系统_01.md | LangGraph 源码分析]

### 4.2 组合使用

```python
from typing_extensions import TypedDict, Required, NotRequired, ReadOnly, Annotated

def add_list(left: list, right: list | None) -> list:
    if right is None:
        return left
    return left + right

class State(TypedDict, total=False):
    # 必需 + 只读 + Reducer
    user_id: Required[ReadOnly[str]]

    # 必需 + Reducer
    messages: Required[Annotated[list[str], add_list]]

    # 可选 + 可空
    session_id: NotRequired[str | None]

    # 可选 + Reducer
    tags: NotRequired[Annotated[list[str], add_list]]

    # 可选（total=False 默认）
    metadata: dict
```

**组合规则**:
1. `Required` / `NotRequired` 控制字段是否必须存在
2. `ReadOnly` 控制字段是否可修改（仅类型提示）
3. `Annotated` 绑定 Reducer 函数
4. `Optional` / `T | None` 控制值是否可以是 `None`

### 4.3 实战示例

```python
from typing_extensions import TypedDict, Required, NotRequired, ReadOnly, Annotated
from langgraph.graph import StateGraph

def add_messages(left: list, right: list | None) -> list:
    if right is None:
        return left
    return left + right

def merge_dict(left: dict, right: dict | None) -> dict:
    if right is None:
        return left
    return {**left, **right}

class ComplexState(TypedDict, total=False):
    # 必需 + 只读
    user_id: Required[ReadOnly[str]]

    # 必需 + Reducer
    messages: Required[Annotated[list[str], add_messages]]

    # 可选 + Reducer
    metadata: NotRequired[Annotated[dict, merge_dict]]

    # 可选 + 可空
    error: NotRequired[str | None]

    # 可选（total=False 默认）
    session_id: str

graph = StateGraph(ComplexState)

def node1(state: ComplexState) -> dict:
    return {
        "messages": ["Hello"],
        "metadata": {"source": "node1"}
    }

def node2(state: ComplexState) -> dict:
    return {
        "messages": ["World"],
        "metadata": {"timestamp": "2026-02-26"},
        "error": None,
        "session_id": "abc123"
    }

graph.add_node("n1", node1)
graph.add_node("n2", node2)
graph.add_edge("n1", "n2")
graph.add_edge("n2", "__end__")
graph.set_entry_point("n1")

app = graph.compile()
result = app.invoke({"user_id": "alice"})

print(result)
# {
#     "user_id": "alice",
#     "messages": ["Hello", "World"],
#     "metadata": {"source": "node1", "timestamp": "2026-02-26"},
#     "error": None,
#     "session_id": "abc123"
# }
```

## 5. 版本兼容性

### 5.1 Python 版本支持

| 特性 | typing_extensions | typing 模块 | Python 版本 |
|------|-------------------|-------------|-------------|
| Annotated | - | 3.9+ | 3.9+ |
| Required | 4.0.0+ | 3.11+ | 3.11+ |
| NotRequired | 4.0.0+ | 3.11+ | 3.11+ |
| ReadOnly | 4.9.0+ | 3.13+ | 3.13+ |
| T \| None | - | 3.10+ | 3.10+ |

[来源: reference/context7_typing_extensions_01.md | typing_extensions 官方文档]

### 5.2 向后兼容策略

```python
# 推荐：使用 typing_extensions 确保兼容性
from typing_extensions import TypedDict, Required, NotRequired, ReadOnly

# Python 3.11+ 可以直接使用 typing
# from typing import TypedDict, Required, NotRequired

# Python 3.13+ 可以使用 ReadOnly
# from typing import ReadOnly

class State(TypedDict):
    name: Required[str]
    age: NotRequired[int]
    # ReadOnly 需要 Python 3.13+ 或 typing_extensions 4.9.0+
    user_id: ReadOnly[str]
```

### 5.3 兼容性问题

**Python < 3.12 的 TypedDict 与 Pydantic 兼容性**:

```python
# 问题：Python < 3.12 时，typing.TypedDict 与 Pydantic 可能冲突
# from typing import TypedDict  # 可能导致错误

# 解决方案：使用 typing_extensions.TypedDict
from typing_extensions import TypedDict
from pydantic import BaseModel

class State(TypedDict):
    name: str

class Model(BaseModel):
    state: dict  # 使用 dict 而非 State
```

## 6. 常见错误与解决

### 6.1 错误：混淆 NotRequired 和 Optional

```python
# ❌ 错误：混淆概念
from typing_extensions import TypedDict, NotRequired

class State(TypedDict):
    name: NotRequired[str | None]  # 冗余

# ✅ 正确：根据需求选择
class State1(TypedDict):
    name: NotRequired[str]  # 字段可以不存在

class State2(TypedDict):
    name: str | None  # 字段必须存在，但可以是 None
```

### 6.2 错误：ReadOnly 运行时强制

```python
# ❌ 错误：期望运行时强制 ReadOnly
from typing_extensions import TypedDict, ReadOnly

class State(TypedDict):
    user_id: ReadOnly[str]

state: State = {"user_id": "alice"}
state["user_id"] = "bob"  # 运行时不会报错，仅类型检查器警告

# ✅ 正确：理解 ReadOnly 仅是类型提示
# 如果需要运行时保护，使用其他机制（如 Pydantic）
```

### 6.3 错误：total 参数与修饰符冲突

```python
# ❌ 错误：total=True 与 NotRequired 冲突理解
from typing_extensions import TypedDict, NotRequired

class State(TypedDict, total=True):
    name: str  # 必需
    age: NotRequired[int]  # 可选（覆盖 total=True）

# ✅ 正确：显式修饰符优先级高于 total
# 上面的代码是正确的，NotRequired 覆盖了 total=True
```

## 7. 最佳实践

### 7.1 选择合适的修饰符

**使用 NotRequired**:
- 字段在某些场景下不需要
- 避免使用 `total=False` 的全局影响

**使用 Optional (T | None)**:
- 字段必须存在，但值可能未知
- 表示"有值但可能是空"的语义

**使用 ReadOnly**:
- 配置对象（初始化后不应修改）
- 不可变状态字段
- 防止意外修改的关键字段

### 7.2 代码可读性

```python
# ❌ 不推荐：使用 total=False 导致不清晰
class State(TypedDict, total=False):
    name: str
    age: str
    email: str

# ✅ 推荐：显式标记每个字段
class State(TypedDict):
    name: str  # 必需
    age: str  # 必需
    email: NotRequired[str]  # 可选
```

### 7.3 与 Pydantic 集成

```python
from pydantic import BaseModel, Field
from typing_extensions import Annotated

def add_messages(left: list, right: list | None) -> list:
    if right is None:
        return left
    return left + right

class State(BaseModel):
    # Pydantic 使用 Field 控制必需性
    messages: Annotated[list[str], add_messages] = Field(default_factory=list)
    user_id: str = Field(..., min_length=1)  # ... 表示必需
    session_id: str | None = Field(default=None)  # 可选
```

## 8. 总结

**核心要点**:
1. `Required` / `NotRequired`: 控制字段是否必须存在
2. `ReadOnly`: 标记只读字段（仅类型提示）
3. `Optional` / `T | None`: 控制值是否可以是 `None`
4. 优先级: `__optional_keys__` > `Required/NotRequired` > dataclass 默认值 > 类型注解
5. 版本兼容性: 使用 `typing_extensions` 确保向后兼容

**关键区别**:
- `NotRequired[T]` vs `T | None`: 字段不存在 vs 值为 None
- `Required` vs `total=True`: 显式标记 vs 全局默认
- `ReadOnly`: 类型提示 vs 运行时强制
