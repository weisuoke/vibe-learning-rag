# 30字核心

**Milvus 2.6 通过 Embedding Functions 实现文本自动向量化，简化数据插入与相似度检索流程（5步→3步）。**

---

## 核心价值

在 Milvus 2.6 (2026) 中，数据插入与查询的核心突破是 **Embedding Functions**：

- **自动向量化**：插入文本时自动转换为向量，无需手动调用外部 API
- **简化流程**：从传统的 5 步简化为 3 步（创建 Collection → 插入文本 → 检索）
- **多提供商支持**：内置 OpenAI、Cohere、Bedrock、Voyage、Jina 等多种 Embedding 提供商
- **Data-in, Data-out**：输入文本，输出文本，向量化过程完全透明

---

## 传统方式 vs 2026 方式

### 传统方式（5步）
```
1. 加载文档
2. 手动调用 Embedding API（外部）
3. 创建 Collection
4. 插入向量
5. 手动创建索引
6. 检索
```

### 2026 Milvus 2.6（3步）
```
1. 创建 Collection（配置 Embedding Function）
2. 插入原始文本（自动向量化 + 自动索引）
3. 检索
```

---

## 为什么重要？

1. **降低复杂度**：开发者无需管理外部 Embedding 服务
2. **提升性能**：减少网络调用，向量化在 Milvus 内部完成
3. **简化维护**：统一管理数据和向量化逻辑
4. **加速开发**：从 100+ 行代码减少到 30-50 行

---

**记住这句话：** Milvus 2.6 的 Embedding Functions 让你专注于业务逻辑，而非向量化细节。
