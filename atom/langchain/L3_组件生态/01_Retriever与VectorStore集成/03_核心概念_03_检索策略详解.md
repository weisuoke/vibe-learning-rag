# Retriever与VectorStore集成 - 核心概念:检索策略详解

> **三种检索策略深度解析**:similarity、MMR、similarity_score_threshold 的原理与应用

---

## 概述

VectorStoreRetriever 支持三种检索策略,每种策略适用于不同的场景。理解这三种策略的原理、参数和适用场景,是优化检索效果的关键。

---

## 三种检索策略对比

| 策略 | 算法 | 核心目标 | 适用场景 | 主要参数 |
|------|------|----------|----------|----------|
| similarity | 余弦相似度 Top-K | 最大相关性 | 通用文档检索 | k |
| mmr | 最大边际相关性 | 相关性+多样性 | 推荐系统、多样化结果 | k, fetch_k, lambda_mult |
| similarity_score_threshold | 相似度阈值过滤 | 质量控制 | 高精度问答 | k, score_threshold |

---

## 策略 1: Similarity Search (相似度搜索)

### 原理

基于向量相似度计算,返回与查询最相似的 Top-K 个文档。

**算法流程**:
```
1. 查询向量化: q_vec = embed(query)
2. 计算相似度: scores = [cosine_similarity(q_vec, d_vec) for d_vec in doc_vecs]
3. 排序取 Top-K: top_k_indices = argsort(scores)[-k:]
4. 返回文档: return [docs[i] for i in top_k_indices]
```

### 相似度计算

**余弦相似度**:
```
cosine_similarity(A, B) = (A · B) / (||A|| * ||B||)
```

**取值范围**: [-1, 1]
- 1: 完全相同
- 0: 正交(无关)
- -1: 完全相反

**归一化到 [0, 1]**:
```python
normalized_score = (cosine_similarity + 1) / 2
# 或
normalized_score = 1 - cosine_distance
```

### 源码实现

```python
def similarity_search(
    self, query: str, k: int = 4, **kwargs: Any
) -> list[Document]:
    """Return docs most similar to query.

    Args:
        query: Input text.
        k: Number of Document objects to return.
        **kwargs: Arguments to pass to the search method.

    Returns:
        List of Document objects most similar to the query.
    """
    # 1. 查询向量化
    embedding = self.embedding.embed_query(query)

    # 2. 计算相似度
    similarity = cosine_similarity([embedding], [doc["vector"] for doc in docs])[0]

    # 3. 排序取 Top-K
    top_k_idx = similarity.argsort()[::-1][:k]

    # 4. 返回文档
    return [docs[idx] for idx in top_k_idx]
```

### 参数详解

#### k - 返回文档数量

**默认值**: 4

**选择建议**:
- **简单问答**: k=3-5
- **复杂分析**: k=10-20
- **推荐系统**: k=20-50

**影响因素**:
1. **LLM 上下文窗口**: k 太大会超出上下文限制
2. **文档质量**: 高质量文档可以用较小的 k
3. **查询复杂度**: 复杂查询需要更多上下文
4. **Token 成本**: k 越大,token 消耗越多

**示例**:
```python
# 简单问答
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 4}
)

# 复杂分析
retriever = vectorstore.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 15}
)
```

#### filter - 元数据过滤

**类型**: dict

**用途**: 根据元数据过滤文档

**示例**:
```python
# 过滤特定来源
retriever = vectorstore.as_retriever(
    search_kwargs={
        "k": 5,
        "filter": {"source": "docs"}
    }
)

# 复杂过滤条件
retriever = vectorstore.as_retriever(
    search_kwargs={
        "k": 5,
        "filter": {
            "source": "docs",
            "year": 2024,
            "category": "technical"
        }
    }
)
```

### 优缺点

**优点**:
- 简单高效
- 计算成本低
- 适用于大多数场景
- 结果可解释性强

**缺点**:
- 可能返回重复或相似的文档
- 无法保证结果多样性
- 对查询质量敏感

### 适用场景

1. **文档问答**: 用户提问,返回相关文档
2. **语义搜索**: 根据语义相似度搜索
3. **内容推荐**: 推荐相似内容
4. **知识库检索**: 企业知识库搜索

## 策略 2: MMR (Maximal Marginal Relevance)

### 原理

MMR 算法在保证相关性的同时,最大化结果的多样性,避免返回重复或过于相似的文档。

**核心公式**:
```
MMR = argmax[λ * Sim(q, d) - (1-λ) * max(Sim(d, d_i))]
```

**参数说明**:
- `λ` (lambda_mult): 多样性参数,范围 [0, 1]
- `Sim(q, d)`: 查询与候选文档的相似度(相关性项)
- `max(Sim(d, d_i))`: 候选文档与已选文档的最大相似度(多样性项)

**平衡机制**:
- λ=1: 最大相关性(退化为相似度搜索)
- λ=0: 最大多样性
- λ=0.5: 平衡相关性和多样性

### 算法流程

```python
def mmr_search(query, k, fetch_k, lambda_mult):
    # 1. 获取候选文档(fetch_k 个)
    candidates = similarity_search(query, k=fetch_k)

    # 2. 初始化
    selected = []

    # 3. 迭代选择 k 个文档
    for _ in range(k):
        best_score = -inf
        best_doc = None

        for doc in candidates:
            if doc in selected:
                continue

            # 计算相关性项
            relevance = similarity(query, doc)

            # 计算多样性项
            if selected:
                diversity = max([similarity(doc, s) for s in selected])
            else:
                diversity = 0

            # 计算 MMR 分数
            mmr_score = lambda_mult * relevance - (1 - lambda_mult) * diversity

            if mmr_score > best_score:
                best_score = mmr_score
                best_doc = doc

        selected.append(best_doc)

    return selected
```

### 源码实现

```python
def max_marginal_relevance_search(
    self,
    query: str,
    k: int = 4,
    fetch_k: int = 20,
    lambda_mult: float = 0.5,
    **kwargs: Any,
) -> list[Document]:
    """Return docs selected using the maximal marginal relevance.

    Maximal marginal relevance optimizes for similarity to query AND diversity
    among selected documents.

    Args:
        query: Text to look up documents similar to.
        k: Number of Document objects to return.
        fetch_k: Number of Document objects to fetch to pass to MMR algorithm.
        lambda_mult: Number between 0 and 1 that determines the degree of
            diversity among the results with 0 corresponding to maximum diversity
            and 1 to minimum diversity.
        **kwargs: Arguments to pass to the search method.

    Returns:
        List of Document objects selected by maximal marginal relevance.
    """
    # 1. 查询向量化
    embedding_vector = self.embedding.embed_query(query)

    # 2. 获取候选文档
    prefetch_hits = self._similarity_search_with_score_by_vector(
        embedding=embedding_vector,
        k=fetch_k,
        **kwargs
    )

    # 3. 使用 MMR 算法选择 k 个文档
    mmr_chosen_indices = maximal_marginal_relevance(
        np.array(embedding_vector, dtype=np.float32),
        [vector for _, _, vector in prefetch_hits],
        k=k,
        lambda_mult=lambda_mult,
    )

    return [prefetch_hits[idx][0] for idx in mmr_chosen_indices]
```

### 参数详解

#### k - 最终返回文档数量

**默认值**: 4

**选择建议**: 与 similarity 策略相同

#### fetch_k - 候选文档数量

**默认值**: 20

**作用**: 从 fetch_k 个候选文档中选择 k 个最终文档

**选择建议**:
- fetch_k >= 2 * k (推荐)
- fetch_k >= 5 * k (高多样性场景)

**影响**:
- fetch_k 太小: 多样性不足
- fetch_k 太大: 计算成本高

**示例**:
```python
# 标准配置
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 10, "fetch_k": 20}
)

# 高多样性配置
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 10, "fetch_k": 50}
)
```

#### lambda_mult - 多样性参数

**默认值**: 0.5

**取值范围**: [0, 1]

**含义**:
- **λ=1**: 最大相关性,最小多样性(等价于 similarity 搜索)
- **λ=0.8-0.9**: 高相关性,低多样性
- **λ=0.5**: 平衡相关性和多样性
- **λ=0.2-0.3**: 低相关性,高多样性
- **λ=0**: 最大多样性,最小相关性

**选择建议**:
- **文档问答**: λ=0.7-0.9 (优先相关性)
- **推荐系统**: λ=0.3-0.5 (平衡)
- **探索性搜索**: λ=0.1-0.3 (优先多样性)

**示例**:
```python
# 高相关性
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 10, "lambda_mult": 0.8}
)

# 高多样性
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 10, "lambda_mult": 0.2}
)
```

### 优缺点

**优点**:
- 结果多样性高
- 避免重复内容
- 适合推荐系统
- 提供更全面的信息

**缺点**:
- 计算成本高(需要计算文档间相似度)
- 可能牺牲部分相关性
- 参数调优复杂

### 适用场景

1. **推荐系统**: 推荐多样化的内容
2. **探索性搜索**: 用户想了解多个方面
3. **摘要生成**: 需要覆盖多个主题
4. **对话系统**: 避免重复回答

## 策略 3: Similarity Score Threshold (阈值过滤)

### 原理

只返回相似度分数高于指定阈值的文档,用于质量控制。

**算法流程**:
```
1. 查询向量化: q_vec = embed(query)
2. 计算相似度: docs_and_scores = [(doc, similarity(q_vec, doc_vec)) for doc in docs]
3. 过滤低分文档: filtered = [(doc, score) for doc, score in docs_and_scores if score >= threshold]
4. 返回文档: return [doc for doc, _ in filtered[:k]]
```

### 源码实现

```python
def similarity_search_with_relevance_scores(
    self,
    query: str,
    k: int = 4,
    **kwargs: Any,
) -> list[tuple[Document, float]]:
    """Return docs and relevance scores in the range [0, 1].

    0 is dissimilar, 1 is most similar.

    Args:
        query: Input text.
        k: Number of Document objects to return.
        **kwargs: Kwargs to be passed to similarity search.
            Should include score_threshold, an optional floating point value
            between 0 to 1 to filter the resulting set of retrieved docs.

    Returns:
        List of tuples of (doc, similarity_score).
    """
    # 1. 提取阈值参数
    score_threshold = kwargs.pop("score_threshold", None)

    # 2. 计算相似度分数
    docs_and_similarities = self._similarity_search_with_relevance_scores(
        query, k=k, **kwargs
    )

    # 3. 过滤低于阈值的文档
    if score_threshold is not None:
        docs_and_similarities = [
            (doc, similarity)
            for doc, similarity in docs_and_similarities
            if similarity >= score_threshold
        ]

        # 4. 警告:没有文档满足阈值
        if len(docs_and_similarities) == 0:
            logger.warning(
                "No relevant docs were retrieved using the "
                "relevance score threshold %s",
                score_threshold,
            )

    return docs_and_similarities
```

### 参数详解

#### k - 最大返回文档数量

**默认值**: 4

**作用**: 限制最大返回数量,即使有更多文档满足阈值

**注意**: 实际返回数量可能小于 k(如果满足阈值的文档不足 k 个)

#### score_threshold - 相似度阈值

**类型**: float

**取值范围**: [0, 1]

**必需**: 使用 similarity_score_threshold 策略时必须提供

**含义**:
- **0.9-1.0**: 极高相似度(几乎完全匹配)
- **0.8-0.9**: 高相似度(强相关)
- **0.7-0.8**: 中等相似度(相关)
- **0.6-0.7**: 低相似度(弱相关)
- **< 0.6**: 很低相似度(可能不相关)

**选择建议**:
- **高精度问答**: threshold=0.8-0.9
- **平衡场景**: threshold=0.7-0.8
- **高召回场景**: threshold=0.6-0.7

**示例**:
```python
# 高精度
retriever = vectorstore.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={
        "k": 10,
        "score_threshold": 0.85
    }
)

# 平衡
retriever = vectorstore.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={
        "k": 10,
        "score_threshold": 0.75
    }
)
```

@model_validator(mode="before")
@classmethod
def validate_search_type(cls, values: dict) -> Any:
    """Validate search type."""
    search_type = values.get("search_type", "similarity")

        score_threshold = values.get("search_kwargs", {}).get("score_threshold")

### 优缺点

**优点**:
- 质量控制严格
- 避免低质量结果

- 阈值难以确定

### 适用场景

1. **高精度问答**: 只返回高质量答案

## 性能对比

### 计算复杂度

| 策略 | 时间复杂度 | 空间复杂度 | 相对性能 |
|------|------------|------------|----------|
| similarity | O(n) | O(1) | 最快 |
| threshold | O(n) | O(1) | 快 |
| mmr | O(k * fetch_k) | O(fetch_k) | 慢 |

**说明**:
- n: 文档总数
- k: 返回文档数量
- fetch_k: 候选文档数量

### 性能优化建议

1. **similarity**: 使用向量索引(HNSW, IVF)加速
2. **threshold**: 先用索引过滤,再计算精确分数
3. **mmr**: 减小 fetch_k,使用近似算法

## 参数调优指南

### 调优流程

```
1. 选择基准策略
   ↓
2. 设置初始参数
   ↓
3. 评估检索质量
   ↓
4. 调整参数
   ↓
5. 重复 3-4 直到满意
```

### 评估指标

1. **相关性**: 返回文档与查询的相关程度
2. **多样性**: 返回文档之间的差异程度
3. **召回率**: 相关文档被检索到的比例
4. **精确率**: 检索到的文档中相关文档的比例
5. **F1 分数**: 召回率和精确率的调和平均

## 常见问题

### Q1: 如何选择合适的策略?

**A**: 根据场景选择:
- 通用场景: similarity
- 需要多样性: mmr
- 需要质量控制: similarity_score_threshold

### Q2: MMR 的 fetch_k 应该设置多大?

**A**: 推荐 fetch_k >= 2 * k,高多样性场景可以设置 fetch_k >= 5 * k。

### Q3: 阈值应该设置多高?

**A**:
- 高精度: 0.8-0.9
- 平衡: 0.7-0.8
- 高召回: 0.6-0.7

建议通过实验确定最优阈值。

### Q4: 为什么 MMR 比 similarity 慢?

**A**: MMR 需要计算文档间的相似度,时间复杂度为 O(k * fetch_k),而 similarity 只需要 O(n)。

### Q5: 如何提高检索速度?

**A**:
1. 使用向量索引(HNSW, IVF)
2. 减小 k 和 fetch_k
3. 使用批量检索
4. 缓存常见查询

## 总结

三种检索策略各有特点:

1. **Similarity**: 简单高效,适合大多数场景
2. **MMR**: 平衡相关性和多样性,适合推荐系统
3. **Threshold**: 严格质量控制,适合高精度场景

选择策略时需要考虑:
- 应用场景
- 性能要求
- 质量要求
- 多样性需求
