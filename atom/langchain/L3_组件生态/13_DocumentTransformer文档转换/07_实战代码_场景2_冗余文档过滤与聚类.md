## 场景描述

### 业务背景

在 RAG 系统中，多路检索（向量检索 + BM25 + 父文档检索）经常返回大量重复或高度相似的文档。
如果直接将这些冗余文档塞进 LLM 的上下文窗口，会浪费 Token、增加成本，甚至降低回答质量。

### 技术挑战

1. **多路检索冗余**：3 个检索器各返回 5 条，可能有 40% 重复
2. **语义重复**：不同表述但含义相同的文档需要识别
3. **多样性不足**：检索结果可能集中在某个子话题
4. **Token 预算有限**：上下文窗口需要高效利用

### 学习目标

- 使用 `EmbeddingsRedundantFilter` 过滤语义重复文档
- 使用 `EmbeddingsClusteringFilter` 聚类选取代表性文档
- 将过滤器集成到 `ContextualCompressionRetriever` 中
- 对比过滤前后的文档多样性

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

```python
# ============================================================
```

## 第四步：组合过滤管道

> 将去重和聚类组合使用，先去重再聚类，效果更好。

### 4.1 DocumentCompressorPipeline 组合

```python
# ============================================================
# 组合管道：去重 → 聚类
# ============================================================

from langchain_community.document_transformers import LongContextReorder

# 第一步：去重
redundant_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.85,
)

# 第二步：聚类
clustering_filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=3,
    num_closest=1,
    sorted=True,
)

# 第三步：长上下文重排序（解决 "Lost in the Middle" 问题）
reordering = LongContextReorder()

# 组合为管道
pipeline = DocumentCompressorPipeline(
    transformers=[redundant_filter, clustering_filter, reordering]
)

# 注意：DocumentCompressorPipeline 的 compress_documents 方法
# 需要一个 query 参数（即使某些转换器不使用它）
compressed_docs = pipeline.compress_documents(
    documents=all_results,
    query="RAG 系统如何处理长文档？"
)

print("=" * 60)
print("组合管道结果：去重 → 聚类 → 重排序")
print("=" * 60)
print(f"输入: {len(all_results)} 篇")
print(f"输出: {len(list(compressed_docs))} 篇")

for i, doc in enumerate(compressed_docs):
    print(f"\n  {i+1}. [{doc.metadata.get('retriever', 'unknown')}] "
          f"{doc.metadata['source']}")
    print(f"     {doc.page_content[:80]}...")
```

### 4.2 集成到 ContextualCompressionRetriever

```python
# ============================================================
# 集成到 ContextualCompressionRetriever
# ============================================================

from langchain_community.vectorstores import Chroma

# 第一步：创建基础向量存储和检索器
# （这里用模拟数据创建一个简单的向量存储）
all_texts = [doc.page_content for doc in all_results]
all_metadatas = [doc.metadata for doc in all_results]

vectorstore = Chroma.from_texts(
    texts=all_texts,
    metadatas=all_metadatas,
    embedding=embeddings,
    collection_name="demo_redundancy_filter",
)

base_retriever = vectorstore.as_retriever(
    search_kwargs={"k": 8}  # 返回较多结果，后续由过滤器精简
)

# 第二步：创建压缩管道
compression_pipeline = DocumentCompressorPipeline(
    transformers=[
        EmbeddingsRedundantFilter(embeddings=embeddings, similarity_threshold=0.85),
        EmbeddingsClusteringFilter(embeddings=embeddings, num_clusters=3, num_closest=1),
        LongContextReorder(),
    ]
)

# 第三步：创建压缩检索器
compression_retriever = ContextualCompressionRetriever(
    base_compressor=compression_pipeline,
    base_retriever=base_retriever,
)

# 第四步：执行检索
query = "RAG 系统如何处理长文档？"

# 对比：基础检索 vs 压缩检索
base_results = base_retriever.invoke(query)
compressed_results = compression_retriever.invoke(query)

print("=" * 60)
print("ContextualCompressionRetriever 集成效果")
print("=" * 60)

print(f"\n基础检索结果: {len(base_results)} 篇")
for i, doc in enumerate(base_results):
    print(f"  {i+1}. {doc.page_content[:60]}...")

print(f"\n压缩检索结果: {len(compressed_results)} 篇")
for i, doc in enumerate(compressed_results):
    print(f"  {i+1}. {doc.page_content[:60]}...")

print(f"\n压缩率: {(1 - len(compressed_results)/len(base_results))*100:.1f}%")

# 清理
vectorstore.delete_collection()
```

## 第五步：效果评估

> 评估过滤前后的文档多样性和信息覆盖度。

```python
# ============================================================
# 多样性评估：计算文档之间的平均相似度
# ============================================================

import numpy as np


def calculate_diversity(docs: list[Document], embeddings_model) -> dict:
    """
    计算文档集合的多样性指标

    多样性 = 1 - 平均余弦相似度
    值越高表示文档越多样
    """
    if len(docs) < 2:
        return {"avg_similarity": 0, "diversity": 1.0, "doc_count": len(docs)}

    # 获取所有文档的嵌入向量
    texts = [doc.page_content for doc in docs]
    vectors = embeddings_model.embed_documents(texts)
    vectors = np.array(vectors)

    # 计算余弦相似度矩阵
    # 归一化
    norms = np.linalg.norm(vectors, axis=1, keepdims=True)
    normalized = vectors / norms

    # 计算相似度矩阵
    similarity_matrix = np.dot(normalized, normalized.T)

    # 取上三角（排除对角线和重复对）
    n = len(docs)
    similarities = []
    for i in range(n):
        for j in range(i + 1, n):
            similarities.append(similarity_matrix[i][j])

    avg_sim = np.mean(similarities)
    diversity = 1 - avg_sim

    return {
        "avg_similarity": float(avg_sim),
        "diversity": float(diversity),
        "doc_count": len(docs),
        "min_similarity": float(np.min(similarities)),
        "max_similarity": float(np.max(similarities)),
    }


# 对比过滤前后的多样性
print("=" * 60)
print("多样性评估")
print("=" * 60)

metrics_before = calculate_diversity(all_results, embeddings)
metrics_after = calculate_diversity(list(compressed_docs), embeddings)

print(f"\n过滤前 ({metrics_before['doc_count']} 篇):")
print(f"  平均相似度: {metrics_before['avg_similarity']:.4f}")
print(f"  多样性得分: {metrics_before['diversity']:.4f}")
print(f"  相似度范围: [{metrics_before['min_similarity']:.4f}, "
      f"{metrics_before['max_similarity']:.4f}]")

print(f"\n过滤后 ({metrics_after['doc_count']} 篇):")
print(f"  平均相似度: {metrics_after['avg_similarity']:.4f}")
print(f"  多样性得分: {metrics_after['diversity']:.4f}")
print(f"  相似度范围: [{metrics_after['min_similarity']:.4f}, "
      f"{metrics_after['max_similarity']:.4f}]")

improvement = metrics_after['diversity'] - metrics_before['diversity']
print(f"\n多样性提升: {improvement:+.4f} "
      f"({'提升' if improvement > 0 else '下降'} {abs(improvement/metrics_before['diversity'])*100:.1f}%)")
```

## 总结与最佳实践

### 过滤器选择决策树

```
多路检索结果去重决策树
│
├── 主要问题是什么？
│   ├── 完全重复/高度相似 → EmbeddingsRedundantFilter
│   ├── 结果缺乏多样性 → EmbeddingsClusteringFilter
│   └── 两者都有 → 组合管道（先去重再聚类）
│
├── Token 预算紧张？
│   ├── 是 → 聚类 (num_clusters=3, num_closest=1)
│   └── 否 → 去重 (threshold=0.85) 即可
│
└── 需要保持排序？
    ├── 是 → EmbeddingsClusteringFilter(sorted=True)
    └── 否 → 默认即可
```

### 关键参数速查

| 过滤器 | 参数 | 推荐值 | 说明 |
|--------|------|--------|------|
| RedundantFilter | `similarity_threshold` | 0.85 | 相似度阈值，越低去重越激进 |
| ClusteringFilter | `num_clusters` | 3-5 | 聚类数，根据主题数量调整 |
| ClusteringFilter | `num_closest` | 1-2 | 每簇取几篇 |
| ClusteringFilter | `sorted` | True | 是否按原始顺序排序 |

### 组合管道推荐配置

```python
# 生产环境推荐配置
pipeline = DocumentCompressorPipeline(
    transformers=[
        EmbeddingsRedundantFilter(embeddings=embeddings, similarity_threshold=0.85),
        EmbeddingsClusteringFilter(embeddings=embeddings, num_clusters=4, num_closest=1, sorted=True),
        LongContextReorder(),  # 解决 "Lost in the Middle"
    ]
)
```

> **一句话总结：** 多路检索必带去重，EmbeddingsRedundantFilter 去掉重复，
> EmbeddingsClusteringFilter 保证多样性，两者组合是 RAG 管道的标配。
