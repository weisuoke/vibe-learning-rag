# 核心概念5：图构建器模式与编译

## 一句话定义

**StateGraph 是 LangGraph 的 Builder Pattern 实现，通过声明式 API 构建图结构，compile() 编译为可执行的 Pregel 引擎——先"画蓝图"，再"盖房子"。**

---

## 双重类比

### 前端类比：React createElement → Virtual DOM → 真实 DOM

React 的渲染流程和 LangGraph 的图构建惊人地相似：

```
React:      JSX → createElement → Virtual DOM → 真实 DOM
LangGraph:  add_node/add_edge → StateGraph → compile() → Pregel 引擎
```

- `StateGraph` 就像 Virtual DOM——它是图的"描述"，还不是可执行的图
- `compile()` 就像 React 的 reconciliation——把描述转化为可执行的结构
- 编译后的图不可变，就像 React 的 fiber tree 在一次渲染中是固定的

### 日常生活类比：建筑蓝图 → 施工 → 建筑物

- `StateGraph` = 建筑蓝图（画图纸阶段，可以随意修改）
- `add_node` = 在蓝图上标注房间（客厅、卧室、厨房）
- `add_edge` = 在蓝图上画走廊连接房间
- `compile()` = 按蓝图施工，建成实际建筑
- 编译后不可变 = 房子盖好了，不能随便拆墙加房间

## StateGraph Builder Pattern 详解

### 什么是 Builder Pattern？

Builder Pattern（构建器模式）是一种设计模式：通过链式调用逐步构建复杂对象，最后一步生成最终产物。

```python
# Builder Pattern 的通用形式
result = Builder()
    .step1(...)
    .step2(...)
    .step3(...)
    .build()  # 最终构建

# LangGraph 的 Builder Pattern
graph = StateGraph(State)          # 创建构建器
graph.add_node("a", func_a)       # 添加节点
graph.add_edge(START, "a")        # 添加边
app = graph.compile()             # 编译（= build）
```

### StateGraph 的完整 API

[来源: sourcecode/langgraph/graph/state.py - StateGraph 类]

#### 1. 创建：StateGraph(State) 指定状态类型

```python
from langgraph.graph import StateGraph
from typing import TypedDict, Annotated
import operator

class MyState(TypedDict):
    messages: Annotated[list[str], operator.add]
    current_step: str

# 创建构建器，绑定状态类型
builder = StateGraph(MyState)
```

状态类型决定了图中所有节点共享的数据结构。`Annotated` + Reducer 定义了并行写入时的合并策略。

#### 2. 添加节点：add_node(name, func)

```python
# 基础签名：(state) -> dict
def my_node(state: MyState) -> dict:
    return {"current_step": "done"}

# 带配置签名：(state, config) -> dict
def node_with_config(state: MyState, config: dict) -> dict:
    model = config.get("configurable", {}).get("model", "default")
    return {"current_step": f"using {model}"}

# 带 writer 签名：(state, *, writer) -> dict
def node_with_writer(state: MyState, *, writer) -> dict:
    writer("processing...")  # 流式输出
    return {"current_step": "streamed"}

builder.add_node("step_a", my_node)
builder.add_node("step_b", node_with_config)
```

[来源: sourcecode/langgraph/graph/_node.py - StateNode 类型别名]

节点函数支持多种签名，LangGraph 会自动检测并适配。

#### 3. 添加边：三种方式

```python
from langgraph.graph import START, END

# 方式1：静态边 - A 之后一定去 B
builder.add_edge("step_a", "step_b")

# 方式2：条件边 - 运行时决定去哪
def route_fn(state: MyState) -> str:
    return "step_a" if state["current_step"] == "retry" else "step_b"

builder.add_conditional_edges("router", route_fn)

# 方式3：顺序添加 - 按顺序串联多个节点
builder.add_sequence(["step_a", "step_b", "step_c"])
# 等价于：
# builder.add_node("step_a", step_a)
# builder.add_node("step_b", step_b)
# builder.add_node("step_c", step_c)
# builder.add_edge("step_a", "step_b")
# builder.add_edge("step_b", "step_c")
```

#### 4. 入口与出口

```python
# 方式1：使用 START/END 常量（推荐）
builder.add_edge(START, "first_node")
builder.add_edge("last_node", END)

# 方式2：使用 set 方法（旧 API，仍然可用）
builder.set_entry_point("first_node")
builder.set_finish_point("last_node")

# 方式3：条件入口
def entry_route(state: MyState) -> str:
    return "fast_path" if state["current_step"] == "simple" else "full_path"

builder.set_conditional_entry_point(entry_route, {
    "fast_path": "step_a",
    "full_path": "step_b",
})
```

[来源: sourcecode/langgraph/graph/state.py - set_entry_point / set_finish_point]

#### 5. 编译：compile() → CompiledStateGraph

```python
# 基础编译
app = builder.compile()

# 带检查点的编译（支持持久化和恢复）
from langgraph.checkpoint.memory import MemorySaver
app = builder.compile(checkpointer=MemorySaver())

# 带中断点的编译（human-in-the-loop）
app = builder.compile(
    checkpointer=MemorySaver(),
    interrupt_before=["human_review"],  # 在这些节点前暂停
)
```

## 编译过程详解（源码层面）

### 两阶段架构

```
StateGraph（蓝图）  →  compile()  →  CompiledStateGraph（可执行图）
                                          ↓ 继承
                                       Pregel（执行引擎）
```

[来源: sourcecode/langgraph/graph/state.py - CompiledStateGraph 类]

### compile() 内部做了什么？

编译过程可以分为三个阶段：

#### 阶段1：附加节点（attach_node）

```python
# 伪代码：compile() 内部
for key, node_spec in self.nodes.items():
    compiled_graph.attach_node(key, node_spec)
    # 创建 PregelNode 对象
    # 配置输入/输出通道
    # 注册触发器
```

每个节点被转化为 `PregelNode`，包含：
- 节点函数（runnable）
- 输入通道映射
- 输出通道映射
- 触发条件

#### 阶段2：附加边（attach_edge / attach_branch）

```python
# 静态边
for start, end in self.edges:
    compiled_graph.attach_edge(start, end)
    # 在 start 的输出通道和 end 的触发器之间建立连接

# 条件边（分支）
for start, branches in self.branches.items():
    for name, branch in branches.items():
        compiled_graph.attach_branch(start, name, branch)
        # 创建 BranchSpec，注册路由函数
```

#### 阶段3：通道（Channel）创建与配置

```python
# 为每个状态字段创建通道
for field_name, field_type in State.__annotations__.items():
    if has_reducer(field_type):
        channel = BinaryOperatorAggregate(reducer)  # 有 Reducer
    else:
        channel = LastValue()  # 无 Reducer，取最后写入的值
```

[来源: sourcecode/langgraph/pregel/_algo.py - 通道与触发器]

通道是节点之间通信的桥梁——节点不直接调用彼此，而是通过通道传递数据。

### 触发器（Trigger）映射

编译后，每个节点都有一组触发器，定义了"什么条件下这个节点会被执行"：

```
trigger_to_nodes 映射：
  通道 "step_a:output" → 触发节点 ["step_b"]
  通道 "router:output" → 触发节点 ["step_a", "step_b", "step_c"]（条件边）
```

这就是 Pregel 引擎的核心机制——基于通道更新触发节点执行，而非显式调用。

## 编译后的图结构

### 核心属性

```python
app = builder.compile()

# 编译后的关键属性
app.nodes       # Dict[str, PregelNode] - 所有节点
app.channels    # Dict[str, BaseChannel] - 所有通道
app.stream_mode # 流式输出模式
```

### 不可变性：编译后图结构固定

```python
builder = StateGraph(State)
builder.add_node("a", func_a)
app = builder.compile()

# ❌ 编译后不能修改图结构
# app.add_node("b", func_b)  # 没有这个方法！

# ✅ 要修改图，必须重新构建和编译
builder.add_node("b", func_b)
app = builder.compile()  # 重新编译
```

**前端类比：** 就像 webpack 打包——`webpack.config.js` 是配置（StateGraph），`webpack build` 是编译（compile），产出的 bundle 是不可变的（CompiledStateGraph）。要改配置就得重新 build。

### BSP 执行模型

编译后的图使用 BSP（Bulk Synchronous Parallel，批量同步并行）模型执行：

```
Step 1: 计划（Plan）
  → 检查哪些通道有更新
  → 确定哪些节点被触发

Step 2: 执行（Execute）
  → 并行执行所有被触发的节点
  → 收集输出

Step 3: 更新（Update）
  → 将输出写入通道
  → 通过 Reducer 合并并行结果
  → 回到 Step 1，直到没有节点被触发
```

[来源: sourcecode/langgraph/pregel/_algo.py - prepare_next_tasks / apply_writes]

## 完整代码示例：从零构建一个动态工作流

```python
"""
图构建器模式实战
场景：构建一个完整的文档处理工作流，展示 StateGraph 的所有核心 API
"""

import operator
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command


# ===== 1. 定义状态 =====

class DocState(TypedDict):
    """文档处理工作流的状态"""
    content: str                                      # 原始内容
    doc_type: str                                     # 文档类型
    chunks: Annotated[list[str], operator.add]        # 分块结果（Reducer 合并）
    summary: str                                      # 摘要
    quality_score: float                              # 质量评分
    messages: Annotated[list[str], operator.add]      # 处理日志


# ===== 2. 定义节点函数 =====

def classify_document(state: DocState) -> dict:
    """分类节点：判断文档类型"""
    content = state["content"]
    if len(content) > 200:
        doc_type = "long"
    elif any(kw in content for kw in ["def ", "class ", "import "]):
        doc_type = "code"
    else:
        doc_type = "short"
    print(f"[分类] 文档类型: {doc_type}")
    return {"doc_type": doc_type, "messages": [f"分类为: {doc_type}"]}


def chunk_long_doc(state: DocState) -> dict:
    """长文档分块"""
    content = state["content"]
    chunk_size = 100
    chunks = [content[i:i+chunk_size] for i in range(0, len(content), chunk_size)]
    print(f"[分块] 长文档分为 {len(chunks)} 块")
    return {"chunks": chunks, "messages": [f"分为 {len(chunks)} 块"]}


def chunk_code_doc(state: DocState) -> dict:
    """代码文档分块：按函数/类分割"""
    content = state["content"]
    # 简化：按换行分块
    chunks = [line for line in content.split("\n") if line.strip()]
    print(f"[分块] 代码文档分为 {len(chunks)} 块")
    return {"chunks": chunks, "messages": [f"代码分为 {len(chunks)} 块"]}


def chunk_short_doc(state: DocState) -> dict:
    """短文档：整体作为一块"""
    print("[分块] 短文档，整体作为一块")
    return {"chunks": [state["content"]], "messages": ["短文档，不分块"]}


def summarize(state: DocState) -> dict:
    """生成摘要"""
    total_chunks = len(state["chunks"])
    summary = f"文档共 {total_chunks} 个分块，总长度 {len(state['content'])} 字符"
    print(f"[摘要] {summary}")
    return {"summary": summary, "messages": [f"摘要: {summary}"]}


def quality_check(state: DocState) -> dict:
    """质量检查"""
    score = min(1.0, len(state["content"]) / 500)
    print(f"[质量] 评分: {score:.2f}")
    return {"quality_score": score, "messages": [f"质量评分: {score:.2f}"]}


# ===== 3. 路由函数 =====

def route_by_doc_type(state: DocState) -> str:
    """根据文档类型路由到不同的分块策略"""
    return {
        "long": "chunk_long",
        "code": "chunk_code",
        "short": "chunk_short",
    }.get(state["doc_type"], "chunk_short")


# ===== 4. 构建图 =====

builder = StateGraph(DocState)

# 添加所有节点
builder.add_node("classify", classify_document)
builder.add_node("chunk_long", chunk_long_doc)
builder.add_node("chunk_code", chunk_code_doc)
builder.add_node("chunk_short", chunk_short_doc)
builder.add_node("summarize", summarize)
builder.add_node("quality_check", quality_check)

# 入口 → 分类
builder.add_edge(START, "classify")

# 条件边：根据文档类型路由到不同分块策略
builder.add_conditional_edges("classify", route_by_doc_type, {
    "chunk_long": "chunk_long",
    "chunk_code": "chunk_code",
    "chunk_short": "chunk_short",
})

# 所有分块节点 → 摘要
builder.add_edge("chunk_long", "summarize")
builder.add_edge("chunk_code", "summarize")
builder.add_edge("chunk_short", "summarize")

# 摘要 → 质量检查 → 结束
builder.add_edge("summarize", "quality_check")
builder.add_edge("quality_check", END)

# 编译
app = builder.compile()


# ===== 5. 运行测试 =====

if __name__ == "__main__":
    print("=== 图构建器模式实战 ===\n")

    # 测试1：短文档
    print("--- 测试1：短文档 ---")
    result = app.invoke({
        "content": "LangGraph 是一个用于构建有状态 AI 工作流的框架。",
        "doc_type": "", "chunks": [], "summary": "",
        "quality_score": 0.0, "messages": [],
    })
    print(f"处理日志: {result['messages']}\n")

    # 测试2：长文档
    print("--- 测试2：长文档 ---")
    long_content = "这是一篇关于 RAG 系统设计的长文档。" * 20
    result = app.invoke({
        "content": long_content,
        "doc_type": "", "chunks": [], "summary": "",
        "quality_score": 0.0, "messages": [],
    })
    print(f"处理日志: {result['messages']}\n")

    # 测试3：代码文档
    print("--- 测试3：代码文档 ---")
    code_content = """import os
def hello():
    print("hello")
class MyClass:
    pass"""
    result = app.invoke({
        "content": code_content,
        "doc_type": "", "chunks": [], "summary": "",
        "quality_score": 0.0, "messages": [],
    })
    print(f"处理日志: {result['messages']}\n")
```

## 与其他 Builder Pattern 的对比

| 框架 | Builder | 构建方法 | 编译/构建 | 产物 |
|------|---------|---------|----------|------|
| LangGraph | `StateGraph(State)` | `add_node`, `add_edge` | `compile()` | `CompiledStateGraph` |
| React | `createElement()` | JSX / createElement | Reconciliation | Fiber Tree |
| SQL | `QueryBuilder()` | `.select()`, `.where()` | `.build()` | SQL 字符串 |
| Docker | `Dockerfile` | `FROM`, `RUN`, `COPY` | `docker build` | Image |

共同特点：
- 声明式 API（描述"要什么"，而非"怎么做"）
- 构建与执行分离
- 产物不可变

## 最佳实践：图构建的组织方式

### 1. 分层组织节点定义

```python
# ===== 状态定义 =====
class State(TypedDict):
    ...

# ===== 节点函数 =====
def node_a(state): ...
def node_b(state): ...

# ===== 路由函数 =====
def route_fn(state): ...

# ===== 图构建 =====
builder = StateGraph(State)
builder.add_node(...)
builder.add_edge(...)
app = builder.compile()
```

### 2. 使用工厂函数创建可复用的子图

```python
def create_processing_subgraph(name: str) -> StateGraph:
    """工厂函数：创建可复用的处理子图"""
    builder = StateGraph(ProcessState)
    builder.add_node("validate", validate_fn)
    builder.add_node("process", process_fn)
    builder.add_edge(START, "validate")
    builder.add_edge("validate", "process")
    builder.add_edge("process", END)
    return builder.compile()
```

### 3. 编译时配置检查点和中断

```python
# 开发环境：无检查点
dev_app = builder.compile()

# 生产环境：带检查点 + 人工审核中断
from langgraph.checkpoint.memory import MemorySaver
prod_app = builder.compile(
    checkpointer=MemorySaver(),
    interrupt_before=["sensitive_operation"],
)
```

### 4. 利用 add_sequence 简化顺序流程

```python
# 冗长写法
builder.add_node("a", func_a)
builder.add_node("b", func_b)
builder.add_node("c", func_c)
builder.add_edge("a", "b")
builder.add_edge("b", "c")

# 简洁写法
builder.add_sequence([("a", func_a), ("b", func_b), ("c", func_c)])
```

## 总结

StateGraph 是 LangGraph 的 Builder Pattern 核心：通过 `add_node` 添加节点、`add_edge` / `add_conditional_edges` 添加边、`compile()` 编译为基于 Pregel 引擎的可执行图。编译过程将声明式的图描述转化为通道-触发器驱动的执行结构，编译后图结构不可变。理解"构建与执行分离"这个设计哲学，是掌握 LangGraph 动态图构建的关键。
