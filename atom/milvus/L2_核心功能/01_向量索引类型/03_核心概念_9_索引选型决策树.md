# 核心概念：索引选型决策树

## 概述

索引选型是向量检索系统设计的关键决策。本文档提供完整的决策树和选型方法论，帮助你根据数据规模、性能需求、成本约束等因素选择最合适的索引类型。

---

## 决策因素

### 1. 数据规模

```
< 1M向量：
- FLAT（100%准确率）
- 简单直接，无需调参

1M-10M向量：
- IVF_FLAT（平衡性能）
- 适度压缩，易于调优

10M-100M向量：
- HNSW（生产标准）
- 高性能，高准确率

100M-1B向量：
- GPU_CAGRA（高性能）
- GPU加速，10x速度

1B+向量：
- RaBitQ（成本优化）
- 72%内存节省
```

### 2. 准确率要求

```
100%准确率：
→ FLAT

95-99%准确率：
→ HNSW, IVF_FLAT, GPU_CAGRA

90-95%准确率：
→ IVF_SQ8

85-95%准确率：
→ IVF_PQ, RaBitQ
```

### 3. 内存限制

```
内存充足（1.5-2x数据大小）：
→ HNSW, GPU_CAGRA

内存一般（1x数据大小）：
→ FLAT, IVF_FLAT

内存受限（0.25x数据大小）：
→ IVF_SQ8

内存极限（0.03x数据大小）：
→ IVF_PQ, RaBitQ
```

### 4. 查询模式

```
纯向量搜索：
→ HNSW

带过滤条件（过滤>50%）：
→ IVF_FLAT

高QPS（>1000）：
→ GPU_CAGRA

批量查询（batch>10）：
→ GPU_CAGRA

单次查询（batch=1）：
→ HNSW
```

### 5. 硬件资源

```
仅CPU：
→ FLAT, IVF系列, HNSW

有GPU：
→ GPU_CAGRA

内存受限：
→ IVF_SQ8, IVF_PQ, RaBitQ

成本敏感：
→ RaBitQ
```

### 6. 更新频率

```
静态数据（无更新）：
→ 所有索引

偶尔更新（每天/每周）：
→ HNSW, IVF系列

频繁更新（实时）：
→ IVF_FLAT
```

## 按规模选择

### < 1M向量

## 常见问题

### Q1：如何快速选择索引？

**A：** 使用3步法。

```
步骤1：确定数据规模
- < 1M → FLAT
- 1M-10M → IVF_FLAT
- 10M-100M → HNSW
- > 100M → GPU_CAGRA或RaBitQ

步骤2：检查约束条件
- 内存受限 → IVF_SQ8或RaBitQ
- 无GPU → 排除GPU_CAGRA
- 频繁更新 → 排除HNSW

步骤3：验证性能
- 小规模测试
- 性能基准测试
- A/B测试
```

### Q2：多个索引都满足需求，如何选择？

**A：** 优先级排序。

```
1. 生产成熟度：HNSW > IVF_FLAT > 其他
2. 调优难度：FLAT > IVF系列 > HNSW > GPU_CAGRA
3. 成本：RaBitQ > IVF_SQ8 > IVF_FLAT > HNSW > GPU_CAGRA
4. 性能：GPU_CAGRA > HNSW > IVF系列 > FLAT
```

### Q3：索引选错了怎么办？

**A：** 渐进式迁移。

```
步骤1：创建新索引
步骤2：双写数据（新旧索引）
步骤3：灰度切换流量
步骤4：监控对比
步骤5：完全切换
步骤6：删除旧索引
```

### Q4：如何验证索引选择是否正确？

**A：** 监控关键指标。

```python
# 监控脚本
def validate_index_choice():
    # 1. 查询延迟
    latency = measure_latency()
    assert latency < target_latency

    # 2. 准确率
    recall = measure_recall()
    assert recall > target_recall

    # 3. 内存使用
    memory = measure_memory()
    assert memory < memory_limit

    # 4. QPS
    qps = measure_qps()
    assert qps > target_qps

    print("Index choice validated!")
```

## 总结

### 核心要点

1. **数据规模是首要因素**：规模决定索引类型
2. **没有完美索引**：只有最适合的索引
3. **权衡三要素**：准确率 ↔ 速度 ↔ 内存
4. **小规模测试**：先测试再部署
5. **持续监控**：监控指标，及时调优

### 快速决策表

| 场景 | 推荐索引 | 理由 |
|------|---------|------|
| 开发测试 | FLAT | 简单直接 |
| 小型应用 | IVF_FLAT | 平衡性能 |
| 生产环境 | HNSW | 标准选择 |
| 高性能 | GPU_CAGRA | 极快速度 |
| 成本优化 | RaBitQ | 内存节省 |

### 选型流程

```
1. 确定数据规模
2. 评估约束条件（内存、GPU、成本）
3. 选择候选索引（2-3个）
4. 小规模测试
5. 性能基准测试
6. A/B测试
7. 生产部署
8. 持续监控
9. 及时调优
```
