# OpenClaw 基础故障排查 - 实战场景 2：网关无法启动排查

## 一、场景概述

**网关无法启动是 OpenClaw 最常见的故障之一，通过系统化的诊断流程，可以快速识别端口冲突、配置错误、权限问题等根本原因并修复。**

**来源**: `reference/source_gateway_troubleshooting.md`

## 二、常见原因分析

### 2.1 三大主要原因

| 原因类型 | 日志特征 | 发生频率 | 修复难度 |
|---------|---------|---------|---------|
| 端口 18789 被占用 | `EADDRINUSE`, `already listening` | 60% | 简单 |
| 配置错误 | `gateway.mode not set`, `Config invalid` | 30% | 中等 |
| 权限不足 | `EACCES`, `permission denied` | 10% | 简单 |

## 三、TypeScript 诊断实现

### 3.1 完整诊断类

```typescript
// gateway-startup-diagnostic.ts
import { spawn } from 'child_process';
import * as fs from 'fs';

interface StartupIssue {
  type: 'port_conflict' | 'config_error' | 'permission' | 'unknown';
  severity: 'high' | 'medium' | 'low';
  message: string;
  solution: string;
  autoFixable: boolean;
}

interface DiagnosticResult {
  canStart: boolean;
  issues: StartupIssue[];
  recommendations: string[];
}

class GatewayStartupDiagnostic {
  async diagnose(): Promise<DiagnosticResult> {
    const issues: StartupIssue[] = [];

    console.log('=== Gateway Startup Diagnostic ===\n');

    // 1. 检查 Gateway 当前状态
    const status = await this.checkGatewayStatus();
    if (status === 'running') {
      console.log('✓ Gateway is already running');
      return {
        canStart: true,
        issues: [],
        recommendations: ['Gateway is healthy']
      };
    }

    // 2. 检查端口占用
    const portIssue = await this.checkPortConflict();
    if (portIssue) {
      issues.push(portIssue);
    }

    // 3. 检查配置
    const configIssues = await this.checkConfiguration();
    issues.push(...configIssues);

    // 4. 检查权限
    const permissionIssue = await this.checkPermissions();
    if (permissionIssue) {
      issues.push(permissionIssue);
    }

    // 5. 生成建议
    const recommendations = this.generateRecommendations(issues);

    return {
      canStart: issues.length === 0,
      issues,
      recommendations
    };
  }

  private async checkGatewayStatus(): Promise<string> {
    try {
      const output = await this.runCommand('openclaw', ['gateway', 'status']);
      
      if (output.includes('Runtime: running')) {
        return 'running';
      } else if (output.includes('Runtime: stopped')) {
        return 'stopped';
      } else {
        return 'unknown';
      }
    } catch {
      return 'stopped';
    }
  }

  private async checkPortConflict(): Promise<StartupIssue | null> {
    console.log('Checking port 18789...');

    try {
      // 检查端口是否被占用
      const output = await this.runCommand('lsof', ['-i', ':18789']);
      
      if (output.trim().length > 0) {
        // 端口被占用
        const lines = output.split('\n');
        const processInfo = lines[1]; // 第一行是header，第二行是进程信息
        
        console.log('✗ Port 18789 is in use');
        console.log(`  Process: ${processInfo}`);

        return {
          type: 'port_conflict',
          severity: 'high',
          message: 'Port 18789 is already in use by another process',
          solution: 'Run: openclaw gateway stop OR kill the process using: lsof -ti:18789 | xargs kill -9',
          autoFixable: true
        };
      } else {
        console.log('✓ Port 18789 is available');
        return null;
      }
    } catch (error) {
      // lsof 命令失败可能意味着端口未被占用
      console.log('✓ Port 18789 is available');
      return null;
    }
  }

  private async checkConfiguration(): Promise<StartupIssue[]> {
    console.log('Checking configuration...');
    const issues: StartupIssue[] = [];

    try {
      const configPath = `${process.env.HOME}/.openclaw/openclaw.json`;
      
      // 检查配置文件是否存在
      if (!fs.existsSync(configPath)) {
        console.log('✗ Config file not found');
        issues.push({
          type: 'config_error',
          severity: 'high',
          message: 'Configuration file not found',
          solution: 'Run: openclaw onboard',
          autoFixable: false
        });
        return issues;
      }

      // 读取配置
      const configContent = fs.readFileSync(configPath, 'utf-8');
      const config = JSON.parse(configContent);

      // 检查 gateway.mode
      if (!config.gateway || !config.gateway.mode) {
        console.log('✗ gateway.mode not set');
        issues.push({
          type: 'config_error',
          severity: 'high',
          message: 'gateway.mode is not configured',
          solution: 'Run: openclaw config set gateway.mode local',
          autoFixable: true
        });
      } else {
        console.log(`✓ gateway.mode = ${config.gateway.mode}`);
      }

      // 检查认证配置（如果绑定非回环地址）
      if (config.gateway?.bind && config.gateway.bind !== '127.0.0.1' && config.gateway.bind !== 'localhost') {
        if (!config.gateway.auth?.token && !config.gateway.auth?.password) {
          console.log('✗ Non-loopback binding requires authentication');
          issues.push({
            type: 'config_error',
            severity: 'high',
            message: 'Non-loopback binding requires authentication',
            solution: 'Run: openclaw config set gateway.auth.token <your-token>',
            autoFixable: false
          });
        }
      }

      if (issues.length === 0) {
        console.log('✓ Configuration is valid');
      }

    } catch (error) {
      console.log('✗ Configuration error');
      issues.push({
        type: 'config_error',
        severity: 'high',
        message: `Configuration error: ${error.message}`,
        solution: 'Run: openclaw doctor --repair',
        autoFixable: true
      });
    }

    return issues;
  }

  private async checkPermissions(): Promise<StartupIssue | null> {
    console.log('Checking permissions...');

    try {
      const configDir = `${process.env.HOME}/.openclaw`;
      
      // 检查配置目录权限
      const stats = fs.statSync(configDir);
      const mode = stats.mode & parseInt('777', 8);
      
      if (mode < parseInt('700', 8)) {
        console.log('✗ Insufficient permissions on config directory');
        return {
          type: 'permission',
          severity: 'medium',
          message: 'Insufficient permissions on ~/.openclaw directory',
          solution: 'Run: chmod 700 ~/.openclaw',
          autoFixable: true
        };
      }

      console.log('✓ Permissions are correct');
      return null;
    } catch (error) {
      console.log('⚠ Could not check permissions');
      return null;
    }
  }

  private generateRecommendations(issues: StartupIssue[]): string[] {
    const recommendations: string[] = [];

    // 按严重程度排序
    const sortedIssues = issues.sort((a, b) => {
      const severityOrder = { high: 0, medium: 1, low: 2 };
      return severityOrder[a.severity] - severityOrder[b.severity];
    });

    // 生成修复步骤
    sortedIssues.forEach((issue, index) => {
      recommendations.push(`${index + 1}. ${issue.solution}`);
    });

    // 添加验证步骤
    if (recommendations.length > 0) {
      recommendations.push('');
      recommendations.push('After fixing, verify with:');
      recommendations.push('  openclaw gateway start');
      recommendations.push('  openclaw gateway status');
    }

    return recommendations;
  }

  async autoFix(issues: StartupIssue[]): Promise<void> {
    console.log('\n=== Attempting Auto-Fix ===\n');

    for (const issue of issues) {
      if (!issue.autoFixable) {
        console.log(`⚠ Cannot auto-fix: ${issue.message}`);
        console.log(`  Manual action required: ${issue.solution}`);
        continue;
      }

      console.log(`Fixing: ${issue.message}...`);

      try {
        switch (issue.type) {
          case 'port_conflict':
            await this.fixPortConflict();
            break;
          case 'config_error':
            await this.fixConfigError(issue);
            break;
          case 'permission':
            await this.fixPermissions();
            break;
        }
        console.log('✓ Fixed');
      } catch (error) {
        console.log(`✗ Failed: ${error.message}`);
      }
    }
  }

  private async fixPortConflict(): Promise<void> {
    // 停止 Gateway
    await this.runCommand('openclaw', ['gateway', 'stop']);
    
    // 等待端口释放
    await this.sleep(2000);
    
    // 如果还有进程占用，强制杀死
    try {
      const pids = await this.runCommand('lsof', ['-ti', ':18789']);
      if (pids.trim()) {
        await this.runCommand('kill', ['-9', ...pids.trim().split('\n')]);
      }
    } catch {
      // 忽略错误
    }
  }

  private async fixConfigError(issue: StartupIssue): Promise<void> {
    if (issue.message.includes('gateway.mode')) {
      await this.runCommand('openclaw', ['config', 'set', 'gateway.mode', 'local']);
    } else {
      await this.runCommand('openclaw', ['doctor', '--repair']);
    }
  }

  private async fixPermissions(): Promise<void> {
    const configDir = `${process.env.HOME}/.openclaw`;
    await this.runCommand('chmod', ['700', configDir]);
  }

  private runCommand(cmd: string, args: string[]): Promise<string> {
    return new Promise((resolve, reject) => {
      const proc = spawn(cmd, args);
      let output = '';

      proc.stdout.on('data', (data) => {
        output += data.toString();
      });

      proc.stderr.on('data', (data) => {
        output += data.toString();
      });

      proc.on('close', (code) => {
        if (code === 0) {
          resolve(output);
        } else {
          reject(new Error(`Command failed with code ${code}`));
        }
      });
    });
  }

  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }
}

// 使用示例
async function main() {
  const diagnostic = new GatewayStartupDiagnostic();
  
  // 运行诊断
  const result = await diagnostic.diagnose();
  
  // 输出结果
  console.log('\n=== Diagnostic Result ===');
  console.log(`Can Start: ${result.canStart ? 'YES' : 'NO'}`);
  
  if (result.issues.length > 0) {
    console.log('\nIssues Found:');
    result.issues.forEach(issue => {
      console.log(`  [${issue.severity.toUpperCase()}] ${issue.message}`);
    });
    
    console.log('\nRecommendations:');
    result.recommendations.forEach(rec => {
      console.log(`  ${rec}`);
    });
    
    // 尝试自动修复
    const autoFixableIssues = result.issues.filter(i => i.autoFixable);
    if (autoFixableIssues.length > 0) {
      await diagnostic.autoFix(autoFixableIssues);
      
      // 重新诊断
      console.log('\n=== Re-diagnosing ===\n');
      const newResult = await diagnostic.diagnose();
      console.log(`Can Start Now: ${newResult.canStart ? 'YES' : 'NO'}`);
    }
  } else {
    console.log('✓ No issues found, gateway should start successfully');
  }
}

main();
```

## 四、Bash 排查脚本

### 4.1 完整排查脚本

```bash
#!/bin/bash
# gateway-startup-troubleshoot.sh

echo "=== OpenClaw Gateway Startup Troubleshooting ==="
echo "Start time: $(date)"
echo ""

# 颜色定义
GREEN='\033[0;32m'
RED='\033[0;31m'
YELLOW='\033[1;33m'
NC='\033[0m'

# 问题计数
ISSUES_FOUND=0

# 1. 检查 Gateway 状态
echo "[1/5] Checking Gateway status..."
STATUS=$(openclaw gateway status 2>&1)

if echo "$STATUS" | grep -q "Runtime: running"; then
  echo -e "${GREEN}✓${NC} Gateway is already running"
  echo "No troubleshooting needed."
  exit 0
elif echo "$STATUS" | grep -q "Runtime: stopped"; then
  echo -e "${YELLOW}⚠${NC} Gateway is stopped"
else
  echo -e "${RED}✗${NC} Gateway status unknown"
fi

# 2. 检查端口占用
echo ""
echo "[2/5] Checking port 18789..."
PORT_CHECK=$(lsof -i :18789 2>&1)

if [ -n "$PORT_CHECK" ] && ! echo "$PORT_CHECK" | grep -q "Cannot open"; then
  echo -e "${RED}✗${NC} Port 18789 is in use"
  echo "  Process info:"
  echo "$PORT_CHECK" | head -n 2
  ISSUES_FOUND=$((ISSUES_FOUND + 1))
  
  # 提供修复建议
  echo ""
  echo "  Fix options:"
  echo "    A) Stop OpenClaw gateway: openclaw gateway stop"
  echo "    B) Kill process: lsof -ti:18789 | xargs kill -9"
else
  echo -e "${GREEN}✓${NC} Port 18789 is available"
fi

# 3. 检查配置
echo ""
echo "[3/5] Checking configuration..."
CONFIG_PATH="$HOME/.openclaw/openclaw.json"

if [ ! -f "$CONFIG_PATH" ]; then
  echo -e "${RED}✗${NC} Config file not found: $CONFIG_PATH"
  echo "  Run: openclaw onboard"
  ISSUES_FOUND=$((ISSUES_FOUND + 1))
else
  # 检查 JSON 语法
  if ! jq empty "$CONFIG_PATH" 2>/dev/null; then
    echo -e "${RED}✗${NC} Config file has syntax errors"
    echo "  Run: openclaw doctor --repair"
    ISSUES_FOUND=$((ISSUES_FOUND + 1))
  else
    # 检查 gateway.mode
    GATEWAY_MODE=$(jq -r '.gateway.mode // empty' "$CONFIG_PATH")
    if [ -z "$GATEWAY_MODE" ]; then
      echo -e "${RED}✗${NC} gateway.mode not set"
      echo "  Run: openclaw config set gateway.mode local"
      ISSUES_FOUND=$((ISSUES_FOUND + 1))
    else
      echo -e "${GREEN}✓${NC} gateway.mode = $GATEWAY_MODE"
    fi
  fi
fi

# 4. 检查权限
echo ""
echo "[4/5] Checking permissions..."
CONFIG_DIR="$HOME/.openclaw"

if [ ! -d "$CONFIG_DIR" ]; then
  echo -e "${RED}✗${NC} Config directory not found"
  echo "  Run: openclaw onboard"
  ISSUES_FOUND=$((ISSUES_FOUND + 1))
else
  PERMS=$(stat -f "%Lp" "$CONFIG_DIR" 2>/dev/null || stat -c "%a" "$CONFIG_DIR" 2>/dev/null)
  if [ "$PERMS" -lt 700 ]; then
    echo -e "${YELLOW}⚠${NC} Insufficient permissions: $PERMS"
    echo "  Run: chmod 700 $CONFIG_DIR"
    ISSUES_FOUND=$((ISSUES_FOUND + 1))
  else
    echo -e "${GREEN}✓${NC} Permissions are correct: $PERMS"
  fi
fi

# 5. 检查日志中的错误
echo ""
echo "[5/5] Checking recent logs..."
LOGS=$(openclaw logs --tail 50 2>&1)

if echo "$LOGS" | grep -q "EADDRINUSE"; then
  echo -e "${RED}✗${NC} Found EADDRINUSE error in logs"
  echo "  Port conflict detected"
  ISSUES_FOUND=$((ISSUES_FOUND + 1))
elif echo "$LOGS" | grep -q "Gateway start blocked"; then
  echo -e "${RED}✗${NC} Found 'Gateway start blocked' in logs"
  echo "  Configuration issue detected"
  ISSUES_FOUND=$((ISSUES_FOUND + 1))
elif echo "$LOGS" | grep -q "EACCES"; then
  echo -e "${RED}✗${NC} Found EACCES error in logs"
  echo "  Permission issue detected"
  ISSUES_FOUND=$((ISSUES_FOUND + 1))
else
  echo -e "${GREEN}✓${NC} No critical errors in recent logs"
fi

# 总结
echo ""
echo "=== Summary ==="
echo "Issues found: $ISSUES_FOUND"

if [ $ISSUES_FOUND -eq 0 ]; then
  echo -e "${GREEN}✓ No issues detected${NC}"
  echo ""
  echo "Try starting the gateway:"
  echo "  openclaw gateway start"
  exit 0
else
  echo -e "${RED}✗ Issues need to be resolved${NC}"
  echo ""
  echo "Quick fix steps:"
  echo "  1. Stop any running instances: openclaw gateway stop"
  echo "  2. Run auto-repair: openclaw doctor --repair"
  echo "  3. Start gateway: openclaw gateway start"
  echo "  4. Verify: openclaw gateway status"
  exit 1
fi
```

### 4.2 一键修复脚本

```bash
#!/bin/bash
# gateway-quick-fix.sh

echo "=== OpenClaw Gateway Quick Fix ==="
echo ""

# 1. 停止 Gateway
echo "[1/4] Stopping gateway..."
openclaw gateway stop
sleep 2

# 2. 清理端口
echo "[2/4] Cleaning up port 18789..."
PIDS=$(lsof -ti:18789 2>/dev/null)
if [ -n "$PIDS" ]; then
  echo "  Killing processes: $PIDS"
  echo "$PIDS" | xargs kill -9
  sleep 1
fi

# 3. 修复配置
echo "[3/4] Repairing configuration..."
openclaw doctor --repair

# 4. 启动 Gateway
echo "[4/4] Starting gateway..."
openclaw gateway start
sleep 3

# 验证
echo ""
echo "=== Verification ==="
openclaw gateway status

if openclaw gateway status | grep -q "Runtime: running"; then
  echo ""
  echo "✓ Gateway started successfully!"
  exit 0
else
  echo ""
  echo "✗ Gateway failed to start"
  echo "Check logs: openclaw logs --follow"
  exit 1
fi
```

## 五、类比理解

### 5.1 后端开发类比

**类比 1：Nginx 启动失败**

```bash

# OpenClaw
openclaw doctor             # 测试配置
openclaw gateway start      # 启动服务
openclaw gateway status     # 检查状态

# 相似的问题：
# 1. 端口被占用（80/443 vs 18789）
# 2. 配置文件错误
# 3. 权限不足
```

**类比 2：Docker 容器启动失败**

```bash
# OpenClaw Gateway ≈ Docker Container

docker ps                   # 查看运行状态
docker logs <container>     # 查看日志
docker start <container>    # 启动容器

# OpenClaw
openclaw gateway status     # 查看运行状态
openclaw logs --follow      # 查看日志

# 相似的排查流程：
# 1. 检查状态
# 2. 查看日志
# 3. 修复问题
# 4. 重新启动
```

### 5.2 日常生活类比

**类比 1：汽车无法启动**

```
OpenClaw Gateway 无法启动 = 汽车无法启动

相似点：
1. 检查电池（端口占用）
2. 检查油量（配置完整性）
3. 检查钥匙（权限）

```

```
Gateway 启动失败 = 门锁打不开

可能原因：
1. 钥匙错误（配置错误）
2. 锁芯卡住（端口占用）
3. 门把手坏了（权限问题）
4. 需要逐个排查
```

### 6.3 如何预防启动失败？

**最佳实践**：

```bash
# 1. 定期健康检查
openclaw doctor

# 2. 升级前备份
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.pre-upgrade

# 3. 升级后验证
openclaw doctor --repair
openclaw gateway restart

# 4. 监控脚本
#!/bin/bash
while true; do
  if ! openclaw gateway status | grep -q "Runtime: running"; then

  fi

done
```

## 七、总结

### 7.1 核心要点

1. **三大主要原因**：端口冲突（60%）、配置错误（30%）、权限问题（10%）
2. **系统化诊断**：状态 → 端口 → 配置 → 权限 → 日志
3. **自动修复能力**：TypeScript + Bash 双重实现
4. **快速修复流程**：停止 → 清理 → 修复 → 启动 → 验证
5. **预防措施**：定期检查、升级备份、监控告警

### 7.2 使用建议

- ✅ 遇到启动失败先运行诊断脚本
- ✅ 优先检查端口占用（最常见）
- ✅ 使用 doctor --repair 自动修复配置
- ✅ 保留配置备份
- ❌ 不要直接编辑配置文件
