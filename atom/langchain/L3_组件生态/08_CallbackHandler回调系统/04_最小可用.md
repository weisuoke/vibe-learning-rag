## 场景1：监控 LLM 调用（最基础）

**需求**：我想看到 LLM 的输入和输出

**最小可用代码**：

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import StdOutCallbackHandler

# 初始化 LLM，传入内置回调处理器
llm = ChatOpenAI(
    temperature=0,
    callbacks=[StdOutCallbackHandler()]  # 打印执行过程
)

# 调用 LLM
response = llm.invoke("什么是 RAG？")
print(f"\n最终答案: {response.content}")
```

**运行效果**：
```
> Entering new ChatOpenAI chain...
Prompt: 什么是 RAG？
> Finished chain.

最终答案: RAG（Retrieval-Augmented Generation）是检索增强生成...
```

**核心要点**：
- 使用内置的 `StdOutCallbackHandler`
- 在初始化时传入 `callbacks` 参数
- 自动打印执行过程

## 场景2：自定义日志回调（最常用）

**需求**：我想记录 LLM 的 token 用量和成本

**最小可用代码**：

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import BaseCallbackHandler
from langchain_core.outputs import LLMResult

class TokenCounterHandler(BaseCallbackHandler):
    """统计 token 用量的回调处理器"""

    def __init__(self):
        self.total_tokens = 0
        self.total_cost = 0

    def on_llm_end(self, response: LLMResult, **kwargs) -> None:
        """LLM 结束时统计 token"""
        # 获取 token 用量
        token_usage = response.llm_output.get("token_usage", {})
        tokens = token_usage.get("total_tokens", 0)

        # 累加 token
        self.total_tokens += tokens

        # 计算成本（假设 $0.002 / 1K tokens）
        cost = (tokens / 1000) * 0.002
        self.total_cost += cost

        print(f"本次调用: {tokens} tokens, ${cost:.4f}")
        print(f"累计: {self.total_tokens} tokens, ${self.total_cost:.4f}")

# 使用自定义回调
handler = TokenCounterHandler()
llm = ChatOpenAI(temperature=0, callbacks=[handler])

# 调用 LLM
llm.invoke("什么是 RAG？")
llm.invoke("RAG 的优势是什么？")

# 查看统计结果
print(f"\n总计: {handler.total_tokens} tokens, ${handler.total_cost:.4f}")
```

**运行效果**：
```
本次调用: 150 tokens, $0.0003
累计: 150 tokens, $0.0003
本次调用: 120 tokens, $0.0002
累计: 270 tokens, $0.0005

总计: 270 tokens, $0.0005
```

**核心要点**：
- 继承 `BaseCallbackHandler`
- 实现 `on_llm_end()` 方法
- 从 `response.llm_output` 获取 token 信息
- 可以累加统计多次调用

## 场景3：RAG 系统监控（最实用）

**需求**：我想监控 RAG 系统的检索和生成过程

**最小可用代码**：

```python
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_core.callbacks import BaseCallbackHandler
from langchain.chains import RetrievalQA
from langchain_core.documents import Document

class RAGMonitorHandler(BaseCallbackHandler):
    """监控 RAG 系统的回调处理器"""

    def on_retriever_end(self, documents, **kwargs) -> None:
        """检索结束时触发"""
        print(f"\n检索到 {len(documents)} 个文档:")
        for i, doc in enumerate(documents[:3], 1):  # 只显示前3个
            print(f"  {i}. {doc.page_content[:50]}...")

    def on_llm_start(self, serialized, prompts, **kwargs) -> None:
        """LLM 开始时触发"""
        print(f"\n开始生成答案...")

    def on_llm_end(self, response, **kwargs) -> None:
        """LLM 结束时触发"""
        tokens = response.llm_output.get("token_usage", {}).get("total_tokens", 0)
        print(f"生成完成，使用 {tokens} tokens")

# 准备测试数据
docs = [
    Document(page_content="RAG 是检索增强生成技术"),
    Document(page_content="RAG 结合了检索和生成两个步骤"),
    Document(page_content="RAG 可以提高 LLM 的准确性")
]

# 创建向量存储
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(docs, embeddings)

# 创建 RAG 链
handler = RAGMonitorHandler()
llm = ChatOpenAI(temperature=0, callbacks=[handler])
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=vectorstore.as_retriever(callbacks=[handler]),
    callbacks=[handler]
)

# 执行查询
result = qa_chain.invoke({"query": "什么是 RAG？"})
print(f"\n最终答案: {result['result']}")
```

**运行效果**：
```
检索到 3 个文档:
  1. RAG 是检索增强生成技术...
  2. RAG 结合了检索和生成两个步骤...
  3. RAG 可以提高 LLM 的准确性...

开始生成答案...
生成完成，使用 180 tokens

最终答案: RAG（Retrieval-Augmented Generation）是一种...
```

**核心要点**：
- 实现 `on_retriever_end()` 监控检索
- 实现 `on_llm_start()` 和 `on_llm_end()` 监控生成
- 同一个回调处理器可以传递给多个组件
- 完整追踪 RAG 管道的执行过程

## 场景4：流式输出（最酷炫）

**需求**：我想实时看到 LLM 生成的每个 token

**最小可用代码**：

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import StreamingStdOutCallbackHandler

# 使用内置的流式回调处理器
llm = ChatOpenAI(
    temperature=0.7,
    streaming=True,  # 启用流式输出
    callbacks=[StreamingStdOutCallbackHandler()]
)

# 调用 LLM，实时输出
print("开始生成:")
response = llm.invoke("用一句话解释 RAG")
print(f"\n\n完成！")
```

**运行效果**：
```
开始生成:
RAG 是一种结合检索和生成的技术，通过从知识库中检索相关信息来增强大语言模型的回答准确性。

完成！
```

**核心要点**：
- 使用 `StreamingStdOutCallbackHandler`
- 设置 `streaming=True`
- 自动实时输出每个 token

## 场景5：错误捕获（最安全）

**需求**：我想捕获 LLM 调用的错误

**最小可用代码**：

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import BaseCallbackHandler

class ErrorHandler(BaseCallbackHandler):
    """捕获错误的回调处理器"""

    def on_llm_error(self, error: BaseException, **kwargs) -> None:
        """LLM 错误时触发"""
        print(f"LLM 错误: {error}")
        print(f"错误类型: {type(error).__name__}")

    def on_chain_error(self, error: BaseException, **kwargs) -> None:
        """Chain 错误时触发"""
        print(f"Chain 错误: {error}")

# 使用错误处理回调
handler = ErrorHandler()
llm = ChatOpenAI(
    temperature=0,
    api_key="invalid_key",  # 故意使用无效 key
    callbacks=[handler]
)

# 尝试调用（会触发错误）
try:
    llm.invoke("测试")
except Exception as e:
    print(f"捕获异常: {e}")
```

**运行效果**：
```
LLM 错误: Incorrect API key provided
错误类型: AuthenticationError
捕获异常: Incorrect API key provided
```

**核心要点**：
- 实现 `on_llm_error()` 捕获 LLM 错误
- 实现 `on_chain_error()` 捕获 Chain 错误
- 可以记录错误日志或发送告警

## 最小可用总结

### 5个核心场景

| 场景 | 使用的回调 | 核心方法 | 代码行数 |
|------|-----------|---------|---------|
| 监控 LLM 调用 | `StdOutCallbackHandler` | 无需实现 | 5行 |
| 统计 token 用量 | 自定义 `BaseCallbackHandler` | `on_llm_end()` | 20行 |
| 监控 RAG 系统 | 自定义 `BaseCallbackHandler` | `on_retriever_end()`, `on_llm_start/end()` | 30行 |
| 流式输出 | `StreamingStdOutCallbackHandler` | 无需实现 | 5行 |
| 错误捕获 | 自定义 `BaseCallbackHandler` | `on_llm_error()`, `on_chain_error()` | 15行 |

### 3个核心步骤

```python
# 步骤1：定义回调处理器
class MyHandler(BaseCallbackHandler):
    def on_llm_end(self, response, **kwargs):
        print("LLM 完成")

# 步骤2：创建回调实例
handler = MyHandler()

# 步骤3：传入 callbacks 参数
llm = ChatOpenAI(callbacks=[handler])
```

### 1个核心原则

**在需要监控的组件初始化时传入 `callbacks` 参数**

## 常见问题

### Q1: 为什么我的回调方法没有被调用？

**A**: 检查以下几点：
1. 是否在组件初始化时传入了 `callbacks` 参数
2. 方法名是否正确（如 `on_llm_end` 而不是 `on_llm_finish`）
3. 是否继承了 `BaseCallbackHandler`

### Q2: 如何同时使用多个回调处理器？

**A**: 传入回调列表：
```python
llm = ChatOpenAI(callbacks=[handler1, handler2, handler3])
```

### Q3: 回调处理器会影响性能吗？

**A**: 影响很小，除非在回调中执行耗时操作（如网络请求、大量计算）。建议：
- 回调逻辑尽量简单
- 耗时操作使用异步或后台任务
