# 核心概念 7：混合查询执行策略

> **目标**：掌握 Milvus 混合查询的执行策略和性能优化方法

---

## 概述

混合查询执行策略决定了标量过滤和向量检索的执行顺序，合理选择策略可以显著提升查询性能和召回率。

**核心价值**：
- 优化查询性能
- 平衡召回率和延迟
- 自动策略选择
- 适应不同场景

## 1. Pre-filtering（前置过滤）

### 1.1 基本原理

**定义**：
- 先应用标量过滤，得到候选集
- 在候选集中进行向量检索
- 减少向量检索的搜索空间

**执行流程**：
```
1. 应用标量过滤 → 候选集
2. 在候选集中进行向量检索 → Top-K 结果
3. 返回结果
```

**示例**：
```python
# 查询：找到价格在100-500元、有库存的相似商品
filter = "price >= 100 AND price <= 500 AND stock > 0"

# 执行流程：
# 1. 标量过滤：100万条 → 10万条（保留10%）
# 2. 向量检索：在10万条中检索 → Top-10
```

### 1.2 适用场景

**高选择性过滤**：
```python
# 选择性 > 50%（过滤掉大部分数据）
"stock > 0"  # 假设只有10%有库存
"vip_level >= 5"  # 假设只有5%是高级VIP
"status == 'active'"  # 假设只有20%是活跃状态
```

**小候选集**：
```python
# 候选集 < 10万条
"brand == 'Apple' AND price >= 5000"  # 候选集：5000条
```

### 1.3 性能特点

**优势**：
- 减少向量检索的搜索空间
- 降低计算成本
- 提高查询速度

**劣势**：
- 如果候选集太小，可能召回不足
- 可能无法返回足够的结果

**性能公式**：
```
查询延迟 = 标量过滤延迟 + 向量检索延迟（候选集）

其中：
向量检索延迟（候选集）< 向量检索延迟（全集）
```

## 2. Post-filtering（后置过滤）

### 2.1 基本原理

**定义**：
- 先进行向量检索，得到 Top-K 结果
- 对结果应用标量过滤
- 可能需要检索更多结果才能满足 limit

**执行流程**：
```
1. 向量检索 → Top-K 结果（K > limit）
2. 应用标量过滤 → 过滤后的结果
3. 如果结果不足，继续检索更多
4. 返回结果
```

**示例**：
```python
# 查询：找到价格在100-500元的相似商品（Top-10）
filter = "price >= 100 AND price <= 500"

# 执行流程：
# 1. 向量检索：Top-100（预留空间）
# 2. 标量过滤：100条 → 80条（保留80%）
# 3. 返回：Top-10
```

### 2.2 适用场景

**低选择性过滤**：
```python
# 选择性 < 50%（保留大部分数据）
"created_at > '2020-01-01'"  # 假设保留90%数据
"price >= 10"  # 假设保留95%数据
"age >= 18"  # 假设保留80%数据
```

**大候选集**：
```python
# 候选集 > 50万条
"category == 'electronics'"  # 候选集：50万条
```

### 2.3 性能特点

**优势**：
- 保证召回率
- 向量检索不受影响
- 适用于低选择性过滤

**劣势**：
- 可能需要检索更多结果才能满足 limit
- 增加向量检索成本

**性能公式**：
```
查询延迟 = 向量检索延迟（全集，K'） + 标量过滤延迟

其中：
K' = limit / 过滤选择性
例如：limit=10, 选择性=50%, 则 K'=20
```

## 3. Hybrid（混合策略）

### 3.1 基本原理

**定义**：
- Milvus 自动根据过滤选择性选择策略
- 选择性高 → Pre-filtering
- 选择性低 → Post-filtering
- 动态调整

**决策逻辑**：
```python
IF 过滤选择性 > 阈值（如50%）:
    使用 Pre-filtering
ELSE:
    使用 Post-filtering
```

### 3.2 选择性计算

**选择性定义**：
```
选择性 = 过滤后数据量 / 总数据量

例如：
总数据量：100万条
过滤后：10万条
选择性 = 10万 / 100万 = 10%（高选择性）
```

**选择性估算**：
```python
# Milvus 使用统计信息估算选择性
# 1. 索引统计
# 2. 历史查询
# 3. 数据分布
```

### 3.3 自动策略选择

**Milvus 自动选择**：
```python
# 用户只需提供 filter，Milvus 自动选择策略
results = client.search(
    collection_name="products",
    data=[query_vector],
    filter="price >= 100 AND price <= 500 AND stock > 0",
    limit=10
)

# Milvus 内部：
# 1. 估算选择性
# 2. 选择策略（Pre-filtering 或 Post-filtering）
# 3. 执行查询
```

## 4. 性能对比

### 4.1 不同选择性下的性能

**测试场景**：100万条记录，Top-10 查询

| 选择性 | Pre-filtering | Post-filtering | Hybrid |
|--------|---------------|----------------|--------|
| 10% | 50ms | 200ms | 50ms |
| 30% | 80ms | 180ms | 80ms |
| 50% | 120ms | 150ms | 120ms |
| 70% | 180ms | 130ms | 130ms |
| 90% | 250ms | 110ms | 110ms |

**结论**：
- 高选择性（< 50%）：Pre-filtering 更快
- 低选择性（> 50%）：Post-filtering 更快
- Hybrid 自动选择最优策略

### 4.2 召回率对比

**测试场景**：100万条记录，Top-10 查询

| 选择性 | Pre-filtering | Post-filtering |
|--------|---------------|----------------|
| 10% | 95% | 100% |
| 30% | 98% | 100% |
| 50% | 99% | 100% |
| 70% | 100% | 100% |
| 90% | 100% | 100% |

**结论**：
- Pre-filtering 在高选择性时可能召回不足
- Post-filtering 始终保证召回率
- 选择性越低，Pre-filtering 召回率越高

## 5. 优化策略

### 5.1 索引优化

**为过滤字段创建索引**：
```python
# 提升标量过滤性能
index_params.add_index("price", "INVERTED")
index_params.add_index("stock", "INVERTED")
index_params.add_index("brand", "BITMAP")
```

**性能提升**：
- 有索引：标量过滤延迟 30ms
- 无索引：标量过滤延迟 1000ms
- 提升：33倍

### 5.2 查询优化

**选择性高的条件在前**：
```python
# 好的写法
"stock > 0 AND price >= 100 AND price <= 500"
# stock > 0 选择性高（10%），先执行

# 不好的写法
"price >= 100 AND price <= 500 AND stock > 0"
# 价格范围选择性低（80%），先执行
```

**使用 IN 替代多个 OR**：
```python
# 不推荐
"city == '北京' OR city == '上海' OR city == '深圳'"

# 推荐
"city IN ['北京', '上海', '深圳']"
```

### 5.3 参数调优

**调整 limit 参数**：
```python
# Post-filtering 时，增加 limit 以保证召回
# 如果选择性 = 50%，limit = 10
# 实际检索：limit = 10 / 0.5 = 20

results = client.search(
    collection_name="products",
    data=[query_vector],
    filter="price >= 100 AND price <= 500",
    limit=10  # Milvus 自动调整实际检索数量
)
```

## 6. 实战示例

### 6.1 高选择性场景（Pre-filtering）

```python
# 场景：查找有库存的高端商品
filter = "stock > 0 AND price >= 5000"
# 选择性：5%（只有5%的商品有库存且价格>=5000）

# 执行策略：Pre-filtering
# 1. 标量过滤：100万条 → 5万条
# 2. 向量检索：在5万条中检索 → Top-10
# 3. 延迟：50ms

results = client.search(
    collection_name="products",
    data=[query_vector],
    filter=filter,
    limit=10
)
```

### 6.2 低选择性场景（Post-filtering）

```python
# 场景：查找最近发布的商品
filter = "publish_date > '2025-01-01'"
# 选择性：80%（80%的商品是最近发布的）

# 执行策略：Post-filtering
# 1. 向量检索：Top-20（预留空间）
# 2. 标量过滤：20条 → 16条（保留80%）
# 3. 返回：Top-10
# 4. 延迟：120ms

results = client.search(
    collection_name="products",
    data=[query_vector],
    filter=filter,
    limit=10
)
```

### 6.3 混合场景（Hybrid）

```python
# 场景：查找特定价格范围的商品
filter = "price >= 100 AND price <= 500"
# 选择性：50%（50%的商品在这个价格范围）

# 执行策略：Hybrid（Milvus 自动选择）
# 1. 估算选择性：50%
# 2. 选择策略：Pre-filtering 或 Post-filtering
# 3. 执行查询

    collection_name="products",
```

## 7. 常见问题

### 7.1 召回不足

**问题**：
```python
# Pre-filtering 导致召回不足
filter = "stock > 0"  # 只有1000条有库存
limit = 10

# 结果：只返回5条（候选集太小）
```

**解决方案**：
```python
# 方案1：放宽过滤条件
filter = "stock >= 0"  # 包含库存为0的商品

# 方案2：增加 limit
limit = 20  # 增加返回数量

# 方案3：使用 Post-filtering
# Milvus 会自动选择 Post-filtering
```

### 7.2 性能问题

**问题**：
```python
# 无索引导致标量过滤慢
filter = "price >= 100 AND price <= 500"
# 延迟：1000ms（无索引）
```

**解决方案**：
```python
# 创建索引
index_params.add_index("price", "INVERTED")
# 延迟：30ms（有索引）
```

### 7.3 选择性估算不准

**问题**：
```python
# Milvus 估算选择性不准，选择了错误的策略
filter = "status == 'active'"
# 实际选择性：10%
# 估算选择性：50%
# 选择策略：Post-filtering（错误）
```

**解决方案**：
```python
# 方案1：更新统计信息
client.flush(collection_name)
client.compact(collection_name)

# 方案2：手动优化查询
# 将选择性高的条件放在前面
filter = "status == 'active' AND price >= 100"
```

## 8. 类比理解

### 前端开发类比

**Pre-filtering = 先筛选再排序**：
```javascript
// JavaScript
const filtered = products.filter(p => p.stock > 0);  // 先筛选
const sorted = filtered.sort((a, b) => similarity(a) - similarity(b));  // 再排序
const top10 = sorted.slice(0, 10);

// Milvus Pre-filtering
filter = "stock > 0"  // 先筛选
// 然后在筛选结果中进行向量检索
```

**Post-filtering = 先排序再筛选**：
```javascript
// JavaScript
const sorted = products.sort((a, b) => similarity(a) - similarity(b));  // 先排序
const top100 = sorted.slice(0, 100);  // 取前100
const filtered = top100.filter(p => p.stock > 0);  // 再筛选
const top10 = filtered.slice(0, 10);

// Milvus Post-filtering
// 先进行向量检索（Top-100）
filter = "stock > 0"  // 然后筛选
```

### 数据库类比

**Pre-filtering = WHERE + ORDER BY**：
```sql
-- SQL
SELECT * FROM products
WHERE stock > 0  -- 先筛选
ORDER BY similarity DESC  -- 再排序
LIMIT 10;

-- Milvus Pre-filtering
filter = "stock > 0"
```

**Post-filtering = ORDER BY + WHERE**：
```sql
-- SQL（伪代码）
SELECT * FROM (
    SELECT * FROM products
    ORDER BY similarity DESC
    LIMIT 100  -- 先排序
) WHERE stock > 0  -- 再筛选
LIMIT 10;

-- Milvus Post-filtering
```
