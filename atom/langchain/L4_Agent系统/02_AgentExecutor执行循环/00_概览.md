# AgentExecutor 执行循环

> AgentExecutor 是 LangChain 的 Agent 执行引擎，通过 Think→Act→Observe 的 while 循环驱动 LLM 自主完成多步推理任务

---

## 知识点定位

- **所属层级**: L4_Agent系统 - 第 2 个知识点
- **重要程度**: 极高（理解 Agent 运行机制的核心）
- **核心文件**: `langchain_classic/agents/agent.py`（AgentExecutor 类，约 61K）
- **核心位置**: AgentExecutor 类定义于 `agent.py:1012`，主循环在 `agent.py:1570`

### 为什么这个知识点重要？

Agent 的"智能"不在于单次 LLM 调用，而在于**循环**。一个 Agent 能自主完成复杂任务，靠的就是 AgentExecutor 内部的 while 循环：

1. 调用 LLM 决定下一步做什么（Think）
2. 执行选中的工具获取结果（Act）
3. 把结果反馈给 LLM（Observe）
4. 重复，直到 LLM 认为任务完成或触发安全限制

这个循环就是 Agent 的"心跳"。不理解它，就无法调试 Agent 行为、优化执行效率、处理异常情况。

### 与 create_agent 的关系

在 LangChain 1.0（2025年10月发布）中，`create_agent()` 基于 LangGraph 取代了 AgentExecutor 作为推荐的 Agent 创建方式。但**执行循环的核心思想完全一致**——都是 Think→Act→Observe 的迭代模式。理解 AgentExecutor 的循环机制，是理解所有 Agent 系统的基础。

## 核心执行流程

```
┌─────────────────────────────────────────────────────────┐
│                   AgentExecutor._call()                 │
│                                                         │
│  初始化: intermediate_steps=[], iterations=0             │
│                                                         │
│  ┌───────────────────────────────────────────────────┐  │
│  │         while _should_continue(iterations):       │  │
│  │                                                   │  │
│  │  ┌─────────────┐                                  │  │
│  │  │   THINK     │  agent.plan(inputs,              │  │
│  │  │  (LLM调用)  │    intermediate_steps)           │  │
│  │  └──────┬──────┘                                  │  │
│  │         │                                         │  │
│  │         ▼                                         │  │
│  │    ┌─────────┐  Yes   ┌──────────────┐           │  │
│  │    │AgentFin-├───────→│ return 最终结果│           │  │
│  │    │ish?     │        └──────────────┘           │  │
│  │    └────┬────┘                                    │  │
│  │         │ No (AgentAction)                        │  │
│  │         ▼                                         │  │
│  │  ┌─────────────┐                                  │  │
│  │  │    ACT      │  tool.run(action.tool_input)     │  │
│  │  │  (执行工具)  │                                  │  │
│  │  └──────┬──────┘                                  │  │
│  │         │                                         │  │
│  │         ▼                                         │  │
│  │  ┌─────────────┐                                  │  │
│  │  │  OBSERVE    │  intermediate_steps.extend(      │  │
│  │  │ (收集结果)   │    AgentStep(action, obs))       │  │
│  │  └──────┬──────┘                                  │  │
│  │         │                                         │  │
│  │         │  iterations += 1                        │  │
│  │         └──────────→ 回到 while 判断              │  │
│  │                                                   │  │
│  └───────────────────────────────────────────────────┘  │
│                                                         │
│  超出限制 → early_stopping_method 处理                   │
│    "force": 返回固定停止消息                              │
│    "generate": 让 LLM 基于已有步骤生成最终答案            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 核心数据类型

执行循环围绕三个核心数据类型运转（定义在 `langchain_core/agents.py`）：

```
AgentAction                AgentFinish               AgentStep
┌──────────────┐          ┌──────────────┐          ┌──────────────┐
│ tool: str    │          │ return_values│          │ action:      │
│ tool_input:  │          │   : dict     │          │  AgentAction │
│   str|dict   │          │ log: str     │          │ observation: │
│ log: str     │          └──────────────┘          │   str        │
└──────────────┘                                    └──────────────┘
  "我要做什么"              "我完成了"                "做了什么+结果"
```

- **AgentAction**: LLM 决定要执行的动作（工具名 + 输入参数）
- **AgentFinish**: LLM 决定任务完成，返回最终答案
- **AgentStep**: 一次完整的 Action + Observation 记录

## 与 RAG 和 AI Agent 的关系

### RAG 场景中的 Agent 循环

在 RAG 系统中，Agent 循环让检索变得"智能"：

```
用户问题: "对比 React 和 Vue 的性能差异"
    │
    ▼
Agent Think: 需要分别检索两个框架的性能数据
    │
    ▼
Agent Act: 调用 retriever.search("React 性能特点")
    │
    ▼
Agent Observe: 获取 React 相关文档片段
    │
    ▼
Agent Think: 还需要 Vue 的数据
    │
    ▼
Agent Act: 调用 retriever.search("Vue 性能特点")
    │
    ▼
Agent Observe: 获取 Vue 相关文档片段
    │
    ▼
Agent Think: 信息足够了，可以生成对比分析
    │
    ▼
AgentFinish: 输出完整的对比分析报告
```

普通 RAG 只做一次检索，而 Agent 驱动的 RAG 可以**多次检索、交叉验证、自主补充信息**，这就是执行循环的价值。

### AI Agent 开发中的核心地位

AgentExecutor 的执行循环是所有 Agent 框架的共同模式：

| 框架 | 循环实现 | 本质 |
|------|----------|------|
| LangChain AgentExecutor | `_call` while 循环 | Think→Act→Observe |
| LangChain create_agent | LangGraph 状态机 | Think→Act→Observe |
| AutoGPT | main loop | Think→Act→Observe |
| BabyAGI | task loop | Think→Act→Observe |
| CrewAI | agent loop | Think→Act→Observe |

**掌握这个循环模式，就掌握了所有 Agent 系统的核心。**
