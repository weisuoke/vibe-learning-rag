# 核心概念 3：守护进程管理

## 概述

守护进程管理是 OpenClaw Gateway 生产部署的核心机制，通过操作系统的服务管理器（macOS 的 launchd 或 Linux 的 systemd）实现 Gateway 的持久运行、自动重启和系统集成。

## 一句话定义

**守护进程管理是将 Gateway 作为系统服务运行的机制，通过 launchd（macOS）或 systemd（Linux）实现开机自启、崩溃自动重启和独立于终端会话的持久运行。**

**参考资料：**
- [OpenClaw GitHub](https://github.com/openclaw/openclaw) - 官方仓库，守护进程实现
- [Gateway Runbook](https://docs.openclaw.ai/gateway) - 守护进程管理文档
- [Getting Started](https://docs.openclaw.ai/start/getting-started) - 快速入门指南

## 第一性原理：为什么需要守护进程？

### 问题根源

生产环境的服务需要满足以下需求：

1. **持久运行**：服务需要 24/7 运行，不能因为终端关闭而停止
2. **自动恢复**：服务崩溃后需要自动重启，无需人工干预
3. **系统集成**：服务需要在系统启动时自动运行
4. **资源管理**：服务需要被操作系统统一管理和监控

前台进程无法满足这些需求，因为它依赖于终端会话。

### 解决方案

操作系统提供了服务管理器来解决这个问题：
- **macOS**: launchd（Launch Daemon）
- **Linux**: systemd（System Daemon）

OpenClaw 通过集成这些服务管理器，实现 Gateway 的守护进程化。

## macOS 守护进程管理（launchd）

### 服务文件位置

```
~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

### 安装守护进程

**通过 Onboarding Wizard（推荐）：**
```bash
openclaw onboard --install-daemon
```

**直接安装：**
```bash
openclaw gateway install
openclaw gateway install --port 18789 --token <token>
```

### 服务管理命令

**OpenClaw CLI（推荐）：**
```bash
# 启动
openclaw gateway start

# 停止
openclaw gateway stop

# 重启
openclaw gateway restart

# 状态
openclaw gateway status

# 卸载
openclaw gateway uninstall
```

**launchctl 原生命令：**
```bash
# 加载服务
launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# 启动服务
launchctl start ai.openclaw.gateway

# 停止服务
launchctl stop ai.openclaw.gateway

# 卸载服务
launchctl unload ~/Library/LaunchAgents/ai.openclaw.gateway.plist

# 查看服务列表
launchctl list | grep openclaw
```

### 服务配置文件示例

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>ai.openclaw.gateway</string>

    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/openclaw</string>
        <string>gateway</string>
        <string>run</string>
    </array>

    <key>RunAtLoad</key>
    <true/>

    <key>KeepAlive</key>
    <true/>

    <key>StandardOutPath</key>
    <string>/Users/username/.openclaw/logs/gateway-stdout.log</string>

    <key>StandardErrorPath</key>
    <string>/Users/username/.openclaw/logs/gateway-stderr.log</string>
</dict>
</plist>
```

**关键配置项：**
- `Label`: 服务唯一标识符
- `ProgramArguments`: 启动命令和参数
- `RunAtLoad`: 系统启动时自动运行
- `KeepAlive`: 崩溃后自动重启
- `StandardOutPath/StandardErrorPath`: 日志文件路径

## Linux 守护进程管理（systemd）

### 服务文件位置

```
~/.config/systemd/user/openclaw-gateway.service
```

### 安装守护进程

**通过 Onboarding Wizard（推荐）：**
```bash
openclaw onboard --install-daemon
```

**启用 lingering（重要）：**
```bash
# 允许用户服务在登出后继续运行
loginctl enable-linger $USER

# 验证 lingering 状态
loginctl show-user $USER | grep Linger
```

### 服务管理命令

**OpenClaw CLI（推荐）：**
```bash
# 启动
openclaw gateway start

# 停止
openclaw gateway stop

# 重启
openclaw gateway restart

# 状态
openclaw gateway status

# 卸载
openclaw gateway uninstall
```

**systemctl 原生命令：**
```bash
# 启用服务（开机自启）
systemctl --user enable openclaw-gateway

# 启动服务
systemctl --user start openclaw-gateway

# 停止服务
systemctl --user stop openclaw-gateway

# 重启服务
systemctl --user restart openclaw-gateway

# 查看状态
systemctl --user status openclaw-gateway

# 查看日志
journalctl --user -u openclaw-gateway -f

# 禁用服务
systemctl --user disable openclaw-gateway
```

### 服务配置文件示例

```ini
[Unit]
Description=OpenClaw Gateway
After=network.target

[Service]
Type=simple
ExecStart=/usr/local/bin/openclaw gateway run
Restart=always
RestartSec=10
StandardOutput=append:/home/username/.openclaw/logs/gateway-stdout.log
StandardError=append:/home/username/.openclaw/logs/gateway-stderr.log

[Install]
WantedBy=default.target
```

**关键配置项：**
- `Description`: 服务描述
- `After`: 依赖的服务（网络启动后）
- `Type`: 服务类型（simple 表示前台进程）
- `ExecStart`: 启动命令
- `Restart`: 重启策略（always 表示总是重启）
- `RestartSec`: 重启延迟（秒）
- `StandardOutput/StandardError`: 日志输出

## 服务生命周期

### 启动流程

```
1. 系统启动 / 用户登录
   ↓
2. launchd/systemd 启动 openclaw-gateway 服务
   ↓
3. Gateway 读取配置 (~/.openclaw/openclaw.json)
   ↓
4. Gateway 绑定端口（默认 18789）
   ↓
5. Gateway 初始化通道（WhatsApp, Telegram 等）
   ↓
6. Gateway 广播 Bonjour 服务（如果启用）
   ↓
7. Gateway 就绪，开始接受连接
```

### 重启流程

```
1. 接收重启信号（SIGUSR1 或手动重启）
   ↓
2. Gateway 停止接受新连接
   ↓
3. Gateway 优雅关闭活动会话
   ↓
4. Gateway 清理资源（Tailscale, Bonjour）
   ↓
5. Gateway 进程退出
   ↓
6. launchd/systemd 检测到进程退出
   ↓
7. launchd/systemd 自动重启 Gateway
   ↓
8. Gateway 重新初始化并就绪
```

### 停止流程

```
1. 接收停止命令（openclaw gateway stop）
   ↓
2. launchd/systemd 发送 SIGTERM 信号
   ↓
3. Gateway 停止接受新连接
   ↓
4. Gateway 优雅关闭活动会话
   ↓
5. Gateway 清理资源
   ↓
6. Gateway 进程退出
   ↓
7. launchd/systemd 不再重启（因为是手动停止）
```

## 日志管理

### 日志文件位置

**OpenClaw 日志：**
```
~/.openclaw/logs/gateway.log
~/.openclaw/logs/gateway-YYYY-MM-DD.log
```

**系统日志（macOS）：**
```
~/Library/LaunchAgents/ai.openclaw.gateway.plist 中配置的路径
```

**系统日志（Linux）：**
```
journalctl --user -u openclaw-gateway
```

### 查看日志

**OpenClaw CLI：**
```bash
# 查看最近日志
openclaw logs

# 实时跟踪
openclaw logs --follow

# 查看最后 100 行
openclaw logs --tail 100
```

**直接查看文件：**
```bash
# 实时查看
tail -f ~/.openclaw/logs/gateway.log

# 查看最后 100 行
tail -n 100 ~/.openclaw/logs/gateway.log
```

**Linux systemd 日志：**
```bash
# 实时查看
journalctl --user -u openclaw-gateway -f

# 查看最后 100 行
journalctl --user -u openclaw-gateway -n 100

# 查看特定时间范围
journalctl --user -u openclaw-gateway --since "2026-02-22 00:00:00"
```

## 故障排查

### 常见问题

**1. 守护进程未启动**

```bash
# 检查服务状态
openclaw gateway status

# 查看日志
openclaw logs --tail 50

# macOS: 检查 launchd
launchctl list | grep openclaw

# Linux: 检查 systemd
systemctl --user status openclaw-gateway
```

**2. 端口已被占用**

```bash
# 检查端口占用
lsof -i :18789

# 停止守护进程
openclaw gateway stop

# 强制释放端口并启动
openclaw gateway run --force
```

**3. 配置验证错误**

```bash
# 运行诊断
openclaw doctor

# 自动修复
openclaw doctor --fix
```

**4. 守护进程崩溃循环**

```bash
# 查看崩溃日志
openclaw logs --tail 200

# 停止守护进程
openclaw gateway stop

# 前台启动调试
openclaw gateway --verbose
```

**5. Linux lingering 未启用**

```bash
# 启用 lingering
loginctl enable-linger $USER

# 验证
loginctl show-user $USER | grep Linger
# 输出应该是: Linger=yes
```

## 多实例管理

### 使用 Profile 运行多个守护进程

**安装多个实例：**
```bash
# 默认实例
openclaw onboard --install-daemon

# 开发实例
openclaw onboard --install-daemon --profile dev

# 救援实例
openclaw onboard --install-daemon --profile rescue
```

**管理多个实例：**
```bash
# 启动默认实例
openclaw gateway start

# 启动开发实例
openclaw gateway start --profile dev

# 查看所有实例状态
openclaw gateway probe
```

**服务文件命名：**
- macOS: `ai.openclaw.gateway.plist`, `ai.openclaw.gateway.dev.plist`
- Linux: `openclaw-gateway.service`, `openclaw-gateway-dev.service`

## 总结

守护进程管理是 OpenClaw Gateway 生产部署的核心：

- **macOS**: 使用 launchd 管理服务

通过守护进程化，Gateway 可以实现持久运行、自动重启和系统集成，满足生产环境的可靠性要求。
