# 核心概念 6：记忆持久化后端

> InMemory / SQLite / PostgreSQL / Redis — 从开发到生产的存储选型

---

## 一句话定义

**记忆持久化后端是 Agent 记忆数据的实际存储位置，决定了系统重启后记忆是否保留、多用户并发性能、以及运维复杂度。**

---

## 为什么要关心持久化后端？

### 开发时的默认陷阱

```python
from langgraph.checkpoint.memory import InMemorySaver

checkpointer = InMemorySaver()  # ← 数据存在 Python 字典里

# 问题：
# 1. 程序重启 → 所有记忆丢失
# 2. 多进程部署 → 每个进程有独立的记忆（不共享）
# 3. 内存有限 → 大量用户会 OOM
```

### 生产环境的要求

```
✅ 持久化：重启后记忆仍在
✅ 共享：多个 Worker 进程共享同一份记忆
✅ 高可用：数据库故障不丢数据
✅ 高性能：并发读写低延迟
✅ 可扩展：用户增长后不需要重新设计
```

---

## 四种 Checkpointer 后端详解

### 1. InMemorySaver（内存）

```python
from langgraph.checkpoint.memory import InMemorySaver

checkpointer = InMemorySaver()
```

**原理**：数据存在 Python 字典中，进程结束即丢失。

```
┌──────────────────┐
│  Python 进程      │
│  ┌──────────────┐ │
│  │ dict = {     │ │
│  │   "thread-1":│ │
│  │     [msg...] │ │
│  │   "thread-2":│ │
│  │     [msg...] │ │
│  │ }            │ │
│  └──────────────┘ │
└──────────────────┘
进程结束 → 全部丢失
```

| 维度 | 说明 |
|------|------|
| **适用** | 开发、测试、原型验证 |
| **性能** | 极快（纯内存操作） |
| **持久化** | ❌ 不持久化 |
| **并发** | ❌ 单进程 |
| **部署** | 零配置 |
| **容量** | 受内存限制 |

---

### 2. SqliteSaver（SQLite）

```python
from langgraph.checkpoint.sqlite import SqliteSaver

# 文件型数据库
checkpointer = SqliteSaver(db_path="./agent_memory.db")

# 或使用连接字符串
checkpointer = SqliteSaver(conn_string="sqlite:///agent_memory.db")
```

**原理**：数据存在本地 SQLite 文件中，进程重启不丢失。

```
┌──────────────────┐     ┌─────────────────┐
│  Python 进程      │────→│ agent_memory.db │
│                  │     │  (本地文件)       │
│  SqliteSaver     │     │  table: checkpts │
│                  │     │  thread_id | data│
└──────────────────┘     └─────────────────┘
进程重启 → 文件还在 → 记忆保留 ✓
```

```python
# 完整使用示例
from langchain.agents import create_agent
from langgraph.checkpoint.sqlite import SqliteSaver

# 创建持久化 Agent
checkpointer = SqliteSaver(db_path="./my_agent.db")

agent = create_agent(
    model="gpt-4.1",
    tools=[],
    checkpointer=checkpointer,
)

config = {"configurable": {"thread_id": "user-123"}}

# 第一次运行
agent.invoke({"messages": "我叫张三"}, config)

# 重启程序后再次运行
# checkpointer = SqliteSaver(db_path="./my_agent.db")  # 重新连接同一文件
agent.invoke({"messages": "我叫什么？"}, config)
# → "你叫张三"  ← 重启后记忆仍在！
```

| 维度 | 说明 |
|------|------|
| **适用** | 轻量生产、单机部署、桌面应用 |
| **性能** | 快（本地文件IO） |
| **持久化** | ✅ 文件持久化 |
| **并发** | ⚠️ 有限（写锁） |
| **部署** | 零配置（无需安装数据库） |
| **容量** | 受磁盘限制 |
| **备份** | 复制文件即可 |

**适用场景**：
- 个人项目 / 内部工具
- 单机部署（1个进程）
- 用户数 < 100

### 3. PostgresSaver（PostgreSQL）

```python
from langgraph.checkpoint.postgres import PostgresSaver

checkpointer = PostgresSaver(
    conn_string="postgresql://user:password@localhost:5432/agent_db"
)

# 异步版本
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver

checkpointer = AsyncPostgresSaver(
    conn_string="postgresql://user:password@localhost:5432/agent_db"
)
```

**原理**：数据存在 PostgreSQL 数据库中，支持高并发和高可用。

```
┌──────────────┐  ┌──────────────┐
│  Worker 1    │──│              │
│  Agent       │  │              │
└──────────────┘  │  PostgreSQL  │
┌──────────────┐  │  Database    │
│  Worker 2    │──│              │
│  Agent       │  │  ┌────────┐  │
└──────────────┘  │  │checkpts│  │
┌──────────────┐  │  │table   │  │
│  Worker 3    │──│  └────────┘  │
│  Agent       │  │              │
└──────────────┘  └──────────────┘
多进程共享 ✓  高可用 ✓  备份恢复 ✓
```

```python
# 生产级配置示例
import os
from langchain.agents import create_agent
from langgraph.checkpoint.postgres import PostgresSaver

DB_URL = os.environ.get(
    "DATABASE_URL",
    "postgresql://agent:secret@db.example.com:5432/agent_production"
)

checkpointer = PostgresSaver(
    conn_string=DB_URL,
    # 连接池配置（高并发）
    # pool_size=20,
    # max_overflow=10,
)

agent = create_agent(
    model="gpt-4.1",
    tools=tools,
    checkpointer=checkpointer,
)

# 多个 Worker 进程可以共享同一个数据库
# 不同用户的 thread_id 互不干扰
```

| 维度 | 说明 |
|------|------|
| **适用** | 企业生产、多用户、高可用 |
| **性能** | 高（连接池 + 索引） |
| **持久化** | ✅ 完整 ACID |
| **并发** | ✅ 高并发支持 |
| **部署** | 需要 PostgreSQL 服务 |
| **容量** | 近乎无限（磁盘） |
| **备份** | pg_dump / 流复制 |
| **运维** | 需要 DBA 知识 |

[来源: atom/langchain/L4_Agent系统/07_Agent Memory集成/reference/search_agent_memory_02.md]

**适用场景**：
- SaaS 产品 / 多用户系统
- 需要高可用（主从复制）
- 用户数 100+
- 已有 PostgreSQL 基础设施

### 4. RedisSaver（Redis）

```python
from langgraph.checkpoint.redis import RedisSaver

checkpointer = RedisSaver(url="redis://localhost:6379/0")

# 带密码
checkpointer = RedisSaver(
    url="redis://:password@redis.example.com:6379/0"
)
```

**原理**：数据存在 Redis 中，极低延迟，支持 TTL 自动过期。

```
┌──────────────┐  ┌──────────────┐
│  Worker 1    │──│              │
│  Agent       │  │    Redis     │
└──────────────┘  │              │
┌──────────────┐  │  key-value   │
│  Worker 2    │──│  in-memory   │
│  Agent       │  │  + 持久化     │
└──────────────┘  │              │
                  │  TTL 支持     │
                  │  亚毫秒延迟    │
                  └──────────────┘
```

```python
# 高性能场景配置
from langchain.agents import create_agent
from langgraph.checkpoint.redis import RedisSaver

checkpointer = RedisSaver(
    url="redis://redis.example.com:6379/0",
    # TTL：对话记忆 24 小时后自动过期
    # ttl=86400,
)

agent = create_agent(
    model="gpt-4.1-mini",  # 低延迟模型
    tools=tools,
    checkpointer=checkpointer,
)
```

| 维度 | 说明 |
|------|------|
| **适用** | 高性能、低延迟、高并发 |
| **性能** | 极快（内存 + 亚毫秒） |
| **持久化** | ⚠️ 需配置 RDB/AOF |
| **并发** | ✅ 极高并发 |
| **部署** | 需要 Redis 服务 |
| **容量** | 受内存限制 |
| **TTL** | ✅ 原生支持自动过期 |

[来源: atom/langchain/L4_Agent系统/07_Agent Memory集成/reference/search_agent_memory_02.md]

**适用场景**：
- 实时对话（聊天机器人、客服）
- 高 QPS（> 1000 req/s）
- 短期对话（不需要永久保存）
- 已有 Redis 基础设施

## Store 的持久化后端

Store（长期记忆）也有类似的后端选择：

```python
# 内存版（开发/测试）
from langgraph.store.memory import InMemoryStore
store = InMemoryStore()

# PostgreSQL 版（生产推荐）
from langgraph.store.postgres import PostgresStore
store = PostgresStore(conn_string=DB_URL)

# 带语义搜索
from langchain.embeddings import init_embeddings
embeddings = init_embeddings("openai:text-embedding-3-small")

store = InMemoryStore(
    index={"embed": embeddings, "dims": 1536}
)
```

## 四种后端横向对比

| 维度 | InMemory | SQLite | PostgreSQL | Redis |
|------|----------|--------|------------|-------|
| **持久化** | ❌ | ✅ 文件 | ✅ ACID | ⚠️ 需配置 |
| **并发** | ❌ 单进程 | ⚠️ 有限 | ✅ 高 | ✅ 极高 |
| **延迟** | ~0ms | ~1ms | ~5ms | ~0.5ms |
| **容量** | 内存 | 磁盘 | 磁盘 | 内存 |
| **部署难度** | ⭐ | ⭐ | ⭐⭐⭐ | ⭐⭐ |
| **运维成本** | 零 | 低 | 中 | 中 |
| **TTL** | ❌ | ❌ | 需实现 | ✅ 原生 |
| **备份** | ❌ | 复制文件 | pg_dump | RDB/AOF |
| **推荐度** | 开发 | 轻量 | **生产首选** | 高性能 |

## 选型决策树

```
你的 Agent 项目处于什么阶段？
│
├── 开发/原型 ──→ InMemorySaver ✓
│
├── 个人项目/内部工具 ──→ SqliteSaver ✓
│
├── 正式产品
│   │
│   ├── 需要高性能/低延迟？
│   │   ├── 是 ──→ RedisSaver ✓
│   │   └── 否 ──→ PostgresSaver ✓
│   │
│   ├── 需要高可用/数据安全？
│   │   └── ──→ PostgresSaver ✓（主从复制）
│   │
│   └── 已有什么基础设施？
│       ├── 已有 PostgreSQL ──→ PostgresSaver ✓
│       ├── 已有 Redis ──→ RedisSaver ✓
│       └── 都没有 ──→ PostgresSaver ✓（推荐）
│
└── 特殊场景
    ├── Serverless (Lambda) ──→ DynamoDB 或 PostgresSaver ✓
    ├── 短期对话（24h过期）──→ RedisSaver + TTL ✓
    └── 边缘设备 ──→ SqliteSaver ✓
```

## 生产部署最佳实践

### 1. 环境分离

```python
import os

def get_checkpointer():
    """根据环境选择 checkpointer"""
    env = os.environ.get("ENV", "development")

    if env == "development":
        from langgraph.checkpoint.memory import InMemorySaver
        return InMemorySaver()

    elif env == "staging":
        from langgraph.checkpoint.sqlite import SqliteSaver
        return SqliteSaver(db_path="./staging_memory.db")

    elif env == "production":
        from langgraph.checkpoint.postgres import PostgresSaver
        db_url = os.environ["DATABASE_URL"]
        return PostgresSaver(conn_string=db_url)

    else:
        raise ValueError(f"Unknown environment: {env}")
```

### 2. 连接池管理（PostgreSQL）

```python
# 生产环境的 PostgreSQL 配置
from langgraph.checkpoint.postgres import PostgresSaver

checkpointer = PostgresSaver(
    conn_string=os.environ["DATABASE_URL"],
    # 连接池配置建议：
    # pool_size: Worker 数 × 2
    # max_overflow: pool_size × 0.5
    # pool_timeout: 30 秒
    # pool_recycle: 3600 秒（防止连接过期）
)
```

### 3. Redis 持久化配置

```bash
# redis.conf 关键配置

# RDB 快照（定时持久化）
save 900 1       # 900秒内有1次写入则快照
save 300 10      # 300秒内有10次写入则快照

# AOF 日志（实时持久化）
appendonly yes
appendfsync everysec   # 每秒同步一次

# 内存限制
maxmemory 2gb
maxmemory-policy allkeys-lru   # 内存满时删除最近最少使用的键
```

### 4. 数据迁移

```python
# 从 SQLite 迁移到 PostgreSQL
async def migrate_checkpoints(
    source: SqliteSaver,
    target: PostgresSaver
):
    """迁移所有检查点数据"""
    # 获取所有 thread
    threads = await source.list_threads()

    for thread_id in threads:
        # 获取该 thread 的所有检查点
        checkpoints = await source.list(
            config={"configurable": {"thread_id": thread_id}}
        )

        for checkpoint in checkpoints:
            # 写入新的后端
            await target.put(
                config={"configurable": {"thread_id": thread_id}},
                checkpoint=checkpoint.checkpoint,
                metadata=checkpoint.metadata,
            )

    print(f"迁移完成：{len(threads)} 个线程")
```

### 5. 监控和清理

```python
# 定期清理过期的对话记忆
async def cleanup_old_threads(
    checkpointer: PostgresSaver,
    max_age_days: int = 30
):
    """清理超过 N 天的对话"""
    # 使用 SQL 直接清理（以 PostgreSQL 为例）
    async with checkpointer.conn.cursor() as cur:
        await cur.execute("""
            DELETE FROM checkpoints
            WHERE created_at < NOW() - INTERVAL '%s days'
        """, (max_age_days,))
        deleted = cur.rowcount
        await checkpointer.conn.commit()

    print(f"清理了 {deleted} 条过期记录")
```

## 与 RAG 开发的关系

| RAG 场景 | 推荐后端 | 理由 |
|----------|----------|------|
| **个人知识库助手** | SQLite | 单用户，简单部署 |
| **团队文档问答** | PostgreSQL | 多用户，需要持久化 |
| **实时客服 RAG** | Redis | 低延迟，短期对话 |
| **企业知识管理** | PostgreSQL + Redis | PG 存长期，Redis 做缓存 |

---

## 本节小结

> **持久化后端决定了 Agent 记忆的「物理基础」。**
> - **InMemorySaver**：开发测试用，零配置
> - **SqliteSaver**：轻量生产，单文件零依赖
> - **PostgresSaver**：**生产首选**，高可用高并发
> - **RedisSaver**：极致性能，适合实时场景
>
> **选型原则：开发用 InMemory → 轻量用 SQLite → 生产用 PostgreSQL → 性能用 Redis**
