## 场景概述

流式解析是 LangChain OutputParser 的核心特性之一，允许在 LLM 生成内容的同时逐步解析输出，而不是等待完整响应。这对于提升用户体验和降低延迟至关重要。

**核心挑战**：
- 如何处理不完整的 JSON/XML 块？
- 如何在流式模式下保持解析的正确性？
- 如何平衡延迟和内存占用？

**两种流式解析模式**：
1. **逐块解析**（BaseTransformOutputParser）：每个块独立解析，适合简单转换
2. **累积解析**（BaseCumulativeTransformOutputParser）：累积缓冲区解析，适合复杂格式

## 示例1：基础流式 JSON 解析

**场景**：使用 `JsonOutputParser` 实现流式 JSON 解析，实时输出部分结果。

**关键技术**：
- `JsonOutputParser` 继承自 `BaseCumulativeTransformOutputParser`
- 支持部分 JSON 解析（`partial=True`）
- 自动处理 Markdown 代码块包裹

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

# 加载环境变量
load_dotenv()

# 初始化模型和解析器
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
parser = JsonOutputParser()

# 创建 prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs JSON."),
    ("user", "Generate a person profile with name, age, occupation, and hobbies (list).")
])

# 构建 LCEL 链
chain = prompt | model | parser

# 流式输出
print("=== 流式 JSON 解析 ===\n")
for chunk in chain.stream({}):
    print(f"Chunk: {chunk}")
    print(f"Type: {type(chunk)}\n")

"""
预期输出（逐步累积）：
Chunk: {}
Type: <class 'dict'>

Chunk: {'name': 'Alice'}
Type: <class 'dict'>

Chunk: {'name': 'Alice', 'age': 30}
Type: <class 'dict'>

Chunk: {'name': 'Alice', 'age': 30, 'occupation': 'Software Engineer'}
Type: <class 'dict'>

Chunk: {'name': 'Alice', 'age': 30, 'occupation': 'Software Engineer', 'hobbies': ['reading']}
Type: <class 'dict'>

Chunk: {'name': 'Alice', 'age': 30, 'occupation': 'Software Engineer', 'hobbies': ['reading', 'hiking', 'coding']}
Type: <class 'dict'>
"""
```

**关键点**：
- 每个 chunk 都是一个完整的（但可能不完整的）字典
- 随着流式输出，字典逐渐完整
- 最后一个 chunk 是完整的 JSON 对象

## 示例2：JSON Patch Diff 模式

**场景**：使用 JSON Patch 模式减少数据传输，只输出变化的部分。

**关键技术**：
- `JsonOutputParser` 的 `diff=True` 参数
- 使用 `jsonpatch` 库生成 diff
- 适用于大型 JSON 对象的流式传输

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
import json

load_dotenv()

# 初始化
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
parser = JsonOutputParser()

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs JSON."),
    ("user", "Generate a detailed product catalog with 3 products (id, name, price, description, tags).")
])

chain = prompt | model | parser

# 流式输出 - Diff 模式
print("=== JSON Patch Diff 模式 ===\n")

# 手动实现 diff 模式（模拟 JsonOutputParser 内部逻辑）
prev_chunk = None
for i, chunk in enumerate(chain.stream({})):
    if prev_chunk is None:
        # 第一个 chunk，输出完整对象
        print(f"Chunk {i}: {json.dumps(chunk, indent=2)}")
        print(f"Size: {len(json.dumps(chunk))} bytes\n")
    else:
        # 后续 chunk，只输出变化
        # 注意：实际的 JsonOutputParser 使用 jsonpatch.make_patch()
        added_keys = set(chunk.keys()) - set(prev_chunk.keys())
        changed_keys = {k for k in chunk.keys() if k in prev_chunk and chunk[k] != prev_chunk[k]}

        diff = {}
        for key in added_keys | changed_keys:
            diff[key] = chunk[key]

        print(f"Chunk {i} (diff): {json.dumps(diff, indent=2)}")
        print(f"Size: {len(json.dumps(diff))} bytes (vs full: {len(json.dumps(chunk))} bytes)\n")

    prev_chunk = chunk

"""
预期输出：
Chunk 0: {}
Size: 2 bytes

Chunk 1 (diff): {"products": []}
Size: 15 bytes (vs full: 15 bytes)

Chunk 2 (diff): {"products": [{"id": 1}]}
Size: 25 bytes (vs full: 25 bytes)

Chunk 3 (diff): {"products": [{"id": 1, "name": "Laptop"}]}
Size: 45 bytes (vs full: 45 bytes)

... (逐步添加字段)
"""
```

**性能优势**：
- 减少网络传输量（特别是大型 JSON）
- 降低客户端解析开销
- 适合实时更新场景

## 示例3：部分解析策略（Partial Parsing）

**场景**：在流式模式下处理不完整的 JSON，使用 `partial=True` 参数。

**关键技术**：
- `parse_result()` 方法的 `partial` 参数
- `parse_partial_json()` 工具函数
- 优雅处理不完整的 JSON 块

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers.json import parse_partial_json

load_dotenv()

# 模拟不完整的 JSON 流
incomplete_json_chunks = [
    '{"name": "Alice"',
    ', "age": 30',
    ', "skills": ["Python"',
    ', "JavaScript"',
    ', "TypeScript"]',
    ', "address": {"city": "NYC"',
    ', "country": "USA"}',
    '}'
]

print("=== 部分 JSON 解析 ===\n")

# 累积缓冲区
buffer = ""
for i, chunk in enumerate(incomplete_json_chunks):
    buffer += chunk
    print(f"Chunk {i}: {repr(chunk)}")
    print(f"Buffer: {repr(buffer)}")

    # 尝试部分解析
    try:
        parsed = parse_partial_json(buffer)
        print(f"Parsed (partial): {parsed}")
    except Exception as e:
        print(f"Parse failed: {e}")

    print()

"""
预期输出：
Chunk 0: '{"name": "Alice"'
Buffer: '{"name": "Alice"'
Parsed (partial): {'name': 'Alice'}

Chunk 1: ', "age": 30'
Buffer: '{"name": "Alice", "age": 30'
Parsed (partial): {'name': 'Alice', 'age': 30}

Chunk 2: ', "skills": ["Python"'
Buffer: '{"name": "Alice", "age": 30, "skills": ["Python"'
Parsed (partial): {'name': 'Alice', 'age': 30, 'skills': ['Python']}

Chunk 3: ', "JavaScript"'
Buffer: '{"name": "Alice", "age": 30, "skills": ["Python", "JavaScript"'
Parsed (partial): {'name': 'Alice', 'age': 30, 'skills': ['Python', 'JavaScript']}

... (逐步完整)
"""
```

**关键点**：
- `parse_partial_json()` 可以解析不完整的 JSON
- 自动补全缺失的括号和引号
- 返回尽可能完整的对象

## 示例4：自定义累积解析器

**场景**：实现一个自定义的累积解析器，用于流式解析 CSV 数据。

**关键技术**：
- 继承 `BaseCumulativeTransformOutputParser`
- 实现 `parse_result()` 方法
- 支持部分解析

```python
from typing import List, Dict, Any
from langchain_core.output_parsers import BaseCumulativeTransformOutputParser
from langchain_core.outputs import Generation

class StreamingCSVParser(BaseCumulativeTransformOutputParser[List[Dict[str, str]]]):
    """流式 CSV 解析器，逐行输出解析结果"""

    def parse_result(self, result: List[Generation], *, partial: bool = False) -> List[Dict[str, str]]:
        """解析 CSV 文本为字典列表"""
        text = result[0].text
        lines = text.strip().split('\n')

        if len(lines) < 1:
            return []

        # 第一行是表头
        headers = [h.strip() for h in lines[0].split(',')]

        # 解析数据行
        rows = []
        for line in lines[1:]:
            # 跳过空行
            if not line.strip():
                continue

            # 分割字段
            fields = [f.strip() for f in line.split(',')]

            # 如果是部分解析且字段不完整，跳过
            if partial and len(fields) < len(headers):
                continue

            # 构建字典
            row = {}
            for i, header in enumerate(headers):
                if i < len(fields):
                    row[header] = fields[i]
                elif not partial:
                    row[header] = ""  # 完整解析时填充空值

            rows.append(row)

        return rows

# 使用示例
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

load_dotenv()

model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
parser = StreamingCSVParser()

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs CSV data."),
    ("user", "Generate a CSV table with 5 employees (name, age, department, salary).")
])

chain = prompt | model | parser

print("=== 流式 CSV 解析 ===\n")
for i, chunk in enumerate(chain.stream({})):
    print(f"Chunk {i}: {len(chunk)} rows")
    for row in chunk:
        print(f"  {row}")
    print()

"""
预期输出：
Chunk 0: 0 rows

Chunk 1: 1 rows
  {'name': 'Alice', 'age': '30', 'department': 'Engineering', 'salary': '120000'}

Chunk 2: 2 rows
  {'name': 'Alice', 'age': '30', 'department': 'Engineering', 'salary': '120000'}
  {'name': 'Bob', 'age': '35', 'department': 'Sales', 'salary': '90000'}

... (逐步添加行)
"""
```

**设计亮点**：
- 支持部分解析（跳过不完整的行）
- 自动处理表头
- 可扩展到其他表格格式

## 示例5：生产级流式解析（错误处理 + 性能优化）

**场景**：构建一个生产级的流式 JSON 解析系统，包含错误处理、性能监控和内存管理。

**关键技术**：
- 异常捕获和恢复
- 性能指标收集
- 内存限制和缓冲区管理

```python
from dotenv import load_dotenv
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.exceptions import OutputParserException
import time
import sys

load_dotenv()

class ProductionStreamingParser:
    """生产级流式解析器包装器"""

    def __init__(self, parser: JsonOutputParser, max_buffer_size: int = 1024 * 1024):
        self.parser = parser
        self.max_buffer_size = max_buffer_size
        self.metrics = {
            "chunks_processed": 0,
            "parse_errors": 0,
            "total_bytes": 0,
            "start_time": None,
            "end_time": None
        }

    def stream_parse(self, chain, input_data: dict):
        """流式解析并收集指标"""
        self.metrics["start_time"] = time.time()
        buffer_size = 0

        try:
            for chunk in chain.stream(input_data):
                self.metrics["chunks_processed"] += 1

                # 估算 chunk 大小
                chunk_size = sys.getsizeof(chunk)
                buffer_size += chunk_size
                self.metrics["total_bytes"] += chunk_size

                # 检查缓冲区大小
                if buffer_size > self.max_buffer_size:
                    raise MemoryError(f"Buffer size exceeded: {buffer_size} > {self.max_buffer_size}")

                # 输出 chunk
                yield chunk

        except OutputParserException as e:
            self.metrics["parse_errors"] += 1
            print(f"Parse error: {e}")
            # 可以选择继续或中断
            raise

        except Exception as e:
            print(f"Unexpected error: {e}")
            raise

        finally:
            self.metrics["end_time"] = time.time()

    def get_metrics(self) -> dict:
        """获取性能指标"""
        duration = self.metrics["end_time"] - self.metrics["start_time"] if self.metrics["end_time"] else 0
        return {
            **self.metrics,
            "duration_seconds": duration,
            "chunks_per_second": self.metrics["chunks_processed"] / duration if duration > 0 else 0,
            "bytes_per_second": self.metrics["total_bytes"] / duration if duration > 0 else 0
        }

# 使用示例
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
parser = JsonOutputParser()

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant that outputs JSON."),
    ("user", "Generate a large dataset with 10 users (id, name, email, address, phone, company).")
])

chain = prompt | model | parser

# 创建生产级解析器
prod_parser = ProductionStreamingParser(parser, max_buffer_size=10 * 1024)  # 10KB limit

print("=== 生产级流式解析 ===\n")

try:
    for i, chunk in enumerate(prod_parser.stream_parse(chain, {})):
        print(f"Chunk {i}: {len(str(chunk))} chars")
        # 在生产环境中，这里可以发送到前端或存储到数据库

except MemoryError as e:
    print(f"Memory limit exceeded: {e}")
except OutputParserException as e:
    print(f"Parse failed: {e}")

# 输出性能指标
print("\n=== Performance Metrics ===")
metrics = prod_parser.get_metrics()
for key, value in metrics.items():
    if isinstance(value, float):
        print(f"{key}: {value:.2f}")
    else:
        print(f"{key}: {value}")

"""
预期输出：
=== 生产级流式解析 ===

Chunk 0: 2 chars
Chunk 1: 15 chars
Chunk 2: 45 chars
... (逐步增长)

=== Performance Metrics ===
chunks_processed: 25
parse_errors: 0
total_bytes: 15234
duration_seconds: 2.34
chunks_per_second: 10.68
bytes_per_second: 6509.40
"""
```

**生产级特性**：
- **错误处理**：捕获并记录解析错误
- **内存管理**：限制缓冲区大小，防止内存溢出
- **性能监控**：收集吞吐量、延迟等指标
- **可观测性**：输出详细的指标用于监控

## 常见错误与解决方案

### 错误1：流式解析时 JSON 不完整

**问题**：
```python
# 错误示例
for chunk in chain.stream({}):
    json.loads(chunk)  # ❌ 会失败，因为 chunk 可能不完整
```

**解决方案**：
```python
# 正确做法：使用 JsonOutputParser
parser = JsonOutputParser()
chain = prompt | model | parser

for chunk in chain.stream({}):
    # chunk 已经是解析后的字典，不需要再 json.loads()
    print(chunk)  # ✅ 正确
```

### 错误2：混淆逐块解析和累积解析

**问题**：
```python
# 错误：期望每个 chunk 都是独立的
for chunk in chain.stream({}):
    process_complete_object(chunk)  # ❌ chunk 可能不完整
```

**解决方案**：
```python
# 正确：理解累积解析的特性
for chunk in chain.stream({}):
    # chunk 是累积的结果，逐步完整
    if is_complete(chunk):
        process_complete_object(chunk)
    else:
        show_partial_result(chunk)  # 显示部分结果
```

### 错误3：内存泄漏

**问题**：
```python
# 错误：无限累积缓冲区
buffer = []
for chunk in chain.stream({}):
    buffer.append(chunk)  # ❌ 内存持续增长
```

**解决方案**：
```python
# 正确：及时清理或限制大小
max_chunks = 100
buffer = []
for chunk in chain.stream({}):
    buffer.append(chunk)
    if len(buffer) > max_chunks:
        buffer.pop(0)  # 移除最旧的 chunk
```

## 性能对比

| 解析模式 | 延迟 | 内存占用 | 适用场景 |
|---------|------|---------|---------|
| 非流式（等待完整响应） | 高（等待全部生成） | 低 | 批处理、离线任务 |
| 逐块解析（BaseTransformOutputParser） | 低（立即输出） | 低 | 简单转换（字符串、数字） |
| 累积解析（BaseCumulativeTransformOutputParser） | 中（需要累积） | 中 | 复杂格式（JSON、XML） |
| JSON Patch Diff | 低（只传输变化） | 低 | 大型 JSON、实时更新 |

1. **选择合适的解析器**：
   - 简单转换 → `StrOutputParser`
   - JSON 解析 → `JsonOutputParser`

2. **启用部分解析**：
   - 在流式场景下使用 `partial=True`
   - 优雅处理不完整的数据

3. **监控性能**：
   - 收集延迟、吞吐量指标
   - 设置内存限制

4. **错误处理**：
   - 捕获 `OutputParserException`
   - 提供降级方案

5. **用户体验**：
   - 实时显示部分结果
   - 使用 loading 状态
   - 提供取消机制
