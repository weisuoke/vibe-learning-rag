# 核心概念 2：共享状态键集成

> 共享状态键集成是父子图通过同名 state key 自动传递状态的机制

---

## 概述

**共享状态键集成是什么？**

共享状态键集成（Shared State Key Integration）是 LangGraph 中最简单的子图集成方式。当子图的 State 中有与父图同名的 key 时，LangGraph 会自动将父图中该 key 的值传入子图，子图更新后也会自动回传给父图。子图中独有的 key（父图中不存在的）则作为私有状态，不会影响父图。

**为什么需要理解共享状态键集成？**

- 这是集成子图最直接、代码量最少的方式
- 不需要编写任何转换函数，LangGraph 自动处理状态映射
- 理解其工作原理可以避免状态传递中的常见 Bug
- 是构建模块化工作流的基础模式

---

## 核心概念：同名 Key 自动传递

### 一句话定义

**父子图中同名的 state key 会自动双向传递，子图的私有 key 留在子图内部，这就是共享状态键集成。**

### 工作原理图示

```
┌──────────────────────────────────────────────────────────┐
│                      父图执行流程                          │
│                                                          │
│  ParentState                                             │
│  ┌─────────────────┐                                     │
│  │ foo: "hello"    │──────────┐                          │
│  │ result: ""      │          │                          │
│  └─────────────────┘          │ 自动传入同名 key: foo     │
│                               ▼                          │
│         ┌────────────────────────────────────┐           │
│         │          子图 (Subgraph)             │           │
│         │                                    │           │
│         │  SubgraphState                     │           │
│         │  ┌──────────────────────────┐      │           │
│         │  │ foo: "hello"  ← 自动传入  │      │           │
│         │  │ bar: ""       ← 私有字段  │      │           │
│         │  └──────────────────────────┘      │           │
│         │           │                        │           │
│         │           ▼ 子图节点处理             │           │
│         │                                    │           │
│         │  SubgraphState（更新后）              │           │
│         │  ┌──────────────────────────┐      │           │
│         │  │ foo: "hello world" ← 更新 │      │           │
│         │  │ bar: "internal"   ← 私有  │      │           │
│         │  └──────────────────────────┘      │           │
│         └────────────────────────────────────┘           │
│                               │                          │
│                               │ foo 自动回传，bar 不回传   │
│                               ▼                          │
│  ParentState（更新后）                                     │
│  ┌──────────────────────────┐                            │
│  │ foo: "hello world" ← 更新 │                            │
│  │ result: ""         ← 不变 │                            │
│  └──────────────────────────┘                            │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 详细解释

#### 规则一：同名 Key 自动传入

当父图调用子图时，LangGraph 检查子图的 state_schema（或 input_schema），找到与父图状态中同名的 key，自动将父图中这些 key 的值传入子图。

```python
# 父图状态：{foo: "hello", result: ""}
# 子图状态定义：{foo: str, bar: str}
# → LangGraph 自动将父图的 foo="hello" 传给子图
# → 子图初始状态：{foo: "hello", bar: ""}（bar 使用默认值）
```

#### 规则二：同名 Key 自动回传

子图执行完毕后，LangGraph 检查子图的输出，将与父图同名的 key 的更新值回传给父图。

```python
# 子图执行后状态：{foo: "hello world", bar: "internal data"}
# 父图接收到的更新：{foo: "hello world"}
# → bar 不在父图状态中，被忽略
```

#### 规则三：私有 Key 完全隔离

子图中存在但父图中不存在的 key 是「私有 key」。私有 key：
- 不会被父图的值初始化（使用 state_schema 的默认值）
- 不会在子图执行后回传给父图
- 子图的多次调用之间不保留（除非使用 Checkpointer）

```python
# 子图有 bar 字段，但父图没有
# → bar 是子图的私有状态
# → 父图完全不知道 bar 的存在
```

### 类比理解

**前端类比**：
- 共享 key = React 父子组件通过同名 props 传递数据
- 私有 key = 子组件的内部 useState，父组件不可见
- 自动回传 = 子组件通过 onChange 回调自动更新父组件的同名状态

**日常生活类比**：
- 共享 key = 总公司和分公司共享的「客户名称」字段
- 私有 key = 分公司内部的「处理进度」「临时备注」等内部字段
- 自动传递 = 总公司把客户名称交给分公司处理，分公司更新后自动回传结果

## 完整代码示例

### 示例一：基础共享状态键集成

```python
"""
共享状态键集成 - 基础示例
演示：父子图通过同名 key 自动传递状态
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END


# ===== 1. 定义父图状态 =====
class ParentState(TypedDict):
    foo: str       # 会与子图共享


# ===== 2. 定义子图状态 =====
class SubgraphState(TypedDict):
    foo: str       # 与父图共享的 key（名称相同）
    bar: str       # 子图私有的 key（父图中不存在）


# ===== 3. 定义子图节点 =====
def subgraph_node_1(state: SubgraphState) -> dict:
    """子图节点1：使用共享 key 和私有 key"""
    print(f"  [子图节点1] 收到 foo = '{state['foo']}'")
    print(f"  [子图节点1] bar 初始值 = '{state.get('bar', '')}'")
    return {
        "foo": state["foo"] + " -> 子图处理中",
        "bar": "子图内部数据"  # 私有 key 只在子图内部使用
    }


def subgraph_node_2(state: SubgraphState) -> dict:
    """子图节点2：继续处理"""
    print(f"  [子图节点2] foo = '{state['foo']}'")
    print(f"  [子图节点2] bar = '{state['bar']}'")  # 可以读到节点1设置的值
    return {
        "foo": state["foo"] + " -> 子图完成",
        "bar": state["bar"] + " (已处理)"
    }


# ===== 4. 构建并编译子图 =====
subgraph_builder = StateGraph(SubgraphState)
subgraph_builder.add_node("sub_node_1", subgraph_node_1)
subgraph_builder.add_node("sub_node_2", subgraph_node_2)
subgraph_builder.add_edge(START, "sub_node_1")
subgraph_builder.add_edge("sub_node_1", "sub_node_2")
subgraph_builder.add_edge("sub_node_2", END)

compiled_subgraph = subgraph_builder.compile()


# ===== 5. 定义父图节点 =====
def parent_node_before(state: ParentState) -> dict:
    """父图前置节点"""
    print(f"[父图前置] foo = '{state['foo']}'")
    return {"foo": "hello from parent"}


def parent_node_after(state: ParentState) -> dict:
    """父图后置节点"""
    print(f"[父图后置] foo = '{state['foo']}'")
    # 注意：这里看不到子图的 bar 字段
    return {"foo": state["foo"] + " -> 父图收尾"}


# ===== 6. 构建父图，将子图作为节点添加 =====
parent_builder = StateGraph(ParentState)
parent_builder.add_node("before", parent_node_before)
parent_builder.add_node("subgraph", compiled_subgraph)  # 直接添加编译后的子图
parent_builder.add_node("after", parent_node_after)

parent_builder.add_edge(START, "before")
parent_builder.add_edge("before", "subgraph")
parent_builder.add_edge("subgraph", "after")
parent_builder.add_edge("after", END)

app = parent_builder.compile()


# ===== 7. 执行并观察结果 =====
print("=" * 60)
print("开始执行父图")
print("=" * 60)

result = app.invoke({"foo": "初始值"})

print("=" * 60)
print(f"最终结果: {result}")
print("=" * 60)
```

**预期输出：**

```
============================================================
开始执行父图
============================================================
[父图前置] foo = '初始值'
  [子图节点1] 收到 foo = 'hello from parent'
  [子图节点1] bar 初始值 = ''
  [子图节点2] foo = 'hello from parent -> 子图处理中'
  [子图节点2] bar = '子图内部数据'
[父图后置] foo = 'hello from parent -> 子图处理中 -> 子图完成'
============================================================
最终结果: {'foo': 'hello from parent -> 子图处理中 -> 子图完成 -> 父图收尾'}
============================================================
```

**关键观察：**
1. 父图的 `foo` 值自动传入子图
2. 子图更新 `foo` 后，更新值自动回传给父图
3. 子图的 `bar` 字段完全不出现在父图的结果中

### 示例二：多个共享 Key

```python
"""
多个共享 Key 的场景
演示：父子图可以共享多个 key
"""

from typing import TypedDict, Annotated
import operator
from langgraph.graph import StateGraph, START, END


# 父图状态
class OrderState(TypedDict):
    order_id: str                                    # 共享
    items: Annotated[list[str], operator.add]         # 共享（使用 reducer 追加）
    total_price: float                                # 共享
    status: str                                       # 共享


# 子图状态：价格计算子图
class PricingState(TypedDict):
    items: Annotated[list[str], operator.add]          # 共享
    total_price: float                                 # 共享
    discount_rate: float                               # 私有：折扣率
    tax_amount: float                                  # 私有：税额


def calculate_base_price(state: PricingState) -> dict:
    """计算基础价格"""
    # 每个商品假设 100 元
    base = len(state["items"]) * 100.0
    return {"total_price": base, "discount_rate": 0.1}


def apply_discount(state: PricingState) -> dict:
    """应用折扣"""
    discounted = state["total_price"] * (1 - state["discount_rate"])
    tax = discounted * 0.13
    return {"total_price": discounted + tax, "tax_amount": tax}


# 构建价格计算子图
pricing_builder = StateGraph(PricingState)
pricing_builder.add_node("base_price", calculate_base_price)
pricing_builder.add_node("discount", apply_discount)
pricing_builder.add_edge(START, "base_price")
pricing_builder.add_edge("base_price", "discount")
pricing_builder.add_edge("discount", END)

compiled_pricing = pricing_builder.compile()


# 父图节点
def create_order(state: OrderState) -> dict:
    return {
        "order_id": "ORD-001",
        "items": ["笔记本", "鼠标", "键盘"],
        "status": "created"
    }


def finalize_order(state: OrderState) -> dict:
    return {"status": f"confirmed (总价: {state['total_price']:.2f})"}


# 构建父图
order_builder = StateGraph(OrderState)
order_builder.add_node("create", create_order)
order_builder.add_node("pricing", compiled_pricing)  # 价格计算子图
order_builder.add_node("finalize", finalize_order)

order_builder.add_edge(START, "create")
order_builder.add_edge("create", "pricing")
order_builder.add_edge("pricing", "finalize")
order_builder.add_edge("finalize", END)

app = order_builder.compile()
result = app.invoke({
    "order_id": "",
    "items": [],
    "total_price": 0.0,
    "status": ""
})
print(result)
# items 和 total_price 共享传递
# discount_rate 和 tax_amount 是子图私有，不出现在结果中
```

**预期输出：**

```
{
    'order_id': 'ORD-001',
    'items': ['笔记本', '鼠标', '键盘'],
    'total_price': 305.1,
    'status': 'confirmed (总价: 305.10)'
}
```

### 示例三：验证私有 Key 的隔离性

```python
"""
验证私有 Key 的完全隔离
演示：子图私有 key 不会泄露到父图
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END


class ParentState(TypedDict):
    shared_data: str


class ChildState(TypedDict):
    shared_data: str       # 共享
    secret_key: str        # 私有：敏感信息
    temp_buffer: str       # 私有：临时缓存


def child_process(state: ChildState) -> dict:
    return {
        "shared_data": f"处理完成: {state['shared_data']}",
        "secret_key": "sk-1234567890",  # 敏感信息
        "temp_buffer": "临时数据不应泄露"
    }


child_builder = StateGraph(ChildState)
child_builder.add_node("process", child_process)
child_builder.add_edge(START, "process")
child_builder.add_edge("process", END)
compiled_child = child_builder.compile()

parent_builder = StateGraph(ParentState)
parent_builder.add_node("child", compiled_child)
parent_builder.add_edge(START, "child")
parent_builder.add_edge("child", END)

app = parent_builder.compile()
result = app.invoke({"shared_data": "原始数据"})

print(f"父图结果: {result}")
print(f"包含 secret_key? {'secret_key' in result}")
print(f"包含 temp_buffer? {'temp_buffer' in result}")
```

**预期输出：**

```
父图结果: {'shared_data': '处理完成: 原始数据'}
包含 secret_key? False
包含 temp_buffer? False
```

**结论**：子图的私有 key（`secret_key`、`temp_buffer`）不会出现在父图的结果中，实现了状态隔离。

## 适用场景

### 场景一：父子图有自然重叠的状态

当父图和子图本来就需要操作相同概念的数据时，共享状态键是最自然的选择。

```
例如：聊天系统
  父图状态：messages, user_id, current_agent
  子图状态：messages, user_id, agent_memory (私有)
  → messages 和 user_id 自动共享
```

### 场景二：简单的数据处理管道

当子图只是对父图中某些字段做处理并更新时，无需手动转换。

```
例如：文本处理
  父图状态：text, language, entities
  子图状态：text, language, ner_cache (私有)
  → text 和 language 自动共享
```

### 场景三：多代理协作

多个代理子图通过共享的 `messages` 列表进行协作。

```
例如：多代理对话
  父图状态：messages (Annotated[list, add_messages])
  代理A子图状态：messages, agent_a_memory (私有)
  代理B子图状态：messages, agent_b_memory (私有)
  → messages 在所有代理间自动共享
```

## 注意事项

### 1. Key 名称必须完全相同

```python
# 错误：名称不同，不会自动传递
class ParentState(TypedDict):
    user_query: str     # 注意名称

class ChildState(TypedDict):
    query: str          # 名称不同！不会自动传递

# 正确：使用完全相同的名称
class ParentState(TypedDict):
    query: str

class ChildState(TypedDict):
    query: str          # 同名，自动传递
```

### 2. 类型必须兼容

```python
# 错误：类型不兼容可能导致运行时错误
class ParentState(TypedDict):
    count: int

class ChildState(TypedDict):
    count: str          # 父图传入 int，子图期望 str → 可能出错

# 正确：使用兼容的类型
class ParentState(TypedDict):
    count: int

class ChildState(TypedDict):
    count: int          # 类型一致
```

### 3. Reducer 的继承与冲突

当父子图的同名 key 都使用了 Reducer（如 `Annotated[list, operator.add]`）时，需要注意：

```python
from typing import Annotated
import operator

# 父图：messages 使用 add reducer
class ParentState(TypedDict):
    messages: Annotated[list[str], operator.add]

# 子图：messages 也使用 add reducer
class ChildState(TypedDict):
    messages: Annotated[list[str], operator.add]
    internal_log: list[str]  # 私有

# 这是可以的：子图内部的 messages 更新通过 add 合并
# 子图返回给父图的 messages 更新也通过父图的 add 合并
```

### 4. 不要依赖私有 Key 跨调用保持状态

```python
# 每次父图调用子图时，子图的私有 key 都会重新初始化
# 除非子图使用了独立的 Checkpointer
child_graph = child_builder.compile(checkpointer=True)  # 独立记忆
```

## 共享状态键 vs 状态转换：何时用哪个

| 维度 | 共享状态键 | 状态转换（包装函数） |
|------|-----------|-------------------|
| **适用场景** | 父子图有部分重叠 key | 父子图 schema 完全不同 |
| **代码量** | 最少，直接 add_node | 需要编写转换函数 |
| **灵活性** | 受限于同名 key | 完全自由映射 |
| **耦合度** | 中等（key 名称耦合） | 低（转换函数解耦） |
| **推荐程度** | 简单场景首选 | 复杂场景或需要解耦时 |

```python
# 共享状态键方式（简单直接）
parent.add_node("child", compiled_child)

# 状态转换方式（灵活但需要更多代码）
def call_child(state: ParentState) -> dict:
    result = compiled_child.invoke({"query": state["user_input"]})
    return {"answer": result["response"]}

parent.add_node("child", call_child)
```

## 小结

| 要素 | 说明 |
|------|------|
| **机制** | 父子图同名 key 自动双向传递 |
| **传入** | 父图的同名 key 值 → 子图初始状态 |
| **回传** | 子图的同名 key 更新 → 父图状态 |
| **私有 key** | 子图独有的 key 不影响父图 |
| **名称要求** | 必须完全相同（区分大小写） |
| **类型要求** | 必须兼容 |
| **优势** | 零代码转换，自动映射 |
| **局限** | 受限于同名 key，灵活性不如状态转换 |
