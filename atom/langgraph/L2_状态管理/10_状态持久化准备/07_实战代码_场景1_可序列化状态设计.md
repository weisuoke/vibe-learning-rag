# 实战代码 - 场景1：可序列化状态设计

## 场景说明

本场景演示如何为 LangGraph 持久化设计可序列化的状态。你将看到：

1. 使用 `TypedDict` 设计可序列化状态（只用基础类型）
2. 使用 `Pydantic BaseModel` 设计带验证的可序列化状态
3. 常见的不可序列化状态设计（文件句柄、连接、lambda）及修复方法
4. 使用 `JsonPlusSerializer` 测试状态是否可序列化
5. 一个完整的图示例，演示可序列化状态与 checkpoint 的配合

**核心原则：** 状态中的每个字段都必须能被序列化为字节，再从字节还原回来。不能序列化的对象（如文件句柄、数据库连接、lambda 函数）不能放进状态。

[来源: reference/source_状态持久化准备_01.md | LangGraph 源码分析]
[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

- JsonPlusSerializer 序列化测试
- 完整图 + checkpointer 实战

安装依赖：uv add langgraph

# 工具函数：测试对象是否可序列化
# ============================================================

serializer = JsonPlusSerializer()


def test_serializable(label: str, obj: Any) -> bool:
    """测试一个对象能否被 JsonPlusSerializer 正确序列化和反序列化

    JsonPlusSerializer 是 LangGraph 所有 checkpointer 的默认序列化器。
    它内部使用 msgpack（而非 JSON），支持的类型比标准 JSON 更广泛：
    - 基础类型：str, int, float, bool, None, list, dict
    - 扩展类型：datetime, UUID, Decimal, set, Enum, Path, Pydantic 模型等
    - 不支持：文件句柄、数据库连接、lambda、生成器等
    """
    try:
        # dumps_typed 返回 (type_tag, bytes)
        # type_tag 告诉反序列化器用什么方式解码，如 "msgpack"、"bytes"、"null"
        type_tag, data = serializer.dumps_typed(obj)

        # loads_typed 根据 type_tag 选择解码方式
        restored = serializer.loads_typed((type_tag, data))

        print(f"  [PASS] {label}")
        print(f"         类型标签: {type_tag}, 字节大小: {len(data)} bytes")
        print(f"         原始值: {obj}")
        print(f"         还原值: {restored}")
        return True
    except Exception as e:
        print(f"  [FAIL] {label}")
        print(f"         错误: {type(e).__name__}: {e}")
        return False

# 定义一个处理阶段的枚举（Enum 可以被序列化）
class ProcessStage(str, Enum):
    LOADING = "loading"
    PROCESSING = "processing"
    COMPLETED = "completed"


class GoodTypedDictState(TypedDict):
    """良好的可序列化状态设计

    规则：只使用可序列化的类型
    - str, int, float, bool, None          → 基础类型，完全支持
    - list, dict, tuple                     → 容器类型，完全支持
    - datetime, Enum, UUID                  → 扩展类型，JsonPlusSerializer 支持
    - Annotated[list, add]                  → Reducer 不影响序列化
    """
    # 基础类型 —— 完全安全
    query: str                                    # 用户查询
    temperature: float                            # LLM 温度参数
    max_retries: int                              # 最大重试次数
    is_complete: bool                             # 是否完成

    # 容器类型 —— 完全安全（内部元素也必须可序列化）
    messages: Annotated[list[str], add]           # 消息列表（带 Reducer）
    metadata: dict[str, str]                      # 元数据字典
    scores: list[float]                           # 评分列表

    # 扩展类型 —— JsonPlusSerializer 特别支持
    created_at: datetime                          # 时间戳
    stage: ProcessStage                           # 枚举阶段

    # 可选字段 —— None 也可以序列化
    error_message: str | None                     # 可能为空的错误信息

from pydantic import BaseModel, Field, field_validator


class DocumentChunk(BaseModel):
    """文档块模型 —— 嵌套的 Pydantic 模型也可以序列化

    JsonPlusSerializer 使用 msgpack 扩展类型 5 处理 Pydantic v2 模型：
    序列化时调用 model_dump()，反序列化时调用 Class(**dict)
    """
    content: str = Field(description="文档内容")
    page_number: int = Field(ge=1, description="页码，从 1 开始")
    score: float = Field(ge=0.0, le=1.0, description="相关性评分")

    @field_validator("content")
    @classmethod
    def content_not_empty(cls, v: str) -> str:
        if not v.strip():
            raise ValueError("文档内容不能为空")
        return v


class GoodPydanticState(BaseModel):
    """使用 Pydantic 定义的可序列化状态

    优势：
    1. 自动类型验证 —— 赋值时检查类型
    2. 字段约束 —— 如 ge=0, le=1 限制范围
    3. 自定义验证器 —— 复杂业务规则
    4. 完全可序列化 —— JsonPlusSerializer 原生支持 Pydantic v2
    """
    query: str = Field(description="用户查询")
    top_k: int = Field(default=5, ge=1, le=100, description="检索数量")
    chunks: list[DocumentChunk] = Field(default_factory=list, description="检索到的文档块")
    total_tokens: int = Field(default=0, ge=0, description="已消耗的 token 数")
    answer: str | None = Field(default=None, description="生成的回答")


# 创建 Pydantic 状态实例

    query="LangGraph 的 checkpoint 机制是什么？",
    top_k=3,
    chunks=[
        DocumentChunk(content="Checkpoint 是 LangGraph 的状态快照机制", page_number=1, score=0.95),
        DocumentChunk(content="每个节点执行后自动保存 checkpoint", page_number=2, score=0.88),
    ],
    total_tokens=150,
    answer=None,
)

print("\n测试 Pydantic 模型的序列化：")

def roundtrip_test(label: str, obj: Any) -> None:
    """完整的序列化往返测试，检查数据是否完整保留"""
    type_tag, data = serializer.dumps_typed(obj)
    restored = serializer.loads_typed((type_tag, data))

    # 检查值是否相等
    is_equal = (obj == restored)

    print(f"  {label}:")
    print(f"    类型标签: {type_tag}")
    print(f"    序列化大小: {len(data)} bytes")
    print(f"    往返一致: {'YES' if is_equal else 'NO'}")
    if not is_equal:
        print(f"    原始: {obj}")
        print(f"    还原: {restored}")
    print()


# 测试各种类型的往返
roundtrip_test("字符串", "Hello, LangGraph!")
roundtrip_test("整数", 42)
roundtrip_test("浮点数", 3.14159)
roundtrip_test("布尔值", True)
roundtrip_test("None", None)
roundtrip_test("列表", [1, "two", 3.0, None, True])
roundtrip_test("嵌套字典", {"a": {"b": {"c": [1, 2, 3]}}})
roundtrip_test("datetime", datetime(2026, 2, 27, 10, 30, 0))
roundtrip_test("Enum", ProcessStage.COMPLETED)
roundtrip_test("Pydantic 模型", DocumentChunk(content="测试内容", page_number=1, score=0.9))

class RAGState(TypedDict):
    """一个简化的 RAG 管道状态 —— 所有字段都可序列化

    设计要点：
    1. query/answer 用 str（基础类型）
    2. documents 用 list[dict]（不用自定义对象，保持简单）
    3. metadata 用 dict（灵活的键值对）
    4. messages 用 Annotated[list, add]（累积模式）
    5. 没有文件句柄、连接、函数等不可序列化对象
    """
    query: str
    documents: list[dict[str, Any]]
    answer: str
    confidence: float
    messages: Annotated[list[str], add]


def retrieve_node(state: RAGState) -> dict:
    """检索节点：模拟从向量数据库检索文档"""
    query = state["query"]
    # 模拟检索结果（实际项目中这里调用向量数据库）
    docs = [
        {"content": f"关于'{query}'的文档1：LangGraph 支持状态持久化", "score": 0.95},
        {"content": f"关于'{query}'的文档2：Checkpoint 保存每步状态", "score": 0.88},
        {"content": f"关于'{query}'的文档3：InMemorySaver 用于开发测试", "score": 0.72},
    ]
    return {
        "documents": docs,
        "messages": [f"检索完成：找到 {len(docs)} 篇相关文档"],
    }


def generate_node(state: RAGState) -> dict:
    """生成节点：模拟 LLM 生成回答"""
    docs = state["documents"]
    query = state["query"]
    # 模拟生成（实际项目中这里调用 LLM）
    context = " | ".join(d["content"] for d in docs[:2])
    answer = f"根据检索到的文档，{query}的答案是：LangGraph 通过 checkpoint 机制实现状态持久化。"
    avg_score = sum(d["score"] for d in docs) / len(docs) if docs else 0
    return {
        "answer": answer,
        "confidence": round(avg_score, 2),
        "messages": [f"生成完成：置信度 {avg_score:.2f}"],
    }


# 构建图
builder = StateGraph(RAGState)
builder.add_node("retrieve", retrieve_node)
builder.add_node("generate", generate_node)
builder.add_edge(START, "retrieve")
builder.add_edge("retrieve", "generate")
builder.add_edge("generate", END)

# 编译时配置 checkpointer
memory = InMemorySaver()
graph = builder.compile(checkpointer=memory)

# 执行图
config = {"configurable": {"thread_id": "rag-demo-001"}}
result = graph.invoke(
    {
        "query": "LangGraph 如何持久化状态",
        "documents": [],
        "answer": "",
        "confidence": 0.0,
        "messages": ["用户发起查询"],
    },
    config,
)

print(f"\n执行结果:")
print(f"  query: {result['query']}")
print(f"  documents 数量: {len(result['documents'])}")
print(f"  answer: {result['answer'][:50]}...")
print(f"  confidence: {result['confidence']}")
print(f"  messages: {result['messages']}")

# 验证 checkpoint 已保存
print(f"\n验证 checkpoint 持久化:")
snapshot = graph.get_state(config)
print(f"  checkpoint_id: {snapshot.config['configurable']['checkpoint_id']}")
print(f"  metadata.source: {snapshot.metadata.get('source')}")
print(f"  metadata.step: {snapshot.metadata.get('step')}")

# 验证状态可以从 checkpoint 恢复
print(f"\n验证状态恢复:")
restored_values = snapshot.values
print(f"  query 一致: {restored_values['query'] == result['query']}")
print(f"  answer 一致: {restored_values['answer'] == result['answer']}")
print(f"  confidence 一致: {restored_values['confidence'] == result['confidence']}")
print(f"  documents 数量一致: {len(restored_values['documents']) == len(result['documents'])}")
print(f"  messages 一致: {restored_values['messages'] == result['messages']}")

# 遍历所有 checkpoint，验证每一步都正确保存
print(f"\n所有 checkpoint 的状态大小:")
for cp in memory.list(config):
    step = cp.metadata.get("step", "?") if cp.metadata else "?"
    source = cp.metadata.get("source", "?") if cp.metadata else "?"
    # 序列化 checkpoint 的 channel_values 来测量大小
    cp_values = cp.checkpoint.get("channel_values", {})
    try:
        _, cp_bytes = serializer.dumps_typed(cp_values)
        size = len(cp_bytes)
    except Exception:
        size = 0
    print(f"  step={step}, source={source}, channel_values 大小: {size} bytes")

print("\n所有 checkpoint 都包含可序列化的状态，持久化验证通过！")

## 关键知识点解析

### 1. JsonPlusSerializer 支持的类型

| 类型分类 | 具体类型 | 序列化方式 |
|----------|---------|-----------|
| 基础类型 | `str`, `int`, `float`, `bool`, `None` | msgpack 原生 |
| 容器类型 | `list`, `dict`, `tuple` | msgpack 原生 |
| 字节类型 | `bytes`, `bytearray` | 直接存储 |
| 时间类型 | `datetime`, `date`, `time`, `timedelta` | msgpack 扩展类型 1/3 |
| 标识类型 | `UUID`, `Decimal`, `Enum` | msgpack 扩展类型 0 |
| 路径类型 | `Path`, `IP` | msgpack 扩展类型 1 |
| 集合类型 | `set`, `frozenset`, `deque` | msgpack 扩展类型 0 |
| 数据类 | `dataclass`, `NamedTuple` | msgpack 扩展类型 2 |
| Pydantic | `BaseModel` (v1/v2) | msgpack 扩展类型 4/5 |
| NumPy | `ndarray` | msgpack 扩展类型 6 |

### 2. 不可序列化对象速查

| 对象类型 | 为什么不行 | 替代方案 |
|----------|-----------|---------|
| 文件句柄 | 绑定操作系统资源，无法跨进程 | 存储文件路径 |
| 数据库连接 | 绑定网络 socket | 存储连接字符串 |
| Lambda/函数 | 包含代码引用，无法安全序列化 | 存储函数名 + 参数 |
| 生成器 | 包含执行状态，无法冻结 | 物化为列表 |
| 线程锁 | 操作系统原语 | 运行时创建 |
| HTTP 客户端 | 包含连接池 | 存储 URL + 配置 |

### 3. 序列化流程图

```
Python 对象
    ↓
JsonPlusSerializer.dumps_typed()
    ↓
判断类型：
  None?        → ("null", b"")
  bytes?       → ("bytes", raw_bytes)
  其他?        → try msgpack 编码
                   ↓
              成功 → ("msgpack", encoded_bytes)
              失败 → pickle_fallback=True?
                       ↓
                  是 → ("pickle", pickled_bytes)  ⚠️ 不安全
                  否 → 抛出异常
```

### 4. TypedDict vs Pydantic 选择指南

| 特性 | TypedDict | Pydantic BaseModel |
|------|-----------|-------------------|
| 运行时验证 | 无 | 有（自动类型检查） |
| 字段约束 | 无 | 有（ge, le, regex 等） |
| 自定义验证器 | 无 | 有（field_validator） |
| 性能 | 更快（无验证开销） | 稍慢（验证开销） |
| 序列化支持 | 完全支持 | 完全支持 |
| 推荐场景 | 简单状态、高性能 | 需要数据验证的场景 |

## 在 RAG 系统中的应用

可序列化状态设计在 RAG 系统中的典型应用：

- **检索结果持久化**：将检索到的文档块存为 `list[dict]`（而非自定义对象），确保 checkpoint 能正确保存
- **对话历史保存**：消息列表使用 `list[str]` 或 `list[dict]`，支持跨会话恢复
- **管道状态恢复**：RAG 管道中断后，从最近的 checkpoint 恢复，无需重新检索
- **多用户隔离**：每个用户的 `thread_id` 对应独立的 checkpoint 链，状态互不干扰
