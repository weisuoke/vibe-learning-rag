# 一句话总结

**Onboarding Wizard 是 OpenClaw 的交互式配置向导，通过问答式流程自动完成 Gateway、Workspace、Channel、Daemon 的全部设置，让用户无需手动编辑配置文件就能从零启动 AI 助手。**

---

## 核心本质

Onboarding Wizard 解决的核心问题是：**降低 OpenClaw 的配置门槛**。

它将复杂的 JSON5 配置文件编辑、守护进程安装、安全凭证生成等技术细节，转化为简单的选择题和填空题，让非技术用户也能快速上手。

---

## 价值主张

| 维度 | 传统方式 | Onboarding Wizard |
|------|---------|------------------|
| **配置方式** | 手动编辑 JSON5 文件 | 交互式问答 |
| **学习成本** | 需要理解配置语法和字段含义 | 只需回答问题 |
| **错误风险** | 语法错误、字段拼写错误、类型错误 | 自动验证输入 |
| **安全凭证** | 手动生成 Token/Password | 自动生成安全凭证 |
| **守护进程** | 手动配置 systemd/launchd | 一键安装 |
| **健康检查** | 手动验证 Gateway 启动 | 自动检测并报告 |

## 适用场景

### ✅ 推荐使用

1. **首次安装 OpenClaw**
2. **不熟悉 JSON5 配置语法**
3. **需要快速体验（QuickStart 模式）**
4. **需要完整控制（Advanced 模式）**
5. **自动化部署脚本（--non-interactive 模式）**

### ⚠️ 可选使用

1. **已有配置文件，只需微调** → 使用 `openclaw configure`
2. **只修改单个字段** → 使用 `openclaw config set`
3. **高级用户，熟悉配置结构** → 直接编辑 `~/.openclaw/openclaw.json`

---

## 核心流程（7 步）

```
1. Model/Auth     → 选择模型提供商（Anthropic/OpenAI/Custom）
2. Workspace      → 设置 Agent 工作目录
3. Gateway        → 配置网关（端口/绑定/认证）
4. Channels       → 连接聊天渠道（WhatsApp/Telegram/Discord）
5. Daemon         → 安装守护进程（LaunchAgent/systemd）
6. Health Check   → 验证 Gateway 启动
7. Skills         → 安装推荐技能
```

## 设计哲学

### 1. 渐进式披露（Progressive Disclosure）

- **QuickStart**：隐藏复杂选项，使用安全默认值
- **Advanced**：暴露全部控制，适合高级用户

### 2. 安全默认值（Secure by Default）

- Gateway 绑定：Loopback (127.0.0.1)
- 认证方式：Token（自动生成）
- DM 策略：Pairing（需要配对码）

### 3. 幂等性（Idempotency）

- 重复运行不会破坏已有配置
- 提供"保留现有值"、"更新值"、"重置"三种选项

### 4. 可恢复性（Recoverability）

- 配置错误时提示运行 `openclaw doctor`
- 提供 `--fix` 选项自动修复常见问题

## 与其他工具的关系

```
首次配置：openclaw onboard
    ↓
部分调整：openclaw configure
    ↓
单字段修改：openclaw config set <key> <value>
    ↓
高级编辑：直接编辑 ~/.openclaw/openclaw.json（支持热重载）
```

## 关键命令

```bash
# 基础用法
openclaw onboard

# 快速模式（最少交互）
openclaw onboard --flow quickstart

# 高级模式（完全控制）
openclaw onboard --flow advanced

# 非交互模式（自动化脚本）
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY"

# 远程 Gateway 配置
openclaw onboard --mode remote \
  --remote-url ws://gateway-host:18789
```

## 常见误区

### ❌ 误区 1：向导会覆盖已有配置

**真相**：向导会检测已有配置，提供"保留"、"更新"、"重置"三种选项。

### ❌ 误区 2：QuickStart 模式功能受限

**真相**：QuickStart 只是使用默认值减少交互，功能完全相同，后续可通过 `openclaw configure` 调整。

### ❌ 误区 3：必须使用向导配置

**真相**：向导是推荐方式，但高级用户可以直接编辑配置文件。

### ❌ 误区 4：非交互模式无法自定义

**真相**：非交互模式支持所有配置选项，通过命令行参数传递。
