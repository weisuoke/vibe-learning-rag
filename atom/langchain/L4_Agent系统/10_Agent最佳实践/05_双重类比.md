# Agent最佳实践 - 双重类比

> 用前端开发 + 日常生活两个视角，让 Agent 最佳实践从"抽象概念"变成"直觉理解"。

---

## 类比总览

| Agent 最佳实践概念 | 前端开发类比 | 日常生活类比 |
|-------------------|------------|-------------|
| **Middleware 架构** | Express.js 中间件 `app.use()` | 工厂流水线上的质检站 |
| **ModelRetryMiddleware** | Axios 的请求重试拦截器 | 电话没打通，再拨一次 |
| **ModelFallbackMiddleware** | CDN 回源（主节点挂了切备节点） | 主路堵车，走备用路线 |
| **ModelCallLimitMiddleware** | 前端防抖/节流 | 银行排号，每人限办 3 笔业务 |
| **PIIMiddleware** | 前端敏感数据脱敏显示 | 快递单上遮住手机号中间四位 |
| **HumanInTheLoopMiddleware** | Git PR Review（合并前需审核） | 签合同前律师审核条款 |
| **ShellToolMiddleware** | Docker 容器化部署 | 化学实验在通风橱里操作 |
| **SummarizationMiddleware** | 浏览器 LocalStorage 清理 | 整理笔记，把 10 页缩到 1 页 |
| **ToolCallLimitMiddleware** | API Rate Limiting（按接口限流） | 自助餐每人限取 3 只螃蟹 |
| **工具 description** | API 文档（Swagger/OpenAPI） | 菜单上菜品的描述和配图 |

## 详细类比

### 1. Middleware 架构 ↔ Express.js 中间件

**前端类比：Express.js 中间件**

```javascript
// Express.js 中间件 — 前端开发者最熟悉的中间件模式
const app = express();

app.use(cors());           // 跨域处理
app.use(rateLimit());       // 限流
app.use(auth());            // 认证
app.use(bodyParser());      // 请求解析

// 每个中间件只管一件事，按需组合
```

```python
# LangChain Middleware — 完全相同的理念
agent = create_agent(
    "gpt-4.1",
    tools=tools,
    middleware=[
        ModelCallLimitMiddleware(),    # 限流 ← 对应 rateLimit()
        ModelRetryMiddleware(),        # 重试
        PIIMiddleware(),               # 安全 ← 对应 auth()
        SummarizationMiddleware(),     # 上下文 ← 对应 bodyParser()
    ],
)
```

**日常生活类比：工厂流水线质检站**

```
原材料 → [检查重量] → [检查颜色] → [检查包装] → 成品
                ↑           ↑           ↑
            质检站 1     质检站 2     质检站 3

用户输入 → [限制检查] → [PII脱敏] → [LLM调用] → [输出脱敏] → 返回结果
                ↑           ↑                        ↑
         Middleware 1   Middleware 2          Middleware 3
```

### 2. ModelRetryMiddleware ↔ Axios 重试拦截器

**前端类比：**

```javascript
// Axios 请求重试拦截器
axios.interceptors.response.use(null, async (error) => {
    if (error.response?.status === 429) {  // 限流
        await sleep(1000);                  // 等 1 秒
        return axios.request(error.config); // 重试
    }
    throw error;
});
```

**日常生活类比：电话没打通**

```
第 1 次拨打 → 占线 → 等 1 秒
第 2 次拨打 → 占线 → 等 2 秒
第 3 次拨打 → 接通了！✅

如果 3 次都没通 → 换个号码打（ModelFallback）
```

### 3. ModelFallbackMiddleware ↔ CDN 回源

**前端类比：CDN 多级回源**

```
用户请求图片 → CDN 节点 A（最近的）
                  ↓ 没有
              CDN 节点 B（次近的）
                  ↓ 没有
              源站服务器（一定有）
                  ↓ 返回图片
```

**日常生活类比：备用路线**

```
上班路线 → 主路（高速公路）
              ↓ 堵车！
           备用路 1（国道）
              ↓ 也堵！
           备用路 2（小路）
              ↓ 虽然慢但能到 ✅
```

### 4. PIIMiddleware ↔ 前端数据脱敏

**前端类比：**

```javascript
// 前端显示手机号时脱敏
function maskPhone(phone) {
    return phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2');
    // 138****5678
}

// 前端显示银行卡时脱敏
function maskCard(card) {
    return '**** **** **** ' + card.slice(-4);
    // **** **** **** 1234
}
```

**日常生活类比：快递单隐私保护**

```
原始信息：张三 13812345678 北京市朝阳区XX小区3号楼501
脱敏后：  张* 138****5678 北京市朝阳区***
```

### 5. HumanInTheLoopMiddleware ↔ Git PR Review

**前端类比：**

```
开发者提交代码 → 创建 Pull Request
                    ↓
              代码审查者 Review
                    ↓
    ┌─────────┼─────────┐
  Approve    Comment    Reject
  (合并)    (修改后合并)  (拒绝)
```

**日常生活类比：签合同前的律师审核**

```
业务部门拟好合同 → 提交给法务审核
                      ↓
              法务律师审查条款
                      ↓
    ┌─────────┼─────────┐
   通过      修改条款     否决
  (签约)    (改后再签)   (不签)
```

### 6. SummarizationMiddleware ↔ 浏览器存储清理

**前端类比：**

```javascript
// LocalStorage 空间不够时的处理策略
if (getStorageSize() > MAX_SIZE * 0.8) {
    // 保留最近的数据，旧数据压缩/删除
    const oldData = getOldEntries();
    const summary = compress(oldData);
    localStorage.setItem('history_summary', summary);
    removeOldEntries();
}
```

**日常生活类比：整理笔记**

```
考试前笔记太多，看不完：
  ├── 第 1 章笔记: 20 页 ──→ 压缩为 2 页要点
  ├── 第 2 章笔记: 15 页 ──→ 压缩为 1.5 页要点
  ├── 第 3 章笔记: 10 页 ──→ 保持原样（最近学的）
  └── 总结笔记: 3.5 + 10 = 13.5 页  ← 能看完了！
```

## 组合类比：完整的 Agent 就像一家公司

```
一家公司（= 一个 Agent）：

CEO（System Prompt）
  → 定义公司使命和工作方式

部门（Tools）
  → 市场部（search 工具）
  → 财务部（calculator 工具）
  → 技术部（code_runner 工具）

管理制度（Middleware）
  → 考勤制度（ModelCallLimitMiddleware）  — 每天最多工作 8 小时
  → 请假补班（ModelRetryMiddleware）      — 失败了可以再试
  → 外包备选（ModelFallbackMiddleware）   — 核心员工请假就用外包
  → 保密协议（PIIMiddleware）             — 客户信息不能泄露
  → 审批流程（HumanInTheLoopMiddleware）  — 大额支出需要老板签字
  → 安全规范（ShellToolMiddleware）        — 危险操作在隔离环境执行
  → 工作汇报（SummarizationMiddleware）   — 定期总结，不让工作日志堆积
```
