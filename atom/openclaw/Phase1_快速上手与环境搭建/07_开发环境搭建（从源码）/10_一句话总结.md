# 一句话总结

**知识点**: 开发环境搭建（从源码）

---

## 核心总结

**OpenClaw 从源码搭建开发环境的本质是：通过 pnpm workspace 管理依赖、tsdown 快速编译、Node.js 22 运行时三层架构，实现高效、可靠、可维护的 monorepo 开发流程。**

---

## 关键要点

### 1. 三层架构

```
依赖管理层（pnpm） → 构建工具链层（tsdown） → 源码结构层（monorepo）
```

### 2. 核心命令

```bash
pnpm install && pnpm ui:build && pnpm build && pnpm gateway:watch
```

### 3. 技术选型逻辑

```
需求 → 约束 → 候选方案 → 权衡 → 最优解
```

---

## 记忆口诀

**pnpm 管依赖，tsdown 快编译，Node 22 来运行，三层架构开发顺。**

---

## 快速回顾

| 层级 | 技术 | 作用 |
|------|------|------|
| 依赖管理层 | pnpm workspace | 管理依赖包 |
| 构建工具链层 | tsdown + TypeScript | 编译代码 |
| 源码结构层 | monorepo | 组织代码 |

---

## 核心洞察

1. **技术选型不是偏好，而是逻辑推导的结果**
2. **三层架构不是设计模式，而是复杂度管理的必然**
3. **工具链不是锦上添花，而是开发效率的基础**
