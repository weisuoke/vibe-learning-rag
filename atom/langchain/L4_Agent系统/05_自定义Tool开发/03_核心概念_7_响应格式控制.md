# 核心概念 7：响应格式控制

> 本文档是【自定义Tool开发】知识点的第7个核心概念，讲解 LangChain 工具如何控制返回数据的格式和结构

---

## 概念定义

**响应格式控制**是指通过 `response_format` 参数控制工具返回值的结构，决定是返回简单字符串（`content`）还是分离的摘要和完整数据（`content_and_artifact`）。

在 Agent 系统中，工具经常需要返回大量数据（如搜索结果、数据库查询），但 LLM 只需要看到摘要。响应格式控制允许我们同时提供：
- **给 LLM 看的摘要**（简洁、易理解）
- **给程序用的完整数据**（结构化、可处理）

---

## 为什么需要响应格式控制？

### 问题场景

假设一个搜索工具返回 100 条结果：

**方式 1：全部返回给 LLM**
```python
@tool
def search(query: str) -> str:
    results = api.search(query, limit=100)
    # 返回所有数据（10000+ tokens）
    return json.dumps(results)
```

**问题**：
- ❌ 浪费 LLM 的 context window
- ❌ 增加推理成本
- ❌ LLM 可能被大量数据淹没

**方式 2：只返回摘要**
```python
@tool
def search(query: str) -> str:
    results = api.search(query, limit=100)
    # 只返回摘要
    return f"Found {len(results)} results. Top result: {results[0]['title']}"
```

**问题**：
- ❌ 程序无法访问完整数据
- ❌ 后续处理需要重新调用 API
- ❌ 丢失了有价值的元数据

**方式 3：使用 content_and_artifact（推荐）**
```python
@tool(response_format="content_and_artifact")
def search(query: str) -> tuple[str, dict]:
    results = api.search(query, limit=100)
    # 返回 (摘要, 完整数据)
    summary = f"Found {len(results)} results. Top: {results[0]['title']}"
    return summary, {"results": results, "metadata": {...}}
```

**优势**：
- ✅ LLM 只看到简洁摘要
- ✅ 程序可以访问完整数据
- ✅ 节省 tokens 和成本
- ✅ 保留所有元数据

## 两种响应格式对比

### 格式 1：content（默认）

**定义**：工具返回值直接作为 `ToolMessage.content`

```python
from langchain.tools import tool

@tool  # 默认 response_format="content"
def simple_calculator(expression: str) -> str:
    """Evaluate mathematical expressions."""
    result = eval(expression)
    return f"Result: {result}"

# 调用
message = simple_calculator.invoke({"expression": "2 + 2"})
print(type(message))  # <class 'langchain_core.messages.ToolMessage'>
print(message.content)  # "Result: 4"
print(message.artifact)  # None
```

**特点**：
- 返回值类型：`str` 或 `dict`
- 适合简单工具
- LLM 直接看到返回值
- 无法分离摘要和数据

[来源: reference/source_tool_development_01.md | LangChain 源码]

---

### 格式 2：content_and_artifact（推荐）

**定义**：工具返回 `tuple[str, Any]`，第一个元素是摘要，第二个是完整数据

```python
from langchain.tools import tool
from typing import Tuple, Dict, Any

@tool(response_format="content_and_artifact")
def advanced_search(query: str, limit: int = 10) -> Tuple[str, Dict[str, Any]]:
    """Search with rich metadata.

    Args:
        query: Search query
        limit: Maximum number of results

    Returns:
        Tuple of (summary, full_results)
    """
    # 模拟搜索 API
    results = [
        {
            "title": f"Result {i}",
            "url": f"https://example.com/{i}",
            "snippet": f"Snippet for result {i}",
            "score": 0.95 - i * 0.05,
        }
        for i in range(limit)
    ]

    # 摘要（给 LLM 看）
    summary = (
        f"Found {len(results)} results for '{query}'. "
        f"Top result: {results[0]['title']} (score: {results[0]['score']:.2f})"
    )

    # 完整数据（给程序用）
    artifact = {
        "query": query,
        "total_results": len(results),
        "results": results,
        "metadata": {
            "search_time_ms": 123,
            "api_version": "v2",
            "timestamp": "2026-03-02T10:00:00Z",
        },
    }

    return summary, artifact

# 调用
message = advanced_search.invoke({"query": "LangChain", "limit": 5})
print(message.content)
# "Found 5 results for 'LangChain'. Top result: Result 0 (score: 0.95)"

print(message.artifact)
# {'query': 'LangChain', 'total_results': 5, 'results': [...], 'metadata': {...}}
```

**特点**：
- 返回值类型：`tuple[str, Any]`
- 第一个元素：`ToolMessage.content`（给 LLM）
- 第二个元素：`ToolMessage.artifact`（给程序）
- 适合返回大量数据的工具

[来源: reference/source_tool_development_01.md | response_format 参数]

## ToolMessage 结构详解

### ToolMessage 的字段

```python
from langchain_core.messages import ToolMessage

# content 模式
message1 = ToolMessage(
    content="Result: 42",
    tool_call_id="call_123",
    name="calculator",
)

# content_and_artifact 模式
message2 = ToolMessage(
    content="Found 10 results",  # 摘要
    artifact={"results": [...]},  # 完整数据
    tool_call_id="call_456",
    name="search",
)
```

**关键字段**：
- `content`：工具返回的主要内容（LLM 可见）
- `artifact`：额外的结构化数据（程序可访问）
- `tool_call_id`：工具调用的唯一标识
- `name`：工具名称

## 实战示例：数据库查询工具

### 场景：查询用户订单

```python
"""
演示：使用 content_and_artifact 返回数据库查询结果
场景：Agent 需要查询用户订单，但只需要看到摘要
"""

from langchain.tools import tool
from typing import Tuple, Dict, Any, List
from pydantic import BaseModel, Field
import json

# ===== 1. 定义输入 Schema =====

class OrderQueryInput(BaseModel):
    user_id: int = Field(description="User ID")
    status: str = Field(
        default="all",
        description="Order status: 'pending', 'completed', 'cancelled', or 'all'"
    )
    limit: int = Field(default=10, ge=1, le=100, description="Max results")

# ===== 2. 创建工具（content_and_artifact 模式）=====

@tool(response_format="content_and_artifact", args_schema=OrderQueryInput)
def query_orders(
    user_id: int,
    status: str = "all",
    limit: int = 10
) -> Tuple[str, Dict[str, Any]]:
    """Query user orders from database.

    Returns:
        Tuple of (summary, full_data)
    """
    # 模拟数据库查询
    all_orders = [
        {
            "order_id": f"ORD{i:04d}",
            "user_id": user_id,
            "status": ["pending", "completed", "cancelled"][i % 3],
            "total": 100 + i * 10,
            "items": [{"product": f"Product {i}", "quantity": 1}],
            "created_at": f"2026-03-{i+1:02d}",
        }
        for i in range(20)
    ]

    # 过滤
    if status != "all":
        filtered = [o for o in all_orders if o["status"] == status]
    else:
        filtered = all_orders

    # 限制数量
    results = filtered[:limit]

    # ===== 摘要（给 LLM 看）=====
    if not results:
        summary = f"No orders found for user {user_id} with status '{status}'"
    else:
        total_amount = sum(o["total"] for o in results)
        summary = (
            f"Found {len(results)} orders for user {user_id}. "
            f"Total amount: ${total_amount}. "
            f"Latest order: {results[0]['order_id']} ({results[0]['status']})"
        )

    # ===== 完整数据（给程序用）=====
    artifact = {
        "user_id": user_id,
        "status_filter": status,
        "total_count": len(filtered),
        "returned_count": len(results),
        "orders": results,
        "statistics": {
            "total_amount": sum(o["total"] for o in results),
            "avg_order_value": sum(o["total"] for o in results) / len(results) if results else 0,
            "status_breakdown": {
                "pending": len([o for o in results if o["status"] == "pending"]),
                "completed": len([o for o in results if o["status"] == "completed"]),
                "cancelled": len([o for o in results if o["status"] == "cancelled"]),
            },
        },
        "metadata": {
            "query_time_ms": 45,
            "database": "orders_db",
        },
    }

    return summary, artifact

# ===== 3. 测试工具 =====
print("=== 测试 1：查询所有订单 ===")
result1 = query_orders.invoke({"user_id": 123, "status": "all", "limit": 5})

print("Content (LLM 看到的):")
print(result1.content)
print("\nArtifact (程序可访问的):")
print(json.dumps(result1.artifact, indent=2))

print("\n=== 测试 2：只查询已完成订单 ===")
result2 = query_orders.invoke({"user_id": 123, "status": "completed", "limit": 3})

print("Content:")
print(result2.content)
print("\nStatistics:")
print(json.dumps(result2.artifact["statistics"], indent=2))

# ===== 4. 在 Agent 中使用 =====
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain_openai import ChatOpenAI
from langchain import hub

async def agent_with_db_tool():
    llm = ChatOpenAI(model="gpt-4", temperature=0)
    tools = [query_orders]

    prompt = hub.pull("hwchase17/openai-functions-agent")
    agent = create_openai_functions_agent(llm, tools, prompt)
    agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

    # Agent 只看到摘要，但程序可以访问完整数据
    result = await agent_executor.ainvoke({
        "input": "Show me the completed orders for user 123"
    })

    print("\n=== Agent 输出 ===")
    print(result["output"])

    # 访问完整数据（如果需要）
    if "intermediate_steps" in result:
        for action, observation in result["intermediate_steps"]:
            if hasattr(observation, "artifact"):
                print("\n=== 完整订单数据 ===")
                print(json.dumps(observation.artifact["orders"], indent=2))

# import asyncio
# asyncio.run(agent_with_db_tool())
```

## 实战示例：RAG 检索工具

### 场景：文档检索返回摘要和原文

```python
"""
演示：RAG 检索工具使用 content_and_artifact
场景：检索相关文档，LLM 看到摘要，程序保留完整文档
"""

from langchain.tools import tool
from langchain_community.vectorstores import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain.schema import Document
from typing import Tuple, Dict, Any, List

@tool(response_format="content_and_artifact")
def rag_search(query: str, top_k: int = 3) -> Tuple[str, Dict[str, Any]]:
    """Search knowledge base for relevant documents.

    Args:
        query: User query
        top_k: Number of documents to retrieve

    Returns:
        Tuple of (summary, full_documents)
    """
    # 初始化向量数据库
    vectorstore = Chroma(
        embedding_function=OpenAIEmbeddings(),
        persist_directory="./chroma_db"
    )

    # 检索文档
    docs = vectorstore.similarity_search_with_score(query, k=top_k)

    # ===== 摘要（给 LLM 看）=====
    # 只包含文档片段，不包含完整内容
    snippets = []
    for doc, score in docs:
        # 截取前 200 字符作为片段
        snippet = doc.page_content[:200] + "..." if len(doc.page_content) > 200 else doc.page_content
        snippets.append(f"[Score: {score:.2f}] {snippet}")

    summary = (
        f"Found {len(docs)} relevant documents:\n\n" +
        "\n\n".join(snippets)
    )

    # ===== 完整数据（给程序用）=====
    artifact = {
        "query": query,
        "total_results": len(docs),
        "documents": [
            {
                "content": doc.page_content,
                "metadata": doc.metadata,
                "score": score,
            }
            for doc, score in docs
        ],
        "retrieval_metadata": {
            "vectorstore": "Chroma",
            "embedding_model": "text-embedding-3-small",
            "top_k": top_k,
        },
    }

    return summary, artifact

# 使用示例
result = rag_search.invoke({"query": "What is LangChain?", "top_k": 3})

print("=== LLM 看到的摘要 ===")
print(result.content)

print("\n=== 程序可访问的完整文档 ===")
for doc in result.artifact["documents"]:
    print(f"\nScore: {doc['score']:.2f}")
    print(f"Content: {doc['content'][:100]}...")
    print(f"Metadata: {doc['metadata']}")
```

## 使用场景对比

### 何时使用 content？

**适合场景**：
- ✅ 简单计算工具（如计算器）
- ✅ 返回值很小（< 500 tokens）
- ✅ 不需要后续处理
- ✅ LLM 需要看到完整结果

**示例**：
```python
@tool  # 默认 content 模式
def add(a: int, b: int) -> str:
    """Add two numbers."""
    return f"{a} + {b} = {a + b}"
```

### 何时使用 content_and_artifact？

**适合场景**：
- ✅ 返回大量数据（如搜索结果）
- ✅ 需要保留元数据
- ✅ 程序需要后续处理
- ✅ 想节省 LLM tokens

**示例**：
```python
@tool(response_format="content_and_artifact")
def search_api(query: str) -> Tuple[str, Dict]:
    """Search API with rich results."""
    results = api.search(query, limit=100)
    summary = f"Found {len(results)} results"
    return summary, {"results": results, "metadata": {...}}
```

## 与 RAG/Agent 的关系

### 在 RAG 系统中的应用

**问题**：RAG 检索可能返回大量文档（10+ 个，每个 1000+ tokens）

**解决方案**：使用 content_and_artifact
- `content`：文档摘要（给 LLM 生成答案）
- `artifact`：完整文档（给程序做引用、评分、重排序）

```python
@tool(response_format="content_and_artifact")
def rag_retrieval(query: str) -> Tuple[str, Dict]:
    """Retrieve documents from knowledge base."""
    docs = vectorstore.similarity_search(query, k=10)

    # 摘要：只包含关键信息
    summary = "\n\n".join([
        f"Doc {i+1}: {doc.page_content[:200]}..."
        for i, doc in enumerate(docs[:3])  # 只给 LLM 看前 3 个
    ])

    # 完整数据：保留所有文档
    artifact = {
        "documents": [doc.dict() for doc in docs],
        "scores": [...],
    }

    return summary, artifact
```

### 在 Agent 系统中的应用

**问题**：Agent 调用工具后，可能需要多次处理结果

**解决方案**：使用 content_and_artifact
- `content`：给 LLM 决策
- `artifact`：给后续工具使用

```python
# 工具 1：搜索（返回 artifact）
@tool(response_format="content_and_artifact")
def search(query: str) -> Tuple[str, Dict]:
    results = api.search(query)
    return f"Found {len(results)} results", {"results": results}

# 工具 2：过滤（使用 artifact）
@tool
def filter_results(criteria: str) -> str:
    # 从上一个工具的 artifact 中获取数据
    # （需要通过 Agent 的 memory 或 state 传递）
    ...
```

## 最佳实践

### 1. 摘要编写原则

**好的摘要**：
- ✅ 简洁（< 200 tokens）
- ✅ 包含关键信息（数量、最重要的结果）
- ✅ 易于 LLM 理解
- ✅ 包含足够的上下文

**示例**：
```python
# ✅ 好的摘要
summary = (
    f"Found {len(results)} products matching '{query}'. "
    f"Top result: {results[0]['name']} (${results[0]['price']}). "
    f"Price range: ${min_price} - ${max_price}."
)

# ❌ 不好的摘要（太简单）
summary = f"Found {len(results)} results"

# ❌ 不好的摘要（太详细）
summary = json.dumps(results)  # 完整数据不应该在摘要中
```

### 2. Artifact 结构设计

**推荐结构**：
```python
artifact = {
    # 核心数据
    "results": [...],

    # 元数据
    "metadata": {
        "query": query,
        "total_count": len(results),
        "api_version": "v2",
        "timestamp": "2026-03-02T10:00:00Z",
    },

    # 统计信息
    "statistics": {
        "avg_score": 0.85,
        "max_score": 0.95,
        "min_score": 0.75,
    },

    # 调试信息（可选）
    "debug": {
        "query_time_ms": 123,
        "cache_hit": False,
    },
}
```

### 3. 类型提示

**始终使用类型提示**，帮助 IDE 和类型检查器：

```python
from typing import Tuple, Dict, Any, List

@tool(response_format="content_and_artifact")
def my_tool(query: str) -> Tuple[str, Dict[str, Any]]:
    """Tool with proper type hints."""
    summary: str = "..."
    artifact: Dict[str, Any] = {...}
    return summary, artifact
```

### 4. 错误处理

**在两个部分都提供错误信息**：

```python
from langchain.tools import ToolException

@tool(response_format="content_and_artifact")
def safe_api_call(url: str) -> Tuple[str, Dict[str, Any]]:
    """API call with error handling."""
    try:
        response = requests.get(url)
        response.raise_for_status()
        data = response.json()

        summary = f"Successfully fetched data from {url}"
        artifact = {"data": data, "status": "success"}
        return summary, artifact

    except requests.HTTPError as e:
        # 错误信息在摘要中
        summary = f"API request failed: {e.response.status_code}"

        # 详细错误在 artifact 中
        artifact = {
            "status": "error",
            "error_type": "HTTPError",
            "error_message": str(e),
            "status_code": e.response.status_code,
        }

        return summary, artifact
```

## 常见陷阱与解决方案

### 陷阱 1：忘记返回 tuple

**错误示例**：
```python
@tool(response_format="content_and_artifact")
def bad_tool(query: str) -> str:  # 错误：返回类型应该是 Tuple
    return "Result"  # 错误：应该返回 tuple
```

**解决方案**：
```python
@tool(response_format="content_and_artifact")
def good_tool(query: str) -> Tuple[str, Dict]:
    return "Summary", {"data": "..."}  # 正确
```

### 陷阱 2：摘要太长

**错误示例**：
```python
@tool(response_format="content_and_artifact")
def bad_tool(query: str) -> Tuple[str, Dict]:
    results = api.search(query, limit=100)
    # 摘要包含所有结果（浪费 tokens）
    summary = json.dumps(results)
    return summary, {"results": results}
```

**解决方案**：
```python
@tool(response_format="content_and_artifact")
def good_tool(query: str) -> Tuple[str, Dict]:
    results = api.search(query, limit=100)
    # 摘要只包含关键信息
    summary = f"Found {len(results)} results. Top: {results[0]['title']}"
    return summary, {"results": results}
```

### 陷阱 3：Artifact 中缺少元数据

**错误示例**：
```python
@tool(response_format="content_and_artifact")
def bad_tool(query: str) -> Tuple[str, Dict]:
    results = api.search(query)
    # Artifact 只有结果，缺少上下文
    return "Found results", {"results": results}
```

**解决方案**：
```python
@tool(response_format="content_and_artifact")
def good_tool(query: str) -> Tuple[str, Dict]:
    results = api.search(query)
    # Artifact 包含完整上下文
    return "Found results", {
        "query": query,
        "results": results,
        "metadata": {"timestamp": "...", "api_version": "v2"},
    }
```

## 性能优化

### Token 使用对比

```python
# 假设搜索返回 100 条结果，每条 200 tokens

# 方式 1：content 模式（全部返回）
@tool
def search_content(query: str) -> str:
    results = api.search(query, limit=100)
    return json.dumps(results)  # 20000 tokens！

# 方式 2：content_and_artifact 模式（只返回摘要）
@tool(response_format="content_and_artifact")
def search_artifact(query: str) -> Tuple[str, Dict]:
    results = api.search(query, limit=100)
    summary = f"Found {len(results)} results"  # 10 tokens
    return summary, {"results": results}

# Token 节省：20000 - 10 = 19990 tokens（99.95% 节省）
```

### 成本对比

假设 GPT-4 价格：$0.03 / 1K tokens

- **content 模式**：20K tokens × $0.03 = **$0.60 / 次**
- **content_and_artifact 模式**：10 tokens × $0.03 = **$0.0003 / 次**

**节省**：$0.60 - $0.0003 = **$0.5997 / 次**（99.95% 节省）
