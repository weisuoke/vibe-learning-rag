## 场景1：动态Schema基础操作

### 核心价值

**问题**：业务需求变化，需要添加新字段，但不想重建Collection。

**解决方案**：使用动态Schema，无需修改Schema即可添加字段。

---

### 最小可用代码

```python
"""
动态Schema基础操作 - 最小可用实现
功能：启用动态Schema、插入动态字段、查询动态字段
"""

from pymilvus import MilvusClient, DataType
import numpy as np

# 1. 连接Milvus
client = MilvusClient("http://localhost:19530")

# 2. 创建启用动态Schema的Collection
collection_name = "dynamic_demo"

# 删除已存在的Collection（如果有）
if client.has_collection(collection_name):
    client.drop_collection(collection_name)

# 创建Schema（启用动态字段）
schema = client.create_schema(
    auto_id=False,
    enable_dynamic_field=True  # 关键：启用动态字段
)

# 添加固定字段
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=128)
schema.add_field(field_name="title", datatype=DataType.VARCHAR, max_length=512)

# 创建索引
index_params = client.prepare_index_params()
index_params.add_index(
    field_name="vector",
    index_type="AUTOINDEX",
    metric_type="COSINE"
)

# 创建Collection
client.create_collection(
    collection_name=collection_name,
    schema=schema,
    index_params=index_params
)

print(f"Collection '{collection_name}' 创建成功（动态Schema已启用）")

# 3. 插入带有动态字段的数据
data = [
    {
        "id": 1,
        "vector": np.random.rand(128).tolist(),
        "title": "Document 1",
        # 动态字段（未在Schema中定义）
        "author": "Alice",
        "category": "tech",
        "tags": ["AI", "ML"]
    },
    {
        "id": 2,
        "vector": np.random.rand(128).tolist(),
        "title": "Document 2",
        # 不同的动态字段
        "author": "Bob",
        "priority": "high",
        "created_at": 1640000000
    },
    {
        "id": 3,
        "vector": np.random.rand(128).tolist(),
        "title": "Document 3",
        # 只有部分动态字段
        "author": "Charlie"
    }
]

insert_result = client.insert(collection_name=collection_name, data=data)
print(f"插入 {len(data)} 条数据（包含动态字段）")

# 4. 查询动态字段
results = client.query(
    collection_name=collection_name,
    filter="id in [1, 2, 3]",
    output_fields=["id", "title", "author", "category", "tags", "priority", "created_at"]
)

print("\n查询结果：")
for result in results:
    print(f"ID: {result['id']}, Title: {result['title']}")
    print(f"  Author: {result.get('author', 'N/A')}")
    print(f"  Category: {result.get('category', 'N/A')}")
    print(f"  Tags: {result.get('tags', 'N/A')}")
    print(f"  Priority: {result.get('priority', 'N/A')}")
    print()

# 5. 过滤动态字段
results = client.query(
    collection_name=collection_name,
    filter='author == "Alice"',
    output_fields=["id", "title", "author", "category"]
)

print("过滤结果（author == 'Alice'）：")
for result in results:
    print(f"ID: {result['id']}, Title: {result['title']}, Author: {result['author']}")

# 清理
client.drop_collection(collection_name)
print(f"\nCollection '{collection_name}' 已删除")
```

**来源**：[来源: reference/context7_pymilvus_01.md], [来源: reference/source_动态Schema_01.md]

---

### 关键要点

1. **启用动态Schema**：`enable_dynamic_field=True`
2. **动态字段自动存储**：存储在隐式的`$meta`字段中
3. **查询动态字段**：直接在`output_fields`中指定字段名
4. **过滤动态字段**：使用标准过滤表达式
5. **缺失字段处理**：使用`.get()`方法避免KeyError

## 场景2：Int8向量实战

### 核心价值

**问题**：大规模向量存储占用大量内存，成本高昂。

**解决方案**：使用Int8向量量化，节省75%内存，提升4x性能。

---

### 最小可用代码

```python
"""
Int8向量实战 - 最小可用实现
功能：Float32向量量化为Int8、创建Int8 Collection、性能对比
"""

from pymilvus import MilvusClient, DataType
import numpy as np
import time

# 1. 连接Milvus
client = MilvusClient("http://localhost:19530")

# 2. Float32向量量化为Int8
def quantize_to_int8(vector_float32):
    """
    将Float32向量量化为Int8向量
    
    量化步骤：
    1. 归一化到[-1, 1]
    2. 映射到[-127, 127]
    3. 转换为int8类型
    """
    # 归一化
    norm = np.linalg.norm(vector_float32)
    if norm == 0:
        return np.zeros(len(vector_float32), dtype=np.int8)
    
    vector_normalized = vector_float32 / norm
    
    # 映射到[-127, 127]
    vector_int8 = np.clip(vector_normalized * 127, -127, 127).astype(np.int8)
    
    return vector_int8

# 测试量化
vector_float32 = np.random.randn(128).astype(np.float32)
vector_int8 = quantize_to_int8(vector_float32)

print("量化示例：")
print(f"Float32向量（前10维）: {vector_float32[:10]}")
print(f"Int8向量（前10维）: {vector_int8[:10]}")
print(f"Float32内存: {vector_float32.nbytes} bytes")
print(f"Int8内存: {vector_int8.nbytes} bytes")
print(f"内存节省: {(1 - vector_int8.nbytes / vector_float32.nbytes) * 100:.1f}%\n")

# 3. 创建Int8向量Collection
collection_name_int8 = "int8_demo"

if client.has_collection(collection_name_int8):
    client.drop_collection(collection_name_int8)

schema = client.create_schema(auto_id=False)
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(
    field_name="vector",
    datatype=DataType.INT8_VECTOR,  # 关键：使用INT8_VECTOR
    dim=128  # 必须是8的倍数
)
schema.add_field(field_name="title", datatype=DataType.VARCHAR, max_length=512)

index_params = client.prepare_index_params()
index_params.add_index(
    field_name="vector",
    index_type="AUTOINDEX",
    metric_type="COSINE"
)

client.create_collection(
    collection_name=collection_name_int8,
    schema=schema,
    index_params=index_params
)

print(f"Int8向量Collection '{collection_name_int8}' 创建成功")

# 4. 插入Int8向量数据
num_vectors = 1000
data_int8 = []

for i in range(num_vectors):
    vector_float32 = np.random.randn(128).astype(np.float32)
    vector_int8 = quantize_to_int8(vector_float32)
    
    data_int8.append({
        "id": i,
        "vector": vector_int8.tolist(),
        "title": f"Document {i}"
    })

start_time = time.time()
client.insert(collection_name=collection_name_int8, data=data_int8)
insert_time = time.time() - start_time

print(f"插入 {num_vectors} 条Int8向量数据，耗时: {insert_time:.2f}秒")

# 5. 性能对比
print(f"\n内存对比（{num_vectors}条向量，128维）：")
float32_memory = num_vectors * 128 * 4  # 4 bytes per dimension
int8_memory = num_vectors * 128 * 1     # 1 byte per dimension
print(f"Float32内存: {float32_memory / 1024:.2f} KB")
print(f"Int8内存: {int8_memory / 1024:.2f} KB")
print(f"内存节省: {(1 - int8_memory / float32_memory) * 100:.1f}%")

# 清理
client.drop_collection(collection_name_int8)
print(f"\nCollections已删除")
```

**来源**：[来源: reference/search_Int8向量_02.md], [来源: reference/source_动态Schema_01.md]

---

### 关键要点

1. **量化方法**：归一化 → 映射到[-127, 127] → 转换为int8
2. **维度限制**：必须是8的倍数（SIMD优化要求）
3. **内存节省**：75%（4 bytes → 1 byte）
4. **性能提升**：4x QPS（生产案例）
5. **精度损失**：5-10%召回率下降（可接受）

## 场景3：AddCollectionField实战

### 核心价值

**问题**：生产环境已有数据，需要添加新字段，但不能重建Collection。

**解决方案**：使用AddCollectionField API渐进式添加字段。

---

### 最小可用代码

```python
"""
AddCollectionField实战 - 最小可用实现
功能：渐进式添加字段、设置nullable和默认值、处理已存在数据
"""

from pymilvus import MilvusClient, DataType
import numpy as np
import time

# 1. 连接Milvus
client = MilvusClient("http://localhost:19530")

# 2. 创建初始Collection
collection_name = "addfield_demo"

if client.has_collection(collection_name):
    client.drop_collection(collection_name)

schema = client.create_schema(auto_id=False)
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=128)
schema.add_field(field_name="title", datatype=DataType.VARCHAR, max_length=512)

index_params = client.prepare_index_params()
index_params.add_index(
    field_name="vector",
    index_type="AUTOINDEX",
    metric_type="COSINE"
)

client.create_collection(
    collection_name=collection_name,
    schema=schema,
    index_params=index_params
)

print(f"Collection '{collection_name}' 创建成功（初始Schema）")

# 3. 插入初始数据
initial_data = [
    {
        "id": i,
        "vector": np.random.rand(128).tolist(),
        "title": f"Document {i}"
    }
    for i in range(100)
]

client.insert(collection_name=collection_name, data=initial_data)
print(f"插入 {len(initial_data)} 条初始数据")

# 4. 添加新字段（nullable + 默认值）
print(f"\n添加新字段 'category'...")

client.add_collection_field(
    collection_name=collection_name,
    field_name="category",
    data_type=DataType.VARCHAR,
    max_length=100,
    nullable=True,  # 必须为True
    default_value="uncategorized"  # 默认值
)

print(f"字段 'category' 添加成功")

# 5. 重新加载Collection（重要！）
client.release_collection(collection_name)
time.sleep(1)
client.load_collection(collection_name)
print(f"Collection重新加载完成")

# 6. 查询已存在数据（新字段值为默认值）
results = client.query(
    collection_name=collection_name,
    filter="id in [0, 1, 2]",
    output_fields=["id", "title", "category"]
)

print(f"\n已存在数据的新字段值：")
for result in results:
    print(f"ID: {result['id']}, Title: {result['title']}, Category: {result.get('category', 'NULL')}")

# 7. 插入新数据（包含新字段）
new_data = [
    {
        "id": 100,
        "vector": np.random.rand(128).tolist(),
        "title": "New Document 1",
        "category": "tech"
    },
    {
        "id": 101,
        "vector": np.random.rand(128).tolist(),
        "title": "New Document 2",
        "category": "business"
    }
]

client.insert(collection_name=collection_name, data=new_data)
print(f"\n插入 {len(new_data)} 条新数据（包含新字段）")

# 8. 查询新数据
results = client.query(
    collection_name=collection_name,
    filter="id >= 100",
    output_fields=["id", "title", "category"]
)

print(f"\n新数据的字段值：")
for result in results:
    print(f"ID: {result['id']}, Title: {result['title']}, Category: {result.get('category', 'NULL')}")

# 清理
client.drop_collection(collection_name)
print(f"\nCollection '{collection_name}' 已删除")
```

**来源**：[来源: reference/context7_milvus_02.md], [来源: reference/search_动态Schema_01.md]

---

### 关键要点

1. **nullable必需**：添加的字段必须设置`nullable=True`
2. **默认值设置**：使用`default_value`为已存在数据设置默认值
3. **重新加载Collection**：添加字段后必须重新加载
4. **暂停写入**：添加字段前建议暂停写入操作（避免并发冲突）
5. **向后兼容**：已存在数据的新字段值为NULL或默认值

## 生产环境注意事项

### 1. 动态Schema性能影响

**问题**（[来源: reference/search_动态Schema_01.md]）：
- group_by对缺失动态字段处理错误（Issue #47438）
- nullable字段聚合慢4倍（Issue #47566）

**解决方案**：
- 核心字段使用固定Schema
- 业务字段使用动态Schema
- 避免对动态字段进行高频聚合

---

### 2. Int8向量维度限制

**问题**（[来源: reference/search_Int8向量_02.md]）：
- Int8向量维度必须是8的倍数（Issue #43466）

**解决方案**：
```python
# 正确：维度是8的倍数
dim = 128  # 128 = 8 * 16
dim = 256  # 256 = 8 * 32
dim = 768  # 768 = 8 * 96

# 错误：维度不是8的倍数
dim = 100  # 报错
dim = 765  # 报错
```

---

### 3. AddCollectionField并发冲突

**问题**（[来源: reference/search_动态Schema_01.md]）：
- 与并发insert操作冲突（Issue #41858）
- HybridSearch可能失败（Issue #43003）

**解决方案**：
```python
# 1. 暂停写入操作
# 2. 添加字段
client.add_collection_field(...)
# 3. 重新加载Collection
client.release_collection(collection_name)
client.load_collection(collection_name)
# 4. 测试所有查询类型
# 5. 恢复写入操作
```
