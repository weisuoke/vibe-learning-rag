# 30字核心

## 核心定义

**Prompt Engineering 是通过精心设计输入指令，让大模型准确理解意图并生成高质量输出的技术。**

---

## 一句话本质

**用人类语言精确"编程"AI，让它按你的意图工作。**

---

## 五大核心技巧

| 技巧 | 核心作用 | RAG 应用 |
|------|---------|---------|
| **角色设定** | 定义 AI 的身份和行为模式 | 让 RAG 助手保持专业一致的回答风格 |
| **指令清晰** | 用明确具体的命令消除歧义 | 精确控制检索和生成的每个步骤 |
| **格式控制** | 规定输出的结构化格式 | 确保 RAG 返回可解析的 JSON/XML |
| **上下文工程** ⭐ | 优化注入的上下文信息 | RAG 的核心：如何组织检索到的文档 |
| **输出质量控制** ⭐ | 约束和验证生成内容 | 防止 RAG 幻觉，确保答案可靠 |

> ⭐ 标记为 2025-2026 年的新重点

---

## 关键术语

### System Prompt（系统提示词）
定义 AI 的全局角色和行为规则，在整个对话中持续生效。

**示例：**
```
你是一个专业的技术文档助手，擅长将复杂概念用简单语言解释。
```

### User Prompt（用户提示词）
用户的具体请求或问题，每次对话可以不同。

**示例：**
```
请解释什么是向量数据库，并给出 Python 示例。
```

### Context Window（上下文窗口）
模型一次能处理的最大 token 数量，包括输入和输出。

**RAG 关键点：** 需要在有限的窗口内塞入最相关的检索内容。

### Few-shot Prompting（少样本提示）
在提示词中提供几个示例，让模型学习期望的输出模式。

**示例：**
```
问题：什么是 Python？
答案：Python 是一种高级编程语言...

问题：什么是 RAG？
答案：
```

### Chain-of-Thought（思维链）
引导模型逐步推理，而非直接给出答案。

**触发词：** "让我们一步步思考"、"请详细解释你的推理过程"

## 为什么重要？

### 对 RAG 开发的价值

1. **检索精度提升 30-50%**
   - 清晰的查询改写指令 → 更准确的向量检索
   - 明确的相关性判断标准 → 更好的文档筛选

2. **生成质量提升 40-60%**
   - 结构化输出格式 → 易于解析和展示
   - 上下文工程优化 → 减少幻觉和错误

3. **用户体验提升 50%+**
   - 一致的角色设定 → 专业可信的助手形象
   - 质量控制机制 → 可靠的答案输出

**来源：** [Source: IBM 2026 Prompt Engineering Guide](https://www.ibm.com/think/prompt-engineering)
