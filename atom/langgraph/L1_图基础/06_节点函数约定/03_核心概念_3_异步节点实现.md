# 核心概念 3：异步节点实现

## 概述

异步节点是 LangGraph 中处理 I/O 密集型操作的核心机制。本文档深入讲解异步节点的实现原理、执行机制、与同步节点的区别，以及常见的陷阱和最佳实践。

---

## 1. async/await 语法基础

### 1.1 Python 异步编程基础

在 LangGraph 中，异步节点使用 Python 的 `async/await` 语法定义：

```python
# 同步节点
def sync_node(state: State) -> dict:
    result = expensive_operation()  # 阻塞执行
    return {"result": result}

# 异步节点
async def async_node(state: State) -> dict:
    result = await async_expensive_operation()  # 非阻塞执行
    return {"result": result}
```

**关键区别**：
- 同步节点：执行时阻塞当前线程，直到操作完成
- 异步节点：执行时释放控制权，允许其他任务并发执行

[来源: reference/source_节点函数约定_01.md]

### 1.2 异步节点的定义约定

LangGraph 支持多种异步节点签名：

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph
from langchain_core.runnables import RunnableConfig

class State(TypedDict):
    messages: list[str]
    result: str

# 基础异步节点
async def basic_async_node(state: State) -> dict:
    result = await some_async_api()
    return {"result": result}

# 带 Config 参数的异步节点
async def async_node_with_config(
    state: State, 
    config: RunnableConfig
) -> dict:
    api_key = config.get("configurable", {}).get("api_key")
    result = await some_async_api(api_key=api_key)
    return {"result": result}

# 带 Runtime 参数的异步节点
from langgraph.types import Runtime

async def async_node_with_runtime(
    state: State,
    runtime: Runtime
) -> dict:
    context = runtime.context
    result = await some_async_api(context=context)
    return {"result": result}
```

## 2. 异步节点的执行机制

### 2.1 自动异步检测

LangGraph 会自动检测节点函数是否为异步函数，并相应地处理：

```python
# 源码片段（简化版）
from langgraph.pregel._call import is_async_callable
import functools
from concurrent.futures import ThreadPoolExecutor

if is_async_callable(func):
    # 异步函数：直接包装
    run = RunnableCallable(
        None, func, name=func.__name__, trace=False, recurse=False
    )
else:
    # 同步函数：包装成异步执行
    afunc = functools.update_wrapper(
        functools.partial(run_in_executor, None, func), func
    )
    run = RunnableCallable(
        func,
        afunc,
        name=func.__name__,
        trace=False,
        recurse=False,
    )
```

**关键机制**：
1. **异步函数**：直接使用 `await` 调用
2. **同步函数**：在线程池中执行，包装成异步接口

[来源: reference/source_节点函数约定_01.md]

### 2.2 异步执行流程

异步节点的执行流程：

```
1. 图调用 ainvoke() 或 astream()
   ↓
2. 检测节点函数类型
   ↓
3a. 异步函数 → 直接 await 执行
3b. 同步函数 → run_in_executor() 包装
   ↓
4. 收集返回值
   ↓
5. 更新状态
   ↓
6. 继续下一个节点
```

### 2.3 并发执行支持

LangGraph 支持多个异步节点并发执行：

```python
from langgraph.graph import StateGraph, START

builder = StateGraph(State)

# 添加多个异步节点
builder.add_node("fetch_user", fetch_user_async)
builder.add_node("fetch_orders", fetch_orders_async)
builder.add_node("fetch_products", fetch_products_async)

# 并发执行
builder.add_edge(START, "fetch_user")
builder.add_edge(START, "fetch_orders")
builder.add_edge(START, "fetch_products")

# 这三个节点会并发执行
graph = builder.compile()
result = await graph.ainvoke({"user_id": 123})
```

## 3. 异步节点 vs 同步节点

### 3.1 性能对比

| 维度 | 同步节点 | 异步节点 |
|------|----------|----------|
| **执行方式** | 阻塞线程 | 非阻塞，事件循环 |
| **并发能力** | 需要多线程/多进程 | 单线程高并发 |
| **资源消耗** | 每个任务占用一个线程 | 共享事件循环 |
| **适用场景** | CPU 密集型 | I/O 密集型 |
| **API 调用** | 阻塞等待响应 | 并发多个请求 |

### 3.2 使用场景选择

**使用异步节点的场景**：
- API 调用（OpenAI、Anthropic 等）
- 数据库查询
- 文件 I/O 操作
- 网络请求
- 需要并发执行的任务

**使用同步节点的场景**：
- CPU 密集型计算
- 简单的数据转换
- 不涉及 I/O 的逻辑处理
- 使用不支持异步的第三方库

[来源: reference/source_节点函数约定_01.md]

### 3.3 代码对比示例

```python
import time
import asyncio
from typing_extensions import TypedDict

class State(TypedDict):
    results: list[str]

# 同步节点：串行执行，总耗时 3 秒
def sync_multi_api_call(state: State) -> dict:
    results = []
    for i in range(3):
        time.sleep(1)  # 模拟 API 调用
        results.append(f"Result {i}")
    return {"results": results}

# 异步节点：并发执行，总耗时 1 秒
async def async_multi_api_call(state: State) -> dict:
    async def fetch(i):
        await asyncio.sleep(1)  # 模拟 API 调用
        return f"Result {i}"
    
    results = await asyncio.gather(
        fetch(0), fetch(1), fetch(2)
    )
    return {"results": list(results)}
```

## 4. 异步生成器的问题

### 4.1 问题描述

社区发现使用 `async function*`（异步生成器）作为节点函数时存在问题：

```python
# ❌ 不推荐：异步生成器节点
async def async_generator_node(state: State):
    """这种写法可能导致函数体不执行"""
    async for item in some_async_generator():
        yield {"item": item}  # yield 可能失效
```

**问题表现**：
- 函数体不执行
- yield 流式输出失效
- 错误处理机制失效

[来源: reference/search_节点函数约定_01.md]

### 4.2 推荐的替代方案

使用普通异步函数 + 流式 API：

```python
# ✅ 推荐：普通异步函数
async def async_streaming_node(state: State) -> dict:
    """使用普通异步函数，通过 astream 实现流式"""
    results = []
    async for item in some_async_generator():
        results.append(item)
    return {"results": results}

# 或者使用 Command 对象控制流式输出
from langgraph.types import Command

async def async_node_with_command(state: State) -> Command:
    result = await some_async_api()
    return Command(
        update={"result": result},
        goto="next_node"
    )
```

[来源: reference/search_节点函数约定_01.md]

### 4.3 社区讨论

GitHub Issue #1158 指出：
> "StateGraph 中使用 async function* 作为节点时，函数体不执行且无 yield 流式，错误处理失效"

**官方建议**：
- 避免使用异步生成器作为节点函数
- 使用普通异步函数 + `astream()` 方法
- 如需流式输出，使用图级别的流式 API

[来源: reference/search_节点函数约定_01.md]

## 5. 异步错误处理

### 5.1 基础错误处理

异步节点中的错误处理与同步节点类似：

```python
async def async_node_with_error_handling(state: State) -> dict:
    try:
        result = await some_async_api()
        return {"result": result, "error": None}
    except Exception as e:
        return {"result": None, "error": str(e)}
```

### 5.2 CancelledError 特殊处理

异步任务取消时需要特殊处理：

```python
import asyncio

async def async_node_with_cancel_handling(state: State) -> dict:
    try:
        result = await some_async_api()
        return {"result": result}
    except asyncio.CancelledError:
        # 特殊处理取消错误
        print("Task cancelled")
        raise  # 重新抛出以确保正确取消
    except Exception as e:
        # 处理其他错误
        return {"error": str(e)}
```

**为什么需要特殊处理**：
- `CancelledError` 会绕过普通异常捕获
- 如果不重新抛出，可能导致状态不一致

[来源: reference/search_节点函数约定_01.md]

### 5.3 流式错误处理

在流式执行中处理异步节点错误：

```python
async def process_with_stream_error_handling():
    """在循环内处理异步节点错误"""
    try:
        async for event in graph.astream_events(input_data):
            try:
                # 处理事件
                process_event(event)
            except Exception as e:
                # 在循环内处理错误
                logger.error(f"Error in stream: {e}")
    except Exception as e:
        # 处理整体错误
        logger.error(f"Stream failed: {e}")
```

**注意事项**：
- try-catch 在 `for await` 循环外部可能失效
- 需要在循环内部处理异步节点错误

[来源: reference/search_节点函数约定_01.md]

## 6. 异步重试机制

### 6.1 RetryPolicy 配置

异步节点支持自动重试：

```python
from langgraph.types import RetryPolicy

async def unreliable_async_api(state: State) -> dict:
    """可能失败的异步 API 调用"""
    result = await some_unreliable_api()
    return {"result": result}

builder.add_node(
    "api_call",
    unreliable_async_api,
    retry_policy=RetryPolicy(
        max_attempts=3,
        initial_interval=1.0,
        backoff_factor=2.0,
        max_interval=10.0,
        jitter=True,
        retry_on=lambda e: isinstance(e, (ConnectionError, TimeoutError))
    )
)
```

[来源: reference/context7_langgraph_01.md]

### 6.2 异步重试实现

异步重试使用 `asyncio.sleep()` 而非 `time.sleep()`：

```python
# 源码片段（简化版）
async def arun_with_retry(task, retry_policy):
    attempts = 0
    while True:
        try:
            return await task.proc.ainvoke(task.input, config)
        except Exception as exc:
            if not should_retry(exc, retry_policy):
                raise
            
            attempts += 1
            if attempts >= retry_policy.max_attempts:
                raise
            
            # 计算退避时间
            interval = retry_policy.initial_interval
            interval = min(
                retry_policy.max_interval,
                interval * (retry_policy.backoff_factor ** (attempts - 1))
            )
            
            # 异步睡眠
            await asyncio.sleep(interval)
```

[来源: reference/source_节点函数约定_02.md]

## 7. 完整代码示例

### 7.1 基础异步节点

```python
import asyncio
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    query: str
    result: str
    error: str | None

async def fetch_data(state: State) -> dict:
    """异步获取数据"""
    query = state["query"]
    
    # 模拟异步 API 调用
    await asyncio.sleep(1)
    result = f"Data for: {query}"
    
    return {"result": result, "error": None}

# 构建图
builder = StateGraph(State)
builder.add_node("fetch", fetch_data)
builder.add_edge(START, "fetch")
builder.add_edge("fetch", END)

graph = builder.compile()

# 异步执行
async def main():
    result = await graph.ainvoke({"query": "test", "result": "", "error": None})
    print(result)

# 运行
asyncio.run(main())
```

### 7.2 并发异步节点

```python
import asyncio
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    user_id: int
    user_data: dict | None
    orders: list | None
    products: list | None

async def fetch_user(state: State) -> dict:
    """获取用户信息"""
    await asyncio.sleep(1)
    return {"user_data": {"id": state["user_id"], "name": "Alice"}}

async def fetch_orders(state: State) -> dict:
    """获取订单信息"""
    await asyncio.sleep(1)
    return {"orders": [{"id": 1}, {"id": 2}]}

async def fetch_products(state: State) -> dict:
    """获取产品信息"""
    await asyncio.sleep(1)
    return {"products": [{"id": 101}, {"id": 102}]}

async def merge_results(state: State) -> dict:
    """合并结果"""
    return {
        "user_data": state["user_data"],
        "orders": state["orders"],
        "products": state["products"]
    }

# 构建图
builder = StateGraph(State)
builder.add_node("fetch_user", fetch_user)
builder.add_node("fetch_orders", fetch_orders)
builder.add_node("fetch_products", fetch_products)
builder.add_node("merge", merge_results)

# 并发执行三个 fetch 节点
builder.add_edge(START, "fetch_user")
builder.add_edge(START, "fetch_orders")
builder.add_edge(START, "fetch_products")

# 等待所有 fetch 完成后合并
builder.add_edge("fetch_user", "merge")
builder.add_edge("fetch_orders", "merge")
builder.add_edge("fetch_products", "merge")
builder.add_edge("merge", END)

graph = builder.compile()

# 执行（三个 fetch 并发，总耗时约 1 秒）
async def main():
    result = await graph.ainvoke({
        "user_id": 123,
        "user_data": None,
        "orders": None,
        "products": None
    })
    print(result)

asyncio.run(main())
```

### 7.3 带错误处理和重试的异步节点

```python
import asyncio
import random
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy

class State(TypedDict):
    query: str
    result: str | None
    error: str | None
    attempts: int

async def unreliable_api_call(state: State) -> dict:
    """模拟不稳定的 API 调用"""
    attempts = state.get("attempts", 0) + 1
    
    try:
        # 50% 概率失败
        if random.random() < 0.5:
            raise ConnectionError("API connection failed")
        
        # 模拟 API 调用
        await asyncio.sleep(0.5)
        result = f"Success after {attempts} attempts"
        
        return {
            "result": result,
            "error": None,
            "attempts": attempts
        }
    except asyncio.CancelledError:
        # 特殊处理取消错误
        print("Task cancelled")
        raise
    except Exception as e:
        # 记录错误但不返回（让 RetryPolicy 处理）
        print(f"Attempt {attempts} failed: {e}")
        raise

# 自定义重试条件
def should_retry(error: Exception) -> bool:
    return isinstance(error, (ConnectionError, TimeoutError))

# 构建图
builder = StateGraph(State)
builder.add_node(
    "api_call",
    unreliable_api_call,
    retry_policy=RetryPolicy(
        max_attempts=5,
        initial_interval=0.5,
        backoff_factor=2.0,
        max_interval=10.0,
        jitter=True,
        retry_on=should_retry
    )
)
builder.add_edge(START, "api_call")
builder.add_edge("api_call", END)

graph = builder.compile()

# 执行
async def main():
    result = await graph.ainvoke({
        "query": "test",
        "result": None,
        "error": None,
        "attempts": 0
    })
    print(f"Final result: {result}")

asyncio.run(main())
```

## 8. 最佳实践

### 8.1 异步节点设计原则

1. **使用普通异步函数**
   - ✅ `async def node(state) -> dict`
   - ❌ `async def node(state)` with `yield`

2. **正确处理 CancelledError**
   ```python
   try:
       result = await api_call()
   except asyncio.CancelledError:
       cleanup()
       raise  # 必须重新抛出
   ```

3. **配置合理的重试策略**
   ```python
   RetryPolicy(
       max_attempts=3,
       initial_interval=1.0,
       backoff_factor=2.0,
       jitter=True
   )
   ```

4. **使用 asyncio.gather() 实现并发**
   ```python
   results = await asyncio.gather(
       api_call_1(),
       api_call_2(),
       api_call_3()
   )
   ```

### 8.2 性能优化建议

1. **批量 API 调用**
   ```python
   async def batch_api_calls(state: State) -> dict:
       tasks = [api_call(item) for item in state["items"]]
       results = await asyncio.gather(*tasks)
       return {"results": results}
   ```

2. **使用连接池**
   ```python
   from aiohttp import ClientSession
   
   async def fetch_with_session(state: State) -> dict:
       async with ClientSession() as session:
           async with session.get(url) as response:
               data = await response.json()
       return {"data": data}
   ```

3. **设置超时**
   ```python
   async def api_call_with_timeout(state: State) -> dict:
       try:
           result = await asyncio.wait_for(
               some_async_api(),
               timeout=5.0
           )
           return {"result": result}
       except asyncio.TimeoutError:
           return {"error": "Timeout"}
   ```

### 8.3 常见陷阱

1. **❌ 在异步节点中使用同步阻塞调用**
   ```python
   async def bad_node(state: State) -> dict:
       time.sleep(1)  # ❌ 阻塞事件循环
       return {"result": "done"}
   ```

2. **❌ 忘记 await**
   ```python
   async def bad_node(state: State) -> dict:
       result = some_async_api()  # ❌ 忘记 await
       return {"result": result}  # result 是 coroutine 对象
   ```

3. **❌ 在流式循环外捕获错误**
   ```python
   # ❌ 错误的错误处理
   try:
       async for event in graph.astream(input):
           process(event)
   except Exception as e:
       handle_error(e)  # 可能捕获不到
   
   # ✅ 正确的错误处理
   async for event in graph.astream(input):
       try:
           process(event)
       except Exception as e:
           handle_error(e)
   ```

## 9. 总结

### 9.1 核心要点

1. **异步节点定义**：使用 `async def` 定义，支持 State、Config、Runtime 参数
2. **自动检测**：LangGraph 自动检测并处理异步/同步节点
3. **并发执行**：异步节点支持高效并发，适合 I/O 密集型任务
4. **避免生成器**：不要使用 `async function*`，使用普通异步函数
5. **错误处理**：正确处理 CancelledError，在流式循环内捕获错误
6. **重试机制**：配置 RetryPolicy 实现自动重试

### 9.2 选择指南

| 场景 | 推荐方案 |
|------|----------|
| API 调用 | 异步节点 + RetryPolicy |
| 数据库查询 | 异步节点 + 连接池 |
| 文件 I/O | 异步节点 + aiofiles |
| CPU 密集型 | 同步节点 + 线程池 |
| 简单逻辑 | 同步节点 |
| 需要并发 | 异步节点 + asyncio.gather() |
