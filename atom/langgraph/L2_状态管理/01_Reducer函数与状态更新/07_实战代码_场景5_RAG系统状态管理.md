## 场景描述

### 业务需求

构建一个智能 RAG（检索增强生成）文档问答系统，需要：
1. **文档检索**：从向量数据库检索相关文档
2. **查询改写**：优化用户查询以提高检索质量
3. **文档评分**：评估检索文档的相关性
4. **答案生成**：基于相关文档生成答案
5. **控制流管理**：根据文档质量决定是否需要网络搜索

### 技术挑战

- 如何管理多个处理阶段的状态？
- 如何累积检索到的文档？
- 如何实现条件路由（文档不足时触发网络搜索）？
- 如何保持查询的演化历史？

### 解决方案

使用 LangGraph 的多种 Reducer 组合：
- **add_messages**：管理对话历史
- **operator.add**：累积文档列表
- **部分状态更新**：只返回需要更新的字段
- **控制流标志**：决定工作流路径

```python
"""
状态定义模块
使用多种 Reducer 管理 RAG 系统状态
"""
from typing import Annotated, TypedDict, List
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages
import operator
from rag_system.models import Document


class RAGState(TypedDict):
    """
    RAG 系统状态

    Attributes:
        messages: 对话历史，使用 add_messages Reducer
        question: 当前问题（会被查询改写更新）
        documents: 检索到的文档列表，使用 operator.add 累积
        generation: 生成的答案
        web_search: 是否需要网络搜索的标志
    """
    messages: Annotated[List[BaseMessage], add_messages]
    question: str
    documents: Annotated[List[Document], operator.add]
    generation: str
    web_search: str  # "Yes" 或 "No"


class SelfRAGState(TypedDict):
    """
    Self-RAG 状态（带自我反思）

    额外字段：
        iterations: 迭代次数
        quality_score: 答案质量分数
    """
    messages: Annotated[List[BaseMessage], add_messages]
    question: str
    documents: Annotated[List[Document], operator.add]
    generation: str
    web_search: str
    iterations: int
    quality_score: float


class CRAGState(TypedDict):
    """
    CRAG 状态（纠正式 RAG）

    额外字段：
        original_question: 原始问题
        rewritten_question: 改写后的问题
        document_relevance: 文档相关性评分
    """
    messages: Annotated[List[BaseMessage], add_messages]
    question: str
    original_question: str
    documents: Annotated[List[Document], operator.add]
    generation: str
    web_search: str
    document_relevance: List[str]  # ["yes", "no", "yes", ...]
```

### 4. 图构建 (rag_system/graph.py)

```python
"""
图构建模块
定义 RAG 系统的工作流
"""
from langgraph.graph import StateGraph, START, END
from rag_system.state import RAGState, CRAGState
from rag_system.nodes import (
    retrieve_documents_node,
    grade_documents_node,
    transform_query_node,
    web_search_node,
    generate_answer_node,
    check_relevance_node
)


def create_basic_rag_graph():
    """
    创建基础 RAG 图

    工作流：
    START -> retrieve -> generate -> END
    """
    builder = StateGraph(RAGState)

    # 添加节点
    builder.add_node("retrieve", retrieve_documents_node)
    builder.add_node("generate", generate_answer_node)

    # 添加边
    builder.add_edge(START, "retrieve")
    builder.add_edge("retrieve", "generate")
    builder.add_edge("generate", END)

    # 编译
    graph = builder.compile()

    return graph


def create_self_rag_graph():
    """
    创建 Self-RAG 图（带文档评分）

    工作流：
    START -> retrieve -> grade -> [generate | transform]
    - 如果文档相关 -> generate -> END
    - 如果文档不相关 -> transform -> retrieve
    """
    builder = StateGraph(RAGState)

    # 添加节点
    builder.add_node("retrieve", retrieve_documents_node)
    builder.add_node("grade", grade_documents_node)
    builder.add_node("transform", transform_query_node)
    builder.add_node("generate", generate_answer_node)

    # 添加边
    builder.add_edge(START, "retrieve")
    builder.add_edge("retrieve", "grade")

    # 条件路由
    def decide_to_generate(state: RAGState) -> str:
        """决定是生成答案还是改写查询"""
        web_search = state.get("web_search", "No")

        if web_search == "Yes":
            print("---DECISION: TRANSFORM QUERY---")
            return "transform"
        else:
            print("---DECISION: GENERATE---")
            return "generate"

    builder.add_conditional_edges(
        "grade",
        decide_to_generate,
        {
            "transform": "transform",
            "generate": "generate"
        }
    )

    builder.add_edge("transform", "retrieve")
    builder.add_edge("generate", END)

    # 编译
    graph = builder.compile()

    return graph


def create_crag_graph():
    """
    创建 CRAG 图（纠正式 RAG，带网络搜索）

    工作流：
    START -> retrieve -> grade -> [generate | web_search]
    - 如果文档相关 -> generate -> END
    - 如果文档不相关 -> web_search -> generate -> END
    """
    builder = StateGraph(RAGState)

    # 添加节点
    builder.add_node("retrieve", retrieve_documents_node)
    builder.add_node("grade", grade_documents_node)
    builder.add_node("web_search", web_search_node)
    builder.add_node("generate", generate_answer_node)

    # 添加边
    builder.add_edge(START, "retrieve")
    builder.add_edge("retrieve", "grade")

    # 条件路由
    def decide_to_search(state: RAGState) -> str:
        """决定是直接生成还是先网络搜索"""
        web_search = state.get("web_search", "No")

        if web_search == "Yes":
            print("---DECISION: WEB SEARCH---")
            return "web_search"
        else:
            print("---DECISION: GENERATE---")
            return "generate"

    builder.add_conditional_edges(
        "grade",
        decide_to_search,
        {
            "web_search": "web_search",
            "generate": "generate"
        }
    )

    builder.add_edge("web_search", "generate")
    builder.add_edge("generate", END)

    # 编译
    graph = builder.compile()

    return graph
```

## 常见问题与解决方案

### 问题 1: 文档重复累积

**问题描述**：
使用 `operator.add` 时，文档会不断累积，可能导致重复。

**解决方案**：
```python
def deduplicate_documents(documents: List[Document]) -> List[Document]:
    """去重文档"""
    seen = set()
    unique_docs = []

    for doc in documents:
        # 使用内容哈希去重
        content_hash = hash(doc.page_content)
        if content_hash not in seen:
            seen.add(content_hash)
            unique_docs.append(doc)

    return unique_docs

# 在节点中使用
def retrieve_node(state: RAGState) -> dict:
    documents = state["documents"]
    new_docs = fetch_documents()

    # 去重
    all_docs = deduplicate_documents(documents + new_docs)

    return {"documents": all_docs}
```

### 问题 2: 状态字段冲突

**问题描述**：
多个节点可能更新相同的字段，导致覆盖。

**解决方案**：
```python
# 使用不同的字段名
class RAGState(TypedDict):
    original_question: str      # 原始问题
    current_question: str       # 当前问题
    rewritten_question: str     # 改写后的问题

# 或者使用历史列表
class RAGState(TypedDict):
    question_history: Annotated[List[str], operator.add]
```

### 问题 3: 控制流标志不生效

**问题描述**：
条件路由的标志没有正确设置。

**解决方案**：
```python

        chunks.append(chunk)
        # 实时返回部分结果

    return {"generation": "".join(chunks)}
```

## 总结

本示例展示了如何使用 LangGraph 构建完整的 RAG 系统：

1. **多 Reducer 组合**：`add_messages` + `operator.add`
