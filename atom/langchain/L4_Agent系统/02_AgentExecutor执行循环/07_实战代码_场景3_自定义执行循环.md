# 实战代码 场景3：自定义执行循环

> 不用 AgentExecutor，手写一个 ReAct 循环——拆开黑盒看清 Think→Act→Observe 到底怎么转的

---

## 场景概述

AgentExecutor 帮你封装了执行循环，但"会用"和"理解"是两回事。本文从零手写一个 ReAct 循环，让你亲手实现：

- LLM 决策（Think）→ 工具执行（Act）→ 结果反馈（Observe）的完整闭环
- 循环终止条件判断
- 错误处理与工具查找
- 中间步骤记录

写完之后，你会发现 AgentExecutor 的源码不再神秘——它就是你写的这个循环加上一堆生产级的防护措施。

## 示例1：最简手写 ReAct 循环

**解决的问题：** 用最少的代码实现一个完整的 Agent 执行循环，理解 AgentExecutor 的核心机制。

```python
"""
手写 ReAct 执行循环（最简版）
演示：不使用 AgentExecutor，手动实现 Think → Act → Observe 循环
"""
from dotenv import load_dotenv
load_dotenv()

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, ToolMessage
from langchain_core.tools import tool


# ===== 1. 定义工具 =====
@tool
def get_weather(city: str) -> str:
    """获取城市天气信息"""
    weather_data = {
        "北京": "晴天 25°C",
        "上海": "多云 22°C",
        "深圳": "阵雨 28°C",
    }
    return weather_data.get(city, f"未找到 {city} 的天气数据")


@tool
def get_population(city: str) -> str:
    """获取城市人口信息"""
    pop_data = {
        "北京": "2189万",
        "上海": "2487万",
        "深圳": "1756万",
    }
    return pop_data.get(city, f"未找到 {city} 的人口数据")


tools = [get_weather, get_population]
tool_map = {t.name: t for t in tools}


# ===== 2. 手写 ReAct 循环 =====
def custom_react_loop(query: str, max_iterations: int = 10):
    """
    手写的 ReAct 执行循环。
    对应 AgentExecutor._call() 的核心逻辑。
    """
    # 绑定工具到 LLM（让 LLM 知道有哪些工具可用）
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0).bind_tools(tools)

    # 初始化消息列表（对应 AgentExecutor 的 inputs）
    messages = [HumanMessage(content=query)]

    # 中间步骤记录（对应 intermediate_steps）
    intermediate_steps = []

    for i in range(max_iterations):
        print(f"\n{'='*50}")
        print(f"迭代 {i + 1}")
        print(f"{'='*50}")

        # ---- Step 1: Think ----
        # 调用 LLM，让它决定下一步做什么
        # 对应 AgentExecutor 中的 agent.plan()
        response = llm.invoke(messages)
        messages.append(response)

        # ---- Step 2: Check ----
        # 检查 LLM 是否决定结束（没有 tool_calls = 任务完成）
        # 对应 AgentExecutor 中的 isinstance(output, AgentFinish)
        if not response.tool_calls:
            print(f"\n[完成] Agent 最终回答:")
            print(f"  {response.content}")
            return {
                "output": response.content,
                "intermediate_steps": intermediate_steps,
            }

        # ---- Step 3: Act ----
        # 执行 LLM 选择的工具
        # 对应 AgentExecutor 中的 _perform_agent_action()
        for tool_call in response.tool_calls:
            tool_name = tool_call["name"]
            tool_input = tool_call["args"]
            print(f"\n[Act] 调用工具: {tool_name}")
            print(f"  输入: {tool_input}")

            # 查找并执行工具
            if tool_name in tool_map:
                observation = tool_map[tool_name].invoke(tool_input)
            else:
                # 对应 AgentExecutor 中的 InvalidTool
                observation = f"错误: 工具 '{tool_name}' 不存在。可用工具: {list(tool_map.keys())}"

            print(f"[Observe] 结果: {observation}")

            # ---- Step 4: Observe ----
            # 将工具结果反馈给 LLM（作为 ToolMessage）
            # 对应 intermediate_steps.append(AgentStep(action, observation))
            messages.append(
                ToolMessage(
                    content=str(observation),
                    tool_call_id=tool_call["id"],
                )
            )
            intermediate_steps.append({
                "tool": tool_name,
                "input": tool_input,
                "observation": observation,
            })

    # 超出最大迭代次数（对应 early_stopping_method="force"）
    print(f"\n[警告] 达到最大迭代次数 {max_iterations}")
    return {
        "output": "Agent stopped due to iteration limit.",
        "intermediate_steps": intermediate_steps,
    }


# ===== 3. 运行 =====
if __name__ == "__main__":
    # 测试1：单工具调用
    print("=" * 60)
    print("测试1：单工具调用")
    print("=" * 60)
    result = custom_react_loop("北京今天天气怎么样？")
    print(f"\n最终输出: {result['output']}")
    print(f"执行步骤数: {len(result['intermediate_steps'])}")

    # 测试2：多工具调用
    print("\n\n" + "=" * 60)
    print("测试2：多工具调用")
    print("=" * 60)
    result = custom_react_loop("告诉我上海的天气和人口")
    print(f"\n最终输出: {result['output']}")
    print(f"执行步骤数: {len(result['intermediate_steps'])}")
```

> **关键洞察：** 整个循环的核心就 4 步——调用 LLM、检查是否结束、执行工具、反馈结果。
> AgentExecutor 的 `_call` 方法做的事情和这里完全一样，只是多了错误处理、回调、日志等生产级功能。

## 示例2：加入错误处理和执行时间限制

**解决的问题：** 理解 AgentExecutor 的 `handle_parsing_errors` 和 `max_execution_time` 是怎么实现的。

```python
"""
带错误处理的 ReAct 循环
演示：手动实现 AgentExecutor 的错误处理和时间限制机制
"""
import time
from dotenv import load_dotenv
load_dotenv()

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, ToolMessage
from langchain_core.tools import tool


@tool
def search_docs(query: str) -> str:
    """搜索知识库文档"""
    docs = {
        "RAG": "RAG 通过检索外部知识增强 LLM 生成，核心流程：索引→检索→生成。",
        "Embedding": "Embedding 将文本映射到向量空间，语义相近的文本距离更近。",
        "Chunking": "文本分块常见策略：固定大小、语义分块、递归分块。",
    }
    for key, value in docs.items():
        if key.lower() in query.lower():
            return value
    return f"未找到与 '{query}' 相关的文档"


@tool
def calculate(expression: str) -> str:
    """计算数学表达式"""
    try:
        result = eval(expression, {"__builtins__": {}})
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


tools = [search_docs, calculate]
tool_map = {t.name: t for t in tools}


def robust_react_loop(
    query: str,
    max_iterations: int = 10,
    max_execution_time: float | None = None,
    handle_errors: bool = True,
    verbose: bool = True,
):
    """
    带错误处理和时间限制的 ReAct 循环。
    对应 AgentExecutor 的完整 _call() 逻辑。

    参数对照：
      max_iterations     → AgentExecutor.max_iterations
      max_execution_time → AgentExecutor.max_execution_time
      handle_errors      → AgentExecutor.handle_parsing_errors
      verbose            → AgentExecutor.verbose
    """
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0).bind_tools(tools)
    messages = [HumanMessage(content=query)]
    intermediate_steps = []
    iterations = 0
    start_time = time.time()

    def _should_continue() -> bool:
        """
        对应 AgentExecutor._should_continue()
        检查迭代次数和执行时间两个条件（OR 关系）
        """
        if iterations >= max_iterations:
            if verbose:
                print(f"[限制] 达到最大迭代次数 {max_iterations}")
            return False
        if max_execution_time is not None:
            elapsed = time.time() - start_time
            if elapsed >= max_execution_time:
                if verbose:
                    print(f"[限制] 达到最大执行时间 {max_execution_time}s")
                return False
        return True

    while _should_continue():
        iterations += 1
        if verbose:
            elapsed = time.time() - start_time
            print(f"\n--- 迭代 {iterations} (已用时 {elapsed:.1f}s) ---")

        try:
            # Think: 调用 LLM
            response = llm.invoke(messages)
            messages.append(response)

        except Exception as e:
            # 对应 handle_parsing_errors 的处理逻辑
            if handle_errors:
                error_msg = f"LLM 调用出错: {str(e)[:200]}。请重试。"
                if verbose:
                    print(f"[错误处理] {error_msg}")
                # 将错误信息作为 observation 反馈给 LLM
                messages.append(HumanMessage(content=error_msg))
                continue
            else:
                raise

        # Check: 是否结束
        if not response.tool_calls:
            if verbose:
                print(f"[完成] {response.content[:100]}...")
            return {
                "output": response.content,
                "intermediate_steps": intermediate_steps,
                "iterations": iterations,
                "time_elapsed": time.time() - start_time,
            }

        # Act + Observe: 执行工具并反馈
        for tool_call in response.tool_calls:
            tool_name = tool_call["name"]
            tool_input = tool_call["args"]

            try:
                if tool_name in tool_map:
                    observation = tool_map[tool_name].invoke(tool_input)
                else:
                    observation = (
                        f"工具 '{tool_name}' 不存在。"
                        f"可用工具: {list(tool_map.keys())}"
                    )
                    if verbose:
                        print(f"[InvalidTool] {observation}")
            except Exception as e:
                # 对应 AgentExecutor 中工具执行异常的处理
                if handle_errors:
                    observation = f"工具执行出错: {str(e)[:200]}"
                    if verbose:
                        print(f"[工具错误] {observation}")
                else:
                    raise

            if verbose:
                print(f"  工具: {tool_name}({tool_input})")
                print(f"  结果: {observation}")

            messages.append(
                ToolMessage(content=str(observation), tool_call_id=tool_call["id"])
            )
            intermediate_steps.append({
                "tool": tool_name,
                "input": tool_input,
                "observation": observation,
            })

    # Early stopping（对应 early_stopping_method="force"）
    return {
        "output": "Agent stopped due to iteration limit or time limit.",
        "intermediate_steps": intermediate_steps,
        "iterations": iterations,
        "time_elapsed": time.time() - start_time,
    }


if __name__ == "__main__":
    # 测试：正常多步推理
    result = robust_react_loop(
        "什么是 RAG？它和 Embedding 有什么关系？",
        max_iterations=10,
        max_execution_time=30.0,
        verbose=True,
    )
    print(f"\n最终输出: {result['output']}")
    print(f"迭代次数: {result['iterations']}")
    print(f"耗时: {result['time_elapsed']:.1f}s")
```

## 示例3：带中间步骤裁剪的循环

**解决的问题：** 理解 AgentExecutor 的 `trim_intermediate_steps` 参数——当对话太长时如何裁剪历史。

```python
"""
带中间步骤裁剪的 ReAct 循环
演示：手动实现 AgentExecutor 的 trim_intermediate_steps 机制

当 Agent 执行很多步时，消息列表会越来越长，可能超出 Context Window。
AgentExecutor 通过 trim_intermediate_steps 参数解决这个问题。
"""
from dotenv import load_dotenv
load_dotenv()

from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, AIMessage, ToolMessage
from langchain_core.tools import tool


@tool
def search(query: str) -> str:
    """搜索信息"""
    return f"关于 '{query}' 的搜索结果：这是一段模拟的搜索内容..."


tools = [search]
tool_map = {t.name: t for t in tools}


def trim_messages(messages: list, keep_last_n: int = 6) -> list:
    """
    裁剪消息列表，只保留系统消息 + 第一条用户消息 + 最近 N 条消息。

    对应 AgentExecutor 的 trim_intermediate_steps 参数：
      - trim_intermediate_steps=5  → 只保留最近 5 步
      - trim_intermediate_steps=fn → 自定义裁剪函数
    """
    if len(messages) <= keep_last_n + 1:
        return messages

    # 保留第一条消息（用户原始问题）+ 最近 N 条
    trimmed = [messages[0]] + messages[-keep_last_n:]
    print(f"  [裁剪] 消息从 {len(messages)} 条裁剪到 {len(trimmed)} 条")
    return trimmed


def react_with_trimming(
    query: str,
    max_iterations: int = 10,
    trim_after: int = 6,
):
    """带消息裁剪的 ReAct 循环"""
    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0).bind_tools(tools)
    messages = [HumanMessage(content=query)]
    intermediate_steps = []

    for i in range(max_iterations):
        print(f"\n--- 迭代 {i + 1} (消息数: {len(messages)}) ---")

        # 裁剪过长的消息列表
        working_messages = trim_messages(messages, keep_last_n=trim_after)

        # Think
        response = llm.invoke(working_messages)
        messages.append(response)

        # Check
        if not response.tool_calls:
            return {
                "output": response.content,
                "intermediate_steps": intermediate_steps,
                "total_messages": len(messages),
            }

        # Act + Observe
        for tc in response.tool_calls:
            obs = tool_map[tc["name"]].invoke(tc["args"]) if tc["name"] in tool_map else "工具不存在"
            print(f"  工具: {tc['name']}({tc['args']}) → {obs[:60]}...")
            messages.append(ToolMessage(content=str(obs), tool_call_id=tc["id"]))
            intermediate_steps.append({"tool": tc["name"], "observation": obs})

    return {"output": "达到迭代限制", "intermediate_steps": intermediate_steps}


if __name__ == "__main__":
    result = react_with_trimming(
        "帮我分别搜索 Python、Java、Go 三种语言的特点，然后做个对比总结",
        max_iterations=10,
        trim_after=6,
    )
    print(f"\n最终输出: {result['output'][:200]}...")
    print(f"总消息数: {result['total_messages']}")
```

### 核心差异

| 方面 | 我们的手写版 | AgentExecutor |
|------|-------------|---------------|
| 消息格式 | 直接用 Messages API | 通过 agent_scratchpad 注入 |
| 错误处理 | 简单 try/except | OutputParserException + InvalidTool + ExceptionTool |
| 工具未找到 | 返回错误字符串 | 使用 InvalidTool 类，格式化错误信息 |
| 回调系统 | 无 | 完整的 CallbackManager 支持 |
| 异步支持 | 无 | `_acall` + `asyncio.gather` 并发执行工具 |
| 迭代器模式 | 无 | AgentExecutorIterator 支持逐步 yield |
| 时间限制 | 简单计时 | `asyncio_timeout` 包装 |

## 从手写循环中学到的关键洞察

### 洞察1：循环的本质是"LLM 驱动的 while"

```
传统编程的 while 循环：程序员写终止条件
Agent 的 while 循环：LLM 决定何时终止（不返回 tool_calls = 结束）
```

这就是 Agent 的"自主性"来源——循环何时结束不是硬编码的，而是 LLM 根据上下文判断的。

### 洞察2：中间步骤就是 Agent 的"工作记忆"

每次迭代，LLM 都能看到之前所有步骤的结果。这就像一个人在做笔记：

```
第1步：我搜索了 X，得到了 A
第2步：我搜索了 Y，得到了 B
第3步：综合 A 和 B，我可以回答了
```

没有中间步骤，Agent 就是"金鱼记忆"——每次迭代都忘了之前做过什么。

### 洞察3：安全限制是必须的

没有 `max_iterations`，一个"犹豫不决"的 LLM 可能永远在循环里转圈。这不是理论问题——在生产环境中，LLM 确实会出现反复调用同一个工具的情况。

## 实战建议

| 场景 | 建议 |
|------|------|
| 学习理解 | 用手写循环，逐步调试 |
| 快速原型 | 用 AgentExecutor，开箱即用 |
| 生产环境 | 用 create_agent（LangGraph），更灵活可控 |
| 需要自定义循环逻辑 | 基于手写循环扩展，或用 LangGraph 自定义节点 |
