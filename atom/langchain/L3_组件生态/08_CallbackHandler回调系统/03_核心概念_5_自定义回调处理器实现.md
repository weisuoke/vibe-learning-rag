# 自定义回调处理器实现

## 1. 【30字核心】

**自定义回调处理器通过继承 BaseCallbackHandler 并重写特定方法，实现日志记录、成本追踪、可观测性集成等自定义功能。**

## 2. 【第一性原理】

### 什么是第一性原理？

**第一性原理**：回到事物最基本的真理，从源头思考问题

### 自定义回调处理器的第一性原理

#### 1. 最基础的定义

**自定义回调处理器 = 继承 BaseCallbackHandler + 重写特定方法**

仅此而已！没有更基础的了。

#### 2. 为什么需要自定义回调处理器？

**核心问题：内置回调处理器功能有限，无法满足所有需求**

想象你在构建 RAG 系统：
- 需要记录每次 LLM 调用的详细日志
- 需要追踪 token 用量和成本
- 需要集成第三方监控平台（Langfuse、DataDog）
- 需要实现自定义的流式输出逻辑

内置回调处理器（如 `StdOutCallbackHandler`）只能打印到控制台，无法满足这些需求。

#### 3. 自定义回调处理器的三层价值

##### 价值1：业务定制

**根据业务需求定制监控逻辑**

```python
class CustomLogHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        # 记录到数据库
        db.insert_log({
            "type": "llm_start",
            "prompt": prompts[0],
            "timestamp": time.time()
        })
```

##### 价值2：成本控制

**实时追踪和控制 API 成本**

```python
class CostLimitHandler(BaseCallbackHandler):
    def __init__(self, max_cost=10.0):
        self.total_cost = 0
        self.max_cost = max_cost

    def on_llm_end(self, response, **kwargs):
        usage = response.llm_output.get('token_usage', {})
        cost = self.calculate_cost(usage)
        self.total_cost += cost

        if self.total_cost > self.max_cost:
            raise Exception(f"成本超限: ${self.total_cost}")
```

##### 价值3：可观测性集成

**与第三方监控平台无缝集成**

```python
class LangfuseHandler(BaseCallbackHandler):
    def __init__(self, api_key):
        self.client = Langfuse(api_key=api_key)

    def on_llm_start(self, serialized, prompts, **kwargs):
        self.client.trace_start(prompts[0])

    def on_llm_end(self, response, **kwargs):
        self.client.trace_end(response)
```

## 3. 【核心概念】

### 核心概念1：继承 BaseCallbackHandler

**自定义回调处理器的基础是继承 BaseCallbackHandler 类**

```python
from langchain_core.callbacks import BaseCallbackHandler

class MyHandler(BaseCallbackHandler):
    """自定义回调处理器"""

    def __init__(self):
        super().__init__()
        # 初始化自定义属性
        self.logs = []

    # 重写特定方法
    def on_llm_start(self, serialized, prompts, **kwargs):
        self.logs.append(f"LLM 开始: {prompts[0]}")

    def on_llm_end(self, response, **kwargs):
        self.logs.append(f"LLM 结束: {response}")
```

**在 RAG 开发中：**
- 创建自定义日志记录器
- 实现成本追踪逻辑
- 集成第三方监控平台

---

### 核心概念2：重写特定回调方法

**根据需求选择性重写回调方法，不需要全部实现**

```python
class TokenCostHandler(BaseCallbackHandler):
    """只关注 token 用量的回调处理器"""

    def __init__(self):
        self.total_tokens = 0
        self.total_cost = 0

    # 只重写 on_llm_end
    def on_llm_end(self, response, **kwargs):
        usage = response.llm_output.get('token_usage', {})
        tokens = usage.get('total_tokens', 0)
        self.total_tokens += tokens

        # 假设 $0.002 per 1K tokens
        self.total_cost += (tokens / 1000) * 0.002

        print(f"累计 tokens: {self.total_tokens}, 成本: ${self.total_cost:.4f}")
```

**常用的回调方法：**
- `on_llm_start` - 记录输入
- `on_llm_end` - 统计 token 和成本
- `on_llm_error` - 错误处理和告警
- `on_llm_new_token` - 流式输出处理

**在 RAG 开发中：**
- 只重写需要的方法，保持代码简洁
- 避免不必要的性能开销

---

### 核心概念3：传递回调处理器

**回调处理器可以在初始化时或运行时传入**

```python
# 方式1：初始化时传入
handler = MyHandler()
llm = ChatOpenAI(callbacks=[handler])
llm.invoke("Hello")

# 方式2：运行时传入（更灵活）
llm = ChatOpenAI()
llm.invoke("Hello", config={"callbacks": [handler]})

# 方式3：多个回调处理器组合
handlers = [
    LogHandler(),
    CostHandler(),
    MetricsHandler()
]
llm = ChatOpenAI(callbacks=handlers)
```

**在 RAG 开发中：**
- 运行时传入更灵活，适合动态配置
- 多个回调处理器组合，分离关注点
- 每个处理器负责一个功能（日志、成本、指标）

## 6. 【反直觉点】

### 误区1：回调处理器可以在并发环境中重用 ❌

**为什么错？**
- 回调处理器可能包含状态（如 `self.total_tokens`）
- 并发请求会导致状态冲突
- trace_id 混乱

**为什么人们容易这样错？**
因为在单线程环境中，重用回调处理器没有问题，所以人们以为在并发环境中也可以重用。

**正确理解：**
```python
# ❌ 错误：重用回调处理器
handler = CostTracker()
for query in queries:
    llm.invoke(query, callbacks=[handler])  # 并发时会冲突

# ✅ 正确：每个请求新建实例
for query in queries:
    handler = CostTracker()
    llm.invoke(query, callbacks=[handler])
```

---

### 误区2：异步回调处理器可以使用普通方法 ❌

**为什么错？**
- 异步回调处理器需要使用 `AsyncCallbackHandler`
- 所有方法都必须是 `async def`
- 否则会阻塞异步执行

**为什么人们容易这样错？**
因为在同步环境中，普通方法可以正常工作，所以人们以为在异步环境中也可以使用普通方法。

**正确理解：**
```python
# ❌ 错误：异步环境使用同步方法
class MyHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        time.sleep(1)  # 会阻塞

# ✅ 正确：使用 AsyncCallbackHandler
from langchain_core.callbacks import AsyncCallbackHandler

class MyAsyncHandler(AsyncCallbackHandler):
    async def on_llm_start(self, serialized, prompts, **kwargs):
        await asyncio.sleep(1)  # 不会阻塞
```

---

### 误区3：回调方法的返回值会被使用 ❌

**为什么错？**
- 回调方法的返回值会被忽略
- 回调方法只用于副作用（日志、统计等）
- 不能通过返回值影响执行流程

**为什么人们容易这样错？**
因为在普通函数中，返回值通常会被使用，所以人们以为回调方法的返回值也会被使用。

**正确理解：**
```python
# ❌ 错误：期望返回值被使用
class MyHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        return "modified_prompt"  # 返回值会被忽略

# ✅ 正确：使用副作用
class MyHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        self.logs.append(prompts[0])  # 副作用
```

## 7. 【实战代码】

```python
"""
自定义回调处理器实战示例
演示：日志记录 + 成本追踪 + 错误处理
"""

import time
from typing import Any
from langchain_core.callbacks import BaseCallbackHandler
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage

# ===== 1. 完整的自定义回调处理器 =====
class ProductionHandler(BaseCallbackHandler):
    """生产级回调处理器：日志 + 成本 + 错误处理"""

    def __init__(self, log_file="llm_calls.log"):
        self.log_file = log_file
        self.start_time = None
        self.total_tokens = 0
        self.total_cost = 0
        self.error_count = 0

    def on_llm_start(self, serialized: dict, prompts: list[str], **kwargs: Any) -> None:
        """LLM 开始时触发"""
        self.start_time = time.time()
        log_entry = f"[{time.strftime('%Y-%m-%d %H:%M:%S')}] LLM 开始\n"
        log_entry += f"Prompt: {prompts[0][:100]}...\n"
        log_entry += f"run_id: {kwargs.get('run_id')}\n"

        with open(self.log_file, "a") as f:
            f.write(log_entry)

    def on_llm_end(self, response, **kwargs: Any) -> None:
        """LLM 结束时触发"""
        elapsed = time.time() - self.start_time
        usage = response.llm_output.get('token_usage', {})
        tokens = usage.get('total_tokens', 0)
        self.total_tokens += tokens

        # 计算成本（假设 $0.002 per 1K tokens）
        cost = (tokens / 1000) * 0.002
        self.total_cost += cost

        log_entry = f"[{time.strftime('%Y-%m-%d %H:%M:%S')}] LLM 结束\n"
        log_entry += f"耗时: {elapsed:.2f}s\n"
        log_entry += f"Tokens: {tokens} (累计: {self.total_tokens})\n"
        log_entry += f"成本: ${cost:.4f} (累计: ${self.total_cost:.4f})\n"
        log_entry += "-" * 50 + "\n"

        with open(self.log_file, "a") as f:
            f.write(log_entry)

    def on_llm_error(self, error: BaseException, **kwargs: Any) -> None:
        """LLM 错误时触发"""
        self.error_count += 1
        log_entry = f"[{time.strftime('%Y-%m-%d %H:%M:%S')}] LLM 错误\n"
        log_entry += f"错误: {error}\n"
        log_entry += f"错误次数: {self.error_count}\n"
        log_entry += "-" * 50 + "\n"

        with open(self.log_file, "a") as f:
            f.write(log_entry)

        # 发送告警（示例）
        if self.error_count > 5:
            print(f"⚠️ 告警: 错误次数超过 5 次！")

    def get_summary(self):
        """获取统计摘要"""
        return {
            "total_tokens": self.total_tokens,
            "total_cost": self.total_cost,
            "error_count": self.error_count
        }

# ===== 2. 使用自定义回调处理器 =====
print("=== 生产级回调处理器 ===")
handler = ProductionHandler(log_file="production.log")
llm = ChatOpenAI(temperature=0, callbacks=[handler])

# 多次调用
queries = [
    "What is 2+2?",
    "Explain Python in one sentence",
    "What is the capital of France?"
]

for query in queries:
    response = llm.invoke([HumanMessage(content=query)])
    print(f"Q: {query}")
    print(f"A: {response.content}\n")

# 打印统计摘要
summary = handler.get_summary()
print(f"\n=== 统计摘要 ===")
print(f"总 tokens: {summary['total_tokens']}")
print(f"总成本: ${summary['total_cost']:.4f}")
print(f"错误次数: {summary['error_count']}")

# ===== 3. 多个回调处理器组合 =====
print("\n=== 多个回调处理器组合 ===")

class SimpleLogHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        print(f"[日志] 开始: {prompts[0][:30]}...")

    def on_llm_end(self, response, **kwargs):
        print(f"[日志] 结束")

class SimpleCostHandler(BaseCallbackHandler):
    def __init__(self):
        self.total_cost = 0

    def on_llm_end(self, response, **kwargs):
        usage = response.llm_output.get('token_usage', {})
        tokens = usage.get('total_tokens', 0)
        cost = (tokens / 1000) * 0.002
        self.total_cost += cost
        print(f"[成本] ${cost:.4f} (累计: ${self.total_cost:.4f})")

# 组合多个处理器
handlers = [
    SimpleLogHandler(),
    SimpleCostHandler()
]

llm = ChatOpenAI(callbacks=handlers)
llm.invoke("Hello, how are you?")

# ===== 4. 运行时传入回调处理器 =====
print("\n=== 运行时传入回调处理器 ===")
llm = ChatOpenAI()
llm.invoke(
    "What is AI?",
    config={"callbacks": [SimpleLogHandler()]}
)
```

### 问题："如何实现一个成本追踪回调处理器？"

**普通回答（❌ 不出彩）：**
"继承 BaseCallbackHandler，重写 on_llm_end 方法，统计 token 用量。"

**出彩回答（✅ 推荐）：**

> **实现成本追踪回调处理器需要考虑三个层面：**
>
> 1. **基础实现**：
>    - 继承 `BaseCallbackHandler`
>    - 重写 `on_llm_end` 方法
>    - 从 `response.llm_output['token_usage']` 提取 token 用量
>    - 根据模型定价计算成本
>
> 2. **生产级增强**：
>    - 支持多种模型的定价（GPT-3.5、GPT-4、Claude 等）
>    - 区分 prompt tokens 和 completion tokens（定价不同）
>    - 记录每次调用的成本明细
>    - 实现成本告警机制（超过阈值时告警）
>
> 3. **并发环境处理**：
>    - 每个请求新建回调处理器实例，避免状态冲突
>    - 使用线程安全的数据结构（如 `threading.Lock`）
>    - 调用 `flush()` 确保数据持久化
>
> **与简单统计的区别**：成本追踪回调处理器不仅统计 token 用量，还要考虑不同模型的定价、并发环境的线程安全、成本告警等生产级需求。
>
> **在 RAG 系统中的应用**：可以追踪检索、重排序、生成等各个环节的成本，实现端到端的成本监控和优化。

**应用：** 每个处理器负责一个功能

### 卡片7：并发环境中的回调处理器

**一句话：** 并发环境中每个请求新建回调处理器实例，避免状态冲突

**举例：**
```python
for query in queries:
    handler = CostTracker()  # 每次新建
    llm.invoke(query, callbacks=[handler])
```

**应用：** 避免并发冲突和 trace_id 混乱

### 卡片8：异步回调处理器

**一句话：** 异步环境使用 AsyncCallbackHandler，所有方法都是 async def

**举例：**
```python
from langchain_core.callbacks import AsyncCallbackHandler

class MyAsyncHandler(AsyncCallbackHandler):
    async def on_llm_start(self, serialized, prompts, **kwargs):
        await asyncio.sleep(1)
```

**应用：** 与 FastAPI 等异步框架集成

### 卡片9：回调方法的返回值会被忽略

**一句话：** 回调方法只用于副作用，返回值会被忽略

**举例：**
```python
def on_llm_start(self, serialized, prompts, **kwargs):
    self.logs.append(prompts[0])  # 副作用
    # 返回值会被忽略
```

**应用：** 使用副作用（日志、统计）而非返回值
