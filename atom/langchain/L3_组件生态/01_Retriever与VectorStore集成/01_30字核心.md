# Retriever与VectorStore集成 - 30字核心

> **一句话精髓**：掌握 LangChain 中 Retriever 与 VectorStore 集成的核心机制

---

## 核心陈述

**VectorStore 通过 as_retriever() 转换为 Retriever，统一检索接口，支持三种检索策略（similarity、MMR、threshold），无缝集成 LCEL 管道。**

---

## 核心价值拆解

### 1. VectorStore 通过 as_retriever() 转换为 Retriever

**核心机制**：
```python
retriever = vectorstore.as_retriever()
```

**价值**：
- 将向量存储转换为标准 Retriever 接口
- 实现了适配器模式，统一检索抽象
- 支持 Runnable 接口，可在 LCEL 管道中使用

### 2. 统一检索接口

**核心接口**：
```python
class BaseRetriever(RunnableSerializable[str, list[Document]]):
    def invoke(self, query: str) -> list[Document]:
        ...
```

**价值**：
- 输入：查询字符串（str）
- 输出：文档列表（list[Document]）
- 标准化：所有 Retriever 实现相同接口

### 3. 支持三种检索策略

**策略选择**：
```python
# 策略 1：相似度搜索（默认）
retriever = vectorstore.as_retriever(search_type="similarity")

# 策略 2：MMR（多样性）
retriever = vectorstore.as_retriever(search_type="mmr")

# 策略 3：阈值过滤（质量控制）
retriever = vectorstore.as_retriever(search_type="similarity_score_threshold")
```

**价值**：
- **similarity**：基础相似度搜索，适合大多数场景
- **MMR**：平衡相关性和多样性，避免结果重复
- **threshold**：只返回高质量结果，适合精准问答

### 4. 无缝集成 LCEL 管道

**LCEL 集成**：
```python
from langchain_core.runnables import RunnablePassthrough
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI

# 构建 RAG 管道
rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
)

# 调用管道
answer = rag_chain.invoke("What is LangChain?")
```

**价值**：
- Retriever 是标准 Runnable，可直接用于 LCEL
- 支持流式、批量、异步等所有 Runnable 特性
- 简化 RAG 应用开发

## 为什么这 30 个字重要？

### 1. 抽象统一

**问题**：不同的向量存储有不同的检索 API
- Pinecone: `index.query()`
- Weaviate: `client.query.get()`
- ChromaDB: `collection.query()`

**解决**：通过 Retriever 抽象统一接口
```python
# 所有 VectorStore 都可以转换为 Retriever
pinecone_retriever = pinecone_vectorstore.as_retriever()
weaviate_retriever = weaviate_vectorstore.as_retriever()
chroma_retriever = chroma_vectorstore.as_retriever()

# 使用相同的接口
docs = retriever.invoke("query")
```

### 2. 策略灵活

**问题**：不同场景需要不同的检索策略

**解决**：通过 search_type 参数灵活切换
```python
# 场景 1：文档问答 - 使用相似度搜索
qa_retriever = vectorstore.as_retriever(search_type="similarity", search_kwargs={"k": 4})

# 场景 2：推荐系统 - 使用 MMR 增加多样性
recommend_retriever = vectorstore.as_retriever(search_type="mmr", search_kwargs={"k": 10, "lambda_mult": 0.3})

# 场景 3：精准问答 - 使用阈值过滤
precise_retriever = vectorstore.as_retriever(
    search_type="similarity_score_threshold",
    search_kwargs={"score_threshold": 0.8}
)
```

### 3. 管道集成

**问题**：如何将检索器集成到复杂的 LLM 应用中？

**解决**：Retriever 实现 Runnable 接口，无缝集成 LCEL
```python
# 简单 RAG
chain = retriever | format_docs | llm

# 复杂 RAG
chain = (
    {
        "context": retriever | format_docs,
        "question": RunnablePassthrough(),
        "history": RunnableLambda(get_history)
    }
    | prompt
    | llm
    | StrOutputParser()
)
```

## 核心设计模式

### 1. 适配器模式

```
VectorStore (多种实现) → as_retriever() → VectorStoreRetriever → BaseRetriever (统一接口)
```

**价值**：将不同的 VectorStore 实现适配到统一的 Retriever 接口

### 2. 策略模式

```python
class VectorStoreRetriever:
    search_type: str  # "similarity" | "mmr" | "similarity_score_threshold"

    def _get_relevant_documents(self, query: str) -> list[Document]:
        if self.search_type == "similarity":
            return self.vectorstore.similarity_search(query, **kwargs)
        elif self.search_type == "mmr":
            return self.vectorstore.max_marginal_relevance_search(query, **kwargs)
        elif self.search_type == "similarity_score_threshold":
            return self.vectorstore.similarity_search_with_relevance_scores(query, **kwargs)
```

**价值**：通过 search_type 参数动态选择检索策略

### 3. Runnable 集成

```python
class BaseRetriever(RunnableSerializable[str, list[Document]]):
    def invoke(self, query: str) -> list[Document]:
        ...

    async def ainvoke(self, query: str) -> list[Document]:
        ...

    def batch(self, queries: list[str]) -> list[list[Document]]:
        ...
```

**价值**：Retriever 是标准 Runnable，支持所有 LCEL 特性

## 实战价值

### 场景 1：快速原型

```python
# 3 行代码构建 RAG
vectorstore = InMemoryVectorStore.from_texts(texts, embeddings)
retriever = vectorstore.as_retriever()
docs = retriever.invoke("query")
```

### 场景 2：策略优化

```python
# 轻松切换检索策略
retriever = vectorstore.as_retriever(
    search_type="mmr",  # 从 similarity 切换到 mmr
    search_kwargs={"k": 10, "lambda_mult": 0.5}
)
```

### 场景 3：LCEL 管道

```python
# 无缝集成到 LCEL 管道
rag_chain = (
    {"context": retriever, "question": RunnablePassthrough()}
    | prompt
    | llm
)
```

### 场景 4：生产部署

```python
# 支持异步、批量、流式
docs = await retriever.ainvoke("query")  # 异步
batch_docs = retriever.batch(["q1", "q2", "q3"])  # 批量
```

## 关键参数速查

### as_retriever() 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `search_type` | str | "similarity" | 检索策略：similarity/mmr/similarity_score_threshold |
| `search_kwargs` | dict | {} | 检索参数字典 |
| `tags` | list[str] | None | 标签列表（用于追踪） |

### search_kwargs 参数

| 参数 | 适用策略 | 默认值 | 说明 |
|------|----------|--------|------|
| `k` | 所有 | 4 | 返回文档数量 |
| `score_threshold` | threshold | 必需 | 相似度阈值（0-1） |
| `fetch_k` | mmr | 20 | MMR 候选文档数量 |
| `lambda_mult` | mmr | 0.5 | MMR 多样性参数（0-1） |
| `filter` | 所有 | None | 元数据过滤条件 |
