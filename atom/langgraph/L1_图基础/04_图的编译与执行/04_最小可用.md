## 核心理念

**20%的核心知识，解决80%的问题**

在 LangGraph 中，图的编译与执行是将声明式的图定义转换为可运行的工作流的关键步骤。掌握以下5个核心知识点，你就能构建大部分实际应用。

## 4.1 基础编译与执行：compile() + invoke()

### 概念说明

**最基础的工作流**：定义图 → 编译 → 执行

```python
# 1. 定义图
builder = StateGraph(State)
builder.add_node("node_a", node_a_func)
builder.add_edge(START, "node_a")
builder.add_edge("node_a", END)

# 2. 编译图
app = builder.compile()

# 3. 执行图
result = app.invoke({"input": "hello"})
```

### 为什么这是最小可用？

- **compile()** 将图定义转换为可执行对象
- **invoke()** 同步执行图并返回最终结果
- 这是最简单、最直接的使用方式

### 代码示例

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END


# 定义状态
class State(TypedDict):
    message: str
    count: int


# 定义节点函数
def process_node(state: State) -> State:
    return {
        "message": state["message"].upper(),
        "count": state["count"] + 1
    }


# 构建图
builder = StateGraph(State)
builder.add_node("process", process_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

# 编译
app = builder.compile()

# 执行
result = app.invoke({"message": "hello", "count": 0})
print(result)
# 输出: {'message': 'HELLO', 'count': 1}
```

### 实际应用场景

- **简单的数据处理流程**：文本清洗、格式转换
- **单次执行的任务**：批量处理、报表生成
- **快速原型验证**：测试图的逻辑是否正确

### 关键要点

1. **编译是必需的**：不能直接执行 StateGraph，必须先 compile()
2. **invoke() 返回最终状态**：包含所有状态字段的字典
3. **同步执行**：会阻塞直到图执行完成

## 4.2 配置 Checkpointer：实现持久化和断点续传

### 概念说明

**Checkpointer 是图的"短期记忆"**：保存执行状态，支持暂停和恢复

```python
from langgraph.checkpoint.memory import MemorySaver

# 编译时配置 checkpointer
checkpointer = MemorySaver()
app = builder.compile(checkpointer=checkpointer)

# 执行时提供 thread_id
config = {"configurable": {"thread_id": "thread-1"}}
result = app.invoke(input_data, config=config)
```

### 为什么需要 Checkpointer？

- **断点续传**：图执行中断后可以恢复
- **状态追踪**：查看图的历史执行状态
- **人机交互**：等待用户输入后继续执行

### 代码示例

```python
from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph, START, END
from typing import TypedDict


class State(TypedDict):
    steps: list[str]
    current: str


def step1(state: State):
    return {"steps": state["steps"] + ["step1"], "current": "step1"}


def step2(state: State):
    return {"steps": state["steps"] + ["step2"], "current": "step2"}


# 构建图
builder = StateGraph(State)
builder.add_node("step1", step1)
builder.add_node("step2", step2)
builder.add_edge(START, "step1")
builder.add_edge("step1", "step2")
builder.add_edge("step2", END)

# 配置 checkpointer
checkpointer = MemorySaver()
app = builder.compile(checkpointer=checkpointer)

# 第一次执行
config = {"configurable": {"thread_id": "my-thread"}}
result1 = app.invoke({"steps": [], "current": ""}, config=config)
print(result1)
# 输出: {'steps': ['step1', 'step2'], 'current': 'step2'}

# 使用相同的 thread_id 可以访问历史状态
# （在实际应用中，可以通过 get_state() 查看历史）
```

### 实际应用场景

- **长时间运行的任务**：数据处理管道、批量任务
- **需要人工审批的流程**：审批工作流、质量检查
- **可恢复的服务**：服务重启后继续执行

### 关键要点

1. **thread_id 是关键**：相同的 thread_id 访问相同的状态
2. **MemorySaver 仅用于开发**：生产环境使用 SqliteSaver 或 PostgresSaver
3. **必须在 compile() 时配置**：不能在 invoke() 时临时添加

## 4.3 配置中断点：interrupt_before/after 实现人机交互

### 概念说明

**静态中断点**：在编译时指定在哪些节点前后暂停执行

```python
# 在 node_a 前中断，在 node_b 后中断
app = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["node_a"],
    interrupt_after=["node_b"]
)

# 第一次执行：运行到中断点
config = {"configurable": {"thread_id": "thread-1"}}
result1 = app.invoke(input_data, config=config)

# 恢复执行：传递 None 作为输入
result2 = app.invoke(None, config=config)
```

### 为什么需要中断点？

- **人机协作**：等待用户确认或输入
- **调试和测试**：在特定节点检查状态
- **分阶段执行**：将长流程拆分成多个阶段

### 代码示例

```python
from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph, START, END
from typing import TypedDict


class State(TypedDict):
    data: str
    approved: bool


def prepare_data(state: State):
    return {"data": "prepared data", "approved": False}


def process_data(state: State):
    return {"data": state["data"] + " -> processed", "approved": True}


# 构建图
builder = StateGraph(State)
builder.add_node("prepare", prepare_data)
builder.add_node("process", process_data)
builder.add_edge(START, "prepare")
builder.add_edge("prepare", "process")
builder.add_edge("process", END)

# 配置中断点：在 process 节点前中断
checkpointer = MemorySaver()
app = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["process"]  # 在处理前等待确认
)

# 第一次执行：运行到中断点
config = {"configurable": {"thread_id": "approval-flow"}}
result1 = app.invoke({"data": "", "approved": False}, config=config)
print("第一次执行结果:", result1)
# 输出: {'data': 'prepared data', 'approved': False}
# 注意：process 节点还没有执行

# 用户确认后，恢复执行
result2 = app.invoke(None, config=config)  # 传递 None
print("恢复执行结果:", result2)
# 输出: {'data': 'prepared data -> processed', 'approved': True}
```

### 实际应用场景

- **审批流程**：等待管理员批准
- **用户输入**：等待用户提供额外信息
- **质量检查**：人工检查中间结果

### 关键要点

1. **必须配置 checkpointer**：中断点依赖持久化
2. **恢复时传递 None**：表示继续执行，不修改输入
3. **使用相同的 thread_id**：确保恢复正确的执行上下文

## 4.4 流式执行：stream() 实时监控

### 概念说明

**流式输出**：逐步观察图的执行过程，而不是等待最终结果

```python
# 流式执行
for chunk in app.stream(input_data):
    print(chunk)  # 每个节点执行后输出一次
```

### 为什么需要流式执行？

- **实时监控**：观察图的执行进度
- **调试和日志**：记录每个节点的输出
- **用户体验**：显示进度条或实时反馈

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict


class State(TypedDict):
    step: int
    result: str


def step1(state: State):
    return {"step": 1, "result": "Step 1 completed"}


def step2(state: State):
    return {"step": 2, "result": "Step 2 completed"}


def step3(state: State):
    return {"step": 3, "result": "Step 3 completed"}


# 构建图
builder = StateGraph(State)
builder.add_node("step1", step1)
builder.add_node("step2", step2)
builder.add_node("step3", step3)
builder.add_edge(START, "step1")
builder.add_edge("step1", "step2")
builder.add_edge("step2", "step3")
builder.add_edge("step3", END)

app = builder.compile()

# 流式执行
print("=== 流式执行 ===")
for chunk in app.stream({"step": 0, "result": ""}):
    print(f"节点输出: {chunk}")

# 输出:
# 节点输出: {'step1': {'step': 1, 'result': 'Step 1 completed'}}
# 节点输出: {'step2': {'step': 2, 'result': 'Step 2 completed'}}
# 节点输出: {'step3': {'step': 3, 'result': 'Step 3 completed'}}
```

### stream_mode 参数

```python
# 1. stream_mode="updates" (默认)
# 输出每个节点的更新
for chunk in app.stream(input_data, stream_mode="updates"):
    print(chunk)  # {'node_name': {...}}

# 2. stream_mode="values"
# 输出每个步骤后的完整状态
for chunk in app.stream(input_data, stream_mode="values"):
    print(chunk)  # {'step': 1, 'result': '...'}
```

### 实际应用场景

- **Web 应用**：显示实时进度
- **调试和开发**：观察每个节点的输出
- **日志记录**：记录详细的执行过程

### 关键要点

1. **stream() 返回迭代器**：需要用 for 循环遍历
2. **每个 chunk 是一个字典**：键是节点名，值是节点输出
3. **stream_mode 控制输出格式**：updates 或 values

## 4.5 持久化模式：durability 参数控制

### 概念说明

**持久化模式**：控制何时将状态保存到 checkpointer

```python
# 同步持久化：每步执行后立即保存
app.invoke(input_data, config=config, durability="sync")

# 异步持久化：并行保存，不阻塞执行（默认）
app.invoke(input_data, config=config, durability="async")

# 退出时持久化：仅在图执行完成时保存
app.invoke(input_data, config=config, durability="exit")
```

### 为什么需要控制持久化模式？

- **可靠性 vs 性能**：同步持久化更可靠，但性能较低
- **不同场景的需求**：关键任务需要同步，高吞吐任务可以异步

### 代码示例

```python
from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph, START, END
from typing import TypedDict
import time


class State(TypedDict):
    counter: int


def increment(state: State):
    time.sleep(0.1)  # 模拟耗时操作
    return {"counter": state["counter"] + 1}


# 构建图
builder = StateGraph(State)
builder.add_node("inc1", increment)
builder.add_node("inc2", increment)
builder.add_node("inc3", increment)
builder.add_edge(START, "inc1")
builder.add_edge("inc1", "inc2")
builder.add_edge("inc2", "inc3")
builder.add_edge("inc3", END)

checkpointer = MemorySaver()
app = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "perf-test"}}

# 测试不同的持久化模式
print("=== 同步持久化 ===")
start = time.time()
result1 = app.invoke({"counter": 0}, config=config, durability="sync")
print(f"耗时: {time.time() - start:.2f}s, 结果: {result1}")

print("\n=== 异步持久化 ===")
config["configurable"]["thread_id"] = "perf-test-2"
start = time.time()
result2 = app.invoke({"counter": 0}, config=config, durability="async")
print(f"耗时: {time.time() - start:.2f}s, 结果: {result2}")

print("\n=== 退出时持久化 ===")
config["configurable"]["thread_id"] = "perf-test-3"
start = time.time()
result3 = app.invoke({"counter": 0}, config=config, durability="exit")
print(f"耗时: {time.time() - start:.2f}s, 结果: {result3}")
```

### 持久化模式对比

| 模式 | 可靠性 | 性能 | 适用场景 |
|------|--------|------|----------|
| `sync` | 最高 | 最低 | 关键任务、金融交易 |
| `async` | 中等 | 中等 | 一般应用（默认） |
| `exit` | 最低 | 最高 | 高吞吐、可重试任务 |

### 实际应用场景

- **sync**：支付流程、订单处理
- **async**：数据处理、内容生成
- **exit**：批量任务、日志分析

### 关键要点

1. **默认是 async**：平衡性能和可靠性
2. **sync 会降低性能**：每步都等待持久化完成
3. **exit 风险最高**：中途失败会丢失所有状态
