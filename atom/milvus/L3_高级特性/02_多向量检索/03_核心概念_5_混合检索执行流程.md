# 核心概念 5：混合检索执行流程

> **知识点定位**：L3_高级特性 > 02_多向量检索 > 核心概念 5
> **学习目标**：掌握 Milvus 多向量混合检索的完整执行流程，理解从定义 AnnSearchRequest 到获取融合结果的全过程

---

## 1. 概述

### 1.1 什么是混合检索执行流程

混合检索执行流程（Hybrid Search Pipeline）是 Milvus 中执行多向量混合检索的完整过程，包括：
- 定义多个 AnnSearchRequest（每个向量字段一个）
- 选择 Ranker（RRFRanker 或 WeightedRanker）
- 调用 hybrid_search API
- 获取融合后的结果

### 1.2 为什么需要混合检索

**单一向量检索的局限性**：
- 稠密向量：擅长语义理解，但可能忽略精确关键词
- 稀疏向量：擅长关键词匹配，但缺乏语义理解
- 单一模态：无法同时检索图片和文本

**混合检索的优势**：
- 结合多种向量类型的优势
- 提高检索准确率和召回率
- 支持多模态检索场景

### 1.3 核心组件

| 组件 | 作用 | 来源 |
|------|------|------|
| **AnnSearchRequest** | 定义单个向量字段的检索请求 | pymilvus |
| **RRFRanker** | 使用 RRF 算法融合结果 | pymilvus |
| **WeightedRanker** | 使用加权求和融合结果 | pymilvus |
| **hybrid_search** | 执行混合检索的主 API | MilvusClient/Collection |

## 2. 执行流程图

### 2.1 完整流程

```
┌─────────────────────────────────────────────────────────────┐
│                    混合检索执行流程                          │
└─────────────────────────────────────────────────────────────┘

步骤 1: 定义 AnnSearchRequest
┌──────────────────────────────────────────────────────────┐
│  req1 = AnnSearchRequest(                                │
│      data=[query_vector_1],                              │
│      anns_field="dense_vector",                          │
│      param={"metric_type": "IP"},                        │
│      limit=10                                            │
│  )                                                       │
│                                                          │
│  req2 = AnnSearchRequest(                                │
│      data=[query_vector_2],                              │
│      anns_field="sparse_vector",                         │
│      param={"metric_type": "IP"},                        │
│      limit=10                                            │
│  )                                                       │
└──────────────────────────────────────────────────────────┘
                          ↓
步骤 2: 选择 Ranker
┌──────────────────────────────────────────────────────────┐
│  选项 A: RRFRanker()                                     │
│  选项 B: WeightedRanker(weight1, weight2)                │
└──────────────────────────────────────────────────────────┘
                          ↓
步骤 3: 调用 hybrid_search
┌──────────────────────────────────────────────────────────┐
│  results = client.hybrid_search(                         │
│      collection_name="my_collection",                    │
│      reqs=[req1, req2],                                  │
│      ranker=ranker,                                      │
│      limit=10,                                           │
│      output_fields=["text", "metadata"]                  │
│  )                                                       │
└──────────────────────────────────────────────────────────┘
                          ↓
步骤 4: 获取融合结果
┌──────────────────────────────────────────────────────────┐
│  for hit in results[0]:                                  │
│      print(f"ID: {hit['id']}")                           │
│      print(f"Score: {hit['distance']}")                  │
│      print(f"Text: {hit['text']}")                       │
└──────────────────────────────────────────────────────────┘
```

### 2.2 内部执行机制

```
Milvus 内部执行流程：

1. 并行执行多个 ANN 检索
   ├─ 检索 dense_vector 字段 → 结果集 A
   └─ 检索 sparse_vector 字段 → 结果集 B

2. Ranker 融合结果
   ├─ RRFRanker: score = 1 / (k + rank)
   └─ WeightedRanker: score = w1 * score1 + w2 * score2

3. 按融合分数排序

4. 返回 Top-K 结果
```

## 3. 步骤详解

### 步骤 1：定义 AnnSearchRequest

**AnnSearchRequest 参数说明**：

| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `data` | List | 查询向量数据 | `[[0.1, 0.2, ...]]` |
| `anns_field` | str | 要检索的向量字段名 | `"dense_vector"` |
| `param` | dict | 检索参数 | `{"metric_type": "IP"}` |
| `limit` | int | 返回结果数量 | `10` |

**示例**：

```python
from pymilvus import AnnSearchRequest

# 定义稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense_embedding],      # 查询向量
    anns_field="dense_vector",         # 向量字段名
    param={"metric_type": "IP"},       # 使用内积
    limit=10                           # 返回 Top-10
)

# 定义稀疏向量检索请求
sparse_req = AnnSearchRequest(
    data=[query_sparse_embedding],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)
```

**关键点**：
- 每个 AnnSearchRequest 对应一个向量字段
- `limit` 参数控制每个检索请求的返回数量
- 最终结果数量由 `hybrid_search` 的 `limit` 参数控制

---

### 步骤 2：选择 Ranker

**Ranker 选择策略**：

| Ranker | 适用场景 | 优势 | 劣势 |
|--------|----------|------|------|
| **RRFRanker** | 不确定权重 | 自动平衡，无需调参 | 无法精确控制权重 |
| **WeightedRanker** | 已知权重 | 精确控制各字段重要性 | 需要手动调参 |

**RRFRanker 示例**：

```python
from pymilvus import RRFRanker

# 创建 RRF 排序器（使用默认 k=60）
ranker = RRFRanker()

# 自定义 k 参数
ranker = RRFRanker(k=100)
```

**RRF 算法原理**：
```
score = 1 / (k + rank)

其中：
- k: 平滑参数（默认 60）
- rank: 结果在原检索中的排名（从 1 开始）
```

**WeightedRanker 示例**：

```python
from pymilvus import WeightedRanker

# 创建加权排序器
ranker = WeightedRanker(
    0.7,  # 第一个检索请求的权重
    1.0   # 第二个检索请求的权重
)
```

**权重配置说明**：
- 权重可以是任意正数，不需要归一化
- 权重顺序与 `reqs` 列表顺序一致
- 权重越大，该检索结果的影响越大

---

### 步骤 3：调用 hybrid_search

**API 签名**：

```python
# MilvusClient 方式
results = client.hybrid_search(
    collection_name: str,
    reqs: List[AnnSearchRequest],
    ranker: Union[RRFRanker, WeightedRanker],
    limit: int,
    output_fields: List[str] = None,
    partition_names: List[str] = None,
    timeout: float = None
)

# Collection 方式
results = collection.hybrid_search(
    reqs: List[AnnSearchRequest],
    rerank: Union[RRFRanker, WeightedRanker],
    limit: int,
    output_fields: List[str] = None
)
```

**参数说明**：

| 参数 | 说明 |
|------|------|
| `collection_name` | Collection 名称（MilvusClient 方式） |
| `reqs` | AnnSearchRequest 列表 |
| `ranker/rerank` | 排序器（RRFRanker 或 WeightedRanker） |
| `limit` | 最终返回结果数量 |
| `output_fields` | 需要返回的字段列表 |
| `partition_names` | 指定检索的分区（可选） |

**完整示例**：

```python
from pymilvus import MilvusClient, AnnSearchRequest, RRFRanker

# 连接 Milvus
client = MilvusClient(uri="http://localhost:19530")

# 定义检索请求
dense_req = AnnSearchRequest(
    data=[query_dense_embedding],
    anns_field="dense_vector",
    param={"metric_type": "IP"},
    limit=10
)

sparse_req = AnnSearchRequest(
    data=[query_sparse_embedding],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 执行混合检索
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(),
    limit=5,  # 最终返回 Top-5
    output_fields=["text", "metadata"]
)
```

## 4. Python 完整示例

### 4.1 基础混合检索（RRFRanker）

```python
from pymilvus import MilvusClient, AnnSearchRequest, RRFRanker
import numpy as np

# 连接 Milvus
client = MilvusClient(uri="http://localhost:19530")

# 准备查询向量
query_dense = np.random.rand(768).tolist()
query_sparse = {0: 0.5, 10: 0.3, 20: 0.8}  # 稀疏向量

# 定义检索请求
dense_req = AnnSearchRequest(
    data=[query_dense],
    anns_field="dense_vector",
    param={"metric_type": "IP", "params": {"nprobe": 10}},
    limit=20
)

sparse_req = AnnSearchRequest(
    data=[query_sparse],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=20
)

# 执行混合检索
results = client.hybrid_search(
    collection_name="hybrid_demo",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=10,
    output_fields=["text", "source"]
)

# 处理结果
print(f"找到 {len(results[0])} 条结果")
for i, hit in enumerate(results[0], 1):
    print(f"{i}. ID={hit['id']}, Score={hit['distance']:.4f}")
    print(f"   Text: {hit['entity']['text'][:100]}...")
```

## 6. 常见场景

### 6.1 BM25 + 稠密向量（全文搜索增强）

**应用场景**：文档问答、知识库检索

**优势**：
- BM25：精确关键词匹配
- 稠密向量：语义理解
- 混合检索：兼顾精确匹配和语义相关性

**示例**：

```python
# 查询文本
query = "如何使用 Milvus 进行混合检索"

# BM25 检索
sparse_req = AnnSearchRequest(
    data=[query],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=10
)

# 稠密向量检索
query_embedding = embedding_model.encode(query)
dense_req = AnnSearchRequest(
    data=[query_embedding],
    anns_field="dense_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 混合检索
results = client.hybrid_search(
    collection_name="docs",
    reqs=[sparse_req, dense_req],
    ranker=RRFRanker(),
    limit=5
)
```

### 6.2 多模态检索（图片 + 文本）

**应用场景**：电商搜索、内容平台

**优势**：
- 同时检索图片和文本
- 提高检索准确率

**示例**：

```python
# 图片向量检索
image_req = AnnSearchRequest(

    param={"metric_type": "L2"},
```

## 7. 最佳实践

### 7.1 参数调优

**limit 参数配置**：
- `AnnSearchRequest.limit`：每个检索请求的返回数量，建议设置为最终结果的 2-5 倍
- `hybrid_search.limit`：最终返回结果数量

**示例**：

```python
# 如果最终需要 Top-10，建议每个请求返回 20-50 条
dense_req = AnnSearchRequest(..., limit=20)
sparse_req = AnnSearchRequest(..., limit=20)

results = client.hybrid_search(..., limit=10)
```

**Ranker 选择**：
- 不确定权重：使用 RRFRanker
- 已知权重：使用 WeightedRanker
- 需要调参：从 RRFRanker 开始，逐步调整到 WeightedRanker

### 7.2 性能优化

**索引优化**：
- 所有向量字段都必须创建索引
- 选择合适的索引类型（HNSW、IVF_FLAT 等）
- 调整索引参数（nlist、M、efConstruction）

**检索优化**：
- 使用 `partition_names` 限制检索范围
- 减少 `output_fields` 数量
- 批量查询时使用异步 API

**示例**：

```python
# 指定分区检索
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(),
    limit=10,
    partition_names=["partition_2024"]  # 只检索特定分区
)
```

## 8. 常见问题

### Q1: 为什么混合检索结果数量少于 limit？

**原因**：
- 多个检索请求返回的结果有重复
- 融合后去重导致结果数量减少

**解决方案**：
- 增加 `AnnSearchRequest.limit` 参数
- 确保每个检索请求返回足够多的结果

### Q2: RRFRanker 和 WeightedRanker 如何选择？

**选择策略**：
- 不确定权重 → RRFRanker
- 已知权重 → WeightedRanker
- 需要调参 → 先用 RRFRanker 测试，再用 WeightedRanker 优化

### Q3: 混合检索的性能如何？

**性能分析**：
- 多个 ANN 检索并行执行
- 融合开销很小（O(n log n)）
- 总体性能接近单个 ANN 检索

### Q4: 可以混合检索 3 个以上的向量字段吗？

**答案**：可以

**示例**：

```python
results = client.hybrid_search(
    collection_name="my_collection",
    reqs=[req1, req2, req3, req4],  # 4 个检索请求
    ranker=WeightedRanker(0.3, 0.3, 0.2, 0.2),  # 4 个权重
    limit=10
)
```

## 9. 总结

### 9.1 核心要点

1. **执行流程**：定义 AnnSearchRequest → 选择 Ranker → 调用 hybrid_search → 处理结果
2. **Ranker 选择**：RRFRanker（自动平衡）vs WeightedRanker（精确控制）
3. **参数配置**：`AnnSearchRequest.limit` 应大于 `hybrid_search.limit`
4. **索引要求**：所有向量字段都必须创建索引

### 9.2 应用场景

- **全文搜索增强**：BM25 + 稠密向量
- **多模态检索**：图片 + 文本
- **多语言检索**：中文向量 + 英文向量
- **多粒度检索**：标题向量 + 内容向量
