# 动态分发与 Trait Object - 一句话总结

> **知识点**: 动态分发与 Trait Object
> **层级**: Phase1_Rust速成基础
> **维度**: 一句话总结
> **目标受众**: 有 TypeScript/前端经验但零 Rust 经验的开发者

---

## 一句话总结

**动态分发通过 `dyn Trait` 的胖指针和 vtable 实现运行时多态，`Box<dyn Trait>` 提供堆上单一所有权、`Arc<dyn Trait>` 提供线程安全共享，受对象安全规则约束，是 ZeroClaw 可插拔 Trait 驱动架构（Provider/Channel/Tool/Memory）的核心机制。**

---

## 关键要点回顾

### 5 个核心概念一句话

| 概念 | 一句话 |
|------|--------|
| Trait Object | `dyn Trait` 通过胖指针（data + vtable）实现类型擦除和运行时方法查找 |
| Box\<dyn Trait\> | 堆上单一所有权的 trait object，用于工厂函数返回和异构集合存储 |
| Arc\<dyn Trait\> | 引用计数的线程安全 trait object，用于多组件共享同一实现 |
| 对象安全 | Trait 需满足 5 条规则（无泛型方法、不返回 Self 等）才能用作 `dyn Trait` |
| 静态 vs 动态 | 默认泛型（零开销），需要异构/运行时选择时用 `dyn`（~1-5ns） |

### 4 个实战场景一句话

| 场景 | 一句话 |
|------|--------|
| 可插拔架构 | `Box<dyn Provider>` + `Vec<Box<dyn Tool>>` 实现 config.toml 驱动的运行时组件切换 |
| 异构集合 | `Vec<Box<dyn Widget>>` 让不同类型通过共同 Trait 统一存储和操作 |
| 高级模式 | `as_any()` 恢复类型、ArcDelegatingTool 桥接 Arc→Box、trait upcasting 简化层次 |
| enum 对比 | 封闭集合用 enum 性能高 5-10 倍，开放集合必须用 `dyn` |

### ZeroClaw 速记

- Agent 持有 `Box<dyn Provider>` + `Vec<Box<dyn Tool>>`
- Memory 通过 `Arc<dyn Memory>` 在多个工具间共享
- 10 个核心 Trait 全部 object-safe，使用 `#[async_trait]`
- 架构边界用动态分发，内部热路径用泛型
