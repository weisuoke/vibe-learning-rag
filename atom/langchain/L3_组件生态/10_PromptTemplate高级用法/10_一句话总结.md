# 10_一句话总结

> 用一句话总结 PromptTemplate 高级用法的本质

[来源: 基于源码分析、Context7 官方文档、社区实践综合总结]

---

## 核心总结

**PromptTemplate 高级用法是通过 Partial Variables 预填充变量、Template Composition 组合模板、多种 Template Formats 灵活表达，实现提示词的模块化复用与动态管理，在 AI Agent 开发中提升提示词工程的可维护性和开发效率。**

---

## 为什么这句话能概括一切？

### 1. 抓住了三大核心能力

**Partial Variables（预填充变量）**：
- 解决了重复传递相同变量的问题
- 支持字符串固定值和函数动态值
- 实现了变量的延迟计算和懒加载

**Template Composition（模板组合）**：
- 使用 `+` 操作符实现模板的灵活组合
- 自动合并 input_variables 和 partial_variables
- 支持 PromptTemplate + str 的混合组合

**Template Formats（模板格式）**：
- f-string：简单快速，默认选择
- mustache：支持嵌套和 section，适合复杂结构
- jinja2：功能强大但需注意安全性

### 2. 点明了核心价值

**模块化复用**：
- 将提示词拆分为可复用的模块
- 通过组合实现复杂提示词的构建
- 减少代码重复，提升可维护性

**动态管理**：
- 运行时动态计算变量值
- 灵活切换不同的模板格式
- 支持从文件加载和验证

### 3. 强调了实际应用价值

**在 AI Agent 开发中**：
- RAG 系统：管理多语言提示词模板
- Agent 系统：构建工具调用和推理模板
- 对话系统：组合系统提示和用户消息
- 批量任务：预填充常量减少重复传递

## 快速参考卡

### 核心 API 速查

```python
# 1. Partial Variables（预填充变量）
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 字符串固定值
prompt = PromptTemplate(
    template="System: {system}\nUser: {query}",
    input_variables=["query"],
    partial_variables={"system": "You are a helpful assistant"}
)

# 函数动态值
prompt = PromptTemplate(
    template="Date: {date}\nQuery: {query}",
    input_variables=["query"],
    partial_variables={"date": lambda: datetime.now().strftime("%Y-%m-%d")}
)

# 2. Template Composition（模板组合）
prefix = PromptTemplate.from_template("System: {system}\n")
suffix = PromptTemplate.from_template("User: {query}")
full_prompt = prefix + suffix  # 自动合并变量

# 3. Template Formats（模板格式）
# f-string（默认）
prompt_f = PromptTemplate.from_template("Say {foo}")

# mustache
prompt_m = PromptTemplate.from_template(
    "Say {{foo}}",
    template_format="mustache"
)

# jinja2
prompt_j = PromptTemplate.from_template(
    "Say {{ foo }}",
    template_format="jinja2"
)
```

### 使用场景速查

| 场景 | 推荐技术 | 原因 |
|------|----------|------|
| 预设系统提示 | Partial Variables（字符串） | 避免重复传递固定值 |
| 动态日期时间 | Partial Variables（函数） | 运行时计算最新值 |
| 多段提示组合 | Template Composition | 模块化管理提示词 |
| 简单变量替换 | f-string | 性能最优，语法简单 |
| 嵌套数据结构 | mustache | 支持对象嵌套访问 |
| 复杂逻辑控制 | jinja2 | 功能强大（注意安全） |
| RAG 多语言模板 | Composition + Partial | 复用基础模板 |
| Agent 工具调用 | Composition + Formats | 灵活组合工具提示 |

### 常见陷阱速查

| 陷阱 | 表现 | 解决方案 |
|------|------|----------|
| partial_variables 冲突 | 组合时报错 | 检查变量名是否重复 |
| template_format 不一致 | 组合失败 | 确保格式一致 |
| ChatPromptTemplate 兼容性 | partial_variables 不生效 | 使用最新版本或避免使用 |
| jinja2 安全风险 | 代码注入 | 不接受不可信输入 |
| mustache 验证限制 | 无法验证模板 | 使用 f-string 或关闭验证 |
| 函数动态值未调用 | 返回函数对象 | 确保传入函数而非函数调用结果 |

快速验证：创建一个带预填充变量和组合的提示词模板

## 最后的话

PromptTemplate 高级用法不是为了炫技，而是为了解决实际问题：

1. **Partial Variables** 解决了重复传递相同变量的痛点
2. **Template Composition** 实现了提示词的模块化管理
3. **Template Formats** 提供了灵活的表达方式

在 AI Agent 开发中，提示词工程是核心能力之一。掌握 PromptTemplate 高级用法，能让你的提示词更加：
- **可维护**：模块化设计，易于修改和扩展
- **可复用**：组合式架构，减少重复代码
- **可测试**：独立模块，便于单元测试
- **可扩展**：灵活格式，适应不同场景

记住：**好的提示词工程不是写出复杂的提示词，而是写出简单、清晰、可维护的提示词。**
