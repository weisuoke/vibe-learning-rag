# 30字核心

## 核心定义

**子图（Subgraph）是将编译后的 StateGraph 作为节点嵌入父图的机制，实现工作流的模块化复用。**

---

## 核心要点拆解

### 1. 子图是什么？

**编译后的 StateGraph 作为节点** -- 子图就是普通的 StateGraph，`compile()` 后通过 `add_node()` 添加到父图，父图把它当作普通节点调度。

### 2. 如何与父图通信？

- **共享键**：父子图 Schema 中同名的键自动双向传递
- **转换函数**：用包装函数手动完成不同 Schema 之间的状态映射
- **Command.PARENT**：子图向父图发送路由指令，实现跨图控制

### 3. 为什么需要模块化？

拆分大图为语义清晰的子模块，同一子图可在多个父图中复用，每个子图可独立编译、独立测试。

---

## 一句话记忆

> 子图 = compile() 后的 StateGraph + add_node() 嵌入父图

---

## 最小示例

```python
from langgraph.graph import START, StateGraph
from typing_extensions import TypedDict

class State(TypedDict):
    value: str

def sub_node(state: State):
    return {"value": state["value"] + " [processed]"}

# 创建并编译子图
sub = StateGraph(State)
sub.add_node("process", sub_node)
sub.add_edge(START, "process")
subgraph = sub.compile()

# 嵌入父图
parent = StateGraph(State)
parent.add_node("subgraph", subgraph)
parent.add_edge(START, "subgraph")
graph = parent.compile()

result = graph.invoke({"value": "hello"})
# {'value': 'hello [processed]'}
```
