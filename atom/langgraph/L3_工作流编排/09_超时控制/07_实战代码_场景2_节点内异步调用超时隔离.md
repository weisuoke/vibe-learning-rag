# 实战代码 · 场景 2：节点内异步调用超时隔离

> **目标**：演示 `asyncio.wait_for()` 如何在 node 内先切断慢调用，并让图继续走 fallback
> **适用**：外部 API / 工具 / LLM 调用不稳定，想避免整个 step 被拖死

---

## 场景说明

这次我们不让 timeout 直接炸整个图，而是：

1. 在 node 内对慢调用做 `asyncio.wait_for()`；
2. 超时后写入 `timed_out=True`；
3. 用条件边把流程路由到 fallback 节点；
4. 图仍然设置 `step_timeout` 作为外层兜底。

这更接近真实生产系统。

---

## 完整代码

```python
"""
场景2：节点内异步调用超时隔离

运行方式：
    python example_node_inner_timeout.py

依赖：
    pip install langgraph
"""

from __future__ import annotations

import asyncio
from typing import Literal

from typing_extensions import TypedDict

from langgraph.graph import END, START, StateGraph


class State(TypedDict):
    query: str
    source: Literal["remote", "cache", "unknown"]
    answer: str
    timed_out: bool


async def fake_remote_search(query: str, delay_s: float) -> str:
    """模拟慢速远程搜索。"""
    await asyncio.sleep(delay_s)
    return f"remote_result_for:{query}"


async def search_node(state: State) -> State:
    """对慢远程调用加 node 内 timeout。"""
    print(f"[search_node] 开始搜索 query={state['query']!r}")

    try:
        # 远程调用需要 2 秒，但这里只给 0.8 秒预算
        result = await asyncio.wait_for(
            fake_remote_search(state["query"], delay_s=2.0),
            timeout=0.8,
        )
        print("[search_node] 远程搜索成功")
        return {
            "query": state["query"],
            "source": "remote",
            "answer": result,
            "timed_out": False,
        }

    except asyncio.TimeoutError:
        print("[search_node] 远程搜索超时，准备走 fallback")
        return {
            "query": state["query"],
            "source": "unknown",
            "answer": "",
            "timed_out": True,
        }


def route_after_search(state: State) -> str:
    return "fallback" if state["timed_out"] else "finish"


def fallback_node(state: State) -> State:
    print("[fallback_node] 返回缓存/降级结果")
    return {
        "query": state["query"],
        "source": "cache",
        "answer": f"cached_result_for:{state['query']}",
        "timed_out": state["timed_out"],
    }


def finish_node(state: State) -> State:
    print("[finish_node] 远程调用成功，直接收尾")
    return state


def build_graph():
    builder = StateGraph(State)
    builder.add_node("search", search_node)
    builder.add_node("fallback", fallback_node)
    builder.add_node("finish", finish_node)

    builder.add_edge(START, "search")
    builder.add_conditional_edges(
        "search",
        route_after_search,
        {
            "fallback": "fallback",
            "finish": "finish",
        },
    )
    builder.add_edge("fallback", END)
    builder.add_edge("finish", END)

    graph = builder.compile()

    # 外层 step 预算仍然存在，但比 node 内 timeout 稍宽
    graph.step_timeout = 3.0
    return graph


async def main() -> None:
    graph = build_graph()

    initial_state: State = {
        "query": "langgraph timeout best practices",
        "source": "unknown",
        "answer": "",
        "timed_out": False,
    }

    result = await graph.ainvoke(initial_state)
    print("\n最终结果：")
    for key, value in result.items():
        print(f"- {key}: {value}")


if __name__ == "__main__":
    asyncio.run(main())
```

## 这个示例为什么重要？

### 1. timeout 没有直接把图炸掉

因为 node 内部已经把“慢远程调用”转成了一个可路由的业务状态：`timed_out=True`。

### 2. fallback 是业务友好的

对于搜索、推荐、摘要等场景，很多时候：
- 返回缓存结果；
- 返回部分结果；
- 返回“稍后重试”；

都比整图直接失败更好。

### 3. `step_timeout` 仍然应该保留

如果 node 自己的逻辑也出了问题，例如：
- fallback 本身卡住；
- 代码路径异常复杂；
- 事件循环被别的任务拖慢；

外层 `step_timeout` 仍然会是最后一道防线。

---

## 这个模式能如何升级？

你可以把 `fake_remote_search()` 替换成：
- `httpx.AsyncClient` 请求；
- 模型 SDK 调用；
- 向量检索接口；
- 远程工具链。

进一步还可以加入：
- `RetryPolicy`；
- 不同错误类型走不同 fallback；
- 记录 timeout 次数与耗时；
- 用 `runtime.context` 注入预算值。

---

## 最关键的设计原则

**越接近慢调用根因的地方，越应该先处理 timeout。**

这就是 node 内 timeout 的价值。

## 变体实现：参数化辅助层

```python
from __future__ import annotations

import asyncio
import time
from dataclasses import dataclass
from typing import Any

@dataclass
class TimeoutBudgets:
    node_timeout_s: float = 1.0
    step_timeout_s: float = 2.0
    consumer_timeout_s: float = 0.5

async def inner_timeout_isolation_demo_remote_call(delay_s: float) -> dict[str, Any]:
    await asyncio.sleep(delay_s)
    return {'ok': True, 'delay_s': delay_s}

async def inner_timeout_isolation_demo_guarded_call(delay_s: float, budgets: TimeoutBudgets) -> dict[str, Any]:
    started_at = time.perf_counter()
    try:
        result = await asyncio.wait_for(inner_timeout_isolation_demo_remote_call(delay_s), timeout=budgets.node_timeout_s)
        result["timed_out"] = False
        result["elapsed_s"] = round(time.perf_counter() - started_at, 3)
        return result
    except asyncio.TimeoutError:
        return {
            "ok": False,
            "timed_out": True,
            "delay_s": delay_s,
            "elapsed_s": round(time.perf_counter() - started_at, 3),
        }

async def inner_timeout_isolation_demo_main() -> None:
    budgets = TimeoutBudgets()
    result = await inner_timeout_isolation_demo_guarded_call(delay_s=1.8, budgets=budgets)
    if result["timed_out"]:
        print("[节点内异步调用超时隔离] 使用 fallback 或降级路径")
    else:
        print("[节点内异步调用超时隔离] 使用主路径结果")
    print(result)

if __name__ == "__main__":
    asyncio.run(inner_timeout_isolation_demo_main())
```
