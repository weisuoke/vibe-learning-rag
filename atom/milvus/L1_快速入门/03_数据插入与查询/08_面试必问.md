## 基础概念题

### Q1: 什么是 Milvus 的 Embedding Functions？它解决了什么问题？

**答案**:

Embedding Functions 是 Milvus 2.6 引入的核心特性，实现了 Data-in, Data-out 模式。

**解决的问题**:
1. **简化开发**: 无需手动调用嵌入 API，Milvus 自动处理
2. **保证一致性**: 插入和查询自动使用同一模型
3. **减少代码量**: 从传统的 5 步流程简化为 3 步

**示例**:
```python
# 传统方式：手动嵌入
embeddings = openai.embed(text)
milvus.insert(embeddings)

# Milvus 2.6：自动嵌入
milvus.insert({"document": text})  # 自动调用嵌入 API
```

**关键点**: Embedding Functions 只处理 Schema 中显式定义的字段，不处理动态字段。

---

### Q2: Insert 和 Upsert 有什么区别？什么时候用哪个？

**答案**:

| 特性 | Insert | Upsert |
|------|--------|--------|
| **主键检查** | 不检查重复 | 检查并更新 |
| **性能** | 更快 | 稍慢 |
| **数据一致性** | 可能重复 | 保证唯一 |
| **适用场景** | 新数据批量导入 | 增量更新、数据同步 |

**使用建议**:
- **Insert**: 首次导入大量新数据，确保没有重复
- **Upsert**: 更新已有数据，或不确定数据是否存在

**陷阱**: Insert 不检查主键重复，可能导致同一主键有多条记录！

---

### Q3: Search 和 Query 有什么区别？

**答案**:

| 特性 | Search | Query |
|------|--------|-------|
| **查询方式** | 向量相似度 | 标量字段过滤 |
| **是否涉及向量** | 是 | 否 |
| **结果排序** | 相似度降序 | 插入顺序或主键 |
| **适用场景** | 语义搜索、相似推荐 | 精确匹配、元数据查询 |

**示例**:
```python
# Search: 找相似的文档
results = client.search(
    data=["What is Milvus?"],  # 语义搜索
    anns_field="dense",
    limit=5
)

# Query: 精确查询
results = client.query(
    filter='category == "tech"',  # 标量过滤
    limit=10
)
```

## 性能优化题

### Q4: 如何优化 Milvus 的插入性能？

**答案**:

**1. 批量插入**:
```python
# ❌ 逐条插入（慢）
for doc in docs:
    client.insert([doc])

# ✅ 批量插入（快）
client.insert(docs)  # 推荐批次大小 1000-5000
```

**2. 使用 Upsert 时注意**:
- Upsert 需要检查主键，比 Insert 慢
- 如果确定是新数据，用 Insert

**3. 合理使用分区**:
- 按业务逻辑分区，减少搜索范围
- 插入时指定分区

**4. 预分配主键**:
- 使用 `auto_id=True` 让 Milvus 自动生成主键

**性能数据**:
- 单条插入: ~1ms/条
- 批量插入(1000条): ~0.1ms/条
- 提升 10 倍

### Q5: 如何优化 Milvus 的搜索性能？

**答案**:

**1. 选择合适的 limit**:
```python
# ❌ 返回过多结果
results = client.search(limit=1000)

# ✅ 只返回需要的结果
results = client.search(limit=10)
```

**2. 使用分区搜索**:
```python
# ❌ 搜索所有分区
results = client.search(...)

# ✅ 只搜索相关分区
results = client.search(partition_names=["tech_docs"])
```

**3. 优化过滤条件**:
- 简单的等值判断比复杂表达式快
- 过滤条件能显著减少候选集时才有效

**4. 批量搜索**:
```python
# ❌ 逐个搜索
for query in queries:
    results = client.search(data=[query])

# ✅ 批量搜索
results = client.search(data=queries)
```

**5. 选择合适的索引**:
- AUTOINDEX: 自动选择最优索引
- 根据数据规模和精度要求调整参数

## 高级特性题

### Q6: 什么是混合搜索？为什么需要它？

**答案**:

混合搜索是在单次查询中结合多种搜索策略，最常见的是 **Dense + Sparse** 混合。

**为什么需要**:

| 场景 | Dense 向量 | Sparse 向量 | 混合搜索 |
|------|-----------|------------|----------|
| "vector database" | ✅ 找到语义相关 | ❌ 可能漏掉精确匹配 | ✅ 最佳 |
| "Milvus 2.6" | ❌ 可能找到 2.5 | ✅ 精确匹配版本号 | ✅ 最佳 |

**实现**:
```python
# Dense 搜索请求
dense_req = AnnSearchRequest(
    data=[query],
    anns_field="dense_vector",
    limit=100
)

# Sparse 搜索请求
sparse_req = AnnSearchRequest(
    data=[query],
    anns_field="sparse_vector",
    limit=100
)

# 混合搜索
results = client.hybrid_search(
    search_requests=[dense_req, sparse_req],
    ranker=RRFRanker(),  # 重排序
    limit=10
)
```

**关键点**: 候选数量(100) ≠ 最终返回数量(10)

### Q7: ANN 和 kNN 有什么区别？为什么 Milvus 使用 ANN？

**答案**:

**kNN (k-Nearest Neighbors)**:
- 精确搜索，遍历所有向量
- 时间复杂度 O(n)
- 100% 召回率
- 不可扩展（大数据下太慢）

**ANN (Approximate Nearest Neighbor)**:
- 近似搜索，使用索引快速定位
- 时间复杂度 O(log n) 或 O(1)
- 95-99% 召回率
- 可扩展（支持亿级向量）

**性能对比**:
- 1亿向量，kNN: ~100s，ANN: ~10ms
- 速度提升 10,000 倍

**为什么使用 ANN**: 在大规模数据下，牺牲 1-5% 的精度换取 10,000 倍的速度是值得的。

## 实战场景题

### Q8: 如何在 RAG 应用中使用 Milvus？

**答案**:

**完整流程**:

```python
# 1. 创建 Collection（配置 Embedding Function）
schema.add_function(Function(
    name="openai_embedding",
    input_field_names=["document"],
    output_field_names=["dense"],
    params={"provider": "openai", "model_name": "text-embedding-3-small"}
))

# 2. 插入文档（自动嵌入）
client.insert("doc_collection", [
    {"document": "Milvus is a vector database..."},
    {"document": "RAG combines retrieval and generation..."}
])

# 3. 查询（自动嵌入查询文本）
results = client.search(
    collection_name="doc_collection",
    data=["What is Milvus?"],  # 原始文本
    anns_field="dense",
    limit=3
)

# 4. 提取上下文
contexts = [hit['entity']['document'] for hit in results[0]]

# 5. 调用 LLM
prompt = f"基于以下上下文回答：\n{contexts}\n\n问题：{question}"
answer = llm.generate(prompt)
```

**关键点**:
- 使用 Embedding Functions 简化流程
- Top-K 通常设置为 3-5
- 使用 COSINE 度量类型

### Q9: 如何处理大规模数据导入？

**答案**:

**方案对比**:

| 数据量 | 推荐方案 | 批次大小 |
|--------|----------|----------|
| < 1万 | Insert | 100-500 |
| 1万-10万 | Insert | 500-1000 |
| 10万-100万 | Insert | 1000-5000 |
| > 100万 | BulkInsert | 文件导入 |

**BulkInsert 示例**:
```python
from pymilvus import utility

# 准备数据文件（JSON 或 Parquet）
# data.json: [{"id": 1, "document": "...", "dense": [...]}, ...]

# 执行批量导入
task_id = utility.do_bulk_insert(
    collection_name="my_collection",
    files=["data.json"]
)

# 检查状态
state = utility.get_bulk_insert_state(task_id)
```

**性能对比**:
- Insert (批量): ~1000 条/秒
- BulkInsert: ~10000 条/秒
- 提升 10 倍

### Q10: 如何调试 Milvus 搜索结果不准确的问题？

**答案**:

**排查步骤**:

**1. 检查度量类型**:
```python
# 确认使用正确的度量类型
search_params = {"metric_type": "COSINE"}  # 推荐用于文本
```

**2. 检查向量归一化**:
```python
# 使用 COSINE 或 IP 时需要归一化
import numpy as np

def normalize(vector):
    norm = np.linalg.norm(vector)
    return (vector / norm).tolist() if norm > 0 else vector
```

**3. 检查嵌入模型一致性**:
```python
# 确保插入和查询使用同一模型
# Embedding Functions 自动保证一致性
```

**4. 检查数据质量**:
```python
# 查看实际存储的向量
results = client.query(
    filter='id == 1',
    output_fields=["document", "dense"]
)
print(results[0]["dense"])  # 检查向量是否正常
```

**5. 调整搜索参数**:
```python
# 增加候选数量
search_params = {
    "metric_type": "COSINE",
    "params": {"nprobe": 20}  # 增大 nprobe 提高召回率
}
```

## 陷阱题

### Q11: 以下代码有什么问题？

```python
# 代码1
client.insert("my_collection", [
    {"id": 1, "document": "text1"}
])
client.insert("my_collection", [
    {"id": 1, "document": "text2"}
])
```

**答案**: Insert 不检查主键重复，会创建两条 id=1 的记录！应该使用 Upsert。

---

### Q12: 以下代码有什么问题？

```python
# 代码2
results = client.search(
    data=[query_vector],
    search_params={"metric_type": "COSINE"},
    limit=10
)

# 判断相似度
if results[0][0]['distance'] < 0.5:
    print("不相似")
```

**答案**: COSINE 的 distance 越大越相似，不是越小！应该判断 `> 0.5`。

---

### Q13: 以下代码有什么问题？

```python
# 代码3
for i in range(10000):
    client.insert("my_collection", [data[i]])
```

**答案**: 逐条插入性能极差！应该批量插入：
```python
client.insert("my_collection", data)
```
