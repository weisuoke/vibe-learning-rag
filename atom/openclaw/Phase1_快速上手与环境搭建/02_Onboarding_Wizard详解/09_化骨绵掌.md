## 技巧 1：使用环境变量预配置向导

### 基础用法

```bash
# 预设所有配置
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENCLAW_GATEWAY_TOKEN="oc_1a2b3c4d5e6f7g8h9i0j"
export OPENCLAW_GATEWAY_PORT="18789"
export OPENCLAW_WORKSPACE="$HOME/.openclaw/workspace"

# 运行向导（自动读取环境变量）
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key
```

### 高级技巧：使用 .env 文件

```bash
# 创建 .env 文件
cat > ~/.openclaw/.env <<'EOF'
ANTHROPIC_API_KEY=sk-ant-...
OPENCLAW_GATEWAY_TOKEN=oc_1a2b3c4d5e6f7g8h9i0j
OPENCLAW_GATEWAY_PORT=18789
OPENCLAW_WORKSPACE=$HOME/.openclaw/workspace
EOF

# 加载并运行
source ~/.openclaw/.env
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key
```

### 专家技巧：多环境配置

```bash
# 开发环境
cat > ~/.openclaw/.env.dev <<'EOF'
ANTHROPIC_API_KEY=sk-ant-dev-...
OPENCLAW_GATEWAY_PORT=18789
OPENCLAW_WORKSPACE=$HOME/.openclaw/workspace-dev
EOF

# 生产环境
cat > ~/.openclaw/.env.prod <<'EOF'
ANTHROPIC_API_KEY=sk-ant-prod-...
OPENCLAW_GATEWAY_PORT=18790
OPENCLAW_WORKSPACE=$HOME/.openclaw/workspace-prod
EOF

# 切换环境
alias openclaw-dev='source ~/.openclaw/.env.dev && openclaw'
alias openclaw-prod='source ~/.openclaw/.env.prod && openclaw'
```

## 技巧 2：自定义向导流程（跳过特定步骤）

### 跳过健康检查（加速部署）

```bash
openclaw onboard --skip-health
```

**使用场景**：
- CI/CD 环境（Gateway 稍后启动）
- 调试配置（不需要立即启动）
- 批量部署（手动验证）

### 跳过 Daemon 安装（容器环境）

```bash
openclaw onboard --no-install-daemon
```

**使用场景**：
- Docker 容器（使用 `docker-compose` 管理）
- Kubernetes（使用 Deployment 管理）
- 开发环境（手动启动 Gateway）

### 跳过 Skills 安装（最小化安装）

```bash
openclaw onboard --skip-skills
```

**使用场景**：
- 最小化安装（后续按需安装）
- 离线环境（无法下载依赖）
- 自定义 Skills（手动安装）

### 组合使用

```bash
# 最小化非交互式配置
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --skip-health \
  --no-install-daemon \
  --skip-skills
```

## 技巧 3：自定义 Provider 高级配置

### Unknown 兼容性自动检测

```bash
# 让 OpenClaw 自动检测 API 兼容性
openclaw onboard --non-interactive \
  --auth-choice custom-api-key \
  --custom-base-url "https://api.groq.com/openai/v1" \
  --custom-model-id "llama-3.1-70b-versatile" \
  --custom-api-key "$GROQ_API_KEY" \
  --custom-compatibility unknown  # 自动检测
```

**支持的自动检测**：
- OpenAI-compatible（OpenRouter、Together AI、Groq）
- Anthropic-compatible（自托管 Claude）

### 多 Provider 配置

```bash
# 配置主 Provider
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY"

# 添加备用 Provider
openclaw config set agents.defaults.model.fallbacks '["openai/gpt-4"]'
openclaw config set env.OPENAI_API_KEY "$OPENAI_API_KEY"

# 验证
openclaw models status --probe
```

### 自定义 Endpoint ID（多个自定义 Provider）

```bash
# Provider 1: Groq
openclaw onboard --non-interactive \
  --auth-choice custom-api-key \
  --custom-base-url "https://api.groq.com/openai/v1" \
  --custom-model-id "llama-3.1-70b-versatile" \
  --custom-api-key "$GROQ_API_KEY" \
  --custom-endpoint-id "groq"

# Provider 2: Together AI
openclaw config set models.providers.together.baseUrl "https://api.together.xyz/v1"
openclaw config set models.providers.together.apiKey "$TOGETHER_API_KEY"
openclaw config set agents.defaults.models.together/llama-3.1-405b.alias "Llama 405B"
```

## 技巧 4：高级 Gateway 配置

### 自定义绑定地址（多网卡环境）

```bash
# 绑定到特定网卡
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --gateway-bind custom \
  --gateway-custom-bind-host "192.168.1.100"
```

**使用场景**：
- 多网卡服务器（内网 + 外网）
- VPN 环境（绑定到 VPN 网卡）
- 容器网络（绑定到 bridge 网络）

### Tailscale 高级配置

```bash
# Tailscale Serve（私有网络）
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --gateway-bind loopback \
  --tailscale-mode serve \
  --tailscale-reset-on-exit false

# Tailscale Funnel（公网暴露）
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --gateway-bind loopback \
  --gateway-auth-mode password \
  --gateway-password "strong-password-here" \
  --tailscale-mode funnel \
  --tailscale-reset-on-exit true
```

**安全建议**：
- Funnel 必须使用强密码
- 启用 `tailscale-reset-on-exit`（退出时清理）
- 定期轮换密码

## 技巧 5：多 Agent 环境高级配置

### 创建隔离的 Agent 环境

```bash
# Agent 1: 工作环境
openclaw agents add work \
  --workspace "$HOME/.openclaw/workspace-work" \
  --model "anthropic/claude-sonnet-4-5" \
  --bind "work:whatsapp:account1"

# Agent 2: 个人环境
openclaw agents add personal \
  --workspace "$HOME/.openclaw/workspace-personal" \
  --model "anthropic/claude-haiku-4" \
  --bind "personal:telegram:account2"

# Agent 3: 实验环境
openclaw agents add experimental \
  --workspace "$HOME/.openclaw/workspace-experimental" \
  --model "openai/gpt-4" \
  --bind "experimental:discord:server1"
```

### 配置 Agent 路由规则

```json5
// ~/.openclaw/openclaw.json
{
  agents: {
    list: [
      { id: "work", workspace: "~/.openclaw/workspace-work" },
      { id: "personal", workspace: "~/.openclaw/workspace-personal" },
      { id: "experimental", workspace: "~/.openclaw/workspace-experimental" },
    ],
  },
  bindings: [
    // WhatsApp 工作账号 → work agent
    { agentId: "work", match: { channel: "whatsapp", accountId: "work-phone" } },
    // Telegram 个人账号 → personal agent
    { agentId: "personal", match: { channel: "telegram", accountId: "personal-bot" } },
    // Discord 服务器 → experimental agent
    { agentId: "experimental", match: { channel: "discord", guildId: "123456789" } },
  ],
}
```

## 技巧 6：配置文件高级技巧

### 使用 $include 分割配置

```json5
// ~/.openclaw/openclaw.json
{
  gateway: { $include: "./config/gateway.json5" },
  agents: { $include: "./config/agents.json5" },
  channels: { $include: "./config/channels.json5" },
  models: { $include: "./config/models.json5" },
}
```

```json5
// ~/.openclaw/config/gateway.json5
{
  port: 18789,
  bind: "loopback",
  auth: {
    mode: "token",
    token: "${OPENCLAW_GATEWAY_TOKEN}",
  },
}
```

**优点**：
- 模块化配置
- 易于版本控制
- 团队协作友好

### 使用环境变量替换

```json5
// ~/.openclaw/openclaw.json
{
  gateway: {
    auth: {
      token: "${OPENCLAW_GATEWAY_TOKEN}",
    },
  },
  env: {
    ANTHROPIC_API_KEY: "${ANTHROPIC_API_KEY}",
    OPENAI_API_KEY: "${OPENAI_API_KEY}",
  },
}
```

**规则**：
- 只支持大写变量名：`[A-Z_][A-Z0-9_]*`
- 缺失变量会报错（启动时）
- 转义：`$${VAR}` → `${VAR}`（字面量）

## 技巧 7：守护进程高级管理

### macOS LaunchAgent 高级配置

```xml
<!-- ~/Library/LaunchAgents/ai.openclaw.gateway.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>ai.openclaw.gateway</string>

  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/openclaw</string>
    <string>gateway</string>
    <string>run</string>
    <string>--port</string>
    <string>18789</string>
  </array>

  <!-- 开机自启 -->
  <key>RunAtLoad</key>
  <true/>

  <!-- 崩溃重启 -->
  <key>KeepAlive</key>
  <dict>
    <key>SuccessfulExit</key>
    <false/>
  </dict>

  <!-- 资源限制 -->
  <key>SoftResourceLimits</key>
  <dict>
    <key>NumberOfFiles</key>
    <integer>4096</integer>
  </dict>

  <!-- 日志轮转 -->
  <key>StandardOutPath</key>
  <string>/Users/username/.openclaw/logs/gateway.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/username/.openclaw/logs/gateway-error.log</string>

  <!-- 环境变量 -->
  <key>EnvironmentVariables</key>
  <dict>
    <key>OPENCLAW_GATEWAY_TOKEN</key>
    <string>oc_1a2b3c4d5e6f7g8h9i0j</string>
  </dict>
</dict>
</plist>
```

### Linux systemd 高级配置

```ini
# ~/.config/systemd/user/openclaw-gateway.service
[Unit]
Description=OpenClaw Gateway
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
ExecStart=/usr/local/bin/openclaw gateway run --port 18789
Restart=always
RestartSec=5

# 资源限制
LimitNOFILE=4096
MemoryMax=2G
CPUQuota=200%

# 环境变量
Environment="OPENCLAW_GATEWAY_TOKEN=oc_1a2b3c4d5e6f7g8h9i0j"
Environment="NODE_ENV=production"

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=openclaw-gateway

[Install]
WantedBy=default.target
```

**高级管理命令**：

```bash
# 查看详细状态
systemctl --user status openclaw-gateway -l

# 查看实时日志
journalctl --user -u openclaw-gateway -f

# 查看资源使用
systemctl --user show openclaw-gateway --property=MemoryCurrent,CPUUsageNSec

# 重新加载配置（不重启）
systemctl --user reload openclaw-gateway

# 优雅重启（等待当前请求完成）
systemctl --user reload-or-restart openclaw-gateway
```

## 技巧 8：性能优化

### 配置热重载优化

```json5
// ~/.openclaw/openclaw.json
{
  gateway: {
    reload: {
      mode: "hot",  // 仅热重载，不自动重启
      debounceMs: 500,  // 增加防抖时间（减少重载频率）
    },
  },
}
```

**优化建议**：
- 开发环境：`mode: "hybrid"`（自动重启）
- 生产环境：`mode: "hot"`（手动重启，避免意外中断）

### Session 管理优化

```json5
{
  session: {
    dmScope: "per-channel-peer",  // 推荐：每个渠道独立会话
    reset: {
      mode: "daily",  // 每日重置（减少内存占用）
      atHour: 4,  // 凌晨 4 点重置（低峰期）
      idleMinutes: 120,  // 2 小时无活动自动重置
    },
  },
}
```

### 日志优化

```json5
{
  logging: {
    level: "info",  // 生产环境：info（减少日志量）
    maxFiles: 7,  // 保留 7 天日志
    maxSize: "100m",  // 单文件最大 100MB
  },
}
```

## 技巧 9：安全加固

```json5
{

    defaults: {
      tools: {
        // 禁用高风险工具

          enabled: true,

        "camera.snap",
        "camera.clip",
        "screen.record",
        "calendar.add",
        "contacts.add",
        "reminders.add",
      ],
    },
  },
}
```

### Channel 安全配置

```json5
{
  channels: {
    whatsapp: {
      dmPolicy: "pairing",  // 需要配对码
      allowFrom: ["+1234567890"],  // 白名单
      groups: {
        "*": {
          requireMention: true,  // 群组需要 @提及
        },
      },
    },
    telegram: {
      dmPolicy: "allowlist",  // 仅白名单
      allowFrom: ["tg:123456789"],
    },
  },

```

## 技巧 11：CI/CD 集成最佳实践

### GitHub Actions 示例

```yaml
name: Deploy OpenClaw

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Install OpenClaw
        run: npm install -g openclaw

      - name: Configure OpenClaw
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          OPENCLAW_GATEWAY_TOKEN: ${{ secrets.OPENCLAW_GATEWAY_TOKEN }}
        run: |
          openclaw onboard --non-interactive \
            --auth-choice anthropic-api-key \
            --anthropic-api-key "$ANTHROPIC_API_KEY" \
            --gateway-token "$OPENCLAW_GATEWAY_TOKEN" \
            --skip-health \
            --no-install-daemon

      - name: Verify Configuration
        run: |
          openclaw doctor
          openclaw config get gateway.port

      - name: Deploy to Server
        run: |
          scp ~/.openclaw/openclaw.json user@server:/home/user/.openclaw/
          ssh user@server 'openclaw gateway restart'
```

### Docker 部署示例

```dockerfile
FROM node:22-alpine

# 安装 OpenClaw
RUN npm install -g openclaw

# 复制配置
COPY openclaw.json /root/.openclaw/openclaw.json

# 启动 Gateway
CMD ["openclaw", "gateway", "run", "--port", "18789"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  openclaw:
    build: .
    ports:
      - "18789:18789"
    environment:
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
      - OPENCLAW_GATEWAY_TOKEN=${OPENCLAW_GATEWAY_TOKEN}
    volumes:
      - ./workspace:/root/.openclaw/workspace
    restart: unless-stopped
```

## 技巧 12：隐藏功能和快捷方式

### 快速重新配置

```bash
# 重新运行向导（保留现有配置）
openclaw onboard

# 只配置特定部分
openclaw configure --section gateway
openclaw configure --section channels
openclaw configure --section models
```

### 配置快照和恢复

```bash
# 备份配置
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.backup

# 恢复配置
cp ~/.openclaw/openclaw.json.backup ~/.openclaw/openclaw.json
openclaw gateway restart

# 版本控制
git init ~/.openclaw
cd ~/.openclaw
git add openclaw.json
git commit -m "Initial config"
```

### 快速切换模型

```bash
# 临时切换模型（单次对话）
openclaw chat --model "openai/gpt-4"

# 永久切换默认模型
openclaw config set agents.defaults.model.primary "openai/gpt-4"
```
