# 核心概念 7：状态更新策略对比

> 深入理解 LangGraph 的两种状态更新策略及其性能差异

[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]
[来源: reference/search_annotated_github_01.md | 2025-2026 年最新实践]

---

## 概述

LangGraph 提供两种状态更新策略：**默认覆盖策略**（无 Annotated）和 **Reducer 策略**（有 Annotated）。理解这两种策略的区别、适用场景和性能差异，是掌握 LangGraph 状态管理的关键。

**核心区别**：
- **覆盖策略**：新值直接替换旧值
- **Reducer 策略**：新值与旧值通过 reducer 函数合并

---

## 1. 默认覆盖策略

### 1.1 基本原理

[来源: reference/context7_langgraph_01.md]

```python
from typing_extensions import TypedDict

class State(TypedDict):
    counter: int  # 无 Annotated，使用覆盖策略
    current_step: str
    result: str
```

**更新行为**：
```python
# 初始状态
state = {"counter": 0, "current_step": "init", "result": ""}

# 节点返回
update = {"counter": 5, "current_step": "step1"}

# 合并后
# state = {"counter": 5, "current_step": "step1", "result": ""}
#         ↑ 直接覆盖    ↑ 直接覆盖           ↑ 保持不变
```

### 1.2 内部实现

[来源: reference/source_annotated_02.md]

```python
from langgraph.channels.last_value import LastValue

# 无 Annotated 的字段使用 LastValue Channel
class LastValue(BaseChannel):
    """存储最后一个值"""

    def update(self, values: Sequence[Value]) -> bool:
        """更新为最后一个值"""
        if not values:
            return False
        self.value = values[-1]  # 直接覆盖
        return True

    def get(self) -> Value:
        """获取当前值"""
        return self.value
```

### 1.3 使用场景

```python
class State(TypedDict):
    # 适合覆盖策略的字段
    current_user: str        # 当前用户
    session_id: str          # 会话 ID
    current_step: int        # 当前步骤
    is_complete: bool        # 是否完成
    last_error: str | None   # 最后的错误
    timestamp: float         # 时间戳
```

**特点**：
- 单一值，不需要累积
- 每次更新都是完整的新值
- 不需要保留历史

## 2. Reducer 策略

### 2.1 基本原理

[来源: reference/context7_langgraph_01.md]

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]  # 使用 Reducer 策略
    counters: Annotated[dict, operator.or_]
```

**更新行为**：
```python
# 初始状态
state = {"messages": ["Hello"], "counters": {"a": 1}}

# 节点返回
update = {"messages": ["World"], "counters": {"b": 2}}

# 合并后
# state = {"messages": ["Hello", "World"], "counters": {"a": 1, "b": 2}}
#         ↑ operator.add 合并          ↑ operator.or_ 合并
```

### 2.2 内部实现

[来源: reference/source_annotated_02.md]

```python
from langgraph.channels.binop import BinaryOperatorAggregate

# 有 Annotated 的字段使用 BinaryOperatorAggregate Channel
class BinaryOperatorAggregate(BaseChannel):
    """使用 reducer 函数合并值"""

    def update(self, values: Sequence[Value]) -> bool:
        """使用 reducer 合并值"""
        if not values:
            return False

        # 第一个值直接赋值
        if self.value is MISSING:
            self.value = values[0]
            values = values[1:]

        # 后续值应用 reducer
        for value in values:
            self.value = self.operator(self.value, value)  # 合并

        return True
```

### 2.3 使用场景

```python
class State(TypedDict):
    # 适合 Reducer 策略的字段
    messages: Annotated[list[BaseMessage], add_messages]  # 对话历史
    tool_calls: Annotated[list[ToolCall], operator.add]  # 工具调用
    reasoning_steps: Annotated[list[str], operator.add]  # 推理步骤
    metadata: Annotated[dict, operator.or_]              # 元数据
    total_tokens: Annotated[int, operator.add]           # 总 token 数
```

**特点**：
- 需要累积的数据
- 每次更新是增量
- 需要保留历史

## 3. 策略对比

### 3.1 功能对比

| 维度 | 覆盖策略 | Reducer 策略 |
|------|----------|--------------|
| **定义方式** | `field: type` | `field: Annotated[type, reducer]` |
| **更新行为** | 直接覆盖 | 通过 reducer 合并 |
| **历史保留** | 不保留 | 保留（取决于 reducer） |
| **Channel 类型** | LastValue | BinaryOperatorAggregate |
| **适用场景** | 单一值 | 累积值 |
| **灵活性** | 低 | 高（可自定义 reducer） |

### 3.2 代码对比

```python
from typing import Annotated, TypedDict
import operator

# 示例 1：覆盖策略
class OverwriteState(TypedDict):
    counter: int

def node1(state: OverwriteState) -> dict:
    return {"counter": 10}

def node2(state: OverwriteState) -> dict:
    return {"counter": 20}

# 执行流程
# 初始: counter = 0
# node1: counter = 10 (覆盖)
# node2: counter = 20 (覆盖)
# 最终: counter = 20

# 示例 2：Reducer 策略
class ReducerState(TypedDict):
    counter: Annotated[int, operator.add]

def node1(state: ReducerState) -> dict:
    return {"counter": 10}

def node2(state: ReducerState) -> dict:
    return {"counter": 20}

# 执行流程
# 初始: counter = 0
# node1: counter = 0 + 10 = 10 (累加)
# node2: counter = 10 + 20 = 30 (累加)
# 最终: counter = 30
```

### 3.3 部分状态更新

[来源: reference/context7_langgraph_01.md]

```python
class State(TypedDict):
    messages: Annotated[list, operator.add]
    counter: int
    user_id: str

# 节点可以只返回部分字段
def node(state: State) -> dict:
    return {"messages": ["new message"]}
    # counter 和 user_id 保持不变

# 这是 TypedDict 的优势
# Pydantic 需要返回完整的模型实例
```

## 4. 性能对比

### 4.1 TypedDict vs Pydantic

[来源: reference/search_annotated_github_01.md | 2025 年基准测试]

```python
import time
from typing import Annotated
from typing_extensions import TypedDict
from pydantic import BaseModel
import operator

# TypedDict 实现
class TypedDictState(TypedDict):
    messages: Annotated[list, operator.add]
    counter: int

# Pydantic 实现
class PydanticState(BaseModel):
    messages: list
    counter: int

# 性能测试
def benchmark_typeddict():
    state = {"messages": [], "counter": 0}
    start = time.time()
    for i in range(10000):
        state = {"messages": state["messages"] + [f"msg{i}"], "counter": i}
    return time.time() - start

def benchmark_pydantic():
    state = PydanticState(messages=[], counter=0)
    start = time.time()
    for i in range(10000):
        state = PydanticState(messages=state.messages + [f"msg{i}"], counter=i)
    return time.time() - start

# 结果（2025 年基准测试）
# TypedDict: ~0.1ms per update
# Pydantic: ~1.0ms per update
# TypedDict 快 10 倍
```

### 4.2 性能差异原因

```python
# TypedDict：零开销
class State(TypedDict):
    messages: Annotated[list, operator.add]

# 运行时就是普通字典
state = {"messages": []}  # 直接操作字典

# Pydantic：验证开销
class State(BaseModel):
    messages: list

# 运行时需要验证
state = State(messages=[])  # 创建模型实例 + 验证
```

**开销来源**：
1. **类型验证**：Pydantic 每次更新都验证类型
2. **模型实例化**：创建 BaseModel 实例有开销
3. **序列化/反序列化**：Pydantic 需要转换数据

### 4.3 性能优化建议

[来源: reference/search_annotated_github_01.md]

```python
# ✅ 推荐：使用 TypedDict
class State(TypedDict):
    messages: Annotated[list, operator.add]
    metadata: dict

# ❌ 不推荐：使用 Pydantic（除非需要验证）
class State(BaseModel):
    messages: list
    metadata: dict

# 如果需要验证，只在边界处使用 Pydantic
from pydantic import BaseModel

class InputModel(BaseModel):
    """输入验证"""
    query: str
    user_id: str

class State(TypedDict):
    """内部状态（无验证）"""
    messages: Annotated[list, operator.add]
    metadata: dict

def validate_input(data: dict) -> InputModel:
    """只在输入时验证"""
    return InputModel(**data)
```

    tool_calls: Annotated[list[ToolCall], operator.add]

## 6. 完整实战示例

### 6.1 对比示例

```python
"""
完整示例：对比两种策略
"""
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
import operator

# 示例 1：覆盖策略
class OverwriteState(TypedDict):
    value: int

def increment_overwrite(state: OverwriteState) -> dict:
    return {"value": state["value"] + 1}

builder1 = StateGraph(OverwriteState)
builder1.add_node("inc1", increment_overwrite)
builder1.add_node("inc2", increment_overwrite)
builder1.add_edge(START, "inc1")
builder1.add_edge("inc1", "inc2")
builder1.add_edge("inc2", END)

graph1 = builder1.compile()
result1 = graph1.invoke({"value": 0})
print(f"覆盖策略结果: {result1['value']}")  # 1 (最后一次更新)

# 示例 2：Reducer 策略
class ReducerState(TypedDict):
    value: Annotated[int, operator.add]

def increment_reducer(state: ReducerState) -> dict:
    return {"value": 1}  # 只返回增量

builder2 = StateGraph(ReducerState)
builder2.add_node("inc1", increment_reducer)
builder2.add_node("inc2", increment_reducer)
builder2.add_edge(START, "inc1")
builder2.add_edge("inc1", "inc2")
builder2.add_edge("inc2", END)

graph2 = builder2.compile()
result2 = graph2.invoke({"value": 0})
print(f"Reducer 策略结果: {result2['value']}")  # 2 (累加)
```

### 6.2 实际应用场景

```python
"""
实际应用：对话系统
"""
from typing import Annotated, TypedDict
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

class ConversationState(TypedDict):
    """对话状态"""
    # Reducer 策略：累积对话历史
    messages: Annotated[list[BaseMessage], add_messages]

    # 覆盖策略：当前状态
    current_user: str
    session_id: str
    turn_count: int
    is_active: bool

def user_input(state: ConversationState) -> dict:
    """用户输入"""
    return {
        "messages": [HumanMessage(content=f"Hello from {state['current_user']}")],
        "turn_count": state["turn_count"] + 1
    }

def ai_response(state: ConversationState) -> dict:
    """AI 响应"""
    return {
        "messages": [AIMessage(content=f"Hi {state['current_user']}!")],
        "turn_count": state["turn_count"] + 1
    }

def should_continue(state: ConversationState) -> str:
    """判断是否继续"""
    if state["turn_count"] >= 4:
        return "end"
    return "continue"

# 构建图
builder = StateGraph(ConversationState)
builder.add_node("user", user_input)
builder.add_node("ai", ai_response)

builder.add_edge(START, "user")
builder.add_edge("user", "ai")
builder.add_conditional_edges(
    "ai",
    should_continue,
    {
        "continue": "user",
        "end": END
    }
)

graph = builder.compile()

# 执行
result = graph.invoke({
    "messages": [],
    "current_user": "Alice",
    "session_id": "session-123",
    "turn_count": 0,
    "is_active": True
})

print("\n=== 对话历史（Reducer 策略）===")
for msg in result["messages"]:
    print(f"{msg.__class__.__name__}: {msg.content}")

print(f"\n=== 当前状态（覆盖策略）===")
print(f"用户: {result['current_user']}")
print(f"会话: {result['session_id']}")
print(f"轮次: {result['turn_count']}")
print(f"活跃: {result['is_active']}")
```

### 6.3 性能测试

```python
"""
性能测试：TypedDict vs Pydantic
"""
import time
from typing import Annotated, TypedDict
from pydantic import BaseModel
import operator

# TypedDict 实现
class TypedDictState(TypedDict):
    items: Annotated[list, operator.add]
    counter: int

# Pydantic 实现
class PydanticState(BaseModel):
    items: list
    counter: int

def benchmark_typeddict(iterations: int = 10000):
    """测试 TypedDict 性能"""
    from langgraph.graph import StateGraph, START, END

    def add_item(state: TypedDictState) -> dict:
        return {"items": ["item"], "counter": state["counter"] + 1}

    builder = StateGraph(TypedDictState)
    builder.add_node("add", add_item)
    builder.add_edge(START, "add")
    builder.add_edge("add", END)
    graph = builder.compile()

    start = time.time()
    for _ in range(iterations):
        graph.invoke({"items": [], "counter": 0})
    return time.time() - start

def benchmark_pydantic(iterations: int = 10000):
    """测试 Pydantic 性能"""
    start = time.time()
    for i in range(iterations):
        state = PydanticState(items=[], counter=0)
        state = PydanticState(items=state.items + ["item"], counter=state.counter + 1)
    return time.time() - start

# 运行测试
print("=== 性能测试 ===")
typeddict_time = benchmark_typeddict(1000)
pydantic_time = benchmark_pydantic(1000)

print(f"TypedDict: {typeddict_time:.4f}s")
print(f"Pydantic: {pydantic_time:.4f}s")
print(f"TypedDict 快 {pydantic_time / typeddict_time:.2f}x")
```

## 7. 常见陷阱

### 7.1 混淆策略

```python
# ❌ 错误：期望累积但没有使用 Annotated
class State(TypedDict):
    messages: list  # 会被覆盖，不会累积

def node(state: State) -> dict:
    return {"messages": ["new message"]}

# 结果：messages 被覆盖为 ["new message"]

# ✅ 正确：使用 Annotated
class State(TypedDict):
    messages: Annotated[list, operator.add]

def node(state: State) -> dict:
    return {"messages": ["new message"]}

# 结果：messages 累积为 [..., "new message"]
```

### 7.2 返回完整状态

```python
# ❌ 错误：返回完整状态导致重复
class State(TypedDict):
    messages: Annotated[list, operator.add]

def node(state: State) -> dict:
    return {"messages": state["messages"] + ["new"]}
    # 会导致 messages 重复

# ✅ 正确：只返回增量
def node(state: State) -> dict:
    return {"messages": ["new"]}
```

### 7.3 性能陷阱

```python
# ❌ 不好：使用 Pydantic（验证开销）
class State(BaseModel):
    messages: list
    counter: int

# ✅ 好：使用 TypedDict
class State(TypedDict):
    messages: Annotated[list, operator.add]
    counter: int
```

## 8. 总结

### 核心要点

1. **两种策略**
   - 覆盖策略：直接替换，适合单一值
   - Reducer 策略：合并累积，适合历史数据

2. **选择标准**
   - 需要累积 → Reducer 策略
   - 单一值 → 覆盖策略

3. **性能优化**
   - 优先使用 TypedDict
   - 避免使用 Pydantic（除非需要验证）
   - TypedDict 比 Pydantic 快 10 倍

4. **最佳实践**
   - 混合使用两种策略
   - 细粒度状态管理
   - 只返回增量更新

### 决策树

```
需要累积数据？
├─ 是 → 使用 Annotated[type, reducer]
│  ├─ 列表 → operator.add
│  ├─ 字典 → operator.or_
│  ├─ 消息 → add_messages
│  └─ 自定义 → 自定义 reducer
└─ 否 → 使用普通类型（覆盖策略）
   └─ 单一值、当前状态、配置等
```

### 性能建议

1. **使用 TypedDict**
   - 零验证开销
   - 支持部分更新
   - 更好的性能

2. **避免深拷贝**
   - 使用浅拷贝
   - 避免 copy.deepcopy()

3. **细粒度状态**
   - 分离不同类型的数据
   - 避免状态膨胀
