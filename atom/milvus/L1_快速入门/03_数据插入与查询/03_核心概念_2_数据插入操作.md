# 核心概念 2: 数据插入操作

> **深入理解 Milvus 的数据插入机制：Insert, Upsert, BulkInsert**

---

## 概述

数据插入是向量数据库的基础操作之一。在 Milvus 中，数据插入不仅仅是简单地存储向量，还涉及 Schema 验证、分区管理、动态字段处理、以及与 Embedding Functions 的集成。本文将深入讲解 Milvus 的三种插入操作及其在 RAG 应用中的最佳实践。

### 核心概念

**Entity（实体）**: Milvus 中的数据记录，包含 Schema 中定义的所有字段
**Insert**: 标准插入操作，不检查主键重复
**Upsert**: 插入或更新操作，自动处理主键冲突
**BulkInsert**: 批量插入操作，适用于大规模数据导入

---

## Entity（实体）的概念

### 什么是 Entity？

在 Milvus 中，**Entity** 是指 Collection 中的一条数据记录，它包含 Schema 中定义的所有字段。同一个 Collection 中的所有 Entity 共享相同的 Schema。

```python
# Entity 示例
entity = {
    "id": 1,                    # 主键字段
    "document": "Milvus...",    # 标量字段
    "dense": [0.1, 0.2, ...],   # 向量字段
    "metadata": {"key": "val"}  # 动态字段（如果启用）
}
```

### Entity 的组成

| 组件 | 说明 | 示例 |
|------|------|------|
| **主键字段** | 唯一标识 Entity | `id: 1` |
| **标量字段** | 存储原始数据 | `document: "text"` |
| **向量字段** | 存储嵌入向量 | `dense: [0.1, 0.2, ...]` |
| **动态字段** | 未在 Schema 中定义的字段 | `color: "red"` |

### Schema 约束

插入 Entity 时必须满足以下约束：
1. **必须包含所有必需字段**（非 auto_id 的主键、所有非可选字段）
2. **字段类型必须匹配** Schema 定义
3. **向量维度必须正确**
4. **字符串长度不能超过** `max_length`

## Insert 操作

### 基本用法

Insert 是最基本的插入操作，将 Entity 添加到 Collection 中。

```python
from pymilvus import MilvusClient

client = MilvusClient(uri="http://localhost:19530")

# 准备数据
data = [
    {"id": 1, "document": "Milvus is a vector database", "dense": [0.1, 0.2, 0.3]},
    {"id": 2, "document": "RAG combines retrieval and generation", "dense": [0.4, 0.5, 0.6]},
    {"id": 3, "document": "Embedding Functions simplify development", "dense": [0.7, 0.8, 0.9]},
]

# 插入数据
result = client.insert(
    collection_name="my_collection",
    data=data
)

print(result)
# 输出: {'insert_count': 3, 'ids': [1, 2, 3]}
```

### Insert 参数详解

```python
client.insert(
    collection_name="my_collection",  # Collection 名称
    data=data,                        # 要插入的数据（列表或字典）
    partition_name=None,              # 可选：指定分区名称
    timeout=None                      # 可选：超时时间（秒）
)
```

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `collection_name` | str | ✅ | Collection 名称 |
| `data` | List[Dict] | ✅ | 要插入的 Entity 列表 |
| `partition_name` | str | ❌ | 分区名称（默认 `_default`） |
| `timeout` | float | ❌ | 超时时间 |

### Insert 返回值

```python
{
    'insert_count': 3,           # 成功插入的 Entity 数量
    'ids': [1, 2, 3]             # 插入的主键列表
}
```

### 重要特性

**1. 不检查主键重复**

```python
# 第一次插入
client.insert("my_collection", [{"id": 1, "document": "text1", "dense": [0.1, 0.2]}])

# 第二次插入相同主键（不会报错，会创建重复数据！）
client.insert("my_collection", [{"id": 1, "document": "text2", "dense": [0.3, 0.4]}])

# 结果：Collection 中有两条 id=1 的记录
```

**⚠️ 警告**: Insert 不检查主键重复，可能导致数据重复和查询结果不一致。如需避免重复，使用 **Upsert** 操作。

**2. 自动填充默认值**

```python
# Schema 定义
schema.add_field("id", DataType.INT64, is_primary=True)
schema.add_field("document", DataType.VARCHAR, max_length=1000)
schema.add_field("score", DataType.FLOAT, default_value=0.0)  # 有默认值

# 插入时不提供 score 字段
client.insert("my_collection", [
    {"id": 1, "document": "text"}  # score 会自动填充为 0.0
])
```

**3. 动态字段支持**

```python
# 启用动态字段
schema.enable_dynamic_field = True

# 插入包含未定义字段的数据
client.insert("my_collection", [
    {
        "id": 1,
        "document": "text",
        "dense": [0.1, 0.2],
        "color": "red",        # 动态字段
        "category": "tech"     # 动态字段
    }
])

# 动态字段存储在 $meta 字段中
```

## Upsert 操作

### 什么是 Upsert？

Upsert = **Up**date + In**sert**，即"插入或更新"操作。如果主键已存在，则更新该 Entity；如果不存在，则插入新 Entity。

### 基本用法

```python
# 第一次插入
client.upsert("my_collection", [
    {"id": 1, "document": "Original text", "dense": [0.1, 0.2]}
])

# 第二次 upsert 相同主键（会更新而非重复）
client.upsert("my_collection", [
    {"id": 1, "document": "Updated text", "dense": [0.3, 0.4]}
])

# 结果：Collection 中只有一条 id=1 的记录，内容为 "Updated text"
```

### Upsert vs Insert

| 特性 | Insert | Upsert |
|------|--------|--------|
| **主键重复检查** | ❌ 不检查 | ✅ 检查并更新 |
| **性能** | 更快 | 稍慢（需要检查） |
| **数据一致性** | 可能重复 | 保证唯一 |
| **适用场景** | 新数据批量导入 | 增量更新、数据同步 |

### 使用场景

**场景1: 增量更新文档**

```python
# RAG 应用中更新文档内容
def update_document(doc_id, new_content):
    # 使用 Upsert 确保不会创建重复记录
    client.upsert("doc_collection", [
        {
            "id": doc_id,
            "document": new_content,
            "updated_at": datetime.now().isoformat()
        }
    ])
```

**场景2: 数据同步**

```python
# 从外部系统同步数据
def sync_from_database(records):
    # 使用 Upsert 处理新增和更新
    entities = [
        {
            "id": record["id"],
            "document": record["content"],
            "metadata": record["meta"]
        }
        for record in records
    ]
    client.upsert("sync_collection", entities)
```

**场景3: 用户反馈更新**

```python
# 根据用户反馈更新文档质量分数
def update_quality_score(doc_id, score):
    # 获取现有文档
    existing = client.query(
        "doc_collection",
        filter=f"id == {doc_id}",
        output_fields=["document", "dense"]
    )[0]

    # 更新分数
    existing["quality_score"] = score
    client.upsert("doc_collection", [existing])
```

## BulkInsert 操作

### 什么是 BulkInsert？

BulkInsert 是专门为大规模数据导入设计的操作，支持从文件（JSON, Parquet）批量导入数据到 Milvus。

### 适用场景

- **初始数据导入**: 首次将大量数据导入 Milvus
- **离线数据处理**: 预处理好的数据批量导入
- **数据迁移**: 从其他系统迁移数据

### 基本用法

```python
from pymilvus import utility

# 准备数据文件（JSON 或 Parquet）
# data.json:
# [
#   {"id": 1, "document": "text1", "dense": [0.1, 0.2]},
#   {"id": 2, "document": "text2", "dense": [0.3, 0.4]},
#   ...
# ]

# 执行 BulkInsert
task_id = utility.do_bulk_insert(
    collection_name="my_collection",
    files=["data.json"],  # 数据文件路径
    partition_name=None   # 可选：指定分区
)

# 检查导入状态
state = utility.get_bulk_insert_state(task_id)
print(f"状态: {state.state_name}, 进度: {state.progress}%")
```

### BulkInsert vs Insert

| 特性 | Insert | BulkInsert |
|------|--------|------------|
| **数据来源** | 内存中的数据 | 文件（JSON, Parquet） |
| **数据量** | 适合小批量 | 适合大批量（百万级+） |
| **性能** | 一般 | 高（并行处理） |
| **内存占用** | 高（数据在内存） | 低（流式处理） |
| **适用场景** | 实时插入 | 离线批量导入 |

## 批量插入策略

### 批次大小选择

```python
# ❌ 低效：逐条插入
for entity in entities:
    client.insert("my_collection", [entity])  # 每次一条

# ✅ 高效：批量插入
batch_size = 1000
for i in range(0, len(entities), batch_size):
    batch = entities[i:i+batch_size]
    client.insert("my_collection", batch)
```

### 推荐批次大小

| 数据规模 | 推荐批次大小 | 说明 |
|----------|--------------|------|
| < 1万 | 100-500 | 小规模，快速插入 |
| 1万-10万 | 500-1000 | 中等规模，平衡性能 |
| 10万-100万 | 1000-5000 | 大规模，优化吞吐量 |
| > 100万 | 使用 BulkInsert | 超大规模，离线导入 |

### 批量插入示例

```python
def batch_insert(collection_name, entities, batch_size=1000):
    """批量插入数据"""
    total = len(entities)
    inserted = 0

    for i in range(0, total, batch_size):
        batch = entities[i:i+batch_size]
        result = client.insert(collection_name, batch)
        inserted += result['insert_count']
        print(f"进度: {inserted}/{total} ({inserted/total*100:.1f}%)")

    return inserted

# 使用示例
entities = [...]  # 10000 条数据
batch_insert("my_collection", entities, batch_size=1000)
```

## 分区插入

### 什么是分区？

分区（Partition）是 Collection 的逻辑划分，可以将数据按业务逻辑分组存储，提升查询效率。

### 分区插入示例

```python
# 创建分区
client.create_partition(
    collection_name="my_collection",
    partition_name="tech_docs"
)

# 插入到指定分区
client.insert(
    collection_name="my_collection",
    data=[{"id": 1, "document": "tech content", "dense": [0.1, 0.2]}],
    partition_name="tech_docs"  # 指定分区
)

# 如果不指定分区，数据会插入到 _default 分区
client.insert(
    collection_name="my_collection",
    data=[{"id": 2, "document": "general content", "dense": [0.3, 0.4]}]
    # 默认插入到 _default 分区
)
```

### 分区的优势

1. **查询加速**: 只搜索特定分区，减少扫描范围
2. **数据管理**: 按业务逻辑组织数据
3. **灵活删除**: 可以删除整个分区

### RAG 应用中的分区策略

```python
# 按文档类型分区
partitions = {
    "technical": ["tech_doc_1.txt", "tech_doc_2.txt"],
    "marketing": ["marketing_1.txt", "marketing_2.txt"],
    "legal": ["legal_1.txt", "legal_2.txt"]
}

for partition_name, docs in partitions.items():
    # 创建分区
    client.create_partition("doc_collection", partition_name)

    # 插入到对应分区
    entities = [process_doc(doc) for doc in docs]
    client.insert("doc_collection", entities, partition_name=partition_name)

# 查询时只搜索相关分区
results = client.search(
    collection_name="doc_collection",
    data=[query_vector],
    partition_names=["technical"],  # 只搜索技术文档
    limit=5
)
```

## 动态字段

### 什么是动态字段？

动态字段允许你插入 Schema 中未定义的字段，这些字段会存储在保留字段 `$meta` 中。

### 启用动态字段

```python
# 创建 Schema 时启用
schema = client.create_schema(enable_dynamic_field=True)

# 或在创建 Collection 时启用
client.create_collection(
    collection_name="my_collection",
    schema=schema,
    enable_dynamic_field=True
)
```

### 使用动态字段

```python
# 插入包含动态字段的数据
client.insert("my_collection", [
    {
        "id": 1,
        "document": "Milvus documentation",
        "dense": [0.1, 0.2, 0.3],
        # 以下是动态字段（未在 Schema 中定义）
        "author": "Milvus Team",
        "category": "database",
        "tags": ["vector", "AI"],
        "created_at": "2026-02-22"
    }
])

# 查询时可以访问动态字段
results = client.query(
    collection_name="my_collection",
    filter="id == 1",
    output_fields=["document", "author", "category"]  # 包含动态字段
)
print(results[0]["author"])  # 输出: "Milvus Team"
```

### 动态字段的限制

1. **不能用于索引**: 动态字段不能建立索引
2. **不能用于向量搜索**: 只能用于标量过滤
3. **性能影响**: 查询动态字段比静态字段稍慢

### 适用场景

✅ **适合使用动态字段**:
- 元数据字段（作者、标签、时间戳）
- 不确定的扩展字段
- 快速原型开发

❌ **不适合使用动态字段**:
- 需要高性能查询的字段
- 需要建立索引的字段
- 核心业务字段

## 错误处理

### 常见错误

**1. Schema 不匹配**

```python
# 错误：缺少必需字段
try:
    client.insert("my_collection", [
        {"id": 1}  # 缺少 document 和 dense 字段
    ])
except Exception as e:
    print(f"错误: {e}")
    # 错误: field 'document' is required
```

**2. 向量维度不匹配**

```python
# 错误：向量维度不正确
try:
    client.insert("my_collection", [
        {"id": 1, "document": "text", "dense": [0.1, 0.2]}  # 应该是 1536 维
    ])
except Exception as e:
    print(f"错误: {e}")
    # 错误: vector dimension mismatch
```

**3. 主键类型错误**

```python
# 错误：主键类型不匹配
try:
    client.insert("my_collection", [
        {"id": "string_id", "document": "text", "dense": [0.1, 0.2]}  # id 应该是 INT64
    ])
except Exception as e:
    print(f"错误: {e}")
    # 错误: field 'id' type mismatch
```

### 错误处理最佳实践

```python
def safe_insert(collection_name, entities, max_retries=3):
    """安全插入，带重试和错误处理"""
    for attempt in range(max_retries):
        try:
            result = client.insert(collection_name, entities)
            return result
        except Exception as e:
            error_msg = str(e)

            # 处理不同类型的错误
            if "dimension mismatch" in error_msg:
                print("错误：向量维度不匹配，请检查数据")
                return None
            elif "field" in error_msg and "required" in error_msg:
                print("错误：缺少必需字段，请检查 Schema")
                return None
            elif "connection" in error_msg:
                if attempt < max_retries - 1:
                    print(f"连接错误，重试 {attempt + 1}/{max_retries}...")
                    time.sleep(2 ** attempt)  # 指数退避
                    continue
                else:
                    print("连接失败，已达最大重试次数")
                    return None
            else:
                print(f"未知错误: {e}")
                return None

    return None
```

## RAG 应用场景

### 场景1: 文档导入

```python
def import_documents(documents):
    """导入文档到 Milvus"""
    entities = []

    for doc in documents:
        entity = {
            "id": doc["id"],
            "document": doc["content"],
            # Embedding Function 会自动生成 dense 字段
            "source": doc["source"],
            "created_at": doc["timestamp"]
        }
        entities.append(entity)

    # 批量插入
    batch_size = 1000
    for i in range(0, len(entities), batch_size):
        batch = entities[i:i+batch_size]
        client.insert("doc_collection", batch)
```

### 场景2: 增量更新

```python
def update_document(doc_id, new_content):
    """更新文档内容"""
    # 使用 Upsert 确保不会创建重复记录
    client.upsert("doc_collection", [
        {
            "id": doc_id,
            "document": new_content,
            "updated_at": datetime.now().isoformat()
        }
    ])
```

### 场景3: 实时数据流

```python
def process_stream(stream):
    """处理实时数据流"""
    buffer = []
    batch_size = 100

    for item in stream:
        entity = {
            "id": item["id"],
            "document": item["text"],
            "timestamp": item["timestamp"]
        }
        buffer.append(entity)

        # 达到批次大小时插入
        if len(buffer) >= batch_size:
            client.insert("stream_collection", buffer)
            buffer = []

    # 插入剩余数据
    if buffer:
        client.insert("stream_collection", buffer)
```

## 性能优化

### 1. 批量插入

```python
# ❌ 低效
for entity in entities:
    client.insert("my_collection", [entity])

# ✅ 高效
client.insert("my_collection", entities)
```

### 2. 并发插入

```python
from concurrent.futures import ThreadPoolExecutor

def insert_batch(batch):
    return client.insert("my_collection", batch)

# 并发插入多个批次
batches = [entities[i:i+1000] for i in range(0, len(entities), 1000)]
with ThreadPoolExecutor(max_workers=4) as executor:
    results = list(executor.map(insert_batch, batches))
```

### 3. 预分配主键

```python
# 使用 auto_id 让 Milvus 自动生成主键
schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)

# 插入时不需要提供 id
client.insert("my_collection", [
    {"document": "text1", "dense": [0.1, 0.2]},
    {"document": "text2", "dense": [0.3, 0.4]}
])
```

## 总结

### 核心要点

1. **Insert**: 标准插入，不检查重复，适合新数据批量导入
2. **Upsert**: 插入或更新，保证主键唯一，适合增量更新
3. **BulkInsert**: 批量导入，适合大规模离线数据导入
4. **批量策略**: 推荐批次大小 1000-5000，避免逐条插入
5. **分区管理**: 按业务逻辑分区，提升查询效率
6. **动态字段**: 灵活扩展，但不能用于索引

### 选择指南

| 场景 | 推荐操作 | 原因 |
|------|----------|------|
| 首次导入大量数据 | BulkInsert | 性能最优 |
| 实时插入新数据 | Insert | 简单高效 |
| 更新已有数据 | Upsert | 避免重复 |
| 数据同步 | Upsert | 处理新增和更新 |
| 小批量插入 | Insert (batch) | 平衡性能 |
