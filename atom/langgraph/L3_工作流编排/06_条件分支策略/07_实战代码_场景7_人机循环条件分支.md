# 实战代码 - 场景7：人机循环条件分支

> 使用 interrupt + Command 实现人工审批分支，掌握 LangGraph 中人机协作的条件路由模式

---

## 场景描述

**场景**：构建一个订单处理系统，根据订单金额进行风险评估，低风险订单自动处理，高风险订单暂停等待人工审批，审批结果再决定后续路由。

**核心挑战**：在自动化工作流中，某些决策必须由人类做出。系统需要在特定条件下"暂停"，等待人工输入后再继续执行。

**与核心概念5的关系**：核心概念5讲解了 Command 的 `goto`、`update`、`resume` 等属性，本场景将 `interrupt()` 和 `Command(resume=...)` 组合成完整的人机循环审批流程。

[来源: 03_核心概念_5_Command高级路由控制.md]

**流程图**：

```
START
  ↓
assess_risk（风险评估）
  ↓
route_by_risk（条件路由）
  ├── risk == "low"  → auto_process（自动处理）→ END
  └── risk != "low"  → human_review（人工审批，interrupt 暂停）
                          ↓
                       Command(resume=...) 恢复
                          ↓
                       route_approval（审批结果路由）
                          ├── approved → execute_order（执行订单）→ END
                          └── rejected → cancel_order（取消订单）→ END
```

**关键机制**：

| 机制 | 说明 |
|------|------|
| `interrupt()` | 在节点内暂停图执行，返回提示信息给调用方 |
| `Command(resume=...)` | 调用方传入人工决策，恢复图执行 |
| `InMemorySaver` | checkpointer，保存暂停时的完整状态 |
| 条件边 | 根据风险等级和审批结果做路由决策 |

## 完整代码

```python
"""
条件分支策略 - 实战场景7：人机循环条件分支
演示：订单审批系统

核心模式：
- 条件路由：根据风险等级分流（自动处理 vs 人工审批）
- interrupt()：高风险订单暂停等待人工决策
- Command(resume=...)：人工审批后恢复执行
- 二次条件路由：根据审批结果决定执行或取消

前置条件：
- pip install langgraph
- 必须启用 checkpointer（InMemorySaver）
"""

from typing import TypedDict, Literal
from langgraph.graph import StateGraph, START, END
from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import InMemorySaver


# ============================================================
# 第一部分：状态定义
# ============================================================

class OrderState(TypedDict):
    """订单处理状态

    字段说明：
    - order_id: 订单编号
    - amount: 订单金额
    - risk_level: 风险等级（low / medium / high）
    - approval_status: 审批状态（pending / approved / rejected）
    - result: 最终处理结果
    """
    order_id: str
    amount: float
    risk_level: str
    approval_status: str
    result: str


# ============================================================
# 第二部分：节点函数
# ============================================================

def assess_risk(state: OrderState) -> dict:
    """风险评估节点

    根据订单金额判断风险等级：
    - 金额 > 10000：高风险
    - 金额 > 1000：中风险
    - 其他：低风险

    实际项目中这里会调用风控模型或规则引擎，
    这里用金额阈值模拟，保证代码可直接运行。
    """
    amount = state["amount"]

    if amount > 10000:
        risk = "high"
    elif amount > 1000:
        risk = "medium"
    else:
        risk = "low"

    print(f"[风险评估] 订单 {state['order_id']}，金额 {amount}，风险等级: {risk}")
    return {"risk_level": risk}


def auto_process(state: OrderState) -> dict:
    """自动处理节点：低风险订单直接通过"""
    result = f"订单 {state['order_id']}（¥{state['amount']}）自动处理完成"
    print(f"[自动处理] {result}")
    return {"result": result, "approval_status": "approved"}


def human_review(state: OrderState) -> dict:
    """人工审批节点

    核心机制：
    1. 调用 interrupt() 暂停图执行
    2. interrupt() 的参数会作为提示信息返回给调用方
    3. 调用方通过 Command(resume=...) 传入审批决策
    4. interrupt() 返回 resume 的值，节点继续执行

    注意：interrupt() 之前的代码在暂停时执行一次，
    恢复后从 interrupt() 返回值处继续，不会重新执行前面的代码。
    """
    print(f"[人工审批] 订单 {state['order_id']} 需要人工审批，暂停等待...")

    # interrupt() 暂停执行，返回提示信息
    # 调用方会收到这个字典作为中断信息
    decision = interrupt({
        "type": "approval_request",
        "order_id": state["order_id"],
        "amount": state["amount"],
        "risk_level": state["risk_level"],
        "message": f"请审批订单 {state['order_id']}，金额 ¥{state['amount']}，风险等级: {state['risk_level']}",
        "options": ["approved", "rejected"],
    })

    # 恢复后，decision 就是 Command(resume=...) 传入的值
    status = decision.get("status", "rejected")
    reason = decision.get("reason", "无")
    print(f"[人工审批] 收到审批结果: {status}，原因: {reason}")

    return {"approval_status": status}


def execute_order(state: OrderState) -> dict:
    """执行订单节点：审批通过后执行"""
    result = f"订单 {state['order_id']}（¥{state['amount']}）审批通过，已执行"
    print(f"[执行] {result}")
    return {"result": result}


def cancel_order(state: OrderState) -> dict:
    """取消订单节点：审批拒绝后取消"""
    result = f"订单 {state['order_id']}（¥{state['amount']}）审批被拒绝，已取消"
    print(f"[取消] {result}")
    return {"result": result}


# ============================================================
# 第三部分：路由函数
# ============================================================

def route_by_risk(state: OrderState) -> Literal["auto_process", "human_review"]:
    """风险路由：低风险自动处理，其他需要人工审批

    使用 Literal 类型注解，LangGraph 自动推断可能的路径，
    图可视化只显示这两条边。
    """
    if state["risk_level"] == "low":
        return "auto_process"
    return "human_review"


def route_approval(state: OrderState) -> Literal["execute_order", "cancel_order"]:
    """审批结果路由：通过则执行，拒绝则取消"""
    if state["approval_status"] == "approved":
        return "execute_order"
    return "cancel_order"


# ============================================================
# 第四部分：构建图
# ============================================================

def build_order_graph():
    """构建订单审批图

    关键点：
    1. 两层条件路由：风险路由 + 审批结果路由
    2. human_review 节点内部使用 interrupt() 暂停
    3. 必须传入 checkpointer，否则 interrupt 无法工作
    """
    builder = StateGraph(OrderState)

    # 添加节点
    builder.add_node("assess_risk", assess_risk)
    builder.add_node("auto_process", auto_process)
    builder.add_node("human_review", human_review)
    builder.add_node("execute_order", execute_order)
    builder.add_node("cancel_order", cancel_order)

    # 入口边
    builder.add_edge(START, "assess_risk")

    # 第一层条件路由：根据风险等级
    builder.add_conditional_edges("assess_risk", route_by_risk)

    # 自动处理 → 结束
    builder.add_edge("auto_process", END)

    # 第二层条件路由：根据审批结果
    builder.add_conditional_edges("human_review", route_approval)

    # 执行/取消 → 结束
    builder.add_edge("execute_order", END)
    builder.add_edge("cancel_order", END)

    # 编译时传入 checkpointer（interrupt 必需）
    checkpointer = InMemorySaver()
    return builder.compile(checkpointer=checkpointer)


# ============================================================
# 第五部分：测试
# ============================================================

if __name__ == "__main__":

    graph = build_order_graph()

    # ===== 测试1：低风险订单（自动处理，不触发 interrupt） =====
    print("=" * 60)
    print("测试1：低风险订单 - 自动处理")
    print("=" * 60)

    config1 = {"configurable": {"thread_id": "order-001"}}
    result1 = graph.invoke(
        {
            "order_id": "ORD-001",
            "amount": 500.0,
            "risk_level": "",
            "approval_status": "pending",
            "result": "",
        },
        config1,
    )
    print(f"最终结果: {result1['result']}")
    print()

    # ===== 测试2：高风险订单（触发 interrupt，需要人工审批） =====
    print("=" * 60)
    print("测试2：高风险订单 - 人工审批（通过）")
    print("=" * 60)

    config2 = {"configurable": {"thread_id": "order-002"}}

    # 第一次 invoke：图会在 human_review 节点暂停
    result2 = graph.invoke(
        {
            "order_id": "ORD-002",
            "amount": 50000.0,
            "risk_level": "",
            "approval_status": "pending",
            "result": "",
        },
        config2,
    )
    # result2 此时是暂停时的状态快照（interrupt 之前的状态）
    print(f"图已暂停，等待人工审批...")
    print(f"当前状态: risk_level={result2['risk_level']}, approval_status={result2['approval_status']}")

    # 查看中断信息（可选）
    state_snapshot = graph.get_state(config2)
    for task in state_snapshot.tasks:
        if hasattr(task, "interrupts") and task.interrupts:
            for intr in task.interrupts:
                print(f"中断信息: {intr.value}")
    print()

    # 模拟人工审批：通过
    print(">>> 人工操作：审批通过")
    result2_resumed = graph.invoke(
        Command(resume={"status": "approved", "reason": "金额合理，客户信用良好"}),
        config2,
    )
    print(f"最终结果: {result2_resumed['result']}")
    print()

    # ===== 测试3：高风险订单（审批拒绝） =====
    print("=" * 60)
    print("测试3：高风险订单 - 人工审批（拒绝）")
    print("=" * 60)

    config3 = {"configurable": {"thread_id": "order-003"}}

    # 第一次 invoke：暂停
    result3 = graph.invoke(
        {
            "order_id": "ORD-003",
            "amount": 99999.0,
            "risk_level": "",
            "approval_status": "pending",
            "result": "",
        },
        config3,
    )
    print(f"图已暂停，等待人工审批...")
    print()

    # 模拟人工审批：拒绝
    print(">>> 人工操作：审批拒绝")
    result3_resumed = graph.invoke(
        Command(resume={"status": "rejected", "reason": "金额异常，疑似欺诈"}),
        config3,
    )
    print(f"最终结果: {result3_resumed['result']}")
    print()

    # ===== 测试4：中风险订单（也需要人工审批） =====
    print("=" * 60)
    print("测试4：中风险订单 - 人工审批（通过）")
    print("=" * 60)

    config4 = {"configurable": {"thread_id": "order-004"}}

    result4 = graph.invoke(
        {
            "order_id": "ORD-004",
            "amount": 5000.0,
            "risk_level": "",
            "approval_status": "pending",
            "result": "",
        },
        config4,
    )
    print(f"图已暂停，等待人工审批...")
    print()

    print(">>> 人工操作：审批通过")
    result4_resumed = graph.invoke(
        Command(resume={"status": "approved", "reason": "常规订单"}),
        config4,
    )
    print(f"最终结果: {result4_resumed['result']}")
```

## 核心机制详解

### 1. interrupt() 的工作原理

```
第一次 invoke（输入初始状态）
  ↓
  assess_risk → route_by_risk → human_review
  ↓
  human_review 内部调用 interrupt({...})
  ↓
  图暂停，状态保存到 checkpointer
  ↓
  invoke 返回当前状态快照（interrupt 之前的状态）

第二次 invoke（传入 Command(resume=...)）
  ↓
  从 checkpointer 恢复状态
  ↓
  interrupt() 返回 resume 的值
  ↓
  human_review 继续执行（处理审批结果）
  ↓
  route_approval → execute_order 或 cancel_order → END
```

### 2. 为什么必须要 checkpointer？

`interrupt()` 暂停图执行后，整个图的状态（包括每个节点的输出、当前执行位置）需要持久化保存。下次 `invoke` 时，图从 checkpointer 恢复到暂停点继续执行。

没有 checkpointer，`interrupt()` 会抛出异常：

```python
# 错误示例：没有 checkpointer
graph = builder.compile()  # 没传 checkpointer
graph.invoke(...)  # interrupt() 会报错
```

### 3. thread_id 的作用

每个 `thread_id` 对应一个独立的执行上下文。不同订单使用不同的 `thread_id`，它们的状态互不干扰：

```python
config1 = {"configurable": {"thread_id": "order-001"}}  # 订单1的上下文
config2 = {"configurable": {"thread_id": "order-002"}}  # 订单2的上下文
```

### 4. 条件分支 + interrupt 的组合模式

本场景展示了两层条件路由的嵌套：

| 层级 | 路由函数 | 条件 | 目标 |
|------|---------|------|------|
| 第一层 | `route_by_risk` | `risk_level == "low"` | `auto_process` |
| 第一层 | `route_by_risk` | `risk_level != "low"` | `human_review`（含 interrupt） |
| 第二层 | `route_approval` | `approval_status == "approved"` | `execute_order` |
| 第二层 | `route_approval` | `approval_status != "approved"` | `cancel_order` |

---

## 注意事项

1. **checkpointer 是必需的**：使用 `interrupt()` 时必须在 `compile()` 时传入 checkpointer，否则会报错
2. **thread_id 必须唯一**：每个独立的工作流实例需要不同的 `thread_id`
3. **interrupt 参数是提示信息**：`interrupt({...})` 的参数会返回给调用方，用于展示审批界面
4. **resume 值就是 interrupt 的返回值**：`Command(resume=X)` 中的 `X` 会成为 `interrupt()` 的返回值
5. **状态不可变原则**：`interrupt()` 之前的代码不会在恢复时重新执行，所以不要在 interrupt 之前做有副作用的操作（如写数据库）

## 扩展思路

- **多级审批**：在 `human_review` 后增加"主管审批"节点，金额超过阈值需要二次审批
- **超时处理**：结合外部定时器，如果人工审批超时则自动拒绝
- **审批历史**：使用 `Annotated[list, operator.add]` 记录每次审批的决策和原因
- **Web 集成**：将 interrupt 信息推送到前端审批页面，用户点击按钮后调用 `Command(resume=...)`
