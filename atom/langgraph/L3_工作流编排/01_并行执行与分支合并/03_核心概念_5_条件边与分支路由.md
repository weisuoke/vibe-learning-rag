# 03_核心概念_5 - 条件边与分支路由

> LangGraph 动态决策的核心机制：通过条件边实现运行时路由和分支控制

---

## 概念定义

**条件边（Conditional Edge）是 LangGraph 中根据状态动态决定下一个执行节点的机制，通过 `add_conditional_edges` 方法和条件函数实现运行时路由，支持返回节点名称、Send 对象或它们的混合列表。**

条件边使得工作流可以：
1. 根据状态动态选择执行路径
2. 实现复杂的分支逻辑
3. 动态创建并行任务
4. 混合使用静态路由和动态并行

这是 LangGraph 实现智能决策和灵活工作流的关键特性。

---

## `add_conditional_edges` 方法

### 方法签名

```python
def add_conditional_edges(
    source: str,                          # 源节点名称
    path: Callable,                       # 条件函数
    path_map: Optional[Dict[str, str]] = None,  # 路径映射（可选）
    then: Optional[str] = None            # 后续节点（可选）
) -> None
```

### 参数说明

#### 1. `source` - 源节点

**类型**: `str`

**作用**: 指定从哪个节点出发的条件边

**示例**:
```python
builder.add_conditional_edges("router", route_function)
```

#### 2. `path` - 条件函数

**类型**: `Callable[[State], Union[str, List[str], Send, List[Send]]]`

**作用**: 根据状态返回下一个节点或 Send 对象

**返回值类型**:
- `str`: 单个节点名称
- `List[str]`: 多个节点名称（并行执行）
- `Send`: 单个 Send 对象
- `List[Send]`: 多个 Send 对象（并行执行）
- `List[Union[str, Send]]`: 混合列表

#### 3. `path_map` - 路径映射（可选）

**类型**: `Dict[str, str]`

**作用**: 将条件函数返回的字符串映射到实际节点名称

**示例**:
```python
def route(state):
    return "continue" if state["count"] < 5 else "end"

builder.add_conditional_edges(
    "node_a",
    route,
    {
        "continue": "node_b",
        "end": END
    }
)
```

#### 4. `then` - 后续节点（可选）

**类型**: `str`

**作用**: 条件边执行完毕后的统一后续节点

---

## 条件函数的编写规范

### 基本规范

**函数签名**:
```python
def condition_function(state: State) -> Union[str, List[str], Send, List[Send]]:
    # 根据状态返回路由决策
    pass
```

**核心要求**:
1. 接收状态作为唯一参数
2. 返回值必须是有效的节点名称或 Send 对象
3. 函数应该是纯函数（无副作用）
4. 避免修改状态

### 返回类型详解

#### 类型1: 返回单个节点名称

```python
from typing import Literal
from typing_extensions import TypedDict

class State(TypedDict):
    count: int

def simple_route(state: State) -> Literal["node_a", "node_b"]:
    """简单的二分支路由"""
    if state["count"] < 5:
        return "node_a"
    else:
        return "node_b"

builder.add_conditional_edges("router", simple_route)
```

**适用场景**: 简单的分支决策

#### 类型2: 返回多个节点名称（并行）

```python
def parallel_route(state: State) -> List[str]:
    """返回多个节点，并行执行"""
    return ["node_a", "node_b", "node_c"]

builder.add_conditional_edges("router", parallel_route)
```

**适用场景**: 静态并行执行多个节点

#### 类型3: 返回 Send 对象列表（动态并行）

```python
from langgraph.types import Send

def dynamic_route(state: State) -> List[Send]:
    """动态创建并行任务"""
    subjects = state["subjects"]
    return [Send("process", {"subject": s}) for s in subjects]

builder.add_conditional_edges("router", dynamic_route, ["process"])
```

**适用场景**: Map-Reduce 模式，动态并行度

#### 类型4: 混合返回（Send + 节点名称）

```python
def mixed_route(state: State) -> List[Union[str, Send]]:
    """混合使用 Send 和节点名称"""
    results = []

    # 动态并行任务
    for item in state["items"]:
        results.append(Send("process_item", {"item": item}))

    # 静态节点
    results.append("logger")

    return results

builder.add_conditional_edges("router", mixed_route, ["process_item", "logger"])
```

**适用场景**: 复杂的混合路由逻辑

## 完整代码示例

### 示例1: 基础条件路由

```python
from typing import Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 状态定义
class State(TypedDict):
    count: int
    message: str

# 节点定义
def increment(state: State):
    """递增计数器"""
    new_count = state["count"] + 1
    print(f"Count: {new_count}")
    return {"count": new_count}

def finish(state: State):
    """完成节点"""
    print(f"Finished with count: {state['count']}")
    return {"message": "Done!"}

# 条件函数
def should_continue(state: State) -> Literal["increment", "finish"]:
    """决定是否继续递增"""
    if state["count"] < 5:
        return "increment"
    else:
        return "finish"

# 构建图
builder = StateGraph(State)
builder.add_node("increment", increment)
builder.add_node("finish", finish)

builder.add_edge(START, "increment")
builder.add_conditional_edges("increment", should_continue)
builder.add_edge("finish", END)

graph = builder.compile()

# 执行
result = graph.invoke({"count": 0, "message": ""})
print(f"Final result: {result}")
```

**输出**:
```
Count: 1
Count: 2
Count: 3
Count: 4
Count: 5
Finished with count: 5
Final result: {'count': 5, 'message': 'Done!'}
```

### 示例2: 动态并行路由（Map-Reduce）

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
import operator

# 状态定义
class OverallState(TypedDict):
    subjects: list[str]
    jokes: Annotated[list[str], operator.add]

class JokeState(TypedDict):
    subject: str

# 节点定义
def generate_topics(state: OverallState):
    """生成主题列表"""
    print("Generating topics...")
    return {"subjects": ["cats", "dogs", "birds"]}

def generate_joke(state: JokeState):
    """为单个主题生成笑话"""
    subject = state["subject"]
    print(f"Generating joke about {subject}")
    joke = f"Why did the {subject} cross the road? To get to the other side!"
    return {"jokes": [joke]}

def summarize(state: OverallState):
    """汇总所有笑话"""
    print(f"\nGenerated {len(state['jokes'])} jokes:")
    for i, joke in enumerate(state["jokes"], 1):
        print(f"{i}. {joke}")
    return {}

# 条件函数：返回 Send 对象列表
def continue_to_jokes(state: OverallState):
    """为每个主题创建一个并行任务"""
    return [Send("generate_joke", {"subject": s}) for s in state["subjects"]]

# 构建图
builder = StateGraph(OverallState)
builder.add_node("generate_topics", generate_topics)
builder.add_node("generate_joke", generate_joke)
builder.add_node("summarize", summarize)

builder.add_edge(START, "generate_topics")
builder.add_conditional_edges("generate_topics", continue_to_jokes, ["generate_joke"])
builder.add_edge("generate_joke", "summarize")
builder.add_edge("summarize", END)

graph = builder.compile()

# 执行
result = graph.invoke({"subjects": [], "jokes": []})
```

**输出**:
```
Generating topics...
Generating joke about cats
Generating joke about dogs
Generating joke about birds

Generated 3 jokes:
1. Why did the cats cross the road? To get to the other side!
2. Why did the dogs cross the road? To get to the other side!
3. Why did the birds cross the road? To get to the other side!
```

### 示例3: 混合路由（Send + 节点名称）

```python
from typing import Annotated, Union
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
import operator

# 状态定义
class State(TypedDict):
    tasks: list[str]
    results: Annotated[list[str], operator.add]
    log: Annotated[list[str], operator.add]

# 节点定义
def prepare(state: State):
    """准备任务"""
    print("Preparing tasks...")
    return {"tasks": ["task1", "task2", "task3"]}

def process_task(state: State):
    """处理单个任务"""
    task = state.get("task", "unknown")
    print(f"Processing {task}")
    return {"results": [f"Result of {task}"]}

def logger(state: State):
    """记录日志"""
    print("Logging execution...")
    return {"log": ["Execution logged"]}

def finalize(state: State):
    """完成"""
    print(f"\nCompleted {len(state['results'])} tasks")
    print(f"Logs: {state['log']}")
    return {}

# 条件函数：混合返回 Send 和节点名称
def route_tasks(state: State) -> list[Union[Send, str]]:
    """混合路由：动态任务 + 静态日志节点"""
    routes = []

    # 为每个任务创建 Send 对象
    for task in state["tasks"]:
        routes.append(Send("process_task", {"task": task}))

    # 添加静态日志节点
    routes.append("logger")

    return routes

# 构建图
builder = StateGraph(State)
builder.add_node("prepare", prepare)
builder.add_node("process_task", process_task)
builder.add_node("logger", logger)
builder.add_node("finalize", finalize)

builder.add_edge(START, "prepare")
builder.add_conditional_edges("prepare", route_tasks, ["process_task", "logger"])
builder.add_edge("process_task", "finalize")
builder.add_edge("logger", "finalize")
builder.add_edge("finalize", END)

graph = builder.compile()

# 执行
result = graph.invoke({"tasks": [], "results": [], "log": []})
```

**输出**:
```
Preparing tasks...
Processing task1
Processing task2
Processing task3
Logging execution...

Completed 3 tasks
Logs: ['Execution logged']
```

### 示例4: 复杂分支决策

```python
from typing import Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 状态定义
class State(TypedDict):
    user_type: str
    score: int
    message: str

# 节点定义
def classify_user(state: State):
    """分类用户"""
    print("Classifying user...")
    return {"user_type": "premium", "score": 85}

def premium_flow(state: State):
    """高级用户流程"""
    print("Premium user flow")
    return {"message": "Welcome, premium user!"}

def standard_flow(state: State):
    """标准用户流程"""
    print("Standard user flow")
    return {"message": "Welcome, standard user!"}

def trial_flow(state: State):
    """试用用户流程"""
    print("Trial user flow")
    return {"message": "Welcome, trial user!"}

def error_flow(state: State):
    """错误处理流程"""
    print("Error flow")
    return {"message": "Error: Invalid user type"}

# 条件函数：多分支决策
def route_by_user_type(state: State) -> Literal["premium", "standard", "trial", "error"]:
    """根据用户类型和分数路由"""
    user_type = state.get("user_type", "")
    score = state.get("score", 0)

    if user_type == "premium" and score >= 80:
        return "premium"
    elif user_type == "standard" or (user_type == "premium" and score < 80):
        return "standard"
    elif user_type == "trial":
        return "trial"
    else:
        return "error"

# 构建图
builder = StateGraph(State)
builder.add_node("classify", classify_user)
builder.add_node("premium", premium_flow)
builder.add_node("standard", standard_flow)
builder.add_node("trial", trial_flow)
builder.add_node("error", error_flow)

builder.add_edge(START, "classify")
builder.add_conditional_edges("classify", route_by_user_type)
builder.add_edge("premium", END)
builder.add_edge("standard", END)
builder.add_edge("trial", END)
builder.add_edge("error", END)

graph = builder.compile()

# 执行
result = graph.invoke({"user_type": "", "score": 0, "message": ""})
print(f"\nResult: {result['message']}")
```

## 高级用法

### 1. 使用 path_map 简化路由

```python
def simple_condition(state: State) -> str:
    """返回简单的字符串标识"""
    if state["count"] < 5:
        return "continue"
    else:
        return "stop"

builder.add_conditional_edges(
    "node_a",
    simple_condition,
    {
        "continue": "node_b",  # 映射到实际节点
        "stop": END            # 映射到 END
    }
)
```

### 2. 条件边 + then 参数

```python
def route(state: State):
    """路由到多个并行节点"""
    return ["node_b", "node_c"]

builder.add_conditional_edges(
    "node_a",
    route,
    then="node_d"  # 所有并行节点完成后执行 node_d
)
```

### 3. 动态决定并行度

```python
def adaptive_route(state: State):
    """根据数据量动态决定并行度"""
    items = state["items"]

    if len(items) <= 3:
        # 少量数据：串行处理
        return "process_serial"
    else:
        # 大量数据：并行处理
        return [Send("process_parallel", {"item": item}) for item in items]

builder.add_conditional_edges("router", adaptive_route, ["process_serial", "process_parallel"])
```

### 4. 嵌套条件路由

```python
def outer_route(state: State):
    """外层路由"""
    if state["stage"] == "preprocessing":
        return "preprocess"
    else:
        return "main_process"

def inner_route(state: State):
    """内层路由（在 main_process 之后）"""
    if state["needs_validation"]:
        return "validate"
    else:
        return END

builder.add_conditional_edges("start", outer_route)
builder.add_conditional_edges("main_process", inner_route)
```

## 最佳实践

### 1. 条件函数设计原则

**保持简单**:
```python
# ✅ 好：简单清晰
def route(state: State) -> str:
    return "node_a" if state["count"] < 5 else "node_b"

# ❌ 差：过于复杂
def route(state: State) -> str:
    if state["count"] < 5:
        if state["flag"]:
            if state["nested"]:
                return "node_a"
            else:
                return "node_b"
        else:
            return "node_c"
    else:
        return "node_d"
```

**使用类型提示**:
```python
from typing import Literal

def route(state: State) -> Literal["node_a", "node_b", END]:
    """明确的返回类型"""
    if state["count"] < 5:
        return "node_a"
    elif state["count"] < 10:
        return "node_b"
    else:
        return END
```

### 2. 错误处理

```python
def safe_route(state: State) -> str:
    """带错误处理的路由函数"""
    try:
        if "count" not in state:
            return "error_handler"

        count = state["count"]
        if count < 0:
            return "error_handler"
        elif count < 5:
            return "node_a"
        else:
            return "node_b"
    except Exception as e:
        print(f"Routing error: {e}")
        return "error_handler"
```

### 3. 日志记录

```python
def logged_route(state: State) -> str:
    """带日志的路由函数"""
    count = state["count"]

    if count < 5:
        print(f"Routing to node_a (count={count})")
        return "node_a"
    else:
        print(f"Routing to node_b (count={count})")
        return "node_b"
```

### 4. 避免副作用

```python
# ❌ 差：修改状态
def bad_route(state: State) -> str:
    state["count"] += 1  # 不要在条件函数中修改状态！
    return "node_a"

# ✅ 好：纯函数
def good_route(state: State) -> str:
    return "node_a" if state["count"] < 5 else "node_b"
```

## 常见误区

### 误区1: 条件函数返回无效节点

```python
# ❌ 错误：返回未定义的节点
def route(state: State) -> str:
    return "nonexistent_node"  # 运行时错误！

# ✅ 正确：确保返回的节点已定义
def route(state: State) -> str:
    return "node_a"  # node_a 必须已通过 add_node 添加
```

### 误区2: 忘记在 path_map 中映射所有可能的返回值

```python
# ❌ 错误：缺少 "stop" 的映射
def route(state: State) -> str:
    return "continue" if state["count"] < 5 else "stop"

builder.add_conditional_edges(
    "node_a",
    route,
    {"continue": "node_b"}  # 缺少 "stop" 的映射！
)

# ✅ 正确：映射所有可能的返回值
builder.add_conditional_edges(
    "node_a",
    route,
    {
        "continue": "node_b",
        "stop": END
    }
)
```

### 误区3: Send 对象的目标节点未声明

```python
# ❌ 错误：未声明 Send 的目标节点
def route(state: State):
    return [Send("process", {"item": i}) for i in state["items"]]

builder.add_conditional_edges("router", route)  # 缺少目标节点声明！

# ✅ 正确：声明 Send 的目标节点
builder.add_conditional_edges("router", route, ["process"])
```

### 误区4: 混合返回类型不一致

```python
# ❌ 错误：返回类型不一致
def route(state: State):
    if state["parallel"]:
        return ["node_a", "node_b"]  # 返回列表
    else:
        return "node_c"  # 返回字符串

# ✅ 正确：统一返回类型
def route(state: State):
    if state["parallel"]:
        return ["node_a", "node_b"]
    else:
        return ["node_c"]  # 也返回列表
```

## 与其他概念的关系

### 条件边 vs 普通边

| 特性 | 普通边 | 条件边 |
|------|--------|--------|
| 路由方式 | 静态 | 动态 |
| 决策时机 | 编译时 | 运行时 |
| 灵活性 | 低 | 高 |
| 性能 | 更快 | 稍慢 |
| 适用场景 | 固定流程 | 动态决策 |

### 条件边 + Send API

```python
# 条件边决定是否并行，Send API 创建并行任务
def smart_route(state: State):
    items = state["items"]

    if len(items) > 10:
        # 大量数据：并行处理
        return [Send("process", {"item": i}) for i in items]
    else:
        # 少量数据：串行处理
        return "process_serial"
```

## 总结

**条件边（Conditional Edge）是 LangGraph 实现动态决策的核心机制**，它通过：

1. **`add_conditional_edges` 方法**：定义条件路由
2. **条件函数**：根据状态返回路由决策
3. **灵活的返回类型**：支持节点名称、Send 对象或混合列表
4. **path_map 映射**：简化路由逻辑

**核心优势**:
- 运行时动态决策
- 支持复杂分支逻辑
- 与 Send API 结合实现动态并行
- 灵活的路由策略

**适用场景**:
- 智能 Agent 路由
- 动态工作流编排
- 条件循环
- 错误处理和恢复
- Map-Reduce 模式

**最佳实践**:
- 保持条件函数简单
- 使用类型提示
- 避免副作用
- 完整的错误处理
- 清晰的日志记录
