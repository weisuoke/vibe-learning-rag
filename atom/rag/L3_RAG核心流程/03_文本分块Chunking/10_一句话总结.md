# 一句话总结

**文本分块（Chunking）是将长文档按策略切分成适合检索的小片段的技术，通过控制块大小和切分边界来平衡检索精度与上下文完整性，是 RAG 系统中连接文档加载与向量化的关键环节，直接决定检索质量和最终答案效果。**

---

## 关键词拆解

| 关键词 | 含义 |
|--------|------|
| **按策略切分** | 不是随意切分，而是有方法论（固定大小、递归字符、语义分块等） |
| **适合检索的小片段** | 大小适中（300-1000字符），语义完整，便于向量化 |
| **平衡精度与完整性** | 块太小精度高但上下文不全，块太大上下文全但噪音多 |
| **关键环节** | 位于文档加载之后、向量化之前，承上启下 |
| **直接决定** | 分块质量是 RAG 效果的基础，垃圾进垃圾出 |

---

## 记忆口诀

```
分块三要素：大小、重叠、切分点
分块三策略：固定、递归、语义
分块三权衡：精度、完整、成本
分块三步骤：选策略、调参数、做评估
```

## 速查配置

```python
# 复制即用的通用配置
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\n\n", "\n", "。", ".", "！", "？", " ", ""]
)

chunks = splitter.split_text(your_text)
```
