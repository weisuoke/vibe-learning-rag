# Agent类型与选择 - 概览

> 选对 Agent 类型，事半功倍 —— 实用选择指南

---

## 知识点概述

在 LangChain 的 Agent 系统中，选择合适的 Agent 类型是构建高效 AI 应用的关键决策。不同的 Agent 类型在推理方式、工具调用机制、模型兼容性上各有特点，选对类型可以显著提升系统的可靠性、降低成本、加快开发速度。

本知识点深入讲解 LangChain 中三大主流 Agent 类型（OpenAI Functions、ReAct、Structured Chat），以及 2026 年推荐的统一创建方式（`create_agent()`），帮助你根据实际场景做出最优选择，并掌握从旧 API 迁移到新 API 的方法。

---

## 为什么这个知识点重要？

### 对 AI Agent 开发的价值

1. **避免选型错误导致的返工**
   - 错误的 Agent 类型会导致工具调用失败、成本激增、性能低下
   - 提前了解各类型特点，一次选对，避免后期重构
   - 节省数周的调试和优化时间

2. **最大化模型能力利用**
   - OpenAI Functions Agent 充分利用函数调用能力（高可靠性）
   - ReAct Agent 适配开源模型（无需函数调用支持）
   - Structured Chat Agent 处理复杂工具参数（多输入场景）
   - 选对类型 = 发挥模型最大潜力

3. **控制成本与性能平衡**
   - OpenAI Functions: 中等成本，高可靠性
   - ReAct: 较高成本（更多 token），高灵活性
   - Structured Chat: 中等成本，支持复杂工具
   - 根据预算和需求做出权衡

4. **拥抱 2026 年最佳实践**
   - 传统 `AgentType` 枚举已弃用（0.1.0 → 1.0 移除）
   - `create_agent()` 成为推荐的统一 API
   - 掌握现代 API，避免使用过时方法
   - 为未来的 LangChain 版本做好准备

### 实际应用场景

- **客服机器人**：OpenAI Functions Agent（高可靠性，结构化工具调用）
- **数据分析 Agent**：Structured Chat Agent（复杂 SQL 参数，多输入工具）
- **开源模型应用**：ReAct Agent（Llama、Mistral 等不支持函数调用）
- **快速原型开发**：`create_agent()` 统一 API（自动选择最佳类型）
- **企业级系统**：根据模型能力和工具复杂度动态选择类型

## 三大 Agent 类型简介

LangChain 提供了三种主流的 Agent 类型，它们在推理方式、工具调用机制、适用场景上各有侧重。

### 1. OpenAI Functions Agent（推荐首选）

**核心特点**：
- 使用模型的原生函数调用能力（Function Calling）
- 结构化的工具调用，可靠性高
- 适合简单到中等复杂度的工具参数
- 支持 OpenAI、Anthropic 等主流模型

**推理方式**：
```
用户输入 → LLM 分析 → 生成函数调用 JSON → 执行工具 → 返回结果
```

**适用场景**：
- 使用支持函数调用的模型（OpenAI gpt-4, gpt-3.5-turbo, Anthropic Claude）
- 工具参数相对简单（1-3 个参数，无复杂嵌套）
- 需要高可靠性和稳定性的生产环境
- 预算充足，追求最佳用户体验

**创建方式**：
```python
from langchain.agents import create_openai_functions_agent, AgentExecutor

agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
```

**来源**：[源码分析 | agent_types.py] + [Context7 文档 | OpenAI Functions Agent]

---

### 2. ReAct Agent（开源模型首选）

**核心特点**：
- 基于 Prompt 的推理-行动循环（Reasoning and Acting）
- 不依赖模型的函数调用能力
- 推理过程显式可见（Thought → Action → Observation）
- 适合任何 LLM，包括开源模型

**推理方式**：
```
用户输入 → Thought (思考) → Action (选择工具) → Observation (观察结果) → 循环 → Final Answer
```

**适用场景**：
- 使用开源模型（Llama、Mistral、Qwen 等）
- 需要显式推理过程（调试、可解释性）
- 工具调用逻辑复杂，需要多步推理
- 预算有限，使用本地模型

**创建方式**：
```python
from langchain.agents import create_react_agent, AgentExecutor

agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
```

**来源**：[源码分析 | agent_types.py] + [Context7 文档 | ReAct Agent]

---

### 3. Structured Chat Agent（复杂工具专用）

**核心特点**：
- 针对聊天模型优化的结构化 Agent
- 支持多输入工具（工具参数复杂、嵌套结构）
- 结合了函数调用和灵活性
- 适合复杂业务逻辑

**推理方式**：
```
用户输入 → LLM 分析 → 生成结构化工具调用 → 执行工具（支持多参数） → 返回结果
```

**适用场景**：
- 工具需要多个输入参数（如：发送邮件需要 to, subject, body, attachments）
- 工具参数有嵌套结构（如：复杂的 JSON 配置）
- 使用支持函数调用的聊天模型
- 业务逻辑复杂，需要精细控制

**创建方式**：
```python
from langchain.agents import create_structured_chat_agent, AgentExecutor

agent = create_structured_chat_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
```

**来源**：[源码分析 | agent_types.py] + [Context7 文档 | Structured Chat Agent]

## 2026 年推荐方式：create_agent()

### 统一的 Agent 创建接口

LangChain 在 2026 年推出了 `create_agent()` 统一 API，它会根据模型能力自动选择最佳的 Agent 类型，简化了开发流程。

**核心优势**：
- **自动选择**：根据模型是否支持函数调用，自动选择 OpenAI Functions 或 ReAct
- **统一接口**：一个函数适配所有模型（OpenAI、Anthropic、开源模型）
- **简化代码**：无需手动判断模型能力和选择 Agent 类型
- **面向未来**：LangChain 官方推荐的现代 API

**使用方式**：
```python
from langchain.agents import create_agent

# 自动选择最佳 Agent 类型
agent = create_agent(
    model,
    tools,
    system_prompt="You are a helpful assistant.",
)

# 直接调用（无需 AgentExecutor）
result = agent.invoke({"messages": [("user", "What's the weather?")]})
```

**自动选择逻辑**：
```
模型支持函数调用？
├─ 是 → 使用 OpenAI Functions Agent（高可靠性）
└─ 否 → 使用 ReAct Agent（兼容性）
```

**来源**：[Context7 文档 | create_agent API]

## 实际应用场景示例

### 场景 1：客服机器人（推荐：OpenAI Functions）

**需求**：
- 查询订单状态
- 修改收货地址
- 申请退款

**选择理由**：
- 工具参数简单（订单号、地址字符串）
- 需要高可靠性（客户服务不能出错）
- 使用 OpenAI gpt-4（支持函数调用）

**代码示例**：
```python
from langchain.agents import create_openai_functions_agent

tools = [查询订单工具, 修改地址工具, 申请退款工具]
agent = create_openai_functions_agent(llm, tools, prompt)
```

---

### 场景 2：数据分析 Agent（推荐：Structured Chat）

**需求**：
- 执行复杂 SQL 查询（多个 WHERE 条件、JOIN、GROUP BY）
- 生成图表（多个配置参数：图表类型、颜色、标签）
- 导出报告（格式、字段选择、过滤条件）

**选择理由**：
- 工具参数复杂（SQL 查询有多个条件，图表配置有嵌套结构）
- 需要精细控制参数验证
- 使用 OpenAI gpt-4（支持函数调用）

**代码示例**：
```python
from langchain.agents import create_structured_chat_agent

tools = [SQL查询工具, 图表生成工具, 报告导出工具]
agent = create_structured_chat_agent(llm, tools, prompt)
```

---

### 场景 3：开源模型应用（推荐：ReAct）

**需求**：
- 使用本地部署的 Llama 3 模型
- 调用搜索引擎、计算器、天气 API
- 成本敏感，不使用商业 API

**选择理由**：
- Llama 3 不支持函数调用
- ReAct 基于 Prompt，兼容任何 LLM
- 推理过程显式，便于调试

**代码示例**：
```python
from langchain.agents import create_react_agent
from langchain_community.llms import Ollama

llm = Ollama(model="llama3")
tools = [搜索工具, 计算器工具, 天气工具]
agent = create_react_agent(llm, tools, prompt)
```

---

### 场景 4：快速原型开发（推荐：create_agent）

**需求**：
- 快速验证 Agent 想法
- 不确定最终使用哪个模型
- 希望代码简洁，易于切换模型

**选择理由**：
- `create_agent()` 自动选择最佳类型
- 统一接口，切换模型无需改代码
- 面向未来的 API

**代码示例**：
```python
from langchain.agents import create_agent

# 自动适配 OpenAI、Anthropic、开源模型
agent = create_agent(model, tools, system_prompt="...")
result = agent.invoke({"messages": [("user", "帮我查询天气")]})
```

## 从旧 API 迁移到新 API

### 旧方式（已弃用）

```python
from langchain.agents import AgentType, initialize_agent

# ❌ 使用 AgentType 枚举（已在 0.1.0 标记为 deprecated）
agent = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.OPENAI_FUNCTIONS,  # 将在 1.0 移除
    verbose=True
)
```

**问题**：
- `AgentType` 枚举将在 LangChain 1.0 版本移除
- `initialize_agent()` 不够灵活，难以自定义
- 不支持最新的 Agent 特性

**来源**：[源码分析 | agent_types.py Deprecation 警告]

---

### 新方式（推荐）

#### 方式 1：使用 create_*_agent() 函数

```python
from langchain.agents import create_openai_functions_agent, AgentExecutor

# ✅ 使用专用创建函数
agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# 调用
result = agent_executor.invoke({"input": "查询天气"})
```

#### 方式 2：使用 create_agent() 统一 API（2026 推荐）

```python
from langchain.agents import create_agent

# ✅ 统一 API，自动选择最佳类型
agent = create_agent(
    model,
    tools,
    system_prompt="You are a helpful assistant.",
)

# 直接调用（无需 AgentExecutor）
result = agent.invoke({"messages": [("user", "查询天气")]})
```

## 2025-2026 最新趋势

### 技术演进

1. **create_agent() 成为主流**
   - 统一 API 简化开发流程
   - 自动选择最佳 Agent 类型
   - LangChain 官方推荐的现代方式

2. **AgentType 枚举被弃用**
   - 0.1.0 版本标记为 deprecated
   - 1.0 版本将完全移除
   - 迁移到函数式 API（create_*_agent）

3. **Tool Calling Agent 崛起**
   - 新一代工具调用机制
   - 更好的跨模型兼容性
   - 未来可能取代 OpenAI Functions

4. **开源模型能力提升**
   - Llama 3、Qwen 2.5 等模型推理能力增强
   - ReAct Agent 在开源模型上表现更好
   - 本地部署成为可行选择

### 常见陷阱（2025-2026）

1. **继续使用 AgentType 枚举**
   - 问题：将在 LangChain 1.0 移除，导致代码失效
   - 解决：立即迁移到 `create_*_agent()` 或 `create_agent()`

2. **错误选择 Agent 类型**
   - 问题：开源模型使用 OpenAI Functions（不支持函数调用）
   - 解决：开源模型必须使用 ReAct Agent

3. **忽略工具参数复杂度**
   - 问题：复杂工具使用 OpenAI Functions（参数解析失败）
   - 解决：复杂工具使用 Structured Chat Agent

4. **过度依赖 verbose=True 调试**
   - 问题：生产环境日志过多，影响性能
   - 解决：使用 LangSmith 进行追踪和调试

## 总结

Agent 类型选择是 LangChain Agent 开发的关键决策。通过掌握三大主流类型（OpenAI Functions、ReAct、Structured Chat）的特点和适用场景，以及 2026 年推荐的 `create_agent()` 统一 API，你可以根据实际需求做出最优选择，快速构建高效、可靠的 AI Agent。

**关键要点**：
- 优先使用 `create_agent()` 统一 API（自动选择）
- OpenAI Functions Agent 适合生产环境（高可靠性）
- ReAct Agent 适合开源模型（无需函数调用支持）
- Structured Chat Agent 适合复杂工具参数（多输入/嵌套）
- 立即迁移旧 API（AgentType 枚举将在 1.0 移除）
- 根据模型能力、工具复杂度、成本预算做出权衡
