# 核心概念 1：过滤表达式语法

> **目标**：掌握 Milvus 标量过滤表达式的完整语法体系

---

## 概述

过滤表达式是 Milvus 标量过滤的核心，通过类似 SQL WHERE 子句的语法实现精准的元数据筛选。

**核心价值**：
- 在向量检索基础上添加业务规则
- 支持复杂的条件组合
- 提供类型安全的查询能力

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

---

## 1. 比较操作符

### 1.1 等于与不等于

**语法**：
```python
# 等于
"field == value"

# 不等于
"field != value"
```

**示例**：
```python
# 字符串比较
"city == '北京'"
"status != 'deleted'"

# 数值比较
"age == 25"
"price != 0"

# 布尔比较
"active == true"
"is_vip != false"
```

**注意事项**：
- 字符串值需要使用单引号或双引号
- 布尔值使用 `true` 或 `false`（小写）
- 数值不需要引号

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

### 1.2 大于与小于

**语法**：
```python
# 大于
"field > value"

# 小于
"field < value"

# 大于等于
"field >= value"

# 小于等于
"field <= value"
```

**示例**：
```python
# 年龄范围
"age > 18"
"age >= 25"
"age < 60"
"age <= 65"

# 价格范围
"price > 100"
"price >= 100.5"
"price < 1000"
"price <= 999.99"

# 时间戳
"created_at > 1640995200"  # Unix timestamp
"updated_at >= 1640995200"
```

**适用类型**：
- 数值类型：INT8, INT16, INT32, INT64, FLOAT, DOUBLE
- 字符串类型：VARCHAR（字典序比较）

[来源: reference/source_expression.md | CompareExpr.cpp]

## 2. 范围操作符

### 2.1 IN 操作符

**语法**：
```python
"field IN [value1, value2, value3, ...]"
```

**示例**：
```python
# 数值集合
"id IN [1, 2, 3, 4, 5]"
"age IN [25, 30, 35, 40]"

# 字符串集合
"city IN ['北京', '上海', '深圳']"
"status IN ['active', 'pending', 'approved']"

# 混合使用
"category IN ['electronics', 'books', 'clothing']"
```

**性能优化**：
- 集合元素数量建议 < 1000
- 大集合考虑使用索引
- 可以与其他条件组合

[来源: reference/context7_pymilvus_scalar_filtering.md | PyMilvus 官方文档]

### 2.2 LIKE 操作符

**语法**：
```python
# 前缀匹配
"field LIKE 'prefix%'"

# 后缀匹配
"field LIKE '%suffix'"

# 包含匹配
"field LIKE '%substring%'"
```

**示例**：
```python
# 前缀匹配
"name LIKE 'John%'"  # John, Johnny, Johnson
"email LIKE 'admin%'"  # admin@example.com, admin123@...

# 后缀匹配
"email LIKE '%@gmail.com'"  # xxx@gmail.com
"filename LIKE '%.pdf'"  # document.pdf, report.pdf

# 包含匹配
"description LIKE '%urgent%'"  # contains "urgent"
"title LIKE '%Python%'"  # contains "Python"
```

**注意事项**：
- 仅支持 VARCHAR 类型
- `%` 表示任意字符（0个或多个）
- 包含匹配性能较差，建议使用索引

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

## 3. NULL 操作符

### 3.1 IS NULL

**语法**：
```python
"field IS NULL"
```

**示例**：
```python
# 查找缺失值
"description IS NULL"
"phone IS NULL"
"optional_field IS NULL"
```

**语义**：
- 字段值为 NULL（未定义）
- 空字符串 `""` 不等于 NULL
- 数值 0 不等于 NULL

### 3.2 IS NOT NULL

**语法**：
```python
"field IS NOT NULL"
```

**示例**：
```python
# 查找已定义值
"description IS NOT NULL"
"email IS NOT NULL"
"verified_at IS NOT NULL"
```

**组合使用**：
```python
# 有描述且价格合理
"description IS NOT NULL AND price > 0"

# 有邮箱或有电话
"email IS NOT NULL OR phone IS NOT NULL"
```

[来源: reference/context7_milvus_boolean_operators.md | Milvus 官方文档]

## 4. 表达式优先级

### 4.1 优先级顺序

```
1. 括号 ()
2. 比较操作符 (==, !=, >, <, >=, <=)
3. NOT
4. AND
5. OR
```

### 4.2 示例

**不使用括号**（可能产生歧义）：
```python
# 可能的理解1：(age > 25 AND city == '北京') OR city == '上海'
# 可能的理解2：age > 25 AND (city == '北京' OR city == '上海')
"age > 25 AND city == '北京' OR city == '上海'"
```

**使用括号**（明确优先级）：
```python
# 明确的理解
"age > 25 AND (city == '北京' OR city == '上海')"
```

**复杂示例**：
```python
# 年龄在25-35之间，且（在北京或上海），且有VIP
"(age >= 25 AND age <= 35) AND (city == '北京' OR city == '上海') AND is_vip == true"
```

## 5. 数据类型支持

### 5.1 标量类型

| 类型 | 支持的操作符 | 示例 |
|------|-------------|------|
| **INT8/16/32/64** | ==, !=, >, <, >=, <=, IN | `age > 25` |
| **FLOAT/DOUBLE** | ==, !=, >, <, >=, <=, IN | `price >= 99.99` |
| **BOOL** | ==, != | `active == true` |
| **VARCHAR** | ==, !=, >, <, >=, <=, IN, LIKE | `name LIKE 'John%'` |

### 5.2 JSON 类型

```python
# JSON 字段访问
"metadata['category'] == 'electronics'"
"product['price'] < 1000"

# 嵌套访问
"user['profile']['age'] > 25"
"data['nested']['value'] != null"
```

### 5.3 ARRAY 类型

```python
# 数组包含
"ARRAY_CONTAINS(tags, 'tech')"
"ARRAY_CONTAINS(categories, 'electronics')"

# 数组长度
"ARRAY_LENGTH(items) > 5"
"ARRAY_LENGTH(history) >= 10"
```

## 7. 性能优化

### 7.1 索引优化

**为常用字段创建索引**：
```python
# 创建标量索引
index_params.add_index(
    field_name="age",
    index_type="INVERTED"
)

index_params.add_index(
    field_name="city",
    index_type="INVERTED"
)
```

**性能提升**：
- 无索引：O(n) 线性扫描
- 有索引：O(log n) 索引查找
- 实际提升：30倍

[来源: reference/search_web_results.md | Milvus 2.6 官方博客]

### 7.2 查询优化

**选择性高的条件放前面**：
```python
# 好的写法（选择性高的条件在前）
"stock > 0 AND price >= 100 AND price <= 500"

# 不好的写法（选择性低的条件在前）
"price >= 100 AND price <= 500 AND stock > 0"
```

**避免复杂的 LIKE 模式**：
```python
# 好的写法（前缀匹配）
"name LIKE 'John%'"

# 不好的写法（包含匹配，性能差）
"name LIKE '%John%'"
```

## 8. 常见错误

### 8.1 语法错误

**错误示例**：
```python
# 错误：字符串没有引号
"city == 北京"  # ❌

# 正确
"city == '北京'"  # ✅

# 错误：布尔值大写
"active == True"  # ❌

# 正确
"active == true"  # ✅
```

### 8.2 类型错误

**错误示例**：
```python
# 错误：字符串与数值比较
"age == '25'"  # ❌

# 正确
"age == 25"  # ✅

# 错误：数值使用 LIKE
"price LIKE '100%'"  # ❌

# 正确
"price >= 100 AND price < 200"  # ✅
```

### 8.3 逻辑错误

**错误示例**：
```python
# 错误：矛盾条件
"age > 30 AND age < 25"  # ❌ 永远为 false

# 错误：冗余条件
"age > 25 OR age > 30"  # ❌ 等价于 age > 25

# 正确
"age >= 25 AND age <= 35"  # ✅
```

## 9. 类比理解

### 前端开发类比

**JavaScript 数组过滤**：
```javascript
// JavaScript
const filtered = users.filter(u =>
    u.age > 25 &&
    ['北京', '上海'].includes(u.city) &&
    u.active === true
);

// Milvus
filter = "age > 25 AND city IN ['北京', '上海'] AND active == true"
```

### SQL 类比

**SQL WHERE 子句**：
```sql
-- SQL
SELECT * FROM users
WHERE age > 25
  AND city IN ('北京', '上海')
  AND active = true;

-- Milvus
filter = "age > 25 AND city IN ['北京', '上海'] AND active == true"
```
