# 最小可用知识

掌握以下内容，就能开始使用 AgentExecutor 执行循环。

> **核心原则：** 20% 的 API 解决 80% 的场景。AgentExecutor 的本质就是一个 while 循环，
> 你只需要理解"思考→行动→观察"这个模式，就能驾驭绝大多数 Agent 场景。

---

## 4.1 理解 ReAct 循环：Think → Act → Observe

**这是你需要记住的第一件事：AgentExecutor 的核心就是一个 while 循环。**

```
用户提问: "北京今天天气怎么样？"
  │
  ▼
┌─────────────────────────────────────────────┐
│              AgentExecutor while 循环         │
│                                              │
│  迭代 1:                                     │
│    Think  → LLM 思考: "我需要查天气"          │
│    Act    → 调用工具: search("北京天气")       │
│    Observe → 工具返回: "晴，25°C"             │
│                                              │
│  迭代 2:                                     │
│    Think  → LLM 思考: "我已经有答案了"         │
│    Finish → 返回最终答案                      │
│                                              │
└─────────────────────────────────────────────┘
  │
  ▼
最终回答: "北京今天天气晴朗，气温 25°C。"
```

### 三个核心数据类型

| 类型 | 含义 | 类比 |
|------|------|------|
| `AgentAction` | Agent 决定执行的动作（工具名 + 输入） | 一条待办指令 |
| `AgentFinish` | Agent 决定结束，返回最终答案 | 任务完成报告 |
| `AgentStep` | 一次动作的完整记录（动作 + 观察结果） | 一条执行日志 |

### 循环的伪代码

```python
# AgentExecutor 的核心逻辑（简化版）
intermediate_steps = []
iterations = 0

while iterations < max_iterations:
    # 1. Think: LLM 根据输入 + 历史步骤做决策
    decision = agent.plan(input, intermediate_steps)

    # 2. 如果 LLM 决定结束 → 返回最终答案
    if isinstance(decision, AgentFinish):
        return decision.return_values

    # 3. Act: 执行 LLM 选择的工具
    observation = tools[decision.tool].run(decision.tool_input)

    # 4. Observe: 记录这一步的结果
    intermediate_steps.append(AgentStep(decision, observation))

    iterations += 1

# 超出迭代限制 → 强制停止
return "Agent stopped due to iteration limit."
```

**关键洞察：** 每次迭代，LLM 都能看到之前所有步骤的结果（intermediate_steps），
这就是 Agent 能"逐步推理"的原因——它在不断积累上下文。

## 4.2 创建 AgentExecutor 的最简代码

**完整可运行的最小示例：**

```python
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate

# 1. 定义工具
@tool
def search(query: str) -> str:
    """Search for information about a topic."""
    return f"搜索结果：关于 '{query}' 的最新信息..."

@tool
def calculate(expression: str) -> str:
    """Calculate a mathematical expression."""
    try:
        return str(eval(expression))
    except Exception as e:
        return f"计算错误: {e}"

# 2. 创建 LLM
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# 3. 创建 Prompt（必须包含 agent_scratchpad）
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个有用的助手，可以搜索信息和做计算。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

# 4. 创建 Agent（决策器）
agent = create_tool_calling_agent(llm, [search, calculate], prompt)

# 5. 创建 AgentExecutor（执行引擎）
executor = AgentExecutor(agent=agent, tools=[search, calculate])

# 6. 运行
result = executor.invoke({"input": "帮我搜索 LangChain 是什么"})
print(result["output"])
```

### 各组件的职责

```
┌──────────────────────────────────────────────────┐
│                  AgentExecutor                    │
│                                                   │
│  ┌─────────────┐    ┌──────────────────────────┐ │
│  │   Agent      │    │       Tools              │ │
│  │  (决策器)    │    │      (工具箱)             │ │
│  │             │    │                           │ │
│  │  LLM + Prompt│   │  search()  calculate()   │ │
│  │  → 决定做什么│    │  → 实际执行操作           │ │
│  └─────────────┘    └──────────────────────────┘ │
│                                                   │
│  while 循环: Agent 决策 → Tool 执行 → 结果反馈    │
└──────────────────────────────────────────────────┘
```

| 组件 | 作用 | 必需？ |
|------|------|--------|
| `llm` | 大语言模型，负责推理 | 是 |
| `prompt` | 提示词模板，必须含 `agent_scratchpad` | 是 |
| `tools` | 工具列表，Agent 可调用的能力 | 是（至少一个） |
| `agent` | 决策器，LLM + Prompt + Tools 的组合 | 是 |
| `executor` | 执行引擎，驱动 while 循环 | 是 |

> **为什么 prompt 必须有 `agent_scratchpad`？** 这是 AgentExecutor 注入中间步骤的位置。
> 每次迭代，之前的 AgentAction + Observation 都会被格式化后放入 `agent_scratchpad`，
> 让 LLM 看到"我之前做了什么、得到了什么结果"。

## 4.3 关键参数：max_iterations 和 handle_parsing_errors

### max_iterations：防止无限循环

```python
# 默认值是 15，大多数场景够用
executor = AgentExecutor(
    agent=agent,
    tools=[search, calculate],
    max_iterations=15,  # 最多迭代 15 次
)

# 简单任务可以调低，节省 token
executor = AgentExecutor(
    agent=agent,
    tools=[search],
    max_iterations=5,  # 简单搜索任务，5 次足够
)

# 复杂推理任务可以调高
executor = AgentExecutor(
    agent=agent,
    tools=[search, calculate],
    max_iterations=25,  # 多步推理，需要更多迭代
)
```

**超出限制时的行为（early_stopping_method）：**

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| `"force"`（默认） | 直接返回固定字符串 "Agent stopped due to iteration limit." | 大多数场景 |
| `"generate"` | 让 LLM 基于已有步骤生成一个总结性答案 | 需要优雅降级时 |

```python
# 优雅降级：即使超时也给出部分答案
executor = AgentExecutor(
    agent=agent,
    tools=[search],
    max_iterations=10,
    early_stopping_method="generate",  # 超限时让 LLM 总结已有信息
)
```

### handle_parsing_errors：处理 LLM 输出格式错误

LLM 有时会输出不符合预期格式的内容（比如工具名拼错、参数格式不对）。
`handle_parsing_errors` 决定如何处理这种情况。

```python
# 方式 1：布尔值（推荐）
executor = AgentExecutor(
    agent=agent,
    tools=[search],
    handle_parsing_errors=True,  # 将错误信息反馈给 LLM，让它自我修正
)

# 方式 2：自定义错误提示
executor = AgentExecutor(
    agent=agent,
    tools=[search],
    handle_parsing_errors="输出格式有误，请严格按照工具调用格式重试。",
)

# 方式 3：自定义处理函数
def my_error_handler(error):
    return f"解析失败: {str(error)[:100]}。请检查工具名和参数格式。"

executor = AgentExecutor(
    agent=agent,
    tools=[search],
    handle_parsing_errors=my_error_handler,
)
```

**处理流程：**

```
LLM 输出 → 解析失败（OutputParserException）
  │
  ├─ handle_parsing_errors=False → 直接抛异常，程序崩溃 ❌
  ├─ handle_parsing_errors=True  → 错误信息作为 observation 反馈给 LLM → LLM 自我修正 ✅
  ├─ handle_parsing_errors="提示" → 自定义提示作为 observation → LLM 重试 ✅
  └─ handle_parsing_errors=fn()  → 函数返回值作为 observation → LLM 重试 ✅
```

> **生产环境建议：** 始终设置 `handle_parsing_errors=True`。LLM 的输出不是 100% 可控的，
> 没有错误处理的 Agent 在生产环境中会频繁崩溃。

### max_execution_time：时间限制

```python
# 限制总执行时间（秒）
executor = AgentExecutor(
    agent=agent,
    tools=[search],
    max_iterations=15,
    max_execution_time=30.0,  # 最多运行 30 秒
)
```

两个限制条件是 OR 关系——任一条件触发就停止循环。

## 4.4 查看执行过程：return_intermediate_steps

**调试 Agent 最重要的参数。** 开启后可以看到 Agent 每一步的思考和行动。

```python
executor = AgentExecutor(
    agent=agent,
    tools=[search, calculate],
    return_intermediate_steps=True,  # 返回中间步骤
    verbose=True,                    # 打印执行过程到控制台
)

result = executor.invoke({"input": "LangChain 是什么？它的 GitHub star 数是多少？"})

# 最终答案
print("最终答案:", result["output"])

# 查看每一步的详细过程
for i, step in enumerate(result["intermediate_steps"]):
    action, observation = step
    print(f"\n--- 步骤 {i + 1} ---")
    print(f"  工具: {action.tool}")
    print(f"  输入: {action.tool_input}")
    print(f"  结果: {observation[:100]}...")
```

**输出示例：**

```
--- 步骤 1 ---
  工具: search
  输入: LangChain 是什么
  结果: LangChain 是一个用于构建 LLM 应用的开源框架...

--- 步骤 2 ---
  工具: search
  输入: LangChain GitHub stars 数量
  结果: LangChain 在 GitHub 上有超过 90k stars...

最终答案: LangChain 是一个用于构建 LLM 应用的开源框架，
目前在 GitHub 上拥有超过 90k stars。
```

### verbose 模式

```python
# verbose=True 会在控制台实时打印执行过程
executor = AgentExecutor(
    agent=agent,
    tools=[search],
    verbose=True,  # 开发调试时开启，生产环境关闭
)
```

verbose 输出格式：

```
> Entering new AgentExecutor chain...

Thought: 我需要搜索 LangChain 的信息
Action: search
Action Input: LangChain 是什么
Observation: LangChain 是一个用于构建 LLM 应用的开源框架...
Thought: 我现在有足够的信息来回答了
Final Answer: LangChain 是一个用于构建 LLM 应用的开源框架。

> Finished chain.
```

## 4.5 流式输出

**让用户看到 Agent 的实时执行过程，而不是干等最终结果。**

```python
# 方式 1：stream() 逐步输出
for chunk in executor.stream({"input": "搜索 RAG 的最新进展"}):
    # chunk 可能包含 actions、steps 或 output
    if "actions" in chunk:
        for action in chunk["actions"]:
            print(f"[调用工具] {action.tool}({action.tool_input})")
    elif "steps" in chunk:
        for step in chunk["steps"]:
            print(f"[工具结果] {step.observation[:80]}...")
    elif "output" in chunk:
        print(f"[最终答案] {chunk['output']}")
```

```python
# 方式 2：使用回调实时监控
from langchain_core.callbacks import StdOutCallbackHandler

result = executor.invoke(
    {"input": "搜索 RAG 的最新进展"},
    config={"callbacks": [StdOutCallbackHandler()]},
)
```

## 4.6 完整示例：RAG 知识库问答 Agent

**把上面的知识点组合起来，构建一个完整的 Agent：**

```python
from langchain.agents import AgentExecutor, create_tool_calling_agent
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.prompts import ChatPromptTemplate


# 1. 定义工具
@tool
def search_knowledge_base(query: str) -> str:
    """Search the internal knowledge base for RAG-related documents.

    Args:
        query: Natural language search query about RAG technology.
    """
    knowledge = {
        "RAG": "RAG（检索增强生成）通过检索外部知识增强 LLM 生成，核心流程：索引→检索→生成。",
        "Embedding": "Embedding 将文本映射到高维向量空间，语义相近的文本在向量空间中距离更近。",
        "Chunking": "文本分块是 RAG 的关键步骤，常见策略：固定大小、语义分块、递归分块。",
    }
    for key, value in knowledge.items():
        if key.lower() in query.lower():
            return value
    return "未找到相关文档，请尝试其他关键词。"


@tool
def calculate(expression: str) -> str:
    """Calculate a mathematical expression.

    Args:
        expression: A valid Python math expression like '2 + 3 * 4'.
    """
    try:
        return str(eval(expression))
    except Exception as e:
        return f"计算错误: {e}"


# 2. 创建 LLM 和 Prompt
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)

prompt = ChatPromptTemplate.from_messages([
    ("system", """你是一个 RAG 技术知识库助手。
优先使用知识库搜索工具获取准确信息，需要计算时使用计算工具。
使用中文回答，简洁专业。"""),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

# 3. 创建 Agent
agent = create_tool_calling_agent(llm, [search_knowledge_base, calculate], prompt)

# 4. 创建 AgentExecutor（生产级配置）
executor = AgentExecutor(
    agent=agent,
    tools=[search_knowledge_base, calculate],
    max_iterations=10,              # 知识库问答不需要太多迭代
    handle_parsing_errors=True,     # 自动处理解析错误
    return_intermediate_steps=True, # 返回中间步骤（调试用）
    verbose=True,                   # 打印执行过程（调试用）
)

# 5. 运行
result = executor.invoke({"input": "什么是 RAG？它的核心流程是什么？"})
print("\n最终答案:", result["output"])
print(f"共执行了 {len(result['intermediate_steps'])} 步")
```
