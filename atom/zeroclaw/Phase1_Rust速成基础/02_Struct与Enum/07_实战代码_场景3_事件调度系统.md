# 实战代码 场景 3：事件调度系统

> **一句话定位：** 用一个 11 变体的 enum 模拟 ZeroClaw 的 ObserverEvent，配合 match 分发 + 统计，体验 Rust enum 在真实事件驱动架构中的威力。

---

## 场景背景

### ZeroClaw 的 ObserverEvent

ZeroClaw 的可观测性模块定义了 `ObserverEvent` enum，包含 11 种事件变体，每种携带不同数据。Agent 运行过程中不断 emit 事件，调度器通过 `match` 将事件分发给对应的处理逻辑。

这是典型的**事件驱动架构**：

```
Agent 运行 --> emit 事件 --> 调度器 match --> 处理器（日志/统计/告警）
```

### TypeScript 中你怎么做

```typescript
// TypeScript: EventEmitter 模式
import { EventEmitter } from "events";

const emitter = new EventEmitter();
emitter.on("agent.started", (data) => console.log(data));
emitter.on("error", (data) => console.error(data));
emitter.emit("agent.started", { agentId: "z-001" });

// 问题 1：事件名是字符串，拼错了不会报错
// 问题 2：data 是 any，没有类型检查
// 问题 3：忘记监听某个事件？运行时才发现
```

Rust 的 enum + match 解决了这三个问题：事件名是编译期类型、数据结构固定、忘记处理任何变体都编译不过。

---

## 第一步：定义事件 Enum

11 个变体，全部使用 struct variant（命名字段），对应 ZeroClaw 的真实设计：

```rust
#[derive(Debug, Clone)]
enum ObserverEvent {
    // --- Agent 生命周期 ---
    AgentStarted { agent_id: String },
    AgentCompleted { agent_id: String, total_turns: u32 },

    // --- 工具调用 ---
    ToolCallStarted { tool_name: String, input: String },
    ToolCallCompleted { tool_name: String, duration_ms: u64 },

    // --- LLM 交互 ---
    LlmRequestStarted { model: String, message_count: usize },
    LlmResponseReceived { model: String, tokens_used: u32 },

    // --- 记忆系统 ---
    MemoryStored { category: String, key: String },

    // --- 安全检查 ---
    SecurityCheckPassed { action: String },
    SecurityCheckFailed { action: String, reason: String },

    // --- 错误与自定义 ---
    Error { source: String, message: String },
    Custom { name: String, data: String },
}
```

> **对比 TypeScript：** 你需要定义 11 个 interface + 一个 union type + 手动维护 `type` 字段。Rust 一个 enum 搞定，编译器自动管理标签。

---

## 第二步：事件分类与过滤

给 enum 加方法，实现分类、严重级别判断、快速过滤：

```rust
impl ObserverEvent {
    fn severity(&self) -> &str {
        match self {
            ObserverEvent::Error { .. } => "error",
            ObserverEvent::SecurityCheckFailed { .. } => "warn",
            _ => "info",
        }
    }

    fn category(&self) -> &str {
        match self {
            ObserverEvent::AgentStarted { .. }
            | ObserverEvent::AgentCompleted { .. } => "agent",

            ObserverEvent::ToolCallStarted { .. }
            | ObserverEvent::ToolCallCompleted { .. } => "tool",

            ObserverEvent::LlmRequestStarted { .. }
            | ObserverEvent::LlmResponseReceived { .. } => "llm",

            ObserverEvent::MemoryStored { .. } => "memory",

            ObserverEvent::SecurityCheckPassed { .. }
            | ObserverEvent::SecurityCheckFailed { .. } => "security",

            ObserverEvent::Error { .. } => "error",
            ObserverEvent::Custom { .. } => "custom",
        }
    }

    fn is_error(&self) -> bool {
        // matches! 宏：快速判断是否匹配某个模式
        matches!(
            self,
            ObserverEvent::Error { .. } | ObserverEvent::SecurityCheckFailed { .. }
        )
    }
}
```

> **`matches!` 宏：** 相当于写一个只返回 `bool` 的 match。TypeScript 里你得写 `event.type === "error" || event.type === "securityCheckFailed"`，拼错字符串不会报错。Rust 拼错变体名直接编译失败。

## 第三步：事件调度器

用 struct 封装调度逻辑，`dispatch` 方法通过 match 格式化每种事件的日志：

```rust
struct EventDispatcher {
    show_debug: bool,
}

impl EventDispatcher {
    fn new(show_debug: bool) -> Self {
        EventDispatcher { show_debug }
    }

    fn dispatch(&self, event: &ObserverEvent) {
        let severity = event.severity();
        let category = event.category();

        let detail = match event {
            ObserverEvent::AgentStarted { agent_id } => {
                format!("Agent '{}' started", agent_id)
            }
            ObserverEvent::AgentCompleted { agent_id, total_turns } => {
                format!("Agent '{}' completed after {} turns", agent_id, total_turns)
            }
            ObserverEvent::ToolCallStarted { tool_name, input } => {
                format!("Tool '{}' called with: {}", tool_name, input)
            }
            ObserverEvent::ToolCallCompleted { tool_name, duration_ms } => {
                format!("Tool '{}' finished in {}ms", tool_name, duration_ms)
            }
            ObserverEvent::LlmRequestStarted { model, message_count } => {
                format!("LLM '{}' request with {} messages", model, message_count)
            }
            ObserverEvent::LlmResponseReceived { model, tokens_used } => {
                format!("LLM '{}' responded, {} tokens", model, tokens_used)
            }
            ObserverEvent::MemoryStored { category, key } => {
                format!("Memory stored: [{}] {}", category, key)
            }
            ObserverEvent::SecurityCheckPassed { action } => {
                format!("Security OK: '{}'", action)
            }
            ObserverEvent::SecurityCheckFailed { action, reason } => {
                format!("Security BLOCKED: '{}' - {}", action, reason)
            }
            ObserverEvent::Error { source, message } => {
                format!("ERROR in '{}': {}", source, message)
            }
            ObserverEvent::Custom { name, data } => {
                format!("Custom '{}': {}", name, data)
            }
        };

        println!("[{:<5}] [{}] {}", severity.to_uppercase(), category, detail);

        if self.show_debug {
            println!("        {:?}", event);
        }
    }
}
```

> **关键点：** match 的 11 个分支缺一不可。如果你以后给 `ObserverEvent` 加了第 12 个变体，所有 match 都会编译报错，强制你更新处理逻辑。TypeScript 的 `switch` 做不到这一点。

## 第四步：事件统计

收集一批事件后，统计各类别数量、错误列表、总 token 消耗和工具耗时：

```rust
struct EventSummary {
    total: usize,
    by_category: Vec<(String, usize)>,
    errors: Vec<String>,
    total_tokens: u32,
    total_tool_duration_ms: u64,
}

fn summarize(events: &[ObserverEvent]) -> EventSummary {
    let mut category_counts = std::collections::HashMap::new();
    let mut errors = Vec::new();
    let mut total_tokens: u32 = 0;
    let mut total_tool_duration_ms: u64 = 0;

    for event in events {
        *category_counts.entry(event.category().to_string()).or_insert(0usize) += 1;

        match event {
            ObserverEvent::Error { source, message } => {
                errors.push(format!("[{}] {}", source, message));
            }
            ObserverEvent::SecurityCheckFailed { action, reason } => {
                errors.push(format!("Security: '{}' - {}", action, reason));
            }
            ObserverEvent::LlmResponseReceived { tokens_used, .. } => {
                total_tokens += tokens_used;
            }
            ObserverEvent::ToolCallCompleted { duration_ms, .. } => {
                total_tool_duration_ms += duration_ms;
            }
            _ => {}
        }
    }

    let mut by_category: Vec<(String, usize)> = category_counts.into_iter().collect();
    by_category.sort_by(|a, b| b.1.cmp(&a.1));

    EventSummary { total: events.len(), by_category, errors, total_tokens, total_tool_duration_ms }
}
```

> **`_ => {}`：** 在统计函数中，我们只关心部分变体的数据。`_` 通配符处理"其余所有情况"。如果你去掉 `_` 且漏了某个变体，编译器会报错。

## 第五步：完整可运行代码

将以上所有部分组合成一个完整的 `main.rs`，用 `rustc main.rs && ./main` 即可运行：

```rust
use std::collections::HashMap;

// ===== 1. 事件定义 =====
#[derive(Debug, Clone)]
enum ObserverEvent {
    AgentStarted { agent_id: String },
    AgentCompleted { agent_id: String, total_turns: u32 },
    ToolCallStarted { tool_name: String, input: String },
    ToolCallCompleted { tool_name: String, duration_ms: u64 },
    LlmRequestStarted { model: String, message_count: usize },

    SecurityCheckPassed { action: String },
    SecurityCheckFailed { action: String, reason: String },
    Error { source: String, message: String },
    Custom { name: String, data: String },
}

// ===== 2. 事件方法 =====
impl ObserverEvent {

        match self {
            ObserverEvent::Error { .. } => "error",
            ObserverEvent::SecurityCheckFailed { .. } => "warn",

// ===== 4. 事件统计 =====
struct EventSummary {
    total: usize,
    by_category: Vec<(String, usize)>,
    errors: Vec<String>,
    total_tokens: u32,
    total_tool_duration_ms: u64,
}

fn summarize(events: &[ObserverEvent]) -> EventSummary {
    let mut counts: HashMap<String, usize> = HashMap::new();
    let mut errors = Vec::new();
    let mut total_tokens: u32 = 0;
    let mut total_tool_ms: u64 = 0;

    for event in events {
        *counts.entry(event.category().to_string()).or_insert(0) += 1;
        match event {
            ObserverEvent::Error { source, message } => {
                errors.push(format!("[{}] {}", source, message));
            }
            ObserverEvent::SecurityCheckFailed { action, reason } => {
                errors.push(format!("Security: '{}' - {}", action, reason));
            }
            ObserverEvent::LlmResponseReceived { tokens_used, .. } => {
                total_tokens += tokens_used;
            }
            ObserverEvent::ToolCallCompleted { duration_ms, .. } => {
                total_tool_ms += duration_ms;
            }
            _ => {}
        }
    }

    let mut by_category: Vec<(String, usize)> = counts.into_iter().collect();
    by_category.sort_by(|a, b| b.1.cmp(&a.1));

    EventSummary {
        total: events.len(),
        by_category,
        errors,
        total_tokens,
        total_tool_duration_ms: total_tool_ms,
    }
}

fn print_summary(s: &EventSummary) {
    println!("\n========== Event Summary ==========");
    println!("Total events: {}", s.total);
    println!("\nBy category:");
    for (cat, count) in &s.by_category {
        println!("  {:<10} : {}", cat, count);
    }
    println!("\nTotal LLM tokens used: {}", s.total_tokens);
    println!("Total tool duration: {}ms", s.total_tool_duration_ms);
    if s.errors.is_empty() {
        println!("\nNo errors!");
    } else {
        println!("\nErrors ({}):", s.errors.len());
        for err in &s.errors {
            println!("  - {}", err);
        }
    }
    println!("====================================");
}

            action: "read_file".to_string(),
        },

            model: "gpt-4".to_string(),
            message_count: 3,
        },
        ObserverEvent::LlmResponseReceived {
            model: "gpt-4".to_string(),
            tokens_used: 150,
        },
        ObserverEvent::ToolCallStarted {
            tool_name: "web_search".to_string(),
            input: "Rust enum tutorial".to_string(),
        },
        ObserverEvent::ToolCallCompleted {
            tool_name: "web_search".to_string(),
            duration_ms: 320,
        },

        ObserverEvent::MemoryStored {
            category: "conversation".to_string(),

            model: "gpt-4".to_string(),
            message_count: 5,
        },
        ObserverEvent::LlmResponseReceived {
            model: "gpt-4".to_string(),
            tokens_used: 280,
        },
        ObserverEvent::SecurityCheckFailed {
            action: "execute_shell".to_string(),
            reason: "Supervised mode requires approval".to_string(),
        },
        ObserverEvent::ToolCallStarted {
            tool_name: "file_write".to_string(),
            input: "output.txt".to_string(),
        },
        ObserverEvent::ToolCallCompleted {
            tool_name: "file_write".to_string(),
            duration_ms: 45,
        },
        ObserverEvent::Error {
            source: "file_write".to_string(),

        ObserverEvent::Custom {
            name: "perf.checkpoint".to_string(),
            data: "turn_2_complete".to_string(),
        },
        ObserverEvent::AgentCompleted {
            agent_id: "zeroclaw-001".to_string(),
            total_turns: 2,
        },

    let dispatcher = EventDispatcher::new(false);
    println!("===== Event Dispatch Log =====\n");
    for event in &events {
        dispatcher.dispatch(event);
    }

    // 过滤出错误事件
    let error_events: Vec<&ObserverEvent> =
        events.iter().filter(|e| e.is_error()).collect();
    println!("\n===== Error Events ({}) =====", error_events.len());
    for e in &error_events {
        println!("  {:?}", e);
    }

    // 打印统计摘要
    let summary = summarize(&events);
    print_summary(&summary);
}
```


## TypeScript 等价实现

同样的事件系统在 TypeScript 中的写法，对比感受差异：

```typescript
// TypeScript: 用 discriminated union 模拟
type ObserverEvent =
  | { type: "AgentStarted"; agentId: string }
  | { type: "AgentCompleted"; agentId: string; totalTurns: number }
  | { type: "ToolCallStarted"; toolName: string; input: string }
  | { type: "ToolCallCompleted"; toolName: string; durationMs: number }
  | { type: "LlmRequestStarted"; model: string; messageCount: number }
  | { type: "LlmResponseReceived"; model: string; tokensUsed: number }
  | { type: "MemoryStored"; category: string; key: string }
  | { type: "SecurityCheckPassed"; action: string }
  | { type: "SecurityCheckFailed"; action: string; reason: string }
  | { type: "Error"; source: string; message: string }
  | { type: "Custom"; name: string; data: string };

function dispatch(event: ObserverEvent): void {
  switch (event.type) {
    case "AgentStarted":
      console.log(`Agent '${event.agentId}' started`);
      break;
    case "AgentCompleted":
      console.log(`Agent completed after ${event.totalTurns} turns`);
      break;
    case "Error":
      console.error(`ERROR in '${event.source}': ${event.message}`);
      break;
    // ... 还有 8 个 case 要写
    // 忘了某个 case？TypeScript 不一定报错
  }
}

// 问题：
// 1. 每个 type 字段是字符串，拼错 "AgentStartd" 不会编译报错
// 2. switch 没有穷尽检查（除非你手动加 exhaustive check）
// 3. 不能给 union type 加方法（severity、category 只能写成独立函数）
```

**Rust vs TypeScript 对比：**

| 对比项 | TypeScript | Rust |
|--------|-----------|------|
| 变体标签 | 手动维护 `type` 字符串 | 编译器自动管理 |
| 穷尽检查 | 需要手动 `never` 技巧 | match 自动强制 |
| 给变体加方法 | 不支持 | `impl ObserverEvent { ... }` |
| 拼错变体名 | 运行时 bug | 编译错误 |
| 新增变体后 | 可能忘记更新 switch | 所有 match 编译报错 |

## 学到了什么

| 知识点 | 本场景中的体现 |
|--------|---------------|
| **复杂 enum（11 变体）** | `ObserverEvent` 每个变体携带不同字段，一个类型表达整个事件系统 |
| **match 处理多变体** | `dispatch` 中 11 个分支，缺一不可，编译器保证穷尽 |
| **`matches!` 宏** | `is_error()` 用一行代码判断是否匹配多个模式 |
| **`\|` 合并模式** | `category()` 中多个变体映射到同一类别 |
| **`..` 忽略字段** | `LlmResponseReceived { tokens_used, .. }` 只取需要的字段 |
| **struct 方法** | `EventDispatcher` 封装调度逻辑，`&self` 引用调用 |
| **`&[ObserverEvent]` 切片** | `summarize` 接受任意长度的事件数组引用 |
| **HashMap 统计** | `entry().or_insert()` 模式做分类计数 |
| **事件驱动模式** | emit -> dispatch -> handle，Rust enum 天然适合这种架构 |

> **核心收获：** 当你的系统有 10+ 种事件/消息/状态时，Rust enum + match 是最安全的建模方式。新增变体时编译器会告诉你所有需要更新的地方 -- 这在大型项目中价值巨大。
