## 1. 为什么需要模型选择与配置？

### 核心问题

在 RAG 开发中，Embedding 模型的选择和配置直接影响：
- **检索质量**：不同模型的语义理解能力差异巨大
- **成本控制**：API 调用费用、存储成本
- **性能表现**：推理速度、并发能力
- **兼容性**：与向量数据库、框架的集成

### 典型场景

```python
# 场景1：高精度场景（客服知识库）
embeddings = OpenAIEmbeddings(model="text-embedding-3-large")

# 场景2：成本敏感场景（大规模文档）
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512  # 降低维度节省成本
)

# 场景3：本地部署场景
embeddings = HuggingFaceEmbeddings(
    model_name="all-MiniLM-L6-v2"
)
```

## 2. OpenAI 模型选择

### 2.1 模型对比

| 模型 | 默认维度 | 性能 | 成本 | 适用场景 |
|------|---------|------|------|---------|
| **text-embedding-3-large** | 3072 | ⭐⭐⭐⭐⭐ | 高 | 高精度检索、企业知识库 |
| **text-embedding-3-small** | 1536 | ⭐⭐⭐⭐ | 低 | 通用场景、大规模应用 |
| **text-embedding-ada-002** | 1536 | ⭐⭐⭐ | 中 | 兼容性场景、旧项目 |

### 2.2 text-embedding-3-large

**特点**：
- 最高精度，MTEB 排行榜前列
- 支持自定义维度（256-3072）
- 适合高价值场景

**使用示例**：
```python
from langchain_openai import OpenAIEmbeddings

# 基础配置
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-large"
)

# 降维配置（节省成本）
embeddings_reduced = OpenAIEmbeddings(
    model="text-embedding-3-large",
    dimensions=1024  # 降低到 1024 维
)

# 测试
vector = embeddings.embed_query("什么是 RAG？")
print(f"向量维度: {len(vector)}")  # 3072
```

**性能数据**（来自 OpenAI 官方）：
- MTEB 平均分：64.6
- 检索任务：54.9
- 分类任务：70.2

### 2.3 text-embedding-3-small

**特点**：
- 性价比最高
- 速度快，适合大规模应用
- 2025-2026 社区推荐首选

**使用示例**：
```python
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512  # 可选：降低维度
)

# 批量处理
texts = ["文档1", "文档2", "文档3"]
vectors = embeddings.embed_documents(texts)
```

**成本对比**（2026年2月价格）：
- text-embedding-3-large: $0.13 / 1M tokens
- text-embedding-3-small: $0.02 / 1M tokens
- **节省 85% 成本**

### 2.4 text-embedding-ada-002

**特点**：
- 旧版模型，兼容性好
- 不支持自定义维度
- 逐渐被淘汰

**使用场景**：
- 旧项目迁移
- 需要与旧数据兼容

```python
embeddings = OpenAIEmbeddings(
    model="text-embedding-ada-002"
)
```

## 3. Sentence Transformers 模型选择

### 3.1 常用模型对比

| 模型 | 维度 | 大小 | 速度 | 适用场景 |
|------|------|------|------|---------|
| **all-MiniLM-L6-v2** | 384 | 80MB | 快 | 通用场景、本地部署 |
| **all-mpnet-base-v2** | 768 | 420MB | 中 | 高精度场景 |
| **paraphrase-multilingual** | 768 | 970MB | 慢 | 多语言场景 |

### 3.2 all-MiniLM-L6-v2（推荐）

**特点**：
- 轻量级，速度快
- 社区最受欢迎
- 适合本地部署

**使用示例**：
```python
from langchain_community.embeddings import HuggingFaceEmbeddings

embeddings = HuggingFaceEmbeddings(
    model_name="all-MiniLM-L6-v2",
    model_kwargs={'device': 'cpu'},
    encode_kwargs={'normalize_embeddings': True}
)

# 测试
vector = embeddings.embed_query("什么是向量检索？")
print(f"向量维度: {len(vector)}")  # 384
```

### 3.3 all-mpnet-base-v2

**特点**：
- 更高精度
- 适合英文场景

```python
embeddings = HuggingFaceEmbeddings(

    model_kwargs={'device': 'cuda'},  # GPU 加速
    encode_kwargs={
        'normalize_embeddings': True,
        'batch_size': 32
```

## 4. 配置参数详解

### 4.1 核心参数总览

```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(
    # 模型选择
    model="text-embedding-3-small",

    # 维度配置（text-embedding-3+ 支持）
    dimensions=512,

    # API 配置
    openai_api_key="sk-...",
    openai_api_base="https://api.openai.com/v1",

    # 性能配置
    chunk_size=1000,
    max_retries=2,
    request_timeout=60,

    # Token 管理
    embedding_ctx_length=8191,
    tiktoken_enabled=True,
    check_embedding_ctx_length=True,

    # 用户体验
    show_progress_bar=False
)
```

### 4.2 dimensions（维度配置）

**功能**：自定义输出向量维度（仅 text-embedding-3+ 支持）

**使用场景**：
- 降低存储成本
- 加快检索速度
- 权衡精度与成本

**示例**：
```python
# 场景1：高精度（默认）
embeddings_full = OpenAIEmbeddings(
    model="text-embedding-3-large"
    # dimensions=3072（默认）
)

# 场景2：平衡（推荐）
embeddings_balanced = OpenAIEmbeddings(
    model="text-embedding-3-large",
    dimensions=1024  # 降低到 1024 维
)

# 场景3：极致成本优化
embeddings_mini = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=256  # 最小 256 维
)

# 性能对比
import time

text = "这是一个测试文档" * 100

# 测试1：3072 维
start = time.time()
v1 = embeddings_full.embed_query(text)
print(f"3072维: {time.time() - start:.3f}s, 大小: {len(v1)}")

# 测试2：1024 维
start = time.time()
v2 = embeddings_balanced.embed_query(text)
print(f"1024维: {time.time() - start:.3f}s, 大小: {len(v2)}")
```

**维度选择建议**：
- **3072维**：金融、法律等高精度场景
- **1024维**：通用企业知识库（推荐）
- **512维**：大规模文档检索
- **256维**：极致成本优化

### 4.3 chunk_size（批量大小）

**功能**：控制每批次处理的文本数量

**默认值**：1000

**使用场景**：
```python
# 场景1：小批量（API 限制严格）
embeddings_small = OpenAIEmbeddings(
    chunk_size=100
)

# 场景2：大批量（高吞吐量）
embeddings_large = OpenAIEmbeddings(
    chunk_size=2000
)

# 实际应用
documents = ["文档" + str(i) for i in range(5000)]

# 批量嵌入
vectors = embeddings_large.embed_documents(documents)
print(f"处理了 {len(vectors)} 个文档")
```

**调优建议**：
- **100-500**：API 限制严格、网络不稳定
- **1000**：默认值，适合大多数场景
- **2000+**：高吞吐量、稳定网络

### 4.4 max_retries（重试机制）

**功能**：API 调用失败时的重试次数

**默认值**：2

**配合参数**：
```python
embeddings = OpenAIEmbeddings(
    max_retries=3,
    retry_min_seconds=4,
    retry_max_seconds=20
)
```

**重试策略**：Exponential Backoff
- 第1次重试：等待 4 秒
- 第2次重试：等待 8 秒
- 第3次重试：等待 16 秒

### 4.5 request_timeout（超时配置）

**功能**：控制 API 请求超时时间

**类型**：
- `float`：统一超时
- `tuple[float, float]`：(连接超时, 读取超时)

```python
# 方式1：统一超时
embeddings = OpenAIEmbeddings(
    request_timeout=60  # 60秒
)

# 方式2：分离超时
embeddings = OpenAIEmbeddings(
    request_timeout=(10, 60)  # 连接10秒，读取60秒
)
```

### 4.6 embedding_ctx_length（上下文长度）

**功能**：单个文本的最大 token 数

**默认值**：8191（ada-002 限制）

**自动分块**：
```python
embeddings = OpenAIEmbeddings(
    embedding_ctx_length=8191,
    tiktoken_enabled=True
)

# 长文本自动分块
long_text = "这是一个很长的文档..." * 10000
vector = embeddings.embed_query(long_text)
# 自动分块 → 加权平均 → 归一化
```

### 4.7 check_embedding_ctx_length（兼容性开关）

**功能**：是否检查 token 长度（用于非 OpenAI 提供商）

**默认值**：True

**使用场景**：见第5节

## 7. Pydantic BaseModel 配置管理

### 7.1 设计原理

OpenAIEmbeddings 继承自 Pydantic BaseModel：

```python
from pydantic import BaseModel, Field
from langchain_core.embeddings import Embeddings

class OpenAIEmbeddings(BaseModel, Embeddings):
    """OpenAI embedding model integration."""

    model: str = "text-embedding-ada-002"
    dimensions: int | None = None
    # ...
```

**优势**：
- 自动类型验证
- 环境变量自动加载
- 序列化支持
- 配置持久化

### 7.2 配置验证

```python
from langchain_openai import OpenAIEmbeddings
from pydantic import ValidationError

# 正确配置
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512
)

# 错误配置（自动验证）
try:
    embeddings = OpenAIEmbeddings(
        model="text-embedding-ada-002",
        dimensions=512  # ada-002 不支持自定义维度
    )
except ValidationError as e:
    print(f"配置错误: {e}")
```

### 7.3 配置序列化

```python
import json

# 创建配置
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    dimensions=512,
    chunk_size=1000
)

# 序列化
config = embeddings.dict()
print(json.dumps(config, indent=2))

# 保存到文件
with open("embedding_config.json", "w") as f:
    json.dump(config, f, indent=2)

# 从文件加载
with open("embedding_config.json", "r") as f:
    config = json.load(f)

embeddings_loaded = OpenAIEmbeddings(**config)
```

### 9.3 多提供商配置

```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.embeddings import HuggingFaceEmbeddings

class EmbeddingFactory:
    """Embedding 工厂类"""

    @staticmethod
    def create_openai(model: str = "text-embedding-3-small"):
        return OpenAIEmbeddings(
            model=model,
            dimensions=512 if "small" in model else 1024
        )

    @staticmethod
    def create_huggingface(model: str = "all-MiniLM-L6-v2"):
        return HuggingFaceEmbeddings(
            model_name=model,
            model_kwargs={'device': 'cpu'},
            encode_kwargs={'normalize_embeddings': True}
        )

    @staticmethod
    def create_custom(base_url: str, model: str):
        return OpenAIEmbeddings(
            model=model,
            openai_api_base=base_url,
            check_embedding_ctx_length=False
        )

# 使用
embeddings_openai = EmbeddingFactory.create_openai()
embeddings_hf = EmbeddingFactory.create_huggingface()
embeddings_custom = EmbeddingFactory.create_custom(
    base_url="http://localhost:11434/v1",
    model="nomic-embed-text"
)
```

### 9.4 生产环境配置

```python
import os
from langchain_openai import OpenAIEmbeddings
from langchain_community.embeddings import CacheBackedEmbeddings
from langchain_community.storage import LocalFileStore

# 环境配置
ENV = os.getenv("ENV", "dev")

# 基础配置
base_config = {
    "dev": {
        "model": "text-embedding-3-small",
        "dimensions": 512,
        "chunk_size": 500,
        "show_progress_bar": True
    },
    "prod": {
        "model": "text-embedding-3-large",
        "dimensions": 1024,
        "chunk_size": 2000,
        "max_retries": 3,
        "request_timeout": (10, 120)
    }
}

# 创建 Embeddings
embeddings = OpenAIEmbeddings(**base_config[ENV])

# 添加缓存
store = LocalFileStore("./embeddings_cache")
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    embeddings,
    store,
    namespace=f"{ENV}-{base_config[ENV]['model']}"
)

# 使用
vector = cached_embeddings.embed_query("测试查询")
print(f"✓ 嵌入成功，维度: {len(vector)}")
```

## 10. 最佳实践总结

### 10.1 模型选择

1. **通用场景**：text-embedding-3-small + dimensions=512
2. **高精度场景**：text-embedding-3-large + dimensions=1024
3. **本地部署**：all-MiniLM-L6-v2
4. **成本优化**：text-embedding-3-small + dimensions=256

### 10.2 配置优化

1. **批量大小**：根据 API 限制调整 chunk_size
2. **重试机制**：生产环境设置 max_retries=3
3. **超时配置**：使用 tuple 分离连接和读取超时
4. **缓存策略**：使用 CacheBackedEmbeddings 避免重复计算

### 10.3 安全建议

1. **密钥管理**：使用环境变量或密钥管理服务
2. **动态获取**：支持密钥轮换
3. **错误处理**：捕获 API 错误并重试
4. **监控告警**：记录 API 调用失败率

### 10.4 性能优化

1. **维度降低**：在精度可接受范围内降低维度
2. **批量处理**：使用 embed_documents 而非循环调用
3. **并发控制**：设置合理的 chunk_size
4. **进度显示**：长时间任务开启 show_progress_bar
