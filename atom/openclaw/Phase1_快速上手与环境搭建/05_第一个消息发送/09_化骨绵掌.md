## 卡片 1：OpenClaw 是什么？

**核心定义**：
OpenClaw 是一个统一的消息发送框架，通过 Gateway 架构支持多个消息平台（WhatsApp、Telegram、Discord 等）。

**关键特性**：
- 统一 CLI 命令：`openclaw message send "Hello"`
- 多通道支持：一套代码操作所有平台
- DM 配对机制：安全的用户验证
- Agent 系统：智能自动回复

**类比**：
就像万能遥控器，一个遥控器控制所有家电。

**快速上手**：
```bash
# 1. 启动 Gateway
openclaw gateway start

# 2. 配对通道
openclaw pairing start

# 3. 发送消息
openclaw message send "Hello!"
```

**记忆口诀**：
"一个 Gateway，多个通道，统一命令，轻松发送"

## 卡片 2：Gateway 架构

**核心概念**：
Gateway 是 OpenClaw 的核心，负责消息路由、队列管理、协议转换。

**架构图**：
```
用户命令 → Gateway → Channel Plugin → 目标平台
```

**Gateway 的职责**：
1. **消息队列**：异步处理，避免阻塞
2. **路由选择**：根据配对信息选择通道
3. **协议转换**：统一格式 → 平台特定格式
4. **错误处理**：重试、降级、监控

**为什么需要 Gateway？**
- 统一入口：用户只需要一个命令
- 协议隔离：不需要了解各平台 API
- 集中管理：配对、路由、监控都在 Gateway

**类比**：
Gateway = Nginx 反向代理 + 消息队列 + 路由器

## 卡片 3：Channel Plugin 系统

**核心概念**：
Channel Plugin 是通道的具体实现，每个平台一个插件。

**插件接口**：
```typescript
interface ChannelPlugin {
  name: string;
  send(message: OutboundMessage): Promise<SendResult>;
  receive(): AsyncIterator<InboundMessage>;
}
```

**支持的通道**：
- ✅ WhatsApp（QR 码配对）
- ✅ Telegram（Bot Token）
- ✅ Discord（Bot Token）
- ✅ Slack（OAuth）
- ⚠️ Microsoft Teams（Beta）

**插件化的优势**：
- 易于扩展：添加新通道只需实现接口
- 解耦：核心逻辑与通道实现分离
- 灵活：可以动态加载/卸载插件

**类比**：
Channel Plugin = 数据库驱动（MySQL、PostgreSQL 等）

## 卡片 4：DM 配对机制

**核心概念**：
DM（Direct Message）配对是 OpenClaw 的安全验证机制。

**配对流程**：
```
1. 运行: openclaw pairing start
2. 生成配对码: PAIR-ABC123
3. 用户通过私信发送配对码给 Bot
4. 系统验证配对码
5. 配对成功，保存配对信息
```

**为什么使用 DM 配对？**
- **安全性**：避免公开暴露配对码
- **用户验证**：确保只有授权用户可以配对
- **简单性**：无需复杂的 OAuth 流程

**配对状态管理**：
- 配对需要定期维护（心跳检测）
- 长时间未使用会失效
- 可以手动删除配对

**类比**：
DM 配对 = 加微信好友（扫码 + 确认）

## 卡片 5：消息发送命令

**基础命令**：
```bash
# 发送文本消息
openclaw message send "Hello, World!"

# 指定通道
openclaw message send "Hello" --channel whatsapp

# 发送 Markdown
openclaw message send "**Bold**" --format markdown

# 发送带附件
openclaw message send "File" --attachment ./file.pdf
```

**高级用法**：
```bash
# 发送到多个通道
openclaw message send "Broadcast" --channel whatsapp --channel telegram

# 静默发送（无通知）
openclaw message send "Silent" --silent

# 回复特定消息
openclaw message send "Reply" --reply-to msg_123
```

**命令参数**：
- `<text>`：消息文本（必需）
- `--channel`：目标通道（可选，默认所有）
- `--format`：消息格式（text/markdown/html）
- `--attachment`：附件路径
- `--silent`：静默发送

## 卡片 6：Agent 系统概览

**核心概念**：
Agent 是自动响应系统，监听入站消息并自动回复。

**三种类型**：

1. **规则型 Agent**：
   - 基于关键词匹配
   - 响应速度快（<100ms）
   - 适合 FAQ

2. **AI 型 Agent**：
   - 使用 LLM（如 GPT-4）
   - 响应灵活，理解上下文
   - 适合复杂问题

3. **混合型 Agent**：
   - 规则优先，AI 兜底
   - 平衡速度和智能
   - 适合通用场景

**Agent 优先级**：
```
规则型 (priority: 1) → 混合型 (priority: 50) → AI 型 (priority: 100)
```

**配置示例**：
```yaml
agents:
  - name: faq-bot
    type: rule
    priority: 1
  - name: ai-assistant
    type: ai
    priority: 100
```

## 卡片 7：规则型 Agent

**核心概念**：
规则型 Agent 基于关键词或正则表达式匹配。

**实现示例**：
```typescript
class RuleAgent {
  private rules = [
    { keywords: ['营业时间'], response: '9:00-18:00' },
    { keywords: ['地址'], response: '北京市朝阳区xxx' }
  ];

  canHandle(message: string): boolean {
    return this.rules.some(rule =>
      rule.keywords.some(kw => message.includes(kw))
    );
  }

  handle(message: string): string {
    const rule = this.rules.find(r =>
      r.keywords.some(kw => message.includes(kw))
    );
    return rule?.response || '无法回答';
  }
}
```

**优势**：
- 响应速度极快（<100ms）
- 准确率 100%（匹配时）
- 成本为零

**劣势**：
- 灵活性低（固定回复）
- 维护成本高（需要更新规则）

**适用场景**：
- FAQ 问答
- 固定流程引导
- 关键词触发

## 卡片 8：AI 型 Agent

**核心概念**：
AI 型 Agent 使用 LLM（如 GPT-4）生成响应。

**实现示例**：
```typescript
class AIAgent {
  async handle(message: string): Promise<string> {
    const response = await openai.chat.completions.create({
      model: 'gpt-4',
      messages: [
        { role: 'system', content: 'You are a helpful assistant.' },
        { role: 'user', content: message }
      ]
    });
    return response.choices[0].message.content;
  }
}
```

**优势**：
- 灵活性高（理解上下文）
- 可以处理复杂问题
- 自动学习

**劣势**：
- 响应速度慢（1-3s）
- 成本高（$0.05/条）
- 可能产生幻觉

**对话上下文管理**：
```typescript
private conversationHistory = new Map<string, Message[]>();

// 保存对话历史
history.push(
  { role: 'user', content: message },
  { role: 'assistant', content: response }
);
```

## 卡片 9：多通道消息发送

**核心概念**：
OpenClaw 支持同时向多个通道发送消息。

**发送策略**：

1. **全部发送（all）**：
```typescript
await sender.send({
  text: 'Broadcast message',
  strategy: 'all'
});
```

2. **主通道优先（primary）**：
```typescript
await sender.send({
  text: 'Primary message',
  strategy: 'primary'
});
```

3. **轮询发送（round-robin）**：
```typescript
await sender.send({
  text: 'Round-robin message',
  strategy: 'round-robin'
});
```

4. **降级策略（fallback）**：
```typescript
try {
  await sendToChannel('whatsapp', message);
} catch (error) {
  await sendToChannel('telegram', message);
}
```

**并行 vs 顺序**：
- 并行：速度快，适合广播
- 顺序：可控性强，避免频率限制

**常见问题和解决方案**：

**问题 1：Gateway 无法启动**
```bash

lsof -ti:3000 | xargs kill -9

# 重启 Gateway
openclaw gateway restart
```

**问题 2：配对失效**
```bash
# 查看配对列表
openclaw pairing list

# 重新配对
openclaw pairing start
```

**问题 3：消息发送失败**
```bash
# 检查 Gateway 状态
openclaw gateway status

# 查看日志
openclaw gateway logs --level error

# 测试发送
```
