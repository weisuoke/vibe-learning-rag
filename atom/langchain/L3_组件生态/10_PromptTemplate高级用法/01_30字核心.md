## 为什么这是核心？

这30个字精准概括了 PromptTemplate 高级用法的三大核心能力和最终目标：

### 1. 部分变量预填充（Partial Variables）

**核心价值：** 避免重复传递相同的变量

在实际的 AI Agent 开发中，很多变量是固定的或共享的（如系统设置、日期、用户信息）。每次调用模板时都传递这些变量会导致代码冗余和维护困难。

**部分变量预填充**允许你在模板创建时就固定这些值，后续调用时只需传递变化的部分。这就像函数的默认参数，大大简化了代码。

```python
# 传统方式：每次都要传递 date 和 language
prompt.format(query="...", date="2026-02-26", language="zh-CN")

# 使用 partial variables：只传递变化的 query
prompt = PromptTemplate(
    template="...",
    partial_variables={"date": get_current_date, "language": "zh-CN"}
)
prompt.format(query="...")  # 简洁！
```

### 2. 模板组合复用（Template Composition）

**核心价值：** 构建模块化的提示词系统

在复杂的 AI Agent 系统中，提示词往往由多个部分组成（系统指令、上下文、用户输入、输出格式等）。如果每次都写完整的提示词，会导致大量重复代码。

**模板组合**允许你将提示词拆分成小的、可复用的模块，然后通过 `+` 操作符组合起来。这就像乐高积木，可以灵活组装。

```python
# 可复用的模块
system_prompt = PromptTemplate.from_template("你是一个{role}助手。\n")
context_prompt = PromptTemplate.from_template("上下文：{context}\n")
query_prompt = PromptTemplate.from_template("问题：{query}")

# 灵活组合
full_prompt = system_prompt + context_prompt + query_prompt
```

### 3. 多格式支持（Template Formats）

**核心价值：** 满足不同场景的需求

不同的场景需要不同的模板能力：
- **简单场景**：f-string 足够（`{variable}`）
- **嵌套数据**：mustache 更强大（`{{obj.field}}`）
- **复杂逻辑**：jinja2 最灵活（支持循环、条件）

**多格式支持**让你可以根据场景选择最合适的工具，而不是被限制在一种格式中。

```python
# f-string：简单快速
PromptTemplate.from_template("Say {foo}")

# mustache：支持嵌套
PromptTemplate.from_template("{{user.name}}", template_format="mustache")

# jinja2：支持逻辑
PromptTemplate.from_template("{% for item in items %}...{% endfor %}",
                             template_format="jinja2")
```

### 4. 构建模块化提示词系统（最终目标）

**核心价值：** 提升提示词工程的可维护性和复用性

前面三个能力的最终目标是：**构建一个模块化、可维护、可复用的提示词系统**。

在企业级 AI Agent 开发中，提示词不是一次性的，而是需要：
- **版本控制**：提示词的演进和回滚
- **团队协作**：多人共同维护提示词库
- **场景复用**：同一个模块在多个场景中使用
- **快速迭代**：修改一个模块，所有使用它的地方都更新

**模块化提示词系统**让这一切成为可能。

```python
# 提示词库（可版本控制）
prompts/
  ├── system/
  │   ├── rag_assistant.txt
  │   └── code_assistant.txt
  ├── context/
  │   ├── document_context.txt
  │   └── conversation_context.txt
  └── output/
      ├── json_format.txt
      └── markdown_format.txt

# 灵活组合使用
system = PromptTemplate.from_file("prompts/system/rag_assistant.txt")
context = PromptTemplate.from_file("prompts/context/document_context.txt")
output = PromptTemplate.from_file("prompts/output/json_format.txt")

prompt = system + context + output  # 模块化组合
```

## 与 AI Agent 开发的联系

### 在 RAG 系统中

**场景：** 构建多语言、多场景的 RAG 问答系统

```python
# 使用 partial variables 管理语言设置
base_prompt = PromptTemplate(
    template="...",
    partial_variables={"language": "zh-CN", "date": get_current_date}
)

# 使用模板组合构建不同场景
qa_prompt = base_prompt + question_template
summary_prompt = base_prompt + summary_template
```

**价值：**
- 语言设置只需配置一次
- 不同场景复用基础模板
- 易于维护和扩展

---

### 在 Agent 系统中

**场景：** 构建具有多种工具的 AI Agent

```python
# 使用 partial variables 管理工具列表
agent_prompt = PromptTemplate(
    template="...",
    partial_variables={
        "tools": get_available_tools,  # 动态获取工具列表
        "date": get_current_date
    }
)

# 使用模板组合构建多步推理
reasoning_prompt = system_prompt + tools_prompt + task_prompt
```

**价值：**
- 工具列表动态更新
- 推理模板模块化
- 易于调试和优化

---

### 在多模态应用中

**场景：** 构建支持文本、图像、视频的多模态 Agent

```python
# 使用多格式支持处理复杂数据
text_prompt = PromptTemplate.from_template("{text}", template_format="f-string")
image_prompt = PromptTemplate.from_template("{{image.url}}", template_format="mustache")
video_prompt = PromptTemplate.from_template(
    "{% for frame in video.frames %}...{% endfor %}",
    template_format="jinja2"
)

# 组合使用
multimodal_prompt = text_prompt + image_prompt + video_prompt
```

**价值：**
- 不同模态使用最合适的格式
- 模块化管理多模态数据
- 易于扩展新的模态
