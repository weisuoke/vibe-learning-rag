## 核心概念一句话总结

### 1. Reducer 函数
**Reducer 是一个接收两个值并返回合并后值的二元函数，签名为 `(Value, Value) -> Value`，用于定义状态字段的更新策略。**

### 2. BinaryOperatorAggregate Channel
**BinaryOperatorAggregate 是 Reducer 的底层执行机制，负责管理状态值的初始化、更新和 Overwrite 处理。**

### 3. 状态合并策略
**有 Reducer 的字段使用函数合并新旧值，无 Reducer 的字段直接替换为新值。**

### 4. 内置 Reducer 函数
**LangGraph 提供三种内置 Reducer：`operator.add`（列表/数值合并）、`operator.or_`（字典合并）、`add_messages`（消息按 ID 合并）。**

### 5. 自定义 Reducer 实现
**自定义 Reducer 必须接收恰好 2 个参数，返回 1 个值，保持纯函数特性，不产生副作用。**

### 6. Overwrite 机制
**使用 `Overwrite(value)` 或 `{"__overwrite__": value}` 可以跳过 Reducer 直接替换状态值，每个 super-step 只能有一个 Overwrite。**

### 7. 部分状态更新
**节点函数签名为 `State -> Partial<State>`，只需返回需要更新的字段，框架自动合并到完整状态。**

### 8. add_messages 深度解析
**add_messages 按消息 ID 合并，相同 ID 的消息会被替换，支持 RemoveMessage 删除特定消息，支持 REMOVE_ALL_MESSAGES 清空历史。**

## 关键技术点速查表

### Reducer 函数签名验证

| 验证项 | 要求 | 错误示例 |
|--------|------|----------|
| 参数数量 | 必须恰好 2 个 | `def r(x): return x` |
| 返回值 | 必须返回 1 个值 | `def r(a, b): pass` |
| 类型一致 | 输入输出类型相同 | `def r(a: int, b: int) -> str` |
| 纯函数 | 不修改输入，无副作用 | `def r(a, b): a.append(b); return a` |

### 内置 Reducer 对比

| Reducer | 数据类型 | 合并策略 | 时间复杂度 | 使用场景 |
|---------|---------|---------|-----------|---------|
| `operator.add` | list, str, int | 拼接/相加 | O(n) | 列表累积、计数器 |
| `operator.or_` | dict | 后者覆盖前者 | O(n) | 配置合并、元数据 |
| `add_messages` | list[Message] | 按 ID 合并 | O(n*m) | 聊天历史、对话管理 |

### 状态更新行为

| 场景 | 有 Reducer | 无 Reducer |
|------|-----------|-----------|
| 节点 A 返回 `{"x": [1]}` | `[1]` | `[1]` |
| 节点 B 返回 `{"x": [2]}` | `[1, 2]` (合并) | `[2]` (替换) |
| 初始值处理 | 第一个值直接赋值 | 直接赋值 |
| 后续值处理 | 调用 Reducer 合并 | 直接替换 |

### Overwrite 机制

| 方式 | 语法 | 说明 |
|------|------|------|
| Overwrite 对象 | `Overwrite(value)` | 显式覆盖 |
| 字典标记 | `{"__overwrite__": value}` | 字典形式覆盖 |
| 限制 | 每个 super-step 只能有一个 | 多个会抛出 InvalidUpdateError |

### add_messages 特殊功能

| 功能 | 语法 | 说明 |
|------|------|------|
| 按 ID 合并 | 自动处理 | 相同 ID 的消息会被替换 |
| 删除消息 | `RemoveMessage(id="1")` | 删除特定 ID 的消息 |
| 清空历史 | `RemoveMessage(id=REMOVE_ALL_MESSAGES)` | 清空所有历史消息 |
| 格式转换 | `add_messages(format="langchain-openai")` | 转换为 OpenAI 格式 |

## 常用代码片段速查

### 1. 基础 Reducer 定义

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator
from langgraph.graph.message import add_messages

class State(TypedDict):
    # 列表累积
    items: Annotated[list, operator.add]

    # 字典合并
    config: Annotated[dict, operator.or_]

    # 消息历史
    messages: Annotated[list, add_messages]

    # 数值累加
    count: Annotated[int, operator.add]

    # 简单覆盖（无 Reducer）
    status: str
```

### 2. 自定义 Reducer

```python
def custom_reducer(old: list, new: list) -> list:
    """去重追加"""
    result = old.copy()
    for item in new:
        if item not in result:
            result.append(item)
    return result

class State(TypedDict):
    unique_items: Annotated[list, custom_reducer]
```

### 3. 使用 Overwrite

```python
from langgraph.types import Overwrite

# 方式 1: Overwrite 对象
def node(state: State) -> dict:
    return {"items": Overwrite([1, 2, 3])}

# 方式 2: 字典标记
def node(state: State) -> dict:
    return {"items": {"__overwrite__": [1, 2, 3]}}
```

### 4. add_messages 用法

```python
from langgraph.graph.message import add_messages, REMOVE_ALL_MESSAGES, RemoveMessage

class ChatState(TypedDict):
    messages: Annotated[list, add_messages]

# 添加消息
def add_msg(state: ChatState) -> dict:
    return {"messages": [("user", "Hello")]}

# 删除特定消息
def remove_msg(state: ChatState) -> dict:
    return {"messages": [RemoveMessage(id="1")]}

# 清空历史
def clear_history(state: ChatState) -> dict:
    return {"messages": [RemoveMessage(id=REMOVE_ALL_MESSAGES)]}
```

### 5. 部分状态更新

```python
# 只更新需要的字段
def node(state: State) -> dict:
    # 只返回需要更新的字段
    return {"status": "completed"}
    # 不需要返回所有字段
```

### 6. 完整示例：聊天机器人

```python
from langgraph.graph import StateGraph
from langgraph.graph.message import add_messages

class ChatState(TypedDict):
    messages: Annotated[list, add_messages]

def chatbot(state: ChatState) -> dict:
    # 模拟 AI 回复
    return {"messages": [("assistant", "Hello! How can I help?")]}

builder = StateGraph(ChatState)
builder.add_node("chatbot", chatbot)
builder.set_entry_point("chatbot")
builder.set_finish_point("chatbot")

graph = builder.compile()

# 执行
result = graph.invoke({"messages": [("user", "Hi")]})
print(result["messages"])
# [HumanMessage("Hi"), AIMessage("Hello! How can I help?")]
```

## 常见问题速查

### Q1: Reducer 什么时候被调用？
**A**: 当多个节点返回同一个状态字段的更新时，Reducer 会被调用来合并这些值。第一个值直接赋值，后续值才使用 Reducer。

### Q2: 如何选择合适的 Reducer？
**A**:
- 列表追加 → `operator.add`
- 字典合并 → `operator.or_`
- 消息列表 → `add_messages`
- 简单覆盖 → 不使用 Reducer

### Q3: Reducer 可以修改输入值吗？
**A**: 不推荐。Reducer 应该是纯函数，不应该修改输入值，而是创建新值返回。

### Q4: 如何清空对话历史？
**A**: 使用 `RemoveMessage(id=REMOVE_ALL_MESSAGES)`。

### Q5: operator.or_ 会递归合并嵌套字典吗？
**A**: 不会。operator.or_ 只合并顶层键，嵌套字典会被整体替换。如需递归合并，使用自定义 Reducer。

## 记忆口诀

**"两值进，一值出，定义合并不糊涂"**

- **两值进**：接收旧值和新值
- **一值出**：返回合并后的值
- **定义合并**：明确指定合并策略
- **不糊涂**：避免状态更新的混乱

## 总结

**Reducer 函数是 LangGraph 状态管理的核心机制**：

1. **定义明确**：`(Value, Value) -> Value` 签名
2. **绑定简单**：`Annotated[type, reducer]`
3. **验证严格**：必须恰好 2 个参数
4. **自动调用**：框架在状态更新时自动执行
5. **纯函数**：不产生副作用
6. **灵活强大**：支持内置和自定义 Reducer
7. **特殊机制**：Overwrite 和部分状态更新
8. **实战导向**：聊天机器人、RAG 系统、配置管理

**关键要点**:
- 优先使用内置 Reducer
- 保持 Reducer 简单、快速、无副作用
- 处理边界情况（None、空值）
- 使用类型注解提高可读性
- 理解有 Reducer 和无 Reducer 的区别
- 掌握 add_messages 的特殊功能
