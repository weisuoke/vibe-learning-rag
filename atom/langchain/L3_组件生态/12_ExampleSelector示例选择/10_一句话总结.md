# ExampleSelector示例选择 - 一句话总结

## 一句话总结（30字以内）

**ExampleSelector 是动态选择 Few-shot 示例的机制，根据输入自动选择最相关示例，提升 Prompt 效果和成本控制。**

## 三个关键要点

### 要点 1：语义相似度选择是核心（⭐⭐⭐⭐⭐）

**核心观点：** SemanticSimilarityExampleSelector 是最常用、效果最好的选择器

**为什么重要？**
- 80% 的场景都使用语义相似度选择
- 基于 Embeddings 和向量相似度，理解语义而非字面匹配
- 2025-2026 年成为主流，社区最佳实践

**快速上手：**
```python
from langchain.prompts.example_selector import SemanticSimilarityExampleSelector
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

# 创建选择器（一行代码）
selector = SemanticSimilarityExampleSelector.from_examples(
    examples=[{"input": "2+2", "output": "4"}, ...],
    embeddings=OpenAIEmbeddings(),
    vectorstore_cls=Chroma,
    k=3  # 推荐 3-5
)

# 使用选择器
selected = selector.select_examples({"input": "3+3"})
```

**在 AI Agent 开发中的应用：**
- 问答系统：根据问题选择相似的 QA 对
- RAG 系统：动态选择 Few-shot 示例优化 Prompt
- 代码生成：根据需求选择相似的代码示例

### 要点 2：与 FewShotPromptTemplate 无缝集成（⭐⭐⭐⭐）

**核心观点：** ExampleSelector 通过 FewShotPromptTemplate 集成到 LangChain 生态

**为什么重要？**
- ExampleSelector 的主要使用方式
- 支持 LCEL 链式调用
- 与 LLM 无缝集成

**快速上手：**
```python
from langchain.prompts import FewShotPromptTemplate, PromptTemplate

# 定义示例模板
example_prompt = PromptTemplate(
    input_variables=["input", "output"],
    template="Input: {input}\nOutput: {output}"
)

# 创建 FewShotPromptTemplate
few_shot_prompt = FewShotPromptTemplate(
    example_selector=selector,  # 使用动态选择器
    example_prompt=example_prompt,
    prefix="Solve the following problems:",
    suffix="Input: {input}\nOutput:",
    input_variables=["input"]
)

# 与 LLM 集成
chain = few_shot_prompt | llm
result = chain.invoke({"input": "5+5"})
```

**在 AI Agent 开发中的应用：**
- Agent Prompt 优化：动态选择最相关的工具使用示例
- 对话系统：根据上下文选择相关的对话示例
- 多模态 Agent：选择包含图像的示例

### 要点 3：k 参数调优是关键（⭐⭐⭐⭐）

**核心观点：** k=3-5 是 2025-2026 年的共识，平衡效果和成本

**为什么重要？**
- k 值直接影响输出质量和 API 成本
- k 值过小：示例太少，模型理解不足
- k 值过大：引入噪音，增加成本

**最佳实践：**

| k 值 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| k=1-2 | 简单任务、示例库小 | 成本低 | 效果可能不佳 |
| k=3-5 | 大多数场景（推荐） | 平衡效果和成本 | - |
| k=6-10 | 复杂任务、需要更多上下文 | 效果可能更好 | 成本高、可能引入噪音 |

**成本对比：**
```
全部示例（100个）：5000 tokens → $0.15（GPT-4）
动态选择（5个）：250 tokens → $0.0075（GPT-4）
节省：95% 成本
```

**在 AI Agent 开发中的应用：**
- 成本优化：减少 Token 使用，降低 API 成本
- 性能优化：减少 Prompt 长度，提高响应速度
- 质量优化：避免无关示例干扰，提高输出质量

## 快速参考卡

### 最常用的代码模板

```python
# 1. 导入
from langchain.prompts.example_selector import SemanticSimilarityExampleSelector
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings
from langchain.prompts import FewShotPromptTemplate, PromptTemplate

# 2. 创建选择器
selector = SemanticSimilarityExampleSelector.from_examples(
    examples=[{"input": "...", "output": "..."}],
    embeddings=OpenAIEmbeddings(model="text-embedding-3-small"),
    vectorstore_cls=Chroma,
    k=3  # 推荐 3-5
)

# 3. 集成到 Prompt
few_shot_prompt = FewShotPromptTemplate(
    example_selector=selector,
    example_prompt=PromptTemplate(...),
    prefix="...",
    suffix="...",
    input_variables=[...]
)

# 4. 使用
chain = few_shot_prompt | llm
result = chain.invoke({"input": "..."})
```

### 核心参数速查表

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `k` | 3-5 | 选择的示例数量 |
| `embeddings` | `OpenAIEmbeddings(model="text-embedding-3-small")` | 成本低、速度快 |
| `vectorstore_cls` | `Chroma` | 易用性好 |

### 常见误区速查

| 误区 | 正确理解 |
|------|----------|
| k 值越大越好 | k=3-5 最佳，过大引入噪音 |
| 不考虑 Token 限制 | 结合长度限制或混合策略 |
| 不缓存 Embeddings | 缓存示例的 Embeddings，只计算一次 |

## 总结

### 核心价值

ExampleSelector 通过动态选择最相关的 Few-shot 示例，实现了：
1. **提高输出质量**：相关示例 → 更好的理解 → 更准确的输出
2. **优化成本**：只选择必要的示例 → 减少 Token → 降低 API 成本（节省 95%）
3. **适应不同场景**：多种选择策略 → 灵活应对不同任务

### 2025-2026 年关键趋势

- **语义相似度成为主流**：SemanticSimilarityExampleSelector 最常用
- **混合策略兴起**：语义相似度 + 长度限制
- **k=3-5 成为共识**：平衡效果和成本
- **性能优化关注**：缓存 Embeddings、批量处理

### 在 AI Agent 开发中的地位

ExampleSelector 是 AI Agent 开发的核心组件：
- **Few-shot Learning**：Agent 快速适应新任务的关键
- **Prompt 优化**：减少 Token、提高响应速度、降低成本
- **多模态支持**：选择包含图像、视频的示例
