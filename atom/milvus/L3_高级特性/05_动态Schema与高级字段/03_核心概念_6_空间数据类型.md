# 核心概念6：空间数据类型

> 本文档详细讲解 Milvus 2.6 空间数据类型的 GIS 函数支持、WKT 格式和地理位置检索应用

---

## 一句话定义

**空间数据类型是 Milvus 支持的地理位置数据类型,通过 WKT 格式表示空间几何对象,提供 9 种 GIS 空间关系操作,适用于地理位置检索、地理围栏和空间分析场景。**

---

## 空间数据类型基础

### 1. Geolocation 数据类型

**定义**（来源：reference/context7_milvus_02.md）:

Milvus 支持多种高级数据类型:
- Sparse Vectors（稀疏向量）
- Binary Vectors（二进制向量）
- JSON Support（JSON 支持）
- Array Support（数组支持）
- Text（文本，开发中）
- **Geolocation（地理位置，开发中）**

**当前状态**:
- Geolocation 数据类型正在开发中（under development）
- 从源码分析中看到 `GISFunctionFilterExpr` 支持 WKT 格式和 9 种 GIS 操作
- 预计在 Milvus 2.6+ 版本中正式发布

---

### 2. WKT 格式

**定义**: WKT (Well-Known Text) 是一种文本标记语言,用于表示空间几何对象。

**支持的几何类型**:

| 几何类型 | WKT 格式 | 示例 |
|---------|---------|------|
| Point | POINT(x y) | POINT(116.4074 39.9042) |
| LineString | LINESTRING(x1 y1, x2 y2, ...) | LINESTRING(0 0, 1 1, 2 2) |
| Polygon | POLYGON((x1 y1, x2 y2, ...)) | POLYGON((0 0, 4 0, 4 4, 0 4, 0 0)) |
| MultiPoint | MULTIPOINT((x1 y1), (x2 y2), ...) | MULTIPOINT((0 0), (1 1)) |
| MultiLineString | MULTILINESTRING((...), (...)) | MULTILINESTRING((0 0, 1 1), (2 2, 3 3)) |
| MultiPolygon | MULTIPOLYGON(((...)), ((...))) | MULTIPOLYGON(((0 0, 4 0, 4 4, 0 4, 0 0))) |

**示例**:

```python
# Point（点）
beijing = "POINT(116.4074 39.9042)"  # 北京天安门

# Polygon（多边形）
beijing_area = "POLYGON((116.3 39.8, 116.5 39.8, 116.5 40.0, 116.3 40.0, 116.3 39.8))"

# LineString（线）
route = "LINESTRING(116.4074 39.9042, 121.4737 31.2304)"  # 北京到上海
```

## GIS 空间关系操作

### 1. GIS 函数定义

**源码定义**（来源：reference/source_动态Schema_01.md）:

```protobuf
message GISFunctionFilterExpr {
  ColumnInfo column_info = 1;
  string wkt_string = 2;  // Well-Known Text 格式
  enum GISOp {
    Invalid = 0;
    Equals = 1;      // 相等
    Touches = 2;     // 接触
    Overlaps = 3;    // 重叠
    Crosses = 4;     // 交叉
    Contains = 5;    // 包含
    Intersects = 6;  // 相交
    Within = 7;      // 在内部
    DWithin = 8;     // 距离内
    STIsValid = 9;   // 验证有效性
  }
  GISOp op = 3;
  double distance = 4;  // DWithin 操作的距离参数
}
```

---

### 2. 9 种 GIS 操作详解

#### 操作1：Equals（相等）

**定义**: 判断两个几何对象是否完全相等。

**应用场景**: 检查两个位置是否完全相同。

**示例**:

```python
# 查询与指定点完全相等的位置
results = client.search(
    collection_name="locations",
    data=[query_vector],
    filter='gis_equals(location, "POINT(116.4074 39.9042)")',
    limit=10
)
```

---

#### 操作2：Touches（接触）

**定义**: 判断两个几何对象是否接触（边界相交但内部不相交）。

**应用场景**: 检查两个区域是否相邻。

**示例**:

```python
# 查询与指定区域接触的区域
results = client.search(
    collection_name="regions",
    data=[query_vector],
    filter='gis_touches(region, "POLYGON((0 0, 4 0, 4 4, 0 4, 0 0))")',
    limit=10
)
```

---

#### 操作3：Overlaps（重叠）

**定义**: 判断两个几何对象是否重叠（部分相交）。

**应用场景**: 检查两个区域是否有重叠部分。

**示例**:

```python
# 查询与指定区域重叠的区域
results = client.search(
    collection_name="regions",
    data=[query_vector],
    filter='gis_overlaps(region, "POLYGON((2 2, 6 2, 6 6, 2 6, 2 2))")',
    limit=10
)
```

---

#### 操作4：Crosses（交叉）

**定义**: 判断两个几何对象是否交叉（线与线、线与面相交）。

**应用场景**: 检查道路是否穿过某个区域。

**示例**:

```python
# 查询与指定线路交叉的区域
results = client.search(
    collection_name="roads",
    data=[query_vector],
    filter='gis_crosses(road, "LINESTRING(0 0, 10 10)")',
    limit=10
)
```

---

#### 操作5：Contains（包含）

**定义**: 判断一个几何对象是否完全包含另一个几何对象。

**应用场景**: 检查某个点是否在某个区域内。

**示例**:

```python
# 查询包含指定点的区域
results = client.search(
    collection_name="regions",
    data=[query_vector],
    filter='gis_contains(region, "POINT(116.4074 39.9042)")',
    limit=10
)
```

---

#### 操作6：Intersects（相交）

**定义**: 判断两个几何对象是否相交（有任何交集）。

**应用场景**: 检查两个区域是否有交集。

**示例**:

```python
# 查询与指定区域相交的区域
results = client.search(
    collection_name="regions",
    data=[query_vector],
    filter='gis_intersects(region, "POLYGON((2 2, 6 2, 6 6, 2 6, 2 2))")',
    limit=10
)
```

---

#### 操作7：Within（在内部）

**定义**: 判断一个几何对象是否完全在另一个几何对象内部。

**应用场景**: 检查某个点是否在某个区域内。

**示例**:

```python
# 查询在指定区域内的点
results = client.search(
    collection_name="locations",
    data=[query_vector],
    filter='gis_within(location, "POLYGON((116.3 39.8, 116.5 39.8, 116.5 40.0, 116.3 40.0, 116.3 39.8))")',
    limit=10
)
```

---

#### 操作8：DWithin（距离内）

**定义**: 判断两个几何对象之间的距离是否在指定范围内。

**应用场景**: 查找附近的位置（如附近的餐厅、酒店）。

**示例**:

```python
# 查询距离指定点 1000 米内的位置
results = client.search(
    collection_name="locations",
    data=[query_vector],
    filter='gis_dwithin(location, "POINT(116.4074 39.9042)", 1000)',  # 1000 米
    limit=10
)
```

**参数说明**:
- `distance`: 距离参数（单位：米）
- 支持任意距离值

---

#### 操作9：STIsValid（验证有效性）

**定义**: 验证几何对象的 WKT 格式是否有效。

**应用场景**: 数据验证、错误检查。

**示例**:

```python
# 验证 WKT 格式是否有效
is_valid = gis_is_valid("POINT(116.4074 39.9042)")  # True
is_valid = gis_is_valid("POINT(116.4074)")  # False（缺少 y 坐标）
```

## 实际应用场景

### 场景1：地理位置检索

**问题**: 查找附近的餐厅、酒店、商店。

**解决方案**:

```python
from pymilvus import MilvusClient, DataType

client = MilvusClient("http://localhost:19530")

# 创建 Schema（假设 Geolocation 已支持）
schema = client.create_schema(auto_id=False)
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="name", datatype=DataType.VARCHAR, max_length=256)
schema.add_field(field_name="location", datatype=DataType.GEOLOCATION)  # 地理位置字段

# 插入数据
data = [
    {
        "id": 1,
        "vector": [0.1] * 768,
        "name": "Restaurant A",
        "location": "POINT(116.4074 39.9042)"  # WKT 格式
    },
    {
        "id": 2,
        "vector": [0.2] * 768,
        "name": "Hotel B",
        "location": "POINT(116.4100 39.9050)"
    }
]

client.insert(collection_name="places", data=data)

# 查询附近 1000 米内的地点
user_location = "POINT(116.4074 39.9042)"
results = client.search(
    collection_name="places",
    data=[[0.1] * 768],
    filter=f'gis_dwithin(location, "{user_location}", 1000)',
    output_fields=["name", "location"],
    limit=10
)

for hit in results[0]:
    print(f"Name: {hit['name']}, Location: {hit['location']}, Distance: {hit['distance']}")
```

### 场景2：地理围栏

**问题**: 检查用户是否进入或离开某个区域。

**解决方案**:

```python
# 定义地理围栏（多边形）
geofence = "POLYGON((116.3 39.8, 116.5 39.8, 116.5 40.0, 116.3 40.0, 116.3 39.8))"

# 检查用户位置是否在围栏内
user_location = "POINT(116.4074 39.9042)"
results = client.search(
    collection_name="users",
    data=[[0.1] * 768],
    filter=f'gis_within(location, "{geofence}")',
    output_fields=["user_id", "location"],
    limit=10
)

# 输出在围栏内的用户
for hit in results[0]:
    print(f"User {hit['user_id']} is inside the geofence")
```

### 场景3：空间分析

**问题**: 分析两个区域是否重叠、相交或相邻。

**解决方案**:

```python
# 定义两个区域
region_a = "POLYGON((0 0, 4 0, 4 4, 0 4, 0 0))"
region_b = "POLYGON((2 2, 6 2, 6 6, 2 6, 2 2))"

# 检查两个区域是否重叠
results = client.search(
    collection_name="regions",
    data=[[0.1] * 768],
    filter=f'gis_overlaps(region, "{region_b}")',
    output_fields=["region_name", "region"],
    limit=10
)

# 输出重叠的区域
for hit in results[0]:
    print(f"Region {hit['region_name']} overlaps with the query region")
```

## 当前限制与未来展望

### 当前限制

**限制1**: Geolocation 数据类型正在开发中
- 当前版本（Milvus 2.6）可能尚未完全支持
- 需要等待官方正式发布

**限制2**: GIS 函数的性能
- 空间计算比标量过滤更复杂
- 大数据量下性能可能下降

**限制3**: 索引支持
- 空间索引（如 R-tree）可能尚未支持
- 需要全表扫描进行空间查询

---

### 未来展望

**展望1**: 完整的 Geolocation 支持
- 正式发布 Geolocation 数据类型
- 提供完整的 GIS 函数库

**展望2**: 空间索引优化
- 支持 R-tree、Quad-tree 等空间索引
- 提升空间查询性能

**展望3**: 更多 GIS 操作
- 支持距离计算（Distance）
- 支持缓冲区分析（Buffer）
- 支持空间聚合（Spatial Aggregation）

## 替代方案（当前可用）

### 方案1：使用标量字段存储经纬度

**实现**:

```python
# 创建 Schema
schema = client.create_schema(auto_id=False)
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="latitude", datatype=DataType.DOUBLE)  # 纬度
schema.add_field(field_name="longitude", datatype=DataType.DOUBLE)  # 经度

# 插入数据
data = [
    {
        "id": 1,
        "vector": [0.1] * 768,
        "latitude": 39.9042,
        "longitude": 116.4074
    }
]

# 查询附近的位置（简化版）
results = client.search(
    collection_name="locations",
    data=[[0.1] * 768],
    filter='latitude >= 39.9 and latitude <= 40.0 and longitude >= 116.4 and longitude <= 116.5',
    limit=10
)
```

**优势**:
- 当前版本即可使用
- 简单易实现

**劣势**:
- 不支持复杂的空间操作
- 精度较低

---

### 方案2：使用 JSON 字段存储 WKT

**实现**:

```python
# 创建 Schema
schema = client.create_schema(auto_id=False)
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="location", datatype=DataType.JSON)  # JSON 字段

# 插入数据
data = [
    {
        "id": 1,
        "vector": [0.1] * 768,
        "location": {
            "type": "Point",
            "coordinates": [116.4074, 39.9042],
            "wkt": "POINT(116.4074 39.9042)"
        }
    }
]

# 查询（在应用层进行空间计算）
results = client.search(
    collection_name="locations",
    data=[[0.1] * 768],
    output_fields=["location"],
    limit=100
)

# 在应用层过滤
from shapely.geometry import Point
from shapely.wkt import loads

user_location = Point(116.4074, 39.9042)
filtered_results = []

for hit in results[0]:
    location_wkt = hit['location']['wkt']
    location = loads(location_wkt)
    distance = user_location.distance(location)
    if distance < 0.01:  # 约 1000 米
        filtered_results.append(hit)
```

**优势**:
- 支持复杂的空间操作（在应用层）
- 灵活性高

**劣势**:
- 需要在应用层进行空间计算
- 性能较低

### 实践3：性能优化

**建议**:
- 使用向量检索缩小候选集
- 在应用层进行精确的空间过滤
- 使用缓存减少重复计算

**示例**:

```python
# 步骤1：使用向量检索缩小候选集
results = client.search(
    collection_name="locations",
    data=[[0.1] * 768],
    limit=100,  # 先检索 100 个候选
    output_fields=["latitude", "longitude"]
)

# 步骤2：在应用层进行精确的空间过滤
from geopy.distance import geodesic

user_location = (39.9042, 116.4074)
filtered_results = []

for hit in results[0]:
    location = (hit['latitude'], hit['longitude'])
    distance = geodesic(user_location, location).meters
    if distance < 1000:  # 1000 米内
        filtered_results.append(hit)
```
