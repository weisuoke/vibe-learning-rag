# 最小可用知识

> 掌握以下内容，就能开始使用 Tools 与函数调用构建能"做事"的 Agent

---

## 核心理念

**20% 的核心知识解决 80% 的问题**

Tool 系统涉及定义方式、Schema 验证、消息协议、工具选择策略等大量细节，但日常开发中你只需要掌握 4 个步骤：**定义工具 → 绑定模型 → 处理调用 → 回传结果**。本文档把这 4 步拆解为可直接复制使用的代码模式。

---

## 4.1 用 @tool 装饰器定义工具（最常用）

### 核心概念

`@tool` 装饰器是定义工具的首选方式，覆盖 80% 以上场景。它的规则极其简单：

**函数名 → 工具名，docstring → 工具描述，类型注解 → 参数 Schema**

三要素齐全，一个完整的 Tool 就定义好了。

### 基础示例

```python
from langchain_core.tools import tool

@tool
def get_weather(city: str) -> str:
    """查询指定城市的实时天气信息"""
    # 实际项目中这里调用天气 API
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云，28°C，湿度 65%",
    }
    return weather_data.get(city, f"{city}：暂无天气数据")

# 验证自动提取的信息
print(get_weather.name)         # "get_weather"
print(get_weather.description)  # "查询指定城市的实时天气信息"

# Tool 也是 Runnable，可以直接调用
result = get_weather.invoke({"city": "北京"})
print(result)  # "晴，25°C，湿度 40%"
```

### 多参数 + docstring 自动提取

当工具有多个参数时，开启 `parse_docstring=True`，LangChain 会从 Google-style docstring 自动提取每个参数的描述：

```python
from langchain_core.tools import tool

@tool(parse_docstring=True)
def search_documents(query: str, top_k: int = 5) -> str:
    """在知识库中搜索与查询相关的文档。

    Args:
        query: 用户的搜索查询文本
        top_k: 返回结果数量，默认 5 条
    """
    return f"搜索 '{query}'，返回前 {top_k} 条结果"

# 参数描述已自动写入 Schema
schema = search_documents.args_schema.model_json_schema()
print(schema["properties"]["query"]["description"])
# → "用户的搜索查询文本"
```

### 记住这个公式

```
@tool（+ parse_docstring=True）
def 函数名(参数: 类型) -> 返回类型:
    """工具描述（给 LLM 看的）

    Args:
        参数: 参数描述（给 LLM 看的）
    """
    return 执行结果
```

## 4.2 用 bind_tools() 绑定工具到模型

### 核心概念

定义好工具后，需要通过 `bind_tools()` 告诉模型"你有哪些工具可以用"。这一步把工具的 Schema（名称、描述、参数）注入到模型的请求中。

### 基础示例

```python
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from dotenv import load_dotenv

load_dotenv()

@tool
def get_weather(city: str) -> str:
    """查询指定城市的实时天气信息"""
    return f"{city}今天晴，25°C"

@tool
def calculator(expression: str) -> str:
    """计算数学表达式的结果"""
    return str(eval(expression))

# 绑定工具到模型
model = ChatOpenAI(model="gpt-4o-mini")
model_with_tools = model.bind_tools([get_weather, calculator])

# model_with_tools 是一个新对象，原 model 不受影响
# 现在模型知道自己可以调用 get_weather 和 calculator 了
```

### 关键点

- `bind_tools()` 返回新对象，不修改原模型
- 传入的是工具列表 `[tool1, tool2, ...]`
- 模型会根据用户问题自动决定是否调用工具、调用哪个

## 4.3 处理 tool_calls 和 ToolMessage

### 核心概念

模型绑定工具后，面对用户问题会做一个决策：

```
用户提问 → 模型思考 → 需要工具？
                        ├─ 是 → 返回 AIMessage(tool_calls=[...])
                        └─ 否 → 返回 AIMessage(content="直接回答")
```

你需要做的是：**检查 `tool_calls` → 执行工具 → 用 `ToolMessage` 把结果传回去**。

### 处理流程代码

```python
from langchain_core.messages import HumanMessage, ToolMessage

# 第一步：发送用户消息，获取模型响应
messages = [HumanMessage(content="北京今天天气怎么样？")]
ai_response = model_with_tools.invoke(messages)

# 第二步：检查模型是否要调用工具
if ai_response.tool_calls:
    print("模型决定调用工具：")
    for tc in ai_response.tool_calls:
        print(f"  工具: {tc['name']}, 参数: {tc['args']}, ID: {tc['id']}")
else:
    print(f"模型直接回答: {ai_response.content}")
```

### 执行工具并回传结果

```python
# 把模型的响应加入消息列表（保持完整对话历史）
messages.append(ai_response)

# 构建工具名 → 工具对象的映射
tool_map = {"get_weather": get_weather, "calculator": calculator}

# 逐个执行工具调用
for tc in ai_response.tool_calls:
    # 执行工具
    tool_result = tool_map[tc["name"]].invoke(tc["args"])

    # 构造 ToolMessage，通过 tool_call_id 关联到对应的请求
    messages.append(ToolMessage(
        content=tool_result,
        tool_call_id=tc["id"]  # 必须与 tc["id"] 匹配
    ))
```

### tool_call_id 的作用

`tool_call_id` 是 ToolCall 和 ToolMessage 之间的"纽带"。模型可能一次调用多个工具，每个 ToolMessage 必须通过 `tool_call_id` 告诉模型"这是哪个调用的结果"：

```
AIMessage.tool_calls[0].id = "call_001"  ←→  ToolMessage(tool_call_id="call_001")
AIMessage.tool_calls[1].id = "call_002"  ←→  ToolMessage(tool_call_id="call_002")
```

## 4.4 完整的最小工作流

### 一个完整的 定义 → 绑定 → 调用 → 回传 示例

```python
"""
Tools 与函数调用 - 最小可用完整示例
演示：定义工具 → 绑定模型 → 处理调用 → 回传结果 → 获取最终回答
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

load_dotenv()

# ===== 第 1 步：定义工具 =====
@tool(parse_docstring=True)
def get_weather(city: str) -> str:
    """查询指定城市的实时天气信息。

    Args:
        city: 要查询天气的城市名称
    """
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云，28°C，湿度 65%",
        "广州": "小雨，30°C，湿度 80%",
    }
    return weather_data.get(city, f"{city}：暂无天气数据")

@tool
def calculator(expression: str) -> str:
    """计算数学表达式的结果，如 '2 + 3 * 4'"""
    try:
        return f"{expression} = {eval(expression)}"
    except Exception as e:
        return f"计算错误：{e}"

# ===== 第 2 步：绑定工具到模型 =====
model = ChatOpenAI(model="gpt-4o-mini")
model_with_tools = model.bind_tools([get_weather, calculator])

# ===== 第 3 步：发送用户消息 =====
messages = [HumanMessage(content="北京今天天气怎么样？")]
ai_response = model_with_tools.invoke(messages)
print(f"模型响应 tool_calls: {ai_response.tool_calls}")

# ===== 第 4 步：执行工具，构造 ToolMessage =====
messages.append(ai_response)

tool_map = {"get_weather": get_weather, "calculator": calculator}
for tc in ai_response.tool_calls:
    result = tool_map[tc["name"]].invoke(tc["args"])
    print(f"工具 {tc['name']} 执行结果: {result}")
    messages.append(ToolMessage(content=result, tool_call_id=tc["id"]))

# ===== 第 5 步：模型基于工具结果生成最终回答 =====
final_response = model_with_tools.invoke(messages)
print(f"最终回答: {final_response.content}")
```

**预期输出：**

```
模型响应 tool_calls: [{'name': 'get_weather', 'args': {'city': '北京'}, 'id': 'call_abc123', 'type': 'tool_call'}]
工具 get_weather 执行结果: 晴，25°C，湿度 40%
最终回答: 北京今天天气晴朗，气温25°C，湿度40%，非常适合户外活动。
```

### 完整消息链路可视化

```
messages 列表的变化过程：

[HumanMessage("北京今天天气怎么样？")]
    ↓ model_with_tools.invoke()
[HumanMessage, AIMessage(tool_calls=[{get_weather, {city:"北京"}, call_001}])]
    ↓ 执行工具 + 构造 ToolMessage
[HumanMessage, AIMessage, ToolMessage("晴，25°C", tool_call_id="call_001")]
    ↓ model_with_tools.invoke()
[HumanMessage, AIMessage, ToolMessage, AIMessage("北京今天天气晴朗...")]
```

### 通用工具调用循环（生产可用模式）

实际项目中，模型可能需要多轮工具调用才能回答问题。以下是一个通用的循环模式：

```python
def run_tool_loop(model_with_tools, tools, user_message: str, max_rounds: int = 5):
    """通用工具调用循环：自动处理多轮工具调用直到模型给出最终回答"""
    tool_map = {t.name: t for t in tools}
    messages = [HumanMessage(content=user_message)]

    for _ in range(max_rounds):
        response = model_with_tools.invoke(messages)
        messages.append(response)

        # 没有 tool_calls → 模型给出了最终回答
        if not response.tool_calls:
            return response.content

        # 有 tool_calls → 执行工具并回传结果
        for tc in response.tool_calls:
            try:
                result = tool_map[tc["name"]].invoke(tc["args"])
            except Exception as e:
                result = f"工具执行失败: {e}"
            messages.append(ToolMessage(content=result, tool_call_id=tc["id"]))

    return "达到最大调用轮次，未能得到最终回答"

# 使用
answer = run_tool_loop(
    model_with_tools,
    tools=[get_weather, calculator],
    user_message="北京天气怎么样？如果气温超过 20 度，帮我算一下 25 * 1.8 + 32 是多少华氏度"
)
print(answer)
```

## 4.5 常用参数速查

### @tool 装饰器参数

| 参数 | 默认值 | 说明 | 使用频率 |
|------|--------|------|----------|
| `parse_docstring` | `False` | 从 Google-style docstring 提取参数描述 | 高 |
| `description` | docstring | 覆盖 docstring 作为工具描述 | 中 |
| `args_schema` | 自动推断 | 自定义 Pydantic BaseModel 作为参数 Schema | 中 |
| `return_direct` | `False` | `True` 时工具结果直接返回用户，跳过 LLM | 低 |
| `response_format` | `"content"` | `"content_and_artifact"` 可分离摘要和完整数据 | 低 |

**最常用组合：**

```python
# 简单工具：无参数装饰器
@tool
def simple_tool(query: str) -> str:
    """工具描述"""
    ...

# 多参数工具：开启 docstring 解析
@tool(parse_docstring=True)
def multi_param_tool(query: str, top_k: int = 5) -> str:
    """工具描述。

    Args:
        query: 参数描述
        top_k: 参数描述
    """
    ...
```

### bind_tools() 参数

| 参数 | 默认值 | 说明 | 使用频率 |
|------|--------|------|----------|
| `tools` | 必填 | 工具列表 | 必用 |
| `tool_choice` | `None`（等同 `"auto"`） | 控制模型如何选择工具 | 中 |

**tool_choice 速查：**

| 值 | 含义 | 典型场景 |
|------|------|----------|
| `"auto"` | 模型自行决定是否调用工具 | 通用对话（默认） |
| `"any"` | 必须调用至少一个工具 | 确保工具被使用 |
| `"none"` | 禁止调用工具 | 临时禁用工具 |
| `"get_weather"` | 必须调用指定名称的工具 | 强制使用特定工具 |

```python
# 默认：模型自动决定
model.bind_tools([get_weather, calculator])

# 强制调用工具（不允许直接回答）
model.bind_tools([get_weather], tool_choice="any")

# 强制调用指定工具
model.bind_tools([get_weather, calculator], tool_choice="get_weather")
```

### ToolMessage 参数

| 参数 | 必填 | 说明 |
|------|------|------|
| `content` | 是 | 工具执行结果（发送给模型的文本） |
| `tool_call_id` | 是 | 关联到对应的 ToolCall（必须与 `tc["id"]` 匹配） |
| `status` | 否 | `"success"`（默认）或 `"error"` |

```python
# 成功
ToolMessage(content="北京：晴，25°C", tool_call_id=tc["id"])

# 失败（告知模型执行出错）
ToolMessage(content="API 超时，请稍后重试", tool_call_id=tc["id"], status="error")
```
