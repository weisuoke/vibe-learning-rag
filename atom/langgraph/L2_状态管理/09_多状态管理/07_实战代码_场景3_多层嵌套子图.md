# 实战代码 - 场景3：多层嵌套子图

## 场景说明

本场景演示 LangGraph 中**三层嵌套子图**（parent → child → grandchild）的状态管理。每一层拥有完全独立的 TypedDict 状态定义，数据通过**显式转换**在层级之间传递。

这是多状态管理中最能体现"子图状态隔离"价值的场景：三层图各自有不同的字段名，互不干扰，通过函数包装器（wrapper function）完成状态映射。

核心知识点：
1. 三层独立 TypedDict——每层有自己的字段名，互不知道对方的存在
2. 函数包装器模式——用普通函数调用子图，手动做状态映射（输入转换 + 输出转换）
3. 命名空间层级——LangGraph 自动为嵌套子图创建 `parent|child|grandchild` 命名空间
4. `subgraphs=True` 流式输出——观察每一层子图的执行过程
5. 状态隔离的好处——修改某一层的 Schema 不影响其他层

[来源: sourcecode/langgraph/libs/langgraph/langgraph/graph/state.py]
[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

```
│  │ parent_1  │───▶│  child   │───▶│ parent_2  │     │
│  └──────────┘    └────┬─────┘    └──────────┘      │
│                       │                             │
│         ┌─────────────▼──────────────┐              │
│         │ Child 图 (ChildState:      │              │
│         │          my_child_key)     │              │
│         │                            │              │
│         │  ┌────────────────────┐    │              │
│         │  │ call_grandchild    │    │              │
│         │  └────────┬───────────┘    │              │
│         │           │                │              │
│         │  ┌────────▼───────────┐    │              │
│         │  │ Grandchild 图      │    │              │
│         │  │ (GrandChildState:  │    │              │
│         │  │  my_grandchild_key)│    │              │
│         │  │                    │    │              │
│         │  │ ┌──────────────┐   │    │              │
│         │  │ │ grandchild_1 │   │    │              │
│         │  │ └──────────────┘   │    │              │
│         │  └────────────────────┘    │              │
│         └────────────────────────────┘              │
└─────────────────────────────────────────────────────┘
```

关键点：三层图各自有独立的 TypedDict，字段名完全不同（`my_key` / `my_child_key` / `my_grandchild_key`）。数据在层级之间通过函数包装器显式转换。

## 完整代码

```python
"""
LangGraph 多状态管理 - 多层嵌套子图
演示：parent → child → grandchild 三层状态管理

每层有独立的 TypedDict 状态，数据通过显式转换传递。
三层图的字段名完全不同，互不知道对方的存在。

核心技术：
- 三层独立 TypedDict（状态隔离）
- 函数包装器模式（状态映射）
- 命名空间层级（自动创建）
- subgraphs=True 流式输出（调试利器）

运行环境：Python 3.13+, langgraph
安装依赖：uv add langgraph
"""

from typing_extensions import TypedDict
from langgraph.graph.state import StateGraph, START, END


# ============================================================
# 1. Grandchild 图（最内层）
# ============================================================

class GrandChildState(TypedDict):
    """最内层状态：只有一个字段 my_grandchild_key

    这个 Schema 完全不知道 Parent 和 Child 的存在。
    它只关心自己的 my_grandchild_key 字段。
    """
    my_grandchild_key: str


def grandchild_1(state: GrandChildState) -> GrandChildState:
    """Grandchild 的唯一节点：追加 ', how are you'

    读取：my_grandchild_key
    写入：my_grandchild_key（覆盖）
    """
    print(f"    [grandchild_1] 收到: {state}")
    return {
        "my_grandchild_key": state["my_grandchild_key"] + ", how are you"
    }


# 构建 Grandchild 图
grandchild_builder = StateGraph(GrandChildState)
grandchild_builder.add_node("grandchild_1", grandchild_1)
grandchild_builder.add_edge(START, "grandchild_1")
grandchild_builder.add_edge("grandchild_1", END)
grandchild_graph = grandchild_builder.compile()

print("=== Grandchild 图构建完成 ===")
print(f"  输入 Schema: GrandChildState (my_grandchild_key)")
print(f"  节点: grandchild_1")


# ============================================================
# 2. Child 图（中间层）
# ============================================================

class ChildState(TypedDict):
    """中间层状态：只有一个字段 my_child_key

    这个 Schema 不知道 Parent 的 my_key，
    也不知道 Grandchild 的 my_grandchild_key。
    它只关心自己的 my_child_key 字段。
    """
    my_child_key: str


def call_grandchild_graph(state: ChildState) -> ChildState:
    """Child 节点：调用 Grandchild 子图

    这是一个「函数包装器」——它的职责是：
    1. 把 ChildState 的字段映射到 GrandChildState 的字段
    2. 调用 grandchild_graph
    3. 把 GrandChildState 的输出映射回 ChildState 的字段

    读取：my_child_key
    写入：my_child_key（覆盖）
    """
    print(f"  [call_grandchild] 收到: {state}")

    # 步骤1：ChildState → GrandChildState（输入映射）
    grandchild_input = {"my_grandchild_key": state["my_child_key"]}

    # 步骤2：调用 Grandchild 子图
    grandchild_output = grandchild_graph.invoke(grandchild_input)

    # 步骤3：GrandChildState → ChildState（输出映射）
    return {
        "my_child_key": grandchild_output["my_grandchild_key"] + " today?"
    }


# 构建 Child 图
child_builder = StateGraph(ChildState)
child_builder.add_node("child_1", call_grandchild_graph)
child_builder.add_edge(START, "child_1")
child_builder.add_edge("child_1", END)
child_graph = child_builder.compile()

print("\n=== Child 图构建完成 ===")
print(f"  输入 Schema: ChildState (my_child_key)")
print(f"  节点: child_1 (内部调用 grandchild_graph)")


# ============================================================
# 3. Parent 图（最外层）
# ============================================================

class ParentState(TypedDict):
    """最外层状态：只有一个字段 my_key

    这个 Schema 不知道 Child 的 my_child_key，
    也不知道 Grandchild 的 my_grandchild_key。
    它只关心自己的 my_key 字段。
    """
    my_key: str


def parent_1(state: ParentState) -> ParentState:
    """Parent 第一个节点：在前面加 'hi '

    读取：my_key
    写入：my_key（覆盖）
    """
    print(f"[parent_1] 收到: {state}")
    return {"my_key": "hi " + state["my_key"]}


def call_child_graph(state: ParentState) -> ParentState:
    """Parent 节点：调用 Child 子图

    又一个「函数包装器」——把 ParentState 映射到 ChildState，
    调用 child_graph，再把结果映射回 ParentState。

    读取：my_key
    写入：my_key（覆盖）
    """
    print(f"[call_child] 收到: {state}")

    # ParentState → ChildState
    child_input = {"my_child_key": state["my_key"]}

    # 调用 Child 子图（Child 内部会再调用 Grandchild）
    child_output = child_graph.invoke(child_input)

    # ChildState → ParentState
    return {"my_key": child_output["my_child_key"]}


def parent_2(state: ParentState) -> ParentState:
    """Parent 最后一个节点：在末尾加 ' bye!'

    读取：my_key
    写入：my_key（覆盖）
    """
    print(f"[parent_2] 收到: {state}")
    return {"my_key": state["my_key"] + " bye!"}


# 构建 Parent 图
parent_builder = StateGraph(ParentState)
parent_builder.add_node("parent_1", parent_1)
parent_builder.add_node("child", call_child_graph)
parent_builder.add_node("parent_2", parent_2)
parent_builder.add_edge(START, "parent_1")
parent_builder.add_edge("parent_1", "child")
parent_builder.add_edge("child", "parent_2")
parent_builder.add_edge("parent_2", END)
parent_graph = parent_builder.compile()

print("\n=== Parent 图构建完成 ===")
print(f"  输入 Schema: ParentState (my_key)")
print(f"  节点: parent_1 → child → parent_2")


# ============================================================
# 4. 测试1：invoke 模式——一次性执行
# ============================================================

print("\n" + "=" * 60)
print("=== 测试1：invoke 模式 - 三层嵌套执行 ===")
print("=" * 60)

result = parent_graph.invoke({"my_key": "Bob"})
print(f"\n最终结果: {result}")
print(f"  输入: 'Bob'")
print(f"  输出: '{result['my_key']}'")


# ============================================================
# 5. 测试2：数据流转全过程追踪
# ============================================================

print("\n" + "=" * 60)
print("=== 测试2：数据流转全过程追踪 ===")
print("=" * 60)

print("""
数据变换过程：

  输入:        "Bob"
     │
  parent_1:    "hi Bob"              ← 前面加 "hi "
     │
  child:       传入 ChildState
     │
  grandchild:  "hi Bob, how are you" ← 追加 ", how are you"
     │
  child 返回:  "hi Bob, how are you today?" ← 追加 " today?"
     │
  parent_2:    "hi Bob, how are you today? bye!" ← 追加 " bye!"
     │
  最终输出:    "hi Bob, how are you today? bye!"
""")


# ============================================================
# 6. 测试3：stream 模式——逐步观察
# ============================================================

print("=" * 60)
print("=== 测试3：stream 模式 - 逐步观察 ===")
print("=" * 60)

for step in parent_graph.stream({"my_key": "Alice"}):
    for node_name, node_output in step.items():
        print(f"\n--- 节点: {node_name} ---")
        for key, value in node_output.items():
            print(f"  {key}: {value}")


# ============================================================
# 7. 测试4：独立测试每一层
# ============================================================

print("\n" + "=" * 60)
print("=== 测试4：独立测试每一层 ===")
print("=" * 60)

# Grandchild 独立运行
gc_result = grandchild_graph.invoke({"my_grandchild_key": "hello"})
print(f"\nGrandchild 独立运行:")
print(f"  输入: 'hello'")
print(f"  输出: '{gc_result['my_grandchild_key']}'")

# Child 独立运行（内部会调用 Grandchild）
c_result = child_graph.invoke({"my_child_key": "hello"})
print(f"\nChild 独立运行:")
print(f"  输入: 'hello'")
print(f"  输出: '{c_result['my_child_key']}'")

# Parent 完整运行
p_result = parent_graph.invoke({"my_key": "hello"})
print(f"\nParent 完整运行:")
print(f"  输入: 'hello'")
print(f"  输出: '{p_result['my_key']}'")

print("\n关键观察：每一层都可以独立测试！")
print("这就是状态隔离的好处——子图不依赖父图的 Schema。")
```

## 关键知识点解析

### 1. 为什么每层需要独立的 TypedDict？

三层图使用了三个完全不同的 TypedDict：

| 层级 | Schema | 字段名 | 职责 |
|------|--------|--------|------|
| Grandchild | `GrandChildState` | `my_grandchild_key` | 最内层处理逻辑 |
| Child | `ChildState` | `my_child_key` | 中间层编排逻辑 |
| Parent | `ParentState` | `my_key` | 最外层业务逻辑 |

这不是巧合，而是**刻意设计**。每层使用不同的字段名，强调了一个核心原则：

> **子图不应该知道父图的 Schema，父图也不应该知道子图的内部结构。**

就像公司的组织架构：总公司（Parent）有自己的报表格式，子公司（Child）有自己的报表格式，子公司的部门（Grandchild）又有自己的格式。它们之间通过"翻译官"（函数包装器）沟通。

### 2. 函数包装器模式详解

函数包装器是多层嵌套的核心模式，它做三件事：

```python
def call_child_graph(state: ParentState) -> ParentState:
    # 1. 输入映射：ParentState → ChildState
    child_input = {"my_child_key": state["my_key"]}

    # 2. 调用子图
    child_output = child_graph.invoke(child_input)

    # 3. 输出映射：ChildState → ParentState
    return {"my_key": child_output["my_child_key"]}
```

为什么不能直接把子图作为节点添加？因为 Schema 不同！如果 Parent 和 Child 有同名字段（共享键），可以直接 `add_node("child", child_graph)` 让 LangGraph 自动映射。但当字段名不同时，必须用函数包装器手动做映射。

**两种子图集成方式对比：**

```python
# 方式1：共享键——字段名相同，自动映射
class ParentState(TypedDict):
    foo: str  # 与子图同名

class SubgraphState(TypedDict):
    foo: str  # 与父图同名
    bar: str  # 子图私有

builder.add_node("sub", subgraph)  # 直接添加，foo 自动传递


# 方式2：函数包装器——字段名不同，手动映射
class ParentState(TypedDict):
    my_key: str  # 父图字段名

class ChildState(TypedDict):
    my_child_key: str  # 子图字段名不同！

def call_child(state: ParentState) -> ParentState:
    result = child_graph.invoke({"my_child_key": state["my_key"]})
    return {"my_key": result["my_child_key"]}

builder.add_node("child", call_child)  # 通过函数包装
```

### 3. 命名空间层级

LangGraph 为嵌套子图自动创建命名空间层级：

```
""                              ← 根图（Parent）
"child:abc123"                  ← 一级子图（Child）
"child:abc123|child_1:def456"   ← 二级子图（Grandchild）
```

命名空间的作用：
- **Checkpoint 隔离**：每层子图的状态快照独立存储
- **调试定位**：通过命名空间可以精确定位问题出在哪一层
- **流式输出**：`subgraphs=True` 时，输出会带上命名空间前缀

源码中的命名空间分隔符：

```python
# langgraph/_internal/_constants.py
NS_SEP = "|"   # 层级分隔：graph|subgraph|subsubgraph
NS_END = ":"   # 命名空间与 task_id 分隔
```

### 4. 数据流转的完整路径

```
输入: {"my_key": "Bob"}

Parent.parent_1:
  读取 my_key = "Bob"
  写入 my_key = "hi Bob"

Parent.child (函数包装器):
  读取 my_key = "hi Bob"
  映射 → {"my_child_key": "hi Bob"}

  Child.child_1 (函数包装器):
    读取 my_child_key = "hi Bob"
    映射 → {"my_grandchild_key": "hi Bob"}

    Grandchild.grandchild_1:
      读取 my_grandchild_key = "hi Bob"
      写入 my_grandchild_key = "hi Bob, how are you"

    映射 ← my_grandchild_key → my_child_key
    追加 " today?"
    写入 my_child_key = "hi Bob, how are you today?"

  映射 ← my_child_key → my_key
  写入 my_key = "hi Bob, how are you today?"

Parent.parent_2:
  读取 my_key = "hi Bob, how are you today?"
  写入 my_key = "hi Bob, how are you today? bye!"

输出: {"my_key": "hi Bob, how are you today? bye!"}
```

### 5. 独立测试的价值

```python
# 每一层都可以独立测试——这是状态隔离的最大好处

# 测试 Grandchild：不需要启动 Parent 和 Child
grandchild_graph.invoke({"my_grandchild_key": "test"})

# 测试 Child：不需要启动 Parent
child_graph.invoke({"my_child_key": "test"})

# 测试 Parent：完整流程
parent_graph.invoke({"my_key": "test"})
```

在 RAG 系统中，这意味着你可以：
- 单独测试检索子图，不需要启动整个 Pipeline
- 单独测试生成子图，用 mock 数据替代检索结果
- 单独测试重排序子图，验证排序逻辑是否正确

## 在 RAG 系统中的应用

三层嵌套在 RAG 系统中的典型场景：

```
Parent（RAG 主流程）
  ├── 查询理解子图（Child 1）
  │     ├── 意图识别子图（Grandchild 1a）
  │     └── 查询改写子图（Grandchild 1b）
  ├── 检索子图（Child 2）
  │     ├── 向量检索子图（Grandchild 2a）
  │     └── 关键词检索子图（Grandchild 2b）
  └── 生成子图（Child 3）
        ├── 上下文组装子图（Grandchild 3a）
        └── 答案生成子图（Grandchild 3b）
```

每一层有自己的状态定义：
- Parent：`question`, `answer`, `sources`
- 检索 Child：`query`, `documents`, `scores`
- 向量检索 Grandchild：`embedding`, `top_k`, `raw_results`

层级之间通过函数包装器做状态映射，修改某一层的内部实现不影响其他层。
