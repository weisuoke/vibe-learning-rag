## 4.1 节点函数的标准写法

**一句话：** 记住一个公式 `State -> dict`，三步走：读取、处理、返回部分更新

```python
from typing_extensions import TypedDict

class State(TypedDict):
    query: str
    result: str

def my_node(state: State) -> dict:
    # 1. 读取需要的状态字段
    query = state["query"]

    # 2. 执行转换逻辑（调用 LLM、检索、计算等）
    result = f"处理结果：{query.upper()}"

    # 3. 只返回要更新的字段（不需要返回 query）
    return {"result": result}
```

**关键点：**
- 节点接收的是完整状态的只读快照，你可以读取任意字段
- 返回值是一个普通 `dict`，只包含你想更新的字段
- 没有出现在返回字典中的字段保持原值不变
- 这就是"部分状态更新"机制——你不需要把整个 state 搬回去

**三步法口诀：读取 → 处理 → 返回部分更新**

**完整可运行示例：**

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    query: str
    result: str

def process_node(state: State) -> dict:
    query = state["query"]
    return {"result": f"已处理: {query.strip().lower()}"}

builder = StateGraph(State)
builder.add_node("process", process_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()
output = graph.invoke({"query": "  什么是 RAG？  "})
print(output["result"])  # 已处理: 什么是 rag？
```

## 4.2 有 Reducer 字段的返回规则

**一句话：** 无 Reducer 的字段返回新值（覆盖），有 Reducer 的字段返回增量（追加）

```python
from typing import Annotated
from typing_extensions import TypedDict
from operator import add
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    count: int                        # 无 reducer → 覆盖
    items: Annotated[list[str], add]  # 有 reducer → 追加

def node_a(state: State) -> dict:
    return {
        "count": 42,           # 直接覆盖为 42
        "items": ["from_a"]    # 追加 "from_a" 到列表
    }

def node_b(state: State) -> dict:
    return {
        "count": 99,           # 覆盖为 99（42 被替换）
        "items": ["from_b"]    # 追加 "from_b"（列表变为 ["from_a", "from_b"]）
    }

builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", END)

graph = builder.compile()
result = graph.invoke({"count": 0, "items": []})
print(result["count"])  # 99（被覆盖）
print(result["items"])  # ['from_a', 'from_b']（被追加）
```

**关键点：**
- 无 Reducer 字段：你返回什么，字段就变成什么（覆盖语义）
- 有 Reducer 字段：你返回的是"增量"，框架用 Reducer 函数合并旧值和新值
- 最常见的错误：对有 Reducer 的字段返回完整列表，导致重复累积

```python
# ❌ 错误：手动拼接旧值 + 新值
def bad_node(state: State) -> dict:
    return {"items": state["items"] + ["new"]}  # 会重复！

# ✅ 正确：只返回增量
def good_node(state: State) -> dict:
    return {"items": ["new"]}  # Reducer 自动追加
```

## 4.3 安全的空值处理

**一句话：** 用 `.get()` 或默认值防御空字段，避免 KeyError 崩溃

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    query: str
    context: str
    error: str

def safe_node(state: State) -> dict:
    # 方式1：用 .get() 提供默认值
    query = state.get("query") or ""

    # 方式2：条件检查后兜底
    if not query.strip():
        return {"error": "查询为空，请提供有效问题"}

    # 正常处理
    return {"context": f"检索到关于 '{query}' 的内容"}

builder = StateGraph(State)
builder.add_node("safe", safe_node)
builder.add_edge(START, "safe")
builder.add_edge("safe", END)

graph = builder.compile()

# 正常输入
result = graph.invoke({"query": "RAG 是什么？", "context": "", "error": ""})
print(result["context"])  # 检索到关于 'RAG 是什么？' 的内容

# 空输入
result = graph.invoke({"query": "", "context": "", "error": ""})
print(result["error"])  # 查询为空，请提供有效问题
```

**关键点：**
- `state["key"]` 在 key 不存在时会抛 `KeyError`
- `state.get("key")` 在 key 不存在时返回 `None`，更安全
- `state.get("key") or ""` 同时处理 `None` 和空字符串
- 养成习惯：对用户输入字段始终做空值防御

## 4.4 使用 Command 同时更新状态和路由

**一句话：** `Command(update={...}, goto="node")` 让你在一次返回中同时完成状态更新和路由跳转

```python
from typing import Annotated, Literal
from typing_extensions import TypedDict
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command

class State(TypedDict):
    query: str
    route: str
    result: str
    logs: Annotated[list[str], add]

def router_node(state: State) -> Command[Literal["search_node", "direct_node"]]:
    query = state["query"]
    if "检索" in query or "搜索" in query:
        return Command(
            update={"route": "search", "logs": ["路由到检索分支"]},
            goto="search_node"
        )
    return Command(
        update={"route": "direct", "logs": ["路由到直接回答分支"]},
        goto="direct_node"
    )

def search_node(state: State) -> dict:
    return {
        "result": f"检索结果：关于 '{state['query']}' 的文档",
        "logs": ["执行了检索"]
    }

def direct_node(state: State) -> dict:
    return {
        "result": f"直接回答：{state['query']}",
        "logs": ["直接生成回答"]
    }

builder = StateGraph(State)
builder.add_node("router", router_node)
builder.add_node("search_node", search_node)
builder.add_node("direct_node", direct_node)
builder.add_edge(START, "router")
builder.add_edge("search_node", END)
builder.add_edge("direct_node", END)

graph = builder.compile()

# 测试检索路径
result = graph.invoke({"query": "检索 RAG 资料", "route": "", "result": "", "logs": []})
print(result["route"])   # search
print(result["result"])  # 检索结果：关于 '检索 RAG 资料' 的文档
print(result["logs"])    # ['路由到检索分支', '执行了检索']

# 测试直接回答路径
result = graph.invoke({"query": "你好", "route": "", "result": "", "logs": []})
print(result["route"])   # direct
print(result["result"])  # 直接回答：你好
print(result["logs"])    # ['路由到直接回答分支', '直接生成回答']
```

**关键点：**
- `Command` 替代了传统的"条件边 + 路由函数"模式，代码更集中
- `update` 参数遵循和普通返回 dict 相同的规则（Reducer 字段返回增量）
- `goto` 指定下一个要执行的节点名称
- 类型注解 `Command[Literal["a", "b"]]` 帮助 IDE 检查路由目标是否合法
- 不需要 `add_conditional_edges`，Command 自带路由能力

## 4.5 不返回 = 不更新

**一句话：** 返回空字典 `{}` 表示不更新任何字段，状态保持原样

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    input: str
    processed: bool
    result: str

def conditional_node(state: State) -> dict:
    # 如果已经处理过，跳过，不更新任何字段
    if state.get("processed"):
        return {}  # 空 dict = 什么都不改

    # 否则执行处理
    return {
        "result": f"处理完成: {state['input']}",
        "processed": True
    }

builder = StateGraph(State)
builder.add_node("process", conditional_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()

# 第一次：会处理
result = graph.invoke({"input": "hello", "processed": False, "result": ""})
print(result["result"])     # 处理完成: hello
print(result["processed"])  # True

# 模拟已处理过的情况
result = graph.invoke({"input": "hello", "processed": True, "result": "旧结果"})
print(result["result"])     # 旧结果（未被更新）
print(result["processed"])  # True（未被更新）
```

**关键点：**
- 返回 `{}` 是完全合法的，框架不会报错
- 空字典意味着"这个节点执行了，但决定不修改任何状态"
- 这在条件跳过、幂等检查、守卫节点等场景中非常有用
- 注意：返回 `{}` 和不返回（返回 `None`）不同——建议始终显式返回 `{}`
