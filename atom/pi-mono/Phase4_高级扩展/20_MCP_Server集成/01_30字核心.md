# 30字核心

> **一句话定义 MCP Server 集成的本质**

---

## 核心定义

**MCP Server 集成是通过标准化协议让 AI Agent 动态发现和调用外部工具与资源的能力扩展机制。**

---

## 关键词解析

| 关键词 | 含义 |
|--------|------|
| **标准化协议** | 基于 JSON-RPC 2.0 的统一通信规范 |
| **动态发现** | Agent 运行时自动识别可用工具，无需硬编码 |
| **外部工具与资源** | 数据库、API、文件系统等外部能力 |
| **能力扩展机制** | 不修改核心代码即可增强 Agent 功能 |

---

## 三个核心维度

```
MCP Server 集成
    ├─ 构建 MCP Server：创建提供工具的服务进程
    ├─ 创建 Pi-mono Extension：开发集成 MCP 的扩展
    └─ 使用现有 Server：配置和调用社区服务器
```

---

## 与传统 API 的本质区别

| 维度 | 传统 API | MCP Server |
|------|---------|-----------|
| **发现方式** | 静态文档 | 动态 Schema |
| **调用方式** | 手动编码 | Agent 自主选择 |
| **描述格式** | 非标准化 | 统一 JSON Schema |
| **设计目标** | 人类开发者 | AI Agent |

---

## 记忆口诀

**"标准协议，动态发现，工具扩展"**

- **标准协议**：JSON-RPC 2.0 统一通信
- **动态发现**：运行时自动识别工具
- **工具扩展**：无侵入式能力增强

---

## 应用场景速览

1. **数据库操作**：通过 MCP Server 让 Agent 查询和修改数据库
2. **文件系统访问**：读写本地文件，无需自定义工具
3. **API 集成**：包装第三方 API 为标准化工具
4. **多工具协作**：组合多个 MCP Server 实现复杂任务
