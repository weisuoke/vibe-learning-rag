# 核心概念 5：语义缓存

## 一句话定义

**语义缓存是基于 Embedding 相似度匹配的缓存机制,不仅匹配精确字符串,还能匹配语义相似的查询,显著提升缓存命中率。**

---

## 详细解释

### 什么是语义缓存?

语义缓存(Semantic Cache)是一种智能缓存机制,它不仅仅匹配完全相同的查询字符串,还能识别语义相似的查询并返回缓存结果。

### 精确匹配 vs 语义匹配

#### 精确匹配缓存(传统方式)

```python
# 查询 1
"What is artificial intelligence?"  # 缓存未命中,调用 API

# 查询 2
"What is AI?"  # 缓存未命中,调用 API(虽然语义相同)

# 查询 3
"What is artificial intelligence?"  # 缓存命中(完全相同)
```

**问题**:语义相同但表述不同的查询无法命中缓存。

#### 语义匹配缓存(语义缓存)

```python
# 查询 1
"What is artificial intelligence?"  # 缓存未命中,调用 API,缓存结果

# 查询 2
"What is AI?"  # 缓存命中!(语义相似度 > 阈值)

# 查询 3
"Explain artificial intelligence"  # 缓存命中!(语义相似)

# 查询 4
"What is machine learning?"  # 缓存未命中(语义不同)
```

**优势**:语义相似的查询可以共享缓存结果,大幅提升命中率。

---

## 语义缓存的工作原理

### 核心流程

```
用户查询
  ↓
计算查询的 Embedding
  ↓
在 Redis 中搜索相似向量
  ↓
找到相似度 > 阈值的缓存?
  ↙        ↘
 是          否
  ↓           ↓
返回缓存结果  调用 LLM API
              ↓
         缓存新结果(存储 Embedding + Response)
              ↓
         返回结果
```

### 关键组件

1. **Embedding 模型**:将查询转换为向量
2. **向量存储**:Redis 作为向量数据库
3. **相似度计算**:余弦相似度或欧氏距离
4. **阈值设置**:决定多相似才算命中

## RedisSemanticCache 实现

### 基本使用

```python
from langchain_redis import RedisSemanticCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI, OpenAIEmbeddings

# 1. 创建 Embedding 模型
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 2. 设置语义缓存
set_llm_cache(RedisSemanticCache(
    embedding=embeddings,
    redis_url="redis://localhost:6379",
    score_threshold=0.9  # 相似度阈值(0-1)
))

# 3. 使用 LLM
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7)

# 首次查询
response1 = llm.invoke("What is artificial intelligence?")
print(f"Response 1: {response1.content[:100]}...")

# 语义相似的查询(命中缓存)
response2 = llm.invoke("What is AI?")
print(f"Response 2: {response2.content[:100]}...")

# 验证是否返回相同结果
print(f"Same response: {response1.content == response2.content}")
```

## 相似度阈值调优

### 阈值对命中率的影响

```python
import time
from langchain_redis import RedisSemanticCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI, OpenAIEmbeddings

def test_threshold(threshold, queries):
    """测试不同阈值的命中率"""
    # 设置缓存
    embeddings = OpenAIEmbeddings()
    set_llm_cache(RedisSemanticCache(
        embedding=embeddings,
        redis_url="redis://localhost:6379",
        score_threshold=threshold
    ))

    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)

    # 首次查询(建立缓存)
    llm.invoke(queries[0])

    # 测试后续查询
    hits = 0
    for query in queries[1:]:
        start = time.time()
        llm.invoke(query)
        elapsed = time.time() - start

        # 如果响应很快,可能是缓存命中
        if elapsed < 0.1:
            hits += 1

    hit_rate = hits / (len(queries) - 1)
    print(f"阈值 {threshold}: 命中率 {hit_rate:.2%}")

# 测试查询
queries = [
    "What is artificial intelligence?",
    "What is AI?",
    "Explain artificial intelligence",
    "Define AI",
    "What does AI mean?",
    "Tell me about artificial intelligence",
]

# 测试不同阈值
for threshold in [0.95, 0.90, 0.85, 0.80]:
    test_threshold(threshold, queries)
```

**典型结果**:

```
阈值 0.95: 命中率 20%  # 太严格,只有几乎相同的查询才命中
阈值 0.90: 命中率 60%  # 平衡,推荐
阈值 0.85: 命中率 80%  # 较宽松,可能返回不太相关的结果
阈值 0.80: 命中率 100% # 太宽松,可能返回错误结果
```

### 阈值选择建议

| 阈值范围 | 适用场景 | 优点 | 缺点 |
|---------|---------|------|------|
| 0.95-1.0 | 精确问答 | 结果准确 | 命中率低 |
| 0.90-0.95 | 通用场景 | 平衡准确性和命中率 | 推荐 |
| 0.85-0.90 | 高命中率需求 | 命中率高 | 可能不够精确 |
| < 0.85 | 不推荐 | 命中率很高 | 结果可能不相关 |

## 语义缓存 vs 精确缓存

### 性能对比

```python
import time
from langchain_core.caches import InMemoryCache
from langchain_redis import RedisSemanticCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI, OpenAIEmbeddings

def benchmark_cache(cache_type, queries):
    """对比不同缓存类型的性能"""
    if cache_type == "exact":
        set_llm_cache(InMemoryCache())
    else:
        embeddings = OpenAIEmbeddings()
        set_llm_cache(RedisSemanticCache(
            embedding=embeddings,
            redis_url="redis://localhost:6379",
            score_threshold=0.9
        ))

    llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.0)

    # 首次查询
    llm.invoke(queries[0])

    # 测试后续查询
    hits = 0
    total_time = 0

    for query in queries[1:]:
        start = time.time()
        llm.invoke(query)
        elapsed = time.time() - start
        total_time += elapsed

        if elapsed < 0.1:
            hits += 1

    hit_rate = hits / (len(queries) - 1)
    avg_time = total_time / (len(queries) - 1)

    print(f"{cache_type.upper()} 缓存:")
    print(f"  命中率: {hit_rate:.2%}")
    print(f"  平均响应时间: {avg_time:.3f}s")
    print()

# 测试查询(包含语义相似的变体)
queries = [
    "What is artificial intelligence?",
    "What is AI?",
    "Explain AI",
    "Define artificial intelligence",
    "What does AI mean?",
]

benchmark_cache("exact", queries)
benchmark_cache("semantic", queries)
```

**典型结果**:

```
EXACT 缓存:
  命中率: 0%
  平均响应时间: 2.340s

SEMANTIC 缓存:
  命中率: 75%
  平均响应时间: 0.580s
```

### 成本对比

**场景**:每天 10,000 次查询,40% 是语义相似的变体

#### 精确缓存

- API 调用:10,000 次(无法识别语义相似)
- 成本($0.01/次):$100/天
- 月成本:$3,000

#### 语义缓存

- API 调用:6,000 次(4,000 次语义命中)
- API 成本:$60/天
- Embedding 成本($0.0001/次):$1/天
- Redis 成本:$50/月
- 月成本:$1,880
- **节省:$1,120/月(37%)**

## GPTCache 集成

### 什么是 GPTCache?

GPTCache 是一个专门为 LLM 设计的语义缓存库,提供了更丰富的功能:

- 多种相似度算法
- 自定义缓存策略
- 缓存预热
- 缓存统计

### 与 LangChain 集成

```python
from gptcache import Cache
from gptcache.adapter.langchain_models import LangChainChat
from gptcache.embedding import OpenAI as GPTCacheOpenAI
from gptcache.manager import CacheBase, VectorBase, get_data_manager
from gptcache.similarity_evaluation.distance import SearchDistanceEvaluation

# 1. 初始化 GPTCache
cache_base = CacheBase('sqlite')
vector_base = VectorBase('faiss', dimension=1536)
data_manager = get_data_manager(cache_base, vector_base)

cache = Cache()
cache.init(
    embedding_func=GPTCacheOpenAI().to_embeddings,
    data_manager=data_manager,
    similarity_evaluation=SearchDistanceEvaluation(),
)

# 2. 创建 LangChain Chat 模型
llm = LangChainChat(cache_obj=cache)

# 3. 使用(自动缓存)
response = llm("What is AI?")
```

### GPTCache 优势

1. **更灵活的相似度算法**:
   - 余弦相似度
   - 欧氏距离
   - 曼哈顿距离
   - 自定义算法

   ```python
       llm(question)
   ```

3. **缓存统计**:

```python
from langchain_redis import RedisSemanticCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_core.prompts import ChatPromptTemplate

    embedding=embeddings,
    redis_url="redis://localhost:6379",
    score_threshold=0.88,  # 客服场景可以稍微宽松
    ttl=86400  # 24小时过期
))

    ("system", "You are a helpful customer service assistant."),
    ("user", "{question}")
])

# 处理用户问题

    response = chain.invoke({"question": question})

print(handle_question("Password reset procedure?"))  # 缓存命中
```

- 命中率从 15%(精确缓存)提升到 65%(语义缓存)

## 语义缓存的局限性

### 1. 额外的 Embedding 计算

```python
# 每次查询都需要计算 Embedding
query = "What is AI?"
embedding = embeddings.embed_query(query)  # 额外的 API 调用

# 成本:
# - Embedding API 调用:$0.0001/次
# - 延迟:~50ms
```

**解决方案**:
- 使用本地 Embedding 模型(如 sentence-transformers)
- 缓存 Embedding 结果

### 2. 可能返回不精确的结果

```python
# 查询 1
"What is the capital of France?"  # 答案:Paris

# 查询 2(语义相似但答案不同)
"What is the capital of Germany?"  # 可能错误返回:Paris
```

**解决方案**:
- 提高相似度阈值(0.92+)
- 在 prompt 中包含关键信息
- 使用混合策略(精确 + 语义)

### 3. 配置复杂度

```python
# 需要调优多个参数
cache = RedisSemanticCache(
    embedding=embeddings,  # 选择合适的 Embedding 模型
    score_threshold=0.9,  # 调优阈值
    ttl=3600,  # 设置过期时间
    namespace="app1"  # 管理命名空间
)
```

**解决方案**:
- 使用默认配置开始
- 根据实际数据调优
- 监控命中率和准确性

## 监控与优化

### 缓存效果监控

```python
import time
from collections import defaultdict
from langchain_redis import RedisSemanticCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI, OpenAIEmbeddings

class MonitoredSemanticCache(RedisSemanticCache):
    """带监控的语义缓存"""

    def __init__(self, **kwargs):
        super().__init__(**kwargs)
        self.stats = defaultdict(int)
        self.response_times = []

    def lookup(self, prompt: str, llm_string: str):
        start = time.time()
        result = super().lookup(prompt, llm_string)
        elapsed = time.time() - start

        if result is not None:
            self.stats['hits'] += 1
            self.stats['semantic_hits'] += 1
        else:
            self.stats['misses'] += 1

        self.response_times.append(elapsed)
        return result

    def get_stats(self):
        """获取统计信息"""
        total = self.stats['hits'] + self.stats['misses']
        hit_rate = self.stats['hits'] / total if total > 0 else 0
        avg_time = sum(self.response_times) / len(self.response_times) if self.response_times else 0

        return {
            'total_queries': total,
            'hits': self.stats['hits'],
            'misses': self.stats['misses'],
            'hit_rate': hit_rate,
            'avg_lookup_time': avg_time,
            'semantic_hits': self.stats['semantic_hits']
        }

# 使用监控缓存
embeddings = OpenAIEmbeddings()
cache = MonitoredSemanticCache(
    embedding=embeddings,
    redis_url="redis://localhost:6379",
    score_threshold=0.9
)
set_llm_cache(cache)

llm = ChatOpenAI(model="gpt-4o-mini")

# 运行一段时间后查看统计
# ... 处理查询 ...

stats = cache.get_stats()
print(f"总查询数: {stats['total_queries']}")
print(f"命中次数: {stats['hits']}")
print(f"命中率: {stats['hit_rate']:.2%}")
print(f"语义命中: {stats['semantic_hits']}")
print(f"平均查询时间: {stats['avg_lookup_time']:.3f}s")
```

### 优化策略

#### 1. 动态阈值调整

```python
def adjust_threshold(hit_rate, accuracy):
    """根据命中率和准确性动态调整阈值"""
    if hit_rate < 0.5 and accuracy > 0.95:
        # 命中率低但准确性高,降低阈值
        return -0.02
    elif hit_rate > 0.8 and accuracy < 0.90:
        # 命中率高但准确性低,提高阈值
        return +0.02
    else:
        return 0

# 定期调整
current_threshold = 0.90
adjustment = adjust_threshold(hit_rate=0.45, accuracy=0.96)
new_threshold = current_threshold + adjustment
```

#### 3. 分层缓存策略

```python
from langchain_core.caches import InMemoryCache
from langchain_redis import RedisSemanticCache
from langchain_core.globals import set_llm_cache
from langchain_openai import ChatOpenAI, OpenAIEmbeddings

class HybridCache:
    """混合缓存:精确缓存 + 语义缓存"""

    def __init__(self):
        self.exact_cache = InMemoryCache(maxsize=100)
        embeddings = OpenAIEmbeddings()
        self.semantic_cache = RedisSemanticCache(
            embedding=embeddings,
            redis_url="redis://localhost:6379",
            score_threshold=0.9
        )

    def lookup(self, prompt: str, llm_string: str):
        # 1. 先查精确缓存(最快)
        result = self.exact_cache.lookup(prompt, llm_string)
        if result is not None:
            return result

        # 2. 再查语义缓存
        result = self.semantic_cache.lookup(prompt, llm_string)
        if result is not None:
            # 将语义命中的结果也存入精确缓存
            self.exact_cache.update(prompt, llm_string, result)
            return result

        return None

    def update(self, prompt: str, llm_string: str, return_val):
        # 同时更新两个缓存
        self.exact_cache.update(prompt, llm_string, return_val)
        self.semantic_cache.update(prompt, llm_string, return_val)

    def clear(self, **kwargs):
        self.exact_cache.clear(**kwargs)
        self.semantic_cache.clear(**kwargs)

# 使用混合缓存
set_llm_cache(HybridCache())
```
