## 练习 1：Move 语义体验

**学习目标：** 理解移动语义，掌握 3 种修复 move 错误的方法。

### 1.1 观察 Move 行为

```rust
// 文件：move_basics.rs
// 运行：rustc move_basics.rs && ./move_basics

fn main() {
    // === 赋值导致 Move ===
    let s1 = String::from("hello, ownership!");
    let s2 = s1; // s1 的所有权移动给 s2
    // println!("{}", s1); // error[E0382]: borrow of moved value: `s1`
    println!("s2 = {}", s2);

    // === 基本类型是 Copy，不会 Move ===
    let n1: i32 = 42;
    let n2 = n1; // i32 实现了 Copy trait，这里是复制
    println!("n1 = {}, n2 = {}", n1, n2); // 两个都能用

    // === 函数传参导致 Move ===
    let greeting = String::from("Hi, Rust!");
    print_message(greeting); // greeting 所有权移入函数
    // println!("{}", greeting); // error[E0382]: borrow of moved value

    // === 整数传参是 Copy ===
    let age: i32 = 25;
    print_number(age);
    println!("age 还能用: {}", age);
}

fn print_message(msg: String) {
    println!("[print_message] {}", msg);
}

fn print_number(num: i32) {
    println!("[print_number] {}", num);
}
```

**预期输出：**
```
s2 = hello, ownership!
n1 = 42, n2 = 42
[print_message] Hi, Rust!
[print_number] 25
age 还能用: 25
```

### 1.2 三种修复 Move 错误的方法

```rust
// 文件：fix_move.rs
// 运行：rustc fix_move.rs && ./fix_move

fn main() {
    // === 方法 1：& 借用（最常用，零开销）===
    let s1 = String::from("借用不转移");
    print_borrowed(&s1);
    println!("s1 还能用: {}", s1);

    // === 方法 2：.clone() 深拷贝 ===
    let s2 = String::from("克隆一份");
    let s3 = s2.clone();
    println!("s2 = {}, s3 = {}", s2, s3); // 两个都有

    // === 方法 3：函数返回所有权 ===
    let s4 = String::from("来回传递");
    let s4 = take_and_give_back(s4);
    println!("s4 = {}", s4); // 所有权回来了

    println!("\n选择建议: 只读->借用 | 需独立副本->clone | 转移后还需用->返回所有权");
}

fn print_borrowed(s: &String) {
    println!("[借用] {}", s);
}

fn take_and_give_back(s: String) -> String {
    println!("[转手] 处理中: {}", s);
    s
}
```

**预期输出：**
```
[借用] 借用不转移
s1 还能用: 借用不转移
s2 = 克隆一份, s3 = 克隆一份
[转手] 处理中: 来回传递
s4 = 来回传递

选择建议: 只读->借用 | 需独立副本->clone | 转移后还需用->返回所有权
```

## 练习 2：借用规则实验

**学习目标：** 理解不可变/可变引用的排斥规则，掌握 NLL 的实际效果。

```rust
// 文件：borrow_rules.rs
// 运行：rustc borrow_rules.rs && ./borrow_rules

fn main() {
    // === 多个不可变引用（OK）===
    println!("=== 多个不可变引用 ===");
    let s = String::from("shared data");
    let r1 = &s;
    let r2 = &s;
    println!("r1={}, r2={}", r1, r2);

    // === 不可变 + 可变 同时存在（编译错误）===
    // let mut s2 = String::from("conflict");
    // let r3 = &s2;
    // let r4 = &mut s2;      // error! 不可变引用还活着
    // println!("{}, {}", r3, r4);

    // === 利用 NLL 让代码通过编译 ===
    println!("\n=== NLL 拯救 ===");
    let mut s3 = String::from("NLL magic");
    let r5 = &s3;
    let r6 = &s3;
    println!("不可变借用: {} and {}", r5, r6);
    // r5/r6 最后一次使用在此，NLL 判定生命周期结束

    let r7 = &mut s3;  // OK! 因为 r5、r6 已经"死了"
    r7.push_str(" - modified");
    println!("可变借用修改后: {}", r7);

    // === 修改函数签名练习 ===
    println!("\n=== 函数签名练习 ===");
    let mut msg = String::from("Hello");
    show(&msg);                    // &String: 只读借用
    println!("show后还能用: {}", msg);
    shout(&mut msg);               // &mut String: 可变借用
    println!("shout后: {}", msg);
    let len = consume(msg);        // String: 获取所有权
    println!("字符串长度: {}", len);
    // println!("{}", msg);        // error! 已被 consume 消费
}

// 修复前: fn show(s: String) -> 会夺走所有权
fn show(s: &String) { println!("[show] {}", s); }

// 修复前: fn shout(s: &String) -> 不可变引用不能修改
fn shout(s: &mut String) {
    s.push_str("!!!");
    println!("[shout] {}", s);
}

fn consume(s: String) -> usize {
    println!("[consume] eating: {}", s);
    s.len()
}
```

**预期输出：**
```
=== 多个不可变引用 ===
r1=shared data, r2=shared data

=== NLL 拯救 ===
不可变借用: NLL magic and NLL magic
可变借用修改后: NLL magic - modified

=== 函数签名练习 ===
[show] Hello
show后还能用: Hello
[shout] Hello!!!
shout后: Hello!!!
[consume] eating: Hello!!!
字符串长度: 8
```

## 练习 3：所有权与数据结构

**学习目标：** 在 struct 上实践 `&self`、`&mut self`、`self` 三种方法签名的区别。

```rust
// 文件：user_profile.rs
// 运行：rustc user_profile.rs && ./user_profile

struct UserProfile {
    name: String,
    email: String,
    login_count: u32,
}

impl UserProfile {
    fn new(name: String, email: String) -> Self {
        UserProfile { name, email, login_count: 0 }
    }

    /// &self: 只读，调用后 struct 仍可用
    fn display(&self) {
        println!("  用户: {} | 邮箱: {} | 登录: {}", self.name, self.email, self.login_count);
    }

    /// &mut self: 可修改，调用后 struct 仍可用
    fn update_name(&mut self, new_name: &str) {
        println!("  [update] {} -> {}", self.name, new_name);
        self.name = String::from(new_name);
    }

    fn increment_login(&mut self) { self.login_count += 1; }

    /// self: 消费结构体，调用后不可再用
    fn into_name(self) -> String {
        println!("  [into_name] 结构体被消费，提取 name");
        self.name
    }
}

fn main() {
    let mut user = UserProfile::new(String::from("Alice"), String::from("alice@example.com"));
    user.display();

    user.update_name("Bob");
    user.increment_login();
    user.increment_login();
    user.display();

    let name = user.into_name(); // user 被消费
    println!("  提取的 name: {}", name);
    // user.display(); // error[E0382]: borrow of moved value: `user`

    println!("\n  &self=只读 | &mut self=可修改 | self=消费");
}
```

**预期输出：**
```
  用户: Alice | 邮箱: alice@example.com | 登录: 0
  [update] Alice -> Bob
  用户: Bob | 邮箱: alice@example.com | 登录: 2
  [into_name] 结构体被消费，提取 name
  提取的 name: Bob

  &self=只读 | &mut self=可修改 | self=消费
```

## 练习 4：综合挑战 - StringCollection

**学习目标：** 综合运用所有权的"获取、借出、归还"。

```rust
// 文件：string_collection.rs
// 运行：rustc string_collection.rs && ./string_collection

struct StringCollection {
    items: Vec<String>,
}

impl StringCollection {
    fn new() -> Self { StringCollection { items: Vec::new() } }

    /// 获取 String 所有权存入集合
    fn add(&mut self, s: String) {
        println!("  [add] \"{}\"", s);
        self.items.push(s);
    }

    /// 只读借用返回 &str，集合保留所有权
    fn get(&self, index: usize) -> Option<&str> {
        self.items.get(index).map(|s| s.as_str())
    }

    /// 移除并归还所有权给调用方
    fn remove(&mut self, index: usize) -> Option<String> {
        if index < self.items.len() {
            Some(self.items.remove(index))
        } else {
            None
        }
    }

    /// 只读检查，不需要所有权
    fn contains(&self, s: &str) -> bool {
        self.items.iter().any(|item| item == s)
    }

    fn len(&self) -> usize { self.items.len() }

    fn display(&self) {
        let joined: Vec<&str> = self.items.iter().map(|s| s.as_str()).collect();
        println!("  集合 [{}项]: {:?}", self.items.len(), joined);
    }
}

fn main() {
    let mut coll = StringCollection::new();

    // add: 所有权转入
    let fruit1 = String::from("apple");
    coll.add(fruit1);
    // println!("{}", fruit1); // error! 已被移走
    coll.add(String::from("banana"));
    coll.add(String::from("cherry"));
    coll.display();

    // get: 只读借用
    println!("\n  索引0: {:?}", coll.get(0));
    println!("  索引99: {:?}", coll.get(99));

    // contains: 只读检查
    println!("\n  包含banana: {}", coll.contains("banana"));
    println!("  包含grape: {}", coll.contains("grape"));

    // remove: 所有权归还
    let removed = coll.remove(1);
    println!("\n  移除: {:?}", removed);
    coll.display();

    if let Some(val) = removed {
        println!("  移除的值还能用: \"{}\", 长度={}", val, val.len());
    }

    // 最终状态
    println!("\n  长度: {}, 包含banana: {}", coll.len(), coll.contains("banana"));

    println!("\n=== 所有权流向 ===");
    println!("  add(String)         -> 调用方交出所有权");
    println!("  get() -> &str       -> 集合借出只读视图");
    println!("  remove() -> String  -> 集合归还所有权");
    println!("  contains(&str)      -> 调用方借出只读视图");
}
```

**预期输出：**
```
  [add] "apple"
  [add] "banana"
  [add] "cherry"
  集合 [3项]: ["apple", "banana", "cherry"]

  索引0: Some("apple")
  索引99: None

  包含banana: true
  包含grape: false

  移除: Some("banana")
  集合 [2项]: ["apple", "cherry"]
  移除的值还能用: "banana", 长度=6

  长度: 2, 包含banana: false

=== 所有权流向 ===
  add(String)         -> 调用方交出所有权
  get() -> &str       -> 集合借出只读视图
  remove() -> String  -> 集合归还所有权
  contains(&str)      -> 调用方借出只读视图
```

## 常见错误速查表

| 错误代码 | 含义 | 快速修复 |
|----------|------|----------|
| **E0382** | 使用已移走的值 | `&` 借用 / `.clone()` / 返回所有权 |
| **E0502** | 不可变与可变借用冲突 | 利用 NLL 调整使用顺序 |
| **E0499** | 多个可变借用冲突 | 确保前一个 `&mut` 用完再创建新的 |
| **E0596** | 未声明 `mut` 却试图可变借用 | 加 `let mut` |
| **E0507** | 从引用后面移出值 | 用 `.clone()` 或改方法签名为 `self` |

### 修复决策树

```
编译错误？
├── "moved value" (E0382)
│   ├── 只需读取？ -> 改用 &借用
│   ├── 需要独立副本？ -> 用 .clone()
│   └── 需要转移后拿回？ -> 函数返回所有权
├── "cannot borrow as mutable" (E0502/E0499)
│   ├── 不可变+可变冲突？ -> 调整顺序，利用 NLL
│   └── 两个可变冲突？ -> 确保前一个借用已结束
├── "not declared as mutable" (E0596) -> 加 let mut
└── "cannot move out of" (E0507) -> 返回 .clone() 或改为 self
```

## 关键收获

1. **Move 是默认行为**：堆数据赋值/传参时所有权转移，栈数据自动 Copy。
2. **三种修复法**：`&` 借用（零开销首选）> `.clone()`（有开销但简单）> 返回所有权。
3. **借用 = 读写锁**：多 `&T` 共存，`&mut T` 独占，NLL 让编译器更智能。
4. **方法签名表意图**：`&self` 只看 / `&mut self` 要改 / `self` 要吃掉。
5. **读懂错误信息**：E0382、E0502、E0596 是最高频的三个，学会读它们解决 80% 问题。
