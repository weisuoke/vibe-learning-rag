# 核心概念 3：文档过滤器

## 一句话定义

**文档过滤器是 LangChain 中基于 Embedding 相似度的 DocumentTransformer，以 EmbeddingsRedundantFilter（冗余过滤）和 EmbeddingsClusteringFilter（聚类过滤）为代表，用于去除重复文档和选择代表性文档，是 RAG 多路检索结果合并后的关键去噪环节。**

[来源: reference/context7_langchain_01.md + reference/source_document_transformer_01.md]

---

## 为什么需要文档过滤？

### 问题：多路检索产生大量重复

在 RAG 系统中，为了提高召回率，常常使用多个检索器并合并结果：

```python
from langchain.retrievers import MergerRetriever

# 多路检索：关键词 + 语义 + MMR
lotr = MergerRetriever(retrievers=[
    bm25_retriever,      # 关键词检索
    vector_retriever,    # 语义检索
    mmr_retriever,       # MMR 多样性检索
])

results = lotr.invoke("什么是 RAG？")
# 问题：三个检索器可能返回大量重复或高度相似的文档！
```

**重复文档的危害：**
- 浪费 LLM 的 Context Window（Token 很贵）
- 重复内容不会增加信息量，反而可能误导模型
- 降低生成回答的多样性和质量

### 解决方案：用 Embedding 过滤器去重

```python
# 过滤前：10 个文档（含大量重复）
# 过滤后：5 个唯一文档（信息密度更高）
```

**LangChain 提供了两种过滤策略：**
1. **EmbeddingsRedundantFilter** — 基于相似度阈值去重
2. **EmbeddingsClusteringFilter** — 基于聚类选择代表性文档

[来源: reference/context7_langchain_01.md | Context7 官方文档]

---

## EmbeddingsRedundantFilter 详解

### 是什么？

`EmbeddingsRedundantFilter` 通过计算文档之间的 **Embedding 余弦相似度**，过滤掉与已有文档过于相似的文档。简单说：如果两个文档的向量相似度超过阈值（默认 0.95），就认为是冗余的，只保留一个。

**核心原理：**
```
文档 A: "RAG 是检索增强生成技术"     → 向量 [0.1, 0.8, 0.3, ...]
文档 B: "RAG 即检索增强生成的技术"   → 向量 [0.1, 0.79, 0.31, ...]
文档 C: "LangChain 是 LLM 开发框架" → 向量 [0.7, 0.2, 0.5, ...]

cosine_similarity(A, B) = 0.98 > 0.95 → B 被过滤（与 A 冗余）
cosine_similarity(A, C) = 0.35 < 0.95 → C 被保留（与 A 不同）

结果：保留 A 和 C，过滤掉 B
```

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import EmbeddingsRedundantFilter
from langchain_openai import OpenAIEmbeddings

# 创建 Embedding 模型
embeddings = OpenAIEmbeddings()

# 创建冗余过滤器
redundant_filter = EmbeddingsRedundantFilter(embeddings=embeddings)

# 准备测试文档（包含重复内容）

    Document(page_content="LangChain 是一个用于开发 LLM 应用的框架。"),
    Document(page_content="向量数据库用于存储和检索高维向量数据。"),
    Document(page_content="LangChain 框架帮助开发者构建 LLM 驱动的应用程序。"),  # 与第 3 个高度相似
]

# 执行过滤
filtered_docs = redundant_filter.transform_documents(docs)

print(f"过滤前: {len(docs)} 个文档")
print(f"过滤后: {len(filtered_docs)} 个文档")
print()
for i, doc in enumerate(filtered_docs):
    print(f"文档 {i+1}: {doc.page_content[:50]}...")
```

### 关键参数

```python
redundant_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,           # 必需：Embedding 模型
    similarity_threshold=0.95,       # 可选：相似度阈值（默认 0.95）
    similarity_fn=cosine_similarity, # 可选：相似度函数（默认余弦相似度）
)
```

**参数详解：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `embeddings` | Embeddings | 必需 | 用于计算文档向量的 Embedding 模型 |
| `similarity_threshold` | float | 0.95 | 相似度阈值，超过此值视为冗余 |
| `similarity_fn` | Callable | cosine_similarity | 相似度计算函数 |

**阈值选择建议：**

| 阈值 | 效果 | 适用场景 |
|------|------|---------|
| 0.99 | 只过滤几乎完全相同的文档 | 精确去重 |
| 0.95（默认） | 过滤高度相似的文档 | 通用场景 |
| 0.90 | 过滤较相似的文档 | 激进去重 |
| 0.85 | 过滤中等相似的文档 | 强调多样性 |

```python
# 宽松过滤：只去掉几乎一模一样的
loose_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.99
)

# 严格过滤：去掉所有相似内容
strict_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.85
)
```

### 工作流程

```
输入文档列表: [D1, D2, D3, D4, D5]
                ↓
1. 计算所有文档的 Embedding 向量
                ↓
2. 逐个检查：
   - D1: 第一个文档，直接保留 → 结果: [D1]
   - D2: 与 D1 相似度 0.98 > 0.95 → 过滤掉
   - D3: 与 D1 相似度 0.35 < 0.95 → 保留 → 结果: [D1, D3]
   - D4: 与 D1 相似度 0.40, 与 D3 相似度 0.30 → 保留 → 结果: [D1, D3, D4]
   - D5: 与 D3 相似度 0.97 > 0.95 → 过滤掉
                ↓
输出: [D1, D3, D4]
```

**注意：** 过滤是按顺序进行的，先出现的文档优先保留。这意味着**文档的输入顺序会影响过滤结果**。

## EmbeddingsClusteringFilter 详解

### 是什么？

`EmbeddingsClusteringFilter` 使用 **KMeans 聚类算法**将文档按语义分组，然后从每个聚类中选择最具代表性的文档。与 RedundantFilter 的"去重"不同，ClusteringFilter 的目标是**选择多样化的代表性文档**。

**核心原理：**
```
输入 10 个文档 → KMeans 聚类为 3 个簇

簇 1（RAG 相关）: [D1, D2, D5, D8]  → 选择最接近簇中心的: D2
簇 2（LLM 相关）: [D3, D6, D9]      → 选择最接近簇中心的: D6
簇 3（向量库相关）: [D4, D7, D10]    → 选择最接近簇中心的: D7

输出: [D2, D6, D7]  — 3 个代表性文档，覆盖 3 个主题
```

### 基础用法

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import EmbeddingsClusteringFilter
from langchain_openai import OpenAIEmbeddings

# 创建 Embedding 模型
embeddings = OpenAIEmbeddings()

# 创建聚类过滤器
clustering_filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=3,     # 聚类数量
    num_closest=1,      # 每个簇选择几个文档
)

# 准备测试文档（多个主题）
docs = [
    # RAG 相关
    Document(page_content="RAG 是检索增强生成技术。"),
    Document(page_content="RAG 系统结合了信息检索和文本生成。"),
    Document(page_content="检索增强生成提高了 LLM 回答的准确性。"),
    # LangChain 相关
    Document(page_content="LangChain 是 LLM 应用开发框架。"),
    Document(page_content="LangChain 提供了丰富的组件生态。"),
    Document(page_content="使用 LangChain 可以快速构建 AI 应用。"),
    # 向量数据库相关
    Document(page_content="Milvus 是高性能向量数据库。"),
    Document(page_content="向量数据库用于存储和检索 Embedding 向量。"),
    Document(page_content="Chroma 是轻量级的向量存储解决方案。"),
]

# 执行聚类过滤
clustered_docs = clustering_filter.transform_documents(docs)

print(f"过滤前: {len(docs)} 个文档")
print(f"过滤后: {len(clustered_docs)} 个文档")
print()
for i, doc in enumerate(clustered_docs):
    print(f"代表文档 {i+1}: {doc.page_content}")
```

**运行输出示例：**
```
过滤前: 9 个文档
过滤后: 3 个文档

代表文档 1: RAG 系统结合了信息检索和文本生成。
代表文档 2: LangChain 提供了丰富的组件生态。
代表文档 3: 向量数据库用于存储和检索 Embedding 向量。
```

**每个主题只保留了一个最具代表性的文档。**

### 关键参数

```python
clustering_filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,    # 必需：Embedding 模型
    num_clusters=10,          # 聚类数量
    num_closest=1,            # 每个簇选择的文档数
    sorted=False,             # 是否按原始检索分数排序
    random_state=42,          # 随机种子（保证可复现）
)
```

**参数详解：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `embeddings` | Embeddings | 必需 | 用于计算文档向量的 Embedding 模型 |
| `num_clusters` | int | 5 | KMeans 聚类的簇数量 |
| `num_closest` | int | 1 | 从每个簇中选择的文档数量 |
| `sorted` | bool | False | 是否按原始检索器的分数排序结果 |
| `random_state` | int | None | 随机种子，保证结果可复现 |

**`sorted` 参数的作用：**

```python
# sorted=False（默认）：按簇的顺序返回
filter_by_cluster = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=10,
    num_closest=1,
)
# 结果顺序：簇 0 的代表 → 簇 1 的代表 → ... → 簇 9 的代表

# sorted=True：按原始检索分数排序
filter_by_score = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=10,
    num_closest=1,
    sorted=True,  # 按原始检索器返回的顺序排序
)
# 结果顺序：按检索相关性从高到低
```

### num_clusters 如何选择？

| 文档数量 | 推荐 num_clusters | 原因 |
|----------|-------------------|------|
| < 10 | 3-5 | 文档少，簇不宜太多 |
| 10-50 | 5-10 | 中等规模，适中聚类 |
| 50-100 | 10-20 | 大规模，需要更多簇 |
| > 100 | 20-50 | 超大规模，按需调整 |

**经验法则：** `num_clusters` 大约为文档数量的 1/3 到 1/5。

## RedundantFilter vs ClusteringFilter：如何选择？

### 对比表

| 特性 | EmbeddingsRedundantFilter | EmbeddingsClusteringFilter |
|------|--------------------------|---------------------------|
| **算法** | 逐对比较余弦相似度 | KMeans 聚类 |
| **目标** | 去除重复文档 | 选择代表性文档 |
| **输出数量** | 不确定（取决于重复程度） | 确定（num_clusters × num_closest） |
| **顺序敏感** | 是（先出现的优先保留） | 否（基于聚类中心距离） |
| **计算复杂度** | O(n²)（逐对比较） | O(n × k × iterations) |
| **适用场景** | 多路检索合并去重 | 大量文档中选择多样化子集 |
| **可控性** | 通过阈值控制 | 通过簇数量控制 |

### 选择指南

**选 EmbeddingsRedundantFilter 的场景：**
- 多个检索器的结果合并后需要去重
- 文档之间有明显的重复（同一内容的不同表述）
- 需要保留尽可能多的唯一信息

```python
# 场景：合并 BM25 + 语义检索的结果，去除重复
filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.95
)
unique_docs = filter.transform_documents(merged_results)
```

**选 EmbeddingsClusteringFilter 的场景：**
- 检索结果太多，需要精选一个子集
- 需要确保结果覆盖多个主题/角度
- 想要固定数量的输出文档

```python
# 场景：从 50 个检索结果中选出 5 个覆盖不同主题的文档
filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=5,
    num_closest=1,
    sorted=True
)
diverse_docs = filter.transform_documents(search_results)
```

## 实战：与 ContextualCompressionRetriever 集成

### 为什么需要 ContextualCompressionRetriever？

文档过滤器本身只是 `DocumentTransformer`，需要手动调用 `transform_documents()`。但在实际 RAG 管道中，我们希望过滤器能**自动集成到检索流程**中。

`ContextualCompressionRetriever` 就是这个桥梁——它包装一个基础检索器，在检索结果返回前自动应用压缩/过滤。

### 基础集成

```python
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain.retrievers import ContextualCompressionRetriever
from langchain_community.document_transformers import (
    EmbeddingsRedundantFilter,
)
from langchain_openai import OpenAIEmbeddings

# ===== 1. 准备基础组件 =====
embeddings = OpenAIEmbeddings()

# 假设已有一个基础检索器（如多路合并检索器）
# base_retriever = MergerRetriever(retrievers=[...])

# ===== 2. 创建过滤器 =====
redundant_filter = EmbeddingsRedundantFilter(embeddings=embeddings)

# ===== 3. 创建压缩管道 =====
pipeline = DocumentCompressorPipeline(
    transformers=[redundant_filter]
)

# ===== 4. 创建压缩检索器 =====
compression_retriever = ContextualCompressionRetriever(
    base_compressor=pipeline,
    base_retriever=base_retriever
)

# ===== 5. 使用 =====
# 检索时自动去重
results = compression_retriever.invoke("什么是 RAG？")
# 返回的结果已经过滤了冗余文档
```

[来源: reference/context7_langchain_01.md | Context7 官方文档]

### 组合多个过滤器

```python
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain.retrievers import ContextualCompressionRetriever
from langchain_community.document_transformers import (
    EmbeddingsRedundantFilter,
    EmbeddingsClusteringFilter,
    LongContextReorder,
)
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings()

# 第 1 步：去除冗余文档
redundant_filter = EmbeddingsRedundantFilter(
    embeddings=embeddings,
    similarity_threshold=0.95
)

# 第 2 步：聚类选择代表性文档
clustering_filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=5,
    num_closest=1,
    sorted=True
)

# 第 3 步：重排序（解决 "Lost in the Middle" 问题）
reordering = LongContextReorder()

# 组合成管道
pipeline = DocumentCompressorPipeline(
    transformers=[
        redundant_filter,    # 先去重
        clustering_filter,   # 再聚类选择
        reordering,          # 最后重排序
    ]
)

# 集成到检索器
compression_retriever = ContextualCompressionRetriever(
    base_compressor=pipeline,
    base_retriever=base_retriever
)

# 使用：一次调用，自动完成 去重 → 聚类 → 重排序
results = compression_retriever.invoke("RAG 系统如何优化检索质量？")
```

**管道执行流程：**
```
base_retriever.invoke("query")
        ↓ 返回 20 个文档
EmbeddingsRedundantFilter
        ↓ 去重后剩 12 个
EmbeddingsClusteringFilter (5 clusters)
        ↓ 聚类后剩 5 个
LongContextReorder
        ↓ 重排序后 5 个（重要的在首尾）
最终返回 5 个高质量、多样化、排序优化的文档
```

## 双重类比

### 类比 1：EmbeddingsRedundantFilter = 数组去重 [...new Set()]

**前端类比：**

```javascript
// JavaScript 数组去重
const items = [1, 2, 2, 3, 3, 3, 4];
const unique = [...new Set(items)];
// 结果: [1, 2, 3, 4]
```

**LangChain 对应：**

```python
# 文档去重（基于语义相似度，而非精确匹配）
filter = EmbeddingsRedundantFilter(embeddings=embeddings)
unique_docs = filter.transform_documents(docs)
```

**区别：** JavaScript 的 `Set` 是精确匹配去重，而 `EmbeddingsRedundantFilter` 是**语义相似度去重**——即使文字不完全一样，只要意思相近就会被过滤。

**日常生活类比：** 就像整理书架——发现有两本讲同一个主题的书（虽然作者不同），只留一本。

---

### 类比 2：EmbeddingsClusteringFilter = 分组取代表

**前端类比：**

```javascript
// 按类别分组，每组取第一个
const items = [
    { category: 'fruit', name: 'apple' },
    { category: 'fruit', name: 'banana' },
    { category: 'veggie', name: 'carrot' },
    { category: 'veggie', name: 'broccoli' },
    { category: 'meat', name: 'chicken' },
];

const representatives = Object.values(
    items.reduce((groups, item) => {
        groups[item.category] = groups[item.category] || item;
        return groups;
    }, {})
);
// 结果: [apple, carrot, chicken] — 每类一个代表
```

**LangChain 对应：**

```python
# 按语义聚类，每簇取一个代表
filter = EmbeddingsClusteringFilter(
    embeddings=embeddings,
    num_clusters=3,
    num_closest=1
)
representatives = filter.transform_documents(docs)
```

**日常生活类比：** 就像班级选课代表——全班 40 人分成 5 个小组，每组选一个代表发言。不是去掉重复的人，而是选出能代表各组观点的人。

---

### 类比 3：DocumentCompressorPipeline = Express 中间件链

**前端类比：**

```javascript
// Express 中间件链：请求依次经过多个中间件
app.use(authenticate);  // 认证
app.use(rateLimit);     // 限流
app.use(compress);      // 压缩
```

**LangChain 对应：**

```python
# DocumentCompressorPipeline：文档依次经过多个过滤器
pipeline = DocumentCompressorPipeline(
    transformers=[
        redundant_filter,   # 去重
        clustering_filter,  # 聚类
        reordering,         # 重排序
    ]
)
```

**日常生活类比：** 就像工厂质检流水线——产品先过 X 光检测（去重），再分类抽检（聚类），最后按优先级排列（重排序）。

## 在 RAG 开发中的应用

### 应用场景 1：多路检索结果合并

```python
"""
场景：BM25 + 语义检索合并后去重
"""
from langchain.retrievers import MergerRetriever
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain.retrievers import ContextualCompressionRetriever
from langchain_community.document_transformers import EmbeddingsRedundantFilter
from langchain_openai import OpenAIEmbeddings

# 合并多个检索器
lotr = MergerRetriever(retrievers=[bm25_retriever, vector_retriever])

# 去重
filter = EmbeddingsRedundantFilter(embeddings=OpenAIEmbeddings())
pipeline = DocumentCompressorPipeline(transformers=[filter])

# 创建压缩检索器
retriever = ContextualCompressionRetriever(
    base_compressor=pipeline,
    base_retriever=lotr
)

# 使用
results = retriever.invoke("什么是 RAG？")
# 自动合并 + 去重
```

### 应用场景 2：大规模文档精选

```python
"""
场景：从 100 个检索结果中选出 10 个覆盖不同主题的文档
```

## 总结

### 两个核心过滤器

| 过滤器 | 算法 | 目标 | 输出数量 |
|--------|------|------|---------|
| EmbeddingsRedundantFilter | 余弦相似度比较 | 去除重复文档 | 不确定 |
| EmbeddingsClusteringFilter | KMeans 聚类 | 选择代表性文档 | 确定（clusters × closest） |

### 选择建议

- **去重** → EmbeddingsRedundantFilter（多路检索合并后）
- **精选** → EmbeddingsClusteringFilter（大量结果中选子集）
- **两者组合** → 先去重再聚类（通过 DocumentCompressorPipeline）

### 关键集成模式

```python
# 标准集成模式
filter = EmbeddingsRedundantFilter(embeddings=embeddings)
pipeline = DocumentCompressorPipeline(transformers=[filter])
retriever = ContextualCompressionRetriever(
    base_compressor=pipeline,
    base_retriever=base_retriever
)
```

### 注意事项

1. 两个过滤器都需要 Embedding 模型，会产生额外的 API 调用成本
2. RedundantFilter 对文档顺序敏感，先出现的优先保留
3. ClusteringFilter 的 `num_clusters` 不能大于文档数量
4. 建议设置 `random_state` 保证聚类结果可复现
