## 卡片1：直觉理解 - CallbackHandler 是什么

**一句话：** CallbackHandler 是 LangChain 的"监控摄像头"，记录执行过程的每一个事件。

**举例：**

想象你在使用 LangChain 调用 LLM：

```python
llm = ChatOpenAI()
result = llm.invoke("什么是 RAG？")
```

**问题**：
- LLM 调用了几次？
- 花费了多少 token？
- 执行过程中有没有错误？

**没有 CallbackHandler**：你完全不知道，就像黑盒。

**有了 CallbackHandler**：
```python
handler = StdOutCallbackHandler()
result = llm.invoke("什么是 RAG？", config={"callbacks": [handler]})

# 输出：
# LLM 开始调用...
# 发送 prompt: 什么是 RAG？
# LLM 完成，token 用量: 150
```

**应用：** 在 RAG 开发中，CallbackHandler 让你看到检索了哪些文档、生成了什么答案、花了多少钱。

## 卡片2：形式化定义 - 事件驱动的函数执行

**一句话：** CallbackHandler = 在特定事件发生时执行的函数。

**核心公式：**
```
事件发生 → 触发回调方法 → 执行自定义逻辑
```

**事件类型：**
- `on_llm_start` - LLM 开始时触发
- `on_llm_end` - LLM 结束时触发
- `on_llm_error` - LLM 错误时触发
- `on_llm_new_token` - 新 token 生成时触发（流式）
- `on_retriever_start` - Retriever 开始时触发
- `on_retriever_end` - Retriever 结束时触发

**代码示例：**
```python
class MyHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        print(f"LLM 开始: {prompts[0][:50]}...")

    def on_llm_end(self, response, **kwargs):
        print(f"LLM 完成: {response.generations[0][0].text[:50]}...")
```

**应用：** 通过实现不同的回调方法，可以在不同的事件点插入自定义逻辑。

## 卡片3：关键概念1 - Mixin 架构

**一句话：** LangChain 使用 6 个 Mixin 组织回调方法，每个 Mixin 负责一类组件。

**6 个 Mixin：**

1. **LLMManagerMixin** - LLM 回调
   - `on_llm_start`, `on_llm_end`, `on_llm_error`, `on_llm_new_token`

2. **ChainManagerMixin** - Chain 和 Agent 回调
   - `on_chain_start`, `on_chain_end`, `on_agent_action`, `on_agent_finish`

3. **ToolManagerMixin** - Tool 回调
   - `on_tool_start`, `on_tool_end`, `on_tool_error`

4. **RetrieverManagerMixin** - Retriever 回调
   - `on_retriever_start`, `on_retriever_end`, `on_retriever_error`

5. **CallbackManagerMixin** - 启动回调
   - 所有组件的 `on_*_start` 方法

6. **RunManagerMixin** - 运行时回调
   - `on_text`, `on_retry`, `on_custom_event`

**BaseCallbackHandler 设计：**
```python
class BaseCallbackHandler(
    LLMManagerMixin,
    ChainManagerMixin,
    ToolManagerMixin,
    RetrieverManagerMixin,
    CallbackManagerMixin,
    RunManagerMixin,
):
    pass
```

**应用：** 通过继承 BaseCallbackHandler，你可以选择性地实现需要的回调方法。

## 卡片4：关键概念2 - 回调生命周期

**一句话：** 每个组件都有完整的生命周期：开始 → 执行 → 结束/错误。

**生命周期模式：**
```
on_*_start → [执行过程] → on_*_end / on_*_error
```

**LLM 生命周期示例：**
```python
class LifecycleHandler(BaseCallbackHandler):
    def on_llm_start(self, serialized, prompts, **kwargs):
        print("1. LLM 开始")
        self.start_time = time.time()

    def on_llm_new_token(self, token, **kwargs):
        print(f"2. 生成 token: {token}")

    def on_llm_end(self, response, **kwargs):
        elapsed = time.time() - self.start_time
        print(f"3. LLM 完成，耗时: {elapsed:.2f}s")

    def on_llm_error(self, error, **kwargs):
        print(f"3. LLM 错误: {error}")
```

**RAG 生命周期：**
```
on_retriever_start → on_retriever_end → on_llm_start → on_llm_new_token → on_llm_end
```

**应用：** 理解生命周期后，你可以在正确的时机插入监控逻辑。

## 卡片5：编程实现 - 自定义回调处理器

**一句话：** 继承 BaseCallbackHandler，实现需要的回调方法。

**实现步骤：**

1. **继承 BaseCallbackHandler**
2. **实现回调方法**
3. **在组件中传入回调**

**完整示例：**
```python
from langchain.callbacks.base import BaseCallbackHandler
from langchain_openai import ChatOpenAI

class TokenCounterHandler(BaseCallbackHandler):
    """统计 token 用量"""

    def __init__(self):
        super().__init__()
        self.total_tokens = 0
        self.total_cost = 0.0

    def on_llm_end(self, response, **kwargs):
        # 获取 token 用量
        usage = response.llm_output.get("token_usage", {})
        tokens = usage.get("total_tokens", 0)
        self.total_tokens += tokens

        # 计算成本（假设 $0.002 / 1K tokens）
        cost = (tokens / 1000) * 0.002
        self.total_cost += cost

        print(f"Token 用量: {tokens}, 成本: ${cost:.4f}")

    def get_summary(self):
        return {
            "total_tokens": self.total_tokens,
            "total_cost": self.total_cost
        }

# 使用
handler = TokenCounterHandler()
llm = ChatOpenAI(callbacks=[handler])
result = llm.invoke("什么是 RAG？")
print(handler.get_summary())
```

**应用：** 自定义回调处理器可以实现任意监控逻辑。

## 卡片6：对比区分 - CallbackHandler vs 日志系统

**一句话：** CallbackHandler 是专门为 LangChain 设计的事件监听系统，比通用日志更强大。

**对比表：**

| 维度 | CallbackHandler | 日志（logging） |
|------|----------------|----------------|
| **定位** | 事件监听系统 | 通用日志记录 |
| **数据类型** | 结构化数据（LLMResult、Document） | 文本信息 |
| **触发方式** | 事件驱动（自动触发） | 主动调用 |
| **访问能力** | 可访问完整的对象（response、documents） | 只能记录字符串 |
| **集成能力** | 支持第三方平台（Langfuse、LangSmith） | 通常输出到文件或控制台 |
| **用途** | 可观测性、调试、监控 | 记录信息 |

**示例对比：**

```python
# 日志方式（有限）
import logging
logging.info("LLM 调用完成")  # 只能记录文本

# CallbackHandler 方式（强大）
def on_llm_end(self, response, **kwargs):
    # 可以访问完整的 response 对象
    tokens = response.llm_output["token_usage"]["total_tokens"]
    text = response.generations[0][0].text
    print(f"Token: {tokens}, 答案: {text[:50]}...")
```

**应用：** 在 LangChain 开发中，优先使用 CallbackHandler 而非日志。

## 卡片7：进阶理解 - 异步回调和流式输出

**一句话：** 异步回调处理器支持 FastAPI 集成和流式输出。

**异步回调处理器：**
```python
from langchain.callbacks.base import AsyncCallbackHandler
import asyncio

class AsyncStreamHandler(AsyncCallbackHandler):
    """异步流式回调"""

    def __init__(self):
        self.queue = asyncio.Queue()

    async def on_llm_new_token(self, token: str, **kwargs):
        # 将 token 放入队列
        await self.queue.put(token)

    async def on_llm_end(self, response, **kwargs):
        # 标记结束
        await self.queue.put(None)

    async def aiter(self):
        """异步迭代器"""
        while True:
            token = await self.queue.get()
            if token is None:
                break
            yield token
```

**FastAPI 集成：**
```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse

app = FastAPI()

@app.post("/chat")
async def chat(request: dict):
    handler = AsyncStreamHandler()
    llm = ChatOpenAI(callbacks=[handler], streaming=True)

    # 启动 LLM 调用
    asyncio.create_task(llm.ainvoke(request["message"]))

    # 返回流式响应
    return StreamingResponse(
        handler.aiter(),
        media_type="text/event-stream"
    )
```

**应用：** 实现类似 ChatGPT 的流式输出体验。

## 卡片9：在 RAG 开发中的使用

**一句话：** CallbackHandler 让 RAG 系统的每个步骤都可观测。

**RAG 监控回调：**
```python
class RAGMonitoringHandler(BaseCallbackHandler):
    """完整的 RAG 监控"""

    def __init__(self):
        self.retrieval_time = 0
        self.generation_time = 0
        self.total_tokens = 0
        self.retrieved_docs = []

    # 1. 监控检索
    def on_retriever_start(self, serialized, query, **kwargs):
        self.retrieval_start = time.time()
        print(f"🔍 开始检索: {query}")

    def on_retriever_end(self, documents, **kwargs):
        self.retrieval_time = time.time() - self.retrieval_start
        self.retrieved_docs = documents
        print(f"✅ 检索完成 ({self.retrieval_time:.2f}s)")
        print(f"   检索到 {len(documents)} 个文档")

        for i, doc in enumerate(documents[:2]):
            print(f"   文档 {i+1}: {doc.page_content[:80]}...")

    # 2. 监控生成
    def on_llm_start(self, serialized, prompts, **kwargs):
        self.generation_start = time.time()
        print(f"\n🤖 开始生成答案")

    def on_llm_end(self, response, **kwargs):
        self.generation_time = time.time() - self.generation_start
        usage = response.llm_output.get("token_usage", {})
        self.total_tokens = usage.get("total_tokens", 0)
        cost = (self.total_tokens / 1000) * 0.002

        print(f"✅ 生成完成 ({self.generation_time:.2f}s)")
        print(f"   Token 用量: {self.total_tokens}")
        print(f"   成本: ${cost:.4f}")

    def get_report(self):
        return {
            "retrieval_time": self.retrieval_time,
            "generation_time": self.generation_time,
            "total_time": self.retrieval_time + self.generation_time,
            "total_tokens": self.total_tokens,
            "num_docs": len(self.retrieved_docs)
        }
```

**应用：** 追踪 RAG 系统的完整执行过程。

## 卡片10：总结与延伸 - 生产环境最佳实践

**一句话：** 生产环境使用 CallbackHandler 的关键是避免并发冲突。

**最佳实践：**

### 1. 并发环境：每个请求新建实例

```python
# ❌ 错误：重用同一个实例
handler = LangfuseCallbackHandler()

@app.post("/chat")
async def chat(request):
    chain.invoke(input, config={"callbacks": [handler]})  # 并发冲突！

# ✅ 正确：每个请求新建实例
@app.post("/chat")
async def chat(request):
    handler = LangfuseCallbackHandler()  # 每次新建
    chain.invoke(input, config={"callbacks": [handler]})
    handler.flush()  # 确保数据持久化
```

### 2. LangGraph：使用 @observe 装饰器

```python
from langfuse.decorators import observe

@observe()
def my_langgraph_node(state):
    # 自动追踪 LangGraph 节点
    return {"output": process(state)}
```

### 3. Agent：追踪工具调用

```python
class AgentToolHandler(BaseCallbackHandler):
    def on_tool_start(self, serialized, input_str, **kwargs):
        print(f"工具调用: {serialized['name']}")
        print(f"参数: {input_str}")

    def on_tool_end(self, output, **kwargs):
        print(f"工具返回: {output}")
```

### 4. 性能优化：选择性监控

```python
class ProductionHandler(BaseCallbackHandler):
    @property
    def ignore_chain(self) -> bool:
        return True  # 生产环境忽略 Chain 回调

    def on_llm_error(self, error, **kwargs):
        # 只监控错误
        logger.error(f"LLM 错误: {error}")
```

**核心要点：**
1. 并发环境每个请求新建实例
2. 使用 `flush()` 确保数据持久化
3. LangGraph 使用 `@observe` 装饰器
4. 生产环境选择性监控，避免性能影响
