# 核心概念 3：数据类型支持

> **目标**：掌握 Milvus 标量过滤支持的数据类型和索引策略

---

## 概述

Milvus 支持丰富的数据类型用于标量过滤，包括基础标量类型、JSON 类型和 ARRAY 类型，每种类型都有对应的索引优化策略。

**核心价值**：
- 支持多种数据类型
- 灵活的索引选择
- 类型安全的查询
- 高效的过滤性能

## 1. 整数类型

### 1.1 支持的整数类型

**类型列表**：
- **INT8**：8位有符号整数（-128 到 127）
- **INT16**：16位有符号整数（-32,768 到 32,767）
- **INT32**：32位有符号整数（-2^31 到 2^31-1）
- **INT64**：64位有符号整数（-2^63 到 2^63-1）

### 1.2 支持的操作符

**比较操作符**：
```python
# 等于与不等于
"age == 25"
"age != 30"

# 大于与小于
"age > 18"
"age < 60"
"age >= 25"
"age <= 35"

# 范围查询
"age >= 25 AND age <= 35"
```

**集合操作符**：
```python
# IN 操作符
"age IN [25, 30, 35, 40]"
"user_id IN [1001, 1002, 1003]"
```

### 1.3 索引类型

**支持的索引**：
- **INVERTED**：倒排索引，适用于精确匹配和范围查询
- **BITMAP**：位图索引，适用于低基数字段
- **STL_SORT**：排序数组，适用于范围查询
- **AUTOINDEX**：自动选择最优索引

**示例**：
```python
# 创建 INVERTED 索引
index_params.add_index(
    field_name="age",
    index_type="INVERTED"
)

# 创建 BITMAP 索引（适用于低基数）
index_params.add_index(
    field_name="status",  # 假设只有几个状态值
    index_type="BITMAP"
)
```

## 2. 浮点类型

### 2.1 支持的浮点类型

**类型列表**：
- **FLOAT**：32位浮点数
- **DOUBLE**：64位浮点数

### 2.2 支持的操作符

**比较操作符**：
```python
# 精确比较（注意浮点精度）
"price == 99.99"
"rating != 0.0"

# 范围查询
"price > 100.0"
"price >= 100.5"
"price < 1000.0"
"price <= 999.99"

# 范围组合
"price >= 100.0 AND price <= 500.0"
"rating >= 4.0 AND rating <= 5.0"
```

**集合操作符**：
```python
# IN 操作符
"price IN [99.99, 199.99, 299.99]"
"rating IN [4.0, 4.5, 5.0]"
```

### 2.3 索引类型

**支持的索引**：
- **INVERTED**：倒排索引
- **STL_SORT**：排序数组
- **AUTOINDEX**：自动选择

**注意事项**：
- 浮点数比较存在精度问题
- 建议使用范围查询而非精确匹配
- BITMAP 索引不适用于浮点类型

## 3. 布尔类型

### 3.1 BOOL 类型

**取值**：
- `true`：真
- `false`：假
- `NULL`：空值（如果字段允许 NULL）

### 3.2 支持的操作符

**比较操作符**：
```python
# 等于与不等于
"active == true"
"deleted != true"
"is_vip == false"
```

**NULL 操作符**：
```python
# NULL 检查
"verified IS NULL"
"verified IS NOT NULL"
```

### 3.3 索引类型

**支持的索引**：
- **INVERTED**：倒排索引
- **BITMAP**：位图索引（推荐，因为只有2-3个值）
- **AUTOINDEX**：自动选择

**推荐**：
- 布尔类型最适合使用 BITMAP 索引
- 基数极低（只有 true/false/NULL）
- BITMAP 索引性能最优

## 4. 字符串类型

### 4.1 VARCHAR 类型

**特点**：
- 可变长度字符串
- 需要指定 `max_length`
- 支持 UTF-8 编码

### 4.2 支持的操作符

**比较操作符**：
```python
# 等于与不等于
"city == '北京'"
"name != 'admin'"

# 字典序比较
"name > 'A'"
"name < 'Z'"
```

**集合操作符**：
```python
# IN 操作符
"city IN ['北京', '上海', '深圳']"
"status IN ['active', 'pending', 'approved']"
```

**模式匹配**：
```python
# LIKE 操作符
"name LIKE 'John%'"        # 前缀匹配
"email LIKE '%@gmail.com'" # 后缀匹配
"title LIKE '%Python%'"    # 包含匹配
```

### 4.3 索引类型

**支持的索引**：
- **INVERTED**：倒排索引，适用于精确匹配
- **BITMAP**：位图索引，适用于低基数字段
- **TRIE**：前缀树，适用于前缀匹配
- **NGRAM**：N-gram 索引，适用于模糊匹配
- **AUTOINDEX**：自动选择

**索引选择建议**：
```python
# 精确匹配 → INVERTED
"city == '北京'"  # 使用 INVERTED 索引

# 低基数 → BITMAP
"status IN ['active', 'pending']"  # 使用 BITMAP 索引

# 前缀匹配 → TRIE
"name LIKE 'John%'"  # 使用 TRIE 索引

# 模糊匹配 → NGRAM
"title LIKE '%Python%'"  # 使用 NGRAM 索引
```

## 5. JSON 类型

### 5.1 JSON 类型特点

**特点**：
- 支持嵌套对象
- 支持数组
- 动态结构
- 灵活的 schema

### 5.2 JSON Path 访问

**语法**：
```python
# 单层访问
"metadata['category'] == 'electronics'"

# 嵌套访问
"metadata['product']['brand'] == 'Apple'"

# 数组访问
"metadata['tags'][0] == 'hot'"

# 混合访问
"metadata['reviews'][0]['rating'] >= 4.0"
```

### 5.3 支持的操作符

**比较操作符**：
```python
# 等于与不等于
"metadata['price'] == 99.99"
"metadata['status'] != 'deleted'"

# 范围查询
"metadata['price'] >= 100 AND metadata['price'] <= 500"
```

**NULL 操作符**：
```python
# NULL 检查
"metadata['description'] IS NULL"
"metadata['tags'] IS NOT NULL"
```

**JSON 函数**：
```python
# JSON_CONTAINS
"JSON_CONTAINS(metadata, 'category')"

# 数组长度
"ARRAY_LENGTH(metadata['tags']) > 5"
```

### 5.4 索引类型

**支持的索引**：
- **INVERTED**：倒排索引（推荐）
- **NGRAM**：N-gram 索引
- **AUTOINDEX**：自动选择

**JSON Path Index**：
```python
# 创建 JSON Path Index
index_params.add_index(
    field_name="metadata",
    index_type="INVERTED",
    params={
        "json_path": "metadata['category']",
        "json_cast_type": "varchar"
    }
)
```

## 6. ARRAY 类型

### 6.1 支持的 ARRAY 类型

**元素类型**：
- **ARRAY<INT32>**：整数数组
- **ARRAY<INT64>**：长整数数组
- **ARRAY<VARCHAR>**：字符串数组

**ARRAY_LENGTH**：
```python
# 检查数组长度
```

### 6.3 索引类型

- **INVERTED**：倒排索引

- **AUTOINDEX**：自动选择

**示例**：
```python
# 创建 ARRAY 字段索引
index_params.add_index(

)
```

## 7. NULL 值处理

### 7.1 NULL 语义

**定义**：
- NULL 表示字段值未定义或缺失
- 空字符串 `""` 不等于 NULL
- 数值 0 不等于 NULL
- 布尔 false 不等于 NULL

### 7.2 NULL 操作符

**IS NULL**：
```python
# 查找 NULL 值
"description IS NULL"
"phone IS NULL"
"optional_field IS NULL"
```

**IS NOT NULL**：
```python
# 查找非 NULL 值
"description IS NOT NULL"
"email IS NOT NULL"
"verified_at IS NOT NULL"
```

### 7.3 NULL 与其他操作符

**比较操作符**：
```python
# NULL 与任何值比较都返回 false
"age == NULL"  # ❌ 错误写法
"age IS NULL"  # ✅ 正确写法

# NULL 不等于任何值（包括 NULL）
"age != NULL"  # ❌ 错误写法
"age IS NOT NULL"  # ✅ 正确写法
```

**组合使用**：
```python
# 有描述且价格合理
"description IS NOT NULL AND price > 0"

# 有邮箱或有电话
"email IS NOT NULL OR phone IS NOT NULL"
```

## 8. 类型转换

### 8.1 JSON Cast 类型

**支持的转换**：
- **BOOL**：转换为布尔类型
- **Double**：转换为双精度浮点数
- **Varchar**：转换为字符串
- **json**：保持 JSON 类型

**示例**：
```python
# 将 JSON 字段转换为 Double
index_params.add_index(
    field_name="metadata",
    index_type="INVERTED",
    params={
        "json_path": "metadata['price']",
        "json_cast_type": "Double"
    }
)

# 将 JSON 字段转换为 Varchar
index_params.add_index(
    field_name="metadata",
    index_type="INVERTED",
    params={
        "json_path": "metadata['category']",
        "json_cast_type": "Varchar"
    }
)
```

### 8.2 类型转换函数

**STRING_TO_DOUBLE**：
```python
# 将字符串转换为 Double
params = {
    "json_path": "metadata['price']",
    "json_cast_type": "double",
    "json_cast_function": "STRING_TO_DOUBLE"
}
```

## 9. 索引选择策略

### 9.1 按数据类型选择

| 数据类型 | 推荐索引 | 适用场景 |
|---------|---------|---------|
| **INT8/16/32/64** | INVERTED | 精确匹配、范围查询 |
| **INT8/16/32/64** | BITMAP | 低基数字段（如状态码） |
| **FLOAT/DOUBLE** | INVERTED | 范围查询 |
| **BOOL** | BITMAP | 布尔值（只有2-3个值） |
| **VARCHAR** | INVERTED | 精确匹配 |
| **VARCHAR** | TRIE | 前缀匹配 |
| **VARCHAR** | NGRAM | 模糊匹配 |
| **JSON** | INVERTED | JSON Path Index |
| **ARRAY** | INVERTED | 数组包含查询 |

### 9.2 按查询模式选择

**精确匹配**：
```python
# 使用 INVERTED 索引
"city == '北京'"
"status == 'active'"
```

**范围查询**：
```python
# 使用 INVERTED 或 STL_SORT 索引
"age >= 25 AND age <= 35"
"price >= 100 AND price <= 500"
```

**低基数字段**：
```python
# 使用 BITMAP 索引
"status IN ['active', 'pending', 'approved']"  # 只有3个值
"gender == 'male'"  # 只有2-3个值
```

**前缀匹配**：
```python
# 使用 TRIE 索引
"name LIKE 'John%'"
"email LIKE 'admin%'"
```

**模糊匹配**：
```python
# 使用 NGRAM 索引
"title LIKE '%Python%'"
"description LIKE '%urgent%'"
```

## 10. 实战示例

### 10.1 电商场景

```python
# Schema 定义
schema.add_field("product_id", DataType.INT64, is_primary=True)
schema.add_field("price", DataType.DOUBLE)
schema.add_field("stock", DataType.INT32)
schema.add_field("brand", DataType.VARCHAR, max_length=100)
schema.add_field("active", DataType.BOOL)
schema.add_field("metadata", DataType.JSON)
schema.add_field("tags", DataType.ARRAY, element_type=DataType.VARCHAR, max_capacity=10)

# 索引创建
index_params.add_index("price", "INVERTED")
index_params.add_index("stock", "INVERTED")
index_params.add_index("brand", "BITMAP")  # 低基数
index_params.add_index("active", "BITMAP")
index_params.add_index("metadata", "INVERTED", params={
    "json_path": "metadata['category']",
    "json_cast_type": "varchar"
})
index_params.add_index("tags", "INVERTED")

# 查询示例
filter = '''
    price >= 100 AND price <= 500
    AND stock > 0
    AND brand IN ['Apple', 'Samsung']
    AND active == true
    AND metadata['category'] == 'electronics'
    AND ARRAY_CONTAINS(tags, 'hot')
'''
```

### 10.2 用户画像场景

```python
# Schema 定义
schema.add_field("user_id", DataType.INT64, is_primary=True)
schema.add_field("age", DataType.INT32)
schema.add_field("city", DataType.VARCHAR, max_length=50)
schema.add_field("vip_level", DataType.INT8)
schema.add_field("is_active", DataType.BOOL)
schema.add_field("profile", DataType.JSON)
schema.add_field("interests", DataType.ARRAY, element_type=DataType.VARCHAR, max_capacity=20)

# 索引创建
index_params.add_index("age", "INVERTED")
index_params.add_index("city", "BITMAP")  # 假设城市数量有限
index_params.add_index("vip_level", "BITMAP")  # 只有几个等级
index_params.add_index("is_active", "BITMAP")
index_params.add_index("profile", "INVERTED", params={
    "json_path": "profile['total_purchase']",
    "json_cast_type": "double"
})
index_params.add_index("interests", "INVERTED")

# 查询示例
filter = '''
    age >= 25 AND age <= 35
    AND city IN ['北京', '上海', '深圳']
    AND vip_level >= 3
    AND is_active == true
    AND profile['total_purchase'] > 10000
    AND ARRAY_CONTAINS(interests, 'tech')
'''
```

## 11. 类比理解

### 前端开发类比

**TypeScript 类型系统**：
```typescript
// TypeScript
interface User {
    id: number;           // INT64
    name: string;         // VARCHAR
    age: number;          // INT32
    active: boolean;      // BOOL
    score: number;        // DOUBLE
    metadata: {           // JSON
        city: string;
        tags: string[];   // ARRAY<VARCHAR>
    };
}

// Milvus
schema.add_field("id", DataType.INT64)
schema.add_field("name", DataType.VARCHAR, max_length=100)
schema.add_field("age", DataType.INT32)
schema.add_field("active", DataType.BOOL)
schema.add_field("score", DataType.DOUBLE)
schema.add_field("metadata", DataType.JSON)
```

### 数据库类比

**SQL 数据类型**：
```sql
-- SQL
CREATE TABLE users (
    id BIGINT,           -- INT64
    name VARCHAR(100),   -- VARCHAR
    age INT,             -- INT32
    active BOOLEAN,      -- BOOL
    score DOUBLE,        -- DOUBLE
    metadata JSON,       -- JSON
    tags TEXT[]          -- ARRAY<VARCHAR>
);

-- Milvus
schema.add_field("id", DataType.INT64)
schema.add_field("name", DataType.VARCHAR, max_length=100)
schema.add_field("age", DataType.INT32)
schema.add_field("active", DataType.BOOL)
schema.add_field("score", DataType.DOUBLE)
schema.add_field("metadata", DataType.JSON)
schema.add_field("tags", DataType.ARRAY, element_type=DataType.VARCHAR)
```
