# 核心概念 6：流式解析机制

> 本文档详细讲解 LangChain OutputParser 的流式解析机制，包括逐块解析、累积解析、部分解析和流式解析工作原理。

---

## 概述

流式解析是 LangChain OutputParser 的核心特性之一，允许在 LLM 生成内容的同时实时解析输出，而不是等待完整响应后再处理。这对于提升用户体验和降低延迟至关重要。

**核心价值**：
- **降低延迟**：用户可以立即看到部分结果
- **提升体验**：流式输出更自然，类似人类对话
- **节省内存**：无需缓存完整输出
- **实时反馈**：可以提前发现问题并中断

**两种流式解析模式**：
1. **逐块解析（BaseTransformOutputParser）**：每个块独立解析，立即输出
2. **累积解析（BaseCumulativeTransformOutputParser）**：累积缓冲区，解析部分结果

---

## 1. BaseTransformOutputParser：逐块解析

### 1.1 核心原理

BaseTransformOutputParser 是流式解析的基类，提供逐块解析能力。每个输入块独立处理，立即输出结果。

**类定义**（来源：source_outputparser_04_转换解析器.md）：

```python
class BaseTransformOutputParser(BaseOutputParser[T]):
    """Base class for an output parser that can handle streaming input."""

    def _transform(
        self,
        input: Iterator[str | BaseMessage],
    ) -> Iterator[T]:
        for chunk in input:
            if isinstance(chunk, BaseMessage):
                yield self.parse_result([ChatGeneration(message=chunk)])
            else:
                yield self.parse_result([Generation(text=chunk)])
```

**工作流程**：
1. 接收输入流（Iterator[str | BaseMessage]）
2. 逐个处理每个块
3. 调用 `parse_result()` 解析
4. 立即 yield 结果

### 1.2 输入类型处理

BaseTransformOutputParser 支持两种输入类型：

**1. 字符串（str）**：
```python
# 来自 LLM 的原始文本输出
for chunk in ["Hello", " ", "world", "!"]:
    yield self.parse_result([Generation(text=chunk)])
```

**2. 消息（BaseMessage）**：
```python
# 来自 ChatModel 的结构化消息
for chunk in [AIMessage(content="Hello"), AIMessage(content=" world")]:
    yield self.parse_result([ChatGeneration(message=chunk)])
```

**为什么需要两种类型？**
- **字符串**：适用于简单的 LLM 文本输出
- **消息**：适用于 ChatModel，包含角色、内容、元数据等

### 1.3 公共接口：transform()

**同步版本**：

```python
@override
def transform(
    self,
    input: Iterator[str | BaseMessage],
    config: RunnableConfig | None = None,
    **kwargs: Any,
) -> Iterator[T]:
    """Transform the input into the output format."""
    yield from self._transform_stream_with_config(
        input, self._transform, config, run_type="parser"
    )
```

**关键特性**：
- 使用 `_transform_stream_with_config()` 包装
- 传递 `run_type="parser"` 用于 LangSmith 追踪
- 支持配置传递（`config`）

**异步版本**：

```python
@override
async def atransform(
    self,
    input: AsyncIterator[str | BaseMessage],
    config: RunnableConfig | None = None,
    **kwargs: Any,
) -> AsyncIterator[T]:
    """Async transform the input into the output format."""
    async for chunk in self._atransform_stream_with_config(
        input, self._atransform, config, run_type="parser"
    ):
        yield chunk
```

### 1.4 实战示例：StrOutputParser

**最简单的流式解析器**：

```python
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

# 创建模型和解析器
model = ChatOpenAI()
parser = StrOutputParser()

# 构建链
chain = model | parser

# 流式输出
for chunk in chain.stream("Tell me a story"):
    print(chunk, end="", flush=True)
    # 输出: "Once", " ", "upon", " ", "a", " ", "time", ...
```

**工作原理**：
1. ChatOpenAI 生成流式输出：`"Once"`, `" "`, `"upon"`, ...
2. StrOutputParser 逐块解析（实际上直接返回）
3. 每个块立即输出到控制台

**优势**：
- 零延迟：每个块立即输出
- 低内存：无需缓存
- 简单高效：适合文本流式输出

## 2. BaseCumulativeTransformOutputParser：累积解析

### 2.1 核心原理

BaseCumulativeTransformOutputParser 继承自 BaseTransformOutputParser，专门用于需要累积缓冲区的解析场景（如 JSON、XML）。

**类定义**（来源：source_outputparser_04_转换解析器.md）：

```python
class BaseCumulativeTransformOutputParser(BaseTransformOutputParser[T]):
    """Base class for an output parser that can handle streaming input."""
```

**关键特性**：
- 继承 BaseTransformOutputParser 的所有能力
- 需要子类实现 `_diff()` 方法（用于 diff 模式）
- 累积缓冲区，解析部分结果

### 2.2 累积解析工作流程

**场景：流式 JSON 解析**

**输入流**：
```
"{\"name\"" → ": \"Alice\"" → ", \"age\": " → "30}"
```

**BaseTransformOutputParser（逐块）**：
```
chunk1: "{\"name\""        → 解析失败 → 不输出
chunk2: ": \"Alice\""      → 解析失败 → 不输出
chunk3: ", \"age\": "      → 解析失败 → 不输出
chunk4: "30}"              → 解析失败 → 不输出
```

**BaseCumulativeTransformOutputParser（累积）**：
```
buffer1: "{\"name\""                    → 部分解析 → {}
buffer2: "{\"name\": \"Alice\""         → 部分解析 → {"name": "Alice"}
buffer3: "{\"name\": \"Alice\", \"age\": " → 部分解析 → {"name": "Alice"}
buffer4: "{\"name\": \"Alice\", \"age\": 30}" → 完整解析 → {"name": "Alice", "age": 30}
```

### 2.3 实战示例：JsonOutputParser

**JsonOutputParser 使用累积解析**：

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import PromptTemplate

# 创建模型和解析器
model = ChatOpenAI()
parser = JsonOutputParser()

# 创建 prompt
prompt = PromptTemplate.from_template(
    "Return a JSON object with name and age for: {person}"
)

# 构建链
chain = prompt | model | parser

# 流式输出（累积）
for chunk in chain.stream({"person": "Alice, 30 years old"}):
    print(chunk)
    # 输出:
    # {}
    # {"name": "Alice"}
    # {"name": "Alice", "age": 30}
```

**工作原理**：
1. ChatOpenAI 生成流式 JSON：`"{"`, `"\"name\""`, `": \"Alice\""`, ...
2. JsonOutputParser 累积缓冲区
3. 每次累积后尝试部分解析（`partial=True`）
4. 输出部分解析结果

### 2.4 diff 模式：JSON Patch

JsonOutputParser 支持 diff 模式，只输出变化部分，减少数据传输。

**实现**（来源：source_outputparser_04_转换解析器.md）：

```python
class JsonOutputParser(BaseCumulativeTransformOutputParser[Any]):
    @override
    def _diff(self, prev: Any | None, next: Any) -> Any:
        return jsonpatch.make_patch(prev, next).patch
```

**使用示例**：

```python
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

# 流式输出（diff 模式）
for chunk in chain.stream({"person": "Alice, 30 years old"}, config={"diff": True}):
    print(chunk)
    # 输出:
    # [{"op": "add", "path": "/name", "value": "Alice"}]
    # [{"op": "add", "path": "/age", "value": 30}]
```

**优势**：
- 减少数据传输量
- 适合大型 JSON 对象
- 客户端可以增量更新

## 3. 部分解析（partial=True）

### 3.1 核心原理

部分解析允许解析器在输入不完整时返回部分结果，而不是抛出异常。

**parse_result() 方法签名**：

```python
def parse_result(
    self,
    result: list[Generation] | list[ChatGeneration],
    *,
    partial: bool = False
) -> T:
    """Parse the result of an LLM call."""
```

**partial 参数**：
- `partial=False`（默认）：要求完整输入，否则抛出异常
- `partial=True`：允许部分输入，返回部分结果

### 3.2 JsonOutputParser 的部分解析

**实现**（来源：source_outputparser_02_JSON解析器.md）：

```python
def parse_result(
    self,
    result: list[Generation] | list[ChatGeneration],
    *,
    partial: bool = False
) -> Any:
    text = self._get_text(result)
    text = text.strip()

    if partial:
        # 部分解析：尝试解析不完整的 JSON
        return parse_partial_json(text)
    else:
        # 完整解析：要求完整的 JSON
        return parse_json_markdown(text)
```

**parse_partial_json() 工作原理**：

```python
def parse_partial_json(s: str) -> Any:
    """Parse a partial JSON string."""
    # 尝试解析完整 JSON
    try:
        return json.loads(s)
    except json.JSONDecodeError:
        # 失败时，尝试修复不完整的 JSON
        # 例如：{"name": "Alice", "age":  → {"name": "Alice"}
        return repair_json(s)
```

### 3.3 实战示例：部分解析

```python
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

# 完整 JSON
result = parser.parse('{"name": "Alice", "age": 30}')
print(result)  # {"name": "Alice", "age": 30}

# 部分 JSON（partial=False）
try:
    result = parser.parse('{"name": "Alice", "age": ')
except Exception as e:
    print(f"Error: {e}")  # JSONDecodeError

# 部分 JSON（partial=True）
result = parser.parse_result(
    [Generation(text='{"name": "Alice", "age": ')],
    partial=True
)
print(result)  # {"name": "Alice"}
```

## 4. 流式解析的两种模式对比

### 4.1 特性对比

| 特性 | BaseTransformOutputParser | BaseCumulativeTransformOutputParser |
|------|---------------------------|-------------------------------------|
| **解析方式** | 逐块独立解析 | 累积缓冲区解析 |
| **输出时机** | 每个块都输出 | 累积到可解析时输出 |
| **适用场景** | 简单转换（如字符串） | 复杂解析（如 JSON、XML） |
| **内存占用** | 低（无缓冲区） | 高（需要缓冲区） |
| **延迟** | 低（立即输出） | 中（需要累积） |
| **错误处理** | 每个块独立处理 | 累积后统一处理 |
| **diff 支持** | 不支持 | 支持（通过 `_diff()` 方法） |

### 4.2 使用场景选择

**使用 BaseTransformOutputParser（逐块解析）**：
- ✅ 简单文本流式输出（StrOutputParser）
- ✅ 每个块可以独立解析
- ✅ 需要最低延迟
- ✅ 内存受限场景

**使用 BaseCumulativeTransformOutputParser（累积解析）**：
- ✅ JSON、XML 等结构化输出
- ✅ 需要部分解析支持
- ✅ 需要 diff 模式
- ✅ 可以容忍一定延迟

## 5. 自定义流式解析器

### 5.1 自定义逐块解析器

```python
from langchain_core.output_parsers import BaseTransformOutputParser

class UpperCaseParser(BaseTransformOutputParser[str]):
    """Convert output to uppercase."""

    def parse(self, text: str) -> str:
        return text.upper()

# 使用
parser = UpperCaseParser()
for chunk in parser.transform(["hello", " ", "world"]):
    print(chunk, end="")  # "HELLO WORLD"
```

### 5.2 自定义累积解析器

```python
from langchain_core.output_parsers import BaseCumulativeTransformOutputParser

class WordCountParser(BaseCumulativeTransformOutputParser[int]):
    """Count words in accumulated text."""

    def parse_result(self, result, *, partial=False):
        text = result[0].text
        return len(text.split())

# 使用
parser = WordCountParser()
for chunk in parser.transform(["hello", " world", " from", " AI"]):
    print(chunk)  # 1, 2, 3, 4
```

### 5.3 实现 diff 模式

```python
from langchain_core.output_parsers import BaseCumulativeTransformOutputParser

class CustomJsonParser(BaseCumulativeTransformOutputParser[dict]):
    """Custom JSON parser with diff support."""

    def parse_result(self, result, *, partial=False):
        text = result[0].text
        if partial:
            return parse_partial_json(text)
        else:
            return json.loads(text)

    def _diff(self, prev: dict | None, next: dict) -> list:
        """Return JSON Patch diff."""
        import jsonpatch
        return jsonpatch.make_patch(prev or {}, next).patch
```

## 6. 与 Runnable 的集成

### 6.1 _transform_stream_with_config()

BaseTransformOutputParser 通过 `_transform_stream_with_config()` 方法与 Runnable 协议集成。

**功能**：
- **配置传递**：将 `config` 传递给解析器
- **可观测性**：通过 `run_type="parser"` 集成 LangSmith 追踪
- **错误处理**：统一的错误处理机制
- **上下文管理**：管理执行上下文

**使用示例**：

```python
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI

model = ChatOpenAI()
parser = JsonOutputParser()

chain = model | parser

# 传递配置
config = {
    "run_name": "json_parsing",
    "tags": ["streaming", "json"],
    "metadata": {"user_id": "123"}
}

for chunk in chain.stream("Return JSON", config=config):
    print(chunk)
```

### 6.2 异步流式解析

```python
import asyncio
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI

async def async_stream():
    model = ChatOpenAI()
    parser = JsonOutputParser()
    chain = model | parser

    async for chunk in chain.astream("Return JSON"):
        print(chunk)

asyncio.run(async_stream())
```

## 7. 性能优化

### 7.1 BaseTransformOutputParser 性能

**优势**：
- **低延迟**：每个块立即输出
- **低内存**：无需缓存完整输出
- **高吞吐**：适合大量小块数据

**适用场景**：
- 实时对话应用
- 流式文本生成
- 需要即时反馈的场景

### 7.2 BaseCumulativeTransformOutputParser 性能

**优势**：
- **部分解析**：可以提前输出部分结果
- **diff 模式**：减少数据传输量
- **错误恢复**：可以处理不完整输入

**劣势**：
- **内存占用**：需要缓存累积缓冲区
- **延迟增加**：需要等待可解析的数据量

**优化建议**：
1. **限制缓冲区大小**：避免内存溢出
2. **及时清理**：解析完成后清理缓冲区
3. **使用 diff 模式**：减少数据传输

## 8. 流式解析最佳实践

### 8.1 选择合适的解析器

| 场景 | 推荐解析器 | 原因 |
|------|-----------|------|
| 简单文本流式 | StrOutputParser | 无需解析，直接输出 |
| JSON 流式 | JsonOutputParser | 支持部分解析 |
| Pydantic 流式 | PydanticOutputParser | 支持部分解析 + 验证 |
| XML 流式 | XMLOutputParser | 使用 XMLPullParser 增量解析 |
| 列表流式 | ListOutputParser | 逐个元素输出 |

### 8.2 错误处理策略

**三层防护**：
1. **Prompt 优化**：在 prompt 中明确指定输出格式
2. **部分解析**：使用 `partial=True` 参数
3. **错误修复**：使用 OutputFixingParser 或 RetryOutputParser（见下一章）

### 8.3 性能优化

**减少延迟**：
- 使用 BaseTransformOutputParser 逐块输出
- 避免累积整个输出再解析
- 使用 BaseCumulativeTransformOutputParser 仅在必要时累积

**减少内存占用**：
- 及时清理缓冲区
- 使用流式输出而不是批量输出
- 限制缓冲区大小

## 9. 架构设计亮点

### 9.1 分层设计

```
BaseOutputParser (基类)
    ↓
BaseTransformOutputParser (流式基类)
    ↓
BaseCumulativeTransformOutputParser (累积流式)
    ↓
JsonOutputParser, PydanticOutputParser, ... (具体实现)
```

**优势**：
- 清晰的职责分离
- 易于扩展和维护
- 复用性高

### 9.2 灵活扩展

**子类只需实现**：
- `parse()` 或 `parse_result()`：解析逻辑
- `_diff()`（可选）：diff 模式支持

**自动获得**：
- 流式解析能力
- 异步支持
- Runnable 集成
- 可观测性

### 9.3 类型安全

```python
class BaseTransformOutputParser(BaseOutputParser[T]):
    """泛型 T 确保类型正确"""
```

**优势**：
- 编译时类型检查
- IDE 自动补全
- 减少运行时错误

## 总结

流式解析是 LangChain OutputParser 的核心特性，通过两种模式（逐块解析和累积解析）满足不同场景需求：

**核心要点**：
1. **BaseTransformOutputParser**：逐块解析，低延迟，适合简单转换
2. **BaseCumulativeTransformOutputParser**：累积解析，支持部分解析和 diff 模式
3. **partial=True**：允许解析不完整输入，提升流式体验
4. **Runnable 集成**：无缝集成到 LCEL 中，支持配置传递和可观测性
5. **性能优化**：根据场景选择合适的解析器，平衡延迟和内存
