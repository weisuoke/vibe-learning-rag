## 场景说明

**用户反馈循环**是 HITL 中最常见的模式之一：

```
生成内容 → 用户审阅 → 不满意？→ 修改 → 用户再审阅 → ... → 满意！→ 结束
```

核心挑战：
- 如何在图中实现"循环"（不是线性流程）
- 如何累积多轮反馈历史
- 如何让用户决定"继续修改"还是"满意退出"

本场景包含 3 个子场景，从简单到复杂逐步演进。

---

## 子场景 1：基础反馈循环

### 思路

最简单的反馈循环：生成 → 反馈 → 修改 → 反馈 → ... → 满意。

关键设计：
- `generate` 节点：模拟内容生成（实际项目中调用 LLM）
- `collect_feedback` 节点：用 `interrupt()` 暂停，等待用户反馈
- 条件边：根据反馈决定是回到 `generate` 还是去 `END`

```
START → generate → collect_feedback → 满意？ → END
                        ↑       │
                        └───────┘  不满意，回到 generate
```

### 完整代码

```python
"""
子场景 1：基础反馈循环
生成内容 → 用户反馈 → 不满意就修改 → 满意就结束
"""

from typing import TypedDict, Literal
from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph, START, END


# ========== 1. 定义状态 ==========

class FeedbackState(TypedDict):
    topic: str              # 用户要求的主题
    draft: str              # 当前草稿
    feedback: str           # 最新反馈
    revision_count: int     # 修改次数


# ========== 2. 定义节点 ==========

def generate(state: FeedbackState):
    """生成/修改内容（实际项目中这里调用 LLM）"""
    revision = state.get("revision_count", 0)

    if revision == 0:
        # 首次生成
        draft = f"【初稿】关于「{state['topic']}」的文章：\n这是一篇介绍 {state['topic']} 的文章..."
    else:
        # 根据反馈修改
        draft = (
            f"【第 {revision} 次修改】关于「{state['topic']}」的文章：\n"
            f"根据反馈「{state['feedback']}」进行了修改...\n"
            f"这是改进后的版本。"
        )

    return {"draft": draft, "revision_count": revision + 1}


def collect_feedback(state: FeedbackState):
    """收集用户反馈：用 interrupt 暂停等待"""
    # interrupt 传递当前草稿给用户审阅
    feedback = interrupt({
        "type": "review_request",
        "draft": state["draft"],
        "revision_count": state["revision_count"],
        "instruction": "请审阅以上内容。输入 'ok' 表示满意，或输入修改意见。",
    })
    return {"feedback": feedback}


# ========== 3. 条件路由 ==========

def should_continue(state: FeedbackState) -> Literal["generate", "__end__"]:
    """根据反馈决定是继续修改还是结束"""
    if state["feedback"].lower().strip() == "ok":
        return "__end__"
    return "generate"


# ========== 4. 构建图 ==========

builder = StateGraph(FeedbackState)
builder.add_node("generate", generate)
builder.add_node("collect_feedback", collect_feedback)

builder.add_edge(START, "generate")
builder.add_edge("generate", "collect_feedback")
builder.add_conditional_edges("collect_feedback", should_continue)

graph = builder.compile(checkpointer=MemorySaver())


# ========== 5. 运行演示 ==========

config = {"configurable": {"thread_id": "feedback-demo-1"}}

# 第一轮：生成初稿
print("=" * 50)
print("第一轮：生成初稿")
print("=" * 50)
result = graph.invoke(
    {"topic": "LangGraph 人机循环", "draft": "", "feedback": "", "revision_count": 0},
    config,
)

# 检查中断状态
snapshot = graph.get_state(config)
for task in snapshot.tasks:
    for intr in task.interrupts:
        print(f"草稿内容: {intr.value['draft']}")
        print(f"修改次数: {intr.value['revision_count']}")

# 第一次反馈：不满意
print("\n" + "=" * 50)
print("用户反馈：需要更详细")
print("=" * 50)
result = graph.invoke(Command(resume="请增加更多技术细节"), config)

# 第二次反馈：满意
snapshot = graph.get_state(config)
for task in snapshot.tasks:
    for intr in task.interrupts:
        print(f"修改后草稿: {intr.value['draft']}")

print("\n" + "=" * 50)
print("用户反馈：满意")
print("=" * 50)
result = graph.invoke(Command(resume="ok"), config)
print(f"最终草稿: {result['draft']}")
print(f"总修改次数: {result['revision_count']}")
```

### 运行输出

```
==================================================
第一轮：生成初稿
==================================================
草稿内容: 【初稿】关于「LangGraph 人机循环」的文章：
这是一篇介绍 LangGraph 人机循环 的文章...
修改次数: 1

==================================================
用户反馈：需要更详细
==================================================
修改后草稿: 【第 1 次修改】关于「LangGraph 人机循环」的文章：
根据反馈「请增加更多技术细节」进行了修改...

==================================================
用户反馈：满意
==================================================
最终草稿: 【第 1 次修改】关于「LangGraph 人机循环」的文章：...
总修改次数: 2
```

### 关键点解析

1. **循环结构**：`collect_feedback` 的条件边可以回到 `generate`，形成循环
2. **interrupt 传递上下文**：把当前草稿和修改次数传给用户，方便审阅
3. **退出条件**：用户输入 `"ok"` 时走向 `END`，否则回到 `generate`
4. **状态累积**：`revision_count` 自动递增，记录修改轮次

## 子场景 2：带历史记录的反馈

### 思路

在基础循环上增加完整的历史记录：
- 保存每一轮的草稿版本
- 保存每一轮的反馈内容
- 可以回溯查看所有修改历程

关键设计：
- 使用 `Annotated[list, operator.add]` 让列表字段自动追加（而非覆盖）
- `draft_history` 记录所有草稿版本
- `feedback_history` 记录所有反馈内容

### 完整代码

```python
"""
子场景 2：带历史记录的反馈循环
累积所有反馈和修改版本，支持回溯查看
"""

from typing import TypedDict, Annotated, Literal
from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph, START, END
import operator


# ========== 1. 定义状态（带历史记录） ==========

class HistoryFeedbackState(TypedDict):
    topic: str
    current_draft: str                                    # 当前草稿
    current_feedback: str                                 # 最新反馈
    revision_count: int                                   # 修改次数
    draft_history: Annotated[list[str], operator.add]     # 所有草稿版本
    feedback_history: Annotated[list[str], operator.add]  # 所有反馈记录


# ========== 2. 定义节点 ==========

def generate_with_history(state: HistoryFeedbackState):
    """生成内容并记录到历史"""
    revision = state.get("revision_count", 0)

    if revision == 0:
        draft = f"【v1 初稿】{state['topic']}：这是初始版本的内容。"
    else:
        feedback = state.get("current_feedback", "")
        draft = (
            f"【v{revision + 1} 修改版】{state['topic']}：\n"
            f"  针对反馈「{feedback}」的改进版本。"
        )

    return {
        "current_draft": draft,
        "revision_count": revision + 1,
        # Annotated[list, operator.add] 会自动追加到列表
        "draft_history": [draft],
    }


def collect_feedback_with_history(state: HistoryFeedbackState):
    """收集反馈并记录到历史"""
    feedback = interrupt({
        "type": "review_with_history",
        "current_draft": state["current_draft"],
        "revision_count": state["revision_count"],
        "total_versions": len(state.get("draft_history", [])),
        "previous_feedback": state.get("feedback_history", []),
        "instruction": "输入 'ok' 表示满意，或输入修改意见继续迭代。",
    })

    return {
        "current_feedback": feedback,
        # 自动追加到反馈历史
        "feedback_history": [feedback],
    }


def should_continue_with_history(
    state: HistoryFeedbackState,
) -> Literal["generate_with_history", "__end__"]:
    """根据反馈决定是否继续"""
    if state["current_feedback"].lower().strip() == "ok":
        return "__end__"
    return "generate_with_history"


# ========== 3. 构建图 ==========

builder = StateGraph(HistoryFeedbackState)
builder.add_node("generate_with_history", generate_with_history)
builder.add_node("collect_feedback_with_history", collect_feedback_with_history)

builder.add_edge(START, "generate_with_history")
builder.add_edge("generate_with_history", "collect_feedback_with_history")
builder.add_conditional_edges(
    "collect_feedback_with_history", should_continue_with_history
)

graph = builder.compile(checkpointer=MemorySaver())


# ========== 4. 运行演示 ==========

config = {"configurable": {"thread_id": "history-demo-1"}}

# 初始化
graph.invoke(
    {
        "topic": "Python 异步编程",
        "current_draft": "",
        "current_feedback": "",
        "revision_count": 0,
        "draft_history": [],
        "feedback_history": [],
    },
    config,
)

# 第一轮反馈：要求更详细
print("第一轮反馈：要求更详细")
graph.invoke(Command(resume="请增加 asyncio 的代码示例"), config)

# 第二轮反馈：要求更简洁
print("第二轮反馈：要求更简洁")
graph.invoke(Command(resume="示例太长了，请精简"), config)

# 第三轮反馈：满意
print("第三轮反馈：满意")
result = graph.invoke(Command(resume="ok"), config)

# ========== 5. 查看完整历史 ==========

print("\n" + "=" * 60)
print("完整修改历史")
print("=" * 60)

print(f"\n总修改次数: {result['revision_count']}")

print("\n--- 草稿版本历史 ---")
for i, draft in enumerate(result["draft_history"], 1):
    print(f"  版本 {i}: {draft}")

print("\n--- 反馈历史 ---")
for i, fb in enumerate(result["feedback_history"], 1):
    print(f"  反馈 {i}: {fb}")

print(f"\n最终版本: {result['current_draft']}")
```

### 运行输出

```
第一轮反馈：要求更详细
第二轮反馈：要求更简洁
第三轮反馈：满意

============================================================
完整修改历史
============================================================

总修改次数: 3

--- 草稿版本历史 ---
  版本 1: 【v1 初稿】Python 异步编程：这是初始版本的内容。
  版本 2: 【v2 修改版】Python 异步编程：针对反馈「请增加 asyncio 的代码示例」的改进版本。
  版本 3: 【v3 修改版】Python 异步编程：针对反馈「示例太长了，请精简」的改进版本。

--- 反馈历史 ---
  反馈 1: 请增加 asyncio 的代码示例
  反馈 2: 示例太长了，请精简
  反馈 3: ok

最终版本: 【v3 修改版】Python 异步编程：...
```

### 关键点解析

1. **`Annotated[list, operator.add]` 的魔力**：
   - 普通字段赋值是覆盖：`{"name": "new"}` 替换旧值
   - `Annotated[list, operator.add]` 是追加：`{"history": ["new"]}` 追加到列表
   - 这是 LangGraph 状态管理的核心技巧

2. **历史记录不会丢失**：每次节点返回 `{"draft_history": [draft]}` 时，新元素被追加到已有列表

3. **中断信息包含历史**：`interrupt()` 传递 `previous_feedback` 让用户看到之前的反馈

## 子场景 3：多阶段反馈

### 思路

真实项目中，内容创作往往分多个阶段，每个阶段都需要独立的反馈循环：

```
阶段 1：大纲反馈（结构是否合理？）
    ↓ 满意
阶段 2：内容反馈（内容是否准确？）
    ↓ 满意
阶段 3：格式反馈（排版是否美观？）
    ↓ 满意
完成！
```

每个阶段内部都是一个"生成→反馈→修改"的循环。

### 完整代码

```python
"""
子场景 3：多阶段反馈
大纲反馈 → 内容反馈 → 格式反馈，每个阶段都有独立的反馈循环
"""

from typing import TypedDict, Annotated, Literal
from langgraph.types import interrupt, Command
from langgraph.checkpoint.memory import MemorySaver
from langgraph.graph import StateGraph, START, END
import operator


# ========== 1. 定义状态 ==========

class MultiStageState(TypedDict):
    topic: str
    current_stage: str                                       # 当前阶段
    outline: str                                             # 大纲
    content: str                                             # 正文
    formatted: str                                           # 格式化版本
    current_feedback: str                                    # 当前反馈
    revision_count: int                                      # 当前阶段修改次数
    stage_history: Annotated[list[dict], operator.add]       # 各阶段历史


# ========== 2. 阶段 1：大纲 ==========

def generate_outline(state: MultiStageState):
    """生成或修改大纲"""
    revision = state.get("revision_count", 0)
    feedback = state.get("current_feedback", "")

    if revision == 0:
        outline = (
            f"# {state['topic']}\n"
            f"## 1. 概述\n"
            f"## 2. 核心原理\n"
            f"## 3. 实战应用\n"
            f"## 4. 总结"
        )
    else:
        outline = (
            f"# {state['topic']}（大纲 v{revision + 1}）\n"
            f"根据反馈「{feedback}」调整了大纲结构。\n"
            f"## 1. 概述\n## 2. 核心原理\n## 3. 实战应用\n## 4. 总结"
        )

    return {
        "outline": outline,
        "current_stage": "outline",
        "revision_count": revision + 1,
    }


def review_outline(state: MultiStageState):
    """大纲审阅"""
    feedback = interrupt({
        "stage": "大纲审阅",
        "content": state["outline"],
        "revision_count": state["revision_count"],
        "instruction": "请审阅大纲。输入 'ok' 进入下一阶段，或输入修改意见。",
    })
    return {"current_feedback": feedback}


def outline_router(state: MultiStageState) -> Literal[
    "generate_outline", "generate_content"
]:
    """大纲阶段路由"""
    if state["current_feedback"].lower().strip() == "ok":
        return "generate_content"
    return "generate_outline"


# ========== 3. 阶段 2：内容 ==========

def generate_content(state: MultiStageState):
    """生成或修改正文"""
    feedback = state.get("current_feedback", "")

    # 进入新阶段时记录上一阶段历史，并重置计数
    stage_entry = []
    if state.get("current_stage") == "outline":
        stage_entry = [{
            "stage": "outline",
            "final_version": state["outline"],
            "revisions": state["revision_count"],
        }]

    revision = 0 if state.get("current_stage") != "content" else state["revision_count"]

    if revision == 0:
        content = f"基于大纲生成的正文内容...\n{state['outline']}\n\n详细内容在这里。"
    else:
        content = f"正文 v{revision + 1}：根据反馈「{feedback}」修改后的内容。"

    return {
        "content": content,
        "current_stage": "content",
        "revision_count": revision + 1,
        "stage_history": stage_entry,
    }


def review_content(state: MultiStageState):
    """正文审阅"""
    feedback = interrupt({
        "stage": "正文审阅",
        "content": state["content"],
        "revision_count": state["revision_count"],
        "instruction": "请审阅正文。输入 'ok' 进入格式化阶段，或输入修改意见。",
    })
    return {"current_feedback": feedback}


def content_router(state: MultiStageState) -> Literal[
    "generate_content", "generate_format"
]:
    """正文阶段路由"""
    if state["current_feedback"].lower().strip() == "ok":
        return "generate_format"
    return "generate_content"


# ========== 4. 阶段 3：格式化 ==========

def generate_format(state: MultiStageState):
    """生成或修改格式"""
    feedback = state.get("current_feedback", "")

    stage_entry = []
    if state.get("current_stage") == "content":
        stage_entry = [{
            "stage": "content",
            "final_version": state["content"],
            "revisions": state["revision_count"],
        }]

    revision = 0 if state.get("current_stage") != "format" else state["revision_count"]

    if revision == 0:
        formatted = f"📄 格式化版本：\n\n{state['content']}\n\n--- 已添加排版和样式 ---"
    else:
        formatted = f"📄 格式 v{revision + 1}：根据反馈「{feedback}」调整了排版。"

    return {
        "formatted": formatted,
        "current_stage": "format",
        "revision_count": revision + 1,
        "stage_history": stage_entry,
    }


def review_format(state: MultiStageState):
    """格式审阅"""
    feedback = interrupt({
        "stage": "格式审阅",
        "content": state["formatted"],
        "revision_count": state["revision_count"],
        "instruction": "请审阅最终格式。输入 'ok' 完成，或输入修改意见。",
    })
    return {"current_feedback": feedback}


def format_router(state: MultiStageState) -> Literal["generate_format", "finalize"]:
    """格式阶段路由"""
    if state["current_feedback"].lower().strip() == "ok":
        return "finalize"
    return "generate_format"


# ========== 5. 完成节点 ==========

def finalize(state: MultiStageState):
    """记录最终阶段历史"""
    return {
        "current_stage": "done",
        "stage_history": [{
            "stage": "format",
            "final_version": state["formatted"],
            "revisions": state["revision_count"],
        }],
    }


# ========== 6. 构建图 ==========

builder = StateGraph(MultiStageState)

# 添加所有节点
builder.add_node("generate_outline", generate_outline)
builder.add_node("review_outline", review_outline)
builder.add_node("generate_content", generate_content)
builder.add_node("review_content", review_content)
builder.add_node("generate_format", generate_format)
builder.add_node("review_format", review_format)
builder.add_node("finalize", finalize)

# 阶段 1：大纲循环
builder.add_edge(START, "generate_outline")
builder.add_edge("generate_outline", "review_outline")
builder.add_conditional_edges("review_outline", outline_router)

# 阶段 2：内容循环
builder.add_edge("generate_content", "review_content")
builder.add_conditional_edges("review_content", content_router)

# 阶段 3：格式循环
builder.add_edge("generate_format", "review_format")
builder.add_conditional_edges("review_format", format_router)

# 完成
builder.add_edge("finalize", END)

graph = builder.compile(checkpointer=MemorySaver())


# ========== 7. 运行演示 ==========

config = {"configurable": {"thread_id": "multi-stage-demo-1"}}

# 初始化
graph.invoke(
    {
        "topic": "LangGraph 入门指南",
        "outline": "",
        "content": "",
        "formatted": "",
        "current_stage": "",
        "current_feedback": "",
        "revision_count": 0,
        "stage_history": [],
    },
    config,
)

# --- 阶段 1：大纲 ---
print("=" * 60)
print("阶段 1：大纲审阅")
print("=" * 60)

# 查看中断信息
snapshot = graph.get_state(config)
for task in snapshot.tasks:
    for intr in task.interrupts:
        print(f"  阶段: {intr.value['stage']}")
        print(f"  内容: {intr.value['content'][:80]}...")

# 大纲反馈：修改一次
print("\n反馈：增加实战案例章节")
graph.invoke(Command(resume="请增加一个实战案例章节"), config)

# 大纲反馈：满意
print("反馈：大纲 OK")
graph.invoke(Command(resume="ok"), config)

# --- 阶段 2：内容 ---
print("\n" + "=" * 60)
print("阶段 2：正文审阅")
print("=" * 60)

# 内容反馈：直接满意
print("反馈：内容 OK")
graph.invoke(Command(resume="ok"), config)

# --- 阶段 3：格式 ---
print("\n" + "=" * 60)
print("阶段 3：格式审阅")
print("=" * 60)

# 格式反馈：满意
print("反馈：格式 OK")
result = graph.invoke(Command(resume="ok"), config)

# --- 查看完整历史 ---
print("\n" + "=" * 60)
print("完整阶段历史")
print("=" * 60)
for entry in result["stage_history"]:
    print(f"\n阶段: {entry['stage']}")
    print(f"  修改次数: {entry['revisions']}")
    print(f"  最终版本: {entry['final_version'][:60]}...")
```

### 关键点解析

1. **多阶段串联**：每个阶段是独立的"生成→审阅→路由"循环，通过条件边串联
2. **阶段切换时记录历史**：进入新阶段时，把上一阶段的最终版本和修改次数存入 `stage_history`
3. **独立的修改计数**：每个阶段有自己的 `revision_count`，进入新阶段时重置
4. **灵活的退出**：每个阶段都可以独立决定"继续修改"或"进入下一阶段"

## 常见问题

### Q1：循环会不会无限执行？

不会。每次循环都会遇到 `interrupt()` 暂停，等待用户输入。只有用户主动输入非 `"ok"` 的反馈才会继续循环。如果担心，可以在条件路由中加上最大修改次数限制：

```python
def should_continue(state: FeedbackState) -> Literal["generate", "__end__"]:
    if state["feedback"].lower().strip() == "ok":
        return "__end__"
    if state["revision_count"] >= 10:  # 最多修改 10 次
        return "__end__"
    return "generate"
```

### Q2：`Annotated[list, operator.add]` 和普通 list 有什么区别？

```python
# 普通 list：每次赋值都是覆盖
class State1(TypedDict):
    history: list[str]
# 节点返回 {"history": ["v2"]} → history 变成 ["v2"]（v1 丢失）

# Annotated list：每次赋值都是追加
class State2(TypedDict):
    history: Annotated[list[str], operator.add]
# 节点返回 {"history": ["v2"]} → history 变成 ["v1", "v2"]（v1 保留）
```

### Q3：多阶段反馈中，如何跳过某个阶段？

可以在审阅节点中增加 `"skip"` 选项：

```python
def review_content(state):
    feedback = interrupt({
        "instruction": "输入 'ok' 进入下一阶段，'skip' 跳过，或输入修改意见。",
        ...
    })
    return {"current_feedback": feedback}

def content_router(state) -> Literal["generate_content", "generate_format"]:
    fb = state["current_feedback"].lower().strip()
    if fb in ("ok", "skip"):
        return "generate_format"
    return "generate_content"
```

## 总结

| 子场景 | 核心技巧 | 适用场景 |
|--------|----------|----------|
| 基础反馈循环 | 条件边形成循环 + interrupt | 简单的"改到满意为止" |
| 带历史记录 | `Annotated[list, operator.add]` 累积历史 | 需要回溯修改过程 |
| 多阶段反馈 | 多个独立循环串联 | 复杂的多步骤创作流程 |

核心模式：**interrupt 暂停 → 用户决策 → 条件路由（继续/退出）→ 状态累积**。
