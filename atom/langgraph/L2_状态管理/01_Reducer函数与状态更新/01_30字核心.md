# Reducer函数与状态更新 - 30字核心

**Reducer函数是定义状态字段更新策略的二元函数，用于合并多个节点返回的值，实现累积型状态管理。**

---

## 核心要素拆解

### 1. 定义：二元函数
- **函数签名**：`(Value, Value) -> Value`
- **接收两个参数**：旧值（left）和新值（right）
- **返回一个值**：合并后的结果

### 2. 作用：定义状态字段更新策略
- **绑定到状态字段**：使用 `Annotated[type, reducer]`
- **控制合并行为**：替换 vs 累积
- **自动调用**：框架在状态更新时自动执行

### 3. 价值：实现累积型状态管理
- **消息历史**：聊天机器人的对话累积
- **文档列表**：RAG 系统的检索结果累积
- **计数器**：多节点的数值累加
- **配置合并**：多节点的配置更新

---

## 一句话记忆法

**"Reducer = 两个值进，一个值出，定义如何合并状态"**

---

## 快速示例

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator

# 定义状态：messages 字段使用 operator.add 作为 Reducer
class State(TypedDict):
    messages: Annotated[list[str], operator.add]  # Reducer 函数
    counter: int  # 无 Reducer，直接替换

# 节点 A 返回
{"messages": ["Hello"], "counter": 1}

# 节点 B 返回
{"messages": ["World"], "counter": 2}

# 最终状态
{
    "messages": ["Hello", "World"],  # 使用 operator.add 合并
    "counter": 2  # 直接替换
}
```

---

## 核心对比

| 场景 | 无 Reducer | 有 Reducer（operator.add） |
|------|-----------|---------------------------|
| 节点 A 返回 | `{"x": [1, 2]}` | `{"x": [1, 2]}` |
| 节点 B 返回 | `{"x": [3, 4]}` | `{"x": [3, 4]}` |
| 最终状态 | `{"x": [3, 4]}` | `{"x": [1, 2, 3, 4]}` |
| 行为 | 直接替换 | 列表合并 |

## 为什么重要？

### 问题：多节点更新同一字段
在 LangGraph 中，多个节点可能同时或先后更新同一个状态字段：

```python
# 场景：两个节点都返回消息
node_a_output = {"messages": ["用户问题"]}
node_b_output = {"messages": ["AI回答"]}

# 问题：最终状态的 messages 应该是什么？
# 选项1：["AI回答"]（替换）
# 选项2：["用户问题", "AI回答"]（合并）
```

### 解决：Reducer 定义合并策略
```python
# 使用 Reducer 明确指定合并策略
class State(TypedDict):
    messages: Annotated[list, operator.add]  # 合并策略

# 结果：["用户问题", "AI回答"]
```

## 三个关键点

### 1. 签名固定
```python
# ✓ 正确：两个参数，返回一个值
def my_reducer(left: list, right: list) -> list:
    return left + right

# ✗ 错误：参数数量不对
def wrong_reducer(value: list) -> list:
    return value
```

### 2. 绑定方式
```python
# 使用 Annotated 绑定
from typing import Annotated

class State(TypedDict):
    field: Annotated[type, reducer_function]
```

### 3. 调用时机
```python
# 框架自动调用，无需手动执行
# 当多个节点更新同一字段时，Reducer 自动合并
```

## 常见 Reducer

| Reducer | 用途 | 示例 |
|---------|------|------|
| `operator.add` | 列表合并、字符串拼接、数值相加 | `[1, 2] + [3, 4] = [1, 2, 3, 4]` |
| `operator.or_` | 字典合并 | `{"a": 1} \| {"b": 2} = {"a": 1, "b": 2}` |
| `add_messages` | 消息列表按 ID 合并 | 相同 ID 的消息会被替换 |
| 自定义函数 | 复杂业务逻辑 | 条件合并、过滤、转换 |

## 实际应用场景

### 场景1：聊天机器人
```python
class State(TypedDict):
    messages: Annotated[list, add_messages]  # 累积对话历史

# 用户输入 → AI 回复 → 用户输入 → AI 回复
# messages 自动累积所有对话
```

### 场景2：RAG 系统
```python
class State(TypedDict):
    documents: Annotated[list, operator.add]  # 累积检索到的文档

# 检索节点1 → 检索节点2 → 检索节点3
# documents 自动合并所有检索结果
```

### 场景3：计数器
```python
class State(TypedDict):
    count: Annotated[int, operator.add]  # 累加计数

# 节点A: count += 1
# 节点B: count += 2
# 最终: count = 3
```

## 与日常生活类比

| LangGraph Reducer | 日常生活类比 | 说明 |
|-------------------|--------------|------|
| Reducer 函数 | 记账规则 | 定义如何累积收支 |
| `operator.add` | 购物车添加商品 | 新商品追加到列表 |
| `operator.or_` | 合并配置文件 | 新配置覆盖旧配置 |
| 无 Reducer | 替换手机壁纸 | 新壁纸直接替换旧壁纸 |

---

## 记忆口诀

**"两值进，一值出，定义合并不糊涂"**

- **两值进**：接收旧值和新值
- **一值出**：返回合并后的值
- **定义合并**：明确指定合并策略
- **不糊涂**：避免状态更新的混乱
