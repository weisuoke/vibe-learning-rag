# 状态转换函数 - 一句话总结

## 核心总结

**状态转换函数是 LangGraph 节点内部将输入状态转换为输出更新的处理逻辑，通过"接收完整状态 → 转换处理 → 返回部分更新"的模式，实现了安全、可组合、可测试的数据流转，是构建状态化工作流的原子操作单元。**

---

## 关键记忆点

| 维度 | 核心要点 |
|------|----------|
| 本质 | 节点函数：`State -> Partial<State>` |
| 输入 | 接收完整状态对象 |
| 输出 | 只返回要更新的字段 |
| 无 Reducer | 返回值直接覆盖 |
| 有 Reducer | 返回增量，由 reducer 合并 |
| Command | 状态更新 + 路由决策一体化 |
| 管道模式 | 多节点串联，每步做一件事 |
| 数据清洗 | 作为图中的节点，可追踪可回溯 |

---

## 速记公式

```
状态转换函数 = 读取状态 + 转换处理 + 返回部分更新
```
