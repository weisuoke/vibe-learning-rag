# 一句话总结：Agent Memory 集成

> Agent Memory 的一切，浓缩成一句话

---

## 核心总结

**Agent Memory = Checkpointer（短期对话记忆）+ Store（长期用户记忆）+ Middleware（记忆管理策略），三者协同让 Agent 在对的时间、对的地方、记住对的东西。**

---

## 每个子概念的一句话总结

| # | 概念 | 一句话 |
|---|------|--------|
| 1 | **Memory 架构演进** | 从 BaseMemory 手动管理，到 Checkpointer + Store 自动双层，三代演进让记忆管理从复杂变简单 |
| 2 | **短期记忆** | Checkpointer + thread_id = 同一会话内自动记住对话，不同会话自动隔离 |
| 3 | **记忆压缩** | SummarizationMiddleware 把 100 条旧消息压缩成 1 条摘要，让长对话永不超限 |
| 4 | **长期记忆** | Store 是跨会话的「通讯录备注」，配合语义搜索让 Agent 在新对话中仍然了解用户 |
| 5 | **工作记忆** | Agent 调用工具的中间结果就是工作记忆——做数学题的草稿纸，用完可以清理 |
| 6 | **持久化后端** | 开发用 InMemory → 轻量用 SQLite → 生产用 PostgreSQL → 高性能用 Redis |

---

## 速记公式

```
Agent 记忆 = checkpointer + store + middleware

  checkpointer → 记住对话（自动）
  store        → 记住用户（跨会话）
  middleware   → 管理记忆（防超限）

  thread_id    → 谁的记忆和谁隔离
  namespace    → 长期记忆的分组方式
```

---

## 最终口诀

```
Checkpointer 管对话，Store 管人，Middleware 管大小。
thread_id 做隔离，PostgreSQL 做持久。
记忆不是越多越好，而是记对的东西。
```
