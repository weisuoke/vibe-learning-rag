## 知识体系

### 核心概念（7个）

```
1. 过滤表达式语法
   - 比较操作符：==, !=, >, <, >=, <=
   - 范围操作符：IN, LIKE
   - NULL 操作符：IS NULL, IS NOT NULL

2. 布尔运算符
   - AND / OR / NOT 操作符
   - 复合条件组合
   - 位图操作优化

3. 数据类型支持
   - 标量类型：INT, FLOAT, DOUBLE, VARCHAR, BOOL
   - JSON 类型：嵌套对象、数组
   - ARRAY 类型：INT32, INT64, VARCHAR 数组

4. JSON Path 语法
   - 路径表达式：json_field['a']['b']
   - 数组访问：json_field['a'][0]
   - 嵌套访问：json_field['a'][0]['b']

5. JSON 索引类型
   - JsonFlatIndex：扁平索引
   - JsonInvertedIndex：倒排索引（推荐）

6. 标量索引优化
   - INVERTED：通用场景
   - BITMAP：低基数字段
   - STL_SORT：范围查询
   - AUTOINDEX：自动选择

7. 混合查询执行策略
   - Pre-filtering：高选择性
   - Post-filtering：低选择性
   - Hybrid：自动选择
```

### 索引选择

```
基数判断：
基数 = DISTINCT(field) / COUNT(*)

索引选择：
IF 基数 < 0.01:
    使用 BITMAP
ELSE:
    使用 INVERTED

选择性判断：
选择性 = 过滤后数据量 / 总数据量

策略选择：
IF 选择性 > 50%:
    使用 Pre-filtering
ELSE:
    使用 Post-filtering
```

## 快速参考

### 创建索引

```python
# 标量索引
index_params.add_index("age", "INVERTED")
index_params.add_index("status", "BITMAP")

# JSON Path Index
index_params.add_index(
    field_name="metadata",
    index_name="category_index",
    index_type="INVERTED",
    params={
        "json_path": "metadata['product']['category']",
        "json_cast_type": "varchar"
    }
)
```

# 标量过滤
filter = "age > 30 AND city == '北京'"

## 常见问题

### Q1: 什么时候使用标量过滤？

**答案**：当你需要精确条件筛选时，如价格范围、库存状态、类别筛选等。

### Q2: JSON Path Index 和标量索引有什么区别？

**答案**：
- 标量索引：用于固定字段（age, price, status）
- JSON Path Index：用于 JSON 字段的特定路径（metadata['category']）

### Q3: 如何选择索引类型？

**答案**：
- 低基数（< 100个值）→ BITMAP
- 中高基数 → INVERTED
- 不确定 → AUTOINDEX

### Q4: 性能提升有多大？

**答案**：
- 标量索引：30 倍
- JSON Path Index：100 倍
