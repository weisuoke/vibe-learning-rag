# Gateway 启动与管理 - 概览

## 知识点概述

**Gateway 启动与管理**是 OpenClaw 多通道 AI 网关的运行时控制系统，涵盖从开发调试到生产部署的全生命周期管理。

本知识点将帮助你掌握：
- Gateway 的两种启动模式（前台/守护进程）
- 端口绑定策略与网络安全配置
- 守护进程的安装、管理与监控
- 日志系统的使用与调试技巧
- 多 Gateway 实例的隔离与管理

## 学习目标

学完本知识点后，你将能够：

1. **启动 Gateway**：理解前台和守护进程两种模式的区别和使用场景
2. **配置端口绑定**：掌握 loopback/lan/tailnet 三种绑定模式的安全策略
3. **管理守护进程**：在 macOS (launchd) 和 Linux (systemd) 上安装和管理 Gateway 服务
4. **调试问题**：使用日志系统快速定位和解决 Gateway 运行问题
5. **多实例管理**：通过 profile 机制运行多个隔离的 Gateway 实例

## 核心概念速览

### 1. Gateway 启动模式

**前台启动（Foreground）**
- 运行在当前终端
- 日志实时输出
- 适合开发调试
- Ctrl+C 停止

**守护进程（Daemon）**
- 后台持续运行
- 系统启动时自动运行
- 适合生产部署
- 通过服务管理

### 2. 端口配置与绑定

**loopback（默认）**
- 只监听 127.0.0.1
- 最安全
- 本地访问

**lan**
- 监听所有接口
- 需要认证
- LAN 访问

**tailnet**
- 只监听 Tailscale 接口
- 安全远程访问
- 端到端加密

### 3. 守护进程管理

**macOS (launchd)**
```bash
openclaw onboard --install-daemon
launchctl start ai.openclaw.gateway
```

**Linux (systemd)**
```bash
openclaw onboard --install-daemon
systemctl --user start openclaw-gateway
```

### 4. 日志与调试

**控制台日志**
```bash
openclaw gateway --verbose
```

**文件日志**
```bash
~/.openclaw/logs/gateway.log
openclaw logs --follow
```

## 常见问题速查

### Q1: Gateway 启动失败？

```bash
# 检查端口占用
lsof -i :18789

# 强制释放端口
openclaw gateway --force

# 运行诊断
openclaw doctor
```

### Q2: 无法从其他设备访问？

```bash
# 检查绑定模式
openclaw gateway status

# 使用 LAN 绑定 + 认证
openclaw gateway --bind lan --token <token>
```

### Q3: 日志在哪里？

```bash
# 控制台日志（前台模式）
openclaw gateway --verbose

# 文件日志（守护进程）
~/.openclaw/logs/gateway.log
openclaw logs --follow
```

### Q4: 如何运行多个 Gateway？

```bash
# 使用不同 profile
openclaw gateway --profile dev --port 18790
openclaw gateway --profile rescue --port 18791
```

### Q5: 守护进程如何管理？

```bash
# macOS
launchctl list | grep openclaw
launchctl start ai.openclaw.gateway

# Linux
systemctl --user status openclaw-gateway
systemctl --user restart openclaw-gateway
```
