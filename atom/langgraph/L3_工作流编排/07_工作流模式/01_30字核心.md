# 07_工作流模式 - 30字核心

> LangGraph L3_工作流编排 | 维度1

**工作流模式是六种经过验证的 AI 编排架构模板，让你根据任务特征直接套用最优的图结构设计。**

---

## 核心要素拆解

### 1. "六种经过验证的架构模板" —— 不用从零设计

这六种模式由 Anthropic 和 LangChain 团队从大量 AI 应用实践中提炼而来，每种解决一类特定问题：

| 模式 | 一句话定义 | 关键词 |
|------|-----------|--------|
| 顺序链式 | 步骤按顺序执行，中间可设 Gate 检查 | 线性、Gate、管道 |
| 路由 | 根据输入特征动态选择执行路径 | 分类、分发、LLM 路由 |
| 并行 | 多个分支同时执行后汇聚结果 | Fan-out、Fan-in、defer |
| 编排者-工作者 | LLM 动态拆解任务并分配执行 | 动态分解、@task、合成 |
| 评估者-优化者 | 生成后评估，不合格则迭代改进 | 循环、反馈、质量门控 |
| Map-Reduce | 运行时动态创建 N 个并行任务 | Send、动态并行、reducer |

### 2. "根据任务特征" —— 选型而非发明

模式选型的核心判断：

```
任务特征              →  对应模式
─────────────────────────────────────
步骤固定、顺序执行     →  顺序链式
输入不同、路径不同     →  路由
子任务独立、可并行     →  并行 / Map-Reduce
任务复杂、需动态拆解   →  编排者-工作者
输出需迭代改进        →  评估者-优化者
```

### 3. "最优的图结构设计" —— 模式即图的形状

每种模式对应一种图的拓扑结构：

```
顺序链式:  ● → ● → ● → ●          (直线)

路由:      ● → ● ─┬→ ●            (分叉)
                   └→ ●

并行:      ● ─┬→ ● ─┬→ ●          (菱形)
               └→ ● ─┘

编排者:    ● → ● ─┬→ ● ─┐         (星形)
                   ├→ ● ─┼→ ●
                   └→ ● ─┘

评估循环:  ● → ● → ● ─┐            (环形)
                ↑      │
                └──────┘

Map-Reduce: ● → ● ═╦→ ● ─╦→ ●    (动态扇形)
                    ╠→ ● ─╣
                    ╚→ ● ─╝
```

## 关键词提取

| 关键词 | 含义 |
|--------|------|
| 工作流模式 | 经过验证的 AI 编排架构模板，解决特定类型的编排问题 |
| Prompt Chaining | 顺序链式，多个 LLM 调用按序执行 |
| Routing | 路由，根据输入动态选择处理路径 |
| Parallelization | 并行，Fan-out 分发 + Fan-in 汇聚 |
| Orchestrator-Worker | 编排者-工作者，LLM 动态分解任务 |
| Evaluator-Optimizer | 评估者-优化者，生成-评估反馈循环 |
| Map-Reduce | Send API 驱动的动态并行 + reducer 聚合 |

---

## 与 RAG/Agent 开发的关联

六大模式在 RAG 和 Agent 系统中的典型映射：

- **顺序链式** → 标准 RAG 管道（查询 → 检索 → 生成）
- **路由** → 问题分类后走不同检索策略
- **并行** → 同时查询多个知识库
- **编排者-工作者** → 复杂问题拆解为多个子问题分别检索
- **评估者-优化者** → Self-RAG（检索质量不够则改写查询重试）
- **Map-Reduce** → 对 N 篇检索文档并行生成摘要后综合
