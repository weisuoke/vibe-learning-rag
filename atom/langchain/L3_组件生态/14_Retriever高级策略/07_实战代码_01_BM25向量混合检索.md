# Retriever高级策略 - 实战代码：BM25+向量混合检索

> **完整可运行的混合检索示例**：使用 EnsembleRetriever 融合 BM25 关键词检索与 ChromaDB 向量检索

---

## 场景描述

构建一个混合检索系统，结合 BM25 关键词检索和 ChromaDB 向量语义检索。BM25 擅长精确关键词匹配（如产品型号、专有名词），向量检索擅长语义理解（如"便宜的笔记本"匹配"高性价比轻薄本"）。两者互补，通过 RRF（Reciprocal Rank Fusion）算法融合排序。

**适用场景：**
- 企业知识库问答（技术文档 + 产品手册）
- 客服系统（精确匹配工单号 + 语义理解用户意图）
- 内部搜索引擎（关键词 + 语义双通道）

### Step 2: 创建 BM25 检索器

```python
"""
```

### Step 3: 创建向量检索器

```python
"""
Step 3: 创建 ChromaDB 向量检索器
向量检索基于语义相似度，擅长理解自然语言意图
"""

from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from dotenv import load_dotenv

# 创建向量存储
vectorstore = Chroma.from_documents(
    documents,
    embedding=OpenAIEmbeddings(),

# 转换为检索器
```

### Step 4: 创建 EnsembleRetriever

```python
"""
Step 4: 使用 EnsembleRetriever 融合两种检索器
核心算法: Weighted Reciprocal Rank Fusion (RRF)
公式: score(doc) = Σ weight_i / (rank_i + c)
"""

from langchain.retrievers import EnsembleRetriever

# 创建混合检索器
# weights: 向量检索 0.6 + BM25 0.4（生产推荐默认值）
ensemble_retriever = EnsembleRetriever(
    retrievers=[vector_retriever, bm25_retriever],
    weights=[0.6, 0.4],
    c=60,  # RRF 常数，原始论文默认值
)

print("=== EnsembleRetriever 创建成功 ===")
print(f"检索器数量: {len(ensemble_retriever.retrievers)}")
print(f"权重分配: 向量={ensemble_retriever.weights[0]}, BM25={ensemble_retriever.weights[1]}")
print(f"RRF 常数 c={ensemble_retriever.c}")
```

### Step 6: 权重调优实验

```python
"""
Step 6: 通过调整权重观察检索效果变化
帮助理解 weights 参数对结果排序的影响
"""

def weight_experiment(query: str, weight_configs: list[tuple[float, float]]):
    """测试不同权重配置的效果"""
    print(f"\n{'='*60}")
    print(f"权重调优实验 - 查询: '{query}'")
    print(f"{'='*60}")

    for vec_w, bm25_w in weight_configs:
        retriever = EnsembleRetriever(
            retrievers=[vector_retriever, bm25_retriever],
            weights=[vec_w, bm25_w],
        )
        results = retriever.invoke(query)
        print(f"\n向量={vec_w}, BM25={bm25_w}:")
        for i, doc in enumerate(results[:3], 1):
            print(f"  {i}. [{doc.metadata['product']}] {doc.page_content[:40]}...")


# 实验: 从纯 BM25 到纯向量
weight_configs = [
    (0.0, 1.0),  # 纯 BM25
    (0.3, 0.7),  # BM25 主导
    (0.5, 0.5),  # 均衡
    (0.6, 0.4),  # 向量略高（推荐默认）
    (1.0, 0.0),  # 纯向量
]

# 精确查询：BM25 权重高更好
weight_experiment("StarPhone 16 A18 芯片", weight_configs)

# 语义查询：向量权重高更好
weight_experiment("拍照好的手机", weight_configs)
```

## 关键参数说明

| 参数 | 默认值 | 说明 | 调优建议 |
|------|--------|------|----------|
| `weights` | 等权重 `[1/n, ...]` | 每个检索器的 RRF 权重 | 向量 0.5-0.7 + BM25 0.3-0.5 |
| `c` | `60` | RRF 常数，平衡高低排名差异 | 保持默认 60 即可 |
| `id_key` | `None` | 去重用的元数据键 | 有唯一 ID 时设置，避免重复文档 |
| BM25 `k` | `4` | BM25 返回文档数 | 设为最终需要数的 2-3 倍 |
| Vector `k` | `4` | 向量检索返回文档数 | 与 BM25 的 k 保持一致 |

### RRF 算法直觉理解

```
score(doc) = Σ weight_i / (rank_i + c)

示例：某文档在向量检索排第1，BM25排第3
  向量贡献 = 0.6 / (1 + 60) = 0.00984
  BM25贡献 = 0.4 / (3 + 60) = 0.00635
  总分 = 0.01619

c=60 的作用：让排名差异的影响变得平滑
  排第1 vs 排第2 的分数差 ≈ 0.00016（很小）
  → 不会因为一个检索器的微小排名差异就大幅改变最终排序
```

## 常见问题

### Q1: BM25Retriever 对中文支持好吗？

默认的 BM25Retriever 使用空格分词，对中文效果一般。生产环境建议自定义分词：

```python
import jieba

def chinese_tokenizer(text: str) -> list[str]:
    """使用 jieba 分词"""
    return list(jieba.cut(text))

bm25_retriever = BM25Retriever.from_documents(
    documents,
    k=4,
    preprocess_func=chinese_tokenizer,  # 自定义分词函数
)
```

### Q2: weights 怎么选？

经验法则：
- **技术文档/产品手册**（关键词多）：BM25 权重高，如 `[0.4, 0.6]`
- **客服对话/自然语言问答**（语义为主）：向量权重高，如 `[0.7, 0.3]`
- **通用场景**：`[0.6, 0.4]` 是社区推荐的默认起点

### Q3: 子检索器的 k 值要一样吗？

不需要，但建议都设为最终需要数的 2-3 倍。比如最终要 top-5，每个子检索器设 k=10-15，让 RRF 有足够的候选文档来融合排序。

### Q4: 能加更多检索器吗？

可以。EnsembleRetriever 支持任意数量的检索器：

```python
ensemble = EnsembleRetriever(
    retrievers=[vector_retriever, bm25_retriever, another_retriever],
    weights=[0.5, 0.3, 0.2],  # 权重之和不必为 1，内部会归一化
)
```
