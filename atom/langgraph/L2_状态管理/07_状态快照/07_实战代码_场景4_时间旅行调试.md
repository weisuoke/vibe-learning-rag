# 实战代码 - 场景4：时间旅行调试

## 场景说明

本场景演示完整的时间旅行调试工作流，模拟一个 RAG Agent 的调试过程。你将看到：

1. 构建一个 RAG-like Agent（查询改写 → 检索 → 评估 → 生成）
2. 执行后发现生成结果不理想
3. 使用 Replay（重放）回到检索步骤观察中间状态
4. 使用 Fork（分叉）修改检索结果
5. 从修改后的状态继续执行，得到更好的结果
6. 对比原始路径和分叉路径的完整执行历史

这是状态快照的终极应用——把 `get_state_history`、`update_state`、`invoke(None)` 串联成完整的调试工作流。

[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]
[来源: reference/search_状态快照_01.md | 社区实践案例]

## 完整代码

```python
"""
LangGraph 状态快照实战 - 场景4：时间旅行调试
演示：完整的 Replay + Fork 调试工作流，模拟 RAG Agent 调试

核心知识点：
- Replay: 从特定 checkpoint 重放执行
- Fork: 修改历史状态后从分叉点继续执行
- 完整调试流程：发现问题 → 定位 → 修改 → 重新执行
- checkpoint_id 在 config 中的使用

运行环境：Python 3.13+, langgraph
安装依赖：uv add langgraph
"""

from typing import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver


# ============================================================
# 1. 定义 RAG Agent 状态
# ============================================================

class RAGAgentState(TypedDict):
    """RAG Agent 状态 —— 模拟完整的检索增强生成流程"""
    query: str                              # 用户原始查询
    rewritten_query: str                    # 改写后的查询
    retrieved_docs: list[str]               # 检索到的文档
    relevance_score: float                  # 文档相关度评分
    answer: str                             # 生成的回答
    messages: Annotated[list[str], add]     # 执行日志（追加模式）
    is_relevant: bool                       # 文档是否足够相关


# ============================================================
# 2. 定义节点函数 —— 模拟 RAG 各步骤
# ============================================================

def query_rewrite(state: RAGAgentState) -> dict:
    """查询改写：优化用户查询以提高检索质量"""
    original = state["query"]
    # 模拟改写：添加关键词扩展
    rewritten = f"{original} 原理 实现 最佳实践"
    return {
        "rewritten_query": rewritten,
        "messages": [f"[改写] '{original}' → '{rewritten}'"],
    }


def retrieve_docs(state: RAGAgentState) -> dict:
    """文档检索：根据改写后的查询检索文档

    故意返回质量不高的结果，用于后续调试演示
    """
    query = state["rewritten_query"]
    # 模拟检索：返回一些不太相关的文档
    docs = [
        "Python 基础教程：变量和数据类型",       # 不相关
        "机器学习入门：监督学习与非监督学习",     # 部分相关
        "向量数据库简介：Milvus 安装指南",       # 部分相关
    ]
    return {
        "retrieved_docs": docs,
        "messages": [f"[检索] 找到 {len(docs)} 篇文档"],
    }


def evaluate_relevance(state: RAGAgentState) -> dict:
    """相关度评估：判断检索结果是否足够好"""
    docs = state["retrieved_docs"]
    # 模拟评估：基于文档内容简单打分
    # 实际场景中会用 LLM 或交叉编码器评估
    score = 0.4  # 故意给低分，模拟检索质量不佳
    is_relevant = score >= 0.7
    return {
        "relevance_score": score,
        "is_relevant": is_relevant,
        "messages": [f"[评估] 相关度={score}, 是否足够={'是' if is_relevant else '否'}"],
    }


def generate_answer(state: RAGAgentState) -> dict:
    """生成回答：基于检索文档生成最终回答"""
    docs = state["retrieved_docs"]
    score = state["relevance_score"]
    query = state["query"]

    if score >= 0.8:
        answer = f"关于'{query}'：基于高质量文档的详细回答。文档覆盖了原理、实现和最佳实践。"
    elif score >= 0.5:
        answer = f"关于'{query}'：基于部分相关文档的回答，可能不够全面。建议补充更多资料。"
    else:
        answer = f"关于'{query}'：抱歉，检索到的文档相关度较低（{score}），回答质量可能不佳。"

    return {
        "answer": answer,
        "messages": [f"[生成] 回答质量: {'高' if score >= 0.8 else '中' if score >= 0.5 else '低'}"],
    }


# ============================================================
# 3. 构建图 —— 线性 RAG 管道
# ============================================================

builder = StateGraph(RAGAgentState)
builder.add_node("query_rewrite", query_rewrite)
builder.add_node("retrieve_docs", retrieve_docs)
builder.add_node("evaluate_relevance", evaluate_relevance)
builder.add_node("generate_answer", generate_answer)

builder.add_edge(START, "query_rewrite")
builder.add_edge("query_rewrite", "retrieve_docs")
builder.add_edge("retrieve_docs", "evaluate_relevance")
builder.add_edge("evaluate_relevance", "generate_answer")
builder.add_edge("generate_answer", END)

memory = InMemorySaver()
graph = builder.compile(checkpointer=memory)


# ============================================================
# 4. 第一次执行 —— 发现问题
# ============================================================

print("=" * 70)
print("  阶段 1：首次执行 RAG Agent")
print("=" * 70)

config = {"configurable": {"thread_id": "debug-session-001"}}

initial_state = {
    "query": "LangGraph 状态快照怎么用",
    "rewritten_query": "",
    "retrieved_docs": [],
    "relevance_score": 0.0,
    "answer": "",
    "messages": [],
    "is_relevant": False,
}

result = graph.invoke(initial_state, config)

print(f"\n执行日志:")
for msg in result["messages"]:
    print(f"  {msg}")

print(f"\n最终回答: {result['answer']}")
print(f"相关度评分: {result['relevance_score']}")
print(f"文档足够相关: {result['is_relevant']}")
print(f"\n诊断: 回答质量低，需要调试检索步骤")


# ============================================================
# 5. Replay（重放）—— 回到检索步骤观察中间状态
# ============================================================

print()
print("=" * 70)
print("  阶段 2：Replay —— 重放到检索步骤")
print("=" * 70)

# 获取完整历史
all_states = list(graph.get_state_history(config))

print(f"\n完整执行历史（{len(all_states)} 个快照）:")
print(f"{'序号':<4} {'source':<8} {'step':<6} {'next':<25} {'说明'}")
print("-" * 75)

for i, state in enumerate(all_states):
    source = state.metadata.get("source", "?")
    step = state.metadata.get("step", "?")
    next_nodes = state.next
    writes = state.metadata.get("writes", {})
    node_name = list(writes.keys())[0] if writes else "initial"
    print(f"{i:<4} {source:<8} {step:<6} {str(next_nodes):<25} {node_name}")

# 找到 retrieve_docs 执行后的快照
retrieve_snapshot = None
for state in all_states:
    writes = state.metadata.get("writes", {})
    if "retrieve_docs" in writes:
        retrieve_snapshot = state
        break

print(f"\n定位到检索步骤快照:")
print(f"  checkpoint_id: {retrieve_snapshot.config['configurable']['checkpoint_id'][:16]}...")
print(f"  检索到的文档:")
for doc in retrieve_snapshot.values["retrieved_docs"]:
    print(f"    - {doc}")

# Replay：使用 checkpoint_id 重放到该步骤
# invoke(None, config_with_checkpoint_id) 会重放到该 checkpoint，然后继续执行
replay_config = {
    "configurable": {
        "thread_id": "debug-session-001",
        "checkpoint_id": retrieve_snapshot.config["configurable"]["checkpoint_id"],
    }
}

print(f"\n执行 Replay（从检索步骤重放）...")
replay_result = graph.invoke(None, config=replay_config)

print(f"\nReplay 结果:")
print(f"  回答: {replay_result['answer']}")
print(f"  说明: Replay 重放了检索之后的步骤（evaluate + generate）")
print(f"  注意: 因为状态没变，结果和原始执行一样")


# ============================================================
# 6. Fork（分叉）—— 修改检索结果后重新执行
# ============================================================

print()
print("=" * 70)
print("  阶段 3：Fork —— 修改检索结果并重新生成")
print("=" * 70)

# 用高质量文档替换原始检索结果
better_docs = [
    "LangGraph 状态快照：get_state() 返回 StateSnapshot，包含 values、next、config 等 8 个字段",
    "LangGraph 时间旅行：通过 get_state_history() 遍历历史，update_state() 修改状态创建分叉",
    "LangGraph Checkpoint：InMemorySaver 在每个 super-step 自动保存状态，支持 Replay 和 Fork",
]

print(f"准备替换的高质量文档:")
for doc in better_docs:
    print(f"  - {doc[:50]}...")

# Fork 操作：update_state 修改检索结果
fork_config = graph.update_state(
    retrieve_snapshot.config,
    values={
        "retrieved_docs": better_docs,
        "messages": ["[Fork] 手动替换为高质量文档"],
    },
    as_node="retrieve_docs",  # 以 retrieve_docs 的身份写入
)

print(f"\nFork 创建的新 checkpoint:")
print(f"  checkpoint_id: {fork_config['configurable']['checkpoint_id'][:16]}...")

# 验证修改后的状态
fork_snapshot = graph.get_state(fork_config)
print(f"  next: {fork_snapshot.next}")
print(f"  metadata.source: {fork_snapshot.metadata.get('source')}")

# 从分叉点继续执行
print(f"\n从分叉点继续执行（evaluate_relevance → generate_answer）...")
fork_result = graph.invoke(None, config=fork_config)

print(f"\nFork 执行结果:")
for msg in fork_result["messages"]:
    print(f"  {msg}")
print(f"\n最终回答: {fork_result['answer']}")
print(f"相关度评分: {fork_result['relevance_score']}")


# ============================================================
# 7. 对比原始路径和分叉路径
# ============================================================

print()
print("=" * 70)
print("  阶段 4：对比原始路径 vs 分叉路径")
print("=" * 70)

print(f"\n{'指标':<16} {'原始路径':<35} {'分叉路径'}")
print("-" * 85)
print(f"{'相关度评分':<16} {result['relevance_score']:<35} {fork_result['relevance_score']}")
print(f"{'文档相关':<16} {str(result['is_relevant']):<35} {str(fork_result['is_relevant'])}")
print(f"{'回答质量':<16} {'低':<35} {'高' if fork_result['relevance_score'] >= 0.8 else '中'}")
print(f"{'文档数量':<16} {len(result['retrieved_docs']):<35} {len(fork_result['retrieved_docs'])}")

print(f"\n原始回答: {result['answer'][:60]}...")
print(f"分叉回答: {fork_result['answer'][:60]}...")


# ============================================================
# 8. 查看分叉后的完整历史
# ============================================================

print()
print("=" * 70)
print("  阶段 5：查看分叉后的完整历史")
print("=" * 70)

# 获取最新的完整历史（包含原始路径和分叉路径）
full_history = list(graph.get_state_history(config))

print(f"\n分叉后的完整历史（{len(full_history)} 个快照）:")
print(f"{'序号':<4} {'source':<8} {'step':<6} {'next':<30} {'说明'}")
print("-" * 80)

for i, state in enumerate(full_history):
    source = state.metadata.get("source", "?")
    step = state.metadata.get("step", "?")
    next_nodes = state.next
    writes = state.metadata.get("writes", {})
    node_name = list(writes.keys())[0] if writes else "initial"

    # 标记分叉点
    marker = ""
    if source == "update":
        marker = " ← Fork 点"
    elif i == 0:
        marker = " ← 最新（分叉路径终点）"

    print(f"{i:<4} {source:<8} {step:<6} {str(next_nodes):<30} {node_name}{marker}")


# ============================================================
# 9. 高级技巧：多次分叉对比
# ============================================================

print()
print("=" * 70)
print("  阶段 6：高级技巧 —— 多次分叉对比")
print("=" * 70)

# 从同一个检索步骤分叉出多个版本，对比不同文档的效果

# 版本 B：使用另一组文档
docs_version_b = [
    "LangGraph 入门：StateGraph 是核心类，用于定义有状态的图",
    "LangGraph 节点：每个节点是一个函数，接收状态返回部分更新",
]

fork_b_config = graph.update_state(
    retrieve_snapshot.config,
    values={
        "retrieved_docs": docs_version_b,
        "messages": ["[Fork-B] 使用简化文档集"],
    },
    as_node="retrieve_docs",
)

fork_b_result = graph.invoke(None, config=fork_b_config)

# 版本 C：使用单篇高质量文档
docs_version_c = [
    "LangGraph 状态快照完全指南：StateSnapshot 包含 values/next/config/metadata/created_at/parent_config/tasks/interrupts 八个字段，通过 get_state() 获取当前快照，get_state_history() 遍历历史，update_state() 修改状态创建分叉，支持 Replay 重放和 Fork 分叉两种时间旅行操作",
]

fork_c_config = graph.update_state(
    retrieve_snapshot.config,
    values={
        "retrieved_docs": docs_version_c,
        "messages": ["[Fork-C] 使用单篇详细文档"],
    },
    as_node="retrieve_docs",
)

fork_c_result = graph.invoke(None, config=fork_c_config)

print(f"\n多版本对比:")
print(f"{'版本':<10} {'文档数':<8} {'相关度':<10} {'回答质量'}")
print("-" * 50)
print(f"{'原始':<10} {len(result['retrieved_docs']):<8} {result['relevance_score']:<10} 低")
print(f"{'Fork-A':<10} {len(fork_result['retrieved_docs']):<8} {fork_result['relevance_score']:<10} 高")
print(f"{'Fork-B':<10} {len(fork_b_result['retrieved_docs']):<8} {fork_b_result['relevance_score']:<10} {'高' if fork_b_result['relevance_score'] >= 0.8 else '中' if fork_b_result['relevance_score'] >= 0.5 else '低'}")
print(f"{'Fork-C':<10} {len(fork_c_result['retrieved_docs']):<8} {fork_c_result['relevance_score']:<10} {'高' if fork_c_result['relevance_score'] >= 0.8 else '中' if fork_c_result['relevance_score'] >= 0.5 else '低'}")

print(f"\n说明: 所有分叉都基于同一个检索步骤的快照，")
print(f"只是替换了不同的文档集，然后各自独立执行后续步骤。")
print(f"这就是时间旅行的核心价值——在不重新执行整个管道的情况下，")
print(f"快速对比不同方案的效果。")


# ============================================================
# 10. 调试工作流总结
# ============================================================

print()
print("=" * 70)
print("  调试工作流总结")
print("=" * 70)

print("""
完整的时间旅行调试流程：

  1. 执行图，发现结果不理想
     └─ graph.invoke(input, config)

  2. 获取历史，定位问题步骤
     └─ states = list(graph.get_state_history(config))
     └─ 遍历 states，通过 writes/next/step 定位目标快照

  3. Replay（重放）—— 观察中间状态
     └─ config_with_cp = {"configurable": {"thread_id": "...", "checkpoint_id": "..."}}
     └─ graph.invoke(None, config=config_with_cp)
     └─ 重放不会改变状态，只是重新执行后续步骤

  4. Fork（分叉）—— 修改状态并重新执行
     └─ new_config = graph.update_state(target_config, values={...}, as_node="...")
     └─ graph.invoke(None, config=new_config)
     └─ 从修改后的状态继续执行，得到新的结果

  5. 对比分析
     └─ 对比原始路径和分叉路径的结果
     └─ 可以多次分叉，对比不同方案

关键 API:
  - graph.get_state(config)          → 获取快照
  - graph.get_state_history(config)  → 遍历历史
  - graph.update_state(config, ...)  → 修改状态（创建分叉）
  - graph.invoke(None, config)       → 从 checkpoint 恢复执行
""")
```

```text
     └─ graph.invoke(input, config)

  2. 获取历史，定位问题步骤
     └─ states = list(graph.get_state_history(config))

  3. Replay（重放）—— 观察中间状态
     └─ graph.invoke(None, config=config_with_checkpoint_id)

  4. Fork（分叉）—— 修改状态并重新执行
     └─ graph.update_state(...) + graph.invoke(None, ...)

  5. 对比分析
```

---

## 关键知识点解析

### 1. Replay vs Fork 对比

| 特性 | Replay（重放） | Fork（分叉） |
|------|---------------|-------------|
| 操作 | `invoke(None, config_with_cp_id)` | `update_state()` + `invoke(None)` |
| 是否修改状态 | 不修改 | 修改 |
| 用途 | 观察中间状态，重新执行后续步骤 | 修改中间状态，探索替代路径 |
| 新 checkpoint | 会创建新的（重新执行的步骤） | 会创建新的（修改 + 重新执行） |
| 结果 | 与原始执行相同（确定性节点） | 可能不同（取决于修改内容） |

### 2. 时间旅行的三个核心 API

```python
# 1. 获取历史 → 找到目标快照
states = list(graph.get_state_history(config))
target = states[N]  # 选择目标快照

# 2. Replay → 从目标快照重放
replay_config = {
    "configurable": {
        "thread_id": "...",
        "checkpoint_id": target.config["configurable"]["checkpoint_id"],
    }
}
graph.invoke(None, config=replay_config)

# 3. Fork → 修改状态后继续执行
fork_config = graph.update_state(
    target.config,
    values={"key": "new_value"},
    as_node="node_name",
)
graph.invoke(None, config=fork_config)
```

### 3. invoke(None) 的含义

当 `invoke` 的第一个参数为 `None` 时，表示不提供新输入，而是从 config 中指定的 checkpoint 恢复执行：

```python
# 提供输入 → 从头开始执行
graph.invoke({"query": "..."}, config)

# 不提供输入 → 从 checkpoint 恢复执行
graph.invoke(None, config=config_with_checkpoint_id)
```

### 4. 调试场景决策树

```
发现问题
  ├─ 想看中间状态？
  │   └─ get_state_history() → 遍历查看
  │
  ├─ 想重新执行后续步骤？
  │   └─ Replay: invoke(None, config_with_cp_id)
  │
  ├─ 想修改中间状态后重新执行？
  │   └─ Fork: update_state() → invoke(None)
  │
  └─ 想对比多种方案？
      └─ 多次 Fork: 从同一个快照分叉多次
```

## 在 RAG 系统中的应用

时间旅行调试在 RAG 系统中的典型场景：

- **检索质量调试**：发现回答不好时，回到检索步骤查看返回了哪些文档，手动替换后重新生成
- **Prompt 调优**：回到生成步骤之前，修改 prompt 模板或参数，对比不同 prompt 的效果
- **错误复现**：通过 checkpoint_id 精确回到出错的状态，复现并分析问题
- **A/B 测试**：从同一个检索结果分叉，用不同的生成策略（temperature、top_k 等）对比效果
- **人工审核**：审核员查看检索结果，不满意时手动修正后继续执行
