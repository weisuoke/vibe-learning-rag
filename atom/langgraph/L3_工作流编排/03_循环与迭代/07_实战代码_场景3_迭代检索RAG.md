## 场景描述

**场景**：构建一个迭代检索 RAG 系统。第一次检索可能不够精准，系统会评估检索结果质量，如果质量不够则改写查询重新检索，直到质量达标或达到最大迭代次数。

**为什么需要迭代检索？**

单次检索的局限性：
- 用户的原始问题可能表述模糊
- 第一次检索的关键词可能不够精准
- 向量相似度高不等于语义真正相关

通过"检索 → 评估 → 改写 → 再检索"的循环，系统可以逐步逼近最佳检索结果。

**特点**：
- 检索-评估-改写的三阶段循环
- `quality_score` 作为业务指标驱动循环终止
- `query_history` 避免重复查询（语义缓存思想）
- 不依赖外部服务，使用模拟数据
- 与 RAG 系统紧密结合

**图结构**：

```
START → retrieve → evaluate → [质量够?] → generate → END
                     ↑            |
                     └── rewrite ←┘
```

[来源: reference/fetch_循环优化_01.md - 语义缓存与状态振荡检测]

## 完整代码

```python
"""
LangGraph 实战代码 - 场景3：迭代检索 RAG
演示：检索-评估-再检索的质量驱动循环

图结构：
  START → retrieve → evaluate → [质量够?] → generate → END
                       ↑            |
                       └── rewrite ←┘

来源：
  - LangGraph 官方文档: 条件边与循环
  - rajatpandit.com: 语义缓存与状态振荡检测
  - ActiveWizards: 质量门控循环模式
"""

from __future__ import annotations

import operator
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END


# ============================================================
# 1. 模拟知识库
# ============================================================
# 模拟一个简单的文档知识库，用于演示检索逻辑
# 实际项目中这里是向量数据库（ChromaDB、Milvus、FAISS 等）

KNOWLEDGE_BASE = {
    "langgraph": [
        "LangGraph 是基于 LangChain 的图编排框架，支持循环和条件分支。",
        "LangGraph 使用 StateGraph 定义有状态的工作流，节点是函数，边是控制流。",
        "LangGraph 的核心优势是支持循环（cycles），这是普通 DAG 框架做不到的。",
    ],
    "循环": [
        "LangGraph 中的循环通过条件边（conditional edges）实现，路由函数决定下一步。",
        "循环的安全机制包括 recursion_limit 和 RemainingSteps 管理值。",
        "防止无限循环的最佳实践是在状态中维护计数器，配合条件边做断路器。",
    ],
    "状态管理": [
        "LangGraph 的状态使用 TypedDict 定义，支持 Annotated 字段和 Reducer 函数。",
        "operator.add 是最常用的 Reducer，用于列表累积。",
    ],
    "RAG": [
        "RAG（Retrieval-Augmented Generation）结合检索和生成，提升 LLM 回答质量。",
        "RAG 的核心流程：文档加载 → 分块 → 向量化 → 检索 → 生成。",
        "迭代检索 RAG 通过多轮检索逐步提升检索质量，是高级 RAG 模式。",
    ],
    "向量检索": [
        "向量检索通过计算 Embedding 之间的相似度找到最相关的文档。",
        "常用的相似度度量包括余弦相似度、L2 距离和内积。",
    ],
}


def simulate_retrieval(query: str) -> list[str]:
    """模拟向量检索（实际项目中调用向量数据库 search 接口）"""
    results = []
    query_lower = query.lower()

    for topic, docs in KNOWLEDGE_BASE.items():
        if topic in query_lower or any(kw in query_lower for kw in topic.split()):
            results.extend(docs)

    if not results:
        results = KNOWLEDGE_BASE.get("RAG", [])[:1]

    return results[:5]


# ============================================================
# 2. 状态定义
# ============================================================
# [来源: context7_langgraph_01.md - State 使用 TypedDict 定义]

class RAGState(TypedDict):
    """迭代检索 RAG 的状态定义"""
    question: str                                          # 用户原始问题
    query: str                                             # 当前检索查询
    documents: list[str]                                   # 检索到的文档
    answer: str                                            # 生成的答案
    quality_score: float                                   # 检索质量评分 0-1
    iteration: int                                         # 当前迭代次数
    query_history: Annotated[list[str], operator.add]      # 查询历史（累积）
    best_documents: list[str]                              # 历史最佳文档集


# 最大迭代次数
MAX_ITERATIONS = 3

# 质量阈值
QUALITY_THRESHOLD = 0.7


# ============================================================
# 3. 检索节点
# ============================================================

def retrieve_node(state: RAGState) -> dict:
    """
    检索节点：根据当前 query 检索文档。

    核心逻辑：
    - 使用当前 query 进行检索
    - 记录查询到 query_history
    """
    query = state["query"]
    iteration = state["iteration"]

    print(f"\n{'='*50}")
    print(f"[检索] 第 {iteration + 1} 次检索")
    print(f"[检索] 查询: {query}")
    print(f"{'='*50}")

    # 检查是否重复查询（语义缓存思想）
    # [来源: fetch_循环优化_01.md - Semantic Cache Divergence Check]
    if query in state.get("query_history", []):
        print(f"[检索] 警告：重复查询 '{query}'，跳过检索使用缓存结果")
        return {
            "query_history": [query],
            "iteration": iteration + 1,
        }

    # 执行检索
    documents = simulate_retrieval(query)

    print(f"[检索] 找到 {len(documents)} 个文档片段")
    for i, doc in enumerate(documents):
        print(f"  [{i+1}] {doc[:60]}...")

    return {
        "documents": documents,
        "query_history": [query],
        "iteration": iteration + 1,
    }


# ============================================================
# 4. 评估节点
# ============================================================

def evaluate_node(state: RAGState) -> dict:
    """评估节点：评估检索结果与问题的相关性。"""
    question = state["question"]
    documents = state["documents"]
    iteration = state["iteration"]

    print(f"\n{'='*50}")
    print(f"[评估] 评估第 {iteration} 次检索结果")

    score = _calculate_relevance_score(question, documents)
    print(f"[评估] 质量评分: {score:.2f} | 阈值: {QUALITY_THRESHOLD} | {'达标' if score >= QUALITY_THRESHOLD else '不达标'}")

    best_docs = state.get("best_documents", [])
    if score > state.get("quality_score", 0):
        best_docs = documents

    return {"quality_score": score, "best_documents": best_docs}


def _calculate_relevance_score(question: str, documents: list[str]) -> float:
    """计算检索结果的相关性评分（实际项目中用 Cross-encoder 或 LLM 评估）"""
    if not documents:
        return 0.0

    score = 0.0

    # 维度1：文档数量
    score += min(len(documents) / 3, 1.0) * 0.3

    # 维度2：关键词覆盖率
    question_keywords = set(question.lower().replace("？", "").replace("?", "").split())
    stop_words = {"的", "是", "在", "了", "和", "与", "如何", "什么", "怎么", "中"}
    question_keywords -= stop_words

    if question_keywords:
        all_doc_text = " ".join(documents).lower()
        covered = sum(1 for kw in question_keywords if kw in all_doc_text)
        score += (covered / len(question_keywords)) * 0.5

    # 维度3：文档多样性
    unique_topics = set()
    for doc in documents:
        for topic in KNOWLEDGE_BASE:
            if topic in doc.lower() or topic in doc:
                unique_topics.add(topic)
    score += min(len(unique_topics) / 2, 1.0) * 0.2

    return min(score, 1.0)


# ============================================================
# 5. 查询改写节点
# ============================================================

def rewrite_query_node(state: RAGState) -> dict:
    """查询改写节点：基于评估结果和历史查询改写检索查询。"""
    question = state["question"]
    current_query = state["query"]
    query_history = state.get("query_history", [])
    iteration = state["iteration"]

    print(f"\n{'='*50}")
    print(f"[改写] 原始: {current_query} | 历史: {query_history}")
    print(f"{'='*50}")

    new_query = _rewrite_query(question, current_query, query_history, iteration)
    print(f"[改写] 新查询: {new_query}")

    return {"query": new_query}


def _rewrite_query(
    question: str, current_query: str, history: list[str], iteration: int,
) -> str:
    """模拟查询改写（实际项目中用 LLM 或 HyDE 策略）"""
    rewrite_strategies = [
        lambda q, _: " ".join([
            w for w in q.replace("？", "").replace("?", "").split()
            if w not in {"的", "是", "在", "了", "和", "与", "如何", "什么", "怎么", "中", "请", "介绍"}
        ]),
        lambda q, curr: f"{curr} 循环 迭代 条件边",
        lambda q, curr: f"LangGraph 循环 状态管理 RAG",
    ]

    strategy_idx = min(iteration - 1, len(rewrite_strategies) - 1)
    new_query = rewrite_strategies[strategy_idx](question, current_query)

    if new_query in history:
        new_query = f"{new_query} 详细"

    return new_query


# ============================================================
# 6. 答案生成节点
# ============================================================

def generate_answer_node(state: RAGState) -> dict:
    """答案生成节点：基于最佳文档集生成最终答案。"""
    question = state["question"]
    documents = state.get("best_documents", state["documents"])
    iteration = state["iteration"]
    quality = state["quality_score"]

    print(f"\n{'='*50}")
    print(f"[生成] 使用 {len(documents)} 个文档 | 质量: {quality:.2f} | 迭代: {iteration}")

    answer = _generate_answer(question, documents)
    print(f"[生成] 答案: {answer[:80]}...")

    return {"answer": answer}


def _generate_answer(question: str, documents: list[str]) -> str:
    """模拟基于文档的答案生成（实际项目中调用 LLM）"""
    if not documents:
        return f"抱歉，未能找到与 '{question}' 相关的信息。"

    context = "\n".join(f"- {doc}" for doc in documents)
    return (
        f"根据检索到的 {len(documents)} 个相关文档：\n{context}\n\n"
        f"综合以上信息，{question.rstrip('？?')}的要点包括：\n"
        f"1. {documents[0][:50]}...\n"
        f"2. {documents[1][:50] if len(documents) > 1 else '（更多细节请参考文档）'}..."
    )


# ============================================================
# 7. 路由函数 — 循环控制
# ============================================================
# [来源: source_循环迭代_01.md - _branch.py L146 路由执行]
# [来源: fetch_循环优化_01.md - 严格单调条件]

def route_after_evaluate(
    state: RAGState,
) -> Literal["rewrite_query", "generate_answer"]:
    """
    路由函数：质量达标或达到最大迭代次数 → 生成答案，否则 → 改写查询。

    关键原则："Every cycle must rely on a strictly monotonic condition."
    [来源: fetch_循环优化_01.md]
    """
    quality = state["quality_score"]
    iteration = state["iteration"]

    # 条件1：质量达标
    if quality >= QUALITY_THRESHOLD:
        print(f"\n[路由] 检索质量达标 ({quality:.2f} >= {QUALITY_THRESHOLD})，生成答案")
        return "generate_answer"

    # 条件2：达到最大迭代次数
    if iteration >= MAX_ITERATIONS:
        print(f"\n[路由] 达到最大迭代次数 {MAX_ITERATIONS}，使用当前最佳结果")
        print(f"[路由] 当前最佳评分: {quality:.2f}")
        return "generate_answer"

    # 默认：继续迭代
    print(f"\n[路由] 质量不足 ({quality:.2f} < {QUALITY_THRESHOLD})，改写查询继续检索...")
    return "rewrite_query"


# ============================================================
# 8. 构建图
# ============================================================
# [来源: context7_langgraph_01.md - StateGraph 构建模式]

def build_iterative_rag_graph() -> StateGraph:
    """构建迭代检索 RAG 图"""

    builder = StateGraph(RAGState)

    # 添加节点
    builder.add_node("retrieve", retrieve_node)
    builder.add_node("evaluate", evaluate_node)
    builder.add_node("rewrite_query", rewrite_query_node)
    builder.add_node("generate_answer", generate_answer_node)

    # 定义边
    builder.add_edge(START, "retrieve")
    builder.add_edge("retrieve", "evaluate")

    # 评估后的条件路由：质量够 → 生成，不够 → 改写
    builder.add_conditional_edges("evaluate", route_after_evaluate)

    # 改写后重新检索（形成循环）
    builder.add_edge("rewrite_query", "retrieve")

    # 生成答案后结束
    builder.add_edge("generate_answer", END)

    return builder.compile()


# ============================================================
# 9. 运行示例
# ============================================================

def main():
    """运行迭代检索 RAG 示例"""

    print("=" * 60)
    print("  迭代检索 RAG 演示")
    print("=" * 60)

    graph = build_iterative_rag_graph()

    # 测试问题：初始查询比较模糊，需要迭代改进
    question = "LangGraph 中如何实现循环？"

    result = graph.invoke({
        "question": question,
        "query": question,          # 初始查询 = 原始问题
        "documents": [],
        "answer": "",
        "quality_score": 0.0,
        "iteration": 0,
        "query_history": [],
        "best_documents": [],
    })

    # 输出最终结果
    print("\n" + "=" * 60)
    print("  最终结果")
    print("=" * 60)
    print(f"原始问题: {result['question']}")
    print(f"迭代次数: {result['iteration']}")
    print(f"最终评分: {result['quality_score']:.2f}")
    print(f"查询历史:")
    for i, q in enumerate(result["query_history"]):
        print(f"  [{i+1}] {q}")
    print(f"\n最终答案:\n{result['answer']}")


if __name__ == "__main__":
    main()
```

## 关键模式解析

### 模式1：query_history 避免重复查询

```python
class RAGState(TypedDict):
    query_history: Annotated[list[str], operator.add]  # 累积所有查询
```

在检索节点中检查是否重复：

```python
if query in state.get("query_history", []):
    print("警告：重复查询，跳过检索使用缓存结果")
    return {"query_history": [query], "iteration": iteration + 1}
```

这是 rajatpandit.com 提出的"语义缓存"思想的简化实现：

> "Has this Graph explicitly called this exact tool, with these exact arguments, within the last 3 turns? If yes, do not execute the call."

[来源: fetch_循环优化_01.md - Semantic Cache Divergence Check]

### 模式2：quality_score 作为业务指标驱动终止

```python
if quality >= QUALITY_THRESHOLD:
    return "generate_answer"  # 质量达标，停止迭代
```

与简单的计数器不同，`quality_score` 是一个有业务含义的指标。这遵循了生产环境的核心原则：

> "Every cycle in a production graph must rely on a strictly monotonic condition to continue. Something in your state must measurably increase or decrease with every turn."

[来源: fetch_循环优化_01.md - The Immutable Production Rule]

### 模式3：best_documents 保留历史最佳

```python
if score > state.get("quality_score", 0):
    best_docs = documents  # 更新最佳文档集
```

即使后续迭代的检索结果变差，系统也能使用历史最佳结果生成答案。这是优雅降级的关键。

## 一次检索 vs 迭代检索对比

| 维度 | 一次检索 | 迭代检索 |
|------|----------|----------|
| 检索质量 | 取决于初始查询 | 逐步优化，质量更高 |
| 延迟 | 低（1次检索） | 较高（2-3次检索） |
| 成本 | 低 | 中等（多次检索 + 评估） |
| 适用场景 | 查询明确、知识库质量高 | 查询模糊、需要高质量答案 |
| 容错性 | 低，首次失败无法恢复 | 高，可自动改进查询 |

**建议**：在生产环境中，可以先做一次检索并评估质量。如果质量已经达标，直接生成答案（快速路径）；如果不达标，再进入迭代循环（慢速路径）。

## RAG 应用联系

迭代检索是 RAG 系统中最实用的循环模式之一。常见变体：

- **Query Expansion**：检索 → 评估 → 扩展查询 → 再检索
- **Multi-hop QA**：第一次检索找到线索 → 基于线索生成新查询 → 检索更深层信息
- **Self-RAG**：生成答案 → 评估是否有幻觉 → 如果有则重新检索
- **Adaptive Retrieval**：根据问题复杂度动态决定检索轮数

在实际 RAG 系统中集成时，将模拟函数替换为：
- `simulate_retrieval` → ChromaDB/Milvus 的 `search()` 接口
- `_calculate_relevance_score` → Cross-encoder 重排序模型或 LLM 评估
- `_rewrite_query` → LLM 查询改写（如 HyDE 策略）
- `_generate_answer` → LLM 基于上下文的答案生成
