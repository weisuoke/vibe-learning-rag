# 实战代码 - 场景2：语义相似度选择（问答系统）

## 场景描述

在问答系统中，我们通常有一个包含大量问答对的示例库。当用户提出新问题时，我们希望：

1. **动态选择最相关的示例**：从示例库中选择与用户问题语义最相似的问答对
2. **提高回答质量**：通过相关示例帮助 LLM 理解问题类型和期望的回答风格
3. **减少无关干扰**：避免使用所有示例，只选择最相关的几个

**实际应用场景：**
- 技术文档问答系统
- 客服机器人
- 教育辅导系统
- 产品知识库问答

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

---

## 核心技术点

### 1. SemanticSimilarityExampleSelector

基于语义相似度动态选择示例的选择器：

```python
from langchain_core.example_selectors import SemanticSimilarityExampleSelector
from langchain_openai import OpenAIEmbeddings
from langchain_chroma import Chroma

selector = SemanticSimilarityExampleSelector.from_examples(
    examples=examples,              # 示例列表
    embeddings=OpenAIEmbeddings(),  # Embedding 模型
    vectorstore_cls=Chroma,         # 向量存储
    k=3                             # 返回 3 个最相似的示例
)
```

[来源: sourcecode/langchain/libs/core/langchain_core/example_selectors/semantic_similarity.py]

### 2. 工作流程

```
用户问题 → Embedding → 向量相似度搜索 → Top-K 示例 → 注入 Prompt → LLM 生成答案
```

### 3. 关键参数

- **k**: 返回的示例数量（推荐 3-5）
- **embeddings**: Embedding 模型（推荐 text-embedding-3-small）
- **vectorstore_cls**: 向量存储后端（Chroma、FAISS、Pinecone）

[来源: reference/search_example_selector_02.md]

## 代码解析

### 1. 示例库设计

```python
qa_examples = [
    {
        "question": "What is LangChain?",
        "answer": "LangChain is a framework..."
    },
    # 更多示例...
]
```

**设计要点：**
- 涵盖多个主题（LangChain、RAG、Vector DB 等）
- 每个示例包含 question 和 answer 字段
- 示例质量直接影响选择效果

[来源: reference/context7_langchain_01.md | LangChain 官方文档]

### 2. 选择器创建

```python
selector = SemanticSimilarityExampleSelector.from_examples(
    examples=qa_examples,
    embeddings=OpenAIEmbeddings(model="text-embedding-3-small"),
    vectorstore_cls=Chroma,
    k=3
)
```

**工作原理：**
1. 将所有示例转换为 embeddings
2. 存储到 Chroma 向量数据库
3. 用户问题转换为 embedding
4. 计算余弦相似度
5. 返回 top-3 最相似的示例

[来源: sourcecode/langchain/libs/core/langchain_core/example_selectors/semantic_similarity.py]

### 3. FewShotPromptTemplate 集成

```python
few_shot_prompt = FewShotPromptTemplate(
    example_selector=selector,  # 动态选择器
    example_prompt=example_prompt,
    prefix="You are a helpful technical assistant...",
    suffix="Question: {question}\nAnswer:",
    input_variables=["question"]
)
```

**Prompt 结构：**
```
prefix
+ 动态选择的示例1
+ 动态选择的示例2
+ 动态选择的示例3
+ suffix（包含用户问题）
```

### 4. 动态示例选择

```python
selected = selector.select_examples({"question": user_question})
```

**选择过程：**
- 输入：`{"question": "What is RAG?"}`
- 输出：3个最相关的问答对
- 自动排序：按相似度从高到低

### 2. k 值动态调整

```python
def dynamic_k_selector(question: str, base_k: int = 3) -> int:
    """根据问题复杂度动态调整 k 值"""
    question_length = len(question.split())

    if question_length < 5:
        return base_k + 2  # 简单问题需要更多示例
    elif question_length > 20:
        return base_k - 1  # 复杂问题减少示例避免混淆
    else:
        return base_k

# 使用
k = dynamic_k_selector(user_question)
selector.k = k
selected = selector.select_examples({"question": user_question})
```

### 3. 与 RAG 系统集成

## 常见问题

### Q1: 如何避免选择过于相似的示例？

**问题：** 选中的3个示例几乎相同，缺乏多样性

**解决方案：** 使用 MaxMarginalRelevanceExampleSelector

```python
from langchain_core.example_selectors import MaxMarginalRelevanceExampleSelector

mmr_selector = MaxMarginalRelevanceExampleSelector.from_examples(
    examples=qa_examples,
    embeddings=embeddings,
    vectorstore_cls=Chroma,
    k=3,
    fetch_k=20  # 先获取20个候选，再选择多样化的3个
)
```

[来源: sourcecode/langchain/libs/core/langchain_core/example_selectors/semantic_similarity.py]

### Q2: 如何控制 Token 成本？

**问题：** 示例太多导致 Token 成本过高

**解决方案：** 结合 LengthBasedExampleSelector

```python
# 先用语义相似度选择候选
semantic_selected = semantic_selector.select_examples({"question": user_q})

# 再用长度限制筛选
length_selector = LengthBasedExampleSelector(
    examples=semantic_selected,
    example_prompt=example_prompt,
    max_length=2000
)

final_examples = length_selector.select_examples({"question": user_q})
```
