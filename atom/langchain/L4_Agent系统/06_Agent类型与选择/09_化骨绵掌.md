## 要点1：Agent类型的本质

### 核心原理（50-100字）

**Agent类型 = LLM与工具的通信协议**

不同Agent类型的本质差异在于LLM如何表达"我要调用工具X，参数是Y"：
- **结构化输出**（OpenAI Functions）：模型直接输出JSON格式的函数调用
- **文本解析**（ReAct）：模型输出自然语言，通过正则表达式解析
- **混合方案**（Structured Chat）：结合结构化输出和灵活参数处理

### 实战应用（50-100字）

在RAG系统中，选择Agent类型直接影响检索工具的调用可靠性：
- 向量检索工具（简单参数）→ OpenAI Functions（最可靠）
- 混合检索工具（复杂过滤条件）→ Structured Chat（支持多参数）
- 开源模型场景 → ReAct（唯一选择）

### 记忆口诀

**"协议决定可靠性，结构化优于文本解析"**

## 要点2：三层决策树

### 核心原理（50-100字）

**Agent选择遵循三层决策逻辑**：

```
第一层：模型能力
├─ 支持函数调用？→ 进入第二层
└─ 不支持？→ ReAct（唯一选择）

第二层：工具复杂度
├─ 参数复杂（>3个或嵌套）？→ Structured Chat
└─ 参数简单？→ 进入第三层

第三层：优先级
├─ 生产环境 → OpenAI Functions
├─ 最新API → Tool Calling (create_agent)
└─ 调试需求 → ReAct
```

### 实战应用（50-100字）

**快速选择公式**：
1. 检查 `hasattr(llm, 'bind_tools')` → 判断函数调用支持
2. 统计工具参数数量 → 判断复杂度
3. 根据场景优先级 → 最终选择

生产环境RAG系统：OpenAI gpt-4 + 简单检索工具 → **OpenAI Functions Agent**

### 记忆口诀

**"能力-复杂度-优先级，三层决策不迷路"**

## 要点3：OpenAI Functions适用场景

### 核心原理（50-100字）

**OpenAI Functions Agent = 函数调用能力 + 简单工具**

核心优势：
- ✅ 可靠性最高（结构化输出，格式正确率99%）
- ✅ Token消耗最少（无需冗长的Thought-Action循环）
- ✅ 响应速度最快（1轮对话完成）

适用条件：
- 模型支持函数调用（OpenAI, Anthropic, Gemini）
- 工具参数简单（1-3个参数，类型简单）

### 实战应用（50-100字）

**典型场景**：
- 客服机器人（查询订单、物流、退款）
- 知识库问答（向量检索、文档查询）
- 代码助手（代码搜索、文档查询）

**代码模板**：
```python
from langchain.agents import create_openai_functions_agent
agent = create_openai_functions_agent(llm, tools, prompt)
```

### 记忆口诀

**"函数调用快又稳，简单工具首选它"**

## 要点4：ReAct适用场景

### 核心原理（50-100字）

**ReAct Agent = Thought-Action-Observation循环 + 任何LLM**

核心优势：
- ✅ 通用性最强（支持所有LLM，包括开源模型）
- ✅ 可解释性最好（推理过程完全透明）
- ✅ 调试最容易（verbose=True查看每步推理）

适用条件：
- 模型不支持函数调用（Llama, Mistral等开源模型）
- 需要查看推理过程（调试、审计、教学）

### 实战应用（50-100字）

**典型场景**：
- 开源模型应用（成本控制）
- 多步推理任务（需要查看中间步骤）
- 开发调试阶段（理解Agent行为）

**代码模板**：
```python
from langchain.agents import create_react_agent
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
```

### 记忆口诀

**"开源模型用ReAct，推理透明易调试"**

---

## 要点5：Structured Chat适用场景

### 核心原理（50-100字）

**Structured Chat Agent = 函数调用 + 复杂参数处理**

核心优势：
- ✅ 支持复杂工具（多个参数、嵌套结构、List/Dict）
- ✅ 格式强约束（减少参数错误）
- ✅ 适合API集成（结构化输入）

适用条件：
- 工具参数复杂（>3个参数或包含嵌套结构）
- 需要精确控制工具调用

### 实战应用（50-100字）

**典型场景**：
- 数据分析（数据库查询，多条件过滤）
- 法律文档分析（条款提取，多字段）
- 复杂API调用（多参数、嵌套配置）

**判断标准**：
```python
# 简单工具（用OpenAI Functions）
@tool
def search(query: str) -> str: ...

# 复杂工具（用Structured Chat）
@tool
def advanced_search(
    query: str,
    filters: List[str],
    date_range: dict,
    max_results: int
) -> str: ...
```

### 记忆口诀

**"参数复杂用Structured，多输入嵌套不怕它"**

## 要点6：create_agent()统一API

### 核心原理（50-100字）

**create_agent() = 智能路由器 + 自动选择**

本质：对现有`create_*_agent()`函数的统一封装，根据模型能力自动选择最佳实现。

```python
def create_agent(llm, tools):
    if hasattr(llm, 'bind_tools'):
        return create_openai_functions_agent(...)
    else:
        return create_react_agent(...)
```

2026年推荐使用，简化决策成本。

### 实战应用（50-100字）

**使用场景**：
- 快速原型开发（减少决策时间）
- 多模型兼容（自动适配不同模型）
- 新手友好（无需理解差异）

**代码模板**：
```python
from langchain.agents import create_agent
agent = create_agent(
    llm,
    tools,
    system_prompt="你是一个助手"
)
```

**注意**：生产环境建议明确使用`create_*_agent()`，获得更细粒度的控制。

### 记忆口诀

**"2026用create_agent，自动选择最省心"**

## 要点7：Agent类型迁移策略

### 核心原理（50-100字）

**Agent类型可以轻松切换，无需重写代码**

迁移步骤：
1. 保持工具定义不变（所有Agent类型共用）
2. 只更改Agent创建函数
3. 调整Prompt格式（如果需要）
4. 测试验证

**关键洞察**：Agent类型不是架构级决策，可以灵活调整。

### 实战应用（50-100字）

**迁移场景**：
- 从旧API迁移到新API（`initialize_agent` → `create_agent`）
- 从OpenAI迁移到开源模型（Functions → ReAct）
- 从简单工具升级到复杂工具（Functions → Structured Chat）

**最佳实践**：
```python
# 封装Agent创建逻辑
class AgentFactory:
    @staticmethod
    def create(agent_type, llm, tools, prompt):
        creators = {
            "openai_functions": create_openai_functions_agent,
            "react": create_react_agent,
            "structured_chat": create_structured_chat_agent,
        }
        return creators[agent_type](llm, tools, prompt)

# 切换只需改一个字符串
agent = AgentFactory.create("openai_functions", llm, tools, prompt)
```

### 记忆口诀

**"工具不变换创建，迁移切换很简单"**

## 要点8：故障排查方法

### 核心原理（50-100字）

**Agent故障的4大类型及解决方案**：

1. **不调用工具** → 改进工具描述，明确使用场景
2. **调用失败** → 检查参数类型，添加错误处理
3. **模型不支持** → 切换到ReAct Agent
4. **无限循环** → 设置`max_iterations`，改进工具返回格式

**调试利器**：`verbose=True`（性能影响<1%，但信息价值巨大）

### 实战应用（50-100字）

**排查流程**：
```python
# 1. 启用verbose查看推理过程
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,  # 关键！
    handle_parsing_errors=True
)

# 2. 设置最大迭代防止无限循环
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=5,
    early_stopping_method="generate"
)

# 3. 测试工具独立运行
result = my_tool.invoke("测试输入")
print(result)  # 确保工具本身可以运行
```

**常见错误速查**：
- 不调用工具 → 工具描述不清晰
- 调用失败 → 参数类型错误
- 不支持函数调用 → 切换到ReAct
- 无限循环 → 没有停止条件

### 记忆口诀

**"verbose开启看推理，max_iterations防死循环"**

## 2026最佳实践速查

### 快速开始（新手）

```python
from langchain.agents import create_agent
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4")
agent = create_agent(llm, tools, system_prompt="你是一个助手")
```

**优点**：最简单，自动选择
**缺点**：失去细粒度控制

---

### 生产环境（推荐）

```python
from langchain.agents import create_openai_functions_agent, AgentExecutor

agent = create_openai_functions_agent(llm, tools, prompt)
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=5,
    handle_parsing_errors=True,
    verbose=True  # 生产环境也建议开启
)
```

**优点**：最可靠，完全控制
**缺点**：需要手动选择类型

---

### 开源模型（成本控制）

```python
from langchain.agents import create_react_agent
from langchain_community.llms import Ollama

llm = Ollama(model="llama3")
agent = create_react_agent(llm, tools, prompt)
```

**优点**：成本最低，兼容性最好
**缺点**：可靠性略低，token消耗高

---

### 复杂工具（数据分析）

```python
from langchain.agents import create_structured_chat_agent

agent = create_structured_chat_agent(llm, complex_tools, prompt)
```

**优点**：支持复杂参数
**缺点**：性能略低于OpenAI Functions
