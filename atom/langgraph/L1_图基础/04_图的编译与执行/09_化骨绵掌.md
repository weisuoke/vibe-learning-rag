## 卡片1：compile() 方法 - 声明式到命令式的转换器

**一句话：** compile() 将声明式的 StateGraph 定义转换为可执行的 CompiledStateGraph 对象。

**核心要点**：
- **输入**：StateGraph（声明式定义）
- **输出**：CompiledStateGraph（可执行对象）
- **作用**：验证图结构、配置运行时特性、生成执行引擎

**代码示例**：
```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    message: str

def node_func(state: State):
    return {"message": "processed"}

# 构建图（声明式）
builder = StateGraph(State)
builder.add_node("process", node_func)
builder.add_edge(START, "process")
builder.add_edge("process", END)

# 编译（转换为可执行对象）
app = builder.compile()
```

**应用**：编译是使用 LangGraph 的第一步，必须先编译才能执行。

**记忆口诀**：声明式定义 → compile() → 可执行对象

## 卡片2：invoke() 方法 - 内部调用 stream() 的便捷封装

**一句话：** invoke() 内部调用 stream()，遍历所有输出并返回最后一个值。

**核心要点**：
- **本质**：invoke() 是 stream() 的语法糖
- **实现**：遍历 stream() 的所有输出，返回最后一个
- **性能**：与 stream() 完全相同，没有性能差异

**源码证据**：
```python
# pregel/main.py:3024
def invoke(self, input, config=None, **kwargs):
    latest = None
    for chunk in self.stream(input, config, **kwargs):
        latest = chunk
    return latest
```

**对比**：
```python
# 方式1：使用 invoke
result = app.invoke({"message": "hello"})

# 方式2：使用 stream（等价）
chunks = []
for chunk in app.stream({"message": "hello"}):
    chunks.append(chunk)
result = chunks[-1]  # 取最后一个
```

**应用**：只需要最终结果时使用 invoke，需要观察过程时使用 stream。

**记忆口诀**：invoke = stream + 取最后一个

## 卡片3：Pregel 算法 - 批量同步并行执行引擎

**一句话：** Pregel 是基于 BSP 模型的批量同步并行执行引擎，将执行组织成超步（supersteps）。

**核心要点**：
- **BSP 模型**：Bulk Synchronous Parallel（批量同步并行）
- **三个阶段**：Plan（选择节点）→ Execution（并行执行）→ Update（更新状态）
- **超步迭代**：重复执行直到满足终止条件

**执行流程**：
```
超步1：
  Plan: 选择可执行的节点（node_a, node_b）
  Execution: 并行执行 node_a 和 node_b
  Update: 更新状态

超步2：
  Plan: 选择下一批节点（node_c）
  Execution: 执行 node_c
  Update: 更新状态

...重复直到结束
```

**关键特点**：
- 同一超步内的节点并行执行
- 超步之间串行执行
- 依赖关系决定超步划分

**应用**：理解 Pregel 算法有助于优化图结构，减少依赖关系，提高并行度。

**记忆口诀**：Plan → Execution → Update → 重复

## 卡片4：Checkpointer - 图的"短期记忆"

**一句话：** Checkpointer 是图的持久化机制，保存执行状态，支持断点续传。

**核心要点**：
- **作用**：保存图的执行状态
- **时机**：每个节点执行后自动保存
- **内容**：当前状态、执行位置、中断信息

**常用实现**：
```python
from langgraph.checkpoint.memory import MemorySaver
from langgraph.checkpoint.sqlite import SqliteSaver

# 开发环境：内存持久化
checkpointer = MemorySaver()

# 生产环境：SQLite 持久化
import sqlite3
checkpointer = SqliteSaver(sqlite3.connect("checkpoints.db"))
```

**配置方式**：
```python
# 编译时配置
app = builder.compile(checkpointer=checkpointer)

# 执行时提供 thread_id
config = {"configurable": {"thread_id": "task-001"}}
result = app.invoke(input_data, config=config)
```

**应用**：长时间运行任务、需要人工审批的流程、可恢复的服务。

**记忆口诀**：Checkpointer = 游戏存档系统

## 卡片5：thread_id - 状态的唯一标识

**一句话：** thread_id 是 Checkpointer 的键，用于标识和恢复不同的执行上下文。

**核心要点**：
- **作用**：作为 Checkpointer 的键，标识不同的执行
- **必需性**：没有 thread_id 就无法持久化
- **隔离性**：不同 thread_id 完全隔离

**关系**：
```
Checkpointer（存储机制） + thread_id（状态标识） = 状态持久化
```

**使用示例**：
```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = builder.compile(checkpointer=checkpointer)

# 不同用户使用不同 thread_id
config1 = {"configurable": {"thread_id": "user-1"}}
config2 = {"configurable": {"thread_id": "user-2"}}

app.invoke({"input": "hello"}, config=config1)  # 用户1的状态
app.invoke({"input": "world"}, config=config2)  # 用户2的状态

# 恢复用户1的状态
app.invoke(None, config=config1)
```

**应用**：多用户系统、会话管理、任务追踪。

**记忆口诀**：thread_id = 存档槽位编号

## 卡片6：中断与恢复 - 暂停 + 等待 + 继续

**一句话：** 中断机制允许图在特定节点暂停执行，等待外部输入后继续。

**核心要点**：
- **静态中断**：编译时配置（interrupt_before/after）
- **动态中断**：使用 interrupt() 函数
- **恢复方式**：使用相同 thread_id 重新调用 invoke()

**静态中断示例**：
```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["approval_node"]  # 在审批节点前中断
)

config = {"configurable": {"thread_id": "flow-1"}}

# 执行到中断点
result1 = app.invoke(input_data, config=config)

# 恢复执行（传递 None）
result2 = app.invoke(None, config=config)
```

**动态中断示例**：
```python
from langgraph.types import interrupt, Command

def approval_node(state):
    decision = interrupt({"question": "Approve?"})
    return Command(goto="proceed" if decision else "cancel")

# 恢复时提供决策
result = app.invoke(Command(resume=True), config=config)
```

**应用**：人工审批、用户输入、外部事件等待。

**记忆口诀**：中断 = 红灯停车，恢复 = 绿灯继续

## 卡片7：流式执行 - 实时观察执行过程

**一句话：** stream() 方法流式输出图的执行过程，每个节点执行后输出一次。

**核心要点**：
- **stream_mode="updates"**：输出每个节点的更新
- **stream_mode="values"**：输出每个步骤后的完整状态
- **用途**：实时监控、调试、日志记录

**使用示例**：
```python
# 流式执行
for chunk in app.stream(input_data):
    print(chunk)

# 输出示例（stream_mode="updates"）：
# {'node_a': {'count': 1}}
# {'node_b': {'count': 2}}
# {'node_c': {'count': 3}}
```

**对比两种模式**：
```python
# updates 模式：只输出更新
for chunk in app.stream(input_data, stream_mode="updates"):
    print(chunk)  # {'node_name': {...}}

# values 模式：输出完整状态
for chunk in app.stream(input_data, stream_mode="values"):
    print(chunk)  # {'count': 1, 'message': '...'}
```

**应用**：Web 应用显示进度、调试观察节点输出、日志记录。

**记忆口诀**：stream = 实时播报执行过程

## 卡片8：持久化模式 - 可靠性 vs 性能的权衡

**一句话：** durability 参数控制何时将状态保存到 Checkpointer，平衡可靠性和性能。

**核心要点**：
- **sync**：每步执行后立即保存（最可靠，最慢）
- **async**：并行保存，不阻塞执行（默认，平衡）
- **exit**：仅在图退出时保存（最快，最不可靠）

**对比表**：
| 模式 | 持久化时机 | 可靠性 | 性能 | 适用场景 |
|------|-----------|--------|------|----------|
| `sync` | 每步立即保存 | 最高 | 最低 | 金融交易 |
| `async` | 并行保存 | 中等 | 中等 | 一般应用 |
| `exit` | 退出时保存 | 最低 | 最高 | 批量任务 |

**使用示例**：
```python
# 关键任务：同步持久化
app.invoke(input_data, config=config, durability="sync")

# 一般任务：异步持久化（默认）
app.invoke(input_data, config=config, durability="async")

# 高吞吐任务：退出时持久化
app.invoke(input_data, config=config, durability="exit")
```

**应用**：根据任务的重要性和性能要求选择合适的模式。

**记忆口诀**：sync = 最安全，async = 平衡，exit = 最快

## 卡片9：编译时 vs 运行时配置 - 固定 vs 动态

**一句话：** 编译时配置在编译后固定，运行时配置每次执行可以不同。

**核心要点**：
- **编译时配置**：checkpointer、interrupt_before/after、cache、store
- **运行时配置**：config（thread_id）、context、durability
- **区别**：编译时配置一次性绑定，运行时配置可动态调整

**对比示例**：
```python
# 编译时配置（固定）
app = builder.compile(
    checkpointer=checkpointer,      # 固定持久化机制
    interrupt_before=["approval"],  # 固定中断点
)

# 运行时配置（动态）
config1 = {"configurable": {"thread_id": "user-1"}}
config2 = {"configurable": {"thread_id": "user-2"}}

app.invoke(input1, config=config1, durability="sync")   # 用户1，同步
app.invoke(input2, config=config2, durability="async")  # 用户2，异步
```

**应用场景**：
- **编译时**：配置图的结构和固定特性
- **运行时**：配置每次执行的具体参数

**记忆口诀**：编译时 = 图的DNA，运行时 = 执行的参数

## 卡片10：实际应用场景 - 从理论到实践

**一句话：** 将编译与执行的知识应用到实际场景，构建生产级工作流。

**场景1：审批工作流**
```python
from langgraph.checkpoint.postgres import PostgresSaver

# 生产环境配置
checkpointer = PostgresSaver(connection_string="...")
app = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["approval"]
)

# 用户提交请求
config = {"configurable": {"thread_id": f"request-{request_id}"}}
result = app.invoke(request_data, config=config)

# 管理员审批后恢复
result = app.invoke(None, config=config)
```

**场景2：数据处理管道**
```python
# 配置断点续传
app = builder.compile(checkpointer=checkpointer)

# 处理大量数据
config = {"configurable": {"thread_id": f"batch-{batch_id}"}}
try:
    result = app.invoke(batch_data, config=config, durability="async")
except Exception as e:
    # 错误后可以恢复
    result = app.invoke(None, config=config)
```

**场景3：实时监控系统**
```python
# 流式输出进度
for chunk in app.stream(input_data, stream_mode="updates"):
    node_name = list(chunk.keys())[0]
    print(f"正在执行: {node_name}")
    # 发送到前端显示进度
```

**最佳实践**：
1. 开发环境使用 MemorySaver，生产环境使用 PostgresSaver
2. 关键任务使用 durability="sync"，一般任务使用默认的 async
3. 使用有意义的 thread_id，便于追踪和调试
4. 流式执行用于实时反馈，invoke 用于获取最终结果

**记忆口诀**：理论 + 实践 = 掌握

### 5个核心概念
1. **compile()**：声明式 → 可执行对象
2. **invoke()**：内部调用 stream()
3. **Pregel**：Plan → Execution → Update
4. **Checkpointer + thread_id**：持久化机制
5. **中断与恢复**：暂停 + 等待 + 继续

### 3个关键配置
1. **编译时**：checkpointer、interrupt_before/after
2. **运行时**：config（thread_id）、durability
3. **流模式**：updates（更新）、values（完整状态）

### 1个核心公式
```
StateGraph（声明式）
  ↓ compile()
CompiledStateGraph（可执行）
  ↓ invoke() / stream()
Result（结果）
```
