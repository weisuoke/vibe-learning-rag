## 概念介绍

### 什么是开发工具链？

**开发工具链**：一组协同工作的工具，用于提升开发效率、代码质量和团队协作。

**OpenClaw 的工具链**：
- **VS Code**：主要开发环境
- **oxlint/oxfmt**：代码质量工具
- **Git hooks**：自动化检查
- **Docker**：容器化开发

**核心目标**：
- 提升开发效率
- 确保代码质量
- 统一团队规范
- 简化部署流程

{
  "editor.formatOnSave": true,
  "files.insertFinalNewline": true,
  "files.trimFinalNewlines": true,
  "[typescript]": {
    "editor.defaultFormatter": "oxc.oxc-vscode"
  },
  "[javascript]": {
    "editor.defaultFormatter": "oxc.oxc-vscode"
  },
  "[json]": {

- **formatOnSave**: 保存时自动格式化
- **defaultFormatter**: 使用 oxc 作为格式化工具
- **useTsgo**: TypeScript 性能优化
- **typeAware**: 启用类型感知 lint

## oxlint/oxfmt 代码质量工具

### 1. oxlint - 快速 Linter

**什么是 oxlint**：
- 基于 Rust 的 TypeScript/JavaScript linter
- 比 ESLint 快 50-100 倍
- 支持 type-aware linting

**安装**：
```bash
pnpm add -D oxlint oxlint-tsgolint
```

**基本使用**：
```bash
# 基础 lint
oxlint

# Type-aware lint
oxlint --type-aware

# Type-aware lint + 类型检查
oxlint --type-aware --type-check

# 自动修复
oxlint --fix
```

---

### 2. Type-Aware Linting

**什么是 Type-Aware Linting**：
- 基于 TypeScript 类型系统进行更精确的代码检查
- 可以检测类型相关的错误

**配置**：
```json
// .oxlintrc.json
{
  "plugins": ["typescript"],
  "rules": {
    "typescript/no-floating-promises": "error",
    "typescript/no-unsafe-assignment": "warn",
    "typescript/no-unsafe-call": "error",
    "typescript/no-unsafe-member-access": "warn"
  }
}
```

**支持的规则**：
- `typescript/no-floating-promises` - 检测未处理的 Promise
- `typescript/no-unsafe-assignment` - 检测不安全的类型赋值
- `typescript/no-unsafe-call` - 检测不安全的函数调用
- `typescript/no-unsafe-member-access` - 检测不安全的成员访问

---

### 3. oxfmt - 快速格式化

**什么是 oxfmt**：
- 基于 Rust 的代码格式化工具
- 比 Prettier 快 10-20 倍
- 兼容 Prettier 配置

**安装**：
```bash
pnpm add -D oxfmt
```

**基本使用**：
```bash
# 格式化文件
oxfmt --write

# 检查格式
oxfmt --check

# 格式化特定文件
oxfmt --write src/**/*.ts
```

**配置**：
```json
// .oxfmtrc.json
{
  "semi": false,
  "singleQuote": true,
  "tabWidth": 2,
  "useTabs": false,
  "trailingComma": "es5",
  "bracketSpacing": true,
  "arrowParens": "always"
}
```

---

### 4. OpenClaw 的使用

**package.json 脚本**：
```json
{
  "scripts": {
    "lint": "oxlint --type-aware",
    "lint:fix": "oxlint --type-aware --fix && pnpm format",
    "format": "oxfmt --write",
    "format:check": "oxfmt --check",
    "check": "pnpm format:check && pnpm tsgo && pnpm lint"
  }
}
```

**使用方式**：
```bash
# 完整检查 (类型 + lint + 格式)
pnpm check

# 仅 lint
pnpm lint

# 自动修复
pnpm lint:fix

# 仅格式化
pnpm format
```

## Git Hooks 自动化检查

### 1. 什么是 Git Hooks？

**Git Hooks**：在 Git 操作（如 commit、push）前后自动执行的脚本。

**常用 Hooks**：
- `pre-commit`：提交前执行
- `pre-push`：推送前执行
- `commit-msg`：提交消息验证

---

### 2. pre-commit Hook

**目的**：在提交前自动检查代码质量

**配置**：
```bash
# .git/hooks/pre-commit
#!/bin/bash

echo "Running pre-commit checks..."

# 格式检查
pnpm format:check
if [ $? -ne 0 ]; then
  echo "❌ Format check failed. Run 'pnpm format' to fix."
  exit 1
fi

# Lint 检查
pnpm lint
if [ $? -ne 0 ]; then
  echo "❌ Lint check failed. Run 'pnpm lint:fix' to fix."
  exit 1
fi

# 类型检查
pnpm tsgo
if [ $? -ne 0 ]; then
  echo "❌ Type check failed."
  exit 1
fi

echo "✅ All checks passed!"
```

**使用 husky**：
```bash
# 安装 husky
pnpm add -D husky

# 初始化
pnpm exec husky init

# 添加 pre-commit hook
echo "pnpm check" > .husky/pre-commit
```

---

### 3. commit-msg Hook

**目的**：验证提交消息格式

**配置**：
```bash
# .husky/commit-msg
#!/bin/bash

commit_msg=$(cat "$1")

# 检查提交消息格式
if ! echo "$commit_msg" | grep -qE "^(feat|fix|docs|style|refactor|test|chore): .+"; then
  echo "❌ Invalid commit message format."
  echo "Format: <type>: <description>"
  echo "Types: feat, fix, docs, style, refactor, test, chore"
  exit 1
fi

echo "✅ Commit message format valid!"
```

## Docker 开发环境

### 1. 多阶段构建

**Dockerfile**：
```dockerfile
# 阶段 1: 构建
FROM node:22-alpine AS builder
WORKDIR /app

# 安装 pnpm
RUN npm install -g pnpm

# 复制依赖文件
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY ui/package.json ./ui/
COPY packages/*/package.json ./packages/
COPY extensions/*/package.json ./extensions/

# 安装依赖
RUN pnpm install --frozen-lockfile

# 复制源码
COPY . .

# 构建
RUN pnpm build

# 阶段 2: 生产
FROM node:22-alpine
WORKDIR /app

# 安装 pnpm
RUN npm install -g pnpm

# 复制构建产物
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./

# 暴露端口
EXPOSE 18789

# 启动命令
CMD ["node", "dist/index.js"]
```

**关键特性**：
- **多阶段构建**：分离构建和运行环境
- **Alpine 镜像**：减小镜像体积
- **缓存优化**：先复制依赖文件，再复制源码

---

### 2. docker-compose 开发环境

**docker-compose.yml**：
```yaml
version: '3.8'

services:
  openclaw:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "18789:18789"
    volumes:
      - ./src:/app/src
      - ./dist:/app/dist
    environment:
      - NODE_ENV=development
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
    command: pnpm gateway:dev

  postgres:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_USER=openclaw
      - POSTGRES_PASSWORD=openclaw
      - POSTGRES_DB=openclaw
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

**使用方式**：
```bash
# 启动所有服务
docker-compose up

# 后台运行
docker-compose up -d

# 查看日志
docker-compose logs -f openclaw

# 停止服务
docker-compose down
```

## 与 OpenClaw 的关系

### 1. 为什么 OpenClaw 使用 oxlint/oxfmt？

**问题**：ESLint 和 Prettier 速度慢

**解决方案**：oxlint/oxfmt
- **更快的速度**：Rust 实现，比 ESLint 快 50-100 倍
- **type-aware linting**：基于类型系统的检查
- **统一工具链**：lint 和 format 使用同一工具

---

### 2. OpenClaw 的开发流程

```
1. 克隆仓库
   ↓
2. 安装依赖 (pnpm install)
   ↓
3. 配置 VS Code (自动加载 .vscode/)
   ↓
4. 开发 (pnpm gateway:dev)
   ↓
5. 提交前检查 (pre-commit hook)
   ↓
6. 提交代码 (git commit)
   ↓
7. CI/CD 检查 (GitHub Actions)
   ↓
8. 部署 (Docker)
```

---

### 3. OpenClaw 的工具链集成

```
VS Code
  ├── oxc.oxc-vscode (格式化)
  ├── TypeScript (类型检查)
  └── Debugger (调试)
       ↓
Git Hooks
  ├── pre-commit (格式 + lint + 类型)
  └── commit-msg (提交消息验证)
       ↓
CI/CD
  ├── pnpm check (完整检查)
  ├── pnpm test (测试)
  └── pnpm build (构建)
       ↓
Docker
  ├── 多阶段构建

```
