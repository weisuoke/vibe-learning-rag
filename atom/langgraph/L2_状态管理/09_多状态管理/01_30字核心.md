# 多状态管理 - 30字核心

**多状态管理是通过 Schema 分离、节点分组和子图隔离，为复杂工作流的不同阶段划定独立状态边界的机制。**

---

## 关键词解析

### 多状态管理 = 图的"权限管理系统"

- **Schema 分离**：用 `input_schema` / `output_schema` 控制图的入口和出口，隐藏内部实现细节
- **节点分组**：每个节点可以声明自己只需要状态的哪些字段，不必接收全部状态
- **子图隔离**：子图拥有完全独立的 Channel 集合，与父图通过共享键或显式调用通信
- **状态边界**：不同阶段的数据互不干扰，内部中间状态不会泄露到外部

---

## 核心公式

```
StateGraph(
    state_schema,        # 内部完整状态（所有字段）
    input_schema,        # 外部输入接口（入口白名单）
    output_schema,       # 外部输出接口（出口白名单）
    context_schema       # 运行时不可变上下文（不创建 Channel）
)
```

```python
# 源码层面：StateGraph 构造函数的四个 Schema 参数
class StateGraph:
    def __init__(
        self,
        state_schema: type[StateT],
        context_schema: type[ContextT] | None = None,
        *,
        input_schema: type[InputT] | None = None,   # 默认 = state_schema
        output_schema: type[OutputT] | None = None,  # 默认 = state_schema
    ) -> None: ...
```

## 与相关概念的区分

| 概念 | 关注点 | 类比 |
|------|--------|------|
| **Reducer** | 同一字段如何合并多个更新 | 快递分拣员 |
| **状态转换函数** | 单个节点如何处理数据 | 工厂加工车间 |
| **状态传递** | 数据如何在节点间流动 | 传送带 |
| **多状态管理** | 不同阶段/模块各自拥有哪些状态 | 公司部门权限管理 |

---

## 记忆锚点

> **多状态管理 = "谁能看到什么、谁能改什么、谁和谁共享什么"**

一个图有四层状态边界：输入接口（外部能传什么）、输出接口（外部能拿什么）、内部完整状态（节点间共享什么）、子图私有状态（子模块自己用什么）。多状态管理就是把这四层边界定义清楚。
