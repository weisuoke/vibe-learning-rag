# 核心概念_3_混合检索执行策略

## 概述

混合检索执行策略是将多个检索结果合并成单一排序列表的关键技术。本文档深入讲解RRF算法、加权排序和重排序机制的原理与实现。

---

## 核心概念1：RRF算法（Reciprocal Rank Fusion）

**RRF是一种基于排名位置而非分数的结果融合算法，通过互逆秩公式公平地合并多个检索结果。**

### RRF算法原理

RRF算法的核心思想是：排名位置比分数更可靠，因为不同检索方式的分数量纲不同，无法直接比较。

**公式：**
```
score_rrf(d) = Σ(1 / (k + rank_i(d)))
```

**参数说明：**
- `d`：文档
- `k`：平滑参数（通常为60）
- `rank_i(d)`：文档d在第i个检索结果中的排名
- `Σ`：对所有检索路径求和

### RRF计算示例

```python
# 向量检索结果
vector_results = {
    "doc1": {"rank": 1, "score": 0.95},
    "doc2": {"rank": 2, "score": 0.88},
    "doc3": {"rank": 3, "score": 0.82},
}

# BM25检索结果
bm25_results = {
    "doc2": {"rank": 1, "score": 8.5},
    "doc1": {"rank": 2, "score": 7.2},
    "doc4": {"rank": 3, "score": 6.8},
}

# RRF计算（k=60）
def calculate_rrf(doc_id, k=60):
    score = 0
    
    # 向量检索贡献
    if doc_id in vector_results:
        rank = vector_results[doc_id]["rank"]
        score += 1 / (k + rank)
    
    # BM25检索贡献
    if doc_id in bm25_results:
        rank = bm25_results[doc_id]["rank"]
        score += 1 / (k + rank)
    
    return score

# 计算所有文档的RRF分数
all_docs = set(vector_results.keys()) | set(bm25_results.keys())
rrf_scores = {doc: calculate_rrf(doc) for doc in all_docs}

# 排序
sorted_docs = sorted(rrf_scores.items(), key=lambda x: x[1], reverse=True)

print("RRF排序结果：")
for doc, score in sorted_docs:
    print(f"{doc}: {score:.6f}")

# 输出：
# doc1: 0.032520 (1/(60+1) + 1/(60+2))
# doc2: 0.032520 (1/(60+2) + 1/(60+1))
# doc3: 0.015873 (1/(60+3) + 0)
# doc4: 0.015873 (0 + 1/(60+3))
```

### RRF在Milvus中的实现

```python
from pymilvus import MilvusClient, AnnSearchRequest, RRFRanker

client = MilvusClient(uri="http://localhost:19530")

# 稠密向量检索
dense_req = AnnSearchRequest(
    data=[dense_query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20
)

# 稀疏向量检索
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20
)

# RRF重排序
results = client.hybrid_search(
    collection_name="documents",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),  # k参数可调
    limit=10,
    output_fields=["text", "title"]
)

# 处理结果
for hits in results:
    for hit in hits:
        print(f"Score: {hit.score:.6f}")
        print(f"Text: {hit.entity.get('text')}")
        print("-" * 50)
```

### RRF参数调优

**k参数的影响：**

| k值 | 特点 | 适用场景 |
|-----|------|---------|
| 10-30 | 排名靠前的文档权重大 | 高质量检索结果 |
| 60 | 平衡（推荐） | 通用场景 |
| 100-200 | 排名靠后的文档也有机会 | 召回率优先 |

```python
# 测试不同k值
for k in [10, 30, 60, 100]:
    results = client.hybrid_search(
        collection_name="documents",
        reqs=[dense_req, sparse_req],
        ranker=RRFRanker(k=k),
        limit=10
    )
    print(f"k={k}: {len(results)} results")
```

## 核心概念2：加权排序（Weighted Ranker）

**加权排序通过为不同检索路径分配权重，实现可控的结果融合。**

### 加权排序原理

加权排序的核心思想是：不同检索路径的重要性不同，应该赋予不同的权重。

**工作流程：**

1. **收集分数**：从每个检索路径获取分数
2. **分数归一化**：使用arctan函数将分数归一化到[0,1]
3. **加权计算**：分数 × 权重
4. **合并排序**：按加权分数排序

**公式：**
```
score_weighted(d) = Σ(w_i × normalize(score_i(d)))

normalize(score) = arctan(score) / (π/2)
```

### 加权排序示例

```python
import math

# 向量检索结果
vector_results = {
    "doc1": 0.92,
    "doc2": 0.88,
    "doc3": 0.85,
}

# BM25检索结果
bm25_results = {
    "doc2": 8.5,
    "doc1": 7.2,
    "doc4": 6.8,
}

# 归一化函数
def normalize_score(score):
    return math.atan(score) / (math.pi / 2)

# 加权计算（向量60%，BM25 40%）
def calculate_weighted(doc_id, w_vector=0.6, w_bm25=0.4):
    score = 0
    
    # 向量检索贡献
    if doc_id in vector_results:
        normalized = normalize_score(vector_results[doc_id])
        score += w_vector * normalized
    
    # BM25检索贡献
    if doc_id in bm25_results:
        normalized = normalize_score(bm25_results[doc_id])
        score += w_bm25 * normalized
    
    return score

# 计算所有文档的加权分数
all_docs = set(vector_results.keys()) | set(bm25_results.keys())
weighted_scores = {doc: calculate_weighted(doc) for doc in all_docs}

# 排序
sorted_docs = sorted(weighted_scores.items(), key=lambda x: x[1], reverse=True)

print("加权排序结果：")
for doc, score in sorted_docs:
    print(f"{doc}: {score:.6f}")
```

### 加权排序在Milvus中的实现

```python
from pymilvus import WeightedRanker

# 加权排序（向量60%，BM25 40%）
results = client.hybrid_search(
    collection_name="documents",
    reqs=[dense_req, sparse_req],
    ranker=WeightedRanker(weights=[0.6, 0.4]),
    limit=10,
    output_fields=["text", "title"]
)
```

### 权重选择策略

**场景1：文本检索（推荐：语义60%，关键词40%）**
```python

```

**场景2：电商搜索（推荐：标题50%，描述30%，图像20%）**
```python

```

**场景3：多模态检索（推荐：均等权重）**

## 核心概念3：重排序策略选择

**重排序策略的选择取决于应用场景、数据特点和性能要求。**

### RRF vs Weighted Ranker对比

| 维度 | RRF | Weighted Ranker |
|-----|-----|----------------|
| **原理** | 基于排名位置 | 基于分数加权 |
| **参数** | k（平滑参数） | weights（权重列表） |
| **调参难度** | 低（k通常为60） | 中（需要调整权重） |
| **适用场景** | 等权重融合 | 已知字段重要性 |
| **优势** | 无需调参，民主化 | 可控制权重 |
| **劣势** | 无法控制权重 | 需要调参 |
| **性能** | 相似 | 相似 |

### 选择决策树

```
是否已知字段重要性？
├─ 是 → Weighted Ranker
│   └─ 权重如何分配？
│       ├─ 根据业务经验
│       ├─ 根据A/B测试
│       └─ 根据用户反馈
│
└─ 否 → RRF
    └─ k参数如何选择？
        ├─ 默认60（推荐）
        ├─ 10-30（高质量优先）
        └─ 100-200（召回率优先）
```

### 实际应用示例

**场景1：文档问答系统（推荐RRF）**
```python
# 语义和关键词同等重要
results = client.hybrid_search(
    collection_name="knowledge_base",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=5
)
```

**场景2：电商商品搜索（推荐Weighted）**
```python
# 标题更重要（60%），描述次之（30%），关键词最后（10%）
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, sparse_req],
    ranker=WeightedRanker(weights=[0.6, 0.3, 0.1]),
    limit=20
)
```

**场景3：多模态检索（推荐RRF）**
```python
# 文本、图像、音频等权重
results = client.hybrid_search(
    collection_name="multimedia",
    reqs=[text_req, image_req, audio_req],
    ranker=RRFRanker(k=60),
    limit=10
)
```

## 混合检索执行流程

### 完整执行流程

```
用户查询
    ↓
┌─────────────────────────────────────┐
│  1. 查询预处理                       │
│  - 文本分析                          │
│  - 生成稠密向量（Embedding）         │
│  - 生成稀疏向量（BM25分词）          │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  2. 并行检索                         │
│  ┌──────────────┐  ┌──────────────┐ │
│  │ 稠密向量检索  │  │ 稀疏向量检索  │ │
│  │ (HNSW索引)   │  │ (倒排索引)    │ │
│  │ TopK=20      │  │ TopK=20      │ │
│  └──────────────┘  └──────────────┘ │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  3. 结果收集                         │
│  - 收集两组结果                      │
│  - 提取排名和分数                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  4. 重排序                           │
│  - 应用RRF或Weighted算法             │
│  - 计算最终分数                      │
│  - 按分数排序                        │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  5. 返回结果                         │
│  - 返回TopK结果                      │
│  - 包含文档内容和分数                │
└─────────────────────────────────────┘
```

### 代码实现

```python
from pymilvus import MilvusClient, AnnSearchRequest, RRFRanker
from sentence_transformers import SentenceTransformer
import time

# 初始化
client = MilvusClient(uri="http://localhost:19530")
model = SentenceTransformer('all-MiniLM-L6-v2')

def hybrid_search_pipeline(query_text, collection_name, top_k=10):
    """完整的混合检索流程"""
    
    # 1. 查询预处理
    print("1. 查询预处理...")
    start_time = time.time()
    
    # 生成稠密向量
    dense_vector = model.encode(query_text)
    print(f"   - 稠密向量维度: {len(dense_vector)}")
    
    # 稀疏向量由Milvus自动生成（BM25）
    print(f"   - 稀疏向量: 自动生成")
    
    preprocess_time = (time.time() - start_time) * 1000
    print(f"   - 预处理耗时: {preprocess_time:.2f}ms")
    
    # 2. 并行检索
    print("\n2. 并行检索...")
    search_start = time.time()
    
    # 稠密向量检索
    dense_req = AnnSearchRequest(
        data=[dense_vector],
        anns_field="dense_vector",
        param={"metric_type": "COSINE", "params": {"ef": 64}},
        limit=20
    )
    
    # 稀疏向量检索
    sparse_req = AnnSearchRequest(
        data=[query_text],
        anns_field="sparse_vector",
        param={"metric_type": "BM25"},
        limit=20
    )
    
    # 3. 结果收集 + 4. 重排序
    print("\n3. 结果收集 + 4. 重排序...")
    results = client.hybrid_search(
        collection_name=collection_name,
        reqs=[dense_req, sparse_req],
        ranker=RRFRanker(k=60),
        limit=top_k,
        output_fields=["text", "title"]
    )
    
    search_time = (time.time() - search_start) * 1000
    print(f"   - 检索+重排序耗时: {search_time:.2f}ms")
    
    # 5. 返回结果
    print(f"\n5. 返回结果 (Top {top_k})...")
    total_time = (time.time() - start_time) * 1000
    print(f"   - 总耗时: {total_time:.2f}ms")
    
    return results

# 使用示例
query = "如何优化Python代码性能？"
results = hybrid_search_pipeline(query, "knowledge_base", top_k=5)

# 展示结果
print("\n" + "="*50)
print("检索结果：")
print("="*50)
for hits in results:
    for i, hit in enumerate(hits, 1):
        print(f"\n{i}. Score: {hit.score:.6f}")
        print(f"   Title: {hit.entity.get('title')}")
        print(f"   Text: {hit.entity.get('text')[:100]}...")
```

## 性能优化

### 1. 并行检索优化

```python
# 确保多个检索请求并行执行
# Milvus内部会自动并行化
results = client.hybrid_search(
    collection_name="documents",
    reqs=[req1, req2, req3],  # 多个请求并行
    ranker=RRFRanker(k=60),
    limit=10
)
```

### 2. TopK调优

```python
# 每个检索路径的TopK应该大于最终TopK
# 推荐：每个路径TopK = 最终TopK × 2

final_top_k = 10

dense_req = AnnSearchRequest(
    data=[dense_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=final_top_k * 2  # 20
)

sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=final_top_k * 2  # 20
)

results = client.hybrid_search(
    collection_name="documents",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=final_top_k  # 10
)
```

### 3. 缓存策略

```python
import hashlib
import json

# 缓存重排序结果
cache = {}

def cached_hybrid_search(query_text, collection_name, top_k=10):
    # 生成缓存键
    cache_key = hashlib.md5(
        f"{query_text}:{collection_name}:{top_k}".encode()
    ).hexdigest()
    
    # 检查缓存
    if cache_key in cache:
        print("Cache hit!")
        return cache[cache_key]
    
    # 执行检索
    results = hybrid_search_pipeline(query_text, collection_name, top_k)
    
    # 缓存结果
    cache[cache_key] = results
    
    return results
```

## 总结

### 核心要点

1. **RRF算法**：基于排名位置，无需调参，民主化融合
2. **加权排序**：基于分数加权，可控制权重，需要调参
3. **策略选择**：根据场景选择RRF或Weighted
4. **性能优化**：并行检索、TopK调优、缓存策略

### 最佳实践

1. **默认使用RRF**：k=60适合大多数场景
2. **已知重要性用Weighted**：根据业务经验分配权重
3. **TopK设置**：每个路径TopK = 最终TopK × 2
4. **性能监控**：跟踪延迟和召回率
5. **A/B测试**：验证重排序策略效果
