## 类比 1：节点返回部分字段

### LangGraph 概念

**节点函数只返回需要更新的字段，未返回的字段保持原值不变。**

[来源: reference/context7_langgraph_01.md]

### 前端类比：React 组件的 setState

**类比对象：** React 类组件的 `setState` 方法

```javascript
// React 类组件
class MyComponent extends React.Component {
  state = {
    count: 0,
    message: "Hello",
    isActive: false
  };

  handleClick = () => {
    // 只更新 count，其他字段保持不变
    this.setState({ count: this.state.count + 1 });
    // message 和 isActive 保持原值
  };
}
```

**相似点：**
- 只需要指定要更新的字段
- 未指定的字段自动保留
- 框架自动合并状态

[来源: CLAUDE_LANGGRAPH.md]

### LangGraph 对应代码

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    count: int
    message: str
    is_active: bool

def my_node(state: State) -> dict:
    # 只更新 count，其他字段保持不变
    return {"count": state["count"] + 1}
    # message 和 is_active 保持原值

# 构建图
graph = StateGraph(State)
graph.add_node("my_node", my_node)
graph.add_edge(START, "my_node")
graph.add_edge("my_node", END)

app = graph.compile()

# 运行
result = app.invoke({
    "count": 0,
    "message": "Hello",
    "is_active": False
})

print(result)
# {'count': 1, 'message': 'Hello', 'is_active': False}
```

[来源: reference/context7_langgraph_01.md]

```
产品状态：
```

## 类比 2：Reducer 函数

### LangGraph 概念

**Reducer 函数定义如何合并当前值和新值，实现增量更新而非覆盖。**

[来源: reference/context7_langgraph_01.md, reference/source_部分状态更新_01.md]

### 前端类比：Redux Reducer

**类比对象：** Redux 的 reducer 函数

```javascript
// Redux reducer
const todoReducer = (state = [], action) => {
  switch (action.type) {
    case 'ADD_TODO':
      // 追加新 todo，而不是替换整个列表
      return [...state, action.payload];
    case 'REMOVE_TODO':
      return state.filter(todo => todo.id !== action.payload);
    default:
      return state;
  }
};

// 使用示例
let todos = [];
todos = todoReducer(todos, { type: 'ADD_TODO', payload: { id: 1, text: 'Learn Redux' } });
// todos = [{ id: 1, text: 'Learn Redux' }]

todos = todoReducer(todos, { type: 'ADD_TODO', payload: { id: 2, text: 'Learn LangGraph' } });
// todos = [{ id: 1, text: 'Learn Redux' }, { id: 2, text: 'Learn LangGraph' }]
```

**相似点：**
- 接收当前状态和新值
- 返回合并后的状态
- 不直接修改原状态

[来源: CLAUDE_LANGGRAPH.md]

### LangGraph 对应代码

```python
from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    # 使用 operator.add 作为 Reducer
    todos: Annotated[list[str], operator.add]

def add_todo_node(state: State) -> dict:
    # 追加新 todo，而不是替换整个列表
    return {"todos": ["Learn LangGraph"]}

# 构建图
graph = StateGraph(State)
graph.add_node("add_todo", add_todo_node)
graph.add_edge(START, "add_todo")
graph.add_edge("add_todo", END)

app = graph.compile()

# 第一次运行
result1 = app.invoke({"todos": ["Learn Redux"]})
print(result1)
# {'todos': ['Learn Redux', 'Learn LangGraph']}

# 第二次运行
result2 = app.invoke({"todos": result1["todos"]})
print(result2)
# {'todos': ['Learn Redux', 'Learn LangGraph', 'Learn LangGraph']}
```

[来源: reference/context7_langgraph_01.md]

### 日常生活类比：收集箱

**类比对象：** 办公室的文件收集箱

想象一个办公室的文件收集箱：

```
收集箱规则：
- 默认规则：新文件替换旧文件（覆盖）
- 累积规则：新文件放在旧文件上面（追加）

场景 1：使用默认规则（覆盖）
- 收集箱里有：报告 A
- 放入：报告 B
- 结果：只有报告 B（报告 A 被替换）

场景 2：使用累积规则（追加）
- 收集箱里有：报告 A
- 放入：报告 B
- 结果：报告 A + 报告 B（都保留）
```

**相似点：**
- Reducer 定义了"放入规则"
- 可以选择替换或累积
- 规则一旦定义，自动应用

## 类比 3：operator.add Reducer

### LangGraph 概念

**`operator.add` 是最常用的 Reducer，用于累加数值、追加列表、拼接字符串。**

[来源: reference/context7_langgraph_01.md]

### 前端类比：数组的 concat 方法

**类比对象：** JavaScript 数组的 `concat` 方法

```javascript
// JavaScript 数组 concat
let items = ['A', 'B'];
items = items.concat(['C', 'D']);
// items = ['A', 'B', 'C', 'D']

// 等价于 operator.add 的行为
// [A, B] + [C, D] = [A, B, C, D]
```

**相似点：**
- 不修改原数组
- 返回新数组
- 自动合并元素

[来源: CLAUDE_LANGGRAPH.md]

### LangGraph 对应代码

```python
from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    items: Annotated[list[str], operator.add]

def add_items_node(state: State) -> dict:
    return {"items": ["C", "D"]}

# 构建图
graph = StateGraph(State)
graph.add_node("add_items", add_items_node)
graph.add_edge(START, "add_items")
graph.add_edge("add_items", END)

app = graph.compile()

# 运行
result = app.invoke({"items": ["A", "B"]})
print(result)
# {'items': ['A', 'B', 'C', 'D']}
```

### 日常生活类比：购物车

**类比对象：** 超市购物车

想象你在超市购物：

```
购物车操作：
- 初始状态：购物车里有 [苹果, 香蕉]
- 添加操作：放入 [牛奶, 面包]

关键点：
```

- operator.add 就像"放入购物车"
- 原有内容保留

## 类比 4：add_messages Reducer

### LangGraph 概念

**`add_messages` 是专门用于消息列表的 Reducer，用于维护对话历史。**

[来源: reference/context7_langgraph_01.md]

### 前端类比：聊天应用的消息列表

**类比对象：** 聊天应用（如微信、Slack）的消息列表

```javascript
// 聊天应用的消息列表
let messages = [
  { id: 1, role: 'user', content: '你好' },
  { id: 2, role: 'assistant', content: '你好！有什么可以帮你的吗？' }
];

// 添加新消息（追加而非替换）
messages.push({ id: 3, role: 'user', content: '今天天气怎么样？' });

// 消息列表保留所有历史
console.log(messages);
// [
//   { id: 1, role: 'user', content: '你好' },
//   { id: 2, role: 'assistant', content: '你好！有什么可以帮你的吗？' },
//   { id: 3, role: 'user', content: '今天天气怎么样？' }
// ]
```

**相似点：**
- 消息追加而非替换
- 保留完整对话历史
- 支持多轮对话

[来源: CLAUDE_LANGGRAPH.md]

### LangGraph 对应代码

```python
from typing import Annotated, Sequence, TypedDict
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph.message import add_messages
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]

def chatbot_node(state: State) -> dict:
    last_message = state["messages"][-1]
    response = f"你说：{last_message.content}"
    return {"messages": [AIMessage(content=response)]}

# 构建图
graph = StateGraph(State)
graph.add_node("chatbot", chatbot_node)
graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)

app = graph.compile()

# 第一轮对话
result1 = app.invoke({"messages": [HumanMessage(content="你好")]})
print([f"{msg.type}: {msg.content}" for msg in result1["messages"]])
# ['human: 你好', 'ai: 你说：你好']

# 第二轮对话（继续追加）
result2 = app.invoke({
    "messages": result1["messages"] + [HumanMessage(content="今天天气怎么样？")]
})
print([f"{msg.type}: {msg.content}" for msg in result2["messages"]])
# ['human: 你好', 'ai: 你说：你好', 'human: 今天天气怎么样？', 'ai: 你说：今天天气怎么样？']
```

[来源: reference/context7_langgraph_01.md]

## 类比 5：覆盖 vs 合并策略

### LangGraph 概念

**没有 Reducer 的字段使用覆盖策略，有 Reducer 的字段使用合并策略。**

[来源: reference/context7_langgraph_01.md]

### 前端类比：变量赋值 vs 数组 push

**类比对象：** JavaScript 的变量赋值和数组操作

```javascript
// 覆盖策略：变量赋值
let status = "pending";
status = "completed";  // 直接覆盖
console.log(status);   // "completed"

// 合并策略：数组 push
let logs = ["step1"];
logs.push("step2");    // 追加而非覆盖
console.log(logs);     // ["step1", "step2"]
```

**相似点：**
- 覆盖策略 = 变量赋值（直接替换）
- 合并策略 = 数组 push（追加）
- 根据需求选择策略

[来源: CLAUDE_LANGGRAPH.md]

### LangGraph 对应代码

```python
from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    # 覆盖策略：没有 Reducer
    status: str
    # 合并策略：有 Reducer
    logs: Annotated[list[str], operator.add]

def step1_node(state: State) -> dict:
    return {
        "status": "step1_completed",
        "logs": ["执行 step1"]
    }

def step2_node(state: State) -> dict:
    return {
        "status": "step2_completed",  # 覆盖 step1_completed
        "logs": ["执行 step2"]         # 追加到 logs
    }

# 构建图
graph = StateGraph(State)
graph.add_node("step1", step1_node)
graph.add_node("step2", step2_node)
graph.add_edge(START, "step1")
graph.add_edge("step1", "step2")
graph.add_edge("step2", END)

app = graph.compile()

# 运行
result = app.invoke({"status": "init", "logs": []})
print(result)
# {'status': 'step2_completed', 'logs': ['执行 step1', '执行 step2']}
```

[来源: reference/context7_langgraph_01.md]

### 日常生活类比：白板 vs 笔记本

**类比对象：** 白板和笔记本的使用方式

```
白板（覆盖策略）：
- 写上"任务：准备会议"
- 擦掉，写上"任务：写报告"
- 结果：只有"任务：写报告"

笔记本（合并策略）：
- 写上"任务 1：准备会议"
- 翻页，写上"任务 2：写报告"
- 结果：两个任务都保留
```

**相似点：**
- 覆盖策略 = 白板（擦掉重写）
- 合并策略 = 笔记本（累积记录）
- 根据需求选择工具

## 类比应用：完整示例

### 场景：多步推理工作流

让我们通过一个完整示例，综合应用所有类比：

```python
"""
完整示例：多步推理工作流
综合应用所有类比概念
"""

from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

# 定义状态（类比：工厂产品状态）
class ReasoningState(TypedDict):
    question: str                                      # 覆盖策略（白板）
    thoughts: Annotated[list[str], operator.add]      # 合并策略（笔记本）
    answer: str                                        # 覆盖策略（白板）

# 节点 1：分析问题（类比：工人 A）
def analyze_node(state: ReasoningState) -> dict:
    # 只返回自己负责的字段
    return {"thoughts": [f"分析：{state['question']}"]}

# 节点 2：推理过程（类比：工人 B）
def reason_node(state: ReasoningState) -> dict:
    # 只返回自己负责的字段
    return {"thoughts": ["推理：基于已有信息..."]}

# 节点 3：得出结论（类比：工人 C）
def conclude_node(state: ReasoningState) -> dict:
    # 更新多个字段
    all_thoughts = "\n".join(state["thoughts"])
    return {
        "thoughts": ["结论：综合分析得出答案"],
        "answer": f"基于以下思考：\n{all_thoughts}\n答案是..."
    }

# 构建图（类比：流水线）
graph = StateGraph(ReasoningState)
graph.add_node("analyze", analyze_node)
graph.add_node("reason", reason_node)
graph.add_node("conclude", conclude_node)
graph.add_edge(START, "analyze")
graph.add_edge("analyze", "reason")
graph.add_edge("reason", "conclude")
graph.add_edge("conclude", END)

app = graph.compile()

# 运行（类比：产品流经流水线）
result = app.invoke({
    "question": "什么是 LangGraph？",
    "thoughts": [],
    "answer": ""
})

print("=== 最终状态 ===")
print(f"问题: {result['question']}")
print(f"思考过程: {result['thoughts']}")
print(f"答案: {result['answer']}")
```

**运行输出：**
```
=== 最终状态 ===
问题: 什么是 LangGraph？
思考过程: ['分析：什么是 LangGraph？', '推理：基于已有信息...', '结论：综合分析得出答案']
答案: 基于以下思考：
分析：什么是 LangGraph？
推理：基于已有信息...
结论：综合分析得出答案
答案是...
```

**类比解释：**
1. **工厂流水线**：每个节点像工人，只处理自己负责的部分
2. **白板 vs 笔记本**：`question` 和 `answer` 使用覆盖策略（白板），`thoughts` 使用合并策略（笔记本）
3. **购物车**：`thoughts` 像购物车，每个节点追加新的思考
4. **Redux reducer**：`operator.add` 定义了如何合并 `thoughts`

[来源: reference/context7_langgraph_01.md]
