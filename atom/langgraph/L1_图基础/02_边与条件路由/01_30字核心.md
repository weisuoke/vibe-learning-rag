## 核心要点

### 什么是边？

边（Edge）是 LangGraph 图中连接节点的有向路径，决定了工作流的执行顺序和流转逻辑。

### 两种边类型

1. **普通边（add_edge）**：固定路由，从起始节点直接连接到结束节点
2. **条件边（add_conditional_edges）**：动态路由，基于状态值选择执行路径

### 核心价值

- **流程控制**：决定节点的执行顺序
- **动态决策**：根据状态值选择不同的执行路径
- **工作流编排**：构建复杂的状态化工作流系统

---

## 快速理解

### 前端类比

边就像前端路由规则：
- **普通边** = 固定路由（`/home` → `HomeComponent`）
- **条件边** = 条件渲染（根据用户权限显示不同组件）

### 日常生活类比

边就像工厂的传送带：
- **普通边** = 固定传送带（产品从 A 工序直接到 B 工序）
- **条件边** = 分拣机（根据产品类型分配到不同工序）

## 基础语法

### 普通边

```python
from langgraph.graph import StateGraph, START, END

graph = StateGraph(State)
graph.add_node("node1", func1)
graph.add_node("node2", func2)

# 固定路由：START → node1 → node2 → END
graph.add_edge(START, "node1")
graph.add_edge("node1", "node2")
graph.add_edge("node2", END)
```

### 条件边

```python
from typing import Literal

def route_func(state: State) -> Literal["path_a", "path_b"]:
    """根据状态值动态选择路径"""
    if state["value"] > 50:
        return "path_a"
    return "path_b"

# 动态路由：根据 route_func 返回值选择路径
graph.add_conditional_edges(
    "node1",
    route_func,
    {"path_a": "node_a", "path_b": "node_b"}
)
```

## 核心区别

| 特性 | 普通边 | 条件边 |
|------|--------|--------|
| 路由类型 | 固定路由 | 动态路由 |
| 目标节点 | 单个固定节点 | 多个动态节点 |
| 决策逻辑 | 无（直接连接） | 路由函数 |
| 使用场景 | 线性流程 | 分支决策 |

## 实际应用场景

### 1. RAG 工作流

```python
# 固定路由：检索 → 评分
graph.add_edge("retrieve", "grade_documents")

# 动态路由：根据文档质量决定下一步
graph.add_conditional_edges(
    "grade_documents",
    decide_to_generate,
    {
        "transform_query": "transform_query",  # 文档不相关，改写查询
        "generate": "generate"                  # 文档相关，直接生成
    }
)
```

### 2. 审批流程

```python
def route_approval(state: State) -> Literal["approved", "rejected", "review"]:
    """根据审批结果路由"""
    if state["score"] > 0.9:
        return "approved"
    elif state["score"] < 0.3:
        return "rejected"
    return "review"

graph.add_conditional_edges(
    "check",
    route_approval,
    {
        "approved": "notify_success",
        "rejected": "notify_failure",
        "review": "human_review"
    }
)
```

### 3. 错误处理

```python
def route_error(state: State) -> Literal["retry", "fallback", END]:
    """根据错误类型决定处理策略"""
    if state.get("retry_count", 0) < 3:
        return "retry"
    elif state.get("has_fallback"):
        return "fallback"
    return END

graph.add_conditional_edges(
    "process",
    route_error,
    {
        "retry": "process",      # 循环重试
        "fallback": "fallback",  # 回退方案
        END: END                 # 结束流程
    }
)
```

## 关键设计原则

### 1. 普通边用于固定流程

当执行顺序确定时，使用普通边：
- 线性流程（A → B → C）
- 并行汇聚（等待多个节点完成）

### 2. 条件边用于动态决策

当需要根据状态值选择路径时，使用条件边：
- 分支逻辑（if-else）
- 多路由决策（switch-case）
- 循环路由（retry/fallback）

### 3. 路由函数保持简单

路由函数应该：
- 逻辑清晰，易于理解
- 避免复杂的嵌套条件
- 使用类型提示明确返回值

## 常见误区

### 误区 1：过度使用条件边

❌ **错误**：所有连接都用条件边
```python
# 不必要的条件边
def always_next(state):
    return "next"

graph.add_conditional_edges("node1", always_next, {"next": "node2"})
```

✅ **正确**：固定流程用普通边
```python
# 简单直接
graph.add_edge("node1", "node2")
```

### 误区 2：路由函数过于复杂

❌ **错误**：复杂的嵌套逻辑
```python
def complex_route(state):
    if state["a"]:
        if state["b"]:
            if state["c"]:
                return "path1"
            else:
                return "path2"
        else:
            return "path3"
    else:
        return "path4"
```

✅ **正确**：扁平化逻辑
```python
def simple_route(state):
    if state["a"] and state["b"] and state["c"]:
        return "path1"
    if state["a"] and state["b"]:
        return "path2"
    if state["a"]:
        return "path3"
    return "path4"
```

### 误区 3：忘记添加结束路径

❌ **错误**：没有结束条件
```python
def route_func(state):
    if state["retry"]:
        return "process"  # 永远循环
    return "process"
```

✅ **正确**：添加结束条件
```python
def route_func(state) -> Literal["process", END]:
    if state.get("retry_count", 0) < 3 and state["retry"]:
        return "process"
    return END
```

## 核心 API

### add_edge

```python
def add_edge(
    self,
    start_key: str | list[str],  # 起始节点（可以是多个）
    end_key: str                  # 结束节点
) -> Self:
    """添加固定路由的边"""
```

**特性**：
- 单起始节点：直接连接
- 多起始节点：等待所有节点完成（AND 逻辑）

### add_conditional_edges

```python
def add_conditional_edges(
    self,
    source: str,                              # 起始节点
    path: Callable[..., Hashable],           # 路由函数
    path_map: dict[Hashable, str] | None     # 路径映射
) -> Self:
    """添加动态路由的边"""
```

**特性**：
- 路由函数接受状态，返回目标节点名称
- 支持多目标路由
- 使用 `Literal` 类型提示明确可能的路由目标

## 快速参考

### 何时使用普通边？

- ✅ 线性流程（A → B → C）
- ✅ 固定的执行顺序
- ✅ 并行汇聚（等待多个节点）

### 何时使用条件边？

- ✅ 分支决策（if-else）
- ✅ 多路由选择（switch-case）
- ✅ 循环路由（retry/fallback）
- ✅ 动态工作流

### 路由函数设计要点

1. **使用类型提示**：`-> Literal["path_a", "path_b"]`
2. **逻辑清晰**：避免复杂嵌套
3. **处理边界**：检查缺失值和异常值
4. **添加日志**：便于调试和追踪
