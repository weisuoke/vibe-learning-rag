# 30字核心

**pnpm monorepo + tsdown 多入口构建 + Vitest 分层测试 + 原生依赖编译 = 可调试的多通道 AI 网关**

---

## 核心要素拆解

### 1. pnpm monorepo
- **作用**: 统一管理多个包（root, ui, packages/*, extensions/*）
- **价值**: 依赖共享、版本统一、构建协调

### 2. tsdown 多入口构建
- **作用**: 8 个独立入口点，TypeScript 到 JavaScript 的高效转换
- **价值**: 模块化构建、tree-shaking、快速编译

### 3. Vitest 分层测试
- **作用**: 5 个独立配置（unit, e2e, live, gateway, extensions）
- **价值**: 隔离测试、并发优化、覆盖率保证

### 4. 原生依赖编译
- **作用**: 9 个原生模块（node-pty, canvas, sharp 等）的跨平台编译
- **价值**: 终端模拟、图像处理、LLM 推理等高级功能

### 5. 可调试的多通道 AI 网关
- **目标**: 从源码到可运行、可调试、可扩展的 OpenClaw 系统
- **价值**: 深入理解架构、定制开发、问题排查

## 与 OpenClaw 的关系

### 为什么 OpenClaw 需要这些？

1. **Monorepo 架构**
   - OpenClaw 有多个包：核心、UI、扩展
   - 需要统一管理依赖和版本

2. **多入口构建**
   - Gateway、CLI、Plugin SDK、Extension API 等
   - 每个入口点独立构建，按需加载

3. **分层测试**
   - 单元测试：核心逻辑
   - E2E 测试：完整流程
   - Live 测试：外部服务集成

4. **原生依赖**
   - 终端模拟（node-pty）：CLI 交互
   - Canvas 渲染（@napi-rs/canvas）：UI 功能
   - 图像处理（sharp）：消息附件
   - LLM 推理（node-llama-cpp）：本地模型
