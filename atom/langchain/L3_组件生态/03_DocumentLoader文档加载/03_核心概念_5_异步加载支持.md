# 核心概念 5: 异步加载支持

> 理解高性能场景的异步模式

---

## 核心问题

为什么 LangChain 的 DocumentLoader 需要异步加载支持?`aload()` 和 `alazy_load()` 如何提升性能?

---

## 什么是异步加载?

### 定义

**异步加载**: 使用 Python 的 `async/await` 语法,在 I/O 等待期间不阻塞事件循环,允许并发处理多个任务。

**对比**:

```python
# 同步加载 - 阻塞
def load_documents():
    docs = []
    for url in urls:  # 串行处理
        response = requests.get(url)  # 阻塞等待
        docs.append(parse(response))
    return docs

# 异步加载 - 非阻塞
async def load_documents():
    tasks = []
    async with aiohttp.ClientSession() as session:
        for url in urls:  # 并发处理
            task = fetch_and_parse(session, url)  # 不阻塞
            tasks.append(task)
        return await asyncio.gather(*tasks)
```

## 为什么需要异步加载?

### 问题 1: I/O 阻塞

**场景**: 从网络加载多个文档

```python
# 同步加载 - 串行等待
import time

start = time.time()
for url in urls:  # 100 个 URL
    response = requests.get(url)  # 每个耗时 1 秒
    process(response)
print(f"耗时: {time.time() - start}秒")
# 输出: 耗时: 100秒 (串行执行)
```

**异步加载解决**:

```python
# 异步加载 - 并发等待
import asyncio
import aiohttp

start = time.time()
async def fetch_all():
    async with aiohttp.ClientSession() as session:
        tasks = [fetch(session, url) for url in urls]
        return await asyncio.gather(*tasks)

docs = asyncio.run(fetch_all())
print(f"耗时: {time.time() - start}秒")
# 输出: 耗时: 2秒 (并发执行,提升 50x)
```

[来源: reference/fetch_7_db4185_04.md | FastAPI + LangChain 异步集成]

### 问题 2: 资源利用率低

**场景**: CPU 在等待 I/O 时空闲

```python
# 同步加载 - CPU 空闲
def load_from_s3():
    for key in keys:
        data = s3.download(key)  # CPU 等待网络 I/O
        # CPU 利用率: 5%
        process(data)
```

**异步加载 - CPU 充分利用**:

```python
# 异步加载 - CPU 处理其他任务
async def load_from_s3():
    tasks = [s3.download_async(key) for key in keys]
    results = await asyncio.gather(*tasks)
    # CPU 利用率: 80%
    for data in results:
        process(data)
```

### 问题 3: 无法集成异步框架

**场景**: FastAPI 异步端点

```python
# 同步加载 - 阻塞事件循环
@app.post("/load")
def load_endpoint():
    loader = PyPDFLoader("file.pdf")
    docs = loader.load()  # ❌ 阻塞事件循环
    return {"docs": docs}

# 异步加载 - 不阻塞
@app.post("/load")
async def load_endpoint():
    loader = PyPDFLoader("file.pdf")
    docs = await loader.aload()  # ✓ 不阻塞
    return {"docs": docs}
```

## 异步加载的实现

### BaseLoader 中的异步方法

```python
class BaseLoader(ABC):
    async def aload(self) -> list[Document]:
        """异步加载所有文档"""
        return [document async for document in self.alazy_load()]

    async def alazy_load(self) -> AsyncIterator[Document]:
        """异步懒加载文档"""
        # 默认实现:在线程池中运行 lazy_load
        iterator = await run_in_executor(None, self.lazy_load)
        done = object()
        while True:
            doc = await run_in_executor(None, next, iterator, done)
            if doc is done:
                break
            yield doc
```

[来源: reference/source_documentloader_01.md | LangChain 源码分析]

### 两种异步实现

**1. 线程池异步(默认)**:

```python
# 默认实现 - 在线程池中运行同步代码
async def alazy_load(self):
    # 将同步 lazy_load 放到线程池
    iterator = await run_in_executor(None, self.lazy_load)
    done = object()
    while True:
        doc = await run_in_executor(None, next, iterator, done)
        if doc is done:
            break
        yield doc
```

**优点**: 不需要修改现有代码
**缺点**: 不是真正的异步,仍然阻塞线程

**2. 真正的异步(覆盖实现)**:

```python
# 真正的异步 - 使用 aiohttp
class AsyncWebLoader(BaseLoader):
    async def alazy_load(self):
        async with aiohttp.ClientSession() as session:
            for url in self.urls:
                async with session.get(url) as response:
                    content = await response.text()
                    yield Document(
                        page_content=content,
                        metadata={"source": url}
                    )
```

**优点**: 真正的非阻塞,性能最佳
**缺点**: 需要使用异步库(aiohttp, aioboto3 等)

## 实际应用

### 场景 1: 批量网络请求

```python
from langchain_community.document_loaders import WebBaseLoader
import asyncio

# 批量加载网页
urls = [
    "https://example.com/page1",
    "https://example.com/page2",
    # ... 100 个 URL
]

async def load_all_pages():
    loaders = [WebBaseLoader(url) for url in urls]

    # 并发加载所有页面
    tasks = [loader.aload() for loader in loaders]
    results = await asyncio.gather(*tasks)

    # 合并结果
    all_docs = []
    for docs in results:
        all_docs.extend(docs)

    return all_docs

# 运行
docs = asyncio.run(load_all_pages())
print(f"加载了 {len(docs)} 个文档")
```

## 性能对比

### 网络 I/O 密集型

```python
import time
import asyncio

# 测试数据
urls = ["https://example.com/page" + str(i) for i in range(50)]

# 同步加载
start = time.time()
docs_sync = []
for url in urls:
    loader = WebBaseLoader(url)
    docs_sync.extend(loader.load())
sync_time = time.time() - start
print(f"同步加载: {sync_time:.2f}秒")
# 输出: 同步加载: 50.00秒

# 异步加载
start = time.time()
async def load_async():
    loaders = [WebBaseLoader(url) for url in urls]
    tasks = [loader.aload() for loader in loaders]
    results = await asyncio.gather(*tasks)
    return [doc for docs in results for doc in docs]

docs_async = asyncio.run(load_async())
async_time = time.time() - start
print(f"异步加载: {async_time:.2f}秒")
# 输出: 异步加载: 2.00秒

print(f"性能提升: {sync_time / async_time:.1f}x")
# 输出: 性能提升: 25.0x
```

### CPU 密集型

```python
# CPU 密集型任务 - 异步优势不明显
import time

# 同步处理
start = time.time()
for i in range(1000):
    # CPU 密集型计算
    result = sum(j**2 for j in range(10000))
sync_time = time.time() - start
print(f"同步处理: {sync_time:.2f}秒")
# 输出: 同步处理: 5.00秒

# 异步处理 - 没有明显提升
start = time.time()
async def process_async():
    tasks = []
    for i in range(1000):
        async def compute():
            return sum(j**2 for j in range(10000))
        tasks.append(compute())
    return await asyncio.gather(*tasks)

asyncio.run(process_async())
async_time = time.time() - start
print(f"异步处理: {async_time:.2f}秒")
# 输出: 异步处理: 5.00秒 (没有提升)
```

**结论**: 异步加载适合 I/O 密集型任务,不适合 CPU 密集型任务。

## 类比理解

### 前端开发类比

**同步加载** 就像 **同步 AJAX**:

```javascript
// 同步 AJAX - 阻塞 UI
const xhr = new XMLHttpRequest();
xhr.open('GET', url, false);  // false = 同步
xhr.send();
// UI 冻结,用户无法操作
```

**异步加载** 就像 **async/await**:

```javascript
// 异步 fetch - 不阻塞 UI
async function loadData() {
  const response = await fetch(url);
  const data = await response.json();
  // UI 不冻结,用户可以继续操作
  return data;
}
```

### 日常生活类比

**同步加载** 就像 **排队买咖啡**:
- 一个人买完,下一个人才能买
- 每个人等待 5 分钟
- 10 个人需要 50 分钟

**异步加载** 就像 **点单后等叫号**:
- 所有人同时点单
- 点完单就可以坐下等
- 10 个人只需要 5 分钟

## 常见误区

### ❌ 误区 1: 异步总是更快

**错误**: 认为异步加载总是比同步快

**正确**: 只有 I/O 密集型任务才能从异步中受益

```python
# I/O 密集型 - 异步更快
async def load_from_network():
    # 网络请求,大量等待时间
    # 异步可以并发处理
    pass

# CPU 密集型 - 异步没有优势
async def compute_heavy():
    # CPU 计算,没有等待时间
    # 异步无法提升性能
    pass
```

### ❌ 误区 2: 异步可以绕过 GIL

**错误**: 认为异步可以绕过 Python 的 GIL(全局解释器锁)

**正确**: 异步仍然受 GIL 限制,只是在 I/O 等待时释放 GIL

```python
# 异步不能绕过 GIL
async def cpu_bound():
    # 仍然受 GIL 限制
    result = sum(i**2 for i in range(10000000))
    return result

# 多进程才能绕过 GIL
from multiprocessing import Pool

def cpu_bound():
    return sum(i**2 for i in range(10000000))

with Pool(4) as pool:
    results = pool.map(cpu_bound, range(4))
```

### ❌ 误区 3: 异步代码更简单

**错误**: 认为异步代码更容易编写

**正确**: 异步代码更复杂,需要理解事件循环

```python
# 同步代码 - 简单直观
def load_docs():
    docs = []
    for url in urls:
        docs.extend(load(url))
    return docs

# 异步代码 - 需要理解 async/await
async def load_docs():
    tasks = []
    for url in urls:
        tasks.append(load_async(url))
    results = await asyncio.gather(*tasks)
    return [doc for docs in results for doc in docs]
```

### 1. 高并发

```python
async def handle_requests(requests):
```

```python
# 立即响应,不等待所有任务完成
async def stream_results():
    async for doc in loader.alazy_load():
```

```python
# 单线程处理多个任务
async def efficient_processing():
```

## 最佳实践

### 1. 优先使用异步(I/O 密集型)

```python
# ✓ 推荐: I/O 密集型使用异步
async def load_from_web():
    loaders = [WebBaseLoader(url) for url in urls]
    tasks = [loader.aload() for loader in loaders]
    return await asyncio.gather(*tasks)

# ✗ 避免: CPU 密集型使用异步
async def compute_embeddings():
    # 没有 I/O 等待,异步无优势
    pass
```

### 2. 控制并发数量

```python
# 限制并发,避免资源耗尽
from asyncio import Semaphore

async def load_with_limit(urls, max_concurrent=10):
    semaphore = Semaphore(max_concurrent)

    async def load_one(url):
        async with semaphore:
            loader = WebBaseLoader(url)
            return await loader.aload()

    tasks = [load_one(url) for url in urls]
    return await asyncio.gather(*tasks)
```

### 3. 错误处理

```python
# 处理单个任务失败
async def load_with_error_handling(urls):
    async def load_one(url):
        try:
            loader = WebBaseLoader(url)
            return await loader.aload()
        except Exception as e:
            print(f"加载失败: {url}, 错误: {e}")
            return []

    tasks = [load_one(url) for url in urls]
    results = await asyncio.gather(*tasks)
    return [doc for docs in results for doc in docs]
```

### 4. 超时控制

```python
# 设置超时,避免无限等待
import asyncio

async def load_with_timeout(url, timeout=10):
    loader = WebBaseLoader(url)
    try:
        return await asyncio.wait_for(
            loader.aload(),
            timeout=timeout
        )
    except asyncio.TimeoutError:
        print(f"加载超时: {url}")
        return []
```

## 性能优化技巧

### 1. 批量处理

```python
# 批量处理,减少开销
async def batch_load(urls, batch_size=10):
    for i in range(0, len(urls), batch_size):
        batch = urls[i:i+batch_size]
        loaders = [WebBaseLoader(url) for url in batch]
```

## 总结

### 核心原理

1. **非阻塞 I/O**: 在等待 I/O 时不阻塞事件循环
2. **并发执行**: 同时处理多个任务
3. **事件驱动**: 基于事件循环调度任务

### 设计优势

| 优势 | 说明 |
|------|------|
| 高并发 | 单线程处理大量请求 |
| 低延迟 | 立即响应,不等待 |
| 资源高效 | 节省内存和 CPU |
| 易于集成 | 与异步框架无缝集成 |

### 适用场景

| 场景 | 是否适用 |
|------|----------|
| 网络请求 | ✓ 非常适合 |
| 文件 I/O | ✓ 适合 |
| 数据库查询 | ✓ 适合 |
| CPU 计算 | ✗ 不适合 |
