# 最小可用知识

## 概述

掌握以下 20% 的核心知识，就能解决 Gateway 启动与管理中 80% 的常见问题。

---

## 1. 基础启动命令

### 前台启动（开发调试）

```bash
# 最简单的启动方式
openclaw gateway

# 详细日志输出
openclaw gateway --verbose

# 开发模式
openclaw gateway --dev
```

### 守护进程（生产部署）

```bash
# 安装守护进程
openclaw onboard --install-daemon

# 启动/停止/重启
openclaw gateway start
openclaw gateway stop
openclaw gateway restart

# 查看状态
openclaw gateway status
```

---

## 2. 端口配置

### 默认端口

```
18789
```

### 更改端口

```bash
# CLI 参数
openclaw gateway --port 8080

# 配置文件
{
  gateway: { port: 8080 }
}
```

### 端口冲突处理

```bash
# 检查端口占用
lsof -i :18789

# 强制释放端口
openclaw gateway --force
```

---

## 3. 绑定模式

### loopback（默认，最安全）

```bash
openclaw gateway --bind loopback
```

**用途：** 本地开发，只允许本机访问

### lan（局域网访问）

```bash
openclaw gateway --bind lan --token <long-random-token>
```

**用途：** 多设备访问，需要强制认证

### tailnet（远程访问）

```bash
openclaw gateway --bind tailnet --tailscale serve
```

**用途：** 安全远程访问，通过 Tailscale

---

## 4. 日志查看

### 实时日志

```bash
# 前台模式（自动显示）
openclaw gateway --verbose

# 守护进程模式
openclaw logs --follow
```

### 历史日志

```bash
# 查看最近日志
openclaw logs

# 查看最后 100 行
openclaw logs --tail 100
```

---

## 5. 故障排查

### 快速诊断

```bash
# 一键诊断
openclaw doctor

# 安全审计
openclaw security audit

# 健康检查
openclaw gateway health
```

### 常见问题

**问题 1：Gateway 无法启动**
```bash
openclaw logs --tail 50
openclaw doctor
```

**问题 2：端口已被占用**
```bash
lsof -i :18789
openclaw gateway --force
```

**问题 3：无法从其他设备访问**
```bash
openclaw gateway --bind lan --token <token>
```

## 6. 多实例管理

### 使用 Profile

```bash
# 默认实例
openclaw gateway

# 开发实例
openclaw gateway --profile dev --port 18790

# 查看所有实例
openclaw gateway probe
```

## 7. 配置文件位置

```
~/.openclaw/openclaw.json
```

### 最小配置

```json5
{
  gateway: {
    port: 18789,
    bind: "loopback",
    auth: {
      mode: "token",
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  }
}
```

## 10. 生产部署流程
