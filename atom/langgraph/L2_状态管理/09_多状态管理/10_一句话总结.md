# 多状态管理 - 一句话总结

## 核心总结

**多状态管理是通过 Input/Output Schema 分离控制图的边界、节点级类型注解实现状态分组、子图独立 Channel 集合实现状态隔离，并以命名空间划分 checkpoint 作用域，为复杂工作流的每个层级划定清晰数据边界的机制。**

---

## 关键记忆点

| 维度 | 核心要点 |
|------|----------|
| 本质 | 图的"权限管理系统"：控制谁能看到什么、谁能改什么 |
| Schema 分离 | input_schema 管入口，output_schema 管出口 |
| 节点分组 | 函数参数类型注解自动推断节点 input_schema |
| 子图隔离 | 独立 Channel 集合，共享键/函数包装/Command.PARENT 通信 |
| 命名空间 | `NS_SEP="\|"` 分层级，`NS_END=":"` 分 task_id |
| context_schema | 不可变运行时上下文，不创建 Channel |
| Channel 池 | 多 Schema 合并到统一池，Schema 是过滤视图 |
| 设计原则 | 最小暴露、单一职责、显式通信 |

---

## 速记公式

```
多状态管理 = Schema 分离（边界控制）+ 节点分组（按需接收）+ 子图隔离（独立空间）
```
