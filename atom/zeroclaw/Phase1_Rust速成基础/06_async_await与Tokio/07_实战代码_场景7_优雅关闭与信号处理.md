## 概述

一个生产级服务不能说关就关——你需要保存状态、关闭数据库连接、flush 日志缓冲区、通知其他服务"我要下线了"。

ZeroClaw Daemon 在收到 `Ctrl+C` 后，会 abort 所有组件（Gateway、Channel、Scheduler、StateWriter），然后等待它们全部退出。本节从零掌握 Rust 中优雅关闭的三种模式。

## 1. 为什么需要优雅关闭？

### 强制 kill vs 优雅关闭

```
强制 kill（kill -9 / 直接关终端）：
  ┌──────────┐
  │  服务运行  │ ──── 💀 立即死亡
  │           │
  │  · 正在写文件（写到一半 → 文件损坏）
  │  · 正在处理请求（客户端收到连接中断）
  │  · 数据库连接未关闭（连接泄漏）
  │  · 缓冲区数据未 flush（日志丢失）
  └──────────┘

优雅关闭（Ctrl+C / SIGTERM）：
  ┌──────────┐
  │  服务运行  │ ──── ⚠️ 收到关闭信号
  │           │       │
  │           │       ▼
  │           │  1. 停止接受新请求
  │           │  2. 完成正在处理的请求
  │           │  3. flush 缓冲区数据
  │           │  4. 关闭数据库连接
  │           │  5. 保存状态
  │           │       │
  │           │       ▼
  │           │  ✅ 安全退出
  └──────────┘
```

### 前端类比

```
Rust 优雅关闭              ←→    前端对应物
──────────────────────────────────────────────────────
ctrl_c().await              ←→    window.addEventListener('beforeunload', ...)
CancellationToken           ←→    AbortController / AbortSignal
token.cancel()              ←→    controller.abort()
token.cancelled().await     ←→    signal.addEventListener('abort', ...)
handle.abort()              ←→    worker.terminate()
drop(tx) → Actor 退出       ←→    removeEventListener + 清理
```

### 日常生活类比

```
强制关闭 = 突然断电
  → 正在编辑的文档丢失
  → 正在下载的文件损坏
  → 洗衣机洗到一半，衣服泡在水里

优雅关闭 = 下班流程
  → 保存所有文档 (flush 缓冲区)
  → 回复最后几封邮件 (完成进行中的请求)
  → 锁好柜子 (关闭数据库连接)
  → 关灯关门 (释放资源)
  → 走人 ✅
```

## 2. 方法1：abort() 模式（简单粗暴）

最简单的关闭方式——直接 abort 所有任务。ZeroClaw Daemon 用的就是这个模式。

### Cargo.toml

```toml
[package]
name = "graceful-shutdown"
version = "0.1.0"
edition = "2021"

[dependencies]
tokio = { version = "1", features = ["full"] }
tokio-util = "0.7"     # 提供 CancellationToken
anyhow = "1"
```

### 代码

```rust
use std::time::Duration;
use tokio::task::JoinHandle;
use anyhow::Result;

/// 模拟一个长运行服务
async fn long_running_service(name: &str) {
    let mut tick = 0u64;
    loop {
        tick += 1;
        println!("  ⚙️  [{}] 正在运行... tick {}", name, tick);
        tokio::time::sleep(Duration::from_secs(1)).await;
    }
    // 注意：这里永远不会执行到
    // 因为 loop 没有 break，只有被 abort 才会停止
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ abort() 模式：ZeroClaw Daemon 风格 ═══\n");

    // ─── 启动多个服务（对应 ZeroClaw 的多组件启动） ───
    let mut handles: Vec<JoinHandle<()>> = vec![];

    handles.push(tokio::spawn(long_running_service("Gateway")));
    handles.push(tokio::spawn(long_running_service("Channel")));
    handles.push(tokio::spawn(long_running_service("Scheduler")));

    println!("  🧠 所有服务已启动");
    println!("  按 Ctrl+C 关闭...\n");

    // ─── 等待 Ctrl+C ───
    // tokio::signal::ctrl_c() 返回一个 Future
    // .await 后会一直等到用户按下 Ctrl+C
    tokio::signal::ctrl_c().await?;

    println!("\n  ⚠️  收到 Ctrl+C，开始关闭...\n");

    // ─── 第一步：abort 所有任务 ───
    // abort() 不会立即停止任务——它只是设置一个标记
    // 任务会在下一个 .await 点被取消
    for handle in &handles {
        handle.abort();
    }
    println!("  📤 已发送 abort 信号给所有服务");

    // ─── 第二步：等待所有任务结束 ───
    // 即使 abort 了，也要 await handle 来确认任务真的结束了
    for (i, handle) in handles.into_iter().enumerate() {
        match handle.await {
            Ok(()) => println!("  ✅ 服务 {} 正常退出", i),
            Err(e) if e.is_cancelled() => println!("  🛑 服务 {} 已取消", i),
            Err(e) => println!("  ❌ 服务 {} 异常: {}", i, e),
        }
    }

    println!("\n  🏁 所有服务已关闭");
    Ok(())
}
```

### abort() 的原理

```
tokio::spawn(async move {
    step_1().await;     // ← abort 后，如果正好在这里 await，任务立即结束
    step_2();           // ← 这是同步代码，即使 abort 了也会执行完
    step_3().await;     // ← 如果 abort 发生在 step_2 期间，会在这个 .await 点取消
});

重点：
1. abort() 只在 .await 点生效
2. 正在执行的同步代码不会被中断
3. abort 后，handle.await 返回 Err(JoinError::Cancelled)
4. 被 abort 的任务不会执行任何 cleanup 代码
   → 这就是 abort 的缺点：无法优雅清理！
```

### abort() 的问题

```
// 如果服务正在做重要操作：

async fn important_service() {
    loop {
        let data = collect_data().await;    // 收集数据
        write_to_database(data).await;      // ← 如果在这里被 abort...
        //                                     → 数据可能写了一半！
        confirm_written().await;            // ← 这行永远不会执行
    }
}

// abort() 相当于 JS 的 worker.terminate()
// 简单粗暴，但无法保证数据完整性
```

## 3. 方法2：CancellationToken 模式（推荐）

用 `tokio_util::sync::CancellationToken` 实现优雅的协作式关闭。

```rust
use std::time::Duration;
use tokio::task::JoinHandle;
use tokio_util::sync::CancellationToken;
use anyhow::Result;

/// 可优雅关闭的服务
/// token.cancelled().await 类似 JS 的 AbortSignal
async fn graceful_service(name: &str, token: CancellationToken) {
    let mut tick = 0u64;

    loop {
        // tokio::select! 同时等待两件事：
        //   1. 取消信号
        //   2. 正常的工作周期
        tokio::select! {
            // 分支1：收到取消信号
            () = token.cancelled() => {
                println!("  ⚠️  [{}] 收到取消信号，开始清理...", name);

                // ─── 清理操作 ───
                // 这里可以做任何必要的清理工作
                println!("  🧹 [{}] 保存状态...", name);
                tokio::time::sleep(Duration::from_millis(200)).await;

                println!("  🧹 [{}] 关闭连接...", name);
                tokio::time::sleep(Duration::from_millis(100)).await;

                println!("  ✅ [{}] 清理完成，安全退出", name);
                return;  // 退出函数 → 任务结束
            }

            // 分支2：正常工作
            _ = tokio::time::sleep(Duration::from_secs(1)) => {
                tick += 1;
                println!("  ⚙️  [{}] 工作中... tick {}", name, tick);
            }
        }
    }
}

/// 带清理回调的服务
async fn service_with_cleanup(
    name: &str,
    token: CancellationToken,
    cleanup_data: String,
) {
    let mut processed = 0u64;

    loop {
        tokio::select! {
            () = token.cancelled() => {
                println!("  ⚠️  [{}] 关闭中，已处理 {} 条数据", name, processed);
                println!("  🧹 [{}] 保存进度: '{}'", name, cleanup_data);
                tokio::time::sleep(Duration::from_millis(150)).await;
                println!("  ✅ [{}] 进度已保存", name);
                return;
            }

            _ = tokio::time::sleep(Duration::from_millis(500)) => {
                processed += 1;
                println!("  ⚙️  [{}] 处理第 {} 条数据", name, processed);
            }
        }
    }
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ CancellationToken 模式：优雅关闭 ═══\n");

    // ─── 创建取消令牌 ───
    // CancellationToken 可以 clone，所有 clone 共享同一个取消状态
    // 类似 JS 的 AbortController：
    //   const controller = new AbortController();
    //   const signal = controller.signal;  // ← 这就是 token.clone()
    let token = CancellationToken::new();

    // ─── 启动服务，每个服务拿一个 token 的 clone ───
    let mut handles: Vec<JoinHandle<()>> = vec![];

    handles.push(tokio::spawn(graceful_service("Gateway", token.clone())));
    handles.push(tokio::spawn(graceful_service("Channel", token.clone())));
    handles.push(tokio::spawn(service_with_cleanup(
        "StateWriter",
        token.clone(),
        "checkpoint_v42".to_string(),
    )));

    println!("  🧠 所有服务已启动（带优雅关闭支持）");
    println!("  按 Ctrl+C 关闭...\n");

    // ─── 等待 Ctrl+C ───
    tokio::signal::ctrl_c().await?;
    println!("\n  ⚠️  收到 Ctrl+C\n");

    // ─── 触发取消（一个 cancel() 通知所有服务） ───
    // 类似 JS: controller.abort()
    token.cancel();
    println!("  📤 已发送取消信号\n");

    // ─── 等待所有服务完成清理 ───
    for (i, handle) in handles.into_iter().enumerate() {
        match handle.await {
            Ok(()) => println!("  ✅ 服务 {} 已安全退出", i),
            Err(e) => println!("  ❌ 服务 {} 异常: {}", i, e),
        }
    }

    println!("\n  🏁 所有服务已优雅关闭");
    Ok(())
}
```

### CancellationToken 详解

```
CancellationToken 的工作原理：

                    token = CancellationToken::new()
                           │
                ┌──────────┼──────────┐
                ▼          ▼          ▼
          token.clone() token.clone() token.clone()
               │          │          │
               ▼          ▼          ▼
         ┌─────────┐ ┌─────────┐ ┌─────────┐
         │ Service1 │ │ Service2 │ │ Service3 │
         │          │ │          │ │          │
         │ select!  │ │ select!  │ │ select!  │
         │  cancel  │ │  cancel  │ │  cancel  │
         │  work    │ │  work    │ │  work    │
         └─────────┘ └─────────┘ └─────────┘
                           ▲
                           │
                    token.cancel()
                    (一次取消所有！)

特性：
  · clone() 是轻量的（内部是 Arc）
  · cancel() 只需调用一次，所有 clone 都会收到通知
  · cancelled().await 在 cancel() 之前会一直等待
  · 如果先 cancel() 后 cancelled().await → 立即返回
  · 可以用 child_token() 创建子令牌（取消父 → 取消子）
```

### child_token() 层级取消

```rust
// 父子令牌：取消父 → 自动取消所有子
let root = CancellationToken::new();

// 子令牌：Gateway 下面有多个子服务
let gateway_token = root.child_token();
let http_token = gateway_token.child_token();
let ws_token = gateway_token.child_token();

// 取消 root → gateway_token, http_token, ws_token 全部取消
// 取消 gateway_token → http_token, ws_token 取消，但 root 不受影响

// 层级结构：
//   root
//   └── gateway_token
//       ├── http_token
//       └── ws_token
//
// 前端类比：DOM 事件冒泡的反向——父节点 abort 会传播到所有子节点
```

## 4. 方法3：Drop Channel 模式（自然关闭）

当所有 `Sender` 被 drop 后，`Receiver` 的 `recv()` 会返回 `None`，这是 Actor 最自然的关闭方式。

```rust
use std::time::Duration;
use tokio::sync::mpsc;
use anyhow::Result;

/// 通过 channel 关闭的 Actor 服务
async fn channel_service(name: &str, mut rx: mpsc::Receiver<String>) {
    println!("  🎬 [{}] 启动，等待消息...", name);

    // 当所有 tx 被 drop 后，recv() 返回 None → 循环结束
    while let Some(msg) = rx.recv().await {
        println!("  📩 [{}] 收到: {}", name, msg);
        tokio::time::sleep(Duration::from_millis(100)).await;
    }

    // rx.recv() 返回 None = 所有 Sender 已断开 = 该关闭了
    println!("  🧹 [{}] 通道已关闭，执行清理...", name);
    tokio::time::sleep(Duration::from_millis(200)).await;
    println!("  ✅ [{}] 清理完成", name);
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ Drop Channel 模式：自然关闭 ═══\n");

    let (tx1, rx1) = mpsc::channel::<String>(32);
    let (tx2, rx2) = mpsc::channel::<String>(32);

    // 启动服务
    let h1 = tokio::spawn(channel_service("ServiceA", rx1));
    let h2 = tokio::spawn(channel_service("ServiceB", rx2));

    // 发送一些消息
    tx1.send("Hello A".to_string()).await?;
    tx2.send("Hello B".to_string()).await?;
    tokio::time::sleep(Duration::from_millis(200)).await;

    tx1.send("Work A".to_string()).await?;
    tx2.send("Work B".to_string()).await?;
    tokio::time::sleep(Duration::from_millis(200)).await;

    println!("\n  ⚠️  Drop 所有 Sender → 服务自然关闭\n");

    // drop Sender → 服务收到 None → 执行清理 → 退出
    drop(tx1);
    drop(tx2);

    // 等待服务完成清理
    h1.await?;
    h2.await?;

    println!("\n  🏁 所有服务已自然关闭");
    Ok(())
}
```

### Drop Channel 的优势

```
Drop Channel 模式的优势：

1. 最符合 Rust 的 RAII 哲学——资源的生命周期自动管理
2. 不需要额外的 CancellationToken 或 abort()
3. Actor 模式天然支持——所有 Handle drop 后 Actor 自动退出
4. 清理代码在 while let 之后自然执行

适用场景：
  · Actor 模式（Handle drop → Actor 退出）
  · 生产者-消费者（生产者结束 → 消费者知道没有更多数据了）
  · 不需要"立即停止"的场景

不适用：
  · 需要立即停止的场景（Drop channel 要等当前消息处理完）
  · 有多个 Sender 时（需要所有 Sender 都 drop 才会触发关闭）
```

## 5. 完整示例：多服务优雅关闭（模拟 ZeroClaw Daemon）

把三种方式组合使用，实现一个接近 ZeroClaw Daemon 的完整关闭流程。

```rust
use std::time::Duration;
use tokio::sync::mpsc;
use tokio::task::JoinHandle;
use tokio_util::sync::CancellationToken;
use anyhow::Result;

// ═══════════════════════════════════════════════════════
// 模拟 ZeroClaw 的各个组件
// ═══════════════════════════════════════════════════════

/// Gateway 服务：处理 HTTP 请求
/// 使用 CancellationToken 优雅关闭
async fn gateway_service(token: CancellationToken) {
    println!("  🌐 [Gateway] 启动，监听 HTTP 请求...");
    let mut request_count = 0u64;

    loop {
        tokio::select! {
            () = token.cancelled() => {
                println!("  🌐 [Gateway] 收到关闭信号");
                println!("  🌐 [Gateway] 完成最后的请求处理...");
                tokio::time::sleep(Duration::from_millis(200)).await;
                println!("  🌐 [Gateway] 已处理 {} 个请求，安全退出", request_count);
                return;
            }

            _ = tokio::time::sleep(Duration::from_millis(800)) => {
                request_count += 1;
                println!("  🌐 [Gateway] 处理请求 #{}", request_count);
            }
        }
    }
}

/// Channel 服务：监听消息平台
/// 使用 CancellationToken + interval（对应 ZeroClaw 的 typing 指示器）
async fn channel_service(name: &str, token: CancellationToken) {
    println!("  📡 [Channel:{}] 启动，监听消息...", name);

    let mut interval = tokio::time::interval(Duration::from_secs(1));
    interval.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Skip);

    let mut msg_count = 0u64;

    loop {
        tokio::select! {
            // 对应 ZeroClaw 源码中的：
            //   () = stop_signal.cancelled() => break,
            () = token.cancelled() => {
                println!("  📡 [Channel:{}] 收到关闭信号", name);
                println!("  📡 [Channel:{}] 发送离线状态...", name);
                tokio::time::sleep(Duration::from_millis(150)).await;
                println!("  📡 [Channel:{}] 已处理 {} 条消息，安全退出", name, msg_count);
                return;
            }

            // 对应 ZeroClaw 的 interval.tick()
            _ = interval.tick() => {
                msg_count += 1;
                println!("  📡 [Channel:{}] 检查新消息... #{}", name, msg_count);
            }
        }
    }
}

/// StateWriter 服务：定期保存状态
/// 使用 mpsc channel 接收数据 + CancellationToken 关闭
async fn state_writer_service(
    mut rx: mpsc::Receiver<String>,
    token: CancellationToken,
) {
    println!("  💾 [StateWriter] 启动，等待数据...");
    let mut buffer: Vec<String> = Vec::new();

    loop {
        tokio::select! {
            () = token.cancelled() => {
                // 关闭时 flush 缓冲区中的所有数据
                println!("  💾 [StateWriter] 收到关闭信号");
                if !buffer.is_empty() {
                    println!("  💾 [StateWriter] Flush 缓冲区 ({} 条数据)...", buffer.len());
                    for item in &buffer {
                        println!("    💾 保存: {}", item);
                    }
                    tokio::time::sleep(Duration::from_millis(300)).await;
                    println!("  💾 [StateWriter] 所有数据已保存");
                } else {
                    println!("  💾 [StateWriter] 缓冲区为空，无需保存");
                }
                return;
            }

            // 接收数据
            data = rx.recv() => {
                match data {
                    Some(item) => {
                        println!("  💾 [StateWriter] 收到数据: {}", item);
                        buffer.push(item);

                        // 缓冲区满 3 条就 flush（简化版）
                        if buffer.len() >= 3 {
                            println!("  💾 [StateWriter] 缓冲区满，执行 flush...");
                            tokio::time::sleep(Duration::from_millis(100)).await;
                            buffer.clear();
                            println!("  💾 [StateWriter] Flush 完成");
                        }
                    }
                    None => {
                        // 所有 Sender 已断开
                        println!("  💾 [StateWriter] 数据通道已关闭");
                        return;
                    }
                }
            }
        }
    }
}

/// 模拟数据产生器（定期向 StateWriter 发送数据）
async fn data_producer(tx: mpsc::Sender<String>, token: CancellationToken) {
    let mut seq = 0u64;

    loop {
        tokio::select! {
            () = token.cancelled() => {
                println!("  📊 [DataProducer] 停止产生数据");
                return;
            }

            _ = tokio::time::sleep(Duration::from_millis(600)) => {
                seq += 1;
                let data = format!("data_#{}", seq);
                if tx.send(data.clone()).await.is_err() {
                    println!("  📊 [DataProducer] StateWriter 已关闭");
                    return;
                }
                println!("  📊 [DataProducer] 产生: {}", data);
            }
        }
    }
}

#[tokio::main]
async fn main() -> Result<()> {
    println!("═══ 完整示例：ZeroClaw 风格优雅关闭 ═══\n");

    // ─── 创建全局取消令牌 ───
    let token = CancellationToken::new();

    // ─── StateWriter 的数据通道 ───
    let (state_tx, state_rx) = mpsc::channel::<String>(32);

    // ─── 启动所有服务 ───
    let mut handles: Vec<(&str, JoinHandle<()>)> = vec![];

    handles.push(("Gateway", tokio::spawn(
        gateway_service(token.clone())
    )));

    handles.push(("Channel:Discord", tokio::spawn(
        channel_service("Discord", token.clone())
    )));

    handles.push(("Channel:Slack", tokio::spawn(
        channel_service("Slack", token.clone())
    )));

    handles.push(("StateWriter", tokio::spawn(
        state_writer_service(state_rx, token.clone())
    )));

    handles.push(("DataProducer", tokio::spawn(
        data_producer(state_tx.clone(), token.clone())
    )));

    // Drop 主函数中的 state_tx clone
    // 这样当 DataProducer 退出后，StateWriter 的 rx 也会收到 None
    drop(state_tx);

    println!("  🧠 ZeroClaw Daemon 已启动（{} 个组件）", handles.len());
    println!("  按 Ctrl+C 优雅关闭...\n");

    // ─── 等待关闭信号 ───
    tokio::signal::ctrl_c().await?;

    println!("\n╔═══════════════════════════════════════════╗");
    println!("║  ⚠️  收到 Ctrl+C，开始优雅关闭流程...      ║");
    println!("╚═══════════════════════════════════════════╝\n");

    // ─── 触发取消 ───
    let shutdown_start = std::time::Instant::now();
    token.cancel();

    // ─── 等待所有组件完成清理 ───
    println!("  ⏳ 等待所有组件完成清理...\n");

    for (name, handle) in handles {
        match handle.await {
            Ok(()) => println!("  ✅ {} 已安全退出", name),
            Err(e) if e.is_cancelled() => println!("  🛑 {} 已取消", name),
            Err(e) => println!("  ❌ {} 异常退出: {}", name, e),
        }
    }

    let elapsed = shutdown_start.elapsed();
    println!("\n╔═══════════════════════════════════════════╗");
    println!("║  🏁 优雅关闭完成（耗时 {:?}）            ║", elapsed);
    println!("╚═══════════════════════════════════════════╝");

    Ok(())
}
```

## 6. ZeroClaw 的关闭流程分析

### Daemon 的 abort 模式

```rust
// ZeroClaw 实际源码（简化版）

pub async fn run(config: Config, host: String, port: u16) -> Result<()> {
    // 启动所有组件
    let mut handles: Vec<JoinHandle<()>> = vec![
        spawn_state_writer(config.clone()),
    ];
    handles.push(spawn_component_supervisor("gateway", ...));
    handles.push(spawn_component_supervisor("channels", ...));
    handles.push(spawn_component_supervisor("heartbeat", ...));
    handles.push(spawn_component_supervisor("scheduler", ...));

    println!("🧠 ZeroClaw daemon started");

    // 等待 Ctrl+C
    tokio::signal::ctrl_c().await?;

    // 简单粗暴：abort 所有组件
    for handle in &handles {
        handle.abort();
    }

    // 等待所有组件结束
    for handle in handles {
        let _ = handle.await;
    }

    Ok(())
}
```

### 实际项目中的最佳实践

```
推荐的组合方式（也是 ZeroClaw 的实际做法）：

1. 全局关闭信号：CancellationToken
   let token = CancellationToken::new();
   tokio::signal::ctrl_c().await?;
   token.cancel();  // 通知所有服务开始关闭

2. 每个服务内部：select! + token.cancelled()
   loop {
       tokio::select! {
           () = token.cancelled() => { cleanup(); return; }
           _ = do_work() => {}
       }
   }

3. 等待超时保护：如果服务 10 秒还没退出，强制 abort
   match tokio::time::timeout(
       Duration::from_secs(10),
       handle.await
   ) {
       Ok(_) => println!("正常退出"),
       Err(_) => {
           handle.abort();  // 超时兜底
           println!("强制取消");
       }
   }

这样你就得到了：
  · 正常情况：CancellationToken → 优雅清理 → 安全退出 ✅
  · 异常情况：超时 → abort 兜底 → 至少不会卡死 ✅
```

        cleanup(),
        new Promise((_, reject) =>

## 9. 速查表

```
信号监听：
  tokio::signal::ctrl_c().await?;          ← 等待 Ctrl+C

abort 关闭：
  handle.abort();                          ← 标记任务取消
  handle.await                             ← 等待任务结束
  Err(e) if e.is_cancelled()               ← 判断是否被取消

CancellationToken：
  let token = CancellationToken::new();    ← 创建令牌
  token.clone()                            ← 复制给各服务
  token.child_token()                      ← 创建子令牌
  token.cancel();                          ← 触发取消
  token.cancelled().await;                 ← 等待取消信号

select! + 取消：
  tokio::select! {
      () = token.cancelled() => { cleanup(); return; }
      result = do_work() => { process(result); }
  }

超时兜底：
  match tokio::time::timeout(dur, handle).await {
      Ok(_)  => { /* 正常退出 */ }
      Err(_) => { handle.abort(); /* 强制取消 */ }
  }

推荐组合：
  ctrl_c().await → token.cancel() → 等待清理 → timeout 兜底 abort
```

### 一句话总结

> **优雅关闭 = 信号监听 + 取消传播 + 清理执行**——用 `tokio::signal::ctrl_c()` 捕获关闭信号，用 `CancellationToken` 广播给所有服务，每个服务在 `select!` 中检测取消并执行清理。ZeroClaw 用 `abort()` 做简单关闭、用 `stop_signal.cancelled()` 做优雅关闭。生产环境推荐 CancellationToken + timeout 兜底 abort 的组合：正常走优雅路径，超时走强制路径，确保服务永远不会卡住。
