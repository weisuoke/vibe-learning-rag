# 核心概念 4：日志与调试

## 概述

日志与调试是 OpenClaw Gateway 运维和故障排查的核心工具，通过多层次的日志系统（控制台日志、文件日志、会话转录）和丰富的调试选项，帮助开发者快速定位和解决问题。

## 一句话定义

**日志与调试是 Gateway 运行状态监控和问题诊断的机制，通过控制台日志（实时反馈）、文件日志（持久化记录）、会话转录（完整历史）和多级日志等级（error/warn/info/debug/trace），实现从开发调试到生产监控的全方位可观测性。**

## 第一性原理：为什么需要日志系统？

### 问题根源

任何运行中的系统都需要回答三个基本问题：

1. **系统在做什么？**（当前状态）
2. **系统做了什么？**（历史记录）
3. **系统为什么这样做？**（决策过程）

没有日志系统，这些问题无法回答，问题排查变成盲人摸象。

### 解决方案

OpenClaw 通过分层的日志系统解决这个问题：
- **控制台日志**：实时反馈当前状态
- **文件日志**：持久化历史记录
- **会话转录**：完整的对话和决策过程
- **日志等级**：控制详细程度

## 日志类型

### 1. 控制台日志（Console Logs）

**定义：** 实时输出到 stdout/stderr 的日志。

**特征：**
- 实时可见
- 彩色输出（TTY）
- 时间戳前缀
- 子系统标识

**查看方式：**
```bash
# 前台启动时自动显示
openclaw gateway --verbose

# 输出示例：
# [2026-02-22T06:45:58.288Z] [gateway] Gateway starting on port 18789
# [2026-02-22T06:45:58.289Z] [gateway] Bind mode: loopback
# [2026-02-22T06:45:58.291Z] [gateway] Gateway ready
```

**使用场景：**
- 开发调试
- 实时监控
- 快速问题定位

### 2. 文件日志（File Logs）

**定义：** 写入磁盘的结构化日志（JSON Lines 格式）。

**位置：**
```
~/.openclaw/logs/gateway.log
~/.openclaw/logs/gateway-YYYY-MM-DD.log
```

**格式：**
```json
{
  "timestamp": "2026-02-22T06:45:58.288Z",
  "level": "info",
  "subsystem": "gateway",
  "message": "Gateway starting on port 18789",
  "context": {
    "port": 18789,
    "bind": "loopback"
  }
}
```

**查看方式：**
```bash
# 使用 OpenClaw CLI
openclaw logs
openclaw logs --follow
openclaw logs --tail 100

# 直接查看文件
tail -f ~/.openclaw/logs/gateway.log
```

**使用场景：**
- 生产监控
- 历史分析
- 自动化解析

### 3. 会话转录（Session Transcripts）

**定义：** 完整的对话历史记录。

**位置：**
```
~/.openclaw/agents/<agentId>/sessions/<sessionKey>/transcript.jsonl
```

**内容：**
- 用户消息
- Agent 响应
- 工具调用
- 模型请求/响应

**查看方式：**
```bash
# 查看会话转录
cat ~/.openclaw/agents/main/sessions/<sessionKey>/transcript.jsonl | jq

# 提取工具调用
cat transcript.jsonl | jq 'select(.type == "tool_use")'
```

**使用场景：**
- 调试 Agent 行为
- 审计工具使用
- 分析对话流程

### 4. 配置审计日志（Config Audit Log）

**定义：** 配置变更历史记录。

**位置：**
```
~/.openclaw/logs/config-audit.jsonl
```

**内容：**
- 配置变更时间
- 变更前后对比
- 验证错误

**使用场景：**
- 追踪配置变更
- 调试配置问题
- 安全审计

## 日志等级

### 等级层次

```
trace (最详细)
  ↓
debug
  ↓
info (默认)
  ↓
warn
  ↓
error (最简洁)
```

### 1. error

**用途：** 错误条件，需要立即关注。

**示例：**
- Gateway 启动失败
- 端口已被占用
- 配置验证错误
- 通道连接失败

**配置：**
```json5
{
  logging: { level: "error" }
}
```

### 2. warn

**用途：** 警告条件，可能需要关注。

**示例：**
- 安全审计警告
- 废弃配置选项
- 性能问题
- 非关键失败

### 3. info（默认）

**用途：** 信息性消息。

**示例：**
- Gateway 启动
- 通道连接
- 会话创建
- 配置重载

### 4. debug

**用途：** 详细调试信息。

**示例：**
- WebSocket 消息
- 工具调用
- 模型请求/响应
- 内部状态变化

**启用方式：**
```bash
openclaw gateway --verbose
```

### 5. trace

**用途：** 非常详细的跟踪信息。

**示例：**
- 底层协议细节
- 原始消息载荷
- 内部函数调用
- 性能指标

## CLI 日志命令

### 查看日志

```bash
# 查看最近日志
openclaw logs

# 实时跟踪
openclaw logs --follow

# 查看最后 N 行
openclaw logs --tail 100

# 查看特定日期
openclaw logs --date 2026-02-22

# JSON 格式输出
openclaw logs --json
```

### Gateway 启动选项

```bash
# 详细日志（debug 级别）
openclaw gateway --verbose

# WebSocket 日志样式
openclaw gateway --ws-log full      # 完整消息
openclaw gateway --ws-log compact   # 紧凑格式
openclaw gateway --ws-log auto      # 自动选择

# 紧凑模式（别名）
openclaw gateway --compact

# 只显示 Claude CLI 日志
openclaw gateway --claude-cli-logs

# 原始流日志
openclaw gateway --raw-stream
openclaw gateway --raw-stream-path /path/to/stream.jsonl
```

## 日志配置

### 基础配置

```json5
{
  logging: {
    level: "info",              // error | warn | info | debug | trace
    redactSensitive: true,      // 脱敏敏感数据
    console: {
      enabled: true,
      timestamp: true,
      subsystem: true,
      color: true
    },
    file: {
      enabled: true,
      path: "~/.openclaw/logs/gateway.log",
      maxSize: "100MB",
      maxFiles: 10,
      rotation: "daily"
    }
  }
}
```

### 高级配置

```json5
{
  logging: {
    level: "debug",
    redactSensitive: true,
    subsystems: {
      gateway: "info",
      channels: "debug",
      agent: "debug",
      tools: "trace"
    },
    console: {
      enabled: true,
      filter: ["gateway", "channels"]  // 只显示特定子系统
    },
    file: {
      enabled: true,
      format: "json",              // json | text
      maxSize: "100MB",
      maxFiles: 10,
      rotation: "daily",           // daily | size | none
      compress: true
    }
  }
}
```

### 敏感数据脱敏

```json5
{
  logging: {
    redactSensitive: true,
    redactPatterns: [
      "password",
      "token",
      "apiKey",
      "secret"
    ]
  }
}
```

**效果：**
```json
// 脱敏前
{"auth": {"token": "sk-ant-1234567890"}}

// 脱敏后
{"auth": {"token": "[REDACTED]"}}
```

## 调试技巧

### 1. 实时日志监控

```bash
# 跟踪日志
openclaw logs --follow

# 过滤错误
openclaw logs --follow | grep "error"

# 过滤特定子系统
openclaw gateway --verbose --claude-cli-logs
```

### 2. WebSocket 消息调试

```bash
# 完整 WebSocket 日志
openclaw gateway --ws-log full

# 紧凑 WebSocket 日志
openclaw gateway --ws-log compact
```

### 3. 原始流日志

```bash
# 记录原始模型流事件
openclaw gateway --raw-stream

# 自定义路径
openclaw gateway --raw-stream-path ~/debug/stream.jsonl
```

### 4. 会话转录分析

```bash
# 查看会话转录
cat ~/.openclaw/agents/main/sessions/<sessionKey>/transcript.jsonl | jq

# 提取工具调用
cat transcript.jsonl | jq 'select(.type == "tool_use")'

# 提取模型响应
cat transcript.jsonl | jq 'select(.type == "text")'
```

### 5. 配置审计分析

```bash
# 查看配置变更
cat ~/.openclaw/logs/config-audit.jsonl | jq

# 查找最近变更
cat config-audit.jsonl | jq 'select(.timestamp > "2026-02-22")'

# 查找验证错误
cat config-audit.jsonl | jq 'select(.error != null)'
```

## 常见调试场景

### 场景 1：Gateway 无法启动

```bash
# 1. 查看日志
openclaw logs --tail 50

# 2. 运行诊断
openclaw doctor

# 3. 检查端口占用
lsof -i :18789

# 4. 前台启动查看详细日志
openclaw gateway --verbose
```

### 场景 2：通道连接问题

```bash
# 1. 检查通道状态
openclaw channels status

# 2. 查看通道日志
openclaw logs --follow | grep "channel"

# 3. 详细日志启动
openclaw gateway --verbose
```

### 场景 3：Agent 无响应

```bash
# 1. 查看会话转录
cat ~/.openclaw/agents/main/sessions/<sessionKey>/transcript.jsonl | jq

# 2. 查看工具调用
openclaw logs --follow | grep "tool"

# 3. 原始流日志
openclaw gateway --verbose --raw-stream
```

### 场景 4：性能问题

```bash
# 1. 启用 trace 日志
openclaw gateway --verbose

# 2. 监控资源使用
top -pid $(pgrep -f "openclaw gateway")

# 3. 检查会话大小
du -sh ~/.openclaw/agents/main/sessions/*
```

### 场景 5：安全审计失败

```bash
# 1. 运行安全审计
openclaw security audit

# 2. 查看审计日志
cat ~/.openclaw/logs/audit.jsonl | jq

# 3. 检查敏感数据泄露
openclaw logs | grep -i "redacted"
```

## 故障排查流程

### 标准流程

```
1. 查看最近日志
   ↓
2. 识别错误模式
   ↓
3. 提升日志等级（debug/trace）
   ↓
4. 重现问题
   ↓
5. 分析详细日志
   ↓
6. 定位根因
   ↓
7. 修复问题
   ↓
8. 验证修复
```

### 快速诊断命令

```bash
# 一键诊断
openclaw doctor

# 安全审计
openclaw security audit

# 健康检查
openclaw gateway health

# 状态检查
openclaw gateway status --deep
```
