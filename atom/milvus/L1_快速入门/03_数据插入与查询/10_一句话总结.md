# 一句话总结

> **用一句话记住数据插入与查询的核心**

---

## 核心总结

**Milvus 2.6 通过 Embedding Functions 让你直接插入和查询原始文本，数据库自动处理向量化，将传统5步流程简化为3步，让RAG开发回归本质——专注数据而非嵌入。**

---

## 拆解记忆

### 关键词提取
1. **Milvus 2.6** - 版本特性
2. **Embedding Functions** - 核心功能
3. **原始文本** - 输入输出
4. **自动向量化** - 核心价值
5. **5步→3步** - 效率提升
6. **专注数据** - 设计哲学

### 记忆口诀
```
文本进，文本出，
向量化，库来做，
五步变三步，
RAG更简单。
```

---

## 三个核心认知

### 1. 是什么？
**Data-in, Data-out 模式** - 输入输出都是原始数据，向量化对用户透明

### 2. 为什么？
**职责归位** - 嵌入生成是数据库的固有职责，不应外包给用户

### 3. 怎么做？
**Embedding Functions** - 在collection schema中配置一次，全局自动生效

## 核心API速记

```python
# 1. 配置（创建collection时）
schema.add_function(
    Function(
        name="embedding",
        provider="openai",
        model="text-embedding-3-small"
    )
)

# 2. 插入（自动嵌入）
client.insert({"document": "原始文本"})

# 3. 查询（自动嵌入）
results = client.search(["查询文本"])
```
