# 核心概念 2：不可变借用（&T）

> **一句话定位**：不可变借用 `&T` 是 Rust 中"只读访问权"的表达 -- 你可以看但不能改，而且编译器保证你看的东西在看的期间不会被任何人偷偷修改。

---

## 1. 什么是不可变借用

### 1.1 借而不改：&T 的含义

在 Rust 中，`&T` 表示对类型 `T` 的一个**不可变引用**（immutable reference）。它的语义是：

- **借**：你不拥有这个值，只是暂时借来用
- **不改**：你承诺在借用期间不会修改这个值

```rust
fn print_name(name: &String) {
    println!("Hello, {}!", name);
    // name.push_str("!"); // 编译错误！不可变引用不能修改值
}

fn main() {
    let my_name = String::from("ZeroClaw");
    print_name(&my_name);       // 借出去，用 & 创建引用
    println!("{}", my_name);     // 借完了，原变量依然有效
}
```

关键操作符：
- `&my_name`：创建一个不可变引用（"借出去"）
- `name: &String`：函数参数接收一个不可变引用（"借进来"）

### 1.2 与 TypeScript 的 readonly 对比

```typescript
// TypeScript: readonly 是"君子协定"
function printConfig(config: Readonly<Config>) {
    // config.host = "evil";  // TypeScript 报错
    (config as any).host = "evil";  // 但运行时可以绕过
}
```

```rust
// Rust: &T 是编译器强制的"铁律"
fn print_config(config: &String) {
    // 没有任何合法方式通过 &T 修改值，不存在 "as any" 逃逸手段
    println!("{}", config);
}
```

| 维度 | TypeScript `readonly` | Rust `&T` |
|------|----------------------|-----------|
| 强制级别 | 类型检查，可绕过 | 编译器硬约束，无法绕过 |
| 并发安全 | 不保证 | 保证（多个 `&T` 可安全并发读） |
| 运行时开销 | 零 | 零 |

### 1.3 引用 vs 指针的区别

Rust 的引用有三个关键保证：永远非空、永远指向有效内存、永远对齐。

| 性质 | C/C++ 指针 | Rust 引用 `&T` | Rust 裸指针 `*const T` |
|------|-----------|----------------|----------------------|
| 可为 null | 是 | 否 | 是 |
| 悬垂（dangling） | 可能 | 编译器阻止 | 可能 |
| 需要手动解引用 | 是 (`*ptr`) | 通常自动 | 是 (`unsafe { *ptr }`) |
| 编译期安全检查 | 无 | 有（借用检查器） | 无（需 unsafe） |

## 2. 不可变借用的规则

### 2.1 规则一：可以同时存在多个 &T

因为大家都只是"看"，所以多少人同时看都没问题：

```rust
fn main() {
    let data = String::from("ZeroClaw Agent Runtime");
    let r1 = &data;
    let r2 = &data;
    let r3 = &data;
    println!("r1: {}, r2: {}, r3: {}", r1, r2, r3); // 全部合法
}
```

### 2.2 规则二：引用必须总是有效（不能悬垂）

```rust
// 编译错误：悬垂引用
fn create_dangling() -> &String {     // 错误！返回了局部变量的引用
    let s = String::from("hello");
    &s  // s 在函数结束时被释放，引用将指向无效内存
}

// 正确做法：返回所有权
fn create_owned() -> String {
    let s = String::from("hello");
    s   // 移动所有权给调用方
}
```

### 2.3 规则三：借用期间所有者不能修改值

```rust
fn main() {
    let mut data = String::from("hello");
    let r = &data;              // 创建不可变引用
    // data.push_str(" world"); // 编译错误！data 被借用期间不能修改
    println!("{}", r);          // r 的最后一次使用

    data.push_str(" world");    // OK！r 不再使用，借用已结束（NLL）
    println!("{}", data);       // "hello world"
}
```

> **为什么这很重要？** 想象你正在遍历一个 Vec，如果遍历过程中有人偷偷往里面加元素，可能导致内存重新分配，迭代器指向无效内存。Rust 在编译期就禁止这种情况。

### 2.4 互斥访问原则

三条规则归纳为一个核心原则：**在任意时刻，要么有任意数量的 `&T`，要么有恰好一个 `&mut T`，但不能同时拥有两种。**

```rust
fn main() {
    let mut s = String::from("hello");

    let r1 = &s;
    let r2 = &s;
    println!("{}, {}", r1, r2);     // 多个 &T -- OK

    let r3 = &mut s;                // 一个 &mut T -- OK（r1, r2 已不再使用）
    r3.push_str(" world");
    println!("{}", r3);
}
```

## 3. 常见的引用类型

### 3.1 &str vs &String

```rust
// 不推荐：只接受 &String
fn greet_narrow(name: &String) {
    println!("Hello, {}!", name);
}

// 推荐：接受 &str（更通用）
fn greet_wide(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    let owned = String::from("ZeroClaw");
    let literal = "World";

    greet_narrow(&owned);     // OK
    // greet_narrow(literal);  // 编译错误！&str 不是 &String

    greet_wide(&owned);       // OK -- String 自动 Deref 为 &str
    greet_wide(literal);      // OK -- &str 本身
    greet_wide(&owned[0..4]); // OK -- 切片也是 &str
}
```

**为什么推荐 `&str`？** 它能接受 `&String`、字符串字面量、字符串切片，接受面最广。类比 TypeScript：就像用 `string` 而不用 `String` 包装类型。

### 3.2 &[T] 切片引用

`&[T]` 是对数组或 Vec 的"窗口视图"，不拥有数据：

```rust
fn sum(numbers: &[i32]) -> i32 {
    numbers.iter().sum()
}

fn main() {
    let vec = vec![1, 2, 3, 4, 5];
    println!("全部: {}", sum(&vec));        // Vec<i32> -> &[i32]
    println!("前三: {}", sum(&vec[0..3]));  // 切片：&[1, 2, 3]

    let array = [10, 20, 30];
    println!("数组: {}", sum(&array));      // [i32; 3] -> &[i32]
}
```

### 3.3 &Path 与 &dyn Trait

```rust
use std::path::{Path, PathBuf};

// &Path：文件路径的引用，类似 &str 对 String 的关系
fn read_config(path: &Path) -> std::io::Result<String> {
    std::fs::read_to_string(path)
}

// &dyn Trait：trait 对象引用，用于运行时多态
fn print_greeting(greeter: &dyn Greet) {
    println!("{}", greeter.hello());
}
// 类比 TS：就像函数参数声明为接口类型 (greeter: Greeter)
```

### 3.4 引用类型速查表

| 引用类型 | 对应所有权类型 | 类比 TypeScript | 使用场景 |
|----------|--------------|----------------|----------|
| `&str` | `String` | `string` | 函数参数中的字符串 |
| `&[T]` | `Vec<T>` | `readonly T[]` | 函数参数中的数组/列表 |
| `&Path` | `PathBuf` | `string`（路径） | 文件路径参数 |
| `&dyn Trait` | `Box<dyn Trait>` | 接口类型参数 | 运行时多态 |

## 4. 解引用（Deref）

### 4.1 自动解引用（Deref Coercion）

Rust 编译器会自动沿着 Deref trait 链转换类型，不需要手动写 `*` 和 `&`：

```rust
fn takes_str(s: &str) { println!("{}", s); }

fn main() {
    let owned = String::from("ZeroClaw");
    takes_str(&owned);  // 编译器自动：&String -> &str（Deref coercion）
}
```

### 4.2 常见的自动转换链

```rust
// String  -> &str      : print_str(&my_string)
// Vec<T>  -> &[T]      : print_slice(&my_vec)
// PathBuf -> &Path     : read_config(&my_pathbuf)
// Box<T>  -> &T        : print_num(&my_box)
// Box<String> -> &str  : 两层 Deref，全自动
```

> **TypeScript 类比**：有点像 JS 的隐式类型转换，但 Rust 的 Deref coercion 是**完全安全且可预测的** -- 只沿着 Deref trait 实现链转换，不会出现 `[] + {} = "[object Object]"` 这种惊喜。

## 5. 非词法生命周期（NLL）

### 5.1 什么是 NLL

NLL（Non-Lexical Lifetimes）让引用的生命周期只延伸到**最后一次使用的位置**，而非到代码块末尾：

```rust
fn main() {
    let mut data = String::from("hello");

    // Rust 2015（非 NLL）：r 的生命周期到 } 才结束，下面会编译错误
    // Rust 2018+（NLL）：r 的生命周期到 println! 就结束了
    let r = &data;
    println!("{}", r);          // r 最后一次使用 <- 生命周期到这里结束

    data.push_str(" world");    // OK！r 已经不再使用
    println!("{}", data);       // "hello world"
}
```

### 5.2 NLL 让更多代码编译通过

```rust
fn main() {
    let mut scores = vec![85, 92, 78, 95, 88];

    // 不可变借用：查找最高分位置
    let max_idx = scores
        .iter()
        .enumerate()
        .max_by_key(|&(_, &score)| score)
        .map(|(idx, _)| idx)
        .unwrap();      // 不可变借用到这里结束（NLL）

    // 可变借用：修改分数
    scores[max_idx] += 10;  // OK！
    println!("更新后: {:?}", scores);
}
```

```rust
    let resolved = resolve_embedding_config(config, embedding_routes, api_key);

```

**学习要点**：
- 五个参数全是不可变借用 -- 工厂函数只需"读取"配置信息
- `&Path` 而非 `&PathBuf`，`Option<&str>` 而非 `Option<&String>` -- 用切片类型更通用
- `&[]` -- 空切片引用字面量

### 6.2 find_tool：生命周期与切片引用的结合

来源：`sourcecode/zeroclaw/src/agent/loop_.rs`

```rust
/// 在工具注册表中按名称查找工具
fn find_tool<'a>(tools: &'a [Box<dyn Tool>], name: &str) -> Option<&'a dyn Tool> {
    tools.iter().find(|t| t.name() == name).map(|t| t.as_ref())
}
```

**逐字拆解**：
- `tools: &'a [Box<dyn Tool>]`：借用工具列表，生命周期标注为 `'a`
- `name: &str`：名称只用来比较，不出现在返回值中，不需要标注生命周期
- `Option<&'a dyn Tool>`：返回的引用生命周期绑定到 `tools` -- 编译器保证返回的引用不会比 tools 活得更久

```typescript
// TypeScript 对比 -- 不需要考虑生命周期
function findTool(tools: Tool[], name: string): Tool | undefined {
    return tools.find(t => t.name === name);
}
```

## 7. 与 TypeScript 对比总结表

| 概念 | TypeScript | Rust |
|------|-----------|------|
| **只读标记** | `readonly` / `Readonly<T>` | `&T`（不可变引用） |
| **强制程度** | 类型检查，可用 `as any` 绕过 | 编译器硬约束，无法绕过 |
| **多个只读引用** | 默认行为（所有变量都是引用） | 显式创建 `&T`，可以有多个 |
| **读写冲突** | 运行时可能出现，不检查 | 编译期禁止（&T 和 &mut T 不能共存） |
| **悬垂引用** | GC 防止 | 编译器防止（借用检查器） |
| **null 引用** | 可能（`undefined`/`null`） | 不可能（`&T` 保证非空） |
| **函数参数** | `fn(x: string)` | `fn(x: &str)` |
| **数组参数** | `fn(arr: readonly number[])` | `fn(arr: &[i32])` |
| **自动转换** | 隐式类型转换（有时意外） | Deref coercion（安全、可预测） |
| **作用域** | GC 管理，不需要关心 | NLL -- 到最后使用处结束 |

## 8. 常见陷阱与解决方案

### 8.1 在借用期间尝试修改

```rust
fn main() {
    let mut names = vec!["Alice", "Bob", "Charlie"];

    // 错误：在迭代（不可变借用）期间修改 Vec
    // for name in &names {
    //     if *name == "Bob" { names.push("Dave"); }  // 编译错误！
    // }

    // 解决：先收集信息，再修改
    let should_add = names.iter().any(|&name| name == "Bob");
    if should_add { names.push("Dave"); }  // OK！迭代已结束
}
```

### 8.2 函数签名过于限制

```rust
// 不好                          // 好
fn process(data: &String) {}    fn process(data: &str) {}
fn analyze(items: &Vec<i32>) {} fn analyze(items: &[i32]) {}
```

### 8.3 不必要的 clone

```rust
fn check_config(config: &str) { println!("Config: {}", config); }

fn main() {
    let config = String::from("debug=true");
    // let copy = config.clone(); check_config(&copy);  // 不好：多余的克隆
    check_config(&config);  // 好：直接借用，零拷贝
}
```

## 9. 速记口诀

```
不可变借用记三条：
  1. 多个 &T 同时借，大家只读都安全
  2. 借用期间禁修改，编译检查不放行
  3. 引用必须总有效，悬垂指针编译拦

函数参数选类型：
  String 参数用 &str，Vec 参数用 &[T]
  PathBuf 参数用 &Path，Deref 转换编译器帮

NLL 让代码更自然：
  引用到最后使用处，不用等到块末尾
```
