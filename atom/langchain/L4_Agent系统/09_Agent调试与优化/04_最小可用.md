### 4.1 一键开启调试模式

**场景**：Agent 输出不符合预期，需要快速看到内部执行过程。

```python
from langchain_core.globals import set_debug, set_verbose

# 方法1：verbose 模式 - 看到链的输入输出
set_verbose(True)

# 方法2：debug 模式 - 看到所有内部细节（包括 prompt 全文）
set_debug(True)

# 使用完毕后关闭
set_debug(False)
set_verbose(False)
```

**记住**：
- `verbose` = 看概要（适合了解执行流程）
- `debug` = 看全部（适合定位具体问题）
- **先开 verbose，不够再开 debug**

### 4.2 用回调追踪 Agent 执行

**场景**：需要记录 Agent 的每一步决策，排查工具调用问题。

```python
from langchain_core.callbacks import StdOutCallbackHandler

# 方法1：全局配置
handler = StdOutCallbackHandler()

# 方法2：单次调用配置（推荐，不影响其他代码）
result = agent.invoke(
    {"input": "今天天气如何？"},
    config={"callbacks": [handler]}
)
```

**自定义回调（最常用模板）**：

```python
from langchain_core.callbacks import BaseCallbackHandler

class SimpleDebugHandler(BaseCallbackHandler):
    """最小化的调试回调，只关注关键事件"""

    def on_llm_start(self, serialized, prompts, **kwargs):
        print(f"🤖 LLM 调用开始，提示词长度: {len(prompts[0])}")

    def on_tool_start(self, serialized, input_str, **kwargs):
        print(f"🔧 调用工具: {serialized.get('name', 'unknown')}")
        print(f"   输入: {input_str[:100]}...")

    def on_tool_error(self, error, **kwargs):
        print(f"❌ 工具报错: {error}")

    def on_agent_action(self, action, **kwargs):
        print(f"🎯 Agent 决策: 使用 {action.tool}")

    def on_agent_finish(self, finish, **kwargs):
        print(f"✅ Agent 完成: {finish.log[:100]}")
```

### 4.3 LangSmith 追踪（生产必备）

**场景**：在生产环境中追踪 Agent 执行，定位线上问题。

```bash
# .env 配置（只需 3 个环境变量）
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=your_api_key
LANGCHAIN_PROJECT=my-agent-project
```

```python
# 代码中无需任何修改！设置环境变量后自动追踪
from dotenv import load_dotenv
load_dotenv()

# 正常调用即可，LangSmith 自动记录
result = agent.invoke({"input": "分析这份数据"})

# 方法2：临时开启追踪（精细控制）
from langchain_core.tracers.context import tracing_v2_enabled

with tracing_v2_enabled(project_name="debug-session"):
    result = agent.invoke({"input": "分析这份数据"})
```

**LangSmith 能看到什么**：
- 完整的调用链路（每一步的输入/输出）
- Token 使用量和成本
- 每一步的延迟
- 错误堆栈信息

### 4.4 LLM 缓存降低成本

**场景**：开发/测试阶段反复调用相同问题，浪费 API 费用。

```python
from langchain_core.globals import set_llm_cache
from langchain_core.caches import InMemoryCache

# 方法1：内存缓存（开发环境推荐）
set_llm_cache(InMemoryCache())

# 方法2：SQLite 缓存（跨会话持久化）
from langchain_community.cache import SQLiteCache
set_llm_cache(SQLiteCache(database_path=".langchain_cache.db"))

# 之后的调用自动命中缓存（相同输入不再调 API）
# 第一次调用：请求 API，~2秒
result1 = llm.invoke("什么是 RAG？")
# 第二次调用：命中缓存，~0.001秒
result2 = llm.invoke("什么是 RAG？")
```

**关键**：缓存的 key 是 `prompt + llm_string`（模型名+参数），同一问题用不同模型不会命中。

### 4.5 防止 Agent 无限循环

**场景**：Agent 不断重复调用工具，陷入死循环。

```python
from langchain_openai import ChatOpenAI
from langchain.agents import create_react_agent, AgentExecutor

# 方法1：设置最大迭代次数（AgentExecutor）
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    max_iterations=10,          # 最多执行 10 步
    max_execution_time=60,      # 最多运行 60 秒
    handle_parsing_errors=True, # 解析错误时不崩溃
)

# 方法2：设置递归限制（LangGraph / Runnable）
result = agent.invoke(
    {"input": "复杂任务"},
    config={"recursion_limit": 25}  # 默认 25，可调低
)
```

**经验法则**：
- 简单问答：`max_iterations=5`
- 多工具协作：`max_iterations=10-15`
- 复杂研究：`max_iterations=20-30`
- 超过 30 步通常说明 prompt 有问题

### 4.6 错误处理三件套

**场景**：Agent 在生产环境中遇到 API 超时、工具报错等问题。

```python
from langchain_openai import ChatOpenAI

# 1. 重试：API 调用失败自动重试
llm = ChatOpenAI(
    model="gpt-4o-mini",
    max_retries=3,              # 最多重试 3 次
    request_timeout=30,         # 单次请求超时 30 秒
)

# 2. 降级：主模型失败时切换到备用模型
main_llm = ChatOpenAI(model="gpt-4o")
backup_llm = ChatOpenAI(model="gpt-4o-mini")

safe_llm = main_llm.with_fallbacks([backup_llm])

# 3. 解析错误处理
agent_executor = AgentExecutor(
    agent=agent,
    tools=tools,
    handle_parsing_errors=True,  # 自动处理输出格式错误
    # 或自定义错误处理
    # handle_parsing_errors="请重新格式化你的回答，使用正确的工具调用格式。"
)
```

## 快速诊断决策树

```
Agent 出问题了？
│
├── 输出不对？
│   ├── 开 set_verbose(True) 看执行流程
│   └── 开 set_debug(True) 看完整 prompt
│
├── 工具调不对？
│   ├── 加 SimpleDebugHandler 看工具调用细节
│   └── 检查工具的 description 是否清晰
│
├── 太慢了？
│   ├── 用 LangSmith 看哪一步最耗时
│   └── 开 LLM 缓存避免重复调用
│
├── 太贵了？
│   ├── 开 LLM 缓存（开发阶段）
│   ├── 简单任务用 gpt-4o-mini
│   └── 用 batch() 批量处理
│
├── 死循环了？
│   ├── 设 max_iterations=10
│   └── 检查 Agent 是否收到工具结果
│
└── 崩溃了？
    ├── 加 max_retries=3
    ├── 加 with_fallbacks()
    └── 加 handle_parsing_errors=True
```
