# 图的编译与执行 - 30字核心

**图的编译与执行是将声明式图定义转换为可执行工作流的过程，通过 compile() 和 invoke() 实现状态化工作流的运行与控制。**

---

## 核心拆解

### compile() - 编译

**将 StateGraph 转换为 CompiledStateGraph 可执行对象**

```python
app = builder.compile(
    checkpointer=checkpointer,      # 持久化
    interrupt_before=["approval"],  # 中断点
)
```

### invoke() - 执行

**使用输入数据运行编译后的图，获取最终结果**

```python
result = app.invoke(
    input_data,
    config={"configurable": {"thread_id": "1"}}
)
```

### Pregel - 引擎

**基于 BSP 模型的批量同步并行执行引擎**

```
Plan → Execution → Update → 重复
```

## 快速上手

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

# 1. 定义状态
class State(TypedDict):
    message: str
    count: int

# 2. 定义节点
def process_node(state: State):
    return {
        "message": f"Processed: {state['message']}",
        "count": state["count"] + 1
    }

# 3. 构建图
builder = StateGraph(State)
builder.add_node("process", process_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

# 4. 编译图
app = builder.compile()

# 5. 执行图
result = app.invoke({
    "message": "Hello",
    "count": 0
})

print(result)
# {'message': 'Processed: Hello', 'count': 1}
```

## 核心价值

### 1. 声明式 → 命令式

**易于理解的声明式定义 → 高效的命令式执行**

```python
# 声明式（易理解）
builder.add_node("step1", func1)
builder.add_edge("step1", "step2")

# 编译成可执行对象（高效）
app = builder.compile()
```

### 2. 灵活控制

**支持多种执行模式**

```python
# 同步执行
result = app.invoke(input)

# 流式执行
for chunk in app.stream(input):
    print(chunk)

# 异步执行
result = await app.ainvoke(input)
```

### 3. 状态持久化

**支持断点续传**

```python
# 配置持久化
app = builder.compile(checkpointer=checkpointer)

# 执行并保存
app.invoke(input, config={"configurable": {"thread_id": "1"}})

# 恢复执行
app.invoke(None, config={"configurable": {"thread_id": "1"}})
```

### 4. 人机交互

**支持执行过程中暂停等待人工输入**

```python
# 配置中断点
app = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["approval"]
)

# 执行到中断点
result = app.invoke(input, config=config)

# 人工处理后恢复
result = app.invoke(None, config=config)
```

## 三个关键问题

### Q1: 为什么需要编译？

**A: 将易于理解的声明式定义转换为高效的可执行对象**

- 声明式定义：易于理解和维护
- 编译后对象：高效执行和优化
- 分离关注点：定义与执行分离

### Q2: invoke 和 stream 有什么区别？

**A: invoke 返回最终结果，stream 流式输出中间过程**

```python
# invoke - 等待完整结果
result = app.invoke(input)
print(result)  # 最终状态

# stream - 流式输出
for chunk in app.stream(input):
    print(chunk)  # 每个节点的输出
```

### Q3: Pregel 算法是什么？

**A: 基于 BSP 模型的批量同步并行执行引擎**

- **Plan**: 选择要执行的节点
- **Execution**: 并行执行节点
- **Update**: 更新状态
- **重复**: 直到满足终止条件

## 实际应用场景

### 场景1: 简单问答

```python
# 编译
app = builder.compile()

# 执行
result = app.invoke({"question": "What is LangGraph?"})
```

### 场景2: 需要审批

```python
# 编译（配置中断点）
app = builder.compile(
    checkpointer=checkpointer,
    interrupt_before=["approval"]
)

# 执行到中断点
result = app.invoke(input, config=config)

# 人工审批后恢复
result = app.invoke(None, config=config)
```

### 场景3: 长时间运行

```python
# 编译（配置持久化）
app = builder.compile(checkpointer=SqliteSaver(...))

# 开始执行
app.invoke(input, config={"configurable": {"thread_id": "1"}})

# 中断后恢复（即使程序重启）
app.invoke(None, config={"configurable": {"thread_id": "1"}})
```

### 场景4: 实时监控

```python
# 流式执行
for chunk in app.stream(input, stream_mode="updates"):
    print(f"Node: {list(chunk.keys())[0]}")
    print(f"Update: {chunk}")
```

## 常见误区

### 误区1: 每次执行都要重新编译 ❌

**正确**: 编译一次，多次使用

```python
# ✅ 正确
app = builder.compile()
result1 = app.invoke(input1)
result2 = app.invoke(input2)

# ❌ 错误
app1 = builder.compile()
result1 = app1.invoke(input1)
app2 = builder.compile()
result2 = app2.invoke(input2)
```

### 误区2: checkpointer 会自动保存 ❌

**正确**: 需要提供 thread_id

```python
# ❌ 错误：没有 thread_id
app.invoke(input)

# ✅ 正确：提供 thread_id
app.invoke(input, config={"configurable": {"thread_id": "1"}})
```

### 误区3: invoke 和 stream 是不同的执行方式 ❌

**正确**: invoke 内部调用 stream

```python
# invoke 的内部实现（简化）
def invoke(self, input):
    latest = None
    for chunk in self.stream(input):
        latest = chunk
    return latest
```

## 核心 API

### compile() 参数

```python
app = builder.compile(
    checkpointer=checkpointer,      # 持久化配置
    interrupt_before=["node1"],     # 节点前中断
    interrupt_after=["node2"],      # 节点后中断
    cache=cache,                    # 缓存配置
    store=store,                    # 存储配置
    debug=True,                     # 调试模式
)
```

### invoke() 参数

```python
result = app.invoke(
    input,                          # 输入数据
    config={                        # 配置
        "configurable": {
            "thread_id": "1"        # 线程 ID
        }
    },
    stream_mode="values",           # 流模式
    durability="async",             # 持久化模式
)
```

### stream() 参数

```python
for chunk in app.stream(
    input,                          # 输入数据
    config=config,                  # 配置
    stream_mode="updates",          # 流模式
):
    print(chunk)
```

## 执行模式对比

| 模式 | 方法 | 返回值 | 使用场景 |
|------|------|--------|----------|
| 同步执行 | `invoke()` | 最终结果 | 简单任务 |
| 流式执行 | `stream()` | 中间过程 | 实时监控 |
| 异步执行 | `ainvoke()` | 最终结果 | 高并发 |
| 异步流式 | `astream()` | 中间过程 | 异步监控 |

---

## 流模式对比

| 模式 | 输出内容 | 使用场景 |
|------|----------|----------|
| `values` | 完整状态 | 查看完整状态 |
| `updates` | 状态更新 | 查看状态变化 |
| `debug` | 调试信息 | 调试和排错 |

---

## 持久化模式对比

| 模式 | 持久化时机 | 性能 | 可靠性 |
|------|-----------|------|--------|
| `sync` | 下一步前完成 | 低 | 高 |
| `async` | 并行持久化 | 高 | 中 |
| `exit` | 退出时持久化 | 最高 | 低 |
