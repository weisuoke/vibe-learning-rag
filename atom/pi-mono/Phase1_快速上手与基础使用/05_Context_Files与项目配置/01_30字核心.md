# 30字核心

**通过 AGENTS.md、settings.json、SYSTEM.md 三个配置文件，让 Pi Agent 理解你的项目规则、开发习惯和团队约定。**

---

## 一句话解释

Context Files 是 Pi Agent 的"项目说明书"，告诉 AI 如何在你的项目中正确工作。

---

## 为什么重要？

没有 Context Files，Pi Agent 就像一个新员工没有入职培训——不知道项目结构、不了解团队规范、不清楚开发流程。

有了 Context Files，Pi Agent 能够：
- 自动遵循项目的代码风格
- 理解项目的目录结构和命名约定
- 知道常用的命令和工作流
- 适应团队的协作方式

---

## 三个核心文件

| 文件 | 作用 | 类比 |
|------|------|------|
| **AGENTS.md** | 项目指令和规范 | 项目 README + 开发手册 |
| **settings.json** | 配置选项（模型、UI、资源） | VS Code 的 settings.json |
| **SYSTEM.md** | 自定义 AI 行为 | 给 AI 的"人设"和"工作守则" |

---

## 快速上手

```bash
# 1. 在项目根目录创建 AGENTS.md
echo "# 项目规范\n\n- 使用 TypeScript\n- 遵循 ESLint 规则" > AGENTS.md

# 2. 创建项目配置目录
mkdir -p .pi

# 3. 创建项目级配置
cat > .pi/settings.json << 'EOF'
{
  "defaultModel": "claude-opus-4",
  "theme": "dark"
}
EOF

# 4. 启动 Pi，配置自动加载
pi
```
