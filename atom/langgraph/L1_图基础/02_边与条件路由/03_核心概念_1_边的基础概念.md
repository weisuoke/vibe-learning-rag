## 核心要点

### 什么是边？

边是图论中的基本概念，在 LangGraph 中表示节点之间的连接关系。每条边都有明确的方向，从起始节点指向结束节点。

### 边的三个核心作用

1. **控制执行顺序**：决定节点的执行先后关系
2. **传递状态**：将状态从一个节点传递到下一个节点
3. **实现决策**：通过条件边实现动态路由

---

## 详细解释

### 1. 边的定义

在 LangGraph 中，边是一个有向连接，包含以下信息：

```python
# 边的基本形式
(start_node, end_node)

# 示例
("retrieve", "grade_documents")  # 从 retrieve 节点到 grade_documents 节点
```

**关键特性**：
- **有向性**：边有明确的方向，从起始节点到结束节点
- **唯一性**：每条边由起始节点和结束节点唯一确定
- **执行语义**：起始节点完成后，结束节点才能开始

[来源: reference/source_edges_01_add_edge.md]

### 2. 边的类型

LangGraph 支持两种类型的边：

#### 类型 1：普通边（Normal Edge）

**定义**：固定路由的边，起始节点总是连接到同一个结束节点。

**语法**：
```python
graph.add_edge(start_key, end_key)
```

**特点**：
- 路由固定，不会改变
- 执行确定，没有分支
- 适用于线性流程

**示例**：
```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    value: int

def step1(state: State) -> dict:
    return {"value": state["value"] * 2}

def step2(state: State) -> dict:
    return {"value": state["value"] + 10}

graph = StateGraph(State)
graph.add_node("step1", step1)
graph.add_node("step2", step2)

# 普通边：固定路由
graph.add_edge(START, "step1")      # START → step1
graph.add_edge("step1", "step2")    # step1 → step2
graph.add_edge("step2", END)        # step2 → END

app = graph.compile()
result = app.invoke({"value": 5})
print(result)  # {'value': 20}
```

**执行流程**：
```
START → step1 → step2 → END
```

[来源: reference/source_edges_01_add_edge.md]

#### 类型 2：条件边（Conditional Edge）

**定义**：动态路由的边，根据状态值选择不同的结束节点。

**语法**：
```python
graph.add_conditional_edges(
    source,           # 起始节点
    path,             # 路由函数
    path_map          # 路径映射
)
```

**特点**：
- 路由动态，根据状态决定
- 执行分支，支持多路由
- 适用于决策场景

**示例**：
```python
from typing import Literal

def route_by_quality(state: State) -> Literal["high", "low"]:
    """根据质量分数路由"""
    if state["quality"] > 0.8:
        return "high"
    return "low"

def process_high(state: State) -> dict:
    return {"result": "High quality processing"}

def process_low(state: State) -> dict:
    return {"result": "Low quality processing"}

graph = StateGraph(State)
graph.add_node("check", lambda s: s)
graph.add_node("high", process_high)
graph.add_node("low", process_low)

# 条件边：动态路由
graph.add_conditional_edges(
    "check",
    route_by_quality,
    {
        "high": "high",
        "low": "low"
    }
)
```

**执行流程**：
```
check → (quality > 0.8) → high
      → (quality ≤ 0.8) → low
```

### 3. 边的存储结构

LangGraph 使用不同的数据结构存储不同类型的边：

#### 存储结构 1：普通边（edges）

**数据结构**：
```python
self.edges: set[tuple[str, str]]
```

**格式**：
```python
{
    ("START", "node1"),
    ("node1", "node2"),
    ("node2", "END")
}
```

**特点**：
- 使用集合（set）存储
- 每个元素是一个元组 `(start, end)`
- 查找效率高（O(1)）

[来源: reference/source_edges_01_add_edge.md]

#### 存储结构 2：等待边（waiting_edges）

**数据结构**：
```python
self.waiting_edges: set[tuple[tuple[str, ...], str]]
```

**格式**：
```python
{
    (("node1", "node2"), "merge"),  # 等待 node1 和 node2 都完成
    (("a", "b", "c"), "final")      # 等待 a、b、c 都完成
}
```

**特点**：
- 用于多起始节点的边
- 第一个元素是起始节点的元组
- 实现 AND 逻辑（等待所有节点完成）

**示例**：
```python
from typing import TypedDict

class State(TypedDict):
    data: list[int]
    sum: int
    product: int

def compute_sum(state: State) -> dict:
    return {"sum": sum(state["data"])}

def compute_product(state: State) -> dict:
    result = 1
    for x in state["data"]:
        result *= x
    return {"product": result}

def merge(state: State) -> dict:
    return state

graph = StateGraph(State)
graph.add_node("sum", compute_sum)
graph.add_node("product", compute_product)
graph.add_node("merge", merge)

# 并行执行
graph.add_edge(START, "sum")
graph.add_edge(START, "product")

# 等待边：等待两个节点都完成
graph.add_edge(["sum", "product"], "merge")

graph.add_edge("merge", END)

app = graph.compile()
result = app.invoke({"data": [2, 3, 4], "sum": 0, "product": 0})
print(result)  # {'data': [2, 3, 4], 'sum': 9, 'product': 24}
```

[来源: reference/source_edges_01_add_edge.md]

#### 存储结构 3：条件边（branches）

**数据结构**：
```python
self.branches: defaultdict[str, dict[str, BranchSpec]]
```

**格式**：
```python
{
    "node1": {
        "condition1": BranchSpec(...),
        "condition2": BranchSpec(...)
    },
    "node2": {
        "router": BranchSpec(...)
    }
}
```

**特点**：
- 两层字典结构
- 第一层键是起始节点名称
- 第二层键是条件边名称
- 值是 BranchSpec 对象（包含路由函数和路径映射）

[来源: reference/source_edges_02_add_conditional_edges.md]

### 4. 边的执行机制

#### 执行时机

边在以下时机执行：

1. **节点完成后**：起始节点执行完成后，立即执行边
2. **状态更新后**：节点更新状态后，边传递新状态
3. **路由决策时**：条件边执行路由函数，决定下一个节点

#### 普通边的执行流程

```python
# 伪代码：边的执行流程
def execute_edge(start_node, end_node, state):
    # 1. 等待起始节点完成
    wait_for_node_completion(start_node)

    # 2. 获取更新后的状态
    updated_state = get_node_output(start_node)

    # 3. 传递状态到结束节点
    pass_state_to_node(end_node, updated_state)

    # 4. 触发结束节点执行
    execute_node(end_node, updated_state)
```

#### 条件边的执行流程

```python
# 伪代码：条件边的执行流程
def execute_conditional_edge(start_node, route_func, path_map, state):
    # 1. 等待起始节点完成
    wait_for_node_completion(start_node)

    # 2. 获取更新后的状态
    updated_state = get_node_output(start_node)

    # 3. 执行路由函数
    route_result = route_func(updated_state)

    # 4. 映射到目标节点
    target_node = path_map[route_result]

    # 5. 传递状态到目标节点
    pass_state_to_node(target_node, updated_state)

    # 6. 触发目标节点执行
    execute_node(target_node, updated_state)
```

### 5. 边的验证规则

LangGraph 在编译时验证边的合法性：

#### 验证规则 1：节点存在性

**规则**：边连接的节点必须已经添加到图中。

**示例**：
```python
# ❌ 错误：node2 未添加
graph.add_node("node1", func1)
graph.add_edge("node1", "node2")  # ValueError: Need to add_node `node2` first

# ✅ 正确：先添加节点
graph.add_node("node1", func1)
graph.add_node("node2", func2)
graph.add_edge("node1", "node2")
```

#### 验证规则 2：END 不能作为起始节点

**规则**：END 是图的终止标记，不能有后续节点。

**示例**：
```python
# ❌ 错误：END 作为起始节点
graph.add_edge(END, "node1")  # ValueError: END cannot be a start node

# ✅ 正确：END 作为结束节点
graph.add_edge("node1", END)
```

#### 验证规则 3：START 不能作为结束节点

**规则**：START 是图的入口标记，不能被其他节点指向。

**示例**：
```python
# ❌ 错误：START 作为结束节点
graph.add_edge("node1", START)  # ValueError: START cannot be an end node

# ✅ 正确：START 作为起始节点
graph.add_edge(START, "node1")
```

#### 验证规则 4：非 StateGraph 不允许重复边

**规则**：非 StateGraph 的图不允许同一个起始节点有多条边。

**示例**：
```python
# ❌ 错误：非 StateGraph 的重复边
from langgraph.graph import Graph
graph = Graph()  # 非 StateGraph
graph.add_edge("node1", "node2")
graph.add_edge("node1", "node3")  # ValueError: Already found path for node 'node1'

# ✅ 正确：StateGraph 支持多边
graph = StateGraph(State)
graph.add_edge("node1", "node2")
graph.add_edge("node1", "node3")  # 合法
```

## 双重类比

### 前端类比

#### 类比 1：普通边 = 固定路由

**前端概念**：React Router 的固定路由
```javascript
<Route path="/home" component={Home} />
<Route path="/about" component={About} />
```

**LangGraph 概念**：
```python
graph.add_edge("home", "about")  # 固定路由
```

**相似性**：
- 都是固定的连接关系
- 执行路径确定
- 没有条件判断

#### 类比 2：条件边 = 条件渲染

**前端概念**：React 的条件渲染
```javascript
{isLoggedIn ? <Dashboard /> : <Login />}
```

**LangGraph 概念**：
```python
def route_func(state):
    if state["is_logged_in"]:
        return "dashboard"
    return "login"

graph.add_conditional_edges("check", route_func, {...})
```

**相似性**：
- 都是基于状态的动态决策
- 根据条件选择不同的路径
- 运行时决定执行哪个分支

#### 类比 3：等待边 = Promise.all()

**前端概念**：等待多个异步操作完成
```javascript
Promise.all([fetchUser(), fetchPosts()])
  .then(([user, posts]) => render(user, posts))
```

**LangGraph 概念**：
```python
graph.add_edge(["fetch_user", "fetch_posts"], "render")
```

**相似性**：
- 都是等待多个操作完成
- 实现 AND 逻辑
- 所有操作完成后才继续

### 日常生活类比

#### 类比 1：普通边 = 工厂传送带

**日常场景**：汽车生产线
- 零件从工序 A 传送到工序 B
- 路径固定，不会改变
- 按顺序执行

**LangGraph 概念**：
```python
graph.add_edge("assemble", "paint")  # 组装 → 喷漆
```

#### 类比 2：条件边 = 机场安检分流

**日常场景**：机场安检
- 根据旅客类型选择不同通道
- 头等舱 → 快速通道
- 经济舱 → 普通通道

**LangGraph 概念**：
```python
def route_passenger(state):
    if state["ticket_class"] == "first":
        return "fast_lane"
    return "normal_lane"
```

#### 类比 3：等待边 = 会议等待

**日常场景**：会议开始前等待所有人到齐
- 等待所有参会者到达
- 所有人到齐后才开始
- 实现 AND 逻辑

**LangGraph 概念**：
```python
graph.add_edge(["person1", "person2", "person3"], "start_meeting")
```

## 实际应用场景

### 场景 1：RAG 工作流

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict, Literal

class RAGState(TypedDict):
    query: str
    documents: list[str]
    quality: float
    answer: str

def retrieve(state: RAGState) -> dict:
    """检索文档"""
    # 模拟检索
    docs = ["doc1", "doc2", "doc3"]
    return {"documents": docs}

def grade(state: RAGState) -> dict:
    """评估文档质量"""
    # 模拟评分
    quality = 0.85
    return {"quality": quality}

def route_after_grade(state: RAGState) -> Literal["generate", "rewrite"]:
    """根据质量路由"""
    if state["quality"] > 0.8:
        return "generate"
    return "rewrite"

def generate(state: RAGState) -> dict:
    """生成答案"""
    answer = f"Answer based on {len(state['documents'])} documents"
    return {"answer": answer}

def rewrite(state: RAGState) -> dict:
    """改写查询"""
    new_query = f"Rewritten: {state['query']}"
    return {"query": new_query}

# 构建图
graph = StateGraph(RAGState)

# 添加节点
graph.add_node("retrieve", retrieve)
graph.add_node("grade", grade)
graph.add_node("generate", generate)
graph.add_node("rewrite", rewrite)

# 添加边
graph.add_edge(START, "retrieve")
graph.add_edge("retrieve", "grade")

# 条件边：根据质量决定路径
graph.add_conditional_edges(
    "grade",
    route_after_grade,
    {
        "generate": "generate",
        "rewrite": "rewrite"
    }
)

# 循环边：改写后重新检索
graph.add_edge("rewrite", "retrieve")
graph.add_edge("generate", END)

# 编译并测试
app = graph.compile()
result = app.invoke({
    "query": "What is LangGraph?",
    "documents": [],
    "quality": 0.0,
    "answer": ""
})
print(result)
```

**关键点**：
- 使用普通边连接固定流程（retrieve → grade）
- 使用条件边实现动态决策（grade → generate/rewrite）
- 使用循环边支持重试机制（rewrite → retrieve）

### 场景 2：多步审批流程

```python
from typing import Literal, TypedDict

class ApprovalState(TypedDict):
    request: dict
    score: float
    status: str

def initial_check(state: ApprovalState) -> dict:
    """初审"""
    # 模拟评分
    score = 0.75
    return {"score": score}

def route_after_check(state: ApprovalState) -> Literal["approve", "reject", "review"]:
    """初审后路由"""
    if state["score"] > 0.9:
        return "approve"
    elif state["score"] < 0.3:
        return "reject"
    return "review"

def approve(state: ApprovalState) -> dict:
    return {"status": "approved"}

def reject(state: ApprovalState) -> dict:
    return {"status": "rejected"}

def human_review(state: ApprovalState) -> dict:
    return {"status": "pending"}

# 构建图
graph = StateGraph(ApprovalState)

# 添加节点
graph.add_node("check", initial_check)
graph.add_node("approve", approve)
graph.add_node("reject", reject)
graph.add_node("review", human_review)

# 添加边
graph.add_edge(START, "check")

# 条件边：多路由决策
graph.add_conditional_edges(
    "check",
    route_after_check,
    {
        "approve": "approve",
        "reject": "reject",
        "review": "review"
    }
)

graph.add_edge("approve", END)
graph.add_edge("reject", END)
graph.add_edge("review", END)

# 编译
app = graph.compile()
```

**关键点**：
- 使用条件边实现多路由决策（approve/reject/review）
- 根据分数阈值选择不同的处理路径
- 支持人机协作（human_review 节点）

## 核心洞察

### 洞察 1：边是控制流的显式表示

传统编程中，控制流隐藏在代码中（if-else、for、while）。LangGraph 将控制流显式表示为边，使得工作流可视化、可分析、可优化。

### 洞察 2：边的类型决定了图的复杂度

- **只有普通边**：线性流程，简单直接
- **包含条件边**：分支流程，支持决策
- **包含等待边**：并行流程，支持汇聚

### 洞察 3：边的组合产生复杂行为

通过组合简单的边，可以构建复杂的工作流：
- 线性 + 分支 = 决策流程
- 分支 + 循环 = 重试机制
- 并行 + 汇聚 = 并行处理
