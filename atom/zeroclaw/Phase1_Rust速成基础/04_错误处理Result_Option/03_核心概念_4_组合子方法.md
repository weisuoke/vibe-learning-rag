## 一句话定义

**组合子是 Result/Option 上的链式变换方法，让你像写 JS 的 `.map().filter().reduce()` 一样处理成功值、错误值和空值——不用到处写 match。**

你已经学了 match 和 `?`，但很多场景既不需要完整的 match 分支，也不需要直接传播错误。你需要的是"对值做一点变换"或"给个默认值"。这就是组合子的用武之地。

## 什么是组合子？

组合子（Combinator）是定义在 `Result<T, E>` 和 `Option<T>` 上的方法，用于对内部值进行变换、链接、过滤或提供默认值。

**TypeScript 开发者的直觉**：你用过 `Array.map()`、`Array.filter()`、`Promise.then()` 吗？Rust 的组合子就是同样的思路，只不过作用对象从数组/Promise 变成了 Result/Option。

```typescript
// TypeScript：数组组合子
const result = [1, 2, 3]
  .map(x => x * 2)        // 变换每个元素
  .filter(x => x > 2)     // 过滤
  .reduce((a, b) => a + b); // 聚合

// TypeScript：Promise 组合子
const data = await fetch(url)
  .then(res => res.json())   // 变换成功值
  .catch(err => defaultData); // 失败时用默认值
```

```rust
// Rust：Result/Option 组合子——同样的链式风格
let port = config.get("port")       // Option<&Value>
    .and_then(|v| v.as_u64())       // Option<u64>
    .map(|p| p as u16)              // Option<u16>
    .unwrap_or(8080);               // u16
```

> **双重类比**
> - **前端类比**：组合子就是 Result/Option 版的 `.map()` / `.then()` / `.catch()` / `??`。
> - **日常生活类比**：组合子就像流水线上的工位——每个工位做一件事（变换、检查、兜底），产品（值）依次经过，最终得到成品。

## Result 组合子

### map() — 变换成功值

对 `Ok(T)` 里的值做变换，`Err` 原样传递。

```rust
let r: Result<i32, String> = Ok(5);
let doubled = r.map(|x| x * 2);  // Ok(10)

let e: Result<i32, String> = Err("fail".into());
let doubled = e.map(|x| x * 2);  // Err("fail") — 闭包不执行
```

```typescript
// TypeScript 等价：Promise.then() 只处理成功
const doubled = await Promise.resolve(5).then(x => x * 2); // 10
```

### map_err() — 变换错误值

对 `Err(E)` 里的错误做变换，`Ok` 原样传递。**ZeroClaw 高频使用。**

```rust
// 基础用法
let r: Result<i32, String> = Err("io error".into());
let mapped = r.map_err(|e| format!("wrapped: {}", e));
// Err("wrapped: io error")
```

**ZeroClaw 实战**：加密库的错误类型没有实现 `std::error::Error`，无法被 `?` 自动转换，需要 `map_err` 手动包装：

```rust
// src/security/secrets.rs
cipher.encrypt(&nonce, plaintext.as_bytes())
    .map_err(|e| anyhow::anyhow!("Encryption failed: {e}"))?;
```

```typescript
// TypeScript 等价：.catch() 变换错误
fetch(url).catch(err => { throw new Error(`Wrapped: ${err}`) });
```

[来源: reference/source_error_handling_01.md - map_err 错误转换]

### and_then() — 链式操作（可能失败的变换）

`map()` 的闭包返回普通值 `U`，`and_then()` 的闭包返回 `Result<U, E>`。避免 `Result<Result<U, E>, E>` 嵌套。

```rust
fn parse_port(s: &str) -> Result<u16, String> {
    s.parse::<u16>().map_err(|e| e.to_string())
}

let r: Result<String, String> = Ok("8080".into());
// map 会得到 Result<Result<u16, String>, String> — 嵌套了！
// and_then 会得到 Result<u16, String> — 展平了
let port = r.and_then(|s| parse_port(&s));  // Ok(8080)
```

```typescript
// TypeScript 等价：Promise.then() 返回新 Promise 时自动展平
const port = await Promise.resolve("8080")
  .then(s => parsePort(s));  // Promise 自动展平
```

**区分 map vs and_then**：
- `map`：闭包返回 `U` → 结果是 `Result<U, E>`（不会失败的变换）
- `and_then`：闭包返回 `Result<U, E>` → 结果是 `Result<U, E>`（可能失败的变换）

### unwrap_or() / unwrap_or_else() — 安全默认值

```rust
// unwrap_or：固定默认值
let port: Result<u16, String> = Err("parse error".into());
let actual = port.unwrap_or(8080);  // 8080

// unwrap_or_else：惰性计算默认值（闭包只在 Err 时执行）
let config: Result<String, std::io::Error> = std::fs::read_to_string("custom.json");
let actual = config.unwrap_or_else(|_| "{}".to_string());
```

```typescript
// TypeScript 等价
const port = parsePort(input) ?? 8080;  // ?? 空值合并
```

### ok() — Result 转 Option（丢弃错误信息）

```rust
let r: Result<i32, String> = Ok(42);
let o: Option<i32> = r.ok();  // Some(42)

let e: Result<i32, String> = Err("fail".into());
let o: Option<i32> = e.ok();  // None（错误信息被丢弃）
```

**使用场景**：你不关心为什么失败，只关心有没有值。ZeroClaw 中 HTTP 头解析就用了这个模式：`.to_str().ok()` 把 Result 转成 Option。

[来源: reference/source_error_handling_01.md - tools/web_fetch.rs 分析]

## Option 组合子

### map() — 变换内部值

```rust
let name: Option<&str> = Some("alice");
let upper = name.map(|n| n.to_uppercase());  // Some("ALICE")

let empty: Option<&str> = None;
let upper = empty.map(|n| n.to_uppercase()); // None（闭包不执行）
```

```typescript
// TypeScript 没有直接等价物，最接近的是：
const upper = name ? name.toUpperCase() : undefined;
```

### and_then() — 链式操作（类比 optional chaining `?.`）

闭包本身返回 `Option<U>`，避免 `Option<Option<U>>` 嵌套。

```rust
let text: Option<&str> = Some("42");
let parsed = text.and_then(|t| t.parse::<i32>().ok());  // Some(42)

let bad: Option<&str> = Some("abc");
let parsed = bad.and_then(|t| t.parse::<i32>().ok());   // None
```

```typescript
// TypeScript 等价：optional chaining
const parsed = text?.match(/^\d+$/)?.[0];  // ?. 遇到 null 就短路
```

**`and_then` 是 Rust 版的 `?.`**——每一步都可能返回 None，遇到 None 就短路，不再执行后续步骤。

### unwrap_or() / unwrap_or_default() — 安全默认值

```rust
// unwrap_or：固定默认值
let flag: Option<bool> = None;
let actual = flag.unwrap_or(false);  // false

// unwrap_or_default：使用类型的 Default trait 实现
let name: Option<String> = None;
let actual = name.unwrap_or_default();  // ""（String 的默认值是空字符串）
// i32 默认 0, bool 默认 false, Vec<T> 默认 []
```

**ZeroClaw 实战**：

```rust
// src/tools/shell.rs — 布尔参数缺失时默认 false
let approved = args.get("approved")
    .and_then(|v| v.as_bool())
    .unwrap_or(false);

// src/tools/web_fetch.rs — 字符串缺失时默认空串
let content_type = response.headers()
    .get(CONTENT_TYPE)
    .and_then(|v| v.to_str().ok())
    .unwrap_or("");
```

```typescript
// TypeScript 等价：?? 空值合并
const approved = args?.approved ?? false;
const contentType = response.headers?.["content-type"] ?? "";
```

[来源: reference/source_error_handling_01.md - 安全默认值 .unwrap_or(false)]

### ok_or() / ok_or_else() — Option 转 Result（ZeroClaw 最常用模式）

把"值可能不存在"升级为"带错误信息的失败"。**这是 ZeroClaw 中出现频率最高的组合子模式。**

```rust
// ok_or：固定错误
let name: Option<&str> = None;
let result = name.ok_or("name is required");  // Err("name is required")

// ok_or_else：惰性构造错误（推荐——只在 None 时才构造错误对象）
let url: Option<&str> = None;
let result = url.ok_or_else(|| anyhow::anyhow!("Missing 'url' parameter"));
// Err(anyhow::Error: Missing 'url' parameter)
```

**为什么推荐 `ok_or_else` 而不是 `ok_or`？**

`ok_or(anyhow!("msg"))` 无论 Some 还是 None 都会执行 `anyhow!("msg")` 构造错误对象。`ok_or_else(|| anyhow!("msg"))` 只在 None 时才执行闭包。当错误构造有开销时（格式化字符串、分配内存），`ok_or_else` 性能更好。

[来源: reference/context7_rust_01.md - Option → Result 转换]

### filter() — 条件过滤

```rust
let age: Option<u32> = Some(15);
let adult = age.filter(|&a| a >= 18);  // None（15 < 18，被过滤掉）

let age: Option<u32> = Some(25);
let adult = age.filter(|&a| a >= 18);  // Some(25)
```

```typescript
// TypeScript 没有直接等价物，最接近的是：
const adult = age !== undefined && age >= 18 ? age : undefined;
```

## ZeroClaw 实战组合子链

### 模式 1：Shell 工具参数提取

这是 ZeroClaw 每个 Tool 实现中都会出现的模式：

```rust
// src/tools/shell.rs
let command = args.get("command")       // Option<&Value>  — HashMap 查找
    .and_then(|v| v.as_str())           // Option<&str>    — JSON Value 转字符串
    .ok_or_else(|| anyhow!("Missing 'command' parameter"))?;  // &str — 失败则传播错误
```

**逐步拆解**：

| 步骤 | 方法 | 输入类型 | 输出类型 | 失败时 |
|------|------|---------|---------|--------|
| 1 | `args.get("command")` | `&Map` | `Option<&Value>` | key 不存在 → None |
| 2 | `.and_then(\|v\| v.as_str())` | `Option<&Value>` | `Option<&str>` | 值不是字符串 → None |
| 3 | `.ok_or_else(\|\| anyhow!(...))` | `Option<&str>` | `Result<&str, anyhow::Error>` | None → Err |
| 4 | `?` | `Result<&str, anyhow::Error>` | `&str` | Err → 函数提前返回 |

```typescript
// TypeScript 等价（如果不用 optional chaining）
const command = args?.command;
if (typeof command !== "string") {
  throw new Error("Missing 'command' parameter");
}
```

[来源: reference/source_error_handling_01.md - Option 组合子链]

### 模式 2：HTTP 头提取

```rust
// src/tools/web_fetch.rs
let content_type = response.headers()
    .get(reqwest::header::CONTENT_TYPE)  // Option<&HeaderValue>
    .and_then(|v| v.to_str().ok())       // Option<&str>  — .ok() 把 Result 转 Option
    .unwrap_or("")                        // &str          — 没有就当空串
    .to_lowercase();
```

**注意 `.ok()` 的妙用**：`v.to_str()` 返回 `Result<&str, ToStrError>`，但我们不关心为什么失败（非 UTF-8 的头值），只关心有没有值。`.ok()` 把 Result 转成 Option，丢弃错误信息，完美融入 Option 组合子链。

[来源: reference/source_error_handling_01.md - tools/web_fetch.rs 分析]

### 模式 3：安全默认值

```rust
// src/tools/shell.rs — 布尔参数
let approved = args.get("approved")     // Option<&Value>
    .and_then(|v| v.as_bool())          // Option<bool>
    .unwrap_or(false);                  // bool — 缺失就是 false

// src/config/schema.rs — 配置字段
let streaming = config.streaming.unwrap_or(false);
let api_url = config.api_url
    .unwrap_or_else(|| "https://api.openai.com/v1".to_string());
```

## 组合子 vs match：何时用哪个？

### 用组合子的场景

```rust
// 1. 简单变换——一行搞定
let doubled = result.map(|x| x * 2);

// 2. 链式操作——多步变换
let value = args.get("key").and_then(|v| v.as_str()).unwrap_or("");

// 3. 提供默认值
let port = config.port.unwrap_or(8080);

// 4. Option → Result 转换
let url = opt_url.ok_or_else(|| anyhow!("Missing URL"))?;
```

### 用 match 的场景

```rust
// 1. Ok 和 Err 需要完全不同的处理逻辑
match self.validate_url(url) {
    Ok(v) => v,
    Err(e) => return Ok(ToolResult { success: false, error: Some(e.to_string()), .. }),
};

// 2. 嵌套 Result 需要多路分支
match tokio::time::timeout(duration, cmd.output()).await {
    Ok(Ok(output)) => { /* 成功 */ },
    Ok(Err(e))     => { /* 执行失败 */ },
    Err(_)         => { /* 超时 */ },
}

// 3. 需要绑定变量做复杂操作
match config.get("mode") {
    Some(mode) if mode == "debug" => enable_debug_logging(),
    Some(mode) => set_mode(mode),
    None => use_default_mode(),
}
```

### 选择指南

| 场景 | 推荐 | 理由 |
|------|------|------|
| 变换值 | `map` / `and_then` | 一行搞定，简洁 |
| 提供默认值 | `unwrap_or` / `unwrap_or_else` | 比 match 简洁 |
| Option → Result | `ok_or_else` | ZeroClaw 标准模式 |
| 两个分支逻辑完全不同 | `match` | 组合子表达不了复杂分支 |
| 三路以上分支 | `match` | 嵌套 Result、guard 条件 |
| 需要提前 return | `match` | 组合子链中不能 return |

## TypeScript 对照表

| Rust 组合子 | TypeScript 等价 | 说明 |
|------------|----------------|------|
| `result.map(\|v\| ...)` | `promise.then(v => ...)` | 变换成功值 |
| `result.map_err(\|e\| ...)` | `promise.catch(e => { throw new Error(...) })` | 变换错误值 |
| `result.and_then(\|v\| ...)` | `promise.then(v => fetchMore(v))` | 链式操作（可能失败） |
| `result.unwrap_or(default)` | `value ?? default` | 失败用默认值 |
| `result.unwrap_or_else(\|e\| ...)` | `value ?? computeDefault()` | 惰性默认值 |
| `result.ok()` | 丢弃 catch 信息 | Result → Option |
| `option.map(\|v\| ...)` | `value?.transform()` | 有值就变换 |
| `option.and_then(\|v\| ...)` | `obj?.nested?.value` | optional chaining |
| `option.unwrap_or(default)` | `value ?? default` | 空值用默认值 |
| `option.unwrap_or_default()` | `value ?? ""` / `value ?? 0` | 用类型默认值 |
| `option.ok_or_else(\|\| err)` | `if (!value) throw new Error(...)` | Option → Result |
| `option.filter(\|v\| cond)` | `value != null && cond(value) ? value : undefined` | 条件过滤 |

**核心对应关系**：
- `map` ≈ `Array.map()` / `Promise.then()`
- `and_then` ≈ `Array.flatMap()` / `Promise.then()` 返回新 Promise
- `unwrap_or` ≈ `??` 空值合并运算符
- `ok_or_else` ≈ `if (!x) throw new Error()`
- `filter` ≈ `Array.filter()`（但作用于单个值）

## 小结

### 速查卡

```
Result 组合子：
  .map(|v| ...)           变换成功值（Ok → Ok，Err 不变）
  .map_err(|e| ...)       变换错误值（Err → Err，Ok 不变）
  .and_then(|v| ...)      链式操作，闭包返回 Result
  .unwrap_or(default)     Err 时用默认值
  .unwrap_or_else(|e| ..) Err 时用闭包计算默认值
  .ok()                   Result → Option（丢弃错误）

Option 组合子：
  .map(|v| ...)           变换内部值（Some → Some，None 不变）
  .and_then(|v| ...)      链式操作，闭包返回 Option
  .unwrap_or(default)     None 时用默认值
  .unwrap_or_default()    None 时用类型 Default 值
  .ok_or(err)             Option → Result（固定错误）
  .ok_or_else(|| err)     Option → Result（惰性错误）★ ZeroClaw 最常用
  .filter(|v| cond)       条件不满足 → None
```

### ZeroClaw 三大组合子模式

| 模式 | 代码 | 场景 |
|------|------|------|
| 参数提取 | `.get("k").and_then(\|v\| v.as_str()).ok_or_else(\|\| anyhow!("..."))?` | 每个 Tool 的 execute 方法 |
| 安全默认值 | `.and_then(\|v\| v.as_bool()).unwrap_or(false)` | 可选布尔/字符串参数 |
| 错误包装 | `.map_err(\|e\| anyhow!("... {e}"))?` | 非标准错误类型转换 |

**记住**：组合子让你用一行代码表达"变换 → 链接 → 兜底"的完整流程。当你发现自己在写只有两行的 match 时，想想能不能用组合子替代。
