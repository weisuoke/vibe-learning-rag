## 1. BM25 算法概述

### 1.1 什么是 BM25？

**BM25（Best Matching 25）** 是一种用于信息检索的排序函数，用于评估文档与查询的相关性。它是 TF-IDF 的改进版本，由 Stephen Robertson 和 Karen Spärck Jones 在 1970-1980 年代开发。

**核心特点：**
- 基于概率检索模型
- 考虑词频饱和效应
- 考虑文档长度归一化
- 广泛应用于搜索引擎（Elasticsearch、Lucene、Milvus）

**在 Milvus 2.6 中的应用：**
- 自动从文本生成稀疏向量
- 支持全文搜索
- 与向量检索混合使用

## 2. 从 TF-IDF 到 BM25 的演进

### 2.1 TF-IDF 基础

**TF-IDF（Term Frequency-Inverse Document Frequency）** 是最经典的文本相关性评分方法。

#### 2.1.1 TF（词频）

**定义：** 词项在文档中出现的频率

```
TF(t, d) = count(t, d) / |d|
```

其中：
- `t`：词项（term）
- `d`：文档（document）
- `count(t, d)`：词项 t 在文档 d 中出现的次数
- `|d|`：文档 d 的总词数

**问题：** 线性增长，没有饱和效应

#### 2.1.2 IDF（逆文档频率）

**定义：** 词项的稀有程度

```
IDF(t) = log(N / df(t))
```

其中：
- `N`：文档总数
- `df(t)`：包含词项 t 的文档数

**作用：** 降低常见词的权重，提升稀有词的权重

#### 2.1.3 TF-IDF 公式

```
TF-IDF(t, d) = TF(t, d) × IDF(t)
```

**问题：**
1. **词频饱和问题**：词频线性增长，没有上限
2. **文档长度问题**：长文档天然得分更高
3. **参数固定**：无法根据数据集调整

---

### 2.2 BM25 的改进

BM25 通过引入两个关键参数解决了 TF-IDF 的问题：
- **k1**：控制词频饱和
- **b**：控制文档长度归一化

---

## 3. BM25 数学公式详解

### 3.1 完整公式

```
BM25(q, d) = Σ IDF(qi) × (f(qi, d) × (k1 + 1)) / (f(qi, d) + k1 × (1 - b + b × |d| / avgdl))
```

其中：
- `q`：查询（query）
- `d`：文档（document）
- `qi`：查询中的第 i 个词项
- `f(qi, d)`：词项 qi 在文档 d 中的词频
- `|d|`：文档 d 的长度（词数）
- `avgdl`：所有文档的平均长度
- `k1`：词频饱和参数（默认 1.2）
- `b`：文档长度归一化参数（默认 0.75）

---

### 3.2 IDF 计算

BM25 使用的 IDF 公式略有不同：

```
IDF(qi) = log((N - df(qi) + 0.5) / (df(qi) + 0.5) + 1)
```

其中：
- `N`：文档总数
- `df(qi)`：包含词项 qi 的文档数
- `+0.5` 和 `+1`：平滑项，避免除零和负值

**与经典 IDF 的区别：**
- 添加平滑项，避免极端值
- 使用 `(N - df + 0.5) / (df + 0.5)` 而非 `N / df`

---

### 3.3 词频饱和机制（k1 参数）

**词频项：**

```
TF_saturated = (f(qi, d) × (k1 + 1)) / (f(qi, d) + k1)
```

**饱和曲线分析：**

当 `f(qi, d)` 增加时：
- `f = 0` → `TF_saturated = 0`
- `f = 1` → `TF_saturated = (k1 + 1) / (k1 + 1) = 1`
- `f → ∞` → `TF_saturated → (k1 + 1)`

**k1 参数的影响：**
- `k1 = 0`：完全忽略词频（类似布尔检索）
- `k1 = 1.2`（默认）：适度饱和
- `k1 → ∞`：接近线性增长（类似 TF-IDF）

**Python 可视化：**

```python
import numpy as np
import matplotlib.pyplot as plt

# 词频范围
f = np.linspace(0, 20, 100)

# 不同 k1 值的饱和曲线
k1_values = [0.5, 1.2, 2.0, 5.0]

plt.figure(figsize=(10, 6))
for k1 in k1_values:
    tf_saturated = (f * (k1 + 1)) / (f + k1)
    plt.plot(f, tf_saturated, label=f'k1={k1}')

plt.xlabel('词频 f(qi, d)')
plt.ylabel('饱和后的词频')
plt.title('BM25 词频饱和曲线')
plt.legend()
plt.grid(True)
plt.show()
```

**观察：**
- k1 越小，饱和越快
- k1 越大，越接近线性增长

---

### 3.4 文档长度归一化（b 参数）

**归一化项：**

```
norm = 1 - b + b × (|d| / avgdl)
```

**完整的分母：**

```
denominator = f(qi, d) + k1 × norm
```

**b 参数的影响：**
- `b = 0`：完全忽略文档长度
- `b = 0.75`（默认）：适度归一化
- `b = 1`：完全归一化（长文档和短文档同等对待）

**文档长度对得分的影响：**

假设 `f(qi, d) = 5`, `k1 = 1.2`, `avgdl = 100`：

| 文档长度 |d| | b=0 | b=0.75 | b=1 |
|------------|-----|--------|-----|
| 50（短）   | 6.2 | 5.48   | 5.17 |
| 100（平均）| 6.2 | 6.2    | 6.2 |
| 200（长）  | 6.2 | 7.7    | 8.7 |

**观察：**
- b=0：文档长度不影响得分
- b=0.75：长文档得分略高
- b=1：长文档得分显著更高

## 4. BM25 参数调优

### 4.1 k1 参数选择

**推荐值：**
- **默认值**：1.2（适用于大多数场景）
- **短文本**：0.8-1.0（如推文、标题）
- **长文本**：1.5-2.0（如文章、书籍）

**调优策略：**
1. 从默认值 1.2 开始
2. 如果高频词权重过高，降低 k1
3. 如果高频词权重过低，提高 k1

### 4.2 b 参数选择

**推荐值：**
- **默认值**：0.75（适用于大多数场景）
- **文档长度差异小**：0.5-0.6
- **文档长度差异大**：0.8-1.0

**调优策略：**
1. 分析文档长度分布
2. 如果长度差异小，降低 b
3. 如果长度差异大，提高 b

**文档长度分析：**

```python
import pandas as pd

# 假设 documents 是文档列表
doc_lengths = [len(doc.split()) for doc in documents]

# 统计分析
df = pd.DataFrame({'length': doc_lengths})
print(df.describe())

# 可视化
import matplotlib.pyplot as plt
plt.hist(doc_lengths, bins=50)
plt.xlabel('文档长度')
plt.ylabel('频数')
plt.title('文档长度分布')
plt.show()

# 计算变异系数（CV）
cv = df['length'].std() / df['length'].mean()
print(f"变异系数: {cv:.2f}")

# 推荐 b 值
if cv < 0.3:
    print("推荐 b=0.5-0.6（长度差异小）")
elif cv < 0.6:
    print("推荐 b=0.75（默认）")
else:
    print("推荐 b=0.8-1.0（长度差异大）")
```

## 5. BM25 在 Milvus 中的实现

### 5.1 源码分析

根据 Milvus 源码（`internal/util/function/bm25_function.go`），BM25 实现流程：

```go
// 1. 分词
tokenStream := tokenizer.NewTokenStream(data[i])

// 2. 哈希（token → uint32）
hash := typeutil.HashString2LessUint32(token)

// 3. 词频统计
embeddingMap[hash] += 1

// 4. 生成稀疏向量
// embeddingMap: map[uint32]float32
```

**关键点：**
- 使用 Analyzer 进行分词
- 使用哈希函数将 token 转换为 uint32
- 词频统计存储在 `map[uint32]float32` 中
- 并发处理（默认 8 个 goroutine）

---

### 5.2 PyMilvus 使用示例

```python
from pymilvus import (
    connections, FieldSchema, CollectionSchema, DataType,
    Collection, Function, FunctionType
)

# 1. 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 2. 定义 Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR)
]

# 3. 创建 BM25 Function
bm25_function = Function(
    name="bm25_fn",
    input_field_names=["content"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)

schema = CollectionSchema(fields=fields, description="BM25 全文搜索")
schema.add_function(bm25_function)

# 4. 创建 Collection
collection = Collection(name="bm25_demo", schema=schema)

# 5. 创建索引（配置 k1 和 b）
index_params = {
    "metric_type": "BM25",
    "index_type": "SPARSE_INVERTED_INDEX",
    "params": {
        "bm25_k1": 1.2,  # 词频饱和参数
        "bm25_b": 0.75   # 文档长度归一化参数
    }
}
collection.create_index("sparse_vector", index_params)

# 6. 插入数据（自动生成稀疏向量）
documents = [
    {"content": "Milvus is a vector database"},
    {"content": "BM25 is a ranking function"},
    {"content": "Sparse vectors are efficient"}
]
collection.insert(documents)
collection.load()

# 7. 搜索
search_params = {"metric_type": "BM25", "params": {}}
results = collection.search(
    data=["vector database"],
    anns_field="sparse_vector",
    param=search_params,
    limit=5,
    output_fields=["content"]
)

for hits in results:
    for hit in hits:
        print(f"ID: {hit.id}, Score: {hit.score}, Content: {hit.entity.get('content')}")
```

## 6. BM25 与 TF-IDF 对比

### 6.1 数学公式对比

| 特性 | TF-IDF | BM25 |
|------|--------|------|
| **词频** | 线性增长 | 饱和增长（k1 控制） |
| **文档长度** | 简单归一化 | 参数化归一化（b 控制） |
| **IDF** | `log(N/df)` | `log((N-df+0.5)/(df+0.5)+1)` |
| **参数** | 无 | k1, b |
| **可调性** | 低 | 高 |

---

### 6.2 性能对比实验

```python
import time
import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer
from pymilvus import connections, Collection

# 准备数据
documents = [...]  # 10000 个文档
queries = [...]    # 100 个查询

# 1. TF-IDF（sklearn）
vectorizer = TfidfVectorizer()

start = time.time()

    scores = (tfidf_matrix * query_vec.T).toarray()

# 2. BM25（Milvus）

for query in queries:
    results = collection.search(
        data=[query],
        anns_field="sparse_vector",
        param={"metric_type": "BM25"},
        limit=10
    )

print(f"TF-IDF 时间: {tfidf_time:.2f}s")
print(f"BM25 时间: {bm25_time:.2f}s")
print(f"加速比: {tfidf_time / bm25_time:.2f}x")
```

**预期结果：**
- Milvus BM25 比 sklearn TF-IDF 快 4-10 倍
- 原因：倒排索引 + WAND 算法优化

## 7. BM25 的局限性与改进

### 7.1 局限性

1. **词袋模型**：忽略词序和语义
2. **稀疏表示**：无法捕捉语义相似性
3. **参数敏感**：需要针对数据集调优
4. **语言依赖**：需要合适的分词器

---

### 7.2 改进方向

#### 7.2.1 BM25+（BM25 Plus）

```
BM25+(q, d) = Σ IDF(qi) × (δ + (f(qi, d) × (k1 + 1)) / (f(qi, d) + k1 × norm))
```

其中 `δ` 是一个小的正数（如 0.5），避免零分。

#### 7.2.2 BM25F（BM25 for Fields）

支持多字段加权：

```
BM25F(q, d) = Σ IDF(qi) × (Σ wj × f(qi, dj) × (k1 + 1)) / (Σ wj × f(qi, dj) + k1)
```

其中 `wj` 是字段 j 的权重。

#### 7.2.3 混合检索（BM25 + 向量检索）

**最佳实践：**
- 使用 BM25 进行关键词匹配
- 使用向量检索进行语义匹配
- 使用 RRFRanker 或 WeightedRanker 融合结果

```python
from pymilvus import AnnSearchRequest, RRFRanker

# 1. BM25 检索
req1 = AnnSearchRequest(
    data=[query],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20
)

# 2. 向量检索
req2 = AnnSearchRequest(
    data=[query_embedding],
    anns_field="dense_vector",
    param={"metric_type": "L2", "params": {"nprobe": 10}},
    limit=20
)

# 3. 混合检索
results = collection.hybrid_search(
    reqs=[req1, req2],
    ranker=RRFRanker(k=60),
    limit=10,
    output_fields=["content"]
)
```

## 8. 实战案例：参数调优实验

### 8.1 实验设计

**目标：** 找到最优的 k1 和 b 参数

**数据集：** 10000 个文档，100 个查询

**评估指标：**
- Recall@10
- Precision@10
- MRR（Mean Reciprocal Rank）

---

### 8.2 实验代码

```python
import numpy as np
from pymilvus import connections, Collection
from sklearn.metrics import recall_score, precision_score

# 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 参数网格
k1_values = [0.8, 1.0, 1.2, 1.5, 2.0]
b_values = [0.5, 0.6, 0.75, 0.85, 1.0]

results = []

for k1 in k1_values:
    for b in b_values:
        # 创建 Collection（使用不同参数）
        collection_name = f"bm25_k1_{k1}_b_{b}"
        # ... (创建 Collection 代码)

        # 创建索引
        index_params = {
            "metric_type": "BM25",
            "index_type": "SPARSE_INVERTED_INDEX",
            "params": {"bm25_k1": k1, "bm25_b": b}
        }
        collection.create_index("sparse_vector", index_params)

        # 评估
        recall_scores = []
        precision_scores = []

        for query, ground_truth in zip(queries, ground_truths):
            search_results = collection.search(
                data=[query],
                anns_field="sparse_vector",
                param={"metric_type": "BM25"},
                limit=10
            )

            predicted = [hit.id for hit in search_results[0]]
            recall = len(set(predicted) & set(ground_truth)) / len(ground_truth)
            precision = len(set(predicted) & set(ground_truth)) / len(predicted)

            recall_scores.append(recall)
            precision_scores.append(precision)

        avg_recall = np.mean(recall_scores)
        avg_precision = np.mean(precision_scores)

        results.append({
            'k1': k1,
            'b': b,
            'recall': avg_recall,
            'precision': avg_precision
        })

        print(f"k1={k1}, b={b}, Recall={avg_recall:.4f}, Precision={avg_precision:.4f}")

# 找到最优参数
best_result = max(results, key=lambda x: x['recall'])
print(f"\n最优参数: k1={best_result['k1']}, b={best_result['b']}")
print(f"Recall@10: {best_result['recall']:.4f}")
print(f"Precision@10: {best_result['precision']:.4f}")
```
