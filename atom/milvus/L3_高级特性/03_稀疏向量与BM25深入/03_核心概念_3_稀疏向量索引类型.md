## 1. 稀疏向量索引概述

### 1.1 为什么需要索引？

**问题：** 稀疏向量检索的挑战
- 维度高（10000+ 维）
- 非零元素少（1-5%）
- 需要快速 top-k 查询

**解决方案：** 倒排索引（Inverted Index）
- 将"文档 → 词项"转换为"词项 → 文档"
- 只存储非零元素
- 支持快速查找

---

### 1.2 Milvus 支持的稀疏向量索引类型

| 索引类型 | 算法 | 特点 | 适用场景 |
|---------|------|------|----------|
| **SPARSE_INVERTED_INDEX** | 倒排索引 | 精度高，速度中等 | 大多数场景 |
| **SPARSE_WAND** | WAND 算法 | 速度快，精度略低 | 性能优先场景 |

## 2. SPARSE_INVERTED_INDEX（倒排索引）

### 2.1 倒排索引原理

**核心思想：** 将"文档 → 词项"的映射转换为"词项 → 文档列表"的映射

**示例：**

假设有 3 个文档：
```
Doc1: "Milvus is a vector database"
Doc2: "BM25 is a ranking function"
Doc3: "Milvus supports vector search"
```

**正向索引（文档 → 词项）：**
```
Doc1 → [Milvus, is, a, vector, database]
Doc2 → [BM25, is, a, ranking, function]
Doc3 → [Milvus, supports, vector, search]
```

**倒排索引（词项 → 文档列表）：**
```
Milvus   → [Doc1, Doc3]
is       → [Doc1, Doc2]
a        → [Doc1, Doc2]
vector   → [Doc1, Doc3]
database → [Doc1]
BM25     → [Doc2]
ranking  → [Doc2]
function → [Doc2]
supports → [Doc3]
search   → [Doc3]
```

---

### 2.2 倒排索引数据结构

**基础结构：**

```python
# 倒排索引结构
inverted_index = {
    term_id: {
        'doc_ids': [doc1, doc2, ...],      # 文档 ID 列表
        'frequencies': [freq1, freq2, ...], # 词频列表
        'positions': [[pos1, pos2], ...]    # 位置列表（可选）
    }
}
```

**示例：**

```python
inverted_index = {
    12345: {  # term_id = hash("Milvus")
        'doc_ids': [1, 3],
        'frequencies': [1, 1],
        'positions': [[0], [0]]
    },
    67890: {  # term_id = hash("vector")
        'doc_ids': [1, 3],
        'frequencies': [1, 1],
        'positions': [[3], [2]]
    }
}
```

---

### 2.3 倒排索引检索流程

**查询：** "Milvus vector"

**步骤：**

1. **分词**：["Milvus", "vector"]
2. **哈希**：[12345, 67890]
3. **查找倒排列表**：
   - term 12345 → [Doc1, Doc3]
   - term 67890 → [Doc1, Doc3]
4. **计算得分**：
   - Doc1: score(12345) + score(67890)
   - Doc3: score(12345) + score(67890)
5. **排序**：返回 top-k 文档

---

### 2.4 PyMilvus 使用示例

```python
from pymilvus import connections, Collection, FieldSchema, CollectionSchema, DataType

# 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 定义 Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR)
]
schema = CollectionSchema(fields=fields)
collection = Collection(name="sparse_inverted_demo", schema=schema)

# 创建 SPARSE_INVERTED_INDEX
index_params = {
    "metric_type": "IP",  # Inner Product
    "index_type": "SPARSE_INVERTED_INDEX",
    "params": {
        "drop_ratio_build": 0.2  # 构建时丢弃 20% 低值条目
    }
}
collection.create_index("sparse_vector", index_params)

# 插入数据
sparse_vectors = [
    {0: 0.5, 100: 0.8, 500: 0.3},
    {10: 0.7, 200: 0.6, 600: 0.4},
    {5: 0.9, 150: 0.5, 700: 0.2}
]
collection.insert([{"sparse_vector": vec} for vec in sparse_vectors])
collection.load()

# 搜索
query_vector = {0: 0.5, 100: 0.8}
search_params = {
    "metric_type": "IP",
    "params": {"drop_ratio_search": 0.2}  # 搜索时丢弃 20% 低值条目
}
results = collection.search(
    data=[query_vector],
    anns_field="sparse_vector",
    param=search_params,
    limit=5
)

for hits in results:
    for hit in hits:
        print(f"ID: {hit.id}, Score: {hit.score}")
```

## 3. SPARSE_WAND（WAND 算法）

### 3.1 WAND 算法原理

**WAND（Weak-AND）** 是一种高效的 top-k 查询算法，通过上界估计跳过不可能进入 top-k 的文档。

**核心思想：**
1. 维护每个词项的最大分数（上界）
2. 使用上界估计跳过不可能进入 top-k 的文档
3. 减少不必要的文档评分

---

### 3.2 WAND 算法流程

**查询：** "Milvus vector"（top-5）

**步骤：**

1. **初始化**：
   - 当前阈值 θ = 0（第 5 名的分数）
   - 词项列表：[Milvus, vector]
   - 上界：[max_score(Milvus), max_score(vector)]

2. **遍历文档**：
   - 对于每个候选文档 d：
     - 计算上界：upper_bound(d) = Σ max_score(term)
     - 如果 upper_bound(d) < θ，跳过该文档
     - 否则，计算实际得分 score(d)

3. **更新阈值**：
   - 如果 score(d) > θ，更新 top-5 列表
   - 更新阈值 θ = 第 5 名的分数

4. **提前终止**：
   - 如果所有剩余文档的上界都 < θ，提前终止

---

### 3.3 WAND vs 倒排索引对比

| 特性 | 倒排索引 | WAND |
|------|----------|------|
| **算法** | 遍历所有候选文档 | 跳过低分文档 |
| **精度** | 100% | 99%+ |
| **速度** | 中等 | 快（2-4x） |
| **内存** | 低 | 中等（需要维护上界） |
| **适用场景** | 精度优先 | 性能优先 |

---

## 4. drop_ratio 参数详解

### 4.1 drop_ratio_build（构建时丢弃）

**作用：** 在构建索引时丢弃低值条目，减少索引大小

**示例：**

```python
# 原始稀疏向量
sparse_vector = {
    0: 0.9,
    100: 0.8,
    200: 0.3,  # 低值
    300: 0.2,  # 低值
    400: 0.1   # 低值
}

# drop_ratio_build = 0.4（丢弃 40% 低值条目）
# 排序：[0.9, 0.8, 0.3, 0.2, 0.1]
# 丢弃：[0.2, 0.1]（最低的 40%）
# 保留：{0: 0.9, 100: 0.8, 200: 0.3}
```

**权衡：**
- drop_ratio_build 越高，索引越小，但精度越低
- drop_ratio_build 越低，索引越大，但精度越高

---

### 4.2 drop_ratio_search（搜索时丢弃）

**作用：** 在搜索时丢弃查询向量的低值条目，加速搜索

**示例：**

```python
# 查询向量
query_vector = {
    0: 0.9,
    100: 0.8,
    200: 0.3,
    300: 0.2,
    400: 0.1
}

# drop_ratio_search = 0.4
# 保留：{0: 0.9, 100: 0.8, 200: 0.3}
# 只使用这 3 个词项进行搜索
```

**权衡：**
- drop_ratio_search 越高，搜索越快，但精度越低
- drop_ratio_search 越低，搜索越慢，但精度越高

---

### 4.3 drop_ratio 调优实验

```python

            data=[query],
            anns_field="sparse_vector",
            param={"metric_type": "IP", "params": {"drop_ratio_search": drop_ratio}},
            limit=10
        )
        search_time = time.time() - start
        search_times.append(search_time)

        # 计算 Recall
        predicted = [hit.id for hit in search_results[0]]
        recall = len(set(predicted) & set(ground_truth)) / len(ground_truth)
        recalls.append(recall)

    avg_time = np.mean(search_times)
    avg_recall = np.mean(recalls)

    results.append({
```

### 5.2 精度对比

```python
from sklearn.metrics import recall_score, precision_score

# 计算 Recall 和 Precision
def evaluate_index(collection, queries, ground_truths):
    recalls = []
    precisions = []

    for query, ground_truth in zip(queries, ground_truths):
        results = collection.search(
            data=[query],
            anns_field="sparse_vector",
            param={"metric_type": "IP"},
            limit=10
        )

        predicted = [hit.id for hit in results[0]]
        recall = len(set(predicted) & set(ground_truth)) / len(ground_truth)
        precision = len(set(predicted) & set(ground_truth)) / len(predicted)

        recalls.append(recall)
        precisions.append(precision)

    return np.mean(recalls), np.mean(precisions)

# 评估
inverted_recall, inverted_precision = evaluate_index(collection_inverted, queries, ground_truths)
wand_recall, wand_precision = evaluate_index(collection_wand, queries, ground_truths)

print(f"SPARSE_INVERTED_INDEX: Recall={inverted_recall:.4f}, Precision={inverted_precision:.4f}")
print(f"SPARSE_WAND: Recall={wand_recall:.4f}, Precision={wand_precision:.4f}")
```

**预期结果：**
```
SPARSE_INVERTED_INDEX: Recall=0.9823, Precision=0.9823
SPARSE_WAND: Recall=0.9756, Precision=0.9756
```

## 6. 索引选择指南

### 6.1 选择 SPARSE_INVERTED_INDEX

**适用场景：**
- 精度优先
- 数据量中等（< 1000万）
- 对搜索速度要求不高

**优点：**
- 精度高（100%）
- 实现简单
- 内存占用低

**缺点：**
- 速度中等

---

### 6.2 选择 SPARSE_WAND

**适用场景：**
- 性能优先
- 数据量大（> 1000万）
- 可以接受略微的精度损失（< 2%）

**优点：**
- 速度快（2-4x）
- 适合大规模数据

**缺点：**
- 精度略低（99%+）
- 内存占用略高

---

### 6.3 决策树

```
是否需要 100% 精度？
├─ 是 → SPARSE_INVERTED_INDEX
└─ 否
   └─ 数据量 > 1000万？
      ├─ 是 → SPARSE_WAND
      └─ 否 → SPARSE_INVERTED_INDEX
```

## 7. 实战案例：BM25 全文搜索

### 7.1 使用 SPARSE_INVERTED_INDEX

```python
from pymilvus import (
    connections, FieldSchema, CollectionSchema, DataType,
    Collection, Function, FunctionType
)

# 1. 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 2. 定义 Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR)
]

# 3. 创建 BM25 Function
bm25_function = Function(
    name="bm25_fn",
    input_field_names=["content"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)

schema = CollectionSchema(fields=fields)
schema.add_function(bm25_function)

# 4. 创建 Collection
collection = Collection(name="bm25_inverted", schema=schema)

# 5. 创建 SPARSE_INVERTED_INDEX
index_params = {
    "metric_type": "BM25",
    "index_type": "SPARSE_INVERTED_INDEX",
    "params": {
        "bm25_k1": 1.2,
        "bm25_b": 0.75,
        "drop_ratio_build": 0.2
    }
}
collection.create_index("sparse_vector", index_params)

# 6. 插入数据
documents = [
    {"content": "Milvus is a vector database"},
    {"content": "BM25 is a ranking function"},
    {"content": "Sparse vectors are efficient"}
]
collection.insert(documents)
collection.load()

# 7. 搜索
results = collection.search(
    data=["vector database"],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=5,
    output_fields=["content"]
)

for hits in results:
    for hit in hits:
        print(f"Score: {hit.score}, Content: {hit.entity.get('content')}")
```
