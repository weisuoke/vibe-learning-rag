## 4.1 普通边（add_edge）- 固定路由的基石

### 概念说明

**普通边是最基础的边类型，用于连接两个节点，实现固定的执行顺序。**

- **API**：`add_edge(start, end)`
- **特征**：确定性路由，节点 A 完成后必定执行节点 B
- **使用场景**：线性流程、确定性步骤

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

class State(TypedDict):
    message: str
    count: int

def step1(state: State) -> dict:
    """第一步：处理输入"""
    return {"message": f"处理: {state['message']}", "count": 1}

def step2(state: State) -> dict:
    """第二步：进一步处理"""
    return {"message": f"{state['message']} -> 完成", "count": state["count"] + 1}

# 构建图
builder = StateGraph(State)
builder.add_node("step1", step1)
builder.add_node("step2", step2)

# 添加普通边：START -> step1 -> step2 -> END
builder.add_edge(START, "step1")
builder.add_edge("step1", "step2")
builder.add_edge("step2", END)

graph = builder.compile()

# 执行
result = graph.invoke({"message": "Hello", "count": 0})
print(result)
# {'message': '处理: Hello -> 完成', 'count': 2}
```

### 实际应用场景

**场景 1：RAG 文档处理流程**
```python
# 文档加载 -> 文本分块 -> 向量化 -> 存储
builder.add_edge(START, "load_documents")
builder.add_edge("load_documents", "chunk_text")
builder.add_edge("chunk_text", "embed_chunks")
builder.add_edge("embed_chunks", "store_vectors")
builder.add_edge("store_vectors", END)
```

**场景 2：数据清洗管道**
```python
# 读取数据 -> 清洗 -> 验证 -> 保存
builder.add_edge(START, "read_data")
builder.add_edge("read_data", "clean_data")
builder.add_edge("clean_data", "validate_data")
builder.add_edge("validate_data", "save_data")
builder.add_edge("save_data", END)
```

### 关键要点

- ✅ **简单直接**：适合 70% 的线性流程
- ✅ **可预测**：执行路径固定，易于调试
- ✅ **高性能**：无需路由判断，执行效率高
- ⚠️ **不灵活**：无法根据状态动态改变流程

## 4.2 条件边（add_conditional_edges）- 动态路由的核心

### 概念说明

**条件边根据状态值动态决定下一个节点，是实现分支逻辑的关键。**

- **API**：`add_conditional_edges(source, path_function, path_map)`
- **特征**：动态路由，根据路由函数返回值决定下一步
- **使用场景**：条件分支、决策逻辑、循环控制

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from typing import Literal
from typing_extensions import TypedDict

class State(TypedDict):
    query: str
    score: float
    result: str

def analyze_query(state: State) -> dict:
    """分析查询质量"""
    # 模拟质量评分
    score = len(state["query"]) / 10.0
    return {"score": score}

def route_by_quality(state: State) -> Literal["high_quality", "low_quality"]:
    """根据质量评分路由"""
    if state["score"] > 0.5:
        return "high_quality"
    return "low_quality"

def handle_high_quality(state: State) -> dict:
    """处理高质量查询"""
    return {"result": f"直接回答: {state['query']}"}

def handle_low_quality(state: State) -> dict:
    """处理低质量查询"""
    return {"result": f"需要改写: {state['query']}"}

# 构建图
builder = StateGraph(State)
builder.add_node("analyze", analyze_query)
builder.add_node("high_quality", handle_high_quality)
builder.add_node("low_quality", handle_low_quality)

# 添加条件边
builder.add_edge(START, "analyze")
builder.add_conditional_edges(
    "analyze",
    route_by_quality,
    {
        "high_quality": "high_quality",
        "low_quality": "low_quality"
    }
)
builder.add_edge("high_quality", END)
builder.add_edge("low_quality", END)

graph = builder.compile()

# 测试高质量查询
result1 = graph.invoke({"query": "What is LangGraph?", "score": 0.0, "result": ""})
print(result1["result"])  # 直接回答: What is LangGraph?

# 测试低质量查询
result2 = graph.invoke({"query": "help", "score": 0.0, "result": ""})
print(result2["result"])  # 需要改写: help
```

### 实际应用场景

**场景 1：RAG 文档质量评估**
```python
def route_by_relevance(state: State) -> Literal["generate", "search_web"]:

    return "search_web"  # 文档不相关，搜索网络

    "grade_documents",

    {
        "generate": "generate_answer",

```

**场景 2：多代理路由**

```python
    """根据问题类型路由到不同代理"""
    if "code" in state["query"].lower():
        return "code_agent"
    elif "document" in state["query"].lower():
        return "doc_agent"
    return "general_agent"

builder.add_conditional_edges(
    "classify_query",
    route_to_agent,
    {
        "code_agent": "code_agent",
        "doc_agent": "doc_agent",
        "general_agent": "general_agent"
    }

```

**场景 3：循环控制**
```python
def should_continue(state: State) -> Literal["continue", "exit"]:
    """判断是否继续迭代"""
    if state["iteration"] < state["max_iterations"] and not state["converged"]:
        return "continue"
    return "exit"

builder.add_conditional_edges(
    "process",
    should_continue,
    {
        "continue": "process",  # 循环回自己
        "exit": END
    }

```

### 关键要点

- ✅ **灵活强大**：适合 25% 的条件分支场景
- ✅ **类型安全**：使用 `Literal` 类型注解提升可维护性
- ✅ **可视化友好**：类型注解帮助自动生成正确的图
- ⚠️ **路由函数必须是纯函数**：只读取状态，不修改状态
- ⚠️ **避免与普通边混用**：同一目标节点不要同时有条件边和普通边

## 4.3 Send 对象 - 动态并行执行

### 概念说明

**Send 对象允许条件边动态创建多个并行任务，每个任务可以接收不同的状态。**

- **API**：条件边返回 `Send(node, state)` 对象
- **特征**：动态并行（fan-out），每个任务独立执行
- **使用场景**：批量处理、并行分析、Map-Reduce 模式

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
from typing_extensions import TypedDict, Annotated
import operator

class OverallState(TypedDict):
    documents: list[str]
    summaries: Annotated[list[str], operator.add]

class DocumentState(TypedDict):
    document: str

def fan_out_documents(state: OverallState) -> list[Send]:
    """为每个文档创建并行任务"""
    return [
        Send("summarize", {"document": doc})
        for doc in state["documents"]
    ]

def summarize(state: DocumentState) -> dict:
    """处理单个文档"""
    summary = f"摘要: {state['document'][:20]}..."
    return {"summaries": [summary]}

# 构建图
builder = StateGraph(OverallState)
builder.add_node("summarize", summarize)

# 使用条件边 + Send 实现动态并行
builder.add_conditional_edges(START, fan_out_documents)
builder.add_edge("summarize", END)

graph = builder.compile()

# 执行
result = graph.invoke({
    "documents": ["Doc1: AI is amazing", "Doc2: LangGraph is powerful", "Doc3: Python is great"],
    "summaries": []
})

print(result["summaries"])
# ['摘要: Doc1: AI is amazi...', '摘要: Doc2: LangGraph i...', '摘要: Doc3: Python is g...']
```

### 实际应用场景

**场景 1：RAG 多文档并行处理**

```python
    """为每个文本块创建并行嵌入任务"""
    return [
        Send("embed_chunk", {"chunk": chunk, "chunk_id": i})
        for i, chunk in enumerate(state["chunks"])
    ]

builder.add_conditional_edges(START, fan_out_chunks)
builder.add_node("embed_chunk", embed_single_chunk)
builder.add_edge("embed_chunk", END)
```

**场景 2：多代理并行分析**

```python
    """将任务分发给多个专业代理"""

        Send("code_agent", {"task": "分析代码质量"}),
        Send("doc_agent", {"task": "检查文档完整性"}),
        Send("test_agent", {"task": "验证测试覆盖率"})
```

**场景 3：批量数据验证**

```python
    """为每条数据创建验证任务"""
    return [
        Send("validate_record", {"record": record})
        for record in state["records"]
    ]
```

### 关键要点

- ✅ **高效并行**：适合 5% 的批量处理场景
- ✅ **灵活状态**：每个任务可以接收不同的状态
- ✅ **自动聚合**：使用 `Annotated[list, operator.add]` 自动收集结果
- ⚠️ **不能发送到 END**：`Send(END, state)` 会报错
- ⚠️ **状态合并**：需要使用 reducer 函数（如 `operator.add`）合并结果

## 4.4 边的选择策略 - 决策树

### 快速决策流程

```
开始选择边类型
    ↓
是否需要根据状态动态决策？
    ├─ 否 → 使用普通边（add_edge）
    │       适用场景：线性流程、固定步骤
    │
    └─ 是 → 是否需要并行处理多个任务？
            ├─ 否 → 使用条件边（add_conditional_edges）
            │       适用场景：条件分支、循环控制
            │
            └─ 是 → 使用条件边 + Send 对象
                    适用场景：批量处理、Map-Reduce
```

### 选择矩阵

| 场景 | 边类型 | 示例 |
|------|--------|------|
| 文档加载 → 分块 → 存储 | 普通边 | `add_edge("load", "chunk")` |
| 根据质量决定是否重写 | 条件边 | `add_conditional_edges("grade", route_fn, {...})` |
| 为每个文档创建摘要任务 | 条件边 + Send | `return [Send("summarize", {"doc": d}) for d in docs]` |
| 循环优化直到收敛 | 条件边（循环） | `"continue": "process"` |
| 多代理协作 | 条件边 | `route_to_agent()` |

### 代码示例：综合应用

```python
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
from typing import Literal
from typing_extensions import TypedDict, Annotated
import operator

class State(TypedDict):
    query: str
    documents: list[str]
    relevance_scores: Annotated[list[float], operator.add]
    answer: str

def retrieve_documents(state: State) -> dict:
    """检索文档（普通边）"""
    docs = ["Doc1", "Doc2", "Doc3"]
    return {"documents": docs}

def fan_out_grading(state: State) -> list[Send]:
    """并行评估每个文档（条件边 + Send）"""
    return [
        Send("grade_document", {"document": doc})
        for doc in state["documents"]
    ]

def grade_document(state: dict) -> dict:
    """评估单个文档"""
    score = 0.8  # 模拟评分
    return {"relevance_scores": [score]}

def route_by_quality(state: State) -> Literal["generate", "search_web"]:
    """根据文档质量路由（条件边）"""
    avg_score = sum(state["relevance_scores"]) / len(state["relevance_scores"])
    if avg_score > 0.7:
        return "generate"
    return "search_web"

def generate_answer(state: State) -> dict:
    """生成答案（普通边）"""
    return {"answer": f"基于文档回答: {state['query']}"}

def search_web(state: State) -> dict:
    """搜索网络（普通边）"""
    return {"answer": f"基于网络回答: {state['query']}"}

# 构建图
builder = StateGraph(State)
builder.add_node("retrieve", retrieve_documents)
builder.add_node("grade_document", grade_document)
builder.add_node("generate", generate_answer)
builder.add_node("search_web", search_web)

# 组合使用不同类型的边
builder.add_edge(START, "retrieve")  # 普通边
builder.add_conditional_edges("retrieve", fan_out_grading)  # 条件边 + Send
builder.add_conditional_edges(
    "grade_document",
    route_by_quality,
    {"generate": "generate", "search_web": "search_web"}
)  # 条件边
builder.add_edge("generate", END)  # 普通边
builder.add_edge("search_web", END)  # 普通边

graph = builder.compile()
```

## 4.5 常见陷阱与避坑指南

### 陷阱 1：条件边和普通边混用

**错误示例**：
```python
# ❌ 错误：同一目标节点同时有条件边和普通边
builder.add_edge("node_a", "node_b")  # 普通边
builder.add_conditional_edges(
    "node_a",
    route_fn,
    {"path1": "node_b"}  # 条件边也指向 node_b
)
# 结果：node_b 会被执行两次！
```

**正确做法**：
```python
# ✅ 正确：只使用一种边类型
builder.add_conditional_edges(
    "node_a",
    route_fn,
    {"path1": "node_b", "path2": "node_c"}
)
```

### 陷阱 2：路由函数修改状态

**错误示例**：
```python
# ❌ 错误：路由函数不应修改状态
def route_fn(state: State) -> Literal["path1", "path2"]:
    state["count"] += 1  # 错误：修改了状态
    return "path1" if state["count"] > 5 else "path2"
```

**正确做法**：
```python
# ✅ 正确：路由函数只读取状态
def route_fn(state: State) -> Literal["path1", "path2"]:
    return "path1" if state["count"] > 5 else "path2"
```

### 陷阱 3：忘记类型注解

**错误示例**：
```python
# ❌ 错误：没有类型注解，图可视化可能不正确
def route_fn(state: State):
    return "path1" if state["score"] > 0.5 else "path2"
```

**正确做法**：
```python
# ✅ 正确：使用 Literal 类型注解
def route_fn(state: State) -> Literal["path1", "path2"]:
    return "path1" if state["score"] > 0.5 else "path2"
```

### 陷阱 4：Send 到 END 节点

**错误示例**：
```python
# ❌ 错误：不能 Send 到 END
def fan_out(state: State) -> list[Send]:
    return [Send(END, {"result": "done"})]  # 错误！
```

**正确做法**：
```python
# ✅ 正确：Send 到普通节点，然后用普通边连接到 END
def fan_out(state: State) -> list[Send]:
    return [Send("process", {"data": d}) for d in state["data"]]

builder.add_edge("process", END)
```

## 这些知识足以做什么？

掌握以上 4 个核心知识点，你可以：

### ✅ 构建 80% 的 LangGraph 工作流

1. **线性 RAG 管道**
   - 文档加载 → 分块 → 向量化 → 存储
   - 使用普通边连接

2. **条件分支 RAG**
   - 检索 → 评估质量 → 决定是否搜索网络
   - 使用条件边实现决策

3. **并行处理 RAG**
   - 批量文档 → 并行摘要 → 聚合结果
   - 使用条件边 + Send 实现并行

4. **多代理系统**
   - 分类查询 → 路由到专业代理 → 汇总结果
   - 使用条件边实现路由

5. **迭代优化流程**
   - 生成 → 评估 → 决定是否继续优化
   - 使用条件边实现循环

### ✅ 避免 90% 的常见错误

- 不会混用条件边和普通边
- 路由函数保持纯函数
- 正确使用类型注解
- 避免 Send 到 END

### ✅ 为后续学习打基础

- 理解边的内部实现机制（通道系统）
- 学习高级边类型（等待边、序列边）
- 掌握复杂路由策略（多条件、嵌套路由）
- 优化图性能（减少不必要的边）

### 路由函数模板

```python
from typing import Literal

# 二分支路由
def route_binary(state: State) -> Literal["path_a", "path_b"]:
    return "path_a" if condition else "path_b"

# 多分支路由
def route_multi(state: State) -> Literal["path1", "path2", "path3"]:
    if condition1:
        return "path1"
    elif condition2:
        return "path2"
    return "path3"

# 循环控制路由
def route_loop(state: State) -> Literal["continue", "exit"]:
    return "continue" if should_continue else "exit"

# 动态并行路由
def route_parallel(state: State) -> list[Send]:
    return [Send("process", {"item": item}) for item in state["items"]]
```

### 常用模式代码片段


## 总结

**记住这个公式：**

```
普通边（70%）+ 条件边（25%）+ Send（5%）= 完整的 LangGraph 工作流
```

**核心原则：**
1. 优先使用普通边（简单直接）
2. 需要决策时使用条件边（灵活强大）
3. 需要并行时使用 Send（高效批量）
4. 避免混用（防止重复执行）
5. 路由函数保持纯函数（易于测试）

**实践建议：**
- 从简单的线性流程开始
- 逐步添加条件分支
- 最后考虑并行优化
- 始终使用类型注解
- 多画图、多测试

掌握这些核心知识，你已经可以构建大多数 LangGraph 应用了！
