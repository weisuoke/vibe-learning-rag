## 卡片1：直觉理解 - 什么是部分状态更新？

**一句话：** 节点函数不需要返回完整状态，只返回需要更新的字段，就像修改表单时只填写变化的字段。

**举例：**

想象你在填写一个长表单，有姓名、年龄、地址、电话等10个字段。传统方式是每次修改都要重新填写所有10个字段，即使只改了电话号码。

部分状态更新就像智能表单：你只需要填写变化的字段（比如只填电话号码），其他字段自动保持原值。

```python
# 传统方式：必须返回完整状态
def update_user_full(state):
    return {
        "name": state["name"],        # 没变也要写
        "age": state["age"],          # 没变也要写
        "address": state["address"],  # 没变也要写
        "phone": "新号码"             # 只有这个变了
    }

# 部分状态更新：只返回变化的字段
def update_user_partial(state):
    return {"phone": "新号码"}  # 其他字段自动保持原值
```

[来源: reference/context7_langgraph_01.md]

**应用：** 在 LangGraph 工作流中，每个节点只关心自己负责的字段，降低耦合，提高可维护性。

## 卡片2：形式化定义 - 精确表述

**一句话：** 部分状态更新是指节点函数返回状态的子集，框架自动将返回值与当前状态合并，未返回的字段保持不变。

**数学表达：**

```
设当前状态为 S = {k1: v1, k2: v2, ..., kn: vn}
节点返回值为 R = {k1: v1', k3: v3'}

合并后状态 S' = {
    k1: merge(v1, v1'),  # 使用 Reducer 合并（如果有）
    k2: v2,              # 未返回，保持原值
    k3: v3',             # 覆盖（如果没有 Reducer）
    ...
}
```

**代码示例：**

```python
from typing import TypedDict

class State(TypedDict):
    field_a: str
    field_b: int
    field_c: list[str]

def node(state: State) -> dict:
    # 只返回需要更新的字段
    return {
        "field_a": "new value",
        "field_b": state["field_b"] + 1
    }
    # field_c 未返回，保持原值不变
```

[来源: reference/source_部分状态更新_01.md]

**应用：** 这是 LangGraph 状态管理的核心机制，支持模块化节点设计。

## 卡片3：关键概念1 - 节点返回部分字段

**一句话：** 节点函数可以返回字典、Pydantic 模型或 Command 对象，只包含需要更新的字段。

**三种返回方式：**

```python
from typing import TypedDict
from pydantic import BaseModel
from langgraph.types import Command

class State(TypedDict):
    messages: list[str]
    counter: int
    status: str

# 方式1：返回字典（最常用）
def node_dict(state: State) -> dict:
    return {"counter": state["counter"] + 1}

# 方式2：返回 Pydantic 模型
class PartialState(BaseModel):
    counter: int

def node_pydantic(state: State) -> PartialState:
    return PartialState(counter=state["counter"] + 1)

# 方式3：返回 Command 对象（同时控制流程）
def node_command(state: State) -> Command:
    return Command(
        goto="next_node",
        update={"counter": state["counter"] + 1}
    )
```

[来源: reference/source_部分状态更新_01.md]

**应用：** 根据场景选择合适的返回方式，字典最灵活，Pydantic 提供类型安全，Command 支持动态路由。

## 卡片4：关键概念2 - Reducer 函数机制

**一句话：** Reducer 函数定义了如何合并新旧值，使用 `Annotated[type, reducer]` 语法绑定到字段上。

**核心原理：**

```python
from typing import Annotated, TypedDict
import operator

class State(TypedDict):
    # 有 Reducer：使用 operator.add 合并
    logs: Annotated[list[str], operator.add]
    # 没有 Reducer：直接覆盖
    status: str

# 执行流程
def node_a(state: State):
    return {"logs": ["A"], "status": "running"}

def node_b(state: State):
    return {"logs": ["B"], "status": "done"}

# 结果：
# logs = ["A", "B"]  # operator.add 追加
# status = "done"    # 覆盖
```

**底层实现（BinaryOperatorAggregate）：**

```python
class BinaryOperatorAggregate:
    def __init__(self, typ: type, operator: Callable):
        self.operator = operator
        self.value = typ()  # 初始化空值

    def update(self, new_value):
        # 使用 Reducer 函数合并
        self.value = self.operator(self.value, new_value)
```

[来源: reference/source_部分状态更新_01.md]

**应用：** Reducer 是实现增量更新的核心，避免数据丢失。

## 卡片5：编程实现 - Annotated 字段定义

**一句话：** 使用 `Annotated` 类型注解为字段指定 Reducer 函数，控制状态更新策略。

**完整示例：**

```python
from typing import Annotated, TypedDict
from langgraph.graph import StateGraph, START, END
import operator

# 1. 定义状态（使用 Annotated 指定 Reducer）
class State(TypedDict):
    messages: Annotated[list[str], operator.add]  # 追加
    counter: Annotated[int, operator.add]         # 累加
    status: str                                    # 覆盖

# 2. 定义节点
def node_a(state: State):
    return {
        "messages": ["Hello from A"],
        "counter": 1,
        "status": "processing"
    }

def node_b(state: State):
    return {
        "messages": ["Hello from B"],
        "counter": 1,
        "status": "done"
    }

# 3. 构建图
builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", END)

graph = builder.compile()

# 4. 执行
result = graph.invoke({"messages": [], "counter": 0, "status": "init"})
print(result)
# {
#     "messages": ["Hello from A", "Hello from B"],  # 追加
#     "counter": 2,                                   # 累加
#     "status": "done"                                # 覆盖
# }
```

[来源: reference/context7_langgraph_01.md]

**应用：** 这是 LangGraph 状态管理的标准模式，适用于大多数场景。

## 卡片6：对比区分 - 覆盖 vs Reducer

**一句话：** 没有 Reducer 的字段使用覆盖策略，有 Reducer 的字段使用合并策略。

**对比表：**

| 特性 | 覆盖策略 | Reducer 策略 |
|------|----------|--------------|
| 定义方式 | `field: type` | `field: Annotated[type, reducer]` |
| 更新行为 | 新值替换旧值 | 使用 Reducer 函数合并 |
| 适用场景 | 状态标志、配置项 | 列表、计数器、消息历史 |
| 数据丢失风险 | 低（单值） | 高（如果误用覆盖） |

**代码对比：**

```python
from typing import Annotated, TypedDict
import operator

class State(TypedDict):
    # 覆盖策略
    status: str
    current_step: int

    # Reducer 策略
    logs: Annotated[list[str], operator.add]
    total_cost: Annotated[float, operator.add]

def node_a(state: State):
    return {
        "status": "running",      # 覆盖
        "current_step": 1,        # 覆盖
        "logs": ["Step 1"],       # 追加
        "total_cost": 0.05        # 累加
    }

def node_b(state: State):
    return {
        "status": "done",         # 覆盖（"running" 丢失）
        "current_step": 2,        # 覆盖（1 丢失）
        "logs": ["Step 2"],       # 追加（保留 "Step 1"）
        "total_cost": 0.03        # 累加（保留 0.05）
    }

# 结果：
# status = "done"                    # 覆盖
# current_step = 2                   # 覆盖
# logs = ["Step 1", "Step 2"]        # 追加
# total_cost = 0.08                  # 累加
```

[来源: reference/context7_langgraph_01.md]

**应用：** 根据数据特性选择合适的策略，避免数据丢失。

## 卡片7：进阶理解 - add_messages 函数

**一句话：** `add_messages` 是专门用于消息列表的 Reducer，支持消息追加、更新和删除。

**核心特性：**

```python
from typing import Annotated, Sequence, TypedDict
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[Sequence[BaseMessage], add_messages]

# 特性1：追加新消息
def chatbot(state: State):
    response = AIMessage(content="Hello!")
    return {"messages": [response]}  # 追加到消息列表

# 特性2：更新现有消息（通过 ID）
def update_message(state: State):
    # 假设要更新第一条消息
    updated = AIMessage(
        content="Updated content",
        id=state["messages"][0].id  # 使用相同 ID
    )
    return {"messages": [updated]}  # 更新而不是追加

# 特性3：删除消息（使用 RemoveMessage）
from langgraph.graph.message import RemoveMessage

def delete_message(state: State):
    return {"messages": [RemoveMessage(id=state["messages"][0].id)]}
```

**与 operator.add 的区别：**

```python
# operator.add：简单拼接
logs: Annotated[list[str], operator.add]
# ["A", "B"] + ["C"] = ["A", "B", "C"]

# add_messages：智能合并
messages: Annotated[Sequence[BaseMessage], add_messages]
# 支持追加、更新、删除
```

[来源: reference/context7_langgraph_01.md]

**应用：** 在对话式工作流中维护消息历史，支持消息编辑和删除。

## 卡片8：高级应用 - Overwrite 模式

**一句话：** 使用 `Overwrite` 可以显式覆盖有 Reducer 的字段，用于特殊场景下的状态重置。

**使用场景：**

```python
from typing import Annotated, TypedDict
from langgraph.channels.binop import Overwrite, OVERWRITE
import operator

class State(TypedDict):
    messages: Annotated[list[str], operator.add]

# 场景1：正常追加
def append_message(state: State):
    return {"messages": ["New message"]}
    # 结果：追加到列表

# 场景2：重置列表（使用 Overwrite）
def reset_messages(state: State):
    return {"messages": Overwrite([])}
    # 结果：清空列表，而不是追加

# 场景3：使用字典形式
def reset_messages_dict(state: State):
    return {"messages": {OVERWRITE: []}}
    # 结果：同上
```

**实际应用示例：**

```python
from langgraph.graph import StateGraph, START, END

class ConversationState(TypedDict):
    messages: Annotated[list[str], operator.add]
    session_id: str

def start_new_session(state: ConversationState):
    # 开始新会话时清空消息历史
    return {
        "messages": Overwrite([]),  # 清空
        "session_id": "new_session_id"
    }

def add_message(state: ConversationState):
    # 正常追加消息
    return {"messages": ["User: Hello"]}

builder = StateGraph(ConversationState)
builder.add_node("start", start_new_session)
builder.add_node("chat", add_message)
builder.add_edge(START, "start")
builder.add_edge("start", "chat")
builder.add_edge("chat", END)

graph = builder.compile()
result = graph.invoke({"messages": ["Old message"], "session_id": "old"})
print(result["messages"])  # ["User: Hello"]（旧消息被清空）
```

[来源: reference/source_部分状态更新_01.md]

**应用：** 在需要重置累积数据的场景中使用，如会话切换、错误恢复。

## 卡片9：在 LangGraph 中的使用 - Pydantic 模型集成

**一句话：** LangGraph 支持 Pydantic 模型作为状态，只更新被显式设置的字段，跳过默认值。

**核心机制（get_update_as_tuples）：**

```python
from pydantic import BaseModel, Field

class UserState(BaseModel):
    name: str = Field(default="")
    age: int = Field(default=0)
    email: str = Field(default="")

# 只更新显式设置的字段
user = UserState(name="Alice", age=30)
# user.model_fields_set = {"name", "age"}
# email 使用默认值，不会被更新

# LangGraph 内部处理
def get_update_as_tuples(input: BaseModel, keys: list[str]):
    keep = input.model_fields_set  # {"name", "age"}
    defaults = {k: v.default for k, v in type(input).model_fields.items()}

    return [
        (k, value)
        for k in keys
        if (value := getattr(input, k)) is not None
        and (value != defaults.get(k) or k in keep)
    ]
```

**实际应用：**

```python
from typing import TypedDict
from pydantic import BaseModel
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    user_name: str
    user_age: int
    user_email: str

class UserUpdate(BaseModel):
    user_name: str = ""
    user_age: int = 0
    user_email: str = ""

def update_user(state: State) -> UserUpdate:
    # 只更新 name 和 age
    return UserUpdate(user_name="Bob", user_age=25)
    # user_email 不会被更新（使用默认值）

builder = StateGraph(State)
builder.add_node("update", update_user)
builder.add_edge(START, "update")
builder.add_edge("update", END)

graph = builder.compile()
result = graph.invoke({
    "user_name": "Alice",
    "user_age": 30,
    "user_email": "alice@example.com"
})
print(result)
# {
#     "user_name": "Bob",              # 更新
#     "user_age": 25,                  # 更新
#     "user_email": "alice@example.com"  # 保持不变
# }
```

[来源: reference/source_部分状态更新_01.md]

**应用：** 使用 Pydantic 模型提供类型安全和数据验证，同时支持部分更新。

## 卡片10：总结与延伸 - Command 对象更新

**一句话：** Command 对象可以同时更新状态和控制流程，实现动态路由。

**核心用法：**

```python
from typing import Annotated, TypedDict
from langgraph.types import Command
from langgraph.graph import StateGraph, START, END
import operator

class State(TypedDict):
    messages: Annotated[list[str], operator.add]
    step: int
    should_continue: bool

def decision_node(state: State) -> Command:
    if state["step"] < 3:
        # 继续处理，跳转到 process 节点
        return Command(
            goto="process",
            update={
                "messages": [f"Step {state['step']}"],
                "step": state["step"] + 1
            }
        )
    else:
        # 完成，跳转到 END
        return Command(
            goto=END,
            update={
                "messages": ["Done"],
                "should_continue": False
            }
        )

def process_node(state: State) -> Command:
    # 处理后返回 decision 节点
    return Command(
        goto="decision",
        update={"messages": ["Processed"]}
    )

# 构建图
builder = StateGraph(State)
builder.add_node("decision", decision_node)
builder.add_node("process", process_node)
builder.add_edge(START, "decision")

graph = builder.compile()
result = graph.invoke({"messages": [], "step": 0, "should_continue": True})
print(result["messages"])
# ["Step 0", "Processed", "Step 1", "Processed", "Step 2", "Processed", "Done"]
```

**Command 对象的优势：**

1. **动态路由**：根据状态决定下一个节点
2. **状态更新**：同时更新状态字段
3. **简化逻辑**：避免使用条件边（conditional edge）

**与条件边的对比：**

```python
# 方式1：使用条件边
def should_continue(state: State) -> str:
    return "process" if state["step"] < 3 else END

builder.add_conditional_edges("decision", should_continue)

# 方式2：使用 Command 对象（更灵活）
def decision_node(state: State) -> Command:
    if state["step"] < 3:
        return Command(goto="process", update={...})
    else:
        return Command(goto=END, update={...})
```

[来源: reference/source_部分状态更新_01.md]

**应用：** 在需要动态路由的复杂工作流中使用，如多步推理、条件分支、循环处理。

### 常见问题速查

**Q1: 什么时候使用 Reducer？**
A: 当字段需要累积数据时（列表、计数器、消息历史）。

**Q2: 如何重置有 Reducer 的字段？**
A: 使用 `Overwrite(value)` 或 `{OVERWRITE: value}`。

**Q3: Pydantic 模型如何部分更新？**
A: 只设置需要更新的字段，其他字段使用默认值。

**Q4: Command 对象什么时候用？**
A: 需要动态路由或同时更新状态和控制流程时。

**Q5: add_messages 和 operator.add 的区别？**
A: add_messages 支持消息更新和删除，operator.add 只是简单拼接。
