## 卡片1：稀疏向量直觉理解

**一句话：** 稀疏向量是只存储非零元素的向量，就像只记录书中出现的关键词。

**举例：**
```python
# 稠密向量：存储所有元素（10000维）
dense_vector = [0, 0, 0.8, 0, 0, ..., 0.3, 0, 0]  # 10000个元素

# 稀疏向量：只存储非零元素
sparse_vector = {2: 0.8, 5005: 0.3, 9999: 0.5}  # 只存储3个元素
```

**应用：** 在 Milvus 2.6 中，BM25 自动生成稀疏向量，用于全文搜索。文档"Kubernetes 集群配置"会生成稀疏向量 `{100: 0.8, 500: 0.3, 1000: 0.5}`，其中 100、500、1000 对应词表中"Kubernetes"、"集群"、"配置"的位置。

**数据来源：** `reference/source_sparse_bm25_01.md` - Milvus 源码分析

## 卡片2：BM25 算法形式化定义

**一句话：** BM25 是 TF-IDF 的改进版，通过 k1 和 b 参数控制词频饱和和文档长度归一化。

**公式：**
```
BM25(q, d) = Σ IDF(qi) × (f(qi, d) × (k1 + 1)) / (f(qi, d) + k1 × (1 - b + b × |d| / avgdl))
```

**参数：**
- **k1**：词频饱和参数（默认 1.2），控制词频增长速度
- **b**：文档长度归一化参数（默认 0.75），控制长文档的惩罚程度
- **IDF**：逆文档频率，降低常见词的权重

**应用：** 在 Milvus 2.6 中，可以通过 `bm25_k1` 和 `bm25_b` 参数调整 BM25 算法的行为。例如，`bm25_k1=1.5` 会让词频增长更快，`bm25_b=0.5` 会减少对长文档的惩罚。

**数据来源：** `reference/context7_pymilvus_01.md` - PyMilvus 官方文档

## 卡片3：稀疏向量数据结构

**一句话：** 稀疏向量使用 `positions[]` + `values[]` 两个数组存储，自动排序，维度 = max(positions) + 1。

**数据结构：**
```python
# Milvus 稀疏向量数据结构（来自源码）
class SparseEmbedding:
    positions: []uint32  # 位置数组（非零元素的索引）
    values: []float32    # 值数组（非零元素的值）
    dim: int             # 维度 = max(positions) + 1
    len: int             # 元素数量 = len(positions)
```

**序列化：**
- 每个元素 8 字节（4 字节位置 + 4 字节值）
- 自动按 position 升序排列
- 例如：`{100: 0.8, 500: 0.3}` 序列化为 `[100, 0.8, 500, 0.3]`（16 字节）

**应用：** 在 Milvus 2.6 中，稀疏向量自动序列化和反序列化，无需手动处理。插入数据时，Milvus 会自动将稀疏向量转换为内部格式。

**数据来源：** `reference/source_sparse_bm25_01.md` - Milvus 源码分析（client/entity/sparse.go）

## 卡片4：WAND 算法原理

**一句话：** WAND（Weak-AND）算法通过上界估计跳过不可能进入 top-k 的文档，实现 4 倍加速。

**核心机制：**
1. **上界估计**：维护每个词项的最大分数
2. **跳跃式遍历**：跳过分数上界低于当前 top-k 最小分数的文档
3. **Block Max-WAND**：维护每个块的最大分数，进一步加速

**对比：**
- **传统倒排索引**：遍历所有包含查询词的文档，计算分数
- **WAND 算法**：只遍历可能进入 top-k 的文档，跳过其他文档

**应用：** 在 Milvus 2.6 中，`SPARSE_WAND` 索引使用 WAND 算法，适用于性能优先的场景。相比 `SPARSE_INVERTED_INDEX`，WAND 索引可以实现 4 倍加速（根据 Timescale 的测试）。

**数据来源：** `reference/search_wand_algorithm_01.md` - WAND 算法与稀疏向量倒排索引

## 卡片5：BM25 Function 实现

**一句话：** BM25 Function 自动从文本生成稀疏向量，流程：分词 → 哈希 → 词频统计 → 稀疏向量。

**实现流程：**
```python
# Milvus 2.6 BM25 Function 实现（来自源码）
# 1. 分词：使用 Analyzer
tokens = analyzer.tokenize("Kubernetes 集群配置")
# ["Kubernetes", "集群", "配置"]

# 2. 哈希：将词转换为 uint32
hashes = [hash("Kubernetes"), hash("集群"), hash("配置")]
# [100, 500, 1000]

# 3. 词频统计：计算每个词的 BM25 分数
sparse_vector = {100: 0.8, 500: 0.3, 1000: 0.5}

# 4. 自动生成稀疏向量
```

**并发处理：**
- Milvus 使用 8 个 goroutine 并发处理
- 适用于大规模文本处理

**应用：** 在 Milvus 2.6 中，只需创建 BM25 Function，插入文本数据时会自动生成稀疏向量，无需手动 embedding。

**数据来源：** `reference/source_sparse_bm25_01.md` - Milvus 源码分析（internal/util/function/bm25_function.go）

## 卡片6：稀疏向量 vs 稠密向量

**一句话：** 稀疏向量基于词频统计（离散），稠密向量基于神经网络（连续），两者互补。

**对比表：**

| 维度 | 稀疏向量 | 稠密向量 |
|------|----------|----------|
| **生成方式** | BM25 算法（词频统计） | Embedding 模型（神经网络） |
| **语义表示** | 关键词匹配（离散） | 语义相似度（连续） |
| **维度** | 10000+（词表大小） | 128-1536（模型输出） |
| **非零元素** | 1-5%（稀疏） | 100%（稠密） |
| **存储** | 400 字节（50 × 8） | 3072 字节（768 × 4） |
| **优势** | 精确关键词匹配 | 理解语义相似度 |
| **局限** | 无法理解同义词 | 对关键词不敏感 |

**应用：** 在 Milvus 2.6 中，混合检索结合稀疏向量和稠密向量，提升 20-40% 的检索准确率。

**数据来源：** `reference/context7_pymilvus_01.md` + `reference/source_sparse_bm25_01.md`

## 卡片7：混合检索策略

**一句话：** 混合检索使用 RRFRanker 或 WeightedRanker 融合稀疏向量和稠密向量的检索结果。

**两种策略：**

**1. RRFRanker（Reciprocal Rank Fusion）**
```python
# 公式：RRF(d) = Σ 1/(k + rank_i(d))
ranker = RRFRanker(k=60)
# 适用场景：不确定权重的场景
```

**2. WeightedRanker（加权融合）**
```python
# 公式：Score = w1 * score1 + w2 * score2
ranker = WeightedRanker(0.7, 0.3)  # 70% 向量，30% BM25
# 适用场景：明确权重的场景
```

**权重调优：**
- 实验驱动：根据具体用例进行实验
- 通常向量检索权重 0.6-0.8，BM25 权重 0.2-0.4

**应用：** 在 Milvus 2.6 中，使用 `hybrid_search` 方法实现混合检索，自动融合多个检索结果。

**数据来源：** `reference/context7_pymilvus_01.md` + `reference/search_hybrid_weight_01.md`

## 卡片8：drop_ratio 参数调优

**一句话：** drop_ratio 参数控制索引大小和搜索精度，通过丢弃低值条目实现性能优化。

**两个参数：**

**1. drop_ratio_build（构建时）**
```python
# 构建索引时丢弃低值条目
index_params = {
    "index_type": "SPARSE_INVERTED_INDEX",
    "metric_type": "BM25",
    "params": {"drop_ratio_build": 0.2}  # 丢弃 20% 最低值
}
# 效果：减少索引大小，加速索引构建
```

**2. drop_ratio_search（搜索时）**
```python
# 搜索时丢弃低值条目
search_params = {
    "metric_type": "BM25",
    "params": {"drop_ratio_search": 0.2}  # 丢弃 20% 最低值
}
# 效果：加速搜索，略微降低精度
```

**权衡：**
- drop_ratio 越大，性能越好，精度越低
- 推荐值：0.1-0.3

**应用：** 在 Milvus 2.6 中，根据场景调整 drop_ratio。性能优先场景使用 0.3，精度优先场景使用 0.1。

**数据来源：** `reference/context7_pymilvus_01.md` - PyMilvus 官方文档
