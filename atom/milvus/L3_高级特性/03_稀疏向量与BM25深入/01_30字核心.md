## 30字核心

**稀疏向量与 BM25 是 Milvus 2.6 实现混合检索的核心技术，结合语义理解与关键词匹配，提升检索准确率 20-40%。**

---

## 核心要素拆解

### 1. 定义部分

**稀疏向量与 BM25** = 稀疏向量数据结构 + BM25 全文搜索算法

- **稀疏向量**：只存储非零元素的向量表示（positions + values）
- **BM25**：改进的 TF-IDF 算法，用于全文搜索和关键词匹配

---

### 2. 核心技术

**混合检索** = 稠密向量（语义相似度）+ 稀疏向量（关键词匹配）

```
查询："如何配置 Kubernetes 集群？"
    ↓
┌─────────────────────────────────────┐
│  稠密向量检索（语义理解）            │
│  找到语义相关的文档                  │
└─────────────────────────────────────┘
    +
┌─────────────────────────────────────┐
│  BM25 检索（关键词匹配）             │
│  找到包含"Kubernetes"、"集群"的文档  │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  混合检索（融合结果）                │
│  RRFRanker 或 WeightedRanker        │
└─────────────────────────────────────┘
    ↓
最终结果（准确率提升 20-40%）
```

---

### 3. 作用/价值

**在 Milvus 2.6 向量检索中的价值：**

1. **补充语义检索的不足**
   - 稠密向量擅长语义相似度，但对关键词匹配不敏感
   - 稀疏向量（BM25）擅长关键词匹配，补充语义检索的盲区

2. **2026 年生产环境标准**
   - Milvus 2.5+ 内置 BM25 全文搜索
   - 自动分词、自动生成稀疏向量
   - 混合检索已成为 RAG 系统的标准配置

3. **性能与成本优化**
   - WAND 算法实现 4 倍加速
   - drop_ratio 参数控制索引大小和搜索精度
   - 适用于大规模文本检索场景

## 与其他概念的对比

### 稀疏向量 vs 稠密向量

| 维度 | 稀疏向量 | 稠密向量 |
|------|----------|----------|
| **存储方式** | 只存储非零元素 | 所有维度都有值 |
| **维度** | 通常 10000+ | 通常 128-1536 |
| **密度** | 1-5% 非零元素 | 100% 非零元素 |
| **擅长场景** | 关键词匹配 | 语义相似度 |
| **典型应用** | BM25 全文搜索 | Embedding 向量检索 |

---

### BM25 vs TF-IDF

| 维度 | BM25 | TF-IDF |
|------|------|--------|
| **算法复杂度** | 引入饱和函数和文档长度归一化 | 简单的词频-逆文档频率 |
| **高频词处理** | 对高频词不敏感（饱和） | 线性增长 |
| **文档长度** | 归一化处理（参数 b） | 不考虑文档长度 |
| **准确率** | 更高 | 较低 |
| **应用场景** | 现代搜索引擎 | 传统信息检索 |

## 核心公式

### BM25 公式

```
BM25(q, d) = Σ IDF(qi) × (f(qi, d) × (k1 + 1)) / (f(qi, d) + k1 × (1 - b + b × |d| / avgdl))
```

**参数说明：**
- `q`：查询
- `d`：文档
- `qi`：查询中的第 i 个词
- `f(qi, d)`：词 qi 在文档 d 中的频率
- `k1`：词频饱和参数（默认 1.2）
- `b`：文档长度归一化参数（默认 0.75）
- `|d|`：文档 d 的长度
- `avgdl`：平均文档长度

---

### 稀疏向量表示

```python
# 稀疏向量格式
sparse_vector = {
    0: 0.5,      # 位置 0，值 0.5
    100: 0.8,    # 位置 100，值 0.8
    500: 0.3     # 位置 500，值 0.3
}

# 等价于稠密向量（10000 维）
# [0.5, 0, 0, ..., 0.8, ..., 0, 0.3, ..., 0]
#  ↑                ↑              ↑
#  位置0            位置100        位置500
```

## 快速代码示例

### 最小可用示例

```python
from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection
from pymilvus import Function, FunctionType

# 1. 连接 Milvus 2.6
connections.connect(host="localhost", port="19530")

# 2. 创建 BM25 Function
bm25_function = Function(
    name="bm25_fn",
    input_field_names=["content"],
    output_field_names="sparse_vector",
    function_type=FunctionType.BM25
)

# 3. 定义 Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=65535),
    FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR)
]
schema = CollectionSchema(fields=fields, functions=[bm25_function])

# 4. 创建 Collection
collection = Collection(name="hybrid_search", schema=schema)

# 5. 创建索引
collection.create_index(
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "BM25",
        "params": {"bm25_k1": 1.2, "bm25_b": 0.75}
    }
)

# 6. 插入数据（自动生成稀疏向量）
data = [
    {"content": "如何配置 Kubernetes 集群？"},
    {"content": "Docker 容器化部署指南"},
    {"content": "Kubernetes 网络配置详解"}
]
collection.insert(data)

# 7. 加载 Collection
collection.load()

# 8. 混合检索
from pymilvus import AnnSearchRequest, RRFRanker

# BM25 检索请求
bm25_req = AnnSearchRequest(
    data=["Kubernetes 集群配置"],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=10
)

# 混合检索（如果有稠密向量字段）
results = collection.hybrid_search(
    reqs=[bm25_req],
    ranker=RRFRanker(k=60),
    limit=5,
    output_fields=["content"]
)

print(f"检索结果：{results[0]}")
```

### 类比记忆

**稀疏向量 = 只记录有内容的页码**
- 一本 10000 页的书，只有 100 页有内容
- 不需要记录所有 10000 页，只记录有内容的 100 页
- 节省存储空间，提升检索效率

**BM25 = 智能的关键词匹配**
- 不是简单的词频统计
- 考虑词频饱和（高频词不会无限增长）
- 考虑文档长度（长文档不会占优势）

**混合检索 = 双重保险**
- 稠密向量：理解"意思"（语义相似度）
- 稀疏向量：匹配"关键词"（精确匹配）
- 两者结合：既理解意思，又匹配关键词
