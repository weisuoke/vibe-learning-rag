# 30字核心

**SKILL.md 标准格式 + 文件系统加载 + 命令式调用，实现 AI Agent 能力的模块化封装与复用。**

---

## 核心要点

Pi-mono 的 Skills 技能包系统是一个模块化的 Agent 能力封装机制，通过 SKILL.md 标准格式 + YAML frontmatter + 文件系统加载，实现 AI Agent 能力的复用、分发和组合。

### 三大核心特性

1. **SKILL.md 标准格式**
   - 使用标准 SKILL.md 文件名
   - YAML frontmatter 定义元数据（name, description）
   - Markdown 正文定义技能逻辑
   - 符合 Agent Skills 开放标准

2. **文件系统加载**
   - 全局技能：`~/.pi/agent/skills/`
   - 项目技能：`./.pi/skills/`
   - 递归扫描：支持子目录组织
   - 自动发现：启动时自动加载

3. **命令式调用**
   - `/skill:name` - 调用技能
   - `/skill:name args` - 传递参数
   - 技能展开：自动注入 System Prompt
   - 技能组合：技能可以调用其他技能

### 为什么是 30 字？

这 30 字浓缩了 Skills 技能包的本质：
- **SKILL.md 标准格式** - 开放标准，跨工具兼容
- **文件系统加载** - 零配置，易于分发
- **命令式调用** - 简洁直观，易于使用
- **模块化封装** - 能力复用，降低复杂度
- **能力复用** - 一次编写，多次使用

### 实际应用场景

1. **代码审查技能**
   ```bash
   /skill:code-review src/components/Button.tsx
   ```
   - 自动分析代码质量
   - 检查潜在问题
   - 提供改进建议

2. **测试驱动开发技能**
   ```bash
   /skill:tdd "实现用户登录功能"
   ```
   - 先写测试用例
   - 再写实现代码
   - 确保测试通过

3. **调试分析技能**
   ```bash
   /skill:debug "用户无法登录"
   ```
   - 系统化排查问题
   - 收集诊断信息
   - 提供解决方案

4. **技能组合**
   ```bash
   /skill:scout-and-plan "添加支付功能"
   ```
   - 探索代码库（调用 explore 技能）
   - 生成实现计划（调用 plan 技能）
   - 执行实现（调用 implement 技能）

## 快速理解

**一句话：** Pi-mono 的 Skills 就像 npm 包之于 Node.js，是 AI Agent 能力的标准化封装和分发单元。

**类比：**
- **TypeScript/Node.js 类比**：就像 npm 包，可以发布、安装、导入和使用
- **日常生活类比**：就像手机 App，每个 App 提供特定功能，可以安装和卸载

**核心价值：**
1. **模块化** - 能力封装，降低复杂度
2. **复用性** - 一次编写，多次使用
3. **可组合** - 技能可以组合成更复杂的工作流
4. **标准化** - 开放标准，跨工具兼容
5. **易分发** - 通过 Git/文件系统轻松分享
