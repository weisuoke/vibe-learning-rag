### 4.1 六大模式速查表

| 模式 | 一句话 | 核心 API | 何时用 |
|------|--------|---------|--------|
| **顺序链式** | A→B→C 线性执行 | `add_edge` / `add_sequence` | 步骤有先后依赖 |
| **路由** | 根据输入选路径 | `add_conditional_edges` | 不同输入需要不同处理 |
| **并行** | 多任务同时跑 | 多条 `add_edge` + reducer | 独立任务需要同时执行 |
| **编排者-工作者** | 动态分任务+并行做 | `@task` + `@entrypoint` | 子任务数量运行时才知道 |
| **评估者-优化者** | 生成→评估→改进循环 | 条件边回指 | 需要迭代提升质量 |
| **Map-Reduce** | 批量数据并行处理 | `Send` API | 对一批数据做相同处理 |

---

### 4.2 模式选择决策树

```
你的任务是什么样的？
│
├─ 步骤之间有严格先后顺序？
│  └─ YES → 顺序链式模式
│
├─ 不同输入需要走不同路径？
│  └─ YES → 路由模式
│
├─ 有多个独立子任务可以同时做？
│  ├─ 子任务数量固定？
│  │  └─ YES → 并行模式
│  └─ 子任务数量运行时才知道？
│     ├─ 需要 LLM 动态分解任务？
│     │  └─ YES → 编排者-工作者模式
│     └─ 只是批量处理数据？
│        └─ YES → Map-Reduce 模式
│
└─ 需要反复改进直到满意？
   └─ YES → 评估者-优化者模式
```

### 4.3 最常用的三个代码模板

#### 模板1：顺序链式（最基础）

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict


class State(TypedDict):
    input: str
    output: str


def step1(state: State):
    return {"output": f"处理1: {state['input']}"}

def step2(state: State):
    return {"output": f"处理2: {state['output']}"}


builder = StateGraph(State)
builder.add_node("step1", step1)
builder.add_node("step2", step2)
builder.add_edge(START, "step1")
builder.add_edge("step1", "step2")
builder.add_edge("step2", END)

graph = builder.compile()
result = graph.invoke({"input": "hello"})
print(result["output"])  # 处理2: 处理1: hello
```

#### 模板2：路由（最常用）

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict


class State(TypedDict):
    input: str
    category: str
    output: str


def classifier(state: State):
    # 实际项目中用 LLM 分类
    if "问题" in state["input"]:
        return {"category": "qa"}
    return {"category": "chat"}

def qa_handler(state: State):
    return {"output": f"回答问题: {state['input']}"}

def chat_handler(state: State):
    return {"output": f"闲聊: {state['input']}"}

def route(state: State):
    return state["category"]


builder = StateGraph(State)
builder.add_node("classifier", classifier)
builder.add_node("qa", qa_handler)
builder.add_node("chat", chat_handler)

builder.add_edge(START, "classifier")
builder.add_conditional_edges("classifier", route, {
    "qa": "qa",
    "chat": "chat",
})
builder.add_edge("qa", END)
builder.add_edge("chat", END)

graph = builder.compile()
```

#### 模板3：Map-Reduce（最灵活）

```python
import operator
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send


class State(TypedDict):
    items: list[str]
    results: Annotated[list[str], operator.add]


class WorkerInput(TypedDict):
    item: str


def splitter(state: State):
    return state  # items 已在输入中

def worker(state: WorkerInput):
    return {"results": [f"处理了: {state['item']}"]}

def reducer(state: State):
    return {"results": state["results"]}

def fan_out(state: State):
    return [Send("worker", {"item": i}) for i in state["items"]]


builder = StateGraph(State)
builder.add_node("splitter", splitter)
builder.add_node("worker", worker)
builder.add_node("reducer", reducer)

builder.add_edge(START, "splitter")
builder.add_conditional_edges("splitter", fan_out, ["worker"])
builder.add_edge("worker", "reducer")
builder.add_edge("reducer", END)

graph = builder.compile()
result = graph.invoke({"items": ["a", "b", "c"]})
print(result["results"])  # ['处理了: a', '处理了: b', '处理了: c']
```

### 4.4 三个必须记住的规则

1. **并行节点必须用 Reducer**：多个节点写同一个字段，没有 reducer 会互相覆盖
   ```python
   # ✅ 正确
   results: Annotated[list, operator.add]
   # ❌ 错误：并行节点写入会覆盖
   result: str
   ```

2. **循环必须有终止条件**：评估者-优化者模式一定要设最大次数
   ```python
   if state["attempts"] >= 3:
       return "done"  # 防止无限循环
   ```

3. **Send 的目标节点必须声明**：`add_conditional_edges` 的第三个参数要列出所有可能的目标
   ```python
   builder.add_conditional_edges("splitter", fan_out, ["worker"])
   #                                                   ^^^^^^^^ 必须声明
   ```

### 4.5 模式组合的基本原则

实际项目中很少只用一种模式，通常是组合使用：

```
常见组合：
- 路由 + 顺序链式：分类后走不同的处理链
- 并行 + 顺序链式：并行处理后串行汇总
- 编排者-工作者 + 评估者-优化者：动态分任务 + 迭代优化
- Map-Reduce + 路由：批量处理中每个 Worker 内部有路由
```
