# 核心概念 1：通道配对机制

> **理解 OpenClaw 如何通过 DM 配对确保消息安全送达**

---

## 概念定义

**通道配对（Channel Pairing）** 是 OpenClaw 中确保消息发送到正确接收者的安全验证机制。通过 **DM（Direct Message，私信）配对**，系统要求发送者和接收者通过私信交换验证码，建立可信的通信连接。

**核心价值**：
- 🔐 **安全性**：防止消息误发到错误的接收者
- ✅ **主动确认**：接收者主动参与验证，而非被动接受
- 🔄 **动态配对**：无需预先配置，支持实时配对
- 🌐 **跨平台**：统一的配对机制适用于所有通道

---

## 为什么需要配对？

### 问题场景

假设你想通过 OpenClaw 向某人的 WhatsApp 发送消息：

```bash
openclaw message send "Hello, this is important!"
```

**问题**：
1. 系统如何知道"某人"是谁？
2. 如何确保消息发送到正确的 WhatsApp 账号？
3. 如何防止消息发送到同名的其他人？

### 传统方案的问题

**方案 A：预先配置**
```yaml
# config.yaml
contacts:
  - name: "John"
    whatsapp: "+1234567890"
```

**问题**：
- 需要手动维护配置文件
- 电话号码可能变化
- 容易出错（输入错误的号码）
- 无法验证号码是否正确

**方案 B：每次手动指定**
```bash
openclaw message send --to "+1234567890" "Hello"
```

**问题**：
- 每次都要输入完整的联系方式
- 容易输入错误
- 无法验证接收者身份

### OpenClaw 的解决方案：DM 配对

**核心思想**：通过私信验证身份

```
1. 系统生成唯一配对码
   ↓
2. 用户通过私信发送配对码给接收者
   ↓
3. 接收者收到配对码，系统验证
   ↓
4. 配对成功，建立可信连接
   ↓
5. 后续消息自动发送到已配对的接收者
```

**优势**：
- ✅ 接收者主动确认，安全性高
- ✅ 无需预先配置
- ✅ 支持动态变化
- ✅ 一次配对，多次使用

#### 步骤 3：生成配对码

```typescript
// src/pairing/pairing-messages.ts:67
export function generatePairingCode(): string {
  // 生成唯一的配对码（8位字母数字组合）
  const characters = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
  let code = '';
  for (let i = 0; i < 8; i++) {
    code += characters.charAt(Math.floor(Math.random() * characters.length));
  }
  return code;
}
```

**配对码特性**：
- 长度：8位
- 字符集：大写字母 + 数字
- 唯一性：每次生成都不同
- 有效期：默认 10 分钟

**输出示例**：
```
✓ Pairing code generated: ABC123XY
✓ Please send this code to the recipient via WhatsApp
✓ Waiting for verification...
```

#### 步骤 4：发送配对码

用户需要手动通过 WhatsApp 将配对码发送给接收者：

```
发送者 → 接收者（WhatsApp）:
"Hi, please pair with OpenClaw using code: ABC123XY"
```

**重要**：
- 必须通过私信（DM）发送，不能在群组中发送
- 配对码区分大小写
- 配对码有时效性（10分钟内有效）

#### 步骤 5：接收者发送配对码

接收者收到配对码后，通过 WhatsApp 回复配对码：

```
接收者 → OpenClaw Bot（WhatsApp）:
"ABC123XY"
```

**系统行为**：
- Gateway 监听所有入站消息
- 检测到配对码格式的消息
- 验证配对码是否有效

**源码参考**：`src/pairing/pairing-messages.ts:89`

```typescript
// src/pairing/pairing-messages.ts:89
export async function verifyPairingCode(
  code: string,
  sender: string,
  channelType: string
): Promise<PairingResult> {
  // 1. 检查配对码是否存在
  const pairing = await findPendingPairing(code);
  if (!pairing) {
    return { success: false, error: 'Invalid pairing code' };
  }

  // 2. 检查配对码是否过期
  if (isPairingExpired(pairing)) {
    return { success: false, error: 'Pairing code expired' };
  }

  // 3. 验证通道类型匹配
  if (pairing.channelType !== channelType) {
    return { success: false, error: 'Channel type mismatch' };
  }

  // 4. 保存配对信息
  await savePairing({
    code,
    sender,
    channelType,
    timestamp: Date.now(),
    status: 'active'
  });

  return { success: true, pairingId: pairing.id };
}
```

#### 步骤 6：验证配对码

**验证逻辑**：

```typescript
// 验证流程

2. 配对码有效性检查

```

**验证成功条件**：
- ✅ 配对码存在且未使用
- ✅ 配对码在有效期内
- ✅ 通道类型匹配
- ✅ 接收者身份可识别

## 安全机制

### 1. 配对码唯一性

```typescript
// src/pairing/pairing-service.ts:112
export async function ensureUniquePairingCode(): Promise<string> {
  let code: string;
  let attempts = 0;
  const maxAttempts = 10;

  do {
    code = generatePairingCode();
    const exists = await checkPairingCodeExists(code);

    if (!exists) {
      return code;
    }

    attempts++;
  } while (attempts < maxAttempts);

  throw new Error('Failed to generate unique pairing code');
}
```

**保证**：
- 每个配对码全局唯一
- 避免配对码冲突
- 最多尝试 10 次生成

### 2. 配对码过期机制

```typescript
// src/pairing/pairing-service.ts:134
export function isPairingExpired(pairing: Pairing): boolean {
  const now = Date.now();
  const createdAt = pairing.createdAt;
  const expiryTime = 10 * 60 * 1000; // 10 分钟

  return (now - createdAt) > expiryTime;
}
```

**特性**：
- 默认有效期：10 分钟
- 过期后自动失效
- 防止配对码被长期滥用

### 3. 通道类型验证

```typescript
// src/pairing/pairing-service.ts:156
export function validateChannelType(
  requestedChannel: string,
  actualChannel: string
): boolean {
  // 确保配对码只能在指定的通道类型中使用
  return requestedChannel === actualChannel;
}
```

**防止**：
- 配对码跨通道使用
- 例如：WhatsApp 的配对码不能在 Telegram 中使用

### 4. 私信验证

```typescript
// src/pairing/pairing-service.ts:178
export function isDirectMessage(message: Message): boolean {
  // 只接受私信中的配对码，拒绝群组消息
  return message.chatType === 'direct' && !message.isGroupMessage;
}
```

**安全性**：
- 只接受私信中的配对码
- 拒绝群组消息中的配对码
- 防止配对码泄露

## 配对机制的优势

### 1. 安全性

**传统方式**：
```yaml
# 配置文件中明文存储联系方式
contacts:
  - name: "John"
    whatsapp: "+1234567890"
    telegram: "@johndoe"
```

**问题**：
- 配置文件可能被泄露
- 无法验证联系方式是否正确
- 容易输入错误

**OpenClaw 方式**：
```bash
# 通过私信验证，无需明文存储
openclaw pairing start
# 接收者主动确认，安全性高
```

**优势**：
- 接收者主动参与验证
- 配对码有时效性
- 只接受私信验证

### 2. 灵活性

**传统方式**：
- 需要预先配置所有联系人
- 联系方式变化需要手动更新
- 添加新联系人需要修改配置

**OpenClaw 方式**：
- 动态配对，无需预先配置
- 支持实时添加新联系人
- 联系方式变化自动更新

### 3. 用户体验

**传统方式**：
```bash
# 每次都要指定完整的联系方式
send-message --to "+1234567890" --channel whatsapp "Hello"
```

**OpenClaw 方式**：
```bash
# 配对后，简单命令即可
openclaw message send "Hello"
# 自动发送到所有已配对的接收者
```

## 常见问题

### Q1: 配对码可以重复使用吗？

**A**: 不可以。每个配对码只能使用一次，验证成功后立即失效。

```typescript
// src/pairing/pairing-service.ts:201
export async function markPairingCodeAsUsed(code: string): Promise<void> {
  await updatePairingStatus(code, 'used');
  // 配对码标记为已使用，无法再次使用
}
```

### Q2: 配对码有效期是多久？

**A**: 默认 10 分钟。可以通过配置修改：

```yaml
# ~/.openclaw/config.yaml
pairing:
  codeExpiryMinutes: 10  # 修改为其他值
```

### Q3: 可以同时配对多个接收者吗？

**A**: 可以。每个通道可以配对一个接收者，不同通道可以配对不同的接收者。

```bash
# 配对 WhatsApp 的 John
openclaw pairing start  # 选择 WhatsApp

# 配对 Telegram 的 Jane
openclaw pairing start  # 选择 Telegram

# 发送消息到所有已配对的接收者
openclaw message send "Hello"
# 消息会发送到 John (WhatsApp) 和 Jane (Telegram)
```

### Q4: 配对信息存储在哪里？

**A**: 存储在本地配置文件中：

```bash
~/.openclaw/pairings.json
```

**安全性**：
- 文件权限：只有当前用户可读写
- 敏感信息加密存储
- 不会上传到云端

### Q5: 如何重新配对？

**A**: 删除旧配对，重新启动配对流程：

```bash
# 删除特定配对
openclaw pairing delete pair_abc123def456

# 或删除所有配对
openclaw pairing clear

# 重新配对
openclaw pairing start
```
