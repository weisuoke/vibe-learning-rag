# 30字核心

**RAG Prompt 是告诉 LLM "基于给定内容回答问题"的指令模板，是控制 RAG 输出质量的关键。**

---

## 拆解理解

| 关键词 | 含义 |
|-------|------|
| **基于给定内容** | LLM 只能使用你注入的检索结果，不能自由发挥 |
| **回答问题** | 明确任务是问答，不是闲聊或创作 |
| **指令模板** | 可复用的结构化格式，不是每次重写 |
| **控制输出质量** | Prompt 设计直接影响回答的准确性和格式 |

---

## 为什么这句话重要？

理解这句话，你就理解了：

1. **RAG 的核心约束** → LLM 必须"基于检索内容"，这是 Prompt 要强调的
2. **Prompt 是唯一控制手段** → 你无法修改 LLM 内部，只能通过 Prompt 引导
3. **模板化的价值** → 好的模板可以复用，减少每次调试的成本

---

## RAG Prompt 的三个组成部分

```
┌─────────────────────────────────────────┐
│           RAG Prompt 结构               │
├─────────────────────────────────────────┤
│  1. 系统提示词（System Prompt）          │
│     → 角色设定 + 行为约束                │
├─────────────────────────────────────────┤
│  2. 上下文注入（Context）                │
│     → 检索到的文档片段                   │
├─────────────────────────────────────────┤
│  3. 用户问题（User Query）               │
│     → 格式化后的用户问题                 │
└─────────────────────────────────────────┘
```
