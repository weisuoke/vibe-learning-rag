# 核心概念 2：端口配置与绑定

## 概述

端口配置与绑定是 OpenClaw Gateway 网络安全的核心机制，决定了 Gateway 如何监听网络连接以及谁可以访问。OpenClaw 提供三种主要绑定模式：**loopback（本地回环）**、**lan（局域网）**和 **tailnet（Tailscale 网络）**，每种模式适用于不同的安全场景和访问需求。

## 一句话定义

**端口配置与绑定是控制 Gateway 网络监听策略的安全机制，通过 loopback（仅本地）、lan（局域网 + 强制认证）、tailnet（Tailscale 安全隧道）三种模式，实现从本地开发到远程访问的分层安全控制。**

---

## 第一性原理：为什么需要端口绑定策略？

### 问题根源

任何网络服务都面临一个基本矛盾：

1. **可访问性需求**：
   - 开发时需要本地访问
   - 生产时需要远程访问
   - 多设备需要跨网络访问

2. **安全性需求**：
   - 防止未授权访问
   - 避免暴露到公网
   - 保护敏感数据

这两个需求天然冲突：越容易访问，越不安全；越安全，越难访问。

### 解决方案

OpenClaw 通过分层的端口绑定策略解决这个矛盾：

- **loopback**：默认最安全，只允许本地访问
- **lan**：扩展到局域网，强制认证
- **tailnet**：通过 Tailscale 实现安全远程访问

每一层都在可访问性和安全性之间找到平衡点。

## 端口配置基础

### 默认端口

OpenClaw Gateway 默认使用端口 **18789**。

**为什么是 18789？**
- 不与常见服务冲突（避开 80, 443, 8080 等）
- 在用户端口范围内（1024-65535）
- 易于记忆

### 端口配置方式

**1. 配置文件：**
```json5
// ~/.openclaw/openclaw.json
{
  gateway: {
    port: 18789
  }
}
```

**2. CLI 参数：**
```bash
openclaw gateway --port 18789
```

**3. 环境变量：**
```bash
export OPENCLAW_GATEWAY_PORT=18789
openclaw gateway
```

**优先级：** CLI 参数 > 环境变量 > 配置文件 > 默认值

### 端口冲突处理

**检查端口占用：**
```bash
# macOS/Linux
lsof -i :18789

# 或使用 OpenClaw 诊断
openclaw gateway probe
```

**强制释放端口：**
```bash
# 自动杀死占用进程并启动
openclaw gateway --force
```

**使用不同端口：**
```bash
# 临时使用其他端口
openclaw gateway --port 18790
```

## 绑定模式详解

### 1. loopback 模式（默认）

#### 定义

**loopback 模式**将 Gateway 绑定到 `127.0.0.1`（localhost），只允许本机访问。

#### 特征

| 特征 | 说明 |
|------|------|
| **监听地址** | 127.0.0.1 |
| **可访问范围** | 仅本机 |
| **安全级别** | ✅ 最高 |
| **认证要求** | 可选（推荐启用） |
| **网络暴露** | 无 |

#### 配置

**配置文件：**
```json5
{
  gateway: {
    port: 18789,
    bind: "loopback"
  }
}
```

**CLI：**
```bash
openclaw gateway --bind loopback
```

#### 使用场景

**场景 1：本地开发**
```bash
# 开发者在本机开发调试
openclaw gateway --bind loopback --verbose
```

**场景 2：单用户桌面使用**
```bash
# 个人电脑上使用 OpenClaw
openclaw gateway --bind loopback
```

**场景 3：通过 SSH 隧道远程访问**
```bash
# 服务器上运行 Gateway（loopback）
openclaw gateway --bind loopback

# 本地通过 SSH 隧道访问
ssh -L 18789:localhost:18789 user@server
```

#### 优点

✅ **最安全**：无网络暴露
✅ **无需认证**：本地访问可信
✅ **简单直接**：默认配置
✅ **适合开发**：快速迭代

#### 缺点

❌ **无法远程访问**：其他设备无法连接
❌ **需要 SSH 隧道**：远程访问需要额外配置
❌ **单机限制**：无法多设备协作

#### 最佳实践

1. **开发环境默认使用 loopback**
   ```bash
   openclaw gateway --bind loopback
   ```

2. **即使 loopback 也建议启用认证**
   ```json5
   {
     gateway: {
       bind: "loopback",
       auth: {
         mode: "token",
         token: "${OPENCLAW_GATEWAY_TOKEN}"
       }
     }
   }
   ```

3. **远程访问使用 SSH 隧道**
   ```bash
   ssh -L 18789:localhost:18789 user@server
   ```

---

### 2. lan 模式（局域网访问）

#### 定义

**lan 模式**将 Gateway 绑定到 `0.0.0.0`（所有接口），允许局域网内的设备访问。

#### 特征

| 特征 | 说明 |
|------|------|
| **监听地址** | 0.0.0.0 |
| **可访问范围** | 局域网内所有设备 |
| **安全级别** | ⚠️ 中等（需要认证） |
| **认证要求** | ✅ 强制（token 或 password） |
| **网络暴露** | 局域网 |

#### 配置

**配置文件：**
```json5
{
  gateway: {
    port: 18789,
    bind: "lan",
    auth: {
      mode: "token",
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  }
}
```

**CLI：**
```bash
openclaw gateway --bind lan --token <long-random-token>
```

#### 认证要求

**OpenClaw 强制要求：**
- lan 模式**必须**启用认证
- 不允许 `auth: "none"` 与 `bind: "lan"` 组合
- Gateway 会拒绝启动如果没有认证

**错误示例：**
```bash
$ openclaw gateway --bind lan --auth none

Error: Refusing to bind gateway to lan without auth.
Set gateway.auth.token/password or pass --token/--password.
```

#### 使用场景

**场景 1：家庭网络多设备访问**
```bash
# 台式机运行 Gateway
openclaw gateway --bind lan --token <token>

# 笔记本、手机通过 LAN 访问
# ws://192.168.1.100:18789
```

**场景 2：办公室内部使用**
```bash
# 服务器运行 Gateway
openclaw gateway --bind lan --token <token>

# 团队成员通过办公网络访问
```

**场景 3：iOS/Android 应用访问**
```bash
# Gateway 绑定 LAN
openclaw gateway --bind lan --token <token>

# 移动应用通过 LAN 连接
```

#### 优点

✅ **多设备访问**：局域网内所有设备可连接
✅ **无需隧道**：直接网络连接
✅ **性能好**：无额外加密开销
✅ **适合团队**：多人协作

#### 缺点

❌ **需要认证**：必须配置 token/password
❌ **局域网限制**：无法跨网络访问
❌ **安全风险**：局域网内可能有不可信设备
❌ **防火墙配置**：可能需要配置防火墙规则

#### 安全最佳实践

**1. 使用强 Token**
```bash
# 生成 32 字符随机 token
openssl rand -base64 32

# 使用环境变量
export OPENCLAW_GATEWAY_TOKEN="<long-random-token>"
openclaw gateway --bind lan
```

**2. 配置防火墙**
```bash
# macOS (pf)
echo "pass in proto tcp from 192.168.1.0/24 to any port 18789" | sudo pfctl -f -

# Linux (ufw)
sudo ufw allow from 192.168.1.0/24 to any port 18789
```

**3. 定期轮换 Token**
```bash
# 每月更换 token
export OPENCLAW_GATEWAY_TOKEN="<new-token>"
openclaw gateway restart
```

**4. 运行安全审计**
```bash
# 检查安全配置
openclaw security audit

# 深度审计
openclaw security audit --deep
```

### 3. tailnet 模式（Tailscale 网络）

#### 定义

**tailnet 模式**将 Gateway 绑定到 Tailscale 虚拟网络接口，只允许 Tailscale 网络内的设备访问。

#### 特征

| 特征 | 说明 |
|------|------|
| **监听地址** | Tailscale 接口（如 100.x.x.x） |
| **可访问范围** | Tailscale 网络内设备 |
| **安全级别** | ✅ 高（端到端加密） |
| **认证要求** | 可选（Tailscale 提供身份认证） |
| **网络暴露** | Tailscale 网络 |

#### 配置

**配置文件：**
```json5
{
  gateway: {
    port: 18789,
    bind: "tailnet",
    tailscale: {
      mode: "serve"  // 或 "funnel"（公网）
    }
  }
}
```

**CLI：**
```bash
openclaw gateway --bind tailnet --tailscale serve
```

#### Tailscale 模式

**1. serve 模式（推荐）**
- 只在 Tailscale 网络内可访问
- 端到端加密
- 需要 Tailscale 客户端

**2. funnel 模式（公网）**
- 通过 Tailscale 暴露到公网
- 仍然加密
- ⚠️ 任何人都可以访问（需要强认证）

#### 使用场景

**场景 1：跨网络远程访问**
```bash
# 家里的服务器运行 Gateway
openclaw gateway --bind tailnet --tailscale serve

# 办公室、咖啡厅通过 Tailscale 访问
# 无需 VPN 或端口转发
```

**场景 2：团队协作**
```bash
# 团队共享 Tailscale 网络
openclaw gateway --bind tailnet --tailscale serve

# 团队成员通过 Tailscale 访问
```

**场景 3：移动设备访问**
```bash
# Gateway 绑定 tailnet
openclaw gateway --bind tailnet --tailscale serve

# iOS/Android 安装 Tailscale 后访问
```

#### 优点

✅ **安全远程访问**：端到端加密
✅ **无需端口转发**：Tailscale 自动穿透 NAT
✅ **跨网络访问**：任何网络都可以访问
✅ **身份认证**：Tailscale 提供身份验证

#### 缺点

❌ **需要 Tailscale**：所有设备需要安装 Tailscale
❌ **依赖第三方**：依赖 Tailscale 服务
❌ **配置复杂**：需要配置 Tailscale 网络
❌ **性能开销**：加密和路由有轻微开销

#### Tailscale 集成

**1. 安装 Tailscale**
```bash
# macOS
brew install tailscale

# Linux
curl -fsSL https://tailscale.com/install.sh | sh

# 启动 Tailscale
sudo tailscale up
```

**2. 配置 Gateway**
```bash
# 启动 Gateway（tailnet 模式）
openclaw gateway --bind tailnet --tailscale serve
```

**3. 信任代理认证（可选）**
```json5
{
  gateway: {
    bind: "tailnet",
    auth: { mode: "trusted-proxy" },
    tailscale: { mode: "serve" }
  }
}
```

**Tailscale 会在 HTTP 头中提供身份信息：**
- `Tailscale-User-Login`: 用户邮箱
- `Tailscale-User-Name`: 用户名
- `Tailscale-User-Profile-Pic`: 头像 URL

### 4. auto 模式（自动选择）

#### 定义

**auto 模式**根据 Tailscale 可用性自动选择绑定模式。

#### 逻辑

```
if Tailscale is running:
    bind to tailnet
else:
    bind to loopback
```

#### 配置

```json5
{
  gateway: {
    bind: "auto"
  }
}
```

#### 使用场景

**场景：灵活部署**
```bash
# 在不同环境自动适配
openclaw gateway --bind auto

# 有 Tailscale：自动使用 tailnet
# 无 Tailscale：自动使用 loopback
```

---

### 5. custom 模式（自定义地址）

#### 定义

**custom 模式**允许绑定到特定 IP 地址。

#### 配置

```json5
{
  gateway: {
    bind: "custom",
    customBind: "192.168.1.100"
  }
}
```

#### 使用场景

**场景：多网卡服务器**
```bash
# 绑定到特定网卡
openclaw gateway --bind custom --custom-bind 192.168.1.100
```

## 绑定模式对比

### 功能对比表

| 特性 | loopback | lan | tailnet | auto | custom |
|------|----------|-----|---------|------|--------|
| **监听地址** | 127.0.0.1 | 0.0.0.0 | Tailscale IP | 自动 | 自定义 |
| **可访问范围** | 仅本机 | 局域网 | Tailscale 网络 | 自动 | 取决于地址 |
| **安全级别** | ✅ 最高 | ⚠️ 中等 | ✅ 高 | 自动 | 取决于配置 |
| **认证要求** | 可选 | ✅ 强制 | 可选 | 自动 | 取决于配置 |
| **远程访问** | ❌ 需要隧道 | ❌ 仅局域网 | ✅ 支持 | 自动 | 取决于配置 |
| **加密** | 无需 | 无 | ✅ 端到端 | 自动 | 无 |
| **配置复杂度** | ✅ 简单 | ⚠️ 中等 | ❌ 复杂 | ✅ 简单 | ⚠️ 中等 |

### 使用场景对比

| 场景 | 推荐模式 | 原因 |
|------|----------|------|
| **本地开发** | loopback | 最安全，无需配置 |
| **家庭网络** | lan + 认证 | 多设备访问，局域网安全 |
| **办公室内部** | lan + 认证 | 团队协作，局域网隔离 |
| **远程访问** | tailnet | 安全加密，跨网络 |
| **移动设备** | tailnet 或 lan | 取决于网络环境 |
| **生产部署** | loopback + SSH 或 tailnet | 最高安全性 |

## 认证策略

### 认证模式

OpenClaw 支持四种认证模式：

#### 1. token 模式（推荐）

**定义：** 使用共享密钥（token）进行认证。

**配置：**
```json5
{
  gateway: {
    auth: {
      mode: "token",
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  }
}
```

**客户端连接：**
```typescript
const client = new GatewayClient({
  url: "ws://127.0.0.1:18789",
  auth: { token: "long-random-token-here" }
});
```

**优点：**
- ✅ 简单直接
- ✅ 易于轮换
- ✅ 适合自动化

**缺点：**
- ❌ 需要安全存储
- ❌ 泄露风险

#### 2. password 模式

**定义：** 使用密码进行认证。

**配置：**
```json5
{
  gateway: {
    auth: {
      mode: "password",
      password: "${OPENCLAW_GATEWAY_PASSWORD}"
    }
  }
}
```

**优点：**
- ✅ 用户友好
- ✅ 易于记忆

**缺点：**
- ❌ 密码强度依赖用户
- ❌ 不适合自动化

#### 3. trusted-proxy 模式

**定义：** 信任反向代理提供的身份信息。

**配置：**
```json5
{
  gateway: {
    auth: { mode: "trusted-proxy" },
    tailscale: { mode: "serve" }
  }
}
```

**使用场景：**
- Tailscale Serve（身份头）
- Nginx 反向代理（X-Forwarded-* 头）

**⚠️ 安全警告：**
- 只能用于可信的反向代理
- 永远不要在公网暴露 trusted-proxy 模式

#### 4. none 模式（危险）

**定义：** 无认证。

**配置：**
```json5
{
  gateway: {
    auth: { mode: "none" }
  }
}
```

**限制：**
- 只能与 `bind: "loopback"` 组合
- Gateway 会拒绝 `bind: "lan"` + `auth: "none"`

**⚠️ 安全警告：**
- 即使 loopback 也不推荐
- 可能被本地恶意软件利用

## 安全最佳实践

### 1. 默认使用 loopback

```json5
{
  gateway: {
    bind: "loopback",
    auth: {
      mode: "token",
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  }
}
```

### 2. lan 模式必须强认证

```json5
{
  gateway: {
    bind: "lan",
    auth: {
      mode: "token",
      token: "<at-least-32-characters-long-random-token>"
    }
  }
}
```

### 3. 使用 Tailscale 进行远程访问

```json5
{
  gateway: {
    bind: "tailnet",
    tailscale: { mode: "serve" },
    auth: { mode: "trusted-proxy" }
  }
}
```

### 4. 配置防火墙规则

```bash
# macOS (pf)
sudo pfctl -e
echo "pass in proto tcp from 192.168.1.0/24 to any port 18789" | sudo pfctl -f -

# Linux (ufw)
sudo ufw allow from 192.168.1.0/24 to any port 18789
sudo ufw enable
```

### 5. 定期运行安全审计

```bash
# 运行安全审计
openclaw security audit

# 深度审计（包含实时探测）
openclaw security audit --deep

# 自动修复问题
openclaw security audit --fix
```

### 6. 使用环境变量存储敏感信息

```bash
# .env 文件
OPENCLAW_GATEWAY_TOKEN=<long-random-token>

# 配置文件引用
{
  gateway: {
    auth: {
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  }
}
```

## 常见安全错误

### 错误 1：lan 模式无认证

**错误配置：**
```json5
{
  gateway: {
    bind: "lan",
    auth: { mode: "none" }
  }
}
```

**结果：** Gateway 拒绝启动

**正确配置：**
```json5
{
  gateway: {
    bind: "lan",
    auth: {
      mode: "token",
      token: "${OPENCLAW_GATEWAY_TOKEN}"
    }
  }
}
```

### 错误 2：弱 Token

**错误配置：**
```json5
{
  gateway: {
    auth: {
      token: "123456"  // 太短！
    }
  }
}
```

**结果：** 安全审计警告

**正确配置：**
```bash
# 生成强 token
openssl rand -base64 32
```

### 错误 3：公网暴露

**错误操作：**
```bash
# 路由器端口转发
0.0.0.0:18789 -> 公网 IP
```

**结果：** 高安全风险

**正确方案：**
```bash
# 使用 Tailscale
openclaw gateway --bind tailnet --tailscale serve
```

### 错误 4：trusted-proxy 公网暴露

**错误配置：**
```json5
{
  gateway: {
    bind: "lan",
    auth: { mode: "trusted-proxy" }
  }
}
```

**结果：** 任何人都可以伪造身份头

**正确配置：**
```json5
{
  gateway: {
    bind: "tailnet",
    auth: { mode: "trusted-proxy" },
    tailscale: { mode: "serve" }
  }
}
```

## 总结

端口配置与绑定是 OpenClaw Gateway 网络安全的基础：

- **loopback**：默认最安全，适合本地开发
- **lan**：局域网访问，强制认证
- **tailnet**：安全远程访问，端到端加密

选择合适的绑定模式和认证策略，可以在可访问性和安全性之间找到最佳平衡点。始终遵循"默认最安全"原则，只在必要时扩展访问范围。
