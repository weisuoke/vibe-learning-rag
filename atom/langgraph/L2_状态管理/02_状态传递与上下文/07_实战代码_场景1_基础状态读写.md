# 实战代码 - 场景1：基础状态读写

> 本文档演示 LangGraph 中最基础的状态读写操作，帮助你理解状态在节点间的传递机制

---

## 场景描述

**目标**：构建一个简单的笑话生成工作流，演示状态在节点间的读写和传递。

**核心知识点**：
- StateGraph 的创建和配置
- 节点如何读取状态（state 参数）
- 节点如何写入状态（返回字典）
- 状态的自动合并机制
- 基础的状态流转路径

**适用场景**：
- 初学者理解 LangGraph 状态管理
- 构建简单的多步骤工作流
- 理解状态传递的基本原理

---

## 核心原理回顾

### 状态传递的基本流程

```
StateGraph.channels (存储)
    ↓
PregelNode.channels (声明输入)
    ↓
ChannelRead.do_read() (读取)
    ↓
PregelNode.bound (节点逻辑)
    ↓
ChannelWrite.do_write() (写入)
    ↓
StateGraph.channels (更新)
```

### 节点与状态的交互

**节点读取状态**：
- 节点函数接收 `state` 参数
- `state` 是一个字典，包含当前所有状态字段
- 节点可以读取任何已定义的状态字段

**节点写入状态**：
- 节点函数返回一个字典
- 返回的字典包含需要更新的状态字段
- LangGraph 自动将返回值合并到状态中

**状态合并规则**：
- 如果字段没有定义 reducer，则直接覆盖
- 如果字段定义了 reducer（如 `add_messages`），则按 reducer 规则合并
- 未返回的字段保持不变

## 实战代码

### 场景1：基础笑话生成器

**功能**：用户输入一个主题，系统生成笑话并评分

```python
"""
场景1：基础状态读写 - 笑话生成器
演示：状态在节点间的读取、写入和传递
"""

import os
from typing import TypedDict
from dotenv import load_dotenv

from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, START, END

# ===== 1. 环境配置 =====
load_dotenv()

# 初始化 LLM
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7
)

# ===== 2. 定义状态 Schema =====
class JokeState(TypedDict):
    """
    笑话生成器的状态定义

    Attributes:
        topic: 用户输入的主题
        joke: 生成的笑话
        score: 笑话的评分（1-10）
        feedback: 评分反馈
    """
    topic: str
    joke: str
    score: int
    feedback: str


# ===== 3. 定义节点函数 =====

def generate_joke(state: JokeState) -> dict:
    """
    节点1：生成笑话

    读取：state["topic"]
    写入：{"joke": "..."}
    """
    print(f"\n[节点1] 正在生成关于 '{state['topic']}' 的笑话...")

    # 从状态中读取主题
    topic = state["topic"]

    # 调用 LLM 生成笑话
    prompt = f"请用中文写一个关于 {topic} 的简短笑话（不超过50字）"
    response = llm.invoke(prompt)
    joke = response.content

    print(f"[节点1] 生成的笑话：{joke}")

    # 返回字典，更新状态
    return {"joke": joke}


def rate_joke(state: JokeState) -> dict:
    """
    节点2：评分笑话

    读取：state["joke"]
    写入：{"score": 8, "feedback": "..."}
    """
    print(f"\n[节点2] 正在评分笑话...")

    # 从状态中读取笑话
    joke = state["joke"]

    # 调用 LLM 评分
    prompt = f"""
    请对以下笑话进行评分（1-10分）并给出简短反馈：

    笑话：{joke}

    请以 JSON 格式返回：{{"score": 8, "feedback": "很有趣"}}
    """
    response = llm.invoke(prompt)

    # 简单解析（实际项目中应使用更健壮的解析方法）
    import json
    try:
        result = json.loads(response.content)
        score = result.get("score", 5)
        feedback = result.get("feedback", "无评价")
    except:
        score = 5
        feedback = "评分失败"

    print(f"[节点2] 评分：{score}/10")
    print(f"[节点2] 反馈：{feedback}")

    # 返回字典，更新状态
    return {
        "score": score,
        "feedback": feedback
    }


# ===== 4. 构建图 =====

# 创建 StateGraph
workflow = StateGraph(JokeState)

# 添加节点
workflow.add_node("generate_joke", generate_joke)
workflow.add_node("rate_joke", rate_joke)

# 定义边（执行顺序）
workflow.add_edge(START, "generate_joke")
workflow.add_edge("generate_joke", "rate_joke")
workflow.add_edge("rate_joke", END)

# 编译图
graph = workflow.compile()


# ===== 5. 运行图 =====

def run_joke_generator(topic: str):
    """运行笑话生成器"""
    print("=" * 60)
    print(f"笑话生成器启动 - 主题：{topic}")
    print("=" * 60)

    # 初始状态
    initial_state = {
        "topic": topic,
        "joke": "",
        "score": 0,
        "feedback": ""
    }

    # 执行图
    final_state = graph.invoke(initial_state)

    # 输出最终结果
    print("\n" + "=" * 60)
    print("最终结果")
    print("=" * 60)
    print(f"主题：{final_state['topic']}")
    print(f"笑话：{final_state['joke']}")
    print(f"评分：{final_state['score']}/10")
    print(f"反馈：{final_state['feedback']}")
    print("=" * 60)

    return final_state


# ===== 6. 测试运行 =====

if __name__ == "__main__":
    # 测试1：程序员笑话
    run_joke_generator("程序员")

    # 测试2：猫咪笑话
    run_joke_generator("猫咪")
```

## 场景2：多步骤数据处理

**功能**：演示状态在多个节点间的累积更新

```python
"""
场景2：多步骤数据处理
演示：状态的累积更新和部分更新
"""

from typing import TypedDict, List

# ===== 1. 定义状态 =====
class DataProcessingState(TypedDict):
    """数据处理状态"""
    raw_data: str           # 原始数据
    cleaned_data: str       # 清洗后的数据
    tokens: List[str]       # 分词结果
    word_count: int         # 词数统计
    summary: str            # 摘要
    processing_steps: List[str]  # 处理步骤记录


# ===== 2. 定义节点 =====

def clean_data(state: DataProcessingState) -> dict:
    """节点1：数据清洗"""
    print("\n[节点1] 数据清洗...")

    raw = state["raw_data"]
    # 简单清洗：去除多余空格
    cleaned = " ".join(raw.split())

    # 记录处理步骤
    steps = state.get("processing_steps", [])
    steps.append("数据清洗完成")

    return {
        "cleaned_data": cleaned,
        "processing_steps": steps
    }


def tokenize(state: DataProcessingState) -> dict:
    """节点2：分词"""
    print("\n[节点2] 分词...")

    cleaned = state["cleaned_data"]
    # 简单分词：按空格分割
    tokens = cleaned.split()

    # 记录处理步骤
    steps = state.get("processing_steps", [])
    steps.append(f"分词完成，共 {len(tokens)} 个词")

    return {
        "tokens": tokens,
        "word_count": len(tokens),
        "processing_steps": steps
    }


def summarize(state: DataProcessingState) -> dict:
    """节点3：生成摘要"""
    print("\n[节点3] 生成摘要...")

    tokens = state["tokens"]
    word_count = state["word_count"]

    # 简单摘要
    summary = f"文本包含 {word_count} 个词，前3个词是：{', '.join(tokens[:3])}"

    # 记录处理步骤
    steps = state.get("processing_steps", [])
    steps.append("摘要生成完成")

    return {
        "summary": summary,
        "processing_steps": steps
    }


# ===== 3. 构建图 =====

workflow2 = StateGraph(DataProcessingState)

workflow2.add_node("clean", clean_data)
workflow2.add_node("tokenize", tokenize)
workflow2.add_node("summarize", summarize)

workflow2.add_edge(START, "clean")
workflow2.add_edge("clean", "tokenize")
workflow2.add_edge("tokenize", "summarize")
workflow2.add_edge("summarize", END)

graph2 = workflow2.compile()


# ===== 4. 运行 =====

def run_data_processing(text: str):
    """运行数据处理"""
    print("=" * 60)
    print("数据处理流程启动")
    print("=" * 60)

    initial_state = {
        "raw_data": text,
        "cleaned_data": "",
        "tokens": [],
        "word_count": 0,
        "summary": "",
        "processing_steps": []
    }

    final_state = graph2.invoke(initial_state)

    print("\n" + "=" * 60)
    print("处理结果")
    print("=" * 60)
    print(f"原始数据：{final_state['raw_data']}")
    print(f"清洗后：{final_state['cleaned_data']}")
    print(f"词数：{final_state['word_count']}")
    print(f"摘要：{final_state['summary']}")
    print("\n处理步骤：")
    for i, step in enumerate(final_state['processing_steps'], 1):
        print(f"  {i}. {step}")
    print("=" * 60)

    return final_state


if __name__ == "__main__":
    run_data_processing("这是   一个   测试   文本   包含   多余   空格")
```

## 核心要点总结

### 1. 状态读取

**方式**：
```python
def my_node(state: MyState) -> dict:
    # 直接访问字段
    value = state["field_name"]
```

**特点**：
- 节点可以读取任何已定义的状态字段
- 读取操作不会修改状态
- 可以读取前序节点写入的数据

### 2. 状态写入

**方式**：
```python
def my_node(state: MyState) -> dict:
    # 返回字典
    return {"field_name": new_value}
```

**特点**：
- 只返回需要更新的字段
- 未返回的字段保持不变
- 返回值自动合并到状态中

### 3. 状态合并

**默认规则**：
- 无 reducer：直接覆盖
- 有 reducer：按 reducer 规则合并

**示例**：
```python
# 无 reducer（覆盖）
class State(TypedDict):
    count: int  # 每次覆盖

# 有 reducer（累加）
from typing import Annotated
from operator import add

class State(TypedDict):
    count: Annotated[int, add]  # 每次累加
```

### 4. 状态流转

**流程**：
```
初始状态 → 节点1 → 中间状态1 → 节点2 → 中间状态2 → ... → 最终状态
```

**特点**：
- 状态在节点间自动传递
- 每个节点都能看到前序节点的更新
- 状态是累积的，不会丢失

## 常见问题

### Q1：节点必须返回所有状态字段吗？

**A**：不需要。节点只返回需要更新的字段，未返回的字段保持不变。

```python
# ✅ 正确：只返回需要更新的字段
def my_node(state: MyState) -> dict:
    return {"field1": value1}

# ❌ 不推荐：返回所有字段（冗余）
def my_node(state: MyState) -> dict:
    return {
        "field1": value1,
        "field2": state["field2"],  # 不需要
        "field3": state["field3"]   # 不需要
    }
```

### Q2：如果节点返回 None 会怎样？

**A**：如果节点返回 None 或空字典，状态不会更新。

```python
def my_node(state: MyState) -> dict:
    if some_condition:
        return {"field": value}
    else:
        return {}  # 状态不变
```

### Q3：可以在节点中直接修改 state 吗？

**A**：不推荐。应该返回新的字典，而不是直接修改 state。

```python
# ❌ 不推荐：直接修改 state
def my_node(state: MyState) -> dict:
    state["field"] = value  # 不推荐
    return state

# ✅ 推荐：返回新字典
def my_node(state: MyState) -> dict:
    return {"field": value}
```

### Q4：如何在节点中访问前序节点的输出？

**A**：直接从 state 中读取即可。

```python
def node2(state: MyState) -> dict:
    # 读取 node1 的输出
    node1_output = state["field_from_node1"]

    # 使用 node1 的输出
    result = process(node1_output)

    return {"field_from_node2": result}
```

## 最佳实践

### 1. 状态设计

**原则**：
- 字段命名清晰，表达意图
- 区分输入、中间、输出字段
- 避免冗余字段

**示例**：
```python
class MyState(TypedDict):
    # 输入字段
    user_input: str

    # 中间字段
    processed_data: str
    intermediate_result: dict

    # 输出字段
    final_output: str
    metadata: dict
```

### 2. 节点设计

**原则**：
- 单一职责：每个节点只做一件事
- 明确输入输出：清楚读取和写入哪些字段
- 保持纯粹：避免副作用

**示例**：
```python
def process_node(state: MyState) -> dict:
    """
    处理节点

    读取：state["input_field"]
    写入：{"output_field": ...}
    """
    # 读取
    input_data = state["input_field"]

    # 处理
    output_data = process(input_data)

    # 写入
    return {"output_field": output_data}
```

### 3. 错误处理

**原则**：
- 在节点中捕获异常
- 将错误信息写入状态
- 使用条件边处理错误

**示例**：
```python
def risky_node(state: MyState) -> dict:
    try:
        result = risky_operation(state["input"])
        return {"result": result, "error": None}
    except Exception as e:
        return {"result": None, "error": str(e)}
```
