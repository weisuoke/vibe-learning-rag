# 30字核心

**传统Memory已弃用，现代方案使用RunnableWithMessageHistory+Session管理或LangGraph Checkpointer，支持持久化存储、异步操作、多用户隔离，LangMem提供长期记忆。**

---

## 核心要点拆解

### 1. 传统Memory已弃用

**关键信息**：
- 所有传统 Memory 类（ConversationBufferMemory、ConversationSummaryMemory 等）已被标记为 `@deprecated`
- 自 0.3.1 版本起弃用，将在 1.0.0 版本移除
- 不支持原生工具调用，会静默失败

**为什么弃用**：
- 架构设计在聊天模型有原生工具调用能力之前
- 无法与现代 LLM 的工具调用功能兼容
- 缺乏 LCEL 管道集成能力

### 2. 现代方案：RunnableWithMessageHistory + Session管理

**核心组件**：
- `InMemoryChatMessageHistory`：内存存储对话历史
- `RunnableWithMessageHistory`：包装 Runnable 自动管理历史
- Session 管理：通过 session ID 隔离不同对话

**核心优势**：
- 实现 Runnable 接口，可在 LCEL 管道中使用
- 支持 LangSmith 追踪和监控
- 统一的参数配置方式
- 支持标准的 invoke、ainvoke、batch 等方法

### 3. LangGraph Checkpointer

**适用场景**：
- 复杂的 Agent 系统
- 需要管理完整图状态
- 多节点工作流

**核心机制**：
- 在每个节点执行后保存图的状态
- 支持跨会话恢复对话
- Thread ID 隔离不同对话
- 支持时间旅行（回溯到历史状态）

### 4. 持久化存储

**支持的存储方案**：
- **Redis**：高性能、支持 TTL、适合生产环境
- **PostgreSQL**：持久化、支持复杂查询、适合长期存储
- **SQLite**：轻量级、无需额外服务、适合开发和小型应用
- **CockroachDB**：分布式、强一致性

### 5. 异步操作

**核心能力**：
- 支持 `ainvoke`、`aadd_message`、`aget_messages` 等异步方法
- 异步连接池管理（asyncpg、aioredis）
- 提高并发性能

### 6. 多用户隔离

**实现方式**：
- Thread ID 隔离：每个用户有独立的 thread ID
- Session 管理：通过 session ID 区分不同会话
- 并发控制：支持多用户并发访问

### 7. LangMem 长期记忆

**2025年新特性**：
- 2025年2月发布的 LangMem SDK
- 提供长期记忆管理
- 语义知识提取
- 跨会话记忆维护
- 与 LangGraph 集成

## 核心价值

这30字核心总结了 LangChain 对话历史管理的完整演进路径：

1. **明确弃用警告**：避免使用已弃用的传统 Memory 类
2. **现代方案指引**：提供两种主流方案（RunnableWithMessageHistory 和 LangGraph Checkpointer）
3. **生产环境能力**：强调持久化存储、异步操作、多用户隔离等生产级特性
4. **前沿技术**：引入 LangMem SDK 长期记忆管理

## 快速决策树

```
需要对话历史管理？
├─ 简单对话系统
│  └─ 使用 RunnableWithMessageHistory + InMemoryChatMessageHistory
│     ├─ 开发环境：内存存储
│     └─ 生产环境：Redis/PostgreSQL 持久化存储
│
└─ 复杂 Agent 系统
   └─ 使用 LangGraph Checkpointer
      ├─ 短期记忆：Checkpointer（InMemorySaver/RedisSaver/PostgresSaver）
      └─ 长期记忆：LangMem SDK
```

## 迁移路径

**从传统 Memory 迁移**：

```python
# 旧方式（已弃用）
from langchain.memory import ConversationBufferMemory
memory = ConversationBufferMemory()

# 新方式（推荐）
from langchain_core.chat_history import InMemoryChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory

store = {}
def get_session_history(session_id: str):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()
    return store[session_id]

conversation = RunnableWithMessageHistory(
    chat_model,
    get_session_history,
)
```
