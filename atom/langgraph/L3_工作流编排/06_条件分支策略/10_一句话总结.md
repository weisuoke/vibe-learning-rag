# 条件分支策略 - 一句话总结

> 条件分支策略是 LangGraph 通过 add_conditional_edges（静态路由）、Send（动态并行）、Command（高级控制）三种机制，让工作流在运行时根据状态智能选择执行路径的核心能力，是构建 Agent 决策逻辑、错误回退和多路并行的基础。

---

## 核心公式

```
条件分支 = 路由函数(状态) → 下一个节点
```

**三种实现方式：**

| 机制 | 本质 | 适用场景 |
|------|------|----------|
| `add_conditional_edges` | 运行时 if-else | 路径已知，单选分支 |
| `Send` | 动态 Map-Reduce | 并行数量运行时决定 |
| `Command` | 路由 + 状态更新 | 决策和更新紧耦合 |

---

## 关键要点回顾

- **条件边 = 运行时 if-else** — 路由函数接收状态，返回节点名
- **Send = 动态 Map-Reduce** — 向多个节点发送独立状态，并行执行
- **Command = 路由 + 状态更新** — 节点内部同时控制去向和状态
- **Literal 类型推断** — 自动推断所有可能路径，简化配置和可视化
- **path_map** — 解耦路由返回值和节点名，方便重构
- **循环守卫 = 安全阀** — 计数器 + recursion_limit + 超时三层防护
- **回退机制 = 韧性保障** — 重试 → 降级 → 人工三级回退
- **Supervisor 模式** — 中央调度 + 专家路由，多 Agent 协作的基础

---

## 使用频率参考

```
add_conditional_edges  ████████████████████  80%
Send                   ███                   15%
Command                █                      5%
```

大多数场景用条件边就够了。Send 用于动态并行，Command 用于路由和状态更新紧耦合的场景。

```
循环加守卫，回退要三级。
```

```
用户提问
  ↓ [条件边] 意图分类
  ├── 知识问答 → [Send] 并行检索 → [条件边] 质量检查 → 生成回答
  ├── 闲聊 → 直接回答
  └── 复杂任务 → [Supervisor] 多 Agent 协作
```
