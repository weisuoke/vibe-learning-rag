# 最小可用：Agent Memory 集成

> 5 个最小知识点，从零开始让 Agent 拥有记忆

---

## 最小知识 1：3 行代码让 Agent 记住对话

### 你只需要知道

**加一个 `checkpointer` + 一个 `thread_id`，Agent 就能记住之前的对话。**

### 最小代码

```python
from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver

# 1. 创建 checkpointer（记忆存储器）
checkpointer = InMemorySaver()

# 2. 创建带记忆的 Agent
agent = create_agent(
    model="gpt-4.1-mini",
    tools=[],
    checkpointer=checkpointer,  # ← 加这一行就有记忆
)

# 3. 用 thread_id 标识对话
config = {"configurable": {"thread_id": "my-chat-001"}}

# 第一次对话
response1 = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫张三，我是程序员"}]},
    config,
)
print(response1["messages"][-1].content)
# → "你好张三！很高兴认识你..."

# 第二次对话（同一 thread_id → 记得上下文）
response2 = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫什么？我做什么工作？"}]},
    config,
)
print(response2["messages"][-1].content)
# → "你叫张三，你是一名程序员"
```

### 关键点

```
记忆 = checkpointer + thread_id

checkpointer ——→ 决定记忆存在哪里（内存/数据库）
thread_id    ——→ 决定哪些对话属于同一会话

没有 checkpointer → Agent 每次对话都是全新的
没有 thread_id    → 所有用户共享同一段记忆（混乱）
```

### 常见错误

```python
# ❌ 错误：没有 checkpointer
agent = create_agent(model="gpt-4.1-mini", tools=[])
# Agent 不会记住任何对话

# ❌ 错误：没有 thread_id
agent.invoke({"messages": [...]})  # 缺少 config
# 报错或使用默认 thread（所有人共享）

# ✅ 正确：两者都有
agent = create_agent(model="gpt-4.1-mini", tools=[], checkpointer=InMemorySaver())
agent.invoke({"messages": [...]}, {"configurable": {"thread_id": "user-001"}})
```

## 最小知识 2：控制记忆长度（防止超限）

### 你只需要知道

**对话太长会超出 Token 限制，用 `SummarizationMiddleware` 自动压缩旧对话。**

### 最小代码

```python
from langchain.agents import create_agent
from langchain.agents.middleware import SummarizationMiddleware
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="gpt-4.1",
    tools=[],
    middleware=[
        SummarizationMiddleware(
            model="gpt-4.1-mini",        # 用便宜模型做摘要
            trigger=("tokens", 4000),     # 超过 4000 tokens 触发
            keep=("messages", 20),        # 保留最近 20 条消息
        )
    ],
    checkpointer=InMemorySaver(),
)

# 正常使用即可，超长时自动压缩
config = {"configurable": {"thread_id": "long-chat"}}

# 即使聊了 100 轮，也不会超出 Token 限制
for i in range(100):
    agent.invoke(
        {"messages": [{"role": "user", "content": f"这是第 {i+1} 轮对话"}]},
        config,
    )
# SummarizationMiddleware 自动处理：
# 1. 检测 Token 总量
# 2. 超过 4000 tokens 时，用 gpt-4.1-mini 摘要旧消息
# 3. 替换为 "对话摘要：..." + 最近 20 条消息
```

### 工作原理

```
对话消息越来越多：

  [msg1, msg2, msg3, ..., msg50]  ← 超过 4000 tokens 了！

SummarizationMiddleware 自动触发：

  [摘要: "用户之前讨论了X和Y", msg31, msg32, ..., msg50]
                                     ↑ 保留最近 20 条

Token 数量又回到合理范围内 ✓
```

### 为什么不直接删除旧消息？

```
直接删除：Agent 完全忘记之前的内容
  "我们之前讨论了什么？" → "我们没有之前的对话"  ← 不好

摘要替代：Agent 保留了概要
  "我们之前讨论了什么？" → "我们讨论了项目架构和技术选型"  ← 好
```

## 最小知识 3：多用户隔离（每人独立记忆）

### 你只需要知道

**不同的 `thread_id` = 不同的记忆空间。用 `user_id + session_id` 组合作为 `thread_id`。**

### 最小代码

```python
from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="gpt-4.1-mini",
    tools=[],
    checkpointer=InMemorySaver(),
)

# 用户 A 的对话
config_a = {"configurable": {"thread_id": "user-alice-session-1"}}
agent.invoke(
    {"messages": [{"role": "user", "content": "我喜欢 Python"}]},
    config_a,
)

# 用户 B 的对话（完全独立）
config_b = {"configurable": {"thread_id": "user-bob-session-1"}}
agent.invoke(
    {"messages": [{"role": "user", "content": "我喜欢 Java"}]},
    config_b,
)

# 用户 A 继续（记得 Python）
agent.invoke(
    {"messages": [{"role": "user", "content": "我喜欢什么语言？"}]},
    config_a,
)
# → "你喜欢 Python"  ← 不会混淆用户 B 的信息

# 用户 B 继续（记得 Java）
agent.invoke(
    {"messages": [{"role": "user", "content": "我喜欢什么语言？"}]},
    config_b,
)
# → "你喜欢 Java"  ← 不会混淆用户 A 的信息
```

### thread_id 设计模式

```python
# 模式 1：每个用户一个长期对话
thread_id = f"user-{user_id}"
# 适合：个人助手，一直聊

# 模式 2：每个会话独立
thread_id = f"user-{user_id}-session-{session_id}"
# 适合：客服系统，每次工单独立

# 模式 3：按场景分离
thread_id = f"user-{user_id}-{scenario}"
# 例：user-alice-coding, user-alice-writing
# 适合：多功能助手
```

### 图解

```
同一个 Agent，不同 thread_id → 完全隔离的记忆

┌─────────────────────────────────────┐
│          Agent（共用）               │
│                                     │
│  thread: user-alice-1               │
│  ┌─────────────────────┐            │
│  │ Alice: 我喜欢Python  │            │
│  │ AI: 好的...         │            │
│  └─────────────────────┘            │
│                                     │
│  thread: user-bob-1                 │
│  ┌─────────────────────┐            │
│  │ Bob: 我喜欢Java     │            │
│  │ AI: 好的...         │            │
│  └─────────────────────┘            │
│                                     │
│  thread: user-alice-2  ← 新会话     │
│  ┌─────────────────────┐            │
│  │ (空)                │            │
│  └─────────────────────┘            │
└─────────────────────────────────────┘
```

## 最小知识 4：生产环境持久化（重启不丢记忆）

### 你只需要知道

**开发用 `InMemorySaver`（重启丢失），生产用 `PostgresSaver`（永久保存）。只需换一行代码。**

### 最小代码

```python
import os

# ============ 开发环境 ============
from langgraph.checkpoint.memory import InMemorySaver
checkpointer_dev = InMemorySaver()
# 数据在内存中，程序重启就没了

# ============ 生产环境 ============
from langgraph.checkpoint.postgres import PostgresSaver
checkpointer_prod = PostgresSaver(
    conn_string=os.environ["DATABASE_URL"]
    # 例："postgresql://user:pass@localhost:5432/agent_db"
)
# 数据在数据库中，永久保存

# ============ 切换只需一行 ============
from langchain.agents import create_agent

# 开发时：
agent = create_agent(
    model="gpt-4.1-mini",
    tools=[],
    checkpointer=checkpointer_dev,   # ← 开发
)

# 上线时：
agent = create_agent(
    model="gpt-4.1-mini",
    tools=[],
    checkpointer=checkpointer_prod,  # ← 只改这一行
)
```

### 环境自动选择

```python
import os

def get_checkpointer():
    """根据环境变量自动选择持久化后端"""
    env = os.environ.get("ENV", "development")

    if env == "development":
        from langgraph.checkpoint.memory import InMemorySaver
        return InMemorySaver()

    elif env == "production":
        from langgraph.checkpoint.postgres import PostgresSaver
        return PostgresSaver(
            conn_string=os.environ["DATABASE_URL"]
        )

    else:
        raise ValueError(f"Unknown env: {env}")

# 使用
checkpointer = get_checkpointer()
agent = create_agent(
    model="gpt-4.1-mini",
    tools=[],
    checkpointer=checkpointer,
)
```

### 四种后端速查

```
InMemorySaver  → 开发/测试（零配置，重启丢失）
SqliteSaver    → 个人项目（文件存储，单机）
PostgresSaver  → 正式产品（推荐首选，高可用）
RedisSaver     → 高性能场景（亚毫秒延迟，需配置持久化）
```

## 最小知识 5：跨会话长期记忆（Store）

### 你只需要知道

**`checkpointer` 管线程内的对话历史（短期记忆），`store` 管跨线程的用户信息（长期记忆）。**

### 最小代码

```python
from langchain.agents import create_agent
from langchain.embeddings import init_embeddings
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.store.memory import InMemoryStore

# 1. 创建带语义搜索的 Store（长期记忆）
embeddings = init_embeddings("openai:text-embedding-3-small")
store = InMemoryStore(
    index={"embed": embeddings, "dims": 1536}
)

# 2. 存入用户信息
store.put(
    ("user-alice", "preferences"), "1",
    {"text": "喜欢 Python，偏好简洁代码风格"}
)
store.put(
    ("user-alice", "preferences"), "2",
    {"text": "正在学习 RAG 开发"}
)

# 3. 创建带双层记忆的 Agent
agent = create_agent(
    model="gpt-4.1",
    tools=[],
    checkpointer=InMemorySaver(),   # 短期记忆（对话历史）
    store=store,                     # 长期记忆（用户信息）
)

# 4. 新会话也能获取用户信息
config_new = {"configurable": {"thread_id": "new-session-999"}}

# Agent 可以从 Store 检索用户偏好
# （具体检索逻辑通过 middleware 或 system prompt 注入）
```

### Checkpointer vs Store 对比

```
Checkpointer（短期记忆）：
  ├── 同一 thread_id 内自动生效
  ├── 存对话消息列表
  ├── 新 thread = 新记忆（不共享）
  └── 类似：今天跟某人的聊天记录

Store（长期记忆）：
  ├── 跨 thread_id 共享
  ├── 存键值对（用户偏好/事实）
  ├── 需要显式读写
  └── 类似：通讯录里这个人的备注
```

### 双层记忆架构图

```
用户 Alice 的记忆体系：

┌────────────────────────────────────────────┐
│  Store（长期记忆）                           │
│  ┌──────────────────────────────────┐      │
│  │ ("user-alice", "preferences")    │      │
│  │   - 喜欢 Python                  │      │
│  │   - 正在学 RAG                   │      │
│  │   - 偏好简洁风格                  │      │
│  └──────────────────────────────────┘      │
│  所有会话共享 ↓                             │
├────────────────────────────────────────────┤
│  Checkpointer（短期记忆）                    │
│                                            │
│  thread-1: [msg1, msg2, msg3...]           │
│  thread-2: [msg1, msg2...]                 │
│  thread-3: [msg1...]                       │
│  每个会话独立 ✓                             │
└────────────────────────────────────────────┘
```

## 与 RAG 开发的关系

| 最小知识 | RAG 应用场景 |
|----------|-------------|
| **知识1** checkpointer | RAG Agent 记住多轮检索上下文 |
