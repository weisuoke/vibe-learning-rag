## 场景描述

### 业务背景

在 RAG 系统中，向量检索只解决了"语义相似"的问题，但实际业务往往需要更精准的过滤：
- 用户问"Python 相关的技术文档"，需要按**语言分类**过滤
- 用户问"最近的正面评价"，需要按**情感倾向**过滤
- 用户问"入门级教程"，需要按**难度等级**过滤

这些结构化元数据（分类、情感、主题、难度）如果靠人工标注，几千篇文档就已经不现实了。

### 技术挑战

1. **人工标注不可扩展**：文档量大时，人工标注成本极高
2. **元数据缺失**：原始文档通常只有文件名和路径，缺少语义级元数据
3. **标注一致性**：不同人标注标准不统一，影响检索质量
4. **成本与精度平衡**：LLM 标注精准但有 API 成本，规则标注免费但能力有限

### 学习目标

- 自定义 `BaseDocumentTransformer` 实现规则型元数据标注
- 使用 `OpenAIMetadataTagger` 实现 LLM 智能标注
- 组合规则标注 + LLM 标注的混合管道
- 在 RAG 检索中利用元数据进行过滤搜索

### Step 2：自定义规则型元数据标注器

规则型标注器零 API 成本，适合提取确定性元数据：

```python
# ============================================================
# Step 2：自定义规则型元数据标注器
# ============================================================
class BasicMetadataTransformer(BaseDocumentTransformer):
    """基于规则的元数据标注器：字数统计、语言检测、长度分类、来源推断"""

    def transform_documents(
        self, documents: Sequence[Document], **kwargs: Any
    ) -> Sequence[Document]:
        """接收文档列表，返回添加了元数据的新文档列表（不修改原始文档）"""
        result = []
        for doc in documents:
            # 复制原有元数据，避免修改原始数据
            new_metadata = {**doc.metadata}
            content = doc.page_content

            # --- 规则1：字数统计 ---
            new_metadata["char_count"] = len(content)
            new_metadata["word_count"] = len(content.split())

            # --- 规则2：语言检测（简单规则） ---
            chinese_chars = sum(1 for c in content if '\u4e00' <= c <= '\u9fff')
            new_metadata["has_chinese"] = chinese_chars > 0
            new_metadata["chinese_ratio"] = round(chinese_chars / max(len(content), 1), 2)

            # --- 规则3：内容长度分类 ---
            if len(content) < 200:
                new_metadata["length_category"] = "short"
            elif len(content) < 500:
                new_metadata["length_category"] = "medium"
            else:
                new_metadata["length_category"] = "long"

            # --- 规则4：来源类型推断 ---
            source = doc.metadata.get("source", "")
            if "blog" in source:
                new_metadata["source_type"] = "blog"
            elif "report" in source:
                new_metadata["source_type"] = "report"
            elif "docs" in source or "tutorial" in source:
                new_metadata["source_type"] = "documentation"
            else:
                new_metadata["source_type"] = "unknown"

            result.append(Document(page_content=content, metadata=new_metadata))

        return result

    # BaseDocumentTransformer 要求实现异步版本
    async def atransform_documents(
        self, documents: Sequence[Document], **kwargs: Any
    ) -> Sequence[Document]:
        return self.transform_documents(documents, **kwargs)


# --- 测试规则型标注器 ---
basic_transformer = BasicMetadataTransformer()
tagged_docs = basic_transformer.transform_documents(documents)

print("=" * 60)
print("规则型标注结果")
print("=" * 60)

    print(f"\n来源: {doc.metadata['source']}")

```

**要点解读：**

- `transform_documents` 是 `BaseDocumentTransformer` 的核心方法
- 规则型标注：**零成本、确定性、毫秒级**，但无法理解语义

### Step 3：OpenAI 智能元数据标注器

LLM 标注器能理解语义，自动提取分类、情感、主题等高级元数据：

```python
# ============================================================
# Step 3：OpenAI 智能元数据标注器
# ============================================================
from langchain_community.document_transformers.openai_functions import (
    OpenAIMetadataTagger,
)
from langchain_openai import ChatOpenAI

# --- 定义元数据 Schema（JSON Schema 格式）---
metadata_schema = {
    "properties": {
        "category": {
            "type": "string",
            "enum": ["技术", "产品", "运营", "教程", "其他"],
            "description": "文档的主题分类",
        },
        "sentiment": {
            "type": "string",
            "enum": ["positive", "negative", "neutral"],
            "description": "文档的整体情感倾向",
        },
        "difficulty": {
            "type": "string",
            "enum": ["beginner", "intermediate", "advanced"],
            "description": "文档内容的难度等级",
        },
        "key_topics": {
            "type": "array",
            "items": {"type": "string"},
            "description": "文档的关键主题词，最多3个",
        },
    },
    "required": ["category", "sentiment", "difficulty", "key_topics"],
}

# --- 创建 LLM 标注器 ---
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
openai_tagger = OpenAIMetadataTagger(schema=metadata_schema, llm=llm)

# --- 执行标注（每篇文档一次 API 请求）---
smart_tagged_docs = openai_tagger.transform_documents(documents)

print("=" * 60)
print("LLM 智能标注结果")
print("=" * 60)
for doc in smart_tagged_docs:
    print(f"\n来源: {doc.metadata['source']}")
    print(f"  分类: {doc.metadata.get('category', 'N/A')}")
    print(f"  情感: {doc.metadata.get('sentiment', 'N/A')}")
    print(f"  难度: {doc.metadata.get('difficulty', 'N/A')}")
    print(f"  主题: {doc.metadata.get('key_topics', [])}")
```

**要点解读：**

- `OpenAIMetadataTagger` 底层使用 Function Calling 提取结构化数据
- Schema 中的 `enum` 约束输出值一致性，`temperature=0` 保证标注可复现

### Step 4：组合管道——规则 + LLM 混合标注

实际项目中，最佳实践是**先用规则标注免费元数据，再用 LLM 标注语义元数据**：

```python
# ============================================================
# Step 4：组合管道——规则 + LLM 混合标注
# ============================================================

class MetadataTaggingPipeline:
    """
    混合元数据标注管道。
    先执行规则标注（免费、快速），再执行 LLM 标注（智能、有成本）。
    类比：工厂流水线——第一道工序自动称重量尺寸，第二道工序质检员判断品质。
    """

    def __init__(self, transformers: list[BaseDocumentTransformer]):
        self.transformers = transformers

    def run(self, documents: Sequence[Document]) -> list[Document]:
        """依次执行所有标注器"""
        current_docs = list(documents)
        for i, transformer in enumerate(self.transformers):
            transformer_name = transformer.__class__.__name__
            print(f"  [{i+1}/{len(self.transformers)}] 执行: {transformer_name}")
            current_docs = transformer.transform_documents(current_docs)
            print(f"    完成，处理了 {len(current_docs)} 篇文档")
        return current_docs


# --- 构建管道 ---
pipeline = MetadataTaggingPipeline(
    transformers=[
        BasicMetadataTransformer(),  # 第一步：规则标注（免费）
        openai_tagger,                # 第二步：LLM 标注（智能）
    ]
)

# --- 执行管道 ---
print("=" * 60)
print("执行混合标注管道")
print("=" * 60)
final_docs = pipeline.run(documents)

# --- 查看完整元数据 ---
print("\n" + "=" * 60)
print("混合标注最终结果")
print("=" * 60)
for doc in final_docs:
    print(f"\n来源: {doc.metadata['source']}")
    # 规则标注的元数据
    print(f"  [规则] 字符数: {doc.metadata.get('char_count')}")
    print(f"  [规则] 长度分类: {doc.metadata.get('length_category')}")
    print(f"  [规则] 来源类型: {doc.metadata.get('source_type')}")
    # LLM 标注的元数据
    print(f"  [LLM]  分类: {doc.metadata.get('category')}")
    print(f"  [LLM]  情感: {doc.metadata.get('sentiment')}")
    print(f"  [LLM]  难度: {doc.metadata.get('difficulty')}")
    print(f"  [LLM]  主题: {doc.metadata.get('key_topics')}")
```

**管道设计的好处：**

| 标注类型 | 速度 | 成本 | 能力 | 适用场景 |
|---------|------|------|------|---------|
| 规则标注 | 毫秒级 | 免费 | 确定性特征 | 字数、语言、来源类型 |
| LLM 标注 | 秒级 | API 费用 | 语义理解 | 分类、情感、主题 |
| 混合管道 | 秒级 | 部分 API 费用 | 两者兼得 | 生产环境推荐 |

### Step 5：在 RAG 检索中使用元数据过滤

标注完元数据后，最关键的一步——在向量检索时利用元数据做过滤：

```python
# ============================================================
# Step 5：在 RAG 检索中使用元数据过滤
# ============================================================

from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")
vectorstore = Chroma.from_documents(
    documents=final_docs,
    embedding=embeddings,
    collection_name="tagged_docs",
)

# --- 场景A：只搜索技术类文档 ---
results_tech = vectorstore.similarity_search(
    query="Python 有什么新特性？",
    k=3,
    filter={"category": "技术"},  # 元数据过滤
)
for doc in results_tech:
    print(f"  [{doc.metadata.get('category')}] {doc.page_content[:50]}...")

# --- 场景B：只搜索正面情感的文档 ---
results_positive = vectorstore.similarity_search(
    query="业务进展如何？",
    k=3,
    filter={"sentiment": "positive"},
)
for doc in results_positive:
    print(f"  [{doc.metadata.get('sentiment')}] {doc.page_content[:50]}...")

# --- 场景C：组合过滤——入门级教程类文档 ---
results_beginner = vectorstore.similarity_search(
    query="RAG 怎么入门？",
    k=3,
    filter={
        "$and": [
            {"difficulty": "beginner"},
            {"source_type": "documentation"},
        ]
    },
)
for doc in results_beginner:
    print(f"  [{doc.metadata.get('difficulty')}|{doc.metadata.get('source_type')}]", end=" ")
    print(f"{doc.page_content[:50]}...")

vectorstore.delete_collection()
```

**元数据过滤的威力：** 没有元数据时，搜索"Python 新特性"可能返回所有提到 Python 的文档。有了 `category: 技术` 的过滤，检索结果精准锁定技术文档。这就是自动元数据标注的核心价值——**把模糊的语义搜索变成精准的结构化 + 语义混合搜索**。

## 关键要点

### 1. 两种标注策略的选择

```
规则型标注 → 确定性特征（字数、语言、来源）→ 免费、快速、100% 一致
LLM 标注  → 语义特征（分类、情感、主题）  → 智能、有成本、需要校验
混合管道  → 先规则后 LLM                  → 生产环境最佳实践
```

### 2. BaseDocumentTransformer 是统一接口

所有标注器都实现同一个接口，这意味着：
- 可以自由组合、替换标注器
- 管道中的每一步都是可插拔的
- 自定义标注器和官方标注器无缝协作

### 3. 元数据是 RAG 检索精度的倍增器

```
纯向量检索：query → embedding → 相似度排序 → 结果（可能包含无关文档）
元数据过滤：query → embedding → 元数据过滤 → 相似度排序 → 精准结果
```

元数据过滤在向量检索之前（或同时）执行，大幅缩小搜索范围，提升检索精度。

### 4. 成本控制是生产环境的必修课

- 对已标注的文档跳过 LLM 调用
- 使用 `gpt-4o-mini` 而非 `gpt-4o` 降低单次成本
- `temperature=0` 保证标注一致性，避免重复标注结果不同

## 常见问题

### Q1：OpenAIMetadataTagger 支持哪些 LLM？

只要是支持 Function Calling 的 OpenAI 兼容模型都可以。通过 `ChatOpenAI` 传入即可。如果使用自定义 API 端点（如 Azure OpenAI），设置 `OPENAI_BASE_URL` 环境变量。

### Q2：Schema 中的 enum 是必须的吗？

不是必须的，但**强烈推荐**。没有 enum 约束时，LLM 可能输出"技术文档"、"tech"、"技术类"等不同表述，导致后续过滤失败。enum 确保输出值的一致性。

### Q3：文档量很大时怎么优化？

三个策略：增量标注（只标注新增文档）、批量处理（分批发送控制并发）、缓存结果（持久化到数据库避免重复计算）。

### Q4：规则型标注器能替代 LLM 标注吗？

不能完全替代。规则型擅长**确定性特征**（字数、格式），LLM 擅长**语义特征**（分类、情感）。两者互补。

### Q5：标注错误怎么办？

LLM 标注不是 100% 准确。建议抽样人工校验、设置 Schema 的 enum 约束减少错误、对结果做后处理校验。

## 与 RAG 系统的关系

```
文档加载 → 文本分块 → 【自动元数据标注】→ 向量存储 → 元数据过滤检索 → LLM 生成
                            ↑
                      本场景的核心位置
```

自动元数据标注处于 RAG 管道的**索引阶段**，在文档存入向量库之前执行。标注的质量直接决定了后续检索的过滤能力。好的元数据标注能让 RAG 系统从"模糊搜索"升级为"精准搜索"。
