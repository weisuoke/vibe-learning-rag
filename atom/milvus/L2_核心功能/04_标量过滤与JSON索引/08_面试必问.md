## 基础概念问题

### Q1: 什么是标量过滤？与向量检索有什么区别？

**答案**：

标量过滤是对结构化字段（如年龄、价格、类别）进行精确条件筛选，向量检索是对向量进行语义相似度搜索。

**区别**：
```
标量过滤：
- 精确匹配：age == 25
- 范围查询：price >= 100 AND price <= 500
- 集合操作：city IN ['北京', '上海']
- 时间复杂度：O(log n)（有索引）

向量检索：
- 语义相似度：找到最相似的向量
- 近似匹配：Top-K 最相似结果
- 距离度量：COSINE、L2、IP
- 时间复杂度：O(log n)（HNSW）
```

**混合查询**：
```python
# 先标量过滤，再向量检索
results = client.search(
    collection_name="products",
    data=[query_vector],
    filter="price >= 100 AND price <= 500 AND stock > 0",  # 标量过滤
    limit=10  # 向量检索 Top-10
)
```

### Q2: Milvus 支持哪些标量索引类型？各有什么特点？

**答案**：

Milvus 支持 6 种标量索引类型：

| 索引类型 | 适用场景 | 时间复杂度 | 内存占用 |
|---------|---------|-----------|---------|
| **INVERTED** | 通用场景 | O(log n) | 中等 |
| **BITMAP** | 低基数字段 | O(1) | 低 |
| **STL_SORT** | 范围查询 | O(log n) | 中等 |
| **AUTOINDEX** | 不确定时 | 自动选择 | 自动 |
| **TRIE** | 前缀匹配 | O(m) | 中等 |
| **NGRAM** | 模糊匹配 | O(n) | 高 |

**选择策略**：
```python
# 低基数字段（< 100个值）→ BITMAP
index_params.add_index("status", "BITMAP")  # 只有3个状态值

# 中高基数字段 → INVERTED
index_params.add_index("age", "INVERTED")  # 年龄范围广

# 前缀匹配 → TRIE
index_params.add_index("name", "TRIE")  # name LIKE 'John%'

# 不确定 → AUTOINDEX
index_params.add_index("field", "AUTOINDEX")  # 让 Milvus 自动选择
```

### Q3: 什么是 JSON Path Index？性能提升有多大？

**答案**：

JSON Path Index 是 Milvus 2.6 的核心特性，为 JSON 字段的特定路径创建索引，实现 100 倍性能提升。

**原理**：
```
无索引：
1. 扫描所有文档
2. 解析每个文档的 JSON 字段
3. 提取嵌套路径值
4. 应用过滤条件
时间复杂度：O(n)

有索引：
1. 查询索引（B-Tree/倒排索引）
2. 直接定位匹配文档
时间复杂度：O(log n)

性能提升：n / log n ≈ 100x（当 n = 100,000）
```

**创建方式**：
```python
index_params.add_index(
    field_name="metadata",
    index_name="category_index",
    index_type="INVERTED",
    params={
        "json_path": "metadata['product']['category']",
        "json_cast_type": "varchar"
    }
)
```

**性能数据**：
- 无索引：10000ms
- 有索引：100ms
- 性能提升：100倍

## 技术深度问题

### Q4: 标量过滤的执行策略有哪些？如何选择？

**答案**：

Milvus 支持 3 种执行策略：Pre-filtering、Post-filtering 和 Hybrid。

**Pre-filtering（前置过滤）**：
```
执行流程：
1. 应用标量过滤 → 候选集
2. 在候选集中进行向量检索 → Top-K

适用场景：
- 高选择性（< 50%）
- 候选集小（< 10万条）

性能：
- 减少向量检索的搜索空间
- 降低计算成本
```

**Post-filtering（后置过滤）**：
```
执行流程：
1. 向量检索 → Top-K'（K' > K）
2. 应用标量过滤 → Top-K

适用场景：
- 低选择性（> 50%）
- 候选集大（> 50万条）

性能：
- 保证召回率
- 向量检索不受影响
```

**Hybrid（混合策略）**：
```
决策逻辑：
IF 过滤选择性 > 50%:
    使用 Pre-filtering
ELSE:
    使用 Post-filtering

优势：
- 自动选择最优策略
- 平衡性能和召回率
```

**性能对比**：
```
高选择性（10%）：
- Pre-filtering: 50ms
- Post-filtering: 200ms

低选择性（90%）：
- Pre-filtering: 250ms
- Post-filtering: 110ms
```

### Q5: 如何优化标量过滤的性能？

**答案**：

**1. 索引优化**：
```python
# 为常用查询字段创建索引
index_params.add_index("age", "INVERTED")
index_params.add_index("city", "BITMAP")  # 低基数

# 性能提升：30-100倍
```

**2. 查询优化**：
```python
# 选择性高的条件在前
filter = "stock > 0 AND price >= 100 AND price <= 500"
# stock > 0 选择性高（10%），先执行

# 使用 IN 替代多个 OR
filter = "city IN ['北京', '上海', '深圳']"  # 推荐
# 而不是：city == '北京' OR city == '上海' OR city == '深圳'

# 避免 NOT 操作符
filter = "age < 60"  # 推荐
# 而不是：NOT (age >= 60)
```

**3. 数据建模优化**：
```python
# 扁平化高频访问字段
# 推荐：
{
    "category": "electronics",  # 顶层字段
    "metadata": {...}
}

# 不推荐：
{
    "metadata": {
        "product": {
            "category": "electronics"  # 深层嵌套
        }
    }
}
```

**4. 批量操作**：
```python
# 批量插入
client.insert(collection_name, data_list)  # 一次插入多条

# 批量查询
query_vectors = [vector1, vector2, vector3]
results = client.search(collection_name, query_vectors, limit=10)
```

### Q6: JSON Path 语法有哪些注意事项？

**答案**：

**1. 使用方括号语法**：
```python
# ❌ 错误：使用点号语法
"metadata.product.category"  # Milvus 不支持

# ✅ 正确：使用方括号语法
"metadata['product']['category']"
```

**2. 必须指定 json_cast_type**：
```python
# ❌ 错误：不指定类型
index_params.add_index(
    field_name="metadata",
    index_type="INVERTED",
    params={
        "json_path": "metadata['price']"
        # 缺少 json_cast_type
    }
)

# ✅ 正确：指定类型
index_params.add_index(
    field_name="metadata",
    index_type="INVERTED",
    params={
        "json_path": "metadata['price']",
        "json_cast_type": "double"  # 必须指定
    }
)
```

**3. 数组操作**：
```python
# 数组包含
"ARRAY_CONTAINS(metadata['tags'], 'hot')"

# 数组长度
"ARRAY_LENGTH(metadata['tags']) >= 2"

# 数组索引访问
"metadata['tags'][0] == 'hot'"
```

**4. NULL 值处理**：
```python
# 查找 NULL 值
"metadata['brand'] IS NULL"

# 查找非 NULL 值
"metadata['brand'] IS NOT NULL"

# 空字符串不等于 NULL
"" != NULL
```

## 实战场景问题

### Q7: 如何实现一个生产级的 RAG 混合检索系统？

**答案**：

**架构设计**：
```
1. 数据层：
   - 向量字段：存储 Embedding
   - 标量字段：存储元数据（category, language, rating）
   - JSON 字段：存储灵活元数据

2. 索引层：
   - 向量索引：AUTOINDEX（COSINE）
   - 标量索引：INVERTED/BITMAP
   - JSON Path Index：为常用路径创建索引

3. 查询层：
   - 混合检索：向量检索 + 标量过滤
   - 自动策略选择：Pre-filtering/Post-filtering
   - 批量查询优化
```

**代码实现**：
```python
class ProductionRAG:
    def __init__(self):
        self.client = MilvusClient("http://localhost:19530")
        self.model = SentenceTransformer('all-MiniLM-L6-v2')
        self._setup_collection()

    def _setup_collection(self):
        """设置集合和索引"""
        schema = self.client.create_schema(auto_id=True)
        schema.add_field("id", DataType.INT64, is_primary=True)
        schema.add_field("vector", DataType.FLOAT_VECTOR, dim=384)
        schema.add_field("text", DataType.VARCHAR, max_length=2000)
        schema.add_field("metadata", DataType.JSON)

        self.client.create_collection("rag_kb", schema=schema)

        # 创建索引
        index_params = self.client.prepare_index_params()
        index_params.add_index("vector", "AUTOINDEX", "COSINE")
        index_params.add_index(
            field_name="metadata",
            index_name="category_index",
            index_type="INVERTED",
            params={
                "json_path": "metadata['category']",
                "json_cast_type": "varchar"
            }
        )
        self.client.create_index("rag_kb", index_params)

    def search(self, query, category=None, language=None, min_rating=None):
        """混合检索"""
        query_vector = self.model.encode(query).tolist()

        # 构建过滤条件
        filters = []
        if category:
            filters.append(f"metadata['category'] == '{category}'")
        if language:
            filters.append(f"metadata['language'] == '{language}'")
        if min_rating:
            filters.append(f"metadata['rating'] >= {min_rating}")

        filter_expr = " AND ".join(filters) if filters else None

        # 执行检索
        results = self.client.search(
            collection_name="rag_kb",
            data=[query_vector],
            filter=filter_expr,
            limit=10,
            output_fields=["text", "metadata"]
        )

        return results
```

**性能优化**：
```python
# 1. 批量查询
def batch_search(self, queries, category=None):
    query_vectors = [self.model.encode(q).tolist() for q in queries]
    return self.client.search("rag_kb", query_vectors, filter=..., limit=10)

# 2. 缓存热点查询
from functools import lru_cache

@lru_cache(maxsize=1000)
def cached_search(self, query, category):
    return self.search(query, category)

# 3. 异步查询
import asyncio

async def async_search(self, query, category):
    return await asyncio.to_thread(self.search, query, category)
```

## 常见误区

### Q10: 常见的标量过滤误区有哪些？

**答案**：

**误区1：索引越多越好**
```python
# ❌ 错误：为所有字段创建索引
index_params.add_index("id", "INVERTED")
index_params.add_index("name", "INVERTED")
index_params.add_index("age", "INVERTED")
# ... 100个字段

# ✅ 正确：只为常用查询字段创建索引
index_params.add_index("id", "INVERTED")  # 主键
index_params.add_index("email", "INVERTED")  # 常用查询
index_params.add_index("status", "BITMAP")  # 常用过滤
```

**误区2：JSON 查询一定很慢**
```python
# 错误认知：JSON 查询慢
# 实际情况：有索引就快

# 无索引：10000ms
# JSON Path Index：100ms
# 性能提升：100倍
```

**误区3：BITMAP 索引适用于所有场景**
```python
# ❌ 错误：高基数字段使用 BITMAP
index_params.add_index("user_id", "BITMAP")  # 100万个值

# ✅ 正确：根据基数选择索引
index_params.add_index("status", "BITMAP")  # 3个值（低基数）
index_params.add_index("user_id", "INVERTED")  # 100万个值（高基数）
```

**误区4：选择性低的条件应该放在前面**
```python
# ❌ 错误：选择性低的条件在前
"price >= 100 AND price <= 500 AND stock > 0"
# 价格范围保留 80% 数据（选择性低）

# ✅ 正确：选择性高的条件在前
"stock > 0 AND price >= 100 AND price <= 500"
# stock > 0 只保留 10% 数据（选择性高）
```
