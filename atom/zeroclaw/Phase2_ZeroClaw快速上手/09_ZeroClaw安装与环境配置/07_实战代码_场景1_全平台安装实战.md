# 实战代码 · 场景 1：全平台安装实战

> **场景目标：** 在 Linux / macOS / Windows / Docker / 低资源设备五个平台上，从零完成 ZeroClaw 的安装、验证和首次运行，最终看到 Agent 回复 "Hello"。

---

## 场景概述

### 你将完成什么？

```
平台选择 → 依赖安装 → ZeroClaw 安装 → 安装验证 → 首次配置 → Hello World
  (1分钟)    (2-5分钟)   (1-30分钟)     (30秒)      (2分钟)     (30秒)
```

### 前置条件

| 条件 | 说明 |
|------|------|
| 终端访问 | 能打开命令行（Terminal / PowerShell / WSL） |
| 网络连接 | 需要下载安装包和调用 LLM API |
| LLM API Key | OpenRouter / OpenAI / Anthropic 任选一个（或用 Ollama 本地免费） |

### 安装方式快速选择指南

```
你的平台是什么？
│
├── macOS
│   ├── 有 Homebrew？ ────→ brew install zeroclaw     (最简单)
│   └── 没有 Homebrew？ ──→ curl 一键脚本              (30秒)
│
├── Linux
│   ├── Ubuntu / Debian ──→ curl 一键脚本              (推荐)
│   ├── Fedora / RHEL ────→ curl 一键脚本              (推荐)
│   └── 树莓派 / ARM ─────→ --prefer-prebuilt          (避免编译)
│
├── Windows
│   ├── 有 WSL？ ─────────→ 按 Linux 流程走            (推荐)
│   └── 原生 Windows ─────→ Visual Studio + cargo      (较复杂)
│
└── Docker ───────────────→ docker build / compose     (隔离环境)
```

### 原生 Windows 安装

#### 步骤 1：安装 Visual Studio Build Tools 2022

1. 下载 [Visual Studio Build Tools 2022](https://visualstudio.microsoft.com/visual-cpp-build-tools/)（免费）
2. 运行安装器
3. 勾选 **"Desktop development with C++"**（桌面 C++ 开发）
4. 确保以下组件已选中：
   - MSVC v143 - VS 2022 C++ x64/x86 build tools

**预期结果：**

[Visual Studio Installer]
✅ Desktop development with C++ — 已安装

   ✅ Windows 11 SDK

### Windows 特有问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `linker 'link.exe' not found` | 未安装 Build Tools | 安装 Visual Studio Build Tools 并勾选 C++ |
| `cargo` 命令不存在 | PATH 未更新 | 关闭并重新打开 PowerShell |
| 编译极慢 | Windows Defender 实时扫描 | 将 `~/.cargo` 和项目目录加入排除列表 |
| 路径含中文报错 | Rust 编译器不支持非 ASCII 路径 | 将项目放在英文路径下（如 `C:\dev\`) |

> **性能提示：** Windows 上编译 Rust 通常比 Linux/macOS 慢 2-3 倍。建议将 `~/.cargo` 目录加入 Windows Defender 排除列表：
> ```
> 设置 → 隐私和安全 → Windows 安全中心 → 病毒和威胁防护 → 管理设置 → 排除 → 添加排除
> ```

### 方式 B：Docker Compose（推荐生产部署）

创建 `docker-compose.yml`：

```yaml
# docker-compose.yml

  zeroclaw:
    build:
      context: .
      target: release           # 使用 distroless 极致小镜像
    environment:
      - ZEROCLAW_API_KEY=${ZEROCLAW_API_KEY}           # 从 .env 文件读取
      - ZEROCLAW_PROVIDER=openrouter
      - ZEROCLAW_MODEL=anthropic/claude-sonnet-4-6

      - "42617:42617"
    volumes:
      - zeroclaw-data:/zeroclaw-data                   # 持久化数据
    restart: unless-stopped

volumes:
  zeroclaw-data:

### 方式 C：使用 Bootstrap 脚本的 Docker 模式

curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --docker

### Docker 多阶段构建解析

ZeroClaw 的 Dockerfile 使用三个 stage，你可以按需选择：

```
docker build --target <stage> .
```

| Stage | 基础镜像 | 用途 | 镜像大小 |
|-------|---------|------|---------|
| `builder` | `rust:1.93-slim` | 编译（仅构建阶段使用） | ~2 GB |
| `dev` | `debian:trixie-slim` | 开发/调试（有 bash、工具） | ~80 MB |
| **`release`** ⭐ | `distroless/cc-debian13` | 生产环境（无 shell，最安全） | ~23 MB |

```bash
# 开发/调试用（可以 exec 进容器排查问题）
docker build -t zeroclaw:dev --target dev .
docker exec -it <container_id> bash

# 生产用（无法 exec，但最安全最小）
docker build -t zeroclaw:release --target release .
```

> **前端类比：** 这和前端 Docker 多阶段构建一模一样：
> ```
> Stage 1 (node:20)     → npm run build  → 构建产物
> Stage 2 (nginx:alpine) → COPY dist     → 生产镜像
> ```
> ZeroClaw 的 distroless 镜像比 alpine 还极端——连 shell 都没有。

[来源: sourcecode/zeroclaw/Dockerfile]

## 场景 1.5：低资源设备（树莓派 / ARM 开发板）

> **预计耗时：** 2-5 分钟（预编译）

### 为什么要单独说？

树莓派等 ARM 设备通常只有 1-4 GB RAM。而 Rust 源码编译需要至少 2 GB RAM + 6 GB 磁盘。在这类设备上直接编译几乎必定失败（被 OOM kill）。

```
树莓派 4 (4GB RAM) 源码编译 ZeroClaw = ❌ 很可能 OOM
树莓派 4 (4GB RAM) 预编译安装 ZeroClaw = ✅ 30 秒搞定
```

### 推荐方案：预编译二进制

```bash
# 方案 A（推荐）：bootstrap 脚本 + --prefer-prebuilt
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --prefer-prebuilt

# 方案 B（更严格）：仅预编译，没有就报错，不尝试源码编译
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --prebuilt-only
```

**预期输出：**

```
🔍 Detecting platform: linux-aarch64 (ARM 64-bit)
📦 Downloading prebuilt binary for aarch64-unknown-linux-gnu...
████████████████████████ 100% | 8.8 MB
🎉 ZeroClaw v0.1.7 installed to ~/.cargo/bin/zeroclaw

Run `zeroclaw onboard` to get started!
```

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh — `--prefer-prebuilt` 和 `--prebuilt-only` 参数]

### 手动下载预编译二进制

如果一键脚本不可用（网络受限），可以手动下载：

```bash
# 确认你的 CPU 架构
uname -m
# 输出 aarch64 = ARM 64 位（树莓派 4/5）
# 输出 armv7l  = ARM 32 位（树莓派 3）

# 根据架构选择下载链接
# 树莓派 4/5 (64位)
TARGET="aarch64-unknown-linux-gnu"

# 树莓派 3 (32位)
# TARGET="armv7-unknown-linux-gnueabihf"

# 下载、解压、安装
curl -LO "https://github.com/zeroclaw-labs/zeroclaw/releases/latest/download/zeroclaw-${TARGET}.tar.gz"
tar xzf "zeroclaw-${TARGET}.tar.gz"
sudo mv zeroclaw /usr/local/bin/

# 验证
zeroclaw --version
```

**预期输出：**

```
$ uname -m
aarch64

$ zeroclaw --version
zeroclaw 0.1.7 (built 2026-02-01, commit abc1234)
```

### 备选方案：交叉编译（在高配机器上编译，拷贝到树莓派）

如果你需要自定义 Feature（如 `hardware` 支持 GPIO），预编译版不包含该功能，需要交叉编译：

```bash
# 在你的 PC/Mac 上执行（而非树莓派上）

# 1. 添加目标平台
rustup target add aarch64-unknown-linux-gnu

# 2. 安装交叉编译工具链（Ubuntu/Debian）
sudo apt install -y gcc-aarch64-linux-gnu

# 3. 编译
cargo build --release --target aarch64-unknown-linux-gnu --features "hardware"

# 4. 拷贝到树莓派
scp target/aarch64-unknown-linux-gnu/release/zeroclaw pi@raspberrypi:~/
```

### 低资源设备性能数据

| 指标 | 树莓派 4 (4GB) | 树莓派 3 (1GB) |
|------|---------------|---------------|
| 二进制大小 | ~8.8 MB | ~8.8 MB |
| 运行内存 | ~4 MB | ~4 MB |
| 启动时间 | ~8ms | ~15ms |
| Agent 响应 | 取决于网络/API | 取决于网络/API |

> ZeroClaw 的运行时资源消耗极低——~4 MB 内存 + <10ms 启动。瓶颈在 LLM API 调用的网络延迟，不在设备性能。

## 常见安装问题排查表

| # | 现象 | 平台 | 原因 | 解决方案 |
|---|------|------|------|---------|
| 1 | `command not found: zeroclaw` | 全平台 | PATH 未更新 | `source ~/.bashrc` 或 `source "$HOME/.cargo/env"` 或重开终端 |
| 2 | `curl: command not found` | Linux | 系统未预装 curl | `sudo apt install curl` |
| 3 | `linker 'cc' not found` | Linux | 缺少构建工具 | `sudo apt install build-essential` |
| 4 | `failed to run custom build command for openssl-sys` | Linux | 缺少 OpenSSL 开发头文件 | `sudo apt install libssl-dev`（Debian）或 `sudo dnf install openssl-devel`（Fedora） |
| 5 | `Killed (signal 9)` | 低配设备 | 内存不足被 OOM kill | 使用 `--prebuilt-only` 或 `cargo build -j 1`（单线程编译） |
| 6 | `requires rustc 1.87.0 or newer` | 全平台 | Rust 版本过低 | `rustup update stable` |
| 7 | `无法验证开发者` 弹窗 | macOS | Gatekeeper 拦截未签名二进制 | `xattr -d com.apple.quarantine $(which zeroclaw)` |
| 8 | 编译极慢（>1小时） | Windows | Windows Defender 实时扫描 | 将 `~/.cargo` 加入排除列表 |
| 9 | Docker 构建失败 | Docker | 网络问题 / 磁盘空间不足 | 检查 `docker system df`，清理旧镜像 `docker system prune` |
| 10 | `zeroclaw doctor` 报 Provider unreachable | 全平台 | 网络问题 / 防火墙 / 代理 | 设置 `export https_proxy=http://127.0.0.1:7890` 或检查 API Key |
| 11 | `pkg-config` 找不到库 | Linux | 缺少 pkg-config 或库未安装 | `sudo apt install pkg-config libssl-dev` |
| 12 | 路径含中文报错 | Windows | Rust 编译器路径问题 | 将项目和 `.cargo` 放在纯英文路径下 |
