# 07_实战代码_场景5_多格式响应工具

> 使用 content_and_artifact 响应格式，展示如何返回摘要和完整数据的最佳实践

---

## 场景说明

**目标**：创建一个搜索工具，既能给 LLM 提供简洁摘要，又能给程序提供完整数据

**适用场景**：
- API 调用返回大量数据，但 LLM 只需要摘要
- 需要保留原始数据供后续处理（如前端展示、数据分析）
- 需要区分"给 AI 看的"和"给程序用的"
- 需要附加元数据（如执行时间、API 版本）

**为什么使用 content_and_artifact**：
- 默认的 `content` 模式只返回字符串，丢失结构化信息
- `content_and_artifact` 模式返回 `(摘要, 完整数据)` 元组
- LLM 看到摘要，程序可以访问完整数据

## 完整代码实现

```python
"""
多格式响应工具 - content_and_artifact 实战
演示：搜索 API、数据摘要、元数据管理
"""

from langchain.tools import tool
from langchain_core.messages import ToolMessage
from typing import Tuple, Dict, Any, List
from datetime import datetime
import json
import time


# ===== 1. 基础示例：搜索工具 =====
@tool(response_format="content_and_artifact")
def search_api(query: str, max_results: int = 10) -> Tuple[str, Dict[str, Any]]:
    """
    搜索 API 工具（模拟）

    Args:
        query: 搜索关键词
        max_results: 最大结果数（1-100）

    Returns:
        Tuple[str, Dict]: (摘要, 完整数据)
        - 摘要：给 LLM 看的简短描述
        - 完整数据：包含所有结果和元数据的字典
    """
    start_time = time.time()

    # 模拟搜索结果
    results = [
        {
            "title": f"LangChain 教程 {i+1}",
            "url": f"https://example.com/tutorial-{i+1}",
            "snippet": f"这是关于 {query} 的教程内容...",
            "score": 0.95 - i * 0.05,
            "published_date": "2026-03-01"
        }
        for i in range(min(max_results, 10))
    ]

    # 计算执行时间
    execution_time = (time.time() - start_time) * 1000  # 毫秒

    # ===== 摘要（给 LLM 看） =====
    summary = f"""找到 {len(results)} 条关于 '{query}' 的结果。

最相关的结果：
- {results[0]['title']} (相关度: {results[0]['score']:.2f})
  {results[0]['snippet']}

其他结果请查看完整数据。"""

    # ===== 完整数据（给程序用） =====
    artifact = {
        "query": query,
        "total_results": len(results),
        "results": results,
        "metadata": {
            "execution_time_ms": round(execution_time, 2),
            "api_version": "v2.1",
            "timestamp": datetime.now().isoformat(),
            "max_results_requested": max_results
        }
    }

    return summary, artifact


# ===== 2. 进阶示例：数据库查询工具 =====
@tool(response_format="content_and_artifact")
def database_query(
    table: str,
    filters: str = "",
    limit: int = 10
) -> Tuple[str, Dict[str, Any]]:
    """
    数据库查询工具（模拟）

    Args:
        table: 表名（users, products, orders）
        filters: 过滤条件（如 "age > 18"）
        limit: 返回记录数

    Returns:
        Tuple[str, Dict]: (摘要, 完整数据)
    """
    # 模拟查询
    mock_data = {
        "users": [
            {"id": 1, "name": "Alice", "age": 25, "email": "alice@example.com"},
            {"id": 2, "name": "Bob", "age": 30, "email": "bob@example.com"},
            {"id": 3, "name": "Charlie", "age": 22, "email": "charlie@example.com"},
        ],
        "products": [
            {"id": 101, "name": "Laptop", "price": 999.99, "stock": 50},
            {"id": 102, "name": "Mouse", "price": 29.99, "stock": 200},
        ],
        "orders": [
            {"id": 1001, "user_id": 1, "product_id": 101, "quantity": 1},
            {"id": 1002, "user_id": 2, "product_id": 102, "quantity": 2},
        ]
    }

    # 获取数据
    data = mock_data.get(table, [])[:limit]

    # ===== 摘要 =====
    if data:
        summary = f"""查询 {table} 表成功，返回 {len(data)} 条记录。

示例记录：
{json.dumps(data[0], indent=2, ensure_ascii=False)}

{'（还有更多记录，请查看完整数据）' if len(data) > 1 else ''}"""
    else:
        summary = f"查询 {table} 表，未找到匹配的记录。"

    # ===== 完整数据 =====
    artifact = {
        "table": table,
        "filters": filters,
        "limit": limit,
        "records": data,
        "metadata": {
            "record_count": len(data),
            "query_time": datetime.now().isoformat(),
            "has_more": len(mock_data.get(table, [])) > limit
        }
    }

    return summary, artifact


# ===== 3. 高级示例：文件分析工具 =====
@tool(response_format="content_and_artifact")
def analyze_file(file_path: str) -> Tuple[str, Dict[str, Any]]:
    """
    文件分析工具（模拟）

    Args:
        file_path: 文件路径

    Returns:
        Tuple[str, Dict]: (摘要, 完整分析结果)
    """
    # 模拟文件分析
    analysis = {
        "file_path": file_path,
        "file_size": 1024 * 512,  # 512 KB
        "line_count": 1500,
        "word_count": 8500,
        "language": "Python",
        "imports": [
            "langchain",
            "openai",
            "pydantic"
        ],
        "functions": [
            {"name": "main", "lines": 50, "complexity": 5},
            {"name": "process_data", "lines": 30, "complexity": 3},
            {"name": "validate_input", "lines": 20, "complexity": 2}
        ],
        "issues": [
            {"line": 45, "severity": "warning", "message": "Unused variable 'temp'"},
            {"line": 120, "severity": "info", "message": "Consider using f-string"}
        ]
    }

    # ===== 摘要 =====
    summary = f"""文件分析完成：{file_path}

基本信息：
- 大小：{analysis['file_size'] / 1024:.1f} KB
- 行数：{analysis['line_count']}
- 语言：{analysis['language']}

代码质量：
- 函数数量：{len(analysis['functions'])}
- 发现问题：{len(analysis['issues'])} 个（{sum(1 for i in analysis['issues'] if i['severity'] == 'warning')} 个警告）

详细分析结果请查看完整数据。"""

    # ===== 完整数据 =====
    artifact = {
        "analysis": analysis,
        "metadata": {
            "analyzer_version": "1.0.0",
            "analysis_time": datetime.now().isoformat(),
            "duration_ms": 234
        }
    }

    return summary, artifact
```

## 关键技术点解析

### 1. response_format 参数

```python
@tool(response_format="content_and_artifact")
def my_tool(param: str) -> Tuple[str, Dict[str, Any]]:
    summary = "简短摘要"
    full_data = {"key": "value"}
    return summary, full_data
```

**两种模式对比**：

| 特性 | content（默认） | content_and_artifact |
|------|----------------|----------------------|
| 返回类型 | `str` | `Tuple[str, Dict]` |
| LLM 看到 | 全部内容 | 只看到摘要（第一个元素） |
| 程序访问 | 只有字符串 | 可访问完整数据（artifact） |
| 适用场景 | 简单文本响应 | 大量数据 + 需要结构化访问 |

### 2. ToolMessage 结构

```python
result = search_api.invoke({"query": "test"})

# ToolMessage 对象
print(result.type)          # "tool"
print(result.content)       # 摘要字符串
print(result.artifact)      # 完整数据字典
print(result.tool_call_id)  # 工具调用 ID
print(result.name)          # 工具名称
```

**ToolMessage 属性**：
- `content`: 给 LLM 看的摘要（字符串）
- `artifact`: 给程序用的完整数据（任意类型，通常是字典）
- `tool_call_id`: 关联的工具调用 ID
- `name`: 工具名称
- `type`: 消息类型（固定为 "tool"）

### 3. 摘要编写最佳实践

```python
# ✅ 好的摘要
summary = f"""找到 {len(results)} 条结果。

最相关：{results[0]['title']}
{results[0]['snippet']}

详细数据请查看 artifact。"""

# ❌ 不好的摘要
summary = json.dumps(results)  # 太长，LLM 难以理解
summary = "成功"  # 太简短，信息不足
```

**摘要原则**：
- 简洁但信息完整（100-300 字）
- 突出最重要的信息
- 提示有更多数据可用
- 使用自然语言，不是 JSON

### 4. artifact 数据结构设计

```python
artifact = {
    # 核心数据
    "results": [...],

    # 元数据（推荐）
    "metadata": {
        "execution_time_ms": 123,
        "api_version": "v2.1",
        "timestamp": "2026-03-02T10:45:30",
        "has_more": False
    }
}
```

**artifact 设计原则**：
- 使用字典结构（易于访问）
- 包含完整的原始数据
- 添加 `metadata` 字段存放元信息
- 保持结构一致性（便于后续处理）

### 5. 在 Agent 中的使用

```python
from langchain.agents import AgentExecutor, create_openai_functions_agent
from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate

# 创建 Agent
llm = ChatOpenAI(model="gpt-4")
tools = [search_api, database_query, analyze_file]

prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个助手，可以使用工具查询信息。"),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}"),
])

agent = create_openai_functions_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools)

# 执行查询
result = agent_executor.invoke({
    "input": "搜索 LangChain 相关资料"
})

# Agent 看到的是摘要
print(result['output'])

# 程序可以访问完整数据
# 注意：需要从 intermediate_steps 中提取
for step in result['intermediate_steps']:
    action, observation = step
    if isinstance(observation, ToolMessage):
        print(f"工具: {observation.name}")
        print(f"完整数据: {observation.artifact}")
```

### 场景 1：简单文本响应 → 使用默认 content

**适用**：
- 响应简短（< 100 字）
- 不需要结构化数据
- LLM 直接使用结果

### 场景 2：大量数据 + 需要摘要 → 使用 content_and_artifact


## 生产级最佳实践

### 1. 错误处理

```python
@tool(response_format="content_and_artifact")
def safe_api_call(endpoint: str) -> Tuple[str, Dict]:
    """安全的 API 调用"""
    try:
        response = requests.get(endpoint, timeout=5)
        response.raise_for_status()

        data = response.json()
        summary = f"API 调用成功，返回 {len(data)} 条数据"

        return summary, {
            "data": data,
            "metadata": {
                "status_code": response.status_code,
                "response_time_ms": response.elapsed.total_seconds() * 1000
            }
        }

    except requests.Timeout:
        return "API 调用超时", {"error": "timeout", "endpoint": endpoint}

    except requests.HTTPError as e:
        return f"API 错误: {e}", {"error": str(e), "status_code": e.response.status_code}
```

```python
        "metadata": {
```

## 总结

**content_and_artifact 模式的核心价值**：
1. **分离关注点**：LLM 看摘要，程序用完整数据
2. **节省 Token**：避免将大量数据传给 LLM
3. **保留信息**：完整数据可用于后续处理
4. **元数据管理**：统一的元数据结构

**使用建议**：
- 数据量 > 1KB → 使用 content_and_artifact
- 需要结构化访问 → 使用 content_and_artifact
- 简单文本响应 → 使用默认 content
- 生产环境 → 始终包含 metadata 字段
