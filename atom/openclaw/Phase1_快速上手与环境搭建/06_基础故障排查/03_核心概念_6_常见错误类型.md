// OpenClaw 错误分类
enum ErrorCategory {
  NO_REPLIES = 'no_replies',                    // 无回复

}

## 三、7 大错误类型详解

### 3.1 错误类型 1：无回复（No Replies）

**症状**：
- 发送消息后无任何回复
- Agent 似乎没有收到消息

**常见原因**：
1. **配对待定**（最常见）
2. **提及门控**（群组消息）
3. **Agent 未启动**
4. **路由配置错误**

**日志特征**：
```bash
# 配对待定
pairing request from user123 on telegram

# 提及门控
drop guild message (mention required)

# Agent 未启动
no agent configured for channel
```

**解决方案**：
```bash
# 1. 检查配对状态
openclaw pairing list <channel>

# 2. 批准配对
openclaw pairing approve <channel> <sender>

# 3. 检查 Agent 状态
openclaw status --all

# 4. 在群组中提及机器人
# 在 Discord/Telegram 中发送：@BotName hello
```

**来源**: `reference/source_channels_troubleshooting.md`

### 3.2 错误类型 2：控制面板无法连接

**症状**：
- 无法访问 OpenClaw 控制面板
- 浏览器显示连接被拒绝

**常见原因**：
1. **Gateway 未运行**
2. **端口 18789 被占用**
3. **防火墙阻止**
4. **认证失败**

**日志特征**：
```bash
# Gateway 未运行
Runtime: stopped

# 端口被占用
EADDRINUSE: address already in use :::18789

# 认证失败
unauthorized: invalid token
```

**解决方案**：
```bash
# 1. 检查 Gateway 状态
openclaw gateway status

# 2. 如果未运行，启动 Gateway
openclaw gateway start

# 3. 如果端口被占用
openclaw gateway stop
lsof -ti:18789 | xargs kill -9
openclaw gateway start

# 4. 检查认证配置
openclaw config get gateway.auth.token
```

**来源**: `reference/source_gateway_troubleshooting.md`

### 3.3 错误类型 3：网关启动失败

**症状**：
- `openclaw gateway start` 无响应
- Gateway 启动后立即停止

**常见原因**：
1. **端口 18789 被占用**（最常见）
2. **配置错误**
3. **gateway.mode 未设置**
4. **权限不足**

**日志特征**：
```bash
# 端口被占用
another gateway instance is already listening on port 18789

# 配置错误
Gateway start blocked: set gateway.mode=local

# 权限不足
EACCES: permission denied
```

**解决方案**：
```bash
# 1. 停止其他实例
openclaw gateway stop

# 2. 检查并修复配置
openclaw doctor --repair

# 3. 设置 gateway.mode
openclaw config set gateway.mode local

# 4. 验证启动
openclaw gateway start
openclaw gateway status
```

**来源**: `reference/source_gateway_troubleshooting.md`

### 3.4 错误类型 4：通道连接但消息不流动

**症状**：
- 通道显示 connected
- 发送消息无回复
- 日志中无错误

**常见原因**：
1. **配对机制阻止**
2. **提及门控阻止**
3. **消息路由失败**
4. **Agent 处理失败**

**日志特征**：
```bash
# 配对阻止
pairing required for sender

# 提及门控
mention required in guild

# 路由失败
no route found for message

# Agent 失败
agent processing failed
```

**解决方案**：
```bash
# 1. 检查配对状态
openclaw pairing list <channel>
openclaw pairing approve <channel> <sender>

# 2. 禁用提及门控（如果需要）
openclaw config set channels.<channel>.requireMention false

# 3. 检查消息路由
openclaw logs --follow | grep routing

# 4. 检查 Agent 状态
openclaw status --all
```

**来源**: `reference/source_channels_troubleshooting.md`

### 3.5 错误类型 5：Cron/心跳未触发

**症状**：
- 定时任务不执行
- 心跳消息不发送

**常见原因**：
1. **Cron 配置错误**
2. **时区问题**
3. **Gateway 重启导致状态丢失**

**日志特征**：
```bash
# Cron 配置错误
invalid cron expression

# 时区问题
cron scheduled for wrong timezone

# 状态丢失
cron state reset after restart
```

**解决方案**：
```bash
# 1. 验证 Cron 表达式
# 使用在线工具验证：https://crontab.guru/

# 2. 检查时区配置
openclaw config get timezone

# 3. 重新配置 Cron
openclaw config set cron.schedule "0 9 * * *"

# 4. 重启 Gateway
openclaw gateway restart
```

**来源**: `reference/source_gateway_troubleshooting.md`

### 3.6 错误类型 6：节点工具失败

**症状**：
- 节点工具调用失败
- 权限请求被拒绝

**常见原因**：
1. **节点离线**
2. **权限未授予**
3. **工具配置错误**

**日志特征**：
```bash
# 节点离线
node offline or unreachable

# 权限未授予
permission denied for tool

# 配置错误
tool not found or misconfigured
```

**解决方案**：
```bash
# 1. 检查节点状态
openclaw nodes status

# 2. 重新连接节点
openclaw nodes reconnect <node-id>

# 3. 授予权限
# 在节点上批准权限请求

# 4. 验证工具配置
openclaw config get nodes.<node-id>.tools
```

**来源**: `reference/source_gateway_troubleshooting.md`

### 3.7 错误类型 7：浏览器工具失败

**症状**：
- 浏览器自动化失败
- 截图或页面操作失败

**常见原因**：
1. **浏览器未启动**
2. **CDP 连接失败**
3. **页面加载超时**

**日志特征**：
```bash
# 浏览器未启动
browser not running

# CDP 连接失败
CDP connection failed

# 页面超时
page load timeout
```

**解决方案**：
```bash
# 1. 检查浏览器状态
openclaw browser status

# 2. 重启浏览器
openclaw browser restart

# 3. 增加超时时间
openclaw config set browser.timeout 30000

# 4. 检查 CDP 端口
lsof -i :9222
```

**来源**: `reference/source_gateway_troubleshooting.md`

## 四、错误识别决策树

```typescript
// 错误识别决策树
function identifyError(symptoms: Symptoms): ErrorCategory {
  // 1. 检查 Gateway 状态
  if (!gatewayRunning()) {
    return ErrorCategory.GATEWAY_STARTUP;
  }

  // 2. 检查控制面板连接
  if (!dashboardAccessible()) {
    return ErrorCategory.DASHBOARD_CONNECTIVITY;
  }

  // 3. 检查通道状态
  if (channelConnected() && !messageFlowing()) {
    return ErrorCategory.CHANNEL_FLOW;
  }

  // 4. 检查回复
  if (!receivingReplies()) {
    return ErrorCategory.NO_REPLIES;
  }

  // 5. 检查自动化
  if (!cronTriggering()) {
    return ErrorCategory.AUTOMATION;
  }

  // 6. 检查工具
  if (nodeToolsFailing()) {
    return ErrorCategory.NODE_TOOLS;
  }

  if (browserToolsFailing()) {
    return ErrorCategory.BROWSER_TOOLS;
  }

  return null; // 未知错误
}
```

## 五、类比理解

### 5.1 后端开发类比

**类比 1：HTTP 状态码**

```typescript
// OpenClaw 错误类型 ≈ HTTP 状态码

// HTTP 状态码
// 400 Bad Request → 配置错误
// 401 Unauthorized → 认证失败
// 403 Forbidden → 权限不足
// 404 Not Found → 路由失败
// 500 Internal Server Error → Gateway 错误
// 503 Service Unavailable → 通道断开

// OpenClaw 错误
// gateway.mode not set → 400 Bad Request
// unauthorized token → 401 Unauthorized
// pairing required → 403 Forbidden
// no route found → 404 Not Found
// gateway crash → 500 Internal Server Error
// channel disconnected → 503 Service Unavailable
```

**类比 2：数据库错误码**

```typescript
// OpenClaw 错误 ≈ 数据库错误码

// MySQL 错误码
// 1045 Access denied → 认证失败
// 1062 Duplicate entry → 端口被占用
// 1146 Table doesn't exist → 配置缺失
// 2002 Connection refused → Gateway 未运行

// OpenClaw 错误
// unauthorized → 1045 Access denied
// EADDRINUSE → 1062 Duplicate entry
// gateway.mode not set → 1146 Table doesn't exist
// connection refused → 2002 Connection refused
```

### 5.2 日常生活类比

**类比 1：邮局投递系统**

```
OpenClaw 错误类型 = 邮局投递问题

1. 无回复 = 信件寄出但无回音
   - 收件人地址错误（配对待定）
   - 需要签收（提及门控）

2. 控制面板无法连接 = 邮局大门紧闭
   - 邮局未营业（Gateway 未运行）
   - 门被锁住（端口被占用）

3. 网关启动失败 = 邮局无法开门
   - 钥匙丢失（配置错误）
   - 门被堵住（端口冲突）

4. 通道消息不流动 = 邮件堆积不派送
   - 需要身份验证（配对）
   - 需要本人签收（提及）

5. Cron 未触发 = 定时派送失败
   - 时钟错误（时区问题）
   - 派送员忘记（配置错误）
```

### 7.2 监控告警

```typescript
// 错误监控脚本
import { spawn } from 'child_process';

interface ErrorAlert {
  type: ErrorCategory;
  severity: 'low' | 'medium' | 'high';
  message: string;
}

class ErrorMonitor {
  private errorPatterns: Map<RegExp, ErrorAlert> = new Map([
    [/EADDRINUSE/, {
      type: ErrorCategory.GATEWAY_STARTUP,
      severity: 'high',
      message: 'Port 18789 is already in use'
    }],
    [/pairing request/, {
      type: ErrorCategory.NO_REPLIES,
      severity: 'medium',
      message: 'Pairing request pending'
    }],
    [/unauthorized/, {
      type: ErrorCategory.DASHBOARD_CONNECTIVITY,
      severity: 'high',
      message: 'Authentication failed'
    }]
  ]);

  async monitor(): Promise<void> {
    const logs = spawn('openclaw', ['logs', '--follow']);

    logs.stdout.on('data', (data) => {
      const lines = data.toString().split('\n');

      for (const line of lines) {
        for (const [pattern, alert] of this.errorPatterns) {
          if (pattern.test(line)) {
            this.handleAlert(alert, line);
          }
        }
      }
    });
  }

  private async handleAlert(alert: ErrorAlert, logLine: string): Promise<void> {
    console.error(`[${alert.severity.toUpperCase()}] ${alert.message}`);
    console.error(`Log: ${logLine}`);

    // 发送告警通知
    await this.sendNotification(alert, logLine);

    // 尝试自动修复
    if (alert.severity === 'high') {
      await this.autoRecover(alert.type);
    }
  }

  private async autoRecover(errorType: ErrorCategory): Promise<void> {
    switch (errorType) {
      case ErrorCategory.GATEWAY_STARTUP:
        await this.runCommand('openclaw', ['gateway', 'stop']);
        await this.sleep(2000);
        await this.runCommand('openclaw', ['gateway', 'start']);
        break;

      case ErrorCategory.NO_REPLIES:
        // 自动批准配对请求
        await this.runCommand('openclaw', ['pairing', 'approve-all']);
        break;

      case ErrorCategory.DASHBOARD_CONNECTIVITY:
        await this.runCommand('openclaw', ['doctor', '--repair']);
        break;
    }
  }

  private runCommand(cmd: string, args: string[]): Promise<void> {
    return new Promise((resolve, reject) => {
      const proc = spawn(cmd, args);
      proc.on('close', (code) => {
        if (code === 0) resolve();
        else reject(new Error(`Command failed with code ${code}`));
      });
    });
  }

  private sleep(ms: number): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, ms));
  }

  private async sendNotification(alert: ErrorAlert, logLine: string): Promise<void> {
    // 实现告警通知（邮件、Slack、钉钉等）
    console.log(`Notification sent: ${alert.message}`);
  }
}

// 启动监控
const monitor = new ErrorMonitor();
monitor.monitor();
```
