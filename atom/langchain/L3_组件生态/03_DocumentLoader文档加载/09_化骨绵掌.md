## 卡片1：直觉理解 - DocumentLoader 是什么

**一句话：** DocumentLoader 是 LangChain 中将各种数据源转换为统一 Document 格式的标准接口，就像一个"万能转换器"。

**举例：**
```python
# 不同数据源，统一接口
pdf_loader = PyPDFLoader("report.pdf")
web_loader = WebBaseLoader("https://example.com")
csv_loader = CSVLoader("data.csv")

# 都使用相同的方法
pdf_docs = pdf_loader.load()
web_docs = web_loader.load()
csv_docs = csv_loader.load()
```

**应用：** 在 RAG 系统中，DocumentLoader 是第一步，负责将各种格式的原始数据转换为统一的 Document 对象，为后续的文本分块、向量化、检索提供标准化输入。

## 卡片2：形式化定义 - BaseLoader 接口

**一句话：** BaseLoader 定义了4个核心方法：`lazy_load()`（必须实现）、`load()`（便利方法）、`alazy_load()`（异步懒加载）、`aload()`（异步加载）。

**举例：**
```python
class BaseLoader(ABC):
    def lazy_load(self) -> Iterator[Document]:
        """核心方法，子类必须实现"""
        raise NotImplementedError

    def load(self) -> list[Document]:
        """便利方法，自动可用"""
        return list(self.lazy_load())

    async def alazy_load(self) -> AsyncIterator[Document]:
        """异步懒加载，有默认实现"""
        ...

    async def aload(self) -> list[Document]:
        """异步加载，自动可用"""
        return [doc async for doc in self.alazy_load()]
```

**应用：** 实现自定义 Loader 时，只需实现 `lazy_load()` 方法，其他方法自动可用。这种设计遵循了"最小实现原则"。

## 卡片3：关键概念1 - Document 与 Blob 的区别

**一句话：** Blob 是原始二进制数据（bytes/str），Document 是结构化文档（文本内容 + 元数据）。

**举例：**
```python
# Blob: 原始数据
blob = Blob(
    data=b"raw binary data",
    path="file.pdf",
    mimetype="application/pdf"
)

# Document: 结构化文档
doc = Document(
    page_content="This is the extracted text",
    metadata={"source": "file.pdf", "page": 1}
)
```

**应用：** 在 LangChain 架构中，Blob → BlobLoader（加载）→ BlobParser（解析）→ Document。这种分离使得加载和解析可以独立替换。

---

## 卡片4：关键概念2 - 懒加载模式

**一句话：** 懒加载使用 Python 生成器逐个返回文档，而不是一次性加载所有数据到内存，避免内存溢出。

**举例：**
```python
# ❌ 急切加载 - 可能内存溢出
def load_all():
    docs = []
    for item in data_source:
        docs.append(process(item))  # 全部加载到内存
    return docs

# ✅ 懒加载 - 内存友好
def lazy_load():
    for item in data_source:
        yield process(item)  # 逐个生成，用完即释放
```

**应用：** 处理大文件（如1000个PDF）时，懒加载可以节省99%的内存占用，支持处理无限大的数据集。

## 卡片5：编程实现 - 实现自定义 Loader

**一句话：** 实现自定义 Loader 只需继承 BaseLoader 并实现 `lazy_load()` 方法。

**举例：**
```python
class MyTextLoader(BaseLoader):
    def __init__(self, file_path: str):
        self.file_path = file_path

    def lazy_load(self) -> Iterator[Document]:
        """只需实现这一个方法"""
        with open(self.file_path, 'r') as f:
            for line_num, line in enumerate(f, 1):
                yield Document(
                    page_content=line.strip(),
                    metadata={"source": self.file_path, "line": line_num}
                )

# 使用
loader = MyTextLoader("file.txt")
docs = loader.load()  # 自动可用
```

**应用：** 当需要从特殊数据源（API、数据库、自定义格式）加载数据时，可以实现自定义 Loader。

## 卡片6：对比区分 - DocumentLoader vs 传统文件读取

**一句话：** DocumentLoader 提供统一接口、自动解析、元数据管理、懒加载支持，而传统方式需要手动处理所有细节。

**举例：**
```python
# 传统方式
with open("file.txt", 'r') as f:
    content = f.read()  # 手动读取
    # 手动解析
    # 手动分块
    # 手动管理元数据

# DocumentLoader 方式
loader = TextLoader("file.txt")
docs = loader.load()  # 自动处理所有细节
chunks = loader.load_and_split()  # 自动分块
```

**应用：** DocumentLoader 的统一接口使得下游组件无需关心数据来源，可以轻松切换数据源。

## 卡片7：进阶理解 - BlobLoader 与 BlobParser 分离

**一句话：** BlobLoader 负责加载原始数据，BlobParser 负责解析数据，职责分离使得加载和解析可以独立替换。

**举例：**
```python
# 架构设计
Blob (原始数据)
  ↓
BlobLoader (加载)
  ↓
BaseBlobParser (解析)
  ↓
Document (结构化文档)

# 实际使用
blob_loader = FileSystemBlobLoader("./docs")
parser = PyPDFParser()

for blob in blob_loader.yield_blobs():
    docs = parser.parse(blob)
```

**应用：** 这种设计允许同一个 BlobLoader 配合不同的 Parser 处理不同格式，或者同一个 Parser 处理来自不同 BlobLoader 的数据。

## 卡片8：高级应用 - 异步批处理

**一句话：** 使用 `aload()` 和 `alazy_load()` 可以实现异步批量加载，提升并发性能。

**举例：**
```python
import asyncio

# 异步批量加载
urls = ["https://example.com/1", "https://example.com/2", ...]
loaders = [WebBaseLoader(url) for url in urls]

# 并发加载，提升性能
results = await asyncio.gather(*[l.aload() for l in loaders])

# 使用 Semaphore 控制并发数
semaphore = asyncio.Semaphore(5)

async def load_with_limit(loader):
    async with semaphore:
        return await loader.aload()

results = await asyncio.gather(*[load_with_limit(l) for l in loaders])
```

**应用：** 在生产环境中，异步批处理可以将加载时间从100秒降低到5秒（20倍提升）。

## 卡片9：在 RAG 中的使用 - RAG 管道集成

**一句话：** DocumentLoader 是 RAG 管道的第一步，负责将原始数据转换为 Document 对象，为后续处理提供标准化输入。

**举例：**
```python
# 完整的 RAG 管道
# 1. 加载文档
loader = DirectoryLoader("./docs", glob="**/*.pdf")
documents = loader.load()

# 2. 文本分块
text_splitter = RecursiveCharacterTextSplitter(chunk_size=1000)
chunks = text_splitter.split_documents(documents)

# 3. 向量化
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_documents(chunks, embeddings)

# 4. 检索
retriever = vectorstore.as_retriever()

# 5. 生成
qa_chain = RetrievalQA.from_chain_type(llm, retriever=retriever)
```

**应用：** DocumentLoader 的元数据（来源、页码、作者）在 RAG 系统中用于溯源和过滤。

## 卡片10：总结与延伸 - 最佳实践

**一句话：** 掌握 DocumentLoader 的核心原则：懒加载优先、统一接口、职责分离、元数据管理。

**举例：**
```python
# 最佳实践总结
# 1. 小文件用 load()，大文件用 lazy_load()
if file_size < 10_000_000:  # 10MB
    docs = loader.load()
else:
    for doc in loader.lazy_load():
        process(doc)

# 2. 批量加载用异步
loaders = [WebBaseLoader(url) for url in urls]
results = await asyncio.gather(*[l.aload() for l in loaders])

# 3. 自定义 Loader 只实现 lazy_load()
class MyLoader(BaseLoader):
    def lazy_load(self) -> Iterator[Document]:
        yield Document(...)

# 4. 保留元数据用于溯源
doc.metadata = {"source": file_path, "page": page_num, "author": author}
```

**应用：** 遵循这些最佳实践可以构建高效、可维护的文档加载系统。
