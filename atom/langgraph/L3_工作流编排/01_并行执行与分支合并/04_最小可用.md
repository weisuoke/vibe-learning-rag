## 4.1 Send API：动态并行的核心

### 概念说明

**Send 是什么？**

`Send` 是 LangGraph 中实现动态并行执行的核心类，用于在条件边中动态创建并行任务。

**核心语法**：
```python
from langgraph.types import Send

Send(node="目标节点名", arg={"传递的状态"})
```

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
from typing_extensions import TypedDict, Annotated
import operator

# 1. 定义状态
class State(TypedDict):
    topics: list[str]  # 输入：主题列表
    jokes: Annotated[list[str], operator.add]  # 输出：笑话列表（自动合并）

# 2. 生成主题
def generate_topics(state: State):
    """生成要处理的主题"""
    return {"topics": ["Python", "JavaScript", "Go"]}

# 3. 条件边：动态创建并行任务
def continue_to_jokes(state: State):
    """为每个主题创建一个并行任务"""
    return [Send("generate_joke", {"topic": topic}) for topic in state["topics"]]

# 4. 并行任务：生成笑话
def generate_joke(state: State):
    """为单个主题生成笑话"""
    topic = state["topic"]
    joke = f"Why did the {topic} developer cross the road? To get to the other side!"
    return {"jokes": [joke]}

# 5. 构建图
builder = StateGraph(State)
builder.add_node("generate_topics", generate_topics)
builder.add_node("generate_joke", generate_joke)

# 6. 添加边
builder.add_edge(START, "generate_topics")
builder.add_conditional_edges(
    "generate_topics",
    continue_to_jokes,
    ["generate_joke"]
)
builder.add_edge("generate_joke", END)

# 7. 编译并运行
graph = builder.compile()
result = graph.invoke({"topics": [], "jokes": []})

print("生成的笑话：")
for joke in result["jokes"]:
    print(f"- {joke}")
```

**运行输出**：
```
生成的笑话：
- Why did the Python developer cross the road? To get to the other side!
- Why did the JavaScript developer cross the road? To get to the other side!
- Why did the Go developer cross the road? To get to the other side!
```

### 实际应用场景

- **文档并行处理**：为每个文档创建一个并行任务
- **多智能体协作**：动态创建多个智能体任务
- **批量 API 调用**：并行调用多个外部 API

## 4.2 Fan-out/Fan-in：静态并行模式

### 概念说明

**Fan-out/Fan-in 是什么？**

- **Fan-out（扇出）**：从一个节点分支到多个并行节点
- **Fan-in（扇入）**：多个并行节点合并到一个节点

**核心语法**：
```python
# Fan-out: 从 'a' 扇出到 'b' 和 'c'
builder.add_edge("a", "b")
builder.add_edge("a", "c")

# Fan-in: 'b' 和 'c' 扇入到 'd'
builder.add_edge("b", "d")
builder.add_edge("c", "d")
```

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict, Annotated
import operator
import time

# 1. 定义状态
class State(TypedDict):
    results: Annotated[list[str], operator.add]

# 2. 定义并行任务
def task_a(state: State):
    """任务 A：模拟耗时操作"""
    time.sleep(0.1)
    return {"results": ["Task A completed"]}

def task_b(state: State):
    """任务 B：模拟耗时操作"""
    time.sleep(0.1)
    return {"results": ["Task B completed"]}

def task_c(state: State):
    """任务 C：模拟耗时操作"""
    time.sleep(0.1)
    return {"results": ["Task C completed"]}

def aggregate(state: State):
    """聚合结果"""
    return {"results": [f"Aggregated {len(state['results'])} results"]}

# 3. 构建图
builder = StateGraph(State)
builder.add_node("task_a", task_a)
builder.add_node("task_b", task_b)
builder.add_node("task_c", task_c)
builder.add_node("aggregate", aggregate)

# 4. Fan-out: 从 START 扇出到三个任务
builder.add_edge(START, "task_a")
builder.add_edge(START, "task_b")
builder.add_edge(START, "task_c")

# 5. Fan-in: 三个任务扇入到 aggregate
builder.add_edge("task_a", "aggregate")
builder.add_edge("task_b", "aggregate")
builder.add_edge("task_c", "aggregate")

# 6. 结束
builder.add_edge("aggregate", END)

# 7. 编译并运行
graph = builder.compile()
start_time = time.time()
result = graph.invoke({"results": []})
elapsed = time.time() - start_time

print("执行结果：")
for r in result["results"]:
    print(f"- {r}")
print(f"\n总耗时: {elapsed:.2f}秒（并行执行，约 0.1 秒）")
```

**运行输出**：
```
执行结果：
- Task A completed
- Task B completed
- Task C completed
- Aggregated 3 results

总耗时: 0.12秒（并行执行，约 0.1 秒）
```

### 实际应用场景

- **固定任务并行**：同时调用多个固定的 API
- **多模型推理**：并行调用多个 LLM 模型
- **数据源聚合**：并行获取多个数据源

## 4.3 状态合并：Reducer 机制

### 概念说明

**Reducer 是什么？**

Reducer 是用于合并并行节点状态更新的函数，使用 `Annotated[type, reducer]` 语法定义。

**核心语法**：
```python
from typing_extensions import Annotated
import operator

# 列表追加
results: Annotated[list[str], operator.add]

# 数值相加
total: Annotated[int, operator.add]

# 自定义 reducer
def merge_dicts(left: dict, right: dict) -> dict:
    return {**left, **right}

data: Annotated[dict, merge_dicts]
```

### 代码示例

```python
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
from typing_extensions import TypedDict, Annotated
import operator

# 1. 定义状态（使用多个 reducer）
class State(TypedDict):
    numbers: list[int]
    results: Annotated[list[int], operator.add]  # 列表追加
    total: Annotated[int, operator.add]  # 数值相加
    count: Annotated[int, operator.add]  # 计数

# 2. 生成数字
def generate_numbers(state: State):
    """生成要处理的数字"""
    return {"numbers": [1, 2, 3, 4, 5]}

# 3. 条件边：动态创建并行任务
def continue_to_process(state: State):
    """为每个数字创建一个并行任务"""
    return [Send("process_number", {"number": n}) for n in state["numbers"]]

# 4. 并行任务：处理数字
def process_number(state: State):
    """处理单个数字"""
    number = state["number"]
    squared = number ** 2
    return {
        "results": [squared],  # 追加到列表
        "total": squared,      # 累加
        "count": 1             # 计数
    }

# 5. 聚合结果
def aggregate(state: State):
    """计算平均值"""
    avg = state["total"] / state["count"] if state["count"] > 0 else 0
    print(f"结果列表: {state['results']}")
    print(f"总和: {state['total']}")
    print(f"数量: {state['count']}")
    print(f"平均值: {avg}")
    return {}

# 6. 构建图
builder = StateGraph(State)
builder.add_node("generate_numbers", generate_numbers)
builder.add_node("process_number", process_number)
builder.add_node("aggregate", aggregate)

builder.add_edge(START, "generate_numbers")
builder.add_conditional_edges(
    "generate_numbers",
    continue_to_process,
    ["process_number"]
)
builder.add_edge("process_number", "aggregate")
builder.add_edge("aggregate", END)

# 7. 编译并运行
graph = builder.compile()
result = graph.invoke({
    "numbers": [],
    "results": [],
    "total": 0,
    "count": 0
})
```

**运行输出**：
```
结果列表: [1, 4, 9, 16, 25]
总和: 55
数量: 5
平均值: 11.0
```

### 实际应用场景

- **结果聚合**：合并多个并行任务的结果
- **统计计算**：累加、计数、平均值等
- **数据合并**：合并多个数据源的数据

## 4.4 Map-Reduce 模式：完整示例

### 概念说明

**Map-Reduce 是什么？**

Map-Reduce 是一种并行处理模式：
- **Map 阶段**：将任务分解为多个子任务并行执行
- **Reduce 阶段**：合并所有子任务的结果

### 代码示例

```python
"""
完整的 Map-Reduce 示例：并行处理文档并统计词频
"""

from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
from typing_extensions import TypedDict, Annotated
import operator
from collections import Counter

# 1. 定义状态
class State(TypedDict):
    documents: list[str]
    word_counts: Annotated[list[dict], operator.add]
    final_counts: dict

# 2. 加载文档
def load_documents(state: State):
    """加载要处理的文档"""
    docs = [
        "Python is great for data science",
        "Python is easy to learn",
        "Data science requires Python skills"
    ]
    return {"documents": docs}

# 3. Map 阶段：条件边
def map_documents(state: State):
    """为每个文档创建一个并行任务"""
    return [
        Send("count_words", {"document": doc})
        for doc in state["documents"]
    ]

# 4. Map 任务：统计单个文档的词频
def count_words(state: State):
    """统计单个文档的词频"""
    doc = state["document"]
    words = doc.lower().split()
    counts = dict(Counter(words))
    return {"word_counts": [counts]}

# 5. Reduce 阶段：合并所有词频
def reduce_counts(state: State):
    """合并所有文档的词频"""
    final_counts = Counter()
    for counts in state["word_counts"]:
        final_counts.update(counts)

    print("=== 词频统计结果 ===")
    for word, count in final_counts.most_common():
        print(f"{word}: {count}")

    return {"final_counts": dict(final_counts)}

# 6. 构建图
builder = StateGraph(State)
builder.add_node("load_documents", load_documents)
builder.add_node("count_words", count_words)
builder.add_node("reduce_counts", reduce_counts)

# 7. 添加边
builder.add_edge(START, "load_documents")
builder.add_conditional_edges(
    "load_documents",
    map_documents,
    ["count_words"]
)
builder.add_edge("count_words", "reduce_counts")
builder.add_edge("reduce_counts", END)

# 8. 编译并运行
graph = builder.compile()
result = graph.invoke({
    "documents": [],
    "word_counts": [],
    "final_counts": {}
})
```

**运行输出**：
```
=== 词频统计结果 ===
python: 3
is: 2
data: 2
science: 2
great: 1
for: 1
easy: 1
to: 1
learn: 1
requires: 1
skills: 1
```

### 实际应用场景

- **文档处理**：并行处理大量文档
- **数据分析**：并行计算统计指标
- **批量推理**：并行调用 LLM 处理多个输入

## 4.5 实际应用：并行 LLM 调用

### 概念说明

**为什么需要并行 LLM 调用？**

在实际应用中，经常需要：
- 同时调用多个 LLM 模型进行对比
- 并行处理多个用户请求
- 同时生成多个候选答案

### 代码示例

```python
"""
实际应用：并行调用多个 LLM 生成摘要
"""

from langgraph.graph import StateGraph, START, END
from langgraph.types import Send
from typing_extensions import TypedDict, Annotated
import operator

# 1. 定义状态
class State(TypedDict):
    text: str
    models: list[str]
    summaries: Annotated[list[dict], operator.add]

# 2. 准备输入
def prepare_input(state: State):
    """准备要处理的文本和模型列表"""
    return {
        "text": "LangGraph is a framework for building stateful, multi-actor applications with LLMs.",
        "models": ["gpt-4", "claude-3", "gemini-pro"]
    }

# 3. 条件边：为每个模型创建并行任务
def fan_out_to_models(state: State):
    """为每个模型创建一个并行任务"""
    return [
        Send("generate_summary", {
            "text": state["text"],
            "model": model
        })
        for model in state["models"]
    ]

# 4. 并行任务：调用单个模型
def generate_summary(state: State):
    """调用单个模型生成摘要（模拟）"""
    text = state["text"]
    model = state["model"]

    # 模拟 LLM 调用（实际应用中替换为真实的 API 调用）
    summary = f"[{model}] Summary: {text[:50]}..."

    return {"summaries": [{
        "model": model,
        "summary": summary
    }]}

# 5. 聚合结果
def aggregate_summaries(state: State):
    """聚合所有模型的摘要"""
    print("=== 并行生成的摘要 ===")
    for item in state["summaries"]:
        print(f"\n模型: {item['model']}")
        print(f"摘要: {item['summary']}")
    return {}

# 6. 构建图
builder = StateGraph(State)
builder.add_node("prepare_input", prepare_input)
builder.add_node("generate_summary", generate_summary)
builder.add_node("aggregate_summaries", aggregate_summaries)

builder.add_edge(START, "prepare_input")
builder.add_conditional_edges(
    "prepare_input",
    fan_out_to_models,
    ["generate_summary"]
)
builder.add_edge("generate_summary", "aggregate_summaries")
builder.add_edge("aggregate_summaries", END)

# 7. 编译并运行
graph = builder.compile()
result = graph.invoke({
    "text": "",
    "models": [],
    "summaries": []
})
```

**运行输出**：
```
=== 并行生成的摘要 ===

模型: gpt-4
摘要: [gpt-4] Summary: LangGraph is a framework for building stateful, m...

模型: claude-3
摘要: [claude-3] Summary: LangGraph is a framework for building stateful, m...

模型: gemini-pro
摘要: [gemini-pro] Summary: LangGraph is a framework for building stateful, m...
```

### 实际应用场景

- **模型对比**：并行调用多个模型进行效果对比
- **A/B 测试**：并行测试不同的 Prompt
- **多候选生成**：生成多个候选答案供用户选择
