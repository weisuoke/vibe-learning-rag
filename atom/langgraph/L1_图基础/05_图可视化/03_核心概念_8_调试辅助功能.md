# 图可视化 - 核心概念8: 调试辅助功能

## 概念定位

**核心概念**: 调试辅助功能
**重要程度**: ⭐⭐⭐ (中)
**使用频率**: 中等
**难度等级**: ⭐⭐ (中等)

## 一句话定义

**调试辅助功能是 LangGraph 在图可视化中提供的特殊标记和元数据,用于标识中断点、延迟节点等调试信息,帮助开发者快速定位和解决问题。**

## 详细解释

### 核心功能

#### 1. 中断点可视化

LangGraph 支持在节点前后设置中断点,并在可视化中显示:

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

class State(TypedDict):
    messages: list[str]

# 创建图
workflow = StateGraph(State)
workflow.add_node("node1", lambda x: x)
workflow.add_node("node2", lambda x: x)
workflow.add_node("node3", lambda x: x)

# 添加边
workflow.add_edge(START, "node1")
workflow.add_edge("node1", "node2")
workflow.add_edge("node2", "node3")
workflow.add_edge("node3", END)

# 设置中断点
app = workflow.compile(
    interrupt_before=["node2"],  # 在 node2 前中断
    interrupt_after=["node3"]    # 在 node3 后中断
)

# 可视化会显示中断点标记
from IPython.display import Image, display
display(Image(app.get_graph().draw_mermaid_png()))
```

**中断点元数据**:
```python
# 节点元数据中的中断点标记
node_metadata = {
    "__interrupt": "before"  # 或 "after" 或 "before,after"
}
```

[来源: reference/source_图可视化_01.md - 调试辅助功能]

#### 2. 延迟节点标记

LangGraph 支持标记延迟执行的节点:

```python
# 延迟节点元数据
node_metadata = {
    "defer": True  # 标记为延迟节点
}
```

**可视化效果**: 延迟节点在图表中会有特殊标记

[来源: reference/source_图可视化_01.md - 支持的特性]

#### 3. 节点元数据

LangGraph 支持在节点上附加元数据,用于调试和可视化:

```python
from langgraph.graph import StateGraph, START, END

workflow = StateGraph(State)

# 添加带元数据的节点
workflow.add_node(
    "process",
    lambda x: x,
    metadata={
        "description": "处理数据",
        "priority": "high",
        "timeout": 30
    }
)
```

### 实际应用场景

#### 场景1: 调试工作流执行

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict
from IPython.display import Image, display

class State(TypedDict):
    messages: list[str]
    step: int

# 创建工作流
workflow = StateGraph(State)

def step1(state):
    print(f"Step 1: {state}")
    return {"messages": state["messages"] + ["step1"], "step": 1}

def step2(state):
    print(f"Step 2: {state}")
    return {"messages": state["messages"] + ["step2"], "step": 2}

def step3(state):
    print(f"Step 3: {state}")
    return {"messages": state["messages"] + ["step3"], "step": 3}

workflow.add_node("step1", step1)
workflow.add_node("step2", step2)
workflow.add_node("step3", step3)

workflow.add_edge(START, "step1")
workflow.add_edge("step1", "step2")
workflow.add_edge("step2", "step3")
workflow.add_edge("step3", END)

# 设置中断点用于调试
app = workflow.compile(
    interrupt_before=["step2"],  # 在 step2 前中断,检查 step1 的输出
    interrupt_after=["step3"]    # 在 step3 后中断,检查最终结果
)

# 可视化显示中断点
print("=== 工作流结构 (带中断点) ===")
display(Image(app.get_graph().draw_mermaid_png()))

# 执行工作流
print("\n=== 执行工作流 ===")
result = app.invoke({"messages": [], "step": 0})
print(f"最终结果: {result}")
```

#### 场景2: 识别性能瓶颈

```python
import time
from langgraph.graph import StateGraph, START, END

# 创建工作流
workflow = StateGraph(State)

def fast_node(state):
    time.sleep(0.1)
    return state

def slow_node(state):
    time.sleep(2.0)  # 模拟慢节点
    return state

workflow.add_node("fast1", fast_node)
workflow.add_node("slow", slow_node, metadata={"performance": "slow"})
workflow.add_node("fast2", fast_node)

workflow.add_edge(START, "fast1")
workflow.add_edge("fast1", "slow")
workflow.add_edge("slow", "fast2")
workflow.add_edge("fast2", END)

app = workflow.compile()

# 可视化显示节点元数据
graph = app.get_graph()
for node in graph.nodes:
    if hasattr(node, 'metadata') and node.metadata:
        print(f"节点 {node.id}: {node.metadata}")

# 显示图表
from IPython.display import Image, display
display(Image(graph.draw_mermaid_png()))
```

#### 场景3: 条件路由调试

```python
from langgraph.graph import StateGraph, START, END

workflow = StateGraph(State)

def router(state):
    """路由函数"""
    if state.get("condition"):
        return "path_a"
    else:
        return "path_b"

workflow.add_node("router", lambda x: x)
workflow.add_node("path_a", lambda x: {"messages": x["messages"] + ["path_a"]})
workflow.add_node("path_b", lambda x: {"messages": x["messages"] + ["path_b"]})

workflow.add_edge(START, "router")
workflow.add_conditional_edges(
    "router",
    router,
    {
        "path_a": "path_a",
        "path_b": "path_b"
    }
)
workflow.add_edge("path_a", END)
workflow.add_edge("path_b", END)

# 设置中断点调试路由逻辑
app = workflow.compile(
    interrupt_after=["router"]  # 在路由后中断,检查路由决策
)

# 可视化显示条件边
from IPython.display import Image, display
display(Image(app.get_graph().draw_mermaid_png()))

# 测试不同条件
print("=== 测试 condition=True ===")
result1 = app.invoke({"messages": [], "condition": True})
print(f"结果: {result1}")

print("\n=== 测试 condition=False ===")
result2 = app.invoke({"messages": [], "condition": False})
print(f"结果: {result2}")
```

#### 场景4: 子图调试

```python
from langgraph.graph import StateGraph, START, END
from IPython.display import Image, display

# 创建子图
sub_workflow = StateGraph(State)
sub_workflow.add_node("sub_step1", lambda x: x)
sub_workflow.add_node("sub_step2", lambda x: x)
sub_workflow.add_edge(START, "sub_step1")
sub_workflow.add_edge("sub_step1", "sub_step2")
sub_workflow.add_edge("sub_step2", END)
sub_app = sub_workflow.compile()

# 创建主图
main_workflow = StateGraph(State)
main_workflow.add_node("main_step1", lambda x: x)
main_workflow.add_node("sub_graph", sub_app)
main_workflow.add_node("main_step2", lambda x: x)

main_workflow.add_edge(START, "main_step1")
main_workflow.add_edge("main_step1", "sub_graph")
main_workflow.add_edge("sub_graph", "main_step2")
main_workflow.add_edge("main_step2", END)

main_app = main_workflow.compile()

# 不展开子图 - 快速查看整体结构
print("=== 简单视图 ===")
display(Image(main_app.get_graph(xray=False).draw_mermaid_png()))

# 展开子图 - 深入调试子图内部
print("\n=== 详细视图 (xray=True) ===")
display(Image(main_app.get_graph(xray=True).draw_mermaid_png()))
```

### 调试技巧

#### 1. 使用中断点逐步调试

```python
# 设置多个中断点
app = workflow.compile(
    interrupt_before=["step1", "step2"],
    interrupt_after=["step3"]
)

# 逐步执行
state = {"messages": []}

# 执行到第一个中断点
state = app.invoke(state)
print(f"中断点 1: {state}")

# 继续执行到下一个中断点
state = app.invoke(state)
print(f"中断点 2: {state}")

# 继续执行到结束
state = app.invoke(state)
print(f"最终结果: {state}")
```

#### 2. 使用可视化验证图结构

```python
# 验证图结构
graph = app.get_graph()

# 检查节点数量
print(f"节点数量: {len(graph.nodes)}")

# 检查边数量
print(f"边数量: {len(graph.edges)}")

# 检查条件边
conditional_edges = [e for e in graph.edges if e.conditional]
print(f"条件边数量: {len(conditional_edges)}")

# 可视化
from IPython.display import Image, display
display(Image(graph.draw_mermaid_png()))
```

#### 3. 使用元数据标记问题节点

```python
# 标记问题节点
workflow.add_node(
    "problematic_node",
    problematic_function,
    metadata={
        "status": "needs_review",
        "issue": "timeout",
        "priority": "high"
    }
)

# 查看元数据
graph = app.get_graph()
for node in graph.nodes:
    if hasattr(node, 'metadata') and node.metadata:
        if node.metadata.get("status") == "needs_review":
            print(f"问题节点: {node.id}")
            print(f"  问题: {node.metadata.get('issue')}")
            print(f"  优先级: {node.metadata.get('priority')}")
```

#### 4. 对比不同版本的图结构

```python
# 版本 1
app_v1 = workflow_v1.compile()

# 版本 2
```

### 常见问题

#### Q1: 如何在可视化中显示中断点?

**A**: 中断点会自动在可视化中显示:
```python
app = workflow.compile(
    interrupt_before=["node1"],
    interrupt_after=["node2"]
)

# 可视化会自动显示中断点标记
display(Image(app.get_graph().draw_mermaid_png()))
```

#### Q2: 如何查看节点的元数据?

**A**:
```python
graph = app.get_graph()
for node in graph.nodes:
    print(f"节点: {node.id}")
    if hasattr(node, 'metadata') and node.metadata:
        print(f"  元数据: {node.metadata}")
```

#### Q3: 如何使用可视化调试条件边?

**A**:
```python
# 1. 可视化显示所有可能的路径
display(Image(app.get_graph().draw_mermaid_png()))

# 2. 设置中断点在条件节点后
app = workflow.compile(interrupt_after=["condition_node"])

# 3. 执行并检查路由决策
result = app.invoke({"input": "test"})
print(f"路由到: {result}")
```

#### Q4: 如何调试子图?

**A**:
```python
# 使用 xray=True 展开子图
display(Image(app.get_graph(xray=True).draw_mermaid_png()))

# 或者单独可视化子图
sub_graph = sub_app.get_graph()
display(Image(sub_graph.draw_mermaid_png()))
```

### 总结

调试辅助功能是 LangGraph 图可视化的重要特性:

1. **核心功能**: 中断点可视化、延迟节点标记、节点元数据
2. **使用场景**: 调试工作流、识别瓶颈、验证路由、子图调试
3. **调试技巧**: 逐步调试、结构验证、元数据标记、版本对比
4. **最佳实践**: 开发阶段使用中断点,调试阶段使用 xray,性能优化标记慢节点

**记住**: 调试辅助功能让 LangGraph 的开发和调试更加高效,是构建复杂工作流的重要工具。
