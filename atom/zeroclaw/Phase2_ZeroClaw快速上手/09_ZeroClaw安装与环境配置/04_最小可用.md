# 04 最小可用 — 20% 核心解决 80% 问题

> **核心理念：** 你不需要读完所有文档才能开始。掌握下面 5 个要点，就能在任何平台上成功安装 ZeroClaw 并跑起你的第一个 Agent。

---

## 为什么需要"最小可用"？

安装一个 CLI 工具本身不难，但 ZeroClaw 涉及 Rust 编译、LLM Provider 配置、环境变量体系等多个环节。初学者最容易犯的错误就是**一头扎进细节**——还没跑起来就开始纠结编译选项和 Feature Flags。

**前端类比：** 想想你第一次用 React 的场景——

```bash
npx create-react-app my-app
cd my-app
npm start
```

三行命令，浏览器就弹出来了。你没读 webpack 配置、没学 JSX 转译原理，但 React 已经跑起来了。**先跑起来，再理解原理**——这就是"最小可用"的核心思路。

ZeroClaw 的安装体验遵循同样的哲学：

| 对比维度 | React 首次体验 | ZeroClaw 首次体验 |
|---------|---------------|-------------------|
| 一键启动 | `npx create-react-app` | `curl ... \| bash` |
| 交互式配置 | 选择模板 (TypeScript?) | `zeroclaw onboard`（选择 Provider） |
| 验证成功 | `npm start` → 浏览器打开 | `zeroclaw agent -m "Hello!"` → 终端回复 |
| 花费时间 | ~2 分钟 | ~2 分钟 |

## 4.1 一键安装（30 秒搞定）

### 做什么

一条命令完成 ZeroClaw CLI 的下载与安装。

### 完整命令

```bash
# 标准安装（推荐，适用于 macOS / Linux / WSL）
curl -fsSL https://zeroclawlabs.ai/install.sh | bash
```

### 这条命令背后发生了什么

安装脚本会按以下顺序自动执行：

```
1. 检测操作系统和 CPU 架构（x86_64 / aarch64 / Apple Silicon）
2. 检查本地是否已安装 Rust
   ├── 已安装 → 跳过
   └── 未安装 → 自动安装 rustup + stable 工具链
3. 查找预编译二进制文件
   ├── 找到 → 直接下载（最快，30秒内完成）
   └── 未找到 → 从源码编译（需要 3-10 分钟）
4. 将 zeroclaw 放入 PATH（通常是 ~/.cargo/bin/）
5. 打印版本号，确认安装成功
```

### 预期输出

```
🔍 Detecting platform: darwin-aarch64 (Apple Silicon)
✅ Rust toolchain found: stable-1.82.0
📦 Downloading prebuilt binary for darwin-aarch64...
████████████████████████ 100% | 23.4 MB
🎉 ZeroClaw v0.9.x installed to ~/.cargo/bin/zeroclaw

Run `zeroclaw onboard` to get started!
```

### 常见变体

```bash
# 低资源系统（如 CI 容器、树莓派）：强制使用预编译，跳过源码编译
curl -fsSL https://zeroclawlabs.ai/install.sh | bash -s -- --prefer-prebuilt

# 指定版本安装
curl -fsSL https://zeroclawlabs.ai/install.sh | bash -s -- --version 0.9.2

# 如果你已有 Rust 环境，也可以直接用 cargo
cargo install zeroclaw-cli
```

### 排错速查

| 现象 | 原因 | 解决 |
|------|------|------|
| `curl: command not found` | 系统没装 curl | `sudo apt install curl`（Ubuntu）或 `brew install curl`（macOS） |
| 下载速度极慢 | 网络问题 | 设置代理：`export https_proxy=http://127.0.0.1:7890` |
| 编译失败 | 缺少系统依赖 | 先运行 `zeroclaw doctor`（如已部分安装）或手动装 `build-essential`、`pkg-config`、`libssl-dev` |
| `Permission denied` | 权限不足 | 不要用 `sudo`，确保 `~/.cargo/bin` 在 PATH 中 |

> **[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]** — 安装脚本的平台检测与 `--prefer-prebuilt` 逻辑均在此文件中实现。

## 4.2 首次配置（zeroclaw onboard）

### 做什么

通过交互式向导完成首次配置，生成配置文件 `~/.zeroclaw/config.toml`。

### 完整命令

```bash
# 交互式（推荐初次使用）
zeroclaw onboard
```

### 交互过程与预期输出

```
🧭 Welcome to ZeroClaw!

? Select your LLM provider:
  ❯ openrouter  (推荐，支持多种模型)
    openai
    anthropic
    ollama      (本地运行，免费)

? Enter your API key: sk-or-v1-xxxxxxxxxxxx

? Select default model:
  ❯ anthropic/claude-sonnet-4-6
    openai/gpt-4o
    meta/llama-3.1-70b

✅ Config saved to ~/.zeroclaw/config.toml
💡 Try: zeroclaw agent -m "What can you do?"
```

### 非交互式（适用于 CI/CD 或脚本化部署）

```bash
# 一行搞定，不会弹出交互提示
zeroclaw onboard --api-key sk-or-v1-xxx --provider openrouter --model anthropic/claude-sonnet-4-6
```

### Provider 选择指南

初学者最常问的问题就是："选哪个 Provider？" 这里给出简单决策树：

```
你想花钱吗？
├── 不想 → 选 ollama（本地模型，完全免费）
└── 可以
    ├── 想用多种模型 → 选 openrouter（一个 key 访问所有模型）
    ├── 只用 GPT → 选 openai
    └── 只用 Claude → 选 anthropic
```

> **前端类比：** `zeroclaw onboard` 就像 `npm init -y`——你可以一路回车用默认值，也可以每一步精心配置。但不管怎样，最终产物就是一个配置文件。

### 配置文件在哪

```bash
cat ~/.zeroclaw/config.toml
```

输出示例：

```toml
# ZeroClaw Configuration (auto-generated by onboard)
api_key = "sk-or-v1-xxxxxxxxxxxx"
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
```

> **[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]** — `onboard` 命令的默认 Provider 列表和配置文件路径定义于引导脚本中。

## 4.3 验证安装（3 条命令确认一切正常）

### 做什么

用三条命令分别验证：CLI 已安装、环境无问题、Agent 能正常调用 LLM。

### 命令 1：版本号

```bash
zeroclaw --version
```

预期输出：

```
zeroclaw 0.9.2 (built 2025-12-01, commit abc1234)
```

✅ **看到版本号** = CLI 二进制安装成功，PATH 配置正确。

### 命令 2：环境诊断

```bash
zeroclaw doctor
```

预期输出：

```
🔍 ZeroClaw Doctor — Environment Check

  ✅ CLI binary:       ~/.cargo/bin/zeroclaw (v0.9.2)
  ✅ Rust toolchain:   stable-1.82.0
  ✅ Config file:      ~/.zeroclaw/config.toml
  ✅ API key:          sk-or-v1-xxxx...xxxx (valid format)
  ✅ Provider:         openrouter (reachable)
  ✅ Model:            anthropic/claude-sonnet-4-6
  ✅ Disk space:       42.3 GB available
  ✅ Network:          API endpoint reachable (latency: 120ms)

All checks passed! 🎉
```

> **前端类比：** `zeroclaw doctor` 就像 `npx envinfo`——帮你一次性检查所有环境依赖，省得到处排查。

### 命令 3：实际调用

```bash
zeroclaw agent -m "Hello! Please introduce yourself briefly."
```

预期输出：

```
🤖 Agent Response:
Hello! I'm a ZeroClaw Agent powered by Claude. I can help you with
coding tasks, file operations, and answering questions. What would
you like to work on?

[tokens: 42 | latency: 1.2s | provider: openrouter]
```

✅ **看到回复** = LLM 调用链路完全通畅（API Key → Provider → Model → 响应）。

### 如果验证失败

| 失败的命令 | 常见原因 | 解决方案 |
|-----------|---------|---------|
| `--version` 报 "command not found" | PATH 未生效 | `source ~/.bashrc` 或重开终端 |
| `doctor` 报 API key invalid | Key 格式错误或过期 | 重新运行 `zeroclaw onboard` |
| `doctor` 报 Provider unreachable | 网络问题 | 检查代理设置、防火墙 |
| `agent -m` 超时 | 模型端限流或网络慢 | 换一个模型试试，或加 `--timeout 30` |

## 4.4 最常用的 3 个环境变量

### 做什么

用环境变量覆盖配置文件中的设置。当你需要**临时切换 Provider** 或在 **CI/CD 环境**中使用时特别有用。

### 完整命令

```bash
# 方式 1：逐个设置（当前终端会话有效）
export ZEROCLAW_API_KEY=sk-or-v1-xxxxxxxxxxxx
export ZEROCLAW_PROVIDER=openrouter
export ZEROCLAW_MODEL=anthropic/claude-sonnet-4-6

# 方式 2：写入 shell 配置文件（永久生效）
echo 'export ZEROCLAW_API_KEY=sk-or-v1-xxxxxxxxxxxx' >> ~/.bashrc
echo 'export ZEROCLAW_PROVIDER=openrouter' >> ~/.bashrc
echo 'export ZEROCLAW_MODEL=anthropic/claude-sonnet-4-6' >> ~/.bashrc
source ~/.bashrc

# 方式 3：单次运行时临时覆盖
ZEROCLAW_MODEL=openai/gpt-4o zeroclaw agent -m "Use GPT for this one"
```

### 优先级规则

配置值的读取遵循"就近覆盖"原则——越具体的越优先：

```
命令行参数 > 环境变量 > 项目配置文件 > 全局配置文件 > 默认值
(最高优先)                                              (最低优先)
```

> **前端类比：** 和 CSS 的特异性(specificity)一模一样——`style="..."` > `.class` > `element`。或者像 Node.js 的 `process.env.PORT || config.port || 3000`，越近的值越优先。

### 预期输出（验证环境变量生效）

```bash
# 检查当前生效的配置
zeroclaw config show
```

```
📋 Current Configuration:

  api_key:   sk-or-v1-xxxx...xxxx  [source: env]
  provider:  openrouter             [source: env]
  model:     anthropic/claude-sonnet-4-6  [source: config.toml]

  Config file: ~/.zeroclaw/config.toml
```

注意 `[source: env]` 标注——说明该值来自环境变量而非配置文件。

### 完整环境变量速查

在"最小可用"阶段你只需要上面 3 个，但这里列出完整参考：

| 环境变量 | 用途 | 示例值 |
|---------|------|-------|
| `ZEROCLAW_API_KEY` | LLM API 密钥 | `sk-or-v1-xxx` |
| `ZEROCLAW_PROVIDER` | LLM 提供商 | `openrouter` / `ollama` |
| `ZEROCLAW_MODEL` | 默认模型 | `anthropic/claude-sonnet-4-6` |
| `ZEROCLAW_LOG_LEVEL` | 日志级别（进阶） | `debug` / `info` / `warn` |
| `ZEROCLAW_CONFIG_DIR` | 配置目录（进阶） | `~/.zeroclaw` |

> **[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]** — 环境变量的读取顺序和优先级逻辑在引导脚本的配置加载模块中定义。

## 4.5 配置文件基础

### 做什么

了解 `~/.zeroclaw/config.toml` 的最小必要字段——当你不想每次都设环境变量时，把配置写到文件里。

### 最小配置文件（3 行就够）

```toml
# ~/.zeroclaw/config.toml — 最小可用配置
api_key = "sk-or-v1-xxxxxxxxxxxx"
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
```

这 3 行的含义：

| 字段 | 作用 | 前端类比 |
|------|------|---------|
| `api_key` | 访问 LLM 的凭证 | `.env` 里的 `REACT_APP_API_KEY` |
| `default_provider` | LLM 服务商 | `package.json` 里的 `proxy` 字段 |
| `default_model` | 使用哪个模型 | `tsconfig.json` 里的 `target` |

### 手动创建配置文件

如果你跳过了 `zeroclaw onboard`，可以手动创建：

```bash
mkdir -p ~/.zeroclaw

cat > ~/.zeroclaw/config.toml << 'EOF'
api_key = "sk-or-v1-xxxxxxxxxxxx"
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
EOF
```

### 项目级配置（可选进阶）

除了全局配置，还可以在项目根目录创建项目级配置：

```bash
# 在项目根目录
cat > .zeroclaw/config.toml << 'EOF'
default_model = "openai/gpt-4o"
EOF
```

项目级配置会覆盖全局配置——适合不同项目用不同模型的场景。

### TOML 格式 30 秒入门

如果你没接触过 TOML，这里是你需要知道的全部：

```toml
# 这是注释

# 字符串用引号
api_key = "sk-xxx"

# 布尔值不用引号
verbose = true

# 数字不用引号
timeout = 30

# 就这些！TOML 比 JSON 简单，比 YAML 不坑
```

> **前端类比：** TOML 之于 Rust 生态，就像 `package.json` 之于 Node.js 生态——几乎所有 Rust 项目的配置都用 TOML。

## 如果只有 5 分钟 — 极速安装指南

> 不看解释，只要命令。**复制粘贴，2 分钟跑起来。**

```bash
# Step 1: 安装（30秒）
curl -fsSL https://zeroclawlabs.ai/install.sh | bash

# Step 2: 重载 PATH（5秒）
source ~/.bashrc  # 或 source ~/.zshrc

# Step 3: 配置（30秒）
zeroclaw onboard --api-key YOUR_API_KEY --provider openrouter

# Step 4: 验证（10秒）
zeroclaw doctor

# Step 5: 开跑！（5秒）
zeroclaw agent -m "Hello! What can you do?"
```

**没有 API Key？用本地模型：**

```bash
# 先装 Ollama（另一个本地 LLM 运行器）
curl -fsSL https://ollama.ai/install.sh | bash
ollama pull llama3.1

# 再配置 ZeroClaw 指向 Ollama
zeroclaw onboard --provider ollama --model llama3.1
# 无需 API Key！
```
