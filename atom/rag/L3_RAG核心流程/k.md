# L3_RAG核心流程 知识点列表

> 掌握 RAG 系统的完整构建流程，从文档到答案的端到端实现

---

## 知识点清单

1. **RAG架构全景** - 理解 RAG 的整体架构、核心组件与数据流
2. **文档加载与解析** - 掌握多格式文档的加载、解析与预处理
3. **文本分块Chunking** - 理解分块策略对检索质量的关键影响
4. **向量存储（RAG视角）** - 掌握面向 RAG 的向量库配置与索引策略
5. **查询处理** - 查询理解、转换与扩展,优化检索效果
6. **检索器设计** - 实现高效的语义检索与相关性匹配
7. **上下文管理** - Context window 优化与文档排序策略
8. **生成与Prompt工程** - 将检索结果组装成 Prompt 并生成高质量答案

---

## 学习顺序建议

```
RAG架构全景 → 文档加载与解析 → 文本分块 → 向量存储 → 查询处理 → 检索器设计 → 上下文管理 → 生成与Prompt工程
     ↓              ↓              ↓          ↓          ↓            ↓            ↓              ↓
  建立全局认知    掌握数据入口    理解分块    配置存储    优化查询      实现检索      优化上下文      完成生成
                                  策略        层                      逻辑          利用
```

**为什么是这个顺序？**

1. **先全景后细节**：理解 RAG 整体架构，才能明白每个组件的作用和位置
2. **按数据流顺序**：文档 → 分块 → 向量化 → 存储 → 查询处理 → 检索 → 上下文管理 → 生成
3. **离线与在线分离**：前4个是离线索引阶段，后4个是在线查询阶段
4. **2025-2026标准流程**：查询处理和上下文管理已是现代RAG的标准环节
