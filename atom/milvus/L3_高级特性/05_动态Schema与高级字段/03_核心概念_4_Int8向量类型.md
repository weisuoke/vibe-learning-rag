# 核心概念4：Int8向量类型

> 本文档详细讲解 Milvus 2.6 Int8 向量类型的内存优化原理、量化方法和生产环境应用

## 一句话定义

**Int8 向量是使用 8 位整数表示向量维度的数据类型，通过标量量化将 Float32 向量压缩为 Int8，实现 75% 内存节省和 4x 性能提升，适合大规模向量存储场景。**

## Int8 向量基础

### 1. 定义与存储格式

**定义**（来源：reference/source_动态Schema_01.md）：

```protobuf
enum VectorType {
  BinaryVector = 0;
  FloatVector = 1;
  Float16Vector = 2;
  BFloat16Vector = 3;
  SparseFloatVector = 4;
  Int8Vector = 5;         // 8位整数向量（内存优化）
  ...
}
```

**存储格式对比**：

| 向量类型 | 每维度字节数 | 768维向量内存 | 相对 Float32 |
|---------|------------|-------------|-------------|
| Float32 | 4 bytes | 3072 bytes | 100% |
| Float16 | 2 bytes | 1536 bytes | 50% |
| Int8 | 1 byte | 768 bytes | 25% |

**示例**：

```python
# Float32 向量（原始）
float32_vector = [0.123, -0.456, 0.789, ...]  # 每个值 4 bytes
# 768 维 = 768 * 4 = 3072 bytes

# Int8 向量（量化后）
int8_vector = [12, -46, 79, ...]  # 每个值 1 byte
# 768 维 = 768 * 1 = 768 bytes（节省 75%）
```

---

### 2. 维度限制

**限制**（来源：reference/search_Int8向量_02.md，Issue #43466）：
- Int8 向量字段的维度必须是 8 的倍数
- 否则会报错或性能下降

**原因**：
- **SIMD 优化**：8 字节对齐，利用 CPU SIMD 指令
- **内存对齐**：提高内存访问效率
- **硬件支持**：现代 CPU 的向量指令集要求

**示例**：

```python
from pymilvus import MilvusClient, DataType

client = MilvusClient("http://localhost:19530")

# ✅ 正确：维度是 8 的倍数
schema = client.create_schema(auto_id=False)
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.INT8_VECTOR, dim=128)  # 128 = 8 * 16
schema.add_field(field_name="vector", datatype=DataType.INT8_VECTOR, dim=256)  # 256 = 8 * 32
schema.add_field(field_name="vector", datatype=DataType.INT8_VECTOR, dim=768)  # 768 = 8 * 96

# ❌ 错误：维度不是 8 的倍数
schema.add_field(field_name="vector", datatype=DataType.INT8_VECTOR, dim=100)  # 100 不是 8 的倍数
schema.add_field(field_name="vector", datatype=DataType.INT8_VECTOR, dim=765)  # 765 不是 8 的倍数
```

## 量化方法

### 1. SQ8 标量量化

**定义**（来源：reference/search_Int8向量_02.md）：
- SQ8 (Scalar Quantization 8-bit) 是将 Float32 向量转换为 Int8 向量的标量量化方法
- 每个维度独立量化，使用线性映射

**量化公式**：

```
Int8_value = round((Float32_value - min_value) / (max_value - min_value) * 255) - 128

其中：
- min_value: 向量中的最小值
- max_value: 向量中的最大值
- 255: Int8 的范围（-128 到 127，共 256 个值）
- -128: 偏移量，使结果在 -128 到 127 范围内
```

**Python 实现**：

```python
import numpy as np

def quantize_to_int8(float32_vector):
    """
    将 Float32 向量量化为 Int8 向量
    """
    # 计算最小值和最大值
    min_val = np.min(float32_vector)
    max_val = np.max(float32_vector)

    # 线性映射到 [0, 255]
    normalized = (float32_vector - min_val) / (max_val - min_val) * 255

    # 转换为 Int8（-128 到 127）
    int8_vector = np.round(normalized).astype(np.int8) - 128

    return int8_vector, min_val, max_val

# 示例
float32_vector = np.array([0.123, -0.456, 0.789, -0.234, 0.567])
int8_vector, min_val, max_val = quantize_to_int8(float32_vector)

print(f"Float32: {float32_vector}")
print(f"Int8: {int8_vector}")
print(f"Min: {min_val}, Max: {max_val}")

# 输出：
# Float32: [ 0.123 -0.456  0.789 -0.234  0.567]
# Int8: [-27 -128  127 -107   95]
# Min: -0.456, Max: 0.789
```

---

### 2. 反量化（可选）

**定义**：将 Int8 向量转换回 Float32 向量（用于验证精度损失）。

**反量化公式**：

```
Float32_value = (Int8_value + 128) / 255 * (max_value - min_value) + min_value
```

**Python 实现**：

```python
def dequantize_from_int8(int8_vector, min_val, max_val):
    """
    将 Int8 向量反量化为 Float32 向量
    """
    # 转换回 [0, 255]
    normalized = (int8_vector.astype(np.float32) + 128) / 255

    # 线性映射回原始范围
    float32_vector = normalized * (max_val - min_val) + min_val

    return float32_vector

# 反量化
reconstructed = dequantize_from_int8(int8_vector, min_val, max_val)

print(f"Original: {float32_vector}")
print(f"Reconstructed: {reconstructed}")
print(f"Error: {np.abs(float32_vector - reconstructed)}")

# 输出：
# Original: [ 0.123 -0.456  0.789 -0.234  0.567]
# Reconstructed: [ 0.12156863 -0.456       0.789      -0.23529412  0.56666667]
# Error: [0.00143137 0.         0.         0.00129412 0.00033333]
```

## 性能与精度权衡

### 1. 内存节省

**对比**（来源：reference/search_Int8向量_02.md）：

| 数据规模 | Float32 内存 | Int8 内存 | 节省比例 |
|---------|-------------|----------|---------|
| 100万条 × 768维 | 2.9 GB | 0.73 GB | 75% |
| 1000万条 × 768维 | 29 GB | 7.3 GB | 75% |
| 1亿条 × 768维 | 290 GB | 72.5 GB | 75% |

**生产案例**（来源：reference/search_Int8向量_02.md，Reddit）：
- Reddit 用户使用 RaBitQ + SQ8 节省 75% RAG 成本
- 1000万条向量从 29 GB 降低到 7.3 GB
- 可以使用更小的机器或存储更多向量

---

### 2. 查询性能提升

**性能对比**（来源：reference/search_Int8向量_02.md）：

| 指标 | Float32 | Int8 (SQ8) | 提升比例 |
|------|---------|-----------|---------|
| QPS（每秒查询数） | 1000 | 4000 | 4x |
| 索引构建时间 | 100s | 80s | 1.25x |
| 内存占用 | 100% | 25% | 4x 节省 |

**原因**：
- 内存访问更快（1 byte vs 4 bytes）
- CPU 缓存命中率更高
- SIMD 指令优化

---

### 3. 精度损失

**召回率下降**（来源：reference/search_Int8向量_02.md）：

| 数据集 | Float32 召回率 | Int8 召回率 | 下降比例 |
|--------|--------------|-----------|---------|
| SIFT1M | 98.5% | 93.2% | 5.3% |
| GIST1M | 97.8% | 92.1% | 5.7% |
| Deep1B | 96.2% | 90.5% | 5.7% |

**精度损失原因**：
- 量化误差：Float32 → Int8 的精度损失
- 距离计算误差：Int8 距离计算不如 Float32 精确
- 边界效应：接近的向量可能被量化为相同的 Int8 值

**可接受性**：
- 5-10% 召回率下降在大多数场景下可接受
- 可以通过增加 top-k 数量来补偿
- 适合对精度要求不是极高的场景

## 在 Milvus 2.6 中使用 Int8 向量

### 1. 创建 Int8 向量 Collection

```python
from pymilvus import MilvusClient, DataType

client = MilvusClient("http://localhost:19530")

# 创建 Schema
schema = client.create_schema(auto_id=False)

# 添加字段
schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True)
schema.add_field(field_name="vector", datatype=DataType.INT8_VECTOR, dim=768)  # Int8 向量
schema.add_field(field_name="text", datatype=DataType.VARCHAR, max_length=65535)

# 创建索引
index_params = client.prepare_index_params()
index_params.add_index(
    field_name="vector",
    index_type="HNSW",  # 支持 Int8 向量
    metric_type="L2",   # 支持 L2, IP, COSINE
    params={"M": 16, "efConstruction": 256}
)

# 创建 Collection
client.create_collection(
    collection_name="int8_documents",
    schema=schema,
    index_params=index_params
)
```

---

### 2. 插入 Int8 向量数据

```python
import numpy as np

# 准备数据
data = []
for i in range(1000):
    # 生成 Float32 向量
    float32_vector = np.random.randn(768).astype(np.float32)

    # 量化为 Int8
    int8_vector, _, _ = quantize_to_int8(float32_vector)

    data.append({
        "id": i,
        "vector": int8_vector.tolist(),  # 转换为列表
        "text": f"Document {i}"
    })

# 插入数据
client.insert(collection_name="int8_documents", data=data)
```

---

### 3. 查询 Int8 向量


## 常见问题

### 问题1：维度不是 8 的倍数

**症状**：创建 Int8 向量字段时报错。

**解决方案**：

```python
# 方案1：调整维度为 8 的倍数
original_dim = 765
adjusted_dim = ((original_dim + 7) // 8) * 8  # 768

# 方案2：填充向量
float32_vector = np.random.randn(765)
padded_vector = np.pad(float32_vector, (0, 3), mode='constant')  # 填充到 768
```

---

### 问题2：精度损失过大

**症状**：召回率下降超过 10%。

**解决方案**：

```python
# 方案1：使用 Float16 代替 Int8
schema.add_field(field_name="vector", datatype=DataType.FLOAT16_VECTOR, dim=768)
```
