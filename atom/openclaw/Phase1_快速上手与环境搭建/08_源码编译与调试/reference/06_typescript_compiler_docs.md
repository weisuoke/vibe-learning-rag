## 1. Generate tsconfig.json with TypeScript 5.9

**Source**: https://www.typescriptlang.org/docs/handbook/release-notes/typescript-5-9

This JSON configuration file is generated by running `tsc --init` in TypeScript 5.9. It includes modern, recommended settings for module resolution, target ECMAScript version, type checking, and JSX handling, aiming to provide a better starting point for new projects.

```json
{
  // Visit https://aka.ms/tsconfig to read more about this file
  "compilerOptions": {
    // File Layout
    // "rootDir": "./src",
    // "outDir": "./dist",
    // Environment Settings
    // See also https://aka.ms/tsconfig_modules
    "module": "nodenext",
    "target": "esnext",
    "types": [],
    // For nodejs:
    // "lib": ["esnext"],
    // "types": ["node"],
    // and npm install -D @types/node
    // Other Outputs
    "sourceMap": true,
    "declaration": true,
    "declarationMap": true,
    // Stricter Typechecking Options
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    // Style Options
    // "noImplicitReturns": true,
    // "noImplicitOverride": true,
    // "noUnusedLocals": true,
    // "noUnusedParameters": true,
    // "noFallthroughCasesInSwitch": true,
    // "noPropertyAccessFromIndexSignature": true,
    // Recommended Options
    "strict": true,
    "jsx": "react-jsx",
    "verbatimModuleSyntax": true,
    "isolatedModules": true,
    "noUncheckedSideEffectImports": true,
    "moduleDetection": "force",
    "skipLibCheck": true,
  }
}
```

## 2. TypeScript Node.js Module Options (JSON)

**Source**: https://www.typescriptlang.org/docs/handbook/modules/guides/choosing-compiler-options

Configuration for TypeScript projects intended to compile and run outputs directly in Node.js. This setup uses the `nodenext` module system to align with Node.js's ES module behavior. It's recommended to set `"type": "module"` in `package.json` or use `.mts` files when emitting ES modules.

```json
{
  "compilerOptions": {
    "module": "nodenext",
    "verbatimModuleSyntax": true
  }
}
```

## 3. Module Resolution with --module node20 (TypeScript)

**Source**: https://www.typescriptlang.org/docs/handbook/release-notes/typescript-5-9

Explains the stable '--module node20' option in TypeScript 5.9, which models Node.js v20 behavior for module settings. This option implies '--target es2023' and is intended to be a stable configuration, unlike '--module nodenext'. It supports requiring ECMAScript modules from CommonJS modules and rejects import assertions.

```typescript
// Example usage in tsconfig.json:
// {
//   "compilerOptions": {
//     "module": "node20",
//     "target": "es2023"
//   }
// }
```

## 4. TypeScript 5.9 - Minimal and Updated `tsc --init`

**Source**: https://www.typescriptlang.org/docs/handbook/release-notes/typescript-5-9

TypeScript 5.9 introduces several key changes to the default `tsconfig.json` generated by `tsc --init`. The `--module` option defaults to `nodenext`, `--target` to `esnext`, and `--jsx` to `react-jsx`, aligning with modern development practices. Additionally, `--moduleDetection` is set to `force` to ensure all implementation files are treated as modules, and `--verbatimModuleSyntax` is enabled for clearer module handling. The `types` array is now empty by default to limit unnecessary declaration file loading from `node_modules/@types`. Other recommended options like `strict`, `isolatedModules`, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`, and `skipLibCheck` are also included to promote robust and maintainable code.

## 5. TypeScript 5.9 Release Notes - Support for `--module node20`

**Source**: https://www.typescriptlang.org/docs/handbook/release-notes/typescript-5-9

TypeScript provides several `node*` options for the `--module` and `--moduleResolution` settings. Most recently, `--module nodenext` has supported the ability to `require()` ECMAScript modules from CommonJS modules, and correctly rejects import assertions (in favor of the standards-bound import attributes). TypeScript 5.9 brings a stable option for these settings called `node20`, intended to model the behavior of Node.js v20. This option is unlikely to have new behaviors in the future, unlike `--module nodenext` or `--moduleResolution nodenext`. Also unlike `nodenext`, specifying `--module node20` will imply `--target es2023` unless otherwise configured. `--module nodenext`, on the other hand, implies the floating `--target esnext`.

## OpenClaw 项目中的 TypeScript 配置

### tsconfig.json

```json
{
  "compilerOptions": {
    "allowImportingTsExtensions": true,
    "allowSyntheticDefaultImports": true,
    "declaration": true,
    "esModuleInterop": true,
    "experimentalDecorators": true,
    "forceConsistentCasingInFileNames": true,
    "lib": ["DOM", "DOM.Iterable", "ES2023", "ScriptHost"],
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "noEmit": true,
    "noEmitOnError": true,
    "outDir": "dist",
    "resolveJsonModule": true,
    "skipLibCheck": true,
    "strict": true,
    "target": "es2023",
    "useDefineForClassFields": false,
    "paths": {
      "openclaw/plugin-sdk": ["./src/plugin-sdk/index.ts"],
      "openclaw/plugin-sdk/*": ["./src/plugin-sdk/*.ts"],
      "openclaw/plugin-sdk/account-id": ["./src/plugin-sdk/account-id.ts"]
    }
  },
  "include": ["src/**/*", "ui/**/*", "extensions/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

**关键配置**:
- **module**: NodeNext (Node.js ES 模块)
- **target**: es2023 (现代 JavaScript)
- **strict**: true (严格类型检查)
- **noEmit**: true (不生成文件,仅类型检查)
- **paths**: 路径别名配置

## 关键配置选项

### module (模块系统)
- **类型**: `'commonjs' | 'esnext' | 'nodenext' | 'node20' | ...`
- **说明**: 指定模块系统
- **推荐**:
  - Node.js 项目: `nodenext` 或 `node20`
  - 浏览器项目: `esnext`

### moduleResolution (模块解析)
- **类型**: `'node' | 'nodenext' | 'bundler'`
- **说明**: 指定模块解析策略
- **推荐**: 与 `module` 保持一致

### target (目标环境)
- **类型**: `'es5' | 'es2015' | 'es2020' | 'es2023' | 'esnext'`
- **说明**: 指定编译目标 JavaScript 版本
- **推荐**: `es2023` (现代环境)

### strict (严格模式)
- **类型**: `boolean`
- **说明**: 启用所有严格类型检查选项
- **默认**: `false`
- **推荐**: `true`

### lib (库文件)
- **类型**: `string[]`
- **说明**: 指定要包含的库文件
- **示例**: `["ES2023", "DOM"]`

### noEmit (不生成文件)
- **类型**: `boolean`
- **说明**: 仅类型检查,不生成输出文件
- **用途**: 配合构建工具使用

### declaration (类型定义)
- **类型**: `boolean`
- **说明**: 生成 .d.ts 类型定义文件
- **默认**: `false`

### sourceMap (源码映射)
- **类型**: `boolean`
- **说明**: 生成 .map 文件
- **默认**: `false`

### paths (路径别名)
- **类型**: `Record<string, string[]>`
- **说明**: 配置路径别名
- **示例**:
```json
{
  "paths": {
    "@/*": ["./src/*"],
    "~/*": ["./"]
  }
}
```

### skipLibCheck (跳过库检查)
- **类型**: `boolean`
- **说明**: 跳过 .d.ts 文件的类型检查
- **推荐**: `true` (提高性能)

### esModuleInterop (ES 模块互操作)
- **类型**: `boolean`
- **说明**: 启用 CommonJS 和 ES 模块互操作
- **推荐**: `true`

### resolveJsonModule (解析 JSON 模块)
- **类型**: `boolean`
- **说明**: 允许导入 .json 文件
- **推荐**: `true`

### isolatedModules (隔离模块)
- **类型**: `boolean`
- **说明**: 确保每个文件可以独立编译
- **推荐**: `true` (配合 tsdown/esbuild)

### verbatimModuleSyntax (逐字模块语法)
- **类型**: `boolean`
- **说明**: 保留 import/export 语法
- **推荐**: `true` (TypeScript 5.0+)

## TypeScript 5.9 新特性

### 1. 默认配置改进

```bash
# 生成现代化的 tsconfig.json
tsc --init
```

**默认值变化**:
- `module`: `nodenext` (之前是 `commonjs`)
- `target`: `esnext` (之前是 `es5`)
- `jsx`: `react-jsx` (之前是 `preserve`)
- `moduleDetection`: `force` (新增)
- `verbatimModuleSyntax`: `true` (新增)

### 2. node20 模块选项

```json
{
  "compilerOptions": {
    "module": "node20",
    "target": "es2023"
  }
}
```

**特点**:
- 稳定的 Node.js v20 行为
- 自动设置 `target` 为 `es2023`
- 不会随 Node.js 更新而变化

### 3. 更严格的类型检查

```json
{
  "compilerOptions": {
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true
  }
}
```
