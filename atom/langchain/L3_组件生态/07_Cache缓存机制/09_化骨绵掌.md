## 卡片 1：缓存的本质

### 一句话核心
缓存是用空间换时间的策略，通过存储计算结果避免重复计算。

### 举例说明
```python
# 无缓存：每次都调用 API
response1 = llm.invoke("What is AI?")  # 耗时 2s，成本 $0.01
response2 = llm.invoke("What is AI?")  # 耗时 2s，成本 $0.01

# 有缓存：第二次从缓存读取
set_llm_cache(InMemoryCache())
response1 = llm.invoke("What is AI?")  # 耗时 2s，成本 $0.01
response2 = llm.invoke("What is AI?")  # 耗时 10ms，成本 $0
```

**性能对比**：
- 无缓存：总耗时 4s，总成本 $0.02
- 有缓存：总耗时 2.01s，总成本 $0.01

### 实际应用
在 RAG 系统中，相同问题的重复查询直接从缓存返回，显著降低成本和延迟。

## 卡片 2：LLM 缓存键设计

### 一句话核心
缓存键由 `(prompt, llm_string)` 组成，确保配置一致性。

### 举例说明
```python
# 缓存键示例
key1 = ("What is AI?", "model=gpt-4,temperature=0.7,max_tokens=100")
key2 = ("What is AI?", "model=gpt-4,temperature=0.9,max_tokens=100")

# key1 和 key2 不同，不会命中同一缓存
```

**为什么需要 llm_string？**
- 相同 prompt 但不同 temperature 会产生不同结果
- 缓存键必须包含配置参数，确保返回正确结果

### 实际应用
相同 prompt 但不同 temperature 不会命中缓存，避免返回不符合预期的结果。

## 卡片 3：Embedding 缓存原理

### 一句话核心
使用文本哈希作为缓存键，避免重复计算 embedding。

### 举例说明
```python
# 文本 -> 哈希 -> 缓存键
text = "hello world"
namespace = "text-embedding-3-small"
key = f"{namespace}{blake2b(text.encode()).hexdigest()}"

# 缓存流程
# 1. 计算文本哈希
# 2. 查询缓存：store.get(key)
# 3. 命中返回，未命中计算并缓存
```

**哈希算法选择**：
- SHA-1：最快，但存在碰撞风险
- BLAKE2B：推荐，安全且性能好
- SHA-256：最安全，但稍慢

### 实际应用
大规模文档处理时，首次计算并缓存，后续直接读取，节省 API 成本。

## 卡片 4：InMemoryCache 实现

### 一句话核心
使用 Python 字典存储，支持 maxsize 限制和 FIFO 淘汰。

### 举例说明
```python
class InMemoryCache(BaseCache):
    def __init__(self, maxsize: int | None = None):
        self._cache: dict[tuple[str, str], RETURN_VAL_TYPE] = {}
        self._maxsize = maxsize

    def update(self, prompt: str, llm_string: str, return_val: RETURN_VAL_TYPE):
        # FIFO 淘汰策略
        if self._maxsize is not None and len(self._cache) == self._maxsize:
            del self._cache[next(iter(self._cache))]  # 删除最早插入的
        self._cache[prompt, llm_string] = return_val
```

**关键点**：
- 使用字典存储，O(1) 查询
- FIFO 淘汰：删除最早插入的项
- 不持久化：进程重启丢失

### 实际应用
开发测试环境快速验证缓存效果，零配置启动。

## 卡片 5：RedisCache 优势

### 一句话核心
高性能、持久化、分布式共享，适合生产环境。

### 举例说明
```python
# 多个服务实例共享 Redis 缓存
# 服务实例 A
set_llm_cache(RedisCache(redis_client))
response = llm.invoke("What is AI?")  # 计算并缓存

# 服务实例 B
set_llm_cache(RedisCache(redis_client))
response = llm.invoke("What is AI?")  # 直接从缓存读取
```

**优势对比**：

| 特性 | InMemoryCache | RedisCache |
|------|---------------|------------|
| 持久化 | ❌ | ✅ |
| 分布式 | ❌ | ✅ |
| 性能 | 最快 | 很快 |
| 配置 | 零配置 | 需要 Redis |

### 实际应用
生产环境 RAG 系统的标准配置，多实例共享缓存，降低整体成本。

## 卡片 6：语义缓存原理

### 一句话核心
基于 embedding 相似度匹配，不仅匹配精确字符串。

### 举例说明
```python
# 标准缓存：精确匹配
query1 = "What is AI?"
query2 = "What is artificial intelligence?"
# 不命中缓存（字符串不同）

# 语义缓存：相似度匹配
set_llm_cache(RedisSemanticCache(
    embedding=OpenAIEmbeddings(),
    redis_url="redis://localhost:6379",
    score_threshold=0.9
))

query1 = "What is AI?"  # 计算并缓存
query2 = "What is artificial intelligence?"  # 相似度 > 0.9，命中缓存
```

**工作流程**：
1. 计算查询的 embedding
2. 在 Redis 中搜索相似向量
3. 相似度超过阈值则返回缓存结果

### 实际应用
问答系统中，相似问题共享缓存结果，显著提升命中率。

## 卡片 7：namespace 隔离

### 一句话核心
使用 namespace 隔离不同模型的缓存，避免冲突。

### 举例说明
```python
# 模型 A 的缓存
embedder_a = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(model="text-embedding-3-small"),
    store,
    namespace="text-embedding-3-small"  # 模型 A 的命名空间
)

# 模型 B 的缓存
embedder_b = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(model="text-embedding-3-large"),
    store,
    namespace="text-embedding-3-large"  # 模型 B 的命名空间
)

# 相同文本，不同模型，缓存键不同
text = "hello world"
key_a = "text-embedding-3-small" + hash(text)
key_b = "text-embedding-3-large" + hash(text)
```

**为什么需要 namespace？**
- 不同模型的 embedding 维度和值不同
- 避免缓存冲突导致返回错误结果

### 实际应用
同一存储后端缓存多个 embedding 模型，通过 namespace 隔离。

## 卡片 8：缓存键规范化

### 一句话核心
剥离消息 ID 等动态字段，确保缓存键一致性。

### 举例说明
```python
# 问题：消息 ID 每次不同
message1 = {"id": "msg_001", "content": "What is AI?"}
message2 = {"id": "msg_002", "content": "What is AI?"}
# 缓存键不同，未命中缓存

# 解决：规范化缓存键，剥离 ID
def normalize_message(message):
    return {"content": message["content"]}  # 只保留内容

key1 = hash(normalize_message(message1))
key2 = hash(normalize_message(message2))
# 缓存键相同，命中缓存
```

**常见动态字段**：
- 消息 ID
- 时间戳
- 请求 ID
- 会话 ID

### 实际应用
聊天模型缓存优化，剥离动态字段提升命中率。

## 卡片 9：批量处理优化

### 一句话核心
使用 mget/mset 批量操作，减少 I/O 次数。

### 举例说明
```python
# 低效：逐个查询
for text in texts:
    vector = store.get(text)  # 100 次 I/O

# 高效：批量查询
vectors = store.mget(texts)  # 1 次 I/O

# 低效：逐个更新
for text, vector in zip(texts, vectors):
    store.set(text, vector)  # 100 次 I/O

# 高效：批量更新
store.mset(zip(texts, vectors))  # 1 次 I/O
```

**性能对比**：
- 逐个操作：100 个文本 = 200 次 I/O
- 批量操作：100 个文本 = 2 次 I/O
- 性能提升：100 倍

### 实际应用
Embedding 缓存批量处理，显著提升性能。

## 卡片 10：缓存监控指标

### 一句话核心
监控命中率、成本节省、响应时间三个核心指标。

### 举例说明
```python
# 监控指标计算
cache_hits = 650
cache_misses = 350
total_requests = 1000

# 1. 命中率
hit_rate = cache_hits / total_requests  # 65%

# 2. 成本节省
api_cost_per_call = 0.01
cost_saved = cache_hits * api_cost_per_call  # $6.50

# 3. 响应时间
avg_cache_hit_time = 10  # ms
avg_cache_miss_time = 2000  # ms
avg_response_time = (cache_hits * avg_cache_hit_time +
                     cache_misses * avg_cache_miss_time) / total_requests
# 706 ms
```

**监控仪表盘**：
- 命中率：65%（目标 > 60%）
- 成本节省：$2000/月
- 响应时间：从 2s 降至 706ms

### 实际应用
持续优化缓存策略，提升 ROI，监控异常情况。

## 常见问题速查

### Q1：如何选择缓存后端？
- 开发测试：InMemoryCache
- 生产环境：RedisCache
- 单机持久化：SQLiteCache

### Q2：如何提升命中率？
- 规范化缓存键
- 使用语义缓存
- 合理设置过期时间

### Q3：如何监控缓存效果？
- 命中率：hits / (hits + misses)
- 成本节省：hits × api_cost
- 响应时间：平均延迟对比

### Q4：缓存失效怎么办？
- 检查缓存键一致性
- 检查配置参数变化
- 检查 Redis 连接状态

### Q5：如何避免内存泄漏？
- 设置 maxsize 限制
- 定期清理过期缓存
- 使用持久化存储
