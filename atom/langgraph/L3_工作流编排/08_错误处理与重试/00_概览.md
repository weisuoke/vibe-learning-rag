# 错误处理与重试 - 概览

> **知识点编号**：L3-08
> **所属层级**：L3_工作流编排
> **前置知识**：L2_状态管理、L3-07_工作流模式

---

## 知识点简介

**错误处理与重试**是 LangGraph 工作流编排中确保系统可靠性的核心机制。在真实的 AI Agent 和 RAG 系统中，网络波动、API 速率限制、模型超时等问题不可避免。LangGraph 提供了内置的 `RetryPolicy` 机制，让开发者可以为每个节点配置智能重试策略，同时通过完善的错误层次体系实现精细化的异常处理。

---

## 为什么这个知识点重要？

| 场景 | 没有错误处理 | 有错误处理 |
|------|-------------|-----------|
| LLM API 返回 429 | 整个工作流崩溃 | 自动退避重试，成功完成 |
| 网络短暂中断 | 用户看到错误页面 | 静默重试，用户无感知 |
| 检索服务不可用 | RAG 管道中断 | 切换到备用检索源 |
| 生产系统高并发 | 级联故障 | 断路器保护，优雅降级 |

---

## 核心内容速览

### 6 个核心概念
1. **RetryPolicy 配置详解** — 6 个参数控制重试行为
2. **异常匹配策略** — 3 种灵活的异常过滤方式
3. **退避策略** — 指数退避 + 抖动防止雷群效应
4. **LangGraph 错误层次体系** — 完整的异常分类与处理优先级
5. **默认重试行为** — 智能区分瞬态错误与编程错误
6. **降级与 Fallback 模式** — 重试耗尽后的优雅降级

### 4 个实战场景
1. **LLM API 调用重试** — 处理速率限制和超时
2. **RAG 检索容错管道** — 多源检索 fallback
3. **多节点差异化重试** — 不同节点不同策略
4. **生产级错误恢复系统** — 断路器 + 检查点恢复
