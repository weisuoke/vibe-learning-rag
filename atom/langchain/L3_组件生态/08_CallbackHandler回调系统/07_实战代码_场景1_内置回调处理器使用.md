# CallbackHandler回调系统 - 实战代码：场景1 内置回调处理器使用

## 场景概述

**场景名称**：内置回调处理器使用

**适用场景**：
- 快速调试 LangChain 应用
- 查看 LLM 和 Chain 的执行过程
- 实时监控流式输出
- 学习回调系统的基础用法

**核心价值**：
- 零配置，开箱即用
- 快速了解 LangChain 内部执行流程
- 为自定义回调处理器打基础

---

## 内置回调处理器介绍

LangChain 提供了两个常用的内置回调处理器：

### 1. StdOutCallbackHandler

**作用**：将 Chain、Agent、Tool 的执行过程打印到标准输出

**适用场景**：
- 调试 Chain 的执行流程
- 查看 Agent 的推理过程
- 监控 Tool 的调用

**输出内容**：
- Chain 开始和结束
- Agent 的动作和完成
- Tool 的输出
- 支持彩色输出

### 2. StreamingStdOutCallbackHandler

**作用**：实时打印 LLM 的流式输出（逐 token 输出）

**适用场景**：
- 流式 LLM 输出
- 实时显示生成过程
- 提升用户体验

**输出内容**：
- 每个新生成的 token
- 实时输出到 stdout

## 完整实战代码

```python
"""
CallbackHandler 实战：内置回调处理器使用
演示：StdOutCallbackHandler 和 StreamingStdOutCallbackHandler 的使用
"""

import os
from dotenv import load_dotenv

# LangChain 核心组件
from langchain_openai import ChatOpenAI
from langchain.prompts import PromptTemplate
from langchain.chains import LLMChain
from langchain.schema import HumanMessage

# 内置回调处理器
from langchain.callbacks import StdOutCallbackHandler
from langchain.callbacks.streaming_stdout import StreamingStdOutCallbackHandler

# 加载环境变量
load_dotenv()

# 确保 API key 已设置
if not os.getenv("OPENAI_API_KEY"):
    raise ValueError("请设置 OPENAI_API_KEY 环境变量")

print("=" * 80)
print("CallbackHandler 实战：内置回调处理器使用")
print("=" * 80)

# ===== 场景1：StdOutCallbackHandler 基础使用 =====
print("\n" + "=" * 80)
print("场景1：StdOutCallbackHandler 基础使用")
print("=" * 80)

# 创建 StdOutCallbackHandler 实例
stdout_handler = StdOutCallbackHandler()

# 创建 LLM（非流式）
llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.7,
    callbacks=[stdout_handler]  # 在初始化时传入回调
)

# 创建 Prompt
prompt = PromptTemplate(
    input_variables=["topic"],
    template="请用一句话解释什么是 {topic}。"
)

# 创建 Chain
chain = LLMChain(
    llm=llm,
    prompt=prompt,
    callbacks=[stdout_handler]  # Chain 也可以传入回调
)

# 执行 Chain
print("\n执行 Chain（StdOutCallbackHandler）：")
result = chain.run(topic="RAG")

print(f"\n最终结果：{result}")

# ===== 场景2：StreamingStdOutCallbackHandler 流式输出 =====
print("\n" + "=" * 80)
print("场景2：StreamingStdOutCallbackHandler 流式输出")
print("=" * 80)

# 创建 StreamingStdOutCallbackHandler 实例
streaming_handler = StreamingStdOutCallbackHandler()

# 创建流式 LLM
streaming_llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.7,
    streaming=True,  # 启用流式输出
    callbacks=[streaming_handler]  # 传入流式回调
)

# 直接调用 LLM（流式输出）
print("\n流式输出（逐 token 显示）：")
messages = [HumanMessage(content="请用三句话介绍 LangChain。")]
response = streaming_llm.invoke(messages)

print(f"\n\n完整响应：{response.content}")

# ===== 场景3：在运行时传入回调 =====
print("\n" + "=" * 80)
print("场景3：在运行时传入回调")
print("=" * 80)

# 创建不带回调的 LLM
llm_no_callback = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.7
)

# 创建不带回调的 Chain
chain_no_callback = LLMChain(
    llm=llm_no_callback,
    prompt=prompt
)

# 在运行时传入回调
print("\n运行时传入 StdOutCallbackHandler：")
result = chain_no_callback.run(
    topic="Embedding",
    callbacks=[stdout_handler]  # 运行时传入
)

print(f"\n最终结果：{result}")

# ===== 场景4：多个回调处理器组合使用 =====
print("\n" + "=" * 80)
print("场景4：多个回调处理器组合使用")
print("=" * 80)

# 创建多个回调处理器
handlers = [
    StdOutCallbackHandler(),
    StreamingStdOutCallbackHandler()
]

# 创建 LLM（同时使用两个回调）
multi_callback_llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.7,
    streaming=True,
    callbacks=handlers  # 传入多个回调
)

# 执行
print("\n同时使用 StdOutCallbackHandler 和 StreamingStdOutCallbackHandler：")
messages = [HumanMessage(content="请用一句话解释什么是向量数据库。")]
response = multi_callback_llm.invoke(messages)

print(f"\n\n完整响应：{response.content}")

# ===== 场景5：Chain 中的回调传递 =====
print("\n" + "=" * 80)
print("场景5：Chain 中的回调传递")
print("=" * 80)

# 创建复杂的 Prompt
complex_prompt = PromptTemplate(
    input_variables=["question", "context"],
    template="""
基于以下上下文回答问题：

上下文：{context}

问题：{question}

答案：
"""
)

# 创建 Chain（带回调）
complex_chain = LLMChain(
    llm=ChatOpenAI(
        model="gpt-3.5-turbo",
        temperature=0.7,
        callbacks=[stdout_handler]
    ),
    prompt=complex_prompt,
    callbacks=[stdout_handler]
)

# 执行
print("\n执行复杂 Chain（回调会追踪整个过程）：")
result = complex_chain.run(
    question="RAG 的核心步骤是什么？",
    context="RAG（Retrieval-Augmented Generation）是一种结合检索和生成的技术，主要包括文档检索、上下文注入和答案生成三个步骤。"
)

print(f"\n最终结果：{result}")

# ===== 场景6：对比有无回调的区别 =====
print("\n" + "=" * 80)
print("场景6：对比有无回调的区别")
print("=" * 80)

# 不带回调的 LLM
print("\n不带回调的执行（看不到内部过程）：")
llm_without = ChatOpenAI(model="gpt-3.5-turbo", temperature=0.7)
result_without = llm_without.invoke([HumanMessage(content="什么是 LLM？")])
print(f"结果：{result_without.content}")

# 带回调的 LLM
print("\n带回调的执行（可以看到内部过程）：")
llm_with = ChatOpenAI(
    model="gpt-3.5-turbo",
    temperature=0.7,
    callbacks=[stdout_handler]
)
result_with = llm_with.invoke([HumanMessage(content="什么是 LLM？")])
print(f"结果：{result_with.content}")

# ===== 场景7：RAG 场景中的回调使用 =====
print("\n" + "=" * 80)
print("场景7：RAG 场景中的回调使用")
print("=" * 80)

# 模拟 RAG 管道
from langchain.schema import Document

# 模拟检索到的文档
retrieved_docs = [
    Document(
        page_content="RAG 是 Retrieval-Augmented Generation 的缩写，是一种结合检索和生成的技术。",
        metadata={"source": "doc1.pdf", "page": 1}
    ),
    Document(
        page_content="RAG 的核心步骤包括：文档检索、上下文注入、答案生成。",
        metadata={"source": "doc2.pdf", "page": 3}
    )
]

# 创建 RAG Prompt
rag_prompt = PromptTemplate(
    input_variables=["context", "question"],
    template="""
基于以下检索到的文档回答问题：

{context}

问题：{question}

答案：
"""
)

# 创建 RAG Chain（带回调）
rag_chain = LLMChain(
    llm=ChatOpenAI(
        model="gpt-3.5-turbo",
        temperature=0.7,
        callbacks=[stdout_handler]
    ),
    prompt=rag_prompt,
    callbacks=[stdout_handler]
)

# 准备上下文
context = "\n\n".join([
    f"文档 {i+1}（来源：{doc.metadata['source']}）：\n{doc.page_content}"
    for i, doc in enumerate(retrieved_docs)
])

# 执行 RAG Chain
print("\n执行 RAG Chain（回调追踪整个 RAG 流程）：")
result = rag_chain.run(
    context=context,
    question="RAG 的核心步骤是什么？"
)

print(f"\n最终答案：{result}")

print("\n" + "=" * 80)
print("实战代码执行完成！")
print("=" * 80)
```

## 代码详解

### 1. StdOutCallbackHandler 使用方式

**初始化时传入**：

```python
llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    callbacks=[StdOutCallbackHandler()]  # 初始化时传入
)
```

**运行时传入**：

```python
chain.run(
    topic="RAG",
    callbacks=[StdOutCallbackHandler()]  # 运行时传入
)
```

**关键点**：
- 可以在 LLM、Chain、Agent 等组件中使用
- 支持初始化时传入和运行时传入
- 会打印 Chain 的开始和结束信息

### 2. StreamingStdOutCallbackHandler 使用方式

**必须启用流式输出**：

```python
streaming_llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    streaming=True,  # 必须启用流式输出
    callbacks=[StreamingStdOutCallbackHandler()]
)
```

**关键点**：
- 必须设置 `streaming=True`
- 只对 LLM 有效（Chain 不支持流式输出）
- 实时打印每个 token

### 3. 多个回调处理器组合

**传入列表**：

```python
handlers = [
    StdOutCallbackHandler(),
    StreamingStdOutCallbackHandler()
]

llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    streaming=True,
    callbacks=handlers  # 传入多个回调
)
```

**关键点**：
- 多个回调处理器会按顺序执行
- 可以组合不同类型的回调处理器
- 每个回调处理器独立工作

### 4. 回调传递机制

**从 Chain 传递到 LLM**：

```python
chain = LLMChain(
    llm=ChatOpenAI(callbacks=[handler1]),  # LLM 的回调
    prompt=prompt,
    callbacks=[handler2]  # Chain 的回调
)
```

**关键点**：
- Chain 的回调会传递给内部的 LLM
- 可以在不同层级设置不同的回调
- 回调会追踪整个执行链路

## 关键要点总结

### StdOutCallbackHandler

**优点**：
- 零配置，开箱即用
- 快速了解执行流程
- 适合调试和学习

**缺点**：
- 只能打印到标准输出
- 无法自定义输出格式
- 不适合生产环境

**适用场景**：
- 开发和调试阶段
- 学习 LangChain 内部机制
- 快速验证逻辑

### StreamingStdOutCallbackHandler

**优点**：
- 实时显示生成过程
- 提升用户体验
- 适合长文本生成

**缺点**：
- 只对 LLM 有效
- 必须启用流式输出
- 无法控制输出格式

**适用场景**：
- 聊天机器人
- 长文本生成
- 实时交互应用

### 回调传递机制

**关键点**：
1. **初始化时传入**：在创建组件时传入 `callbacks` 参数
2. **运行时传入**：在调用 `run()` 或 `invoke()` 时传入 `callbacks` 参数
3. **多层传递**：Chain 的回调会传递给内部的 LLM
4. **多个回调**：可以传入多个回调处理器（列表形式）

## 常见问题

### Q1：为什么 StreamingStdOutCallbackHandler 没有输出？

**原因**：没有启用流式输出

**解决方案**：

```python
# ❌ 错误：没有启用流式输出
llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    callbacks=[StreamingStdOutCallbackHandler()]
)

# ✅ 正确：启用流式输出
llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    streaming=True,  # 必须启用
    callbacks=[StreamingStdOutCallbackHandler()]
)
```

### Q2：如何在 Chain 中使用流式输出？

**答案**：Chain 不支持流式输出，只能在 LLM 层面使用

```python
# ❌ 错误：Chain 不支持流式输出
chain = LLMChain(
    llm=llm,
    prompt=prompt,
    streaming=True  # Chain 没有这个参数
)

# ✅ 正确：在 LLM 层面启用流式输出
streaming_llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    streaming=True,
    callbacks=[StreamingStdOutCallbackHandler()]
)

chain = LLMChain(llm=streaming_llm, prompt=prompt)
```

### Q3：如何同时使用多个回调处理器？

**答案**：传入列表

```python
handlers = [
    StdOutCallbackHandler(),
    StreamingStdOutCallbackHandler(),
    CustomCallbackHandler()  # 自定义回调
]

llm = ChatOpenAI(
    model="gpt-3.5-turbo",
    streaming=True,
    callbacks=handlers  # 传入列表
)
```

### Q4：回调处理器会影响性能吗？

**答案**：影响很小，但在生产环境建议使用异步回调

```python
# 同步回调（开发环境）
from langchain.callbacks import StdOutCallbackHandler

# 异步回调（生产环境）
from langchain.callbacks import AsyncCallbackHandler
```
