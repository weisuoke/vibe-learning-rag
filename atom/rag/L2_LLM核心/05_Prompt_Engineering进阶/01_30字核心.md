# 30字核心

**进阶 Prompt 技巧通过示例引导、分步推理、格式约束，让 LLM 输出更准确、更可控、更稳定。**

---

## 拆解理解

| 关键词 | 含义 |
|-------|------|
| **示例引导** | Few-shot：给 LLM 看几个例子，它就知道你想要什么 |
| **分步推理** | Chain-of-Thought：让 LLM 一步步思考，而不是直接给答案 |
| **格式约束** | 结构化输出：强制 LLM 输出 JSON、表格等固定格式 |
| **更准确** | 复杂问题的正确率提升 |
| **更可控** | 输出格式符合预期 |
| **更稳定** | 多次调用结果一致 |

---

## 三大进阶技巧速览

```
┌─────────────────────────────────────────────────────────────┐
│                    进阶 Prompt 三件套                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Few-shot              CoT                  结构化输出        │
│  ────────              ───                  ────────         │
│  "看这几个例子"         "让我们一步步想"      "输出 JSON"       │
│                                                              │
│  解决：                解决：                解决：            │
│  LLM 不知道           LLM 推理              LLM 输出          │
│  你想要什么格式        容易跳步出错          格式不稳定         │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

---

## 为什么这句话重要？

理解这句话，你就理解了：

1. **基础 vs 进阶的区别**
   - 基础：告诉 LLM "做什么"（角色、任务、约束）
   - 进阶：告诉 LLM "怎么做"（示例、推理步骤、输出格式）

2. **进阶技巧的本质**
   - 不是更复杂的 Prompt，而是更聪明的引导方式
   - 用最小的改动，获得最大的效果提升

3. **在 RAG 中的价值**
   - Few-shot：统一 RAG 回答风格
   - CoT：处理需要多文档综合的复杂问题
   - 结构化输出：让 RAG 输出可被程序解析

---

## 一个对比示例

```python
# ❌ 基础 Prompt（可能输出格式不稳定）
prompt_basic = """
根据文档回答问题。
文档：{context}
问题：{query}
"""

# ✅ 进阶 Prompt（Few-shot + 结构化输出）
prompt_advanced = """
根据文档回答问题，按 JSON 格式输出。

示例：
问题：产品保修期是多久？
输出：{"answer": "产品保修期为1年", "confidence": "high", "source": "保修政策第3条"}

问题：{query}
文档：{context}
输出：
"""
```
