# 核心概念 3：状态设计规范

> 本文档是"08_图基础最佳实践"知识点的核心概念之一，详细讲解 LangGraph 中状态设计的规范和最佳实践。

---

## 什么是状态设计？

**状态设计**指的是如何定义和管理图中节点之间共享的数据结构。

在 LangGraph 中，状态是节点之间通信的核心机制。良好的状态设计直接影响：
- 图的可维护性和可扩展性
- 节点之间的数据传递效率
- 并行执行时的数据一致性
- 持久化和恢复的可靠性

---

## 状态定义方式

### TypedDict 定义

**推荐用于简单状态**：

```python
from typing_extensions import TypedDict, Annotated
from operator import add

class State(TypedDict):
    """基础状态定义"""
    query: str                          # 用户查询
    documents: Annotated[list, add]     # 检索到的文档（使用 reducer）
    response: str                       # 生成的回复
    metadata: dict                      # 元数据
```

**优点**：
- ✅ 轻量级，性能好
- ✅ 与 LangGraph 原生集成
- ✅ 支持 Annotated[type, reducer] 语法
- ✅ 类型提示清晰

**缺点**：
- ❌ 缺少运行时验证
- ❌ 不支持复杂的数据验证
- ❌ 不支持默认值

### Pydantic 定义

**推荐用于复杂状态**：

```python
from pydantic import BaseModel, Field
from typing import List, Dict, Optional

class State(BaseModel):
    """Pydantic 状态定义"""
    query: str = Field(..., description="用户查询")
    documents: List[Dict] = Field(default_factory=list, description="检索到的文档")
    response: Optional[str] = Field(None, description="生成的回复")
    metadata: Dict = Field(default_factory=dict, description="元数据")

    class Config:
        arbitrary_types_allowed = True
```

**优点**：
- ✅ 运行时数据验证
- ✅ 支持默认值和 Field 配置
- ✅ 丰富的验证器和转换器
- ✅ 自动生成 JSON Schema

**缺点**：
- ❌ 性能略低于 TypedDict
- ❌ 需要额外的依赖
- ❌ 与 Annotated[type, reducer] 集成较复杂

### 选择建议

```python
# ✅ 简单状态：使用 TypedDict
class SimpleState(TypedDict):
    input: str
    output: str

# ✅ 复杂状态：使用 Pydantic
class ComplexState(BaseModel):
    user_id: str = Field(..., min_length=1)
    email: str = Field(..., regex=r"^[\w\.-]+@[\w\.-]+\.\w+$")
    age: int = Field(..., ge=0, le=150)
    preferences: Dict = Field(default_factory=dict)
```

## Reducer 函数设计

### 什么是 Reducer？

**Reducer 函数**定义了如何聚合多个节点对同一状态字段的更新。

```python
from typing_extensions import Annotated

# 没有 reducer：后面的更新会覆盖前面的
class State1(TypedDict):
    value: int  # 覆盖模式

# 有 reducer：多个更新会被聚合
class State2(TypedDict):
    value: Annotated[list, add]  # 追加模式
```

### 内置 Reducer

```python
from operator import add
from typing_extensions import Annotated, TypedDict

class State(TypedDict):
    # 1. 列表追加
    messages: Annotated[list, add]

    # 2. 字符串拼接
    text: Annotated[str, add]

    # 3. 数值相加
    count: Annotated[int, add]
```

### 自定义 Reducer

```python
from typing import Dict, Any

# 1. 字典合并 reducer
def merge_dict(a: Dict, b: Dict) -> Dict:
    """合并两个字典"""
    return {**a, **b}

# 2. 列表去重 reducer
def unique_list(a: list, b: list) -> list:
    """合并列表并去重"""
    seen = set(a)
    result = list(a)
    for item in b:
        if item not in seen:
            result.append(item)
            seen.add(item)
    return result

# 3. 最大值 reducer
def max_value(a: float, b: float) -> float:
    """保留最大值"""
    return max(a, b)

# 使用自定义 reducer
class State(TypedDict):
    metadata: Annotated[Dict, merge_dict]
    keywords: Annotated[list, unique_list]
    score: Annotated[float, max_value]
```

### Reducer 最佳实践

```python
# ✅ 好的 reducer：纯函数
def good_reducer(a: list, b: list) -> list:
    """纯函数，无副作用"""
    return a + b

# ❌ 不好的 reducer：有副作用
def bad_reducer(a: list, b: list) -> list:
    """修改输入参数，有副作用"""
    a.extend(b)  # 修改了 a
    return a

# ✅ 好的 reducer：处理 None
def safe_reducer(a: list | None, b: list | None) -> list:
    """安全处理 None 值"""
    if a is None:
        a = []
    if b is None:
        b = []
    return a + b

# ❌ 不好的 reducer：不处理 None
def unsafe_reducer(a: list, b: list) -> list:
    """假设输入不为 None"""
    return a + b  # 如果 a 或 b 为 None 会报错
```

## 状态字段命名规范

### 命名原则

**使用清晰、描述性的名称**：

```python
# ✅ 好的命名
class State(TypedDict):
    user_query: str              # 清晰表达含义
    search_results: list         # 描述性强
    llm_response: str            # 明确来源
    is_valid: bool               # 布尔值用 is_ 前缀
    num_retries: int             # 数量用 num_ 前缀

# ❌ 不好的命名
class State(TypedDict):
    q: str                       # 太简短
    data: list                   # 太泛化
    result: str                  # 不明确
    flag: bool                   # 不清楚含义
    count: int                   # 计数什么？
```

class State(TypedDict):

## 状态更新模式

### 部分更新 vs 完整更新

```python
from typing_extensions import TypedDict

class State(TypedDict):
    query: str
    documents: list
    response: str
    metadata: dict

# ✅ 推荐：部分更新
def search_node(state: State) -> dict:
    """只返回需要更新的字段"""
    results = search(state["query"])
    return {"documents": results}  # 只更新 documents

# ❌ 不推荐：完整更新
def search_node_bad(state: State) -> State:
    """返回完整状态"""
    results = search(state["query"])
    return {
        "query": state["query"],        # 不必要的复制
        "documents": results,
        "response": state["response"],  # 不必要的复制
        "metadata": state["metadata"]   # 不必要的复制
    }
```

### 条件更新

```python
def conditional_update_node(state: State) -> dict:
    """根据条件决定更新哪些字段"""
    updates = {}

    # 条件1：如果查询为空，设置错误
    if not state["query"]:
        updates["error"] = "Query is empty"
        return updates

    # 条件2：如果文档为空，执行搜索
    if not state["documents"]:
        updates["documents"] = search(state["query"])

    # 条件3：如果需要生成回复
    if state.get("needs_response"):
        updates["response"] = generate_response(state)

    return updates
```

## Context Schema 设计

### 什么是 Context Schema？

**Context Schema** 用于传递不可变的运行时参数，这些参数在整个图执行过程中保持不变。

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph
from langgraph.runtime import Runtime

# 状态：可变的数据
class State(TypedDict):
    query: str
    results: list

# 上下文：不可变的参数
class Context(TypedDict):
    user_id: str
    session_id: str
    db_connection: str
    api_key: str

# 创建图
graph = StateGraph(state_schema=State, context_schema=Context)

# 节点访问上下文
def search_node(state: State, *, runtime: Runtime[Context]) -> dict:
    """访问运行时上下文"""
    user_id = runtime.context["user_id"]
    api_key = runtime.context["api_key"]

    results = search(state["query"], user_id=user_id, api_key=api_key)
    return {"results": results}
```

### Context vs State

```python
# ✅ 适合放在 Context 中
class Context(TypedDict):
    user_id: str              # 用户标识（不变）
    session_id: str           # 会话标识（不变）
    db_conn: str              # 数据库连接（不变）
    api_key: str              # API 密钥（不变）
    config: dict              # 配置参数（不变）

# ✅ 适合放在 State 中
class State(TypedDict):
    query: str                # 用户输入（可变）
    documents: list           # 检索结果（可变）
    response: str             # 生成回复（可变）
    metadata: dict            # 元数据（可变）
```

## 多用户状态隔离

### 使用 thread_id 隔离

```python
from langgraph.checkpoint.memory import InMemorySaver

# 创建 checkpointer
checkpointer = InMemorySaver()

# 编译图
graph = builder.compile(checkpointer=checkpointer)

# 为不同用户创建隔离的会话
def invoke_for_user(user_id: str, session_id: str, input_data: dict):
    """为特定用户执行图"""
    config = {
        "configurable": {
            "thread_id": f"user_{user_id}_session_{session_id}"
        }
    }
    return graph.invoke(input_data, config=config)

# 用户1的会话
result1 = invoke_for_user("user_001", "session_001", {"query": "问题1"})

# 用户2的会话（完全隔离）
result2 = invoke_for_user("user_002", "session_001", {"query": "问题2"})
```

### 状态隔离最佳实践

```python
from typing_extensions import TypedDict
from langgraph.checkpoint.postgres import PostgresSaver

class MultiUserState(TypedDict):
    user_id: str              # 用户标识
    session_id: str           # 会话标识
    query: str
    history: list
    response: str

# 使用 PostgreSQL 持久化
checkpointer = PostgresSaver(
    connection_string="postgresql://user:pass@localhost/dbname"
)

graph = builder.compile(checkpointer=checkpointer)

# 最佳实践：组合 user_id 和 session_id
def create_thread_id(user_id: str, session_id: str) -> str:
    """创建唯一的 thread_id"""
    return f"user_{user_id}_session_{session_id}"

# 执行
config = {
    "configurable": {
        "thread_id": create_thread_id("user_001", "session_001")
    }
}
result = graph.invoke({"query": "问题"}, config=config)
```

## 状态验证与类型安全

### TypedDict 类型检查

```python
from typing_extensions import TypedDict
from typing import get_type_hints

class State(TypedDict):
    query: str
    count: int
    is_valid: bool

def validate_state(state: dict) -> bool:
    """验证状态是否符合 TypedDict 定义"""
    hints = get_type_hints(State)

    for key, expected_type in hints.items():
        if key not in state:
            print(f"Missing key: {key}")
            return False

        actual_value = state[key]
        if not isinstance(actual_value, expected_type):
            print(f"Type mismatch for {key}: expected {expected_type}, got {type(actual_value)}")
            return False

    return True

# 测试
valid_state = {"query": "test", "count": 5, "is_valid": True}
invalid_state = {"query": "test", "count": "5", "is_valid": True}  # count 类型错误

print(validate_state(valid_state))    # True
print(validate_state(invalid_state))  # False
```

### Pydantic 自动验证

```python
from pydantic import BaseModel, Field, validator

class State(BaseModel):
    """带验证的状态"""
    query: str = Field(..., min_length=1, max_length=1000)
    count: int = Field(..., ge=0, le=100)
    email: str = Field(..., regex=r"^[\w\.-]+@[\w\.-]+\.\w+$")

    @validator("query")
    def query_not_empty(cls, v):
        """自定义验证器"""
        if not v.strip():
            raise ValueError("Query cannot be empty")
        return v.strip()

# 自动验证
try:
    state = State(query="  ", count=5, email="test@example.com")
except ValueError as e:
    print(f"Validation error: {e}")
```

## 完整示例：状态设计实践

```python
"""
状态设计规范实战示例
演示：如何设计和使用状态
"""

from typing_extensions import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.runtime import Runtime

# ===== 1. 状态定义 =====

class RAGState(TypedDict):
    """RAG 系统状态"""
    # 输入
    query: str

    # 检索阶段
    documents: Annotated[list, add]  # 使用 reducer 聚合多个检索源

    # 重排序阶段
    reranked_docs: list

    # 生成阶段
    context: str
    response: str

    # 元数据
    metadata: dict

# ===== 2. 上下文定义 =====

class RAGContext(TypedDict):
    """运行时上下文（不可变）"""
    user_id: str
    session_id: str
    api_key: str
    max_docs: int

# ===== 3. 自定义 Reducer =====

def merge_metadata(a: dict, b: dict) -> dict:
    """合并元数据"""
    return {**a, **b}

class EnhancedRAGState(TypedDict):
    """增强的 RAG 状态"""
    query: str
    documents: Annotated[list, add]
    reranked_docs: list
    context: str
    response: str
    metadata: Annotated[dict, merge_metadata]  # 使用自定义 reducer

# ===== 4. 节点定义 =====

def retrieve_documents(state: EnhancedRAGState, *, runtime: Runtime[RAGContext]) -> dict:
    """
    检索文档节点
    - 访问运行时上下文
    - 返回部分状态更新
    """
    print(f"检索文档: {state['query']}")

    # 访问上下文
    max_docs = runtime.context["max_docs"]
    user_id = runtime.context["user_id"]

    # 模拟检索
    docs = [
        {"content": f"文档{i}", "score": 0.9 - i * 0.1}
        for i in range(max_docs)
    ]

    # 返回部分更新
    return {
        "documents": docs,
        "metadata": {
            "num_retrieved": len(docs),
            "user_id": user_id
        }
    }

def rerank_documents(state: EnhancedRAGState) -> dict:
    """重排序节点"""
    print("重排序文档")

    # 只访问需要的字段
    docs = state["documents"]

    # 重排序
    reranked = sorted(docs, key=lambda x: x["score"], reverse=True)[:3]

    return {
        "reranked_docs": reranked,
        "metadata": {"num_reranked": len(reranked)}
    }

def build_context(state: EnhancedRAGState) -> dict:
    """构建上下文节点"""
    print("构建上下文")

    context = "\n\n".join([
        f"文档 {i+1}: {doc['content']}"
        for i, doc in enumerate(state["reranked_docs"])
    ])

    return {
        "context": context,
        "metadata": {"context_length": len(context)}
    }

def generate_response(state: EnhancedRAGState) -> dict:
    """生成回复节点"""
    print("生成回复")

    # 模拟 LLM 调用
    response = f"基于 {len(state['reranked_docs'])} 个文档的回复"

    return {
        "response": response,
        "metadata": {"response_length": len(response)}
    }

# ===== 5. 构建图 =====

# 创建图
graph = StateGraph(
    state_schema=EnhancedRAGState,
    context_schema=RAGContext
)

# 添加节点
graph.add_node("retrieve", retrieve_documents)
graph.add_node("rerank", rerank_documents)
graph.add_node("build_context", build_context)
graph.add_node("generate", generate_response)

# 添加边
graph.add_edge(START, "retrieve")
graph.add_edge("retrieve", "rerank")
graph.add_edge("rerank", "build_context")
graph.add_edge("build_context", "generate")
graph.add_edge("generate", END)

# 编译图（带 checkpointer）
checkpointer = InMemorySaver()
app = graph.compile(checkpointer=checkpointer)

# ===== 6. 执行 =====

if __name__ == "__main__":
    # 初始状态
    initial_state = {
        "query": "什么是 LangGraph？",
        "documents": [],
        "reranked_docs": [],
        "context": "",
        "response": "",
        "metadata": {}
    }

    # 运行时上下文
    context = {
        "user_id": "user_001",
        "session_id": "session_001",
        "api_key": "sk-xxx",
        "max_docs": 5
    }

    # 配置
    config = {
        "configurable": {
            "thread_id": f"user_{context['user_id']}_session_{context['session_id']}"
        }
    }

    # 执行
    result = app.invoke(initial_state, config=config, context=context)

    print("\n=== 执行结果 ===")
    print(f"查询: {result['query']}")
    print(f"检索文档数: {len(result['documents'])}")
    print(f"重排序后文档数: {len(result['reranked_docs'])}")
    print(f"上下文长度: {len(result['context'])}")
    print(f"回复: {result['response']}")
    print(f"元数据: {result['metadata']}")
```

**运行输出**：
```
检索文档: 什么是 LangGraph？
重排序文档
构建上下文
生成回复

=== 执行结果 ===
查询: 什么是 LangGraph？
检索文档数: 5
重排序后文档数: 3
上下文长度: 45
回复: 基于 3 个文档的回复
元数据: {'num_retrieved': 5, 'user_id': 'user_001', 'num_reranked': 3, 'context_length': 45, 'response_length': 18}
```

## 状态设计反模式

### 反模式1：在状态中存储大对象

```python
# ❌ 不好：存储大对象
class BadState(TypedDict):
    query: str
    raw_file_content: bytes  # 可能很大
    model_weights: dict      # 可能很大

# ✅ 好：只存储引用
class GoodState(TypedDict):
    query: str
    file_path: str           # 存储路径
    model_id: str            # 存储 ID
```

### 反模式2：状态字段过多

```python
# ❌ 不好：字段过多
class BadState(TypedDict):
    field1: str
    field2: str
    field3: str
    # ... 50 个字段

# ✅ 好：分组管理
class GoodState(TypedDict):
    input_data: dict         # 输入相关字段
    processing_data: dict    # 处理相关字段
    output_data: dict        # 输出相关字段
```

### 反模式3：直接修改状态

```python
# ❌ 不好：直接修改状态
def bad_node(state: State) -> dict:
    state["documents"].append(new_doc)  # 直接修改
    return state

# ✅ 好：返回新的状态更新
def good_node(state: State) -> dict:
    return {"documents": state["documents"] + [new_doc]}
```

### 反模式4：缺少类型定义

```python
# ❌ 不好：使用 dict
def bad_node(state: dict) -> dict:
    return {"result": state["input"]}

# ✅ 好：使用 TypedDict
def good_node(state: State) -> dict:
    return {"result": state["input"]}
```

## 最佳实践总结

### 状态设计清单

- [ ] 是否使用了 TypedDict 或 Pydantic 定义状态？
- [ ] 状态字段命名是否清晰描述性强？
- [ ] 是否为需要聚合的字段配置了 Reducer？
- [ ] Reducer 函数是否是纯函数？
- [ ] 是否避免在状态中存储大对象？
- [ ] 是否使用 Context Schema 传递不可变参数？
- [ ] 是否使用 thread_id 实现多用户隔离？
- [ ] 节点是否返回部分状态更新而非完整状态？
- [ ] 是否避免直接修改状态对象？
- [ ] 是否有状态验证机制？

### 核心原则

1. **类型安全**：使用 TypedDict 或 Pydantic 定义状态
2. **清晰命名**：字段名称应该清晰描述其含义
3. **合理 Reducer**：为需要聚合的字段配置 Reducer
4. **部分更新**：节点只返回需要更新的字段
5. **不可变性**：不要直接修改状态对象
6. **轻量状态**：避免在状态中存储大对象
7. **上下文分离**：使用 Context Schema 传递不可变参数
8. **会话隔离**：使用 thread_id 实现多用户隔离
9. **验证机制**：实施状态验证确保数据正确性
10. **持久化友好**：状态应该易于序列化和反序列化

## 在 LangGraph 中的应用

### 与 Checkpoint 的关系

状态设计直接影响 Checkpoint 的效率：

```python
# ✅ 好的状态设计：易于序列化
class GoodState(TypedDict):
    query: str
    results: list[dict]  # 简单的数据结构

# ❌ 不好的状态设计：难以序列化
class BadState(TypedDict):
    query: str
    model: object        # 复杂对象
    connection: object   # 数据库连接
```

### 与并行执行的关系

Reducer 函数确保并行节点的状态更新正确聚合：

```python
from operator import add

class State(TypedDict):
    results: Annotated[list, add]  # 并行节点的结果会被聚合

# 节点 A 和 B 并行执行
def node_a(state: State) -> dict:
    return {"results": ["A"]}

def node_b(state: State) -> dict:
    return {"results": ["B"]}

# 最终状态: {"results": ["A", "B"]}
```

### 与人机循环的关系

状态设计影响人机循环的交互体验：

```python
from langgraph.types import interrupt

class InteractiveState(TypedDict):
    query: str
    draft_response: str
    approved: bool
    feedback: str

def review_node(state: InteractiveState) -> dict:
    """人工审核节点"""
    # 展示草稿给用户
    approved = interrupt({
        "message": "请审核回复",
        "draft": state["draft_response"]
    })

    return {"approved": approved}
```
