## 场景1：简单状态更新节点

### 1.1 基础状态更新

最简单的节点函数：接收状态，返回部分更新。

[来源: reference/context7_langgraph_01.md]

```python
"""
场景1.1：基础状态更新节点
演示最简单的节点函数实现
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# 定义状态结构
class State(TypedDict):
    """图的状态定义"""
    message: str
    count: int

def simple_node(state: State) -> dict:
    """
    简单的状态更新节点
    
    Args:
        state: 当前图状态
        
    Returns:
        dict: 部分状态更新（只返回需要更新的字段）
        
    说明：
        - 节点函数只需返回需要更新的字段
        - 不需要返回完整的状态对象
        - 返回的字段会与现有状态合并
    """
    # 读取当前状态
    current_message = state["message"]
    current_count = state["count"]
    
    # 执行业务逻辑
    new_message = f"{current_message} - processed"
    new_count = current_count + 1
    
    # 返回部分状态更新
    return {
        "message": new_message,
        "count": new_count
    }

# 构建图
builder = StateGraph(State)
builder.add_node("process", simple_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()

# 运行示例
result = graph.invoke({
    "message": "Hello",
    "count": 0
})

print(f"Result: {result}")
# 输出: {'message': 'Hello - processed', 'count': 1}
```

**关键点**：
- 节点函数签名：`def node(state: State) -> dict`
- 只返回需要更新的字段
- 返回值会自动合并到状态中

[来源: reference/source_节点函数约定_01.md]

### 1.2 部分状态更新

演示只更新部分字段的场景。

```python
"""
场景1.2：部分状态更新
演示节点函数可以只更新部分字段
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    """包含多个字段的状态"""
    user_input: str
    processed_text: str
    word_count: int
    timestamp: str

def text_processor(state: State) -> dict:
    """
    文本处理节点 - 只更新处理相关的字段
    
    Args:
        state: 当前状态
        
    Returns:
        dict: 只返回 processed_text 和 word_count
        
    说明：
        - 不需要返回 user_input 和 timestamp
        - 未返回的字段保持不变
    """
    text = state["user_input"]
    
    # 处理文本
    processed = text.strip().lower()
    words = len(processed.split())
    
    # 只返回需要更新的字段
    return {
        "processed_text": processed,
        "word_count": words
    }

def timestamp_updater(state: State) -> dict:
    """
    时间戳更新节点 - 只更新时间戳字段
    
    Args:
        state: 当前状态
        
    Returns:
        dict: 只返回 timestamp
    """
    from datetime import datetime
    
    return {
        "timestamp": datetime.now().isoformat()
    }

# 构建图
builder = StateGraph(State)
builder.add_node("process_text", text_processor)
builder.add_node("update_timestamp", timestamp_updater)

builder.add_edge(START, "process_text")
builder.add_edge("process_text", "update_timestamp")
builder.add_edge("update_timestamp", END)

graph = builder.compile()

# 运行示例
result = graph.invoke({
    "user_input": "  Hello World  ",
    "processed_text": "",
    "word_count": 0,
    "timestamp": ""
})

print(f"User Input: {result['user_input']}")
print(f"Processed: {result['processed_text']}")
print(f"Word Count: {result['word_count']}")
print(f"Timestamp: {result['timestamp']}")
```

**关键点**：
- 每个节点只更新自己负责的字段
- 未返回的字段保持原值
- 支持链式更新

[来源: reference/context7_langgraph_01.md]

## 场景2：使用 Config 参数的节点

### 2.1 访问配置信息

节点函数可以接收 `RunnableConfig` 参数来访问运行时配置。

[来源: reference/source_节点函数约定_01.md]

```python
"""
场景2.1：使用 Config 参数访问配置信息
演示如何在节点函数中访问运行时配置
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_core.runnables import RunnableConfig

class State(TypedDict):
    """状态定义"""
    message: str
    user_id: str
    run_id: str

def config_aware_node(state: State, config: RunnableConfig) -> dict:
    """
    配置感知节点 - 可以访问运行时配置
    
    Args:
        state: 当前状态
        config: 运行时配置对象
        
    Returns:
        dict: 状态更新
        
    说明：
        - config 参数提供运行时配置信息
        - 可以访问 run_id, tags, metadata 等
        - 用于日志记录、追踪、调试等场景
    """
    # 访问配置信息
    run_id = config.get("run_id", "unknown")
    tags = config.get("tags", [])
    metadata = config.get("metadata", {})
    
    # 从 configurable 中获取自定义配置
    configurable = config.get("configurable", {})
    user_id = configurable.get("user_id", "anonymous")
    
    # 记录日志
    print(f"[Run ID: {run_id}] Processing for user: {user_id}")
    print(f"[Tags: {tags}]")
    print(f"[Metadata: {metadata}]")
    
    # 更新状态
    return {
        "message": f"Processed by {user_id}",
        "user_id": user_id,
        "run_id": run_id
    }

# 构建图
builder = StateGraph(State)
builder.add_node("process", config_aware_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()

# 运行示例 - 传递配置
result = graph.invoke(
    {
        "message": "Hello",
        "user_id": "",
        "run_id": ""
    },
    config={
        "tags": ["production", "api"],
        "metadata": {"source": "web_app"},
        "configurable": {
            "user_id": "user_123"
        }
    }
)

print(f"\nResult: {result}")
```

**关键点**：
- 节点函数签名：`def node(state: State, config: RunnableConfig) -> dict`
- `config` 参数提供运行时配置信息
- 可以访问 `run_id`, `tags`, `metadata`, `configurable` 等

[来源: reference/source_节点函数约定_01.md]

### 2.2 使用 Configurable 参数

演示如何使用 `configurable` 传递自定义参数。

```python
"""
场景2.2：使用 Configurable 传递自定义参数
演示如何通过 configurable 传递业务参数
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_core.runnables import RunnableConfig

class State(TypedDict):
    """状态定义"""
    text: str
    language: str
    translated_text: str

def translator_node(state: State, config: RunnableConfig) -> dict:
    """
    翻译节点 - 从 config 中获取目标语言
    
    Args:
        state: 当前状态
        config: 运行时配置
        
    Returns:
        dict: 包含翻译结果的状态更新
        
    说明：
        - 从 config.configurable 中获取目标语言
        - 支持运行时动态配置
        - 适合需要外部参数的场景
    """
    # 从 configurable 中获取参数
    configurable = config.get("configurable", {})
    target_language = configurable.get("target_language", "en")
    
    # 模拟翻译（实际应该调用翻译 API）
    text = state["text"]
    
    # 简单的模拟翻译
    translations = {
        "en": f"[EN] {text}",
        "zh": f"[中文] {text}",
        "ja": f"[日本語] {text}"
    }
    
    translated = translations.get(target_language, text)
    
    return {
        "language": target_language,
        "translated_text": translated
    }

# 构建图
builder = StateGraph(State)
builder.add_node("translate", translator_node)
builder.add_edge(START, "translate")
builder.add_edge("translate", END)

graph = builder.compile()

# 运行示例 - 翻译成中文
result_zh = graph.invoke(
    {
        "text": "Hello World",
        "language": "",
        "translated_text": ""
    },
    config={
        "configurable": {
            "target_language": "zh"
        }
    }
)

print(f"Chinese: {result_zh['translated_text']}")

# 运行示例 - 翻译成日文
result_ja = graph.invoke(
    {
        "text": "Hello World",
        "language": "",
        "translated_text": ""
    },
    config={
        "configurable": {
            "target_language": "ja"
        }
    }
)

print(f"Japanese: {result_ja['translated_text']}")
```

**关键点**：
- 使用 `config.get("configurable", {})` 获取自定义参数
- 支持运行时动态配置
- 适合需要外部参数的场景（如 API keys, 模型参数等）

## 场景3：使用 Runtime 参数的节点

### 3.1 访问 Runtime Context

节点函数可以接收 `Runtime` 参数来访问运行时上下文。

[来源: reference/source_节点函数约定_01.md]

```python
"""
场景3.1：使用 Runtime 参数访问运行时上下文
演示如何在节点函数中访问 Runtime Context
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Runtime

class State(TypedDict):
    """状态定义"""
    message: str
    context_value: float

class MyContext(TypedDict):
    """自定义上下文类型"""
    multiplier: float
    debug: bool

def runtime_aware_node(state: State, runtime: Runtime[MyContext]) -> dict:
    """
    Runtime 感知节点 - 可以访问运行时上下文
    
    Args:
        state: 当前状态
        runtime: 运行时对象，包含 context
        
    Returns:
        dict: 状态更新
        
    说明：
        - runtime.context 提供自定义上下文数据
        - 用于跨节点共享数据
        - 不会被持久化到 checkpoint
    """
    # 访问 Runtime Context
    multiplier = runtime.context.get("multiplier", 1.0)
    debug = runtime.context.get("debug", False)
    
    if debug:
        print(f"[DEBUG] Multiplier: {multiplier}")
        print(f"[DEBUG] Current state: {state}")
    
    # 使用 context 中的值
    result = state.get("context_value", 0) * multiplier
    
    return {
        "message": f"Processed with multiplier {multiplier}",
        "context_value": result
    }

# 构建图
builder = StateGraph(State)
builder.add_node("process", runtime_aware_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()

# 运行示例 - 传递 Runtime Context
# 注意：Runtime Context 的传递方式取决于具体的执行环境
# 这里展示概念性示例
result = graph.invoke({
    "message": "Hello",
    "context_value": 10.0
})

print(f"Result: {result}")
```

**关键点**：
- 节点函数签名：`def node(state: State, runtime: Runtime[Context]) -> dict`
- `runtime.context` 提供运行时上下文数据
- 用于跨节点共享数据，不会被持久化

[来源: reference/source_节点函数约定_01.md]

### 3.2 Runtime Context 实战示例

演示 Runtime Context 在实际场景中的使用。

```python
"""
场景3.2：Runtime Context 实战示例
演示如何使用 Runtime Context 共享配置和资源
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import Runtime

class State(TypedDict):
    """状态定义"""
    query: str
    result: str
    api_calls: int

class AppContext(TypedDict):
    """应用上下文"""
    api_key: str
    max_retries: int
    timeout: float

def api_caller_node(state: State, runtime: Runtime[AppContext]) -> dict:
    """
    API 调用节点 - 使用 Runtime Context 中的配置
    
    Args:
        state: 当前状态
        runtime: 运行时对象
        
    Returns:
        dict: 状态更新
        
    说明：
        - 从 runtime.context 获取 API 配置
        - 避免在状态中传递敏感信息（如 API key）
        - 支持动态配置
    """
    # 从 Runtime Context 获取配置
    api_key = runtime.context.get("api_key", "")
    max_retries = runtime.context.get("max_retries", 3)
    timeout = runtime.context.get("timeout", 30.0)
    
    query = state["query"]
    
    # 模拟 API 调用
    print(f"Calling API with key: {api_key[:8]}...")
    print(f"Max retries: {max_retries}, Timeout: {timeout}s")
    
    # 模拟结果
    result = f"API result for: {query}"
    
    return {
        "result": result,
        "api_calls": state.get("api_calls", 0) + 1
    }

def result_processor_node(state: State, runtime: Runtime[AppContext]) -> dict:
    """
    结果处理节点 - 也可以访问相同的 Runtime Context
    
    Args:
        state: 当前状态
        runtime: 运行时对象
        
    Returns:
        dict: 状态更新
    """
    # 访问相同的 Runtime Context
    api_key = runtime.context.get("api_key", "")
    
    result = state["result"]
    processed = f"[Processed with key {api_key[:8]}...] {result}"
    
    return {
        "result": processed
    }

# 构建图
builder = StateGraph(State)
builder.add_node("call_api", api_caller_node)
builder.add_node("process_result", result_processor_node)

builder.add_edge(START, "call_api")
builder.add_edge("call_api", "process_result")
builder.add_edge("process_result", END)

graph = builder.compile()

# 运行示例
# 注意：实际使用时需要通过特定方式传递 Runtime Context
result = graph.invoke({
    "query": "What is LangGraph?",
    "result": "",
    "api_calls": 0
})

print(f"\nFinal Result: {result['result']}")
print(f"Total API Calls: {result['api_calls']}")
```

**关键点**：
- Runtime Context 用于共享配置和资源
- 避免在状态中传递敏感信息
- 所有节点可以访问相同的 Runtime Context

### 综合示例：文档处理工作流

结合所有场景的完整示例。

```python
"""
综合示例：文档处理工作流
演示节点函数的各种参数使用方式
"""

from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END
from langchain_core.runnables import RunnableConfig
import operator

class State(TypedDict):
    """文档处理状态"""
    document: str
    word_count: int
    processed: bool
    errors: Annotated[list[str], operator.add]  # 使用 Reducer
    metadata: dict

def load_document(state: State) -> dict:
    """
    场景1：简单状态更新
    加载文档内容
    """
    # 模拟加载文档
    document = "This is a sample document for processing."
    
    return {
        "document": document,
        "processed": False
    }

def count_words(state: State, config: RunnableConfig) -> dict:
    """
    场景2：使用 Config 参数
    统计词数，并记录日志
    """
    # 从 config 获取运行信息
    run_id = config.get("run_id", "unknown")
    configurable = config.get("configurable", {})
    verbose = configurable.get("verbose", False)
    
    # 统计词数
    document = state["document"]
    word_count = len(document.split())
    
    if verbose:
        print(f"[Run {run_id}] Word count: {word_count}")
    
    return {
        "word_count": word_count,
        "metadata": {
            "run_id": run_id,
            "word_count": word_count
        }
    }

def validate_document(state: State, config: RunnableConfig) -> dict:
    """
    场景2：使用 Config 参数
    验证文档，记录错误
    """
    configurable = config.get("configurable", {})
    min_words = configurable.get("min_words", 5)
    
    word_count = state["word_count"]
    errors = []
    
    if word_count < min_words:
        errors.append(f"Document too short: {word_count} < {min_words}")
    
    return {
        "errors": errors,
        "processed": len(errors) == 0
    }

# 构建图
builder = StateGraph(State)

# 添加节点
builder.add_node("load", load_document)
builder.add_node("count", count_words)
builder.add_node("validate", validate_document)

# 添加边
builder.add_edge(START, "load")
builder.add_edge("load", "count")
builder.add_edge("count", "validate")
builder.add_edge("validate", END)

# 编译图
graph = builder.compile()

# 运行示例1：正常文档
print("=== Example 1: Normal Document ===")
result1 = graph.invoke(
    {
        "document": "",
        "word_count": 0,
        "processed": False,
        "errors": [],
        "metadata": {}
    },
    config={
        "configurable": {
            "verbose": True,
            "min_words": 5
        }
    }
)

print(f"Processed: {result1['processed']}")
print(f"Word Count: {result1['word_count']}")
print(f"Errors: {result1['errors']}")
print(f"Metadata: {result1['metadata']}")

# 运行示例2：文档太短
print("\n=== Example 2: Short Document ===")
result2 = graph.invoke(
    {
        "document": "",
        "word_count": 0,
        "processed": False,
        "errors": [],
        "metadata": {}
    },
    config={
        "configurable": {
            "verbose": True,
            "min_words": 20  # 设置更高的最小词数
        }
    }
)

print(f"Processed: {result2['processed']}")
print(f"Word Count: {result2['word_count']}")
print(f"Errors: {result2['errors']}")
```

**关键点**：
- 结合了简单状态更新、Config 参数、Reducer 等特性
- 展示了完整的工作流构建过程
- 演示了如何通过 config 传递参数

## 总结

本文档展示了 LangGraph 节点函数的三种基础实现模式：

1. **简单状态更新**：`def node(state: State) -> dict`
   - 最基础的节点函数形式
   - 只返回需要更新的字段
   - 适合纯业务逻辑处理

2. **使用 Config 参数**：`def node(state: State, config: RunnableConfig) -> dict`
   - 访问运行时配置信息
   - 获取 run_id, tags, metadata
   - 通过 configurable 传递自定义参数

3. **使用 Runtime 参数**：`def node(state: State, runtime: Runtime[Context]) -> dict`
   - 访问运行时上下文
   - 跨节点共享数据和资源
   - 避免在状态中传递敏感信息

**最佳实践**：
- 优先使用简单状态更新，保持节点函数纯净
- 需要配置信息时使用 Config 参数
- 需要共享资源时使用 Runtime 参数
- 使用类型注解提高代码可读性
