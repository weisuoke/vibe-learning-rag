# 实战代码 场景2：异构集合与工厂模式

> **目标**：深入掌握 `Vec<Box<dyn Trait>>` 异构集合和返回 `Box<dyn Trait>` 的工厂函数模式
> **运行**：`cargo run`（纯标准库，无外部依赖）

---

## 场景说明

在 UI 框架、插件系统、规则引擎等场景中，你经常需要：

1. **把不同类型放进同一个集合** — 异构集合
2. **根据配置字符串在运行时创建不同类型** — 工厂模式
3. **对集合统一遍历、过滤、操作** — 不关心具体类型

这正是 `Vec<Box<dyn Trait>>` + 工厂函数的核心用途。

```
┌─ Vec<Box<dyn Widget>> ──────────────────────────┐
│                                                   │
│  [0] Box<dyn Widget> → Button { "Submit", 120 }   │
│  [1] Box<dyn Widget> → TextInput { "Name", 200 }  │
│  [2] Box<dyn Widget> → Checkbox { true, "Agree" } │
│                                                   │
│  三种不同类型，统一为 16 字节胖指针               │
└───────────────────────────────────────────────────┘
```

## 完整代码

```rust
// ============================================================
// 异构集合与工厂模式演示
// ============================================================

use std::fmt;

// ============================================================
// 第一部分：定义 Widget Trait
// ============================================================

/// Widget Trait — UI 组件的行为契约
/// Send + Sync：确保可在线程间传递（对象安全的常见约束）
trait Widget: Send + Sync {
    /// 组件类型名称
    fn name(&self) -> &str;

    /// 渲染为字符串（模拟 HTML 输出）
    fn render(&self) -> String;

    /// 组件宽度（像素）
    fn width(&self) -> u32;
}

// 为所有 Widget 添加 Debug 风格的打印
impl fmt::Display for dyn Widget {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "<{} width={}>{}</{}>",
            self.name(), self.width(), self.render(), self.name())
    }
}

// ============================================================
// 第二部分：实现三种不同的 Widget
// ============================================================

/// 按钮组件
struct Button {
    label: String,
    width: u32,
}

impl Widget for Button {
    fn name(&self) -> &str { "Button" }
    fn render(&self) -> String {
        format!("[🔘 {}]", self.label)
    }
    fn width(&self) -> u32 { self.width }
}

/// 文本输入框
struct TextInput {
    placeholder: String,
    width: u32,
}

impl Widget for TextInput {
    fn name(&self) -> &str { "TextInput" }
    fn render(&self) -> String {
        format!("[📝 ___{}___]", self.placeholder)
    }
    fn width(&self) -> u32 { self.width }
}

/// 复选框
struct Checkbox {
    checked: bool,
    label: String,
}

impl Widget for Checkbox {
    fn name(&self) -> &str { "Checkbox" }
    fn render(&self) -> String {
        let icon = if self.checked { "☑" } else { "☐" };
        format!("[{} {}]", icon, self.label)
    }
    fn width(&self) -> u32 { 80 }  // 固定宽度
}

/// 分隔线（零字段类型，仍可放进异构集合！）
struct Divider;

impl Widget for Divider {
    fn name(&self) -> &str { "Divider" }
    fn render(&self) -> String { "────────────".to_string() }
    fn width(&self) -> u32 { 0 }  // 占满整行
}

// ============================================================
// 第三部分：异构集合 — 三种不同类型统一存储
// ============================================================

/// 构建一个包含多种 Widget 的异构集合
fn build_form() -> Vec<Box<dyn Widget>> {
    vec![
        // 四种完全不同的类型，统一为 Box<dyn Widget>
        Box::new(TextInput {
            placeholder: "Your name...".into(),
            width: 200,
        }),
        Box::new(TextInput {
            placeholder: "Your email...".into(),
            width: 200,
        }),
        Box::new(Checkbox {
            checked: false,
            label: "I agree to terms".into(),
        }),
        Box::new(Divider),
        Box::new(Button {
            label: "Submit".into(),
            width: 120,
        }),
        Box::new(Button {
            label: "Cancel".into(),
            width: 100,
        }),
    ]
}

// ============================================================
// 第四部分：遍历异构集合 — for / map / filter
// ============================================================

/// 渲染所有 Widget
fn render_all(widgets: &[Box<dyn Widget>]) {
    println!("  ┌─ Form ──────────────────────────┐");
    for (i, widget) in widgets.iter().enumerate() {
        // widget: &Box<dyn Widget>
        // widget.render() → 通过 vtable 动态分发到具体类型的 render()
        println!("  │ [{}] {} → {}", i, widget.name(), widget.render());
    }
    println!("  └─────────────────────────────────┘");
}

/// 筛选宽度大于指定值的 Widget
fn filter_wide(widgets: &[Box<dyn Widget>], min_width: u32) -> Vec<&dyn Widget> {
    widgets.iter()
        .filter(|w| w.width() > min_width)
        .map(|w| w.as_ref())  // Box<dyn Widget> → &dyn Widget
        .collect()
}

/// 计算所有 Widget 的总宽度
fn total_width(widgets: &[Box<dyn Widget>]) -> u32 {
    widgets.iter()
        .map(|w| w.width())  // 每个调用都是动态分发
        .sum()
}

// ============================================================
// 第五部分：工厂函数 — 从配置创建 Widget
// ============================================================

/// Widget 配置（模拟从 JSON/TOML 解析的数据）
struct WidgetConfig {
    widget_type: String,
    label: String,
    width: Option<u32>,
    checked: Option<bool>,
}

/// 单个 Widget 工厂函数
/// 返回 Result<Box<dyn Widget>, String> — 运行时决定具体类型
fn create_widget(config: &WidgetConfig) -> Result<Box<dyn Widget>, String> {
    match config.widget_type.as_str() {
        "button" => Ok(Box::new(Button {
            label: config.label.clone(),
            width: config.width.unwrap_or(100),
        })),
        "input" => Ok(Box::new(TextInput {
            placeholder: config.label.clone(),
            width: config.width.unwrap_or(150),
        })),
        "checkbox" => Ok(Box::new(Checkbox {
            checked: config.checked.unwrap_or(false),
            label: config.label.clone(),
        })),
        "divider" => Ok(Box::new(Divider)),
        unknown => Err(format!("Unknown widget type: '{}'", unknown)),
    }
    // 每个分支返回不同的具体类型，但统一为 Box<dyn Widget>
    // 编译器在这里做类型擦除：具体类型信息存入 vtable
}

/// 批量创建 — 从配置数组构建 UI
fn create_ui(configs: &[WidgetConfig]) -> Vec<Box<dyn Widget>> {
    configs.iter()
        .filter_map(|config| {
            match create_widget(config) {
                Ok(widget) => Some(widget),
                Err(e) => {
                    eprintln!("  ⚠ Skipping: {}", e);
                    None
                }
            }
        })
        .collect()
}

// ============================================================
// 第六部分：ZeroClaw 的工厂模式对照
// ============================================================

// 以下代码模拟 ZeroClaw 的 create_memory_with_builders 模式
// 真实代码见：src/memory/mod.rs

/// Memory Trait（简化版）
trait Memory: Send + Sync {
    fn name(&self) -> &str;
    fn store(&self, key: &str, value: &str);
    fn recall(&self, query: &str) -> Vec<String>;
}

/// 三种 Memory 实现
struct SqliteMemory { db_path: String }
struct MarkdownMemory { dir: String }
struct NoneMemory;

impl Memory for SqliteMemory {
    fn name(&self) -> &str { "sqlite" }
    fn store(&self, key: &str, value: &str) {
        println!("    [SQLite:{}] INSERT '{}' → '{}'", self.db_path, key, value);
    }
    fn recall(&self, query: &str) -> Vec<String> {
        vec![format!("[SQLite] Result for '{}'", query)]
    }
}

impl Memory for MarkdownMemory {
    fn name(&self) -> &str { "markdown" }
    fn store(&self, key: &str, value: &str) {
        println!("    [Markdown:{}] Write '{}.md' → '{}'", self.dir, key, value);
    }
    fn recall(&self, query: &str) -> Vec<String> {
        vec![format!("[Markdown] Result for '{}'", query)]
    }
}

impl Memory for NoneMemory {
    fn name(&self) -> &str { "none" }
    fn store(&self, _key: &str, _value: &str) {
        println!("    [NoneMemory] Discarded (no-op)");
    }
    fn recall(&self, _query: &str) -> Vec<String> { vec![] }
}

/// Memory 后端分类
enum MemoryBackendKind { Sqlite, Markdown, None }

fn classify_memory_backend(name: &str) -> MemoryBackendKind {
    match name {
        "sqlite" => MemoryBackendKind::Sqlite,
        "markdown" => MemoryBackendKind::Markdown,
        _ => MemoryBackendKind::None,
    }
}

/// ZeroClaw 风格的工厂函数
/// 输入：config 字符串 → 输出：Box<dyn Memory>（类型擦除）
fn create_memory(backend_name: &str, workspace: &str) -> Box<dyn Memory> {
    match classify_memory_backend(backend_name) {
        MemoryBackendKind::Sqlite => Box::new(SqliteMemory {
            db_path: format!("{}/memory.db", workspace),
        }),
        MemoryBackendKind::Markdown => Box::new(MarkdownMemory {
            dir: format!("{}/memories", workspace),
        }),
        MemoryBackendKind::None => Box::new(NoneMemory),
    }
}

// ============================================================
// 第七部分：动态添加和移除
// ============================================================

/// 演示运行时动态修改异构集合
fn dynamic_modify_demo(widgets: &mut Vec<Box<dyn Widget>>) {
    println!("\n  --- 动态添加 ---");

    // push：运行时添加新组件
    widgets.push(Box::new(Button {
        label: "New Button".into(),
        width: 110,
    }));
    println!("  ✅ push: 添加了 'New Button'，共 {} 个", widgets.len());

    // retain：移除所有宽度为 0 的组件（如 Divider）
    let before = widgets.len();
    widgets.retain(|w| w.width() > 0);
    println!("  ✅ retain: 移除宽度=0 的组件，{} → {} 个", before, widgets.len());

    // 可以在任意位置插入
    widgets.insert(0, Box::new(TextInput {
        placeholder: "Search...".into(),
        width: 250,
    }));
    println!("  ✅ insert(0): 在开头插入 'Search...'，共 {} 个", widgets.len());
}

// ============================================================
// 第八部分：main — 把所有部分串起来
// ============================================================

fn main() {
    println!("==============================================");
    println!("  异构集合与工厂模式演示");
    println!("==============================================\n");

    // ---- 1. 手动构建异构集合 ----
    println!("=== 1. 手动构建异构集合 ===");
    let widgets = build_form();
    render_all(&widgets);

    // ---- 2. 遍历、过滤、聚合 ----
    println!("\n=== 2. 集合操作 ===");
    let wide = filter_wide(&widgets, 100);
    println!("  宽度 > 100px 的组件:");
    for w in &wide {
        println!("    - {} ({}px)", w.name(), w.width());
    }
    println!("  所有组件总宽度: {}px", total_width(&widgets));

    // ---- 3. 工厂模式：从配置创建 ----
    println!("\n=== 3. 工厂模式 ===");
    let configs = vec![
        WidgetConfig { widget_type: "input".into(), label: "Username".into(), width: Some(180), checked: None },
        WidgetConfig { widget_type: "input".into(), label: "Password".into(), width: Some(180), checked: None },
        WidgetConfig { widget_type: "checkbox".into(), label: "Remember me".into(), width: None, checked: Some(true) },
        WidgetConfig { widget_type: "slider".into(), label: "Volume".into(), width: None, checked: None },  // ← 未知类型！
        WidgetConfig { widget_type: "divider".into(), label: "".into(), width: None, checked: None },
        WidgetConfig { widget_type: "button".into(), label: "Login".into(), width: Some(120), checked: None },
    ];

    let ui = create_ui(&configs);
    println!("  从 {} 个配置创建了 {} 个 Widget:", configs.len(), ui.len());
    render_all(&ui);

    // ---- 4. ZeroClaw Memory 工厂 ----
    println!("\n=== 4. ZeroClaw Memory 工厂 ===");
    let backends = ["sqlite", "markdown", "none", "redis"];
    for backend in &backends {
        let memory = create_memory(backend, "/workspace");
        println!("  Backend '{}' → Memory.name() = '{}'", backend, memory.name());
        memory.store("today", "learned trait objects");
        let results = memory.recall("trait");
        if results.is_empty() {
            println!("    recall → (empty)");
        } else {
            for r in &results {
                println!("    recall → {}", r);
            }
        }
    }

    // ---- 5. 动态添加和移除 ----
    println!("\n=== 5. 动态修改集合 ===");
    let mut widgets = build_form();
    println!("  初始: {} 个 Widget", widgets.len());
    dynamic_modify_demo(&mut widgets);
    render_all(&widgets);

    // ---- 6. 内存布局 ----
    println!("\n=== 6. 内存布局 ===");
    println!("  Box<dyn Widget>:    {} bytes (胖指针: data + vtable)",
        std::mem::size_of::<Box<dyn Widget>>());
    println!("  Box<Button>:        {} bytes (瘦指针: data only)",
        std::mem::size_of::<Box<Button>>());
    println!("  Box<dyn Memory>:    {} bytes (胖指针)",
        std::mem::size_of::<Box<dyn Memory>>());
    println!("  Vec<Box<dyn Widget>>: {} bytes (Vec 本身: ptr+len+cap)",
        std::mem::size_of::<Vec<Box<dyn Widget>>>());
}
```

## 代码解析：8 个关键点

| # | 关键点 | 代码位置 | 说明 |
|---|--------|---------|------|
| 1 | **Trait 定义行为契约** | `trait Widget` | `Send + Sync` + `&self` → 对象安全 |
| 2 | **异构集合统一存储** | `build_form()` | 4 种不同类型 → `Vec<Box<dyn Widget>>` |
| 3 | **iter() 统一遍历** | `render_all()` | 不知道具体类型，通过 vtable 调用正确方法 |
| 4 | **filter() 统一筛选** | `filter_wide()` | `w.width()` 是动态分发，对调用者透明 |
| 5 | **工厂函数** | `create_widget()` | match 分支返回不同类型 → 统一为 `Box<dyn Widget>` |
| 6 | **批量创建** | `create_ui()` | `filter_map` 跳过未知类型，收集成功的 Widget |
| 7 | **ZeroClaw 风格** | `create_memory()` | 与真实的 `create_memory_with_builders` 同模式 |
| 8 | **动态增删** | `push/retain/insert` | 运行时修改集合，Agent 零修改 |

## 与 TypeScript 对照

```typescript
// TypeScript 版本 — 异构集合 + 工厂模式

interface Widget {
  name(): string;
  render(): string;
  width(): number;
}

class Button implements Widget {
  constructor(private label: string, private w: number) {}
  name() { return "Button"; }
  render() { return `[🔘 ${this.label}]`; }
  width() { return this.w; }
}

class TextInput implements Widget {
  constructor(private placeholder: string, private w: number) {}
  name() { return "TextInput"; }
  render() { return `[📝 ${this.placeholder}]`; }
  width() { return this.w; }
}

class Checkbox implements Widget {
  constructor(private checked: boolean, private label: string) {}
  name() { return "Checkbox"; }
  render() { return `[${this.checked ? "☑" : "☐"} ${this.label}]`; }
  width() { return 80; }
}

// ---- 异构集合 ----
// TypeScript：直接用 Widget[]，不需要 Box！
const widgets: Widget[] = [
  new Button("Submit", 120),
  new TextInput("Name...", 200),
  new Checkbox(true, "Agree"),
];

// ---- 遍历 + 过滤 ----
widgets.forEach(w => console.log(`${w.name()}: ${w.render()}`));
const wide = widgets.filter(w => w.width() > 100);

// ---- 工厂函数 ----
// TypeScript：直接返回 Widget，不需要 Box！
function createWidget(config: WidgetConfig): Widget {
  switch (config.type) {
    case "button": return new Button(config.label, config.width ?? 100);
    case "input": return new TextInput(config.label, config.width ?? 150);
    case "checkbox": return new Checkbox(config.checked ?? false, config.label);
    default: throw new Error(`Unknown: ${config.type}`);
  }
}

// ---- 动态添加 ----
widgets.push(new Button("New", 110));       // 直接 push，同 Rust
widgets.splice(2, 1);                        // 移除第 3 个
```

### 关键差异对照表

| 概念 | Rust | TypeScript | 为什么不同 |
|------|------|------------|-----------|
| 异构集合 | `Vec<Box<dyn Widget>>` | `Widget[]` | TS 有 GC，自动堆分配 |
| 工厂返回 | `Box<dyn Widget>` | `Widget` | TS 接口 = 默认动态分发 |
| 筛选 | `.filter(\|w\| w.width() > 100)` | `.filter(w => w.width() > 100)` | 语法不同，语义相同 |
| 动态添加 | `widgets.push(Box::new(...))` | `widgets.push(new ...)` | Rust 需显式装箱 |
| 动态移除 | `widgets.retain(\|w\| ...)` | `widgets.filter(w => ...)` | Rust 原地修改 vs TS 创建新数组 |
| 错误处理 | `Result<Box<dyn Widget>, String>` | `throw new Error(...)` | Rust 用返回值，TS 用异常 |

> **一句话对比**：TypeScript 中 `Widget[]` 天然就是异构集合，因为所有对象都在堆上。Rust 需要 `Vec<Box<dyn Widget>>` 显式装箱，但好处是你**完全控制**内存布局和分发方式。
