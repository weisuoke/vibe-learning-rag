# 动态图构建 - 一句话总结

> 用一句话总结 LangGraph 动态图构建的核心本质

---

## 一句话总结

**动态图构建是 LangGraph 通过条件边、Send API 和 Command API 三大机制，让工作流在运行时根据状态、数据和 AI 决策自主选择执行路径的能力，是构建智能代理系统的核心基础。**

---

## 展开解读

### 关键词拆解

**条件边（Conditional Edges）—— 运行时路径选择**
通过 `add_conditional_edges(source, path_fn, path_map)` 注册路由函数，在运行时根据当前状态返回目标节点名。这是最常用的动态路由方式，覆盖 80% 的分支场景。路由函数是纯函数，接收 state，返回字符串，简单直接。

**Send API —— 运行时动态并行**
`Send(node, arg)` 在运行时创建任意数量的并行任务，每个 Send 携带独立状态。任务数量完全由运行时数据决定，编译时未知。配合 `Annotated[list, operator.add]` reducer 合并并行结果，实现完整的 map-reduce 模式。

**Command API —— 状态更新 + 路由控制一体化**
节点返回 `Command(update=..., goto=..., graph=...)` 在一次操作中同时更新状态、指定下一个节点、甚至跨图导航。`graph=Command.PARENT` 是子图跳回父图的唯一通道，是多代理协作系统的关键机制。

**运行时自主选择执行路径**
"动态"的核心含义：图结构在 `compile()` 后不可变，但运行时根据状态数据选择走哪条路径。编译时定义所有可能性，运行时选择实际路径——这是 LangGraph 动态图的根本设计哲学。

### 记忆口诀

```
条件边选路走，
Send 开并行，
Command 管全局。
编译定可能，运行选路径。
```

---

## 三大机制速记

### 核心公式

```
动态图 = 条件边(固定分支) + Send(动态并行) + Command(自主导航)
```

### 一张表记住三种机制

| 机制 | 一句话 | 典型场景 | API |
|------|--------|----------|-----|
| 条件边 | 根据状态选路径 | if/else 分支、意图路由 | `add_conditional_edges(src, fn)` |
| Send | 运行时创建并行任务 | 批量处理、map-reduce | `Send(node, arg)` |
| Command | 改状态 + 选路径一步到位 | 多代理切换、复杂决策 | `Command(update=, goto=)` |

### 选型速查

```
简单分支 → 条件边
动态并行 → Send
状态+路由 → Command
跨图导航 → Command + PARENT
AI 决策  → LLM + 结构化输出 + 条件边
```

## 核心代码速查

### 条件边

```python
from langgraph.graph import StateGraph, START, END

# 基础用法
def route(state):
    if state["intent"] == "search":
        return "search_node"
    return "chat_node"

builder.add_conditional_edges("classifier", route)

# 带 path_map
builder.add_conditional_edges("classifier", route, {
    "search": "search_node",
    "chat": "chat_node",
})
```

### Send API

```python
from langgraph.types import Send
import operator
from typing import Annotated

class State(TypedDict):
    items: list[str]
    results: Annotated[list[str], operator.add]  # reducer 必须有

def fan_out(state):
    return [Send("worker", {"item": x}) for x in state["items"]]

builder.add_conditional_edges("splitter", fan_out)
```

### Command API

```python
from langgraph.types import Command
from typing import Literal

# 基础用法
def node(state) -> Command[Literal["approve", "reject"]]:
    if state["score"] > 0.8:
        return Command(update={"status": "pass"}, goto="approve")
    return Command(update={"status": "fail"}, goto="reject")

# 跨图导航
def subgraph_node(state) -> Command[Literal["other_agent"]]:
    return Command(goto="other_agent", graph=Command.PARENT)
```

### LLM 路由

```python
from pydantic import BaseModel

class Route(BaseModel):
    next_step: Literal["search", "calculate", "respond"]

router = llm.with_structured_output(Route)

def llm_router(state):
    decision = router.invoke(state["messages"])
    return decision.next_step

builder.add_conditional_edges("planner", llm_router)
```

### 图构建三步曲

```python
# 1. 创建
builder = StateGraph(State)

# 2. 组装
builder.add_node("name", function)
builder.add_edge(START, "name")
builder.add_conditional_edges("name", route_fn)

# 3. 编译
graph = builder.compile()
result = graph.invoke(initial_state)
```

## 学习检查清单

### 基础能力（必须掌握）

- [ ] 能用一句话解释"动态图"的"动态"指什么
- [ ] 能写出 `add_conditional_edges` 的基本用法
- [ ] 能写出路由函数（接收 state，返回节点名）
- [ ] 理解 `compile()` 后图结构不可变，但路径是动态的
- [ ] 能用 `StateGraph` 三步曲构建完整的图

### 进阶能力（建议掌握）

- [ ] 能用 `Send` 实现 map-reduce 并行模式
- [ ] 能用 `Command` 同时更新状态和控制路由
- [ ] 理解 `Annotated[list, operator.add]` reducer 的作用
- [ ] 能结合 LLM 结构化输出实现智能路由
- [ ] 知道 `path_map` 参数的用途

### 高级能力（可选掌握）

- [ ] 能用 `Command(graph=Command.PARENT)` 实现跨图导航
- [ ] 理解 Pregel 执行引擎的 BSP 模型
- [ ] 能组合多种动态机制构建复杂工作流
- [ ] 知道 Command 的 goto 和静态边同时存在的陷阱
- [ ] 能设计生产级的动态路由系统

### 三个必须记住的规则

1. **图结构编译后不可变** —— "动态"是选路径，不是改图
2. **Command 别配静态边** —— 两者会同时执行
3. **Send 必须配 Reducer** —— 否则并行结果互相覆盖

## 快速参考卡

### 核心 API 一览

```python
# ===== 导入 =====
from langgraph.graph import StateGraph, START, END
from langgraph.types import Send, Command
from typing import Literal, Annotated
import operator

# ===== 条件边 =====
builder.add_conditional_edges("source", route_fn)
builder.add_conditional_edges("source", route_fn, {"key": "node"})

# ===== Send =====
Send("target_node", {"key": "value"})
# 在条件边路由函数中返回 [Send(...), Send(...)]

# ===== Command =====
Command(update={"k": "v"}, goto="next")
Command(goto="other", graph=Command.PARENT)
Command(goto=[Send("w", {"t": x}) for x in items])

# ===== 图构建 =====
builder = StateGraph(State)
builder.add_node("name", func)
builder.add_edge(START, "name")
builder.add_edge("name", END)
graph = builder.compile()
result = graph.invoke(state)

# ===== Reducer =====
class State(TypedDict):
    results: Annotated[list, operator.add]
```

### 三种机制的返回值

```python
# 条件边路由函数返回值
def route(state) -> str:           # 单路：返回节点名
def route(state) -> list[str]:     # 多路：返回节点名列表
def route(state) -> list[Send]:    # Send：返回 Send 列表

# Command 返回值
def node(state) -> Command[Literal["a", "b"]]:
    return Command(update={...}, goto="a")
```
