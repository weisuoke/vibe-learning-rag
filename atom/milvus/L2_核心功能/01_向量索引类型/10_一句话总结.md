# 一句话总结

**向量索引类型是组织和检索向量的不同算法（FLAT、IVF、HNSW、GPU_CAGRA、RaBitQ等），通过权衡准确率、速度、内存实现十亿级向量高效检索，是Milvus 2.6构建高性能RAG系统的核心基础。**

- **应用场景**：RAG文档检索、图像搜索、推荐系统

- **可扩展性**：支持百亿级向量检索
- **高性能**：毫秒级响应（HNSW ~5ms，GPU_CAGRA ~1ms）

### 实际价值

## 3个核心理解

### 理解1：索引是性能与成本的权衡艺术
- **没有完美索引**：只有最适合的索引
- **三要素博弈**：准确率 ↔ 速度 ↔ 内存
- **场景决定选择**：RAG系统用HNSW，推荐系统用IVF_PQ，图像搜索用GPU_CAGRA

### 理解2：索引类型决定系统规模上限
- **FLAT**：适合 < 1M 向量（原型验证）
- **HNSW**：适合 1M-100M 向量（生产标准）
- **GPU_CAGRA + RaBitQ**：适合 100M-10B 向量（超大规模）

### 理解3：2026年索引选型新趋势
- **混合索引**：GPU_CAGRA（构建）+ CPU（查询）= 性能 + 成本
- **极致压缩**：RaBitQ 1-bit量化 = 72%内存节省 + <2%准确率损失
- **生产标准**：HNSW成为RAG系统默认选择

## 对比记忆

### 传统数据库索引 vs 向量索引

| 维度 | 传统索引 | 向量索引 |
|------|---------|----------|
| **数据类型** | 标量 | 高维向量 |
| **比较方式** | 精确匹配 | 相似度计算 |
| **索引结构** | B树、哈希 | 聚类、图、量化 |
| **查询目标** | 找精确值 | 找最相似k个 |
| **性能瓶颈** | 磁盘I/O | 距离计算 |
| **典型算法** | B+树 | FLAT、IVF、HNSW、CAGRA |

## 核心API快速参考

### 创建索引
```python
# FLAT索引
index_params = {"index_type": "FLAT", "metric_type": "L2"}

# HNSW索引（生产推荐）
index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {"M": 16, "efConstruction": 200}
}

# GPU_CAGRA索引（高性能）
index_params = {
    "index_type": "GPU_CAGRA",
    "metric_type": "L2",
    "params": {"intermediate_graph_degree": 128}
}

# RaBitQ索引（成本优化）
index_params = {
    "index_type": "IVF_RABITQ",
    "metric_type": "L2",
    "params": {"nlist": 1024}
}
```

### 搜索参数
```python
# HNSW搜索
search_params = {"metric_type": "L2", "params": {"ef": 100}}

# IVF搜索
search_params = {"metric_type": "L2", "params": {"nprobe": 16}}

# GPU_CAGRA搜索
search_params = {"metric_type": "L2", "params": {"itopk_size": 128}}
```

## 5个核心操作

### 1. 创建索引
```python
collection.create_index(field_name="embeddings", index_params=index_params)
```

### 2. 加载索引
```python
collection.load()
```

### 3. 搜索向量
```python
results = collection.search(data=query_vectors, anns_field="embeddings", param=search_params, limit=10)
```

### 4. 查看索引信息
```python
index_info = collection.index()
```

### 5. 删除索引
```python
collection.drop_index()
```
