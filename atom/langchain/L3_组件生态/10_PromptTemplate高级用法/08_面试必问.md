## 问题1："LangChain 的 PromptTemplate 支持哪些高级用法？"

### 普通回答（❌ 不出彩）

"PromptTemplate 支持变量替换、模板组合、从文件加载等功能。"

### 出彩回答（✅ 推荐）

> **PromptTemplate 的高级用法主要包括三个层面：**
>
> **1. Partial Variables（部分变量预填充）**
> - 支持字符串固定值：预设常量如系统设置、日期
> - 支持函数动态值：运行时计算如 `get_current_date()`
> - 减少重复传递相同变量，提升代码复用性
>
> **2. Template Composition（模板组合）**
> - 使用 `+` 操作符组合多个模板
> - 自动合并 `input_variables` 和 `partial_variables`
> - 支持 PromptTemplate + str 的灵活组合
> - 要求两个模板的 `template_format` 必须一致
>
> **3. 多种模板格式**
> - f-string（默认）：简单快速，适合基础场景
> - mustache：支持嵌套变量和 section，适合复杂结构
> - jinja2：功能强大但有安全风险，需谨慎使用
>
> **在实际工作中的应用**：
> - RAG 系统：使用 partial variables 预填充系统提示词和日期
> - Agent 系统：使用模板组合构建多步推理提示词
> - 多语言场景：使用 mustache 模板支持嵌套变量

**代码示例**：

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 1. Partial Variables - 函数动态值
def get_current_date():
    return datetime.now().strftime("%Y-%m-%d")

prompt = PromptTemplate(
    template="Today is {date}. {task}",
    input_variables=["task"],
    partial_variables={"date": get_current_date}
)

# 2. Template Composition
system_prompt = PromptTemplate.from_template("You are a {role}.")
user_prompt = PromptTemplate.from_template("Question: {question}")
combined = system_prompt + user_prompt

print(combined.format(role="assistant", question="What is AI?"))
```

## 问题2："Partial Variables 的两种方式有什么区别？什么时候用哪种？"

### 普通回答（❌ 不出彩）

"一种是字符串，一种是函数。字符串用于固定值，函数用于动态值。"

### 出彩回答（✅ 推荐）

> **Partial Variables 的两种方式本质区别在于求值时机：**
>
> **1. 字符串固定值（立即求值）**
> - 在模板创建时就确定值
> - 适用场景：系统设置、API 密钥、固定提示词
> - 优点：性能好，无额外计算开销
> - 缺点：无法动态更新
>
> ```python
> prompt = PromptTemplate(
>     template="{system_msg} {query}",
>     input_variables=["query"],
>     partial_variables={"system_msg": "You are a helpful assistant."}
> )
> ```
>
> **2. 函数动态值（延迟求值）**
> - 在模板调用时才计算值
> - 适用场景：当前时间、随机数、上下文相关的值
> - 优点：每次调用都是最新值
> - 缺点：有函数调用开销
>
> ```python
> from datetime import datetime
>
> def get_timestamp():
>     return datetime.now().isoformat()
>
> prompt = PromptTemplate(
>     template="[{timestamp}] {message}",
>     input_variables=["message"],
>     partial_variables={"timestamp": get_timestamp}  # 注意：传函数对象，不是调用结果
> )
> ```
>
> **选择标准**：
> - 值是否会变化？→ 会变化用函数，不变用字符串
> - 是否需要实时性？→ 需要用函数
> - 是否有计算成本？→ 有成本且不常变用字符串

**常见陷阱**：

```python
# ❌ 错误：传入函数调用结果，而非函数对象
partial_variables={"date": get_current_date()}  # 只会在创建时调用一次

# ✅ 正确：传入函数对象
partial_variables={"date": get_current_date}  # 每次 format 时都会调用
```

1. ✅ 说明本质区别（求值时机）
2. ✅ 提供明确的选择标准

## 问题3："模板组合时如何处理变量冲突？"

### 普通回答（❌ 不出彩）

"如果两个模板有相同的变量，会报错。"

### 出彩回答（✅ 推荐）

> **模板组合时的变量处理遵循以下规则：**
>
> **1. input_variables 的合并（取并集）**
> - 自动合并两个模板的 input_variables
> - 相同变量名不会冲突，只保留一个
> - 最终需要用户提供所有 input_variables 的值
>
> **2. partial_variables 的合并（检查冲突）**
> - 如果两个模板有相同的 partial_variables 键，会抛出 ValueError
> - 错误信息："Cannot have same variable partialed twice."
> - 这是为了避免歧义：不清楚应该使用哪个值
>
> **3. template_format 的一致性要求**
> - 两个模板的 template_format 必须相同
> - 否则抛出 ValueError: "Cannot add templates of different formats"
> - 这是因为不同格式的语法不兼容

**源码实现**（来自 prompt.py:142-184）：

```python
def __add__(self, other: Any) -> PromptTemplate:
    if isinstance(other, PromptTemplate):
        # 检查格式一致性
        if self.template_format != other.template_format:
            raise ValueError("Cannot add templates of different formats")

        # 合并 input_variables（取并集）
        input_variables = list(
            set(self.input_variables) | set(other.input_variables)
        )

        # 合并 partial_variables（检查冲突）
        partial_variables = dict(self.partial_variables.items())
        for k, v in other.partial_variables.items():
            if k in partial_variables:
                raise ValueError("Cannot have same variable partialed twice.")
            partial_variables[k] = v

        return PromptTemplate(
            template=self.template + other.template,
            input_variables=input_variables,
            partial_variables=partial_variables,
            template_format=self.template_format,
        )
```

**实际应用示例**：

```python
from langchain_core.prompts import PromptTemplate

# ✅ 正确：input_variables 可以重复
prompt1 = PromptTemplate.from_template("Hello {name}.")
prompt2 = PromptTemplate.from_template("Your role is {name}.")
combined = prompt1 + prompt2  # 成功，name 只需提供一次

# ❌ 错误：partial_variables 不能重复
prompt1 = PromptTemplate(
    template="{system} {query}",
    input_variables=["query"],
    partial_variables={"system": "System A"}
)
prompt2 = PromptTemplate(
    template="{system} {task}",
    input_variables=["task"],
    partial_variables={"system": "System B"}
)
# combined = prompt1 + prompt2  # 抛出 ValueError

# ✅ 解决方案：使用不同的变量名
prompt2 = PromptTemplate(
    template="{system2} {task}",
    input_variables=["task"],
    partial_variables={"system2": "System B"}
)
combined = prompt1 + prompt2  # 成功
```

## 问题4："什么时候应该使用 Mustache 模板而不是默认的 f-string？"

### 普通回答（❌ 不出彩）

"Mustache 支持嵌套变量，f-string 不支持。"

### 出彩回答（✅ 推荐）

> **选择模板格式的决策树：**
>
> **使用 f-string（默认）的场景：**
> - 简单的变量替换（90% 的场景）
> - 性能敏感的应用
> - 团队熟悉 Python f-string 语法
> - 需要模板验证功能
>
> **使用 Mustache 的场景：**
> - 需要嵌套变量访问：`{{user.profile.name}}`
> - 需要条件渲染：`{{#is_premium}} Premium content {{/is_premium}}`
> - 需要列表迭代：`{{#items}} {{name}} {{/items}}`
> - 跨语言模板共享（Mustache 是语言无关的）
> - 不需要模板验证（Mustache 不支持验证）
>
> **使用 Jinja2 的场景：**
> - 需要复杂的逻辑控制（if/for/filter）
> - 需要模板继承和宏
> - 已有 Jinja2 模板资产
> - ⚠️ 注意：有安全风险，不要接受不可信输入

**对比示例**：

```python
from langchain_core.prompts import PromptTemplate

# 场景1：简单变量替换 → f-string
prompt = PromptTemplate.from_template("Hello {name}!")

# 场景2：嵌套变量 → Mustache
prompt = PromptTemplate.from_template(
    "User: {{user.name}}, Email: {{user.email}}",
    template_format="mustache"
)
result = prompt.format(user={"name": "Alice", "email": "alice@example.com"})

# 场景3：条件渲染 → Mustache
prompt = PromptTemplate.from_template(
    "{{#is_premium}}Premium: {{feature}}{{/is_premium}}",
    template_format="mustache"
)
result = prompt.format(is_premium=True, feature="Advanced Analytics")

# 场景4：复杂逻辑 → Jinja2
prompt = PromptTemplate.from_template(
    "{% if score > 80 %}Excellent{% elif score > 60 %}Good{% else %}Need improvement{% endif %}",
    template_format="jinja2"
)
```

**性能对比**（基于社区测试）：
- f-string: 最快（基准）
- Mustache: 约慢 2-3 倍
- Jinja2: 约慢 5-10 倍

2. ✅ 列出具体使用场景

## 问题5："在 RAG 系统中，如何优雅地管理多个提示词模板？"

### 普通回答（❌ 不出彩）

"可以把模板存在文件里，然后用 from_file 加载。"

### 出彩回答（✅ 推荐）

> **RAG 系统的提示词模板管理最佳实践：**
>
> **1. 分层管理策略**
> - 系统层：通用系统提示词（角色定义、行为规范）
> - 任务层：特定任务模板（检索、生成、评估）
> - 用户层：用户自定义模板
>
> **2. 使用 Partial Variables 预填充公共部分**
> - 系统设置、日期、版本信息等
> - 减少每次调用的参数传递
>
> **3. 使用模板组合构建复杂提示词**
> - 基础模板 + 任务特定模板
> - 支持灵活组合和复用
>
> **4. 从文件加载支持多语言**
> - 按语言组织模板文件
> - 运行时动态加载

**完整实现示例**：

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime
from pathlib import Path

class RAGPromptManager:
    """RAG 系统提示词管理器"""

    def __init__(self, template_dir: str = "prompts"):
        self.template_dir = Path(template_dir)
        self._cache = {}

    def get_system_prompt(self, language: str = "zh") -> PromptTemplate:
        """获取系统提示词（带 partial variables）"""
        return PromptTemplate(
            template="你是一个专业的 AI 助手。当前日期：{date}\n{task_instruction}",
            input_variables=["task_instruction"],
            partial_variables={"date": lambda: datetime.now().strftime("%Y-%m-%d")}
        )

    def get_retrieval_prompt(self) -> PromptTemplate:
        """获取检索提示词"""
        return PromptTemplate.from_template(
            "基于以下上下文回答问题：\n\n{context}\n\n问题：{question}"
        )

    def get_generation_prompt(self) -> PromptTemplate:
        """获取生成提示词"""
        return PromptTemplate.from_template(
            "请根据检索结果生成回答：\n{retrieved_docs}\n\n要求：{requirements}"
        )

    def build_rag_prompt(self) -> PromptTemplate:
        """组合完整的 RAG 提示词"""
        system = self.get_system_prompt()
        retrieval = self.get_retrieval_prompt()

        # 使用模板组合
        return system + retrieval

    def load_from_file(self, template_name: str, language: str = "zh") -> PromptTemplate:
        """从文件加载模板（支持多语言）"""
        cache_key = f"{template_name}_{language}"
        if cache_key in self._cache:
            return self._cache[cache_key]

        file_path = self.template_dir / language / f"{template_name}.txt"
        prompt = PromptTemplate.from_file(str(file_path))
        self._cache[cache_key] = prompt
        return prompt

# 使用示例
manager = RAGPromptManager()

# 1. 获取组合提示词
rag_prompt = manager.build_rag_prompt()
result = rag_prompt.format(
    task_instruction="回答用户问题",
    context="LangChain 是一个 AI 应用开发框架...",
    question="什么是 LangChain？"
)

# 2. 从文件加载多语言模板
zh_prompt = manager.load_from_file("greeting", language="zh")
en_prompt = manager.load_from_file("greeting", language="en")
```

```
│   ├── retrieval.txt
│   └── generation.txt
└── en/
    ├── greeting.txt
    ├── retrieval.txt
    └── generation.txt
```

## 常见陷阱总结

### 陷阱1：Partial Variables 传入函数调用结果

```python
# ❌ 错误
partial_variables={"date": get_current_date()}  # 只在创建时调用一次

# ✅ 正确
partial_variables={"date": get_current_date}  # 每次 format 时调用
```

### 陷阱2：模板组合时 partial_variables 冲突

```python
# ❌ 错误：两个模板有相同的 partial_variables
prompt1 = PromptTemplate(..., partial_variables={"system": "A"})
prompt2 = PromptTemplate(..., partial_variables={"system": "B"})
combined = prompt1 + prompt2  # ValueError

# ✅ 正确：使用不同的变量名
prompt2 = PromptTemplate(..., partial_variables={"system2": "B"})
```

### 陷阱3：混用不同的 template_format

```python
# ❌ 错误
prompt1 = PromptTemplate.from_template("Hello {name}", template_format="f-string")
prompt2 = PromptTemplate.from_template("{{greeting}}", template_format="mustache")
combined = prompt1 + prompt2  # ValueError

# ✅ 正确：使用相同格式
prompt2 = PromptTemplate.from_template("Goodbye {name}", template_format="f-string")
```

### 陷阱4：Mustache 模板无法验证

```python
# ❌ 错误：Mustache 不支持验证
prompt = PromptTemplate(
    template="{{name}}",
    input_variables=["name"],
    template_format="mustache",
    validate_template=True  # 会抛出 ValueError
)

# ✅ 正确：关闭验证
prompt = PromptTemplate(
    template="{{name}}",
    input_variables=["name"],
    template_format="mustache",
    validate_template=False
)
```
