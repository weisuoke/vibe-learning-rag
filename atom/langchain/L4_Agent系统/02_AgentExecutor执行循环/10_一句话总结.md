# 【一句话总结】AgentExecutor 执行循环

---

**AgentExecutor 执行循环是 LangChain Agent 的核心引擎，通过 while 循环驱动"思考→行动→观察"的 ReAct 迭代，配合 max_iterations 和错误自修复机制确保安全可控，是理解所有 AI Agent 系统的基础模式。**

---

## 展开理解

AgentExecutor 的本质极其简单：一个 while 循环。每次迭代中，`agent.plan()` 调用 LLM 做出决策——要么返回 `AgentAction` 表示"我需要调用某个工具"，要么返回 `AgentFinish` 表示"我有最终答案了"。工具执行的结果作为 observation 被记录到 `intermediate_steps` 中，供下一次 plan() 参考。这就是 ReAct 模式的完整实现。

循环控制是生产环境的生命线。`max_iterations` 防止 Agent 陷入无限循环，`max_execution_time` 防止单次请求耗时过长，`handle_parsing_errors` 把 LLM 的格式错误转化为 observation 让 Agent 自我修正而不是直接崩溃。这三个配置缺一不可。

从架构演进的角度看，AgentExecutor 的 while 循环正在被 LangGraph 的 StateGraph 状态机所替代（`create_react_agent`），但核心模式完全一致——调用 LLM 决策、执行工具、反馈结果、重复。理解了 AgentExecutor 的执行循环，就掌握了所有 AI Agent 系统的底层逻辑。
