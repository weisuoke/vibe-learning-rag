## 一、依赖安装问题

### 1.1 Node.js 版本不符合

**错误**:
```
ERR_PNPM_UNSUPPORTED_ENGINE  Unsupported environment (project requires node >=22.12.0)
```

**解决方案**:
```bash
# 检查当前版本
node --version

# 使用 asdf 安装
asdf install nodejs 22.12.0
asdf global nodejs 22.12.0

# 或使用 nvm
nvm install 22.12.0
nvm use 22.12.0

# 验证
node --version
```

---

### 1.2 pnpm 未安装

**错误**:
```
command not found: pnpm
```

**解决方案**:
```bash
# 启用 corepack（Node.js 22 内置）
corepack enable pnpm

# 验证
pnpm --version
```

---

### 1.3 依赖安装失败

**错误**:
```
ERR_PNPM_FETCH_404  GET https://registry.npmjs.org/xxx: Not Found
```

**解决方案**:
```bash
# 清理缓存
pnpm store prune

# 使用镜像
pnpm config set registry https://registry.npmmirror.com

# 重新安装
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

---

## 二、编译问题

### 2.1 TypeScript 类型错误

**错误**:
```
error TS2792: Cannot find module 'openclaw/plugin-sdk'
```

**解决方案**:
```bash
# 检查 tsconfig.json 的 paths 配置
cat tsconfig.json | grep -A 5 paths

# 重新编译
pnpm build
```

---

### 2.2 tsdown 编译失败

**错误**:
```
Error: Cannot find module 'tsdown'
```

**解决方案**:
```bash
# 重新安装依赖
rm -rf node_modules
pnpm install

# 验证 tsdown 安装
pnpm list tsdown

# 重新编译
pnpm build
```

---

### 2.3 UI 构建失败

**错误**:
```
Error: Cannot find module 'vite'
```

**解决方案**:
```bash
# 进入 UI 目录
cd ui

# 安装依赖
pnpm install

# 构建
pnpm build

# 返回根目录
cd ..
```

## 三、运行时问题

### 3.1 端口被占用

**错误**:
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方案**:
```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=3001 pnpm gateway:watch
```

---

### 3.2 模块找不到

**错误**:
```
Error: Cannot find module './dist/index.js'
```

**解决方案**:
```bash
# 检查编译产物
ls -la dist/

# 重新编译
pnpm build

# 运行
pnpm gateway:watch
```

---

### 3.3 环境变量缺失

**错误**:
```
Error: OPENAI_API_KEY is not defined
```

**解决方案**:
```bash
# 创建 .env 文件
cp .env.example .env

# 编辑配置
vim .env

# 添加 API Key
echo "OPENAI_API_KEY=sk-..." >> .env
```

## 四、Git 问题

### 4.1 克隆失败

**错误**:
```
fatal: unable to access 'https://github.com/openclaw/openclaw.git/': Could not resolve host
```

**解决方案**:
```bash
# 检查网络连接
ping github.com

# 使用 SSH
git clone git@github.com:openclaw/openclaw.git

# 或配置代理
git config --global http.proxy http://proxy:port
```

---

### 4.2 权限问题

**错误**:
```
Permission denied (publickey)
```

**解决方案**:
```bash
# 生成 SSH 密钥
ssh-keygen -t ed25519 -C "your_email@example.com"

# 添加到 GitHub
cat ~/.ssh/id_ed25519.pub
# 复制并添加到 GitHub Settings > SSH Keys
```

## 五、性能问题

### 5.1 编译速度慢

**问题**: 编译时间超过 10 秒

**解决方案**:
```bash
# 使用增量编译
# tsdown.config.ts
export default defineConfig({
  incremental: true,
  clean: false
});

# 使用并行编译
pnpm build --parallel
```

---

### 5.2 内存占用高

**问题**: Node.js 进程占用大量内存

**解决方案**:
```bash
# 增加内存限制
NODE_OPTIONS="--max-old-space-size=4096" pnpm dev

# 或在 package.json 中配置
{
  "scripts": {
    "dev": "NODE_OPTIONS='--max-old-space-size=4096' node scripts/run-node.mjs"
  }
}
```

## 七、完整排查流程

### 7.1 排查脚本

```bash
#!/bin/bash

echo "=== OpenClaw 环境排查 ==="

# 1. 检查 Node.js
echo "1. 检查 Node.js..."
if command -v node &> /dev/null; then
    NODE_VERSION=$(node --version)
    echo "✅ Node.js: $NODE_VERSION"
else
    echo "❌ Node.js 未安装"
    exit 1
fi

# 2. 检查 pnpm
echo "2. 检查 pnpm..."
if command -v pnpm &> /dev/null; then
    PNPM_VERSION=$(pnpm --version)
    echo "✅ pnpm: $PNPM_VERSION"
else
    echo "❌ pnpm 未安装"
    exit 1
fi

# 3. 检查依赖
echo "3. 检查依赖..."
if [ -d "node_modules" ]; then
    echo "✅ 依赖已安装"
else
    echo "❌ 依赖未安装"
    exit 1
fi

# 4. 检查编译产物
echo "4. 检查编译产物..."
if [ -d "dist" ]; then
    echo "✅ 编译产物存在"
else
    echo "❌ 编译产物不存在"
    exit 1
fi

# 5. 检查端口
echo "5. 检查端口..."
if lsof -i :3000 &> /dev/null; then
    echo "⚠️  端口 3000 被占用"
else
    echo "✅ 端口 3000 可用"
fi

echo "=== 排查完成 ==="
```
