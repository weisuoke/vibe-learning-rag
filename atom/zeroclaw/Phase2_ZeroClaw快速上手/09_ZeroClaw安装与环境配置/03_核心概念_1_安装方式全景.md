# 核心概念 1：安装方式全景

> **一句话定义：** ZeroClaw 提供 5 种安装方式（一键脚本、Homebrew、Cargo Install、预编译二进制、Docker），覆盖从"30 秒快速体验"到"定制化源码编译"的全部场景，最终目标都是把一个 ~8.8 MB 的二进制放到系统 PATH 中。

---

## 1. 为什么有 5 种安装方式？

### 1.1 根本原因：Rust 是编译型语言

在 `02_第一性原理.md` 中我们推导过：Rust 编译出的二进制是**平台特定的机器码**。不同操作系统 + CPU 架构需要不同的二进制文件。这意味着不可能像 `npm install -g` 那样"下载源码 → 解释器运行"一步到位。

```
同一份源码 → 6 种二进制产物
├── x86_64-unknown-linux-gnu      (Linux PC / 服务器)
├── aarch64-unknown-linux-gnu     (Linux ARM64 / 树莓派 4+)
├── armv7-unknown-linux-gnueabihf (Linux ARM 32 位 / 树莓派 3)
├── x86_64-apple-darwin           (macOS Intel)
├── aarch64-apple-darwin          (macOS Apple Silicon)
└── x86_64-pc-windows-msvc        (Windows PC)
```

> **前端类比：** Node.js 不需要这么多安装方式，因为 `node` 运行时帮你抹平了平台差异。但你有没有遇到过 `node-gyp` 编译原生模块失败的情况？那正是"编译型代码"在 Node.js 世界中的体现——ZeroClaw 整个程序都是编译型的，所以这个问题被放大到了安装层面。

### 1.2 不同用户、不同场景

| 用户画像 | 核心诉求 | 最佳安装方式 |
|---------|---------|------------|
| 第一次听说 ZeroClaw，想试试 | 30 秒装好、开箱即用 | 一键脚本 |
| macOS/Linux 日常开发者 | 用包管理器统一管理 | Homebrew |
| Rust 开发者 / 想定制 Feature | 已有 Rust 环境、想控制编译选项 | Cargo Install |
| 运维工程师 / 离线环境 | 不想装任何工具链 | 预编译二进制 |
| 服务器 24/7 部署 / 隔离环境 | 容器化、可复现 | Docker |

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

## 2. 五种安装方式详解

### 2.1 方式一：一键安装脚本 ⭐ 最推荐

**适用场景：** 首次安装、快速体验、不想操心依赖的用户

#### 基本命令

```bash
# 方式 A：官网安装（推荐）
curl -fsSL https://zeroclawlabs.ai/install.sh | bash

# 方式 B：GitHub 源安装
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash
```

#### Bootstrap 脚本做了什么？

这个脚本是一个**智能安装器**，它会自动完成以下工作：

```
bootstrap.sh 执行流程
│
├── 1. 预检 (Pre-flight Checks)
│   ├── RAM 检查：最低 2 GB（ZEROCLAW_BOOTSTRAP_MIN_RAM_MB=2048）
│   ├── 磁盘检查：最低 6 GB（ZEROCLAW_BOOTSTRAP_MIN_DISK_MB=6144）
│   └── 网络连通性检查
│
├── 2. 平台检测
│   ├── 检测操作系统（Linux / macOS / Windows）
│   ├── 检测 CPU 架构（x86_64 / aarch64 / armv7）
│   └── 组合成 target triple（如 aarch64-apple-darwin）
│
├── 3. 安装策略选择
│   ├── 默认：优先预编译二进制 → 不可用时回退源码编译
│   ├── --prefer-prebuilt：强制优先预编译
│   ├── --prebuilt-only：仅预编译（不回退）
│   └── --force-source-build：强制源码编译
│
├── 4. 依赖安装（如需要）
│   ├── --install-system-deps：自动装系统构建依赖
│   └── --install-rust：自动装 Rust 工具链
│
├── 5. 安装执行
│   ├── 预编译路径：下载 → 解压 → 放到 PATH
│   └── 源码路径：cargo install --locked
│
├── 6. 验证
│   └── zeroclaw --version
│
└── 7. 可选后续
    └── --onboard：安装后自动运行 zeroclaw onboard
```

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

#### 关键参数速查

```bash
# 交互式引导（适合新手，一步步问你要怎么装）
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --guided

# 全自动安装（CI/CD 场景，不问任何问题）
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- \
  --install-system-deps \
  --install-rust \
  --prefer-prebuilt \
  --onboard \
  --api-key "sk-or-v1-your-key" \
  --provider openrouter

# 低资源设备（树莓派等，避免本地编译吃光内存）
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --prebuilt-only

# Docker 模式
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --docker
```

| 参数 | 作用 | 使用场景 |
|------|------|---------|
| `--guided` | 交互式引导，一步步问你 | 第一次安装 |
| `--install-system-deps` | 自动安装系统构建依赖 | 全新系统 |
| `--install-rust` | 自动安装 Rust 工具链 | 没有 Rust |
| `--prefer-prebuilt` | 优先下载预编译二进制 | 低资源系统 |
| `--prebuilt-only` | 仅预编译（失败则中止） | 不想编译 |
| `--force-source-build` | 强制从源码编译 | 需要定制 Feature |
| `--docker` | Docker 模式安装 | 容器化部署 |
| `--onboard` | 安装后自动执行 onboard | 一步到位 |
| `--api-key <key>` | 非交互式传入 API Key | CI/CD |
| `--provider <id>` | 指定 Provider（默认 openrouter） | 非交互式 |

[来源: sourcecode/zeroclaw/scripts/bootstrap.sh]

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 一行命令搞定 | ❌ 需要信任远程脚本（`curl | bash`） |
| ✅ 自动检测平台 | ❌ 首次源码编译耗时 30-40 分钟 |
| ✅ 智能回退策略 | ❌ 需要网络 |
| ✅ 支持全自动（CI/CD 友好） | |

### 2.2 方式二：Homebrew（macOS / Linux）

**适用场景：** 已经用 Homebrew 管理工具的 macOS / Linux 开发者

#### 命令

```bash
# 安装
brew install zeroclaw

# 更新
brew upgrade zeroclaw

# 卸载
brew uninstall zeroclaw

# 查看版本
zeroclaw --version
```

> **前端类比：** `brew install node` 的体验——Homebrew 帮你管理安装、更新、卸载。`brew upgrade` 就像 `nvm install --lts` + 自动切换。

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 统一管理（brew upgrade 一键更新全部） | ❌ 需要先安装 Homebrew |
| ✅ 干净卸载 | ❌ 可能不是最新版（等 formula 更新） |
| ✅ 自动处理依赖 | ❌ 仅 macOS / Linux |
| ✅ 不需要 Rust 工具链 | ❌ 无法自定义 Feature Flags |

### 2.3 方式三：Cargo Install（Rust 开发者）

**适用场景：** 已安装 Rust、想自定义 Feature Flags、或想查看 / 修改源码的开发者

#### 基本命令

```bash
# 从 crates.io 安装（发布版本）
cargo install zeroclaw

# 从源码安装（开发版本）
git clone https://github.com/zeroclaw-labs/zeroclaw.git
cd zeroclaw
cargo install --path . --force --locked
```

#### 带 Feature Flags 安装

这是 Cargo Install 的**独特优势**——你可以在编译时选择要包含哪些功能：

```bash
# 默认安装（最小体积，~8.8 MB）
cargo install zeroclaw

# 带 Matrix 通道支持
cargo install zeroclaw --features "channel-matrix"

# 带硬件控制（树莓派 GPIO）
cargo install zeroclaw --features "hardware"

# 带 PostgreSQL 内存后端
cargo install zeroclaw --features "memory-postgres"

# 多个 Feature 组合
cargo install zeroclaw --features "channel-matrix,memory-postgres,browser-native"

# 全功能安装（体积最大）
cargo install zeroclaw --features "full"
```

| Feature Flag | 功能 | 新增依赖 |
|-------------|------|---------|
| `hardware` | USB + 串口支持 | nusb, tokio-serial |
| `channel-matrix` | Matrix 协议通道 | matrix-sdk |
| `channel-lark` | 飞书通道 | prost |
| `memory-postgres` | PostgreSQL 内存后端 | postgres |
| `observability-otel` | OpenTelemetry 可观测性 | opentelemetry |
| `peripheral-rpi` | 树莓派 GPIO | rppal |
| `browser-native` | 浏览器自动化 | fantoccini |
| `sandbox-landlock` | Landlock 沙箱(Linux) | - |
| `sandbox-bubblewrap` | Bubblewrap 沙箱(Linux) | - |
| `probe` | STM32 探测 | probe-rs (~50 deps) |
| `rag-pdf` | PDF 文档 RAG | pdf-extract |
| `whatsapp-web` | WhatsApp 客户端 | wa-rs |

[来源: sourcecode/zeroclaw/Cargo.toml]

> **前端类比：** `cargo install --features "channel-matrix"` 类似于想象中的 `npm install -g zeroclaw --with=matrix-channel`——但区别是巨大的。npm 的可选依赖（`optionalDependencies`）仍然会下载到 node_modules 中，只是运行时可能不加载。而 Cargo 的 Feature Flags 是**编译时排除**——未选择的功能代码根本不存在于最终二进制中，所以体积可以做到极致小。

#### `--locked` 参数

```bash
cargo install --path . --force --locked
# --locked = 使用 Cargo.lock 锁定的依赖版本（类似 npm ci 而非 npm install）
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 可自定义 Feature Flags | ❌ 需要先安装 Rust 工具链 |
| ✅ 可从最新源码编译 | ❌ 编译耗时 30-40 分钟 |
| ✅ Rust 开发者最熟悉的方式 | ❌ 需要系统构建依赖（gcc, openssl-dev 等） |
| ✅ `--locked` 保证可复现构建 | ❌ 需要 2GB+ RAM |

### 2.4 方式四：预编译二进制

**适用场景：** 离线环境、不想安装任何工具链、快速在服务器上部署

#### 下载与安装

```bash
# 确定你的平台 target triple（取消注释你需要的行）
# TARGET="aarch64-apple-darwin"            # macOS Apple Silicon
# TARGET="x86_64-apple-darwin"             # macOS Intel
# TARGET="x86_64-unknown-linux-gnu"        # Linux x86_64
# TARGET="aarch64-unknown-linux-gnu"       # Linux ARM64
# TARGET="armv7-unknown-linux-gnueabihf"   # Linux ARM 32 位
# TARGET="x86_64-pc-windows-msvc"          # Windows

# 下载 → 解压 → 放到 PATH
curl -LO "https://github.com/zeroclaw-labs/zeroclaw/releases/latest/download/zeroclaw-${TARGET}.tar.gz"
tar xzf "zeroclaw-${TARGET}.tar.gz"
sudo mv zeroclaw /usr/local/bin/       # 或 ~/.local/bin/

# 验证
zeroclaw --version
```

#### 支持的 6 个平台

| 平台 | Target Triple | 适用设备 |
|------|--------------|---------|
| Linux x86_64 | `x86_64-unknown-linux-gnu` | 标准 PC / 服务器 |
| Linux ARM64 | `aarch64-unknown-linux-gnu` | ARM64 服务器 / 树莓派 4+ |
| Linux ARM32 | `armv7-unknown-linux-gnueabihf` | 树莓派 3 |
| macOS Intel | `x86_64-apple-darwin` | Intel Mac |
| macOS ARM | `aarch64-apple-darwin` | Apple Silicon (M1/M2/M3+) |
| Windows | `x86_64-pc-windows-msvc` | Windows PC |

[来源: sourcecode/zeroclaw/.github/workflows/release.yml]

> **前端类比：** 这就像去 [VS Code 官网](https://code.visualstudio.com/) 下载安装包——你选择你的操作系统，下载对应的安装包，解压就能用。不需要 Node.js、不需要 npm、不需要任何运行时。

> **日常生活类比：** 这像在宜家买**已经组装好的**家具（而不是扁平包装自己装）。直接搬回家就能用，但选择有限——你无法定制颜色或尺寸（无法自定义 Feature Flags）。

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 不需要任何工具链 | ❌ 无法自定义 Feature Flags |
| ✅ 下载即用（秒级） | ❌ 只有默认功能集 |
| ✅ 适合离线环境 | ❌ 需要自己管理更新 |
| ✅ 最小依赖 | ❌ 需要知道自己的 target triple |

### 2.5 方式五：Docker

**适用场景：** 服务器 24/7 部署、需要隔离环境、CI/CD 流水线

#### 基本使用

```bash
# 方式 A：使用 Dockerfile 本地构建
git clone https://github.com/zeroclaw-labs/zeroclaw.git
cd zeroclaw
docker build -t zeroclaw-bootstrap:local .
docker run -it zeroclaw-bootstrap:local

# 方式 B：使用 bootstrap.sh 的 Docker 模式
curl -LsSf https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/scripts/bootstrap.sh | bash -s -- --docker
```

#### 多阶段构建

ZeroClaw 的 Dockerfile 使用**三阶段构建**：

| Stage | 基础镜像 | 用途 | 体积 |
|-------|---------|------|------|
| builder | `rust:1.93-slim` | 编译 Release 二进制 | ~2 GB |
| dev | `debian:trixie-slim` | 开发 / 调试环境 | ~80-100 MB |
| **release** ⭐ | `distroless/cc-debian13` | 生产环境（无 shell，只有二进制） | ~20-30 MB |

[来源: sourcecode/zeroclaw/Dockerfile]

> **前端类比：** 这和前端应用的 Docker 多阶段构建完全一样的思路：
> ```dockerfile
> # Stage 1: 用 node:20 构建（包含 npm, webpack 等构建工具）
> FROM node:20 AS builder
> RUN npm run build
>
> # Stage 2: 用 nginx:alpine 运行（只有 nginx + 静态文件）
> FROM nginx:alpine
> COPY --from=builder /app/dist /usr/share/nginx/html
> ```
> ZeroClaw 的 distroless 镜像甚至比 alpine 更极端——连 shell 都没有，只有 zeroclaw 二进制。

#### Docker Compose 生产部署示例

```yaml
services:
  zeroclaw:
    build: { context: ., target: release }
    environment:
      - ZEROCLAW_API_KEY=${ZEROCLAW_API_KEY}
      - ZEROCLAW_PROVIDER=openrouter
    ports: ["42617:42617"]
    volumes: [zeroclaw-data:/root/.zeroclaw]
    restart: unless-stopped
volumes:
  zeroclaw-data:
```

#### 优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 完全隔离（不污染宿主环境） | ❌ 需要安装 Docker |
| ✅ 可复现（Dockerfile = 安装说明书） | ❌ 首次构建耗时较长 |
| ✅ distroless 生产镜像极致安全 | ❌ 开发调试不如本地方便 |
| ✅ 适合服务器 24/7 部署 | ❌ 学习 Docker 有一定门槛 |
| ✅ Docker Compose 一键编排 | |

## 3. 安装方式决策流程图

遇到"该选哪种方式"时，按以下流程判断：

```
你要做什么？
│
├── 只是想快速试试 ZeroClaw？
│   └── ✅ 一键脚本: curl -fsSL https://zeroclawlabs.ai/install.sh | bash
│
├── 已经用 Homebrew 管理工具？
│   └── ✅ Homebrew: brew install zeroclaw
│
├── 已有 Rust 环境？
│   │
│   ├── 需要自定义 Feature Flags？
│   │   └── ✅ Cargo Install + --features: cargo install zeroclaw --features "..."
│   │
│   └── 不需要定制？
│       └── ✅ Cargo Install: cargo install zeroclaw
│
├── 没有 Rust、不想装 Homebrew、也不想 curl | bash？
│   └── ✅ 预编译二进制: 下载 tar.gz → 解压 → 放到 PATH
│
├── 在服务器上 24/7 部署？
│   └── ✅ Docker: docker compose up -d
│
├── 离线环境 / 无网络？
│   └── ✅ 预编译二进制: 提前下载好，U 盘拷贝过去
│
└── 树莓派 / 低资源设备？
    │
    ├── 有对应预编译版本？
    │   └── ✅ 预编译二进制（--prebuilt-only）
    │
    └── 没有 / 需要硬件 Feature？
        └── ✅ 交叉编译 或 在设备上 cargo install --features "hardware"
```

## 6. 在 ZeroClaw 开发中的应用

安装方式的选择贯穿整个 ZeroClaw 使用周期：

| 开发阶段 | 推荐安装方式 | 原因 |
|---------|------------|------|
| **初次体验** | 一键脚本 | 最快上手，30 秒搞定 |
| **日常开发** | Homebrew / Cargo | 方便更新，Cargo 可定制 Feature |
| **贡献源码** | Cargo（从 git clone） | 需要修改代码、本地编译测试 |
| **CI/CD** | 预编译二进制 / Docker | 速度快、可复现 |
| **服务器部署** | Docker | 隔离环境、自动重启 |
| **嵌入式设备** | 预编译 / 交叉编译 | 设备资源有限 |

### 安装方式与 ZeroClaw 架构的关系

安装方式决定了两件事：**功能集**（编译时 Feature Flags）和**部署形态**（本地 vs 容器）。默认安装包含核心功能，Cargo Install 可通过 `--features` 增加 Matrix 通道、硬件控制、PostgreSQL 内存等模块。

## 7. 常见问题速查

| 问题 | 解决方案 |
|------|---------|
| `command not found: zeroclaw` | 检查 PATH，确认二进制位置在 PATH 中 |
| 一键脚本 RAM 检查失败 | 使用 `--prebuilt-only` 跳过源码编译 |
| Cargo Install 编译失败 | 检查系统依赖是否安装（gcc, openssl-dev 等） |
| 不知道选哪种安装方式 | 新手用一键脚本，有 Homebrew 用 Homebrew |
| macOS 提示"无法验证开发者" | `xattr -d com.apple.quarantine $(which zeroclaw)` |
| 想更新到最新版 | Homebrew: `brew upgrade`，Cargo: 重新 `cargo install` |
