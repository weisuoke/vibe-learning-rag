## 1. 核心操作速查

### 1.1 Insert（插入数据）

**最常用的操作，必须掌握。**

```python
from pymilvus import MilvusClient
import numpy as np

client = MilvusClient("http://localhost:19530")

# 批量插入数据
data = [
    {"id": 1, "vector": np.random.rand(128).tolist(), "name": "Item 1", "price": 99.99},
    {"id": 2, "vector": np.random.rand(128).tolist(), "name": "Item 2", "price": 199.99},
    {"id": 3, "vector": np.random.rand(128).tolist(), "name": "Item 3", "price": 299.99}
]

result = client.insert(
    collection_name="my_collection",
    data=data
)

print(f"Inserted {result['insert_count']} records")
```

**关键点**：
- 批量插入比单条插入快 10-100 倍
- 推荐批量大小：1000-10000 条

### 1.2 Search（向量检索）

**RAG 系统的核心操作，必须掌握。**

```python
# 向量检索
query_vector = np.random.rand(128).tolist()

results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,  # 返回 Top-10 结果
    output_fields=["id", "name", "price"]  # 返回的字段
)

# 处理结果
for hits in results:
    for hit in hits:
        print(f"ID: {hit['id']}, Score: {hit['score']}, Name: {hit['entity']['name']}")
```

**关键点**：
- `limit` 控制返回结果数量
- `output_fields` 控制返回的字段（不要返回向量字段，影响性能）

### 1.3 Query（精确查询）

**元数据过滤的核心操作，必须掌握。**

```python
# 按主键查询
results = client.query(
    collection_name="my_collection",
    ids=[1, 2, 3],
    output_fields=["id", "name", "price"]
)

# 按表达式查询
results = client.query(
    collection_name="my_collection",
    filter="price > 100 and status == 'active'",
    output_fields=["id", "name", "price"],
    limit=10
)
```

**关键点**：
- Query 用于精确查找，不涉及向量计算
- 支持复杂的布尔表达式

### 1.4 Delete（删除数据）

**数据清理的核心操作，必须掌握。**

```python
# 按主键删除
result = client.delete(
    collection_name="my_collection",
    ids=[1, 2, 3]
)

# 按表达式删除
result = client.delete(
    collection_name="my_collection",
    filter="status == 'archived'"
)

print(f"Deleted {result['delete_count']} records")
```

**关键点**：
- Delete 是逻辑删除，需要 Compaction 才能回收空间
- 批量删除比单条删除快得多

### 1.5 Upsert（插入或更新）

**数据同步的核心操作，按需掌握。**

```python
# Upsert 数据
data = [
    {"id": 1, "vector": np.random.rand(128).tolist(), "name": "Updated Item 1", "price": 89.99},
    {"id": 4, "vector": np.random.rand(128).tolist(), "name": "New Item 4", "price": 399.99}
]

result = client.upsert(
    collection_name="my_collection",
    data=data
)

print(f"Upserted {result['upsert_count']} records")
```

**关键点**：
- Upsert 性能比 Insert 慢（需要先查询）
- 仅在必要时使用（如数据同步）

## 2. 必须掌握的 5 个概念

### 2.1 Load/Release

**Collection 必须 Load 才能执行 Search 和 Query。**

```python
from pymilvus import Collection

collection = Collection("my_collection")

# Load Collection（加载到内存）
collection.load()

# 执行 Search
results = collection.search(...)

# Release Collection（释放内存）
collection.release()
```

### 2.2 一致性级别

**控制读操作能看到多新的数据。**

```python
# Bounded Consistency（推荐，平衡性能和准确性）
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Bounded"  # 推荐
)

# Strong Consistency（保证读取最新数据）
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Strong"  # 实时推荐系统使用
)
```

**选择建议**：
- 一般 RAG 应用：使用 `Bounded`
- 实时推荐系统：使用 `Strong`
- 离线分析：使用 `Eventually`

### 2.3 批量操作

**批量操作性能远优于单条操作。**

```python
# ✅ 推荐：批量插入
batch_size = 1000
for i in range(0, len(data), batch_size):
    batch = data[i:i+batch_size]
    client.insert(collection_name="my_collection", data=batch)

# ❌ 不推荐：单条插入
for item in data:
    client.insert(collection_name="my_collection", data=[item])
```

### 2.4 Compaction

**删除操作后需要 Compaction 才能回收空间。**

```python
from pymilvus import Collection

collection = Collection("my_collection")

# 删除数据
client.delete(collection_name="my_collection", filter="status == 'archived'")

# 执行 Compaction 回收空间
collection.compact()
collection.wait_for_compaction_completed()
```

### 2.5 标量过滤

**在向量检索时添加标量字段过滤条件。**

```python
# 带标量过滤的检索
results = client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    filter="price > 100 and status == 'active'",  # 标量过滤
    output_fields=["id", "name", "price"]
)
```

## 3. 最常用的代码模板

### 3.1 RAG 文档检索模板

```python
from pymilvus import MilvusClient
from openai import OpenAI
import numpy as np

# 初始化客户端
milvus_client = MilvusClient("http://localhost:19530")
openai_client = OpenAI()

# 1. 用户查询
user_query = "What is the capital of France?"

# 2. 生成查询向量
response = openai_client.embeddings.create(
    model="text-embedding-3-small",
    input=user_query
)
query_vector = response.data[0].embedding

# 3. 向量检索
results = milvus_client.search(
    collection_name="documents",
    data=[query_vector],
    limit=5,
    filter="doc_type == 'knowledge_base'",  # 可选的标量过滤
    output_fields=["id", "title", "content"],
    consistency_level="Bounded"  # 推荐
)

# 4. 构建上下文
context = "\n\n".join([hit['entity']['content'] for hit in results[0]])

# 5. 生成回答
response = openai_client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {user_query}"}
    ]
)

print(response.choices[0].message.content)
```

### 3.2 批量数据导入模板

```python
from pymilvus import Collection, FieldSchema, CollectionSchema, DataType
import numpy as np

# 1. 创建 Collection
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=False),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
    FieldSchema(name="name", dtype=DataType.VARCHAR, max_length=100)
]
schema = CollectionSchema(fields=fields)
collection = Collection(name="my_collection", schema=schema)

# 2. 批量插入数据
batch_size = 10000
total_records = 100000

for i in range(0, total_records, batch_size):
    data = [
        {"id": j, "vector": np.random.rand(128).tolist(), "name": f"Item {j}"}
        for j in range(i, min(i + batch_size, total_records))
    ]
    collection.insert(data)
    print(f"Inserted {min(i + batch_size, total_records)} / {total_records} records")

# 3. 创建索引
index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="vector", index_params=index_params)
print("Index created")

# 4. Load Collection
collection.load()
print("Collection loaded")
```

### 3.3 数据更新模板

```python
# 1. 查询现有数据
results = milvus_client.query(
    collection_name="my_collection",
    ids=[1, 2, 3],
    output_fields=["id", "name", "price"]
)

# 2. 更新数据（使用 Upsert）
updated_data = [
    {"id": 1, "name": "Updated Name", "price": 99.99},  # 仅更新标量字段
    {"id": 2, "name": "Updated Name 2", "price": 199.99}
]

result = milvus_client.upsert(
    collection_name="my_collection",
    data=updated_data
)

print(f"Updated {result['upsert_count']} records")

# 3. 执行 Compaction（确保数据一致性）
from pymilvus import Collection
collection = Collection("my_collection")
collection.compact()
collection.wait_for_compaction_completed()
```

### 3.4 数据清理模板

```python
from pymilvus import Collection

collection = Collection("my_collection")

# 1. 删除过期数据
result = milvus_client.delete(
    collection_name="my_collection",
    filter="expired_at < '2026-01-01'"
)
print(f"Deleted {result['delete_count']} expired records")

# 2. 执行 Compaction 回收空间
collection.compact()
collection.wait_for_compaction_completed()
print("Compaction completed")

# 3. 验证删除结果
num_entities = collection.num_entities
print(f"Remaining entities: {num_entities}")
```

# 1. 从外部数据源获取数据
external_data = fetch_data_from_external_source()

for batch in batch_data(external_data, batch_size=1000):

## 5. 性能优化速查

### 5.1 批量操作优化

```python
# ✅ 推荐：批量插入
batch_size = 10000
for i in range(0, len(data), batch_size):
    batch = data[i:i+batch_size]
    milvus_client.insert(collection_name="my_collection", data=batch)

# ❌ 不推荐：单条插入
for item in data:
    milvus_client.insert(collection_name="my_collection", data=[item])
```

**性能提升**：10-100 倍

### 5.2 减少返回字段

```python
# ✅ 推荐：只返回需要的字段
results = milvus_client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    output_fields=["id", "name"]  # 不返回向量字段
)

# ❌ 不推荐：返回所有字段（包括向量）
results = milvus_client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    output_fields=["*"]  # 包括向量字段，影响性能
)
```

**性能提升**：2-5 倍

### 5.3 使用合适的一致性级别

```python
# ✅ 推荐：一般 RAG 应用使用 Bounded
results = milvus_client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Bounded"  # 平衡性能和准确性
)

# ❌ 不推荐：非必要时使用 Strong
results = milvus_client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Strong"  # 性能较低
)
```

**性能提升**：2-3 倍

## 6. 错误处理速查

### 6.1 连接错误

```python
from pymilvus import connections
import time

def connect_with_retry(max_retries=3):
    for i in range(max_retries):
        try:
            connections.connect("default", host="localhost", port="19530")
            return True
        except Exception as e:
            print(f"Connection failed: {e}")
            if i < max_retries - 1:
                time.sleep(5)
    return False
```

### 6.2 插入错误

```python
def insert_with_retry(collection, data, max_retries=3):
    for i in range(max_retries):
        try:
            return collection.insert(data)
        except Exception as e:
            print(f"Insert failed: {e}")
            if i < max_retries - 1:
                time.sleep(2)
            else:
                raise
```

### 6.3 Search 返回空结果

```python
# 确保 Collection 已加载
collection.load()

# 使用 Strong Consistency
results = milvus_client.search(
    collection_name="my_collection",
    data=[query_vector],
    limit=10,
    consistency_level="Strong"  # 确保读取最新数据
)
```

## 8. 总结

**掌握以下内容，就能开始使用 Milvus 2.6 的数据管理 CRUD 操作：**

**5 个核心操作**：
1. Insert - 批量插入数据
2. Search - 向量检索
3. Query - 精确查询
4. Delete - 删除数据
5. Upsert - 插入或更新（按需）

**5 个必须掌握的概念**：
1. Load/Release - Collection 必须 Load 才能检索
2. 一致性级别 - 推荐使用 Bounded
3. 批量操作 - 性能远优于单条操作
4. Compaction - 删除后需要 Compaction 回收空间
5. 标量过滤 - 在检索时添加过滤条件

**3 个性能优化技巧**：
1. 批量操作（10-100 倍提升）
2. 减少返回字段（2-5 倍提升）
3. 使用合适的一致性级别（2-3 倍提升）

- 构建基础的 RAG 文档问答系统
