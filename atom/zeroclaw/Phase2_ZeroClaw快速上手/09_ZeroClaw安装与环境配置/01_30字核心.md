## 30 字核心

这句话浓缩了安装配置的全部本质。下面逐词拆解。

---

## 一、为什么是"Rust 编译产物"？

### 1.1 编译型 vs 解释型：安装的本质差异

| 维度 | 解释型安装（npm install） | 编译型安装（cargo install） |
|------|------------------------|--------------------------|
| **代表** | Node.js / Python | Rust / Go / C++ |
| **安装产物** | 源码 + node_modules（几百 MB） | 单一二进制文件（~8.8 MB） |
| **运行时依赖** | 需要 Node.js 运行时 | 零运行时依赖 |
| **安装速度** | 快（下载即可） | 慢（需要编译）或快（预编译下载） |
| **运行速度** | JIT 编译，启动慢 | 原生机器码，启动 <10ms |

**前端类比：** `npm install -g typescript` 安装的是 TS 源码，运行时仍需 Node.js 解释执行。而 `cargo install zeroclaw` 安装的是一个编译好的原生二进制——就像你把一个 `.exe` 放到了 PATH 里，双击就跑，不需要任何运行时。

### 1.2 ZeroClaw 的安装产物有多小？

```
┌────────────────────────────────────────────────────────┐
│  ZeroClaw 安装产物：                                     │
│                                                         │
│  单个二进制文件                                           │
│  ├── 大小: ~8.8 MB（3 张高清照片）                        │
│  ├── 运行内存: <5 MB（比一个空白浏览器标签页还小）          │
│  ├── 启动时间: <10 ms（人类眨眼 300ms，它启动 30 次）      │
│  └── 运行时依赖: 零                                      │
│                                                         │
│  对比 Node.js 项目：                                     │
│  ├── node_modules: 200-500 MB                           │
│  ├── 运行内存: 50-100 MB                                 │
│  ├── 启动时间: 500ms-2s                                  │
│  └── 运行时依赖: Node.js (~100 MB)                       │
└────────────────────────────────────────────────────────┘
```

## 二、什么是"部署到目标平台"？

### 2.1 五种安装方式 = 五种部署路径

ZeroClaw 提供 5 种安装方式，覆盖 6 大平台（Linux/macOS/Windows × x86_64/ARM64/ARMv7）：

| 方式 | 命令 | 前端类比 | 适用场景 |
|------|------|---------|---------|
| 一键脚本 | `curl ... \| bash` | `npx create-react-app` | 快速体验 |
| Homebrew | `brew install zeroclaw` | `brew install node` | macOS/Linux 日常 |
| Cargo Install | `cargo install zeroclaw` | `npm install -g` | Rust 开发者 |
| Docker | `docker compose up -d` | Docker 部署前端 | 隔离/服务器 |
| 预编译二进制 | 下载 tar.gz 解压 | 下载 VS Code | 无 Rust 环境 |

[来源: reference/source_安装配置_01.md]

### 2.2 安装的本质：把二进制放到 PATH

```bash
# 不管哪种方式，最终结果都是：
which zeroclaw
# /usr/local/bin/zeroclaw  ← 一个可执行文件

zeroclaw --version
# zeroclaw v0.1.7          ← 直接运行，无需运行时
```

## 三、什么是"分层配置"？

### 3.1 四层配置优先级

```
环境变量 (ZEROCLAW_*) ──→ 最高优先级（临时覆盖）
        ↑
命令行参数 (--model xxx) ─→ 次高优先级（本次执行）
        ↑
config.toml              ─→ 持久化配置（项目级/全局级）
        ↑
默认值                    ─→ 最低优先级（内置在代码中）
```

**前端类比：** 这就像 CSS 优先级 `!important > inline style > ID > class > 标签`，或 Next.js 的环境变量覆盖：`.env.local > .env.production > .env`。

[来源: reference/source_安装配置_01.md]

### 3.2 配置决定"谁来演"

ZeroClaw 的 Trait 驱动架构意味着：**安装是"搭舞台"，配置是"选演员"。**

```toml
# config.toml — "选演员"
default_provider = "openrouter"     # Provider 选角：谁提供 AI 能力
default_model = "anthropic/claude-sonnet-4-6"  # 具体演员

[gateway]
port = 42617                        # Channel 入口：观众从哪里看

[security]
allowed_directories = ["/home/user/projects"]  # 安全策略：演出边界
```

## 四、什么是"驱动 Agent 运行"？

安装和配置的最终目的只有一个——让 Agent 跑起来：

```bash
# 安装 → 配置 → 运行 → Agent 开始工作
zeroclaw agent --message "帮我分析这个代码文件"
```

```
安装完成 + 配置就绪
        │
        v
┌─────────────────────────────────────────────────┐
│  ZeroClaw Agent 运行时                            │
│                                                   │
│  Provider (AI 模型) ←── config.toml 选定           │
│       ↕                                           │
│  Agent Loop (ReAct) ←── 内置核心循环               │
│       ↕                                           │
│  Tools (工具)       ←── 通过 Feature Flags 裁剪    │
│       ↕                                           │
│  Memory (记忆)      ←── 通过 config.toml 配置后端  │
│       ↕                                           │
│  Channel (通道)     ←── 通过 config.toml 选择接入   │
└─────────────────────────────────────────────────┘
```

## 五、记忆口诀

### 安装三步走

> **装二进制、写配置、跑 Agent。**

### 配置优先级

> **环境覆命令，命令覆文件，文件覆默认。**

### 一句话总结

> **安装是"搭舞台"（获取二进制），配置是"选演员"（决定 Provider/Channel/Tool/Memory 的具体实现），最终目的是让 AI Agent 在这个舞台上演出。**
