## 技巧1：流式解析性能优化 - JSON Patch diff 模式

### 场景

在 RAG 应用中，LLM 生成大型 JSON 对象（如包含多个文档的检索结果），流式传输时累积缓冲区导致内存占用高，用户体验差。

```python
# 问题场景：大型 JSON 对象流式解析
from langchain_core.output_parsers import JsonOutputParser
from langchain_openai import ChatOpenAI

model = ChatOpenAI()
parser = JsonOutputParser()
chain = model | parser

# 生成包含 100 个文档的 JSON
for chunk in chain.stream("Return 100 documents with title and summary"):
    print(chunk)
    # 输出: {}, {"docs": [{"title": "Doc1"}]}, {"docs": [{"title": "Doc1"}, {"title": "Doc2"}]}, ...
    # 问题：每次都传输完整对象，数据量大，延迟高
```

### 问题

**累积模式的性能瓶颈**：
- 每次输出都包含完整的 JSON 对象
- 数据传输量随对象大小线性增长
- 内存占用高，网络带宽浪费

### 解决方案

**使用 JSON Patch diff 模式**：

```python
from langchain_core.output_parsers import JsonOutputParser

# 启用 diff 模式
parser = JsonOutputParser(diff=True)
chain = model | parser

# 流式输出（diff 模式）
for patch in chain.stream("Return 100 documents"):
    print(patch)
    # 输出: [{"op": "add", "path": "/docs/0", "value": {"title": "Doc1"}}]
    # 输出: [{"op": "add", "path": "/docs/1", "value": {"title": "Doc2"}}]
    # 只传输差异，数据量小
```

**应用 JSON Patch**：

```python
import jsonpatch

current_state = {}
for patch in chain.stream("Return 100 documents"):
    # 应用 patch 到当前状态
    current_state = jsonpatch.apply_patch(current_state, patch)
    print(f"Current state: {current_state}")
```

### 原理

**JSON Patch 工作原理**（来源：source_outputparser_02_JSON解析器.md）：

```python
# JsonOutputParser 的 _diff() 方法
def _diff(self, prev: Any | None, next: Any) -> Any:
    return jsonpatch.make_patch(prev, next).patch
```

**性能对比**：

| 模式 | 数据传输量 | 内存占用 | 延迟 |
|------|-----------|---------|------|
| 累积模式 | O(n²) | 高 | 高 |
| diff 模式 | O(n) | 低 | 低 |

**适用场景**：
- 大型 JSON 对象（> 10KB）
- 网络带宽受限
- 需要实时更新 UI

**来源**：
- `reference/source_outputparser_02_JSON解析器.md` - JSON Patch 实现
- `reference/source_outputparser_04_转换解析器.md` - 流式解析机制

## 技巧2：错误处理三层防护 - 生产级鲁棒性

### 场景

生产环境中，LLM 输出格式不稳定，解析失败导致系统崩溃，影响用户体验。

```python
# 问题场景：解析失败导致系统崩溃
from langchain_core.output_parsers import PydanticOutputParser
from pydantic import BaseModel

class Person(BaseModel):
    name: str
    age: int

parser = PydanticOutputParser(pydantic_object=Person)
chain = model | parser

try:
    result = chain.invoke("Extract: Alice is thirty years old")
except Exception as e:
    print(f"系统崩溃: {e}")
    # 输出: 系统崩溃: Invalid json output: {"name": "Alice", "age": "thirty"}
```

### 问题

**单层错误处理的脆弱性**：
- LLM 输出格式不稳定（年龄是字符串而非数字）
- 解析失败直接抛出异常
- 没有重试或修复机制

### 解决方案

**三层防护策略**（来源：search_outputparser_02_流式解析错误处理.md）：

#### 第一层：Prompt 优化

```python
from langchain_core.prompts import ChatPromptTemplate

# 在 Prompt 中明确指定输出格式
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant. Always return valid JSON."),
    ("user", "{input}\n\n{format_instructions}")
])

format_instructions = parser.get_format_instructions()
chain = prompt | model | parser

# 提高成功率
result = chain.invoke({
    "input": "Extract: Alice is thirty years old",
    "format_instructions": format_instructions
})
```

#### 第二层：部分解析

```python
# 使用 partial=True 参数
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

# 流式解析时启用部分解析
for chunk in chain.stream("Extract: Alice is thirty years old"):
    if chunk:  # 检查是否有结果
        print(f"部分结果: {chunk}")
    # 即使解析失败，也不会抛出异常
```

#### 第三层：错误修复

```python
from langchain.output_parsers import OutputFixingParser, RetryOutputParser

# 方案1：OutputFixingParser（使用 LLM 修复）
fixing_parser = OutputFixingParser.from_llm(
    parser=PydanticOutputParser(pydantic_object=Person),
    llm=ChatOpenAI()
)

# 方案2：RetryOutputParser（重试机制）
retry_parser = RetryOutputParser.from_llm(
    parser=PydanticOutputParser(pydantic_object=Person),
    llm=ChatOpenAI(),
    max_retries=3
)

# 使用修复解析器
chain = model | fixing_parser
result = chain.invoke("Extract: Alice is thirty years old")
# 自动修复：{"name": "Alice", "age": "thirty"} → {"name": "Alice", "age": 30}
```

### 原理

**三层防护的协同作用**：

```
输入 → [第一层：Prompt 优化] → LLM 输出
                                    ↓
                        [第二层：部分解析] → 部分结果
                                    ↓
                        [第三层：错误修复] → 最终结果
```

**性能对比**：

| 策略 | 成功率 | 延迟 | 成本 |
|------|--------|------|------|
| 无防护 | 60% | 低 | 低 |
| Prompt 优化 | 80% | 低 | 低 |
| + 部分解析 | 90% | 中 | 低 |
| + 错误修复 | 99% | 高 | 高 |

**来源**：
- `reference/search_outputparser_02_流式解析错误处理.md` - 三层防护策略
- `reference/search_outputparser_01_Reddit最佳实践.md` - 社区最佳实践

## 技巧3：自定义 OutputParser 开发 - 处理特殊格式

### 场景

需要解析 LLM 输出的特殊格式（如 YAML、TOML、自定义格式），现有解析器不满足需求。

```python
# 问题场景：需要解析 YAML 格式
from langchain_openai import ChatOpenAI

model = ChatOpenAI()
result = model.invoke("Return a YAML config with database settings")
print(result.content)
# 输出:
# database:
#   host: localhost
#   port: 5432
#   name: mydb

# 现有解析器无法处理 YAML
```

### 问题

**现有解析器的局限性**：
- JsonOutputParser：只支持 JSON
- PydanticOutputParser：只支持 JSON + Pydantic 验证
- XMLOutputParser：只支持 XML
- 没有 YAML、TOML 等格式的解析器

### 解决方案

**自定义 OutputParser**（来源：source_outputparser_01_基础架构.md）：

```python
from langchain_core.output_parsers import BaseOutputParser
from typing import Any
import yaml

class YAMLOutputParser(BaseOutputParser[dict]):
    """自定义 YAML 解析器"""

    def parse(self, text: str) -> dict:
        """解析 YAML 格式的输出"""
        try:
            # 去除 Markdown 代码块
            if "```yaml" in text:
                text = text.split("```yaml")[1].split("```")[0]
            elif "```" in text:
                text = text.split("```")[1].split("```")[0]

            # 解析 YAML
            return yaml.safe_load(text.strip())
        except yaml.YAMLError as e:
            raise ValueError(f"Invalid YAML output: {text}") from e

    def get_format_instructions(self) -> str:
        """返回格式指令"""
        return """Please return the output in YAML format.
Example:
```yaml
database:
  host: localhost
  port: 5432
  name: mydb
```"""

    @property
    def _type(self) -> str:
        """返回解析器类型"""
        return "yaml"

# 使用自定义解析器
parser = YAMLOutputParser()
chain = model | parser

result = chain.invoke("Return a YAML config with database settings")
print(result)
# 输出: {'database': {'host': 'localhost', 'port': 5432, 'name': 'mydb'}}
```

**支持流式解析**：

```python
from langchain_core.output_parsers import BaseCumulativeTransformOutputParser

class StreamingYAMLOutputParser(BaseCumulativeTransformOutputParser[dict]):
    """支持流式解析的 YAML 解析器"""

    def parse_result(self, result, *, partial: bool = False) -> dict:
        """解析结果（支持部分解析）"""
        text = result[0].text

        if partial:
            try:
                return yaml.safe_load(text.strip())
            except yaml.YAMLError:
                return {}  # 返回空字典而不是抛出异常
        else:
            return yaml.safe_load(text.strip())

    def get_format_instructions(self) -> str:
        return """Please return the output in YAML format."""

# 使用流式解析器
parser = StreamingYAMLOutputParser()
chain = model | parser

for chunk in chain.stream("Return a YAML config"):
    print(f"Chunk: {chunk}")
```

### 原理

**自定义 OutputParser 的核心方法**（来源：source_outputparser_01_基础架构.md）：

```python
class BaseOutputParser(BaseGenerationOutputParser):
    """OutputParser 基类"""

    @abstractmethod
    def parse(self, text: str) -> T:
        """解析文本输出"""
        pass

    def get_format_instructions(self) -> str:
        """返回格式指令（可选）"""
        return ""

    @property
    def _type(self) -> str:
        """返回解析器类型（用于序列化）"""
        return "custom"
```

**实现步骤**：
1. 继承 `BaseOutputParser` 或 `BaseCumulativeTransformOutputParser`
2. 实现 `parse()` 方法（必需）
3. 实现 `get_format_instructions()` 方法（可选）
4. 实现 `_type` 属性（可选）

**来源**：
- `reference/source_outputparser_01_基础架构.md` - OutputParser 基础架构

## 技巧4：Markdown 包裹问题处理 - 自动检测与去除

### 场景

LLM 经常将 JSON 输出包裹在 Markdown 代码块中，导致 JsonOutputParser 解析失败。

```python
# 问题场景：Markdown 包裹导致解析失败
from langchain_core.output_parsers import JsonOutputParser

parser = JsonOutputParser()

# LLM 输出（包含 Markdown 代码块）
llm_output = """
Here is the JSON:
```json
{"name": "Alice", "age": 30}
```
"""

try:
    result = parser.parse(llm_output)
except Exception as e:
    print(f"解析失败: {e}")
    # 输出: 解析失败: Invalid json output: Here is the JSON:...
```

### 问题

**Markdown 包裹的常见形式**：
- ` ```json ... ``` `
- ` ``` ... ``` `
- ` ```python ... ``` `（错误的语言标记）

### 解决方案

**JsonOutputParser 已内置支持**（来源：source_outputparser_02_JSON解析器.md）：

```python
from langchain_core.utils.json import parse_json_markdown

# JsonOutputParser 内部使用 parse_json_markdown()
def parse_result(self, result, *, partial: bool = False):
    text = result[0].text
    text = text.strip()

    if partial:
        try:
            return parse_json_markdown(text)  # 自动处理 Markdown 包裹
        except JSONDecodeError:
            return None
    else:
        try:
            return parse_json_markdown(text)
        except JSONDecodeError as e:
            raise OutputParserException(f"Invalid json output: {text}") from e
```

**手动使用 parse_json_markdown()**：

```python
from langchain_core.utils.json import parse_json_markdown

# 手动解析包含 Markdown 的 JSON
llm_output = """
```json
{"name": "Alice", "age": 30}
```
"""

result = parse_json_markdown(llm_output)
print(result)
# 输出: {'name': 'Alice', 'age': 30}
```

**自定义解析器支持 Markdown**：

```python
from langchain_core.output_parsers import BaseOutputParser
from langchain_core.utils.json import parse_json_markdown

class CustomJSONParser(BaseOutputParser[dict]):
    """自定义 JSON 解析器（支持 Markdown 包裹）"""

    def parse(self, text: str) -> dict:
        """解析 JSON（自动处理 Markdown 包裹）"""
        try:
            return parse_json_markdown(text)
        except Exception as e:
            raise ValueError(f"Invalid JSON: {text}") from e

# 使用自定义解析器
parser = CustomJSONParser()
result = parser.parse("""
```json
{"name": "Alice", "age": 30}
```
""")
print(result)
# 输出: {'name': 'Alice', 'age': 30}
```

### 原理

**parse_json_markdown() 工作原理**：

```python
def parse_json_markdown(text: str) -> dict:
    """从 Markdown 代码块中提取 JSON"""

    # 1. 检测 Markdown 代码块
    if "```json" in text:
        # 提取 ```json ... ``` 中的内容
        text = text.split("```json")[1].split("```")[0]
    elif "```" in text:
        # 提取 ``` ... ``` 中的内容
        text = text.split("```")[1].split("```")[0]

    # 2. 去除空白字符
    text = text.strip()

    # 3. 解析 JSON
    return json.loads(text)
```

**支持的格式**：
- ` ```json ... ``` ` → 提取 JSON
- ` ``` ... ``` ` → 提取内容
- 纯 JSON → 直接解析

## 实战建议

### 1. 选择合适的优化策略

**决策树**：

```
是否需要流式解析大型 JSON？
├─ 是 → 使用 JSON Patch diff 模式
└─ 否 → 是否需要高鲁棒性？
    ├─ 是 → 使用三层防护策略
    └─ 否 → 是否需要特殊格式？
        ├─ 是 → 自定义 OutputParser
        └─ 否 → 使用现有解析器
```

### 2. 性能优化优先级

**优先级排序**：
1. **Prompt 优化**：成本低，效果好（成功率 +20%）
2. **Markdown 处理**：自动支持，无需额外工作
3. **部分解析**：流式场景必备
4. **JSON Patch diff**：大型对象场景（> 10KB）
5. **错误修复**：最后防线（成本高）

### 3. 生产环境检查清单

**部署前检查**：
- [ ] Prompt 中包含格式指令
- [ ] 启用部分解析（流式场景）
- [ ] 配置错误修复机制
- [ ] 测试 Markdown 包裹场景
- [ ] 监控解析成功率
- [ ] 设置超时和重试

## 避坑指南

### 1. 不要过度优化

**错误做法**：
```python
# ❌ 所有场景都使用 JSON Patch diff
parser = JsonOutputParser(diff=True)
# 小型 JSON 对象不需要 diff 模式
```

**正确做法**：
```python
# ✅ 根据对象大小选择模式
if estimated_size > 10_000:  # 10KB
    parser = JsonOutputParser(diff=True)
else:
    parser = JsonOutputParser()
```

### 2. 不要忽略错误处理

**错误做法**：
```python
# ❌ 没有错误处理
result = parser.parse(text)
```

**正确做法**：
```python
# ✅ 使用三层防护
try:
    result = parser.parse(text)
except OutputParserException as e:
    # 使用 OutputFixingParser 修复
    result = fixing_parser.parse(text)
```

### 3. 不要重复造轮子

**错误做法**：
```python
# ❌ 手动处理 Markdown 包裹
if "```json" in text:
    text = text.split("```json")[1].split("```")[0]
result = json.loads(text)
```

**正确做法**：
```python
# ✅ 使用内置工具
from langchain_core.utils.json import parse_json_markdown
result = parse_json_markdown(text)
```
