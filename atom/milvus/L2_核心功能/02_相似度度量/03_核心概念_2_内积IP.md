# 核心概念2：内积IP（Inner Product）

> 内积是测量向量点积的度量方式，同时考虑方向和长度，是推荐系统和归一化向量检索的首选

---

## 一句话定义

**内积（Inner Product，IP）是计算两个向量对应元素乘积之和的度量方式，值越大表示越相似，特别适合推荐系统和归一化向量场景。**

---

## 数学定义

### 公式

对于两个n维向量 **A = (a₀, a₁, ..., aₙ₋₁)** 和 **B = (b₀, b₁, ..., bₙ₋₁)**：

```
IP(A, B) = Σ(aᵢ × bᵢ)
         = a₀×b₀ + a₁×b₁ + ... + aₙ₋₁×bₙ₋₁
```

### 几何意义

内积可以表示为：

```
IP(A, B) = ||A|| × ||B|| × cos(θ)
```

其中：
- `||A||` 和 `||B||` 是向量的长度（模）
- `θ` 是两个向量之间的夹角

**物理意义**：A在B方向上的投影长度 × B的长度

---

## 核心特征

### 1. 值域特征

- **值域**：(-∞, +∞)（理论上无界）
- **实际值域**：[-1, 1]（Milvus归一化后）
- **相似性**：值越大，越相似
- **正值**：夹角 < 90°（同向）
- **负值**：夹角 > 90°（反向）
- **零值**：夹角 = 90°（正交）

### 2. 与COSINE的关系

**关键洞察**：对于归一化向量，IP = COSINE

```python
import numpy as np

# 原始向量
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

# 归一化
vec_a_norm = vec_a / np.linalg.norm(vec_a)
vec_b_norm = vec_b / np.linalg.norm(vec_b)

# 内积
ip = np.dot(vec_a_norm, vec_b_norm)

# 余弦相似度
cosine = np.dot(vec_a, vec_b) / (np.linalg.norm(vec_a) * np.linalg.norm(vec_b))

print(f"IP: {ip:.6f}")        # 0.974632
print(f"COSINE: {cosine:.6f}") # 0.974632
# 完全相同！
```

**来源**：[Milvus Official Docs](https://milvus.io/docs/metric.md)

> "If you use IP to calculate similarities between embeddings, you must normalize your embeddings. After normalization, the inner product equals cosine similarity."

---

## 详细解释

### 工作原理（步骤拆解）

**步骤1：对齐向量**

```
向量A: [1, 2, 3]
向量B: [2, 3, 4]
```

**步骤2：逐维度相乘**

```
乘积: [1×2, 2×3, 3×4] = [2, 6, 12]
```

**步骤3：求和**

```
内积: 2 + 6 + 12 = 20
```

### Python实现

```python
import numpy as np

def inner_product(a, b):
    """
    计算内积（点积）

    参数:
        a: numpy数组，向量A
        b: numpy数组，向量B

    返回:
        float: 内积值
    """
    # 方法1：手动计算
    products = a * b
    ip = np.sum(products)

    return ip

# 方法2：使用numpy内置函数
def inner_product_numpy(a, b):
    return np.dot(a, b)

# 示例
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

print(f"内积（手动）: {inner_product(vec_a, vec_b):.4f}")      # 20.0000
print(f"内积（numpy）: {inner_product_numpy(vec_a, vec_b):.4f}") # 20.0000
```

## Milvus中的内积

### 创建Collection时指定

```python
from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection

# 连接Milvus
connections.connect("default", host="localhost", port="19530")

# 定义Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=128)
]
schema = CollectionSchema(fields, description="内积示例")
collection = Collection(name="ip_demo", schema=schema)

# 创建索引（指定IP度量）
index_params = {
    "index_type": "HNSW",
    "metric_type": "IP",  # ← 指定内积
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="embedding", index_params=index_params)

print("✅ Collection创建成功，使用内积度量")
```

### 检索时使用

```python
# 加载Collection
collection.load()

# 准备查询向量（归一化）
query_embedding = np.array([[0.1, 0.2, 0.3, ...]])  # 128维
query_embedding = query_embedding / np.linalg.norm(query_embedding)

# 检索参数
search_params = {
    "metric_type": "IP",
    "params": {"ef": 100}
}

# 执行检索
results = collection.search(
    data=query_embedding,
    anns_field="embedding",
    param=search_params,
    limit=10,
    output_fields=["id"]
)

# 输出结果
for hits in results:
    for hit in hits:
        print(f"ID: {hit.id}, IP分数: {hit.distance:.4f}")
```

## 何时使用内积

### ✅ 适用场景

#### 1. 向量已归一化

```python
# 场景：文本Embedding（通常归一化）
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
embedding = model.encode("文本示例")

# 检查是否归一化
norm = np.linalg.norm(embedding)
print(f"向量长度: {norm:.6f}")  # 接近1.0

# ✅ 使用IP（等价于COSINE，但更快）
```

**原因**：归一化后IP = COSINE，但IP计算更快（省略归一化步骤）

**来源**：[Milvus CheatSheet](https://github.com/milvus-io/bootcamp/blob/master/bootcamp/MilvusCheatSheet.md)

> "Most search algorithms work best with normalized embeddings data. **'IP' (inner product) and 'COSINE' are equivalent when the vectors are normalized.**"

---

#### 2. 推荐系统（向量长度有意义）

```python
# 场景：用户-物品推荐
# 向量长度代表偏好强度或置信度

user_embedding = np.array([0.5, 0.3, 0.8])    # 用户偏好
item_embedding = np.array([0.6, 0.4, 0.7])    # 物品特征

# 使用IP
ip_score = np.dot(user_embedding, item_embedding)
print(f"推荐分数: {ip_score:.4f}")  # 0.86

# IP分数高 → 推荐该物品
```

**原因**：IP同时考虑方向（偏好匹配）和长度（偏好强度）

**来源**：[Zilliz Blog](https://zilliz.com/blog/similarity-metrics-for-vector-search)

> "In a recommender system based on collaborative filtering and matrix factorization, every user and every item has an embedding, and the model is learned such that the dot product between a user embedding and an item embedding is a good predictor of the rating."

---

#### 3. Embedding模型使用点积训练

```python
# 场景：模型训练时使用点积损失
# 推理时应使用相同的度量方式

# 示例：msmarco-bert-base-dot-v5
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('msmarco-bert-base-dot-v5')
query_emb = model.encode("查询文本")
doc_emb = model.encode("文档文本")

# 使用IP（模型训练时的度量）
ip_score = np.dot(query_emb, doc_emb)
```

**原因**：度量方式应与模型训练时的损失函数一致

**来源**：[Zilliz Blog](https://zilliz.com/blog/similarity-metrics-for-vector-search)

> "You're likely to encounter many Large Language Models (LLMs) that use dot product for training. For example, the **msmarco-bert-base-dot-v5** model on Hugging Face specifies the 'suitable scoring functions' to be only dot product."

---

### ❌ 不适用场景

#### 1. 使用图索引（HNSW）且向量未归一化

```python
# ❌ 不推荐：IP + HNSW + 未归一化向量

# 原因：IP不满足三角不等式
# 图索引（如HNSW）依赖三角不等式进行剪枝优化
# 使用IP可能导致检索质量下降
```

**来源**：[Zilliz Blog](https://zilliz.com/blog/similarity-metrics-for-vector-search)

> "One thing to keep in mind with Inner Product is that it doesn't follow the triangle inequality. Larger lengths (large magnitudes) are prioritized. This means we should be careful when using IP with Inverted File Index or a graph index like HNSW."

---

#### 2. 只关注语义方向，不关注长度

```python
# 场景：文档语义相似度（长度无意义）

doc1_embedding = np.array([0.1, 0.2, 0.3])
doc2_embedding = np.array([1.0, 2.0, 3.0])  # doc1的10倍

# IP分数
ip = np.dot(doc1_embedding, doc2_embedding)
print(f"IP: {ip:.4f}")  # 1.4000（受长度影响）

# COSINE分数
cosine = np.dot(doc1_embedding, doc2_embedding) / (
    np.linalg.norm(doc1_embedding) * np.linalg.norm(doc2_embedding)
)
print(f"COSINE: {cosine:.4f}")  # 1.0000（只看方向）

# ✅ 推荐使用COSINE
```

## 与其他度量的对比

### IP vs COSINE

| 维度 | 内积IP | 余弦相似度COSINE |
|-----|--------|-----------------|
| **考虑因素** | 方向 + 长度 | 只考虑方向 |
| **归一化向量** | 等价于COSINE ✅ | 推荐 ✅ |
| **未归一化向量** | 推荐 ✅ | 可用 |
| **值域** | (-∞, +∞) | [-1, 1] |
| **相似性** | 越大越相似 | 越大越相似 |
| **计算成本** | 低 | 中（需要归一化） |
| **推荐系统** | 推荐 ✅ | 不推荐 |

**示例对比**：

```python
# 两个向量：方向相同，长度不同
vec_a = np.array([1, 2, 3])
vec_b = np.array([2, 4, 6])  # vec_a的2倍

# 内积
ip = np.dot(vec_a, vec_b)
print(f"IP: {ip:.4f}")  # 28.0000（受长度影响）

# 余弦相似度
cosine = np.dot(vec_a, vec_b) / (np.linalg.norm(vec_a) * np.linalg.norm(vec_b))
print(f"COSINE: {cosine:.4f}")  # 1.0000（只看方向）
```

---

### IP vs L2

| 维度 | 内积IP | L2距离 |
|-----|--------|--------|
| **考虑因素** | 方向 + 长度 | 方向 + 长度 |
| **归一化向量** | 等价于COSINE | 不推荐 |
| **未归一化向量** | 推荐 ✅ | 推荐 ✅ |
| **值域** | (-∞, +∞) | [0, +∞) |
| **相似性** | 越大越相似 | 越小越相似 |
| **计算成本** | 低 | 中 |

**关系公式**（归一化向量）：

```
L2² = 2 - 2×IP
```

**示例验证**：

```python
# 归一化向量
vec_a = np.array([1, 2, 3])
vec_b = np.array([2, 3, 4])

vec_a_norm = vec_a / np.linalg.norm(vec_a)
vec_b_norm = vec_b / np.linalg.norm(vec_b)

# L2距离平方
l2_squared = np.sum((vec_a_norm - vec_b_norm) ** 2)

# 内积
ip = np.dot(vec_a_norm, vec_b_norm)

# 验证关系
print(f"L2²: {l2_squared:.6f}")
print(f"2 - 2×IP: {2 - 2*ip:.6f}")
print(f"差异: {abs(l2_squared - (2 - 2*ip)):.10f}")  # 接近0
```

## RAG应用场景

### 场景1：推荐系统（协同过滤）

**需求**：根据用户历史行为推荐相似物品

```python
from pymilvus import connections, Collection
import numpy as np

# 连接Milvus
connections.connect("default", host="localhost", port="19530")

# 创建推荐系统Collection（使用IP）
collection = Collection("recommendation_system")
collection.load()

# 用户偏好向量（未归一化，长度代表偏好强度）
user_preference = np.array([0.8, 0.5, 0.9, 0.3, ...])  # 128维

# 使用IP检索
search_params = {"metric_type": "IP", "params": {"ef": 100}}
results = collection.search(
    data=[user_preference],
    anns_field="item_embedding",
    param=search_params,
    limit=10
)

# 输出推荐物品
for hit in results[0]:
    print(f"物品ID: {hit.id}, 推荐分数: {hit.distance:.4f}")
```

**为什么使用IP**：
- 用户偏好向量未归一化
- 向量长度代表偏好强度（高置信度 vs 低置信度）
- IP同时考虑偏好方向和强度

### 场景2：语义检索（归一化Embedding）

**需求**：文档语义相似度检索

```python
# 使用归一化Embedding模型
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')

# 查询文本
query = "如何优化Milvus检索性能？"
query_embedding = model.encode(query)

# 检查归一化
norm = np.linalg.norm(query_embedding)
print(f"向量长度: {norm:.6f}")  # 接近1.0

search_params = {"metric_type": "IP", "params": {"ef": 100}}
results = collection.search(
    data=[query_embedding],
    anns_field="doc_embedding",
    param=search_params,
    limit=10
)

# 输出相似文档
for hit in results[0]:
    print(f"文档ID: {hit.id}, 相似度: {hit.distance:.4f}")
```

**为什么使用IP**：
- Embedding模型输出已归一化
- IP = COSINE（结果相同）
- IP计算更快（省略归一化步骤）

## 2025-2026最佳实践

### 1. 归一化向量优先使用IP

**决策流程**：

```python
def select_metric_for_normalized_vectors():
    """归一化向量的度量选择"""

    # 1. 检查是否归一化
    sample_embedding = model.encode("测试文本")
    norm = np.linalg.norm(sample_embedding)

    if abs(norm - 1.0) < 1e-6:
        # 归一化向量
        return "IP"  # 推荐 ✅（比COSINE更快）
    else:
        # 未归一化向量
        if has_magnitude_meaning():
            return "IP"  # 长度有意义
        else:
            return "COSINE"  # 只关注方向
```

> "💡👉🏼 **'COSINE' works best for most use cases.** Most search algorithms work best with normalized embeddings data. **'IP' (inner product) and 'COSINE' are equivalent when the vectors are normalized.**"

## 常见误区

### 误区1：IP总是比COSINE快 ❌

**错误观点**：\"IP计算更简单，总是比COSINE快\"

**正确理解**：
- 只有归一化向量时，IP才比COSINE快
- 未归一化向量时，两者计算成本相近
- 速度差异主要在归一化步骤

---

### 误区2：IP适合所有场景 ❌

**错误观点**：\"IP同时考虑方向和长度，比COSINE更全面\"

**正确理解**：
- 只有向量长度有意义时才使用IP
- 文本语义检索通常只关注方向，应使用COSINE
- 推荐系统才需要考虑长度（偏好强度）

---

### 误区3：IP不能用于图索引 ❌

**错误观点**：\"IP不满足三角不等式，完全不能用于HNSW\"

**正确理解**：
- 归一化向量 + IP + HNSW 是可行的
- 未归一化向量 + IP + HNSW 可能影响检索质量
- 关键是向量是否归一化
