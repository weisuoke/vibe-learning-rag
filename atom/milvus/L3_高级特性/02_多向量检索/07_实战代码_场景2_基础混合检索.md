# 实战代码 - 场景2：基础混合检索（RRFRanker）

## 场景说明

**应用场景**：文档问答系统中的混合检索

在实际的 RAG 系统中，我们经常需要同时使用多个向量字段进行检索：
- **稠密向量**：捕获语义关系（如 OpenAI Embedding）
- **稀疏向量**：精确关键词匹配（如 BM25）

使用 **RRFRanker（Reciprocal Rank Fusion）** 可以自动平衡多个检索结果，无需手动调整权重。

**RRF 算法原理**：
```
score = 1 / (k + rank)
```
- `k`：平滑参数，默认值为 60
- `rank`：结果在各自检索中的排名
- 融合多个检索结果的分数，自动平衡

**适用场景**：
- 不确定各向量字段的重要性
- 需要快速实现混合检索
- 希望自动平衡多个检索结果

```python
def create_collection_with_multi_vectors():
    """
    创建包含多个向量字段的 Collection
    - dense_vector: 稠密向量（FloatVector）
    - sparse_vector: 稀疏向量（SparseFloatVector）
    """
    client = MilvusClient(uri=MILVUS_URI)

    # 删除已存在的 Collection
    if client.has_collection(COLLECTION_NAME):
        client.drop_collection(COLLECTION_NAME)
        print(f"✓ 已删除旧的 Collection: {COLLECTION_NAME}")

    # 定义 Schema
    schema = client.create_schema(
        auto_id=True,
        enable_dynamic_field=False,
    )

    # 添加字段
    schema.add_field(
        field_name="id",
        datatype=DataType.INT64,
        is_primary=True,
        auto_id=True,
    )

    schema.add_field(
        field_name="text",
        datatype=DataType.VARCHAR,
        max_length=1000,
    )

    # 稠密向量字段
    schema.add_field(
        field_name="dense_vector",
        datatype=DataType.FLOAT_VECTOR,
        dim=DIM,
    )

    # 稀疏向量字段
    schema.add_field(
        field_name="sparse_vector",
        datatype=DataType.SPARSE_FLOAT_VECTOR,
    )

    print(f"✓ Schema 定义完成")
    print(f"  - 稠密向量字段: dense_vector (dim={DIM})")
    print(f"  - 稀疏向量字段: sparse_vector")

    # 创建 Collection
    client.create_collection(
        collection_name=COLLECTION_NAME,
        schema=schema,
    )

    print(f"✓ Collection 创建成功: {COLLECTION_NAME}")

    return client


# 执行创建
client = create_collection_with_multi_vectors()
```

```python
def generate_dense_vector(dim):
    """生成归一化的稠密向量"""
    vec = np.random.rand(dim).astype(np.float32)
    vec = vec / np.linalg.norm(vec)  # 归一化
    return vec.tolist()


def generate_sparse_vector():
    """
    生成稀疏向量（模拟 BM25 输出）
    稀疏向量格式：{index: value}
    """
    # 随机选择 5-10 个非零位置
    num_nonzero = random.randint(5, 10)
    indices = random.sample(range(1000), num_nonzero)

    # 生成稀疏向量字典
    sparse_dict = {}
    for idx in indices:
        sparse_dict[idx] = random.uniform(0.1, 1.0)

    return sparse_dict
```

### 7. 执行混合检索（RRFRanker）

```python
def hybrid_search_with_rrf(client, query_dense, query_sparse, top_k=10):
    """
    使用 RRFRanker 执行混合检索

    参数:
        client: MilvusClient 实例
        query_dense: 查询的稠密向量
        query_sparse: 查询的稀疏向量
        top_k: 返回结果数量
    """
    print(f"\n执行混合检索 (RRFRanker)...")
    print(f"  - Top K: {top_k}")

    # 1. 定义稠密向量检索请求
    dense_req = AnnSearchRequest(
        data=[query_dense],
        anns_field="dense_vector",
        param={"metric_type": "IP"},
        limit=top_k,
    )

    # 2. 定义稀疏向量检索请求
    sparse_req = AnnSearchRequest(
        data=[query_sparse],
        anns_field="sparse_vector",
        param={"metric_type": "IP"},
        limit=top_k,
    )

    # 3. 创建 RRF 排序器
    ranker = RRFRanker()

    # 4. 执行混合检索
    results = client.hybrid_search(
        collection_name=COLLECTION_NAME,
        reqs=[dense_req, sparse_req],
        ranker=ranker,
        limit=top_k,
        output_fields=["text"],
    )

    print(f"✓ 混合检索完成")
    print(f"  - 返回结果数: {len(results[0])}")

    return results


# 生成查询向量
query_dense_vector = generate_dense_vector(DIM)
query_sparse_vector = generate_sparse_vector()

# 执行混合检索
search_results = hybrid_search_with_rrf(
    client,
    query_dense_vector,
    query_sparse_vector,
    top_k=10,
)
```

## 代码解析

### 1. 多向量字段定义

```python
# 稠密向量字段
schema.add_field(

    datatype=DataType.FLOAT_VECTOR,
```

**关键点**：
- 一个 Collection 可以包含多个向量字段
- 稠密向量需要指定维度 `dim`
- 稀疏向量不需要指定维度（动态维度）

### 2. 索引创建要求

```python
# 所有向量字段都必须创建索引
dense_index_params.add_index(
    field_name="dense_vector",
    index_type="FLAT",
    metric_type="IP",
)

sparse_index_params.add_index(
    field_name="sparse_vector",
    index_type="SPARSE_INVERTED_INDEX",
    metric_type="IP",
)
```

**关键点**：
- **所有向量字段都必须创建索引**才能加载 Collection
- 稠密向量使用 `FLAT`、`IVF_FLAT`、`HNSW` 等索引
- 稀疏向量使用 `SPARSE_INVERTED_INDEX` 索引

### 3. AnnSearchRequest 定义

```python
dense_req = AnnSearchRequest(
    data=[query_dense],           # 查询向量
    anns_field="dense_vector",    # 向量字段名
    param={"metric_type": "IP"},  # 检索参数
    limit=top_k,                  # 返回结果数量
)
```

**关键点**：
- `data`：查询向量数据（列表形式）
- `anns_field`：要检索的向量字段名
- `param`：检索参数（metric_type、nprobe 等）
- `limit`：每个检索请求的返回数量

### 4. RRFRanker 使用

```python
# 创建 RRF 排序器
ranker = RRFRanker()

# 执行混合检索
results = client.hybrid_search(
    collection_name=COLLECTION_NAME,
    reqs=[dense_req, sparse_req],  # 多个检索请求
    ranker=ranker,                 # RRF 排序器
    limit=top_k,                   # 最终返回数量
    output_fields=["text"],        # 返回字段
)
```

**关键点**：
- `reqs`：多个 AnnSearchRequest 列表
- `ranker`：RRFRanker 实例
- `limit`：最终融合后的返回数量
- RRF 自动平衡多个检索结果，无需手动设置权重

### 5. RRF 算法原理

```
score = 1 / (k + rank)
```

**示例**：
- 假设 k=60（默认值）
- 稠密向量检索中排名第 1：score_dense = 1 / (60 + 1) = 0.0164
- 稀疏向量检索中排名第 3：score_sparse = 1 / (60 + 3) = 0.0159
- 最终分数：score_final = score_dense + score_sparse = 0.0323

**优势**：
- 自动平衡多个检索结果
- 不需要手动调整权重
- 对排名靠前的结果给予更高权重

## 最佳实践

### 1. 何时使用 RRFRanker

**适合场景**：
- 不确定各向量字段的重要性
- 需要快速实现混合检索
- 希望自动平衡多个检索结果
- 多模态检索（图片 + 文本）

**不适合场景**：
- 已知各向量字段的精确权重
- 需要精细控制融合策略
- 某个向量字段明显更重要

### 2. 索引选择建议

```python
# 稠密向量索引选择
- FLAT: 精确检索，适合小数据集（< 100万）
- IVF_FLAT: 平衡性能和召回率，适合中等数据集
- HNSW: 高性能检索，适合大数据集（> 100万）

# 稀疏向量索引
- SPARSE_INVERTED_INDEX: 稀疏向量专用索引
```

### 3. 性能优化

