# 化骨绵掌

## 概述

将 Gateway 启动与管理拆分成 10 个 2 分钟知识卡片，每个卡片独立完整，形成递进关系。

---

## 卡片 1：Gateway 是什么

**一句话：** Gateway 是 OpenClaw 的 WebSocket 服务器，负责多通道消息路由和 Agent 集成。

**举例：**
```bash
# 启动 Gateway
openclaw gateway

# Gateway 监听 18789 端口
# 接收来自 WhatsApp、Telegram 等通道的消息
# 路由到 Agent 处理
# 返回响应
```

**应用：** Gateway 是 OpenClaw 的控制平面核心，所有消息都通过 Gateway 路由。

---

## 卡片 2：前台启动模式

**一句话：** 前台启动将 Gateway 运行在当前终端，日志实时输出，适合开发调试。

**举例：**
```bash
# 前台启动
openclaw gateway --verbose

# 特点：
# - 日志实时可见
# - Ctrl+C 停止
# - 终端关闭进程停止
```

**应用：** 开发环境使用前台启动，快速迭代，实时查看日志。

---

## 卡片 3：守护进程模式

**一句话：** 守护进程将 Gateway 作为系统服务运行，独立于终端，支持自动重启。

**举例：**
```bash
# 安装守护进程
openclaw onboard --install-daemon

# 管理守护进程
openclaw gateway start
openclaw gateway stop
openclaw gateway restart
```

**应用：** 生产环境使用守护进程，24/7 运行，自动恢复。

---

## 卡片 4：端口配置

**一句话：** Gateway 默认监听 18789 端口，可通过配置或 CLI 参数修改。

**举例：**
```bash
# 使用默认端口
openclaw gateway

# 指定端口
openclaw gateway --port 8080

# 配置文件
{
  gateway: { port: 18789 }
}
```

**应用：** 多实例部署时使用不同端口避免冲突。

---

## 卡片 5：loopback 绑定模式

**一句话：** loopback 模式只监听 127.0.0.1，只允许本机访问，最安全。

**举例：**
```bash
# loopback 绑定（默认）
openclaw gateway --bind loopback

# 只有本机可以连接
# ws://127.0.0.1:18789
```

**应用：** 开发环境和单机部署使用 loopback，最高安全性。

---

## 卡片 6：lan 绑定模式

**一句话：** lan 模式监听所有接口（0.0.0.0），允许局域网访问，强制认证。

**举例：**
```bash
# lan 绑定 + 认证
openclaw gateway --bind lan --token <token>

# 局域网设备可以连接
# ws://192.168.1.100:18789
```

**应用：** 家庭网络或办公室内部使用 lan 模式，多设备访问。

---

## 卡片 7：tailnet 绑定模式

**一句话：** tailnet 模式只监听 Tailscale 接口，通过加密隧道实现安全远程访问。

**举例：**
```bash
# tailnet 绑定
openclaw gateway --bind tailnet --tailscale serve

# 通过 Tailscale 网络访问
# 端到端加密
```

**应用：** 跨网络远程访问使用 tailnet，安全加密。

---

## 卡片 8：日志系统

**一句话：** Gateway 提供控制台日志（实时）、文件日志（持久化）和会话转录（完整历史）。

**举例：**
```bash
# 控制台日志（前台模式）
openclaw gateway --verbose

# 文件日志（守护进程）
openclaw logs --follow

# 会话转录
~/.openclaw/agents/main/sessions/<sessionKey>/transcript.jsonl
```

**应用：** 开发时用控制台日志，生产时用文件日志，调试时查看会话转录。

---

## 卡片 9：多实例管理

**一句话：** 通过 profile 机制运行多个隔离的 Gateway 实例，使用不同端口和配置。

**举例：**
```bash
# 默认实例
openclaw gateway

# 开发实例
openclaw gateway --profile dev --port 18790

# 救援实例
openclaw gateway --profile rescue --port 18791
```

**应用：** 开发/生产环境隔离，或运行备用 Gateway。

---

## 卡片 10：故障排查流程

**一句话：** 使用 `openclaw doctor` 诊断问题，查看日志定位原因，前台启动详细调试。

**举例：**
```bash
# 1. 快速诊断
openclaw doctor

# 2. 查看日志
openclaw logs --tail 50

# 3. 前台调试
openclaw gateway --verbose

# 4. 安全审计
openclaw security audit
```

**应用：** 遇到问题时按此流程排查，快速定位和解决。
