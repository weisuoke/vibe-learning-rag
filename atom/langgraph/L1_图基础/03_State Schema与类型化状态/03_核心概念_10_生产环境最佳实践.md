# 核心概念10：生产环境最佳实践

> 在生产环境中使用State Schema的最佳实践和注意事项

## 概念定义

**生产环境最佳实践是在实际应用中使用State Schema时需要遵循的规范、模式和优化策略，确保系统的可靠性、性能和可维护性。**

## 状态设计原则

### 1. 最小化状态

```python
# ✓ 只保留必要的状态
class State(TypedDict):
    query: str
    answer: str

# ❌ 保留过多状态
class State(TypedDict):
    query: str
    answer: str
    temp_data1: dict
    temp_data2: list
    debug_info: dict
    # 太多不必要的字段
```

### 2. 明确状态生命周期

```python
class State(TypedDict):
    # 输入状态（整个流程）
    user_id: str
    query: str

    # 中间状态（特定节点间）
    retrieved_docs: list

    # 输出状态（返回给用户）
    answer: str
```

### 3. 避免大对象

```python
# ✓ 存储引用或ID
class State(TypedDict):
    document_ids: list[str]

# ❌ 存储完整对象
class State(TypedDict):
    documents: list[dict]  # 可能很大
```

## 性能优化

### 1. 使用TypedDict而非Pydantic

```python
# ✓ 性能更好
from typing import TypedDict

class State(TypedDict):
    messages: list

# ❌ 性能稍慢（除非需要验证）
from pydantic import BaseModel

class State(BaseModel):
    messages: list
```

### 2. 控制Reducer复杂度

```python
# ✓ 简单高效的Reducer
def simple_reducer(existing: list, new: list) -> list:
    return existing + new

# ❌ 复杂低效的Reducer
def complex_reducer(existing: list, new: list) -> list:
    # 复杂的去重、排序、过滤逻辑
    # 可能影响性能
    pass
```

### 3. 批量更新

```python
# ✓ 一次返回多个更新
def node(state: State) -> dict:
    return {
        "field1": value1,
        "field2": value2,
        "field3": value3
    }

# ❌ 多次小更新（低效）
# 避免在节点内部多次更新
```

## 错误处理

### 1. 验证输入

```python
def validate_input(state: State) -> dict:
    """验证输入状态"""
    if not state.get("query"):
        raise ValueError("query is required")
    if not state.get("user_id"):
        raise ValueError("user_id is required")
    return {}
```

### 2. 优雅降级

```python
def node(state: State) -> dict:
    try:
        result = risky_operation()
        return {"result": result}
    except Exception as e:
        # 记录错误但不中断流程
        return {
            "result": "default_value",
            "errors": [str(e)]
        }
```

### 3. 错误追踪

```python
class State(TypedDict):
    errors: Annotated[list, operator.add]
    warnings: Annotated[list, operator.add]

def node(state: State) -> dict:
    try:
        result = operation()
        return {"result": result}
    except Exception as e:
        return {
            "errors": [{
                "node": "node_name",
                "error": str(e),
                "timestamp": time.time()
            }]
        }
```

## 监控与日志

### 1. 状态日志

```python
class State(TypedDict):
    logs: Annotated[list, operator.add]

def node(state: State) -> dict:
    start_time = time.time()
    result = process()
    duration = time.time() - start_time

    return {
        "result": result,
        "logs": [{
            "node": "process",
            "duration": duration,
            "timestamp": start_time

```

```python
import time

def monitored_node(state: State) -> dict:
```

## 安全性

### 1. 敏感数据处理

```python
class State(TypedDict):
    user_id: str
    # ❌ 不要存储敏感信息
    # password: str
    # credit_card: str

    # ✓ 存储引用或加密后的数据
    user_token: str
```

### 2. 输入验证

```python
from pydantic import BaseModel, Field, validator

class State(BaseModel):
    user_id: str = Field(pattern=r"^user_\d+$")
    email: str = Field(pattern=r"^[\w\.-]+@[\w\.-]+\.\w+$")

    @validator("user_id")
    def validate_user_id(cls, v):
        if not v.startswith("user_"):
            raise ValueError("Invalid user_id format")
        return v
```

### 3. 权限检查

```python
def check_permissions(state: State) -> dict:
    """检查用户权限"""
    user_id = state["user_id"]
    if not has_permission(user_id, "access_resource"):
        raise PermissionError("Access denied")
    return {}
```

## 可维护性

### 1. 清晰的命名

```python

class ChatState(TypedDict):
    user_query: str
    llm_response: str
    conversation_history: list

# ❌ 模糊的命名
class State(TypedDict):
    q: str
    r: str
    h: list
```

### 2. 文档化

```python
class State(TypedDict):
    """
    聊天机器人状态

    Attributes:
        messages: 对话历史，按时间顺序排列
        user_id: 用户唯一标识符
        context: 当前对话上下文

    user_id: str
    context: str
    metadata: dict
```

### 3. 版本管理

```python
class StateV1(TypedDict):
    query: str
    answer: str

class StateV2(TypedDict):
    query: str
    answer: str
    metadata: dict  # 新增字段

# 迁移函数
def migrate_v1_to_v2(state_v1: StateV1) -> StateV2:
    return {
        **state_v1,
        "metadata": {}
    }
```

### 3. 超时控制

```python
def timeout_handler(signum, frame):
    raise TimeoutError("Operation timed out")

def node_with_timeout(state: State) -> dict:
    signal.signal(signal.SIGALRM, timeout_handler)
    signal.alarm(30)  # 30秒超时

    try:
        result = long_operation()
        signal.alarm(0)  # 取消超时
```

## 常见陷阱

### 1. 状态膨胀

```python
# ❌ 状态不断增长
class State(TypedDict):
    logs: Annotated[list, operator.add]  # 无限增长

# ✓ 控制状态大小
class State(TypedDict):
    logs: Annotated[list, keep_last_n(100)]
```

### 2. 并发问题

```python
# ❌ 无Reducer的并行更新
class State(TypedDict):
    status: str  # 并行更新会导致不确定结果

# ✓ 使用Reducer或避免并行更新
class State(TypedDict):
    statuses: Annotated[list, operator.add]
```

### 3. 内存泄漏

```python
# ❌ 保留大对象引用
class State(TypedDict):
    large_data: list  # 可能导致内存泄漏

# ✓ 及时清理或使用引用
class State(TypedDict):
    data_id: str  # 只保留ID
```
