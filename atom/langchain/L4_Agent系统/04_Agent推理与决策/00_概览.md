# Agent推理与决策 - 概览

> Agent 推理与决策是 LLM 从"文本生成器"变成"问题解决者"的核心机制——通过 Thought→Action→Observation 循环，让 LLM 能够分步思考、调用工具、基于反馈迭代，最终解决复杂问题。

---

## 知识点定位

- **所属层级**: L4_Agent系统 - 第 4 个知识点
- **重要程度**: 极高（Agent 智能行为的核心引擎）
- **核心文件**: `langchain/agents/react/agent.py`（ReAct Agent 工厂函数）
- **输出解析**: `langchain/agents/output_parsers/`（5 种解析器）
- **推理提示**: `langchain/agents/mrkl/prompt.py`、`langchain_core/prompts/`

### 为什么这个知识点重要？

LLM 本身是一个"一次性文本生成器"——给它一个问题，它生成一个回答，就结束了。但现实中的问题往往需要**多步推理**：先查资料，再分析，再计算，最后总结。

**推理与决策机制**就是让 LLM 具备这种"多步思考"能力的关键。它决定了：
1. Agent 如何**分解复杂问题**（推理）
2. Agent 如何**选择下一步行动**（决策）
3. Agent 如何**从工具反馈中学习**（迭代）
4. Agent 如何**知道何时停止**（终止判断）


如果说 AgentExecutor 是"引擎"，Tools 是"手脚"，那么**推理与决策就是"大脑"**——它决定了 Agent 怎么想、怎么选、怎么判断。

## 核心概念速览

本知识点包含 **5 个核心概念**，覆盖推理与决策的完整知识面：

### 概念 1: ReAct 推理模式

ReAct = Reasoning + Acting。LLM 在 Thought（思考）和 Action（行动）之间交替循环，通过 Observation（观察）获取真实信息反馈推理。这是 Agent 推理的经典范式。

### 概念 2: 思维链与 Scratchpad

Chain-of-Thought 让 LLM "展示推理过程"，Scratchpad（草稿纸）将历史推理步骤格式化后喂回 LLM，实现跨轮次的迭代推理记忆。

### 概念 3: 输出解析与决策提取

5 种 OutputParser 将 LLM 的自由文本输出解析为结构化决策：AgentAction（继续调用工具）或 AgentFinish（返回最终答案）。解析器是推理循环的"翻译官"。

### 概念 4: 推理提示工程

MRKL 四段式 Prompt 结构（prefix + tools + format_instructions + suffix）、Stop Sequence 防幻觉、Few-Shot 示例引导推理格式。Prompt 设计直接决定推理质量。

### 概念 5: 推理策略对比与选型

三种主流推理策略的全面对比：ReAct 文本推理（经典）、Tool Calling 工具调用（现代主流）、XML 推理（特定模型）。包含选型决策树和 2025-2026 趋势分析。

## 与 RAG 和 AI Agent 的关系

### RAG 场景中的推理

在 RAG 系统中，推理与决策让 Agent 能够**智能地编排检索和生成**：

```
用户: "对比 React 和 Vue 的状态管理方案，给出选型建议"
    │
    ▼
Thought: 这是一个对比类问题，需要分别检索两个框架的信息
    │
    ▼
Action: retriever_tool("React 状态管理 Redux Zustand")
Observation: React 生态有 Redux、Zustand、Jotai...
    │
    ▼
Thought: 拿到了 React 的信息，现在需要查 Vue 的
    │
    ▼
Action: retriever_tool("Vue 状态管理 Pinia Vuex")
Observation: Vue 3 推荐 Pinia，Vue 2 用 Vuex...
    │
    ▼
Thought: 两边信息都有了，可以做对比分析了
    │
    ▼
Final Answer: React 推荐 Zustand（轻量）或 Redux Toolkit（复杂场景）...
```

没有推理能力，RAG 只能做"一问一检索一回答"；有了推理，RAG 能做**多步检索、交叉分析、综合判断**。
