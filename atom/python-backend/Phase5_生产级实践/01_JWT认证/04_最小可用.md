## 核心知识点

### 4.1 JWT 的三部分结构

**一句话：** JWT 由 Header（头部）、Payload（载荷）、Signature（签名）三部分组成，用点号分隔。

```python
# JWT 格式
jwt_token = "Header.Payload.Signature"

# 实际例子
jwt_token = "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxMjMsImV4cCI6MTcwMDAwMDAwMH0.5K8v7J9mN2pQ3rT6wX8yZ1aB4cD7eF0gH"
#            ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
#            Header (Base64)                   Payload (Base64)                    Signature (Base64)
```

**三部分的作用：**
- **Header**：描述 Token 类型和签名算法
- **Payload**：存储用户信息和声明（如 user_id、过期时间）
- **Signature**：签名，防止 Token 被篡改

### 4.2 生成 JWT Token

**使用 python-jose 库生成 JWT：**

```python
from jose import jwt
from datetime import datetime, timedelta

# 配置
SECRET_KEY = "your-secret-key-keep-it-safe"  # 密钥，必须保密
ALGORITHM = "HS256"  # 签名算法

def create_access_token(user_id: int) -> str:
    """生成 Access Token"""
    # 1. 准备 Payload
    payload = {
        "user_id": user_id,
        "exp": datetime.utcnow() + timedelta(minutes=15)  # 15分钟后过期
    }

    # 2. 生成 JWT
    token = jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

    return token


# 使用示例
token = create_access_token(user_id=123)
print(f"生成的 Token: {token}")
```

**输出：**
```
生成的 Token: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2VyX2lkIjoxMjMsImV4cCI6MTcwOTI4MDAwMH0.abc123...
```

### 4.3 验证 JWT Token

**验证 Token 并解析用户信息：**

```python
from jose import jwt, JWTError
from fastapi import HTTPException

def verify_token(token: str) -> dict:
    """验证 Token 并返回 Payload"""
    try:
        # 解码并验证 Token
        payload = jwt.decode(
            token,
            SECRET_KEY,
            algorithms=[ALGORITHM]
        )
        return payload

    except jwt.ExpiredSignatureError:
        # Token 已过期
        raise HTTPException(status_code=401, detail="Token 已过期")

    except JWTError:
        # Token 无效（签名错误、格式错误等）
        raise HTTPException(status_code=401, detail="无效的 Token")


# 使用示例
try:
    payload = verify_token(token)
    print(f"用户 ID: {payload['user_id']}")
except HTTPException as e:
    print(f"验证失败: {e.detail}")
```

### 4.4 用户登录流程

**实现登录端点，验证密码并返回 JWT：**

```python
from fastapi import FastAPI, HTTPException
from passlib.context import CryptContext
from pydantic import BaseModel

app = FastAPI()

# 密码哈希工具
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 模拟用户数据库
fake_users_db = {
    "alice@example.com": {
        "user_id": 123,
        "username": "alice",
        "hashed_password": pwd_context.hash("password123")  # 实际中从数据库读取
    }
}


class LoginRequest(BaseModel):
    email: str
    password: str


@app.post("/login")
async def login(request: LoginRequest):
    """登录端点"""
    # 1. 查找用户
    user = fake_users_db.get(request.email)
    if not user:
        raise HTTPException(status_code=401, detail="用户名或密码错误")

    # 2. 验证密码
    if not pwd_context.verify(request.password, user["hashed_password"]):
        raise HTTPException(status_code=401, detail="用户名或密码错误")

    # 3. 生成 JWT
    access_token = create_access_token(user_id=user["user_id"])

    # 4. 返回 Token
    return {
        "access_token": access_token,
        "token_type": "bearer"
    }
```

**测试登录：**
```bash
curl -X POST "http://localhost:8000/login" \
  -H "Content-Type: application/json" \
  -d '{"email": "alice@example.com", "password": "password123"}'
```

**响应：**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

### 4.5 使用依赖注入保护路由

**创建依赖函数，自动验证 Token：**

```python
from fastapi import Depends, Header

async def get_current_user(authorization: str = Header()) -> dict:
    """
    依赖函数：从请求头中提取并验证 Token

    请求头格式：Authorization: Bearer <token>
    """
    # 1. 检查格式
    if not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="无效的认证格式")

    # 2. 提取 Token
    token = authorization.replace("Bearer ", "")

    # 3. 验证 Token
    payload = verify_token(token)

    return payload


# 受保护的端点
@app.get("/profile")
async def get_profile(current_user: dict = Depends(get_current_user)):
    """
    需要登录才能访问的端点
    FastAPI 会自动调用 get_current_user 验证 Token
    """
    return {
        "message": f"欢迎，用户 {current_user['user_id']}",
        "user_id": current_user["user_id"]
    }


# AI Agent 对话端点（需要认证）
@app.post("/agent/chat")
async def chat(
    message: str,
    current_user: dict = Depends(get_current_user)
):
    """
    AI Agent 对话端点，需要登录
    """
    user_id = current_user["user_id"]

    # 调用 AI Agent（这里简化）
    response = f"AI 回复给用户 {user_id}: {message}"

    return {"response": response}
```

**测试受保护端点：**
```bash
# 1. 先登录获取 Token
TOKEN=$(curl -X POST "http://localhost:8000/login" \
  -H "Content-Type: application/json" \
  -d '{"email": "alice@example.com", "password": "password123"}' \
  | jq -r '.access_token')

# 2. 使用 Token 访问受保护端点
curl -X GET "http://localhost:8000/profile" \
  -H "Authorization: Bearer $TOKEN"
```

**响应：**
```json
{
  "message": "欢迎，用户 123",
  "user_id": 123
}
```

## 完整最小可用示例

**将以上知识点整合成一个完整的示例：**

```python
"""
JWT 认证最小可用示例
演示：用户登录 → 获取 Token → 访问受保护端点
"""

from fastapi import FastAPI, Depends, HTTPException, Header
from jose import jwt, JWTError
from passlib.context import CryptContext
from pydantic import BaseModel
from datetime import datetime, timedelta

# ===== 配置 =====
SECRET_KEY = "your-secret-key-keep-it-safe"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 15

app = FastAPI()
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 模拟用户数据库
fake_users_db = {
    "alice@example.com": {
        "user_id": 123,
        "username": "alice",
        "hashed_password": pwd_context.hash("password123")
    }
}


# ===== 数据模型 =====
class LoginRequest(BaseModel):
    email: str
    password: str


class TokenResponse(BaseModel):
    access_token: str
    token_type: str = "bearer"


# ===== JWT 工具函数 =====
def create_access_token(user_id: int) -> str:
    """生成 Access Token"""
    payload = {
        "user_id": user_id,
        "exp": datetime.utcnow() + timedelta(minutes=ACCESS_TOKEN_EXPIRE_MINUTES)
    }
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)


def verify_token(token: str) -> dict:
    """验证 Token"""
    try:
        payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
        return payload
    except jwt.ExpiredSignatureError:
        raise HTTPException(status_code=401, detail="Token 已过期")
    except JWTError:
        raise HTTPException(status_code=401, detail="无效的 Token")


# ===== 依赖注入 =====
async def get_current_user(authorization: str = Header()) -> dict:
    """从请求头中提取并验证 Token"""
    if not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="无效的认证格式")

    token = authorization.replace("Bearer ", "")
    return verify_token(token)


# ===== 路由端点 =====

@app.post("/login", response_model=TokenResponse)
async def login(request: LoginRequest):
    """登录端点：验证密码并返回 JWT"""
    # 1. 查找用户
    user = fake_users_db.get(request.email)
    if not user:
        raise HTTPException(status_code=401, detail="用户名或密码错误")

    # 2. 验证密码
    if not pwd_context.verify(request.password, user["hashed_password"]):
        raise HTTPException(status_code=401, detail="用户名或密码错误")

    # 3. 生成并返回 Token
    access_token = create_access_token(user_id=user["user_id"])
    return TokenResponse(access_token=access_token)


@app.get("/profile")
async def get_profile(current_user: dict = Depends(get_current_user)):
    """受保护端点：需要登录"""
    return {
        "message": f"欢迎，用户 {current_user['user_id']}",
        "user_id": current_user["user_id"]
    }


@app.post("/agent/chat")
async def chat(
    message: str,
    current_user: dict = Depends(get_current_user)
):
    """AI Agent 对话端点：需要登录"""
    user_id = current_user["user_id"]

    # 这里简化 AI Agent 调用
    response = f"AI 回复给用户 {user_id}: 你说了 '{message}'"

    return {"response": response}


@app.get("/public")
async def public_endpoint():
    """公开端点：无需登录"""
    return {"message": "这是公开端点，任何人都能访问"}


# ===== 运行服务器 =====
if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

**运行示例：**

```bash

pip install fastapi uvicorn python-jose[cryptography] passlib[bcrypt]

# 2. 运行服务器
python app.py

# 3. 测试公开端点（无需登录）
curl http://localhost:8000/public

# 4. 测试登录
curl -X POST http://localhost:8000/login \
  -H "Content-Type: application/json" \
  -d '{"email": "alice@example.com", "password": "password123"}'

# 5. 使用 Token 访问受保护端点
TOKEN="<从登录响应中复制 access_token>"
curl http://localhost:8000/profile \
  -H "Authorization: Bearer $TOKEN"

# 6. 调用 AI Agent
curl -X POST "http://localhost:8000/agent/chat?message=你好" \
  -H "Authorization: Bearer $TOKEN"
```

## 常见问题

### Q1: SECRET_KEY 应该怎么设置？

```python
# ❌ 不要硬编码
SECRET_KEY = "my-secret-key"

# ✅ 从环境变量读取
import os
SECRET_KEY = os.getenv("SECRET_KEY")

# ✅ 生成随机密钥
import secrets
SECRET_KEY = secrets.token_urlsafe(32)
```

### Q2: Token 过期时间应该设置多久？

```python
# 推荐配置
ACCESS_TOKEN_EXPIRE_MINUTES = 15   # 短期，15分钟
REFRESH_TOKEN_EXPIRE_DAYS = 7      # 长期，7天（后续学习）

# 高安全场景
ACCESS_TOKEN_EXPIRE_MINUTES = 5    # 5分钟

# 内部工具
ACCESS_TOKEN_EXPIRE_MINUTES = 60   # 1小时
```

### Q3: 如何在前端存储 Token？

```javascript
// ❌ 不推荐：localStorage（容易被 XSS 攻击）
localStorage.setItem('access_token', token);

// ✅ 推荐：HttpOnly Cookie（后端设置）
// 前端无需手动存储，浏览器自动管理
```

### Q4: 如何处理 Token 过期？

```python
# 前端处理
async function callAPI() {
  try {
    const response = await fetch('/api/data', {
      headers: { 'Authorization': `Bearer ${token}` }
    });
    return response.json();
  } catch (error) {
    if (error.status === 401) {
      // Token 过期，跳转到登录页
      window.location.href = '/login';
    }
  }
}
```
