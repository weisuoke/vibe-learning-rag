# 实战代码 - 场景2：子图嵌套状态管理

## 场景说明

本场景演示 LangGraph 中父子图的状态管理——当一个图作为另一个图的节点时，如何处理两者之间的状态转换与通信。

LangGraph 提供两种子图集成方式：

1. **方式A：直接添加编译子图**——子图作为编译后的对象直接添加为节点，父子图通过"共享键"（同名字段）自动通信
2. **方式B：函数包装器**——用一个普通函数包装子图调用，手动完成父子图之间的状态转换

两种方式各有适用场景：方式A 简洁但要求父子图有共享字段；方式B 灵活但需要手动映射。

[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]
[来源: reference/source_多状态管理_01.md | LangGraph 源码分析]

---

## 核心知识点

| 知识点 | 说明 |
|--------|------|
| 子图状态隔离 | 子图拥有完全独立的 Channel 集合，与父图互不干扰 |
| 共享键通信 | 父子图中同名字段自动传递数据（方式A） |
| 私有键隔离 | 子图中不与父图同名的字段，父图完全看不到 |
| 函数包装器 | 手动调用子图并映射状态（方式B） |
| `builder.add_node("name", compiled_subgraph)` | 直接添加编译子图的语法 |
| 命名空间隔离 | 子图的 Checkpoint 在独立命名空间中，格式为 `node:task_id` |

## 完整代码

```python
"""
LangGraph 多状态管理 - 子图嵌套状态管理
演示：父子图状态转换的两种方式——共享键 vs 函数包装器

核心知识点：
- 方式A：直接添加编译子图，通过共享键自动通信
- 方式B：函数包装器，手动映射父子图状态
- 子图的私有字段对父图不可见
- 两种方式的适用场景对比

运行环境：Python 3.13+, langgraph
安装依赖：uv add langgraph
运行方式：python examples/langgraph/subgraph_state.py
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END


# ============================================================
# 方式A：直接添加编译子图（共享键通信）
# ============================================================

print("=" * 60)
print("=== 方式A：直接添加编译子图（共享键通信）===")
print("=" * 60)


# ----- 1. 定义子图的状态 -----

class SubgraphStateA(TypedDict):
    """子图状态

    foo: 与父图共享的键（同名同类型）
    bar: 子图私有的键（父图看不到）
    """
    foo: str    # 共享键：父图也有这个字段
    bar: str    # 私有键：只在子图内部使用


# ----- 2. 定义子图的节点 -----

def sub_node_1(state: SubgraphStateA) -> dict:
    """子图节点1：读取共享键 foo，写入私有键 bar"""
    print(f"    sub_node_1 收到: {state}")
    return {"bar": state["foo"] + " bar"}


def sub_node_2(state: SubgraphStateA) -> dict:
    """子图节点2：读取私有键 bar，写回共享键 foo"""
    print(f"    sub_node_2 收到: {state}")
    return {"foo": state["bar"] + " baz"}


# ----- 3. 构建并编译子图 -----

sub_builder_a = StateGraph(SubgraphStateA)
sub_builder_a.add_node("sub_node_1", sub_node_1)
sub_builder_a.add_node("sub_node_2", sub_node_2)
sub_builder_a.add_edge(START, "sub_node_1")
sub_builder_a.add_edge("sub_node_1", "sub_node_2")
sub_builder_a.add_edge("sub_node_2", END)

subgraph_a = sub_builder_a.compile()


# ----- 4. 定义父图的状态 -----

class ParentStateA(TypedDict):
    """父图状态

    foo: 与子图共享的键
    parent_data: 父图私有的键（子图看不到）
    """
    foo: str
    parent_data: str


# ----- 5. 定义父图的节点 -----

def parent_node_1(state: ParentStateA) -> dict:
    """父图节点1：准备数据"""
    print(f"  parent_node_1 收到: {state}")
    return {"foo": "hello from parent"}


def parent_node_3(state: ParentStateA) -> dict:
    """父图节点3：处理子图返回的结果"""
    print(f"  parent_node_3 收到: {state}")
    return {"parent_data": f"父图处理完成: {state['foo']}"}


# ----- 6. 构建父图，直接添加编译子图 -----

parent_builder_a = StateGraph(ParentStateA)
parent_builder_a.add_node("parent_node_1", parent_node_1)
parent_builder_a.add_node("subgraph", subgraph_a)     # 直接传入编译后的子图！
parent_builder_a.add_node("parent_node_3", parent_node_3)

parent_builder_a.add_edge(START, "parent_node_1")
parent_builder_a.add_edge("parent_node_1", "subgraph")
parent_builder_a.add_edge("subgraph", "parent_node_3")
parent_builder_a.add_edge("parent_node_3", END)

parent_graph_a = parent_builder_a.compile()


# ----- 7. 执行并观察 -----

print("\n--- 执行父图 ---")
result_a = parent_graph_a.invoke({
    "foo": "",
    "parent_data": "",
})

print(f"\n最终输出: {result_a}")
print(f"foo 的值: {result_a['foo']}")
print(f"parent_data 的值: {result_a['parent_data']}")
print("注意：子图的 bar 字段不在父图输出中——它是子图的私有状态！")


# ----- 8. stream 观察完整流程 -----

print("\n--- stream 观察完整流程 ---")
for step in parent_graph_a.stream({
    "foo": "",
    "parent_data": "",
}):
    for node_name, node_output in step.items():
        print(f"\n  节点 [{node_name}] 返回:")
        for key, value in node_output.items():
            print(f"    {key}: {value!r}")


# ============================================================
# 方式B：函数包装器（手动状态映射）
# ============================================================

print()
print("=" * 60)
print("=== 方式B：函数包装器（手动状态映射）===")
print("=" * 60)


# ----- 1. 定义子图的状态（与父图完全不同的字段名）-----

class SubgraphStateB(TypedDict):
    """子图状态：字段名与父图完全不同

    这种情况下不能用方式A（没有共享键），
    必须用函数包装器手动映射。
    """
    input_text: str
    processed_text: str


# ----- 2. 定义子图的节点 -----

def sub_process(state: SubgraphStateB) -> dict:
    """子图处理节点"""
    print(f"    sub_process 收到: {state}")
    text = state["input_text"]
    return {"processed_text": f"[已处理] {text.upper()}"}


# ----- 3. 构建并编译子图 -----

sub_builder_b = StateGraph(SubgraphStateB)
sub_builder_b.add_node("process", sub_process)
sub_builder_b.add_edge(START, "process")
sub_builder_b.add_edge("process", END)

subgraph_b = sub_builder_b.compile()


# ----- 4. 定义父图的状态 -----

class ParentStateB(TypedDict):
    """父图状态：字段名与子图完全不同"""
    query: str
    result: str
    step: str


# ----- 5. 定义父图的节点（包含函数包装器）-----

def prepare_node(state: ParentStateB) -> dict:
    """准备节点"""
    print(f"  prepare 收到: {state}")
    return {"step": "prepared"}


def call_subgraph(state: ParentStateB) -> dict:
    """函数包装器：手动调用子图并映射状态

    这是方式B的核心——在普通函数中：
    1. 从父图状态中提取数据
    2. 转换为子图的输入格式
    3. 调用子图
    4. 从子图输出中提取数据
    5. 转换为父图的更新格式
    """
    print(f"  call_subgraph 收到: {state}")

    # 步骤1-2：父图状态 → 子图输入
    sub_input = {"input_text": state["query"], "processed_text": ""}

    # 步骤3：调用子图
    print("  --- 进入子图 ---")
    sub_output = subgraph_b.invoke(sub_input)
    print("  --- 离开子图 ---")

    # 步骤4-5：子图输出 → 父图更新
    return {
        "result": sub_output["processed_text"],
        "step": "subgraph_done",
    }


def finalize_node(state: ParentStateB) -> dict:
    """最终处理节点"""
    print(f"  finalize 收到: {state}")
    return {
        "result": f"最终结果: {state['result']}",
        "step": "finalized",
    }


# ----- 6. 构建父图 -----

parent_builder_b = StateGraph(ParentStateB)
parent_builder_b.add_node("prepare", prepare_node)
parent_builder_b.add_node("subgraph", call_subgraph)    # 函数包装器作为普通节点
parent_builder_b.add_node("finalize", finalize_node)

parent_builder_b.add_edge(START, "prepare")
parent_builder_b.add_edge("prepare", "subgraph")
parent_builder_b.add_edge("subgraph", "finalize")
parent_builder_b.add_edge("finalize", END)

parent_graph_b = parent_builder_b.compile()


# ----- 7. 执行并观察 -----

print("\n--- 执行父图 ---")
result_b = parent_graph_b.invoke({
    "query": "hello langgraph",
    "result": "",
    "step": "init",
})

print(f"\n最终输出: {result_b}")

# 方式A vs 方式B 对比实验
# ============================================================

print()
print("=" * 60)
print("=== 方式A vs 方式B 对比 ===")
print("=" * 60)

print("""
┌─────────────────────┬──────────────────────┬──────────────────────┐
│ 对比维度            │ 方式A：直接添加子图   │ 方式B：函数包装器     │
├─────────────────────┼──────────────────────┼──────────────────────┤
│ 语法                │ add_node("n", graph) │ add_node("n", func)  │
│ 状态映射            │ 自动（共享键）        │ 手动（显式转换）      │
│ 前提条件            │ 父子图有同名字段      │ 无限制               │
│ 子图私有字段        │ 自动隔离             │ 自动隔离              │
│ 灵活性              │ 低（字段名耦合）      │ 高（完全解耦）        │
│ 代码量              │ 少                   │ 多（需要映射函数）     │
│ 子图可复用性        │ 需要字段名匹配        │ 任意子图都可接入      │
│ stream 可见性       │ 子图节点可见          │ 只看到包装函数        │
│ Checkpoint 支持     │ 子图有独立命名空间    │ 子图无独立命名空间     │
└─────────────────────┴──────────────────────┴──────────────────────┘
""")

# 示例3：共享键的数据流向详解
# ============================================================

print("=" * 60)
print("=== 示例3：共享键的数据流向详解 ===")
print("=" * 60)


class ChildState(TypedDict):
    """子图状态"""
    shared: str       # 共享键
    child_only: str   # 子图私有


class ParentState3(TypedDict):
    """父图状态"""
    shared: str        # 共享键
    parent_only: str   # 父图私有


def child_node(state: ChildState) -> dict:
    """子图节点：修改共享键和私有键"""
    print(f"    child_node 收到: {state}")
    print(f"    child_node 能看到 shared: {state['shared']!r}")
    print(f"    child_node 能看到 child_only: {state['child_only']!r}")
    return {
        "shared": state["shared"] + " → child修改",
        "child_only": "子图私有数据",
    }


child_builder = StateGraph(ChildState)
child_builder.add_node("child", child_node)
child_builder.add_edge(START, "child")
child_builder.add_edge("child", END)
child_graph = child_builder.compile()


def before_node(state: ParentState3) -> dict:
    print(f"  before 收到: {state}")
    return {"shared": "父图数据"}


def after_node(state: ParentState3) -> dict:
    print(f"  after 收到: {state}")
    print(f"  after 能看到 shared: {state['shared']!r}")
    print(f"  after 能看到 parent_only: {state['parent_only']!r}")
    # 注意：这里看不到 child_only！
    return {"parent_only": f"父图处理: {state['shared']}"}


parent_builder_3 = StateGraph(ParentState3)
parent_builder_3.add_node("before", before_node)
parent_builder_3.add_node("child_graph", child_graph)
parent_builder_3.add_node("after", after_node)

parent_builder_3.add_edge(START, "before")
parent_builder_3.add_edge("before", "child_graph")
parent_builder_3.add_edge("child_graph", "after")
parent_builder_3.add_edge("after", END)

parent_3 = parent_builder_3.compile()

print("\n--- 执行并观察共享键流向 ---")
result_3 = parent_3.invoke({"shared": "初始值", "parent_only": ""})
print(f"\n最终输出: {result_3}")
print(f"shared 经历了: '初始值' → '父图数据' → '父图数据 → child修改'")
print(f"child_only 对父图不可见，parent_only 对子图不可见")

# 子图使用 Input/Output Schema 控制边界
class SubInput(TypedDict):
    task: str

class SubOutput(TypedDict):
    result: str

class SubInternal(TypedDict):
    task: str
    result: str
    temp: str    # 内部临时字段


def sub_step_1(state: SubInput) -> dict:
    print(f"    sub_step_1 收到: {state}")
    return {"temp": f"处理中: {state['task']}"}


def sub_step_2(state: SubInternal) -> dict:
    print(f"    sub_step_2 收到: {state}")
    return {"result": f"完成: {state['temp']}"}


sub_builder_4 = StateGraph(
    SubInternal,
    input_schema=SubInput,
    output_schema=SubOutput,
)
sub_builder_4.add_node("step_1", sub_step_1)
sub_builder_4.add_node("step_2", sub_step_2)
sub_builder_4.add_edge(START, "step_1")
sub_builder_4.add_edge("step_1", "step_2")
sub_builder_4.add_edge("step_2", END)

sub_graph_4 = sub_builder_4.compile()


# 父图通过函数包装器调用带 Schema 的子图
class ParentState4(TypedDict):
    user_query: str
    final_answer: str


def parent_prepare(state: ParentState4) -> dict:
    print(f"  parent_prepare 收到: {state}")
    return {}


def parent_call_sub(state: ParentState4) -> dict:
    """函数包装器：调用带 Input/Output Schema 的子图"""
    print(f"  parent_call_sub 收到: {state}")

    # 只需要传 SubInput 的字段
    sub_result = sub_graph_4.invoke({"task": state["user_query"]})

    # sub_result 只包含 SubOutput 的字段
    print(f"  子图返回: {sub_result}")
    print(f"  注意：子图的 temp 字段不在返回值中！")

    return {"final_answer": sub_result["result"]}


parent_builder_4 = StateGraph(ParentState4)
parent_builder_4.add_node("prepare", parent_prepare)
parent_builder_4.add_node("call_sub", parent_call_sub)

parent_builder_4.add_edge(START, "prepare")
parent_builder_4.add_edge("prepare", "call_sub")
parent_builder_4.add_edge("call_sub", END)

parent_graph_4 = parent_builder_4.compile()

print("\n--- 执行 ---")
result_4 = parent_graph_4.invoke({
    "user_query": "分析这段文本",
    "final_answer": "",
})
print(f"\n最终输出: {result_4}")
print("双重隔离：子图的 temp 被 SubOutput 过滤，父图只拿到 result")

# 示例5：RAG 场景——检索子图 + 生成子图
# ============================================================

print()
print("=" * 60)
print("=== 示例5：RAG 场景——检索子图 + 生成子图 ===")
print("=" * 60)


# ----- 检索子图 -----

class RetrievalState(TypedDict):
    """检索子图的状态"""
    query: str              # 共享键：查询
    documents: list[str]    # 共享键：检索结果
    # 子图私有
    raw_results: list[str]
    scores: list[float]


def embed_node(state: RetrievalState) -> dict:
    """Embedding 节点"""
    query = state["query"]
    # 模拟向量检索
    raw = [f"doc_A({query})", f"doc_B({query})", f"doc_C({query})"]
    scores = [0.95, 0.82, 0.65]
    print(f"    embed: 检索到 {len(raw)} 个结果")
    return {"raw_results": raw, "scores": scores}


def rerank_node(state: RetrievalState) -> dict:
    """重排序节点"""
    # 只保留分数 > 0.8 的结果
    filtered = [
        doc for doc, score in zip(state["raw_results"], state["scores"])
        if score > 0.8
    ]
    print(f"    rerank: 过滤后保留 {len(filtered)} 个结果")
    return {"documents": filtered}


retrieval_builder = StateGraph(RetrievalState)
retrieval_builder.add_node("embed", embed_node)
retrieval_builder.add_node("rerank", rerank_node)
retrieval_builder.add_edge(START, "embed")
retrieval_builder.add_edge("embed", "rerank")
retrieval_builder.add_edge("rerank", END)

retrieval_graph = retrieval_builder.compile()


# ----- 生成子图 -----

class GenerationState(TypedDict):
    """生成子图的状态"""
    query: str              # 共享键
    documents: list[str]    # 共享键
    answer: str             # 共享键
    # 子图私有
    prompt_text: str


def build_prompt_node(state: GenerationState) -> dict:
    """构建 Prompt"""
    docs_text = "\n".join(state["documents"])
    prompt = f"问题：{state['query']}\n参考文档：\n{docs_text}\n请回答："
    print(f"    build_prompt: 构建了 {len(prompt)} 字符的 Prompt")
    return {"prompt_text": prompt}


def generate_answer_node(state: GenerationState) -> dict:
    """生成回答"""
    # 模拟 LLM 生成
    answer = f"基于 {len(state['documents'])} 篇文档的回答：{state['query']}的答案是..."
    print(f"    generate: 生成了回答")
    return {"answer": answer}


generation_builder = StateGraph(GenerationState)
generation_builder.add_node("build_prompt", build_prompt_node)
generation_builder.add_node("generate", generate_answer_node)
generation_builder.add_edge(START, "build_prompt")
generation_builder.add_edge("build_prompt", "generate")
generation_builder.add_edge("generate", END)

generation_graph = generation_builder.compile()


# ----- 父图：RAG Pipeline -----

class RAGPipelineState(TypedDict):
    """RAG Pipeline 状态：只包含共享键"""
    query: str
    documents: list[str]
    answer: str


def preprocess_query(state: RAGPipelineState) -> dict:
    """预处理查询"""
    cleaned = state["query"].strip().lower()
    print(f"  preprocess: '{state['query']}' → '{cleaned}'")
    return {"query": cleaned}


rag_builder = StateGraph(RAGPipelineState)
rag_builder.add_node("preprocess", preprocess_query)
rag_builder.add_node("retrieval", retrieval_graph)     # 检索子图
rag_builder.add_node("generation", generation_graph)   # 生成子图

rag_builder.add_edge(START, "preprocess")
rag_builder.add_edge("preprocess", "retrieval")
rag_builder.add_edge("retrieval", "generation")
rag_builder.add_edge("generation", END)

rag_pipeline = rag_builder.compile()

print("\n--- 执行 RAG Pipeline ---")
rag_result = rag_pipeline.invoke({
    "query": "  什么是向量数据库  ",
    "documents": [],
    "answer": "",
})

print(f"\n最终输出:")
print(f"  query: {rag_result['query']}")
print(f"  documents: {rag_result['documents']}")
print(f"  answer: {rag_result['answer']}")
print("\n关键观察：")
print("  - 检索子图的 raw_results、scores 对父图不可见")
print("  - 生成子图的 prompt_text 对父图不可见")
print("  - 父图只通过 query、documents、answer 三个共享键与子图通信")
```

## 代码解析

### 1. 方式A：共享键通信的数据流

```
父图 ParentStateA          子图 SubgraphStateA
┌──────────────────┐       ┌──────────────────┐
│ foo: str    ─────┼──────►│ foo: str (共享)   │
│ parent_data: str │       │ bar: str (私有)   │
└──────────────────┘       └──────────────────┘
        ▲                          │
        │    foo 的值自动回传       │
        └──────────────────────────┘
```

当使用 `builder.add_node("subgraph", compiled_subgraph)` 时：
- LangGraph 自动找到父子图中同名的字段（`foo`）
- 进入子图时：父图的 `foo` 值自动传给子图的 `foo`
- 离开子图时：子图的 `foo` 值自动回传给父图的 `foo`
- 子图的 `bar` 字段对父图完全不可见

### 2. 方式B：函数包装器的数据流

```
父图 ParentStateB          函数包装器           子图 SubgraphStateB
┌──────────────┐     ┌──────────────────┐     ┌──────────────────┐
│ query: str   │────►│ 手动映射：        │────►│ input_text: str  │
│ result: str  │◄────│ query→input_text  │◄────│ processed: str   │
│ step: str    │     │ processed→result  │     └──────────────────┘
└──────────────┘     └──────────────────┘
```

函数包装器的核心职责：
1. 从父图状态中提取需要的字段
2. 转换为子图期望的输入格式
3. 调用 `subgraph.invoke()`
4. 从子图输出中提取需要的字段
5. 转换为父图期望的更新格式

### 3. stream 可见性差异

方式A 的 stream 输出中，子图作为一个整体节点出现：
```python
# 方式A：stream 看到 "subgraph" 节点返回共享键的最终值
{"subgraph": {"foo": "hello from parent bar baz"}}
```

方式B 的 stream 输出中，只看到包装函数：
```python
# 方式B：stream 看到 "subgraph" 节点返回映射后的值
{"subgraph": {"result": "[已处理] HELLO LANGGRAPH", "step": "subgraph_done"}}
```

如果需要观察子图内部的执行过程，方式A 可以使用 `stream(subgraphs=True)` 参数。

### 4. Checkpoint 命名空间

方式A 中，子图有独立的命名空间：
```
""                          # 父图根命名空间
"subgraph:abc123"           # 子图命名空间
"subgraph:abc123|sub_node_1:def456"  # 子图内部节点
```

方式B 中，子图没有独立命名空间（因为它只是一个普通函数调用），子图的中间状态不会被 Checkpoint 记录。

---

## 关键要点

1. **方式A（直接添加）适用于**：父子图有共享字段、需要 Checkpoint 支持、需要 stream 观察子图内部
2. **方式B（函数包装器）适用于**：父子图字段名不同、需要复杂的状态转换逻辑、复用第三方子图
3. **共享键是自动的**：同名字段自动双向传递，无需任何配置
4. **私有键是隔离的**：子图的非共享字段对父图完全不可见，反之亦然
5. **双重隔离**：子图可以同时使用 Input/Output Schema + 共享键隔离，实现多层数据保护
6. **RAG 场景推荐**：检索和生成作为独立子图，通过 query/documents/answer 三个共享键通信

---

## 扩展思考

1. **如果父子图的共享键类型不同会怎样？** 编译时不会报错，但运行时可能因为类型不匹配导致意外行为。建议始终保持共享键的类型一致。

2. **子图能否修改父图的非共享字段？** 不能通过共享键机制修改。但可以使用 `Command(graph=Command.PARENT, update={...})` 显式更新父图状态。

3. **方式A 和方式B 能否混用？** 可以。同一个父图中，有些节点用方式A，有些用方式B，完全没问题。

4. **三层嵌套怎么处理？** 每层子图都是独立的 StateGraph，通过共享键或函数包装器逐层通信。命名空间会自动嵌套：`"level1:id1|level2:id2|level3:id3"`。
