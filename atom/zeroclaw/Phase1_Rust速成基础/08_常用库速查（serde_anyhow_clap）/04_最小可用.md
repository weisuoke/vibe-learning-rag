## 4.1 serde 最小可用

### 只需记住一行 derive

```rust
use serde::{Serialize, Deserialize};

#[derive(Serialize, Deserialize, Debug)]
struct Config {
    name: String,
    #[serde(default)]
    port: u16,
}
```

**TypeScript 对照**：这一行 `#[derive(Serialize, Deserialize)]` 等于同时完成了：
```typescript
// TypeScript 需要做三件事：
interface Config { name: string; port?: number; }  // 1. 类型定义
const config = JSON.parse(str);                     // 2. 解析（无类型安全）
const validated = configSchema.parse(config);        // 3. zod 校验
// Rust 一行 derive 全部搞定，而且编译时保证类型安全
```

### 三个最常用属性

日常开发 90% 的情况只用这三个：

```rust
#[derive(Serialize, Deserialize, Debug)]
struct AppConfig {
    name: String,

    #[serde(default)]              // ① 缺失时用默认值（最高频！）
    port: u16,                     //   port 缺失 → 0

    #[serde(rename = "api_key")]   // ② 字段名映射（对接外部 API）
    key: String,                   //   Rust 用 key，JSON/TOML 用 api_key

    #[serde(skip)]                 // ③ 跳过序列化（运行时计算的字段）
    cache: Vec<String>,            //   不读也不写
}
```

**速记口诀**：`default` 填缺省，`rename` 改名字，`skip` 不参与。

### 一个格式库就够用

```rust
// 方式一：从 TOML 配置文件读取（ZeroClaw 的主要方式）
let config: Config = toml::from_str(toml_str)?;

// 方式二：从 JSON 读取（API 响应）
let config: Config = serde_json::from_str(json_str)?;

// 方式三：写成 JSON
let json = serde_json::to_string(&config)?;
```

### 一个宏：快速构建 JSON

```rust
use serde_json::json;

// json! 宏 = JavaScript 对象字面量
let request = json!({
    "model": "gpt-4",
    "messages": [
        {"role": "user", "content": "Hello!"}
    ]
});
// 等价于 TS: const request = { model: "gpt-4", messages: [...] }
```

> **总结**：`derive` + `default/rename/skip` + `from_str` + `json!` = serde 日常全部。

## 4.2 anyhow 最小可用

### 函数返回 `anyhow::Result<T>`

```rust
use anyhow::{Result, Context, bail};

// ① 返回值用 anyhow::Result<T> — 能接住任何错误
fn load_config(path: &str) -> Result<String> {
    // ② 用 ? 自动传播错误
    let content = std::fs::read_to_string(path)?;
    Ok(content)
}
```

**TypeScript 对照**：
```typescript
// TS 的 try/catch 是可选的，不写也能编译
function loadConfig(path: string): string {
    return fs.readFileSync(path, 'utf-8');  // 可能抛异常，但编译器不管
}
// Rust 的 Result 是强制的 — 不处理 ? 就编译不过
```

### 用 `.context("说明")` 给错误加上下文

```rust
fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .context("读取配置文件失败")?;          // 出错时：读取配置文件失败 → 原始IO错误

    let config: Config = toml::from_str(&content)
        .context("解析配置文件失败")?;          // 出错时：解析配置文件失败 → 原始解析错误

    Ok(config)
}
```

**为什么重要？** 没有 `.context()` 你只看到 `No such file or directory`；加了 `.context()` 你看到 `读取配置文件失败: No such file or directory` — 瞬间知道哪一步出了问题。

### 用 `bail!` 提前返回错误

```rust
fn validate(config: &Config) -> Result<()> {
    if config.name.is_empty() {
        bail!("配置名称不能为空");     // 等价于 TS 的 throw new Error("...")
    }
    if config.port == 0 {
        bail!("端口号不能为 0");
    }
    Ok(())
}
```

### 用 `ensure!` 做条件断言

```rust
fn validate(config: &Config) -> Result<()> {
    // ensure! = if !condition { bail!(msg) } 的简写
    ensure!(!config.name.is_empty(), "配置名称不能为空");
    ensure!(config.port > 0, "端口号必须大于 0");
    Ok(())
}
```

> **总结**：`Result<T>` + `?` + `.context()` + `bail!` = anyhow 日常全部。

## 4.3 clap 最小可用

### 定义 `#[derive(Parser)]` struct

```rust
use clap::Parser;

#[derive(Parser, Debug)]
#[command(name = "mytool", version, about = "My CLI tool")]
struct Cli {
    /// The name to greet        ← 文档注释 = --help 文本
    #[arg(short, long)]          ← short: -n, long: --name
    name: String,

    /// Enable verbose output
    #[arg(short, long)]
    verbose: bool,

    /// Port number
    #[arg(short, long, default_value = "8080")]
    port: u16,
}

fn main() {
    let cli = Cli::parse();      // 一行完成解析！
    println!("Hello, {}!", cli.name);
}
```

**TypeScript 对照**：
```typescript
// yargs 需要链式调用
const argv = yargs(process.argv.slice(2))
    .option('name', { type: 'string', demandOption: true })
    .option('verbose', { type: 'boolean', default: false })
    .option('port', { type: 'number', default: 8080 })
    .parse();
// Rust clap：类型定义即参数定义，编译时保证类型安全
```

### 用 `#[command(subcommand)]` 定义子命令

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
struct Cli {
    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// Start the server
    Serve {
        #[arg(long, default_value = "8080")]
        port: u16,
    },
    /// Run in agent mode
    Agent {
        #[arg(long)]
        model: Option<String>,
    },
}

fn main() {
    let cli = Cli::parse();
    match cli.command {
        Commands::Serve { port } => println!("Serving on port {}", port),
        Commands::Agent { model } => println!("Agent mode, model: {:?}", model),
    }
}
```

使用方式：
```bash
$ mytool serve --port 3000
$ mytool agent --model gpt-4
$ mytool --help              # 自动生成帮助信息
```

> **总结**：`#[derive(Parser)]` struct + `#[derive(Subcommand)]` enum + `Cli::parse()` = clap 日常全部。

## 4.4 tracing 最小可用

### 五个日志宏（从低到高）

```rust
use tracing::{trace, debug, info, warn, error};

trace!("极细节日志，一般不开");           // 最低级别
debug!("调试信息: value = {}", val);      // 开发时用
info!("服务启动，端口 {}", port);         // 正常运行信息
warn!("警告：连接池快满了 {}/100", n);    // 需要注意
error!("数据库连接失败: {}", err);        // 出错了！
```

**TypeScript 对照**：
```typescript
// console.log 没有级别控制 → 生产环境满屏日志
console.log("debug info");    // 无法关闭
console.warn("warning");      // 无法过滤

// winston/pino 有级别但需要手动配置
logger.debug("debug info");   // ≈ debug!()
logger.info("server start");  // ≈ info!()
logger.warn("pool full");     // ≈ warn!()
logger.error("db failed");    // ≈ error!()
```

### 一行初始化

```rust
fn main() {
    // 方式一：最简初始化（默认输出 INFO 及以上）
    tracing_subscriber::fmt::init();

    // 方式二：支持环境变量控制级别（推荐）
    tracing_subscriber::fmt()
        .with_env_filter(
            tracing_subscriber::EnvFilter::from_default_env()
        )
        .init();

    info!("App started");
}
```

### 环境变量控制日志级别

```bash
# 只看 warn 和 error
RUST_LOG=warn cargo run

# 看 debug 及以上
RUST_LOG=debug cargo run

# 只看某个模块的 debug
RUST_LOG=my_app::config=debug cargo run

# 等价于 Node.js 的：
# LOG_LEVEL=debug node app.js
```

> **总结**：`info!/warn!/error!` + `fmt::init()` + `RUST_LOG` = tracing 日常全部。

## 4.5 综合最小可用示例

一个完整的小程序，同时使用四个库：

```rust
// Cargo.toml 依赖：
// serde = { version = "1.0", features = ["derive"] }
// serde_json = "1.0"
// toml = "1.0"
// anyhow = "1.0"
// clap = { version = "4.5", features = ["derive"] }
// tracing = "0.1"
// tracing-subscriber = { version = "0.3", features = ["env-filter"] }

use anyhow::{Context, Result, bail};
use clap::{Parser, Subcommand};
use serde::{Deserialize, Serialize};
use tracing::{info, warn, error};

// ===== serde: 配置结构 =====
#[derive(Serialize, Deserialize, Debug)]
struct Config {
    name: String,
    #[serde(default = "default_port")]
    port: u16,
    #[serde(default)]
    debug: bool,
}

fn default_port() -> u16 { 8080 }

// ===== clap: CLI 定义 =====
#[derive(Parser)]
#[command(name = "miniapp", version, about = "A mini demo app")]
struct Cli {
    /// Path to config file
    #[arg(short, long, default_value = "config.toml")]
    config: String,

    #[command(subcommand)]
    command: Commands,
}

#[derive(Subcommand)]
enum Commands {
    /// Start the server
    Serve,
    /// Show current config
    Show,
}

// ===== anyhow: 错误处理 =====
fn load_config(path: &str) -> Result<Config> {
    let content = std::fs::read_to_string(path)
        .context(format!("无法读取配置文件: {}", path))?;

    if content.trim().is_empty() {
        bail!("配置文件为空");
    }

    let config: Config = toml::from_str(&content)
        .context("解析 TOML 配置失败")?;

    Ok(config)
}

// ===== 主函数：四库协作 =====
fn main() -> Result<()> {
    // 1. tracing: 初始化日志
    tracing_subscriber::fmt()
        .with_env_filter(
            tracing_subscriber::EnvFilter::from_default_env()
        )
        .init();

    // 2. clap: 解析命令行参数
    let cli = Cli::parse();
    info!("CLI 参数解析完成");

    // 3. serde + anyhow: 加载配置
    let config = load_config(&cli.config)?;
    info!(name = %config.name, port = config.port, "配置加载成功");

    // 4. clap: 分发子命令
    match cli.command {
        Commands::Serve => {
            info!("启动服务器: {}:{}", config.name, config.port);
            // 实际启动服务器的代码...
        }
        Commands::Show => {
            let json = serde_json::to_string_pretty(&config)
                .context("序列化配置失败")?;
            println!("{}", json);
        }
    }

    Ok(())
}
```

使用方式：
```bash
# 启动服务
$ RUST_LOG=info cargo run -- serve --config app.toml

# 显示配置
$ cargo run -- show

# 查看帮助
$ cargo run -- --help
```

## 最小可用速查表

```
库         你只需要记住的                               TS 等价物
────────────────────────────────────────────────────────────────────
serde      #[derive(Serialize, Deserialize)]           interface + JSON.parse + zod
           #[serde(default/rename/skip)]               .default() / 字段映射
           serde_json::from_str / toml::from_str       JSON.parse()
           json!({...})                                 对象字面量 {...}

anyhow     fn foo() -> anyhow::Result<T>               async function foo(): Promise<T>
           .context("msg")?                             catch(e) { throw new Error("msg", {cause:e}) }
           bail!("msg")                                 throw new Error("msg")
           ensure!(cond, "msg")                         if (!cond) throw new Error("msg")

clap       #[derive(Parser)] struct Cli                yargs 定义
           #[arg(short, long)]                         .option('name', {...})
           #[derive(Subcommand)] enum Commands         .command('serve', ...)
           Cli::parse()                                .parse()

tracing    info!() / warn!() / error!()                 logger.info/warn/error()
           tracing_subscriber::fmt::init()              winston.createLogger()
           RUST_LOG=debug                               LOG_LEVEL=debug
```
