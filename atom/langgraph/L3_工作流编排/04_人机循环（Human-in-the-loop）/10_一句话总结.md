# 10_一句话总结

> 用一句话总结 LangGraph 人机循环（Human-in-the-loop）的核心本质

---

## 一句话总结

**LangGraph 人机循环通过 `interrupt(value)` 在节点内暂停图执行并传递上下文给人类，借助 Checkpoint 持久化状态实现跨时间等待，最终由 `Command(resume=value)` 注入人类决策恢复运行，让 AI 工作流在关键节点获得人类判断力。**

---

## 展开解读

### 关键词拆解

**interrupt(value) 暂停并传递上下文**
节点内调用 `interrupt()` 抛出 `GraphInterrupt` 异常，图执行立即暂停。value 参数携带中断原因和上下文信息发送给客户端，让人类清楚知道"AI 在问什么"。这是整个 HITL 机制的触发点。

**Checkpoint 持久化状态**
中断发生时，Checkpointer 自动将图的完整状态（节点执行进度、State 数据）写入存储。这意味着即使服务重启、等待数小时甚至数天，工作流状态也不会丢失。没有 Checkpointer，interrupt 根本无法工作。

**Command(resume=value) 注入决策恢复运行**
人类做出决策后，通过 `Command(resume=value)` 将决策值传回图。恢复时节点从头重新执行，`interrupt()` 不再暂停而是直接返回 resume 值，工作流继续向下游节点推进。

**关键节点获得人类判断力**
HITL 的核心价值不是让人类做所有事，而是让 AI 在"高风险、高不确定性"的决策点停下来请求人类介入——审批转账、确认工具调用、验证生成内容。这是 AI 安全与可控的关键保障。

### 记忆口诀

```
interrupt 问人停一停，
checkpoint 存档不会丢，
Command resume 接着走，
人机协作最佳流。
```

### 核心公式

```
HITL = interrupt(传值暂停) + Checkpoint(状态持久化) + Command(resume恢复)
```

展开为执行时序：

```
graph.invoke(input, config)
  → 节点执行到 interrupt(value)
  → GraphInterrupt 异常冒泡
  → Checkpointer 保存状态
  → 返回 __interrupt__ 给客户端
  → 人类查看 value，做出决策
  → graph.invoke(Command(resume=决策), config)
  → 节点从头重新执行，interrupt() 返回 resume 值
  → 后续节点继续运行
```
