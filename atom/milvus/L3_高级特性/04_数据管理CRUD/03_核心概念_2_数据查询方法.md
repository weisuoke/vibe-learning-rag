# 核心概念 2：数据查询方法（Query）

> 本文档详细讲解 Milvus 2.6 中的 Query 精确查询操作

---

## 概述

Query 是 Milvus 中基于主键或标量过滤表达式的精确查询操作，与向量相似度检索（Search）不同，Query 用于获取满足特定条件的实体数据。

---

## 1. Query 的定义

**Query 是基于主键或标量字段的精确查询操作，返回满足条件的完整实体数据。**

### 核心特征

1. **精确匹配**：基于主键或标量字段的精确查询，不涉及向量相似度计算
2. **完整返回**：返回满足条件的完整实体数据（包括向量和标量字段）
3. **高性能**：基于索引的查询，性能优于全表扫描
4. **灵活过滤**：支持复杂的布尔表达式过滤

### 与 Search 的区别

| 特性 | Query | Search |
|------|-------|--------|
| **查询方式** | 基于主键/标量字段 | 基于向量相似度 |
| **返回结果** | 精确匹配的实体 | 相似度排序的实体 |
| **性能特点** | 索引查询，快速 | 向量计算，相对慢 |
| **应用场景** | 精确查找、元数据过滤 | 语义检索、相似度匹配 |
| **是否需要向量** | 不需要 | 需要 |

## 2. Query 的核心概念

### 2.1 基于主键的查询

**通过主键 ID 直接查询实体数据。**

```python
from pymilvus import MilvusClient

client = MilvusClient("http://localhost:19530")

# 基于主键 ID 查询
results = client.query(
    collection_name="my_collection",
    ids=[1, 2, 3],  # 主键 ID 列表
    output_fields=["id", "vector", "name", "price"]
)

for result in results:
    print(f"ID: {result['id']}, Name: {result['name']}, Price: {result['price']}")
```

**特点**：
- 最快的查询方式
- 直接通过主键索引定位
- 适合已知 ID 的场景

### 2.2 基于表达式的查询

**通过标量字段的过滤表达式查询实体数据。**

```python
# 基于表达式查询
results = client.query(
    collection_name="my_collection",
    filter="price > 100 and status == 'active'",  # 过滤表达式
    output_fields=["id", "name", "price", "status"],
    limit=10  # 限制返回数量
)

for result in results:
    print(f"ID: {result['id']}, Name: {result['name']}, Price: {result['price']}")
```

**支持的表达式**：
- 比较运算符：`>`, `<`, `>=`, `<=`, `==`, `!=`
- 逻辑运算符：`and`, `or`, `not`
- 成员运算符：`in`, `not in`
- 字符串匹配：`like`

### 2.3 输出字段选择

**通过 `output_fields` 参数选择返回的字段。**

```python
# 只返回特定字段
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    output_fields=["id", "name", "price"]  # 只返回这些字段
)

# 返回所有字段（包括向量）
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    output_fields=["*"]  # 通配符，返回所有字段
)
```

**注意事项**：
- 默认不返回向量字段（除非显式指定）
- 使用 `*` 返回所有字段（包括向量）
- 减少返回字段可以提升性能

## 3. Query 的实现原理

### 3.1 查询流程

```
用户请求
   ↓
Query Coordinator（查询协调器）
   ↓
Query Node（查询节点）
   ↓
索引查找 + 数据读取
   ↓
结果返回
```

### 3.2 索引加速

**Query 操作依赖标量字段索引加速查询。**

```python
from pymilvus import Collection, FieldSchema, CollectionSchema, DataType

# 创建 Collection 时定义标量索引
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True),
    FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
    FieldSchema(name="price", dtype=DataType.FLOAT),
    FieldSchema(name="status", dtype=DataType.VARCHAR, max_length=50)
]

schema = CollectionSchema(fields=fields)
collection = Collection(name="my_collection", schema=schema)

# 为标量字段创建索引
collection.create_index(
    field_name="price",
    index_params={"index_type": "STL_SORT"}  # 标量索引
)

collection.create_index(
    field_name="status",
    index_params={"index_type": "INVERTED"}  # 倒排索引
)
```

**索引类型**：
- **STL_SORT**：适合数值字段（Int, Float）
- **INVERTED**：适合字符串字段（VarChar）
- **TRIE**：适合前缀匹配

### 3.3 一致性级别

**Query 操作支持不同的一致性级别。**

```python
# Strong Consistency（强一致性）
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    consistency_level="Strong"  # 保证读取最新数据
)

# Bounded Consistency（有界一致性）
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    consistency_level="Bounded"  # 允许一定延迟
)

# Eventually Consistency（最终一致性）
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    consistency_level="Eventually"  # 性能最高，但可能读到旧数据
)
```

**一致性级别对比**：

| 一致性级别 | 数据新鲜度 | 性能 | 适用场景 |
|-----------|-----------|------|---------|
| **Strong** | 最新数据 | 较低 | 金融、交易系统 |
| **Bounded** | 允许延迟 | 中等 | 一般应用 |
| **Eventually** | 可能旧数据 | 最高 | 分析、统计 |

## 4. Query 的高级用法

### 4.1 分页查询

```python
# 分页查询
page_size = 100
offset = 0

while True:
    results = client.query(
        collection_name="my_collection",
        filter="price > 100",
        output_fields=["id", "name", "price"],
        limit=page_size,
        offset=offset  # 偏移量
    )

    if not results:
        break

    for result in results:
        print(f"ID: {result['id']}, Name: {result['name']}")

    offset += page_size
```

### 4.2 复杂表达式查询

```python
# 复杂表达式
results = client.query(
    collection_name="my_collection",
    filter="""
        (price > 100 and price < 500)
        and status in ['active', 'pending']
        and name like 'Product%'
    """,
    output_fields=["id", "name", "price", "status"]
)
```

### 4.3 动态字段查询

```python
# 查询动态字段（Milvus 2.6 支持）
results = client.query(
    collection_name="my_collection",
    filter="metadata['category'] == 'electronics'",  # 动态字段过滤
    output_fields=["id", "name", "metadata"]
)

for result in results:
    print(f"ID: {result['id']}, Metadata: {result['metadata']}")
```

## 5. Query 的性能优化

### 5.1 创建标量索引

```python
# 为常用查询字段创建索引
collection.create_index(
    field_name="price",
    index_params={"index_type": "STL_SORT"}
)

collection.create_index(
    field_name="status",
    index_params={"index_type": "INVERTED"}
)
```

### 5.2 减少返回字段

```python
# 只返回需要的字段
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    output_fields=["id", "name"]  # 不返回向量字段
)
```

### 5.3 使用合适的一致性级别

```python
# 分析场景使用 Eventually Consistency
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    consistency_level="Eventually"  # 性能最高
)
```

```python

results = client.query(
    collection_name="documents",
    filter="doc_type == 'pdf' and created_at > '2025-01-01'",
```

```python
    collection_name="my_collection",
```

## 7. Query 的常见问题

### 7.1 Query 返回空结果

**原因**：
- 数据未加载到内存（未执行 Load）
- 一致性级别导致数据不可见
- 过滤表达式错误

**解决方案**：
```python
# 确保 Collection 已加载
collection.load()

# 使用 Strong Consistency
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    consistency_level="Strong"
)
```

### 7.2 Query 性能慢

**原因**：
- 未创建标量索引
- 返回字段过多（包括向量）
- 过滤条件复杂

**解决方案**：
```python
# 创建索引
collection.create_index(
    field_name="price",
    index_params={"index_type": "STL_SORT"}
)

# 减少返回字段
results = client.query(
    collection_name="my_collection",
    filter="price > 100",
    output_fields=["id", "name"]  # 不返回向量
)
```

### 7.3 Query 表达式语法错误

**常见错误**：
```python
# ❌ 错误：使用 Python 语法
filter="price > 100 && status == 'active'"  # && 不支持

# ✅ 正确：使用 Milvus 表达式语法
filter="price > 100 and status == 'active'"
```

## 8. 源码实现（Go 客户端）

根据 `client/milvusclient/read.go` 的源码分析：

```go
func (c *Client) Query(ctx context.Context, option QueryOption, callOptions ...grpc.CallOption) (ResultSet, error) {
    startTime := time.Now()
    collectionName := option.CollectionName()
    result := ResultSet{}

    // 获取 Collection 信息
    collection, err := c.getCollection(ctx, option.CollectionName())
    if err != nil {
        return result, err
    }

    // 构建 Query 请求
    req, err := option.QueryRequest(collection)
    if err != nil {
        return result, err
    }

    // 调用 gRPC 服务
    err = c.callService(func(milvusService milvuspb.MilvusServiceClient) error {
        resp, err := milvusService.Query(ctx, req, callOptions...)
        err = merr.CheckRPCCall(resp, err)
        if err != nil {
            return err
        }

        // 解析结果
        result, err = c.handleQueryResult(collection.Schema, resp)
        return err
    })

    // 记录操作
    c.recordOperation("Query", collectionName, startTime, err)
    return result, err
}
```

**关键点**：
- Query 返回单个 `ResultSet`（与 Search 返回 `[]ResultSet` 不同）
- 基于主键或表达式的精确查询
- 支持输出字段选择

## 9. 总结

**Query 是 Milvus 中基于主键或标量字段的精确查询操作，具有以下特点：**

1. **精确匹配**：基于主键或标量字段的精确查询
2. **高性能**：依赖标量索引加速查询
3. **灵活过滤**：支持复杂的布尔表达式
4. **完整返回**：返回满足条件的完整实体数据
5. **一致性保证**：支持多种一致性级别

**与 Search 的互补**：
- Query 用于精确查找和元数据过滤
- Search 用于语义检索和相似度匹配
- 两者结合可以实现强大的混合检索
