## 4.1 基础状态定义与传递

### 核心概念

**状态（State）** 是节点间共享数据的载体，通过 TypedDict 定义。

**关键点：**
- 使用 TypedDict 定义状态结构
- 节点返回部分状态更新
- 框架自动合并状态

### 代码示例

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

# 1. 定义状态
class State(TypedDict):
    messages: Annotated[list, add_messages]  # 使用 reducer
    user_name: str
    step_count: int

# 2. 定义节点（只返回需要更新的字段）
def greeting_node(state: State) -> State:
    """节点只需返回部分状态更新"""
    return {
        "messages": [{"role": "assistant", "content": f"你好，{state['user_name']}！"}],
        "step_count": state.get("step_count", 0) + 1
    }

def process_node(state: State) -> State:
    return {
        "messages": [{"role": "assistant", "content": "正在处理..."}],
        "step_count": state["step_count"] + 1
    }

# 3. 构建图
builder = StateGraph(State)
builder.add_node("greeting", greeting_node)
builder.add_node("process", process_node)
builder.add_edge(START, "greeting")
builder.add_edge("greeting", "process")
builder.add_edge("process", END)

graph = builder.compile()

# 4. 调用
result = graph.invoke({
    "messages": [],
    "user_name": "张三",
    "step_count": 0
})

print(f"消息数量: {len(result['messages'])}")
print(f"步骤数: {result['step_count']}")
```

**运行结果：**
```
消息数量: 2
步骤数: 2
```

### 实际应用场景

- **聊天机器人**：messages 存储对话历史
- **多步推理**：step_count 跟踪执行步骤
- **工作流状态**：user_name 等元数据传递

## 4.2 节点读写状态的基本模式

### 核心概念

**节点函数签名：** `def node(state: State) -> State`

**三种常见模式：**
1. **读取状态** → 处理 → **返回更新**
2. **条件读取** → 分支逻辑
3. **累积更新** → 使用 Reducer

### 代码示例

```python
from typing import TypedDict, Annotated
from operator import add

# 定义状态（使用不同的 Reducer）
class WorkflowState(TypedDict):
    query: str  # 单值字段（覆盖更新）
    results: Annotated[list, add]  # 列表字段（累加更新）
    metadata: dict  # 字典字段（覆盖更新）

# 模式 1：读取 → 处理 → 返回更新
def search_node(state: WorkflowState) -> WorkflowState:
    """读取 query，返回 results"""
    query = state["query"]
    # 模拟搜索
    new_results = [
        {"title": f"结果1 for {query}", "score": 0.9},
        {"title": f"结果2 for {query}", "score": 0.8}
    ]
    return {"results": new_results}  # 累加到 results

# 模式 2：条件读取 → 分支逻辑
def filter_node(state: WorkflowState) -> WorkflowState:
    """根据条件过滤结果"""
    results = state["results"]
    # 过滤低分结果
    filtered = [r for r in results if r["score"] > 0.85]
    return {"results": filtered, "metadata": {"filtered_count": len(filtered)}}

# 模式 3：累积更新
def enrich_node(state: WorkflowState) -> WorkflowState:
    """累积添加更多结果"""
    additional_results = [{"title": "额外结果", "score": 0.95}]
    return {"results": additional_results}  # 自动累加

# 构建图
from langgraph.graph import StateGraph, START, END

builder = StateGraph(WorkflowState)
builder.add_node("search", search_node)
builder.add_node("filter", filter_node)
builder.add_node("enrich", enrich_node)

builder.add_edge(START, "search")
builder.add_edge("search", "filter")
builder.add_edge("filter", "enrich")
builder.add_edge("enrich", END)

graph = builder.compile()

# 调用
result = graph.invoke({
    "query": "LangGraph 教程",
    "results": [],
    "metadata": {}
})

print(f"最终结果数: {len(result['results'])}")
print(f"元数据: {result['metadata']}")
```

**运行结果：**
```
最终结果数: 3
元数据: {'filtered_count': 2}
```

### 实际应用场景

- **搜索系统**：累积搜索结果
- **数据处理管道**：逐步过滤和转换数据
- **多源聚合**：从多个节点收集数据

## 4.3 RunnableConfig 上下文传递

### 核心概念

**RunnableConfig** 是 LangGraph 传递运行时配置的标准方式。

**常用配置项：**
- `thread_id`：线程标识（用于持久化）
- `tags`：追踪标签
- `configurable`：自定义配置

### 代码示例

```python
from typing import TypedDict
from langchain_core.runnables import RunnableConfig
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    message: str
    processed: bool

def process_node(state: State, config: RunnableConfig) -> State:
    """节点可以访问 RunnableConfig"""
    # 读取配置
    thread_id = config.get("configurable", {}).get("thread_id", "unknown")
    tags = config.get("tags", [])

    print(f"线程 ID: {thread_id}")
    print(f"标签: {tags}")

    return {
        "message": f"处理完成 (thread: {thread_id})",
        "processed": True
    }

# 构建图
builder = StateGraph(State)
builder.add_node("process", process_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()

# 调用时传递配置
result = graph.invoke(
    {"message": "测试消息", "processed": False},
    config={
        "configurable": {"thread_id": "user-123"},
        "tags": ["production", "v1.0"]
    }
)

print(f"结果: {result['message']}")
```

**运行结果：**
```
线程 ID: user-123
标签: ['production', 'v1.0']
结果: 处理完成 (thread: user-123)
```

### 实际应用场景

- **多租户系统**：通过 thread_id 隔离用户数据
- **可观测性**：通过 tags 追踪请求
- **A/B 测试**：通过 configurable 传递实验配置

## 4.4 Runtime Context 使用

### 核心概念

**Runtime Context** 用于传递不可变的运行时配置（如 API key、数据库连接）。

**关键区别：**
- **State**：可变，节点间共享和更新
- **Runtime Context**：不可变，只读访问

### 代码示例

```python
from typing import TypedDict
from dataclasses import dataclass
from langgraph.graph import StateGraph, START, END
from langgraph.runtime import Runtime

# 1. 定义 Context Schema
@dataclass
class Context:
    api_key: str
    model_name: str
    max_retries: int

# 2. 定义 State
class State(TypedDict):
    query: str
    response: str

# 3. 节点访问 Runtime Context
def llm_node(state: State, runtime: Runtime[Context]) -> State:
    """通过 runtime.context 访问上下文"""
    # 读取配置
    api_key = runtime.context.api_key
    model_name = runtime.context.model_name
    max_retries = runtime.context.max_retries

    print(f"使用模型: {model_name}")
    print(f"最大重试: {max_retries}")

    # 模拟 LLM 调用
    response = f"使用 {model_name} 处理: {state['query']}"

    return {"response": response}

# 4. 构建图（指定 context_schema）
builder = StateGraph(State, context_schema=Context)
builder.add_node("llm", llm_node)
builder.add_edge(START, "llm")
builder.add_edge("llm", END)

graph = builder.compile()

# 5. 调用时传递 context
result = graph.invoke(
    {"query": "什么是 LangGraph？", "response": ""},
    context=Context(
        api_key="sk-xxx",
        model_name="gpt-4",
        max_retries=3
    )
)

print(f"响应: {result['response']}")
```

**运行结果：**
```
使用模型: gpt-4
最大重试: 3
响应: 使用 gpt-4 处理: 什么是 LangGraph？
```

### 实际应用场景

- **动态 LLM 选择**：根据用户选择不同的模型
- **数据库连接传递**：避免在状态中传递连接对象
- **敏感信息管理**：API key 不出现在状态中

## 4.5 多状态 Schema 基础应用

### 核心概念

**多状态 Schema** 允许定义不同的输入输出接口，隐藏内部状态。

**基础模式：**
- **InputState**：定义输入接口
- **OutputState**：定义输出接口
- **OverallState**：内部完整状态

### 代码示例

```python
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义三种状态
class InputState(TypedDict):
    """外部输入接口"""
    user_query: str

class OutputState(TypedDict):
    """外部输出接口"""
    answer: str

class OverallState(TypedDict):
    """内部完整状态"""
    user_query: str
    processed_query: str
    search_results: list
    answer: str

# 2. 节点使用不同的状态
def preprocess_node(state: InputState) -> OverallState:
    """输入 InputState，输出 OverallState"""
    return {
        "processed_query": state["user_query"].lower().strip()
    }

def search_node(state: OverallState) -> OverallState:
    """内部节点使用 OverallState"""
    results = [f"结果 for {state['processed_query']}"]
    return {"search_results": results}

def answer_node(state: OverallState) -> OutputState:
    """输出 OutputState"""
    return {
        "answer": f"找到 {len(state['search_results'])} 个结果"
    }

# 3. 构建图（指定 input_schema 和 output_schema）
builder = StateGraph(
    OverallState,
    input_schema=InputState,
    output_schema=OutputState
)

builder.add_node("preprocess", preprocess_node)
builder.add_node("search", search_node)
builder.add_node("answer", answer_node)

builder.add_edge(START, "preprocess")
builder.add_edge("preprocess", "search")
builder.add_edge("search", "answer")
builder.add_edge("answer", END)

graph = builder.compile()

# 4. 调用（只传入 InputState 字段）
result = graph.invoke({"user_query": "  LangGraph 教程  "})

# 5. 返回（只包含 OutputState 字段）
print(f"返回结果: {result}")
print(f"内部状态不可见: {'processed_query' not in result}")
```

**运行结果：**
```
返回结果: {'answer': '找到 1 个结果'}
内部状态不可见: True
```

### 实际应用场景

- **API 服务**：隐藏内部实现细节
- **模块化设计**：清晰的输入输出接口
- **安全性**：敏感信息不暴露给外部

## 常见错误与解决

### 错误 1：忘记使用 Reducer

```python
# ❌ 错误：列表字段没有使用 Reducer
class State(TypedDict):
    messages: list  # 每次覆盖，不是累加

# ✅ 正确：使用 Reducer
class State(TypedDict):
    messages: Annotated[list, add_messages]
```

### 错误 2：节点返回完整状态

```python
# ❌ 错误：返回完整状态（冗余）
def node(state: State) -> State:
    return {
        "field1": state["field1"],  # 不需要
        "field2": "new_value"
    }

# ✅ 正确：只返回更新的字段
def node(state: State) -> State:
    return {"field2": "new_value"}
```

### 错误 3：在状态中传递敏感信息

```python
# ❌ 错误：API key 在状态中
class State(TypedDict):
    api_key: str  # 不安全

# ✅ 正确：使用 Runtime Context
@dataclass
class Context:
    api_key: str

def node(state: State, runtime: Runtime[Context]) -> State:
    api_key = runtime.context.api_key
```
