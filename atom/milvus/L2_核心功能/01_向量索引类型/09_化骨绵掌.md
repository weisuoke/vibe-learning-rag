## 知识卡片1：索引本质

### 核心要点
**向量索引 = 数据结构 + 算法 = 空间换时间/精度换速度**

### 关键理解
- 索引不是魔法，是权衡艺术
- 没有完美索引，只有最适合的索引
- 三要素博弈：准确率 ↔ 速度 ↔ 内存

### 记忆口诀
```
索引本质三权衡
准确速度内存量
场景决定选哪样
```

### 快速判断
```
要准确 → FLAT (100%)
要速度 → GPU_CAGRA (10x)
要省钱 → RaBitQ (32x压缩)
要平衡 → HNSW (生产标准)
```

## 知识卡片2：FLAT索引

### 核心要点
**FLAT = 暴力搜索 = O(n) = 100%准确 = 最慢**

### 关键参数
```python
index_params = {
    "index_type": "FLAT",
    "metric_type": "L2",
    "params": {}  # 无参数
}
```

### 适用场景
- ✅ < 100K向量
- ✅ 开发测试
- ✅ 准确率基准
- ❌ 生产环境

### 性能数据
```
10K:  ~5ms
100K: ~50ms
1M:   ~500ms (不推荐)
```

## 知识卡片3：IVF系列

### 核心要点
**IVF = k-means聚类 + 倒排索引 = O(nprobe*n/nlist)**

### 三种变体
```
IVF_FLAT: 无压缩, 95-99%准确, 1x内存
IVF_SQ8:  4x压缩, 93-98%准确, 0.25x内存
IVF_PQ:   32x压缩, 85-95%准确, 0.03x内存
```

### 关键参数
```python
# 构建参数
nlist = sqrt(n) to 4*sqrt(n)  # 聚类数

# 查询参数
nprobe = 8 to 128  # 搜索聚类数
```

### 选型决策
```
内存充足 → IVF_FLAT
内存受限 → IVF_SQ8
内存极限 → IVF_PQ
```

---

## 知识卡片4：HNSW图索引

### 核心要点
**HNSW = 多层图 + 贪婪搜索 = O(log n) = 生产标准**

### 关键参数
```python
# 构建参数
M = 16              # 连接数 (8-64)
efConstruction = 200  # 构建质量 (100-400)

# 查询参数
ef = 100  # 搜索质量 (50-400)
```

### 参数调优
```
M越大:
  ✅ 准确率越高
  ❌ 内存越大
  ❌ 构建越慢

ef越大:
  ✅ 准确率越高
  ❌ 查询越慢
```

### 性能数据
```
1M:   ~5ms, 98%准确率
10M:  ~10ms, 98%准确率
100M: ~20ms, 98%准确率
```

## 知识卡片5：GPU CAGRA

### 核心要点
**GPU_CAGRA = GPU并行 + 图索引 = 10x速度提升**

### 关键特性
- 构建速度：12-15x faster (vs CPU HNSW)
- 查询速度：10x faster (批量查询)
- 混合模式：GPU构建 + CPU查询

### 适用场景
```
✅ 大规模: 10M-1B向量
✅ 高QPS: > 1000
✅ 批量查询: batch > 10
❌ 单次查询: batch = 1 (GPU开销大)
```

### 配置示例
```python
index_params = {
    "index_type": "GPU_CAGRA",
    "metric_type": "L2",
    "params": {
        "intermediate_graph_degree": 128
    }
}
```

## 知识卡片6：RaBitQ量化

### 核心要点
**RaBitQ = 1-bit量化 + 残差补偿 = 72%内存节省 + <2%准确率损失**

### 技术原理
```
1. 量化: FP32 → 1-bit (32x压缩)
2. 粗筛: 用量化数据快速筛选
3. 精排: 用残差补偿精确排序
```

### 性能数据
```
内存: 512MB → 16MB (97%节省)
速度: 4x faster
准确率: 98% (<2%损失)
```

### 适用场景
```
✅ 十亿级: 1B-10B向量
✅ 成本优化
✅ 内存受限
```

## 知识卡片7：索引参数调优

### HNSW调优
```
# 平衡配置 (推荐)
M=16, efConstruction=200, ef=100

# 高性能配置
M=32, efConstruction=400, ef=200

# 低内存配置
M=8, efConstruction=100, ef=50
```

### IVF调优
```
# nlist推荐值
nlist = sqrt(n)

# nprobe推荐值
nprobe = 16-32 (平衡)
nprobe = 64-128 (高准确率)
```

### 调优原则
```
1. 先用默认值
2. 准确率不够 → 增加ef/nprobe
3. 速度太慢 → 减少ef/nprobe
4. 内存不够 → 减少M或用量化
```

## 记忆技巧

### 联想记忆
- **FLAT** = 平的 = 没有结构 = 暴力搜索
- **IVF** = Inverted File = 倒排索引 = 聚类
- **HNSW** = Hierarchical = 层次结构 = 多层图
- **CAGRA** = CUDA Accelerated = GPU加速
- **RaBitQ** = Residual-Aware Binary = 残差补偿

### 口诀记忆
```
FLAT最准最简单，小数据集来帮忙
IVF聚类分三种，压缩程度各不同
HNSW图索引最强，生产环境是标准
GPU加速CAGRA，十倍速度不是梦
RaBitQ量化极致，十亿规模省成本
```

### 对比记忆
```
准确率: FLAT > HNSW > IVF_FLAT > IVF_SQ8 > IVF_PQ
速度:   GPU_CAGRA > HNSW > IVF > FLAT
内存:   RaBitQ < IVF_PQ < IVF_SQ8 < IVF_FLAT = FLAT < HNSW
```

### 场景记忆
```
开发测试 → FLAT (简单)
生产环境 → HNSW (标准)
高性能   → GPU_CAGRA (快)
成本优化 → RaBitQ (省)
```
