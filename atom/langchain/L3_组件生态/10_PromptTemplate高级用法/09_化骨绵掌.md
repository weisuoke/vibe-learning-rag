## 卡片1：直觉理解 - PromptTemplate高级用法是什么？

**一句话**：PromptTemplate高级用法是通过预填充变量、组合模板、选择格式来提升提示词管理效率和复用性的技术。

**类比**：
- **前端类比**：就像 React 的 HOC（高阶组件），通过组合和预设 props 来复用组件逻辑
- **日常类比**：就像填空题模板，可以预先填好部分答案，组合多个模板，选择不同的填空格式

**核心价值**：
- 减少重复代码
- 提升模板复用性
- 支持复杂场景

**应用**：在 RAG 系统中，使用 partial variables 预填充系统提示词和日期，使用模板组合构建多步推理提示词。

---

## 卡片2：形式化定义 - 三大核心技术

**一句话**：PromptTemplate高级用法包括 Partial Variables（部分变量）、Template Composition（模板组合）、Template Formats（模板格式）三大核心技术。

**精确表述**：

1. **Partial Variables**：
   ```python
   partial_variables: Mapping[str, Any] = Field(default_factory=dict)
   ```
   预填充模板变量，支持字符串固定值和函数动态值。

2. **Template Composition**：
   ```python
   def __add__(self, other: Any) -> PromptTemplate
   ```
   使用 `+` 操作符组合多个模板，自动合并变量。

3. **Template Formats**：
   ```python
   template_format: PromptTemplateFormat = "f-string"  # "f-string" | "mustache" | "jinja2"
   ```
   支持三种模板格式，适应不同场景。

**应用**：在 Agent 系统中，使用模板组合构建系统提示词 + 任务提示词 + 工具提示词的复杂结构。

## 卡片3：Partial Variables - 字符串固定值

**一句话**：字符串固定值在模板创建时就确定，适用于系统设置、常量等不变的值。

**代码示例**：
```python
from langchain_core.prompts import PromptTemplate

# 预填充系统设置
prompt = PromptTemplate(
    template="{system_msg}\n\nUser: {query}",
    input_variables=["query"],
    partial_variables={"system_msg": "You are a helpful AI assistant."}
)

# 使用时只需提供 query
result = prompt.format(query="What is LangChain?")
print(result)
# Output:
# You are a helpful AI assistant.
#
# User: What is LangChain?
```

**关键点**：
- 立即求值（创建时确定）
- 性能好，无额外开销
- 适用于不变的值

**应用**：在 RAG 系统中，预填充系统角色定义、API 版本、固定的上下文信息。

---

## 卡片4：Partial Variables - 函数动态值

**一句话**：函数动态值在模板调用时才计算，适用于当前时间、随机数等需要实时更新的值。

**代码示例**：
```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 定义动态值函数
def get_current_date():
    return datetime.now().strftime("%Y-%m-%d")

# 预填充动态值
prompt = PromptTemplate(
    template="[{date}] {message}",
    input_variables=["message"],
    partial_variables={"date": get_current_date}  # 注意：传函数对象，不是调用结果
)

# 每次调用都会获取最新日期
result1 = prompt.format(message="First message")
result2 = prompt.format(message="Second message")
```

**关键点**：
- 延迟求值（调用时计算）
- 每次调用都是最新值
- 有函数调用开销

**常见陷阱**：
```python
# ❌ 错误：传入函数调用结果
partial_variables={"date": get_current_date()}  # 只会在创建时调用一次

# ✅ 正确：传入函数对象
partial_variables={"date": get_current_date}  # 每次 format 时都会调用
```

**应用**：在日志系统中，自动添加时间戳；在 A/B 测试中，动态选择提示词版本。

## 卡片5：Template Composition - 基础组合

**一句话**：使用 `+` 操作符组合多个模板，自动合并 input_variables 和 partial_variables。

**代码示例**：
```python
from langchain_core.prompts import PromptTemplate

# 定义基础模板
system_prompt = PromptTemplate.from_template("You are a {role}.")
user_prompt = PromptTemplate.from_template("Question: {question}")

# 使用 + 操作符组合
combined = system_prompt + user_prompt

# 使用组合后的模板
result = combined.format(role="helpful assistant", question="What is AI?")
print(result)
# Output:
# You are a helpful assistant.Question: What is AI?
```

**关键点**：
- 自动合并 input_variables（取并集）
- 自动合并 partial_variables（检查冲突）
- 要求 template_format 一致

**应用**：在 RAG 系统中，组合系统提示词 + 检索上下文 + 用户问题，构建完整的提示词。

## 卡片6：Template Composition - 变量合并规则

**一句话**：模板组合时，input_variables 取并集不冲突，partial_variables 必须无重复键。

**源码实现**（来自 prompt.py:142-184）：
```python
# input_variables 合并（取并集）
input_variables = list(
    set(self.input_variables) | set(other.input_variables)
)

# partial_variables 合并（检查冲突）
partial_variables = dict(self.partial_variables.items())
for k, v in other.partial_variables.items():
    if k in partial_variables:
        raise ValueError("Cannot have same variable partialed twice.")
    partial_variables[k] = v
```

**实际示例**：
```python
# ✅ 正确：input_variables 可以重复
prompt1 = PromptTemplate.from_template("Hello {name}.")
prompt2 = PromptTemplate.from_template("Your role is {name}.")
combined = prompt1 + prompt2  # 成功，name 只需提供一次

# ❌ 错误：partial_variables 不能重复
prompt1 = PromptTemplate(
    template="{system} {query}",
    input_variables=["query"],
    partial_variables={"system": "System A"}
)
prompt2 = PromptTemplate(
    template="{system} {task}",
    input_variables=["task"],
    partial_variables={"system": "System B"}
)
# combined = prompt1 + prompt2  # 抛出 ValueError
```

**应用**：在多模块系统中，确保不同模块的 partial_variables 使用不同的键名，避免冲突。

## 卡片7：Template Formats - 三种格式对比

**一句话**：f-string 简单快速，mustache 支持嵌套，jinja2 功能强大但有安全风险。

**对比表格**：

| 格式 | 语法 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| f-string | `{var}` | 简单、快速、支持验证 | 不支持嵌套和逻辑 | 90%的基础场景 |
| mustache | `{{var}}` | 支持嵌套、section、跨语言 | 不支持验证、稍慢 | 复杂数据结构 |
| jinja2 | `{{ var }}` | 功能强大、支持逻辑控制 | 安全风险、最慢 | 复杂逻辑场景 |

**代码示例**：
```python
from langchain_core.prompts import PromptTemplate

# f-string（默认）
prompt1 = PromptTemplate.from_template("Hello {name}!")

# mustache（嵌套变量）
prompt2 = PromptTemplate.from_template(
    "User: {{user.name}}, Email: {{user.email}}",
    template_format="mustache"
)

# jinja2（逻辑控制）
prompt3 = PromptTemplate.from_template(
    "{% if score > 80 %}Excellent{% else %}Good{% endif %}",
    template_format="jinja2"
)
```

**应用**：在多语言系统中，使用 mustache 模板支持嵌套的国际化配置；在复杂规则引擎中，使用 jinja2 实现条件渲染。

## 卡片8：Mustache 高级特性

**一句话**：Mustache 支持嵌套变量、section 变量、no escape 三大高级特性。

**三大特性**：

1. **嵌套变量**：`{{obj.bar}}`
   ```python
   prompt = PromptTemplate.from_template(
       "Name: {{user.name}}, Age: {{user.age}}",
       template_format="mustache"
   )
   result = prompt.format(user={"name": "Alice", "age": 30})
   ```

2. **Section 变量**：`{{#foo}} {{bar}} {{/foo}}`
   ```python
   prompt = PromptTemplate.from_template(
       "{{#is_premium}}Premium: {{feature}}{{/is_premium}}",
       template_format="mustache"
   )
   result = prompt.format(is_premium=True, feature="Advanced Analytics")
   ```

3. **No escape**：`{{{foo}}}`（不转义 HTML）
   ```python
   prompt = PromptTemplate.from_template(
       "HTML: {{{html_content}}}",
       template_format="mustache"
   )
   result = prompt.format(html_content="<b>Bold</b>")
   ```

**应用**：在内容生成系统中，使用 section 变量实现条件渲染；在 HTML 生成中，使用 no escape 保留原始标签。

## 卡片9：实战模式 - RAG 提示词管理器

**一句话**：构建一个完整的 RAG 提示词管理器，支持分层管理、预填充、组合、多语言。

**完整实现**：
```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime
from pathlib import Path

class RAGPromptManager:
    """RAG 系统提示词管理器"""

    def __init__(self, template_dir: str = "prompts"):
        self.template_dir = Path(template_dir)
        self._cache = {}

    def get_system_prompt(self) -> PromptTemplate:
        """获取系统提示词（带 partial variables）"""
        return PromptTemplate(
            template="你是一个专业的 AI 助手。当前日期：{date}\n{task_instruction}",
            input_variables=["task_instruction"],
            partial_variables={"date": lambda: datetime.now().strftime("%Y-%m-%d")}
        )

    def get_retrieval_prompt(self) -> PromptTemplate:
        """获取检索提示词"""
        return PromptTemplate.from_template(
            "基于以下上下文回答问题：\n\n{context}\n\n问题：{question}"
        )

    def build_rag_prompt(self) -> PromptTemplate:
        """组合完整的 RAG 提示词"""
        return self.get_system_prompt() + self.get_retrieval_prompt()

# 使用示例
manager = RAGPromptManager()
rag_prompt = manager.build_rag_prompt()
result = rag_prompt.format(
    task_instruction="回答用户问题",
    context="LangChain 是一个 AI 应用开发框架...",
    question="什么是 LangChain？"
)
```

**应用**：在生产环境中，使用管理器统一管理所有提示词模板，支持版本控制、A/B 测试、多语言。

## 常见问题 FAQ

### Q1：什么时候用字符串固定值，什么时候用函数动态值？

**A**：
- **字符串固定值**：值不会变化，如系统设置、API 密钥、固定提示词
- **函数动态值**：值需要实时更新，如当前时间、随机数、上下文相关的值

**决策标准**：
- 值是否会变化？→ 会变化用函数，不变用字符串
- 是否需要实时性？→ 需要用函数
- 是否有计算成本？→ 有成本且不常变用字符串

---

### Q2：模板组合时如何避免变量冲突？

**A**：
- **input_variables**：可以重复，自动取并集
- **partial_variables**：不能重复，会抛出 ValueError

**解决方案**：
- 使用不同的变量名（如 `system1`、`system2`）
- 在组合前检查变量名是否冲突
- 使用命名空间前缀（如 `module1_system`、`module2_system`）

---

### Q3：什么时候应该使用 Mustache 而不是 f-string？

**A**：
- **使用 f-string**：简单的变量替换（90%的场景）
- **使用 Mustache**：需要嵌套变量、条件渲染、列表迭代
- **使用 Jinja2**：需要复杂的逻辑控制、模板继承

**性能对比**：
- f-string: 最快（基准）
- Mustache: 约慢 2-3 倍
- Jinja2: 约慢 5-10 倍

---

### Q4：如何在生产环境中管理提示词模板？

**A**：
1. **分层管理**：系统层、任务层、用户层
2. **版本控制**：使用 Git 管理模板文件
3. **缓存机制**：避免重复加载和解析
4. **多语言支持**：按语言组织模板文件
5. **A/B 测试**：动态加载不同版本的模板
6. **监控优化**：使用 LangSmith 监控提示词性能

---

### Q5：Jinja2 模板的安全风险是什么？

**A**：
- **代码注入风险**：不可信的模板可能包含恶意代码
- **信息泄露风险**：模板可能访问敏感的环境变量

**防范措施**：
- 使用 `SandboxedEnvironment`
- 不要接受来自不可信源的模板
- 验证和清理用户输入
- 限制模板的访问权限
