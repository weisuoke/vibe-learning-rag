# 组件组合最佳实践 - 一句话总结

## 一句话总结

**组件组合最佳实践是通过串行(|)、并行({})、条件(Branch)、降级(Fallback)、重试(Retry)等模式将 LangChain 组件高效组装的方法论，核心是选对组件、用对模式、处理好异常、优化好性能，在 AI 应用开发中实现可靠、高效、可观测的生产级管道。**

---

## 核心要点回顾

### 5个核心概念

| 概念 | 一句话 |
|------|--------|
| 组件选择决策框架 | 根据复杂度选择 SDK/LCEL/LangGraph |
| 组合模式全景 | 串行\|、并行{}、条件Branch、透传Passthrough |
| 错误处理策略 | Retry处理瞬态错误，Fallback处理持续故障 |
| 性能优化模式 | 流式降延迟、batch提吞吐、缓存省成本 |
| 可观测性与调试 | Callbacks监控、LangSmith追踪、Graph可视化 |

### 4个实战场景

| 场景 | 核心模式 |
|------|----------|
| RAG管道 | RunnableParallel(question + context) \| prompt \| model \| parser |
| 错误处理 | model.with_retry().with_fallbacks([backup]) |
| 性能优化 | stream + batch + cache + RunnableParallel |
| 生产级管道 | configurable + callbacks + retry + fallback |

### 3个关键原则

1. **重试范围要小** - 只在最可能失败的组件上添加重试
2. **流式要注意阻断** - RunnableLambda 会阻断流式，用 RunnableGenerator 替代
3. **不要过度使用** - 简单任务直接用 SDK，复杂任务才用 LangChain

## 记忆口诀

> **选组件、连管道、处异常、优性能、看监控**

- **选** - 根据场景选择合适的组件
- **连** - 用 | {} Branch 等模式连接
- **处** - Retry + Fallback 处理异常
- **优** - stream + batch + cache 优化性能
- **看** - Callbacks + LangSmith 监控运行

## 快速检查清单

- [ ] 是否选择了合适的组件？（不过度使用 LangChain）
- [ ] 是否使用了正确的组合模式？（串行/并行/条件）
- [ ] 是否在 Model 上添加了 retry？
- [ ] 是否配置了 fallback 降级？
- [ ] 是否使用了流式输出？（实时场景）
- [ ] 是否使用了 batch？（批量场景）
- [ ] 是否配置了缓存？（重复查询场景）
- [ ] 是否添加了可观测性？（Callbacks/LangSmith）
- [ ] 是否用 get_graph() 验证了管道结构？
