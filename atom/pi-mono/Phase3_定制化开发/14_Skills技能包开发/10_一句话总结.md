# 一句话总结

**Skills 是 AI Agent 能力的标准化封装单元，通过 SKILL.md 文件 + 文件系统加载 + 命令式调用，实现能力的模块化、复用和组合，就像 npm 包之于 Node.js。**

---

## 核心本质

Skills 技能包系统解决了 AI Agent 开发中的核心问题：**如何将 Agent 能力标准化、模块化、可复用化？**

### 三个关键要素

1. **标准化封装**
   - SKILL.md 文件格式
   - YAML frontmatter 元数据
   - Markdown 正文逻辑
   - 符合开放标准

2. **零配置加载**
   - 文件系统自动扫描
   - 全局 + 项目级别
   - 递归子目录支持
   - 启动时自动发现

3. **简洁调用**
   - `/skill:name` 命令
   - 参数传递支持
   - 自动展开注入
   - 技能组合能力

## 类比理解

### TypeScript/Node.js 类比

```typescript
// npm 包
import { Button } from '@ui/components';

// Skills 技能包
/skill:code-review src/components/Button.tsx
```

**相似点：**
- 都是能力的封装单元
- 都支持发布和分发
- 都可以被导入和使用
- 都支持版本管理

**不同点：**
- npm 包是代码库，Skills 是提示词库
- npm 包通过 import，Skills 通过 `/skill:` 命令
- npm 包在编译时，Skills 在运行时

### 日常生活类比

**Skills 就像手机 App：**
- 每个 App 提供特定功能（每个 Skill 提供特定能力）
- 可以从应用商店下载（可以从 GitHub 下载）
- 安装后即可使用（放到 skills 目录即可使用）
- 可以组合使用（可以在 Skill 中调用其他 Skill）

## 核心价值

### 1. 模块化

**问题：** Agent 能力混杂在一起，难以管理。

**解决：** 每个 Skill 封装一个独立能力。

```
❌ 单体 Agent
- 代码审查 + 测试 + 调试 + 重构 + ...
- 所有逻辑混在一起
- 难以维护和更新

✅ 模块化 Skills
- /skill:code-review - 代码审查
- /skill:tdd - 测试驱动开发
- /skill:debug - 调试分析
- /skill:refactor - 代码重构
- 每个 Skill 独立维护
```

### 2. 复用性

**问题：** 相同的提示词在多个项目中重复编写。

**解决：** 一次编写，多次使用。

```
项目 A: /skill:code-review
项目 B: /skill:code-review
项目 C: /skill:code-review
→ 同一个 Skill，在不同项目中复用
```

### 3. 可组合

**问题：** 复杂工作流难以实现。

**解决：** Skills 可以组合成更复杂的工作流。

```markdown
<!-- scout-and-plan/SKILL.md -->
1. 调用 /skill:explore 探索代码库
2. 调用 /skill:plan 生成实现计划
3. 调用 /skill:implement 执行实现
```

### 4. 标准化

**问题：** 不同工具的技能格式不兼容。

**解决：** SKILL.md 开放标准，跨工具兼容。

```
Pi-mono Skills ←→ Claude Code Skills
     ↓                    ↓
Vercel Agent Skills ←→ Anthropic Skills
```

### 5. 易分发

**问题：** 技能难以分享和传播。

**解决：** 通过 Git/文件系统轻松分发。

```bash
# 安装技能
git clone https://github.com/user/my-skills ~/.pi/agent/skills/my-skills

# 使用技能
/skill:my-custom-skill
```
