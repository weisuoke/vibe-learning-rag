# OpenClaw 基础故障排查 - 核心概念 7：自动修复机制

## 一、概念定义

**自动修复机制是 OpenClaw doctor 命令的核心能力，通过配置迁移、键值清理、环境冲突解决和备份恢复，实现配置问题的一键自动修复。**

## 二、核心价值

### 2.1 为什么需要自动修复？

**问题场景**：
- 手动修复配置容易出错
- 配置迁移步骤复杂
- 不知道如何修复
- 修复后可能引入新问题

**自动修复的价值**：
- ✅ 一键修复常见问题
- ✅ 修复前自动备份
- ✅ 标准化修复流程
- ✅ 提供详细修复报告
- ✅ 支持回滚恢复

**来源**: `reference/source_doctor_command.md`

```typescript
// 清理无效配置键
function removeUnknownKeys(config: any, schema: Schema): any {
  const validKeys = Object.keys(schema);
  const unknownKeys = Object.keys(config).filter(k => !validKeys.includes(k));

  for (const key of unknownKeys) {
    delete config[key];
    logger.info(`Removed unknown key: ${key}`);
  }

  return config;
}
```

```typescript
function addMissingKeys(config: any, schema: Schema): any {
  for (const [key, definition] of Object.entries(schema)) {
    if (definition.required && !(key in config)) {
      config[key] = definition.default || generateDefault(definition.type);
      logger.info(`Added missing key: ${key}`);
    }
  }

  return config;
}
```

```typescript
function fixInvalidValues(config: any, schema: Schema): any {
  for (const [key, definition] of Object.entries(schema)) {
    const value = config[key];

    if (!validateValue(value, definition)) {
      config[key] = definition.default;
```

## 四、工作原理

### 4.1 修复流程实现

```typescript
// 自动修复完整流程
async function runRepair(): Promise<RepairResult> {
  const results: RepairAction[] = [];

  // 1. 备份配置
  const backupPath = '~/.openclaw/openclaw.json.bak';
  await backupConfig(backupPath);
  results.push({ action: 'backup', status: 'success', path: backupPath });

  // 2. 加载配置
  const config = await loadConfig();

  // 3. 运行诊断
  const issues = await diagnose(config);

  // 4. 执行修复
  for (const issue of issues) {
    const fix = await applyFix(config, issue);
    results.push(fix);
  }

  // 5. 保存配置
  await saveConfig(config);

  // 6. 验证修复
  const verification = await diagnose(config);

  return {
    success: verification.length === 0,
    actions: results,
    remainingIssues: verification
  };
}
```

### 4.2 备份机制

```typescript
// 配置备份实现
async function backupConfig(backupPath: string): Promise<void> {
  const configPath = '~/.openclaw/openclaw.json';

  // 1. 读取当前配置
  const config = await fs.readFile(configPath, 'utf-8');

  // 2. 添加备份元数据
  const backup = {
    timestamp: Date.now(),
    version: getCurrentVersion(),
    config: JSON.parse(config)
  };

  // 3. 保存备份
  await fs.writeFile(backupPath, JSON.stringify(backup, null, 2));

  logger.info(`Backup saved to ${backupPath}`);
}

// 恢复备份
async function restoreBackup(backupPath: string): Promise<void> {
  const backup = JSON.parse(await fs.readFile(backupPath, 'utf-8'));
  const configPath = '~/.openclaw/openclaw.json';

  await fs.writeFile(configPath, JSON.stringify(backup.config, null, 2));

  logger.info(`Config restored from ${backupPath}`);
}
```

### 4.3 修复策略选择

```typescript
// 修复策略决策
function selectRepairStrategy(issue: Issue): RepairStrategy {
  switch (issue.type) {
    case 'config_version':
      return new MigrationStrategy();

    case 'unknown_key':
      return new RemoveKeyStrategy();

    case 'missing_key':
      return new AddKeyStrategy();

    case 'invalid_value':
      return new FixValueStrategy();

    case 'env_conflict':
      return new ResolveEnvStrategy();

    default:
      return new ManualFixStrategy();
  }
}

// 应用修复策略
async function applyFix(config: any, issue: Issue): Promise<RepairAction> {
  const strategy = selectRepairStrategy(issue);

  try {
    await strategy.apply(config, issue);
    return {
      action: strategy.name,
      status: 'success',
      issue: issue.type
    };
  } catch (error) {
    return {
      action: strategy.name,
      status: 'failed',
      issue: issue.type,
      error: error.message
    };
  }
}
```

```bash
# 4. 重新验证
```

### 6.2 如何回滚修复？

```bash

git checkout openclaw.json
```

## 七、最佳实践

### 7.1 定期健康检查

```bash
#!/bin/bash
# health-check.sh

# 每周运行健康检查和自动修复

echo "Running OpenClaw health check..."
openclaw doctor

if [ $? -ne 0 ]; then
  echo "Issues found, running auto-repair..."
  openclaw doctor --repair

  if [ $? -eq 0 ]; then
    echo "✓ All issues fixed"
  else
    echo "✗ Some issues remain, manual intervention required"
    exit 1
  fi
fi

echo "✓ OpenClaw is healthy"
```

```yaml

    - cron: '0 0 * * *'  # 每天运行

  health-check:
    runs-on: ubuntu-latest
    steps:
      - name: Install OpenClaw
        run: npm install -g openclaw@latest

      - name: Run Doctor
        run: openclaw doctor --non-interactive

      - name: Auto Repair
        if: failure()
```
