## 什么是"最小可用"？

在向量存储后端选择中，"最小可用"指的是：
- **快速决策**：5分钟内做出合理选择
- **避免过度工程**：不追求完美，追求够用
- **渐进式升级**：从简单开始，根据需求演进

## 核心知识点 1：三个关键问题

### 问题 1：数据规模是多少？

这是最重要的决策因素。

```python
def quick_backend_recommendation(num_documents: int) -> str:
    """根据数据规模快速推荐后端"""
    if num_documents < 10000:
        return "Chroma"
    elif num_documents < 100000:
        return "Qdrant"
    else:
        return "Milvus 或 Pinecone"

# 示例
print(quick_backend_recommendation(5000))   # 输出: Chroma
print(quick_backend_recommendation(50000))  # 输出: Qdrant
print(quick_backend_recommendation(500000)) # 输出: Milvus 或 Pinecone
```

**为什么数据规模最重要？**
- 小规模（< 10K）：任何方案都能工作，选最简单的
- 中等规模（10K-100K）：需要考虑性能，但不需要分布式
- 大规模（> 100K）：必须考虑扩展性和分布式能力

**[来源: reference/search_rag_selection_criteria_02.md | 数据规模标准]**

### 问题 2：有运维能力吗？

运维能力决定了你能否使用自托管方案。

```python
def recommend_by_ops_capability(
    num_documents: int,
    has_ops_team: bool
) -> str:
    """根据运维能力推荐后端"""
    if num_documents < 10000:
        return "Chroma（无需运维）"

    if has_ops_team:
        return "Qdrant 或 Milvus（自托管）"
    else:
        return "Pinecone（托管服务）"

# 示例
print(recommend_by_ops_capability(50000, has_ops_team=True))
# 输出: Qdrant 或 Milvus（自托管）

print(recommend_by_ops_capability(50000, has_ops_team=False))
# 输出: Pinecone（托管服务）
```

**运维能力包括：**
- 服务器部署和维护
- 监控和告警
- 备份和恢复
- 性能调优

### 问题 3：预算是多少？

预算决定了托管服务 vs 自托管的选择。

```python
def recommend_by_budget(
    num_documents: int,
    monthly_budget_usd: int
) -> str:
    """根据预算推荐后端"""
    if num_documents < 10000:
        return "Chroma（免费）"

    # 估算托管服务成本（Pinecone）
    pinecone_cost = (num_documents / 1000000) * 70  # $70/百万向量/月

    if monthly_budget_usd >= pinecone_cost:
        return "Pinecone（托管，省心）"
    else:
        return "Qdrant 或 Milvus（自托管，成本低）"

# 示例
print(recommend_by_budget(100000, monthly_budget_usd=100))
# 输出: Qdrant 或 Milvus（自托管，成本低）

print(recommend_by_budget(100000, monthly_budget_usd=500))
# 输出: Pinecone（托管，省心）
```

**成本对比：**
- **免费方案**：Chroma, FAISS（本地运行）
- **低成本**：Qdrant, Milvus（自托管，服务器成本）
- **中高成本**：Pinecone（托管，按使用付费）

**[来源: reference/search_rag_selection_criteria_02.md | 成本标准]**

### 5分钟决策流程

```
开始
  ↓
数据规模？
  ├─ < 10K → Chroma（零配置，快速上手）
  ├─ 10K-100K → 有运维能力？
  │              ├─ 有 → Qdrant（性能好，易部署）
  │              └─ 无 → Pinecone（托管服务）
  └─ > 100K → 预算充足？
              ├─ 是 → Pinecone（托管，自动扩展）
              └─ 否 → Milvus（自托管，企业级）
```

### 决策函数实现

```python
def quick_decision(
    num_documents: int,
    has_ops_team: bool = False,
    monthly_budget_usd: int = 0
) -> dict:
    """快速决策：推荐向量存储后端

    Args:
        num_documents: 文档数量
        has_ops_team: 是否有运维团队
        monthly_budget_usd: 月度预算（美元）

    Returns:
        推荐结果字典
    """
    if num_documents < 10000:
        return {
            "backend": "Chroma",
            "reason": "小规模数据，零配置，快速上手",
            "deployment": "本地部署",
            "cost": "免费"
        }

    elif num_documents < 100000:
        if has_ops_team:
            return {
                "backend": "Qdrant",
                "reason": "中等规模，性能好，易部署",
                "deployment": "自托管（Docker）",
                "cost": "$50-200/月（服务器）"
            }
        else:
            return {
                "backend": "Pinecone",
                "reason": "中等规模，无需运维",
                "deployment": "托管服务",
                "cost": "$70-200/月（按使用付费）"
            }

    else:  # > 100K
        if monthly_budget_usd >= 500:
            return {
                "backend": "Pinecone",
                "reason": "大规模，托管服务，自动扩展",
                "deployment": "托管服务",
                "cost": "$500+/月（按使用付费）"
            }
        else:
            return {
                "backend": "Milvus",
                "reason": "大规模，自托管，成本可控",
                "deployment": "自托管（Kubernetes）",
                "cost": "$500-2000/月（服务器集群）"
            }

# 示例使用
result = quick_decision(
    num_documents=50000,
    has_ops_team=True,
    monthly_budget_usd=100
)

print(f"推荐后端: {result['backend']}")
print(f"理由: {result['reason']}")
print(f"部署方式: {result['deployment']}")
print(f"成本: {result['cost']}")

# 输出:
# 推荐后端: Qdrant
# 理由: 中等规模，性能好，易部署
# 部署方式: 自托管（Docker）
# 成本: $50-200/月（服务器）
```

### 场景 1：Chroma（< 10K 文档）

```python
from langchain_chroma import Chroma
from langchain_openai import OpenAIEmbeddings
from langchain_core.documents import Document

# 1. 初始化（零配置）
vector_store = Chroma(
    collection_name="my_docs",
    embedding_function=OpenAIEmbeddings(),
    persist_directory="./chroma_db"  # 持久化

# 2. 添加文档

    Document(page_content="文档1", metadata={"source": "file1"}),
    Document(page_content="文档2", metadata={"source": "file2"})

# 3. 检索
results = vector_store.similarity_search("查询", k=5)
for doc in results:
    print(doc.page_content)
```

- 零配置：无需安装额外服务
- 持久化：数据保存到本地
- 功能完整：支持元数据过滤、MMR 检索

## 核心知识点 4：统一接口模式

### 为什么需要统一接口？

**问题**：不同后端 API 不同，切换成本高

**解决**：使用 LangChain 统一接口

```python
from langchain_core.vectorstores import VectorStore
from langchain_openai import OpenAIEmbeddings

def create_vector_store(backend: str, **kwargs) -> VectorStore:
    """工厂函数：创建向量存储

    Args:
        backend: 后端类型（chroma, qdrant, pinecone）
        **kwargs: 后端特定参数

    Returns:
        VectorStore 实例
    """
    embeddings = OpenAIEmbeddings()

    if backend == "chroma":
        from langchain_chroma import Chroma
        return Chroma(
            collection_name=kwargs.get("collection_name", "default"),
            embedding_function=embeddings,
            persist_directory=kwargs.get("persist_directory", "./chroma_db")
        )

    elif backend == "qdrant":
        from langchain_qdrant import QdrantVectorStore
        return QdrantVectorStore.from_documents(
            documents=kwargs.get("documents", []),
            embedding=embeddings,
            url=kwargs.get("url", "http://localhost:6333"),
            collection_name=kwargs.get("collection_name", "default")
        )

    elif backend == "pinecone":
        from langchain_pinecone import PineconeVectorStore
        from pinecone import Pinecone

        pc = Pinecone(api_key=kwargs.get("api_key"))
        index = pc.Index(kwargs.get("index_name"))

        return PineconeVectorStore(
            index=index,
            embedding=embeddings
        )

    else:
        raise ValueError(f"不支持的后端: {backend}")

# 使用示例
# 开发环境：使用 Chroma
dev_store = create_vector_store(
    backend="chroma",
    collection_name="dev_docs"
)

# 生产环境：使用 Qdrant
prod_store = create_vector_store(
    backend="qdrant",
    url="http://qdrant:6333",
    collection_name="prod_docs"
)

# 切换后端只需更改一个参数
```

**优势**：
- 降低迁移成本
- 支持渐进式升级
- 保持架构灵活性

## 核心知识点 5：渐进式升级路径

### 从开发到生产的演进

```python
import os
from typing import Literal

def get_vector_store_for_env(
    env: Literal["dev", "test", "prod"] = "dev"
):
    """根据环境返回合适的向量存储"""

    if env == "dev":
        # 开发环境：Chroma（零配置）
        from langchain_chroma import Chroma
        return Chroma(
            collection_name="dev_collection",
            embedding_function=OpenAIEmbeddings(),
            persist_directory="./chroma_db"
        )

    elif env == "test":
        # 测试环境：Chroma（持久化测试）
        from langchain_chroma import Chroma
        return Chroma(
            collection_name="test_collection",
            embedding_function=OpenAIEmbeddings(),
            persist_directory="./test_chroma_db"
        )

    elif env == "prod":
        # 生产环境：Qdrant（高性能）
        from langchain_qdrant import QdrantVectorStore
        return QdrantVectorStore.from_documents(
            documents=[],
            embedding=OpenAIEmbeddings(),
            url=os.getenv("QDRANT_URL", "http://qdrant:6333"),
            collection_name="prod_collection"
        )

# 使用
env = os.getenv("ENV", "dev")
vector_store = get_vector_store_for_env(env)
```

**升级路径**：
```
阶段 1：开发（Chroma）
  ↓ 数据量增长到 10K
阶段 2：测试（Chroma + 持久化）
  ↓ 数据量增长到 100K
阶段 3：生产（Qdrant）
  ↓ 数据量增长到 1M
阶段 4：大规模（Milvus/Pinecone）
```
