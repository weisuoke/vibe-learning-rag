# OpenClaw 安装与配置 - 一句话总结

**OpenClaw 通过 npm/pnpm 全局安装到 Node.js 22+ 环境，使用 Onboarding Wizard 交互式配置 Gateway 和通道，配置文件存储在 ~/.openclaw/ 目录，支持前台调试和守护进程两种运行模式，是构建多通道 AI 应用的基础设施。**

---

## 核心要点回顾

### 安装
- **方式**：npm/pnpm 全局安装
- **命令**：`npm install -g openclaw@latest`
- **验证**：`openclaw --version`

### 环境
- **要求**：Node.js 22.0.0+
- **推荐**：Node.js 22.18.0+（原生 TypeScript）
- **管理**：nvm、asdf、Volta

### 配置
- **位置**：`~/.openclaw/`
- **文件**：`settings.json`、`gateway.log`
- **方式**：`openclaw onboard` 交互式向导

### 运行
- **前台模式**：`openclaw gateway start`（调试用）
- **守护进程**：`openclaw gateway start -d`（生产用）
- **状态检查**：`openclaw gateway status`

### 价值
- **多通道统一**：Claude、GPT、Gemini 等统一接口
- **本地网关**：隐私保护、成本控制
- **基础设施**：构建 AI 应用的基石

## 最小可用示例

```bash
# 1. 安装 OpenClaw
npm install -g openclaw@latest

# 2. 运行 Onboarding Wizard
openclaw onboard
# 按提示输入：
# - Claude API Key
# - Gateway 端口（默认 3000）
# - 启用的通道（Claude、GPT 等）

# 3. 启动 Gateway
openclaw gateway start

# 4. 测试连接（新终端）
curl http://localhost:3000/health
# 预期输出：{"status":"ok","version":"1.2.3"}
```

## 常见问题速查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `openclaw: command not found` | PATH 未配置 | 重新安装或手动添加 PATH |
| `Node.js version too old` | Node.js < 22 | 升级到 Node.js 22+ |
| `Gateway failed to start` | 端口被占用 | 修改 `~/.openclaw/settings.json` 中的端口 |
| `API key invalid` | 配置错误 | 重新运行 `openclaw onboard` |
