# OpenClaw 基础故障排查 - 核心概念 5：配置验证

## 一、概念定义

**配置验证是 OpenClaw 故障排查的预防性机制，通过语法检查、值验证、版本兼容性检查和环境变量冲突检测，确保配置文件的正确性和一致性。**

**来源**: `reference/source_doctor_command.md`

---

## 二、核心价值

### 2.1 为什么需要配置验证？

**问题场景**：
- 手动编辑配置文件出错
- 升级后配置不兼容
- 环境变量覆盖配置
- 配置值类型错误
- 必需字段缺失

**配置验证的价值**：
- ✅ 启动前发现配置错误
- ✅ 自动迁移旧版本配置
- ✅ 检测环境变量冲突
- ✅ 验证配置值有效性
- ✅ 提供详细的错误信息

**来源**: `reference/source_doctor_command.md`

### 2.2 配置验证的四个层次

#### 层次 1：语法验证

```typescript
// 配置文件语法验证
interface SyntaxValidation {
  valid: boolean;
  error?: {
    line: number;
    column: number;
    message: string;
  };
}

// 检查 JSON 语法
function validateSyntax(configPath: string): SyntaxValidation {
  try {
    const content = fs.readFileSync(configPath, 'utf-8');
    JSON.parse(content);
    return { valid: true };
  } catch (error) {
    return {
      valid: false,
      error: {
        line: error.lineNumber,
        column: error.columnNumber,
        message: error.message
      }
    };
  }
}
```

**来源**: `reference/source_doctor_command.md`

#### 层次 2：值验证

```typescript
// 配置值验证
interface ValueValidation {
  field: string;
  valid: boolean;
  error?: string;
  suggestion?: string;
}

// 验证配置值
function validateValues(config: Config): ValueValidation[] {
  const results: ValueValidation[] = [];

  // 验证 gateway.mode
  if (!['local', 'remote'].includes(config.gateway.mode)) {
    results.push({
      field: 'gateway.mode',
      valid: false,
      error: `Invalid mode: ${config.gateway.mode}`,
      suggestion: 'Must be "local" or "remote"'
    });
  }

  // 验证 gateway.port
  if (config.gateway.port < 1024 || config.gateway.port > 65535) {
    results.push({
      field: 'gateway.port',
      valid: false,
      error: `Invalid port: ${config.gateway.port}`,
      suggestion: 'Must be between 1024 and 65535'
    });
  }

  return results;
}
```

**来源**: `reference/source_doctor_command.md`

#### 层次 3：版本兼容性

```typescript
// 版本兼容性检查
interface VersionCheck {
  currentVersion: string;
  requiredVersion: string;
  compatible: boolean;
  migration?: ConfigMigration;
}

// 检查配置版本
function checkVersion(config: Config): VersionCheck {
  const currentVersion = config.version || 'v1';
  const requiredVersion = 'v2';

  if (currentVersion === requiredVersion) {
    return {
      currentVersion,
      requiredVersion,
      compatible: true
    };
  }

  // 需要迁移
  return {
    currentVersion,
    requiredVersion,
    compatible: false,
    migration: getMigration(currentVersion, requiredVersion)
  };
}
```

#### 层次 4：环境变量冲突

```typescript
// 环境变量冲突检测
interface EnvConflict {
  variable: string;
  envValue: string;
  configValue: string;
  conflict: boolean;
}

// 检测环境变量冲突
function detectEnvConflicts(config: Config): EnvConflict[] {
  const conflicts: EnvConflict[] = [];

  // 检查 OPENCLAW_GATEWAY_TOKEN
  const envToken = process.env.OPENCLAW_GATEWAY_TOKEN;
  if (envToken && envToken !== config.gateway.auth.token) {
    conflicts.push({
      variable: 'OPENCLAW_GATEWAY_TOKEN',
      envValue: envToken,
      configValue: config.gateway.auth.token,
      conflict: true
    });
  }

  return conflicts;
}
```

## 四、工作原理

### 4.1 配置验证流程

```typescript
// 配置验证完整流程
async function validateConfig(configPath: string): Promise<ValidationResult> {
  const results: ValidationIssue[] = [];

  // 1. 语法验证
  const syntaxCheck = validateSyntax(configPath);
  if (!syntaxCheck.valid) {
    return {
      valid: false,
      issues: [{
        type: 'syntax',
        severity: 'fatal',
        message: syntaxCheck.error.message
      }]
    };
  }

  // 2. 加载配置
  const config = loadConfig(configPath);

  // 3. 值验证
  const valueChecks = validateValues(config);
  results.push(...valueChecks.filter(c => !c.valid).map(c => ({
    type: 'value',
    severity: 'error',
    field: c.field,
    message: c.error,
    suggestion: c.suggestion
  })));

  // 4. 版本检查
  const versionCheck = checkVersion(config);
  if (!versionCheck.compatible) {
    results.push({
      type: 'version',
      severity: 'warning',
      message: `Config version ${versionCheck.currentVersion} is outdated`,
      suggestion: 'Run doctor --repair to migrate'
    });
  }

  // 5. 环境变量冲突
  const envConflicts = detectEnvConflicts(config);
  results.push(...envConflicts.map(c => ({
    type: 'env_conflict',
    severity: 'warning',
    message: `${c.variable} overrides config`,
    suggestion: `Clear environment variable: launchctl unsetenv ${c.variable}`
  })));

  return {
    valid: results.length === 0,
    issues: results
  };
}
```

### 4.2 配置迁移机制

```typescript
// 配置迁移实现
interface ConfigMigration {
  from: string;
  to: string;
  changes: ConfigChange[];
}

interface ConfigChange {
  type: 'rename' | 'remove' | 'add' | 'transform';
  path: string;
  oldValue?: any;
  newValue?: any;
}

// v1 → v2 迁移
const v1ToV2Migration: ConfigMigration = {
  from: 'v1',
  to: 'v2',
  changes: [
    {
      type: 'rename',
      path: 'gateway.token',
      newValue: 'gateway.auth.token'
    },
    {
      type: 'add',
      path: 'gateway.auth.mode',
      newValue: 'token'
    },
    {
      type: 'remove',
      path: 'gateway.password'
    }
  ]
};

// 执行迁移
function migrateConfig(config: any, migration: ConfigMigration): any {
  const newConfig = { ...config };

  for (const change of migration.changes) {
    switch (change.type) {
      case 'rename':
        const value = getPath(newConfig, change.path);
        setPath(newConfig, change.newValue, value);
        deletePath(newConfig, change.path);
        break;

      case 'add':
        setPath(newConfig, change.path, change.newValue);
        break;

      case 'remove':
        deletePath(newConfig, change.path);
        break;

      case 'transform':
        const oldValue = getPath(newConfig, change.path);
        const transformed = change.transform(oldValue);
        setPath(newConfig, change.path, transformed);
        break;
    }
  }

  newConfig.version = migration.to;
  return newConfig;
}
```

### 6.1 如何手动验证配置？

```bash
# 方法 1：使用 doctor
openclaw doctor

# 方法 2：使用 jq 验证 JSON
cat ~/.openclaw/openclaw.json | jq .

# 方法 3：使用 Node.js

```

**来源**: `reference/source_doctor_command.md`

### 6.2 配置迁移会丢失数据吗？

**答案**：不会。doctor 在修复前会自动备份配置。

```bash
# 备份位置
~/.openclaw/openclaw.json.bak

# 如果需要恢复
cp ~/.openclaw/openclaw.json.bak ~/.openclaw/openclaw.json
```

**来源**: `reference/source_doctor_command.md`

### 6.3 如何禁用环境变量？

```bash
# macOS (launchctl)
launchctl unsetenv OPENCLAW_GATEWAY_TOKEN
launchctl unsetenv OPENCLAW_GATEWAY_PASSWORD

# Linux/WSL (当前 shell)
unset OPENCLAW_GATEWAY_TOKEN
unset OPENCLAW_GATEWAY_PASSWORD

# 永久禁用：编辑 ~/.bashrc 或 ~/.zshrc
# 删除相关的 export 语句
```
