# Retriever高级策略 - 一句话总结

## 一句话总结

**Retriever高级策略是通过 EnsembleRetriever 混合检索、ContextualCompression 重排序、MultiQuery 查询扩展、ParentDocument 父文档检索和 SelfQuery 自查询五种策略的灵活组合，将基础检索器升级为生产级高精度检索管道，在 RAG 系统中实现精确度与召回率的最优平衡。**

## 关键词解读

| 关键词 | 含义 | 为什么重要 |
|--------|------|------------|
| EnsembleRetriever | 用 RRF 算法融合多个检索器结果 | 混合检索的核心实现，BM25+向量双保险 |
| ContextualCompression | 装饰器模式叠加重排序/压缩 | 两阶段检索范式，粗筛+精排提升精确率 |
| MultiQuery | LLM 生成多个查询变体 | 查询扩展经典技术，提高召回率 |
| ParentDocument | 小块检索、大块返回 | 解决检索粒度与上下文完整性的矛盾 |
| SelfQuery | LLM 自动拆分语义+元数据过滤 | 结构化检索，精确过滤特定条件 |
| RRF | Reciprocal Rank Fusion | 排名倒数加权融合，多检索器投票机制 |
| 两阶段检索 | 先粗筛(k=20)再精排(top_n=5) | 生产级 RAG 的标准架构 |

---

## 核心公式

```
生产级检索管道 = MultiQuery(可选) → Ensemble(BM25 + Vector) → Rerank → Top-K

RRF 分数 = Σ weight_i / (rank_i + 60)

检索质量 = 召回率(Ensemble 保证) × 精确率(Rerank 保证)
```

## 快速参考

```python
# 最小可用：混合检索 + 重排序（生产级起点）
from langchain.retrievers import EnsembleRetriever, ContextualCompressionRetriever
from langchain_community.retrievers import BM25Retriever
from langchain_community.document_compressors import FlashrankRerank

ensemble = EnsembleRetriever(
    retrievers=[bm25_retriever, vector_retriever],
    weights=[0.4, 0.6],
)
final = ContextualCompressionRetriever(
    base_compressor=FlashrankRerank(top_n=5),
    base_retriever=ensemble,
)
results = final.invoke("你的查询")
```
