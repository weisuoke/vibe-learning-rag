# 节点函数约定 - 一句话总结

## 核心总结

**节点函数是 LangGraph 中定义状态转换逻辑的纯函数，通过标准化的签名约定（接收 State，返回 dict/Command）实现类型安全、灵活更新和容错性，是构建状态化工作流的基本单元。**

[来源: reference/source_节点函数约定_01.md, reference/context7_langgraph_01.md]

---

## 关键要点提炼

### 1. 函数签名约定

**核心**：`def node(state: State) -> dict`

- **必需参数**：State（当前状态）
- **可选参数**：Config（运行时配置）、Runtime（运行时上下文）
- **返回值类型**：dict（部分更新）、Command（流程控制）、None（不更新）、list（多个操作）
- **自动推断**：框架自动推断第一个参数类型作为 input_schema

[来源: reference/source_节点函数约定_01.md]

---

### 2. 部分状态更新机制

**核心**：只返回需要更新的字段，框架自动合并

```python
# ✅ 正确：只返回需要更新的字段
def my_node(state: State) -> dict:
    return {"x": state["x"] + 1}  # 其他字段保持不变

# ❌ 错误：返回完整状态
def bad_node(state: State) -> State:
    return state  # 会覆盖其他节点的更新
```

**关键点**：
- 使用 Reducer 函数控制合并逻辑（如 `add` 用于列表追加）
- 未返回的字段保持原值不变
- 支持多个节点并行更新不同字段

[来源: reference/source_节点函数约定_01.md, reference/context7_langgraph_01.md]

---

### 3. 异步节点支持

**核心**：框架自动检测并处理同步/异步函数

```python
# 同步节点：自动包装成异步执行
def sync_node(state: State) -> dict:
    result = sync_operation()
    return {"result": result}

# 异步节点：直接使用
async def async_node(state: State) -> dict:
    result = await async_operation()
    return {"result": result}
```

**关键点**：
- 同步函数在线程池中执行（`run_in_executor`）
- 异步函数直接在事件循环中执行
- **不推荐使用异步生成器**（函数体可能不执行）

[来源: reference/source_节点函数约定_01.md, reference/search_节点函数约定_01.md]

---

### 4. 错误处理与重试

**核心**：通过 RetryPolicy 配置自动重试

```python
from langgraph.types import RetryPolicy

builder.add_node(
    "api_call",
    my_node,
    retry_policy=RetryPolicy(
        max_attempts=3,
        initial_interval=1.0,
        backoff_factor=2.0,
        retry_on=lambda e: isinstance(e, (ConnectionError, TimeoutError))
    )
)
```

**关键点**：
- 支持指数退避算法
- 支持自定义重试条件（异常类型或 Callable）
- 支持抖动（jitter）避免雷鸣群效应
- **CancelledError 需要单独处理**（不是 Exception 子类）

[来源: reference/source_节点函数约定_02.md, reference/context7_langgraph_01.md, reference/search_节点函数约定_01.md]

---

### 5. Command 对象控制流程

**核心**：使用 Command 对象动态控制流程跳转

```python
from langgraph.types import Command
from typing import Literal

def router_node(state: State) -> Command[Literal["next", "end"]]:
    if state["x"] > 10:
        return Command(goto="end")
    return Command(goto="next", update={"x": state["x"] + 1})
```

**关键点**：
- 可以同时更新状态和跳转节点
- 使用 Literal 类型注解确保类型安全
- 支持父图通信（`Command.PARENT`）
- 指向无效节点会抛出 `InvalidUpdateError`

[来源: reference/source_节点函数约定_01.md, reference/search_节点函数约定_01.md]

---

### 6. 配置与上下文传递

**核心**：通过 Config 和 Runtime 参数传递配置和上下文

```python
from langgraph.types import RunnableConfig, Runtime

# 使用 Config 访问配置
def node_with_config(state: State, config: RunnableConfig) -> dict:
    api_key = config["configurable"]["api_key"]
    return {"result": call_api(api_key)}

# 使用 Runtime 访问运行时上下文
def node_with_runtime(state: State, runtime: Runtime[Context]) -> dict:
    user_id = runtime.context.get("user_id")
    return {"user_id": user_id}
```

**关键点**：
- Config 用于静态配置（API keys、超时设置等）
- Runtime 用于动态上下文（用户 ID、请求 ID 等）
- 避免污染状态结构

[来源: reference/source_节点函数约定_01.md, reference/context7_langgraph_01.md]

### 7. 常见误区

**8 大常见误区**：

1. **异步生成器可以作为节点函数** → 不支持，函数体不执行
2. **节点函数必须返回完整状态** → 只需返回需要更新的字段
3. **所有异常都会自动重试** → 需要配置 RetryPolicy
4. **streamEvents 中的错误会自动传播** → 需要在循环内处理错误
5. **CancelledError 会被正常捕获** → 需要单独处理
6. **Command 指向无效节点会静默失败** → 会抛出 InvalidUpdateError
7. **不理解自动类型推断机制** → 框架自动推断第一个参数类型
8. **同步函数不能作为节点函数** → 会自动包装成异步执行

[来源: reference/search_节点函数约定_01.md, reference/source_节点函数约定_01.md]

### 8. 设计哲学

**四大设计哲学**：

1. **约定优于配置**：通过合理的约定减少配置，让开发者专注于业务逻辑
2. **声明式编程**：描述"做什么"而不是"怎么做"
3. **渐进式复杂度**：简单场景简单用，复杂场景有能力
4. **容错优先**：假设失败会发生，提前设计容错机制

[来源: reference/source_节点函数约定_02.md]

## 快速参考

### 节点函数模板

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义状态
class State(TypedDict):
    field1: str
    field2: int

# 2. 定义节点函数
def my_node(state: State) -> dict:
    """节点函数说明"""
    return {"field2": state["field2"] + 1}

# 3. 构建图
builder = StateGraph(State)
builder.add_node("my_node", my_node)
builder.add_edge(START, "my_node")
builder.add_edge("my_node", END)

# 4. 编译并执行
graph = builder.compile()
result = graph.invoke({"field1": "input", "field2": 0})
```

### 核心要点速查表

| 要点 | 说明 | 示例 |
|------|------|------|
| **函数签名** | `def node(state: State) -> dict` | 第一个参数是 State，返回 dict |
| **返回值** | 只返回需要更新的字段 | `return {"count": 1}` |
| **异步节点** | 使用 `async def` | `async def node(state: State) -> dict` |
| **错误处理** | 配置 RetryPolicy | `retry_policy=RetryPolicy(max_attempts=3)` |
| **流程控制** | 使用 Command 对象 | `return Command(goto="next")` |
| **配置传递** | 使用 Config 参数 | `def node(state: State, config: RunnableConfig)` |

## 总结

节点函数约定是 LangGraph 的核心抽象，它通过标准化的接口实现了：

1. **类型安全**：自动推断输入输出类型
2. **灵活更新**：部分状态更新机制
3. **容错性**：自动重试和错误处理
4. **流程控制**：Command 对象动态路由
5. **异步支持**：自动检测同步/异步函数

掌握节点函数约定是掌握 LangGraph 的第一步，也是构建健壮状态化工作流的基础。
