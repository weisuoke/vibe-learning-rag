## 卡片 1：边的直觉理解

**一句话：** 边是连接节点的有向路径，控制图的执行流程。

**举例：**
```python
graph.add_edge(START, "node1")  # START → node1
graph.add_edge("node1", "node2")  # node1 → node2
graph.add_edge("node2", END)  # node2 → END
```

**类比：**
- **前端类比**：路由规则（React Router 的路由配置）
- **日常生活类比**：传送带（工厂流水线上的传送带）

**应用：** 在 LangGraph 中，边定义了节点的执行顺序，是构建工作流的基础。

[来源: reference/source_edges_01_add_edge.md]

## 卡片 2：普通边 vs 条件边

**一句话：** 普通边是固定路由，条件边是动态路由。

**对比：**
```python
# 普通边：固定路由
graph.add_edge("node1", "node2")  # node1 完成后总是执行 node2

# 条件边：动态路由
graph.add_conditional_edges("node1", route_func)  # 根据状态决定下一个节点
```

**关键区别：**
- 普通边：编译时确定路径
- 条件边：运行时动态决策

**应用：** 固定流程用普通边，需要决策的流程用条件边。

[来源: reference/source_edges_01_add_edge.md, reference/source_edges_02_add_conditional_edges.md]

## 卡片 3：路由函数的核心规则

**一句话：** 路由函数是纯函数，只读取状态不修改状态。

**规则：**
```python
# ✅ 正确：只读取状态
def route_func(state: State) -> Literal["path_a", "path_b"]:
    if state["score"] > 0.5:
        return "path_a"
    return "path_b"

# ❌ 错误：修改状态
def route_func(state: State) -> str:
    state["counter"] += 1  # 不要这样做！
    return "path_a"
```

**为什么：**
- 路由函数的职责是决策，不是修改状态
- 状态修改应该在节点函数中完成

**应用：** 保持路由函数的纯函数特性，确保可预测性和可测试性。

[来源: reference/context7_langgraph_01_edges_routing.md]

## 卡片 4：类型提示的重要性

**一句话：** 使用 `Literal` 类型提示明确可能的路由目标。

**示例：**
```python
from typing import Literal

def route_func(state: State) -> Literal["approve", "review", "reject"]:
    """使用 Literal 明确可能的返回值"""
    score = state["score"]
    if score > 0.8:
        return "approve"
    elif score > 0.5:
        return "review"
    else:
        return "reject"
```

**好处：**
- IDE 提供代码补全
- 编译时发现类型错误
- 图可视化更准确

**应用：** 始终为路由函数添加 `Literal` 类型提示。

[来源: reference/source_edges_02_add_conditional_edges.md]

## 卡片 5：path_map 的三种形式

**一句话：** path_map 将路由函数返回值映射到节点名称。

**三种形式：**
```python
# 1. None（默认）：路由函数直接返回节点名称
graph.add_conditional_edges("node1", route_func)

# 2. dict（映射）：将逻辑键映射到节点名称
graph.add_conditional_edges(
    "node1",
    route_func,
    {"yes": "node2", "no": "node3"}
)

# 3. list（列表）：用于图可视化和验证
graph.add_conditional_edges(
    "node1",
    route_func,
    ["node2", "node3", "node4"]
)
```

**应用：** 使用 dict 形式实现路由逻辑与节点名称的解耦。

[来源: reference/source_edges_02_add_conditional_edges.md]

## 卡片 6：等待边的 AND 逻辑

**一句话：** 等待边（多起始节点）使用 AND 逻辑，必须等待所有节点完成。

**示例：**
```python
graph.add_edge(["node_a", "node_b", "node_c"], "merge")
# merge 节点会等待 node_a、node_b、node_c 全部完成
```

**执行流程：**
```
START
  ↓
┌─────┬─────┬─────┐
│node_a│node_b│node_c│  (并行执行)
└─────┴─────┴─────┘
  ↓     ↓     ↓
  └─────┼─────┘  (等待所有节点完成)
        ↓
      merge  (所有节点完成后才执行)
```

**应用：** 用于并行执行后的同步点。

[来源: reference/source_edges_01_add_edge.md]

## 卡片 7：简单分支模式（if-else）

**一句话：** 使用条件边实现 if-else 决策逻辑。

**模式：**
```python
def route_func(state: State) -> Literal["path_a", "path_b"]:
    if condition:
        return "path_a"
    return "path_b"

graph.add_conditional_edges(
    "source",
    route_func,
    {"path_a": "node_a", "path_b": "node_b"}
)
```

**应用场景：**
- 内容审核（通过/拒绝）
- 用户验证（有效/无效）
- 数据检查（正常/异常）

**应用：** 最常见的条件路由模式，适合二选一决策。

[来源: reference/context7_langgraph_01_edges_routing.md]

## 卡片 8：多路由模式（switch-case）

**一句话：** 使用条件边实现 switch-case 多路由决策。

**模式：**
```python
def route_func(state: State) -> Literal["path_a", "path_b", "path_c", END]:
    if condition_a:
        return "path_a"
    elif condition_b:
        return "path_b"
    elif condition_c:
        return "path_c"
    return END

graph.add_conditional_edges("source", route_func)
```

**应用场景：**
- 质量分级（高/中/低）
- 任务分类（类型A/B/C）
- 优先级路由（紧急/普通/低优先级）

**应用：** 适合多选一决策，支持多个目标节点。

[来源: reference/context7_langgraph_01_edges_routing.md]

## 卡片 9：循环路由模式（retry/fallback）

**一句话：** 使用条件边实现循环重试和回退机制。

**模式：**
```python
def route_func(state: State) -> Literal["retry", "fallback", END]:
    # 检查退出条件
    if state.get("retry_count", 0) >= 3:
        return END

    # 决定是否重试
    if should_retry(state):
        return "retry"
    elif should_fallback(state):
        return "fallback"
    return END

graph.add_conditional_edges(
    "process",
    route_func,
    {
        "retry": "process",  # 循环回自己
        "fallback": "fallback_node",
        END: END
    }
)
```

**关键点：**
- 必须有退出条件（避免无限循环）
- 在状态中添加计数器
- 提供回退路径

**应用：** 错误处理、重试机制、降级策略。

[来源: reference/context7_langgraph_01_edges_routing.md]

## 卡片 10：边的最佳实践总结

**一句话：** 遵循设计原则，构建清晰可维护的路由逻辑。

**五大原则：**

1. **类型安全**：使用 `Literal` 类型提示
2. **纯函数设计**：路由函数只读取状态
3. **边界处理**：处理缺失值和异常值
4. **清晰逻辑**：使用扁平化的条件判断
5. **可观测性**：添加日志和调试信息

**常见误区：**
- ❌ 路由函数修改状态
- ❌ 等待边理解为 OR 逻辑
- ❌ 返回未定义的节点
- ❌ 循环路由没有退出条件

**设计模式：**
- 简单分支（if-else）
- 多路由（switch-case）
- 循环路由（retry/fallback）
- 入口路由（START 条件边）
- 多层路由（嵌套决策）

**应用：** 遵循这些原则，构建健壮的状态化工作流系统。

[来源: reference/context7_langgraph_01_edges_routing.md, reference/search_edges_02_reddit.md]

### 普通边语法
```python
# 单起始节点
graph.add_edge("node1", "node2")

# 多起始节点（等待边）

# 快捷方法
graph.add_edge(START, "node1")  # 入口

```

### 条件边语法
```python
# 基础语法
def route_func(state: State) -> Literal["path_a", "path_b"]:
    return "path_a" if condition else "path_b"

graph.add_conditional_edges("source", route_func)

# 使用 path_map
graph.add_conditional_edges(
    "source",
    route_func,
    {"path_a": "node_a", "path_b": "node_b"}
)
```

### 路由函数模板
```python
def route_func(state: State) -> Literal["path_a", "path_b", END]:
    """路由函数模板"""
    # 1. 获取状态值（安全获取）
    value = state.get("key", default_value)

    # 2. 处理边界情况
    if value < 0 or value > 1:
        return END

    # 3. 路由决策
    if value > 0.5:
        return "path_a"
    return "path_b"
```

### 常见模式速查
```python
# 简单分支（if-else）
def route(state): return "a" if cond else "b"

# 多路由（switch-case）
def route(state):
    if cond_a: return "a"
    elif cond_b: return "b"
    elif cond_c: return "c"
    return END

# 循环路由（retry）
def route(state):
    if state.get("retry_count", 0) < 3:
        return "retry"
    return END
```
