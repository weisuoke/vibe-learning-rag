## Common Planning Patterns

### Pattern 1: Specification-First
1. Write detailed specification
2. Break down into tasks
3. Create implementation plan
4. Execute incrementally
5. Review and iterate

### Pattern 2: Plan-Execute-Review
1. Analyze requirements
2. Create execution plan
3. Implement in chunks
4. Review each chunk
5. Refine and continue

### Pattern 3: Autonomous Planning
1. Agent analyzes task
2. Agent creates plan
3. Agent executes plan
4. Agent monitors progress
5. Agent adapts as needed

## Best Practices from 2025-2026

### Planning Phase
- **Be Specific**: Clear, detailed specifications
- **Break Down**: Small, manageable tasks
- **Document**: Write plans to files
- **Review**: Human review of plans
- **Iterate**: Refine plans based on feedback

### Execution Phase
- **Incremental**: Build in small chunks
- **Test-Driven**: Write tests alongside code
- **Review**: Continuous code review
- **Adapt**: Adjust plan as needed
- **Document**: Keep plans updated

### Tools & Techniques
- **File-Based Plans**: Maximum observability
- **Version Control**: Track plan evolution
- **Checkpoints**: Regular review points
- **Rollback**: Easy to revert if needed
- **Collaboration**: Share plans with team
