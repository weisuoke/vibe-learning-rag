# 核心概念4：缓存机制

> 理解 LangChain Embedding 的缓存优化策略

---

## 什么是 Embedding 缓存机制？

**一句话定义：** Embedding 缓存机制通过存储已计算的向量结果，避免重复调用 API，显著降低成本和延迟。

### 为什么需要缓存？

在 RAG 系统中，Embedding 计算是一个高成本操作：

```
成本分析：
- OpenAI text-embedding-ada-002: $0.0001 / 1K tokens
- 1万个文档（平均500 tokens）= $0.50
- 重复计算10次 = $5.00

时间分析：
- 单次 API 调用延迟：100-300ms
- 1万个文档串行计算：16-50分钟
- 批量处理（100个/批）：1-3分钟
```

**缓存的价值**：
- **成本节省**：避免重复 API 调用，节省 90%+ 成本
- **性能提升**：从缓存读取比 API 调用快 100-1000 倍
- **开发体验**：快速迭代，无需等待重复计算

## 直觉理解

### 类比1：前端开发 - 浏览器缓存

```javascript
// 没有缓存：每次都请求服务器
async function getImage(url) {
  const response = await fetch(url);  // 每次都下载
  return response.blob();
}

// 有缓存：第一次请求后缓存
const imageCache = new Map();
async function getImageCached(url) {
  if (imageCache.has(url)) {
    return imageCache.get(url);  // 从缓存读取（快）
  }
  const response = await fetch(url);  // 第一次下载
  const blob = await response.blob();
  imageCache.set(url, blob);  // 存入缓存
  return blob;
}
```

**Embedding 缓存类似**：
- 文本 = URL
- Embedding 向量 = 图片数据
- 第一次计算后缓存，后续直接读取

### 类比2：日常生活 - 字典查词

```
没有缓存：
你：查"apple"的意思
字典：翻到A部分，找到apple，告诉你"苹果"
你：查"apple"的意思（第二次）
字典：再翻一次，再告诉你"苹果"

有缓存：
你：查"apple"的意思
字典：翻到A部分，找到apple，告诉你"苹果"
你：在笔记本上记下"apple = 苹果"（缓存）
你：查"apple"的意思（第二次）
你：直接看笔记本，"苹果"（从缓存读取）
```

## CacheBackedEmbeddings 设计

### 装饰器模式

LangChain 使用装饰器模式实现缓存，不侵入原始 Embeddings 实现：

```python
from langchain.embeddings import CacheBackedEmbeddings
from langchain_openai import OpenAIEmbeddings
from langchain.storage import LocalFileStore

# 原始 Embeddings（未缓存）
base_embeddings = OpenAIEmbeddings()

# 添加缓存层（装饰器）
store = LocalFileStore("./embedding_cache")
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    namespace="openai-ada-002"
)

# 使用方式完全相同
vectors = cached_embeddings.embed_documents(["hello", "world"])
```

**设计优势**：
- **透明性**：使用方式与原始 Embeddings 完全相同
- **灵活性**：可以包装任何 Embeddings 实现
- **可组合性**：可以与其他装饰器组合使用

### 核心组件

```
┌─────────────────────────────────────────┐
│     CacheBackedEmbeddings（装饰器）      │
├─────────────────────────────────────────┤
│  - underlying_embeddings: Embeddings    │  ← 原始 Embeddings
│  - document_embedding_store: BaseStore  │  ← 文档缓存
│  - query_embedding_store: BaseStore?    │  ← 查询缓存（可选）
│  - batch_size: int?                     │  ← 批量大小
└─────────────────────────────────────────┘
```

## 缓存键生成策略

### 哈希算法选择

LangChain 支持多种哈希算法生成缓存键：

```python
from langchain.embeddings import CacheBackedEmbeddings

# SHA-1（默认，快速但不抗碰撞）
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    key_encoder="sha1"  # 默认
)

# BLAKE2b（推荐，快速且安全）
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    key_encoder="blake2b"
)

# SHA-256（安全但较慢）
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    key_encoder="sha256"
)
```

### 算法对比

| 算法 | 速度 | 安全性 | 适用场景 |
|------|------|--------|----------|
| SHA-1 | 最快 | 低（存在碰撞风险） | 开发环境、非敏感数据 |
| BLAKE2b | 快 | 高（抗碰撞） | **生产环境推荐** |
| SHA-256 | 中等 | 高（抗碰撞） | 安全敏感场景 |
| SHA-512 | 慢 | 最高 | 极高安全要求 |

### 安全性考虑

**SHA-1 的风险**（来自源码注释）：

```python
# 源码警告
warnings.warn(
    "Using default key encoder: SHA-1 is *not* collision-resistant. "
    "While acceptable for most cache scenarios, a motivated attacker "
    "can craft two different payloads that map to the same cache key. "
    "If that risk matters in your environment, supply a stronger "
    "encoder (e.g. SHA-256 or BLAKE2) via the `key_encoder` argument.",
    category=UserWarning,
)
```

**实际建议**：
- **开发环境**：SHA-1（性能优先）
- **生产环境**：BLAKE2b（平衡性能和安全）
- **金融/医疗**：SHA-256 或 SHA-512（安全优先）

## 缓存策略：文档 vs 查询

### 默认行为

```python
class CacheBackedEmbeddings:
    def __init__(
        self,
        underlying_embeddings: Embeddings,
        document_embedding_store: BaseStore,  # 必需
        query_embedding_store: BaseStore | None = None,  # 可选
    ):
        ...
```

**设计理由**：
- **文档嵌入**：默认缓存（文档相对固定，重复计算多）
- **查询嵌入**：默认不缓存（查询多样化，缓存命中率低）

### 何时缓存查询？

```python
# 场景1：固定查询集（评估、测试）
test_queries = ["如何使用RAG？", "什么是向量数据库？"]
query_store = LocalFileStore("./query_cache")

cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    document_store,
    query_embedding_cache=query_store  # 启用查询缓存
)

# 场景2：Few-shot 示例（固定的示例查询）
few_shot_queries = load_few_shot_examples()
```

**查询缓存的权衡**：

| 维度 | 优点 | 缺点 |
|------|------|------|
| 命中率 | 固定查询集命中率高 | 用户查询多样化，命中率低 |
| 存储 | 节省重复查询成本 | 占用存储空间 |
| 维护 | 简化测试流程 | 需要定期清理 |

## 批量处理与优化

### 批量获取与更新

```python
def embed_documents(self, texts: list[str]) -> list[list[float]]:
    # 1. 批量获取缓存（mget）
    vectors = self.document_embedding_store.mget(texts)

    # 2. 找出缺失的文本
    missing_indices = [i for i, v in enumerate(vectors) if v is None]

    # 3. 分批计算缺失的嵌入
    for batch_indices in batch_iterate(self.batch_size, missing_indices):
        missing_texts = [texts[i] for i in batch_indices]
        missing_vectors = self.underlying_embeddings.embed_documents(missing_texts)

        # 4. 批量更新缓存（mset）
        self.document_embedding_store.mset(
            list(zip(missing_texts, missing_vectors))
        )

        # 5. 更新结果
        for index, vector in zip(batch_indices, missing_vectors):
            vectors[index] = vector

    return vectors
```

### 性能优化点

**1. 批量操作（mget/mset）**

```python
# ❌ 低效：逐个获取
for text in texts:
    vector = store.get(text)  # N次I/O

# ✅ 高效：批量获取
vectors = store.mget(texts)  # 1次I/O
```

**2. 分批处理（batch_size）**

```python
# 场景：10000个文档，其中5000个未缓存

# 不分批：一次性计算5000个
# - 内存占用：5000 * 1536 * 4 bytes ≈ 30MB
# - API限制：可能超过单次请求限制

# 分批：每批100个
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    batch_size=100  # 每批100个
)
# - 内存占用：100 * 1536 * 4 bytes ≈ 600KB
# - API友好：符合速率限制
```

**3. 增量计算**

```python
# 第一次：计算并缓存1000个文档
docs_v1 = load_documents()  # 1000个
vectors_v1 = cached_embeddings.embed_documents(docs_v1)
# → 1000次API调用

# 第二次：新增100个文档
docs_v2 = docs_v1 + load_new_documents()  # 1100个
vectors_v2 = cached_embeddings.embed_documents(docs_v2)
# → 只有100次API调用（1000个从缓存读取）
```

## 序列化与反序列化

### JSON 序列化

LangChain 使用 JSON 序列化向量数据：

```python
def _value_serializer(value: Sequence[float]) -> bytes:
    """将向量序列化为字节"""
    return json.dumps(value).encode()

def _value_deserializer(serialized_value: bytes) -> list[float]:
    """将字节反序列化为向量"""
    return json.loads(serialized_value.decode())
```

**为什么选择 JSON？**

| 维度 | JSON | Pickle | NumPy |
|------|------|--------|-------|
| 跨语言 | ✅ 支持 | ❌ Python专用 | ⚠️ 部分支持 |
| 安全性 | ✅ 安全 | ❌ 不安全 | ✅ 安全 |
| 可读性 | ✅ 人类可读 | ❌ 二进制 | ❌ 二进制 |
| 性能 | ⚠️ 中等 | ✅ 快 | ✅ 快 |
| 体积 | ⚠️ 较大 | ✅ 小 | ✅ 小 |

**设计权衡**：
- 优先考虑安全性和兼容性
- 牺牲部分性能和存储效率
- 适合大多数 RAG 场景

### EncoderBackedStore 包装

```python
from langchain.storage import EncoderBackedStore

# 将 ByteStore 包装为 BaseStore[str, list[float]]
document_embedding_store = EncoderBackedStore[str, list[float]](
    document_embedding_cache,  # ByteStore
    key_encoder,               # 键编码器（哈希函数）
    _value_serializer,         # 值序列化器
    _value_deserializer,       # 值反序列化器
)
```

**包装器的作用**：
1. **类型转换**：ByteStore → BaseStore[str, list[float]]
2. **键编码**：文本 → 哈希值
3. **值序列化**：向量 → 字节
4. **值反序列化**：字节 → 向量

## from_bytes_store 工厂方法

### 便捷构造器

```python
@classmethod
def from_bytes_store(
    cls,
    underlying_embeddings: Embeddings,
    document_embedding_cache: ByteStore,
    *,
    namespace: str = "",
    batch_size: int | None = None,
    query_embedding_cache: bool | ByteStore = False,
    key_encoder: Callable[[str], str] | Literal["sha1", "blake2b", "sha256", "sha512"] = "sha1",
) -> CacheBackedEmbeddings:
    """从 ByteStore 创建 CacheBackedEmbeddings"""
```

**参数说明**：

| 参数 | 类型 | 说明 |
|------|------|------|
| `underlying_embeddings` | Embeddings | 原始 Embeddings 实现 |
| `document_embedding_cache` | ByteStore | 文档缓存存储 |
| `namespace` | str | 命名空间（区分不同模型） |
| `batch_size` | int? | 批量大小 |
| `query_embedding_cache` | bool/ByteStore | 查询缓存（可选） |
| `key_encoder` | str/Callable | 哈希算法 |

### 使用示例

```python
from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings

# 1. 创建原始 Embeddings
base_embeddings = OpenAIEmbeddings(model="text-embedding-ada-002")

# 2. 创建缓存存储
store = LocalFileStore("./embedding_cache")

# 3. 使用工厂方法创建缓存 Embeddings
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    namespace="ada-002",  # 命名空间
    batch_size=100,       # 批量大小
    key_encoder="blake2b" # 哈希算法
)

# 4. 使用（与原始 Embeddings 完全相同）
documents = ["文档1", "文档2", "文档3"]
vectors = cached_embeddings.embed_documents(documents)
```

## 异步支持

### 异步实现

```python
async def aembed_documents(self, texts: list[str]) -> list[list[float]]:
    """异步嵌入文档"""
    # 1. 异步批量获取缓存
    vectors = await self.document_embedding_store.amget(texts)

    # 2. 找出缺失的文本
    missing_indices = [i for i, v in enumerate(vectors) if v is None]

    # 3. 分批异步计算
    for batch_indices in batch_iterate(self.batch_size, missing_indices):
        missing_texts = [texts[i] for i in batch_indices]

        # 异步调用底层 Embeddings
        missing_vectors = await self.underlying_embeddings.aembed_documents(missing_texts)

        # 4. 异步批量更新缓存
        await self.document_embedding_store.amset(
            list(zip(missing_texts, missing_vectors))
        )

        # 5. 更新结果
        for index, vector in zip(batch_indices, missing_vectors):
            vectors[index] = vector

    return vectors
```

### 性能优势

```python
import asyncio
from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings

# 创建缓存 Embeddings
store = LocalFileStore("./cache")
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store
)

# 同步版本
def sync_embed():
    documents = [f"文档{i}" for i in range(1000)]
    vectors = cached_embeddings.embed_documents(documents)
    return vectors

# 异步版本
async def async_embed():
    documents = [f"文档{i}" for i in range(1000)]
    vectors = await cached_embeddings.aembed_documents(documents)
    return vectors

# 性能对比
import time

# 同步：约 10-30 秒（取决于缓存命中率）
start = time.time()
sync_embed()
print(f"同步耗时: {time.time() - start:.2f}s")

# 异步：约 5-15 秒（更快）
start = time.time()
asyncio.run(async_embed())
print(f"异步耗时: {time.time() - start:.2f}s")
```

## 在 RAG 中的应用

### 场景1：文档索引构建

```python
from langchain.embeddings import CacheBackedEmbeddings
from langchain.storage import LocalFileStore
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma

# 1. 创建缓存 Embeddings
store = LocalFileStore("./embeddings_cache")
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    namespace="openai-ada-002"
)

# 2. 加载文档
from langchain_community.document_loaders import TextLoader
loader = TextLoader("documents.txt")
documents = loader.load()

# 3. 构建向量索引（第一次：计算并缓存）
vectorstore = Chroma.from_documents(
    documents,
    cached_embeddings,
    persist_directory="./chroma_db"
)

# 4. 后续运行（从缓存读取，节省成本）
vectorstore = Chroma.from_documents(
    documents,
    cached_embeddings,  # 从缓存读取
    persist_directory="./chroma_db"
)
```

### 场景2：增量索引更新

```python
# 初始文档
initial_docs = load_documents("batch1")
vectorstore = Chroma.from_documents(
    initial_docs,
    cached_embeddings
)

# 新增文档（只计算新文档的嵌入）
new_docs = load_documents("batch2")
all_docs = initial_docs + new_docs

# 增量更新（initial_docs 从缓存读取，new_docs 计算并缓存）
vectorstore = Chroma.from_documents(
    all_docs,
    cached_embeddings
)
```

### 场景3：开发迭代

```python
# 开发阶段：频繁修改代码，但文档不变
documents = load_documents()

# 第一次运行：计算并缓存
vectorstore = Chroma.from_documents(documents, cached_embeddings)
results = vectorstore.similarity_search("查询1")

# 修改代码后重新运行：从缓存读取（快速迭代）
vectorstore = Chroma.from_documents(documents, cached_embeddings)
results = vectorstore.similarity_search("查询2")
```

## 常见问题

### 问题1：缓存未命中

**症状**：每次运行都重新计算嵌入

**原因**：
1. 文本内容发生变化（空格、换行符）
2. namespace 不一致
3. 哈希算法不一致

**解决方案**：
```python
# 1. 标准化文本
def normalize_text(text: str) -> str:
    return text.strip().replace("\n", " ")

documents = [normalize_text(doc) for doc in raw_documents]

# 2. 固定 namespace
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    namespace="my-project-v1"  # 固定 namespace
)

# 3. 固定哈希算法
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    key_encoder="blake2b"  # 固定算法
)
```

### 问题2：缓存占用空间过大

**症状**：缓存目录占用数GB空间

**原因**：
1. 缓存了大量低价值数据
2. 未定期清理旧缓存

**解决方案**：
```python
# 1. 只缓存文档，不缓存查询
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    query_embedding_cache=False  # 不缓存查询
)

# 2. 定期清理缓存
import shutil
import os

def clean_old_cache(cache_dir: str, days: int = 30):
    """清理超过 N 天的缓存"""
    import time
    now = time.time()
    for filename in os.listdir(cache_dir):
        filepath = os.path.join(cache_dir, filename)
        if os.path.getmtime(filepath) < now - days * 86400:
            os.remove(filepath)

clean_old_cache("./embedding_cache", days=30)
```

### 问题3：版本兼容性

**症状**：升级 LangChain 后缓存失效

**原因**：
1. 序列化格式变化
2. 哈希算法变化

**解决方案**：
```python
# 使用版本化的 namespace
from langchain import __version__

cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    base_embeddings,
    store,
    namespace=f"ada-002-langchain-{__version__}"  # 版本化
)
```
