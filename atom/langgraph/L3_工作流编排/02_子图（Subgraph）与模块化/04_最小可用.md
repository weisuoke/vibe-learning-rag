# 子图（Subgraph）与模块化 - 最小可用

> 掌握以下内容，就能开始使用子图构建模块化工作流

---

## 核心理念

**20% 的核心知识解决 80% 的问题**

子图的最小可用知识集中在 4 个操作上：
1. 定义子图的 StateGraph 并 compile()
2. 通过 add_node() 将编译后的子图添加到父图
3. 理解共享状态键的自动传递机制
4. 使用 input_schema/output_schema 控制子图接口

---

## 4.1 定义子图并编译

子图就是一个普通的 StateGraph，定义完节点和边之后调用 `compile()` 即可。

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 子图的状态定义
class SubState(TypedDict):
    query: str
    result: str

# 子图的节点
def process(state: SubState):
    return {"result": f"processed: {state['query']}"}

# 构建并编译子图
sub_builder = StateGraph(SubState)
sub_builder.add_node(process)
sub_builder.add_edge(START, "process")
sub_builder.add_edge("process", END)
compiled_sub = sub_builder.compile()
```

编译后的子图是一个 `CompiledStateGraph` 实例，既可以独立运行，也可以嵌入父图。

---

## 4.2 将子图添加到父图

把编译后的子图像普通节点一样用 `add_node()` 添加到父图。

```python
class ParentState(TypedDict):
    query: str
    result: str
    final_answer: str

def summarize(state: ParentState):
    return {"final_answer": f"Answer: {state['result']}"}

# 构建父图
parent_builder = StateGraph(ParentState)
parent_builder.add_node("sub_workflow", compiled_sub)  # 子图作为节点
parent_builder.add_node(summarize)
parent_builder.add_edge(START, "sub_workflow")
parent_builder.add_edge("sub_workflow", "summarize")
parent_builder.add_edge("summarize", END)

graph = parent_builder.compile()
result = graph.invoke({"query": "hello"})
# result: {'query': 'hello', 'result': 'processed: hello', 'final_answer': 'Answer: processed: hello'}
```

---

## 4.3 共享状态键的自动传递

当父图和子图的 State 中有**同名字段**时，LangGraph 会自动在两者之间传递数据，无需手动转换。

```python
# 父图状态
class ParentState(TypedDict):
    query: str          # 同名键 -> 自动传给子图
    result: str         # 同名键 -> 自动从子图接收
    metadata: dict      # 子图中没有 -> 不传递，但保留

# 子图状态
class SubState(TypedDict):
    query: str          # 同名键 -> 自动从父图接收
    result: str         # 同名键 -> 自动传回父图
    internal: int       # 父图中没有 -> 子图私有
```

**规则很简单**：同名键自动共享，不同名键各自私有。这是最常用的集成方式。

---

## 4.4 使用 input/output_schema 控制接口

当需要精确控制子图接收和返回哪些字段时，使用 `input` 和 `output` 参数。

```python
# 定义输入接口：子图只接收 query
class SubInput(TypedDict):
    query: str

# 定义输出接口：子图只返回 result
class SubOutput(TypedDict):
    result: str

# 子图的完整内部状态
class SubState(TypedDict):
    query: str
    result: str
    internal_steps: list  # 私有字段，不会暴露给父图

# 使用 input/output 参数构建子图
sub_builder = StateGraph(SubState, input=SubInput, output=SubOutput)
sub_builder.add_node(process)
sub_builder.add_edge(START, "process")
sub_builder.add_edge("process", END)
compiled_sub = sub_builder.compile()
```

`input` 限制子图接收的字段，`output` 限制子图返回的字段。内部状态对父图完全不可见。

## 这些知识足以做什么？

掌握以上 4 个核心操作，你可以：

- **拆分复杂工作流**：将 10+ 节点的大图拆分为多个 3-5 节点的子图
- **实现模块复用**：同一个子图在不同父图中使用
- **隔离内部状态**：子图的临时变量不污染父图
- **构建 RAG 管道**：检索子图、生成子图、审核子图各自独立
- **多人协作开发**：团队成员各自负责不同子图，通过接口集成
