# 状态转换函数 - 30字核心

**状态转换函数是节点内部将输入状态转换为输出更新的处理逻辑，是 LangGraph 数据流转的核心引擎。**

---

## 关键词解析

### 状态转换函数 = 节点的"加工车间"

- **状态转换**：接收完整状态，读取所需字段，经过业务逻辑处理后产出新值
- **函数**：签名固定为 `(State) -> dict`，接收完整状态，返回部分更新
- **节点内部**：每个节点就是一个状态转换函数，是图执行的最小计算单元

---

## 核心公式

```
输入状态(State) → 节点函数(转换逻辑) → 输出更新(Partial<State>)
```

```python
# 源码层面：节点函数的标准签名
def my_node(state: State) -> dict:
    # 1. 读取需要的字段
    messages = state["messages"]
    # 2. 执行转换逻辑
    result = process(messages)
    # 3. 返回部分更新（只返回变化的字段）
    return {"result": result}
```

[来源: sourcecode/langgraph/libs/langgraph/langgraph/graph/state.py]

---

## 与相关概念的区分

| 概念 | 关注点 | 类比 |
|------|--------|------|
| **Reducer** | 如何合并多个更新到同一字段 | 快递分拣员 |
| **状态转换函数** | 节点内部如何处理数据并产出更新 | 工厂加工车间 |
| **状态传递** | 数据如何在节点间流动 | 传送带 |
| **部分状态更新** | 只返回变化字段的机制 | 只汇报变更项 |

---

## 记忆锚点

> **状态转换函数 = "接收原料 → 加工处理 → 输出成品"**

每个节点做三件事：从状态中取数据、处理数据、把结果写回状态。框架负责剩下的一切——状态保留、Reducer 合并、节点调度。
