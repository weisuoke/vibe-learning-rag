# 01_30字核心

## 核心定义

**LangGraph 的状态传递与上下文是通过 Channel 机制和 RunnableConfig 实现节点间数据流转和运行时配置传递的核心系统。**

---

## 详细解释

### 什么是状态传递与上下文？

在 LangGraph 中，状态传递与上下文是两个紧密相关但又有明确区分的概念：

**状态传递（State Passing）**：
- 指的是**可变数据**在节点之间的流转
- 通过 **Channel** 机制实现存储和传递
- 节点通过 **ChannelRead** 读取状态，通过 **ChannelWrite** 写入状态
- 状态在执行过程中会被不断更新和合并

**上下文传递（Context Passing）**：
- 指的是**不可变配置**在运行时的传递
- 通过 **RunnableConfig** 作为载体
- 包含 **Runtime Context**（运行时上下文）和配置信息
- 适合传递数据库连接、用户 ID、模型配置等不变的信息

### 核心价值

1. **解耦节点逻辑**：节点不需要知道状态如何存储，只需声明读写需求
2. **灵活的数据流**：支持复杂的状态流转路径，包括条件分支、循环、并行
3. **类型安全**：通过 TypedDict 定义状态 Schema，提供类型检查
4. **可追溯性**：所有状态变化都可以通过 Checkpoint 机制追踪
5. **上下文隔离**：State 和 Context 分离，避免混淆可变和不可变数据

### 在 LangGraph 中的体现

```python
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.runtime import Runtime
from dataclasses import dataclass

# 1. 定义状态 Schema（可变数据）
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]  # 使用 reducer 累积消息
    user_query: str                          # 用户查询
    search_results: list                     # 搜索结果
    final_answer: str                        # 最终答案

# 2. 定义上下文 Schema（不可变配置）
@dataclass
class Context:
    user_id: str
    model_name: str
    temperature: float

# 3. 节点函数：接收 state 和 runtime context
def search_node(state: AgentState, runtime: Runtime[Context]) -> dict:
    """搜索节点：读取状态，执行搜索，返回部分更新"""
    query = state["user_query"]
    user_id = runtime.context.user_id  # 从 context 获取用户 ID

    # 执行搜索逻辑
    results = perform_search(query, user_id)

    # 返回部分状态更新（只更新 search_results）
    return {"search_results": results}

def answer_node(state: AgentState, runtime: Runtime[Context]) -> dict:
    """回答节点：读取搜索结果，生成答案"""
    results = state["search_results"]
    model = runtime.context.model_name

    # 生成答案
    answer = generate_answer(results, model)

    # 返回部分状态更新
    return {"final_answer": answer}

# 4. 构建图
builder = StateGraph(AgentState, context_schema=Context)
builder.add_node("search", search_node)
builder.add_node("answer", answer_node)
builder.add_edge(START, "search")
builder.add_edge("search", "answer")
builder.add_edge("answer", END)

graph = builder.compile()

# 5. 执行图：传递初始状态和上下文
result = graph.invoke(
    {"user_query": "什么是 LangGraph？"},  # 初始状态
    context=Context(                        # 运行时上下文
        user_id="user_123",
        model_name="gpt-4",
        temperature=0.7
    )
)

print(result["final_answer"])
```

### 状态流转路径

```
初始状态
    ↓
StateGraph.channels（存储）
    ↓
PregelNode.channels（声明输入）
    ↓
ChannelRead.do_read()（读取）
    ↓
节点函数执行（处理）
    ↓
ChannelWrite.do_write()（写入）
    ↓
StateGraph.channels（更新）
    ↓
下一个节点...
```

### 关键区别

| 维度 | State（状态） | Context（上下文） |
|------|--------------|------------------|
| **可变性** | 可变（Mutable） | 不可变（Immutable） |
| **传递方式** | Channel 机制 | RunnableConfig |
| **更新方式** | 节点返回部分更新 | 运行时传入，不可修改 |
| **典型用途** | 消息历史、中间结果、工作流状态 | 用户 ID、模型配置、数据库连接 |
| **生命周期** | 随执行不断变化 | 整个执行期间保持不变 |
| **类型定义** | TypedDict | dataclass 或 TypedDict |

## 为什么需要这个机制？

### 问题场景

在构建复杂的 AI 工作流时，我们面临以下挑战：

1. **多步骤协作**：多个节点需要共享数据
2. **状态管理**：需要追踪工作流的执行状态
3. **配置传递**：需要在不污染状态的情况下传递配置
4. **可恢复性**：需要支持断点续传和错误恢复
5. **类型安全**：需要在编译时检查状态结构

### 解决方案

LangGraph 的状态传递与上下文机制提供了：

1. **清晰的数据流**：通过 Channel 机制明确状态的读写
2. **配置隔离**：通过 Context 分离可变和不可变数据
3. **灵活的更新策略**：通过 Reducer 函数控制状态合并逻辑
4. **持久化支持**：通过 Checkpoint 机制支持状态快照
5. **类型安全**：通过 TypedDict 和类型注解提供编译时检查

class MultiAgentState(TypedDict):
    task: str
    research_data: dict
    draft_content: str
    review_feedback: str
    final_output: str

@dataclass
class AgentContext:
    user_id: str
    project_id: str
    llm_config: dict

# 研究 Agent
def research_agent(state: MultiAgentState, runtime: Runtime[AgentContext]):
    task = state["task"]
    # 使用 context 中的配置
    llm = init_llm(runtime.context.llm_config)
    data = llm.research(task)
    return {"research_data": data}

# 写作 Agent
def writer_agent(state: MultiAgentState, runtime: Runtime[AgentContext]):
    data = state["research_data"]
    draft = generate_draft(data)
    return {"draft_content": draft}

# 审核 Agent
def reviewer_agent(state: MultiAgentState, runtime: Runtime[AgentContext]):
    draft = state["draft_content"]

    return {"review_feedback": feedback}

### 场景 2：RAG 系统

```python
class RAGState(TypedDict):
    query: str
    retrieved_docs: list
    reranked_docs: list
    generated_answer: str

@dataclass
class RAGContext:
    user_id: str
    collection_name: str
    top_k: int

def retrieval_node(state: RAGState, runtime: Runtime[RAGContext]):
    query = state["query"]
    collection = runtime.context.collection_name
    top_k = runtime.context.top_k

    docs = vector_search(query, collection, top_k)
    return {"retrieved_docs": docs}

def rerank_node(state: RAGState):
    docs = state["retrieved_docs"]
    reranked = rerank_documents(docs)
    return {"reranked_docs": reranked}

def generation_node(state: RAGState, runtime: Runtime[RAGContext]):
    docs = state["reranked_docs"]
    query = state["query"]

    answer = generate_answer(query, docs)
    return {"generated_answer": answer}
```
