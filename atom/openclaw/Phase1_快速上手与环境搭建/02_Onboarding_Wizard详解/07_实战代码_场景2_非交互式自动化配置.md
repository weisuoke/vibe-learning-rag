# 实战代码 场景2：非交互式自动化配置

**本文档演示非交互式模式的完整使用，包括环境变量配置、CI/CD集成和自动化部署脚本。**

---

## 场景概述

**目标**：使用非交互式模式自动化配置 OpenClaw，适用于 CI/CD、批量部署和脚本化安装。

**适用场景**：
- CI/CD 流水线自动部署
- 批量服务器配置
- Docker 容器初始化
- 自动化测试环境搭建

**核心优势**：
- 无需人工交互
- 可重复执行
- 易于版本控制
- 支持批量部署

## 非交互式模式基础

### 基本语法

```bash
openclaw onboard --non-interactive [选项]
```

**关键参数**：

| 参数 | 说明 | 必需 |
|------|------|------|
| `--non-interactive` | 启用非交互式模式 | 是 |
| `--auth-choice` | 认证方式选择 | 是 |
| `--anthropic-api-key` | Anthropic API 密钥 | 条件 |
| `--anthropic-model` | Anthropic 模型 | 否 |
| `--workspace` | Workspace 路径 | 否 |
| `--skip-health` | 跳过健康检查 | 否 |
| `--no-install-daemon` | 不安装守护进程 | 否 |

### 认证方式选项

```bash
# Anthropic
--auth-choice anthropic-api-key

# OpenAI
--auth-choice openai-api-key

# 自定义 Provider
--auth-choice custom-api-key

# Z.AI
--auth-choice zai-api-key
--auth-choice zai-coding-global
--auth-choice zai-coding-cn
```

## 环境变量配置

### 方式 1：使用环境变量

```bash
# 设置 API 密钥
export ANTHROPIC_API_KEY="sk-ant-api03-..."

# 运行非交互式配置
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key
```

**说明**：
- 如果设置了环境变量，`--anthropic-api-key` 参数可省略
- 向导会自动读取环境变量

### 方式 2：使用 .env 文件

```bash
# 创建 .env 文件
cat > ~/.openclaw/.env <<EOF
ANTHROPIC_API_KEY=sk-ant-api03-...
OPENAI_API_KEY=sk-...
CUSTOM_API_KEY=...
EOF

# 加载环境变量
source ~/.openclaw/.env

# 运行配置
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key
```

### 方式 3：直接传递参数

```bash
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "sk-ant-api03-..." \
  --anthropic-model "claude-sonnet-4-5"
```

## 完整配置示例

### 示例 1：Anthropic 基础配置

```bash
#!/bin/bash
# setup-openclaw-anthropic.sh

set -e

# 配置参数
ANTHROPIC_API_KEY="${ANTHROPIC_API_KEY:-sk-ant-api03-...}"
ANTHROPIC_MODEL="${ANTHROPIC_MODEL:-claude-sonnet-4-5}"
WORKSPACE="${WORKSPACE:-$HOME/.openclaw/workspace}"

# 运行非交互式配置
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --anthropic-model "$ANTHROPIC_MODEL" \
  --workspace "$WORKSPACE"

# 验证安装
openclaw status
openclaw health

echo "OpenClaw 配置完成！"
```

**使用方法**：

```bash
# 设置环境变量
export ANTHROPIC_API_KEY="sk-ant-..."

# 运行脚本
chmod +x setup-openclaw-anthropic.sh
./setup-openclaw-anthropic.sh
```

```bash
#!/bin/bash
# setup-openclaw-openai.sh

set -e

openclaw onboard --non-interactive \
  --auth-choice openai-api-key \
  --openai-api-key "$OPENAI_API_KEY" \

```

### 示例 3：自定义 Provider 配置

```bash
#!/bin/bash
# setup-openclaw-custom.sh

set -e

# 自定义 Provider 配置
CUSTOM_BASE_URL="${CUSTOM_BASE_URL:-https://llm.example.com/v1}"
CUSTOM_MODEL_ID="${CUSTOM_MODEL_ID:-foo-large}"
CUSTOM_API_KEY="${CUSTOM_API_KEY:-custom-key-...}"
CUSTOM_COMPATIBILITY="${CUSTOM_COMPATIBILITY:-openai}"

openclaw onboard --non-interactive \
  --auth-choice custom-api-key \
  --custom-base-url "$CUSTOM_BASE_URL" \
  --custom-model-id "$CUSTOM_MODEL_ID" \
  --custom-api-key "$CUSTOM_API_KEY" \
  --custom-compatibility "$CUSTOM_COMPATIBILITY"

openclaw status
```

**说明**：
- `--custom-compatibility`：兼容性模式（`openai` 或 `anthropic`）
- `--custom-base-url`：自定义 API 端点
- `--custom-model-id`：自定义模型 ID

## CI/CD 集成

### GitHub Actions 示例

```yaml
# .github/workflows/deploy-openclaw.yml
name: Deploy OpenClaw

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Install OpenClaw CLI
        run: |
          curl -fsSL https://openclaw.ai/install.sh | bash
          echo "$HOME/.openclaw/bin" >> $GITHUB_PATH

      - name: Configure OpenClaw
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          openclaw onboard --non-interactive \
            --auth-choice anthropic-api-key \
            --anthropic-api-key "$ANTHROPIC_API_KEY" \
            --workspace "$HOME/.openclaw/workspace" \
            --skip-health \
            --no-install-daemon

      - name: Verify installation
        run: |
          openclaw status
          openclaw config get agents.defaults.model.primary

      - name: Run tests
        run: |
          # 运行测试脚本
          ./scripts/test-openclaw.sh
```

```yaml
  only:
    - main
```

### Docker 容器初始化

```dockerfile
# Dockerfile
FROM ubuntu:22.04

# 安装依赖
RUN apt-get update && apt-get install -y \
    curl \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# 安装 OpenClaw CLI
RUN curl -fsSL https://openclaw.ai/install.sh | bash

# 设置环境变量
ENV PATH="/root/.openclaw/bin:$PATH"

# 复制配置脚本
COPY setup-openclaw.sh /usr/local/bin/
RUN chmod +x /usr/local/bin/setup-openclaw.sh

# 启动脚本
CMD ["/usr/local/bin/setup-openclaw.sh"]
```

**setup-openclaw.sh**：

```bash
#!/bin/bash
set -e

# 检查必需的环境变量
if [ -z "$ANTHROPIC_API_KEY" ]; then
  echo "Error: ANTHROPIC_API_KEY is not set"
  exit 1
fi

# 配置 OpenClaw
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --workspace "/root/.openclaw/workspace" \
  --skip-health \
  --no-install-daemon

# 启动 Gateway
openclaw gateway run --foreground
```

**使用方法**：

```bash
# 构建镜像
docker build -t openclaw:latest .

# 运行容器
docker run -d \
  -e ANTHROPIC_API_KEY="sk-ant-..." \
  -p 18789:18789 \
  openclaw:latest
```

## 高级自动化脚本

### 多环境配置脚本

```bash
#!/bin/bash
# setup-openclaw-multi-env.sh

set -e

# 环境选择
ENVIRONMENT="${1:-development}"

case "$ENVIRONMENT" in
  development)
    ANTHROPIC_API_KEY="$DEV_ANTHROPIC_API_KEY"
    WORKSPACE="$HOME/.openclaw/workspace-dev"
    PORT=18789
    ;;
  staging)
    ANTHROPIC_API_KEY="$STAGING_ANTHROPIC_API_KEY"
    WORKSPACE="$HOME/.openclaw/workspace-staging"
    PORT=18790
    ;;
  production)
    ANTHROPIC_API_KEY="$PROD_ANTHROPIC_API_KEY"
    WORKSPACE="$HOME/.openclaw/workspace-prod"
    PORT=18791
    ;;
  *)
    echo "Unknown environment: $ENVIRONMENT"
    exit 1
    ;;
esac

echo "Configuring OpenClaw for $ENVIRONMENT environment..."

# 配置 OpenClaw
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --workspace "$WORKSPACE"

# 修改端口
openclaw config set gateway.port "$PORT"

# 重启 Gateway
openclaw gateway restart

echo "OpenClaw configured for $ENVIRONMENT on port $PORT"
```

**使用方法**：

```bash
# 配置开发环境
./setup-openclaw-multi-env.sh development

# 配置生产环境
./setup-openclaw-multi-env.sh production
```

### 批量服务器部署脚本

```bash
#!/bin/bash
# deploy-openclaw-batch.sh

set -e

# 服务器列表
SERVERS=(
  "server1.example.com"
  "server2.example.com"
  "server3.example.com"
)

# API 密钥
ANTHROPIC_API_KEY="${ANTHROPIC_API_KEY}"

# 部署函数
deploy_to_server() {
  local server=$1
  echo "Deploying to $server..."

  ssh "$server" bash <<EOF
    set -e

    # 安装 OpenClaw CLI
    curl -fsSL https://openclaw.ai/install.sh | bash
    export PATH="\$HOME/.openclaw/bin:\$PATH"

    # 配置 OpenClaw
    openclaw onboard --non-interactive \
      --auth-choice anthropic-api-key \
      --anthropic-api-key "$ANTHROPIC_API_KEY" \
      --workspace "\$HOME/.openclaw/workspace"

    # 验证
    openclaw status
EOF

  echo "Deployed to $server successfully!"
}

# 批量部署
for server in "${SERVERS[@]}"; do
  deploy_to_server "$server" &
done

# 等待所有部署完成
wait

echo "All servers deployed successfully!"
```

## 故障排查

### 问题 1：环境变量未识别

**症状**：

```
Error: ANTHROPIC_API_KEY is required but not provided
```

**解决方案**：

```bash
# 1. 检查环境变量
echo $ANTHROPIC_API_KEY

# 2. 如果未设置，显式传递
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "sk-ant-..."

# 3. 或使用 .env 文件
source ~/.openclaw/.env
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key
```

### 问题 2：非交互式模式失败

**症状**：

```
Error: Cannot prompt in non-interactive mode
```

**原因**：缺少必需参数

**解决方案**：

```bash
# 检查缺少的参数
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "sk-ant-..." \
  --workspace "$HOME/.openclaw/workspace"  # 添加缺少的参数
```

### 问题 3：CI/CD 中 Gateway 启动失败

**症状**：

```
Error: Gateway did not become ready
```

**解决方案**：

```bash
# 跳过健康检查（CI/CD 环境）
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --skip-health \
  --no-install-daemon
```

### 问题 4：Docker 容器中权限问题

**症状**：

```
Error: Permission denied: /root/.openclaw/workspace
```

**解决方案**：

```dockerfile
# Dockerfile 中添加
RUN mkdir -p /root/.openclaw/workspace && \
    chmod -R 755 /root/.openclaw
```

```bash
# verify-openclaw.sh

  --anthropic-api-key "$ANTHROPIC_API_KEY"
```

## 完整部署示例

### 生产环境部署脚本

```bash
#!/bin/bash
# production-deploy.sh

set -e

# 配置参数
ENVIRONMENT="production"
ANTHROPIC_API_KEY="${ANTHROPIC_API_KEY}"
WORKSPACE="/opt/openclaw/workspace"
LOG_FILE="/var/log/openclaw-deploy.log"

# 日志函数
log() {
  echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"
}

# 错误处理
error_exit() {
  log "ERROR: $1"
  exit 1
}

# 检查必需的环境变量
if [ -z "$ANTHROPIC_API_KEY" ]; then
  error_exit "ANTHROPIC_API_KEY is not set"
fi

log "Starting OpenClaw deployment for $ENVIRONMENT..."

# 1. 安装 OpenClaw CLI
log "Installing OpenClaw CLI..."
curl -fsSL https://openclaw.ai/install.sh | bash || error_exit "Failed to install OpenClaw CLI"
export PATH="$HOME/.openclaw/bin:$PATH"

# 2. 配置 OpenClaw
log "Configuring OpenClaw..."
openclaw onboard --non-interactive \
  --auth-choice anthropic-api-key \
  --anthropic-api-key "$ANTHROPIC_API_KEY" \
  --workspace "$WORKSPACE" || error_exit "Failed to configure OpenClaw"

# 3. 验证配置
log "Verifying configuration..."
openclaw status || error_exit "Gateway is not running"
openclaw health || error_exit "Health check failed"

# 4. 配置生产环境参数
log "Configuring production parameters..."
openclaw config set gateway.bind "lan"
openclaw config set gateway.auth.mode "password"

# 5. 重启 Gateway
log "Restarting Gateway..."
openclaw gateway restart || error_exit "Failed to restart Gateway"

# 6. 最终验证
log "Final verification..."
sleep 5
openclaw status || error_exit "Final status check failed"

log "OpenClaw deployment completed successfully!"
```

## 总结

非交互式自动化配置的核心要点：

1. **环境变量优先**：使用环境变量管理敏感信息
2. **参数完整性**：确保所有必需参数都提供
3. **错误处理**：使用 `set -e` 和错误检查
4. **验证配置**：配置后立即验证
5. **日志记录**：记录所有操作日志
6. **幂等性**：脚本可重复执行

完成这些步骤后，你就可以在 CI/CD 流水线中自动化部署 OpenClaw 了！🎉
