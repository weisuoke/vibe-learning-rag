# 核心概念 4：配置文件系统

> **一句话定义：** 配置文件系统是 ZeroClaw 的"持久化控制面板"——通过 TOML 格式的 `config.toml` 文件，将 Gateway、安全策略、Agent 编排、内存后端、通道配置等十余个功能域的设置集中管理，并与环境变量协同组成"环境变量 → 活跃工作区 → 默认配置文件 → 代码默认值"的四层解析优先级链。

---

## 1. TOML 格式速成（给前端开发者）

**TOML**（Tom's Obvious, Minimal Language）是 Rust 生态的"标配"配置格式（就像 JSON 之于 JavaScript）。

```toml
# 这是注释（JSON 不支持注释，这是选 TOML 的原因之一）
name = "zeroclaw"         # 字符串必须双引号
port = 42617              # 整数不加引号
temperature = 0.7         # 浮点数
enabled = true            # 布尔值（小写）
items = ["a", "b", "c"]   # 数组

[gateway]                 # 节/表 = JSON 的嵌套对象 { "gateway": {} }
port = 42617
host = "[::]"
```

### TOML vs JSON 对照（一眼看懂）

| 概念 | TOML | JSON | 注意 |
|------|------|------|------|
| 字符串 | `key = "value"` | `"key": "value"` | TOML 必须双引号 |
| 对象/节 | `[section]` | `"section": {}` | 方括号声明节名 |
| 嵌套 | `[a.b]` | `"a":{"b":{}}` | 点号表示层级 |
| 注释 | `# 注释` | 不支持 | TOML 的一大优势 |

> **前端类比：** TOML 就是"能写注释的 JSON + 用 `[section]` 代替大括号嵌套"。`Cargo.toml` 就是 TOML——你在 Phase 1 已经见过了。

> **日常生活类比：** JSON 像填表格（引号逗号一个不对就报错），TOML 像写备忘录（简洁直观还能加注释）。

## 2. config.toml 完整结构

ZeroClaw 的 `config.toml` 分为**顶层字段**和**10+ 个功能节**。[来源: sourcecode/zeroclaw/src/config/mod.rs]

```toml
# ===== 顶层字段（全局设置）=====
workspace_dir = "/path/to/workspace"
config_path = "/path/to/config.toml"       # 自动写入，通常不手动设
api_key = "sk-..."                          # ⚠️ 建议走环境变量
api_url = "http://custom-endpoint"          # 代理/私有部署时用
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
default_temperature = 0.7

[gateway]          # HTTP 网关：端口、绑定地址
[autonomy]         # Agent 自主操作安全等级
[security]         # 文件/网络访问边界
[runtime]          # native vs Docker 运行时
[reliability]      # 重试、降级、回退模型
[scheduler]        # 定时任务（Cron）
[agent]            # 多 Agent 协作编排
[skills]           # Skills 搜索路径、社区仓库
[memory]           # 对话记忆存储（sqlite/postgres）
[channels_config]  # Telegram/Discord/Slack 等通道
[observability]    # 日志级别、指标导出
```

### 2.1 顶层字段（最常用 ⭐）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `workspace_dir` | String | `~/.zeroclaw` | 工作区根目录 |
| `api_key` | String | 无 | API Key（**建议走环境变量**） |
| `api_url` | String | Provider 默认 | 自定义 API 端点 |
| `default_provider` | String | `"openrouter"` | 默认 LLM Provider |
| `default_model` | String | 无 | 默认模型 ID |
| `default_temperature` | Float | `0.7` | 温度（0.0-2.0） |

```toml
# 最小可用 config.toml（只需 3 个字段，api_key 走环境变量）
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
default_temperature = 0.7
```

### 2.2 各节速查

| 节名 | 功能 | 前端类比 | 日常生活类比 |
|------|------|---------|------------|
| `[gateway]` | 端口、绑定地址、公网开关 | `devServer` 配置 | 门牌号和门锁 |
| `[autonomy]` | Agent 自主执行权限等级 | — | 给孩子的零花钱额度 |
| `[security]` | 文件/网络访问白名单 | CSP 策略 / `.eslintrc` | 小区门禁规则 |
| `[reliability]` | 重试次数、降级回退 | `axios-retry` | 快递重发策略 |
| `[memory]` | 对话持久化（sqlite/postgres） | localStorage vs DB | 笔记本 |
| `[channels_config]` | 消息通道接入 | API routes / webhooks | 手机通讯 App |
| `[skills]` | 扩展技能加载 | `plugins` 数组 | 手机 App Store |
| `[scheduler]` | 定时任务 | cron jobs | 闹钟/日历提醒 |
| `[agent]` | 多 Agent 协作 | 微前端编排 | 团队分工 |
| `[observability]` | 日志与指标 | Sentry / DataDog | 仪表盘监控 |

```toml
# [gateway] 示例
[gateway]
port = 42617          # 默认端口（类似 PORT=3000 npm start）
host = "[::]"         # IPv6 全地址 = 监听所有网卡
allow_public_bind = true  # 生产环境开启，开发环境默认 false
```

## 3. 配置解析优先级

### 3.1 四层优先级链（查找配置文件位置）

```
最高                                                        最低
 │                                                           │
 ▼                                                           ▼
ZEROCLAW_CONFIG_DIR → ZEROCLAW_WORKSPACE → active_workspace.toml → ~/.zeroclaw/
```

[来源: sourcecode/zeroclaw/src/config/mod.rs]

```rust
// 简化后的配置目录解析逻辑
fn resolve_config_dir() -> PathBuf {
    // 第1优先：ZEROCLAW_CONFIG_DIR 环境变量
    if let Ok(dir) = env::var("ZEROCLAW_CONFIG_DIR") {
        return PathBuf::from(dir);
    }
    // 第2优先：ZEROCLAW_WORKSPACE 环境变量
    if let Ok(dir) = env::var("ZEROCLAW_WORKSPACE") {
        return PathBuf::from(dir);
    }
    // 第3优先：活跃工作区标记文件
    let default_dir = home_dir().join(".zeroclaw");
    let marker = default_dir.join("active_workspace.toml");
    if marker.exists() {
        if let Some(path) = read_active_workspace(&marker) {
            return path;
        }
    }
    // 第4优先：默认路径
    default_dir  // ~/.zeroclaw
}
```

### 3.2 配置值的优先级（找到文件后，字段值从哪来）

```
环境变量（最高） → 配置文件字段 → 代码默认值（最低）

API Key:   ZEROCLAW_API_KEY → API_KEY → config.toml.api_key → 无（报错）
Port:      ZEROCLAW_GATEWAY_PORT → PORT → config.toml.[gateway].port → 42617
Provider:  ZEROCLAW_PROVIDER → PROVIDER → config.toml.default_provider → "openrouter"
```

> **前端类比：** 这和 TypeScript 查找 `tsconfig.json` 的逻辑类似——先看 `TS_NODE_PROJECT` 环境变量，再沿目录树找。Next.js 端口优先级也是 `PORT` 环境变量 → 配置文件 → 默认 `3000`。

### 3.3 排查"配置不生效"

```bash
# 第1步：确认读取的是哪个配置文件
echo $ZEROCLAW_CONFIG_DIR     # 如果有值，优先读这个目录
echo $ZEROCLAW_WORKSPACE      # 其次读这个目录
cat ~/.zeroclaw/active_workspace.toml  # 检查活跃工作区指向

# 第2步：确认环境变量没有覆盖配置文件
env | grep ZEROCLAW           # 环境变量优先级 > 配置文件

# 第3步：检查 TOML 语法
#   ❌ key = value        （字符串没加引号）
#   ❌ port = "42617"     （端口应该是数字不是字符串）
#   ✅ key = "value"      ✅ port = 42617
```

| 不生效原因 Top 3 | 解决方案 |
|------------------|---------|
| 环境变量覆盖了配置文件 | `unset ZEROCLAW_API_KEY` |
| 修改了错误路径的 config.toml | 按上述步骤确认实际路径 |
| TOML 语法错误 | 检查引号和类型匹配 |

## 4. 配置文件的生成和管理

### 4.1 zeroclaw onboard（推荐）

```bash
# 交互式引导
zeroclaw onboard

# 非交互式（脚本自动化）
zeroclaw onboard --api-key "sk-or-v1-your-key" --provider openrouter

# 安装时一并完成
./zeroclaw_install.sh --onboard --api-key "sk-or-v1-your-key" --provider openrouter
```

onboard 会：创建 `~/.zeroclaw/` → 询问 Provider/Key/Model → 生成 `config.toml` → 验证 API 可用。

> **前端类比：** `zeroclaw onboard` = `npx create-next-app`——交互式生成项目配置。

### 4.2 手动编辑（完整模板）

```toml
# ~/.zeroclaw/config.toml - 完整模板（复制后按需修改）
# 未注释 = 常用，注释掉 = 按需启用

default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
default_temperature = 0.7
# api_key = "sk-..."              # ⚠️ 建议走环境变量
# api_url = "http://custom-endpoint"

[gateway]
port = 42617
host = "[::]"
# allow_public_bind = false       # 生产环境改为 true

# [reliability]
# retry_max = 3

# [memory]
# provider = "sqlite"

# [skills]
# open_skills_enabled = false

# [observability]
# log_level = "info"
```

### 4.3 版本控制

```bash
git add config.toml.example      # ✅ 提交模板（无敏感信息）
echo "config.toml" >> .gitignore  # ❌ 不提交实际配置
# 新成员：cp config.toml.example config.toml → 填入自己的配置
```

## 5. 多工作区管理

### 5.1 场景：多个 ZeroClaw 项目需要独立配置

```
~/.zeroclaw/                        # 默认工作区
├── config.toml
├── active_workspace.toml           # "指针文件"，指向活跃工作区
└── data/

~/projects/customer-bot/.zeroclaw/  # 客服项目（用 OpenRouter）
└── config.toml

~/projects/code-helper/.zeroclaw/   # 代码助手（用 Ollama 本地）
└── config.toml
```

### 5.2 active_workspace.toml

```toml
# ~/.zeroclaw/active_workspace.toml — 标记当前活跃工作区
workspace_dir = "/home/user/projects/customer-bot/.zeroclaw"
```

### 5.3 切换方式

```bash
# 临时切换（当前终端）
export ZEROCLAW_WORKSPACE="/home/user/projects/code-helper/.zeroclaw"

# 单次命令
ZEROCLAW_WORKSPACE="/home/user/projects/customer-bot/.zeroclaw" \
  zeroclaw agent -m "Answer customer query"

# 持久切换（修改 active_workspace.toml）
```

> **前端类比：** 像 `nvm use 18` / `nvm use 20` 切换 Node 版本，`active_workspace.toml` 就是 `.nvmrc`。

> **日常生活类比：** 手机切换工作/生活模式——不同模式不同设置。

## 6. 配置最佳实践

### 环境变量 vs 配置文件分工

```
环境变量：秘密 + 易变                配置文件：稳定 + 复杂
├── API Key（敏感）               ├── 复杂嵌套结构（channels/security）
├── 跨环境变化的值                ├── 不常变化的默认值
├── CI/CD 注入                   ├── 需要注释说明的配置
└── 临时覆盖测试                  └── 团队共享的模板
```

### 分层管理

```bash
# 层1：config.toml → 稳定基线（可提交 Git 模板）
# 层2：.env / .bashrc → 个人/环境专属值（不提交）
# 层3：命令行前缀 → 单次临时覆盖
ZEROCLAW_TEMPERATURE=0 zeroclaw agent -m "Be precise"
```

## 7. 在 ZeroClaw 开发中的应用

| 阶段 | config.toml | 环境变量 |
|------|------------|---------|
| **首次体验** | `zeroclaw onboard` 自动生成 | `ZEROCLAW_API_KEY` |
| **日常开发** | 固定 Provider + 模型 | API Key 走 `.bashrc` |
| **多项目** | 每个项目独立 config.toml | `ZEROCLAW_WORKSPACE` 切换 |
| **Docker** | 最小 config 或全走环境变量 | `--env-file` 注入 |
| **CI/CD** | 无 config.toml | GitHub Secrets |

```
用户输入 → 配置系统 [环境变量 > config.toml > 默认值] → 各子系统
                                                      ├─ Gateway    ← [gateway]
                                                      ├─ Agent      ← [agent] + [autonomy]
                                                      ├─ Provider   ← default_provider
                                                      ├─ Memory     ← [memory]
                                                      └─ Channels   ← [channels_config]
```

> **核心理解：** config.toml 是 ZeroClaw 所有子系统的"配置中心"。理解它的结构，就理解了 ZeroClaw 的功能全景。
