## 知识点概述

### 什么是 Milvus Embedding Functions？

**Milvus 2.6 Embedding Functions 是内置的文本向量化功能，支持10个主流embedding提供商，让RAG开发者无需手动管理embedding API调用。**

### 核心价值

1. **简化开发流程**
   - 无需手动调用embedding API
   - 自动处理批量数据
   - 统一的接口设计

2. **生产级特性**
   - 并行执行支持
   - 自动批处理
   - 错误处理机制
   - 凭证管理

3. **多提供商支持**
   - 10个主流提供商
   - 灵活切换
   - 统一配置

## 架构设计

### 核心组件

```
┌─────────────────────────────────────────────────────────┐
│                   FunctionExecutor                       │
│  (并行执行多个embedding函数)                              │
└────────────────┬────────────────────────────────────────┘
                 │
                 ├─── TextEmbeddingFunction 1
                 │    └─── Provider (OpenAI/Cohere/...)
                 │
                 ├─── TextEmbeddingFunction 2
                 │    └─── Provider (VoyageAI/...)
                 │
                 └─── TextEmbeddingFunction N
                      └─── Provider (...)
```

### Provider Pattern设计

所有提供商实现统一接口：

```go
type textEmbeddingProvider interface {
    MaxBatch() int                                          // 最大批量大小
    CallEmbedding(ctx, texts, mode) (any, error)          // 调用embedding API
    FieldDim() int64                                       // 输出维度
}
```

### 执行模式

1. **InsertMode（插入模式）**
   - 用于数据插入操作
   - 某些提供商使用"document"类型
   - 批量处理优化

2. **SearchMode（搜索模式）**
   - 用于查询操作
   - 某些提供商使用"query"类型
   - 单条或小批量处理

## 关键特性

### 1. 自动批处理

```python
# Milvus自动将大量数据分批处理
collection.insert([
    {"text": "doc1"},
    {"text": "doc2"},
    # ... 10000条数据
])
# Milvus自动按MaxBatch分批调用embedding API
```

### 2. 并行执行

```python
# 多个embedding函数并行执行
executor.ProcessInsert(data)
# 使用goroutines并行调用多个embedding函数
```

### 3. 凭证管理优先级

```
函数参数 > YAML配置 > 环境变量
```

### 4. 维度控制

支持维度控制的提供商：
- OpenAI (text-embedding-3系列)
- VoyageAI (voyage-3系列)
- Cohere (embed-v3系列)
- VertexAI (text-embedding-004)
- DashScope (text-embedding-v3)

### 5. 量化支持

支持Int8输出的提供商：
- VoyageAI
- Cohere

## 常见误区

### 误区1：批量大小越大越好 ❌

**为什么错？**
- 批量大小受API限制
- 超过MaxBatch会报错
- 不同提供商限制不同

**正确理解**：
- 批量大小应≤MaxBatch
- Milvus自动分批处理
- 选择批量大的提供商可提升吞吐量

### 误区2：所有提供商都支持批处理 ❌

**为什么错？**
- Bedrock不支持批处理（MaxBatch=1）
- 每条数据需单独API调用
- 性能显著低于其他提供商

**正确理解**：
- 查看MaxBatch值
- Bedrock适合低频场景
- 高频场景选择其他提供商

### 误区3：Embedding Functions会增加显著延迟 ❌

**为什么错？**
- Milvus引入的额外开销<10ms
- 主要延迟来自embedding API本身
- 并行执行可提升整体性能

**正确理解**：
- 延迟主要取决于提供商
- 地理位置影响大于模型结构
- 选择就近的提供商

## 生产实践建议

### 1. 提供商选择

```python
# 开发环境：使用OpenAI（稳定可靠）
dev_config = {
    "provider": "openai",
    "model_name": "text-embedding-3-small"
}

# 生产环境：根据场景选择
prod_config = {
    "provider": "voyageai",  # 大规模RAG
    "model_name": "voyage-3-large"
}
```

### 2. 错误处理

```python
try:
    collection.insert(data)
except MilvusException as e:
    if "empty string" in str(e):
        # 过滤空字符串
        data = [d for d in data if d["text"].strip()]
        collection.insert(data)
    elif "batch size" in str(e):
        # 减小批量大小
        for batch in chunks(data, max_batch // 2):
            collection.insert(batch)
```

### 3. 性能优化

```python
# 1. 使用批量大的提供商
# OpenAI/VoyageAI/VertexAI: 128条/批

# 2. 并行处理多个collection
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=4) as executor:
    futures = [
        executor.submit(coll.insert, batch)
        for coll, batch in zip(collections, batches)
    ]
```

### 4. 成本控制

```python
# 1. 使用较小的模型
config = {
    "provider": "openai",
    "model_name": "text-embedding-3-small",  # 而非3-large
    "dim": 512  # 降低维度
}

# 2. 缓存embedding结果
embedding_cache = {}

def get_embedding_with_cache(text):
    if text in embedding_cache:
        return embedding_cache[text]
    result = collection.insert([{"text": text}])
    embedding_cache[text] = result
    return result
```

## 快速开始

### 最小可用示例

```python
from pymilvus import MilvusClient, Function, DataType, FunctionType

# 1. 创建客户端
client = MilvusClient("http://localhost:19530")

# 2. 定义embedding函数
openai_ef = Function(
    name="openai_ef",
    function_type=FunctionType.TEXTEMBEDDING,
    input_field_names=["text"],
    output_field_names=["vector"],
    params={
        "provider": "openai",
        "model_name": "text-embedding-3-small",
        "api_key": "sk-xxx",
        "dim": 512
    }
)

# 3. 创建schema
schema = client.create_schema()
schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
schema.add_field("text", DataType.VARCHAR, max_length=1000)
schema.add_field("vector", DataType.FLOAT_VECTOR, dim=512)
schema.add_function(openai_ef)

# 4. 创建collection
client.create_collection("my_rag", schema=schema)

# 5. 插入数据（自动embedding）
client.insert("my_rag", [
    {"text": "Milvus is a vector database"},
    {"text": "RAG combines retrieval and generation"}
])

# 6. 搜索（自动embedding查询文本）
results = client.search(
    "my_rag",
    data=["What is Milvus?"],
    anns_field="vector",
    limit=5
)
```
