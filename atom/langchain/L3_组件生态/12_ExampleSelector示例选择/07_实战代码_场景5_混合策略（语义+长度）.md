## 场景概述

在实际生产环境中，单一的示例选择策略往往无法满足需求：
- **只用语义相似度**：可能选择过多示例，超过模型的 context window
- **只用长度限制**：可能选择不相关的示例，影响效果

**混合策略**结合两者优势：先用语义相似度选择最相关的示例，再用长度限制控制总量，确保既相关又不超过 token 限制。

### 为什么需要混合策略？

**问题场景：**
```python
# 场景1：只用语义相似度
semantic_selector = SemanticSimilarityExampleSelector(k=10)
# 问题：10个示例可能超过4000 tokens，超出模型限制

# 场景2：只用长度限制
length_selector = LengthBasedExampleSelector(max_length=2000)
# 问题：可能选择不相关的示例，影响效果

# 场景3：混合策略（最佳实践）
hybrid_selector = SemanticLengthExampleSelector(
    k=10,  # 先选择10个最相关的
    max_length=2000  # 再控制总长度不超过2000
)
# 优势：既保证相关性，又控制长度
```

## 核心原理

### 混合策略的工作流程

```
用户查询
    ↓
1. 语义相似度选择
   - 计算查询与所有示例的相似度
   - 选择 top-k 个最相关的示例
    ↓
2. 长度过滤
   - 计算已选示例的总长度
   - 如果超过 max_length，移除最后的示例
   - 重复直到总长度 <= max_length
    ↓
3. 返回最终示例
   - 既相关又不超过长度限制
```

```python
class SemanticLengthExampleSelector(BaseExampleSelector):
    """结合语义相似度和长度限制的混合选择器"""

    def __init__(
        self,
        vectorstore: VectorStore,
        k: int = 5,
        max_length: int = 2000,
        get_text_length: Callable[[str], int] = len,
    ):
        self.vectorstore = vectorstore
        self.k = k
        self.max_length = max_length
        self.get_text_length = get_text_length

    def select_examples(self, input_variables: dict) -> list[dict]:
        # 1. 语义相似度选择
        candidates = self.vectorstore.similarity_search(
            input_variables["input"],
            k=self.k
        )

        # 2. 长度过滤
        selected = []
        current_length = 0

        for doc in candidates:
            example_length = self.get_text_length(doc.page_content)
            if current_length + example_length <= self.max_length:
                selected.append(doc.metadata)
                current_length += example_length
            else:
                break

        return selected
```

```python
class SemanticLengthExampleSelector(BaseExampleSelector):
    """结合语义相似度和长度限制的混合选择器

    工作流程：
    1. 使用语义相似度选择 top-k 个最相关的示例
    2. 按顺序添加示例，直到达到长度限制
    3. 返回既相关又不超过长度限制的示例
    """

    def __init__(
        self,
        vectorstore: VectorStore,
        k: int = 5,
        max_length: int = 2000,
        get_text_length: Callable[[str], int] = None,
        example_prompt: PromptTemplate = None,
    ):
        """初始化混合选择器

        Args:
            vectorstore: 向量存储
            k: 语义相似度选择的候选数量
            max_length: 最大长度限制（字符数）
            get_text_length: 自定义长度计算函数
            example_prompt: 示例格式化模板（用于计算长度）
        """
        self.vectorstore = vectorstore
        self.k = k
        self.max_length = max_length
        self.get_text_length = get_text_length or self._default_length
        self.example_prompt = example_prompt

    @staticmethod
    def _default_length(text: str) -> int:
        """默认长度计算：字符数"""
        return len(text)

    def add_example(self, example: Dict[str, str]) -> None:
        """添加示例到向量存储"""
        # 将示例转换为文本
        text = " ".join(example.values())
        # 添加到向量存储
        self.vectorstore.add_texts([text], metadatas=[example])

    def select_examples(self, input_variables: Dict[str, str]) -> List[Dict]:
        """选择示例

        Args:
            input_variables: 输入变量（包含用户查询）

        Returns:
            选择的示例列表
        """
        # 1. 语义相似度选择
        query = input_variables.get("input", "")
        candidate_docs = self.vectorstore.similarity_search(query, k=self.k)

        # 2. 长度过滤
        selected_examples = []
        current_length = 0

        for doc in candidate_docs:
            # 计算示例长度
            if self.example_prompt:
                # 使用模板格式化后计算长度
                formatted = self.example_prompt.format(**doc.metadata)
                example_length = self.get_text_length(formatted)
            else:
                # 直接计算原始文本长度
                example_length = self.get_text_length(doc.page_content)

            # 检查是否超过长度限制
            if current_length + example_length <= self.max_length:
                selected_examples.append(doc.metadata)
                current_length += example_length
            else:
                # 超过限制，停止添加
                break

        return selected_examples

    @classmethod
    def from_examples(
        cls,
        examples: List[Dict[str, str]],
        embeddings,
        vectorstore_cls,
        k: int = 5,
        max_length: int = 2000,
        get_text_length: Callable[[str], int] = None,
        example_prompt: PromptTemplate = None,
        **vectorstore_kwargs,
    ):
        """从示例列表创建选择器"""
        # 将示例转换为文本
        texts = [" ".join(ex.values()) for ex in examples]

        # 创建向量存储
        vectorstore = vectorstore_cls.from_texts(
            texts, embeddings, metadatas=examples, **vectorstore_kwargs
        )

        return cls(
            vectorstore=vectorstore,
            k=k,
            max_length=max_length,
            get_text_length=get_text_length,
            example_prompt=example_prompt,
        )

def count_tokens(text: str) -> int:
    """简单的 token 计数（实际应使用 tiktoken）"""
    # 简化版本：1个中文字符 ≈ 2 tokens，1个英文单词 ≈ 1.3 tokens
    chinese_chars = sum(1 for c in text if '\u4e00' <= c <= '\u9fff')
    other_chars = len(text) - chinese_chars
    return int(chinese_chars * 2 + other_chars * 0.3)
```

## 关键要点

### 1. 混合策略的核心优势

**平衡相关性和长度：**
- 先用语义相似度选择最相关的示例
- 再用长度限制控制总量
- 确保既相关又不超过 token 限制

### 2. 参数配置建议

**k 参数（候选数量）：**
```python
# k 太小：可能错过相关示例
hybrid_selector = SemanticLengthExampleSelector(k=3, max_length=500)

# k 太大：计算成本高，但选择空间大
hybrid_selector = SemanticLengthExampleSelector(k=20, max_length=500)

# 推荐配置
hybrid_selector = SemanticLengthExampleSelector(
    k=8,  # 先选择 8 个候选
    max_length=500  # 再控制总长度
)
```

**max_length 参数（最大长度）：**
```python
# 根据模型 context window 调整
# GPT-3.5: 4096 tokens → max_length=2000-3000 字符
# GPT-4: 8192 tokens → max_length=4000-6000 字符
# Claude: 100k tokens → max_length=8000-10000 字符
```

### 3. 长度计算方式

**字符数 vs Token 数：**
```python
# 方式1：字符数（简单但不精确）
def char_length(text: str) -> int:
    return len(text)

# 方式2：Token 数（精确但需要额外库）
import tiktoken

def token_length(text: str) -> int:
    encoding = tiktoken.encoding_for_model("gpt-3.5-turbo")
    return len(encoding.encode(text))

# 方式3：估算（折中方案）
def estimated_token_length(text: str) -> int:
    chinese_chars = sum(1 for c in text if '\u4e00' <= c <= '\u9fff')
    other_chars = len(text) - chinese_chars
    return int(chinese_chars * 2 + other_chars * 0.3)
```

### 4. 适用场景

**适合使用混合策略的场景：**
- 生产环境（需要控制成本）
- 示例长度差异大
- 需要严格控制 Prompt 长度
- 对 token 使用量敏感

**不适合使用混合策略的场景：**
- 示例库很小（< 5 个）
- 所有示例长度相近
- 不关心 token 成本
- 对性能要求极高（混合策略比纯语义相似度慢）
