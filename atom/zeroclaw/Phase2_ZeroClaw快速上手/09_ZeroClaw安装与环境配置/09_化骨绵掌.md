### 卡片1：ZeroClaw 是什么？

**一句话：** ZeroClaw 是一个 100% Rust 编写的轻量级 AI Agent 运行时，一个二进制文件解决所有问题。

**举例：**

```
┌─────────────────────────────────────────────────┐
│  ZeroClaw 核心数据                                │
│                                                   │
│  📦 二进制大小:  ~8.8 MB  (3 张高清照片)           │
│  🧠 运行内存:   <5 MB    (比一个空白浏览器标签小)   │
│  ⚡ 启动时间:   <10 ms   (人类眨眼 300ms)          │
│  🔧 运行时依赖: 零        (不需要 Node/Python/JVM) │
│  🖥️  支持平台:  6 个      (Linux/macOS/Windows     │
│                            × x86_64/ARM64/ARMv7)  │
└─────────────────────────────────────────────────┘
```

**前端对比：** 一个典型的 Node.js CLI 工具需要 Node.js 运行时（~100 MB）+ `node_modules`（200-500 MB），启动需要 500ms-2s。ZeroClaw 只有一个 8.8 MB 的文件，启动快 30 倍以上。

**应用：** 安装 ZeroClaw 就是获取这一个二进制文件并放到 PATH 中。理解"它就是一个可执行文件"，是理解所有安装方式的基础。

### 卡片2：为什么安装不是 npm install？

**一句话：** ZeroClaw 是编译型产物（机器码），不是解释型脚本，所以"安装"的含义完全不同。

**举例：**

```
解释型安装（你熟悉的）：              编译型安装（ZeroClaw）：
npm install -g typescript              方式 A: 下载预编译二进制（推荐）
  │                                       │
  ├── 下载 JS 源码到全局目录              ├── 下载已编译好的机器码
  ├── 需要 Node.js 运行时解释执行         ├── 直接运行，无需任何运行时
  └── 产物: 大量 .js 文件                 └── 产物: 一个可执行文件

                                       方式 B: cargo install（从源码编译）
                                          │
                                          ├── 拉取 Rust 源码
                                          ├── 调用 rustc 编译成机器码
                                          └── 产物: 一个可执行文件（慢但可定制）
```

**关键区别：**

| 维度 | `npm install -g` | ZeroClaw 推荐安装 |
|------|-------------------|-------------------|
| 产物 | 源码 + node_modules | 单一二进制文件 |
| 运行时 | 需要 Node.js | 不需要任何运行时 |
| 启动速度 | 500ms-2s | <10ms |

**应用：** 这就是为什么 ZeroClaw 提供 5 种安装方式而不只是一个 `npm install`——编译型工具的分发逻辑和解释型完全不同。推荐方式（一键脚本、Homebrew、预编译下载）都是直接下载编译好的文件，和 `cargo install` 的编译过程无关。

### 卡片3：5 种安装方式速查

**一句话：** 5 种方式对应 5 个场景，90% 的人用前两种就够了。

**举例：**

| # | 方式 | 一行命令 | 适用场景 | 耗时 |
|---|------|---------|---------|------|
| 1 | **一键脚本** ⭐ | `curl -fsSL https://zeroclawlabs.ai/install.sh \| bash` | 首次安装、快速体验 | ~30s |
| 2 | **Homebrew** | `brew install zeroclaw` | macOS/Linux 日常 | ~15s |
| 3 | **预编译二进制** | 从 GitHub Releases 下载解压 | 离线环境、CI/CD | ~10s |
| 4 | **Docker** | `docker compose up -d` | 服务器 24/7 部署 | ~30s |
| 5 | **Cargo Install** | `cargo install zeroclaw` | Rust 开发者、定制 Feature | 3-15min |

**决策树：**

```
你要做什么？
├── 第一次用 ZeroClaw ──────→ ① 一键脚本
├── macOS/Linux 日常 ───────→ ② Homebrew
├── 没有网络 / CI 环境 ─────→ ③ 预编译下载
├── 生产服务器部署 ──────────→ ④ Docker
└── 需要自定义 Feature ─────→ ⑤ Cargo Install
```

**应用：** 不要纠结"用哪种方式"——第一次就用一键脚本，30 秒搞定。后面有需要了再换。就像你第一次学 React 不需要纠结 Vite 还是 Webpack。

### 卡片4：一键安装背后发生了什么？

**一句话：** `bootstrap.sh` 是一个智能安装脚本，按"检测→预检→安装→验证"四步自动完成所有操作。

**举例：**

```
curl -fsSL https://zeroclawlabs.ai/install.sh | bash

                    ┌──────────────────────┐
Step 1: 检测        │ 检测 OS + CPU 架构    │
                    │ darwin-aarch64?       │
                    │ linux-x86_64?         │
                    └──────────┬───────────┘
                               ↓
                    ┌──────────────────────┐
Step 2: 预检        │ Rust 已安装？          │
                    │ RAM ≥ 2GB？           │
                    │ 磁盘 ≥ 6GB？          │
                    └──────────┬───────────┘
                               ↓
                    ┌──────────────────────┐
Step 3: 安装        │ 有预编译版本？         │
                    │ ├── 是 → 直接下载     │
                    │ └── 否 → 源码编译     │
                    └──────────┬───────────┘
                               ↓
                    ┌──────────────────────┐
Step 4: 验证        │ zeroclaw --version    │
                    │ 输出版本号 = 成功 ✅   │
                    └──────────────────────┘
```

**前端类比：** 就像 `npx create-react-app` 的背后也不是简单的 `npm install`——它会检测 npm 版本、选择模板、安装依赖、生成项目结构。一键安装脚本做的事情类似，只是最终产物是一个二进制文件而非一个项目目录。

**应用：** 了解这个流程有助于排错。如果安装失败，你知道是哪一步出了问题——是平台不支持（Step 1）、资源不足（Step 2）、还是网络问题（Step 3）。加 `--prefer-prebuilt` 可以跳过编译步骤。

### 卡片5：config.toml 速成

**一句话：** `~/.zeroclaw/config.toml` 只需要 3 行就能让 ZeroClaw 跑起来，TOML 比 JSON 简单。

**举例：**

```toml
# ~/.zeroclaw/config.toml — 最小可用配置（只需要这 3 行）
api_key = "sk-or-v1-xxxxxxxxxxxx"
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
```

**TOML vs JSON — 30 秒对比：**

```
TOML（ZeroClaw 用的）：         JSON（你熟悉的）：
─────────────────────          ─────────────────────
api_key = "sk-xxx"             {
port = 42617                     "api_key": "sk-xxx",
verbose = true                   "port": 42617,
                                 "verbose": true
                               }

区别：
✅ TOML: 不用大括号、不用逗号、支持注释（# 开头）
❌ JSON: 需要大括号、逗号、不支持注释
```

**前端类比：** `config.toml` 之于 ZeroClaw，就像 `next.config.js` 之于 Next.js——所有运行时行为的集中配置。TOML 格式之于 Rust 生态，就像 `package.json`（JSON）之于 Node.js 生态——几乎所有 Rust 项目都用 TOML。

**应用：** 你可以用 `zeroclaw onboard` 自动生成这个文件（推荐），也可以手动创建。只要有这 3 行，Agent 就能启动。后面需要更多配置时，在这个文件里加 `[gateway]`、`[security]` 等分区即可。

### 卡片6：环境变量的 3 条黄金规则

**一句话：** Key 走环境变量（安全）、Provider 走 config.toml（持久）、临时测试走命令行内联（灵活）。

**举例：**

```bash
# ━━━ 规则 1: 安全敏感信息 → 环境变量 ━━━
# 原因：不落盘、不提交 Git、进程退出即消失
export ZEROCLAW_API_KEY="sk-or-v1-abc123"

# ━━━ 规则 2: 持久化偏好 → config.toml ━━━
# 原因：设置一次永久生效，结构化可读，可团队共享模板
# ~/.zeroclaw/config.toml:
#   default_provider = "openrouter"
#   default_model = "anthropic/claude-sonnet-4-6"

# ━━━ 规则 3: 临时覆盖 → 命令行内联 ━━━
# 原因：只影响本次执行，不改变任何持久配置
ZEROCLAW_MODEL=openai/gpt-4o zeroclaw agent -m "用 GPT-4o 试试"
```

**前端类比：**

```
规则 1: GitHub Secrets / CI 环境变量   → 存放 API Key
规则 2: next.config.js / tsconfig.json → 存放项目配置
规则 3: npm start -- --port 4000       → 临时用不同端口
```

**应用：** 这 3 条规则解决了最常见的新手困惑——"API Key 应该放哪里？"。记住口诀：**密钥走 env，偏好走 toml，临时走 inline**。如果你把 API Key 写进了 config.toml 并提交到 Git，那就和在前端代码里硬编码 `REACT_APP_API_KEY` 一样危险。

### 卡片7：Feature Flags = 编译时菜单

**一句话：** Feature Flags 是 ZeroClaw 的"编译时菜单"——按需点菜，不点也能吃饱（默认安装已覆盖 90% 场景）。

**举例：**

```
┌─────────────────────────────────────────────────┐
│  ZeroClaw 编译时菜单                               │
├─────────────────────────────────────────────────┤
│                                                   │
│  🍚 默认套餐（0 Feature，已包含）：                  │
│     ✅ CLI 交互通道                                 │
│     ✅ Agent 核心循环 (ReAct)                       │
│     ✅ 基础工具集（文件、Shell、代码分析）            │
│     ✅ 所有 LLM Provider                           │
│     ✅ SQLite 内存后端                              │
│     → 90% 的用户到这里就够了                         │
│                                                   │
│  🍖 可选加菜（需要 --features 启用）：               │
│     · hardware          — USB/串口硬件控制          │
│     · channel-matrix    — Matrix 聊天协议           │
│     · channel-discord   — Discord 机器人            │
│     · memory-redis      — Redis 分布式记忆          │
│     · memory-postgres   — PostgreSQL 存储           │
│     · ... 共 12 个可选 Feature                      │
│                                                   │
│  ⚠️ 注意：加菜 = 编译进二进制，吃不了不能退          │
│     （没有运行时 tree-shaking！）                    │
└─────────────────────────────────────────────────┘
```

```bash
# 默认安装（推荐，最小体积）
cargo install zeroclaw

# 只加你需要的菜
cargo install zeroclaw --features "channel-matrix"

# ❌ 不推荐：全部加菜（体积翻倍、编译时间 ×5、可能依赖报错）
cargo install zeroclaw --all-features
```

**前端类比：** 类似于 webpack 的 `externals` 配置——你可以选择哪些模块打包进 bundle、哪些排除。但区别是：前端有运行时 tree-shaking 兜底（装了不用也不影响 bundle），Rust 的 Feature 一旦启用就编译进去了，无法运行时移除。

**应用：** 如果你用的是一键脚本或 Homebrew 安装，Feature Flags 和你没关系——预编译版本已经做好了选择。只有从源码 `cargo install` 时才需要关心 Feature Flags。**大多数前端开发者用默认安装就完全够了。**

### 卡片8：配置优先级链

**一句话：** ZeroClaw 的配置像 CSS 层叠一样，有明确的优先级——越"近"的越优先。

**举例：**

```
优先级从高到低：

① 环境变量          →  最高优先（临时覆盖一切）
   ZEROCLAW_MODEL=gpt-4o

② 命令行参数        →  次高（只影响本次执行）
   --model gpt-4o

③ 项目 config.toml  →  中等（项目级配置）
   .zeroclaw/config.toml

④ 全局 config.toml  →  较低（用户级配置）
   ~/.zeroclaw/config.toml

⑤ 代码默认值        →  最低（内置在源码中）
   DEFAULT_MODEL = "gpt-3.5-turbo"
```

**验证当前生效的配置：**

```bash
zeroclaw config show
# default_model = "gpt-4o"  [source: env]        ← 一目了然
# default_provider = "openrouter"  [source: config.toml]
```

**前端类比：**

```
CSS 优先级：          ZeroClaw 配置优先级：
─────────────────    ─────────────────────
!important         = 环境变量（最高）
inline style       = 命令行参数
#id                = 项目配置文件
.class             = 全局配置文件
element            = 代码默认值（最低）
```

**应用：** 遇到"明明改了配置却不生效"的问题时，先运行 `zeroclaw config show` 查看每个值的来源。99% 的情况是更高优先级的配置覆盖了你的修改——就像你改了 `.class` 的 CSS 却被 `#id` 覆盖了一样。

### 卡片9：多平台一次编写

**一句话：** ZeroClaw 通过 Target Triple（目标三元组）实现"一份源码，编译到 6 个平台"。

**举例：**

```
Target Triple = OS + CPU 架构 + 附加信息

ZeroClaw 支持的 6 个平台：

  x86_64-unknown-linux-gnu       → Intel/AMD Linux
  aarch64-unknown-linux-gnu      → ARM64 Linux（树莓派 4+、云服务器）
  armv7-unknown-linux-gnueabihf  → ARMv7 Linux（树莓派 3、旧设备）
  x86_64-apple-darwin            → Intel Mac
  aarch64-apple-darwin           → Apple Silicon Mac (M1/M2/M3/M4)
  x86_64-pc-windows-msvc         → Windows
```

**交叉编译示例（在 Mac 上编译 Linux 版本）：**

```bash
# 安装目标平台工具链
rustup target add x86_64-unknown-linux-gnu

# 交叉编译
cargo build --release --target x86_64-unknown-linux-gnu
# → 产出 Linux 可执行文件，可以直接部署到服务器
```

**前端类比：** 类似于 Electron 的多平台打包——`electron-builder` 可以在 macOS 上同时打包 `.dmg`（Mac）、`.deb`（Linux）、`.exe`（Windows）。ZeroClaw 的 CI/CD 用 GitHub Actions 在多个矩阵环境中并行编译，产出所有平台的预编译二进制。

**应用：** 作为**使用者**，你不需要关心交叉编译——一键安装脚本会自动下载对应你平台的预编译版本。但如果你要在嵌入式设备（如树莓派）上运行 ZeroClaw，或者为不同平台做 CI/CD 构建，就需要理解 Target Triple 的概念。

### 卡片10：从安装到 Hello World

**一句话：** 4 步完成从零到第一次 Agent 对话——安装、配置、验证、运行。

**举例：**

```bash
# ━━━ Step 1: 安装（30 秒）━━━
curl -fsSL https://zeroclawlabs.ai/install.sh | bash
source ~/.bashrc   # 或 source ~/.zshrc，让 PATH 生效

# ━━━ Step 2: 配置（30 秒）━━━
zeroclaw onboard
# 交互式选择 Provider → 输入 API Key → 选择模型
# 自动生成 ~/.zeroclaw/config.toml

# ━━━ Step 3: 验证（10 秒）━━━
zeroclaw --version   # 确认 CLI 安装成功
zeroclaw doctor      # 确认环境 + API 连通

# ━━━ Step 4: Hello World！（5 秒）━━━
zeroclaw agent -m "Hello! What can you do?"
# 🤖 看到 Agent 回复 = 一切就绪！
```

**没有 API Key？用免费本地模型：**

```bash
# 先装 Ollama（本地 LLM 运行器）
curl -fsSL https://ollama.ai/install.sh | bash
ollama pull llama3.1

# 再配 ZeroClaw 指向 Ollama（不需要 API Key！）
zeroclaw onboard --provider ollama --model llama3.1
zeroclaw agent -m "Hello!"
```

**前端类比：**

```
ZeroClaw 首次体验               React 首次体验
────────────────────────        ────────────────────
curl ... | bash                 npx create-react-app my-app
zeroclaw onboard                cd my-app
zeroclaw doctor                 （自动完成）
zeroclaw agent -m "Hello!"     npm start → 浏览器打开
────────────────────────        ────────────────────
总耗时: ~2 分钟                  总耗时: ~2 分钟
```

**应用：** 这就是"最小路径"——不看任何原理文档，2 分钟跑起来。**先跑通再学原理**是最高效的学习策略。跑通之后，回头阅读其他卡片理解"为什么"，会事半功倍。
