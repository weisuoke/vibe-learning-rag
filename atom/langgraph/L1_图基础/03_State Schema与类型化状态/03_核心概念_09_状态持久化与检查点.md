# 核心概念09：状态持久化与检查点

> 使用Checkpointer保存和恢复图的执行状态

## 概念定义

**状态持久化通过Checkpointer机制保存图的执行状态，支持中断恢复、时间旅行调试和多轮对话，是LangGraph实现有状态应用的核心。**

## 为什么需要持久化

### 问题场景

```python
# 无持久化的问题
app = graph.compile()

# 第一次运行
result1 = app.invoke({"messages": ["hello"]})

# 第二次运行 - 状态丢失
result2 = app.invoke({"messages": ["world"]})
# 无法访问第一次的状态
```

**持久化解决了状态保存和恢复的问题。**

## Checkpointer基础

### 1. MemorySaver

```python
from langgraph.checkpoint.memory import MemorySaver

# 创建内存检查点
checkpointer = MemorySaver()

# 编译图
app = graph.compile(checkpointer=checkpointer)
```

**特点**：
- 存储在内存中
- 进程重启后丢失
- 适合开发和测试

### 2. SqliteSaver

```python
from langgraph.checkpoint.sqlite import SqliteSaver

# 创建SQLite检查点
checkpointer = SqliteSaver.from_conn_string("checkpoints.db")

app = graph.compile(checkpointer=checkpointer)
```

**特点**：
- 持久化到磁盘
- 进程重启后保留
- 适合生产环境

### 3. PostgresSaver

```python
from langgraph.checkpoint.postgres import PostgresSaver

# 创建PostgreSQL检查点
checkpointer = PostgresSaver.from_conn_string(
    "postgresql://user:pass@localhost/db"
)

app = graph.compile(checkpointer=checkpointer)
```

**特点**：
- 分布式存储
- 高可用性
- 适合大规模生产

## 使用检查点

### 1. 基础使用

```python
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)

# 第一次运行
config = {"configurable": {"thread_id": "1"}}
result1 = app.invoke({"messages": ["hello"]}, config)

# 第二次运行 - 继续之前的状态
result2 = app.invoke({"messages": ["world"]}, config)
```

### 2. 多线程管理

```python
# 线程1
config1 = {"configurable": {"thread_id": "user1"}}
result1 = app.invoke({"messages": ["hello"]}, config1)

# 线程2
config2 = {"configurable": {"thread_id": "user2"}}
result2 = app.invoke({"messages": ["hi"]}, config2)

# 两个线程的状态独立
```

### 3. 获取状态

```python
# 获取当前状态
state = app.get_state(config)
print(state.values)  # 当前状态值
print(state.next)    # 下一个要执行的节点
```

### 4. 更新状态

```python
# 手动更新状态
app.update_state(
    config,
    {"messages": ["manual update"]}
)
```

## 中断与恢复

### 1. 中断前

```python
app = graph.compile(
    checkpointer=checkpointer,
    interrupt_before=["human_review"]
)

# 运行到human_review前中断
result = app.invoke({"task": "review this"}, config)
```

### 2. 中断后

```python
app = graph.compile(
    checkpointer=checkpointer,
    interrupt_after=["critical_step"]
)

# 运行完critical_step后中断
result = app.invoke({"task": "process"}, config)
```

### 3. 恢复执行

```python
# 中断后恢复
result = app.invoke(None, config)  # 从中断点继续
```

## 时间旅行

### 1. 查看历史

```python
# 获取所有检查点
history = app.get_state_history(config)

for state in history:
    print(f"Step: {state.metadata['step']}")
    print(f"Values: {state.values}")
```

### 2. 回到过去

```python
# 获取特定检查点
checkpoint_id = "checkpoint_123"
state = app.get_state(config, checkpoint_id=checkpoint_id)

# 从该检查点继续
result = app.invoke(None, config, checkpoint_id=checkpoint_id)
```

## 实际应用

### 1. 多轮对话

```python
from typing import TypedDict, Annotated
import operator
from langgraph.checkpoint.memory import MemorySaver

class ChatState(TypedDict):
    messages: Annotated[list, operator.add]

graph = StateGraph(ChatState)

def chat_node(state: ChatState) -> dict:
    messages = state["messages"]
    response = llm.invoke(messages)
    return {"messages": [response]}

graph.add_node("chat", chat_node)
graph.set_entry_point("chat")
graph.set_finish_point("chat")

checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)

# 多轮对话
config = {"configurable": {"thread_id": "conversation1"}}

# 第一轮
app.invoke({"messages": [{"role": "user", "content": "Hello"}]}, config)

# 第二轮 - 保留历史
app.invoke({"messages": [{"role": "user", "content": "How are you?"}]}, config)
```

### 2. 人工审核工作流

```python
class ReviewState(TypedDict):
    content: str
    status: str
    feedback: str

graph = StateGraph(ReviewState)

def auto_process(state: ReviewState) -> dict:
    return {"status": "pending_review"}

def human_review(state: ReviewState) -> dict:
    # 等待人工审核
    return {}

def finalize(state: ReviewState) -> dict:
    return {"status": "completed"}

graph.add_node("process", auto_process)
graph.add_node("review", human_review)
graph.add_node("finalize", finalize)

graph.add_edge("process", "review")
graph.add_edge("review", "finalize")

graph.set_entry_point("process")
graph.set_finish_point("finalize")

# 在review前中断
app = graph.compile(
    checkpointer=checkpointer,
    interrupt_before=["review"]
)

# 提交任务
config = {"configurable": {"thread_id": "task1"}}
app.invoke({"content": "review this", "status": "new"}, config)

# 人工审核后继续
app.update_state(config, {"feedback": "approved"})
app.invoke(None, config)
```

### 3. 长时间运行任务

```python
class TaskState(TypedDict):
    task_id: str
    progress: int
    result: str

def long_task(state: TaskState) -> dict:
    # 模拟长时间任务
    for i in range(10):
        # 每步保存检查点
        yield {"progress": i * 10}
    return {"result": "completed", "progress": 100}

graph.add_node("task", long_task)
app = graph.compile(checkpointer=checkpointer)

# 运行任务
config = {"configurable": {"thread_id": "long_task1"}}
for chunk in app.stream({"task_id": "task1", "progress": 0}, config):
    print(f"Progress: {chunk}")
```

## 性能优化

### 1. 批量保存

```python
# 配置批量保存
checkpointer = SqliteSaver.from_conn_string(
    "checkpoints.db",
    batch_size=100
)
```

### 2. 压缩存储

```python
# 启用压缩
checkpointer = SqliteSaver.from_conn_string(
    "checkpoints.db",
    compress=True
)
```

### 3. 定期清理

```python
# 清理旧检查点
def cleanup_old_checkpoints(days=7):
    cutoff = time.time() - (days * 24 * 3600)
    checkpointer.delete_before(cutoff)
```

## 最佳实践

### 1. 使用唯一thread_id

```python
# ✓ 使用用户ID或会话ID
config = {"configurable": {"thread_id": f"user_{user_id}"}}

# ❌ 使用固定ID
config = {"configurable": {"thread_id": "default"}}
```

### 2. 定期保存

```python
# ✓ 关键步骤后保存
def critical_step(state):
    result = process()
    # 自动保存检查点
    return {"result": result}
```

### 3. 错误恢复

```python
try:
    result = app.invoke(input, config)
except Exception as e:
    # 从最后的检查点恢复
    state = app.get_state(config)
    print(f"Failed at: {state.next}")
    # 修复后重试
    result = app.invoke(None, config)
```

## 总结

**状态持久化的核心**：
- Checkpointer保存执行状态
- 支持中断和恢复
- 实现多轮对话
- 时间旅行调试

**关键点**：
- 使用唯一thread_id
- 选择合适的Checkpointer
- 定期清理旧检查点
