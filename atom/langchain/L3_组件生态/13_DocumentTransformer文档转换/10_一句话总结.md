# DocumentTransformer文档转换 - 一句话总结

## 一句话总结

**DocumentTransformer 是 LangChain 中对文档序列进行变换的统一抽象接口，支持清洗、过滤、增强、重排序等操作，通过 Pipeline 组合构建文档预处理管道，直接决定 RAG 系统的检索质量上限。**

## 三个关键要点

### 要点 1：统一接口是核心设计（最重要）

**核心观点：** `Sequence[Document] → Sequence[Document]` 这个统一签名，让所有转换器可以像乐高积木一样自由组合。

**为什么重要？**
- 所有转换器（清洗、分块、过滤、重排序）共享同一个接口
- DocumentCompressorPipeline 正是基于这个统一接口实现管道串联
- TextSplitter 也是 DocumentTransformer 的子类，天然可以参与管道组合

**快速上手：**
```python
from langchain.retrievers.document_compressors import DocumentCompressorPipeline
from langchain_community.document_transformers import (
    EmbeddingsRedundantFilter,
    LongContextReorder,
)

# 任意转换器都能串联——因为接口统一
pipeline = DocumentCompressorPipeline(
    transformers=[
        EmbeddingsRedundantFilter(embeddings=embeddings),
        LongContextReorder(),
    ]
)
```

**在 RAG 中的应用：**
- 文档预处理：清洗 → 分块 → 存入向量库
- 检索后优化：去重 → 聚类 → 重排序 → 送入 LLM
- 不同场景配置不同 Pipeline，灵活应对业务需求

---

### 要点 2：检索后优化是被低估的提升点（高价值）

**核心观点：** 大多数人只关注 embedding 模型和分块策略，却忽略了检索后的文档优化——冗余过滤、聚类选择、上下文重排序。

**为什么重要？**
- EmbeddingsRedundantFilter 去重可提升信息密度 30-40%
- EmbeddingsClusteringFilter 聚类可提升结果多样性 20-30%
- LongContextReorder 重排序可提升准确率 10-15%（解决 Lost in the Middle）

**快速上手：**
```python
from langchain.retrievers import ContextualCompressionRetriever

# 将优化管道集成到检索器
compression_retriever = ContextualCompressionRetriever(
    base_compressor=pipeline,
    base_retriever=vector_retriever,
)

# 检索时自动执行优化
results = compression_retriever.invoke("查询内容")
```

**在 RAG 中的应用：**
- 知识库问答：去重 + 重排序，提升回答准确性
- 多文档摘要：聚类选择，确保覆盖多个主题
- 长文档处理：重排序，确保关键信息不被 LLM 忽略

---

### 要点 3：自定义转换器满足业务需求（实用）

**核心观点：** 继承 BaseDocumentTransformer 实现 `transform_documents` 方法，就能创建满足特定业务需求的转换器。

**为什么重要？**
- 内置转换器覆盖通用场景，但企业级应用往往有特殊需求
- 敏感信息脱敏、质量评分、业务规则过滤等都需要自定义
- 自定义转换器可以无缝接入 Pipeline

**快速上手：**
```python
from langchain_core.documents.transformers import BaseDocumentTransformer
from langchain_core.documents import Document
from typing import Any, Sequence

class QualityFilter(BaseDocumentTransformer):
    """过滤低质量文档"""

    def __init__(self, min_length: int = 50):
        self.min_length = min_length

    def transform_documents(
        self, documents: Sequence[Document], **kwargs: Any
    ) -> Sequence[Document]:
        return [
            doc for doc in documents
            if len(doc.page_content) >= self.min_length
        ]
```

**在 RAG 中的应用：**
- 合规需求：GDPR 数据脱敏转换器
- 质量控制：过滤过短、乱码、重复的文档
- 业务增强：自动添加来源标注、时间戳等元数据
