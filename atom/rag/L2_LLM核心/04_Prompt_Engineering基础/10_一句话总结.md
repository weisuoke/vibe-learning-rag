# 一句话总结

## 核心总结

**Prompt Engineering 是用精确的人类语言"编程"大模型，通过角色设定、清晰指令、格式控制、上下文工程和质量控制五大技巧，让 AI 准确理解意图并生成高质量输出，是 RAG 系统从"能用"到"好用"的关键。**

---

## 记住这三点

### 1. Prompt 是 AI 的"编程语言"

就像前端开发用 JavaScript 控制浏览器行为，Prompt Engineering 用自然语言控制 AI 行为。

**关键差异：**
- 传统编程：语法严格，逻辑确定
- Prompt 编程：语义模糊，需要精心设计消除歧义

### 2. RAG 的成败取决于 Prompt 质量

| 差的 Prompt | 好的 Prompt | 效果差异 |
|------------|------------|---------|
| "总结这些文档" | "作为技术文档专家，用3个要点总结以下内容，每点不超过50字" | 准确度 +40% |
| "回答问题" | "基于检索到的上下文回答，如果上下文不足请明确说明" | 幻觉率 -60% |
| 返回自由文本 | 返回 JSON 格式 `{"answer": "...", "confidence": 0.9}` | 可解析性 100% |

**来源：** [Source: IBM 2026 Prompt Engineering Guide](https://www.ibm.com/think/prompt-engineering)

### 3. 2025-2026 的新重点：上下文工程

传统 Prompt Engineering 关注"怎么问"，现代 Prompt Engineering 更关注"给什么上下文"。

**RAG 的核心挑战：**
- 检索到 10 篇文档，但 Context Window 只能放 3 篇 → 如何选择？
- 文档总长度 5000 tokens，但只有 2000 tokens 预算 → 如何压缩？
- 多轮对话历史 + 检索内容 → 如何平衡？

**解决方案：** Context Engineering（上下文工程）

**来源：** [Source: Context Engineering - The Next Frontier](https://www.deepset.ai/blog/context-engineering-the-next-frontier-beyond-prompt-engineering)

## 实践建议

### 立即可用的技巧

```python
# ❌ 差的 Prompt
prompt = "总结文档"

# ✅ 好的 Prompt（应用五大技巧）
prompt = """
你是一个专业的技术文档分析师（角色设定）。

请按以下步骤总结文档（指令清晰）：
1. 提取3个核心要点
2. 每个要点不超过50字
3. 如果文档不包含技术内容，返回"非技术文档"

返回格式（格式控制）：
{
  "summary": ["要点1", "要点2", "要点3"],
  "is_technical": true
}

上下文（上下文工程）：
{retrieved_docs}

约束（输出质量控制）：
- 只基于上下文回答，不要编造信息
- 如果上下文不足，明确说明
"""
```

## 最后的话

Prompt Engineering 不是"玄学"，而是一门系统化的工程学科。

**核心原则：**
1. **明确性** > 简洁性：宁可啰嗦也要清晰
2. **结构化** > 自由文本：用格式约束输出
3. **上下文质量** > 上下文数量：精选比堆砌更重要
4. **迭代优化** > 一次完美：通过测试不断改进

**记住：** 好的 Prompt 能让普通模型表现出色，差的 Prompt 会让顶级模型表现平庸。
