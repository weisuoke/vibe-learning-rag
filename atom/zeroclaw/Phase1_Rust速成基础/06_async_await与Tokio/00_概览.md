# async/await 与 Tokio - 概览

> **一句话定位**：Rust 的 `async/await` 配合 Tokio 运行时实现零开销异步 IO，`async fn` 编译为惰性状态机、由 Tokio 调度执行，是 ZeroClaw 整个异步架构（1939 个 async fn、124 个 #[async_trait]）的底层引擎。

## 2. 为什么前端开发者需要学 Rust async/await？

### 2.1 你已经很熟悉异步编程了

作为前端/TypeScript 开发者，你每天都在写 `async/await`。好消息是——Rust 的 `async/await` **语法几乎一样**：

```typescript
// TypeScript — 你已经很熟悉了
async function fetchUser(id: string): Promise<User> {
    const response = await fetch(`/api/users/${id}`);
    const user = await response.json();
    return user;
}
```

```rust
// Rust — 语法几乎相同！
async fn fetch_user(id: &str) -> Result<User> {
    let response = reqwest::get(&format!("/api/users/{}", id)).await?;
    let user = response.json::<User>().await?;
    Ok(user)
}
```

### 2.2 但底层机制完全不同

| 维度 | JavaScript async/await | Rust async/await |
|------|----------------------|------------------|
| **执行时机** | 调用即执行（eager） | 惰性的，`.await` 或 `poll` 才执行（lazy） |
| **运行时** | V8 引擎内置事件循环 | 需要显式选择运行时（Tokio） |
| **取消** | 困难，需 AbortController | drop Future 即取消 |
| **返回类型** | `Promise<T>` 一个类型 | `impl Future<Output = T>` 编译器生成状态机 |
| **线程模型** | 单线程事件循环 | 多线程工作窃取调度器 |
| **Send 约束** | 不存在（单线程） | 跨线程 spawn 需要 `Send + 'static` |
| **开销** | 微任务队列 + 堆分配 | 零开销抽象，编译为状态机 enum |

**最关键的区别**：

```typescript
// JavaScript — 调用 fetchUser() 的瞬间就开始请求了
const promise = fetchUser("123");  // ← 已经在执行了！
// 即使你永远不 await 它，请求也已经发出去了
```

```rust
// Rust — 调用 fetch_user() 只是创建了一个"计划"，什么都没执行
let future = fetch_user("123");    // ← 什么都没发生！只是一个状态机实例
// 必须 .await 或传给执行器才会真正开始
future.await?;                      // ← 现在才开始执行！
```

> **类比**：JavaScript 的 Promise 像在线点外卖——下单那刻厨房就开火了。Rust 的 Future 像一张食谱卡——你可以传递它、复制它、扔掉它，只有厨师（执行器）拿起它开始做时才会开火。

### 2.3 为什么 ZeroClaw 需要 async？

ZeroClaw 是一个 AI Agent 框架，核心工作是**大量 IO 密集型操作**：

- 调用 LLM API（Anthropic/OpenAI）→ 每次 100ms-30s
- 执行 Shell 命令 → 等待进程输出
- 读写文件系统 → 磁盘 IO
- 收发消息（Telegram/Discord）→ 网络 IO
- 存取记忆（SQLite/Postgres）→ 数据库 IO

如果用同步代码，Agent 在等 LLM 回复时就只能**干等着**，什么都做不了。async/await 让它在等待时可以：同时接收新消息、执行工具、监控组件健康……

### 3.3 Tokio Feature 配置

ZeroClaw 使用了几乎所有 Tokio feature：

```toml
tokio = { version = "1.42", default-features = false, features = [
    "rt-multi-thread",  # 多线程工作窃取调度器
    "macros",           # #[tokio::main], #[tokio::test]
    "time",             # sleep, interval, timeout
    "net",              # TCP/UDP 网络
    "io-util",          # IO 读写工具
    "sync",             # mpsc, oneshot, Mutex 通道和锁
    "process",          # 子进程管理（Shell 工具）
    "io-std",           # stdin/stdout
    "fs",               # 异步文件系统
    "signal"            # Ctrl+C 信号处理
] }
```
