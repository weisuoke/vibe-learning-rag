> **目标受众**: 有 TypeScript/前端经验但零 Rust 经验的开发者

## 场景目标

**从零创建一个 `mini-agent` 命令行工具**，它能：
1. 从 TOML 配置文件读取设定（使用 `serde` + `toml`）
2. 用清晰的模块结构组织代码（config / utils / agent）
3. 正确处理错误（使用 `anyhow`）
4. 包含单元测试

---

## Step 1：初始化项目

```bash
cargo new mini-agent    # = npm init -y + 自动生成骨架
cd mini-agent
```

**生成的目录结构**（`cargo run` 确认可运行）：

```
mini-agent/
├── Cargo.toml      ← 项目配置（= package.json）
├── .gitignore       ← 自动生成
└── src/
    └── main.rs      ← 入口文件（= index.ts）
```

---

## Step 2：添加依赖

```bash
cargo add serde --features derive    # 序列化/反序列化框架
cargo add toml                       # TOML 文件解析
cargo add anyhow                     # 简化错误处理
```

**添加后的 Cargo.toml**：

```toml
[package]
name = "mini-agent"
version = "0.1.0"
edition = "2024"

[dependencies]
anyhow = "1"                                      # 万能 Error 类型
serde = { version = "1", features = ["derive"] }   # 自动派生序列化
toml = "0.8"                                       # TOML 解析器
```

与 `npm install` 不同，`cargo add` 不会立即下载——依赖会在首次 `cargo build` 时自动下载 [^1]。

---

## Step 3：创建模块结构

```bash
touch src/config.rs src/agent.rs src/utils.rs
```

```
src/
├── main.rs        ← 入口点：声明模块 + 组装运行
├── config.rs      ← 配置模块：读取和解析 TOML
├── agent.rs       ← Agent 模块：核心业务逻辑
└── utils.rs       ← 工具模块：辅助函数
```

**核心区别**：TypeScript 中文件存在就能 `import`；Rust 中**必须在 `main.rs` 里用 `mod` 显式声明**，否则编译器不会编译这些文件 [^2]。

---

## Step 4：实现配置模块（config.rs）

先在项目根目录创建 **config.toml**：

```toml
[agent]
name = "mini-agent"
version = "0.1.0"
max_retries = 3
verbose = true
```

**src/config.rs**：

```rust
use anyhow::{Context, Result};
use serde::Deserialize;
use std::fs;

/// Agent 配置段（对应 TOML 中的 [agent] 表）
#[derive(Debug, Deserialize)]
pub struct AgentConfig {
    pub name: String,
    pub version: String,
    pub max_retries: u32,
    pub verbose: bool,
}

/// 顶层配置结构（对应整个 TOML 文件）
#[derive(Debug, Deserialize)]
pub struct Config {
    pub agent: AgentConfig,
}

impl Config {
    /// 从 TOML 文件加载配置
    pub fn load(path: &str) -> Result<Self> {
        let content = fs::read_to_string(path)
            .with_context(|| format!("无法读取配置文件: {}", path))?;
        let config: Config = toml::from_str(&content)
            .with_context(|| format!("TOML 解析失败: {}", path))?;
        Ok(config)
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_load_valid_config() {
        let toml_str = r#"
            [agent]
            name = "test-agent"
            version = "0.0.1"
            max_retries = 5
            verbose = false
        "#;
        let config: Config = toml::from_str(toml_str).unwrap();
        assert_eq!(config.agent.name, "test-agent");
        assert_eq!(config.agent.max_retries, 5);
        assert!(!config.agent.verbose);
    }

    #[test]
    fn test_load_missing_field() {
        let toml_str = r#"
            [agent]
            name = "test"
        "#;
        let result: Result<Config, _> = toml::from_str(toml_str);
        assert!(result.is_err());
    }
}
```

**关键设计**：
- `#[derive(Deserialize)]`——让 serde 自动生成反序列化代码
- `anyhow::Result<T>` = `Result<T, anyhow::Error>`，省去自定义错误类型
- `.with_context(|| ...)`——为错误添加上下文，调试时能看到**哪一步**出错

## Step 5：实现工具模块（utils.rs）

**src/utils.rs**：

```rust
/// pub 函数——其他模块可通过 crate::utils::format_timestamp 访问
pub fn format_timestamp() -> String {
    let now = std::time::SystemTime::now()
        .duration_since(std::time::UNIX_EPOCH)
        .unwrap_or_default();
    format!("[{}s]", now.as_secs())
}

/// pub(crate)：仅 crate 内部可用，外部库不能调用
pub(crate) fn truncate(s: &str, max_len: usize) -> String {
    if s.len() <= max_len {
        s.to_string()
    } else {
        format!("{}...", &s[..max_len])
    }
}

/// 私有——仅 utils 模块内部使用（没有 pub）
fn _internal_helper() -> &'static str {
    "这个函数外部看不到"
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_format_timestamp() {
        let ts = format_timestamp();
        assert!(ts.starts_with('['));
        assert!(ts.ends_with("s]"));
    }

    #[test]
    fn test_truncate_short_string() {
        assert_eq!(truncate("hello", 10), "hello");
    }

    #[test]
    fn test_truncate_long_string() {
        assert_eq!(truncate("hello world", 5), "hello...");
    }
}
```

**可见性三级对比**：

```rust
pub fn format_timestamp()      // 完全公开        = TypeScript export function
pub(crate) fn truncate()       // crate 内部      ≈ 不 export 的函数
fn _internal_helper()          // 私有（模块内）   = TypeScript 无精确等价
```

## Step 6：实现 Agent 模块（agent.rs）

**src/agent.rs**：

```rust
use crate::config::Config;     // crate:: = 从项目根路径开始（≈ @/ 路径别名）
use crate::utils;              // 引入整个 utils 模块

pub struct Agent {
    config: Config,            // 私有字段——外部不能直接 agent.config
    run_count: u32,
}

impl Agent {
    pub fn new(config: Config) -> Self {
        Agent { config, run_count: 0 }
    }

    pub fn run(&mut self) {
        self.run_count += 1;
        let ts = utils::format_timestamp();
        let cfg = &self.config.agent;

        println!("{} {} v{} 启动（第 {} 次运行）",
            ts, cfg.name, cfg.version, self.run_count);

        if cfg.verbose {
            println!("  [详细] max_retries = {}", cfg.max_retries);
        }

        self.execute_tasks();
        println!("{} {} 运行完成 ✓", utils::format_timestamp(), cfg.name);
    }

    /// 私有方法——仅 Agent 内部使用
    fn execute_tasks(&self) {
        let tasks = vec!["检查环境", "加载工具", "就绪"];
        for (i, task) in tasks.iter().enumerate() {
            let status = utils::truncate(task, 20);
            println!("  [{}/{}] {}", i + 1, tasks.len(), status);
        }
    }

    pub fn run_count(&self) -> u32 {
        self.run_count
    }
}

#[cfg(test)]
mod tests {
    use super::*;

    fn test_config() -> Config {
        let toml_str = r#"
            [agent]
            name = "test-agent"
            version = "0.0.1"
            max_retries = 1
            verbose = false
        "#;
        toml::from_str(toml_str).unwrap()
    }

    #[test]
    fn test_agent_creation() {
        let agent = Agent::new(test_config());
        assert_eq!(agent.run_count(), 0);
    }

    #[test]
    fn test_agent_run_increments_count() {
        let mut agent = Agent::new(test_config());
        agent.run();
        assert_eq!(agent.run_count(), 1);
        agent.run();
        assert_eq!(agent.run_count(), 2);
    }
}
```

**struct 字段私有是 Rust 的封装理念**——外部通过 `pub fn` 方法访问，内部可自由修改。ZeroClaw 的 `Agent` 也遵循这一模式 [^3]。

## Step 7：组装 main.rs

**src/main.rs**：

```rust
// ─── Step 1：声明模块（必须！Rust 不会自动发现文件）─── //
mod config;     // → 编译器去找 src/config.rs
mod agent;      // → 编译器去找 src/agent.rs
mod utils;      // → 编译器去找 src/utils.rs

// ─── Step 2：导入需要的类型 ─── //
use config::Config;
use agent::Agent;

fn main() {
    // Step 3：加载配置
    let config = Config::load("config.toml").unwrap_or_else(|err| {
        eprintln!("❌ 启动失败: {:#}", err);   // {:#} 打印完整错误链
        std::process::exit(1);
    });

    println!("✅ 配置加载成功: {:?}", config.agent.name);

    // Step 4：创建并运行 Agent
    let mut agent = Agent::new(config);
    agent.run();
    println!("\n📊 总共运行 {} 次", agent.run_count());
}
```

**main.rs 的三个职责**：`mod` 声明模块 → `use` 导入类型 → `fn main()` 组装运行。

## 完整项目结构回顾

```
mini-agent/
├── Cargo.toml             ← [package] + [dependencies]: serde, toml, anyhow
├── Cargo.lock             ← 依赖版本锁定（首次 build 后自动生成）
├── config.toml            ← 应用配置文件
├── target/                ← 编译产物（.gitignore 已忽略）
└── src/
    ├── main.rs            ← 3 个 mod 声明 + fn main()
    ├── config.rs          ← Config/AgentConfig struct + load() + 2 tests
    ├── agent.rs           ← Agent struct + new()/run() + 2 tests
    └── utils.rs           ← pub/pub(crate)/private 三级可见性 + 3 tests
```

## 知识点速览

| 知识点 | 项目中的体现 |
|--------|------------|
| `cargo new` / `cargo add` | 初始化项目 + 添加依赖 |
| `mod` 声明 | main.rs 的三行 `mod xxx;` |
| `use crate::` 路径 | agent.rs 引用 config 和 utils |
| `pub` / `pub(crate)` / 私有 | utils.rs 的三级可见性 |
| struct 字段可见性 | Agent 的私有字段 + pub 方法 |
| `#[derive(Deserialize)]` | Config 的自动反序列化 |
| `#[cfg(test)]` | 每个文件的测试模块 |
| `anyhow::Result` | Config::load 的错误处理 |

## 与 ZeroClaw 的对比

mini-agent 用的 `mod` / `use` / `pub` 模式，和 ZeroClaw 的 30+ 模块用的是**同一套规则**。区别只在规模：ZeroClaw 用目录模块（`agent/mod.rs`）代替文件模块（`agent.rs`），因为每个模块下还有子模块 [^3]。
