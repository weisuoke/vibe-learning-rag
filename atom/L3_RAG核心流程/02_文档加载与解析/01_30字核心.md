# 30字核心

**文档加载与解析是将 PDF、Word 等多格式文档转换为纯文本的过程，是 RAG 系统数据入口的关键环节。**

---

## 拆解理解

| 关键词 | 含义 |
|-------|------|
| **文档加载（Loading）** | 读取文件内容到内存中 |
| **解析（Parsing）** | 从文件格式中提取纯文本和结构信息 |
| **多格式** | PDF、Word、HTML、Markdown、TXT 等 |
| **纯文本** | 去除格式后的文字内容，可供后续处理 |
| **数据入口** | RAG 系统离线索引阶段的第一步 |

---

## 为什么这句话重要？

理解这句话，你就理解了：

1. **文档加载是 RAG 的起点** → 没有高质量的文本输入，后续一切都是空谈
2. **解析质量决定检索质量** → 垃圾进，垃圾出（Garbage In, Garbage Out）
3. **不同格式需要不同策略** → PDF 解析和 Markdown 解析完全不同

---

## 文档加载的一句话公式

```
文档加载与解析 = 读取文件 + 提取文本 + 保留元数据
```

用前端的话说：

```
文档加载 = fs.readFile() + 解析器处理 + 返回结构化数据
```
