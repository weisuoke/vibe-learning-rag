# 核心概念10：与原生结构化输出对比

> 理解 OutputParser 与 with_structured_output() 的区别，选择最适合的方法

---

## 概述

在 LangChain 中获取结构化输出有两种主要方法：

1. **现代方法（2025+）**：使用 `with_structured_output()` + LLM 原生支持
2. **传统方法**：使用 `OutputParser` 手动解析

本文档详细对比这两种方法，帮助你在实际项目中做出正确选择。

---

## 1. 两种方法的核心区别

### 1.1 现代方法：with_structured_output()

**定义**：
> `with_structured_output()` 是 LangChain 提供的方法，利用 LLM 的原生结构化输出能力（如 OpenAI 的 Function Calling）直接生成符合 schema 的输出。

**工作原理**：

```python
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

class Person(BaseModel):
    """Information about a person."""
    name: str = Field(description="The person's name")
    age: int = Field(description="The person's age")

# 使用 with_structured_output()
llm = ChatOpenAI(model="gpt-4.1")
structured_llm = llm.with_structured_output(Person, method="json_schema")

# 直接调用，返回 Pydantic 对象
result = structured_llm.invoke("Michael is 35 years old")
print(result)  # Person(name="Michael", age=35)
```

**关键特性**：
- LLM 内部处理结构化输出（通过 Function Calling）
- 自动验证和类型转换
- 自动错误检测和重试
- 无需手动解析

### 1.2 传统方法：OutputParser

**定义**：
> `OutputParser` 是 LangChain 的早期解决方案，通过手动解析 LLM 的文本输出来获取结构化数据。

**工作原理**：

```python
from langchain_core.output_parsers import PydanticOutputParser
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field

class Person(BaseModel):
    """Information about a person."""
    name: str = Field(description="The person's name")
    age: int = Field(description="The person's age")

# 使用 PydanticOutputParser
parser = PydanticOutputParser(pydantic_object=Person)
llm = ChatOpenAI(model="gpt-4.1")

# 获取格式指令
format_instructions = parser.get_format_instructions()

# 构建 Prompt（需要包含格式指令）
prompt = f"Extract info: Michael is 35 years old\n\n{format_instructions}"

# 调用链
chain = llm | parser
result = chain.invoke(prompt)
print(result)  # Person(name="Michael", age=35)
```

**关键特性**：
- LLM 输出普通文本（通常是 JSON）
- 手动解析和验证
- 需要在 Prompt 中注入格式指令
- 需要显式错误处理

## 2. 完整对比表格

| 特性 | with_structured_output() | OutputParser |
|------|--------------------------|--------------|
| **实现方式** | LLM 原生支持（Function Calling） | 手动解析 LLM 输出 |
| **支持的模型** | 仅支持结构化输出的模型（GPT-4, Claude 3+） | 所有 LLM 模型 |
| **Prompt 工程** | 无需格式指令（LLM 自动处理） | 需要注入格式指令 |
| **错误检测** | 自动（LLM 内置） | 手动（解析时检测） |
| **错误反馈** | 自动（ToolMessage） | 手动（抛出异常） |
| **重试机制** | 自动（Agent 循环） | 手动（需要实现） |
| **验证** | Pydantic 自动验证 | Pydantic 自动验证 |
| **流式支持** | 部分支持（逐字段） | 完全支持（逐块/累积） |
| **性能** | 更快（原生支持） | 较慢（需要解析） |
| **灵活性** | 受限于 LLM 能力 | 完全可控 |
| **自定义解析** | 不支持 | 完全支持 |
| **Markdown 代码块** | 不需要处理 | 自动处理 |
| **部分解析** | 不支持 | 支持（`partial=True`） |
| **JSON Patch** | 不支持 | 支持（流式 diff） |
| **成本** | 可能更高（Function Calling） | 标准成本 |
| **调试难度** | 较难（黑盒） | 较易（可见输出） |
| **代码复杂度** | 简单（一行代码） | 中等（需要链式组合） |

## 3. 何时使用 with_structured_output()

### 3.1 推荐场景

✅ **使用 with_structured_output() 当：**

1. **模型支持原生结构化输出**
   - OpenAI GPT-4.1+
   - Anthropic Claude 3+
   - Google Gemini 2.0+

2. **需要简单快速的实现**
   ```python
   # 一行代码搞定
   structured_llm = llm.with_structured_output(MyModel)
   result = structured_llm.invoke(input)
   ```

3. **需要自动错误处理和重试**
   - LLM 自动检测格式错误
   - 自动重试直到成功
   - 无需手动实现重试逻辑

4. **追求最佳性能**
   - 原生支持比手动解析快
   - 减少 token 消耗（无需格式指令）

5. **生产环境的标准场景**
   - 数据提取
   - 表单填充
   - 结构化问答

### 3.2 代码示例

```python
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field
from typing import List

class Movie(BaseModel):
    """A movie with details."""
    title: str = Field(..., description="The title of the movie")
    year: int = Field(..., description="The year the movie was released")
    director: str = Field(..., description="The director of the movie")
    rating: float = Field(..., description="The movie's rating out of 10")

class MovieList(BaseModel):
    """A list of movies."""
    movies: List[Movie] = Field(..., description="List of movies")

# 创建结构化 LLM
llm = ChatOpenAI(model="gpt-4.1")
structured_llm = llm.with_structured_output(MovieList, method="json_schema")

# 直接调用
result = structured_llm.invoke(
    "List 3 Christopher Nolan movies with their details"
)

print(result)
# MovieList(movies=[
#     Movie(title="Inception", year=2010, director="Christopher Nolan", rating=8.8),
#     Movie(title="Interstellar", year=2014, director="Christopher Nolan", rating=8.6),
#     Movie(title="The Dark Knight", year=2008, director="Christopher Nolan", rating=9.0)
# ])
```

### 3.3 流式支持

```python
# 流式结构化输出（逐字段返回）
for chunk in structured_llm.stream("Extract person info: Alice is 30"):
    print(chunk)
# 输出:
# {"name": "Alice"}
# {"name": "Alice", "age": 30}
```

## 4. 何时使用 OutputParser

### 4.1 推荐场景

✅ **使用 OutputParser 当：**

1. **模型不支持原生结构化输出**
   - 开源模型（Llama, Mistral）
   - 旧版本 GPT-3.5
   - 自定义微调模型

2. **需要自定义解析逻辑**
   ```python
   class CustomParser(BaseOutputParser):
       def parse_result(self, result):
           # 自定义解析逻辑
           return custom_logic(result)
   ```

3. **需要额外验证或处理**
   - 复杂的业务规则验证
   - 数据清洗和转换
   - 多步骤处理流程

4. **需要完整的流式支持**
   - 逐块解析（`BaseTransformOutputParser`）
   - 累积解析（`BaseCumulativeTransformOutputParser`）
   - JSON Patch diff 模式

5. **需要处理 Markdown 代码块**
   ```python
   # OutputParser 自动处理 Markdown 代码块
   # LLM 输出:
   # ```json
   # {"name": "Alice", "age": 30}
   # ```
   # 自动提取 JSON
   ```

6. **需要调试和可观测性**
   - 可以看到 LLM 的原始输出
   - 可以记录解析过程
   - 可以自定义错误处理

### 4.2 代码示例

```python
from langchain_core.output_parsers import PydanticOutputParser
from langchain_openai import ChatOpenAI
from langchain_core.prompts import PromptTemplate
from pydantic import BaseModel, Field

class Person(BaseModel):
    """Information about a person."""
    name: str = Field(description="The person's name")
    age: int = Field(description="The person's age")
    occupation: str = Field(description="The person's occupation")

# 创建 Parser
parser = PydanticOutputParser(pydantic_object=Person)

# 获取格式指令
format_instructions = parser.get_format_instructions()

# 创建 Prompt 模板
prompt = PromptTemplate(
    template="Extract person information from the text.\n{format_instructions}\n\nText: {text}",
    input_variables=["text"],
    partial_variables={"format_instructions": format_instructions}
)

# 创建链
llm = ChatOpenAI(model="gpt-4.1")
chain = prompt | llm | parser

# 调用
result = chain.invoke({"text": "Alice is 30 years old and works as a software engineer"})
print(result)
# Person(name="Alice", age=30, occupation="software engineer")
```

### 4.3 流式解析示例

```python
from langchain_core.output_parsers import JsonOutputParser

# 创建 JSON Parser（支持流式）
parser = JsonOutputParser()

# 流式解析
for chunk in chain.stream({"text": "..."}):
    print(chunk)
# 输出部分 JSON 对象
```

### 4.4 自定义 OutputParser

```python
from langchain_core.output_parsers import BaseOutputParser
from typing import List

class CustomListParser(BaseOutputParser[List[str]]):
    """自定义列表解析器"""

    def parse_result(self, result: List[Generation], *, partial: bool = False) -> List[str]:
        """解析 LLM 输出为列表"""
        text = result[0].text.strip()

        # 自定义解析逻辑
        lines = text.split('\n')
        items = [line.strip('- ').strip() for line in lines if line.strip()]

        return items

    def get_format_instructions(self) -> str:
        """返回格式指令"""
        return "Please return a list of items, one per line, prefixed with '-'."

# 使用自定义 Parser
parser = CustomListParser()
chain = llm | parser
result = chain.invoke("List 3 programming languages")
print(result)  # ['Python', 'JavaScript', 'Java']
```

## 5. 性能与灵活性权衡

### 5.1 性能对比

| 维度 | with_structured_output() | OutputParser |
|------|--------------------------|--------------|
| **响应速度** | ⚡⚡⚡ 快（原生支持） | ⚡⚡ 中等（需要解析） |
| **Token 消耗** | ⚡⚡⚡ 少（无格式指令） | ⚡⚡ 多（需要格式指令） |
| **成功率** | ⚡⚡⚡ 高（LLM 保证） | ⚡⚡ 中等（依赖 Prompt） |
| **流式性能** | ⚡⚡ 中等（逐字段） | ⚡⚡⚡ 好（逐块/累积） |

### 5.2 灵活性对比

| 维度 | with_structured_output() | OutputParser |
|------|--------------------------|--------------|
| **自定义解析** | ❌ 不支持 | ✅ 完全支持 |
| **额外验证** | ⚠️ 有限（Pydantic） | ✅ 完全支持 |
| **错误处理** | ⚠️ 自动（不可控） | ✅ 完全可控 |
| **调试能力** | ❌ 黑盒 | ✅ 完全可见 |
| **模型兼容** | ⚠️ 仅部分模型 | ✅ 所有模型 |

### 5.3 成本对比

```python
# with_structured_output() - 更少 token
# Prompt: "Extract person info: Alice is 30"
# 无需格式指令，直接调用 Function Calling

# OutputParser - 更多 token
# Prompt: "Extract person info: Alice is 30\n\n
# The output should be formatted as a JSON instance that conforms to the JSON schema below.
# As an example, for the schema {...}
# the object {...} is a well-formatted instance of the schema.
# The output is parsed and type-checked according to the provided schema instance.
# Here is the output schema:
# {...}"
```

**成本估算**：
- `with_structured_output()`：约 50-100 tokens（输入）
- `OutputParser`：约 150-300 tokens（输入，包含格式指令）

## 6. 错误处理对比

### 6.1 with_structured_output() 错误处理

**自动错误处理流程**：

```python
from langchain.agents import create_agent
from langchain.agents.structured_output import ToolStrategy

# 自动错误处理（默认启用）
agent = create_agent(
    model="gpt-5",
    response_format=ToolStrategy(Person)  # handle_errors=True（默认）
)

# 错误处理流程：
# 1. LLM 返回错误格式
# 2. 系统自动生成 ToolMessage 提示错误
# 3. LLM 根据错误反馈重新生成
# 4. 重复直到成功或达到最大重试次数
```

**错误处理示例**（来自 Context7 文档）：

```typescript
// 错误处理流程示例
const result = await agent.invoke({
    messages: [
        {
        role: "user",
        content: "Extract info: John Doe (john@email.com) is organizing Tech Conference on March 15th",
        },
    ],
});

/**
 * 错误处理流程：
 * 1. 模型错误：返回多个工具调用（ContactInfo 和 EventDetails）
 * 2. 错误反馈：系统自动生成 ToolMessage 提示错误
 * 3. 模型重试：模型根据错误反馈重新生成正确的单个输出
 * 4. 成功返回：最终返回正确的结构化响应
 */
```

### 6.2 OutputParser 错误处理

**手动错误处理**：

```python
from langchain_core.output_parsers import PydanticOutputParser, OutputParserException

parser = PydanticOutputParser(pydantic_object=Person)

try:
    result = chain.invoke(input)
except OutputParserException as e:
    # 手动处理错误
    print(f"解析失败: {e}")
    print(f"原始输出: {e.llm_output}")

    # 选项1：使用 OutputFixingParser 自动修复
    from langchain.output_parsers import OutputFixingParser
    fixing_parser = OutputFixingParser.from_llm(parser=parser, llm=llm)
    result = fixing_parser.parse(e.llm_output)

    # 选项2：使用 RetryOutputParser 重试
    from langchain.output_parsers import RetryOutputParser
    retry_parser = RetryOutputParser.from_llm(parser=parser, llm=llm)
    result = retry_parser.parse_with_prompt(e.llm_output, prompt)
```

**OutputFixingParser 示例**：

```python
from langchain.output_parsers import OutputFixingParser

# 创建自动修复 Parser
fixing_parser = OutputFixingParser.from_llm(parser=parser, llm=llm)

# 自动修复格式错误
# 如果 LLM 输出: {"name": "Alice", "age": "30"}  # age 是字符串
# OutputFixingParser 会自动调用 LLM 修复为: {"name": "Alice", "age": 30}
result = fixing_parser.parse(bad_output)
```

## 7. 实战选择指南

### 7.1 决策树

```
开始
  ↓

  ↓ 是                    ↓ 否
需要自定义解析逻辑？      使用 OutputParser
  ↓ 否        ↓ 是
使用 with_    使用 OutputParser
structured_
output()
```

### 7.2 场景推荐

| 场景 | 推荐方法 | 理由 |
|------|----------|------|
| **生产环境数据提取** | with_structured_output() | 性能好、成功率高、代码简单 |
| **开源模型集成** | OutputParser | 开源模型不支持 Function Calling |
| **复杂业务规则验证** | OutputParser | 需要自定义验证逻辑 |
| **实时流式应用** | OutputParser | 完整的流式支持 |
| **快速原型开发** | with_structured_output() | 代码简单、快速实现 |
| **需要调试和监控** | OutputParser | 可见原始输出、可控错误处理 |
| **多模型兼容** | OutputParser | 适用于所有模型 |
| **成本敏感场景** | with_structured_output() | 更少 token 消耗 |

### 7.3 混合使用策略

```python
from langchain_openai import ChatOpenAI
from langchain_core.output_parsers import PydanticOutputParser

def get_structured_output(model_name: str, schema: type[BaseModel], input: str):
    """根据模型选择合适的方法"""
    llm = ChatOpenAI(model=model_name)

    # 支持原生结构化输出的模型
    if model_name in ["gpt-4.1", "gpt-4", "claude-3-opus"]:
        structured_llm = llm.with_structured_output(schema, method="json_schema")
        return structured_llm.invoke(input)

    # 不支持的模型，回退到 OutputParser
    else:
        parser = PydanticOutputParser(pydantic_object=schema)
        format_instructions = parser.get_format_instructions()
        prompt = f"{input}\n\n{format_instructions}"
        chain = llm | parser
        return chain.invoke(prompt)

# 使用
result = get_structured_output("gpt-4.1", Person, "Alice is 30")
```

## 10. 总结

**核心要点**：

1. **优先使用 with_structured_output()**：如果模型支持，这是最简单、最快速的方法
2. **回退到 OutputParser**：当模型不支持或需要自定义逻辑时使用
3. **性能 vs 灵活性**：with_structured_output() 性能更好，OutputParser 更灵活
4. **错误处理**：with_structured_output() 自动处理，OutputParser 需要手动处理
5. **流式支持**：OutputParser 提供更完整的流式支持

**一句话总结**：
> **with_structured_output() 是现代 LangChain 应用的首选方法，但 OutputParser 在需要自定义解析逻辑、支持所有模型或完整流式功能时仍然不可或缺。**
