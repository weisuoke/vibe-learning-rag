# 实战代码 - 场景4：复杂状态管理

## 场景概述

本场景演示如何在复杂的 AI Agent 系统中管理多字段状态，实现高效、可维护的状态架构。

**核心目标**：
- 掌握多字段状态设计模式
- 实现数据分离与隔离
- 避免状态膨胀问题
- 优化性能与可维护性

**适用场景**：
- ReAct Agent 系统
- 多步推理系统
- 复杂工作流管理
- 生产级 AI 应用

[来源: reference/search_annotated_github_01.md]

## 原理讲解

### 1. 状态膨胀问题

```python
# ❌ 问题：所有数据混在一起
class BadState(TypedDict):
    data: Annotated[list, operator.add]  # 包含消息、工具结果、日志等

# 问题：
# 1. 难以管理和查询
# 2. 序列化开销大
# 3. Token 浪费
# 4. 可维护性差
```

**状态膨胀的后果**：
- 每次更新都需要序列化整个状态
- LLM 上下文被无关数据占用
- 难以追踪特定类型的数据
- 性能下降

[来源: reference/search_annotated_reddit_01.md]

### 2. 数据分离原则

```python
# ✅ 解决方案：分离不同类型的数据
class GoodState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    tool_results: Annotated[list[ToolResult], operator.add]
    reasoning_steps: Annotated[list[str], operator.add]
    metadata: dict
```

**分离的优势**：
- 清晰的数据边界
- 独立的更新策略
- 选择性序列化
- 更好的性能

[来源: reference/search_annotated_github_01.md]

### 3. 状态设计模式

| 模式 | 用途 | 示例 |
|------|------|------|
| **分层状态** | 继承复用 | BaseState → AgentState → WorkflowState |
| **命名空间状态** | 逻辑隔离 | user_*, system_*, agent_* |
| **事件驱动状态** | 历史追踪 | events: list[Event] |
| **细粒度状态** | 精确控制 | 每种数据独立字段 |

[来源: reference/search_annotated_github_01.md]

## 完整代码示例

### 示例 1：ReAct Agent 状态管理

```python
"""
ReAct Agent 状态管理
演示如何分离消息、工具结果、推理步骤
"""

from typing_extensions import TypedDict, Annotated
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
import operator


# 1. 定义工具结果类型
class ToolResult(TypedDict):
    """工具调用结果"""
    tool_name: str
    input: dict
    output: str


# 2. 定义 ReAct Agent 状态
class ReActState(TypedDict):
    """
    ReAct Agent 状态
    - 分离消息、工具结果、推理步骤
    - 避免状态膨胀
    """
    # 对话消息（用于 LLM 上下文）
    messages: Annotated[list[BaseMessage], add_messages]

    # 工具调用结果（独立存储）
    tool_results: Annotated[list[ToolResult], operator.add]

    # 推理步骤（用于调试和追踪）
    reasoning_steps: Annotated[list[str], operator.add]

    # 元数据（覆盖策略）
    current_step: str
    iteration: int


# 3. 定义节点
def think_node(state: ReActState) -> dict:
    """思考节点：生成推理步骤"""
    return {
        "reasoning_steps": [f"思考: 分析问题 '{state['messages'][-1].content}'"],
        "current_step": "thinking"
    }


def act_node(state: ReActState) -> dict:
    """行动节点：调用工具"""
    tool_result: ToolResult = {
        "tool_name": "search",
        "input": {"query": "LangGraph"},
        "output": "LangGraph 是一个用于构建状态化 AI 应用的框架"
    }

    return {
        "tool_results": [tool_result],
        "reasoning_steps": [f"行动: 调用工具 {tool_result['tool_name']}"],
        "current_step": "acting"
    }


def observe_node(state: ReActState) -> dict:
    """观察节点：处理工具结果"""
    last_result = state["tool_results"][-1]

    return {
        "messages": [AIMessage(content=f"观察到: {last_result['output']}")],
        "reasoning_steps": [f"观察: 工具返回结果"],
        "current_step": "observing",
        "iteration": state.get("iteration", 0) + 1
    }


# 4. 构建图
def create_react_graph():
    """创建 ReAct 图"""
    builder = StateGraph(ReActState)

    builder.add_node("think", think_node)
    builder.add_node("act", act_node)
    builder.add_node("observe", observe_node)

    builder.add_edge(START, "think")
    builder.add_edge("think", "act")
    builder.add_edge("act", "observe")
    builder.add_edge("observe", END)

    return builder.compile()


# 5. 运行
def main():
    """主函数"""
    graph = create_react_graph()

    result = graph.invoke({
        "messages": [HumanMessage(content="什么是 LangGraph？")],
        "tool_results": [],
        "reasoning_steps": [],
        "current_step": "",
        "iteration": 0
    })

    print("=" * 60)
    print("ReAct Agent 状态管理")
    print("=" * 60)

    print(f"\n当前步骤: {result['current_step']}")
    print(f"迭代次数: {result['iteration']}")

    print(f"\n消息列表 ({len(result['messages'])} 条):")
    for msg in result["messages"]:
        role = "Human" if isinstance(msg, HumanMessage) else "AI"
        print(f"  [{role}] {msg.content}")

    print(f"\n工具结果 ({len(result['tool_results'])} 条):")
    for tr in result["tool_results"]:
        print(f"  {tr['tool_name']}: {tr['output'][:50]}...")

    print(f"\n推理步骤 ({len(result['reasoning_steps'])} 步):")
    for i, step in enumerate(result["reasoning_steps"], 1):
        print(f"  {i}. {step}")


if __name__ == "__main__":
    main()
```

**运行输出**：

```
============================================================
ReAct Agent 状态管理
============================================================

当前步骤: observing
迭代次数: 1

消息列表 (2 条):
  [Human] 什么是 LangGraph？
  [AI] 观察到: LangGraph 是一个用于构建状态化 AI 应用的框架

工具结果 (1 条):
  search: LangGraph 是一个用于构建状态化 AI 应用的框架...

推理步骤 (3 步):
  1. 思考: 分析问题 '什么是 LangGraph？'
  2. 行动: 调用工具 search
  3. 观察: 工具返回结果
```

**关键观察**：
- 消息、工具结果、推理步骤分别存储
- 每种数据有独立的更新策略
- 状态清晰，易于调试

### 示例 2：分层状态设计

```python
"""
分层状态设计
演示状态继承和复用
"""

from typing_extensions import TypedDict, Annotated
from langchain_core.messages import BaseMessage
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
import operator


# 1. 基础状态
class BaseState(TypedDict):
    """基础状态：所有状态的基类"""
    messages: Annotated[list[BaseMessage], add_messages]


# 2. Agent 状态（继承基础状态）
class AgentState(BaseState):
    """Agent 状态：添加工具相关字段"""
    tool_calls: Annotated[list[dict], operator.add]
    tool_results: Annotated[list[dict], operator.add]


# 3. Workflow 状态（继承 Agent 状态）
class WorkflowState(AgentState):
    """Workflow 状态：添加工作流字段"""
    steps: Annotated[list[str], operator.add]
    current_phase: str


# 4. 定义节点
def base_node(state: BaseState) -> dict:
    """基础节点：只使用基础状态"""
    return {"messages": []}


def agent_node(state: AgentState) -> dict:
    """Agent 节点：使用 Agent 状态"""
    return {
        "tool_calls": [{"tool": "search", "args": {}}],
        "tool_results": [{"result": "data"}]
    }


def workflow_node(state: WorkflowState) -> dict:
    """Workflow 节点：使用完整状态"""
    return {
        "steps": ["Step 1: Initialize"],
        "current_phase": "execution"
    }


# 5. 构建图
def create_layered_graph():
    """创建分层图"""
    builder = StateGraph(WorkflowState)

    builder.add_node("base", base_node)
    builder.add_node("agent", agent_node)
    builder.add_node("workflow", workflow_node)

    builder.add_edge(START, "base")
    builder.add_edge("base", "agent")
    builder.add_edge("agent", "workflow")
    builder.add_edge("workflow", END)

    return builder.compile()


def main():
    """主函数"""
    graph = create_layered_graph()

    result = graph.invoke({
        "messages": [],
        "tool_calls": [],
        "tool_results": [],
        "steps": [],
        "current_phase": ""
    })

    print("=" * 60)
    print("分层状态设计")
    print("=" * 60)
    print(f"\n当前阶段: {result['current_phase']}")
    print(f"工具调用: {len(result['tool_calls'])} 次")
    print(f"工作流步骤: {len(result['steps'])} 步")


if __name__ == "__main__":
    main()
```

### 示例 3：命名空间状态

```python
"""
命名空间状态
使用前缀分离不同命名空间的数据
"""

from typing_extensions import TypedDict, Annotated
from langchain_core.messages import BaseMessage
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
import operator


class NamespacedState(TypedDict):
    """命名空间状态"""
    # 用户命名空间
    user_messages: Annotated[list[BaseMessage], add_messages]
    user_metadata: dict

    # 系统命名空间
    system_logs: Annotated[list[str], operator.add]
    system_metrics: dict

    # Agent 命名空间
    agent_actions: Annotated[list[dict], operator.add]
    agent_state: str


def user_node(state: NamespacedState) -> dict:
    """用户节点：只更新用户命名空间"""
    return {
        "user_metadata": {"user_id": "123", "session": "abc"}
    }


def system_node(state: NamespacedState) -> dict:
    """系统节点：只更新系统命名空间"""
    return {
        "system_logs": ["System initialized"],
        "system_metrics": {"cpu": 50, "memory": 1024}
    }


def agent_node(state: NamespacedState) -> dict:
    """Agent 节点：只更新 Agent 命名空间"""
    return {
        "agent_actions": [{"action": "search", "target": "docs"}],
        "agent_state": "active"
    }


def create_namespaced_graph():
    """创建命名空间图"""
    builder = StateGraph(NamespacedState)

    builder.add_node("user", user_node)
    builder.add_node("system", system_node)
    builder.add_node("agent", agent_node)

    builder.add_edge(START, "user")
    builder.add_edge("user", "system")
    builder.add_edge("system", "agent")
    builder.add_edge("agent", END)

    return builder.compile()


def main():
    """主函数"""
    graph = create_namespaced_graph()

    result = graph.invoke({
        "user_messages": [],
        "user_metadata": {},
        "system_logs": [],
        "system_metrics": {},
        "agent_actions": [],
        "agent_state": ""
    })

    print("=" * 60)
    print("命名空间状态")
    print("=" * 60)
    print(f"\n用户元数据: {result['user_metadata']}")
    print(f"系统日志: {result['system_logs']}")
    print(f"Agent 状态: {result['agent_state']}")


if __name__ == "__main__":
    main()
```

### 示例 4：事件驱动状态

```python
"""
事件驱动状态
记录所有状态变更为事件
"""

from typing_extensions import TypedDict, Annotated
from datetime import datetime
import operator
from langgraph.graph import StateGraph, START, END


class Event(TypedDict):
    """事件定义"""
    type: str
    data: dict
    timestamp: str


class EventDrivenState(TypedDict):
    """事件驱动状态"""
    events: Annotated[list[Event], operator.add]
    current_state: str


def create_event(event_type: str, data: dict) -> Event:
    """创建事件"""
    return {
        "type": event_type,
        "data": data,
        "timestamp": datetime.now().isoformat()
    }


def init_node(state: EventDrivenState) -> dict:
    """初始化节点"""
    return {
        "events": [create_event("INIT", {"status": "started"})],
        "current_state": "initialized"
    }


def process_node(state: EventDrivenState) -> dict:
    """处理节点"""
    return {
        "events": [create_event("PROCESS", {"items": 10})],
        "current_state": "processing"
    }


def complete_node(state: EventDrivenState) -> dict:
    """完成节点"""
    return {
        "events": [create_event("COMPLETE", {"success": True})],
        "current_state": "completed"
    }


def create_event_graph():
    """创建事件图"""
    builder = StateGraph(EventDrivenState)

    builder.add_node("init", init_node)
    builder.add_node("process", process_node)
    builder.add_node("complete", complete_node)

    builder.add_edge(START, "init")
    builder.add_edge("init", "process")
    builder.add_edge("process", "complete")
    builder.add_edge("complete", END)

    return builder.compile()


def main():
    """主函数"""
    graph = create_event_graph()

    result = graph.invoke({
        "events": [],
        "current_state": ""
    })

    print("=" * 60)
    print("事件驱动状态")
    print("=" * 60)
    print(f"\n当前状态: {result['current_state']}")
    print(f"\n事件历史 ({len(result['events'])} 个事件):")
    for event in result["events"]:
        print(f"  [{event['type']}] {event['data']} @ {event['timestamp']}")


if __name__ == "__main__":
    main()
```

## 性能优化技巧

### 1. 选择性序列化

```python
class OptimizedState(TypedDict):
    # 需要持久化的数据
    messages: Annotated[list[BaseMessage], add_messages]

    # 不需要持久化的临时数据（使用 EphemeralValue）
    temp_results: list  # 不使用 Annotated，每次重新计算
```

### 2. 限制列表大小

```python
def create_limited_reducer(max_size: int):
    def reducer(old: list, new: list) -> list:
        combined = old + new
        return combined[-max_size:]
    return reducer

class State(TypedDict):
    recent_items: Annotated[list, create_limited_reducer(100)]
```

### 3. 使用 TypedDict 而非 Pydantic

```python
# ✅ 快：TypedDict（~0.1ms per update）
class State(TypedDict):
    data: Annotated[list, operator.add]

# ❌ 慢：Pydantic（~1.0ms per update）
from pydantic import BaseModel
class State(BaseModel):
    data: list
```

## 实际应用场景

### 场景：生产级 ReAct Agent

```python
"""
生产级 ReAct Agent
完整的状态管理实现
"""

from typing_extensions import TypedDict, Annotated
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages
import operator


class ProductionReActState(TypedDict):
    """生产级 ReAct Agent 状态"""
    # 核心数据
    messages: Annotated[list[BaseMessage], add_messages]
    tool_results: Annotated[list[dict], operator.add]
    reasoning_steps: Annotated[list[str], operator.add]

    # 元数据
    user_id: str
    session_id: str
    iteration: int
    max_iterations: int

    # 控制流
    should_continue: bool
    error: str | None


def create_limited_message_reducer(max_messages: int):
    """创建限制消息数量的 Reducer"""
    def reducer(old: list, new: list) -> list:
        from langgraph.graph.message import add_messages
        merged = add_messages(old, new)
        if len(merged) > max_messages:
            # 保留系统消息 + 最近的消息
            system_msgs = [m for m in merged if m.type == "system"]
            recent_msgs = [m for m in merged if m.type != "system"][-max_messages:]
            return system_msgs + recent_msgs
        return merged
    return reducer


class OptimizedState(TypedDict):
    """优化的状态：限制消息数量"""
    messages: Annotated[list[BaseMessage], create_limited_message_reducer(50)]
    tool_results: Annotated[list[dict], operator.add]
```

## 常见陷阱与解决方案

### 陷阱 1：状态膨胀

```python
# ❌ 问题：所有数据混在一起
class BadState(TypedDict):
    data: Annotated[list, operator.add]

# ✅ 解决：分离不同类型的数据
class GoodState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    tool_results: Annotated[list[dict], operator.add]
    logs: Annotated[list[str], operator.add]
```

### 陷阱 2：无限增长

```python
# ❌ 问题：列表无限增长
class BadState(TypedDict):
    items: Annotated[list, operator.add]

# ✅ 解决：限制大小
def limited_add(max_size: int):
    def reducer(old: list, new: list) -> list:
        return (old + new)[-max_size:]
    return reducer

class GoodState(TypedDict):
    items: Annotated[list, limited_add(100)]
```

### 陷阱 3：过度使用 Pydantic

```python
# ❌ 慢：Pydantic 验证开销
from pydantic import BaseModel
class State(BaseModel):
    data: list

# ✅ 快：TypedDict
class State(TypedDict):
    data: Annotated[list, operator.add]
```

## 总结

### 核心要点

1. **数据分离**：不同类型的数据使用独立字段
2. **限制大小**：防止状态无限增长
3. **命名空间**：使用前缀组织相关数据
4. **性能优化**：使用 TypedDict、限制列表大小
5. **事件驱动**：记录状态变更历史

### 设计模式

- 分层状态：继承复用
- 命名空间状态：逻辑隔离
- 事件驱动状态：历史追踪
- 细粒度状态：精确控制

### 最佳实践

- 分离消息、工具结果、推理步骤
- 限制列表最大长度
- 使用 TypedDict 而非 Pydantic
- 选择性序列化
- 定期清理历史数据
