# 30字核心

**多向量检索通过多个向量字段并行检索，使用 Ranker 融合结果，实现多模态和混合语义检索。**

---

## 核心要素拆解

### 1. 多向量检索是什么？

**多个向量字段的并行检索**
- 一个 Collection 可以包含多个向量字段（稠密向量、稀疏向量、二值向量等）
- 每个向量字段独立配置维度和索引类型
- 支持同时检索多个向量字段并融合结果

### 2. 多向量检索的核心组件

**AnnSearchRequest + Ranker**
- `AnnSearchRequest`：定义单个向量字段的检索请求（查询向量、字段名、检索参数）
- `RRFRanker`：使用 Reciprocal Rank Fusion 算法自动平衡多个检索结果
- `WeightedRanker`：使用加权求和方式精确控制各向量字段的重要性

### 3. 多向量检索的应用场景

**多模态与混合语义检索**
- 多模态检索：图片 + 文本混合检索（电商、内容平台）
- 全文搜索增强：BM25 稀疏向量 + 稠密向量混合检索（文档问答）
- 多语言检索：中文向量 + 英文向量混合检索（跨语言搜索）
- 多粒度检索：标题向量 + 内容向量混合检索（精准匹配）

---

## 与其他概念的关系

### 多向量检索 vs 单向量检索

| 维度 | 单向量检索 | 多向量检索 |
|------|-----------|-----------|
| 向量字段数量 | 1 个 | 多个（2-N 个） |
| 检索方式 | 单一向量检索 | 并行检索 + 结果融合 |
| 应用场景 | 单一语义检索 | 多模态、混合语义检索 |
| API | `search()` | `hybrid_search()` |

### RRFRanker vs WeightedRanker

| 维度 | RRFRanker | WeightedRanker |
|------|-----------|----------------|
| 融合算法 | Reciprocal Rank Fusion | 加权求和 |
| 权重配置 | 自动平衡（k=60） | 手动指定权重 |
| 适用场景 | 不确定权重的场景 | 已知权重的场景 |
| 参数调整 | 调整 k 参数 | 调整各字段权重 |

---

## 一句话记忆法

**多向量检索 = 多个向量字段 + 并行检索 + Ranker 融合**

## 快速验证

```python
from pymilvus import connections, Collection, AnnSearchRequest, RRFRanker

# 连接 Milvus
connections.connect("default", host="localhost", port="19530")

# 假设已有包含多个向量字段的 Collection
collection = Collection("multi_vector_collection")

# 定义稠密向量检索请求
dense_req = AnnSearchRequest(
    data=[query_dense_embedding],      # 查询向量
    anns_field="dense_vector",         # 向量字段名
    param={"metric_type": "IP"},       # 检索参数
    limit=10                           # 返回结果数量
)

# 定义稀疏向量检索请求
sparse_req = AnnSearchRequest(
    data=[query_sparse_embedding],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 使用 RRF 融合结果
results = collection.hybrid_search(
    reqs=[dense_req, sparse_req],      # 多个检索请求
    ranker=RRFRanker(),                # RRF 排序器
    limit=10,                          # 最终返回数量
    output_fields=["text", "metadata"]
)

# 使用加权融合（可选）
from pymilvus import WeightedRanker
results = collection.hybrid_search(
    reqs=[sparse_req, dense_req],
    ranker=WeightedRanker(0.7, 1.0),   # 稀疏权重0.7，稠密权重1.0
    limit=10
)
```

## 核心价值

1. **多模态检索**：支持图片 + 文本等多模态数据的混合检索
2. **检索质量提升**：BM25 稀疏向量 + 稠密向量兼顾语义理解和精确匹配
3. **灵活融合策略**：RRFRanker 自动平衡，WeightedRanker 精确控制
4. **索引独立管理**：每个向量字段独立配置索引类型和参数

---

## 关键约束

1. **索引要求**：所有向量字段都必须创建索引才能加载 Collection
2. **数据完整性**：插入数据时需要为所有向量字段提供数据
3. **维度独立**：不同向量字段可以有不同的维度和类型
4. **Ranker 选择**：根据场景选择合适的 Ranker（RRF 或 Weighted）
