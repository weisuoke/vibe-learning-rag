# 实战代码 场景2：函数调用完整流程

> 演示 bind_tools → 模型调用 → 解析 tool_calls → 执行工具 → ToolMessage 回传 → 最终回答的完整链路，覆盖单工具、多工具并行、tool_choice 控制和通用循环封装

---

## 场景概述

场景1 聚焦 Tool 的定义和直接调用，不涉及 LLM。本场景进入**真正的函数调用流程**：把工具绑定到模型，让 LLM 自己决定调不调、调哪个、传什么参数，然后我们执行工具、把结果喂回去，LLM 再给出最终回答。

这就是 OpenAI Function Calling / Tool Use 的核心机制，也是构建 Agent 的基础。

每个示例都是完整可运行的 Python 代码，复制即用。

## 示例1：单工具调用完整流程（逐步拆解）

**解决的问题：** 从零走通一次完整的函数调用——发消息、拿到 tool_calls、执行工具、构造 ToolMessage、发回模型、拿到最终回答。每一步都打印中间状态，让你看清整个数据流。

```python
"""
单工具调用完整流程：逐步拆解每个环节
目标：理解 HumanMessage → AIMessage(tool_calls) → ToolMessage → AIMessage(content) 的完整链路
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

load_dotenv()

# ===== 第一步：定义工具 =====
@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气信息。"""
    # 模拟天气数据（实际项目中调用天气 API）
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云，28°C，湿度 65%",
        "深圳": "阵雨，30°C，湿度 80%",
    }
    return weather_data.get(city, f"{city}：暂无天气数据")


@tool
def calculate(expression: str) -> str:
    """计算数学表达式并返回结果。"""
    try:
        result = eval(expression)  # 示例用，生产环境应使用安全解析器
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


# ===== 第二步：绑定工具到模型 =====
model = ChatOpenAI(model="gpt-4o-mini")
model_with_tools = model.bind_tools([get_weather, calculate])

# ===== 第三步：发送消息，获取模型响应 =====
messages = [HumanMessage(content="北京今天天气怎么样？")]
print(">>> 用户消息:", messages[0].content)

ai_response = model_with_tools.invoke(messages)
print("\n>>> 模型响应类型:", type(ai_response).__name__)
print(">>> content:", repr(ai_response.content))
print(">>> tool_calls:", ai_response.tool_calls)

# ===== 第四步：检查是否有工具调用 =====
if ai_response.tool_calls:
    tc = ai_response.tool_calls[0]
    print(f"\n>>> 模型要求调用工具:")
    print(f"    名称: {tc['name']}")
    print(f"    参数: {tc['args']}")
    print(f"    ID:   {tc['id']}")

    # ===== 第五步：执行工具 =====
    # 构建工具名称到工具对象的映射
    tools_map = {"get_weather": get_weather, "calculate": calculate}
    tool_result = tools_map[tc["name"]].invoke(tc["args"])
    print(f"\n>>> 工具执行结果: {tool_result}")

    # ===== 第六步：构造 ToolMessage =====
    # tool_call_id 必须与 AIMessage 中的 tc["id"] 对应
    tool_msg = ToolMessage(content=tool_result, tool_call_id=tc["id"])
    print(f"\n>>> ToolMessage:")
    print(f"    content: {tool_msg.content}")
    print(f"    tool_call_id: {tool_msg.tool_call_id}")

    # ===== 第七步：把完整对话历史发回模型 =====
    messages.append(ai_response)  # AIMessage（包含 tool_calls）
    messages.append(tool_msg)     # ToolMessage（工具执行结果）

    print(f"\n>>> 当前消息列表（{len(messages)} 条）:")
    for i, msg in enumerate(messages):
        print(f"    [{i}] {type(msg).__name__}: {msg.content[:50]}...")

    final_response = model_with_tools.invoke(messages)
    print(f"\n>>> 最终回答: {final_response.content}")
else:
    # 模型认为不需要调用工具，直接回答
    print(f"\n>>> 直接回答: {ai_response.content}")
```

## 示例2：多工具并行调用

**解决的问题：** 用户一句话涉及多个工具时，LLM 会在一次响应中返回多个 tool_calls。我们需要全部执行，把所有 ToolMessage 一起发回去。

```python
"""
多工具并行调用：一个问题触发多个工具
LLM 在一次响应中返回多个 tool_calls，我们全部执行后一起发回
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

load_dotenv()


@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气信息。"""
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云，28°C，湿度 65%",
        "深圳": "阵雨，30°C，湿度 80%",
    }
    return weather_data.get(city, f"{city}：暂无天气数据")


@tool
def calculate(expression: str) -> str:
    """计算数学表达式并返回结果。"""
    try:
        result = eval(expression)
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


model = ChatOpenAI(model="gpt-4o-mini")
model_with_tools = model.bind_tools([get_weather, calculate])

# 一个问题同时需要天气和计算
messages = [HumanMessage(content="北京和上海今天天气怎么样？另外帮我算一下 125 * 8 + 320")]
print(">>> 用户消息:", messages[0].content)

ai_response = model_with_tools.invoke(messages)
print(f"\n>>> 模型返回了 {len(ai_response.tool_calls)} 个工具调用:")
for i, tc in enumerate(ai_response.tool_calls):
    print(f"    [{i}] {tc['name']}({tc['args']})")

# 构建工具映射
tools_map = {"get_weather": get_weather, "calculate": calculate}

# 执行所有工具调用，收集 ToolMessage
messages.append(ai_response)

for tc in ai_response.tool_calls:
    result = tools_map[tc["name"]].invoke(tc["args"])
    print(f"\n>>> 执行 {tc['name']}({tc['args']}) → {result}")
    messages.append(ToolMessage(content=str(result), tool_call_id=tc["id"]))

# 所有 ToolMessage 发回模型
print(f"\n>>> 消息列表共 {len(messages)} 条，发回模型...")
final_response = model_with_tools.invoke(messages)
print(f"\n>>> 最终回答:\n{final_response.content}")
```

### 预期输出

```
>>> 用户消息: 北京和上海今天天气怎么样？另外帮我算一下 125 * 8 + 320

>>> 模型返回了 3 个工具调用:
    [0] get_weather({'city': '北京'})
    [1] get_weather({'city': '上海'})
    [2] calculate({'expression': '125 * 8 + 320'})

>>> 执行 get_weather({'city': '北京'}) → 晴，25°C，湿度 40%

>>> 执行 get_weather({'city': '上海'}) → 多云，28°C，湿度 65%

>>> 执行 calculate({'expression': '125 * 8 + 320'}) → 1320

>>> 消息列表共 6 条，发回模型...

>>> 最终回答:
以下是查询结果：

1. **北京天气**：晴天，气温 25°C，湿度 40%
2. **上海天气**：多云，气温 28°C，湿度 65%
3. **计算结果**：125 × 8 + 320 = 1320
```

### 并行调用的消息结构

```
messages 列表最终状态（6 条消息）：

[0] HumanMessage:  "北京和上海今天天气怎么样？另外帮我算一下..."
[1] AIMessage:     tool_calls: [get_weather(北京), get_weather(上海), calculate(125*8+320)]
[2] ToolMessage:   "晴，25°C，湿度 40%"          (tool_call_id → 对应北京)
[3] ToolMessage:   "多云，28°C，湿度 65%"         (tool_call_id → 对应上海)
[4] ToolMessage:   "1320"                         (tool_call_id → 对应计算)
[5] AIMessage:     "以下是查询结果：..."           (最终回答)
```

> **关键记忆点：** 多个 tool_calls 对应多个 ToolMessage，通过 `tool_call_id` 一一配对。所有 ToolMessage 必须在同一轮发回，不能只发一部分。

## 示例3：tool_choice 参数控制工具选择策略

**解决的问题：** 默认情况下 LLM 自己决定调不调工具（auto）。但有时你需要强制调用、或指定调用某个工具。`tool_choice` 参数提供了这种控制能力。

```python
"""
tool_choice 参数演示：控制 LLM 的工具选择行为
三种模式：auto（默认）、any（强制调用）、指定工具
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage

load_dotenv()


@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气信息。"""
    return f"{city}：晴，25°C"


@tool
def calculate(expression: str) -> str:
    """计算数学表达式并返回结果。"""
    return str(eval(expression))


model = ChatOpenAI(model="gpt-4o-mini")
tools = [get_weather, calculate]

# ========================================
# 模式1：tool_choice="auto"（默认）
# LLM 自己决定是否调用工具
# ========================================
print("=" * 50)
print("模式1: auto（默认）— LLM 自行决定")
print("=" * 50)

model_auto = model.bind_tools(tools, tool_choice="auto")

# 不需要工具的问题 → LLM 直接回答
resp1 = model_auto.invoke([HumanMessage(content="你好，你是谁？")])
print(f"问题: '你好，你是谁？'")
print(f"tool_calls: {resp1.tool_calls}")
print(f"content: {resp1.content[:60]}...")

# 需要工具的问题 → LLM 调用工具
resp2 = model_auto.invoke([HumanMessage(content="北京天气怎么样？")])
print(f"\n问题: '北京天气怎么样？'")
print(f"tool_calls: {resp2.tool_calls}")

# ========================================
# 模式2：tool_choice="any"（强制调用）
# LLM 必须调用至少一个工具，不能直接回答
# ========================================
print(f"\n{'=' * 50}")
print("模式2: any — 强制调用工具")
print("=" * 50)

model_any = model.bind_tools(tools, tool_choice="any")

# 即使不需要工具，也会被强制调用
resp3 = model_any.invoke([HumanMessage(content="你好，你是谁？")])
print(f"问题: '你好，你是谁？'")
print(f"tool_calls: {resp3.tool_calls}")
print(f"content: {repr(resp3.content)}")

# ========================================
# 模式3：指定工具名称（强制调用特定工具）
# ========================================
print(f"\n{'=' * 50}")
print("模式3: 指定工具 — 强制调用 get_weather")
print("=" * 50)

model_forced = model.bind_tools(tools, tool_choice="get_weather")

# 即使问的是计算问题，也会被强制调用 get_weather
resp4 = model_forced.invoke([HumanMessage(content="帮我算 1+1")])
print(f"问题: '帮我算 1+1'")
print(f"tool_calls: {resp4.tool_calls}")
```

### 预期输出

```
==================================================
模式1: auto（默认）— LLM 自行决定
==================================================
问题: '你好，你是谁？'
tool_calls: []
content: 你好！我是一个 AI 助手，可以帮你查天气和做计算。有什么...

问题: '北京天气怎么样？'
tool_calls: [{'name': 'get_weather', 'args': {'city': '北京'}, 'id': 'call_xxx'}]

==================================================
模式2: any — 强制调用工具
==================================================
问题: '你好，你是谁？'
tool_calls: [{'name': 'get_weather', 'args': {'city': '北京'}, 'id': 'call_yyy'}]
content: ''

==================================================
模式3: 指定工具 — 强制调用 get_weather
==================================================
问题: '帮我算 1+1'
tool_calls: [{'name': 'get_weather', 'args': {'city': '...'}, 'id': 'call_zzz'}]
```

### tool_choice 速查表

```
tool_choice 值          │ 行为                          │ 使用场景
────────────────────────┼───────────────────────────────┼──────────────────────
"auto"（默认）          │ LLM 自行决定是否调用          │ 通用对话 Agent
"any"                   │ 必须调用至少一个工具          │ 确保每次都执行操作
"none"                  │ 禁止调用任何工具              │ 纯文本回答场景
```

## 示例4：不需要工具时的处理

**解决的问题：** 不是每个问题都需要工具。当 LLM 判断可以直接回答时，`tool_calls` 为空列表，`content` 包含回答文本。你的代码必须正确处理这两种情况。

```python
"""
处理"不需要工具"的情况
LLM 可能直接回答，也可能调用工具——代码必须兼容两种路径
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

load_dotenv()


@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气信息。"""
    return f"{city}：晴，25°C"


model = ChatOpenAI(model="gpt-4o-mini")
model_with_tools = model.bind_tools([get_weather])
tools_map = {"get_weather": get_weather}

# 测试两种问题
questions = [
    "你好，请做个自我介绍",     # 不需要工具
    "深圳今天天气怎么样？",     # 需要工具
]

for question in questions:
    print(f"\n{'=' * 50}")
    print(f"问题: {question}")
    messages = [HumanMessage(content=question)]
    response = model_with_tools.invoke(messages)

    if response.tool_calls:
        # 路径 A：需要工具
        print(f"→ 走工具路径，调用 {len(response.tool_calls)} 个工具")
        messages.append(response)
        for tc in response.tool_calls:
            result = tools_map[tc["name"]].invoke(tc["args"])
            messages.append(ToolMessage(content=str(result), tool_call_id=tc["id"]))
        final = model_with_tools.invoke(messages)
        print(f"→ 最终回答: {final.content}")
    else:
        # 路径 B：直接回答
        print(f"→ 走直接回答路径")
        print(f"→ 回答: {response.content}")
```

### 预期输出

```
==================================================
问题: 你好，请做个自我介绍
→ 走直接回答路径
→ 回答: 你好！我是一个 AI 助手，可以帮你查询天气信息。有什么需要帮忙的吗？

==================================================
问题: 深圳今天天气怎么样？
→ 走工具路径，调用 1 个工具
→ 最终回答: 深圳今天的天气是晴天，气温 25°C。
```

> **关键记忆点：** 永远先检查 `response.tool_calls` 是否为空。这是区分"需要工具"和"直接回答"的唯一判据。

## 示例5：完整的循环调用封装（通用 Agent Loop）

**解决的问题：** 前面的示例都是手动处理一轮工具调用。但实际场景中，LLM 可能需要多轮工具调用才能回答一个问题（比如先查天气，再根据天气做计算）。这个封装实现了通用的循环调用逻辑。

```python
"""
通用 Agent Loop：自动处理多轮工具调用
支持：单工具、多工具并行、多轮迭代、最大轮次限制
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

load_dotenv()


# ===== 定义工具 =====
@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气信息。"""
    weather_data = {
        "北京": "晴，25°C，湿度 40%",
        "上海": "多云，28°C，湿度 65%",
        "深圳": "阵雨，30°C，湿度 80%",
        "广州": "晴，32°C，湿度 55%",
    }
    return weather_data.get(city, f"{city}：暂无天气数据")


@tool
def calculate(expression: str) -> str:
    """计算数学表达式并返回结果。支持加减乘除和括号。"""
    try:
        result = eval(expression)
        return str(result)
    except Exception as e:
        return f"计算错误: {e}"


# ===== 核心：通用循环调用函数 =====
def run_agent_loop(
    model_with_tools,
    messages: list,
    tools_map: dict,
    max_iterations: int = 5,
    verbose: bool = True,
) -> str:
    """
    通用的工具调用循环。

    Args:
        model_with_tools: 绑定了工具的模型
        messages: 初始消息列表
        tools_map: 工具名称 → 工具对象的映射
        max_iterations: 最大迭代次数（防止无限循环）
        verbose: 是否打印中间过程

    Returns:
        模型的最终文本回答
    """
    for i in range(max_iterations):
        if verbose:
            print(f"\n--- 第 {i + 1} 轮 ---")

        # 调用模型
        response = model_with_tools.invoke(messages)
        messages.append(response)

        # 检查是否需要调用工具
        if not response.tool_calls:
            if verbose:
                print(f"模型直接回答，循环结束")
            return response.content

        # 执行所有工具调用
        if verbose:
            print(f"模型请求调用 {len(response.tool_calls)} 个工具:")

        for tc in response.tool_calls:
            tool_name = tc["name"]
            tool_args = tc["args"]

            if verbose:
                print(f"  → {tool_name}({tool_args})")

            if tool_name not in tools_map:
                result = f"错误：未知工具 '{tool_name}'"
            else:
                result = tools_map[tool_name].invoke(tool_args)

            if verbose:
                print(f"    结果: {result}")

            messages.append(
                ToolMessage(content=str(result), tool_call_id=tc["id"])
            )

    return "达到最大迭代次数，未能获得最终回答"


# ===== 使用示例 =====
model = ChatOpenAI(model="gpt-4o-mini")
tools = [get_weather, calculate]
model_with_tools = model.bind_tools(tools)
tools_map = {t.name: t for t in tools}

# 测试1：单工具调用
print("=" * 60)
print("测试1：单工具调用")
print("=" * 60)

answer1 = run_agent_loop(model_with_tools, messages1, tools_map)
print(f"\n最终回答: {answer1}")

# 测试2：多工具并行调用
print(f"\n{'=' * 60}")
print("测试2：多工具并行调用")
print("=" * 60)

answer2 = run_agent_loop(model_with_tools, messages2, tools_map)
print(f"\n最终回答: {answer2}")

# 测试3：不需要工具
print(f"\n{'=' * 60}")
print("测试3：不需要工具")
print("=" * 60)
messages3 = [HumanMessage(content="你好，你能做什么？")]
answer3 = run_agent_loop(model_with_tools, messages3, tools_map)
print(f"\n最终回答: {answer3}")
```

### run_agent_loop 流程图

```
run_agent_loop(messages, max_iterations=5)
    │
    ├─ 第 1 轮 ─────────────────────────────────────────
    │   model.invoke(messages)
    │       │
    │       ├─ tool_calls 不为空？
    │       │   ├─ YES → 执行所有工具 → 追加 ToolMessage → 继续循环
    │       │   └─ NO  → return response.content（结束）
    │       │
    ├─ 第 2 轮 ─────────────────────────────────────────
    │   model.invoke(messages)  ← 包含上一轮的 ToolMessage
    │       │
    │       ├─ tool_calls 不为空？
    │       │   ├─ YES → 执行所有工具 → 追加 ToolMessage → 继续循环
    │       │   └─ NO  → return response.content（结束）
    │       │
    ├─ ... 最多 max_iterations 轮 ...
    │
    └─ 超过最大轮次 → return "达到最大迭代次数"
```

## 示例6：错误处理与边界情况

**解决的问题：** 工具执行可能失败（网络超时、参数错误、未知工具）。把错误信息作为 ToolMessage 返回给 LLM，让它自行决定如何处理——重试、换个参数、或告诉用户出错了。

```python
"""
错误处理：工具执行失败时的正确做法
核心原则：把错误信息作为 ToolMessage 返回给 LLM，让它自行处理
"""
import os
from dotenv import load_dotenv
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langchain_core.messages import HumanMessage, ToolMessage

load_dotenv()


@tool
def get_weather(city: str) -> str:
    """获取指定城市的当前天气信息。"""
    supported = {"北京", "上海", "深圳"}
    if city not in supported:
        raise ValueError(f"不支持的城市: {city}，目前仅支持: {', '.join(supported)}")
    return f"{city}：晴，25°C"


@tool
def calculate(expression: str) -> str:
    """计算数学表达式并返回结果。"""
    result = eval(expression)
    return str(result)


def run_agent_loop_safe(model_with_tools, messages, tools_map, max_iterations=5):
    """带错误处理的 Agent Loop"""
    for i in range(max_iterations):
        response = model_with_tools.invoke(messages)
        messages.append(response)

        if not response.tool_calls:
            return response.content

        for tc in response.tool_calls:
            tool_name = tc["name"]
            tool_args = tc["args"]

            # 错误处理：未知工具
            if tool_name not in tools_map:
                error_msg = f"错误：工具 '{tool_name}' 不存在"
                messages.append(ToolMessage(content=error_msg, tool_call_id=tc["id"]))
                continue

            # 错误处理：工具执行异常
            try:
                result = tools_map[tool_name].invoke(tool_args)
                messages.append(
                    ToolMessage(content=str(result), tool_call_id=tc["id"])
                )
            except Exception as e:
                error_msg = f"工具执行失败: {type(e).__name__}: {e}"
                print(f"  [错误] {tool_name}({tool_args}) → {error_msg}")
                messages.append(
                    ToolMessage(content=error_msg, tool_call_id=tc["id"])
                )

    return "达到最大迭代次数"


# ===== 测试：查询不支持的城市 =====
model = ChatOpenAI(model="gpt-4o-mini")
tools = [get_weather, calculate]
model_with_tools = model.bind_tools(tools)
tools_map = {t.name: t for t in tools}

messages = [HumanMessage(content="成都今天天气怎么样？")]
print("问题:", messages[0].content)
answer = run_agent_loop_safe(model_with_tools, messages, tools_map)
print(f"回答: {answer}")
```

### 预期输出

```
问题: 成都今天天气怎么样？
  [错误] get_weather({'city': '成都'}) → 工具执行失败: ValueError: 不支持的城市: 成都，目前仅支持: 北京, 上海, 深圳
回答: 抱歉，目前我只能查询北京、上海和深圳的天气信息，暂不支持成都。您可以尝试查询这三个城市的天气。
```

> **关键记忆点：** 永远不要因为工具报错就中断整个流程。把错误信息作为 ToolMessage 发回给 LLM，它通常能给出合理的兜底回答。这比直接抛异常给用户好得多。

## 完整流程速查

```
函数调用完整流程（5 步）：

1. 定义工具        @tool / StructuredTool / BaseTool
2. 绑定到模型      model.bind_tools([tools], tool_choice="auto")
3. 调用模型        response = model.invoke(messages)
4. 执行工具        result = tools_map[tc["name"]].invoke(tc["args"])
5. 回传结果        ToolMessage(content=result, tool_call_id=tc["id"])

循环 3-5 直到 response.tool_calls 为空
```

### 常见错误排查

| 错误现象 | 原因 | 解决方案 |
|----------|------|----------|
| `tool_call_id` 不匹配 | ToolMessage 的 id 与 AIMessage 中的不一致 | 用 `tc["id"]` 而不是自己生成 |
| 模型不调用工具 | Tool 描述不清晰，LLM 不知道该用 | 改善 docstring 描述 |
| 模型总是调用工具 | 使用了 `tool_choice="any"` | 改为 `"auto"` 或不传 |
| 多工具只返回部分结果 | 只发了部分 ToolMessage | 确保每个 tool_call 都有对应的 ToolMessage |
| 无限循环 | LLM 反复调用工具 | 设置 `max_iterations` 上限 |

## 关键要点总结

1. **消息流是核心**：HumanMessage → AIMessage(tool_calls) → ToolMessage → AIMessage(content)，理解这个流程就理解了函数调用的本质。
2. **模型不执行工具**：模型只输出"我想调用什么"，执行是你的代码做的。这是安全边界。
3. **tool_call_id 是配对键**：每个 ToolMessage 必须通过 `tool_call_id` 与对应的 tool_call 配对，否则模型无法关联请求和结果。
4. **错误也是 ToolMessage**：工具报错时，把错误信息作为 ToolMessage 发回，让 LLM 自行处理。
5. **Agent Loop 是 Agent 的基础**：`run_agent_loop` 就是最简 Agent——循环调用模型和工具直到得到最终回答。
