# PromptTemplate高级用法 - 第一性原理

## 什么是第一性原理?

**第一性原理**：回到事物最基本的真理,从源头思考问题,而不是通过类比或经验来推理。

在物理学中,第一性原理意味着从最基本的物理定律出发推导现象。在软件工程中,第一性原理意味着从最基本的需求和约束出发,重新思考解决方案。

## PromptTemplate高级用法的第一性原理

### 1. 最基础的定义

**PromptTemplate高级用法 = 模板的可组合性 + 变量的可预填充性**

仅此而已!没有更基础的了。

让我们拆解这个定义:

- **模板的可组合性**: 将多个小模板组合成大模板的能力
- **变量的可预填充性**: 提前设置某些变量值,避免每次都传递的能力

这两个特性解决了模板复用和维护的根本问题。

### 2. 为什么需要PromptTemplate高级用法?

**核心问题**: 在实际的 AI Agent 开发中,我们面临三个根本性挑战:

#### 挑战1: 模板重复与维护困难

```python
# ❌ 没有高级用法时的困境
prompt1 = PromptTemplate.from_template(
    "You are a helpful assistant. Today is 2026-02-26. User: {query}"
)

prompt2 = PromptTemplate.from_template(
    "You are a helpful assistant. Today is 2026-02-26. Translate: {text}"
)

prompt3 = PromptTemplate.from_template(
    "You are a helpful assistant. Today is 2026-02-26. Summarize: {content}"
)
```

**问题**:
- 系统提示词重复了3次
- 日期重复了3次
- 如果要修改系统提示词,需要改3个地方
- 如果要更新日期,需要改3个地方

#### 挑战2: 动态值的重复计算

```python
# ❌ 每次调用都要重新计算日期
from datetime import datetime

def create_prompt(query: str) -> str:
    current_date = datetime.now().strftime("%Y-%m-%d")  # 每次都计算
    return f"Today is {current_date}. User: {query}"

# 调用100次就计算100次日期
for i in range(100):
    prompt = create_prompt(f"Query {i}")
```

**问题**:
- 相同的计算重复执行
- 性能浪费
- 代码冗余

#### 挑战3: 模板层次结构难以管理

```python
# ❌ 扁平化的模板难以维护
long_prompt = """
System: You are a helpful assistant.
Context: {context}
History: {history}
Tools: {tools}
User: {query}
"""
```

**问题**:
- 所有内容混在一起
- 难以复用部分内容
- 难以测试单个部分
- 难以根据场景动态组合

### 3. PromptTemplate高级用法的三层价值

#### 价值1: 模板复用 - DRY原则的体现

**DRY (Don't Repeat Yourself)** 是软件工程的基本原则。

```python
# ✅ 使用模板组合实现复用
from langchain_core.prompts import PromptTemplate

# 定义可复用的基础模板
system_template = PromptTemplate.from_template(
    "You are a helpful assistant."
)

date_template = PromptTemplate.from_template(
    "Today is {date}."
)

# 组合成不同的完整模板
query_prompt = system_template + date_template + PromptTemplate.from_template(
    "User: {query}"
)

translate_prompt = system_template + date_template + PromptTemplate.from_template(
    "Translate: {text}"
)
```

**价值体现**:
- 系统提示词只定义一次
- 日期模板只定义一次
- 修改一处,所有地方生效
- 符合单一职责原则

[来源: reference/source_prompttemplate_01.md | LangChain源码分析]

#### 价值2: 变量预填充 - 减少认知负担

**认知负担**: 每次使用模板时需要记住和传递的变量数量。

```python
# ✅ 使用partial_variables预填充
from datetime import datetime

def get_current_date():
    return datetime.now().strftime("%Y-%m-%d")

prompt = PromptTemplate(
    template="Today is {date}. User: {query}",
    input_variables=["query"],
    partial_variables={"date": get_current_date}  # 预填充日期
)

# 使用时只需要传递query,不需要关心date
result = prompt.format(query="What's the weather?")
# 输出: "Today is 2026-02-26. User: What's the weather?"
```

**价值体现**:
- 使用者只需关心核心变量 (query)
- 系统变量 (date) 自动处理
- 降低使用门槛
- 减少出错可能

[来源: reference/search_partial_01.md | LangChain社区实践]

#### 价值3: 延迟计算 - 性能优化

**延迟计算 (Lazy Evaluation)**: 只在真正需要时才计算值。

```python
# ✅ 函数形式的partial_variables实现延迟计算
def expensive_computation():
    # 模拟耗时计算
    import time
    time.sleep(1)
    return "computed_value"

# 创建模板时不会执行expensive_computation
prompt = PromptTemplate(
    template="Result: {result}, Query: {query}",
    input_variables=["query"],
    partial_variables={"result": expensive_computation}  # 传递函数,不是函数调用结果
)

# 只有在format时才执行expensive_computation
output = prompt.format(query="test")  # 此时才计算
```

**价值体现**:
- 避免不必要的计算
- 提升应用启动速度
- 支持动态值 (如当前时间)
- 符合按需加载原则

### 4. 从第一性原理推导AI Agent开发

让我们从最基本的需求出发,推导为什么AI Agent开发需要PromptTemplate高级用法:

**推理链:**

```
1. AI Agent需要与LLM交互
   ↓
2. 交互需要构造Prompt
   ↓
3. Prompt包含固定部分(系统提示)和动态部分(用户输入)
   ↓
4. 固定部分在多个场景中重复出现
   ↓
5. 需要一种机制来复用固定部分 → 模板组合
   ↓
6. 某些动态部分(如日期)在同一会话中保持不变
   ↓
7. 需要一种机制来预填充这些变量 → partial_variables
   ↓
8. 某些值需要在使用时才计算(如当前时间)
   ↓
9. 需要支持函数形式的预填充 → 延迟计算
   ↓
10. 最终形成: PromptTemplate高级用法 = 组合 + 预填充 + 延迟计算
```

**实际应用场景推导:**

#### 场景1: RAG系统

```python
# RAG系统的Prompt结构
# 基础层: 系统角色定义
system_prompt = PromptTemplate.from_template(
    "You are a helpful AI assistant with access to a knowledge base."
)

# 上下文层: 检索到的文档
context_prompt = PromptTemplate.from_template(
    "\n\nRelevant documents:\n{context}"
)

# 交互层: 用户问题
user_prompt = PromptTemplate.from_template(
    "\n\nUser question: {question}"
)

# 组合成完整的RAG Prompt
rag_prompt = system_prompt + context_prompt + user_prompt
```

**推导逻辑**:
- RAG需要注入检索到的上下文 → 需要模板组合
- 系统角色在所有查询中相同 → 可以复用
- 上下文和问题是动态的 → 需要变量占位符

[来源: reference/search_composition_01.md | LangChain社区实践]

#### 场景2: 多语言Agent

```python
# 多语言Agent的Prompt结构
from datetime import datetime

def get_current_time():
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

# 预填充当前时间和语言设置
multilang_prompt = PromptTemplate(
    template="""
System: You are a multilingual assistant.
Current time: {current_time}
Language: {language}
User: {query}
""",
    input_variables=["query"],
    partial_variables={
        "current_time": get_current_time,  # 动态计算
        "language": "English"  # 固定值
    }
)

# 使用时只需传递query
response = multilang_prompt.format(query="Hello")
```

**推导逻辑**:
- 时间需要实时获取 → 函数形式的partial
- 语言设置在会话中固定 → 字符串形式的partial
- 用户输入是唯一的动态变量 → input_variables

#### 场景3: Agent工具调用

```python
# Agent工具调用的Prompt结构
# 工具描述模板(可复用)
tools_template = PromptTemplate.from_template(

    "\n\nThink step by step about how to answer the question."
```

**推导逻辑**:
- Agent需要知道可用工具 → 需要工具描述模板
- 推理模式在所有任务中相同 → 可以复用推理模板
- 不同Agent可能有不同的系统提示 → 需要模块化组合

## 与传统方法的对比

### 传统方法: 字符串拼接

```python
# ❌ 传统方法
def create_prompt(query: str, context: str) -> str:
    system = "You are a helpful assistant."
    date = datetime.now().strftime("%Y-%m-%d")
    return f"{system}\nToday is {date}.\nContext: {context}\nUser: {query}"
```

**问题**:
- 所有逻辑混在一起
- 难以测试单个部分
- 难以复用
- 难以维护

### 高级方法: 模板组合 + 预填充

```python
# ✅ 高级方法
from langchain_core.prompts import PromptTemplate
from datetime import datetime

def get_date():
    return datetime.now().strftime("%Y-%m-%d")

# 模块化定义
system_template = PromptTemplate.from_template("You are a helpful assistant.")
date_template = PromptTemplate(
    template="Today is {date}.",
    input_variables=[],
    partial_variables={"date": get_date}
)
context_template = PromptTemplate.from_template("Context: {context}")
user_template = PromptTemplate.from_template("User: {query}")

# 组合
full_prompt = system_template + date_template + context_template + user_template

# 使用
result = full_prompt.format(context="...", query="...")
```

**优势**:
- 每个部分独立定义和测试
- 可以灵活组合
- 易于维护和扩展
- 符合SOLID原则

## 核心设计原则

PromptTemplate高级用法体现了以下软件工程原则:

### 1. 单一职责原则 (SRP)

每个模板只负责一个职责:
- 系统提示模板只定义角色
- 日期模板只处理时间
- 上下文模板只注入检索结果

### 2. 开闭原则 (OCP)

对扩展开放,对修改关闭:
- 可以通过组合添加新功能
- 不需要修改现有模板

### 3. 依赖倒置原则 (DIP)

依赖抽象而非具体实现:
- 通过变量占位符定义接口
- 具体值在运行时注入

### 4. 组合优于继承

通过组合实现功能扩展:
- 使用 `+` 操作符组合模板
- 避免复杂的继承层次

## 实现原理的第一性思考

### 为什么使用 `+` 操作符?

```python
# Python的操作符重载
class PromptTemplate:
    def __add__(self, other):
        # 组合两个模板
        return combined_template
```

**第一性思考**:
- 数学中: `a + b` 表示两个元素的组合
- 字符串中: `"a" + "b"` 表示拼接
- 模板中: `template1 + template2` 表示组合

这是一种**自然的语义映射**,符合人类直觉。

[来源: reference/source_prompttemplate_01.md:28-59 | LangChain源码分析]

### 为什么区分字符串和函数形式的partial?

```python
# 字符串形式: 立即求值
partial_variables={"language": "English"}

# 函数形式: 延迟求值
partial_variables={"date": get_current_date}
```

**第一性思考**:
- 某些值在创建时就确定 (如配置) → 字符串形式
- 某些值需要在使用时才确定 (如时间) → 函数形式

这是**求值时机**的根本区别。
