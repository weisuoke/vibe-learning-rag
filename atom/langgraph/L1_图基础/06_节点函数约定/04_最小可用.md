## 4.1 最简单的节点函数定义

### 核心概念

节点函数就是一个**普通的 Python 函数**，接收状态（State），返回状态更新（dict）。

**最简签名**：
```python
def node_function(state: State) -> dict:
    return {"key": "value"}
```

[来源: reference/context7_langgraph_01.md]

### 完整示例

```python
from typing_extensions import TypedDict

# 1. 定义状态类型
class State(TypedDict):
    message: str
    count: int

# 2. 定义节点函数
def my_node(state: State) -> dict:
    """最简单的节点函数"""
    # 读取当前状态
    current_count = state["count"]

    # 返回状态更新（只返回需要更新的字段）
    return {
        "count": current_count + 1,
        "message": f"处理了 {current_count + 1} 次"
    }
```

[来源: reference/context7_langgraph_01.md]

### 关键点

1. **第一个参数必须是 State 类型**
2. **返回值是 dict**，只包含需要更新的字段
3. **不要返回完整状态对象**，只返回变化的部分

**类比理解**：
- **前端类比**：就像 React 的 setState，只传入需要更新的字段
- **日常生活类比**：就像填写表单，只填写需要修改的项目

[来源: reference/source_节点函数约定_01.md]

## 4.2 部分状态更新机制

### 核心概念

节点函数返回的 dict 会**自动合并**到现有状态中，而不是替换整个状态。

**更新规则**：
- 返回的字段会更新到状态中
- 未返回的字段保持不变
- 使用 Reducer 函数控制合并逻辑

[来源: reference/source_节点函数约定_01.md]

### 完整示例

```python
from typing_extensions import TypedDict
from typing import Annotated
from operator import add

# 定义状态（带 Reducer）
class State(TypedDict):
    messages: Annotated[list[str], add]  # 使用 add reducer，追加而非替换
    count: int                            # 默认 reducer，直接替换

# 节点函数 1：添加消息
def add_message(state: State) -> dict:
    """只返回需要更新的字段"""
    return {
        "messages": ["新消息"]  # 会被追加到列表，而不是替换
    }

# 节点函数 2：增加计数
def increment_count(state: State) -> dict:
    """只更新 count 字段"""
    return {
        "count": state["count"] + 1  # 会直接替换 count 值
    }
    # 注意：没有返回 messages，所以 messages 保持不变

# 使用示例
initial_state = {
    "messages": ["初始消息"],
    "count": 0
}

# 执行 add_message 后：
# {"messages": ["初始消息", "新消息"], "count": 0}

# 执行 increment_count 后：
# {"messages": ["初始消息", "新消息"], "count": 1}
```

[来源: reference/source_节点函数约定_01.md, reference/context7_langgraph_01.md]

### 关键点

1. **默认行为**：直接替换字段值
2. **Reducer 函数**：自定义合并逻辑（如 `add` 用于列表追加）
3. **未返回的字段**：保持原值不变

**常见错误**：
```python
# ❌ 错误：返回完整状态
def bad_node(state: State) -> State:
    state["count"] = state["count"] + 1
    return state  # 错误：不要返回完整状态

# ✅ 正确：返回部分更新
def good_node(state: State) -> dict:
    return {"count": state["count"] + 1}  # 正确：只返回需要更新的字段
```

[来源: reference/context7_langgraph_01.md]

## 4.3 将节点添加到图

### 核心概念

使用 `StateGraph.add_node()` 方法将节点函数添加到图中。

**基础语法**：
```python
builder.add_node("节点名称", 节点函数)
```

[来源: reference/source_节点函数约定_01.md]

### 完整示例

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

# 1. 定义状态
class State(TypedDict):
    input: str
    output: str

# 2. 定义节点函数
def process_input(state: State) -> dict:
    """处理输入"""
    return {"output": f"处理结果: {state['input']}"}

# 3. 创建图并添加节点
builder = StateGraph(State)
builder.add_node("process", process_input)  # 添加节点

# 4. 连接节点
builder.add_edge(START, "process")  # 从开始节点到 process
builder.add_edge("process", END)    # 从 process 到结束节点

# 5. 编译图
graph = builder.compile()

# 6. 执行图
result = graph.invoke({"input": "Hello"})
print(result)  # {"input": "Hello", "output": "处理结果: Hello"}
```

[来源: reference/context7_langgraph_01.md]

### 关键点

1. **节点名称**：字符串，用于标识节点
2. **节点函数**：可调用对象，接收 State 返回 dict
3. **连接节点**：使用 `add_edge` 连接节点
4. **编译图**：使用 `compile()` 生成可执行图

**类比理解**：
- **前端类比**：就像注册路由，`add_node` 相当于 `router.get("/path", handler)`
- **日常生活类比**：就像在流水线上安装工序站点

[来源: reference/source_节点函数约定_01.md]

## 4.4 完整可运行示例

### 场景：简单的问答系统

构建一个最简单的问答系统，包含 3 个节点：
1. **接收问题**：处理用户输入
2. **生成回答**：调用 LLM 生成回答
3. **格式化输出**：格式化最终输出

[来源: reference/context7_langgraph_01.md]

### 完整代码

```python
from langgraph.graph import StateGraph, START, END
from typing_extensions import TypedDict

# ============ 1. 定义状态 ============
class QAState(TypedDict):
    question: str      # 用户问题
    answer: str        # LLM 回答
    formatted: str     # 格式化后的输出

# ============ 2. 定义节点函数 ============

def receive_question(state: QAState) -> dict:
    """节点1：接收并处理问题"""
    question = state["question"].strip()
    print(f"[节点1] 接收问题: {question}")
    return {"question": question}

def generate_answer(state: QAState) -> dict:
    """节点2：生成回答（模拟 LLM 调用）"""
    question = state["question"]
    # 模拟 LLM 调用（实际应用中替换为真实 LLM）
    answer = f"这是对 '{question}' 的回答"
    print(f"[节点2] 生成回答: {answer}")
    return {"answer": answer}

def format_output(state: QAState) -> dict:
    """节点3：格式化输出"""
    formatted = f"Q: {state['question']}\nA: {state['answer']}"
    print(f"[节点3] 格式化输出:\n{formatted}")
    return {"formatted": formatted}

# ============ 3. 构建图 ============

# 创建图
builder = StateGraph(QAState)

# 添加节点
builder.add_node("receive", receive_question)
builder.add_node("generate", generate_answer)
builder.add_node("format", format_output)

# 连接节点（线性流程）
builder.add_edge(START, "receive")
builder.add_edge("receive", "generate")
builder.add_edge("generate", "format")
builder.add_edge("format", END)

# 编译图
graph = builder.compile()

# ============ 4. 执行图 ============

# 输入问题
input_state = {"question": "什么是 LangGraph?"}

# 执行图
result = graph.invoke(input_state)

# 输出结果
print("\n=== 最终结果 ===")
print(result["formatted"])
```

[来源: reference/context7_langgraph_01.md]

### 运行输出

```
[节点1] 接收问题: 什么是 LangGraph?
[节点2] 生成回答: 这是对 '什么是 LangGraph?' 的回答
[节点3] 格式化输出:
Q: 什么是 LangGraph?
A: 这是对 '什么是 LangGraph?' 的回答

=== 最终结果 ===
Q: 什么是 LangGraph?
A: 这是对 '什么是 LangGraph?' 的回答
```

### 关键点

1. **状态流转**：每个节点接收当前状态，返回更新
2. **线性流程**：使用 `add_edge` 连接节点形成流程
3. **状态累积**：每个节点的更新会累积到状态中
4. **最终结果**：`invoke()` 返回最终状态

[来源: reference/context7_langgraph_01.md]

## 4.5 避免 3 个最常见错误

### 错误 1：返回完整状态对象

**错误示例**：
```python
# ❌ 错误
def bad_node(state: State) -> State:
    state["count"] = state["count"] + 1
    return state  # 错误：返回完整状态
```

**正确示例**：
```python
# ✅ 正确
def good_node(state: State) -> dict:
    return {"count": state["count"] + 1}  # 正确：只返回更新
```

**原因**：LangGraph 使用 Reducer 函数合并状态更新，返回完整状态会覆盖其他节点的更新。

[来源: reference/context7_langgraph_01.md]

---

### 错误 2：修改状态对象

**错误示例**：
```python
# ❌ 错误
def bad_node(state: State) -> dict:
    state["count"] += 1  # 错误：直接修改状态
    return {"count": state["count"]}
```

**正确示例**：
```python
# ✅ 正确
def good_node(state: State) -> dict:
    new_count = state["count"] + 1  # 正确：不修改原状态
    return {"count": new_count}
```

**原因**：状态应该是不可变的，直接修改可能导致意外的副作用。

[来源: reference/source_节点函数约定_01.md]

---

### 错误 3：忘记返回值

**错误示例**：
```python
# ❌ 错误
def bad_node(state: State):
    result = state["count"] + 1
    # 忘记返回值
```

**正确示例**：
```python
# ✅ 正确
def good_node(state: State) -> dict:
    result = state["count"] + 1
    return {"count": result}  # 正确：返回更新
```

**原因**：节点函数必须返回状态更新，否则状态不会改变。

[来源: reference/context7_langgraph_01.md]

## 快速参考卡片

### 节点函数模板

```python
from typing_extensions import TypedDict

# 1. 定义状态
class State(TypedDict):
    field1: str
    field2: int

# 2. 定义节点函数
def my_node(state: State) -> dict:
    """节点函数说明"""
    # 读取状态
    value = state["field1"]

    # 处理逻辑
    result = process(value)

    # 返回更新（只返回需要更新的字段）
    return {"field2": result}

# 3. 添加到图
from langgraph.graph import StateGraph, START, END

builder = StateGraph(State)
builder.add_node("my_node", my_node)
builder.add_edge(START, "my_node")
builder.add_edge("my_node", END)

graph = builder.compile()

# 4. 执行
result = graph.invoke({"field1": "input"})
```

[来源: reference/context7_langgraph_01.md, reference/source_节点函数约定_01.md]

### 核心要点速查

| 要点 | 说明 | 示例 |
|------|------|------|
| **函数签名** | `def node(state: State) -> dict` | 第一个参数是 State，返回 dict |
| **返回值** | 只返回需要更新的字段 | `return {"count": 1}` |
| **状态更新** | 自动合并到现有状态 | 未返回的字段保持不变 |
| **添加节点** | `builder.add_node(name, func)` | 节点名称 + 节点函数 |
| **连接节点** | `builder.add_edge(from, to)` | 定义节点执行顺序 |
| **编译执行** | `graph = builder.compile()` | 生成可执行图 |

[来源: reference/context7_langgraph_01.md, reference/source_节点函数约定_01.md]

---

### 常见错误速查

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| 返回完整状态 | 会覆盖其他节点更新 | 只返回需要更新的字段 |
| 直接修改状态 | 破坏不可变性 | 创建新值而不是修改原值 |
| 忘记返回值 | 状态不会更新 | 始终返回 dict |
| 返回 None | 状态不会更新 | 返回空 dict `{}` 或有效更新 |

[来源: reference/context7_langgraph_01.md]
