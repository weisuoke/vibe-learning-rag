## 完整脚本

将以上步骤整合为一个完整的脚本：

```bash
#!/bin/bash

# OpenClaw 快速上手脚本
# 用途：从克隆到首次运行的自动化脚本

set -e  # 遇到错误立即退出

echo "=== OpenClaw 快速上手 ==="
echo ""

# 步骤 1：克隆仓库
echo "[1/6] 克隆仓库..."
if [ -d "openclaw" ]; then
  echo "目录已存在，跳过克隆"
  cd openclaw
else
  git clone https://github.com/openclaw/openclaw.git
  cd openclaw
fi
echo "✓ 克隆完成"
echo ""

# 步骤 2：检查环境
echo "[2/6] 检查环境..."

# 检查 Node.js 版本
NODE_VERSION=$(node --version | cut -d'v' -f2 | cut -d'.' -f1)
if [ "$NODE_VERSION" -lt 22 ]; then
  echo "❌ Node.js 版本过低: $(node --version)"
  echo "需要 Node.js 22+，请使用 nvm install 22"
  exit 1
fi
echo "✓ Node.js 版本: $(node --version)"

# 检查 pnpm
if ! command -v pnpm &> /dev/null; then
  echo "❌ pnpm 未安装"
  echo "正在安装 pnpm..."
  npm install -g pnpm
fi
echo "✓ pnpm 版本: $(pnpm --version)"
echo ""

# 步骤 3：安装依赖
echo "[3/6] 安装依赖..."
echo "这可能需要 10-15 分钟（包含原生依赖编译）"
pnpm install
echo "✓ 依赖安装完成"
echo ""

# 步骤 4：构建项目
echo "[4/6] 构建项目..."
echo "这可能需要 2-3 分钟"
pnpm build
echo "✓ 构建完成"
echo ""

# 步骤 5：运行测试
echo "[5/6] 运行测试..."
pnpm test:fast
echo "✓ 测试通过"
echo ""

# 步骤 6：启动 Gateway
echo "[6/6] 启动 Gateway..."
echo "Gateway 将在 http://localhost:18789 启动"
echo "按 Ctrl+C 停止"
echo ""
pnpm gateway:dev
```

**使用方式**：
```bash
# 保存为 quickstart.sh
chmod +x quickstart.sh
./quickstart.sh
```

## 故障排查

### 问题 1：依赖安装超时

```bash
# 症状
ERR_PNPM_FETCH_TIMEOUT  Request to https://registry.npmjs.org/... timed out

# 解决方案：使用国内镜像
pnpm config set registry https://registry.npmmirror.com

# 重新安装
pnpm install
```

### 问题 2：磁盘空间不足

```bash
# 症状
ENOSPC: no space left on device

# 检查磁盘空间
df -h

# 清理 pnpm 缓存
pnpm store prune

# 清理 node_modules
rm -rf node_modules
pnpm install
```

### 问题 3：端口被占用

```bash
# 症状
Error: listen EADDRINUSE: address already in use :::18789

# 查找占用端口的进程
lsof -i :18789

# 杀死进程
kill -9 <PID>

# 或使用其他端口
OPENCLAW_PORT=18790 pnpm gateway:dev
```

### 问题 4：权限问题

```bash
# 症状（macOS/Linux）
EACCES: permission denied

# 解决方案：修复权限
sudo chown -R $(whoami) ~/.pnpm-store
sudo chown -R $(whoami) node_modules

# 重新安装
pnpm install
```

## 核心洞察

### 1. 依赖安装是最耗时的环节

**原因**：
- 9 个原生依赖需要编译
- 需要下载和编译 C++ 源码
- 跨平台编译工具链复杂

**优化**：
- 使用国内镜像加速下载
- 确保编译工具链完整
- 使用 `--prefer-offline` 利用缓存

---

### 2. 构建流程有 9 步

**不是简单的 tsc 编译**：
- Canvas A2UI 打包
- tsdown 构建（8 个入口点）
- 生成类型定义
- 复制资源文件
- 写入元数据

---

### 3. 验证很重要

**不要假设成功**：
- 检查构建产物
- 运行测试
- 验证 Gateway 健康状态
- 测试 API 端点

---

## 一句话总结

**OpenClaw 快速上手的关键是：确保环境正确（Node.js 22+, pnpm, 编译工具链），耐心等待依赖安装（10-15分钟），验证每个步骤的输出。**
