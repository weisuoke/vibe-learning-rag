## 类比1:Retriever抽象层

**场景**:前端调用不同的后端服务

```javascript
// 问题:不同的后端服务有不同的API

  method: 'POST',

  method: 'POST',
  body: JSON.stringify({query: '...', limit: 5})

  async query(text, k) {
    // 统一接口,内部适配不同服务
```

**对应关系**:

## 类比2:as_retriever()转换

### 前端开发类比:适配器模式

**场景**:将旧API适配到新接口

```javascript
// 旧API:不同的方法签名
class OldVectorStore {
  searchSimilar(vector, count) { /* ... */ }
  searchMMR(vector, count, diversity) { /* ... */ }
}

// 新接口:统一的方法签名
interface Retriever {
  query(text: string, options: object): Document[]
}

// 适配器:将旧API适配到新接口
class VectorStoreAdapter implements Retriever {
  constructor(private vectorStore: OldVectorStore) {}

  query(text: string, options: object): Document[] {
    const vector = embed(text);
    if (options.type === 'similarity') {
      return this.vectorStore.searchSimilar(vector, options.k);
    } else if (options.type === 'mmr') {
      return this.vectorStore.searchMMR(vector, options.k, options.diversity);
    }
  }
}

// 工厂方法:简化创建
class OldVectorStore {
  asRetriever(options): Retriever {
    return new VectorStoreAdapter(this, options);
  }
}

// 使用
const vectorStore = new OldVectorStore();
const retriever = vectorStore.asRetriever({type: 'similarity', k: 5});
const docs = retriever.query('query');
```

**对应关系**:
- `OldVectorStore` = `VectorStore`
- `VectorStoreAdapter` = `VectorStoreRetriever`
- `asRetriever()` = `as_retriever()`
- `Retriever` = `BaseRetriever`

## 类比3:三种检索策略

### 前端开发类比:搜索排序算法

#### Similarity:相似度排序

```javascript
// 场景:电商网站搜索
function searchProducts(query, k) {
  // 1. 计算相似度
  const scores = products.map(p => similarity(query, p.name));

  // 2. 排序
  const sorted = products.sort((a, b) => scores[b] - scores[a]);

  // 3. 返回Top-K
  return sorted.slice(0, k);
}

// 使用
const results = searchProducts('手机', 10);
// 返回:最相似的10个商品
```

**特点**:
- 简单直接
- 只考虑相关性
- 可能返回重复或相似的结果

---

#### MMR:多样性排序

```javascript
// 场景:推荐系统
function recommendProducts(query, k, diversity) {
  // 1. 获取候选商品
  const candidates = searchProducts(query, k * 5);

  // 2. 迭代选择
  const selected = [];
  for (let i = 0; i < k; i++) {
    let bestScore = -Infinity;
    let bestProduct = null;

    for (const product of candidates) {
      if (selected.includes(product)) continue;

      // 相关性分数
      const relevance = similarity(query, product.name);

      // 多样性分数(与已选商品的差异)
      const diversity = selected.length > 0
        ? Math.min(...selected.map(s => similarity(product, s)))
        : 0;

      // MMR分数
      const mmrScore = diversity * relevance + (1 - diversity) * diversity;

      if (mmrScore > bestScore) {
        bestScore = mmrScore;
        bestProduct = product;
      }
    }

    selected.push(bestProduct);
  }

  return selected;
}

// 使用
const results = recommendProducts('手机', 10, 0.5);
// 返回:既相关又多样的10个商品
```

**特点**:
- 平衡相关性和多样性
- 避免重复推荐
- 适合推荐系统

---

#### Threshold:阈值过滤

```javascript
// 场景:高质量搜索
function searchHighQuality(query, threshold) {
  // 1. 计算相似度
  const results = products.map(p => ({
    product: p,
    score: similarity(query, p.name)
  }));

  // 2. 过滤低分结果
  const filtered = results.filter(r => r.score >= threshold);

  // 3. 排序返回
  return filtered
    .sort((a, b) => b.score - a.score)
    .map(r => r.product);
}

// 使用
const results = searchHighQuality('手机', 0.8);
// 返回:相似度>=0.8的所有商品
```

**特点**:
- 质量控制
- 可能返回很少或没有结果
- 适合高精度场景

### 日常生活类比:找餐厅

#### Similarity:按距离排序

**场景**:找最近的餐厅

**流程**:
1. 打开地图App
2. 搜索"餐厅"
3. 按距离排序
4. 选择最近的5家

**特点**:
- 简单直接
- 只考虑距离
- 可能都是同一类型的餐厅

**对应**:`search_type="similarity"`

---

#### MMR:多样化选择

**场景**:找不同类型的餐厅

**流程**:
1. 搜索"餐厅"
2. 获取候选列表(20家)
3. 选择过程:
   - 第1家:最近的餐厅(中餐)
   - 第2家:较近且不同类型(西餐)
   - 第3家:较近且不同类型(日料)
   - ...
4. 最终选择5家不同类型的餐厅

**特点**:
- 既考虑距离又考虑多样性
- 避免都是同一类型
- 提供更多选择

**对应**:`search_type="mmr"`

---

#### Threshold:高评分过滤

**场景**:只找高评分餐厅

**流程**:
1. 搜索"餐厅"
2. 设置评分阈值(>=4.5星)
3. 过滤低评分餐厅
4. 返回所有高评分餐厅

**特点**:
- 质量保证
- 可能找不到餐厅
- 适合对质量要求高的场景

**对应**:`search_type="similarity_score_threshold"`

## 类比4:LCEL管道集成

### 前端开发类比:Promise链式调用

**场景**:数据处理管道

```javascript
// 传统方式:嵌套回调
fetchData(url, (data) => {
  processData(data, (processed) => {
    saveData(processed, (result) => {
      console.log(result);
    });
  });
});

// Promise链式调用
fetch(url)
  .then(response => response.json())
  .then(data => processData(data))
  .then(processed => saveData(processed))
  .then(result => console.log(result));

// 管道操作符(提案)
const result = url
  |> fetch
  |> (response => response.json())
  |> processData
  |> saveData;
```

**对应关系**:
- Promise链 = LCEL管道
- `.then()` = `|`操作符
- 数据流转 = 文档流转

---

**LangChain LCEL示例**:

```python
# 传统方式:嵌套调用
docs = retriever.invoke(query)
context = format_docs(docs)
prompt_value = prompt.invoke({"context": context, "question": query})
answer = llm.invoke(prompt_value)

# LCEL管道
answer = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
).invoke(query)
```

## 类比6:批量处理

### 前端开发类比:批量API请求

**场景**:批量获取用户信息

```javascript
// 低效:逐个请求
const users = [];
for (const id of userIds) {
  const user = await fetch(`/api/users/${id}`);
  users.push(user);
}

// 高效:批量请求
const users = await fetch('/api/users/batch', {
  method: 'POST',
  body: JSON.stringify({ids: userIds})
});

// Promise.all并发
const users = await Promise.all(
  userIds.map(id => fetch(`/api/users/${id}`))
);
```

**对应关系**:
- 逐个请求 = 逐个调用`invoke()`
- 批量请求 = 调用`batch()`
- `Promise.all` = 异步批量处理

## 类比8:错误处理

### 前端开发类比:API错误处理

**场景**:调用API

```javascript
// 基础错误处理
try {
  const data = await fetch('/api/search');
  return data;
} catch (error) {
  console.error('Search failed:', error);
  return [];  // 返回空结果
}

// 高级错误处理
async function safeSearch(query, retries = 3) {
  for (let i = 0; i < retries; i++) {
    try {
      const data = await fetch('/api/search', {
        body: JSON.stringify({query})
      });
      return data;
    } catch (error) {
      if (i === retries - 1) {
        // 最后一次重试失败,返回降级结果
        return getFallbackResults(query);
      }
      // 等待后重试
      await sleep(1000 * (i + 1));
    }
  }
}
```

**对应关系**:
- `try-catch` = 错误捕获
- 重试机制 = 容错处理
- 降级结果 = 备用检索器
