# 30字核心

**Pi AI 统一 LLM API 是通过 Provider Adapter 模式抽象多个 LLM 提供商的接口，实现一套代码调用 25+ 模型的极简架构。**

---

## 为什么需要统一 API？

在 AI Agent 开发中，你可能需要：
- 在 Claude、GPT-4、Gemini 之间切换模型
- 根据任务复杂度选择不同的模型
- 在成本和性能之间做权衡
- 避免被单一 Provider 锁定

**问题**：每个 LLM Provider 的 API 格式都不同：
```typescript
// OpenAI 格式
{ role: 'user', content: 'Hello' }

// Anthropic 格式
{ role: 'user', content: [{ type: 'text', text: 'Hello' }] }

// Google 格式
{ parts: [{ text: 'Hello' }] }
```

**Pi AI 的解决方案**：一套统一的内部格式，自动转换。

---

## 核心价值

1. **开发效率**：写一次代码，支持所有模型
2. **灵活切换**：一行配置切换 Provider
3. **极简设计**：零学习成本，符合直觉
4. **生产就绪**：支持流式、工具调用、成本追踪
