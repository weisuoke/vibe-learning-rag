# 核心概念2：内置回调处理器

> LangChain 提供的开箱即用的回调处理器实现

---

## 概述

**内置回调处理器**是 LangChain 预先实现的回调处理器类，提供常见的回调功能，无需自定义即可直接使用。

**为什么需要内置回调处理器？**
- **快速开发**：无需从零实现，直接使用
- **最佳实践**：经过验证的实现模式
- **功能覆盖**：满足大部分常见需求
- **学习参考**：作为自定义回调的参考实现

---

## 核心概念1：StdOutCallbackHandler（标准输出回调）

**一句话定义：** 将 LLM、Chain、Agent、Tool 的执行过程打印到标准输出的回调处理器。

### 设计目的

StdOutCallbackHandler 是最基础的回调处理器，主要用于：
- **开发调试**：查看执行流程
- **日志记录**：简单的控制台日志
- **演示展示**：展示 LangChain 的执行过程

### 实现原理

```python
# 源码简化版本
class StdOutCallbackHandler(BaseCallbackHandler):
    """打印到标准输出的回调处理器"""

    def on_chain_start(
        self, serialized: dict, inputs: dict, **kwargs
    ) -> None:
        """Chain 开始时打印"""
        class_name = serialized.get("name", serialized.get("id", ["<unknown>"])[-1])
        print(f"\n\n\033[1m> Entering new {class_name} chain...\033[0m")

    def on_chain_end(self, outputs: dict, **kwargs) -> None:
        """Chain 结束时打印"""
        print("\n\033[1m> Finished chain.\033[0m")

    def on_agent_action(self, action: AgentAction, **kwargs) -> Any:
        """Agent 执行动作时打印"""
        print(action.log)

    def on_tool_end(
        self, output: str, color: str | None = None, **kwargs
    ) -> None:
        """Tool 结束时打印"""
        print(f"\n{output}")
```

### 使用场景

**场景1：调试 Chain 执行流程**
```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import StdOutCallbackHandler
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

    input_variables=["product"],
    template="为 {product} 公司起一个好名字"
)

# 使用 StdOutCallbackHandler

    {"product": "彩色袜子"},
    config={"callbacks": [handler]}
)

# 输出示例：
# > Entering new LLMChain chain...
# Prompt after formatting:
# 为 彩色袜子 公司起一个好名字
# > Finished chain.
```


## 核心概念2：StreamingStdOutCallbackHandler（流式输出回调）

**一句话定义：** 实时将 LLM 生成的每个 token 打印到标准输出的回调处理器。

### 设计目的

StreamingStdOutCallbackHandler 专门用于流式输出场景：
- **实时反馈**：用户立即看到生成内容
- **改善体验**：避免长时间等待
- **流式应用**：聊天机器人、实时问答

### 实现原理

```python
# 源码简化版本
import sys

class StreamingStdOutCallbackHandler(BaseCallbackHandler):
    """流式输出回调处理器"""

    def on_llm_new_token(self, token: str, **kwargs: Any) -> None:
        """每生成一个新 token 时调用"""
        sys.stdout.write(token)  # 写入标准输出
        sys.stdout.flush()       # 立即刷新缓冲区
```

**关键设计点：**
1. **只实现 on_llm_new_token**：专注于流式输出
2. **使用 sys.stdout.write**：而非 print，避免自动换行
3. **立即 flush**：确保 token 立即显示

### 使用场景

**场景1：基础流式输出**
```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import StreamingStdOutCallbackHandler

# 初始化 LLM，启用流式输出

    temperature=0,
    streaming=True,  # 必须启用 streaming
    callbacks=[StreamingStdOutCallbackHandler()]
)

# 调用 LLM
response = llm.invoke("讲一个长故事")

# 输出效果：
# 从前有座山，山里有座庙...（逐字显示）
```

**场景2：Chain 中的流式输出**

**场景3：Agent 中的流式输出（需要自定义）**
```python
# Agent 的流式输出需要自定义回调处理器
class CustomStreamingHandler(StreamingStdOutCallbackHandler):
    def __init__(self):
        super().__init__()
        self.content = ""
        self.final_answer = False

    def on_llm_new_token(self, token: str, **kwargs) -> None:
        """只输出 Final Answer 部分"""
        self.content += token

        # 检测到 Final Answer 标记
        if "Final Answer" in self.content:
            self.final_answer = True
            self.content = ""

        # 只输出 Final Answer 的内容
        if self.final_answer:
            if '"action_input": "' in self.content:
                if token not in ["}"]:
                    sys.stdout.write(token)
                    sys.stdout.flush()

# 使用自定义流式处理器
agent = initialize_agent(
    tools=tools,
    llm=ChatOpenAI(streaming=True),
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION
)

agent.run("查询问题", callbacks=[CustomStreamingHandler()])
```

### 与 FastAPI 集成

**流式 API 端点**
```python
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from langchain.callbacks.base import BaseCallbackHandler
import asyncio
from queue import Queue

class AsyncStreamingCallbackHandler(BaseCallbackHandler):
    """异步流式回调处理器"""

    def __init__(self):
        self.queue = Queue()

    def on_llm_new_token(self, token: str, **kwargs) -> None:
        """将 token 放入队列"""
        self.queue.put(token)

    def on_llm_end(self, response, **kwargs) -> None:
        """结束标记"""
        self.queue.put(None)

app = FastAPI()

@app.get("/chat")
async def chat_stream(query: str):
    """流式聊天端点"""
    handler = AsyncStreamingCallbackHandler()

    # 异步生成器
    async def generate():
        # 启动 LLM 调用（在后台线程）
        llm = ChatOpenAI(streaming=True, callbacks=[handler])
        asyncio.create_task(
            asyncio.to_thread(llm.invoke, query)
        )

        # 从队列中读取 token
        while True:
            token = await asyncio.to_thread(handler.queue.get)
            if token is None:
                break
            yield token

    return StreamingResponse(generate(), media_type="text/plain")
```

### 优缺点

**优点：**
- ✅ 实时反馈，用户体验好
- ✅ 实现简单，易于使用
- ✅ 适合聊天、问答场景

**缺点：**
- ❌ 需要 LLM 支持流式输出
- ❌ Agent 场景需要自定义
- ❌ 只能输出到控制台

## 核心概念3：其他内置回调处理器

### FileCallbackHandler（文件回调）

**一句话定义：** 将回调信息写入文件的回调处理器。

**使用场景：**
```python
from langchain_community.callbacks import FileCallbackHandler
from pathlib import Path

# 创建文件回调处理器
log_file = Path("langchain_log.txt")
handler = FileCallbackHandler(log_file)

# 使用
chain.invoke(
    {"input": "查询"},
    config={"callbacks": [handler]}
)

# 日志会写入 langchain_log.txt
```

**适用场景：**
- 持久化日志记录
- 离线分析执行过程
- 生产环境日志

### UsageMetadataCallbackHandler（使用统计回调）

**一句话定义：** 统计 LLM 调用的 token 用量和成本的回调处理器。

**使用场景：**
```python
from langchain_core.callbacks import UsageMetadataCallbackHandler

# 创建使用统计回调
handler = UsageMetadataCallbackHandler()

# 使用
result = llm.invoke(
    "查询问题",
    config={"callbacks": [handler]}
)

# 获取统计信息
print(f"Prompt tokens: {handler.prompt_tokens}")
print(f"Completion tokens: {handler.completion_tokens}")
print(f"Total tokens: {handler.total_tokens}")
print(f"Estimated cost: ${handler.total_cost:.4f}")
```

**适用场景：**
- 成本追踪
- 性能监控
- 预算控制

### LangChainTracer（LangSmith 追踪）

**一句话定义：** 将执行追踪发送到 LangSmith 平台的回调处理器。

**使用场景：**
```python
import os
from langchain.callbacks.tracers import LangChainTracer

# 配置 LangSmith
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "your-api-key"

# 创建追踪器
tracer = LangChainTracer(project_name="my-rag-project")

# 使用
chain.invoke(
    {"input": "查询"},
    config={"callbacks": [tracer]}
)

# 追踪数据会发送到 LangSmith 平台
```

**适用场景：**
- 生产环境可观测性
- 性能分析
- 调试复杂链路

## 内置回调处理器对比

| 回调处理器 | 主要用途 | 输出目标 | 适用场景 | 生产环境 |
|-----------|---------|---------|---------|---------|
| **StdOutCallbackHandler** | 调试日志 | 控制台 | 开发调试 | ❌ |
| **StreamingStdOutCallbackHandler** | 流式输出 | 控制台 | 实时反馈 | ⚠️ |
| **FileCallbackHandler** | 持久化日志 | 文件 | 日志记录 | ✅ |
| **UsageMetadataCallbackHandler** | 成本统计 | 内存 | 成本追踪 | ✅ |
| **LangChainTracer** | 可观测性 | LangSmith | 生产监控 | ✅ |

## 实战代码示例

```python
"""
内置回调处理器完整示例
演示：StdOutCallbackHandler、StreamingStdOutCallbackHandler、FileCallbackHandler
"""

import os
from pathlib import Path
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import (
    StdOutCallbackHandler,
    StreamingStdOutCallbackHandler
)
from langchain_community.callbacks import FileCallbackHandler
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

# ===== 1. 基础回调：StdOutCallbackHandler =====
print("=== 示例1：StdOutCallbackHandler ===")

llm = ChatOpenAI(temperature=0)
prompt = PromptTemplate(
    input_variables=["product"],
    template="为 {product} 公司起一个好名字"
)
chain = LLMChain(llm=llm, prompt=prompt)

# 使用标准输出回调
stdout_handler = StdOutCallbackHandler()
result = chain.invoke(
    {"product": "AI 助手"},
    config={"callbacks": [stdout_handler]}
)

print(f"\n结果: {result['text']}\n")

# ===== 2. 流式回调：StreamingStdOutCallbackHandler =====
print("=== 示例2：StreamingStdOutCallbackHandler ===")

streaming_llm = ChatOpenAI(
    temperature=0,
    streaming=True,  # 必须启用
    callbacks=[StreamingStdOutCallbackHandler()]
)

print("流式输出: ", end="")
response = streaming_llm.invoke("用一句话介绍 LangChain")
print("\n")

# ===== 3. 文件回调：FileCallbackHandler =====
print("=== 示例3：FileCallbackHandler ===")

log_file = Path("langchain_demo.log")
file_handler = FileCallbackHandler(log_file)

result = chain.invoke(
    {"product": "智能客服"},
    config={"callbacks": [file_handler]}
)

print(f"日志已写入: {log_file}")
print(f"日志内容:\n{log_file.read_text()}\n")

# ===== 4. 组合使用多个回调 =====
print("=== 示例4：组合使用多个回调 ===")

# 同时使用标准输出和文件回调
combined_callbacks = [
    StdOutCallbackHandler(),
    FileCallbackHandler("combined.log")
]

result = chain.invoke(
    {"product": "RAG 系统"},
    config={"callbacks": combined_callbacks}
)

print(f"\n结果: {result['text']}")

# ===== 5. RAG 场景示例 =====
print("\n=== 示例5：RAG 场景中的回调使用 ===")

    "CallbackHandler 用于监控 LangChain 的执行过程"

    texts=texts,
    embedding=OpenAIEmbeddings()
)

    llm=ChatOpenAI(

    callbacks=[StdOutCallbackHandler()]
```

## 关键要点总结

1. **StdOutCallbackHandler**：开发调试的首选，打印执行流程
2. **StreamingStdOutCallbackHandler**：实时反馈，改善用户体验
3. **FileCallbackHandler**：持久化日志，适合生产环境
4. **UsageMetadataCallbackHandler**：成本追踪，预算控制
5. **LangChainTracer**：可观测性，生产监控

**选择建议：**
- 开发阶段：StdOut + Streaming
- 测试阶段：File + UsageMetadata
- 生产阶段：File + UsageMetadata + Tracer

**在 RAG 开发中：**
- 调试检索：使用 StdOutCallbackHandler
- 流式问答：使用 StreamingStdOutCallbackHandler
- 成本控制：使用 UsageMetadataCallbackHandler
- 生产监控：使用 LangChainTracer
