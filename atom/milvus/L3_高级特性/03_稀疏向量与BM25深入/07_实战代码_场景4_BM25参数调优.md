## 场景描述

你需要为一个**技术文档搜索系统**调优 BM25 参数，在准确率和性能之间找到最佳平衡点。

# ===== 3. 评估函数 =====
def evaluate_search(collection: Collection, queries: List[Dict], top_k: int = 5) -> Dict:
    """评估搜索性能"""
    total_recall = 0
    total_precision = 0
    total_f1 = 0

    for query_data in queries:
        query_text = query_data["query"]
        relevant_docs = set(query_data["relevant_docs"])

        # 执行搜索
        results = collection.search(
            data=[query_text],
            anns_field="sparse_vector",
            param={"metric_type": "BM25"},
            limit=top_k,
            output_fields=["doc_id"]
        )

        # 获取返回的文档 ID
        retrieved_docs = set([hit.entity.get("doc_id") for hit in results[0]])

        # 计算指标
        true_positives = len(relevant_docs & retrieved_docs)
        recall = true_positives / len(relevant_docs) if relevant_docs else 0
        precision = true_positives / len(retrieved_docs) if retrieved_docs else 0
        f1 = 2 * (precision * recall) / (precision + recall) if (precision + recall) > 0 else 0

        total_recall += recall
        total_precision += precision
        total_f1 += f1

    num_queries = len(queries)
    return {
        "recall": total_recall / num_queries,
        "precision": total_precision / num_queries,
        "f1_score": total_f1 / num_queries
    }

## 关键点解析

### 1. BM25 参数含义

**bm25_k1（词频饱和参数）**：
- 控制词频对分数的影响
- k1 越大，词频影响越大
- 推荐范围：1.0-2.0
- 默认值：1.2

**bm25_b（文档长度归一化参数）**：
- 控制文档长度对分数的影响
- b 越大，长文档惩罚越重
- 推荐范围：0.3-1.0
- 默认值：0.75

**drop_ratio_build（索引构建丢弃比例）**：
- 减少索引大小，加速构建
- 丢弃权重最低的 N% 元素
- 推荐范围：0.0-0.3
- 默认值：0.0

### 2. 评估指标

**Recall（召回率）**：
- 公式：`Recall = TP / (TP + FN)`
- 含义：相关文档中被检索到的比例
- 越高越好

**Precision（精确率）**：
- 公式：`Precision = TP / (TP + FP)`
- 含义：检索结果中相关文档的比例
- 越高越好

**F1-Score（F1 分数）**：
- 公式：`F1 = 2 * (Precision * Recall) / (Precision + Recall)`
- 含义：Precision 和 Recall 的调和平均
- 综合评估指标

### 3. 参数调优策略

**网格搜索**：
- 定义参数网格
- 遍历所有参数组合
- 选择最优参数

**实验驱动**：
- 准备测试集（查询 + Ground Truth）
- 评估不同参数组合
- 选择 F1-Score 最高的配置

**权衡考虑**：
- 精度 vs 性能
- Recall vs Precision
- 索引大小 vs 搜索速度

### 4. 调优建议

**短文档（< 100 词）**：
- k1 = 1.0-1.2
- b = 0.3-0.5
- drop_ratio = 0.0-0.1

**长文档（> 500 词）**：
- k1 = 1.5-2.0
- b = 0.75-1.0
- drop_ratio = 0.1-0.2

**代码搜索**：
- k1 = 1.0（精确匹配）
- b = 0.5（长度影响小）
- drop_ratio = 0.0（精度优先）
