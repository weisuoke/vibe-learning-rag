# Agent Memory集成 - 30字核心

## 核心概念（30字）

**Agent Memory集成 是让无状态的 LLM Agent 拥有记忆能力的机制，通过 checkpointer（短期）和 store（长期）实现跨轮次上下文保持。**

---

## 为什么这30字重要？

这30字揭示了 Agent Memory 的三个本质：

1. **问题根源**：LLM 本身是无状态的，每次调用都是"初次见面"
2. **解决方案**：通过外部存储（checkpointer + store）弥补无状态缺陷
3. **核心价值**：让 Agent 的多轮对话自然流畅，像"有记忆的人"一样工作

### 核心公式

```
Agent Memory = 外部存储 + 自动加载/保存 + 注入到 Prompt

具体实现：
短期记忆（checkpointer）→ 同一会话内的对话历史
长期记忆（store）        → 跨会话的用户偏好和知识
```

---

## 展开说明

### 1. Memory 的两层架构

LangGraph 2026 将记忆分为清晰的两层：

| 层级 | 机制 | 作用域 | 类比 |
|------|------|--------|------|
| **短期记忆** | Checkpointer | 单次会话（thread 内） | 聊天时的上下文 |
| **长期记忆** | Store | 跨会话（thread 间） | 用户档案/偏好 |

### 2. Memory 的三阶段生命周期

```
1. Load  → 加载历史记忆到当前 prompt
2. Save  → 保存本轮交互到存储
3. Clear → 清除记忆（可选）
```

### 3. 为什么 RAG 开发需要 Agent Memory？

- **多轮检索场景**：用户说"换一个来源再查一下"，Agent 需要记住之前查的是什么
- **个性化问答**：记住用户偏好（"我只看中文论文"），后续检索自动过滤
- **长对话知识积累**：前面检索到的信息不需要反复查询

## 记忆技巧

### 口诀

**"短期靠 checkpoint，长期靠 store，中间件管压缩"**

- **checkpoint**：像游戏存档，自动保存每一步
- **store**：像笔记本，跨会话记录重要信息
- **中间件**：像编辑，决定哪些记忆要保留、哪些要压缩

### 场景联想

想象你在和一个客服对话：
- **没有记忆**：每次你说话，客服都问"请问您是哪位？"→ 体验极差
- **短期记忆**：同一次通话中客服记得你说了什么 → checkpointer
- **长期记忆**：下次来电客服还记得你的历史问题 → store

## 与 RAG 开发的联系

在 RAG Agent 系统中，Memory 集成是实现"智能知识助手"的关键：

```python
# 典型场景：带记忆的 RAG Agent
# 第1轮：用户问"什么是 Transformer？"
#   → Agent 检索并回答
#   → Memory 保存：用户在学 NLP 基础

# 第2轮：用户问"它和 RNN 有什么区别？"
#   → Memory 加载：知道"它"指 Transformer
#   → Agent 精准检索 Transformer vs RNN
#   → 无需用户重复说明背景
```
