# 核心概念3：节点函数协议

> **来源**：基于 LangGraph 源码分析（_node.py:16-93）、Context7 官方文档、社区实践案例

---

## 概述

节点函数协议定义了 LangGraph 中节点函数的各种签名规范，是实现灵活节点设计的核心机制。

**核心价值**：
- 支持 9 种不同的节点签名，满足不同场景需求
- 提供依赖注入机制（config、writer、store、runtime）
- 无缝集成 LangChain Runnable
- 自动处理同步和异步节点

---

## 核心概念1：9种节点签名协议

### 什么是节点签名协议？

**节点签名协议**定义了节点函数可以接受哪些参数、返回什么类型的值。LangGraph 通过 Protocol 类型定义了 9 种不同的节点签名，框架会自动识别节点函数的签名并注入相应的依赖。

**来源**：`sourcecode/langgraph/libs/langgraph/langgraph/graph/_node.py:16-93`

### 9种节点签名详解

#### 1. 基础节点：`_Node`

**定义**：
```python
class _Node(Protocol[NodeInputT_contra]):
    def __call__(self, state: NodeInputT_contra) -> Any: ...
```

**特点**：
- 最简单的节点签名
- 只接受 state 参数
- 返回部分状态更新（dict）

**使用场景**：
- 简单的状态转换
- 不需要访问运行时信息
- 纯函数式节点

**示例**：
```python
from typing_extensions import TypedDict

class State(TypedDict):
    count: int
    message: str

def increment_node(state: State) -> dict:
    """基础节点：只接受 state"""
    return {"count": state["count"] + 1}
```

**来源**：源码 _node.py:16-18

---

#### 2. 带 Config 的节点：`_NodeWithConfig`

**定义**：
```python
class _NodeWithConfig(Protocol[NodeInputT_contra]):
    def __call__(self, state: NodeInputT_contra, config: RunnableConfig) -> Any: ...
```

**特点**：
- 接受 state 和 config 参数
- config 包含运行时配置信息（metadata、tags、callbacks 等）
- 支持 LangSmith 追踪

**使用场景**：
- 需要访问运行时配置
- 需要自定义 callbacks
- 需要传递 metadata

**示例**：
```python
from langchain_core.runnables import RunnableConfig

def node_with_config(state: State, config: RunnableConfig) -> dict:
    """带 config 的节点：可以访问运行时配置"""
    # 访问配置信息
    run_id = config.get("run_id")
    tags = config.get("tags", [])

    print(f"Run ID: {run_id}, Tags: {tags}")
    return {"message": f"Processed with config"}
```

**来源**：源码 _node.py:21-23

---

#### 3. 带 Writer 的节点：`_NodeWithWriter`

**定义**：
```python
class _NodeWithWriter(Protocol[NodeInputT_contra]):
    def __call__(self, state: NodeInputT_contra, *, writer: StreamWriter) -> Any: ...
```

**特点**：
- 接受 state 和 writer 参数（关键字参数）
- writer 用于流式输出中间结果
- 支持实时进度反馈

**使用场景**：
- 长时间运行的任务
- 需要实时反馈进度
- 流式输出结果

**示例**：
```python
from langgraph.pregel import StreamWriter

def node_with_writer(state: State, *, writer: StreamWriter) -> dict:
    """带 writer 的节点：可以流式输出中间结果"""
    # 流式输出进度
    writer("开始处理...")
    # 模拟长时间任务
    for i in range(5):
        writer(f"进度: {(i+1)*20}%")
    writer("处理完成！")

    return {"message": "Done"}
```

**来源**：源码 _node.py:26-28

#### 4. 带 Store 的节点：`_NodeWithStore`

**定义**：
```python
class _NodeWithStore(Protocol[NodeInputT_contra]):
    def __call__(self, state: NodeInputT_contra, *, store: BaseStore) -> Any: ...
```

**特点**：
- 接受 state 和 store 参数（关键字参数）
- store 用于持久化存储
- 支持跨节点共享数据

**使用场景**：
- 需要持久化数据
- 跨节点共享信息
- 缓存中间结果

**示例**：
```python
from langgraph.store.base import BaseStore

def node_with_store(state: State, *, store: BaseStore) -> dict:
    """带 store 的节点：可以访问持久化存储"""
    # 从 store 读取数据
    cached_data = store.get("cache_key")

    if cached_data is None:
        # 计算并缓存
        result = expensive_computation()
        store.put("cache_key", result)
    else:
        result = cached_data

    return {"message": f"Result: {result}"}
```

#### 9. 带 Runtime 的节点：`_NodeWithRuntime`（推荐）

**定义**：
```python
class _NodeWithRuntime(Protocol[NodeInputT_contra, ContextT]):
    def __call__(
        self, state: NodeInputT_contra, *, runtime: Runtime[ContextT]
    ) -> Any: ...
```

**特点**：
- 新版本推荐的签名
- runtime 封装了 config、writer、store、context
- 统一的依赖注入接口

**使用场景**：
- 新项目推荐使用
- 需要访问运行时上下文
- 简化依赖注入

**示例**：
```python
from langgraph.runtime import Runtime
from typing_extensions import TypedDict

class Context(TypedDict):
    user_id: str
    api_key: str

def node_with_runtime(state: State, *, runtime: Runtime[Context]) -> dict:
    """带 runtime 的节点：统一的依赖注入接口"""
    # 访问上下文
    user_id = runtime.context.get("user_id")

    # 访问 config
    run_id = runtime.config.get("run_id")

    # 流式输出
    runtime.writer(f"Processing for user: {user_id}")

    # 访问 store
    cached = runtime.store.get(f"user_{user_id}_cache")

    return {"message": f"Processed for {user_id}"}
```

**来源**：源码 _node.py:68-71

### 节点签名对比表

| 签名类型 | state | config | writer | store | runtime | 使用场景 |
|---------|-------|--------|--------|-------|---------|---------|
| `_Node` | ✅ | ❌ | ❌ | ❌ | ❌ | 简单状态转换 |
| `_NodeWithConfig` | ✅ | ✅ | ❌ | ❌ | ❌ | 需要配置信息 |
| `_NodeWithWriter` | ✅ | ❌ | ✅ | ❌ | ❌ | 流式输出 |
| `_NodeWithStore` | ✅ | ❌ | ❌ | ✅ | ❌ | 持久化存储 |
| `_NodeWithWriterStore` | ✅ | ❌ | ✅ | ✅ | ❌ | 流式 + 存储 |
| `_NodeWithConfigWriter` | ✅ | ✅ | ✅ | ❌ | ❌ | 配置 + 流式 |
| `_NodeWithConfigStore` | ✅ | ✅ | ❌ | ✅ | ❌ | 配置 + 存储 |
| `_NodeWithConfigWriterStore` | ✅ | ✅ | ✅ | ✅ | ❌ | 全功能 |
| `_NodeWithRuntime` | ✅ | ❌ | ❌ | ❌ | ✅ | 推荐使用 |

## 核心概念2：节点输入输出规范

### 输入规范

**节点函数的输入**：
1. **必需参数**：`state` - 完整的状态对象
2. **可选参数**：`config` - 运行时配置
3. **关键字参数**：`writer`、`store`、`runtime` - 依赖注入

**类型约束**：
- `state` 的类型必须与 StateGraph 的 `state_schema` 兼容
- 可以使用 `input_schema` 参数自定义节点的输入类型

**示例**：
```python
from typing_extensions import TypedDict

class State(TypedDict):
    x: int
    y: int

class NodeInput(TypedDict):
    x: int  # 只需要 x 字段

def my_node(state: NodeInput) -> dict:
    """节点只需要部分状态"""
    return {"y": state["x"] * 2}

# 添加节点时指定 input_schema
builder.add_node(my_node, input_schema=NodeInput)
```

**来源**：Context7 官方文档

---

### 输出规范

**节点函数的输出**：
- 返回 `dict` 类型，表示部分状态更新
- 返回的字段会与当前状态合并
- 使用 reducer 函数控制合并策略

**部分更新机制**：
```python
class State(TypedDict):
    count: int
    message: str
    items: list

def node_a(state: State) -> dict:
    # 只更新 count 字段
    return {"count": state["count"] + 1}

def node_b(state: State) -> dict:
    # 只更新 message 字段
    return {"message": "Updated"}

def node_c(state: State) -> dict:
    # 更新多个字段
    return {
        "count": state["count"] + 10,
        "items": state["items"] + ["new_item"]
    }
```

**来源**：Context7 官方文档

---

### Reducer 函数

**Reducer 函数**控制状态字段的更新策略：

```python
import operator
from typing import Annotated

class State(TypedDict):
    # 使用 operator.add 作为 reducer，列表变为 append-only
    aggregate: Annotated[list, operator.add]

def node_a(state: State) -> dict:
    return {"aggregate": ["A"]}  # 追加 ["A"]

def node_b(state: State) -> dict:
    return {"aggregate": ["B"]}  # 追加 ["B"]

# 最终 state["aggregate"] = ["A", "B"]
```

**常用 reducer**：
- `operator.add`：列表追加、数字相加
- `lambda a, b: b`：直接替换（默认行为）
- 自定义函数：实现复杂合并逻辑

**来源**：Context7 官方文档

## 核心概念3：Runnable集成与异步支持

### Runnable 集成

**LangChain Runnable** 可以直接作为节点添加到 StateGraph：

```python
from langchain_core.runnables import RunnableLambda
from langgraph.graph import StateGraph, START

class State(TypedDict):
    text: str

# 创建 Runnable
runnable = RunnableLambda(lambda x: {"text": x["text"].upper()})

# 直接作为节点添加
builder = StateGraph(State)
builder.add_node("uppercase", runnable)
builder.add_edge(START, "uppercase")
```

**支持的 Runnable 类型**：
- `RunnableLambda`：函数包装
- `RunnableSequence`：链式组合
- `RunnableParallel`：并行执行
- LangChain 的 LLM、Chain、Tool 等

**来源**：Context7 官方文档、源码 _node.py:74-81

---

### 异步节点支持

**异步节点函数**：LangGraph 自动处理异步节点的执行

```python
import asyncio
from typing_extensions import TypedDict

class State(TypedDict):
    result: str

async def async_node(state: State) -> dict:
    """异步节点函数"""
    # 模拟异步操作
    await asyncio.sleep(1)
    result = await fetch_data_from_api()
    return {"result": result}

# 添加异步节点（与同步节点相同）
builder.add_node("async_node", async_node)
```

**异步执行方法**：
```python
# 使用 ainvoke 执行异步图
result = await graph.ainvoke({"result": ""})

# 使用 astream 流式执行
async for chunk in graph.astream({"result": ""}):
    print(chunk)
```

## 扩展概念：StateNodeSpec 数据类

### StateNodeSpec 定义

**StateNodeSpec** 是节点规范的数据类，封装了节点的所有配置信息：

```python
@dataclass(slots=True)
class StateNodeSpec(Generic[NodeInputT, ContextT]):
    runnable: StateNode[NodeInputT, ContextT]  # 节点函数或 Runnable
    metadata: dict[str, Any] | None            # 节点元数据
    input_schema: type[NodeInputT]             # 输入 schema
    retry_policy: RetryPolicy | Sequence[RetryPolicy] | None  # 重试策略
    cache_policy: CachePolicy | None           # 缓存策略
    ends: tuple[str, ...] | dict[str, str] | None  # 目标节点
    defer: bool = False                        # 是否延迟执行
```

**来源**：源码 _node.py:84-93

---

### 字段说明

#### 1. runnable
- 节点函数或 Runnable 对象
- 支持 9 种节点签名

#### 2. metadata
- 节点元数据
- 用于追踪、日志、调试

#### 3. input_schema
- 节点输入类型
- 默认使用图的 state_schema

#### 4. retry_policy
- 重试策略
- 支持多个重试策略

#### 5. cache_policy
- 缓存策略
- 控制节点结果缓存

#### 6. ends
- 目标节点
- 用于图渲染

#### 7. defer
- 是否延迟执行
- 用于优化执行顺序

## 实战示例

### 示例1：基础节点

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    count: int

def increment(state: State) -> dict:
    return {"count": state["count"] + 1}

builder = StateGraph(State)
builder.add_node("increment", increment)
builder.add_edge(START, "increment")
builder.add_edge("increment", END)

graph = builder.compile()
result = graph.invoke({"count": 0})
print(result)  # {"count": 1}
```

---

### 示例2：带 Runtime 的节点

```python
from langgraph.runtime import Runtime
from typing_extensions import TypedDict

class State(TypedDict):
    message: str

class Context(TypedDict):
    user_id: str

def process_with_runtime(state: State, *, runtime: Runtime[Context]) -> dict:
    user_id = runtime.context.get("user_id")
    runtime.writer(f"Processing for user: {user_id}")
    return {"message": f"Hello, {user_id}!"}

builder = StateGraph(State, context_schema=Context)
builder.add_node("process", process_with_runtime)
builder.add_edge(START, "process")
builder.add_edge("process", END)

graph = builder.compile()
result = graph.invoke({"message": ""}, context={"user_id": "alice"})
print(result)  # {"message": "Hello, alice!"}
```

## 总结

### 核心要点

1. **9种节点签名**：从简单到复杂，满足不同场景需求
2. **推荐使用 Runtime**：新项目优先使用 `_NodeWithRuntime` 签名
3. **部分更新机制**：节点只需返回需要更新的字段
4. **Runnable 集成**：无缝集成 LangChain 生态
5. **异步支持**：自动处理异步节点执行

### 最佳实践

1. **简单场景**：使用基础节点签名（`_Node`）
2. **需要配置**：使用 `_NodeWithConfig`
3. **流式输出**：使用 `_NodeWithWriter`
4. **持久化**：使用 `_NodeWithStore`
5. **新项目**：使用 `_NodeWithRuntime`（推荐）
