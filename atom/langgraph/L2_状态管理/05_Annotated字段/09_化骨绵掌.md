## 卡片1：直觉理解 - Annotated 是什么？

**一句话：** Annotated 是 Python 的类型装饰器，在 LangGraph 中用来给状态字段附加"合并规则"，控制状态更新时是覆盖还是累积。

**举例：**

```python
from typing import Annotated
import operator

# 没有 Annotated：每次覆盖
class State1(TypedDict):
    messages: list  # 新值直接覆盖旧值

# 有 Annotated：每次累加
class State2(TypedDict):
    messages: Annotated[list, operator.add]  # 新值追加到旧值
```

**应用：** 对话系统需要累积消息历史，而不是每次覆盖，这时就需要 Annotated。

**类比：** 就像购物车，没有 Annotated 是"清空购物车再放新商品"，有 Annotated 是"把新商品加到购物车里"。

## 卡片2：形式化定义 - Annotated 的语法结构

**一句话：** `Annotated[type, metadata]` 是 Python 3.9+ 的标准语法，第一个参数是类型，后面的参数是元数据。

**举例：**

```python
from typing import Annotated, get_type_hints

class State(TypedDict):
    x: Annotated[int, "这是元数据"]
    y: Annotated[list, operator.add, "可以有多个元数据"]

# 解析类型提示
hints = get_type_hints(State, include_extras=True)
print(hints['x'].__metadata__)  # ("这是元数据",)
print(hints['x'].__origin__)    # int
```

**应用：** LangGraph 通过 `get_type_hints(include_extras=True)` 获取 Annotated 的元数据，提取 reducer 函数。

**关键点：**
- `__metadata__`：元数据元组
- `__origin__`：原始类型
- `include_extras=True`：必须设置才能保留 Annotated 信息

## 卡片3：Reducer 函数签名 - 两个参数的魔法

**一句话：** Reducer 函数必须是 `(old_value, new_value) -> merged_value`，恰好两个位置参数，返回合并后的值。

**举例：**

```python
# ✅ 正确的 Reducer 签名
def merge_lists(old: list, new: list) -> list:
    return old + new

def merge_dedup(old: list, new: list) -> list:
    seen = set(old)
    return old + [x for x in new if x not in seen]

# ❌ 错误的签名
def bad_reducer(value):  # 只有一个参数
    return value

def bad_reducer2(old, new, extra):  # 三个参数
    return old + new
```

**应用：** LangGraph 使用 `inspect.signature()` 验证 reducer 签名，不符合要求会抛出 `ValueError`。

**验证代码：**

```python
from inspect import signature

sig = signature(merge_lists)
params = list(sig.parameters.values())
print(len(params))  # 必须是 2
```

## 卡片4：operator.add - 最常用的内置 Reducer

**一句话：** `operator.add` 是 Python 内置的加法运算符，用于列表拼接、字符串拼接、数字相加。

**举例：**

```python
import operator

# 列表拼接
operator.add([1, 2], [3, 4])  # [1, 2, 3, 4]

# 字符串拼接
operator.add("Hello ", "World")  # "Hello World"

# 数字相加
operator.add(10, 20)  # 30

# 在 LangGraph 中使用
class State(TypedDict):
    messages: Annotated[list, operator.add]
    logs: Annotated[list[str], operator.add]
```

**应用：** 对话系统、多步推理、日志累积等场景都使用 `operator.add`。

**注意：** `operator.add` 等价于 `lambda x, y: x + y`，但更简洁。

## 卡片5：add_messages - 智能消息管理

**一句话：** `add_messages` 是 LangGraph 预定义的 reducer，支持按 ID 更新消息，而不是简单追加。

**举例：**

```python
from langgraph.graph.message import add_messages
from langchain_core.messages import HumanMessage, AIMessage

# 场景1：追加新消息
msgs1 = [HumanMessage(content="Hello", id="1")]
msgs2 = [AIMessage(content="Hi!", id="2")]
result = add_messages(msgs1, msgs2)
# [HumanMessage(id="1"), AIMessage(id="2")]

# 场景2：按 ID 更新消息
msgs1 = [HumanMessage(content="Hello", id="1")]
msgs2 = [HumanMessage(content="Hello again", id="1")]
result = add_messages(msgs1, msgs2)
# [HumanMessage(content="Hello again", id="1")]  # 覆盖了旧消息
```

**应用：** 对话系统中，用户可能修改之前的消息，`add_messages` 会自动更新而不是重复追加。

**对比：**
- `operator.add`：总是追加，可能重复
- `add_messages`：按 ID 去重更新

## 卡片6：自定义 Reducer - 三种实现方式

**一句话：** 自定义 Reducer 可以用 lambda、普通函数、工厂函数三种方式实现。

**举例：**

```python
# 方式1：Lambda 函数
class State1(TypedDict):
    value: Annotated[str, lambda x, y: y or x]  # 优先使用新值

# 方式2：普通函数
def merge_with_dedup(old: list, new: list) -> list:
    seen = set(old)
    return old + [x for x in new if x not in seen]

class State2(TypedDict):
    items: Annotated[list, merge_with_dedup]

# 方式3：工厂函数（带参数）
def merge_with_limit(max_size: int):
    def reducer(old: list, new: list) -> list:
        combined = old + new
        return combined[-max_size:]  # 只保留最后 max_size 个
    return reducer

class State3(TypedDict):
    recent: Annotated[list, merge_with_limit(100)]
```

**应用：**
- Lambda：简单逻辑
- 普通函数：复杂逻辑
- 工厂函数：需要配置参数

## 卡片7：BinaryOperatorAggregate - 内部实现机制

**一句话：** `BinaryOperatorAggregate` 是 LangGraph 内部的 Channel 类，负责存储状态值并应用 reducer 函数。

**举例：**

```python
# 简化的实现逻辑
class BinaryOperatorAggregate:
    def __init__(self, typ, operator):
        self.operator = operator
        self.value = typ()  # 初始化为空值

    def update(self, values):
        if self.value is MISSING:
            self.value = values[0]  # 第一个值直接赋值
            values = values[1:]

        for value in values:
            self.value = self.operator(self.value, value)  # 应用 reducer
```

**应用：** 理解内部机制有助于调试状态更新问题。

**关键点：**
1. 第一个值直接赋值，不调用 reducer
2. 后续值通过 reducer 合并
3. 支持 `Overwrite` 覆盖模式

## 卡片8：状态更新策略对比 - 覆盖 vs 累积

**一句话：** 无 Annotated 是覆盖模式，有 Annotated 是累积模式，选择取决于业务需求。

**举例：**

```python
class State(TypedDict):
    # 覆盖模式：适合单值状态
    current_step: str
    user_id: str
    timestamp: float

    # 累积模式：适合列表/历史
    messages: Annotated[list, operator.add]
    steps: Annotated[list[str], operator.add]

    # 条件覆盖：自定义逻辑
    result: Annotated[str, lambda x, y: y or x]

# 节点返回
def node(state):
    return {
        "current_step": "step2",  # 直接覆盖
        "messages": [new_message],  # 追加到列表
        "result": "new_result"  # 条件覆盖
    }
```

**应用：**
- 覆盖：配置、状态标记、单值数据
- 累积：消息、日志、步骤、工具结果

**性能对比：**
- 覆盖：O(1)
- 累积：O(n)，n 是列表长度

## 卡片9：常见陷阱与解决方案

**一句话：** 三大陷阱：可变对象共享、签名错误、类型不匹配。

**举例：**

```python
# 陷阱1：可变对象共享 ❌
def bad_reducer(old: list, new: list) -> list:
    old.extend(new)  # 修改了原始列表
    return old

# 解决方案 ✅
def good_reducer(old: list, new: list) -> list:
    return old + new  # 创建新列表

# 陷阱2：签名错误 ❌
def bad_reducer(value):
    return value

# 解决方案 ✅
def good_reducer(old, new):
    return old + new

# 陷阱3：类型不匹配 ❌
class State(TypedDict):
    messages: Annotated[list, operator.or_]  # or_ 用于字典

# 解决方案 ✅
class State(TypedDict):
    messages: Annotated[list, operator.add]
    metadata: Annotated[dict, operator.or_]
```

**应用：** 遇到状态更新错误时，首先检查这三个陷阱。

## 卡片10：实战应用场景 - 从简单到复杂

**一句话：** Annotated 字段在对话系统、ReAct Agent、多步推理等场景中都有应用。

**举例：**

```python
# 场景1：对话系统
class ConversationState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    user_id: str
    session_id: str

# 场景2：ReAct Agent（避免 token 膨胀）
class ReActState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    tool_results: Annotated[list[ToolResult], operator.add]  # 分离工具结果
    reasoning: Annotated[list[str], operator.add]

# 场景3：多步推理
class ReasoningState(TypedDict):
    question: str
    thoughts: Annotated[list[str], operator.add]
    actions: Annotated[list[dict], operator.add]
    observations: Annotated[list[str], operator.add]
    answer: str

# 场景4：限制大小（避免内存溢出）
def merge_with_limit(max_size: int):
    def reducer(old: list, new: list) -> list:
        combined = old + new
        return combined[-max_size:]
    return reducer

class State(TypedDict):
    recent_messages: Annotated[list, merge_with_limit(100)]
```

**应用：** 根据业务需求选择合适的 reducer 策略。

**最佳实践：**
1. 分离不同类型的数据到独立字段
2. 使用限制大小的 reducer 避免内存溢出
3. 使用 TypedDict 而非 Pydantic（性能更好）
4. 为 Annotated 字段指定明确的类型

## 快速参考

### 常用 Reducer 速查表

| Reducer | 用途 | 示例 |
|---------|------|------|
| `operator.add` | 列表/字符串拼接 | `[1, 2] + [3]` = `[1, 2, 3]` |
| `operator.or_` | 字典合并 | `{a: 1} \| {b: 2}` = `{a: 1, b: 2}` |
| `add_messages` | 消息列表管理 | 按 ID 更新或追加 |
| `lambda x, y: y or x` | 优先使用新值 | 新值非空则使用新值 |
| `lambda x, y: y if y is not None else x` | 条件覆盖 | 新值非 None 则覆盖 |

### 签名验证代码

```python
from inspect import signature

def is_valid_reducer(func):
    sig = signature(func)
    params = list(sig.parameters.values())
    return sum(
        p.kind in (p.POSITIONAL_ONLY, p.POSITIONAL_OR_KEYWORD)
        for p in params
    ) == 2

# 测试
print(is_valid_reducer(operator.add))  # True
print(is_valid_reducer(lambda x: x))   # False
```

### 调试技巧

```python
# 1. 打印状态更新
def debug_reducer(old, new):
    print(f"Old: {old}, New: {new}")
    result = old + new
    print(f"Result: {result}")
    return result

# 2. 检查类型提示
from typing import get_type_hints

hints = get_type_hints(State, include_extras=True)
for name, typ in hints.items():
    if hasattr(typ, '__metadata__'):
        print(f"{name}: {typ.__metadata__}")

# 3. 验证 reducer 签名
from inspect import signature

def validate_state(state_class):
    hints = get_type_hints(state_class, include_extras=True)
    for name, typ in hints.items():
        if hasattr(typ, '__metadata__'):
            reducer = typ.__metadata__[-1]
            if callable(reducer):
                sig = signature(reducer)
                params = list(sig.parameters.values())
                if len(params) != 2:
                    print(f"❌ {name}: Invalid reducer signature")
                else:
                    print(f"✅ {name}: Valid reducer")
```
