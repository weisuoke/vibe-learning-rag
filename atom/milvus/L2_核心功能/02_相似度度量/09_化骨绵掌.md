## 卡片1：相似度度量的本质

**一句话：** 相似度度量是将"相似"这个抽象概念转换为可计算数值的数学函数

**核心概念：**

```
输入：两个向量 A 和 B
输出：一个实数（相似度分数）
目的：量化"接近程度"
```

**为什么需要？**

计算机只能处理数字，无法直接理解"相似"。相似度度量提供了统一的比较标准。

**举例：**

```python
# 人类的直觉
"苹果" 和 "橙子" 很相似（都是水果）

# 计算机的度量
embedding_apple = [0.2, 0.8, 0.1, ...]
embedding_orange = [0.3, 0.7, 0.15, ...]
cosine_similarity = 0.95  # 高相似度
```

**应用：** 文档检索、推荐系统、图像搜索、去重检测

## 卡片2：L2距离（欧几里得距离）

**一句话：** L2距离测量向量之间的直线距离，就像地图上两点间的直线距离

**数学公式：**

```
L2(A, B) = √(Σ(ai - bi)²)
```

**几何意义：** 高维空间中两点的直线距离（勾股定理的推广）

**特点：**
- 值域：[0, +∞)
- 距离越小，越相似
- 考虑绝对位置差异

**Python实现：**

```python
import numpy as np

def l2_distance(a, b):
    return np.sqrt(np.sum((a - b) ** 2))

vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])
print(f"L2距离: {l2_distance(vec_a, vec_b):.4f}")  # 1.7321
```

**适用场景：** 图像检索、坐标数据、向量长度有意义的场景

## 卡片3：余弦相似度（COSINE）

**一句话：** 余弦相似度测量向量之间的夹角，只关注方向，忽略长度

**数学公式：**

```
COSINE(A, B) = (A·B) / (||A|| × ||B||)
```

**几何意义：** 向量夹角的余弦值

**特点：**
- 值域：[-1, 1]
- 值越大，越相似（1表示完全同向）
- 只看方向，不看长度

**Python实现：**

```python
def cosine_similarity(a, b):
    dot_product = np.dot(a, b)
    norm_a = np.linalg.norm(a)
    norm_b = np.linalg.norm(b)
    return dot_product / (norm_a * norm_b)

vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 4.0, 6.0])  # vec_a的2倍
print(f"余弦相似度: {cosine_similarity(vec_a, vec_b):.4f}")  # 1.0000
```

**适用场景：** 文本语义检索（80%的场景推荐使用）

## 卡片4：内积（IP）

**一句话：** 内积测量向量的点积，同时考虑方向和长度

**数学公式：**

```
IP(A, B) = Σ(ai × bi)
```

**几何意义：** A在B上的投影长度 × B的长度

**特点：**
- 值域：(-∞, +∞)
- 值越大，越相似
- 考虑长度和方向

**与COSINE的关系：**

```python
# 归一化后：COSINE ≈ IP
vec_a_norm = vec_a / np.linalg.norm(vec_a)
vec_b_norm = vec_b / np.linalg.norm(vec_b)

cosine = cosine_similarity(vec_a_norm, vec_b_norm)
ip = np.dot(vec_a_norm, vec_b_norm)
print(f"COSINE: {cosine:.6f}, IP: {ip:.6f}")  # 完全相同
```

**适用场景：** 推荐系统（向量长度代表偏好强度）

## 卡片5：归一化的影响

**一句话：** 归一化消除向量长度的影响，使所有向量长度都变为1

**归一化公式：**

```
A_norm = A / ||A||
```

**关键关系：**

```
归一化后：
- COSINE ≈ IP（完全等价）
- L2 = √(2 - 2×COSINE)（有单调关系）
```

**实践意义：**

```python
# 检查Embedding模型输出是否归一化
embedding = model.encode("文本")
norm = np.linalg.norm(embedding)
print(f"向量长度: {norm:.6f}")

if abs(norm - 1.0) < 1e-6:
    print("已归一化 → 推荐使用COSINE")
else:
    print("未归一化 → 根据场景选择")
```

**应用：** 大多数文本Embedding模型输出已归一化，推荐使用COSINE

## 卡片6：二值向量度量（HAMMING）

**一句话：** HAMMING距离测量二值向量中不同位的数量

**数学定义：**

```
HAMMING(A, B) = Σ(ai ≠ bi)
```

**特点：**
- 只适用于二值向量（0/1）
- 位级比较，速度快
- 值越小，越相似

**Python实现：**

```python
# 二值向量
vec_a = np.array([1, 0, 1, 0], dtype=np.uint8)
vec_b = np.array([1, 1, 0, 0], dtype=np.uint8)

# HAMMING距离
hamming = np.sum(vec_a != vec_b)
print(f"HAMMING距离: {hamming}")  # 2（有2位不同）
```

**适用场景：** 图像哈希去重、DNA序列比较、二值特征匹配

## 卡片7：BM25全文检索

**一句话：** BM25是基于词频统计的全文检索算法，关注关键词精确匹配

**核心思想：**

```
BM25得分 = Σ(IDF × TF × 调节因子)

IDF：逆文档频率（稀有词权重高）
TF：词频（出现次数）
调节因子：考虑文档长度
```

**与向量检索的区别：**

| 维度 | BM25 | 向量检索 |
|-----|------|---------|
| 原理 | 词频统计 | 语义Embedding |
| 优势 | 精确匹配关键词 | 理解语义相似 |
| 劣势 | 无法理解语义 | 可能漏掉关键词 |

**2026年标准：** BM25 + 向量混合检索

**应用：** 文档检索、知识库搜索、智能客服

## 卡片8：混合检索的必要性

**一句话：** 混合检索结合BM25（精确匹配）和向量检索（语义理解），是2026年生产标准

**为什么需要混合检索？**

```
查询："Milvus向量数据库的性能优化"

文档1："Milvus性能优化指南"
- BM25：高（关键词匹配）
- 向量：高（语义相似）
→ 两者都认为相关 ✅

文档2："如何提升向量数据库的速度"
- BM25：低（缺少"Milvus"、"性能"）
- 向量：高（语义："提升速度" ≈ "性能优化"）
→ BM25漏掉了 ❌

文档3："Milvus向量数据库的安装教程"
- BM25：中（包含"Milvus"、"向量"）
- 向量：低（语义不相关）
→ BM25误判 ❌
```

**Milvus实现：**

```python
from pymilvus import AnnSearchRequest, RRFRanker

# 向量检索 + BM25
results = collection.hybrid_search(
    reqs=[vector_search, bm25_search],
    rerank=RRFRanker(),
    limit=10
)
```

**应用：** 所有生产环境的文本检索系统

## 卡片9：度量选型决策树

**一句话：** 根据数据类型、归一化状态和业务场景，系统化选择度量方式

**3步选型法：**

```
步骤1：检查数据类型
├─ 稠密向量（float） → 继续步骤2
├─ 二值向量（binary） → HAMMING/JACCARD
└─ 文本关键词 → BM25

步骤2：检查Embedding模型
├─ 输出已归一化 → COSINE（推荐）
└─ 输出未归一化 → 继续步骤3

步骤3：检查业务场景
├─ 语义检索（文档问答） → COSINE
├─ 推荐系统（偏好强度） → IP
└─ 图像检索（绝对位置） → L2
```

**快速记忆：**
- **不确定用什么？→ 用COSINE**
- **文本检索？→ 用COSINE**
- **推荐系统？→ 用IP**
- **图像去重？→ 用HAMMING**

**验证方法：** 基准测试对比不同度量方式的Recall和Precision

## 卡片10：Milvus中的度量配置

**一句话：** 在Milvus中，度量方式在创建索引时指定，一旦指定就固定了

**完整示例：**

```python
from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection

# 1. 连接Milvus
connections.connect("default", host="localhost", port="19530")

# 2. 定义Schema
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=128)
]
schema = CollectionSchema(fields, description="相似度检索")
collection = Collection(name="demo", schema=schema)

# 3. 创建索引（指定度量方式）
index_params = {
    "index_type": "HNSW",
    "metric_type": "COSINE",  # ← 关键：指定度量方式
    "params": {"M": 16, "efConstruction": 200}
}
collection.create_index(field_name="embedding", index_params=index_params)

# 4. 检索（使用固定的度量方式）
search_params = {"metric_type": "COSINE", "params": {"ef": 100}}
results = collection.search(
    data=[query_embedding],
    anns_field="embedding",
    param=search_params,
    limit=10
)
```

**重要提醒：**
- 度量方式在创建索引时固定
- 更换度量方式需要重建索引
- 不同度量方式的索引结构不兼容

**应用：** 所有Milvus向量检索场景

## 快速复习卡

### 核心公式速查

| 度量方式 | 公式 | 值域 | 相似性 |
|---------|------|------|--------|
| **L2** | √(Σ(ai-bi)²) | [0, +∞) | 越小越相似 |
| **COSINE** | (A·B)/(‖A‖×‖B‖) | [-1, 1] | 越大越相似 |
| **IP** | Σ(ai×bi) | (-∞, +∞) | 越大越相似 |
| **HAMMING** | Σ(ai≠bi) | [0, n] | 越小越相似 |

---

### 场景选型速查

| 场景 | 推荐度量 | 原因 |
|-----|---------|------|
| 文档问答 | COSINE | 语义相似，Embedding归一化 |
| 商品推荐 | IP | 考虑偏好强度 |
| 图像检索 | L2 | 绝对位置有意义 |
| 图像去重 | HAMMING | 二值哈希，快速比较 |
| 混合检索 | BM25+COSINE | 2026年标准 |

---

### 常见误区速查

| 误区 | 正确理解 |
|-----|---------|
| "归一化后L2=COSINE" | 归一化后COSINE≈IP，不是L2 |
| "度量可以随时更换" | 度量在创建索引时固定，更换需重建 |
| "BM25可以替代向量检索" | 两者互补，需要混合检索 |
| "向量维度越高越好" | 维度取决于模型，不是越高越好 |
