# 最小可用知识

掌握以下内容，就能开始使用 create_agent 构建智能代理。

> **核心原则：** 20% 的 API 解决 80% 的场景。create_agent 的设计哲学就是"简单的事情简单做"，
> 你只需要掌握 6 个用法，就能覆盖绝大多数 Agent 开发需求。

---

## 4.1 最简 Agent 创建（3 行代码）

**这是你需要记住的第一件事：创建一个 Agent 只需要 3 行代码。**

```python
from langchain.agents import create_agent

agent = create_agent("openai:gpt-4o", tools=[])
result = agent.invoke({"messages": [{"role": "user", "content": "你好"}]})
print(result["messages"][-1].content)
```

| 要素 | 说明 |
|------|------|
| `"openai:gpt-4o"` | 字符串格式的模型标识符，`provider:model_name` |
| `tools=[]` | 工具列表，空列表 = 纯对话 Agent |
| `agent.invoke()` | 同步调用，传入 messages 列表 |
| 返回值 | 包含 `messages` 的字典，最后一条是 AI 回复 |

### 模型标识符速查

```python
agent = create_agent("openai:gpt-4o", tools=[])       # OpenAI
agent = create_agent("openai:gpt-4o-mini", tools=[])   # OpenAI 轻量版
agent = create_agent("anthropic:claude-sonnet-4-20250514", tools=[])  # Anthropic

# 也可以直接传入模型实例（需要精细控制 temperature 等参数时）
from langchain_openai import ChatOpenAI
model = ChatOpenAI(model="gpt-4o", temperature=0)
agent = create_agent(model, tools=[])
```

> **注意输入格式：** `{"messages": [...]}`，不是旧版的 `{"input": "..."}`。
> `create_agent()` 返回 `CompiledStateGraph`，拥有 `.invoke()` / `.stream()` / `.ainvoke()` / `.astream()` 四个执行方法。

## 4.2 定义工具（函数即工具）

**普通 Python 函数就是工具。** 不需要继承基类，不需要装饰器，只要有类型注解和 docstring。

```python
def search_web(query: str) -> str:
    """Search the web for information.

    Args:
        query: The search query string.
    """
    return f"搜索结果：关于 '{query}' 的信息..."

def calculate(expression: str) -> str:
    """Calculate a mathematical expression.

    Args:
        expression: A valid Python math expression like '2 + 3 * 4'.
    """
    try:
        return str(eval(expression))
    except Exception as e:
        return f"计算错误: {e}"

# 多工具 Agent —— LLM 会自动选择合适的工具
agent = create_agent("openai:gpt-4o", tools=[search_web, calculate])
```

### 工具定义三要素

| 要素 | 作用 | 示例 |
|------|------|------|
| **类型注解** | 告诉 LLM 参数类型 | `query: str` |
| **docstring** | 告诉 LLM 工具用途 | `"""Search the web..."""` |
| **返回值** | 工具执行结果 | `return "结果..."` |

> **为什么 docstring 这么重要？** LLM 通过阅读 docstring 来决定"什么时候该调用这个工具"。
> docstring 写得越清晰，工具选择就越准确。

## 4.3 添加系统提示（System Prompt）

**系统提示定义了 Agent 的"人设"和行为边界。**

```python
# 好的系统提示：明确角色 + 行为约束 + 输出格式
agent = create_agent(
    "openai:gpt-4o",
    tools=[search_web, calculate],
    system_prompt="""你是一个数据分析助手。

## 你的职责
- 帮助用户分析数据、计算指标
- 需要实时信息时使用搜索工具

## 回答规范
- 使用中文回答
- 不确定时明确说明
""",
)

# 差的系统提示：太模糊
agent = create_agent(
    "openai:gpt-4o",
    system_prompt="你是一个助手。",  # 太笼统，LLM 不知道该怎么表现
)
```

`system_prompt` 支持三种格式：字符串（最常用）、`SystemMessage` 对象、或不传（使用模型默认行为）。

## 4.4 流式输出（Streaming）

**用户体验的关键：不要让用户盯着空白屏幕等待。**

```python
# stream_mode="updates" 是最常用的模式
for chunk in agent.stream(
    {"messages": [{"role": "user", "content": "什么是 RAG？"}]},
    stream_mode="updates",
):
    for node_name, node_output in chunk.items():
        print(f"[{node_name}]", node_output["messages"][-1].content[:100])
```

带工具调用时，chunk 会按执行顺序依次输出：agent 节点（决定调用工具） -> tools 节点（工具结果） -> agent 节点（最终回答）。

### stream_mode 对比

| stream_mode | 返回内容 | 适合场景 |
|-------------|----------|----------|
| `"updates"` | 每个节点的增量更新 | 最常用，展示 Agent 执行过程 |
| `"values"` | 每步的完整状态 | 需要完整状态快照时 |
| `"messages"` | token 级流式 | 聊天界面逐字显示（打字机效果） |

异步版本用 `agent.astream()`，适合 Web 应用（FastAPI 等）。

## 4.5 添加简单 Middleware

**Middleware 是 create_agent 的灵魂，但入门只需要掌握装饰器写法。**

```python
from langchain.agents.middleware import before_model, after_model


@before_model
def log_before(state, runtime):
    """在每次调用模型前打印日志"""
    print(f"即将调用模型，当前消息数: {len(state['messages'])}")


@after_model
def log_after(state, runtime):
    """在每次模型返回后打印日志"""
    last_msg = state["messages"][-1]
    if hasattr(last_msg, "tool_calls") and last_msg.tool_calls:
        print(f"调用工具: {[tc['name'] for tc in last_msg.tool_calls]}")
    elif hasattr(last_msg, "content") and last_msg.content:
        print(f"AI 回复: {last_msg.content[:50]}...")


agent = create_agent(
    "openai:gpt-4o",
    tools=[search_web],
    middleware=[log_before, log_after],
)
```

### 装饰器速查表

| 装饰器 | 触发时机 | 典型用途 |
|--------|----------|----------|
| `@before_model` | 调用 LLM 之前 | 日志、输入验证、消息过滤 |
| `@after_model` | LLM 返回之后 | 日志、输出过滤、结果缓存 |
| `@before_agent` | Agent 开始执行前 | 初始化、权限检查 |
| `@after_agent` | Agent 执行完毕后 | 清理、统计、审计 |

### 常用内置 Middleware

```python
from langchain.agents.middleware import (
    ModelCallLimitMiddleware,   # 限制最多调用模型 N 次（防止无限循环）
    ModelRetryMiddleware,       # 模型调用失败时自动重试
    HumanInTheLoopMiddleware,   # 工具调用前需要人工确认
)

agent = create_agent(
    "openai:gpt-4o",
    tools=[search_web],
    middleware=[
        ModelCallLimitMiddleware(max_calls=10),
        ModelRetryMiddleware(max_retries=3),
    ],
)
```

## 4.6 持久化对话（Checkpointer）

**让 Agent 记住之前的对话，实现多轮交互。**

```python
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    "openai:gpt-4o",
    tools=[search_web],
    checkpointer=InMemorySaver(),  # 生产环境换 PostgresSaver
)

# 使用 thread_id 维持对话
config = {"configurable": {"thread_id": "user-123"}}
result1 = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫 Alice"}]}, config,
)
result2 = agent.invoke(
    {"messages": [{"role": "user", "content": "我叫什么名字？"}]}, config,
)
# Agent 会记住 "Alice"，因为 checkpointer 保存了对话状态
```

同一个 `thread_id` 下的所有调用共享对话历史。不同 `thread_id` 互相隔离。

## 4.7 完整示例：RAG 知识库助手

**把上面 6 个知识点组合起来，构建一个完整的 Agent：**

```python
from langchain.agents import create_agent
from langchain.agents.middleware import before_model, after_model, ModelRetryMiddleware
from langgraph.checkpoint.memory import InMemorySaver


# 1. 定义工具
def search_knowledge_base(query: str) -> str:
    """Search the internal knowledge base for relevant documents.

    Args:
        query: Natural language search query.
    """
    knowledge = {
        "RAG": "RAG 通过检索外部知识增强 LLM 生成，解决幻觉和知识过时问题。",
        "Embedding": "Embedding 将文本映射到高维向量空间，语义相近的文本距离更近。",
    }
    for key, value in knowledge.items():
        if key.lower() in query.lower():
            return value
    return "未找到相关文档。"


# 2. 定义 Middleware
@before_model
def log_request(state, runtime):
    user_msgs = [m for m in state["messages"] if hasattr(m, "type") and m.type == "human"]
    if user_msgs:
        print(f"[用户] {user_msgs[-1].content[:60]}")

@after_model
def log_response(state, runtime):
    last = state["messages"][-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        print(f"[工具] {[tc['name'] for tc in last.tool_calls]}")
    elif hasattr(last, "content") and last.content:
        print(f"[回复] {last.content[:60]}...")


# 3. 创建 Agent
agent = create_agent(
    "openai:gpt-4o",
    tools=[search_knowledge_base],
    system_prompt="""你是一个 RAG 技术知识库助手。
优先使用知识库搜索工具获取准确信息。使用中文，简洁专业。""",
    middleware=[log_request, log_response, ModelRetryMiddleware(max_retries=2)],
    checkpointer=InMemorySaver(),
)


# 4. 多轮对话
config = {"configurable": {"thread_id": "demo_001"}}

result = agent.invoke(
    {"messages": [{"role": "user", "content": "什么是 RAG？"}]}, config,
)
print(result["messages"][-1].content)

result = agent.invoke(
    {"messages": [{"role": "user", "content": "它和 Embedding 有什么关系？"}]}, config,
)
print(result["messages"][-1].content)
```
