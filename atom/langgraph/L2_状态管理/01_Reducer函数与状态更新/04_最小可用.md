## 示例 1：最简单的 Reducer（5 分钟入门）

### 目标
理解 Reducer 的最基本用法：列表追加

### 完整代码

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import operator

# 1. 定义状态 Schema
class State(TypedDict):
    # 使用 operator.add 作为 Reducer
    messages: Annotated[list[str], operator.add]

# 2. 定义节点函数
def node_a(state: State) -> dict:
    """节点 A：返回一条消息"""
    print(f"节点 A 收到的状态: {state}")
    return {"messages": ["Hello from A"]}

def node_b(state: State) -> dict:
    """节点 B：返回另一条消息"""
    print(f"节点 B 收到的状态: {state}")
    return {"messages": ["Hello from B"]}

# 3. 构建图
builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)

# 4. 添加边
builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", END)

# 5. 编译图
graph = builder.compile()

# 6. 执行图
result = graph.invoke({"messages": []})

# 7. 查看结果
print("\n最终状态:")
print(result)
# {'messages': ['Hello from A', 'Hello from B']}
```

### 执行输出

```
节点 A 收到的状态: {'messages': []}
节点 B 收到的状态: {'messages': ['Hello from A']}

最终状态:
{'messages': ['Hello from A', 'Hello from B']}
```

### 关键点解析

1. **Annotated[list[str], operator.add]**
   - `list[str]`: 字段类型是字符串列表
   - `operator.add`: Reducer 函数，用于合并列表

2. **节点返回部分状态**
   - 节点只返回 `{"messages": [...]}`
   - 不需要返回完整状态

3. **自动合并**
   - 节点 A 返回 `["Hello from A"]`
   - 节点 B 返回 `["Hello from B"]`
   - Reducer 自动合并：`[] + ["Hello from A"] + ["Hello from B"]`

## 示例 2：对比有无 Reducer（理解核心价值）

### 场景：两个节点都更新同一个字段

#### 2.1 无 Reducer（直接替换）

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 状态定义：无 Reducer
class StateWithoutReducer(TypedDict):
    messages: list[str]  # 无 Reducer

def node_a(state: StateWithoutReducer) -> dict:
    return {"messages": ["A"]}

def node_b(state: StateWithoutReducer) -> dict:
    return {"messages": ["B"]}

# 构建图
builder = StateGraph(StateWithoutReducer)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", END)

graph = builder.compile()

# 执行
result = graph.invoke({"messages": []})
print("无 Reducer 结果:", result)
# {'messages': ['B']}  # 节点 B 的值覆盖了节点 A 的值
```

#### 2.2 有 Reducer（自动合并）

```python
from typing import Annotated
import operator

# 状态定义：有 Reducer
class StateWithReducer(TypedDict):
    messages: Annotated[list[str], operator.add]  # 有 Reducer

def node_a(state: StateWithReducer) -> dict:
    return {"messages": ["A"]}

def node_b(state: StateWithReducer) -> dict:
    return {"messages": ["B"]}

# 构建图
builder = StateGraph(StateWithReducer)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", END)

graph = builder.compile()

# 执行
result = graph.invoke({"messages": []})
print("有 Reducer 结果:", result)
# {'messages': ['A', 'B']}  # 自动合并
```

### 对比总结

| 特性 | 无 Reducer | 有 Reducer |
|------|-----------|-----------|
| 行为 | 直接替换 | 自动合并 |
| 节点 A 返回 | `["A"]` | `["A"]` |
| 节点 B 返回 | `["B"]` | `["B"]` |
| 最终结果 | `["B"]` | `["A", "B"]` |
| 数据丢失 | 是（丢失 A） | 否 |

## 示例 3：使用 add_messages（聊天机器人场景）

### 目标
使用内置的 `add_messages` Reducer 管理对话历史

### 完整代码

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_core.messages import HumanMessage, AIMessage

# 1. 定义状态
class ChatState(TypedDict):
    messages: Annotated[list, add_messages]

# 2. 定义聊天节点
def chatbot(state: ChatState) -> dict:
    """模拟 AI 回复"""
    # 获取最后一条用户消息
    last_message = state["messages"][-1]
    user_input = last_message.content
    
    # 生成 AI 回复
    ai_response = f"你说了：{user_input}。我理解了！"
    
    return {"messages": [AIMessage(content=ai_response)]}

# 3. 构建图
builder = StateGraph(ChatState)
builder.add_node("chatbot", chatbot)
builder.add_edge(START, "chatbot")
builder.add_edge("chatbot", END)

graph = builder.compile()

# 4. 第一轮对话
result1 = graph.invoke({
    "messages": [HumanMessage(content="你好")]
})

print("第一轮对话:")
for msg in result1["messages"]:
    print(f"{msg.__class__.__name__}: {msg.content}")

# 5. 第二轮对话（基于第一轮的结果）
result2 = graph.invoke({
    "messages": result1["messages"] + [HumanMessage(content="今天天气怎么样？")]
})

print("\n第二轮对话:")
for msg in result2["messages"]:
    print(f"{msg.__class__.__name__}: {msg.content}")
```

### 执行输出

```
第一轮对话:
HumanMessage: 你好
AIMessage: 你说了：你好。我理解了！

第二轮对话:
HumanMessage: 你好
AIMessage: 你说了：你好。我理解了！
HumanMessage: 今天天气怎么样？
AIMessage: 你说了：今天天气怎么样？。我理解了！
```

### 关键点

1. **add_messages 的特性**
   - 自动管理消息列表
   - 按 ID 合并（相同 ID 的消息会被替换）
   - 支持 LangChain 的消息类型

2. **对话历史累积**
   - 每次调用都会累积消息
   - 支持多轮对话

## 示例 4：自定义 Reducer（去重列表）

### 目标
实现一个自定义 Reducer，追加列表时自动去重

### 完整代码

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义自定义 Reducer
def unique_list_reducer(left: list, right: list) -> list:
    """
    自定义 Reducer：合并列表并去重
    
    Args:
        left: 旧值（当前状态中的列表）
        right: 新值（节点返回的列表）
    
    Returns:
        合并后的去重列表
    """
    # 合并两个列表
    combined = left + right
    
    # 去重（保持顺序）
    seen = set()
    result = []
    for item in combined:
        if item not in seen:
            seen.add(item)
            result.append(item)
    
    return result

# 2. 定义状态
class State(TypedDict):
    items: Annotated[list[str], unique_list_reducer]

# 3. 定义节点
def node_a(state: State) -> dict:
    return {"items": ["apple", "banana"]}

def node_b(state: State) -> dict:
    return {"items": ["banana", "cherry"]}  # banana 重复

def node_c(state: State) -> dict:
    return {"items": ["cherry", "date"]}  # cherry 重复

# 4. 构建图
builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_node("c", node_c)

builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", "c")
builder.add_edge("c", END)

graph = builder.compile()

# 5. 执行
result = graph.invoke({"items": []})

print("最终结果（已去重）:")
print(result)
# {'items': ['apple', 'banana', 'cherry', 'date']}
```

### 执行输出

```
最终结果（已去重）:
{'items': ['apple', 'banana', 'cherry', 'date']}
```

### 关键点

1. **Reducer 函数签名**
   - 必须接收 2 个参数：`(left, right)`
   - 必须返回 1 个值

2. **去重逻辑**
   - 使用 `set` 跟踪已见过的元素
   - 保持元素的原始顺序

3. **自动调用**
   - 框架在每次状态更新时自动调用 Reducer
   - 无需手动管理去重逻辑

## 示例 5：混合使用 Reducer 和非 Reducer 字段

### 目标
在同一个状态中同时使用有 Reducer 和无 Reducer 的字段

### 完整代码

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import operator

# 1. 定义状态（混合字段）
class MixedState(TypedDict):
    # 有 Reducer：累积消息
    messages: Annotated[list[str], operator.add]
    
    # 有 Reducer：累加计数
    count: Annotated[int, operator.add]
    
    # 无 Reducer：直接替换状态
    status: str

# 2. 定义节点
def node_a(state: MixedState) -> dict:
    return {
        "messages": ["A started"],
        "count": 1,
        "status": "processing"
    }

def node_b(state: MixedState) -> dict:
    return {
        "messages": ["B processing"],
        "count": 2,
        "status": "processing"  # 保持不变
    }

def node_c(state: MixedState) -> dict:
    return {
        "messages": ["C completed"],
        "count": 3,
        "status": "completed"  # 更新状态
    }

# 3. 构建图
builder = StateGraph(MixedState)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_node("c", node_c)

builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", "c")
builder.add_edge("c", END)

graph = builder.compile()

# 4. 执行
result = graph.invoke({
    "messages": [],
    "count": 0,
    "status": "init"
})

print("最终结果:")
print(f"messages: {result['messages']}")
print(f"count: {result['count']}")
print(f"status: {result['status']}")
```

### 执行输出

```
最终结果:
messages: ['A started', 'B processing', 'C completed']
count: 6
status: completed
```

### 关键点

1. **messages 字段（有 Reducer）**
   - 使用 `operator.add` 合并
   - 结果：`['A started', 'B processing', 'C completed']`

2. **count 字段（有 Reducer）**
   - 使用 `operator.add` 累加
   - 结果：`0 + 1 + 2 + 3 = 6`

3. **status 字段（无 Reducer）**
   - 直接替换
   - 结果：`'completed'`（最后一个节点的值）

## 常见问题

### Q1: 第一个值会调用 Reducer 吗？

**A**: 不会。第一个值直接赋值，不调用 Reducer。

```python
# 初始状态: {"items": []}
# 节点 A 返回: {"items": [1]}
# 结果: {"items": [1]}  # 直接赋值，不调用 Reducer

# 节点 B 返回: {"items": [2]}
# 结果: {"items": [1, 2]}  # 调用 Reducer: [1] + [2]
```

### Q2: 如何处理 None 值？

**A**: 在 Reducer 中添加 None 检查。

```python
def safe_reducer(old: list | None, new: list | None) -> list:
    if old is None:
        old = []
    if new is None:
        new = []
    return old + new
```

### Q3: Reducer 可以修改输入值吗？

**A**: 不推荐。Reducer 应该是纯函数。

```python
# ❌ 错误：修改输入
def bad_reducer(old: list, new: list) -> list:
    old.extend(new)  # 修改了 old
    return old

# ✅ 正确：创建新值
def good_reducer(old: list, new: list) -> list:
    return old + new  # 创建新列表
```
