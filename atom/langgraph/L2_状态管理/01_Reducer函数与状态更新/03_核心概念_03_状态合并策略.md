## 1. 概念定义

### 什么是状态合并策略？

**状态合并策略是 LangGraph 中定义如何聚合多个节点返回的状态更新值的规则和机制。**

当多个节点同时或顺序更新同一个状态字段时，LangGraph 需要决定如何合并这些更新值。状态合并策略就是这个决策的规则集合。

### 核心特征

1. **自动聚合**: 框架自动合并多个节点的返回值
2. **Reducer 驱动**: 使用 Reducer 函数定义合并逻辑
3. **顺序保证**: 合并顺序遵循图的执行顺序
4. **初始值特殊处理**: 第一个值直接赋值，不调用 Reducer

## 2. 合并场景

### 2.1 顺序执行合并

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph
import operator

class State(TypedDict):
    items: Annotated[list, operator.add]

def node_a(state: State) -> dict:
    return {"items": [1, 2]}

def node_b(state: State) -> dict:
    return {"items": [3, 4]}

# 构建图：A -> B
builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_edge("__start__", "a")
builder.add_edge("a", "b")
builder.add_edge("b", "__end__")

graph = builder.compile()

# 执行
result = graph.invoke({"items": []})
print(result)
# {'items': [1, 2, 3, 4]}
```

**合并流程**:
```
初始状态: {"items": []}
  ↓
node_a 返回: {"items": [1, 2]}
  ↓
合并: [] + [1, 2] = [1, 2]  (第一个值直接赋值)
  ↓
node_b 返回: {"items": [3, 4]}
  ↓
合并: [1, 2] + [3, 4] = [1, 2, 3, 4]  (调用 Reducer)
  ↓
最终状态: {"items": [1, 2, 3, 4]}
```

### 2.2 并行执行合并

```python
from langgraph.graph import StateGraph
import operator

class State(TypedDict):
    items: Annotated[list, operator.add]

def node_a(state: State) -> dict:
    return {"items": [1, 2]}

def node_b(state: State) -> dict:
    return {"items": [3, 4]}

def node_c(state: State) -> dict:
    return {"items": [5, 6]}

# 构建图：A, B, C 并行执行
builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_node("c", node_c)
builder.add_edge("__start__", "a")
builder.add_edge("__start__", "b")
builder.add_edge("__start__", "c")
builder.add_edge("a", "__end__")
builder.add_edge("b", "__end__")
builder.add_edge("c", "__end__")

graph = builder.compile()

# 执行
result = graph.invoke({"items": []})
print(result)
# {'items': [1, 2, 3, 4, 5, 6]}  (顺序可能不同)
```

**合并流程**:
```
初始状态: {"items": []}
  ↓
node_a, node_b, node_c 并行执行
  ↓
收集返回值: [[1, 2], [3, 4], [5, 6]]
  ↓
第一个值直接赋值: [1, 2]
  ↓
应用 Reducer: [1, 2] + [3, 4] = [1, 2, 3, 4]
  ↓
应用 Reducer: [1, 2, 3, 4] + [5, 6] = [1, 2, 3, 4, 5, 6]
  ↓
最终状态: {"items": [1, 2, 3, 4, 5, 6]}
```

## 3. Reducer 调用时机

### 3.1 调用规则

**Reducer 在以下情况被调用**:
1. 当前状态字段已有值（不是 MISSING）
2. 节点返回该字段的新值
3. 新值不是 Overwrite

**Reducer 不被调用的情况**:
1. 第一个更新值（当前值是 MISSING）
2. 新值是 Overwrite

### 3.2 调用时机示例

```python
import operator

class State(TypedDict):
    items: Annotated[list, operator.add]

# 场景 1: 第一个值
channel = BinaryOperatorAggregate(list, operator.add)
channel.value = MISSING

channel.update([[1, 2]])
# Reducer 不被调用，直接赋值: channel.value = [1, 2]

# 场景 2: 后续值
channel.update([[3, 4]])
# Reducer 被调用: channel.value = operator.add([1, 2], [3, 4]) = [1, 2, 3, 4]

# 场景 3: Overwrite
from langgraph.types import Overwrite
channel.update([Overwrite([5, 6])])
# Reducer 不被调用，直接覆盖: channel.value = [5, 6]
```

## 4. 初始值处理

### 4.1 类型实例化

```python
# 可实例化的类型
list()    # []
dict()    # {}
set()     # set()
int()     # 0
str()     # ""

# 无法实例化的类型
class CustomType:
    def __init__(self, required_arg):
        pass

CustomType()  # TypeError -> 标记为 MISSING
```

### 4.2 MISSING 标记

```python
# 定义
MISSING = object()

# 使用
if channel.value is MISSING:
    # 第一个值直接赋值
    channel.value = values[0]
else:
    # 后续值使用 Reducer
    channel.value = reducer(channel.value, values[0])
```

### 4.3 初始值策略

```python
from typing import Annotated
import operator

class State(TypedDict):
    # 策略 1: 类型可实例化，自动创建空值
    items: Annotated[list, operator.add]  # 初始值: []

    # 策略 2: 显式提供初始值
    counter: Annotated[int, operator.add]  # 初始值: 0

    # 策略 3: 无法实例化，标记为 MISSING
    custom: Annotated[CustomType, my_reducer]  # 初始值: MISSING

# 执行
result = graph.invoke({"items": [], "counter": 0})
# items: [] -> 已初始化
# counter: 0 -> 已初始化
# custom: MISSING -> 未初始化
```

## 5. 合并顺序

### 5.1 顺序执行

```python
# 图结构: A -> B -> C
# 合并顺序: A, B, C

# 执行流程
初始值: []
  ↓
A 返回: [1]
合并: [] + [1] = [1]  (第一个值直接赋值)
  ↓
B 返回: [2]
合并: [1] + [2] = [1, 2]  (调用 Reducer)
  ↓
C 返回: [3]
合并: [1, 2] + [3] = [1, 2, 3]  (调用 Reducer)
  ↓
最终值: [1, 2, 3]
```

### 5.2 并行执行

```python
# 图结构: A, B, C 并行
# 合并顺序: 不确定（取决于执行完成顺序）

# 可能的执行流程 1
初始值: []
  ↓
A 完成: [1]
合并: [] + [1] = [1]  (第一个值直接赋值)
  ↓
B 完成: [2]
合并: [1] + [2] = [1, 2]  (调用 Reducer)
  ↓
C 完成: [3]
合并: [1, 2] + [3] = [1, 2, 3]  (调用 Reducer)
  ↓
最终值: [1, 2, 3]

# 可能的执行流程 2
初始值: []
  ↓
B 完成: [2]
合并: [] + [2] = [2]  (第一个值直接赋值)
  ↓
A 完成: [1]
合并: [2] + [1] = [2, 1]  (调用 Reducer)
  ↓
C 完成: [3]
合并: [2, 1] + [3] = [2, 1, 3]  (调用 Reducer)
  ↓
最终值: [2, 1, 3]
```

**注意**: 并行执行时，合并顺序不确定。如果需要确定的顺序，使用顺序执行或自定义 Reducer。

## 6. 不同 Reducer 的合并行为

### 6.1 operator.add (列表)

```python
import operator

class State(TypedDict):
    items: Annotated[list, operator.add]

# 合并行为: 列表拼接
[1, 2] + [3, 4] = [1, 2, 3, 4]
```

### 6.2 operator.or_ (字典)

```python
import operator

class State(TypedDict):
    config: Annotated[dict, operator.or_]

# 合并行为: 字典合并，后者覆盖前者
{"a": 1, "b": 2} | {"b": 3, "c": 4} = {"a": 1, "b": 3, "c": 4}
```

### 6.3 add_messages (消息列表)

```python
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list, add_messages]

# 合并行为: 按 ID 合并，相同 ID 的消息会被替换
[
    HumanMessage(id="1", content="A"),
    AIMessage(id="2", content="B")
] + [
    HumanMessage(id="1", content="C"),  # 替换 ID="1" 的消息
    AIMessage(id="3", content="D")      # 追加新消息
] = [
    HumanMessage(id="1", content="C"),
    AIMessage(id="2", content="B"),
    AIMessage(id="3", content="D")
]
```

### 6.4 自定义 Reducer

```python
def max_reducer(old: int, new: int) -> int:
    """保留较大的值"""
    return max(old, new)

class State(TypedDict):
    max_score: Annotated[int, max_reducer]

# 合并行为: 保留最大值
max(5, 3) = 5
max(5, 8) = 8
```

## 8. 常见问题

### Q1: 并行执行时如何保证合并顺序？

**A**: 使用自定义 Reducer 或添加排序逻辑。

```python
def sorted_list_reducer(old: list, new: list) -> list:
    """合并后排序"""
    result = old + new
    return sorted(result, key=lambda x: x.get("order", 0))

class State(TypedDict):
    items: Annotated[list, sorted_list_reducer]
```

### Q2: 如何避免重复值？

**A**: 使用自定义 Reducer 去重。

```python
def unique_list_reducer(old: list, new: list) -> list:
    """去重追加"""
    result = old.copy()
    for item in new:
        if item not in result:
            result.append(item)
    return result

class State(TypedDict):
    unique_items: Annotated[list, unique_list_reducer]
```

### Q3: 如何处理冲突的更新？

**A**: 使用 Overwrite 或自定义冲突解决策略。

```python
# 方式 1: 使用 Overwrite
from langgraph.types import Overwrite
return {"value": Overwrite(new_value)}

# 方式 2: 自定义冲突解决
def conflict_resolver(old: dict, new: dict) -> dict:
    """自定义冲突解决策略"""
    result = old.copy()
    for key, value in new.items():
        if key in result:
            # 冲突解决逻辑
            result[key] = resolve_conflict(result[key], value)
        else:
            result[key] = value
    return result
```

### Q4: 第一个值为什么不调用 Reducer？

**A**: 因为 Reducer 需要两个参数（旧值和新值），如果没有旧值，就无法调用 Reducer。

```python
# 场景
channel.value = MISSING  # 没有旧值

# 如果调用 Reducer
operator.add(???, [1, 2])  # 第一个参数是什么？

# 解决方案：第一个值直接赋值
channel.value = [1, 2]  # 不调用 Reducer
```

## 9. 最佳实践

### 9.1 选择合适的 Reducer

```python
# ✅ 列表追加：使用 operator.add
items: Annotated[list, operator.add]

# ✅ 字典合并：使用 operator.or_
config: Annotated[dict, operator.or_]

# ✅ 消息列表：使用 add_messages
messages: Annotated[list, add_messages]

# ✅ 数值累加：使用 operator.add
total: Annotated[int, operator.add]
```

### 9.2 处理并行执行

```python
# ❌ 坏：依赖合并顺序
def bad_reducer(old: list, new: list) -> list:
    # 假设 new 总是在 old 之后
    return old + new

# ✅ 好：不依赖合并顺序
def good_reducer(old: list, new: list) -> list:
    # 合并后排序
    result = old + new
    return sorted(result, key=lambda x: x["timestamp"])
```

### 9.3 避免副作用

```python
# ❌ 坏：修改输入值
def bad_reducer(old: list, new: list) -> list:
    old.extend(new)  # 修改了 old
    return old

# ✅ 好：创建新值
def good_reducer(old: list, new: list) -> list:
    return old + new  # 创建新列表
```

### 9.4 提供初始值

```python
# ✅ 好：显式提供初始值
result = graph.invoke({
    "items": [],
    "config": {},
    "counter": 0
})

# ❌ 坏：依赖自动初始化
result = graph.invoke({})  # 可能导致 MISSING
```

## 10. 与前端开发的类比

### Redux 状态合并

**LangGraph 状态合并** 类似于 **Redux 的 combineReducers**：

| LangGraph | Redux |
|-----------|-------|
| 多个节点更新同一字段 | 多个 action 更新同一状态 |
| Reducer 合并值 | Reducer 合并状态 |
| 顺序/并行执行 | 同步/异步 dispatch |
| Overwrite 覆盖 | 直接设置状态 |

```python
# LangGraph
class State(TypedDict):
    items: Annotated[list, operator.add]

# Redux (JavaScript)
const reducer = (state = [], action) => {
    switch (action.type) {
        case 'ADD_ITEMS':
            return [...state, ...action.payload];
        default:
            return state;
    }
};
```

## 11. 总结

**状态合并策略是 LangGraph 状态管理的核心机制**：

1. **自动聚合**: 框架自动合并多个节点的返回值
2. **Reducer 驱动**: 使用 Reducer 函数定义合并逻辑
3. **初始值特殊处理**: 第一个值直接赋值，不调用 Reducer
4. **顺序保证**: 顺序执行时合并顺序确定，并行执行时不确定

**关键要点**:
- 理解 Reducer 的调用时机
- 掌握初始值处理策略
- 处理并行执行的不确定性
- 选择合适的 Reducer 函数
