# 03_核心概念_3 - 递归限制与安全机制

> LangGraph 循环的安全阀：防止无限循环的多层防护体系

---

## 概念定义

**递归限制（recursion_limit）是 LangGraph 防止无限循环的核心安全机制，通过限制图执行的最大超级步数（supersteps），配合 `IsLastStep`、`RemainingSteps` 等管理值实现主动式安全控制。**

LangGraph 的循环能力是其核心优势，但也带来了无限循环的风险。框架提供了从被动（抛异常）到主动（感知剩余步数）的完整安全体系：

1. `recursion_limit` — 硬性上限，超出即报错
2. `GraphRecursionError` — 超限时抛出的异常
3. `IsLastStep` — 节点感知"这是最后一步吗"
4. `RemainingSteps` — 节点感知"还剩几步"

---

## recursion_limit 配置

### 基本用法

`recursion_limit` 通过 `config` 字典传入，控制图可以执行的最大超级步数。

```python
# 默认值为 25
graph.invoke(inputs)

# 自定义递归限制
graph.invoke(inputs, config={"recursion_limit": 10})

# 流式处理同样支持
for event in graph.stream(inputs, config={"recursion_limit": 50}):
    print(event)
```

### 源码实现：stop 的计算

源码位置：`_loop.py` L1119-L1120（`SyncPregelLoop.__enter__`）

```python
# _loop.py L1119-L1120
self.step = self.checkpoint_metadata["step"] + 1
self.stop = self.step + self.config["recursion_limit"] + 1
```

**关键公式：`stop = step + recursion_limit + 1`**

这意味着实际允许执行的超级步数 = `recursion_limit + 1`（包含 input step）。

### step 与 stop 的关系图解

```
假设 recursion_limit = 5，从 checkpoint step=0 开始：

初始化：step = 0 + 1 = 1, stop = 1 + 5 + 1 = 7

step=1: tick() → 1 > 7? No  → 执行任务 → after_tick() → step=2
step=2: tick() → 2 > 7? No  → 执行任务 → after_tick() → step=3
step=3: tick() → 3 > 7? No  → 执行任务 → after_tick() → step=4
step=4: tick() → 4 > 7? No  → 执行任务 → after_tick() → step=5
step=5: tick() → 5 > 7? No  → 执行任务 → after_tick() → step=6
step=6: tick() → 6 > 7? No  → 执行任务 → after_tick() → step=7
step=7: tick() → 7 > 7? No  → 执行任务 → after_tick() → step=8
step=8: tick() → 8 > 7? Yes → status="out_of_steps" → GraphRecursionError!

实际执行了 7 个 superstep（step 1~7），即 recursion_limit + 1 = 6 个 superstep
（第一个 step 是 input 处理）
```

### tick() 中的限制检查

源码位置：`_loop.py` L459-L536

```python
# _loop.py L459
def tick(self) -> bool:
    """Execute a single iteration of the Pregel loop.
    Returns: True if more iterations are needed.
    """
    # 1. 检查是否超出步数限制
    if self.step > self.stop:
        self.status = "out_of_steps"
        return False

    # 2. 准备下一批任务
    self.tasks = prepare_next_tasks(
        self.checkpoint,
        self.checkpoint_pending_writes,
        self.nodes,
        self.channels,
        self.managed,
        self.config,
        self.step,
        self.stop,
        for_execution=True,
    )

    # 3. 没有任务则正常结束
    if not self.tasks:
        self.status = "done"
        return False

    return True
```

`tick()` 返回 `False` 有两种情况：
- `status = "done"` — 正常结束（没有待执行的任务）
- `status = "out_of_steps"` — 超出限制（触发 `GraphRecursionError`）

## GraphRecursionError

### 定义与继承关系

源码位置：`errors.py` L45-L65

```python
# errors.py L45
class GraphRecursionError(RecursionError):
    """Raised when the graph has exhausted the maximum number of steps.

    This prevents infinite loops. To increase the maximum number of steps,
    run your graph with a config specifying a higher `recursion_limit`.

    Examples:
        graph = builder.compile()
        graph.invoke(
            {"messages": [("user", "Hello, world!")]},
            {"recursion_limit": 1000},
        )
    """
    pass
```

**继承链：** `GraphRecursionError` → `RecursionError`（Python 内置）→ `RuntimeError` → `Exception`

### 触发时机

在主循环中，当 `tick()` 返回 `False` 且 `status == "out_of_steps"` 时触发：

```python
# main.py L2643-L2674（简化）
while loop.tick():
    # 执行任务...
    loop.after_tick()

# 循环结束后检查状态
if loop.status == "out_of_steps":
    msg = create_error_message(
        message=(
            f"Recursion limit of {config['recursion_limit']} reached "
            "without hitting a stop condition. You can increase the "
            "limit by setting the `recursion_limit` config key."
        ),
        error_code=ErrorCode.GRAPH_RECURSION_LIMIT,
    )
    raise GraphRecursionError(msg)
```

### 捕获与处理

```python
"""
被动式处理：捕获 GraphRecursionError
"""
from langgraph.errors import GraphRecursionError

try:
    result = graph.invoke(
        {"messages": ["开始处理"]},
        config={"recursion_limit": 5},
    )
    print(f"正常完成: {result}")
except GraphRecursionError as e:
    print(f"递归限制触发: {e}")
    # 可以在这里执行降级逻辑
    # 但此时图的状态已经丢失（除非使用了 checkpointer）
```

**注意：** 被动捕获 `GraphRecursionError` 是不推荐的做法，因为：
1. 异常抛出时，当前执行的中间状态可能丢失
2. 无法在图内部执行优雅的收尾逻辑
3. 用户体验差 — 看到的是错误而非结果

## IsLastStep 管理值

### 定义

源码位置：`is_last_step.py` L1-L15

```python
# is_last_step.py
class IsLastStepManager(ManagedValue[bool]):
    @staticmethod
    def get(scratchpad: PregelScratchpad) -> bool:
        return scratchpad.step == scratchpad.stop - 1

IsLastStep = Annotated[bool, IsLastStepManager]
```

`IsLastStep` 返回 `True` 当且仅当当前步是最后一个允许执行的步。

### 使用方式

通过类型注解将 `IsLastStep` 注入到状态中：

```python
from langgraph.managed.is_last_step import IsLastStep

class State(TypedDict):
    messages: list[str]
    is_last_step: IsLastStep  # 自动注入，无需手动设置

def my_node(state: State):
    if state["is_last_step"]:
        # 这是最后一步了，必须给出最终答案
        return {"messages": ["强制最终答案"]}
    # 正常处理
    return {"messages": ["继续思考..."]}
```

### 计算逻辑

```
假设 recursion_limit=5, 初始 step=1, stop=7

step=1: is_last_step = (1 == 7-1) = (1 == 6) = False
step=2: is_last_step = (2 == 6) = False
...
step=6: is_last_step = (6 == 6) = True  ← 最后一步！
step=7: is_last_step = (7 == 6) = False  ← 但这一步已经不会执行了
                                           因为 tick() 检查 7 > 7 为 False
```

**注意：** `IsLastStep` 只在 `step == stop - 1` 时为 `True`，这是最后一个可以执行节点逻辑的步。

## RemainingSteps 管理值

### 定义

源码位置：`is_last_step.py` L17-L24

```python
# is_last_step.py
class RemainingStepsManager(ManagedValue[int]):
    @staticmethod
    def get(scratchpad: PregelScratchpad) -> int:
        return scratchpad.stop - scratchpad.step

RemainingSteps = Annotated[int, RemainingStepsManager]
```

`RemainingSteps` 返回从当前步到停止点的剩余步数。

### PregelScratchpad

`RemainingSteps` 和 `IsLastStep` 都依赖 `PregelScratchpad`，它是执行引擎的内部状态：

```python
# _scratchpad.py L8
@dataclasses.dataclass
class PregelScratchpad:
    step: int                              # 当前步数
    stop: int                              # 最大步数
    call_counter: Callable[[], int]        # 调用计数器
    interrupt_counter: Callable[[], int]   # 中断计数器
    get_null_resume: Callable[[bool], Any]
    resume: list[Any]
    subgraph_counter: Callable[[], int]    # 子图计数器
```

### 使用方式与计算示例

```python
from langgraph.managed.is_last_step import RemainingSteps

class State(TypedDict):
    messages: list[str]
    remaining_steps: RemainingSteps  # 自动注入

def my_node(state: State):
    remaining = state["remaining_steps"]
    print(f"剩余步数: {remaining}")

    if remaining <= 2:
        return {"messages": ["步数不足，返回当前结果"]}
    return {"messages": ["继续处理..."]}
```

**计算示例：**
```
假设 recursion_limit=5, 初始 step=1, stop=7

step=1: remaining = 7 - 1 = 6
step=2: remaining = 7 - 2 = 5
step=3: remaining = 7 - 3 = 4
step=4: remaining = 7 - 4 = 3
step=5: remaining = 7 - 5 = 2  ← 推荐在这里开始降级
step=6: remaining = 7 - 6 = 1  ← IsLastStep = True
step=7: remaining = 7 - 7 = 0  ← tick() 返回 False，不执行
```

### 完整示例：RemainingSteps 驱动的循环

```python
"""
使用 RemainingSteps 实现自适应循环
循环会根据剩余步数自动调整行为
"""
import operator
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.managed.is_last_step import RemainingSteps


class State(TypedDict):
    aggregate: Annotated[list, operator.add]
    remaining_steps: RemainingSteps


def node_a(state: State):
    remaining = state["remaining_steps"]
    print(f'Node A: aggregate={state["aggregate"]}, remaining={remaining}')
    return {"aggregate": ["A"]}


def node_b(state: State):
    print(f'Node B: aggregate={state["aggregate"]}')
    return {"aggregate": ["B"]}


def route(state: State) -> Literal["b", "__end__"]:
    """当剩余步数 <= 2 时终止循环"""
    if state["remaining_steps"] <= 2:
        return END
    return "b"


builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)

builder.add_edge(START, "a")
builder.add_conditional_edges("a", route)
builder.add_edge("b", "a")

graph = builder.compile()

# 不同的 recursion_limit 会导致不同的迭代次数
print("=== recursion_limit=4 ===")
result = graph.invoke({"aggregate": []}, {"recursion_limit": 4})
print(f"结果: {result['aggregate']}\n")

print("=== recursion_limit=8 ===")
result = graph.invoke({"aggregate": []}, {"recursion_limit": 8})
print(f"结果: {result['aggregate']}")
```

**运行输出：**
```
=== recursion_limit=4 ===
Node A: aggregate=[], remaining=5
Node B: aggregate=['A']
Node A: aggregate=['A', 'B'], remaining=3
结果: ['A', 'B', 'A']

=== recursion_limit=8 ===
Node A: aggregate=[], remaining=9
Node B: aggregate=['A']
Node A: aggregate=['A', 'B'], remaining=7
Node B: aggregate=['A', 'B', 'A']
Node A: aggregate=['A', 'B', 'A', 'B'], remaining=5
Node B: aggregate=['A', 'B', 'A', 'B', 'A']
Node A: aggregate=['A', 'B', 'A', 'B', 'A', 'B'], remaining=3
结果: ['A', 'B', 'A', 'B', 'A', 'B', 'A']
```

## 主动式 vs 被动式处理

### 被动式：捕获 GraphRecursionError

```python
"""
被动式处理（不推荐）：等异常发生后再处理
"""
from langgraph.errors import GraphRecursionError


def run_with_reactive_handling(graph, inputs, limit=5):
    """被动式：捕获异常"""
    try:
        result = graph.invoke(inputs, config={"recursion_limit": limit})
        return {"status": "success", "result": result}
    except GraphRecursionError:
        # 问题：此时图的中间状态已丢失
        # 无法获取到循环中积累的部分结果
        return {"status": "error", "result": "递归限制触发，无法返回结果"}
```

**被动式的问题：**
1. 异常抛出时，中间状态丢失（除非使用 checkpointer）
2. 无法在图内部执行收尾逻辑
3. 用户看到的是错误，而非部分结果
4. 无法区分"差一点就完成"和"完全没进展"

### 主动式：使用 RemainingSteps（推荐）

```python
"""
主动式处理（推荐）：在图内部感知剩余步数，提前降级
"""
import operator
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.managed.is_last_step import RemainingSteps


class AgentState(TypedDict):
    messages: Annotated[list[str], operator.add]
    remaining_steps: RemainingSteps


def agent_node(state: AgentState):
    """Agent 节点：感知剩余步数，调整行为"""
    remaining = state["remaining_steps"]

    if remaining <= 2:
        # 步数不足，快速总结
        return {"messages": [f"[Agent] 步数不足({remaining})，生成最终摘要"]}

    # 正常处理
    return {"messages": [f"[Agent] 正常推理 (剩余{remaining}步)"]}


def route(state: AgentState) -> Literal["agent", "fallback"]:
    if state["remaining_steps"] <= 2:
        return "fallback"
    return "agent"


def fallback_node(state: AgentState):
    """Fallback：返回尽力而为的结果"""
    return {"messages": ["[Fallback] 返回当前最佳结果，循环正常结束"]}


builder = StateGraph(AgentState)
builder.add_node("agent", agent_node)
builder.add_node("fallback", fallback_node)

builder.add_edge(START, "agent")
builder.add_conditional_edges("agent", route)
builder.add_edge("fallback", END)

graph = builder.compile()

# 主动式：图正常完成，不抛异常
result = graph.invoke({"messages": []}, {"recursion_limit": 5})
print("执行结果:")
for msg in result["messages"]:
    print(f"  {msg}")
```

**运行输出：**
```
执行结果:
  [Agent] 正常推理 (剩余6步)
  [Agent] 正常推理 (剩余4步)
  [Agent] 步数不足(2)，生成最终摘要
  [Fallback] 返回当前最佳结果，循环正常结束
```

### 对比总结

| 维度 | 被动式（GraphRecursionError） | 主动式（RemainingSteps） |
|------|------------------------------|--------------------------|
| 触发时机 | 超出限制后 | 接近限制前 |
| 中间状态 | 丢失（除非有 checkpointer） | 完整保留 |
| 用户体验 | 看到错误信息 | 看到部分结果 |
| 收尾逻辑 | 无法在图内执行 | 可以路由到 fallback 节点 |
| 推荐度 | 仅作兜底 | 官方推荐 |
| 实现复杂度 | 简单（try/except） | 稍复杂（需要 fallback 节点） |

**官方推荐：** Context7 文档明确推荐使用 `RemainingSteps` 的主动式方法，因为它允许在图逻辑内部实现优雅降级。

## PregelLoop tick() 完整执行流程

理解递归限制的工作原理，需要了解 PregelLoop 的完整迭代流程。

### 流程图

```
用户调用 graph.invoke(inputs, config)
  │
  ▼
PregelLoop.__enter__()
  ├── 加载 checkpoint（如果有 checkpointer）
  ├── 初始化 channels
  ├── step = checkpoint_metadata["step"] + 1
  ├── stop = step + recursion_limit + 1    ← 计算停止点
  └── _first(): 处理输入
        │
        ▼
  ┌──→ tick()
  │     ├── step > stop?
  │     │     ├── Yes → status = "out_of_steps", return False
  │     │     └── No  → 继续
  │     ├── prepare_next_tasks()
  │     │     └── 根据 channel 状态确定要执行的节点
  │     ├── 没有任务?
  │     │     ├── Yes → status = "done", return False
  │     │     └── No  → 继续
  │     ├── interrupt_before 检查
  │     └── return True
  │           │
  │           ▼
  │     runner.tick()
  │     └── 并行执行所有任务（节点函数）
  │           │
  │           ▼
  │     after_tick()
  │     ├── apply_writes()
  │     │     ├── 将节点输出写入 channels
  │     │     ├── 写入 "branch:to:X" → X 节点下一轮被触发
  │     │     └── 写入 TASKS (Send) → 新并行任务被创建
  │     ├── _put_checkpoint()
  │     │     └── step += 1                ← 步数递增
  │     └── interrupt_after 检查
  │           │
  └──────────┘ (回到 tick)

循环结束:
  ├── status == "done"         → 正常返回结果
  ├── status == "out_of_steps" → 抛出 GraphRecursionError
  └── GraphInterrupt           → 被 _suppress_interrupt 处理
```

### 关键源码片段

**主循环驱动**（`main.py` L2643-L2674，简化）：

```python
while loop.tick():
    # 执行缓存命中的任务
    for task in loop.match_cached_writes():
        loop.output_writes(task.id, task.writes, cached=True)

    # 执行所有待执行的任务
    for _ in runner.tick(
        [t for t in loop.tasks.values() if not t.writes],
        timeout=self.step_timeout,
    ):
        yield from _output(...)

    # 迭代后处理
    loop.after_tick()

# 循环结束后的状态检查
if loop.status == "out_of_steps":
    raise GraphRecursionError(...)
```

**step 递增**（`_loop.py` L811-L813）：

```python
# _loop.py L811
def _put_checkpoint(self, ...):
    if not exiting:
        self.step += 1
```

## 实战：完整的安全循环模式

```python
"""
生产级安全循环模式：结合所有安全机制
1. 业务终止条件（质量达标）
2. 计数器上限（防止业务逻辑 bug）
3. RemainingSteps 优雅降级（系统级安全阀）
4. GraphRecursionError 兜底（最后防线）
"""
import operator
from typing import Annotated, Literal
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.managed.is_last_step import RemainingSteps
from langgraph.errors import GraphRecursionError


class SafeLoopState(TypedDict):
    content: str
    quality_score: float
    iteration_count: int
    exit_reason: str
    log: Annotated[list[str], operator.add]
    remaining_steps: RemainingSteps


def process_node(state: SafeLoopState):
    """处理节点"""
    iteration = state["iteration_count"] + 1
    remaining = state["remaining_steps"]

    # 模拟质量递增
    score = min(0.15 * iteration + 0.1, 0.95)

    return {
        "content": f"内容 v{iteration}",
        "quality_score": score,
        "iteration_count": iteration,
        "log": [f"迭代{iteration}: 质量={score:.2f}, 剩余步数={remaining}"],
    }


def route(state: SafeLoopState) -> Literal["process", "fallback", "__end__"]:
    """三层终止条件"""
    # 第一层：业务条件 — 质量达标
    if state["quality_score"] >= 0.85:
        return END

    # 第二层：计数器上限
    if state["iteration_count"] >= 10:
        return END

    # 第三层：系统安全阀 — 剩余步数不足
    if state["remaining_steps"] <= 2:
        return "fallback"

    return "process"


def fallback_node(state: SafeLoopState):
    """优雅降级节点"""
    return {
        "exit_reason": "graceful_degradation",
        "log": [f"优雅降级: 返回质量={state['quality_score']:.2f}的结果"],
    }


builder = StateGraph(SafeLoopState)
builder.add_node("process", process_node)
builder.add_node("fallback", fallback_node)

builder.add_edge(START, "process")
builder.add_conditional_edges("process", route)
builder.add_edge("fallback", END)

graph = builder.compile()


# === 场景 1：正常完成（质量达标退出）===
print("=== 场景 1: 正常完成 ===")
result = graph.invoke(
    {
        "content": "",
        "quality_score": 0.0,
        "iteration_count": 0,
        "exit_reason": "success",
        "log": [],
    },
    config={"recursion_limit": 25},
)
for entry in result["log"]:
    print(f"  {entry}")
print(f"  退出原因: {result['exit_reason']}")


# === 场景 2：优雅降级（recursion_limit 较小）===
print("\n=== 场景 2: 优雅降级 ===")
result = graph.invoke(
    {
        "content": "",
        "quality_score": 0.0,
        "iteration_count": 0,
        "exit_reason": "",
        "log": [],
    },
    config={"recursion_limit": 4},
)
for entry in result["log"]:
    print(f"  {entry}")
print(f"  退出原因: {result['exit_reason']}")
```

**运行输出：**
```
=== 场景 1: 正常完成 ===
  迭代1: 质量=0.25, 剩余步数=26
  迭代2: 质量=0.40, 剩余步数=24
  迭代3: 质量=0.55, 剩余步数=22
  迭代4: 质量=0.70, 剩余步数=20
  迭代5: 质量=0.85, 剩余步数=18
  退出原因: success

=== 场景 2: 优雅降级 ===
  迭代1: 质量=0.25, 剩余步数=5
  迭代2: 质量=0.40, 剩余步数=3
  优雅降级: 返回质量=0.40的结果
  退出原因: graceful_degradation
```

## 在 RAG 开发中的应用

### ReAct Agent 的安全循环

```python
from langgraph.managed.is_last_step import RemainingSteps

class RAGAgentState(TypedDict):
    messages: list
    remaining_steps: RemainingSteps

def agent_node(state: RAGAgentState):
    remaining = state["remaining_steps"]

    if remaining <= 2:
        # 步数不足，强制生成最终答案（不再调用工具）
        return {"messages": ["基于已有信息生成最终答案..."]}

    # 正常的 ReAct 循环：调用工具获取信息
    return {"messages": ["调用检索工具..."]}
```

### 推荐的 recursion_limit 设置

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 简单 RAG 问答 | 5-10 | 检索 + 生成，不需要太多迭代 |
| ReAct Agent | 15-25 | 可能需要多次工具调用 |
| 自校正循环 | 10-15 | 通常 3-5 次迭代就够 |
| 多 Agent 协作 | 25-50 | 多个 Agent 交互，步数消耗快 |
| 复杂研究任务 | 50-100 | 深度搜索和分析 |
