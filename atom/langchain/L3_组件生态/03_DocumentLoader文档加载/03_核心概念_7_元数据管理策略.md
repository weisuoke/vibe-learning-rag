# 核心概念 7: 元数据管理策略

> 理解元数据在文档检索中的关键作用

---

## 核心问题

为什么 LangChain 的 DocumentLoader 强调元数据管理?元数据如何提升 RAG 系统的检索质量?

---

## 什么是元数据?

### 定义

**元数据(Metadata)**: 描述文档本身的数据,包括来源、作者、时间、页码等信息。

**对比**:

```python
# 没有元数据 - 只有内容
doc = Document(page_content="Python is a programming language.")

# 有元数据 - 内容 + 描述信息
doc = Document(
    page_content="Python is a programming language.",
    metadata={
        "source": "python_tutorial.pdf",
        "page": 1,
        "author": "John Doe",
        "created_at": "2025-01-01"
    }
)
```

---

## 为什么需要元数据?

### 问题 1: 无法追溯来源

**场景**: 检索到相关文档,但不知道来自哪里

```python
# 没有元数据
docs = loader.load()
for doc in docs:
    print(doc.page_content)
    # 问题: 这段内容来自哪个文件?哪一页?

# 有元数据
docs = loader.load()
for doc in docs:
    print(f"内容: {doc.page_content}")
    print(f"来源: {doc.metadata['source']}")
    print(f"页码: {doc.metadata['page']}")
```

### 问题 2: 无法过滤检索

**场景**: 只想检索特定类型的文档

```python
# 没有元数据 - 无法过滤
results = vector_store.similarity_search("Python tutorial")
# 返回所有相关文档,无法区分类型

# 有元数据 - 可以过滤
results = vector_store.similarity_search(
    "Python tutorial",
    filter={"doc_type": "tutorial", "language": "en"}
)
# 只返回英文教程
```

### 问题 3: 无法去重

**场景**: 同一文档被多次加载

```python
# 没有元数据 - 无法去重
docs1 = loader1.load()
docs2 = loader2.load()
all_docs = docs1 + docs2
# 可能包含重复文档

# 有元数据 - 可以去重
docs1 = loader1.load()
docs2 = loader2.load()
seen = set()
unique_docs = []
for doc in docs1 + docs2:
    doc_id = (doc.metadata["source"], doc.metadata["page"])
    if doc_id not in seen:
        seen.add(doc_id)
        unique_docs.append(doc)
```

## 元数据的来源

### 1. 文件系统元数据

```python
from langchain_community.document_loaders import PyPDFLoader

loader = PyPDFLoader("report.pdf")
docs = loader.load()

# 自动添加的元数据
for doc in docs:
    print(doc.metadata)
    # {
    #     "source": "report.pdf",
    #     "page": 0,
    #     "total_pages": 10
    # }
```

### 2. 文档内容元数据

```python
from langchain_community.document_loaders import UnstructuredPDFLoader

loader = UnstructuredPDFLoader("report.pdf")
docs = loader.load()

# 从文档内容提取的元数据
for doc in docs:
    print(doc.metadata)
    # {
    #     "source": "report.pdf",
    #     "page": 0,
    #     "title": "Annual Report 2025",
    #     "author": "John Doe",
    #     "created_date": "2025-01-01"
    # }
```

### 3. 自定义元数据

```python
from langchain_community.document_loaders import TextLoader

loader = TextLoader("article.txt")
docs = loader.load()

# 添加自定义元数据
for doc in docs:
    doc.metadata["category"] = "technology"
    doc.metadata["language"] = "en"
    doc.metadata["priority"] = "high"
```

### 4. 运行时元数据

```python
from datetime import datetime

loader = DirectoryLoader("docs/")
docs = loader.load()

# 添加运行时元数据
for doc in docs:
    doc.metadata["loaded_at"] = datetime.now().isoformat()
    doc.metadata["loader_version"] = "1.0.0"
```

## 元数据的传递

### 加载阶段

```python
# Blob 的元数据
blob = Blob.from_path(
    "file.pdf",
    metadata={"source_type": "local", "priority": "high"}
)

# Parser 继承 Blob 的元数据
parser = PyPDFParser()
docs = list(parser.lazy_parse(blob))

for doc in docs:
    print(doc.metadata)
    # {
    #     "source": "file.pdf",
    #     "source_type": "local",  # 继承自 Blob
    #     "priority": "high",      # 继承自 Blob
    #     "page": 0                # Parser 添加
    # }
```

[来源: reference/source_documentloader_01.md | LangChain 源码分析]

### 分割阶段

```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

# 加载文档
loader = PyPDFLoader("report.pdf")
docs = loader.load()

# 分割文档 - 元数据会被保留
splitter = RecursiveCharacterTextSplitter(chunk_size=1000)
chunks = splitter.split_documents(docs)

for chunk in chunks:
    print(chunk.metadata)
    # {
    #     "source": "report.pdf",  # 保留原始元数据
    #     "page": 0,               # 保留原始元数据
    #     "chunk_index": 0         # 分割器添加
    # }
```

### 向量化阶段

```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma

# 向量化 - 元数据会被存储
embeddings = OpenAIEmbeddings()
vector_store = Chroma.from_documents(chunks, embeddings)

# 检索时元数据会被返回
results = vector_store.similarity_search("Python tutorial")
for result in results:
    print(result.metadata)
    # {
    #     "source": "report.pdf",
    #     "page": 0,
    #     "chunk_index": 0
    # }
```

## 元数据在检索中的应用

### 1. 过滤检索

```python
# 只检索特定来源的文档
results = vector_store.similarity_search(
    "Python tutorial",
    filter={"source": "python_tutorial.pdf"}
)

# 只检索特定时间范围的文档
results = vector_store.similarity_search(
    "Python tutorial",
    filter={
        "created_at": {"$gte": "2025-01-01", "$lte": "2025-12-31"}
    }
)

# 组合过滤条件
results = vector_store.similarity_search(
    "Python tutorial",
    filter={
        "category": "tutorial",
        "language": "en",
        "priority": "high"
    }
)
```

### 2. 结果排序

```python
# 按优先级排序
results = vector_store.similarity_search("Python tutorial", k=10)
sorted_results = sorted(
    results,
    key=lambda doc: doc.metadata.get("priority", 0),
    reverse=True
)

# 按时间排序
sorted_results = sorted(
    results,
    key=lambda doc: doc.metadata.get("created_at", ""),
    reverse=True
)
```

### 3. 结果去重

```python
# 按来源去重
seen_sources = set()
unique_results = []
for doc in results:
    source = doc.metadata["source"]
    if source not in seen_sources:
        seen_sources.add(source)
        unique_results.append(doc)

# 按内容哈希去重
import hashlib

seen_hashes = set()
unique_results = []
for doc in results:
    content_hash = hashlib.md5(doc.page_content.encode()).hexdigest()
    if content_hash not in seen_hashes:
        seen_hashes.add(content_hash)
        unique_results.append(doc)
```

### 4. 引用生成

```python
# 生成引用信息
for doc in results:
    citation = f"[{doc.metadata['source']}, p.{doc.metadata['page']}]"
    print(f"{doc.page_content}\n{citation}")

# 输出:
# Python is a programming language.
# [python_tutorial.pdf, p.1]
```

## 实际应用

### 场景 1: 多文档问答系统

```python
from langchain_community.document_loaders import DirectoryLoader
from langchain_openai import OpenAIEmbeddings, ChatOpenAI
from langchain_community.vectorstores import Chroma
from langchain.chains import RetrievalQA

# 加载多个文档
loader = DirectoryLoader("docs/", glob="**/*.pdf")
docs = loader.load()

# 添加文档类型元数据
for doc in docs:
    if "tutorial" in doc.metadata["source"]:
        doc.metadata["doc_type"] = "tutorial"
    elif "reference" in doc.metadata["source"]:
        doc.metadata["doc_type"] = "reference"
    else:
        doc.metadata["doc_type"] = "other"

# 分割并向量化
chunks = loader.load_and_split()
embeddings = OpenAIEmbeddings()
vector_store = Chroma.from_documents(chunks, embeddings)

# 构建 QA 链
llm = ChatOpenAI()
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=vector_store.as_retriever()
)

# 查询时可以过滤文档类型
result = qa_chain.invoke({
    "query": "How to install Python?",
    "filter": {"doc_type": "tutorial"}
})

# 显示引用
print(result["result"])
print("\nSources:")
for doc in result["source_documents"]:
    print(f"- {doc.metadata['source']} (p.{doc.metadata['page']})")
```

### 场景 2: 时间敏感的文档检索

```python
from datetime import datetime, timedelta

# 加载文档并添加时间戳
loader = DirectoryLoader("news/")
docs = loader.load()

for doc in docs:
    # 从文件名提取日期
    filename = doc.metadata["source"]
    date_str = filename.split("_")[0]  # 假设格式: 2025-01-01_news.txt
    doc.metadata["published_date"] = date_str

# 向量化
chunks = loader.load_and_split()
vector_store = Chroma.from_documents(chunks, embeddings)

# 只检索最近 7 天的新闻
seven_days_ago = (datetime.now() - timedelta(days=7)).strftime("%Y-%m-%d")
results = vector_store.similarity_search(
    "Python news",
    filter={"published_date": {"$gte": seven_days_ago}}
)
```

### 场景 3: 多语言文档检索

```python
# 加载多语言文档
loader = DirectoryLoader("docs/")
docs = loader.load()

# 添加语言元数据
for doc in docs:
    if "en" in doc.metadata["source"]:
        doc.metadata["language"] = "en"
    elif "zh" in doc.metadata["source"]:
        doc.metadata["language"] = "zh"
    elif "ja" in doc.metadata["source"]:
        doc.metadata["language"] = "ja"

# 向量化
chunks = loader.load_and_split()
vector_store = Chroma.from_documents(chunks, embeddings)

# 只检索中文文档
results = vector_store.similarity_search(
    "Python 教程",
    filter={"language": "zh"}
)
```

### 场景 4: 权限控制

```python
# 加载文档并添加权限元数据
loader = DirectoryLoader("docs/")
docs = loader.load()

for doc in docs:
    if "public" in doc.metadata["source"]:
        doc.metadata["access_level"] = "public"
    elif "internal" in doc.metadata["source"]:
        doc.metadata["access_level"] = "internal"
    else:
        doc.metadata["access_level"] = "private"

# 向量化
chunks = loader.load_and_split()
vector_store = Chroma.from_documents(chunks, embeddings)

# 根据用户权限检索
def search_with_permission(query, user_level):
    allowed_levels = {
        "public": ["public"],
        "internal": ["public", "internal"],
        "admin": ["public", "internal", "private"]
    }

    results = vector_store.similarity_search(
        query,
        filter={"access_level": {"$in": allowed_levels[user_level]}}
    )
    return results

# 公开用户只能看到公开文档
public_results = search_with_permission("Python tutorial", "public")

# 管理员可以看到所有文档
admin_results = search_with_permission("Python tutorial", "admin")
```

## 类比理解

### 前端开发类比

**元数据** 就像 **HTTP 响应头**:

```javascript
// 没有元数据 - 只有内容
const response = "Hello, world!"

// 有元数据 - 内容 + 描述信息
const response = {
  body: "Hello, world!",
  headers: {
    "Content-Type": "text/plain",
    "Content-Length": "13",
    "Last-Modified": "2025-01-01",
    "Cache-Control": "max-age=3600"
  }
}
```

### 日常生活类比

**元数据** 就像 **图书馆的索引卡**:

**没有元数据** = 只有书的内容:
- 无法知道作者
- 无法知道出版日期
- 无法按类别查找

**有元数据** = 书 + 索引卡:
- 作者: 张三
- 出版日期: 2025-01-01
- 类别: 计算机
- ISBN: 978-1234567890

## 常见误区

### ❌ 误区 1: 元数据不重要

**错误**: 认为只要有内容就够了

**正确**: 元数据对检索质量至关重要

```python
# ✗ 错误: 忽略元数据
docs = loader.load()
vector_store = Chroma.from_documents(docs, embeddings)
results = vector_store.similarity_search("Python tutorial")
# 无法知道结果来自哪里

# ✓ 正确: 保留元数据
docs = loader.load()
vector_store = Chroma.from_documents(docs, embeddings)
results = vector_store.similarity_search("Python tutorial")
for result in results:
    print(f"来源: {result.metadata['source']}")
```

### ❌ 误区 2: 元数据会被自动保留

**错误**: 认为元数据会自动传递

**正确**: 需要确保元数据在每个阶段都被保留

```python
# ✗ 错误: 手动分割丢失元数据
docs = loader.load()
chunks = []
for doc in docs:
    # 手动分割,丢失元数据
    for i in range(0, len(doc.page_content), 1000):
        chunk = doc.page_content[i:i+1000]
        chunks.append(Document(page_content=chunk))

# ✓ 正确: 使用 TextSplitter 保留元数据
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=1000)
chunks = splitter.split_documents(docs)
# 元数据被自动保留
```

### ❌ 误区 3: 元数据越多越好

**错误**: 添加大量无用的元数据

**正确**: 只添加有用的元数据

```python
# ✗ 错误: 添加无用元数据
doc.metadata["random_id"] = "abc123"
doc.metadata["internal_flag"] = True
doc.metadata["debug_info"] = "..."

# ✓ 正确: 只添加有用元数据
doc.metadata["source"] = "report.pdf"
doc.metadata["page"] = 1
doc.metadata["category"] = "tutorial"
doc.metadata["language"] = "en"
```

## 最佳实践

### 1. 标准化元数据字段

```python
# 定义标准元数据字段
STANDARD_METADATA = {
    "source": str,      # 文件路径
    "page": int,        # 页码
    "category": str,    # 类别
    "language": str,    # 语言
    "created_at": str,  # 创建时间
}

# 验证元数据
def validate_metadata(doc):
    for key, value_type in STANDARD_METADATA.items():
        if key in doc.metadata:
            if not isinstance(doc.metadata[key], value_type):
                raise ValueError(f"Invalid type for {key}")
```

### 2. 使用命名空间

```python
# 使用命名空间避免冲突
doc.metadata["file.source"] = "report.pdf"
doc.metadata["file.size"] = 1024
doc.metadata["content.language"] = "en"
doc.metadata["content.category"] = "tutorial"
doc.metadata["system.loaded_at"] = "2025-01-01"
```

### 3. 元数据索引

```python
# 为常用元数据字段创建索引
vector_store = Chroma.from_documents(
    chunks,
    embeddings,
    collection_metadata={
        "indexed_fields": ["source", "category", "language"]
    }
)
```

### 4. 元数据清理

```python
# 清理无用的元数据
def clean_metadata(doc):
    # 移除内部字段
    doc.metadata.pop("_internal_id", None)
    doc.metadata.pop("_debug_info", None)

    # 标准化字段名
    if "src" in doc.metadata:
        doc.metadata["source"] = doc.metadata.pop("src")

    return doc

docs = [clean_metadata(doc) for doc in docs]
```

## 总结

### 核心作用

1. **可追溯性**: 知道内容来自哪里
2. **可过滤性**: 按条件筛选文档
3. **可排序性**: 按优先级排序
4. **可去重性**: 避免重复文档

### 设计优势

| 优势 | 说明 |
|------|------|
| 提升检索质量 | 通过过滤提高相关性 |
| 增强可解释性 | 提供引用来源 |
| 支持权限控制 | 按权限过滤文档 |
| 便于调试 | 追踪文档来源 |

### 关键字段

| 字段 | 说明 | 示例 |
|------|------|------|
| source | 文件路径 | "report.pdf" |
| page | 页码 | 1 |
| category | 类别 | "tutorial" |
| language | 语言 | "en" |
| created_at | 创建时间 | "2025-01-01" |
