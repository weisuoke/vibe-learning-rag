# 30字核心

**Markdown 模板 + Bash 风格变量插值 + 多源加载，实现提示词复用与团队协作的极简模板系统。**

---

## 核心要点

Pi-mono 的 Prompt Templates 是一个极简但强大的模板系统，通过 Markdown 文件 + YAML frontmatter + Bash 风格变量替换，实现提示词的复用、版本控制和团队协作。

### 三大核心特性

1. **Markdown 模板**
   - 使用标准 Markdown 格式
   - YAML frontmatter 定义元数据
   - 纯文本，易于版本控制

2. **Bash 风格变量插值**
   - `$1, $2, ...` - 位置参数
   - `$@` / `$ARGUMENTS` - 所有参数
   - `${@:N}` / `${@:N:L}` - 参数切片
   - **不是** `{{variable}}` 语法

3. **多源加载**
   - 全局模板：`~/.pi/agent/prompts/`
   - 项目模板：`./.pi/prompts/`
   - 显式路径：通过 CLI 或配置指定
   - 优先级：项目 > 全局 > 显式

### 为什么是 30 字？

这 30 字浓缩了 Prompt Templates 的本质：
- **Markdown 模板** - 格式简单，易于编写和维护
- **Bash 风格变量插值** - 熟悉的语法，强大的参数处理
- **多源加载** - 灵活的组织方式，支持全局和项目级别
- **提示词复用** - 避免重复，提高效率
- **团队协作** - 版本控制，共享最佳实践

### 与其他模板系统的区别

| 特性 | Pi-mono Templates | Handlebars/Jinja | LangChain PromptTemplate |
|------|-------------------|------------------|--------------------------|
| 语法 | Bash 风格 (`$1`, `$@`) | `{{variable}}` | `{variable}` |
| 格式 | Markdown + YAML | 任意文本 | Python 字符串 |
| 加载 | 文件系统多源 | 手动加载 | 代码定义 |
| 调用 | `/template-name args` | 编程调用 | 编程调用 |
| 版本控制 | 原生支持 | 需要额外管理 | 代码版本控制 |

### 实际应用场景

1. **代码审查模板**
   ```bash
   /pr https://github.com/user/repo/pull/123
   ```
   - 自动分析 PR 内容
   - 生成结构化审查报告

2. **问题分析模板**
   ```bash
   /issue https://github.com/user/repo/issues/456
   ```
   - 提取问题关键信息
   - 提供解决方案建议

3. **自定义工作流模板**
   ```bash
   /scout-and-plan "实现用户认证功能"
   ```
   - 探索代码库
   - 生成实现计划

3. **AI Agent 工作流标准化**

   - 参数化配置，适应不同场景

## 快速理解

**一句话：** Pi-mono 的 Prompt Templates 就像 Bash 脚本的参数传递 + Markdown 文档的可读性 + Git 的版本控制，三者结合实现提示词的工程化管理。

**类比：**
- **TypeScript/Node.js 类比**：就像 Handlebars/EJS 模板，但使用 Bash 风格的变量替换
- **日常生活类比**：就像邮件模板，填入收件人、主题等变量后发送

**核心价值：**
1. **复用性** - 一次编写，多次使用
2. **可维护性** - 版本控制，易于更新
3. **团队协作** - 共享最佳实践
4. **标准化** - 统一提示词格式
5. **灵活性** - 支持参数化配置
