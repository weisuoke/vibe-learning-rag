# 核心概念 2: 构建工具链层

**知识点**: 开发环境搭建（从源码）
**层级**: 中层编译系统

---

## 一、概念定位

### 1.1 在三层架构中的位置

```
┌─────────────────────────────────────┐
│     依赖管理层（pnpm workspace）      │
└─────────────────────────────────────┘
              ↓ 提供依赖
┌─────────────────────────────────────┐
│     构建工具链层（TypeScript）        │ ← 当前层
│  - tsconfig.json 编译配置            │
│  - tsdown 快速编译器                 │
│  - 构建脚本和输出管理                │
└─────────────────────────────────────┘
              ↓ 编译代码
┌─────────────────────────────────────┐
│     源码结构层（Monorepo）           │
└─────────────────────────────────────┘
```

### 1.2 核心职责

构建工具链层负责：
1. **代码编译**：将 TypeScript 编译为 JavaScript
2. **类型检查**：确保类型安全
3. **声明文件生成**：生成 .d.ts 类型声明
4. **Source Map 生成**：便于调试

## 二、TypeScript 编译配置

### 2.1 tsconfig.json 核心配置

**OpenClaw 的 tsconfig.json**:

```json
{
  "compilerOptions": {
    "target": "ES2023",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "lib": ["ES2023"],
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "paths": {
      "openclaw/plugin-sdk": ["./src/plugin-sdk/index.ts"]
    }
  },
  "include": ["src/**/*", "ui/**/*", "extensions/**/*"],
  "exclude": ["node_modules", "dist", "**/*.test.ts"]
}
```

**数据来源**: `sourcecode/openclaw/tsconfig.json`

---

### 2.2 编译选项详解

#### target: ES2023

**作用**: 指定编译后的 JavaScript 版本

**为什么选择 ES2023**:
- Node.js 22 完全支持 ES2023
- 使用最新的 JavaScript 特性
- 无需额外的 polyfill

**数据来源**: `reference/03_typescript_5.9.3_docs.md` - target 配置

---

#### module: NodeNext

**作用**: 指定模块系统

**为什么选择 NodeNext**:
- 完全支持 Node.js 的 ESM 模块系统
- 自动处理 .mjs 和 .cjs 文件
- 支持 package.json 的 "type": "module"

**关键点**:
```json
{
  "module": "NodeNext",
  "moduleResolution": "NodeNext"
}
```

**注意**: `module` 和 `moduleResolution` 必须同时设置为 `NodeNext`

**数据来源**: `reference/03_typescript_5.9.3_docs.md` - module 配置

---

#### strict: true

**作用**: 启用所有严格类型检查选项

**包含的选项**:
- `noImplicitAny`: 禁止隐式 any 类型
- `strictNullChecks`: 严格的 null 检查
- `strictFunctionTypes`: 严格的函数类型检查
- `strictBindCallApply`: 严格的 bind/call/apply 检查
- `strictPropertyInitialization`: 严格的属性初始化检查
- `noImplicitThis`: 禁止隐式 this
- `alwaysStrict`: 始终使用严格模式

**优势**:
- 提前发现潜在错误
- 提高代码质量
- 更好的 IDE 支持

---

#### declaration: true

**作用**: 生成 .d.ts 类型声明文件

**示例**:

```typescript
// src/index.ts
export function hello(name: string): string {
  return `Hello, ${name}!`;
}
```

**生成的 .d.ts**:

```typescript
// dist/index.d.ts
export declare function hello(name: string): string;
```

**用途**:
- 为库提供类型定义
- 支持 TypeScript 项目使用
- 提供 IDE 自动补全

---

#### sourceMap: true

**作用**: 生成 source map 文件

**示例**:

```
dist/
├── index.js
├── index.js.map      ← source map
├── index.d.ts
└── index.d.ts.map    ← declaration map
```

**用途**:
- 调试时映射到原始 TypeScript 代码
- 错误堆栈显示原始代码位置
- 提高开发体验

---

#### paths 路径映射

**作用**: 配置模块别名

**OpenClaw 的配置**:

```json
{
  "paths": {
    "openclaw/plugin-sdk": ["./src/plugin-sdk/index.ts"]
  }
}
```

**使用示例**:

```typescript
// 不使用 paths
import { Plugin } from '../../../src/plugin-sdk/index.ts';

// 使用 paths
import { Plugin } from 'openclaw/plugin-sdk';
```

**优势**:
- 简化导入路径
- 避免相对路径地狱
- 提高代码可读性

## 三、tsdown 编译器

### 3.1 什么是 tsdown？

**定义**: tsdown 是基于 Rolldown 和 Oxc 的高性能 TypeScript 编译器

**核心特性**:
1. **极快的编译速度**：比 tsc 快数倍
2. **完整的类型支持**：生成 .d.ts 文件
3. **简洁的配置**：开箱即用

**数据来源**: `reference/05_typescript_build_github.md` - tsdown 介绍

---

### 3.2 tsdown vs tsc vs esbuild

**性能对比**:

| 工具 | 编译速度 | 类型检查 | DTS 生成 | 配置复杂度 |
|------|----------|----------|----------|------------|
| tsc | 慢 | ✅ | ✅ | 高 |
| esbuild | 快 | ❌ | ❌ | 中 |
| tsdown | 最快 | ✅ | ✅ | 低 |

**数据来源**: `reference/05_typescript_build_github.md` - 性能对比

---

### 3.3 tsdown 配置

**OpenClaw 的 tsdown.config.ts**:

```typescript
import { defineConfig } from 'tsdown';

export default defineConfig({
  entry: {
    index: 'src/index.ts',
    'plugin-sdk': 'src/plugin-sdk/index.ts',
  },
  format: ['esm'],
  dts: true,
  sourcemap: true,
  outDir: 'dist',
  clean: true,
});
```

**配置说明**:
- `entry`: 多入口配置
- `format`: 输出格式（ESM）
- `dts`: 生成类型声明
- `sourcemap`: 生成 source map
- `outDir`: 输出目录
- `clean`: 构建前清理输出目录

**数据来源**: `reference/05_typescript_build_github.md` - tsdown 配置

---

### 3.4 tsdown 工作原理

**编译流程**:

```
TypeScript 源码
    ↓
Oxc 解析（AST）
    ↓
Rolldown 打包
    ↓
生成 JavaScript
    ↓
生成 .d.ts
    ↓
生成 source map
    ↓
输出到 dist/
```

**关键技术**:
- **Oxc**: 用 Rust 编写的超快 JavaScript 工具链
- **Rolldown**: 用 Rust 编写的 Rollup 替代品

**数据来源**: `reference/05_typescript_build_github.md` - tsdown 原理

## 四、构建脚本

### 4.1 package.json 构建脚本

**OpenClaw 的构建脚本**:

```json
{
  "scripts": {
    "ui:build": "pnpm --filter ./ui build",
    "build": "tsdown",
    "dev": "node scripts/run-node.mjs",
    "gateway:dev": "pnpm dev",
    "gateway:watch": "pnpm dev",
    "check": "oxfmt --check && tsc --noEmit && oxlint",
    "test": "vitest"
  }
}
```

**数据来源**: `sourcecode/openclaw/package.json` - scripts

---

### 4.2 构建流程

**完整构建流程**:

```bash
# 1. 构建 UI
pnpm ui:build

# 2. 编译 TypeScript
pnpm build

# 3. 验证
pnpm check

# 4. 测试
pnpm test
```

**详细说明**:

#### ui:build

```bash
pnpm --filter ./ui build
```

**作用**:
- 只构建 UI workspace
- 使用 Vite 构建 React 应用
- 输出到 ui/dist/

---

#### build

```bash
tsdown
```

**作用**:
- 编译所有 TypeScript 代码
- 生成 .d.ts 类型声明
- 生成 source map
- 输出到 dist/

---

#### check

```bash
oxfmt --check && tsc --noEmit && oxlint
```

**作用**:
- `oxfmt --check`: 检查代码格式
- `tsc --noEmit`: 类型检查（不生成文件）
- `oxlint`: 代码检查

**为什么分离类型检查和编译**:
- tsdown 负责快速编译
- tsc 负责严格的类型检查
- 各司其职，提高效率

---

### 4.3 开发模式脚本

**run-node.mjs 脚本**:

```javascript
// scripts/run-node.mjs
import { spawn } from 'child_process';
import { watch } from 'chokidar';

// 监听源码变化
const watcher = watch('src/**/*.ts', {
  ignored: /(^|[\/\\])\../,
  persistent: true
});

// 文件变化时重新编译
watcher.on('change', (path) => {
  console.log(`File ${path} has been changed`);

  // 重新编译
  const build = spawn('pnpm', ['build'], {
    stdio: 'inherit'
  });

  build.on('close', (code) => {
    if (code === 0) {
      console.log('Build successful');
    }
  });
});
```

**数据来源**: `sourcecode/openclaw/scripts/run-node.mjs`

```json
{
```

## 六、常见问题

### 6.1 编译错误

**问题**: tsc 编译失败

**可能原因**:
1. 类型错误
2. 配置错误
3. 依赖缺失

**解决方案**:

```bash
# 检查类型错误
pnpm check

# 查看详细错误
tsc --noEmit

# 清理缓存
rm -rf dist node_modules
pnpm install
pnpm build
```

**数据来源**: `reference/06_openclaw_dev_setup_github_reddit.md` - 故障排查

---

### 6.2 tsdown vs tsc 选择

**问题**: 什么时候用 tsdown，什么时候用 tsc？

**答案**:
- **tsdown**: 用于构建（快速编译）
- **tsc**: 用于类型检查（严格检查）

**推荐做法**:

```json
{
  "scripts": {
    "build": "tsdown",
    "type-check": "tsc --noEmit",
    "check": "pnpm type-check && pnpm lint"
  }
}
```

---

### 6.3 source map 不生效

**问题**: 调试时无法映射到原始代码

**可能原因**:
1. `sourceMap: false`
2. source map 文件缺失
3. 路径配置错误

**解决方案**:

```json
{
  "compilerOptions": {
    "sourceMap": true,
    "inlineSources": true
  }
}
```

---

## 七、最佳实践

### 7.1 分离类型检查和编译

**推荐**:

```json
{
  "scripts": {
    "build": "tsdown",
    "type-check": "tsc --noEmit",
    "dev": "tsdown --watch"
  }
}
```

**优势**:
- 编译速度快（tsdown）
- 类型检查严格（tsc）
- 开发体验好（watch 模式）

---

### 7.2 使用 strict 模式

**推荐**:

```json
{
  "compilerOptions": {
    "strict": true
  }
}
```

**优势**:
- 提前发现错误
- 提高代码质量
- 更好的 IDE 支持

---

### 7.3 生成 source map

**推荐**:

```json
{
  "compilerOptions": {
    "sourceMap": true,
    "declarationMap": true
  }
}
```

**优势**:
- 便于调试
- 错误堆栈清晰
- 提高开发效率

---

### 7.4 使用路径映射

**推荐**:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "openclaw/plugin-sdk": ["src/plugin-sdk/index.ts"]
    }
  }
}
```

**优势**:
- 简化导入路径
- 避免相对路径地狱
- 提高代码可读性

## 八、性能优化

### 8.1 增量编译

**tsdown 自动支持增量编译**:

```bash
# watch 模式自动增量编译
pnpm dev
```

**优势**:
- 只编译变更的文件
- 大幅提高编译速度
- 提高开发效率

---

### 8.2 并行编译

**使用 Turborepo 并行编译**:

```json
{
  "$schema": "https://turbo.build/schema.json",
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**"]
    }
  }
}
```

**优势**:
- 并行编译多个包
- 利用多核 CPU
- 大幅提高构建速度

**数据来源**: `reference/07_pnpm_workspace_best_practices.md` - Turborepo

---

### 8.3 缓存优化

**tsdown 自动缓存**:

```typescript
export default defineConfig({

});
```

**优势**:
- 利用缓存加速编译
- 减少重复工作
- 提高构建速度

## 九、与其他层的关系
