# 实战代码 · 场景 1：单节点步级超时保护

> **目标**：最小可复现 `graph.step_timeout` 的行为
> **适用**：理解“step timeout ≠ node 内部 timeout”之前，先抓住最基本的执行感受

---

## 场景说明

这是一个最简单的 LangGraph 图：

- 只有一个慢节点；
- 节点内部 `sleep(2)`；
- 图设置 `step_timeout = 1`；
- 观察同步 `invoke()` 如何抛 `TimeoutError`。

虽然它看起来像“node timeout”，但我们要借这个最小例子建立正确直觉：

**它本质上仍然是 step budget 触发。**

---

## 完整代码

```python
"""
场景1：单节点步级超时保护

运行方式：
    python example_step_timeout.py

依赖：
    pip install langgraph
"""

from __future__ import annotations

import time
from typing_extensions import TypedDict

from langgraph.graph import END, START, StateGraph


class State(TypedDict):
    task_name: str
    status: str
    elapsed_s: float


def slow_work(state: State) -> State:
    """模拟一个明显超出 step 预算的阻塞任务。"""
    started_at = time.perf_counter()
    print(f"[slow_work] 开始执行: {state['task_name']}")

    # 模拟慢任务
    time.sleep(2.0)

    elapsed = time.perf_counter() - started_at
    print(f"[slow_work] 执行完成，耗时 {elapsed:.2f}s")

    return {
        "task_name": state["task_name"],
        "status": "completed",
        "elapsed_s": round(elapsed, 3),
    }


def build_graph():
    builder = StateGraph(State)
    builder.add_node("slow_work", slow_work)
    builder.add_edge(START, "slow_work")
    builder.add_edge("slow_work", END)

    graph = builder.compile()

    # 关键配置：步级超时只有 1 秒
    graph.step_timeout = 1
    return graph


def main() -> None:
    graph = build_graph()

    input_state: State = {
        "task_name": "slow-report-generation",
        "status": "pending",
        "elapsed_s": 0.0,
    }

    print("=" * 70)
    print("开始执行图：step_timeout = 1s，slow_work = 2s")
    print("=" * 70)

    started_at = time.perf_counter()

    try:
        result = graph.invoke(input_state)
        print("图执行成功：", result)
    except TimeoutError as exc:
        total_elapsed = time.perf_counter() - started_at
        print("\n[捕获到超时]")
        print("异常类型:", type(exc).__name__)
        print("异常信息:", str(exc))
        print(f"外层观察到的总耗时: {total_elapsed:.2f}s")

        print("\n解释：")
        print("1. LangGraph 等待当前 step 完成。")
        print("2. step_timeout = 1 秒，但 slow_work 需要约 2 秒。")
        print("3. 因此当前 step 在预算耗尽后抛 TimeoutError。")


if __name__ == "__main__":
    main()
```


注意：
- 具体耗时会有少量浮动；
- 这个示例是同步阻塞 node，因此从业务感受上很像“node timeout”；
- 但语义上仍然是“单 step 预算被耗尽”。

## 你应该从这个示例学到什么？

### 1. `step_timeout` 足够做第一道兜底

如果你完全没有 timeout，这是最应该先加的一层保护。

### 2. 但它不够表达 I/O 细粒度信息

这个示例只知道：
- 慢了；
- 超时了。

它不知道：
- connect 慢；
- read 慢；
- tool 某一步慢；
- 哪个外部依赖在拖累。

### 3. 所以真正生产里还要继续细化到节点内部

这也是下一个场景要做的事：

> 在 node 里先做自己的 timeout，再把 `step_timeout` 当外层预算。

---

## 适合什么时候用这个模式？

- 本地开发时快速发现卡死 step；
- 简单内部工具图做第一层保护；
- 测试环境里验证 timeout 是否真的生效；
- 教学和排障时做最小可复现。

---

## 不适合什么时候只用它？

- 节点内部有复杂 HTTP / DB / SDK 调用；
- 多代理 / 子图 / 长任务；
- 需要 fallback 或 partial result；
- 需要分清 connect/read/pool 等不同慢点。

这时候请继续看场景 2 和场景 4。

## 变体实现：参数化辅助层

```python
from __future__ import annotations

import asyncio
import time
from dataclasses import dataclass
from typing import Any

@dataclass
class TimeoutBudgets:
    node_timeout_s: float = 1.0
    step_timeout_s: float = 2.0
    consumer_timeout_s: float = 0.5

async def single_step_timeout_demo_remote_call(delay_s: float) -> dict[str, Any]:
    await asyncio.sleep(delay_s)
    return {'ok': True, 'delay_s': delay_s}

async def single_step_timeout_demo_guarded_call(delay_s: float, budgets: TimeoutBudgets) -> dict[str, Any]:
    started_at = time.perf_counter()
    try:
        result = await asyncio.wait_for(single_step_timeout_demo_remote_call(delay_s), timeout=budgets.node_timeout_s)
        result["timed_out"] = False
        result["elapsed_s"] = round(time.perf_counter() - started_at, 3)
        return result
    except asyncio.TimeoutError:
        return {
            "ok": False,
            "timed_out": True,
            "delay_s": delay_s,
            "elapsed_s": round(time.perf_counter() - started_at, 3),
        }

async def single_step_timeout_demo_main() -> None:
    budgets = TimeoutBudgets()
    result = await single_step_timeout_demo_guarded_call(delay_s=1.8, budgets=budgets)
    if result["timed_out"]:
        print("[单节点步级超时保护] 使用 fallback 或降级路径")
    else:
        print("[单节点步级超时保护] 使用主路径结果")
    print(result)

if __name__ == "__main__":
    asyncio.run(single_step_timeout_demo_main())
```
