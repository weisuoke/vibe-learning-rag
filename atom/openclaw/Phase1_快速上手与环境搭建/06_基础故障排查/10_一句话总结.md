# OpenClaw 基础故障排查 - 一句话总结

**OpenClaw 基础故障排查是通过 60 秒标准化诊断流程（status → gateway → doctor → logs）和 openclaw doctor 自动修复工具，快速定位并解决 Gateway 启动、通道连接、配置错误等常见问题的系统化方法，核心价值在于降低故障排查门槛、提高修复效率、实现知识沉淀，使初学者也能在 60 秒内独立解决 80% 的常见问题。**

---

## 核心要点回顾

### 1. 60 秒诊断流程

```bash
openclaw status
openclaw gateway status
openclaw doctor
openclaw doctor --repair  # 如果发现问题
openclaw logs --follow    # 如果问题仍存在
```

### 2. 核心工具

- **openclaw doctor** - 自动化健康检查和修复
- **openclaw logs** - 实时日志分析
- **openclaw gateway status** - 网关状态监控
- **openclaw channels status** - 通道连接诊断

### 3. 常见问题

| 问题 | 快速修复 |
|------|---------|
| Gateway 无法启动 | `openclaw doctor --repair` |
| 端口 18789 冲突 | `openclaw gateway stop` |
| 配置错误 | `openclaw doctor --repair` |
| 通道无回复 | `openclaw pairing approve` |

### 4. 核心价值

- ✅ 60 秒内定位大部分问题
- ✅ 自动修复常见配置错误
- ✅ 降低故障排查门槛
- ✅ 标准化流程可复用
