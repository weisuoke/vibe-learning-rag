# Annotated字段 - 知识点概览

## 知识点简介

**Annotated字段是 LangGraph 状态管理的核心机制，通过 Python 的 `Annotated` 类型注解为状态字段附加 Reducer 函数，实现灵活的状态更新策略。**

在 LangGraph 中，状态管理不仅仅是简单的值覆盖，而是可以通过 Reducer 函数实现累积、合并、去重等复杂的更新逻辑。这种设计让 LangGraph 能够优雅地处理对话历史、多步推理、工具调用结果等需要累积的数据。

[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

### 核心语法

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]  # 使用 Reducer 累积
    counter: int  # 默认覆盖
```

### 为什么重要？

1. **状态化工作流的基础**：LangGraph 的状态管理依赖 Annotated 字段实现复杂的状态更新逻辑
2. **声明式设计**：通过类型注解声明状态更新策略，代码简洁易读
3. **灵活性**：支持内置 Reducer、自定义 Reducer、Lambda 函数等多种方式
4. **性能优化**：TypedDict + Annotated 比 Pydantic 更轻量，支持部分状态更新

[来源: reference/search_annotated_github_01.md]

## 8个核心概念

### 1. Annotated类型注解语法

**`Annotated[type, metadata]` 是 Python 3.9+ 引入的类型注解语法，用于为类型附加元数据。**

在 LangGraph 中，元数据通常是一个 Reducer 函数，用于定义状态更新策略。

```python
from typing import Annotated
import operator

# 基本语法
messages: Annotated[list, operator.add]

# 内部结构
# - __origin__: list (原始类型)
# - __metadata__: (operator.add,) (元数据元组)
```

**关键特性**：
- `get_type_hints(include_extras=True)` 可以获取完整的 Annotated 信息
- 元数据存储在 `__metadata__` 属性中
- 支持多个元数据项，LangGraph 使用最后一个可调用对象作为 Reducer

[来源: reference/source_annotated_02.md]

### 2. Reducer函数签名与验证

**Reducer 函数必须接受两个参数 `(old_value, new_value)` 并返回合并后的值。**

LangGraph 使用 `inspect.signature()` 验证 Reducer 函数签名，确保其符合要求。

```python
# ✅ 正确的 Reducer 签名
def valid_reducer(old: list, new: list) -> list:
    return old + new

# ❌ 错误的签名（只有一个参数）
def invalid_reducer(value):
    return value
```

**验证逻辑**（源码）：
```python
def _is_field_binop(typ: type[Any]) -> BinaryOperatorAggregate | None:
    if hasattr(typ, "__metadata__"):
        meta = typ.__metadata__
        if len(meta) >= 1 and callable(meta[-1]):
            sig = signature(meta[-1])
            params = list(sig.parameters.values())
            if sum(p.kind in (p.POSITIONAL_ONLY, p.POSITIONAL_OR_KEYWORD) for p in params) == 2:
                return BinaryOperatorAggregate(typ, meta[-1])
            else:
                raise ValueError(f"Invalid reducer signature. Expected (a, b) -> c. Got {sig}")
    return None
```

[来源: reference/source_annotated_02.md]

### 3. 内置Reducer函数

**LangGraph 提供了三个常用的内置 Reducer 函数。**

| Reducer | 用途 | 示例 |
|---------|------|------|
| `operator.add` | 列表/字符串拼接 | `[1, 2] + [3]` = `[1, 2, 3]` |
| `operator.or_` | 字典合并 | `{a: 1} \| {b: 2}` = `{a: 1, b: 2}` |
| `add_messages` | 消息列表管理（按 ID 更新） | 智能合并对话历史 |

**使用示例**：
```python
from typing import Annotated
from typing_extensions import TypedDict
import operator
from langgraph.graph.message import add_messages

class State(TypedDict):
    # 列表累积
    steps: Annotated[list[str], operator.add]

    # 字典合并
    metadata: Annotated[dict, operator.or_]

    # 消息管理
    messages: Annotated[list[BaseMessage], add_messages]
```

**add_messages 特性**：
- 按消息 ID 更新现有消息
- 自动追加新消息
- 支持 `RemoveMessage` 删除消息

[来源: reference/context7_langgraph_01.md | LangGraph 官方文档]

### 4. 自定义Reducer函数

**可以编写自定义 Reducer 函数实现特定的合并逻辑。**

**模式1：简单函数**
```python
def merge_with_dedup(old: list, new: list) -> list:
    """去重合并"""
    seen = set(old)
    return old + [x for x in new if x not in seen]

class State(TypedDict):
    items: Annotated[list, merge_with_dedup]
```

**模式2：Lambda 函数**
```python
class State(TypedDict):
    # 优先使用新值
    value: Annotated[str, lambda x, y: y or x]

    # 条件覆盖
    data: Annotated[dict | None, lambda x, y: y if y is not None else x]
```

**模式3：Reducer 工厂函数**
```python
def merge_with_limit(max_size: int):
    """限制大小的合并"""
    def reducer(old: list, new: list) -> list:
        combined = old + new
        return combined[-max_size:]
    return reducer

class State(TypedDict):
    recent_messages: Annotated[list, merge_with_limit(100)]
```

### 5. BinaryOperatorAggregate实现

**`BinaryOperatorAggregate` 是 LangGraph 内部用于实现 Reducer 逻辑的 Channel 类。**

**核心实现**（源码）：
```python
class BinaryOperatorAggregate(Generic[Value], BaseChannel[Value, Value, Value]):
    def __init__(self, typ: type[Value], operator: Callable[[Value, Value], Value]):
        super().__init__(typ)
        self.operator = operator
        self.value = typ()  # 初始化为空值

    def update(self, values: Sequence[Value]) -> bool:
        if not values:
            return False
        if self.value is MISSING:
            self.value = values[0]  # 第一个值直接赋值
            values = values[1:]
        for value in values:
            self.value = self.operator(self.value, value)  # 应用 Reducer
        return True
```

**关键设计**：
1. **初始化策略**：第一个值直接赋值，后续值通过 operator 合并
2. **类型转换**：自动将抽象类型（如 `Sequence`）转换为具体类型（如 `list`）
3. **Overwrite 支持**：可以使用 `Overwrite(value)` 强制覆盖状态

[来源: reference/source_annotated_02.md]

### 6. 类型提示解析机制

**LangGraph 使用 `get_type_hints(include_extras=True)` 解析 Annotated 字段。**

**解析流程**：
```
用户定义状态
    ↓
class State(TypedDict):
    messages: Annotated[list, operator.add]
    ↓
StateGraph.__init__()
    ↓
_get_channels(schema)
    ↓
get_type_hints(schema, include_extras=True)
    ↓
返回: {'messages': Annotated[list, operator.add]}
    ↓
_get_channel('messages', Annotated[list, operator.add])
    ↓
_is_field_binop(Annotated[list, operator.add])
    ↓
检查 __metadata__ = (operator.add,)
    ↓
验证 operator.add 签名: (a, b) -> c
    ↓
创建 BinaryOperatorAggregate(list, operator.add)
    ↓
存储到 self.channels['messages']
```

**处理优先级**：
1. Managed value（如 `IsLastStep`）
2. 显式 Channel（如 `EphemeralValue`）
3. Binop/Reducer 函数（如 `operator.add`）
4. 默认 LastValue（覆盖模式）

[来源: reference/source_annotated_02.md]

### 7. 状态更新策略对比

**LangGraph 支持两种状态更新策略：默认覆盖和 Reducer 合并。**

| 策略 | 定义方式 | 更新行为 | 适用场景 |
|------|----------|----------|----------|
| **默认覆盖** | `counter: int` | 新值直接覆盖旧值 | 简单标量值 |
| **Reducer 合并** | `messages: Annotated[list, operator.add]` | 通过 Reducer 函数合并 | 需要累积的数据 |

**示例对比**：
```python
class State(TypedDict):
    # 覆盖策略
    current_step: str
    counter: int

    # Reducer 策略
    messages: Annotated[list, operator.add]
    metadata: Annotated[dict, operator.or_]

# 节点返回
def node(state):
    return {
        "current_step": "step2",  # 覆盖
        "counter": state["counter"] + 1,  # 覆盖
        "messages": [new_message],  # 累积（不需要 state["messages"] +）
        "metadata": {"new_key": "value"}  # 合并
    }
```

**性能对比**（2025年基准测试）：
- TypedDict + Annotated: ~0.1ms per update
- Pydantic: ~1.0ms per update

[来源: reference/search_annotated_github_01.md]

### 8. Overwrite覆盖机制

**`Overwrite` 类允许在使用 Reducer 的字段上强制覆盖状态。**

**使用方式**：
```python
from langgraph.channels.binop import Overwrite

# 方式1：使用 Overwrite 类
return {"messages": Overwrite([new_message])}

# 方式2：使用字典语法
from langgraph.constants import OVERWRITE
return {"messages": {OVERWRITE: [new_message]}}
```

**冲突检测**：
```python
# ❌ 错误：一个 super-step 中只能有一个 Overwrite
if seen_overwrite:
    raise InvalidUpdateError("Can receive only one Overwrite value per super-step.")
```

**使用场景**：
- 重置累积的状态
- 清空消息历史
- 强制覆盖而非合并

[来源: reference/source_annotated_02.md]

## 4个实战场景

### 场景1：基础Annotated使用

**使用 `operator.add` 累积列表数据。**

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator
from langgraph.graph import START, END, StateGraph

class State(TypedDict):
    messages: Annotated[list[str], operator.add]
    counter: int

def first_node(state: State) -> dict:
    return {"messages": ["Hello from first node"], "counter": state["counter"] + 1}

def second_node(state: State) -> dict:
    return {"messages": ["Hello from second node"], "counter": state["counter"] + 1}

# 构建图
builder = StateGraph(State)
builder.add_node("first", first_node)
builder.add_node("second", second_node)
builder.add_edge(START, "first")
builder.add_edge("first", "second")
builder.add_edge("second", END)

graph = builder.compile()

# 执行
result = graph.invoke({"messages": [], "counter": 0})
print(result)
# {'messages': ['Hello from first node', 'Hello from second node'], 'counter': 2}
```

### 场景2：消息列表管理

**使用 `add_messages` 管理对话历史，支持按 ID 更新消息。**

```python
from typing import Annotated
from typing_extensions import TypedDict
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph.message import add_messages

class ConversationState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    user_id: str

def chatbot_node(state: ConversationState) -> dict:
    # 只返回新消息，add_messages 会自动合并
    response = AIMessage(content="Hello! How can I help you?")
    return {"messages": [response]}

# add_messages 特性演示
msgs1 = [HumanMessage(content="Hello", id="1")]
msgs2 = [AIMessage(content="Hi there!", id="2")]
result = add_messages(msgs1, msgs2)
# [HumanMessage(content='Hello', id='1'), AIMessage(content='Hi there!', id='2')]

# 覆盖现有消息
msgs3 = [HumanMessage(content="Hello again", id="1")]
result = add_messages(result, msgs3)
# [HumanMessage(content='Hello again', id='1'), AIMessage(content='Hi there!', id='2')]
```

### 场景3：自定义Reducer实战

**编写自定义 Reducer 实现去重、限制大小等逻辑。**

```python
from typing import Annotated
from typing_extensions import TypedDict

# 去重合并
def merge_with_dedup(old: list, new: list) -> list:
    seen = set(old)
    return old + [x for x in new if x not in seen]

# 限制大小
def merge_with_limit(max_size: int):
    def reducer(old: list, new: list) -> list:
        combined = old + new
        return combined[-max_size:]
    return reducer

# 条件覆盖
def conditional_merge(old: dict, new: dict) -> dict:
    """只合并非空值"""
    result = old.copy()
    for k, v in new.items():
        if v is not None:
            result[k] = v
    return result

class State(TypedDict):
    unique_items: Annotated[list, merge_with_dedup]
    recent_messages: Annotated[list, merge_with_limit(100)]
    config: Annotated[dict, conditional_merge]
```

### 场景4：复杂状态管理

**分离不同类型的数据，避免状态膨胀。**

```python
from typing import Annotated
from typing_extensions import TypedDict
import operator
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages

class ToolCall(TypedDict):
    tool: str
    args: dict

class ToolResult(TypedDict):
    tool: str
    result: str

class ReActState(TypedDict):
    # 分离不同类型的数据
    messages: Annotated[list[BaseMessage], add_messages]
    tool_calls: Annotated[list[ToolCall], operator.add]
    tool_results: Annotated[list[ToolResult], operator.add]
    reasoning_steps: Annotated[list[str], operator.add]

    # 元数据（覆盖模式）
    user_id: str
    session_id: str
    current_step: str

def agent_node(state: ReActState) -> dict:
    # 只返回新增的数据
    return {
        "reasoning_steps": ["Analyzing user query..."],
        "tool_calls": [{"tool": "search", "args": {"query": "..."}}],
        "current_step": "tool_execution"
    }
```

**优势**：
- 避免所有数据混在一起
- 更好的可维护性
- 支持选择性序列化
- 减少 token 消耗
