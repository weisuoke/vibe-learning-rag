## 步骤 2：基础断点调试

### 2.1 设置断点

**示例代码**：`src/core/config.ts`

```typescript
// src/core/config.ts
export function loadConfig(path: string): Config {
  // 在这里设置断点（点击行号左侧）
  const raw = fs.readFileSync(path, 'utf-8')

  // 设置条件断点：只在 path 包含 'test' 时暂停
  const config = JSON.parse(raw)

  // 设置日志断点：不暂停，只输出日志
  return validateConfig(config)
}
```

**断点类型**：
1. **普通断点**：点击行号左侧，红点表示断点
2. **条件断点**：右键断点 → Edit Breakpoint → 输入条件
3. **日志断点**：右键断点 → Edit Breakpoint → Log Message

{

### 场景：调试 RAG 检索问题

**问题描述**：用户查询 "OpenClaw 如何配置" 返回了不相关的结果。

**调试步骤**：

```typescript
// 1. 在 processMessage 设置断点
// src/core/agent.ts
export async function processMessage(text: string) {
  // 断点1：检查输入
  console.log('Query:', text)

  // 断点2：检查 embedding
  const embedding = await generateEmbedding(text)
  console.log('Embedding:', embedding.slice(0, 5))

  // 断点3：检查检索结果
  const results = await searchVectorStore(embedding, { limit: 5 })
  console.log('Results:', results.map(r => ({
    score: r.score,
    text: r.text.slice(0, 50)
  })))

  // 断点4：检查 rerank 结果
  const reranked = await rerank(results, text)
  console.log('Reranked:', reranked.map(r => ({
    score: r.score,
    text: r.text.slice(0, 50)
  })))

  // 断点5：检查上下文
  const context = reranked.slice(0, 3).map(r => r.text).join('\n\n')
  console.log('Context length:', context.length)

  // 断点6：检查 LLM 响应
  const response = await callLLM({ context, query: text })
  console.log('Response:', response.slice(0, 100))

  return response
}
```

## 核心洞察

### 1. 调试不是猜测

**表面**：看到错误就改代码

**实际**：
- 设置断点，观察变量
- 单步执行，理解流程
- 分析调用栈，找到根因
- 验证修复，确保正确

---

### 2. 工具链很重要

**表面**：用 console.log 调试

**实际**：
- VS Code 断点调试更高效
- Source map 确保准确性
- 调试配置提升体验
- 性能工具定位瓶颈

---

### 3. 异步调试需要技巧

**表面**：async/await 很简单

**实际**：
- 理解 Promise 链
- 查看异步调用栈
- 处理并发问题
- 避免竞态条件

---

## 一句话总结

**OpenClaw 开发调试的关键是：配置 VS Code 调试环境，掌握断点、变量检查、调用栈分析技巧，结合性能工具定位问题。**
