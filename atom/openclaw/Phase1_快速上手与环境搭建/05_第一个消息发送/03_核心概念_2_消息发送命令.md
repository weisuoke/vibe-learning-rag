# 核心概念 2：消息发送命令

> **深入理解 `openclaw message send` 命令的使用和实现**

---

## 概念定义

**消息发送命令（Message Send Command）** 是 OpenClaw 提供的核心 CLI 命令，用于向已配对的通道发送消息。通过 `openclaw message send` 命令，用户可以用统一的接口向 WhatsApp、Telegram、Discord 等多个平台发送消息。

**核心价值**：
- 🎯 **统一接口**：一条命令支持所有通道
- 🚀 **简单易用**：无需编写代码，直接命令行操作
- 🔄 **自动路由**：自动选择已配对的通道
- 📊 **实时反馈**：立即显示发送状态

---

## 命令语法

### 基础语法

```bash
openclaw message send <text> [options]
```

### 参数说明

| 参数 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `<text>` | string | ✅ | 要发送的消息内容 |
| `--channel` | string | ❌ | 指定通道类型（whatsapp、telegram、discord） |
| `--to` | string | ❌ | 指定接收者（覆盖配对设置） |
| `--pairing-id` | string | ❌ | 指定配对 ID |
| `--format` | string | ❌ | 消息格式（text、markdown、html） |
| `--attachment` | string | ❌ | 附件文件路径 |
| `--silent` | boolean | ❌ | 静默模式，不显示输出 |
| `--dry-run` | boolean | ❌ | 模拟发送，不实际发送 |

## 使用示例

```bash

# 输出:

# ✓ Recipient: +1234567890
# ✓ Message ID: msg_abc123
# ✓ Status: delivered
```

**说明**：
- 自动发送到所有已配对的通道
- 如果有多个配对，会发送到所有通道

### 示例 2：指定通道发送

```bash
# 只发送到 WhatsApp

# 只发送到 Discord
openclaw message send "Hello, Discord!" --channel discord
```

**说明**：
- 使用 `--channel` 参数指定通道
- 只会发送到指定的通道

```bash

```

### 示例 4：发送 Markdown 格式消息


**支持的 Markdown 语法**：
- `**bold**` - 粗体

- `[text](url)` - 链接

- ` ```code block``` ` - 代码块

### 示例 5：发送带附件的消息

```bash
# 发送图片

# 发送文档
openclaw message send "Here's the report" --attachment ./report.pdf

# 发送多个附件
openclaw message send "Multiple files" \
  --attachment ./file1.pdf \
  --attachment ./file2.png
```

**支持的附件类型**：
- 图片：`.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`
- 文档：`.pdf`, `.doc`, `.docx`, `.txt`
- 音频：`.mp3`, `.wav`, `.ogg`
- 视频：`.mp4`, `.mov`, `.avi`

### 示例 6：指定接收者

```bash

# 使用配对 ID

```

### 示例 7：模拟发送（Dry Run）

```bash
# 模拟发送，不实际发送消息

# ✓ Channel: WhatsApp

# ✓ Message: "Test message"
```

**用途**：
- 测试命令是否正确
- 验证配对是否有效
- 检查消息格式

### 示例 8：静默模式


## 命令实现原理

### 命令注册

**源码位置**：`src/cli/program/message/register.send.ts`

```typescript
// src/cli/program/message/register.send.ts:23
import { Command } from 'commander';
import { sendMessage } from '../../commands/message';

export function registerSendCommand(program: Command): void {
  program
    .command('send')
    .description('Send a message to paired channels')
    .argument('<text>', 'Message text to send')
    .option('--channel <type>', 'Specify channel type (whatsapp, telegram, discord)')
    .option('--to <recipient>', 'Specify recipient (overrides pairing)')
    .option('--pairing-id <id>', 'Specify pairing ID')
    .option('--format <format>', 'Message format (text, markdown, html)', 'text')
    .option('--attachment <path>', 'Attachment file path (can be used multiple times)', collect, [])
    .option('--silent', 'Silent mode, no output')
    .option('--dry-run', 'Simulate sending without actually sending')
    .action(async (text: string, options: SendOptions) => {
      try {
        await sendMessage(text, options);
      } catch (error) {
        console.error('Failed to send message:', error.message);
        process.exit(1);
      }
    });
}

// 收集多个附件
function collect(value: string, previous: string[]): string[] {
  return previous.concat([value]);
}
```

**关键点**：
- 使用 `commander` 库注册命令
- 支持多个选项参数
- 错误处理和退出码

### 命令执行

**源码位置**：`src/commands/message.ts`

```typescript
// src/commands/message.ts:34
import { OutboundSendService } from '../infra/outbound/outbound-send-service';
import { PairingService } from '../pairing/pairing-service';
import { MessageActionRunner } from '../infra/outbound/message-action-runner';

export interface SendOptions {
  channel?: string;
  to?: string;
  pairingId?: string;
  format?: 'text' | 'markdown' | 'html';
  attachment?: string[];
  silent?: boolean;
  dryRun?: boolean;
}

export async function sendMessage(text: string, options: SendOptions): Promise<void> {
  // 1. 验证输入
  if (!text || text.trim().length === 0) {
    throw new Error('Message text cannot be empty');
  }

  // 2. 获取配对信息
  const pairings = await getPairings(options);
  if (pairings.length === 0) {
    throw new Error('No active pairings found. Please run "openclaw pairing start" first.');
  }

  // 3. 准备消息
  const message = prepareMessage(text, options);

  // 4. 发送消息
  const results = await Promise.all(
    pairings.map(pairing => sendToChannel(pairing, message, options))
  );

  // 5. 显示结果
  if (!options.silent) {
    displayResults(results);
  }
}

// 获取配对信息
async function getPairings(options: SendOptions): Promise<Pairing[]> {
  const pairingService = new PairingService();

  // 如果指定了配对 ID，只使用该配对
  if (options.pairingId) {
    const pairing = await pairingService.getPairingById(options.pairingId);
    return pairing ? [pairing] : [];
  }

  // 如果指定了通道，只使用该通道的配对
  if (options.channel) {
    return await pairingService.getPairingsByChannel(options.channel);
  }

  // 否则，使用所有活跃的配对
  return await pairingService.getActivePairings();
}

// 准备消息
function prepareMessage(text: string, options: SendOptions): OutboundMessage {
  return {
    text,
    format: options.format || 'text',
    attachments: options.attachment || [],
    timestamp: Date.now()
  };
}

// 发送到通道
async function sendToChannel(
  pairing: Pairing,
  message: OutboundMessage,
  options: SendOptions
): Promise<SendResult> {
  // Dry run 模式
  if (options.dryRun) {
    return {
      success: true,
      channel: pairing.channelType,
      recipient: pairing.channelId,
      messageId: 'dry_run',
      status: 'simulated'
    };
  }

  // 实际发送
  const actionRunner = new MessageActionRunner();
  return await actionRunner.send({
    channelType: pairing.channelType,
    channelId: pairing.channelId,
    message
  });
}

// 显示结果
function displayResults(results: SendResult[]): void {
  console.log('✓ Message sent successfully');

  for (const result of results) {
    console.log(`✓ Channel: ${result.channel}`);
    console.log(`✓ Recipient: ${result.recipient}`);
    console.log(`✓ Message ID: ${result.messageId}`);
    console.log(`✓ Status: ${result.status}`);
  }
}
```

### 消息发送流程

```
用户命令
    ↓
CLI 解析参数
    ↓
验证输入
    ↓
获取配对信息
    ├─ 指定配对 ID？ → 使用该配对
    ├─ 指定通道？ → 使用该通道的所有配对
    └─ 否则 → 使用所有活跃配对
    ↓
准备消息
    ├─ 格式化文本
    ├─ 处理附件
    └─ 添加元数据
    ↓
发送消息
    ├─ Dry Run？ → 模拟发送
    └─ 否则 → 实际发送
    ↓
显示结果
    ├─ 静默模式？ → 不显示
    └─ 否则 → 显示详细信息
```

## 消息发送服务

### OutboundSendService

**源码位置**：`src/infra/outbound/outbound-send-service.ts`

```typescript
// src/infra/outbound/outbound-send-service.ts:45
export class OutboundSendService {
  private channelPlugins: Map<string, ChannelPlugin>;

  constructor() {
    this.channelPlugins = new Map();
    this.loadChannelPlugins();
  }

  // 发送消息
  async send(request: SendRequest): Promise<SendResult> {
    // 1. 选择通道插件
    const plugin = this.selectChannelPlugin(request.channelType);
    if (!plugin) {
      throw new Error(`Channel plugin not found: ${request.channelType}`);
    }

    // 2. 转换消息格式
    const platformMessage = this.convertMessage(request.message, plugin);

    // 3. 发送到平台
    try {
      const result = await plugin.send(platformMessage);
      return {
        success: true,
        channel: request.channelType,
        recipient: request.channelId,
        messageId: result.id,
        status: 'delivered'
      };
    } catch (error) {
      return {
        success: false,
        channel: request.channelType,
        recipient: request.channelId,
        error: error.message,
        status: 'failed'
      };
    }
  }

  // 选择通道插件
  private selectChannelPlugin(channelType: string): ChannelPlugin | undefined {
    return this.channelPlugins.get(channelType);
  }

  // 转换消息格式
  private convertMessage(message: OutboundMessage, plugin: ChannelPlugin): any {
    // 根据通道类型转换消息格式
    switch (plugin.name) {
      case 'whatsapp':
        return this.convertToWhatsApp(message);
      case 'telegram':
        return this.convertToTelegram(message);
      case 'discord':
        return this.convertToDiscord(message);
      default:
        return message;
    }
  }

  // 转换为 WhatsApp 格式
  private convertToWhatsApp(message: OutboundMessage): any {
    return {
      to: message.recipient,
      body: message.text,
      // WhatsApp 特定字段
    };
  }

  // 转换为 Telegram 格式
  private convertToTelegram(message: OutboundMessage): any {
    return {
      chat_id: message.recipient,
      text: message.text,
      parse_mode: message.format === 'markdown' ? 'Markdown' : undefined,
      // Telegram 特定字段
    };
  }

  // 转换为 Discord 格式
  private convertToDiscord(message: OutboundMessage): any {
    return {
      content: message.text,
      // Discord 特定字段
    };
  }

  // 加载通道插件
  private loadChannelPlugins(): void {
    // 动态加载所有通道插件
    const plugins = [
      new WhatsAppPlugin(),
      new TelegramPlugin(),
      new DiscordPlugin(),
      // ... 更多插件
    ];

    for (const plugin of plugins) {
      this.channelPlugins.set(plugin.name, plugin);
    }
  }
}
```

```bash
# 从文件读取消息列表
while IFS= read -r message; do
```

```bash
0 9 * * * openclaw message send "Good morning!"
```

### 3. 条件发送

```bash
# 根据条件发送不同的消息
if [ $(date +%u) -eq 1 ]; then
  openclaw message send "Happy Monday!"
else
  openclaw message send "Have a great day!"
fi
```

### 4. 集成到脚本

```bash
#!/bin/bash

# 监控脚本
CPU_USAGE=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | cut -d'%' -f1)

if (( $(echo "$CPU_USAGE > 80" | bc -l) )); then
  openclaw message send "⚠️ Alert: CPU usage is ${CPU_USAGE}%"
```

### 5. 错误处理

```bash
# 捕获错误并重试
MAX_RETRIES=3
RETRY_COUNT=0

while [ $RETRY_COUNT -lt $MAX_RETRIES ]; do
  if openclaw message send "Important message" --silent; then
    echo "Message sent successfully"
    break
  else
    echo "Failed to send message, retrying..."
    RETRY_COUNT=$((RETRY_COUNT + 1))
    sleep 5
  fi
done

if [ $RETRY_COUNT -eq $MAX_RETRIES ]; then
  echo "Failed to send message after $MAX_RETRIES attempts"
  exit 1
fi
```

## 错误处理

### 常见错误

#### 错误 1：没有活跃的配对

```bash
$ openclaw message send "Hello"
✗ Error: No active pairings found
✗ Please run "openclaw pairing start" first
```

**解决方案**：
```bash
# 启动配对流程
openclaw pairing start
```

#### 错误 2：消息为空

```bash
$ openclaw message send ""
✗ Error: Message text cannot be empty
```

**解决方案**：
```bash
# 提供有效的消息内容
openclaw message send "Valid message"
```

#### 错误 3：附件不存在

```bash
$ openclaw message send "File" --attachment ./missing.pdf
✗ Error: Attachment file not found: ./missing.pdf
```

**解决方案**：
```bash
# 检查文件路径
ls -la ./missing.pdf

# 使用正确的路径
openclaw message send "File" --attachment ./correct/path/file.pdf
```

#### 错误 4：通道不支持

```bash
$ openclaw message send "Hello" --channel unknown
✗ Error: Channel plugin not found: unknown
✗ Supported channels: whatsapp, telegram, discord, slack
```

**解决方案**：
```bash
# 使用支持的通道
openclaw message send "Hello" --channel whatsapp
```

#### 错误 5：发送失败

```bash
$ openclaw message send "Hello"
✗ Error: Failed to send message
✗ Reason: Network timeout
```

**解决方案**：
```bash
# 检查网络连接
ping google.com

# 检查 Gateway 状态
openclaw gateway status

# 重试发送
openclaw message send "Hello"
```

### 2. 消息队列

```typescript
// 使用队列避免频率限制
class MessageQueue {
  private queue: Message[] = [];
  private processing = false;

  async enqueue(message: Message): Promise<void> {
    this.queue.push(message);
    if (!this.processing) {
      await this.process();
    }
  }

  private async process(): Promise<void> {
    this.processing = true;
    while (this.queue.length > 0) {
      const message = this.queue.shift();
      await this.send(message);
      await this.delay(1000); // 1秒延迟
    }
    this.processing = false;
  }
}
```

```typescript
class PairingCache {

  private ttl = 5 * 60 * 1000; // 5分钟

  async get(id: string): Promise<Pairing | undefined> {
    const cached = this.cache.get(id);
    if (cached && !this.isExpired(cached)) {
      return cached;
    }

    if (pairing) {
      this.cache.set(id, pairing);

    return pairing;
```
