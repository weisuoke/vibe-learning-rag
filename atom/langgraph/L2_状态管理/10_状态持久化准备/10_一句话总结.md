# 状态持久化准备 - 一句话总结

## 核心总结

**状态持久化准备是从设计之初就确保状态可序列化（Pydantic/dataclass 替代普通类）、大小可控（消息修剪与外部存储卸载）、敏感数据安全（EncryptedSerializer 加密而非仅靠 SecretStr），通过三层管道（Channel→Serializer→Storage）为 LangGraph 的容错恢复、时间旅行和人机协作奠定基础的工程实践。**

---

## 关键记忆点

| 维度 | 核心要点 |
|------|----------|
| 本质 | 从第一天就为"保存和恢复"设计状态 |
| 可序列化 | Pydantic/dataclass/TypedDict 可以，普通自定义类不行 |
| 序列化器 | JsonPlusSerializer 名字叫 Json，实际用 msgpack |
| 类型标签 | `dumps_typed()` 返回 `(type_tag, bytes)`，自描述格式 |
| 大小控制 | 消息历史 + 工具输出是两大杀手，必须有修剪策略 |
| 敏感数据 | SecretStr 只防日志，EncryptedSerializer 才防存储 |
| 三层架构 | 数据层(Channel) + 序列化层(Serializer) + 存储层(Saver) |
| 加密机制 | AES-EAX 认证加密，type_tag 用 `+` 分隔实现向后兼容 |

---

## 速记公式

```
状态持久化准备 = 可序列化设计 + 大小控制 + 敏感数据保护
```
