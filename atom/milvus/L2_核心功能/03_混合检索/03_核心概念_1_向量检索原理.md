# 核心概念_1_向量检索原理

## 概述

向量检索是混合检索的核心组成部分之一，负责捕获查询和文档之间的语义相似性。本文档深入讲解向量检索的原理、类型和实现。

---

## 核心概念1：稠密向量检索（Dense Vector Search）

**稠密向量检索是将文本转换为固定维度的连续向量，通过计算向量间的距离或相似度来衡量语义相关性的检索方式。**

### 什么是稠密向量？

稠密向量（Dense Vector）是指向量的每个维度都有非零值的向量表示。

**示例：**
```python
# 稠密向量示例（768维）
dense_vector = [0.23, -0.45, 0.67, 0.12, ..., 0.89]  # 768个浮点数

# 特点：
# - 每个维度都有值（非零）
# - 维度固定（如768、1536等）
# - 值连续（浮点数）
```

### 稠密向量的生成

稠密向量通过Embedding模型生成：

```python
from sentence_transformers import SentenceTransformer

# 加载Embedding模型
model = SentenceTransformer('all-MiniLM-L6-v2')

# 生成稠密向量
text = "Python机器学习库"
dense_vector = model.encode(text)

print(f"向量维度: {len(dense_vector)}")  # 384
print(f"前5个值: {dense_vector[:5]}")
# 输出: [0.23, -0.45, 0.67, 0.12, 0.34]
```

### 稠密向量检索原理

**核心思想：** 语义相似的文本在向量空间中距离更近

```python
from pymilvus import MilvusClient, AnnSearchRequest

client = MilvusClient(uri="http://localhost:19530")

# 查询向量
query_text = "如何学习Python"
query_vector = model.encode(query_text)

# 稠密向量检索
search_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="dense_vector",
    param={
        "metric_type": "COSINE",  # 余弦相似度
        "params": {"ef": 64}      # HNSW参数
    },
    limit=10
)

results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="dense_vector",
    limit=10,
    output_fields=["text"]
)

# 结果：
# 1. "Python入门教程"        (score: 0.92)
# 2. "Python编程指南"        (score: 0.88)
# 3. "机器学习Python实战"    (score: 0.85)
```

### 相似度度量方式

Milvus支持三种稠密向量相似度度量：

| 度量方式 | 公式 | 值域 | 适用场景 |
|---------|------|------|---------|
| **L2距离** | √(Σ(xi-yi)²) | [0, +∞] | 图像检索、推荐系统 |
| **内积IP** | Σ(xi×yi) | [-∞, +∞] | 归一化向量 |
| **余弦相似度COSINE** | Σ(xi×yi) / (‖x‖×‖y‖) | [-1, 1] | 文本语义检索（推荐） |

**代码示例：**
```python
# L2距离
search_params_l2 = {
    "metric_type": "L2",
    "params": {"ef": 64}
}

# 内积IP
search_params_ip = {
    "metric_type": "IP",
    "params": {"ef": 64}
}

# 余弦相似度（推荐用于文本）
search_params_cosine = {
    "metric_type": "COSINE",
    "params": {"ef": 64}
}
```

### 在混合检索中的应用

```python
from pymilvus import AnnSearchRequest, RRFRanker

# 稠密向量检索（语义匹配）
dense_req = AnnSearchRequest(
    data=[dense_query_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20  # 召回20个候选
)

# 与BM25检索结合
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20
)

# 混合检索
results = client.hybrid_search(
    collection_name="documents",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=10
)
```

## 核心概念2：稀疏向量检索（Sparse Vector Search）

**稀疏向量检索是将文本转换为高维稀疏向量（大部分维度为0），通过BM25算法计算关键词匹配度的检索方式。**

### 什么是稀疏向量？

稀疏向量（Sparse Vector）是指向量的大部分维度为0，只有少数维度有非零值的向量表示。

**示例：**
```python
# 稀疏向量示例（字典格式）
sparse_vector = {
    27: 0.5,      # 第27维：权重0.5
    100: 0.3,     # 第100维：权重0.3
    5369: 0.6     # 第5369维：权重0.6
}

# 特点：
# - 大部分维度为0（未存储）
# - 只存储非零维度
# - 维度可变（取决于词汇表大小）
```

### 稀疏向量的生成（BM25）

Milvus 2.5+内置BM25，自动生成稀疏向量：

```python
from pymilvus import MilvusClient, DataType, Function, FunctionType

client = MilvusClient(uri="http://localhost:19530")

# 创建Schema
schema = client.create_schema()

schema.add_field(field_name="id", datatype=DataType.INT64, is_primary=True, auto_id=True)
schema.add_field(field_name="text", datatype=DataType.VARCHAR, max_length=1000, enable_analyzer=True)
schema.add_field(field_name="dense_vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="sparse_vector", datatype=DataType.SPARSE_FLOAT_VECTOR)

# 定义BM25函数（自动生成稀疏向量）
bm25_function = Function(
    name="text_bm25_emb",
    input_field_names=["text"],
    output_field_names=["sparse_vector"],
    function_type=FunctionType.BM25
)
schema.add_function(bm25_function)

# 创建索引
index_params = client.prepare_index_params()

index_params.add_index(
    field_name="dense_vector",
    index_type="HNSW",
    metric_type="COSINE",
    params={"M": 16, "efConstruction": 200}
)

index_params.add_index(
    field_name="sparse_vector",
    index_type="SPARSE_INVERTED_INDEX",
    metric_type="BM25"
)

# 创建Collection
client.create_collection(
    collection_name="hybrid_demo",
    schema=schema,
    index_params=index_params
)
```

### BM25算法原理

**BM25（Best Matching 25）** 是一种基于概率的排序函数，用于计算查询和文档的相关性。

**公式：**
```
BM25(D, Q) = Σ IDF(qi) × (f(qi, D) × (k1 + 1)) / (f(qi, D) + k1 × (1 - b + b × |D| / avgdl))
```

**参数：**
- `D`：文档
- `Q`：查询
- `qi`：查询中的第i个词
- `f(qi, D)`：词qi在文档D中的频率
- `|D|`：文档D的长度
- `avgdl`：平均文档长度
- `k1`：词频饱和参数（通常为1.2-2.0）
- `b`：长度归一化参数（通常为0.75）
- `IDF(qi)`：逆文档频率

**直觉理解：**
- 词频越高，分数越高（但有饱和效应）
- 文档越短，分数越高（长度惩罚）
- 词越稀有（IDF高），分数越高

### 稀疏向量检索原理

```python
# 查询文本
query_text = "Python 3.11新特性"

# 稀疏向量检索（BM25）
sparse_req = AnnSearchRequest(
    data=[query_text],  # 直接传入文本
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=10
)

results = client.search(
    collection_name="hybrid_demo",
    data=[query_text],
    anns_field="sparse_vector",
    limit=10,
    output_fields=["text"]
)

# 结果：
# 1. "Python 3.11新特性详解"    (score: 8.5)
# 2. "Python 3.11发布说明"      (score: 7.2)
# 3. "Python新版本特性"         (score: 5.8)
```

### 稠密向量 vs 稀疏向量

| 维度 | 稠密向量 | 稀疏向量 |
|-----|---------|---------|
| **表示方式** | 连续浮点数 | 稀疏键值对 |
| **维度** | 固定（768、1536等） | 可变（词汇表大小） |
| **生成方式** | Embedding模型 | BM25算法 |
| **语义理解** | ✅ 强 | ❌ 弱 |
| **关键词匹配** | ❌ 弱 | ✅ 强 |
| **同义词处理** | ✅ 好 | ❌ 差 |
| **精确术语** | ❌ 差 | ✅ 好 |
| **内存占用** | 高 | 低 |
| **检索速度** | 中 | 快 |

## 核心概念3：多向量检索（Multi-Vector Search）

**多向量检索是在同一个Collection中对多个向量字段同时进行ANN检索，并通过重排序策略合并结果的检索方式。**

### 什么是多向量检索？

多向量检索允许一个文档有多个向量表示，每个向量捕获不同的信息维度。

**应用场景：**
1. **文本+图像**：产品描述（文本向量）+ 产品图片（图像向量）
2. **标题+正文**：文章标题（向量1）+ 文章正文（向量2）
3. **稠密+稀疏**：语义向量（稠密）+ 关键词向量（稀疏）

### Schema设计

```python
from pymilvus import MilvusClient, DataType

client = MilvusClient(uri="http://localhost:19530")

schema = client.create_schema()

# 主键
schema.add_field(field_name="product_id", datatype=DataType.INT64, is_primary=True)

# 文本字段
schema.add_field(field_name="title", datatype=DataType.VARCHAR, max_length=500)
schema.add_field(field_name="description", datatype=DataType.VARCHAR, max_length=2000)

# 多个向量字段
schema.add_field(field_name="title_vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="description_vector", datatype=DataType.FLOAT_VECTOR, dim=768)
schema.add_field(field_name="image_vector", datatype=DataType.FLOAT_VECTOR, dim=512)
schema.add_field(field_name="sparse_vector", datatype=DataType.SPARSE_FLOAT_VECTOR)

# 元数据
schema.add_field(field_name="category", datatype=DataType.VARCHAR, max_length=100)
schema.add_field(field_name="price", datatype=DataType.FLOAT)
```

### 多向量检索实现

```python
from pymilvus import AnnSearchRequest, WeightedRanker

# 标题向量检索
title_req = AnnSearchRequest(
    data=[title_query_vector],
    anns_field="title_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20
)

# 描述向量检索
description_req = AnnSearchRequest(
    data=[description_query_vector],
    anns_field="description_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20
)

# 图像向量检索
image_req = AnnSearchRequest(
    data=[image_query_vector],
    anns_field="image_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20
)

# 稀疏向量检索
sparse_req = AnnSearchRequest(
    data=[query_text],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20
)

# 多向量混合检索（加权排序）
# 权重：标题40%、描述30%、图像20%、关键词10%
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, image_req, sparse_req],
    ranker=WeightedRanker(weights=[0.4, 0.3, 0.2, 0.1]),
    limit=10,
    output_fields=["title", "description", "category", "price"]
)
```

### 重排序策略选择

**RRF（Reciprocal Rank Fusion）：**
- 适用场景：多个向量字段等权重
- 优势：无需调参，民主化融合
- 劣势：无法控制权重

```python
from pymilvus import RRFRanker

# RRF重排序
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, image_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=10
)
```

**Weighted Ranker（加权排序）：**
- 适用场景：已知字段重要性
- 优势：可控制权重
- 劣势：需要调参

```python
from pymilvus import WeightedRanker

# 加权排序
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, image_req, sparse_req],
    ranker=WeightedRanker(weights=[0.4, 0.3, 0.2, 0.1]),
    limit=10
)
```

## 向量检索性能优化

### 1. 索引选择

**HNSW（推荐）：**
- 适用场景：10M-100M向量
- 优势：高召回率、低延迟
- 参数：M=16, efConstruction=200, ef=64

```python
index_params.add_index(
    field_name="dense_vector",
    index_type="HNSW",
    metric_type="COSINE",
    params={"M": 16, "efConstruction": 200}
)

search_params = {"ef": 64}
```

**IVF_FLAT：**
- 适用场景：1M-10M向量
- 优势：平衡性能和内存
- 参数：nlist=1024, nprobe=16

```python
index_params.add_index(
    field_name="dense_vector",
    index_type="IVF_FLAT",
    metric_type="COSINE",
    params={"nlist": 1024}
)

search_params = {"nprobe": 16}
```

**GPU_CAGRA（高性能）：**
- 适用场景：100M-1B向量
- 优势：GPU加速，10x速度提升
- 参数：intermediate_graph_degree=128

```python
index_params.add_index(
    field_name="dense_vector",
    index_type="GPU_CAGRA",
    metric_type="COSINE",
    params={"intermediate_graph_degree": 128}
)
```

### 2. 批量检索

```python
# 批量查询（提升吞吐量）
query_vectors = [vector1, vector2, vector3, ...]

results = client.search(
    collection_name="documents",
    data=query_vectors,  # 多个查询向量
    anns_field="dense_vector",
    limit=10
)
```

### 3. 预过滤 vs 后过滤

**Pre-filter（推荐用于高选择性）：**
```python
# 先过滤，再检索（选择性<20%）
results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="dense_vector",
    filter="category == 'electronics' AND price < 1000",  # 高选择性
    limit=10
)
```

**Post-filter（推荐用于低选择性）：**
```python
# 先检索，再过滤（选择性>50%）
results = client.search(
    collection_name="documents",
    data=[query_vector],
    anns_field="dense_vector",
    limit=100  # 召回更多候选
)

# 应用层过滤
filtered_results = [r for r in results if r.entity.get("price") < 1000][:10]
```

## 实际应用示例

### 示例1：文档问答系统

```python
from pymilvus import MilvusClient, AnnSearchRequest, RRFRanker
from sentence_transformers import SentenceTransformer

# 初始化
client = MilvusClient(uri="http://localhost:19530")
model = SentenceTransformer('all-MiniLM-L6-v2')

# 用户查询
user_query = "如何优化Python代码性能？"

# 生成稠密向量
dense_vector = model.encode(user_query)

# 稠密向量检索（语义匹配）
dense_req = AnnSearchRequest(
    data=[dense_vector],
    anns_field="dense_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20
)

# 稀疏向量检索（关键词匹配）
sparse_req = AnnSearchRequest(
    data=[user_query],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20
)

# 混合检索
results = client.hybrid_search(
    collection_name="knowledge_base",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),
    limit=5,
    output_fields=["content", "title"]
)

# 提取上下文
context = "\n\n".join([hit.entity.get("content") for hits in results for hit in hits])

# LLM生成答案
from openai import OpenAI
llm = OpenAI()

response = llm.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "你是一个Python专家，根据提供的上下文回答问题。"},
        {"role": "user", "content": f"Context:\n{context}\n\nQuestion: {user_query}\n\nAnswer:"}
    ]
)

print(response.choices[0].message.content)
```

### 示例2：电商商品搜索

```python
# 用户搜索："红色连衣裙 夏季"
query = "红色连衣裙 夏季"

# 生成查询向量
query_vector = model.encode(query)

# 多向量检索
title_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="title_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20,
    expr="category == 'clothing' AND season == 'summer'"  # 标量过滤
)

description_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="description_vector",
    param={"metric_type": "COSINE", "params": {"ef": 64}},
    limit=20,
    expr="category == 'clothing' AND season == 'summer'"
)

sparse_req = AnnSearchRequest(
    data=[query],
    anns_field="sparse_vector",
    param={"metric_type": "BM25"},
    limit=20,
    expr="category == 'clothing' AND season == 'summer'"
)

# 混合检索（标题60%、描述30%、关键词10%）
results = client.hybrid_search(
    collection_name="products",
    reqs=[title_req, description_req, sparse_req],
    ranker=WeightedRanker(weights=[0.6, 0.3, 0.1]),
    limit=20,
    output_fields=["title", "price", "image_url", "rating"]
)

# 展示结果
for hits in results:
    for hit in hits:
        print(f"商品: {hit.entity.get('title')}")
        print(f"价格: ¥{hit.entity.get('price')}")
        print(f"评分: {hit.entity.get('rating')}")
        print(f"相关度: {hit.score:.2f}")
        print("-" * 50)
```

## 总结

### 核心要点

1. **稠密向量检索**：语义匹配，适合理解用户意图
2. **稀疏向量检索**：关键词匹配，适合精确术语
3. **多向量检索**：多维度信息融合，提升检索质量
4. **重排序策略**：RRF（等权重）vs Weighted（可控权重）
5. **性能优化**：索引选择、批量检索、过滤策略

### 最佳实践

1. **文本检索**：稠密向量（COSINE）+ 稀疏向量（BM25）
2. **多模态检索**：多个向量字段 + 加权排序
3. **索引选择**：HNSW（推荐）、IVF_FLAT（平衡）、GPU_CAGRA（高性能）
4. **过滤策略**：Pre-filter（高选择性）、Post-filter（低选择性）
