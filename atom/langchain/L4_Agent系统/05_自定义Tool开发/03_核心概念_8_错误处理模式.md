# 核心概念 8：错误处理模式

## 一句话定义

**错误处理模式是 LangChain 工具开发中处理执行失败的机制，通过 ToolException 和 handle_tool_error 配置，让工具错误不会中断 Agent 循环，而是作为观察返回给 Agent 进行自我修正。**

---

## 为什么需要错误处理？

在 Agent 系统中，工具调用可能因为各种原因失败：
- 外部 API 不可用
- 参数验证失败
- 网络超时
- 权限不足
- 数据格式错误

**传统做法的问题**：
```python
@tool
def risky_api_call(query: str) -> str:
    """调用外部 API"""
    response = requests.get(f"https://api.example.com?q={query}")
    # 如果 API 失败，抛出异常会直接中断整个 Agent 循环
    response.raise_for_status()  # ❌ 会中断 Agent
    return response.text
```

**LangChain 的解决方案**：
- 使用 `ToolException` 标记可恢复的错误
- 错误信息作为观察返回给 Agent
- Agent 可以根据错误信息调整策略
- 整个对话流程不会中断

---

## 核心机制详解

### 1. ToolException 特殊异常

**定义**（来自源码）：
```python
class ToolException(Exception):
    """Exception thrown when a tool execution error occurs.

    This exception allows tools to signal errors without stopping the agent.

    The error is handled according to the tool's `handle_tool_error` setting,
    and the result is returned as an observation to the agent.
    """
```

**关键特性**：
- 继承自 Python 标准 `Exception`
- 被 LangChain 特殊处理（不会中断 Agent）
- 错误信息会被格式化为 ToolMessage 返回

**使用示例**：
```python
from langchain.tools import tool, ToolException

@tool
def safe_api_call(query: str) -> str:
    """安全的 API 调用工具"""
    try:
        response = requests.get(
            f"https://api.example.com?q={query}",
            timeout=5
        )
        response.raise_for_status()
        return response.json()["result"]
    except requests.Timeout:
        # 抛出 ToolException 不会中断 Agent
        raise ToolException("API 超时，请稍后重试")
    except requests.HTTPError as e:
        raise ToolException(f"API 返回错误: {e.response.status_code}")
    except KeyError:
        raise ToolException("API 返回数据格式错误")
```

**Agent 看到的错误信息**：
```
Tool: safe_api_call
Input: {"query": "test"}
Observation: Error: API 超时，请稍后重试
```

Agent 可以根据这个观察决定：
- 重试工具调用
- 使用其他工具
- 向用户请求更多信息
- 调整查询参数

### 2. handle_tool_error 配置

**三种错误处理策略**：

#### 策略 1：默认行为（传播异常）
```python
@tool
def default_error_tool(param: str) -> str:
    """默认错误处理"""
    # 普通异常会中断 Agent
    raise ValueError("This will stop the agent")
```

#### 策略 2：返回固定错误消息
```python
@tool(handle_tool_error=True)
def simple_error_tool(param: str) -> str:
    """简单错误处理"""
    # 任何异常都会被捕获，返回通用错误消息
    raise ValueError("Some error")
    # Agent 看到: "Error: Some error"
```

#### 策略 3：自定义错误处理函数
```python
def custom_error_handler(error: Exception) -> str:
    """自定义错误处理逻辑"""
    if isinstance(error, requests.Timeout):
        return "API 超时，建议使用缓存数据或稍后重试"
    elif isinstance(error, requests.HTTPError):
        status_code = error.response.status_code
        if status_code == 429:
            return "API 速率限制，请等待 60 秒后重试"
        elif status_code == 401:
            return "API 认证失败，请检查 API 密钥配置"
        else:
            return f"API 错误 {status_code}，请联系管理员"
    else:
        return f"未知错误: {str(error)}"

@tool(handle_tool_error=custom_error_handler)
def advanced_error_tool(query: str) -> str:
    """高级错误处理"""
    response = requests.get(f"https://api.example.com?q={query}")
    response.raise_for_status()
    return response.text
```

### 3. 错误传播 vs 错误捕获

**对比表**：

| 场景 | 使用方式 | Agent 行为 | 适用场景 |
|------|----------|-----------|----------|
| **致命错误** | 抛出普通异常 | 立即停止 | 配置错误、权限不足 |
| **可恢复错误** | 抛出 ToolException | 继续执行 | API 超时、数据不存在 |
| **简单捕获** | handle_tool_error=True | 返回通用错误 | 快速原型开发 |
| **智能处理** | handle_tool_error=func | 返回定制错误 | 生产环境 |

**决策树**：
```
工具执行失败
    ↓
是否可以恢复？
    ├─ 否 → 抛出普通异常（中断 Agent）
    │       例如：API 密钥未配置
    │
    └─ 是 → 抛出 ToolException（不中断）
            例如：API 临时不可用
                ↓
            需要定制错误消息？
                ├─ 否 → handle_tool_error=True
                └─ 是 → handle_tool_error=custom_func
```

## 完整代码示例

### 示例 1：数据库查询工具（生产级错误处理）

```python
from langchain.tools import tool, ToolException
from typing import List, Dict, Any
import sqlalchemy
from sqlalchemy.exc import OperationalError, ProgrammingError

def db_error_handler(error: Exception) -> str:
    """数据库错误处理器"""
    if isinstance(error, OperationalError):
        return "数据库连接失败，请检查网络或稍后重试"
    elif isinstance(error, ProgrammingError):
        return "SQL 语法错误，请检查查询参数"
    elif isinstance(error, ToolException):
        # ToolException 直接返回原始消息
        return str(error)
    else:
        return f"数据库查询失败: {type(error).__name__}"

@tool(handle_tool_error=db_error_handler)
def query_database(
    table: str,
    columns: List[str],
    limit: int = 10
) -> str:
    """查询数据库表

    Args:
        table: 表名（仅支持 users, products, orders）
        columns: 要查询的列名列表
        limit: 返回结果数量限制（1-100）
    """
    # 1. 参数验证（业务逻辑错误 → ToolException）
    allowed_tables = ['users', 'products', 'orders']
    if table not in allowed_tables:
        raise ToolException(
            f"表名 '{table}' 不在白名单中。允许的表: {', '.join(allowed_tables)}"
        )

    if not 1 <= limit <= 100:
        raise ToolException("limit 必须在 1-100 之间")

    # 2. 数据库操作（技术错误 → 普通异常，由 handler 处理）
    engine = sqlalchemy.create_engine("postgresql://...")

    # 使用参数化查询防止 SQL 注入
    cols = ", ".join(columns)
    query = f"SELECT {cols} FROM {table} LIMIT :limit"

    with engine.connect() as conn:
        result = conn.execute(
            sqlalchemy.text(query),
            {"limit": limit}
        )
        rows = result.fetchall()

    # 3. 结果格式化
    if not rows:
        raise ToolException(f"表 '{table}' 中没有数据")

    return f"查询到 {len(rows)} 条记录:\n" + "\n".join(str(row) for row in rows)
```

**测试场景**：
```python
# 场景 1：参数验证失败
query_database.invoke({"table": "admin", "columns": ["*"]})
# Agent 看到: "表名 'admin' 不在白名单中。允许的表: users, products, orders"

# 场景 2：数据库连接失败
# Agent 看到: "数据库连接失败，请检查网络或稍后重试"

# 场景 3：成功查询
query_database.invoke({"table": "users", "columns": ["id", "name"], "limit": 5})
# Agent 看到: "查询到 5 条记录:\n(1, 'Alice')\n(2, 'Bob')..."
```

### 示例 2：外部 API 工具（重试逻辑）

```python
import time
import requests
from langchain.tools import tool, ToolException

@tool
def weather_api_with_retry(location: str, max_retries: int = 3) -> str:
    """获取天气信息（带重试）

    Args:
        location: 城市名称
        max_retries: 最大重试次数（默认 3 次）
    """
    api_key = os.getenv("WEATHER_API_KEY")
    if not api_key:
        # 配置错误 → 普通异常（中断 Agent）
        raise ValueError("WEATHER_API_KEY 环境变量未设置")

    url = f"https://api.weather.com/v1/current"
    params = {"q": location, "key": api_key}

    for attempt in range(max_retries):
        try:
            response = requests.get(url, params=params, timeout=5)
            response.raise_for_status()
            data = response.json()

            return f"天气: {data['condition']}, 温度: {data['temp']}°C"

        except requests.Timeout:
            if attempt < max_retries - 1:
                # 还有重试机会，等待后重试
                time.sleep(2 ** attempt)  # 指数退避
                continue
            else:
                # 重试耗尽 → ToolException
                raise ToolException(
                    f"API 超时（已重试 {max_retries} 次），请稍后再试"
                )

        except requests.HTTPError as e:
            status_code = e.response.status_code
            if status_code == 404:
                # 城市不存在 → ToolException
                raise ToolException(f"未找到城市 '{location}'，请检查拼写")
            elif status_code == 429:
                # 速率限制 → ToolException
                raise ToolException("API 速率限制，请等待 60 秒")
            else:
                # 其他 HTTP 错误 → ToolException
                raise ToolException(f"API 错误 {status_code}")

        except KeyError as e:
            # 数据格式错误 → ToolException
            raise ToolException(f"API 返回数据缺少字段: {e}")
```

### 示例 3：文件操作工具（安全检查）

```python
import os
from pathlib import Path
from langchain.tools import tool, ToolException

@tool
def read_file_safe(file_path: str) -> str:
    """安全读取文件内容

    Args:
        file_path: 文件路径（相对于工作目录）
    """
    # 1. 路径安全检查
    base_dir = Path("/app/data")  # 限制访问范围
    full_path = (base_dir / file_path).resolve()

    # 防止路径遍历攻击
    if not str(full_path).startswith(str(base_dir)):
        raise ToolException(
            f"拒绝访问: 文件路径超出允许范围"
        )

    # 2. 文件存在性检查
    if not full_path.exists():
        raise ToolException(f"文件不存在: {file_path}")

    if not full_path.is_file():
        raise ToolException(f"路径不是文件: {file_path}")

    # 3. 文件大小检查
    max_size = 10 * 1024 * 1024  # 10MB
    if full_path.stat().st_size > max_size:
        raise ToolException(
            f"文件过大（>{max_size/1024/1024}MB），请使用分块读取工具"
        )

    # 4. 读取文件
    try:
        with open(full_path, 'r', encoding='utf-8') as f:
            content = f.read()
        return f"文件内容（{len(content)} 字符）:\n{content}"
    except UnicodeDecodeError:
        raise ToolException("文件编码错误，仅支持 UTF-8 文本文件")
    except PermissionError:
        raise ToolException("权限不足，无法读取文件")
```

## 在 Agent 循环中的工作流程

### 正常流程（无错误）
```
User: "查询 users 表的前 5 条记录"
    ↓
Agent: 调用 query_database(table="users", columns=["*"], limit=5)
    ↓
Tool: 返回 "查询到 5 条记录: ..."
    ↓
Agent: 将结果返回给用户
```

### 错误恢复流程（ToolException）
```
User: "查询 admin 表的数据"
    ↓
Agent: 调用 query_database(table="admin", columns=["*"])
    ↓
Tool: 抛出 ToolException("表名 'admin' 不在白名单中...")
    ↓
Agent: 收到错误观察
    ↓
Agent: 重新思考 → "用户可能想查询管理员信息，应该查询 users 表并过滤 role='admin'"
    ↓
Agent: 调用 query_database(table="users", columns=["*"])
    ↓
Tool: 返回成功结果
    ↓
Agent: 将结果返回给用户
```

### 致命错误流程（普通异常）
```
User: "查询数据库"
    ↓
Agent: 调用 query_database(...)
    ↓
Tool: 抛出 ValueError("WEATHER_API_KEY 环境变量未设置")
    ↓
Agent: 立即停止，向用户报告配置错误
```

## 最佳实践

### 1. 错误分类原则

**可恢复错误（使用 ToolException）**：
- 用户输入错误（参数不合法）
- 外部服务临时不可用
- 数据不存在或格式错误
- 速率限制、超时

**不可恢复错误（使用普通异常）**：
- 配置错误（API 密钥缺失）
- 权限不足
- 代码逻辑错误
- 系统资源耗尽

### 2. 错误消息编写技巧

**❌ 不好的错误消息**：
```python
raise ToolException("Error")  # 太模糊
raise ToolException("API call failed")  # 没有上下文
raise ToolException(str(e))  # 技术细节过多
```

**✅ 好的错误消息**：
```python
raise ToolException(
    f"未找到城市 '{location}'，请检查拼写或尝试使用英文名称"
)
# 包含：问题描述 + 上下文 + 解决建议
```

### 3. 日志记录

```python
import logging

logger = logging.getLogger(__name__)

@tool
def logged_tool(param: str) -> str:
    """带日志的工具"""
    try:
        result = risky_operation(param)
        logger.info(f"Tool succeeded: {param}")
        return result
    except Exception as e:
        # 记录详细错误（包括堆栈）
        logger.error(f"Tool failed: {param}", exc_info=True)
        # 返回用户友好的错误消息
        raise ToolException(f"操作失败: {type(e).__name__}")
```

### 4. 错误处理测试

```python
import pytest

    """测试工具错误处理"""
```

```python
@tool
def retrieve_documents(query: str, top_k: int = 5) -> str:
```

## 双重类比

### 类比 1：错误处理 vs 前端错误边界

**前端类比**：React Error Boundary
```javascript
// React Error Boundary
class ErrorBoundary extends React.Component {
  componentDidCatch(error, errorInfo) {
    // 捕获子组件错误，显示降级 UI
    this.setState({ hasError: true });
  }

  render() {
    if (this.state.hasError) {
      return <h1>Something went wrong.</h1>;
    }
    return this.props.children;
  }
}
```

**LangChain 对应**：
```python
@tool(handle_tool_error=True)  # 类似 Error Boundary
def component_tool(param: str) -> str:
    # 错误不会传播到父级（Agent）
    raise ValueError("Component error")
```

**相似性**：
- 都是捕获子级错误
- 都提供降级体验
- 都不会中断整个应用

### 类比 2：ToolException vs HTTP 状态码

**日常生活类比**：餐厅点餐
- **404 Not Found** = ToolException("菜品已售罄")
  - 服务员告诉你，你可以点其他菜
  - 不会赶你出餐厅
- **500 Internal Server Error** = ValueError("厨房失火")
  - 餐厅关闭，所有顾客离开
  - 无法继续服务

**LangChain 对应**：
```python
# 404 类型错误（可恢复）
raise ToolException("数据不存在，请尝试其他查询")

# 500 类型错误（不可恢复）
raise ValueError("数据库连接池耗尽")
```

### 类比 3：错误处理策略 vs 异常处理模式

**前端类比**：Promise 错误处理
```javascript
// 策略 1：传播错误
fetch(url).then(data => process(data));  // 错误会传播

// 策略 2：捕获并返回默认值
fetch(url).catch(() => "default value");

// 策略 3：自定义错误处理
fetch(url).catch(error => {
  if (error.status === 404) return "Not found";
  if (error.status === 500) return "Server error";
  throw error;
});
```

**LangChain 对应**：
```python
# 策略 1：传播错误
@tool
def tool1(param: str) -> str:
    raise ValueError("Error")  # 中断 Agent

# 策略 2：捕获并返回通用消息
@tool(handle_tool_error=True)
def tool2(param: str) -> str:
    raise ValueError("Error")  # 返回 "Error: ..."

# 策略 3：自定义错误处理
@tool(handle_tool_error=custom_handler)
def tool3(param: str) -> str:
    raise ValueError("Error")  # 调用 custom_handler
```

## 反直觉点

### 误区 1：所有错误都应该抛出 ToolException ❌

**为什么错？**
- 配置错误、权限错误等不可恢复错误应该立即停止
- 让 Agent 继续尝试只会浪费 token 和时间

**正确理解**：
```python
@tool
def smart_error_tool(param: str) -> str:
    # 配置错误 → 普通异常（立即停止）
    if not os.getenv("API_KEY"):
        raise ValueError("API_KEY 未配置，请联系管理员")

    # 业务错误 → ToolException（让 Agent 处理）
    if param not in valid_params:
        raise ToolException(f"参数 '{param}' 无效，有效值: {valid_params}")
```

### 误区 2：错误消息越详细越好 ❌

**为什么错？**
- 过多技术细节会混淆 Agent
- 堆栈跟踪对 LLM 没有意义

**正确理解**：
```python
# ❌ 不好：技术细节过多
raise ToolException(f"Traceback: {traceback.format_exc()}")

# ✅ 好：用户友好的消息
raise ToolException("API 调用失败，请检查网络连接")

# 技术细节记录到日志
logger.error("API call failed", exc_info=True)
```

### 误区 3：handle_tool_error=True 是万能的 ❌

**为什么错？**
- 通用错误消息缺乏上下文
- Agent 无法根据错误类型调整策略

**正确理解**：
```python
# ❌ 不好：所有错误都返回通用消息
@tool(handle_tool_error=True)
def generic_tool(param: str) -> str:
    # 超时、404、500 都返回 "Error: ..."
    ...

# ✅ 好：根据错误类型返回定制消息
@tool(handle_tool_error=custom_handler)
def smart_tool(param: str) -> str:
    # 超时 → "请稍后重试"
    # 404 → "数据不存在，请检查参数"
    # 500 → "服务器错误，请联系管理员"
    ...
```
