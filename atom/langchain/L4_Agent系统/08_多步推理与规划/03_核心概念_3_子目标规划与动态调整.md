# 核心概念 3：子目标规划与动态调整

> LangChain Agent 系统 · 第 8 个知识点 · 核心概念 3/5

---

## 一句话定义

**子目标规划是为每个子任务设定明确目标和优先级的过程，动态调整（Re-planning）则是在执行过程中根据反馈修改计划的能力。**

---

## 为什么需要动态调整？

```
静态计划的问题：
  Plan: [搜索A, 搜索B, 对比, 写报告]
  执行搜索A → 发现A已停止维护 → 计划失效！
  继续执行搜索B → 结果与预期不同 → 计划更加偏离！
  对比 → 数据不匹配 → 输出质量很差 ❌

动态调整的优势：
  Plan: [搜索A, 搜索B, 对比, 写报告]
  执行搜索A → 发现A已停止维护
  Re-plan: [搜索A的替代品C, 搜索B, 对比B和C, 写报告]
  继续执行 → 输出质量高 ✅
```

**核心洞察：真实世界的任务充满不确定性，没有动态调整的规划系统注定脆弱。**

## 子目标的设定原则

### SMART 原则（适配 Agent 场景）

```
S - Specific  （具体）：不是"了解框架"，而是"搜索 FastAPI 的异步特性"
M - Measurable （可衡量）：有明确的完成标志，如"获取到 3 个性能指标"
A - Actionable （可执行）：对应具体工具调用，如"search('FastAPI benchmark')"
R - Relevant   （相关）：与最终目标直接相关
T - Time-bound （有限制）：设定最大重试次数或超时
```

### 用 Pydantic 定义子目标

```python
from pydantic import BaseModel, Field
from typing import List, Optional
from enum import Enum

class GoalStatus(str, Enum):
    PENDING = "pending"
    IN_PROGRESS = "in_progress"
    COMPLETED = "completed"
    FAILED = "failed"
    SKIPPED = "skipped"

class SubGoal(BaseModel):
    """子目标定义"""
    id: int = Field(description="子目标编号")
    title: str = Field(description="子目标标题（动词开头）")
    success_criteria: str = Field(description="完成标志")
    tool: str = Field(description="需要使用的工具")
    dependencies: List[int] = Field(
        default_factory=list,
        description="依赖的前置子目标"
    )
    priority: int = Field(
        default=1,
        description="优先级（1最高，5最低）"
    )
    max_retries: int = Field(default=2, description="最大重试次数")
    status: GoalStatus = Field(default=GoalStatus.PENDING)
    result: Optional[str] = Field(default=None, description="执行结果")
```

## 优先级排序策略

### 策略1：依赖优先（最常用）

```python
def topological_sort(goals: List[SubGoal]) -> List[SubGoal]:
    """根据依赖关系进行拓扑排序"""
    # 构建依赖图
    in_degree = {g.id: len(g.dependencies) for g in goals}
    graph = {g.id: [] for g in goals}
    for g in goals:
        for dep in g.dependencies:
            graph[dep].append(g.id)

    # BFS 拓扑排序
    queue = [gid for gid, deg in in_degree.items() if deg == 0]
    sorted_ids = []

    while queue:
        # 同一层级按 priority 排序
        queue.sort(key=lambda gid: next(
            g.priority for g in goals if g.id == gid
        ))
        current = queue.pop(0)
        sorted_ids.append(current)
        for neighbor in graph[current]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    goal_map = {g.id: g for g in goals}
    return [goal_map[gid] for gid in sorted_ids]
```

## 动态调整（Re-planning）机制

### Re-planning 触发条件

```python
class ReplanTrigger(str, Enum):
    """何时触发重新规划"""
    STEP_FAILED = "step_failed"          # 某个步骤执行失败
    UNEXPECTED_RESULT = "unexpected"      # 结果与预期不符
    NEW_INFO = "new_info"                # 发现新的关键信息
    GOAL_CHANGED = "goal_changed"        # 用户修改了目标
    TIMEOUT = "timeout"                  # 执行超时
    ALL_DONE = "all_done"               # 所有步骤完成，检查是否真的完成
```

### Re-planning 策略

```python
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field
from typing import List, Literal

class ReplanDecision(BaseModel):
    """重规划决策"""
    action: Literal["continue", "replan", "abort", "complete"] = Field(
        description="决策：继续执行/重新规划/终止/完成"
    )
    reason: str = Field(description="决策理由")
    new_goals: List[str] = Field(
        default_factory=list,
        description="如果重新规划，新的子目标列表"
    )
    final_answer: str = Field(
        default="",
        description="如果完成，最终答案"
    )

replanner_llm = ChatOpenAI(model="gpt-4o", temperature=0)
replanner = replanner_llm.with_structured_output(ReplanDecision)

def evaluate_and_replan(state: dict) -> ReplanDecision:
    """评估执行状态，决定是否重规划"""
    return replanner.invoke(f"""
    你是一个任务规划评估器。根据当前执行状态，决定下一步行动。

    原始目标：{state['objective']}
    已完成步骤：{state['completed_steps']}
    当前步骤结果：{state['current_result']}
    剩余计划：{state['remaining_plan']}

    决策规则：
    - 如果当前步骤成功且有剩余计划 → continue
    - 如果当前步骤失败且可以绕过 → replan（生成替代方案）
    - 如果当前步骤失败且无法继续 → abort
    - 如果所有步骤完成且目标达成 → complete
    """)
```

## 三种 Re-planning 模式

### 模式1：完全重规划

**整个计划推翻重来，生成全新计划。**

```
原计划: [A, B, C, D]
A 执行完毕 → 发现问题
完全重规划: [A结果, X, Y, Z]  ← 全新的后续步骤
```

**适用**：发现原方向完全错误

### 模式2：增量调整

**只修改失败步骤及其后续影响步骤。**

```
原计划: [A, B, C, D]
B 执行失败
增量调整: [A结果, B', C, D]  ← 只替换 B
```

**适用**：个别步骤失败但整体方向正确

### 模式3：插入补充

**保持原计划不变，插入额外步骤。**

```
原计划: [A, B, C, D]
A 执行后发现需要额外信息
插入补充: [A结果, A+, B, C, D]  ← 插入 A+
```

**适用**：发现遗漏但不影响整体规划

### 代码实现

```python
class ReplanMode(str, Enum):
    FULL = "full_replan"        # 完全重规划
    INCREMENTAL = "incremental" # 增量调整
    INSERT = "insert"           # 插入补充

class ReplanResult(BaseModel):
    mode: ReplanMode
    new_plan: List[str]
    explanation: str

def smart_replan(state: dict) -> ReplanResult:
    """根据失败严重程度选择重规划模式"""
    replanner = llm.with_structured_output(ReplanResult)
    return replanner.invoke(f"""
    根据以下情况选择最合适的重规划模式：

    - full_replan：整体方向错误，需要完全重新规划
    - incremental：个别步骤失败，替换失败步骤即可
    - insert：发现遗漏信息，需要插入额外步骤

    目标：{state['objective']}
    已完成：{state['completed']}
    失败步骤：{state['failed_step']}
    失败原因：{state['failure_reason']}
    剩余计划：{state['remaining']}
    """)
```

## 在 LangGraph 中实现动态调整

```python
from typing import TypedDict, List, Annotated
from langgraph.graph import StateGraph, END
import operator

class PlanState(TypedDict):
    objective: str
    plan: List[str]
    step_index: int
    completed: Annotated[List[tuple], operator.add]
    response: str

def execute_node(state: PlanState):
    """执行当前步骤"""
    step = state["plan"][state["step_index"]]
    result = executor_llm.invoke(f"执行：{step}")
    return {
        "completed": [(step, result.content)],
        "step_index": state["step_index"] + 1,
    }

def replan_node(state: PlanState):
    """检查并决定是否重规划"""
    decision = evaluate_and_replan({
        "objective": state["objective"],
        "completed_steps": state["completed"],
        "current_result": state["completed"][-1] if state["completed"] else None,
        "remaining_plan": state["plan"][state["step_index"]:],
    })

    if decision.action == "complete":
        return {"response": decision.final_answer}
    elif decision.action == "replan":
        return {"plan": decision.new_goals, "step_index": 0}
    return {}  # continue

def router(state: PlanState):
    if state.get("response"):
        return "end"
    if state["step_index"] < len(state["plan"]):
        return "execute"
    return "replan"

# 构建图
graph = StateGraph(PlanState)
graph.add_node("execute", execute_node)
graph.add_node("replan", replan_node)
graph.set_entry_point("execute")
graph.add_conditional_edges("execute", router,
    {"execute": "execute", "replan": "replan", "end": END})
graph.add_conditional_edges("replan", router,
    {"execute": "execute", "replan": "replan", "end": END})
app = graph.compile()
```

## 常见陷阱与解决方案

| 陷阱 | 说明 | 解决方案 |
|------|------|---------|
| **过度重规划** | 每一步都重新规划，效率极低 | 设定重规划阈值，只在异常时触发 |
| **重规划循环** | 重规划后再次失败，无限循环 | 限制最大重规划次数（2-3次） |
| **忽略已完成步骤** | 重规划时丢弃已有成果 | 将已完成结果作为上下文传入 |
| **目标漂移** | 多次调整后偏离原始目标 | 每次重规划都检查与原始目标的一致性 |
| **子目标粒度不均** | 有的太粗有的太细 | 确保每个子目标 1-2 次工具调用 |

---

## 关键要点

1. **子目标要 SMART**：具体、可衡量、可执行、相关、有限制
2. **优先级靠依赖关系**：拓扑排序确保执行顺序正确
3. **三种重规划模式**：完全重规划、增量调整、插入补充
4. **触发条件明确**：步骤失败、意外结果、新信息、目标变更
5. **防止过度重规划**：限制次数，保持与原始目标的一致性
