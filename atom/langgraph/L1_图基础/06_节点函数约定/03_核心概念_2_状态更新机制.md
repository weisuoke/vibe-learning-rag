# 核心概念 2：状态更新机制

## 概述

状态更新机制是 LangGraph 节点函数的核心约定之一。理解状态如何更新、如何合并、如何通过 Command 控制流程，是掌握 LangGraph 的关键。

本文档深入讲解：
1. 部分更新 vs 完整更新
2. Reducer 函数的作用
3. Command 对象的使用
4. 状态更新的底层实现

---

## 1. 部分更新 vs 完整更新

### 1.1 部分更新机制

LangGraph 的节点函数采用**部分更新**机制，这是一个重要的设计决策。

**核心原则**：
- 节点函数只需返回需要更新的状态字段
- 不需要返回完整的状态对象
- 返回的字段会与现有状态合并

[来源: reference/context7_langgraph_01.md]

**示例**：

```python
from typing_extensions import TypedDict

class State(TypedDict):
    question: str
    documents: list[str]
    answer: str
    error: str | None

def retrieve(state: State) -> dict:
    """
    检索文档节点

    只返回需要更新的字段：documents
    不需要返回完整的 State
    """
    question = state["question"]

    # 执行检索
    documents = retriever.invoke(question)

    # 只返回需要更新的字段
    return {"documents": documents}  # 不需要返回 question, answer, error

def generate(state: State) -> dict:
    """
    生成答案节点

    只返回需要更新的字段：answer
    """
    question = state["question"]
    documents = state["documents"]

    # 生成答案
    answer = llm.invoke(f"Question: {question}\nContext: {documents}")

    # 只返回需要更新的字段
    return {"answer": answer}
```

[来源: reference/context7_langgraph_01.md]

### 1.2 完整更新的场景

虽然推荐使用部分更新，但在某些场景下需要返回完整状态：

**场景 1：状态重置**

```python
def reset_state(state: State) -> State:
    """完全重置状态"""
    return {
        "question": "",
        "documents": [],
        "answer": "",
        "error": None
    }
```

**场景 2：条件路由函数**

```python
def decide_to_generate(state: State) -> str:
    """
    条件路由函数不更新状态
    只返回下一个节点名称
    """
    error = state["error"]
    iterations = state["iterations"]

    if error == "no" or iterations == max_iterations:
        return "end"
    else:
        return "generate"
```

[来源: reference/context7_langgraph_01.md]

### 1.3 底层实现：_get_updates

LangGraph 内部通过 `_get_updates` 函数处理状态更新：

```python
def _get_updates(
    input: None | dict | Any,
) -> Sequence[tuple[str, Any]] | None:
    """
    从节点返回值中提取状态更新

    支持的返回值类型：
    1. None - 不更新状态
    2. dict - 部分状态更新
    3. Command - 控制流程
    4. list[Command | dict] - 多个更新
    5. Annotated 自定义类型
    """
    if input is None:
        return None
    elif isinstance(input, dict):
        # 提取需要更新的字段
        return [(k, v) for k, v in input.items() if k in output_keys]
    elif isinstance(input, Command):
        # 处理 Command 对象
        if input.graph == Command.PARENT:
            return None
        return [
            (k, v) for k, v in input._update_as_tuples() if k in output_keys
        ]
    elif isinstance(input, (list, tuple)) and input and any(isinstance(i, Command) for i in input):
        # 处理多个更新
        updates: list[tuple[str, Any]] = []
        for i in input:
            if isinstance(i, Command):
                if i.graph == Command.PARENT:
                    continue
                updates.extend(
                    (k, v) for k, v in i._update_as_tuples() if k in output_keys
                )
            else:
                updates.extend(_get_updates(i) or ())
        return updates
    elif (t := type(input)) and get_cached_annotated_keys(t):
        # 处理 Annotated 类型
        return get_update_as_tuples(input, output_keys)
    else:
        raise InvalidUpdateError(f"Expected dict, got {input}")
```

[来源: reference/source_节点函数约定_01.md]

**关键点**：
1. 自动过滤不在 `output_keys` 中的字段
2. 支持多种返回值类型
3. 处理 Command 对象的特殊逻辑
4. 类型安全检查

## 2. Reducer 函数的作用

### 2.1 什么是 Reducer

Reducer 函数定义了如何合并状态更新。类似于 Redux 中的 reducer，它控制状态的更新逻辑。

**类比**：
- **前端类比**：Redux reducer - 定义状态如何响应 action
- **日常类比**：银行账户规则 - 定义存款如何累加、取款如何扣减

[来源: CLAUDE_LANGGRAPH.md]

### 2.2 默认 Reducer 行为

LangGraph 的默认 Reducer 行为：

```python
# 默认行为：直接覆盖
state["key"] = new_value

# 对于 dict 类型的字段
state = {
    "question": "What is RAG?",  # 旧值
    "answer": ""
}

# 节点返回
return {"question": "What is LangGraph?"}  # 新值

# 结果：直接覆盖
state = {
    "question": "What is LangGraph?",  # 覆盖
    "answer": ""
}
```

### 2.3 自定义 Reducer

通过 `Annotated` 定义自定义 Reducer：

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    # 默认覆盖行为
    question: str

    # 自定义累加行为
    messages: Annotated[list[str], add]

    # 自定义计数行为
    iterations: Annotated[int, lambda x, y: x + y]

# 使用示例
def node1(state: State) -> dict:
    return {"messages": ["Message 1"]}

def node2(state: State) -> dict:
    return {"messages": ["Message 2"]}

# 执行结果
# 初始状态: {"messages": []}
# node1 后: {"messages": ["Message 1"]}
# node2 后: {"messages": ["Message 1", "Message 2"]}  # 累加而非覆盖
```

### 2.4 常见 Reducer 模式

**模式 1：列表累加**

```python
from operator import add

class State(TypedDict):
    documents: Annotated[list[str], add]

# 多个节点可以累加文档
def retrieve_web(state: State) -> dict:
    return {"documents": ["web doc 1", "web doc 2"]}

def retrieve_db(state: State) -> dict:
    return {"documents": ["db doc 1", "db doc 2"]}

# 结果：["web doc 1", "web doc 2", "db doc 1", "db doc 2"]
```

**模式 2：计数器**

```python
class State(TypedDict):
    attempts: Annotated[int, lambda x, y: x + y]

def retry_node(state: State) -> dict:
    return {"attempts": 1}  # 每次重试 +1
```

**模式 3：错误收集**

```python
class State(TypedDict):
    errors: Annotated[list[str], add]

def node_with_error_handling(state: State) -> dict:
    try:
        result = risky_operation()
        return {"result": result}
    except Exception as e:
        return {"errors": [str(e)]}
```

## 3. Command 对象的使用

### 3.1 Command 对象概述

`Command` 对象用于控制图的执行流程，同时可以携带状态更新。

**核心功能**：
1. 跳转到指定节点（`goto`）
2. 携带状态更新（`update`）
3. 与父图通信（`ParentCommand`）

[来源: reference/source_节点函数约定_01.md]

### 3.2 基础 Command 使用

```python
from langgraph.types import Command
from typing import Literal

def decision_node(state: State) -> Command[Literal["process", "end"]]:
    """
    根据条件决定下一步
    """
    if state["x"] > 10:
        # 跳转到 "end" 节点，不更新状态
        return Command(goto="end")
    else:
        # 跳转到 "process" 节点，同时更新状态
        return Command(
            goto="process",
            update={"x": state["x"] + 1}
        )
```

[来源: reference/source_节点函数约定_01.md]

### 3.3 Command 的返回值处理

```python
# Command 对象的内部结构
class Command:
    goto: str | list[str]  # 目标节点
    update: dict | None    # 状态更新
    graph: str | None      # 目标图（用于子图）

    PARENT = "__parent__"  # 特殊标记：父图

# 底层处理逻辑
if isinstance(input, Command):
    if input.graph == Command.PARENT:
        # 不更新当前图的状态
        return None
    return [
        (k, v) for k, v in input._update_as_tuples() if k in output_keys
    ]
```

[来源: reference/source_节点函数约定_01.md]

### 3.4 多个 Command 返回

```python
def multi_command_node(state: State) -> list[Command | dict]:
    """
    返回多个更新和命令
    """
    return [
        {"status": "processing"},  # 状态更新
        Command(goto="next", update={"step": 1}),  # 命令 1
        Command(goto="log", update={"timestamp": time.time()})  # 命令 2
```

### 3.5 ParentCommand 使用

```python
from langgraph.types import Command

def subgraph_node(state: State) -> Command:
    """
    子图节点向父图发送命令
    """
    if state["should_exit"]:
        # 向父图发送命令
        return Command(
            graph=Command.PARENT,
            goto="parent_node",
            update={"result": state["result"]}
        )
    else:
        # 在子图内跳转
        return Command(goto="next_subgraph_node")
```

### 3.6 Command 错误处理

```python
def safe_command_node(state: State) -> Command[Literal["valid", "invalid"]]:
    """
    使用类型注解确保 Command 指向有效节点
    """
    if validate(state):
        return Command(goto="valid")
    else:
        return Command(goto="invalid")
```

## 4. 实战代码示例

### 4.1 完整示例：RAG 系统状态更新

```python
from typing_extensions import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command
from typing import Literal

# 定义状态
class RAGState(TypedDict):
    question: str
    documents: Annotated[list[str], add]  # 累加文档
    answer: str
    error: str | None
    iterations: Annotated[int, lambda x, y: x + y]  # 累加迭代次数

# 节点 1：检索文档
def retrieve(state: RAGState) -> dict:
    """
    部分更新：只返回 documents
    """
    print(f"---RETRIEVE: {state['question']}---")
    question = state["question"]

    # 模拟检索
    documents = [f"Doc about {question}"]

    return {"documents": documents}

# 节点 2：生成答案
def generate(state: RAGState) -> dict:
    """
    部分更新：返回 answer 和 iterations
    """
    print("---GENERATE---")
    question = state["question"]
    documents = state["documents"]

    # 模拟生成
    answer = f"Answer based on {len(documents)} documents"

    return {
        "answer": answer,
        "iterations": 1  # 累加迭代次数
    }

# 节点 3：检查答案
def check_answer(state: RAGState) -> Command[Literal["generate", "end"]]:
    """
    使用 Command 控制流程
    """
    print("---CHECK ANSWER---")
    iterations = state["iterations"]
    max_iterations = 3

    # 模拟检查
    is_valid = len(state["answer"]) > 10

    if is_valid or iterations >= max_iterations:
        # 跳转到结束，更新 error 状态
        return Command(
            goto="end",
            update={"error": "no" if is_valid else "max_iterations"}
        )
    else:
        # 重新生成，更新 error 状态
        return Command(
            goto="generate",
            update={"error": "invalid_answer"}
        )

# 构建图
builder = StateGraph(RAGState)
builder.add_node("retrieve", retrieve)
builder.add_node("generate", generate)
builder.add_node("check", check_answer)

builder.add_edge(START, "retrieve")
builder.add_edge("retrieve", "generate")
builder.add_edge("generate", "check")
# check 节点通过 Command 控制流程，不需要显式添加边

graph = builder.compile()

# 执行
result = graph.invoke({
    "question": "What is LangGraph?",
    "documents": [],
    "answer": "",
    "error": None,
    "iterations": 0
})

print(f"Final result: {result}")
# 输出：
# {
#     "question": "What is LangGraph?",
#     "documents": ["Doc about What is LangGraph?"],
#     "answer": "Answer based on 1 documents",
#     "error": "no",
#     "iterations": 1
# }
```

### 4.2 错误处理与状态更新

```python
class ErrorState(TypedDict):
    result: str | None
    errors: Annotated[list[str], add]  # 累加错误
    attempts: Annotated[int, lambda x, y: x + y]

def risky_operation(state: ErrorState) -> dict:
    """
    错误处理节点
    """
    try:
        # 模拟可能失败的操作
        result = perform_operation()
        return {
            "result": result,
            "attempts": 1
        }
    except Exception as e:
        # 累加错误信息
        return {
            "errors": [str(e)],
            "attempts": 1
        }

def check_errors(state: ErrorState) -> Command[Literal["retry", "fail", "success"]]:
    """
    根据错误状态决定下一步
    """
    if state["result"] is not None:
        return Command(goto="success")
    elif state["attempts"] >= 3:
        return Command(goto="fail")
    else:
        return Command(goto="retry")
```

## 5. 最佳实践

### 5.1 状态更新原则

1. **优先使用部分更新**
   ```python
   # ✅ 推荐
   return {"answer": answer}

   # ❌ 不推荐
   return {
       "question": state["question"],
       "documents": state["documents"],
       "answer": answer,
       "error": state["error"]
   }
   ```

2. **使用 Reducer 处理累加逻辑**
   ```python
   # ✅ 推荐：使用 Reducer
   messages: Annotated[list[str], add]

   # ❌ 不推荐：手动合并
   return {"messages": state["messages"] + new_messages}
   ```

3. **使用 Command 控制复杂流程**
   ```python
   # ✅ 推荐：Command 同时更新状态和控制流程
   return Command(goto="next", update={"status": "done"})

   # ❌ 不推荐：分开处理
   # 无法在条件边中更新状态
   ```

### 5.2 类型安全

```python
from typing import Literal

# ✅ 使用 Literal 限制 Command 目标
def node(state: State) -> Command[Literal["valid_node1", "valid_node2"]]:
    return Command(goto="valid_node1")

# ❌ 不使用类型注解，容易出错
def node(state: State) -> Command:
    return Command(goto="typo_node")  # 运行时才发现错误
```

### 5.3 错误处理

```python
# ✅ 在状态中记录错误
class State(TypedDict):
    errors: Annotated[list[str], add]

def node(state: State) -> dict:
    try:
        result = operation()
        return {"result": result}
    except Exception as e:
        return {"errors": [str(e)]}

# ❌ 直接抛出异常（除非配置了 RetryPolicy）
def node(state: State) -> dict:
    result = operation()  # 可能抛出异常
    return {"result": result}
```

## 6. 常见误区

### 误区 1：必须返回完整状态

```python
# ❌ 错误：返回完整状态
def node(state: State) -> State:
    return {
        "question": state["question"],
        "documents": state["documents"],
        "answer": new_answer,  # 只有这个字段变化
        "error": state["error"]
    }

# ✅ 正确：只返回变化的字段
def node(state: State) -> dict:
    return {"answer": new_answer}
```

### 误区 2：Reducer 会自动应用

```python
# ❌ 错误：期望自动累加
class State(TypedDict):
    messages: list[str]  # 没有 Reducer

def node(state: State) -> dict:
    return {"messages": ["new"]}  # 会覆盖而非累加

# ✅ 正确：显式定义 Reducer
class State(TypedDict):
    messages: Annotated[list[str], add]
```

### 误区 3：Command 可以指向任意节点

```python
# ❌ 错误：指向不存在的节点
return Command(goto="nonexistent_node")  # 运行时错误

# ✅ 正确：使用类型注解确保安全
def node(state: State) -> Command[Literal["node1", "node2"]]:
    return Command(goto="node1")
```

## 7. 总结

状态更新机制是 LangGraph 的核心设计：

1. **部分更新**：节点只返回需要更新的字段，简化代码
2. **Reducer 函数**：定义状态合并逻辑，支持累加、覆盖等模式
3. **Command 对象**：控制流程跳转，同时携带状态更新
4. **类型安全**：使用 TypedDict 和 Literal 确保类型安全

**关键要点**：
- 优先使用部分更新
- 使用 Reducer 处理累加逻辑
- 使用 Command 控制复杂流程
- 使用类型注解确保安全

通过掌握这些机制，你可以构建灵活、可维护的状态化工作流。
