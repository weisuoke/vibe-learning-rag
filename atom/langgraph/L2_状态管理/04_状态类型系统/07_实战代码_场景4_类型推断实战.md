# 实战代码 - 场景4：类型推断实战

## 场景概述

LangGraph 提供了强大的类型推断机制，可以自动从节点函数签名推断输入类型。本场景展示如何利用类型推断简化代码，以及如何在推断失败时正确处理。

**核心价值**：
- 减少样板代码，让类型系统自动工作
- 理解推断优先级和失败场景
- 掌握类型提示最佳实践

**适用场景**：
- 快速原型开发
- 类型安全的节点定义
- 复杂的多类型参数图

---

## 核心概念

### 1. 类型推断机制

LangGraph 使用 Python 的 `get_type_hints()` 从函数签名提取类型信息：

```python
# 推断优先级（从高到低）
1. 显式指定的 input_schema 参数
2. 从函数第一个参数推断的类型
3. 图的 state_schema（默认）
```

[来源: sourcecode/langgraph/libs/langgraph/langgraph/graph/state.py | LangGraph 源码]

### 2. 推断条件

类型推断成功需要满足：
- 函数有类型注解
- 第一个参数的类型是一个类（class）
- 该类有 `__annotations__` 属性

### 3. 推断失败场景

以下情况推断会失败：
- 函数没有类型注解
- 类型注解不是类（如 `dict`、`Any`）
- 循环引用导致 `NameError`
- 类型提示语法错误

## 实战代码 1：基础类型推断

### 场景：自动推断节点输入类型

```python
"""
基础类型推断示例
展示 LangGraph 如何自动推断节点输入类型
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END


# 定义状态类型
class State(TypedDict):
    """图的主状态"""
    messages: list[str]
    count: int


class ProcessState(TypedDict):
    """处理节点的输入状态（State 的子集）"""
    messages: list[str]


# 节点函数 - 自动推断输入类型
def process_node(state: ProcessState) -> dict:
    """
    LangGraph 会自动推断：
    - input_schema = ProcessState
    - 只接收 messages 字段
    """
    print(f"处理节点接收到: {state}")
    return {
        "messages": state["messages"] + ["processed"],
        "count": 1
    }


def aggregate_node(state: State) -> dict:
    """
    LangGraph 会自动推断：
    - input_schema = State
    - 接收完整状态
    """
    print(f"聚合节点接收到: {state}")
    return {
        "count": state["count"] + len(state["messages"])
    }


# 构建图
def build_graph():
    """构建使用类型推断的图"""
    graph = StateGraph(State)

    # 添加节点 - 不需要显式指定 input_schema
    graph.add_node("process", process_node)
    graph.add_node("aggregate", aggregate_node)

    # 添加边
    graph.add_edge(START, "process")
    graph.add_edge("process", "aggregate")
    graph.add_edge("aggregate", END)

    return graph.compile()


# 测试
if __name__ == "__main__":
    app = build_graph()

    result = app.invoke({
        "messages": ["hello"],
        "count": 0
    })

    print(f"\n最终结果: {result}")
    # 输出:
    # 处理节点接收到: {'messages': ['hello']}
    # 聚合节点接收到: {'messages': ['hello', 'processed'], 'count': 1}
    # 最终结果: {'messages': ['hello', 'processed'], 'count': 3}
```

**关键点**：
- `process_node` 只声明需要 `messages`，LangGraph 自动推断 `input_schema=ProcessState`
- `aggregate_node` 声明需要完整 `State`，自动推断 `input_schema=State`
- 无需手动指定 `input_schema` 参数

## 实战代码 2：显式指定 input_schema

### 场景：覆盖自动推断

```python
"""
显式指定 input_schema
在需要精确控制时覆盖自动推断
"""

from typing_extensions import TypedDict, NotRequired
from langgraph.graph import StateGraph, START, END


class State(TypedDict):
    """完整状态"""
    query: str
    context: list[str]
    answer: str
    metadata: NotRequired[dict]


class QueryInput(TypedDict):
    """查询输入（最小化）"""
    query: str


class ContextInput(TypedDict):
    """上下文输入"""
    query: str
    context: list[str]


# 节点函数 - 使用通用类型注解
def retrieve_node(state: dict) -> dict:
    """
    使用 dict 类型注解，无法自动推断
    需要显式指定 input_schema
    """
    print(f"检索节点接收到: {state}")
    return {
        "context": [f"context for: {state['query']}"]
    }


def generate_node(state: dict) -> dict:
    """
    使用 dict 类型注解，无法自动推断
    需要显式指定 input_schema
    """
    print(f"生成节点接收到: {state}")
    return {
        "answer": f"Answer based on: {state['context']}"
    }


# 构建图 - 显式指定 input_schema
def build_graph_explicit():
    """构建显式指定类型的图"""
    graph = StateGraph(State)

    # 显式指定 input_schema
    graph.add_node(
        "retrieve",
        retrieve_node,
        input_schema=QueryInput  # 只接收 query
    )

    graph.add_node(
        "generate",
        generate_node,
        input_schema=ContextInput  # 接收 query 和 context
    )

    graph.add_edge(START, "retrieve")
    graph.add_edge("retrieve", "generate")
    graph.add_edge("generate", END)

    return graph.compile()


# 测试
if __name__ == "__main__":
    app = build_graph_explicit()

    result = app.invoke({
        "query": "What is LangGraph?",
        "context": [],
        "answer": "",
        "metadata": {"source": "test"}
    })

    print(f"\n最终结果: {result}")
    # 输出:
    # 检索节点接收到: {'query': 'What is LangGraph?'}
    # 生成节点接收到: {'query': 'What is LangGraph?', 'context': ['context for: What is LangGraph?']}
```

**关键点**：
- 使用 `dict` 类型注解时，必须显式指定 `input_schema`
- 显式指定优先级最高，会覆盖自动推断
- 适用于需要精确控制输入的场景

## 实战代码 3：推断失败处理

### 场景：处理类型推断失败

```python
"""
推断失败处理
展示如何处理类型推断失败的情况
"""

from typing import Any
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import warnings


class State(TypedDict):
    """图状态"""
    data: str
    result: str


# 场景 1：无类型注解 - 推断失败
def node_no_annotation(state):
    """没有类型注解，推断失败，使用 state_schema"""
    return {"result": f"processed: {state['data']}"}


# 场景 2：Any 类型 - 推断失败
def node_any_type(state: Any) -> dict:
    """Any 类型，推断失败，使用 state_schema"""
    return {"result": f"processed: {state['data']}"}


# 场景 3：dict 类型 - 推断失败
def node_dict_type(state: dict) -> dict:
    """dict 类型，推断失败，使用 state_schema"""
    return {"result": f"processed: {state['data']}"}


# 场景 4：正确的类型注解 - 推断成功
def node_typed(state: State) -> dict:
    """TypedDict 类型，推断成功"""
    return {"result": f"processed: {state['data']}"}


# 构建图 - 展示不同推断结果
def build_graph_inference_test():
    """测试类型推断的不同场景"""
    graph = StateGraph(State)

    # 所有节点都会回退到 State（因为推断失败）
    graph.add_node("no_annotation", node_no_annotation)
    graph.add_node("any_type", node_any_type)
    graph.add_node("dict_type", node_dict_type)
    graph.add_node("typed", node_typed)

    # 构建线性流程
    graph.add_edge(START, "no_annotation")
    graph.add_edge("no_annotation", "any_type")
    graph.add_edge("any_type", "dict_type")
    graph.add_edge("dict_type", "typed")
    graph.add_edge("typed", END)

    return graph.compile()


# 推断失败检测工具
def check_type_inference(func, expected_schema):
    """
    检查函数是否能被正确推断

    Args:
        func: 节点函数
        expected_schema: 期望的 schema

    Returns:
        bool: 是否能正确推断
    """
    from typing import get_type_hints
    import inspect

    try:
        # 尝试获取类型提示
        hints = get_type_hints(func)

        # 获取第一个参数名
        sig = inspect.signature(func)
        first_param = next(iter(sig.parameters.keys()))

        # 检查第一个参数的类型
        if first_param in hints:
            param_type = hints[first_param]

            # 检查是否是类且有 __annotations__
            if isinstance(param_type, type) and hasattr(param_type, "__annotations__"):
                print(f"✓ {func.__name__}: 推断成功 -> {param_type.__name__}")
                return True
            else:
                print(f"✗ {func.__name__}: 推断失败 (类型不是 TypedDict)")
                return False
        else:
            print(f"✗ {func.__name__}: 推断失败 (无类型注解)")
            return False

    except Exception as e:
        print(f"✗ {func.__name__}: 推断失败 ({type(e).__name__})")
        return False


# 测试
if __name__ == "__main__":
    print("=== 类型推断检测 ===")
    check_type_inference(node_no_annotation, State)
    check_type_inference(node_any_type, State)
    check_type_inference(node_dict_type, State)
    check_type_inference(node_typed, State)

    print("\n=== 运行图 ===")
    app = build_graph_inference_test()
    result = app.invoke({"data": "test", "result": ""})
    print(f"最终结果: {result}")
```

**输出示例**：
```
=== 类型推断检测 ===
✗ node_no_annotation: 推断失败 (无类型注解)
✗ node_any_type: 推断失败 (类型不是 TypedDict)
✗ node_dict_type: 推断失败 (类型不是 TypedDict)
✓ node_typed: 推断成功 -> State

=== 运行图 ===
最终结果: {'data': 'test', 'result': 'processed: test'}
```

**关键点**：
- 推断失败时，LangGraph 回退到使用图的 `state_schema`
- 推断失败不会报错，但会失去类型精确控制
- 使用检测工具可以提前发现推断问题

## 实战代码 4：方法和可调用对象推断

### 场景：支持多种可调用类型

```python
"""
方法和可调用对象的类型推断
LangGraph 支持函数、方法、可调用对象
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END


class State(TypedDict):
    """图状态"""
    value: int
    log: list[str]


class InputState(TypedDict):
    """输入状态"""
    value: int


# 场景 1：普通函数
def function_node(state: InputState) -> dict:
    """普通函数 - 推断成功"""
    return {
        "value": state["value"] + 1,
        "log": [f"function: {state['value']}"]
    }


# 场景 2：类方法
class NodeClass:
    """节点类"""

    def method_node(self, state: InputState) -> dict:
        """实例方法 - 推断成功"""
        return {
            "value": state["value"] * 2,
            "log": [f"method: {state['value']}"]
        }

    @staticmethod
    def static_node(state: InputState) -> dict:
        """静态方法 - 推断成功"""
        return {
            "value": state["value"] + 10,
            "log": [f"static: {state['value']}"]
        }

    @classmethod
    def class_node(cls, state: InputState) -> dict:
        """类方法 - 推断成功"""
        return {
            "value": state["value"] - 5,
            "log": [f"class: {state['value']}"]
        }


# 场景 3：可调用对象
class CallableNode:
    """可调用对象"""

    def __call__(self, state: InputState) -> dict:
        """__call__ 方法 - 推断成功"""
        return {
            "value": state["value"] ** 2,
            "log": [f"callable: {state['value']}"]
        }


# 构建图
def build_graph_callable_types():
    """测试不同可调用类型的推断"""
    graph = StateGraph(State)

    # 创建实例
    node_obj = NodeClass()
    callable_obj = CallableNode()

    # 添加不同类型的节点
    graph.add_node("function", function_node)
    graph.add_node("method", node_obj.method_node)
    graph.add_node("static", NodeClass.static_node)
    graph.add_node("class", NodeClass.class_node)
    graph.add_node("callable", callable_obj)

    # 构建流程
    graph.add_edge(START, "function")
    graph.add_edge("function", "method")
    graph.add_edge("method", "static")
    graph.add_edge("static", "class")
    graph.add_edge("class", "callable")
    graph.add_edge("callable", END)

    return graph.compile()


# 测试
if __name__ == "__main__":
    app = build_graph_callable_types()

    result = app.invoke({
        "value": 5,
        "log": []
    })

    print(f"最终结果: {result}")
    print(f"执行日志: {result['log']}")
    # 输出:
    # 最终结果: {'value': 441, 'log': ['function: 5', 'method: 6', 'static: 12', 'class: 22', 'callable: 17']}
```

**关键点**：
- LangGraph 支持函数、实例方法、静态方法、类方法、可调用对象
- 所有类型都能正确推断第一个参数的类型
- 推荐使用普通函数或可调用对象（更清晰）

## 实战代码 5：泛型类型推断

### 场景：多类型参数的推断

```python
"""
泛型类型推断
展示如何在多类型参数图中使用推断
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END


# 定义多个类型
class State(TypedDict):
    """主状态"""
    query: str
    result: str


class Context(TypedDict):
    """上下文"""
    user_id: str
    session_id: str


class Input(TypedDict):
    """输入"""
    query: str


class Output(TypedDict):
    """输出"""
    result: str


# 节点函数 - 使用不同的类型参数
def input_node(state: Input) -> dict:
    """
    输入节点
    推断: input_schema = Input
    """
    print(f"输入节点: {state}")
    return {"query": state["query"].upper()}


def process_node(state: State, context: Context) -> dict:
    """
    处理节点（带上下文）
    推断: input_schema = State
    context 通过 context_schema 传递
    """
    print(f"处理节点: state={state}, context={context}")
    return {
        "result": f"Processed '{state['query']}' for user {context['user_id']}"
    }


def output_node(state: State) -> Output:
    """
    输出节点
    推断: input_schema = State, output_schema = Output
    """
    print(f"输出节点: {state}")
    return {"result": state["result"]}


# 构建图 - 使用多类型参数
def build_graph_generic():
    """构建多类型参数图"""
    graph = StateGraph(
        state_schema=State,
        context_schema=Context,
        input_schema=Input,
        output_schema=Output
    )

    # 添加节点 - 自动推断
    graph.add_node("input", input_node)
    graph.add_node("process", process_node)
    graph.add_node("output", output_node)

    # 构建流程
    graph.add_edge(START, "input")
    graph.add_edge("input", "process")
    graph.add_edge("process", "output")
    graph.add_edge("output", END)

    return graph.compile()


# 测试
if __name__ == "__main__":
    app = build_graph_generic()

    # 调用时提供 context
    result = app.invoke(
        {"query": "hello"},
        config={
            "configurable": {
                "user_id": "user123",
                "session_id": "session456"
            }
        }
    )

    print(f"\n最终结果: {result}")
    # 输出:
    # 输入节点: {'query': 'hello'}
    # 处理节点: state={'query': 'HELLO', 'result': ''}, context={'user_id': 'user123', 'session_id': 'session456'}
    # 输出节点: {'query': 'HELLO', 'result': "Processed 'HELLO' for user user123"}
    # 最终结果: {'result': "Processed 'HELLO' for user user123"}
```

**关键点**：
- 图级别的 `input_schema` 和 `output_schema` 控制边界
- 节点级别的推断控制内部类型
- `context_schema` 不参与推断，通过 config 传递

## 类型提示最佳实践

### 1. 优先使用 TypedDict

```python
# ✓ 推荐：使用 TypedDict
from typing_extensions import TypedDict

class State(TypedDict):
    field: str

def node(state: State) -> dict:
    return {"field": "value"}

# ✗ 不推荐：使用 dict
def node(state: dict) -> dict:  # 无法推断
    return {"field": "value"}
```

### 2. 显式类型注解

```python
# ✓ 推荐：显式类型注解
def node(state: State) -> dict:
    return {"field": "value"}

# ✗ 不推荐：无类型注解
def node(state):  # 无法推断
    return {"field": "value"}
```

### 3. 避免循环引用

```python
# ✓ 推荐：使用字符串引用
from __future__ import annotations

class State(TypedDict):
    next_state: State | None  # 使用字符串引用

# ✗ 不推荐：直接引用（可能导致 NameError）
class State(TypedDict):
    next_state: "State" | None  # 可能推断失败
```

### 4. 使用 input_schema 覆盖

```python
# ✓ 推荐：需要精确控制时显式指定
graph.add_node(
    "node",
    node_func,
    input_schema=CustomInput  # 覆盖推断
)

# ✗ 不推荐：依赖推断但类型不明确
def node(state: dict) -> dict:  # 推断失败
    pass
graph.add_node("node", node)
```

### 5. 使用类型检查工具

```python
# ✓ 推荐：使用 mypy 或 pyright 检查
# mypy --strict your_graph.py

# ✓ 推荐：使用运行时检查
from typing import get_type_hints
hints = get_type_hints(node_func)
print(hints)  # 验证类型注解
```

## 常见问题

### Q1: 为什么我的节点没有被正确推断？

**A**: 检查以下几点：
1. 函数是否有类型注解
2. 类型是否是 TypedDict（不是 dict 或 Any）
3. 是否有循环引用导致 NameError
4. 使用 `get_type_hints()` 验证类型提示

### Q2: 推断失败会报错吗？

**A**: 不会。推断失败时，LangGraph 会回退到使用图的 `state_schema`，不会抛出异常。

### Q3: 如何调试类型推断问题？

**A**: 使用以下方法：
```python
from typing import get_type_hints
import inspect

# 检查类型提示
hints = get_type_hints(node_func)
print(f"类型提示: {hints}")

# 检查函数签名
sig = inspect.signature(node_func)
print(f"函数签名: {sig}")
```

### Q4: 显式指定和推断哪个优先级高？

**A**: 显式指定的 `input_schema` 优先级最高，会覆盖自动推断。

### Q5: 可以混合使用推断和显式指定吗？

**A**: 可以。不同节点可以使用不同的方式，LangGraph 会分别处理。

## 性能考虑

### 1. 推断开销

类型推断发生在图构建时（`add_node`），不影响运行时性能：

```python
# 构建时推断（一次性）
graph.add_node("node", node_func)  # 推断发生在这里

# 运行时无推断开销
app.invoke({"data": "test"})  # 无推断开销
```

### 2. 类型检查开销

LangGraph 不进行运行时类型检查，类型仅用于：
- 确定节点接收哪些字段
- 静态类型检查（mypy/pyright）

### 3. 推荐策略

- 开发阶段：使用自动推断（快速迭代）
- 生产阶段：显式指定（明确意图）
- 性能敏感：两者性能相同，选择更清晰的方式

## 总结

### 核心要点

1. **自动推断**：LangGraph 从函数签名自动推断输入类型
2. **推断优先级**：显式 > 推断 > state_schema
3. **推断条件**：需要 TypedDict 类型注解
4. **推断失败**：回退到 state_schema，不报错
5. **最佳实践**：优先使用 TypedDict，需要精确控制时显式指定
