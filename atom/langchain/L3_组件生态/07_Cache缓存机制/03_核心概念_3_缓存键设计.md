# 核心概念3：缓存键设计

> 缓存键生成规则、规范化策略与最佳实践

---

## 概述

缓存键设计是缓存机制的核心，决定了缓存的命中率和正确性。LangChain使用不同的缓存键策略来处理LLM缓存和Embedding缓存。

---

## 1. LLM缓存键设计

### 缓存键组成

```python
# LLM缓存键 = (prompt, llm_string)
cache_key = (prompt, llm_string)
```

**prompt**：
- 对于普通LLM：提示词字符串
- 对于Chat模型：序列化后的消息列表

**llm_string**：
- LLM配置的字符串表示
- 包含：模型名、温度、max_tokens、停止词等

### llm_string生成逻辑

```python
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7,
    max_tokens=1000,
    stop=["\n\n"]
)

# llm_string示例（简化）
llm_string = "model=gpt-4o-mini,temperature=0.7,max_tokens=1000,stop=['\n\n']"
```

**关键参数**：
- `model`: 模型名称
- `temperature`: 温度参数
- `max_tokens`: 最大token数
- `stop`: 停止词列表
- `top_p`: 核采样参数
- `frequency_penalty`: 频率惩罚
- `presence_penalty`: 存在惩罚

### Chat模型的prompt序列化

```python
from langchain_core.messages import HumanMessage, SystemMessage, AIMessage

# 消息列表
messages = [
    SystemMessage(content="You are a helpful assistant."),
    HumanMessage(content="What is AI?"),
    AIMessage(content="AI stands for Artificial Intelligence."),
    HumanMessage(content="Tell me more.")
]

# 序列化为字符串（简化示例）
prompt = """System: You are a helpful assistant.
Human: What is AI?
AI: AI stands for Artificial Intelligence.
Human: Tell me more."""
```

**序列化规则**：
- 按消息顺序拼接
- 包含角色标识（System/Human/AI）
- 保留消息内容
- 消息ID不包含在序列化中（2025年修复）

## 2. Embedding缓存键设计

### 缓存键组成

```python
# Embedding缓存键 = namespace + hash(text)
cache_key = f"{namespace}{hash(text)}"
```

**namespace**：
- 隔离不同embedding模型的缓存
- 通常使用模型名称
- 避免模型切换时的缓存冲突

**hash(text)**：
- 文本内容的哈希值
- 支持多种哈希算法
- 确保唯一性

### 哈希算法选择

```python
import hashlib

def generate_cache_key(text: str, namespace: str, algorithm: str) -> str:
    """生成缓存键"""

    if algorithm == "sha1":
        hash_value = hashlib.sha1(text.encode('utf-8')).hexdigest()
    elif algorithm == "blake2b":
        hash_value = hashlib.blake2b(text.encode('utf-8')).hexdigest()
    elif algorithm == "sha256":
        hash_value = hashlib.sha256(text.encode('utf-8')).hexdigest()
    elif algorithm == "sha512":
        hash_value = hashlib.sha512(text.encode('utf-8')).hexdigest()
    else:
        raise ValueError(f"Unsupported algorithm: {algorithm}")

    return f"{namespace}{hash_value}"

# 示例
text = "Hello, world!"
namespace = "text-embedding-3-small"

key_sha1 = generate_cache_key(text, namespace, "sha1")
key_blake2b = generate_cache_key(text, namespace, "blake2b")

print(f"SHA-1:    {key_sha1}")
print(f"BLAKE2B:  {key_blake2b}")
```

**算法对比**：

| 算法 | 速度 | 安全性 | 碰撞风险 | 哈希长度 | 推荐场景 |
|------|------|--------|----------|----------|----------|
| SHA-1 | 最快 | 低 | 高 | 40字符 | 开发测试 |
| BLAKE2B | 快 | 高 | 极低 | 128字符 | 生产环境（推荐） |
| SHA-256 | 中等 | 高 | 极低 | 64字符 | 生产环境 |
| SHA-512 | 慢 | 最高 | 极低 | 128字符 | 高安全要求 |

## 3. 缓存键规范化问题

### 问题1：消息ID影响缓存命中（2025年修复）

**问题描述**：
```python
from langchain_core.messages import HumanMessage

# 两次调用，消息ID不同
message1 = HumanMessage(content="Hello", id="msg_001")
message2 = HumanMessage(content="Hello", id="msg_002")

# 问题：消息ID不同导致缓存未命中
# 即使content相同，缓存键也不同
```

**解决方案**（2025年GitHub #33883）：
- 在缓存键生成前剥离消息ID
- 只使用消息内容和角色
- 确保相同内容命中缓存

**修复后的行为**：
```python
# 修复后：消息ID不影响缓存键
message1 = HumanMessage(content="Hello", id="msg_001")
message2 = HumanMessage(content="Hello", id="msg_002")

# 缓存键相同，命中缓存 ✅
```

### 问题2：total_cost注入破坏缓存键（2025年GitHub #35308）

**问题描述**：
```python
# 首次调用
response1 = llm.invoke("Hello")
# 缓存键：("Hello", "model=gpt-4o-mini,...")

# 缓存命中后，注入total_cost
# response1.usage_metadata.total_cost = 0.01

# 再次调用
response2 = llm.invoke("Hello")
# 问题：usage_metadata变化导致缓存键不同
```

**解决方案**：
- 规范化usage_metadata
- 在缓存键生成前移除动态字段
- 只保留静态配置参数

### 问题3：空格和换行影响缓存

**问题描述**：
```python
# 三个看似相同的prompt
prompt1 = "What is AI?"
prompt2 = "What is AI? "  # 末尾多一个空格
prompt3 = "What is AI?\n"  # 末尾多一个换行

# 问题：三个prompt的缓存键都不同
# 导致缓存未命中
```

**解决方案**：
```python
def normalize_prompt(prompt: str) -> str:
    """规范化prompt"""
    # 去除首尾空白
    prompt = prompt.strip()

    # 统一换行符
    prompt = prompt.replace('\r\n', '\n')

    # 压缩多个空格为一个
    import re
    prompt = re.sub(r'\s+', ' ', prompt)

    return prompt

# 使用规范化
prompt1 = normalize_prompt("What is AI?")
prompt2 = normalize_prompt("What is AI? ")
prompt3 = normalize_prompt("What is AI?\n")

# 结果：三个prompt相同，缓存命中 ✅
```

## 4. namespace设计最佳实践

### 为什么需要namespace？

**问题场景**：
```python
# 场景：切换embedding模型
embedder1 = OpenAIEmbeddings(model="text-embedding-ada-002")
embedder2 = OpenAIEmbeddings(model="text-embedding-3-small")

# 相同文本
text = "Hello, world!"

# 问题：两个模型的embedding不同
# 但如果没有namespace，缓存键相同
# 导致错误地返回旧模型的embedding
```

**解决方案**：
```python
# 使用namespace隔离
cached_embedder1 = CacheBackedEmbeddings.from_bytes_store(
    embedder1,
    store,
    namespace="text-embedding-ada-002"  # 模型1的namespace
)

cached_embedder2 = CacheBackedEmbeddings.from_bytes_store(
    embedder2,
    store,
    namespace="text-embedding-3-small"  # 模型2的namespace
)

# 结果：两个模型的缓存互不干扰 ✅
```

### namespace命名规范

```python
# ✅ 推荐：使用模型名称
namespace = underlying_embeddings.model
# 示例："text-embedding-3-small"

# ✅ 推荐：包含版本信息
namespace = f"{model_name}-v{version}"
# 示例："text-embedding-3-small-v1"

# ✅ 推荐：包含应用标识
namespace = f"{app_name}-{model_name}"
# 示例："rag-app-text-embedding-3-small"

# ❌ 避免：使用通用名称
namespace = "embeddings"  # 太通用，容易冲突

# ❌ 避免：不设置namespace
namespace = ""  # 没有隔离
```

### 多应用共享存储

```python
# 场景：多个应用共享Redis
# 应用A
cached_embedder_a = CacheBackedEmbeddings.from_bytes_store(
    embeddings,
    redis_store,
    namespace="app-a:text-embedding-3-small"
)

# 应用B
cached_embedder_b = CacheBackedEmbeddings.from_bytes_store(
    embeddings,
    redis_store,
    namespace="app-b:text-embedding-3-small"
)

# 结果：两个应用的缓存互不干扰
```

## 5. 缓存键冲突检测

### 检测SHA-1碰撞风险

```python
import hashlib
from collections import defaultdict

def detect_collisions(texts: list[str], algorithm: str = "sha1") -> dict:
    """检测哈希碰撞"""
    hash_to_texts = defaultdict(list)

    for text in texts:
        if algorithm == "sha1":
            hash_value = hashlib.sha1(text.encode()).hexdigest()
        elif algorithm == "blake2b":
            hash_value = hashlib.blake2b(text.encode()).hexdigest()
        else:
            raise ValueError(f"Unsupported algorithm: {algorithm}")

        hash_to_texts[hash_value].append(text)

    # 找出碰撞
    collisions = {h: texts for h, texts in hash_to_texts.items() if len(texts) > 1}

    return {
        "total_texts": len(texts),
        "unique_hashes": len(hash_to_texts),
        "collisions": len(collisions),
        "collision_details": collisions
    }

# 测试
texts = [f"Document {i}" for i in range(10000)]
result_sha1 = detect_collisions(texts, "sha1")
result_blake2b = detect_collisions(texts, "blake2b")

print(f"SHA-1 碰撞: {result_sha1['collisions']}")
print(f"BLAKE2B 碰撞: {result_blake2b['collisions']}")
```

### 监控缓存键分布

```python
class CacheKeyMonitor:
    """缓存键监控器"""

    def __init__(self):
        self.key_counts = defaultdict(int)
        self.key_sizes = []

    def record_key(self, key: str):
        """记录缓存键"""
        self.key_counts[key] += 1
        self.key_sizes.append(len(key))

    def get_stats(self) -> dict:
        """获取统计信息"""
        return {
            "total_keys": len(self.key_counts),
            "total_accesses": sum(self.key_counts.values()),
            "avg_key_size": sum(self.key_sizes) / len(self.key_sizes) if self.key_sizes else 0,
            "max_key_size": max(self.key_sizes) if self.key_sizes else 0,
            "duplicate_keys": sum(1 for count in self.key_counts.values() if count > 1),
            "top_keys": sorted(self.key_counts.items(), key=lambda x: x[1], reverse=True)[:10]
        }

# 使用示例
monitor = CacheKeyMonitor()

# 记录缓存键
for text in texts:
    key = generate_cache_key(text, "model-name", "blake2b")
    monitor.record_key(key)

# 查看统计
stats = monitor.get_stats()
print(f"总缓存键数: {stats['total_keys']}")
print(f"平均键长度: {stats['avg_key_size']:.2f}")
print(f"重复键数: {stats['duplicate_keys']}")
```

## 6. 自定义缓存键编码器

### 实现自定义编码器

```python
from typing import Callable

def create_custom_key_encoder(
    namespace: str,
    prefix: str = "",
    suffix: str = ""
) -> Callable[[str], str]:
    """创建自定义键编码器"""

    def key_encoder(text: str) -> str:
        # 1. 规范化文本
        text = text.strip().lower()

        # 2. 生成哈希
        hash_value = hashlib.blake2b(text.encode()).hexdigest()

        # 3. 组合键
        key = f"{prefix}{namespace}:{hash_value}{suffix}"

        return key

    return key_encoder

# 使用自定义编码器
custom_encoder = create_custom_key_encoder(
    namespace="my-app",
    prefix="cache:",
    suffix=":v1"
)

cached_embedder = CacheBackedEmbeddings.from_bytes_store(
    underlying_embeddings,
    store,
    namespace="my-app",
    key_encoder=custom_encoder  # 使用自定义编码器
)
```

### 带版本控制的编码器

```python
def create_versioned_key_encoder(
    namespace: str,
    version: str
) -> Callable[[str], str]:
    """创建带版本控制的键编码器"""

    def key_encoder(text: str) -> str:
        # 包含版本信息
        versioned_text = f"{version}:{text}"
        hash_value = hashlib.blake2b(versioned_text.encode()).hexdigest()
        return f"{namespace}:v{version}:{hash_value}"

    return key_encoder

# 使用版本控制
encoder_v1 = create_versioned_key_encoder("my-app", "1.0")
encoder_v2 = create_versioned_key_encoder("my-app", "2.0")

# 版本1的缓存
cached_embedder_v1 = CacheBackedEmbeddings.from_bytes_store(
    embeddings,
    store,
    namespace="my-app",
    key_encoder=encoder_v1
)

# 版本2的缓存（不会与v1冲突）
cached_embedder_v2 = CacheBackedEmbeddings.from_bytes_store(
    embeddings,
    store,
    namespace="my-app",
    key_encoder=encoder_v2
)
```

### 场景3：环境隔离（开发/测试/生产）

import os

def create_env_cached_embedder(
    embeddings: Embeddings,
    store: BaseStore
) -> CacheBackedEmbeddings:

## 9. 常见问题

### Q1: 为什么缓存没有命中？

**检查清单**：
1. prompt是否完全相同？（包括空格、换行）
2. llm_string是否相同？（配置参数）
3. namespace是否正确？
4. 缓存是否被清空？

**调试方法**：
```python
# 打印缓存键
print(f"Prompt: {repr(prompt)}")
print(f"LLM String: {llm_string}")
print(f"Cache Key: {(prompt, llm_string)}")
```

### Q2: 如何处理哈希碰撞？

**解决方案**：
1. 使用更安全的哈希算法（BLAKE2B、SHA-256）
2. 监控碰撞情况
3. 必要时使用自定义编码器

### Q3: namespace应该多细粒度？

**建议**：
- 最小粒度：模型名称
- 推荐粒度：模型名称 + 版本
- 多租户：租户ID + 模型名称
- 多环境：环境 + 模型名称
