# 核心概念3:回调管理器系统

> LangChain 的回调管理器负责组织、传递和执行回调处理器

---

## 概述

**回调管理器系统**是 LangChain 中管理回调处理器生命周期的核心机制，负责回调的注册、传递、执行和层次化管理。

**为什么需要回调管理器？**
- **统一管理**：集中管理多个回调处理器
- **层次传递**：支持父子链路的回调传递
- **生命周期**：管理回调的开始、结束、错误
- **并发安全**：处理异步和并发场景

---

## 核心概念1：CallbackManager（回调管理器）

**一句话定义：** 负责注册和执行多个回调处理器的管理器类。

### 设计架构

```python
# 源码简化版本
class CallbackManager(BaseCallbackManager):
    """同步回调管理器"""
    
    def __init__(
        self,
        handlers: List[BaseCallbackHandler],
        inheritable_handlers: List[BaseCallbackHandler] | None = None,
        parent_run_id: UUID | None = None,
        tags: List[str] | None = None,
        metadata: Dict[str, Any] | None = None,
    ):
        self.handlers = handlers
        self.inheritable_handlers = inheritable_handlers or []
        self.parent_run_id = parent_run_id
        self.tags = tags or []
        self.metadata = metadata or {}
```

### 核心职责

**1. 回调处理器注册**
```python
from langchain_core.callbacks import CallbackManager, StdOutCallbackHandler
from langchain_community.callbacks import FileCallbackHandler

# 创建回调管理器
manager = CallbackManager(
    handlers=[
        StdOutCallbackHandler(),
        FileCallbackHandler("log.txt")
    ]
)
```

**2. 回调事件分发**
```python
# 管理器会将事件分发给所有注册的处理器
manager.on_llm_start(serialized={}, prompts=["query"], run_id=run_id)
# 所有处理器的 on_llm_start 方法都会被调用
```

**3. 可继承处理器**
```python
# 可继承处理器会传递给子链
manager = CallbackManager(
    handlers=[handler1],  # 仅当前链使用
    inheritable_handlers=[handler2]  # 会传递给子链
)
```

### AsyncCallbackManager（异步回调管理器）

```python
class AsyncCallbackManager(BaseCallbackManager):
    """异步回调管理器"""
    
    async def on_llm_start(self, serialized, prompts, **kwargs):
        """异步执行所有处理器的回调"""
        await asyncio.gather(
            *[
                handler.on_llm_start(serialized, prompts, **kwargs)
                for handler in self.handlers
            ]
        )
```

**使用场景：**
```python
from langchain_core.callbacks import AsyncCallbackManager

# 异步回调管理器
async_manager = AsyncCallbackManager(
    handlers=[AsyncCallbackHandler()]
)

# 在异步环境中使用
await llm.ainvoke("query", config={"callbacks": async_manager})
```

## 核心概念2：RunManager（运行管理器）

**一句话定义：** 管理单次运行（LLM、Chain、Tool、Retriever）的回调生命周期。

### 层次结构

LangChain 为不同组件提供专门的 RunManager：

```python
# 源码架构
class CallbackManagerForLLMRun(RunManager):
    """LLM 运行管理器"""
    
    def on_llm_new_token(self, token: str, **kwargs) -> None:
        """处理新 token"""
        for handler in self.handlers:
            handler.on_llm_new_token(token, **kwargs)
    
    def on_llm_end(self, response: LLMResult, **kwargs) -> None:
        """处理 LLM 结束"""
        for handler in self.handlers:
            handler.on_llm_end(response, **kwargs)
    
    def on_llm_error(self, error: Exception, **kwargs) -> None:
        """处理 LLM 错误"""
        for handler in self.handlers:
            handler.on_llm_error(error, **kwargs)

class CallbackManagerForChainRun(RunManager):
    """Chain 运行管理器"""
    
    def on_chain_end(self, outputs: Dict, **kwargs) -> None:
        for handler in self.handlers:
            handler.on_chain_end(outputs, **kwargs)
    
    def on_chain_error(self, error: Exception, **kwargs) -> None:
        for handler in self.handlers:
            handler.on_chain_error(error, **kwargs)
    
    def on_agent_action(self, action: AgentAction, **kwargs) -> None:
        for handler in self.handlers:
            handler.on_agent_action(action, **kwargs)

class CallbackManagerForToolRun(RunManager):
    """Tool 运行管理器"""
    
    def on_tool_end(self, output: str, **kwargs) -> None:
        for handler in self.handlers:
            handler.on_tool_end(output, **kwargs)

class CallbackManagerForRetrieverRun(RunManager):
    """Retriever 运行管理器"""
    
    def on_retriever_end(self, documents: List[Document], **kwargs) -> None:
        for handler in self.handlers:
            handler.on_retriever_end(documents, **kwargs)
```

### 生命周期管理

**完整的运行生命周期：**

```python
# 1. 开始阶段
run_manager = callback_manager.on_llm_start(
    serialized={"name": "ChatOpenAI"},
    prompts=["query"],
    run_id=UUID("..."),
    parent_run_id=None
)

# 2. 执行阶段
run_manager.on_llm_new_token("token1")
run_manager.on_llm_new_token("token2")

# 3. 结束阶段
run_manager.on_llm_end(response)

# 或错误阶段
run_manager.on_llm_error(exception)
```

### 在 RAG 开发中的应用

```python
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import StdOutCallbackHandler

# 创建 LLM
llm = ChatOpenAI()

# 回调会自动通过 RunManager 管理
response = llm.invoke(
    "query",
    config={"callbacks": [StdOutCallbackHandler()]}
)

# 内部流程：
# 1. CallbackManager 创建 CallbackManagerForLLMRun
# 2. RunManager 调用 on_llm_start
# 3. RunManager 调用 on_llm_new_token (如果流式)
# 4. RunManager 调用 on_llm_end
```

## 核心概念3：回调传递机制

**一句话定义：** 回调处理器在父子链路之间的传递和继承机制。

### 传递规则

**1. 父子链传递**

```python
from langchain.chains import LLMChain, SequentialChain

# 父链的回调会传递给子链
parent_chain = SequentialChain(
    chains=[child_chain1, child_chain2],
    callbacks=[handler]  # 会传递给所有子链
)
```

**2. run_id 和 parent_run_id**

```python
# 回调系统使用 UUID 追踪父子关系
class CallbackHandler(BaseCallbackHandler):
    def on_chain_start(self, serialized, inputs, run_id, parent_run_id, **kwargs):
        print(f"Chain {run_id} started")
        print(f"Parent: {parent_run_id}")
```

**层次结构示例：**
```
SequentialChain (run_id=A, parent_run_id=None)
├── LLMChain1 (run_id=B, parent_run_id=A)
│   └── ChatOpenAI (run_id=C, parent_run_id=B)
└── LLMChain2 (run_id=D, parent_run_id=A)
    └── ChatOpenAI (run_id=E, parent_run_id=D)
```

**3. 可继承 vs 不可继承**

```python
from langchain_core.callbacks import CallbackManager

manager = CallbackManager(
    handlers=[handler1],  # 仅当前层使用
    inheritable_handlers=[handler2]  # 传递给子链
)
```

### trace_as_chain_group（链组追踪）

**一句话定义：** 将多个独立调用组合成一个追踪组的上下文管理器。

**使用场景：**
```python
from langchain_core.callbacks import trace_as_chain_group

# 将多个独立调用组合成一个追踪组
with trace_as_chain_group("my_workflow") as group_manager:
    # 第一步：检索
    docs = retriever.invoke(
        "query",
        config={"callbacks": group_manager}
    )
    
    # 第二步：生成
    response = llm.invoke(
        f"Context: {docs}\nQuery: query",
        config={"callbacks": group_manager}
    )

# 所有调用会被追踪为一个组
```

**源码简化版本：**
```python
@contextmanager
def trace_as_chain_group(
    group_name: str,
    callback_manager: CallbackManager | None = None,
    *,
    inputs: Dict[str, Any] | None = None,
    project_name: str | None = None,
    tags: List[str] | None = None,
    metadata: Dict[str, Any] | None = None,
) -> Generator[CallbackManagerForChainGroup, None, None]:
    """创建链组追踪上下文"""
    
    # 创建组管理器
    group_cm = CallbackManagerForChainGroup(
        group_name=group_name,
        handlers=callback_manager.handlers if callback_manager else [],
        tags=tags,
        metadata=metadata
    )
    
    # 调用 on_chain_start
    group_cm.on_chain_start(
        serialized={"name": group_name},
        inputs=inputs or {}
    )
    
    try:
        yield group_cm
    finally:
        # 调用 on_chain_end
        group_cm.on_chain_end(outputs={})
```

## 回调管理器对比

| 管理器类型 | 用途 | 生命周期 | 适用场景 |
|-----------|------|---------|---------|
| **CallbackManager** | 管理多个回调处理器 | 整个应用 | 全局配置 |
| **AsyncCallbackManager** | 异步回调管理 | 整个应用 | 异步环境 |
| **CallbackManagerForLLMRun** | 管理单次 LLM 调用 | 单次调用 | LLM 执行 |
| **CallbackManagerForChainRun** | 管理单次 Chain 执行 | 单次执行 | Chain 执行 |
| **CallbackManagerForToolRun** | 管理单次 Tool 调用 | 单次调用 | Tool 执行 |
| **CallbackManagerForRetrieverRun** | 管理单次检索 | 单次检索 | 检索执行 |
| **CallbackManagerForChainGroup** | 管理链组追踪 | 上下文范围 | 多步骤流程 |

## 实战代码示例

```python
"""
回调管理器系统完整示例
演示：CallbackManager、RunManager、回调传递、trace_as_chain_group
"""

import os
from uuid import uuid4
from langchain_openai import ChatOpenAI
from langchain_core.callbacks import (
    CallbackManager,
    StdOutCallbackHandler,
    BaseCallbackHandler
)
from langchain_core.callbacks import trace_as_chain_group
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

# ===== 1. 自定义回调处理器（追踪层次结构）=====
class HierarchyCallbackHandler(BaseCallbackHandler):
    """追踪回调层次结构的处理器"""
    
    def __init__(self):
        self.indent_level = 0
    
    def _print(self, message: str):
        print("  " * self.indent_level + message)
    
    def on_chain_start(self, serialized, inputs, run_id, parent_run_id, **kwargs):
        self._print(f"[Chain Start] {serialized.get('name', 'Unknown')}")
        self._print(f"  run_id: {run_id}")
        self._print(f"  parent_run_id: {parent_run_id}")
        self.indent_level += 1
    
    def on_chain_end(self, outputs, run_id, parent_run_id, **kwargs):
        self.indent_level -= 1
        self._print(f"[Chain End] run_id: {run_id}")
    
    def on_llm_start(self, serialized, prompts, run_id, parent_run_id, **kwargs):
        self._print(f"[LLM Start] {serialized.get('name', 'Unknown')}")
        self._print(f"  run_id: {run_id}")
        self._print(f"  parent_run_id: {parent_run_id}")
        self.indent_level += 1
    
    def on_llm_end(self, response, run_id, parent_run_id, **kwargs):
        self.indent_level -= 1
        self._print(f"[LLM End] run_id: {run_id}")

# ===== 2. CallbackManager 基础使用 =====
print("=== 示例1：CallbackManager 基础使用 ===\n")

# 创建回调管理器
manager = CallbackManager(
    handlers=[
        StdOutCallbackHandler(),
        HierarchyCallbackHandler()
    ]
)

llm = ChatOpenAI(temperature=0)
prompt = PromptTemplate(
    input_variables=["topic"],
    template="用一句话介绍 {topic}"
)
chain = LLMChain(llm=llm, prompt=prompt)

# 使用回调管理器
result = chain.invoke(
    {"topic": "LangChain"},
    config={"callbacks": manager}
)

print(f"\n结果: {result['text']}\n")

# ===== 3. 回调传递机制 =====
print("=== 示例2：回调传递机制 ===\n")

from langchain.chains import SequentialChain

# 创建子链
chain1 = LLMChain(
    llm=llm,
    prompt=PromptTemplate(
        input_variables=["topic"],
        template="列出 {topic} 的3个特点"
    ),
    output_key="features"
)

chain2 = LLMChain(
    llm=llm,
    prompt=PromptTemplate(
        input_variables=["features"],
        template="总结这些特点: {features}"
    ),
    output_key="summary"
)

# 创建父链
parent_chain = SequentialChain(
    chains=[chain1, chain2],
    input_variables=["topic"],
    output_variables=["summary"]
)

# 父链的回调会传递给子链
result = parent_chain.invoke(
    {"topic": "RAG"},
    config={"callbacks": [HierarchyCallbackHandler()]}
)

print(f"\n结果: {result['summary']}\n")

# ===== 4. trace_as_chain_group 使用 =====
print("=== 示例3：trace_as_chain_group 使用 ===\n")

# 模拟 RAG 流程
with trace_as_chain_group(
    "rag_workflow",
    tags=["rag", "demo"],
    metadata={"user": "demo_user"}
) as group:
    print("[Group Start] rag_workflow\n")
    
    # 步骤1：检索（模拟）
    print("  [Step 1] 检索文档...")
    docs = ["文档1内容", "文档2内容"]
    
    # 步骤2：生成
    print("  [Step 2] 生成答案...")
    context = "\n".join(docs)
    response = llm.invoke(
        f"基于以下内容回答问题:\n{context}\n\n问题: 什么是 RAG?",
        config={"callbacks": group}
    )
    
    print(f"\n  答案: {response.content}\n")
    
    print("[Group End] rag_workflow\n")

# ===== 5. 异步回调管理器 =====
print("=== 示例4：异步回调管理器 ===\n")

import asyncio
from langchain_core.callbacks import AsyncCallbackManager

async def async_example():
    """异步回调示例"""
    
    # 创建异步回调管理器
    async_manager = AsyncCallbackManager(
        handlers=[HierarchyCallbackHandler()]
    )
    
    # 异步调用
    result = await llm.ainvoke(
        "用一句话介绍异步编程",
        config={"callbacks": async_manager}
    )
    
    print(f"\n结果: {result.content}\n")

# 运行异步示例
asyncio.run(async_example())

print("=== 示例完成 ===")
```

## 关键要点总结

1. **CallbackManager**：管理多个回调处理器，支持可继承和不可继承
2. **RunManager**：管理单次运行的回调生命周期（LLM、Chain、Tool、Retriever）
3. **回调传递**：通过 run_id 和 parent_run_id 追踪父子关系
4. **trace_as_chain_group**：将多个独立调用组合成一个追踪组
5. **异步支持**：AsyncCallbackManager 处理异步场景

**在 RAG 开发中：**
- 使用 CallbackManager 统一管理回调
- 使用 trace_as_chain_group 追踪完整 RAG 流程
- 使用 RunManager 追踪每个组件的执行
- 使用 parent_run_id 分析调用链路
