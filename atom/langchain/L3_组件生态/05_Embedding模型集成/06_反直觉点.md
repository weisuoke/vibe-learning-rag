# 反直觉点

## 误区1：embed_documents 和 embed_query 应该返回完全相同的结果

❌ **错误观点**
认为两个方法本质相同，只是输入形式不同（批量 vs 单个）。

**为什么错**
LangChain 的设计允许文档和查询使用不同的嵌入策略：
- OpenAI 的某些模型对查询有特殊优化
- 某些模型对文档和查询使用不同的 prompt
- 允许未来扩展（如查询重写、查询增强）

**为什么容易这样错**
直观上认为"嵌入"是一个确定的操作，输入相同应该输出相同。

✅ **正确理解**
```python
# 同一文本在不同方法中可能有不同处理
embeddings = OpenAIEmbeddings()

# 文档嵌入：可能添加 "Represent this document for retrieval" 前缀
doc_vector = embeddings.embed_documents(["hello world"])[0]

# 查询嵌入：可能添加 "Represent this query for retrieval" 前缀
query_vector = embeddings.embed_query("hello world")

# 向量可能不同！这是设计特性，不是 bug
```

## 误区2：使用 CacheBackedEmbeddings 就能显著提升性能

❌ **错误观点**
只要包装 Embeddings 就能自动获得性能提升。

**为什么错**
性能提升取决于多个因素：
- 缓存命中率（查询重复度）
- 文档解析速度（可能是瓶颈）
- 批量处理效率
- 存储 I/O 开销

**为什么容易这样错**
缓存通常能提升性能，所以假设它总是有效的。

✅ **正确理解**
```python
# 缓存只在查询重复时有效
store = LocalFileStore("./cache")
cached_embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    namespace="openai-ada-002"
)

# 场景1：高重复查询 → 缓存有效
vectors = cached_embeddings.embed_documents(["doc1", "doc2"])  # 计算
vectors = cached_embeddings.embed_documents(["doc1", "doc2"])  # 从缓存读取 ✓

# 场景2：低重复查询 → 缓存无效
vectors = cached_embeddings.embed_documents(["unique_doc_1"])  # 计算
vectors = cached_embeddings.embed_documents(["unique_doc_2"])  # 计算
# 缓存命中率低，性能提升不明显
```

## 误区3：默认的 SHA-1 哈希足够安全

❌ **错误观点**
使用默认的 SHA-1 算法是安全的。

**为什么错**
SHA-1 存在碰撞风险（虽然对大多数缓存场景可接受）。LangChain 源码明确警告：
> "SHA-1 is *not* collision-resistant. A motivated attacker could craft two different payloads that map to the same cache key."

**为什么容易这样错**
SHA-1 是常见的哈希算法，且 LangChain 默认使用它。

✅ **正确理解**
```python
# 根据场景选择哈希算法
from langchain_classic.embeddings import CacheBackedEmbeddings
from langchain_classic.storage import LocalFileStore

store = LocalFileStore("./cache")

# 场景1：一般应用 → SHA-1 足够（性能优先）
embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    key_encoder="sha1"  # 默认，快速
)

# 场景2：安全敏感应用 → 使用更强的算法
embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    key_encoder="blake2b"  # 密码学强度，更安全
)

# 场景3：高安全需求 → SHA-256
embeddings = CacheBackedEmbeddings.from_bytes_store(
    OpenAIEmbeddings(),
    store,
    key_encoder="sha256"  # 最安全但较慢
)
```

---

## 总结

这三个误区的共同点是**过度简化**了 LangChain 的设计：
1. 认为接口应该完全一致（忽视了灵活性设计）
2. 认为缓存总是有效（忽视了命中率的影响）
3. 认为默认配置总是最优（忽视了安全性权衡）

关键是理解 LangChain 的**设计哲学**：提供灵活性和合理默认值，但需要根据实际场景调整。
