## 示例 1：TypedDict 基础状态定义

**最简单的状态定义方式**

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义状态类型
class State(TypedDict):
    """最简单的状态定义"""
    message: str
    count: int

# 2. 定义节点函数
def process_node(state: State) -> State:
    """节点函数：接收状态，返回部分状态更新"""
    return {
        "message": f"Processed: {state['message']}",
        "count": state["count"] + 1
    }

# 3. 创建图
graph = StateGraph(State)
graph.add_node("process", process_node)
graph.add_edge(START, "process")
graph.add_edge("process", END)

# 4. 编译并运行
app = graph.compile()
result = app.invoke({"message": "Hello", "count": 0})
print(result)
# 输出: {'message': 'Processed: Hello', 'count': 1}
```

**关键点**：
- TypedDict 定义状态结构
- 节点函数返回部分状态（只更新需要改变的字段）
- 状态自动合并

## 示例 2：带 Reducer 的状态累积

**使用 Annotated 绑定 Reducer 函数**

```python
from typing_extensions import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END

# 1. 定义 Reducer 函数
def add_messages(left: list, right: list) -> list:
    """累积消息列表"""
    return left + right

# 2. 定义状态（使用 Annotated 绑定 Reducer）
class State(TypedDict):
    messages: Annotated[list[str], add_messages]  # 累积模式
    user_id: str  # 普通字段（覆盖模式）

# 3. 定义节点
def node1(state: State) -> State:
    return {"messages": ["Message from node1"]}

def node2(state: State) -> State:
    return {"messages": ["Message from node2"]}

# 4. 创建图
graph = StateGraph(State)
graph.add_node("node1", node1)
graph.add_node("node2", node2)
graph.add_edge(START, "node1")
graph.add_edge("node1", "node2")
graph.add_edge("node2", END)

# 5. 运行
app = graph.compile()
result = app.invoke({"messages": [], "user_id": "user123"})
print(result["messages"])
# 输出: ['Message from node1', 'Message from node2']
```

**关键点**：
- `Annotated[type, reducer]` 绑定累积逻辑
- Reducer 函数接受两个参数：当前值和新值
- 未绑定 Reducer 的字段使用覆盖模式

[来源: reference/source_状态类型系统_01.md | LangGraph 源码]

## 示例 3：Pydantic 模型状态

**使用 Pydantic 进行运行时验证**

```python
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph, START, END

# 1. 定义 Pydantic 状态
class State(BaseModel):
    """带验证的状态定义"""
    query: str = Field(..., min_length=1, max_length=1000)
    user_id: str = Field(..., pattern=r'^[a-zA-Z0-9_-]+$')
    count: int = Field(default=0, ge=0)

# 2. 定义节点
def validate_node(state: State) -> dict:
    """节点返回字典，LangGraph 会自动验证"""
    return {
        "query": f"Validated: {state.query}",
        "count": state.count + 1
    }

# 3. 创建图
graph = StateGraph(State)
graph.add_node("validate", validate_node)
graph.add_edge(START, "validate")
graph.add_edge("validate", END)

# 4. 运行（自动验证）
app = graph.compile()

# 成功案例
result = app.invoke({"query": "Hello", "user_id": "user_123"})
print(result.query)  # 输出: Validated: Hello

# 失败案例（会抛出 ValidationError）
try:
    app.invoke({"query": "", "user_id": "user_123"})  # query 太短
except Exception as e:
    print(f"Validation failed: {e}")
```

**关键点**：
- Pydantic 提供运行时类型验证
- `Field()` 添加约束条件
- 验证失败会抛出 `ValidationError`

[来源: reference/context7_pydantic_01.md | Pydantic 官方文档]

## 示例 4：可选字段与默认值

**使用 Required/NotRequired 控制字段必需性**

```python
from typing_extensions import TypedDict, NotRequired, Required
from langgraph.graph import StateGraph, START, END

# 1. 定义带可选字段的状态
class State(TypedDict, total=False):  # total=False: 所有字段默认可选
    message: Required[str]  # 显式标记为必需
    count: Required[int]    # 显式标记为必需
    metadata: NotRequired[dict]  # 可选字段
    timestamp: NotRequired[str]  # 可选字段

# 2. 定义节点
def process_node(state: State) -> State:
    result = {
        "message": f"Processed: {state['message']}",
        "count": state["count"] + 1
    }
    # 只在存在时更新 metadata
    if "metadata" in state:
        result["metadata"] = {**state["metadata"], "processed": True}
    return result

# 3. 创建图
graph = StateGraph(State)
graph.add_node("process", process_node)
graph.add_edge(START, "process")
graph.add_edge("process", END)

# 4. 运行（可选字段可以省略）
app = graph.compile()

# 只提供必需字段
result1 = app.invoke({"message": "Hello", "count": 0})
print(result1)
# 输出: {'message': 'Processed: Hello', 'count': 1}

# 提供可选字段
result2 = app.invoke({
    "message": "Hello",
    "count": 0,
    "metadata": {"source": "user"}
})
print(result2)
# 输出: {'message': 'Processed: Hello', 'count': 1, 'metadata': {'source': 'user', 'processed': True}}
```

**关键点**：
- `total=False` 使所有字段默认可选
- `Required[T]` 显式标记必需字段
- `NotRequired[T]` 显式标记可选字段

[来源: reference/context7_typing_extensions_01.md | typing_extensions 官方文档]

## 示例 5：泛型类型系统

**分离输入、输出和内部状态**

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义不同的状态类型
class InternalState(TypedDict):
    """内部状态（完整信息）"""
    messages: list[str]
    user_id: str
    internal_data: dict

class InputState(TypedDict):
    """输入状态（用户提供）"""
    query: str
    user_id: str

class OutputState(TypedDict):
    """输出状态（返回给用户）"""
    response: str

# 2. 定义节点（使用内部状态）
def process_node(state: InternalState) -> InternalState:
    return {
        "messages": state["messages"] + [state.get("query", "")],
        "internal_data": {"processed": True}
    }

def format_output(state: InternalState) -> InternalState:
    return {
        "response": f"Processed {len(state['messages'])} messages"
    }

# 3. 创建图（指定输入和输出类型）
graph = StateGraph(
    state_schema=InternalState,
    input_schema=InputState,
    output_schema=OutputState
)
graph.add_node("process", process_node)
graph.add_node("format", format_output)
graph.add_edge(START, "process")
graph.add_edge("process", "format")
graph.add_edge("format", END)

# 4. 运行
app = graph.compile()

# 输入只需要 InputState 的字段
result = app.invoke({"query": "Hello", "user_id": "user123"})

# 输出只包含 OutputState 的字段
print(result)
# 输出: {'response': 'Processed 1 messages'}
```

**关键点**：
- `state_schema`: 内部状态（节点使用）
- `input_schema`: 输入状态（用户提供）
- `output_schema`: 输出状态（返回给用户）
- 自动类型转换和过滤

[来源: reference/source_状态类型系统_01.md | LangGraph 源码]

## 示例 6：类型推断

**自动推断节点输入类型**

```python
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END

# 1. 定义多个状态类型
class FullState(TypedDict):
    message: str
    count: int
    metadata: dict

class PartialState(TypedDict):
    message: str
    count: int

# 2. 定义节点（显式类型注解）
def node_with_full_state(state: FullState) -> FullState:
    """自动推断输入类型为 FullState"""
    return {
        "message": f"Full: {state['message']}",
        "metadata": {"processed": True}
    }

def node_with_partial_state(state: PartialState) -> PartialState:
    """自动推断输入类型为 PartialState"""
    return {
        "message": f"Partial: {state['message']}",
        "count": state["count"] + 1
    }

# 3. 创建图（不指定 input_schema，自动推断）
graph = StateGraph(FullState)

# 添加节点时，LangGraph 会从函数签名推断输入类型
graph.add_node("full", node_with_full_state)
graph.add_node("partial", node_with_partial_state)

graph.add_edge(START, "full")
graph.add_edge("full", "partial")
graph.add_edge("partial", END)

# 4. 运行
app = graph.compile()
result = app.invoke({
    "message": "Hello",
    "count": 0,
    "metadata": {}
})
print(result)
```

**关键点**：
- LangGraph 从函数签名的第一个参数推断输入类型
- 推断优先级：显式 `input_schema` > 函数签名 > 图的 `state_schema`
- 类型推断提高代码可读性

[来源: reference/source_状态类型系统_01.md | LangGraph 源码]

## 示例 7：混合使用 TypedDict 和 Pydantic

**边界验证 + 内部性能优化**

```python
from typing_extensions import TypedDict
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph, START, END

# 1. 输入使用 Pydantic（边界验证）
class InputState(BaseModel):
    query: str = Field(..., min_length=1, max_length=1000)
    user_id: str = Field(..., pattern=r'^[a-zA-Z0-9_-]+$')

# 2. 内部使用 TypedDict（性能优化）
class InternalState(TypedDict):
    messages: list[str]
    user_id: str
    processed: bool

# 3. 输出使用 TypedDict（简单结构）
class OutputState(TypedDict):
    response: str
    message_count: int

# 4. 定义节点
def process_node(state: InternalState) -> InternalState:
    return {
        "messages": state["messages"] + ["Processed"],
        "processed": True
    }

def format_output(state: InternalState) -> InternalState:
    return {
        "response": f"Processed {len(state['messages'])} messages",
        "message_count": len(state["messages"])
    }

# 5. 创建图
graph = StateGraph(
    state_schema=InternalState,
    input_schema=InputState,
    output_schema=OutputState
)
graph.add_node("process", process_node)
graph.add_node("format", format_output)
graph.add_edge(START, "process")
graph.add_edge("process", "format")
graph.add_edge("format", END)

# 6. 运行
app = graph.compile()

# 输入会被 Pydantic 验证
result = app.invoke({"query": "Hello", "user_id": "user_123"})
print(result)
# 输出: {'response': 'Processed 1 messages', 'message_count': 1}
```

**关键点**：
- 输入边界使用 Pydantic 验证
- 内部状态使用 TypedDict 提高性能
- 输出使用简单类型
- 最佳实践：验证在边界，性能在内部

[来源: reference/search_状态类型系统_01.md | 社区最佳实践]

## 示例 8：完整的 RAG 状态系统

**实战：文档问答系统的状态定义**

```python
from typing_extensions import TypedDict, Annotated
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph, START, END

# 1. 定义 Reducer
def add_messages(left: list, right: list) -> list:
    return left + right

# 2. 输入状态（Pydantic 验证）
class RAGInput(BaseModel):
    query: str = Field(..., min_length=1, description="用户查询")
    user_id: str = Field(..., description="用户ID")
    top_k: int = Field(default=5, ge=1, le=20, description="检索数量")

# 3. 内部状态（TypedDict 性能）
class RAGState(TypedDict):
    query: str
    user_id: str
    top_k: int
    messages: Annotated[list[dict], add_messages]  # 累积消息
    documents: list[dict]  # 检索到的文档
    context: str  # 拼接的上下文
    response: str  # 最终回答

# 4. 输出状态
class RAGOutput(TypedDict):
    response: str
    source_count: int

# 5. 定义节点
def retrieve_node(state: RAGState) -> RAGState:
    """检索节点"""
    # 模拟检索
    docs = [
        {"content": f"Document {i}", "score": 0.9 - i * 0.1}
        for i in range(state["top_k"])
    ]
    return {
        "documents": docs,
        "messages": [{"role": "system", "content": "Retrieved documents"}]
    }

def generate_node(state: RAGState) -> RAGState:
    """生成节点"""
    context = "\n".join([doc["content"] for doc in state["documents"]])
    response = f"Based on {len(state['documents'])} documents: Answer to '{state['query']}'"
    return {
        "context": context,
        "response": response,
        "messages": [{"role": "assistant", "content": response}]
    }

def format_output_node(state: RAGState) -> RAGState:
    """格式化输出"""
    return {
        "response": state["response"],
        "source_count": len(state["documents"])
    }

# 6. 创建图
graph = StateGraph(
    state_schema=RAGState,
    input_schema=RAGInput,
    output_schema=RAGOutput
)
graph.add_node("retrieve", retrieve_node)
graph.add_node("generate", generate_node)
graph.add_node("format", format_output_node)
graph.add_edge(START, "retrieve")
graph.add_edge("retrieve", "generate")
graph.add_edge("generate", "format")
graph.add_edge("format", END)

# 7. 运行
app = graph.compile()
result = app.invoke({
    "query": "What is LangGraph?",
    "user_id": "user123",
    "top_k": 3
})
print(result)
# 输出: {'response': "Based on 3 documents: Answer to 'What is LangGraph?'", 'source_count': 3}
```

**关键点**：
- 输入验证：Pydantic 确保参数合法
- 内部状态：TypedDict 提高性能
- 消息累积：Annotated + Reducer
- 输出过滤：只返回必要字段
- 完整的 RAG 流程：检索 → 生成 → 格式化

[来源: reference/search_状态类型系统_01.md | RAG 实战案例]

## 快速参考

### TypedDict vs Pydantic 选择指南

| 场景 | 推荐类型 | 原因 |
|------|---------|------|
| 内部状态 | TypedDict | 性能好，无验证开销 |
| API 输入 | Pydantic | 运行时验证，防止错误 |
| 简单结构 | TypedDict | 语法简洁 |
| 复杂验证 | Pydantic | 支持约束、自定义验证器 |
| 高并发 | TypedDict | 无验证开销 |

### 常用 Reducer 函数

```python
# 列表累积
def add_list(left: list, right: list) -> list:
    return left + right

# 字典合并
def merge_dict(left: dict, right: dict) -> dict:
    return {**left, **right}

# 数值累加
def add_number(left: int, right: int) -> int:
    return left + right

# 保留最新值
def keep_latest(left: any, right: any) -> any:
    return right
```

### 类型修饰符速查

```python
from typing_extensions import Required, NotRequired, ReadOnly

class State(TypedDict, total=False):
    required_field: Required[str]      # 必需字段
    optional_field: NotRequired[int]   # 可选字段
    readonly_field: ReadOnly[str]      # 只读字段（Python 3.13+）
```
