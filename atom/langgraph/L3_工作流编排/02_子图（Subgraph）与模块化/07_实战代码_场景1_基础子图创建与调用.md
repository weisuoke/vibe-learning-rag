# 实战代码 - 场景1：基础子图创建与调用

> 构建文档处理系统，掌握子图的两种集成方式：共享状态键与状态转换

---

## 场景概述

**目标**：创建一个文档处理系统，父图负责接收和输出，子图负责文本处理，掌握子图的两种核心集成方式。

**系统架构**：
```
父图：接收文档 → [子图：文本清洗 → 分词 → 关键词提取] → 生成摘要
```

**核心步骤**：
1. 定义子图的 State Schema
2. 构建子图（文本处理流水线）
3. 定义父图的 State Schema
4. 方式1：共享状态键集成（直接添加编译后的子图）
5. 方式2：状态转换集成（包装函数手动转换状态）
6. 对比两种方式的运行结果

**适用场景**：
- 将复杂流程拆分为独立模块
- 多个工作流复用同一子图
- RAG 系统中文档预处理流水线

## 完整代码示例

### Part 1：子图定义（文本处理流水线）

```python
"""
基础子图创建与调用 - Part 1：子图定义
演示：构建独立的文本处理子图

来源：
- Context7 官方文档：https://docs.langchain.com/oss/python/langgraph/use-subgraphs
- LangGraph 源码分析：state.py, types.py
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import re


# ===== 1. 定义子图的 State Schema =====
print("=" * 60)
print("Part 1：构建文本处理子图")
print("=" * 60)

print("\n=== 步骤1：定义子图 State Schema ===")


class TextProcessState(TypedDict):
    """
    子图状态：文本处理流水线的共享数据

    字段说明：
    - raw_text: 原始文本（与父图共享的 key）
    - cleaned_text: 清洗后的文本（子图私有）
    - tokens: 分词结果（子图私有）
    - keywords: 提取的关键词（与父图共享的 key）
    """
    raw_text: str       # 共享 key：父图传入
    cleaned_text: str   # 私有 key：仅子图内部使用
    tokens: list[str]   # 私有 key：仅子图内部使用
    keywords: list[str] # 共享 key：子图传回父图


print("  TextProcessState 定义完成")
print("  - 共享 key: raw_text, keywords")
print("  - 私有 key: cleaned_text, tokens")


# ===== 2. 定义子图节点函数 =====
print("\n=== 步骤2：定义子图节点函数 ===")


def clean_text(state: TextProcessState) -> dict:
    """
    节点1：文本清洗
    - 去除多余空白
    - 去除特殊字符
    - 统一为小写

    类比：
    - 前端：sanitize 用户输入
    - 日常：洗菜，去掉泥土和杂质
    """
    raw = state["raw_text"]
    print(f"  [清洗节点] 接收原始文本: '{raw[:50]}...'")

    # 去除多余空白
    cleaned = re.sub(r'\s+', ' ', raw).strip()
    # 去除特殊字符（保留中文、英文、数字、空格）
    cleaned = re.sub(r'[^\w\s\u4e00-\u9fff]', '', cleaned)
    # 统一小写
    cleaned = cleaned.lower()

    print(f"  [清洗节点] 清洗完成: '{cleaned[:50]}...'")
    return {"cleaned_text": cleaned}


def tokenize(state: TextProcessState) -> dict:
    """
    节点2：分词
    - 按空格分割英文
    - 过滤停用词
    - 过滤短词

    类比：
    - 前端：split() 拆分字符串
    - 日常：把一段话拆成一个个词语
    """
    text = state["cleaned_text"]
    print(f"  [分词节点] 接收清洗文本: '{text[:50]}...'")

    # 简单的英文停用词
    stop_words = {
        "the", "a", "an", "is", "are", "was", "were", "in", "on",
        "at", "to", "for", "of", "with", "and", "or", "but", "not",
        "this", "that", "it", "be", "as", "by", "from", "has", "have"
    }

    # 按空格分词并过滤
    words = text.split()
    tokens = [w for w in words if w not in stop_words and len(w) > 2]

    print(f"  [分词节点] 分词完成: {len(tokens)} 个有效词")
    return {"tokens": tokens}


def extract_keywords(state: TextProcessState) -> dict:
    """
    节点3：关键词提取
    - 统计词频
    - 取 Top-5 高频词作为关键词

    类比：
    - 前端：数组 reduce 统计频率
    - 日常：找出文章中出现最多的词
    """
    tokens = state["tokens"]
    print(f"  [关键词节点] 接收 {len(tokens)} 个 token")

    # 统计词频
    freq: dict[str, int] = {}
    for token in tokens:
        freq[token] = freq.get(token, 0) + 1

    # 按频率排序，取 Top-5
    sorted_words = sorted(freq.items(), key=lambda x: x[1], reverse=True)
    keywords = [word for word, count in sorted_words[:5]]

    print(f"  [关键词节点] 提取关键词: {keywords}")
    return {"keywords": keywords}


print("  clean_text: 文本清洗")
print("  tokenize: 分词处理")
print("  extract_keywords: 关键词提取")


# ===== 3. 构建并编译子图 =====
print("\n=== 步骤3：构建并编译子图 ===")

subgraph_builder = StateGraph(TextProcessState)

# 添加节点
subgraph_builder.add_node("clean_text", clean_text)
subgraph_builder.add_node("tokenize", tokenize)
subgraph_builder.add_node("extract_keywords", extract_keywords)

# 添加边：线性流水线
subgraph_builder.add_edge(START, "clean_text")
subgraph_builder.add_edge("clean_text", "tokenize")
subgraph_builder.add_edge("tokenize", "extract_keywords")
subgraph_builder.add_edge("extract_keywords", END)

# 编译子图
text_process_subgraph = subgraph_builder.compile()

print("  子图结构: START -> clean_text -> tokenize -> extract_keywords -> END")
print("  编译完成")


# ===== 4. 单独测试子图 =====
print("\n=== 步骤4：单独测试子图 ===\n")

test_input = {
    "raw_text": "LangGraph is a framework for building   stateful, multi-actor applications with LLMs. LangGraph provides tools for creating agentic workflows and multi-agent systems.",
    "cleaned_text": "",
    "tokens": [],
    "keywords": [],
}

result = text_process_subgraph.invoke(test_input)

print(f"\n  子图输出:")
print(f"  - keywords: {result['keywords']}")
print(f"  - tokens 数量: {len(result['tokens'])}")

print("\n" + "=" * 60)
print("Part 1 完成：子图独立运行成功")
print("=" * 60)
```

### Part 2：方式1 - 共享状态键集成

```python
"""
基础子图创建与调用 - Part 2：共享状态键集成
演示：父子图通过同名 key 自动传递状态

核心原理：
- 父图和子图有同名的 state key 时，LangGraph 自动传递
- 子图私有 key 不会影响父图
- 这是最简单、代码量最少的集成方式

来源：
- Context7 官方文档：共享 key 自动传递
- 源码分析：state.py add_node 方法
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import re


# ===== 1. 子图定义（同 Part 1，此处完整展示）=====
print("=" * 60)
print("Part 2：共享状态键集成")
print("=" * 60)


class TextProcessState(TypedDict):
    raw_text: str       # 共享 key
    cleaned_text: str   # 子图私有
    tokens: list[str]   # 子图私有
    keywords: list[str] # 共享 key


def clean_text(state: TextProcessState) -> dict:
    raw = state["raw_text"]
    cleaned = re.sub(r'\s+', ' ', raw).strip()
    cleaned = re.sub(r'[^\w\s\u4e00-\u9fff]', '', cleaned)
    cleaned = cleaned.lower()
    print(f"  [子图-清洗] 完成")
    return {"cleaned_text": cleaned}


def tokenize(state: TextProcessState) -> dict:
    text = state["cleaned_text"]
    stop_words = {"the", "a", "an", "is", "are", "was", "in", "on", "at",
                  "to", "for", "of", "with", "and", "or", "but", "not",
                  "this", "that", "it", "be", "as", "by", "from", "has", "have"}
    words = text.split()
    tokens = [w for w in words if w not in stop_words and len(w) > 2]
    print(f"  [子图-分词] 完成: {len(tokens)} 个词")
    return {"tokens": tokens}


def extract_keywords(state: TextProcessState) -> dict:
    tokens = state["tokens"]
    freq: dict[str, int] = {}
    for token in tokens:
        freq[token] = freq.get(token, 0) + 1
    sorted_words = sorted(freq.items(), key=lambda x: x[1], reverse=True)
    keywords = [word for word, count in sorted_words[:5]]
    print(f"  [子图-关键词] 完成: {keywords}")
    return {"keywords": keywords}


# 构建并编译子图
subgraph_builder = StateGraph(TextProcessState)
subgraph_builder.add_node("clean_text", clean_text)
subgraph_builder.add_node("tokenize", tokenize)
subgraph_builder.add_node("extract_keywords", extract_keywords)
subgraph_builder.add_edge(START, "clean_text")
subgraph_builder.add_edge("clean_text", "tokenize")
subgraph_builder.add_edge("tokenize", "extract_keywords")
subgraph_builder.add_edge("extract_keywords", END)
text_process_subgraph = subgraph_builder.compile()


# ===== 2. 定义父图的 State Schema =====
print("\n=== 步骤1：定义父图 State Schema ===")


class ParentState(TypedDict):
    """
    父图状态：文档处理系统

    关键点：
    - raw_text 和 keywords 与子图同名 → 自动传递
    - summary 是父图独有字段 → 子图看不到
    """
    raw_text: str       # 与子图共享 → 自动传入子图
    keywords: list[str] # 与子图共享 → 子图处理后自动回传
    summary: str        # 父图独有 → 子图不可见


print("  ParentState 定义完成")
print("  - 共享 key: raw_text（传入子图）, keywords（子图回传）")
print("  - 独有 key: summary（仅父图使用）")


# ===== 3. 定义父图节点 =====
print("\n=== 步骤2：定义父图节点 ===")


def receive_document(state: ParentState) -> dict:
    """
    父图节点1：接收文档
    准备 raw_text，后续自动传入子图
    """
    doc = state["raw_text"]
    print(f"  [父图-接收] 文档长度: {len(doc)} 字符")
    return {"raw_text": doc}


def generate_summary(state: ParentState) -> dict:
    """
    父图节点3：根据关键词生成摘要
    此时 keywords 已经由子图处理完毕并自动回传
    """
    keywords = state["keywords"]
    print(f"  [父图-摘要] 收到关键词: {keywords}")

    # 模拟摘要生成（不依赖外部 API）
    summary = f"本文档的核心主题围绕 {', '.join(keywords[:3])} 展开，" \
              f"共涉及 {len(keywords)} 个关键概念。"

    print(f"  [父图-摘要] 生成完成")
    return {"summary": summary}


# ===== 4. 构建父图（共享状态键方式）=====
print("\n=== 步骤3：构建父图（共享状态键方式）===")

parent_builder = StateGraph(ParentState)

# 添加节点
parent_builder.add_node("receive_document", receive_document)
# 关键：直接将编译后的子图作为节点添加
parent_builder.add_node("text_process", text_process_subgraph)
parent_builder.add_node("generate_summary", generate_summary)

# 添加边
parent_builder.add_edge(START, "receive_document")
parent_builder.add_edge("receive_document", "text_process")   # → 子图
parent_builder.add_edge("text_process", "generate_summary")   # 子图 → 摘要
parent_builder.add_edge("generate_summary", END)

# 编译父图
parent_graph = parent_builder.compile()

print("  父图结构:")
print("  START -> receive_document -> text_process(子图) -> generate_summary -> END")
print("  编译完成")


# ===== 5. 执行父图 =====
print("\n=== 步骤4：执行父图 ===\n")

document = (
    "Retrieval Augmented Generation (RAG) is a technique that combines "
    "information retrieval with text generation. RAG systems first retrieve "
    "relevant documents from a knowledge base, then use the retrieved context "
    "to generate accurate and grounded responses. The retrieval component "
    "typically uses vector similarity search with embeddings, while the "
    "generation component leverages large language models. RAG is particularly "
    "useful for building question answering systems, chatbots, and knowledge "
    "management applications."
)

initial_state = {
    "raw_text": document,
    "keywords": [],
    "summary": "",
}

print(f"输入文档: '{document[:80]}...'\n")

result = parent_graph.invoke(initial_state)

print(f"\n{'='*40}")
print(f"最终结果:")
print(f"  - 关键词: {result['keywords']}")
print(f"  - 摘要: {result['summary']}")

print("\n" + "=" * 60)
print("Part 2 完成：共享状态键集成成功")
print("=" * 60)
```

### Part 3：方式2 - 状态转换集成

```python
"""
基础子图创建与调用 - Part 3：状态转换集成
演示：当父子图状态结构不同时，用包装函数手动转换状态

核心原理：
- 父图和子图的 State Schema 完全不同
- 通过包装函数在调用前后手动转换状态
- 更灵活，适合状态结构差异大的场景

来源：
- Context7 官方文档：状态转换方式
- 源码分析：add_node 接受普通函数
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
import re


# ===== 1. 子图定义（与 Part 1 相同）=====
print("=" * 60)
print("Part 3：状态转换集成")
print("=" * 60)


class TextProcessState(TypedDict):
    raw_text: str
    cleaned_text: str
    tokens: list[str]
    keywords: list[str]


def clean_text(state: TextProcessState) -> dict:
    raw = state["raw_text"]
    cleaned = re.sub(r'\s+', ' ', raw).strip()
    cleaned = re.sub(r'[^\w\s\u4e00-\u9fff]', '', cleaned)
    cleaned = cleaned.lower()
    print(f"  [子图-清洗] 完成")
    return {"cleaned_text": cleaned}


def tokenize(state: TextProcessState) -> dict:
    text = state["cleaned_text"]
    stop_words = {"the", "a", "an", "is", "are", "was", "in", "on", "at",
                  "to", "for", "of", "with", "and", "or", "but", "not",
                  "this", "that", "it", "be", "as", "by", "from", "has", "have"}
    words = text.split()
    tokens = [w for w in words if w not in stop_words and len(w) > 2]
    print(f"  [子图-分词] 完成: {len(tokens)} 个词")
    return {"tokens": tokens}


def extract_keywords(state: TextProcessState) -> dict:
    tokens = state["tokens"]
    freq: dict[str, int] = {}
    for token in tokens:
        freq[token] = freq.get(token, 0) + 1
    sorted_words = sorted(freq.items(), key=lambda x: x[1], reverse=True)
    keywords = [word for word, count in sorted_words[:5]]
    print(f"  [子图-关键词] 完成: {keywords}")
    return {"keywords": keywords}


subgraph_builder = StateGraph(TextProcessState)
subgraph_builder.add_node("clean_text", clean_text)
subgraph_builder.add_node("tokenize", tokenize)
subgraph_builder.add_node("extract_keywords", extract_keywords)
subgraph_builder.add_edge(START, "clean_text")
subgraph_builder.add_edge("clean_text", "tokenize")
subgraph_builder.add_edge("tokenize", "extract_keywords")
subgraph_builder.add_edge("extract_keywords", END)
text_process_subgraph = subgraph_builder.compile()


# ===== 2. 定义完全不同的父图 State =====
print("\n=== 步骤1：定义父图 State（与子图完全不同）===")


class DocumentState(TypedDict):
    """
    父图状态：注意 key 名称与子图完全不同

    - document_content: 原始内容（子图中叫 raw_text）
    - extracted_tags: 提取的标签（子图中叫 keywords）
    - final_summary: 最终摘要（子图没有）
    - doc_metadata: 文档元信息（子图没有）
    """
    document_content: str     # 对应子图的 raw_text
    extracted_tags: list[str] # 对应子图的 keywords
    final_summary: str
    doc_metadata: dict


print("  DocumentState 定义完成")
print("  - document_content → 子图 raw_text（需要转换）")
print("  - extracted_tags → 子图 keywords（需要转换）")
print("  - final_summary, doc_metadata：父图独有")


# ===== 3. 定义包装函数（状态转换的核心）=====
print("\n=== 步骤2：定义状态转换包装函数 ===")


def call_text_process_subgraph(state: DocumentState) -> dict:
    """
    包装函数：手动进行父图 ↔ 子图的状态转换

    工作流程：
    1. 从父图状态提取数据，构造子图输入
    2. 调用子图 invoke
    3. 从子图输出提取数据，构造父图更新

    类比：
    - 前端：API 适配器，将外部数据转为内部格式
    - 日常：翻译官，在两种语言之间转换
    """
    print(f"  [包装函数] 开始状态转换")

    # 步骤1：父图状态 → 子图输入
    subgraph_input = {
        "raw_text": state["document_content"],  # 字段名映射
        "cleaned_text": "",
        "tokens": [],
        "keywords": [],
    }
    print(f"  [包装函数] 父图 document_content → 子图 raw_text")

    # 步骤2：调用子图
    subgraph_output = text_process_subgraph.invoke(subgraph_input)
    print(f"  [包装函数] 子图执行完成")

    # 步骤3：子图输出 → 父图状态更新
    parent_update = {
        "extracted_tags": subgraph_output["keywords"],  # 字段名映射
    }
    print(f"  [包装函数] 子图 keywords → 父图 extracted_tags")

    return parent_update


print("  call_text_process_subgraph 定义完成")
print("  转换映射:")
print("    document_content → raw_text（传入）")
print("    keywords → extracted_tags（传回）")


# ===== 4. 定义父图其他节点 =====
print("\n=== 步骤3：定义父图其他节点 ===")


def prepare_document(state: DocumentState) -> dict:
    """父图节点1：准备文档，添加元信息"""
    content = state["document_content"]
    metadata = {
        "length": len(content),
        "word_count": len(content.split()),
        "source": "user_input",
    }
    print(f"  [父图-准备] 文档 {metadata['word_count']} 词")
    return {"doc_metadata": metadata}


def create_summary(state: DocumentState) -> dict:
    """父图节点3：生成摘要"""
    tags = state["extracted_tags"]
    metadata = state["doc_metadata"]
    summary = (
        f"文档概要（{metadata['word_count']} 词）："
        f"核心主题为 {', '.join(tags[:3])}，"
        f"共提取 {len(tags)} 个关键标签。"
    )
    print(f"  [父图-摘要] 生成完成")
    return {"final_summary": summary}


# ===== 5. 构建父图（状态转换方式）=====
print("\n=== 步骤4：构建父图（状态转换方式）===")

parent_builder = StateGraph(DocumentState)

parent_builder.add_node("prepare_document", prepare_document)
# 关键：使用包装函数而非直接添加子图
parent_builder.add_node("process_text", call_text_process_subgraph)
parent_builder.add_node("create_summary", create_summary)

parent_builder.add_edge(START, "prepare_document")
parent_builder.add_edge("prepare_document", "process_text")
parent_builder.add_edge("process_text", "create_summary")
parent_builder.add_edge("create_summary", END)

parent_graph = parent_builder.compile()

print("  父图结构:")
print("  START -> prepare_document -> process_text(包装函数) -> create_summary -> END")
print("  编译完成")


# ===== 6. 执行父图 =====
print("\n=== 步骤5：执行父图 ===\n")

document = (
    "Retrieval Augmented Generation (RAG) is a technique that combines "
    "information retrieval with text generation. RAG systems first retrieve "
    "relevant documents from a knowledge base, then use the retrieved context "
    "to generate accurate and grounded responses. The retrieval component "
    "typically uses vector similarity search with embeddings, while the "
    "generation component leverages large language models. RAG is particularly "
    "useful for building question answering systems, chatbots, and knowledge "
    "management applications."
)

initial_state = {
    "document_content": document,
    "extracted_tags": [],
    "final_summary": "",
    "doc_metadata": {},
}

print(f"输入文档: '{document[:80]}...'\n")

result = parent_graph.invoke(initial_state)

print(f"\n{'='*40}")
print(f"最终结果:")
print(f"  - 标签: {result['extracted_tags']}")
print(f"  - 摘要: {result['final_summary']}")
print(f"  - 元信息: {result['doc_metadata']}")

print("\n" + "=" * 60)
print("Part 3 完成：状态转换集成成功")
print("=" * 60)
```

  ParentState 定义完成

## 核心知识点解析

### 1. 两种集成方式对比

| 对比维度 | 共享状态键 | 状态转换 |
|----------|-----------|---------|
| **代码量** | 少（自动传递） | 多（需写包装函数） |
| **灵活性** | 低（必须同名 key） | 高（任意映射） |
| **耦合度** | 较高（父子图 key 绑定） | 低（通过包装函数解耦） |
| **适用场景** | 父子图结构相似 | 父子图结构差异大 |
| **子图复用** | 需要父图配合命名 | 任意父图均可复用 |
| **调试难度** | 低（直观） | 中（需检查转换逻辑） |

**来源：Context7 官方文档、源码分析 state.py**

### 2. 共享状态键的工作原理

```
父图 State                子图 State
┌──────────────────┐      ┌──────────────────┐
│ raw_text: "..."  │─────>│ raw_text: "..."   │ 同名 key 自动传入
│ keywords: []     │      │ keywords: [...]   │
│ summary: ""      │      │ cleaned_text: ""  │ 私有 key 不回传
└──────────────────┘      │ tokens: []        │
         ^                └──────────────────┘
         │                         │
         └─── keywords 自动回传 ────┘
```

**关键规则**：
- 同名 key 自动双向传递
- 子图私有 key 留在子图内部
- 父图独有 key 子图不可见

### 3. 状态转换的工作原理

```
父图 State                      包装函数                    子图 State
┌─────────────────┐    ┌─────────────────────┐    ┌──────────────────┐
│ document_content│───>│ 提取 + 重命名         │───>│ raw_text          │
│ extracted_tags  │    │ document_content      │    │ keywords          │
│ final_summary   │    │   → raw_text         │    │ cleaned_text      │
│ doc_metadata    │    │                      │    │ tokens            │
└─────────────────┘    │ keywords             │    └──────────────────┘
         ^             │   → extracted_tags   │             │
         │             └─────────────────────┘             │
         └──────────────── 手动映射回传 ────────────────────┘
```

### 4. 如何选择集成方式

**选择共享状态键**：
- 父子图有明确的共享数据
- 子图专为当前父图设计
- 追求简洁代码

**选择状态转换**：
- 子图需要在多个不同父图中复用
- 父子图的状态结构差异大
- 需要在传递过程中做数据加工

**来源：Context7 官方文档**

## 常见问题

### Q1: 子图的私有 key 需要在父图中定义吗？

**答案**：不需要。子图私有 key（如 `cleaned_text`、`tokens`）只在子图内部使用，父图完全不知道它们的存在。

### Q2: 共享状态键方式中，子图能修改父图的独有 key 吗？

**答案**：不能。子图只能看到和修改与自己同名的 key。父图的 `summary` 对子图不可见。

### Q3: 状态转换方式中，子图知道自己被包装了吗？

**答案**：不知道。子图通过 `invoke()` 被调用，它只看到标准的输入和输出，不知道外面有包装函数。这是良好的封装。

### Q4: 两种方式可以混合使用吗？

**答案**：可以。同一个父图中，某些子图用共享状态键，某些用状态转换，完全兼容。

## 最佳实践

### 1. 子图设计原则：高内聚低耦合

```python
# 好的设计：子图有清晰的输入输出接口
class SubgraphState(TypedDict):
    input_data: str     # 明确的输入
    internal_cache: str # 内部使用
    output_result: str  # 明确的输出
```

### 2. 优先使用共享状态键

如果父子图有自然共享的数据，优先使用共享状态键方式，代码更简洁。

### 3. 包装函数要做好错误处理

```python
def call_subgraph(state: ParentState) -> dict:
    try:
        result = subgraph.invoke({...})
        return {"output": result["key"]}
    except Exception as e:
        print(f"子图执行失败: {e}")
        return {"output": "fallback_value"}
```

### 4. 给子图添加 input_schema 和 output_schema

```python
# 明确子图的输入输出接口
subgraph = StateGraph(
    InternalState,
    input_schema=SubgraphInput,
    output_schema=SubgraphOutput,
)
```
