# 核心概念 2：文档内容转换器

## 一句话定义

**文档内容转换器是 LangChain 中专门处理文档内容格式的 DocumentTransformer，以 BeautifulSoupTransformer 和 Html2TextTransformer 为代表，将 HTML 等富文本转换为干净的纯文本，是 RAG 管道中网页数据清洗的核心工具。**

[来源: sourcecode/langchain + reference/context7_langchain_01.md]

---

## 为什么 HTML 清洗在 RAG 中至关重要？

### 问题：网页抓取的原始内容充满噪音

当你用 `WebBaseLoader` 抓取网页时，得到的是原始 HTML：

```html
<html>
<head><script>analytics.track('page_view')</script></head>
<body>
  <nav>Home | About | Contact</nav>
  <div class="content">
    <h1>什么是 RAG？</h1>
    <p>RAG（检索增强生成）是一种结合信息检索与文本生成的技术。</p>
    <p>它通过从知识库中检索相关文档来增强 LLM 的回答质量。</p>
  </div>
  <footer>Copyright 2026</footer>
  <script>console.log('loaded')</script>
  <style>.content { font-size: 14px; }</style>
</body>
</html>
```

**如果直接把这些内容送入 Embedding 模型：**
- `<script>` 和 `<style>` 标签会污染语义
- 导航栏、页脚等无关内容会干扰检索
- HTML 标签本身占用 Token 但不提供语义价值
- 向量化后的结果质量大幅下降

### 解决方案：用 DocumentTransformer 清洗内容

```python
# 清洗前：充满 HTML 标签和噪音
raw_content = "<h1>什么是 RAG？</h1><p>RAG 是...</p><script>...</script>"

# 清洗后：干净的纯文本
clean_content = "什么是 RAG？\nRAG 是..."
```

**LangChain 提供了两个主要的 HTML 清洗转换器：**
1. **BeautifulSoupTransformer** — 精细控制，提取特定标签
2. **Html2TextTransformer** — 简单直接，HTML 转 Markdown/纯文本

[来源: reference/search_document_transformer_01.md | 2026年最佳实践]

---

## BeautifulSoupTransformer 详解

### 是什么？

`BeautifulSoupTransformer` 基于 Python 的 `beautifulsoup4` 库，可以从 HTML 文档中**精确提取指定标签的内容**，同时自动移除 `<script>`、`<style>` 等噪音元素。

**核心能力：**
- 提取特定 HTML 标签（如 `<p>`、`<h1>`、`<article>`）
- 自动移除脚本和样式标签
- 可配置要移除的标签列表
- 保留文本结构（标签之间用换行分隔）

### 安装依赖

```bash
pip install beautifulsoup4 lxml
```

### 基础用法

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import BeautifulSoupTransformer

# 模拟从网页加载的 HTML 文档
html_docs = [
    Document(
        page_content="""
        <html>
        <head><title>RAG 教程</title></head>
        <body>
            <nav>首页 | 教程 | 关于</nav>
            <h1>什么是 RAG？</h1>
            <p>RAG（Retrieval-Augmented Generation）是一种结合检索与生成的技术。</p>
            <p>它通过从知识库中检索相关文档来增强 LLM 的回答质量。</p>
            <div class="sidebar">相关推荐：LangChain 入门</div>
            <script>console.log('tracking');</script>
            <style>.sidebar { color: gray; }</style>
            <footer>Copyright 2026</footer>
        </body>
        </html>
        """,
        metadata={"source": "https://example.com/rag-tutorial"}
    )
]

# 创建转换器
bs_transformer = BeautifulSoupTransformer()

# 只提取 <p> 和 <h1> 标签的内容
cleaned_docs = bs_transformer.transform_documents(
    html_docs,
    tags_to_extract=["p", "h1"]
)

print(cleaned_docs[0].page_content)
```

**运行输出：**
```
什么是 RAG？ RAG（Retrieval-Augmented Generation）是一种结合检索与生成的技术。 它通过从知识库中检索相关文档来增强 LLM 的回答质量。
```

**注意：** 导航栏、侧边栏、脚注、脚本、样式全部被移除，只保留了 `<h1>` 和 `<p>` 中的文本。

[来源: reference/context7_langchain_01.md | Context7 官方文档]

---

### 配置参数

```python
# 默认行为：提取所有文本内容
bs_transformer = BeautifulSoupTransformer()
result = bs_transformer.transform_documents(docs)

# 指定要提取的标签
result = bs_transformer.transform_documents(
    docs,
    tags_to_extract=["p", "h1", "h2", "h3", "li", "article"]
)

# 指定要移除的标签（默认移除 script 和 style）
result = bs_transformer.transform_documents(
    docs,
    unwanted_tags=["script", "style", "nav", "footer", "aside"]
)
```

**常用标签提取策略：**

| 场景 | 推荐提取的标签 | 原因 |
|------|---------------|------|
| 博客文章 | `["p", "h1", "h2", "h3", "li"]` | 正文段落和标题 |
| 技术文档 | `["p", "h1", "h2", "h3", "code", "pre", "li"]` | 包含代码块 |
| 新闻页面 | `["p", "h1", "article"]` | 文章主体 |
| 产品页面 | `["p", "h1", "h2", "span", "div.description"]` | 产品描述 |

### 与 WebBaseLoader 配合使用

```python
from langchain_community.document_loaders import WebBaseLoader
from langchain_community.document_transformers import BeautifulSoupTransformer

# 第 1 步：加载网页
loader = WebBaseLoader(["https://example.com/article1", "https://example.com/article2"])
raw_docs = loader.load()
print(f"加载了 {len(raw_docs)} 个原始文档")

# 第 2 步：用 BeautifulSoup 清洗
bs_transformer = BeautifulSoupTransformer()
cleaned_docs = bs_transformer.transform_documents(
    raw_docs,
    tags_to_extract=["p", "h1", "h2", "h3"]
)

# 第 3 步：查看清洗效果
for doc in cleaned_docs:
    print(f"来源: {doc.metadata['source']}")
    print(f"清洗前长度: {len(raw_docs[0].page_content)}")
    print(f"清洗后长度: {len(doc.page_content)}")
    print(f"内容预览: {doc.page_content[:200]}...")
    print()
```

## Html2TextTransformer 详解

### 是什么？

`Html2TextTransformer` 基于 `html2text` 库，将 HTML 文档转换为**格式化的 Markdown 文本**或纯文本。相比 BeautifulSoup，它更简单直接——不需要指定标签，直接把整个 HTML 转成可读文本。

**核心能力：**
- HTML → Markdown 格式转换
- 自动处理链接、图片、列表等 HTML 元素
- 保留文档结构（标题层级、列表缩进）
- 配置简单，开箱即用

### 安装依赖

```bash
pip install html2text
```

### 基础用法

```python
from langchain_core.documents import Document
from langchain_community.document_transformers import Html2TextTransformer

# 模拟 HTML 文档
html_docs = [
    Document(
        page_content="""
        <html>
        <body>
            <h1>LangChain 快速入门</h1>
            <p>LangChain 是一个用于开发 <strong>LLM 应用</strong>的框架。</p>
            <h2>核心组件</h2>
            <ul>
                <li>Model I/O：模型输入输出</li>
                <li>Retrieval：检索模块</li>
                <li>Agents：智能代理</li>
            </ul>
            <p>更多信息请访问 <a href="https://langchain.com">官网</a>。</p>
            <script>alert('hello');</script>
        </body>
        </html>
        """,
        metadata={"source": "https://example.com/langchain-intro"}
    )
]

# 创建转换器
html2text_transformer = Html2TextTransformer()

# 执行转换
text_docs = html2text_transformer.transform_documents(html_docs)

print(text_docs[0].page_content)
```

**运行输出：**
```
# LangChain 快速入门

LangChain 是一个用于开发 **LLM 应用**的框架。

## 核心组件

  * Model I/O：模型输入输出
  * Retrieval：检索模块
  * Agents：智能代理

更多信息请访问 [官网](https://langchain.com)。
```

**注意：** HTML 被转换成了格式良好的 Markdown，标题、加粗、列表、链接都被保留了，而 `<script>` 标签被自动移除。

[来源: reference/context7_langchain_01.md | Context7 官方文档]

---

### 配置选项

`Html2TextTransformer` 底层使用 `html2text.HTML2Text`，支持多种配置：

```python
from langchain_community.document_transformers import Html2TextTransformer

# 默认配置：转换为 Markdown
transformer = Html2TextTransformer()

# 忽略链接（不保留 href）
transformer = Html2TextTransformer(ignore_links=True)

# 忽略图片
transformer = Html2TextTransformer(ignore_images=True)

# 同时忽略链接和图片（最干净的纯文本）
transformer = Html2TextTransformer(
    ignore_links=True,
    ignore_images=True
)
```

**配置对比：**

| 配置 | 输入 HTML | 输出文本 |
|------|----------|---------|
| 默认 | `<a href="url">链接</a>` | `[链接](url)` |
| `ignore_links=True` | `<a href="url">链接</a>` | `链接` |
| 默认 | `<img src="img.png" alt="图片">` | `![图片](img.png)` |
| `ignore_images=True` | `<img src="img.png" alt="图片">` | （被移除） |

## BeautifulSoup vs Html2Text：如何选择？

### 对比表

| 特性 | BeautifulSoupTransformer | Html2TextTransformer |
|------|------------------------|---------------------|
| **底层库** | beautifulsoup4 | html2text |
| **输出格式** | 纯文本（标签内容拼接） | Markdown 格式文本 |
| **精细控制** | 高（可指定提取/移除的标签） | 低（整体转换） |
| **配置复杂度** | 中等 | 简单 |
| **保留结构** | 部分（取决于提取的标签） | 好（标题、列表、链接） |
| **适用场景** | 需要精确提取特定内容 | 需要保留文档结构 |
| **性能** | 较快 | 较快 |

### 选择指南

**选 BeautifulSoupTransformer 的场景：**
- 网页结构复杂，只需要特定区域的内容（如文章正文）
- 需要移除特定的噪音元素（如广告、侧边栏）
- 对提取内容有精确控制需求

```python
# 场景：只要文章正文，不要导航和侧边栏
bs_transformer = BeautifulSoupTransformer()
result = bs_transformer.transform_documents(
    docs,
    tags_to_extract=["article", "p", "h1", "h2"],
    unwanted_tags=["nav", "aside", "footer"]
)
```

**选 Html2TextTransformer 的场景：**
- 需要保留文档的层级结构（标题、列表）
- 输出需要人类可读（Markdown 格式）
- 快速清洗，不需要精细控制
- 后续还要做 Markdown 分块（如 `MarkdownHeaderTextSplitter`）

```python
# 场景：快速清洗，保留结构，后续用 Markdown 分块
html2text = Html2TextTransformer()
markdown_docs = html2text.transform_documents(docs)

# 然后用 Markdown 分块器按标题分块
from langchain_text_splitters import MarkdownHeaderTextSplitter
md_splitter = MarkdownHeaderTextSplitter(
    headers_to_split_on=[("#", "h1"), ("##", "h2")]
)
```

## 实战：完整的网页清洗管道

```python
"""
网页文档清洗实战
场景：从网页抓取内容 → 清洗 HTML → 分块 → 准备存入向量库
"""

from langchain_core.documents import Document
from langchain_community.document_transformers import (
    BeautifulSoupTransformer,
    Html2TextTransformer,
)
from langchain_text_splitters import RecursiveCharacterTextSplitter


# ===== 模拟从网页加载的 HTML 文档 =====
html_docs = [
    Document(
        page_content="""
        <html>
        <body>
            <nav><a href="/">首页</a> | <a href="/docs">文档</a></nav>
            <main>
                <h1>RAG 系统架构指南</h1>
                <p>RAG（Retrieval-Augmented Generation）是当前最流行的 LLM 应用架构之一。
                本文将详细介绍 RAG 系统的核心组件和最佳实践。</p>

                <h2>1. 文档加载</h2>
                <p>文档加载是 RAG 管道的第一步。支持 PDF、Word、HTML、Markdown 等多种格式。
                推荐使用 LangChain 的 DocumentLoader 体系。</p>

                <h2>2. 文本分块</h2>
                <p>将长文档切分为适合 Embedding 模型处理的小段。
                推荐使用 RecursiveCharacterTextSplitter，chunk_size 设为 400-512 token。</p>

                <h2>3. 向量存储</h2>
                <p>将文本块转换为向量并存储到向量数据库中。
                常用的向量数据库包括 Chroma、Milvus、Pinecone 等。</p>
            </main>
            <aside>
                <h3>相关文章</h3>
                <ul><li>LangChain 入门</li><li>Milvus 教程</li></ul>
            </aside>
            <footer>Copyright 2026 RAG Academy</footer>
            <script>trackPageView();</script>
            <style>body { font-family: sans-serif; }</style>
        </body>
        </html>
        """,
        metadata={"source": "https://example.com/rag-guide", "title": "RAG 架构指南"}
    )
]


# ===== 方案 A：BeautifulSoup 精确提取 =====
print("=" * 50)
print("方案 A：BeautifulSoup 精确提取")
print("=" * 50)

bs_transformer = BeautifulSoupTransformer()
bs_result = bs_transformer.transform_documents(
    html_docs,
    tags_to_extract=["h1", "h2", "p"]
)

print(f"清洗后内容长度: {len(bs_result[0].page_content)} 字符")
print(f"内容预览:\n{bs_result[0].page_content[:300]}")
print()


# ===== 方案 B：Html2Text 保留结构 =====
print("=" * 50)
print("方案 B：Html2Text 保留 Markdown 结构")
print("=" * 50)

h2t_transformer = Html2TextTransformer()
h2t_result = h2t_transformer.transform_documents(html_docs)

print(f"清洗后内容长度: {len(h2t_result[0].page_content)} 字符")
print(f"内容预览:\n{h2t_result[0].page_content[:300]}")
print()


# ===== 后续步骤：分块 =====
print("=" * 50)
print("后续步骤：文本分块")
print("=" * 50)

splitter = RecursiveCharacterTextSplitter(
    chunk_size=200,
    chunk_overlap=20,
)

# 对方案 A 的结果分块
chunks_a = splitter.transform_documents(bs_result)
print(f"方案 A 分块结果: {len(chunks_a)} 个片段")

# 对方案 B 的结果分块

# 查看分块内容
```

## 双重类比

### 类比 1：HTML 清洗 = 前端的 DOMPurify

**前端类比：**

前端开发中，为了防止 XSS 攻击，会用 DOMPurify 清洗用户输入的 HTML：

```javascript
// 前端：清洗不安全的 HTML
import DOMPurify from 'dompurify';
const cleanHtml = DOMPurify.sanitize(dirtyHtml);
// 移除 <script>、事件处理器等危险内容
```

**LangChain 对应：**

```python
# RAG：清洗噪音 HTML
from langchain_community.document_transformers import BeautifulSoupTransformer
bs = BeautifulSoupTransformer()
clean_docs = bs.transform_documents(html_docs, tags_to_extract=["p", "h1"])
# 移除 <script>、<style>、导航栏等噪音内容
```

**相似点：** 都是从 HTML 中移除不需要的内容，只保留有价值的部分。区别在于前端关注安全，RAG 关注语义质量。

---

### 类比 2：BeautifulSoup vs Html2Text = querySelector vs innerText

**前端类比：**

```javascript
// querySelector：精确选择特定元素
const paragraphs = document.querySelectorAll('p, h1, h2');
const text = Array.from(paragraphs).map(el => el.textContent).join('\n');

// innerText：获取整个元素的可见文本
const text = document.body.innerText;
```

**LangChain 对应：**

```python
# BeautifulSoup：精确提取特定标签（类似 querySelector）
bs.transform_documents(docs, tags_to_extract=["p", "h1", "h2"])

# Html2Text：转换整个文档为文本（类似 innerText）
h2t.transform_documents(docs)
```

**日常生活类比：**
- **BeautifulSoup** 像用镊子挑选——精确但需要你知道要什么
- **Html2Text** 像用筛子过滤——简单粗暴但效果不错

## 在 RAG 开发中的应用

### 典型场景：知识库网页抓取管道

```
网页 URL → WebBaseLoader → BeautifulSoup/Html2Text → TextSplitter → Embedding → VectorStore
                ↓                    ↓                      ↓
           原始 HTML            干净文本               文本片段
```

### 2026 年最佳实践

根据最新的 RAG 管道实践，推荐的文档转换流程：

1. **加载**：`WebBaseLoader` 抓取网页
2. **清洗**：`Html2TextTransformer`（通用场景）或 `BeautifulSoupTransformer`（精确控制）
3. **分块**：`RecursiveCharacterTextSplitter`（400-512 token + 10-20% 重叠）
4. **过滤**：`EmbeddingsRedundantFilter`（去除重复片段）
5. **嵌入**：`OpenAIEmbeddings` 或 `sentence-transformers`

## 总结

### 两个核心转换器

| 转换器 | 底层库 | 输出格式 | 适用场景 |
|--------|--------|---------|---------|
| BeautifulSoupTransformer | beautifulsoup4 | 纯文本 | 精确提取特定标签 |
| Html2TextTransformer | html2text | Markdown | 快速清洗保留结构 |

### 选择建议

- **需要精确控制** → BeautifulSoupTransformer
- **快速清洗 + 保留结构** → Html2TextTransformer
- **不确定** → 先用 Html2TextTransformer，不满意再换 BeautifulSoup

### 关键要点

1. HTML 清洗是 RAG 网页数据处理的必要步骤
2. 两个转换器都实现了 `BaseDocumentTransformer` 接口
3. 都通过 `transform_documents()` 调用，可以无缝接入管道
4. 清洗后的文档可以直接传给 TextSplitter 分块
