## 误区1：函数形式的 Partial Variables 在模板创建时就执行了 ❌

### 错误观点

很多开发者认为，当你使用函数作为 `partial_variables` 的值时，函数会在**模板创建时**立即执行并缓存结果。

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 错误理解：认为 get_date 会在这里执行一次，然后缓存结果
prompt = PromptTemplate(
    template="Date: {date}, Query: {query}",
    input_variables=["query"],
    partial_variables={
        "date": lambda: datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    }
)

# 错误理解：认为两次调用会得到相同的时间
result1 = prompt.format(query="First")
result2 = prompt.format(query="Second")
```

### 为什么错？

**函数形式的 Partial Variables 是在每次 `format()` 或 `invoke()` 时才执行的，不是在模板创建时执行。**

这是一种**延迟计算（Lazy Evaluation）**机制，确保每次调用都能获取最新的动态值。

[来源: reference/search_partial_01.md:86-93]

### 为什么人们容易这样错？

**心理原因：直觉上的"初始化思维"**

在大多数编程场景中，我们习惯了"初始化时设置值"的模式：

```python
# 常见的初始化模式
class Config:
    def __init__(self):
        self.created_at = datetime.now()  # 在初始化时执行一次

config = Config()
# 之后 config.created_at 永远是创建时的时间
```

这种思维定式让我们自然地认为 `partial_variables` 中的函数也会在模板创建时执行。

**认知偏差：函数 vs 函数调用**

```python
# 这是函数对象（不执行）
partial_variables={"date": get_date}

# 这是函数调用（立即执行）
partial_variables={"date": get_date()}
```

很多人混淆了"传递函数对象"和"传递函数调用结果"的区别。

### 正确理解

**函数形式的 Partial Variables 是延迟计算的，每次调用都会重新执行。**

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime
import time

# ===== 正确理解：函数在每次 format 时执行 =====

prompt = PromptTemplate(
    template="Timestamp: {timestamp}, Query: {query}",
    input_variables=["query"],
    partial_variables={
        "timestamp": lambda: datetime.now().strftime("%Y-%m-%d %H:%M:%S.%f")[:-3]
    }
)

# 第一次调用
result1 = prompt.format(query="First query")
print(result1)
# 输出: Timestamp: 2026-02-26 10:30:15.123, Query: First query

# 等待 2 秒
time.sleep(2)

# 第二次调用（时间会更新！）
result2 = prompt.format(query="Second query")
print(result2)
# 输出: Timestamp: 2026-02-26 10:30:17.456, Query: Second query
#       ↑ 注意时间变化了！

# ===== 对比：字符串形式的 Partial Variables（固定值）=====

prompt_fixed = PromptTemplate(
    template="Timestamp: {timestamp}, Query: {query}",
    input_variables=["query"],
    partial_variables={
        "timestamp": datetime.now().strftime("%Y-%m-%d %H:%M:%S")  # 立即执行
    }
)

result3 = prompt_fixed.format(query="First")
time.sleep(2)
result4 = prompt_fixed.format(query="Second")
# 两次调用的时间戳相同（因为在模板创建时就固定了）
```

**实际应用场景：**

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime
import random

# RAG 系统的查询日志模板
log_template = PromptTemplate(
    template="""
=== Query Log ===
Timestamp: {timestamp}
Request ID: {request_id}
User: {user}
Query: {query}
==================
""",
    input_variables=["user", "query"],
    partial_variables={
        # 每次查询都生成新的时间戳和请求 ID
        "timestamp": lambda: datetime.now().isoformat(),
        "request_id": lambda: f"req_{random.randint(10000, 99999)}"
    }
)

# 第一次查询
print(log_template.format(user="Alice", query="What is RAG?"))
# 输出:
# === Query Log ===
# Timestamp: 2026-02-26T10:30:15.123456
# Request ID: req_45678
# User: Alice
# Query: What is RAG?
# ==================

# 第二次查询（时间戳和请求 ID 都会更新）
print(log_template.format(user="Bob", query="How to use LangChain?"))
# 输出:
# === Query Log ===
# Timestamp: 2026-02-26T10:30:16.789012  ← 时间变了
# Request ID: req_92341                   ← ID 变了
# User: Bob
# Query: How to use LangChain?
# ==================
```

## 误区2：使用 `+` 操作符组合模板总是能成功 ❌

### 错误观点

很多开发者认为，任意两个 `PromptTemplate` 都可以用 `+` 操作符组合，就像字符串拼接一样简单。

```python
from langchain_core.prompts import PromptTemplate

# 错误理解：认为任意模板都能组合
template1 = PromptTemplate.from_template("Hello {name}")
template2 = PromptTemplate.from_template("Goodbye {{name}}", template_format="mustache")

# 错误理解：认为这样可以工作
combined = template1 + template2  # 实际上会报错！
```

### 为什么错？

**模板组合有严格的限制条件：**

1. **两个模板的 `template_format` 必须一致**
2. **两个模板的 `partial_variables` 不能有相同的键**

如果违反这些规则，会抛出 `ValueError`。

[来源: reference/source_prompttemplate_01.md:28-59]

### 为什么人们容易这样错？

**心理原因：字符串拼接的类比误导**

在 Python 中，字符串拼接非常自由：

```python
# 字符串拼接总是成功
str1 = "Hello"
str2 = "World"
result = str1 + " " + str2  # 总是可以工作
```

这种经验让我们误以为 `PromptTemplate` 的 `+` 操作符也同样自由。

**认知偏差：忽略类型系统**

开发者往往关注"内容"（模板字符串），而忽略了"元数据"（`template_format`、`partial_variables`）的一致性要求。

### 正确理解

**模板组合需要满足严格的兼容性条件。**

```python
from langchain_core.prompts import PromptTemplate

# ===== 错误示例1：template_format 不一致 =====

template1 = PromptTemplate.from_template(
    "Hello {name}",
    template_format="f-string"
)

template2 = PromptTemplate.from_template(
    "Goodbye {{name}}",
    template_format="mustache"
)

try:
    combined = template1 + template2
except ValueError as e:
    print(f"错误: {e}")
    # 输出: 错误: Cannot add templates of different formats

# ===== 错误示例2：partial_variables 冲突 =====

template3 = PromptTemplate(
    template="System: {system_name}\n",
    input_variables=[],
    partial_variables={"system_name": "RAG Assistant"}
)

template4 = PromptTemplate(
    template="Version: {system_name} v2.0\n",
    input_variables=[],
    partial_variables={"system_name": "Different Name"}  # 冲突！
)

try:
    combined = template3 + template4
except ValueError as e:
    print(f"错误: {e}")
    # 输出: 错误: Cannot have same variable partialed twice.

# ===== 正确示例1：相同的 template_format =====

template5 = PromptTemplate.from_template(
    "System: {role}\n",
    template_format="f-string"
)

template6 = PromptTemplate.from_template(
    "User: {query}\n",
    template_format="f-string"  # 相同格式
)

combined_correct = template5 + template6
print(combined_correct.format(role="assistant", query="Hello"))
# 输出:
# System: assistant
# User: Hello

# ===== 正确示例2：不冲突的 partial_variables =====

template7 = PromptTemplate(
    template="Date: {date}\n",
    input_variables=[],
    partial_variables={"date": "2026-02-26"}
)

template8 = PromptTemplate(
    template="Version: {version}\n",
    input_variables=[],
    partial_variables={"version": "v1.0"}  # 不同的键，不冲突
)

combined_correct2 = template7 + template8
print(combined_correct2.format())
# 输出:
# Date: 2026-02-26
# Version: v1.0
```

**实际应用：安全的模板组合模式**

```python
from langchain_core.prompts import PromptTemplate

# ===== 模式1：统一使用默认格式（f-string）=====

def create_rag_prompt(include_context: bool = True):
    """根据需求动态组合 RAG 提示词"""

    # 基础模块（都使用 f-string）
    system_module = PromptTemplate.from_template(
        "System: You are a {role}.\n"
    )

    context_module = PromptTemplate.from_template(
        "Context: {context}\n"
    )

    query_module = PromptTemplate.from_template(
        "User: {query}\nAssistant:"
    )

    # 根据需求组合
    if include_context:
        return system_module + context_module + query_module
    else:
        return system_module + query_module

# 使用
prompt_with_context = create_rag_prompt(include_context=True)
prompt_without_context = create_rag_prompt(include_context=False)

# ===== 模式2：使用工厂函数确保一致性 =====

def create_template(text: str, format: str = "f-string"):
    """工厂函数，确保所有模板使用相同格式"""
    return PromptTemplate.from_template(text, template_format=format)

# 所有模板都使用相同格式
t1 = create_template("Part 1: {var1}\n")
t2 = create_template("Part 2: {var2}\n")
t3 = create_template("Part 3: {var3}\n")

# 安全组合
combined = t1 + t2 + t3
```

## 误区3：Mustache 模板和 f-string 模板功能相同，只是语法不同 ❌

### 错误观点

很多开发者认为，Mustache 模板（`{{var}}`）和 f-string 模板（`{var}`）只是语法上的差异，功能完全相同。

```python
from langchain_core.prompts import PromptTemplate

# 错误理解：认为这两个模板功能相同
f_string_template = PromptTemplate.from_template(
    "Hello {name}",
    template_format="f-string"
)

mustache_template = PromptTemplate.from_template(
    "Hello {{name}}",
    template_format="mustache"
)
```

### 为什么错？

**Mustache 模板和 f-string 模板有本质区别：**

1. **功能差异**：
   - f-string：简单变量替换
   - Mustache：支持嵌套变量、条件渲染（section）、循环等高级特性

2. **验证能力差异**：
   - f-string：支持模板验证（`validate_template=True`）
   - Mustache：**不支持模板验证**

3. **安全性差异**：
   - f-string：相对安全
   - Jinja2：功能强大但有安全风险（可能导致代码执行）

[来源: reference/source_prompttemplate_01.md:100-173]

### 为什么人们容易这样错？

**心理原因：表面相似性误导**

在简单场景下，两种模板看起来功能相同：

```python
# 简单变量替换时，两者看起来一样
f_string_template.format(name="Alice")  # "Hello Alice"
mustache_template.format(name="Alice")  # "Hello Alice"
```

这种表面相似性让人误以为它们完全等价。

**认知偏差：忽略边界情况**

开发者往往只测试简单场景，而忽略了复杂场景下的差异（如嵌套变量、条件渲染、模板验证）。

### 正确理解

**不同模板格式有不同的能力和限制，应根据场景选择。**

```python
from langchain_core.prompts import PromptTemplate

# ===== 差异1：嵌套变量支持 =====

# f-string：不支持嵌套变量
f_string_template = PromptTemplate.from_template(
    "User: {user.name}",  # 这样写会报错
    template_format="f-string"
)

# Mustache：支持嵌套变量
mustache_template = PromptTemplate.from_template(
    "User: {{user.name}}",  # 支持嵌套
    template_format="mustache"
)

result = mustache_template.format(user={"name": "Alice", "age": 30})
print(result)  # 输出: User: Alice

# ===== 差异2：条件渲染（Section）=====

# f-string：不支持条件渲染
# 只能通过 Python 代码实现

# Mustache：原生支持条件渲染
mustache_conditional = PromptTemplate.from_template(
    """
User: {{name}}
{{#is_premium}}
Premium Features: Enabled
Access Level: Gold
{{/is_premium}}
{{^is_premium}}
Premium Features: Disabled
Access Level: Free
{{/is_premium}}
""",
    template_format="mustache"
)

# 高级用户
result1 = mustache_conditional.format(name="Alice", is_premium=True)
print(result1)
# 输出:
# User: Alice
# Premium Features: Enabled
# Access Level: Gold

# 普通用户
result2 = mustache_conditional.format(name="Bob", is_premium=False)
print(result2)
# 输出:
# User: Bob
# Premium Features: Disabled
# Access Level: Free

# ===== 差异3：模板验证能力 =====

# f-string：支持验证
f_string_validated = PromptTemplate(
    template="Hello {name}, you are {age} years old",
    input_variables=["name", "age"],
    template_format="f-string",
    validate_template=True  # 可以验证
)

# Mustache：不支持验证
try:
    mustache_validated = PromptTemplate(
        template="Hello {{name}}, you are {{age}} years old",
        input_variables=["name", "age"],
        template_format="mustache",
        validate_template=True  # 会报错
    )
except ValueError as e:
    print(f"错误: {e}")
    # 输出: 错误: Mustache templates cannot be validated.

# ===== 差异4：Jinja2 的安全风险 =====

# Jinja2：功能强大但有安全风险
jinja_template = PromptTemplate.from_template(
    """
User: {{ name }}
{% if age >= 18 %}
Status: Adult
{% else %}
Status: Minor
{% endif %}
""",
    template_format="jinja2"
)

result = jinja_template.format(name="Alice", age=25)
print(result)
# 输出:
# User: Alice
# Status: Adult

# ⚠️ 安全警告：不要接受来自不可信源的 jinja2 模板
# 可能导致任意 Python 代码执行！
```

**选择指南：**

| 场景 | 推荐格式 | 原因 |
|------|---------|------|
| 简单变量替换 | f-string | 简单、快速、支持验证 |
| 需要嵌套变量 | Mustache | 支持 `{{obj.field}}` |
| 需要条件渲染 | Mustache 或 Jinja2 | 支持 section 或 if 语句 |
| 需要复杂逻辑 | Jinja2 | 功能最强大（但注意安全） |
| 需要模板验证 | f-string | Mustache 不支持验证 |
| 不可信输入 | f-string | 最安全 |

**实际应用：根据场景选择格式**

```python
from langchain_core.prompts import PromptTemplate

# ===== 场景1：简单 RAG（使用 f-string）=====

simple_rag = PromptTemplate.from_template(
    """
Context: {context}
Question: {question}
Answer:
""",
    template_format="f-string"
)

# ===== 场景2：多级权限系统（使用 Mustache）=====

permission_template = PromptTemplate.from_template(
    """
User: {{user.name}}
Role: {{user.role}}

{{#user.is_admin}}
Admin Panel: Accessible
System Settings: Editable
{{/user.is_admin}}

{{^user.is_admin}}
Admin Panel: Restricted
System Settings: Read-only
{{/user.is_admin}}
""",
    template_format="mustache"
)

result = permission_template.format(
    user={
        "name": "Alice",
        "role": "Admin",
        "is_admin": True
    }
)

# ===== 场景3：动态报告生成（使用 Jinja2）=====

report_template = PromptTemplate.from_template(
    """
# Report for {{ user_name }}

## Statistics
- Total Queries: {{ total_queries }}
- Success Rate: {{ success_rate }}%

## Status
{% if success_rate >= 90 %}
✅ Excellent performance!
{% elif success_rate >= 70 %}
⚠️ Good, but room for improvement.
{% else %}
❌ Needs attention.
{% endif %}
""",
    template_format="jinja2"
)

result = report_template.format(
    user_name="Alice",
    total_queries=100,
    success_rate=95
)
```

## 误区总结表

| 误区 | 错误观点 | 正确理解 | 关键点 |
|------|---------|---------|--------|
| **函数 Partial Variables** | 在模板创建时执行 | 在每次 format/invoke 时执行 | 延迟计算机制 |
| **模板组合** | 任意模板都能组合 | 需要 format 一致且 partial 不冲突 | 严格兼容性检查 |
| **模板格式** | 只是语法差异 | 功能、验证能力、安全性都不同 | 根据场景选择 |

## 避坑指南

### 1. 使用函数 Partial Variables 时

✅ **正确做法：**
```python
# 需要动态值时使用函数
partial_variables={"timestamp": lambda: datetime.now().isoformat()}
```

❌ **错误做法：**
```python
# 误以为函数只执行一次
partial_variables={"timestamp": datetime.now().isoformat()}  # 这是立即执行
```

### 2. 组合模板时

✅ **正确做法：**
```python
# 确保所有模板使用相同格式
t1 = PromptTemplate.from_template("...", template_format="f-string")
t2 = PromptTemplate.from_template("...", template_format="f-string")
combined = t1 + t2
```

❌ **错误做法：**
```python
# 混合不同格式
t1 = PromptTemplate.from_template("...", template_format="f-string")
t2 = PromptTemplate.from_template("...", template_format="mustache")
combined = t1 + t2  # 报错！
```

### 3. 选择模板格式时

✅ **正确做法：**
```python
# 根据需求选择格式
if need_nested_variables:
    format = "mustache"
elif need_complex_logic:
    format = "jinja2"
else:
    format = "f-string"  # 默认选择
```

❌ **错误做法：**
```python
# 盲目使用复杂格式
format = "jinja2"  # 简单场景不需要
```
