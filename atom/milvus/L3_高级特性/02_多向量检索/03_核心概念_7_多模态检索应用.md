# 核心概念 7：多模态检索应用

> 本文档详细讲解如何使用 Milvus 多向量检索实现图片 + 文本多模态检索，包括 CLIP 模型集成、多模态向量字段定义、检索与融合策略。

---

## 1. 什么是多模态检索？

### 1.1 定义

**多模态检索（Multimodal Search）** 是指在同一个检索系统中，同时处理和检索多种类型的数据（如文本、图片、音频、视频等），并将它们的检索结果融合在一起。

在 Milvus 中，多模态检索通过以下方式实现：
- 为不同模态的数据定义独立的向量字段
- 使用多模态模型（如 CLIP）生成统一语义空间的向量
- 通过多向量混合检索同时查询多个模态
- 使用 Ranker 融合多模态检索结果

### 1.2 核心优势

**1. 跨模态语义理解**
- 文本查询可以检索到相关图片
- 图片查询可以检索到相关文本
- 实现"以图搜文"、"以文搜图"、"图文混合搜索"

**2. 更丰富的检索体验**
- 用户可以用自然语言描述想要的图片
- 用户可以上传图片找到相关的文本描述
- 提升搜索的灵活性和准确性

**3. 统一的语义空间**
- 多模态模型（如 CLIP）将不同模态映射到同一向量空间
- 不同模态的数据可以直接计算相似度
- 简化了跨模态检索的实现

## 2. 多模态模型：CLIP

### 2.1 CLIP 模型简介

**CLIP（Contrastive Language-Image Pre-training）** 是 OpenAI 开发的多模态模型，能够理解图片和文本之间的语义关系。

**核心特点**：
- **统一向量空间**：将图片和文本映射到同一个向量空间
- **零样本学习**：无需微调即可处理新的图片-文本对
- **强大的泛化能力**：在大规模数据上预训练，适用于多种场景

**工作原理**：
```
文本 "a dog playing in the park" → Text Encoder → 512维向量
图片 [dog_image.jpg]           → Image Encoder → 512维向量

相似度计算：cosine_similarity(text_vector, image_vector)
```

### 2.2 CLIP 模型的使用

**安装依赖**：
```bash
pip install transformers torch pillow
```

**基础使用示例**：
```python
from transformers import CLIPProcessor, CLIPModel
from PIL import Image
import torch

# 加载 CLIP 模型
model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

# 文本向量化
text = "a dog playing in the park"
text_inputs = processor(text=[text], return_tensors="pt", padding=True)
text_features = model.get_text_features(**text_inputs)
text_vector = text_features.detach().numpy()[0]  # 512维向量

# 图片向量化
image = Image.open("dog.jpg")
image_inputs = processor(images=image, return_tensors="pt")
image_features = model.get_image_features(**image_inputs)
image_vector = image_features.detach().numpy()[0]  # 512维向量

# 计算相似度
similarity = torch.nn.functional.cosine_similarity(
    text_features, image_features
).item()
print(f"相似度: {similarity}")
```

## 3. 多模态向量字段定义

### 3.1 Schema 设计

在 Milvus 中，为图片和文本分别定义向量字段：

```python
from pymilvus import MilvusClient, DataType

# 连接 Milvus
client = MilvusClient(uri="http://localhost:19530")

# 定义 Schema
schema = MilvusClient.create_schema(
    auto_id=True,
    enable_dynamic_field=True
)

# 添加主键字段
schema.add_field(
    field_name="id",
    datatype=DataType.INT64,
    is_primary=True,
    auto_id=True
)

# 添加文本字段
schema.add_field(
    field_name="text",
    datatype=DataType.VARCHAR,
    max_length=1000
)

# 添加图片路径字段
schema.add_field(
    field_name="image_path",
    datatype=DataType.VARCHAR,
    max_length=500
)

# 添加文本向量字段（CLIP Text Encoder）
schema.add_field(
    field_name="text_vector",
    datatype=DataType.FLOAT_VECTOR,
    dim=512  # CLIP ViT-B/32 输出 512 维
)

# 添加图片向量字段（CLIP Image Encoder）
schema.add_field(
    field_name="image_vector",
    datatype=DataType.FLOAT_VECTOR,
    dim=512  # CLIP ViT-B/32 输出 512 维
)

print("Schema 定义完成")
```

### 3.2 索引创建

**关键要求**：所有向量字段都必须创建索引才能加载 Collection。

```python
# 为文本向量创建索引
index_params_text = MilvusClient.prepare_index_params()
index_params_text.add_index(
    field_name="text_vector",
    index_type="IVF_FLAT",
    metric_type="COSINE",
    params={"nlist": 128}
)

# 为图片向量创建索引
index_params_image = MilvusClient.prepare_index_params()
index_params_image.add_index(
    field_name="image_vector",
    index_type="IVF_FLAT",
    metric_type="COSINE",
    params={"nlist": 128}
)

# 创建 Collection
client.create_collection(
    collection_name="multimodal_collection",
    schema=schema,
    index_params=index_params_text
)

# 为图片向量字段创建索引
client.create_index(
    collection_name="multimodal_collection",
    index_params=index_params_image
)

print("索引创建完成")
```

## 4. 数据插入

### 4.1 批量向量化与插入

```python
from transformers import CLIPProcessor, CLIPModel
from PIL import Image
import os

# 加载 CLIP 模型
model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

def encode_text(text):
    """文本向量化"""
    inputs = processor(text=[text], return_tensors="pt", padding=True)
    features = model.get_text_features(**inputs)
    return features.detach().numpy()[0].tolist()

def encode_image(image_path):
    """图片向量化"""
    image = Image.open(image_path)
    inputs = processor(images=image, return_tensors="pt")
    features = model.get_image_features(**inputs)
    return features.detach().numpy()[0].tolist()

# 准备数据
data = [
    {
        "text": "a dog playing in the park",
        "image_path": "images/dog_park.jpg"
    },
    {
        "text": "a cat sleeping on the sofa",
        "image_path": "images/cat_sofa.jpg"
    },
    {
        "text": "a bird flying in the sky",
        "image_path": "images/bird_sky.jpg"
    }
]

# 向量化并插入
entities = []
for item in data:
    text_vector = encode_text(item["text"])
    image_vector = encode_image(item["image_path"])

    entities.append({
        "text": item["text"],
        "image_path": item["image_path"],
        "text_vector": text_vector,
        "image_vector": image_vector
    })

# 批量插入
client.insert(
    collection_name="multimodal_collection",
    data=entities
```

## 5. 多模态检索实现

### 5.1 文本查询（以文搜图 + 以文搜文）

```python
from pymilvus import AnnSearchRequest, RRFRanker

def multimodal_text_search(query_text, limit=5):
    """使用文本查询，同时检索文本和图片"""

    # 1. 将查询文本向量化
    query_vector = encode_text(query_text)

    # 2. 定义文本向量检索请求
    text_search_req = AnnSearchRequest(
        data=[query_vector],
        anns_field="text_vector",
        param={"metric_type": "COSINE", "params": {"nprobe": 10}},
        limit=limit
    )

    # 3. 定义图片向量检索请求（使用相同的查询向量）
    image_search_req = AnnSearchRequest(
        data=[query_vector],
        anns_field="image_vector",
        param={"metric_type": "COSINE", "params": {"nprobe": 10}},
        limit=limit
    )

    # 4. 使用 RRF 融合结果
    results = client.hybrid_search(
        collection_name="multimodal_collection",
        reqs=[text_search_req, image_search_req],
        ranker=RRFRanker(),
        limit=limit,
        output_fields=["text", "image_path"]
    )

    return results[0]

# 测试：以文搜图 + 以文搜文
query = "a cute animal playing outdoors"
results = multimodal_text_search(query)

print(f"查询: {query}\n")
for i, hit in enumerate(results):
    print(f"结果 {i+1}:")
    print(f"  文本: {hit['entity']['text']}")
    print(f"  图片: {hit['entity']['image_path']}")
    print(f"  距离: {hit['distance']}\n")
```

### 5.3 加权融合策略

使用 `WeightedRanker` 可以精确控制文本和图片的权重：

```python
from pymilvus import WeightedRanker

def multimodal_weighted_search(query_text, text_weight=0.7, image_weight=0.3, limit=5):
    """使用加权融合策略的多模态检索"""

    # 1. 将查询文本向量化
    query_vector = encode_text(query_text)

    # 2. 定义检索请求
    text_search_req = AnnSearchRequest(
        data=[query_vector],
        anns_field="text_vector",
        param={"metric_type": "COSINE", "params": {"nprobe": 10}},
        limit=limit
    )

    image_search_req = AnnSearchRequest(
        data=[query_vector],
        anns_field="image_vector",
        param={"metric_type": "COSINE", "params": {"nprobe": 10}},
        limit=limit
    )

    # 3. 使用加权排序器
    ranker = WeightedRanker(text_weight, image_weight)

    # 4. 执行混合检索
    results = client.hybrid_search(
        collection_name="multimodal_collection",
        reqs=[text_search_req, image_search_req],
        ranker=ranker,
        limit=limit,
        output_fields=["text", "image_path"]
    )

    return results[0]

# 测试：更重视文本匹配
query = "a dog playing"
results = multimodal_weighted_search(query, text_weight=0.8, image_weight=0.2)

print(f"查询: {query} (文本权重: 0.8, 图片权重: 0.2)\n")
for i, hit in enumerate(results):
    print(f"结果 {i+1}:")
    print(f"  文本: {hit['entity']['text']}")
    print(f"  图片: {hit['entity']['image_path']}")
    print(f"  距离: {hit['distance']}\n")
```

## 6. 完整实战示例

### 6.1 电商图文搜索系统

```python
import os
from pymilvus import MilvusClient, DataType, AnnSearchRequest, RRFRanker
from transformers import CLIPProcessor, CLIPModel
from PIL import Image

class MultimodalSearchSystem:
    """多模态搜索系统"""

    def __init__(self, collection_name="ecommerce_multimodal"):
        self.collection_name = collection_name
        self.client = MilvusClient(uri="http://localhost:19530")

        # 加载 CLIP 模型
        self.model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
        self.processor = CLIPProcessor.from_pretrained("openai/clip-vit-base-patch32")

        self._create_collection()

    def _create_collection(self):
        """创建 Collection"""
        if self.client.has_collection(self.collection_name):
            self.client.drop_collection(self.collection_name)

        # 定义 Schema
        schema = MilvusClient.create_schema(auto_id=True, enable_dynamic_field=True)
        schema.add_field("id", DataType.INT64, is_primary=True, auto_id=True)
        schema.add_field("product_name", DataType.VARCHAR, max_length=500)
        schema.add_field("description", DataType.VARCHAR, max_length=2000)
        schema.add_field("image_url", DataType.VARCHAR, max_length=500)
        schema.add_field("price", DataType.FLOAT)
        schema.add_field("text_vector", DataType.FLOAT_VECTOR, dim=512)
        schema.add_field("image_vector", DataType.FLOAT_VECTOR, dim=512)

        # 创建索引
        index_params = MilvusClient.prepare_index_params()
        index_params.add_index("text_vector", "IVF_FLAT", "COSINE", {"nlist": 128})
        index_params.add_index("image_vector", "IVF_FLAT", "COSINE", {"nlist": 128})

        # 创建 Collection
        self.client.create_collection(
            collection_name=self.collection_name,
            schema=schema,
            index_params=index_params
        )

    def encode_text(self, text):
        """文本向量化"""
        inputs = self.processor(text=[text], return_tensors="pt", padding=True)
        features = self.model.get_text_features(**inputs)
        return features.detach().numpy()[0].tolist()

    def encode_image(self, image_path):
        """图片向量化"""
        image = Image.open(image_path)
        inputs = self.processor(images=image, return_tensors="pt")
        features = self.model.get_image_features(**inputs)
        return features.detach().numpy()[0].tolist()

    def add_products(self, products):
        """批量添加商品"""
        entities = []
        for product in products:
            # 组合文本：商品名 + 描述
            combined_text = f"{product['product_name']} {product['description']}"
            text_vector = self.encode_text(combined_text)
            image_vector = self.encode_image(product['image_path'])

            entities.append({
                "product_name": product['product_name'],
                "description": product['description'],
                "image_url": product['image_url'],
                "price": product['price'],
                "text_vector": text_vector,
                "image_vector": image_vector
            })

        self.client.insert(collection_name=self.collection_name, data=entities)
        print(f"添加 {len(entities)} 个商品")

    def search_by_text(self, query_text, limit=10):
        """文本搜索"""
        query_vector = self.encode_text(query_text)

        text_req = AnnSearchRequest(
            data=[query_vector],
            anns_field="text_vector",
            param={"metric_type": "COSINE", "params": {"nprobe": 10}},
            limit=limit
        )

        image_req = AnnSearchRequest(
            data=[query_vector],
            anns_field="image_vector",
            param={"metric_type": "COSINE", "params": {"nprobe": 10}},
            limit=limit
        )

        results = self.client.hybrid_search(
            collection_name=self.collection_name,
            reqs=[text_req, image_req],
            ranker=RRFRanker(),
            limit=limit,
            output_fields=["product_name", "description", "image_url", "price"]
        )

        return results[0]

    def search_by_image(self, image_path, limit=10):
        """图片搜索"""
        query_vector = self.encode_image(image_path)

        text_req = AnnSearchRequest(
            data=[query_vector],
            anns_field="text_vector",
            param={"metric_type": "COSINE", "params": {"nprobe": 10}},
            limit=limit
        )

        image_req = AnnSearchRequest(
            data=[query_vector],
            anns_field="image_vector",
            param={"metric_type": "COSINE", "params": {"nprobe": 10}},
            limit=limit
        )

        results = self.client.hybrid_search(
            collection_name=self.collection_name,
            reqs=[text_req, image_req],
            ranker=RRFRanker(),
            limit=limit,
            output_fields=["product_name", "description", "image_url", "price"]
        )

        return results[0]

# 使用示例
if __name__ == "__main__":
    # 初始化系统
    system = MultimodalSearchSystem()

    # 添加商品数据
    products = [
        {
            "product_name": "红色连衣裙",
            "description": "优雅的红色连衣裙，适合夏季穿着",
            "image_path": "products/red_dress.jpg",
            "image_url": "https://example.com/red_dress.jpg",
            "price": 299.0
        },
        {
            "product_name": "蓝色牛仔裤",
            "description": "经典蓝色牛仔裤，百搭款式",
            "image_path": "products/blue_jeans.jpg",
            "image_url": "https://example.com/blue_jeans.jpg",
            "price": 199.0
        }
    ]

    system.add_products(products)

    # 文本搜索
    print("\n=== 文本搜索：红色裙子 ===")
    results = system.search_by_text("红色裙子", limit=5)
    for i, hit in enumerate(results):
        print(f"{i+1}. {hit['entity']['product_name']} - ¥{hit['entity']['price']}")
        print(f"   {hit['entity']['description']}")
        print(f"   相似度: {1 - hit['distance']:.4f}\n")

    # 图片搜索
    print("\n=== 图片搜索 ===")
    results = system.search_by_image("query_images/dress.jpg", limit=5)
    for i, hit in enumerate(results):
        print(f"{i+1}. {hit['entity']['product_name']} - ¥{hit['entity']['price']}")
        print(f"   {hit['entity']['description']}")
        print(f"   相似度: {1 - hit['distance']:.4f}\n")
```

## 7. 应用场景

### 7.1 电商搜索
- **以文搜图**：用户输入"红色连衣裙"，返回相关商品图片

## 8. 性能优化建议

### 8.1 模型选择
- **CLIP ViT-B/32**：512维，速度快，适合大规模应用
- **CLIP ViT-L/14**：768维，精度高，适合高质量要求
- **中文 CLIP**：针对中文场景优化的模型

### 8.2 索引优化
- 使用 `IVF_FLAT` 或 `HNSW` 索引
- 根据数据规模调整 `nlist` 参数
- 考虑使用 GPU 加速向量检索

### 8.3 权重调整
- 根据业务场景调整文本和图片的权重
- 使用 A/B 测试找到最优权重配置
- 考虑用户反馈动态调整权重

## 9. 总结

多模态检索是 Milvus 多向量检索的重要应用场景，通过以下步骤实现：

1. **模型选择**：使用 CLIP 等多模态模型生成统一语义空间的向量
2. **Schema 设计**：为不同模态定义独立的向量字段
3. **数据插入**：批量向量化并插入多模态数据
4. **混合检索**：使用 `AnnSearchRequest` 定义多个检索请求
5. **结果融合**：使用 `RRFRanker` 或 `WeightedRanker` 融合结果

**关键要点**：
- 所有向量字段都必须创建索引
- CLIP 模型将图片和文本映射到同一向量空间
- RRFRanker 适合不确定权重的场景
- WeightedRanker 适合已知权重的场景
