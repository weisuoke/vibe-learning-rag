# OpenClaw 基础故障排查 - 实战场景 5：配置验证与修复

## 一、场景概述

**场景描述**：使用 `openclaw doctor` 命令验证和修复 OpenClaw 配置问题

**适用场景**：
- 升级后配置不兼容
- 手动编辑配置出错
- 环境变量覆盖配置
- 配置文件损坏
- 权限配置错误

**核心工具**：
- `openclaw doctor` - 健康检查
- `openclaw doctor --repair` - 自动修复
- `openclaw doctor --deep` - 深度诊断

## 二、TypeScript 配置验证器实现

### 2.1 配置验证系统

```typescript
// config-validator.ts
import * as fs from 'fs/promises';
import * as path from 'path';
import { homedir } from 'os';

// 验证层次
enum ValidationLayer {
  SYNTAX = 'syntax',           // 层次1：语法验证
  VALUES = 'values',           // 层次2：值验证
  VERSION = 'version',         // 层次3：版本兼容性
  ENVIRONMENT = 'environment'  // 层次4：环境变量冲突
}

// 验证问题
interface ValidationIssue {
  type: string;
  severity: 'fatal' | 'error' | 'warning';
  message: string;
  field?: string;
  value?: any;
}

// 验证结果
interface ValidationResult {
  valid: boolean;
  layer: ValidationLayer;
  issues: ValidationIssue[];
  suggestions: string[];
}

// 配置验证器
class ConfigValidator {
  private configPath: string;

  constructor(configPath?: string) {
    this.configPath = configPath || path.join(homedir(), '.openclaw', 'openclaw.json');
  }

  // 完整验证流程
  async validate(): Promise<ValidationResult[]> {
    const results: ValidationResult[] = [];

    // 1. 语法验证
    const syntaxResult = await this.validateSyntax();
    results.push(syntaxResult);

    if (!syntaxResult.valid) {
      return results; // 语法错误，停止后续验证
    }

    // 2. 加载配置
    const content = await fs.readFile(this.configPath, 'utf-8');
    const config = JSON.parse(content);

    // 3. 值验证
    results.push(await this.validateValues(config));

    // 4. 版本验证
    results.push(await this.validateVersion(config));

    // 5. 环境变量验证
    results.push(await this.validateEnvironment(config));

    return results;
  }

  // 层次1：语法验证
  private async validateSyntax(): Promise<ValidationResult> {
    try {
      const content = await fs.readFile(this.configPath, 'utf-8');
      JSON.parse(content);

      return {
        valid: true,
        layer: ValidationLayer.SYNTAX,
        issues: [],
        suggestions: []
      };
    } catch (error: any) {
      return {
        valid: false,
        layer: ValidationLayer.SYNTAX,
        issues: [{
          type: 'syntax_error',
          severity: 'fatal',
          message: `JSON syntax error: ${error.message}`
        }],
        suggestions: ['Run: openclaw doctor --repair']
      };
    }
  }

  // 层次2：值验证
  private async validateValues(config: any): Promise<ValidationResult> {
    const issues: ValidationIssue[] = [];

    // 验证 gateway.mode
    if (!['local', 'remote'].includes(config.gateway?.mode)) {
      issues.push({
        type: 'invalid_value',
        severity: 'error',
        field: 'gateway.mode',
        value: config.gateway?.mode,
        message: `Invalid gateway.mode: ${config.gateway?.mode}`
      });
    }

    // 验证 gateway.port
    const port = config.gateway?.port;
    if (port && (port < 1024 || port > 65535)) {
      issues.push({
        type: 'invalid_value',
        severity: 'error',
        field: 'gateway.port',
        value: port,
        message: `Port must be between 1024 and 65535, got ${port}`
      });
    }

    // 验证 gateway.auth.token
    if (config.gateway?.mode === 'local' && !config.gateway?.auth?.token) {
      issues.push({
        type: 'missing_value',
        severity: 'warning',
        field: 'gateway.auth.token',
        message: 'Authentication token not set'
      });
    }

    return {
      valid: issues.length === 0,
      layer: ValidationLayer.VALUES,
      issues,
      suggestions: issues.length > 0 ? ['Run: openclaw doctor --repair'] : []
    };
  }

  // 层次3：版本兼容性
  private async validateVersion(config: any): Promise<ValidationResult> {
    const currentVersion = config.version || 'v1';
    const requiredVersion = 'v2';

    if (currentVersion === requiredVersion) {
      return {
        valid: true,
        layer: ValidationLayer.VERSION,
        issues: [],
        suggestions: []
      };
    }

    return {
      valid: false,
      layer: ValidationLayer.VERSION,
      issues: [{
        type: 'version_mismatch',
        severity: 'warning',
        message: `Config version ${currentVersion} is outdated (required: ${requiredVersion})`
      }],
      suggestions: [
        'Run: openclaw doctor --repair',
        'This will migrate your config to the latest version'
      ]
    };
  }

  // 层次4：环境变量冲突
  private async validateEnvironment(config: any): Promise<ValidationResult> {
    const issues: ValidationIssue[] = [];

    // 检查 OPENCLAW_GATEWAY_TOKEN
    const envToken = process.env.OPENCLAW_GATEWAY_TOKEN;
    if (envToken && envToken !== config.gateway?.auth?.token) {
      issues.push({
        type: 'env_conflict',
        severity: 'warning',
        field: 'OPENCLAW_GATEWAY_TOKEN',
        message: 'Environment variable OPENCLAW_GATEWAY_TOKEN overrides config'
      });
    }

    // 检查 OPENCLAW_GATEWAY_PASSWORD
    const envPassword = process.env.OPENCLAW_GATEWAY_PASSWORD;
    if (envPassword && envPassword !== config.gateway?.auth?.password) {
      issues.push({
        type: 'env_conflict',
        severity: 'warning',
        field: 'OPENCLAW_GATEWAY_PASSWORD',
        message: 'Environment variable OPENCLAW_GATEWAY_PASSWORD overrides config'
      });
    }

    return {
      valid: issues.length === 0,
      layer: ValidationLayer.ENVIRONMENT,
      issues,
      suggestions: issues.length > 0 ? [
        'Clear environment variables:',
        '  macOS: launchctl unsetenv OPENCLAW_GATEWAY_TOKEN',
        '  Linux: unset OPENCLAW_GATEWAY_TOKEN'
      ] : []
    };
  }

  // 打印验证报告
  printReport(results: ValidationResult[]): void {
    console.log('=== OpenClaw Configuration Validation Report ===\n');

    for (const result of results) {
      const status = result.valid ? '✓' : '✗';
      console.log(`${status} ${result.layer.toUpperCase()} Validation`);

      if (result.issues.length > 0) {
        for (const issue of result.issues) {
          const icon = issue.severity === 'fatal' ? '✗' :
                      issue.severity === 'error' ? '⚠' : 'ℹ';
          console.log(`  ${icon} ${issue.message}`);
        }
      }

      if (result.suggestions.length > 0) {
        console.log('\n  Suggestions:');
        for (const suggestion of result.suggestions) {
          console.log(`    - ${suggestion}`);
        }
      }

      console.log('');
    }

    const allValid = results.every(r => r.valid);
    if (allValid) {
      console.log('✓ All checks passed!');
    } else {
      console.log('✗ Issues found. Run "openclaw doctor --repair" to fix.');
    }
  }
}

// 使用示例
async function main() {
  const validator = new ConfigValidator();
  const results = await validator.validate();
  validator.printReport(results);
}

main().catch(console.error);
```

## 三、TypeScript 配置修复器实现

### 3.1 自动修复系统

```typescript
// config-repairer.ts
import * as fs from 'fs/promises';
import * as path from 'path';
import { homedir } from 'os';
import { randomBytes } from 'crypto';

// 配置修复器
class ConfigRepairer {
  private configPath: string;
  private backupPath: string;

  constructor(configPath?: string) {
    this.configPath = configPath || path.join(homedir(), '.openclaw', 'openclaw.json');
    this.backupPath = `${this.configPath}.bak`;
  }

  // 完整修复流程
  async repair(): Promise<void> {
    console.log('Starting config repair...\n');

    // 1. 备份
    await this.backup();

    // 2. 语法修复
    const syntaxFixed = await this.repairSyntax();
    if (!syntaxFixed) {
      console.error('Cannot proceed with repair due to syntax errors');
      return;
    }

    // 3. 加载配置
    const content = await fs.readFile(this.configPath, 'utf-8');
    let config = JSON.parse(content);

    // 4. 版本迁移
    const currentVersion = config.version || 'v1';
    const requiredVersion = 'v2';

    if (currentVersion !== requiredVersion) {
      config = await this.migrate(config, currentVersion, requiredVersion);
    }

    // 5. 值修复
    config = await this.repairValues(config);

    // 6. 保存配置
    await fs.writeFile(this.configPath, JSON.stringify(config, null, 2));
    console.log('\n✓ All issues fixed!');
  }

  // 备份配置
  private async backup(): Promise<void> {
    const content = await fs.readFile(this.configPath, 'utf-8');
    const config = JSON.parse(content);

    const backup = {
      timestamp: Date.now(),
      version: config.version || 'v1',
      config
    };

    await fs.writeFile(this.backupPath, JSON.stringify(backup, null, 2));
    console.log(`✓ Backup saved to ${this.backupPath}`);
  }

  // 修复语法错误
  private async repairSyntax(): Promise<boolean> {
    try {
      const content = await fs.readFile(this.configPath, 'utf-8');

      // 尝试修复常见语法错误
      let fixed = content
        .replace(/,(\s*[}\]])/g, '$1')  // 移除尾随逗号
        .replace(/'/g, '"')              // 单引号转双引号
        .replace(/(\w+):/g, '"$1":');    // 键名加引号

      JSON.parse(fixed); // 验证修复后的 JSON

      await fs.writeFile(this.configPath, fixed);
      console.log('✓ Config syntax fixed');
      return true;
    } catch (error) {
      console.error('✗ Cannot auto-fix syntax error');
      return false;
    }
  }

  // 修复无效值
  private async repairValues(config: any): Promise<any> {
    // 修复 gateway.mode
    if (!['local', 'remote'].includes(config.gateway?.mode)) {
      config.gateway = config.gateway || {};
      config.gateway.mode = 'local';
      console.log('✓ Set gateway.mode to "local"');
    }

    // 修复 gateway.port
    const port = config.gateway?.port;
    if (!port || port < 1024 || port > 65535) {
      config.gateway.port = 18789;
      console.log('✓ Set gateway.port to 18789');
    }

    // 生成 auth.token（如果缺失）
    if (!config.gateway?.auth?.token) {
      config.gateway.auth = config.gateway.auth || {};
      config.gateway.auth.token = this.generateToken();
      console.log('✓ Generated gateway.auth.token');
    }

    return config;
  }

  // 配置迁移
  private async migrate(config: any, fromVersion: string, toVersion: string): Promise<any> {
    console.log(`Migrating config from ${fromVersion} to ${toVersion}...`);

    if (fromVersion === 'v1' && toVersion === 'v2') {
      // v1 → v2 迁移

      // 1. 重命名 gateway.token → gateway.auth.token
      if (config.gateway?.token) {
        config.gateway.auth = config.gateway.auth || {};
        config.gateway.auth.token = config.gateway.token;
        delete config.gateway.token;
        console.log('✓ Migrated gateway.token → gateway.auth.token');
      }

      // 2. 添加 gateway.auth.mode
      if (!config.gateway?.auth?.mode) {
        config.gateway.auth = config.gateway.auth || {};
        config.gateway.auth.mode = 'token';
        console.log('✓ Added gateway.auth.mode');
      }

      // 3. 移除废弃的 gateway.password
      if (config.gateway?.password) {
        delete config.gateway.password;
        console.log('✓ Removed deprecated gateway.password');
      }

      // 4. 更新版本号
      config.version = 'v2';
      console.log('✓ Updated version to v2');
    }

    return config;
  }

  // 恢复备份
  async restore(): Promise<void> {
    const content = await fs.readFile(this.backupPath, 'utf-8');
    const backup = JSON.parse(content);

    await fs.writeFile(this.configPath, JSON.stringify(backup.config, null, 2));
    console.log(`✓ Config restored from ${this.backupPath}`);
  }

  // 生成token
  private generateToken(): string {
    return `sk-ant-${Date.now()}-${randomBytes(8).toString('hex')}`;
  }
}

// 使用示例
async function main() {
  const repairer = new ConfigRepairer();
  await repairer.repair();
}

main().catch(console.error);
```

## 四、类比理解

### 4.1 后端开发类比

**类比 1：Nginx 配置测试**

```
OpenClaw doctor ≈ nginx -t

相似点：
1. 语法检查 - 验证配置文件格式
2. 值验证 - 检查配置项的有效性
3. 安全重载 - 修复后重启服务

OpenClaw:
openclaw doctor           # 测试配置
openclaw doctor --repair  # 修复并重启

Nginx:
nginx -t                    # 测试配置
nginx -t && nginx -s reload # 测试并重载
```

**类比 2：Terraform Validate**

```
OpenClaw doctor ≈ Terraform validate

相似点：
1. 配置验证 - 检查配置语法和值
2. 自动修复 - 修复常见问题
3. 变更预览 - 显示将要修复的内容

OpenClaw:
openclaw doctor           # 验证配置
openclaw doctor --repair  # 应用修复

Terraform:
terraform validate  # 验证配置
terraform plan      # 预览变更
```

**来源**: 后端开发经验

## 六、总结

### 6.1 核心要点

1. **4层验证系统**：语法、值、版本、环境
2. **自动修复能力**：doctor --repair 一键修复
3. **安全备份机制**：修复前自动备份
4. **配置迁移**：自动迁移旧版本配置
5. **环境变量检测**：识别配置覆盖问题

### 6.2 使用建议

- ✅ 升级后立即运行 doctor
- ✅ 修改配置后验证
- ✅ 定期运行健康检查
- ✅ 保留配置备份
- ❌ 不要跳过验证直接启动
