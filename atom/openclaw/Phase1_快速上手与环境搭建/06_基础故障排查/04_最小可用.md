## 一、核心诊断命令（5个）

### 1.1 openclaw status - 快速状态检查

```bash
openclaw status
```

**作用**：快速查看系统整体状态

**输出示例**：
```
✓ Gateway: running
✓ Channels: 2 connected
✓ Provider: anthropic configured
⚠ Updates: v2026.2.23 available
```

**何时使用**：
- 第一时间检查系统状态
- 快速判断是否有明显问题

**来源**: `reference/source_troubleshooting_main.md`

### 1.2 openclaw gateway status - 网关状态

```bash
openclaw gateway status
```

**作用**：检查 Gateway 运行状态和端口

**输出示例**：
```
Runtime: running
RPC probe: ok
Port: 18789
Dashboard: http://127.0.0.1:18789/
```

**何时使用**：
- Gateway 无法启动
- 端口冲突问题
- 连接问题

**来源**: `reference/source_gateway_troubleshooting.md`

### 1.3 openclaw doctor - 健康检查

```bash
openclaw doctor
```

**作用**：自动检测配置和服务问题

**输出示例**：
```
✓ Config syntax valid
✓ Gateway mode configured
✗ gateway.auth.token not set

Issues found: 1
Run 'openclaw doctor --repair' to fix
```

**何时使用**：
- 升级后验证
- 配置错误检测
- 定期健康检查

**来源**: `reference/source_doctor_command.md`

### 1.4 openclaw logs --follow - 实时日志

```bash
openclaw logs --follow
```

**作用**：实时查看系统日志

**何时使用**：
- 问题正在发生时
- 需要看到错误消息
- 调试问题

**来源**: `reference/source_debugging_tools.md`

### 1.5 openclaw doctor --repair - 自动修复

```bash
openclaw doctor --repair
```

**作用**：自动修复常见配置问题

**输出示例**：
```
✓ Backup saved
✓ Config migrated v1 → v2
✓ Fixed gateway.mode
✓ All issues fixed!
```

**何时使用**：
- doctor 发现问题后
- 升级后配置不兼容
- 配置文件损坏

**来源**: `reference/source_doctor_command.md`

## 二、60秒诊断流程

**标准化诊断流程，按顺序执行：**

```bash
# 1. 快速状态
openclaw status

# 2. 网关状态
openclaw gateway status

# 3. 健康检查
openclaw doctor

# 4. 如果有问题，自动修复
openclaw doctor --repair

# 5. 查看日志（如果问题仍存在）
openclaw logs --follow
```

**这个流程能解决的问题**：
- ✅ 配置错误（80%）
- ✅ 端口冲突（90%）
- ✅ 服务未启动（95%）
- ✅ 升级后问题（85%）

**来源**: `reference/source_troubleshooting_main.md`

## 三、常见问题快速修复

### 3.1 Gateway 无法启动

**症状**：运行 `openclaw gateway start` 无响应

**快速修复**：
```bash
# 1. 检查状态
openclaw gateway status

# 2. 如果显示端口冲突，停止其他实例
openclaw gateway stop

# 3. 重新启动
openclaw gateway start

# 4. 验证
openclaw gateway status
```

**来源**: `reference/search_openclaw_troubleshooting.md`

### 3.2 配置错误

**症状**：Gateway 启动失败，提示配置错误

**快速修复**：
```bash
# 一键修复
openclaw doctor --repair
```

**来源**: `reference/source_doctor_command.md`

### 3.3 通道无回复

**症状**：通道显示已连接，但发送消息无回复

**快速修复**：
```bash
# 1. 检查通道状态
openclaw channels status --probe

# 2. 检查配对状态
openclaw pairing list <channel>

# 3. 如果需要批准
openclaw pairing approve <channel> <sender>
```

**来源**: `reference/source_channels_troubleshooting.md`

## 四、关键概念速查

### 4.1 端口 18789

**OpenClaw Gateway 的默认端口**

```bash
# 检查端口占用
openclaw gateway status

# 如果端口被占用
openclaw gateway stop
```

### 4.2 配对机制（Pairing）

**DM 发送者需要批准才能交互**

```bash
# 查看待批准列表
openclaw pairing list <channel>

# 批准发送者
openclaw pairing approve <channel> <sender>
```

### 4.3 提及门控（Mention Gating）

**群组消息需要提及机器人才触发**

```bash
# 检查配置
openclaw config get channels.<channel>.requireMention

# 关闭提及要求
openclaw config set channels.<channel>.requireMention false
```

## 五、日志特征识别

### 5.1 常见日志特征

| 日志特征 | 问题类型 | 快速修复 |
|---------|---------|---------|
| `EADDRINUSE` | 端口被占用 | `openclaw gateway stop` |
| `unauthorized` | 认证失败 | `openclaw doctor --repair` |
| `pairing required` | 配对待定 | `openclaw pairing approve` |
| `mention required` | 提及门控 | 提及机器人或关闭门控 |
| `gateway.mode=local` | 配置错误 | `openclaw doctor --repair` |
