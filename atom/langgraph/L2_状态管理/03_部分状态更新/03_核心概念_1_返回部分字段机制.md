# 核心概念 1：返回部分字段机制

## 概念定义

**返回部分字段机制**：节点函数只需要返回需要更新的状态字段，未返回的字段自动保持原值不变的状态更新机制。

**一句话：** 节点函数返回字典时，只包含要改变的字段，框架自动将其与当前状态合并。

**来源：** [reference/context7_langgraph_01.md - 节点函数返回部分状态]

---

## 核心原理

### 1. 基本工作流程

```
当前状态 + 节点返回的部分状态 → 新状态
```

**流程图：**
```
┌─────────────────┐
│  当前状态        │
│  {              │
│    "a": 1,      │
│    "b": 2,      │
│    "c": 3       │
│  }              │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  节点返回        │
│  {              │
│    "b": 20      │  ← 只返回要改变的字段
│  }              │
└────────┬────────┘
         │
         ↓ (框架自动合并)
┌─────────────────┐
│  新状态          │
│  {              │
│    "a": 1,      │  ← 保持不变
│    "b": 20,     │  ← 更新
│    "c": 3       │  ← 保持不变
│  }              │
└─────────────────┘
```

**来源：** [reference/source_部分状态更新_01.md - 节点返回值处理]

---

### 2. 支持的返回值类型

LangGraph 支持三种返回值类型：

#### 类型 1：字典（最常用）

```python
def node_function(state: State) -> dict:
    return {"field_name": "new_value"}
```

**特点：**
- 最简单、最直观
- 直接返回要更新的字段
- 适用于大多数场景

---

#### 类型 2：Pydantic 模型

```python
from pydantic import BaseModel

class StateUpdate(BaseModel):
    field_name: str

def node_function(state: State) -> StateUpdate:
    return StateUpdate(field_name="new_value")
```

**特点：**
- 类型安全
- 只更新被显式设置的字段（`model_fields_set`）
- 跳过默认值字段

**来源：** [reference/source_部分状态更新_01.md - Pydantic 模型的部分更新]

---

#### 类型 3：Command 对象

```python
from langgraph.types import Command

def node_function(state: State) -> Command:
    return Command(
        goto="next_node",
        update={"field_name": "new_value"}
    )
```

**特点：**
- 同时控制流程和状态
- 可以指定下一个节点
- 适用于复杂的条件路由

**来源：** [reference/source_部分状态更新_01.md - Command 对象更新]

---

## 详细解释

### 为什么需要部分字段返回？

**问题场景：**

假设我们有一个复杂的状态：

```python
class State(TypedDict):
    messages: list[str]           # 对话历史（可能有100条消息）
    user_profile: dict            # 用户信息
    context: dict                 # 上下文信息
    metadata: dict                # 元数据
    current_step: str             # 当前步骤
    error: str | None             # 错误信息
```

**如果没有部分字段返回：**

```python
def node_a(state: State) -> State:
    # 只想更新 current_step，但必须返回完整状态
    return {
        "messages": state["messages"],        # 必须复制
        "user_profile": state["user_profile"], # 必须复制
        "context": state["context"],          # 必须复制
        "metadata": state["metadata"],        # 必须复制
        "current_step": "completed",          # 只改这个！
        "error": state["error"]               # 必须复制
    }
```

**问题：**
1. 代码冗长：每个节点都要写大量重复代码
2. 容易出错：可能忘记复制某个字段
3. 性能浪费：复制大量不需要改变的数据
4. 维护困难：状态字段增加时，所有节点都要修改

**有了部分字段返回：**

```python
def node_a(state: State) -> dict:
    # 只返回需要改变的字段
    return {"current_step": "completed"}
```

**优势：**
1. 代码简洁：只写需要改变的字段
2. 不会出错：未返回的字段自动保留
3. 性能优化：只传递变化的数据
4. 易于维护：状态字段增加不影响现有节点

**来源：** [reference/search_部分状态更新_01.md - 部分状态更新机制]

## 代码示例

### 示例 1：基础部分字段返回

```python
"""
演示：基础的部分字段返回机制
"""
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# ===== 1. 定义状态 =====
class State(TypedDict):
    counter: int
    message: str
    status: str

# ===== 2. 定义节点函数 =====
def node_a(state: State) -> dict:
    """只更新 counter 字段"""
    print(f"Node A - 当前状态: {state}")
    return {"counter": state["counter"] + 1}

def node_b(state: State) -> dict:
    """只更新 message 字段"""
    print(f"Node B - 当前状态: {state}")
    return {"message": "Hello from Node B"}

def node_c(state: State) -> dict:
    """只更新 status 字段"""
    print(f"Node C - 当前状态: {state}")
    return {"status": "completed"}

# ===== 3. 构建图 =====
builder = StateGraph(State)
builder.add_node("a", node_a)
builder.add_node("b", node_b)
builder.add_node("c", node_c)

builder.add_edge(START, "a")
builder.add_edge("a", "b")
builder.add_edge("b", "c")
builder.add_edge("c", END)

graph = builder.compile()

# ===== 4. 执行图 =====
initial_state = {
    "counter": 0,
    "message": "Initial message",
    "status": "pending"
}

result = graph.invoke(initial_state)
print(f"\n最终状态: {result}")
```

**运行输出：**
```
Node A - 当前状态: {'counter': 0, 'message': 'Initial message', 'status': 'pending'}
Node B - 当前状态: {'counter': 1, 'message': 'Initial message', 'status': 'pending'}
Node C - 当前状态: {'counter': 1, 'message': 'Hello from Node B', 'status': 'pending'}

最终状态: {'counter': 1, 'message': 'Hello from Node B', 'status': 'completed'}
```

**关键观察：**
- Node A 只返回 `{"counter": 1}`，但 `message` 和 `status` 保持不变
- Node B 只返回 `{"message": "Hello from Node B"}`，但 `counter` 和 `status` 保持不变
- Node C 只返回 `{"status": "completed"}`，但 `counter` 和 `message` 保持不变

**来源：** [reference/context7_langgraph_01.md - 节点函数返回部分状态]

### 示例 2：多字段同时更新

```python
"""
演示：节点可以同时更新多个字段
"""
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    step: int
    message: str
    data: list[int]
    error: str | None

def process_node(state: State) -> dict:
    """同时更新多个字段"""
    return {
        "step": state["step"] + 1,
        "message": f"Processed step {state['step'] + 1}",
        "data": state["data"] + [state["step"] + 1]
    }
    # 注意：error 字段未返回，保持原值

# 构建图
builder = StateGraph(State)
builder.add_node("process", process_node)
builder.add_edge(START, "process")
builder.add_edge("process", END)
graph = builder.compile()

# 执行
initial_state = {
    "step": 0,
    "message": "Start",
    "data": [],
    "error": None
}

result = graph.invoke(initial_state)
print(f"最终状态: {result}")
# 输出: {'step': 1, 'message': 'Processed step 1', 'data': [1], 'error': None}
```

**关键点：**
- 节点可以同时返回多个字段
- 未返回的字段（如 `error`）保持原值

### 示例 3：使用 Pydantic 模型

```python
"""
演示：使用 Pydantic 模型进行部分更新
"""
from typing import TypedDict
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph, START, END

# ===== 1. 定义状态 =====
class State(TypedDict):
    name: str
    age: int
    email: str
    phone: str

# ===== 2. 定义 Pydantic 更新模型 =====
class UserUpdate(BaseModel):
    name: str | None = None
    age: int | None = None
    email: str | None = None
    phone: str | None = None

# ===== 3. 定义节点函数 =====
def update_name(state: State) -> dict:
    """只更新 name 字段"""
    update = UserUpdate(name="Alice")
    # 只返回被显式设置的字段
    return {k: v for k, v in update.model_dump().items() if k in update.model_fields_set}

def update_email(state: State) -> dict:
    """只更新 email 字段"""
    update = UserUpdate(email="alice@example.com")
    return {k: v for k, v in update.model_dump().items() if k in update.model_fields_set}

# ===== 4. 构建图 =====
builder = StateGraph(State)
builder.add_node("update_name", update_name)
builder.add_node("update_email", update_email)

builder.add_edge(START, "update_name")
builder.add_edge("update_name", "update_email")
builder.add_edge("update_email", END)

graph = builder.compile()

# ===== 5. 执行图 =====
initial_state = {
    "name": "Unknown",
    "age": 0,
    "email": "unknown@example.com",
    "phone": "000-0000"
}

result = graph.invoke(initial_state)
print(f"最终状态: {result}")
# 输出: {'name': 'Alice', 'age': 0, 'email': 'alice@example.com', 'phone': '000-0000'}
```

**关键点：**
- 使用 `model_fields_set` 只返回被显式设置的字段
- 未设置的字段（如 `age`, `phone`）保持原值

**来源：** [reference/source_部分状态更新_01.md - get_update_as_tuples 函数]

### 示例 4：使用 Command 对象

```python
"""
演示：使用 Command 对象同时更新状态和控制流程
"""
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, START, END
from langgraph.types import Command

class State(TypedDict):
    value: int
    status: str

def check_value(state: State) -> Command[Literal["process", "skip"]]:
    """根据条件决定下一个节点"""
    if state["value"] > 10:
        return Command(
            goto="skip",
            update={"status": "skipped"}
        )
    else:
        return Command(
            goto="process",
            update={"status": "processing"}
        )

def process_node(state: State) -> dict:
    return {
        "value": state["value"] * 2,
        "status": "processed"
    }

def skip_node(state: State) -> dict:
    return {"status": "skipped_completed"}

# 构建图
builder = StateGraph(State)
builder.add_node("check", check_value)
builder.add_node("process", process_node)
builder.add_node("skip", skip_node)

builder.add_edge(START, "check")
builder.add_edge("process", END)
builder.add_edge("skip", END)

graph = builder.compile()

# 测试 1：value <= 10
result1 = graph.invoke({"value": 5, "status": "init"})
print(f"测试 1 结果: {result1}")
# 输出: {'value': 10, 'status': 'processed'}

# 测试 2：value > 10
result2 = graph.invoke({"value": 15, "status": "init"})
print(f"测试 2 结果: {result2}")
# 输出: {'value': 15, 'status': 'skipped_completed'}
```

**关键点：**
- `Command` 对象可以同时更新状态和控制流程
- `goto` 参数指定下一个节点
- `update` 参数指定要更新的状态字段

**来源：** [reference/source_部分状态更新_01.md - Command 对象]

## 在实际应用中的使用

### 应用场景 1：对话系统

```python
"""
实际应用：对话系统中的部分状态更新
"""
from typing import TypedDict, Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages

class ConversationState(TypedDict):
    messages: Annotated[list[str], add_messages]
    user_intent: str
    context: dict
    current_step: str

def understand_intent(state: ConversationState) -> dict:
    """只更新 user_intent"""
    # 模拟意图识别
    last_message = state["messages"][-1] if state["messages"] else ""
    intent = "book_flight" if "flight" in last_message.lower() else "unknown"
    return {"user_intent": intent}

def extract_context(state: ConversationState) -> dict:
    """只更新 context"""
    # 模拟上下文提取
    context = {"destination": "New York", "date": "2026-03-01"}
    return {"context": context}

def generate_response(state: ConversationState) -> dict:
    """只更新 messages 和 current_step"""
    response = f"I understand you want to {state['user_intent']}. Let me help you."
    return {
        "messages": [response],
        "current_step": "completed"
    }

# 构建图
builder = StateGraph(ConversationState)
builder.add_node("understand", understand_intent)
builder.add_node("extract", extract_context)
builder.add_node("respond", generate_response)

builder.add_edge(START, "understand")
builder.add_edge("understand", "extract")
builder.add_edge("extract", "respond")
builder.add_edge("respond", END)

graph = builder.compile()

# 执行
initial_state = {
    "messages": ["I want to book a flight"],
    "user_intent": "",
    "context": {},
    "current_step": "init"
}

result = graph.invoke(initial_state)
print(f"最终状态: {result}")
```

**优势：**
- 每个节点只关注自己的职责
- 状态自动累积和传递
- 代码简洁、易于维护

**来源：** [reference/context7_langgraph_01.md - 实际应用示例]

### 应用场景 2：数据处理管道

```python
"""
实际应用：数据处理管道中的部分状态更新
"""
from typing import TypedDict, Annotated
import operator

class PipelineState(TypedDict):
    raw_data: list[dict]
    processed_data: list[dict]
    errors: Annotated[list[str], operator.add]
    stats: dict

def validate_data(state: PipelineState) -> dict:
    """只更新 errors"""
    errors = []
    for item in state["raw_data"]:
        if "id" not in item:
            errors.append(f"Missing 'id' in item: {item}")
    return {"errors": errors}

def transform_data(state: PipelineState) -> dict:
    """只更新 processed_data"""
    processed = [
        {**item, "processed": True}
        for item in state["raw_data"]
        if "id" in item
    ]
    return {"processed_data": processed}

def calculate_stats(state: PipelineState) -> dict:
    """只更新 stats"""
    stats = {
        "total": len(state["raw_data"]),
        "valid": len(state["processed_data"]),
        "errors": len(state["errors"])
    }
    return {"stats": stats}

# 构建图
builder = StateGraph(PipelineState)
builder.add_node("validate", validate_data)
builder.add_node("transform", transform_data)
builder.add_node("stats", calculate_stats)

builder.add_edge(START, "validate")
builder.add_edge("validate", "transform")
builder.add_edge("transform", "stats")
builder.add_edge("stats", END)

graph = builder.compile()

# 执行
initial_state = {
    "raw_data": [
        {"id": 1, "name": "Alice"},
        {"name": "Bob"},  # 缺少 id
        {"id": 3, "name": "Charlie"}
    ],
    "processed_data": [],
    "errors": [],
    "stats": {}
}

result = graph.invoke(initial_state)
print(f"最终状态: {result}")
```

**优势：**
- 数据处理是流式的
- 错误信息累积（使用 `operator.add` Reducer）
- 每个阶段独立、可测试

**来源：** [reference/search_部分状态更新_01.md - 状态合并机制]

## 与 Reducer 函数的关系

部分字段返回机制与 Reducer 函数紧密配合：

### 默认行为：覆盖

```python
class State(TypedDict):
    counter: int  # 没有 Reducer，默认覆盖

def node(state: State) -> dict:
    return {"counter": 10}  # 直接覆盖原值
```

---

### 使用 Reducer：聚合

```python
from typing import Annotated
import operator

class State(TypedDict):
    counter: Annotated[int, operator.add]  # 使用 operator.add Reducer

def node(state: State) -> dict:
    return {"counter": 10}  # 累加到原值

# 如果原值是 5，新值是 10，最终值是 15
```

**关键点：**
- 部分字段返回决定**返回哪些字段**
- Reducer 函数决定**如何合并字段**

**来源：** [reference/context7_langgraph_01.md - 状态更新语义]

## 最佳实践

### 1. 节点函数应该是纯函数

```python
# ✅ 好的做法：纯函数
def good_node(state: State) -> dict:
    return {"field": "new_value"}

# ❌ 坏的做法：直接修改状态
def bad_node(state: State) -> dict:
    state["field"] = "new_value"  # 不要这样做！
    return state
```

**原因：**
- 纯函数易于测试
- 纯函数没有副作用
- 纯函数易于并行

**来源：** [reference/search_部分状态更新_01.md - 节点如纯函数]

---

### 2. 只返回需要改变的字段

```python
# ✅ 好的做法：只返回需要改变的字段
def good_node(state: State) -> dict:
    return {"status": "completed"}

# ❌ 坏的做法：返回所有字段
def bad_node(state: State) -> dict:
    return {
        "status": "completed",
        "messages": state["messages"],  # 不需要返回
        "context": state["context"]     # 不需要返回
    }
```

**原因：**
- 代码更简洁
- 性能更好
- 易于维护

---

### 3. 使用类型注解

```python
# ✅ 好的做法：使用类型注解
def good_node(state: State) -> dict:
    return {"field": "value"}

# ❌ 坏的做法：没有类型注解
def bad_node(state):
    return {"field": "value"}
```

**原因：**
- 编辑器可以提供类型检查
- 代码更易读
- 减少错误

## 常见误区

### 误区 1：必须返回完整状态

**错误观点：** "节点函数必须返回完整的状态对象"

**正确理解：** 节点函数只需要返回需要改变的字段，未返回的字段自动保留。

```python
# ❌ 错误：返回完整状态
def node(state: State) -> dict:
    return {
        "field1": state["field1"],
        "field2": "new_value",
        "field3": state["field3"]
    }

# ✅ 正确：只返回需要改变的字段
def node(state: State) -> dict:
    return {"field2": "new_value"}
```

---

### 误区 2：返回 None 会清空字段

**错误观点：** "返回 `None` 会清空字段"

**正确理解：** 返回 `None` 会将字段设置为 `None`，而不是清空。如果不想改变字段，不要返回它。

```python
# ❌ 错误：返回 None 清空字段
def node(state: State) -> dict:
    return {"field": None}  # 将字段设置为 None

# ✅ 正确：不返回字段保持原值
def node(state: State) -> dict:
    return {}  # 不改变任何字段
```

---

### 误区 3：部分字段返回不支持 Reducer

**错误观点：** "部分字段返回不支持 Reducer 函数"

**正确理解：** 部分字段返回与 Reducer 函数完美配合，Reducer 决定如何合并字段。

```python
from typing import Annotated
import operator

class State(TypedDict):
    messages: Annotated[list[str], operator.add]

# ✅ 正确：部分字段返回 + Reducer
def node(state: State) -> dict:
    return {"messages": ["new message"]}  # 追加到列表
```

**来源：** [reference/search_部分状态更新_01.md - Annotated Reducers]

## 总结

**返回部分字段机制是 LangGraph 状态管理的核心特性：**

1. **简洁性**：节点只返回需要改变的字段
2. **正确性**：未返回的字段自动保留
3. **灵活性**：支持字典、Pydantic 模型、Command 对象
4. **可组合性**：与 Reducer 函数完美配合

**关键要点：**
- 节点函数是纯函数
- 只返回需要改变的字段
- 使用类型注解
- 与 Reducer 函数配合使用

**这个机制让状态管理变得简单、直观、不易出错。**
