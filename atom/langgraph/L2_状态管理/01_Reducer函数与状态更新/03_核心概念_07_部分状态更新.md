## 1. 概念定义

### 什么是部分状态更新？

**部分状态更新是 LangGraph 中节点只返回需要更新的状态字段，而不是返回完整状态对象的机制。**

在 LangGraph 中，节点函数的签名是 `State -> Partial<State>`，这意味着节点可以选择性地更新状态字段，而不需要返回所有字段。

### 核心特征

1. **选择性更新**: 只返回需要更新的字段
2. **简洁高效**: 减少不必要的数据传递
3. **灵活性高**: 可以添加新字段或更新现有字段
4. **与 Reducer 协作**: 有 Reducer 的字段会合并，无 Reducer 的字段会覆盖

## 2. 节点签名

### 2.1 标准签名

```python
from typing_extensions import TypedDict

class State(TypedDict):
    field1: str
    field2: list
    field3: dict

def node(state: State) -> dict:
    """
    节点签名: State -> Partial<State>

    Args:
        state: 完整的状态对象

    Returns:
        部分状态更新（只包含需要更新的字段）
    """
    return {"field1": "new_value"}  # 只返回需要更新的字段
```

### 2.2 完整示例

```python
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph
import operator

class State(TypedDict):
    messages: Annotated[list, operator.add]
    query: str
    documents: list
    generation: str

def retrieve_docs(state: State) -> dict:
    """只更新 documents 字段"""
    query = state["query"]
    docs = search_documents(query)
    return {"documents": docs}

def generate_answer(state: State) -> dict:
    """只更新 generation 字段"""
    docs = state["documents"]
    query = state["query"]
    answer = generate(query, docs)
    return {"generation": answer}

# 构建图
builder = StateGraph(State)
builder.add_node("retrieve", retrieve_docs)
builder.add_node("generate", generate_answer)
builder.add_edge("__start__", "retrieve")
builder.add_edge("retrieve", "generate")
builder.add_edge("generate", "__end__")

graph = builder.compile()

# 执行
result = graph.invoke({
    "messages": [],
    "query": "What is RAG?",
    "documents": [],
    "generation": ""
})

print(result)
# {
#     "messages": [],
#     "query": "What is RAG?",
#     "documents": [Document(...)],  # 被 retrieve_docs 更新
#     "generation": "RAG is..."       # 被 generate_answer 更新
# }
```

## 3. 返回策略

### 3.1 只返回需要更新的字段

```python
def node(state: State) -> dict:
    # ✅ 正确：只返回需要更新的字段
    return {"field1": new_value}

    # ❌ 错误：不需要返回所有字段
    return {
        "field1": new_value,
        "field2": state["field2"],  # 不需要
        "field3": state["field3"],  # 不需要
    }
```

### 3.2 添加新字段

```python
def node(state: State) -> dict:
    """添加新字段到状态"""
    return {
        "new_field": "value",
        "timestamp": datetime.now()
    }

# 结果：状态中会添加 new_field 和 timestamp 字段
```

### 3.3 更新多个字段

```python
def node(state: State) -> dict:
    """同时更新多个字段"""
    return {
        "field1": "value1",
        "field2": "value2",
        "field3": "value3"
    }
```

### 3.4 条件更新

```python
def node(state: State) -> dict:
    """根据条件决定更新哪些字段"""
    result = {}

    if condition1:
        result["field1"] = "value1"

    if condition2:
        result["field2"] = "value2"

    return result
```

## 4. 与 Reducer 的交互

### 4.1 有 Reducer 的字段：合并

```python
class State(TypedDict):
    items: Annotated[list, operator.add]  # 有 Reducer

def node_a(state: State) -> dict:
    return {"items": [1, 2]}

def node_b(state: State) -> dict:
    return {"items": [3, 4]}

# 结果: items = [1, 2, 3, 4]  (使用 operator.add 合并)
```

### 4.2 无 Reducer 的字段：覆盖

```python
class State(TypedDict):
    status: str  # 无 Reducer

def node_a(state: State) -> dict:
    return {"status": "processing"}

def node_b(state: State) -> dict:
    return {"status": "completed"}

# 结果: status = "completed"  (直接覆盖)
```

### 4.3 混合场景

```python
class State(TypedDict):
    items: Annotated[list, operator.add]  # 有 Reducer
    status: str  # 无 Reducer

def node_a(state: State) -> dict:
    return {
        "items": [1, 2],
        "status": "processing"
    }

def node_b(state: State) -> dict:
    return {
        "items": [3, 4],
        "status": "completed"
    }

# 结果:
# items = [1, 2, 3, 4]  (合并)
# status = "completed"  (覆盖)
```

### 5.5 RAG 完整流程

```python
from langgraph.graph import StateGraph

class RAGState(TypedDict):
    query: str
    documents: Annotated[list, operator.add]
    generation: str
    web_search: str  # 控制流标志

def retrieve_docs(state: RAGState) -> dict:
    """检索文档"""
    query = state["query"]
    docs = retriever.invoke(query)
    return {"documents": docs}

def grade_documents(state: RAGState) -> dict:
    """评分文档"""
    query = state["query"]
    documents = state["documents"]

    filtered_docs = []
    web_search = "No"

    for doc in documents:
        if is_relevant(doc, query):
            filtered_docs.append(doc)
        else:
            web_search = "Yes"

    return {
        "documents": filtered_docs,
        "web_search": web_search
    }

def web_search(state: RAGState) -> dict:
    """网络搜索"""
    query = state["query"]
    web_docs = web_search_tool.invoke(query)
    return {"documents": web_docs}

def generate(state: RAGState) -> dict:
    """生成答案"""
    query = state["query"]
    documents = state["documents"]
    generation = rag_chain.invoke({"query": query, "documents": documents})
    return {"generation": generation}

# 构建图
builder = StateGraph(RAGState)
builder.add_node("retrieve", retrieve_docs)
builder.add_node("grade", grade_documents)
builder.add_node("web_search", web_search)
builder.add_node("generate", generate)

builder.add_edge("__start__", "retrieve")
builder.add_edge("retrieve", "grade")
builder.add_conditional_edges(
    "grade",
    lambda state: "web_search" if state["web_search"] == "Yes" else "generate"
)
builder.add_edge("web_search", "generate")
builder.add_edge("generate", "__end__")

graph = builder.compile()
```

## 6. 常见问题

### Q1: 必须返回所有字段吗？

**A**: 不需要。只返回需要更新的字段即可。

```python
# ✅ 正确：只返回需要更新的字段
def node(state: State) -> dict:
    return {"field1": "value"}

# ❌ 不推荐：返回所有字段
def node(state: State) -> dict:
    return {
        "field1": "value",
        "field2": state["field2"],
        "field3": state["field3"]
    }
```

### Q2: 可以返回空字典吗？

**A**: 可以。返回空字典表示不更新任何字段。

```python
def node(state: State) -> dict:
    if not should_update():
        return {}  # 不更新任何字段
    return {"field1": "value"}
```

### Q3: 如何删除字段？

**A**: 不能直接删除字段。可以设置为 None 或使用 Overwrite。

```python
# 方式 1: 设置为 None
def node(state: State) -> dict:
    return {"field1": None}

# 方式 2: 使用 Overwrite（对于有 Reducer 的字段）
from langgraph.types import Overwrite
def node(state: State) -> dict:
    return {"items": Overwrite([])}
```

### Q4: 部分更新会影响性能吗？

**A**: 不会。部分更新反而更高效，因为减少了数据传递。

```python
# ✅ 高效：只传递需要的数据
def node(state: State) -> dict:
    return {"field1": "value"}

# ❌ 低效：传递所有数据
def node(state: State) -> dict:
    return state  # 传递整个状态对象
```

## 7. 最佳实践

### 7.1 只返回需要更新的字段

```python
# ✅ 好：简洁明了
def node(state: State) -> dict:
    return {"field1": new_value}

# ❌ 坏：冗余
def node(state: State) -> dict:
    return {
        "field1": new_value,
        "field2": state["field2"],
        "field3": state["field3"]
    }
```

### 7.2 使用类型注解

```python
from typing import TypedDict

class State(TypedDict):
    field1: str
    field2: list

def node(state: State) -> dict:
    """带类型注解的节点"""
    return {"field1": "value"}
```

### 7.3 添加文档字符串

```python
def node(state: State) -> dict:
    """
    更新 field1 字段

    Args:
        state: 当前状态

    Returns:
        部分状态更新
    """
    return {"field1": "value"}
```

### 7.4 处理边界情况

```python
def node(state: State) -> dict:
    """处理边界情况"""
    result = {}

    # 条件更新
    if state.get("field1"):
        result["field2"] = process(state["field1"])

    # 默认值
    if "field3" not in state:
        result["field3"] = "default"

    return result
```

## 8. 与前端开发的类比

### React setState

**部分状态更新** 类似于 **React 的 setState 部分更新**：

```python
# LangGraph
def node(state: State) -> dict:
    return {"field1": "value"}

# React (JavaScript)
this.setState({field1: "value"});  // 只更新 field1
```

### Redux Partial Update

**部分状态更新** 类似于 **Redux 的部分状态更新**：

```python
# LangGraph
return {"field1": "value"}

# Redux (JavaScript)
return {
    ...state,
    field1: "value"
};
```

## 10. 总结

**部分状态更新是 LangGraph 状态管理的核心机制**：

1. **选择性更新**: 只返回需要更新的字段
2. **简洁高效**: 减少不必要的数据传递
3. **灵活性高**: 可以添加新字段或更新现有字段
4. **与 Reducer 协作**: 有 Reducer 的字段会合并，无 Reducer 的字段会覆盖

**关键要点**:
- 节点签名: `State -> Partial<State>`
- 只返回需要更新的字段
- 有 Reducer 的字段会合并，无 Reducer 的字段会覆盖
- 可以添加新字段到状态

**使用场景**:
- 查询改写（只更新 query）
- 文档过滤（只更新 documents）
- 添加元数据（添加新字段）
- 控制流标志（添加控制字段）
