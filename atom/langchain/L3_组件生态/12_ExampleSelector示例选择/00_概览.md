### 1.2 为什么需要 ExampleSelector？

**问题场景：**
```python
# ❌ 静态示例的问题
examples = [
    {"input": "2+2", "output": "4"},
    {"input": "What is the moon?", "output": "A celestial body"},
    {"input": "Write a poem", "output": "Roses are red..."}
]

# 用户问题："3+3 等于多少？"
# 问题：所有示例都会被使用，包括不相关的诗歌和月亮
```

**ExampleSelector 的解决方案：**
```python
# ✅ 动态选择相关示例
selector = SemanticSimilarityExampleSelector.from_examples(
    examples=examples,
    embeddings=OpenAIEmbeddings(),
    vectorstore_cls=Chroma,
    k=2  # 只选择最相关的 2 个
)

# 用户问题："3+3 等于多少？"
# 结果：自动选择数学相关的示例（"2+2"），忽略无关示例
```

## 四、重点与难点

### 4.1 核心重点

#### 重点 1：SemanticSimilarityExampleSelector（⭐⭐⭐⭐⭐）

**为什么重要？**
- 最常用的选择器（80% 的场景）
- 效果最好（基于语义理解）
- 2025-2026 年成为主流

**核心用法：**
```python
from langchain.prompts.example_selector import SemanticSimilarityExampleSelector
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

# 创建选择器
selector = SemanticSimilarityExampleSelector.from_examples(
    examples=[
        {"input": "2+2", "output": "4"},
        {"input": "2+3", "output": "5"},
        {"input": "What is the moon?", "output": "A celestial body"}
    ],
    embeddings=OpenAIEmbeddings(),
    vectorstore_cls=Chroma,
    k=2  # 选择 2 个最相关的示例
)

# 使用选择器
selected = selector.select_examples({"input": "3+3"})
# 结果：自动选择数学相关的示例
```

### 4.2 核心难点

#### 难点 1：理解语义相似度选择的原理（⭐⭐⭐⭐）

**难在哪里？**
- 涉及 Embeddings 和向量相似度
- 需要理解余弦相似度计算
- 需要理解 VectorStore 的作用

**突破方法：**
```python
# 手动实现语义相似度选择（理解原理）
import numpy as np
from openai import OpenAI

client = OpenAI()

def find_similar(examples, query, k=3):
    """手动实现语义相似度选择"""
    # 1. 提取所有示例的输入 + 查询
    inputs = [e["input"] for e in examples] + [query]

    # 2. 生成 embeddings
    response = client.embeddings.create(
        input=inputs,
        model="text-embedding-3-small"
    )
    vectors = np.array([e.embedding for e in response.data])

    # 3. 计算余弦相似度
    query_vector = vectors[-1]
    example_vectors = vectors[:-1]
    similarities = example_vectors.dot(query_vector)

    # 4. 返回 top-k 最相似的示例
    top_k_indices = np.argsort(-similarities)[:k]
    return [examples[i] for i in top_k_indices]
```

#### 难点 2：混合策略的实现（⭐⭐⭐⭐）

**难在哪里？**
- 需要同时考虑相关性和长度
- LangChain 原生不支持混合策略
- 需要使用社区扩展或自定义实现

**突破方法：**
```python
# 方法 1：使用社区扩展项目（TypeScript）
from langchain_semantic_length_example_selector import SemanticLengthExampleSelector

selector = SemanticLengthExampleSelector(
    vectorstore=vectorstore,
    k=5,  # 语义相似度选择 5 个
    max_length=2000  # 长度限制
)

# 方法 2：自定义实现（Python）
class CustomExampleSelector(BaseExampleSelector):
    def select_examples(self, input_variables):
        # 1. 先用语义相似度选择
        semantic_examples = self.semantic_selector.select_examples(input_variables)

        # 2. 再用长度限制过滤
        filtered_examples = []
        total_length = 0
        for example in semantic_examples:
            example_length = len(example["input"]) + len(example["output"])
            if total_length + example_length <= self.max_length:
                filtered_examples.append(example)
                total_length += example_length

        return filtered_examples
```

#### 难点 3：性能优化（⭐⭐⭐）

**难在哪里？**
- Embeddings 计算成本高
- 向量相似度搜索耗时
- 需要缓存策略

**突破方法：**
```python
# 1. 缓存 Embeddings
from functools import lru_cache

@lru_cache(maxsize=1000)
def get_embedding(text):
    return embeddings.embed_query(text)

# 2. 使用更快的向量存储
# FAISS（内存中，速度快）
from langchain.vectorstores import FAISS
vectorstore = FAISS.from_texts(texts, embeddings)

# 3. 批量处理
selector.add_examples(examples)  # 批量添加
```

## 七、常见误区与陷阱

### 误区 1：k 值越大越好 ❌

**错误观点：**
"选择更多示例（k=10 或更多）会让模型理解更好"

**为什么错？**
- 过多示例会引入噪音
- 增加 token 成本
- 可能超过 context window

**正确理解：**
k=3-5 通常效果最好，需要根据场景调整。

[来源: reference/search_example_selector_02.md | 常见误区]

### 误区 2：不考虑 token 限制 ❌

**错误观点：**
"只关注相关性，不考虑长度"

**为什么错？**
- 可能超过模型的 context window
- 增加 API 成本
- 影响响应速度

**正确理解：**
结合长度限制或使用混合策略。

[来源: reference/search_example_selector_02.md | 常见误区]

### 误区 3：不缓存 embeddings ❌

**错误观点：**
"每次都重新计算 embeddings"

**为什么错？**
- 重复计算浪费时间和成本
- 影响性能
- 增加 API 调用次数

**正确理解：**
缓存示例的 embeddings，只计算一次。

[来源: reference/search_example_selector_02.md | 常见误区]
