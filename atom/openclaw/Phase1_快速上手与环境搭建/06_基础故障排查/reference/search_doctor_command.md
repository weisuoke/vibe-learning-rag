## 核心功能总结

### doctor 命令的主要用途
1. **健康检查** - 验证配置和服务状态
2. **自动修复** - 使用 --repair 或 --fix 标志
3. **配置迁移** - 升级后的配置迁移
4. **配置清理** - 删除无效配置键
5. **配置备份** - 修复前自动备份

### 常用选项
- `openclaw doctor` - 基础健康检查
- `openclaw doctor --repair` - 自动修复
- `openclaw doctor --fix` - 修复别名
- `openclaw doctor --deep` - 深度诊断

### 使用场景
1. 升级后验证
2. 安装后检查
3. 配置错误修复
4. 服务冲突解决
5. 权限问题诊断

### 社区最佳实践
