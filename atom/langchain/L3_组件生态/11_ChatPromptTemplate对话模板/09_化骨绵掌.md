## 一、核心概念速查卡

### 卡片 1: 三大核心特征

```
┌─────────────────────────────────────────────┐
│ ChatPromptTemplate 核心三要素                │
├─────────────────────────────────────────────┤
│ 1. 多角色消息模板                            │
│    - System/Human/AI/Chat 四种角色           │
│    - 输出消息对象列表，非字符串              │
│    - 结构化对话管理                          │
│                                             │
│ 2. 历史占位符 (MessagesPlaceholder)          │
│    - 动态注入对话历史                        │
│    - optional=True 支持首次对话              │
│    - n_messages 限制历史长度                 │
│                                             │
│ 3. 标准化接口                                │
│    - LCEL 管道操作符 |                       │
│    - 与 ChatModel/Memory/Agent 无缝集成      │
│    - 可组合、可复用、可测试                  │
└─────────────────────────────────────────────┘
```

---

### 卡片 2: 四种消息角色

```
┌──────────────────────────────────────────────────────┐
│ 消息角色类型                                          │
├──────────────────────────────────────────────────────┤
│ SystemMessage                                        │
│ - 定义 AI 角色和行为规则                             │
│ - 设置任务背景和约束                                 │
│ - 优先级最高，通常放在最前面                         │
│                                                      │
│ HumanMessage                                         │
│ - 用户输入和问题                                     │
│ - 支持文本和多模态（文本+图片）                      │
│ - 别名: UserMessage                                  │
│                                                      │
│ AIMessage                                            │
│ - AI 回复和 Few-shot 示例                            │
│ - 引导 LLM 的回答格式                                │
│ - 别名: AssistantMessage                             │
│                                                      │
│ ChatMessage                                          │
│ - 自定义角色（如 tool, function）                    │
│ - 灵活扩展对话场景                                   │
│ - 需要指定 role 参数                                 │
└──────────────────────────────────────────────────────┘
```

---

### 卡片 3: MessagesPlaceholder 参数

```
┌──────────────────────────────────────────────────────┐
│ MessagesPlaceholder 核心参数                          │
├──────────────────────────────────────────────────────┤
│ variable_name: str                                   │
│ - 占位符的变量名                                     │
│ - 调用时通过此名称传入消息列表                       │
│                                                      │
│ optional: bool = False                               │
│ - 是否可选（默认必需）                               │
│ - True: 可以不提供值                                 │
│ - False: 必须提供值，否则报错                        │
│                                                      │
│ n_messages: int | None = None                        │
│ - 限制消息数量                                       │
│ - 取最后 N 条消息（不是前 N 条！）                   │
│ - None: 不限制                                       │
└──────────────────────────────────────────────────────┘
```

## 二、对比速查表

### 表 1: ChatPromptTemplate vs PromptTemplate

| 维度 | PromptTemplate | ChatPromptTemplate |
|------|----------------|-------------------|
| **输出格式** | StringPromptValue | ChatPromptValue |
| **输出内容** | 字符串 | 消息对象列表 |
| **角色支持** | 无 | System/Human/AI/Chat |
| **对话历史** | 手动拼接 | MessagesPlaceholder |
| **适用场景** | 文本生成、简单提示 | 对话系统、RAG、Agent |
| **LLM 集成** | 需要转换 | 直接使用 |
| **变量替换** | 支持 | 支持 |
| **模板组合** | 不支持 | 支持 + 操作符 |
| **Few-shot** | 手动拼接 | AI 角色消息 |
| **多模态** | 不支持 | 支持（文本+图片） |

---

### 表 2: 消息格式对比

| 格式 | 语法 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **元组格式** | `("human", "Hello")` | 简洁易读 | 功能有限 | 简单对话 |
| **消息对象** | `HumanMessage(content="Hello")` | 功能完整 | 代码较长 | 固定消息 |
| **模板对象** | `HumanMessagePromptTemplate.from_template("{input}")` | 灵活强大 | 最复杂 | 复杂模板 |

---

### 表 3: 创建方法对比

| 方法 | 语法 | 输出 | 适用场景 |
|------|------|------|----------|
| **from_messages()** | `ChatPromptTemplate.from_messages([...])` | 多角色对话 | 复杂对话、RAG、Agent |
| **from_template()** | `ChatPromptTemplate.from_template("...")` | 单个 Human 消息 | 简单问答 |
| **构造函数** | `ChatPromptTemplate(messages=[...])` | 自定义配置 | 动态构建 |

## 三、API 速查表

### 核心 API

```python
# 1. 创建模板
ChatPromptTemplate.from_messages(messages: List)
ChatPromptTemplate.from_template(template: str)

# 2. 格式化
prompt.invoke(kwargs: Dict) -> ChatPromptValue
prompt.format_messages(**kwargs) -> List[BaseMessage]

# 3. 组合
prompt1 + prompt2  # 合并两个模板
prompt.append(message)  # 追加单个消息
prompt.extend(messages)  # 追加多个消息

# 4. 预填充
prompt.partial(**kwargs) -> ChatPromptTemplate

# 5. 属性
prompt.input_variables  # 输入变量列表
prompt.messages  # 消息模板列表
```

---

### MessagesPlaceholder API

```python
# 创建
MessagesPlaceholder(
    variable_name: str,
    optional: bool = False,
    n_messages: int | None = None
)

# 使用
prompt.invoke({
    "variable_name": [
        ("human", "Hello"),
        ("ai", "Hi!")
    ]
})
```

---

### LCEL 集成

```python
# 基础链
chain = prompt | llm

# 带输出解析
chain = prompt | llm | output_parser

# 带历史管理
from langchain_core.runnables.history import RunnableWithMessageHistory

conversation = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history"
)
```

## 四、常用模式速查

### 模式 1: 基础问答

```python
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    ("human", "{question}")
])
```

**适用**: 简单问答、信息查询

---

### 模式 2: 带历史对话

```python
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a helpful assistant."),
    MessagesPlaceholder("history", optional=True),
    ("human", "{input}")
])
```

**适用**: 聊天机器人、客服系统

## 六、反模式警示

### 反模式 1: 混用 PromptTemplate

```python
# ❌ 错误
prompt = PromptTemplate.from_template("System: ...\nHuman: {input}")
chain = prompt | ChatOpenAI()

# ✅ 正确
prompt = ChatPromptTemplate.from_messages([
    ("system", "..."),
    ("human", "{input}")
])
chain = prompt | ChatOpenAI()
```

---

### 反模式 2: 忘记 optional=True

```python
# ❌ 错误：首次对话会报错
prompt = ChatPromptTemplate.from_messages([
    ("system", "..."),
    MessagesPlaceholder("history"),  # 必需
    ("human", "{input}")
])

# ✅ 正确
prompt = ChatPromptTemplate.from_messages([
    ("system", "..."),
    MessagesPlaceholder("history", optional=True),
    ("human", "{input}")
])
```

---

### 反模式 3: 误解 n_messages

```python
# ❌ 错误理解：以为取前 5 条
MessagesPlaceholder("history", n_messages=5)

# ✅ 正确理解：取最后 5 条
# history[-5:]
```

---

### 反模式 4: 组合前使用 partial

```python
# ❌ 可能有问题
prompt1 = ChatPromptTemplate.from_messages([
    ("system", "You are {role}")
]).partial(role="assistant")

prompt2 = ChatPromptTemplate.from_messages([
    ("human", "{input}")
])

combined = prompt1 + prompt2  # partial 可能丢失

# ✅ 更安全
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are {role}"),
    ("human", "{input}")
]).partial(role="assistant")
```

## 七、故障排查指南

### 问题 1: KeyError: 'variable_name'

**原因**: 必需变量未提供

**解决**:
```python
# 方案 1: 提供变量
prompt.invoke({"variable_name": value})

# 方案 2: 使用 optional=True
MessagesPlaceholder("variable_name", optional=True)
```

---

### 问题 2: partial_variables 不生效

**原因**: 版本兼容性问题 (v0.1.9)

**解决**:
```python
# 方案 1: 升级到 v0.2.x+
pip install langchain-core>=0.2.0

# 方案 2: 直接传值
prompt.invoke({"role": "assistant", "input": "..."})

# 方案 3: 使用函数
prompt.partial(time=lambda: datetime.now())
```

---

### 问题 3: 历史消息格式错误

**原因**: 消息格式不正确

**解决**:
```python
# ✅ 支持的格式
history = [
    ("human", "Hello"),  # 元组
    HumanMessage(content="Hello"),  # 消息对象
    {"role": "human", "content": "Hello"}  # 字典
]
```

---

### 问题 4: 上下文窗口溢出

**原因**: 历史消息太长

**解决**:
```python
# 方案 1: 使用 n_messages
MessagesPlaceholder("history", n_messages=10)

# 方案 2: 手动截取
history = history[-10:]

# 方案 3: 使用摘要
from langchain.memory import ConversationSummaryMemory
```

## 十、快速决策树

### 决策 1: 选择模板类型

```
需要对话系统？
├─ 是 → ChatPromptTemplate
└─ 否 → PromptTemplate
```

---

### 决策 2: 选择创建方法

```
需要多角色？
├─ 是 → from_messages()
└─ 否 → from_template()
```

---

### 决策 3: 是否需要历史

```
需要记住上下文？
├─ 是 → 使用 MessagesPlaceholder
│   └─ 首次对话？
│       ├─ 是 → optional=True
│       └─ 否 → optional=False
└─ 否 → 不使用 MessagesPlaceholder
```

---

### 决策 4: 历史长度控制

```
历史消息很长？
├─ 是 → 使用 n_messages 或摘要
└─ 否 → 不限制
```
