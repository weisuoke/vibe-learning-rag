## 4.1 节点返回部分字段

### 核心概念

**节点函数只需要返回需要更新的字段，未返回的字段自动保持原值。**

这是部分状态更新最基础的用法，也是最常用的模式。

[来源: reference/context7_langgraph_01.md]

### 基础语法

```python
from typing import TypedDict

class State(TypedDict):
    field_a: str
    field_b: int
    field_c: list[str]

def my_node(state: State) -> dict:
    # 只返回需要更新的字段
    return {"field_b": state["field_b"] + 1}
    # field_a 和 field_c 保持不变
```

**关键点：**
- 返回值是字典（dict）
- 只包含需要更新的字段
- 未返回的字段保持原值

[来源: reference/source_部分状态更新_01.md]

### 实际应用示例

```python
"""
最小可用示例 1：节点返回部分字段
演示：多步处理工作流，每个节点只更新自己负责的字段
"""

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# 定义状态
class ProcessState(TypedDict):
    input_text: str
    cleaned_text: str
    word_count: int
    summary: str

# 节点 1：清理文本
def clean_text(state: ProcessState) -> dict:
    cleaned = state["input_text"].strip().lower()
    return {"cleaned_text": cleaned}
    # input_text, word_count, summary 保持不变

# 节点 2：统计词数
def count_words(state: ProcessState) -> dict:
    words = state["cleaned_text"].split()
    return {"word_count": len(words)}
    # input_text, cleaned_text, summary 保持不变

# 节点 3：生成摘要
def generate_summary(state: ProcessState) -> dict:
    summary = f"Text has {state['word_count']} words"
    return {"summary": summary}
    # input_text, cleaned_text, word_count 保持不变

# 构建图
graph = StateGraph(ProcessState)
graph.add_node("clean", clean_text)
graph.add_node("count", count_words)
graph.add_node("summarize", generate_summary)
graph.add_edge(START, "clean")
graph.add_edge("clean", "count")
graph.add_edge("count", "summarize")
graph.add_edge("summarize", END)

app = graph.compile()

# 运行
result = app.invoke({
    "input_text": "  Hello World  ",
    "cleaned_text": "",
    "word_count": 0,
    "summary": ""
})

print("=== 最终状态 ===")
print(f"原始文本: {result['input_text']}")
print(f"清理后: {result['cleaned_text']}")
print(f"词数: {result['word_count']}")
print(f"摘要: {result['summary']}")
```

**运行输出：**
```
=== 最终状态 ===
原始文本:   Hello World
清理后: hello world
词数: 2
摘要: Text has 2 words
```

**为什么这个示例重要？**
- 每个节点职责清晰，只关注自己负责的字段
- 不需要手动复制状态
- 避免状态丢失
- 代码简洁易维护

[来源: reference/context7_langgraph_01.md]

## 4.2 使用 Annotated 定义 Reducer

### 核心概念

**使用 `Annotated[type, reducer]` 定义字段的更新策略，实现增量合并而非覆盖。**

这是部分状态更新的核心机制，决定了如何合并当前值和新值。

[来源: reference/context7_langgraph_01.md, reference/source_部分状态更新_01.md]

### 基础语法

```python
from typing import Annotated, TypedDict
import operator

class State(TypedDict):
    # 使用 Reducer：追加到列表
    items: Annotated[list[str], operator.add]
    # 没有 Reducer：覆盖原值
    counter: int
```

**关键点：**
- `Annotated[type, reducer]` 语法
- Reducer 接收两个参数：当前值和新值
- 返回合并后的值
- 没有 Reducer 的字段使用覆盖策略

[来源: reference/source_部分状态更新_01.md]

### Reducer 工作原理

```python
# Reducer 函数签名
def reducer(current_value, new_value):
    return merged_value

# operator.add 的行为
# 对于列表：[1, 2] + [3, 4] = [1, 2, 3, 4]
# 对于数值：5 + 3 = 8
# 对于字符串："hello" + " world" = "hello world"
```

[来源: reference/source_部分状态更新_01.md]

### 实际应用示例

```python
"""
最小可用示例 2：使用 Annotated 定义 Reducer
演示：累加数值、追加列表
"""

from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

# 定义状态（使用 Reducer）
class CounterState(TypedDict):
    # 使用 operator.add 累加数值
    total: Annotated[int, operator.add]
    # 使用 operator.add 追加列表
    items: Annotated[list[str], operator.add]
    # 没有 Reducer，使用覆盖
    last_item: str

# 节点 1：处理 item A
def process_a(state: CounterState) -> dict:
    return {
        "total": 10,        # 累加到 total
        "items": ["A"],     # 追加到 items
        "last_item": "A"    # 覆盖 last_item
    }

# 节点 2：处理 item B
def process_b(state: CounterState) -> dict:
    return {
        "total": 20,        # 累加到 total
        "items": ["B"],     # 追加到 items
        "last_item": "B"    # 覆盖 last_item
    }

# 节点 3：处理 item C
def process_c(state: CounterState) -> dict:
    return {
        "total": 30,        # 累加到 total
        "items": ["C"],     # 追加到 items
        "last_item": "C"    # 覆盖 last_item
    }

# 构建图
graph = StateGraph(CounterState)
graph.add_node("a", process_a)
graph.add_node("b", process_b)
graph.add_node("c", process_c)
graph.add_edge(START, "a")
graph.add_edge("a", "b")
graph.add_edge("b", "c")
graph.add_edge("c", END)

app = graph.compile()

# 运行
result = app.invoke({
    "total": 0,
    "items": [],
    "last_item": ""
})

print("=== 最终状态 ===")
print(f"总计: {result['total']}")           # 0 + 10 + 20 + 30 = 60
print(f"项目列表: {result['items']}")        # [] + ["A"] + ["B"] + ["C"] = ["A", "B", "C"]
print(f"最后一项: {result['last_item']}")    # "C" (覆盖)
```

**运行输出：**
```
=== 最终状态 ===
总计: 60
项目列表: ['A', 'B', 'C']
最后一项: C
```

**为什么这个示例重要？**
- 展示了 Reducer 的累加效果
- 对比了覆盖和合并策略
- 适用于计数、日志收集等场景

[来源: reference/context7_langgraph_01.md]

## 4.3 使用 operator.add 实现列表追加

### 核心概念

**`operator.add` 是最常用的 Reducer，用于累加数值、追加列表、拼接字符串。**

这是实际应用中最常见的场景，掌握这个就能解决大部分问题。

[来源: reference/context7_langgraph_01.md]

### 基础语法

```python
from typing import Annotated
import operator

class State(TypedDict):
    # 数值累加
    count: Annotated[int, operator.add]
    # 列表追加
    logs: Annotated[list[str], operator.add]
    # 字符串拼接
    text: Annotated[str, operator.add]
```

**operator.add 的行为：**
- 数值：`5 + 3 = 8`
- 列表：`[1, 2] + [3, 4] = [1, 2, 3, 4]`
- 字符串：`"hello" + " world" = "hello world"`

[来源: reference/source_部分状态更新_01.md]

### 实际应用示例

```python
"""
最小可用示例 3：使用 operator.add 实现列表追加
演示：多步推理工作流，收集每一步的思考过程
"""

from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

# 定义状态
class ReasoningState(TypedDict):
    question: str
    # 使用 operator.add 追加思考步骤
    thoughts: Annotated[list[str], operator.add]
    answer: str

# 节点 1：分析问题
def analyze(state: ReasoningState) -> dict:
    thought = f"分析问题：{state['question']}"
    return {"thoughts": [thought]}

# 节点 2：推理过程
def reason(state: ReasoningState) -> dict:
    thought = "推理：基于已有信息进行逻辑推导"
    return {"thoughts": [thought]}

# 节点 3：得出结论
def conclude(state: ReasoningState) -> dict:
    thought = "结论：综合以上分析得出答案"
    all_thoughts = "\n".join(state["thoughts"])
    answer = f"基于以下思考：\n{all_thoughts}\n答案是：LangGraph 是一个状态化工作流框架"
    return {
        "thoughts": [thought],
        "answer": answer
    }

# 构建图
graph = StateGraph(ReasoningState)
graph.add_node("analyze", analyze)
graph.add_node("reason", reason)
graph.add_node("conclude", conclude)
graph.add_edge(START, "analyze")
graph.add_edge("analyze", "reason")
graph.add_edge("reason", "conclude")
graph.add_edge("conclude", END)

app = graph.compile()

# 运行
result = app.invoke({
    "question": "什么是 LangGraph？",
    "thoughts": [],
    "answer": ""
})

print("=== 推理过程 ===")
for i, thought in enumerate(result["thoughts"], 1):
    print(f"{i}. {thought}")

print("\n=== 最终答案 ===")
print(result["answer"])
```

**运行输出：**
```
=== 推理过程 ===
1. 分析问题：什么是 LangGraph？
2. 推理：基于已有信息进行逻辑推导
3. 结论：综合以上分析得出答案

=== 最终答案 ===
基于以下思考：
分析问题：什么是 LangGraph？
推理：基于已有信息进行逻辑推导
答案是：LangGraph 是一个状态化工作流框架
```

**为什么这个示例重要？**
- 展示了多步推理的典型模式
- 每个节点追加自己的思考步骤
- 适用于 Chain-of-Thought、Agent 推理等场景

[来源: reference/context7_langgraph_01.md]

## 4.4 使用 add_messages 维护对话历史

### 核心概念

**`add_messages` 是专门用于消息列表的 Reducer，用于维护对话历史。**

这是 LangGraph 中最常用的 Reducer，几乎所有对话式应用都会用到。

[来源: reference/context7_langgraph_01.md]

### 基础语法

```python
from typing import Annotated, Sequence, TypedDict
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages

class State(TypedDict):
    # 使用 add_messages 维护对话历史
    messages: Annotated[Sequence[BaseMessage], add_messages]
```

**add_messages 的特点：**
- 专门用于消息列表
- 追加新消息而不是替换
- 支持消息去重和更新
- 用于维护对话历史

[来源: reference/context7_langgraph_01.md]

### 实际应用示例

```python
"""
最小可用示例 4：使用 add_messages 维护对话历史
演示：简单的对话机器人
"""

from typing import Annotated, Sequence, TypedDict
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage
from langgraph.graph.message import add_messages
from langgraph.graph import StateGraph, START, END

# 定义状态
class ChatState(TypedDict):
    # 使用 add_messages 维护对话历史
    messages: Annotated[Sequence[BaseMessage], add_messages]

# 聊天节点
def chatbot(state: ChatState) -> dict:
    # 获取最后一条用户消息
    last_message = state["messages"][-1]

    # 生成回复
    if isinstance(last_message, HumanMessage):
        response = f"你说：{last_message.content}"
    else:
        response = "我不明白"

    # 返回新消息（会追加到历史中）
    return {"messages": [AIMessage(content=response)]}

# 构建图
graph = StateGraph(ChatState)
graph.add_node("chatbot", chatbot)
graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)

app = graph.compile()

# 第一轮对话
print("=== 第一轮对话 ===")
result1 = app.invoke({
    "messages": [HumanMessage(content="你好")]
})
for msg in result1["messages"]:
    role = "用户" if isinstance(msg, HumanMessage) else "AI"
    print(f"{role}: {msg.content}")

# 第二轮对话（继续追加）
print("\n=== 第二轮对话 ===")
result2 = app.invoke({
    "messages": result1["messages"] + [HumanMessage(content="再见")]
})
for msg in result2["messages"]:
    role = "用户" if isinstance(msg, HumanMessage) else "AI"
    print(f"{role}: {msg.content}")
```

**运行输出：**
```
=== 第一轮对话 ===
用户: 你好
AI: 你说：你好

=== 第二轮对话 ===
用户: 你好
AI: 你说：你好
用户: 再见
AI: 你说：再见
```

**为什么这个示例重要？**
- 展示了对话历史的维护
- 每次调用都追加新消息
- 适用于聊天机器人、对话式 Agent 等场景

[来源: reference/context7_langgraph_01.md]

## 4.5 理解覆盖 vs 合并策略

### 核心概念

**没有 Reducer 的字段使用覆盖策略，有 Reducer 的字段使用合并策略。**

理解这个区别是正确使用部分状态更新的关键。

[来源: reference/context7_langgraph_01.md, reference/source_部分状态更新_01.md]

### 对比表格

| 特性 | 覆盖策略 | 合并策略 |
|------|---------|---------|
| 定义方式 | `field: type` | `field: Annotated[type, reducer]` |
| 更新行为 | 直接替换原值 | 使用 Reducer 合并 |
| 适用场景 | 状态标志、配置项 | 累加、追加、合并 |
| 示例 | `counter: int` | `counter: Annotated[int, operator.add]` |

[来源: reference/context7_langgraph_01.md]

### 实际应用示例

```python
"""
最小可用示例 5：理解覆盖 vs 合并策略
演示：对比两种策略的行为差异
"""

from typing import Annotated, TypedDict
import operator
from langgraph.graph import StateGraph, START, END

# 定义状态
class CompareState(TypedDict):
    # 覆盖策略
    status: str
    # 合并策略
    logs: Annotated[list[str], operator.add]

# 节点 1
def step1(state: CompareState) -> dict:
    return {
        "status": "step1_done",
        "logs": ["执行 step1"]
    }

# 节点 2
def step2(state: CompareState) -> dict:
    return {
        "status": "step2_done",  # 覆盖 step1_done
        "logs": ["执行 step2"]   # 追加到 logs
    }

# 节点 3
def step3(state: CompareState) -> dict:
    return {
        "status": "step3_done",  # 覆盖 step2_done
        "logs": ["执行 step3"]   # 追加到 logs
    }

# 构建图
graph = StateGraph(CompareState)
graph.add_node("step1", step1)
graph.add_node("step2", step2)
graph.add_node("step3", step3)
graph.add_edge(START, "step1")
graph.add_edge("step1", "step2")
graph.add_edge("step2", "step3")
graph.add_edge("step3", END)

app = graph.compile()

# 运行
result = app.invoke({
    "status": "init",
    "logs": []
})

print("=== 最终状态 ===")
print(f"状态（覆盖）: {result['status']}")  # 只保留最后一个值
print(f"日志（合并）: {result['logs']}")    # 保留所有值
```

**运行输出：**
```
=== 最终状态 ===
状态（覆盖）: step3_done
日志（合并）: ['执行 step1', '执行 step2', '执行 step3']
```

**为什么这个示例重要？**
- 清晰展示了两种策略的差异
- 帮助理解何时使用哪种策略
- 避免常见的误用

[来源: reference/context7_langgraph_01.md]
