## 1. 理解 Embeddings 接口的两个方法

**核心概念**：LangChain 将 Embedding 分为两个操作：
- `embed_documents()` - 批量嵌入文档（用于索引构建）
- `embed_query()` - 单个查询嵌入（用于检索）

**为什么分离？** 某些模型对文档和查询使用不同的 prompt，允许针对性优化。

**代码示例**：
```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 批量嵌入文档（索引阶段）
docs = ["Python 是编程语言", "JavaScript 用于前端开发"]
doc_vectors = embeddings.embed_documents(docs)
print(f"文档向量维度: {len(doc_vectors[0])}")  # 512

# 单个查询嵌入（检索阶段）
query = "什么是编程语言？"
query_vector = embeddings.embed_query(query)
print(f"查询向量维度: {len(query_vector)}")  # 512
```

**实际应用**：RAG 系统中，文档在离线阶段批量嵌入存储，查询在线实时嵌入后检索。

## 2. 选择合适的 Embedding 模型

**两大方案对比**：

| 方案 | 模型 | 优点 | 缺点 | 场景 |
|------|------|------|------|------|
| **OpenAI** | text-embedding-3-small (512维) | 效果好、维护少 | 需要 API Key、有成本 | 生产环境、对质量要求高 |
| **本地** | all-MiniLM-L6-v2 (384维) | 免费、隐私、快速 | 效果一般、需要 GPU | 开发、内部系统 |

**代码示例**：
```python
# 方案 1: OpenAI（推荐生产环境）
from langchain_openai import OpenAIEmbeddings
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 方案 2: 本地模型（推荐开发环境）
from langchain_community.embeddings import HuggingFaceEmbeddings
embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")
```

**实际应用**：开发时用本地模型快速迭代，生产环境切换到 OpenAI 获得更好效果。

## 3. 批量处理与性能优化

**核心要点**：使用 `batch()` 方法和并发控制提高效率。

**代码示例**：
```python
from langchain_openai import OpenAIEmbeddings

embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 批量嵌入 1000 个文档
documents = [f"文档 {i}" for i in range(1000)]

# 方式 1: 直接调用（自动批处理）
vectors = embeddings.embed_documents(documents)

# 方式 2: 使用 batch() 控制并发
vectors = embeddings.batch(
    documents,
    config={"max_concurrency": 5}  # 限制并发数避免速率限制
)

print(f"生成了 {len(vectors)} 个向量")
```

**实际应用**：大规模文档索引时，合理设置 `max_concurrency` 避免 API 限流。

## 4. 在 RAG 中的完整流程

**核心流程**：文档索引 → 向量存储 → 查询检索 → 生成答案

**代码示例**：
```python
from langchain_openai import OpenAIEmbeddings
from langchain_community.vectorstores import Chroma

# 初始化 Embeddings
embeddings = OpenAIEmbeddings(model="text-embedding-3-small")

# 1. 文档索引阶段（离线）
documents = [
    "Python 是一种高级编程语言",
    "JavaScript 主要用于前端开发",
    "Go 语言以并发性能著称"
]
vector_store = Chroma.from_texts(documents, embeddings)

# 2. 查询检索阶段（在线）
query = "哪种语言适合并发编程？"
results = vector_store.similarity_search(query, k=2)

for doc in results:
    print(f"相关文档: {doc.page_content}")
```

**实际应用**：完整的知识库问答系统，用户提问 → 检索相关文档 → 生成答案。
