## 30 字核心

> **动态分发通过 `dyn Trait` 胖指针实现运行时多态，`Box<dyn T>` 和 `Arc<dyn T>` 是 ZeroClaw 可插拔架构的基石。**

这句话浓缩了 Rust 运行时多态的全部要点。下面逐词拆解。

---

## 一、关键词拆解

### 1.1 动态分发（Dynamic Dispatch）

"分发"指的是**调用方法时，如何找到要执行的那个具体函数**。

- **静态分发**：编译时就确定调用哪个函数（泛型单态化，上一节学的）
- **动态分发**：运行时通过函数指针表（vtable）查找要调用的函数

```rust
// 静态分发 -- 编译时确定
fn greet(tool: &impl Tool) { /* 编译器知道 tool 的具体类型 */ }

// 动态分发 -- 运行时确定
fn greet(tool: &dyn Tool) { /* 编译器不知道具体类型，通过 vtable 查找 */ }
```

### 1.2 `dyn Trait` 胖指针

`dyn Trait` 是一个**胖指针**（fat pointer），包含两个指针：

```
普通指针（瘦指针）：
┌──────────────┐
│ data pointer │  →  指向数据
└──────────────┘
     8 字节

dyn Trait（胖指针）：
┌──────────────┐
│ data pointer │  →  指向数据（ShellTool 实例）
├──────────────┤
│ vtable ptr   │  →  指向方法表（name/execute/...函数指针）
└──────────────┘
     16 字节
```

**类型擦除**：`dyn Tool` 不再知道具体类型是 `ShellTool` 还是 `FileTool`，只知道它「能做 Tool 能做的事」。具体是谁，由 vtable 中的函数指针决定。

### 1.3 运行时多态

"运行时多态"指的是**同一段代码，运行时可以调用不同的实现**：

```rust
fn run_tool(tool: &dyn Tool, args: Value) -> Result<ToolResult> {
    // 这一行代码可能调用 ShellTool::execute，也可能调用 FileTool::execute
    // 取决于 tool 实际指向的对象
    tool.execute(args)
}
```

### 1.4 `Box<dyn T>` -- 堆上的唯一拥有者

`Box<dyn Tool>` = `Box`（堆分配 + 单一所有权） + `dyn Tool`（类型擦除 + vtable）。

**用途**：当一个值只有一个拥有者时使用。ZeroClaw 中 Agent 独占其 Provider。

### 1.5 `Arc<dyn T>` -- 线程安全的共享拥有者

`Arc<dyn Memory>` = `Arc`（引用计数 + 线程安全共享） + `dyn Memory`（类型擦除 + vtable）。

**用途**：当多个地方需要共享同一个实例时使用。ZeroClaw 中多个 MemoryTool 共享同一个 Memory 后端。

### 1.6 可插拔架构的基石

"可插拔"意味着**不改代码，只改配置，就能切换实现**。`Box<dyn Provider>` 让 Agent 不需要知道自己用的是 Anthropic 还是 Ollama，只需要知道它"能提供 LLM 对话能力"。

## 二、30 字核心的三层含义

### 第一层：它是什么（机制）

动态分发是一种**方法调用机制**。当你写 `tool.execute(args)` 时，编译器不在编译时确定调用哪个函数，而是在运行时通过 vtable 查表找到对应的函数指针，然后调用它。

```rust
// 编译器生成的伪代码（简化）
fn call_execute(tool: &dyn Tool, args: Value) -> Result<ToolResult> {
    let vtable = tool.vtable_ptr;              // 取出 vtable 指针
    let execute_fn = vtable[EXECUTE_INDEX];     // 从表中取出函数指针
    execute_fn(tool.data_ptr, args)             // 通过函数指针调用
}
```

### 第二层：为什么需要（问题）

静态分发（泛型）有一个根本限制：**编译时必须知道具体类型**。

```rust
// 静态分发 -- 编译时必须知道 T 是什么
fn use_tool<T: Tool>(tool: T) { /* ... */ }

// 问题：你不能把 ShellTool 和 FileTool 放进同一个 Vec
let tools: Vec<???> = vec![ShellTool::new(), FileTool::new()]; // 类型不同！

// 动态分发解决了这个问题
let tools: Vec<Box<dyn Tool>> = vec![
    Box::new(ShellTool::new()),   // ShellTool → Box<dyn Tool>
    Box::new(FileTool::new()),    // FileTool  → Box<dyn Tool>
];
```

此外，当类型取决于用户配置时（比如 config.toml 中写了 `provider = "anthropic"`），编译时根本无法确定类型，只能用动态分发。

### 第三层：ZeroClaw 如何应用（实践）

ZeroClaw 的四大 Trait 全部通过动态分发实现可插拔：

```rust
// Agent 结构体 -- 全部使用 dyn Trait
struct Agent {
    provider: Box<dyn Provider>,     // config.toml: provider = "anthropic"
    tools: Vec<Box<dyn Tool>>,       // 20+ 种工具动态组装
}

// Memory 共享 -- Arc<dyn Memory>
let memory: Arc<dyn Memory> = match config.memory_backend.as_str() {
    "sqlite"   => Arc::new(SqliteMemory::new()?),
    "postgres" => Arc::new(PostgresMemory::new()?),
    _          => Arc::new(NoneMemory::new()),
};
// 多个 MemoryTool 共享同一个 memory 实例
```

## 三、TypeScript 对照理解

```typescript
// ===== TypeScript：天然就是动态分发 =====
interface Tool {
  name(): string;
  execute(args: any): Promise<ToolResult>;
}

class ShellTool implements Tool {
  name() { return "shell"; }
  async execute(args: any) { /* ... */ }
}

// TS 中变量类型是接口，运行时多态是免费的
const tool: Tool = new ShellTool();  // 无需 Box，JS 引擎自动处理
const tools: Tool[] = [new ShellTool(), new FileTool()];  // 无需 Box

// 共享引用也是免费的
const memory = new SqliteMemory();
const store = new MemoryStoreTool(memory);   // 自动共享引用
const recall = new MemoryRecallTool(memory); // 自动共享引用
```

```rust
// ===== Rust：需要显式声明动态分发 =====
// Box<dyn Tool> 对应 TS 的 tool: Tool
let tool: Box<dyn Tool> = Box::new(ShellTool::new());

// Vec<Box<dyn Tool>> 对应 TS 的 tools: Tool[]
let tools: Vec<Box<dyn Tool>> = vec![
    Box::new(ShellTool::new()),
    Box::new(FileTool::new()),
];

// Arc<dyn Memory> 对应 TS 的共享引用
let memory: Arc<dyn Memory> = Arc::new(SqliteMemory::new()?);
let store = MemoryStoreTool::new(memory.clone());   // 显式 clone 引用计数
let recall = MemoryRecallTool::new(memory.clone()); // 显式 clone 引用计数
```

**一句话对比**：TypeScript 默认就是动态分发（JS 引擎在底层处理），Rust 默认是静态分发，需要用 `dyn` 关键字显式选择动态分发。

## 四、ZeroClaw 中的体现

```rust
// Agent 核心结构 -- 动态分发的集大成者
struct Agent {
    // ① Box<dyn Provider> -- 独占一个 LLM 提供者
    //    可能是 Anthropic、OpenAI、Ollama...由 config.toml 决定
    provider: Box<dyn Provider>,

    // ② Vec<Box<dyn Tool>> -- 异构工具集合
    //    ShellTool、FileTool、SearchTool...不同类型放一个 Vec
    tools: Vec<Box<dyn Tool>>,
}

// 工具注册表 -- 工厂模式 + Box<dyn Tool>
pub fn default_tools_with_runtime(
    security: Arc<SecurityPolicy>,
    runtime: Arc<dyn RuntimeAdapter>,
) -> Vec<Box<dyn Tool>> {
    vec![
        Box::new(ShellTool::new(security.clone(), runtime)),
        Box::new(FileReadTool::new(security.clone())),
        Box::new(FileWriteTool::new(security.clone())),
        Box::new(FileEditTool::new(security.clone())),
        Box::new(GlobSearchTool::new(security.clone())),
        Box::new(ContentSearchTool::new(security)),
    ]
}

// Memory 共享 -- Arc<dyn Memory> 被多个工具共用
pub fn all_tools(memory: Arc<dyn Memory>, ...) -> Vec<Box<dyn Tool>> {
    vec![
        Box::new(MemoryStoreTool::new(memory.clone(), ...)),  // 共享
        Box::new(MemoryRecallTool::new(memory.clone())),      // 共享
        Box::new(MemoryForgetTool::new(memory, ...)),         // 最后一个不需要 clone
    ]
}
```

## 五、记忆口诀

> **`dyn` 擦类型，vtable 查方法。Box 独占，Arc 共享，配置驱动选实现。**

- `dyn` 擦类型：`dyn Tool` 擦除了 `ShellTool` / `FileTool` 的具体类型信息
- vtable 查方法：运行时通过函数指针表找到要调用的方法
- Box 独占：`Box<dyn Provider>` -- Agent 独占一个 Provider
- Arc 共享：`Arc<dyn Memory>` -- 多个 MemoryTool 共享一个 Memory
- 配置驱动选实现：config.toml 决定用哪个 Provider/Memory/Tool

---

## 总结

```
  动态分发通过 dyn Trait 胖指针实现运行时多态，
  Box<dyn T> 和 Arc<dyn T> 是 ZeroClaw 可插拔架构的基石。

  dyn Trait  ────→ 胖指针（data + vtable），类型擦除
  运行时多态 ───→ 同一接口，运行时调用不同实现
  Box<dyn T> ───→ 堆上单一所有权，用于工厂返回和异构集合
  Arc<dyn T> ───→ 线程安全共享，用于跨任务共享实例
  可插拔架构 ───→ config.toml 驱动，运行时动态组装

  ZeroClaw 因此实现：10 个核心 Trait | 58+ 个实现 | 配置驱动的运行时切换
```
