# 所有权与借用 —— 最小可用

> **一句话定位：** 掌握这 6 个关键点，你就能解决 80% 的所有权与借用编译错误，剩下的 20% 在实战中慢慢积累。

---

## 核心思想

Rust 的所有权系统初看令人畏惧，但真正高频使用的知识点只有一小部分。本文档提炼出 **6 个关键点**，它们构成了日常 Rust 编程中所有权相关操作的核心。

**给 TypeScript 开发者的定心丸：** 你在 JS/TS 中从来不需要想"这个对象谁来释放"，因为 GC 帮你搞定了。在 Rust 中，编译器帮你搞定——但它需要你通过类型签名把意图说清楚。本文档教你最高效的"说清楚"方式。

## 第一部分：20% 核心知识清单

### 关键点 1：记住所有权三规则

**口诀：一值一主人，转手即无权，出域即归零。**

这三条规则解决了 50% 的编译错误。每当编译器报所有权相关错误时，先回忆这三条：

```
规则 1：每个值有且仅有一个所有者
规则 2：赋值 / 传参会转移所有权（移动），原变量失效
规则 3：所有者离开作用域，值被自动释放（drop）
```

```rust
fn main() {
    let name = String::from("hello");  // name 是 "hello" 的所有者
    let name2 = name;                  // 所有权转移给 name2
    // println!("{}", name);           // 编译错误！name 已经无效
    println!("{}", name2);             // OK，name2 是当前所有者
}   // name2 离开作用域，"hello" 被释放
```

**关键区别：** TypeScript 中 `const b = a` 是共享引用（两个变量指向同一块内存），Rust 中是移动所有权（原变量失效）。这消除了"谁来释放"的歧义。

> **注意：** 基本类型（i32、f64、bool、char 等）实现了 `Copy` trait，赋值时会复制而非移动，和 TypeScript 原始类型行为一致。

### 关键点 2：函数参数用 & 借用

**80% 的情况用 `&T`（只读借用），这是最常见的参数传递方式。**

| 场景 | 用什么 | 含义 | 频率 |
|------|--------|------|------|
| 只需要读数据 | `&T` | 借来看看 | ~70% |
| 需要修改数据 | `&mut T` | 借来改改 | ~15% |
| 需要存储/消费数据 | `T` | 拿走不还 | ~15% |

```rust
// 只读：最常见
fn print_name(name: &str) {
    println!("Hello, {}!", name);
}

// 需要修改：加 mut
fn add_greeting(name: &mut String) {
    name.push_str(", welcome!");
}

// 需要存储：获取所有权（构造函数常用）
struct User { name: String }

impl User {
    fn new(name: String) -> Self { User { name } }
}

fn main() {
    let mut name = String::from("Alice");
    print_name(&name);           // 借用：name 仍然有效
    add_greeting(&mut name);     // 可变借用：name 被修改
    let user = User::new(name);  // 所有权转移：name 不再有效
    println!("{}", user.name);   // OK
}
```

**决策流程：** 只需要看看 → `&T` | 需要改一下 → `&mut T` | 需要存起来 → `T`

### 关键点 3：编译器报错时的三步解决法

```
步骤 1：看报错类型
  → "value used after move"      = 移动问题
  → "cannot borrow as mutable"   = 借用冲突
  → "does not live long enough"  = 生命周期问题

步骤 2：针对移动问题
  → 方案 A：加 & 改为借用（首选）
  → 方案 B：加 .clone() 复制一份（次选）

步骤 3：针对借用冲突
  → 调整代码顺序，让不可变借用先用完
  → 利用 NLL：引用在最后使用处自动结束
  → 把操作拆分到不同的作用域块 {}
```

**修复移动错误：**

```rust
// 错误：所有权被移走后还在用
let data = String::from("important");
process(data);
println!("{}", data);  // 编译错误！

// 修复 A：改为借用（首选）
fn process(s: &str) { println!("Processing: {}", s); }
process(&data);        // data 仍有效

// 修复 B：clone（初学时大胆用）
process(data.clone()); // 传副本，data 不受影响
```

**修复借用冲突：**

```rust
// 错误：不可变借用存在时试图可变借用
let mut numbers = vec![1, 2, 3];
let first = &numbers[0];  // 不可变借用
numbers.push(4);           // 可变借用——冲突！
println!("{}", first);

// 修复：让不可变借用先用完（NLL）
let first = &numbers[0];
println!("{}", first);  // first 的借用到此结束
numbers.push(4);        // 现在可以了
```

> **NLL（Non-Lexical Lifetimes）：** 引用的生命周期在最后一次使用处结束，而非块的末尾。

### 关键点 4：String vs &str 快速选择

**速记口诀：存用 String，看用 &str。**

| 场景 | 选什么 | 原因 |
|------|--------|------|
| 函数参数 | `&str` | 调用方传 String 或字面量都能用 |
| 函数返回值 | `String` | 需要拥有数据才能返回 |
| struct 字段 | `String` | struct 需要拥有自己的数据 |
| 字符串字面量 | `&str` | `"hello"` 的类型就是 `&str` |
| 需要修改字符串 | `String` | `&str` 是不可变的切片 |

```rust
// 参数用 &str（最通用），返回用 String
fn greet(name: &str) -> String {
    format!("Hello, {}!", name)
}

fn main() {
    let result1 = greet("Alice");           // 字面量 → OK
    let name = String::from("Bob");
    let result2 = greet(&name);             // &String → 自动转为 &str → OK
    println!("{}, {}", result1, result2);
}
```

**为什么分两种？** `String` 在堆上分配内存，需要有人负责释放。`&str` 只是指向现有数据的"视图"，不涉及内存管理。TypeScript 只有 `string` 一种，因为 GC 统一管理。

### 关键点 5：Clone 是你的朋友

**初学 Rust 时，不要害怕 `.clone()`。** 社区共识：先让代码编译通过，等理解所有权后再优化。

| 类型 | clone 成本 | 说明 |
|------|-----------|------|
| `Arc<T>` / `Rc<T>` | 非常便宜 | 只增加引用计数 |
| `i32`, `f64`, `&str` | 免费 | Copy 类型，自动复制 |
| `String` / `Vec<T>` | 较贵 | 深拷贝堆数据 |

```rust
use std::sync::Arc;

fn main() {
    // Arc::clone() —— 便宜！推荐写法
    let shared = Arc::new(String::from("shared data"));
    let clone1 = Arc::clone(&shared);

    // String::clone() —— 初学时大胆用
    let original = String::from("hello");
    let copy = original.clone();
    println!("{}, {}", original, copy);  // 两者都有效
}
```

**进阶提示：** 熟悉后可以用 `&T` 借用代替 clone，或用 `Cow<'a, str>` 实现"需要时才 clone"。

### 关键点 6：常用模式速记

先记住"什么场景用什么"，理解可以后来再深入：

| 模式 | 场景 | 一句话说明 |
|------|------|-----------|
| `&[T]` | 函数参数接收切片 | 通用只读序列访问，Vec 和数组都能传 |
| `Vec<Box<dyn Trait>>` | 异构集合 | 存不同类型但实现同一 Trait 的对象 |
| `Arc<T>` | 多线程共享只读 | 原子引用计数，线程安全 |
| `Arc<Mutex<T>>` | 多线程共享可变 | Arc 负责共享，Mutex 负责互斥 |

```rust
use std::sync::{Arc, Mutex};

// 切片参数：Vec 和数组都能传
fn sum(numbers: &[i32]) -> i32 { numbers.iter().sum() }

fn main() {
    let v = vec![1, 2, 3];
    println!("sum: {}", sum(&v));  // Vec → OK
    println!("sum: {}", sum(&[4, 5, 6]));  // 数组 → OK

    // 多线程共享可变数据
    let counter = Arc::new(Mutex::new(0));
    let c = Arc::clone(&counter);
    let handle = std::thread::spawn(move || {
        *c.lock().unwrap() += 1;
    });
    handle.join().unwrap();
    println!("Counter: {}", *counter.lock().unwrap());
}
```

## 第二部分：速查卡片

> 每张卡片 2-3 行，遇到概念时快速翻阅。

**所有权（Ownership）** —— 谁负责释放这块内存。每个值有且仅有一个所有者，所有者离开作用域时值被释放。

**移动（Move）** —— 所有权从一个变量转移到另一个变量。原变量立即失效，不能再使用。

**借用（Borrow）** —— 临时访问别人拥有的数据，不获取所有权。分为共享借用 `&T` 和独占借用 `&mut T`。

**`&T`（共享引用）** —— 只读访问，可以同时有任意多个。口诀：多人同时看书。

**`&mut T`（独占引用）** —— 读写访问，同一时刻只能有一个，不能和 `&T` 共存。口诀：一人独占白板。

**`'a`（生命周期注解）** —— 告诉编译器"这个引用至少活多久"的标签。大多数情况编译器能自动推断。

**`Box<T>`** —— 堆上的唯一所有者。常用于 trait 对象和递归数据结构。

**`Arc<T>`** —— 线程安全的共享所有者。原子引用计数，`Arc::clone()` 很便宜。

**`Mutex<T>`** —— 互斥锁保护的可变数据。`.lock()` 获取锁后才能访问。

**`clone()`** —— 显式深拷贝。Rust 不隐式复制堆数据，你必须明确说"我要复制一份"。TS 类比：`structuredClone(obj)`。

## 第三部分：从 TypeScript 到 Rust 的心智转换表

| 维度 | TypeScript 思维 | Rust 思维 | 转换要点 |
|------|----------------|-----------|---------|
| 内存管理 | GC 自动回收 | 编译器通过所有权规则确定释放时机 | 每个值有且仅有一个主人 |
| 变量赋值 | 共享引用（对象）/ 复制（原始值） | 默认移动（堆数据）/ 复制（栈数据） | 赋值后原变量可能失效 |
| 函数传参 | 引用传递（对象） | 默认移动；用 `&` 借用 | 要思考"给出去还能不能用" |
| null | `null` / `undefined` / `?.` | `Option<T>` + `match` / `if let` / `?` | 编译器强制处理空值 |
| 错误处理 | `try/catch` | `Result<T, E>` + `?` | 错误是返回值，不是异常 |
| 字符串 | 只有 `string` | `String`（拥有）vs `&str`（借用） | 存用 String，看用 &str |
| 数组 | `Array<T>` 随意读写 | `Vec<T>`（拥有）vs `&[T]`（借用切片） | 同字符串的拥有/借用模式 |
| 可变性 | 默认可变 | 默认不可变（除非 `mut`） | 需要改才加 `mut` |
| 并发安全 | 单线程事件循环 | 编译器保证数据竞争被捕获 | 共享用 Arc，可变用 Mutex |
| 多态 | `class` + 继承 | `struct` + `trait`（组合优于继承） | 没有继承，用 trait |

**最关键的心智转变：**

> 在 TypeScript 中，你把精力放在"逻辑对不对"。
> 在 Rust 中，你还要多想一步——"数据归谁管"。
> 这不是额外负担，而是让编译器帮你找到 bug 的方式。

## 第四部分：实战检查清单

### 写代码前

- [ ] 我清楚每个变量的所有权归谁？
- [ ] 我的数据需要跨函数使用吗？（考虑借用 vs 移动）
- [ ] 我的数据需要跨线程使用吗？（考虑 Arc）
- [ ] struct 的字段用 String 还是 &str？（通常用 String）

### 写函数时

- [ ] 函数参数是获取所有权（`T`）还是借用（`&T` / `&mut T`）？
- [ ] 默认选 `&T`（只读借用），只有需要修改时用 `&mut T`
- [ ] 只有构造函数 / 消费型函数才用 `T`（获取所有权）
- [ ] 字符串参数用 `&str` 而不是 `&String`
- [ ] 切片参数用 `&[T]` 而不是 `&Vec<T>`

### 编译报错时

- [ ] 是 `move` 错误？→ 加 `&` 改为借用，或加 `.clone()`
- [ ] 是 `borrow` 冲突？→ 调整代码顺序或拆分作用域
- [ ] 是 `lifetime` 错误？→ 检查引用是否活得够久，考虑返回 String 而非 &str
- [ ] 实在搞不定？→ 先 `.clone()` 让它编译通过，之后再优化

### 多线程场景

- [ ] 需要跨线程只读共享？→ `Arc<T>`
- [ ] 需要跨线程修改？→ `Arc<Mutex<T>>`
- [ ] 需要跨线程传递所有权？→ `mpsc` 通道
- [ ] 读多写少？→ 考虑 `Arc<RwLock<T>>`

## 第五部分：常见编译错误速查

| 错误信息 | 原因 | 解决方案 |
|----------|------|---------|
| `value used here after move` | 变量已被移动 | 用 `&` 借用或 `.clone()` |
| `cannot borrow as mutable` | 存在不可变借用时试图可变借用 | 调整代码顺序，让不可变借用先结束 |
| `cannot borrow as mutable more than once` | 同时两个可变借用 | 拆分作用域或改用 `RefCell` |
| `does not live long enough` | 引用指向的值已释放 | 延长值的生命周期或返回所有权 |
| `missing lifetime specifier` | 编译器无法推断生命周期 | 添加 `'a` 注解或返回 String |
| `expected String, found &str` | 类型不匹配 | `.to_string()` 或 `String::from()` |
| `expected &str, found String` | 类型不匹配 | 用 `&` 取引用或 `.as_str()` |

## 总结

**记住这 6 个关键点，你就能写出 80% 的 Rust 代码：**

1. **三规则口诀：** 一值一主人，转手即无权，出域即归零
2. **参数选择：** 读 → `&T`，写 → `&mut T`，存 → `T`
3. **报错三步法：** 看类型 → move 加 `&` / clone → borrow 调顺序
4. **字符串选择：** 存用 `String`，看用 `&str`
5. **Clone 是朋友：** 初学大胆用，熟了再优化
6. **常用模式：** `Arc<Mutex<T>>` 多线程，`Vec<Box<dyn Trait>>` 异构集合

**给自己的许可：**

> 初学 Rust 时，允许自己多用 `.clone()`，允许自己写"不够优雅"的代码。
> 编译器是你的老师，不是你的敌人。每一个编译错误都在教你如何写出更安全的代码。
> 先让代码跑起来，再让代码跑得好。
