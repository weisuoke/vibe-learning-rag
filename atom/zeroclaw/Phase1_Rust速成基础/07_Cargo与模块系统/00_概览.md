# Cargo 与模块系统 - 概览

> **一句话定位**：Cargo 是 Rust 的构建系统与包管理器，模块系统是代码组织机制，二者结合让 ZeroClaw 把 30+ 模块、15+ feature flags、2 个 crate 组织成一个可维护的 workspace 工程。

## 2. 为什么前端开发者需要学 Cargo 与模块系统？

### 2.1 你已经很熟悉类似的工具了

作为前端/TypeScript 开发者，你每天都在用 npm + ES Module。好消息是——Cargo 和 Rust 模块系统与它们**高度对应**：

```
npm / TypeScript                    Cargo / Rust
─────────────────────────────────────────────────────
package.json         ←→            Cargo.toml
node_modules/        ←→            ~/.cargo/registry/
npm install          ←→            cargo add
npm run build        ←→            cargo build
npm test             ←→            cargo test
package-lock.json    ←→            Cargo.lock
npmjs.com            ←→            crates.io
import/export        ←→            mod/use/pub
monorepo workspace   ←→            Cargo workspace
```

### 2.2 但有几个关键差异

| 维度 | npm/TypeScript | Cargo/Rust |
|------|---------------|------------|
| **构建** | 需要 webpack/vite/esbuild | Cargo 内置编译器集成 |
| **类型** | TS 类型独立于运行时 | 类型是编译的一部分 |
| **条件编译** | webpack DefinePlugin / 环境变量 | Feature flags（编译时移除代码） |
| **可见性** | export 控制 | `pub` / `pub(crate)` / 私有 三级 |
| **模块声明** | 文件即模块（自动发现） | 必须显式 `mod` 声明 |
| **构建产物** | JS bundle | 原生二进制 / 库 |
| **锁文件** | 总是需要 | 二进制项目需要，库项目可选 |

**最关键的差异**：

```typescript
// TypeScript — 文件存在就自动成为模块
// 直接 import 即可
import { UserService } from './services/user';  // ← 文件存在就能导入
```

```rust
// Rust — 文件存在 ≠ 模块存在，必须显式声明！
// 1. 先在 lib.rs 或 mod.rs 中声明模块
pub mod services;    // ← 声明 services 模块（对应 services/ 目录或 services.rs）

// 2. 在 services/mod.rs 中声明子模块
pub mod user;        // ← 声明 user 子模块（对应 services/user.rs）

// 3. 然后才能 use
use crate::services::user::UserService;
```

### 2.3 为什么 ZeroClaw 需要良好的模块组织？

ZeroClaw 是一个大型项目：

- **30+ 个顶层模块**（agent、channels、providers、tools…）
- **40+ 个工具实现**（Shell、HTTP、文件读写…）
- **20+ 个通道实现**（Telegram、Discord、Slack…）
- **13+ 个 LLM 提供商**（OpenAI、Anthropic、Ollama…）
- **15+ 个 feature flags**（按需编译硬件支持、数据库后端…）
- **2 个 crate**（主 crate + robot-kit 子 crate）

没有清晰的模块组织，这个项目根本无法维护。Cargo 和模块系统是让这一切井然有序的骨架。

## 6. 知识地图

```
Cargo 与模块系统
│
├── Cargo 构建系统 ──────────────────────────────────────────────
│   ├── Cargo.toml ─────────── 项目配置（= package.json）
│   │   ├── [package]  ──── name, version, edition, rust-version
│   │   ├── [dependencies]  ── 运行时依赖
│   │   ├── [dev-dependencies] ── 开发/测试依赖
│   │   ├── [build-dependencies] ── 构建脚本依赖
│   │   ├── [features]  ──── 条件编译开关
│   │   └── [profile.*]  ── 编译优化配置
│   │
│   ├── Cargo.lock ─────────── 锁定依赖版本（= package-lock.json）
│   ├── Cargo 命令 ─────────── build / run / test / add / update
│   └── crates.io ──────────── Rust 包注册中心（= npmjs.com）
│
├── Workspace ───────────────────────────────────────────────────
│   ├── 虚拟 manifest ──────── 根 Cargo.toml 只含 [workspace]
│   ├── workspace.dependencies ── 集中管理共享依赖版本
│   ├── workspace.package ──── 共享元数据（edition, license…）
│   ├── members ────────────── 列出所有成员 crate
│   └── resolver ───────────── 特性解析策略（v2/v3）
│
├── 模块系统 ────────────────────────────────────────────────────
│   ├── mod 声明 ───────────── 显式声明模块（文件不会自动成为模块！）
│   ├── 文件映射规则 ────────── foo.rs 或 foo/mod.rs
│   ├── 路径引用 ───────────── crate:: / super:: / self::
│   ├── 可见性 ─────────────── pub / pub(crate) / pub(super) / 私有
│   └── use 导入 ───────────── use crate::module::Type;
│
├── Feature Flags ───────────────────────────────────────────────
│   ├── 定义 ───────────────── [features] 表
│   ├── 可选依赖 ───────────── optional = true + dep: 前缀
│   ├── 条件编译 ───────────── #[cfg(feature = "name")]
│   ├── 默认特性 ───────────── default = [...]
│   └── 特性统一 ───────────── workspace 内自动取并集
│
└── Re-export 与 API 设计 ───────────────────────────────────────
    ├── pub use ────────────── 重新导出（= export { x } from '...'）
    ├── 选择性暴露 ─────────── 只 re-export 核心类型
    └── 扁平 API ───────────── 用户不需要知道内部层级
```

## 7. 核心概念速览

### 7.1 Cargo.toml — Rust 的 package.json

```toml
# ZeroClaw 的 Cargo.toml（简化版）
[package]
name = "zeroclaw"
version = "0.1.7"
edition = "2024"
rust-version = "1.87"
license = "MIT OR Apache-2.0"

[dependencies]
tokio = { version = "1.42", features = ["rt-multi-thread", "macros"] }
serde = { version = "1.0", features = ["derive"] }

[dev-dependencies]
tempdir = "0.3"

[features]
default = []                          # 最小构建，默认不启用任何可选特性
hardware = ["dep:nusb", "dep:tokio-serial"]
```

### 7.2 Workspace — 多 crate 项目

```toml
# 根 Cargo.toml
[workspace]
members = [".", "crates/robot-kit"]   # 两个 crate 共享构建
resolver = "2"                         # 更好的 feature 解析
```

等同于 npm workspace 的 `workspaces` 字段：
```json
// package.json（类比）
{ "workspaces": [".", "packages/robot-kit"] }
```

### 7.3 模块系统 — 显式声明

```rust
// src/lib.rs — ZeroClaw 的模块声明（30+ 个）
pub mod agent;         // → src/agent/mod.rs 或 src/agent.rs
pub mod channels;      // → src/channels/mod.rs
pub mod config;        // → src/config.rs 或 src/config/mod.rs
pub mod providers;     // → src/providers/mod.rs
pub mod tools;         // → src/tools/mod.rs
// ...
```

### 7.4 Feature Flags — 编译时开关

```toml
[features]
hardware = ["dep:nusb", "dep:tokio-serial"]    # 启用 USB 硬件支持
channel-matrix = ["dep:matrix-sdk"]             # 启用 Matrix 协议
```

```rust
// 代码中用 #[cfg] 条件编译
#[cfg(feature = "hardware")]
pub mod hardware;     // 只在启用 hardware feature 时编译
```

### 7.5 Re-export — 清理公共 API

```rust
// src/lib.rs
pub use config::Config;              // 用户直接 use zeroclaw::Config
pub use agent::{Agent, AgentBuilder};
pub use tools::Tool;
pub use channels::Channel;
pub use providers::Provider;
```

等同于 TypeScript 的 barrel export：
```typescript
// index.ts（类比）
export { Config } from './config';
export { Agent, AgentBuilder } from './agent';
```

## 10. npm 与 Cargo 对照速查表

这张表帮助你快速建立 npm → Cargo 的心智映射：

| 你在 npm 中做的事 | Cargo 中对应的操作 |
|------|------|
| `npm init` | `cargo new my-project` 或 `cargo init` |
| `npm install lodash` | `cargo add lodash`（注意：Rust 包名不同） |
| `npm install --save-dev jest` | `cargo add --dev jest`（dev-dependency） |
| `npm run build` | `cargo build` |
| `npm run build -- --release` | `cargo build --release` |
| `npm test` | `cargo test` |
| `npm start` | `cargo run` |
| 编辑 `package.json` | 编辑 `Cargo.toml` |
| `npm publish` | `cargo publish` |
| `npx create-react-app` | `cargo generate`（需要安装） |
| 查看 npmjs.com | 查看 crates.io 或 lib.rs |
| `import { x } from 'pkg'` | `use pkg::x;` |
| `export default class X {}` | `pub struct X {}` |
| `export { X } from './module'` | `pub use module::X;` |
| monorepo `workspaces: [...]` | `[workspace] members = [...]` |
| `.env` + `process.env.X` | Feature flags + `#[cfg(feature = "x")]` |
