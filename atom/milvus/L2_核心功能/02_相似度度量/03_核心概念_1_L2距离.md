# 核心概念1：L2距离（欧几里得距离）

> L2距离是最直观的向量距离度量方式，测量高维空间中两点之间的直线距离

---

## 一句话定义

**L2距离（Euclidean Distance）是测量向量端点之间直线距离的度量方式，就像地图上两点间的直线距离，值越小表示越相似。**

---

## 数学定义

### 公式

对于两个n维向量 **a = (a₀, a₁, ..., aₙ₋₁)** 和 **b = (b₀, b₁, ..., bₙ₋₁)**：

```
L2(a, b) = √(Σ(aᵢ - bᵢ)²)
         = √((a₀-b₀)² + (a₁-b₁)² + ... + (aₙ₋₁-bₙ₋₁)²)
```

### 几何意义

L2距离是**勾股定理在高维空间的推广**，测量的是两个向量端点之间的直线距离。

```
2D空间示例：
点A(1, 2) 和 点B(4, 6)
L2距离 = √((4-1)² + (6-2)²) = √(9 + 16) = √25 = 5
```

---

## 核心特征

### 1. 值域特征

- **值域**：[0, +∞)
- **相似性**：值越小，越相似
- **0表示**：两个向量完全相同
- **无上界**：距离可以无限大

### 2. 敏感性特征

**对规模敏感**：向量的绝对值大小会影响距离

```python
import numpy as np

# 示例：方向相同，长度不同
vec_a = np.array([1, 2, 3])
vec_b = np.array([2, 4, 6])  # vec_a的2倍

l2_distance = np.linalg.norm(vec_a - vec_b)
print(f"L2距离: {l2_distance:.4f}")  # 3.7417（距离较大）

# 即使方向完全相同，长度差异也会导致较大的L2距离
```

**对位置敏感**：关注向量的绝对位置差异

---

## 详细解释

### 工作原理（步骤拆解）

**步骤1：对齐向量**

```
向量A: [1, 2, 3]
向量B: [2, 3, 4]
```

**步骤2：逐维度相减**

```
差值: [1-2, 2-3, 3-4] = [-1, -1, -1]
```

**步骤3：平方每个差值**

```
平方: [(-1)², (-1)², (-1)²] = [1, 1, 1]
```

**步骤4：求和**

```
总和: 1 + 1 + 1 = 3
```

**步骤5：开平方根**

```
L2距离: √3 ≈ 1.732
```

### Python实现

```python
import numpy as np

def l2_distance(a, b):
    """
    计算L2距离（欧几里得距离）

    参数:
        a: numpy数组，向量A
        b: numpy数组，向量B

    返回:
        float: L2距离
    """
    # 方法1：手动计算
    diff = a - b
    squared_diff = diff ** 2
    sum_squared = np.sum(squared_diff)
    distance = np.sqrt(sum_squared)

    return distance

# 方法2：使用numpy内置函数
def l2_distance_numpy(a, b):
    return np.linalg.norm(a - b)

# 示例
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

print(f"L2距离（手动）: {l2_distance(vec_a, vec_b):.4f}")
print(f"L2距离（numpy）: {l2_distance_numpy(vec_a, vec_b):.4f}")
```

**输出：**
```
L2距离（手动）: 1.7321
L2距离（numpy）: 1.7321
```

## Milvus中的L2距离

### Milvus优化：跳过平方根

**关键洞察**：排序结果不受平方根影响

```python
# 三个向量与查询向量的距离
distances = [1.5, 2.0, 3.0]

# 开平方根后的排序
sorted_with_sqrt = sorted(distances)  # [1.5, 2.0, 3.0]

# 不开平方根的排序
distances_squared = [d**2 for d in distances]  # [2.25, 4.0, 9.0]
sorted_without_sqrt = sorted(distances_squared)  # [2.25, 4.0, 9.0]

# 排序顺序完全一致！
```

**Milvus实现**：

```python
# Milvus实际计算的是平方距离
def milvus_l2_distance(a, b):
    """Milvus的L2距离计算（跳过平方根）"""
    return np.sum((a - b) ** 2)  # 不开平方根

# 示例
vec_a = np.array([1.0, 2.0, 3.0])
vec_b = np.array([2.0, 3.0, 4.0])

print(f"Milvus L2距离: {milvus_l2_distance(vec_a, vec_b):.4f}")  # 3.0000
print(f"标准L2距离: {np.linalg.norm(vec_a - vec_b):.4f}")        # 1.7321
```

**优势**：
- ✅ 降低计算成本（省略平方根运算）
- ✅ 提升检索速度
- ✅ 保持排序结果一致

**来源**：[Zilliz Blog - Similarity Metrics](https://zilliz.com/blog/similarity-metrics-for-vector-search)

> "Milvus skips the square root because the square-rooted and un-square-rooted rank order is the same. This way, we can skip an operation and get the same result, lowering latency and cost and increasing throughput."

## 何时使用L2距离

### ✅ 适用场景

#### 1. 向量未归一化

```python
# 场景：图像特征提取
# 特征向量未归一化，长度有意义

image_feature_1 = np.array([10.5, 20.3, 15.7, ...])  # 未归一化
image_feature_2 = np.array([12.1, 18.9, 16.2, ...])  # 未归一化

# 使用L2距离
l2_dist = np.linalg.norm(image_feature_1 - image_feature_2)
```

**原因**：L2距离考虑向量的绝对位置差异，适合未归一化的数据。

---

#### 2. 向量长度有业务含义

```python
# 场景：用户行为强度
# 向量长度代表行为频率或强度

user_behavior_1 = np.array([5, 10, 3])   # 浏览5次，点击10次，购买3次
user_behavior_2 = np.array([50, 100, 30]) # 浏览50次，点击100次，购买30次

# 使用L2距离
l2_dist = np.linalg.norm(user_behavior_1 - user_behavior_2)
# 距离较大，反映行为强度差异
```

**原因**：L2距离同时考虑方向和长度，能够捕捉行为强度的差异。

---

#### 3. Embedding模型使用L2损失训练

```python
# 场景：模型训练时使用L2损失
# 推理时应使用相同的度量方式

# 训练时的损失函数
def l2_loss(pred, target):
    return np.sum((pred - target) ** 2)

# 推理时使用L2距离
l2_dist = np.linalg.norm(embedding_1 - embedding_2)
```

**原因**：度量方式应与模型训练时的损失函数一致。

**来源**：[Milvus AI Quick Reference](https://milvus.io/ai-quick-reference/how-does-the-distance-metric-used-cosine-vs-l2-interplay-with-the-embedding-model-choice)

> "L2与嵌入模型训练目标匹配至关重要，若模型使用L2损失则推荐L2，否则不匹配可能导致次优检索结果。"

---

### ❌ 不适用场景

#### 1. 向量已归一化

```python
# 场景：文本Embedding（通常归一化）
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')
embedding = model.encode("文本示例")

# 检查是否归一化
norm = np.linalg.norm(embedding)
print(f"向量长度: {norm:.6f}")  # 接近1.0

# ❌ 不推荐使用L2
# ✅ 推荐使用COSINE
```

**原因**：归一化向量的长度都是1，L2距离失去意义。

**2025-2026最佳实践**：[Milvus CheatSheet](https://github.com/milvus-io/bootcamp/blob/master/bootcamp/MilvusCheatSheet.md)

> "Most search algorithms work best with normalized embeddings data. This means L2 metric is useless (since all vectors have same length). **Only choose metric='L2' if you plan to keep your embeddings unnormalized.**"

---

#### 2. 只关注语义方向

```python
# 场景：文档语义相似度
# 只关注语义方向，不关注文档长度

doc1_embedding = np.array([0.1, 0.2, 0.3])
doc2_embedding = np.array([0.2, 0.4, 0.6])  # doc1的2倍

# L2距离较大（因为长度不同）
l2_dist = np.linalg.norm(doc1_embedding - doc2_embedding)
print(f"L2距离: {l2_dist:.4f}")  # 0.3742

# 但语义方向完全相同
cosine_sim = np.dot(doc1_embedding, doc2_embedding) / (
    np.linalg.norm(doc1_embedding) * np.linalg.norm(doc2_embedding)
)
print(f"余弦相似度: {cosine_sim:.4f}")  # 1.0000

# ❌ L2距离误判为不相似
# ✅ COSINE正确识别为相似
```

**原因**：L2距离对长度敏感，会误判方向相同但长度不同的向量。

## 与其他度量的对比

### L2 vs COSINE

| 维度 | L2距离 | 余弦相似度 |
|-----|--------|-----------|
| **考虑因素** | 方向 + 长度 | 只考虑方向 |
| **归一化向量** | 不推荐 | 推荐 ✅ |
| **未归一化向量** | 推荐 ✅ | 可用 |
| **值域** | [0, +∞) | [-1, 1] |
| **相似性** | 越小越相似 | 越大越相似 |
| **计算成本** | 低 | 中（需要归一化） |

**示例对比**：

```python
import numpy as np

# 两个向量：方向相同，长度不同
vec_a = np.array([1, 2, 3])
vec_b = np.array([2, 4, 6])  # vec_a的2倍

# L2距离
l2 = np.linalg.norm(vec_a - vec_b)
print(f"L2距离: {l2:.4f}")  # 3.7417（距离较大）

# 余弦相似度
cosine = np.dot(vec_a, vec_b) / (np.linalg.norm(vec_a) * np.linalg.norm(vec_b))
print(f"余弦相似度: {cosine:.4f}")  # 1.0000（完全相似）

# 结论：L2认为不相似，COSINE认为相似
```

---

### L2 vs IP（内积）

| 维度 | L2距离 | 内积IP |
|-----|--------|--------|
| **考虑因素** | 方向 + 长度 | 方向 + 长度 |
| **归一化向量** | 不推荐 | 等价于COSINE |
| **未归一化向量** | 推荐 ✅ | 推荐 ✅ |
| **值域** | [0, +∞) | (-∞, +∞) |
| **相似性** | 越小越相似 | 越大越相似 |
| **计算成本** | 中 | 低 |

**关系公式**（归一化向量）：

```
L2² = 2 - 2×COSINE = 2 - 2×IP
```

**示例验证**：

```python
# 归一化向量
vec_a = np.array([1, 2, 3])
vec_b = np.array([2, 3, 4])

vec_a_norm = vec_a / np.linalg.norm(vec_a)
vec_b_norm = vec_b / np.linalg.norm(vec_b)

# L2距离平方
l2_squared = np.sum((vec_a_norm - vec_b_norm) ** 2)

# 内积
ip = np.dot(vec_a_norm, vec_b_norm)

# 验证关系
print(f"L2²: {l2_squared:.6f}")
print(f"2 - 2×IP: {2 - 2*ip:.6f}")
print(f"差异: {abs(l2_squared - (2 - 2*ip)):.10f}")  # 接近0
```

def select_metric(embedding_model):

## 常见误区

### 误区1：L2总是比COSINE好 ❌

**错误观点**："L2是最常用的度量，应该总是使用L2"

**正确理解**：
- L2只适合未归一化向量
- 对于归一化向量（如文本Embedding），COSINE更合适
- 2025-2026标准：80%场景推荐COSINE

---

### 误区2：归一化后L2=COSINE ❌

**错误观点**："向量归一化后，L2和COSINE结果一样"

**正确理解**：
- 归一化后：**COSINE ≈ IP**（内积），不是L2
- L2和COSINE的排序结果可能不同
- 关系公式：L2² = 2 - 2×COSINE（归一化向量）

---

### 误区3：L2距离可以直接比较 ❌

**错误观点**："L2距离0.5和COSINE相似度0.5可以比较"

**正确理解**：
- 不同度量的分数不可比较
- L2距离：值域[0, +∞)，越小越相似
- COSINE：值域[-1, 1]，越大越相似
