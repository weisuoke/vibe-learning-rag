## 30 字核心概述

**Struct 组合数据字段（AND 关系），Enum 表达互斥变体（OR 关系），match 穷尽处理保证安全。**

你在 TypeScript 中用 `interface` 组合字段、用 `union type` 表达多选一、用 `switch` 分支处理 --
Rust 的 Struct + Enum + match 做的是同样的事，但**编译器会替你检查每一种情况是否都处理了**。

## 为什么要学：TypeScript 开发者的动机

### 你已经会的

作为 TypeScript 开发者，你每天都在用以下概念：

```typescript
// TypeScript: interface 定义数据结构
interface User {
  name: string;
  age: number;
}

// TypeScript: union type 表达多选一
type Message =
  | { type: "text"; content: string }
  | { type: "image"; url: string }
  | { type: "file"; path: string; size: number };

// TypeScript: switch 分支处理
function handle(msg: Message) {
  switch (msg.type) {
    case "text": console.log(msg.content); break;
    case "image": console.log(msg.url); break;
    // 忘记处理 "file"？TypeScript 不会报错！
  }
}
```

### Rust 做得更好的地方

| 痛点 | TypeScript | Rust |
|------|------------|------|
| 忘记处理某个分支 | 运行时才发现 | **编译时报错** |
| 字段可能是 `null` / `undefined` | 容易忘记检查 | **Option\<T\> 强制处理** |
| union type 没有统一方法 | 每个 case 重复写 | **enum 上可以 impl 方法** |
| interface 字段全可选时的混乱 | `Partial<T>` 只是语法糖 | **Builder 模式 + Option\<T\>** |
| class 继承链过深 | 调试困难 | **struct 组合 + trait，无继承** |

**一句话**：Rust 的 Struct + Enum + match 是 TypeScript interface + union + switch 的**编译时安全升级版**。

## ZeroClaw 中的应用概览

ZeroClaw 是一个 Rust 编写的 AI Agent 运行时。Struct 和 Enum 是其**最核心的数据建模工具**。

### 使用统计

| 模块 | Struct 数量 | Enum 数量 | 关键类型 |
|------|------------|-----------|---------|
| **Config（配置系统）** | 70+ | 7+ | 全部在 `config/schema.rs`，用 serde 反序列化 TOML/YAML |
| **Agent（核心代理）** | 5+ | - | `Agent`（运行时核心）、`AgentBuilder`（Builder 模式） |
| **Provider（LLM 接口）** | 3+ | 2+ | `ChatMessage` struct、`ConversationMessage` enum |
| **Security（安全策略）** | 3+ | 2+ | `SecurityPolicy` struct、`AutonomyLevel` enum |
| **Memory（记忆系统）** | 2+ | 1+ | `MemoryEntry` struct、`MemoryCategory` enum |
| **Observability（可观测性）** | - | 1+ | `ObserverEvent` enum（11 个变体！） |
| **Tools（工具系统）** | 3+ | - | `ToolResult` struct、`ToolSpec` struct |
| **Identity（身份标识）** | 5+ | - | `AieosIdentity` 深度嵌套 struct |
| **Channels（通信通道）** | 3+ | - | `ChannelMessage`、`SendMessage` struct |

### 关键代码片段预览

**1. Agent struct -- 组合多个 trait object**

```rust
// src/agent/agent.rs
pub struct Agent {
    provider: Box<dyn Provider>,        // LLM 提供者（多态）
    memory: Arc<dyn Memory>,            // 记忆系统（线程安全共享）
    tools: Vec<Box<dyn Tool>>,          // 工具列表（动态分发）
    security_policy: SecurityPolicy,    // 安全策略（值类型）
    identity: AieosIdentity,            // 身份标识（嵌套 struct）
    // ...更多字段
}
```

> **TypeScript 对比**：相当于 `class Agent { provider: Provider; memory: Memory; tools: Tool[]; ... }`
> 但 Rust 不用 class，用 struct + trait 实现同样效果。

**2. ConversationMessage enum -- 3 种消息变体**

```rust
// src/providers/traits.rs
pub enum ConversationMessage {
    Chat(ChatMessage),                          // 普通聊天消息
    AssistantToolCalls(Vec<ToolCall>),           // AI 调用工具
    ToolResults(Vec<ToolResult>),                // 工具返回结果
}
```

> **TypeScript 对比**：
> ```typescript
> type ConversationMessage =
>   | { type: "Chat"; data: ChatMessage }
>   | { type: "AssistantToolCalls"; data: ToolCall[] }
>   | { type: "ToolResults"; data: ToolResult[] };
> ```

**3. AutonomyLevel enum -- 安全等级**

```rust
// src/security/policy.rs
pub enum AutonomyLevel {
    ReadOnly,       // 只读：AI 不能执行任何操作
    Supervised,     // 监督：AI 执行前需人类确认
    Full,           // 完全：AI 自主执行所有操作
}
```

> 如果你用 TypeScript 写 `autonomy: "readonly" | "supervised" | "full"`，
> 写错成 `"readnoly"` 不会有编译错误。Rust enum 在编译时就杜绝了拼写错误。

**4. ObserverEvent enum -- 11 个变体，每个携带不同数据**

```rust
// src/observability/traits.rs（简化）
pub enum ObserverEvent {
    AgentStarted { agent_id: String },
    ToolCallStarted { tool_name: String, input: Value },
    ToolCallCompleted { tool_name: String, result: ToolResult, duration: Duration },
    LlmRequestStarted { model: String, message_count: usize },
    LlmResponseReceived { model: String, tokens_used: u32 },
    MemoryStored { category: MemoryCategory, key: String },
    SecurityCheckFailed { action: String, reason: String },
    Error { source: String, message: String },
    // ...更多变体
}
```

> 这就是 Rust enum 的威力：**每个变体可以携带完全不同的数据结构**，
> 而 `match` 会强制你处理所有 11 种情况，一个都不能漏。

**5. AgentBuilder -- Builder 模式（所有字段 Option\<T\>）**

```rust
// src/agent/agent.rs（简化）
pub struct AgentBuilder {
    provider: Option<Box<dyn Provider>>,
    memory: Option<Arc<dyn Memory>>,
    tools: Option<Vec<Box<dyn Tool>>>,
    security_policy: Option<SecurityPolicy>,
    // ...每个字段都是 Option<T>
}

impl AgentBuilder {
    pub fn new() -> Self {
        AgentBuilder {
            provider: None,
            memory: None,
            tools: None,
            security_policy: None,
        }
    }

    pub fn provider(mut self, p: Box<dyn Provider>) -> Self {
        self.provider = Some(p);
        self             // 返回 self 实现链式调用
    }

    pub fn build(self) -> Result<Agent, String> {
        Ok(Agent {
            provider: self.provider.ok_or("provider is required")?,
            memory: self.memory.ok_or("memory is required")?,
            tools: self.tools.unwrap_or_default(),
            security_policy: self.security_policy.unwrap_or_default(),
        })
    }
}
```

> **TypeScript 对比**：类似于你用过的 `new AgentBuilder().setProvider(p).setMemory(m).build()`。
> 但 Rust 用 `Option<T>` 明确表达"这个字段可能还没设置"，build() 时**编译器帮你检查**。

## TypeScript -> Rust 概念映射表

这是本知识点最关键的一张表。学习过程中随时回来查：

| TypeScript 概念 | Rust 对应 | 关键区别 |
|-----------------|----------|---------|
| `interface User { name: string }` | `struct User { name: String }` | Rust struct 有所有权语义 |
| `class User { constructor() {} }` | `struct User` + `impl User { fn new() }` | Rust 分离数据与行为 |
| `type Status = "active" \| "inactive"` | `enum Status { Active, Inactive }` | Rust enum 是编译时类型安全 |
| 带数据的 union type | `enum Msg { Text(String), File { path: String } }` | Rust enum 变体可携带不同数据 |
| `switch (msg.type) { case ... }` | `match msg { Msg::Text(s) => ... }` | Rust match 强制穷尽 |
| `msg?.content` (可选链) | `if let Some(content) = msg { ... }` | Rust 用 Option 代替 null |
| `Partial<Config>` | `struct ConfigBuilder { field: Option<T> }` | Rust 用 Option 逐字段可选 |
| `class extends BaseClass` | Rust **没有继承**，用 struct 组合 + trait | 组合优于继承 |
| `implements Interface` | `impl Trait for Struct` | 类似但更灵活 |
| `readonly` 属性 | 默认不可变，需 `mut` 才可变 | Rust 默认安全 |
| `private` / `public` | 默认私有，需 `pub` 才公有 | Rust 默认安全 |
| `Record<string, any>` | `HashMap<String, Value>` | Rust 需要明确值类型 |
| `Array<T>` | `Vec<T>` | 功能等价 |
| `T \| null` | `Option<T>` | Rust 无 null，用 Option |
| `T \| Error` | `Result<T, E>` | Rust 用 Result 处理错误 |
| `#[decorator]` | `#[derive(Debug, Clone)]` | Rust 用属性宏自动生成代码 |
