## 一、源码级关键机制

### 1.1 类型推断机制

**核心源码**（state.py:698-743）：

```python
if (
    isfunction(action)
    or ismethod(action)
    or ismethod(getattr(action, "__call__", None))
) and (
    hints := get_type_hints(getattr(action, "__call__"))
    or get_type_hints(action)
):
    if input_schema is None:
        first_parameter_name = next(
            iter(
                inspect.signature(
                    cast(FunctionType, action)
                ).parameters.keys()
            )
        )
        if input_hint := hints.get(first_parameter_name):
            if isinstance(input_hint, type) and get_type_hints(input_hint):
                inferred_input_schema = input_hint
```

[来源: reference/source_节点函数约定_01.md]

**关键洞察**：
- LangGraph 通过 `get_type_hints()` 自动推断节点函数的输入类型
- 第一个参数的类型被推断为 `input_schema`
- 支持 TypedDict 和 Annotated 类型

**实战应用**：

```python
from typing_extensions import TypedDict, Annotated
from operator import add

class State(TypedDict):
    messages: Annotated[list[str], add]  # 自动推断 + add reducer
    count: int

def my_node(state: State) -> dict:
    # LangGraph 自动推断 input_schema = State
    return {"count": state["count"] + 1}
```

### 1.2 重试机制的指数退避算法

**核心源码**（_retry.py:90-101）：

```python
# 指数退避计算
interval = matching_policy.initial_interval
interval = min(
    matching_policy.max_interval,
    interval * (matching_policy.backoff_factor ** (attempts - 1)),
)

# 添加抖动（jitter）避免雷鸣群效应
sleep_time = (
    interval + random.uniform(0, 1) if matching_policy.jitter else interval
)
```

[来源: reference/source_节点函数约定_02.md]

**算法分析**：

| 尝试 | 公式 | 示例（1s, 2x） |
|-----|------|---------------|
| 1 | initial | 1s |
| 2 | initial × 2^1 | 2s |
| 3 | initial × 2^2 | 4s |
| 4 | initial × 2^3 | 8s |

**生产级配置**：

```python
# API 限流场景
retry_policy = RetryPolicy(
    max_attempts=5,
    initial_interval=1.0,
    backoff_factor=2.0,
    max_interval=30.0,
    jitter=True,  # 启用抖动
    retry_on=lambda e: "rate limit" in str(e).lower()
)
```

### 1.3 异步节点执行机制

**核心源码**（_call.py:175-192）：

```python
if is_async_callable(func):
    # 异步函数：直接包装
    run = RunnableCallable(None, func, name=func.__name__)
else:
    # 同步函数：包装成异步执行
    afunc = functools.update_wrapper(
        functools.partial(run_in_executor, None, func), func
    )
    run = RunnableCallable(func, afunc, name=func.__name__)
```

[来源: reference/source_节点函数约定_01.md]

**关键洞察**：
- 异步函数直接包装
- 同步函数通过 `run_in_executor` 在线程池执行
- I/O 密集型优先使用异步，CPU 密集型使用同步

**最佳实践**：

```python
# ✅ I/O 密集型：使用异步
async def fetch_data(state: State) -> dict:
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as resp:
            return {"data": await resp.json()}

# ✅ CPU 密集型：使用同步
def process_data(state: State) -> dict:
    result = heavy_computation(state["data"])
    return {"result": result}

# ❌ 避免：异步生成器（可能不执行）
async def my_generator(state: State):
    async for item in some_async_generator():
        yield {"item": item}  # 可能失效
```

## 二、高级技巧

### 2.1 自定义重试条件

**技巧1：基于异常消息**

```python
def should_retry_on_rate_limit(error: Exception) -> bool:
    """只在遇到限流错误时重试"""
    error_msg = str(error).lower()
    return any(keyword in error_msg for keyword in [
        "rate limit", "too many requests", "quota exceeded"
    ])

retry_policy = RetryPolicy(
    max_attempts=5,
    initial_interval=1.0,
    backoff_factor=2.0,
    retry_on=should_retry_on_rate_limit
)
```

**技巧2：组合多个重试策略**

```python
retry_policies = [
    RetryPolicy(
        max_attempts=5,
        initial_interval=1.0,
        backoff_factor=2.0,
        retry_on=lambda e: "rate limit" in str(e).lower()
    ),
    RetryPolicy(
        max_attempts=3,
        initial_interval=0.5,
        backoff_factor=1.5,
        retry_on=(ConnectionError, TimeoutError)
    )
]

builder.add_node("api_call", api_call_node, retry_policy=retry_policies)
```

### 2.2 Command 对象高级用法

**技巧1：条件跳转 + 状态更新**

```python
from langgraph.types import Command
from typing import Literal

def smart_router(state: State) -> Command[Literal["process", "skip", "retry"]]:
    """智能路由：根据状态决定下一步"""
    if state.get("error"):
        return Command(
            goto="retry",
            update={"error": None, "retry_count": state.get("retry_count", 0) + 1}
        )
    elif state.get("data_ready"):
        return Command(goto="process")
    else:
        return Command(goto="skip", update={"skipped": True})
```

**技巧2：父图通信**

```python
def subgraph_node(state: State) -> Command:
    """子图节点：向父图发送命令"""
    if state["should_exit_to_parent"]:
        return Command(
            graph=Command.PARENT,
            goto="parent_node",
            update={"result": state["result"]}
        )
    return Command(goto="next_in_subgraph")
```

```python
import asyncio

async def cancellable_node(state: State) -> dict:
    """正确处理取消错误"""
    try:
```

### 2.4 性能优化技巧

**技巧1：避免不必要的状态复制**

```python
# ❌ 低效：复制整个状态
def inefficient_node(state: State) -> dict:
    new_state = state.copy()
    new_state["count"] += 1
    return new_state

# ✅ 高效：只返回更新
def efficient_node(state: State) -> dict:
    return {"count": state["count"] + 1}
```

**技巧2：使用 Reducer 函数优化列表操作**

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    # ✅ 高效：使用 add reducer 追加
    messages: Annotated[list[str], add]

def add_message(state: State) -> dict:
    # 只返回新消息，自动追加到列表
    return {"messages": ["new message"]}
```

### 3.1 分层错误处理

**实践：三层错误处理策略**

```python
# 层1：节点内部处理预期错误
async def api_call_node(state: State) -> dict:
    try:
        result = await api_call()
        return {"result": result, "error": None}
    except ValidationError as e:
        return {"result": None, "error": str(e)}

# 层2：RetryPolicy 处理临时错误
builder.add_node(
    "api_call",
    api_call_node,
    retry_policy=RetryPolicy(
        max_attempts=3,
        retry_on=(ConnectionError, TimeoutError)
    )
)

# 层3：图级别处理致命错误
try:
    result = graph.invoke(input)
except Exception as e:
    logger.error(f"Graph execution failed: {e}")
    send_alert(e)
```

### 3.2 监控和调试

**实践：结构化日志**

```python
import logging
import json

logger = logging.getLogger(__name__)

def logged_node(state: State, config: RunnableConfig) -> dict:
    """带结构化日志的节点"""
    thread_id = config.get("configurable", {}).get("thread_id")

    logger.info(json.dumps({
```

```python
from typing import Protocol

class APIClient(Protocol):
    async def call(self, query: str) -> dict: ...

def create_api_node(client: APIClient):
    """工厂函数：注入依赖"""
```

## 四、总结

### 核心要点

1. **类型推断**：利用 Python 类型注解实现自动 schema 推断
2. **重试机制**：指数退避 + 抖动避免雷鸣群效应
3. **异步执行**：同步/异步节点自动适配，混用无障碍
4. **错误处理**：分层处理（节点内 + RetryPolicy + 图级别）
5. **性能优化**：部分状态更新 + Reducer 函数 + 缓存
6. **生产实践**：监控、日志、安全、可测试性

1. 阅读 LangGraph 源码：`langgraph/graph/state.py`
