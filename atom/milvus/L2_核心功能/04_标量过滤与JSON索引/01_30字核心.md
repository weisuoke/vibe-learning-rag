## 核心拆解

### 1. 标量过滤（8字）

**关键词**：布尔表达式、元数据筛选、精准匹配

**解释**：
- **布尔表达式**：使用 AND/OR/NOT 组合过滤条件
- **元数据筛选**：基于标量字段（年龄、价格、类别等）过滤
- **精准匹配**：支持比较、范围、集合、NULL 等操作符

**为什么重要**：
向量检索返回语义相似结果，但往往需要结合业务规则（价格范围、库存状态、用户权限等）进一步筛选。标量过滤提供精准的业务逻辑控制。

---

### 2. JSON Path Index（8字）

**关键词**：嵌套JSON、路径索引、100倍提升

**解释**：
- **嵌套JSON**：支持复杂的JSON结构（对象、数组、多层嵌套）
- **路径索引**：为特定JSON路径创建索引（如 `metadata['category']`）
- **100倍提升**：相比全表扫描，性能提升100倍

**为什么重要**：
现代应用的元数据通常以JSON格式存储，包含丰富的嵌套信息。JSON Path Index使得复杂元数据查询变得高效可行。

---

### 3. 混合检索标准（8字）

**关键词**：向量+标量、生产环境、2026标准

**解释**：
- **向量+标量**：语义检索与业务规则结合
- **生产环境**：企业级应用的必备能力
- **2026标准**：从高级特性变为基础配置

**为什么重要**：
纯向量检索无法满足实际业务需求。混合检索（向量+标量+JSON）成为RAG系统、推荐系统、企业搜索的标准配置。

---

### 4. 性能提升（6字）

**关键词**：30倍、100倍、索引优化

**解释**：
- **标量索引**：点查询性能提升30倍
- **JSON Path Index**：嵌套查询性能提升100倍
- **索引优化**：INVERTED、BITMAP、STL_SORT等多种选择

**为什么重要**：
性能是生产环境的核心指标。通过合理的索引策略，可以在保证功能的同时实现极致性能。

## 核心价值主张

### 1. 解决纯向量检索的局限

**问题**：
- 无法应用业务规则（价格范围、库存状态）
- 无法处理权限控制（用户级别、数据可见性）
- 无法支持复杂元数据查询

**解决**：
标量过滤 + JSON Path Index 提供完整的元数据查询能力。

### 2. 支持复杂业务场景

**场景1：电商推荐**
```python
# 向量检索：找相似商品
# 标量过滤：价格范围、库存状态
# JSON过滤：品牌、类别、标签
filter = '''
    price >= 100 AND price <= 500
    AND stock > 0
    AND metadata['brand'] IN ['Apple', 'Samsung']
    AND ARRAY_CONTAINS(metadata['tags'], 'hot')
'''
```

**场景2：企业知识库**
```python

    department == 'engineering'
    AND access_level <= user_level
    AND metadata['doc_type'] == 'technical'
    AND metadata['author']['team'] == 'backend'
'''
```

## 实战示例

### 查询："推荐500元以内的白色降噪耳机"

**向量检索**（语义理解）：
```python
query = "白色降噪耳机"
# 返回：降噪耳机、白色耳机、舒适耳机
```

**标量过滤**（价格范围）：
```python
filter = "price <= 500 AND stock > 0"
# 筛选：有库存且价格合适的商品
```

**JSON过滤**（复杂属性）：
```python
filter += " AND metadata['color'] == 'white'"
filter += " AND ARRAY_CONTAINS(metadata['features'], 'noise_cancelling')"
# 筛选：白色 + 降噪功能
```

**最终结果**：
```
Top 1: 白色降噪耳机 ¥399（语义+价格+属性都匹配）
Top 2: 白色舒适耳机 ¥299（语义+价格+颜色匹配）
Top 3: 降噪耳机 ¥499（语义+价格+功能匹配）
```

## 核心技术栈

### Milvus 实现

```python
from pymilvus import MilvusClient

client = MilvusClient("http://localhost:19530")

# 1. 标量过滤
results = client.search(
    collection_name="products",
    data=[query_vector],
    filter="price >= 100 AND price <= 500 AND stock > 0",
    limit=10
)

# 2. JSON Path Index 创建
index_params.add_index(
    field_name="metadata",
    index_type="INVERTED",
    params={
        "json_path": "metadata['category']",
        "json_cast_type": "varchar"
    }
)

# 3. 混合检索
results = client.search(
    collection_name="products",
    data=[query_vector],
    filter='''
        price <= 500
        AND metadata['color'] == 'white'
        AND ARRAY_CONTAINS(metadata['features'], 'noise_cancelling')
    ''',
    limit=5
)
```
