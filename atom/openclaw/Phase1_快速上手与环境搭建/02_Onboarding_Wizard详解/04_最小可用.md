# 最小可用 - 从零到首次对话

**本文档提供最快路径：从全新安装到完成首次 AI 对话，只需 5 分钟。**

---

## 目标

完成本文档后，你将：

1. ✅ 成功运行 `openclaw onboard` 向导
2. ✅ 配置 Gateway 和基础认证
3. ✅ 启动 Gateway 守护进程
4. ✅ 完成首次 AI 对话（通过 Dashboard 或 Channel）

---

## 前置条件

### 必需

- ✅ 已安装 OpenClaw CLI（参考 Phase1_01_安装与环境配置）
- ✅ Node.js 22+ 已安装
- ✅ 有效的 API 密钥（Anthropic 或 OpenAI）

### 可选

- Telegram Bot Token（用于 Telegram 集成）
- WhatsApp 手机（用于 WhatsApp 集成）

---

## 最小可用路径（5 分钟）

### 步骤 1：准备 API 密钥（1 分钟）

```bash
# 方式 1：设置环境变量（推荐）
export ANTHROPIC_API_KEY="sk-ant-..."

# 方式 2：稍后在向导中输入
# 向导会提示你输入 API 密钥
```

**获取 API 密钥**：
- Anthropic: https://console.anthropic.com/settings/keys
- OpenAI: https://platform.openai.com/api-keys

## 常见问题排查

### 问题 1：API 密钥未识别

**症状**：
```
◆  Anthropic API key
│  [blank]
```

**解决方案**：
```bash
# 确认环境变量已设置
echo $ANTHROPIC_API_KEY

# 如果未设置，手动输入或重新设置
export ANTHROPIC_API_KEY="sk-ant-..."
openclaw onboard --flow quickstart
```

---

### 问题 2：Gateway 启动失败

**症状**：
```
◇  Health check
│  Starting Gateway...
│  Gateway did not become ready.
```

**解决方案**：

```bash
# 检查端口占用
lsof -i :18789

# 如果端口被占用，杀死进程或使用其他端口
openclaw onboard --flow advanced
# 选择其他端口（如 18790）

# 检查 Gateway 日志
openclaw logs gateway

# 手动启动 Gateway（调试）
openclaw gateway run --foreground
```

---

### 问题 3：systemd 安装失败（Linux）

**症状**：
```
◇  Gateway service
│  Systemd user services are unavailable. Skipping service install.
```

**解决方案**：

```bash
# 安装 dbus-user-session
sudo apt install -y dbus-user-session

# 启用 lingering
sudo loginctl enable-linger $USER

# 重新登录
exit
# SSH 重新连接

# 验证 systemd 可用
systemctl --user status

# 重新运行向导
openclaw onboard --flow quickstart
```

**详细排查**：查看 `temp/troubleshooting/systemd_issue_1818.md`

---

### 问题 4：macOS Gateway 未就绪

**症状**：
```
◇  Health check
│  Gateway never becomes ready (Setup Wizard stuck on Retry)
```

**解决方案**：

```bash
# 确认 CLI 在 PATH 中
which openclaw

# 检查 Accessibility 权限
# System Settings → Privacy & Security → Accessibility
# 启用 OpenClaw

# 手动启动 Gateway
openclaw gateway run --foreground

# 查看日志
openclaw logs gateway
```

**详细排查**：查看 `temp/troubleshooting/macos_gateway_readiness_issue_6156.md`

---

### 问题 5：Model/Auth 步骤被跳过

**症状**：
向导直接跳到 "How do you want to hatch your bot?"，未提示配置模型和 API 密钥。

**解决方案**：

```bash
# 方式 1：手动配置 API 密钥
openclaw config set env.ANTHROPIC_API_KEY "sk-ant-..."
openclaw gateway restart

# 方式 2：使用 configure 命令
openclaw configure
openclaw gateway restart

# 方式 3：降级到工作版本
npm uninstall -g openclaw
npm install -g openclaw@2026.2.9
openclaw onboard
```

**详细排查**：查看 `temp/troubleshooting/model_auth_skip_issue_16134.md`

## 配置文件位置

向导完成后，配置文件位于：

```bash
~/.openclaw/openclaw.json          # 主配置文件
~/.openclaw/workspace/             # Agent 工作目录
~/.openclaw/workspace/.openclaw/bootstrap.md  # Bootstrap 文件
```

**查看配置**：
```bash
# 查看完整配置
cat ~/.openclaw/openclaw.json

# 查看特定字段
openclaw config get gateway.port
openclaw config get agents.defaults.model.primary
```

## 守护进程管理

### macOS（LaunchAgent）

```bash
# 查看服务状态
launchctl list | grep openclaw

# 启动服务
launchctl start ai.openclaw.gateway

# 停止服务
launchctl stop ai.openclaw.gateway

# 重启服务
openclaw gateway restart

# 卸载服务
openclaw gateway uninstall
```

**服务文件位置**：
```
~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

### Linux（systemd）

```bash
# 查看服务状态
systemctl --user status openclaw-gateway

# 启动服务
systemctl --user start openclaw-gateway

# 停止服务
systemctl --user stop openclaw-gateway

# 重启服务
systemctl --user restart openclaw-gateway

# 查看日志
journalctl --user -u openclaw-gateway -f

# 卸载服务
openclaw gateway uninstall
```

**服务文件位置**：
```
~/.config/systemd/user/openclaw-gateway.service
```
