## 类比1：Partial Variables（部分变量预填充）

### 目标概念

**Partial Variables** 允许你预先填充模板中的部分变量，避免每次调用时重复传递相同的值。

[来源: reference/search_partial_01.md | LangChain 官方文档]

### 前端类比：React 的 defaultProps

在 React 中，你可以为组件设置默认属性，这样使用组件时就不需要每次都传递这些属性。

```jsx
// React 组件的 defaultProps
function UserCard({ name, role, company }) {
  return <div>{name} - {role} at {company}</div>;
}

UserCard.defaultProps = {
  company: "TechCorp",  // 默认公司名
  role: "Developer"     // 默认角色
};

// 使用时只需传递 name
<UserCard name="Alice" />
// 等价于
<UserCard name="Alice" role="Developer" company="TechCorp" />
```

**LangChain 的 Partial Variables 类似：**

```python
from langchain_core.prompts import PromptTemplate

# 预填充 company 和 role
prompt = PromptTemplate(
    template="Employee: {name}, Role: {role}, Company: {company}",
    input_variables=["name"],
    partial_variables={
        "company": "TechCorp",
        "role": "Developer"
    }
)

# 使用时只需传递 name
result = prompt.format(name="Alice")
# 输出: "Employee: Alice, Role: Developer, Company: TechCorp"
```

[来源: reference/source_prompttemplate_01.md:69-85]

### 日常生活类比：快递单模板

想象你经常从同一个地址寄快递，快递单上有很多固定信息：

**传统方式（每次都填写所有信息）：**
```
寄件人：张三
寄件地址：北京市朝阳区xxx街道xxx号
寄件电话：138xxxxxxxx
收件人：______（每次填写）
收件地址：______（每次填写）
收件电话：______（每次填写）
```

**使用 Partial Variables（预填充固定信息）：**
```
寄件人：张三 ✓（已预填）
寄件地址：北京市朝阳区xxx街道xxx号 ✓（已预填）
寄件电话：138xxxxxxxx ✓（已预填）
收件人：______（只需填这个）
收件地址：______（只需填这个）
收件电话：______（只需填这个）
```

**优势：**
- 减少重复劳动
- 降低出错概率
- 提高填写效率

### 代码对比示例

**❌ 不使用 Partial Variables（每次都传递所有变量）：**

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

prompt = PromptTemplate(
    template="""
System: {system_name}
Date: {date}
User Query: {query}
""",
    input_variables=["system_name", "date", "query"]
)

# 每次调用都要传递 system_name 和 date
result1 = prompt.format(
    system_name="RAG Assistant",
    date=datetime.now().strftime("%Y-%m-%d"),
    query="What is LangChain?"
)

result2 = prompt.format(
    system_name="RAG Assistant",  # 重复传递
    date=datetime.now().strftime("%Y-%m-%d"),  # 重复传递
    query="How to use PromptTemplate?"
)
```

**✅ 使用 Partial Variables（预填充固定变量）：**

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 预填充固定变量
prompt = PromptTemplate(
    template="""
System: {system_name}
Date: {date}
User Query: {query}
""",
    input_variables=["query"],
    partial_variables={
        "system_name": "RAG Assistant",
        "date": lambda: datetime.now().strftime("%Y-%m-%d")  # 函数动态值
    }
)

# 只需传递变化的变量
result1 = prompt.format(query="What is LangChain?")
result2 = prompt.format(query="How to use PromptTemplate?")
```

## 类比2：Template Composition（模板组合）

### 目标概念

**Template Composition** 允许你使用 `+` 操作符将多个模板组合成一个更大的模板，自动合并变量。

[来源: reference/search_composition_01.md:45-52]

### 前端类比：Express 中间件链

在 Express.js 中，你可以将多个中间件函数组合成一个处理链：

```javascript
// Express 中间件组合
const express = require('express');
const app = express();

// 中间件1：日志记录
const logger = (req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
};

// 中间件2：身份验证
const auth = (req, res, next) => {
  if (req.headers.authorization) {
    next();
  } else {
    res.status(401).send('Unauthorized');
  }
};

// 组合中间件
app.use(logger);  // 先执行日志
app.use(auth);    // 再执行认证
```

**LangChain 的 Template Composition 类似：**

```python
from langchain_core.prompts import PromptTemplate

# 模板1：系统设置
system_template = PromptTemplate.from_template(
    "System: You are a {role}.\n"
)

# 模板2：用户查询
user_template = PromptTemplate.from_template(
    "User: {query}\n"
)

# 使用 + 操作符组合
combined = system_template + user_template

# 自动合并变量：["role", "query"]
result = combined.format(role="helpful assistant", query="What is AI?")
```

[来源: reference/source_prompttemplate_01.md:28-59]

### 日常生活类比：乐高积木

想象你在用乐高积木搭建一个房子：

**单独的积木块：**
```
[地基块] - 包含地面和基础
[墙壁块] - 包含四面墙
[屋顶块] - 包含屋顶和烟囱
```

**组合积木：**
```
[地基块]
   +
[墙壁块]
   +
[屋顶块]
   =
[完整的房子]
```

**特点：**
- 每个积木块独立设计
- 可以自由组合
- 组合后形成完整结构
- 可以重复使用积木块

### 代码对比示例

**❌ 不使用 Template Composition（手动拼接字符串）：**

```python
from langchain_core.prompts import PromptTemplate

# 手动拼接模板字符串
template_str = """
System: You are a {role}.
Context: {context}
User: {query}
Assistant:
"""

prompt = PromptTemplate(
    template=template_str,
    input_variables=["role", "context", "query"]
)

# 问题：难以复用、难以维护
```

**✅ 使用 Template Composition（模块化组合）：**

```python
from langchain_core.prompts import PromptTemplate

# 模块化设计
system_prompt = PromptTemplate.from_template(
    "System: You are a {role}.\n"
)

context_prompt = PromptTemplate.from_template(
    "Context: {context}\n"
)

user_prompt = PromptTemplate.from_template(
    "User: {query}\nAssistant:\n"
)

# 灵活组合
# 场景1：带上下文
full_prompt = system_prompt + context_prompt + user_prompt

# 场景2：不带上下文
simple_prompt = system_prompt + user_prompt

# 场景3：添加新模块
history_prompt = PromptTemplate.from_template(
    "History: {history}\n"
)
chat_prompt = system_prompt + history_prompt + user_prompt
```

## 类比3：Template Formats（模板格式）

### 目标概念

LangChain 支持三种模板格式：**f-string**（默认）、**mustache**、**jinja2**，每种格式有不同的语法和能力。

[来源: reference/source_prompttemplate_01.md:100-142]

### 前端类比：CSS 预处理器

在前端开发中，有多种 CSS 预处理器可选：

```css
/* 原生 CSS（类似 f-string）*/
.button {
  color: #333;
  background: #fff;
}

/* SCSS（类似 mustache）*/
$primary-color: #333;
.button {
  color: $primary-color;
  &:hover {
    color: darken($primary-color, 10%);
  }
}

/* Stylus（类似 jinja2）*/
primary-color = #333
.button
  color primary-color
  &:hover
    color darken(primary-color, 10%)
```

**LangChain 的三种模板格式：**

```python
from langchain_core.prompts import PromptTemplate

# 1. f-string（默认，简单快速）
prompt1 = PromptTemplate.from_template(
    "Hello {name}!",
    template_format="f-string"
)

# 2. mustache（支持嵌套和条件）
prompt2 = PromptTemplate.from_template(
    "Hello {{name}}! {{#premium}}You are a premium user.{{/premium}}",
    template_format="mustache"
)

# 3. jinja2（功能强大但有安全风险）
prompt3 = PromptTemplate.from_template(
    "Hello {{ name }}! {% if premium %}You are a premium user.{% endif %}",
    template_format="jinja2"
)
```

[来源: reference/source_prompttemplate_01.md:113-142]

### 日常生活类比：填空题的三种形式

**形式1：简单填空（f-string）**
```
我的名字是 ___，我今年 ___ 岁。
```
- 最简单直接
- 只能填入值
- 适合大多数场景

**形式2：条件填空（mustache）**
```
我的名字是 ___。
[如果是学生] 我在 ___ 学校读书。
[如果是职员] 我在 ___ 公司工作。
```
- 支持条件显示
- 支持嵌套结构
- 更灵活

**形式3：编程填空（jinja2）**
```
我的名字是 ___。
{% if age < 18 %}
  我是未成年人。
{% else %}
  我是成年人，今年 {{ age }} 岁。
{% endif %}
```
- 支持复杂逻辑
- 功能最强大
- 但也最复杂

### 代码对比示例

```python
from langchain_core.prompts import PromptTemplate

# ===== 1. f-string 格式 =====
print("=== f-string 格式 ===")

f_prompt = PromptTemplate.from_template(
    "User: {name}\nRole: {role}",
    template_format="f-string"
)

print(f_prompt.format(name="Alice", role="Developer"))
# 输出:
# User: Alice
# Role: Developer

# ===== 2. mustache 格式 =====
print("\n=== mustache 格式 ===")

mustache_prompt = PromptTemplate.from_template(
    """
User: {{name}}
{{#is_premium}}
Premium Features: Enabled
{{/is_premium}}
{{^is_premium}}
Premium Features: Disabled
{{/is_premium}}
""",
    template_format="mustache"
)

print(mustache_prompt.format(name="Alice", is_premium=True))
# 输出:
# User: Alice
# Premium Features: Enabled

# ===== 3. jinja2 格式 =====
print("\n=== jinja2 格式 ===")

jinja_prompt = PromptTemplate.from_template(
    """
User: {{ name }}
{% if age >= 18 %}
Status: Adult ({{ age }} years old)
{% else %}
Status: Minor ({{ age }} years old)
{% endif %}
""",
    template_format="jinja2"
)

print(jinja_prompt.format(name="Alice", age=25))
# 输出:
# User: Alice
# Status: Adult (25 years old)
```

## 类比4：动态 Partial Variables（函数动态值）

### 目标概念

Partial Variables 不仅支持固定字符串值，还支持**函数动态值**，在每次调用时自动计算。

[来源: reference/search_partial_01.md:86-93]

### 前端类比：React Hooks 的 useMemo

在 React 中，`useMemo` 可以缓存计算结果，但在依赖变化时重新计算：

```jsx
import { useMemo } from 'react';

function UserProfile({ userId }) {
  // 动态计算当前时间
  const currentTime = useMemo(() => {
    return new Date().toLocaleString();
  }, []); // 空依赖数组，每次渲染都重新计算

  return (
    <div>
      <p>User ID: {userId}</p>
      <p>Loaded at: {currentTime}</p>
    </div>
  );
}
```

**LangChain 的函数动态值类似：**

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 使用函数动态获取当前时间
prompt = PromptTemplate(
    template="Query at {timestamp}: {query}",
    input_variables=["query"],
    partial_variables={
        "timestamp": lambda: datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    }
)

# 每次调用都会重新计算时间
result1 = prompt.format(query="First query")
# 输出: "Query at 2026-02-26 10:30:15: First query"

import time
time.sleep(2)

result2 = prompt.format(query="Second query")
# 输出: "Query at 2026-02-26 10:30:17: Second query"
# 注意时间变化了！
```

### 日常生活类比：智能手表的动态显示

想象你的智能手表表盘：

**静态信息（字符串固定值）：**
```
品牌：Apple Watch
型号：Series 9
```

**动态信息（函数动态值）：**
```
当前时间：[自动更新]
心率：[实时监测]
步数：[实时统计]
```

**特点：**
- 静态信息一次设置，永不改变
- 动态信息每次查看都重新计算
- 无需手动更新

### 代码对比示例

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime
import random

# ===== 场景：RAG 系统的查询日志 =====

# 创建带动态变量的模板
log_prompt = PromptTemplate(
    template="""
=== Query Log ===
Timestamp: {timestamp}
Session ID: {session_id}
User Query: {query}
System Version: {version}
==================
""",
    input_variables=["query"],
    partial_variables={
        # 动态值：每次调用时重新计算
        "timestamp": lambda: datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
        "session_id": lambda: f"session_{random.randint(1000, 9999)}",
        # 静态值：固定不变
        "version": "v1.0.0"
    }
)

# 第一次查询
print(log_prompt.format(query="What is RAG?"))
# 输出:
# === Query Log ===
# Timestamp: 2026-02-26 10:30:15
# Session ID: session_3847
# User Query: What is RAG?
# System Version: v1.0.0
# ==================

import time
time.sleep(1)

# 第二次查询（时间和 session_id 自动更新）
print(log_prompt.format(query="How to use LangChain?"))
# 输出:
# === Query Log ===
# Timestamp: 2026-02-26 10:30:16  ← 时间变了
# Session ID: session_7291        ← session_id 变了
# User Query: How to use LangChain?
# System Version: v1.0.0          ← version 不变
# ==================
```

## 类比5：模板组合时的变量合并

### 目标概念

使用 `+` 操作符组合模板时，LangChain 会自动合并 `input_variables` 和 `partial_variables`，并检查冲突。

[来源: reference/source_prompttemplate_01.md:28-59]

### 前端类比：Webpack 配置合并

在 Webpack 中，你可以合并多个配置对象：

```javascript
const { merge } = require('webpack-merge');

// 基础配置
const baseConfig = {
  entry: './src/index.js',
  module: {
    rules: [{ test: /\.js$/, use: 'babel-loader' }]
  }
};

// 开发配置
const devConfig = {
  mode: 'development',
  devtool: 'source-map'
};

// 合并配置
const finalConfig = merge(baseConfig, devConfig);
// 结果：包含所有配置项
```

**LangChain 的模板合并类似：**

```python
from langchain_core.prompts import PromptTemplate

# 模板1：包含 role 变量
template1 = PromptTemplate(
    template="System: {role}\n",
    input_variables=["role"]
)

# 模板2：包含 query 变量
template2 = PromptTemplate(
    template="User: {query}\n",
    input_variables=["query"]
)

# 合并模板：自动合并变量
combined = template1 + template2
# combined.input_variables = ["role", "query"]

result = combined.format(role="assistant", query="Hello")
```

### 日常生活类比：拼图游戏

想象你在玩拼图：

**拼图块1：**
```
[左上角] - 包含天空和云朵
需要的颜色：蓝色、白色
```

**拼图块2：**
```
[右上角] - 包含太阳
需要的颜色：黄色、橙色
```

**拼图块3：**
```
[左下角] - 包含草地
需要的颜色：绿色
```

**合并拼图：**
```
[完整图片]
需要的颜色：蓝色、白色、黄色、橙色、绿色
（自动合并所有需要的颜色）
```

### 代码对比示例

```python
from langchain_core.prompts import PromptTemplate

# ===== 场景：构建 RAG 系统的提示词 =====

# 模块1：系统角色
system_module = PromptTemplate(
    template="System: You are a {role}.\n",
    input_variables=["role"]
)

# 模块2：上下文信息
context_module = PromptTemplate(
    template="Context: {context}\n",
    input_variables=["context"]
)

# 模块3：用户查询
query_module = PromptTemplate(
    template="User: {query}\nAssistant:",
    input_variables=["query"]
)

# 组合模板：自动合并变量
rag_prompt = system_module + context_module + query_module

# 检查合并后的变量
print("合并后的变量:", rag_prompt.input_variables)
# 输出: ['role', 'context', 'query']

# 使用合并后的模板
result = rag_prompt.format(
    role="RAG assistant",
    context="LangChain is a framework for building LLM applications.",
    query="What is LangChain?"
)

print(result)
# 输出:
# System: You are a RAG assistant.
# Context: LangChain is a framework for building LLM applications.
# User: What is LangChain?
# Assistant:
```

**⚠️ 注意：Partial Variables 冲突检查**

```python
from langchain_core.prompts import PromptTemplate

# 模板1：预填充 system_name
template1 = PromptTemplate(
    template="System: {system_name}\n",
    input_variables=[],
    partial_variables={"system_name": "RAG Assistant"}
)

# 模板2：也预填充 system_name（冲突！）
template2 = PromptTemplate(
    template="Version: {system_name} v2.0\n",
    input_variables=[],
    partial_variables={"system_name": "Different Name"}
)

# 尝试合并会报错
try:
    combined = template1 + template2
except ValueError as e:
    print(f"错误: {e}")
    # 输出: 错误: Cannot have same variable partialed twice.
```

## 类比总结表

| 高级特性 | 前端类比 | 日常生活类比 | 核心价值 |
|---------|---------|-------------|---------|
| **Partial Variables** | React defaultProps | 快递单预填充 | 减少重复传递固定值 |
| **Template Composition** | Express 中间件链 | 乐高积木组合 | 模块化设计，灵活组合 |
| **Template Formats** | CSS 预处理器 | 填空题的三种形式 | 不同场景选择合适格式 |
| **函数动态值** | React useMemo | 智能手表动态显示 | 自动计算实时数据 |
| **变量合并** | Webpack 配置合并 | 拼图游戏 | 自动合并变量，检查冲突 |

## 实战应用场景

### 场景1：多语言 RAG 系统

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 基础模板（可复用）
base_template = PromptTemplate(
    template="System: {system_role}\nLanguage: {language}\n",
    input_variables=[],
    partial_variables={
        "system_role": "Multilingual RAG Assistant",
        "language": "Auto-detect"
    }
)

# 中文查询模板
chinese_query = PromptTemplate.from_template(
    "用户问题：{query}\n回答："
)

# 英文查询模板
english_query = PromptTemplate.from_template(
    "User Query: {query}\nAnswer:"
)

# 组合：中文 RAG
chinese_rag = base_template + chinese_query

# 组合：英文 RAG
english_rag = base_template + english_query

# 使用
print(chinese_rag.format(query="什么是 LangChain？"))
print(english_rag.format(query="What is LangChain?"))
```

### 场景2：带时间戳的日志系统

```python
from langchain_core.prompts import PromptTemplate
from datetime import datetime

# 日志模板（自动添加时间戳）
log_template = PromptTemplate(
    template="[{timestamp}] {level}: {message}",
    input_variables=["level", "message"],
    partial_variables={
        "timestamp": lambda: datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    }
)

# 使用
print(log_template.format(level="INFO", message="User query received"))
print(log_template.format(level="ERROR", message="Database connection failed"))
```
