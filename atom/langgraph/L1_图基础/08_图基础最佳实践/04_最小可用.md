## 4.1 状态定义：TypedDict + Reducer

### 核心概念

**状态（State）** 是图中所有节点共享的数据结构，使用 TypedDict 定义类型，使用 Reducer 函数聚合更新。

### 基础用法

```python
from typing_extensions import TypedDict, Annotated
from operator import add

# 定义状态
class State(TypedDict):
    messages: Annotated[list[str], add]  # 使用 add reducer 追加
    user_id: str                          # 普通字段，会被覆盖
    count: int                            # 普通字段，会被覆盖
```

**关键规则**：
- 没有 Reducer 的字段：后面的值覆盖前面的值
- 有 Reducer 的字段：使用 Reducer 函数聚合多个更新

### 代码示例

```python
"""
状态定义与更新示例
演示：TypedDict + Reducer 的基础用法
"""

from typing_extensions import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END

# ===== 1. 定义状态 =====
class WorkflowState(TypedDict):
    # 使用 add reducer，多个节点的更新会追加
    messages: Annotated[list[str], add]

    # 普通字段，后面的值覆盖前面的值
    user_id: str
    status: str
    count: int

# ===== 2. 定义节点 =====
def node_a(state: WorkflowState):
    """节点 A：返回部分状态更新"""
    return {
        "messages": ["A executed"],
        "count": 1
    }

def node_b(state: WorkflowState):
    """节点 B：返回部分状态更新"""
    return {
        "messages": ["B executed"],
        "count": 2  # 会覆盖 node_a 的 count
    }

# ===== 3. 构建图 =====
graph = StateGraph(WorkflowState)
graph.add_node("node_a", node_a)
graph.add_node("node_b", node_b)
graph.add_edge(START, "node_a")
graph.add_edge("node_a", "node_b")
graph.add_edge("node_b", END)

app = graph.compile()

# ===== 4. 执行 =====
result = app.invoke({
    "messages": [],
    "user_id": "user_123",
    "status": "pending",
    "count": 0
})

print("最终状态:")
print(f"messages: {result['messages']}")  # ["A executed", "B executed"] - 追加
print(f"count: {result['count']}")        # 2 - 覆盖
```

**运行输出**：
```
最终状态:
messages: ['A executed', 'B executed']
count: 2
```

## 4.2 节点函数规范：返回部分状态更新

### 核心概念

**节点函数** 接收当前状态，返回部分状态更新（不需要返回完整状态）。

### 基础用法

```python
def my_node(state: State) -> dict:
    """
    节点函数签名：
    - 输入：完整的 State
    - 输出：部分状态更新（dict）
    """
    # 访问状态
    user_input = state["user_input"]

    # 执行业务逻辑
    result = process(user_input)

    # 返回部分更新
    return {"result": result, "status": "completed"}
```

**关键规则**：
- 节点函数应该是纯函数或幂等的
- 不要直接修改输入的 state
- 只返回需要更新的字段

### 代码示例

```python
"""
节点函数规范示例
演示：多种节点签名和返回方式
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_core.runnables import RunnableConfig

class State(TypedDict):
    input: str
    result: str
    metadata: dict

# ===== 1. 基础节点：只接收 state =====
def basic_node(state: State) -> dict:
    """最简单的节点函数"""
    return {
        "result": f"Processed: {state['input']}",
        "metadata": {"node": "basic"}
    }

# ===== 2. 带配置的节点：接收 state + config =====
def node_with_config(state: State, config: RunnableConfig) -> dict:
    """可以访问运行时配置"""
    thread_id = config.get("configurable", {}).get("thread_id", "default")
    return {
        "result": f"Thread {thread_id}: {state['input']}",
        "metadata": {"thread_id": thread_id}
    }

# ===== 3. 条件节点：返回不同的更新 =====
def conditional_node(state: State) -> dict:
    """根据条件返回不同的更新"""
    if len(state["input"]) > 10:
        return {"result": "Long input", "metadata": {"length": "long"}}
    else:
        return {"result": "Short input", "metadata": {"length": "short"}}

# ===== 4. 构建图 =====
graph = StateGraph(State)
graph.add_node("basic", basic_node)
graph.add_node("with_config", node_with_config)
graph.add_node("conditional", conditional_node)

graph.add_edge(START, "basic")
graph.add_edge("basic", "with_config")
graph.add_edge("with_config", "conditional")
graph.add_edge("conditional", END)

app = graph.compile()

# ===== 5. 执行 =====
result = app.invoke(
    {"input": "Hello", "result": "", "metadata": {}},
    config={"configurable": {"thread_id": "thread_123"}}
)

print(f"Result: {result['result']}")
print(f"Metadata: {result['metadata']}")
```

**运行输出**：
```
Result: Short input
Metadata: {'length': 'short'}
```

## 4.3 错误处理：RetryPolicy

### 核心概念

**RetryPolicy** 用于处理临时性错误（网络错误、服务暂时不可用），自动重试失败的节点。

### 基础用法

```python
from langgraph.types import RetryPolicy

# 创建重试策略
retry_policy = RetryPolicy(
    max_attempts=3,              # 最大重试次数
    initial_interval=1.0,        # 初始重试间隔（秒）
    backoff_factor=2.0,          # 退避因子（指数退避）
    jitter=True,                 # 添加随机抖动
    retry_on=ValueError          # 只重试 ValueError
)

# 为节点配置重试策略
graph.add_node("api_call", api_call_node, retry_policy=retry_policy)
```

**关键规则**：
- 只重试临时性错误（网络错误、5xx 状态码）
- 不要重试编程错误（ValueError、TypeError）
- 使用指数退避避免雷鸣群效应

### 代码示例

```python
"""
错误处理与重试策略示例
演示：RetryPolicy 的配置和使用
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy
import random

class State(TypedDict):
    input: str
    result: str
    attempts: int

# ===== 1. 模拟可能失败的节点 =====
attempt_count = 0

def unreliable_api_call(state: State):
    """模拟不稳定的 API 调用"""
    global attempt_count
    attempt_count += 1

    # 前两次失败，第三次成功
    if attempt_count < 3:
        raise ConnectionError(f"API call failed (attempt {attempt_count})")

    return {
        "result": f"Success after {attempt_count} attempts",
        "attempts": attempt_count
    }

# ===== 2. 配置重试策略 =====
retry_policy = RetryPolicy(
    max_attempts=3,              # 最多重试 3 次
    initial_interval=0.1,        # 初始间隔 0.1 秒
    backoff_factor=2.0,          # 指数退避：0.1, 0.2, 0.4...
    jitter=False,                # 测试时关闭抖动
    retry_on=ConnectionError     # 只重试 ConnectionError
)

# ===== 3. 构建图 =====
graph = StateGraph(State)
graph.add_node("api_call", unreliable_api_call, retry_policy=retry_policy)
graph.add_edge(START, "api_call")
graph.add_edge("api_call", END)

app = graph.compile()

# ===== 4. 执行 =====
try:
    result = app.invoke({"input": "test", "result": "", "attempts": 0})
    print(f"Result: {result['result']}")
    print(f"Total attempts: {result['attempts']}")
except Exception as e:
    print(f"Failed: {e}")
```

**运行输出**：
```
Result: Success after 3 attempts
Total attempts: 3
```

### 实际应用场景

## 4.4 图结构设计：Builder 模式

### 核心概念

**StateGraph** 使用 Builder 模式构建图，必须调用 `.compile()` 才能执行。

### 基础用法

```python
from langgraph.graph import StateGraph, START, END

# 1. 创建 Builder
graph = StateGraph(State)

# 2. 添加节点
graph.add_node("node_a", node_a)
graph.add_node("node_b", node_b)

# 3. 添加边
graph.add_edge(START, "node_a")
graph.add_edge("node_a", "node_b")
graph.add_edge("node_b", END)

# 4. 编译成可执行图
app = graph.compile()

# 5. 执行
result = app.invoke({"input": "test"})
```

**关键规则**：
- StateGraph 是 Builder，不能直接执行
- 必须调用 `.compile()` 创建可执行图
- 编译后的图是不可变的

### 代码示例

```python
"""
图结构设计示例
演示：Builder 模式 + 条件路由 + 并行执行
"""

from typing_extensions import TypedDict, Literal
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    input: str
    path: str
    results: list[str]

# ===== 1. 定义节点 =====
def start_node(state: State):
    return {"path": "started"}

def route_decision(state: State) -> Literal["path_a", "path_b"]:
    """条件路由：根据输入长度决定路径"""
    if len(state["input"]) > 5:
        return "path_a"
    return "path_b"

def path_a_node(state: State):
    return {"results": ["Path A executed"], "path": "a"}

def path_b_node(state: State):
    return {"results": ["Path B executed"], "path": "b"}

def end_node(state: State):
    return {"path": "completed"}

# ===== 2. 构建图（Builder 模式）=====
graph = StateGraph(State)

# 添加节点
graph.add_node("start", start_node)
graph.add_node("path_a", path_a_node)
graph.add_node("path_b", path_b_node)
graph.add_node("end", end_node)

# 添加边
graph.add_edge(START, "start")

# 添加条件边
graph.add_conditional_edges(
    "start",
    route_decision,
    {
        "path_a": "path_a",
        "path_b": "path_b"
    }
)

graph.add_edge("path_a", "end")
graph.add_edge("path_b", "end")
graph.add_edge("end", END)

# ===== 3. 编译 =====
app = graph.compile()

# ===== 4. 执行不同的输入 =====
print("=== 短输入 ===")
result1 = app.invoke({"input": "hi", "path": "", "results": []})
print(f"Path: {result1['path']}")
print(f"Results: {result1['results']}")

print("\n=== 长输入 ===")
result2 = app.invoke({"input": "hello world", "path": "", "results": []})
print(f"Path: {result2['path']}")
print(f"Results: {result2['results']}")
```

**运行输出**：
```
=== 短输入 ===
Path: completed
Results: ['Path B executed']

=== 长输入 ===
Path: completed
Results: ['Path A executed']
```

### 实际应用场景

## 4.5 Context Schema：运行时上下文

### 核心概念

**Context Schema** 用于传递不可变的运行时参数（user_id、配置、数据库连接等），不会被 Checkpoint 保存。

### 基础用法

```python
from langgraph.graph import StateGraph
from langgraph.runtime import Runtime

# 定义 Context
class Context(TypedDict):
    user_id: str
    db_conn: object

# 创建图时指定 context_schema
graph = StateGraph(state_schema=State, context_schema=Context)

# 节点函数访问 context
def node(state: State, runtime: Runtime[Context]):
    user_id = runtime.context["user_id"]
    db_conn = runtime.context["db_conn"]
    # 使用 context...
    return {"result": f"User {user_id}"}

# 调用时传入 context
result = app.invoke(
    {"input": "test"},
    context={"user_id": "user_123", "db_conn": db}
)
```

**关键规则**：
- Context 在整个图执行过程中保持不变
- Context 不会被 Checkpoint 保存
- 适合存储：user_id、配置、数据库连接

### 代码示例

```python
"""
Context Schema 示例
演示：运行时上下文的使用
"""

from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.runtime import Runtime

# ===== 1. 定义 State 和 Context =====
class State(TypedDict):
    input: str
    result: str

class Context(TypedDict):
    user_id: str
    api_key: str
    max_retries: int

# ===== 2. 定义使用 Context 的节点 =====
def process_node(state: State, runtime: Runtime[Context]):
    """访问运行时上下文"""
    user_id = runtime.context["user_id"]
    api_key = runtime.context["api_key"]
    max_retries = runtime.context["max_retries"]

    return {
        "result": f"Processed for user {user_id} with {max_retries} retries"
    }

# ===== 3. 构建图 =====
graph = StateGraph(state_schema=State, context_schema=Context)
graph.add_node("process", process_node)
graph.add_edge(START, "process")
graph.add_edge("process", END)

app = graph.compile()

# ===== 4. 执行时传入 context =====
result = app.invoke(
    {"input": "test", "result": ""},
    context={
        "user_id": "user_123",
        "api_key": "sk-xxx",
        "max_retries": 3
    }
)

print(f"Result: {result['result']}")
```

**运行输出**：
```
Result: Processed for user user_123 with 3 retries
```
