# 实战代码 - 场景4：MMR选择（提高多样性）

**知识点：** ExampleSelector示例选择
**场景：** 使用 MaxMarginalRelevanceExampleSelector 提高示例多样性
**难度：** ⭐⭐⭐⭐
**代码行数：** ~400行

---

## 场景概述

在 Few-shot learning 中，如果选择的示例过于相似，会导致模型学习到的模式过于单一，影响泛化能力。**MaxMarginalRelevanceExampleSelector（MMR选择器）** 通过平衡相关性和多样性，选择既相关又多样化的示例，提高 Few-shot learning 的效果。

### 为什么需要 MMR 选择器？

**问题场景：**
```python
# 假设我们有一个客服问答系统
examples = [
    {"input": "如何退款？", "output": "请联系客服申请退款"},
    {"input": "怎么退钱？", "output": "请联系客服申请退款"},
    {"input": "退款流程是什么？", "output": "请联系客服申请退款"},
    {"input": "如何修改订单？", "output": "请在订单详情页点击修改"},
    {"input": "如何取消订单？", "output": "请在订单详情页点击取消"},
]

# 用户查询："我想退款"
# 语义相似度选择器会选择前3个示例（都是关于退款的）
# 但这3个示例几乎一模一样，没有提供多样化的信息！
```

**MMR 的解决方案：**
- 先选择最相关的示例
- 然后在剩余示例中选择与已选示例差异最大的
- 平衡相关性和多样性

---

## 核心原理

### MMR 算法

**公式：**
```
MMR = λ * Sim(query, doc) - (1-λ) * max(Sim(doc, selected_docs))
```

**参数说明：**
- `λ`：平衡因子（0-1之间）
  - λ=1：只考虑相关性（退化为语义相似度选择）
  - λ=0：只考虑多样性
  - λ=0.5：平衡相关性和多样性
- `Sim(query, doc)`：查询与文档的相似度
- `max(Sim(doc, selected_docs))`：文档与已选文档的最大相似度

**选择过程：**
1. 从候选集中选择与查询最相似的文档
2. 对于剩余文档，计算 MMR 分数
3. 选择 MMR 分数最高的文档
4. 重复步骤 2-3，直到选够 k 个文档

### LangChain 中的 MMR 实现

```python
class MaxMarginalRelevanceExampleSelector(_VectorStoreExampleSelector):
    """基于最大边际相关性选择示例

    论文：https://arxiv.org/pdf/2211.13892.pdf
    """

    fetch_k: int = 20  # 先获取 20 个候选
    k: int = 4  # 最终返回 4 个

    def select_examples(self, input_variables: dict[str, str]) -> list[dict]:
        # 使用 MMR 算法选择示例
        example_docs = self.vectorstore.max_marginal_relevance_search(
            self._example_to_text(input_variables, self.input_keys),
            k=self.k,
            fetch_k=self.fetch_k,  # 先获取更多候选
        )
        return self._documents_to_examples(example_docs)
```

**关键参数：**
- `fetch_k`：先从向量存储中获取 fetch_k 个最相似的候选
- `k`：从候选中使用 MMR 算法选择 k 个最终示例
- `fetch_k` 越大，多样性越好，但计算成本越高

def calculate_diversity(examples: List[Dict[str, str]]) -> float:
    """计算示例集的多样性（平均余弦距离）"""
    from sklearn.metrics.pairwise import cosine_similarity
    import numpy as np

    # 获取所有示例的 embeddings
    texts = [ex["input"] for ex in examples]
    embeddings_list = embeddings.embed_documents(texts)

    # 计算两两之间的余弦相似度
    similarity_matrix = cosine_similarity(embeddings_list)

    # 计算平均相似度（排除对角线）
    n = len(examples)
    total_similarity = (similarity_matrix.sum() - n) / (n * (n - 1))

    # 多样性 = 1 - 平均相似度
    diversity = 1 - total_similarity

    return diversity

# 对比语义相似度和 MMR 的多样性

mmr_examples = mmr_selector.select_examples({"input": test_query})

print(f"语义相似度选择器的多样性：{semantic_diversity:.4f}")
print(f"MMR 选择器的多样性：{mmr_diversity:.4f}")
print(f"多样性提升：{(mmr_diversity - semantic_diversity) / semantic_diversity * 100:.2f}%\n")

## 关键要点

### 1. MMR 的核心优势

**平衡相关性和多样性：**
- 不仅选择最相关的示例
- 还确保示例之间有足够的差异
- 避免信息冗余

### 2. fetch_k 参数的重要性

**fetch_k 的作用：**
```python
# fetch_k 太小：多样性不足
mmr_selector = MaxMarginalRelevanceExampleSelector(
    k=4,
    fetch_k=5,  # 只从 5 个候选中选择，多样性有限
)

# fetch_k 太大：计算成本高
mmr_selector = MaxMarginalRelevanceExampleSelector(
    k=4,
    fetch_k=50,  # 从 50 个候选中选择，计算量大
)

# 推荐配置
mmr_selector = MaxMarginalRelevanceExampleSelector(
    k=4,
    fetch_k=10,  # fetch_k = 2-3 * k
)
```

### 3. 适用场景

**适合使用 MMR 的场景：**
- 示例库中有大量相似示例
- 需要展示多样化的解决方案
- 避免模型学习到过于单一的模式
- 对话系统（增加回复的多样性）

**不适合使用 MMR 的场景：**
- 示例库很小（< 10 个）
- 所有示例都高度相关且必要
- 对性能要求极高的场景
