# OpenClaw 安装与配置 - 核心概念 1：npm 全局安装

**核心概念：** npm/pnpm 全局安装机制和 PATH 环境变量

本文档深入讲解 npm/pnpm 全局安装的工作原理，帮助你理解 `npm install -g openclaw` 背后发生了什么。

---

## 1. npm 全局安装的本质

### 1.1 什么是全局安装？

**全局安装** = 将包安装到系统级目录，使其可执行文件在任何位置都能调用

```bash
# 本地安装（项目依赖）
npm install openclaw
# 安装到：./node_modules/openclaw/

# 全局安装（系统工具）
npm install -g openclaw
# 安装到：/usr/local/lib/node_modules/openclaw/
```

### 1.2 全局安装的三个关键步骤

```typescript
// npm install -g openclaw 的执行流程

// 步骤 1：下载包
// npm 从 registry 下载 openclaw@latest.tgz

// 步骤 2：解压到全局目录
// 解压到：/usr/local/lib/node_modules/openclaw/

// 步骤 3：创建可执行文件链接
// 读取 package.json 的 bin 字段
// 创建符号链接：/usr/local/bin/openclaw -> ../lib/node_modules/openclaw/dist/cli.js
```

### 1.3 全局目录位置

**不同平台的全局目录：**

```typescript
import { execSync } from 'child_process';
import * as os from 'os';

/**
 * 获取 npm 全局目录
 */
function getNpmGlobalPaths(): {
  prefix: string;
  nodeModules: string;
  bin: string;
} {
  const platform = os.platform();

  // 获取 npm prefix（全局安装根目录）
  const prefix = execSync('npm config get prefix', {
    encoding: 'utf-8'
  }).trim();

  // 不同平台的目录结构
  let nodeModules: string;
  let bin: string;

  if (platform === 'win32') {
    // Windows
    nodeModules = `${prefix}\\node_modules`;
    bin = prefix; // Windows 的 bin 就是 prefix
  } else {
    // macOS / Linux
    nodeModules = `${prefix}/lib/node_modules`;
    bin = `${prefix}/bin`;
  }

  return { prefix, nodeModules, bin };
}

// 示例输出
const paths = getNpmGlobalPaths();
console.log('npm prefix:', paths.prefix);
// macOS: /usr/local
// Linux: /usr/local 或 /usr
// Windows: C:\Users\<user>\AppData\Roaming\npm

console.log('全局 node_modules:', paths.nodeModules);
// macOS: /usr/local/lib/node_modules
// Linux: /usr/local/lib/node_modules
// Windows: C:\Users\<user>\AppData\Roaming\npm\node_modules

console.log('全局 bin:', paths.bin);
// macOS: /usr/local/bin
// Linux: /usr/local/bin
// Windows: C:\Users\<user>\AppData\Roaming\npm
```

## 2. package.json 的 bin 字段

### 2.1 bin 字段的作用

**bin 字段** 告诉 npm 哪些文件应该作为可执行命令

```json
// openclaw/package.json
{
  "name": "openclaw",
  "version": "1.2.3",
  "bin": {
    "openclaw": "./dist/cli.js"
  }
}
```

**含义：**
- 命令名：`openclaw`
- 可执行文件：`./dist/cli.js`（相对于 package.json）

### 2.2 单个命令 vs 多个命令

```json
// 单个命令（简写形式）
{
  "name": "openclaw",
  "bin": "./dist/cli.js"
  // 命令名默认为包名：openclaw
}

// 多个命令（对象形式）
{
  "name": "openclaw",
  "bin": {
    "openclaw": "./dist/cli.js",
    "openclaw-gateway": "./dist/gateway.js",
    "openclaw-agent": "./dist/agent.js"
  }
}
```

### 2.3 可执行文件的 Shebang

**Shebang** 告诉操作系统用哪个解释器执行文件

```javascript
#!/usr/bin/env node
// ^^^ Shebang 行，必须在文件第一行

// OpenClaw CLI 入口文件
import { Command } from 'commander';

const program = new Command();
program
  .name('openclaw')
  .version('1.2.3')
  .description('OpenClaw CLI');

program.parse();
```

**为什么用 `#!/usr/bin/env node` 而不是 `#!/usr/bin/node`？**
- `#!/usr/bin/node`：硬编码 node 路径，可能不存在
- `#!/usr/bin/env node`：在 PATH 中查找 node，更灵活

## 3. 符号链接机制

### 3.1 什么是符号链接？

**符号链接（Symbolic Link）** = 指向另一个文件的快捷方式

```bash
# 查看 openclaw 命令的符号链接
ls -l $(which openclaw)

# 输出示例（macOS）：
# /usr/local/bin/openclaw -> ../lib/node_modules/openclaw/dist/cli.js
```

### 3.2 符号链接的创建过程

```typescript
import * as fs from 'fs';
import * as path from 'path';

/**
 * 模拟 npm 创建符号链接的过程
 */
function createBinLink(
  packageName: string,
  binName: string,
  binPath: string,
  globalBinDir: string,
  globalNodeModules: string
): void {
  // 1. 可执行文件的完整路径
  const targetPath = path.join(globalNodeModules, packageName, binPath);

  // 2. 符号链接的路径
  const linkPath = path.join(globalBinDir, binName);

  // 3. 计算相对路径（符号链接使用相对路径更灵活）
  const relativePath = path.relative(globalBinDir, targetPath);

  // 4. 创建符号链接
  if (fs.existsSync(linkPath)) {
    fs.unlinkSync(linkPath); // 删除旧链接
  }

  fs.symlinkSync(relativePath, linkPath);

  // 5. 设置可执行权限（Unix/Linux）
  if (process.platform !== 'win32') {
    fs.chmodSync(linkPath, 0o755); // rwxr-xr-x
  }

  console.log(`✅ 创建符号链接: ${linkPath} -> ${relativePath}`);
}

// 示例：为 OpenClaw 创建符号链接
createBinLink(
  'openclaw',                    // 包名
  'openclaw',                    // 命令名
  'dist/cli.js',                 // 可执行文件路径
  '/usr/local/bin',              // 全局 bin 目录
  '/usr/local/lib/node_modules'  // 全局 node_modules 目录
);

// 输出：
// ✅ 创建符号链接: /usr/local/bin/openclaw -> ../lib/node_modules/openclaw/dist/cli.js
```

### 3.3 Windows 的特殊处理

Windows 不支持 Unix 风格的符号链接，npm 使用 **批处理文件** 和 **PowerShell 脚本**

```batch
REM openclaw.cmd（批处理文件）
@ECHO off
SETLOCAL
SET "NODE_EXE=%~dp0\node.exe"
IF NOT EXIST "%NODE_EXE%" (
  SET "NODE_EXE=node"
)

"%NODE_EXE%" "%~dp0\node_modules\openclaw\dist\cli.js" %*
```

```powershell
# openclaw.ps1（PowerShell 脚本）
#!/usr/bin/env pwsh
$basedir=Split-Path $MyInvocation.MyCommand.Definition -Parent

$exe=""
if ($PSVersionTable.PSVersion -lt "6.0" -or $IsWindows) {
  $exe=".exe"
}
$ret=0

& "node$exe" "$basedir/node_modules/openclaw/dist/cli.js" $args
exit $ret
```

## 4. PATH 环境变量

### 4.1 什么是 PATH？

**PATH** = 操作系统查找可执行文件的目录列表

```bash
# 查看 PATH 环境变量
echo $PATH

# 输出示例（macOS）：
# /usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
```

### 4.2 PATH 的查找顺序

```typescript
import { execSync } from 'child_process';

/**
 * 模拟 Shell 在 PATH 中查找命令的过程
 */
function findCommandInPath(command: string): string | null {
  // 1. 获取 PATH 环境变量
  const pathEnv = process.env.PATH || '';

  // 2. 分割成目录列表
  const pathDirs = pathEnv.split(process.platform === 'win32' ? ';' : ':');

  // 3. 按顺序查找
  for (const dir of pathDirs) {
    const fullPath = path.join(dir, command);

    // 检查文件是否存在且可执行
    try {
      const stats = fs.statSync(fullPath);
      if (stats.isFile() && (stats.mode & 0o111)) {
        return fullPath; // 找到了！
      }
    } catch {
      // 文件不存在，继续查找
    }
  }

  return null; // 未找到
}

// 示例：查找 openclaw 命令
const openclawPath = findCommandInPath('openclaw');
if (openclawPath) {
  console.log(`✅ 找到 openclaw: ${openclawPath}`);
} else {
  console.log('❌ 未找到 openclaw');
}
```

### 4.3 如何将目录添加到 PATH？

**临时添加（当前会话）：**
```bash
# macOS / Linux
export PATH="/usr/local/bin:$PATH"

# Windows (CMD)
set PATH=C:\Users\<user>\AppData\Roaming\npm;%PATH%

# Windows (PowerShell)
$env:PATH = "C:\Users\<user>\AppData\Roaming\npm;$env:PATH"
```

**永久添加（所有会话）：**
```bash
# macOS / Linux (Bash)
echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# macOS / Linux (Zsh)
echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# Windows
# 通过"系统属性" -> "环境变量"手动添加
```

## 5. pnpm 全局安装的区别

### 5.1 pnpm 的存储机制

**pnpm** 使用 **内容寻址存储（Content-Addressable Storage）**

```bash
# pnpm 全局存储位置
pnpm store path
# 输出：~/.pnpm-store/v3

# pnpm 全局安装位置
pnpm root -g
# 输出：~/.pnpm-global/5/node_modules
```

### 5.2 pnpm 的硬链接机制

```typescript
/**
 * pnpm 的存储结构
 */
interface PnpmStore {
  // 内容寻址存储（所有包的唯一副本）
  store: {
    // 包内容按哈希存储
    'openclaw@1.2.3': '/Users/username/.pnpm-store/v3/files/00/abc123...',
  };

  // 全局安装（硬链接到 store）
  global: {
    'openclaw': {
      // 硬链接到 store 中的文件
      'dist/cli.js': 'hardlink -> store/openclaw@1.2.3/dist/cli.js',
    },
  };
}
```

**硬链接 vs 符号链接：**
| 特性 | 硬链接 | 符号链接 |
|------|--------|----------|
| 指向 | 文件内容（inode） | 文件路径 |
| 删除源文件 | 硬链接仍然有效 | 符号链接失效 |
| 跨文件系统 | 不支持 | 支持 |
| 磁盘空间 | 不占用额外空间 | 占用少量空间 |

### 5.3 pnpm vs npm 全局安装对比

```typescript
/**
 * 对比 npm 和 pnpm 的全局安装
 */
function compareGlobalInstall(): void {

  console.log('链接方式: 符号链接');
  console.log('磁盘占用: 每个包独立存储');
```

## 6. 全局安装的完整流程

### 6.1 npm install -g openclaw 的完整流程

```typescript
import { execSync } from 'child_process';
import * as fs from 'fs';
import * as path from 'path';
import * as https from 'https';
import * as tar from 'tar';

/**
 * 模拟 npm install -g openclaw 的完整流程
 */
async function simulateGlobalInstall(packageName: string): Promise<void> {
  console.log(`📦 开始全局安装 ${packageName}...\n`);

  // ===== 步骤 1：获取包信息 =====
  console.log('步骤 1：从 npm registry 获取包信息');
  const registryUrl = `https://registry.npmjs.org/${packageName}`;
  const packageInfo = await fetchJson(registryUrl);
  const latestVersion = packageInfo['dist-tags'].latest;
  const tarballUrl = packageInfo.versions[latestVersion].dist.tarball;

  console.log(`  版本: ${latestVersion}`);
  console.log(`  下载地址: ${tarballUrl}\n`);

  // ===== 步骤 2：下载 tarball =====
  console.log('步骤 2：下载 tarball');
  const tarballPath = `/tmp/${packageName}-${latestVersion}.tgz`;
  await downloadFile(tarballUrl, tarballPath);
  console.log(`  已下载到: ${tarballPath}\n`);

  // ===== 步骤 3：解压到全局目录 =====
  console.log('步骤 3：解压到全局 node_modules');
  const globalNodeModules = execSync('npm root -g', {
    encoding: 'utf-8'
  }).trim();
  const installDir = path.join(globalNodeModules, packageName);

  // 删除旧版本
  if (fs.existsSync(installDir)) {
    fs.rmSync(installDir, { recursive: true });
  }

  // 解压
  await tar.extract({
    file: tarballPath,
    cwd: globalNodeModules,
    strip: 1 // 去掉 package/ 前缀
  });

  console.log(`  已解压到: ${installDir}\n`);

  // ===== 步骤 4：读取 package.json =====
  console.log('步骤 4：读取 package.json');
  const packageJsonPath = path.join(installDir, 'package.json');
  const packageJson = JSON.parse(fs.readFileSync(packageJsonPath, 'utf-8'));

  console.log(`  包名: ${packageJson.name}`);
  console.log(`  版本: ${packageJson.version}`);
  console.log(`  bin: ${JSON.stringify(packageJson.bin)}\n`);

  // ===== 步骤 5：创建符号链接 =====
  console.log('步骤 5：创建符号链接');
  const globalBin = execSync('npm bin -g', { encoding: 'utf-8' }).trim();

  if (typeof packageJson.bin === 'string') {
    // 单个命令
    createSymlink(
      packageJson.name,
      packageJson.bin,
      installDir,
      globalBin
    );
  } else if (typeof packageJson.bin === 'object') {
    // 多个命令
    for (const [binName, binPath] of Object.entries(packageJson.bin)) {
      createSymlink(binName, binPath as string, installDir, globalBin);
    }
  }

  // ===== 步骤 6：运行 postinstall 脚本 =====
  if (packageJson.scripts && packageJson.scripts.postinstall) {
    console.log('\n步骤 6：运行 postinstall 脚本');
    execSync('npm run postinstall', {
      cwd: installDir,
      stdio: 'inherit'
    });
  }

  console.log('\n✅ 全局安装完成！');
  console.log(`\n验证安装:`);
  console.log(`  ${packageJson.name} --version`);
}

/**
 * 创建符号链接
 */
function createSymlink(
  binName: string,
  binPath: string,
  installDir: string,
  globalBin: string
): void {
  const targetPath = path.join(installDir, binPath);
  const linkPath = path.join(globalBin, binName);

  // 计算相对路径
  const relativePath = path.relative(globalBin, targetPath);

  // 创建符号链接
  if (fs.existsSync(linkPath)) {
    fs.unlinkSync(linkPath);
  }

  fs.symlinkSync(relativePath, linkPath);

  // 设置可执行权限
  if (process.platform !== 'win32') {
    fs.chmodSync(targetPath, 0o755);
  }

  console.log(`  ✅ ${binName} -> ${relativePath}`);
}

// 辅助函数
async function fetchJson(url: string): Promise<any> {
  return new Promise((resolve, reject) => {
    https.get(url, (res) => {
      let data = '';
      res.on('data', (chunk) => (data += chunk));
      res.on('end', () => resolve(JSON.parse(data)));
      res.on('error', reject);
    });
  });
}

async function downloadFile(url: string, dest: string): Promise<void> {
  return new Promise((resolve, reject) => {
    const file = fs.createWriteStream(dest);
    https.get(url, (res) => {
      res.pipe(file);
      file.on('finish', () => {
        file.close();
        resolve();
      });
      file.on('error', reject);
    });
  });
}

// 运行示例
simulateGlobalInstall('openclaw');
```

```
步骤 2：下载 tarball

  已解压到: /usr/local/lib/node_modules/openclaw
```

## 7. 常见问题与解决方案

### 7.1 问题：openclaw: command not found

**原因：** 全局 bin 目录不在 PATH 中

**解决方案：**
```bash
# 1. 查看全局 bin 目录
npm bin -g
# 输出：/usr/local/bin

# 2. 检查 PATH
echo $PATH | grep "/usr/local/bin"

# 3. 如果不在 PATH 中，添加
echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### 7.2 问题：权限错误（EACCES）

**原因：** 没有权限写入全局目录

**解决方案：**
```bash
# 方案 1：使用 sudo（不推荐）
sudo npm install -g openclaw

# 方案 2：修改 npm prefix（推荐）
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

# 方案 3：使用 nvm（最推荐）
nvm install 22
nvm use 22
npm install -g openclaw  # 无需 sudo
```

### 7.3 问题：多个版本冲突

**原因：** 全局只能安装一个版本

**解决方案：**
```bash
# 方案 1：使用 npx（临时使用特定版本）
npx openclaw@1.2.3 --version

# 方案 2：使用 nvm 切换 Node.js 版本
nvm use 20  # 使用 Node.js 20 的全局包
nvm use 22  # 使用 Node.js 22 的全局包

# 方案 3：使用别名
npm install -g openclaw@1.2.3
mv $(npm bin -g)/openclaw $(npm bin -g)/openclaw-1.2.3
npm install -g openclaw@latest
# 现在可以同时使用 openclaw 和 openclaw-1.2.3
```
