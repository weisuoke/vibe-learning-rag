## 30 字核心

## 一、serde — 数据的通用翻译官

**一句话**：serde 让 struct 自动获得序列化/反序列化能力，一个 derive 搞定 JSON、TOML、YAML 等所有格式。

```rust
// 只需一行 derive，struct 就能「变成」JSON / TOML / 任意格式
#[derive(Serialize, Deserialize)]
struct Config {
    name: String,
    port: u16,
}

// 从 TOML 读入
let config: Config = toml::from_str("name = 'app'\nport = 8080")?;
// 写成 JSON
let json = serde_json::to_string(&config)?;
```

**前端对照**：
```
TypeScript                           Rust + serde
─────────────                        ──────────────
JSON.parse(str)                  →   serde_json::from_str::<T>(str)
JSON.stringify(obj)              →   serde_json::to_string(&obj)
zod.object({ name: z.string() }) →  #[derive(Deserialize)] struct T { name: String }
```

**ZeroClaw 中的体现**：`config/schema.rs` 中 20+ 个配置段全部使用 `#[derive(Serialize, Deserialize)]` + 属性宏（`#[serde(default)]`、`#[serde(alias)]`、`#[serde(skip)]`）。

## 二、anyhow — 错误处理的瑞士军刀

**一句话**：anyhow 让你不再纠结错误类型，`anyhow::Result<T>` 能接住任何错误，`.context()` 自动构建错误链。

```rust
use anyhow::{Result, bail, ensure, Context};

fn load_config(path: &str) -> Result<Config> {
    let content = fs::read_to_string(path)
        .context("failed to read config file")?;    // 附上下文
    ensure!(!content.is_empty(), "config is empty"); // 条件断言
    let config: Config = toml::from_str(&content)
        .context("failed to parse config")?;         // 附上下文
    Ok(config)
}
```

**前端对照**：
```
TypeScript                           Rust + anyhow
─────────────                        ──────────────
throw new Error("msg")           →   bail!("msg")
if (!ok) throw new Error("msg")  →   ensure!(ok, "msg")
catch (e) { throw new Error(     →   .context("msg")?
  "context", { cause: e }) }
Promise<T>                       →   anyhow::Result<T>
```

**ZeroClaw 中的体现**：几乎所有函数返回 `anyhow::Result<T>`，四大 trait（Tool/Memory/Channel/Provider）统一使用 `anyhow::Result`。

---

## 三、clap — 声明式 CLI 构建器

**一句话**：clap 用 derive 宏把 struct/enum 变成 CLI 参数定义，编译时自动生成解析逻辑和帮助信息。

```rust
use clap::{Parser, Subcommand};

#[derive(Parser)]
#[command(name = "mytool", version, about)]
struct Cli {
    #[arg(short, long)]
    verbose: bool,
    #[command(subcommand)]
    command: Cmd,
}

#[derive(Subcommand)]
enum Cmd {
    /// Run the server
    Serve { #[arg(long, default_value = "8080")] port: u16 },
    /// Run tests
    Test,
}

fn main() { let cli = Cli::parse(); }
```

**前端对照**：
```
TypeScript / yargs                   Rust + clap
─────────────                        ──────────────
yargs.command('serve', ...)      →   #[derive(Subcommand)] enum { Serve }
.option('port', { default: 8080 }) → #[arg(long, default_value = "8080")]
.parse()                         →   Cli::parse()
.help()                          →   自动生成（/// 注释 = help 文本）
```

**ZeroClaw 中的体现**：`main.rs` 中定义了 30+ 子命令（Onboard、Agent、Service、Config…），用 `Parser` + `Subcommand` + `ValueEnum` 三个 derive 宏实现。

---

## 四、tracing — 结构化日志系统

**一句话**：tracing 用宏替代 `println!`，支持日志级别过滤和结构化输出，是 Rust 项目的标准日志方案。

```rust
use tracing::{info, warn, debug, error};

info!("Server started on port {}", port);
debug!(user_id = %id, "Processing request");     // 结构化字段
warn!("Rate limit: {}/100", count);
error!("Connection failed: {}", err);
```

**前端对照**：
```
TypeScript / winston                 Rust + tracing
─────────────                        ──────────────
console.log()                    →   info!() / debug!()
logger.warn()                    →   warn!()
logger.error()                   →   error!()
LOG_LEVEL=debug                  →   RUST_LOG=debug
```

**ZeroClaw 中的体现**：各模块广泛使用 `tracing::debug!`、`tracing::info!`、`tracing::warn!` 输出运行时信息。

## 五、前端对照速查表

| 你在 TS 中做的事 | Rust + 四大库对应 |
|-----------------|------------------|
| `JSON.parse(str)` | `serde_json::from_str::<T>(&str)?` |
| `JSON.stringify(obj)` | `serde_json::to_string(&obj)?` |
| `zod.object({...}).parse(data)` | `#[derive(Deserialize)]` + `serde_json::from_str` |
| `try { ... } catch (e) { throw new Error("ctx", { cause: e }) }` | `something().context("ctx")?` |
| `throw new Error("msg")` | `bail!("msg")` |
| `if (!ok) throw ...` | `ensure!(ok, "msg")` |
| `yargs.command('cmd').parse()` | `#[derive(Parser)]` + `Cli::parse()` |
| `commander.option('-v, --verbose')` | `#[arg(short, long)] verbose: bool` |
| `winston.info("msg")` | `info!("msg")` |
| `LOG_LEVEL=debug node app.js` | `RUST_LOG=debug cargo run` |

## 六、TypeScript → Rust 心智模型转换

### 最核心的思维转变

```
TypeScript 世界：                    Rust + 四大库世界：
─────────────                        ──────────────
运行时解析 + 运行时校验               编译时代码生成 + 编译时类型检查
  JSON.parse → any → zod 校验           serde derive → 类型安全的解析
  运行时才知道字段对不对               编译时就知道字段对不对

可选的错误处理                        强制的错误处理
  catch 写不写都行                       Result 不处理编译不过
  error 是 any                          error 有类型 + 上下文链

命令式 API 调用                       声明式 derive 宏
  yargs.command().option().parse()       #[derive(Parser)] struct Cli {}
  代码定义行为                          类型定义行为，代码由编译器生成
```

### 一个等式记住全部

```
serde  = JSON.parse + JSON.stringify + zod    （数据层）
anyhow = try/catch 增强版 + error-cause chain  （错误层）
clap   = yargs + commander + 自动 --help       （入口层）
tracing = winston + 日志级别过滤               （观测层）
```
