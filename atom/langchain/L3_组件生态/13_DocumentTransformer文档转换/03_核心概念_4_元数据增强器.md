# 核心概念 4：元数据增强器

## 概念定义

**元数据增强器是通过 AI 或规则自动从文档内容中提取结构化元数据的转换器，让"哑文档"变成"自带标签的智能文档"，大幅提升 RAG 检索的精度和可过滤性。**

[来源: Context7 + 网络搜索]

在 RAG 系统中，文档不仅仅是一段文本。如果每个文档都带有"主题"、"语言"、"情感"等标签，检索时就可以先按标签过滤，再做向量搜索，效果远好于纯向量检索。

---

## 为什么元数据在 RAG 中至关重要？

### 问题场景

```python
# 场景：你有一个多语言、多主题的知识库
documents = [
    Document(page_content="Python 是一种解释型编程语言..."),
    Document(page_content="React is a JavaScript library for building UIs..."),
    Document(page_content="今日股市大涨，上证指数突破3500点..."),
    Document(page_content="The latest AI research shows that..."),
]

# 用户问："Python 有哪些特性？"
# 纯向量检索可能返回：
# 1. Python 文档 ✅
# 2. React 文档 ❌（都是编程相关，向量相似）
# 3. AI 研究文档 ❌（包含技术词汇，向量相似）

# 如果有元数据标签：
# topic=programming, language=zh → 精准过滤后再检索
```

### 元数据的三大价值

| 价值 | 说明 | RAG 场景 |
|------|------|----------|
| **过滤** | 按标签缩小检索范围 | 只搜中文文档、只搜技术文档 |
| **路由** | 根据标签选择不同处理策略 | 不同主题用不同 Prompt |
| **上下文** | 为 LLM 提供额外信息 | 告诉 LLM 文档来源、时间、可信度 |

### 手动 vs 自动标注

```python
# ❌ 手动标注：费时费力，不可扩展
doc = Document(
    page_content="Python 是一种解释型编程语言...",
    metadata={"topic": "programming", "language": "zh", "sentiment": "neutral"}
    # 每篇文档都要人工标注？不现实！
)

# ✅ 自动标注：用 AI 自动提取元数据
# OpenAIMetadataTagger / DoctranPropertyExtractor
# 批量处理，自动化，可扩展
```

## OpenAIMetadataTagger：AI 驱动的元数据标注

### 核心原理

**OpenAIMetadataTagger** 利用 OpenAI 的 Function Calling 能力，根据你定义的 JSON Schema，自动从文档内容中提取结构化元数据。

```
文档内容 → OpenAI API (Function Calling) → 结构化元数据 → 写入 metadata
```

[来源: Context7 + 网络搜索]

### 工作流程

```
1. 你定义 JSON Schema（描述要提取什么元数据）
   ↓
2. OpenAIMetadataTagger 将文档内容 + Schema 发送给 OpenAI
   ↓
3. OpenAI 通过 Function Calling 返回结构化数据
   ↓
4. 结构化数据写入文档的 metadata 字段
```

### 基础使用

```python
from langchain_core.documents import Document
from langchain_community.document_transformers.openai_functions import (
    create_metadata_tagger,
)
from langchain_openai import ChatOpenAI

# ===== 1. 定义元数据 Schema =====
# 告诉 AI 你想提取什么信息
metadata_schema = {
    "properties": {
        "topic": {
            "type": "string",
            "enum": ["technology", "finance", "health", "education", "other"],
            "description": "The main topic of the document",
        },
        "language": {
            "type": "string",
            "enum": ["zh", "en", "ja", "ko"],
            "description": "The language of the document",
        },
        "sentiment": {
            "type": "string",
            "enum": ["positive", "negative", "neutral"],
            "description": "The overall sentiment of the document",
        },
        "key_entities": {
            "type": "array",
            "items": {"type": "string"},
            "description": "Key entities mentioned in the document",
        },
    },
    "required": ["topic", "language", "sentiment"],
}

# ===== 2. 创建标注器 =====
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
tagger = create_metadata_tagger(
    metadata_schema=metadata_schema,
    llm=llm,
)

# ===== 3. 准备文档 =====
documents = [
    Document(page_content="Python 3.12 引入了新的类型语法，让类型注解更简洁。"),
    Document(page_content="Today's stock market saw a significant rally..."),
    Document(page_content="最新研究表明，每天运动30分钟可以降低心脏病风险。"),
]

# ===== 4. 自动标注 =====
tagged_documents = tagger.transform_documents(documents)

# ===== 5. 查看结果 =====
for doc in tagged_documents:
    print(f"内容: {doc.page_content[:30]}...")
    print(f"元数据: {doc.metadata}")
    print("---")

# 预期输出：
# 内容: Python 3.12 引入了新的类型语法...
# 元数据: {'topic': 'technology', 'language': 'zh', 'sentiment': 'neutral',
#          'key_entities': ['Python', 'Python 3.12']}
# ---
# 内容: Today's stock market saw a sig...
# 元数据: {'topic': 'finance', 'language': 'en', 'sentiment': 'positive',
#          'key_entities': ['stock market']}
# ---
# 内容: 最新研究表明，每天运动30分钟可以降低...
# 元数据: {'topic': 'health', 'language': 'zh', 'sentiment': 'positive',
#          'key_entities': ['心脏病']}
```

### 使用 Pydantic 定义 Schema（推荐）

```python
from pydantic import BaseModel, Field
from typing import List, Optional

# 用 Pydantic 定义更清晰的 Schema
class DocumentMetadata(BaseModel):
    """文档元数据结构"""
    topic: str = Field(
        description="文档主题",
        enum=["technology", "finance", "health", "education", "other"]
    )
    language: str = Field(
        description="文档语言",
        enum=["zh", "en", "ja", "ko"]
    )
    sentiment: str = Field(
        description="文档情感倾向",
        enum=["positive", "negative", "neutral"]
    )
    key_entities: List[str] = Field(
        default_factory=list,
        description="文档中提到的关键实体"
    )
    summary: Optional[str] = Field(
        default=None,
        description="文档的一句话摘要"
    )

# 使用 Pydantic Schema 创建标注器
tagger = create_metadata_tagger(
    metadata_schema=DocumentMetadata,
    llm=llm,
)
```

## DoctranPropertyExtractor：属性提取器

### 核心原理

**DoctranPropertyExtractor** 使用 Doctran 库从文档中提取自定义属性。与 OpenAIMetadataTagger 类似，但通过 Doctran 库提供了更灵活的属性定义方式。

[来源: sourcecode/langchain]

### 基础使用

```python
from langchain_community.document_transformers import DoctranPropertyExtractor
from langchain_core.documents import Document

# ===== 1. 定义要提取的属性 =====
properties = [
    {
        "name": "category",
        "description": "The category of the document content",
        "type": "string",
        "enum": ["技术", "金融", "医疗", "教育"],
        "required": True,
    },
    {
        "name": "confidence_level",
        "description": "How confident the content appears to be",
        "type": "string",
        "enum": ["high", "medium", "low"],
        "required": True,
    },
    {
        "name": "target_audience",
        "description": "The intended audience for this content",
        "type": "string",
        "required": False,
    },
]

# ===== 2. 创建属性提取器 =====
property_extractor = DoctranPropertyExtractor(
    properties=properties,
    openai_api_key="your-api-key",  # 需要 OpenAI API key
)

# ===== 3. 提取属性 =====
documents = [
    Document(page_content="本教程面向 Python 初学者，介绍基础语法和数据类型。"),
]

extracted_docs = property_extractor.transform_documents(documents)

for doc in extracted_docs:
    print(f"提取的属性: {doc.metadata}")

# 预期输出：
# 提取的属性: {
#     'category': '技术',
#     'confidence_level': 'high',
#     'target_audience': 'Python 初学者'
# }
```

### OpenAIMetadataTagger vs DoctranPropertyExtractor

| 特性 | OpenAIMetadataTagger | DoctranPropertyExtractor |
|------|---------------------|--------------------------|
| **底层机制** | OpenAI Function Calling | Doctran 库 |
| **Schema 定义** | JSON Schema / Pydantic | 属性列表 |
| **灵活性** | 高（支持嵌套结构） | 中等 |
| **依赖** | `langchain-openai` | `doctran` |
| **推荐场景** | 生产环境 | 快速原型 |

## DoctranQATransformer：问答对提取

### 核心原理

**DoctranQATransformer** 自动从文档内容中提取问答对（Q&A pairs），并存储在文档的 metadata 中。这在 RAG 系统中非常有用——可以用提取的问题来增强检索匹配。

[来源: Context7]

### 工作流程

```
文档内容: "Python 是一种解释型语言，支持多种编程范式..."
    ↓
DoctranQATransformer 分析
    ↓
提取的 Q&A 对:
  Q: "Python 是什么类型的语言？"
  A: "Python 是一种解释型语言。"
  Q: "Python 支持哪些编程范式？"
  A: "Python 支持多种编程范式。"
    ↓
存储到 metadata["questions_and_answers"]
```

### 基础使用

```python
from langchain_community.document_transformers import DoctranQATransformer
from langchain_core.documents import Document

# ===== 1. 准备文档 =====
sample_text = """
向量数据库是专门为存储和检索高维向量数据而设计的数据库系统。
与传统关系型数据库不同，向量数据库使用近似最近邻（ANN）算法
来实现高效的相似度搜索。常见的向量数据库包括 Milvus、Pinecone、
Weaviate 和 ChromaDB。在 RAG 系统中，向量数据库是存储文档
Embedding 的核心组件。
"""

documents = [Document(page_content=sample_text)]

# ===== 2. 创建 QA 转换器 =====
qa_transformer = DoctranQATransformer(
    openai_api_key="your-api-key",
)

# ===== 3. 提取问答对 =====
transformed_docs = qa_transformer.transform_documents(documents)

# ===== 4. 查看结果 =====
for doc in transformed_docs:
    qa_pairs = doc.metadata.get("questions_and_answers", [])
    print(f"提取了 {len(qa_pairs)} 个问答对：")
    for qa in qa_pairs:
        print(f"  Q: {qa['question']}")
        print(f"  A: {qa['answer']}")
        print()

# 预期输出：
# 提取了 3 个问答对：
#   Q: 什么是向量数据库？
#   A: 向量数据库是专门为存储和检索高维向量数据而设计的数据库系统。
#
#   Q: 向量数据库使用什么算法实现相似度搜索？
#   A: 向量数据库使用近似最近邻（ANN）算法来实现高效的相似度搜索。
#
#   Q: 常见的向量数据库有哪些？
#   A: 常见的向量数据库包括 Milvus、Pinecone、Weaviate 和 ChromaDB。
```

### QA 提取在 RAG 中的应用

```python
# 场景：用 QA 对增强检索
# 原始文档可能用陈述句描述知识
# 但用户通常用疑问句提问
# QA 提取可以弥补这个"表述鸿沟"

# 1. 原始文档
doc = Document(page_content="Python 支持列表推导式语法。")
# 向量化后，与 "列表推导式怎么用？" 的相似度可能不够高

# 2. 提取 QA 后
# Q: "Python 支持什么语法特性？" → 与用户问题更匹配
# Q: "什么是列表推导式？" → 直接匹配用户可能的提问方式

# 3. 实际应用：将 QA 对也存入向量库
# 检索时同时搜索原文和 QA 对，提升召回率
```

## 实战：完整的自动标注管道

```python
"""
元数据增强器实战示例
演示：自动标注文档的主题、语言、情感
"""

from langchain_core.documents import Document
from langchain_openai import ChatOpenAI
from langchain_community.document_transformers.openai_functions import (
    create_metadata_tagger,
)
from pydantic import BaseModel, Field
from typing import List, Optional

# ===== 1. 定义元数据 Schema =====
class RAGDocumentMetadata(BaseModel):
    """RAG 文档元数据"""
    topic: str = Field(
        description="文档主题分类",
        json_schema_extra={"enum": ["技术", "金融", "医疗", "教育", "法律", "其他"]}
    )
    language: str = Field(
        description="文档语言",
        json_schema_extra={"enum": ["zh", "en"]}
    )
    sentiment: str = Field(
        description="情感倾向",
        json_schema_extra={"enum": ["positive", "negative", "neutral"]}
    )
    difficulty: str = Field(
        description="内容难度",
        json_schema_extra={"enum": ["beginner", "intermediate", "advanced"]}
    )
    key_topics: List[str] = Field(
        default_factory=list,
        description="关键主题词（最多5个）"
    )

# ===== 2. 创建标注器 =====
llm = ChatOpenAI(model="gpt-4o-mini", temperature=0)
tagger = create_metadata_tagger(
    metadata_schema=RAGDocumentMetadata,
    llm=llm,
)

# ===== 3. 准备测试文档 =====
documents = [
    Document(
        page_content="Python 的 asyncio 模块提供了异步编程的基础设施。"
                     "通过 async/await 语法，开发者可以编写高效的并发代码。",
        metadata={"source": "python_docs.pdf", "page": 42},
    ),
    Document(
        page_content="2024年第四季度，公司营收同比增长25%，"
                     "净利润达到历史新高，市场前景乐观。",
        metadata={"source": "annual_report.pdf", "page": 5},
    ),
    Document(
        page_content="This beginner's guide covers the basics of machine learning, "
                     "including supervised and unsupervised learning algorithms.",
        metadata={"source": "ml_guide.pdf", "page": 1},
    ),
]

# ===== 4. 执行标注 =====
tagged_docs = tagger.transform_documents(documents)

# ===== 5. 查看结果 =====
for i, doc in enumerate(tagged_docs):
    print(f"文档 {i+1}:")
    print(f"  来源: {doc.metadata.get('source', 'unknown')}")
    print(f"  主题: {doc.metadata.get('topic', 'N/A')}")
    print(f"  语言: {doc.metadata.get('language', 'N/A')}")
    print(f"  情感: {doc.metadata.get('sentiment', 'N/A')}")
    print(f"  难度: {doc.metadata.get('difficulty', 'N/A')}")
    print(f"  关键词: {doc.metadata.get('key_topics', [])}")
    print()

# 预期输出：
# 文档 1:
#   来源: python_docs.pdf
#   主题: 技术
#   语言: zh
#   情感: neutral
#   难度: intermediate
#   关键词: ['Python', 'asyncio', '异步编程', 'async/await']
#
# 文档 2:
#   来源: annual_report.pdf
#   主题: 金融
#   语言: zh
#   情感: positive
#   难度: beginner
#   关键词: ['营收', '净利润', '市场前景']
#
# 文档 3:
#   来源: ml_guide.pdf
#   主题: 技术
#   语言: en
#   情感: neutral
#   难度: beginner
#   关键词: ['machine learning', 'supervised learning', 'unsupervised learning']

# ===== 6. 利用元数据进行过滤检索 =====
# 标注完成后，可以在向量库中利用元数据过滤
print("=== 元数据过滤示例 ===")

# 只检索中文技术文档
zh_tech_docs = [
    doc for doc in tagged_docs
    if doc.metadata.get("language") == "zh"
    and doc.metadata.get("topic") == "技术"
]
print(f"中文技术文档: {len(zh_tech_docs)} 篇")

# 只检索初学者内容
beginner_docs = [
    doc for doc in tagged_docs
    if doc.metadata.get("difficulty") == "beginner"
]
print(f"初学者文档: {len(beginner_docs)} 篇")
```

## Schema 设计最佳实践

### 1. 使用枚举值限制选项

```python
# ✅ 好的设计：使用枚举值
"topic": {
    "type": "string",
    "enum": ["technology", "finance", "health"],  # 有限选项
    "description": "The main topic",
}

# ❌ 不好的设计：开放式字符串
"topic": {
    "type": "string",
    "description": "The main topic",  # AI 可能返回任意值
}
```

### 2. 提供清晰的 description

```python
# ✅ 好的描述
"sentiment": {
    "type": "string",
    "enum": ["positive", "negative", "neutral"],
    "description": "The overall emotional tone of the document. "
                   "Use 'positive' for optimistic/encouraging content, "
                   "'negative' for critical/pessimistic content, "
                   "'neutral' for factual/objective content.",
}

# ❌ 模糊的描述
"sentiment": {
    "type": "string",
    "description": "sentiment",  # 太模糊
}
```

### 3. 控制 Schema 复杂度

```python
# ✅ 适度的 Schema（5-8 个字段）
metadata_schema = {
    "properties": {
        "topic": {...},
        "language": {...},
        "sentiment": {...},
        "difficulty": {...},
        "key_entities": {...},
    }
}

# ❌ 过于复杂的 Schema（15+ 个字段）
# 字段越多，AI 提取的准确率越低，成本越高
```

### 4. 区分必需和可选字段

```python
metadata_schema = {
    "properties": {
        "topic": {...},       # 必需：每个文档都应该有主题
        "language": {...},    # 必需：每个文档都有语言
        "sentiment": {...},   # 可选：不是所有文档都需要情感分析
        "key_entities": {...},# 可选：实体提取可能不总是需要
    },
    "required": ["topic", "language"],  # 只标记真正必需的
}
```

## 常见误区

### 误区 1：元数据标注应该在检索时实时进行 ❌

**为什么错？**
- 元数据标注需要调用 LLM API，有延迟和成本
- 应该在文档入库时（离线）完成标注
- 检索时直接使用已标注的元数据过滤

**正确做法：**
```python
# ✅ 正确：入库时标注（离线）
# 步骤 1: 加载文档
# 步骤 2: 自动标注元数据
# 步骤 3: 存入向量库（带元数据）
# 步骤 4: 检索时用元数据过滤

# ❌ 错误：检索时标注（在线）
# 每次检索都调用 LLM 标注 → 慢且贵
```

### 误区 2：元数据越多越好 ❌

**为什么错？**
- 字段越多，AI 提取准确率越低
- 存储和索引成本增加
- 过滤条件过多反而降低召回率

**正确做法：**
```python
# ✅ 只提取对检索有用的元数据
# 问自己：这个字段会用来过滤或路由吗？
# 如果不会，就不要提取

# 推荐的核心字段：
# - topic（主题）→ 用于过滤
# - language（语言）→ 用于过滤
# - source_type（来源类型）→ 用于路由
# - date（日期）→ 用于时间过滤
```

### 误区 3：OpenAIMetadataTagger 只能用 OpenAI ❌

**为什么错？**
- `create_metadata_tagger` 接受任何 `ChatModel`
- 可以使用任何支持 Function Calling 的模型

**正确理解：**
```python
# 使用 OpenAI
from langchain_openai import ChatOpenAI
llm = ChatOpenAI(model="gpt-4o-mini")

# 使用其他支持 Function Calling 的模型也可以
# 关键是模型要支持 tool/function calling 能力
tagger = create_metadata_tagger(metadata_schema=schema, llm=llm)
```

## 双重类比

### 前端类比：自动标签系统

**元数据增强器 = 前端的自动分类标签**

```javascript
// 前端场景：博客文章自动打标签
const article = {
  content: "React 18 introduces concurrent features...",
  // 手动打标签？太累了
};

// 自动标签系统（类似 OpenAIMetadataTagger）
const autoTag = (article) => ({
  ...article,
  tags: {
    category: "frontend",
    framework: "react",
    level: "advanced",
  },
});

// 有了标签，搜索和过滤就方便了
const results = articles.filter(a => a.tags.category === "frontend");
```

### 日常生活类比：图书馆编目

```
场景：图书馆收到一批新书

没有编目（没有元数据）：
- 书随便放在架子上
- 找书只能一本本翻看
- 效率极低

有编目（有元数据）：
- 每本书都有分类号、作者、主题词
- 按分类号找到对应书架
- 按主题词精确定位
- 效率极高

OpenAIMetadataTagger = 自动编目机器人
- 读取书的内容
- 自动生成分类号和主题词
- 贴上标签，放到正确的位置
```

## 总结

元数据增强器是 RAG 系统中提升检索精度的关键工具：

| 转换器 | 用途 | 适用场景 |
|--------|------|----------|
| **OpenAIMetadataTagger** | AI 自动提取结构化元数据 | 生产环境，需要精确标注 |
| **DoctranPropertyExtractor** | 自定义属性提取 | 快速原型，灵活定义 |
| **DoctranQATransformer** | 提取问答对 | 增强检索匹配，弥补表述鸿沟 |

**核心原则：**
1. 在入库时离线标注，不要在检索时实时标注
2. Schema 设计要精简，只提取对检索有用的字段
3. 使用枚举值限制选项，提高提取准确率
4. 注意 API 调用成本，先去重再标注
