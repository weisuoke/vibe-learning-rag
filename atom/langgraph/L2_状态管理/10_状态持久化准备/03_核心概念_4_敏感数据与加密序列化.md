# 核心概念 4：敏感数据与加密序列化

## 概述

LangGraph 的 checkpoint 机制会将完整状态持久化到数据库。这意味着状态中的**所有数据**——包括用户的个人信息（PII）、API 密钥、密码等敏感信息——都会以序列化形式存储。如果不加保护，这些数据就是明文的。本节介绍 LangGraph 的加密序列化机制，以及如何在生产环境中保护敏感数据。

## Pydantic SecretStr：一个常见的误解

### 什么是 SecretStr？

Pydantic 提供了 `SecretStr` 和 `SecretBytes` 类型，用于标记敏感字段。它们的特点是在打印和日志中自动隐藏值：

```python
"""
Pydantic SecretStr 的行为
"""
from pydantic import BaseModel, SecretStr

class Config(BaseModel):
    api_key: SecretStr
    database_url: SecretStr

config = Config(
    api_key="sk-abc123secret",
    database_url="postgresql://user:password@host/db"
)

# ✅ 打印时自动隐藏
print(config.api_key)           # SecretStr('**********')
print(config.database_url)      # SecretStr('**********')
print(repr(config))             # 不会泄露密钥

# 需要实际值时显式提取
actual_key = config.api_key.get_secret_value()
print(actual_key)               # sk-abc123secret
```

### 关键陷阱：SecretStr 不等于加密！

这是一个非常重要的误解。LangGraph 的 `JsonPlusSerializer` 在序列化 Pydantic 对象时，会调用 `get_secret_value()` 提取实际值：

```python
"""
⚠️ 关键陷阱：SecretStr 在序列化时会变成明文
"""

# JsonPlusSerializer 的处理逻辑（简化版）：
# 1. 遇到 Pydantic v2 模型
# 2. 调用 model_dump() 获取字典
# 3. SecretStr 字段通过 get_secret_value() 提取明文
# 4. 明文值被 msgpack 编码存储

# 结果：
# - 日志中看不到密钥 ✅（SecretStr 的设计目的）
# - 数据库中存的是明文 ❌（除非启用加密！）

print("SecretStr 的保护范围：")
print("  ✅ print() 输出隐藏")
print("  ✅ repr() 输出隐藏")
print("  ✅ 日志记录隐藏")
print("  ❌ 序列化后仍是明文")
print("  ❌ checkpoint 数据库中是明文")
print()
print("结论：SecretStr 防的是日志泄露，不防数据库泄露")
print("要保护 checkpoint 中的数据，必须使用 EncryptedSerializer")
```

## EncryptedSerializer：真正的加密保护

### 设计模式：装饰器

`EncryptedSerializer` 采用**装饰器模式**——它不是一个独立的序列化器，而是包装任何现有的 `SerializerProtocol`，在序列化和反序列化的过程中添加加密/解密层。

```
原始流程：
  obj → JsonPlusSerializer.dumps_typed() → (type_tag, bytes) → 存储

加密流程：
  obj → JsonPlusSerializer.dumps_typed() → (type_tag, bytes)
      → CipherProtocol.encrypt(bytes) → (type_tag+cipher_name, ciphertext)
      → 存储
```

[来源: reference/source_状态持久化准备_01.md | serde/encrypted.py: 装饰器模式]

### 类型标签格式

加密后的类型标签使用 `+` 分隔符标记加密方法：

```
未加密：  "msgpack"
加密后：  "msgpack+aes"

未加密：  "json"
加密后：  "json+aes"

未加密：  "bytes"
加密后：  "bytes+aes"
```

这个设计是**向后兼容**的：
- 没有 `+` 的标签 → 视为未加密，直接反序列化
- 有 `+` 的标签 → 先解密，再反序列化

```python
"""
类型标签格式演示
"""

def parse_type_tag(tag: str) -> tuple[str, str | None]:
    """解析类型标签，分离序列化格式和加密方法"""
    if "+" in tag:
        serde_type, cipher_name = tag.split("+", 1)
        return serde_type, cipher_name
    return tag, None

# 测试
examples = ["msgpack", "msgpack+aes", "json", "json+aes", "bytes+aes"]
for tag in examples:
    serde_type, cipher = parse_type_tag(tag)
    status = f"加密方式: {cipher}" if cipher else "未加密"
    print(f"  '{tag}' → 序列化: {serde_type}, {status}")
```

## AES-EAX 认证加密

### 为什么选择 AES-EAX？

LangGraph 使用 **AES-EAX** 模式进行加密。这是一种**认证加密**（Authenticated Encryption）模式，同时提供：

- **机密性（Confidentiality）**：数据被加密，无法被未授权者读取
- **完整性（Integrity）**：数据被篡改后会被检测到

```
普通加密（如 AES-CBC）：
  只保证机密性 → 攻击者可以修改密文而不被发现

认证加密（AES-EAX）：
  机密性 + 完整性 → 修改密文会导致解密失败（tag 验证不通过）
```

### 线格式（Wire Format）

每个加密后的数据块包含三部分：

```
┌──────────┬──────────┬─────────────────┐
│ nonce    │ tag      │ ciphertext      │
│ (16字节) │ (16字节) │ (变长)          │
└──────────┴──────────┴─────────────────┘

总大小 = 16 + 16 + 原始数据大小 = 原始数据 + 32 字节开销
```

- **nonce（16字节）**：随机数，确保相同明文每次加密产生不同密文
- **tag（16字节）**：认证标签，用于验证数据完整性
- **ciphertext（变长）**：加密后的数据

```python
"""
AES-EAX 线格式解析（概念演示）
"""

def explain_wire_format(encrypted_data: bytes):
    """解析加密数据的线格式"""
    if len(encrypted_data) < 32:
        print("数据太短，不是有效的加密数据")
        return

    nonce = encrypted_data[:16]
    tag = encrypted_data[16:32]
    ciphertext = encrypted_data[32:]

    print(f"线格式解析：")
    print(f"  nonce (16B):      {nonce.hex()[:32]}...")
    print(f"  tag (16B):        {tag.hex()[:32]}...")
    print(f"  ciphertext ({len(ciphertext)}B): {ciphertext.hex()[:32]}...")
    print(f"  总大小:           {len(encrypted_data)} bytes")
    print(f"  加密开销:         32 bytes (固定)")

# 模拟一个加密数据块
import os
fake_encrypted = os.urandom(16) + os.urandom(16) + os.urandom(100)
explain_wire_format(fake_encrypted)
```

## CipherProtocol：可插拔加密算法

### 接口定义

LangGraph 定义了 `CipherProtocol`，允许你使用任何加密算法：

```python
"""
CipherProtocol 接口
"""
from typing import Protocol

class CipherProtocol(Protocol):
    """加密协议 - 任何实现了 encrypt/decrypt 的对象都可以使用"""

    def encrypt(self, data: bytes) -> bytes:
        """加密字节数据"""
        ...

    def decrypt(self, data: bytes) -> bytes:
        """解密字节数据"""
        ...
```

### 自定义加密器示例

```python
"""
自定义加密器示例（概念演示）
使用 Python 标准库的简单 XOR 加密（仅用于演示，不要在生产中使用！）
"""

class SimpleXORCipher:
    """
    简单 XOR 加密器（仅用于演示！）

    生产环境请使用 AES-EAX（from_pycryptodome_aes）
    """

    def __init__(self, key: bytes):
        self.key = key

    def encrypt(self, data: bytes) -> bytes:
        """XOR 加密"""
        key_len = len(self.key)
        return bytes(b ^ self.key[i % key_len] for i, b in enumerate(data))

    def decrypt(self, data: bytes) -> bytes:
        """XOR 解密（与加密相同）"""
        return self.encrypt(data)  # XOR 是自反的


# 演示
cipher = SimpleXORCipher(key=b"demo-key-12345!!")
original = b"Hello, this is sensitive data!"
encrypted = cipher.encrypt(original)
decrypted = cipher.decrypt(encrypted)

print(f"原始数据:   {original}")
print(f"加密后:     {encrypted[:30]}...")
print(f"解密后:     {decrypted}")
print(f"验证一致:   {original == decrypted}")
print()
print("⚠️ XOR 加密极不安全，仅用于演示 CipherProtocol 接口")
print("生产环境必须使用 AES-EAX！")
```

### 设置加密持久化

```python
"""
完整示例：设置加密的 checkpoint 持久化
"""
import os
from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END, add_messages
from langgraph.checkpoint.memory import MemorySaver
from langchain_core.messages import HumanMessage, AIMessage

# ===== 步骤 1：定义包含敏感数据的状态 =====

class SecureAgentState(TypedDict):
    messages: Annotated[list, add_messages]
    customer_email: str      # PII
    query_result: str

# ===== 步骤 2：设置加密序列化器 =====

# 方式 A：使用 pycryptodome（推荐）
# 需要安装：pip install pycryptodome
#
# from langgraph.checkpoint.serde.encrypted import EncryptedSerializer
# from langgraph.checkpoint.serde.jsonplus import JsonPlusSerializer
#
# # 设置密钥（生产中用环境变量）
# os.environ["LANGGRAPH_AES_KEY"] = os.urandom(32).hex()
#
# # 创建加密序列化器
# encrypted_serde = EncryptedSerializer.from_pycryptodome_aes()
#
# # 创建带加密的 checkpointer
# checkpointer = MemorySaver(serde=encrypted_serde)

# 方式 B：不加密（仅用于开发/演示）
checkpointer = MemorySaver()

# ===== 步骤 3：构建图 =====

def handle_query(state: SecureAgentState) -> dict:
    """处理用户查询"""
    messages = state["messages"]
    email = state.get("customer_email", "unknown")
    query = messages[-1].content if messages else ""

    print(f"处理来自 {email} 的查询: {query}")

    return {
        "messages": [AIMessage(content=f"已处理您的查询: {query}")],
        "query_result": f"查询结果: 关于 '{query}' 的回答",
    }

graph = StateGraph(SecureAgentState)
graph.add_node("handle", handle_query)
graph.add_edge(START, "handle")
graph.add_edge("handle", END)

# 编译时传入 checkpointer
app = graph.compile(checkpointer=checkpointer)

# ===== 步骤 4：执行 =====

config = {"configurable": {"thread_id": "secure-thread-001"}}

result = app.invoke(
    {
        "messages": [HumanMessage(content="查询我的订单状态")],
        "customer_email": "user@example.com",
        "query_result": "",
    },
    config=config,
)

print(f"\n执行结果:")
print(f"  消息数: {len(result['messages'])}")
print(f"  邮箱: {result['customer_email']}")
print(f"  查询结果: {result['query_result']}")

# 如果使用了 EncryptedSerializer：
# - 数据库中存储的是加密后的字节
# - 类型标签变为 "msgpack+aes"
# - 读取时自动解密
# - 对应用代码完全透明
```

## 与 RAG 系统的关系

在 RAG 系统中，敏感数据保护尤为重要：

1. **用户查询内容**：用户可能在查询中包含敏感信息（如"我的社保号是 XXX，帮我查询..."）
2. **检索结果**：从知识库检索到的文档可能包含机密信息
3. **API 密钥**：调用 LLM 和 Embedding 服务的密钥不应出现在状态中
4. **多租户场景**：不同用户的数据必须隔离，加密是额外的保护层

**最佳实践**：
- API 密钥通过环境变量传递，永远不放入状态
- 用户 PII 在处理完成后从状态中清除
- 生产环境始终启用 `EncryptedSerializer`
- 配合外部存储卸载（核心概念 3），减少 checkpoint 中的敏感数据量
