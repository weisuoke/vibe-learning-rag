# 常用库速查 - 一句话总结

## 核心总结

**serde/anyhow/clap/tracing 是 Rust 生态的"基础设施四件套"：serde 用 derive 宏实现编译时安全的数据格式转换，anyhow 用错误链实现上下文丰富的错误传播，clap 用 derive 宏实现类型安全的 CLI 解析，tracing 用结构化日志实现生产级可观测性。它们共同构成了 ZeroClaw 的 I/O 基础设施层，是阅读和贡献源码的必备知识。**

## 每个库一句话

| 库 | 一句话 | 前端等价 |
|---|---|---|
| serde | 编译时生成的万能数据格式翻译器 | JSON.parse/stringify + zod 合体 |
| anyhow | 自动收集面包屑的万能错误容器 | try/catch + Sentry breadcrumbs |
| clap | 类型安全的命令行菜单系统 | yargs + TypeScript 类型检查 |
| tracing | 可编程的飞机黑匣子 | console.log + winston + OpenTelemetry |

## 记忆口诀

**"序错命日"** → **序**列化(serde) → **错**误处理(anyhow) → **命**令行(clap) → **日**志(tracing)

或者记住前端对照：
**"zod + sentry + yargs + winston"** → 这就是 Rust 的 serde + anyhow + clap + tracing
