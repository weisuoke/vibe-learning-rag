# 所有权与借用 - 概览

> **一句话定位**：所有权是 Rust 的灵魂机制 -- 它用编译期规则替代垃圾回收器，在零运行时开销的前提下杜绝内存安全问题。

---

## 1. 为什么 Rust 需要所有权系统？

### 1.1 内存管理的三条路

| 路线 | 代表语言 | 机制 | 代价 |
|------|----------|------|------|
| 手动管理 | C / C++ | 程序员负责 malloc/free | 悬垂指针、双重释放、内存泄漏 |
| 垃圾回收（GC） | JS / TS / Go / Java | 运行时自动回收 | GC 暂停、内存开销、不可预测延迟 |
| **所有权系统** | **Rust** | **编译期规则自动管理** | **学习曲线陡峭，但零运行时代价** |

Rust 选择了第三条路：**在编译期就把内存问题全部解决**。

### 1.2 它解决了什么问题？

所有权系统在**编译阶段**就杜绝了以下问题（不是运行时检测，是直接不让你编译通过）：

- **悬垂指针**（Dangling Pointer）：引用已释放的内存
- **双重释放**（Double Free）：同一块内存被释放两次
- **数据竞争**（Data Race）：多线程同时读写同一数据
- **Use-After-Free**：释放后继续使用
- **内存泄漏**：大部分情况下自动管理（离开作用域即释放）

> **对 ZeroClaw 的意义**：作为 AI Agent 运行时，ZeroClaw 需要同时管理多个异步任务、共享工具注册表、跨线程传递安全策略。所有权系统保证这些操作在编译期就是安全的。

## 2. TypeScript/JavaScript vs Rust：GC 与所有权的对比

### 2.1 对象传递：复制引用 vs 移动所有权

**TypeScript（GC 管理）：**
```typescript
const config = { host: "localhost", port: 8080 };
const backup = config;      // backup 和 config 指向同一个对象
console.log(config.host);   // 完全没问题，config 仍然有效
```

**Rust（所有权管理）：**
```rust
let config = String::from("localhost:8080");
let backup = config;        // 所有权从 config 移动到 backup
// println!("{}", config);  // 编译错误！config 已经无效了
println!("{}", backup);     // OK，backup 是新的所有者
```

### 2.2 函数传参：传引用 vs 转移所有权

**TypeScript：**
```typescript
function processConfig(cfg: Config) {
    console.log(cfg.host);  // cfg 只是对象的另一个引用
}
const config = { host: "localhost" };
processConfig(config);
console.log(config.host);   // 没问题，config 还在
```

**Rust：**
```rust
fn process_config(cfg: String) {
    println!("{}", cfg);    // cfg 获取了所有权
}
let config = String::from("localhost");
process_config(config);     // config 的所有权移入函数
// println!("{}", config);  // 编译错误！config 已被移走
```

### 2.3 核心思维转变

| 维度 | TypeScript | Rust |
|------|-----------|------|
| 谁管理内存？ | GC（运行时） | 编译器（编译时） |
| 对象赋值 | 复制引用（共享） | 移动所有权（独占） |
| 函数传参 | 传引用副本 | 默认移动所有权 |
| 共享数据 | 天然共享，随便传 | 需显式选择：`&T`、`Clone`、`Arc` |
| 可变性 | 默认可变（`let`） | 默认不可变，需 `let mut` |
| 并发安全 | 靠程序员小心 | 编译器强制检查 |
| 内存释放 | GC 不确定何时回收 | 离开作用域立即释放 |

**核心转变**：从"数据是共享的，我需要时再加锁"转变为"数据是独占的，我需要时再共享"。

## 3. 核心概念列表

本知识点包含 6 个核心概念，覆盖 Rust 所有权系统的完整图景。

### 3.1 所有权三规则与移动语义

> 文件：`03_核心概念_1_所有权三规则与移动语义.md`

**三条铁律**：(1) 每个值有且仅有一个所有者 (2) 同一时刻只能有一个所有者 (3) 所有者离开作用域，值被自动 drop。

**核心内容**：Move 语义、Copy trait（栈上小数据自动复制）、Clone trait（显式深拷贝）、Drop trait（自定义清理）。

**ZeroClaw 体现**：
```rust
// DelegateTool 构造函数获取 HashMap 所有权，包进 Arc 实现共享
pub fn new(
    agents: HashMap<String, DelegateAgentConfig>,  // 获取所有权
    security: Arc<SecurityPolicy>,
) -> Self {
    Self { agents: Arc::new(agents), security }  // 移动进 Arc
}
```
[来源: sourcecode/zeroclaw/src/tools/delegate.rs]

### 3.2 不可变借用 &T

> 文件：`03_核心概念_2_不可变借用.md`

**核心理念**：不转移所有权，只读访问。多个 `&T` 可同时存在，但 `&T` 存在时不能创建 `&mut T`。

**ZeroClaw 体现**：
```rust
pub fn create_memory(
    config: &MemoryConfig,      // 不可变借用：只读取配置
    workspace_dir: &Path,       // 不可变借用：只读取路径
    api_key: Option<&str>,      // 不可变借用：只读取密钥
) -> anyhow::Result<Box<dyn Memory>> { /* ... */ }
```
[来源: sourcecode/zeroclaw/src/memory/mod.rs]

### 3.3 可变借用 &mut T

> 文件：`03_核心概念_3_可变借用.md`

**核心理念**：独占的可写访问权。同一时刻只能有一个 `&mut T`，且不能与 `&T` 共存。这是 Rust 在编译期防止数据竞争的核心机制。

**ZeroClaw 体现**：
```rust
pub fn apply_env_overrides(&mut self) { /* 可变 self 就地修改配置 */ }

pub fn advance_step(&mut self, run_id: &str, result: SopStepResult) -> Result<SopRunAction> {
    // 可变 self 更新步骤进度
}
```
[来源: sourcecode/zeroclaw/src/config/schema.rs, src/sop/engine.rs]

### 3.4 生命周期注解

> 文件：`03_核心概念_4_生命周期注解.md`

**核心理念**：告诉编译器"这个引用至少活多久"，防止悬垂引用。大多数情况编译器自动推导（省略规则），少数情况需要手写 `'a`。

**ZeroClaw 体现**：
```rust
// 返回引用的生命周期 'a 绑定到输入切片，保证不会悬垂
fn find_tool<'a>(tools: &'a [Box<dyn Tool>], name: &str) -> Option<&'a dyn Tool> { /* ... */ }

// Provider 中：转换后的工具规格仍然引用原始数据
fn convert_tools<'a>(tools: Option<&'a [ToolSpec]>) -> Option<Vec<NativeToolSpec<'a>>> { /* ... */ }
```
[来源: sourcecode/zeroclaw/src/agent/loop_.rs, src/providers/anthropic.rs]

### 3.5 智能指针与所有权共享 (Box, Arc, Rc)

> 文件：`03_核心概念_5_智能指针与所有权共享.md`

当"一个所有者"不够用时，用智能指针突破限制：

| 智能指针 | 用途 | 线程安全 | 开销 |
|----------|------|----------|------|
| `Box<T>` | 堆分配 + 单一所有权 | 是 | 最小 |
| `Rc<T>` | 共享所有权（单线程） | 否 | 引用计数 |
| `Arc<T>` | 共享所有权（多线程） | 是 | 原子引用计数 |

**ZeroClaw 体现**：
```rust
// Box<dyn Tool>: 异构工具集合
pub fn default_tools(security: Arc<SecurityPolicy>) -> Vec<Box<dyn Tool>> {
    vec![
        Box::new(ShellTool::new(security.clone(), runtime)),
        Box::new(FileReadTool::new(security.clone())),  // Arc::clone 只增加引用计数
    ]
}

// 嵌套 Arc: 委托工具共享工具列表
pub struct DelegateTool {
    agents: Arc<HashMap<String, DelegateAgentConfig>>,
    parent_tools: Arc<Vec<Arc<dyn Tool>>>,  // Arc 嵌套 Arc
}
```
[来源: sourcecode/zeroclaw/src/tools/mod.rs, src/tools/delegate.rs]

### 3.6 内部可变性 (Mutex, RefCell, Cell)

> 文件：`03_核心概念_6_内部可变性.md`

**核心理念**：通过 `&self`（不可变引用）修改内部数据。绕过编译期借用检查，改为运行时检查。

| 类型 | 用途 | 线程安全 |
|------|------|----------|
| `Cell<T>` | 简单值（Copy 类型） | 否 |
| `RefCell<T>` | 复杂值（运行时 borrow 检查） | 否 |
| `Mutex<T>` | 线程安全（运行时 lock） | 是 |

**ZeroClaw 体现**：
```rust
// Mutex 内部可变性：&self 也能修改
pub struct ActionTracker {
    actions: Mutex<Vec<Instant>>,
}
pub fn record(&self) -> usize {          // 注意：&self 不是 &mut self
    let mut actions = self.actions.lock();
    actions.push(Instant::now());
    actions.len()
}

// 异步 Mutex：Arc<Mutex<T>> 是异步共享可变状态的标准模式
pub struct DriveTool {
    last_command: Arc<Mutex<Option<std::time::Instant>>>,
}
```
[来源: sourcecode/zeroclaw/src/security/policy.rs, crates/robot-kit/src/drive.rs]

## 6. 在 ZeroClaw 中的应用概览

所有权系统贯穿 ZeroClaw 的每个模块：

| 模块 | 所有权模式 | 说明 |
|------|-----------|------|
| **工具注册** | `Vec<Box<dyn Tool>>` | Box 单一所有权，存储异构工具类型 |
| **安全策略共享** | `Arc<SecurityPolicy>` | 所有工具共享同一个策略（引用计数） |
| **工具执行** | `&self.execute()` | 不可变借用，工具执行不修改自身状态 |
| **线程安全约束** | `Tool: Send + Sync` | Trait 约束保证所有工具可跨线程使用 |
| **工具查找** | `find_tool<'a>` | 生命周期保证返回引用不超过工具列表存活时间 |
| **配置修改** | `&mut self` | 可变借用就地修改配置 |
| **策略枚举** | `#[derive(Copy)]` | Copy 类型，赋值时自动复制 |
| **行为追踪** | `Mutex<Vec<Instant>>` | 内部可变性，`&self` 也能修改 |
| **异步状态** | `Arc<Mutex<Option<Instant>>>` | 跨 async 任务共享可变状态 |

[来源: sourcecode/zeroclaw/src/tools/mod.rs, src/security/policy.rs, src/agent/loop_.rs, crates/robot-kit/src/drive.rs, crates/robot-kit/src/traits.rs]

### 所有权决策速查表

```
需要传递数据？
├── 只需要读取 ──────────> &T（不可变借用）
├── 需要修改 ────────────> &mut T（可变借用）
├── 需要拥有数据 ────────> T（移动所有权）
│
需要存储异构类型？ ──────> Box<dyn Trait>
│
需要多处共享？
├── 单线程 ──────────────> Rc<T>
├── 多线程 ──────────────> Arc<T>
│
需要通过 &self 修改？
├── 单线程 ──────────────> RefCell<T>
├── 多线程（同步）───────> std::sync::Mutex<T>
├── 多线程（异步）───────> tokio::sync::Mutex<T>
│
需要多处共享 + 修改？ ──> Arc<Mutex<T>>（ZeroClaw 最常用模式）
```
