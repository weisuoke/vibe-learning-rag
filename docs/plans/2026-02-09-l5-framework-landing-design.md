## 设计目标

为 RAG 开发学习项目的最后一层（L5）设计完整的知识点体系，覆盖从框架使用到生产落地的全流程。

## 核心决策

### 1. 知识点范围扩展

**原始规划（CLAUDE.md）：** 3 个知识点
- LangChain核心概念
- LlamaIndex核心概念
- 端到端项目实战

**最终设计：** 6 个知识点（新增 3 个生产化相关）
- 01_LangChain核心概念
- 02_LlamaIndex核心概念
- 03_端到端项目实战
- 04_RAG API服务化 ⭐ 新增
- 05_部署与监控 ⭐ 新增
- 06_生产环境优化 ⭐ 新增

**扩展理由：**
- L5 作为"框架与落地"层，仅覆盖框架使用不足以支撑"落地"目标
- 实际 RAG 项目需要 API 服务化、部署运维、性能优化等生产化能力
- 用户明确需求：增加部署与运维相关知识点

### 2. 学习顺序设计

```
框架基础（1-2）→ 全流程实战（3）→ 服务化（4）→ 部署运维（5-6）
```

**设计原则：**
1. **框架先行**：先掌握工具（LangChain/LlamaIndex），再用工具实战
2. **实战串联**：用端到端项目串联 L1-L4 所有知识点
3. **渐进生产化**：API 服务化 → 部署监控 → 性能优化，逐步迈向生产环境

**为什么不是"框架 → API化 → 实战 → 生产化"？**
- 端到端实战需要先理解完整流程，再考虑服务化
- API 服务化是对已有系统的封装，需要先有可运行的原型

### 3. 知识点详细设计

#### 01_LangChain核心概念
- **定位**：通用 LLM 应用框架
- **核心内容**：Chain/Agent/Memory 架构，LCEL 表达式语言
- **RAG 应用**：可组合的 RAG Pipeline

#### 02_LlamaIndex核心概念
- **定位**：RAG 专用框架
- **核心内容**：数据连接器、索引类型、查询引擎
- **RAG 应用**：高效的数据索引与检索系统

#### 03_端到端项目实战
- **定位**：串联所有知识点的综合实战
- **核心内容**：需求分析 → 架构设计 → 实现 → 测试 → 优化
- **产出**：可运行的完整 RAG 应用原型

#### 04_RAG API服务化
- **定位**：将 RAG 系统封装为可调用的服务
- **核心内容**：FastAPI 封装、流式响应、异步处理、接口设计
- **产出**：可对外提供服务的 REST API

#### 05_部署与监控
- **定位**：将服务部署到生产环境
- **核心内容**：Docker 容器化、云部署方案、日志管理、监控告警
- **产出**：稳定运行的线上服务

#### 06_生产环境优化
- **定位**：提升生产系统的性能与成本效益
- **核心内容**：向量库缓存、并发处理、延迟优化、成本控制
- **产出**：高性能低成本的生产系统

## 与其他层级的关系

### 前置知识（L1-L4）
- **L1_NLP基础**：BPE分词、Embedding、语义相似度
- **L2_LLM核心**：Transformer、API调用、Token/Context、Prompt Engineering
- **L3_RAG核心流程**：RAG架构、文档加载、分块、向量存储、检索、生成
- **L4_RAG进阶优化**：混合检索、ReRank、Query改写、幻觉检测、评估调优

### 后续学习方向
- 实际项目中持续迭代与优化
- 探索更多框架（Haystack、Semantic Kernel 等）
- 多模态 RAG、Graph RAG 等前沿方向
