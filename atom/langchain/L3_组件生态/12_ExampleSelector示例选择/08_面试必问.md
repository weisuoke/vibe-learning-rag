## 问题 1：什么是 ExampleSelector？为什么需要它？（初级）

### 问题

"请解释一下 LangChain 中的 ExampleSelector 是什么？为什么我们需要它？"

### 普通回答（❌ 不出彩）

"ExampleSelector 是 LangChain 中用来选择示例的工具，可以帮助我们动态选择 Few-shot 示例。"

### 出彩回答（✅ 推荐）

> **ExampleSelector 是 LangChain 中用于动态选择 Few-shot 示例的抽象接口，它解决了三个核心问题：**
>
> 1. **示例相关性问题**：当我们有大量示例时（比如 100 个问答对），不可能把所有示例都放入 Prompt。ExampleSelector 可以根据用户输入动态选择最相关的 3-5 个示例，提高 Few-shot learning 的效果。
>
> 2. **Token 限制问题**：LLM 都有 context window 限制（如 GPT-4 的 8K/32K tokens）。ExampleSelector 可以根据长度限制自动调整示例数量，避免超过 token 限制。
>
> 3. **示例多样性问题**：有时我们需要平衡相关性和多样性，避免选择过于相似的示例。ExampleSelector 提供了 MMR（Max Marginal Relevance）等策略来解决这个问题。
>
> **与静态示例的区别**：
> - 静态示例：`FewShotPromptTemplate(examples=[ex1, ex2, ex3])`，所有查询使用相同示例
> - 动态示例：`FewShotPromptTemplate(example_selector=selector)`，根据输入动态选择
>
> **在 RAG 系统中的应用**：在 RAG 系统中，我们可以使用 ExampleSelector 根据用户查询动态选择最相关的问答示例，优化 Prompt 质量，提高回答准确性。

## 问题 2：SemanticSimilarityExampleSelector 的工作原理是什么？（初级）

### 问题

"请详细解释 SemanticSimilarityExampleSelector 的工作原理，它是如何选择示例的？"

### 普通回答（❌ 不出彩）

"SemanticSimilarityExampleSelector 使用语义相似度来选择示例，它会计算输入和示例之间的相似度，然后返回最相似的几个。"

### 出彩回答（✅ 推荐）

> **SemanticSimilarityExampleSelector 基于语义相似度选择示例，工作流程分为两个阶段：**
>
> **阶段 1：初始化阶段（Indexing）**
> 1. 将所有示例转换为文本（通过 `_example_to_text` 方法）
> 2. 使用 Embeddings 模型（如 OpenAI text-embedding-3-small）生成向量
> 3. 将向量存储到 VectorStore（如 Chroma、FAISS）中
> 4. 示例的元数据（如 input、output）也一起存储
>
> **阶段 2：查询阶段（Retrieval）**
> 1. 将用户输入转换为文本
> 2. 生成输入的 embedding 向量
> 3. 在 VectorStore 中进行相似度搜索（通常使用余弦相似度）
> 4. 返回 top-k 最相似的示例
>
> **核心代码逻辑**：
> ```python
> def select_examples(self, input_variables: dict[str, str]) -> list[dict]:
>     # 1. 将输入转换为文本
>     query_text = self._example_to_text(input_variables, self.input_keys)
>     
>     # 2. 使用向量存储进行相似度搜索
>     example_docs = self.vectorstore.similarity_search(
>         query_text,
>         k=self.k,  # 返回 top-k 个示例
>         **self.vectorstore_kwargs
>     )
>     
>     # 3. 从文档中提取示例
>     return self._documents_to_examples(example_docs)
> ```
>
> **关键参数**：
> - `k=4`：返回的示例数量（默认 4 个）
> - `input_keys`：只基于部分输入字段进行检索
> - `example_keys`：只返回示例的部分字段
>
> **在 RAG 系统中的应用**：在 RAG 问答系统中，我们可以根据用户问题的语义相似度选择最相关的历史问答对，帮助 LLM 更好地理解问题类型和回答风格。

## 问题 3：静态示例和动态示例的区别是什么？各自适用于什么场景？（初级）

### 问题

"在 LangChain 的 FewShotPromptTemplate 中，静态示例和动态示例有什么区别？各自适用于什么场景？"

### 普通回答（❌ 不出彩）

"静态示例是固定的，动态示例是根据输入选择的。静态示例简单，动态示例更灵活。"

### 出彩回答（✅ 推荐）

> **静态示例和动态示例的核心区别在于示例选择的时机和策略：**
>
> **静态示例（Fixed Examples）**：
> ```python
> # 所有查询使用相同的示例
> prompt = FewShotPromptTemplate(
>     examples=[
>         {"input": "2+2", "output": "4"},
>         {"input": "2+3", "output": "5"},
>     ],
>     example_prompt=example_prompt,
>     ...
> )
> ```
>
> **动态示例（Dynamic Examples）**：
> ```python
> # 根据输入动态选择示例
> selector = SemanticSimilarityExampleSelector.from_examples(
>     examples=[...],  # 示例池
>     embeddings=OpenAIEmbeddings(),
>     vectorstore_cls=Chroma,
>     k=3
> )
> prompt = FewShotPromptTemplate(
>     example_selector=selector,
>     example_prompt=example_prompt,
>     ...
> )
> ```
>
> **对比表格**：
>
> | 维度 | 静态示例 | 动态示例 |
> |------|----------|----------|
> | **选择时机** | 定义时固定 | 运行时动态选择 |
> | **示例数量** | 固定 | 可变（受 k 值或长度限制） |
> | **相关性** | 可能不相关 | 高度相关 |
> | **性能开销** | 无额外开销 | 需要 embedding + 检索 |
> | **适用场景** | 示例少、通用性强 | 示例多、需要个性化 |
> | **Token 使用** | 固定 | 可优化 |
>
> **适用场景**：
>
> **静态示例适用于**：
> 1. 示例数量很少（< 5 个）
> 2. 所有示例都高度相关且通用
> 3. 任务简单，不需要个性化
> 4. 性能要求高，不能有额外开销
>
> **动态示例适用于**：
> 1. 示例数量较多（> 10 个）
> 2. 需要根据输入选择最相关的示例
> 3. 需要控制 Token 数量
> 4. RAG 系统中的 Few-shot 优化
>
> **在 RAG 系统中的应用**：在 RAG 问答系统中，我们通常使用动态示例选择。例如，当用户问"如何优化向量检索性能？"时，系统会从示例池中选择与性能优化相关的问答对，而不是随机选择。

## 问题 4：LengthBasedExampleSelector 如何控制 Token 数量？（中级）

### 问题

"请解释 LengthBasedExampleSelector 是如何控制 Token 数量的？如何自定义长度计算函数？"

### 普通回答（❌ 不出彩）

"LengthBasedExampleSelector 通过设置 max_length 参数来限制示例的总长度，避免超过 token 限制。"

### 出彩回答（✅ 推荐）

> **LengthBasedExampleSelector 通过动态调整示例数量来控制 Token 数量，核心机制如下：**
>
> **工作原理**：
> 1. 预计算每个示例的长度（在初始化时）
> 2. 计算输入的长度
> 3. 从剩余长度中按顺序选择示例，直到达到 max_length 限制
>
> **核心代码逻辑**：
> ```python
> def select_examples(self, input_variables: dict[str, str]) -> list[dict]:
>     # 1. 计算输入的长度
>     inputs = " ".join(input_variables.values())
>     remaining_length = self.max_length - self.get_text_length(inputs)
>     
>     # 2. 按顺序选择示例，直到达到长度限制
>     i = 0
>     examples = []
>     while remaining_length > 0 and i < len(self.examples):
>         new_length = remaining_length - self.example_text_lengths[i]
>         if new_length < 0:
>             break  # 超过限制，停止选择
>         examples.append(self.examples[i])
>         remaining_length = new_length
>         i += 1
>     
>     return examples
> ```
>
> **默认长度计算函数**：
> ```python
> def _get_length_based(text: str) -> int:
>     return len(re.split(r"\n| ", text))  # 按空格和换行符分割
> ```
>
> **自定义长度计算函数**：
> ```python
> import tiktoken
>
> # 使用 tiktoken 精确计算 token 数量
> def get_token_length(text: str) -> int:
>     encoding = tiktoken.encoding_for_model("gpt-4")
>     return len(encoding.encode(text))
>
> # 创建选择器
> selector = LengthBasedExampleSelector(
>     examples=examples,
>     example_prompt=example_prompt,
>     max_length=2048,  # 最大 token 数
>     get_text_length=get_token_length  # 自定义长度计算
> )
> ```
>
> **关键参数**：
> - `max_length=2048`：最大长度限制（默认按单词数，可自定义为 token 数）
> - `get_text_length`：长度计算函数（可自定义）
> - `example_prompt`：示例格式化模板（用于计算示例长度）
>
> **在 RAG 系统中的应用**：在 RAG 系统中，我们可以根据检索到的文档长度动态调整示例数量。例如，如果检索到的文档很长，就减少示例数量；如果文档较短，就增加示例数量，确保不超过模型的 context window。

## 问题 5：MaxMarginalRelevanceExampleSelector 如何平衡相关性和多样性？（中级）

### 问题

"请解释 MaxMarginalRelevanceExampleSelector 是如何平衡相关性和多样性的？fetch_k 参数的作用是什么？"

### 普通回答（❌ 不出彩）

"MaxMarginalRelevanceExampleSelector 使用 MMR 算法来选择示例，可以避免选择过于相似的示例。"

### 出彩回答（✅ 推荐）

> **MaxMarginalRelevanceExampleSelector 使用 MMR（Max Marginal Relevance）算法平衡相关性和多样性，核心思想是：**
>
> **MMR 算法原理**：
> 1. 先选择与查询最相关的示例
> 2. 后续选择时，既要考虑与查询的相关性，又要考虑与已选示例的差异性
> 3. 通过权衡这两个因素，避免选择过于相似的示例
>
> **工作流程**：
> ```python
> def select_examples(self, input_variables: dict[str, str]) -> list[dict]:
>     # 使用 MMR 算法选择示例
>     example_docs = self.vectorstore.max_marginal_relevance_search(
>         self._example_to_text(input_variables, self.input_keys),
>         k=self.k,        # 最终返回 4 个示例
>         fetch_k=self.fetch_k,  # 先获取 20 个候选
>     )
>     return self._documents_to_examples(example_docs)
> ```
>
> **fetch_k 参数的作用**：
> - `fetch_k=20`：先从向量存储中获取 20 个最相关的候选示例
> - `k=4`：从这 20 个候选中使用 MMR 算法选择 4 个最终示例
> - **为什么需要 fetch_k？** 因为 MMR 算法需要在一个候选集中进行选择，fetch_k 越大，多样性越好，但性能开销也越大
>
> **MMR 算法公式**：
> ```
> MMR = argmax[λ * Sim(Di, Q) - (1-λ) * max Sim(Di, Dj)]
>                                      j∈S
> 
> 其中：
> - Di: 候选示例
> - Q: 查询
> - S: 已选示例集合
> - λ: 权衡参数（通常为 0.5）
> ```
>
> **对比 SemanticSimilarityExampleSelector**：
>
> | 维度 | SemanticSimilarityExampleSelector | MaxMarginalRelevanceExampleSelector |
> |------|-----------------------------------|-------------------------------------|
> | **选择策略** | 只考虑相关性 | 平衡相关性和多样性 |
> | **示例多样性** | 可能选择相似示例 | 保证示例多样性 |
> | **性能开销** | 较低 | 较高（需要 fetch_k 个候选） |
> | **适用场景** | 一般场景 | 需要多样性的场景 |
>
> **在 RAG 系统中的应用**：在 RAG 问答系统中，如果用户问题比较宽泛（如"如何优化 RAG 系统？"），使用 MMR 选择器可以选择涵盖不同优化方向的示例（如检索优化、生成优化、评估优化），而不是只选择检索优化相关的示例。

### 为什么这个回答出彩？

1. ✅ 详细说明了 MMR 算法原理
2. ✅ 解释了 fetch_k 参数的作用
3. ✅ 提供了对比表格，展示了差异
4. ✅ 联系了 RAG 开发实际应用

## 问题 6：NGramOverlapExampleSelector 的工作原理是什么？（中级）

### 问题

"请解释 NGramOverlapExampleSelector 是如何工作的？它与 SemanticSimilarityExampleSelector 有什么区别？"

### 普通回答（❌ 不出彩）

"NGramOverlapExampleSelector 使用 N-gram 重叠来选择示例，它比语义相似度选择器更轻量。"

### 出彩回答（✅ 推荐）

> **NGramOverlapExampleSelector 基于 N-gram 重叠度选择示例，是一种轻量级的文本相似度计算方法，不依赖向量化：**
>
> **工作原理**：
> 1. 将输入和示例都分解为 N-gram（连续的 n 个词）
> 2. 计算输入和每个示例之间的 N-gram 重叠数量
> 3. 按重叠度排序，返回 top-k 个示例
>
> **核心算法**：
> ```python
> def ngram_overlap_score(input_text: str, example_text: str, n: int = 2) -> float:
>     """计算 N-gram 重叠度"""
>     # 1. 生成 N-grams
>     input_ngrams = set(ngrams(input_text.split(), n))
>     example_ngrams = set(ngrams(example_text.split(), n))
>
>     # 2. 计算重叠
>     overlap = len(input_ngrams & example_ngrams)
>
>     # 3. 归一化
>     if len(input_ngrams) == 0:
>         return 0.0
>     return overlap / len(input_ngrams)
> ```
>
> **与 SemanticSimilarityExampleSelector 的对比**：
>
> | 维度 | NGramOverlapExampleSelector | SemanticSimilarityExampleSelector |
> |------|----------------------------|-----------------------------------|
> | **相似度计算** | N-gram 重叠（词级别） | 语义相似度（向量级别） |
> | **依赖** | 无需 Embeddings | 需要 Embeddings 模型 |
> | **性能** | 快速（无需向量计算） | 较慢（需要向量计算） |
> | **准确性** | 适合字面相似 | 适合语义相似 |
> | **成本** | 无额外成本 | 需要 Embedding API 调用 |
> | **适用场景** | 关键词匹配、代码示例 | 自然语言理解 |
>
> **适用场景**：
> - 关键词匹配场景（如代码搜索）
> - 不需要深度语义理解
> - 对成本敏感（无需 Embedding API）
> - 对延迟敏感（无需向量计算）
>
> **在 AI Agent 开发中的应用**：在代码生成 Agent 中，NGramOverlapExampleSelector 可以快速匹配相似的代码片段，无需调用 Embedding API，降低成本和延迟。

2. ✅ 提供了核心算法示例

## 问题 7：如何自定义 ExampleSelector？（中级-高级）

"如果现有的 ExampleSelector 都不满足需求，如何自定义一个 ExampleSelector？请提供完整的实现示例。"

### 出彩回答（✅ 推荐）

>
> **步骤 1：理解 BaseExampleSelector 接口**
> ```python
> from langchain_core.example_selectors import BaseExampleSelector
> from typing import Dict, List
>
> class BaseExampleSelector(ABC):
>     @abstractmethod
>     def add_example(self, example: Dict[str, str]) -> None:
>         """添加示例到存储"""
>         pass
>
>     @abstractmethod
>     def select_examples(self, input_variables: Dict[str, str]) -> List[Dict]:
>         """根据输入选择示例"""
>         pass
> ```
>
> **步骤 2：实现自定义选择器（示例：基于关键词匹配）**
> ```python
> from langchain_core.example_selectors import BaseExampleSelector
> from typing import Dict, List
>
> class KeywordExampleSelector(BaseExampleSelector):
>     """基于关键词匹配的示例选择器
>
>     适用场景：
>     - 需要精确关键词匹配
>     - 不需要语义理解
