# ZeroClaw 安装与环境配置 - 概览

> **一句话定位**：安装与环境配置是使用 ZeroClaw 的第一步 -- 它提供 5 种安装方式、完整的环境变量体系和 TOML 配置系统，让你在 5 分钟内从零到一个可运行的 AI Agent 运行时。

---

## 1. 知识点简介

### 1.1 这是什么？

ZeroClaw 是一个 **100% Rust 编写的轻量级 AI Agent 运行时**。它的安装与环境配置涵盖三个层面：

- **获取二进制**：如何把 ZeroClaw 装到你的机器上（5 种方式，覆盖 6 大平台）
- **环境变量配置**：通过 `ZEROCLAW_*` 前缀的环境变量控制运行行为
- **配置文件管理**：通过 `~/.zeroclaw/config.toml` 持久化配置

与传统的"下载安装包 → 下一步 → 完成"不同，ZeroClaw 的安装配置体系更像一个**分层架构**：不同安装方式适配不同场景，环境变量和配置文件有明确的优先级关系，Feature Flags 让你按需裁剪功能。

### 1.2 核心数据一览

| 指标 | 数值 | 类比 |
|------|------|------|
| 二进制大小 | ~8.8 MB | 约等于 3 张高清照片 |
| 运行内存 | <5 MB | 比一个空白浏览器标签页还小 |
| 启动时间 | <10 ms | 人类眨眼 300ms，它启动 30 次 |
| 最新稳定版 | v0.1.7 | 2026 年 2 月发布 |
| 支持平台 | 6 个 | Linux/macOS/Windows，x86_64/ARM64/ARMv7 |
| Feature Flags | 12 个 | 按需开启硬件、通道、存储等模块 |

### 1.3 为什么它这么小？

ZeroClaw 的 Release 构建使用了极致优化策略：

```toml
# Cargo.toml [profile.release]
opt-level = "z"        # 体积优化（而非速度优化）
lto = "fat"            # 全量链接时优化
codegen-units = 1      # 单编译单元（更小但编译更慢）
strip = true           # 去除调试符号
panic = "abort"        # 不展开调用栈，直接中止
```

这就像前端的 `webpack production` 模式开启了 tree-shaking + minify + gzip 全家桶。

## 2. 为什么需要学习这个知识点？

### 2.1 它是一切的起点

```
安装配置 ──→ 首次对话 ──→ CLI 使用 ──→ Provider 配置 ──→ Channel 配对 ──→ ...
   ↑
  你在这里
```

没有正确的安装和配置，后续所有知识点都无法实践。

### 2.2 ZeroClaw 的配置体系不是"装好就完了"

与简单的 `npm install -g some-cli` 不同，ZeroClaw 的配置涉及：

- **多层 Provider 配置**：你需要决定用哪个 AI 模型后端（OpenRouter / Anthropic / Ollama 本地）
- **安全策略**：默认限制在工作区目录内操作
- **Feature 裁剪**：在嵌入式设备上可能只需要核心功能
- **多环境管理**：开发/测试/生产用不同配置

### 2.3 理解配置优先级避免"玄学问题"

ZeroClaw 的配置解析有明确的优先级链：

```
环境变量 > 命令行参数 > config.toml > 默认值
  (最高)                                (最低)
```

**前端类比**：这就像 CSS 的优先级 `!important > inline style > ID > class > 标签`。不理解优先级，你会遇到"明明改了配置却不生效"的问题。

**日常生活类比**：类似于搬进新家的优先级 -- 你自己带来的家具 > 房东提供的家具 > 精装房自带的设施 > 毛坯房的基础设施。

### 概念 1：安装方式全景（5 种）

> 文件：`03_核心概念_1_安装方式全景.md`

**5 种安装方式**，从简到繁，覆盖所有场景：

| 方式 | 命令 | 适用场景 | 前端类比 |
|------|------|----------|----------|
| 一键脚本 | `curl ... \| bash` | 快速体验 | `npx create-react-app` |
| Homebrew | `brew install zeroclaw` | macOS/Linux 日常 | `brew install node` |

**ZeroClaw 体现**：
```bash
# 最推荐的方式：一键安装（自动检测平台、优先预编译）
curl -fsSL https://zeroclawlabs.ai/install.sh | bash
```

### 概念 2：系统要求与平台依赖

> 文件：`03_核心概念_2_系统要求与平台依赖.md`

**核心要点**：Rust MSRV（最低支持版本）1.87、编译需 2GB+ RAM、各平台需安装不同的构建依赖。

**前端类比**：类似于 `node >= 18` 的引擎要求 + `npm install` 前需要安装 Python/C++ 编译工具（node-gyp）。

### 概念 3：环境变量体系

> 文件：`03_核心概念_3_环境变量体系.md`

**核心要点**：统一的 `ZEROCLAW_*` 前缀、分为核心运行时 / Gateway / 存储 / 代理 / Provider API Keys 五大类。

```bash
# 核心环境变量示例
export ZEROCLAW_API_KEY="sk-or-v1-..."     # API 密钥
export ZEROCLAW_PROVIDER="openrouter"       # 默认 Provider
export ZEROCLAW_MODEL="anthropic/claude-sonnet-4-6"  # 默认模型
export ZEROCLAW_WORKSPACE="~/.zeroclaw"     # 工作区目录
```

**前端类比**：类似于 `.env` 文件 + `process.env.NEXT_PUBLIC_*` 的环境变量命名规范。

### 概念 4：配置文件系统

> 文件：`03_核心概念_4_配置文件系统.md`

**核心要点**：TOML 格式的 `~/.zeroclaw/config.toml`、分区配置（gateway / security / agent / memory 等）、解析优先级链。

```toml
# ~/.zeroclaw/config.toml 核心结构
default_provider = "openrouter"
default_model = "anthropic/claude-sonnet-4-6"
default_temperature = 0.7

[gateway]
port = 42617
host = "[::]"

[security]
# ...

[agent]
# ...
```

**前端类比**：类似于 `vite.config.ts` / `next.config.js`，集中管理项目配置。

### 概念 5：Feature Flags 与构建优化

> 文件：`03_核心概念_5_Feature_Flags与构建优化.md`

**核心要点**：12 个可选 Feature（hardware、channel-matrix、memory-postgres 等），按需编译、减小体积。

```bash
# 只编译需要的 Feature
cargo install zeroclaw --features "channel-matrix,memory-postgres"

# 默认构建（无额外 Feature）= 最小体积
cargo install zeroclaw
```

**前端类比**：类似于 webpack 的 `externals` + `splitChunks`，或 Next.js 的选择性 import。

### 概念 6：版本管理与多平台构建

> 文件：`03_核心概念_6_版本管理与多平台构建.md`

**核心要点**：rustup 工具链管理、MSRV 约束、CI/CD 自动化多目标交叉编译。

**前端类比**：类似于 `nvm` 管理 Node.js 版本 + GitHub Actions 自动构建多平台产物。

## 5. TypeScript/Node.js vs ZeroClaw：安装配置对照

作为有前端背景的开发者，你可以通过以下对照快速建立直觉：

| 维度 | TypeScript/Node.js 世界 | ZeroClaw/Rust 世界 |
|------|------------------------|-------------------|
| 安装运行时 | `nvm install 20` | `rustup install stable` |
| 安装全局工具 | `npm install -g typescript` | `cargo install zeroclaw` |
| 一键脚手架 | `npx create-next-app` | `curl ... \| bash` + `zeroclaw onboard` |
| 包管理器 | npm / yarn / pnpm | Cargo |
| 配置文件 | `package.json` + `.env` | `config.toml` + `ZEROCLAW_*` 环境变量 |
| 构建优化 | webpack production mode | `[profile.release]` opt-level/lto/strip |
| 可选功能 | `npm install --save-optional` | `cargo install --features "..."` |
| 容器化 | `Dockerfile` + `docker-compose.yml` | 同样，`Dockerfile` + `docker compose` |
| 版本锁定 | `package-lock.json` | `Cargo.lock` + `--locked` |
| 版本管理 | `.nvmrc` | `rust-toolchain.toml` |

### Rust 安装核心概念（快速入门）

如果你还没有安装 Rust，以下是你需要知道的：

```bash
# 安装 Rust（类似于安装 nvm + node）
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 验证安装
rustc --version    # 类似于 node --version
cargo --version    # 类似于 npm --version
```

- **rustup** = nvm（管理 Rust 版本）
- **rustc** = node（编译器/运行时）
- **cargo** = npm（包管理 + 构建 + 运行）

## 10. 在 ZeroClaw 开发中的应用概览

安装与配置不是一次性的事情，它贯穿整个 ZeroClaw 使用和开发周期：

| 应用场景 | 涉及的配置 | 说明 |
|---------|-----------|------|
| **本地开发** | 一键安装 + config.toml | 快速搭建开发环境 |
| **Provider 切换** | 环境变量 + config.toml | 从 OpenRouter 切到 Ollama 本地 |
| **服务器部署** | Docker + 环境变量 | 24/7 运行的生产环境 |
| **嵌入式设备** | Feature Flags + 交叉编译 | 树莓派等低资源设备 |
| **团队协作** | config.toml + .env | 共享配置模板，隐藏 API Keys |
| **CI/CD 自动化** | 环境变量 + 多目标构建 | GitHub Actions 自动构建 |
| **故障排查** | 配置优先级 + 日志 | 定位"配置不生效"问题 |

### 安装方式决策速查

```
你要做什么？
├── 快速体验 ──────────────→ 一键脚本 (curl | bash)
├── macOS/Linux 日常使用 ──→ Homebrew (brew install)
├── 已有 Rust 环境 ────────→ Cargo Install
├── 服务器 24/7 部署 ──────→ Docker
├── 无网络环境 / 离线 ─────→ 预编译二进制下载
└── 需要自定义 Feature ────→ 源码编译 (git clone + cargo build)
```
