# 核心概念 4：执行监控与自我反思

> LangChain Agent 系统 · 第 8 个知识点 · 核心概念 4/5

---

## 一句话定义

**执行监控是跟踪每个子任务执行状态的机制，自我反思（Reflection/Reflexion）是 Agent 对自身输出进行评估和改进的能力，两者结合让 Agent 能够自我纠错。**

---

## 为什么需要自我反思？

```
没有反思：
  Plan → Execute → 输出结果（可能有错误但不知道）
  用户: "你的分析遗漏了成本对比" ← 事后才发现

有反思：
  Plan → Execute → Reflect（自我评估）→ 发现遗漏 → 补充执行 → 输出
  Agent 自己发现了遗漏并修正 ← 自我纠错
```

**人类也一样**：写完文章后自己检查一遍（Reflection），请别人审稿（LLM-as-Judge），把错误记下来下次避免（Reflexion）。

---

## 三种反思机制

### 机制1：Reflection（基础反思）

**原理**：Agent 对自己的输出进行自我评估，发现问题后改进。

```python
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field
from typing import List

class ReflectionResult(BaseModel):
    """反思结果"""
    is_satisfactory: bool = Field(description="输出是否满意")
    issues: List[str] = Field(
        default_factory=list,
        description="发现的问题列表"
    )
    suggestions: List[str] = Field(
        default_factory=list,
        description="改进建议"
    )
    score: int = Field(description="质量评分 1-10")

llm = ChatOpenAI(model="gpt-4o", temperature=0)
reflector = llm.with_structured_output(ReflectionResult)

def reflect(task: str, output: str) -> ReflectionResult:
    """对 Agent 的输出进行反思"""
    return reflector.invoke(f"""
    你是一个严格的质量审查员。请评估以下输出的质量。

    原始任务：{task}
    Agent 输出：{output}

    评估维度：
    1. 完整性：是否回答了所有方面？
    2. 准确性：信息是否正确？
    3. 相关性：是否紧扣任务？
    4. 可操作性：建议是否具体可执行？

    如果评分低于 7 分，请列出问题和改进建议。
    """)
```

### 带反思的执行循环

```python
def execute_with_reflection(task: str, max_iterations: int = 3):
    """带反思的执行循环"""
    executor = ChatOpenAI(model="gpt-4o-mini")

    output = executor.invoke(f"请完成任务：{task}").content

    for i in range(max_iterations):
        # 反思
        reflection = reflect(task, output)
        print(f"迭代 {i+1}: 评分 {reflection.score}/10")

        if reflection.is_satisfactory:
            print(f"✅ 质量达标，结束反思")
            return output

        # 根据反思结果改进
        print(f"  问题: {reflection.issues}")
        output = executor.invoke(f"""
        请改进以下输出。

        原始任务：{task}
        当前输出：{output}
        发现问题：{reflection.issues}
        改进建议：{reflection.suggestions}

        请生成改进后的输出。
        """).content

    return output
```

### 机制2：Reflexion（增强反思）

**原理**：在基础反思之上，增加外部验证和经验记忆。

```
Reflection:  生成 → 自我评估 → 改进 → 再评估
                     （内部视角）

Reflexion:   生成 → 自我评估 → 外部验证 → 记忆经验 → 改进
                     （内部+外部视角+长期学习）
```

```python
class ReflexionMemory(BaseModel):
    """Reflexion 经验记忆"""
    task_type: str = Field(description="任务类型")
    mistake: str = Field(description="犯过的错误")
    lesson: str = Field(description="学到的经验")
    correct_approach: str = Field(description="正确的做法")

# 经验记忆库
experience_memory: List[ReflexionMemory] = []

def reflexion_loop(task: str, max_trials: int = 3):
    """Reflexion 循环：执行 → 评估 → 记忆 → 改进"""
    executor = ChatOpenAI(model="gpt-4o-mini")

    for trial in range(max_trials):
        # 1. 执行（带入历史经验）
        relevant_lessons = [
            m.lesson for m in experience_memory
            if m.task_type in task
        ]
        prompt = f"任务：{task}"
        if relevant_lessons:
            prompt += f"\n历史经验教训：{relevant_lessons}"

        output = executor.invoke(prompt).content

        # 2. 自我评估
        reflection = reflect(task, output)

        # 3. 外部验证（如果有）
        external_check = external_validate(task, output)

        # 4. 如果通过，返回结果
        if reflection.is_satisfactory and external_check.passed:
            return output

        # 5. 记忆经验
        lesson = ReflexionMemory(
            task_type=extract_task_type(task),
            mistake=str(reflection.issues),
            lesson=f"避免：{reflection.issues}。应该：{reflection.suggestions}",
            correct_approach=str(reflection.suggestions),
        )
        experience_memory.append(lesson)
        print(f"试验 {trial+1}: 记录经验 → {lesson.lesson}")

    return output
```

### 机制3：LLM-as-Judge（LLM 评判）

**原理**：使用独立的 LLM 作为"评委"，评估 Agent 输出质量。

```python
class JudgeVerdict(BaseModel):
    """评判结果"""
    criteria_scores: dict = Field(
        description="各维度评分，如 {'完整性': 8, '准确性': 7}"
    )
    overall_score: float = Field(description="综合评分 0-10")
    pass_threshold: bool = Field(description="是否通过质量阈值")
    feedback: str = Field(description="详细反馈")

judge_llm = ChatOpenAI(model="gpt-4o", temperature=0)
judge = judge_llm.with_structured_output(JudgeVerdict)

def llm_judge(task: str, output: str, criteria: List[str]) -> JudgeVerdict:
    """LLM 评判 Agent 输出"""
    return judge.invoke(f"""
    你是一个公正的评委。请根据以下标准评估输出质量。

    任务：{task}
    输出：{output}

    评估标准：
    {chr(10).join(f'- {c}' for c in criteria)}

    每个标准打分 0-10，然后给出综合评分。
    综合评分 >= 7 视为通过。
    """)

# 使用示例
verdict = llm_judge(
    task="对比 Python 和 Rust 的 Web 开发优劣",
    output=agent_output,
    criteria=["完整性", "准确性", "平衡性", "实用性"]
)
```

## 三种机制对比

| 维度 | Reflection | Reflexion | LLM-as-Judge |
|------|-----------|-----------|--------------|
| **评估方式** | 自我评估 | 自我 + 外部 | 独立评委 |
| **记忆能力** | 无 | 有（经验库） | 无 |
| **客观性** | 低（自己评自己） | 中等 | 高（独立评估） |
| **LLM 调用** | 1-2 次/轮 | 2-3 次/轮 | 1 次/轮 |
| **适用场景** | 简单任务改进 | 重复性任务 | 关键质量把关 |
| **典型用法** | 代码自检 | 学习类任务 | 最终输出审核 |

## 执行监控的实现

### 状态追踪器

```python
from datetime import datetime
from typing import Optional

class StepExecution(BaseModel):
    """步骤执行记录"""
    step_id: int
    task: str
    status: str  # pending, running, completed, failed
    start_time: Optional[datetime] = None
    end_time: Optional[datetime] = None
    result: Optional[str] = None
    error: Optional[str] = None
    retries: int = 0

class ExecutionMonitor:
    """执行监控器"""

    def __init__(self, max_retries: int = 2):
        self.executions: List[StepExecution] = []
        self.max_retries = max_retries

    def start_step(self, step_id: int, task: str):
        execution = StepExecution(
            step_id=step_id,
            task=task,
            status="running",
            start_time=datetime.now(),
        )
        self.executions.append(execution)
        return execution

    def complete_step(self, step_id: int, result: str):
        execution = self._get(step_id)
        execution.status = "completed"
        execution.end_time = datetime.now()
        execution.result = result

    def fail_step(self, step_id: int, error: str) -> bool:
        """标记步骤失败，返回是否可以重试"""
        execution = self._get(step_id)
        execution.retries += 1
        if execution.retries <= self.max_retries:
            execution.status = "running"
            return True  # 可以重试
        execution.status = "failed"
        execution.error = error
        return False  # 不能再重试

    def get_summary(self) -> dict:
        """获取执行摘要"""
        return {
            "total": len(self.executions),
            "completed": sum(1 for e in self.executions if e.status == "completed"),
            "failed": sum(1 for e in self.executions if e.status == "failed"),
            "running": sum(1 for e in self.executions if e.status == "running"),
        }

    def _get(self, step_id: int) -> StepExecution:
        return next(e for e in self.executions if e.step_id == step_id)
```

## Self-Critique 模式

**自我批评**是一种特殊的反思模式，Agent 主动挑战自己的输出。

```python
class SelfCritique(BaseModel):
    """自我批评结果"""
    original_claim: str = Field(description="原始论点")
    counter_argument: str = Field(description="反面论点")
    weakness: str = Field(description="论证弱点")
    revised_claim: str = Field(description="修正后的论点")

critic = llm.with_structured_output(SelfCritique)

def self_critique(claim: str) -> SelfCritique:
    """自我批评：主动挑战自己的结论"""
    return critic.invoke(f"""
    你是一个严厉的批评者。请对以下论点提出反面意见。

    论点：{claim}

    要求：
    1. 提出最强的反面论点
    2. 找出论证中最大的弱点
    3. 给出更平衡的修正论点
    """)
```

## 常见陷阱

| 陷阱 | 说明 | 解决方案 |
|------|------|---------|
| **反思循环** | 反思后改进，改进后又发现新问题 | 限制最大反思次数（2-3次） |
| **过度自我肯定** | LLM 倾向于认为自己的输出是好的 | 用独立 LLM 或提高评估标准 |
| **反思成本过高** | 每次都用大模型反思，成本翻倍 | 只在关键输出时反思 |
| **忽略外部信号** | 只做内部反思，不看外部反馈 | 结合 Reflexion 的外部验证 |

---

## 关键要点

1. **三种机制**：Reflection（自我评估）、Reflexion（+外部验证+记忆）、LLM-as-Judge（独立评委）
2. **Reflection 最轻量**：适合快速自检，1-2 次 LLM 调用
3. **Reflexion 能学习**：通过经验记忆避免重复犯错
4. **LLM-as-Judge 最客观**：独立评估，适合关键质量把关
5. **执行监控是基础设施**：追踪状态、管理重试、生成摘要
6. **限制反思次数**：2-3 次最佳，避免无限循环
