# 一句话总结

**Milvus 2.6 的 Embedding Functions 让数据插入与检索从"手动向量化 + 复杂流程"变成"插入文本 + 自动检索"，3 步完成 RAG 核心流程。**

---

## 核心要点

- **Embedding Functions**：自动文本向量化，无需外部 API
- **Data-in, Data-out**：输入文本，输出文本，向量化透明
- **简化流程**：5 步 → 3 步（创建 Collection → 插入文本 → 检索）
- **多提供商**：OpenAI、Cohere、Bedrock、Voyage、Jina 等
- **混合检索**：向量检索 + BM25 + 过滤表达式
- **生产就绪**：批量插入、性能优化、错误处理

---

## 学习路径

1. **理解 Embedding Functions**：自动向量化的核心机制
2. **掌握数据插入**：Insert、Upsert、Bulk Insert
3. **掌握相似度检索**：Search、Query、Hybrid Search
4. **实战应用**：基础检索 → 批量优化 → 混合检索 → RAG 系统

---

## 关键对比

| 维度 | 传统方式 | 2026 Milvus 2.6 |
|------|----------|------------------|
| **向量化** | 手动调用外部 API | Embedding Functions 自动 |
| **流程步骤** | 5-6 步 | 3 步 |
| **代码复杂度** | 100-150 行 | 30-50 行 |
| **维护成本** | 管理外部服务 | 统一管理 |
| **性能** | 多次网络调用 | 内部优化 |

---

**最终目标：** 用最少的代码，最简单的流程，构建生产级 RAG 检索系统。
