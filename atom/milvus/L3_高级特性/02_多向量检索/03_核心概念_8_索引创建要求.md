# 核心概念 8：索引创建要求

> **核心约束**：在多向量检索场景中，所有向量字段都必须创建索引，否则 Collection 无法加载。

---

## 1. 核心约束说明

### 1.1 强制索引要求

在 Milvus 多向量检索中，存在一个**硬性约束**：

```
所有向量字段都必须创建索引，才能成功加载 Collection
```

**来源**：源码分析 `hybridsearch_test.go:98-148`

```go
// 尝试在未创建索引的情况下加载 Collection
loadStatus, err := c.MilvusClient.LoadCollection(ctx, &milvuspb.LoadCollectionRequest{
    DbName:         dbName,
    CollectionName: collectionName,
})
s.NoError(err)
s.Error(merr.Error(loadStatus))  // 预期失败 - 因为向量字段没有索引
```

### 1.2 部分索引的问题

如果只为部分向量字段创建索引，会导致：

- **LoadCollection 失败**：无法将 Collection 加载到内存
- **检索不可用**：无法执行任何检索操作
- **错误信息不明确**：可能只提示"加载失败"，不会明确指出缺少索引

**错误示例**：

```python
# 假设有 3 个向量字段：dense_vector, sparse_vector, binary_vector
# 只为 dense_vector 创建索引
client.create_index(
    collection_name="multi_vector_collection",
    field_name="dense_vector",
    index_params={"index_type": "IVF_FLAT", "metric_type": "L2", "params": {"nlist": 128}}
)

# 尝试加载 Collection - 会失败
client.load_collection("multi_vector_collection")  # ❌ 失败：sparse_vector 和 binary_vector 没有索引
```

### 1.3 正确的索引创建流程

```
1. 定义 Schema（包含多个向量字段）
2. 创建 Collection
3. 为所有向量字段创建索引（必须）
4. 插入数据
5. 加载 Collection（此时才能成功）
6. 执行混合检索
```

## 2. 不同向量类型的索引要求

### 2.1 FloatVector（稠密向量）

**支持的索引类型**：
- `FLAT`：暴力搜索，精度最高
- `IVF_FLAT`：倒排索引，平衡精度和速度
- `IVF_SQ8`：标量量化，节省内存
- `IVF_PQ`：乘积量化，极致压缩
- `HNSW`：图索引，速度最快
- `DISKANN`：磁盘索引，超大规模

**推荐配置**：

```python
# 小规模数据（< 100万）
float_index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {
        "M": 16,
        "efConstruction": 200
    }
}

# 大规模数据（> 100万）
float_index_params = {
    "index_type": "IVF_FLAT",
    "metric_type": "L2",
    "params": {
        "nlist": 1024
    }
}
```

### 2.2 SparseFloatVector（稀疏向量）

**支持的索引类型**：
- `SPARSE_INVERTED_INDEX`：稀疏倒排索引（推荐）
- `SPARSE_WAND`：WAND 算法优化

**推荐配置**：

```python
sparse_index_params = {
    "index_type": "SPARSE_INVERTED_INDEX",
    "metric_type": "IP",  # 稀疏向量通常使用内积
    "params": {
        "drop_ratio_build": 0.2  # 构建时丢弃低权重项
    }
}
```

### 2.3 BinaryVector（二值向量）

**支持的索引类型**：
- `BIN_FLAT`：二值暴力搜索
- `BIN_IVF_FLAT`：二值倒排索引

**推荐配置**：

```python
binary_index_params = {
    "index_type": "BIN_IVF_FLAT",
    "metric_type": "JACCARD",  # 或 HAMMING
    "params": {
        "nlist": 128
    }
}
```

### 2.4 BFloat16Vector（半精度向量）

**支持的索引类型**：
- 与 FloatVector 相同，但内存占用减半

**推荐配置**：

```python
bfloat16_index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {
        "M": 16,
        "efConstruction": 200
    }
}
```

## 3. 完整代码示例

### 3.1 Python SDK 示例

```python
from pymilvus import (
    MilvusClient,
    DataType,
    CollectionSchema,
    FieldSchema,
)

# 1. 连接 Milvus
client = MilvusClient(uri="http://localhost:19530")

# 2. 定义 Schema（包含多个向量字段）
fields = [
    FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
    FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=1000),
    FieldSchema(name="dense_vector", dtype=DataType.FLOAT_VECTOR, dim=768),
    FieldSchema(name="sparse_vector", dtype=DataType.SPARSE_FLOAT_VECTOR),
    FieldSchema(name="binary_vector", dtype=DataType.BINARY_VECTOR, dim=128),
]

schema = CollectionSchema(
    fields=fields,
    description="Multi-vector collection with proper indexing"
)

# 3. 创建 Collection
collection_name = "multi_vector_indexed"
client.create_collection(
    collection_name=collection_name,
    schema=schema
)

# 4. 为所有向量字段创建索引（必须）
print("Creating indexes for all vector fields...")

# 4.1 为稠密向量创建索引
client.create_index(
    collection_name=collection_name,
    field_name="dense_vector",
    index_params={
        "index_type": "HNSW",
        "metric_type": "L2",
        "params": {
            "M": 16,
            "efConstruction": 200
        }
    }
)
print("✓ Dense vector index created")

# 4.2 为稀疏向量创建索引
client.create_index(
    collection_name=collection_name,
    field_name="sparse_vector",
    index_params={
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "IP",
        "params": {
            "drop_ratio_build": 0.2
        }
    }
)
print("✓ Sparse vector index created")

# 4.3 为二值向量创建索引
client.create_index(
    collection_name=collection_name,
    field_name="binary_vector",
    index_params={
        "index_type": "BIN_IVF_FLAT",
        "metric_type": "JACCARD",
        "params": {
            "nlist": 128
        }
    }
)
print("✓ Binary vector index created")

# 5. 插入数据
import numpy as np
from scipy.sparse import csr_matrix

data = [
    {
        "text": f"Document {i}",
        "dense_vector": np.random.rand(768).tolist(),
        "sparse_vector": csr_matrix(np.random.rand(1, 1000)),
        "binary_vector": bytes(np.random.randint(0, 2, 16))  # 128 bits = 16 bytes
    }
    for i in range(1000)
]

client.insert(collection_name=collection_name, data=data)
print(f"✓ Inserted {len(data)} entities")

# 6. 加载 Collection（现在可以成功）
client.load_collection(collection_name=collection_name)
print("✓ Collection loaded successfully")

# 7. 验证索引状态
indexes = client.list_indexes(collection_name=collection_name)
print(f"\nIndexes created: {indexes}")

# 8. 执行混合检索
from pymilvus import AnnSearchRequest, RRFRanker

# 定义检索请求
dense_req = AnnSearchRequest(
    data=[np.random.rand(768).tolist()],
    anns_field="dense_vector",
    param={"metric_type": "L2", "params": {"ef": 100}},
    limit=10
)

sparse_req = AnnSearchRequest(
    data=[csr_matrix(np.random.rand(1, 1000))],
    anns_field="sparse_vector",
    param={"metric_type": "IP"},
    limit=10
)

# 执行混合检索
results = client.hybrid_search(
    collection_name=collection_name,
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(),
    limit=5,
    output_fields=["text"]
)

print(f"\n✓ Hybrid search completed, found {len(results[0])} results")
```

### 3.2 Go SDK 示例

```go
package main

import (
    "context"
    "fmt"
    "log"

    "github.com/milvus-io/milvus-sdk-go/v2/client"
    "github.com/milvus-io/milvus-sdk-go/v2/entity"
)

func main() {
    ctx := context.Background()

    // 1. 连接 Milvus
    c, err := client.NewClient(ctx, client.Config{
        Address: "localhost:19530",
    })
    if err != nil {
        log.Fatal("failed to connect to Milvus:", err)
    }
    defer c.Close()

    // 2. 定义 Schema
    schema := &entity.Schema{
        CollectionName: "multi_vector_indexed_go",
        Description:    "Multi-vector collection with proper indexing",
        Fields: []*entity.Field{
            {
                Name:       "id",
                DataType:   entity.FieldTypeInt64,
                PrimaryKey: true,
                AutoID:     true,
            },
            {
                Name:     "text",
                DataType: entity.FieldTypeVarChar,
                TypeParams: map[string]string{
                    "max_length": "1000",
                },
            },
            {
                Name:     "dense_vector",
                DataType: entity.FieldTypeFloatVector,
                TypeParams: map[string]string{
                    "dim": "768",
                },
            },
            {
                Name:     "sparse_vector",
                DataType: entity.FieldTypeSparseVector,
            },
            {
                Name:     "binary_vector",
                DataType: entity.FieldTypeBinaryVector,
                TypeParams: map[string]string{
                    "dim": "128",
                },
            },
        },
    }

    // 3. 创建 Collection
    err = c.CreateCollection(ctx, schema, 2)
    if err != nil {
        log.Fatal("failed to create collection:", err)
    }
    fmt.Println("✓ Collection created")

    // 4. 为所有向量字段创建索引
    collectionName := "multi_vector_indexed_go"

    // 4.1 稠密向量索引
    denseIdx, err := entity.NewIndexHNSW(entity.L2, 16, 200)
    if err != nil {
        log.Fatal("failed to create dense index:", err)
    }
    err = c.CreateIndex(ctx, collectionName, "dense_vector", denseIdx, false)
    if err != nil {
        log.Fatal("failed to create dense index:", err)
    }
    fmt.Println("✓ Dense vector index created")

    // 4.2 稀疏向量索引
    sparseIdx, err := entity.NewIndexSparseInverted(entity.IP, 0.2)
    if err != nil {
        log.Fatal("failed to create sparse index:", err)
    }
    err = c.CreateIndex(ctx, collectionName, "sparse_vector", sparseIdx, false)
    if err != nil {
        log.Fatal("failed to create sparse index:", err)
    }
    fmt.Println("✓ Sparse vector index created")

    // 4.3 二值向量索引
    binaryIdx, err := entity.NewIndexBinIvfFlat(entity.JACCARD, 128)
    if err != nil {
        log.Fatal("failed to create binary index:", err)
    }
    err = c.CreateIndex(ctx, collectionName, "binary_vector", binaryIdx, false)
    if err != nil {
        log.Fatal("failed to create binary index:", err)
    }
    fmt.Println("✓ Binary vector index created")

    // 5. 加载 Collection
    err = c.LoadCollection(ctx, collectionName, false)
    if err != nil {
        log.Fatal("failed to load collection:", err)
    }
    fmt.Println("✓ Collection loaded successfully")

    // 6. 验证索引状态
    indexes, err := c.DescribeIndex(ctx, collectionName, "dense_vector")
    if err != nil {
        log.Fatal("failed to describe index:", err)
    }
    fmt.Printf("\nDense vector index: %+v\n", indexes)
}
```

## 4. 索引创建最佳实践

### 4.1 索引创建顺序

```python
# 推荐顺序：
# 1. 创建 Collection
# 2. 为所有向量字段创建索引
# 3. 插入数据
# 4. 加载 Collection

# ❌ 错误顺序：
# 1. 创建 Collection
# 2. 插入数据
# 3. 创建索引（数据已存在，索引构建会很慢）
# 4. 加载 Collection
```

### 4.2 索引参数选择

**小规模数据（< 100万）**：
```python
# 优先考虑精度和速度
index_params = {
    "index_type": "HNSW",
    "metric_type": "L2",
    "params": {"M": 16, "efConstruction": 200}
}
```

**中等规模数据（100万 - 1000万）**：
```python
# 平衡精度、速度和内存
index_params = {
    "index_type": "IVF_FLAT",
    "metric_type": "L2",
    "params": {"nlist": 1024}
}
```

**大规模数据（> 1000万）**：
```python
# 优先考虑内存和速度
index_params = {
    "index_type": "IVF_PQ",
    "metric_type": "L2",
    "params": {"nlist": 2048, "m": 8, "nbits": 8}
}
```

### 4.3 索引构建监控

```python
import time

def wait_for_index_building(client, collection_name, field_name, timeout=300):
    """等待索引构建完成"""
    start_time = time.time()

    while time.time() - start_time < timeout:
        # 检查索引状态
        index_info = client.describe_index(
            collection_name=collection_name,
            field_name=field_name
        )

        if index_info["state"] == "Finished":
            print(f"✓ Index on {field_name} built successfully")
            return True

        print(f"Building index on {field_name}... ({int(time.time() - start_time)}s)")
        time.sleep(5)

    raise TimeoutError(f"Index building timeout for {field_name}")

# 使用示例
client.create_index(collection_name, "dense_vector", dense_index_params)
wait_for_index_building(client, collection_name, "dense_vector")
```

## 5. 常见错误和解决方案

### 5.1 错误：LoadCollection 失败

**错误信息**：
```
failed to load collection: index not found for field 'sparse_vector'
```

**原因**：部分向量字段没有创建索引

**解决方案**：
```python
# 检查所有向量字段是否都有索引
vector_fields = ["dense_vector", "sparse_vector", "binary_vector"]

for field in vector_fields:
    try:
        index_info = client.describe_index(collection_name, field)
        print(f"✓ {field}: {index_info['index_type']}")
    except Exception as e:
        print(f"✗ {field}: No index found")
        # 创建缺失的索引
        client.create_index(collection_name, field, get_index_params(field))
```

### 5.2 错误：索引类型不匹配

**错误信息**：
```
invalid index type 'IVF_FLAT' for field type 'SPARSE_FLOAT_VECTOR'
```

**原因**：为稀疏向量使用了稠密向量的索引类型

**解决方案**：
```python
# 根据向量类型选择正确的索引
def get_index_params(field_name, field_type):
    if field_type == DataType.FLOAT_VECTOR:
        return {
            "index_type": "HNSW",
            "metric_type": "L2",
            "params": {"M": 16, "efConstruction": 200}
        }
    elif field_type == DataType.SPARSE_FLOAT_VECTOR:
        return {
            "index_type": "SPARSE_INVERTED_INDEX",
            "metric_type": "IP",
            "params": {"drop_ratio_build": 0.2}
        }
    elif field_type == DataType.BINARY_VECTOR:
        return {
            "index_type": "BIN_IVF_FLAT",
            "metric_type": "JACCARD",
            "params": {"nlist": 128}
        }
```

### 5.3 错误：度量类型不匹配

**错误信息**：
```
invalid metric type 'L2' for sparse vector
```

**原因**：稀疏向量通常使用 IP（内积），不支持 L2

**解决方案**：
```python
# 稀疏向量必须使用 IP
sparse_index_params = {
    "index_type": "SPARSE_INVERTED_INDEX",
    "metric_type": "IP",  # ✓ 正确
    # "metric_type": "L2",  # ✗ 错误
    "params": {"drop_ratio_build": 0.2}
}
```

## 6. 索引管理工具函数

### 6.1 批量创建索引

```python
def create_all_indexes(client, collection_name, index_configs):
    """
    为 Collection 的所有向量字段批量创建索引

    Args:
        client: MilvusClient 实例
        collection_name: Collection 名称
        index_configs: 索引配置字典
            {
                "field_name": {
                    "index_type": "HNSW",
                    "metric_type": "L2",
                    "params": {...}
                }
            }
    """
    success_count = 0
    failed_fields = []

    for field_name, index_params in index_configs.items():
        try:
            client.create_index(
                collection_name=collection_name,
                field_name=field_name,
                index_params=index_params
            )
            print(f"✓ Index created for {field_name}")
            success_count += 1
        except Exception as e:
            print(f"✗ Failed to create index for {field_name}: {e}")
            failed_fields.append(field_name)

    print(f"\nSummary: {success_count}/{len(index_configs)} indexes created")

    if failed_fields:
        raise Exception(f"Failed to create indexes for: {failed_fields}")

    return True

# 使用示例
index_configs = {
    "dense_vector": {
        "index_type": "HNSW",
        "metric_type": "L2",
        "params": {"M": 16, "efConstruction": 200}
    },
    "sparse_vector": {
        "index_type": "SPARSE_INVERTED_INDEX",
        "metric_type": "IP",
        "params": {"drop_ratio_build": 0.2}
    },
    "binary_vector": {
        "index_type": "BIN_IVF_FLAT",
        "metric_type": "JACCARD",
        "params": {"nlist": 128}
    }
}

create_all_indexes(client, "multi_vector_collection", index_configs)
```

### 6.2 验证索引完整性

```python
def verify_all_indexes(client, collection_name):
    """
    验证 Collection 的所有向量字段是否都有索引

    Returns:
        bool: 所有向量字段都有索引返回 True，否则返回 False
    """
    # 获取 Schema
    schema = client.describe_collection(collection_name)

    # 找出所有向量字段
    vector_fields = []
    for field in schema["fields"]:
        if field["type"] in [
            DataType.FLOAT_VECTOR,
            DataType.BINARY_VECTOR,
            DataType.SPARSE_FLOAT_VECTOR,
            DataType.BFLOAT16_VECTOR
        ]:
            vector_fields.append(field["name"])

    print(f"Found {len(vector_fields)} vector fields: {vector_fields}")

    # 检查每个向量字段是否有索引
    missing_indexes = []
    for field_name in vector_fields:
        try:
            index_info = client.describe_index(collection_name, field_name)
            print(f"✓ {field_name}: {index_info['index_type']}")
        except Exception:
            print(f"✗ {field_name}: No index")
            missing_indexes.append(field_name)

    if missing_indexes:
        print(f"\n⚠️  Missing indexes for: {missing_indexes}")
        return False

    print("\n✓ All vector fields have indexes")
    return True

# 使用示例
if not verify_all_indexes(client, "multi_vector_collection"):
    print("Please create missing indexes before loading the collection")
```

## 7. 性能优化建议

### 7.1 索引构建性能

```python
# 1. 先创建索引，再插入数据（推荐）
client.create_index(collection_name, "dense_vector", index_params)
client.insert(collection_name, data)  # 数据插入时自动更新索引

# 2. 先插入数据，再创建索引（不推荐）
client.insert(collection_name, data)
client.create_index(collection_name, "dense_vector", index_params)  # 需要扫描所有数据
```

### 7.2 内存优化

```python
# 使用量化索引减少内存占用
quantized_index_params = {
    "index_type": "IVF_SQ8",  # 标量量化
    "metric_type": "L2",
    "params": {"nlist": 1024}
}

# 或使用乘积量化
pq_index_params = {
    "index_type": "IVF_PQ",  # 乘积量化
    "metric_type": "L2",
    "params": {
        "nlist": 1024,
        "m": 8,      # 子向量数量
        "nbits": 8   # 每个子向量的位数
    }
}
```

### 7.3 检索性能

```python
# 调整检索参数以平衡精度和速度
search_params = {
    "metric_type": "L2",

# 或对于 IVF 索引

    "metric_type": "L2",
```

## 8. 总结

### 8.1 核心要点

1. **强制要求**：所有向量字段都必须创建索引
2. **类型匹配**：不同向量类型使用不同的索引类型
3. **度量选择**：稀疏向量使用 IP，稠密向量使用 L2/IP/COSINE
4. **创建顺序**：先创建索引，再插入数据，最后加载 Collection

### 8.2 索引类型速查表

| 向量类型 | 推荐索引 | 度量类型 | 适用场景 |
|---------|---------|---------|---------|
| FloatVector | HNSW | L2/IP/COSINE | 小规模高精度 |
| FloatVector | IVF_FLAT | L2/IP/COSINE | 中等规模平衡 |
| FloatVector | IVF_PQ | L2/IP/COSINE | 大规模内存优化 |
| SparseFloatVector | SPARSE_INVERTED_INDEX | IP | BM25/稀疏向量 |
| BinaryVector | BIN_IVF_FLAT | JACCARD/HAMMING | 二值特征 |
| BFloat16Vector | HNSW | L2/IP/COSINE | 半精度向量 |

### 8.3 检查清单

在执行混合检索前，确保：

- [ ] 所有向量字段都已创建索引
- [ ] 索引类型与向量类型匹配
- [ ] 度量类型正确配置
- [ ] 索引构建已完成
- [ ] Collection 已成功加载
- [ ] 验证索引状态正常
